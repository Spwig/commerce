---
title: 翻訳サービス
---

翻訳サービスは、ストアの商品説明、ページ内容、ブログ投稿、SEOフィールド、その他の販売業者のコンテンツに対してAIで翻訳を提供します。翻訳はローカルのサーバー上または外部プロバイダを通じて実行されるため、コンテンツはプライベートのまま、翻訳は数秒で完了します。

![Language management](/static/core/admin/img/help/translation-service/language-management.webp)

## How It Works

1. ストアで**言語を有効化**します（例: 英語、ドイツ語、日本語）
2. コンテンツ（商品、ページ、ブログ投稿）を作成または編集する際は、デフォルトの言語で作成します
3. 任意の翻訳可能なフィールドで**Translate**をクリックして、アクティブな言語にAIによる翻訳を生成します
4. 翻訳はオリジナルのコンテンツと一緒に保存され、訪問者の言語に基づいて自動的に提供されます

## Managing Languages

**Settings > Languages**に移動してストアの言語を管理します。

### Language Dashboard

ダッシュボードには以下が表示されます：
- **Total Languages** — システムで利用可能なすべての言語（100以上）
- **Active Languages** — 現在ストアで有効になっている言語
- **Model Coverage** — インストールされた翻訳モデルがサポートしている言語数

### Activating Languages

1. **Available Languages**列で言語を探します
2. 言語をクリックして**Active Languages**列に移動させます
3. 言語はすぐに翻訳に使用可能になり、ストアの言語切り替え機能に表示されます

### Default Language

1つの言語が**デフォルト**としてマークされています。これは：
- コンテンツを作成する際の言語
- 翻訳が存在しない場合の代替
- 訪問者が選択していない場合に表示される言語

## Translation Models

Spwigには、ローカルのAI翻訳エンジンが含まれており、これは完全にあなたのサーバー上で実行されます — 外部サービスにデータを送信することはありません。

### Available Models

| Model | Languages | Speed | Quality |
|-------|-----------|-------|---------|
| **M2M100-418M** | 100 | Fast | 一般的な言語ペアに適しています |
| **M2M100-1.2B** | 100 | Moderate | 質が高いが、リソース使用量が高め |
| **NLLB-200** | 200+ | Moderate | 最も広範なカバレッジ、レアな言語も含む |

### Model Selection

言語管理ページは、インストールされているモデルとその言語カバレッジを表示します。モデルはCTranslate2を使用してローカルサービスとして効率的な推論を実行します。

## External Providers

クラウドベースの翻訳を好む、または特定の言語品質が必要なストアのために、Spwigは外部の翻訳プロバイダをサポートしています。

| Provider | Description |
|----------|-------------|
| **DeepL** | ヨーロッパおよびアジアの言語に対するプレミアム品質の翻訳 |
| **Google Translate** | ニューラルマシン翻訳を用いた広範な言語カバレッジ |
| **Azure Translator** | Microsoftのニューラル翻訳サービス |
| **AWS Translate** | Amazonの機械翻訳、カスタム用語のサポートあり |

### Connecting a Provider

1. **Settings > Translation Providers**に移動します
2. プロバイダを選択し、APIキーを入力します
3. プロバイダを翻訳エンジンとしての優先設定にします
4. 外部プロバイダがローカルモデルの代わりに翻訳を実行します

外部プロバイダとローカルモデルを併用できます — 例えば、ヨーロッパ語をDeepLで翻訳し、他のすべてをローカルモデルで翻訳します。

## Translating Content

### Field-Level Translation

翻訳可能なフィールド（商品名、説明、SEOタイトルなど）は、フィールドの隣に**翻訳ボタン**を表示します。クリックして：

1. **すべてのアクティブな言語に翻訳** — 一度にすべてのアクティブな言語に翻訳を生成します
2. **特定の言語に翻訳** — 翻訳する個々の言語を選択します

翻訳はエディタの言語タブに表示されます。機械翻訳を確認し、必要に応じて手動で編集できます。

### Bulk Translation Jobs

大量のコンテンツを翻訳する場合は、**翻訳ジョブ**を使用してください：

1. **Settings > Translation Jobs**に移動します
2. 新しいジョブを作成し、以下を選択します：
   - **コンテンツタイプ** — 商品、ページ、ブログ投稿、カテゴリなど
   - **ソース言語** — 翻訳元の言語
   - **ターゲット言語** — 翻訳先の1つ以上の言語
   - **範囲** — すべてのコンテンツ、または未翻訳のフィールドのみ
3. ジョブを送信 — タスクキューを介してバックグラウンドで実行されます
4. ジョブリストで進行状況を確認します（キューイング → 処理中 → 完了）

大量のジョブは、新しい言語を有効化し、カタログ全体を一度に翻訳したいときに役立ちます。

## Translation Management

### Reviewing Translations

各翻訳フィールドは以下を追跡します：
- **翻訳ステータス** — フィールドが機械翻訳済み、手動で編集済み、または存在しないか
- **ロックステータス** — ロックされた翻訳は、将来的な機械翻訳によって上書きされません
- **最後に翻訳された** — 翻訳が最後に生成または編集された時刻

### Locking Translations

機械翻訳を手動で編集して改善した場合、**ロック**を設定して、バッチ翻訳が実行されるたびに上書きされないようにします。ロックされたフィールドは、自動翻訳の対象外になります。

### Translation Coverage

カバレッジトラッカーは、各言語に対してコンテンツの翻訳がどの割合であるかを表示します。**Settings > Languages**に移動して以下を確認できます：
- 言語ごとの完了率
- どのコンテンツタイプにギャップがあるか
- まだ翻訳が必要なフィールド

## UI Translation Overrides

商品やページのコンテンツだけでなく、**フロントエンドインターフェースの文字列**（ボタン、ラベル、メッセージ、その他のUIテキスト）の翻訳をカスタマイズすることもできます。

**Settings > UI Overrides**に移動して：
1. 特定の文字列（例: "Add to Cart"）を検索します
2. 各言語で希望の翻訳を入力します
3. 保存 — オーバーライドはすぐに有効になります

カスタマイズ可能なフロントエンド文字列は約300あります。オーバーライドはデフォルトの翻訳よりも優先されます。

## Tips

- 顧客が実際に使用している言語のみを最初に有効化してください — いつでも後で追加できます。
- 日常的な翻訳には**ローカルAIモデル**を使用してください — これは高速、プライベート、かつ1つの翻訳あたりのコストがありません。
- キーのヨーロッパ語で最高品質の翻訳が必要な場合は、**DeepL**を検討してください — 一般的なモデルよりも自然な翻訳を一貫して生成します。
- **機械翻訳を常に確認**してください — 商品名、ブランド用語、マーケティングコピーなど。AIは技術的なコンテンツをうまく処理しますが、創造的なテキストのニュアンスを欠く可能性があります。
- **ロック**を設定して、手動で改善した翻訳を保護してください — バッチ翻訳が実行されるたびに上書きされないようにします。
- 新しい言語を有効化する際には、**バッチ翻訳ジョブ**を使用してカタログ全体を一度に翻訳するのではなく、商品ごとに個別に翻訳する必要はありません。
- **UIオーバーライド**をカスタマイズして、ブランドのトーンに合わせてください — たとえば、"Add to Cart"を"Buy Now"に変更して、それがストアに合っている場合など。
