---
title: บริการแปลภาษา
---

บริการแปลภาษาช่วยให้คุณแปลคำอธิบายสินค้า ข้อความหน้าเว็บ บล็อก ฟิลด์ SEO และเนื้อหาอื่นๆ ของร้านค้าโดยใช้ AI บริการแปลภาษาทำงานบนเซิร์ฟเวอร์ของคุณเองหรือผ่านผู้ให้บริการภายนอก ดังนั้นเนื้อหาของคุณจึงยังคงเป็นส่วนตัว และการแปลจะเกิดขึ้นทันทีภายในไม่กี่วินาที

![การจัดการภาษา](/static/core/admin/img/help/translation-service/language-management.webp)

## วิธีการทำงาน

1. คุณ **เปิดใช้งานภาษา** สำหรับร้านค้าของคุณ (เช่น ภาษาอังกฤษ เยอรมัน ญี่ปุ่น)
2. เมื่อคุณสร้างหรือแก้ไขเนื้อหา (สินค้า หน้าเว็บ บล็อก) คุณจะเขียนในภาษาเริ่มต้นของคุณ
3. คลิก **Translate** ที่ฟิลด์ใดๆ ที่สามารถแปลได้ เพื่อสร้างการแปลด้วย AI ไปยังภาษาที่เปิดใช้งานอยู่
4. การแปลจะถูกเก็บไว้ร่วมกับเนื้อหาต้นฉบับ และจะถูกส่งออกโดยอัตโนมัติตามภาษาของผู้เยี่ยมชม

## การจัดการภาษา

ไปที่ **Settings > Languages** เพื่อจัดการภาษาของร้านค้าของคุณ

### แดชบอร์ดภาษา

แดชบอร์ดแสดง:
- **Total Languages** — ทั้งหมดภาษาที่มีอยู่ในระบบ (มากกว่า 100 ภาษา)
- **Active Languages** — ภาษาที่เปิดใช้งานสำหรับร้านค้าของคุณในขณะนี้
- **Model Coverage** — จำนวนภาษาที่โมเดลแปลภาษาที่ติดตั้งไว้รองรับ

### การเปิดใช้งานภาษา

1. ค้นหาภาษาในคอลัมน์ **Available Languages**
2. คลิกที่ภาษาเพื่อเคลื่อนย้ายไปยังคอลัมน์ **Active Languages**
3. ภาษาจะพร้อมใช้งานสำหรับการแปลทันที และจะปรากฏในตัวเลือกเปลี่ยนภาษาของร้านค้าของคุณ

### ภาษาเริ่มต้น

หนึ่งภาษาจะถูกตั้งค่าเป็น **ภาษาเริ่มต้น** ซึ่งคือ:
- ภาษาที่คุณเขียนเนื้อหา
- ภาษาที่ใช้เป็นตัวสำรองเมื่อไม่มีการแปลสำหรับเนื้อหา
- ภาษาที่แสดงเมื่อผู้เยี่ยมชมยังไม่ได้เลือกภาษาของตนเอง

## โมเดลแปลภาษา

Spwig มีเครื่องมือแปลภาษา AI แบบท้องถิ่นที่ทำงานทั้งหมดบนเซิร์ฟเวอร์ของคุณ — ไม่มีข้อมูลใดถูกส่งไปยังบริการภายนอก

### โมเดลที่มีอยู่

| Model | Languages | Speed | Quality |
|-------|-----------|-------|---------|
| **M2M100-418M** | 100 | Fast | เหมาะสำหรับคู่ภาษาทั่วไป |
| **M2M100-1.2B** | 100 | Moderate | คุณภาพดีขึ้น ใช้ทรัพยากรมากขึ้น |
| **NLLB-200** | 200+ | Moderate | ครอบคลุมมากที่สุด รวมถึงภาษาที่พบได้น้อย |

### การเลือกโมเดล

หน้าจัดการภาษาจะแสดงว่าโมเดลใดที่ติดตั้งไว้และครอบคลุมภาษาใดบ้าง โมเดลทำงานเป็นบริการท้องถิ่นโดยใช้ CTranslate2 เพื่อให้การคำนวณมีประสิทธิภาพ

## ผู้ให้บริการแปลภาษาภายนอก

สำหรับร้านค้าที่ต้องการใช้บริการแปลภาษาแบบคลาวด์ หรือต้องการคุณภาพภาษาเฉพาะ  Spwig รองรับผู้ให้บริการแปลภาษาภายนอก

| Provider | Description |
|----------|-------------|
| **DeepL** | คุณภาพการแปลแบบพรีเมียมสำหรับภาษาในยุโรปและเอเชีย |
| **Google Translate** | ครอบคลุมภาษาหลากหลายโดยใช้การแปลเครื่องประสาท |
| **Azure Translator** | บริการแปลภาษาแบบเครื่องประสาทของ Microsoft |
| **AWS Translate** | การแปลภาษาของ Amazon พร้อมการสนับสนุนคำศัพท์เฉพาะ |

### การเชื่อมต่อกับผู้ให้บริการ

1. ไปที่ **Settings > Translation Providers**
2. เลือกผู้ให้บริการและใส่ API key ของคุณ
3. ตั้งค่าผู้ให้บริการเป็นเครื่องมือแปลภาษาที่ต้องการ
4. การแปลจะใช้ผู้ให้บริการภายนอกแทนโมเดลท้องถิ่น

คุณสามารถใช้ผู้ให้บริการแปลภาษาภายนอกร่วมกับโมเดลท้องถิ่นได้ — เช่น ใช้ DeepL สำหรับภาษาในยุโรป และใช้โมเดลท้องถิ่นสำหรับทุกอย่างอื่น

## การแปลเนื้อหา

### การแปลระดับฟิลด์

ฟิลด์ที่สามารถแปลได้ (ชื่อสินค้า คำอธิบาย หัวข้อ SEO ฯลฯ) จะแสดงปุ่ม **translate** อยู่ข้างๆ ฟิลด์ คลิกเพื่อ:

1. **แปลไปยังภาษาทั้งหมดที่เปิดใช้งาน** — สร้างการแปลสำหรับทุกภาษาที่เปิดใช้งานในครั้งเดียว
2. **แปลไปยังภาษาเฉพาะ** — เลือกภาษาแต่ละภาษาเพื่อแปล

การแปลจะปรากฏในแท็บภาษาของตัวแก้ไข คุณสามารถตรวจสอบและแก้ไขการแปลด้วยเครื่องมือได้ด้วยตนเอง

### การแปลแบบกลุ่ม

สำหรับเนื้อหาจำนวนมาก ให้ใช้ **งานแปลแบบกลุ่ม**:

1. ไปที่ **Settings > Translation Jobs**
2. สร้างงานใหม่โดยเลือก:
   - **ประเภทเนื้อหา** — สินค้า หน้าเว็บ บล็อก หมวดหมู่ ฯลฯ
   - **ภาษาต้นทาง** — ภาษาที่ต้องการแปลจาก
   - **ภาษาปลายทาง** — หนึ่งหรือหลายภาษาที่ต้องการแปลไป
   - **ขอบเขต** — ทั้งหมด หรือเฉพาะฟิลด์ที่ยังไม่ได้แปล
3. ส่งงาน — งานจะทำงานในพื้นหลังผ่านคิวงาน
4. ติดตามความคืบหน้าในรายการงาน (คิว → กำลังประมวลผล → สำเร็จ)

งานแปลแบบกลุ่มมีประโยชน์เมื่อคุณเปิดใช้งานภาษาใหม่ และต้องการแปลแคตตาล็อกทั้งหมดในครั้งเดียว

## การจัดการการแปล

### การตรวจสอบการแปล

แต่ละฟิลด์ที่แปลจะติดตาม:
- **สถานะการแปล** — ว่าฟิลด์นั้นถูกแปลโดยเครื่องมือ ถูกแก้ไขด้วยตนเอง หรือยังไม่มีการแปล
- **สถานะการล็อก** — การแปลที่ถูกล็อกจะไม่ถูกทับโดยการแปลของเครื่องมือในอนาคต
- **วันที่แปลล่าสุด** — เมื่อการแปลถูกสร้างหรือแก้ไขครั้งสุดท้าย

### การล็อกการแปล

หากคุณแก้ไขการแปลของเครื่องมือด้วยตนเองเพื่อปรับปรุงมัน **ล็อก** ฟิลด์เพื่อป้องกันไม่ให้ถูกทับเมื่อครั้งต่อไปที่มีการแปลแบบกลุ่ม การแปลที่ถูกล็อกจะถูกข้ามในระหว่างการแปลอัตโนมัติ

### การครอบคลุมการแปล

ตัวติดตามการครอบคลุมจะแสดงเปอร์เซ็นต์ของเนื้อหาที่ถูกแปลสำหรับแต่ละภาษา ไปที่ **Settings > Languages** เพื่อดู:
- เปอร์เซ็นต์ความสมบูรณ์ต่อภาษา
- ประเภทเนื้อหาใดที่ยังมีช่องว่าง
- ฟิลด์ใดที่ยังต้องการการแปล

## การปรับแต่งการแปล UI

นอกเหนือจากเนื้อหาสินค้าและหน้าเว็บ คุณยังสามารถปรับแต่งการแปลของ **ข้อความ UI ด้านหน้า** — ปุ่ม ป้ายกำกับ ข้อความ และข้อความ UI อื่นๆ ที่แสดงให้ผู้เยี่ยมชมเห็น

ไปที่ **Settings > UI Overrides** เพื่อ:
1. ค้นหาข้อความเฉพาะ (เช่น "Add to Cart")
2. ใส่การแปลที่คุณต้องการสำหรับแต่ละภาษา
3. บันทึก — การปรับแต่งจะมีผลทันที

มีข้อความด้านหน้าประมาณ 300 รายการที่สามารถปรับแต่งได้ การปรับแต่งจะมีความสำคัญเหนือกว่าการแปลเริ่มต้น

## เคล็ดลับ

- เริ่มต้นด้วยการเปิดใช้งานภาษาที่ลูกค้าของคุณใช้จริง — คุณสามารถเพิ่มภาษาเพิ่มเติมได้ตลอดเวลา
- ใช้ **โมเดล AI ท้องถิ่น** สำหรับการแปลประจำวัน — มันเร็ว เป็นส่วนตัว และไม่มีค่าใช้จ่ายต่อการแปล
- พิจารณา **DeepL** หากคุณต้องการคุณภาพสูงสุดสำหรับภาษาหลักในยุโรป — มันสร้างการแปลที่เป็นธรรมชาติมากกว่าโมเดลทั่วไป
- ตรวจสอบ **การแปลของเครื่องมือ** สำหรับชื่อสินค้า คำศัพท์แบรนด์ และข้อความการตลาด — AI จัดการเนื้อหาทางเทคนิคได้ดี แต่อาจพลาดความละเอียดอ่อนในข้อความสร้างสรรค์
- **ล็อก** การแปลใดๆ ที่คุณได้ปรับปรุงด้วยตนเองเพื่อป้องกันไม่ให้ถูกทับในระหว่างการแปลแบบกลุ่ม
- ใช้ **งานแปลแบบกลุ่ม** เมื่อเปิดใช้งานภาษาใหม่เพื่อแปลแคตตาล็อกทั้งหมดในครั้งเดียวแทนการแปลสินค้าแต่ละชิ้น
- ปรับแต่ง **การแปล UI** เพื่อให้สอดคล้องกับเสียงของแบรนด์คุณ — ตัวอย่างเช่น เปลี่ยน "Add to Cart" เป็น "Buy Now" หากเหมาะสมกับร้านค้าของคุณมากกว่า
