---
title: UI 번역 관리
---

UI 번역 페이지는 각 언어에서 프론트엔드 인터페이스 문자열—버튼, 라벨, 오류 메시지 및 기타 UI 텍스트—의 표시 방식을 사용자 정의할 수 있도록 합니다. 제품 또는 페이지 콘텐츠 번역과 달리 이러한 요소는 고객이 온라인 스토어 전반에서 볼 수 있는 고정된 인터페이스 요소입니다. 브랜드의 목소리와 일치하거나 특정 대상 독자에게 더 명확하게 보이도록 사용자 정의할 수 있습니다.

이 페이지는 모든 번역 가능한 UI 문자열을 표시하고 Spwig에서 제공하는 기본 번역을 사용자 정의할 수 있도록 합니다.

## UI 번역 이해

UI 번역은 스토어 인터페이스를 구성하는 텍스트 문자열입니다:

**UI 문자열 예시**:
- 버튼: "Add to Cart", "Checkout", "Search"
- 라벨: "Price", "Quantity", "Shipping Address"
- 메시지: "Item added to cart", "Order confirmed", "Invalid email address"
- 네비게이션: "Home", "Shop", "Contact Us"
- 폼 필드: "Email", "Password", "First Name"

Spwig은 모든 지원되는 언어에서 약 300개의 UI 문자열에 대한 기본 번역을 포함하고 있습니다. UI 번역 페이지는 이러한 기본값을 사용자 정의 번역으로 대체할 수 있도록 합니다.

## UI 번역을 사용자 정의하는 이유

**브랜드 목소리**: "Add to Cart"을 "Buy Now" 또는 "Get Yours"으로 변경하여 브랜드의 성격과 일치시킬 수 있습니다

**지역적 차이**: 특정 시장에 맞게 번역을 조정할 수 있습니다 (영국 영어 대 미국 영어, 유럽 스페인어 대 중남미 스페인어)

**명확성**: 기본 번역이 제품이나 대상 독자에게 의미가 없다면 더 명확한 텍스트로 대체할 수 있습니다

**산업별 용어**: 고객이 기대하는 용어를 사용하세요 (예: 서비스 기반 스토어의 경우 "Book Appointment" 대신 "Add to Cart")

## 문자열 검색

특정 UI 문자열을 찾으려면 검색 상자를 사용하세요:

**영어 텍스트로 검색**: "add to cart"을 입력하여 해당 버튼의 번역을 찾습니다

**번역으로 검색**: 어떤 언어의 텍스트를 입력하여 일치하는 번역을 찾습니다

**키로 검색**: 번역 키(예: `cart.add_item`)를 알고 있다면 직접 검색합니다

문자열을 입력할 때마다 페이지가 즉시 업데이트되어 일치하는 문자열만 표시됩니다.

## 번역 세부 정보 보기

각 UI 문자열은 다음과 같은 정보를 표시합니다:

**영어 원본 텍스트** - 기본 영어 버전(참고 기준)

**번역 키** - 코드에서 사용되는 내부 식별자(예: `cart.add_to_cart`)

**언어 열** - 활성화된 각 언어의 현재 번역

**대체 상태** - 번역을 사용자 정의했는지 여부(대체된 경우 강조 표시)

## 번역 대체 생성

UI 문자열의 번역을 사용자 정의하려면:

1. **문자열 찾기**: 검색을 사용하여 (예: "add to cart" 검색)
2. **사용자 정의할 언어 셀 클릭**: 
3. **팝업 편집기에서 사용자 정의 번역 입력**: 
4. **저장**: 사용자 정의 대체가 즉시 적용됩니다

기본 번역은 보존됩니다 - 사용자 정의 대체가 우선권을 가집니다.

## 기본값으로 복원

사용자 정의 대체를 제거하고 기본 번역으로 복원하려면:

1. **대체된 번역 클릭**: (이들은 강조 표시됨)
2. **편집기에서 "기본값으로 복원" 클릭**: 
3. **확인**: 기본 번역이 즉시 복원됩니다

다른 언어의 대체를 영향받지 않고 개별 언어 대체를 복원할 수 있습니다.

## 대체 상태로 필터링

필터 드롭다운을 사용하여 다음을 보세요:

**모든 문자열** - 시스템의 모든 UI 문자열 (~300개 총)

**대체된 문자열만** - 사용자가 사용자 정의 번역을 생성한 문자열

**기본값만** - Spwig의 기본 번역을 여전히 사용하는 문자열

이 기능은 사용자가 사용자 정의한 문자열을 검토하고 틈을 파악하는 데 도움이 됩니다.

## 일반적인 사용자 정의 예시

| 영어 기본값 | 사용자 정의 대체 | 사용 사례 |
|----------------|----------------|----------|
| Add to Cart | Buy Now | 더 직접적인 콜 투 액션 |
| Checkout | Secure Checkout | 보안 강조 |
| Search | Find Products | 전자상거래에 더 구체적 |
| Contact Us | Get in Touch | 더 친근한 어조 |
| Subscribe | Join Our Newsletter | 더 명확한 가치 제안 |

## 번역 검증

사용자 정의 번역을 입력할 때 다음을 검증하세요:

**UI 공간에 맞는 길이**: 번역은 영어보다 길거나 짧을 수 있습니다 (예: 독일어 단어는 종종 더 길음)

**의미 유지**: 번역에서 기능을 변경하지 마세요 (예: "Cancel" 버튼은 "Delete"가 되어서는 안 됨)

**일관된 용어**: 인터페이스 전반에서 반복되는 용어에 동일한 번역을 사용하세요

**적절한 공식성**: 대상 시장의 어조에 맞추세요 (공식적인 대 대화체)

## 다국어 일관성

여러 언어에 대해 문자열을 사용자 정의할 때:

1. **기본 언어부터 시작**: 기준을 설정하세요
2. **기타 언어를 사용자 정의**하여 동일한 의도를 유지하세요
3. **각 언어에서 테스트**하여 레이아웃과 의미를 확인하세요
4. **가능한 경우 원어민 사용자**를 사용하여 비영어 사용자 정의를 검토하세요

언어별로 불일치한 사용자 정의는 고객 경험에 혼란을 초래합니다.

## 대량 내보내기/불러오기

대규모 사용자 정의가 필요한 경우 내보내기/불러오기 워크플로를 고려하세요:

1. **현재 번역 내보내기**: JSON 또는 CSV로
2. **스프레드시트 또는 텍스트 편집기에서 편집**: 대량 변경이 더 쉬움
3. **업데이트된 번역 다시 불러오기**: 시스템으로

이 워크플로는 대규모 번역 프로젝트를 관리하는 데 사용할 수 있는 번역 작업 페이지를 통해 제공됩니다.

## 팁

- **사용자 정의하기 전에 검색**: 올바른 문자열을 편집하고 있는지 확인하세요. 유사한 문자열은 다른 목적을 위해 사용됩니다
- **저장 후 프론트엔드에서 테스트**: 사용자 정의 번역이 실제 UI에 올바르게 표시되는지 확인하세요
- **번역을 간결하게 유지**: 버튼과 라벨은 짧을수록 좋습니다
- **사용자 정의를 기록**: 특정 문자열을 사용자 정의한 이유를 기록하여 향후 참조용으로 보관하세요
- **일관된 용어 사용**: "Cart"을 "Basket"으로 사용자 정의했다면 관련된 모든 문자열에서 일관되게 사용하세요
- **모바일 레이아웃 고려**: 긴 번역은 작은 화면에서 줄바꿈하거나 잘릴 수 있으니 주의하세요
- **언어 업데이트 후 검토**: Spwig이 새로운 기본 번역을 추가할 때마다 검토하고 사용자 정의하여 일관성을 유지하세요

기억하세요: 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어는 보존 규칙에 따라 정확히 유지하세요.
