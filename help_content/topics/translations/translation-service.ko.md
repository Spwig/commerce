title: 번역 서비스
---

번역 서비스는 상점의 제품 설명, 페이지 콘텐츠, 블로그 게시물, SEO 필드 및 기타 판매자 콘텐츠에 대한 AI 기반 번역을 제공합니다. 번역은 서버 로컬에서 실행되거나 외부 제공업체를 통해 실행되며, 콘텐츠는 비공개 상태를 유지하고 번역은 몇 초 안에 이루어집니다.

![언어 관리](/static/core/admin/img/help/translation-service/language-management.webp)

## 작동 방식

1. 상점에 **언어를 활성화**합니다(예: 영어, 독일어, 일본어)
2. 콘텐츠(제품, 페이지, 블로그 게시물)를 생성하거나 편집할 때 기본 언어로 작성합니다
3. 번역 가능한 필드에 있는 **번역**을 클릭하여 활성화된 언어로 AI 번역을 생성합니다
4. 번역은 원본 콘텐츠와 함께 저장되며, 방문자의 언어에 따라 자동으로 제공됩니다

## 언어 관리

**설정 > 언어**로 이동하여 상점의 언어를 관리합니다.

### 언어 대시보드

대시보드는 다음과 같은 정보를 표시합니다:
- **총 언어 수** — 시스템에서 사용 가능한 모든 언어(100개 이상)
- **활성 언어** — 현재 상점에서 활성화된 언어
- **모델 커버리지** — 설치된 번역 모델이 지원하는 언어 수

### 언어 활성화

1. **사용 가능한 언어** 열에서 언어를 찾습니다
2. 언어를 클릭하여 **활성 언어** 열로 이동합니다
3. 언어는 즉시 번역에 사용 가능하며, 상점의 언어 전환기에서 표시됩니다

### 기본 언어

한 가지 언어가 **기본**으로 표시됩니다. 이는 다음과 같습니다:
- 콘텐츠를 작성하는 언어
- 번역이 존재하지 않을 때의 대체 언어
- 방문자가 선호도를 선택하지 않은 경우 표시되는 언어

## 번역 모델

Spwig은 외부 서비스로 데이터를 전송하지 않고 서버 로컬에서 전체적으로 실행되는 로컬 AI 번역 엔진을 포함합니다.

### 사용 가능한 모델

| 모델 | 언어 | 속도 | 품질 |
|-------|-----------|-------|---------|
| **M2M100-418M** | 100 | 빠름 | 일반적인 언어 쌍에 적합 |
| **M2M100-1.2B** | 100 | 중간 | 더 높은 품질, 더 높은 자원 사용 |
| **NLLB-200** | 200+ | 중간 | 최고의 커버리지, 드문 언어 포함 |

### 모델 선택

언어 관리 페이지는 설치된 모델과 그 언어 커버리지를 보여줍니다. 모델은 CTranslate2를 사용하여 효율적인 추론을 위해 로컬 서비스로 실행됩니다.

## 외부 제공업체

클라우드 기반 번역을 선호하거나 특정 언어 품질이 필요한 경우, Spwig은 외부 번역 제공업체를 지원합니다.

| 제공업체 | 설명 |
|----------|-------------|
| **DeepL** | 유럽 및 아시아 언어에 대한 프리미엄 번역 품질 |
| **Google Translate** | 신경 기계 번역을 사용한 넓은 언어 커버리지 |
| **Azure Translator** | 마이크로소프트의 신경 번역 서비스 |
| **AWS Translate** | 커스텀 용어 지원을 포함한 아마존의 기계 번역 |

### 제공업체 연결

1. **설정 > 번역 제공업체**로 이동합니다
2. 제공업체를 선택하고 API 키를 입력합니다
3. 제공업체를 기본 번역 엔진으로 설정합니다
4. 번역은 외부 제공업체를 사용하여 수행됩니다

외부 제공업체와 로컬 모델을 함께 사용할 수 있습니다. 예를 들어, 유럽 언어에 대해 DeepL을 사용하고 나머지 모든 언어에 대해 로컬 모델을 사용할 수 있습니다.

## 콘텐츠 번역

### 필드 수준 번역

번역 가능한 필드(제품 이름, 설명, SEO 제목 등)는 필드 옆에 **번역 버튼**을 표시합니다. 클릭하면 다음과 같은 작업을 수행할 수 있습니다:

1. **모든 활성 언어로 번역** — 모든 활성 언어에 대해 한 번에 번역을 생성합니다
2. **특정 언어로 번역** — 번역할 개별 언어를 선택합니다

번역은 편집기의 언어 탭에 나타납니다. 기계 번역을 검토하고 수동으로 편집할 수 있습니다.

### 대량 번역 작업

대량의 콘텐츠를 번역하려면 **번역 작업**을 사용합니다:

1. **설정 > 번역 작업**으로 이동합니다
2. 새 작업을 생성하고 다음을 선택합니다:
   - **콘텐츠 유형** — 제품, 페이지, 블로그 게시물, 카테고리 등
   - **소스 언어** — 번역할 언어
   - **대상 언어** — 하나 이상의 언어로 번역할 언어
   - **범위** — 모든 콘텐츠 또는 번역되지 않은 필드만
3. 작업을 제출합니다 — 작업은 작업 대기열을 통해 백그라운드에서 실행됩니다
4. 작업 목록에서 진행 상황을 모니터링합니다(대기 중 → 처리 중 → 완료됨)

대량 작업은 새 언어를 활성화하고 전체 카탈로그를 한 번에 번역하려는 경우에 유용합니다.

## 번역 관리

### 번역 검토

각 번역된 필드는 다음과 같은 정보를 추적합니다:
- **번역 상태** — 필드가 기계 번역, 수동 편집 또는 누락되었는지 여부
- **잠금 상태** — 잠긴 번역은 향후 기계 번역으로 덮어쓰지 않습니다
- **마지막 번역** — 번역이 마지막으로 생성되거나 편집된 시간

### 번역 잠금

기계 번역을 수동으로 편집하여 개선한 경우, **잠금**을 설정하여 대량 번역이 다시 실행될 때 필드가 덮어쓰지 않도록 합니다. 잠긴 필드는 자동 번역 중에 건너뜁니다.

### 번역 커버리지

커버리지 추적기는 각 언어에 대해 콘텐츠의 번역 비율을 보여줍니다. **설정 > 언어**로 이동하여 다음과 같은 정보를 확인할 수 있습니다:
- 언어별 완료 비율
- 콘텐츠 유형의 누락된 부분
- 여전히 번역이 필요한 필드

## UI 번역 오버라이드

제품 및 페이지 콘텐츠를 넘어, **프론트엔드 인터페이스 문자열**의 번역을 커스터마이징할 수 있습니다. 버튼, 라벨, 메시지 및 기타 방문자에게 표시되는 UI 텍스트 등입니다.

**설정 > UI 오버라이드**로 이동하여:
1. 특정 문자열(예: "Add to Cart")을 검색합니다
2. 각 언어에 대해 선호하는 번역을 입력합니다
3. 저장 — 오버라이드는 즉시 적용됩니다

커스터마이징 가능한 프론트엔드 문자열은 약 300개입니다. 오버라이드는 기본 번역보다 우선 적용됩니다.

## 팁

- 고객이 실제로 사용하는 언어만 먼저 활성화하세요 — 나중에 언제든지 추가할 수 있습니다.
- 일상적인 번역에는 **로컬 AI 모델**을 사용하세요 — 빠르고, 개인 정보 보호되고, 번역당 비용이 없습니다.
- 주요 유럽 언어에 대해 최고의 품질이 필요한 경우 **DeepL**을 고려하세요 — 일반 모델보다 더 자연스러운 번역을 일관되게 생성합니다.
- 제품 이름, 브랜드 용어 및 마케팅 콘텐츠의 **기계 번역을 검토**하세요 — AI는 기술 콘텐츠를 잘 처리하지만 창의적인 텍스트의 세부 사항을 놓칠 수 있습니다.
- 수동으로 개선한 번역은 **잠금**을 설정하여 대량 번역 작업 중에 덮어쓰지 않도록 보호하세요.
- 새 언어를 활성화할 때 **대량 번역 작업**을 사용하여 전체 카탈로그를 한 번에 번역하는 것이 좋습니다.
- **UI 오버라이드**를 사용하여 브랜드의 목소리와 일치하도록 커스터마이징하세요 — 예를 들어, "Add to Cart"을 "Buy Now"로 변경하는 것이 상점에 더 적합하다면 그렇게 하세요.

기억하세요: 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어를 보존 규칙에 따라 정확히 유지하세요.
