---
title: Translation Jobs
---

Translation Jobs ช่วยให้คุณแปลเนื้อหาจำนวนมากได้อย่างอัตโนมัติ แทนที่จะแปลสินค้าทีละชิ้นด้วยตนเอง คุณสามารถสร้างงานที่แปลแคตตาล็อกทั้งหมด หรือชุดย่อยเฉพาะได้ในพื้นหลัง งานเหล่านี้ทำงานแบบไม่ส่งผลต่อการทำงานของคุณ ดังนั้นคุณจึงสามารถดำเนินการต่อได้ขณะที่ระบบแปลข้อมูลหลายร้อยหรือหลายพันชุดอัตโนมัติ

ใช้ Translation Jobs เมื่อเปิดใช้งานภาษาใหม่ นำเข้าสินค้าใหม่ หรือแก้ไขเนื้อหาที่ยังไม่ได้แปล

## Translation Jobs คืออะไร?

งานแปลคืองานที่ทำงานในพื้นหลังที่:

1. **สแกนเนื้อหา** สำหรับฟิลด์ที่สามารถแปลได้ (สินค้า หน้าเพจ โพสต์บล็อก ฯลฯ)
2. **ระบุฟิลด์ที่ยังไม่ได้แปลหรือล้าสมัย** ตามขอบเขตของงานของคุณ
3. **ส่งฟิลด์ไปยังเครื่องมือแปล** (โมเดล AI ภายใน หรือผู้ให้บริการภายนอก)
4. **บันทึกการแปล** กลับไปยังเนื้อหาของคุณ
5. **รายงานความสำเร็จ** พร้อมสถิติของฟิลด์ที่แปลแล้ว

งานทำงานผ่านระบบคิวงาน Celery ดังนั้นจึงไม่บล็อกอินเทอร์เฟซการจัดการของคุณ

## เมื่อใดควรใช้ Translation Jobs

**เปิดใช้งานภาษาใหม่**:
- เปิดใช้งานภาษาเยอรมันเป็นภาษาใหม่
- สร้างงาน: แปลสินค้าทั้งหมดจากภาษาอังกฤษไปยังภาษาเยอรมัน
- ผลลัพธ์: แคตตาล็อกทั้งหมดพร้อมใช้งานในภาษาเยอรมันภายในไม่กี่นาที/ชั่วโมง (ขึ้นอยู่กับขนาด)

**การนำเข้าสินค้าใหม่**:
- นำเข้าสินค้าใหม่ 500 รายการในภาษาอังกฤษ
- สร้างงาน: แปลสินค้าใหม่ไปยังภาษาทั้งหมดที่เปิดใช้งาน
- ผลลัพธ์: สินค้าใหม่พร้อมใช้งานทันทีในตลาดทั้งหมด

**แก้ไขช่องว่างที่ยังไม่ได้แปล**:
- รายงานการครอบคลุมแสดงว่าสินค้ามีเพียง 60% ที่แปลเป็นภาษาฝรั่งเศส
- สร้างงาน: แปลฟิลด์สินค้าที่ยังไม่ได้แปลเป็นภาษาฝรั่งเศสเท่านั้น
- ผลลัพธ์: การครอบคลุมของภาษาฝรั่งเศสเพิ่มขึ้นเป็น ~100%

**อัปเดตการแปลที่ล้าสมัย**:
- โมเดลการแปลได้รับการปรับปรุงหรือมีผู้ให้บริการใหม่
- สร้างงาน: แปลสินค้าทั้งหมดใหม่เป็นภาษาสเปน
- ผลลัพธ์: การแปลภาษาสเปนทั่วแคตตาล็อกมีคุณภาพสูงขึ้น

## การสร้างงานแปล

ไปที่ **Settings > Translation Jobs** และคลิก **+ Create Job**

### การตั้งค่างาน

**ชื่องาน** - ป้ายกำกับที่อธิบายได้ (เช่น "แปลสินค้าเป็นภาษาเยอรมัน", "โพสต์บล็อกใหม่ - ทุกภาษา")

**ประเภทเนื้อหา** - สิ่งที่ต้องการแปล:
- สินค้า
- หมวดหมู่สินค้า
- หน้าเพจ
- โพสต์บล็อก
- ข้อมูล SEO
- แม่แบบอีเมล
- ทุกประเภทเนื้อหา

**ภาษาต้นทาง** - ภาษาที่คุณกำลังแปลจาก (มักเป็นภาษาเริ่มต้นของคุณ)

**ภาษาเป้าหมาย** - หนึ่งหรือหลายภาษาที่คุณต้องการแปลไป (เลือกหลายภาษาเพื่อแปลแบบขนาน)

**ขอบเขต** - ชุดย่อยของเนื้อหา:
- **ทั้งหมด** - แปลทุกอย่างไม่ว่าจะมีการแปลก่อนหน้านี้แล้วหรือไม่
- **เฉพาะที่ยังไม่ได้แปล** - เสิร์ฟข้ามฟิลด์ที่มีการแปลแล้ว
- **สร้าง/อัปเดตตั้งแต่วันที่** - เนื้อหาใหม่หรือที่เปลี่ยนแปลงล่าสุดเท่านั้น
- **รายการเฉพาะ** - เลือกสินค้า/หน้าเพจแต่ละรายการ (การกรองขั้นสูง)

**เครื่องมือแปล** - บริการที่ต้องการใช้:
- โมเดล AI ภายใน (ค่าเริ่มต้น ไม่มีค่าใช้จ่าย API)
- ผู้ให้บริการภายนอกเฉพาะ (DeepL, Google, Azure, AWS)
- เลือกอัตโนมัติ (ใช้การตั้งค่าที่กำหนดไว้ก่อนหน้า)

**ล็อกการแปล** - กำหนดว่าจะล็อกฟิลด์ที่แปลแล้วไม่ให้ถูกเขียนทับโดยอัตโนมัติในอนาคต (มีประโยชน์สำหรับการแปลที่ได้รับการตรวจสอบแล้ว)

### ตัวเลือกขั้นสูง

**ข้ามฟิลด์ที่ล็อก** - หากเปิดใช้งาน จะเคารพการแปลที่ล็อกไว้ก่อนหน้านี้ (แนะนำให้เปิดใช้งาน)

**ทับการแปลที่มีอยู่** - แปลใหม่แม้จะมีการแปลอยู่แล้ว (ใช้เพื่อปรับปรุงคุณภาพ)

**ตัวกรองฟิลด์** - แปลเฉพาะฟิลด์ที่กำหนด (เช่น ชื่อและคำอธิบายสินค้า ข้ามคุณสมบัติ)

**ขนาดแบตช์** - จำนวนรายการที่ต้องการประมวลผลในครั้งเดียว (ค่าเริ่มต้น: 50 เพิ่มขึ้นเพื่อให้ประมวลผลเร็วขึ้นหากเซิร์ฟเวอร์สามารถรองรับได้)

**ลำดับความสำคัญ** - งานที่มีลำดับความสำคัญสูงจะดำเนินการก่อนงานทั่วไป (ควรใช้แบบประหยัด)

## วงจรชีวิตและการสถานะของงาน

งานจะผ่านสถานะเหล่านี้:

**Queued** - งานถูกสร้างขึ้นแล้ว กำลังรอให้ worker รับงาน

**Processing** - worker กำลังแปลเนื้อหาอย่างกระตือรือร้น

**Completed** - การแปลทั้งหมดเสร็จสิ้นอย่างสำเร็จ

**Failed** - งานพบข้อผิดพลาด (ตรวจสอบบันทึกข้อผิดพลาด)

**Cancelled** - ถูกหยุดโดยผู้ดูแลระบบ

**Paused** - ถูกหยุดชั่วคราว (สามารถเริ่มต้นใหม่ได้)

## การติดตามความคืบหน้าของงาน

หน้ารายละเอียดงานแสดง:

**แถบความคืบหน้า** - ร้อยละที่ทำเสร็จแล้ว

**สถิติ**:
- จำนวนรายการทั้งหมดที่ต้องการแปล
- จำนวนรายการที่ทำเสร็จแล้ว
- จำนวนรายการที่เหลืออยู่
- เวลาที่คาดว่าจะใช้

**บันทึกแบบเรียลไทม์** - ลำดับเหตุการณ์การแปล (มีประโยชน์สำหรับการแก้ไขปัญหา)

**จำนวนข้อผิดพลาด** - จำนวนฟิลด์ที่ไม่สามารถแปลได้ (พร้อมเหตุผล)

## ผลลัพธ์และสถิติของงาน

เมื่องานเสร็จสิ้น หน้าผลลัพธ์จะแสดง:

**สรุป**:
- จำนวนฟิลด์ทั้งหมดที่ประมวลผล
- จำนวนฟิลด์ที่แปลสำเร็จ
- จำนวนการแปลที่ล้มเหลว
- จำนวนที่ข้าม (มีการแปลแล้ว ถูกล็อก หรือถูกตัดออกโดยตัวกรอง)

**การวิเคราะห์รายรายการ**:
- สินค้า/หน้าเพจใดที่ถูกแปล
- จำนวนฟิลด์ต่อรายการ
- ข้อผิดพลาดใดที่เกิดขึ้น

**ตัวชี้วัดประสิทธิภาพ**:
- เวลาทั้งหมดที่ผ่านมา
- จำนวนการแปลต่อวินาทีเฉลี่ย
- เครื่องมือแปลที่ใช้

## การจัดการการแปลที่ล้มเหลว

หากมีการแปลบางส่วนล้มเหลว:

**ตรวจสอบบันทึกข้อผิดพลาด** - ระบุฟิลด์ใดที่ล้มเหลวและเหตุผล

**สาเหตุการล้มเหลวที่พบบ่อย**:
- ถูกจำกัดการใช้งาน API (ผู้ให้บริการภายนอก)
- เวลาหมดของเครื่องมือแปล (เนื้อหาที่ยาวมาก)
- รูปแบบฟิลด์ไม่ถูกต้อง (ข้อผิดพลาดการวิเคราะห์ JSON)
- โมเดลไม่รองรับคู่ภาษา

**ตัวเลือกการลองอีกครั้ง**:
- แก้ไขปัญหาที่เกิดขึ้นเบื้องต้น
- สร้างงานใหม่สำหรับรายการที่ล้มเหลวเท่านั้น
- ใช้เครื่องมือแปลที่แตกต่างกัน

## การยกเลิกและการหยุดงาน

**ยกเลิก** - หยุดงานทันที ทิ้งการแปลที่กำลังดำเนินอยู่ (การแปลที่เสร็จสิ้นแล้วจะถูกบันทึกไว้)

**หยุดชั่วคราว** - หยุดงานชั่วคราว สามารถเริ่มต้นใหม่ได้ในภายหลังจากที่หยุดไว้

**เริ่มต้นใหม่** - ดำเนินการต่องานที่หยุดชั่วคราว

ใช้การหยุดชั่วคราว/เริ่มต้นใหม่เมื่อคุณต้องการปลดปล่อยทรัพยากรเซิร์ฟเวอร์ชั่วคราว

## กลยุทธ์งานแบบกลุ่ม

**กลยุทธ์ 1: ภาษาต่อภาษา**:
- สร้างงานแยกต่างหากสำหรับแต่ละภาษาเป้าหมาย
- ง่ายต่อการติดตามความคืบหน้าต่อภาษา
- สามารถกำหนดลำดับความสำคัญให้กับภาษาที่สำคัญได้
- กระจายโหลดตามเวลา

**กลยุทธ์ 2: ทั้งหมดในครั้งเดียว**:
- งานเดียวที่แปลไปยังภาษาทั้งหมดที่เปิดใช้งาน
- สรุปเสร็จเร็วกว่า
- โหลดเซิร์ฟเวอร์สูงในระหว่างการประมวลผล
- จัดการงานได้ง่ายขึ้น

**กลยุทธ์ 3: ประเภทเนื้อหาต่อประเภทเนื้อหา**:
- แปลสินค้าก่อน (ลำดับความสำคัญสูงสุด)
- จากนั้นแปลหมวดหมู่, หน้าเพจ, โพสต์บล็อก
- ช่วยให้เปิดตัวได้อย่างค่อยเป็นค่อยไป
- ง่ายต่อการทดสอบและตรวจสอบการแปล

เลือกตามความสามารถของเซิร์ฟเวอร์ ความเร่งด่วน และขนาดแคตตาล็อกของคุณ

## การกำหนดเวลาทำงาน

กำหนดงานที่เกิดขึ้นซ้ำเพื่อจัดการเนื้อหาใหม่โดยอัตโนมัติ:

**งานรายวัน** - แปลสินค้าที่สร้างหรืออัปเดตในช่วง 24 ชั่วโมงที่ผ่านมา

**งานรายสัปดาห์** - แก้ไขช่องว่างการแปลรายสัปดาห์

**หลังการนำเข้า** - เริ่มงานอัตโนมัติหลังการนำเข้าสินค้าแบบกลุ่ม

**เมื่อเปิดใช้งานภาษา** - สร้างงานอัตโนมัติเมื่อคุณเปิดใช้งานภาษาใหม่

งานที่กำหนดเวลาช่วยให้การแปลเป็นปัจจุบันโดยไม่ต้องแทรกแซงด้วยตนเอง

## ข้อพิจารณาด้านประสิทธิภาพ

**โมเดล AI ภายใน**:
- ~100-500 การแปลต่อวินาที (ขึ้นอยู่กับเซิร์ฟเวอร์)
- ใช้ทรัพยากร CPU ขณะประมวลผล
- ไม่มีข้อจำกัดการใช้งาน API
- ฟรี (ไม่มีค่าใช้จ่ายต่อการแปล)

**ผู้ให้บริการภายนอก**:
- ข้อจำกัดการใช้งานแตกต่างกัน (DeepL: 500k ตัวอักษรต่อเดือนในระดับฟรี)
- ความล่าช้าของ API เพิ่มขึ้น
- คุณภาพดีกว่าแต่ต้องเสียค่าใช้จ่าย
- ข้อจำกัดการร้องขอพร้อมกัน

**งานขนาดใหญ่** (>10,000 ฟิลด์):
- ดำเนินการในช่วงเวลาที่ไม่เป็นช่วงเร่งด่วน
- ตรวจสอบทรัพยากรเซิร์ฟเวอร์
- พิจารณาแบ่งงานเป็นแบตช์ขนาดเล็กกว่า
- ทดสอบกับชุดย่อยก่อน

## เคล็ดลับ

- **เริ่มจากงานขนาดเล็ก** - ทดสอบงานกับชุดย่อย (เช่น 10 สินค้า) ก่อนที่จะแปลแคตตาล็อกทั้งหมด
- **ใช้ขอบเขต "เฉพาะที่ยังไม่ได้แปล"** - ทำให้เร็วขึ้นและหลีกเลี่ยงการแปลซ้ำเนื้อหาที่ดีอยู่แล้ว
- **ติดตามงานแรกอย่างใกล้ชิด** - ตรวจสอบข้อผิดพลาดหรือปัญหาคุณภาพก่อนเริ่มงานขนาดใหญ่
- **กำหนดงานในช่วงเวลาที่มีการจราจรต่ำ** - การแปลใช้ทรัพยากร CPU/API มาก
- **ล็อกการแปลที่ตรวจสอบแล้ว** - ป้องกันไม่ให้งานแปลขนาดใหญ่เขียนทับการแก้ไขของคุณ
- **ให้งานมีเป้าหมายชัดเจน** - งานขนาดเล็กและมีเป้าหมายชัดเจนจะแก้ไขปัญหาง่ายกว่างานขนาดใหญ่ที่แปลทุกอย่าง
- **ตรวจสอบตัวอย่างหลังจากเสร็จสิ้น** - ตรวจสอบการแปลแบบสุ่มเพื่อความคุณภาพก่อนพิจารณาว่างานสำเร็จ
- **สำรองข้อมูลก่อนงานขนาดใหญ่** - ในกรณีที่คุณต้องการยกเลิกการแปลแบบกลุ่ม
