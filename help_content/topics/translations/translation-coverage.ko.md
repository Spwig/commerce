---
title: 번역 범위
---

번역 범위 페이지는 상점의 콘텐츠가 각 활성 언어로 얼마나 번역되었는지를 종합적으로 보여줍니다. 이 대시보드를 사용하여 번역 공백을 식별하고, 진행 상황을 추적하며, 새로운 시장에 출시하기 전에 완전한 다국어 커버리지를 보장할 수 있습니다.

범위는 모든 번역 가능한 콘텐츠 유형(상품, 카테고리, 페이지, 블로그 게시물, SEO 필드 등)에 걸쳐 계산됩니다.

## 범위 지표 이해

**전체 범위 비율** - 특정 언어에 대해 모든 번역 가능한 필드가 번역된 비율

**콘텐츠 유형별 분석** - 각 유형의 콘텐츠(상품, 페이지, 블로그 게시물 등)별 범위 비율

**필드 수준 세부 정보** - 번역된 필드와 누락된 필드가 무엇인지

**총 번역 가능한 필드 수** - 비율 계산에 사용되는 분모

## 범위 대시보드 개요

대시보드는 다음과 같이 표시됩니다:

**언어 카드** - 활성 언어별로 전체 범위 비율을 보여주는 카드 하나씩

**콘텐츠 유형 테이블** - 상품, 카테고리, 페이지, 블로그 게시물, SEO 메타데이터 등에 따른 상세 분석

**번역 누락 수** - 각 범주에서 아직 번역이 필요한 필드 수

**완료 추세** - 시간에 따른 진행 상황(추적 기능이 활성화된 경우)

## 범위 비율 해석

| 범위 | 상태 | 필요한 조치 |
|------|-----|-----------|
| **95-100%** | 우수 | 남은 공백을 검토하세요. 의도적으로 번역되지 않은 경우가 있을 수 있습니다. |
| **80-94%** | 양호 | 주요 누락 콘텐츠를 식별하고 번역하세요. |
| **60-79%** | 보통 | 높은 가시성 콘텐츠(홈페이지, 상품 이름 등)를 우선순위로 처리하세요. |
| **<60%** | 개선 필요 | 대량 번역 작업을 실행하거나 번역가를 고용하세요. |

기억하세요: 100% 커버리지는 항상 필수적이지 않습니다. 일부 필드(SKUs, 내부 메모, 기술 ID 등)는 번역이 필요하지 않습니다.

## 콘텐츠 유형 분석

**상품** - 일반적으로 가장 큰 범주:
- 상품 이름 및 설명
- 변형 이름(크기, 색상 옵션)
- 속성 값
- 사용자 정의 필드
- SEO 제목 및 메타 설명

**카테고리** - 탐색 및 브라우징:
- 카테고리 이름 및 설명
- SEO 메타데이터
- 사용자 정의 카테고리 필드

**페이지** - 정적 콘텐츠:
- 페이지 제목 및 본문 콘텐츠
- SEO 메타데이터
- 사용자 정의 페이지 필드

**블로그 게시물** - 콘텐츠 마케팅:
- 게시물 제목 및 콘텐츠
- 요약 및 요약
- SEO 메타데이터

**기타 콘텐츠**:
- 상점 정책(배송, 반품, 개인정보 보호)
- 이메일 템플릿
- 폼 라벨 및 메시지

## 번역 공백 식별

어떤 콘텐츠 유형을 클릭하여 세부 필드 수준의 범위를 보세요:

**누락 필드 보기** - 정확히 어떤 상품/페이지/게시물이 번역되지 않은 필드를 보여줍니다.

**필드 우선순위** - 필드가 고객 대면 필드인지 내부 필드인지 표시합니다.

**대량 작업** - 여러 항목을 한 번에 번역할 수 있습니다.

## 범위를 사용한 번역 작업 계획

**새로운 언어 출시**:
1. 기존 언어의 범위를 확인합니다(기준선으로 사용)
2. 새 언어를 활성화합니다
3. 새 언어의 범위는 0%에서 시작합니다
4. 핵심 콘텐츠를 대량 번역하는 작업을 생성합니다
5. 작업이 완료될 때까지 범위를 모니터링합니다
6. 주요 콘텐츠를 수동으로 검토하고 정제합니다
7. 출시 전에 80% 이상을 목표로 합니다

**기존 언어 유지**:
1. 매달 범위를 검토합니다
2. 새 상품이 전체 범위를 낮출 수 있습니다
3. 상품 생성 시 자동 번역을 설정합니다
4. 주기적으로 작업을 실행하여 누락된 부분을 확인합니다

## 번역 작업 우선순위

고가치 콘텐츠에 우선적으로 집중하세요:

**우선순위 1 - 고객 대면 텍스트**:
- 상품 이름 및 주요 설명
- 홈페이지 및 메인 네비게이션
- 결제 흐름 페이지
- 핵심 정책(배송, 반품)

**우선순위 2 - 지원 콘텐츠**:
- 블로그 게시물 및 마케팅 콘텐츠
- 카테고리 설명
- 상품 변형 세부 정보
- 이메일 템플릿

**우선순위 3 - 선택적 콘텐츠**:
- 확장된 상품 사양
- 내부 메모 및 태그
- 관리자 전용 필드

## 범위 필터

필터를 사용하여 특정 콘텐츠에 집중하세요:

**언어별** - 한 번에 하나의 언어의 범위를 보여줍니다

**콘텐츠 유형별** - 상품만, 페이지만 등으로 필터링합니다

**완료도별** - 완료된 항목만, 완료되지 않은 항목만, 또는 모든 항목을 보여줍니다

**날짜 범위별** - 특정 기간 동안 생성/업데이트된 콘텐츠의 범위

## 범위 보기에서 대량 번역

공백을 식별하면 범위 페이지에서 직접 번역 작업을 시작할 수 있습니다:

1. **낮은 범위의 콘텐츠 유형을 선택**합니다(예: "상품: 45% 범위")
2. **"누락 필드 번역"을 클릭**합니다
3. **대상 언어를 선택**합니다
4. **작업 제출** - 오직 누락된 필드만 번역합니다
5. **작업 모니터링** - 번역 작업 페이지에서 작업을 모니터링합니다
6. **범위로 돌아가** - 업데이트된 비율을 확인합니다

이러한 집중적인 접근 방식은 모든 것을 다시 번역하는 것보다 더 효율적입니다.

## 범위 대비 품질

**높은 범위는 품질을 보장하지 않습니다**. 스토어는 검토되지 않은 빈약한 기계 번역으로 100% 범위를 가질 수 있습니다.

**권장 워크플로**:
1. 대량 작업을 사용하여 빠르게 높은 범위를 달성하세요(수량)
2. 핵심 콘텐츠를 수동으로 검토하고 정제하세요(품질)
3. 정제된 번역을 잠급니다. 이는 그들을 보존하기 위해 필요합니다.
4. 주기적으로 번역의 정확성을 감사합니다.

## 시간에 따른 범위 추적

범위 추적을 활성화하여 추세를 모니터링하세요:

**주간 스냅샷** - 매주 범위 비율을 기록합니다

**성장 차트** - 100% 범위에 대한 진행 상황을 시각화합니다

**알림** - 범위가 감소할 때 알림(예: 번역되지 않은 상품이 많이 추가됨)

이것은 캐탈로그가 성장하면서 번역 품질을 유지하는 데 도움이 됩니다.

## 팁

- **범위 목표 설정** - 새 언어를 공개하기 전에 최소 80%를 목표로 하세요
- **콘텐츠 업데이트 후 모니터링** - 100개의 새 상품을 추가하면 범위가 떨어지며 번역이 완료될 때까지 유지됩니다
- **고객 대면 콘텐츠에 우선순위를 두세요** - 내부 필드는 기다릴 수 있습니다
- **범위를 사용하여 번역 예산을 정당화하세요** - 이해관계자에게 정확히 얼마나 많은 작업이 남았는지 보여줍니다
- **정기적인 범위 검토를 예약하세요** - 월별 점검은 번역 부채가 쌓이지 않도록 합니다
- **분석과 결합하세요** - 실제 트래픽이 있는 언어에 콘텐츠를 번역하세요
- **100%에 집착하지 마세요** - 일부 필드(SKUs, 내부 코드 등)는 번역이 필요하지 않습니다

기억하세요: 모든 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어를 보존 규칙에 따라 정확히 유지하세요.
