title: Охват перевода
---

Страница Охвата перевода предоставляет подробный обзор того, насколько содержимое вашего магазина переведено на каждый активный язык. Используйте этот дашборд для выявления пробелов в переводе, отслеживания прогресса и обеспечения полного многоязычного охвата перед запуском в новых рынках.

Охват рассчитывается для всех переводимых типов содержимого: товары, категории, страницы, блог-посты, SEO-поля и многое другое.

## Понимание метрик охвата

**Общий процент охвата** - Процент всех переводимых полей, переведённых на определённый язык

**Разбивка по типам содержимого** - Проценты охвата для каждого типа содержимого (товары, страницы, блог-посты и т.д.)

**Детализация на уровне полей** - Какие конкретные поля переведены, а какие отсутствуют

**Общее количество переводимых полей** - Знаменатель, используемый для расчёта процентов

## Обзор дашборда охвата

Дашборд отображает:

**Карты языков** - Одна карта на каждый активный язык, показывающая общий процент охвата

**Таблицы по типам содержимого** - Подробный разбор по товарам, категориям, страницам, блог-постам, метаданным SEO и т.д.

**Количество пропущенных переводов** - Сколько полей всё ещё требуют перевода в каждой категории

**Тенденции завершения** - Прогресс со временем (если отслеживание включено)

## Интерпретация процентов охвата

| Охват | Статус | Действие, необходимое |
|----------|--------|---------------|
| **95-100%** | Отлично | Проверьте оставшиеся пробелы, возможно, они намеренно не переведены |
| **80-94%** | Хорошо | Определите и переведите ключевые пропущенные элементы |
| **60-79%** | Средне | Приоритетно переведите высокодоступные элементы (главная страница, названия товаров) |
| **<60%** | Требует работы | Запустите массовые задания по переводу или наймите переводчиков |

Помните: 100% охвата не всегда необходимо. Некоторые поля (SKU, внутренние заметки, технические идентификаторы) не требуют перевода.

## Разбивка по типам содержимого

**Товары** - Самая большая категория обычно:
- Названия и описания товаров
- Названия вариантов (размеры, цвета)
- Значения атрибутов
- Пользовательские поля
- SEO-заголовки и мета-описания

**Категории** - Навигация и поиск:
- Названия и описания категорий
- SEO-метаданные
- Пользовательские поля категорий

**Страницы** - Статическое содержимое:
- Заголовки страниц и основной контент
- SEO-метаданные
- Пользовательские поля страниц

**Блог-посты** - Контент-маркетинг:
- Заголовки постов и содержимое
- Выдержки и резюме
- SEO-метаданные

**Другое содержимое**:
- Политики магазина (доставка, возвраты, конфиденциальность)
- Шаблоны электронной почты
- Метки и сообщения форм

## Выявление пробелов в переводе

Нажмите на любой тип содержимого, чтобы увидеть подробную информацию на уровне полей:

**Просмотр пропущенных полей** - Показывает, какие именно товары/страницы/посты имеют непереведённые поля

**Приоритет полей** - Показывает, какие поля предназначены для клиентов, а какие внутренние

**Массовые действия** - Выберите несколько элементов для одновременного перевода

## Использование охвата для планирования работы по переводу

**Запуск нового языка**:
1. Проверьте охват для существующих языков (как базовая линия)
2. Активируйте новый язык
3. Охват для нового языка начинается с 0%
4. Создайте задание перевода для массового перевода основного содержимого
5. Отслеживайте охват по мере завершения задания
6. Вручную проверьте и уточните ключевые элементы
7. Цель - достичь 80% и более перед запуском

**Поддержание существующих языков**:
1. Проверяйте охват ежемесячно
2. Новые товары могут снизить общий охват
3. Настройте автоматический перевод при создании товара
4. Периодически запускайте задания, чтобы выявить любые пробелы

## Приоритет работы по переводу

Сначала сосредоточьтесь на содержимом с высоким влиянием:

**Приоритет 1 - Текст, предназначенный для клиентов**:
- Названия товаров и основные описания
- Главная страница и основная навигация
- Страницы процесса оформления заказа
- Ключевые политики (доставка, возвраты)

**Приоритет 2 - Поддерживающее содержимое**:
- Блог-посты и маркетинговое содержимое
- Описания категорий
- Детали вариантов товаров
- Шаблоны электронной почты

**Приоритет 3 - Опциональное содержимое**:
- Расширенные спецификации товаров
- Внутренние заметки и теги
- Поля, доступные только в админ-панели

## Фильтры охвата

Используйте фильтры, чтобы сосредоточиться на конкретном содержимом:

**По языку** - Просмотр охвата для одного языка за раз

**По типу содержимого** - Фильтрация только товаров, только страниц и т.д.

**По полноте** - Показывать только завершённые элементы, только незавершённые элементы или все элементы

**По диапазону дат** - Охват содержимого, созданного/обновлённого в определённый период

## Массовый перевод из представления охвата

Когда вы выявляете пробелы, запускайте задания по переводу напрямую со страницы охвата:

1. **Выберите тип содержимого** с низким охватом (например, "Товары: 45% охвата")
2. **Нажмите "Перевести пропущенные поля"**
3. **Выберите целевой язык(ы)**
4. **Отправьте задание** - Переводит только пропущенные поля
5. **Отслеживайте задание** на странице заданий по переводу
6. **Вернитесь к охвату**, чтобы увидеть обновлённые проценты

Этот целевой подход более эффективен, чем повторный перевод всего содержимого.

## Охват vs Качество

**Высокий охват не гарантирует качество**. Магазин может иметь 100% охвата с непроверенными машинными переводами, которые плохи.

**Рекомендуемый рабочий процесс**:
1. Используйте массовые задания, чтобы быстро достичь высокого охвата (количество)
2. Вручную проверьте и уточните ключевые элементы (качество)
3. Заблокируйте уточнённые переводы, чтобы сохранить их
4. Периодически проверяйте переводы на точность

## Отслеживание охвата со временем

Включите отслеживание охвата, чтобы отслеживать тенденции:

**Еженедельные снимки** - Процент охвата записывается каждую неделю

**Графики роста** - Визуализация прогресса к 100% охвата

**Уведомления** - Уведомления, когда охват падает (например, добавлено много непереведённых товаров)

Это помогает сохранять качество перевода по мере роста вашего каталога.

## Советы

- **Установите цели охвата** - Цель: достичь минимум 80% перед публичным запуском нового языка
- **Отслеживайте после обновления содержимого** - Добавление 100 новых товаров снижает охват до тех пор, пока они не будут переведены
- **Сначала сосредоточьтесь на клиентском содержимом** - Внутренние поля могут подождать
- **Используйте охват для обоснования бюджета на перевод** - Покажите заинтересованным сторонам, сколько работы остаётся
- **Планируйте регулярные проверки охвата** - Ежемесячные проверки предотвращают накопление задолженности по переводу
- **Сочетайте с аналитикой** - Переводите контент на языки, где у вас есть реальный трафик
- **Не зацикливайтесь на 100%** - Некоторые поля (SKU, внутренние коды) не требуют перевода
