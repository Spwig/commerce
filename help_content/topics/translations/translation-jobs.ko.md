---
title: 번역 작업
---

번역 작업은 대량의 콘텐츠를 자동으로 번역하는 기능입니다. 하나씩 제품을 수동으로 번역하는 대신, 전체 카탈로그 또는 특정 부분을 배경에서 번역하는 작업을 생성할 수 있습니다. 작업은 비동기식으로 실행되므로 수백 또는 수천 개의 필드가 자동으로 번역되는 동안에도 계속 작업할 수 있습니다.

새로운 언어를 활성화하거나 새로운 제품을 가져오거나 번역되지 않은 콘텐츠를 처리할 때 번역 작업을 사용합니다.

## 번역 작업이란 무엇인가요?

번역 작업은 다음을 수행하는 배경 작업입니다:

1. **콘텐츠 스캔** - 번역 가능한 필드(제품, 페이지, 블로그 게시물 등)를 찾아냅니다.
2. **번역되지 않은 또는 오래된 필드 식별** - 작업 범위에 따라 번역되지 않은 필드를 식별합니다.
3. **필드를 번역 엔진으로 전송** - 로컬 AI 모델 또는 외부 제공업체로 필드를 전송합니다.
4. **번역 저장** - 번역된 내용을 다시 저장합니다.
5. **완료 보고** - 번역된 필드에 대한 통계를 포함한 완료를 보고합니다.

작업은 Celery 작업 대기열을 통해 실행되므로 관리자 인터페이스를 차단하지 않습니다.

## 번역 작업을 언제 사용해야 하나요?

**새로운 언어 출시**:
- 독일어를 새로운 언어로 활성화
- 작업 생성: 영어에서 독일어로 모든 제품 번역
- 결과: 몇 분 또는 몇 시간 안에 전체 카탈로그가 독일어로 제공됩니다(크기에 따라 다름)

**새로운 제품 가져오기**:
- 영어로 500개의 새로운 제품 가져오기
- 작업 생성: 새로운 제품을 모든 활성 언어로 번역
- 결과: 모든 시장에서 즉시 새로운 재고가 제공됩니다

**번역 누락 보완**:
- 보고서에 따르면 제품은 프랑스어로 오직 60%만 번역됨
- 작업 생성: 프랑스어로 번역되지 않은 제품 필드만 번역
- 결과: 프랑스어 보완률이 약 100%에 도달함

**구식 번역 업데이트**:
- 번역 모델이 개선되었거나 새로운 제공업체가 사용 가능
- 작업 생성: 모든 제품을 스페인어로 재번역
- 결과: 카탈로그 전반에 걸쳐 품질이 높은 스페인어 번역

## 번역 작업 생성

**설정 > 번역 작업**으로 이동하고 **+ 작업 생성**을 클릭합니다.

### 작업 구성

**작업 이름** - 설명적인 라벨(예: "제품을 독일어로 번역", "새로운 블로그 게시물 - 모든 언어")

**콘텐츠 유형** - 무엇을 번역할 것인가:
- 제품
- 제품 카테고리
- 페이지
- 블로그 게시물
- SEO 메타데이터
- 이메일 템플릿
- 모든 콘텐츠 유형

**소스 언어** - 번역할 언어(보통 기본 언어)

**대상 언어** - 번역할 언어(여러 개 선택하여 병렬 번역)

**범위** - 콘텐츠의 어떤 부분을 번역할 것인가:
- **모든 항목** - 기존 번역과 관계없이 모든 것을 번역
- **번역되지 않은 항목만** - 이미 번역된 필드를 건너뜁니다.
- **특정 날짜 이후 생성/업데이트된 항목** - 오직 새롭거나 최근에 변경된 콘텐츠만
- **특정 항목** - 개별 제품/페이지 선택(고급 필터링)

**번역 엔진** - 어떤 서비스를 사용할 것인가:
- 로컬 AI 모델(기본값, API 비용 없음)
- 특정 외부 제공업체(DeepL, Google, Azure, AWS)
- 자동 선택(설정된 선호도 사용)

**번역 잠금** - 번역된 필드가 미래의 자동 덮어쓰기로부터 잠금되어 있는지 여부(검토된 번역에 유용)

### 고급 옵션

**잠긴 필드 건너뛰기** - 활성화된 경우 기존의 잠긴 번역을 존중함(권장)

**기존 내용 덮어쓰기** - 번역이 이미 존재하더라도 재번역(품질 개선에 사용)

**필드 필터** - 특정 필드만 번역(예: 제품 이름과 설명, 속성은 건너뛰기)

**배치 크기** - 한 번에 처리할 항목 수(기본값: 50, 서버가 처리할 수 있다면 더 빠르게 처리하기 위해 증가)

**우선순위** - 높은 우선순위 작업은 일반 우선순위 작업보다 먼저 실행됨(적절히 사용)

## 작업 생명주기 및 상태

작업은 다음 상태를 거칩니다:

**대기 중** - 작업이 생성되었으며, 작업자에게 할당될 때까지 대기 중입니다.

**처리 중** - 작업자가 콘텐츠를 번역하고 있습니다.

**완료됨** - 모든 번역이 성공적으로 완료되었습니다.

**실패** - 작업 중 오류가 발생했습니다(에러 로그 확인)

**취소됨** - 관리자가 수동으로 중지했습니다.

**일시 중지됨** - 일시적으로 중단됨(재개 가능)

## 작업 진행 상황 모니터링

작업 세부 정보 페이지는 다음과 같은 내용을 보여줍니다:

**진행률 표시줄** - 완료된 비율

**통계**:
- 번역할 총 항목 수
- 완료된 항목 수
- 남은 항목 수
- 예상 소요 시간

**실시간 로그** - 번역 활동 스트림(문제 해결에 유용)

**에러 수** - 번역에 실패한 필드 수(이유 포함)

## 작업 결과 및 통계

작업이 완료되면 결과 페이지는 다음과 같은 내용을 보여줍니다:

**요약**:
- 처리된 총 필드 수
- 성공적으로 번역된 필드 수
- 실패한 번역 수
- 건너뛴 항목(이미 번역됨, 잠금됨, 필터에 의해 제외됨)

**항목별 분석**:
- 번역된 제품/페이지는 무엇인지
- 항목별 필드 수
- 발생한 오류

**성능 지표**:
- 총 소요 시간
- 초당 평균 번역 수
- 사용된 번역 엔진

## 실패한 번역 처리

일부 번역이 실패한 경우:

**에러 로그 검토** - 실패한 필드와 그 이유를 식별합니다.

**일반적인 실패 원인**:
- API 요청 제한(외부 제공업체)
- 번역 엔진 타임아웃(매우 긴 텍스트)
- 유효하지 않은 필드 형식(JSON 파싱 오류)
- 모델이 언어 쌍을 지원하지 않음

**재시도 옵션**:
- 근본적인 문제를 수정
- 오직 실패한 항목만을 위한 새 작업 생성
- 다른 번역 엔진 사용

## 작업 취소 및 일시 중지

**취소** - 작업을 즉시 중단하고 진행 중인 번역을 버립니다(이미 완료된 번역은 저장됨)

**일시 중지** - 작업을 일시적으로 중단하고 나중에 다시 시작할 수 있습니다.

**재개** - 일시 중지된 작업을 계속합니다.

작업을 일시 중지/재개하는 것은 서버 자원을 잠시 해제할 때 사용합니다.

## 대량 작업 전략

**전략 1: 언어별로**:
- 각 대상 언어에 대해 별도의 작업 생성
- 언어별로 진행 상황을 더 쉽게 모니터링할 수 있음
- 중요한 언어를 우선순위로 설정할 수 있음
- 시간에 따라 부하를 분산시킴

**전략 2: 모두 한 번에**:
- 모든 활성 언어로 번역하는 단일 작업
- 전체 완료가 더 빠름
- 처리 중에 서버 부하가 더 큼
- 작업 관리가 더 간단함

**전략 3: 콘텐츠 유형별로**:
- 제품 먼저 번역(가장 높은 우선순위)
- 그 후 카테고리, 페이지, 블로그 게시물
- 점진적인 출시 가능
- 번역을 테스트하고 확인하는 것이 더 쉬움

서버 용량, 긴급성 및 카탈로그 크기에 따라 전략을 선택하세요.

## 작업 일정

정기적으로 반복되는 작업을 예약하여 새로운 콘텐츠를 자동으로 처리할 수 있습니다:

**일일 작업** - 지난 24시간 동안 생성/업데이트된 제품을 번역

**주간 작업** - 매주 번역 누락을 보완

**가져오기 후** - 대량 제품 가져오기 후 자동으로 작업을 트리거

**언어 활성화 시** - 새로운 언어를 활성화할 때 자동으로 작업 생성

예약된 작업은 수동 개입 없이 번역을 최신 상태로 유지합니다.

## 성능 고려사항

**로컬 AI 모델**:
- ~100-500 번역/초(서버에 따라 다름)
- 처리 중 CPU 집약적
- API 요청 제한 없음
- 무료(번역당 비용 없음)

**외부 제공업체**:
- 요청 제한이 다름(DeepL: 무료 계층에서 월 500,000자)
- API 지연 시간이 추가 부하
- 품질이 더 좋지만 비용이 들음
- 동시 요청 제한

**대량 작업**(10,000 필드 이상):
- 비수요 시간에 실행
- 서버 자원 모니터링
- 작은 배치로 나누는 것을 고려
- 전체 작업 전에 일부분으로 테스트

## 팁

- **작은 규모부터 시작** - 전체 카탈로그 번역 전에 일부분(예: 10개 제품)으로 작업을 테스트하세요
- **"번역되지 않은 항목만" 범위 사용** - 더 빠르고 이미 좋은 콘텐츠를 다시 번역하는 것을 피합니다
- **첫 번째 작업을 면밀히 모니터링** - 더 큰 작업을 시작하기 전에 오류나 품질 문제를 확인하세요
- **저조한 트래픽 시에 작업 예약** - 번역은 CPU/API 집약적입니다
- **검토된 번역 잠금** - 대량 작업이 수동 편집을 덮어쓰지 않도록 합니다
- **작업을 집중화** - 더 작은, 특정 작업이 대규모 "모든 것을 번역" 작업보다 문제 해결이 더 쉽습니다
- **작업 완료 후 샘플 검토** - 작업이 성공적으로 완료되었는지 확인하기 전에 무작위 번역을 확인하세요
- **중요한 작업 전에 내보내기/백업** - 대량 변경을 되돌릴 필요가 있을 경우

기억하세요: 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어는 보존 규칙에 따라 정확히 유지하세요.
