---
title: การจัดการการแปล UI
---

หน้าการแปล UI ช่วยให้คุณสามารถปรับแต่งวิธีที่สตริงอินเทอร์เฟซด้านหน้า—ปุ่ม, ป้ายกำกับ, ข้อความข้อผิดพลาด, และสตริง UI อื่นๆ—ปรากฏในแต่ละภาษา ต่างจากข้อความการแปลของสินค้าหรือหน้าเว็บ นี่คือองค์ประกอบอินเทอร์เฟซที่คงที่ที่ลูกค้าจะเห็นตลอดทั้งร้านของคุณ ปรับแต่งมันเพื่อให้สอดคล้องกับเสียงของแบรนด์คุณ หรือเพื่อเพิ่มความชัดเจนสำหรับกลุ่มเป้าหมายเฉพาะของคุณ

หน้านี้แสดงสตริง UI ที่สามารถแปลได้ทั้งหมด และอนุญาตให้คุณทับ書きการแปลเริ่มต้นที่ Spwig ให้มา

## การทำความเข้าใจการแปล UI

การแปล UI คือสตริงข้อความที่ประกอบขึ้นเป็นอินเทอร์เฟซของร้านคุณ:

**ตัวอย่างของสตริง UI**:
- ปุ่ม: "เพิ่มลงในตะกร้า", "ชำระเงิน", "ค้นหา"
- ป้ายกำกับ: "ราคา", "จำนวน", "ที่อยู่การจัดส่ง"
- ข้อความ: "เพิ่มสินค้าลงในตะกร้า", "ยืนยันคำสั่งซื้อ", "ที่อยู่อีเมลไม่ถูกต้อง"
- การนำทาง: "หน้าแรก", "ร้านค้า", "ติดต่อเรา"
- ฟิลด์ฟอร์ม: "อีเมล", "รหัสผ่าน", "ชื่อ"

Spwig มีการแปลเริ่มต้นสำหรับสตริง UI ประมาณ 300 รายการในทุกภาษาที่รองรับ หน้าการแปล UI ช่วยให้คุณทับ書きการแปลเริ่มต้นใดๆ ด้วยการแปลเฉพาะของคุณเอง

## ทำไมถึงต้องปรับแต่งการแปล UI

**เสียงของแบรนด์**: เปลี่ยน "เพิ่มลงในตะกร้า" เป็น "ซื้อทันที" หรือ "ได้รับของคุณ" เพื่อให้สอดคล้องกับบุคลิกของแบรนด์คุณ

**ความแตกต่างตามภูมิภาค**: ปรับการแปลสำหรับตลาดเฉพาะ (ภาษาอังกฤษแบบบริติช vs แบบอเมริกัน, ภาษาสเปนในยุโรป vs ภาษาสเปนในอเมริกาละติน)

**ความชัดเจน**: หากการแปลเริ่มต้นไม่เข้าใจกับสินค้าหรือกลุ่มเป้าหมายของคุณ ให้เปลี่ยนมันด้วยข้อความที่ชัดเจนขึ้น

**คำศัพท์เฉพาะอุตสาหกรรม**: ใช้คำศัพท์ที่ลูกค้าของคุณคาดหวัง (เช่น "จองนัดหมาย" แทน "เพิ่มลงในตะกร้า" สำหรับร้านค้าที่เน้นบริการ)

## การค้นหาสตริง

ใช้กล่องค้นหาเพื่อหาสตริง UI ที่เฉพาะเจาะจง:

**ค้นหาด้วยข้อความภาษาอังกฤษ**: พิมพ์ "เพิ่มลงในตะกร้า" เพื่อหาการแปลของปุ่มนั้น

**ค้นหาด้วยการแปล**: พิมพ์ข้อความในภาษาใดก็ได้เพื่อหาการแปลที่ตรงกัน

**ค้นหาด้วยคีย์**: หากคุณทราบคีย์การแปล (เช่น `cart.add_item`) ค้นหาโดยตรง

หน้าจะอัปเดตทันทีขณะที่คุณพิมพ์ แสดงเฉพาะสตริงที่ตรงกับการค้นหา

## การดูรายละเอียดการแปล

แต่ละสตริง UI จะแสดง:

**ข้อความแหล่งภาษาอังกฤษ** - รุ่นภาษาอังกฤษเริ่มต้น (จุดอ้างอิงของคุณ)

**คีย์การแปล** - ตัวระบุภายในที่ใช้ในโค้ด (เช่น `cart.add_to_cart`)

**คอลัมน์ภาษา** - การแปลปัจจุบันสำหรับแต่ละภาษาที่เปิดใช้งาน

**สถานะการทับ書き** - ว่าคุณปรับแต่งการแปลหรือไม่ (จะถูกเน้นหากได้ทับ書き)

## การสร้างการทับ書きการแปล

เพื่อปรับแต่งการแปลของสตริง UI:

1. **ค้นหาสตริง** ด้วยการค้นหา (เช่น ค้นหา "เพิ่มลงในตะกร้า")
2. **คลิกเซลล์ภาษา** ที่คุณต้องการปรับแต่ง
3. **ป้อนการแปลเฉพาะของคุณ** ในตัวแก้ไขที่ปรากฏ
4. **บันทึก** - การทับ書きของคุณจะมีผลทันที

การแปลเริ่มต้นเดิ้ลจะถูกเก็บไว้ - คุณกำลังสร้างการทับ書きที่มีความสำคัญเหนือกว่า

## การกลับไปใช้การแปลเริ่มต้น

เพื่อลบการทับ書きเฉพาะและกู้คืนการแปลเริ่มต้น:

1. **คลิกการแปลที่ได้ทับ書き** (จะถูกเน้น)
2. **คลิก "กลับไปใช้การแปลเริ่มต้น"** ในตัวแก้ไข
3. **ยืนยัน** - การแปลเริ่มต้นจะได้รับการกู้คืนทันที

คุณสามารถกลับไปใช้การแปลเฉพาะของภาษาได้โดยไม่กระทบต่อการทับ書きในภาษาอื่น

## การกรองตามสถานะการทับ書き

ใช้เมนูกรองเพื่อแสดง:

**ทุกสตริง** - สตริง UI ทั้งหมดในระบบ (~300 รายการรวม)

**เฉพาะสตริงที่ได้ทับ書き** - สตริงที่คุณสร้างการแปลเฉพาะ

**เฉพาะการแปลเริ่มต้น** - สตริงที่ยังใช้การแปลเริ่มต้นของ Spwig

นี่ช่วยให้คุณตรวจสอบสตริงที่คุณปรับแต่งแล้ว และระบุช่องว่าง

## การตรวจสอบความถูกต้องของการแปล

เมื่อป้อนการแปลเฉพาะ ให้ตรวจสอบว่า:

**ความยาวเหมาะสมกับพื้นที่ UI** - การแปลอาจยาวหรือสั้นกว่าภาษาอังกฤษ (คำภาษาเยอรมันมักยาวกว่า)

**รักษาความหมาย** - อย่าเปลี่ยนฟังก์ชันในข้อความการแปล (ปุ่ม "ยกเลิก" ไม่ควรพิมพ์ว่า "ลบ")

**ใช้คำศัพท์ที่สอดคล้องกัน** - ใช้การแปลเดียวกันสำหรับคำที่ซ้ำกันตลอดทั้งอินเทอร์เฟซ

**ระดับความเป็นทางการที่เหมาะสม** - ให้ความเหมาะสมกับโทนเสียงของตลาดเป้าหมาย (ทางการ vs ไม่ทางการ)

## ความสอดคล้องหลายภาษา

เมื่อปรับแต่งสตริงสำหรับหลายภาษา:

1. **เริ่มต้นด้วยภาษาเริ่มต้นของคุณ** - ตั้งค่าพื้นฐาน
2. **ปรับแต่งภาษาอื่นๆ** ให้สอดคล้องกับเจตนารมณ์เดียวกัน
3. **ทดสอบในแต่ละภาษา** เพื่อตรวจสอบการจัดวางและความหมาย
4. **ใช้ผู้พูดภาษาพื้นเมือง** เมื่อเป็นไปได้เพื่อตรวจสอบการแปลที่ไม่ใช่ภาษาอังกฤษ

การปรับแต่งที่ไม่สอดคล้องกันในหลายภาษาจะสร้างประสบการณ์ลูกค้าที่สับสน

## การส่งออก/นำเข้าแบบกลุ่ม

สำหรับการปรับแต่งที่กว้างขวาง ให้พิจารณาใช้กระบวนการทำงานส่งออก/นำเข้า:

1. **ส่งออก** การแปลปัจจุบันเป็น JSON หรือ CSV
2. **แก้ไขในตาราง** หรือตัวแก้ไขข้อความ (ง่ายขึ้นสำหรับการเปลี่ยนแปลงแบบกลุ่ม)
3. **นำเข้า** การแปลที่อัปเดตกลับเข้าสู่ระบบ

กระบวนการทำงานนี้สามารถใช้ได้ผ่านหน้า Translation Jobs เพื่อจัดการโครงการแปลขนาดใหญ่

## เคล็ดลับ

- **ค้นหาก่อนปรับแต่ง** - ตรวจสอบว่าคุณกำลังแก้ไขสตริงที่ถูกต้อง; บางสตริงที่คล้ายกันมีวัตถุประสงค์ต่างกัน
- **ทดสอบในด้านหน้าหลังจากบันทึก** - ตรวจสอบว่าการแปลเฉพาะของคุณปรากฏใน UI จริงอย่างถูกต้อง
- **รักษาการแปลให้กระชับ** - การแปลที่สั้นกว่ามักจะดีกว่าสำหรับปุ่มและป้ายกำกับ
- **จดบันทึกการทับ書きของคุณ** - บันทึกเหตุผลที่คุณปรับแต่งสตริงเฉพาะเพื่อใช้อ้างอิงในอนาคต
- **ใช้คำศัพท์ที่สอดคล้องกัน** - หากคุณปรับแต่ง "ตะกร้า" เป็น "ตะกร้าสินค้า" ให้ทำแบบเดียวกันในสตริงที่เกี่ยวข้องทั้งหมด
- **พิจารณาการจัดวางบนมือถือ** - การแปลที่ยาวอาจทำให้เกิดการห่อหรือตัดทอนบนหน้าจอขนาดเล็ก
- **ตรวจสอบหลังการอัปเดตภาษา** - เมื่อ Spwig เพิ่มการแปลเริ่มต้นใหม่ ตรวจสอบและปรับแต่งเพื่อรักษาความสอดคล้อง
