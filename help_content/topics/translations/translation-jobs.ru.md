---
title: Задачи перевода
---

Задачи перевода автоматизируют перевод больших объемов контента. Вместо того чтобы вручную переводить продукты по одному, создайте задачу, которая будет переводить весь ваш каталог или определенные подмножества в фоновом режиме. Задачи выполняются асинхронно, поэтому вы можете продолжать работу, пока сотни или тысячи полей автоматически переводятся.

Используйте задачи перевода при активации новых языков, импорте новых продуктов или при устранении пробелов в не переведенном контенте.

## Что такое задачи перевода?

Задача перевода - это фоновая задача, которая:

1. **Сканирует контент** на наличие полей, подлежащих переводу (продукты, страницы, блог-посты и т.д.)
2. **Определяет непереведенные или устаревшие поля** на основе области действия задачи
3. **Отправляет поля в движок перевода** (локальная модель ИИ или внешний провайдер)
4. **Сохраняет переведенные поля** обратно в ваш контент
5. **Отчитывается о завершении** с статистикой по переведенным полям

Задачи выполняются через очередь задач Celery, поэтому они не блокируют ваш интерфейс администратора.

## Когда использовать задачи перевода

**Запуск нового языка**:
- Активируйте немецкий язык как новый язык
- Создайте задачу: перевести все продукты с английского на немецкий
- Результат: весь каталог доступен на немецком языке в течение минут/часов (в зависимости от размера)

**Импорт новых продуктов**:
- Импортируйте 500 новых продуктов на английском языке
- Создайте задачу: перевести новые продукты на все активные языки
- Результат: новый ассортимент сразу же доступен во всех регионах

**Устранение пробелов**:
- Отчет о покрытии показывает, что продукты переведены только на 60% на французский язык
- Создайте задачу: перевести недостающие поля продуктов на французский язык
- Результат: покрытие французским языком увеличивается до ~100%

**Обновление устаревших переводов**:
- Модель перевода улучшена или доступен новый провайдер
- Создайте задачу: пере-перевести все продукты на испанский язык
- Результат: повышение качества испанских переводов по всему каталогу

## Создание задачи перевода

Перейдите в **Настройки > Задачи перевода** и нажмите **+ Создать задачу**.

### Настройка задачи

**Название задачи** - описательная метка (например, "Перевести продукты на немецкий", "Новые блог-посты - все языки")

**Тип контента** - что переводить:
- Продукты
- Категории продуктов
- Страницы
- Блог-посты
- SEO-метаданные
- Шаблоны электронной почты
- Все типы контента

**Исходный язык** - язык, с которого производится перевод (обычно ваш язык по умолчанию)

**Целевой язык(ы)** - один или несколько языков, на которые производится перевод (выберите несколько для параллельного перевода)

**Область действия** - какое подмножество контента:
- **Все элементы** - перевести все, независимо от существующих переводов
- **Только непереведенные** - пропустить поля, которые уже имеют переводы
- **Создано/обновлено с даты** - только новые или недавно измененные элементы
- **Конкретные элементы** - выберите отдельные продукты/страницы (расширенный фильтр)

**Движок перевода** - какой сервис использовать:
- Локальная модель ИИ (по умолчанию, без затрат на API)
- Конкретный внешний провайдер (DeepL, Google, Azure, AWS)
- Автоматически выбрать (использует настроенную предпочтение)

**Заблокировать переводы** - блокировать переведенные поля от автоматического перезаписывания в будущем (полезно для проверенных переводов)

### Расширенные параметры

**Пропустить заблокированные поля** - если включено, уважает существующие заблокированные переводы (рекомендуется)

**Перезаписать существующие** - повторно перевести, даже если переводы уже существуют (используйте для улучшения качества)

**Фильтры полей** - переводить только определенные поля (например, названия и описания продуктов, пропустить атрибуты)

**Размер пакета** - сколько элементов обрабатывать за раз (по умолчанию: 50, увеличьте для более быстрой обработки, если сервер может это обрабатывать)

**Приоритет** - задачи высокого приоритета выполняются до задач с обычным приоритетом (используйте редко)

## Жизненный цикл и состояние задачи

Задачи проходят через следующие состояния:

**В очереди** - задача создана, ожидает, пока рабочий процесс не возьмет ее

**Обработка** - рабочий процесс активно переводит контент

**Завершено** - все переводы успешно завершены

**Ошибка** - задача столкнулась с ошибками (проверьте журнал ошибок)

**Отменено** - задача была вручную остановлена администратором

**Приостановлено** - временно приостановлено (можно возобновить)

## Отслеживание прогресса задачи

На странице деталей задачи отображается:

**Полоса прогресса** - процент выполнения

**Статистика**:
- Общее количество элементов для перевода
- Элементы, выполненные
- Оставшиеся элементы
- Оценочное время выполнения

**Журнал в реальном времени** - поток активности перевода (полезно для устранения неполадок)

**Количество ошибок** - сколько полей не удалось перевести (с указанием причин)

## Результаты и статистика задачи

Когда задача завершается, страница результатов показывает:

**Обзор**:
- Общее количество обработанных полей
- Успешно переведено
- Неудачные переводы
- Пропущено (уже переведено, заблокировано или исключено фильтрами)

**Разбивка по элементам**:
- Какие продукты/страницы были переведены
- Сколько полей на каждый элемент
- Любые возникшие ошибки

**Метрики производительности**:
- Общее время, прошедшее
- Среднее количество переводов в секунду
- Использованный движок перевода

## Обработка неудачных переводов

Если некоторые переводы неудачны:

**Проверьте журнал ошибок** - определяет, какие поля не удалось перевести и почему

**Частые причины сбоев**:
- Превышение лимита запросов API (внешний провайдер)
- Тайм-аут движка перевода (очень длинный текст)
- Недопустимый формат поля (ошибка парсинга JSON)
- Модель не поддерживает пару языков

**Варианты повтора**:
- Исправьте подлежащую решению проблему
- Создайте новую задачу только для неудачных элементов
- Используйте другой движок перевода

## Отмена и приостановка задач

**Отменить** - немедленно останавливает задачу, отбрасывает любые в процессе переводы (сохраненные переводы остаются)

**Приостановить** - временно останавливает задачу, можно возобновить позже с того места, где она остановилась

**Возобновить** - продолжает приостановленную задачу

Используйте приостановку/возобновление, когда вам нужно временно освободить ресурсы сервера.

## Стратегии массовых задач

**Стратегия 1: Язык за языком**:
- Создайте отдельные задачи для каждого целевого языка
- Проще отслеживать прогресс по каждому языку
- Можно приоритизировать важные языки
- Распределяет нагрузку со временем

**Стратегия 2: Все сразу**:
- Одна задача, переводящая на все активные языки
- Быстрее завершается в целом
- Высокая нагрузка на сервер во время обработки
- Проще управление задачами

**Стратегия 3: По типу контента**:
- Сначала переведите продукты (наивысший приоритет)
- Затем категории, страницы, блог-посты
- Позволяет постепенное внедрение
- Проще тестировать и проверять переводы

Выберите стратегию на основе пропускной способности вашего сервера, срочности и размера каталога.

## Планирование задач

Планируйте повторяющиеся задачи для автоматической обработки нового контента:

**Ежедневные задачи** - переводят продукты, созданные/обновленные в течение последних 24 часов

**Еженедельные задачи** - еженедельно устраняют пробелы в переводах

**После импорта** - автоматически запускает задачу после массового импорта продуктов

**При активации языка** - автоматически создает задачу при активации нового языка

Планированные задачи поддерживают актуальность переводов без вмешательства.

## Рекомендации по производительности

**Локальная модель ИИ**:
- ~100-500 переводов в секунду (зависит от сервера)
- Высокая нагрузка на процессор во время обработки
- Нет ограничений на частоту запросов API
- Бесплатно (без затрат на каждый перевод)

**Внешние провайдеры**:
- Ограничения на частоту запросов различаются (DeepL: 500k символов в месяц на бесплатном тарифе)
- Задержка API добавляет накладные расходы
- Лучшее качество, но с затратами
- Ограничения на одновременные запросы

**Большие задачи** (>10,000 полей):
- Запускайте во время непиковых часов
- Отслеживайте ресурсы сервера
- Рассмотрите разделение на более мелкие пакеты
- Сначала протестируйте на подмножестве

## Советы

- **Начните с малого** - протестируйте задачи на подмножестве (например, 10 продуктов) перед запуском перевода всего каталога
- **Используйте область действия "Только непереведенные"** - быстрее и избегает повторного перевода уже хорошего контента
- **Близко отслеживайте первую задачу** - следите за ошибками или проблемами качества перед запуском больших задач
- **Планируйте задачи во время низкого трафика** - перевод требует интенсивного использования процессора/API
- **Заблокируйте проверенные переводы** - предотвращает массовые задачи от перезаписи ваших вручную внесенных изменений
- **Держите задачи сфокусированными** - более мелкие, целевые задачи легче диагностировать, чем огромные задачи "перевести все"
- **Проверьте образцы после завершения** - проверьте случайные переводы на качество перед тем, как считать задачу успешной
- **Экспортируйте/создайте резервные копии перед крупными задачами** - на случай, если вам нужно отменить массовые изменения
