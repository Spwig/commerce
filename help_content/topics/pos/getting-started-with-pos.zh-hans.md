Spwig POS 可将任何平板电脑或浏览器转变为完整的店内收银机 —— 与您的产品目录、库存和订单历史连接。此检查表将引导您从全新安装到完成您的第一笔交易。每一步都链接到一个专门的主题，如果您需要完整详细信息，可以查看。

![POS仪表板](/static/core/admin/img/help/pos/getting-started-dashboard.webp)

## 第1步：为商店位置启用POS

POS终端与一个物理商店位置相关联。在Spwig中，商店位置是标记为零售位置的仓库。

1. 在您的管理侧边栏中导航到 **目录 > 仓库**。
2. 打开您要用作商店的仓库，或创建一个新的仓库。
3. 勾选 **零售位置** 开关，并输入一个 **POS显示名称**（例如，“High Street Store”）。此名称会显示在收据和终端选择器中。
4. 保存仓库。

如果您有多个商店或希望将它们分组以进行区域报告，请先在 **POS > 商店组** 创建一个 **商店组**，然后将每个仓库分配到该组。商店组允许您设置共享的货币、时区和收据模板，所有组内的位置都会继承这些设置。

## 第2步：创建或验证至少一个具有POS访问权限的员工账户

您的员工使用与Spwig管理后台相同的凭据登录POS。


# 第3步：注册您的第一个POS终端

一个终端代表一个单独的收银台或设备。您可以在管理员中注册它，然后使用一次性配对代码将物理设备与其配对。

1. 导航到 **POS > POS终端**，然后点击 **+ 添加POS终端**。
2. 为终端命名（例如，"前台收银台"），并将其分配给步骤1中启用的商店位置。
3. 保存终端。Spwig会生成一个 **8个字符的配对代码** — 您可以在终端的详细信息页面上看到它。
4. 在您要用作收银台的设备上，打开浏览器并前往 `/pos/`。
5. 当提示时输入配对代码。设备现在已与此终端链接。

配对代码仅限一次性使用。如果您需要重新配对设备，请在管理员中打开终端，然后点击 **重新生成配对代码**。

有关硬件配置选项（收据打印机、条码扫描器、钱箱），请参阅 [POS终端设置](pos-terminal-setup)。

## 第4步：配置支付提供商

支付提供商将您的读卡器连接到支付网络，例如Stripe Terminal或Square。使用5步设置向导输入您的凭据。

1. 导航到 **POS > 支付提供商**，然后点击 **配置提供商**。
2. 向导将在 `/admin/pos/terminal-provider/wizard/step1/` 处打开。

![支付提供商向导](/static/core/admin/img/help/pos/getting-started-provider-wizard-step1.webp)

3. 选择您的提供商（例如，**Stripe Terminal**），然后按照屏幕上的说明完成所有五个步骤：选择提供商 → 设置说明 → 输入凭据 → 测试连接 → 配置位置。
4. 绿色的 **已连接** 标签确认集成已上线。

如果你只需要现金和手动刷卡功能，请选择 **Manual** 作为提供商 — 不需要凭据。

如需查看每个支持提供商的详细凭据字段，请参阅 [POS 支付提供商设置](pos-payment-provider-setup)。

## 第 5 步：配对卡读卡器

在连接支付提供商后，你可以使用 3 步读卡器向导，将物理卡读卡器配对到你的终端之一。

1. 导航到 **POS > 卡读卡器**，然后点击 **添加读卡器**。
2. 读卡器向导从 `/admin/pos/reader/wizard/step1/` 开始。
3. 选择你的提供商，然后选择 **注册新设备**（输入读卡器屏幕上显示的代码）或 **发现现有设备**（Spwig 会获取已与提供商注册的读卡器）。
4. 在最后一步，将读卡器分配给你在第 3 步中创建的终端。

每个终端支持一个分配的卡读卡器。你可以随时从卡读卡器列表中重新分配读卡器。

## 第 6 步：设计你的收据（第一天可选）

Spwig 会自动生成默认的收据模板。你可以立即开始销售，无需对其进行任何操作 — 默认情况下会打印你的商店名称、地址、明细销售、支付方式以及一个“感谢您的购买！”的页脚。

当你准备好自定义时，请前往 **POS > 收据模板**。选项包括你的标志、税号、二维码推广、退货政策以及纸张宽度（适用于热敏打印机的 58mm 或 80mm）。你可以为每个商店或每个商店组创建不同的模板。

## 第 7 步：开启你的第一个班次

班次用于跟踪谁处理了销售以及抽屉中应有多少现金。收银员在 POS 本身上开启和关闭班次。

1. 在已配对的设备上，前往 `/pos/` 并使用员工凭据登录。
2. 选择终端和商店位置。
3. Spwig 会提示你 **清点初始零钱** — 输入抽屉中已有的现金金额（如果抽屉为空，请输入 `0`）。
4. 点击 **开启班次**。现在收银机已准备好进行销售。

有关班次、现金流动和对账报告的完整说明，请参阅[管理POS班次](pos-shifts)。

## 第8步：完成您的第一笔销售

一旦班次开启，销售操作就非常简单：

1. 通过名称搜索产品、扫描条形码或浏览分类，将商品添加到购物车。
2. 如有必要，应用折扣或优惠券代码。
3. 点击**Charge**开始支付。选择支付方式（现金、通过读卡器的银行卡，或分账支付）。
4. 对于银行卡支付，读卡器会提示顾客刷卡或插入卡片。
5. 收据会自动打印（或显示数字收据选项）。订单会实时保存到您的订单历史记录中。

## 第9步：在一天结束时关闭班次

关闭班次会锁定收银机并生成对账摘要。

1. 在POS菜单中，点击**Close Shift**。
2. 清点抽屉中的现金，并在提示时输入总额。
3. Spwig会根据班次开始时的备用金、现金销售以及班次期间的任何现金流动计算预期现金，并向您显示差异。
4. 确认关闭。班次报告将被保存，并在您的管理员界面中**POS > Shifts**下可见。

将一天中从抽屉中取出或添加的现金记录为**现金流动**（通过班次菜单），而不是调整关闭时的现金总额——这有助于保持对账的准确性。

## 小贴士

- 在营业的第一天之前，请完成步骤1到5。

步骤6到9可以在当天完成。
- 使用一个强大但容易记住的员工密码——POS员工在收银机上输入凭据，因此过于复杂的密码会减慢他们的操作。
- 如果卡读卡器未在线上显示，请点击**Sync readers**在卡读卡器页面上，从您的提供商处获取最新的状态。
- 在繁忙的营业期之前，使用$0.01的测试交易测试整个流程（开启班次 → 销售 → 收据 → 关闭班次）。
- POS支持离线操作以进行基本的现金销售。

刷卡终端支付需要互联网连接来授权。
- 您可以在一个商店位置拥有多个终端——在管理后台中添加一个新的终端记录，并将其配对到不同的设备。
