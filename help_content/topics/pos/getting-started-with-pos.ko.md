Spwig POS는 태블릿 또는 브라우저를 실제 매장의 전체 계산대로 변환할 수 있습니다. 제품 카탈로그, 재고, 주문 내역과 연결되어 있습니다. 이 확인 목록은 신규 설치부터 첫 번째 판매를 처리하는 과정을 안내합니다. 각 단계는 추가 정보가 필요할 경우 해당 주제에 링크되어 있습니다.

![POS 대시보드](/static/core/admin/img/help/pos/getting-started-dashboard.webp)

## 단계 1: 매장 위치에 POS 활성화

POS 단말기는 물리적인 매장 위치와 연결되어 있습니다. Spwig에서 매장 위치는 판매 위치로 표시된 창고입니다.

1. 관리자 사이드바에서 **Catalog > Warehouses**로 이동합니다.
2. 사용할 창고를 열거나 새 창고를 만듭니다.
3. **Retail location** 토글을 체크하고 **POS display name**을 입력합니다 (예: "High Street Store"). 이 이름은 영수증과 단말기 선택기에 표시됩니다.
4. 창고를 저장합니다.

여러 매장이 있거나 지역별 보고를 위해 그룹화하고 싶다면 먼저 **POS > Store Groups**에서 **Store Group**을 생성한 후 각 창고를 그룹에 할당합니다. 매장 그룹은 그룹 내 모든 위치가 상속받는 공유 통화, 시간대, 영수증 템플릿을 설정할 수 있습니다.

## 단계 2: POS 액세스가 있는 직원 계정을 최소 하나 생성하거나 확인합니다

직원은 Spwig 관리자에 로그인하는 동일한 자격 증명을 사용하여 POS에 로그인합니다.

# 단계 3: 첫 번째 POS 단말기 등록

단말기는 하나의 계산대 또는 장치를 나타냅니다. 먼저 관리자에서 단말기를 등록한 후, 일회용 쌍방향 코드를 사용하여 물리적 장치를 연결합니다.

1. **POS > POS 단말기**로 이동하고 **+ POS 단말기 추가**를 클릭합니다.
2. 단말기에 이름을 지정합니다(예: "전면 계산대") 그리고 단계 1에서 활성화한 매장 위치에 할당합니다.
3. 단말기를 저장합니다. Spwig은 **8자리 쌍방향 코드**를 생성합니다 — 단말기의 상세 페이지에서 이를 확인할 수 있습니다.
4. 계산대로 사용할 장치에서 브라우저를 열고 `/pos/`로 이동합니다.
5. 요청 시 쌍방향 코드를 입력합니다. 이제 장치가 이 단말기와 연결되었습니다.

쌍방향 코드는 일회용입니다. 장치를 다시 연결해야 하는 경우, 관리자에서 단말기를 열고 **쌍방향 코드 재생성**을 클릭합니다.

하드웨어 구성 옵션(영수증 프린터, 바코드 스캐너, 현금 캐비닛)은 [POS 단말기 설정](pos-terminal-setup)을 참조하십시오.

## 단계 4: 결제 제공업체 구성

결제 제공업체는 카드 리더기를 Stripe Terminal 또는 Square와 같은 결제 네트워크에 연결합니다. 5단계 설정 마법사를 사용하여 자격 증명을 입력합니다.

1. **POS > 결제 제공업체**로 이동하고 **제공업체 구성**을 클릭합니다.
2. 마법사는 `/admin/pos/terminal-provider/wizard/step1/`에서 열립니다.

![결제 제공업체 마법사](/static/core/admin/img/help/pos/getting-started-provider-wizard-step1.webp)

3. 제공업체(예: **Stripe Terminal**)를 선택하고 화면의 지시사항에 따라 5단계를 모두 수행합니다: 제공업체 선택 → 설정 지시사항 → 자격 증명 입력 → 연결 테스트 → 위치 구성.
4. 녹색 **연결됨** 배지가 통합이 완료되었음을 나타냅니다.

현금 및 수동 카드 입력만 필요하다면, 제공업체로 **Manual**을 선택하세요 — 자격 증명이 필요하지 않습니다.

각 지원되는 제공업체의 자격 증명 필드에 대한 자세한 내용은 [POS Payment Provider Setup](pos-payment-provider-setup)을 참조하세요.

## Step 5: 카드 리더기와 연결하기

결제 제공업체가 연결된 후, 3단계 리더기 마법사를 사용하여 물리적 카드 리더기를 터미널 중 하나에 연결할 수 있습니다.

1. **POS > Card Readers**로 이동하고 **Add reader**를 클릭합니다.
2. 리더기 마법사는 `/admin/pos/reader/wizard/step1/`에서 시작됩니다.
3. 제공업체를 선택한 후, **Register new device** (리더기 화면에 표시된 코드를 입력) 또는 **Discover existing** (Spwig이 제공업체에 이미 등록된 리더기를 가져옵니다)를 선택합니다.
4. 마지막 단계에서는 Step 3에서 생성한 터미널에 리더기를 할당합니다.

각 터미널은 하나의 할당된 카드 리더기를 지원합니다. 언제든지 Card Readers 목록에서 리더기를 재할당할 수 있습니다.

## Step 6: 영수증 디자인 (1일차에는 선택 사항)

Spwig은 기본 영수증 템플릿을 자동으로 생성합니다. 이를 만지지 않고 즉시 판매를 시작할 수 있습니다 — 기본 템플릿은 가게 이름, 주소, 항목별 판매 내역, 결제 방식, 그리고 "구매 감사합니다!"라는 푸터를 인쇄합니다.

사용자 정의를 준비할 때는 **POS > Receipt Templates**로 이동하세요. 옵션에는 로고, 세금 번호, QR 코드 홍보, 반품 정책, 종이 너비(열전도 프린터용 58mm 또는 80mm)가 포함됩니다. 매장별 또는 매장 그룹별로 별도의 템플릿을 생성할 수 있습니다.

## Step 7: 첫 번째 시프트 열기

시프트는 판매를 처리한 사람과 현금 캐비닛에 있어야 할 금액을 추적합니다. 직원은 POS 자체에서 시프트를 열고 닫습니다.

1. 연결된 장치에서 `/pos/`로 이동하고 직원 자격 증명으로 로그인합니다.
2. 터미널과 가게 위치를 선택합니다.
3. Spwig은 **시작 현금을 계수하라**라고 요청합니다 — 캐비닛에 이미 있는 현금 금액을 입력합니다(캐비닛이 비어 있다면 `0`을 입력합니다).
4. **Shift Open**을 탭합니다. 이제 판매가 준비되었습니다.

직접 판매 시프트, 현금 이동 및 대조 보고서에 대한 자세한 설명은 [Managing POS Shifts](pos-shifts)를 참조하세요.

## 단계 8: 첫 번째 판매를 완료하세요

시프트가 열려 있는 경우 판매는 간단합니다:

1. 제품 이름으로 검색하거나 바코드를 스캔하거나 카테고리를 탐색하여 장바구니에 항목을 추가합니다.
2. 필요한 경우 할인 또는 쿠폰 코드를 적용합니다.
3. **결제**를 탭하여 결제를 시작합니다. 결제 방식(현금, 리더를 통한 카드, 또는 분할 결제)을 선택합니다.
4. 카드 결제의 경우 리더가 고객에게 카드를 탭하거나 삽입하도록 요청합니다.
5. 영수증이 자동으로 인쇄되거나 디지털 영수증 옵션이 표시됩니다. 주문은 실시간으로 주문 내역에 저장됩니다.

## 단계 9: 영업일 종료 시 시프트를 종료하세요

시프트를 종료하면 계산대가 잠기고 대조 요약이 생성됩니다.

1. POS 메뉴에서 **시프트 종료**를 탭합니다.
2. 계산대에 있는 현금을 세고 프롬프트가 나타날 때 총액을 입력합니다.
3. Spwig는 시프트 시작 시의 초기 현금, 현금 판매 및 시프트 중의 현금 이동을 기반으로 예상 현금을 계산하고 차이를 표시합니다.
4. 확인하여 종료합니다. 시프트 보고서는 관리자 계정의 **POS > 시프트**에서 저장되고 표시됩니다.

하루 동안 계산대에서 현금을 제거하거나 추가한 경우, **현금 이동**으로 기록(시프트 메뉴를 통해)해야 하며, 종료 현금 수량을 조정하는 대신 이렇게 하면 대조가 정확하게 유지됩니다.

## 팁

- 거래 시작 전에 단계 1에서 5까지 완료하세요.

- 단계 6에서 9는 거래 당일에 수행할 수 있습니다.
- 강력하지만 기억하기 쉬운 직원 비밀번호를 사용하세요. POS 직원은 계산대에서 자격 증명을 입력하므로 지나치게 복잡한 비밀번호는 그들을 느리게 만듭니다.
- 카드 리더가 온라인 상태가 아닌 경우, 카드 리더 페이지에서 **리더 동기화**를 클릭하여 제공업체에서 최신 상태를 가져옵니다.
- 바쁜 거래 기간 전에 $0.01 테스트 거래를 사용하여 전체 흐름(시프트 열기 → 판매 → 영수증 → 시프트 종료)을 테스트하세요.
- POS는 기본적인 현금 판매에 대해 오프라인으로 작동합니다.

카드 단말기 결제는 인증을 위해 인터넷 연결이 필요합니다.
- 한 가게 위치에 여러 단말기를 가질 수 있습니다 — 관리자에서 새 단말기 기록을 추가하고 다른 장치에 연결하세요.
