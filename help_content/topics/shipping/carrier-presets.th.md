---
title: การตั้งค่าผู้ให้บริการจัดส่งล่วงหน้า
---

การตั้งค่าผู้ให้บริการจัดส่งล่วงหน้า กำหนดผู้ให้บริการจัดส่งแบบมือถือ (DHL, FedEx, UPS, ผู้ให้บริการแบบกำหนดเอง) สำหรับการจัดส่งที่สร้างโดยไม่มีการผสาน API — แต่ละการตั้งค่าจะให้โลโก้ผู้ให้บริการ แม่แบบ URL ติดตาม และการตั้งค่าการแสดงผล ตัวตั้งค่าระบบ (DHL, FedEx, UPS, USPS) ได้รับการตั้งค่าล่วงหน้าและไม่สามารถลบได้ ในขณะที่ตัวตั้งค่าแบบกำหนดเองอนุญาตให้ผู้ขายเพิ่มผู้ให้บริการจัดส่งในระดับภูมิภาคหรือเฉพาะทาง ตัวตั้งค่าจะเชื่อมโยงกับการจัดส่งแบบมือถือที่ผู้ขายป้อนหมายเลขติดตามด้วยตนเองแทนที่จะซื้อฉลากผ่าน API ของผู้ให้บริการ

ใช้ตัวตั้งค่าผู้ให้บริการเมื่อสร้างการจัดส่งแบบมือถือ หรือเมื่อคุณต้องการลิงก์ติดตามโดยไม่มีการผสาน API แบบเต็มรูปแบบ

## ตัวตั้งค่าระบบ vs ตัวตั้งค่าแบบกำหนดเอง

**ตัวตั้งค่าระบบ** (ติดตั้งไว้ล่วงหน้า):
- DHL, FedEx, UPS, USPS, Royal Mail, Canada Post, Australia Post
- ไม่สามารถลบได้ (is_system=True)
- สามารถเปลี่ยนแม่แบบ URL ติดตามหรือโลโก้ได้
- ให้แม่แบบ URL ติดตามเริ่มต้น

**ตัวตั้งค่าแบบกำหนดเอง** (สร้างโดยผู้ขาย):
- ผู้ให้บริการแบบภูมิภาค (OnTrac, LaserShip, บริการไปรษณีย์ในระดับภูมิภาค)
- ผู้ให้บริการแบบเฉพาะทาง (ขนส่งสินค้าขนาดใหญ่, การจัดส่งแบบมือถือ)
- สามารถแก้ไขหรือลบได้
- จำเป็นต้องใช้แม่แบบ URL ติดตามแบบกำหนดเอง

---

## การตั้งค่าตัวตั้งค่าผู้ให้บริการจัดส่ง

แต่ละตัวตั้งค่ากำหนด:

**การตั้งค่าพื้นฐาน**:
- **ชื่อ**: ชื่อแสดงผลของผู้ให้บริการ (เช่น, "DHL Express", "Local Courier")
- **รหัส**: ตัวระบุภายใน (เช่น, "dhl", "local_courier")
- **โลโก้**: ภาพโลโก้ของผู้ให้บริการ (ไม่จำเป็น, จะใช้ไอคอนหากไม่มีการอัปโหลด)
- **ไอคอน**: ไอคอน FontAwesome ใช้เป็นตัวเลือก (เช่น, "fa-truck")
- **เปิดใช้งาน**: เปิด/ปิดการแสดงผล

**การตั้งค่าการติดตาม**:
- **แม่แบบ URL การติดตาม**: รูปแบบ URL ที่มีตัวย่อ {tracking_id}
- **การติดตาม URL แบบกำหนดเอง**: URL แบบกำหนดเอง (จะทับการตั้งค่าเริ่มต้น)

**การตั้งค่าระบบ** (เฉพาะตัวตั้งค่าระบบ):
- **เป็นระบบ**: ไม่สามารถลบได้
- **เป็นค่าเริ่มต้น**: มีค่าเริ่มต้นหนึ่งค่าต่อประเภทผู้ให้บริการ

---

## แม่แบบ URL การติดตาม

URL การติดตามใช้ตัวย่อ {tracking_id}:

**ตัวอย่าง**:

DHL: `https://www.dhl.com/en/express/tracking.html?AWB={tracking_id}`

FedEx: `https://www.fedex.com/fedextrack/?tracknumbers={tracking_id}`

UPS: `https://www.ups.com/track?tracknum={tracking_id}`

USPS: `https://tools.usps.com/go/TrackConfirmAction?tLabels={tracking_id}`

แบบกำหนดเอง: `https://track.localcourier.com/tracking/{tracking_id}`

**วิธีการทำงาน**:
1. ผู้ขายสร้างการจัดส่งโดยใช้หมายเลขติดตาม "1234567890"
2. ระบบจะแทนที่ {tracking_id} ด้วยหมายเลขจริง
3. ลูกค้าคลิกลิงก์ติดตาม → ถูกเปลี่ยนเส้นทางไปยังหน้าผู้ให้บริการ
4. ผลลัพธ์: `https://www.dhl.com/en/express/tracking.html?AWB=1234567890`

---

## การสร้างตัวตั้งค่าผู้ให้บริการจัดส่งแบบกำหนดเอง

**ขั้นตอนทีละขั้นตอน**:

1. ไปที่ ตั้งค่า > การจัดส่ง > ตัวตั้งค่าผู้ให้บริการจัดส่ง
2. คลิก "เพิ่มตัวตั้งค่าผู้ให้บริการจัดส่ง"
3. ใส่ชื่อ (เช่น, "OnTrac")
4. ใส่รหัส (slug: "ontrac")
5. ไม่จำเป็น: อัปโหลดภาพโลโก้
6. เลือกไอคอน (fa-truck, fa-shipping-fast, ฯลฯ)
7. ใส่แม่แบบ URL การติดตามโดยใช้ {tracking_id}
8. เปิดใช้งาน = ใช่
9. บันทึก

**ตัวอย่าง - OnTrac**:
```
ชื่อ: OnTrac
รหัส: ontrac
URL การติดตาม: https://www.ontrac.com/tracking.asp?tracking_number={tracking_id}
ไอคอน: fa-truck
เปิดใช้งาน: ใช่
```

---

## การทับ URL การติดตามของตัวตั้งค่าระบบ

ตัวตั้งค่าระบบสามารถมีการทับ URL การติดตามได้:

**กรณีการใช้งาน**: บัญชีผู้ให้บริการของคุณมีพอร์ทัลติดตามแบบพิเศษ

**วิธีการทับ**:
1. แก้ไขตัวตั้งค่าระบบ (เช่น, DHL)
2. ใส่ URL ที่ทับในช่อง "URL การติดตามแบบกำหนดเอง"
3. การทับจะมีความสำคัญมากกว่าแม่แบบเริ่มต้น
4. บันทึก

**ตัวอย่าง**:
```
ระบบ: DHL
URL ค่าเริ่มต้น: https://www.dhl.com/en/express/tracking.html?AWB={tracking_id}
URL ที่ทับ: https://track.dhl.com/special-account/{tracking_id}
ผลลัพธ์: URL ที่ทับจะถูกใช้สำหรับการจัดส่ง DHL ทั้งหมด
```

---

## โลโก้ผู้ให้บริการ

**แนวทางการใช้โลโก้**:
- รูปแบบ: PNG หรือ SVG (แนะนำให้ใช้ SVG เพื่อการขยาย)
- ขนาด: แนะนำให้ใช้ขนาด 200×60px
- พื้นหลัง: โปร่งใสหรือสีขาว
- สี: สีเต็มรูปแบบของแบรนด์ผู้ให้บริการ

**ไอคอนสำรอง**:
หากไม่ได้อัปโหลดโลโก้ ระบบจะแสดงไอคอน FontAwesome:
- fa-truck (ค่าเริ่มต้น)
- fa-shipping-fast (แบบเร่งด่วน)
- fa-plane (การขนส่งทางอากาศ)
- fa-box (พัสดุ)

---

## การใช้ตัวตั้งค่าผู้ให้บริการใน การจัดส่ง

เมื่อสร้างการจัดส่งแบบมือถือ:

1. คำสั่งซื้อ > รายละเอียดคำสั่งซื้อ > สร้างการจัดส่ง
2. เลือกโหมด "การจัดส่งแบบมือถือ"
3. เลือกผู้ให้บริการจากกล่องแสดงตัวเลือกตัวตั้งค่าผู้ให้บริการ
4. ใส่หมายเลขติดตาม
5. ไม่จำเป็น: ทับ URL การติดตามสำหรับการจัดส่งนี้
6. บันทึก

**การแสดงผลการจัดส่ง**:
- แสดงโลโก้ผู้ให้บริการ (หรือไอคอน)
- แสดงหมายเลขติดตาม
- ลิงก์ติดตามที่คลิกได้ (ใช้แม่แบบ URL ตัวตั้งค่า)

---

## ผู้ให้บริการเริ่มต้น

สามารถตั้งค่าตัวตั้งค่าผู้ให้บริการหนึ่งตัวเป็นค่าเริ่มต้นต่อระบบ:

**กรณีการใช้งาน**: ผู้ให้บริการที่ใช้บ่อยที่สุดจะถูกเลือกโดยอัตโนมัติในระหว่างการสร้างการจัดส่ง

**วิธีตั้งค่า**:
1. แก้ไขตัวตั้งค่าผู้ให้บริการ
2. ตรวจสอบ "เป็นค่าเริ่มต้น"
3. บันทึก
4. ค่าเริ่มต้นก่อนหน้า (ถ้ามี) จะถูกยกเลิกโดยอัตโนมัติ

**อนุญาตให้ตั้งค่าค่าเริ่มต้นได้เพียงหนึ่งตัวเท่านั้น** — การตั้งค่าค่าเริ่มต้นใหม่จะลบค่าเริ่มต้นก่อนหน้า

---

## เคล็ดลับ

- **ใช้ชื่อที่อธิบายได้ชัดเจน** — "DHL Express" ดีกว่า "DHL"
- **ทดสอบ URL การติดตาม** — ตรวจสอบว่าแม่แบบทำงานได้กับหมายเลขติดตามจริง
- **อัปโหลดโลโก้ผู้ให้บริการ** — ดูเป็นมืออาชีพในอีเมลลูกค้า
- **อย่าลบตัวตั้งค่าระบบ** — มันได้รับการตั้งค่าล่วงหน้าอย่างถูกต้อง
- **ใช้การทับอย่างระมัดระวัง** — ใช้เพียงเมื่อผู้ให้บริการเปลี่ยนระบบติดตาม
- **ตั้งค่าค่าเริ่มต้นสำหรับผู้ให้บริการหลัก** — ช่วยประหยัดเวลาในการสร้างการจัดส่ง
- **เก็บตัวตั้งค่าให้เปิดใช้งานอยู่** — ปิดใช้งานก็ได้หากผู้ให้บริการยุติการให้บริการ
- **จัดทำเอกสารสำหรับผู้ให้บริการแบบกำหนดเอง** — เพิ่มหมายเหตุเกี่ยวกับผู้ให้บริการในระดับภูมิภาค
