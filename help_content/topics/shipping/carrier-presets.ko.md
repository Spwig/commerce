---
title: 운송사 사전 설정
---

운송사 사전 설정은 API 통합 없이 생성된 배송에 대해 수작업 운송사(DHL, FedEx, UPS, 사용자 정의 운송사)를 정의합니다. 각 사전 설정은 운송사 로고, 추적 URL 템플릿 및 표시 설정을 제공합니다. 시스템 사전 설정(DHL, FedEx, UPS, USPS)은 미리 구성되어 있으며 삭제할 수 없지만, 사용자 정의 사전 설정은 판매자가 지역 또는 전문 운송사를 추가할 수 있습니다. 사전 설정은 판매자가 추적 번호를 수작업으로 입력하는 수작업 배송과 연결됩니다.

API 통합이 완전하지 않은 경우 추적 링크가 필요한 경우 또는 수작업 배송을 생성할 때 운송사 사전 설정을 사용합니다.

## 시스템 사전 설정 vs 사용자 정의 사전 설정

**시스템 사전 설정**(사전 설치):
- DHL, FedEx, UPS, USPS, Royal Mail, Canada Post, Australia Post
- 삭제할 수 없음(is_system=True)
- 추적 URL 또는 로고를 재정의할 수 있음
- 기본 추적 URL 템플릿 제공

**사용자 정의 사전 설정**(판매자 생성):
- 지역 운송사(OnTrac, LaserShip, 지역 우체국)
- 전문 운송사(화물, 화이트 글로브 배송)
- 편집 또는 삭제 가능
- 수작업 추적 URL 템플릿 필요

---

## 운송사 사전 설정 구성

각 사전 설정은 다음과 같이 정의합니다:

**기본 설정**:
- **이름**: 운송사 표시 이름(예: "DHL Express", "지역 배송업체")
- **코드**: 내부 식별자(예: "dhl", "local_courier")
- **로고**: 운송사 로고 이미지(선택 사항, 제공되지 않으면 아이콘 사용)
- **아이콘**: FontAwesome 아이콘을 대체(예: "fa-truck")
- **활성**: 가시성 토글

**추적 구성**:
- **추적 URL 템플릿**: {tracking_id} 플레이스홀더가 있는 URL 패턴
- **추적 URL 재정의**: 사용자 정의 URL(기본 템플릿 재정의)

**시스템 설정**(시스템 사전 설정만):
- **시스템**: 삭제할 수 없음
- **기본값**: 운송사 유형별 하나의 기본값

---

## 추적 URL 템플릿

추적 URL은 {tracking_id} 플레이스홀더를 사용합니다:

**예시**:

DHL: `https://www.dhl.com/en/express/tracking.html?AWB={tracking_id}`

FedEx: `https://www.fedex.com/fedextrack/?tracknumbers={tracking_id}`

UPS: `https://www.ups.com/track?tracknum={tracking_id}`

USPS: `https://tools.usps.com/go/TrackConfirmAction?tLabels={tracking_id}`

사용자 정의: `https://track.localcourier.com/tracking/{tracking_id}`

**작동 방식**:
1. 판매자가 추적 번호 "1234567890"으로 배송을 생성합니다
2. 시스템은 {tracking_id}를 실제 번호로 대체합니다
3. 고객이 추적 링크를 클릭하면 운송사 사이트로 리디렉션됩니다
4. 결과: `https://www.dhl.com/en/express/tracking.html?AWB=1234567890`

---

## 사용자 정의 운송사 사전 설정 생성

**단계별**:

1. 설정 > 배송 > 운송사 사전 설정으로 이동
2. "운송사 사전 설정 추가"를 클릭
3. 이름 입력(예: "OnTrac")
4. 코드 입력(스러그: "ontrac")
5. 선택 사항: 로고 이미지 업로드
6. 아이콘 선택(fa-truck, fa-shipping-fast 등)
7. {tracking_id}를 포함한 추적 URL 템플릿 입력
8. 활성 토글 = 예
9. 저장

**예시 - OnTrac**:
```
이름: OnTrac
코드: ontrac
추적 URL: https://www.ontrac.com/tracking.asp?tracking_number={tracking_id}
아이콘: fa-truck
활성: 예
```

## 시스템 사전 설정의 추적 URL 재정의

시스템 사전 설정은 추적 URL 재정의가 가능합니다:

**사용 사례**: 운송사 계정에 특별한 추적 포털이 있습니다

**재정의 방법**:
1. 시스템 사전 설정 편집(예: DHL)
2. "추적 URL 재정의" 필드에 재정의 URL 입력
3. 재정의는 기본 템플릿보다 우선시 됩니다
4. 저장

**예시**:
```
시스템: DHL
기본 URL: https://www.dhl.com/en/express/tracking.html?AWB={tracking_id}
재정의 URL: https://track.dhl.com/special-account/{tracking_id}
결과: 모든 DHL 배송에 재정의 URL 사용
```

## 운송사 로고

**로고 가이드라인**:
- 형식: PNG 또는 SVG(SVG가 확장성 때문에 선호됨)
- 크기: 200×60px 권장
- 배경: 투명 또는 하얀색
- 색상: 전체 색상의 운송사 브랜딩

**대체 아이콘**:
로고가 업로드되지 않은 경우 시스템은 FontAwesome 아이콘을 표시합니다:
- fa-truck(기본)
- fa-shipping-fast(특급)
- fa-plane(항공 화물)
- fa-box(택배)

## 운송사 사전 설정을 사용한 배송 생성

수작업 배송을 생성할 때:

1. 주문 > 주문 상세 > 배송 생성
2. "수작업 배송" 모드 선택
3. 사전 설정 드롭다운에서 운송사 선택
4. 추적 번호 입력
5. 선택 사항: 이 배송의 추적 URL 재정의
6. 저장

**배송 표시**:
- 운송사 로고 표시(또는 아이콘)
- 추적 번호 표시
- 클릭 가능한 추적 링크(사전 설정 URL 템플릿 사용)

---

## 기본 운송사

시스템당 하나의 사전 설정을 기본값으로 설정할 수 있습니다:

**사용 사례**: 대부분 사용되는 운송사가 배송 생성 시 자동으로 선택됩니다

**설정 방법**:
1. 운송사 사전 설정 편집
2. "기본값" 체크
3. 저장
4. 이전 기본값(있는 경우) 자동 해제

**기본값은 하나만 허용** - 새 기본값을 설정하면 이전 기본값 플래그가 제거됩니다.

---

## 팁

- **설명적인 이름 사용** - "DHL Express"가 "DHL"보다 좋음
- **추적 URL 테스트** - 실제 추적 번호로 템플릿이 작동하는지 확인
- **운송사 로고 업로드** - 고객 이메일에서 전문적인 외관 제공
- **시스템 사전 설정 삭제 금지** - 정확하게 사전 구성되어 있음
- **재정의는 적게 사용** - 운송사가 추적 시스템을 변경할 때만 사용
- **주요 운송사에 기본값 설정** - 배송 생성 시 시간 절약
- **사전 설정은 활성 상태 유지** - 운송사가 중단될 경우에만 비활성화
- **사용자 정의 운송사에 대한 문서화** - 지역 운송사에 대한 메모 추가

기억하세요: 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어는 보존 규칙에 따라 정확히 유지합니다.
