# 梱包の設定

梱包は、料金計算および自動梱包用に事前に定義された箱や封筒のサイズを定義します。内部寸法（使用可能なスペース）、壁厚み（外部寸法、運送会社API用）、重量制限、梱包コストを指定します。運送会社は外部寸法を使用して、正確な料金提示用の寸法重量を計算します。梱包には、カート商品に最適な梱包組み合わせを自動選択するためのバインパッキングアルゴリズムの優先順位が設定されています。

カスタムAPIを使用してリアルタイム料金を取得するか、正確な寸法重量計算が必要な場合に梱包を設定します。

## 梱包設定

各梱包は以下を定義します：

**寸法**:
- **内部長さ**: 使用可能なスペース内 (cm)
- **内部幅**: 使用可能なスペース内 (cm)
- **内部高さ**: 使用可能なスペース内 (cm)
- **壁厚み**:梱包材の厚み (cm)

**外部寸法** (自動計算):
```
外部長さ = 内部長さ + (2 × 壁厚み)
外部幅 = 内部幅 + (2 × 壁厚み)
外部高さ = 内部高さ + (2 × 壁厚み)
```

**重量とコスト**:
- **空梱包重量**: 空の梱包の重量 (グラム)
- **最大重量**: 最大容量 (グラム)
- **コスト**:梱包材コスト (コスト最適化用)

**プロパティ**:
- **名前**: 梱包識別子 (例: "小箱", "大封筒")
- **タイプ**: 箱 or 封筒
- **優先度**: 自動梱包選択順序 (数字が小さいほど優先度が高くなります)
- **有効**: 可視性の切り替え

---

## 外部寸法の重要性

運送会社は外部寸法から**寸法重量**を計算します：

**寸法重量計算式**:
```
寸法重量 = (長さ × 幅 × 高さ) / 除数

一般的な除数:
- DHL: 5000
- FedEx/UPS: 5000 (国内), 6000 (国際)
```

**例**:
```
小箱:
内部: 20cm × 15cm × 10cm
壁厚み: 0.5cm
外部: 21cm × 16cm × 11cm

寸法重量 = (21 × 16 × 11) / 5000 = 0.74kg

実際の重量 = 0.5kg → 運送会社は0.74kg（寸法重量が高いため）で請求
```

**正確性の重要性**: 不正確な寸法 → 違った料金提示 → 顧客が過剰請求または不足請求される可能性があります。

## 一般的な梱包サイズ

### 小サイズの緩衝封筒

```
内部: 25cm × 18cm × 2cm
壁厚み: 0.3cm
最大重量: 500g
タイプ: 封筒
使用: 文書、本、ジュエリー
```

### 小箱

```
内部: 20cm × 15cm × 10cm
壁厚み: 0.5cm
最大重量: 5kg
タイプ: 箱
使用: 小型電子機器、化粧品、アクセサリー
```

### 中箱

```
内部: 30cm × 25cm × 20cm
壁厚み: 0.5cm
最大重量: 15kg
タイプ: 箱
使用: 衣類、靴、キッチン用品
```

### 大箱

```
内部: 45cm × 35cm × 30cm
壁厚み: 0.6cm
最大重量: 30kg
タイプ: 箱
使用: 複数品目、大量の商品、大型電子機器
```

## 自動梱包アルゴリズム

システムはカートアイテムに対して自動的に梱包を選択します：

**動作方法**:
1. カートアイテムの総体積を計算
2. 梱包を優先度でソート（数字が小さい順）
3. 品物を単一梱包に詰める
4. 詰められなかった場合、次の梱包サイズを試す
5. 単一梱包に詰められない場合、複数梱包を組み合わせる
6. `optimize_for`設定に基づいて最適化

**最適化モード**:
- **コスト**:梱包コストを最小化
- **体積**: 空間の無駄を最小化
- **数量**: 梱包数量を最小化

**例**:
```
カートアイテム:
- アイテムA: 10cm × 8cm × 5cm, 200g
- アイテムB: 15cm × 12cm × 8cm, 400g

梱包（優先度順）:
1. 小箱 (20×15×10, 優先度=1)
2. 中箱 (30×25×20, 優先度=2)

アルゴリズム:
小箱を試す: 両方の品物が入る
結果: 1×小箱（数量最適化）
```

---

## 梱包優先度

**優先度が梱包順序を決定します**:

優先度1（最も高い）: 小型梱包を最初に試す
優先度10: 大型梱包は最後の手段

**戦略**:
- 小型梱包 = 低い優先度数字（1-3）
- 中型梱包 = 中程度の優先度（4-6）
- 大型梱包 = 高い優先度数字（7-10）

**理由**: 最小の梱包から始めて必要に応じて拡大 → 配送コストを最小化します。

---

## 壁厚みの正確性

実際の梱包を測定してください：

**測定方法**:
1. 空の箱を取得
2. 内部寸法（内部）を測定
3. 外部寸法（外部）を測定
4. 計算: `(外部 - 内部) / 2 = 壁厚み`

**例**:
```
内部幅: 20cm
外部幅: 21cm
壁厚み: (21 - 20) / 2 = 0.5cm
```

**一般的な厚み**:
- 緩衝封筒: 0.2-0.4cm
- 単層紙パッケージ: 0.4-0.6cm
- 二層紙パッケージ: 0.8-1.0cm

## 梱包プリセットの作成

**ステップバイステップ**:

1. 設定 > 配送 > 梱包の設定
2. 「配送梱包を追加」をクリック
3. 名前を入力（例: "中箱"）
4. タイプを選択（箱 or 封筒）
5. 内部寸法を入力（L × W × H cm）
6. 壁厚みを入力（cm）
7. システムが外部寸法を自動計算
8. 空梱包重量を入力（グラム）
9. 最大重量を入力（容量）
10. オプション: コストを入力（コスト最適化用）
11. 優先度を設定（1-10）
12. 有効を切り替え = はい
13. 保存

---

## 梱包選択のテスト

**手動テスト**:
1. テストカートに商品を追加
2. 支払いに進む
3. リアルタイム配送方法を選択（梱包を使用）
4. 適切な料金が返されることを確認
5. 運送会社の応答を確認（APIログに選択された梱包が表示）

**自動梱包プレビュー**:
- 一部の配送会社アカウントでは梱包の詳細が表示されます
- カートに選択された梱包を確認
- 最適な梱包が選択されていることを確認

---

## ヒント

- **正確に測定** - 不正確な寸法 → 違った料金提示
- **壁厚みを含める** - 寸法重量の計算に不可欠
- **3-4サイズから始める** - 小型、中型、大型でほとんどのシナリオをカバー
- **現実的な最大重量を設定** - 箱の容量、理論的限界ではない
- **優先度を賢く設定** - 小型箱は優先度1、大型箱は優先度10
- **実際の製品でテスト** - 自動梱包が正しいサイズを選択するか確認
- **梱包が変更されたら更新** - 新しいサプライヤー = 再測定
- **特殊品を考慮** - 破損防止が必要な品は特定の箱サイズが必要
- **有効な梱包を最小限に保つ** - 過多なオプションは自動梱包アルゴリズムを遅くする
- **梱包を記録** - どの製品がどの梱包に合うかメモ
