# 运输包装

运输包装定义了预设的盒子和信封尺寸，用于运费计算和自动打包——指定内部尺寸（可用空间）、壁厚（外部尺寸用于承运商API）、重量限制和包装成本。承运商使用外部尺寸计算体积重量以获得准确的运费报价。包装具有优先级排序，用于自动打包算法，该算法会自动选择最佳包装组合以适应购物车商品。

在使用承运商API进行实时运费计算或需要精确体积重量计算时配置包装。

## 包装配置

每个包装定义：

**尺寸**：
- **内部长度**：内部可用空间（厘米）
- **内部宽度**：内部可用空间（厘米）
- **内部高度**：内部可用空间（厘米）
- **壁厚**：包装材料厚度（厘米）

**外部尺寸**（自动计算）：
```
外部长度 = 内部长度 + (2 × 壁厚)
外部宽度 = 内部宽度 + (2 × 壁厚)
外部高度 = 内部高度 + (2 × 壁厚)
```

**重量与成本**：
- **空箱重量**：空包装重量（克）
- **最大重量**：最大负载能力（克）
- **成本**：包装材料成本（用于成本优化）

**属性**：
- **名称**：包装标识符（例如，"小型纸箱"，"大型信封"）
- **类型**：纸箱或信封
- **优先级**：自动打包选择顺序（数字越低优先级越高）
- **激活**：切换可用性

---

## 为什么外部尺寸重要

承运商从外部尺寸计算**体积重量**：

**体积重量公式**：
```
体积重量 = (长 × 宽 × 高) / 除数

常见除数：
- DHL：5000
- FedEx/UPS：5000（国内），6000（国际）
```

**示例**：
```
小型纸箱：
内部：20cm × 15cm × 10cm
壁厚：0.5cm
外部：21cm × 16cm × 11cm

体积重量 = (21 × 16 × 11) / 5000 = 0.74kg

如果实际重量 = 0.5kg → 承运商按0.74kg（体积重量更高）收费
```

**为什么准确性重要**：尺寸不准确 → 运费报价错误 → 客户被多收费或少收费。

## 自动打包算法

系统会自动为购物车商品选择包装：

**工作原理**：
1. 计算购物车商品总体积
2. 按优先级对包装排序（数字最小优先）
3. 尝试将商品装入单个包装
4. 如果装不下，尝试下一个包装尺寸
5. 如果没有单个包装能装下，组合多个包装
6. 根据 `optimize_for` 设置进行优化

**优化模式**：
- **成本**：最小化包装成本
- **体积**：最小化空间浪费
- **数量**：最小化包装数量

**示例**：
```
购物车商品：
- 商品A：10cm × 8cm × 5cm，200g
- 商品B：15cm × 12cm × 8cm，400g

包装（按优先级）：
1. 小型纸箱（20×15×10，优先级=1）
2. 中型纸箱（30×25×20，优先级=2）

算法：
尝试小型纸箱：两个商品都能装下
结果：1×小型纸箱（按数量优化）
```

---

## 包装优先级

**优先级决定打包顺序**：

优先级1（最高）：先尝试小型包装
优先级10：大型包装作为最后选择

**策略**：
- 小型包装 = 低优先级数字（1-3）
- 中型包装 = 中等优先级（4-6）
- 大型包装 = 高优先级数字（7-10）

**原因**：从小型包装开始，按需扩展 → 最小化运费成本。

---

## 壁厚准确性

测量实际包装：

**测量方法**：
1. 取一个空箱子
2. 测量内部尺寸（内部）
3. 测量外部尺寸（外部）
4. 计算：`(外部 - 内部) / 2 = 壁厚`

**示例**：
```
内部宽度：20cm
外部宽度：21cm
壁厚：(21 - 20) / 2 = 0.5cm
```

**常见壁厚**：
- 加垫信封：0.2-0.4cm
- 单层纸板：0.4-0.6cm
- 双层纸板：0.8-1.0cm

## 创建包装预设

**逐步操作**：

1. 设置 > 运输 > 运输包装
2. 点击“添加运输包装”
3. 输入名称（例如，“中型纸箱”）
4. 选择类型（纸箱或信封）
5. 输入内部尺寸（L × W × H 以厘米为单位）
6. 输入壁厚（厘米）
7. 系统自动计算外部尺寸
8. 输入空箱重量（空包装重量以克为单位）
9. 输入最大重量（负载能力以克为单位）
10. 可选：输入成本（用于成本优化）
11. 设置优先级（1-10）
12. 切换激活 = 是
13. 保存

2. 进入结账流程

4. 验证返回的运费是否合理

**自动打包预览**：

- 查看购物车所选的包装
- 验证最佳打包

## 小贴士

- **准确测量** - 尺寸不准确 → 承运商运费错误
- **包含壁厚** - 体积重量的关键
- **从3-4种尺寸开始** - 小型、中型、大型覆盖大多数场景
- **设置现实的最大重量** - 包装容量，而非理论极限
- **明智地设置优先级** - 小型纸箱优先级1，大型纸箱优先级10
- **用实际商品测试** - 验证自动打包选择正确尺寸
- **包装变更时更新** - 新供应商 = 重新测量尺寸
- **考虑特殊商品** - 易碎品可能需要特定尺寸的纸箱
- **保持激活包装最小化** - 太多选项会减慢自动打包算法
- **记录包装信息** - 注明哪些商品适合哪些包装
