# 配送業者テンプレート

配送業者テンプレートは、API統合なしで作成された出荷物用の手動配送業者（DHL、FedEx、UPS、カスタム配送業者）を定義します。各テンプレートは配送業者のロゴ、追跡URLテンプレート、および表示設定を提供します。システムテンプレート（DHL、FedEx、UPS、USPS）は事前に構成されており、削除できませんが、カスタムテンプレートは販売業者が地域または専門的な配送業者を追加できるようにします。テンプレートは、販売業者が手動で追跡番号を入力する手動出荷に関連付けられます。

手動出荷を作成するか、完全なAPI統合なしで追跡リンクが必要な場合に配送業者テンプレートを使用してください。

## システムテンプレートとカスタムテンプレートの違い

**システムテンプレート**（事前インストール）:
- DHL、FedEx、UPS、USPS、Royal Mail、Canada Post、Australia Post
- 削除できません（is_system=True）
- 追跡URLまたはロゴを上書きできます
- デフォルトの追跡URLテンプレートが提供されます

**カスタムテンプレート**（販売業者が作成）:
- 地域配送業者（OnTrac、LaserShip、地域郵便局）
- 専門配送業者（貨物、白手袋配送）
- 編集または削除可能
- 手動で追跡URLテンプレートが必要

---

## 配送業者テンプレートの設定

各テンプレートは以下を定義します:

**基本設定**:
- **名前**: 配送業者の表示名（例: "DHL Express", "ローカルカーシア"）
- **コード**: 内部識別子（例: "dhl", "local_courier"）
- **ロゴ**: 配送業者ロゴ画像（任意、指定がない場合はアイコンを使用）
- **アイコン**: FontAwesomeアイコン（代替）（例: "fa-truck"）
- **有効**: 視覚表示の切り替え

**追跡設定**:
- **追跡URLテンプレート**: {tracking_id} プレースホルダーを含むURLパターン
- **追跡URL上書き**: カスタムURL（デフォルトテンプレートを上書き）

**システム設定**（システムテンプレートのみ）:
- **システム**: 削除できません
- **デフォルト**: 各配送業者タイプごとに1つ

---

## 追跡URLテンプレート

追跡URLは{tracking_id}プレースホルダーを使用します:

**例**:

DHL: `https://www.dhl.com/en/express/tracking.html?AWB={tracking_id}`

FedEx: `https://www.fedex.com/fedextrack/?tracknumbers={tracking_id}`

UPS: `https://www.ups.com/track?tracknum={tracking_id}`

USPS: `https://tools.usps.com/go/TrackConfirmAction?tLabels={tracking_id}`

カスタム: `https://track.localcourier.com/tracking/{tracking_id}`

**仕組み**:
1. 販売業者が追跡番号 "1234567890" で出荷を作成
2. システムは{tracking_id}を実際の番号に置き換え
3. 顧客が追跡リンクをクリック → 配送業者のサイトにリダイレクト
4. 結果: `https://www.dhl.com/en/express/tracking.html?AWB=1234567890`

## カスタム配送業者テンプレートの作成

**ステップバイステップ**:

1. **設定 > 配送 > 配送業者テンプレート**に移動
2. "配送業者テンプレートを追加"をクリック
3. 名前を入力（例: "OnTrac"）
4. コードを入力（スラッグ: "ontrac"）
5. 任意: ロゴ画像をアップロード
6. アイコンを選択（fa-truck、fa-shipping-fastなど）
7. {tracking_id}を含む追跡URLテンプレートを入力
8. 有効 = はい
9. 保存

**例 - OnTrac**:
```
名前: OnTrac
コード: ontrac
追跡URL: https://www.ontrac.com/tracking.asp?tracking_number={tracking_id}
アイコン: fa-truck
有効: はい
```

## システムテンプレートの追跡URLを上書き

システムテンプレートには追跡URLの上書きが可能です:

**使用例**: あなたの配送業者アカウントに専用の追跡ポータルがある場合

**上書き方法**:
1. システムテンプレートを編集（例: DHL）
2. "追跡URL上書き"フィールドに上書きURLを入力
3. 上書きURLはデフォルトテンプレートを上回ります
4. 保存

**例**:
```
システム: DHL
デフォルトURL: https://www.dhl.com/en/express/tracking.html?AWB={tracking_id}
上書きURL: https://track.dhl.com/special-account/{tracking_id}
結果: 上書きURLがDHL出荷すべてに使用されます
```

## 配送業者ロゴ

**ロゴガイドライン**:
- 形式: PNGまたはSVG（拡大縮小のためSVGが推奨）
- サイズ: 200×60pxが推奨
- 背景色: 透明または白
- 色: フルカラーの配送業者ブランド

**代替アイコン**:
ロゴがアップロードされていない場合、システムはFontAwesomeアイコンを表示します:
- fa-truck（デフォルト）
- fa-shipping-fast（エクスプレス）
- fa-plane（航空貨物）
- fa-box（荷物）

---

## 配送業者テンプレートの使用

手動出荷を作成する場合:

1. 注文 > 注文詳細 > 出荷を作成
2. "手動出荷"モードを選択
3. プレセットドロップダウンから配送業者を選択
4. 追跡番号を入力
5. 任意: この出荷用の追跡URLを上書き
6. 保存

**出荷表示**:
- 配送業者ロゴが表示されます（またはアイコン）
- 追跡番号が表示されます
- クリック可能な追跡リンク（テンプレートURLを使用）

---

## デフォルト配送業者

システムごとに1つのテンプレートをデフォルトに設定できます:

**使用例**: 最もよく使用される配送業者が出荷作成時に自動選択される

**デフォルトを設定する方法**:
1. 配送業者テンプレートを編集
2. "デフォルト"をチェック
3. 保存
4. 以前のデフォルト（存在する場合）が自動的に解除されます

**1つのデフォルトのみ許可** - 新しいデフォルトを設定すると、以前のデフォルトフラグが解除されます。

---

## ヒント

- **説明的な名前を使用** - "DHL Express" は "DHL" よりも良い
- **追跡URLをテスト** - 実際の追跡番号でテンプレートが動作するか確認
- **配送業者ロゴをアップロード** - 顧客メールで専門的な外観を提供
- **システムテンプレートを削除しない** - これらは事前に正しく構成されています
- **上書きは慎重に使用** - 配送業者が追跡システムを変更した場合のみ
- **主要な配送業者のデフォルトを設定** - 出荷作成時に時間を節約
- **テンプレートを有効に保つ** - 配送業者が廃止された場合のみ無効化
- **カスタム配送業者を文書化** - 地域配送業者に関するメモを追加
