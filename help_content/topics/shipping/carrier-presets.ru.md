---
title: Настройки доставщиков
---

Настройки доставщиков определяют ручные доставщики (DHL, FedEx, UPS, пользовательские доставщики) для отправлений, созданных без интеграции API — каждая настройка предоставляет логотип доставщика, шаблон URL-адреса отслеживания и настройки отображения. Системные настройки (DHL, FedEx, UPS, USPS) предварительно настроены и не могут быть удалены, в то время как пользовательские настройки позволяют торговцам добавлять региональные или специализированные доставщики. Настройки связываются с ручными отправлениями, где торговцы вручную вводят номера отслеживания вместо покупки этикеток через API-интерфейсы поставщиков.

Используйте настройки доставщиков при создании ручных отправлений или когда вы хотите получить ссылки на отслеживание без полной интеграции API.

## Системные настройки vs пользовательские настройки

**Системные настройки** (Предустановленные):
- DHL, FedEx, UPS, USPS, Royal Mail, Canada Post, Australia Post
- Не могут быть удалены (is_system=True)
- Можно изменить шаблон URL-адреса отслеживания или логотип
- Предоставляются стандартные шаблоны URL-адресов отслеживания

**Пользовательские настройки** (Созданные торговцем):
- Региональные доставщики (OnTrac, LaserShip, региональные почтовые службы)
- Специализированные доставщики (грузовые перевозки, доставка с белыми перчатками)
- Можно редактировать или удалить
- Требуется вручную ввести шаблон URL-адреса отслеживания

## Настройка настроек доставщиков

Каждая настройка определяет:

**Базовые настройки**:
- **Название**: Отображаемое имя доставщика (например, "DHL Express", "Локальный курьер")
- **Код**: Внутренний идентификатор (например, "dhl", "local_courier")
- **Логотип**: Изображение логотипа доставщика (необязательно, используется иконка, если не предоставлено)
- **Иконка**: Иконка FontAwesome в качестве резервного варианта (например, "fa-truck")
- **Активный**: Переключатель видимости

**Настройка отслеживания**:
- **Шаблон URL-адреса отслеживания**: URL-шаблон с заполнителем {tracking_id}
- **Переопределение URL-адреса отслеживания**: Пользовательский URL (переопределяет стандартный шаблон)

**Системные настройки** (только для системных настроек):
- **Системная**: Не может быть удалена
- **По умолчанию**: Один по умолчанию на тип доставщика

## Шаблоны URL-адресов отслеживания

URL-адреса отслеживания используют заполнитель {tracking_id}:

**Примеры**:

DHL: `https://www.dhl.com/en/express/tracking.html?AWB={tracking_id}`

FedEx: `https://www.fedex.com/fedextrack/?tracknumbers={tracking_id}`

UPS: `https://www.ups.com/track?tracknum={tracking_id}`

USPS: `https://tools.usps.com/go/TrackConfirmAction?tLabels={tracking_id}`

Пользовательский: `https://track.localcourier.com/tracking/{tracking_id}`

**Как это работает**:
1. Торговец создает отправление с номером отслеживания "1234567890"
2. Система заменяет {tracking_id} на фактический номер
3. Клиент нажимает на ссылку отслеживания → перенаправляется на сайт доставщика
4. Результат: `https://www.dhl.com/en/express/tracking.html?AWB=1234567890`

## Создание пользовательской настройки доставщика

**Пошаговая инструкция**:

1. Перейдите в Настройки > Доставка > Настройки доставщика
2. Нажмите "Добавить настройку доставщика"
3. Введите имя (например, "OnTrac")
4. Введите код (slug: "ontrac")
5. Необязательно: загрузите изображение логотипа
6. Выберите иконку (fa-truck, fa-shipping-fast и т.д.)
7. Введите шаблон URL-адреса отслеживания с {tracking_id}
8. Переключите активный = Да
9. Сохраните

**Пример - OnTrac**:
```
Название: OnTrac
Код: ontrac
URL отслеживания: https://www.ontrac.com/tracking.asp?tracking_number={tracking_id}
Иконка: fa-truck
Активный: Да
```

## Переопределение URL-адреса отслеживания системной настройки

Системные настройки могут иметь переопределение URL-адреса отслеживания:

**Сценарий использования**: Учетная запись вашего доставщика имеет специальный портал отслеживания

**Как переопределить**:
1. Редактируйте системную настройку (например, DHL)
2. Введите переопределенный URL в поле "Переопределение URL-адреса отслеживания"
3. Переопределение имеет приоритет над стандартным шаблоном
4. Сохраните

**Пример**:
```
Система: DHL
Стандартный URL: https://www.dhl.com/en/express/tracking.html?AWB={tracking_id}
Переопределенный URL: https://track.dhl.com/special-account/{tracking_id}
Результат: Переопределенный URL используется для всех отправлений DHL
```

## Логотипы доставщиков

**Рекомендации по логотипам**:
- Формат: PNG или SVG (SVG предпочтительнее из-за масштабируемости)
- Размер: рекомендуется 200×60px
- Фон: прозрачный или белый
- Цвет: полный цвет брендинга доставщика

**Резервная иконка**:
Если не загружен логотип, система отображает иконку FontAwesome:
- fa-truck (по умолчанию)
- fa-shipping-fast (экспресс-доставка)
- fa-plane (авиакурьерские перевозки)
- fa-box (посылка)

## Использование настроек доставщиков в отправлениях

При создании ручной отправки:

1. Заказы > Детали заказа > Создать отправление
2. Выберите режим "Ручная отправка"
3. Выберите доставщика из выпадающего списка настроек
4. Введите номер отслеживания
5. Необязательно: переопределите URL-адрес отслеживания для этой отправки
6. Сохраните

**Отображение отправки**:
- Показывается логотип доставщика (или иконка)
- Отображается номер отслеживания
- Кликаемая ссылка отслеживания (использует шаблон URL-адреса настройки)

## Стандартный доставщик

Одну настройку можно установить как стандартную для системы:

**Сценарий использования**: Наиболее часто используемый доставщик автоматически выбирается при создании отправки

**Как установить**:
1. Редактируйте настройку доставщика
2. Отметьте "По умолчанию"
3. Сохраните
4. Предыдущий стандартный (если был) автоматически снимается

**Разрешено только одно стандартное значение** — установка нового стандарта удаляет предыдущий флаг стандарта.

## Советы

- **Используйте описательные названия** — "DHL Express" лучше, чем "DHL"
- **Проверьте URL-адреса отслеживания** — Убедитесь, что шаблон работает с реальными номерами отслеживания
- **Загрузите логотипы доставщиков** — Профессиональный вид в электронных письмах клиентов
- **Не удаляйте системные настройки** — Они правильно предварительно настроены
- **Используйте переопределение с осторожностью** — Только когда доставщик меняет систему отслеживания
- **Установите стандартный для основного доставщика** — Экономит время при создании отправки
- **Оставляйте настройки активными** — Отключайте только если доставщик прекращает деятельность
- **Документируйте пользовательские доставщики** — Добавьте примечания о региональных доставщиках
