---
title: 배송 포장
---

배송 포장은 요금 계산 및 자동 포장에 사용되는 사전 정의된 상자 및 봉투 크기를 정의합니다. 내부 차원(사용 가능한 공간), 벽 두께(배송 업체 API에 사용되는 외부 차원), 무게 한계, 포장 비용을 지정합니다. 운송업체는 외부 차원을 기반으로 차원 무게를 계산하여 정확한 요금 견적을 제공합니다. 포장은 자동 포장 알고리즘에서 사용되는 우선순위 순서를 가집니다. 이 알고리즘은 장바구니 상품에 적합한 최적의 포장 조합을 자동으로 선택합니다.

운송업체 API를 사용하여 실시간 요금을 사용하거나 정확한 차원 무게 계산이 필요한 경우 포장을 구성합니다.

## 포장 구성

각 포장은 다음과 같이 정의합니다:

**차원**:
- **내부 길이**: 내부 사용 가능한 공간 (cm)
- **내부 너비**: 내부 사용 가능한 공간 (cm)
- **내부 높이**: 내부 사용 가능한 공간 (cm)
- **벽 두께**: 포장 재료 두께 (cm)

**외부 차원** (자동 계산):
```
외부 길이 = 내부 길이 + (2 × 벽 두께)
외부 너비 = 내부 너비 + (2 × 벽 두께)
외부 높이 = 내부 높이 + (2 × 벽 두께)
```

**무게 및 비용**:
- **공차 무게**: 빈 포장 무게 (그램)
- **최대 무게**: 최대 하중 용량 (그램)
- **비용**: 포장 재료 비용 (비용 최적화용)

**속성**:
- **이름**: 포장 식별자 (예: "작은 상자", "대형 봉투")
- **유형**: 상자 또는 봉투
- **우선순위**: 자동 포장 선택 순서 (숫자가 작을수록 우선순위가 높음)
- **활성**: 가용성 토글

---

## 외부 차원의 중요성

운송업체는 외부 차원을 기반으로 **차원 무게**를 계산합니다:

**차원 무게 공식**:
```
차원 무게 = (길이 × 너비 × 높이) / 분자

일반적인 분자:
- DHL: 5000
- FedEx/UPS: 5000 (국내), 6000 (해외)
```

**예시**:
```
작은 상자:
내부: 20cm × 15cm × 10cm
벽 두께: 0.5cm
외부: 21cm × 16cm × 11cm

차원 무게 = (21 × 16 × 11) / 5000 = 0.74kg

실제 무게 = 0.5kg → 운송업체는 0.74kg(차원 무게가 더 높음)으로 청구
```

**정확도의 중요성**: 차원이 잘못되면 요금 견적이 잘못되어 고객이 과다 또는 부과금을 받을 수 있습니다.

## 일반적인 포장 크기

### 작은 패드 봉투

```
내부: 25cm × 18cm × 2cm
벽 두께: 0.3cm
최대 무게: 500g
유형: 봉투
사용: 문서, 책, 보석
```

### 작은 상자

```
내부: 20cm × 15cm × 10cm
벽 두께: 0.5cm
최대 무게: 5kg
유형: 상자
사용: 작은 전자기기, 화장품, 액세서리
```

### 중간 상자

```
내부: 30cm × 25cm × 20cm
벽 두께: 0.5cm
최대 무게: 15kg
유형: 상자
사용: 옷, 신발, 주방 용품
```

### 대형 상자

```
내부: 45cm × 35cm × 30cm
벽 두께: 0.6cm
최대 무게: 30kg
유형: 상자
사용: 대량 상품, 여러 제품, 대형 전자기기
```

## 자동 포장 알고리즘

시스템은 자동으로 장바구니 상품에 적합한 포장을 선택합니다:

**작동 방식**:
1. 장바구니 상품의 총 부피 계산
2. 포장을 우선순위에 따라 정렬 (숫자가 작은 것부터)
3. 단일 포장에 상품을 넣어보기
4. 맞지 않으면 다음 포장 크기 시도
5. 단일 포장에 맞지 않으면 여러 포장 조합 시도
6. `optimize_for` 설정에 따라 최적화

**최적화 모드**:
- **비용**: 포장 비용 최소화
- **부피**: 낭비된 공간 최소화
- **개수**: 포장 개수 최소화

**예시**:
```
장바구니 상품:
- 항목 A: 10cm × 8cm × 5cm, 200g
- 항목 B: 15cm × 12cm × 8cm, 400g

포장 (우선순위별):
1. 작은 상자 (20×15×10, 우선순위=1)
2. 중간 상자 (30×25×20, 우선순위=2)

알고리즘:
작은 상자 시도: 두 항목 모두 들어감
결과: 1× 작은 상자 (개수 최적화)
```

---

## 포장 우선순위

**우선순위는 포장 순서를 결정합니다**:

우선순위 1 (가장 높음): 작은 포장 먼저 시도
우선순위 10: 큰 포장은 마지막으로 사용

**전략**:
- 작은 포장 = 낮은 우선순위 숫자 (1-3)
- 중간 포장 = 중간 우선순위 (4-6)
- 큰 포장 = 높은 우선순위 숫자 (7-10)

**이유**: 가장 작은 포장부터 시작하여 필요할 경우 확장 → 배송 비용 최소화.

---

## 벽 두께 정확도

실제 포장 측정:

**측정 방법**:
1. 빈 상자 가져오기
2. 내부 차원 측정 (내부)
3. 외부 차원 측정 (외부)
4. 계산: `(외부 - 내부) / 2 = 벽 두께`

**예시**:
```
내부 너비: 20cm
외부 너비: 21cm
벽 두께: (21 - 20) / 2 = 0.5cm
```

**일반적인 두께**:
- 패드 봉투: 0.2-0.4cm
- 단일 벽 종이 상자: 0.4-0.6cm
- 이중 벽 종이 상자: 0.8-1.0cm

## 포장 사전 설정 생성

**단계별**:

1. 설정 > 배송 > 배송 포장
2. "배송 포장 추가" 클릭
3. 이름 입력 (예: "중간 상자")
4. 유형 선택 (상자 또는 봉투)
5. 내부 차원 입력 (cm 단위)
6. 벽 두께 입력 (cm)
7. 시스템이 외부 차원 자동 계산
8. 공차 무게 입력 (빈 포장 무게, 그램)
9. 최대 무게 입력 (하중 용량, 그램)
10. 선택적: 비용 입력 (비용 최적화용)
11. 우선순위 설정 (1-10)
12. 활성화 = 예
13. 저장

## 포장 선택 테스트

**수동 테스트**:
1. 테스트 장바구니에 상품 추가
2. 결제로 이동
3. 실시간 배송 방법 선택 (포장 사용)
4. 합리적인 요금 반환 확인
5. 운송업체 응답 확인 (API 로그에서 선택된 포장 확인)

**자동 포장 미리보기**:
- 일부 배송 제공업체 계정은 포장 분해표시를 보여줍니다
- 장바구니에 선택된 포장 확인
- 최적의 포장 확인

## 팁

- **정확히 측정하세요** - 차원이 잘못되면 운송업체 요금이 잘못됩니다
- **벽 두께 포함하세요** - 차원 무게에 필수적
- **3-4개 크기로 시작하세요** - 작은, 중간, 큰 크기로 대부분의 시나리오를 커버
- **현실적인 최대 무게 설정** - 상자 용량, 이론적 한계가 아님
- **우선순위를 잘 설정하세요** - 작은 상자 우선순위 1, 큰 상자 우선순위 10
- **실제 상품으로 테스트하세요** - 자동 포장이 올바른 크기를 선택하는지 확인
- **포장 변경 시 업데이트하세요** - 새로운 공급업체 = 차원 재측정
- **특수 상품 고려하세요** - 깨지기 쉬운 상품은 특정 상자 크기가 필요할 수 있음
- **활성 포장 최소화하세요** - 너무 많은 옵션은 자동 포장 알고리즘을 느리게 함
- **포장 문서화하세요** - 어떤 상품이 어떤 포장에 들어가는지 기록
