---
title: 被遗弃的购物车
---

当已登录的客户将商品添加到购物车但未在24小时内完成结账时，就会创建一个被遗弃的购物车。Spwig会自动跟踪这些购物车，以便您了解流失的收入，识别客户离开的原因模式，并采取行动挽回销售。

导航到 **客户 > 被遗弃的购物车** 以查看所有记录的遗弃情况。

## 在被遗弃的购物车列表中可以看到的内容

列表视图会以一目了然的方式显示每个被遗弃的购物车，包含以下信息：

| 列 | 描述 |
|---|---|
| **客户** | 客户的姓名和电子邮件 |
| **被遗弃时间** | 购物车被标记为被遗弃的日期和时间 |
| **总价值** | 购物车在被遗弃时的商品货币价值 |
| **总商品数** | 购物车中的商品数量 |
| **预估原因** | Spwig 对购物车被遗弃原因的最佳猜测 |
| **恢复状态** | 该购物车是否已被恢复（转换为已完成的订单） |
| **自遗弃以来的天数** | 购物车被遗弃多久了 |

### 过滤被遗弃的购物车

使用右侧的筛选器来缩小列表范围：

- **预估原因** — 按遗弃原因进行筛选（例如，仅显示预估原因为高运费的购物车）
- **已恢复** — 筛选仅显示已恢复或未恢复的购物车
- **被遗弃时间** — 按日期范围筛选，以关注最近的遗弃情况或特定活动期间

## 理解遗弃原因

Spwig 会为每个遗弃情况记录一个预估原因。这些原因基于结账过程中捕获的信号，并不保证完全准确，但它们为诊断流失模式提供了一个有用的起点。

| 原因 | 可能表明的问题 |
|---|---|
| **未知** | 没有捕获到具体信号 —— 最常见的原因 |
| **运费过高** | 客户可能因结账时显示的运费而却步 |
| **总价过高** | 订单总价可能高于预期 |
| **结账问题** | 客户在结账过程中遇到了问题 |
| **支付失败** | 尝试支付但失败了 |
| **比价购物** | 客户可能访问此网站是为了比较价格 |
| **保存以备后用** | 客户有意将商品保存以备将来再次访问 |

如果你看到大量具有相同原因的购物车 —— 例如，大量“运费过高”导致的购物车放弃 —— 这是一个值得调查的信号，可以检查你的运费设置或结账展示方式。

## 查看单个被放弃的购物车

点击列表中的任何一行以打开详细视图。你将看到：

- **放弃详情** —— 客户信息、购物车参考编号、放弃时间以及预估原因
- **购物车摘要** —— 放弃时的商品数量和总价值
- **恢复跟踪** —— 购物车是否已被恢复、恢复时间以及它转换成的订单

**购物车**字段直接链接到底层的购物车记录，因此你可以确切地看到购物车中包含哪些商品。

## 恢复流程

Spwig会跟踪每个被放弃的购物车是否最终转换为完成的订单。当客户返回并从被放弃的购物车完成购买时，该记录会自动标记为**已恢复**，并链接到生成的订单。

**已发送恢复邮件**计数器显示已向客户发送了多少封自动恢复邮件。这有助于你了解你的电子邮件活动是否促使客户返回。

### 手动恢复操作


未完成的购物车视图是只读的 —— 它是历史记录，而不是编辑购物车内容的工具。

要对未完成的购物车采取行动：

1. 从未完成的购物车记录中记下客户的电子邮件地址
2. 使用您的电子邮件系统或营销工具发送个性化的消息
3. 考虑附加优惠券代码，以激励客户完成购买
4. 在接下来的几天内监控 **已恢复** 的状态，以查看您的跟进是否有效

## 分析购物车放弃趋势

定期查看未完成的购物车列表，以检查结账流程的健康状况：

- 未完成数量的突然激增可能表明结账或支付存在技术问题
- 未恢复购物车中持续较高的购物车价值代表了您最有机会恢复的客户群体
- 随时间比较已恢复与未恢复购物车的比例，以衡量恢复邮件的效果

每个客户个人资料的 **客户分析** 部分还会显示他们的个人购物车放弃率，因此您可以识别那些经常将商品加入购物车但很少完成购买的客户。

## 小贴士

- 按 **总价值**（降序）排序，以识别最有价值的购物车，这些购物车应优先进行个性化跟进
- 使用 **放弃时间** 日期筛选器来查看特定活动或促销期间的购物车放弃情况 —— 限时促销期间的激增可能意味着您的促销吸引了浏览者而非买家
- 将未完成的购物车数据与优惠券活动结合使用：向拥有高价值未恢复购物车的客户发送限时折扣码，以制造紧迫感
- 被放弃超过7天的购物车很可能无法自行恢复 —— 如果启用了恢复邮件，这些购物车需要最多的关注
- 未注册的访客客户不会出现在未完成的购物车中 —— 此跟踪功能仅适用于已注册账户的客户
