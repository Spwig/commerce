---
title: 遺棄購物車
---

當已登入的客戶將商品加入購物車但未在24小時內完成結帳時，就會產生遺棄購物車。Spwig 會自動追蹤這些購物車，讓您了解損失的收入、識別客戶離棄的原因模式，並採取行動恢復銷售。

導航至 **客戶 > 遺棄購物車** 以查看所有記錄的遺棄情況。

## 在遺棄購物車清單中可以看到什麼

清單視圖會顯示每個遺棄的購物車，並提供以下資訊：

| 欄位 | 說明 |
|---|---|
| **客戶** | 客戶的姓名和電子郵件 |
| **遺棄時間** | 購物車被標記為遺棄的日期和時間 |
| **總價值** | 遺棄時購物車中商品的金額價值 |
| **總商品數** | 購物車中的商品數量 |
| **預估原因** | Spwig 對購物車被遺棄原因的最佳猜測 |
| **恢復狀態** | 這個購物車是否已被恢復（轉換為已完成的訂單） |
| **遺棄天數** | 購物車被遺棄多久前 |

### 過濾遺棄購物車

使用右側的過濾器來縮小清單範圍：

- **預估原因** — 按遺棄原因過濾（例如，僅顯示預估原因為運費高昂的購物車）
- **已恢復** — 過濾以僅顯示已恢復或未恢復的購物車
- **遺棄時間** — 按日期範圍過濾，專注於近期遺棄或特定活動期間的遺棄

## 理解遺棄原因

Spwig 會為每個遺棄情況記錄一個預估原因。這些原因基於結帳過程中捕獲的信號，並不能保證完全準確，但它們為診斷離棄模式提供了一個有用的起點。

| 原因 | 可能表示的含義 |
|---|---|
| **未知** | 未捕獲到特定信號 — 最常見的原因 |
| **運費太高** | 顧客可能因結帳時顯示的運費而卻步 |
| **總金額太高** | 訂單總金額可能高於預期 |
| **結帳問題** | 顧客在結帳過程中遇到了問題 |
| **付款失敗** | 有付款嘗試但失敗了 |
| **比價購物** | 顧客可能來訪比較價格 |
| **稍後再購** | 顧客有意將商品保存以備將來再次訪問 |

如果你看到大量具有相同原因的購物車 — 例如，大量「運費太高」的購物車被放棄 — 這是一個值得在運費設置或結帳展示中進行調查的信號。

## 查看單個放棄的購物車

點擊列表中的任何一行以打開詳細視圖。你將看到：

- **放棄詳情** — 顧客、購物車參考、放棄時間和預估原因
- **購物車摘要** — 放棄時的商品數量和總價值
- **恢復追蹤** — 購物車是否已恢復、恢復時間以及轉換為哪個訂單

**購物車**欄位直接連結到底層的購物車記錄，因此你可以看到購物車中確切的商品。

## 恢復流程

Spwig 會追蹤每個放棄的購物車是否最終轉換為完成的訂單。當顧客返回並從放棄的購物車完成購買時，記錄會自動標記為 **已恢復**，並與產生的訂單連結。

**已發送恢復郵件**計數器顯示已自動發送到顧客的恢復郵件數量。這有助於你了解電子郵件活動是否促使顧客返回。

### 手動恢復操作


# 放棄購物車檢視

放棄購物車的檢視功能僅為只讀 —— 它是對發生事件的記錄，而非編輯購物車內容的工具。

要對放棄的購物車採取行動：

1. 從放棄的購物車記錄中記下客戶的電子郵件地址
2. 使用您的電子郵件系統或行銷工具，發送個性化的訊息
3. 考慮附加優惠券碼，以激勵客戶完成購買
4. 在接下來的幾天內監控 **已恢復** 的狀態，以查看您的聯繫是否有效

## 分析購物車放棄趨勢

定期查看放棄的購物車清單，作為對結帳流程健康狀況的檢查：

- 放棄數量的突然增加可能表示結帳或付款存在技術問題
- 在未恢復的購物車中持續高價值的購物車，代表您最有機會恢復的客戶群
- 隨時間比較已恢復與未恢復購物車的比例，以衡量恢復電子郵件的效果

每個客戶個人資料的 **客戶分析** 區域也會顯示他們個人的購物車放棄率，因此您可以識別那些經常加入購物車但很少完成購買的客戶。

## 小技巧

- 按 **總價值**（降序）排序，以識別值得優先進行個人聯繫的高價值購物車
- 使用 **放棄時間** 日期篩選器來查看特定活動或促銷期間的放棄情況 —— 在閃購活動期間出現高峰，可能表示您的促銷吸引了瀏覽者而非買家
- 將放棄的購物車資料與優惠券活動結合：向擁有高價值未恢復購物車的客戶發送有限時間的折扣碼，以創造緊迫感
- 放棄超過 7 天的購物車很可能無法自行恢復 —— 如果已啟用恢復電子郵件，這些購物車需要最多的關注
- 未註冊帳號的訪客客戶不會出現在放棄的購物車中 —— 此追蹤僅適用於已註冊帳號的客戶
