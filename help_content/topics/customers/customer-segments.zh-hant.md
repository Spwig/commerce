---
title: 客戶群組
---

客戶群組讓您可以根據客戶的購買行為，自動將客戶分類到有意義的群組中。一旦客戶被分群，您可以利用這些群組來集中您的行銷努力 — 例如，向VIP客戶提供忠誠獎勵，或向一段時間內未購買的客戶發送回購活動。

Spwig 會根據每個客戶的指標評估群組條件，並將其分配到他們符合的最高優先級群組。當客戶資料更新時，這一切會自動發生。

## 可用的群組類型

Spwig 內建了一組群組類型。每種群組類型都有固定的內部識別碼，但您可以自訂顯示名稱、描述、條件和顏色，以符合您對客戶的思考方式。

| 群組類型 | 典型用途 |
|---|---|
| **訪客客戶** | 沒有建立帳號就完成結帳的客戶 |
| **新客戶** | 最近完成首筆購買的客戶 |
| **常客** | 有穩定購買記錄的客戶 |
| **常客** | 頻繁購買的客戶（訂單之間的時間較短） |
| **高價值客戶** | 總消費金額較高的客戶 |
| **VIP客戶** | 您最有價值且忠誠的客戶 |
| **折扣購物者** | 倾向於在促銷期間購買的客戶 |
| **有風險客戶** | 一段時間內未購買的客戶 |
| **不活躍客戶** | 一段長時間未出現的客戶 |

## 理解群組條件

每個群組都由一組條件定義。Spwig 會根據每個客戶的儲存指標檢查這些條件。群組內的所有條件都會被組合 — 客戶必須符合所有設定的條件才能符合資格。

### 消費條件

- **最低總消費** — 客戶在所有已完成的訂單中必須至少消費此金額
- **最高總消費** — 客戶的總消費金額不得超過此金額

使用消費範圍來識別特定層級。

例如，將 Min 設為 $500，Max 設為 $2,000，將會針對中階客戶。

### 訂單數量標準

- **最少訂單數** — 顧客必須至少有這麼多已完成的訂單
- **最多訂單數** — 顧客的已完成訂單數不能超過這個數字

將最少訂單數與消費金額最低標準結合，是定義高階客戶的可靠方式：他們經常購買 *且* 消費慷慨。

### 最近購買標準

- **上次購買後的最少天數** — 顧客最近的訂單必須至少是這麼多天前
- **上次購買後的最多天數** — 顧客最近的訂單必須是在這麼多天內

最近購買標準對於高風險和不活躍的客戶群體至關重要。例如，將最少天數設為 90，最多天數設為 365，可以識別出那些已經變得沉默但尚未完全流失的客戶。

## 段落優先順序

當顧客符合多個段落時，具有 **最高優先順序** 值的段落會被選中。您可以在段落表單的 **顯示設定** 區域中為每個段落設定優先順序。

**Guest Customer** 段落總是會被最先評估，與優先順序無關，因為訪客狀態是由帳號類型決定，而不是購買標準。

## 查看和管理段落

導航至 **Customers > Customer Segments** 來查看所有已配置的段落。清單會顯示每個段落的顯示名稱、內部類型、指定的顏色、優先順序、目前符合條件的顧客數量，以及段落是否處於啟用狀態。

![Customer Segments List](/static/core/admin/img/help/customer-segments/segments-list.webp)

### 建立或編輯段落

1.

導航至 **Customers > Customer Segments**
2.

點擊現有的段落進行編輯，或點擊 **+ Add Customer Segment** 來建立新的段落
3.

保留所有 markdown 格式、圖片路徑、程式碼區塊和技術術語。

# 填寫 **Segment Information** 細節卡
   - **Name** — 從下拉選單中選擇內部分段類型
   - **Display Name** — 在管理介面中顯示的可讀名稱（例如："VIP Customers"）
   - **Description** — 一個簡短的內部說明，解釋這個分段所代表的內容
4.

根據相關細節卡設定條件：
   - **Criteria - Spending** — 總消費的最低和最高金額
   - **Criteria - Orders** — 訂單數量的最低和最高數
   - **Criteria - Recency** — 上次購買後的最少和最多天數
5.

設定 **Display Settings**（顯示設定）：
   - **Color** — 用於在清單中視覺化識別此分段的十六進制顏色
   - **Priority** — 數字越高，表示此分段會優先評估
   - **Is Active** — 取消勾選以停用此分段而不刪除它
6.

點擊 **Save** 以套用更改

### 示例：設定 VIP 分段

這是一個高價值 VIP 分段的實際示例：

| Field | Value |
|---|---|
| Name | `vip` |
| Display Name | VIP Customers |
| Min Total Spent | $1,000 |
| Min Orders | 5 |
| Max Days Since Last Purchase | 180 |
| Priority | 90 |
| Color | `#FFD700` |

這表示：如果顧客至少消費了 $1,000、至少下了 5 個訂單，且在過去 6 個月內有購買記錄，則該顧客符合 VIP 標準。

### 示例：設定 At Risk 分段

| Field | Value |
|---|---|
| Name | `at_risk` |
| Display Name | At Risk |
| Min Days Since Last Purchase | 60 |
| Max Days Since Last Purchase | 180 |
| Priority | 30 |
| Color | `#FF6B35` |

## 使用分段進行針對性行銷

分段會在管理介面的顧客個人資料中顯示，因此您的團隊可以立即知道每個顧客屬於哪個層級。利用這些資訊來：

- **執行針對性優惠券活動** — 建立僅限特定客戶群體的優惠券，然後使用您的電子郵件系統僅發送到該群組
- **優先提供支援** — 標記VIP或高價值客戶，讓您的團隊能為他們提供優先服務
- **規劃重新參與策略** — 定期審查「風險客戶」和「不活躍客戶」群組，以識別需要回復電子郵件或特別優惠的客戶
- **調整市場營銷支出** — 透過分析哪些客戶群體來自不同渠道，將獲客預算集中在能帶來高價值客戶的渠道上

## 小技巧

- 先使用內建的群組類型，再建立自定義條件 — 它們已經涵蓋了大多數常見的分群需求
- 定期查看每個群組的客戶數量；如果VIP群組沒有任何客戶，或「風險客戶」群組快速增長，都值得進一步調查
- 請謹慎使用 **Priority（優先順序）** 欄位 — 如果某位客戶符合多個群組的條件（例如同時符合「常客」和「高價值」），優先順序較高的群組會被選中
- 如果目前沒有使用某個群組，請先停用它，而不是刪除 — 之後可以重新啟用，而無需重新設定條件
- 群組條件會根據儲存的客戶指標進行檢查，這些指標會自動重新計算。如果群組數量看起來過時，可以從管理員介面的 **Customer Metrics（客戶指標）** 部分重新計算指標
