---
title: 客户群体
---

客户群体功能可让您根据客户的购买行为，自动将客户分类到有意义的群体中。一旦客户被分组，您可以利用这些群体集中营销资源——例如，为VIP客户提供忠诚度奖励，或向一段时间内未购买的客户发送召回活动。

Spwig 会根据每个客户的指标评估分组条件，并将其分配到他们符合的最高优先级的群体中。当客户数据更新时，这一过程会自动发生。

## 可用的分组类型

Spwig 内置了一组分组类型。每种分组类型都有一个固定的内部标识符，但您可以自定义显示名称、描述、条件和颜色，以符合您对客户群体的看法。

| 分组类型 | 典型用途 |
|---|---|
| **访客客户** | 未创建账户即完成结账的客户 |
| **新客户** | 最近完成首次购买的客户 |
| **常客** | 具有稳定购买历史的客户 |
| **频繁购买者** | 经常购买（订单间隔时间短）的客户 |
| **高价值客户** | 总消费金额高的客户 |
| **VIP客户** | 您最有价值和最忠诚的客户 |
| **价格敏感型客户** | 倾向于在促销期间购买的客户 |
| **风险客户** | 一段时间内未购买的客户 |
| **不活跃客户** | 一段时间内未出现的客户 |

## 了解分组条件

每个分组由一组条件定义。Spwig 会将这些条件与每个客户的存储指标进行比对。分组内的所有条件都会被组合——客户必须满足所有设定的条件才能符合该分组。

### 消费条件

- **最低总消费额** — 客户在所有已完成的订单中必须至少花费此金额
- **最高总消费额** — 客户在所有已完成的订单中花费不得超过此金额

使用消费范围来识别特定层级。

例如，将最低设置为 500 美元，最高设置为 2,000 美元，可以针对中端客户。

### 订单数量标准

- **最低订单数** — 客户必须至少有这么多已完成的订单
- **最高订单数** — 客户的已完成订单数不得超过这个数值

将最低订单数与最低消费额结合使用，是定义 VIP 客户的一种可靠方式：他们经常购买 *并且* 消费慷慨。

### 最近购买标准

- **最近购买的最少天数** — 客户最近一次订单必须至少是这个天数之前
- **最近购买的最多天数** — 客户最近一次订单必须在这一天数范围内

最近购买标准对于识别高风险和不活跃的客户群体至关重要。例如，将最少天数设置为 90，最多天数设置为 365，可以识别出那些已经变得不活跃但尚未完全流失的客户。

## 分段优先级

当客户符合多个分段时，具有 **最高优先级** 值的分段将被选中。你可以在分段表单的 **显示设置** 部分为每个分段设置优先级。

**访客客户** 分段始终优先评估，不受优先级顺序的影响，因为访客状态是根据账户类型而非购买标准来确定的。

## 查看和管理分段

导航到 **客户 > 客户分段**，查看所有已配置的分段。列表中会显示每个分段的显示名称、内部类型、分配的颜色、优先级、当前匹配客户的数量，以及该分段是否处于活动状态。

![客户分段列表](/static/core/admin/img/help/customer-segments/segments-list.webp)

### 创建或编辑分段

1.

导航到 **客户 > 客户分段**
2.

点击现有分段进行编辑，或点击 **+ 添加客户分段** 创建新分段
3.


填写 **Segment Information**（分段信息）标签页：
   - **Name** — 从下拉菜单中选择内部分段类型
   - **Display Name** — 在管理后台中显示的人类可读名称（例如："VIP Customers"）
   - **Description** — 一个简短的内部说明，解释此分段所代表的内容
4.

在相关标签页中设置条件：
   - **Criteria - Spending** — 总消费的最小和最大值
   - **Criteria - Orders** — 订单数量的最小和最大值
   - **Criteria - Recency** — 自上次购买以来的最小和最大天数
5.

配置 **Display Settings**（显示设置）：
   - **Color** — 用于在列表中直观标识此分段的十六进制颜色
   - **Priority** — 数字越高，此分段的优先级越高
   - **Is Active** — 取消勾选以禁用该分段而不删除它
6.

点击 **Save** 以应用更改

### 示例：配置一个 VIP 分段

这是一个高价值 VIP 分段的现实示例：

| Field | Value |
|---|---|
| Name | `vip` |
| Display Name | VIP Customers |
| Min Total Spent | $1,000 |
| Min Orders | 5 |
| Max Days Since Last Purchase | 180 |
| Priority | 90 |
| Color | `#FFD700` |

这意味着：如果客户至少消费了 $1,000，至少下了 5 个订单，并且在最近 6 个月内有购买记录，则该客户符合 VIP 条件。

### 示例：配置一个 At Risk 分段

| Field | Value |
|---|---|
| Name | `at_risk` |
| Display Name | At Risk |
| Min Days Since Last Purchase | 60 |
| Max Days Since Last Purchase | 180 |
| Priority | 30 |
| Color | `#FF6B35` |

## 使用分段进行定向营销

分段会在管理后台的客户资料中显示，因此您的团队可以立即知道每个客户属于哪个层级。利用这些信息：

- **开展定向优惠券活动** — 创建仅限特定客户群体使用的优惠券，然后使用您的电子邮件系统仅发送给该群体
- **优先提供支持** — 标记VIP或高价值客户，以便您的团队为其提供优先服务
- **制定重新激活计划** — 定期查看“风险客户”和“不活跃客户”分组，以识别需要回访邮件或特别优惠的客户
- **调整营销预算** — 通过分析哪些客户群体是由哪些渠道转化而来的，将获取客户的预算重点放在能带来高价值客户的渠道上

## 小贴士

- 在创建自定义条件之前，先使用内置的分组类型 — 它们可以满足最常见的分组需求
- 定期查看每个分组的客户数量；如果VIP分组中没有客户，或“风险客户”分组快速增长，都值得进一步调查
- 谨慎使用**优先级**字段 — 如果某些分组的条件存在重叠（例如，一位客户同时符合“经常购买者”和“高价值客户”），优先级较高的分组将被选中
- 如果当前不使用某些分组，应将其停用而不是删除 — 以后可以随时重新启用，无需重新配置条件
- 分组条件是基于存储的客户指标进行检查的，这些指标会自动重新计算。如果发现分组数量看起来过时，可以从管理后台的“客户指标”部分重新计算指标
