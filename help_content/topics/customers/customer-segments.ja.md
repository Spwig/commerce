---
title: 顧客セグメント
---

顧客セグメントを使用すると、購入行動に基づいて顧客を意味のあるグループに自動的に分類できます。顧客がセグメントに分類されると、そのグループを活用してマーケティング活動に集中できます。例えば、VIP顧客にロイヤルティ報酬を提供したり、長期間購入していない顧客に対してリカバリーキャンペーンを送信したりできます。

Spwigは、各顧客のメトリクスに対してセグメントの条件を評価し、該当する最も優先度の高いセグメントに顧客を割り当てます。これは、顧客データが更新されるたびに自動的に行われます。

## 利用可能なセグメントタイプ

Spwigには、いくつかの組み込みのセグメントタイプが用意されています。各セグメントタイプには固定された内部識別子がありますが、表示名、説明、条件、色をカスタマイズして、顧客に対するあなたの考え方と一致させることができます。

| セグメントタイプ | 一般的な用途 |
|---|---|
| **ゲスト顧客** | アカウントを作成せずにチェックアウトした顧客 |
| **新規顧客** | 最近初めて購入した顧客 |
| **常連顧客** | 定期的に購入している顧客 |
| **頻繁な購入者** | 注文の間隔が短い、よく購入する顧客 |
| **高価値顧客** | 総支出額が高い顧客 |
| **VIP顧客** | 最も価値があり、忠誠心が高い顧客 |
| **セール好き** | 割引セール時に購入する傾向がある顧客 |
| **リスクのある顧客** | ある程度の期間購入していない顧客 |
| **非アクティブ顧客** | 長期間活動していない顧客 |

## セグメント条件の理解

各セグメントは、条件の組み合わせによって定義されます。Spwigは、各顧客の保存されたメトリクスに対してこれらの条件をチェックします。セグメント内のすべての条件は組み合わされており、顧客が設定されたすべての条件を満たす必要があります。

### 購入条件

- **最低総購入額** — 顧客はすべての完了した注文でこの金額以上を購入している必要があります
- **最大総購入額** — 顧客はこの金額を超えて購入してはいけません

購入範囲を使用して特定の階層を特定できます。

例えば、Minを$500、Maxを$2,000に設定すると、中堅層の顧客を対象にできます。

### 注文数の基準

- **Min Orders** — 顧客はこの数以上の完了した注文が必要です
- **Max Orders** — 顧客はこの数以下の完了した注文を保持している必要があります

Min Ordersと支出の最小値を組み合わせることは、VIP顧客を定義する信頼できる方法です。彼らは頻繁に購入し、*かつ* 大きな金額を費やす傾向があります。

### 最近の購入に関する基準

- **Min Days Since Last Purchase** — 顧客の最新の注文はこの日数以上前になければなりません
- **Max Days Since Last Purchase** — 顧客の最新の注文はこの日数以内でなければなりません

最近の購入に関する基準は、リスクのあるセグメントや非アクティブなセグメントを特定するために不可欠です。例えば、Min Daysを90日、Max Daysを365日に設定すると、静かに活動していないが完全に失われたわけではない顧客を特定できます。

## セグメントの優先順位

顧客が1つ以上のセグメントに該当する場合、**最も高い優先順位**を持つセグメントが適用されます。セグメントの優先順位は、セグメントフォームの**Display Settings**セクションで各セグメントごとに設定できます。

**Guest Customer**セグメントは、優先順位の順序に関係なく常に最初に評価されます。これは、ゲストステータスが購入基準ではなくアカウントタイプによって決まるためです。

## セグメントの表示と管理

**Customers > Customer Segments**にナビゲートして、設定されたすべてのセグメントを確認できます。リストには、各セグメントの表示名、内部タイプ、割り当てられた色、優先順位、現在該当する顧客数、セグメントがアクティブかどうかが表示されます。

![Customer Segments List](/static/core/admin/img/help/customer-segments/segments-list.webp)

### セグメントの作成または編集

1. 

**Customers > Customer Segments**にナビゲートします
2. 

既存のセグメントを編集するにはクリックし、または**+ Add Customer Segment**をクリックして新しいセグメントを作成します
3. 

すべてのマークダウンフォーマット、画像パス、コードブロック、技術用語を保持してください。

**セグメント情報**タブを入力してください:
   - **名前** — ドロップダウンから内部セグメントタイプを選択
   - **表示名** — 管理画面に表示される人間が読みやすい名前（例: "VIP顧客"）
   - **説明** — このセグメントが何を表しているかを示す簡単な内部のノート
4.

関連するタブにわたって基準を設定してください:
   - **基準 - 購入額** — 総購入額の最小値と最大値
   - **基準 - 注文** — 注文数の最小値と最大値
   - **基準 - 最近性** — 最後の購入から経過した日数の最小値と最大値
5.

**表示設定**を構成してください:
   - **色** — このセグメントをリストで視覚的に識別するために使用される16進色
   - **優先度** — 数値が高いほど、このセグメントが最初に評価されます
   - **有効** — 有効にする代わりにセグメントを無効にするためにチェックを外してください
6.

変更を適用するために**保存**をクリックしてください

### 例: VIPセグメントの構成

ここに高価値のVIPセグメントの現実的な例を示します:

| フィールド | 値 |
|---|---|
| 名前 | `vip` |
| 表示名 | VIP顧客 |
| 最小総購入額 | $1,000 |
| 最小注文数 | 5 |
| 最後の購入からの最大日数 | 180 |
| 優先度 | 90 |
| 色 | `#FFD700` |

これは、顧客が少なくとも$1,000を購入し、少なくとも5回の注文を出し、最後の購入が6か月以内である場合、その顧客がVIPとして資格があることを示しています。

### 例: At Riskセグメントの構成

| フィールド | 値 |
|---|---|
| 名前 | `at_risk` |
| 表示名 | At Risk |
| 最小日数（最後の購入から） | 60 |
| 最大日数（最後の購入から） | 180 |
| 優先度 | 30 |
| 色 | `#FF6B35` |

## セグメントを用いたターゲティングマーケティング

セグメントは、管理画面全体の顧客プロファイルに表示されるため、チームは各顧客がどのレベルに属しているかをすぐに把握できます。この情報を使用して:


- **ターゲットされたクーポンキャンペーンの実施** — 特定のセグメントに属する顧客に限定したクーポンを作成し、メールシステムを使ってそのグループにのみ送信します
- **サポートの優先順位付け** — VIPまたは高価値の顧客をマークして、チームが優先的なサポートを提供できるようにします
- **再エンゲージメントの計画** — At Risk（リスクのある）およびInactive（アクティブでない）セグメントを定期的に確認し、リカウントメールや特別なオファーが必要な顧客を特定します
- **マーケティング予算の調整** — 高価値顧客を獲得するチャネルに獲得予算を集中させ、どのコホートセグメントがそれらを変換しているかを分析します

## Tips

- まず、ビルトインのセグメントタイプから始め、カスタムの基準を作成する前にそれらを使用してください — これらは、箱に含まれている最も一般的なセグメンテーションのニーズをカバーしています
- 各セグメントの顧客数を定期的に確認してください。VIPセグメントに顧客が0人いる、またはAt Riskセグメントが急速に増加していることは、どちらも調査価値があります
- **Priority（優先度）**フィールドを意図的に使用してください — セグメント間で基準が重複している場合（例: ある顧客がFrequent Buyer（頻繁な購入者）とHigh Value（高価値）の両方に該当する場合）、優先度が高いセグメントが適用されます
- 現在使用していないセグメントは削除する代わりに無効化してください — 後で再び有効化することができ、基準を再構成する必要はありません
- セグメントの基準は、保存された顧客メトリクスに対してチェックされ、自動的に再計算されます。セグメントの数が古びているように見える場合は、管理者のCustomer Metrics（顧客メトリクス）セクションからメトリクスを再計算できます
