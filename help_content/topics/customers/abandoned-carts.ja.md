---
title: 放棄されたカート
---

ログインした顧客が商品をカートに追加した後、24時間以内にチェックアウトを完了しない場合、放棄されたカートが作成されます。Spwigはこれらのカートを自動的に追跡し、失われた売上を理解し、顧客が離れる理由のパターンを特定し、売上の回復に向けた行動を取ることができます。

**顧客 > 放棄されたカート**に移動して、すべての記録された放棄を表示してください。

## 放棄されたカート一覧で確認できる内容

一覧表示では、以下の情報を一目で確認できる放棄されたカートが表示されます：

| 列 | 説明 |
|---|---|
| **顧客** | 顧客の名前とメールアドレス |
| **放棄された時刻** | カートが放棄されたとマークされた日時 |
| **総価値** | 放棄時のカート内の商品の金額 |
| **総アイテム数** | カート内の商品数 |
| **推定理由** | Spwigがカートが放棄されたと考える理由 |
| **回復状態** | このカートが回復された（完了した注文に変換された）かどうか |
| **放棄からの日数** | カートが放棄された期間 |

### 放棄されたカートのフィルタリング

右側のフィルタを使用して、リストを絞り込むことができます：

- **推定理由** — 放棄理由でフィルタ（例：推定理由が送料が高かった場合のみ表示）
- **回復済み** — 回復済みまたは未回復のカートのみを表示するフィルタ
- **放棄された時刻** — 日付範囲でフィルタし、最近の放棄または特定のキャンペーン期間に焦点を当てる

## 放棄理由の理解

Spwigは、各放棄に対して推定された理由を記録します。これらの理由は、チェックアウトプロセス中にキャプチャされたシグナルに基づいており、正確であるとは保証されていませんが、離脱パターンの診断のための有用な出発点を提供します。

| 理由 | 何を示しているか |
|---|---|
| **不明** | 特定の信号がキャプチャされなかった — 最も一般的な理由 |
| **送料が高すぎる** | 顧客がチェックアウト時に表示された送料によって購入をためらった可能性がある |
| **合計が高すぎる** | 注文の合計金額が予想より高かった可能性がある |
| **チェックアウトの問題** | 顧客がチェックアウトプロセス中に問題に遭遇した |
| **支払い失敗** | 支払いが試みられたが失敗した |
| **価格比較** | 顧客はおそらく価格を比較するために訪れた |
| **後で保存** | 顧客が意図的にアイテムを今後の訪問のために保存した |

同じ理由で放棄されたカートが大量にある場合 — 例えば、「送料が高すぎる」放棄が顕著に集まっている場合 — それは送料設定やチェックアウトの表示方法を調査する価値のあるシグナルです。

## 個別の放棄されたカートの確認

リスト内の任意の行をクリックして詳細ビューを開きます。以下を確認できます：

- **放棄の詳細** — 顧客、カートの参照、放棄された時間、および推定された理由
- **カートの概要** — 放棄時のアイテム数と合計金額
- **回復の追跡** — カートが回復されたかどうか、回復された時間、およびどの注文に変換されたか

**カート**フィールドは、下層のカートレコードに直接リンクしており、カートに含まれていた正確な商品を確認できます。

## 回復ワークフロー

Spwigは、各放棄されたカートが最終的に完成した注文に変換されたかどうかを追跡します。顧客が戻ってきて放棄されたカートから購入を完了すると、そのレコードは自動的に**回復済み**としてマークされ、生成された注文はリンクされます。

**回復用メール送信数**のカウンターは、このカートに対して顧客に自動的に送信された回復メールの数を示します。これにより、メールキャンペーンが顧客の再訪を促しているかどうかを理解できます。

### 手動の回復アクション

放棄されたカートのビューは読み取り専用です — これは過去の出来事の記録であり、カートの内容を編集するためのツールではありません。

放棄されたカートに対してアクションを実行するには:

1. 放棄されたカートのレコードから顧客のメールアドレスを確認する
2. ご自分のメールシステムやマーケティングツールを使用して、パーソナライズされたメッセージを送信する
3. 顧客に購入を完了させるインセンティブとして、割引コードを添付することを検討する
4. その後数日間、**Recovered**（回復）のステータスを確認して、リーチが成功したかどうかを確認する

## カート放棄の傾向の分析

チェックアウトプロセスの健康診断として、定期的に放棄されたカートリストを確認してください:

- チェックアウトまたは支払いに関する技術的な問題が原因で、放棄数が急激に増加する可能性があります
- 回復されていないカートの価値が常に高い場合、これは最も回復の可能性が高いセグメントを示しています
- 回復したカートと回復されていないカートの比率を時間経過に応じて比較することで、回復メールの効果を測定できます

各顧客プロファイルの**Customer Analytics**（顧客分析）セクションには、その顧客の個人的なカート放棄率も表示されています。これにより、カートに頻繁に商品を追加するが、購入をほとんど完了しない顧客を特定できます。

## ヒント

- **Total Value**（総価値）で降順に並べ替えることで、個人的なアプローチを優先すべき高価値のカートを特定できます
- **Abandoned At**（放棄日時）の日付フィルタを使用して、特定のキャンペーンやプロモーション期間の放棄を確認できます — フラッシュセール中に急増した場合、プロモーションは閲覧者ではなく購入者を引き寄せている可能性があります
- 放棄されたカートのデータと割引キャンペーンを組み合わせて使用してください: 回復されていない高価値のカートを持つ顧客に、期限付きの割引コードを送信して、緊急性を生み出してください
- 7日以上放棄されたカートは、自分では回復する可能性が低いです — 回復メールが有効になっている場合、これらのカートが最も注意を必要とするものです
- ゲスト顧客は放棄されたカートに表示されません — このトラッキングは登録アカウントを持つ顧客にのみ適用されます
