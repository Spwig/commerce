---
title: Полная миграция системы
---

Полная миграция системы переносит весь ваш магазин — настройки, товары, клиентов, заказы, медиафайлы и все остальные данные — с одной установки Spwig на другую. Используйте это при переходе на новый сервер или при создании полной копии вашего магазина.

## Когда использовать полную миграцию

- **Перемещение сервера**: Перенос вашего магазина на новый провайдера хостинга или сервер
- **Создание среды для тестирования**: Настройка полной среды тестирования из рабочей среды
- **Восстановление после катастрофы**: Восстановление полного магазина из резервной копии

Полная миграция включает всё, что делает синхронизация настроек, плюс все транзакционные данные (товары, клиенты, заказы, отзывы, запасы, медиа и т.д.).

## Что мигрируется

Полная миграция может переносить все категории настроек плюс следующие категории данных:

| Категория | Описание |
|----------|-------------|
| **Установленные компоненты** | Темы, интеграции провайдеров и утилитарные компоненты с их пакетными файлами |
| **Товары, категории и бренды** | Товары, варианты, изображения, категории, бренды и атрибуты |
| **Библиотека медиа** | Все загруженные медиафайлы и ресурсы |
| **Клиенты и адреса** | Аккаунты клиентов, профили и адреса |
| **История заказов** | Заказы, товары в заказах и транзакционные записи |
| **Отзывы о товарах** | Отзывы клиентов и оценки |
| **Уровни запасов** | Количество запасов по складам и точки reorder |
| **Цифровые товары и лицензии** | Цифровые активы, шаблоны лицензий и пулы лицензий |
| **Подарочные карты и использование купонов** | Балансы подарочных карт и записи использования купонов |
| **Кредит магазина и кошельки** | Балансы кошельков клиентов и история транзакций |
| **Участники программы лояльности** | Участники программы лояльности, баллы, транзакции и значки |
| **Активные подписки** | Планы подписок, активные подписки и история оплат |
| **Отправки и отслеживание** | Записи отправок и события отслеживания |
| **Возвраты, возвраты и заметки к заказу** | Записи возвратов, запросы на возврат и заметки |
| **Участники аффилиатов** | Аккаунты аффилиатов, коды рефералов и история комиссий |

## Пошаговое руководство

### Шаг 1: Подключение к исходной системе

1. Перейдите в **Data Migration > Spwig-to-Spwig Sync** в боковом меню администратора
2. Нажмите **Start Full Migration**
3. Подключитесь к исходному магазину (магазин, с которого вы мигрируете):
   - Введите URL исходного магазина
   - Вставьте токен синхронизации из исходного магазина
   - Назовите подключение (например, "Old Production Server")
4. Нажмите **Test Connection**, чтобы проверить подключение
5. Нажмите **Next**

> **Важно:** Полная миграция всегда **загружает** данные с подключенного магазина в этот магазин. Запускайте мастер на **целевом** (новом) магазине.

### Шаг 2: Выбор области

Выберите, какие категории данных включить в миграцию. Категории организованы в группы:

- **Настройки**: Настройка магазина, темы, провайдеры, контент
- **Данные**: Товары, клиенты, заказы, медиа и другие транзакционные данные

Некоторые категории имеют зависимости (например, Заказы зависят от Клиентов и Товаров). Зависимости автоматически включаются, когда вы выбираете категорию.

Категории со специальными индикаторами:
- **Иконка ключа**: Содержит учетные данные, которые передаются безопасно
- **Иконка файла**: Включает двоичные файлы (изображения, медиа, пакеты)
- **Иконка предупреждения**: Особые соображения для рабочих сред

### Шаг 3: Предварительные проверки

Перед началом миграции автоматические предварительные проверки проверяют:

- **Состояние подключения**: Исходный магазин доступен и аутентифицирован
- **Совместимость версий**: Оба магазина работают на совместимых версиях Spwig
- **Доступное место на диске**: Достаточно места для хранения медиафайлов
- **Готовность базы данных**: Целевая база данных может принять данные

Если какие-либо проверки не пройдут, вы увидите конкретные рекомендации по устранению проблемы перед продолжением.

### Шаг 4: Ход миграции

Миграция выполняется в фоновом режиме. Вы можете безопасно уйти отсюда — процесс продолжится.


Страница прогресса показывает:
- Общий процент с оценкой оставшегося времени
- Статус завершения по категориям
- Живой журнал активности с деталями передачи
- Статистика передачи медиафайлов (файлы и байты, переданные) для категории медиа

Для крупных магазинов с множеством товаров и медиафайлов миграция может занять некоторое время. Фаза передачи медиа обычно является самой длительной.

### Шаг 5: Результаты

После завершения миграции страница результатов показывает:

- Общая статистика (перенесенные, пропущенные, неудачные элементы)
- Разбивка по категориям со статусом
- Подробности ошибок для любых неудачных элементов

## Проверочный список после миграции

После успешной миграции выполните следующие шаги на вашем новом магазине:

1. **Активируйте лицензию** на новой установке
2. **Вновь введите учетные данные провайдера платежей**, которые были пропущены во время миграции (ключи для песочницы/теста не передаются в продакшн)
3. **Настройте DNS**, чтобы ваш домен указывал на новый сервер
4. **Проверьте процесс оформления заказа** с помощью тестового заказа
5. **Убедитесь, что отправка электронной почты** работает корректно
6. **Проверьте медиафайлы** и изображения, чтобы убедиться, что они загружаются правильно

## Откат

После завершения полной миграции у вас есть **24 часа**, чтобы выполнить откат. Откат удаляет все перенесенные данные из целевого магазина, возвращая его в состояние до миграции.

Чтобы выполнить откат:
1. Перейдите на страницу результатов или на панель синхронизации
2. Нажмите **Откат миграции** и подтвердите
3. Дождитесь завершения отката

> **Предупреждение:** Откат навсегда удаляет все перенесенные данные. Любые изменения, внесенные в целевой магазин после миграции (новые заказы, регистрации клиентов и т.д.), также будут затронуты.

Через 24 часа опция отката истекает.

## Советы

- **Запускайте на целевом магазине**: Мастер полной миграции должен запускаться на **новом** магазине, получая данные с старого
- **Мигрируйте в чистую установку**: Для наилучших результатов запустите миграцию на свежей установке Spwig перед запуском
- **Проверьте объем диска**: Убедитесь, что целевой сервер имеет достаточно места для хранения всех медиафайлов
- **Оставьте источник включенным**: Не выключайте исходный магазин, пока не проверите, что все работает на целевом сервере
- **Планируйте переход DNS**: После проверки миграции обновите свои DNS-записи, чтобы они указывали на новый сервер
