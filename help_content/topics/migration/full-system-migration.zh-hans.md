---
title: 完整系统迁移
---

完整系统迁移会将您的整个商店——设置、产品、客户、订单、媒体文件以及所有其他数据——从一个 Spwig 安装迁移到另一个安装。当您要迁移到新服务器或设置商店的完整副本时，请使用此功能。

## 何时使用完整迁移

- **服务器迁移**：将商店迁移到新的托管提供商或服务器
- **创建预发布副本**：从生产环境设置完整的预发布环境
- **灾难恢复**：从备份实例恢复完整的商店

完整迁移包括设置同步所包含的所有内容，再加上所有交易数据（产品、客户、订单、评论、库存、媒体等）。

## 会迁移哪些内容

完整迁移可以迁移所有设置类别以及以下数据类别：

保留所有 markdown 格式、图片路径、代码块和专业术语。

| 分类 | 描述 |
|----------|-------------|
| **已安装的组件** | 主题、提供商集成和实用组件及其包文件 |
| **产品、分类与品牌** | 产品、变体、图片、分类、品牌和属性 |
| **媒体库** | 所有上传的媒体文件和资源 |
| **客户与地址** | 客户账户、个人资料和地址 |
| **订单历史** | 订单、订单项和交易记录 |
| **产品评论** | 客户评论和评分 |
| **库存水平** | 每个仓库的库存数量和补货点 |
| **数字产品与许可证** | 数字资产、许可证模板和许可证池 |
| **礼品卡与优惠券使用** | 礼品卡余额和优惠券使用记录 |
| **商店信用与钱包** | 客户钱包余额和交易历史 |
| **忠诚度计划成员** | 忠诚度成员、积分、交易和徽章 |
| **活跃订阅** | 订阅计划、活跃订阅和计费历史 |
| **发货与跟踪** | 发货记录和跟踪事件 |
| **退款、退货与订单备注** | 退款记录、退货请求和备注 |
| **联盟成员** | 联盟账户、推荐码和佣金历史 |

## 分步指南

### 步骤 1：连接到源实例

1. 在管理侧边栏中导航到 **数据迁移 > Spwig-to-Spwig 同步**
2. 点击 **开始完整迁移**
3. 连接到源商店（您要从中迁移的商店）：
   - 输入源商店的 URL
   - 从源商店粘贴同步令牌
   - 为连接命名（例如，"旧生产服务器"）
4. 点击 **测试连接** 以验证
5. 点击 **下一步**

> **重要提示：** 完整迁移始终会从连接的商店 **拉取** 数据到此商店。请在 **目标**（新）商店上运行此向导。

### 步骤 2：选择范围

选择要包含在迁移中的数据类别。类别按组组织：

- **设置**: 商店配置、主题、提供商、内容
- **数据**: 产品、客户、订单、媒体和其他交易数据

某些类别有依赖关系（例如，订单依赖于客户和产品）。选择类别时会自动包含依赖项。

具有特殊标识的类别：
- **密钥图标**: 包含通过安全方式传输的凭证
- **文件图标**: 包含二进制文件（图片、媒体、软件包）
- **警告图标**: 需要特别注意生产环境

### 第 3 步：预检查

在迁移开始之前，自动预检查将验证：

- **连接状态**: 源商店可以访问并已通过身份验证
- **版本兼容性**: 两个商店运行的是兼容的 Spwig 版本
- **磁盘空间**: 有足够的存储空间用于媒体文件
- **数据库准备情况**: 目标数据库可以接收数据

如果任何检查失败，您将看到有关如何解决问题的具体指导，然后再继续。

### 第 4 步：迁移进度

迁移在后台运行。您可以安全地离开页面——进程将继续进行。

进度页面显示：
- 总体百分比和剩余预估时间
- 每个类别的完成状态
- 实时活动日志和传输详情
- 媒体传输统计信息（媒体类别中传输的文件和字节数）

对于产品和媒体文件数量较多的大型商店，迁移可能需要一些时间。媒体传输阶段通常是最长的。

### 第 5 步：结果

迁移完成后，结果页面显示：

- 汇总统计信息（已迁移、已跳过、失败的项目）
- 每个类别的详细信息和状态
- 任何失败项目的错误详情

## 迁移后检查清单

成功迁移后，在您的新商店上完成以下步骤：

1. 在新安装上 **激活许可证**
2. **重新输入支付提供商凭据**，这些凭据在迁移过程中被跳过（沙箱/测试密钥不会转移到生产环境）
3. **配置DNS**，将您的域名指向新服务器
4. **使用测试订单测试结账流程**
5. **验证电子邮件发送** 是否正常工作
6. **检查媒体文件** 和图片是否加载正常

## 回滚

在完成完整迁移后，您有 **24小时** 的时间进行回滚。回滚会从目标商店中删除所有迁移的数据，将其恢复到迁移前的状态。

要回滚：
1. 转到结果页面或同步仪表板
2. 点击 **回滚迁移** 并确认
3. 等待回滚完成

> **警告：** 回滚会永久删除所有迁移的数据。迁移后在目标商店上进行的任何更改（如新订单、客户注册等）也将会受到影响。

24小时后，回滚选项将过期。

## 小贴士

- **在目标商店上运行**：完整迁移向导应在 **新** 商店上运行，从旧商店拉取数据
- **迁移到干净的安装**：为了获得最佳效果，在上线前应在全新的 Spwig 安装上运行迁移
- **检查磁盘空间**：确保目标商店有足够的存储空间存储所有媒体文件
- **保持源商店运行**：在确认目标商店上一切正常之前，请不要关闭源商店
- **规划DNS过渡**：在验证迁移后，更新DNS记录以指向新服务器
