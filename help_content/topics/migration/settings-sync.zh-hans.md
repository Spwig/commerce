---
title: 设置同步
---

设置同步允许你在两个 Spwig 安装之间复制商店配置。这对于维护预发布和生产环境非常理想，你可以在预发布环境中配置和测试更改，然后再将其部署到你的实时商店。

## 何时使用设置同步

- **预发布到生产**：在预发布商店中配置设置，然后将其推送到生产环境
- **生产到预发布**：将生产环境的设置拉取到预发布环境，以创建匹配的环境
- **备份配置**：将生产环境的设置拉取到备份实例中，作为安全措施

设置同步仅处理配置数据——它不会传输产品、客户、订单或媒体文件。如需完整的数据传输，请改用完整系统迁移。

## 可同步的内容

设置同步支持以下类别：

| 分组 | 类别 |
|-------|-----------|
| **设置** | 站点设置、税与货币、税率、语言、博客设置、社交分享、销售区域与仓库、搜索配置、自定义字段、员工角色、客户分析 |
| **设计** | 设计与主题、页眉/页脚/菜单 |
| **提供商** | 邮件、短信/WhatsApp、支付提供商、物流、SEO 提供商、产品数据源、博客社交连接器、POS 配置 |
| **内容** | 页面与模板、博客文章、公告、表单、产品集合 |
| **电商** | 电商规则（优惠券、促销、忠诚度、订阅）、联盟计划、Webhook 与集成 |

> **注意**：包含凭证的类别（支付提供商、物流账户等）会用钥匙图标标记。API 密钥和密钥会安全传输，但可能需要重新输入以进行基于 OAuth 的集成。

## 分步指南

### 第 1 步：建立连接

1.

在管理侧边栏中导航到 **数据迁移 > Spwig-to-Spwig 同步**
2.

点击 **开始设置同步**
3.

保留所有 markdown 格式、图片路径、代码块和专业术语。

选择已保存的连接或创建新的连接：
   - 输入远程商店的URL（例如，`https://staging.yourstore.com`）
   - 粘贴在远程商店上生成的同步令牌
   - 为连接命名以描述其用途
   - 设置角色（预发布、生产、备份或其它）
4.

点击 **测试连接** 以验证其是否正常工作
5.

点击 **下一步** 继续

### 第2步：选择类别和方向

**方向：**
- **拉取** -- 从连接的商店复制设置到此商店
- **推送** -- 从此商店复制设置到连接的商店

**同步模式：**
- **添加与更新** -- 添加新项目并更新现有项目，但从不删除任何内容。这是最安全的选项。
- **精确复制** -- 使目标与源完全一致，包括删除目标中存在但源中不存在的项目。请谨慎使用。

选择您要包含的类别，然后点击 **下一步**。

### 第3步：预览更改

在任何更改应用之前，您将看到详细的预览，显示每个类别将确切添加、修改或删除的内容。请仔细查看。

如果推送到生产环境连接，您需要确认了解更改将影响您的实时商店。

准备好后点击 **开始同步**。

### 第4步：监控进度

同步在后台运行。您可以安全地离开进度页面——同步将继续运行。

进度页面显示：
- 整体完成百分比及剩余预估时间
- 每个类别的进度及成功/失败数量
- 可展开的实时活动日志，以查看详细输出

## 回滚

同步完成后，您有 **24小时** 的时间回滚更改。回滚将恢复所有受影响设置的先前状态。

要回滚：
1. 转到 **同步仪表板**
2. 找到已完成的任务
3. 点击 **回滚** 并确认

24小时后，回滚选项将过期，更改将变为永久性。

## 小贴士

保留所有markdown格式、图片路径、代码块和专业术语。

- **先在预发布环境进行测试**：在将更改推送到生产环境之前，始终先同步到预发布环境以验证结果
- **使用添加与更新模式**：这是最安全的模式，因为它从不会删除现有数据
- **仔细检查预览**：差异预览会准确显示在应用任何更改之前会发生哪些变化
- **生产连接会显示警告**：当将更改推送到标记为生产环境的连接时，需要额外的安全确认
