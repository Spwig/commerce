---
title: 동기화 토큰 관리
---

동기화 토큰은 두 개의 Spwig 설치 간에 통신할 수 있는 보안 인증 정보입니다. 설정을 동기화하거나 데이터를 이전하려면 먼저 **수신** 스토어에서 토큰을 생성하고 해당 토큰을 **발신** 스토어에 제공해야 합니다.

## 동기화 토큰 작동 방식

동기화 토큰은 두 개의 Spwig 설치 간 요청을 인증하는 일회성으로 표시되는 API 키입니다. 연결을 설정할 때 원격 스토어는 이 토큰을 사용하여 자신의 스토어에서 읽거나 쓰는 권한이 있음을 증명합니다.

- 토큰은 **연결 대상** (대상) 스토어에서 생성됩니다
- 각 토큰은 생성 직후에만 한 번 표시됩니다
- 토큰은 언제든지 취소하여 즉시 접근을 차단할 수 있습니다
- 한 스토어는 여러 연결에 사용할 수 있는 활성 토큰을 여러 개 가질 수 있습니다

## 토큰 생성

1. 관리자 사이드바에서 **Data Migration > Spwig-to-Spwig Sync**로 이동합니다
2. 동기화 대시보드에서 **Manage Tokens**을 클릭합니다
3. 토큰에 설명을 위한 이름을 입력합니다 (예: "Staging Server" 또는 "Production Sync")
4. **Generate Token**을 클릭합니다
5. **토큰을 즉시 복사**합니다 -- 다시 표시되지 않습니다

> **중요:** 토큰을 안전하게 저장하세요. 토큰을 잃어버리면 새로운 토큰을 생성해야 합니다.

## 토큰 사용

대상 스토어에서 토큰을 받은 후:

1. 연결을 시작할 스토어의 **Spwig-to-Spwig Sync** 대시보드로 이동합니다
2. 새로운 **Settings Sync** 또는 **Full Migration**을 시작합니다
3. 연결 단계에서 대상 스토어의 URL을 입력하고 토큰을 붙여넣습니다
4. **Test Connection**을 클릭하여 작동 여부를 확인합니다
5. 연결은 향후 사용을 위해 저장됩니다

## 토큰 취소

토큰이 유출되거나 더 이상 필요하지 않은 경우:

1. 동기화 대시보드에서 **Manage Tokens**으로 이동합니다
2. 취소하려는 토큰을 찾습니다
3. **Revoke** 버튼을 클릭합니다
4. 취소를 확인합니다

토큰을 취소하면 즉시 효과가 발생합니다. 해당 토큰을 사용하는 모든 활성 연결은 작동을 멈추고 새로운 토큰으로 다시 구성해야 합니다.

## 최선의 실천 방법

모든 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어를 유지합니다.

- **명확하게 이름을 지정하여** 각 토큰이 어떤 연결에 속하는지 알 수 있도록 합니다
- **사용하지 않는 토큰을 취소**하여 보안 노출을 최소화합니다
- 여러 매장을 연결할 때 각 매장에 대해 **별도의 토큰을 생성**하고, 여러 매장에 걸쳐 하나의 토큰을 공유하지 않습니다
- 보안 절차의 일환으로 **정기적으로 토큰을 재생성**하고, 특히 직원 변경 후에는 더욱 주의해야 합니다
