---
title: 전체 시스템 마이그레이션
---

전체 시스템 마이그레이션은 하나의 Spwig 설치에서 다른 설치로 가게의 모든 데이터 - 설정, 상품, 고객, 주문, 미디어 파일 및 기타 모든 데이터 -를 이전합니다. 새로운 서버로 이전하거나 가게의 완전한 복사본을 설정할 때 이 기능을 사용합니다.

## 전체 마이그레이션 사용 시기

- **서버 이전**: 새로운 호스팅 제공업체 또는 서버로 가게 이전
- **스테이징 복사본 생성**: 프로덕션 환경에서 완전한 스테이징 환경 설정
- **재해 복구**: 백업 인스턴스에서 전체 가게 복원

전체 마이그레이션은 설정 동기화가 수행하는 모든 작업을 포함하며, 거래 데이터(상품, 고객, 주문, 리뷰, 재고, 미디어 등)도 포함합니다.

## 마이그레이션되는 항목

전체 마이그레이션은 모든 설정 범주와 다음 데이터 범주를 이전할 수 있습니다:

| 범주 | 설명 |
|------|------|
| 설정 | 사용자 인터페이스, 주문 처리, 결제 게이트웨이, 이메일 설정 등 |
| 상품 | 상품 정보, 가격, 재고, 이미지, 설명, 카테고리 |
| 고객 | 고객 정보, 주문 내역, 이메일, 주소, 프로필 |
| 주문 | 주문 세부 정보, 결제 상태, 배송 상태, 주문 내역 |
| 리뷰 | 고객 리뷰 및 평점 |
| 미디어 | 이미지, 동영상, 문서 등 모든 미디어 파일 |
| 기타 데이터 | 로그, 설정, 캐시, 세션 데이터 등 |

모든 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어를 유지합니다.

| 카테고리 | 설명 |
|----------|-------------|
| **설치된 구성 요소** | 테마, 제공업체 통합 및 유틸리티 구성 요소와 그 패키지 파일 |
| **제품, 카테고리 및 브랜드** | 제품, 변형, 이미지, 카테고리, 브랜드 및 속성 |
| **미디어 라이브러리** | 업로드된 모든 미디어 파일 및 자산 |
| **고객 및 주소** | 고객 계정, 프로필 및 주소 |
| **주문 내역** | 주문, 주문 항목 및 거래 기록 |
| **제품 리뷰** | 고객 리뷰 및 평점 |
| **재고 수준** | 창고별 재고 수량 및 재주문 수준 |
| **디지털 제품 및 라이선스** | 디지털 자산, 라이선스 템플릿 및 라이선스 풀 |
| **기프트 카드 및 쿠폰 사용 내역** | 기프트 카드 잔액 및 쿠폰 사용 기록 |
| **스토어 크레딧 및 지갑** | 고객 지갑 잔액 및 거래 내역 |
| **로열티 프로그램 멤버** | 로열티 멤버, 포인트, 거래 및 배지 |
| **활성 구독** | 구독 플랜, 활성 구독 및 청구 내역 |
| **배송 및 추적** | 배송 기록 및 추적 이벤트 |
| **환불, 반품 및 주문 메모** | 환불 기록, 반품 요청 및 메모 |
| **아フィ리에이트 멤버** | 아피리에이트 계정, 추천 코드 및 수수료 내역 |

## 단계별 가이드

### 단계 1: 소스 인스턴스에 연결

1. 관리 사이드바에서 **데이터 마이그레이션 > Spwig-to-Spwig Sync**로 이동
2. **전체 마이그레이션 시작**을 클릭
3. 소스 스토어(마이그레이션을 **할** 스토어)에 연결합니다:
   - 소스 스토어의 URL을 입력합니다
   - 소스 스토어에서 복사한 동기화 토큰을 붙여넣습니다
   - 연결을 이름으로 지정합니다(예: "구형 프로덕션 서버")
4. **연결 테스트**를 클릭하여 확인합니다
5. **다음**을 클릭합니다

> **중요:** 전체 마이그레이션은 항상 연결된 스토어에서 데이터를 **끌어옵니다**. 이 스토어에 마이그레이션 마법사를 실행합니다. **목적지**(새로운) 스토어에서 실행하세요.

### 단계 2: 범위 선택

마이그레이션에 포함할 데이터 카테고리를 선택합니다. 카테고리는 그룹으로 구성되어 있습니다:

- **설정**: 저장소 구성, 테마, 제공업체, 콘텐츠
- **데이터**: 제품, 고객, 주문, 미디어 및 기타 거래 데이터

일부 카테고리는 의존성이 있습니다(예: 주문은 고객과 제품에 의존합니다). 카테고리를 선택할 때 자동으로 의존성이 포함됩니다.

특수 지표가 있는 카테고리:
- **키 아이콘**: 보안적으로 전송되는 자격 증명을 포함합니다
- **파일 아이콘**: 이진 파일(이미지, 미디어, 패키지)을 포함합니다
- **경고 아이콘**: 프로덕션 환경에서 고려해야 할 특별한 사항이 있습니다

### 단계 3: 사전 점검

이전에 마이그레이션이 시작되기 전, 자동 사전 점검이 확인합니다:

- **연결 상태**: 소스 저장소가 도달 가능하고 인증되었습니다
- **버전 호환성**: 두 저장소 모두 호환 가능한 Spwig 버전을 실행 중입니다
- **디스크 공간**: 미디어 파일을 저장할 충분한 저장 공간이 있습니다
- **데이터베이스 준비**: 대상 데이터베이스가 데이터를 수신할 수 있습니다

점검 중 하나라도 실패하면 진행하기 전에 문제를 해결하는 방법에 대한 구체적인 지침을 보게 됩니다.

### 단계 4: 마이그레이션 진행 상황

마이그레이션은 백그라운드에서 실행됩니다. 안전하게 다른 페이지로 이동할 수 있습니다 - 프로세스는 계속됩니다.

진행 상황 페이지는 다음과 같은 정보를 표시합니다:
- 전체 진행률과 예상 남은 시간
- 카테고별 완료 상태
- 전송 세부 정보가 포함된 실시간 활동 로그
- 미디어 카테고리에 대한 미디어 전송 통계(전송된 파일 및 바이트 수)

많은 제품과 미디어 파일이 있는 대규모 저장소의 경우 마이그레이션에 시간이 소요될 수 있습니다. 미디어 전송 단계는 일반적으로 가장 오래 걸립니다.

### 단계 5: 결과

마이그레이션이 완료되면 결과 페이지는 다음과 같은 정보를 표시합니다:

- 요약 통계(이동, 건너뛴, 실패한 항목)
- 카테고별 세부 정보 및 상태
- 실패한 항목의 오류 세부 정보

## 마이그레이션 후 확인 목록

성공적인 마이그레이션 후, 새 저장소에서 다음 단계를 완료하세요:

1. 새 설치에서 **라이선스를 활성화**하세요
2. 마이그레이션 중 건너뛴 **결제 제공업체 자격 증명을 다시 입력**하세요 (샌드박스/테스트 키는 프로덕션으로 이전되지 않습니다)
3. **DNS를 구성**하여 도메인을 새 서버로 지정하세요
4. 테스트 주문으로 **체크아웃 흐름을 테스트**하세요
5. **이메일 전송**이 올바르게 작동하는지 확인하세요
6. **미디어 파일**과 이미지가 올바르게 로드되는지 확인하세요

## 롤백

풀 마이그레이션이 완료된 후, **24시간** 동안 롤백을 수행할 수 있습니다. 롤백은 대상 스토어에서 마이그레이션된 모든 데이터를 삭제하고, 마이그레이션 전 상태로 복원합니다.

롤백을 수행하려면:
1. 결과 페이지 또는 Sync 대시보드로 이동하세요
2. **마이그레이션 롤백**을 클릭하고 확인하세요
3. 롤백이 완료될 때까지 기다리세요

> **경고:** 롤백은 마이그레이션된 모든 데이터를 영구적으로 삭제합니다. 마이그레이션 후 대상 스토어에서 이루어진 변경 사항(새로운 주문, 고객 가입 등)도 영향을 받습니다.

24시간이 지나면 롤백 옵션이 만료됩니다.

## 팁

- **대상 스토어에서 실행**: 풀 마이그레이션 마법사는 **새로운** 스토어에서 실행되어야 하며, 이전 스토어에서 데이터를 가져옵니다
- **깨끗한 설치로 마이그레이션**: 최상의 결과를 위해, 라이브 전에 신선한 Spwig 설치에서 마이그레이션을 실행하세요
- **디스크 공간 확인**: 대상 스토어가 모든 미디어 파일을 저장할 수 있는 충분한 저장 공간이 있는지 확인하세요
- **소스 실행 유지**: 대상 스토어에서 모든 것이 작동하는지 확인할 때까지 소스 스토어를 종료하지 마세요
- **DNS 전환 계획**: 마이그레이션을 확인한 후 DNS 레코드를 새 서버로 업데이트하는 계획을 세우세요
