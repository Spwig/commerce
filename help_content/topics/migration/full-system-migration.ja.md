---
title: フルシステム移行
---

フルシステム移行は、Spwigの1つのインストールから別のインストールに、ストアのすべてのデータ（設定、商品、顧客、注文、メディアファイル、その他のすべてのデータ）を移行します。新しいサーバーへの移行や、ストアの完全なコピーの作成時に使用してください。

## フル移行の使用タイミング

- **サーバーの移転**: ストアを新しいホスティングプロバイダーやサーバーに移動する場合
- **ステージングコピーの作成**: 本番環境から完全なステージング環境を設定する場合
- **災害復旧**: バックアップインスタンスから完全なストアを復元する場合

フル移行は、設定同期（Settings Sync）が行うすべての内容に加え、トランザクショナルデータ（商品、顧客、注文、レビュー、在庫、メディアなど）も含みます。

## 移行される内容

フル移行はすべての設定カテゴリと以下のデータカテゴリを移行できます：

| カテゴリ | 説明 |
|----------|-------------|
| **インストールされたコンポーネント** | テーマ、プロバイダ統合、およびユーティリティコンポーネントとそのパッケージファイル |
| **商品、カテゴリ、ブランド** | 商品、バリエーション、画像、カテゴリ、ブランド、および属性 |
| **メディアライブラリ** | すべてのアップロードされたメディアファイルとアセット |
| **顧客と住所** | 顧客アカウント、プロフィール、および住所 |
| **注文履歴** | 注文、注文項目、および取引履歴 |
| **商品レビュー** | 顧客のレビューと評価 |
| **在庫レベル** | ワークショップごとの在庫数量と再注文ポイント |
| **デジタル商品とライセンス** | デジタルアセット、ライセンステンプレート、およびライセンスプール |
| **ギフトカードとクーポンの使用** | ギフトカード残高とクーポン使用履歴 |
| **ストアクレジットとウォレット** | 顧客のウォレット残高と取引履歴 |
| **ロイヤルティプログラム会員** | ロイヤルティ会員、ポイント、取引、およびバッジ |
| **アクティブなサブスクリプション** | サブスクリプションプラン、アクティブなサブスクリプション、および請求履歴 |
| **配送と追跡** | 配送記録と追跡イベント |
| **返金、返品、および注文ノート** | 返金記録、返品依頼、およびノート |
| **アフィリエイト会員** | アフィリエイトアカウント、リファラルコード、およびコミッション履歴 |

## ステップバイステップガイド

### ステップ1: ソースインスタンスに接続

1. 管理サイドバーで **Data Migration > Spwig-to-Spwig Sync** に移動します
2. **Start Full Migration** をクリックします
3. ソースストア（移行する**元の**ストア）に接続します:
   - ソースストアのURLを入力します
   - ソースストアから取得した同期トークンを貼り付けます
   - 接続を名付けます（例: "Old Production Server"）
4. **Test Connection** をクリックして確認します
5. **Next** をクリックします

> **重要:** Full Migration は常に接続されたストアからこのストアにデータを**引き出す**ものです。ウィザードを**宛先**（新しい）ストアで実行してください。

### ステップ2: スコープの選択

移行に含めるデータカテゴリを選択します。カテゴリはグループに分類されています:

- **設定**: ストアの構成、テーマ、プロバイダ、コンテンツ
- **データ**: 商品、顧客、注文、メディア、その他のトランザクションデータ

いくつかのカテゴリには依存関係があります（例: 注文は顧客と商品に依存しています）。カテゴリを選択すると、依存関係が自動的に含まれます。

特殊なアイコンを持つカテゴリ:
- **鍵アイコン**: セキュアに転送される資格情報が含まれています
- **ファイルアイコン**: 二進ファイル（画像、メディア、パッケージ）が含まれています
- **警告アイコン**: 本番環境での特別な考慮事項があります

### ステップ3: 事前チェック

移行を開始する前に、自動的な事前チェックが実行され、以下を確認します:

- **接続状態**: ソースストアがアクセス可能で認証済みであるか
- **バージョン互換性**: 両方のストアが互換性のあるSpwigバージョンで動作しているか
- **ディスク容量**: メディアファイルのための十分なストレージが利用可能か
- **データベースの準備状態**: 宛先データベースがデータを受信可能か

チェックに失敗した場合、進行を続ける前に、問題を解決するための具体的なガイドが表示されます。

### ステップ4: 移行の進行状況

移行はバックグラウンドで実行されます。安全に他のページに移動できます。プロセスは継続されます。

進行状況ページには以下が表示されます:
- 全体の進捗率と残りの推定時間
- カテゴリーごとの完了状態
- 転送詳細を含むライブアクティビティログ
- メディアカテゴリのメディア転送統計（転送されたファイル数とバイト数）

多くの商品とメディアファイルを持つ大規模なストアの場合、移行には時間がかかる場合があります。メディア転送フェーズが通常最も時間がかかります。

### ステップ5: 結果

移行が完了した後、結果ページには以下が表示されます:

- サマリ統計（移行済み、スキップされた、失敗した項目）
- カテゴリーごとの詳細とステータス
- すべての失敗した項目のエラー詳細

## 移行後のチェックリスト

成功した移行の後、新しいストアで以下の手順を完了してください:

1. **新しいインストールでライセンスをアクティベート**
2. **移行中にスキップされた支払いプロバイダーの資格情報を再入力**（サンドボックス/テストキーは本番環境に転送されません）
3. **DNSを構成**して、ドメインを新しいサーバーにポイントします
4. **テスト注文でチェックアウトフローをテスト**
5. **メール送信が**正しく動作することを確認
6. **メディアファイルと画像が**正常に読み込まれているかを確認

## ロールバック

フル移行が完了した後、**24時間**以内にロールバックできます。ロールバックは、宛先ストアから移行されたすべてのデータを削除し、移行前の状態に戻します。

ロールバックするには:
1. 結果ページまたはSyncダッシュボードにアクセスします
2. **移行のロールバック**をクリックして確認します
3. ロールバックが完了するのを待ちます

> **警告:** ロールバックは移行されたすべてのデータを永久に削除します。移行後、宛先ストアで行われた変更（新しい注文、顧客の登録など）も影響を受けます。

24時間経過後、ロールバックオプションは失効します。

## ヒント

- **宛先ストアで実行する**: フル移行ウィザードは、**新しい**ストアで実行し、古いストアからデータを取得します
- **クリーンなインストールに移行する**: 最良の結果を得るために、本番運用開始前に、Spwigの**新しい**インストールで移行を実行することをお勧めします
- **ディスク容量を確認する**: 宛先ストアにすべてのメディアファイルを保存する十分なストレージがあることを確認してください
- **ソースを稼働させたままにする**: 宛先ストアですべてが正常に動作することを確認するまで、ソースストアをシャットダウンしないでください
- **DNSの移行を計画する**: 移行を確認した後、DNSレコードを新しいサーバーにポイントするように更新してください
