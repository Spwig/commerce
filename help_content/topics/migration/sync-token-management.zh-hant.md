---
title: 同步令牌管理
---

同步令牌是安全凭证，允许两个 Spwig 安装实例之间进行通信。在您能够同步设置或在商店之间迁移数据之前，需要在 **接收** 商店上生成令牌，并将其提供给 **发送** 商店。

## 同步令牌的工作原理

同步令牌是一次性可见的 API 密钥，用于在两个 Spwig 安装实例之间验证请求。当您设置连接时，远程商店会使用此令牌来证明它有权从您的商店读取或写入数据。

- 令牌是在将要 **连接到** 的商店（目标）上生成的
- 每个令牌只能在生成后立即查看一次
- 任何时候都可以撤销令牌以立即切断访问权限
- 一个商店可以为不同的连接拥有多个活动令牌

## 生成令牌

1. 在管理侧边栏中导航到 **数据迁移 > Spwig-to-Spwig 同步**
2. 在同步仪表板上点击 **管理令牌**
3. 为令牌输入一个描述性名称（例如，"预发布服务器" 或 "生产同步"）
4. 点击 **生成令牌**
5. **立即复制令牌** -- 它将不再显示

> **重要提示：** 请安全地存储令牌。如果您丢失了它，将需要生成一个新的。

## 使用令牌

一旦从目标商店获得令牌：

1. 转到将要发起连接的商店的 **Spwig-to-Spwig 同步** 仪表板
2. 开始新的 **设置同步** 或 **完整迁移**
3. 在连接步骤中，输入目标商店的 URL 并粘贴令牌
4. 点击 **测试连接** 以验证其是否有效
5. 连接将被保存以供将来使用

## 撤销令牌

如果令牌被泄露或不再需要：

1. 在同步仪表板上转到 **管理令牌**
2. 找到要撤销的令牌
3. 点击 **撤销** 按钮
4. 确认撤销

撤销令牌会立即生效。任何使用该令牌的活动连接将停止工作，并需要使用新令牌重新配置。

## 最佳实践

保留所有 markdown 格式、图片路径、代码块和技术术语。

- **為每個令牌提供描述性的名稱**，以便你知道每個令牌屬於哪個連接
- **撤銷未使用的令牌**，以最小化安全風險
- **為每個連接的商店生成獨立的令牌**，而不是在多個商店之間共用一個令牌
- **定期重新生成令牌**，作為您安全流程的一部分，特別是在員工變動後
