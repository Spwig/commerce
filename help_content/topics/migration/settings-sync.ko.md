---
title: 설정 동기화
---

설정 동기화는 두 개의 Spwig 설치 간에 가게 구성 설정을 복사할 수 있게 해줍니다. 이는 스테이징 환경과 프로덕션 환경을 유지하는 데 이상적입니다. 스테이징에서 변경 사항을 구성하고 테스트한 후 실제 가게에 배포하기 전에 사용합니다.

## 설정 동기화 사용 시기

- **스테이징에서 프로덕션으로**: 스테이징 가게에서 설정을 구성한 후 이를 프로덕션으로 전송합니다.
- **프로덕션에서 스테이징으로**: 프로덕션 설정을 스테이징으로 가져와서 일치하는 환경에서 시작합니다.
- **설정 백업**: 프로덕션에서 설정을 백업 인스턴스로 가져와 보호 장치로 사용합니다.

설정 동기화는 구성 데이터만 처리합니다. 제품, 고객, 주문, 미디어 파일은 전송하지 않습니다. 완전한 데이터 전송이 필요한 경우 대신 전체 시스템 마이그레이션을 사용하세요.

## 동기화 가능한 항목

설정 동기화는 다음 범주를 지원합니다:

| 그룹 | 범주 |
|-------|-----------|
| **설정** | 사이트 설정, 세금 및 통화, 세금율, 언어, 블로그 설정, 소셜 공유, 판매 지역 및 창고, 검색 설정, 커스텀 필드, 직원 역할, 고객 분석 |
| **디자인** | 디자인 및 테마, 헤더/푸터/메뉴 |
| **제공업체** | 이메일, SMS/WhatsApp, 결제 제공업체, 배송, SEO 제공업체, 제품 피드, 블로그 소셜 커넥터, POS 설정 |
| **콘텐츠** | 페이지 및 템플릿, 블로그 게시물, 공지사항, 폼, 제품 컬렉션 |
| **상거래** | 상거래 규칙 (쿠폰, 프로모션, 충성도, 구독), 파트너 프로그램, 웹훅 및 통합 |

> **참고:** 자격 증명을 포함한 범주(결제 제공업체, 배송 계정 등)는 키 아이콘으로 표시됩니다. API 키 및 비밀은 보안적으로 전송되지만 OAuth 기반 통합에 대해 다시 입력해야 할 수 있습니다.

## 단계별 가이드

### 단계 1: 연결 설정

1.

관리자 사이드바에서 **데이터 마이그레이션 > Spwig-to-Spwig 동기화**로 이동합니다.
2.

**설정 동기화 시작**을 클릭합니다.
3.

모든 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어를 유지합니다.

저장된 연결을 선택하거나 새 연결을 만듭니다:
   - 원격 스토어의 URL을 입력하세요 (예: `https://staging.yourstore.com`)
   - 원격 스토어에서 생성된 동기화 토큰을 붙여넣으세요
   - 연결에 설명적인 이름을 지정하세요
   - 역할을 설정하세요 (Staging, Production, Backup 또는 Other)
4.

**연결 테스트**를 클릭하여 작동 여부를 확인하세요
5.

**다음**을 클릭하여 계속하세요

### 단계 2: 범주 및 방향 선택

**방향:**
- **Pull** -- 연결된 스토어에서 이 스토어로 설정을 복사합니다
- **Push** -- 이 스토어에서 연결된 스토어로 설정을 복사합니다

**동기화 모드:**
- **추가 및 업데이트** -- 새 항목을 추가하고 기존 항목을 업데이트하지만, 아무것도 삭제하지 않습니다. 이는 가장 안전한 옵션입니다.
- **정확한 복사** -- 대상이 소스와 정확히 일치하도록 하며, 대상에 있지만 소스에 없는 항목을 제거합니다. 주의해서 사용하세요.

필요한 범주를 선택한 후 **다음**을 클릭하세요.

### 단계 3: 변경 사항 미리보기

변경 사항이 적용되기 전에, 각 범주에 대해 정확히 무엇이 추가, 수정, 또는 제거될지 상세한 미리보기를 볼 수 있습니다. 신중하게 검토하세요.

생산 연결로 푸시하는 경우, 변경 사항이 실제 스토어에 영향을 줄 것임을 이해했다는 확인이 필요합니다.

준비가 되면 **동기화 시작**을 클릭하세요.

### 단계 4: 진행 상황 모니터링

동기화는 백그라운드에서 실행됩니다. 진행 페이지에서 벗어나도 안전합니다 -- 동기화는 계속 실행됩니다.

진행 페이지는 다음과 같은 정보를 표시합니다:
- 전체 완료 비율과 예상 남은 시간
- 범주별 진행 상황과 성공/실패 수
- 확장하여 상세한 출력을 볼 수 있는 실시간 활동 로그

## 롤백

동기화가 완료된 후, 변경 사항을 롤백할 수 있는 **24시간**이 있습니다. 롤백은 모든 영향을 받은 설정의 이전 상태를 복원합니다.

롤백 방법:
1. **동기화 대시보드**로 이동하세요
2. 완료된 작업을 찾으세요
3. **롤백**을 클릭하고 확인하세요

24시간 후, 롤백 옵션이 만료되고 변경 사항은 영구적으로 적용됩니다.

## 팁

모든 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어를 유지하세요.

- **먼저 스테이징 환경에서 테스트하세요**: 프로덕션으로 푸시하기 전에 결과를 확인하기 위해 먼저 스테이징 환경으로 동기화하세요
- **Add & Update 모드 사용**: 이 모드는 기존 데이터를 삭제하지 않기 때문에 가장 안전한 모드입니다
- **미리보기를 신중히 확인하세요**: diff 미리보기에서는 변경 사항이 적용되기 전에 정확히 무엇이 변경되는지 보여줍니다
- **프로덕션 연결은 경고를 표시합니다**: 프로덕션으로 푸시할 때 추가적인 안전 확인이 필요합니다
