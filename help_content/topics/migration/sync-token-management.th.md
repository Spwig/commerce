---
title: การจัดการ Token การซิงค์
---

Token การซิงค์คือข้อมูลประจำตัวที่ปลอดภัย ซึ่งอนุญาตให้การติดตั้ง Spwig สองแห่งสามารถสื่อสารกันได้ ก่อนที่คุณจะสามารถซิงค์การตั้งค่าหรือย้ายข้อมูลระหว่างร้านค้า คุณจำเป็นต้องสร้าง Token ที่ร้านค้า **ปลายทาง** และให้ Token นั้นแก่ร้านค้า **ต้นทาง**

## วิธีการทำงานของ Token การซิงค์

Token การซิงค์คือคีย์ API ที่สามารถมองเห็นได้เพียงครั้งเดียว ซึ่งใช้ยืนยันการร้องขอระหว่างการติดตั้ง Spwig สองแห่ง เมื่อคุณตั้งค่าการเชื่อมต่อ ร้านค้าที่อยู่ไกลจะใช้ Token นี้เพื่อแสดงว่ามีสิทธิ์ในการอ่านหรือเขียนข้อมูลในร้านค้าของคุณ

- Token ถูกสร้างขึ้นที่ร้านค้าที่จะ **เชื่อมต่อเข้ามา** (เป้าหมาย)
- Token แต่ละตัวสามารถมองเห็นได้เพียงครั้งเดียวทันทีหลังจากสร้าง
- Token สามารถถูกยกเลิกได้ทุกเมื่อเพื่อตัดการเชื่อมต่อทันที
- ร้านค้าสามารถมี Token ที่ใช้งานได้หลายตัวสำหรับการเชื่อมต่อที่แตกต่างกัน

## การสร้าง Token

1. ไปที่ **Data Migration > Spwig-to-Spwig Sync** บนแถบด้านข้างของแดชบอร์ดผู้ดูแลระบบ
2. คลิก **Manage Tokens** บนแดชบอร์ดการซิงค์
3. ใส่ชื่อที่อธิบายได้ชัดเจนสำหรับ Token (เช่น "Staging Server" หรือ "Production Sync")
4. คลิก **Generate Token**
5. **คัดลอก Token ทันที** -- Token จะไม่แสดงอีกครั้ง

> **สำคัญ:** จัดเก็บ Token อย่างปลอดภัย หากคุณสูญเสียมัน คุณจะต้องสร้าง Token ใหม่

## การใช้ Token

เมื่อคุณมี Token จากร้านค้าปลายทาง:

1. ไปที่แดชบอร์ด **Spwig-to-Spwig Sync** บนร้านค้าที่จะเริ่มการเชื่อมต่อ
2. เริ่ม **Settings Sync** หรือ **Full Migration** ใหม่
3. ในขั้นตอนการเชื่อมต่อ ใส่ URL ของร้านค้าปลายทางและวาง Token
4. คลิก **Test Connection** เพื่อตรวจสอบว่าการเชื่อมต่อทำงานได้
5. การเชื่อมต่อจะถูกบันทึกไว้เพื่อใช้งานในอนาคต

## การยกเลิก Token

หาก Token ถูกเปิดเผยหรือไม่จำเป็นอีกต่อไป:

1. ไปที่ **Manage Tokens** บนแดชบอร์ดการซิงค์
2. ค้นหา Token ที่คุณต้องการยกเลิก
3. คลิกปุ่ม **Revoke**
4. ยืนยันการยกเลิก

การยกเลิก Token จะมีผลทันที การเชื่อมต่อที่ใช้งาน Token นั้นอยู่จะหยุดทำงานและต้องตั้งค่าใหม่ด้วย Token ใหม่
