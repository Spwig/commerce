---
title: 데이터베이스 백업
---

정기적인 백업은 하드웨어 고장, 실수로 인한 삭제 및 기타 예상치 못한 사건으로부터 가게의 데이터 — 주문, 고객, 제품 및 설정 —을 보호합니다. Spwig의 백업 시스템은 수동으로 백업을 생성할 수 있으며, 자동 백업 일정을 설정하고, 로컬에 백업을 다운로드할 수 있으며, 저장된 백업에서 복원하거나, Amazon S3 또는 Google Drive와 같은 원격 저장소 위치로 백업을 복사할 수 있습니다.

**Management > System Metrics**으로 이동하여 도구 모음 링크를 사용하여 백업 도구에 접근합니다.

![System Dashboard with backup tools](/static/core/admin/img/help/database-backups/system-dashboard.webp)

## 수동 백업 생성

중요한 변경 사항을 수행하기 전에 백업을 실행하세요 — 예를 들어, 제품 임포트, 테마 업데이트 또는 플랫폼 업그레이드 등입니다.

1. **Management > System Metrics**으로 이동합니다.
2. 도구 모음에서 **Create Full Backup**을 클릭합니다.
3. 백업에 대한 설명적인 **이름**을 입력합니다 (예: `before-july-import`).
4. 이 백업을 생성한 이유를 상기시키기 위해 선택적으로 **설명**을 추가합니다.
5. **백업 유형**을 선택합니다:
   - **Full System** — 데이터베이스와 모든 미디어 파일을 백업합니다 (권장)
   - **Database Only** — 업로드된 이미지 및 파일을 제외하고 가게 데이터만 백업합니다.
6. **압축** 방식을 선택합니다 (`gzip`이 기본값이며 대부분의 가게에 적합합니다).
7. **Backup**을 클릭합니다.

Spwig은 백업을 백그라운드에서 생성합니다. 진행 상태 표시기가 현재 단계를 보여줍니다. 완료되면 백업은 **Database Backups** 목록에 **Completed** 상태와 파일 크기와 함께 나타납니다.

## 백업 다운로드

완료된 백업을 다운로드하여 컴퓨터에 로컬 복사본을 유지할 수 있습니다.

1. **Management > Database Backups**으로 이동합니다.
2. 다운로드하려는 백업을 찾습니다.
3. 해당 백업 옆의 **Download** 버튼을 클릭합니다.

백업 파일은 압축된 아카이브로 다운로드됩니다. 안전한 장소 — 예를 들어, 별도의 장치 또는 클라우드 저장소 —에 저장하여 서버와 독립적인 복사본을 유지하세요.

## 자동 백업 일정 설정

자동 백업은 배경에서 실행되며, 당신의 데이터는 수동 백업을 생성하는 것을 잊었어도 보호됩니다.

1. **Management > System Metrics**으로 이동하세요
2. **Backup Schedule**을 클릭하세요
3. **Enable Automatic Backups**을 선택하세요
4. **Frequency**를 설정하세요:
   - **Daily** — 지정한 시간에 하루에 한 번 실행됩니다
   - **Weekly** — 선택한 요일에 주에 한 번 실행됩니다
   - **Monthly** — 월의 특정 날에 실행됩니다
5. 백업이 실행되어야 할 **Time**을 설정하세요 (서버 시간, 일반적으로 UTC — 03:00 AM은 낮은 트래픽 시간입니다)
6. **Backup Type**을 선택하세요 (Full System 또는 Database Only)
7. **Retention Days**를 설정하세요 — 이 날짜보다 오래된 백업은 자동으로 삭제됩니다 (기본값: 30일)
8. 선택적으로 **Encrypt Backup**을 선택하여 백업 파일을 저장 시 암호화하세요
9. 원격 저장소 위치가 구성되어 있다면, **Remote Destinations**에서 이를 선택하여 예약된 백업을 자동으로 업로드하세요
10. **Save Schedule**을 클릭하세요

**Next Run** 타임스탬프는 즉시 업데이트되고, 다음 자동 백업이 실행될 시간을 표시합니다.

## 백업에서 복원하기

복원은 현재 가게 데이터를 백업의 내용으로 대체합니다. 이 기능은 데이터 손실에서 복구하거나 원치 않는 변경을 되돌리기 위해 사용합니다.

> **중요:** 복원은 현재 데이터를 백업 데이터로 대체합니다. 복원 중에는 가게가 유지보수 모드에 들어갑니다. 복원을 실행하기 전에 팀에 알리세요.

1. **Management > System Metrics**으로 이동하세요
2. 툴바에서 **Restore**을 클릭하세요
3. 복원 목록은 사용 가능한 모든 백업과 그 날짜 및 크기를 보여줍니다
4. 사용하고자 하는 백업 옆의 **Restore**을 클릭하세요
5. 확인 화면을 검토하세요 — 이 화면은 정확히 무엇이 대체될지를 나열합니다
6. 확인 문구가 요청된다면 입력한 후 **Execute Restore**을 클릭하세요

Spwig는 복원이 단계를 거치며 진행되는 동안 진행률 표시줄을 보여줍니다 (현재 상태 백업, 원격 저장소에서 백업 다운로드, 데이터베이스 복원, 미디어 파일 복원).

완료되면 매장은 자동으로 유지보수 모드를 종료합니다。

## 원격 저장소 설정

원격 저장소는 백업을 자동으로 외부 목적지로 복사합니다 — Amazon S3, Google Drive, Dropbox 또는 SFTP 서버입니다. 이는 서버 수준의 장애로부터 보호합니다。

1. **관리 > 시스템 메트릭**으로 이동합니다
2. **원격 저장소**를 클릭합니다
3. **목적지 추가**를 클릭합니다
4. 설정 마법사는 세 가지 단계를 안내합니다:
   - **단계 1**: 저장소 유형을 선택합니다 (S3, Google Drive, Dropbox 또는 SFTP)
   - **단계 2**: 선택한 제공업체의 자격 증명을 입력합니다 (아래 세부 정보를 참조하세요)
   - **단계 3**: 목적지를 이름 지정하고 연결을 테스트합니다
5. 연결 테스트가 성공하면 **저장**을 클릭합니다

### Amazon S3 (및 S3 호환 서비스)

필요한 사항:
- **액세스 키 ID** 및 **비밀 액세스 키**를 AWS IAM 사용자에서 가져옵니다
- **버킷 이름** — 백업을 업로드할 S3 버킷
- **지역** — 버킷이 위치한 AWS 지역 (예: `us-east-1`)
- 선택적으로 **프리픽스** (버킷 내의 폴더 경로, 예: `spwig-backups/`)

S3 호환 서비스 (Backblaze B2, Wasabi, MinIO 등)는 동일한 방식으로 작동합니다 — 요청 시 사용자 지정 엔드포인트 URL을 입력합니다。

### Google Drive

자격 증명 단계에서 **Google과 연결**을 클릭합니다. Spwig은 Google OAuth 창을 엽니다 — 로그인하고 파일 업로드에 대한 권한을 부여합니다. 복사할 자격 증명이 없습니다。

### Dropbox

자격 증명 단계에서 **Dropbox와 연결**을 클릭합니다. Dropbox에 로그인하고 액세스를 승인합니다. 백업은 Dropbox의 `Apps/Spwig` 폴더에 업로드됩니다。

### SFTP

필요한 사항:
- **SFTP 서버의 호스트 이름**
- **포트** (기본값: 22)
- **사용자 이름** 및 **비밀번호** (또는 SSH 개인 키)
- **원격 경로** — 서버에 백업을 업로드할 디렉터리

### 목적지를 기본으로 설정

**원격 저장소** 페이지에서, 목적지 옆의 토글을 클릭하여 해당 목적지를 **기본**으로 설정합니다。


기본 대상은 매번 선택할 필요 없이 모든 백업 — 수동 백업과 예약 백업 —을 자동으로 받습니다.

## 팁

- 제품 임포트, 테마 편집, 플랫폼 업그레이드 또는 할인 캠페인과 같은 중요한 변경 사항 전에 수동 백업을 실행하세요
- 서버에 문제가 생겨도 백업이 유지되도록 최소한 하나의 원격 저장소 대상을 설정하세요
- **보존 일수** 설정은 로컬 백업이 얼마나 오래 유지되는지를 제어합니다 — 대부분의 매장에서는 30일이 합리적인 기본값이지만, 저장 공간이 허용된다면 이를 늘릴 수 있습니다
- 복원 후에는 매장의 유지보수 모드를 수동으로 해제하기 전에 몇 개의 주문과 제품을 확인하여 데이터가 올바르게 보이는지 확인하세요
- 암호화된 백업은 보안을 한 층 더 강화하지만 복원하려면 복호화 키가 필요합니다 — 이를 잃지 마세요
