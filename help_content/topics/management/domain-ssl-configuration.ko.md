---
title: 도메인 및 SSL 구성
---

이 가이드는 Spwig 스토어에 사용자 정의 도메인을 연결하고 보안 HTTPS 액세스를 위한 SSL 인증서를 설정하는 방법을 설명합니다. 설치 중에 도메인을 구성하거나 나중에 추가할 수 있습니다.

## 설치 후 도메인 추가

Spwig을 설치할 때 도메인 없이 설치한 경우(서버 IP 주소를 사용하여), 언제든지 도메인을 추가할 수 있습니다.

### 단계 1: DNS 설정

도메인 등록업체 또는 DNS 제공업체에서:

1. 도메인(또는 서브도메인)을 서버 IP 주소로 가리키는 **A 레코드**를 생성합니다.
2. `shop.example.com`과 같은 서브도메인을 사용하는 경우, `shop`에 대한 A 레코드를 생성합니다.
3. DNS 전파를 기다리세요 — 일반적으로 5~60분이 소요됩니다.

DNS 레코드가 작동하는지 확인합니다:

```bash
dig +short shop.example.com
```

이 명령어는 서버 IP 주소를 반환해야 합니다.

### 단계 2: 도메인 구성 스크립트 실행

서버에 SSH 접속하고 Spwig 설치 디렉터리로 이동합니다:

```bash
./configure-domain.sh
```

스크립트는 다음과 같은 작업을 수행합니다:

1. 도메인 이름을 요청합니다.
2. DNS가 서버로 가리키는지 확인합니다.
3. 스토어 구성 업데이트
4. Let's Encrypt에서 무료 SSL 인증서를 가져옵니다.
5. 웹 서버를 HTTPS 사용으로 구성합니다.
6. 관련 서비스를 재시작합니다.

이제 스토어는 `https://yourdomain.com`에서 접근 가능합니다.

### 단계 3: 스토어 설정 업데이트

도메인을 추가한 후 관리자 패널에 로그인하고 **Store Settings**로 이동합니다. **Store URL**이 새 도메인과 일치하는지 확인합니다. 이는 이메일, 영수증 및 링크가 올바른 주소를 사용하도록 보장합니다.

## SSL 인증서

### 자동 SSL (Let's Encrypt)

**독립 모드**에서 설치자는 Let's Encrypt에서 무료 SSL 인증서를 자동으로 가져옵니다. 이러한 인증서는 다음과 같은 특징을 가지고 있습니다:

- 모든 주요 브라우저에서 신뢰됨
- 90일 유효 기간
- 자동 갱신 — 매일 갱신 확인이 실행되며, 인증서가 30일 미만 남았을 때 자동으로 갱신됨
- 정확한 도메인을 커버함 (예: `shop.example.com`)

수동으로 갱신을 관리할 필요가 없습니다.

### 자체 서명 인증서


일부 상황에서는 Spwig가 자체 서명 인증서를 사용합니다:

- **로컬 모드** 설치(개발/테스트)
- Let's Encrypt가 서버에 도달할 수 없는 경우(포트 80이 방화벽에 차단됨, DNS가 아직 전파되지 않음)
- 도메인을 구성하지 않은 경우(IP 전용 액세스)

자체 서명 인증서는 트래픽을 암호화하지만 브라우저에 의해 신뢰되지 않으므로 방문자는 보안 경고를 보게 됩니다. 이는 테스트에 적합하지만 프로덕션에서는 사용하지 않아야 합니다.

### Sidecar 모드 SSL

**sidecar 모드**에서 기존 웹 서버(Apache, Nginx, Caddy 등)가 SSL 종료를 처리합니다. Spwig는 프록시 뒤의 HTTP 포트에서 실행됩니다. 일반적으로 주 웹 서버에서 SSL을 구성하듯이 구성하면 됩니다.

설치 프로그램은 웹 서버에 추가할 수 있는 프록시 구성 블록을 생성합니다. Nginx의 경우 다음과 유사합니다:

```nginx
location / {
    proxy_pass http://127.0.0.1:8080;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

## 도메인 변경

다른 도메인으로 전환하려면:

1. 새 도메인의 DNS를 설정(A 레코드가 서버로 가리킴)
2. 새 도메인으로 `./configure-domain.sh`를 다시 실행
3. 스크립트는 모든 설정을 업데이트하고 새 인증서를 취득한 후 서비스를 재시작합니다
4. 관리 패널의 **Store Settings**에서 새 URL로 업데이트

설정이 업데이트되면 이전 도메인은 더 이상 작동하지 않습니다.

## 문제 해결

### "DNS 검증 실패"

configure-domain 스크립트는 인증서를 요청하기 전에 도메인이 서버로 가리키는지 확인합니다. 이 검사가 실패하면:

- `dig +short yourdomain.com`으로 A 레코드가 올바른지 확인합니다
- DNS 전파가 완료될 때까지 몇 분 더 기다립니다
- 정확한 도메인 또는 서브도메인(와일드카드가 아님)을 구성하고 있는지 확인합니다

### "Let's Encrypt 요청 제한 도달"

Let's Encrypt는 도메인당 주간 기준으로 인증서 요청을 5회로 제한합니다. 이 제한에 도달하면:


- 7일 후 다시 시도하세요
- 이 기간 동안 다른 서브도메인을 사용하세요
- 기다리는 동안 HTTP 또는 자체 서명 인증서를 통해 가게에 접근할 수 있습니다

### "포트 80에 도달할 수 없음"

Let's Encrypt는 도메인 소유권을 확인하기 위해 서버의 포트 80에 연결해야 합니다. 다음을 확인하세요:

- 방화벽이 포트 80의 입구 TCP 트래픽을 허용하고 있는지
- 포트 80을 차단하는 다른 애플리케이션이 없는지
- 클라우드 제공업체의 보안 그룹 또는 네트워크 방화벽이 포트 80을 허용하고 있는지

### 인증서 갱신 실패

자동 갱신이 실패할 경우 인증서는 90일 후 만료됩니다. 수동으로 갱신하려면:

```bash
docker exec spwig_nginx certbot renew
docker exec spwig_nginx nginx -s reload
```

이 명령이 실패할 경우 갱신 로그를 확인하여 세부 정보를 확인하세요. 가장 흔한 원인은 초기 설치 후 방화벽 변경으로 인해 포트 80이 차단된 경우입니다.
