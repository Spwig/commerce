---
title: التحديثات والصيانة
---

تتلقى Spwig تحديثات منتظمة تحتوي على ميزات جديدة وتحسينات في الأداء وإصلاحات الأمان. يغطي هذا الدليل كيفية ترقية تثبيتك، واستخدام أداة التشخيص، وتنفيذ مهام الصيانة.

## ترقية Spwig

### قبل أن تبدأ الترقية

1. **أنشئ نسخة احتياطية** — انتقل إلى **الإدارة > مؤشرات النظام > إنشاء نسخة احتياطية كاملة** أو قم بتشغيل نسخة احتياطية من خلال سطر الأوامر. هذه هي وسيلة الأمان الخاصة بك في حالة حدوث أي شيء غير متوقع.
2. **تحقق من الإصدار الحالي** — يظهر في **الإدارة > مؤشرات النظام** أو في قاعدة بيانات المشرف.
3. **اقرأ ملاحظات الإصدار** — متاحة في لوحة إدارة Spwig تحت **الإدارة > تحديثات المكونات** عند اكتشاف إصدار جديد.

### تنفيذ الترقية

ابدأ جلسة SSH على خادمك وانتقل إلى دليل تثبيت Spwig (عادة `/opt/spwig`):

```bash
./upgrade.sh
```

برنامج الترقية:

1. **التحقق المبدئي** — يتحقق من مساحة القرص وصحة Docker وحالة الخدمة
2. **اختبار التحديثات القاعدة البيانات** — يختبر أن التغييرات في قاعدة البيانات ستُطبَّق بشكل نظيف دون إجراء أي تغييرات فعليّة
3. **الدخول إلى وضع الصيانة** — يظهر متجرك صفحة الصيانة للمستخدمين أثناء الترقية
4. **إنشاء نسخة احتياطية** — نسخة احتياطية تلقائية قبل إجراء أي تغييرات
5. **إنهاء العمال الخلفية** — ينتظر حتى انتهاء المهام الجارية (إرسال البريد الإلكتروني، الترجمات) بشكل سلس
6. **تنزيل الصور الجديدة** — يقوم بتنزيل التطبيق المحدث من سجل Spwig
7. **تطبيق تحديثات قاعدة البيانات** — يقوم بتحديث هيكل قاعدة البيانات الخاصة بك إلى الإصدار الجديد
8. **إعادة تشغيل الخدمات** — يبدأ التطبيق بإصداره الجديد
9. **التحقق من الصحة** — يتحقق من أن جميع الخدمات تعمل بشكل صحيح
10. **الخروج من وضع الصيانة** — يعود متجرك إلى العمل

إذا فشل التحقق من الصحة بعد الترقية، يقوم البرنامج **بإعادة التراجع تلقائيًا** إلى الإصدار السابق ويستعيد النسخة الاحتياطية.

### خيارات الترقية

```bash
./upgrade.sh              # ترقية معيارية مع وضع الصيانة
./upgrade.sh --dry-run    # تحقق من التغييرات التي ستتم دون تطبيقها
```

## الأداة التشخيصية

تتضمن Spwig أداة تشخيصية مدمجة تتحقق من جميع تثبيتاتك من أجل اكتشاف المشكلات:

```bash
./doctor.sh
```

تتحقق الأداة من:

| الفئة | ما تتحقق منه |
|----------|---------------|
| **النظام** | مساحة القرص، استخدام الذاكرة العشوائية، حمل المعالج |
| **الحاويات** | صحة محرك Docker، حالات الحاويات، إصدارات الصور |
| **قاعدة البيانات** | اتصال PostgreSQL، حالة الهجرة، صحة حوض الاتصال |
| **الذاكرة المؤقتة** | اتصال Redis، استخدام الذاكرة |
| **تخزين الكائنات** | اتصال MinIO، توفر الحاويات |
| **الشبكة** | حل DNS، توفر المنافذ، صلاحية شهادة SSL |
| **التطبيق** | نقاط نهاية صحة الخدمة، حالة العمال الخلفية |

يظهر كل فحص نتيجة ناجحة أو فاشلة مع التفاصيل إذا كانت هناك مشكلة.

### الوضع التصحيح التلقائي

للتعامل مع المشكلات الشائعة، يمكن للأداة محاولة إصلاحات تلقائية:

```bash
./doctor.sh --fix
```

يمكن للتصحيح التلقائي حل:

- الحاويات المتوقفة (إعادة تشغيلها)
- الاتصالات القديمة مع قاعدة البيانات (إعادة تدوير حوض الاتصال)
- الشهادات SSL المنتهية الصلاحية (تشغيل التجديد)
- القرص الكامل الناتج عن صور Docker القديمة (إزالة الصور غير المستخدمة)

تشرح الأداة دائمًا ما ستقوم بإصلاحه قبل اتخاذ أي إجراء.

## الوضع الصيانة

يعرض الوضع الصيانة الزوار صفحة "المتجر غير متاح مؤقتًا" أثناء إجراء التغييرات. ويبقى لوحة إدارة المشرف متاحة.

### تمكين الوضع الصيانة

من لوحة إدارة المشرف: **إعدادات المتجر > الصيانة > تمكين وضع الصيانة**

أو من سطر الأوامر:

```bash
docker exec spwig_shop python manage.py maintenance on
```

### تعطيل الوضع الصيانة

من لوحة إدارة المشرف: قم بتعطيل مفتاح وضع الصيانة.

أو من سطر الأوامر:

```bash
./go-live.sh
```

### تجاوز الوصول أثناء الصيانة

بينما يكون وضع الصيانة نشطًا، يمكنك الوصول إلى المتجر بشكل طبيعي بإضافة معلمة سرية إلى عنوان URL. يتم عرض السر الخاص بالتجاوز في ملف التكوين `.env` الخاص بك تحت `MAINTENANCE_SECRET`.

## إدارة الخدمات

### عرض حالة الخدمة

تحقق من حالة جميع خدمات Spwig:

```bash
docker compose ps
```

هذا يعرض كل خدمة، وحالتها (تشغيل، توقف، إعادة التشغيل)، وحالة صحتها.

### عرض السجلات

تحقق من سجلات خدمة معينة:

```bash
docker logs spwig_shop          # سجلات التطبيق
docker logs spwig_celery         # سجلات العمال الخلفية
docker logs spwig_nginx          # سجلات الوصول إلى خادم الويب
docker logs spwig_db             # سجلات قاعدة البيانات
```

أضف `--tail 100` لعرض آخر 100 سطر، أو `--follow` لمتابعة السجلات في الوقت الفعلي.

### إعادة تشغيل خدمة

إذا كانت هناك حاجة لإعادة تشغيل خدمة معينة:

```bash
docker compose restart shop      # إعادة تشغيل التطبيق
docker compose restart celery    # إعادة تشغيل العمال الخلفية
docker compose restart nginx     # إعادة تشغيل خادم الويب
```

لإعادة تشغيل جميع الخدمات:

```bash
docker compose restart
```

## تحديثات المكونات

تتميز Spwig بسوق للمكونات حيث يمكنك تثبيت قوالب ومزودي الدفع وتكاملات الشحن وتوسعات أخرى. يتم تحديث المكونات بشكل مستقل عن منصة النظام الأساسية.

انتقل إلى **الإدارة > تحديثات المكونات** للتحقق من تحديثات المكونات المتاحة. يتم تنزيل التحديثات وتطبيقها تلقائيًا عندما تصدق عليها.

## نصائح

- **قم بالتحديث بانتظام** - البقاء على أحدث إصدار يضمن لك وجود إصلاحات الأمان ووصولك إلى الميزات الجديدة
- **ابدأ دائمًا بالنسخ الاحتياطي** - على الرغم من أن نسخة التحديث تنشئ نسخة احتياطية تلقائية، فإن امتلاك نسخة احتياطية الخاصة بك يوفر سلامة إضافية
- **قم بتشغيل الطبيب بعد المشكلات** - إذا تصرف متجرك بشكل غير متوقع، فإن `./doctor.sh` هو أسرع طريقة لتحديد المشكلات
- **جدولة التحديثات في أوقات منخفضة المرور** - وضع الصيانة يقطع الوصول المؤقت للعملاء، لذا قم بالتحديث في أوقات هادئة
- **احتفظ بمساحة القرص متوفرة** - تحتاج التحديثات إلى مساحة مؤقتة لصور جديدة والنسخ الاحتياطية. حافظ على وجود ما لا يقل عن 5 جيجابايت خالية.
