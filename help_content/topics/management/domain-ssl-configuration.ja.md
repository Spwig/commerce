---
title: ドメインとSSLの設定
---

このガイドでは、カスタムドメインをSpwigストアに接続し、セキュアなHTTPSアクセスのためにSSL証明書を設定する方法を説明します。ドメインの設定はインストール時に実施するか、後から追加することもできます。

一部の状況では、Spwigは自己署名証明書を使用します:

- **ローカルモード**のインストール（開発/テスト）
- Let's Encryptがサーバーに到達できない場合（ポート80がファイアウォールでブロックされている、DNSがまだ伝播していない）
- ドメインが設定されていない場合（IPのみでのアクセス）

自己署名証明書は通信を暗号化しますが、ブラウザから信頼されていません — 訪問者はセキュリティ警告を表示します。これはテストには許容されますが、本番環境では使用しないでください。

### SidecarモードSSL

**Sidecarモード**では、既存のWebサーバー（Apache、Nginx、Caddyなど）がSSL終了を処理します。Spwigはプロキシの後ろでHTTPポート上に実行されます。通常通りメインのWebサーバーでSSLを構成してください。

インストーラーは、Webサーバーに追加できるプロキシ構成ブロックを生成します。Nginxの場合、以下のように見えます:

```nginx
location / {
    proxy_pass http://127.0.0.1:8080;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

## ドメインの変更

別のドメインに切り替えるには:

1. 新しいドメインのDNSを設定（サーバーへのAレコード）
2. 新しいドメインで`./configure-domain.sh`を再度実行
3. スクリプトはすべての構成を更新し、新しい証明書を取得し、サービスを再起動します
4. 管理パネルの**Store Settings**で新しいURLを更新します

構成が更新されると、古いドメインは動作しなくなります。

## トラブルシューティング

### "DNS validation failed"

configure-domainスクリプトは証明書を要求する前に、ドメインがサーバーを指していることを確認します。このチェックが失敗した場合:

- `dig +short yourdomain.com`でAレコードが正しいか確認してください
- DNS伝播のために数分間待ってください
- 正確なドメインまたはサブドメイン（ワイルドカードではない）を構成しているか確認してください

### "Let's Encrypt rate limit reached"

Let's Encryptはドメインごとに週に5回の証明書要求を制限しています。この制限に達した場合:


- 7日間待ってから再度試してください
- その間、別のサブドメインを使用してください
- 待っている間、ストアはHTTP経由または自己署名証明書を使用してアクセス可能です

### "ポート80に到達できません"

Let's Encryptは、ドメイン所有権を確認するためにサーバーのポート80に接続する必要があります。以下の点を確認してください:

- ファイアウォールがポート80のインバウンドTCPを許可していますか
- 他のアプリケーションがポート80をブロックしていないか確認してください
- クラウドプロバイダーのセキュリティグループまたはネットワークファイアウォールがポート80を許可していますか

### 証明書の更新失敗

自動更新が失敗した場合、証明書は90日後に期限切れになります。手動で更新するには:

```bash
docker exec spwig_nginx certbot renew
docker exec spwig_nginx nginx -s reload
```

この操作が失敗した場合は、更新ログを確認してください。最も一般的な原因は、初期インストール後にファイアウォールの変更によりポート80がブロックされていることです。
