---
title: 升級與維護
---

Spwig 會定期接收包含新功能、效能改進和安全性修復的更新。本指南說明如何升級您的安裝、使用診斷工具以及處理維護任務。

## 升級 Spwig

### 升級前

1. **建立備份** — 轉到 **管理 > 系統指標 > 建立完整備份** 或從命令列執行備份腳本。如果發生任何問題，這是您的安全網。
2. **檢查當前版本** — 可在 **管理 > 系統指標** 或管理員儀表板頁腳中看到。
3. **閱讀發行說明** — 當檢測到新版本時，可在管理面板的 **管理 > 組件更新** 下找到。

### 執行升級

透過 SSH 登入您的伺服器，並導航到您的 Spwig 安裝目錄（通常為 `/opt/spwig`）：

```bash
./upgrade.sh
```

升級腳本：

1. **預飛行檢查** — 驗證磁碟空間、Docker 健康狀態和服務狀態
2. **乾運行資料庫遷移** — 測試資料庫變更是否能順利套用，而實際上不進行任何更改
3. **進入維護模式** — 在升級期間，您的商店會向訪客顯示維護頁面
4. **建立備份** — 在進行更改前自動建立安全備份
5. **排空背景工作人員** — 等待進行中的任務（郵件發送、翻譯）順利完成
6. **拉取新映像** — 從 Spwig 倉庫下載更新後的應用程式
7. **套用資料庫遷移** — 更新資料庫結構以適應新版本
8. **重新啟動服務** — 使用新版本啟動應用程式
9. **健康檢查** — 驗證所有服務是否正確運行
10. **退出維護模式** — 您的商店重新在線

如果升級後健康檢查失敗，腳本會 **自動回滾** 到上一個版本並恢復備份。

### 升級選項

```bash
./upgrade.sh              # 帶維護模式的標準升級
./upgrade.sh --dry-run    # 檢查會有什麼改變，但不實際套用
```

## 診斷工具

Spwig 內建了一個診斷工具，用來檢查整個安裝是否存在問題：

```bash
./doctor.sh
```

doctor 會檢查以下項目：

| 分類 | 檢查內容 |
|----------|---------------|
| **系統** | 硬碟空間、RAM 使用量、CPU 負載 |
| **Docker** | Docker 引擎狀態、容器狀態、影像版本 |
| **資料庫** | PostgreSQL 連線性、遷移狀態、連線池健康狀態 |
| **快取** | Redis 連線性、記憶體使用量 |
| **物件儲存** | MinIO 連線性、儲存桶可存取性 |
| **網路** | DNS 解析、埠可存取性、SSL 證書有效性 |
| **應用程式** | 服務健康狀態端點、背景工作狀態 |

每個檢查項目都會顯示通過或失敗的結果，如果出現問題，也會顯示詳細資訊。

### 自動修復模式

針對常見問題，doctor 可嘗試自動修復：

```bash
./doctor.sh --fix
```

自動修復可以解決以下問題：

- 停止的容器（重新啟動）
- 過期的資料庫連線（重新建立連線池）
- 過期的 SSL 證書（觸發更新）
- 因舊的 Docker 影像導致硬碟空間滿（清除未使用的影像）

doctor 在採取行動前，始終會說明它將修復的內容。

## 維護模式

維護模式會在您進行更改時，向訪客顯示「商店暫時不可用」頁面。您的管理面板仍然可以存取。

### 啟用維護模式

從管理面板：**商店設定 > 維護 > 啟用維護模式**

或從命令列：

```bash
docker exec spwig_shop python manage.py maintenance on
```

### 停用維護模式

從管理面板：將維護模式開關關閉。

或從命令列：

```bash
./go-live.sh
```

### 維護期間繞過存取

維護模式啟用時，您可以在 URL 中加入一個秘密參數，以正常存取商店。繞過秘密會顯示在您的 `.env` 配置文件中，位於 `MAINTENANCE_SECRET` 下。

## 管理服務

### 查看服務狀態

# 檢查所有 Spwig 服務的狀態：

```bash
docker compose ps
```

這會顯示每個服務的狀態（運行中、已停止、重新啟動中）及其健康狀態。

### 查看日誌

查看特定服務的日誌：

```bash
docker logs spwig_shop          # 應用程式日誌
docker logs spwig_celery         # 背景工作程式日誌
docker logs spwig_nginx          # 網站伺服器存取日誌
docker logs spwig_db             # 資料庫日誌
```

加入 `--tail 100` 可查看最後 100 行，或加入 `--follow` 以即時監控日誌。

### 重新啟動服務

如果需要重新啟動特定服務：

```bash
docker compose restart shop      # 重新啟動應用程式
docker compose restart celery    # 重新啟動背景工作程式
docker compose restart nginx     # 重新啟動網站伺服器
```

要重新啟動所有服務：

```bash
docker compose restart
```

## 組件更新

Spwig 提供一個組件市場，您可以在其中安裝主題、付款提供者、運送整合以及其他擴展。組件的更新與核心平台獨立進行。

導覽至 **管理 > 組件更新** 以檢查可用的組件更新。當您批准更新時，更新會自動下載並套用。

## 小技巧

- **定期升級** — 保持最新版本可確保您獲得安全性修復和新功能的使用權
- **先進行備份** — 雖然升級腳本會自動建立備份，但擁有自己的備份可提供額外的安全性
- **出現問題後執行 doctor 檢查** — 如果您的商店行為異常，`./doctor.sh` 是最快識別問題的方法
- **在低流量時間安排升級** — 維護模式會短暫中斷客戶的存取，因此請在流量較少的時間進行升級
- **保持磁碟空間可用** — 升級需要臨時空間來存放新映像和備份。請確保至少有 5 GB 的可用空間。
