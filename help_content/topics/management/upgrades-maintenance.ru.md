---
title: Обновления и техническое обслуживание
---

Spwig получает регулярные обновления с новыми функциями, улучшениями производительности и исправлениями безопасности. Это руководство описывает, как обновить вашу установку, использовать инструмент диагностики и выполнять задачи технического обслуживания.

## Обновление Spwig

### Перед обновлением

1. **Создайте резервную копию** — перейдите в **Управление > Системные метрики > Создать полную резервную копию** или запустите скрипт резервного копирования из командной строки. Это ваша защита, если что-либо пойдет не так.
2. **Проверьте текущую версию** — видна в **Управление > Системные метрики** или в нижней части панели администратора.
3. **Прочтите заметки о выпуске** — доступны в панели администратора под **Управление > Обновления компонентов**, когда обнаружена новая версия.

### Запуск обновления

SSH-доступ к вашему серверу и переход в директорию установки Spwig (обычно `/opt/spwig`):

```bash
./upgrade.sh
```

Скрипт обновления:

1. **Предварительные проверки** — проверяет наличие дискового пространства, состояние Docker и статус служб
2. **Тестирование миграций базы данных** — проверяет, будут ли изменения в базе данных применены корректно, не внося изменений
3. **Включение режима технического обслуживания** — ваш магазин отображает страницу технического обслуживания для посетителей во время обновления
4. **Создание резервной копии** — автоматическая резервная копия перед внесением изменений
5. **Очистка фоновых рабочих процессов** — ждет завершения текущих задач (отправка электронной почты, переводы) вежливо
6. **Загрузка новых образов** — загружает обновленное приложение из реестра Spwig
7. **Применение миграций базы данных** — обновляет схему базы данных для новой версии
8. **Перезапуск служб** — запускает приложение с новой версией
9. **Проверка состояния** — проверяет, что все службы работают корректно
10. **Выход из режима технического обслуживания** — ваш магазин снова онлайн

Если проверка состояния завершится неудачно после обновления, скрипт **автоматически откатывается** до предыдущей версии и восстанавливает резервную копию.

### Варианты обновления

```bash
./upgrade.sh              # Стандартное обновление с режимом технического обслуживания
./upgrade.sh --dry-run    # Проверка изменений без их применения
```

## Инструмент диагностики

Spwig включает встроенную диагностическую утилиту, которая проверяет всю вашу установку на наличие проблем:

```bash
./doctor.sh
```

Доктор проверяет:

| Категория | Что проверяет |
|----------|---------------|
| **Система** | Дисковое пространство, использование ОЗУ, нагрузка на ЦП |
| **Docker** | Состояние движка Docker, состояние контейнеров, версии образов |
| **База данных** | Подключение к PostgreSQL, статус миграции, состояние пула подключений |
| **Кэш** | Подключение к Redis, использование памяти |
| **Объектное хранилище** | Подключение к MinIO, доступность бакета |
| **Сеть** | Разрешение DNS, доступность портов, действительность сертификата SSL |
| **Приложение** | Точки проверки состояния службы, статус фоновых рабочих процессов |

Каждая проверка показывает результат прохождения/провала с деталями, если что-то не так.

### Режим автоматического исправления

Для распространенных проблем доктор может попытаться автоматически исправить их:

```bash
./doctor.sh --fix
```

Автоматическое исправление может решить:

- Остановленные контейнеры (перезапускает их)
- Устаревшие соединения с базой данных (пересоздает пул подключений)
- Истекшие сертификаты SSL (инициирует обновление)
- Полный диск из-за старых образов Docker (очищает неиспользуемые образы)

Доктор всегда объясняет, что он исправит, прежде чем предпринять действия.

## Режим технического обслуживания

Режим технического обслуживания показывает посетителям страницу "магазин временно недоступен", пока вы вносите изменения. Ваша панель администратора остается доступной.

### Включение режима технического обслуживания

С помощью панели администратора: **Настройки магазина > Техническое обслуживание > Включить режим технического обслуживания**

Или из командной строки:

```bash
docker exec spwig_shop python manage.py maintenance on
```

### Отключение режима технического обслуживания

С помощью панели администратора: переключите режим технического обслуживания в положение "выключено".

Или из командной строки:

```bash
./go-live.sh
```

### Обход доступа в режиме технического обслуживания

Пока режим технического обслуживания включен, вы можете получить доступ к магазину в обычном режиме, добавив секретный параметр в URL. Секрет обхода отображается в вашем файле конфигурации `.env` под `MAINTENANCE_SECRET`.

## Управление службами

### Просмотр статуса служб

Проверьте состояние всех служб Spwig:

```bash
docker compose ps
```

Это показывает каждую службу, её состояние (запущено, остановлено, перезапускается) и её статус здоровья.

### Просмотр логов

Проверьте логи для конкретной службы:

```bash
docker logs spwig_shop          # Логи приложения
docker logs spwig_celery         # Логи фоновых рабочих
docker logs spwig_nginx          # Логи доступа веб-сервера
docker logs spwig_db             # Логи базы данных
```

Добавьте `--tail 100`, чтобы увидеть последние 100 строк, или `--follow`, чтобы просматривать логи в реальном времени.

### Перезапуск службы

Если требуется перезапустить конкретную службу:

```bash
docker compose restart shop      # Перезапустить приложение
docker compose restart celery    # Перезапустить фоновые рабочие
docker compose restart nginx     # Перезапустить веб-сервер
```

Чтобы перезапустить все службы:

```bash
docker compose restart
```

## Обновления компонентов

Spwig имеет маркетплейс компонентов, где вы можете установить темы, платежные провайдеры, интеграции доставки и другие расширения. Компоненты обновляются независимо от ядра платформы.

Перейдите в **Управление > Обновления компонентов**, чтобы проверить доступные обновления компонентов. Обновления автоматически скачиваются и применяются, когда вы их одобряете.

## Советы

- **Регулярно обновляйтесь** — оставаясь на последней версии вы получаете исправления безопасности и доступ к новым функциям
- **Сначала делайте резервную копию** — даже если скрипт обновления создаёт автоматическую резервную копию, наличие собственной даёт дополнительную защиту
- **Запускайте doctor после проблем** — если ваш магазин ведёт себя неожиданно, `./doctor.sh` — самый быстрый способ определить проблемы
- **Планируйте обновления в часы низкого трафика** — режим обслуживания кратковременно прерывает доступ клиентов, поэтому обновляйте в тихие часы
- **Сохраняйте свободное место на диске** — обновления требуют временного места для новых образов и резервных копий. Сохраняйте как минимум 5 ГБ свободного места.
