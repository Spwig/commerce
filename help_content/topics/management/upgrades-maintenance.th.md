---
title: การอัปเกรดและการบำรุงรักษา
---

Spwig ได้รับการอัปเดตเป็นประจำด้วยคุณสมบัติใหม่ ๆ การปรับปรุงประสิทธิภาพ และการแก้ไขความปลอดภัย คู่มือนี้ครอบคลุมวิธีการอัปเกรดการติดตั้งของคุณ การใช้เครื่องมือตรวจสอบ และการจัดการงานบำรุงรักษา

## การอัปเกรด Spwig

### ก่อนที่คุณจะอัปเกรด

1. **สร้างสำเนาสำรอง** — ไปที่ **Management > System Metrics > Create Full Backup** หรือรันสคริปต์สำเนาสำรองจากบรรทัดคำสั่ง นี่คือเครือข่ายความปลอดภัยของคุณหากเกิดข้อผิดพลาดใด ๆ
2. **ตรวจสอบเวอร์ชันปัจจุบัน** — แสดงใน **Management > System Metrics** หรือที่ส่วนท้ายของแดชบอร์ดผู้ดูแลระบบ
3. **อ่านหมายเหตุการปล่อย** — สามารถเข้าถึงได้ในแผงควบคุมผู้ดูแลระบบภายใต้ **Management > Component Updates** เมื่อตรวจพบเวอร์ชันใหม่

### การรันการอัปเกรด

SSH เข้าสู่เซิร์ฟเวอร์ของคุณและนำทางไปยังไดเรกทอรีการติดตั้ง Spwig ของคุณ (โดยทั่วไปคือ `/opt/spwig`):

```bash
./upgrade.sh
```

สคริปต์อัปเกรด:

1. **การตรวจสอบก่อนบิน** — ตรวจสอบพื้นที่จัดเก็บ สถานะของ Docker และสถานะบริการ
2. **การย้ายฐานข้อมูลแบบไม่เปลี่ยนแปลง** — ทดสอบว่าการเปลี่ยนแปลงฐานข้อมูลจะถูกนำไปใช้ได้อย่างถูกต้องโดยไม่เปลี่ยนแปลงอะไรจริง ๆ
3. **เข้าสู่โหมดบำรุงรักษา** — ร้านค้าของคุณจะแสดงหน้าบำรุงรักษาให้ผู้เยี่ยมชมระหว่างการอัปเกรด
4. **สร้างสำเนาสำรอง** — สำเนาสำรองความปลอดภัยอัตโนมัติก่อนที่จะทำการเปลี่ยนแปลง
5. **ระบายงานพื้นหลัง** — รอให้งานที่กำลังดำเนินอยู่ (การส่งอีเมล การแปล) จบลงอย่างเป็นทางการ
6. **ดึงภาพใหม่** — ดาวน์โหลดแอปพลิเคชันที่อัปเดตจากคลัง Spwig
7. **นำฐานข้อมูลไปใช้** — อัปเดตโครงสร้างฐานข้อมูลของคุณสำหรับเวอร์ชันใหม่
8. **รีสตาร์ทบริการ** — เปิดใช้งานแอปพลิเคชันด้วยเวอร์ชันใหม่
9. **ตรวจสอบสุขภาพ** — ตรวจสอบว่าบริการทั้งหมดทำงานได้อย่างถูกต้อง
10. **ออกจากโหมดบำรุงรักษา** — ร้านค้าของคุณกลับมาออนไลน์อีกครั้ง

หากการตรวจสอบสุขภาพล้มเหลวหลังการอัปเกรด สคริปต์จะ **กลับไปยังเวอร์ชันก่อนหน้าอัตโนมัติ** และกู้คืนสำเนาสำรอง

### ตัวเลือกการอัปเกรด

```bash
./upgrade.sh              # การอัปเกรดมาตรฐานด้วยโหมดบำรุงรักษา
./upgrade.sh --dry-run    # ตรวจสอบสิ่งที่จะเปลี่ยนโดยไม่ได้ใช้งาน
```

## เครื่องมือตรวจสอบระบบ

Spwig มีเครื่องมือตรวจสอบระบบในตัวที่ตรวจสอบการติดตั้งทั้งหมดของคุณเพื่อหาปัญหา:

```bash
./doctor.sh
```

เครื่องมือ doctor ตรวจสอบ:

| หมวดหมู่ | สิ่งที่ตรวจสอบ |
|----------|---------------|
| **ระบบ** | พื้นที่จัดเก็บข้อมูล, การใช้หน่วยความจำ RAM, โหลด CPU |
| **Docker** | ความสมบูรณ์ของ Docker engine, สถานะของคอนเทนเนอร์, เวอร์ชันของภาพ |
| **ฐานข้อมูล** | ความเชื่อมต่อ PostgreSQL, สถานะการอัปเกรด, ความสมบูรณ์ของส่วนต่อเชื่อม |
| **แคช** | ความเชื่อมต่อ Redis, การใช้หน่วยความจำ |
| **การจัดเก็บข้อมูล** | ความเชื่อมต่อ MinIO, การเข้าถึงบั๊กเก็ต |
| **เครือข่าย** | การแก้ไข DNS, ความพร้อมใช้งานของพอร์ต, ความถูกต้องของใบรับรอง SSL |
| **แอปพลิเคชัน** | จุดสิ้นสุดของสถานะบริการ, สถานะของงานพื้นหลัง |

ผลการตรวจสอบแต่ละรายการจะแสดงผลลัพธ์ผ่าน/ไม่ผ่านพร้อมรายละเอียดหากมีปัญหา

### โหมดซ่อมอัตโนมัติ

สำหรับปัญหาทั่วไป เครื่องมือ doctor สามารถลองซ่อมอัตโนมัติได้:

```bash
./doctor.sh --fix
```

โหมดซ่อมอัตโนมัติสามารถแก้ไข:

- คอนเทนเนอร์ที่หยุดทำงาน (เริ่มต้นใหม่)
- การเชื่อมต่อฐานข้อมูลที่ล้าสมัย (รีไซเคิลส่วนต่อเชื่อม)
- ใบรับรอง SSL ที่หมดอายุ (เริ่มต้นการต่ออายุ)
- ดิสก์เต็มจากภาพ Docker ที่เก่า (ลบภาพที่ไม่ได้ใช้งาน)

เครื่องมือ doctor จะอธิบายเสมอว่ามันจะแก้ไขอะไรก่อนที่จะดำเนินการ

## โหมดบำรุงรักษา

โหมดบำรุงรักษาจะแสดงหน้า "ร้านค้าชั่วคราวไม่สามารถใช้งานได้" ขณะที่คุณทำการเปลี่ยนแปลง แผงควบคุมผู้ดูแลยังคงสามารถเข้าถึงได้

### เปิดใช้งานโหมดบำรุงรักษา

จากแผงควบคุมผู้ดูแล: **การตั้งค่าร้านค้า > การบำรุงรักษา > เปิดใช้งานโหมดบำรุงรักษา**

หรือจากบรรทัดคำสั่ง:

```bash
docker exec spwig_shop python manage.py maintenance on
```

### ปิดใช้งานโหมดบำรุงรักษา

จากแผงควบคุมผู้ดูแล: ปิดสวิตช์โหมดบำรุงรักษา

หรือจากบรรทัดคำสั่ง:

```bash
./go-live.sh
```

### เข้าถึงร้านค้าขณะอยู่ในโหมดบำรุงรักษา

ขณะที่โหมดบำรุงรักษาเปิดอยู่ คุณสามารถเข้าถึงร้านค้าได้ตามปกติโดยการเพิ่มพารามิเตอร์ลับใน URL คีย์ลับสำหรับการข้ามจะแสดงในไฟล์กำหนดค่า `.env` ของคุณภายใต้ `MAINTENANCE_SECRET`

## การจัดการบริการ

### การดูสถานะบริการ

# การตรวจสอบสถานะบริการ Spwig ทั้งหมด:

```bash
docker compose ps
```

สิ่งนี้แสดงให้เห็นแต่ละบริการ สถานะ (กำลังทำงาน หยุดทำงาน กำลังเริ่มต้นใหม่) และสถานะสุขภาพของบริการ

### การดูบันทึก

ตรวจสอบบันทึกสำหรับบริการเฉพาะ:

```bash
docker logs spwig_shop          # บันทึกแอปพลิเคชัน
docker logs spwig_celery         # บันทึกของเวิร์กเกอร์พื้นหลัง
docker logs spwig_nginx          # บันทึกการเข้าถึงเซิร์ฟเวอร์เว็บ
docker logs spwig_db             # บันทึกฐานข้อมูล
```

เพิ่ม `--tail 100` เพื่อดูบรรทัดสุดท้าย 100 บรรทัด หรือ `--follow` เพื่อดูบันทึกแบบเรียลไทม์

### การเริ่มต้นใหม่บริการ

หากต้องการเริ่มต้นใหม่บริการเฉพาะ:

```bash
docker compose restart shop      # เริ่มต้นใหม่แอปพลิเคชัน
docker compose restart celery    # เริ่มต้นใหม่เวิร์กเกอร์พื้นหลัง
docker compose restart nginx     # เริ่มต้นใหม่เซิร์ฟเวอร์เว็บ
```

เพื่อเริ่มต้นใหม่ทั้งหมดบริการ:

```bash
docker compose restart
```

## การอัปเดตส่วนประกอบ

Spwig มีตลาดส่วนประกอบที่คุณสามารถติดตั้งธีม ผู้ให้บริการการชำระเงิน การผสานการจัดส่ง และส่วนขยายอื่น ๆ ได้ ส่วนประกอบอัปเดตได้แยกจากแพลตฟอร์มหลัก

ไปที่ **Management > Component Updates** เพื่อตรวจสอบการอัปเดตส่วนประกอบที่มีอยู่ การอัปเดตจะถูกดาวน์โหลดและนำไปใช้โดยอัตโนมัติเมื่อคุณอนุมัติ

## เคล็ดลับ

- **อัปเกรดเป็นประจำ** — การอยู่ในเวอร์ชันล่าสุดช่วยให้คุณมีการแก้ไขความปลอดภัยและเข้าถึงคุณสมบัติใหม่ ๆ
- **สำรองข้อมูลก่อนเสมอ** — แม้ว่าสคริปต์อัปเกรดจะสร้างสำรองข้อมูลอัตโนมัติ แต่การสำรองข้อมูลของคุณเองจะเพิ่มความปลอดภัย
- **รัน doctor หลังจากปัญหา** — หากร้านค้าของคุณทำงานผิดปกติ `./doctor.sh` เป็นวิธีที่เร็วที่สุดในการระบุปัญหา
- **กำหนดเวลาอัปเกรดในช่วงเวลาที่มีการจราจรต่ำ** — โหมดบำรุงรักษาจะขัดขวางการเข้าถึงของลูกค้าชั่วคราว ดังนั้นจึงควรอัปเกรดในช่วงเวลาที่เงียบสงบ
- **รักษาพื้นที่ดิสก์ให้พร้อมใช้งาน** — การอัปเกรดต้องการพื้นที่ชั่วคราวสำหรับภาพใหม่และสำรองข้อมูล จัดเตรียมพื้นที่ว่างอย่างน้อย 5 GB
