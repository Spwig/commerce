---
title: 업그레이드 및 유지보수
---

Spwig은 새로운 기능, 성능 개선 및 보안 수정을 포함한 정기적인 업데이트를 받습니다. 이 가이드는 설치 업그레이드, 진단 도구 사용 및 유지보수 작업을 수행하는 방법을 다룹니다.

## Spwig 업그레이드

### 업그레이드하기 전

1. **백업 생성** — **관리 > 시스템 메트릭 > 전체 백업 생성**으로 이동하거나 명령줄에서 백업 스크립트를 실행합니다. 문제가 발생할 경우 이는 안전망 역할을 합니다.
2. **현재 버전 확인** — **관리 > 시스템 메트릭** 또는 관리 대시보드 하단에 표시됩니다.
3. **릴리스 노트 확인** — 새 버전이 감지되면 관리 패널의 **관리 > 구성 요소 업데이트**에서 확인할 수 있습니다.

### 업그레이드 실행

서버에 SSH 접속하고 Spwig 설치 디렉터리(보통 `/opt/spwig`)로 이동합니다:

```bash
./upgrade.sh
```

업그레이드 스크립트:

1. **사전 점검** — 디스크 공간, 도커 상태 및 서비스 상태를 확인합니다.
2. **데이터베이스 마이그레이션 사전 실행** — 데이터베이스 변경 사항이 실제로 변경되지 않고 깨끗하게 적용될 수 있는지 테스트합니다.
3. **유지보수 모드 진입** — 업그레이드 중 방문자에게 유지보수 페이지를 표시합니다.
4. **백업 생성** — 변경 사항을 적용하기 전에 자동으로 안전한 백업을 생성합니다.
5. **배경 작업자 정리** — 진행 중인 작업(이메일 전송, 번역 등)이 완료될 때까지 기다립니다.
6. **새 이미지 다운로드** — Spwig 레지스트리에서 업데이트된 애플리케이션을 다운로드합니다.
7. **데이터베이스 마이그레이션 적용** — 새 버전에 맞게 데이터베이스 스키마를 업데이트합니다.
8. **서비스 재시작** — 새 버전으로 애플리케이션을 시작합니다.
9. **건강 상태 확인** — 모든 서비스가 올바르게 실행되고 있는지 확인합니다.
10. **유지보수 모드 종료** — 상점이 다시 온라인 상태가 됩니다.

업그레이드 후 건강 상태 확인이 실패하면 스크립트는 **이전 버전으로 자동 롤백**하고 백업을 복원합니다.

### 업그레이드 옵션

```bash
./upgrade.sh              # 유지보수 모드를 사용한 표준 업그레이드
./upgrade.sh --dry-run    # 변경 사항을 적용하지 않고 확인
```


## 진단 도구

Spwig에는 설치 전체를 확인하는 내장 진단 도구가 포함되어 있습니다:

```bash
./doctor.sh
```

doctor는 다음을 확인합니다:

| 카테고리 | 확인 내용 |
|----------|---------------|
| **시스템** | 디스크 공간, RAM 사용량, CPU 부하 |
| **도커** | 도커 엔진 상태, 컨테이너 상태, 이미지 버전 |
| **데이터베이스** | PostgreSQL 연결, 마이그레이션 상태, 연결 풀 상태 |
| **캐시** | Redis 연결, 메모리 사용량 |
| **객체 저장소** | MinIO 연결, 버킷 접근성 |
| **네트워크** | DNS 해석, 포트 접근성, SSL 인증서 유효성 |
| **애플리케이션** | 서비스 상태 엔드포인트, 백그라운드 워커 상태 |

각 확인은 문제가 있는 경우 세부 정보와 함께 통과/실패 결과를 보여줍니다.

### 자동 복구 모드

일반적인 문제에 대해 doctor는 자동 복구를 시도할 수 있습니다:

```bash
./doctor.sh --fix
```

자동 복구는 다음과 같은 문제를 해결할 수 있습니다:

- 중지된 컨테이너(재시작)
- 오래된 데이터베이스 연결(연결 풀 재생성)
- 만료된 SSL 인증서(재발행)
- 오래된 도커 이미지로 인한 디스크 공간 부족(사용하지 않는 이미지 삭제)

doctor는 조치를 취하기 전에 무엇을 수정할지 항상 설명합니다.

## 유지보수 모드

유지보수 모드는 변경 사항을 적용하는 동안 방문자에게 "일시적으로 가게가 사용 불가합니다" 페이지를 표시합니다. 관리자 패널은 여전히 접근 가능합니다.

### 유지보수 모드 활성화

관리자 패널에서: **가게 설정 > 유지보수 > 유지보수 모드 활성화**

또는 명령줄에서:

```bash
docker exec spwig_shop python manage.py maintenance on
```

### 유지보수 모드 비활성화

관리자 패널에서: 유지보수 모드 전환 스위치를 끕니다.

또는 명령줄에서:

```bash
./go-live.sh
```

### 유지보수 중 우회 접근

유지보수 모드가 활성화된 동안 URL에 비밀 파라미터를 추가하여 일반적으로 가게에 접근할 수 있습니다. 우회 비밀은 `.env` 구성 파일의 `MAINTENANCE_SECRET` 아래에 표시됩니다.

## 서비스 관리

### 서비스 상태 확인

# 서비스 상태 확인

모든 Spwig 서비스의 상태를 확인합니다:

```bash
docker compose ps
```

이 명령은 각 서비스, 상태(실행 중, 중지됨, 재시작 중), 및 건강 상태를 표시합니다.

### 로그 확인

특정 서비스의 로그를 확인합니다:

```bash
docker logs spwig_shop          # 애플리케이션 로그
docker logs spwig_celery         # 백그라운드 워커 로그
docker logs spwig_nginx          # 웹 서버 접근 로그
docker logs spwig_db             # 데이터베이스 로그
```

`--tail 100`을 추가하여 마지막 100줄을 확인하거나, `--follow`를 추가하여 실시간으로 로그를 확인할 수 있습니다.

### 서비스 재시작

특정 서비스를 재시작해야 하는 경우:

```bash
docker compose restart shop      # 애플리케이션 재시작
docker compose restart celery    # 백그라운드 워커 재시작
docker compose restart nginx     # 웹 서버 재시작
```

모든 서비스를 재시작하려면:

```bash
docker compose restart
```

## 구성 요소 업데이트

Spwig은 구성 요소 시장 플랫폼을 제공하여 테마, 결제 제공업체, 배송 통합, 기타 확장 기능을 설치할 수 있습니다. 구성 요소는 핵심 플랫폼과 독립적으로 업데이트됩니다.

**Management > Component Updates**로 이동하여 사용 가능한 구성 요소 업데이트를 확인합니다. 승인하면 업데이트가 자동으로 다운로드 및 적용됩니다.

## 팁

- **정기적으로 업그레이드** — 최신 버전을 사용하면 보안 패치와 새로운 기능에 접근할 수 있습니다
- **항상 백업을 먼저 수행하세요** — 업그레이드 스크립트는 자동 백업을 생성하지만, 자신의 백업은 추가적인 안전성을 제공합니다
- **문제 발생 후 doctor 실행** — 매장이 예상치 못하게 작동할 경우, `./doctor.sh`는 문제를 가장 빠르게 식별할 수 있습니다
- **저조한 트래픽 시간에 업그레이드 예약** — 유지보수 모드는 고객 접근을 잠시 중단하므로, 조용한 시간대에 업그레이드하세요
- **디스크 공간 확보** — 업그레이드는 새로운 이미지와 백업을 위한 임시 공간이 필요합니다. 최소 5 GB 이상의 여유 공간을 유지하세요.
