---
title: ショップダッシュボード
---

ショップダッシュボードは、店舗のパフォーマンスに関する包括的なビューを提供します。収益、注文、人気商品、訪問者のトラフィックなど、すべてが1か所に集約されています。これにより、何が売れているのか、顧客がどこから来ているのか、そして店舗のトレンドが時間とともにどう変化しているのかを理解できます。

**管理 > システムメトリクス**にナビゲートし、ツールバーから**ショップダッシュボード**をクリックしてください。

![ショップダッシュボード概観](/static/core/admin/img/help/shop-dashboard/overview.webp)

## 時期の選択

ダッシュボードは、選択された期間に基づいてすべてのメトリクスをフィルタリングします。ページの上部にある期間セレクタを使用して、以下から選択してください：

| 期間 | 表示内容 |
|------|---------|
| 今日 | 今日と昨日の比較 |
| 今週 | 月曜日から今日と先週の比較 |
| 今月 | 今月と先月の比較 |
| 今年 | 年間累計と昨年同日の比較 |
| 過去30日 | 30日間のローリングウィンドウ |
| 過去90日 | 90日間のローリングウィンドウ |
| カスタム | 特定の開始日と終了日を入力 |

ほとんどのビューは、過去の同等期間と比較した**比較**を表示します。これにより、パフォーマンスが改善しているか、低下しているかを確認できます。現在の数値のみを表示したい場合は、**Compare**のスイッチをオフにします。

## アクションカード

ダッシュボードの上部に配置されたアクションカードは、今すぐ注意が必要な項目を強調表示します：

- **未完了の注文** — 出荷を待っている注文
- **放棄されたカート** — 顧客が商品を追加したがチェックアウトを完了しなかったセッション
- **未読メッセージ** — 回答を待っている顧客からの問い合わせ
- **在庫不足アラート** — 在庫が不足している商品

任意のアクションカードをクリックして、関連する管理セクションに直接移動できます。

## 売上パフォーマンス

売上パフォーマンスセクションは、選択された期間における主要な収益指標を表示します：

- **総収益** — 減額前の総売上
- **総注文数** — 完了した注文の数
- **平均注文価格** — 収益を注文数で割った値
- **純利益** — 収益から商品の原価と費用を差し引いた金額（設定されている場合）

# 図

各図は、比較期間からの変化を示す矢印とパーセンテージを表示しています。

## 時間ごとの売上チャート

メインのチャートは、選択した期間における売上または注文をプロットします。Spwigは最も有用なグループ化を自動的に選択します：

- 短い期間（1週間まで）は日ごとにグループ化
- 中程度の期間（3か月まで）は週ごとにグループ化
- 長い期間は月ごとにグループ化

チャートの上にある**グループ化**コントロールを使用して、グループ化を上書きできます。任意のポイントにカーソルを合わせて、その日の正確な値を確認できます。

## トップ商品

トップ商品テーブルは、選択した期間における売上高順に商品をリストします。各行は次の情報を表示します：

- **商品名**
- **販売数量**
- **生成された売上高**

これを使用して、最も優れた商品を特定し、プロモーションや在庫補充に注力すべき場所を判断してください。

## 訪問者分析

訪問者分析セクションは、あなたのストアを訪れた人数とその行動方法を示します：

- **総訪問者数** — ストアフロントへのユニークな訪問者数
- **ページビュー数** — 見られたページの合計数
- **離脱率** — 1ページのみを見た訪問者のパーセンテージ
- **時間ごとのビュー** — 選択した期間におけるトラフィック量を示すチャート

**地理**パネルは、訪問者の来源地を示し、国と（利用可能な場合）都市ごとに細分化されています。

## トラフィックソース

トラフィックソースパネルは、訪問者がストアをどのように見つけたかを示します：

| ソース | 説明 |
|--------|-------------|
| Direct | URLをタイプしたかブックマークを使用した訪問者 |
| Organic search | 検索エンジンからの訪問者 |
| Social | SNSプラットフォームからの訪問者 |
| Referral | 他のウェブサイトからリンクされた訪問者 |
| Email | メール内のリンクをクリックした訪問者 |

この情報を使用して、どのマーケティングチャネルが最も多くのトラフィックをもたらしているか、どこに投資すべきかを理解してください。

## 買い物変換漏斗

変換漏斗は、訪問者がブラウジングから購入完了に至る過程を示します：

1. **訪問者** — 総合的なユニーク訪問者数

2. **商品閲覧数** — 少なくとも1つの商品を閲覧した訪問者

3. **カートに追加** — 商品をカートに追加した訪問者

4. **チェックアウト開始** — チェックアウトプロセスを開始した訪問者

5. **注文完了** — 注文を完了した訪問者

各ステップの％はドロップオフ率を示しています。"カートに追加"から"チェックアウト開始"の間の大きなドロップは、チェックアウトフローに摩擦があることを示しています。

## チケットのパフォーマンス

チケットキャンペーンを実施している場合、このセクションは選択した期間中のチケットのパフォーマンスを示します:

- **総使用回数** — チケットが使用された回数

- **提供された総割引額** — すべてのチケット割引が適用された合計額

- **チケット付き売上高** — チケットを含む注文からの総売上高

## 顧客セグメント

顧客セグメントパネルは、顧客ベースをグループに分類します:

- **新規顧客** — 選択期間内に初めて購入した顧客

- **リピーター顧客** — 以前購入したことがある顧客

- **アカウントなしのチェックアウト** — アカウントを作成せずに行われた注文

新規顧客とリピーター顧客の比率を理解することで、マーケティング（獲得）またはロイヤルティプログラム（リテンション）にどのくらい投資するかを判断できます。

## アフィリエイトとロイヤルティのサマリ

ストアにアフィリエイトプログラムまたはロイヤルティプログラムが有効な場合、ここにサマリされたパフォーマンスメトリクスが表示されます — 総手数料、発行された総ポイント、および上位のアフィリエイターまたはポイント利用者。

## ヒント

すべてのMarkdownのフォーマット、画像のパス、コードブロック、および技術用語を保持してください。

- 月初めにダッシュボードを確認して、週次レビューを行いましょう — 「今週」の期間は、最近のパフォーマンスの明確な概要を提供します
- 特定のキャンペーンの影響を測定するには、**カスタム**の日付範囲を使用してください: キャンペーン期間の開始日と終了日を設定します
- コンバージョン・ファンネルが**チェックアウト開始**で大幅に減少している場合、チェックアウトフローを簡素化するか、信頼バッジを追加してみましょう
- 注文キャンセル数が高く、コンバージョン率が低い場合、価格や送料の問題を示している可能性があります — チェックアウトのコストを確認してください
- **今年**の期間を使用して、年間比較を行い、ビジネスの季節的なパターンを理解しましょう
- 在庫再補充の重要な決定を行う前に、上位商品のテーブルをエクスポートまたはスクリーンショットして、正しい数量を注文しているか確認してください
