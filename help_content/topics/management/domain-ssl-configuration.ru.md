---
title: Настройка домена и SSL
---

Этот руководство объясняет, как подключить пользовательский домен к вашему магазину Spwig и настроить сертификаты SSL для безопасного доступа по HTTPS. Вы можете настроить домен во время установки или добавить его позже.

## Добавление домена после установки

Если вы установили Spwig без домена (используя IP-адрес сервера), вы можете добавить его в любое время.

### Шаг 1: Настройка DNS

С вашим регистратором доменов или поставщиком DNS:

1. Создайте **A-запись**, которая направляет ваш домен (или поддомен) на IP-адрес вашего сервера
2. Если вы используете поддомен, например, `shop.example.com`, создайте A-запись для `shop`
3. Подождите распространения DNS — это обычно занимает 5–60 минут

Проверьте, работает ли DNS-запись:

```bash
dig +short shop.example.com
```

Это должно вернуть IP-адрес вашего сервера.

### Шаг 2: Запуск скрипта настройки домена

SSH-подключение к вашему серверу и переход в директорию установки Spwig:

```bash
./configure-domain.sh
```

Скрипт выполнит следующие действия:

1. Спросит имя вашего домена
2. Проверит, указывает ли DNS на ваш сервер
3. Обновит конфигурацию магазина
4. Получит бесплатный сертификат SSL от Let's Encrypt
5. Настроит веб-сервер для использования HTTPS
6. Перезапустит соответствующие службы

Ваш магазин теперь доступен по адресу `https://yourdomain.com`.

### Шаг 3: Обновление настроек магазина

После добавления домена войдите в панель администратора и перейдите в **Настройки магазина**. Убедитесь, что **URL магазина** совпадает с вашим новым доменом. Это гарантирует, что электронные письма, счета и ссылки используют правильный адрес.

## Сертификаты SSL

### Автоматический SSL (Let's Encrypt)

В **режиме standalone** установщик автоматически получает бесплатный сертификат SSL от Let's Encrypt. Эти сертификаты:

- Доверены всем основным браузерам
- Действительны в течение 90 дней
- Обновляются автоматически — проверка обновления выполняется ежедневно, и сертификаты обновляются, когда у них остается менее 30 дней
- Охватывают ваш точный домен (например, `shop.example.com`)

Вам не нужно вручную управлять обновлением.

### Самоподписанные сертификаты

В некоторых ситуациях Spwig использует самоподписанный сертификат вместо этого:

- **Режим локального** установки (разработка/тестирование)
- Когда Let's Encrypt не может достичь вашего сервера (ограничение порта 80 брандмауэром, DNS еще не распространился)
- Когда не настроен домен (доступ только по IP-адресу)

Самоподписанные сертификаты шифруют трафик, но не доверены браузерами — посетители увидят предупреждение о безопасности. Это приемлемо для тестирования, но не должно использоваться в продакшн.

### SSL в режиме sidecar

В **режиме sidecar** ваш существующий веб-сервер (Apache, Nginx, Caddy и т.д.) обрабатывает завершение SSL. Spwig работает на порту HTTP за вашим прокси-сервером. Настройте SSL на своем основном веб-сервере, как обычно.

Установщик генерирует блок конфигурации прокси, который вы можете добавить в ваш веб-сервер. Для Nginx он выглядит примерно так:

```nginx
location / {
    proxy_pass http://127.0.0.1:8080;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

## Изменение вашего домена

Чтобы перейти на другой домен:

1. Настройте DNS для нового домена (A-запись, указывающая на ваш сервер)
2. Запустите `./configure-domain.sh` снова с новым доменом
3. Скрипт обновит все конфигурации, получит новый сертификат и перезапустит службы
4. Обновите **Настройки магазина** в панели администратора с новым URL

Старый домен перестанет работать после обновления конфигурации.

## Устранение неполадок

### "DNS-валидация не удалась"

Скрипт configure-domain проверяет, указывает ли ваш домен на ваш сервер, прежде чем запрашивать сертификат. Если эта проверка не проходит:

- Проверьте правильность A-записи с помощью `dig +short yourdomain.com`
- Подождите еще несколько минут для распространения DNS
- Убедитесь, что вы настраиваете точный домен или поддомен (не дикий символ *)

### "Достигнут лимит Let's Encrypt"

Let's Encrypt ограничивает запросы сертификатов до 5 на домен в неделю. Если вы достигли этого лимита:



- Подождите 7 дней перед повторной попыткой
- Временно используйте другой поддомен
- Магазин остается доступен через HTTP или с самоподписанным сертификатом, пока вы ждете

### "Порт 80 недоступен"

Let's Encrypt должен подключиться к вашему серверу через порт 80, чтобы проверить владение доменом. Убедитесь, что:

- Ваш брандмауэр позволяет входящий TCP-трафик на порт 80
- Нет другого приложения, блокирующего порт 80
- Группа безопасности или сетевой брандмауэр вашего провайдера облака разрешает порт 80

### Ошибки обновления сертификата

Если автоматическое обновление не удается, сертификат истечет через 90 дней. Чтобы обновить вручную:

```bash
docker exec spwig_nginx certbot renew
docker exec spwig_nginx nginx -s reload
```

Проверьте журнал обновления для получения подробной информации, если это не сработает. Наиболее распространенная причина — блокировка порта 80 брандмауэром после первоначальной установки.
