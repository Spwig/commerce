---
title: अपग्रेड एवं रखरखाव
---

Spwig नए फीचर, प्रदर्शन सुधार और सुरक्षा फिक्स के साथ नियमित अपडेट प्राप्त करता है। यह पृष्ठ आपके इंस्टॉलेशन को अपग्रेड करने, डायग्नोस्टिक टूल का उपयोग करने और रखरखाव कार्यों के संचालन के बारे में बताता है।

## Spwig को अपग्रेड करें

### अपग्रेड करने से पहले

1. **बैकअप बनाएं** — **मैनेजमेंट > सिस्टम मीट्रिक्स > फुल बैकअप बनाएं** जाएं या कमांड लाइन से बैकअप स्क्रिप्ट चलाएं। यह आपका सुरक्षा जाल है अगर कुछ गलत हो जाता है।
2. **वर्तमान संस्करण जांचें** — **मैनेजमेंट > सिस्टम मीट्रिक्स** में दिखाई देता है या एडमिन डैशबोर्ड के फूटर में।
3. **रिलीज नोट्स पढ़ें** — जब एक नई संस्करण प्रदर्शित होती है तो एडमिन पैनल में **मैनेजमेंट > कॉम्पोनेंट अपडेट्स** के तहत उपलब्ध है।

### अपग्रेड को चलाएं

अपने सर्वर में SSH करें और अपने Spwig इंस्टॉलेशन डायरेक्टरी (आमतौर पर `/opt/spwig`) में जाएं:

```bash
./upgrade.sh
```

अपग्रेड स्क्रिप्ट:

1. **प्रारंभिक जांच** — डिस्क स्पेस, डॉकर के स्वास्थ्य और सेवा के स्थिति की जांच करता है
2. **सूखे चलाने के डेटाबेस माइग्रेशन** — डेटाबेस में परिवर्तन लागू करने के बिना शुद्ध रूप से लागू होंगे यह परीक्षण करता है
3. **रखरखाव मोड में प्रवेश** — अपग्रेड के दौरान आपका स्टोर आगंतुकों के लिए एक रखरखाव पृष्ठ दिखाता है
4. **एक बैकअप बनाता है** — परिवर्तन करने से पहले सुरक्षा के लिए स्वचालित बैकअप बनाता है
5. **पृष्ठभूमि कार्यकर्ताओं को खाली करता है** — चल रहे कार्यों (ईमेल भेजने, अनुवाद) को सुगम रूप से समाप्त करने के लिए इंतजार करता है
6. **नए इमेज खींचता है** — Spwig रजिस्ट्री से अपडेटेड एप्लिकेशन को डाउनलोड करता है
7. **डेटाबेस माइग्रेशन लागू करता है** — नए संस्करण के लिए आपके डेटाबेस स्कीमा को अपडेट करता है
8. **सेवाओं को पुनः शुरू करता है** — नए संस्करण के साथ एप्लिकेशन को शुरू करता है
9. **स्वास्थ्य परीक्षण** — सभी सेवाओं के सही रूप से चल रहे होने की जांच करता है
10. **रखरखाव मोड से बाहर निकलता है** — आपका स्टोर फिर से ऑनलाइन हो जाता है

अगर अपग्रेड के बाद स्वास्थ्य परीक्षण विफल हो जाता है, तो स्क्रिप्ट पूर्ववर्ती संस्करण में **स्वचालित रूप से वापस लौट जाता है** और बैकअप को पुनर्स्थापित करता है।

### अपग्रेड विकल्प

```bash
./upgrade.sh              # रखरखाव मोड के साथ मानक अपग्रेड
./upgrade.sh --dry-run    # बिना लागू करे देखें कि क्या बदलेगा
```

## डायग्नोस्टिक टूल

Spwig में एक बिल्ट-इन डायग्नोस्टिक टूल शामिल है जो आपकी पूरी इंस्टॉलेशन की जांच करता है:

```bash
./doctor.sh
```

डॉक्टर निम्न चेक करता है:

| श्रेणी | यह क्या जांचता है |
|----------|---------------|
| **सिस्टम** | डिस्क स्पेस, रैम उपयोग, सीपीयू लोड |
| **डॉकर** | डॉकर इंजन के स्वास्थ्य, कंटेनर स्थिति, इमेज संस्करण |
| **डेटाबेस** | पोस्टग्रेस कनेक्टिविटी, माइग्रेशन स्थिति, कनेक्शन पूल स्वास्थ्य |
| **कैश** | रेडिस कनेक्टिविटी, मेमोरी उपयोग |
| **ऑब्जेक्ट स्टोरेज** | मिनिओ कनेक्टिविटी, बकेट पहुंच क्षमता |
| **नेटवर्क** | डीएनएस रिज़ॉल्यूशन, पोर्ट पहुंच क्षमता, एसएसएल सर्टिफिकेट वैधता |
| **एप्लिकेशन** | सेवा स्वास्थ्य एंडपॉइंट, पृष्ठभूमि कार्यकर्ता स्थिति |

प्रत्येक जांच में एक पास/फेल परिणाम होता है और यदि कुछ गलत है तो विवरण दिखाता है।

### ऑटो-फिक्स मोड

सामान्य समस्याओं के लिए, डॉक्टर स्वचालित मरम्मत का प्रयास कर सकता है:

```bash
./doctor.sh --fix
```

स्वचालित मरम्मत निम्नलिखित को हल कर सकता है:

- बंद कंटेनर (उन्हें पुनः शुरू करता है)
- अमूल्य डेटाबेस कनेक्शन (कनेक्शन पूल को फिर से बनाता है)
- अतीत के डॉकर इमेज से पूरा हार्ड डिस्क (अप्रयुक्त इमेज को हटा देता है)

डॉक्टर हमेशा कार्रवाई करने से पहले यह बताता है कि वह क्या मरम्मत करेगा।

## रखरखाव मोड

रखरखाव मोड आप बदलाव करते समय आगंतुकों को एक "स्टोर अस्थायी रूप से अवामिल है" पृष्ठ दिखाता है। आपका प्रशासन पैनल अभी भी उपलब्ध रहता है।

### रखरखाव मोड को सक्रिय करें

प्रशासन पैनल से: **स्टोर सेटिंग्स > रखरखाव > रखरखाव मोड को सक्रिय करें**

या कमांड लाइन से:

```bash
docker exec spwig_shop python manage.py maintenance on
```

### रखरखाव मोड को अक्षम करें

प्रशासन पैनल से: रखरखाव मोड के स्विच को बंद करें।

या कमांड लाइन से:

```bash
./go-live.sh
```

### रखरखाव के दौरान पहुँच को बायपास करें

maintenances mode सक्रिय होने के दौरान, आप URL में एक गुप्त पैरामीटर जोड़कर स्टोर को सामान्य रूप से पहुँच सकते हैं।

बायपास गुप्त कुंजी आपके `.env` विन्यास फ़ाइल में `MAINTENANCE_SECRET` के तहत दिखाई देती है।

## सेवाओं का प्रबंधन

### सेवा स्थिति देखें

सभी Spwig सेवाओं की स्थिति जांचें:

```bash
docker compose ps
```

यह प्रत्येक सेवा, उसकी स्थिति (चल रहा, बंद, पुनरारंभ), और स्वास्थ्य स्थिति दिखाता है।

### लॉग देखें

एक विशिष्ट सेवा के लॉग जांचें:

```bash
docker logs spwig_shop          # एप्लिकेशन लॉग्स
docker logs spwig_celery         # पृष्ठभूमि कार्यकर्ता लॉग्स
docker logs spwig_nginx          # वेब सर्वर पहुँच लॉग्स
docker logs spwig_db             # डेटाबेस लॉग्स
```

`--tail 100` जोड़कर अंतिम 100 पंक्तियों को देखें, या `--follow` वास्तविक समय में लॉग्स की निगरानी करें।

### एक सेवा को पुनरारंभ करें

यदि एक विशिष्ट सेवा को पुनरारंभ करने की आवश्यकता है:


```bash
docker compose restart shop      # एप्लिकेशन को पुनरारंभ करें
docker compose restart celery    # पृष्ठभूमि कार्यकर्ताओं को पुनरारंभ करें
docker compose restart nginx     # वेब सर्वर को पुनरारंभ करें
```

सभी सेवाओं को पुनरारंभ करें:

```bash
docker compose restart
```

## घटक अपडेट

Spwig में एक घटक बाजार है जहां आप थीम, भुगतान प्रदाताओं, डिलीवरी समायोजन और अन्य विस्तारों को इंस्टॉल कर सकते हैं। घटक अपडेट कोर प्लेटफॉर्म से स्वतंत्र रूप से अपडेट किया जाता है।

**प्रबंधन > घटक अपडेट** जाएं उपलब्ध घटक अपडेट की जांच करने के लिए। आप उन्हें अनुमोदित करते समय अपडेट स्वचालित रूप से डाउनलोड और लागू किए जाते हैं।

## सुझाव

सभी मार्कडाउन फॉर्मेटिंग, इमेज पथ, कोड ब्लॉक और तकनीकी शब्दों को संरक्षित रखें।

- **अपग्रेड करते रहें** — नवीनतम संस्करण पर रहने से आपको सुरक्षा सुधार और नए फीचर्स के लिए एक्सेस मिलता है
- **पहले हमेशा बैकअप बनाएं** — यद्यपि अपग्रेड स्क्रिप्ट एक ऑटोमैटिक बैकअप बनाता है, तो अपना बैकअप अतिरिक्त सुरक्षा प्रदान करता है
- **समस्याओं के बाद डॉक्टर चलाएं** — यदि आपका स्टोर अप्रत्याशित रूप से काम करता है, तो `./doctor.sh` समस्याओं को पहचानने के लिए सबसे तेज़ तरीका है
- **कम ट्रैफिक वाले समय पर अपग्रेड करें** — रखरखाव मोड ग्राहक एक्सेस को अस्थायी रूप से बाधित करता है, इसलिए शांत घंटों में अपग्रेड करें
- **डिस्क स्पेस उपलब्ध रखें** — अपग्रेड के लिए नए इमेज और बैकअप के लिए अस्थायी स्थान की आवश्यकता होती है।

कम से कम 5 जीबी स्वतंत्र रखें।
