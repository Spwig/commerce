---
title: डोमेन & SSL सेटअप
---

इस गाइड में बताया गया है कि आप अपने Spwig स्टोर के लिए एक कस्टम डोमेन कैसे जोड़ सकते हैं और सुरक्षित HTTPS पहुँच के लिए SSL प्रमाणपत्र कैसे सेट कर सकते हैं। आप इसे इंस्टॉलेशन के दौरान या बाद में भी एक कर सकते हैं।

## इंस्टॉलेशन के बाद डोमेन जोड़ें

यदि आपने Spwig को डोमेन के बिना (सर्वर के IP पते के साथ) इंस्टॉल किया है, तो आप कभी भी एक डोमेन जोड़ सकते हैं।

### चरण 1: DNS के सेटअप करें

अपने डोमेन रजिस्टर या DNS प्रोवाइडर के साथ:

1. अपने डोमेन (या सब-डोमेन) को अपने सर्वर के IP पते के लिए एक **A रिकॉर्ड** बनाएं
2. यदि आप `shop.example.com` जैसे सब-डोमेन का उपयोग कर रहे हैं, तो `shop` के लिए A रिकॉर्ड बनाएं
3. DNS प्रसार के लिए प्रतीक्षा करें — यह आमतौर पर 5–60 मिनट लेता है

DNS रिकॉर्ड के काम करने की जांच करें:

```bash
dig +short shop.example.com
```

यह आपके सर्वर के IP पते को लौटाना चाहिए।

### चरण 2: डोमेन सेटअप स्क्रिप्ट चलाएं

अपने सर्वर में SSH और अपने Spwig इंस्टॉलेशन डायरेक्टरी में जाएं:

```bash
./configure-domain.sh
```

स्क्रिप्ट निम्न करेगा:

1.

अपने डोमेन के नाम के लिए पूछेगा
2.

सभी मार्कडाउन फॉर्मेटिंग, इमेज पथ, कोड ब्लॉक और तकनीकी शब्दों को सुरक्षित रखें।

DNS की जांच करें कि यह आपके सर्वर पर निर्देशित है
3.

स्टोर के विन्यास को अपडेट करें
4.

Let's Encrypt से एक मुफ्त SSL प्रमाणपत्र प्राप्त करें
5.

वेब सर्वर को HTTPS के लिए विन्यास करें
6.

संबंधित सेवाओं को पुनः शुरू करें

अब आपका स्टोर `https://yourdomain.com` पर उपलब्ध है।

### चरण 3: स्टोर सेटिंग्स अपडेट करें

डोमेन जोड़ने के बाद, अपने प्रशासन पैनल में लॉग इन करें और **स्टोर सेटिंग्स** पर जाएं। जांच करें कि **स्टोर URL** आपके नए डोमेन के साथ मेल खाता है। यह ईमेल, बिल, और लिंक के लिए सही पता का उपयोग सुनिश्चित करता है।

## SSL प्रमाणपत्र

### स्वतंत्र SSL (Let's Encrypt)

**एकल मोड** में, इंस्टॉलर ऑटोमैटिक रूप से Let's Encrypt से एक मुफ्त SSL प्रमाणपत्र प्राप्त करता है। ये प्रमाणपत्र:

- सभी प्रमुख ब्राउज़रों द्वारा विश्वसनीय होते हैं
- 90 दिनों के लिए वैध होते हैं
- स्वचालित रूप से नवीनीकरण होता है - एक नवीनीकरण जांच प्रतिदिन चलती है, और प्रमाणपत्र तब नवीनीकृत किए जाते हैं जब वे 30 दिन से कम अवधि के लिए वैध होते हैं
- आपके ठीक डोमेन को कवर करते हैं (जैसे `shop.example.com`)

आपको नवीनीकरण को मैन्युअल रूप से प्रबंधित करने की आवश्यकता नहीं होती।

### स्व-हस्ताक्षरित प्रमाणपत्र


कुछ स्थितियों में, Spwig एक स्व-हस्त-हस्ताक्षरित प्रमाणपत्र का उपयोग करता है:

- **लोकल मोड** स्थापना (विकास/परीक्षण)
- जब Let's Encrypt आपके सर्वर तक पहुँच नहीं सकता (पोर्ट 80 के ब्लॉकिंग फ़ायरवॉल, DNS अभी तक प्रसारित नहीं हुआ)
- जब कोई डोमेन निर्मित नहीं है (केवल IP पहुँच)

स्व-हस्ताक्षरित प्रमाणपत्र ट्रैफिक को एन्क्रिप्ट करता है लेकिन ब्राउज़र द्वारा विश्वास नहीं किया जाता है — यात्री एक सुरक्षा चेतावनी देखेंगे। यह परीक्षण के लिए स्वीकार्य है लेकिन उत्पादन में इसका उपयोग नहीं किया जाना चाहिए।

### साइडकार मोड SSL

**साइडकार मोड** में, आपके वर्तमान वेब सर्वर (Apache, Nginx, Caddy, आदि) SSL समाप्ति का प्रबंधन करता है। Spwig अपने प्रॉक्सी के पीछे एक HTTP पोर्ट पर चलता है। अपने मुख्य वेब सर्वर पर SSL को आपके आम तरीके से निर्मित करें।

इंस्टॉलर एक प्रॉक्सी निर्माण ब्लॉक उत्पन्न करता है जिसे आप अपने वेब सर्वर में जोड़ सकते हैं। Nginx के लिए, यह इस प्रकार दिखता है:


```nginx
location / {
    proxy_pass http://127.0.0.1:8080;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

## अपने डोमेन को बदलें

अपने डोमेन को बदलने के लिए:

1. नए डोमेन के लिए DNS सेटअप करें (ए रिकॉर्ड आपके सर्वर के लिए पॉइंट करता है)
2. नए डोमेन के साथ `./configure-domain.sh` फिर से चलाएं
3. स्क्रिप्ट सभी संरचना को अपडेट करता है, एक नई प्रमाणपत्र प्राप्त करता है और सेवाओं को पुनः शुरू करता है
4. प्रशासन पैनल में **स्टोर सेटिंग्स** को अपडेट करें और नए URL के साथ अपडेट करें

आपके पुराने डोमेन एक बार डोमेन संरचना अपडेट कर दिए जाने के बाद काम नहीं करेगा।

## ट्राउबलशूटिंग

### "DNS वैधता विफल"

कॉन्फ़िगर-डोमेन स्क्रिप्ट एक प्रमाणपत्र के लिए अनुरोध करने से पहले यह जांचता है कि आपका डोमेन आपके सर्वर के लिए पॉइंट करता है। यदि यह जांच विफल हो जाती है:


- `dig +short yourdomain.com` के साथ A रिकॉर्ड की पुष्टि करें
- DNS प्रसार के लिए कुछ और मिनट इंतजार करें
- यह चेक करें कि आप ठीक डोमेन या सबडोमेन (वाइल्डकार्ड नहीं) को निर्माण कर रहे हैं

### "Let's Encrypt दर लिमिट पहुँच गई"

Let's Encrypt प्रति डोमेन प्रति सप्ताह 5 सर्टिफिकेट अनुरोध के लिए सीमा निर्धारित करता है। यदि आप इस सीमा पर पहुँच जाते हैं:

- पुनः प्रयास करने से पहले 7 दिन इंतजार करें
- इस बीच एक अलग सबडोमेन का उपयोग करें
- इंतजार के दौरान, दुकान HTTP के माध्यम से या स्व-हस्ताक्षरित सर्टिफिकेट के साथ उपलब्ध रहती है

### "पोर्ट 80 पहुँच नहीं हो सकता"

Let's Encrypt को डोमेन के मालिकता की पुष्टि के लिए अपने सर्वर पर पोर्ट 80 पर कनेक्ट करना आवश्यक है। निश्चित करें:

- आपका फ़ायरवॉल पोर्ट 80 पर आने वाले TCP के लिए अनुमति देता है
- कोई अन्य एप्लिकेशन पोर्ट 80 को ब्लॉक नहीं कर रहा है
- आपके क्लाउड प्रोवाइडर के सुरक्षा समूह या नेटवर्क फ़ायरवॉल पोर्ट 80 की अनुमति देता है

### सर्टिफिकेट नवीनीकरण विफलता

यदि स्वचालित नवीनीकरण विफल हो जाता है, तो 90 दिनों के बाद सर्टिफिकेट समाप्त हो जाएगा। मनमाने तरीके से नवीनीकरण करने के लिए:

```bash
docker exec spwig_nginx certbot renew
docker exec spwig_nginx nginx -s reload
```

यदि यह विफल हो जाता है, तो नवीनीकरण लॉग में विवरण देखें।

सबसे आम कारण शुरुआती इंस्टॉलेशन के बाद फायरवॉल में पोर्ट 80 के ब्लॉक होना हो सकता है।
