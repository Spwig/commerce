---
title: アップグレードとメンテナンス
---

Spwigは定期的に新機能、パフォーマンスの改善、セキュリティの修正を含むアップデートを受けます。このガイドでは、インストールのアップグレード方法、診断ツールの使用方法、メンテナンスタスクの処理方法について説明します。

## Spwigのアップグレード

### アップグレードする前に

1. **バックアップを作成** — **管理 > システムメトリクス > フルバックアップの作成**にアクセスするか、コマンドラインからバックアップスクリプトを実行してください。何か問題が起きた場合の安全網になります。
2. **現在のバージョンを確認** — **管理 > システムメトリクス**または管理ダッシュボードのフッターに表示されます。
3. **リリースノートを読む** — 新しいバージョンが検出された場合、管理パネルの**管理 > コンポーネントアップデート**で利用可能です。

### アップグレードの実行

サーバーにSSH接続し、Spwigのインストールディレクトリ（通常は`/opt/spwig`）に移動します：

```bash
./upgrade.sh
```

アップグレードスクリプト:

1. **事前チェック** — ディスク容量、Dockerの状態、サービスの状態を確認します。
2. **データベースマイグレーションのドライラン** — 実際に変更せずにデータベースの変更がクリーンに適用されることをテストします。
3. **メンテナンスモードに入ります** — アップグレード中、店舗は訪問者にメンテナンスページを表示します。
4. **バックアップを作成** — 変更を行う前の自動的な安全バックアップを作成します。
5. **バックグラウンドワーカーをドレイン** — 進行中のタスク（メール送信、翻訳）がスムーズに完了するのを待ちます。
6. **新しいイメージをプル** — Spwigレジストリから更新されたアプリケーションをダウンロードします。
7. **データベースマイグレーションを適用** — 新しいバージョン用にデータベーススキーマを更新します。
8. **サービスを再起動** — 新しいバージョンでアプリケーションを起動します。
9. **健康チェック** — すべてのサービスが正常に動作していることを確認します。
10. **メンテナンスモードを終了** — 店舗はオンラインに戻ります。

アップグレード後に健康チェックが失敗した場合、スクリプトは**自動的に前のバージョンにロールバック**し、バックアップを復元します。

### アップグレードオプション

```bash
./upgrade.sh              # メンテナンスモード付きの標準アップグレード
./upgrade.sh --dry-run    # 変更内容を確認するだけ、適用はしない
```


## ディアグノスティックツール

Spwigには、インストール全体をチェックする組み込みの診断ツールが含まれています：

```bash
./doctor.sh
```

doctorは以下をチェックします：

| カテゴリ | チェック内容 |
|----------|---------------|
| **システム** | ディスク容量、RAM使用量、CPU負荷 |
| **Docker** | Dockerエンジンの状態、コンテナの状態、イメージバージョン |
| **データベース** | PostgreSQLの接続性、マイグレーション状態、接続プールの状態 |
| **キャッシュ** | Redisの接続性、メモリ使用量 |
| **オブジェクトストレージ** | MinIOの接続性、バケットのアクセス可能性 |
| **ネットワーク** | DNS解決、ポートの可アクセス性、SSL証明書の有効性 |
| **アプリケーション** | サービスの健康状態エンドポイント、バックグラウンドワーカーの状態 |

各チェックでは、問題がある場合の詳細とともにパス/失敗の結果を表示します。

### 自動修復モード

一般的な問題に対して、doctorは自動的な修復を試みることができます：

```bash
./doctor.sh --fix
```

自動修復は以下を解決できます：

- 停止しているコンテナ（再起動）
- 古くなったデータベース接続（接続プールを再生成）
- 期限切れのSSL証明書（更新をトリガー）
- 古いDockerイメージによるディスクのフル（使用されていないイメージを削除）

doctorは常にアクションを実行する前に、何を修復するかを説明します。

## 保守モード

保守モードでは、変更を加える間、訪問者に「一時的に店舗が利用不可です」ページを表示します。管理パネルは引き続きアクセス可能です。

### 保守モードの有効化

管理パネルから：**ストア設定 > 保守 > 保守モードを有効にする**

またはコマンドラインから：

```bash
docker exec spwig_shop python manage.py maintenance on
```

### 保守モードの無効化

管理パネルから：保守モードのスイッチをオフにします。

またはコマンドラインから：

```bash
./go-live.sh
```

### 保守モード中のアクセスバイパス

保守モードが有効になっている間、URLにシークレットパラメータを追加することで、通常通り店舗にアクセスできます。バイパスシークレットは、`.env`設定ファイルの`MAINTENANCE_SECRET`の下に表示されます。

## サービスの管理

### サービスの状態の確認

# スピグのサービスの状態を確認する

すべてのスピグサービスの状態を確認します：

```bash
docker compose ps
```

これは、各サービスの状態（実行中、停止中、再起動中）と健康状態を表示します。

### ログの確認

特定のサービスのログを確認します：

```bash
docker logs spwig_shop          # アプリケーションログ
docker logs spwig_celery         # バックグラウンドワーカーログ
docker logs spwig_nginx          # Webサーバーのアクセスログ
docker logs spwig_db             # データベースログ
```

`--tail 100`を追加して最後の100行を表示するか、`--follow`を追加してリアルタイムでログを監視できます。

### サービスの再起動

特定のサービスを再起動する必要がある場合：

```bash
docker compose restart shop      # アプリケーションの再起動
docker compose restart celery    # バックグラウンドワーカーの再起動
docker compose restart nginx     # Webサーバーの再起動
```

すべてのサービスを再起動するには：

```bash
docker compose restart
```

## コンポーネントの更新

スピグには、テーマ、支払いプロバイダー、配送統合、その他の拡張機能をインストールできるコンポーネントマーケットプレイスが用意されています。コンポーネントはコアプラットフォームとは独立して更新されます。

**管理 > コンポーネントの更新**に移動して、利用可能なコンポーネント更新を確認してください。更新は、承認したときに自動的にダウンロードおよび適用されます。

## ヒント

- **定期的にアップグレードする** — 最新バージョンに保つことで、セキュリティ修正と新しい機能へのアクセスが保証されます
- **常にバックアップを取る** — アップグレードスクリプトは自動バックアップを作成しますが、自分のバックアップがあると追加の安全性が得られます
- **問題が発生した後はdoctorを実行する** — ストアの挙動が予期しない場合、`./doctor.sh`は問題を特定する最も速い方法です
- **低トラフィック時間にアップグレードをスケジュールする** — メンテナンスモードは一時的に顧客のアクセスを中断するため、忙しくない時間にアップグレードしてください
- **ディスクスペースを確保する** — アップグレードには新しいイメージとバックアップ用の一時スペースが必要です。少なくとも5 GBの空き容量を維持してください。
