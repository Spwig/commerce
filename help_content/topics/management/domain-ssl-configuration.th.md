---
title: การตั้งค่าโดเมนและ SSL
---

คู่มือนี้อธิบายวิธีการเชื่อมต่อโดเมนเฉพาะของคุณกับร้านค้า Spwig และตั้งค่าใบรับรอง SSL เพื่อเข้าถึง HTTPS อย่างปลอดภัย คุณสามารถตั้งค่าโดเมนขณะติดตั้งหรือเพิ่มภายหลังได้

## การเพิ่มโดเมนหลังการติดตั้ง

หากคุณติดตั้ง Spwig โดยไม่มีโดเมน (โดยใช้ที่อยู่ IP ของเซิร์ฟเวอร์) คุณสามารถเพิ่มได้ทุกเมื่อ

### ขั้นตอนที่ 1: ตั้งค่า DNS

ด้วยผู้จัดการโดเมนหรือผู้ให้บริการ DNS ของคุณ:

1. สร้าง **A record** ที่ชี้โดเมน (หรือซับโดเมน) ของคุณไปยังที่อยู่ IP ของเซิร์ฟเวอร์
2. หากใช้ซับโดเมนเช่น `shop.example.com` ให้สร้าง A record สำหรับ `shop`
3. รอให้ DNS แพร่กระจาย — ปกติจะใช้เวลา 5–60 นาที

ตรวจสอบว่าบันทึก DNS ทำงานได้:

```bash
dig +short shop.example.com
```

นี่ควรจะคืนค่าที่อยู่ IP ของเซิร์ฟเวอร์ของคุณ

### ขั้นตอนที่ 2: รันสคริปต์การตั้งค่าโดเมน

เข้าสู่ระบบผ่าน SSH ไปยังเซิร์ฟเวอร์ของคุณและไปยังไดเรกทอรีการติดตั้ง Spwig ของคุณ:

```bash
./configure-domain.sh
```

สคริปต์จะ:

1. ถามชื่อโดเมนของคุณ
2. ตรวจสอบว่า DNS ชี้ไปยังเซิร์ฟเวอร์ของคุณ
3. อัปเดตการตั้งค่าร้านค้า
4. รับใบรับรอง SSL ฟรีจาก Let's Encrypt
5. ตั้งค่าเซิร์ฟเวอร์เว็บให้ใช้ HTTPS
6. เริ่มต้นบริการที่เกี่ยวข้องใหม่

ร้านค้าของคุณสามารถเข้าถึงได้ที่ `https://yourdomain.com` ตอนนี้

### ขั้นตอนที่ 3: อัปเดตการตั้งค่าร้านค้า

หลังจากเพิ่มโดเมน ให้เข้าสู่แดชบอร์ดผู้ดูแลระบบของคุณและไปที่ **Store Settings** ตรวจสอบให้แน่ใจว่า **Store URL** ตรงกับโดเมนใหม่ของคุณ นี่จะทำให้แน่ใจว่าอีเมล ใบแจ้งหนี้ และลิงก์ใช้ที่อยู่ที่ถูกต้อง

## ใบรับรอง SSL

### SSL อัตโนมัติ (Let's Encrypt)

ในโหมด **standalone** ตัวติดตั้งจะได้รับใบรับรอง SSL ฟรีจาก Let's Encrypt อัตโนมัติ ใบรับรองเหล่านี้:

- ได้รับการรับรองจากเบราว์เซอร์หลักทั้งหมด
- มีอายุ 90 วัน
- อัปเดตอัตโนมัติ — การตรวจสอบการอัปเดตจะดำเนินการทุกวัน และใบรับรองจะถูกอัปเดตเมื่อเหลือเวลาอีกน้อยกว่า 30 วัน
- ครอบคลุมโดเมนของคุณอย่างแม่นยำ (เช่น `shop.example.com`)

คุณไม่จำเป็นต้องจัดการการอัปเดตด้วยตนเอง

### ใบรับรองที่เซ็นด้วยตนเอง


ในบางสถานการณ์ Spwig จะใช้ใบรับรองที่เซลฟ์-เซ็น (self-signed certificate) แทน:

- **โหมดท้องถิ่น** (การติดตั้งสำหรับการพัฒนา/ทดสอบ)
- เมื่อ Let's Encrypt ไม่สามารถเข้าถึงเซิร์ฟเวอร์ของคุณได้ (การปิดกั้นพอร์ต 80 โดยไฟร์วอลล์, DNS ยังไม่ได้เผยแพร่)
- เมื่อไม่มีโดเมนที่กำหนด (การเข้าถึงผ่าน IP เท่านั้น)

ใบรับรองที่เซลฟ์-เซ็นจะเข้ารหัสข้อมูลการสื่อสาร แต่ไม่ได้รับการรับรองจากเบราว์เซอร์ — ผู้เยี่ยมชมจะเห็นข้อเตือนด้านความปลอดภัย นี่เป็นเรื่องที่ยอมรับได้สำหรับการทดสอบ แต่ไม่ควรใช้ในสภาพแวดล้อมการผลิต

### SSL ในโหมด Sidecar

ใน **โหมด Sidecar** เซิร์ฟเวอร์เว็บของคุณที่มีอยู่แล้ว (เช่น Apache, Nginx, Caddy, ฯลฯ) จะจัดการการสิ้นสุด SSL (SSL termination) Spwig จะทำงานบนพอร์ต HTTP ที่อยู่ด้านหลังของ proxy ของคุณ ตั้งค่า SSL บนเซิร์ฟเวอร์เว็บหลักของคุณตามปกติที่คุณเคยทำ

ตัวติดตั้งจะสร้างบล็อกการตั้งค่า proxy ที่คุณสามารถเพิ่มเข้าไปในเซิร์ฟเวอร์เว็บของคุณ สำหรับ Nginx จะดูคล้ายกับ:

```nginx
location / {
    proxy_pass http://127.0.0.1:8080;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

## การเปลี่ยนโดเมนของคุณ

เพื่อเปลี่ยนไปใช้โดเมนอื่น:

1. ตั้งค่า DNS สำหรับโดเมนใหม่ (บันทึก A record ที่ชี้ไปยังเซิร์ฟเวอร์ของคุณ)
2. รัน `./configure-domain.sh` อีกครั้งพร้อมโดเมนใหม่
3. สคริปต์จะอัปเดตการตั้งค่าทั้งหมด ขอใบรับรองใหม่ และเริ่มต้นบริการใหม่
4. อัปเดต **การตั้งค่าร้านค้า** ในแดชบอร์ดผู้ดูแลระบบด้วย URL ใหม่

โดเมนเก่าของคุณจะหยุดทำงานเมื่อการตั้งค่าได้รับการอัปเดตแล้ว

## การแก้ไขปัญหา

### "DNS validation failed"

สคริปต์ configure-domain จะตรวจสอบว่าโดเมนของคุณชี้ไปยังเซิร์ฟเวอร์ของคุณก่อนที่จะขอใบรับรอง หากการตรวจสอบล้มเหลว:

- ตรวจสอบว่า A record ถูกต้องโดยใช้ `dig +short yourdomain.com`
- รออีกสักพักเพื่อให้ DNS ได้เผยแพร่
- ตรวจสอบว่าคุณกำลังตั้งค่าโดเมนหรือซับโดเมนที่ถูกต้อง (ไม่ใช่ wildcards)

### "Let's Encrypt rate limit reached"

Let's Encrypt จำกัดการขอใบรับรองเป็น 5 ครั้งต่อโดเมนต่อสัปดาห์ หากคุณใช้ถึงขีดจำกัดนี้:


- รอ 7 วันก่อนที่จะลองอีกครั้ง
- ใช้ subdomain ที่แตกต่างกันในช่วงระหว่างนี้
- ร้านค้ายังสามารถเข้าถึงได้ผ่าน HTTP หรือใช้ใบรับรองที่เซ็นชื่อเองในขณะที่คุณรออยู่

### "พอร์ต 80 ไม่สามารถเข้าถึงได้"

Let's Encrypt ต้องเชื่อมต่อกับเซิร์ฟเวอร์ของคุณผ่านพอร์ต 80 เพื่อยืนยันการเป็นเจ้าของโดเมน โปรดตรวจสอบ:

- ไฟร์วอลล์ของคุณอนุญาตให้มีการเข้ามาผ่าน TCP บนพอร์ต 80
- ไม่มีแอปพลิเคชันอื่นที่บล็อกพอร์ต 80
- กลุ่มความปลอดภัยหรือไฟร์วอลล์เครือข่ายของผู้ให้บริการคลาวด์ของคุณอนุญาตให้ใช้พอร์ต 80

### การต่ออายุใบรับรองล้มเหลว

หากการต่ออายุอัตโนมัติล้มเหลว ใบรับรองจะหมดอายุหลังจาก 90 วัน เพื่อต่ออายุแบบมือถือ:

```bash
docker exec spwig_nginx certbot renew
docker exec spwig_nginx nginx -s reload
```

ตรวจสอบบันทึกการต่ออายุเพื่อดูรายละเอียดหากการดำเนินการนี้ล้มเหลว สาเหตุที่พบบ่อยที่สุดคือพอร์ต 80 ถูกบล็อกโดยการเปลี่ยนแปลงไฟร์วอลล์หลังจากการติดตั้งเริ่มต้น
