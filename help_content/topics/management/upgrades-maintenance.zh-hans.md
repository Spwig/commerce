---
title: 升级与维护
---

Spwig 会定期接收包含新功能、性能改进和安全修复的更新。本指南介绍了如何升级您的安装、使用诊断工具以及处理维护任务。

## 升级 Spwig

### 升级前

1. **创建备份** — 转到 **管理 > 系统指标 > 创建完整备份** 或从命令行运行备份脚本。如果出现问题，这是您的安全网。
2. **检查当前版本** — 可在 **管理 > 系统指标** 或管理员仪表板页脚中查看。
3. **阅读发布说明** — 当检测到新版本时，可在管理员面板的 **管理 > 组件更新** 下找到。

### 运行升级

通过 SSH 登录到您的服务器，并导航到 Spwig 安装目录（通常为 `/opt/spwig`）：

```bash
./upgrade.sh
```

升级脚本：

1. **预检** — 检查磁盘空间、Docker 健康状况和服务状态
2. **数据库迁移预演** — 测试数据库更改是否可以干净地应用，而不会实际更改任何内容
3. **进入维护模式** — 在升级期间，您的商店会向访客显示维护页面
4. **创建备份** — 在进行更改之前自动创建安全备份
5. **清空后台工作线程** — 等待正在进行的任务（如发送电子邮件、翻译）优雅完成
6. **拉取新镜像** — 从 Spwig 注册表下载更新后的应用程序
7. **应用数据库迁移** — 更新数据库模式以适应新版本
8. **重启服务** — 使用新版本启动应用程序
9. **健康检查** — 验证所有服务是否正常运行
10. **退出维护模式** — 您的商店重新上线

如果升级后健康检查失败，脚本会 **自动回滚** 到上一个版本并恢复备份。

### 升级选项

```bash
./upgrade.sh              # 带维护模式的标准升级
./upgrade.sh --dry-run    # 检查将发生哪些更改，但不实际应用
```

## 诊断工具

Spwig 内置了一个诊断工具，用于检查整个安装是否存在任何问题：

```bash
./doctor.sh
```

诊断工具会检查以下内容：

| 类别 | 检查内容 |
|----------|---------------|
| **系统** | 磁盘空间、RAM 使用情况、CPU 负载 |
| **Docker** | Docker 引擎健康状况、容器状态、镜像版本 |
| **数据库** | PostgreSQL 连接性、迁移状态、连接池健康状况 |
| **缓存** | Redis 连接性、内存使用情况 |
| **对象存储** | MinIO 连接性、存储桶可访问性 |
| **网络** | DNS 解析、端口可访问性、SSL 证书有效性 |
| **应用** | 服务健康检查端点、后台工作进程状态 |

每个检查都会显示通过/失败的结果，并在出现问题时提供详细信息。

### 自动修复模式

对于常见问题，诊断工具可以尝试自动修复：

```bash
./doctor.sh --fix
```

自动修复可以解决以下问题：

- 停止的容器（重新启动它们）
- 过时的数据库连接（重新生成连接池）
- 过期的 SSL 证书（触发续订）
- 由于旧的 Docker 镜像导致磁盘满（清理未使用的镜像）

诊断工具在采取行动前总会解释它将修复的内容。

## 维护模式

在维护模式下，访客会看到一个“商店暂时不可用”的页面，而您可以在该模式下进行更改。您的管理员面板仍然可以访问。

### 启用维护模式

从管理员面板：**商店设置 > 维护 > 启用维护模式**

或者从命令行：

```bash
docker exec spwig_shop python manage.py maintenance on
```

### 禁用维护模式

从管理员面板：关闭维护模式开关。

或者从命令行：

```bash
./go-live.sh
```

### 维护期间绕过访问

在维护模式激活期间，您可以通过在 URL 中添加一个秘密参数来正常访问商店。绕过密钥显示在您的 `.env` 配置文件中的 `MAINTENANCE_SECRET` 下。

## 管理服务

### 查看服务状态

# 服务状态检查

查看所有 Spwig 服务的状态：

```bash
docker compose ps
```

这会显示每个服务的状态（运行中、已停止、正在重启）及其健康状态。

### 查看日志

查看特定服务的日志：

```bash
docker logs spwig_shop          # 应用日志
docker logs spwig_celery         # 后台工作程序日志
docker logs spwig_nginx          # Web 服务器访问日志
docker logs spwig_db             # 数据库日志
```

添加 `--tail 100` 可查看最后 100 行，或添加 `--follow` 以实时查看日志。

### 重启服务

如果需要重启特定服务：

```bash
docker compose restart shop      # 重启应用
docker compose restart celery    # 重启后台工作程序
docker compose restart nginx     # 重启 Web 服务器
```

要重启所有服务：

```bash
docker compose restart
```

## 组件更新

Spwig 提供了一个组件市场，您可以在其中安装主题、支付提供商、配送集成和其他扩展。组件的更新与核心平台独立进行。

导航到 **管理 > 组件更新** 以检查可用的组件更新。批准后，更新将自动下载并应用。

## 建议

- **定期升级** —— 保持最新版本可确保您获得安全补丁并使用新功能
- **始终先备份** —— 尽管升级脚本会创建自动备份，但拥有自己的备份可提供额外的安全保障
- **出现问题后运行 doctor** —— 如果您的商店行为异常，`./doctor.sh` 是快速识别问题的最快方法
- **在低流量时段安排升级** —— 维护模式会短暂中断客户访问，因此请在低流量时段升级
- **保持磁盘空间可用** —— 升级需要临时空间来存储新镜像和备份。请保持至少 5 GB 的可用空间。
