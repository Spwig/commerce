---
title: 顧客ウォレット
---

顧客ウォレットは、顧客が今後の注文で使用できる残高を持つストアクレジットシステムです。ストアクレジットは、返金、紹介報酬、プロモーションキャンペーン、またはあなたのチームが行った手動の調整の結果として追加されます。顧客はチェックアウト時にウォレットの残高を適用して、支払う金額を減らすことができます。

**顧客 > 顧客ウォレット**にナビゲートして、ウォレットを表示および管理してください。

## ウォレット残高の理解

各顧客ウォレットは4つの残高数値を表示します：

| 残高 | 説明 |
|---|---|
| **利用可能残高** | 顧客が現在チェックアウト時に使用できる金額 |
| **保留残高** | まだ使用できないクレジット — 例えば、確認期間内にある返金 |
| **累計クレジット** | このウォレットにこれまでにクレジットされた総額、すべての過去のクレジットを含む |
| **累計使用額** | 顧客がすべての注文からウォレットから使用した総額 |

チェックアウト時に重要なのは利用可能残高のみです。保留クレジットは保留期間が終了すると利用可能になります。

## 顧客のウォレットの表示

1. **顧客 > 顧客ウォレット**にナビゲートします
2. 検索フィールドを使用して、名前またはメールアドレスで顧客を検索します
3. ウォレットエントリをクリックして詳細ビューを開きます

詳細ビューは上部に現在の残高を表示し、下部に完全な取引履歴を表示します。**最後にクレジットされた時刻**と**最後に使用された時刻**のタイムスタンプは、ウォレットが最後にアクティブだった時間を示します。

### ウォレットリストのフィルタリング

**アクティブ**フィルタを使用して、アクティブなウォレットと凍結されたウォレットを分離します。アクティブでないとマークされたウォレットは、ポジティブな残高がある場合でもチェックアウトで使用できません。

## 取引履歴の読み取り

ウォレット残高のすべての変更は、個別の取引として記録されます。取引履歴は完全で永久な台帳であり、取引は編集または削除されることはありません。エラーを修正する必要がある場合は、代わりに新しい補正取引が追加されます。

各取引は次の情報を表示します：

| フィールド | 説明 |
|---|---|
| **タイプ** | クレジット、デビット、リファンド、調整、または逆転 |
| **金額** | この取引の金額（常に正の数で表示されます） |
| **取引後の残高** | この取引が適用された後のウォレット残高 |
| **ソース** | クレジットまたはデビットの発生元 |
| **ステータス** | 完了、保留、または逆転 |
| **説明** | 取引の簡単な説明 |
| **参照ID** | 元のレコードへのリンク（例：注文番号または報酬ID） |
| **作成日時** | 取引が記録された日時 |

### 取引タイプの説明

- **クレジット** — ウォレットに資金が追加されたもの（リファンド、プロモーション、または手動調整から）
- **デビット** — チェックアウト時に使用された資金
- **リファンド** — 返品またはキャンセルされた注文の結果としてウォレットに追加されたクレジット
- **調整** — あなたのチームによって行われた手動の修正
- **逆転** — 以前の取引をキャンセルする取引

### 取引ソースの説明

- **注文リファンド** — 注文がウォレットに戻されたときに発行されたクレジット
- **リファラル報酬** — リファラルプログラムを通じて獲得されたクレジット
- **プロモーション** — マーケティングキャンペーンの一環として与えられたクレジット
- **手動調整** — スタッフメンバーによって直接追加または削除されたクレジット
- **注文支払い** — 注文を支払うためにチェックアウト時に使用された資金

## 手動ウォレット調整

ウォレット詳細ビューから直接資金を追加または削除することはできません — ウォレット取引は、関連プロセス（リファンド、報酬、プロモーション）を通じて作成されます。ただし、適切な権限を持つスタッフは、**ウォレット取引**セクションを通じて手動調整取引を作成できます。

**顧客 > ウォレット取引**に移動し、**+ ウォレット取引の追加**を使用して、他のソースに該当しないクレジットを適用する必要がある場合（例：サービスに関する苦情後の善意クレジット）。

手動調整を作成する際には、以下の手順に従ってください：

1.


調整している**Wallet**を選びます（顧客のメールアドレスで検索）
2.

**Transaction Type**を`Adjustment`に設定します
3.

**Source**を`Manual Adjustment`に設定します
4.

**Amount**を入力します — 方向に関係なく常に正の数値を使用してください
5.

**Status**を`Completed`に設定して、即時クレジットを実行します
6.

**Description**に、理由を明確に記入してください — これはトランザクション履歴で表示されます
7.

**Save**をクリックします

> **注意:** Walletトランザクションは変更不可なので、保存する前に金額とWalletを再度確認してください。間違いがある場合、修正するために逆トランザクションを作成する必要があります。

## Walletの凍結

顧客がWallet残高を使用できないようにする必要がある場合 — 例えば、詐欺調査中の場合 — Walletを削除したり残高を削除したりすることなく、無効化できます。

1. 顧客のWallet詳細ビューを開きます
2. **Active**のトグルを外します
3. **Save**をクリックします

残高は保持され、いつでもWalletを再アクティブ化できます。無効な間、顧客はチェックアウト時にWallet残高を使用できません。

## すべてのトランザクションの閲覧

ストア全体のWallet活動のビューが必要な場合は、**Customers > Wallet Transactions**に移動してください。このリストはすべての顧客Walletのトランザクションを表示し、以下のフィルターが利用可能です:

- **Transaction Type** — クレジット、デビット、調整など、タイプでフィルター
- **Source** — トランザクションの発生元でフィルター
- **Status** — 完了、保留、逆転など、ステータスでフィルター
- **Date** — 上部の日付階層を使用して特定の日、月、年を詳しく表示

トランザクションリストは読み取り専用です — このビューからトランザクションは編集または削除できません。

## Tips

すべてのMarkdownの書式、画像パス、コードブロック、技術用語を保持してください。

- **Lifetime Credited** と **Lifetime Used** を確認して、顧客がストアクレジットをどのくらい積極的に使用しているかを把握してください — 大きな未使用残高は、顧客がその存在を忘れている可能性を示しています
- 顧客が残高が間違っていると報告した場合、残高が時間とともにどのように変化したかを正確に追跡するために、完全な取引履歴を確認してください — 各エントリの **Balance After** 列により、これは簡単になります
- ウォレットクレジットを顧客のリテンションツールとして活用してください — 困難な注文経験後に示す善意のクレジットは、返金よりもコストが低く、顧客があなたのストアで継続的に購入を続けるようにすることが可能です
- 冻結されたウォレットは、永久に残高を保持します — 一時的にウォレットを無効にした場合、問題が解決したときに再アクティベートすることを忘れないでください
- 各取引の **Reference ID** は、元のレコードにリンクしており、クレジットまたはデビットが適用された理由を他の場所を検索することなく簡単に確認できます
