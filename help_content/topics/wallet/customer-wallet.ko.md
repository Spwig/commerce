---
title: 고객 지갑
---

고객 지갑은 고객이 미래 주문에 사용할 수 있는 잔액을 제공하는 가상의 신용 시스템입니다. 매장 신용은 환불, 추천 보상, 프로모션 캠페인, 또는 귀하의 팀이 수동으로 조정한 경우에 추가될 수 있습니다. 고객은 결제 시 지갑 잔액을 적용하여 지불 금액을 줄일 수 있습니다.

**고객 > 고객 지갑**으로 이동하여 지갑을 확인하고 관리할 수 있습니다.

## 지갑 잔액 이해

각 고객 지갑은 네 가지 잔액 수치를 보여줍니다:

| 잔액 | 설명 |
|---|---|
| **사용 가능한 잔액** | 고객이 현재 결제 시 사용할 수 있는 금액 |
| **대기 잔액** | 아직 사용할 수 없는 신용 — 예를 들어, 확인 기간이 아직 남은 환불 |
| **총 신용 금액** | 이 지갑에 지금까지 추가된 총 금액, 과거 모든 신용 포함 |
| **총 사용 금액** | 고객이 모든 주문에서 지갑에서 사용한 총 금액 |

결제 시에는 **사용 가능한 잔액**이 유일하게 중요한 수치입니다. 대기 신용은 대기 기간이 끝나면 사용 가능하게 됩니다.

## 고객의 지갑 보기

1. **고객 > 고객 지갑**으로 이동합니다.
2. 이름 또는 이메일로 고객을 검색할 수 있는 검색 필드를 사용합니다.
3. 지갑 항목을 클릭하여 상세 보기로 이동합니다.

상세 보기에서는 상단에 현재 잔액이 표시되고, 아래에는 전체 거래 내역이 표시됩니다. **최근 신용 시간** 및 **최근 사용 시간** 타임스탬프는 지갑이 마지막으로 활성화된 시점을 알려줍니다.

### 지갑 목록 필터링

**활성** 필터를 사용하여 활성 지갑과 동결된 지갑을 구분할 수 있습니다. 비활성화된 지갑은 긍정적인 잔액이 있더라도 결제 시 사용할 수 없습니다.

## 거래 내역 읽기

지갑 잔액의 모든 변경은 개별 거래로 기록됩니다. 거래 내역은 완전하고 영구적인 장부이며, 거래는 결코 편집되거나 삭제되지 않습니다. 오류를 수정해야 하는 경우, 대신 새로운 보상 거래가 추가됩니다.

각 거래는 다음과 같은 정보를 보여줍니다:

| 필드 | 설명 |
|---|---|
| **타입** | 신용, 차감, 환불, 조정, 또는 취소 |
| **금액** | 이 거래의 금액 (항상 양수로 표시됨) |
| **거래 후 잔액** | 이 거래가 적용된 후 지갑 잔액 |
| **출처** | 신용 또는 차감이 발생한 위치 |
| **상태** | 완료, 대기, 또는 취소 |
| **설명** | 거래에 대한 간단한 설명 |
| **참조 ID** | 원본 기록에 대한 링크 (예: 주문 번호 또는 보상 ID) |
| **생성 시간** | 거래가 기록된 시간 |

### 거래 유형 설명

- **신용** — 지갑에 추가된 자금 (환불, 프로모션, 또는 수동 조정으로 인해)
- **차감** — 체크아웃 시 사용된 자금
- **환불** — 반품 또는 주문 취소로 인해 추가된 신용
- **조정** — 팀원이 수동으로 수행한 수정
- **취소** — 이전 항목을 무효화하는 거래

### 거래 출처 설명

- **주문 환불** — 주문이 지갑으로 환불될 때 제공되는 신용
- **추천 보상** — 추천 프로그램을 통해 얻은 신용
- **프로모션** — 마케팅 캠페인의 일부로 제공된 신용
- **수동 조정** — 직원이 직접 추가하거나 제거한 신용
- **주문 결제** — 주문을 결제하기 위해 체크아웃 시 사용된 자금

## 수동 지갑 조정

지갑 세부 정보 뷰에서 직접 자금을 추가하거나 제거할 수 없습니다 — 지갑 거래는 관련된 프로세스(환불, 보상, 프로모션)를 통해 생성됩니다. 그러나 적절한 권한을 가진 직원은 **지갑 거래** 섹션을 통해 수동 조정 거래를 생성할 수 있습니다.

**고객 > 지갑 거래**로 이동하고, 다른 출처에 해당하지 않는 신용을 적용해야 하는 경우 **+ 지갑 거래 추가**를 사용하세요 — 예를 들어, 서비스 불만 후 제공되는 친절한 신용입니다.

수동 조정을 생성할 때:

1.

조정 중인 **Wallet**을 선택하세요 (고객 이메일로 검색)
2.

**Transaction Type**을 `Adjustment`로 설정하세요
3.

**Source**을 `Manual Adjustment`로 설정하세요
4.

**Amount**을 입력하세요 — 방향과 관계없이 항상 양수를 입력하세요
5.

**Status**을 `Completed`로 설정하여 즉시 크레딧을 적용하세요
6.

**Description**을 추가하세요 — 이는 거래 내역에서 보입니다. 원인을 명확하게 설명하세요
7.

**Save**를 클릭하세요

> **참고:** Wallet 거래는 수정할 수 없으므로, 저장하기 전에 금액과 Wallet을 다시 확인하세요. 실수한 경우, 이를 수정하려면 반전 거래를 생성해야 합니다.

## Wallet 동결

고객이 Wallet 잔액을 사용하지 못하게 하려는 경우 — 예를 들어, 사기 조사 중 — Wallet을 삭제하거나 잔액을 제거하지 않고 비활성화할 수 있습니다.

1. 고객의 Wallet 상세 보기로 이동하세요
2. **Active** 토글을 해제하세요
3. **Save**를 클릭하세요

잔액은 보존되고, 언제든지 Wallet을 다시 활성화할 수 있습니다. 비활성 상태일 때 고객은 체크아웃 시 Wallet 잔액을 사용할 수 없습니다.

## 모든 거래 보기

모든 고객의 Wallet 활동을 전체적으로 보려면 **Customers > Wallet Transactions**으로 이동하세요. 이 목록은 모든 고객의 Wallet에 있는 모든 거래를 보여주며, 다음 필터 옵션이 있습니다:

- **Transaction Type** — 크레딧, 디빗, 조정 등으로 필터
- **Source** — 거래가 발생한 위치로 필터
- **Status** — 완료, 대기, 반전으로 필터
- **Date** — 상단의 날짜 계층 구조를 사용하여 특정 날짜, 월, 또는 연도로 세부 정보를 확인하세요

거래 목록은 읽기 전용입니다 — 이 뷰에서 거래를 편집하거나 삭제할 수 없습니다.

## 팁

모든 마크다운 형식, 이미지 경로, 코드 블록 및 기술 용어를 보존하세요.

- **생애 주기 적립액**과 **생애 주기 사용액**을 확인하여 고객이 얼마나 적극적으로 스토어 크레딧을 사용하는지 파악할 수 있습니다. 사용되지 않은 금액이 많다면 고객이 이를 잊어버렸을 가능성이 있습니다.
- 고객이 잔액이 잘못 표시된다고 보고할 경우, 전체 거래 내역을 확인하여 시간에 따라 잔액이 어떻게 변경되었는지 정확히 추적하세요. 각 항목의 **거래 후 잔액** 열은 이 작업을 쉽게 만들어 줍니다.
- 월렛 크레딧을 고객 유지 도구로 활용하세요. 어려운 주문 경험 후에 고객에게 친절한 크레딧을 제공하면 환불보다 비용이 적게 들며 고객이 계속해서 스토어에서 소비하도록 유도할 수 있습니다.
- 동결된 월렛은 잔액이 영구적으로 유지되며 만료 기간이 없습니다. 월렛을 일시적으로 비활성화한 경우, 문제가 해결되면 다시 활성화를 잊지 마세요.
- 각 거래의 **참조 ID**는 원본 기록으로 돌아가며, 이는 다른 곳을 검색할 필요 없이 크레딧 또는 차감이 왜 적용되었는지 쉽게 확인할 수 있도록 해줍니다.
