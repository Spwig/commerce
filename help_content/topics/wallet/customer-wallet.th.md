---
title: กระเป๋าเงินลูกค้า
---

กระเป๋าเงินลูกค้าเป็นระบบเครดิตร้านค้าที่ให้ลูกค้ามียอดเงินคงเหลือที่สามารถใช้จ่ายในคำสั่งซื้อในอนาคตได้ เครดิตร้านค้าสามารถเพิ่มได้จากการคืนเงิน รางวัลจากการแนะนำ แคมเปญโปรโมชัน หรือการปรับเปลี่ยนด้วยตนเองที่ทีมของคุณทำ ลูกค้าสามารถใช้ยอดเงินในกระเป๋าเงินเพื่อลดจำนวนเงินที่ต้องจ่ายได้ที่หน้าชำระเงิน

ไปที่ **ลูกค้า > กระเป๋าเงินลูกค้า** เพื่อดูและจัดการกระเป๋าเงิน

## การเข้าใจยอดคงเหลือกระเป๋าเงิน

กระเป๋าเงินแต่ละใบแสดงยอดคงเหลือ 4 รายการ:

| ยอดคงเหลือ | คำอธิบาย |
|---|---|
| **ยอดคงเหลือที่ใช้ได้** | จำนวนเงินที่ลูกค้าสามารถใช้จ่ายได้ทันทีที่หน้าชำระเงิน |
| **ยอดคงเหลือที่รอการใช้งาน** | เงินเครดิตที่ยังไม่สามารถใช้ได้ เช่น การคืนเงินที่ยังอยู่ในช่วงยืนยัน |
| **ยอดเครดิตตลอดชีวิต** | จำนวนเงินทั้งหมดที่เคยถูกเครดิตเข้ากระเป๋าเงินนี้ รวมถึงเครดิตทั้งหมดในอดีต |
| **ยอดใช้ตลอดชีวิต** | จำนวนเงินทั้งหมดที่ลูกค้าใช้จากกระเป๋าเงินนี้ในทุกคำสั่งซื้อ |

ยอดคงเหลือที่ใช้ได้เป็นตัวเลขเดียวที่สำคัญที่สุดที่หน้าชำระเงิน ยอดเครดิตที่รอการใช้งานจะกลายเป็นยอดที่ใช้ได้เมื่อช่วงเวลาที่รอผ่านไปแล้ว

## การดูกระเป๋าเงินของลูกค้า

1. ไปที่ **ลูกค้า > กระเป๋าเงินลูกค้า**
2. ใช้ช่องค้นหาเพื่อค้นหาลูกค้าตามชื่อหรืออีเมล
3. คลิกที่รายการกระเป๋าเงินเพื่อเปิดดูรายละเอียด

หน้ารายละเอียดแสดงยอดคงเหลือปัจจุบันด้านบนและประวัติการทำธุรกรรมทั้งหมดด้านล่าง ตัวบ่งชี้ **Last Credited At** และ **Last Used At** จะบอกคุณว่ากระเป๋าเงินถูกใช้งานครั้งสุดท้ายเมื่อใด

### การกรองรายการกระเป๋าเงิน

ใช้ตัวกรอง **Active** เพื่อแยกกระเป๋าเงินที่ใช้งานได้จากกระเป๋าเงินที่ถูกแช่แข็ง กระเป๋าเงินที่ถูกทำเครื่องหมายว่าไม่ใช้งานไม่สามารถใช้ได้ที่หน้าชำระเงินแม้ว่าจะมียอดคงเหลือเป็นบวก

## การอ่านประวัติการทำธุรกรรม

ทุกการเปลี่ยนแปลงในยอดคงเหลือกระเป๋าเงินจะถูกบันทึกเป็นธุรกรรมแต่ละรายการ ประวัติการทำธุรกรรมเป็นบัญชีที่สมบูรณ์และถาวร — ธุรกรรมจะไม่มีการแก้ไขหรือลบ หากต้องการแก้ไขข้อผิดพลาด จะเพิ่มธุรกรรมชดเชยใหม่แทน

แต่ละธุรกรรมแสดง:

| Field | Description |
|---|---|
| **Type** | Credit, Debit, Refund, Adjustment, หรือ Reversal |
| **Amount** | ค่าของธุรกรรมนี้ (จะแสดงเป็นจำนวนบวกเสมอ) |
| **Balance After** | ยอดเงินในกระเป๋าหลังจากธุรกรรมนี้ถูกประมวลผล |
| **Source** | ที่มาของเครดิตหรือเดบิต |
| **Status** | Completed, Pending, หรือ Reversed |
| **Description** | คำอธิบายสั้น ๆ เกี่ยวกับธุรกรรม |
| **Reference ID** | ลิงก์ไปยังบันทึกต้นทาง (เช่น หมายเลขคำสั่งซื้อหรือ ID ของรางวัล) |
| **Created At** | เมื่อธุรกรรมถูกบันทึก |

### การอธิบายประเภทของธุรกรรม

- **Credit** — เงินที่ถูกเพิ่มเข้าไปในกระเป๋า (จากคืนเงิน โปรโมชัน หรือการปรับปรุงด้วยมือ)
- **Debit** — เงินที่ถูกใช้ในการชำระเงินที่เคาน์เตอร์
- **Refund** — เงินที่ถูกเพิ่มเข้ามาโดยเฉพาะเนื่องจากคำสั่งซื้อที่ถูกคืนหรือยกเลิก
- **Adjustment** — การแก้ไขด้วยมือที่ทำโดยทีมของคุณ
- **Reversal** — ธุรกรรมที่ยกเลิกการบันทึกก่อนหน้า

### การอธิบายแหล่งที่มาของธุรกรรม

- **Order Refund** — เงินที่ได้รับเมื่อคืนเงินเข้ากระเป๋าหลังจากคำสั่งซื้อ
- **Referral Reward** — เงินที่ได้รับผ่านโปรแกรมแนะนำ
- **Promotion** — เงินที่ได้รับเป็นส่วนหนึ่งของแคมเปญการตลาด
- **Manual Adjustment** — เงินที่ถูกเพิ่มหรือลบโดยตรงโดยเจ้าหน้าที่
- **Order Payment** — เงินที่ใช้ในการชำระเงินสำหรับคำสั่งซื้อที่เคาน์เตอร์

## การปรับปรุงกระเป๋าเงินแบบมือถือ

คุณไม่สามารถเพิ่มหรือลบเงินโดยตรงจากหน้ารายละเอียดกระเป๋าเงิน — ธุรกรรมกระเป๋าเงินถูกสร้างขึ้นผ่านกระบวนการที่เกี่ยวข้อง (การคืนเงิน รางวัล โปรโมชัน) อย่างไรก็ตาม เจ้าหน้าที่ที่มีสิทธิ์เหมาะสมสามารถสร้างธุรกรรมการปรับปรุงแบบมือถือผ่านส่วน **Wallet Transactions**

ไปที่ **Customers > Wallet Transactions** และใช้ **+ Add Wallet Transaction** หากคุณต้องการเพิ่มเครดิตที่ไม่ตรงกับแหล่งอื่น — เช่น เงินคืนด้วยความกรุณาหลังจากร้องเรียนเกี่ยวกับบริการ

เมื่อสร้างการปรับปรุงแบบมือถือ:

1.

เลือก **Wallet** ที่คุณกำลังปรับ (ค้นหาด้วยอีเมลลูกค้า)
2.

ตั้งค่า **Transaction Type** เป็น `Adjustment`
3.

ตั้งค่า **Source** เป็น `Manual Adjustment`
4.

กรอก **Amount** — เสมอเป็นจำนวนบวกไม่ว่าจะเป็นการเพิ่มหรือลด
5.

ตั้งค่า **Status** เป็น `Completed` เพื่อให้เครดิตทันที
6.

เพิ่ม **Description** ที่ชัดเจนอธิบายเหตุผล — สิ่งนี้จะปรากฏในประวัติการทำธุรกรรม\ n7.

คลิก **Save**

> **หมายเหตุ:** เนื่องจากธุรกรรม Wallet ไม่สามารถแก้ไขได้ โปรดตรวจสอบจำนวนและ Wallet ให้แน่ใจก่อนบันทึก หากคุณทำผิดพลาด คุณจะต้องสร้างธุรกรรมการคืนเงินเพื่อแก้ไขมัน

## การแช่แข็ง Wallet

หากคุณต้องการป้องกันไม่ให้ลูกค้าใช้ยอดเงินใน Wallet — เช่น ในระหว่างการสืบสวนการฉ้อโกง — คุณสามารถปิดใช้งานมันโดยไม่ต้องลบหรือลบยอดเงิน

1. เปิดหน้ารายละเอียด Wallet ของลูกค้า
2. คลิกปุ่ม **Active** เพื่อปิดการใช้งาน
3. คลิก **Save**

ยอดเงินจะถูกเก็บไว้และ Wallet สามารถเปิดใช้งานได้ทุกเมื่อ ขณะที่ไม่ได้ใช้งาน ลูกค้าจะไม่สามารถใช้ยอดเงินใน Wallet ในการชำระเงินได้

## การดูธุรกรรมทั้งหมด

สำหรับมุมมองแบบองค์กรเกี่ยวกับกิจกรรม Wallet ให้ไปที่ **Customers > Wallet Transactions** รายการนี้แสดงธุรกรรมทั้งหมดใน Wallet ของลูกค้าทั้งหมด โดยมีตัวกรองดังนี้:

- **Transaction Type** — กรองตามเครดิต เดบิต การปรับปรุง ฯลฯ
- **Source** — กรองตามแหล่งที่ธุรกรรมเริ่มต้น
- **Status** — กรองตามสถานะที่เสร็จสิ้น รอ หรือคืนเงิน
- **Date** — ใช้ลำดับชั้นของวันที่ด้านบนเพื่อดูข้อมูลเฉพาะวัน เดือน หรือปี

รายการธุรกรรมเป็นแบบอ่านเฉพาะ — ธุรกรรมไม่สามารถแก้ไขหรือลบได้จากมุมมองนี้

## เคล็ดลับ

รักษาการจัดรูปแบบ Markdown ทั้งหมด ค่าเส้นทางรูปภาพ บล็อกโค้ด และคำศัพท์ทางเทคนิคไว้

- ตรวจสอบ **Lifetime Credited** และ **Lifetime Used** เพื่อทำความเข้าใจว่าลูกค้าใช้เครดิตร้านค้าของตนอย่างกระตือรือร้นเพียงใด — ยอดคงเหลือที่ไม่ได้ใช้จำนวนมากอาจบ่งบอกว่าลูกค้าลืมว่ามีอยู่
- หากลูกค้ารายงานว่ายอดคงเหลือดูผิดปกติ ให้ตรวจสอบประวัติการทำธุรกรรมทั้งหมดเพื่อติดตามว่ายอดคงเหลือเปลี่ยนแปลงอย่างไรตามเวลาที่ผ่านมา คอลัมน์ **Balance After** ที่ปรากฏในแต่ละรายการทำให้สิ่งนี้เป็นเรื่องง่าย
- ใช้เครดิตกระเป๋าเงินเป็นเครื่องมือรักษาลูกค้า — เงินคืนเพื่อความเป็นมิตรหลังประสบการณ์การสั่งซื้อที่ยากลำบากอาจมีค่าใช้จ่ายน้อยกว่าการคืนเงิน ขณะที่ยังคงให้ลูกค้าใช้จ่ายในร้านค้าของคุณ
- กระเป๋าเงินที่ถูกแช่แข็งจะรักษายอดคงเหลือไว้ตลอดไป ไม่มีการหมดอายุ — หากคุณปิดใช้งานกระเป๋าเงินชั่วคราว จำไว้ว่าต้องเปิดใช้งานใหม่เมื่อปัญหาได้รับการแก้ไขแล้ว
- **Reference ID** ที่ปรากฏในแต่ละการทำธุรกรรมจะเชื่อมโยงกลับไปยังบันทึกต้นทาง ทำให้สามารถตรวจสอบได้อย่างตรงไปตรงมาว่าเหตุใดเครดิตหรือการหักเงินจึงถูกนำไปใช้โดยไม่จำเป็นต้องค้นหาที่อื่น
