---
title: 配送業者
---

配送業者は、リアルタイムの送料計算、ラベル生成、パッケージの追跡のために、ストアを配送業者のAPIに接続します。Spwigは世界中の主要な配送業者をサポートしており、API統合ができない配送業者に対しては、手動で送料表を設定することもできます。

![配送業者](/static/core/admin/img/help/shipping-providers/provider-list.webp)

## 利用可能な配送業者

| 配送業者 | 地域 | 主な機能 |
|---------|---------|-------------|
| **FedEx** | グローバル | リアルタイムの送料、ラベル印刷、追跡、複数パッケージ |
| **UPS** | グローバル | リアルタイムの送料、ラベル印刷、追跡、住所検証 |
| **USPS** | アメリカ合衆国 | 国内および国際送料、追跡 |
| **NinjaVan** | 東南アジア | 最後のマイル配送、現金払いサポート |
| **Canada Post** | カナダ | 国内および国際、荷物および手紙の送料 |
| **Australia Post** | オーストラリア | 国内および国際、荷物およびエクスプレス |

## 配送業者の接続

**設定 > 配送業者**に移動し、**Providerを接続**をクリックしてセットアップウィザードを起動します。

### ステップ1: 配送業者の選択

利用可能な配送業者から選択してください。各カードには、該当する配送業者のサポート地域と機能が表示されます。

### ステップ2: 設定手順

配送業者固有の設定ガイドを確認してください:
- 配送業者で開発者/ビジネスアカウントを作成する方法
- API資格情報をどこで見つけるか
- 必要なアカウント設定（例: 輸送者番号、メーター番号）

### ステップ3: 資格情報の入力

配送業者のアカウント用のAPI資格情報を入力します。必要な項目は配送業者によって異なります:

- **APIキー / シークレット** — 認証資格情報
- **アカウント番号** — 配送業者のアカウントまたは輸送者番号
- **メーター番号** — 一部の配送業者（例: FedEx）で必須
- **サンドボックスモード** — 本番環境に移行する前に、配送業者のサンドボックスAPIでテストするには有効にしてください

### ステップ4: 接続のテスト

**接続をテスト**をクリックして資格情報を確認します。ウィザードは以下を確認します:
- API認証が成功している
- アカウントの権限が有効
- 料金照会が予期された結果を返す

### ステップ5: 設定と保存

設定を確定してください:
- **有効** — 輸送業者を有効または無効にします
- **表示名** — チェックアウト時に顧客に表示される名前
- **出荷元住所** — 輸送料の計算に使用される倉庫または履行住所

## 出荷地域

出荷地域は、輸送料の計算用の地理的エリアを定義します。**設定 > 出荷地域**に移動して管理してください。

### 地域の作成

1. **+ 地域の追加**をクリックします
2. 地域に名前を付けます（例: "国内", "ヨーロッパ", "アジア太平洋"）
3. 地域のカバレッジを定義するために、以下の1つまたは複数を使用します:
   - **国** — 特定の国を選択
   - **州/県** — 国内での特定の地域に限定
   - **郵便番号パターン** — パターンを使用して郵便/ZIPコードを一致させます（例: "90*" はロサンゼルス地域を指します）
4. **優先度**を設定します — 地域が重複している場合、最も高い優先度の地域が使用されます

### 地域の一致

顧客がチェックアウト時に配送先住所を入力すると、システムは以下のように処理します:
1. 郵便番号パターンを最初にチェック（最も具体的）
2. 次に州/県の一致
3. 次に国の一致
4. 最も高い優先度の一致した地域を使用

## 出荷プロモーション

出荷プロモーションは、条件付きの変更を出荷料金に適用します。**設定 > 出荷プロモーション**に移動して構成してください。

### プロモーションタイプ

| プロモーションタイプ | 説明 |
|-----------|-------------|
| **割引%** | 輸送料をパーセンテージで減らす |
| **固定割引** | 輸送料を固定金額で減らす |
| **コストの上書き** | 輸送料を特定の金額に上書き |
| **無料配送** | 輸送料をゼロに設定 |
| **上乗せ%** | レートにパーセンテージの上乗せを追加 |
| **固定上乗せ** | レートに固定金額の上乗せを追加 |

### 条件

各プロモーションは、満たすべき1つ以上の条件を持つことができます:

| 条件 | 例 |
|-----------|---------|
| **カートの金額** | 100ドル以上の注文で無料配送 |
| **総重量** | 30kg以上の注文に手数料を適用 |
| **商品数** | 5つ以上の商品がある注文に割引 |
| **配送地域** | 国内配送のみにプロモーションを適用 |
| **配送方法** | 特定の配送方法に適用 |
| **商品** | 特定の商品に特別な料金 |
| **顧客グループ** | VIP顧客は無料配送 |
| **日付範囲** | 休日前の配送プロモーション |

### プロモーションの優先順位

- プロモーションは優先順位の順番で評価されます（数字が小さい順）
- **他のプロモーションを停止** — 有効にすると、このプロモーションが一致した場合、他のプロモーションはチェックされません
- 複数のプロモーションが重複して適用可能（例：10％の割引プロモーションと無料配送の金額条件プロモーション）

## レートテーブル

レートテーブルは、注文属性に基づいて段階的な価格を提供します。**設定 > 配送レートテーブル**に移動して構成してください。

### テーブルの種類

以下に基づいてレートの段階を作成します：
- **重量** — 総注文重量に基づく価格段階（例：0〜1kg = 5ドル、1〜5kg = 10ドル）
- **注文金額** — カートのサブトータルに基づく価格段階
- **数量** — 商品数に基づく価格段階

### レートテーブルの作成

1. **+ レートテーブルを追加**をクリック
2. テーブルに名前をつけて、段階タイプを選択
3. 最小/最大範囲と価格を含む段階を追加
4. レートテーブルを配送地域に割り当て

カスタムの価格構造を定義し、キャリアのAPIレートを使用しない場合に、レートテーブルは非常に役立ちます。

## 配送パッケージ

正確なレート計算のために標準的な梱包サイズを定義します。**設定 > 配送パッケージ**に移動してください。

各パッケージタイプで以下を設定します：
- **名前** — 説明（例："小箱", "大サイズフラットレート"）
- **寸法** — 長さ、幅、高さ
- **最大重量** — パッケージが保持できる最大重量
- **デフォルト** — 特定の梱包が割り当てられていない場合にこのパッケージを使用

キャリアはパッケージの寸法を基に寸法重量を計算し、これにより配送料金に影響を与えることがあります。

## 手動の配送業者（配送業者のプリセット）

API連携ができない配送業者の場合、手動で配送業者のプリセットを作成します。

1. **設定 > 配送業者のプリセット** にアクセスします
2. **+ プリセットの追加** をクリックします
3. 次の設定を行います：
   - **配送業者の名前** — チェックアウト画面に表示される表示名
   - **追跡URLテンプレート** — `{tracking_number}` というプレースホルダーを含むURLパターン（例：`https://track.carrier.com/?id={tracking_number}`）
   - **推定到着日** — 顧客に表示する到着日範囲
4. 価格設定用の料金表と組み合わせます

手動の配送業者は、ライブAPI連携なしで追跡リンクと到着日の見積もりを提供します。

## 複数倉庫の配送

複数の倉庫をお持ちの場合、配送は異なる出発地から計算されます：

- **国別倉庫** — 特定の国に倉庫を割り当てて、配送距離を短縮します
- **代替チェーン** — 主倉庫が在庫切れのときにどの倉庫が配送するかを定義します
- **商品ごとの割り当て** — 一部の商品は特定の倉庫からしか配送されません

システムは、顧客の所在地と商品の在庫状況に基づいて最適な倉庫を自動的に選択します。

## ヒント

- 可能であれば、**ライブレート**用の配送業者のAPIを接続してください — フラットレート表よりも正確で、重量、寸法、目的地に応じて調整されます。
- **"Rest of World"** という配送地域を、特定の地域にカバーされていない国すべてをカバーするためのキャッチオールとして作成してください。
- **無料配送** のプロモーションタイプを、カートの金額条件付きで使用して、販売促進に利用してください（例："75ドル以上の注文で無料配送"）。
- 本番運用前に、さまざまな住所とカート内容で配送料金の計算をテストしてください。
- **配送業者のプリセット** を、API連携ができないローカルの配送業者に追跡URLテンプレートを設定しておきましょう — 顧客は依然として追跡リンクを取得できます。
- **配送パッケージ** を使用して、FedExやUPSなどの配送業者から正確な寸法重量料金を取得してください。
