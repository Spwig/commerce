---
title: 배송 제공업체
---

배송 제공업체는 매장과 운송사 API를 연결하여 실시간 배송 요금, 라벨 생성 및 패키지 추적 기능을 제공합니다. Spwig은 전 세계 주요 운송사를 지원하며, API 통합이 없는 운송사에 대해 수동 요금표를 설정할 수도 있습니다.

![배송 제공업체](/static/core/admin/img/help/shipping-providers/provider-list.webp)

## 사용 가능한 운송사

| 운송사 | 지역 | 주요 기능 |
|---------|---------|-------------|
| **FedEx** | 글로벌 | 실시간 요금, 라벨 인쇄, 추적, 다중 패키지 |
| **UPS** | 글로벌 | 실시간 요금, 라벨 인쇄, 추적, 주소 검증 |
| **USPS** | 미국 | 국내 및 국제 요금, 추적 |
| **NinjaVan** | 동남아시아 | 최종 배송, 현금 대금 지불 지원 |
| **Canada Post** | 캐나다 | 국내 및 국제, 소포 및 편지 요금 |
| **Australia Post** | 호주 | 국내 및 국제, 소포 및 익스프레스 |

## 운송사 연결

**설정 > 배송 제공업체**로 이동하여 **제공업체 연결**을 클릭하여 설정 마법사를 시작합니다.

### 단계 1: 제공업체 선택

사용 가능한 배송 운송사 중에서 선택합니다. 각 카드는 운송사가 지원하는 지역과 기능을 보여줍니다.

### 단계 2: 설정 지침

운송사별 설정 가이드를 확인하세요:
- 운송사와 함께 개발자/사업 계정을 생성하는 방법
- API 인증 정보를 어디에서 찾는지
- 필요한 계정 설정(예: 운송사 번호, 미터 번호)

### 단계 3: 인증 정보 입력

운송사 계정의 API 인증 정보를 입력합니다. 필요한 필드는 운송사에 따라 달라집니다:

- **API 키 / 비밀번호** — 인증 정보
- **계정 번호** — 운송사 계정 또는 운송사 번호
- **미터 번호** — 일부 운송사(예: FedEx)에서 필요함
- **샌드박스 모드** — 실제 배송 전 운송사의 샌드박스 API로 테스트

### 단계 4: 연결 테스트

**연결 테스트**를 클릭하여 인증 정보를 확인합니다. 마법사는 다음을 확인합니다:
- API 인증이 성공했는지
- 계정 권한이 유효한지
- 요금 조회가 예상 결과를 반환하는지

### 단계 5: 설정 및 저장

설정을 완료합니다:
- **활성화** — 운송사를 활성화하거나 비활성화함
- **표시 이름** — 체크아웃 시 고객에게 표시되는 이름
- **출발지 주소** — 요금 계산을 위한 창고 또는 충족 주소

## 배송 지역

배송 지역은 요금 계산을 위한 지리적 영역을 정의합니다. **설정 > 배송 지역**으로 이동하여 관리할 수 있습니다.

### 지역 생성

1. **+ 지역 추가**를 클릭합니다
2. 지역에 이름을 지정합니다(예: "국내", "유럽", "아시아 태평양")
3. 다음 중 하나 이상으로 지역의 범위를 정의합니다:
   - **국가** — 특정 국가를 선택합니다
   - **주/도** — 국가 내 특정 지역으로 좁힙니다
   - **우편번호 패턴** — 패턴을 사용하여 우편/ZIP 코드를 일치시킵니다(예: "90*"은 로스앤젤레스 지역을 참조)
4. **우선순위**를 설정합니다 — 지역이 겹칠 경우 가장 높은 우선순위 지역이 사용됩니다

### 지역 일치

고객이 체크아웃 시 배송 주소를 입력하면 시스템은 다음과 같이 작동합니다:
1. 우편번호 패턴을 먼저 확인합니다(가장 구체적)
2. 그 다음 주/도 일치를 확인합니다
3. 그 다음 국가 일치를 확인합니다
4. 가장 높은 우선순위로 일치하는 지역을 사용합니다

## 배송 규칙

배송 규칙은 배송 요금에 조건부 수정을 적용합니다. **설정 > 배송 규칙**으로 이동하여 설정합니다.

### 규칙 유형

| 규칙 유형 | 설명 |
|-----------|-------------|
| **할인 %** | 배송 요금을 퍼센트로 줄입니다 |
| **고정 할인** | 배송 요금을 고정 금액으로 줄입니다 |
| **설정 비용** | 요금을 특정 금액으로 대체합니다 |
| **무료 배송** | 배송 비용을 0으로 설정합니다 |
| **추가 요금 %** | 요금에 퍼센트 추가 요금을 추가합니다 |
| **추가 요금 고정** | 요금에 고정 추가 요금을 추가합니다 |

### 조건

각 규칙은 충족해야 하는 하나 이상의 조건을 가질 수 있습니다:

| 조건 | 예 |
|-----------|---------|
| **카트 금액** | 100달러 이상 주문 시 무료 배송 |
| **총 무게** | 30kg 이상 주문 시 추가 요금 |
| **상품 수량** | 5개 이상 상품 주문 시 할인 |
| **배송 지역** | 국내 배송에만 규칙 적용 |
| **배송 방법** | 특정 운송사 방법에 적용 |
| **상품** | 특정 상품에 대한 특별 요금 |
| **고객 그룹** | VIP 고객에게 무료 배송 제공 |
| **날짜 범위** | 연휴 배송 프로모션 |

### 규칙 우선순위

- 규칙은 우선순위 순서로 평가됩니다(가장 낮은 숫자부터)
- **추가 규칙 중단** — 활성화되면 이 규칙이 일치하면 추가 규칙이 확인되지 않습니다
- 여러 규칙이 중첩될 수 있습니다(예: 10% 할인 규칙과 무료 배송 기준 규칙)

## 요금 표

요금 표는 주문 속성에 따라 계층식 가격을 제공합니다. **설정 > 배송 요금 표**로 이동하여 설정합니다.

### 표 유형

다음 기준으로 요금 계층을 생성합니다:
- **무게** — 총 주문 무게에 따라 가격 계층(예: 0-1kg = 5달러, 1-5kg = 10달러)
- **주문 금액** — 카트 합계에 따라 가격 계층
- **수량** — 상품 수량에 따라 가격 계층

### 요금 표 생성

1. **+ 요금 표 추가**를 클릭합니다
2. 표 이름을 지정하고 계층 유형을 선택합니다
3. 최소/최대 범위와 가격을 가진 계층을 추가합니다
4. 요금 표를 배송 지역에 할당합니다

요금 표는 운송사 API 요금을 사용하지 않고 자체 가격 구조를 정의할 때 유용합니다.

## 배송 패키지

정확한 요금 계산을 위해 표준 포장 크기를 정의합니다. **설정 > 배송 패키지**로 이동합니다.

각 패키지 유형에 대해 다음을 설정합니다:
- **이름** — 설명(예: "작은 상자", "대형 평가 요금")
- **크기** — 길이, 너비, 높이
- **최대 무게** — 패키지가 견딜 수 있는 최대 무게
- **기본** — 특정 포장이 할당되지 않은 경우 이 패키지를 사용합니다

운송사는 패키지의 크기를 기준으로 무게 계산에 사용합니다. 이는 배송 요금에 영향을 줄 수 있습니다.

## 수동 운송사(운송사 사전 설정)

API 통합이 없는 운송사의 경우 수동 운송사 사전 설정을 생성합니다:

1. **설정 > 운송사 사전 설정**으로 이동합니다
2. **+ 사전 설정 추가**를 클릭합니다
3. 다음을 구성합니다:
   - **운송사 이름** — 체크아웃에 표시되는 이름
   - **추적 URL 템플릿** — {tracking_number} 플레이스홀더가 포함된 URL 패턴(예: `https://track.carrier.com/?id={tracking_number}`)
   - **예상 배송일** — 고객에게 표시할 배송 시간 범위
4. 가격을 위해 요금 표와 연결합니다

수동 운송사는 실시간 API 통합 없이도 추적 링크와 배송 예상 시간을 제공합니다.

## 다중 창고 배송

다중 창고가 있는 경우 배송은 다른 출발지에서 계산될 수 있습니다:

- **국가별 창고** — 특정 국가에 창고를 할당하여 더 짧은 배송 거리
- **대체 체인** — 주 창고가 재고가 없을 때 어떤 창고가 배송하는지 정의
- **상품별 할당** — 일부 상품은 특정 창고에서만 배송할 수 있음

시스템은 고객의 위치와 상품 재고에 따라 최적의 창고를 자동으로 선택합니다.

## 팁

- 가능한 한 운송사 API를 연결하여 **실시간 요금**을 사용하세요 — 이는 평면 요금표보다 더 정확하며 무게, 크기 및 목적지를 고려합니다.
- **"Rest of World"** 배송 지역을 생성하여 특정 지역이 아닌 국가를 위한 잔여 영역으로 사용하세요.
- **무료 배송** 규칙 유형을 사용하여 카트 금액 조건을 함께 설정하여 판매 유도(예: "75달러 이상 주문 시 무료 배송")
- 실제 출시 전 다양한 주소 및 카트 내용으로 배송 요금 계산을 테스트하세요.
- API 통합이 없는 로컬 운송사에 대해 **운송사 사전 설정**을 설정하여 추적 URL 템플릿을 포함하세요 — 고객은 여전히 추적 링크를 받습니다.
- **배송 패키지**를 사용하여 FedEx 및 UPS와 같은 운송사에서 정확한 차원 무게 요금을 받습니다.
