---
title: 배송 제공업체
---

배송 제공업체는 스토어를 운송사 API에 연결하여 실시간 배송 비용, 라벨 생성 및 패키지 추적을 제공합니다. Spwig은 전 세계 주요 운송사를 지원하며, API 통합이 없는 운송사에 대해 수동 비용표를 설정할 수도 있습니다.

![배송 제공업체](/static/core/admin/img/help/shipping-providers/provider-list.webp)

## 사용 가능한 운송사

| 운송사 | 지역 | 주요 기능 |
|---------|---------|-------------|
| **FedEx** | 글로벌 | 실시간 비용, 라벨 인쇄, 추적, 다중 패키지 |
| **UPS** | 글로벌 | 실시간 비용, 라벨 인쇄, 추적, 주소 검증 |
| **USPS** | 미국 | 국내 및 국제 비용, 추적 |
| **NinjaVan** | 동남아시아 | 최종 마일 배송, 현금 대금 지원 |
| **Canada Post** | 캐나다 | 국내 및 국제, 패키지 및 편지 비용 |
| **Australia Post** | 호주 | 국내 및 국제, 패키지 및 익스프레스 |

## 운송사 연결

**설정 > 배송 제공업체**로 이동하여 **Provider 연결**을 클릭하여 설정 마법사를 시작합니다.

### 단계 1: 제공업체 선택

사용 가능한 배송 제공업체 중에서 선택합니다. 각 카드는 제공업체가 지원하는 지역 및 기능을 표시합니다.

### 단계 2: 설정 지침

운송사별 설정 가이드를 확인하세요:
- 운송사와 함께 개발자/사업 계정을 만드는 방법
- API 자격 증명을 어디에서 찾을 수 있는지
- 필요한 계정 설정 (예: 발송자 번호, 미터 번호)

### 단계 3: 자격 증명 입력

운송사 계정의 API 자격 증명을 입력합니다. 필요한 필드는 제공업체에 따라 다릅니다:

- **API 키 / 비밀** — 인증 자격 증명
- **계정 번호** — 운송사 계정 또는 발송자 번호
- **미터 번호** — 일부 운송사에서 필요 (예: FedEx)
- **샌드박스 모드** — 실제 배송 전에 운송사의 샌드박스 API로 테스트할 수 있도록 활성화

### 단계 4: 연결 테스트

**연결 테스트**를 클릭하여 자격 증명을 확인합니다. 마법사는 다음을 확인합니다:
- API 인증 성공
- 계정 권한이 유효함
- 비용 조회가 예상 결과를 반환함

### 단계 5: 구성 및 저장

설정을 완료하세요:
- **활성화** — 운송업체를 활성화하거나 비활성화합니다
- **표시 이름** — 결제 시 고객에게 표시되는 이름
- **기본 주소** — 요금 계산을 위한 창고 또는 출고 주소

## 배송 지역

배송 지역은 요금 계산을 위한 지리적 영역을 정의합니다. **설정 > 배송 지역**으로 이동하여 관리할 수 있습니다.

### 지역 생성

1. **+ 지역 추가**를 클릭합니다
2. 지역에 이름을 지정합니다 (예: "국내", "유럽", "아시아 태평양")
3. 다음 중 하나 이상을 사용하여 지역의 범위를 정의합니다:
   - **국가** — 특정 국가를 선택합니다
   - **주/도** — 국가 내 특정 지역으로 좁힙니다
   - **우편 번호 패턴** — 패턴을 사용하여 우편/ZIP 코드를 일치시킵니다 (예: "90*"은 로스앤젤레스 지역에 해당)
4. **우선순위**를 설정합니다 — 지역이 겹칠 경우 가장 높은 우선순위의 지역이 사용됩니다

### 지역 일치

고객이 결제 시 배송 주소를 입력하면 시스템은 다음과 같이 작동합니다:
1. 우편 번호 패턴을 먼저 확인합니다 (가장 구체적)
2. 그 다음 주/도 일치를 확인합니다
3. 그 다음 국가 일치를 확인합니다
4. 가장 높은 우선순위의 일치 지역을 사용합니다

## 배송 프로모션

배송 프로모션은 배송 요금에 조건부 수정을 적용합니다. **설정 > 배송 프로모션**으로 이동하여 구성할 수 있습니다.

### 프로모션 유형

| 프로모션 유형 | 설명 |
|-----------|-------------|
| **할인 %** | 배송 요금을 일정 비율만큼 감소시킵니다 |
| **할인 고정** | 배송 요금을 고정 금액만큼 감소시킵니다 |
| **비용 대체** | 특정 금액으로 요금을 대체합니다 |
| **무료 배송** | 배송 비용을 0으로 설정합니다 |
| **추가 요금 %** | 요금에 일정 비율의 추가 요금을 적용합니다 |
| **추가 요금 고정** | 요금에 고정 금액의 추가 요금을 적용합니다 |

### 조건

각 프로모션은 충족해야 하는 하나 이상의 조건을 가질 수 있습니다:

| 조건 | 예시 |
|-----------|---------|
| **카트 가치** | 100달러 이상 주문 시 무료 배송 |
| **총 무게** | 30kg 이상 주문 시 추가 요금 |
| **상품 수량** | 5개 이상 주문 시 할인 |
| **배송 지역** | 국내 배송에만 프로모션 적용 |
| **배송 방식** | 특정 운송업체 방식에 적용 |
| **상품** | 특정 상품에 대한 특별 가격 |
| **고객 그룹** | VIP 고객에게 무료 배송 제공 |
| **날짜 범위** | 연휴 배송 프로모션 |

### 프로모션 우선순위

- 프로모션은 우선순위 순서에 따라 평가됩니다 (숫자가 낮은 순으로)
- **추가 프로모션 중단** — 활성화 시 이 프로모션이 일치하면 추가 프로모션은 확인하지 않습니다
- 여러 프로모션은 중첩될 수 있습니다 (예: 10% 할인 프로모션과 무료 배송 기준 프로모션)

## 요금표

요금표는 주문 속성에 따라 계층식 가격을 제공합니다. **설정 > 배송 요금표**로 이동하여 구성할 수 있습니다.

### 표 유형

다음 기준에 따라 요금 계층을 생성합니다:
- **무게** — 총 주문 무게에 따라 가격 계층 (예: 0-1kg = 5달러, 1-5kg = 10달러)
- **주문 가치** — 카트 소계에 따라 가격 계층
- **수량** — 항목 수에 따라 가격 계층

### 요금표 생성

1. **+ 요금표 추가**를 클릭합니다
2. 표 이름을 지정하고 계층 유형을 선택합니다
3. 최소/최대 범위와 가격을 포함한 계층을 추가합니다
4. 요금표를 배송 지역에 할당합니다

요금표는 운송업체 API 요금을 사용하지 않고 자체 가격 구조를 정의할 때 유용합니다.

## 배송 패키지

정확한 요금 계산을 위해 표준 포장 크기를 정의합니다. **설정 > 배송 패키지**로 이동합니다.

각 패키지 유형에 대해 다음을 설정합니다:
- **이름** — 설명 (예: "작은 상자", "대형 고정 요금")
- **크기** — 길이, 너비, 높이
- **최대 무게** — 패키지가 견딜 수 있는 최대 무게
- **기본값** — 특정 포장이 할당되지 않은 경우 이 패키지를 사용합니다

운송업체는 패키지 크기를 기반으로 차원 무게 계산을 수행하며, 이는 배송 요금에 영향을 줄 수 있습니다.

## 수동 운송업체 (운송업체 사전 설정)

API 통합이 없는 운송업체의 경우 수동 운송업체 사전 설정을 생성합니다:

1. **설정 > 운송업체 사전 설정**으로 이동
2. **+ 사전 설정 추가**를 클릭
3. 구성:
   - **운송업체 이름** — 체크아웃에서 표시할 이름
   - **추적 URL 템플릿** — `{tracking_number}` 플레이스홀더가 포함된 URL 패턴 (예: `https://track.carrier.com/?id={tracking_number}`)
   - **예상 배송 시간** — 고객에게 표시할 배송 시간 범위
4. 가격 책정을 위해 가격표와 연결

수동 운송업체는 실시간 API 통합 없이도 추적 링크와 배송 예상 시간을 제공합니다.

## 다중 창고 배송

다중 창고를 사용하는 경우, 배송은 다른 출발지에서 계산될 수 있습니다:

- **국가별 창고** — 특정 국가에 창고를 할당하여 배송 거리를 줄입니다
- **대체 체인** — 주 창고가 재고가 없을 때 어떤 창고가 배송하는지 정의합니다
- **제품별 할당** — 일부 제품은 특정 창고에서만 배송될 수 있습니다

시스템은 고객의 위치와 제품 재고 상태에 따라 최적의 창고를 자동으로 선택합니다.

## 팁

- 가능한 한 **실시간 가격**을 위해 운송업체 API를 연결하세요 — 이는 평균 가격표보다 더 정확하며, 무게, 크기 및 목적지에 따라 조정됩니다.
- **"Rest of World"** 배송 구역을 특정 구역이 아닌 국가를 위한 대체 구역으로 생성하세요.
- **무료 배송** 프로모션 유형을 사용하여 장바구니 금액 조건을 설정하여 판매 유인을 제공할 수 있습니다 (예: "75달러 이상 주문 시 무료 배송").
- 실제 출시 전 다양한 주소와 장바구니 내용으로 배송 가격 계산을 테스트하세요.
- API 통합이 없는 지역 운송업체에 대해 **운송업체 사전 설정**을 생성하여 추적 URL 템플릿을 설정하세요 — 고객은 여전히 추적 링크를 받습니다.
- **배송 패키지**를 사용하여 FedEx 및 UPS와 같은 운송업체에서 정확한 차원 무게 가격을 받을 수 있습니다.
