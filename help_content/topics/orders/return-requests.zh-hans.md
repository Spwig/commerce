---
title: 退货请求与处理
---

退货请求跟踪客户从发起退货到退款完成的整个流程——客户选择退货商品并提供原因，商家批准或拒绝请求，生成退货标签，检查退回商品，并处理退款。该流程会经历9个状态阶段（待处理 → 已批准 → 标签已发送 → 运输中 → 已接收 → 已检查 → 完成/拒绝/取消），并包含商品级别的退货原因、检查备注和可选的重置费用。

使用此管理员页面可以高效地查看、批准和处理客户的退货请求。

## 退货请求流程

**9阶段流程**：

### 1. 待处理（客户发起）

客户提交退货请求：
- 从订单中选择商品
- 为每个商品提供退货原因
- 可选客户备注
- 状态：`pending`

### 2. 已批准/已拒绝（商家审核）

商家审核请求：
- **批准**：允许退货，进入标签生成阶段
- **拒绝**：拒绝退货并提供拒绝原因
- 状态：`approved` 或 `rejected`

### 3. 标签已发送（退货运输）

生成退货标签：
- 商家创建退货发货（可选）
- 退货标签通过电子邮件发送给客户
- 客户将商品退回
- 状态：`label_sent`

### 4. 运输中（客户发货）

客户发货：
- 跟踪显示运输状态
- 通过承运商webhook自动更新状态
- 状态：`in_transit`

### 5. 已接收（到达仓库）

商品到达：
- 仓库扫描包裹
- 商品入库
- 状态：`received`

### 6. 已检查（质量检查）

商家检查商品：
- 记录商品状况（优秀/良好/可接受/损坏/有缺陷）
- 添加检查备注
- 如适用，应用重置费用
- 状态：`inspected`

### 7. 已完成（退款处理）

退款处理：
- 创建相关退款
- 处理付款
- 退货关闭
- 状态：`completed`

**替代结果**：
- **已取消**：客户在发货前取消
- **已拒绝**：商家审核后拒绝

## 退货请求处理

**分步操作**：

**步骤1：查看待处理请求**
- 导航至 订单 > 退货请求
- 过滤状态 = "待处理"
- 点击请求查看详情

**步骤2：评估请求**
- 查看订单详情
- 检查退货原因
- 验证是否符合退货政策（在退货窗口内，商品是否符合退货条件）

**步骤3：批准或拒绝**
- 点击 "批准" 接受退货
- 或点击 "拒绝" 并输入拒绝原因
- 保存决定

**步骤4：生成退货标签**（如果已批准）
- 点击 "创建退货发货"
- 选择承运商/服务
- 系统生成退货标签
- 标签自动通过电子邮件发送给客户
- 状态 → `label_sent`

**步骤5：监控运输**
- 通过承运商webhook自动同步跟踪更新
- 当承运商扫描包裹时，状态自动推进至 `in_transit`

**步骤6：接收商品**
- 当商品到达时，点击 "标记为已接收"
- 状态 → `received`

**步骤7：检查商品**
- 打开退货请求
- 从下拉菜单中选择商品状况：
  - 优秀（如新，可重新销售）
  - 良好（轻微磨损，可重新销售）
  - 可接受（明显磨损，可折扣重新销售）
  - 损坏（不可重新销售）
  - 有缺陷（制造缺陷）
- 添加检查备注
- 可选：应用重置费用（百分比或固定金额）
- 状态 → `inspected`

**步骤8：处理退款**
- 点击 "创建退款"
- 系统计算退款金额：
  - 原始商品价格
  - 减去重置费用（如果适用）
  - 减去不可退运费（如果适用）
- 创建退款（链接到退货请求）
- 状态 → `completed`

## 商品级别的退货原因

客户为每个商品选择原因：

**常见原因**：
- 收到错误商品
- 商品有缺陷/损坏
- 改变主意/不再需要
- 商品与描述不符
- 找到更优惠的价格
- 误购
- 质量未达预期

**用于**：
- 分析（跟踪常见退货原因）
- 质量控制（识别有缺陷的商品）
- 流程改进（减少可预防的退货）

## 重置费用

应用费用以抵消退货处理成本：

**配置**：
- **类型**：百分比（例如，15%）或固定（例如，$5）
- **适用条件**：非缺陷退货，已打开的商品，特殊订单

**示例**：
```
原始购买：$100
重置费用：15%
退款金额：$85
```

**最佳实践**：
- 明确传达重置费用政策
- 不适用于有缺陷的商品
- 考虑为VIP客户提供豁免

## 退货检查指南

建立一致的检查标准：

**优秀**：
- 未打开的原始包装
- 没有明显磨损
- 所有配件齐全
- 可以以原价重新销售

**良好**：
- 已打开但使用极少
- 包装轻微磨损
- 所有组件齐全
- 可以以原价重新销售

**可接受**：
- 明显使用/磨损
- 包装损坏
- 缺失非必需配件
- 可以以折扣重新销售

**损坏**：
- 物理损坏
- 缺失部件
- 无法重新销售
- 需要丢弃或修理

**有缺陷**：
- 制造缺陷
- 功能故障
- 保修索赔
- 退回制造商

## 退货运输选项

**选项1：客户支付退货运费**
- 不提供退货标签
- 客户选择自己的承运商
- 手动输入跟踪号

**选项2：商家提供预付标签**
- 通过承运商账户生成退货标签
- 运费从退款中扣除或由商家承担
- 自动同步跟踪信息

**选项3：免费退货运输**
- 商家承担退货运费
- 提高客户满意度
- 增加退货率（需权衡利弊）

## 过滤与报告

**有用过滤器**：
- 状态：待处理（需处理）
- 日期范围：最近30天
- 订单：特定订单查询
- 原因：跟踪退货原因

**退货分析**：
- 按商品的退货率
- 最常见的退货原因
- 平均处理时间（待处理 → 完成）
- 重置费用收入

---

## 小贴士

- **设定明确的退货政策** - 传达退货窗口（30天），条件，费用
- **及时处理请求** - 在24小时内响应待处理请求
- **彻底检查** - 记录状况以避免争议
- **跟踪退货原因** - 使用数据改进产品/描述
- **尽可能自动化** - 承运商webhook自动更新运输状态
- **与客户沟通** - 在每个状态变化时发送电子邮件更新
- **公平处理重置费用** - 一致应用，对缺陷商品豁免
- **监控退货欺诈** - 标记退货次数过多的客户
- **改进包装** - 减少与损坏相关的退货
- **及时更新库存** - 检查后恢复库存
- **从模式中学习** - 某些产品的高退货率可能表明质量问题
