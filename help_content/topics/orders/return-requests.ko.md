---
title: 반품 요청 및 처리
---

반품 요청은 고객이 반품을 시작하여 환불 완료까지의 과정을 추적합니다. 고객은 반품할 항목과 이유를 선택하고, 판매자는 요청을 승인하거나 거부하며, 반품 라벨을 생성하고 반품된 항목을 점검하고 환불을 처리합니다. 이 워크플로우는 9개의 상태 단계(대기 → 승인 → 라벨_전송 → 이동 중 → 수령 → 점검 → 완료/거부/취소)를 통해 항목별 반품 이유, 점검 메모, 선택적 재고 재입고 비용을 포함합니다.

이 관리 페이지를 사용하여 고객의 반품 요청을 효율적으로 검토, 승인 및 처리할 수 있습니다.

## 반품 요청 워크플로우

**9단계 프로세스**:

### 1. 대기 (고객이 시작)

고객이 반품 요청을 제출합니다:
- 주문에서 항목 선택
- 항목별 반품 이유 제공
- 선택적 고객 메모
- 상태: `pending`

### 2. 승인/거부 (판매자가 검토)

판매자가 요청을 검토합니다:
- **승인**: 반품 허용, 라벨 생성으로 진행
- **거부**: 반품 거부 및 거부 이유 제공
- 상태: `approved` 또는 `rejected`

### 3. 라벨_전송 (반품 배송)

반품 라벨 생성:
- 판매자가 반품 배송(선택사항) 생성
- 고객에게 반품 라벨 이메일 발송
- 고객이 항목을 다시 보내기
- 상태: `label_sent`

### 4. 이동 중 (고객이 배송)

고객이 항목을 배송:
- 추적 정보가 이동을 표시
- 운송사 웹훅에서 자동 상태 업데이트
- 상태: `in_transit`

### 5. 수령 (창고에 도착)

항목 도착:
- 창고가 배송을 스캔
- 항목 확인
- 상태: `received`

### 6. 점검 (품질 검사)

판매자가 항목을 점검:
- 항목 상태 기록(완전/좋음/가능/손상/결함)
- 점검 메모 추가
- 재고 재입고 비용 적용(적용 가능한 경우)
- 상태: `inspected`

### 7. 완료 (환불 처리)

환불 발급:
- 관련 환불 생성
- 결제 처리
- 반품 종료
- 상태: `completed`

**대체 결과**:
- **취소**: 배송 전 고객이 요청 취소
- **거부**: 검토 후 판매자가 반품 거부

## 반품 요청 처리

**단계별**:

**단계 1: 대기 요청 검토**
- 주문 > 반품 요청으로 이동
- 상태 = "대기"로 필터링
- 요청 클릭하여 세부 정보 보기

**단계 2: 요청 평가**
- 주문 세부 정보 검토
- 반품 이유 확인
- 반품 정책 준수 여부 확인(반품 기간 내, 항목 적격 여부)

**단계 3: 승인 또는 거부**
- "승인" 클릭하여 반품 허용
- 또는 "거부" 클릭 및 거부 이유 입력
- 결정 저장

**단계 4: 반품 라벨 생성** (승인 시)
- "반품 배송 생성" 클릭
- 운송사/서비스 선택
- 시스템이 반품 라벨 생성
- 라벨이 고객에게 자동 이메일 발송
- 상태 → `label_sent`

**단계 5: 이동 모니터링**
- 운송사 웹훅에서 추적 업데이트 자동 동기화
- 운송사가 패키지 스캔 시 상태 자동으로 `in_transit`으로 업데이트

**단계 6: 항목 수령**
- 항목 도착 시 "수령 표시" 클릭
- 상태 → `received`

**단계 7: 항목 점검**
- 반품 요청 열기
- 드롭다운에서 항목 상태 선택:
  - 완전 (새로 보임, 재판매 가능)
  - 좋음 (최소 사용, 재판매 가능)
  - 가능 (가시적 사용, 할인 적용 후 재판매 가능)
  - 손상 (재판매 불가)
  - 결함 (제조 결함)
- 점검 메모 추가
- 선택적: 재고 재입고 비용 적용 (백분율 또는 고정)
- 상태 → `inspected`

**단계 8: 환불 처리**
- "환불 생성" 클릭
- 시스템이 환불 금액 계산:
  - 원래 항목 가격
  - 재고 재입고 비용 (적용 시) 빼기
  - 비환불 운송비 (적용 시) 빼기
- 환불 생성 (반품 요청과 연결)
- 상태 → `completed`

---

## 항목별 반품 이유

고객은 항목별로 이유를 선택합니다:

**일반적인 이유**:
- 잘못 받은 항목
- 항목 손상/결함
- 마음 바뀜/더 이상 필요 없음
- 항목 설명과 일치하지 않음
- 더 나은 가격 찾음
- 실수로 주문
- 예상 품질과 차이

**이유 사용 목적**:
- 분석 (일반적인 반품 원인 추적)
- 품질 관리 (결함 제품 식별)
- 프로세스 개선 (예방 가능한 반품 감소)

## 재고 재입고 비용

반품 처리 비용을 상쇄하기 위해 비용 적용:

**설정**:
- **유형**: 비율 (예: 15%) 또는 고정 (예: $5)
- **적용 시점**: 비결함 반품, 개봉 항목, 특별 주문

**예시**:
```
원래 구매: $100
재고 재입고 비용: 15%
환불 금액: $85
```

**최선의 실천**:
- 재고 재입고 비용 정책을 명확히 전달
- 결함 항목에 비용 적용하지 않음
- VIP 고객에게 면제 고려

---

## 반품 점검 가이드라인

일관된 점검 기준을 수립:

**완전**:
- 개봉되지 않은 원래 포장
- 가시적 마모 없음
- 모든 액세서리 포함
- 전가격으로 완전히 재판매 가능

**좋음**:
- 개봉되었지만 최소 사용
- 최소 포장 마모
- 모든 구성 요소 포함
- 전가격으로 재판매 가능

**가능**:
- 가시적 사용/마모
- 포장 손상
- 필수 액세서리 누락
- 할인 적용 후 재판매 가능

**손상**:
- 물리적 손상
- 부품 누락
- 재판매 불가
- 폐기 또는 수리 필요

**결함**:
- 제조 결함
- 기능 실패
- 보증 청구
- 제조사로 반품

## 반품 배송 옵션

**옵션 1: 고객이 반품 배송 비용 지불**
- 반품 라벨 제공 없음
- 고객이 자체 운송사 선택
- 수동 추적 번호 입력

**옵션 2: 판매자가 사전 지급 라벨 제공**
- 제공업체 계정을 통해 반품 라벨 생성
- 환불에서 비용 차감 또는 판매자가 부담
- 추적 자동 동기화

**옵션 3: 무료 반품 배송**
- 판매자가 반품 배송 비용 부담
- 고객 만족도 향상
- 반품률 증가 (거래-offs 고려)

## 필터링 및 보고

**유용한 필터**:
- 상태: 대기 (조치 필요)
- 날짜 범위: 최근 30일
- 주문: 특정 주문 검색
- 이유: 반품 원인 추적

**반품 분석**:
- 제품별 반품률
- 가장 흔한 반품 이유
- 평균 처리 시간 (대기 → 완료)
- 재고 재입고 비용 수익

---

## 팁

- **명확한 반품 정책 설정** - 윈도우(30일), 조건, 비용 통신
- **요청을 신속히 처리** - 대기 요청에 24시간 이내 응답
- **점검 철저히** - 분쟁 방지 위해 상태 기록
- **반품 이유 추적** - 데이터를 활용하여 제품/설명 개선
- **가능한 경우 자동화** - 운송사 웹훅으로 이동 상태 자동 업데이트
- **고객과 소통** - 상태 변경 시 이메일 업데이트
- **재고 재입고 비용에 공정한 적용** - 일관성 있게 적용, 결함 시 면제
- **반품 사기 모니터링** - 과도한 반품을 하는 고객 표시
- **포장 개선** - 손상 관련 반품 감소
- **재고 즉시 업데이트** - 점검 후 재고 복구
- **패턴에서 배우기** - 특정 제품의 고반품률은 품질 문제를 나타낼 수 있음
