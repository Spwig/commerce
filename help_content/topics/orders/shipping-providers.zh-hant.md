---
title: 運遞服務商
---

運遞服務商可將您的商店連接到運遞商的 API，以實現即時運費、標籤生成和包裹追蹤。Spwig 支援全球主要的運遞商，也允許您為沒有 API 整合的運遞商設定手動費用表。

![運遞服務商](/static/core/admin/img/help/shipping-providers/provider-list.webp)

## 可用的運遞商

| 運遞商 | 地區 | 主要功能 |
|---------|---------|-------------|
| **FedEx** | 全球 | 即時費用、標籤列印、追蹤、多包裹 |
| **UPS** | 全球 | 即時費用、標籤列印、追蹤、地址驗證 |
| **USPS** | 美國 | 國內及國際費用、追蹤 |
| **NinjaVan** | 東南亞 | 最後一哩配送、現金到付支援 |
| **Canada Post** | 加拿大 | 國內及國際、包裹及信件費用 |
| **Australia Post** | 澳洲 | 國內及國際、包裹及快遞 |

## 連接運遞商

前往 **設定 > 運遞服務商**，並點擊 **連接服務商** 以啟動設定向導。

### 第 1 步：選擇服務商

從可用的運遞商中選擇。每個卡片都會顯示該運遞商支援的地區和功能。

### 第 2 步：設定說明

查看特定運遞商的設定指南：
- 如何在運遞商處建立開發者/商業帳號
- 如何找到您的 API 認證
- 所需帳號設定（例如：發貨人編號、計量編號）

### 第 3 步：輸入認證

輸入您運遞商帳號的 API 認證。所需欄位因運遞商而異：

- **API 金鑰 / 密碼** — 認證憑證
- **帳號編號** — 您的運遞商帳號或發貨人編號
- **計量編號** — 某些運遞商需要（例如：FedEx）
- **沙盒模式** — 啟用後可在運遞商的沙盒 API 上進行測試，再正式上線

### 第 4 步：測試連接

點擊 **測試連接** 以驗證您的認證。向導會確認：
- API 認證成功
- 帳號權限有效
- 費用查詢返回預期結果

### 第5步：配置和保存

完成设置：
- **启用** — 啟用或停用運送商
- **顯示名稱** — 在結帳時顯示給客戶的名稱
- **原產地地址** — 用於計算運費的倉庫或履行地址

## 運送區域

運送區域定義用於計算運費的地理區域。請前往 **設定 > 運送區域** 來管理它們。

### 建立區域

1. 點擊 **+ 添加區域**
2. 為區域命名（例如，"國內"、"歐洲"、"亞太地區"）
3. 使用以下一種或多種方式定義區域的範圍：
   - **國家** — 選擇特定國家
   - **州/省份** — 縮小到國家內的特定區域
   - **郵政編碼模式** — 使用模式匹配郵政/ZIP 編碼（例如，"90*" 用於洛杉磯地區）
4. 設定 **優先順序** — 當區域重疊時，優先順序最高的區域會被使用

### 區域匹配

當客戶在結帳時輸入運送地址時，系統會：
1. 首先檢查郵政編碼模式（最特定）
2. 然後檢查州/省份匹配
3. 然後檢查國家匹配
4. 使用優先順序最高的匹配區域

## 運送優惠

運送優惠會根據條件對運費進行修改。請前往 **設定 > 運送優惠** 來配置它們。

### 優惠類型

| 優惠類型 | 說明 |
|-----------|-------------|
| **折扣百分比** | 將運費減少一定百分比 |
| **固定折扣** | 將運費減少固定金額 |
| **覆蓋成本** | 用特定金額覆蓋運費 |
| **免運費** | 將運費設為零 |
| **附加費用百分比** | 對運費增加一定百分比的附加費用 |
| **固定附加費用** | 對運費增加固定附加費用 |

### 條件

每個優惠可以有一個或多個必須滿足的條件：

| 條件 | 範例 |
|-----------|---------|
| **購物車金額** | 訂單滿 $100 免運費 |
| **總重量** | 訂單超過 30 公斤收取附加費用 |
| **商品數量** | 訂單包含 5 項以上商品可享折扣 |
| **運送區域** | 僅適用於國內運送的優惠活動 |
| **運送方式** | 適用於特定運送方式 |
| **商品** | 特定商品的特殊價格 |
| **客戶群組** | VIP 客戶享有免運費 |
| **日期範圍** | 节日運送優惠 |

### 優惠優先順序

- 優惠會按照優先順序進行評估（數字越小優先順序越高）
- **停止進一步優惠** — 啟用後，若此優惠符合條件，將不再檢查其他優惠
- 多個優惠可以疊加（例如，10% 折扣優惠加上免運費門檻優惠）

## 費率表

費用表根據訂單屬性提供階梯定價。請前往 **設定 > 運費費用表** 進行設定。

### 表格類型

根據以下條件建立費用階梯：
- **重量** — 按總訂單重量設定價格階梯（例如，0-1 公斤 = $5，1-5 公斤 = $10）
- **訂單金額** — 按購物車小計設定價格階梯
- **數量** — 按商品數量設定價格階梯

### 建立費用表

1. 點擊 **+ 新增費用表**
2. 為費用表命名並選擇階梯類型
3. 添加包含最小/最大範圍和價格的階梯
4. 將費用表分配給運送區域

當你不使用運送商 API 費率，而是希望自行定義價格結構時，費用表會很有幫助。

## 運送包裝

定義標準包裝尺寸以進行準確的費用計算。請前往 **設定 > 運送包裝**。

對於每種包裝類型，請設定：
- **名稱** — 描述（例如，"小盒子"，"大平郵包裝"）
- **尺寸** — 長度、寬度、高度
- **最大重量** — 包裝可承載的最大重量
- **預設** — 當未指定特定包裝時使用此包裝

運送商會使用包裝尺寸進行體積重量計算，這可能會影響運費。

## 手動運輸商（運輸商預設）

對於沒有 API 整合的運輸商，請建立手動運輸商預設：

1. 進入 **設定 > 運輸商預設**
2. 點擊 **+ 新增預設**
3. 設定：
   - **運輸商名稱** — 用於結帳介面顯示的名稱
   - **追蹤 URL 模板** — 帶有 `{tracking_number}` 占位符的 URL 模式（例如，`https://track.carrier.com/?id={tracking_number}`）
   - **預估送達時間** — 顯示給客戶的送達時間範圍
4. 與定價表配對

手動運輸商可在不使用即時 API 整合的情況下提供追蹤連結和送達預估。

## 多倉庫運輸

如果您有多個倉庫，運輸費用可以根據不同的出貨地點進行計算：

- **特定國家的倉庫** — 將倉庫指派給特定國家，以縮短運輸距離
- **備用鏈** — 當主要倉庫缺貨時，定義哪個倉庫負責出貨
- **按產品指派** — 某些產品可能只能從特定倉庫出貨

系統會根據客戶的位置和產品庫存自動選擇最佳倉庫。

## 小提示

- 當有可能時，請連接運輸商的 API 以取得 **即時費用** — 它們比固定費用表更準確，並會根據重量、尺寸和目的地進行調整。
- 建立一個 **「其他地區」** 的運輸區域作為一個萬用區域，用於未被特定區域涵蓋的國家。
- 使用 **免運費** 的促銷類型，並設定購物車金額條件作為銷售激勵（例如，「購物滿 $75 免運費」）。
- 在正式上線前，使用不同的地址和購物車內容測試運費計算。
- 設定 **運輸商預設**，並為任何沒有 API 整合的本地運輸商建立追蹤 URL 模板 — 這樣客戶仍然可以獲得追蹤連結。
- 使用 **運輸包裝** 來從 FedEx 和 UPS 等運輸商處獲得準確的尺寸重量費用。
