title: वितरण प्रदाता

वितरण प्रदाता आपके स्टोर को वितरक API के साथ जोड़ते हैं ताकि वास्तविक वितरण दर, लेबल उत्पादन और पैकेज ट्रैकिंग के लिए जानकारी प्राप्त की जा सके। Spwig विश्व के प्रमुख वितरकों के समर्थन के साथ आता है और ऐसे वितरकों के लिए मैनुअल दर तालिकाएं भी बनाने देता है जिनके पास API एकीकरण नहीं है।

![वितरण प्रदाता](/static/core/admin/img/help/shipping-providers/provider-list.webp)

## उपलब्ध वितरक

| वितरक | क्षेत्र | मुख्य विशेषताएं |
|---------|---------|-------------|
| **FedEx** | वैश्विक | वास्तविक दर, लेबल प्रिंटिंग, ट्रैकिंग, बहु-पैकेज |
| **UPS** | वैश्विक | वास्तविक दर, लेबल प्रिंटिंग, ट्रैकिंग, पता सत्यापन |
| **USPS** | संयुक्त राज्य अमेरिका | घरेलू और अंतरराष्ट्रीय दर, ट्रैकिंग |
| **NinjaVan** | दक्षिण पूर्व एशिया | अंतिम मील डिलीवरी, कैश ऑन डिलीवरी समर्थन |
| **Canada Post** | कनाडा | घरेलू और अंतरराष्ट्रीय, पैकेज और पत्र दर |
| **Australia Post** | ऑस्ट्रेलिया | घरेलू और अंतरराष्ट्रीय, पैकेज और एक्सप्रेस |

## एक वितरक को जोड़ें

**सेटिंग > शिपिंग प्रोवाइडर** जाएं और **प्रोवाइडर कनेक्ट करें** पर क्लिक करके सेटअप विजार्ड लॉन्च करें।

### चरण 1: प्रोवाइडर चुनें

उपलब्ध शिपिंग कैरियर्स में से चुनें। प्रत्येक कार्ड कैरियर द्वारा समर्थित क्षेत्रों और विशेषताओं को दिखाता है।

### चरण 2: सेटअप निर्देश

कैरियर के विशिष्ट सेटअप गाइड की समीक्षा करें:
- कैरियर के साथ एक डेवलपर/व्यवसाय खाता बनाने के तरीके
- अपने API ज्वॉइन्ट्स कहां पाए जा सकते हैं
- आवश्यक खाता सेटिंग (जैसे, शिपर संख्या, मीटर संख्या)

### चरण 3: ज्वॉइन्ट्स दर्ज करें

अपने कैरियर खाते के लिए API ज्वॉइन्ट्स दर्ज करें। आवश्यक क्षेत्र विभिन्न कैरियर्स द्वारा भिन्न होते हैं:

- **API कुंजी / रहस्य** — प्रमाणिकरण ज्वॉइन्ट्स
- **खाता संख्या** — आपका कैरियर खाता या शिपर संख्या
- **मीटर संख्या** — कुछ कैरियर्स द्वारा आवश्यक (जैसे, फेडेक्स)
- **सैंडबॉक मोड** — लाइव होने से पहले कैरियर के सैंडबॉक API के साथ परीक्षण करने के लिए इसे एनेबल करें

### चरण 4: कनेक्शन परीक्षण करें

**कनेक्शन परीक्षण** पर क्लिक करके अपने ज्वॉइन्ट्स की पुष्टि करें।

मैजिक निश्चित करता है:
- API प्रमाणीकरण सफल होता है
- खाता अनुमतियाँ वैध हैं
- दर प्रश्न अपेक्षित परिणाम लौटाते हैं

### चरण 5: सेटिंग करें और सहेजें

सेटिंग्स को अंतिम रूप दें:
- **सक्रिय** — डिलीवरी को एनेबल या डिसएबल करें
- **दिखाए जाने वाला नाम** — चेकआउट पर ग्राहकों के लिए दिखाया जाने वाला नाम
- **मूल पता** — दर गणना के लिए वर्कशॉप या फुलफिलमेंट पता

## शिपिंग जोन

शिपिंग जोन दर गणना के लिए भौगोलिक क्षेत्रों को परिभाषित करते हैं। **सेटिंग्स > शिपिंग जोन** जाएं उनका प्रबंधन करने के लिए।

### एक जोन बनाएं

1. **+ जोन जोड़ें** पर क्लिक करें
2. जोन के लिए एक नाम दें (उदाहरण के लिए, "देशीय", "यूरोप", "एशियाई प्रशांत")
3. जोन के कवरेज को एक या अधिक के उपयोग से परिभाषित करें:
   - **देश** — विशिष्ट देशों का चयन करें
   - **राज्य/प्रांत** — एक देश के भीतर विशिष्ट क्षेत्रों तक सीमित करें
   - **पोस्टल कोड पैटर्न** — पोस्टल/ZIP कोड के मैच के लिए पैटर्न का उपयोग करें (उदाहरण के लिए, "90*" लॉस एंजिल्स क्षेत्र के लिए)
4. **तीव्रता** को सेट करें — जब जोन ओवरलैप होते हैं, तो सबसे उच्च प्राथमिकता वाला जोन उपयोग किया जाता है

### जोन मैचिंग

जब एक ग्राहक चेकआउट पर अपना शिपिंग पता दर्ज करता है, तो प्रणाली:
1.

पहले डाक कोड पैटर्न की जांच करती है (सबसे विशिष्ट)
2.

फिर राज्य/प्रांत मैच करती है
3.

फिर देश मैच करती है
4.

उच्चतम-प्राथमिकता वाले मैचिंग क्षेत्र का उपयोग करती है

## शिपिंग प्रोमोशन

शिपिंग प्रोमोशन शिपिंग दरों पर शर्तों वाले संशोधक लागू करते हैं। **सेटिंग्स > शिपिंग प्रोमोशन** जाएं और उन्हें विनिर्माण करें।

### प्रोमोशन प्रकार

| प्रोमोशन प्रकार | विवरण |
|-----------|-------------|
| **डिस्काउंट %** | शिपिंग दर में प्रतिशत कमी करें |
| **डिस्काउंट फिक्स्ड** | शिपिंग दर में एक निश्चित राशि कम करें |
| **ओवरराइड कॉस्ट** | दर को एक विशिष्ट राशि से बदल दें |
| **फ्री शिपिंग** | शिपिंग लागत को शून्य करें |
| **सर्चाज %** | दर में प्रतिशत सर्चाज जोड़ें |
| **सर्चाज फिक्स्ड** | दर में एक निश्चित सर्चाज जोड़ें |

### शर्तें

प्रत्येक प्रोमोशन में एक या अधिक शर्तें हो सकती हैं जो पूरी होनी चाहिए:

| शर्त | उदाहरण |
|-----------|---------|
| **कार्ट की कीमत** | $100 से ऊपर के ऑर्डर पर मुफ्त शिपिंग |
| **कुल वजन** | 30 किलोग्राम से ऊपर के ऑर्डर पर एक अतिरिक्त शुल्क |
| **आइटम की संख्या** | 5 या उससे अधिक आइटम वाले ऑर्डर पर छूट |
| **शिपिंग ज़ोन** | विशेष रूप से घरेलू शिपमेंट पर प्रोमोशन लागू करें |
| **शिपिंग विधि** | विशिष्ट वाहक विधियों पर लागू करें |
| **उत्पाद** | विशिष्ट उत्पादों के लिए विशेष दर |
| **ग्राहक समूह** | वीआईपी ग्राहकों को मुफ्त शिपिंग मिलता है |
| **तिथि की अवधि** | छुट्टी के शिपिंग प्रोमोशन |

### प्रोमोशन प्राथमिकता

- प्रोमोशन की प्राथमिकता के क्रम में मूल्यांकन किया जाता है (सबसे कम संख्या से शुरू)
- **अगले प्रोमोशन बंद करें** — जब यह चालू है, तो यदि यह प्रोमोशन मिलता है, तो अगले प्रोमोशन की जांच नहीं की जाती
- एक से अधिक प्रोमोशन संयोजित हो सकते हैं (उदाहरण के लिए, 10% छूट प्रोमोशन और मुफ्त शिपिंग द्वारा छूट प्रोमोशन)

## दर टेबल

दर टेबल ऑर्डर गुणों के आधार पर टियर्ड मूल्य देते हैं। **सेटिंग्स > शिपिंग दर टेबल** जाएं और उन्हें विनिर्माण करें।

### टेबल प्रकार


कीमत टियर्स बनाएं इसके आधार पर:
- **वजन** — कुल ऑर्डर वजन के आधार पर कीमत टियर्स (जैसे, 0-1 किलोग्राम = 5 डॉलर, 1-5 किलोग्राम = 10 डॉलर)
- **ऑर्डर मूल्य** — कार्ट उपज के आधार पर कीमत टियर्स
- **मात्रा** — आइटम की संख्या के आधार पर कीमत टियर्स

### एक रेट टेबल बनाएं

1. **+ Add Rate Table** पर क्लिक करें
2. टेबल का नाम दें और टियर प्रकार चुनें
3. न्यूनतम/अधिकतम रेंज और कीमतों के साथ टियर्स जोड़ें
4. एक शिपिंग जोन के लिए रेट टेबल को निर्धारित करें

जब आप कैरियर API दरों का उपयोग नहीं करते हैं और अपनी अपनी कीमत ढांचा परिभाषित करना चाहते हैं, तो रेट टेबल उपयोगी होते हैं।

## शिपिंग पैकेज

सटीक रेट गणना के लिए मानक पैकेज आकारों को परिभाषित करें। **सेटिंग्स > शिपिंग पैकेज** पर जाएं।

प्रत्येक पैकेज प्रकार के लिए, निम्नलिखित सेट करें:
- **नाम** — विवरण (जैसे, "छोटा बॉक्स", "बड़ा फ्लैट रेट")
- **आयाम** — लंबाई, चौड़ाई, ऊंचाई
- **अधिकतम वजन** — पैकेज द्वारा धारण किए जा सकने वाले अधिकतम वजन
- **डिफ़ॉल्ट** — जब कोई विशिष्ट पैकेज निर्धारित नहीं होता है, तो इस पैकेज का उपयोग करें

कैरियर पैकेज आयामों का उपयोग आयामी वजन की गणना के लिए करते हैं, जो शिपिंग दरों पर प्रभाव डाल सकता है।

## मैनुअल कैरियर (कैरियर प्रीसेट)

API एकीकरण के बिना कैरियर के लिए मैनुअल कैरियर प्रीसेट बनाएं:

1. **सेटिंग्स > कैरियर प्रीसेट** पर जाएं
2. **+ प्रीसेट जोड़ें** पर क्लिक करें
3. निर्माण करें:
   - **कैरियर का नाम** — चेकआउट पर प्रदर्शित नाम
   - **ट्रैकिंग URL टेम्पलेट** — `{tracking_number}` प्लेसहोल्डर के साथ URL पैटर्न (उदाहरण, `https://track.carrier.com/?id={tracking_number}`)
   - **अनुमानित डिलीवरी** — ग्राहकों के लिए प्रदर्शित डिलीवरी समय के बीच का अंतराल
4. मूल्य निर्धारण के लिए एक दर तालिका के साथ जोड़ें

मैनुअल कैरियर लाइव API एकीकरण के बिना ट्रैकिंग लिंक और डिलीवरी अनुमान प्रदान करते हैं।

## बहु-वर्कशॉप शिपिंग

यदि आपके पास कई वर्कशॉप हैं, तो शिपिंग को अलग-अलग मूल से गणना की जा सकती है:

- **देश-विशेष वर्कशॉप** — छोटे शिपिंग दूरी के लिए विशिष्ट देशों के लिए वर्कशॉप निर्धारित करें
- **बैकअप चेन** — प्राथमिक वर्कशॉप स्टॉक में खाली होने पर जो वर्कशॉप शिपिंग करता है उसे परिभाषित करें
- **प्रति-उत्पाद निर्धारण** — कुछ उत्पाद केवल विशिष्ट वर्कशॉप से शिपिंग के लिए ही हो सकते हैं

सिस्टम ग्राहक के स्थान और उत्पाद उपलब्धता के आधार पर सबसे अच्छा वर्कहाउस ऑटोमैटिक रूप से चुनता है।

## सुझाव

- जब भी संभव हो, **लाइव दर** के लिए कैरियर के APIs के साथ जुड़ें — वे फ्लैट-रेट टेबल की तुलना में अधिक सटीक होते हैं और भार, आयाम और गंतव्य के अनुसार समायोजित होते हैं।
- विशिष्ट क्षेत्रों द्वारा कवर न किए गए देशों के लिए एक **"Rest of World"** शिपिंग क्षेत्र बनाएं।
- बिक्री के एक प्रोत्साहन के रूप में **मुफ्त शिपिंग** प्रोमोशन प्रकार का उपयोग करें जिसमें कार्ट मूल्य की शर्त होती है (उदाहरण के लिए, "75 डॉलर से अधिक के ऑर्डर पर मुफ्त शिपिंग")।
- लाइव होने से पहले अलग-अलग पतों और कार्ट सामग्री के साथ शिपिंग दर की गणना करें।
- किसी भी स्थानीय कैरियर के लिए **कैरियर प्रीसेट्स** के साथ ट्रैकिंग URL टेम्पलेट्स सेट करें जिनके पास API इंटीग्रेशन नहीं है — ग्राहक अभी भी ट्रैकिंग लिंक प्राप्त करते हैं।
- **शिपिंग पैकेज** का उपयोग करें ताकि फेडेक्स और यूपीएस जैसे कैरियरों से आयामी भार मूल्य के लिए अधिक सटीक दर प्राप्त की जा सके।
