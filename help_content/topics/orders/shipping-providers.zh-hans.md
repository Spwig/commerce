---
title: 运输服务提供商
---

运输服务提供商将您的商店连接到承运商的API，以获取实时运费、生成标签和跟踪包裹。Spwig支持全球主要的承运商，也允许您为没有API集成的承运商设置手动费率表。

![运输服务提供商](/static/core/admin/img/help/shipping-providers/provider-list.webp)

## 可用承运商

| 承运商 | 地区 | 主要功能 |
|---------|---------|-------------|
| **FedEx** | 全球 | 实时运费、标签打印、跟踪、多包裹 |
| **UPS** | 全球 | 实时运费、标签打印、跟踪、地址验证 |
| **USPS** | 美国 | 国内和国际运费、跟踪 |
| **NinjaVan** | 东南亚 | 最后一公里配送、支持货到付款 |
| **Canada Post** | 加拿大 | 国内和国际、包裹和信件运费 |
| **Australia Post** | 澳大利亚 | 国内和国际、包裹和快递 |

## 连接承运商

导航到 **设置 > 运输服务提供商**，然后点击 **连接服务提供商** 以启动设置向导。

### 第1步：选择服务提供商

从可用的运输服务提供商中进行选择。每张卡片显示承运商支持的地区和功能。

### 第2步：设置说明

查看特定承运商的设置指南：
- 如何在承运商处创建开发/企业账户
- 如何找到您的API凭证
- 所需的账户设置（例如，发货人编号、计量编号）

### 第3步：输入凭证

输入您承运商账户的API凭证。所需字段因承运商而异：

- **API密钥/密钥** — 认证凭证
- **账户编号** — 您的承运商账户或发货人编号
- **计量编号** — 某些承运商需要（例如，FedEx）
- **沙盒模式** — 启用后，可在承运商的沙盒API上进行测试，然后再上线

### 第4步：测试连接

点击 **测试连接** 以验证您的凭证。向导将确认：
- API认证成功
- 账户权限有效
- 费率查询返回预期结果

### 第5步：配置和保存

完成设置：
- **启用** — 启用或禁用承运商
- **显示名称** — 结账时向客户显示的名称
- **原产地地址** — 用于运费计算的仓库或配送地址

## 运费区域

运费区域定义用于运费计算的地理区域。导航到 **设置 > 运费区域** 来管理它们。

### 创建区域

1. 点击 **+ 添加区域**
2. 为区域命名（例如，"国内"，"欧洲"，"亚太地区"）
3. 使用以下一项或多项定义区域的覆盖范围：
   - **国家** — 选择特定国家
   - **州/省** — 将范围缩小到国家内的特定地区
   - **邮政编码模式** — 使用模式匹配邮政/ZIP编码（例如，"90*" 用于洛杉矶地区）
4. 设置 **优先级** — 当区域重叠时，优先级最高的区域将被使用

### 区域匹配

当客户在结账时输入他们的送货地址时，系统：
1. 首先检查邮政编码模式（最具体）
2. 然后检查州/省匹配
3. 然后检查国家匹配
4. 使用优先级最高的匹配区域

## 运费促销

运费促销会对运费应用条件性修改。导航到 **设置 > 运费促销** 来配置它们。

### 促销类型

| 促销类型 | 描述 |
|-----------|-------------|
| **折扣百分比** | 按百分比减少运费 |
| **固定折扣** | 按固定金额减少运费 |
| **覆盖成本** | 用特定金额覆盖运费 |
| **免运费** | 将运费成本设为零 |
| **附加百分比** | 向运费添加百分比附加费 |
| **固定附加费** | 向运费添加固定附加费 |

### 条件

每个促销可以有一个或多个必须满足的条件：

| 条件 | 示例 |
|-----------|---------|
| **购物车金额** | 订单金额超过 100 美元免运费 |
| **总重量** | 订单重量超过 30 公斤加收运费 |
| **商品数量** | 订单商品数量达到 5 件以上可享受折扣 |
| **配送区域** | 仅适用于国内配送 |
| **配送方式** | 适用于特定承运商配送方式 |
| **商品** | 特定商品的特殊价格 |
| **客户群体** | VIP 客户享受免运费 |
| **日期范围** | 节假日配送促销 |

### 促销优先级

- 促销按优先级顺序进行评估（数字越小优先级越高）
- **停止后续促销** — 启用后，如果此促销匹配，将不再检查其他促销
- 多个促销可以叠加（例如，10% 的折扣促销加上免运费门槛促销）

## 计费表

计费表根据订单属性提供分层定价。导航到 **设置 > 配送计费表** 进行配置。

### 表格类型

根据以下内容创建计费层级：
- **重量** — 按总订单重量分层定价（例如，0-1 公斤 = 5 美元，1-5 公斤 = 10 美元）
- **订单金额** — 按购物车小计分层定价
- **数量** — 按商品数量分层定价

### 创建计费表

1. 点击 **+ 添加计费表**
2. 为表格命名并选择层级类型
3. 添加带有最小/最大范围和价格的层级
4. 将计费表分配给一个配送区域

当不使用承运商 API 计费并希望自定义定价结构时，计费表非常有用。

## 配送包装

定义标准包装尺寸以实现准确的计费计算。导航到 **设置 > 配送包装**。

对于每种包装类型，请设置：
- **名称** — 描述（例如，"小盒子"，"大平邮"）
- **尺寸** — 长度、宽度、高度
- **最大重量** — 包装可承受的最大重量
- **默认** — 当未分配特定包装时使用此包装

承运商使用包装尺寸进行体积重量计算，这可能会影响运费。

## 手动承运商（承运商预设）

对于没有 API 集成的承运商，创建手动承运商预设：

1. 导航到 **设置 > 承运商预设**
2. 点击 **+ 添加预设**
3. 配置：
   - **承运商名称** — 结账时显示的名称
   - **跟踪 URL 模板** — 带有 `{tracking_number}` 占位符的 URL 模式（例如，`https://track.carrier.com/?id={tracking_number}`）
   - **预估送达时间** — 显示给客户的送达时间范围
4. 与一个计费表配对以进行定价

手动承运商在没有实时 API 集成的情况下提供跟踪链接和送达预估。

## 多仓库发货

如果您有多个仓库，发货可以从不同的原点计算：

- **特定国家的仓库** — 将仓库分配给特定国家以缩短发货距离
- **备用链** — 定义当主要仓库缺货时使用哪个仓库发货
- **按产品分配** — 某些产品可能只能从特定仓库发货

系统会根据客户的地理位置和产品可用性自动选择最佳仓库。

## 小贴士

- 尽可能连接承运商的 API 以获取 **实时运费** — 它们比固定费率表更准确，并会根据重量、尺寸和目的地进行调整。
- 创建一个 **"世界其他地区"** 的发货区域作为未被特定区域覆盖国家的兜底选项。
- 使用 **免运费** 促销类型并设置购物车金额条件作为销售激励（例如，"购物满 $75 免运费"）。
- 在上线之前，使用不同的地址和购物车内容测试运费计算。
- 为任何没有 API 集成的本地承运商设置 **承运商预设**，并带有跟踪 URL 模板 — 客户仍然可以获得跟踪链接。
- 使用 **发货包裹** 从 FedEx 和 UPS 等承运商获取准确的尺寸重量计费。
