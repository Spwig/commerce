title: Поставщики доставки
---

Поставщики доставки соединяют ваш магазин с API транспортных компаний для получения актуальных цен на доставку, генерации этикеток и отслеживания посылок. Spwig поддерживает основных транспортных компаний по всему миру и также позволяет настроить ручные таблицы тарифов для компаний без интеграции API.

![Поставщики доставки](/static/core/admin/img/help/shipping-providers/provider-list.webp)

## Доступные транспортные компании

| Транспортная компания | Регионы | Основные функции |
|---------------------|---------|------------------|
| **FedEx** | Глобальный | Актуальные цены, печать этикеток, отслеживание, несколько посылок |
| **UPS** | Глобальный | Актуальные цены, печать этикеток, отслеживание, проверка адреса |
| **USPS** | США | Внутренние и международные тарифы, отслеживание |
| **NinjaVan** | Юго-Восточная Азия | Доставка на последнем этапе, поддержка оплаты наличными |
| **Canada Post** | Канада | Внутренние и международные, тарифы на посылки и письма |
| **Australia Post** | Австралия | Внутренние и международные, посылки и экспресс-доставка |

## Подключение транспортной компании

Перейдите в **Настройки > Поставщики доставки** и нажмите **Подключить поставщика**, чтобы запустить мастер настройки.

### Шаг 1: Выберите поставщика

Выберите из доступных транспортных компаний. Каждая карточка показывает регионы и функции, поддерживаемые поставщиком.

### Шаг 2: Инструкции по настройке

Просмотрите инструкцию по настройке, специфичную для поставщика:
- Как создать разработчика/бизнес-аккаунт у поставщика
- Где найти ваши учетные данные API
- Требуемые настройки аккаунта (например, номер отправителя, номер счетчика)

### Шаг 3: Введите учетные данные

Введите учетные данные API для вашего аккаунта поставщика. Требуемые поля различаются в зависимости от поставщика:

- **API-ключ / секрет** — учетные данные для аутентификации
- **Номер аккаунта** — ваш номер аккаунта или номер отправителя у поставщика
- **Номер счетчика** — требуется некоторыми поставщиками (например, FedEx)
- **Режим песочницы** — включите, чтобы протестировать с API песочницы поставщика перед запуском

### Шаг 4: Тестирование подключения

Нажмите **Проверить подключение**, чтобы проверить ваши учетные данные. Мастер подтверждает:
- Успешная аутентификация API
- Действительны ли разрешения аккаунта
- Запросы тарифов возвращают ожидаемые результаты

### Шаг 5: Настройка и сохранение

Завершите настройку:
- **Активен** — включить или отключить поставщика
- **Отображаемое имя** — имя, которое будет показано покупателям при оформлении заказа
- **Адрес отправки** — склад или адрес выполнения заказа для расчета тарифов

## Зоны доставки

Зоны доставки определяют географические зоны для расчета тарифов. Перейдите в **Настройки > Зоны доставки**, чтобы управлять ими.

### Создание зоны

1. Нажмите **+ Добавить зону**
2. Дайте зоне имя (например, "Внутренние", "Европа", "Азиатско-Тихоокеанский регион")
3. Определите зону покрытия с помощью одного или нескольких из следующих:
   - **Страны** — выберите конкретные страны
   - **Штаты/провинции** — сузьте до конкретных регионов внутри страны
   - **Шаблоны почтовых индексов** — сопоставьте почтовые/ZIP-коды с помощью шаблонов (например, "90*" для района Лос-Анджелеса")
4. Установите **Приоритет** — при пересечении зон используется зона с наивысшим приоритетом

### Сопоставление зон

Когда клиент вводит свой адрес доставки при оформлении заказа, система:
1. Сначала проверяет шаблоны почтовых индексов (наиболее конкретные)
2. Затем сопоставляет штаты/провинции
3. Затем сопоставляет страны
4. Использует зону с наивысшим приоритетом, соответствующую сопоставлению

## Акции на доставку

Акции на доставку применяют условные модификаторы к тарифам на доставку. Перейдите в **Настройки > Акции на доставку**, чтобы настроить их.

### Типы акций

| Тип акции | Описание |
|-----------|-------------|
| **Скидка в процентах** | Уменьшает стоимость доставки на определенный процент |
| **Фиксированная скидка** | Уменьшает стоимость доставки на фиксированную сумму |
| **Переопределение стоимости** | Заменяет стоимость на конкретную сумму |
| **Бесплатная доставка** | Устанавливает стоимость доставки в ноль |
| **Надбавка в процентах** | Добавляет процентную надбавку к стоимости |
| **Фиксированная надбавка** | Добавляет фиксированную надбавку к стоимости |

### Условия

Каждая акция может иметь одно или несколько условий, которые должны быть выполнены:

| Условие | Пример |
|-----------|---------|
| **Значение корзины** | Бесплатная доставка для заказов свыше 100 $ |
| **Общий вес** | Дополнительная плата за заказы свыше 30 кг |
| **Количество товаров** | Скидка на заказы с 5 и более товарами |
| **Зона доставки** | Применить акцию только к отечественным доставкам |
| **Способ доставки** | Применить к конкретным способам доставки |
| **Товары** | Специальные цены на конкретные товары |
| **Группа клиентов** | Клиенты VIP получают бесплатную доставку |
| **Диапазон дат** | Акции на доставку в праздничные дни |

### Приоритет акций

- Акции оцениваются в порядке приоритета (сначала с наименьшим номером)
- **Остановить дальнейшие акции** — При включении, если эта акция совпадает, дальнейшие акции не проверяются
- Несколько акций могут накладываться друг на друга (например, акция со скидкой 10% плюс акция на бесплатную доставку при определенном пороге)

## Таблицы тарифов

Таблицы тарифов предоставляют иерархическую цену на основе атрибутов заказа. Перейдите в **Настройки > Таблицы тарифов на доставку**, чтобы настроить их.

### Типы таблиц

Создайте уровни тарифов на основе:
- **Вес** — уровни цен по общему весу заказа (например, 0-1 кг = 5 $, 1-5 кг = 10 $)
- **Значение заказа** — уровни цен по подытогу корзины
- **Количество** — уровни цен по количеству товаров

### Создание таблицы тарифов

1. Нажмите **+ Добавить таблицу тарифов**
2. Назовите таблицу и выберите тип уровня
3. Добавьте уровни с минимальными/максимальными диапазонами и ценами
4. Назначьте таблицу тарифов зоне доставки

Таблицы тарифов полезны, когда вы не используете тарифы API перевозчика и хотите определить собственную структуру цен.

## Упаковки для доставки

Определите стандартные размеры упаковок для точных расчетов тарифов. Перейдите в **Настройки > Упаковки для доставки**.

Для каждого типа упаковки задайте:
- **Название** — описание (например, "Маленькая коробка", "Большая плоская ставка")
- **Размеры** — длина, ширина, высота
- **Максимальный вес** — максимальный вес, который может вместить упаковка
- **По умолчанию** — использовать эту упаковку, когда не назначена конкретная упаковка

Перевозчики используют размеры упаковки для расчета объемного веса, что может повлиять на тарифы доставки.

## Ручные перевозчики (предустановки перевозчика)

Для перевозчиков без интеграции API создайте ручные предустановки перевозчика:

1. Перейдите в **Настройки > Предустановки перевозчика**
2. Нажмите **+ Добавить предустановку**
3. Настройте:
   - **Название перевозчика** — отображаемое название на этапе оформления заказа
   - **Шаблон URL отслеживания** — шаблон URL с заполнителем {tracking_number} (например, `https://track.carrier.com/?id={tracking_number}`)
   - **Ожидаемое время доставки** — диапазон времени доставки, который будет отображаться клиентам
4. Сопоставьте с таблицей тарифов для ценообразования

Ручные перевозчики предоставляют ссылки для отслеживания и оценки времени доставки без живой интеграции API.

## Многоскладская доставка

Если у вас есть несколько складов, доставка может рассчитываться из разных мест:

- **Склад по стране** — назначьте склады для конкретных стран для более коротких расстояний доставки
- **Цепочка резервного копирования** — определите, какой склад будет использоваться, когда основной склад вышел из строя
- **Назначение по продукту** — некоторые товары могут доставляться только с определенных складов

Система автоматически выбирает лучший склад на основе местоположения клиента и наличия товара.

## Советы

- Подключите API перевозчиков для **живых тарифов**, если это возможно — они более точны, чем плоские таблицы тарифов и учитывают вес, размеры и место назначения.
- Создайте зону доставки **"Остальной мир"** как универсальную для стран, не охваченных конкретными зонами.
- Используйте тип акции **Бесплатная доставка** с условием значения корзины как стимул для продаж (например, "Бесплатная доставка для заказов свыше 75 $")
- Тестируйте расчеты тарифов доставки с разными адресами и содержимым корзины перед запуском.
- Настройте **Предустановки перевозчика** с шаблонами URL отслеживания для любых местных перевозчиков, у которых нет интеграции API — клиенты все равно получают ссылки для отслеживания.
- Используйте **Упаковки для доставки**, чтобы получить точные цены на объемный вес от перевозчиков, таких как FedEx и UPS.
