---
title: 退貨申請與處理流程
---

退貨申請追蹤顧客從申請到退款完成的整個退貨過程—顧客選擇退貨商品並提供原因，商家批准或拒絕申請，生成退貨標籤，檢查退回商品，並處理退款。工作流程會經歷9個狀態階段（待處理 → 已批准 → 已發送標籤 → 運送中 → 已收到 → 已檢查 → 已完成/拒絕/取消），並包含商品層級的退貨原因、檢查筆記以及可選的重新入庫費用。

使用此管理頁面高效審核、批准並處理顧客退貨申請。

## 退貨申請工作流程

**9階段流程**：

### 1. 待處理（顧客啟動）

顧客提交退貨申請：
- 從訂單中選擇商品
- 提供每個商品的退貨原因
- 可選顧客備註
- 狀態：`pending`

### 2. 已批准/已拒絕（商家審核）

商家審核申請：
- **批准**：允許退貨，進入標籤生成階段
- **拒絕**：拒絕退貨並提供拒絕原因
- 狀態：`approved` 或 `rejected`

### 3. 已發送標籤（退貨運輸）

生成退貨標籤：
- 商家創建退貨運輸（可選）
- 退貨標籤郵件發送給顧客
- 顧客寄回商品
- 狀態：`label_sent`

### 4. 運送中（顧客寄送）

顧客寄送商品：
- 跟蹤顯示運送狀態
- 載運商的webhook自動更新狀態
- 狀態：`in_transit`

### 5. 已收到（到達倉庫）

商品到達：
- 倉庫掃描運送包裝
- 商品檢查入庫
- 狀態：`received`

### 6. 已檢查（品質檢查）

商家檢查商品：
- 記錄商品狀況（優良/良好/可接受/損壞/瑕疵）
- 增加檢查筆記
- 如適用，套用重新入庫費用
- 狀態：`inspected`

### 7. 已完成（退款處理）

發放退款：
- 創建相關退款
- 處理付款
- 退貨完成
- 狀態：`completed`

**替代結果**：
- **已取消**：顧客在寄送前取消
- **已拒絕**：商家審核後拒絕

## 處理退貨申請

**逐步操作**：

**步驟1：審核待處理申請**
- 前往 訂單 > 退貨申請
- 過濾狀態 = "待處理"
- 點擊申請以查看詳細資訊

**步驟2：評估申請**
- 審核訂單細節
- 檢查退貨原因
- 驗證是否符合退貨政策（在退貨期限內，商品是否符合資格）

**步驟3：批准或拒絕**
- 點擊 "批准" 以接受退貨
- 或點擊 "拒絕" 並輸入拒絕原因
- 儲存決定

**步驟4：生成退貨標籤**（如已批准）
- 點擊 "創建退貨運輸"
- 選擇運輸商/服務
- 系統生成退貨標籤
- 標籤自動郵件發送給顧客
- 狀態 → `label_sent`

**步驟5：監控運送狀態**
- 跟蹤狀態自動同步來自運輸商的webhook
- 當運輸商掃描包裹時，狀態自動進展至 `in_transit`

**步驟6：接收商品**
- 當商品到達時，點擊 "標記為已收到"
- 狀態 → `received`

**步驟7：檢查商品**
- 打開退貨申請
- 從下拉選單中選擇商品狀況：
  - 優良（如全新，可重新銷售）
  - 良好（輕微使用，可重新銷售）
  - 可接受（明顯磨損，可折扣重新銷售）
  - 損壞（不可重新銷售）
  - 瑕疵（製造缺陷）
- 增加檢查筆記
- 可選：套用重新入庫費用（百分比或固定）
- 狀態 → `inspected`

**步驟8：處理退款**
- 點擊 "創建退款"
- 系統計算退款金額：
  - 原始商品價格
  - 減去重新入庫費用（如適用）
  - 減去運費（如不可退）
- 創建退款（連結至退貨申請）
- 狀態 → `completed`

## 商品層級退貨原因

顧客為每個商品選擇原因：

**常見原因**：
- 收到錯誤商品
- 商品損壞/瑕疵
- 改變主意/不再需要
- 商品與描述不符
- 發現更好的價格
- 錯誤下單
- 品質不符合預期

**用途**：
- 分析（追蹤常見退貨原因）
- 品質控制（識別瑕疵商品）
- 流程改進（減少可預防的退貨）

---

## 重新入庫費用

套用費用以抵消退貨處理成本：

**設定**：
- **類型**：百分比（例如，15%）或固定（例如，$5）
- **套用時機**：非瑕疵退貨、已開封商品、特殊訂購

**範例**：
```
原購買金額：$100
重新入庫費用：15%
退款金額：$85
```

**最佳實踐**：
- 明確溝通重新入庫費用政策
- 不套用瑕疵商品
- 考慮對VIP顧客免除此費用

## 退貨檢查指南

建立一致的檢查標準：

**優良**：
- 未拆封的原包裝
- 無可見磨損
- 所有配件齊全
- 可以全價重新銷售

**良好**：
- 已拆封但使用極少
- 輕微包裝磨損
- 所有組件齊備
- 可以全價重新銷售

**可接受**：
- 可見使用/磨損
- 包裝損壞
- 缺少非必要配件
- 可以折扣重新銷售

**損壞**：
- 物理損壞
- 缺失零件
- 不可重新銷售
- 需要丟棄或修理

**瑕疵**：
- 製造缺陷
- 功能故障
- 保固申訴
- 退回製造商

## 退貨運輸選項

**選項1：顧客支付退貨運費**
- 不提供退貨標籤
- 顧客選擇自己的運輸商
- 手動輸入跟蹤號碼

**選項2：商家提供預付標籤**
- 透過運輸商帳號生成退貨標籤
- 費用從退款中扣除或由商家承擔
- 跟蹤狀態自動同步

**選項3：免退貨運費**
- 商家承擔退貨運費
- 提升顧客滿意度
- 增加退貨率（考慮權衡）

---

## 過濾與報告

**有用的過濾器**：
- 狀態：待處理（需處理）
- 日期範圍：最近30天
- 訂單：特定訂單查詢
- 原因：追蹤退貨原因

**退貨分析**：
- 每個商品的退貨率
- 最常見的退貨原因
- 平均處理時間（待處理 → 完成）
- 重新入庫費用收入

---

## 小技巧

- **設定明確的退貨政策** - 通報期限（30天）、條件、費用
- **及時處理申請** - 在24小時內回應待處理申請
- **仔細檢查** - 記錄狀況以避免爭議
- **追蹤退貨原因** - 使用數據改進產品/描述
- **盡可能自動化** - 載運商的webhook自動更新運送狀態
- **與顧客溝通** - 在每個狀態變更時郵件通知
- **公平套用重新入庫費用** - 一致性套用，瑕疵商品免除此費用
- **監控退貨詐騙** - 將退貨過多的顧客標記
- **改善包裝** - 減少因損壞導致的退貨
- **及時更新庫存** - 檢查後恢復庫存
- **從模式中學習** - 特定產品退貨率高可能表示品質問題
