---
title: 管理客户账户
---

客户账户允许商家跟踪客户信息、订单历史和偏好。在管理侧边栏中导航至 **客户 > 所有客户** 以管理客户账户。

![添加客户](/static/core/admin/img/help/managing-customer-accounts/add-customer.webp)

## 理解客户账户与客户资料

**客户账户** 是存储在用户模型中的登录凭证（电子邮件/密码）。**客户资料** 存储额外的客户信息，如电话号码、出生日期、偏好和分析数据。每个客户账户都有一个对应的资料，用于存储这些扩展数据。

在管理客户时，您正在处理与后台用户账户关联的客户资料。

## 查看所有客户

客户列表显示所有注册客户的关键指标：

| 列 | 描述 |
|--------|-------------|
| **用户** | 客户姓名和电子邮件地址 |
| **附属状态** | 客户是否也是附属合作伙伴 |
| **客户价值** | 客户总消费金额（颜色编码） |
| **客户分段** | RFM分段（冠军、忠诚、风险等） |
| **总订单数** | 完成的订单数量 |
| **上次订单天数** | 上次购买的近期性 |
| **VIP客户** | 如果客户被标记为VIP则显示徽章 |

### 筛选客户

使用筛选侧边栏缩小列表范围：

- **附属状态** — 是附属、非附属、附属待处理、激活、暂停、拒绝
- **仪表板布局** — 客户偏好的仪表板布局
- **订阅新闻通讯** — 客户是否选择加入新闻通讯
- **营销电子邮件** — 客户是否选择加入营销电子邮件
- **创建时间** — 按注册日期筛选

### 搜索客户

使用搜索栏通过以下方式查找客户：
- 用户名
- 电子邮件地址
- 名字
- 姓氏
- 电话号码

## 查看客户详情

点击客户姓名以查看其完整资料。客户详情页面显示：

![客户详情](/static/core/admin/img/help/managing-customer-accounts/customer-detail.webp)

### 客户信息部分

基本联系方式和账户状态：
- **用户** — 链接到底层用户账户
- **电话** — 客户电话号码
- **出生日期** — 用于年龄验证和生日促销

### 仪表板偏好

客户如何自定义其账户仪表板：
- **仪表板布局** — 网格、列表或紧凑视图
- **显示订单历史** — 是否在仪表板上显示订单历史
- **显示愿望清单** — 是否在仪表板上显示愿望清单
- **显示最近商品** — 是否显示最近查看的商品
- **显示推荐商品** — 是否显示产品推荐

### 通信偏好

客户选择加入各种通信的状况：
- **订阅新闻通讯** — 选择加入常规新闻通讯
- **营销电子邮件** — 选择加入促销电子邮件
- **订单通知** — 选择加入订单状态更新

### 客户分析

客户行为和价值的只读摘要：
- **客户分析摘要** — RFM评分、分段、终身价值
- **购买行为摘要** — 订单频率、平均订单价值、首选分类
- **参与度摘要** — 最后登录、电子邮件打开率、网站活动

这些分析字段是自动生成的，不能手动编辑。详情请参阅 [理解客户分析](customer-analytics.md)。

## 创建客户账户

商家可以手动为客户账户创建电话订单、店内自提或预注册批发客户。

1. 点击顶部右侧的 **+ 添加客户资料**
2. 填写必填和可选字段：

| 字段 | 必填 | 描述 |
|-------|----------|-------------|
| **用户** | 是 | 选择现有用户账户或创建新账户 |
| **电话** | 否 | 客户电话号码 |
| **出生日期** | 否 | 用于年龄验证和生日活动 |
| **订阅新闻通讯** | 否 | 将客户加入新闻通讯 |
| **营销电子邮件** | 否 | 将客户加入营销电子邮件 |

### 添加资料时创建新用户

如果客户尚未拥有用户账户：
1. 点击 **用户** 字段旁边的 **+** 图标
2. 输入客户的 **电子邮件地址**（这将成为其用户名）
3. 可选输入 **名字** 和 **姓氏**
4. 可选设置 **密码**
5. 如果未设置密码，请勾选 **发送密码重置电子邮件**
6. 保存用户账户
7. 完成客户资料字段
8. 点击 **保存**

### 欢迎电子邮件

创建客户账户后：
- 如果设置了密码，客户可以立即使用该密码登录
- 如果未设置密码，系统会发送密码重置电子邮件，客户可以自行设置密码
- 您可以通过 **营销 > 电子邮件活动** 手动触发欢迎电子邮件

## 编辑客户信息

要更新客户详细信息：
1. 导航至 **客户 > 所有客户**
2. 点击客户姓名
3. 修改要更新的字段
4. 点击 **保存**

### 您可以编辑的内容

**联系方式：**
- 名字（通过用户账户）
- 电子邮件地址（通过用户账户）
- 电话号码
- 出生日期

**偏好：**
- 新闻通讯订阅状态
- 营销电子邮件加入
- 订单通知偏好
- 仪表板布局和可见性设置

### 您无法编辑的内容

这些字段基于客户行为自动生成：
- 总消费金额/客户价值
- 订单数量
- 客户分段（冠军、忠诚、风险等）
- RFM评分
- 终身价值预测
- 最后订单日期
- 分析摘要

如果这些字段显示不正确，请检查底层订单数据或在 **客户 > 分析** → **重新计算指标** 中触发手动重新计算。

## 客户备注

添加关于客户的内部备注，以跟踪支持问题、VIP请求或后续任务。

### 添加备注

1. 打开客户资料
2. 滚动到 **客户备注** 部分（可能是单独的标签页）
3. 点击 **+ 添加备注**
4. 填写备注详情：

| 字段 | 描述 |
|-------|-------------|
| **备注类型** | 一般、支持问题、投诉、表扬、VIP服务、需要后续跟进、支付问题、发货问题 |
| **标题** | 备注的简短摘要 |
| **内容** | 详细备注内容 |
| **需要后续跟进** | 勾选是否需要操作 |
| **后续跟进日期** | 需要跟进的日期 |
| **已完成** | 勾选当后续跟进完成时 |

### 备注类型

| 类型 | 用例 |
|------|----------|
| **一般备注** | 任何关于客户的普通观察 |
| **支持问题** | 支持工单或问题的记录 |
| **投诉** | 用于跟踪和解决的客户投诉 |
| **表扬** | 关于客户或他们对您的反馈的积极反馈 |
| **VIP服务** | 为VIP客户请求的特殊处理 |
| **需要后续跟进** | 需要在特定日期完成的任务 |
| **支付问题** | 关于支付问题或争议的备注 |
| **发货问题** | 关于发货问题或特殊配送请求的备注 |

### 查看备注历史

所有备注按时间顺序显示在客户的资料中。每个备注显示：
- 创建日期和时间
- 创建者（员工姓名）
- 备注类型徽章
- 标题和内容
- 如果适用，后续跟进状态

### 内部备注与客户可见备注

所有客户备注默认为 **仅内部** — 客户永远看不到这些备注。它们仅用于商家团队的沟通。

如果需要与客户沟通，请使用 **营销 > 电子邮件活动** 中的电子邮件系统，或在特定订单上添加订单评论。

## 将访客转换为注册客户

当有人在未创建账户的情况下完成结账时，系统会自动生成访客客户。他们的用户名遵循 `guest_10374` 的模式，其中数字是唯一ID。

要将访客转换为注册客户：

1. 导航至 **客户 > 所有客户**
2. 通过订单电子邮件地址搜索访客
3. 点击访客客户资料
4. 点击 **用户** 链接以编辑底层用户账户
5. 将 **用户名** 从 `guest_10374` 更改为客户的实际电子邮件地址
6. 将 **电子邮件** 更改为匹配
7. 可选添加 **名字** 和 **姓氏**
8. 勾选 **发送密码重置电子邮件** 以便客户设置密码
9. 点击 **保存**

现在客户可以使用其电子邮件地址登录，并在其订单历史中看到之前的访客订单。

### 为什么要将访客转换为注册客户？

- 访客订单不计入客户分析或分段
- 访客无法跟踪订单或访问订单历史
- 转换访客可增加注册客户数量并提高分析准确性
- 注册客户更有可能进行重复购买

## 停用账户与删除账户

### 停用客户账户

停用会阻止登录同时保留所有数据：

1. 打开客户的资料
2. 点击 **用户** 链接以编辑用户账户
3. **取消勾选“激活”**
4. 点击 **保存**

**会发生什么：**
- 客户无法登录
- 订单历史保留
- 客户之后可以通过再次勾选“激活”重新启用
- 分析和指标保持不变

**使用停用的情况：**
- 由于支付争议暂时停用账户
- 阻止滥用客户
- 客户要求停止接收访问权限但不删除数据

### 删除客户账户

删除会移除账户并可能导致订单历史孤立：

1. 打开客户的资料
2. 滚动到底部并点击 **删除**
3. 确认删除

**会发生什么：**
- 客户账户被永久删除
- 客户资料被删除
- 订单历史可能孤立（订单存在但未与客户关联）
- 无法撤销

**使用删除的情况：**
- GDPR/CCPA数据删除请求（先导出数据）
- 应该从未存在过的测试账户
- 由于错误创建的重复账户

### GDPR合规性

在响应GDPR请求删除客户账户之前：

1. 导航至 **客户 > 所有客户**
2. 选择客户
3. 使用 **导出数据** 操作生成完整数据导出
4. 如果客户请求了，将导出发送给客户
5. 然后继续删除

导出包括：客户资料、订单历史、地址、备注和分析数据。

## 小贴士

- **使用筛选器识别高价值客户** — 按客户价值筛选以找到您的冠军和VIP
- **定期查看客户备注** — 每周至少检查一次开放的后续任务
- **不要手动编辑分析** — 让系统自动计算RFM评分和分段
- **主动将访客转换为注册客户** — 在访客完成第二次购买后，主动联系并提供创建正式账户的机会
- **使用停用而非删除** — 停用保留数据，如有必要可以撤销
- **在支持电话中添加备注** — 记录支持互动，以便其他团队成员了解背景
- **设置后续跟进日期** — 使用备注中的后续任务系统确保没有遗漏
- **尊重沟通偏好** — 不要向已选择退出的客户发送营销电子邮件
