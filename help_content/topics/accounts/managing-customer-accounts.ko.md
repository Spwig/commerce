---
title: 고객 계정 관리
---

고객 계정은 판매자가 고객 정보, 주문 내역 및 선호도를 추적할 수 있도록 합니다. 관리자 사이드바에서 **고객 > 모든 고객**으로 이동하여 고객 계정을 관리할 수 있습니다.

![고객 추가](/static/core/admin/img/help/managing-customer-accounts/add-customer.webp)

## 고객 계정과 고객 프로필 이해

**고객 계정**은 사용자 모델에 저장된 로그인 자격 증명(이메일/비밀번호)입니다. **고객 프로필**은 전화번호, 생년월일, 선호도 및 분석과 같은 추가 고객 정보를 저장합니다. 모든 고객 계정에는 이 확장 데이터를 저장하는 프로필이 있습니다.

관리자에서 고객을 관리할 때는 고객 프로필을 다루고 있으며, 이는 뒷면에서 사용자 계정과 연결됩니다.

## 모든 고객 보기

고객 목록은 주요 지표와 함께 등록된 모든 고객을 보여줍니다:

| 열 | 설명 |
|-----|------|
| **사용자** | 고객 이름 및 이메일 주소 |
| **소속 상태** | 고객이 또한 소속 파트너인지 여부 |
| **고객 가치** | 고객이 총 지출 금액 (색상 코드로 표시) |
| **고객 세그먼트** | RFM 세그먼트 (챔피언, 충성, 위험 등) |
| **총 주문 수** | 완료된 주문 수 |
| **최근 주문 일수** | 마지막 구매의 신선도 |
| **VIP 고객** | 고객이 VIP로 표시된 경우 배지 |

### 고객 필터

필터 사이드바를 사용하여 목록을 좁힐 수 있습니다:

- **소속 상태** — 소속, 비소속, 소속 대기, 활성, 일시 중지, 거부
- **대시보드 레이아웃** — 고객의 선호 대시보드 레이아웃
- **뉴스레터 구독** — 고객이 뉴스레터에 가입했는지 여부
- **마케팅 이메일** — 고객이 마케팅 이메일에 가입했는지 여부
- **생성일** — 등록 날짜로 필터

### 고객 검색

검색 바를 사용하여 고객을 다음과 같이 검색할 수 있습니다:

- 사용자 이름
- 이메일 주소
- 이름
- 성
- 전화 번호

## 고객 세부 정보 보기

고객의 이름을 클릭하여 그들의 전체 프로필을 볼 수 있습니다. 고객 세부 정보 페이지는 다음과 같은 정보를 보여줍니다:

![고객 세부 정보](/static/core/admin/img/help/managing-customer-accounts/customer-detail.webp)

### 고객 정보 섹션

기본 연락처 세부 정보 및 계정 상태:
- **사용자** — 기저 사용자 계정으로의 링크
- **전화** — 고객의 전화 번호
- **생년월일** — 나이 확인 및 생일 캠페인을 위해

### 대시보드 선호도

고객이 계정 대시보드를 어떻게 커스터마이징했는지:
- **대시보드 레이아웃** — 격자, 목록 또는 압축 뷰
- **주문 내역 표시** — 대시보드에 주문 내역이 표시되는지 여부
- **위시리스트 표시** — 대시보드에 위시리스트가 표시되는지 여부
- **최근 본 상품 표시** — 최근 본 상품이 표시되는지 여부
- **상품 추천 표시** — 상품 추천이 표시되는지 여부

### 커뮤니케이션 선호도

고객의 다양한 커뮤니케이션에 대한 가입 상태:
- **뉴스레터 구독** — 일반 뉴스레터에 가입함
- **마케팅 이메일** — 프로모션 이메일에 가입함
- **주문 알림** — 주문 상태 업데이트에 가입함

### 고객 분석

고객 행동 및 가치에 대한 읽기 전용 요약:
- **고객 분석 요약** — RFM 점수, 세그먼트, 수명 가치
- **구매 행동 요약** — 주문 빈도, 평균 주문 가치, 선호 카테고리
- **참여 요약** — 마지막 로그인, 이메일 열람률, 사이트 활동

이 분석 필드는 자동으로 계산되며 수동으로 편집할 수 없습니다. 자세한 내용은 [고객 분석 이해](customer-analytics.md)를 참조하십시오.

## 고객 계정 생성

판매자는 전화 주문, 매장 픽업, 또는 대량 고객을 사전 등록하기 위해 고객 계정을 수동으로 생성할 수 있습니다.

1. 상단 오른쪽에 **+ 고객 프로필 추가**를 클릭합니다
2. 필요한 필드 및 선택 필드를 입력합니다:

| 필드 | 필수 | 설명 |
|------|------|------|
| **사용자** | 예 | 기존 사용자 계정을 선택하거나 새 계정을 생성합니다 |
| **전화** | 아니오 | 고객의 전화 번호 |
| **생년월일** | 아니오 | 나이 확인 및 생일 캠페인을 위해 |
| **뉴스레터 구독** | 아니오 | 고객을 뉴스레터에 가입합니다 |
| **마케팅 이메일** | 아니오 | 고객을 마케팅 이메일에 가입합니다 |

### 프로필 추가 시 새 사용자 생성

고객이 아직 사용자 계정이 없는 경우:
1. 사용자 필드 옆의 **+** 아이콘을 클릭합니다
2. 고객의 **이메일 주소**를 입력합니다 (이것은 그들의 사용자 이름이 됩니다)
3. 필요에 따라 **이름** 및 **성**을 입력합니다
4. 필요에 따라 **비밀번호**를 설정합니다
5. 비밀번호를 설정하지 않았다면 **비밀번호 재설정 이메일 보내기**를 체크합니다
6. 사용자 계정을 저장합니다
7. 고객 프로필 필드를 완료합니다
8. **저장**을 클릭합니다

## 고객 정보 편집

고객 세부 정보를 업데이트하려면:
1. **고객 > 모든 고객**으로 이동합니다
2. 고객의 이름을 클릭합니다
3. 업데이트하고자 하는 필드를 수정합니다
4. **저장**을 클릭합니다

### 편집 가능한 항목

**연락처 세부 정보:**
- 이름 (사용자 계정을 통해)
- 이메일 주소 (사용자 계정을 통해)
- 전화 번호
- 생년월일

**선호도:**
- 뉴스레터 구독 상태
- 마케팅 이메일 가입
- 주문 알림 선호도
- 대시보드 레이아웃 및 가시성 설정

### 편집 불가 항목

이 필드는 고객 행동을 기반으로 자동으로 계산됩니다:
- 총 지출 / 고객 가치
- 주문 수
- 고객 세그먼트 (챔피언, 충성, 위험 등)
- RFM 점수
- 수명 가치 예측
- 마지막 주문 날짜
- 분석 요약

이 필드가 잘못된 것으로 보이는 경우, 기저 주문 데이터를 확인하거나 **고객 > 분석** → **지표 재계산**에서 수동으로 재계산을 트리거합니다.

## 고객 메모

고객에 대한 내부 메모를 추가하여 지원 문제, VIP 요청 또는 후속 작업을 추적할 수 있습니다.

### 메모 추가

1. 고객 프로필을 엽니다
2. **고객 메모** 섹션으로 스크롤합니다 (개별 탭일 수 있음)
3. **+ 메모 추가**를 클릭합니다
4. 메모 세부 정보를 입력합니다:

| 필드 | 설명 |
|------|------|
| **메모 유형** | 일반, 지원 문제, 불만, 칭찬, VIP 서비스, 후속 작업 필요, 결제 문제, 배송 문제 |
| **제목** | 메모의 짧은 요약 |
| **내용** | 상세한 메모 내용 |
| **후속 작업 필요** | 이 작업이 필요한 경우 체크 |
| **후속 작업 날짜** | 후속 작업을 마무리할 날짜 |
| **완료됨** | 후속 작업이 완료된 경우 체크 |

### 메모 유형

| 유형 | 사용 사례 |
|------|----------|
| **일반 메모** | 고객에 대한 일반적인 관찰 |
| **지원 문제** | 지원 티켓 또는 문제의 기록 |
| **불만** | 추적 및 해결을 위한 고객 불만 |
| **칭찬** | 고객 또는 그들의 피드백에 대한 긍정적인 피드백 |
| **VIP 서비스** | VIP 고객의 특별한 처리 요청 |
| **후속 작업 필요** | 특정 날짜까지 처리해야 하는 작업 |
| **결제 문제** | 결제 문제 또는 분쟁에 대한 메모 |
| **배송 문제** | 배송 문제 또는 특별한 배송 요청에 대한 메모 |

### 메모 기록 보기

모든 메모는 고객 프로필에서 시계열 순서로 나타납니다. 각 메모는 다음과 같은 정보를 보여줍니다:
- 생성 날짜 및 시간
- 생성자 (직원 이름)
- 메모 유형 배지
- 제목 및 내용
- 필요한 경우 후속 작업 상태

### 내부 메모 vs 고객에게 보이는 메모

모든 고객 메모는 기본적으로 **내부용**입니다 — 고객은 이러한 메모를 절대 보지 못합니다. 이는 판매자 팀 간의 커뮤니케이션을 위해만 사용됩니다.

고객과의 커뮤니케이션이 필요한 경우, **마케팅 > 이메일 캠페인**에서 이메일 시스템을 사용하거나 해당 주문에 주문 주석을 추가하세요.

## 방문자 고객을 등록 고객으로 전환

체크아웃 시 계정을 생성하지 않은 경우 자동으로 생성된 방문자 고객은 `guest_10374` 패턴을 따르며, 숫자는 고유 ID입니다.

방문자를 등록 고객으로 전환하려면:
1. **고객 > 모든 고객**으로 이동합니다
2. 주문 이메일 주소로 방문자를 검색합니다
3. 방문자 고객 프로필을 클릭합니다
4. **사용자** 링크를 클릭하여 기저 사용자 계정을 편집합니다
5. **사용자 이름**을 `guest_10374`에서 고객의 실제 이메일 주소로 변경합니다
6. **이메일**을 일치시킵니다
7. 필요에 따라 **이름** 및 **성**을 추가합니다
8. 고객이 비밀번호를 설정할 수 있도록 **비밀번호 재설정 이메일 보내기**를 체크합니다
9. **저장**을 클릭합니다

이제 고객은 이메일 주소로 로그인할 수 있으며, 이전 방문자 주문을 주문 내역에서 볼 수 있습니다.

### 왜 방문자 고객을 전환해야 하는가?

- 방문자 주문은 고객 분석 또는 세그먼트에 포함되지 않습니다
- 방문자는 주문을 추적하거나 주문 내역에 접근할 수 없습니다
- 방문자를 전환하면 등록 고객 수가 증가하고 분석 정확도가 향상됩니다
- 등록 고객은 반복 구매를 할 가능성이 더 높습니다

## 계정 비활성화 vs 계정 삭제

### 고객 계정 비활성화

비활성화는 데이터를 유지하면서 로그인을 방지합니다:

1. 고객 프로필을 엽니다
2. **사용자** 링크를 클릭하여 사용자 계정을 편집합니다
3. **활성**을 선택 해제합니다
4. **저장**을 클릭합니다

**발생하는 일:**
- 고객이 로그인할 수 없음
- 주문 내역은 유지됨
- 이후 다시 활성화할 수 있음
- 분석 및 지표는 그대로 유지됨

**비활성화를 사용하는 경우:**
- 결제 분쟁으로 인한 임시 중지
- 악의적인 고객 차단
- 고객이 접근을 중단하되 데이터를 삭제하지 않는 요청

### 고객 계정 삭제

삭제는 계정을 제거하고 주문 내역을 고아화할 수 있습니다:

1. 고객 프로필을 엽니다
2. 하단으로 스크롤하여 **삭제**를 클릭합니다
3. 삭제를 확인합니다

**발생하는 일:**
- 고객 계정이 영구적으로 제거됨
- 고객 프로필이 삭제됨
- 주문 내역이 고아화될 수 있음 (주문이 존재하지만 고객과 연결되지 않음)
- 되돌릴 수 없음

**삭제를 사용하는 경우:**
- GDPR/CCPA 데이터 삭제 요청 (먼저 데이터 내보내기)
- 존재해서는 안 되는 테스트 계정
- 실수로 생성된 중복 계정

### GDPR 준수

GDPR 요청에 따라 고객 계정을 삭제하기 전에:

1. **고객 > 모든 고객**으로 이동합니다
2. 고객을 선택합니다
3. **데이터 내보내기** 작업을 사용하여 완전한 데이터 내보내기를 생성합니다
4. 고객이 요청한 경우 내보내기를 고객에게 전송합니다
5. 그 후 삭제를 진행합니다

내보내기에는 포함된 항목: 고객 프로필, 주문 내역, 주소, 메모 및 분석 데이터

## 팁

- **고객 필터를 사용하여 고가치 고객을 식별하세요** — 고객 가치로 필터하여 챔피언 및 VIP를 찾습니다
- **고객 메모를 정기적으로 확인하세요** — 최소 주간으로 열려 있는 후속 작업을 확인합니다
- **분석을 수동으로 편집하지 마세요** — 시스템이 RFM 점수 및 세그먼트를 자동으로 계산하도록 합니다
- **방문자를 적극적으로 전환하세요** — 방문자가 두 번째 구매를 마친 후, 적절한 계정을 생성하는 것을 제안하세요
- **삭제 대신 비활성화를 사용하세요** — 데이터를 유지하고 필요 시 다시 활성화할 수 있습니다
