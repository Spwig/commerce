---
title: 顧客アカウントの管理
---

顧客アカウントは、小売業者が顧客情報を、注文履歴、および好みを追跡するためのものです。管理画面のサイドバーで **顧客 > すべての顧客** にアクセスして顧客アカウントを管理してください。

![顧客の追加](/static/core/admin/img/help/managing-customer-accounts/add-customer.webp)

## 顧客アカウントと顧客プロファイルの理解

**顧客アカウント** は、Userモデルに保存されているログイン資格情報（メール/パスワード）です。**顧客プロファイル** は、電話番号、生年月日、好み、分析などの追加の顧客情報を保存します。すべての顧客アカウントには、この拡張データを保存するプロファイルが対応しています。

管理画面で顧客を管理する際は、バックエンドでUserアカウントにリンクしている顧客プロファイルを操作しています。

## すべての顧客を表示する

顧客リストは、登録済みの顧客と主要なメトリクスを表示します：

| カラム | 説明 |
|--------|-------------|
| **ユーザー** | 顧客名とメールアドレス |
| **アフィリエイトステータス** | 顧客がアフィリエイトパートナーでもあるかどうか |
| **顧客価値** | 顧客がこれまでに支払った総額（カラーコード付き） |
| **顧客セグメント** | RFMセグメント（チャンピオン、忠実、リスクありなど） |
| **注文数** | 完了した注文の数 |
| **最終注文日からの日数** | 最後の購入の新しさ |
| **VIP顧客** | VIPとしてマークされている顧客のバッジ |

### 顧客のフィルタリング

フィルターサイドバーを使用してリストを絞り込みます：

- **アフィリエイトステータス** — アフィリエイト、非アフィリエイト、アフィリエイト保留、アクティブ、停止、拒否
- **ダッシュボードレイアウト** — 顧客が好むダッシュボードレイアウト
- **ニュースレター登録** — 顧客がニュースレターに登録しているかどうか
- **マーケティングメール** — 顧客がマーケティングメールに登録しているかどうか
- **作成日時** — 登録日でフィルタリング

### 顧客の検索

検索バーを使用して顧客を以下で検索できます：
- ユーザー名
- メールアドレス
- 名前
- 苗字
- 電話番号

## 顧客の詳細を表示する

顧客の名前をクリックして、そのフルプロファイルを表示します。顧客詳細ページには以下が表示されます：

![顧客詳細](/static/core/admin/img/help/managing-customer-accounts/customer-detail.webp)

### 顧客情報セクション

基本的な連絡先情報とアカウントステータス：
- **ユーザー** — 下層のUserアカウントへのリンク
- **電話** — 顧客の電話番号
- **生年月日** — 年齢認証と誕生日キャンペーン用

### ダッシュボードの設定

顧客がアカウントダッシュボードをどのようにカスタマイズしたか：
- **ダッシュボードレイアウト** — グリッド、リスト、またはコンパクト表示
- **注文履歴を表示** — ダッシュボードに注文履歴が表示されるかどうか
- **お気に入りリストを表示** — ダッシュボードにお気に入りリストが表示されるかどうか
- **最近見た商品を表示** — 最近見た商品が表示されるかどうか
- **商品の推奨を表示** — 商品の推奨が表示されるかどうか

### コミュニケーションの設定

顧客がさまざまな通信に登録しているステータス：
- **ニュースレター登録** — 一般ニュースレターに登録済み
- **マーケティングメール** — プロモーションメールに登録済み
- **注文通知** — 注文ステータスの更新に登録済み

### 顧客分析

顧客の行動と価値の読み取り専用サマリ：
- **顧客分析サマリ** — RFMスコア、セグメント、ライフタイムバリュー
- **購入行動サマリ** — 注文頻度、平均注文価格、好むカテゴリ
- **エンゲージメントサマリ** — 最後のログイン、メール開封率、サイト活動

これらの分析フィールドは自動的に計算され、手動で編集することはできません。詳細については [顧客分析の理解](customer-analytics.md) をご覧ください。

## 顧客アカウントの作成

小売業者は、電話注文、店舗でのピックアップ、または小売向けの事前登録顧客のために顧客アカウントを手動で作成できます。

1. トップ右の **+ 顧客プロファイルの追加** をクリックします
2. 必要なフィールドと任意のフィールドを入力します：

| フィールド | 必須 | 説明 |
|-------|----------|-------------|
| **ユーザー** | はい | 既存のUserアカウントを選択するか、新しいアカウントを作成します |
| **電話** | いいえ | 顧客の電話番号 |
| **生年月日** | いいえ | 年齢認証と誕生日キャンペーン用 |
| **ニュースレター登録** | いいえ | 顧客をニュースレターに登録します |
| **マーケティングメール** | いいえ | 顧客をマーケティングメールに登録します |

### プロファイルの追加時に新しいUserを作成する

顧客がまだUserアカウントを持っていない場合：
1. Userフィールドの隣の **+** アイコンをクリックします
2. 顧客の **メールアドレス** を入力します（これがユーザー名になります）
3. 必要に応じて **名前** と **苗字** を入力します
4. 必要に応じて **パスワード** を設定します
5. パスワードを設定しなかった場合は **パスワードリセットメールを送信** にチェックを入れます
6. Userアカウントを保存します
7. 顧客プロファイルのフィールドを完了します
8. **保存** をクリックします

### ウェルカムメール

顧客アカウントを作成した後：
- パスワードを設定した場合、顧客はそのパスワードで即座にログインできます
- パスワードを設定しなかった場合、システムは顧客が独自のパスワードを設定できるようにパスワードリセットメールを送信します
- **マーケティング > メールキャンペーン** から、手動でウェルカムメールをトリガーできます

## 顧客情報の編集

顧客の詳細を更新するには：
1. **顧客 > すべての顧客** にアクセスします
2. 顧客の名前をクリックします
3. 更新したいフィールドを編集します
4. **保存** をクリックします

### 編集可能な内容

**連絡先情報：**
- 名前（Userアカウントを通して）
- メールアドレス（Userアカウントを通して）
- 電話番号
- 生年月日

**設定：**
- ニュースレターの登録ステータス
- マーケティングメールの登録
- 注文通知設定
- ダッシュボードレイアウトと可視性設定

### 編集できない内容

顧客の行動に基づいて自動的に計算されるこれらのフィールド：
- 総支払額 / 顧客価値
- 注文数
- 顧客セグメント（チャンピオン、忠実、リスクありなど）
- RFMスコア
- ライフタイムバリューピレディクション
- 最終注文日
- 分析サマリ

これらのフィールドが正しく表示されていない場合は、下層の注文データを確認するか、**顧客 > 分析** → **メトリクスの再計算** で手動で再計算をトリガーしてください。

## 顧客ノート

顧客に関する内部ノートを追加して、サポート問題、VIPリクエスト、フォローアップタスクを追跡してください。

### ノートの追加

1. 顧客のプロファイルを開きます
2. **顧客ノート** セクションにスクロールします（別のタブである場合もあります）
3. **+ ノートの追加** をクリックします
4. ノートの詳細を入力します：

| フィールド | 説明 |
|-------|-------------|
| **ノートタイプ** | 一般、サポート問題、苦情、称賛、VIPサービス、フォローアップが必要、支払い問題、配送問題 |
| **タイトル** | ノートの短い要約 |
| **内容** | 詳細なノート内容 |
| **フォローアップが必要** | これに行動が必要かどうかをチェック |
| **フォローアップ日** | フォローアップの日付 |
| **完了** | フォローアップが完了したときにチェック |

### ノートタイプ

| タイプ | 使用ケース |
|------|----------|
| **一般ノート** | 顧客に関する任意の一般観察 |
| **サポート問題** | サポートチケットまたは問題の記録 |
| **苦情** | 顧客の苦情を追跡および解決するため |
| **称賛** | 顧客または顧客のフィードバックに関するポジティブなフィードバック |
| **VIPサービス** | VIP顧客の特別な処理リクエスト |
| **フォローアップが必要** | 特定の日付までに行動が必要なタスク |
| **支払い問題** | 支払い問題または紛争に関するノート |
| **配送問題** | 配送問題または特別な配達リクエストに関するノート |

### ノート履歴の表示

すべてのノートは、顧客のプロファイル上に時系列順で表示されます。各ノートには以下が表示されます：
- 作成日時
- 作成者（スタッフ名）
- ノートタイプバッジ
- タイトルと内容
- 必要に応じてフォローアップステータス

### 内部ノートと顧客が見えるノート

すべての顧客ノートは、**内部のみ** としてデフォルトで設定されています — 顧客はこれらのノートを決して見ません。これは、小売業チーム間のコミュニケーションのためにのみ使用されます。

顧客と通信する必要がある場合は、**マーケティング > メールキャンペーン** でメールシステムを使用するか、特定の注文に注文コメントを追加してください。

## ゲストを登録顧客に変換する

チェックアウト時にアカウントを作成せずに購入したゲスト顧客は自動的に作成されます。ゲストのユーザー名は `guest_10374` のような形式で、数字は一意のIDです。

ゲストを登録顧客に変換するには：
1. **顧客 > すべての顧客** にアクセスします
2. 注文のメールアドレスでゲストを検索します
3. ゲスト顧客プロファイルをクリックします
4. **ユーザー** リンクをクリックして下層のUserアカウントを編集します
5. ユーザー名を `guest_10374` から顧客の実際のメールアドレスに変更します
6. そのメールアドレスに一致するようにメールアドレスを変更します
7. 必要に応じて **名前** と **苗字** を追加します
8. 顧客がパスワードを設定できるように **パスワードリセットメールを送信** にチェックを入れます
9. **保存** をクリックします

顧客は今やメールアドレスでログインでき、過去のゲスト注文を注文履歴で見ることができます。

### ゲスト顧客を変換する理由

- ゲスト注文は顧客分析またはセグメントにカウントされません
- ゲストは注文履歴にアクセスしたり、注文を追跡したりできません
- ゲストを変換すると、登録顧客数が増加し、分析の正確性が向上します
- 登録顧客は繰り返し購入する可能性が高くなります

## アカウントの無効化と削除

### 顧客アカウントの無効化

無効化はログインを防止しつつすべてのデータを保持します：

1. 顧客のプロファイルを開きます
2. **ユーザー** リンクをクリックしてUserアカウントを編集します
3. **「アクティブ」を外す**
4. **保存** をクリックします

**発生する事象：**
- 顧客はログインできません
- 注文履歴は保持されます
- 顧客は後で「アクティブ」を再度チェックして再アクティブ化できます
- 分析とメトリクスは保持されます

**無効化を使用する理由：**
- 支払い紛争による一時的なアカウントの停止
- 悪質な顧客のブロック
- 顧客がアクセスを停止したいが、データを削除したくない場合

### 顧客アカウントの削除

削除はアカウントを永久に削除し、注文履歴を孤立させる可能性があります：

1. 顧客のプロファイルを開きます
2. 最下部にスクロールして **削除** をクリックします
3. 削除を確認します

**発生する事象：**
- 顧客アカウントは永久に削除されます
- 顧客プロファイルが削除されます
- 注文履歴が孤立する可能性があります（注文は存在しますが、顧客にリンクされていません）
- これを行うことは取り消せません

**削除を使用する理由：**
- GDPR/CCPAデータ削除リクエスト（削除前にデータをエクスポート）
- 存在すべきでないテストアカウント
- 一時的に誤って作成された複製アカウント

### GDPR準拠

GDPRの要求に応じて顧客アカウントを削除する前に：

1. **顧客 > すべての顧客** にアクセスします
2. 顧客を選択します
3. **データのエクスポート** アクションを使用して完全なデータエクスポートを生成します
4. 顧客が要求した場合はエクスポートを顧客に送信します
5. その後削除を進めてください

エクスポートには以下が含まれます：顧客プロファイル、注文履歴、住所、ノート、および分析データ。

## ヒント

- **フィルタを使用して高価値顧客を特定する** — 顧客価値でフィルタしてチャンピオンやVIPを特定してください
- **顧客ノートを定期的に確認する** — 週に少なくとも1回、開いているフォローアップタスクを確認してください
- **分析を手動で編集しないでください** — RFMスコアとセグメントをシステムが自動的に計算させてください
- **ゲストを積極的に変換する** — ゲストが2回目の購入をした後、適切なアカウントを作成する提案を出してください
- **削除の代わりに無効化を使用する** — 無効化はデータを保持し、必要に応じて逆転させることができます
- **サポートコール中にノートを追加する** — 他のチームメンバーに文脈を提供するためにサポートインタラクションを文書化してください
- **フォローアップ日を設定する** — ノートのフォローアップタスクシステムを使用して、何も漏れることのないようにしてください
- **コミュニケーションの設定を尊重する** — ニュースレターに登録していない顧客にマーケティングメールを送らないでください
