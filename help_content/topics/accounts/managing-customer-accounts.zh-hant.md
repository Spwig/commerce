---
title: 管理客戶帳號
---

客戶帳號讓商家可以追蹤客戶資訊、訂購歷史和偏好設定。在管理員側邊欄中導航至 **Customers > All Customers** 來管理客戶帳號。

![Add Customer](/static/core/admin/img/help/managing-customer-accounts/add-customer.webp)

## 了解客戶帳號與客戶個人資料的區別

**客戶帳號** 是儲存在 User 模型中的登入憑證（電子郵件/密碼）。**客戶個人資料** 儲存額外的客戶資訊，如電話號碼、生日、偏好設定和分析數據。每個客戶帳號都有一個對應的個人資料，用來儲存這些擴展資料。

當您在管理員中管理客戶時，您實際上是在操作與後端 User 帳號連結的客戶個人資料。

## 查看所有客戶

客戶列表顯示所有已註冊的客戶及其關鍵指標：

| 欄位 | 描述 |
|--------|-------------|
| **用戶** | 客戶姓名和電子郵件地址 |
| **合作夥伴狀態** | 客戶是否也是合作夥伴 |
| **客戶價值** | 客戶總消費金額（彩色標示） |
| **客戶分群** | RFM 分群（冠軍、忠實、有風險等） |
| **總訂單數** | 已完成的訂單數量 |
| **上次訂購天數** | 上次購買的近期性 |
| **VIP 客戶** | 若客戶被標記為 VIP，會顯示徽章 |

### 篩選客戶

使用篩選側邊欄來縮小列表範圍：

- **合作夥伴狀態** — 是合作夥伴、不是合作夥伴、待審核合作夥伴、已啟用、已停用、已拒絕
- **儀表板佈局** — 客戶偏好的儀表板佈局
- **訂閱電子報** — 客戶是否選擇訂閱電子報
- **營銷郵件** — 客戶是否選擇接收營銷郵件
- **創建時間** — 依註冊日期篩選

### 搜尋客戶

使用搜尋欄按以下方式查找客戶：

- 使用者名稱
- 電子郵件地址
- 姓氏
- 名字
- 電話號碼

## 查看客戶詳情

點擊客戶姓名以查看其完整個人資料。客戶詳情頁面顯示：

![Customer Detail](/static/core/admin/img/help/managing-customer-accounts/customer-detail.webp)

### 客戶資訊區

基本聯絡資訊和帳號狀態：
- **用戶** — 連結至底層的 User 帳號
- **電話** — 客戶的電話號碼
- **生日** — 用於年齡驗證和生日促銷

### 儀表板偏好

客戶如何自訂其帳號儀表板：
- **儀表板佈局** — 網格、清單或緊湊視圖
- **顯示訂購歷史** — 是否在儀表板上顯示訂購歷史
- **顯示願望清單** — 是否在儀表板上顯示願望清單
- **顯示最近商品** — 是否顯示最近瀏覽過的商品
- **顯示推薦商品** — 是否顯示商品推薦

### 通訊偏好

客戶對各種通訊的選擇狀態：
- **訂閱電子報** — 選擇接收一般電子報
- **營銷郵件** — 選擇接收促銷郵件
- **訂單通知** — 選擇接收訂單狀態更新

### 客戶分析

客戶行為和價值的只讀摘要：
- **客戶分析摘要** — RFM 分數、分群、終身價值
- **購買行為摘要** — 訂購頻率、平均訂購價值、偏好的類別
- **參與度摘要** — 最後登入時間、電子郵件開啟率、網站活動

這些分析欄位是自動計算的，無法手動編輯。詳見 [了解客戶分析](customer-analytics.md)。

## 建立客戶帳號

商家可以手動為電話訂購、門市自取或預註冊批發客戶建立客戶帳號。

1. 點擊頂部右側的 **+ Add Customer Profile**
2. 填寫必填和選填欄位：

| 欄位 | 必填 | 描述 |
|-------|----------|-------------|
| **用戶** | 是 | 選擇現有的 User 帳號或建立新帳號 |
| **電話** | 否 | 客戶的電話號碼 |
| **生日** | 否 | 用於年齡驗證和生日活動 |
| **訂閱電子報** | 否 | 讓客戶訂閱電子報 |
| **營銷郵件** | 否 | 讓客戶接收營銷郵件 |

### 在新增個人資料時建立新用戶

如果客戶還沒有 User 帳號：
1. 點擊用戶欄位旁的 **+** 圖示
2. 輸入客戶的 **電子郵件地址**（這會成為他們的使用者名稱）
3. 可選輸入 **名字** 和 **姓氏**
4. 可選設定 **密碼**
5. 如果未設定密碼，勾選 **Send password reset email**
6. 儲存 User 帳號
7. 完成客戶個人資料欄位
8. 點擊 **Save**

### 歡迎郵件

建立客戶帳號後：
- 如果您設定了密碼，客戶可以立即使用該密碼登入
- 如果未設定密碼，系統會發送密碼重設郵件，讓客戶自行設定密碼
- 您可以透過電子郵件系統在 **Marketing > Email Campaigns** 手動觸發歡迎郵件

## 編輯客戶資訊

要更新客戶資訊：
1. 導航至 **Customers > All Customers**
2. 點擊客戶姓名
3. 修改您要更新的欄位
4. 點擊 **Save**

### 可編輯的內容

**聯絡資訊：**
- 姓名（透過 User 帳號）
- 電子郵件地址（透過 User 帳號）
- 電話號碼
- 生日

**偏好設定：**
- 電子報訂閱狀態
- 營銷郵件選擇
- 訂單通知偏好
- 儀表板佈局和可見性設定

### 不可編輯的內容

這些欄位是根據客戶行為自動計算的：
- 總消費金額 / 客戶價值
- 訂單數量
- 客戶分群（冠軍、忠實、有風險等）
- RFM 分數
- 終身價值預測
- 最後訂購日期
- 分析摘要

如果這些欄位顯示不正確，請檢查底層的訂單資料，或在 **Customers > Analytics** → **Recalculate Metrics** 觸發手動重新計算。

## 客戶備註

添加內部備註以追蹤支援問題、VIP 請求或後續任務。

### 添加備註

1. 打開客戶個人資料
2. 滾動到 **Customer Notes** 區域（可能為獨立標籤）
3. 點擊 **+ Add Note**
4. 填寫備註細節：

| 欄位 | 描述 |
|-------|-------------|
| **備註類型** | 一般、支援問題、投訴、讚譽、VIP 服務、需要後續、付款問題、運送問題 |
| **標題** | 備註的簡短摘要 |
| **內容** | 詳細備註內容 |
| **需要後續** | 勾選是否需要處理 |
| **後續日期** | 需要後續的日期 |
| **完成** | 勾選後續完成時 |

### 備註類型

| 類型 | 使用情境 |
|------|----------|
| **一般備註** | 關於客戶的任何一般觀察 |
| **支援問題** | 支援票務或問題的記錄 |
| **投訴** | 用於追蹤和解決的客戶投訴 |
| **讚譽** | 關於客戶或他們對您的正面反饋 |
| **VIP 服務** | VIP 客戶的特殊處理請求 |
| **需要後續** | 需要在特定日期處理的任務 |
| **付款問題** | 關於付款問題或爭議的備註 |
| **運送問題** | 關於運送問題或特殊送貨請求的備註 |

### 查看備註歷史

所有備註會按時間順序顯示在客戶個人資料中。每個備註顯示：
- 建立日期和時間
- 建立者（員工姓名）
- 備註類型徽章
- 標題和內容
- 如果適用，後續狀態

### 內部備註與客戶可見備註

所有客戶備註預設為 **僅內部** — 客戶從來不會看到這些備註。它們僅供商家團隊溝通使用。

如果需要與客戶溝通，請使用 **Marketing > Email Campaigns** 的電子郵件系統，或在特定訂單上添加訂單備註。

## 將訪客轉換為註冊客戶

當某人完成結帳但未建立帳號時，會自動創建訪客客戶。其使用者名稱遵循 `guest_10374` 的格式，其中數字是唯一 ID。

要將訪客轉換為註冊客戶：

1. 導航至 **Customers > All Customers**
2. 透過他們的訂單電子郵件地址搜尋訪客
3. 點擊訪客客戶個人資料
4. 點擊 **User** 連結以編輯底層的 User 帳號
5. 將 **使用者名稱** 從 `guest_10374` 改為客戶的真實電子郵件地址
6. 將 **電子郵件** 更改為匹配
7. 可選添加 **名字** 和 **姓氏**
8. 勾選 **Send password reset email** 以便客戶設定密碼
9. 點擊 **Save**

客戶現在可以使用電子郵件地址登入，並會在訂購歷史中看到他們之前的訪客訂單。

### 為什麼要轉換訪客客戶？

- 訪客訂單不計入客戶分析或分群
- 訪客無法追蹤訂單或查看訂購歷史
- 轉換訪客可增加註冊客戶數量並提高分析準確性
- 註冊客戶更有可能進行重複購買

## 停用帳號 vs 刪除帳號

### 停用客戶帳號

停用會阻止登入同時保留所有資料：

1. 打開客戶個人資料
2. 點擊 **User** 連結以編輯 User 帳號
3. **取消勾選 "Active"**
4. 點擊 **Save**

**發生什麼：**
- 客戶無法登入
- 訂購歷史被保留
- 後續可以透過再次勾選 "Active" 來重新啟用客戶
- 分析和指標保持完整

**使用停用的情況：**
- 因付款爭議暫時停用帳號
- 阻止濫用的客戶
- 客戶要求停止接收存取權但不刪除資料

### 刪除客戶帳號

刪除會移除帳號並可能導致訂購歷史孤立：

1. 打開客戶個人資料
2. 滾動到底部並點擊 **Delete**
3. 確認刪除

**發生什麼：**
- 客戶帳號被永久刪除
- 客戶個人資料被刪除
- 訂購歷史可能孤立（訂單存在但未連結到客戶）
- 無法撤銷

**使用刪除的情況：**
- GDPR/CCPA 資料刪除請求（先匯出資料）
- 應該從未存在過的測試帳號
- 錯誤建立的重複帳號

### GDPR 合規性

在響應 GDPR 請求刪除客戶帳號之前：

1. 導航至 **Customers > All Customers**
2. 選擇客戶
3. 使用 **Export Data** 動作生成完整的資料匯出
4. 如果客戶要求，將匯出資料發送給客戶
5. 然後進行刪除

匯出資料包含：客戶個人資料、訂購歷史、地址、備註和分析資料。

## 小技巧

- **使用篩選來識別高價值客戶** — 透過客戶價值篩選來找出您的冠軍和 VIP 客戶
- **定期查看客戶備註** — 每週至少檢查一次開放的後續任務
- **不要手動編輯分析** — 讓系統自動計算 RFM 分數和分群
- **主動轉換訪客** — 當訪客完成第二次購買時，主動聯繫並提供建立正式帳號的選擇
- **使用停用而非刪除** — 停用會保留資料，如需可逆轉
- **在支援通話中添加備註** — 記錄支援互動，讓其他團隊成員有背景資訊
- **設置後續日期** — 使用備註中的後續任務系統確保沒有遺漏
- **尊重通訊偏好** — 絕對不要向已選擇退出的客戶發送營銷郵件
