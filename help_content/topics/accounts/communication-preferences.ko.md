## 커뮤니케이션 설정

커뮤니케이션 설정은 고객이 가게에서 받는 이메일과 SMS 메시지의 종류를 조절할 수 있도록 합니다. 이 시스템은 GDPR 준수를 보장하고, 모든 채널에서 고객의 커뮤니케이션 설정을 존중하도록 도와줍니다.

관리자 사이드바에서 **Customers > Communication Preferences**로 이동하여 고객의 커뮤니케이션 설정을 관리할 수 있습니다.

## 커뮤니케이션 설정 이해

커뮤니케이션 설정 시스템은 고객이 받는 메시지에 대한 세부적인 제어를 제공합니다. 이는 다음과 같습니다:

- **거래 이메일** — 필수 주문 확인, 배송 업데이트, 계정 보안 이메일 (항상 활성화됨)
- **마케팅 이메일** — 뉴스레터, 프로모션, 제품 추천 (사전 동의 필요)
- **앱별 알림** — 블로그 게시물, 충성도 포인트, 추천 보상, 파트너십 수수료
- **SMS 알림** — 텍스트 메시지 알림 (TCPA에 따라 명시적 동의 필요)

모든 마케팅 커뮤니케이션은 고객의 동의와 이메일 확인을 필요로 하여 GDPR 준수를 보장합니다.

## 설정 유형 설명

### 거래 통신 (항상 활성화됨)

거래 메시지는 고객의 계정과 주문에 필수적이므로 고객이 **비활성화할 수 없습니다**:

| 유형 | 설명 | 예시 |
|------|-------------|----------|
| **주문 확인** | 주문이 생성될 때 확인 | 주문 #12345가 수신됨 |
| **배송 업데이트** | 주문 상태가 변경될 때 알림 | 주문이 배송됨 |
| **결제 확인** | 결제 수령, 환불 처리 | $49.99 결제 확인 |
| **계정 보안** | 비밀번호 재설정, 이메일 확인 | 비밀번호 재설정 |

### 마케팅 커뮤니케이션 (사전 동의 필요)

마케팅 메시지는 고객의 동의와 이메일 확인을 필요로 합니다:

| 유형 | 설명 | 기본값 |
|------|-------------|---------|
| **뉴스레터** | 일반 뉴스레터 및 업데이트 | 비활성화 |
| **프로모션 제안** | 할인, 특별 제안 | 비활성화 |
| **제품 추천** | 맞춤형 제품 제안 | 비활성화 |
| **재입고 알림** | 제품 재입고 알림 | 비활성화 |

고객은 마케팅 이메일을 받기 전에 **이메일 주소를 확인**해야 합니다 (GDPR 이중 동의 요구 사항).

### 앱별 설정

고객은 특정 기능에서의 알림을 제어할 수 있습니다:

**블로그 알림**
- 새 블로그 게시물 (즉시, 주간 요약, 월간 요약)
- 카테고리별 구독
- 빈도 설정

**충성도 프로그램**
- 포인트 획득 알림
- 레벨 업그레이드
- 보상 해제
- 곧 만료되는 포인트
- 생일 보너스
- 캠페인 제안

**추천 프로그램**
- 보상 발급 (추천자 및 추천 받은 사람)
- 성공적인 추천 가입
- 곧 만료되는 보상
- 추천 초대

**파트너십 프로그램**
- 수수료 획득
- 수수료 승인 또는 거부
- 지급 처리, 완료, 실패
- 월별 성과 보고서

### SMS 알림 (명시적 동의 필요)

모든 SMS 알림은 TCPA 규정에 따라 **명시적 동의**가 필요합니다. 고객은 반드시 SMS 동의 상자에 체크해야 합니다:

- **거래 SMS** — 주문 배송, 도착 (동의 필요)
- **마케팅 SMS** — 프로모션, 특별 제안 (별도 동의 필요)

거래 SMS도 동의가 필요한 이유는, 요청 없는 텍스트 메시지의 전송이 이메일보다 엄격하게 규제되기 때문입니다.
