通信設定

通信設定は、顧客がストアから受け取るメールやSMSメッセージを制御できるようにします。このシステムはGDPRの遵守を保証し、すべてのチャネルで顧客の通信設定を尊重するのを助けます。

管理サイドバーで **Customers > Communication Preferences** にアクセスして、顧客の通信設定を管理してください。

## 通信設定の理解

通信設定システムは、顧客が受信するメッセージを細かく制御できるようにします。これは以下の内容を含みます：

- **取引メール** — 注文確定、配送更新、アカウントセキュリティメール（常に有効）
- **マーケティングメール** — ニュースレター、プロモーション、製品の推奨（オプトインが必要）
- **アプリ固有の通知** — ブログ記事、ロイヤルティポイント、リファラル報酬、アフィリエイト報酬
- **SMS通知** — テキストメッセージ通知（TCPAに従って明示的なオプトインが必要）

すべてのマーケティング通信は顧客の同意とメール確認が必要で、GDPRの遵守を保証します。

## 通信設定タイプの説明

### 取引通信（常に有効）

取引メッセージは、顧客のアカウントおよび注文にとって重要です。これらのメッセージは顧客によって**無効にすることはできません**：

| Type | Description | Examples |
|------|-------------|----------|
| **注文確定** | 注文が確定したときの確認 | 注文番号12345が受け取られました |
| **配送状況の更新** | 注文のステータスが変更されたときの通知 | ご注文は発送されました |
| **支払い確認** | 支払いの受領、返金の処理 | $49.99の支払いが確認されました |
| **アカウントセキュリティ** | パスワードのリセット、メール確認 | パスワードをリセットしてください |

### マーケティング通信（オプトインが必要）

マーケティングメッセージは顧客の同意とメール確認が必要です：

| Type | Description | Default |
|------|-------------|---------|
| **ニュースレター** | 一般的なニュースレターおよび更新 | オプトアウト |
| **プロモーションオファー** | 営業、割引、特別オファー | オプトアウト |
| **製品の推奨** | パーソナライズされた製品提案 | オプトアウト |
| **在庫に戻る** | 商品が在庫に戻ったときの通知 | オプトアウト |

顧客は、マーケティングメールを受信する前に**メールアドレスを確認する必要があります**（GDPRの二重オプトイン要件）。

### アプリ固有の設定

顧客は特定の機能からの通知を制御できます：

**ブログ通知**
- 新しいブログ記事の公開（即時、週次、月次の要約）
- カテゴリ固有の購読
- 頻度設定

**ロイヤルティプログラム**
- ポイント獲得通知
- ランクアップ
- レワードの解放
- 期限が近いポイント
- 誕生日ボーナス
- キャンペーンオファー

**リファラルプログラム**
- レワードの発行（リファラーおよびリファーリー）
- 成功したリファラル登録
- 期限が近いレワード
- リファラル招待

**アフィリエイトプログラム**
- コミッションの獲得
- コミッションの承認または拒否
- 支払いの処理、完了、失敗
- 月次のパフォーマンスレポート

### SMS通知（明示的なオプトインが必要）

すべてのSMS通知はTCPA規則に従って**明示的なオプトイン**が必要です。顧客はアクティブにSMSオプトインボックスをチェックする必要があります：

- **取引SMS** — 注文の発送、到着（オプトインが必要）
- **マーケティングSMS** — プロモーション、特別オファー（個別のオプトインが必要）

取引SMSもオプトインが必要です。これは、不要なテキストメッセージの送信がメールよりも厳しく規制されているためです。

## 管理画面で顧客の設定を管理する

### すべての設定を表示する

**Customers > Communication Preferences**にアクセスしてすべての顧客の設定を表示します：

| Column | Description |
|--------|-------------|
| **ユーザーのメール** | 顧客のメールアドレス（ユーザー管理画面にリンク） |
| **メールのステータス** | メールが有効な場合は緑色の✓、無効な場合は灰色の○ |
| **SMSのステータス** | SMSが有効な場合は緑色の✓、無効な場合は灰色の○ |
| **マーケティングのステータス** | 「Opted In」または「Opted Out」のバッジ |
| **確認ステータス** | 📧✓ がメールが確認済み、📱✓ がSMSが確認済み |
| **同意の出典** | 顧客が同意した場所（登録、チェックアウト、設定センター、API、移行） |
| **更新日時** | 設定が最後に変更された時間 |

### 設定をフィルタリングする

フィルターサイドバーを使用して顧客を検索してください：

- **メール有効** — はい/いいえ
- **SMS有効** — はい/いいえ
- **メールマーケティング** — はい/いいえ（マーケティングにオプトイン）
- **SMSマーケティング** — はい/いいえ（SMSマーケティングにオプトイン）
- **メール確認済み** — はい/いいえ（メールアドレスを確認済み）
- **SMS確認済み** — はい/いいえ（電話番号を確認済み）
- **同意の出典** — 登録、チェックアウト、設定センター、API、移行
- **言語コード** — 通信用の希望言語

### 設定を検索する

以下で顧客を検索できます：
- ユーザーのメール
- ユーザー名
- お名前
- ご姓
- サブスクリプショントークン

### バッチアクション

複数の顧客を選択してバッチアクションを適用します：

**✓ メールを確認済みとしてマーク**
- 顧客のメールアドレスを手動で確認します
- 他のシステムから顧客をインポートした場合に便利です
- 設定キャッシュを無効にして変更をすぐに適用します

**🚫 すべてのマーケティングからサブスクリプション解除**
- すべてのマーケティング通信（メール、SMS、すべてのアプリ）を無効にします
- 取引メールは有効のままです
- 顧客が完全にサブスクリプション解除を要求した場合に使用します
- GDPRの同意撤回権を尊重します

**📥 CSVに設定をエクスポート**
- 顧客の設定をスプレッドシートにエクスポートします
- すべての設定フィールドとアプリ固有の設定を含みます
- 合規性監査と分析に役立ちます
- 形式：CSVヘッダー付き

## 顧客自身の設定センター

顧客は、ログインしているときに `/accounts/preferences/` で自分の設定を管理できます。
