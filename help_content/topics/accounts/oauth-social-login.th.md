OAuth และการเข้าสู่ระบบผ่านโซเชียลช่วยให้ลูกค้าสามารถเข้าสู่ระบบในร้านค้าของคุณโดยใช้บัญชีที่มีอยู่แล้วของ Google, Apple หรือ Microsoft — ไม่จำเป็นต้องสร้างและจำรหัสผ่านอีกต่อไป

![การตั้งค่า OAuth](/static/core/admin/img/help/oauth-social-login/oauth-settings.webp)

## OAuth / การเข้าสู่ระบบผ่านโซเชียลคืออะไร?

OAuth เป็นมาตรฐานการตรวจสอบความถูกต้องที่ปลอดภัย ซึ่งอนุญาตให้ลูกค้าเข้าสู่ระบบโดยใช้ข้อมูลประจำตัวจากผู้ให้บริการที่น่าเชื่อถือ เช่น Google, Apple หรือ Microsoft

### ประโยชน์

- **การชำระเงินที่รวดเร็วขึ้น** — ลูกค้าสามารถข้ามแบบฟอร์มการลงทะเบียนและเข้าสู่ระบบด้วยการคลิกเพียงครั้งเดียว
- **ลดความยุ่งยาก** — ไม่จำเป็นต้องสร้างรหัสผ่าน การส่งอีเมลยืนยัน หรือการดำเนินการเมื่อลืมรหัสผ่าน
- **การแปลงที่ดีขึ้น** — การศึกษาแสดงให้เห็นว่าการเข้าสู่ระบบผ่านโซเชียลสามารถเพิ่มอัตราการแปลงได้ 20-40%
- **ความปลอดภัยที่ดีขึ้น** — ข้อมูลประจำตัวจะไม่ผ่านร้านค้าของคุณ การตรวจสอบความถูกต้องจะถูกดำเนินการโดยผู้ให้บริการ
- **ความไว้วางใจของลูกค้า** — ลูกค้ามีความไว้วางใจผู้ให้บริการที่มีชื่อเสียงในการใช้ข้อมูลการเข้าสู่ระบบของพวกเขา

### วิธีการทำงาน

1. ลูกค้าคลิกที่ปุ่ม "Sign in with Google" (หรือ Apple/Microsoft) บนหน้าเข้าสู่ระบบของคุณ
2. พวกเขาจะถูกเปลี่ยนเส้นทางไปยังหน้าเข้าสู่ระบบของผู้ให้บริการที่ปลอดภัย
3. ลูกค้าตรวจสอบความถูกต้องด้วยข้อมูลประจำตัวของผู้ให้บริการ
4. ผู้ให้บริการส่งข้อมูลตัวตนที่ได้รับการยืนยันกลับไปยังร้านค้าของคุณ
5. ลูกค้าจะถูกเข้าสู่ระบบโดยอัตโนมัติ

เมื่อเข้าสู่ระบบครั้งแรก บัญชีลูกค้าใหม่จะถูกสร้างขึ้นโดยอัตโนมัติโดยใช้อีเมลและข้อมูลโปรไฟล์จากผู้ให้บริการ

## ผู้ให้บริการที่รองรับ

Spwig รองรับผู้ให้บริการ OAuth 3 รายหลัก:

| ผู้ให้บริการ | กรณีการใช้งาน | ข้อกำหนดของข้อมูลประจำตัว |
|----------|----------|------------------------|
| **Google** | ยอดนิยมที่สุด ง่ายต่อการตั้งค่า | Client ID, Client Secret |
| **Apple** | จำเป็นสำหรับแอป iOS ให้ความสำคัญกับความเป็นส่วนตัว | Client ID, Team ID, Key ID, Private Key |
| **Microsoft** | ลูกค้าองค์กร ผู้ใช้ Office 365 | Client ID, Client Secret, Tenant ID |

คุณสามารถเปิดใช้งานได้หนึ่ง สอง หรือทั้งสามผู้ให้บริการ แต่ละผู้ให้บริการทำงานได้อย่างอิสระ

## การตั้งค่า OAuth ของ Google

OAuth ของ Google เป็นตัวเลือกยอดนิยมและง่ายต่อการตั้งค่าที่สุด

### ข้อกำหนดเบื้องต้น

- บัญชี Google
- การเข้าถึง Google Cloud Console

### การตั้งค่าขั้นตอนโดยขั้นตอน

1. **ไปที่การตั้งค่า OAuth**
   - ไปที่ **Settings > Store Settings** ในแดชบอร์ดผู้ดูแลของคุณ
   - ค้นหาไปยังส่วน **OAuth Providers**
   - คลิก **Configure Google**

2. **สร้างโครงการ Google Cloud**
   - เยี่ยมชม [Google Cloud Console](https://console.cloud.google.com/)
   - คลิก **Create Project**
   - ใส่ชื่อโครงการ (เช่น "My Store OAuth")
   - คลิก **Create**

3. **เปิดใช้งาน Google+ API**
   - บนแถบด้านซ้าย ไปที่ **APIs & Services > Library**
   - ค้นหา "Google+ API"
   - คลิก **Enable**

4. **สร้างข้อมูลประจำตัว OAuth**
   - ไปที่ **APIs & Services > Credentials**
   - คลิก **Create Credentials > OAuth client ID**
   - เลือกประเภทแอปพลิเคชัน: **Web application**
   - ใส่ชื่อ (เช่น "Store Login")

5. **ตั้งค่า Redirect URI**
   - ภายใต้ **Authorized redirect URIs** เพิ่ม:
     ```
     https://yourdomain.com/accounts/google/login/callback/
     ```
   - แทนที่ `yourdomain.com` ด้วยโดเมนจริงของคุณ
   - คลิก **Create**

6. **คัดลอกข้อมูลประจำตัว**
   - คัดลอก **Client ID** และ **Client Secret** จากหน้าต่างป็อปอัป

7. **ใส่ข้อมูลประจำตัวใน Spwig**
   - กลับไปที่การตั้งค่า OAuth ของ Spwig
   - วาง Client ID และ Client Secret
   - คลิก **Save**
   - เปิดใช้งาน **Enable Google OAuth** เพื่อเปิดใช้งาน

### การทดสอบ

- เยี่ยมชมหน้าเข้าสู่ระบบของร้านค้าของคุณ
- หาปุ่ม "Sign in with Google"
- คลิกและตรวจสอบความถูกต้องด้วยบัญชี Google ของคุณ
- คุณควรจะเข้าสู่ระบบและถูกเปลี่ยนเส้นทางไปยังแดชบอร์ดลูกค้าของคุณ

## การตั้งค่า OAuth ของ Apple

OAuth ของ Apple มีความซับซ้อนมากกว่า Google เนื่องจากใช้ระบบตรวจสอบความถูกต้องที่ใช้คีย์

### ข้อกำหนดเบื้องต้น

- บัญชีผู้พัฒนา Apple (ต้องสมัครสมาชิกแบบเสียค่าใช้จ่าย)
- การเข้าถึงพอร์ทัลผู้พัฒนา Apple

### การตั้งค่าขั้นตอนโดยขั้นตอน

1. **ไปที่การตั้งค่า OAuth**
   - ไปที่ **Settings > Store Settings > OAuth Providers**
   - คลิก **Configure Apple**

2. **สร้าง Service ID**
   - เข้าสู่ [Apple Developer](https://developer.apple.com/account/)
   - ไปที่ **Certificates, Identifiers & Profiles**
   - คลิก **Identifiers** และจากนั้นคลิกปุ่ม **+**
   - เลือก **Services IDs** และคลิก **Continue**
   - ใส่คำอธิบาย (เช่น "Store Login")
   - ใส่ตัวระบุ (เช่น `com.yourstore.login`)
   - คลิก **Continue** และจากนั้น **Register**

3. **ตั้งค่า Service ID**
   - คลิก Service ID ที่คุณสร้างใหม่
   - ตรวจสอบ **Sign In with Apple**
   - คลิก **Configure**
   - เพิ่มโดเมนและ URL คืนค่า:
     - **Domains**: `yourdomain.com`
     - **Return URLs**: `https://yourdomain.com/accounts/apple/login/callback/`
   - คลิก **Save** และจากนั้น **Continue** และ **Save** อีกครั้ง

4. **สร้าง Key**
   - บนแถบด้านซ้าย คลิก **Keys** และจากนั้นคลิกปุ่ม **+**
   - ใส่ชื่อคีย์ (เช่น "Store OAuth Key")
   - ตรวจสอบ **Sign In with Apple**
   - คลิก **Configure** และเลือก Primary App ID ของคุณ
   - คลิก **Save** แล้ว **Continue** และ **Register**
   - **ดาวน์โหลดไฟล์คีย์** (.p8) — คุณไม่สามารถดาวน์โหลดมันอีกครั้ง

5. **รวบรวมข้อมูลที่จำเป็น**
   คุณจำเป็นต้องมี:
   - **Client ID (Service ID)**: ตัวระบุที่คุณสร้าง (เช่น `com.yourstore.login`)
   - **Team ID**: พบได้ที่มุมขวาบนของพอร์ทัลผู้พัฒนา Apple
   - **Key ID**: แสดงเมื่อคุณสร้างคีย์
   - **Private Key**: ข้อมูลของไฟล์ .p8 ที่คุณดาวน์โหลดมา

6. **ใส่ข้อมูลประจำตัวใน Spwig**
   - กลับไปที่การตั้งค่า OAuth ของ Spwig
   - วาง Client ID, Team ID และ Key ID
   - เปิดไฟล์ .p8 ในตัวแก้ไขข้อความและคัดลอกเนื้อหาของมัน
   - วางคีย์ทั้งหมด (รวมหัวข้อ) ลงในช่อง Private Key
   - คลิก **Save**
   - เปิดใช้งาน **Enable Apple OAuth** เพื่อเปิดใช้งาน

### การทดสอบ

- เยี่ยมชมหน้าเข้าสู่ระบบของร้านค้าของคุณบนอุปกรณ์ที่มี Apple ID
- คลิก "Sign in with Apple"
- ตรวจสอบความถูกต้องด้วย Apple ID ของคุณ
- คุณควรจะเข้าสู่ระบบได้สำเร็จ

## การตั้งค่า OAuth ของ Microsoft

OAuth ของ Microsoft เป็นตัวเลือกที่เหมาะสำหรับร้านค้าที่มุ่งเน้นลูกค้าองค์กรที่ใช้ Office 365 หรือ Azure AD

### ข้อกำหนดเบื้องต้น

- บัญชี Microsoft
- การเข้าถึง Azure Portal

### การตั้งค่าขั้นตอนโดยขั้นตอน

1. **ไปที่การตั้งค่า OAuth**
   - ไปที่ **Settings > Store Settings > OAuth Providers**
   - คลิก **Configure Microsoft**

2. **ลงทะเบียนแอปพลิเคชันใน Azure**
   - เยี่ยมชม [Azure Portal](https://portal.azure.com/)
   - ไปที่ **Azure Active Directory > App registrations**
   - คลิก **New registration**
   - ใส่ชื่อ (เช่น "Store OAuth")
   - เลือก **Accounts in any organizational directory and personal Microsoft accounts**
   - ภายใต้ **Redirect URI** เลือก **Web** และใส่:
     ```
     https://yourdomain.com/accounts/microsoft/login/callback/
     ```
   - คลิก **Register**

3. **คัดลอก Application ID**
   - บนหน้าสรุปแอปพลิเคชัน คัดลอก **Application (client) ID**

4. **สร้าง Client Secret**
   - บนแถบด้านซ้าย คลิก **Certificates & secrets**
   - คลิก **New client secret**
   - ใส่คำอธิบาย (เช่น "OAuth Secret")
   - เลือกช่วงเวลาหมดอายุ (แนะนำ: 24 เดือน)
   - คลิก **Add**
   - **คัดลอกค่าความลับทันที** — มันจะไม่แสดงอีกครั้ง

5. **ใส่ข้อมูลประจำตัวใน Spwig**
   - กลับไปที่การตั้งค่า OAuth ของ Spwig
   - วาง Application (client) ID เป็น Client ID
   - วางค่าความลับเป็น Client Secret
   - ใส่ Tenant ID (สำหรับแอปพลิเคชันแบบ single-tenant; ปล่อยว่างสำหรับ multi-tenant) ได้ตามต้องการ
   - คลิก **Save**
   - เปิดใช้งาน **Enable Microsoft OAuth** เพื่อเปิดใช้งาน

### การทดสอบ

- เยี่ยมชมหน้าเข้าสู่ระบบของร้านค้าของคุณ
- คลิก "Sign in with Microsoft"
- ตรวจสอบความถูกต้องด้วยบัญชี Microsoft ของคุณ
- คุณควรจะเข้าสู่ระบบได้สำเร็จ

## การจัดการการเชื่อมต่อ OAuth

### ความคิดเห็นของลูกค้า

ลูกค้าสามารถดูและจัดการผู้ให้บริการ OAuth ที่เชื่อมต่อได้จากแดชบอร์ดบัญชีของพวกเขา:

- ไปที่ **My Account > Connected Accounts**
- ดูผู้ให้บริการที่เชื่อมต่อ (Google, Apple, Microsoft)
- ตัดการเชื่อมต่อผู้ให้บริการโดยคลิก **Disconnect**
- เชื่อมต่ออีกครั้งโดยเข้าสู่ระบบด้วยผู้ให้บริการนั้นอีกครั้ง

### ผู้ให้บริการหลายราย

บัญชีลูกค้าเดียวสามารถเชื่อมต่อกับผู้ให้บริการ OAuth หลายรายได้ ตัวอย่างเช่น ลูกค้าสามารถเชื่อมต่อทั้ง Google และ Apple ไปยังบัญชีเดียวกัน

หากลูกค้าพยายามเข้าสู่ระบบด้วยผู้ให้บริการ OAuth ที่แตกต่างกันโดยใช้อีเมลเดียวกัน Spwig จะเชื่อมต่อกับบัญชีที่มีอยู่ของพวกเขาโดยอัตโนมัติ

### การจัดการโดยผู้ดูแล

ในฐานะผู้ดูแล คุณสามารถดูการเชื่อมต่อ OAuth ของลูกค้า:

- ไปที่ **Customers > Customers**
- เปิดบันทึกลูกค้า
- ค้นหาไปยังส่วน **Connected Accounts**
- ดูผู้ให้บริการที่เชื่อมต่อและวันที่ที่เชื่อมต่อ

คุณไม่สามารถตัดการเชื่อมต่อผู้ให้บริการแทนลูกค้าได้ — พวกเขาต้องทำมันด้วยตนเองเพื่อความปลอดภัย

## การแก้ไขปัญหา

### Redirect URI ไม่ตรงกัน

**ข้อผิดพลาด**: "Redirect URI mismatch" หรือ "Invalid redirect_uri"

**วิธีแก้ไข**:
- ตรวจสอบให้แน่ใจว่า Redirect URI ในการตั้งค่าผู้ให้บริการตรงกับที่ใน Spwig อย่างแน่นอน
- ตรวจสอบว่ามีการใช้เครื่องหมาย / ที่ปลายทาง — ต้องตรงกัน
- ตรวจสอบให้แน่ใจว่าคุณใช้ `https://` (ไม่ใช่ `http://`)
- ล้างแคชของเบราว์เซอร์และลองอีกครั้ง

### ข้อมูลประจำตัวไม่ถูกต้อง

**ข้อผิดพลาด**: "Invalid client ID" หรือ "Authentication failed"

**วิธีแก้ไข**:
- ตรวจสอบให้แน่ใจว่าคุณคัดลอก Client ID และ Client Secret ได้อย่างถูกต้อง
- ตรวจสอบว่าไม่มีช่องว่างเพิ่มเติมหรือการขึ้นบรรทัดใหม่
- ตรวจสอบให้แน่ใจว่าข้อมูลประจำตัวมาจากโครงการ/แอปพลิเคชันที่ถูกต้อง
- สำหรับ Apple ตรวจสอบให้แน่ใจว่า Private Key มีเนื้อหาทั้งหมดของไฟล์ .p8

### API ของผู้ให้บริการไม่ได้เปิดใช้งาน

**ข้อผิดพลาด**: "API not enabled" หรือ "Access not configured"

**วิธีแก้ไข**:
- สำหรับ Google: ตรวจสอบให้แน่ใจว่าคุณเปิดใช้งาน Google+ API ในโครงการ Google Cloud ของคุณ
- สำหรับ Microsoft: ตรวจสอบให้แน่ใจว่าการลงทะเบียนแอปพลิเคชันได้รับการอนุมัติและเปิดใช้งาน
- สำหรับ Apple: ตรวจสอบให้แน่ใจว่า "Sign In with Apple" เปิดใช้งานสำหรับ Service ID ของคุณ

### ต้องการ SSL

**ข้อผิดพลาด**: "OAuth requires HTTPS" หรือ "Insecure redirect URI"

**วิธีแก้ไข**:
- ผู้ให้บริการ OAuth ต้องการ SSL/TLS (HTTPS) เพื่อความปลอดภัย
- ตรวจสอบให้แน่ใจว่าร้านค้าของคุณมีใบรับรอง SSL ที่ถูกต้องติดตั้ง
- อัปเดต Redirect URI ของคุณให้ใช้ `https://` แทนที่จะใช้ `http://`
- หากคุณกำลังทดสอบในท้องถิ่น ใช้บริการเช่น ngrok เพื่อสร้างท่อกลับด้าน HTTPS

### ปุ่มไม่แสดงขึ้นมา

**ปัญหา**: ปุ่ม "Sign in with Google/Apple/Microsoft" ไม่แสดงขึ้นมาบนหน้าเข้าสู่ระบบ

**วิธีแก้ไข**:
- ตรวจสอบให้แน่ใจว่าผู้ให้บริการได้รับการเปิดใช้งานในการตั้งค่า OAuth
- ล้างแคชของเบราว์เซอร์และรีเฟรชหน้า
- ตรวจสอบให้แน่ใจว่าธีมของคุณมีเทมเพลตสำหรับการเข้าสู่ระบบผ่านโซเชียล
- ตรวจสอบคอนโซลของเบราว์เซอร์เพื่อหาข้อผิดพลาดของ JavaScript

## เคล็ดลับและแนวทางปฏิบัติที่ดีที่สุด

### ความปลอดภัย

- **หมุนเวียนคีย์ความลับเป็นประจำ** — อัปเดต Client Secrets ทุก 12-24 เดือน
- **ตรวจสอบการเข้าสู่ระบบล้มเหลว** — ตรวจสอบรูปแบบการตรวจสอบความถูกต้องที่ไม่ปกติ
- **ใช้ข้อมูลประจำตัวแยกต่างหากต่อสภาพแวดล้อม** — ใช้ข้อมูลประจำตัวที่แตกต่างกันสำหรับสภาพแวดล้อมการทดสอบและสภาพแวดล้อมการผลิต
- **จำกัด Redirect URI** — เพิ่มเฉพาะ URI ที่คุณต้องการ

### ประสบการณ์ผู้ใช้

- **เปิดใช้งานผู้ให้บริการทั้งสาม** — ให้ลูกค้ามีทางเลือก; ประชากรที่แตกต่างกันมีความชอบผู้ให้บริการที่แตกต่างกัน
- **วางปุ่มอย่างเด่นชัด** — ปุ่มการเข้าสู่ระบบผ่านโซเชียลควรอยู่เหนือแบบฟอร์มอีเมล/รหัสผ่าน
- **ใช้แบรนด์ที่เป็นที่รู้จัก** — รักษาสไตล์ปุ่มมาตรฐานของ Google/Apple/Microsoft
- **ทดสอบบนมือถือ** — การดำเนินการ OAuth ทำงานต่างกันบนเบราว์เซอร์มือถือ

### การปฏิบัติตามกฎหมาย

- **นโยบายความเป็นส่วนตัว** — เปิดเผยว่าคุณใช้ผู้ให้บริการ OAuth และข้อมูลใดที่คุณได้รับ
- **ข้อกำหนดและเงื่อนไข** — ปฏิบัติตามข้อกำหนดของผู้ให้บริการ (Google, Apple, Microsoft แต่ละรายมีข้อกำหนดของตนเอง)
- **การลดข้อมูล** — ขอข้อมูลโปรไฟล์ที่คุณต้องการจริงๆ เท่านั้น
