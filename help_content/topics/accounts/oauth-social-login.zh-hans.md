OAuth和社交登录允许客户使用现有的Google、Apple或Microsoft账户登录到您的商店——无需再创建和记住另一个密码。

![OAuth设置](/static/core/admin/img/help/oauth-social-login/oauth-settings.webp)

## 什么是OAuth / 社交登录？

OAuth是一种安全的认证标准，允许客户使用来自Google、Apple或Microsoft等可信提供商的凭证登录。

### 优势

- **更快的结账流程** — 客户可以跳过注册表单，一键登录
- **减少摩擦** — 无需创建密码、验证电子邮件或忘记密码流程
- **更好的转化率** — 研究表明，社交登录可以将转化率提高20-40%
- **增强的安全性** — 凭证不会通过您的商店传递；认证由提供商处理
- **客户信任** — 客户信任已建立的提供商管理他们的登录凭证

### 它是如何工作的

1. 客户在您的登录页面上点击“使用Google登录”（或Apple/Microsoft）
2. 他们会被重定向到提供商的安全登录页面
3. 客户使用提供商的凭证进行认证
4. 提供商将验证后的身份信息发送回您的商店
5. 客户自动登录

首次登录时，会使用客户从提供商处获得的电子邮件和个人资料信息自动创建一个新客户账户。

## 受支持的提供商

Spwig支持三个主要的OAuth提供商：

| 提供商 | 用例 | 凭证要求 |
|----------|----------|------------------------|
| **Google** | 最受欢迎，最易于设置 | 客户端ID，客户端密钥 |
| **Apple** | iOS应用必需，注重隐私 | 客户端ID，团队ID，密钥ID，私钥 |
| **Microsoft** | 企业客户，Office 365用户 | 客户端ID，客户端密钥，租户ID |

您可以启用一个、两个或所有三个提供商。每个提供商独立运行。

## 设置Google OAuth

Google OAuth是最受欢迎的选项，也是最容易配置的。

### 先决条件

- 一个Google账户
- 访问Google Cloud Console的权限

### 分步设置

1. **导航到OAuth设置**
   - 在您的管理面板中，前往 **设置 > 商店设置**
   - 滚动到 **OAuth提供商** 部分
   - 点击 **配置Google**

2. **创建Google Cloud项目**
   - 访问 [Google Cloud Console](https://console.cloud.google.com/)
   - 点击 **创建项目**
   - 输入项目名称（例如，"My Store OAuth"）
   - 点击 **创建**

3. **启用Google+ API**
   - 在左侧边栏，前往 **API和服务 > 图书馆**
   - 搜索"Google+ API"
   - 点击 **启用**

4. **创建OAuth凭证**
   - 前往 **API和服务 > 凭证**
   - 点击 **创建凭证 > OAuth客户端ID**
   - 选择应用程序类型：**Web应用**
   - 输入名称（例如，"Store Login"）

5. **配置重定向URI**
   - 在 **授权的重定向URI** 下，添加：
     ```
     https://yourdomain.com/accounts/google/login/callback/
     ```
   - 将`yourdomain.com`替换为您的实际域名
   - 点击 **创建**

6. **复制凭证**
   - 从弹出窗口中复制 **客户端ID** 和 **客户端密钥**

7. **在Spwig中输入凭证**
   - 返回到您的Spwig管理OAuth设置
   - 粘贴客户端ID和客户端密钥
   - 点击 **保存**
   - 切换 **启用Google OAuth** 以激活

### 测试

- 访问您的商店前端登录页面
- 查找"使用Google登录"按钮
- 点击它并使用您的Google账户进行认证
- 您应该会登录并被重定向到客户仪表板

## 设置Apple OAuth

由于Apple使用基于密钥的认证系统，Apple OAuth比Google更复杂。

### 先决条件

- 一个Apple开发者账户（需要付费会员）
- 访问Apple开发者门户的权限

### 分步设置

1. **导航到OAuth设置**
   - 前往 **设置 > 商店设置 > OAuth提供商**
   - 点击 **配置Apple**

2. **创建服务ID**
   - 登录 [Apple开发者](https://developer.apple.com/account/)
   - 前往 **证书、标识符和配置文件**
   - 点击 **标识符** 然后 **+** 按钮
   - 选择 **服务ID** 并点击 **继续**
   - 输入描述（例如，"Store Login"）
   - 输入标识符（例如，`com.yourstore.login`）
   - 点击 **继续** 然后 **注册**

3. **配置服务ID**
   - 点击您新创建的服务ID
   - 勾选 **使用Apple登录**
   - 点击 **配置**
   - 添加您的域名和返回URL：
     - **域名**：`yourdomain.com`
     - **返回URL**：`https://yourdomain.com/accounts/apple/login/callback/`
   - 点击 **保存** 然后 **继续** 并再次 **保存**

4. **创建密钥**
   - 在左侧边栏，点击 **密钥** 然后 **+** 按钮
   - 输入密钥名称（例如，"Store OAuth密钥"）
   - 勾选 **使用Apple登录**
   - 点击 **配置** 并选择您的主应用程序ID
   - 点击 **保存**，然后 **继续** 并 **注册**
   - **下载密钥文件** (.p8) — 您无法再次下载它

5. **收集所需信息**
   您需要：
   - **客户端ID（服务ID）**：您创建的标识符（例如，`com.yourstore.login`）
   - **团队ID**：在Apple开发者门户右上角找到
   - **密钥ID**：在创建密钥时显示
   - **私钥**：您下载的.p8文件的内容

6. **在Spwig中输入凭证**
   - 返回到Spwig OAuth设置
   - 粘贴客户端ID、团队ID和密钥ID
   - 在文本编辑器中打开.p8文件并复制其内容
   - 将整个密钥（包括标题）粘贴到私钥字段中
   - 点击 **保存**
   - 切换 **启用Apple OAuth** 以激活

### 测试

- 在具有Apple ID的设备上访问您的商店前端登录页面
- 点击"使用Apple登录"
- 使用Apple ID进行认证
- 您应该能够成功登录

## 设置Microsoft OAuth

Microsoft OAuth适合针对使用Office 365或Azure AD的企业客户商店。

### 先决条件

- 一个Microsoft账户
- 访问Azure门户的权限

### 分步设置

1. **导航到OAuth设置**
   - 前往 **设置 > 商店设置 > OAuth提供商**
   - 点击 **配置Microsoft**

2. **在Azure中注册应用程序**
   - 访问 [Azure门户](https://portal.azure.com/)
   - 前往 **Azure Active Directory > 应用注册**
   - 点击 **新注册**
   - 输入名称（例如，"Store OAuth"）
   - 选择 **任何组织目录中的帐户和个人Microsoft帐户**
   - 在 **重定向URI** 下，选择 **Web** 并输入：
     ```
     https://yourdomain.com/accounts/microsoft/login/callback/
     ```
   - 点击 **注册**

3. **复制应用程序ID**
   - 在应用程序概述页面上，复制 **应用程序（客户端）ID**

4. **创建客户端密钥**
   - 在左侧边栏，点击 **证书与密钥**
   - 点击 **新客户端密钥**
   - 输入描述（例如，"OAuth密钥"）
   - 选择过期时间（推荐：24个月）
   - 点击 **添加**
   - **立即复制密钥值** — 它将不再显示

5. **在Spwig中输入凭证**
   - 返回到Spwig OAuth设置
   - 将应用程序（客户端）ID粘贴为客户端ID
   - 将密钥值粘贴为客户端密钥
   - 可选地输入租户ID（用于单租户应用；多租户应用留空）
   - 点击 **保存**
   - 切换 **启用Microsoft OAuth** 以激活

### 测试

- 访问您的商店前端登录页面
- 点击"使用Microsoft登录"
- 使用Microsoft账户进行认证
- 您应该能够成功登录

## 管理OAuth连接

### 客户视角

客户可以从他们的账户仪表板查看和管理已连接的OAuth提供商：

- 导航到 **我的账户 > 已连接的账户**
- 查看已连接的提供商（Google、Apple、Microsoft）
- 点击 **断开连接** 以断开某个提供商
- 通过再次使用该提供商登录来重新连接

### 多个提供商

一个客户账户可以连接到多个OAuth提供商。例如，客户可以将Google和Apple连接到同一个账户。

如果客户尝试使用相同的电子邮件地址和不同的OAuth提供商登录，Spwig会自动将其链接到其现有账户。

### 管理员管理

作为管理员，您可以查看客户的OAuth连接：

- 前往 **客户 > 客户**
- 打开客户记录
- 滚动到 **已连接的账户** 部分
- 查看已连接的提供商以及连接时间

您不能代表客户断开提供商——出于安全原因，客户必须自己执行此操作。

## 故障排除

### 重定向URI不匹配

**错误**："重定向URI不匹配"或"无效的redirect_uri"

**解决方案**：
- 确保提供商设置中的重定向URI与Spwig中的完全一致
- 检查尾随斜杠——它们必须匹配
- 确认您使用的是`https://`（而不是`http://`）
- 清除浏览器缓存并重试

### 无效的凭证

**错误**："无效的客户端ID"或"认证失败"

**解决方案**：
- 再次确认您是否正确复制了客户端ID和客户端密钥
- 确保没有多余的空格或换行符
- 确认凭证来自正确的项目/应用程序
- 对于Apple，确保私钥包含.p8文件的全部内容

### 提供商API未启用

**错误**："API未启用"或"访问未配置"

**解决方案**：
- 对于Google：确保在Google Cloud项目中启用了Google+ API
- 对于Microsoft：确认您的应用程序注册已获得批准并处于活动状态
- 对于Apple：检查您的服务ID是否启用了"使用Apple登录"

### 需要SSL

**错误**："OAuth需要HTTPS"或"不安全的重定向URI"

**解决方案**：
- OAuth提供商要求SSL/TLS（HTTPS）以确保安全
- 确保您的商店安装了有效的SSL证书
- 将重定向URI更新为使用`https://`而不是`http://`
- 如果在本地测试，使用类似ngrok的服务创建HTTPS隧道

### 按钮未显示

**问题**："使用Google/Apple/Microsoft登录"按钮未显示在登录页面上

**解决方案**：
- 确认OAuth设置中已启用提供商
- 清除浏览器缓存并刷新页面
- 检查您的主题是否包含社交登录模板
- 查看浏览器控制台是否有JavaScript错误

## 建议和最佳实践

### 安全性

- **定期轮换密钥** — 每12-24个月更新一次客户端密钥
- **监控失败的登录尝试** — 注意异常的认证模式
- **每个环境使用不同的凭证** — 阶段环境和生产环境使用不同的凭证
- **限制重定向URI** — 仅添加您需要的确切URI

### 用户体验

- **启用所有三个提供商** — 给客户选择权；不同的人群可能偏好不同的提供商
- **突出显示按钮** — 社交登录按钮应放在电子邮件/密码表单上方
- **使用可识别的品牌** — 保持Google/Apple/Microsoft的标准按钮样式
- **在移动设备上测试** — 移动浏览器上的OAuth流程有所不同

### 合规性

- **隐私政策** — 披露您使用OAuth提供商以及您接收的数据
- **服务条款** — 遵守提供商的条款（Google、Apple、Microsoft各有要求）
- **数据最小化** — 仅请求您实际需要的个人资料信息

### 测试清单

在上线之前，测试以下内容：

- [ ] 使用每个提供商在桌面端登录
- [ ] 使用每个提供商在移动端登录
- [ ] 首次登录（账户创建）
- [ ] 后续登录（账户链接）
- [ ] 使用相同电子邮件在不同提供商上登录
- [ ] 断开并重新连接提供商
- [ ] 确保非OAuth用户的密码重置流程仍然有效
