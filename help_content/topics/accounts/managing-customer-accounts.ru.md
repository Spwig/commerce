---
title: Управление клиентскими аккаунтами
---

Клиентские аккаунты позволяют продавцам отслеживать информацию о клиентах, историю заказов и предпочтения. Перейдите в **Клиенты > Все клиенты** в боковом меню администратора, чтобы управлять клиентскими аккаунтами.

![Добавить клиента](/static/core/admin/img/help/managing-customer-accounts/add-customer.webp)

## Понимание клиентских аккаунтов и профилей клиентов

**Клиентские аккаунты** — это учетные данные для входа (электронная почта/пароль), хранящиеся в модели User. **Профили клиентов** хранят дополнительную информацию о клиентах, такую как номер телефона, дата рождения, предпочтения и аналитика. У каждого клиентского аккаунта есть соответствующий профиль, который хранит эту расширенную информацию.

Когда вы управляете клиентами в админ-панели, вы работаете с профилями клиентов, которые связываются с учетными записями пользователей в фоновом режиме.

## Просмотр всех клиентов

Список клиентов показывает всех зарегистрированных клиентов с ключевыми метриками:

| Столбец | Описание |
|--------|-------------|
| **Пользователь** | Имя клиента и адрес электронной почты |
| **Статус аффилиат** | Является ли клиент аффилиатом |
| **Значение клиента** | Общая сумма, потраченная клиентом (цветовая кодировка) |
| **Сегмент клиента** | Сегмент RFM (Чемпион, Лояльный, На грани, и т.д.) |
| **Общее количество заказов** | Количество завершенных заказов |
| **Дней с момента последнего заказа** | Недавность последней покупки |
| **VIP-клиент** | Значок, если клиент помечен как VIP |

### Фильтрация клиентов

Используйте боковую панель фильтра, чтобы сузить список:

- **Статус аффилиат** — Является аффилиатом, Не является аффилиатом, Ожидание подтверждения аффилиата, Активный, Приостановлен, Отклонен
- **Разметка дашборда** — Предпочтительный вид дашборда клиента
- **Подписка на рассылку** — Подписан ли клиент на рассылку
- **Рекламные письма** — Подписан ли клиент на рекламные письма
- **Создан** — Фильтрация по дате регистрации

### Поиск клиентов

Используйте строку поиска, чтобы найти клиентов по:
- Имя пользователя
- Адрес электронной почты
- Имя
- Фамилия
- Номер телефона

## Просмотр информации о клиенте

Нажмите на имя клиента, чтобы просмотреть его полный профиль. Страница деталей клиента показывает:

![Детали клиента](/static/core/admin/img/help/managing-customer-accounts/customer-detail.webp)

### Раздел информации о клиенте

Базовые контактные данные и статус аккаунта:
- **Пользователь** — Ссылка на базовый аккаунт пользователя
- **Телефон** — Номер телефона клиента
- **Дата рождения** — Для проверки возраста и кампаний по дню рождения

### Предпочтения дашборда

Как клиент настроил свой дашборд аккаунта:
- **Разметка дашборда** — Сетка, список или компактный вид
- **Показать историю заказов** — Показывать историю заказов на дашборде
- **Показать закладки** — Показывать закладки на дашборде
- **Показать недавние товары** — Показывать недавно просмотренные товары
- **Показать рекомендации** — Показывать рекомендации по товарам

### Предпочтения коммуникации

Статус подписки клиента на различные коммуникации:
- **Подписка на рассылку** — Подписан на общие рассылки
- **Рекламные письма** — Подписан на рекламные письма
- **Уведомления о заказе** — Подписан на обновления статуса заказа

### Аналитика клиента

Только для чтения, резюме поведения и значения клиента:
- **Краткое содержание аналитики клиента** — Оценки RFM, сегмент, общая ценность
- **Краткое содержание поведения покупок** — Частота заказов, средняя сумма заказа, предпочитаемые категории
- **Краткое содержание вовлеченности** — Последний вход, процент открытия писем, активность на сайте

Эти поля аналитики вычисляются автоматически и не могут быть вручную отредактированы. Для деталей см. [Понимание аналитики клиентов](customer-analytics.md).

## Создание клиентского аккаунта

Продавцы могут вручную создавать клиентские аккаунты для заказов по телефону, самовывоза или предварительной регистрации оптовых клиентов.

1. Нажмите **+ Добавить профиль клиента** в правом верхнем углу
2. Заполните обязательные и необязательные поля:

| Поле | Обязательно | Описание |
|-------|----------|-------------|
| **Пользователь** | Да | Выберите существующий аккаунт пользователя или создайте новый |
| **Телефон** | Нет | Номер телефона клиента |
| **Дата рождения** | Нет | Для проверки возраста и кампаний по дню рождения |
| **Подписка на рассылку** | Нет | Подписка клиента на рассылку |
| **Рекламные письма** | Нет | Подписка клиента на рекламные письма |

### Создание нового пользователя при добавлении профиля

Если у клиента еще нет аккаунта пользователя:
1. Нажмите значок **+** рядом с полем Пользователь
2. Введите **адрес электронной почты** клиента (он становится его именем пользователя)
3. Необязательно введите **имя** и **фамилию**
4. Необязательно установите **пароль**
5. Поставьте галочку **Отправить письмо с сбросом пароля**, если вы не установили пароль
6. Сохраните аккаунт пользователя
7. Завершите заполнение полей профиля клиента
8. Нажмите **Сохранить**

### Приветственные письма

После создания клиентского аккаунта:
- Если вы установили пароль, клиент может сразу войти в систему с этим паролем
- Если вы не установили пароль, система отправит письмо с сбросом пароля, чтобы клиент мог установить свой собственный пароль
- Вы можете вручную запустить приветственное письмо через систему электронной почты в разделе **Маркетинг > Рассылки электронной почты**

## Редактирование информации о клиенте

Чтобы обновить детали клиента:
1. Перейдите в **Клиенты > Все клиенты**
2. Нажмите имя клиента
3. Измените поля, которые хотите обновить
4. Нажмите **Сохранить**

### Что можно редактировать

**Контактные данные:**
- Имя (через аккаунт пользователя)
- Адрес электронной почты (через аккаунт пользователя)
- Номер телефона
- Дата рождения

**Предпочтения:**
- Статус подписки на рассылку
- Подписка на рекламные письма
- Предпочтения уведомлений о заказе
- Разметка дашборда и настройки видимости

### Что нельзя редактировать

Эти поля вычисляются автоматически на основе поведения клиента:
- Общая сумма / Значение клиента
- Количество заказов
- Сегмент клиента (Чемпион, Лояльный, На грани, и т.д.)
- Оценки RFM
- Прогнозы общей ценности
- Дата последнего заказа
- Краткие аналитические отчеты

Если эти поля отображаются неправильно, проверьте исходные данные о заказах или запустите ручное пересчитание в разделе **Клиенты > Аналитика** → **Пересчитать метрики**.

## Заметки о клиентах

Добавьте внутренние заметки о клиентах для отслеживания проблем с поддержкой, запросов VIP или задач, требующих последующего действия.

### Добавление заметки

1. Откройте профиль клиента
2. Прокрутите до раздела **Заметки о клиентах** (может быть отдельной вкладкой)
3. Нажмите **+ Добавить заметку**
4. Заполните детали заметки:

| Поле | Описание |
|-------|-------------|
| **Тип заметки** | Общая, Проблема с поддержкой, Жалоба, Похвала, Служба VIP, Требуется следующее действие, Проблема с оплатой, Проблема с доставкой |
| **Заголовок** | Краткое резюме заметки |
| **Содержание** | Подробное содержание заметки |
| **Требуется следующее действие** | Поставьте галочку, если это требует действий |
| **Дата следующего действия** | Дата, к которой требуется следующее действие |
| **Выполнено** | Поставьте галочку, когда следующее действие выполнено |

### Типы заметок

| Тип | Случаи использования |
|------|----------|
| **Общая заметка** | Любое общее наблюдение о клиенте |
| **Проблема с поддержкой** | Запись о тикете поддержки или проблеме |
| **Жалоба** | Жалоба клиента для отслеживания и решения |
| **Похвала** | Положительная обратная связь о клиенте или его обратной связи о вас |
| **Служба VIP** | Запросы на специальное обслуживание для VIP-клиентов |
| **Требуется следующее действие** | Задачи, требующие действий к определенной дате |
| **Проблема с оплатой** | Заметки о проблемах с оплатой или спорах |
| **Проблема с доставкой** | Заметки о проблемах с доставкой или особых запросах на доставку |

### История заметок

Все заметки отображаются в хронологическом порядке на профиле клиента. Каждая заметка показывает:
- Дата и время создания
- Создано (имя сотрудника)
- Значок типа заметки
- Заголовок и содержание
- Статус следующего действия, если применимо

### Внутренние заметки против заметок, видимых клиентам

Все заметки о клиентах по умолчанию являются **внутренними** — клиенты никогда не видят эти заметки. Они предназначены исключительно для коммуникации внутри команды продавца.

Если вам нужно связаться с клиентом, используйте систему электронной почты в разделе **Маркетинг > Рассылки электронной почты** или добавьте комментарий к конкретному заказу.

## Преобразование гостей в зарегистрированных клиентов

Гости создаются автоматически, когда кто-то завершает оформление заказа без создания аккаунта. Их имя пользователя соответствует шаблону `guest_10374`, где число — это уникальный идентификатор.

Чтобы преобразовать гостя в зарегистрированного клиента:

1. Перейдите в **Клиенты > Все клиенты**
2. Найдите гостя по адресу электронной почты заказа
3. Нажмите на профиль гостя
4. Нажмите ссылку **Пользователь**, чтобы отредактировать базовый аккаунт пользователя
5. Измените **имя пользователя** с `guest_10374` на реальный адрес электронной почты клиента
6. Измените **электронную почту**, чтобы она совпадала
7. Необязательно добавьте **имя** и **фамилию**
8. Поставьте галочку **Отправить письмо с сбросом пароля**, чтобы клиент мог установить пароль
9. Нажмите **Сохранить**

Теперь клиент может войти в систему с адресом электронной почты и увидит свои предыдущие заказы в качестве гостя в истории заказов.

### Почему преобразовывать гостей в клиентов?

- Заказы гостей не учитываются в аналитике клиентов и сегментах
- Гости не могут отслеживать заказы или просматривать историю заказов
- Преобразование гостей увеличивает количество зарегистрированных клиентов и улучшает точность аналитики
- Зарегистрированные клиенты чаще совершают повторные покупки

## Отключение и удаление аккаунтов

### Отключение аккаунта клиента

Отключение предотвращает вход в систему, сохраняя при этом все данные:

1. Откройте профиль клиента
2. Нажмите ссылку **Пользователь**, чтобы отредактировать аккаунт пользователя
3. **Снимите галочку "Активный"**
4. Нажмите **Сохранить**

**Что происходит:**
- Клиент не может войти в систему
- История заказов сохраняется
- Клиента можно позже снова активировать, поставив галочку "Активный"
- Аналитика и метрики остаются неизменными

**Используйте отключение для:**
- Временного приостановления аккаунтов из-за споров по оплате
- Блокировки агрессивных клиентов
- Клиентов, которые запросили прекращение доступа, но не хотят удалить данные

### Удаление аккаунта клиента

Удаление удаляет аккаунт и может оставить заказы без связи с клиентом:

1. Откройте профиль клиента
2. Прокрутите вниз и нажмите **Удалить**
3. Подтвердите удаление

**Что происходит:**
- Аккаунт клиента полностью удаляется
- Профиль клиента удаляется
- История заказов может быть оставлена без связи (заказы существуют, но не связаны с клиентом)
- Действие необратимо

**Используйте удаление для:**
- Запросов на удаление данных в соответствии с GDPR/CCPA (сначала экспортируйте данные)
- Тестовых аккаунтов, которые никогда не должны существовать
- Дублирующихся аккаунтов, созданных случайно

### Соответствие GDPR

Перед удалением аккаунта клиента в ответ на запрос в соответствии с GDPR:

1. Перейдите в **Клиенты > Все клиенты**
2. Выберите клиента
3. Используйте действие **Экспортировать данные**, чтобы сгенерировать полный экспорт данных
4. Отправьте экспорт клиенту, если он запросил это
5. Затем выполните удаление

Экспорт включает: профиль клиента, историю заказов, адреса, заметки и аналитические данные.

## Советы

- **Используйте фильтры для выявления высокорентабельных клиентов** — Фильтруйте по Значению клиента, чтобы найти ваших чемпионов и VIP-клиентов
- **Регулярно проверяйте заметки о клиентах** — Проверяйте открытые задачи по следующему действию как минимум раз в неделю
- **Не редактируйте аналитику вручную** — Позвольте системе автоматически вычислять оценки RFM и сегменты
- **Активно преобразуйте гостей** — После того как гость совершит второй заказ, свяжитесь с ним и предложите создать полноценный аккаунт
- **Используйте отключение вместо удаления** — Отключение сохраняет данные и может быть отменено, если это необходимо
- **Добавляйте заметки во время звонков поддержки** — Документируйте взаимодействие с поддержкой, чтобы другие члены команды имели контекст
- **Установите даты следующего действия** — Используйте систему задач следующего действия в заметках, чтобы ничего не упустить
- **Уважайте предпочтения по коммуникации** — Никогда не отправляйте рекламные письма клиентам, которые отписались
