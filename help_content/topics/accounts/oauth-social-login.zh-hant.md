title: OAuth與社交登錄設置
---

OAuth和社交登錄允許客戶使用現有的Google、Apple或Microsoft賬戶登錄您的商店——無需再創建和記住另一個密碼。

![OAuth設置](/static/core/admin/img/help/oauth-social-login/oauth-settings.webp)

## 什麼是OAuth / 社交登錄？

OAuth是一種安全的身份驗證標準，允許客戶使用來自可信提供商（如Google、Apple或Microsoft）的憑證登錄。

### 優點

- **更快捷的結帳流程** — 客戶可以跳過註冊表單，點擊一下即可登錄
- **減少摩擦** — 無需創建密碼、驗證電子郵件或遺忘密碼流程
- **更好的轉化率** — 研究顯示社交登錄可以提高轉化率20-40%
- **增強安全性** — 憑證不會經過您的商店；身份驗證由提供商處理
- **客戶信任** — 客戶信任已建立的提供商管理其登錄憑證

### 它是如何工作的

1. 客戶在您的登錄頁面上點擊"使用Google登錄"（或Apple/Microsoft）
2. 他們會被重定向到提供商的安全登錄頁面
3. 客戶使用提供商的憑證進行身份驗證
4. 提供商將驗證的身分信息發送回您的商店
5. 客戶自動登錄

首次登錄時，會自動使用提供商的電子郵件和個人資料信息創建一個新的客戶賬戶。

## 支持的提供商

Spwig支持三個主要的OAuth提供商：

| 提供商 | 使用場景 | 憑證要求 |
|----------|----------|------------------------|
| **Google** | 最受歡迎，最易於設置 | 客戶端ID，客戶端密鑰 |
| **Apple** | iOS應用程序必需，注重隱私 | 客戶端ID，團隊ID，密鑰ID，私鑰 |
| **Microsoft** | 企業客戶，Office 365用戶 | 客戶端ID，客戶端密鑰，租戶ID |

您可以啟用一個、兩個或所有三個提供商。每個都獨立運作。

## 設置Google OAuth

Google OAuth是最受歡迎的選項，也是最易於配置的。

### 先決條件

- 一個Google賬戶
- 可訪問Google Cloud Console

### 分步設置

1. **導航到OAuth設置**
   - 前往管理面板中的 **設置 > 商店設置**
   - 滾動到 **OAuth提供商** 部分
   - 點擊 **配置Google**

2. **創建Google Cloud項目**
   - 訪問 [Google Cloud Console](https://console.cloud.google.com/)
   - 點擊 **創建項目**
   - 輸入項目名稱（例如，"My Store OAuth"）
   - 點擊 **創建**

3. **啟用Google+ API**
   - 在左側菜單欄，前往 **APIs & Services > Library**
   - 搜索"Google+ API"
   - 點擊 **啟用**

4. **創建OAuth憑證**
   - 前往 **APIs & Services > Credentials**
   - 點擊 **創建憑證 > OAuth客戶端ID**
   - 選擇應用類型：**Web應用程序**
   - 輸入名稱（例如，"Store Login"）

5. **配置重定向URI**
   - 在 **授權的重定向URI** 下，添加：
     ```
     https://yourdomain.com/accounts/google/login/callback/
     ```
   - 將 `yourdomain.com` 替換為您的實際域名
   - 點擊 **創建**

6. **複製憑證**
   - 從彈出窗口中複製 **客戶端ID** 和 **客戶端密鑰**

7. **在Spwig中輸入憑證**
   - 返回到Spwig管理面板的OAuth設置
   - 黏貼客戶端ID和客戶端密鑰
   - 點擊 **保存**
   - 切換 **啟用Google OAuth** 以激活

### 測試

- 訪問您的商店前端登錄頁面
- 查找"使用Google登錄"按鈕
- 點擊它並使用您的Google賬戶進行身份驗證
- 您應該會登錄並被重定向到客戶儀表板

## 設置Apple OAuth

Apple OAuth由於其基於密鑰的身份驗證系統比Google更複雜。

### 先決條件

- 一個Apple開發者賬戶（需要付費會員）
- 可訪問Apple開發者門戶

### 分步設置

1. **導航到OAuth設置**
   - 前往 **設置 > 商店設置 > OAuth提供商**
   - 點擊 **配置Apple**

2. **創建服務ID**
   - 登錄 [Apple Developer](https://developer.apple.com/account/)
   - 前往 **證書、標識符和配置文件**
   - 點擊 **標識符** 然後點擊 **+** 按鈕
   - 選擇 **服務ID** 並點擊 **繼續**
   - 輸入描述（例如，"Store Login"）
   - 輸入標識符（例如，`com.yourstore.login`）
   - 點擊 **繼續** 然後 **註冊**

3. **配置服務ID**
   - 點擊您剛創建的服務ID
   - 勾選 **使用Apple登錄**
   - 點擊 **配置**
   - 添加您的域名和返回URL：
     - **域名**：`yourdomain.com`
     - **返回URL**：`https://yourdomain.com/accounts/apple/login/callback/`
   - 點擊 **保存** 然後 **繼續** 和 **保存** 再次

4. **創建密鑰**
   - 在左側菜單欄，點擊 **密鑰** 然後點擊 **+** 按鈕
   - 輸入密鑰名稱（例如，"Store OAuth Key"）
   - 勾選 **使用Apple登錄**
   - 點擊 **配置** 並選擇您的主應用ID
   - 點擊 **保存**，然後 **繼續** 和 **註冊**
   - **下載密鑰文件** (.p8) — 您不能再次下載它

5. **收集所需信息**
   您需要：
   - **客戶端ID**（服務ID）：您創建的標識符（例如，`com.yourstore.login`）
   - **團隊ID**：在Apple開發者門戶右上角找到
   - **密鑰ID**：創建密鑰時顯示的
   - **私鑰**：您下載的.p8文件的內容

6. **在Spwig中輸入憑證**
   - 返回到Spwig OAuth設置
   - 黏貼客戶端ID、團隊ID和密鑰ID
   - 在文本編輯器中打開.p8文件並複製其內容
   - 將整個密鑰（包括標頭）粘貼到私鑰字段中
   - 點擊 **保存**
   - 切換 **啟用Apple OAuth** 以激活

### 測試

- 在具有Apple ID的設備上訪問您的商店前端登錄頁面
- 點擊"使用Apple登錄"
- 使用Apple ID進行身份驗證
- 您應該會成功登錄

## 設置Microsoft OAuth

Microsoft OAuth對於針對使用Office 365或Azure AD的企業客戶的商店來說是理想的。

### 先決條件

- 一個Microsoft賬戶
- 可訪問Azure Portal

### 分步設置

1. **導航到OAuth設置**
   - 前往 **設置 > 商店設置 > OAuth提供商**
   - 點擊 **配置Microsoft**

2. **在Azure中註冊應用程序**
   - 訪問 [Azure Portal](https://portal.azure.com/)
   - 前往 **Azure Active Directory > 應用程序註冊**
   - 點擊 **新註冊**
   - 輸入名稱（例如，"Store OAuth"）
   - 選擇 **任何組織目錄中的賬戶和個人Microsoft賬戶**
   - 在 **重定向URI** 下，選擇 **Web** 並輸入：
     ```
     https://yourdomain.com/accounts/microsoft/login/callback/
     ```
   - 點擊 **註冊**

3. **複製應用程序ID**
   - 在應用程序概覽頁面上，複製 **應用程序（客戶端）ID**

4. **創建客戶端密鑰**
   - 在左側菜單欄，點擊 **證書與密鑰**
   - 點擊 **新客戶端密鑰**
   - 輸入描述（例如，"OAuth密鑰"）
   - 選擇過期時間（建議：24個月）
   - 點擊 **添加**
   - **立即複製密鑰值** — 它將不再顯示

5. **在Spwig中輸入憑證**
   - 返回到Spwig OAuth設置
   - 將應用程序（客戶端）ID粘貼為客戶端ID
   - 將密鑰值粘貼為客戶端密鑰
   - 可選地輸入租戶ID（對於單租戶應用程序；多租戶應用程序請留空）
   - 點擊 **保存**
   - 切換 **啟用Microsoft OAuth** 以激活

### 測試

- 訪問您的商店前端登錄頁面
- 點擊"使用Microsoft登錄"
- 使用Microsoft賬戶進行身份驗證
- 您應該會成功登錄

## 管理OAuth連接

### 客戶視圖

客戶可以從其賬戶儀表板查看和管理他們連接的OAuth提供商：

- 前往 **我的賬戶 > 連接的賬戶**
- 查看已連接的提供商（Google、Apple、Microsoft）
- 點擊 **斷開連接** 來斷開一個提供商
- 通過再次使用該提供商登錄來重新連接

### 多個提供商

一個客戶賬戶可以連接到多個OAuth提供商。例如，一個客戶可以將Google和Apple連接到同一個賬戶。

如果一個客戶嘗試使用相同的電子郵件地址和不同的OAuth提供商登錄，Spwig會自動將其連接到他們現有的賬戶。

### 管理員管理

作為管理員，您可以查看客戶的OAuth連接：

- 前往 **客戶 > 客戶**
- 打開一個客戶記錄
- 滾動到 **連接的賬戶** 部分
- 查看已連接的提供商以及連接時間

您不能代表客戶斷開提供商——出于安全原因，他們必須自己操作。

## 故障排除

### 重定向URI不匹配

**錯誤**："重定向URI不匹配"或"無效的redirect_uri"

**解決方案**：
- 確保提供商設置中的重定向URI與Spwig中的完全一致
- 檢查是否有尾隨斜杠——必須匹配
- 確認您使用的是`https://`（而不是`http://`）
- 清除瀏覽器緩存並重試

### 無效的憑證

**錯誤**："無效的客戶端ID"或"身份驗證失敗"

**解決方案**：
- 再次確認您正確複製了客戶端ID和客戶端密鑰
- 確保沒有額外的空格或換行符
- 確認憑證來自正確的項目/應用
- 對於Apple，確保私鑰包含.p8文件的全部內容

### 提供商API未啟用

**錯誤**："API未啟用"或"訪問未配置"

**解決方案**：
- 對於Google：確保您在Google Cloud項目中啟用了Google+ API
- 對於Microsoft：確認您的應用程序註冊已獲批准並處於活動狀態
- 對於Apple：檢查您的服務ID是否啟用了"使用Apple登錄"

### 需要SSL

**錯誤**："OAuth需要HTTPS"或"不安全的重定向URI"

**解決方案**：
- OAuth提供商需要SSL/TLS（HTTPS）以確保安全
- 確保您的商店安裝了有效的SSL證書
- 更新您的重定向URI以使用`https://`而不是`http://`
- 如果在本地測試，使用像ngrok這樣的服務創建HTTPS隧道

### 按鈕未顯示

**問題**："使用Google/Apple/Microsoft登錄"按鈕未出現在登錄頁面

**解決方案**：
- 確認OAuth設置中已啟用提供商
- 清除瀏覽器緩存並刷新頁面
- 檢查您的主題是否包含社交登錄模板
- 查看瀏覽器控制台是否有JavaScript錯誤

## 小技巧與最佳實踐

### 安全性

- **定期輪換密鑰** — 每12-24個月更新一次客戶端密鑰
- **監控失敗的登錄嘗試** — 留意異常的身份驗證模式
- **每個環境使用不同的憑證** — 隨機生成測試和生產環境的憑證
- **限制重定向URI** — 只添加您需要的確切URI

### 用戶體驗

- **啟用所有三個提供商** — 給客戶選擇的自由；不同的人群可能偏好不同的提供商
- **突出顯示按鈕** — 社交登錄按鈕應位於電子郵件/密碼表單上方
- **使用可識別的品牌** — 保持Google/Apple/Microsoft標準的按鈕樣式
- **在移動設備上測試** — 移動瀏覽器上的OAuth流程與桌面瀏覽器不同

### 合規性

- **隱私政策** — 公開說明您使用OAuth提供商以及您接收的數據
- **服務條款** — 遵守提供商的條款（Google、Apple、Microsoft各有要求）
- **數據最小化** — 只請求您實際需要的個人資料信息

### 測試清單

在上線前，測試：

- [ ] 使用每個提供商在桌面端登錄
- [ ] 使用每個提供商在移動端登錄
- [ ] 首次登錄（賬戶創建）
- [ ] 後續登錄（賬戶連接）
- [ ] 使用相同電子郵件在不同提供商上登錄
- [ ] 斷開並重新連接提供商
- [ ] 非OAuth用戶的密碼重置流程仍然有效
