---
title: Атрибуты товаров
---

Атрибуты товаров определяют измерения, по которым товар может различаться — например, Размер, Цвет или Материал. После создания атрибута и его возможных значений вы можете назначить его любому переменному товару, и Spwig сгенерирует селектор вариантов, который клиенты используют при оформлении заказа.

Перейдите в **Каталог > Атрибуты товаров**, чтобы управлять атрибутами и их значениями.

## Как работают атрибуты

Атрибуты можно повторно использовать во всем вашем каталоге. Вы создаете их один раз и назначаете столько товаров, сколько нужно. Каждый атрибут имеет:

- **Имя**, которое его идентифицирует (например, "Размер")
- **Тип отображения**, который контролирует, как будет отображаться селектор на странице товара
- Одно или несколько **значений**, представляющих доступные варианты (например, "Маленький", "Средний", "Большой")

Когда вы назначаете атрибут товару, вы также указываете, какие из его значений доступны для этого конкретного товара. Это означает, что атрибут "Размер" может иметь значения от S до 3XL, но конкретная футболка может предлагать только S, M и L.

## Типы отображения атрибутов

Поле **Тип** на атрибуте контролирует, как будет отображаться элемент выбора на странице товара в вашем интернет-магазине:

| Тип | Внешний вид | Лучше подходит для |
|---|---|---|
| **Список с выпадающим меню** | Выпадающее меню, которое открывает клиент для выбора значения | Атрибуты с множеством значений (например, диапазон размеров с 10+ размерами) |
| **Цветовые образцы** | Цветные круги или квадраты, которые кликабельны | Атрибуты цвета, где визуальная идентификация помогает |
| **Группа кнопок** | Круглые кнопки, отображаемые в строку | Атрибуты с небольшим количеством значений (например, S, M, L, XL) |
| **Переключатели** | Традиционный список переключателей | Любой атрибут, где вы хотите, чтобы список был понятным и доступным |

Выберите тип отображения, который соответствует тому, как ваши клиенты воспринимают атрибут. Для цвета образцы почти всегда лучше, чем выпадающий список. Для размера группы кнопок работают хорошо, если количество вариантов меньше 8.

## Создание атрибута

1. Перейдите в **Каталог > Атрибуты товаров**
2. Нажмите **+ Добавить атрибут товара**
3. Введите **Имя** (например, `Размер`, `Цвет`, `Материал`)
4. **Slug** заполняется автоматически — вы можете оставить его как есть
5. Выберите **Тип** (Список с выпадающим меню, Цветовые образцы, Группа кнопок или Переключатели)
6. Поставьте галочку **Обязательно** если клиенты должны выбрать этот атрибут, прежде чем добавить товар в корзину — это подходит для большинства атрибутов размера и цвета
7. Установите **Порядок сортировки** — атрибуты с меньшими числами отображаются первыми в селекторе вариантов на странице товара
8. Добавьте значения атрибута напрямую в раздел **Значения** (см. ниже)
9. Нажмите **Сохранить**

## Добавление значений атрибута

Значения атрибута — это отдельные варианты внутри атрибута. Вы можете добавить их напрямую при создании или редактировании атрибута, используя форму значений в нижней части страницы деталей атрибута.

Для каждого значения:

- **Значение** — метка отображения (например, `Маленький`, `Красный`, `Хлопок`)
- **Slug** — заполняется автоматически из значения; используется в URL и идентификаторах вариантов
- **Цвет в HEX** — относится только к атрибутам типа **Цветовые образцы**. Введите код цвета в формате HEX (например, `#FF0000` для красного), чтобы образец отображал правильный цвет.
- **Порядок сортировки** — контролирует порядок, в котором значения отображаются в селекторе. Назначьте меньшие числа тем значениям, которые вы хотите отобразить первыми.

### Логическая сортировка значений

Для атрибутов размера установите порядок сортировки так, чтобы размеры шли от малого к большему:

| Значение | Порядок сортировки |
|---|---|
| XS | 1 |
| S | 2 |
| M | 3 |
| L | 4 |
| XL | 5 |
| 2XL | 6 |

Для атрибутов цвета вы можете сортировать их алфавитно или группировать похожие цвета — что бы ни было наиболее логичным для ваших клиентов.

## Управление значениями атрибутов отдельно

Вы также можете управлять значениями атрибутов независимо в **Каталог > Значения атрибутов**. Этот список полезен, когда вам нужно найти или обновить конкретное значение по всему каталогу, не открывая каждый атрибут отдельно. Список можно фильтровать по имени атрибута.

## Назначение атрибутов товарам

Атрибуты назначаются на уровне продукта, а не глобально.

Чтобы добавить атрибут к продукту:

1. Перейдите в **Каталог > Продукты** и откройте переменный продукт
2. Вкладка **Вариации**, найдите раздел **Атрибуты**
3. Выберите атрибут, который хотите добавить
4. Выберите, какие значения этого атрибута доступны для этого продукта
5. Сохраните продукт — Spwig сгенерирует соответствующие комбинации вариантов

Для подробного руководства по настройке вариантов продукта см. раздел **Варианты продукта** в справке.

## Практические примеры

### Пример: Атрибут размера одежды

| Поле | Значение |
|---|---|
| Название | Размер |
| Тип | Группа кнопок |
| Обязательное | Да |
| Порядок сортировки | 1 |
| Значения | XS (1), S (2), M (3), L (4), XL (5), 2XL (6) |

### Пример: Атрибут цветовой палитры

| Поле | Значение |
|---|---|
| Название | Цвет |
| Тип | Цветовая палитра |
| Обязательное | Да |
| Порядок сортировки | 2 |
| Значения | Черный (#000000), Белый (#FFFFFF), Синий (#001F5B), Красный (#CC0000) |

### Пример: Атрибут материала

| Поле | Значение |
|---|---|
| Название | Материал |
| Тип | Список с выпадающим меню |
| Обязательное | Нет |
| Порядок сортировки | 3 |
| Значения | 100% Хлопок, Смесь хлопка/полиэстера, Мериносовая шерсть, Лен |

## Советы

- Создавайте атрибуты, которые представляют реальные решения покупателей — если покупателям не нужно выбирать, это, возможно, не нужно делать атрибутом
- Используйте последовательное названия в вашем каталоге: если некоторые продукты используют «Цвет» и другие «Color», покупателям и вашей команде будет сложно понять несоответствие
- Порядок сортировки как у атрибутов, так и у значений имеет значение — поставьте наиболее важный атрибут первым (обычно Размер или Цвет) и упорядочьте значения логической последовательностью
- Тип Цветовая палитра требует точных шестнадцатеричных кодов; протестируйте цвета в цветовом выборщике браузера перед сохранением, чтобы убедиться, что палитра соответствует реальному цвету продукта
- Если вам нужно переименовать атрибут (например, с «Color» на «Colour»), обновите поле **Название**, а не создавайте новый атрибут — изменение названия не влияет на существующие назначения продуктов
