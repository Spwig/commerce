---
title: 판매 지역
---

판매 지역은 가게의 지리적 시장 영역을 정의하고 각 지역에서 제공할 제품을 제어할 수 있게 해줍니다. 이 기능은 여러 국가나 지역에서 제품을 판매하고, 지역별 통화, 제품 카탈로그, 재고 가용성을 다르게 관리해야 할 때 유용합니다.

## 판매 지역이란 무엇인가요?

판매 지역은 하나 이상의 국가로 구성된 이름이 지정된 지리적 영역입니다. 각 지역에는 기본 통화, 우선순위, 그리고 하나 이상의 창고와 연결될 수 있습니다. 고객이 가게를 탐색할 때 Spwig는 고객의 위치를 기반으로 지역을 결정하고 적절한 통화와 제품 가시성 규칙을 적용합니다.

일반적인 사용 사례:
- 각 국가의 고객에게 해당 지역에서만 제공 가능한 제품만 표시
- 지역별 기본 통화 설정 (예: 뉴질랜드 고객에게 NZD)
- 각 지역의 주문을 처리할 창고를 제어
- 특정 시장에서 아직 제공되지 않은 제품을 숨김

## 판매 지역 생성

1. **카탈로그 > 판매 지역**으로 이동
2. **+ 판매 지역 추가**를 클릭
3. 지역 정보를 입력합니다:

| 필드 | 설명 | 예시 |
|-------|-------------|---------|
| **지역 이름** | 이 지역의 표시 이름 | `아시아-태평양` |
| **지역 코드** | 이 지역의 짧고 고유한 식별자 | `APAC` |
| **국가** | 이 지역에 포함된 ISO 국가 코드 | `["NZ", "AU", "SG", "FJ"]` |
| **기본 통화** | 이 지역의 ISO 통화 코드 | `NZD` |
| **우선순위** | 우선순위가 높은 지역은 먼저 일치합니다 | `10` |
| **활성화** | 이 지역이 현재 사용 중인지 여부 | 체크됨 |

4. **저장**을 클릭

### 국가 코드

국가를 입력할 때 두 글자의 ISO 코드로 구성된 JSON 목록을 사용합니다. 예시:
- 뉴질랜드와 호주: `["NZ", "AU"]`
- 싱가포르만: `["SG"]`
- 유럽 전체: `["DE", "FR", "IT", "ES", "NL", "BE", "AT", "CH", "SE", "NO", "DK", "FI", "PL"]`

### 우선순위

고객의 국가가 하나 이상의 지역과 일치하는 경우, 우선순위 숫자가 높은 지역이 사용됩니다.

# 더 구체적인 지역에 우선순위를 높이기
더 구체적인 지역에 더 높은 우선순위를 설정하세요(예: `NZ`에 우선순위 20을, `APAC`에 우선순위 10을 설정하여 뉴질랜드 고객이 먼저 NZ 지역에 매칭되도록 합니다).

## 지역에 따른 제품 가시성 제어
기본적으로 모든 제품은 모든 지역에서 보입니다. 특정 지역에 제품을 제한하려면 **제품 지역 가시성** 기록을 사용하세요.

### 특정 지역에 제품을 제한
1. **카탈로그 > 제품 지역 가시성**으로 이동하세요
2. **+ 제품 지역 가시성 추가**를 클릭하세요
3. **제품**을 선택하세요
4. **지역**을 선택하세요
5. 필요에 따라 **가시성**을 켜거나 끄세요
6. **저장**을 클릭하세요

제품에 어떤 가시성 기록이 존재하면 Spwig은 규칙을 적용합니다. 가시성 기록이 없는 제품은 모든 지역에서 보입니다.

### 일반적인 패턴
**한 지역만 제한**
지원하려는 지역 수만큼 가시성 기록을 추가하고, 허용된 지역에 대해 **가시성**을 `예`로 설정하세요. 다른 지역에 있는 고객은 제품을 보지 못합니다.

**한 지역에서 제외**
제외하려는 지역에 대한 단일 가시성 기록을 추가하고, **가시성**을 `아니요`로 설정하세요. 제품은 다른 모든 지역에서 보입니다.

### 제품 페이지에서 가시성을 편집
제품 편집 폼에서 직접 지역 가시성을 관리할 수도 있습니다. 제품의 **지역 가시성** 섹션에서 해당 제품의 모든 지역과 그에 따른 가시성 설정을 보여주는 인라인 테이블을 찾을 수 있습니다.

## 지역별 통화
각 지역에는 기본 통화가 있습니다. 해당 지역에서 브라우징하는 고객은 지역의 통화로 가격을 보게 됩니다. 사용되는 통화는 체크아웃 시 결정됩니다.

여러 통화에서 가격을 설정하려면 **설정 > 환율**에서 환율을 구성하세요. 가격은 자동으로 변환되거나 통화별로 수동으로 설정할 수 있습니다.

## 창고를 지역에 연결하기
창고는 **카탈로그 > 창고**에서 창고를 생성하거나 편집할 때 지역에 연결됩니다.

# 각 창고는 하나의 지역에 속하며, 이는 주문을 처리하기 위해 어떤 지역의 재고가 사용되는지를 결정합니다.

창고에 대한 자세한 내용은 **재고 및 창고** 도움말 주제를 참조하십시오.

## 팁

- 지역 코드는 짧고 설명적이어야 합니다(`NZ`, `APAC`, `EU`, `US`) — 이는 내부 및 로그에서 사용됩니다.
- 더 작고 구체적인 지역에 더 높은 우선순위 숫자를 사용하여, 더 넓은 포괄적인 지역보다 우선시되도록 합니다.
- 한 국가만 판매하는 경우 지역을 구성할 필요가 없습니다 — Spwig은 단일 글로벌 카탈로그와 함께 잘 작동합니다.
- 관리자에서 특정 지역으로 필터링하여 가상으로 상점을 미리보며 지역 기반 가시성을 테스트할 수 있습니다.
- 제품 가시성 기록은 제품을 제한하고자 할 때만 생성해야 합니다. 제품에 가시성 기록이 없는 경우 제품은 전 세계적으로 사용 가능합니다.
- 새로운 지역을 추가할 때마다 기존 제품 제한이 올바른지 확인하기 위해 가시성 규칙을 검토하십시오.
