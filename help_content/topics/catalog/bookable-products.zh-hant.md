---
title: 可預訂商品
---

可預訂商品讓客戶在購買時預約特定的日期和時間。這支援預約、租賃、課程、活動和住宿預訂——所有內容均可直接在您的 Spwig 管理介面中管理。

## 預訂類型

| 類型 | 最適合用於 |
|------|----------|
| **預約** | 服務：諮詢、理髮、個人訓練 |
| **租賃** | 設備租賃、車輛租賃、房間租賃 |
| **課程 / 工作坊** | 有固定容量的團體課程 |
| **住宿** | 有多個夜晚的住宿，包含入住/退房時間 |
| **活動** | 有票務的一次性或重複活動 |

## 設置可預訂商品

### 第 1 步：建立商品

1. 點擊 **Products > All Products**，然後點擊 **+ Add Product**
2. 將 **Product Type** 設為 **Booking Product**
3. 填寫標準商品欄位（名稱、描述、價格）
4. 儲存商品

### 第 2 步：配置預訂設定

儲存後，商品編輯表單上會出現 **Booking Configuration** 區塊。填寫預訂設定：

#### 預訂類型和時長

- **Booking Type** — 選擇最符合您服務類型的選項（預約、租賃、課程等）
- **Duration Type** — 選擇 **Fixed Duration** 用於固定時長的課程，或選擇 **Customer Selects Duration** 讓客戶選擇他們需要的時長
- **Duration** 和 **Duration Unit** — 設定時長（例如，`60` 分鐘，`1` 小時，`2` 天）
- **Min/Max Duration** — 如果客戶可以選擇時長，請設定允許的範圍

#### 緩衝時間

預訂之間會自動加入緩衝時間，以進行準備或清潔：
- **Buffer Before** — 預訂開始前保留的分鐘數
- **Buffer After** — 預訂結束後保留的分鐘數

例如，一個 60 分鐘的按摩預約，預約後有 15 分鐘的緩衝時間，可以讓您有 15 分鐘為下一位客戶做準備。

#### 預訂提前時間窗口


- **最少預約提前時間** — 顧客必須預訂的提前時間（例如，`24 小時`，因此不允許當天預約）
- **最多預約範圍** — 顧客可以預訂的未來時間範圍（例如，`365 天`）

#### 容量

- **每個時段最多預約數** — 對於課程和活動，設定同一時段可以預訂的顧客數量。設定為 `1` 可用於私人預約。

#### 確認

- **需要手動確認** — 勾選後，預約不會自動確認。您必須從預約清單中手動批准每個預約。當您希望在確認前審核顧客時會很有用。

#### 取消政策

- **允許取消** — 顧客是否可以取消預約
- **取消截止時間** — 顧客可以在預約前多少小時/天取消（例如，`24 小時`）

#### 日曆顯示

顧客在產品頁面上如何選擇日期和時間：

| 顯示模式 | 最佳用途 |
|---------|---------|
| **日曆視圖** | 一般用途 — 完整的月曆 |
| **日期選擇器** | 簡單的單日期選擇 |
| **可用日期下拉選單** | 可用性時段有限的產品 |
| **日期範圍選擇器** | 住宿和多日租賃 |

#### 抵押金

若要在結帳時要求押金而非全額付款：
1. 勾選 **啟用押金**
2. 將 **押金類型** 設為 **固定金額** 或 **總金額的百分比**
3. 輸入 **押金金額**（例如，`50` 表示 $50，或 `25` 表示 25%）

#### 住宿專用設定

對於住宿預約，會顯示額外欄位：
- **入住時間** 和 **退房時間** — 財產的標準時間
- **標準入住人數** — 基準價格中包含的預設客人數量

### 第 3 步：新增預約資源（可選）

資源是會分配給預約的實體物品或員工 — 例如，"房間 1"、"場地 A" 或 "教練 Sam"。

1.

在產品編輯表單中，前往 **預約資源** 區域
2.

點擊 **新增資源**
3.


為資源指定 **名稱**，並設置其 **容量**（可同時處理的預訂數量）
4.

選擇性地添加資源圖片

資源讓您可以根據單個資產或員工追蹤可用性，而不僅僅是根據時間段。

### 第 4 步：設置可用性規則

可用性規則定義了可以進行預訂的時間：

1. 在產品的 **可用性** 區域，點擊 **新增可用性規則**
2. 選擇此規則適用的 **資源**
3. 設置可用的 **週幾**
4. 設置可用時間段的 **開始時間** 和 **結束時間**
5. 選擇性地設置日期範圍（**生效日期** / **截止日期**）以處理季節性可用性
6. 儲存

## 查看和管理預訂

### 預訂清單

導航至 **目錄 > 預訂** 以查看所有預訂。您可以按以下條件篩選：
- 狀態（待確認、已確認、已取消、已完成、未到場）
- 產品
- 日期範圍

### 預訂狀態

| 狀態 | 含義 |
|------|------|
| **待確認** | 等待手動批准（如果需要確認） |
| **已確認** | 預訂已確認且處於活動狀態 |
| **已取消** | 預訂已被客戶或您取消 |
| **已完成** | 預訂日期已過且已完成 |
| **未到場** | 客戶未出席 |

### 確認待確認的預訂

1. 從 **目錄 > 預訂** 開啟預訂
2. 將 **狀態** 改為 **已確認**
3. 儲存 — 客戶會自動收到確認電子郵件

### 取消預訂

1. 開啟預訂
2. 將 **狀態** 改為 **已取消**
3. 輸入 **取消原因**（會顯示在客戶的電子郵件中）
4. 儲存

## 管理等候名單

當時間段已滿時，客戶可以將自己加入等候名單。Spwig 會在取消導致出現空位時自動通知等候名單上的客戶。

### 查看等候名單

導航至 **目錄 > 預訂等候名單** 以查看所有等候名單條目。

每個項目顯示：
- 顧客姓名和電子郵件
- 產品和期望日期
- 狀態：**等待中**、**已通知**、**轉為預訂** 或 **已過期**

### 等候名單狀態

| 狀態 | 含義 |
|--------|---------|
| **等待中** | 顧客已排隊，尚未有可用時段 |
| **已通知** | 顧客已收到有關可用時段的電子郵件 |
| **轉為預訂** | 顧客已佔用時段並完成預訂 |
| **已過期** | 期望日期已過，但沒有可用時段 |

### 手動通知等候名單上的顧客

如果您想在自動通知之前聯繫特定的等候名單顧客：
1. 打開等候名單項目
2. 複製他們的電子郵件地址並直接聯繫他們
3. 一旦他們完成預訂，他們的等候名單項目狀態會更新為 **轉為預訂**

## 小技巧

- 對於高價值預訂（例如攝影課程、私人活動），啟用手動確認，這樣您可以在承諾之前檢查可用性並匹配需求。
- 起始時請設定充足的緩衝時間 — 一旦您了解實際的週轉需求，總是可以減少它。
- 對於團體課程，請將 **每個時段的最大預訂數** 設為課程容量，並啟用等候名單，這樣受歡迎的課程會自動建立隊伍。
- 對於住宿產品，請使用日期範圍選擇器顯示模式 — 顧客期望一起選擇到達和離開日期。
- 設定最低預訂通知時間，以防止最後一刻的預訂，如果您需要準備時間（例如自訂餐飲訂單的 48 小時最低預訂通知）。
- 在旺季時定期查看您的等候名單 — 手動聯繫等候名單上的顧客可以比自動通知更快填補取消的預訂。
