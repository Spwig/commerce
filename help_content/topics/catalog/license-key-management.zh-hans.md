---
title: 许可证密钥管理
---

许可证密钥管理让您能够控制软件许可证密钥的生成、存储以及在客户购买数字产品时的交付方式。Spwig 支持内置的密钥生成、预加载的密钥池以及与外部许可证管理服务的集成。

## 概览

在 Spwig 中有三种管理许可证密钥的方法：

| 方法 | 最适合 | 
|--------|---------| 
| **许可证模板** | 在购买时自动生成符合自定义格式的唯一密钥 | 
| **许可证池** | 提前批量生成密钥以进行批量分发 | 
| **外部提供商** | 将密钥生成和管理委托给第三方服务，如 Keygen.sh | 

这些方法可以结合使用——例如，一个池可以使用自定义模板来定义密钥格式，并可选择将生成的密钥同步到外部提供商。

## 许可证密钥模板

许可证密钥模板定义了生成密钥的 *格式*。模板使用带有占位符的模式，Spwig 在生成时会填充这些占位符。

### 创建模板

1. 导航到 **目录 > 许可证密钥模板**
2. 点击 **+ 添加许可证密钥模板**
3. 输入一个 **名称**（例如，`Standard App License`）
4. 使用占位符配置 **模式**（见下文）
5. 如需设置 **前缀** 和 **后缀**（例如，前缀为 `MYAPP` 会为每个密钥添加 `MYAPP-`）
6. 选择 **分隔符** 字符（默认：`-`）
7. 设置 **字符集** —— 用于随机部分的字符。默认值排除了容易混淆的字符，如 `0` 和 `O`、`1` 和 `I`。
8. 设置 **最小/最大长度** 以进行验证
9. 点击 **保存**

### 模式占位符

保留所有 markdown 格式、图片路径、代码块和专业术语。

| 占位符 | 描述 | 示例输出 |
|-------------|-------------|---------------|
| `{RANDOM:N}` | 从字符集中的 N 个随机字符 | `{RANDOM:5}` → `K7JXQ` |
| `{CHECKSUM:N}` | 验证用的 N 位校验和 | `{CHECKSUM:2}` → `47` |
| `{PREFIX}` | 模板的前缀值 | `MYAPP` |
| `{SUFFIX}` | 模板的后缀值 | `PRO` |
| `{ORDER_ID}` | 订单号 | `10045` |
| `{PRODUCT_SKU}` | 产品的 SKU | `SOFTPRO` |
| `{DATE:FORMAT}` | 格式化的日期 | `{DATE:YYMMDD}` → `260318` |

**示例模式**: `{PREFIX}-{RANDOM:5}-{RANDOM:5}-{RANDOM:5}-{CHECKSUM:2}`

这会生成类似以下的密钥: `MYAPP-K7JXQ-M3TPR-9BWKN-47`

| 字段 | 描述 |
|-------|-------------|
| **名称** | 描述性名称（例如，`Retail Pack Q1 2026`） |
| **产品** | 这些密钥对应的产品 |
| **许可证模板** | 密钥格式的模板（默认使用产品的模板） |
| **总密钥数** | 要生成的密钥数量 |
| **密钥类型** | 永久、订阅或试用 |
| **最大激活次数** | 每个密钥可以激活的设备数量 |
| **激活后过期天数** | 从首次激活起许可证过期的天数（留空表示不过期） |
| **池过期时间** | 此池中未使用的密钥在此日期之后将失效 |
| **同步到提供商** | 可选地将生成的密钥同步到外部许可证提供商 |

4. 点击 **保存** —— Spwig 开始在后台生成密钥

### 池状态

| 状态 | 含义 |
|--------|---------|
| **生成中** | 密钥正在后台创建 |
| **已就绪** | 所有密钥已生成并可供分发 |
| **已耗尽** | 所有密钥已分配给订单 |
| **已过期** | 池的过期日期已过 |

### 监控池

池列表显示已分发的密钥数量与生成的总密钥数量。打开一个池以查看完整的密钥列表及其各自的狀態。

## 外部许可证提供商

外部提供商是第三方许可证管理服务，负责处理密钥生成和激活跟踪。当客户完成购买时，Spwig 会与提供商通信以生成并注册密钥。

### 支持的提供商

| 提供商 | 类型 |
|----------|------|
| **Spwig 内置许可证服务器** | 内置 —— 无需外部账户 |
| **Keygen.sh** | 基于云的许可证管理 API |
| **LicenseSpring** | 企业许可证管理 |
| **Cryptlex** | 支持离线的许可证管理 |
| **自定义 API** | 任何基于 REST 的许可证系统 |

### 连接提供商

1.

导航到 **目录 > 许可证提供商**
2.

点击 **+ 添加许可证提供商**
3.


填写提供商详细信息：

| 字段 | 描述 |
|-------|-------------|
| **名称** | 为此连接指定一个标签（例如，`Keygen Production`） |
| **提供商类型** | 从支持的提供商中选择 |
| **API 端点** | 提供商的 API 基础 URL |
| **API 密钥** | 提供商的认证密钥 |
| **API 密钥** | 如果提供商需要的话 |

4. 配置同步行为：
   - **在订单时同步** — 当客户完成购买时自动同步
   - **在激活时同步** — 向提供商报告设备激活
   - **在停用时同步** — 报告停用（适用于许可证转移和退款）
   - **双向同步** — 允许提供商通过 Webhook 更新 Spwig 记录

5. 点击 **保存**，然后点击 **测试连接** 以验证凭据是否有效

### 连接状态

每个提供商显示三种连接状态之一：

| 状态 | 含义 |
|--------|---------|
| **未测试** | 连接尚未验证 |
| **已连接** | 上次测试成功 |
| **错误** | 连接测试失败 — 请检查错误信息 |

### 同步现有许可证

要手动将现有许可证密钥推送到提供商（用于初始设置或失败同步后），请从提供商列表中使用 **现在同步** 操作。

## 监控同步活动

导航到 **目录 > 外部许可证同步** 以查看同步日志。每条记录显示：
- 被同步的许可证密钥
- 发送到的提供商
- 方向（Spwig → 提供商 或 提供商 → Spwig）
- 状态（待处理、成功、失败）
- 失败同步的错误详情

失败的同步会自动重试。您也可以通过编辑记录并清除错误来强制重试。

## 小贴士

保留所有 Markdown 格式、图片路径、代码块和专业术语。

- 使用默认字符集 (`ABCDEFGHJKLMNPQRSTUVWXYZ23456789`) 以避免客户经常误读的模糊字符 — 它排除了 `0`、`O`、`1` 和 `I`。
- 在您的模板模式中添加 `{CHECKSUM}` 段，以便客户和支持团队可以快速检测到拼写错误的密钥。
- 对于高销量产品，使用池而不是按需生成，以确保结账时密钥可以立即交付。
- 对于季节性或限时密钥批次，设置 **Pool Expires At**，以便自动使旧的未使用的密钥失效。
- 在设置后以及任何凭证更改后，始终测试提供者的连接 — 断开的连接意味着客户无法收到他们的密钥。
- 如果使用双向同步，请将提供者的 webhook URL 配置为指向您商店的许可证 webhook 端点。
