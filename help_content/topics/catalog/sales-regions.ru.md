---
title: Регионы продаж
---

Регионы продаж позволяют определить географические рынки для вашего магазина и управлять тем, какие товары доступны в каждом регионе. Это полезно, когда вы продаете в нескольких странах или территориях и вам нужны разные каталоги товаров, региональные валюты или наличие товара в зависимости от локации.

## Что такое регион продаж?

Регион продаж — это названная географическая область, состоящая из одной или нескольких стран. Каждый регион имеет валюту по умолчанию, приоритет и может быть связан с одним или несколькими складами. Когда клиент просматривает ваш магазин, Spwig определяет его регион на основе его местоположения и применяет соответствующие правила валюты и видимости товаров.

Общие случаи использования:
- Отображение товаров, доступных только в определенной стране, для клиентов этой страны
- Назначение валюты по умолчанию, специфичной для региона (например, NZD для клиентов Новой Зеландии)
- Управление тем, какие склады выполняют заказы для каждого региона
- Скрытие товаров, которые еще недоступны в определенных регионах

## Создание региона продаж

1. Перейдите в **Каталог > Регионы продаж**
2. Нажмите **+ Добавить регион продаж**
3. Заполните детали региона:

| Параметр | Описание | Пример |
|---------|---------|--------|
| **Название региона** | Отображаемое имя этого региона | `Asia-Pacific` |
| **Код региона** | Короткий уникальный идентификатор | `APAC` |
| **Страны** | Коды стран ISO, включенные в этот регион | `["NZ", "AU", "SG", "FJ"]` |
| **Валюта по умолчанию** | Код валюты ISO для этого региона | `NZD` |
| **Приоритет** | Регионы с более высоким приоритетом сопоставляются первыми | `10` |
| **Активен** | Определяет, используется ли этот регион в данный момент | Отмечено |

4. Нажмите **Сохранить**

### Коды стран

Введите страны в виде JSON-списка из двухбуквенных кодов ISO. Например:
- Новая Зеландия и Австралия: `["NZ", "AU"]`
- Только Сингапур: `["SG"]`
- Все Европы: `["DE", "FR", "IT", "ES", "NL", "BE", "AT", "CH", "SE", "NO", "DK", "FI", "PL"]`

### Приоритет

Если страна клиента совпадает с несколькими регионами, используется регион с самым высоким приоритетом. Установите более высокий приоритет для более конкретных регионов (например, назначьте `NZ` приоритет 20 и `APAC` приоритет 10, чтобы клиенты Новой Зеландии сначала сопоставлялись с регионом NZ).

## Управление видимостью товаров по регионам

По умолчанию каждый товар виден во всех регионах. Чтобы ограничить товар определенными регионами, используйте записи **Видимость региона товара**.

### Ограничение товара определенными регионами

1. Перейдите в **Каталог > Видимость региона товара**
2. Нажмите **+ Добавить видимость региона товара**
3. Выберите **Товар**
4. Выберите **Регион**
5. Установите **Видимость** включенной или выключенной в зависимости от необходимости
6. Нажмите **Сохранить**

Как только для товара существует любая запись видимости, Spwig применяет правила. Товары без записей видимости остаются видимыми везде.

### Общие сценарии

**Ограничение только одним регионом**

Добавьте одну запись видимости на каждый регион, который вы хотите поддерживать, установив **Видимость** в `Да` для разрешенных регионов. Клиенты из других регионов не увидят товар.

**Исключение из одного региона**

Добавьте одну запись видимости для региона, который вы хотите исключить, и установите **Видимость** в `Нет`. Товар остается видимым во всех других регионах.

### Редактирование видимости с страницы товара

Вы также можете управлять видимостью региона напрямую с формы редактирования товара. В разделе **Видимость региона** товара вы найдете встроенную таблицу, показывающую все регионы и настройки видимости для этого товара.

## Валюта региона

Каждый регион имеет валюту по умолчанию. Клиенты, просматривающие магазин из этого региона, видят цены, отображенные в валюте региона. Валюта, используемая при оформлении заказа, определяется на этапе оформления заказа.

Чтобы настроить цены в нескольких валютах, настройте курсы обмена в разделе **Настройки > Курсы обмена**. Цены могут автоматически конвертироваться или устанавливаться вручную для каждой валюты.

## Связь складов с регионами

Склады связываются с регионами, когда вы создаете или редактируете склад в разделе **Каталог > Склады**. Каждый склад принадлежит одному региону, который управляет наличием товара этого региона для выполнения заказов.

Для получения дополнительной информации о складах см. раздел **Инвентарь и склады** в справке.

## Советы

- Держите коды регионов короткими и описательными (`NZ`, `APAC`, `EU`, `US`) — они используются внутренне и в логах.
- Используйте более высокие числа приоритета для меньших и более конкретных регионов, чтобы они имели приоритет над более широкими регионами-заглушками.
- Если вы продаете только в одной стране, настройка регионов вовсе не требуется — Spwig отлично работает с одним глобальным каталогом.
- Проверяйте видимость по регионам, предварительно просмотрев магазин, фильтруя его по конкретному региону в административной панели.
- Записи видимости продуктов создаются только в том случае, если вы хотите ограничить доступ к продуктам. Оставление продукта без записей видимости делает его доступным для всех.
- Проверяйте свои правила видимости каждый раз, когда добавляете новый регион, чтобы убедиться, что существующие ограничения для продуктов корректны.
