---
title: ライセンスキーの管理
---

ライセンスキーの管理は、デジタル製品を購入した顧客に対して、ソフトウェアライセンスキーがどのように生成され、保存され、配布されるかを制御する機能です。Spwigは、組み込みのキー生成、事前に読み込まれたキーのプール、および外部のライセンス管理サービスとの統合をサポートしています。

## 概要

Spwigでは、ライセンスキーを管理するための3つの方法があります:

| 方法 | 最適な用途 |
|--------|---------|
| **ライセンステンプレート** | 購入時にカスタム形式で一意のキーを自動生成する |
| **ライセンスプール** | 事前にキーのバッチを事前に生成して、大量配布用に準備する |
| **外部プロバイダ** | Keygen.shなどの第三者サービスにキーの生成と管理を委譲する |

これらの方法は組み合わせて使用できます。例えば、プールはカスタムテンプレートを使用してキーの形式を定義し、生成されたキーを外部プロバイダに同期することも選択できます。

## ライセンスキーのテンプレート

ライセンスキーのテンプレートは、生成されたキーの*形式*を定義します。テンプレートは、Spwigが生成時に埋め込むプレースホルダーを使用したパターンを使用します。

### テンプレートの作成

1. **カタログ > ライセンスキーのテンプレート**に移動します
2. **+ ライセンスキーのテンプレートの追加**をクリックします
3. **名前**を入力します（例: `Standard App License`）
4. プレースホルダーを使用して**パターン**を構成します（下記を参照）
5. 必要に応じて**プレフィックス**と**サフィックス**を設定します（例: プレフィックス`MYAPP`はすべてのキーに`MYAPP-`を追加します）
6. **セパレータ**文字を選択します（デフォルト: `-`）
7. **文字セット**を設定します — ランダムなセグメントに使用される文字。デフォルトでは、`0`と`O`、`1`と`I`などの曖昧な文字は除外されます
8. 検証用に**最小/最大長**を設定します
9. **保存**をクリックします

### パターンのプレースホルダー

| Placeholder | Description | Example output |
|-------------|-------------|---------------|
| `{RANDOM:N}` | N random characters from the character set | `{RANDOM:5}` → `K7JXQ` |
| `{CHECKSUM:N}` | N-digit checksum for validation | `{CHECKSUM:2}` → `47` |
| `{PREFIX}` | The template's prefix value | `MYAPP` |
| `{SUFFIX}` | The template's suffix value | `PRO` |
| `{ORDER_ID}` | The order number | `10045` |
| `{PRODUCT_SKU}` | The product's SKU | `SOFTPRO` |
| `{DATE:FORMAT}` | Formatted date | `{DATE:YYMMDD}` → `260318` |

**Example pattern**: `{PREFIX}-{RANDOM:5}-{RANDOM:5}-{RANDOM:5}-{CHECKSUM:2}`

This produces keys like: `MYAPP-K7JXQ-M3TPR-9BWKN-47`

### Previewing keys

After saving a template, a **Generate Sample Key** action is available on the template list. Use this to verify your pattern produces keys in the expected format before assigning the template to a product.

## License pools

A license pool is a batch of pre-generated keys for a product. Pools are useful when:
- You need keys for physical packaging (retail boxes, printed cards)
- You work with resellers who need batches of keys
- You want keys generated in advance rather than on demand

### Creating a license pool

1. Navigate to **Catalog > License Pools**
2. Click **+ Add License Pool**
3. Fill in the pool details:

| フィールド | 説明 |
|-------|-------------|
| **名前** | 説明的な名前（例: `Retail Pack Q1 2026`） |
| **製品** | これらのキーが対象となる製品 |
| **ライセンステンプレート** | キーの形式のテンプレート（デフォルトは製品のテンプレート） |
| **合計キー数** | 生成するキーの数 |
| **キーの種類** | 永続的、サブスクリプション、またはトライアル |
| **最大アクティベーション数** | 各キーがアクティベートできるデバイスの数 |
| **アクティベーション後有効期限（日数）** | 最初のアクティベーションからライセンスが有効になる日数（有効期限なしの場合は空欄にします） |
| **プールの有効期限** | このプールの未使用キーが無効になる日付 |
| **プロバイダに同期** | 生成されたキーを外部のライセンスプロバイダに同期する（オプション） |

4. **保存**をクリック — Spwig がバックグラウンドでキーの生成を開始します

### プールの状態

| ステータス | 意味 |
|--------|---------|
| **生成中** | キーがバックグラウンドで作成されている |
| **準備完了** | すべてのキーが生成され、配布可能 |
| **枯渇** | すべてのキーが注文に割り当てられた |
| **期限切れ** | プールの有効期限日が過ぎた |

### プールの監視

プール一覧は、配布されたキー数と生成された合計キー数を表示します。プールを開くと、キーのフルリストと個々のステータスが表示されます。

## 外部ライセンスプロバイダ

外部プロバイダは、キーの生成とアクティベーションの追跡を処理するサードパーティのライセンス管理サービスです。顧客が購入を完了すると、Spwig はプロバイダと通信してキーを生成し、登録します。

### 対応プロバイダ

| プロバイダ | タイプ |
|----------|------|
| **Spwig 内蔵ライセンスサーバー** | 内蔵 — 外部アカウント不要 |
| **Keygen.sh** | クラウドベースのライセンス管理 API |
| **LicenseSpring** | エンタープライズライセンス管理 |
| **Cryptlex** | オフラインサポート付きライセンス管理 |
| **カスタム API** | 任意の REST ベースのライセンスシステム |

### プロバイダの接続

1.

**カタログ > ライセンスプロバイダ**に移動
2.

**+ ライセンスプロバイダの追加**をクリック
3.


プロバイダの詳細を入力してください：

| フィールド | 説明 |
|-------|-------------|
| **名前** | この接続のラベル（例：`Keygen Production`） |
| **プロバイダタイプ** | サポートされているプロバイダから選択 |
| **APIエンドポイント** | プロバイダのAPIベースURL |
| **APIキー** | プロバイダの認証キー |
| **APIシークレット** | プロバイダが要求している場合 |

4. 同期の動作を構成する：
   - **注文時に同期** — 顧客が購入を完了すると自動的に同期します
   - **アクティベーション時に同期** — デバイスのアクティベーションをプロバイダに報告します
   - **デアクティベーション時に同期** — デアクティベーションを報告します（ライセンスの転送や払い戻しに役立ちます）
   - **双方向同期** — プロバイダがWebhookを通じてSpwigのレコードを更新できるようにします

5. **保存**をクリックし、その後 **接続をテスト**をクリックして資格情報を確認してください

### 接続状態

各プロバイダは3つの接続状態のいずれかを示します：

| ステータス | 意味 |
|--------|---------|
| **テストされていない** | 接続がまだ確認されていません |
| **接続済み** | 最後のテストが成功しました |
| **エラー** | 接続テストに失敗しました — エラーメッセージを確認してください |

### 既存のライセンスの同期

既存のライセンスキーをプロバイダに手動でプッシュするには（初期設定または同期に失敗した後）、プロバイダリストから **今すぐ同期**アクションを使用してください。

## 同期アクティビティの監視

**カタログ > 外部ライセンス同期**に移動して、同期ログを確認してください。各レコードには以下が表示されます：
- 同期されたライセンスキー
- 送信されたプロバイダ
- 方向（Spwig → プロバイダ または プロバイダ → Spwig）
- ステータス（保留中、成功、失敗）
- 失敗した同期のエラー詳細

失敗した同期は自動的に再試行されます。また、レコードを編集してエラーをクリアすることによって、再試行を強制することもできます。

## ヒント

すべてのMarkdownの書式、画像のパス、コードブロック、および技術用語を保持してください。

- デフォルトの文字セット (`ABCDEFGHJKLMNPQRSTUVWXYZ23456789`) を使用して、顧客がよく誤読する曖昧な文字（`0`、`O`、`1`、`I`）を除外してください。
- テンプレートパターンに `{CHECKSUM}` セグメントを追加して、顧客およびサポートチームが誤って入力されたキーを迅速に検出できるようにしてください。
- 高量の商品の場合、オンデマンド生成ではなくプールを使用して、チェックアウト時にキーが即座に配信されるようにしてください。
- 季節限定または時間限定のキーのバッチでは、**Pool Expires At** を設定して、古い未使用のキーが自動的に無効になるようにしてください。
- 設定後および資格情報の変更後には常にプロバイダ接続をテストしてください — 接続が破損していると、顧客がキーを受信できなくなります。
- 双方向同期を使用する場合、プロバイダの webhook URL を、ストアのライセンス webhook エンドポイントに指向するように設定してください。
