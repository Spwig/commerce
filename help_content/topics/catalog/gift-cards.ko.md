---
title: 선물 카드
---

선물 카드는 고객이 선물로 보낼 수 있는 가상의 상품권을 구매할 수 있도록 해줍니다. 수신자는 이메일로 고유한 코드를 받아 결제 시에 사용할 수 있습니다.

![선물 카드 관리](/static/core/admin/img/help/gift-cards/gift-card-list.webp)

## 금액 유형

고객이 선물 카드 금액을 선택하는 방식을 제어합니다:

| 유형 | 설명 |
|------|-------------|
| **고정 금액** | 고객은 사전 설정된 금액(예: $25, $50, $100)을 선택합니다 |
| **사용자 정의 금액** | 고객은 최소/최대 범위 내의 금액을 입력할 수 있습니다 |
| **둘 다** | 사전 설정된 금액과 사용자 정의 금액 옵션을 함께 제공합니다 |

## 선물 카드 상품 생성

### 단계 1: 상품 설정

1. **상품 > 모든 상품**으로 이동하고 **+ 상품 추가**를 클릭합니다
2. **상품 유형**을 **선물 카드**로 설정합니다
3. 상품 이름과 설명을 입력합니다
4. 금액 설정을 구성합니다:
   - **금액 유형**을 선택합니다 (고정, 사용자 정의, 또는 둘 다)
   - 고정 금액인 경우 사용 가능한 금액을 설정합니다
   - 사용자 정의 금액인 경우 **최소** 및 **최대** 허용 금액을 설정합니다
5. **만료 일수**를 설정합니다 (0 = 만료되지 않음) — 이는 선물 카드가 구매 후 유효한 기간을 결정합니다
6. 상품을 저장하고 게시합니다

### 단계 2: 게시 및 판매

게시된 후, 선물 카드는 다른 상품과 마찬가지로 쇼핑몰에 표시됩니다. 고객은 이를 찾아볼 수 있으며, 금액을 선택하고 장바구니에 추가할 수 있습니다.

## 선물 카드 생명 주기

선물 카드는 다음과 같은 생명 주기를 따릅니다:

1. **구매** — 고객이 선물 카드 상품을 구매하고 수신자 정보를 제공합니다
2. **배송** — 수신자에게 선물 카드 코드가 포함된 이메일이 자동으로 전송됩니다
3. **사용** — 수신자가 체크아웃 시 코드를 입력하여 잔액을 적용합니다
4. **잔액 추적** — 각 사용 시 잔액이 차감되어 0에 도달할 때까지 유지됩니다

## 고객 구매 흐름

고객이 선물 카드를 구매할 때:

1. **금액 선택** — 사전 설정된 금액을 선택하거나 사용자 정의 금액을 입력합니다
2. **수신자 정보** — 수신자의 이메일 주소와 이름을 입력합니다
3. **개인 메시지** — 배송 이메일에 포함할 선택적 메시지를 추가합니다
4. **보내는 사람 이름** — 이메일에 보내는 사람의 이름을 제공합니다
5. **예약 배송** — 선택적으로 이메일을 미래의 날짜에 예약할 수 있습니다 (예: 생일)
6. **체크아웃** — 다른 상품과 마찬가지로 구매를 완료합니다

## 자동 배송

구매 후, 선물 카드는 자동으로 배송됩니다:

- 수신자에게 스타일화된 이메일이 전송되며 다음을 포함합니다:
  - 고유한 선물 카드 코드
  - 선물 카드의 금액
  - 보내는 사람의 개인 메시지
  - 잔액을 확인할 수 있는 링크
- 예약 배송이 설정된 경우, 이메일은 지정된 날짜와 시간에 전송됩니다
- 보내는 사람에게 선물 카드의 세부 정보가 포함된 주문 확인 이메일이 전송됩니다

## 관리자에서 선물 카드 관리

**상품 > 선물 카드**로 이동하여 모든 선물 카드를 관리합니다:

### 통계 대시보드

페이지 상단에 네 개의 카드가 주요 지표를 보여줍니다:

- **총 선물 카드** — 발행된 선물 카드의 총 수
- **활성** — 현재 활성화된 잔액이 있는 카드
- **총 잔액** — 모든 카드의 잔액 합계
- **부분적으로 사용됨** — 부분적으로 사용된 카드

### 필터

선물 카드를 다음과 같이 필터링할 수 있습니다:

- **검색** — 코드, 이메일 또는 수신자 이름으로 검색
- **상태** — 활성, 비활성, 만료, 전체 사용, 부분적으로 사용됨
- **잔액** — 잔액 있음 또는 잔액 없음
- **생성일** — 시간 기간 (오늘, 이번 주, 이번 달, 올해)

### 선물 카드 세부 정보

각 선물 카드는 다음과 같은 정보를 보여줍니다:

- **코드** — 고유한 사용 코드 (예: GC-XXXX-XXXX-XXXX)
- **수신자** — 이메일 및 이름
- **상태 배지** — 현재 상태와 색상 코드
- **잔액 / 초기 금액 / 사용 금액** — 사용 비율을 포함한 재정 요약
- **중요한 날짜** — 생성일, 발행일, 최초 사용일
- **보내는 사람** — 선물 카드를 구매한 사람

### 작업

각 선물 카드에 대해 다음과 같은 작업을 수행할 수 있습니다:

- **편집** — 선물 카드 세부 정보를 확인하고 수정합니다
- **거래 내역 보기** — 전체 거래 내역을 확인합니다
- **이메일 재발송** — 수신자에게 배송 이메일을 다시 전송합니다
- **비활성화** — 카드를 비활성화합니다 (잔액은 유지되지만 사용할 수 없습니다)

## 체크아웃 시 사용

고객이 체크아웃 시 선물 카드 코드를 입력할 때:

1. 코드가 검증됩니다 (활성, 만료되지 않음, 잔액이 있음)
2. 사용 가능한 잔액이 표시됩니다
3. 잔액이 주문 총액에 적용됩니다
4. 잔액이 주문 총액을 완전히 커버하는 경우 추가 결제가 필요하지 않습니다
5. 잔액이 주문 총액보다 적은 경우 고객은 나머지 금액을 지불해야 합니다
6. 거래가 기록되고 잔액이 업데이트됩니다

## 환불 처리

선물 카드를 사용한 주문을 환불할 때:

- **사용되지 않은 선물 카드** — 선물 카드를 완전히 비활성화합니다
- **부분적으로 사용된 카드** — 거래를 통해 잔액을 수동으로 조정해야 합니다
- **전액 환불** — 환불 거래를 통해 선물 카드 잔액에 금액을 다시 신용합니다

## 팁

- 지역 선물 카드 규정을 준수하기 위해 합리적인 만료 기간(예: 365일)을 설정하세요 — 일부 지역에서는 최소 유효 기간이 필요합니다.
- **둘 다** 금액 유형을 사용하여 편의성(사전 설정된 금액)과 유연성(사용자 정의 금액)을 제공하세요.
- 총 잔액 지표를 정기적으로 모니터링하세요 — 이는 귀하의 회계에서 미지급 부채를 나타냅니다.
- 계절적인 프로모션을 위해 예약 배송을 사용하세요 — 고객은 선물 카드를 미리 구매하고 정확한 날짜에 배송될 수 있습니다.
- 출시 전에 테스트 주문을 통해 전체 흐름(구매, 이메일 배송, 사용)을 테스트하세요.
- 여러 국가의 고객에게 판매하는 경우, 특정 통화로 선물 카드를 발행할 수 있습니다 — 자세한 내용은 **다국어 선물 카드** 도움말 주제를 참조하세요.
