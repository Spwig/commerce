---
title: Инвентарь и склады
---

Система складов позволяет управлять запасами в нескольких местах, устанавливать приоритеты выполнения заказов и отслеживать уровни запасов в режиме реального времени. Перейдите в **Настройки > Управление лицензиями** в боковой панели или получите доступ к складам через вкладку запасов товара.

![Список складов](/static/core/admin/img/help/inventory-warehouses/warehouse-list.webp)

## Склады

### Список складов

Страница складов отображает все ваши складские локации в виде карточек с:

- **Название и код** — Идентификатор склада (например, «Основной склад», код «MAIN-WH»)
- **Регион продаж** — Привязка к географическому региону
- **Индикаторы статуса** — Активный/неактивный, розничная точка
- **Статистика** — Количество товаров на складе, приоритет выполнения, процент буферного запаса
- **Местоположение** — Город и страна
- **Последнее обновление** — Когда уровни запасов были изменены в последний раз

### Создание склада

1. Нажмите **+ Добавить склад**
2. Заполните данные склада:
   - **Название** — Описательное наименование (например, «Склад Восточного побережья США»)
   - **Код** — Короткий уникальный идентификатор (например, «US-EAST»)
   - **Регион продаж** — Назначить географический регион для маршрутизации выполнения заказов
   - **Адрес** — Полный адрес склада для расчёта стоимости доставки
3. Настройте параметры:
   - **Активный** — Включить для участия в выполнении заказов
   - **Розничная точка** — Отметить, если этот склад также работает как физический магазин
   - **Приоритет выполнения** — Большие числа = более высокий приоритет для выполнения заказов
   - **Буферный запас** — Процент запасов, резервируемый как страховой буфер
4. Нажмите **Сохранить**

### Приоритет выполнения

При поступлении заказа система выбирает оптимальный склад на основе:

1. **Значение приоритета** — Склады с более высоким приоритетом предпочтительны
2. **Наличие запасов** — Должно быть достаточно товара
3. **Соответствие региона** — Склады в регионе покупателя предпочтительны

Например, если у вас есть склад в США (приоритет 100) и склад в ЕС (приоритет 60), заказы из США будут выполняться сначала со склада в США.

### Буферный запас

Буферный запас резервирует процент от инвентаря, который не будет продаваться онлайн. Это полезно для:
- Физических розничных магазинов, которым нужен выставочный товар
- Страхового запаса для предотвращения перепродажи
- Зарезервированного инвентаря для оптовых заказов

Буфер в 10% при 100 единицах означает, что только 90 единиц доступны для онлайн-заказов.

## Складские единицы

Складские единицы представляют фактические запасы конкретного товара на конкретном складе.

### Просмотр уровней запасов

1. Нажмите на **значок запасов** на любой карточке склада, чтобы увидеть складские единицы
2. Или перейдите на вкладку **Запасы** товара, чтобы увидеть запасы по всем складам

Каждая складская единица показывает:
- **Название товара** и вариант (если применимо)
- **В наличии** — Общий физический инвентарь
- **Распределено** — Количество, зарезервированное для ожидающих заказов
- **Доступно** — В наличии минус распределено (то, что можно продать)

### Добавление запасов

1. В представлении запасов склада нажмите **Добавить складскую единицу**
2. Выберите товар и вариант
3. Введите количество **в наличии**
4. Сохраните

### Движение запасов

Каждое изменение инвентаря отслеживается как **движение запасов**:

| Тип движения | Описание |
|-------------|----------|
| **Приход** | Новый товар получен от поставщика |
| **Продажа** | Запас списан для выполненного заказа |
| **Возврат** | Товар возвращён покупателем |
| **Корректировка** | Ручная коррекция (расхождение при подсчёте) |
| **Перемещение** | Перемещение между складами |
| **Резервирование** | Временное удержание для активной корзины |

Движения запасов обеспечивают полный аудиторский след изменений инвентаря.

## Отслеживание запасов товаров

### Включение отслеживания запасов

На вкладке **Запасы** товара:

1. Включите **Отслеживать запасы** для активации управления складом
2. Установите **Порог низкого запаса** — срабатывает оповещение, когда запас падает ниже этого уровня
3. Настройте **Разрешить предзаказы**, если хотите принимать заказы при отсутствии товара на складе

### Мультискладские запасы

Когда отслеживание запасов включено, вкладка «Запасы» показывает уровни запасов по всем складам в сводной таблице:

- Общее количество в наличии по всем локациям
- Разбивка по складам
- Доступные количества после резервирований и распределений

## Оповещения о низком запасе

Система автоматически отслеживает уровни запасов и оповещает вас, когда:
- Товар падает ниже своего **порога низкого запаса**
- Товар достигает **нулевого доступного запаса**

Оповещения о низком запасе отображаются в:
- **Панели управления магазина** в разделе «Требуются действия»
- Списке товаров с визуальным индикатором

## Советы

- Начните с одного склада и добавляйте новые по мере роста бизнеса.
- Устанавливайте приоритеты выполнения на основе скорости и стоимости доставки в каждый регион.
- Используйте буферные запасы для розничных точек, чтобы обеспечить наличие выставочного товара.
- Регулярно проверяйте движения запасов для выявления потерь или расхождений.
- Устанавливайте пороги низкого запаса на основе времени пополнения — если на пополнение уходит 2 недели, установите порог, покрывающий 2 недели продаж.
- Включите отслеживание запасов до запуска магазина, чтобы избежать перепродажи.
