---
title: "商品バンドル"
---

商品バンドルでは、事前に組み合わせた商品パッケージをバンドル価格で販売できます。これは、ギフトセット、スターターキット、または割引価格でまとめて提供したい商品の組み合わせに最適です。

![Bundle components admin](/static/core/admin/img/help/product-bundles/bundle-components.webp)

## 価格戦略

バンドル価格の計算方法を選択します：

| 戦略 | 説明 |
|------|------|
| **固定価格** | コンポーネントの価格に関係なく、バンドル全体に1つの固定価格を設定。 |
| **パーセント割引** | コンポーネント価格の合計からパーセンテージ割引として自動的に価格を計算。 |
| **コンポーネントの合計** | バンドル価格はすべてのコンポーネント価格の合計に等しい（割引なしのグループ表示に便利）。 |

## バンドルの作成

### ステップ1：商品を作成する

1. **商品 > すべての商品** に移動し、**+ 商品を追加** をクリックします
2. **商品タイプ** を **商品バンドル** に設定します
3. バンドル名、説明、画像を入力します
4. 商品を保存します

### ステップ2：コンポーネントを追加する

**バンドルアイテム** タブに切り替えて、バンドルに商品を追加します：

1. **+ コンポーネントを追加** をクリックします
2. ドロップダウンから商品を検索して選択します
3. 各コンポーネントの **数量** を設定します（例：スキンケアセットのフェイスマスク2枚）
4. **表示順** を設定して表示順序を制御します
5. 必要に応じてコンポーネントを **オプション** としてマークします（顧客が除外可能）
6. コンポーネントがバリエーション商品の場合、以下のいずれかを選択します：
   - **固定バリアント** — すべての顧客が同じバリアントを受け取る
   - **バリアント選択を許可** — 顧客がチェックアウト時に好みのバリアントを選択

下部のサマリーには、**コンポーネント合計** 数と **バンドル価値**（コンポーネント価格の合計）が表示されます。

### ステップ3：価格を設定する

**価格設定** タブに切り替えます：

1. **バンドル価格戦略** を選択します
2. **固定価格** の場合 — バンドル価格を直接入力します
3. **パーセント割引** の場合 — 割引率を設定します（例：15%オフ）
4. **コンポーネントの合計** の場合 — 価格は自動的に計算されます

## バンドルに含められるもの

| 商品タイプ | コンポーネントにできるか？ |
|-----------|------------------------|
| シンプル商品 | はい |
| バリエーション商品 | はい（固定バリアントまたは顧客選択） |
| デジタル商品 | はい |
| カスタマイズ可能商品 | いいえ |
| 構成可能な商品 | いいえ |
| 商品バンドル | いいえ（バンドルはネストできません） |
| ギフトカード | いいえ |

## 在庫管理

バンドルの在庫はコンポーネントを通じて管理されます：

- バンドルが購入可能であるためには、**すべてのコンポーネントが在庫にある** 必要があります
- バンドルが注文されると、各コンポーネント商品から個別に在庫が差し引かれます
- いずれかのコンポーネントが在庫切れになると、バンドルは利用不可になります
- コンポーネントの在庫レベルはチェックアウト時にリアルタイムで確認されます

## オプションコンポーネント

コンポーネントを **オプション** としてマークすると、顧客がバンドルをカスタマイズできます：

- オプションコンポーネントはデフォルトで含まれていますが、顧客が削除できます
- オプションコンポーネントが除外されると、バンドル価格がそれに応じて調整されます
- 少なくとも1つのコンポーネントは非オプション（必須）でなければなりません

## 顧客体験

顧客がストアフロントでバンドルを閲覧すると：

1. **コンポーネントリスト** — すべての含まれる商品が画像と数量とともに表示されます
2. **バンドル割引** — 個別購入と比較した割引額が表示されます
3. **バリアント選択** — バリアント選択が有効なコンポーネントでは、顧客が好みのオプションを選択します
4. **オプションアイテム** — 顧客がオプションコンポーネントのオン/オフを切り替えられます
5. **一括カート追加** — バンドル全体が1つのアイテムとして追加されます

## ヒント

- 最も柔軟な価格設定にはパーセント割引戦略を使用しましょう — コンポーネントの価格が変更されると自動的に調整されます。
- バンドル購入を促進するために、商品説明に割引額を目立つように表示しましょう。
- 最良の顧客体験のために、バンドルは3〜5個のコンポーネントに抑えましょう。アイテムが多すぎると圧倒的に感じることがあります。
- オプションコンポーネントを使用して、同じバンドルの「ベーシック」版と「プレミアム」版を提供しましょう。
- すべてのコンポーネント商品がまだアクティブで在庫にあることを定期的に確認しましょう。
