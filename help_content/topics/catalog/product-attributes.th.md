---
title: คุณสมบัติของสินค้า
---

คุณสมบัติของสินค้า (Product attributes) กำหนดมิติที่สินค้าสามารถเปลี่ยนแปลงได้ — เช่น ขนาด (Size), สี (Colour), หรือวัสดุ (Material) เมื่อคุณสร้างคุณสมบัติและค่าที่เป็นไปได้แล้ว คุณสามารถกำหนดให้มันใช้กับสินค้าแบบปรับแต่งได้ (variable product) และ Spwig จะสร้างตัวเลือกสำหรับการปรับแต่งที่ลูกค้าใช้ในขั้นตอนการชำระเงิน

ไปที่ **Catalog > Product Attributes** เพื่อจัดการคุณสมบัติและค่าของมัน

## วิธีการทำงานของคุณสมบัติ

คุณสมบัติสามารถใช้ร่วมกับแคตตาล็อกของคุณได้ทั้งหมด คุณสร้างมันเพียงครั้งเดียวและสามารถกำหนดให้ใช้กับสินค้าได้มากเท่าที่คุณต้องการ คุณสมบัติแต่ละตัวมี:

- **ชื่อ** ที่ระบุตัวมัน (เช่น "Size")
- **ประเภทการแสดงผล** ที่ควบคุมว่าตัวเลือกจะแสดงอย่างไรบนหน้าสินค้า
- **ค่า** หนึ่งหรือมากกว่าที่แสดงถึงตัวเลือกที่มีอยู่ (เช่น "Small", "Medium", "Large")

เมื่อคุณกำหนดคุณสมบัติให้กับสินค้า คุณยังต้องระบุว่าค่าใดที่สามารถใช้ได้สำหรับสินค้าตัวนั้นโดยเฉพาะ ซึ่งหมายความว่าคุณสมบัติ "Size" อาจมีค่าตั้งแต่ S ถึง 3XL แต่เสื้อผ้าตัวหนึ่งอาจมีเฉพาะ S, M และ L เท่านั้น

## ประเภทการแสดงผลของคุณสมบัติ

ฟิลด์ **Type** บนคุณสมบัติควบคุมว่าตัวเลือกจะแสดงอย่างไรบนหน้าสินค้าของร้านค้าของคุณ:

| Type | Appearance | Best for |
|---|---|---|
| **Dropdown Select** | เมนูแบบเลื่อนลงที่ลูกค้าเปิดเพื่อเลือกค่า | คุณสมบัติที่มีค่าจำนวนมาก (เช่น ช่วงขนาดที่มีมากกว่า 10 ขนาด) |
| **Color Swatch** | วงกลมหรือสี่เหลี่ยมสีที่ลูกค้าคลิก | คุณสมบัติสีที่การระบุด้วยภาพช่วยได้ |
| **Button Group** | ปุ่มรูปทรงยาวยื่นที่แสดงเป็นแถว | คุณสมบัติที่มีจำนวนค่าไม่มากนัก (เช่น S, M, L, XL) |
| **Radio Buttons** | รายการปุ่มเลือกแบบดั้งเดิม | คุณสมบัติใด ๆ ที่คุณต้องการให้แสดงในรูปแบบรายการที่ชัดเจนและเข้าถึงได้ง่าย |

เลือกประเภทการแสดงผลที่ตรงกับวิธีที่ลูกค้าของคุณคิดถึงคุณสมบัติ สำหรับสี สวัตช์สีมักจะดีกว่าเมนูแบบเลื่อนลงเสมอ สำหรับขนาด กลุ่มปุ่มจะทำงานได้ดีเมื่อมีตัวเลือกน้อยกว่า 8 ตัว

## การสร้างคุณสมบัติ

1.

ไปที่ **Catalog > Product Attributes**
2.

คลิก **+ Add Product Attribute**
3.

ใส่ **Name** (เช่น `Size`, `Colour`, `Material`)
4.

**Slug** จะถูกเติมอัตโนมัติ — คุณสามารถปล่อยไว้แบบนั้นได้
5.

เลือก **Type** (Dropdown, Color Swatch, Button Group, หรือ Radio Buttons)
6.

ตรวจสอบ **Is Required** หากลูกค้าต้องเลือกคุณสมบัตินี้ก่อนที่จะสามารถเพิ่มสินค้าลงในตะกร้า — สิ่งนี้เหมาะสมกับคุณสมบัติขนาดและสีส่วนใหญ่
7.

ตั้งค่า **Sort Order** — คุณสมบัติที่มีเลขต่ำกว่าจะปรากฏก่อนในตัวเลือกการเปลี่ยนรูปแบบบนหน้าสินค้า\น8.

เพิ่มค่าคุณสมบัติโดยตรงในส่วน **Values** (ดูข้างล่าง)
9.

คลิก **Save**

## การเพิ่มค่าคุณสมบัติ

ค่าคุณสมบัติคือตัวเลือกแต่ละตัวภายในคุณสมบัติ คุณสามารถเพิ่มค่าเหล่านี้โดยตรงขณะสร้างหรือแก้ไขคุณสมบัติ โดยใช้ฟอร์มค่าที่อยู่ด้านล่างของหน้ารายละเอียดคุณสมบัติ

สำหรับค่าแต่ละค่า:

- **Value** — ป้ายกำกับที่แสดง (เช่น `Small`, `Red`, `Cotton`)
- **Slug** — ถูกเติมอัตโนมัติจากค่า; ใช้ใน URL และตัวระบุรูปแบบ
- **Color Hex** — สำคัญเฉพาะสำหรับคุณสมบัติประเภท **Color Swatch** เท่านั้น ใส่รหัสสีในรูปแบบ Hex (เช่น `#FF0000` สำหรับสีแดง) เพื่อให้ส่วนตัวเลือกแสดงสีที่ถูกต้อง
- **Sort Order** — ควบคุมลำดับที่ค่าปรากฏในตัวเลือก กำหนดเลขที่ต่ำกว่าสำหรับค่าที่คุณต้องการให้ปรากฏก่อน

### การจัดลำดับค่าอย่างมีเหตุผล

สำหรับคุณสมบัติขนาด ให้ตั้งค่าลำดับการจัดเรียงเพื่อให้ขนาดเรียงจากเล็กไปใหญ่:

| Value | Sort Order |
|---|---|
| XS | 1 |
| S | 2 |
| M | 3 |
| L | 4 |
| XL | 5 |
| 2XL | 6 |

สำหรับคุณสมบัติสี คุณอาจจัดเรียงตามอักษรหรือจัดกลุ่มสีที่คล้ายกันเข้าด้วยกัน — ไม่ว่าจะเป็นวิธีใดก็ตามที่มีความหมายมากที่สุดสำหรับลูกค้าของคุณ

## การจัดการค่าคุณสมบัติแยกต่างหาก

คุณยังสามารถจัดการค่าคุณสมบัติได้อย่างเป็นอิสระที่ **Catalog > Attribute Values** รายการนี้มีประโยชน์เมื่อคุณต้องการค้นหาหรืออัปเดตค่าเฉพาะในแคตตาล็อกของคุณโดยไม่ต้องเปิดคุณสมบัติแต่ละตัว รายการนี้สามารถกรองได้ตามชื่อคุณสมบัติ

## การกำหนดคุณสมบัติให้กับสินค้า

คุณสมบัติถูกกำหนดที่ระดับสินค้า ไม่ใช่ระดับทั่วไป เพื่อเพิ่มคุณสมบัติให้กับสินค้า:

1. ไปที่ **Catalog > Products** และเปิดสินค้าแบบมีตัวเลือก
2. ในแท็บ **Variations** หาส่วน **Attributes**
3. เลือกคุณสมบัติที่ต้องการเพิ่ม
4. เลือกค่าของคุณสมบัติที่สามารถใช้ได้กับสินค้านี้
5. บันทึกสินค้า — Spwig จะสร้างการรวมตัวเลือกที่เกี่ยวข้อง

สำหรับคำแนะนำที่ละเอียดเกี่ยวกับการตั้งค่าตัวเลือกสินค้า โปรดดูหัวข้อช่วยเหลือ **Product Variants**

## ตัวอย่างการใช้งาน

### ตัวอย่าง: คุณสมบัติขนาดเสื้อผ้า

| Field | Value |
|---|---|
| Name | Size |
| Type | Button Group |
| Is Required | Yes |
| Sort Order | 1 |
| Values | XS (1), S (2), M (3), L (4), XL (5), 2XL (6) |

### ตัวอย่าง: คุณสมบัติสีตัวอย่าง

| Field | Value |
|---|---|
| Name | Colour |
| Type | Color Swatch |
| Is Required | Yes |
| Sort Order | 2 |
| Values | Black (#000000), White (#FFFFFF), Navy (#001F5B), Red (#CC0000) |

### ตัวอย่าง: คุณสมบัติวัสดุ

| Field | Value |
|---|---|
| Name | Material |
| Type | Dropdown Select |
| Is Required | No |
| Sort Order | 3 |
| Values | 100% Cotton, Cotton/Polyester Blend, Merino Wool, Linen |

## เคล็ดลับ

รักษาการจัดรูปแบบ markdown ทั้งหมด ค่าเส้นทางรูปภาพ บล็อกโค้ด และคำศัพท์ทางเทคนิคไว้

- สร้างคุณลักษณะที่สะท้อนการตัดสินใจซื้อจริงของลูกค้า — หากลูกค้าไม่จำเป็นต้องเลือกมัน มันอาจไม่จำเป็นต้องเป็นคุณลักษณะ
- ใช้ชื่อที่สอดคล้องกันทั่วทั้งแคตตาล็อกของคุณ: หากสินค้าบางส่วนใช้ "Colour" และสินค้าอื่นใช้ "Color" ลูกค้าและทีมของคุณจะพบความไม่สอดคล้องนี้เป็นเรื่องสับสน
- ลำดับการจัดเรียงของคุณลักษณะและค่ามีความสำคัญ — ให้คุณลักษณะที่สำคัญที่สุดอยู่ด้านหน้า (โดยทั่วไปคือ Size หรือ Colour) และจัดเรียงค่าตามลำดับที่มีเหตุผล
- ประเภท Color Swatch ต้องใช้ค่า hex ที่ถูกต้อง; ทดสอบสีในเครื่องมือเลือกสีของเบราว์เซอร์ก่อนบันทึกเพื่อให้แน่ใจว่าสีที่แสดงตรงกับสีของสินค้าจริง
- หากคุณต้องการเปลี่ยนชื่อคุณลักษณะ (เช่น จาก "Color" เป็น "Colour") ให้อัปเดตฟิลด์ **Name** แทนที่จะสร้างคุณลักษณะใหม่ — การเปลี่ยนชื่อไม่ส่งผลต่อการกำหนดค่าสินค้าที่มีอยู่แล้ว
