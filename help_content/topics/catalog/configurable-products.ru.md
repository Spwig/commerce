---
title: "Конфигурируемые товары"
---

Конфигурируемые товары позволяют покупателям собрать свой собственный продукт, выбирая опции из различных слотов конфигурации. Это идеально подходит для товаров на заказ, таких как пользовательские ПК, персонализированные подарочные наборы или мебель на заказ, где каждый компонент является реальным товаром в вашем каталоге.

![Product configurator admin](/static/core/admin/img/help/configurable-products/product-configurator.webp)

## Как это работает

Конфигурируемый товар состоит из **слотов** (категорий выбора) и **опций** (фактических товаров, которые покупатели могут выбрать). Например, пользовательский ПК может иметь слоты для Процессора, Видеокарты, Оперативной памяти и Хранилища — каждый слот содержит несколько вариантов товаров на выбор.

## Стратегии ценообразования

Выберите, как рассчитывается итоговая цена:

| Стратегия | Описание |
|-----------|----------|
| **Сумма компонентов** | Итоговая цена = сумма цен всех выбранных опций. Базовая цена не требуется. |
| **Базовая цена + корректировки** | Начинается с базовой цены товара, затем добавляются/вычитаются ценовые корректировки для каждой опции. |
| **Фиксированная цена** | Одна единая цена независимо от того, какие опции выбирает покупатель. |

## Настройка конфигурируемого товара

### Шаг 1: Создание товара

1. Перейдите в **Товары > Все товары** и нажмите **+ Добавить товар**
2. Установите **Тип товара** как **Конфигурируемый товар**
3. Выберите **Стратегию ценообразования** (Сумма компонентов — наиболее распространённая)
4. Заполните название товара, описание и другие основные данные
5. Сохраните товар

### Шаг 2: Добавление слотов конфигурации

После сохранения переключитесь на вкладку **Конфигурация** для настройки слотов.

1. Нажмите **+ Добавить слот** для создания новой категории конфигурации
2. Для каждого слота настройте:
   - **Название** — Что видит покупатель (например, «Процессор», «Цвет»)
   - **Иконка** — Класс иконки Font Awesome для визуальной идентификации
   - **Обязательный** — Должен ли покупатель сделать выбор
   - **Мин/Макс выборов** — Сколько опций покупатель может выбрать (по умолчанию: ровно 1)
   - **Порядок сортировки** — Контролирует порядок отображения слотов в мастере конфигурации

### Шаг 3: Добавление опций к каждому слоту

Каждому слоту нужны товарные опции для выбора покупателями:

1. Нажмите **Управление опциями** на слоте
2. Найдите и добавьте существующие товары из вашего каталога
3. Для каждой опции настройте:
   - **Корректировка цены** — Сумма для добавления или вычитания (используется с ценообразованием Базовая цена + корректировки)
   - **По умолчанию** — Предварительно выбрать эту опцию при загрузке конфигуратора
   - **Популярное** — Показать значок «Популярное» для помощи покупателям в принятии решения
   - **Количество** — Сколько единиц этого компонента включено
   - **Теги совместимости** — Теги, используемые для пакетной генерации правил совместимости

**Совет:** Компонентные товары можно скрыть из витрины, установив флажок **Скрыть из витрины** на вкладке Основная информация компонентного товара. Это сохраняет их доступность как опций конфигуратора без загромождения вашего каталога товаров.

### Шаг 4: Определение правил совместимости

Правила совместимости предотвращают выбор покупателями несовместимых комбинаций:

| Тип правила | Описание |
|-------------|----------|
| **Требует** | При выборе опции A только перечисленные опции доступны в целевом слоте |
| **Исключает** | При выборе опции A перечисленные опции скрываются из целевого слота |

Чтобы добавить правила:

1. Прокрутите до раздела **Правила совместимости** на вкладке Конфигурация
2. Нажмите **+ Добавить правило**
3. Выберите **исходную опцию** (триггер)
4. Выберите **тип правила** (Требует или Исключает)
5. Выберите **целевой слот** и **затронутые опции**

Вы также можете автоматически генерировать правила из тегов совместимости, назначенных опциям, что быстрее при управлении большим количеством комбинаций.

### Шаг 5: Создание предустановок (необязательно)

Предустановки — это заранее собранные конфигурации, которые дают покупателям быструю отправную точку:

1. Прокрутите до раздела **Предустановки конфигурации**
2. Нажмите **+ Добавить предустановку**
3. Задайте предустановке название и описание (например, «Игровая сборка», «Бюджетный вариант»)
4. Выберите опции для каждого слота
5. При необходимости загрузите изображение предварительного просмотра и отметьте как **Рекомендуемое**

Покупатели могут начать с предустановки и затем настроить отдельные слоты по своему вкусу.

## Опыт покупателя

Когда покупатель просматривает конфигурируемый товар в вашей витрине:

1. **Интерфейс мастера** — Слоты представлены как шаги, направляющие покупателя через каждый выбор
2. **Фильтрация** — Несовместимые опции автоматически скрываются на основе правил совместимости
3. **Значки популярности** — Опции, отмеченные как популярные, отображают значок для помощи в принятии решения
4. **Предустановки** — Рекомендуемые предустановки отображаются как варианты быстрого старта
5. **Обновление цен** — Общая цена обновляется в реальном времени по мере выбора опций
6. **Сводка** — Шаг обзора показывает все выбранные опции перед добавлением в корзину

## Советы

- Начните со стратегии ценообразования «Сумма компонентов» — она наиболее интуитивна для покупателей и проще всего в обслуживании.
- Используйте правила совместимости для предотвращения недопустимых конфигураций, а не полагайтесь на знания покупателя.
- Создайте 2-3 предустановки для самых популярных конфигураций, чтобы снизить усталость от принятия решений.
- Скрывайте компонентные товары из витрины, если они должны быть доступны только через конфигуратор.
- Протестируйте полный процесс конфигурации на фронтенде после настройки, чтобы убедиться, что все правила работают как ожидается.
