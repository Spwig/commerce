สินค้าดิจิทัลช่วยให้คุณสามารถขายไฟล์ดาวน์โหลดได้ ใบอนุญาตซอฟต์แวร์ และสินค้าที่ไม่ใช่สินค้าทางกายภาพอื่นๆ Spwig รองรับสินค้าดิจิทัลแบบอิสระ รวมถึงสินค้าแบบผสมผสานที่รวมทั้งสินค้าทางกายภาพและสินค้าดิจิทัล

![ผู้ให้บริการใบอนุญาต](/static/core/admin/img/help/digital-products/license-providers.webp)

## ประเภทของสินค้าดิจิทัล

### สินค้าดิจิทัลแบบอิสระ

ตั้งค่า **ประเภทสินค้า** เป็น **สินค้าดิจิทัล** สำหรับสินค้าที่เป็นดิจิทัลโดยสมบูรณ์:
- แอปพลิเคชันซอฟต์แวร์
- หนังสืออิเล็กทรอนิกส์และ PDF
- เพลงและไฟล์เสียง
- งานศิลปะดิจิทัลและเทมเพลต

### สินค้าแบบผสมผสาน

ประเภทสินค้าใดก็ตามสามารถรวมการส่งสินค้าดิจิทัลได้โดยการตรวจสอบ **เป็นสินค้าดิจิทัล** บนแท็บข้อมูลพื้นฐาน ซึ่งมีประโยชน์สำหรับ:
- **สินค้าดิจิทัลแบบปรับแต่งได้** — ซอฟต์แวร์ที่มีรุ่นพื้นฐาน/โปร/อีนเตอร์ไพรส์
- **สินค้าดิจิทัลแบบปรับแต่งได้** — สินค้าดิจิทัลที่ออกแบบเอง
- **ชุดสินค้าทางกายภาพ + ดิจิทัล** — หนังสือที่รวมการดาวน์โหลดดิจิทัล

## การตั้งค่าสินค้าดิจิทัล

### ขั้นตอนที่ 1: สร้างสินค้า

1. ไปที่ **สินค้า > ทั้งหมดสินค้า** และคลิก **+ เพิ่มสินค้า**
2. ตั้งค่า **ประเภทสินค้า** เป็น **สินค้าดิจิทัล** (หรือตรวจสอบ **เป็นสินค้าดิจิทัล** บนประเภทสินค้าอื่น)
3. กรอกข้อมูลสินค้า (ชื่อ, คำอธิบาย, ราคา)
4. บันทึกสินค้า

### ขั้นตอนที่ 2: เพิ่มไฟล์ที่ดาวน์โหลดได้

1. ไปที่แท็บ **สต็อกสินค้า** ของสินค้า
2. ในส่วน **ไฟล์ดิจิทัล** คุณสามารถอัปโหลดไฟล์ที่ลูกค้าจะได้รับหลังจากการซื้อ
3. สำหรับแต่ละไฟล์ คุณสามารถตั้งค่า:
   - **ชื่อไฟล์** — ชื่อที่แสดงให้ลูกค้าเห็น
   - **ข้อจำกัดการดาวน์โหลด** — จำนวนครั้งสูงสุดที่ไฟล์สามารถดาวน์โหลดได้ (0 = ไม่จำกัด)
   - **จำนวนวันหมดอายุ** — จำนวนวันที่ลิงก์ดาวน์โหลดยังคงใช้งานได้

### ขั้นตอนที่ 3: ตั้งค่าการจัดส่งใบอนุญาต (ไม่จำเป็น)

หากสินค้าดิจิทัลของคุณต้องการใบอนุญาต:

1. ไปที่ **การตั้งค่า > การจัดการใบอนุญาต**
2. เชื่อมต่อกับผู้ให้บริการใบอนุญาต (ดูข้อมูลด้านล่าง)
3. บนฟอร์มแก้ไขสินค้า ระบุผู้ให้บริการใบอนุญาต

## ผู้ให้บริการใบอนุญาต

ผู้ให้บริการใบอนุญาตคือบริการภายนอกที่สร้างและจัดการใบอนุญาตซอฟต์แวร์อัตโนมัติเมื่อลูกค้าซื้อสินค้าของคุณ

### ประเภทผู้ให้บริการที่มีอยู่

| ผู้ให้บริการ | คำอธิบาย |
|----------|-------------|
| **เซิร์ฟเวอร์ใบอนุญาตที่มีในตัวของ Spwig** | การสร้างใบอนุญาตแบบง่ายที่มีในแพลตฟอร์ม |
| **Keygen.sh** | API การจัดการใบอนุญาตที่มีคุณสมบัติครบครัน |
| **LicenseSpring** | การจัดการใบอนุญาตสำหรับองค์กร |
| **Cryptlex** | การจัดการใบอนุญาตซอฟต์แวร์ที่รองรับการใช้งานแบบออฟไลน์ |
| **API แบบกำหนดเอง** | เชื่อมต่อกับระบบใบอนุญาตใดๆ ผ่าน REST API |

### การเชื่อมต่อกับผู้ให้บริการใบอนุญาต

1. ไปที่ **การตั้งค่า > การจัดการใบอนุญาต**
2. คลิก **เชื่อมต่อกับผู้ให้บริการ**
3. ดำเนินการตามเวิร์กฟล로ว์การตั้งค่า:
   - **ขั้นตอนที่ 1** — เลือกประเภทผู้ให้บริการ
   - **ขั้นตอนที่ 2** — ตั้งค่าการตั้งค่าทั่วไป
   - **ขั้นตอนที่ 3** — ป้อนข้อมูลเข้าถึง API
4. ทดสอบการเชื่อมต่อเพื่อยืนยันว่าทำงานได้
5. บันทึกการตั้งค่า

### บัตรผู้ให้บริการ

ผู้ให้บริการที่เชื่อมต่อแต่ละรายแสดง:
- **ป้ายสถานะ** — สถานะเปิดใช้งาน/ปิดใช้งานและสถานะการเชื่อมต่อ
- **จุดปลายทาง API** — URL ของเซิร์ฟเวอร์ที่ตั้งค่าไว้
- **ความสามารถในการซิงค์** — รองรับการซิงค์การสั่งซื้อ การเปิดใช้งาน และการปิดใช้งาน
- **ปุ่มดำเนินการ** — ตั้งค่า, ทดสอบ และ ซิงค์ทันที

### ความสามารถในการซิงค์

ผู้ให้บริการใบอนุญาตสามารถซิงค์ได้ในเหตุการณ์สามแบบ:

- **การสั่งซื้อ** — สร้างใบอนุญาตอัตโนมัติเมื่อลูกค้าทำรายการซื้อเสร็จสิ้น
- **การเปิดใช้งาน** — ติดตามเมื่อลูกค้าเปิดใช้งานใบอนุญาต
- **การปิดใช้งาน** — จัดการการปิดใช้งานใบอนุญาตสำหรับการคืนเงินหรือการโอนย้าย

## ประสบการณ์ของลูกค้า

### หลังจากการซื้อ

เมื่อลูกค้าซื้อสินค้าดิจิทัล:

1. **การยืนยันคำสั่งซื้อ** — แสดงว่ามีการจัดส่งดิจิทัลที่รวมอยู่
2. **การจัดส่งผ่านอีเมล** — ลิงก์ดาวน์โหลดและ/หรือใบอนุญาตจะถูกส่งอัตโนมัติ
3. **หน้าบัญชี** — ลูกค้าสามารถเข้าถึงการดาวน์โหลดได้จากแดชบอร์ดบัญชีของพวกเขา
4. **หน้าดาวน์โหลด** — ลิงก์ดาวน์โหลดที่มีความปลอดภัยและมีระยะเวลาจำกัด

### ความปลอดภัยในการดาวน์โหลด

การดาวน์โหลดไฟล์ดิจิทัลได้รับการปกป้องโดย:
- โทเค็นดาวน์โหลดที่มีความเฉพาะเจาะจงและมีระยะเวลาจำกัด
- ข้อจำกัดการดาวน์โหลดที่สามารถกำหนดได้ (ถ้าต้องการ)
- วันหมดอายุหลังจากนั้นลิงก์จะไม่สามารถใช้งานได้
- ต้องเข้าสู่ระบบ (สำหรับลูกค้าที่ลงทะเบียน)

## คำแนะนำ

- ตั้งค่าข้อจำกัดการดาวน์โหลดให้เหมาะสม (3-5 ครั้ง) เพื่อป้องกันการใช้ในทางที่ผิดขณะที่ยังอนุญาตให้ดาวน์โหลดซ้ำได้
- ใช้จำนวนวันหมดอายุที่ตรงกับระยะเวลาการสนับสนุนของคุณ (เช่น 365 วันสำหรับการเข้าถึงหนึ่งปี)
- ทดสอบการซื้อทั้งหมดด้วยคำสั่งซื้อทดสอบเพื่อให้แน่ใจว่าลิงก์ดาวน์โหลดและใบอนุญาตถูกส่งได้อย่างถูกต้อง
- สำหรับสินค้าซอฟต์แวร์ เชื่อมต่อกับผู้ให้บริการใบอนุญาตเพื่อสร้างใบอนุญาตอัตโนมัติแทนการจัดการใบอนุญาตด้วยตนเอง
- ใช้คุณสมบัติสินค้าแบบผสมผสานเมื่อขายสินค้าทางกายภาพที่รวมสินค้าดิจิทัลเพิ่มเติม (เช่น หนังสือพิมพ์ + PDF)
