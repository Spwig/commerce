---
title: Управление лицензионными ключами
---

Управление лицензионными ключами позволяет контролировать, как создаются, хранятся и доставляются лицензионные ключи для программного обеспечения, когда клиенты покупают цифровые продукты. Spwig поддерживает встроенную генерацию ключей, заранее загруженные пулы ключей и интеграции с внешними службами управления лицензиями.

## Обзор

В Spwig есть три способа управления лицензионными ключами:

| Метод | Лучше подходит для |
|--------|---------|
| **Шаблоны лицензий** | Автоматически генерировать уникальные ключи в пользовательском формате в момент покупки |
| **Пулы лицензий** | Предварительно сгенерировать пакет ключей для массового распространения |
| **Внешние провайдеры** | Делегировать генерацию и управление ключами сторонней службе, например Keygen.sh |

Эти методы можно комбинировать — например, пул может использовать пользовательский шаблон для определения формата ключа и, при необходимости, синхронизировать сгенерированные ключи с внешним провайдером.

## Шаблоны лицензионных ключей

Шаблон лицензионного ключа определяет *формат* сгенерированных ключей. Шаблоны используют паттерн с заполнителями, которые Spwig заполняет в момент генерации.

### Создание шаблона

1. Перейдите в **Каталог > Шаблоны лицензионных ключей**
2. Нажмите **+ Добавить шаблон лицензионного ключа**
3. Введите **Название** (например, `Стандартная лицензия приложения`)
4. Настройте **Паттерн** с использованием заполнителей (см. ниже)
5. Установите **Префикс** и **Суффикс**, если это необходимо (например, префикс `MYAPP` добавляет `MYAPP-` к каждому ключу)
6. Выберите **Разделитель** (по умолчанию: `-`)
7. Установите **Набор символов** — символы, используемые для случайных сегментов. По умолчанию исключаются неоднозначные символы, такие как `0` и `O`, `1` и `I`
8. Установите **Минимальную/максимальную длину** для проверки
9. Нажмите **Сохранить**

### Заполнители паттерна

| Заполнитель | Описание | Пример вывода |
|-------------|-------------|---------------|
| `{RANDOM:N}` | N случайных символов из набора символов | `{RANDOM:5}` → `K7JXQ` |
| `{CHECKSUM:N}` | N-значная контрольная сумма для проверки | `{CHECKSUM:2}` → `47` |
| `{PREFIX}` | Значение префикса шаблона | `MYAPP` |
| `{SUFFIX}` | Значение суффикса шаблона | `PRO` |
| `{ORDER_ID}` | Номер заказа | `10045` |
| `{PRODUCT_SKU}` | SKU продукта | `SOFTPRO` |
| `{DATE:FORMAT}` | Форматированная дата | `{DATE:YYMMDD}` → `260318` |

**Пример паттерна**: `{PREFIX}-{RANDOM:5}-{RANDOM:5}-{RANDOM:5}-{CHECKSUM:2}`

Это создает ключи вроде: `MYAPP-K7JXQ-M3TPR-9BWKN-47`

### Предварительный просмотр ключей

После сохранения шаблона, на списке шаблонов становится доступным действие **Сгенерировать образец ключа**. Используйте это, чтобы проверить, что ваш паттерн создает ключи в ожидаемом формате, прежде чем назначить шаблон продукту.

## Пулы лицензий

Пул лицензий — это пакет предварительно сгенерированных ключей для продукта. Пулы полезны, когда:
- Вам нужны ключи для физической упаковки (розничные коробки, напечатанные карты)
- Вы работаете с дистрибьюторами, которым нужны пакеты ключей
- Вы хотите, чтобы ключи генерировались заранее, а не по требованию

### Создание пула лицензий

1. Перейдите в **Каталог > Пулы лицензий**
2. Нажмите **+ Добавить пул лицензий**
3. Заполните детали пула:

| Поле | Описание |
|-------|-------------|
| **Название** | Описательное название (например, `Розничный пакет Q1 2026`) |
| **Продукт** | Продукт, для которого предназначены эти ключи |
| **Шаблон лицензии** | Шаблон для формата ключа (по умолчанию используется шаблон продукта) |
| **Общее количество ключей** | Сколько ключей сгенерировать |
| **Тип лицензии** | Бессрочная, подписка или пробная |
| **Максимальное количество активаций** | Сколько устройств может активировать каждый ключ |
| **Истекает через (дней)** | Количество дней, через которое лицензия истекает после первой активации (оставьте пустым, чтобы не истекала) |
| **Пул истекает к** | Дата, после которой неиспользованные ключи из этого пула станут недействительными |
| **Синхронизировать с провайдером** | Опционально синхронизировать сгенерированные ключи с внешним провайдером лицензий |

4. Нажмите **Сохранить** — Spwig начинает генерировать ключи в фоновом режиме

### Состояние пула

| Статус | Значение |
|--------|---------|
| **Генерация** | Ключи создаются в фоновом режиме |
| **Готово** | Все ключи сгенерированы и доступны для распространения |
| **Использовано** | Все ключи были назначены заказам |
| **Истекло** | Дата истечения срока действия пула прошла |

### Мониторинг пула

Список пулов показывает, сколько ключей было распределено по сравнению с общим количеством сгенерированных ключей. Откройте пул, чтобы увидеть полный список ключей и их индивидуальные статусы.

## Внешние поставщики лицензий

Внешние поставщики — это сторонние службы управления лицензиями, которые отвечают за генерацию ключей и отслеживание их активации. Когда клиент завершает покупку, Spwig взаимодействует с поставщиком для генерации и регистрации ключа.

### Поддерживаемые поставщики

| Поставщик | Тип |
|----------|------|
| **Встроенный сервер лицензий Spwig** | Встроенный — не требуется внешняя учетная запись |
| **Keygen.sh** | Облачная система управления лицензиями на основе API |
| **LicenseSpring** | Предприятиевая система управления лицензиями |
| **Cryptlex** | Управление лицензиями с поддержкой автономного режима |
| **Пользовательский API** | Любая система лицензии на основе REST |

### Подключение поставщика

1. Перейдите в **Каталог > Поставщики лицензий**
2. Нажмите **+ Добавить поставщика лицензий**
3. Заполните детали поставщика:

| Параметр | Описание |
|-------|-------------|
| **Название** | Метка для этой связи (например, `Keygen Production`) |
| **Тип поставщика** | Выберите из поддерживаемых поставщиков |
| **Конечная точка API** | Базовый URL API поставщика |
| **Ключ API** | Ключ аутентификации для поставщика |
| **Секрет API** | Если это требуется от поставщика |

4. Настройте поведение синхронизации:
   - **Синхронизировать при заказе** — автоматически синхронизировать, когда клиент завершает покупку
   - **Синхронизировать при активации** — сообщить о активации устройств поставщику
   - **Синхронизировать при деактивации** — сообщить о деактивациях (полезно для передачи лицензий и возвратов)
   - **Двусторонняя синхронизация** — разрешить поставщику обновлять записи Spwig через веб-хуки

5. Нажмите **Сохранить**, затем нажмите **Проверить соединение**, чтобы убедиться, что учетные данные работают

### Статус соединения

Каждый поставщик показывает один из трех статусов соединения:

| Статус | Значение |
|--------|---------|
| **Не проверено** | Соединение еще не проверялось |
| **Подключено** | Последняя проверка была успешной |
| **Ошибка** | Проверка соединения завершилась неудачно — проверьте сообщение об ошибке |

### Синхронизация существующих лицензий

Чтобы вручную отправить существующие лицензионные ключи поставщику (для первоначальной настройки или после неудачной синхронизации), используйте действие **Синхронизировать сейчас** из списка поставщиков.

## Мониторинг активности синхронизации

Перейдите в **Каталог > Внешняя синхронизация лицензий**, чтобы просмотреть журнал синхронизации. Каждая запись показывает:
- Лицензионный ключ, который был синхронизирован
- Поставщика, которому он был отправлен
- Направление (Spwig → Поставщик или Поставщик → Spwig)
- Статус (В ожидании, Успешно, Неудачно)
- Подробности ошибки для неудачных синхронизаций

Неудачные синхронизации автоматически повторяются. Вы также можете принудительно повторить попытку, отредактировав запись и очистив ошибку.

## Советы

- Используйте стандартный набор символов (`ABCDEFGHJKLMNPQRSTUVWXYZ23456789`), чтобы избежать неоднозначных символов, которые часто неправильно читаются клиентами — он исключает `0`, `O`, `1` и `I`.
- Добавьте сегмент `{CHECKSUM}` в ваш шаблонный паттерн, чтобы клиенты и ваша служба поддержки могли быстро обнаружить опечатки в ключах.
- Для продуктов с высоким объемом используйте пул вместо генерации по запросу, чтобы гарантировать мгновенную доставку ключей при оформлении заказа.
- Установите **Pool Expires At** для сезонных или временных пакетов ключей, чтобы старые неиспользуемые ключи автоматически становились недействительными.
- Всегда проверяйте соединение с поставщиком после настройки и после любых изменений учетных данных — разорванное соединение означает, что клиенты не получают свои ключи.
- Если вы используете двустороннюю синхронизацию, настройте URL-адрес веб-хука вашего поставщика, чтобы он указывал на конечную точку веб-хука лицензии вашего магазина.
