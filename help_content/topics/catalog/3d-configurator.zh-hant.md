---
title: 3D產品配置器
---

3D配置器讓您的客戶可以直接在產品頁面上，透過互動式3D查看器查看可配置的產品。當客戶選擇選項（例如顏色、材質或組件變體）時，3D模型會即時更新以反映他們的選擇。在支援的行動裝置上，客戶還可以透過增強現實（AR）查看產品，在購買前將其虛擬放置在自己的空間中。

3D配置器與可配置產品一起使用。每個可配置產品可以有一個3D場景配置，將GLB模型文件連結到產品的配置選項。

## 在開始之前

要設置3D場景，您需要：

- 已經在您的目錄中創建的**可配置產品**
- 已上傳到您的媒體庫作為GLB文件的**基礎3D模型**——這是預設顯示的組裝模型
- 可選地，用於幾何替換的其他GLB文件（例如，不同的頸部形狀），以及用於材質變體的紋理圖片

如果您尚未創建可配置產品及其配置選項，請先完成這一步，再設置3D場景。

## 建立場景配置

1. 進入 **目錄 > 3D場景配置**
2. 點擊 **+ 添加3D場景配置**
3. 選擇這個場景所屬的 **產品** — 只能選擇可配置產品
4. 從您的媒體庫中選擇 **基礎3D模型** — 這是預設載入的GLB文件
5. 設置查看器設定（請參閱下文）
6. 儲存記錄

儲存後，**節點樹**欄位會自動填寫。這是從您的GLB文件中解析出的場景圖 — 它列出模型中每個命名的節點，您在添加節點映射時將會引用這些節點。

## 查看器設定

這些設定控制3D查看器在您的產品頁面上的顯示方式。

### 相機和照明


| 欄位 | 說明 | 預設值 |
|-------|-------------|---------|
| **攝影機軌道** | 攝影機起始位置，格式為 `angle elevation distance`（例如，`0deg 75deg 2m`） | `0deg 75deg 2m` |
| **攝影機目標** | 攝影機所觀看的點，以模型中心為基準的公尺數（例如，`0m 0m 0m`） | `0m 0m 0m` |
| **環境圖片** | 從您的媒體庫中使用的 HDR 圖片，用於基於圖像的照明 — 可提供更真實的反射和陰影效果 | 無 |
| **曝光** | 场景的整体亮度 — 值越低越暗，值越高越亮 | `1.0` |

### 陰影

| 欄位 | 說明 | 預設值 |
|-------|-------------|---------|
| **陰影強度** | 模型下方陰影的強度 — `0` 表示無陰影，`1` 表示全強度 | `0.5` |
| **陰影柔化** | 陰影邊緣的模糊程度 — `0` 表示清晰，`1` 表示非常柔化 | `0.5` |

### 色調校正

| 欄位 | 說明 |
|-------|-------------|
| **色調映射** | 應用於場景的色彩校正算法。**Commerce** 會產生鮮豔且適合產品的色彩。**Neutral** 是色彩準確的。**ACES** 會產生電影般的視覺效果。 |
| **光暈強度** | 為模型自發光的部分添加光暈效果。`0` 會禁用光暈。`1` 到 `5` 的值會產生從微妙到戲劇性的光暈效果。 |

### 行為與背景

| 欄位 | 說明 | 預設值 |
|-------|-------------|---------|
| **自動旋轉** | 模型在載入時是否緩慢旋轉以吸引顧客的注意 | 啟用 |
| **啟用 AR** | 支援裝置上的顧客是否會看到 **在 AR 中查看** 按鈕 | 啟用 |
| **背景** | 觀看器的背景顏色或 CSS 渐變 — 輸入十六進位顏色（例如，`#f5f5f5`）或 CSS 渐變值 | `#ffffff` |

### 縮略圖

**縮略圖** 欄位保存 3D 觀看器的預覽截圖，會在觀看器載入前顯示。

您可以从实时产品页面截取屏幕截图，并将其上传到您的媒体库，然后在此处链接以获得更流畅的页面加载体验。

## 啟用和停用3D查看器

**啟用**開關控制是否在產品頁面上顯示3D查看器。當停用時，產品會回退到標準的2D圖片配置器。這讓您可以在讓其對客戶可見之前準備場景配置。

## 將配置選項連接到3D操作

一旦基礎場景配置完成，您可以將每個配置選項連接到3D模型中的視覺變化。這些連結稱為**節點映射**，並在場景配置表單底部的**節點映射**部分中添加。

### 節點映射欄位

| 欄位 | 說明 |
|-------|-------------|
| **選項槽** | 觸發此更改的配置選項（例如，"紅色皮革"） |
| **操作類型** | 發生的視覺更改（請參閱下方的操作類型） |
| **目標節點** | 更改的場景圖節點名稱 — 從您的**節點樹**中列出的名稱中選擇 |
| **操作資料** | 與操作相關的資料，例如顏色十六進制代碼、紋理URL或GLB文件URL |
| **排序順序** | 控制相同選項的多個映射應用的順序 |

### 操作類型

| 操作 | 功能 |
|--------|-------------|
| **材質顏色** | 更改目標節點上材質的顏色 — 在**操作資料**中提供十六進制顏色代碼 |
| **材質紋理** | 更換應用到材質上的紋理 — 在**操作資料**中連結到紋理圖片資源 |
| **幾何替換** | 用不同的GLB文件替換模型的一部分 — 用於結構更改，例如不同的手柄形狀 |
| **可見性** | 在場景中顯示或隱藏節點 — 在**操作資料**中設置 `visible: true` 或 `visible: false` |

一個選項槽可以添加多個映射。

例如，選擇「藍色牛仔布」可能會同時改變材質顏色 *並* 隱藏一個皮革縫線節點。

## 幾何形狀資源

如果您的配置包含 **幾何形狀替換** 動作，您需要將替換的 GLB 檔案註冊為幾何形狀資源。這些資源會在場景配置表單的 **幾何形狀資源** 區域中新增。

| 欄位 | 說明 |
|-------|-------------|
| **標籤** | 這個幾何形狀資源的描述性名稱，例如："V 頸領" |
| **GLB 檔案** | 來自您的媒體庫的替換 GLB 檔案 |
| **目標節點** | 這個幾何形狀替換基礎模型中的哪個節點 |

在儲存幾何形狀資源後，其節點名稱會從 GLB 中解析並儲存在 **節點資料** 中，使其在您的映射中可用作目標節點。

## 紋理資源

用於 **材質紋理** 映射的紋理圖片可以註冊為紋理資源，以便更輕鬆地引用。這些資源會在 **紋理資源** 區域中新增。

| 欄位 | 說明 |
|-------|-------------|
| **標籤** | 描述性名稱，例如："紅色皮革" |
| **紋理圖片** | 來自您的媒體庫的紋理圖片 |
| **紋理類型** | 這個紋理適用於哪個 PBR 通道 — 基底顏色、法線圖、粗糙度圖、金屬度圖、環境遮蔽或發光圖 |

## 範例：可配置的外套與顏色選項

**情境：** 一件可以訂購為黑色、海軍藍或勃艮第紅的外套，每種顏色都應用於外套的網格模型。

**設定：**

1. 為外套產品建立場景配置，使用組裝好的外套 GLB 作為基礎模型
2. 將 **色調映射** 設為 Commerce，並將 **自動旋轉** 設為開啟
3. 在節點映射中新增三個項目 — 每個顏色選項一個：

| Slot Option | Action Type | Target Node | Action Data |
|-------------|-------------|-------------|-------------|
| Black | Material Color | JacketBody | `{"color": "#1a1a1a"}` |
| Navy | Material Color | JacketBody | `{"color": "#1b2a4a"}` |
| Burgundy | Material Color | JacketBody | `{"color": "#6b2737"}` |

當客戶在產品頁面上選擇 Navy 時，查看器會立即將 JacketBody 材料更新為深藍色。

## Tips

- 在建立 3D 模型時，請清楚命名 GLB 節點 — 類似 "JacketBody" 或 "CollarMesh" 的節點名稱比自動產生的名稱如 "Mesh_023" 更容易使用
- 對於大多數產品，請使用 **Commerce** 色調映射 — 它針對生動且吸引人的產品展示進行了優化
- 對於預設攝影機角度已經展示最重要特徵的產品，請停用 **Auto Rotate**，以避免在載入時讓客戶感到困惑
- 在推廣 AR 按鈕之前，請在實際的手機設備上測試它 — AR 的可用性取決於客戶的設備和瀏覽器（支援 WebXR 的 iOS Safari 和 Android Chrome 是最可靠的）
- 為每個場景配置上傳 **Thumbnail** 圖像 — 這樣可以防止 3D 查看器載入時閃過一個空白的白盒子
- 如果 3D 查看器尚未準備好，請透過 **Enabled** 切換鈕停用它，讓客戶看到標準的圖像配置器
