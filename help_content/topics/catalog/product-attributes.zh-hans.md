title: 产品属性
---

产品属性定义了产品可以变化的维度 —— 例如，尺寸、颜色或材质。一旦你创建了一个属性及其可能的值，就可以将其分配给任何可变产品，Spwig 将生成客户在结账时使用的变体选择器。

导航到 **目录 > 产品属性** 来管理属性及其值。

## 属性的工作原理

属性可以在你的整个目录中重复使用。你只需创建一次，然后根据需要分配给尽可能多的产品。每个属性包含：

- 一个 **名称**，用于标识它（例如，"尺寸"）
- 一个 **显示类型**，控制属性选择器在产品页面上的显示方式
- 一个或多个 **值**，表示可用的选项（例如，"小号"、"中号"、"大号"）

当你将一个属性分配给一个产品时，你还需要指定该产品可用的值。这意味着一个 "尺寸" 属性可能有 S 到 3XL 的值，但一件特定的T恤可能只提供 S、M 和 L。

## 属性显示类型

属性上的 **类型** 字段控制选择小部件在您的商店前端产品页面上的显示方式：

| 类型 | 显示方式 | 最适合用于 |
|---|---|---|
| **下拉选择** | 客户打开下拉菜单以选择一个值 | 具有大量值的属性（例如，包含 10 个以上尺寸的尺寸范围） |
| **颜色色块** | 客户点击的彩色圆圈或方块 | 颜色属性，视觉识别有帮助时 |
| **按钮组** | 以内联方式显示的圆角按钮 | 具有少量值的属性（例如，S、M、L、XL） |
| **单选按钮** | 传统的单选按钮列表 | 任何你希望有清晰、易用列表布局的属性 |

选择与客户对属性的思维方式相匹配的显示类型。对于颜色，色块几乎总是比下拉菜单更好。对于尺寸，当选项少于 8 个时，按钮组效果很好。

## 创建属性

1.

转到 **目录 > 产品属性**
2.

点击 **+ 添加产品属性**
3.

输入 **名称**（例如，`Size`，`Colour`，`Material`）
4.

**Slug** 会自动填充 —— 你可以保持原样
5.

选择 **类型**（下拉菜单、颜色色块、按钮组或单选按钮）
6.

如果客户在将产品添加到购物车之前必须选择此属性，请勾选 **必填** —— 这适用于大多数尺寸和颜色属性
7.

设置 **排序顺序** —— 排序数字较小的属性会在产品页面的变体选择器中首先显示
8.

在 **值** 部分直接添加属性值（见下文）
9.

点击 **保存**

## 添加属性值

属性值是属性中的单个选项。你可以在创建或编辑属性时，使用属性详情页面底部的内联值表单直接添加它们。

对于每个值：

- **值** —— 显示标签（例如，`Small`，`Red`，`Cotton`）
- **Slug** —— 从值自动生成；用于 URL 和变体标识符
- **颜色十六进制** —— 仅适用于 **颜色色块** 类型的属性。输入一个十六进制颜色代码（例如，`#FF0000` 表示红色），以便色块显示正确的颜色。
- **排序顺序** —— 控制值在选择器中显示的顺序。将较小的数字分配给你希望首先显示的值。

### 逻辑排序值

对于尺寸属性，设置排序顺序，使尺寸从小到大排列：

| 值 | 排序顺序 |
|---|---|
| XS | 1 |
| S | 2 |
| M | 3 |
| L | 4 |
| XL | 5 |
| 2XL | 6 |

对于颜色属性，你可以按字母顺序排序或将相似的颜色分组 —— 无论哪种方式对你客户来说最有意义。

## 单独管理属性值

你还可以在 **目录 > 属性值** 中单独管理属性值。此列表在你需要在目录中查找或更新特定值而无需逐一打开每个属性时非常有用。该列表可以通过属性名称进行筛选。

## 为产品分配属性

属性是在产品级别分配的，而不是全局分配的。要将属性添加到产品中：

1. 导航到 **目录 > 产品** 并打开一个可变产品
2. 在 **变体** 选项卡中，找到 **属性** 部分
3. 选择要添加的属性
4. 选择该产品可用的属性值
5. 保存产品 —— Spwig 将生成相应的变体组合

有关设置产品变体的详细指南，请参阅 **产品变体** 帮助主题。

## 实际示例

### 示例：服装尺寸属性

| 字段 | 值 |
|---|---|
| 名称 | Size |
| 类型 | 按钮组 |
| 是否必填 | 是 |
| 排序顺序 | 1 |
| 值 | XS (1), S (2), M (3), L (4), XL (5), 2XL (6) |

### 示例：颜色色板属性

| 字段 | 值 |
|---|---|
| 名称 | Colour |
| 类型 | 颜色色板 |
| 是否必填 | 是 |
| 排序顺序 | 2 |
| 值 | Black (#000000), White (#FFFFFF), Navy (#001F5B), Red (#CC0000) |

### 示例：材质属性

| 字段 | 值 |
|---|---|
| 名称 | Material |
| 类型 | 下拉选择 |
| 是否必填 | 否 |
| 排序顺序 | 3 |
| 值 | 100% Cotton, Cotton/Polyester Blend, Merino Wool, Linen |

## 小贴士

保留所有markdown格式、图片路径、代码块和专业术语。

- 创建能够体现客户真实购买决策的属性 —— 如果客户不需要选择它，那么可能不需要将其设为属性
- 在您的商品目录中使用一致的命名方式：如果某些商品使用 "Colour"，而另一些使用 "Color"，客户和您的团队会感到困惑
- 属性和值的排序顺序很重要 —— 将最重要的属性放在前面（通常是尺寸或颜色），并按逻辑顺序排列值
- 颜色色块类型需要准确的十六进制代码；在保存之前，使用浏览器的颜色选择器测试颜色，以确保色块与实际产品颜色一致
- 如果需要重命名属性（例如，从 "Color" 改为 "Colour"），请更新 **名称** 字段，而不是创建新的属性 —— 更改名称不会影响现有产品的分配
