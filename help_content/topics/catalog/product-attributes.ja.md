---
title: 製品属性
---

製品属性は、製品が変化できる次元を定義します — 例えば、サイズ、色、または素材です。属性とその可能な値を作成した後、変数製品のいずれかに割り当てることができます。Spwigは、顧客がチェックアウト時に使用する変更選択肢を生成します。

**カタログ > 製品属性**に移動して、属性とその値を管理してください。

## 属性の仕組み

属性は、カタログ全体で再利用可能です。一度作成し、必要なだけの製品に割り当てることができます。各属性には以下があります:

- 識別するための**名前**（例: "Size"）
- 製品ページ上でのセレクターの表示方法を制御する**表示タイプ**
- 利用可能なオプションを表す**1つ以上の値**（例: "Small", "Medium", "Large"）

属性を製品に割り当てると、その製品で利用可能な値を指定することも必要です。これは、"Size"属性がSから3XLまでの値を持つ場合でも、特定のTシャツはS、M、Lのみを提供する可能性があることを意味します。

## 属性の表示タイプ

属性の**Type**フィールドは、ストアフロントの製品ページ上での選択ウィジェットの表示方法を制御します:

| Type | Appearance | Best for |
|---|---|---|
| **Dropdown Select** | 顧客が開いて値を選択するドロップダウンメニュー | 複数の値を持つ属性（例: 10以上のサイズを持つサイズ範囲） |
| **Color Swatch** | 顧客がクリックする色付きの円や正方形 | 視覚的な識別が役立つ色属性 |
| **Button Group** | 並べて表示されるピル型ボタン | 値が少ない属性（例: S、M、L、XL） |
| **Radio Buttons** | 伝統的なラジオボタンリスト | リストレイアウトを明確かつアクセス可能な形式で表示したい属性 |

顧客が属性についてどのように考えるかに合わせて、表示タイプを選択してください。色の場合、スワッチはドロップダウンよりも常に優れています。サイズの場合、選択肢が8未満の場合はボタングループが効果的です。

## 属性の作成

1.

ナビゲートして **カタログ > 商品属性** にアクセス

## 商品に属性を割り当てる

属性はグローバルではなく、商品レベルで割り当てられます。商品に属性を追加するには:

1. **カタログ > 商品** に移動し、変更可能な商品を開きます
2. **バリエーション** タブで、**属性** セクションを探します
3. 追加したい属性を選択します
4. この商品で利用可能な属性の値を選択します
5. 商品を保存すると、Spwig が対応するバリアントの組み合わせを生成します

商品バリアントの設定に関する詳細なガイドについては、**Product Variants** ヘルプトピックを参照してください。

## 実用例

### 例: サイズ属性

| Field | Value |
|---|---|
| Name | Size |
| Type | Button Group |
| Is Required | Yes |
| Sort Order | 1 |
| Values | XS (1), S (2), M (3), L (4), XL (5), 2XL (6) |

### 例: カラースウォッチ属性

| Field | Value |
|---|---|
| Name | Colour |
| Type | Color Swatch |
| Is Required | Yes |
| Sort Order | 2 |
| Values | Black (#000000), White (#FFFFFF), Navy (#001F5B), Red (#CC0000) |

### 例: 材質属性

| Field | Value |
|---|---|
| Name | Material |
| Type | Dropdown Select |
| Is Required | No |
| Sort Order | 3 |
| Values | 100% Cotton, Cotton/Polyester Blend, Merino Wool, Linen |

## ヒント

すべてのマークダウン形式、画像パス、コードブロック、技術用語を保持してください。

- 顧客が実際に購入決定に関与する属性を作成してください — 顧客が選択する必要がない場合は、属性にする必要はありません
- カタログ全体で一貫した命名規則を使用してください — 一部の商品が「Colour」を使用し、他の商品が「Color」を使用していると、顧客やチームにとって混乱を引き起こします
- 属性と値の並び順は重要です — 通常、サイズやカラーなどの重要な属性を最初に配置し、値は論理的な順序で並べてください
- カラースウォッチタイプは正確なHEXコードを必要とします — 保存する前にブラウザのカラーピッカーで色をテストし、スウォッチが実際の商品の色と一致することを確認してください
- 属性名を変更する必要がある場合（例: 「Color」から「Colour」に変更する場合）、新しい属性を作成する代わりに**名前**フィールドを更新してください — 名前の変更は既存の商品の割り当てに影響を与えません
