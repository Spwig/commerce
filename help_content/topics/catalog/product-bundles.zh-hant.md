---
title: "商品套裝"
---

商品套裝允許您以捆綁價格銷售預先組合的商品包。這非常適合禮品套裝、入門套件或任何您想以折扣價一起提供的商品組合。

![套裝組件管理](/static/core/admin/img/help/product-bundles/bundle-components.webp)

## 定價策略

選擇套裝價格的計算方式：

| 策略 | 描述 |
|----------|-------------|
| **固定價格** | 為整個套裝設定一個統一價格，不受組件價格影響。 |
| **百分比折扣** | 根據組件價格合計自動計算折扣後的價格。 |
| **組件價格總和** | 套裝價格等於所有組件價格的總和（適用於不打折的分組展示）。 |

## 建立套裝

### 第 1 步：建立商品

1. 導覽至**商品 > 所有商品**，然後點擊**+ 新增商品**
2. 將**商品類型**設定為**商品套裝**
3. 填寫套裝名稱、描述和圖片
4. 儲存商品

### 第 2 步：新增組件

切換到**套裝項目**索引標籤來新增商品到您的套裝：

1. 點擊**+ 新增組件**
2. 從下拉選單中搜尋並選擇一個商品
3. 設定每個組件的**數量**（例如，護膚套裝中的 2 片面膜）
4. 設定**排序順序**來控制顯示順序
5. 可選將組件標記為**可選**（客戶可以排除它）
6. 如果組件是多規格商品，選擇以下之一：
   - **固定規格** — 所有客戶獲得相同的規格
   - **允許規格選擇** — 客戶在結帳時選擇他們喜歡的規格

底部的摘要顯示**組件總數**和**套裝價值**（組件價格總和）。

### 第 3 步：設定定價

切換到**定價**索引標籤：

1. 選擇您的**套裝定價策略**
2. 對於**固定價格** — 直接輸入套裝價格
3. 對於**百分比折扣** — 設定折扣百分比（例如 15% 折扣）
4. 對於**組件價格總和** — 價格自動計算

## 可捆綁的商品類型

| 商品類型 | 可作為組件？ |
|-------------|-------------------|
| 簡單商品 | 是 |
| 多規格商品 | 是（固定規格或客戶選擇） |
| 數位商品 | 是 |
| 可自訂商品 | 否 |
| 可配置商品 | 否 |
| 商品套裝 | 否（套裝不能巢狀） |
| 禮品卡 | 否 |

## 庫存管理

套裝庫存透過其組件進行管理：

- 套裝可購買的前提是**所有組件都有庫存**
- 當套裝被下單時，庫存從每個組件商品中單獨扣減
- 如果任何組件缺貨，套裝將變為不可用
- 結帳過程中會即時檢查組件庫存水準

## 可選組件

將組件標記為**可選**以讓客戶自訂他們的套裝：

- 可選組件預設包含在內，但客戶可以將其移除
- 當可選組件被排除時，套裝價格會相應調整
- 至少一個組件必須是非可選的（必選）

## 客戶體驗

當客戶在店面檢視套裝時：

1. **組件列表** — 所有包含的商品及其圖片和數量一起顯示
2. **套裝節省** — 顯示與單獨購買相比的折扣金額
3. **規格選擇** — 對於啟用了規格選擇的組件，客戶選擇他們喜歡的選項
4. **可選項目** — 客戶可以開啟或關閉可選組件
5. **一鍵加入購物車** — 整個套裝作為一個項目新增

## 提示

- 使用百分比折扣策略獲得最靈活的定價——當組件價格變化時它會自動調整。
- 在商品描述中突出顯示節省金額，以鼓勵購買套裝。
- 將套裝控制在 3-5 個組件以獲得最佳客戶體驗。太多項目可能會讓人感到不知所措。
- 使用可選組件來提供同一套裝的「基礎版」和「進階版」。
- 定期檢查所有組件商品是否仍處於活躍狀態且有庫存。
