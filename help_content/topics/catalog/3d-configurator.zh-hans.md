---
title: 3D产品配置器
---

3D配置器允许您的客户在产品页面上直接通过交互式3D查看器查看可配置产品。当客户选择选项（如颜色、材质或组件变体）时，3D模型会实时更新以反映他们的选择。在支持的移动设备上，客户还可以通过增强现实（AR）查看产品，在购买前将其虚拟放置在自己的空间中。

3D配置器与可配置产品一起工作。每个可配置产品可以有一个3D场景配置，将GLB模型文件链接到产品的配置选项。

## 开始之前

要设置3D场景，您需要：

- 已在您的目录中创建的**可配置产品**
- 作为GLB文件上传到您的媒体库的**基础3D模型**——这是默认显示的组装模型
- 可选地，用于几何替换的附加GLB文件（例如，不同的领口形状），以及用于材质变体的纹理图像

如果您尚未创建可配置产品及其配置选项，请先完成此步骤，然后再设置3D场景。

## 创建场景配置

1. 导航到 **目录 > 3D场景配置**
2. 点击 **+ 添加3D场景配置**
3. 选择此场景所属的 **产品** — 仅可配置产品可用
4. 从您的媒体库中选择 **基础3D模型** — 这是默认加载的GLB文件
5. 配置查看器设置（见下文）
6. 保存记录

保存后，**节点树**字段会自动填充。这是从您的GLB文件中提取的解析场景图——它列出了模型中的每个命名节点，您在添加节点映射时将引用这些节点。

## 查看器设置

这些设置控制3D查看器在您的产品页面上的显示方式。

### 摄像机和照明


| 字段 | 描述 | 默认值 |
|-------|-------------|---------|
| **相机轨道** | 相机的起始位置，格式为 `angle elevation distance`（例如，`0deg 75deg 2m`） | `0deg 75deg 2m` |
| **相机目标** | 相机注视的点，以模型中心为基准的米数（例如，`0m 0m 0m`） | `0m 0m 0m` |
| **环境图像** | 从您的媒体库中使用的HDR图像，用于基于图像的照明——提供更逼真的反射和阴影 | 无 |
| **曝光** | 场景的整体亮度——较低的值更暗，较高的值更亮 | `1.0` |

### 阴影

| 字段 | 描述 | 默认值 |
|-------|-------------|---------|
| **阴影强度** | 模型下方阴影的强度——`0` 表示无阴影，`1` 表示全强度 | `0.5` |
| **阴影柔和度** | 阴影边缘的模糊程度——`0` 表示清晰，`1` 表示非常柔和 | `0.5` |

### 色彩校正

| 字段 | 描述 |
|-------|-------------|
| **色调映射** | 应用于场景的色彩校正算法。**商业** 会产生鲜艳、适合产品的颜色。**中性** 是色彩准确的。**ACES** 会呈现出电影般的视觉效果。 |
| **辉光强度** | 为模型中自发光（自带光源）的部分添加辉光效果。`0` 禁用辉光。`1` 到 `5` 的值会产生从微妙到戏剧性的辉光效果。 |

### 行为和背景

| 字段 | 描述 | 默认值 |
|-------|-------------|---------|
| **自动旋转** | 模型在加载时是否缓慢旋转以吸引顾客的注意 | 开启 |
| **启用AR** | 是否在支持的设备上让客户看到 **在AR中查看** 按钮 | 开启 |
| **背景** | 查看器的背景颜色或CSS渐变——输入一个十六进制颜色（例如，`#f5f5f5`）或一个CSS渐变值 | `#ffffff` |

### 缩略图

**缩略图** 字段保存了3D查看器的预览截图，在查看器加载之前显示。

你可以从实时产品页面截取屏幕截图，并将其上传到你的媒体库，然后在此处链接以获得更流畅的页面加载体验。

## 启用和禁用3D查看器

**启用**切换按钮控制是否在产品页面上显示3D查看器。当禁用时，产品将回退到标准的2D图像配置器。这使你可以在将其对客户可见之前准备场景配置。

## 将配置选项连接到3D操作

一旦基础场景配置完成，你可以将每个配置槽选项链接到3D模型中的视觉变化。这些链接称为**节点映射**，在场景配置表单底部的**节点映射**部分中添加。

### 节点映射字段

| 字段 | 描述 |
|-------|-------------|
| **槽选项** | 触发此更改的配置选项（例如，"红色皮革"） |
| **操作类型** | 发生的视觉更改（请参阅下方的操作类型） |
| **目标节点** | 发生更改的场景图节点的名称 — 从你的**节点树**中列出的名称中选择 |
| **操作数据** | 操作特定数据，例如颜色十六进制代码、纹理URL或GLB文件URL |
| **排序顺序** | 控制相同选项的多个映射的应用顺序 |

### 操作类型

| 操作 | 功能 |
|--------|-------------|
| **材质颜色** | 更改目标节点上材质的颜色 — 在**操作数据**中提供一个十六进制颜色代码 |
| **材质纹理** | 更换应用于材质的纹理 — 在**操作数据**中链接到纹理图像资源 |
| **几何替换** | 用不同的GLB文件替换模型的一部分 — 适用于结构更改，例如不同的手柄形状 |
| **可见性** | 显示或隐藏场景中的节点 — 在**操作数据**中设置 `visible: true` 或 `visible: false` |

一个槽选项可以添加多个映射。

例如，选择“蓝色牛仔布”可能会同时更改材质颜色*并*隐藏皮革装饰节点。

## 几何体资源

如果您的配置包含 **Geometry Swap** 操作，则需要将替换的 GLB 文件注册为几何体资源。这些资源在场景配置表单的 **Geometry Assets** 部分中添加。

| Field | Description |
|-------|-------------|
| **Label** | 该几何体资源的描述性名称，例如 "V领领口" |
| **GLB File** | 来自您的媒体库的替换 GLB 文件 |
| **Target Node** | 此几何体替换基础模型中的哪个节点 |

保存几何体资源后，其节点名称将从 GLB 中解析并存储在 **Node Data** 中，使其在您的映射中可用作目标节点。

## 纹理资源

在 **Material Texture** 映射中使用的纹理图像可以注册为纹理资源，以便更方便地引用。这些资源在 **Texture Assets** 部分中添加。

| Field | Description |
|-------|-------------|
| **Label** | 描述性名称，例如 "红色皮革" |
| **Texture Image** | 来自您的媒体库的纹理图像 |
| **Texture Type** | 该纹理适用的 PBR 通道 — 基色、法线贴图、粗糙度贴图、金属度贴图、环境遮蔽或发光贴图 |

## 示例：可配置的夹克，带颜色选项

**场景：** 一种可以订购为黑色、海军蓝或勃艮第红的夹克，每种颜色应用于夹克主体网格。

**设置：**

1. 为夹克产品创建场景配置，使用组装好的夹克 GLB 作为基础模型
2. 将 **Tone Mapping** 设置为 Commerce，并将 **Auto Rotate** 设置为开启
3. 在 Node Mappings 中添加三个条目 — 每个颜色选项一个：

| Slot Option | Action Type | Target Node | Action Data |
|-------------|-------------|-------------|-------------|
| Black | Material Color | JacketBody | `{"color": "#1a1a1a"}` |
| Navy | Material Color | JacketBody | `{"color": "#1b2a4a"}` |
| Burgundy | Material Color | JacketBody | `{"color": "#6b2737"}` |

当客户在产品页面上选择Navy时，查看器会立即将JacketBody材质更新为海军蓝颜色。

## Tips

- 在创建3D模型时，要清楚地命名GLB节点——像“JacketBody”或“CollarMesh”这样的节点名称比自动生成的名称（如“Mesh_023”）更容易使用
- 对于大多数产品，使用**Commerce**色调映射——它针对鲜艳、吸引人的产品展示进行了优化
- 对于默认相机角度已经展示了最重要功能的产品，禁用**Auto Rotate**，以避免在加载时让客户感到困惑
- 在推广AR按钮之前，在实际移动设备上测试它——AR的可用性取决于客户的设备和浏览器（iOS Safari和Android Chrome支持WebXR的版本最为可靠）
- 为每个场景配置上传**Thumbnail**图像——这可以防止3D查看器加载时出现空白方框
- 如果3D查看器尚未准备就绪，请通过**Enabled**切换按钮将其禁用，以便客户看到标准的图像配置器
