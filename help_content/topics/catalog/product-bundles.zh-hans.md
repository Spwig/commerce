---
title: "商品套装"
---

商品套装允许您以捆绑价格销售预先组合的商品包。这非常适合礼品套装、入门套件或任何您想以折扣价一起提供的商品组合。

![套装组件管理](/static/core/admin/img/help/product-bundles/bundle-components.webp)

## 定价策略

选择套装价格的计算方式：

| 策略 | 描述 |
|----------|-------------|
| **固定价格** | 为整个套装设置一个统一价格，不受组件价格影响。 |
| **百分比折扣** | 根据组件价格合计自动计算折扣后的价格。 |
| **组件价格总和** | 套装价格等于所有组件价格的总和（适用于不打折的分组展示）。 |

## 创建套装

### 第 1 步：创建商品

1. 导航到**商品 > 所有商品**，然后点击**+ 添加商品**
2. 将**商品类型**设置为**商品套装**
3. 填写套装名称、描述和图片
4. 保存商品

### 第 2 步：添加组件

切换到**套装项目**选项卡来添加商品到您的套装：

1. 点击**+ 添加组件**
2. 从下拉菜单中搜索并选择一个商品
3. 设置每个组件的**数量**（例如，护肤套装中的 2 片面膜）
4. 设置**排序顺序**来控制显示顺序
5. 可选将组件标记为**可选**（客户可以排除它）
6. 如果组件是多规格商品，选择以下之一：
   - **固定规格** — 所有客户获得相同的规格
   - **允许规格选择** — 客户在结账时选择他们喜欢的规格

底部的摘要显示**组件总数**和**套装价值**（组件价格总和）。

### 第 3 步：配置定价

切换到**定价**选项卡：

1. 选择您的**套装定价策略**
2. 对于**固定价格** — 直接输入套装价格
3. 对于**百分比折扣** — 设置折扣百分比（例如 15% 折扣）
4. 对于**组件价格总和** — 价格自动计算

## 可捆绑的商品类型

| 商品类型 | 可作为组件？ |
|-------------|-------------------|
| 简单商品 | 是 |
| 多规格商品 | 是（固定规格或客户选择） |
| 数字商品 | 是 |
| 可定制商品 | 否 |
| 可配置商品 | 否 |
| 商品套装 | 否（套装不能嵌套） |
| 礼品卡 | 否 |

## 库存管理

套装库存通过其组件进行管理：

- 套装可购买的前提是**所有组件都有库存**
- 当套装被下单时，库存从每个组件商品中单独扣减
- 如果任何组件缺货，套装将变为不可用
- 结账过程中会实时检查组件库存水平

## 可选组件

将组件标记为**可选**以让客户自定义他们的套装：

- 可选组件默认包含在内，但客户可以将其移除
- 当可选组件被排除时，套装价格会相应调整
- 至少一个组件必须是非可选的（必选）

## 客户体验

当客户在店面查看套装时：

1. **组件列表** — 所有包含的商品及其图片和数量一起显示
2. **套装节省** — 显示与单独购买相比的折扣金额
3. **规格选择** — 对于启用了规格选择的组件，客户选择他们喜欢的选项
4. **可选项目** — 客户可以开启或关闭可选组件
5. **一键加入购物车** — 整个套装作为一个项目添加

## 提示

- 使用百分比折扣策略获得最灵活的定价——当组件价格变化时它会自动调整。
- 在商品描述中突出显示节省金额，以鼓励购买套装。
- 将套装控制在 3-5 个组件以获得最佳客户体验。太多项目可能会让人感到不知所措。
- 使用可选组件来提供同一套装的"基础版"和"高级版"。
- 定期检查所有组件商品是否仍处于活跃状态且有库存。
