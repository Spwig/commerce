title: إدارة مفاتيح الترخيص
---

تتيح لك إدارة مفاتيح الترخيص التحكم في طريقة إنشاء مفاتيح ترخيص البرامج، وتخزينها، وتوزيعها على العملاء عند شراء المنتجات الرقمية. يدعم Spwig إنشاء المفاتيح مدمجًا، وools مسبقة التحميل، وتكاملات مع خدمات إدارة الترخيص الخارجية.

| Placeholder | Description | Example output |
|-------------|-------------|---------------|
| `{RANDOM:N}` | N random characters from the character set | `{RANDOM:5}` → `K7JXQ` |
| `{CHECKSUM:N}` | N-digit checksum for validation | `{CHECKSUM:2}` → `47` |
| `{PREFIX}` | The template's prefix value | `MYAPP` |
| `{SUFFIX}` | The template's suffix value | `PRO` |
| `{ORDER_ID}` | The order number | `10045` |
| `{PRODUCT_SKU}` | The product's SKU | `SOFTPRO` |
| `{DATE:FORMAT}` | Formatted date | `{DATE:YYMMDD}` → `260318` |

**Example pattern**: `{PREFIX}-{RANDOM:5}-{RANDOM:5}-{RANDOM:5}-{CHECKSUM:2}`

This produces keys like: `MYAPP-K7JXQ-M3TPR-9BWKN-47`

### Previewing keys

After saving a template, a **Generate Sample Key** action is available on the template list. Use this to verify your pattern produces keys in the expected format before assigning the template to a product.

## License pools

A license pool is a batch of pre-generated keys for a product. Pools are useful when:
- You need keys for physical packaging (retail boxes, printed cards)
- You work with resellers who need batches of keys
- You want keys generated in advance rather than on demand

### Creating a license pool

1. Navigate to **Catalog > License Pools**
2. Click **+ Add License Pool**
3. Fill in the pool details:

| الحقل | الوصف |
|-------|-------------|
| **الاسم** | اسم وصفي (مثلاً: `Retail Pack Q1 2026`) |
| **المنتج** | المنتج الذي هذه المفاتيح الخاصة به |
| **قالب الترخيص** | نموذج لتنسيق المفتاح (يُحدد بشكل افتراضي نموذج المنتج) |
| **إجمالي المفاتيح** | عدد المفاتيح التي يجب إنشاؤها |
| **نوع المفتاح** | دائم، اشتراك، أو تجريب |
| **عدد التنشيطات القصوى** | عدد الأجهزة التي يمكن للمفتاح تنشيطها عليها |
| **الانتهاء بعد الأيام** | عدد الأيام التي يستمر فيها الترخيص حتى ينتهي بعد التنشيط الأول (اتركه فارغًا لعدم وجود انتهاء) |
| **انتهاء المجموعة في** | التاريخ الذي يصبح فيه المفاتيح غير المستخدمة من هذه المجموعة غير صالحة |
| **التوافق مع مزود خارجي** | خياري للتوافق مع مزود ترخيص خارجي |

4. اضغط على **حفظ** — يبدأ Spwig بإنشاء المفاتيح في الخلفية

### حالة المجموعة

| الحالة | المعنى |
|--------|---------|
| **جارٍ الإنشاء** | يتم إنشاء المفاتيح في الخلفية |
| **جاهز** | تم إنشاء جميع المفاتيح وتم إعدادها للتسليم |
| **مُستهلك** | تم تخصيص جميع المفاتيح للطلبات |
| **منتهية الصلاحية** | تجاوزت تاريخ انتهاء صلاحية المجموعة |

### مراقبة مجموعة

تُظهر قائمة المجموعات عدد المفاتيح التي تم توزيعها مقابل إجمالي المفاتيح المُنشَأة. افتح مجموعة لرؤية القائمة الكاملة للمفاتيح وأحجامها الفردية.

## مزودي الترخيص الخارجي

يمثل مزودو الترخيص الخارجي خدمات إدارة ترخيص طرف ثالث تتعامل مع إنشاء المفاتيح ومتابعة تنشيطها. عندما يكمل العميل عملية الشراء، يقوم Spwig بالاتصال بالمزود لإنشاء وتسجيل المفتاح.

### المزودون المدعومون

| المزود | النوع |
|----------|------|
| **خادم الترخيص الداخلي لـ Spwig** | داخلي — لا حاجة لحساب خارجي |
| **Keygen.sh** | واجهة برمجة تطبيقات إدارة ترخيص سحابية |
| **LicenseSpring** | إدارة ترخيص للشركات |
| **Cryptlex** | إدارة ترخيص مع دعم عدم الاتصال |
| **واجهة برمجة تطبيقات مخصصة** | أي نظام ترخيص قائم على REST |

### الاتصال بمزود

1.

اذهب إلى **الكتالوج > مزودي الترخيص**
2.

اضغط على **+ إضافة مزود ترخيص**
3.


املأ تفاصيل مزود الخدمة:

| الحقل | الوصف |
|-------|-------------|
| **الاسم** | تسمية لهذا الاتصال (مثلاً: `Keygen Production`) |
| **نوع المزود** | اختر من بين المزودين المدعومين |
| **نقطة النهاية الخاصة بـ API** | عنوان URL الأساسي لـ API للمزود |
| **مفتاح API** | مفتاح المصادقة للمزود |
| **سر API** | إذا طلبته المزود |

4. تكوين سلوك التزامن:
   - **التزامن عند الطلب** — اتزامن تلقائياً عند إكمال العميل لطلب شراء
   - **التزامن عند التفعيل** — تقرير تفعيلات الأجهزة إلى المزود
   - **التزامن عند إلغاء التفعيل** — تقرير إلغاء التفعيلات (مفيد لنقل التراخيص والاستردادات)
   - **التزامن ثنائي الاتجاه** — السماح للمزود بتحديث سجلات Spwig عبر ويب هوك

5. اضغط على **حفظ**، ثم اضغط على **اختبار الاتصال** للتحقق من أن بيانات الاعتماد تعمل

### حالة الاتصال

يظهر كل مزود من بين ثلاث حالات اتصال:

| الحالة | المعنى |
|--------|---------|
| **لم يتم الاختبار بعد** | لم يتم التحقق من الاتصال بعد |
| **متصلاً** | كان الاختبار الأخير ناجحاً |
| **خطأ** | فشل اختبار الاتصال — تحقق من رسالة الخطأ |

### التزامن مع التراخيص الموجودة

للحصول على مفاتيح الترخيص الموجودة يدويًا إلى مزود (للإعداد الأولي أو بعد فشل التزامن)، استخدم إجراء **التزامن الآن** من قائمة مزودي الخدمة.

## مراقبة أنشطة التزامن

انتقل إلى **الكتالوج > التزامن مع التراخيص الخارجية** لمراجعة سجل التزامن. تظهر كل سجل:
- المفتاح الخاص بالترخيص الذي تم التزامنه
- المزود الذي تم إرساله إليه
- الاتجاه (Spwig → المزود أو المزود → Spwig)
- الحالة (معلقة، نجاح، فشل)
- تفاصيل الخطأ للتزامنات الفاشلة

تتم إعادة المحاولة تلقائياً لفشل التزامن. يمكنك أيضاً إجبار إعادة المحاولة من خلال تعديل السجل ومسح الخطأ.

## نصائح

احتفظ بجميع التنسيقات المرئية، مسارات الصور، كتل الشفرة، والمصطلحات الفنية.

- استخدم المجموعة الافتراضية من الأحرف (`ABCDEFGHJKLMNPQRSTUVWXYZ23456789`) لتجنب الأحرف التي قد يخطئ العملاء في قراءتها — حيث يتم استبعاد `0`، و`O`، و`1`، و`I`.
- أضف قطعة `{CHECKSUM}` إلى نمط القالب الخاص بك بحيث يمكن للعملاء وفريق الدعم الخاص بك تحديد المفاتيح المكتوبة بشكل خاطئ بسرعة.
- بالنسبة للمنتجات ذات الحجم الكبير، استخدم مجموعة من المفاتيح بدلاً من إنشاء المفاتيح عند الطلب لضمان توصيل المفاتيح فورًا عند الخروج.
- ضع **Pool Expires At** على دفعات المفاتيح الموسمية أو ذات المدة الزمنية المحدودة بحيث تُلغى المفاتيح القديمة غير المستخدمة تلقائيًا.
- قم دائمًا بإجراء اختبار للتوصيل مع مزود الخدمة بعد الإعداد وبعد أي تغييرات في بيانات الاعتماد — توقف التوصيل يعني أن العملاء لا يتلقون مفاتيحهم.
- إذا كنت تستخدم التزامن ثنائي الاتجاه، قم بتكوين عنوان URL لـ webhook لمزودك ليشير إلى نقطة نهاية webhook الخاصة بترخيص متجرك.
