---
title: การจัดการคีย์ใบอนุญาต
---

การจัดการคีย์ใบอนุญาตช่วยให้คุณสามารถควบคุมวิธีการสร้าง จัดเก็บ และส่งคีย์ใบอนุญาตซอฟต์แวร์ไปยังลูกค้าเมื่อพวกเขาซื้อสินค้าดิจิทัล Spwig รองรับการสร้างคีย์ในตัว การโหลดคีย์ล่วงหน้า และการผสานกับบริการจัดการคีย์ภายนอก

## ภาพรวม

มีสามวิธีในการจัดการคีย์ใบอนุญาตใน Spwig:

| วิธี | เหมาะสำหรับ |
|--------|---------|
| **เทมเพลตคีย์ใบอนุญาต** | สร้างคีย์ที่ไม่ซ้ำกันในรูปแบบที่กำหนดเองโดยอัตโนมัติในขณะที่ลูกค้าซื้อ |
| **คีย์แบบรวมกัน** | สร้างคีย์ล่วงหน้าเป็นชุดเพื่อแจกจ่ายจำนวนมาก |
| **ผู้ให้บริการภายนอก** | มอบหมายให้ผู้ให้บริการภายนอก เช่น Keygen.sh สร้างและจัดการคีย์ |

วิธีเหล่านี้สามารถใช้ร่วมกันได้ - ตัวอย่างเช่น คีย์แบบรวมกันสามารถใช้เทมเพลตที่กำหนดเองเพื่อกำหนดรูปแบบคีย์ และสามารถส 同步 คีย์ที่สร้างไปยังผู้ให้บริการภายนอกได้

## เทมเพลตคีย์ใบอนุญาต

เทมเพลตคีย์ใบอนุญาตกำหนด *รูปแบบ* ของคีย์ที่สร้าง เทมเพลตใช้รูปแบบที่มีตัวแทนที่ Spwig จะเติมในขณะสร้าง

### การสร้างเทมเพลต

1. ไปที่ **Catalog > License Key Templates**
2. คลิก **+ Add License Key Template**
3. ใส่ **ชื่อ** (เช่น `Standard App License`)
4. ตั้งค่า **รูปแบบ** โดยใช้ตัวแทนที่ (ดูข้างล่าง)
5. ตั้งค่า **ตัวนำ** และ **ตัวท้าย** หากจำเป็น (เช่น ตัวนำ `MYAPP` จะเพิ่ม `MYAPP-` ไปยังคีย์ทุกตัว)
6. เลือกตัวอักษร **ตัวคั่น** (ค่าเริ่มต้น: `-`)
7. ตั้งค่าชุดตัวอักษร **Character Set** - ตัวอักษรที่ใช้ในส่วนสุ่ม ค่าเริ่มต้นจะไม่รวมตัวอักษรที่คล้ายกัน เช่น `0` และ `O`, `1` และ `I`
8. ตั้งค่า **ความยาวขั้นต่ำ/สูงสุด** เพื่อตรวจสอบความถูกต้อง
9. คลิก **บันทึก**

### ตัวแทนในรูปแบบ


| Placeholder | Description | Example output |
|-------------|-------------|---------------|
| `{RANDOM:N}` | N random characters from the character set | `{RANDOM:5}` → `K7JXQ` |
| `{CHECKSUM:N}` | N-digit checksum for validation | `{CHECKSUM:2}` → `47` |
| `{PREFIX}` | The template's prefix value | `MYAPP` |
| `{SUFFIX}` | The template's suffix value | `PRO` |
| `{ORDER_ID}` | The order number | `10045` |
| `{PRODUCT_SKU}` | The product's SKU | `SOFTPRO` |
| `{DATE:FORMAT}` | Formatted date | `{DATE:YYMMDD}` → `260318` |

**Example pattern**: `{PREFIX}-{RANDOM:5}-{RANDOM:5}-{RANDOM:5}-{CHECKSUM:2}`

This produces keys like: `MYAPP-K7JXQ-M3TPR-9BWKN-47`

### Previewing keys

After saving a template, a **Generate Sample Key** action is available on the template list. Use this to verify your pattern produces keys in the expected format before assigning the template to a product.

## License pools

A license pool is a batch of pre-generated keys for a product. Pools are useful when:
- You need keys for physical packaging (retail boxes, printed cards)
- You work with resellers who need batches of keys
- You want keys generated in advance rather than on demand

### Creating a license pool

1. Navigate to **Catalog > License Pools**
2. Click **+ Add License Pool**
3. Fill in the pool details:

| ฟิลด์ | คำอธิบาย |
|-------|-------------|
| **ชื่อ** | ชื่อที่อธิบายได้ (เช่น, `Retail Pack Q1 2026`) |
| **ผลิตภัณฑ์** | ผลิตภัณฑ์ที่คีย์เหล่านี้ใช้กับ |
| **เทมเพลตใบอนุญาต** | เทมเพลตสำหรับรูปแบบคีย์ (ค่าเริ่มต้นคือเทมเพลตของผลิตภัณฑ์) |
| **จำนวนคีย์ทั้งหมด** | จำนวนคีย์ที่ต้องการสร้าง |
| **ประเภทคีย์** | คีย์แบบถาวร, แบบสมัครสมาชิก หรือแบบทดลอง |
| **จำนวนการเปิดใช้งานสูงสุด** | จำนวนอุปกรณ์ที่แต่ละคีย์สามารถเปิดใช้งานได้ |
| **หมดอายุหลังจาก (วัน)** | จำนวนวันที่ใบอนุญาตจะหมดอายุหลังจากเปิดใช้งานครั้งแรก (เว้นว่างหากไม่ต้องการกำหนดวันหมดอายุ) |
| **หมดอายุของ Pool** | วันที่หลังจากนั้นคีย์ที่ยังไม่ได้ใช้จาก Pool นี้จะไม่สามารถใช้งานได้ |
| **ส 同步到提供商** | สามารถส 同步生成的密钥到外部许可证提供商 |

4. คลิก **บันทึก** — Spwig จะเริ่มสร้างคีย์ในพื้นหลัง

### สถานะของ Pool

| สถานะ | ความหมาย |
|--------|---------|
| **กำลังสร้าง** | คีย์กำลังถูกสร้างในพื้นหลัง |
| **พร้อมใช้งาน** | คีย์ทั้งหมดถูกสร้างแล้วและพร้อมสำหรับการแจกจ่าย |
| **หมดสต็อก** | คีย์ทั้งหมดถูกกำหนดให้กับคำสั่งซื้อแล้ว |
| **หมดอายุ** | วันหมดอายุของ Pool ได้ผ่านไปแล้ว |

### การติดตาม Pool

รายการ Pool แสดงจำนวนคีย์ที่ได้แจกจ่ายแล้วเทียบกับจำนวนคีย์ทั้งหมดที่สร้างขึ้น คลิกเปิด Pool เพื่อดูรายการคีย์ทั้งหมดและสถานะของแต่ละคีย์

## ผู้ให้บริการใบอนุญาตภายนอก

ผู้ให้บริการภายนอกคือบริการจัดการใบอนุญาตของบุคคลที่สามที่จัดการการสร้างคีย์และการติดตามการเปิดใช้งาน เมื่อลูกค้าเสร็จสิ้นการซื้อ Spwig จะสื่อสารกับผู้ให้บริการเพื่อสร้างและลงทะเบียนคีย์

### ผู้ให้บริการที่รองรับ

| ผู้ให้บริการ | ประเภท |
|----------|------|
| **Spwig Built-in License Server** | ติดตั้งในตัว — ไม่จำเป็นต้องมีบัญชีภายนอก |
| **Keygen.sh** | API จัดการใบอนุญาตแบบคลาวด์ |
| **LicenseSpring** | จัดการใบอนุญาตสำหรับองค์กร |
| **Cryptlex** | จัดการใบอนุญาตพร้อมการสนับสนุนแบบออฟไลน์ |
| **Custom API** | ระบบจัดการใบอนุญาตแบบ REST ใดๆ ก็ได้ |

### การเชื่อมต่อกับผู้ให้บริการ

1.

ไปที่ **Catalog > License Providers**
2.

คลิก **+ Add License Provider**
3.


กรุณากรอกรายละเอียดผู้ให้บริการ:

| Field | Description |
|-------|-------------|
| **Name** | ป้ายกำกับสำหรับการเชื่อมต่อ (เช่น `Keygen Production`) |
| **Provider Type** | เลือกจากผู้ให้บริการที่รองรับ |
| **API Endpoint** | URL ฐานของ API ของผู้ให้บริการ |
| **API Key** | คีย์การรับรองสำหรับผู้ให้บริการ |
| **API Secret** | หากผู้ให้บริการต้องการ

4. ตั้งค่าพฤติกรรมการซิงค์:
   - **Sync on Order** — ซิงค์อัตโนมัติเมื่อลูกค้าสั่งซื้อสินค้าเสร็จสิ้น
   - **Sync on Activation** — รายงานการเปิดใช้งานอุปกรณ์ไปยังผู้ให้บริการ
   - **Sync on Deactivation** — รายงานการปิดใช้งาน (มีประโยชน์สำหรับการโอนสิทธิ์และคืนเงิน)
   - **Bidirectional Sync** — อนุญาตให้ผู้ให้บริการอัปเดตข้อมูล Spwig ผ่าน webhook

5. คลิก **Save**, แล้วคลิก **Test Connection** เพื่อตรวจสอบว่าข้อมูลยืนยันได้หรือไม่

### Connection status

ผู้ให้บริการแต่ละรายแสดงสถานะการเชื่อมต่อหนึ่งในสามสถานะต่อไปนี้:

| Status | Meaning |
|--------|---------|
| **Not Tested** | การเชื่อมต่ยังไม่ได้รับการตรวจสอบ |
| **Connected** | การทดสอบครั้งสุดท้ายสำเร็จ |
| **Error** | การทดสอบการเชื่อมต่อล้มเหลว — ตรวจสอบข้อความข้อผิดพลาด

### Syncing existing licenses

เพื่อส่งลิขสิทธิ์ที่มีอยู่ไปยังผู้ให้บริการด้วยตนเอง (สำหรับการตั้งค่าเริ่มต้นหรือหลังจากซิงค์ล้มเหลว) ให้ใช้การดำเนินการ **Sync Now** จากรายการผู้ให้บริการ

## Monitoring sync activity

ไปที่ **Catalog > External License Syncs** เพื่อดูบันทึกการซิงค์ แต่ละบันทึกแสดง:
- คีย์ลิขสิทธิ์ที่ซิงค์
- ผู้ให้บริการที่ส่งไป
- ทิศทาง (Spwig → Provider หรือ Provider → Spwig)
- สถานะ (Pending, Success, Failed)
- รายละเอียดข้อผิดพลาดสำหรับการซิงค์ที่ล้มเหลว

การซิงค์ที่ล้มเหลวจะถูกลองอีกครั้งโดยอัตโนมัติ คุณยังสามารถบังคับให้ลองอีกครั้งโดยการแก้ไขบันทึกและลบข้อผิดพลาด

## Tips

รักษาการจัดรูปแบบ markdown, ค่าเส้นทางรูปภาพ, บล็อกโค้ด และคำศัพท์ทางเทคนิคทั้งหมดไว้

- ใช้ชุดตัวอักษรมาตรฐาน (`ABCDEFGHJKLMNPQRSTUVWXYZ23456789`) เพื่อหลีกเลี่ยงตัวอักษรที่อาจก่อให้เกิดความสับสนแก่ลูกค้า — ชุดนี้ไม่รวม `0`, `O`, `1`, และ `I`
- เพิ่มส่วน `{CHECKSUM}` ลงในรูปแบบเทมเพลตของคุณ เพื่อให้ลูกค้าและทีมสนับสนุนของคุณสามารถตรวจสอบการพิมพ์ผิดของคีย์ได้อย่างรวดเร็ว
- สำหรับสินค้าที่มีปริมาณสูง ให้ใช้ระบบสุ่มคีย์แบบ Pool แทนการสร้างแบบเรียลไทม์ เพื่อให้แน่ใจว่าคีย์จะถูกส่งไปยังลูกค้าทันทีที่ชำระเงิน
- ตั้งค่า **Pool Expires At** สำหรับคีย์ที่มีช่วงเวลาจำกัดหรือตามฤดูกาล เพื่อให้คีย์ที่ไม่ได้ใช้งานอีกต่อไปถูกยกเลิกโดยอัตโนมัติ
- ตรวจสอบการเชื่อมต่อกับผู้ให้บริการหลังจากตั้งค่าและหลังจากเปลี่ยนข้อมูลการรับรองใด ๆ — การเชื่อมต่อที่เสียหายหมายถึงลูกค้าจะไม่ได้รับคีย์ของพวกเขา
- หากใช้การซิงค์แบบสองทาง ให้ตั้งค่า URL webhook ของผู้ให้บริการให้ชี้ไปยังจุดปลายทาง webhook สำหรับใบอนุญาตของร้านค้าของคุณ
