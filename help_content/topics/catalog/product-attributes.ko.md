title: 상품 속성
---

상품 속성은 상품이 달라질 수 있는 차원을 정의합니다. 예를 들어, 크기(Size), 색상(Colour), 재질(Material) 등이 있습니다. 속성과 가능한 값을 생성한 후에는 해당 속성을 어떤 가변 상품에도 할당할 수 있으며, Spwig은 고객이 결제 시 사용하는 변형 선택기를 생성합니다.

**카탈로그 > 상품 속성**으로 이동하여 속성과 그 값들을 관리합니다.

## 속성의 작동 방식

속성은 전체 카탈로그에서 재사용할 수 있습니다. 한 번 생성한 후 필요한 만큼의 상품에 할당할 수 있습니다. 각 속성에는 다음과 같은 항목이 있습니다:

- **이름**으로 식별되는 이름(예: "Size")
- 상품 페이지에 선택기가 어떻게 표시되는지를 제어하는 **표시 유형**
- 사용 가능한 옵션을 나타내는 하나 이상의 **값**(예: "Small", "Medium", "Large")

속성을 상품에 할당할 때, 해당 상품에 사용 가능한 값들을 지정해야 합니다. 이는 "Size" 속성에 S부터 3XL까지의 값들이 있을 수 있지만, 특정 티셔츠는 S, M, L만 제공할 수 있다는 의미입니다.

## 속성 표시 유형

속성의 **유형** 필드는 상점 프론트 페이지의 상품 페이지에 선택 위젯이 어떻게 표시되는지를 제어합니다:

| 유형 | 표시 방식 | 최적의 사용 사례 |
|---|---|---|
| **드롭다운 선택** | 고객이 값을 선택하기 위해 열 수 있는 드롭다운 메뉴 | 많은 값이 있는 속성(예: 10개 이상의 크기 범위) |
| **색상 스와치** | 고객이 클릭할 수 있는 색상 원 또는 정사각형 | 시각적 식별이 도움이 되는 색상 속성 |
| **버튼 그룹** | 줄을 따라 표시되는 원형 버튼 | 값이 적은 속성(예: S, M, L, XL) |
| **라디오 버튼** | 전통적인 라디오 버튼 목록 | 명확하고 접근성이 좋은 목록 레이아웃이 필요한 속성 |

고객이 속성을 어떻게 생각하는지에 맞는 표시 유형을 선택하세요. 색상의 경우, 드롭다운보다 스와치가 거의 항상 더 좋습니다. 크기의 경우, 선택지가 8개 미만인 경우 버튼 그룹이 잘 작동합니다.

## 제품 속성 추가

제품 속성 값은 속성 내부의 개별 옵션입니다. 속성을 생성하거나 편집할 때 속성 상세 페이지 하단의 인라인 값 폼을 사용하여 직접 추가할 수 있습니다.

각 값에 대해:

- **Value** — 표시 라벨 (예: `Small`, `Red`, `Cotton`)
- **Slug** — 값에서 자동으로 채워지며 URL 및 변형 식별자에 사용됩니다.
- **Color Hex** — **Color Swatch** 유형 속성에만 관련됩니다. 스와치가 올바른 색상을 표시하도록 하기 위해 16진 색상 코드(예: `#FF0000`은 빨강)를 입력합니다.
- **Sort Order** — 선택자에서 값이 나타나는 순서를 제어합니다. 먼저 나타나기를 원하는 값에 더 작은 숫자를 할당합니다.

### 논리적인 순서로 값 정렬

크기 속성의 경우, 작은 크기에서 큰 크기로 정렬 순서를 설정합니다:

| Value | Sort Order |
|---|---|
| XS | 1 |
| S | 2 |
| M | 3 |
| L | 4 |
| XL | 5 |
| 2XL | 6 |

색상 속성의 경우, 알파벳 순으로 정렬하거나 유사한 색상을 그룹화할 수 있습니다 — 고객에게 가장 의미 있는 방식으로 정렬합니다.

## 별도로 속성 값 관리

**Catalog > Attribute Values**에서 속성 값을 독립적으로 관리할 수도 있습니다. 이 목록은 캐탈로그에서 특정 값을 찾거나 업데이트할 때 각 속성을 개별적으로 열지 않고도 유용합니다. 이 목록은 속성 이름으로 필터링할 수 있습니다.

## 제품에 속성 할당

속성은 전역이 아닌 제품 수준에서 할당됩니다. 제품에 속성을 추가하려면:

1. **카탈로그 > 제품**으로 이동하고 가변 제품을 엽니다
2. **변경 사항** 탭에서 **속성** 섹션을 찾습니다
3. 추가하려는 속성을 선택합니다
4. 이 제품에 사용할 속성의 값을 선택합니다
5. 제품을 저장하면 Spwig는 대응하는 변형 조합을 생성합니다

제품 변형을 설정하는 방법에 대한 자세한 지침은 **제품 변형** 도움 주제를 참조하십시오.

## 실용적인 예시

### 예시: 옷 사이즈 속성

| 필드 | 값 |
|---|---|
| 이름 | Size |
| 유형 | 버튼 그룹 |
| 필수 여부 | 예 |
| 정렬 순서 | 1 |
| 값 | XS (1), S (2), M (3), L (4), XL (5), 2XL (6) |

### 예시: 색상 스와치 속성

| 필드 | 값 |
|---|---|
| 이름 | Colour |
| 유형 | Color Swatch |
| 필수 여부 | 예 |
| 정렬 순서 | 2 |
| 값 | Black (#000000), White (#FFFFFF), Navy (#001F5B), Red (#CC0000) |

### 예시: 재질 속성

| 필드 | 값 |
|---|---|
| 이름 | Material |
| 유형 | 드롭다운 선택 |
| 필수 여부 | 아니오 |
| 정렬 순서 | 3 |
| 값 | 100% Cotton, Cotton/Polyester Blend, Merino Wool, Linen |

## 팁

모든 마크다운 형식, 이미지 경로, 코드 블록 및 기술 용어를 유지합니다.

- 고객이 실제로 구매 결정에 영향을 주는 속성을 생성하세요 — 고객이 선택할 필요가 없는 경우, 그 속성은 필요하지 않을 수 있습니다
- 카탈로그 전반에서 일관된 이름을 사용하세요: 일부 제품이 "Colour"를 사용하고 다른 제품이 "Color"를 사용하는 경우, 고객과 팀이 이 불일치를 혼동할 수 있습니다
- 속성과 값의 정렬 순서가 중요합니다 — 일반적으로 가장 중요한 속성(보통은 Size 또는 Colour)을 먼저 배치하고, 값을 논리적인 순서로 정렬하세요
- Color Swatch 유형은 정확한 헥스 코드가 필요합니다; 저장하기 전에 브라우저의 색상 선택기를 사용해 색상을 테스트하여 스와치가 실제 제품 색상과 일치하는지 확인하세요
- 속성을 이름을 변경해야 하는 경우(예: "Color"에서 "Colour"로), 새로운 속성을 생성하는 대신 **Name** 필드를 업데이트하세요 — 이름을 변경하는 것은 기존 제품 할당에 영향을 주지 않습니다
