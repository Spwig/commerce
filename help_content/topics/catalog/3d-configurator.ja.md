---
title: 3D製品構成設定
---

3D構成設定は、顧客が製品ページ上でインタラクティブな3Dビューアーを使用して、設定可能な製品を表示できるようにします。顧客が色、素材、またはコンポーネントのバリエーションなどのオプションを選択するたびに、3Dモデルはリアルタイムで更新され、顧客の選択を反映します。サポートされているモバイルデバイスでは、顧客は購入する前に、拡張現実（AR）で製品を自分の空間に仮置きして視認することもできます。

3D構成設定は、設定可能な製品と連携して動作します。各設定可能な製品には、製品の設定オプションにGLBモデルファイルをリンクする1つの3Dシーン設定が存在できます。

## 事前準備

3Dシーンを設定するには、以下が必要です：

- カタログにすでに作成された **設定可能な製品**
- **ベースの3Dモデル** としてメディアライブラリにアップロードされたGLBファイル — これは、デフォルトで表示される組み立て済みモデルです
- 必要に応じて、ジオメトリ交換用の追加GLBファイル（例：異なる襟の形状）や、素材のバリエーション用のテクスチャ画像

設定可能な製品とその設定オプションを作成していない場合は、3Dシーンの設定を開始する前にそれを作成してください。

## シーン設定の作成

1. **カタログ > 3Dシーン設定** に移動します
2. **+ 3Dシーン設定の追加** をクリックします
3. このシーンが所属する **製品** を選択します — 設定可能な製品のみが利用可能です
4. メディアライブラリから **ベースの3Dモデル** を選択します — これは、デフォルトで読み込まれるGLBファイルです
5. ビューアーの設定を構成します（下記を参照）
6. レコードを保存します

保存後、**ノードツリー** フィールドが自動的に埋め込まれます。これは、あなたのGLBファイルから抽出された解析済みシーングラフです — モデル内のすべての名前付きノードを一覧表示し、ノードマッピングを追加するときに参照します。

## ビューアー設定

これらの設定は、製品ページ上に3Dビューアーが表示される方法を制御します。

### カメラと照明


| フィールド | 説明 | デフォルト |
|-------|-------------|---------|
| **カメラ軌道** | カメラの初期位置で、`angle elevation distance`の形式で指定します（例: `0deg 75deg 2m`） | `0deg 75deg 2m` |
| **カメラ対象** | モデルの中心からメートル単位で測ったカメラが注視する点（例: `0m 0m 0m`） | `0m 0m 0m` |
| **環境画像** | メディアライブラリから選んだHDR画像で、画像ベースの照明を使用します — 反射や影がより現実的になります | None |
| **露出** | シーン全体の明るさ — 値が低いほど暗く、高いほど明るくなります | `1.0` |

### シャドウ

| フィールド | 説明 | デフォルト |
|-------|-------------|---------|
| **シャドウの強さ** | モデルの下に現れるシャドウの強さ — `0`はシャドウなし、`1`は最大の強さ | `0.5` |
| **シャドウの柔らかさ** | シャドウのエッジのぼかし具合 — `0`ははっきり、`1`は非常に柔らかくぼかされます | `0.5` |

### カラーグレーディング

| フィールド | 説明 |
|-------|-------------|
| **トーンマッピング** | シーンに適用されるカラーグレーディングアルゴリズム。**Commerce**は商品に適した鮮やかな色を生成します。**Neutral**は正確な色を再現します。**ACES**は映画のような映像的な見た目を与えます。 |
| **ブルームの強さ** | モデルの自発的に光る部分に光のエフェクトを加えます。`0`でブルームを無効にします。`1`から`5`の値は、控えめな光から劇的な光まで変化します。 |

### 行為と背景

| フィールド | 説明 | デフォルト |
|-------|-------------|---------|
| **自動回転** | モデルが読み込まれるときにゆっくり回転して、顧客の注意を引くかどうか | On |
| **AR有効** | サポートされているデバイスの顧客が**ARで見る**ボタンを表示するかどうか | On |
| **背景** | ビューアーの背景色またはCSSグラデーション — 16進数の色（例: `#f5f5f5`）またはCSSグラデーション値を入力します | `#ffffff` |

### サムネイル

**サムネイル**フィールドは、3Dビューアーのプレビューとして表示されるスクリーンショットを保持します。ビューアーが読み込まれる前に表示されます。

ライブ商品ページからスクリーンショットをキャプチャし、メディアライブラリにアップロードして、ここにリンクすることで、ページの読み込みがよりスムーズになります。

## 3Dビューアーの有効化と無効化

**有効**のトグルは、商品ページに3Dビューアーが表示されるかどうかを制御します。無効にすると、商品は標準的な2D画像設定ツールに戻ります。これにより、顧客に表示する前にシーン設定を準備できます。

## 設定オプションを3Dアクションに接続する

ベースのシーンが設定された後、各設定スロットオプションを3Dモデルの視覚的な変更にリンクできます。これらのリンクは**ノードマッピング**と呼ばれ、シーン設定フォームの下部にある**ノードマッピング**セクションで追加します。

### ノードマッピングフィールド

| フィールド | 説明 |
|-------|-------------|
| **スロットオプション** | この変更をトリガーする設定オプション（例: "レッドレザー"） |
| **アクションタイプ** | 視覚的な変更が何であるか（下記のアクションタイプを参照） |
| **ターゲットノード** | 変更されるシーングラフノードの名前 — **ノードツリー**にリストされている名前から選択 |
| **アクションデータ** | 色のHEXコード、テクスチャURL、またはGLBファイルURLなどのアクション固有のデータ |
| **並べ替え順序** | 同じオプションの複数のマッピングが適用される順序を制御 |

### アクションタイプ

| アクション | その動作 |
|--------|-------------|
| **マテリアルカラー** | ターゲットノードのマテリアルの色を変更 — **アクションデータ**にHEXカラーを提供 |
| **マテリアルテクスチャ** | マテリアルに適用されているテクスチャを交換 — **アクションデータ**にテクスチャ画像アセットへのリンクを提供 |
| **ジオメトリ交換** | モデルの一部を別のGLBファイルで置き換え — ハンドルの形状などの構造的な変更に有用 |
| **可視性** | シーン内のノードを表示または非表示にする — **アクションデータ**に `visible: true` または `visible: false` を設定 |

1つのスロットオプションに対して複数のマッピングを追加できます。

たとえば、「ブルー・デニム」を選択すると、素材の色が変更される*かつ*革のトリムノードが同時に非表示になる場合があります。

## Geometry assets

構成に**Geometry Swap**アクションが含まれている場合、置き換え用のGLBファイルを**Geometry Assets**として登録する必要があります。これらは、シーン構成フォームの**Geometry Assets**セクションに追加されます。

| Field | Description |
|-------|-------------|
| **Label** | このgeometry assetの説明的な名前、例: "V-Neck Collar" |
| **GLB File** | メディアライブラリから取得した置き換え用GLBファイル |
| **Target Node** | このgeometryが置き換えるベースモデルのノード |

Geometry Assetを保存すると、GLBからノード名が解析され、**Node Data**に保存され、マッピングでターゲットノードとして使用可能になります。

## Texture assets

**Material Texture**マッピングで使用されるテクスチャ画像は、参照しやすくするために**Texture Assets**として登録できます。これらは**Texture Assets**セクションに追加されます。

| Field | Description |
|-------|-------------|
| **Label** | 説明的な名前、例: "Red Leather" |
| **Texture Image** | メディアライブラリから取得したテクスチャ画像 |
| **Texture Type** | このテクスチャが適用されるPBRチャネル — Base Color、Normal Map、Roughness Map、Metalness Map、Ambient Occlusion、またはEmissive Map |

## Example: configurable jacket with colour options

**Scenario:** 黒、ネイビー、ボルドーの3色から選べるジャケットで、各色がジャケットのボディメッシュに適用されます。

**Setup:**

1. ジャケット製品のシーン構成を作成し、組み立てられたジャケットGLBをベースモデルとして指定します
2. **Tone Mapping**をCommerceに、**Auto Rotate**をonに設定します
3. Node Mappingsで3つのエントリを追加します — 各色オプションごとに1つ:


| Slot Option | Action Type | Target Node | Action Data |
|-------------|-------------|-------------|-------------|
| Black | Material Color | JacketBody | `{"color": "#1a1a1a"}` |
| Navy | Material Color | JacketBody | `{"color": "#1b2a4a"}` |
| Burgundy | Material Color | JacketBody | `{"color": "#6b2737"}` |

顧客が商品ページでNavyを選択すると、ビューアーは即座にJacketBodyの素材をネイビーの色に更新します。

## Tips

- 3Dモデルを作成する際、GLBノードの名前を明確に指定してください — "JacketBody"や"CollarMesh"などの名前は、"Mesh_023"などの自動生成された名前よりも扱いがはるかに簡単です
- 多くの商品に対して**Commerce**トーンマッピングを使用してください — これは、魅力的で引き込まれる商品プレゼンテーションに最適化されています
- デフォルトのカメラ角度がすでに重要な特徴を表示している商品では、**Auto Rotate**を無効にしてください — これにより、ロード時に顧客が混乱しないようにします
- ARボタンを実際のモバイルデバイスでテストしてから公開してください — ARの利用可能性は顧客のデバイスとブラウザに依存しており、WebXRをサポートするiOS SafariとAndroid Chromeが最も信頼性が高いです
- すべてのシーン構成に対して**Thumbnail**画像をアップロードしてください — これにより、3Dビューアーが読み込まれる間に空白の白いボックスが一瞬表示されるのを防げます
- 3Dビューアーがまだ準備されていない場合は、**Enabled**トグルを使用して無効にしてください — これにより、顧客は標準的な画像設定ツールを表示します
