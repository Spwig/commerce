---
title: 3D 제품 구성기
---

3D 구성기는 고객이 제품 페이지에서 직접 상호작용하는 3D 뷰어를 통해 구성 가능한 제품을 볼 수 있도록 합니다. 고객이 색상, 재질, 구성 요소 변형과 같은 옵션을 선택할 때, 3D 모델은 실시간으로 고객의 선택을 반영하여 업데이트됩니다. 지원되는 모바일 기기에서는 고객이 구매하기 전에 제품을 증강현실(AR)에서 가상으로 자신의 공간에 배치할 수도 있습니다.

3D 구성기는 구성 가능한 제품과 함께 작동합니다. 각 구성 가능한 제품은 하나의 3D 시나리오 구성이 있을 수 있으며, 이 구성은 제품의 구성 옵션에 GLB 모델 파일을 연결합니다.

## 시작하기 전에

3D 시나리오를 설정하려면 다음이 필요합니다:

- 이미 카탈로그에 생성된 **구성 가능한 제품**
- **기본 3D 모델** — 이는 기본적으로 표시되는 조립된 모델로, 미디어 라이브러리에 GLB 파일로 업로드되어야 합니다
- 선택적으로, 기하학적 교체를 위한 추가 GLB 파일(예: 다른 목소리 형태) 및 재질 변형을 위한 텍스처 이미지

만약 구성 가능한 제품과 그 구성 옵션을 아직 생성하지 않았다면, 먼저 3D 시나리오를 설정하기 전에 이를 생성해야 합니다.

## 시나리오 구성 생성

1. **카탈로그 > 3D 시나리오 구성**으로 이동합니다
2. **+ 3D 시나리오 구성 추가**를 클릭합니다
3. 이 시나리오가 속한 **제품**을 선택합니다 — 구성 가능한 제품만 선택할 수 있습니다
4. 미디어 라이브러리에서 **기본 3D 모델**을 선택합니다 — 이는 기본적으로 로드되는 GLB 파일입니다
5. 뷰어 설정을 구성합니다(아래 참조)
6. 기록을 저장합니다

기록을 저장한 후 **노드 트리** 필드가 자동으로 채워집니다. 이는 GLB 파일에서 추출된 파싱된 시나리오 그래프이며, 모델 내부의 모든 이름이 지정된 노드를 나열합니다. 노드 매핑을 추가할 때 이 노드를 참조하게 됩니다.

## 뷰어 설정

이 설정은 3D 뷰어가 제품 페이지에 어떻게 표시되는지를 제어합니다.

### 카메라 및 조명


| 필드 | 설명 | 기본값 |
|-------|-------------|---------|
| **카메라 궤도** | `각도 높이 거리` 형식의 시작 카메라 위치 (예: `0deg 75deg 2m`) | `0deg 75deg 2m` |
| **카메라 대상** | 모델 중심에서 미터 단위로 측정된 카메라가 바라보는 지점 (예: `0m 0m 0m`) | `0m 0m 0m` |
| **환경 이미지** | 이미지 기반 조명을 위해 미디어 라이브러리에서 가져온 HDR 이미지 — 더 현실적인 반사와 그림자를 제공합니다 | 없음 |
| **노출** | 장면의 전체 밝기 — 낮은 값은 어둡고, 높은 값은 밝습니다 | `1.0` |

### 그림자

| 필드 | 설명 | 기본값 |
|-------|-------------|---------|
| **그림자 강도** | 모델 아래에 떨어지는 그림자의 강도 — `0`은 그림자가 없고, `1`은 최대 강도입니다 | `0.5` |
| **그림자 부드러움** | 그림자 가장자리가 얼마나 흐릿한지 — `0`은 날카로우며, `1`은 매우 부드럽습니다 | `0.5` |

### 색상 보정

| 필드 | 설명 |
|-------|-------------|
| **톤 매핑** | 장면에 적용되는 색상 보정 알고리즘. **Commerce**는 생동감 있고 제품에 적합한 색상을 제공합니다. **Neutral**은 색상 정확도를 유지합니다. **ACES**는 영화 같은 시네마틱 효과를 제공합니다. |
| **블룸 강도** | 모델의 자가 조명 부분에 빛나는 효과를 추가합니다. `0`은 블룸을 비활성화합니다. `1`에서 `5` 사이의 값은 미묘한 빛나는 효과에서 극적인 효과까지 제공합니다. |

### 행동 및 배경

| 필드 | 설명 | 기본값 |
|-------|-------------|---------|
| **자동 회전** | 모델이 로드될 때 천천히 회전하여 고객의 주의를 끌도록 하는지 여부 | On |
| **AR 활성화** | 지원되는 장치에서 고객이 **AR에서 보기** 버튼을 볼 수 있는지 여부 | On |
| **배경** | 뷰어의 배경 색상 또는 CSS 그라디언트 — 16진수 색상 (예: `#f5f5f5`) 또는 CSS 그라디언트 값을 입력합니다 | `#ffffff` |

### 썸네일

**썸네일** 필드는 3D 뷰어의 미리보기 스크린샷을 저장합니다. 뷰어가 로드되기 전에 표시됩니다.

라이브 제품 페이지에서 스크린샷을 캡처하여 미디어 라이브러리에 업로드한 후 여기에 링크하여 페이지 로드 경험을 더 부드럽게 만들 수 있습니다.

## 3D 뷰어 활성화 및 비활성화

**활성화** 토글은 제품 페이지에 3D 뷰어가 표시되는지 여부를 제어합니다. 비활성화되면 제품은 표준 2D 이미지 구성기로 돌아갑니다. 이는 제품을 고객에게 표시하기 전에 시나리오 구성이 완료되었는지 확인할 수 있습니다.

## 구성 옵션을 3D 액션과 연결하기

기본 시나리오가 구성되면 각 구성 슬롯 옵션을 3D 모델의 시각적 변경으로 연결할 수 있습니다. 이러한 연결은 **노드 매핑**(Node Mappings)이라고 하며, 시나리오 구성 폼 하단의 **노드 매핑**(Node Mappings) 섹션에서 추가합니다.

### 노드 매핑 필드

| 필드 | 설명 |
|-------|-------------|
| **슬롯 옵션** | 이 변경을 트리거하는 구성 옵션(예: "레드 레더") |
| **액션 유형** | 시각적 변경이 무엇인지(아래의 액션 유형을 참조) |
| **대상 노드** | 변경되는 시나리오 그래프 노드의 이름 — **노드 트리**(Node Tree)에 나열된 이름 중에서 선택 |
| **액션 데이터** | 색상 헥스 코드, 텍스처 URL 또는 GLB 파일 URL과 같은 액션에 따른 데이터 |
| **정렬 순서** | 동일한 옵션에 대한 여러 매핑이 적용되는 순서를 제어 |

### 액션 유형

| 액션 | 기능 |
|--------|-------------|
| **Material Color** | 대상 노드의 재질 색상을 변경 — **액션 데이터**에 헥스 색상을 제공 |
| **Material Texture** | 재질에 적용된 텍스처를 교체 — **액션 데이터**에 텍스처 이미지 자산을 연결 |
| **Geometry Swap** | 모델의 일부를 다른 GLB 파일로 교체 — 다른 핸들 모양과 같은 구조적 변경에 유용 |
| **Visibility** | 시나리오에서 노드를 표시하거나 숨김 — **액션 데이터**에 `visible: true` 또는 `visible: false`를 설정 |

하나의 슬롯 옵션에 여러 매핑을 추가할 수 있습니다.

예를 들어, "Blue Denim"을 선택하면 재질 색상 *및* 동시에 가죽 트림 노드를 숨길 수 있습니다.

## Geometry assets

구성에 **Geometry Swap** 작업이 포함되어 있다면, 대체 GLB 파일을 **Geometry Assets**로 등록해야 합니다. 이는 시나리오 구성 폼의 **Geometry Assets** 섹션에 추가됩니다.

| Field | Description |
|-------|-------------|
| **Label** | 이 geometry asset에 대한 설명적인 이름, 예: "V-Neck Collar" |
| **GLB File** | 미디어 라이브러리에서 가져온 대체 GLB 파일 |
| **Target Node** | 이 geometry가 대체하는 기본 모델의 노드 |

Geometry Asset을 저장한 후, GLB에서 노드 이름이 파싱되어 **Node Data**에 저장되며, 매핑에서 대상 노드로 사용할 수 있게 됩니다.

## Texture assets

**Material Texture** 매핑에서 사용하는 텍스처 이미지는 더 쉽게 참조할 수 있도록 **Texture Assets**로 등록할 수 있습니다. 이는 **Texture Assets** 섹션에 추가됩니다.

| Field | Description |
|-------|-------------|
| **Label** | 설명적인 이름, 예: "Red Leather" |
| **Texture Image** | 미디어 라이브러리에서 가져온 텍스처 이미지 |
| **Texture Type** | 이 텍스처가 적용되는 PBR 채널 — Base Color, Normal Map, Roughness Map, Metalness Map, Ambient Occlusion, 또는 Emissive Map |

## 예시: 색상 옵션을 가진 조정 가능한 자켓

**시나리오:** Black, Navy, Burgundy 색상으로 주문 가능한 자켓이며, 각 색상은 자켓 본체 메시에 적용됩니다.

**설정:**

1. 자켓 제품에 대한 시나리오 구성에서 조립된 자켓 GLB를 기본 모델로 설정합니다.
2. **Tone Mapping**을 Commerce로 설정하고 **Auto Rotate**을 켜줍니다.
3. Node Mappings에서 세 가지 항목을 추가합니다 — 각 색상 옵션 하나씩:


| 슬롯 옵션 | 액션 유형 | 대상 노드 | 액션 데이터 |
|-------------|-------------|-------------|-------------|
| Black | Material Color | JacketBody | `"{\"color\": \"#1a1a1a\"}"` |
| Navy | Material Color | JacketBody | `"{\"color\": \"#1b2a4a\"}"` |
| Burgundy | Material Color | JacketBody | `"{\"color\": \"#6b2737\"}"` |

고객이 제품 페이지에서 Navy를 선택하면, 뷰어는 즉시 JacketBody 재질을 네이비 색으로 업데이트합니다.

## Tips

- 3D 모델을 생성할 때 GLB 노드를 명확하게 이름지어주세요 — "JacketBody" 또는 "CollarMesh"와 같은 노드 이름은 "Mesh_023"과 같은 자동 생성 이름보다 훨씬 작업하기 쉬운데요
- 대부분의 제품에 대해 **Commerce** 톤 매핑을 사용하세요 — 이 것은 생동감 있고 매력적인 제품 전시를 위해 조정된 것입니다
- 기본 카메라 각도가 이미 중요한 기능을 보여주는 제품의 경우 **Auto Rotate**를 비활성화하세요 — 로드 시 고객을 혼란스럽게 하지 않도록 하세요
- AR 버튼을 실제로 모바일 기기에서 테스트한 후에 홍보하기 전에 — AR 사용 가능 여부는 고객의 기기와 브라우저에 따라 달라지며 (WebXR 지원이 있는 iOS Safari 및 Android Chrome이 가장 신뢰할 수 있습니다)
- 모든 시나리오 구성에 대해 **Thumbnail** 이미지를 업로드하세요 — 이는 3D 뷰어가 로드될 때 빈 하얀 박스가 깜빡이는 것을 방지합니다
- 3D 뷰어가 아직 준비되지 않은 경우 **Enabled** 토글을 사용해 비활성화하세요 — 고객은 표준 이미지 구성기 대신 보게 됩니다
