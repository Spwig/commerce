---
title: 라이선스 키 관리
---

라이선스 키 관리는 디지털 제품을 구매할 때 소프트웨어 라이선스 키가 어떻게 생성, 저장 및 고객에게 전달되는지를 제어할 수 있게 해줍니다. Spwig는 내장된 키 생성, 사전 로드된 키 풀, 외부 라이선스 관리 서비스와의 통합을 지원합니다.

## 개요

Spwig에서 라이선스 키를 관리하는 방법은 총 세 가지입니다:

| 방법 | 최적의 사용 사례 |
|--------|---------|
| **라이선스 템플릿** | 구매 시 사용자 정의 형식으로 고유한 키를 자동으로 생성할 때 |
| **라이선스 풀** | 대량 분배를 위해 사전에 키 배치를 미리 생성할 때 |
| **외부 제공업체** | Keygen.sh와 같은 제3자 서비스에 키 생성 및 관리를 위임할 때 |

이 방법들은 조합하여 사용할 수 있습니다. 예를 들어, 풀은 사용자 정의 템플릿을 사용하여 키 형식을 정의하고, 생성된 키를 외부 제공업체와 동기화할 수 있습니다.

## 라이선스 키 템플릿

라이선스 키 템플릿은 생성된 키의 *형식*을 정의합니다. 템플릿은 Spwig이 생성 시 채우는 플레이스홀더가 포함된 패턴을 사용합니다.

### 템플릿 생성

1. **카탈로그 > 라이선스 키 템플릿**으로 이동합니다.
2. **+ 라이선스 키 템플릿 추가**를 클릭합니다.
3. **이름**을 입력합니다 (예: `Standard App License`)
4. 플레이스홀더를 사용하여 **패턴**을 구성합니다 (아래 참조)
5. 필요 시 **프리픽스** 및 **서픽스**를 설정합니다 (예: 프리픽스 `MYAPP`은 모든 키에 `MYAPP-`을 추가합니다)
6. **구분자** 문자를 선택합니다 (기본값: `-`)
7. **문자 집합**을 설정합니다 - 랜덤 섹션에 사용되는 문자. 기본값은 `0` 및 `O`, `1` 및 `I`와 같은 모호한 문자를 제외합니다.
8. 검증을 위한 **최소/최대 길이**를 설정합니다.
9. **저장**을 클릭합니다.

### 패턴 플레이스홀더

모든 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어는 보존합니다.

| Placeholder | Description | Example output |
|-------------|-------------|---------------|
| `{RANDOM:N}` | N random characters from the character set | `{RANDOM:5}` → `K7JXQ` |
| `{CHECKSUM:N}` | N-digit checksum for validation | `{CHECKSUM:2}` → `47` |
| `{PREFIX}` | The template's prefix value | `MYAPP` |
| `{SUFFIX}` | The template's suffix value | `PRO` |
| `{ORDER_ID}` | The order number | `10045` |
| `{PRODUCT_SKU}` | The product's SKU | `SOFTPRO` |
| `{DATE:FORMAT}` | Formatted date | `{DATE:YYMMDD}` → `260318` |

**Example pattern**: `{PREFIX}-{RANDOM:5}-{RANDOM:5}-{RANDOM:5}-{CHECKSUM:2}`

This produces keys like: `MYAPP-K7JXQ-M3TPR-9BWKN-47`

### Previewing keys

After saving a template, a **Generate Sample Key** action is available on the template list. Use this to verify your pattern produces keys in the expected format before assigning the template to a product.

## License pools

A license pool is a batch of pre-generated keys for a product. Pools are useful when:
- You need keys for physical packaging (retail boxes, printed cards)
- You work with resellers who need batches of keys
- You want keys generated in advance rather than on demand

### Creating a license pool

1. Navigate to **Catalog > License Pools**
2. Click **+ Add License Pool**
3. Fill in the pool details:

| 필드 | 설명 |
|-------|-------------|
| **이름** | 설명적인 이름 (예: `Retail Pack Q1 2026`) |
| **제품** | 이 키가 적용되는 제품 |
| **라이선스 템플릿** | 키 형식을 위한 템플릿 (기본값은 제품의 템플릿) |
| **총 키 수** | 생성할 키 수 |
| **키 유형** | 영구, 구독, 또는 시험판 |
| **최대 활성화 수** | 각 키가 활성화할 수 있는 기기 수 |
| **활성화 후 만료 일수** | 첫 번째 활성화 후 라이선스가 만료되기까지의 일수 (만료 없이 남겨두려면 빈칸으로 둡니다) |
| **풀 만료 일자** | 이 풀에서 사용되지 않은 키가 유효하지 않게 되는 날짜 |
| **프로바이더와 동기화** | 생성된 키를 외부 라이선스 프로바이더와 선택적으로 동기화 |

4. **저장**을 클릭하세요 — Spwig은 키를 백그라운드에서 생성합니다

### 풀 상태

| 상태 | 의미 |
|--------|---------|
| **생성 중** | 키가 백그라운드에서 생성되고 있습니다 |
| **준비 완료** | 모든 키가 생성되어 분배할 준비가 되었습니다 |
| **모든 키 사용 완료** | 모든 키가 주문에 할당되었습니다 |
| **만료됨** | 풀의 만료 일자가 지났습니다 |

### 풀 모니터링

풀 목록은 분배된 키 수와 생성된 총 키 수를 보여줍니다. 풀을 열면 키의 전체 목록과 각각의 상태를 볼 수 있습니다.

## 외부 라이선스 프로바이더

외부 프로바이더는 키 생성 및 활성화 추적을 처리하는 제3자 라이선스 관리 서비스입니다. 고객이 구매를 완료하면 Spwig는 프로바이더와 통신하여 키를 생성하고 등록합니다.

### 지원되는 프로바이더

| 프로바이더 | 유형 |
|----------|------|
| **Spwig 내장 라이선스 서버** | 내장 — 외부 계정이 필요하지 않음 |
| **Keygen.sh** | 클라우드 기반 라이선스 관리 API |
| **LicenseSpring** | 기업용 라이선스 관리 |
| **Cryptlex** | 오프라인 지원을 포함한 라이선스 관리 |
| **사용자 정의 API** | 모든 REST 기반 라이선스 시스템 |

### 프로바이더 연결

1.

**카탈로그 > 라이선스 프로바이더**로 이동하세요
2.

**+ 라이선스 프로바이더 추가**를 클릭하세요
3.


공급업체 세부 정보를 입력합니다:

| 필드 | 설명 |
|-------|-------------|
| **이름** | 이 연결에 대한 라벨(예: `Keygen Production`) |
| **공급업체 유형** | 지원되는 공급업체 중 하나를 선택합니다 |
| **API 엔드포인트** | 공급업체의 API 기본 URL |
| **API 키** | 공급업체 인증 키 |
| **API 시크릿** | 공급업체가 요구하는 경우 |

4. 동기화 동작을 구성합니다:
   - **주문 시 동기화** — 고객이 구매를 완료할 때 자동으로 동기화합니다
   - **활성화 시 동기화** — 장치 활성화를 공급업체에 보고합니다
   - **비활성화 시 동기화** — 비활성화를 보고합니다(라이선스 이전 및 환불에 유용)
   - **양방향 동기화** — 공급업체가 웹훅을 통해 Spwig 기록을 업데이트하도록 허용합니다

5. **저장**을 클릭한 후 **연결 테스트**를 클릭하여 자격 증명이 작동하는지 확인합니다

### 연결 상태

각 공급업체는 세 가지 연결 상태 중 하나를 표시합니다:

| 상태 | 의미 |
|--------|---------|
| **테스트되지 않음** | 연결이 아직 확인되지 않았습니다 |
| **연결됨** | 마지막 테스트가 성공했습니다 |
| **에러** | 연결 테스트가 실패했습니다 — 오류 메시지를 확인하세요 |

### 기존 라이선스 동기화

초기 설정 또는 실패한 동기화 후 기존 라이선스 키를 수동으로 공급업체에 전송하려면 공급업체 목록에서 **지금 동기화** 작업을 사용합니다.

## 동기화 활동 모니터링

**카탈로그 > 외부 라이선스 동기화**로 이동하여 동기화 로그를 확인합니다. 각 기록은 다음과 같습니다:
- 동기화된 라이선스 키
- 전송된 공급업체
- 방향 (Spwig → 공급업체 또는 공급업체 → Spwig)
- 상태 (대기 중, 성공, 실패)
- 실패한 동기화의 오류 세부 정보

실패한 동기화는 자동으로 재시도됩니다. 기록을 편집하고 오류를 지우면 강제로 재시도할 수도 있습니다.

## 팁

모든 마크다운 형식, 이미지 경로, 코드 블록 및 기술 용어를 유지합니다.

- 고객이 자주 잘못 읽는 모호한 문자를 피하기 위해 기본 문자 집합(`ABCDEFGHJKLMNPQRSTUVWXYZ23456789`)을 사용하세요 — `0`, `O`, `1`, `I`는 제외합니다.
- 템플릿 패턴에 `{CHECKSUM}` 세그먼트를 추가하여 고객과 지원 팀이 잘못 입력된 키를 빠르게 감지할 수 있도록 해주세요.
- 고부하 제품의 경우, 즉시 제공을 보장하기 위해 온디맨드 생성 대신 풀을 사용하세요.
- 계절적 또는 시간 제한이 있는 키 배치에 대해 **Pool Expires At**을 설정하여 오래된 미사용 키가 자동으로 무효화되도록 해주세요.
- 설정 후 및 자격 증명 변경 후 항상 제공자 연결을 테스트하세요 — 연결이 끊어지면 고객이 키를 받지 못하게 됩니다.
- 양방향 동기화를 사용하는 경우, 제공자의 웹후크 URL을 가게의 라이선스 웹후크 엔드포인트로 설정하도록 구성하세요.
