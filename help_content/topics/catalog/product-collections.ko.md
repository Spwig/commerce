---
title: 상품 컬렉션
---

컬렉션은 판매처에서 상품을 함께 표시할 수 있도록 상품을 그룹화하는 데 사용됩니다. 카테고리와 달리 전체 카탈로그를 영구적인 계층 구조로 정리하는 것과는 달리 컬렉션은 특정 목적을 위해 생성하는 유연하고 편집된 그룹입니다. 컬렉션은 신상품을 강조하거나 계절 캠페인을 위한 항목을 보여주거나, 최고 인기 상품을 수작업으로 선별한 선택을 제시할 수 있습니다.

**카탈로그 > 컬렉션**으로 이동하여 컬렉션을 관리하세요.

## 컬렉션 vs 카테고리

카테고리와 컬렉션 모두 상품을 그룹화하지만 목적은 다릅니다:

| | 카테고리 | 컬렉션 |
|---|---|---|
| **목적** | 영구적인 카탈로그 구조 | 유연하고 편집된 그룹 |
| **계층 구조** | 예 — 중첩된 부모/자식 구조 | 아니요 — 평평한 그룹 |
| **그룹당 상품 수** | 각 상품은 하나의 카테고리에 속함 | 하나의 상품이 여러 컬렉션에 나타날 수 있음 |
| **일반적인 사용** | 쇼핑 내비게이션 메뉴, 부서별 검색 | 랜딩 페이지, 캠페인, 특별 추천 세트 |

카테고리를 "쇼핑몰이 어떻게 구성되어 있는지"에 사용하고, 컬렉션을 "지금 강조하고 싶은 항목"에 사용하세요.

## 컬렉션 유형

컬렉션을 생성할 때, 상품 목록을 관리하고자 하는 방식에 맞는 유형을 선택하세요:

| 유형 | 상품 추가 방식 |
|---|---|
| **수동 선택** | 정확히 어떤 상품이 나타나는지 하나씩 선택하여 추가 |
| **자동 규칙** | 정의한 기준에 따라 자동으로 상품이 추가 |
| **추천 상품** | 편집된 기사 형식의 선택, 수동으로 관리 |
| **계절별** | 시간 기반 선택, 일반적으로 캠페인을 위해 수동으로 관리 |

수동 및 추천 유형은 정확한 제어를 제공합니다. 자동 컬렉션은 카탈로그가 성장하면서도 지속적인 유지보수가 필요 없습니다.

## 컬렉션 생성

1.

**카탈로그 > 컬렉션**으로 이동하세요
2.

**+ 컬렉션 추가**를 클릭하세요
3.

모든 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어를 유지하세요.

# 컬렉션 생성 가이드

**기본 정보** 섹션을 작성하세요:
   - **이름** — 스토어프론트에서 표시될 컬렉션의 이름
   - **Slug** — 컬렉션 페이지의 URL 경로(이름에서 자동으로 채워짐; 사용자 정의 가능)
   - **설명** — 컬렉션의 스토어프론트 페이지에 표시되는 설명
4.

**컬렉션 유형**을 선택하세요
5.

제품을 추가하세요:
   - **수동 선택** 및 **추천 제품** 유형의 경우: **제품** 필드를 사용하여 제품을 검색하고 추가하세요
   - **자동** 유형의 경우: **자동 기준** 필드에서 기준을 정의하세요
6.

이미지 업로드:
   - **이미지** — 목록 페이지 및 썸네일에 사용되는 주 컬렉션 이미지
   - **배너 이미지** — 컬렉션 페이지 상단에 표시되는 더 넓은 배너 이미지
7.

**SEO** 필드를 구성하세요(선택 사항이지만 권장됨):
   - **메타 제목** — 검색 결과에 표시되는 페이지 제목
   - **메타 설명** — 제목 아래에 표시되는 설명
8.

**표시 옵션**을 설정하세요:
   - **활성 상태** — 컬렉션이 스토어프론트에 표시되는지 여부를 제어합니다
   - **추천 표시** — 테마에서 추천 위치에 표시될 컬렉션을 표시합니다
   - **정렬 순서** — 목록 페이지에서 컬렉션의 표시 순서를 제어합니다(숫자가 낮을수록 먼저 표시됨)
9.

**저장**을 클릭하세요

## 컬렉션에 제품 추가

수동 컬렉션의 경우, **제품** 자동 완성 필드를 사용하여 카탈로그에서 제품을 검색하고 항목을 선택하세요. 필요한 만큼 많은 제품을 추가할 수 있습니다 — 제한이 없습니다.

제품은 여러 컬렉션에 동시에 속할 수 있습니다. 예를 들어, 제품은 "여름 세일" 컬렉션과 "베스트셀러" 컬렉션 모두에 속할 수 있으며, 이는 충돌이 없습니다.

## 스토어프론트에서 컬렉션 표시

각 컬렉션은 자동으로 `/collection/{slug}/`에 해당하는 페이지를 가지게 됩니다. 네비게이션 메뉴, 페이지 빌더 또는 프로모션 배너에서 컬렉션 페이지로 링크할 수 있습니다.

**특징 표시** 플래그는 주제가 특징적인 위치에 나타나는 컬렉션을 결정하는 데 사용됩니다. 예를 들어, 홈페이지의 강조된 컬렉션 그리드입니다.

주제 문서를 확인하여 특징 컬렉션이 정확히 어떻게 표시되는지 알아보세요.

## 컬렉션 가시성 관리

- **활성화**는 컬렉션 페이지가 대중에게 공개되는지 여부를 제어합니다. 비활성화된 컬렉션은 고객에게 숨겨지지만 관리자에서 보관되어 나중에 다시 활성화할 수 있습니다.
- **정렬 순서**는 컬렉션이 목록 페이지에 나타나는 순서를 결정합니다. 먼저 나타나기를 원하는 컬렉션에 더 작은 숫자를 할당하세요.

## 컬렉션의 SEO

각 컬렉션에는 자체 **메타 제목** 및 **메타 설명** 필드가 있습니다. 이 필드는 누군가 컬렉션 페이지를 찾을 때 검색 엔진 결과에 표시되는 내용을 제어합니다. 이 필드를 비워두면 주제가 일반적으로 컬렉션 이름과 설명으로 돌아갑니다.

좋은 컬렉션 SEO 제목은 설명적이고 구체적이어야 합니다:
- "2026 여름 드레스 — 꽃무늬 및 가벼운 스타일"은 "여름 컬렉션"보다 더 잘 작동합니다.
- "남성용 러닝 신발 — 가벼운 및 통기성"은 "러닝 신발"보다 더 잘 작동합니다.

## 팁

- 컬렉션 이름은 짧고 명확하게 유지하세요 — 이는 쇼핑몰 내비게이션에서 페이지 제목 및 링크 텍스트로 나타납니다.
- 계절 또는 캠페인 컬렉션을 사용하고 시작 및 종료 계획을 세우세요: 컬렉션을 생성하고 캠페인 시작 시 활성화하고, 종료 시 비활성화(삭제하지 않고)하여 나중에 참조할 수 있도록 하세요.
- **정렬 순서** 필드는 의도적으로 설정하는 것이 좋습니다 — 기본값은 모든 컬렉션에 0이며, 이는 알파벳순으로 정렬하게 됩니다.

# 특정 숫자를 할당하여 어떤 컬렉션이 가장 두드러지게 표시되는지 제어
- 제품이 없는 컬렉션은 고객에게 빈 페이지를 표시합니다 — 활성화하기 전에 제품을 추가하거나, 컬렉션이 준비될 때까지 비활성 상태로 두는 것이 좋습니다
- 진짜로 강조하고 싶은 컬렉션만 **Is Featured** 플래그를 체크하세요; 대부분의 테마는 소수의 컬렉션에만 특징 슬롯을 예약하고, 너무 많은 컬렉션에 플래그를 지정하면 표시가 혼잡해 보일 수 있습니다
