---
title: 3D-конфигуратор продукта
---

3D-конфигуратор позволяет вашим клиентам просматривать настраиваемые продукты в интерактивном 3D-просмотрщике прямо на странице продукта. По мере того как клиенты выбирают опции — такие как цвета, материалы или варианты компонентов — 3D-модель обновляется в реальном времени, чтобы отразить их выбор. На поддерживаемых мобильных устройствах клиенты также могут просматривать продукт в дополненной реальности (AR), размещая его виртуально в своем пространстве перед покупкой.

3D-конфигуратор работает с настраиваемыми продуктами. Каждый настраиваемый продукт может иметь одну конфигурацию 3D-сцены, которая связывает файл модели GLB с вариантами конфигурации продукта.

## Перед началом

Чтобы настроить 3D-сцену, вам понадобится:

- **настраиваемый продукт**, уже созданный в вашем каталоге
- **базовая 3D-модель**, загруженная в вашу библиотеку медиа как файл GLB — это собранный модель, которая отображается по умолчанию
- по желанию, дополнительные файлы GLB для замены геометрии (например, разные формы воротников), и текстурные изображения для вариантов материалов

Если вы еще не создали настраиваемый продукт и его варианты конфигурации, сделайте это сначала, прежде чем настраивать 3D-сцену.

## Создание конфигурации сцены

1. Перейдите в **Каталог > Конфигурации 3D-сцен**
2. Нажмите **+ Добавить конфигурацию 3D-сцены**
3. Выберите **Продукт**, к которому относится эта сцена — доступны только настраиваемые продукты
4. Выберите **Базовую 3D-модель** из вашей библиотеки медиа — это файл GLB, который загружается по умолчанию
5. Настройте параметры просмотра (см. ниже)
6. Сохраните запись

После сохранения поле **Дерево узлов** заполняется автоматически. Это синтаксически разобранный граф сцены, извлеченный из вашего файла GLB — он перечисляет все именованные узлы внутри модели, которые вы будете использовать при добавлении сопоставлений узлов.

## Настройки просмотра

Эти настройки управляют тем, как 3D-просмотрщик отображается на странице продукта.

### Камера и освещение

| Параметр | Описание | Значение по умолчанию |
|---------|---------|------------------|
| **Орбита камеры** | Начальная позиция камеры в формате `угол подъем расстояние` (например, `0deg 75deg 2m`) | `0deg 75deg 2m` |
| **Цель камеры** | Точка, на которую смотрит камера, в метрах от центра модели (например, `0m 0m 0m`) | `0m 0m 0m` |
| **Изображение окружения** | HDR-изображение из вашей библиотеки медиа, используемое для освещения на основе изображения — обеспечивает более реалистичные отражения и тени | Нет |
| **Экспозиция** | Общая яркость сцены — меньшие значения темнее, большие значения ярче | `1.0` |

### Тени

| Параметр | Описание | Значение по умолчанию |
|---------|---------|------------------|
| **Интенсивность тени** | Сила тени, отбрасываемой моделью — `0` означает отсутствие тени, `1` — полную интенсивность | `0.5` |
| **Размытость тени** | Степень размытости краев тени — `0` означает резкие, `1` — очень размытые | `0.5` |

### Цветовая коррекция

| Параметр | Описание |
|---------|---------|
| **Маппинг тона** | Алгоритм цветовой коррекции, применяемый к сцене. **Коммерческий** производит яркие, ориентированные на продукт цвета. **Нейтральный** обеспечивает точность цвета. **ACES** дает кинематографический вид фильма. |
| **Сила свечения** | Добавляет эффект свечения к излучающим (самосветящимся) частям модели. `0` отключает свечение. Значения между `1` и `5` производят незначительное до драматичное свечение. |

### Поведение и фон

| Параметр | Описание | Значение по умолчанию |
|---------|---------|------------------|
| **Автовращение** | Определяет, вращается ли модель медленно при загрузке, чтобы привлечь внимание клиента | Включено |
| **Включение AR** | Определяет, видят ли клиенты на поддерживаемых устройствах кнопку **Просмотреть в AR** | Включено |
| **Фон** | Цвет или градиент CSS фона просмотра — введите шестнадцатеричный цвет (например, `#f5f5f5`) или значение градиента CSS | `#ffffff` |

### Миниатюра

Поле **Миниатюра** содержит предварительный снимок экрана 3D-просмотрщика, который отображается до загрузки просмотра. Вы можете сделать снимок экрана с живой страницы продукта и загрузить его в вашу библиотеку медиа, а затем связать его здесь для более плавного опыта загрузки страницы.

## Включение и отключение 3D-просмотрщика

Переключатель **Включено** управляет тем, отображается ли 3D-просмотрщик на странице продукта.

Когда функция отключена, продукт переходит к стандартному 2D-конфигуратору изображений.

Это позволяет вам подготовить настройку сцены до того, как она станет видна клиентам.

## Связь параметров конфигурации с 3D-действиями

После настройки базовой сцены вы можете связать каждый параметр слота конфигурации с визуальным изменением в 3D-модели. Эти связи называются **Node Mappings** и добавляются в раздел **Node Mappings**, расположенном в нижней части формы настройки сцены.

### Поля сопоставления узлов

| Поле | Описание |
|-------|-------------|
| **Опция слота** | Параметр конфигурации, который запускает это изменение (например, "Красная кожа") |
| **Тип действия** | Какое визуальное изменение происходит (см. типы действий ниже) |
| **Целевой узел** | Имя узла сцены, который изменяется — выберите из названий, перечисленных в вашем **Node Tree** |
| **Данные действия** | Данные, специфичные для действия, такие как код цвета в формате HEX, URL текстуры или URL файла GLB |
| **Порядок сортировки** | Управляет порядком применения нескольких сопоставлений для одной и той же опции |

### Типы действий

| Действие | Что оно делает |
|--------|-------------|
| **Цвет материала** | Изменяет цвет материала на целевом узле — укажите цвет в формате HEX в **Данных действия** |
| **Текстура материала** | Заменяет текстуру, применённую к материалу — свяжите с текстурным изображением в **Данных действия** |
| **Замена геометрии** | Заменяет часть модели на другой файл GLB — полезно для структурных изменений, например, для другого формы ручки |
| **Видимость** | Показывает или скрывает узел в сцене — установите `visible: true` или `visible: false` в **Данных действия** |

Можно добавить несколько сопоставлений для одной опции слота. Например, выбор "Синее деним" может изменить цвет материала *и* скрыть узел отделки из кожи одновременно.

## Геометрические активы

Если ваша конфигурация включает действия **Замена геометрии**, вам нужно зарегистрировать заменяемые файлы GLB как геометрические активы. Они добавляются в раздел **Геометрические активы** на форме настройки сцены.

| Поле | Описание |
|-------|-------------|
| **Метка** | Описательное имя этого геометрического актива, например, "V-образный воротник" |
| **Файл GLB** | Заменяемый файл GLB из вашей медиатеки |
| **Целевой узел** | Какой узел в базовой модели заменяет эта геометрия |

После сохранения геометрического актива, имена узлов извлекаются из GLB и сохраняются в **Node Data**, делая их доступными как целевые узлы в ваших сопоставлениях.

## Текстурные активы

Текстурные изображения, используемые в сопоставлениях **Текстура материала**, могут быть зарегистрированы как текстурные активы для удобного обращения. Они добавляются в раздел **Текстурные активы**.

| Поле | Описание |
|-------|-------------|
| **Метка** | Описательное имя, например, "Красная кожа" |
| **Текстурное изображение** | Текстурное изображение из вашей медиатеки |
| **Тип текстуры** | Канал PBR, к которому применяется эта текстура — Базовый цвет, Нормальный карт, Карты шероховатости, Карты металлического оттенка, Окклюзия окружающего освещения или Эмиссионная карта |

## Пример: настраиваемый жилет с цветовыми вариантами

**Сценарий:** Жилет, который можно заказать в чёрном, синем или бордовом цвете, с каждым цветом, применённым к сетке тела жилета.

**Настройка:**

1. Создайте настройку сцены для продукта-жилета с собранным GLB-файлом жилета в качестве базовой модели
2. Установите **Tone Mapping** в Commerce и **Auto Rotate** в включено
3. В разделе Node Mappings добавьте три записи — по одной на каждый цветовой вариант:

| Опция слота | Тип действия | Целевой узел | Данные действия |
|-------------|-------------|-------------|-------------|
| Черный | Цвет материала | JacketBody | `{"color": "#1a1a1a"}` |
| Синий | Цвет материала | JacketBody | `{"color": "#1b2a4a"}` |
| Бордовый | Цвет материала | JacketBody | `{"color": "#6b2737"}` |

Когда клиент выбирает синий цвет на странице продукта, просмотрщик мгновенно обновляет материал JacketBody до синего цвета.

## Советы

Сохраните всю разметку Markdown, пути к изображениям, блоки кода и технические термины.

- Называйте узлы GLB ясно, когда создаете свою 3D-модель — такие названия узлов, как "JacketBody" или "CollarMesh", намного проще в использовании, чем автоматически сгенерированные названия вроде "Mesh_023"
- Используйте **Commerce** для отображения тональности цвета большинства продуктов — он настроен для яркого и привлекательного представления продукта
- Отключите **Auto Rotate** для продуктов, где угол камеры по умолчанию уже показывает наиболее важные характеристики, чтобы избежать запутанности клиента при загрузке
- Проверьте кнопку AR на реальном мобильном устройстве перед продвижением — доступность AR зависит от устройства и браузера клиента (iOS Safari и Android Chrome с поддержкой WebXR являются наиболее надежными)
- Загрузите изображение **Thumbnail** для каждой конфигурации сцены — это предотвращает появление белого пустого квадрата при загрузке 3D-просмотрщика
- Если 3D-просмотрщик еще не готов, отключите его с помощью переключателя **Enabled**, чтобы клиенты видели стандартный конфигуратор изображений вместо этого
