---
title: 商品バリエーション
---

商品バリエーションを使用すると、1つの商品をサイズ、色、素材など複数のオプションで提供でき、それぞれに独自のSKU、価格、在庫レベルを設定できます。任意の**バリアブル商品**に移動し、**バリエーション**タブをクリックしてください。

![商品バリエーション](/static/core/admin/img/help/product-variants/product-variants.webp)

## バリエーションの理解

**バリアブル商品**は、複数のバリエーションをサポートする商品タイプです。例えば、Tシャツは以下のように展開できます：
- **カラー**: ブルー、レッド、グリーン
- **サイズ**: S、M、L、XL

各組み合わせ（例：「ブルー / Lサイズ」）は、独自の在庫と価格を持つ個別のバリエーションになります。

## バリアブル商品の設定

### ステップ1：商品タイプの設定

1. 商品編集フォームを開きます（または新しい商品を作成します）
2. **基本情報**タブで、**商品タイプ**を**バリアブル商品**に設定します
3. 商品を保存します

### ステップ2：属性の定義

属性は、バリエーションを区別するオプションです（例：サイズ、カラー）。

1. **バリエーション**タブに移動します
2. **商品属性**セクションで、**+ 属性を追加**をクリックして既存の属性を割り当てるか、**新規作成**をクリックして新しい属性を定義します
3. 各属性に対して、利用可能な値を指定します（例：S、M、L）

### ステップ3：バリエーションの作成

1. **商品バリエーション**セクションで、**+ 新しいバリエーションを追加**をクリックします
2. 各バリエーションを設定します：
   - **名前** — 説明的なラベル（例：「ブルー」、「Lサイズ / レッド」）
   - **SKU** — 一意の在庫管理コード
   - **価格** — バリエーション固有の価格（ベース商品と異なる設定が可能）
   - **在庫** — 現在の在庫レベル
3. 必要な各バリエーションについて繰り返します

## バリエーションの管理

### バリエーション詳細

各バリエーションカードには以下が表示されます：
- **名前**と**SKU** — 識別情報
- **価格** — 現在の販売価格
- **在庫レベル** — ステータスインジケーター付きの利用可能数量（在庫あり / 在庫少 / 在庫切れ）

バリエーションカードをクリックすると、展開してすべての詳細を編集できます。

### バリエーション固有の設定

各バリエーションには独自の設定が可能です：

| 設定 | 説明 |
|------|------|
| **価格** | ベース商品の価格を上書き |
| **比較価格** | 取り消し線付きのセール価格を表示 |
| **SKU** | 在庫管理用の一意の識別子 |
| **在庫レベル** | 独立した在庫追跡 |
| **重量** | 配送料計算用 |
| **画像** | バリエーション固有の商品画像 |

### バリエーションの編集

1. バリエーションカードの**編集アイコン**をクリックします
2. 必要なフィールドを変更します
3. **保存**をクリックして更新します

### バリエーションの削除

1. バリエーションカードの**削除アイコン**をクリックします
2. 削除を確認します

**注意：** バリエーションを削除すると、その在庫記録も削除されます。この操作は元に戻せません。

## 属性

### 属性とは？

属性は再利用可能なオプション定義です。「サイズ」という属性を「S、M、L、XL」の値で作成すると、任意のバリアブル商品に割り当てることができます。

### 属性の作成

1. バリエーションタブの商品属性セクションで**新規作成**をクリックします
2. 属性名を入力します（例：「カラー」）
3. 値を追加します（例：「レッド」、「ブルー」、「グリーン」）
4. 属性を保存します

### 属性の割り当て

属性は複数の商品に割り当てることができます。同じ「サイズ」属性をTシャツ、パンツ、シューズで使用できます。

## ストアフロントでの表示

ストアフロントでは、バリアブル商品は以下のように表示されます：
- 各属性のオプションセレクター（ドロップダウンまたはスウォッチ）
- バリエーション選択時の自動価格更新
- バリエーションごとの在庫状況
- バリエーション固有の画像

## ヒント

- 統一されたショッピング体験のために、商品間で一貫した属性名を使用しましょう。
- プロセスを効率化するために、バリエーションを作成する前にすべての属性を設定しましょう。
- お客様が注文内容を正確に確認できるよう、バリエーション固有の画像をアップロードしましょう。
- SKUは体系的に保ちましょう（例：「TSHIRT-BLUE-L」）。在庫管理が容易になります。
- バリエーションの比較価格を使用して、サイズ別やカラー別のセールを実施しましょう。
