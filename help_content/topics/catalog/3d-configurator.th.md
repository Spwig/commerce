---
title: ตัวกำหนดค่าผลิตภัณฑ์ 3D
---

ตัวกำหนดค่า 3D ช่วยให้ลูกค้าของคุณสามารถดูผลิตภัณฑ์ที่สามารถกำหนดค่าได้ผ่านตัวดู 3D แบบโต้ตอบโดยตรงบนหน้าผลิตภัณฑ์ เมื่อลูกค้าเลือกตัวเลือกต่าง ๆ เช่น สี วัสดุ หรือการเปลี่ยนรูปแบบส่วนประกอบ โมเดล 3D จะอัปเดตแบบเรียลไทม์เพื่อสะท้อนการเลือกของพวกเขา บนอุปกรณ์มือถือที่รองรับ ลูกค้าสามารถดูผลิตภัณฑ์ในสภาพแวดล้อมเสมือนจริง (AR) ได้โดยวางมันไว้ในพื้นที่ของตนเองก่อนที่จะซื้อ

ตัวกำหนดค่า 3D ทำงานกับผลิตภัณฑ์ที่สามารถกำหนดค่าได้ ผลิตภัณฑ์ที่สามารถกำหนดค่าได้แต่ละตัวสามารถมีการตั้งค่าฉาก 3D หนึ่งชุดที่เชื่อมโยงไฟล์โมเดล GLB กับตัวเลือกการกำหนดค่าของผลิตภัณฑ์

## ก่อนเริ่มต้น

เพื่อตั้งค่าฉาก 3D คุณจำเป็นต้องมี:

- ผลิตภัณฑ์ที่สามารถกำหนดค่าได้ **ที่สร้างไว้แล้ว** ในแคตตาล็อกของคุณ
- โมเดล 3D **พื้นฐาน** ที่อัปโหลดไปยังไลบรารีสื่อของคุณในรูปแบบไฟล์ GLB — นี่คือโมเดลที่ประกอบแล้วซึ่งจะแสดงขึ้นมาเป็นค่าเริ่มต้น
- ไฟล์ GLB เพิ่มเติมสำหรับการเปลี่ยนรูปทรงเรขาคณิต (เช่น รูปร่างคอที่แตกต่างกัน) และภาพข้อความสำหรับการเปลี่ยนแปลงวัสดุ (ถ้ามี)

หากคุณยังไม่ได้สร้างผลิตภัณฑ์ที่สามารถกำหนดค่าได้และตัวเลือกการกำหนดค่าของมัน ให้ทำสิ่งนั้นก่อนที่จะตั้งค่าฉาก 3D

## การสร้างการตั้งค่าฉาก

1. ไปที่ **Catalog > 3D Scene Configurations**
2. คลิก **+ Add 3D Scene Configuration**
3. เลือก **Product** ที่ฉากนี้เกี่ยวข้องกับ — เฉพาะผลิตภัณฑ์ที่สามารถกำหนดค่าได้เท่านั้นที่สามารถเลือกได้
4. เลือก **Base 3D Model** จากไลบรารีสื่อของคุณ — นี่คือไฟล์ GLB ที่โหลดเป็นค่าเริ่มต้น
5. ตั้งค่าตัวตั้งค่าตัวดู (ดูข้างล่าง)
6. บันทึกข้อมูล

หลังจากบันทึก ฟิลด์ **Node Tree** จะถูกเติมขึ้นอัตโนมัติ นี่คือกราฟฉากที่ถูกวิเคราะห์แล้วจากไฟล์ GLB ของคุณ — มันจะรายการโหนดที่มีชื่อทั้งหมดภายในโมเดล ซึ่งคุณจะอ้างอิงเมื่อเพิ่มการจับคู่โหนด

## ตัวตั้งค่าตัวดู

ตัวตั้งค่าเหล่านี้ควบคุมว่าตัวดู 3D จะปรากฏอย่างไรบนหน้าผลิตภัณฑ์ของคุณ

### กล้องและแสง

| Field | Description | Default |
|-------|-------------|---------|
| **Camera Orbit** | ตำแหน่งเริ่มต้นของกล้องในรูปแบบ `angle elevation distance` (เช่น `0deg 75deg 2m`) | `0deg 75deg 2m` |
| **Camera Target** | จุดที่กล้องมองไป วัดเป็นเมตรจากศูนย์กลางโมเดล (เช่น `0m 0m 0m`) | `0m 0m 0m` |
| **Environment Image** | ภาพ HDR จากไลบรารีสื่อของคุณ ใช้สำหรับการส่องแสงแบบอิงภาพ — ให้การสะท้อนและการเงาที่ดูเป็นธรรมชาติมากขึ้น | None |
| **Exposure** | ความสว่างโดยรวมของฉาก — ค่าต่ำกว่าจะมืดขึ้น ค่าสูงกว่าจะสว่างขึ้น | `1.0` |

### Shadows

| Field | Description | Default |
|-------|-------------|---------|
| **Shadow Intensity** | ความเข้มของเงาที่ตกใต้โมเดล — `0` หมายถึงไม่มีเงา `1` หมายถึงความเข้มสูงสุด | `0.5` |
| **Shadow Softness** | ความเบลอของขอบเงา — `0` หมายถึงคมชัด `1` หมายถึงความเบลอสูงสุด | `0.5` |

### Colour grading

| Field | Description |
|-------|-------------|
| **Tone Mapping** | อัลกอริทึมการปรับโทนสีที่ใช้กับฉาก **Commerce** สร้างสีที่สดใสและเหมาะกับสินค้า **Neutral** ให้สีที่แม่นยำ **ACES** ให้ลุคภาพยนตร์แบบ cinematic |
| **Bloom Strength** | เพิ่มเอฟเฟกต์แสงสว่างให้กับส่วนของโมเดลที่มีแสงของตัวเอง `0` ปิดเอฟเฟกต์ ค่าระหว่าง `1` ถึง `5` สร้างเอฟเฟกต์แสงที่นุ่มนวลถึงเด่นชัด |

### Behaviour and background

| Field | Description | Default |
|-------|-------------|---------|
| **Auto Rotate** | กำหนดว่าโมเดลจะหมุนช้าๆ เมื่อโหลดเพื่อดึงความสนใจของลูกค้า | On |
| **AR Enabled** | กำหนดว่าลูกค้าที่ใช้อุปกรณ์ที่รองรับจะเห็นปุ่ม **View in AR** | On |
| **Background** | สีพื้นหลังหรือค่าความชันของ CSS ของผู้ชม — ใส่ค่าสีในรูปแบบ hex (เช่น `#f5f5f5`) หรือค่าความชันของ CSS | `#ffffff` |

### Thumbnail

ฟิลด์ **Thumbnail** ถือภาพหน้าต่างของผู้ชม 3D ที่แสดงก่อนที่ผู้ชมจะโหลด

คุณสามารถถ่ายภาพหน้าสินค้าแบบเรียลไทม์และอัปโหลดไปยังไลบรารีสื่อของคุณ แล้วเชื่อมโยงมันที่นี่เพื่อให้ประสบการณ์การโหลดหน้าเว็บที่ราบรื่นยิ่งขึ้น

## การเปิดใช้งานและปิดใช้งาน 3D viewer

ปุ่ม **Enabled** ควบคุมว่า 3D viewer จะถูกแสดงบนหน้าสินค้าหรือไม่ เมื่อถูกปิดใช้งาน สินค้าจะกลับไปใช้ตัวกำหนดรูปภาพ 2D แบบมาตรฐาน นี่ช่วยให้คุณเตรียมการตั้งค่าสถานการณ์ได้ก่อนที่จะเปิดให้ลูกค้าเห็น

## การเชื่อมต่อตัวเลือกการกำหนดค่ากับการกระทำใน 3D

เมื่อตั้งค่าสถานการณ์พื้นฐานเสร็จแล้ว คุณสามารถเชื่อมต่อตัวเลือกการกำหนดค่าแต่ละช่องกับการเปลี่ยนแปลงทางภาพในโมเดล 3D ลิงก์เหล่านี้เรียกว่า **Node Mappings** และถูกเพิ่มในส่วน **Node Mappings** ที่ด้านล่างของฟอร์มการตั้งค่าสถานการณ์

### ฟิลด์ของ Node mapping

| Field | Description |
|-------|-------------|
| **Slot Option** | ตัวเลือกการกำหนดค่าที่กระตุ้นการเปลี่ยนแปลงนี้ (เช่น "Red Leather") |
| **Action Type** | การเปลี่ยนแปลงทางภาพที่เกิดขึ้น (ดูประเภทการกระทำด้านล่าง) |
| **Target Node** | ชื่อของโหนดในกราฟสถานการณ์ที่เปลี่ยนแปลง — เลือกจากชื่อที่ปรากฏใน **Node Tree** ของคุณ |
| **Action Data** | ข้อมูลเฉพาะของการกระทำ เช่น รหัสสีในรูปแบบ hex, URL ของภาพผิว, หรือ URL ของไฟล์ GLB |
| **Sort Order** | ควบคุมลำดับที่การกำหนดค่าหลายรายการสำหรับตัวเลือกเดียวกันถูกนำไปใช้ |

### ประเภทการกระทำ

| Action | What it does |
|--------|-------------|
| **Material Color** | เปลี่ยนสีของวัสดุในโหนดเป้าหมาย — ให้รหัสสีในรูปแบบ hex ใน **Action Data** |
| **Material Texture** | เปลี่ยนผิวที่ใช้กับวัสดุ — เชื่อมโยงไปยังทรัพยากรภาพผิวใน **Action Data** |
| **Geometry Swap** | แทนที่ส่วนหนึ่งของโมเดลด้วยไฟล์ GLB ที่แตกต่างกัน — มีประโยชน์สำหรับการเปลี่ยนแปลงโครงสร้าง เช่น รูปร่างของด้ามจับที่แตกต่างกัน |
| **Visibility** | แสดงหรือซ่อนโหนดในสถานการณ์ — ตั้งค่า `visible: true` หรือ `visible: false` ใน **Action Data** |

สามารถเพิ่มการกำหนดค่าหลายรายการสำหรับช่องเดียวกันได้

ตัวอย่างเช่น การเลือก "Blue Denim" อาจเปลี่ยนสีวัสดุ *และ* ซ่อนโหนดการตกแต่งหนังในเวลาเดียวกัน

## Geometry assets

หากการตั้งค่าของคุณมีการกระทำ **Geometry Swap** คุณจำเป็นต้องลงทะเบียนไฟล์ GLB ที่ใช้แทนเป็น Geometry Assets นี่จะถูกเพิ่มในส่วน **Geometry Assets** ของฟอร์มการตั้งค่าฉาก

| Field | Description |
|-------|-------------|
| **Label** | ชื่อที่อธิบายสำหรับ geometry asset นี้ เช่น "V-Neck Collar" |
| **GLB File** | ไฟล์ GLB ที่ใช้แทนจากไลบรารีสื่อของคุณ |
| **Target Node** | โหนดใดในโมเดลพื้นฐานที่ geometry นี้ใช้แทน |

หลังจากบันทึก Geometry Asset ชื่อโหนดจะถูกวิเคราะห์จาก GLB และถูกเก็บไว้ใน **Node Data** ทำให้สามารถใช้เป็นโหนดเป้าหมายในการกำหนดค่าของคุณได้

## Texture assets

ภาพข้อความที่ใช้ใน **Material Texture** mappings สามารถลงทะเบียนเป็น Texture Assets เพื่อการอ้างอิงที่ง่ายขึ้น นี่จะถูกเพิ่มในส่วน **Texture Assets**

| Field | Description |
|-------|-------------|
| **Label** | ชื่อที่อธิบาย เช่น "Red Leather" |
| **Texture Image** | ภาพข้อความจากไลบรารีสื่อของคุณ |
| **Texture Type** | ช่อง PBR ที่ข้อความนี้ใช้ — Base Color, Normal Map, Roughness Map, Metalness Map, Ambient Occlusion, หรือ Emissive Map |

## ตัวอย่าง: แจ็คเก็ตที่สามารถปรับแต่งได้พร้อมตัวเลือกสี

**สถานการณ์:** แจ็คเก็ตที่สามารถสั่งซื้อได้ในสีดำ, น้ำเงิน, หรือ burgundy โดยแต่ละสีจะถูกนำไปใช้กับ mesh ของตัวแจ็คเก็ต

**การตั้งค่า:**

1. สร้างการตั้งค่าฉากสำหรับผลิตภัณฑ์แจ็คเก็ตโดยใช้ไฟล์ GLB ที่ประกอบแล้วเป็นโมเดลพื้นฐาน
2. ตั้งค่า **Tone Mapping** เป็น Commerce และ **Auto Rotate** เป็น on
3. ใน Node Mappings เพิ่มรายการสามรายการ — หนึ่งรายการต่อตัวเลือกสี:

| ตัวเลือกส插槽 | ประเภทการกระทำ | โหนดเป้าหมาย | ข้อมูลการกระทำ |
|-------------|-------------|-------------|-------------|
| Black | สีวัสดุ | JacketBody | `{"color": "#1a1a1a"}` |
| Navy | สีวัสดุ | JacketBody | `{"color": "#1b2a4a"}` |
| Burgundy | สีวัสดุ | JacketBody | `{"color": "#6b2737"}` |

เมื่อลูกค้าเลือก Navy บนหน้าสินค้า ผู้ชมจะอัปเดตวัสดุ JacketBody เป็นสีน้ำเงินทันที

## เคล็ดลับ

- เมื่อสร้างโมเดล 3D ให้ตั้งชื่อโหนด GLB ให้ชัดเจน — ชื่อโหนดเช่น "JacketBody" หรือ "CollarMesh" ใช้งานง่ายกว่าชื่อที่สร้างโดยอัตโนมัติ เช่น "Mesh_023"
- ใช้การแมปโทน **Commerce** สำหรับสินค้าส่วนใหญ่ — มันถูกปรับแต่งสำหรับการนำเสนอสินค้าที่สดใสและน่าดึงดูด
- ปิดใช้งาน **Auto Rotate** สำหรับสินค้าที่มุมกล้องเริ่มต้นแสดงคุณสมบัติสำคัญอยู่แล้ว เพื่อหลีกเลี่ยงการสับสนลูกค้าเมื่อโหลด
- ทดสอบปุ่ม AR บนอุปกรณ์มือถือจริงก่อนโปรโมต — การใช้งาน AR ขึ้นอยู่กับอุปกรณ์และเบราว์เซอร์ของลูกค้า (iOS Safari และ Android Chrome ที่มีการสนับสนุน WebXR เป็นตัวที่น่าเชื่อถือที่สุด)
- อัปโหลดภาพ **Thumbnail** สำหรับการตั้งค่าสถานการณ์ทุกแบบ — เพื่อป้องกันไม่ให้กล่องสีขาวว่างเปล่ากระพริบขณะที่ผู้ชม 3D กำลังโหลด
- หากผู้ชม 3D ยังไม่พร้อม ให้ปิดใช้งานด้วยตัวเลือก **Enabled** เพื่อให้ลูกค้าเห็นตัวปรับแต่งภาพมาตรฐานแทน
