---
title: 訪問者分析
---

訪問者分析は、顧客が店舗を通じてどのように移動しているかの明確な概要を提供します。どのページが最も多くの訪問を引き寄せているのか、全体的なトラフィックの時間ごとの傾向、顧客が使用するデバイス、そして新規訪問者とリピーター訪問者の比較など、すべての情報を、外部の分析ツールを必要とすることなく確認できます。

## 分析画面の概要

GeoIPシステムがアクティブになると、店舗は自動的に訪問者の活動を追跡します。データは3つのビューに整理され、それぞれが異なるレベルの詳細情報を提供します。

### 日次のトラフィックサマリー

**顧客 > 日次のトラフィック統計**に移動して、各日の店舗全体のトラフィックを確認してください。各行は1つのカレンダーデイを表し、以下を表示します：

| 列 | それが教えてくれること |
|----|---------------------|
| **日付** | トラフィックが記録された日 |
| **総閲覧数** | ボットを含むすべてのページ閲覧数 |
| **ユニーク訪問者** | セッションごとに区別された訪問者 |
| **ボット閲覧数** | クローラーおよび自動ツールからの閲覧数 |
| **新規訪問者** | 以前の履歴がないセッション |
| **リピーター訪問者** | 以前に見た訪問者からのセッション |
| **デスクトップ閲覧数** | デスクトップブラウザからの閲覧数 |
| **モバイル閲覧数** | モバイルデバイスからの閲覧数 |
| **タブレット閲覧数** | タブレットデバイスからの閲覧数 |

リストの上部にある日付階層ナビゲーションを使用して、特定の月または年へ迅速にジャンプできます。合計値は自動的なバックグラウンドプロセスによって1日ごとに更新され、当日の数値は次の日の朝に表示されます。

### ページごとの統計

**顧客 > 日次のページ統計**に移動して、個々のページごとのトラフィックを確認してください。各行は1日の1つのURLパスを表示し、特定のページの時間ごとのパフォーマンスを比較できます。

| 列 | それが何を示すか |
|--------|-------------------|
| **日付** | これらの統計が適用される日 |
| **URLパス** | 正規化されたページパス（例: `/products/blue-widget`） |
| **ビュー数** | その日のそのページの総ビュー数 |
| **ユニーク訪問者** | そのページを閲覧した異なる訪問者 |
| **ボットビュー** | そのページからのボットによるビュー数 |
| **エントリ** | このページでセッションが開始された回数（このページが彼らの着地ページだった） |

**URLパス**の検索ボックスを使用して、特定のページの統計を検索してください。たとえば、`/products/`を検索してすべての商品ページのトラフィックを確認したり、特定の商品スラッグを検索して1つの商品に焦点を当てたりできます。

### 個別ページビューイベント

**顧客 > ページビュー**に移動して、追跡されたすべてのページナビゲーションの生のログを確認してください。これは読み取り専用のレコードであり、エントリを追加または編集することはできません。特定のセッションを調査したり、トラッキングが正しく記録されていることを確認するために使用してください。

各レコードは以下を示します：
- **URLパス** — 訪問されたページ
- **セッション** — 訪問者のセッションの短い識別子
- **ソース** — 訪問がヘッドレスフロントエンドから来たか、標準のストアフロントから来たか
- **ボットか** — 訪問者が自動トラフィックとして識別されたかどうか
- **エントリページか** — これは彼らのセッションの最初のページかどうか
- **タイムスタンプ** — 訪問の正確な時間

サイドバーのフィルターを使用して **ボットか**、**ソース**、および **エントリページか** でフィルターをかけ、上部の日付階層を使用して日付でナビゲートできます。

## トラフィックの傾向を読む

毎日のトラフィックサマリーは、傾向を把握するための最良のツールです。以下のようなパターンを確認してください：

- **プロモーションの実施やマーケティングメールの送信後のトラフィックの急増**
- **数週間または数ヶ月にわたる徐々な成長** — あなたのストアが有機的な露出を獲得している場合
- **週末と平日との比較** — 顧客が最もアクティブな時間帯を把握するため
- **モバイルとデスクトップの割合** — モバイル最適化されたデザインの変更を優先するかどうかを判断するため

「**新規訪問者**」と「**リピーター訪問者**」の列は、顧客のリテンション状況を教えてくれます。

健全な店舗では、両方の訪問者が混在しているのが一般的です。新規訪問者の割合が高い場合は、獲得力が強いことを示し、リピーターの割合が高い場合は、顧客のロイヤルティが高まっていることを示します。

## トップページの特定

ページごとの統計ビューは、デフォルトで表示回数の降順に並べ替えられており、特定の日のトラフィックを最も多く引き起こしているページをすぐに確認できます。以下を確認してください：

- **高エントリ、低表示ページ** — 検索や広告から訪問者が集まるが、注目を保つのが難しいページ
- **高表示回数でユニーク訪問者が多いページ** — 人気のある目的地ページで、常に新鮮さを保つ価値がある
- **表示回数が増加している商品ページ** — 検索の可視性が向上している可能性のある商品

### 例：商品のトラフィックを確認する

先週、ベストセラーリングの商品がどれだけのトラフィックを獲得したかを確認するには、以下を行います：

1. **顧客 > 日次ページ統計** に移動します
2. 日付階層を使用して、関連する週を選択します
3. 検索ボックスに、商品のURLスラッグを入力します（例: `/blue-widget`）
4. 表示されている日付にわたって **Views（表示回数）**、**Unique Visitors（ユニーク訪問者）**、**Entries（エントリ数）** を確認します

## 訪問者の位置情報データ

**顧客 > 訪問者位置** に移動して、訪問者の位置情報をセッションレベルで確認できます。各レコードは1つの訪問者セッションを表し、以下を含みます：

- 国と都市（GeoIPシステムによって自動的に解決）
- デバイスの種類（デスクトップ、モバイル、タブレット）
- 訪問者が選択した通貨と言語の設定
- UTMキャンペーンの属性（ソース、媒体、キャンペーン名）
- ボットと管理者トラフィックのフラグ

国、デバイスの種類、UTMソース、ボットまたは管理者スタッフであるかどうかで訪問者をフィルタリングできます。**Is Bot（ボットかどうか）** フィルタを false に設定して、本格的な顧客トラフィックに焦点を当て、**Is Admin Traffic（管理者トラフィックかどうか）** フィルタを使用して、自分のテストセッションを分析から除外できます。

## ヒント

すべてのマークダウンの書式、画像のパス、コードブロック、技術用語は保持してください。

- ボットの閲覧数は個別に追跡され、ユニークな訪問者数から自動的に除外されます — あなたのトラフィックの数値は、実際の顧客の活動を反映しています
- ページごとの統計にある **Entries** 列は、検索や広告からストアのフロントドアとして機能するページを示しています; これらのページの最適化は最も大きな影響を与えます
- **UTM Source** で訪問者の所在地をフィルタリングして、特定のマーケティングチャネル（例: メールニュースレターまたはGoogle広告）が実際にどのくらいのトラフィックを送っているかを測定できます
- 日次の統計は夜間に集計されます — その日のトラフィックを確認したい場合は、直接ページビューのログをご利用ください
- 日次のサマリーにおけるデバイス別の分析は、デザイン作業の優先順位を決めるのに役立ちます; 訪問者の半数以上がモバイル端末からアクセスしている場合、製品ページやチェックアウト画面が小さな画面でも見やすく表示されているかを確認してください
