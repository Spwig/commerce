---
title: 訪客分析
---

訪客分析讓您清楚了解顧客是如何在您的商店中流動的。您可以查看哪些頁面吸引了最多的訪問量，整體流量隨時間的變化趨勢，顧客使用的設備類型，以及新訪客與回訪顧客的比較——所有這些都無需任何外部分析工具。

## 分析螢幕概覽

當您在 GeoIP 系統啟用後，商店會自動追蹤訪客活動。數據分為三個視圖，每個視圖提供不同程度的細節。

### 日流量摘要

導航至 **Customers > Daily Traffic Stats**，查看您商店每天的整體流量。每一行代表一個日曆日，並顯示：

| 欄位 | 它告訴您什麼 |
|--------|-------------------|
| **日期** | 流量被記錄的日期 |
| **總瀏覽次數** | 所有頁面瀏覽次數，包括機器人 |
| **獨特訪客** | 不同的訪客（按會話計算） |
| **機器人瀏覽次數** | 來自爬蟲和自動化工具的瀏覽次數 |
| **新訪客** | 沒有先前歷史記錄的會話 |
| **回訪訪客** | 來自之前看到過的訪客的會話 |
| **桌面瀏覽次數** | 來自桌面瀏覽器的瀏覽次數 |
| **行動裝置瀏覽次數** | 來自行動設備的瀏覽次數 |
| **平板瀏覽次數** | 來自平板設備的瀏覽次數 |

使用列表頂部的日期層級導航，快速跳轉到特定月份或年份。總數每天通過自動化的背景流程更新，因此當天的數據將在第二天早上出現。

### 每頁統計數據

導航至 **Customers > Daily Page Stats**，查看按個別頁面分解的流量。每一行顯示一天中的一個 URL 路徑，因此您可以比較特定頁面隨時間的表現。

| 欄位 | 它告訴你什麼 |
|--------|-------------------|
| **日期** | 這些統計數據適用的日期 |
| **URL 路徑** | 正規化的頁面路徑（例如，`/products/blue-widget`） |
| **瀏覽次數** | 該頁面當天的總瀏覽次數 |
| **獨特訪客** | 查看該頁面的獨特訪客數 |
| **機器人瀏覽次數** | 來自機器人的該頁面瀏覽次數 |
| **入口頁面** | 在此頁面開始的會話數（這是他們的首頁） |

使用 **URL 路徑** 搜尋框來查找特定頁面的統計數據。例如，搜尋 `/products/` 來查看所有產品頁面的流量，或搜尋特定產品的 slug 來專注於某個項目。

### 個別頁面瀏覽事件

導航至 **客戶 > 頁面瀏覽**，以查看所有追蹤頁面導航的原始日誌。這是一個只讀記錄 —— 您無法新增或編輯條目。使用它來調查特定會話，或驗證追蹤是否正確記錄。

每個記錄顯示：
- **URL 路徑** — 被訪問的頁面
- **會話** — 訪客會話的簡短識別碼
- **來源** — 訪問來自無頭前端還是標準商店前端
- **是否為機器人** — 訪客是否被識別為自動流量
- **是否為入口頁面** — 這是否是他們會話中的第一頁
- **時間戳** — 訪問的精確時間

您可以使用側邊欄篩選器根據 **是否為機器人**、**來源** 和 **是否為入口頁面** 進行篩選，並使用頂部的日期層級按日期導航。

## 閱讀流量趨勢

每日流量摘要是您發現趨勢的最佳工具。尋找以下模式：

- **流量高峰** — 在運行促銷活動或發送市場郵件後
- **逐漸增長** — 隨著商店獲得有機曝光度，在數週和數月內的逐步增長
- **週末與工作日模式** — 了解客戶最活躍的時間
- **行動裝置與桌面分佈** — 决定是否優先進行行動優化的設計更改

「新訪客」和「回訪客」這兩欄資料一起告訴你，你在保留客戶方面的表現如何。

一個健康的商店通常會看到這兩者的混合情況——新訪客比例高表示獲客能力強，而回訪客比例更高則表示客戶忠誠度正在建立。

## 識別你的頂級頁面

按每頁的統計資料視圖，預設是按瀏覽次數由高到低排序，這會立即顯示出當天流量最高的頁面。請留意以下內容：

- **高入口、低瀏覽次數的頁面** — 這些頁面可能從搜尋或廣告吸引來訪客，但可能無法留住他們的注意力
- **瀏覽次數高且獨特訪客數多的頁面** — 這些受歡迎的目的地頁面值得保持新鮮
- **瀏覽次數上升的產品頁面** — 這些產品可能正在提高搜尋能見度

### 範例：查找產品的流量

要查看上週你最暢銷的產品獲得多少流量：

1. 點擊 **Customers > Daily Page Stats**
2. 使用日期層級選擇相關的週
3. 在搜尋框中輸入產品的 URL 標籤（例如 `/blue-widget`）
4. 查看所顯示日期內的 **Views**（瀏覽次數）、**Unique Visitors**（獨特訪客）和 **Entries**（入口次數）

## 訪客位置資料

點擊 **Customers > Visitor Locations**，可以看到訪客所在位置的會話級別視圖。每個記錄代表一個訪客會話，包含以下資訊：

- 國家和城市（由 GeoIP 系統自動解析）
- 裝置類型（桌面、行動、平板）
- 訪客選擇的貨幣和語言偏好
- UTM 廣告活動歸因（來源、媒介、活動名稱）
- 機器人和管理員流量標記

你可以根據國家、裝置類型、UTM 來源，以及訪客是否為機器人或管理員來篩選訪客。將 **Is Bot** 篩選設定為 false，可以專注於真實的客戶流量，並使用 **Is Admin Traffic** 篩選來排除自己的測試會話。

- 機器人流量會被獨立追蹤，並自動排除在獨特訪客數之外 —— 你的流量數據反映的是真實的客戶活動
- 每頁統計中的 **Entries** 欄位告訴你哪些頁面是商店的入口，來自搜尋和廣告；優化這些頁面會產生最大的影響
- 透過 **UTM Source** 篩選訪客位置，以衡量特定的行銷渠道（例如電子郵件簡訊或 Google 廣告）實際上帶來了多少流量
- 每日統計數據會在當天晚上進行彙總 —— 如果你需要查看當天的流量，請直接使用頁面瀏覽記錄
- 每日摘要中的設備分類幫助你優先處理設計工作；如果你超過一半的訪問來自行動裝置，請確保產品頁面和結帳流程在小螢幕上看起來很棒
