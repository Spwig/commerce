---
title: GeoIP 设置
---

GeoIP 可让您的商店根据访客的 IP 地址自动检测其所在位置。这为您的商店提供了基于地理位置的功能 —— 从默认显示正确的货币，到运行地理业务规则，再到在您的分析中查看国家层面的细分数据。

您的商店已预配置了 Spwig GeoIP 服务，因此地理位置检测功能开箱即用。您也可以连接其他提供商以提高准确性，使用自行下载的数据库，或依赖 CDN 的标头以实现零延迟查询。

## 提供商的工作原理

导航到 **客户 > GeoIP 提供商**，查看为您的商店配置的提供商。每个提供商都使用不同的方法处理 IP 到位置的查询。当访客到达时，您的商店会按照优先级顺序查询活动的提供商，并使用第一个成功的结果。

可以同时激活多个提供商 —— 优先级较低的数字会先尝试。如果最高优先级的提供商失败或未返回数据，则会自动尝试下一个提供商。

### 可用的提供商类型

| 提供商 | 描述 |
|----------|-------------|
| **Spwig GeoIP** | 通过 Spwig 服务的默认基于云的查找。无需设置。 |
| **MaxMind GeoLite2** | 来自 MaxMind 的离线数据库。高精度。需要一个免费的许可证密钥。 |
| **DB-IP Lite** | 来自 DB-IP 的离线数据库。从其网站下载。 |
| **IP2Location LITE** | 来自 IP2Location 的离线数据库。需要免费注册。 |
| **CDN Edge Headers** | 读取由您的 CDN（例如 Cloudflare）注入的位置标头。零延迟。 |
| **Browser Hints** | 使用浏览器提供的时区/语言作为软位置信号。 |
| **Custom Provider** | 从 Spwig 组件市场安装的提供商组件。 |

## 添加提供商

### 使用 Spwig GeoIP 服务（默认）

Spwig GeoIP 提供商会在新安装时自动添加。

检查它是否出现在列表中，并确保 **已激活** 已勾选。

无需进行其他配置。

### 添加 MaxMind GeoLite2 数据库

MaxMind 提供了一个免费的离线数据库，可以在不将查询发送到外部服务的情况下提供准确的结果。

1. 在 maxmind.com 注册一个免费账户并生成许可证密钥
2. 导航到 **客户 > GeoIP 提供商**，然后点击 **+ 添加 GeoIP 提供商**
3. 填写表单：
   - **名称**：`MaxMind GeoLite2`（或任何描述性名称）
   - **提供商类型**：MaxMind GeoLite2
   - **是否启用**：勾选
   - **优先级**：`1`（低于 Spwig 默认值以首先尝试，或高于默认值以用作后备）
   - **许可证密钥**：粘贴您的 MaxMind 许可证密钥
   - **数据库 URL**：从您的 MaxMind 账户仪表板下载的 URL
4. 点击 **保存**

保存后，在列表中选择提供商，并使用 **更新所选提供商数据库** 操作来验证数据库 URL 是否可访问。

### 添加 CDN 边缘头

如果您的商店位于注入地理位置头的 CDN 后面（例如 Cloudflare 的 `CF-IPCountry`），您可以使用这些头来实现即时、零延迟的国家检测。

1. 导航到 **客户 > GeoIP 提供商**，然后点击 **+ 添加 GeoIP 提供商**
2. 将 **提供商类型** 设置为 **CDN 边缘头**
3. 将 **优先级** 设置为 `0`（最高优先级，因为头是最快的来源）
4. 在 **配置** 字段中，指定您的 CDN 使用的头：
   ```json
   {
     "header_name": "CF-IPCountry"
   }
   ```
5. 点击 **保存**

### 测试提供商

添加提供商后，您可以验证其是否正常工作：

1. 在 GeoIP 提供商列表中，使用复选框选择提供商
2. 打开 **操作** 下拉菜单，选择 **测试所选提供商**
3. 点击 **开始**

Spwig 将为已知的 IP 地址（Google 的公共 DNS，`8.8.8.8`）发送测试查询，并向您显示结果。成功的测试将显示返回的国家和响应时间（以毫秒为单位）。

## 设置提供商优先级

当有多个提供商处于活动状态时，**优先级**字段控制首先尝试哪一个。

较小的数字表示更高的优先级。

例如，要首先使用CDN头（最快），然后回退到Spwig GeoIP：

| 提供商 | 优先级 |
|----------|----------|
| CDN边缘头 | 0 |
| Spwig GeoIP | 10 |

你可以在列表视图中直接编辑优先级——**优先级**列支持内联编辑。

## 监控提供商性能

每个提供商记录都会跟踪其自身的准确性统计信息：

- **总查询次数** — 尝试查询IP的总次数
- **成功查询次数** — 返回结果的查询次数
- **失败查询次数** — 返回无数据或错误的查询次数
- **平均响应时间（ms）** — 响应时间的平均值（以毫秒为单位）
- **准确性** — 成功查询的百分比

如果某个提供商的准确性较低或响应时间较高，考虑调整其优先级或将其禁用，改用性能更好的选项。

## 国家映射

导航到 **客户 > 国家映射**，以配置每个国家的默认设置，包括货币、语言、税和运费。每个国家条目控制以下内容：

- **默认货币** — 为来自该国家的访客预选的货币
- **默认语言** — 展示给来自该国家的访客的语言
- **税率** — 为该国家应用的默认税率百分比
- **是否为欧盟成员国** / **是否需要增值税** — 用于欧盟税务合规逻辑
- **运费区域** — 将国家链接到一个运费区域
- **支持货到付款** — 为该国家启用货到付款

你可以在列表中直接编辑 **是否启用**、**默认货币** 和 **默认语言** 字段，而无需打开每条记录。

## 小贴士

- Spwig GeoIP 提供商无需配置即可立即使用 — 仅在需要更高精度或离线操作时添加额外的提供商
- 如果您使用 Cloudflare，CDN Edge Headers 提供商是最佳选择：它不会增加延迟，也不会占用任何 API 配额
- 仅保留您实际需要的提供商 — 如果第一个提供商已成功，拥有多个活动提供商并不会提高精度
- 每周检查一次精度统计信息，并禁用成功率低于 80% 的任何提供商
- 国家映射用作默认值；客户始终可以在商店前端手动更改其货币和语言
