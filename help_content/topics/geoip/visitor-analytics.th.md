---
title: การวิเคราะห์ผู้เยี่ยมชม
---

การวิเคราะห์ผู้เยี่ยมชมให้คุณมีภาพรวมที่ชัดเจนว่าลูกค้ากำลังเคลื่อนไหวอย่างไรในร้านของคุณ คุณสามารถเห็นหน้าใดที่ดึงดูดการเยี่ยมชมมากที่สุด แนวโน้มการจราจรโดยรวมในช่วงเวลา หน้าจอที่ลูกค้าใช้ และเปรียบเทียบผู้เยี่ยมชมใหม่กับผู้เยี่ยมชมที่เคยมา — ทั้งหมดนี้โดยไม่จำเป็นต้องใช้เครื่องมือวิเคราะห์ภายนอกใด ๆ

## ภาพรวมหน้าจอการวิเคราะห์

ร้านของคุณจะติดตามกิจกรรมของผู้เยี่ยมชมโดยอัตโนมัติเมื่อระบบ GeoIP เปิดใช้งาน ข้อมูลถูกจัดระเบียบเป็น 3 แบบทัศนคติ แต่ละแบบให้ระดับรายละเอียดที่แตกต่างกัน

### สรุปการจราจรรายวัน

ไปที่ **Customers > Daily Traffic Stats** เพื่อดูการจราจรโดยรวมของร้านของคุณสำหรับแต่ละวัน แต่ละแถวแสดงถึงหนึ่งวันในปฏิทิน และแสดง:

| คอลัมน์ | สิ่งที่มันบอกคุณ |
|--------|-------------------|
| **วันที่** | วันที่การจราจรถูกบันทึก |
| **จำนวนการดูทั้งหมด** | การดูหน้าทั้งหมด รวมทั้งบอท |
| **ผู้เยี่ยมชมที่ไม่ซ้ำกัน** | ผู้เยี่ยมชมที่ไม่ซ้ำกัน (ตามเซสชัน) |
| **การดูจากบอท** | การดูจากเครื่องมือค้นหาและเครื่องมืออัตโนมัติ |
| **ผู้เยี่ยมชมใหม่** | เซสชันที่ไม่มีประวัติมาก่อน |
| **ผู้เยี่ยมชมที่เคยมา** | เซสชันจากผู้เยี่ยมชมที่เคยมา |
| **การดูจากเดสก์ท็อป** | การดูจากเบราว์เซอร์เดสก์ท็อป |
| **การดูจากมือถือ** | การดูจากอุปกรณ์มือถือ |
| **การดูจากแท็บเล็ต** | การดูจากอุปกรณ์แท็บเล็ต |

ใช้การนำทางลำดับชั้นของวันที่ด้านบนของรายการเพื่อกระโดดไปยังเดือนหรือปีเฉพาะได้อย่างรวดเร็ว ยอดรวมจะอัปเดตทุกวันผ่านกระบวนการพื้นหลังอัตโนมัติ ดังนั้นตัวเลขสำหรับวันนี้จะปรากฏในวันพรุ่งนี้

### สถิติรายหน้า

ไปที่ **Customers > Daily Page Stats** เพื่อดูการจราจรที่ถูกแยกตามหน้าแต่ละหน้า แต่ละแถวแสดงหนึ่งเส้นทาง URL ในหนึ่งวัน ดังนั้นคุณสามารถเปรียบเทียบประสิทธิภาพของหน้าเฉพาะในช่วงเวลา

| คอลัมน์ | สิ่งที่มันบอกคุณ |
|--------|-------------------|
| **วันที่** | วันที่สถิติเหล่านี้ใช้ได้ |
| **URL Path** | ค่าเส้นทางหน้าเว็บที่ได้รับการปรับมาตรฐาน (เช่น `/products/blue-widget`) |
| **จำนวนการเข้าชม** | จำนวนการเข้าชมรวมสำหรับหน้าเว็บนั้นในวันนั้น |
| **ผู้เยี่ยมชมที่ไม่ซ้ำกัน** | ผู้เยี่ยมชมที่ไม่ซ้ำกันที่เข้าชมหน้าเว็บนั้น |
| **จำนวนการเข้าชมจากบอท** | จำนวนการเข้าชมจากบอทในหน้าเว็บนั้น |
| **จำนวนการเริ่มเซสชัน** | จำนวนเซสชันที่เริ่มต้นที่หน้าเว็บนี้ (เป็นหน้าแรกที่ผู้เยี่ยมชมเข้ามา) |

ใช้กล่องค้นหา **URL Path** เพื่อค้นหาสถิติสำหรับหน้าเว็บเฉพาะ ตัวอย่างเช่น ค้นหา `/products/` เพื่อดูการจราจรของหน้าสินค้าทั้งหมด หรือค้นหา slug สินค้าเฉพาะเพื่อโฟกัสที่สินค้าหนึ่งเดียว

### เหตุการณ์การเข้าชมหน้าเว็บแต่ละหน้า

ไปที่ **Customers > Page Views** เพื่อดูบันทึกการเข้าชมหน้าเว็บที่ติดตามทุกครั้ง นี่เป็นบันทึกแบบอ่านเท่านั้น — คุณไม่สามารถเพิ่มหรือแก้ไขรายการได้ ใช้มันเพื่อตรวจสอบเซสชันเฉพาะหรือตรวจสอบว่าการติดตามบันทึกข้อมูลได้ถูกต้องหรือไม่

แต่ละรายการแสดง:
- **URL Path** — หน้าเว็บที่ถูกเข้าชม
- **Session** — ตัวระบุสั้นสำหรับเซสชันของผู้เยี่ยมชม
- **Source** — ว่าการเข้าชมมาจาก frontend แบบ headless หรือร้านค้ามาตรฐาน
- **Is Bot** — ว่าผู้เยี่ยมชมถูกระบุว่าเป็นการจราจรอัตโนมัติหรือไม่
- **Is Entry Page** — ว่าหน้านี้เป็นหน้าแรกในเซสชันของพวกเขาหรือไม่
- **Timestamp** — เวลาที่แม่นยำของการเข้าชม

คุณสามารถกรองตาม **Is Bot**, **Source**, และ **Is Entry Page** โดยใช้ตัวกรองในแถบด้านข้าง และนำทางตามวันที่โดยใช้ลำดับชั้นของวันที่ด้านบน

## การอ่านแนวโน้มการจราจร

สรุปการจราจรรายวันคือเครื่องมือที่ดีที่สุดของคุณในการสังเกตแนวโน้ม มองหาลักษณะเช่น:

- **การเพิ่มขึ้นของผู้เยี่ยมชม** หลังจากดำเนินการส่งเสริมการขายหรือส่งอีเมลการตลาด
- **การเติบโตอย่างค่อยเป็นค่อยไป** ในช่วงสัปดาห์และเดือนเมื่อร้านค้าของคุณได้รับการมองเห็นแบบอินทรีย์มากขึ้น
- **รูปแบบการเปรียบเทียบวันหยุดสุดสัปดาห์กับวันทำการ** เพื่อเข้าใจว่าลูกค้าของคุณมีกิจกรรมสูงสุดเมื่อใด
- **การแบ่งสัดส่วนระหว่างมือถือกับเดสก์ท็อป** เพื่อตัดสินใจว่าจะให้ความสำคัญกับการปรับปรุงการออกแบบสำหรับมือถือมากขึ้นหรือไม่

คอลัมน์ **New Visitors** และ **Returning Visitors** ร่วมกันบอกคุณว่าคุณรักษาลูกค้าได้ดีแค่ไหน

ร้านค้าที่แข็งแรงมักจะมีทั้งสองประเภท — จำนวนผู้เข้าชมใหม่มากขึ้นแสดงถึงการดึงดูดลูกค้าที่แข็งแกร่ง ในขณะที่จำนวนผู้เข้าชมที่กลับมาสูงขึ้นแสดงถึงการสร้างความภักดีของลูกค้า

## การระบุหน้าเว็บยอดนิยม

การดูสถิติรายหน้าเว็บ ซึ่งเรียงลำดับตามจำนวนการเข้าชมจากมากไปน้อย (ค่าเริ่มต้น) จะแสดงให้คุณเห็นทันทีว่าหน้าเว็บใดที่สร้างการเข้าชมมากที่สุดในแต่ละวัน ให้สังเกต:

- **หน้าเว็บที่มีการเข้าชมสูงแต่การดูต่ำ** — หน้าเว็บที่ดึงดูดผู้เข้าชมจากค้นหาหรือโฆษณา แต่อาจไม่สามารถดึงความสนใจไว้ได้นาน
- **หน้าเว็บที่มีการดูสูงและมีผู้เข้าชมที่ไม่ซ้ำกันจำนวนมาก** — หน้าเว็บยอดนิยมที่ควรรักษาความสดใหม่ไว้
- **หน้าเว็บสินค้าที่มีจำนวนการดูเพิ่มขึ้น** — สินค้าที่อาจเริ่มมีการปรากฏในผลการค้นหา

### ตัวอย่าง: การค้นหาการเข้าชมของสินค้า

เพื่อตรวจสอบว่าสินค้าขายดีของคุณได้รับการเข้าชมเท่าไรในสัปดาห์ที่แล้ว:

1. ไปที่ **Customers > Daily Page Stats**
2. ใช้ชุดวันที่เพื่อเลือกสัปดาห์ที่เกี่ยวข้อง
3. ใส่ slug ของ URL สินค้าในช่องค้นหา (เช่น `/blue-widget`)
4. ตรวจสอบ **Views**, **Unique Visitors**, และ **Entries** ตลอดช่วงวันที่แสดง

## ข้อมูลตำแหน่งของผู้เข้าชม

ไปที่ **Customers > Visitor Locations** เพื่อดูมุมมองระดับเซสชันว่าผู้เข้าชมของคุณอยู่ที่ไหน แต่ละบันทึกแสดงถึงเซสชันของผู้เข้าชมหนึ่งคน และรวมถึง:

- ประเทศและเมือง (ระบุโดยระบบ GeoIP อัตโนมัติ)
- ประเภทอุปกรณ์ (เดสก์ท็อป, มือถือ, แท็บเล็ต)
- ความชอบด้านสกุลเงินและภาษาที่ผู้เข้าชมเลือก
- การระบุแคมเปญ UTM (แหล่งที่มา, ประเภท, ชื่อแคมเปญ)
- ป้ายระบุการเข้าชมจากบอทและผู้ดูแลระบบ

คุณสามารถกรองผู้เข้าชมตามประเทศ, ประเภทอุปกรณ์, แหล่งที่มา UTM และว่าพวกเขาเป็นบอทหรือพนักงานผู้ดูแลระบบหรือไม่ ใช้ตัวกรอง **Is Bot** ตั้งค่าเป็น false เพื่อเน้นการเข้าชมจากลูกค้าจริง และตัวกรอง **Is Admin Traffic** เพื่อไม่รวมเซสชันการทดสอบของคุณเองในการวิเคราะห์

## เคล็ดลับ

รักษาการจัดรูปแบบ markdown ทั้งหมด ชื่อเส้นทางภาพ บล็อกโค้ด และคำศัพท์ทางเทคนิคไว้

- การเข้าชมจากบอทจะถูกติดตามแยกต่างหากและไม่รวมในจำนวนผู้เข้าชมที่แตกต่างกันโดยอัตโนมัติ — ตัวเลขการเข้าชมของคุณสะท้อนกิจกรรมของลูกค้าจริง
- คอลัมน์ **Entries** ในสถิติรายหน้าจะบอกคุณว่าหน้าใดที่ทำหน้าที่เป็นหน้าแรกของร้านค้าของคุณจากค้นหาและโฆษณา; การปรับปรุงหน้าเหล่านี้มีผลกระทบมากที่สุด
- กรองตำแหน่งของผู้เข้าชมโดย **UTM Source** เพื่อวัดว่าช่องทางการตลาดเฉพาะ (เช่น จดหมายข่าวอีเมลหรือโฆษณาของ Google) ส่งการเข้าชมมากน้อยแค่ไหน
- สถิติรายวันจะถูกรวบรวมในช่วงคืน — หากคุณต้องการตรวจสอบการเข้าชมในวันเดียวกัน ให้ใช้บันทึก Page Views โดยตรง
- การแบ่งประเภทอุปกรณ์ในสรุปรายวันช่วยให้คุณจัดลำดับความสำคัญในการออกแบบงาน; หากการเข้าชมมากกว่าครึ่งหนึ่งเป็นจากมือถือ ให้ตรวจสอบให้แน่ใจว่าหน้าสินค้าและหน้าชำระเงินของคุณดูดีบนหน้าจอขนาดเล็ก
