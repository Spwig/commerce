---
title: Правила бизнеса на основе расположения
---

Правила бизнеса на основе расположения позволяют автоматически выполнять действия, когда посетитель приходит из определенной страны, региона или типа устройства. Вы можете использовать правила для установки валюты для клиентов из определенного региона, перенаправления посетителей на локализованную страницу, отображения рекламного баннера или ограничения доступа к определенному контенту.

Правила оцениваются в порядке приоритета каждый раз, когда устанавливается сессия посетителя. Когда правило совпадает, его настроенные действия выполняются немедленно.

## Как работают правила бизнеса

Каждое правило состоит из двух частей:

- **Условия** — критерии, которые должны быть выполнены, чтобы правило сработало (например, "посетитель из Германии")
- **Действия** — что происходит, когда все условия совпадают (например, "установить валюту в EUR")

Условия и действия хранятся в виде объектов JSON в форме правила. Spwig оценивает все активные правила в порядке приоритета (сначала с наименьшим числом) и применяет те, которые совпадают.

## Перейти к правилам бизнеса

Перейдите в **Customers > Business Rules**, чтобы увидеть все настроенные правила. Список показывает имя каждого правила, статус, приоритет, количество раз, когда оно срабатывало, и когда оно последний раз сработало.

Щелкните любое правило, чтобы просмотреть или изменить его, или нажмите **+ Add Business Rule**, чтобы создать новое.

## Создание правила бизнеса

### Шаг 1: базовая информация

Заполните идентификационные данные правила:

- **Name** — понятное и описательное имя (например, `Set EUR for Eurozone`)
- **Description** — необязательные примечания, объясняющие цель правила
- **Is Active** — установите этот флажок, чтобы включить правило; снимите его, чтобы приостановить правило без его удаления
- **Priority** — правила с меньшими числами выполняются первыми; используйте `10`, `20`, `30`, чтобы оставить место для будущих правил

### Шаг 2: определение условий

В поле **Conditions** введите объект JSON, описывающий, когда должно сработать правило. Все условия в объекте должны быть истинными, чтобы правило совпадало.

#### Доступные ключи условий

| Условие | Формат | Пример |
|-----------|--------|---------|
| `country_in` | Массив кодов стран ISO | `["DE", "FR", "IT"]` |
| `country_not_in` | Массив кодов стран ISO | `["US", "CA"]` |
| `region_in` | Массив названий регионов | `["Bavaria", "Catalonia"]` |
| `region_not_in` | Массив названий регионов | `["Quebec"]` |
| `is_mobile` | Булево значение | `true` |
| `is_vpn` | Булево значение | `false` |

#### Примеры условий

Посетители из Германии, Франции или Италии:
```json
{
  "country_in": ["DE", "FR", "IT"]
}
```

Посетители из США, использующие мобильное устройство:
```json
{
  "country_in": ["US"],
  "is_mobile": true
}
```

Посетители из вне Европейского Союза:
```json
{
  "country_not_in": ["AT","BE","BG","CY","CZ","DE","DK","EE","ES","FI","FR","GR","HR","HU","IE","IT","LT","LU","LV","MT","NL","PL","PT","RO","SE","SI","SK"]
}
```

### Шаг 3: определение действий

В поле **Actions** введите объект JSON, описывающий, что должно произойти, когда правило сработает.

#### Доступные ключи действий

| Действие | Формат | Описание |
|--------|--------|-------------|
| `set_currency` | Строковый код валюты | Предварительно выберите валюту для посетителя |
| `set_language` | Строковый код языка | Установите язык отображения |
| `show_banner` | Булево значение | Активировать рекламный баннер |
| `redirect_to` | Строковый путь URL | Перенаправить посетителя на другой URL |

#### Примеры действий

Установить валюту в евро:
```json
{
  "set_currency": "EUR"
}
```

Перенаправить на локализованную страницу приветствия:
```json
{
  "redirect_to": "/de/"
}
```

Установить валюту и язык одновременно:
```json
{
  "set_currency": "GBP",
  "set_language": "en"
}
```

## Практические примеры

### Пример: правило валюты для Еврозоны

**Сценарий:** Автоматически показывать цены в евро посетителям из стран Еврозоны.

| Поле | Значение |
|-------|-------|
| Name | `Eurozone — Set EUR` |
| Priority | `10` |
| Is Active | Отмечено |
| Conditions | `{"country_in": ["AT","BE","DE","ES","FI","FR","GR","IE","IT","LU","NL","PT"]}` |
| Actions | `{"set_currency": "EUR"}` |

### Пример: правило цен для Великобритании

**Сценарий:** Показывать цены в фунтах стерлингов посетителям из Великобритании.

| Параметр | Значение |
|-------|-------|
| Название | `UK — Set GBP` |
| Приоритет | `20` |
| Активен | Отмечено |
| Условия | `"{\"country_in\": [\"GB\"]}"` |
| Действия | `"{\"set_currency\": \"GBP\"}"` |

### Пример: перенаправление на локализованный раздел магазина

**Сценарий:** Отправляйте посетителей из Австралии на отдельную австралийскую страницу.

| Параметр | Значение |
|-------|-------|
| Название | `Australia — Redirect` |
| Приоритет | `30` |
| Активен | Отмечено |
| Условия | `"{\"country_in\": [\"AU\"]}"` |
| Действия | `"{\"redirect_to\": \/au\/}"` |

## Тестирование правил

Вы можете проверить, соответствует ли правило ожидаемому профилю посетителя, не дожидаясь реального трафика:

1. В списке бизнес-правил выберите правило с помощью его флажка
2. Откройте выпадающий список **Действие** и выберите **Тестировать выбранные правила**
3. Нажмите **Go**

Spwig оценит правило по образцовому профилю посетителя из США и сообщит, совпадает ли оно, и какие действия были бы активированы.

## Отслеживание активности правил

Столбец **Triggered** в списке правил показывает, сколько раз каждое правило сработало. Нажмите на правило, чтобы увидеть метку времени **Last Triggered** в разделе Статистика.

Используйте действие **Reset statistics**, чтобы обнулить счетчики срабатываний, если вы хотите начать измерение с определенной даты после внесения изменений в правило.

## Советы

- Устанавливайте приоритеты с промежутками (10, 20, 30), а не последовательными числами (1, 2, 3), чтобы можно было вставить новые правила позже, не перенумеровывая все
- Правила срабатывают в порядке приоритета, и применяются все совпадающие правила — если два правила устанавливают валюту, действие правила с более низким приоритетом (большим номером) будет применено последним
- Используйте переключатель **Is Active**, чтобы временно приостановить правило во время акций, не удаляя конфигурацию
- Всегда тестируйте новое правило перед активацией в рабочей среде, чтобы убедиться, что условия корректны
- Детекция VPN (`"is_vpn": true`) доступна, если вы хотите применить разное обращение к посетителям, скрывающим свое местоположение, но имейте в виду, что некоторые законные клиенты используют VPN для приватности
