---
title: GeoIP 설정
---

GeoIP은 방문자의 IP 주소를 기반으로 각 방문자가 어디서 오는지 자동으로 감지할 수 있도록 합니다. 이 기능은 가게 전반에 걸쳐 위치 기반 기능을 가능하게 합니다 - 기본 통화 표시부터 지리적 비즈니스 규칙 실행, 분석에서 국가 수준의 분석까지.

가게는 기본적으로 Spwig GeoIP 서비스로 사전 구성되어 있으므로 지리적 감지가 즉시 작동합니다. 더 높은 정확도를 위해 추가 제공업체를 연결하거나, 직접 다운로드한 데이터베이스를 사용하거나, CDN에서 제공하는 헤더에 의존하여 지연 없는 조회를 수행할 수도 있습니다.

## 제공업체 작동 방식

**고객 > GeoIP 제공업체**로 이동하여 가게에 구성된 제공업체를 확인할 수 있습니다. 각 제공업체는 다른 방법을 사용하여 IP-위치 조회를 처리합니다. 방문자가 도착하면 가게는 우선순위 순서에 따라 활성화된 제공업체를 쿼리하고 첫 번째 성공적인 결과를 사용합니다.

여러 제공업체가 동시에 활성화될 수 있습니다 - 우선순위가 낮은 번호부터 시도됩니다. 최고 우선순위 제공업체가 실패하거나 데이터를 반환하지 않으면 다음 제공업체가 자동으로 시도됩니다.

### 사용 가능한 제공업체 유형

| 제공업체 | 설명 |
|----------|-------------|
| **Spwig GeoIP** | Spwig 서비스를 통한 기본 클라우드 기반 조회. 설정이 필요 없습니다. |
| **MaxMind GeoLite2** | MaxMind에서 제공하는 오프라인 데이터베이스. 높은 정확도. 무료 라이선스 키가 필요합니다. |
| **DB-IP Lite** | DB-IP에서 제공하는 오프라인 데이터베이스. 그들의 웹사이트에서 다운로드할 수 있습니다. |
| **IP2Location LITE** | IP2Location에서 제공하는 오프라인 데이터베이스. 무료 등록이 필요합니다. |
| **CDN Edge Headers** | CDN(예: Cloudflare)에서 주입된 위치 헤더를 읽습니다. 지연 시간이 없습니다. |
| **Browser Hints** | 브라우저에서 제공하는 시간대/언어를 사용하여 부드러운 위치 신호로 사용합니다. |
| **Custom Provider** | Spwig 구성 요소 시장에서 설치한 제공업체 구성 요소입니다. |

## 제공업체 추가

### Spwig GeoIP 서비스 사용(기본)

Spwig GeoIP 제공업체는 새 설치 시 자동으로 추가됩니다.

리스트에 표시되어 있고 **활성화됨**이 확인되어 있는지 확인합니다.

추가 구성은 필요 없습니다.

### MaxMind GeoLite2 데이터베이스 추가

MaxMind는 외부 서비스에 조회를 보내지 않고도 정확한 결과를 제공하는 무료 오프라인 데이터베이스를 제공합니다.

1. maxmind.com에서 무료 계정을 등록하고 라이선스 키를 생성합니다
2. **Customers > GeoIP Providers**로 이동하고 **+ Add GeoIP Provider**를 클릭합니다
3. 폼을 작성합니다:
   - **Name**: `MaxMind GeoLite2` (또는 설명이 가능한 이름)
   - **Provider Type**: MaxMind GeoLite2
   - **Is Active**: 체크
   - **Priority**: `1` (Spwig 기본값보다 낮게 설정하여 먼저 사용하거나, 대체로 사용하려면 더 높게 설정)
   - **License Key**: MaxMind 라이선스 키를 붙여넣습니다
   - **Database URL**: MaxMind 계정 대시보드에서 다운로드 URL을 입력합니다
4. **Save**를 클릭합니다

저장 후, 목록에서 제공업체를 선택하고 **Update selected provider databases** 액션을 사용하여 데이터베이스 URL이 접근 가능한지 확인합니다.

### CDN 엣지 헤더 추가

스토어가 Cloudflare의 `CF-IPCountry`와 같은 지리 정보 헤더를 삽입하는 CDN 뒤에 위치한 경우, 이러한 헤더를 사용하여 즉시, 지연 시간이 없는 국가 감지가 가능합니다.

1. **Customers > GeoIP Providers**로 이동하고 **+ Add GeoIP Provider**를 클릭합니다
2. **Provider Type**을 **CDN Edge Headers**로 설정합니다
3. **Priority**를 `0`으로 설정합니다 (가장 높은 우선순위, 헤더는 가장 빠른 소스이기 때문)
4. **Config** 필드에서 CDN이 사용하는 헤더를 지정합니다:
   ```json
   {
     "header_name": "CF-IPCountry"
   }
   ```
5. **Save**를 클릭합니다

### 제공업체 테스트

제공업체를 추가한 후, 제공업체가 올바르게 작동하는지 확인할 수 있습니다:

1. GeoIP Providers 목록에서 제공업체를 체크박스를 사용하여 선택합니다
2. **Action** 드롭다운을 열고 **Test selected providers**를 선택합니다
3. **Go**를 클릭합니다

Spwig는 알려진 IP 주소(예: Google의 공개 DNS, `8.8.8.8`)에 대한 테스트 조회를 보내고 결과를 표시합니다. 성공적인 테스트는 반환된 국가와 응답 시간(밀리초 단위)을 표시합니다.

## 제공업체 우선순위 설정

# 여러 제공업체가 활성화된 경우
**우선순위** 필드는 먼저 시도할 제공업체를 결정합니다.

작은 숫자일수록 우선순위가 높습니다.

예를 들어, CDN 헤더를 먼저 사용하고 (가장 빠름) 실패 시 Spwig GeoIP로 전환하려면 다음과 같이 설정할 수 있습니다:

| 제공업체 | 우선순위 |
|----------|----------|
| CDN Edge Headers | 0 |
| Spwig GeoIP | 10 |

리스트 뷰에서 직접 우선순위를 편집할 수 있습니다 — **우선순위** 열은 인라인으로 편집할 수 있습니다.

## 제공업체 성능 모니터링

각 제공업체 기록은 자체 정확도 통계를 추적합니다:

- **총 조회 수** — 시도된 IP 조회 수의 총합
- **성공한 조회 수** — 결과를 반환한 조회 수
- **실패한 조회 수** — 데이터가 없거나 오류가 발생한 조회 수
- **평균 응답 시간 (ms)** — 밀리초 단위의 평균 응답 시간
- **정확도** — 성공한 조회 수의 비율

제공업체가 낮은 정확도나 높은 응답 시간을 보일 경우, 우선순위를 조정하거나 더 높은 성능을 보이는 옵션으로 전환하는 것을 고려하세요.

## 국가 매핑

**고객 > 국가 매핑**으로 이동하여 통화, 언어, 세금, 배송에 대한 국가별 기본값을 구성할 수 있습니다. 각 국가 항목은 다음을 제어합니다:

- **기본 통화** — 해당 국가에서 방문자에게 미리 선택된 통화
- **기본 언어** — 해당 국가에서 방문자에게 표시되는 언어
- **세율** — 해당 국가에 적용되는 기본 세율
- **EU 회원** / **VAT 필요 여부** — EU 세금 규정 논리에 사용됨
- **배송 구역** — 국가를 배송 구역에 연결
- **COD 지원** — 해당 국가에 대한 현금 결제 옵션을 활성화

**활성화 여부**, **기본 통화**, **기본 언어** 필드는 각 기록을 열지 않고도 리스트에서 직접 편집할 수 있습니다.

## 팁
모든 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어를 유지하세요.

- Spwig GeoIP 제공업체는 구성 없이 즉시 작동합니다 — 더 높은 정확도나 오프라인 작업이 필요할 경우에만 추가 제공업체를 사용하세요
- Cloudflare을 사용하는 경우, CDN Edge Headers 제공업체가 최상의 선택입니다: 이는 지연 시간을 추가하지 않으며 어떤 API 할당량에도 영향을 주지 않습니다
- 실제로 필요한 제공업체만 활성화해 두세요 — 첫 번째 제공업체가 성공하면 많은 활성화된 제공업체가 정확도를 향상시키지 않습니다
- 매주 정확도 통계를 확인하고 성공률이 80% 미만인 제공업체를 비활성화하세요
- 국가 매핑은 기본값으로 사용되며, 고객은 언제든지 쇼핑몰에서 통화와 언어를 수동으로 변경할 수 있습니다
