---
title: 방문자 분석
---

방문자 분석은 고객이 가게를 어떻게 이동하는지에 대한 명확한 그림을 제공합니다. 어떤 페이지가 가장 많은 방문을 끌어들이는지, 전체 트래픽이 시간에 따라 어떻게 변화하는지, 고객이 사용하는 기기 종류, 그리고 새 방문자와 재방문 방문자 간의 비교 등을 볼 수 있습니다. 외부 분석 도구를 사용할 필요 없이 말이죠.

## 분석 화면 개요

GeoIP 시스템이 활성화되면 가게는 방문자 활동을 자동으로 추적합니다. 데이터는 세 가지 뷰로 구성되어 있으며, 각 뷰는 다른 수준의 세부 정보를 제공합니다.

### 일일 트래픽 요약

**고객 > 일일 트래픽 통계**로 이동하여 매일 가게의 전체 트래픽을 확인할 수 있습니다. 각 행은 달력의 하루를 나타내며 다음을 보여줍니다:

| 열 | 무엇을 알려주는가 |
|----|------------------|
| **날짜** | 트래픽이 기록된 날 |
| **총 조회수** | 봇 포함 모든 페이지 조회 |
| **유니크 방문자** | 세션 기준으로 구분된 방문자 |
| **봇 조회수** | 크롤러 및 자동 도구에서의 조회 |
| **신규 방문자** | 이전 기록이 없는 세션 |
| **재방문 방문자** | 이전에 본 방문자에서 온 세션 |
| **데스크탑 조회수** | 데스크탑 브라우저에서의 조회 |
| **모바일 조회수** | 모바일 기기에서의 조회 |
| **태블릿 조회수** | 태블릿 기기에서의 조회 |

목록 상단의 날짜 계층 탐색을 사용하여 특정 월이나 연도로 빠르게 이동할 수 있습니다. 총합은 자동화된 백그라운드 프로세스를 통해 매일 업데이트되며, 현재 날짜의 수치는 다음 날 아침에 나타납니다.

### 페이지별 통계

**고객 > 일일 페이지 통계**로 이동하여 개별 페이지별로 트래픽을 분석할 수 있습니다. 각 행은 하루 동안 하나의 URL 경로를 보여주므로, 시간에 따라 특정 페이지의 성과를 비교할 수 있습니다.

| 컬럼 | 무엇을 알려주는가 |
|--------|-------------------|
| **날짜** | 이 통계가 적용되는 날 |
| **URL 경로** | 정규화된 페이지 경로 (예: `/products/blue-widget`) |
| **조회수** | 해당 페이지의 당일 총 조회수 |
| **고유 방문자** | 해당 페이지를 본 고유 방문자 수 |
| **봇 조회수** | 해당 페이지에서의 봇 조회수 |
| **세션 시작 수** | 이 페이지에서 세션이 시작된 횟수 (이 페이지가 방문자의 시작 페이지였음) |

**URL 경로** 검색 상자를 사용하여 특정 페이지의 통계를 찾을 수 있습니다. 예를 들어, `/products/`를 검색하여 모든 제품 페이지 트래픽을 확인하거나, 특정 제품 슬러그를 검색하여 하나의 항목에 집중할 수 있습니다.

### 개별 페이지 조회 이벤트

**고객 > 페이지 조회**로 이동하여 추적된 모든 페이지 네비게이션의 원본 로그를 확인할 수 있습니다. 이는 읽기 전용 기록입니다 — 항목을 추가하거나 편집할 수 없습니다. 특정 세션을 조사하거나 추적 기록이 올바르게 기록되는지 확인하는 데 사용할 수 있습니다.

각 기록은 다음과 같은 정보를 보여줍니다:
- **URL 경로** — 방문한 페이지
- **세션** — 방문자의 세션을 위한 짧은 식별자
- **출처** — 방문이 헤드리스 프론트엔드에서 온 것인지 표준 스토어프론트에서 온 것인지
- **봇 여부** — 방문자가 자동 트래픽으로 식별되었는지 여부
- **입장 페이지 여부** — 이 페이지가 세션의 첫 번째 페이지였는지 여부
- **시간戳** — 방문의 정확한 시간

**봇 여부**, **출처**, **입장 페이지 여부**에 따라 사이드바 필터를 사용하여 필터링할 수 있으며, 상단의 날짜 계층을 사용하여 날짜로 이동할 수 있습니다.

## 트래픽 트렌드 확인

일일 트래픽 요약은 트렌드를 파악하는 데 가장 유용한 도구입니다. 다음과 같은 패턴을 확인하세요:

- **프로모션 실행 또는 마케팅 이메일 전송 후 트래픽 급증**
- **가장 최근의 유기적 가시성 증가에 따른 수주 및 달 단위 점진적 성장**
- **주말 대 평일 패턴** — 고객이 가장 활동적인 시간을 파악하기 위해
- **모바일 대 데스크탑 분할** — 모바일 최적화된 디자인 변경을 우선시할지 결정하기 위해

**신규 방문자** 및 **재방문자** 열은 고객 유지율을 얼마나 잘 유지하고 있는지를 알려줍니다.

건강한 가게는 일반적으로 두 가지 모두의 혼합을 보입니다. 신규 방문자가 많다는 것은 강력한 유입을 의미하고, 재방문 비율이 높다는 것은 고객 충성도가 형성되고 있음을 나타냅니다.

## 주요 페이지 식별

페이지별 통계 뷰는 기본값으로 뷰 수를 내림차순으로 정렬되어 있으며, 특정 날짜에 가장 많은 트래픽을 유발하는 페이지를 즉시 보여줍니다. 다음을 확인하세요:

- **높은 진입, 낮은 뷰 페이지** — 검색 또는 광고에서 방문자를 유치하지만 주의를 유지하지 못하는 페이지
- **높은 뷰 수와 많은 고유 방문자 수를 가진 페이지** — 인기 있는 목적지 페이지로 유지보수가 필요한 페이지
- **뷰 수가 증가하는 상품 페이지** — 검색 가시성이 증가하고 있는 상품

### 예시: 상품의 트래픽 확인

지난 주에 가장 잘 팔리는 상품이 받은 트래픽을 확인하려면:

1. **고객 > 일일 페이지 통계**로 이동합니다.
2. 날짜 계층 구조를 사용하여 관련 주를 선택합니다.
3. 검색 상자에 상품의 URL 슬러그를 입력합니다 (예: `/blue-widget`)
4. 표시된 날짜에 대해 **뷰**, **고유 방문자**, **진입**을 검토합니다.

## 방문자 위치 데이터

**고객 > 방문자 위치**로 이동하여 방문자의 위치에 대한 세션 수준의 뷰를 확인할 수 있습니다. 각 레코드는 하나의 방문자 세션을 나타내며 다음을 포함합니다:

- 국가 및 도시 (GeoIP 시스템에 의해 자동으로 결정됨)
- 기기 유형 (데스크탑, 모바일, 태블릿)
- 방문자가 선택한 통화 및 언어 선호도
- UTM 캠페인 속성 (소스, 매체, 캠페인 이름)
- 봇 및 관리자 트래픽 플래그

국가, 기기 유형, UTM 소스, 봇 또는 관리자 직원 여부에 따라 방문자를 필터링할 수 있습니다. **봇 여부** 필터를 false로 설정하여 진짜 고객 트래픽에 집중하고, **관리자 트래픽** 필터를 사용하여 자신의 테스트 세션을 분석에서 제외할 수 있습니다.

## 팁

모든 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어를 유지하세요.

- 봇 트래픽은 별도로 추적되며 고유 방문자 수에서 자동으로 제외됩니다 — 귀하의 트래픽 수치는 실제 고객 활동을 반영합니다
- 페이지별 통계의 **Entries** 열은 검색 및 광고를 통해 가게의 정문 역할을 하는 페이지를 보여줍니다; 이러한 페이지를 최적화하는 것이 가장 큰 영향을 미칩니다
- **UTM Source**로 방문자 위치를 필터링하여 특정 마케팅 채널(예: 이메일 뉴스레터 또는 구글 광고)이 실제로 얼마나 많은 트래픽을 보내는지 측정할 수 있습니다
- 일일 통계는 밤에 집계됩니다 — 동일 날짜의 트래픽을 확인하려면 페이지 뷰 로그를 직접 사용하세요
- 일일 요약의 기기 분석은 디자인 작업 우선순위를 결정하는 데 도움이 됩니다; 방문자의 절반 이상이 모바일이라면 제품 페이지와 결제 과정이 작은 화면에서도 잘 보이는지 확인하세요
