---
title: 访客分析
---

访客分析可让您清楚地了解客户如何在您的商店中流动。您可以查看哪些页面吸引了最多的访问量，整体流量如何随时间变化，客户使用的设备类型，以及新访客与回头客的对比——所有这些都不需要任何外部分析工具。

## 分析屏幕概述

一旦启用了GeoIP系统，您的商店会自动跟踪访客活动。数据被组织成三个视图，每个视图提供不同级别的详细信息。

### 每日流量摘要

导航到 **客户 > 每日流量统计**，查看您商店每天的整体流量。每一行代表一个日历日，并显示：

| 列 | 它告诉您什么 |
|--------|-------------------|
| **日期** | 流量记录的日期 |
| **总浏览量** | 所有页面浏览量，包括机器人 |
| **独立访客** | 不同的访客（按会话） |
| **机器人浏览量** | 来自爬虫和自动化工具的浏览量 |
| **新访客** | 没有历史记录的会话 |
| **回头客** | 来自之前见过的访客的会话 |
| **桌面浏览量** | 来自桌面浏览器的浏览量 |
| **移动设备浏览量** | 来自移动设备的浏览量 |
| **平板浏览量** | 来自平板设备的浏览量 |

使用列表顶部的日期层级导航，快速跳转到特定的月份或年份。总计每天通过自动化后台进程更新一次，因此当天的数据将在第二天早上出现。

### 每页统计

导航到 **客户 > 每日页面统计**，查看按单个页面细分的流量。每一行显示某一天的一个URL路径，因此您可以随时间比较特定页面的性能。

| 列 | 它告诉你的信息 |
|--------|-------------------|
| **日期** | 这些统计数据适用的日期 |
| **URL路径** | 规范化的页面路径（例如，`/products/blue-widget`） |
| **浏览量** | 该页面当天的总浏览量 |
| **独立访客** | 查看该页面的不同访客 |
| **机器人浏览量** | 该页面来自机器人的浏览量 |
| **入口** | 在此页面开始的会话数量（即用户首次访问的页面） |

使用 **URL路径** 搜索框查找特定页面的统计数据。例如，搜索 `/products/` 以查看所有产品页面的流量，或搜索特定产品slug以专注于某一个商品。

### 单个页面浏览事件

导航到 **客户 > 页面浏览**，以查看所有跟踪页面导航的原始日志。这是一个只读记录 —— 你不能添加或编辑条目。使用它来调查特定会话或验证跟踪是否正确记录。

每条记录显示：
- **URL路径** —— 被访问的页面
- **会话** —— 访客会话的简短标识符
- **来源** —— 访问是来自无头前端还是标准商店前端
- **是否为机器人** —— 访客是否被识别为自动化流量
- **是否为入口页面** —— 这是否是会话中的第一页
- **时间戳** —— 访问的准确时间

你可以通过 **是否为机器人**、**来源** 和 **是否为入口页面** 使用侧边栏筛选器进行筛选，并通过顶部的日期层级按日期导航。

## 分析流量趋势

每日流量摘要是你发现趋势的最佳工具。寻找以下模式：

- **流量高峰**：在进行促销活动或发送营销邮件后
- **逐渐增长**：随着商店获得有机曝光，几周或几个月内的增长
- **周末与工作日模式**：了解客户最活跃的时间
- **移动端与桌面端比例**：决定是否优先进行移动端优化设计的更改

**新访客** 和 **回头客** 两列数据一起告诉你你在客户留存方面的表现。

一个健康的店铺通常两者兼有 —— 较高的新访客比例表明获取客户的能力很强，而较高的回头客比例则表明客户忠诚度正在建立。

## 识别你的热门页面

按页面浏览量降序排列（默认）的每页统计视图，可以立即显示你当天流量最高的页面。注意以下内容：

- **高入口、低浏览量页面** —— 通过搜索或广告吸引访客但可能无法留住他们注意力的页面
- **浏览量高且独立访客多的页面** —— 受欢迎的目的地页面，值得保持内容更新
- **浏览量上升的产品页面** —— 可能正在获得更多搜索曝光的产品

### 示例：查找产品的流量

要查看上周你最畅销产品的流量情况：

1. 导航到 **Customers > Daily Page Stats**
2. 使用日期层级选择相关的一周
3. 在搜索框中输入产品的 URL slug（例如，`/blue-widget`）
4. 查看所显示日期内的 **Views**（浏览量）、**Unique Visitors**（独立访客）和 **Entries**（入口数）

## 访客位置数据

导航到 **Customers > Visitor Locations**，查看访客的会话级位置信息。每条记录代表一个访客会话，包括：

- 国家和城市（由 GeoIP 系统自动解析）
- 设备类型（台式机、手机、平板）
- 访客选择的货币和语言偏好
- UTM 活动归因（来源、媒介、活动名称）
- 机器人和管理员流量标志

你可以按国家、设备类型、UTM 来源以及访客是否为机器人或管理员进行筛选。将 **Is Bot** 筛选设置为 false，以专注于真实的客户流量，并使用 **Is Admin Traffic** 筛选来排除你自己的测试会话。

## 小贴士

保留所有 Markdown 格式、图片路径、代码块和专业术语。

- 机器人流量会单独跟踪，并自动排除在独立访客统计之外 —— 您的流量数据反映了真实的客户活动
- 每页统计中的 **Entries** 列会告诉您哪些页面充当了您商店的入口，来自搜索和广告；优化这些页面会产生最大的影响
- 通过 **UTM Source** 过滤访客位置，以衡量特定营销渠道（例如电子邮件通讯或 Google 广告）实际带来的流量
- 每日统计数据会在夜间汇总 —— 如果您需要查看当天的流量，请直接使用页面浏览日志
- 每日摘要中的设备分析有助于您优先安排设计工作；如果超过一半的访问量来自移动设备，请确保您的产品页面和结账页面在小屏幕上显示良好
