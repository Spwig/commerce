---
title: आधारित व्यवसाय के नियम स्थान
---

स्थान आधारित व्यवसाय के नियम आपको एक विशिष्ट देश, क्षेत्र या उपकरण प्रकार से आए एक यात्री के आगमन पर स्वचालित रूप से कार्रवाई करने की अनुमति देते हैं। आप नियमों का उपयोग एक विशिष्ट क्षेत्र से ग्राहकों के लिए एक मुद्रा को सेट करने, यात्रियों को एक स्थानीयकृत पृष्ठ पर रीडायरेक्ट करने, एक प्रोमोशनल बैनर दिखाने या कुछ विशिष्ट सामग्री के लिए पहुंच को सीमित करने के लिए कर सकते हैं।

हर बार एक यात्री सत्र स्थापित होते समय नियमों का प्राथमिकता क्रम में मूल्यांकन किया जाता है। जब एक नियम मिलता है, तो उसके विनिर्मित कार्रवाई तुरंत किए जाते हैं।

## व्यवसाय नियम कैसे काम करते हैं

प्रत्येक नियम दो भागों से बना होता है:

- **शर्तें** — जो शर्तों को पूरा करने के बाद नियम के ट्रिगर होना चाहिए (उदाहरण के लिए, "यात्री जर्मनी से है")
- **कार्रवाई** — जब सभी शर्तें मिलती हैं तो क्या होता है (उदाहरण के लिए, "मुद्रा को EUR में सेट करें")

शर्तें और कार्रवाई नियम फॉर्म में JSON ऑब्जेक्ट के रूप में संग्रहित की जाती हैं। Spwig सभी सक्रिय नियमों का मूल्यांकन प्राथमिकता क्रम में (पहले सबसे कम संख्या) करता है और कोई भी मिलता है उसका लागू करता है।

## व्यवसाय नियमों तक पहुंचना


एक व्यवसाय नियम बनाने के लिए **ग्राहक > व्यवसाय नियम** जाएं।

इस सूची में प्रत्येक नियम के नाम, स्थिति, प्राथमिकता, इसके द्वारा कितनी बार ट्रिगर किया गया है, और इसके अंतिम ट्रिगर के समय दिखाया गया है।

किसी भी नियम पर क्लिक करके इसे देखें या एडिट करें, या **+ व्यवसाय नियम जोड़ें** पर क्लिक करके एक नए नियम को बनाएं।

## व्यवसाय नियम बनाना

### चरण 1: आधारभूत जानकारी

नियम के पहचान विवरण भरें:

- **नाम** — एक स्पष्ट और विवरणात्मक नाम (उदाहरण के लिए, `Set EUR for Eurozone`)
- **विवरण** — नियम के उद्देश्य को समझाने के लिए वैकल्पिक टिप्पणियाँ
- **सक्रिय है** — इसे चेक करें ताकि नियम सक्रिय हो जाए; इसे अनचेक करें ताकि इसे बिना डिलीट किए बंद कर दिया जाए
- **प्राथमिकता** — छोटी संख्या पहले चलती है; भविष्य के नियमों के लिए जगह छोड़ने के लिए `10`, `20`, `30` का उपयोग करें

### चरण 2: शर्तों की परिभाषा

**शर्तों** के क्षेत्र में एक JSON ऑब्जेक्ट दर्ज करें जो बताता है कि नियम कब ट्रिगर होना चाहिए। ऑब्जेक्ट में सभी शर्तों के होना आवश्यक है ताकि नियम मिले।

चरण 3: कार्रवाइयाँ परिभाषित करें

**कार्रवाइयाँ** क्षेत्र में एक JSON ऑब्जेक्ट दर्ज करें जो नियम के ट्रिगर होने पर क्या होना चाहिए।

#### उपलब्ध कार्रवाई कुंजियाँ

| कार्रवाई | प्रारूप | विवरण |
|--------|--------|-------------|
| `set_currency` | मुद्रा कोड स्ट्रिंग | यात्री के लिए मुद्रा को पूर्व-निर्धारित करें |
| `set_language` | भाषा कोड स्ट्रिंग | प्रदर्शन भाषा को सेट करें |
| `show_banner` | बूलियन | एक प्रोमोशनल बैनर को ट्रिगर करें |
| `redirect_to` | URL पथ स्ट्रिंग | यात्री को एक अलग URL पर रीडायरेक्ट करें |

#### उदाहरण कार्रवाई

यूरो में मुद्रा सेट करें:
```json
{
  "set_currency": "EUR"
}
```

एक स्थानीयकृत लैंडिंग पृष्ठ पर रीडायरेक्ट करें:
```json
{
  "redirect_to": "/de/"
}
```

मुद्रा और भाषा दोनों को एक साथ सेट करें:
```json
{
  "set_currency": "GBP",
  "set_language": "en"
}
```

## व्यावहारिक उदाहरण

### उदाहरण: यूरोजोन मुद्रा नियम

**स्थिति:** यूरोजोन देशों से आए यात्रियों के लिए ऑटोमैटिक रूप से यूरो मूल्य दिखाएं।

| Field | Value |
|-------|-------|
| Name | `Eurozone — Set EUR` |
| Priority | `10` |
| Is Active | Checked |
| Conditions | `{"country_in": ["AT","BE","DE","ES","FI","FR","GR","IE","IT","LU","NL","PT"]}` |
| Actions | `{"set_currency": "EUR"}` |

### उदाहरण: UK की कीमत नियम

**स्थिति:** UK से आए यात्रियों के लिए GBP कीमत दिखाएं।

| Field | Value |
|-------|-------|
| Name | `UK — Set GBP` |
| Priority | `20` |
| Is Active | Checked |
| Conditions | `{"country_in": ["GB"]}` |
| Actions | `{"set_currency": "GBP"}` |

### उदाहरण: एक स्थानीय स्टोर अनुभाग में रीडायरेक्ट करें

**स्थिति:** ऑस्ट्रेलिया से आए यात्रियों को एक विशिष्ट ऑस्ट्रेलियाई पृष्ठ पर भेजें।

| Field | Value |
|-------|-------|
| Name | `Australia — Redirect` |
| Priority | `30` |
| Is Active | Checked |
| Conditions | `{"country_in": ["AU"]}` |
| Actions | `{"redirect_to": "/au/"}` |

## नियमों का परीक्षण

आप वास्तविक ट्रैफिक के बिना एक नियम के अपेक्षित यात्री प्रोफ़ाइल के साथ मिलता है यह जांच सकते हैं:

1.

व्यवसाय नियमों की सूची में, चेकबॉक्स के उपयोग से नियम का चयन करें
2.

**कार्रवाई** ड्रॉपडाउन खोलें और **चुनिंदा नियमों का परीक्षण** चुनें
3.

**जाओ** पर क्लिक करें

Spwig एक नमूना यूएस-आधारित यात्री प्रोफ़ाइल के खिलाफ नियम का मूल्यांकन करेगा और बताएगा कि यह मिलता है या नहीं और कौन-सी कार्रवाई ट्रिगर होती।

## नियम गतिविधि की निगरानी

नियमों की सूची में **ट्रिगर किया गया** स्तंभ बताता है कि प्रत्येक नियम कितनी बार ट्रिगर हुआ है। एक नियम पर क्लिक करें ताकि सांख्यिकी खंड में **अंतिम ट्रिगर** के समय देख सकें।

यदि आप एक नियम के बदलाव के बाद एक विशिष्ट तारीख से मापन शुरू करना चाहते हैं, तो **सांख्यिकी रीसेट** कार्रवाई का उपयोग करें ताकि ट्रिगर गिनती शून्य हो जाए।

## सुझाव

सभी मार्कडाउन फॉर्मेटिंग, इमेज पथ, कोड ब्लॉक और तकनीकी शब्दों को संरक्षित रखें।

- अगले नियमों के बिना अंतर (10, 20, 30) के साथ प्राथमिकताएं निर्धारित करें, बजाय क्रमागत संख्याओं (1, 2, 3) के, ताकि आप बाद में नए नियमों को जोड़ सकें बिना सभी के पुनर्नंबरिंग के।
- नियम प्राथमिकता के क्रम में लागू होते हैं और सभी मिलते-जुलते नियम लागू किए जाते हैं - यदि दो नियम दोनों मुद्रा को सेट करते हैं, तो निम्न-प्राथमिकता (उच्च-संख्या) नियम की कार्रवाई अंतिम रूप से लागू की जाएगी।
- **सक्रिय है** स्विच का उपयोग अभियान के दौरान एक नियम को अस्थायी रूप से रोकने के लिए करें बिना निर्माण के हटाए बिना।
- एक नए नियम को लाइव वातावरण में सक्रिय करने से पहले इसे हमेशा परीक्षण करें ताकि शर्तें सही हों।
- वीपीएन पहचान (`"is_vpn": true`) उपलब्ध है यदि आप अपने स्थान को छिपाने वाले यात्रियों के लिए अलग उपचार लागू करना चाहते हैं, लेकिन ध्यान रखें कि कुछ वैध ग्राहक गोपनीयता के लिए वीपीएन का उपयोग करते हैं।
