---
title: กฎธุรกิจตามตำแหน่ง
---

กฎธุรกิจตามตำแหน่งช่วยให้คุณสามารถดำเนินการอัตโนมัติเมื่อผู้เยี่ยมชมมาถึงจากประเทศ ภูมิภาค หรือประเภทอุปกรณ์ที่กำหนดไว้ คุณสามารถใช้กฎเพื่อตั้งค่าสกุลเงินสำหรับลูกค้าจากภูมิภาคเฉพาะ ทิ้งผู้เยี่ยมชมไปยังหน้าที่ปรับแต่งตามภูมิภาค แสดงแถบส่งเสริมการขาย หรือจำกัดการเข้าถึงเนื้อหาบางอย่าง

กฎจะถูกประเมินตามลำดับความสำคัญทุกครั้งที่เซสชันของผู้เยี่ยมชมถูกสร้างขึ้น เมื่อกฎตรงกับเงื่อนไข ผลการตั้งค่าที่กำหนดไว้จะถูกดำเนินการทันที

## วิธีการทำงานของกฎธุรกิจ

แต่ละกฎประกอบด้วยสองส่วน:

- **เงื่อนไข** — เงื่อนไขที่ต้องตรงกันเพื่อให้กฎทำงาน (เช่น "ผู้เยี่ยมชมมาจากเยอรมนี")
- **การดำเนินการ** — สิ่งที่เกิดขึ้นเมื่อเงื่อนไขทั้งหมดตรงกัน (เช่น "ตั้งค่าสกุลเงินเป็น EUR")

เงื่อนไขและการดำเนินการจะถูกเก็บเป็นวัตถุ JSON ในฟอร์มกฎ Spwig จะประเมินกฎที่เปิดใช้งานทั้งหมดตามลำดับความสำคัญ (เริ่มจากตัวเลขที่ต่ำที่สุด) และนำไปใช้กับกฎที่ตรงกัน

## การนำทางไปยังกฎธุรกิจ

ไปที่ **ลูกค้า > กฎธุรกิจ** เพื่อดูกฎที่คุณตั้งค่าไว้ทั้งหมด รายการจะแสดงชื่อ สถานะ ลำดับความสำคัญ จำนวนครั้งที่กฎถูกเปิดใช้งาน และเวลาที่กฎถูกเปิดใช้งานล่าสุด

คลิกกฎใดกฎหนึ่งเพื่อดูหรือแก้ไข หรือคลิก **+ เพิ่มกฎธุรกิจ** เพื่อสร้างกฎใหม่

## การสร้างกฎธุรกิจ

### ขั้นตอนที่ 1: ข้อมูลพื้นฐาน

กรอกข้อมูลระบุตัวตนของกฎ:

- **ชื่อ** — ชื่อที่ชัดเจนและอธิบายได้ (เช่น `ตั้ง EUR สำหรับยูโรโซน`)
- **คำอธิบาย** — หมายเหตุเพิ่มเติมที่อธิบายจุดประสงค์ของกฎ (ไม่บังคับ)
- **เปิดใช้งาน** — เลือกเพื่อเปิดใช้งานกฎ ไม่เลือกเพื่อหยุดใช้งานโดยไม่ลบมัน
- **ลำดับความสำคัญ** — ตัวเลขที่ต่ำกว่าจะทำงานก่อน ใช้ `10`, `20`, `30` เพื่อให้พื้นที่สำหรับกฎในอนาคต

### ขั้นตอนที่ 2: กำหนดเงื่อนไข

ในช่อง **เงื่อนไข** ใส่วัตถุ JSON ที่อธิบายว่าเมื่อใดกฎควรทำงาน ทุกเงื่อนไขในวัตถุต้องเป็นจริงเพื่อให้กฎตรงกัน

#### คีย์เงื่อนไขที่มีอยู่

| ข้อกำหนด | รูปแบบ | ตัวอย่าง |
|-----------|--------|---------|
| `country_in` | อาเรย์ของรหัสประเทศ ISO | `['DE', 'FR', 'IT']` |
| `country_not_in` | อาเรย์ของรหัสประเทศ ISO | `['US', 'CA']` |
| `region_in` | อาเรย์ของชื่อภูมิภาค | `['Bavaria', 'Catalonia']` |
| `region_not_in` | อาเรย์ของชื่อภูมิภาค | `['Quebec']` |
| `is_mobile` | บูลีน | `true` |
| `is_vpn` | บูลีน | `false` |

#### ตัวอย่างข้อกำหนด

ผู้เยี่ยมชมจากเยอรมนี ฝรั่งเศส หรืออิตาลี:
```json
{
  "country_in": ["DE", "FR", "IT"]
}
```

ผู้เยี่ยมชมจากสหรัฐอเมริกาที่ใช้อุปกรณ์มือถือ:
```json
{
  "country_in": ["US"],
  "is_mobile": true
}
```

ผู้เยี่ยมชมที่ไม่ใช่จากสหภาพยุโรป:
```json
{
  "country_not_in": ["AT","BE","BG","CY","CZ","DE","DK","EE","ES","FI","FR","GR","HR","HU","IE","IT","LT","LU","LV","MT","NL","PL","PT","RO","SE","SI","SK"]
}
```

### ขั้นตอนที่ 3: กำหนดการกระทำ

ในช่อง **Actions** ให้ใส่วัตถุ JSON ที่อธิบายว่าควรทำอะไรเมื่อกฎถูกกระตุ้น

#### คีย์การกระทำที่มีอยู่

| การกระทำ | รูปแบบ | คำอธิบาย |
|--------|--------|-------------|
| `set_currency` | รหัสสกุลเงินเป็นสตริง | เลือกสกุลเงินล่วงหน้าสำหรับผู้เยี่ยมชม |
| `set_language` | รหัสภาษาเป็นสตริง | ตั้งค่าภาษาที่แสดงผล |
| `show_banner` | บูลีน | เปิดใช้งานแบนเนอร์โปรโมชัน |
| `redirect_to` | สตริงของเส้นทาง URL | ทิ้งผู้เยี่ยมชมไปยัง URL ที่แตกต่างกัน |

#### ตัวอย่างการกระทำ

ตั้งค่าสกุลเงินเป็นยูโร:
```json
{
  "set_currency": "EUR"
}
```

ทิ้งไปยังหน้าแรกที่ปรับแต่งตามภูมิภาค:
```json
{
  "redirect_to": "/de/"
}
```

ตั้งค่าสกุลเงินและภาษาพร้อมกัน:
```json
{
  "set_currency": "GBP",
  "set_language": "en"
}
```

## ตัวอย่างการใช้งานจริง

### ตัวอย่าง: กฎสกุลเงินยูโรโซน

**สถานการณ์:** แสดงราคาในยูโรอัตโนมัติสำหรับผู้เยี่ยมชมจากประเทศในยูโรโซน


| Field | Value |
|-------|-------|
| Name | `Eurozone — Set EUR` |
| Priority | `10` |
| Is Active | Checked |
| Conditions | `{"country_in": ["AT","BE","DE","ES","FI","FR","GR","IE","IT","LU","NL","PT"]}` |
| Actions | `{"set_currency": "EUR"}` |

### Example: UK pricing rule

**Scenario:** Show GBP pricing to visitors from the United Kingdom.

| Field | Value |
|-------|-------|
| Name | `UK — Set GBP` |
| Priority | `20` |
| Is Active | Checked |
| Conditions | `{"country_in": ["GB"]}` |
| Actions | `{"set_currency": "GBP"}` |

### Example: redirect to a localized store section

**Scenario:** Send visitors from Australia to a dedicated Australian page.

| Field | Value |
|-------|-------|
| Name | `Australia — Redirect` |
| Priority | `30` |
| Is Active | Checked |
| Conditions | `{"country_in": ["AU"]}` |
| Actions | `{"redirect_to": "/au/"}` |

## Testing rules

You can verify that a rule matches the expected visitor profile without waiting for real traffic:

1. In the Business Rules list, select the rule using its checkbox
2. Open the **Action** dropdown and choose **Test selected rules**
3. Click **Go**

Spwig will evaluate the rule against a sample US-based visitor profile and report whether it matched and which actions would have been triggered.

## Monitoring rule activity

The **Triggered** column in the rules list shows how many times each rule has fired. Click a rule to see the **Last Triggered** timestamp in the Statistics section.

Use the **Reset statistics** action to zero out trigger counts if you want to start measuring from a specific date after making changes to a rule.

## Tips

Preserve all markdown formatting, image paths, code blocks, and technical terms.

- ตั้งลำดับความสำคัญโดยใช้ช่วง (10, 20, 30) แทนเลขลำดับที่ต่อเนื่องกัน (1, 2, 3) เพื่อที่คุณจะสามารถเพิ่มกฎใหม่ได้ในภายหลังโดยไม่ต้องเปลี่ยนลำดับทั้งหมด
- กฎจะถูกใช้งานตามลำดับความสำคัญ และกฎทั้งหมดที่ตรงกับเงื่อนไขจะถูกนำไปใช้ — หากมีกฎสองกฎที่ตั้งค่าสกุลเงินเดียวกัน กฎที่มีลำดับความสำคัญต่ำกว่า (เลขมากกว่า) จะถูกนำไปใช้เป็นครั้งสุดท้าย
- ใช้ปุ่ม **Is Active** เพื่อหยุดใช้งานกฎชั่วคราวในช่วงโปรโมชันโดยไม่ต้องลบการตั้งค่า
- ทดสอบกฎใหม่เสมอ ก่อนที่จะเปิดใช้งานในสภาพแวดล้อมจริง เพื่อให้มั่นใจว่าเงื่อนไขถูกต้อง
- การตรวจจับ VPN (`"is_vpn": true`) พร้อมใช้งาน หากคุณต้องการให้ใช้การจัดการที่แตกต่างกันกับผู้เยี่ยมชมที่ใช้ VPN เพื่อซ่อนตำแหน่งของตนเอง แต่โปรดจำไว้ว่าลูกค้าบางคนใช้ VPN เพื่อความเป็นส่วนตัว
