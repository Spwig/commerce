---
title: 基于位置的商业规则
---

基于位置的商业规则允许您在访客从特定国家、地区或设备类型访问时自动执行操作。您可以使用规则为来自特定地区的客户设置货币，将访客重定向到本地化页面，显示促销横幅，或限制对某些内容的访问。

每次建立访客会话时，规则都会按照优先级顺序进行评估。当规则匹配时，其配置的操作会立即执行。

## 商业规则的工作原理

每条规则由两部分组成：

- **条件** — 规则触发所需满足的标准（例如，"访客来自德国"）
- **操作** — 当所有条件匹配时发生的事情（例如，"将货币设置为EUR"）

条件和操作以JSON对象的形式存储在规则表单中。Spwig按照优先级顺序（优先级数字较小的先执行）评估所有活动规则，并应用匹配的规则。

## 导航到商业规则

导航到 **Customers > Business Rules** 以查看所有配置的规则。列表显示每条规则的名称、状态、优先级、触发次数以及最后一次触发的时间。

点击任意规则以查看或编辑它，或点击 **+ Add Business Rule** 创建新规则。

## 创建商业规则

### 第1步：基本信息

填写规则的标识信息：

- **名称** — 一个清晰且具有描述性的名称（例如，`Set EUR for Eurozone`）
- **描述** — 可选的说明，解释规则的目的
- **启用** — 勾选以启用规则；取消勾选以暂停规则而不删除它
- **优先级** — 数字较小的优先执行；使用 `10`、`20`、`30` 为未来规则留出空间

### 第2步：定义条件

在 **Conditions** 字段中，输入一个描述规则应何时触发的JSON对象。对象中的所有条件都必须为真，规则才能匹配。

#### 可用的条件键


| Condition | Format | Example |
|-----------|--------|---------|
| `country_in` | ISO国家代码数组 | `['DE', 'FR', 'IT']` |
| `country_not_in` | ISO国家代码数组 | `['US', 'CA']` |
| `region_in` | 区域名称数组 | `['Bavaria', 'Catalonia']` |
| `region_not_in` | 区域名称数组 | `['Quebec']` |
| `is_mobile` | 布尔值 | `true` |
| `is_vpn` | 布尔值 | `false` |

#### Example conditions

来自德国、法国或意大利的访客：
```json
{
  "country_in": ["DE", "FR", "IT"]
}
```

来自美国且使用移动设备的访客：
```json
{
  "country_in": ["US"],
  "is_mobile": true
}
```

来自欧洲联盟以外的访客：
```json
{
  "country_not_in": ["AT","BE","BG","CY","CZ","DE","DK","EE","ES","FI","FR","GR","HR","HU","IE","IT","LT","LU","LV","MT","NL","PL","PT","RO","SE","SI","SK"]
}
```

### Step 3: define actions

在**Actions**字段中，输入一个JSON对象，描述规则触发时应执行的操作。

#### Available action keys

| Action | Format | Description |
|--------|--------|-------------|
| `set_currency` | 货币代码字符串 | 为访客预选一种货币 |
| `set_language` | 语言代码字符串 | 设置显示语言 |
| `show_banner` | 布尔值 | 触发促销横幅 |
| `redirect_to` | URL路径字符串 | 将访客重定向到不同的URL |

#### Example actions

将货币设置为欧元：
```json
{
  "set_currency": "EUR"
}
```

重定向到本地化的落地页：
```json
{
  "redirect_to": "/de/"
}
```

同时设置货币和语言：
```json
{
  "set_currency": "GBP",
  "set_language": "en"
}
```

## Practical examples

### Example: Eurozone currency rule

**Scenario:** 自动向来自欧元区国家的访客显示欧元价格。

| 字段 | 值 |
|-------|-------|
| 名称 | `Eurozone — Set EUR` |
| 优先级 | `10` |
| 是否启用 | 已勾选 |
| 条件 | `"{\"country_in\": [\"AT\",\"BE\",\"DE\",\"ES\",\"FI\",\"FR\",\"GR\",\"IE\",\"IT\",\"LU\",\"NL\",\"PT\"]}"` |
| 操作 | `"{\"set_currency\": \"EUR\"}"` |

### 示例：英国定价规则

**场景：** 向来自英国的访客显示英镑定价。

| 字段 | 值 |
|-------|-------|
| 名称 | `UK — Set GBP` |
| 优先级 | `20` |
| 是否启用 | 已勾选 |
| 条件 | `"{\"country_in\": [\"GB\"]}"` |
| 操作 | `"{\"set_currency\": \"GBP\"}"` |

### 示例：重定向到本地化商店部分

**场景：** 将来自澳大利亚的访客重定向到专门的澳大利亚页面。

| 字段 | 值 |
|-------|-------|
| 名称 | `Australia — Redirect` |
| 优先级 | `30` |
| 是否启用 | 已勾选 |
| 条件 | `"{\"country_in\": [\"AU\"]}"` |
| 操作 | `"{\"redirect_to\": \/au\/}"` |

## 测试规则

你可以无需等待真实流量，即可验证规则是否匹配预期的访客档案：

1. 在业务规则列表中，使用复选框选择规则
2. 打开 **Action** 下拉菜单，选择 **Test selected rules**
3. 点击 **Go**

Spwig 将根据一个基于美国的示例访客档案评估规则，并报告是否匹配以及会触发哪些操作。

## 监控规则活动

规则列表中的 **Triggered** 列显示每个规则触发的次数。点击规则以在统计信息部分查看 **Last Triggered** 时间戳。

如果要在更改规则后从特定日期开始测量，可以使用 **Reset statistics** 操作将触发次数重置为零。

## 小贴士

保留所有 markdown 格式、图片路径、代码块和专业术语。

- 使用间隔优先级（10、20、30）而不是连续数字（1、2、3）来设置优先级，这样可以在不重新编号所有规则的情况下，后续插入新规则
- 规则按照优先级顺序触发，所有匹配的规则都会被应用 —— 如果两个规则都设置了货币，优先级较低（数字较大）的规则的操作将最后被应用
- 使用 **是否启用** 开关，在促销期间临时暂停规则，而无需删除配置
- 在将新规则激活到生产环境之前，始终先进行测试，确保条件设置正确
- 如果您希望对伪装位置的访客采取不同的处理方式，可以使用VPN检测（`"is_vpn": true`），但请注意，一些合法客户出于隐私原因会使用VPN
