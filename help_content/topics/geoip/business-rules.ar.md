---
title: قواعد العمل بناءً على الموقع
---

تتيح لك قواعد العمل بناءً على الموقع اتخاذ إجراءات تلقائية عندما يصل الزائر من دولة أو منطقة أو نوع جهاز معين. يمكنك استخدام القواعد لتحديد العملة لعملاء من منطقة معينة، أو توجيه الزوار إلى صفحة محلية، أو عرض لافتة ترويجية، أو تقييد الوصول إلى محتوى معين.

تتم تقييم القواعد حسب الأولوية كل مرة يتم إنشاء جلسة زائر. عندما تتطابق القاعدة، يتم تنفيذ الإجراءات المُكوَّنة فورًا.

## كيف تعمل قواعد العمل

تتكون كل قاعدة من جزأين:

- **الشروط** — المعايير التي يجب أن تُحقَّق لتفعيل القاعدة (مثلاً: "الزائر من ألمانيا")
- **الإجراءات** — ما يحدث عندما تتطابق جميع الشروط (مثلاً: "تحديد العملة إلى EUR")

تخزن الشروط والإجراءات ككائنات JSON في نموذج القاعدة. يقوم Spwig بتقييم جميع القواعد النشطة حسب أولويتها (أولوية الأرقام الأصغر أولاً) ويطبق أي منها تتطابق.

## التنقل إلى قواعد العمل

انتقل إلى **العملاء > قواعد العمل** لعرض جميع قواعدك المُكوَّنة. تعرض القائمة اسم كل قاعدة، الحالة، الأولوية، عدد مرات تفعيلها، ووقت آخر تفعيل.

انقر على أي قاعدة لعرضها أو تعديلها، أو انقر على **+ إضافة قاعدة عمل** لإنشاء واحدة جديدة.

## إنشاء قاعدة عمل

### الخطوة 1: المعلومات الأساسية

املأ تفاصيل هوية القاعدة:

- **الاسم** — اسم واضح ووصفية (مثلاً: `تحديد EUR للمنطقة الأوروبية`)
- **الوصف** — ملاحظات اختيارية تشرح هدف القاعدة
- **الحالة النشطة** — اختر هذا لتفعيل القاعدة؛ ألغِ اختياره لتعليقها دون حذفها
- **الأولوية** — الأرقام الأصغر تُنفذ أولاً؛ استخدم `10`، `20`، `30` لترك مساحة لقواعد مستقبلية

### الخطوة 2: تحديد الشروط

في حقل **الشروط**، أدخل كائن JSON يصف متى يجب أن تُفعَّل القاعدة. يجب أن تكون جميع الشروط في الكائن صحيحة لتفعيل القاعدة.

#### مفاتيح الشروط المتاحة

| الشرط | التنسيق | المثال |
|-----------|--------|---------|
| `country_in` | مصفوفة من رموز الدول ISO | `['DE', 'FR', 'IT']` |
| `country_not_in` | مصفوفة من رموز الدول ISO | `['US', 'CA']` |
| `region_in` | مصفوفة من أسماء المناطق | `['Bavaria', 'Catalonia']` |
| `region_not_in` | مصفوفة من أسماء المناطق | `['Quebec']` |
| `is_mobile` | قيمة منطقية | `true` |
| `is_vpn` | قيمة منطقية | `false` |

#### أمثلة على الشروط

الزوار من ألمانيا وفرنسا وإيطاليا:
```json
{
  "country_in": ["DE", "FR", "IT"]
}
```

الزوار من الولايات المتحدة الذين يستخدمون أجهزة محمولة:
```json
{
  "country_in": ["US"],
  "is_mobile": true
}
```

الزوار خارج الاتحاد الأوروبي:
```json
{
  "country_not_in": ["AT","BE","BG","CY","CZ","DE","DK","EE","ES","FI","FR","GR","HR","HU","IE","IT","LT","LU","LV","MT","NL","PL","PT","RO","SE","SI","SK"]
}
```

### الخطوة 3: تحديد الإجراءات

في حقل **الإجراءات**، أدخل كائن JSON يصف ما يجب أن يحدث عندما يُفعّل القاعدة.

#### مفاتيح الإجراءات المتاحة

| الإجراء | التنسيق | الوصف |
|--------|--------|-------------|
| `set_currency` | سلسلة رمز العملة | تحديد العملة المفضلة للزائر مسبقًا |
| `set_language` | سلسلة رمز اللغة | تحديد لغة العرض |
| `show_banner` | قيمة منطقية | تشغيل لافتة ترويجية |
| `redirect_to` | سلسلة مسار URL | توجيه الزائر إلى عنوان URL مختلف |

#### أمثلة على الإجراءات

تحديد العملة إلى اليورو:
```json
{
  "set_currency": "EUR"
}
```

توجيه الزائر إلى صفحة و landing محليّة:
```json
{
  "redirect_to": "/de/"
}
```

تحديد العملة واللغة معًا:
```json
{
  "set_currency": "GBP",
  "set_language": "en"
}
```

## أمثلة عملية

### مثال: قاعدة تحديد العملة لمنطقة اليورو

**الموقف:** عرض الأسعار باليورو تلقائيًا للزوار من دول منطقة اليورو.


| الحقل | القيمة |
|-------|-------|
| الاسم | `Eurozone — Set EUR` |
| الأولوية | `10` |
| النشاط | مُشَكَّل |
| الشروط | `{"country_in": ["AT","BE","DE","ES","FI","FR","GR","IE","IT","LU","NL","PT"]}` |
| الإجراءات | `{"set_currency": "EUR"}` |

### مثال: قاعدة سعر المملكة المتحدة

**الموقف:** إظهار الأسعار بالجنيه البريطاني للمستخدمين من المملكة المتحدة.

| الحقل | القيمة |
|-------|-------|
| الاسم | `UK — Set GBP` |
| الأولوية | `20` |
| النشاط | مُشَكَّل |
| الشروط | `{"country_in": ["GB"]}` |
| الإجراءات | `{"set_currency": "GBP"}` |

### مثال: توجيه الزوار إلى قسم متجر محلي

**الموقف:** إرسال الزوار من أستراليا إلى صفحة أسترالية مخصصة.

| الحقل | القيمة |
|-------|-------|
| الاسم | `Australia — Redirect` |
| الأولوية | `30` |
| النشاط | مُشَكَّل |
| الشروط | `{"country_in": ["AU"]}` |
| الإجراءات | `{"redirect_to": "/au/"}` |

## اختبار القواعد

يمكنك التحقق من أن القاعدة تتطابق مع الملف الشخصي المتوقع للزائر دون الانتظار لحركة المرور الفعلية:

1. في قائمة قواعد العمل، اختر القاعدة باستخدام مربع الاختيار الخاص بها
2. افتح قائمة **الإجراء** واختر **اختبار القواعد المحددة**
3. اضغط على **الذهاب**

سيقوم Spwig بتقييم القاعدة ضد ملف شخصي زائر أمريكي نموذجي ويبين ما إذا كانت تتطابق وما هي الإجراءات التي كانت ستُفعَّل.

## مراقبة نشاط القواعد

يُظهر عمود **المُفعَّلة** في قائمة القواعد عدد مرات تفعيل كل قاعدة. اضغط على القاعدة لرؤية **وقت التفعيل الأخير** في قسم الإحصائيات.

استخدم إجراء **إعادة تعيين الإحصائيات** لمسح عدادات التفعيل إذا أردت البدء في القياس من تاريخ معين بعد إجراء تغييرات على القاعدة.

## نصائح

احتفظ بجميع التنسيقات المرتبطة بالملف، مسارات الصور، كتل الشفرة، والكلمات الفنية.

- استخدم الأولويات مع فجوات (10، 20، 30) بدلًا من الأرقام التسلسلية (1، 2، 3) بحيث يمكنك إدراج قواعد جديدة لاحقًا دون إعادة ترقيم كل شيء
- تُطبق القواعد حسب ترتيب الأولوية، وجميع القواعد المطابقة تُطبق — إذا كانت قاعدتان يُحددان العملة، فإن الإجراء الخاص بالقاعدة ذات الأولوية الأقل (رقم أعلى) سيتم تطبيقه في النهاية
- استخدم مفتاح **Is Active** لتعليق قاعدة مؤقتًا أثناء الحملات الترويجية دون حذف التكوين
- قم دائمًا بإجراء اختبار لقاعدة جديدة قبل تفعيلها في بيئة حية للتأكد من صحة الشروط
- اكتشاف الـVPN (`"is_vpn": true`) متاح إذا كنت ترغب في تطبيق معاملة مختلفة على الزوار الذين يستخدمون الـVPN لإخفاء مواقعهم، ولكن تذكّر أن بعض العملاء الشرعيين يستخدمون الـVPN للخصوصية
