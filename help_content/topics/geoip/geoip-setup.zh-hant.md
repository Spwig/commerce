---
title: GeoIP 設置
---

GeoIP 讓您的商店能夠根據訪客的 IP 位址自動偵測其來源地。這項功能會在您的商店中啟用基於地理位置的功能 — 從預設顯示正確的貨幣，到執行地理商業規則，再到在您的分析中查看國家層級的資料分解。

您的商店預設已配置 Spwig GeoIP 服務，因此地理位置偵測功能開箱即用。您也可以連接其他提供商以提高準確度，使用自行下載的資料庫，或依賴 CDN 的標頭來實現零延遲查詢。

## 提供商的工作方式

導航至 **顧客 > GeoIP 提供商**，查看為您的商店配置的提供商。每個提供商都使用不同的方法來處理 IP 到位置的查詢。當訪客到達時，您的商店會按照優先順序查詢活躍的提供商，並使用第一個成功的結果。

可以同時啟用多個提供商 — 優先順序較低的數字會先嘗試。如果最高優先順序的提供商失敗或沒有返回資料，系統會自動嘗試下一個提供商。

### 可用的提供商類型

| 提供商 | 說明 |
|----------|-------------|
| **Spwig GeoIP** | 預設透過 Spwig 服務的雲端查詢。無需設置。 |
| **MaxMind GeoLite2** | 來自 MaxMind 的離線資料庫。準確度高。需要一個免費的授權金鑰。 |
| **DB-IP Lite** | 來自 DB-IP 的離線資料庫。從他們的網站下載。 |
| **IP2Location LITE** | 來自 IP2Location 的離線資料庫。需要免費註冊。 |
| **CDN Edge Headers** | 讀取由您的 CDN（例如 Cloudflare）注入的位置標頭。零延遲。 |
| **Browser Hints** | 使用瀏覽器提供的時區/語言作為軟性位置信號。 |
| **Custom Provider** | 從 Spwig 組件市場安裝的提供商組件。 |

## 添加提供商

### 使用 Spwig GeoIP 服務（預設）

Spwig GeoIP 提供商會在新安裝時自動添加。

確認它出現在列表中，並確認 **Is Active** 已勾選。

不需要進行額外的配置。

### 添加 MaxMind GeoLite2 數據庫

MaxMind 提供了一個免費的離線數據庫，可以在不將查詢發送到外部服務的情況下提供準確的結果。

1. 在 maxmind.com 注冊一個免費賬戶並生成許可證密鑰
2. 轉到 **Customers > GeoIP Providers**，然後點擊 **+ Add GeoIP Provider**
3. 填寫表單：
   - **Name**: `MaxMind GeoLite2`（或任何描述性名稱）
   - **Provider Type**: MaxMind GeoLite2
   - **Is Active**: 勾選
   - **Priority**: `1`（低於 Spwig 預設值以首先嘗試，或設為更高值作為備用）
   - **License Key**: 貼上你的 MaxMind 許可證密鑰
   - **Database URL**: 從你的 MaxMind 帳戶儀表板下載的 URL
4. 點擊 **Save**

保存後，選擇列表中的提供商，並使用 **Update selected provider databases** 操作來驗證數據庫 URL 是否可訪問。

### 添加 CDN 網關頭部

如果你的商店位於一個會注入地理定位頭部的 CDN 後面（例如 Cloudflare 的 `CF-IPCountry`），你可以使用這些頭部來實現即時、零延遲的國家檢測。

1. 轉到 **Customers > GeoIP Providers**，然後點擊 **+ Add GeoIP Provider**
2. 將 **Provider Type** 設置為 **CDN Edge Headers**
3. 將 **Priority** 設置為 `0`（最高優先級，因為頭部是最快的來源）
4. 在 **Config** 欄位中，指定你的 CDN 使用的頭部：
   ```json
   {
     "header_name": "CF-IPCountry"
   }
   ```
5. 點擊 **Save**

### 測試提供商

添加提供商後，你可以驗證它是否正常工作：

1. 在 GeoIP 提供商列表中，使用複選框選擇提供商
2. 打開 **Action** 下拉菜單，選擇 **Test selected providers**
3. 點擊 **Go**

Spwig 將為一個已知的 IP 地址（Google 的公共 DNS，`8.8.8.8`）發送測試查詢，並向你顯示結果。成功的測試會顯示返回的國家和響應時間（以毫秒為單位）。

## 設置提供商優先級

# 當多個提供商處於啟用狀態時，**優先級**欄位會控制先嘗試哪一個。

數字越小，優先級越高。

例如，要首先使用 CDN 頭部（最快），然後回退到 Spwig GeoIP：

| 提供商 | 優先級 |
|----------|----------|
| CDN Edge Headers | 0 |
| Spwig GeoIP | 10 |

您可以在列表視圖中直接編輯優先級 — **優先級**欄位可以內聯編輯。

## 監控提供商性能

每個提供商記錄都會追蹤自己的準確性統計數據：

- **總查詢數** — 尝試查詢 IP 的總數
- **成功查詢數** — 返回結果的查詢數
- **失敗查詢數** — 返回無數據或錯誤的查詢數
- **平均回應時間（毫秒）** — 回應時間的平均值（以毫秒為單位）
- **準確率** — 成功查詢的百分比

如果某個提供商顯示準確率低或回應時間高，請考慮調整其優先級或禁用它，改用表現更好的選項。

## 國家映射

導航至 **Customers > Country Mappings**，以配置每個國家的預設值，包括貨幣、語言、稅收和運輸。每個國家條目控制：

- **預設貨幣** — 為來自該國家的訪客預設選擇的貨幣
- **預設語言** — 顯示給來自該國家的訪客的語言
- **稅率** — 應用於該國家的預設稅率百分比
- **是否為 EU 成員** / **是否需要 VAT** — 用於 EU 稅務合規邏輯
- **運輸區域** — 將國家與運輸區域連結
- **支援現金付款** — 啟用該國家的現金付款功能

您可以在列表中直接編輯 **是否啟用**、**預設貨幣** 和 **預設語言** 欄位，而無需打開每個記錄。

## 小技巧

保留所有 markdown 格式、圖片路徑、程式碼塊和技術術語。

- Spwig GeoIP 提供商無需配置即可立即使用 — 僅在需要更高準確度或離線操作時添加額外提供商
- 如果您使用 Cloudflare，CDN Edge Headers 提供商是最佳選擇：它不會增加延遲，也不會佔用任何 API 配額
- 僅保留您實際需要的提供商 — 如果第一個提供商已經成功，擁有許多活動提供商不會提高準確度
- 每週檢查準確度統計數據，並停用成功率低於 80% 的任何提供商
- 國家映射用作預設值；客戶始終可以在商店前端手動更改貨幣和語言
