---
title: 위치 기반 비즈니스 규칙
---

위치 기반 비즈니스 규칙은 방문자가 특정 국가, 지역 또는 기기 유형에서 올 때 자동으로 조치를 취할 수 있도록 합니다. 규칙을 사용하여 특정 지역의 고객에게 통화를 설정하거나, 방문자를 지역화된 페이지로 리디렉션하거나, 프로모션 배너를 표시하거나, 특정 콘텐츠에 대한 액세스를 제한할 수 있습니다.

규칙은 방문자 세션이 시작될 때마다 우선순위 순서에 따라 평가됩니다. 규칙이 일치하면 구성된 액션들이 즉시 실행됩니다.

## 비즈니스 규칙 작동 방식

각 규칙은 두 부분으로 구성됩니다:

- **조건** — 규칙이 트리거되기 위해 충족해야 하는 기준 (예: "방문자가 독일에서 온 경우")
- **액션** — 모든 조건이 일치할 때 발생하는 일 (예: "통화를 EUR로 설정")

조건과 액션은 규칙 폼에서 JSON 객체로 저장됩니다. Spwig는 활성화된 모든 규칙을 우선순위 순서 (낮은 숫자부터)에 따라 평가하고 일치하는 규칙을 적용합니다.

## 비즈니스 규칙으로 이동

**고객 > 비즈니스 규칙**으로 이동하여 구성된 모든 규칙을 볼 수 있습니다. 목록에는 각 규칙의 이름, 상태, 우선순위, 트리거된 횟수, 마지막으로 실행된 시간이 표시됩니다.

어떤 규칙을 클릭하여 보기 또는 편집하거나, **+ 비즈니스 규칙 추가**를 클릭하여 새 규칙을 생성할 수 있습니다.

## 비즈니스 규칙 생성

### 단계 1: 기본 정보

규칙의 식별 정보를 입력합니다:

- **이름** — 명확하고 설명적인 이름 (예: `유로존에서 EUR 설정`)
- **설명** — 규칙의 목적을 설명하는 선택적 메모
- **활성화** — 이 체크박스를 선택하여 규칙을 활성화하고, 비활성화하여 규칙을 일시 중지할 수 있습니다.
- **우선순위** — 낮은 숫자부터 실행되며, `10`, `20`, `30`과 같은 숫자를 사용하여 향후 규칙을 추가할 여유 공간을 두세요.

### 단계 2: 조건 정의

**조건** 필드에 규칙이 실행되어야 하는 시기를 설명하는 JSON 객체를 입력합니다. 객체 내의 모든 조건이 참이어야 규칙이 일치합니다.

#### 사용 가능한 조건 키

| 조건 | 형식 | 예시 |
|-----------|--------|---------|
| `country_in` | ISO 국가 코드 배열 | `['DE', 'FR', 'IT']` |
| `country_not_in` | ISO 국가 코드 배열 | `['US', 'CA']` |
| `region_in` | 지역 이름 배열 | `['Bavaria', 'Catalonia']` |
| `region_not_in` | 지역 이름 배열 | `['Quebec']` |
| `is_mobile` | 불리언 | `true` |
| `is_vpn` | 불리언 | `false` |,

#### 예시 조건

독일, 프랑스, 이탈리아에서 온 방문자:
```json
{
  "country_in": ["DE", "FR", "IT"]
}
```

미국에서 온 모바일 기기 사용자:
```json
{
  "country_in": ["US"],
  "is_mobile": true
}
```

유럽 연합 외부에서 온 방문자:
```json
{
  "country_not_in": ["AT","BE","BG","CY","CZ","DE","DK","EE","ES","FI","FR","GR","HR","HU","IE","IT","LT","LU","LV","MT","NL","PL","PT","RO","SE","SI","SK"]
}
```

### 단계 3: 액션 정의

**액션** 필드에 규칙이 트리거될 때 발생해야 하는 작업을 설명하는 JSON 객체를 입력합니다.

#### 사용 가능한 액션 키

| 액션 | 형식 | 설명 |
|--------|--------|-------------|
| `set_currency` | 통화 코드 문자열 | 방문자에게 미리 선택된 통화를 설정합니다 |
| `set_language` | 언어 코드 문자열 | 표시 언어를 설정합니다 |
| `show_banner` | 불리언 | 프로모션 배너를 트리거합니다 |
| `redirect_to` | URL 경로 문자열 | 방문자를 다른 URL로 리디렉션합니다 |

#### 예시 액션

통화를 유로로 설정:
```json
{
  "set_currency": "EUR"
}
```

로컬라이즈된 랜딩 페이지로 리디렉션:
```json
{
  "redirect_to": "/de/"
}
```

통화와 언어를 함께 설정:
```json
{
  "set_currency": "GBP",
  "set_language": "en"
}
```

## 실용적인 예시

### 예시: 유로존 통화 규칙

**시나리오:** 유로존 국가에서 온 방문자에게 자동으로 유로 가격을 표시합니다.

| 필드 | 값 |
|-------|-------|
| 이름 | `Eurozone — Set EUR` |
| 우선순위 | `10` |
| 활성화 | 체크됨 |
| 조건 | `"{\"country_in\": [\"AT\",\"BE\",\"DE\",\"ES\",\"FI\",\"FR\",\"GR\",\"IE\",\"IT\",\"LU\",\"NL\",\"PT\"]}"` |
| 액션 | `"{\"set_currency\": \"EUR\"}"` |

### 예시: UK 가격 규칙

**시나리오:** 영국의 방문자에게 GBP 가격을 표시합니다.

| 필드 | 값 |
|-------|-------|
| 이름 | `UK — Set GBP` |
| 우선순위 | `20` |
| 활성화 | 체크됨 |
| 조건 | `"{\"country_in\": [\"GB\"]}"` |
| 액션 | `"{\"set_currency\": \"GBP\"}"` |

### 예시: 지역화된 상점 섹션으로 리디렉션

**시나리오:** 호주 방문자를 전용 호주 페이지로 이동시킵니다.

| 필드 | 값 |
|-------|-------|
| 이름 | `Australia — Redirect` |
| 우선순위 | `30` |
| 활성화 | 체크됨 |
| 조건 | `"{\"country_in\": [\"AU\"]}"` |
| 액션 | `"{\"redirect_to\": \/au\/}"` |

## 규칙 테스트

실제 트래픽을 기다리지 않고 규칙이 예상된 방문자 프로필과 일치하는지 확인할 수 있습니다:

1. 비즈니스 규칙 목록에서 규칙을 체크박스를 사용해 선택합니다
2. **액션** 드롭다운을 열고 **테스트 선택된 규칙**을 선택합니다
3. **Go**를 클릭합니다

Spwig은 미국 기반 샘플 방문자 프로필에 대해 규칙을 평가하고 일치 여부와 어떤 액션이 트리거되었는지 보고합니다.

## 규칙 활동 모니터링

규칙 목록의 **Triggered** 열은 각 규칙이 실행된 횟수를 보여줍니다. 규칙을 클릭하면 **Statistics** 섹션에서 **Last Triggered** 타임스탬프를 볼 수 있습니다.

규칙을 수정한 후 특정 날짜부터 측정을 시작하려면 **Reset statistics** 액션을 사용해 트리거 횟수를 0으로 리셋할 수 있습니다.

## 팁

모든 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어를 유지합니다.

- 간격이 있는 우선순위(10, 20, 30)를 사용하여 순차적인 숫자(1, 2, 3) 대신 사용하되, 나중에 새 규칙을 삽입할 때 모든 항목을 다시 번호 매기지 않도록 하세요
- 규칙은 우선순위 순서대로 실행되며, 모든 일치하는 규칙이 적용됩니다. 두 규칙이 모두 통화를 설정하는 경우, 우선순위가 낮은(숫자가 높은) 규칙의 동작이 마지막에 적용됩니다
- **활성화** 토글을 사용하여 프로모션 중 규칙을 일시적으로 중지할 수 있으며, 구성 자체를 삭제하지 않아도 됩니다
- 실제 환경에서 규칙을 활성화하기 전에 항상 새 규칙을 테스트하여 조건이 올바른지 확인하세요
- 방문자가 위치를 숨기기 위해 VPN을 사용하는 경우에 다른 처리를 적용하고 싶다면, VPN 감지(`"is_vpn": true`)가 사용 가능합니다. 다만 일부 정당한 고객이 프라이버시를 위해 VPN을 사용할 수 있음을 고려해야 합니다
