---
title: 位置ベースのビジネスルール
---

位置ベースのビジネスルールを使用すると、特定の国、地域、またはデバイスタイプから訪問者が来たときに自動的にアクションを実行できます。ルールを使用して、特定の地域の顧客に対して通貨を設定したり、訪問者をローカライズされたページにリダイレクトしたり、プロモーションバナーを表示したり、特定のコンテンツへのアクセスを制限したりできます。

ルールは、訪問者のセッションが確立されるたびに優先順位の順序で評価されます。ルールに一致した場合、設定されたアクションがすぐに実行されます。

## ビジネスルールの仕組み

各ルールは2つの部分で構成されています:

- **条件** — ルールがトリガーされるために満たすべき基準（例: 「訪問者はドイツから来ている」）
- **アクション** — すべての条件が一致したときに発生する内容（例: 「通貨をEURに設定する」）

条件とアクションは、ルールフォーム内でJSONオブジェクトとして保存されます。Spwigは、すべてのアクティブなルールを優先順位の順序（最も低い数字から）で評価し、一致するものを適用します。

## ビジネスルールへの移動

**顧客 > ビジネスルール**に移動して、設定されたすべてのルールを確認できます。リストには、各ルールの名前、ステータス、優先順位、トリガーされた回数、最後に発火した時間が表示されます。

任意のルールをクリックして表示または編集できます。または、**+ ビジネスルールの追加**をクリックして新しいルールを作成できます。

## ビジネスルールの作成

### ステップ1: 基本情報

ルールの識別情報を入力してください:

- **名前** — 明確で説明的な名前（例: `Set EUR for Eurozone`）
- **説明** — ルールの目的を説明するオプションのノート
- **有効** — このチェックボックスをオンにしてルールを有効にします。オフにすると、削除せずに一時停止できます。
- **優先順位** — 数値が小さいほど先に実行されます。将来的なルールのための余地を残すために、`10`、`20`、`30`を使用してください。

### ステップ2: 条件の定義

**条件**フィールドに、ルールが発火するタイミングを記述したJSONオブジェクトを入力してください。オブジェクト内のすべての条件が真である場合に限り、ルールが一致します。

#### 利用可能な条件キー


| Condition | Format | Example |
|-----------|--------|---------|
| `country_in` | ISO国コードの配列 | `['DE', 'FR', 'IT']` |
| `country_not_in` | ISO国コードの配列 | `['US', 'CA']` |
| `region_in` | 地域名の配列 | `['Bavaria', 'Catalonia']` |
| `region_not_in` | 地域名の配列 | `['Quebec']` |
| `is_mobile` | ブール値 | `true` |
| `is_vpn` | ブール値 | `false` |

#### Example conditions

ドイツ、フランス、イタリアからの訪問者:
```json
{
  "country_in": ["DE", "FR", "IT"]
}
```

アメリカ合衆国からの訪問者で、モバイル端末を使用している場合:
```json
{
  "country_in": ["US"],
  "is_mobile": true
}
```

ヨーロッパ連合（EU）以外からの訪問者:
```json
{
  "country_not_in": ["AT","BE","BG","CY","CZ","DE","DK","EE","ES","FI","FR","GR","HR","HU","IE","IT","LT","LU","LV","MT","NL","PL","PT","RO","SE","SI","SK"]
}
```

### Step 3: define actions

**Actions**フィールドに、ルールがトリガーされたときに何が起こるべきかを記述するJSONオブジェクトを入力してください。

#### Available action keys

| Action | Format | Description |
|--------|--------|-------------|
| `set_currency` | 通貨コード文字列 | 訪問者に事前に通貨を設定する |
| `set_language` | 言語コード文字列 | 表示言語を設定する |
| `show_banner` | ブール値 | プロモーションバナーをトリガーする |
| `redirect_to` | URLパス文字列 | 訪問者を別のURLにリダイレクトする |

#### Example actions

通貨をユーロに設定:
```json
{
  "set_currency": "EUR"
}
```

ローカライズされたランディングページにリダイレクト:
```json
{
  "redirect_to": "/de/"
}
```

通貨と言語を同時に設定:
```json
{
  "set_currency": "GBP",
  "set_language": "en"
}
```

## Practical examples

### Example: Eurozone currency rule

**Scenario:** Eurozoneの国からの訪問者に自動的にユーロ価格を表示する。


| フィールド | 値 |
|-------|-------|
| 名前 | `Eurozone — Set EUR` |
| 優先度 | `10` |
| 有効 | チェック済み |
| 条件 | `"{\"country_in\": [\"AT\",\"BE\",\"DE\",\"ES\",\"FI\",\"FR\",\"GR\",\"IE\",\"IT\",\"LU\",\"NL\",\"PT\"]}"` |
| アクション | `"{\"set_currency\": \"EUR\"}"` |

### 例: UK価格設定ルール

**シナリオ:** イギリスからの訪問者にGBP価格を表示します。

| フィールド | 値 |
|-------|-------|
| 名前 | `UK — Set GBP` |
| 優先度 | `20` |
| 有効 | チェック済み |
| 条件 | `"{\"country_in\": [\"GB\"]}"` |
| アクション | `"{\"set_currency\": \"GBP\"}"` |

### 例: ローカライズされたストアセクションへのリダイレクト

**シナリオ:** オーストラリアからの訪問者を専用のオーストラリアページにリダイレクトします。

| フィールド | 値 |
|-------|-------|
| 名前 | `Australia — Redirect` |
| 優先度 | `30` |
| 有効 | チェック済み |
| 条件 | `"{\"country_in\": [\"AU\"]}"` |
| アクション | `"{\"redirect_to\": \/au\/}"` |

## ルールのテスト

実際のトラフィックを待たずに、ルールが予期した訪問者プロファイルと一致するかを確認できます：

1. ビジネスルール一覧で、ルールのチェックボックスを選択します
2. **アクション**ドロップダウンを開き、**選択したルールをテスト**を選択します
3. **Go**をクリックします

Spwigは、サンプルの米国ベースの訪問者プロファイルに対してルールを評価し、一致したかどうかを報告し、どのアクションがトリガーされたかを示します。

## ルールアクティビティの監視

ルール一覧の**Triggered**列は、各ルールが何回発火したかを示しています。ルールをクリックして、統計セクションの**Last Triggered**タイムスタンプを表示します。

**Reset statistics**アクションを使用して、ルールを変更した後、特定の日付から測定を開始したい場合に、トリガー数をゼロにリセットできます。

## ヒント

すべてのマークダウンフォーマット、画像パス、コードブロック、および技術用語を保持してください。

- 10, 20, 30 などのギャップのある優先度を順番の数字 (1, 2, 3) ではなく設定して、後で新しいルールを挿入するときにすべての番号を再編成する必要がないようにする
- ルールは優先度順に実行され、すべての一致するルールが適用されます — 2つのルールがどちらも通貨を設定している場合、優先度が低い（番号が大きい）ルールのアクションが最後に適用されます
- **有効**トグルを使用して、プロモーション中に設定を削除せずにルールを一時的に停止できます
- 本番環境でルールを有効にする前に、常に新しいルールをテストして、条件が正しいことを確認してください
- ビプライバシーのためにVPNを使用する正当な顧客もいることを考慮して、訪問者の位置情報を隠すために異なる処理を適用したい場合に限り、VPN検出 (`"is_vpn": true`) が利用可能です
