---
title: 직원 역할 및 권한
---

직원 역할은 각 팀원이 관리자 패널과 POS 단말기에서 볼 수 있고 할 수 있는 것을 정확히 제어할 수 있습니다. 특정한 권한을 가진 역할을 정의한 후 직원들에게 할당합니다. 사용자는 여러 역할을 가질 수 있으며, 그들의 실제 권한은 할당된 모든 역할의 조합입니다.

![직원 역할](/static/core/admin/img/help/staff-roles/role-list.webp)

## 작동 방식

1. **역할**을 생성하여 특정한 권한 집합을 정의합니다 (예: "주문 관리자", "계산원")
2. 각 역할은 두 가지 유형의 접근을 제어합니다: **관리자 패널 권한** 및 **POS 권한**
3. 직원의 프로필 페이지에서 **역할을 할당**합니다
4. 직원의 실제 권한은 그들이 가진 모든 역할의 **합집합**입니다 — 어떤 역할이 접근을 허용하면 사용자는 해당 접근을 가집니다
5. 권한은 성능을 위해 **캐시**되며, 역할이 변경되면 자동으로 갱신됩니다

## 사전 정의된 역할

Spwig은 가장 일반적인 팀 구조를 커버하는 7개의 기본 역할을 포함합니다. 이 역할은 삭제할 수 없지만, 더 구체적인 요구사항을 위해 사용자 정의 역할을 생성할 수 있습니다.

| 역할 | 접근 | 설명 |
|------|--------|-------------|
| **가게 소유자** | 관리자 + POS | 모든 항목에 대한 전체 접근 권한. 주요 가게 관리자용입니다. |
| **가게 관리자** | 관리자 + POS | 일상적인 운영 — 제품, 주문, 고객, 마케팅, 검색에 대한 전체 접근. 디자인, 이메일, 결제, 설정은 읽기 전용입니다. |
| **콘텐츠 편집자** | 관리자 | 페이지, 블로그 게시물, 디자인, 미디어를 관리합니다. 제품은 읽기 전용입니다. |
| **주문 관리자** | 관리자 | 주문, 배송, 반품, 고객 서비스를 처리합니다. 제품은 읽기 전용입니다. |
| **마케팅 관리자** | 관리자 | 프로모션, 쿠폰, 파트너, 충성도, 추천 프로그램을 관리합니다. 제품, 고객, 미디어는 읽기 전용입니다. |
| **계산원** | POS 전용 | 프론트라인 POS 직원. 판매 처리 및 기프트 카드 잔액 확인이 가능합니다. 할인, 환불, 현금 관리는 불가능합니다. |
| **고급 계산원** | POS 전용 | 경험 많은 POS 직원. 환불 처리, 할인 적용(최대 25%), 현금 관리, 시프트 종료가 가능합니다. |

## 사용자 정의 역할 생성

**설정 > 직원 역할**로 이동하여 **역할 추가**를 클릭합니다.

### 일반 설정

| 설정 | 설명 |
|---------|-------------|
| **표시 이름** | 관리자에서 표시되는 역할 이름 (예: "창고 직원") |
| **설명** | 이 역할이 무엇을 위한 것인지 간단한 설명 |
| **정렬 순서** | 역할 목록에서 표시 순서를 결정합니다 |
| **아이콘** | 역할을 시각적으로 식별할 수 있는 20개의 아이콘 중 선택 |
| **배지 색상** | 역할 배지에 사용되는 색상 (파랑, 녹색, 주황색, 빨강, 청록, 회색) |
| **관리자 패널** | 이 역할이 관리자 백엔드 접근을 허용하는지 토글 |
| **POS 단말기** | 이 역할이 POS 단말기 접근을 허용하는지 토글 |

### 관리자 권한 범주

관리자 권한 탭은 플랫폼 기능을 13개 범주로 정리합니다. 각 범주에 대해 세 가지 접근 수준 중 하나를 설정합니다:

- **없음** — 이 영역에 대한 접근 없음 (메뉴 항목은 숨김)
- **읽기** — 읽기 전용 접근 (데이터를 볼 수 있지만 변경할 수 없음)
- **전체** — 전체 접근 (데이터를 볼 수 있고, 생성, 편집, 삭제 가능)

![권한 범주](/static/core/admin/img/help/staff-roles/permission-categories.webp)

| 범주 | 제어 항목 |
|----------|-----------------|
| **제품 카탈로그** | 제품, 카테고리, 브랜드, 속성, 재고, 창고, 디지털 자산 |
| **주문 및 충족** | 주문, 환불, 반품, 배송, 배송 설정 |
| **고객** | 고객 프로필, 세그먼트, 분석 |
| **콘텐츠 및 페이지** | 페이지, 블로그 게시물, 공지, 폼 |
| **디자인 및 테마** | 테마, 헤더/푸터 템플릿, 메뉴, 디자인 토큰, 사용자 정의 CSS |
| **마케팅 및 프로모션** | 프로모션, 쿠폰, 파트너, 충성도, 추천, 제품 피드 |
| **미디어 라이브러리** | 이미지, 동영상, 폴더, 태그 |
| **이메일 시스템** | 이메일 계정, 템플릿, 전송 대기열 |
| **결제 및 청구** | 결제 제공자, 거래, 웹후크, 구독, 환율 |
| **검색** | 검색 설정, 동의어, 리디렉션, 분석 |
| **가게 설정** | 사이트 설정, 지리적 위치, 국가 매핑, 비즈니스 규칙 |
| **POS 관리** | POS 단말기, 시프트, 현금 이동, 영수증 템플릿 |
| **사용자 및 역할** | 직원 사용자 계정, 역할, API 토큰 |

사용자가 여러 역할을 가진 경우 **가장 높은** 접근 수준이 적용됩니다. 예를 들어, 역할 A가 제품에 대해 "읽기"를 허용하고 역할 B가 "전체"를 허용하면 사용자는 "전체" 접근을 가집니다.

### POS 권한 플래그

역할이 POS 접근을 허용하는 경우, POS 권한 탭은 POS 운영자가 할 수 있는 것을 정확히 세부 조정할 수 있습니다. 이는 관리자 권한과 별도이며, POS 단말기에서 확인됩니다.

![POS 권한](/static/core/admin/img/help/staff-roles/pos-permissions.webp)

| 그룹 | 권한 | 설명 |
|-------|-----------|-------------|
| **일반** | POS 접근 | 모든 POS 시스템을 사용할 수 있음 |
| **판매 및 할인** | 수동 할인 | 수동 항목 또는 장바구니 수준의 할인을 적용할 수 있음 |
| | 최대 할인 % | 허용되는 최대 할인 비율 (0–100) |
| | 가격 재설정 | 판매대에서 제품 가격을 재설정할 수 있음 |
| **환불 및 무효화** | 환불 처리 | POS 주문에 대해 환불을 처리할 수 있음 |
| | 주문 무효화 | 현재 시프트에서 POS 주문을 무효화할 수 있음 |
| **기프트 카드** | 기프트 카드 발행 | 판매대에서 새로운 기프트 카드를 발행할 수 있음 |
| | 기프트 카드 잔액 확인 | 기프트 카드 잔액을 확인할 수 있음 |
| **현금 관리** | 현금 관리 | 현금 입금 및 출금 작업을 수행할 수 있음 |
| | 현금 인출 | 판매 없이 현금 인출을 할 수 있음 |
| | 시프트 종료 | 시프트를 종료하고 현금 정산을 수행할 수 있음 |
| **보고서** | POS 보고서 보기 | 시프트 보고서 및 판매 요약을 볼 수 있음 |
| **재고** | 재고 조정 | 재고 수준을 조정할 수 있음 (수령, 손상, 재점검, 반품) |

불리언 권한의 경우, 사용자의 역할 중 **어떤 역할이라도** 이를 활성화하면 사용자는 해당 권한을 가집니다. 최대 할인 %의 경우, 모든 역할 중 **가장 높은 값**이 적용됩니다.

## 직원 관리

**설정 > 직원 관리**로 이동하여 팀을 보고 관리합니다.

### 직원 목록

직원 목록은 직원 접근이 있는 모든 사용자를 보여줍니다. 각 직원에 대해 다음과 같은 정보를 볼 수 있습니다:
- **이름 및 이메일**
- **할당된 역할** (색상 배지로 표시)
- **접근 유형** — 관리자 전용, POS 전용, 또는 둘 다
- **2FA 상태** — 두 요인 인증이 활성화되어 있는지 여부
- **활성/비활성** 상태

필터를 사용하여 역할, 접근 유형, 또는 2FA 상태에 따라 좁혀볼 수 있습니다.

### 직원에게 역할 할당

1. 직원을 클릭하여 프로필을 열기
2. **역할** 섹션에서 사용 가능한 각 역할에 대한 카드를 볼 수 있습니다
3. 역할 카드의 토글을 클릭하여 역할을 할당하거나 제거합니다
4. 변경 사항은 즉시 적용되며, 저장 버튼이 필요 없습니다
5. 아래의 **실제 권한 요약**은 할당된 모든 역할의 결과를 보여줍니다

### 새 직원 추가

1. **설정 > 직원 관리**로 이동하여 **직원 추가**를 클릭합니다
2. 사용자의 이메일, 이름, 성을 입력합니다
3. 임시 비밀번호를 설정합니다
4. 하나 이상의 역할을 할당합니다
5. 사용자는 이제 역할이 제공하는 접근을 통해 로그인할 수 있습니다

## 역할 복제

기존 역할을 기반으로 새 역할을 생성하려면:

1. 복사할 역할을 열어주세요
2. 페이지 하단의 **역할 복제**를 클릭합니다
3. 동일한 모든 권한을 가진 새 역할이 생성됩니다
4. 이름을 바꾸고 필요한 경우 권한을 조정합니다
5. 새 역할을 저장합니다

이 기능은 기존 역할과 유사하지만 약간 다른 역할이 필요할 때 유용합니다 — 예를 들어, "주니어 매니저"는 "가게 매니저"를 기반으로 하지만 더 적은 권한을 가집니다.

## 권한 적용 방식

### 관리자 패널

- **메뉴 가시성** — 사용자가 "없음" 접근을 가진 범주에 대해 사이드바 섹션은 숨겨집니다
- **페이지 접근** — 제한된 페이지에 접근하려 할 때 권한 오류가 표시됩니다
- **작업 제한** — "읽기" 접근을 가진 경우 편집 및 삭제 버튼은 숨겨지고 저장 작업은 차단됩니다
- **슈퍼유저 우회** — 슈퍼유저 계정은 역할 할당과 관계없이 항상 전체 접근을 가집니다

### POS 단말기

- **로그인 게이트** — 최소 하나의 역할이 "POS 단말기"를 활성화한 사용자만 POS에 로그인할 수 있습니다
- **기능 토글** — POS 버튼 및 작업(환불, 할인, 무효화 등)은 사용자의 합산된 POS 권한에 따라 표시되거나 숨겨집니다
- **할인 한도** — 최대 할인 %는 POS 운영자가 적용할 수 있는 할인의 하드한 한도를 설정합니다
- **API 강제** — 모든 POS 권한은 API 레이어에서 서버 측에서 확인되며, UI에서만 확인하는 것이 아닙니다

## 팁

- **사전 정의된 역할부터 시작하세요** — 7개의 기본 역할은 대부분의 팀 구조를 커버합니다. 더 구체적인 접근 제어가 필요할 때만 사용자 정의 역할을 생성하세요.
- **복제 기능을 사용하세요** — 기존 역할과 유사한 역할이 필요할 경우, 복제하여 조정하세요. 처음부터 만드는 대신에요.
- **필요할 때 여러 역할을 할당하세요** — 주문과 마케팅을 모두 처리하는 직원은 "주문 관리자" 및 "마케팅 관리자" 역할을 모두 할당받을 수 있습니다. 권한은 자동으로 결합됩니다.
- **관리자 및 POS 접근을 분리하세요** — 계산원은 일반적으로 관리자 접근이 필요하지 않으며, 사무직은 POS 접근이 필요하지 않습니다. 접근 토글을 사용하여 정리된 상태를 유지하세요.
- **POS 직원에게 할인 한도를 설정하세요** — 최대 할인 %는 계산원이 과도한 할인을 적용하는 것을 방지합니다. 할인을 적용하지 않으려면 0으로 설정하거나, 고급 직원에게 10–25%와 같은 합리적인 한도를 설정하세요.
- **정기적으로 역할을 검토하세요** — 팀이 성장하면서 역할 할당을 감사하여 직원들이 업무에 필요한 최소한의 접근을 가진지 확인하세요. 직원들이 직무를 변경할 때 역할을 제거하세요.
- **민감한 역할에 2FA를 활성화하세요** — 결제, 설정, 사용자 관리에 접근할 수 있는 직원들은 보안을 위해 두 요인 인증을 활성화해야 합니다.
