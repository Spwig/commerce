付款服务提供商设置

付款服务提供商设置允许您配置PayPal和Airwallex以实现自动联盟会员付款。本指南将向您展示如何连接您的付款服务提供商账户，配置webhook，并测试您的集成。

## 受支持的付款服务提供商

Spwig与两个付款服务提供商集成，以实现自动联盟会员付款：

| 提供商 | 付款方式 | 处理方式 | 批量支持 | 最佳用途 |
|----------|----------------|------------|---------------|----------|
| **PayPal** | PayPal账户转账 | 基于API | 是（最多15,000） | 大多数联盟会员，全球覆盖 |
| **Airwallex** | 国际银行转账 | 基于API | 否（单个） | 银行转账，国际付款 |

### 关键差异

**PayPal付款**：
- 要求联盟会员拥有PayPal账户（付款电子邮件）
- 可一次处理多达15,000笔付款的批量
- 处理速度更快（1-2个工作日）
- 设置复杂度较低
- 费用：约2%或每笔付款0.25-1.00美元
- 整个批量的单一webhook

**Airwallex**：
- 支持直接银行转账
- 逐个处理单笔付款
- 处理时间更长（2-5个工作日）
- 支持多种货币和国家
- 费用因目的地国家而异
- 每笔付款的单独webhook

您可以配置这两个提供商，并让联盟会员选择他们偏好的付款方式。

## 为什么使用付款服务提供商？

与手动付款相比，集成付款服务提供商具有显著优势：

- **自动化处理** — 无需手动输入数据或执行付款
- **批量效率** — 一键处理数十或数百笔付款
- **webhook确认** — 付款完成后自动更新状态
- **减少错误** — 系统在处理前验证账户详细信息
- **审计跟踪** — 交易和提供商响应的完整记录
- **更快的付款** — 联盟会员更快收到资金
- **可扩展性** — 在不增加管理工作的前提下处理日益增长的联盟会员计划

如果没有提供商集成，您必须通过银行或PayPal仪表板手动处理每笔付款，然后返回Spwig将付款标记为完成。

## PayPal设置

按照以下步骤配置PayPal付款以实现自动联盟会员付款。

### 先决条件

在开始之前，您需要：
- 一个PayPal商业账户（个人账户无法使用付款API）
- 访问PayPal开发人员仪表板
- 生产环境的付款API批准（在沙箱测试后）

### 第1步：创建PayPal应用

1. **导航**到[PayPal开发人员仪表板](https://developer.paypal.com/dashboard/)
2. **使用**您的PayPal商业账户登录
3. **点击**左侧边栏的**我的应用和凭证**
4. **选择****实时**选项卡（或沙箱用于测试）
5. **点击****创建应用**
6. **输入应用名称**（例如，"Spwig联盟会员付款"）
7. **选择应用类型**：商户
8. **点击****创建应用**

PayPal会生成您的凭证。

### 第2步：获取API凭证

创建应用后：

1. **复制客户端ID** — 长的字母数字字符串
2. **点击****显示**在Secret下方
3. **复制客户端密钥** — 保持此密钥保密
4. **注意模式** — 沙箱或实时

### 第3步：启用付款功能

PayPal应用需要明确的权限来使用付款功能：

1. **滚动到**您的应用的**功能**部分
2. **查找****付款**功能
3. **点击****添加**如果尚未启用
4. **提交以获得批准**如果使用实时模式（批准需要1-2个工作日）

### 第4步：在Spwig中添加提供商

现在将PayPal账户添加到Spwig：

1. **导航**到**设置 > 付款服务提供商**
2. **点击****+ 添加PayPal账户**
3. **填写表单**：
   - **账户名称**：描述性标签（例如，"主PayPal账户"）
   - **客户端ID**：从PayPal开发人员仪表板粘贴
   - **客户端密钥**：从PayPal开发人员仪表板粘贴
   - **模式**：选择沙箱（测试）或生产（实时）
   - **是否启用**：勾选以启用
4. **点击保存**

Spwig通过请求访问令牌来验证凭证。如果验证失败，请再次检查您的客户端ID和密钥。

### 第5步：测试连接

验证您的PayPal集成：

1. 在**联盟会员计划 > 付款**中创建测试付款
2. 使用您自己的PayPal电子邮件作为收件人
3. 设置金额为$0.01（如果在生产环境）或任意金额（如果沙箱）
4. 使用提供商处理
5. 检查PayPal账户中的入账
6. 验证webhook更新Spwig中的付款状态

如果使用沙箱模式，请在[PayPal沙箱](https://developer.paypal.com/dashboard/accounts)创建测试PayPal账户以接收测试付款。

## Airwallex设置

Airwallex支持国际银行转账，适用于偏好直接存款的联盟会员。

### 先决条件

在开始之前，您需要：
- 一个Airwallex账户（在[airwallex.com](https://www.airwallex.com)创建）
- 验证的商业账户状态
- 启用API访问（如需，联系Airwallex支持）
- Airwallex账户中足够的余额

### 第1步：生成API凭证

1. **登录**到[Airwallex仪表板](https://www.airwallex.com/app/)
2. **导航**到**设置 > API密钥**
3. **点击****创建API密钥**
4. **输入描述**："Spwig联盟会员付款"
5. **选择权限**：启用**付款**（读取和写入）
6. **点击****生成**
7. **复制API密钥** — 仅显示一次
8. **复制客户端ID** — 与密钥一起显示

### 第2步：注意您的环境

Airwallex提供两种环境：

- **演示**：用于测试虚假交易
- **生产**：用于真实资金转账

确保您知道您的API密钥属于哪个环境。

### 第3步：在Spwig中添加提供商

将Airwallex账户添加到Spwig：

1. **导航**到**设置 > 付款服务提供商**
2. **点击****+ 添加Airwallex账户**
3. **填写表单**：
   - **账户名称**：描述性标签（例如，"Airwallex EUR账户"）
   - **API密钥**：从Airwallex仪表板粘贴
   - **客户端ID**：从Airwallex仪表板粘贴
   - **环境**：选择演示或生产
   - **是否启用**：勾选以启用
4. **点击保存**

Spwig通过查询您的账户余额来验证凭证。

### 第4步：验证支持的国家

Airwallex支持许多国家的转账，但并非所有国家。在[Airwallex覆盖范围](https://www.airwallex.com/global-business-account/global-transfers)页面上确认您的联盟会员的国家是否受支持。

常见的支持国家包括：
- 美国
- 英国
- 欧盟国家
- 澳大利亚
- 加拿大
- 新加坡
- 香港

### 第5步：测试银行转账

测试您的Airwallex集成：

1. 为具有银行详细信息的联盟会员创建测试付款
2. 如果在生产模式中，使用小额金额（$1-$5）
3. 使用提供商处理
4. 检查Airwallex仪表板上的交易
5. 等待webhook确认
6. 验证Spwig中的付款完成

演示模式立即处理。生产模式需要2-5个工作日。

## 提供商选择逻辑

当您处理付款时，Spwig会根据联盟会员的付款方式自动选择适当的提供商。

### 选择流程

1. **检查联盟会员的付款方式**：
   - 如果设置了`payment_email` → 联盟会员偏好PayPal
   - 如果设置了银行详细信息 → 联盟会员偏好银行转账
2. **匹配到提供商**：
   - PayPal电子邮件 → 使用启用的PayPal提供商账户
   - 银行详细信息 → 使用启用的Airwallex提供商账户
3. **如果首选提供商未配置，回退到第一个可用的**
4. **如果没有匹配的提供商，显示错误**

### 多个提供商账户

您可以为同一提供商配置多个账户（例如，用于不同地区的两个PayPal账户）。Spwig会选择第一个匹配付款方式的启用账户。要控制使用哪个账户，请在管理员列表中重新排序，或仅启用一个账户。

## 测试付款集成

在处理实际付款给联盟会员之前，始终测试您的提供商集成。

### 沙箱/演示模式测试

1. **将提供商设置为沙箱模式**（PayPal沙箱或Airwallex演示）
2. **创建测试联盟会员**并使用测试付款详细信息
3. **创建测试佣金**并批准它们
4. **创建包含这些佣金的测试付款**
5. **使用操作菜单通过提供商处理**
6. **监控Celery日志**以查看API请求
7. **检查提供商仪表板**上的交易
8. **等待webhook**以更新付款状态
9. **验证佣金标记为已支付**

### 生产测试

在上线前：

1. **在提供商设置中切换到生产模式**
2. **创建一个小的测试付款**给自己（$0.01-$1.00）
3. **处理它**并等待完成
4. **验证资金是否收到**在您自己的账户中
5. **检查webhook是否触发**并更新状态
6. **查看提供商交易费用**

### 常见测试问题

| 问题 | 原因 | 解决方案 |
|-------|-------|----------|
| "无效的凭证" | 错误的API密钥或模式不匹配 | 重新检查凭证，确认沙箱与生产 |
| webhook从未触发 | 未在提供商中配置URL | 在提供商仪表板中添加webhook URL |
| 付款停留在处理中 | webhook签名失败 | 检查webhook密钥是否匹配 |
| 没有可用的提供商 | 无启用的提供商用于付款方式 | 启用至少一个提供商账户 |

## 批量处理（PayPal）

PayPal支持批量处理以提高效率和节省成本。

### 批量处理的工作原理

当您选择多个付款并点击**使用提供商处理**时：

1. Spwig将所有PayPal付款分组为一个批次
2. 系统发送一个包含所有付款详情的API请求（最多15,000）
3. PayPal将整个批次作为单个交易处理
4. webhook返回批次结果
5. Spwig根据批次响应更新所有付款

### 批量处理的优势

- **减少API调用** — 一个请求处理数百笔付款
- **降低费用** — 一些PayPal费用结构更倾向于批量处理
- **更快的处理** — 整个批次并行执行
- **单一webhook** — 更容易监控和记录

### 批量处理限制

PayPal施加这些限制：
- 每个批次最多15,000个收款人
- 每个批次总金额上限为$100,000
- 处理通常在几分钟内完成

如果您超过15,000笔付款，Spwig会自动拆分为多个批次。

## 个体处理（Airwallex）

Airwallex逐个处理付款，这带来了不同的权衡。

### 个体处理的工作原理

当您处理Airwallex付款时：

1. 系统为每个付款发送单独的API请求
2. Airwallex将转账单独排队
3. 每个转账独立完成（2-5天）
4. 每个转账完成后触发单独的webhook
5. Spwig在webhook到达时更新付款

### 个体处理的优势

- **更好的错误隔离** — 一个失败不会阻塞其他
- **按付款跟踪** — 每个交易的ID
- **更多付款详情** — 每个转账的银行特定信息
- **灵活的时间** — 转账以不同速度完成

### 处理时间

与PayPal的即时批量处理不同，Airwallex转账需要更长时间：
- 国内转账：1-2个工作日
- 国际转账：3-5个工作日
- 一些国家：最多7个工作日

根据您的计划条款相应地设置联盟会员的期望。

## webhook配置

webhook允许在提供商完成交易时自动更新付款状态。

### webhook URL格式

在提供商仪表板中配置此URL：

```
https://yourdomain.com/api/payout-providers/{provider}/webhook/
```

将{provider}替换为：
- `paypal`用于PayPal webhook
- `airwallex`用于Airwallex webhook

示例：
- `https://shop.example.com/api/payout-providers/paypal/webhook/`
- `https://shop.example.com/api/payout-providers/airwallex/webhook/`

### PayPal webhook设置

1. **导航**到[PayPal开发人员仪表板](https://developer.paypal.com/dashboard/)
2. **点击**您的应用名称
3. **滚动到****webhook**部分
4. **点击****添加webhook**
5. **输入webhook URL**（格式如上）
6. **选择事件**：
   - `PAYMENT.PAYOUTSBATCH.SUCCESS`
   - `PAYMENT.PAYOUTSBATCH.DENIED`
   - `PAYMENT.PAYOUTS-ITEM.SUCCEEDED`
   - `PAYMENT.PAYOUTS-ITEM.FAILED`
7. **点击保存**

PayPal提供一个webhook签名密钥。Spwig使用此密钥验证webhook的真实性。

### Airwallex webhook设置

1. **导航**到[Airwallex仪表板](https://www.airwallex.com/app/)
2. **进入****设置 > webhook**
3. **点击****创建webhook**
4. **输入webhook URL**（格式如上）
5. **选择事件**：
   - `transfer.created`
   - `transfer.completed`
   - `transfer.failed`
6. **点击创建**

Airwallex使用您的API密钥对webhook进行签名。

### webhook安全

webhook通过以下机制进行验证：

- **签名验证** — 提供商使用密钥对webhook负载进行签名
- **时间戳检查** — 拒绝旧的webhook（防止重放攻击）
- **IP白名单（可选）** — 限制到提供商IP范围
- **HTTPS必需** — webhook仅在SSL上工作

在生产环境中，**永远不要禁用签名验证**。

### 测试webhook

大多数提供商提供webhook测试工具：

**PayPal**：在开发人员仪表板中使用"模拟器"来触发测试webhook

**Airwallex**：在演示模式中创建测试转账并观察webhook

您还可以在Spwig的**设置 > 系统日志**中检查webhook日志（如果启用了日志记录）。

## 故障排除

### 无效凭证错误

**症状**：保存提供商账户时出现"认证失败"

**原因**：
- 错误的客户端ID或密钥
- 在生产模式中使用沙箱凭证（或相反）
- API密钥过期或被撤销
- 账户未验证

**解决方案**：
- 从提供商仪表板重新复制凭证
- 确认模式匹配（沙箱与生产）
- 重新生成API密钥
- 联系提供商支持以确认账户状态

### 未收到webhook

**症状**：付款状态卡在"处理中"无限期

**原因**：
- 未在提供商仪表板中配置webhook URL
- HTTPS证书无效
- 防火墙阻止提供商IP
- webhook签名验证失败

**解决方案**：
- 仔细检查提供商设置中的webhook URL
- 确认SSL证书有效
- 在防火墙中白名单提供商IP范围
- 检查Celery日志中的签名错误
- 使用提供商的模拟器工具测试webhook

### 付款失败

**症状**：付款状态变为"失败"并带有错误信息

**原因**：
- 无效的联盟会员付款详细信息（错误的电子邮件或银行账户）
- 提供商账户余额不足
- 收款账户无法接收付款
- 国家不支持（Airwallex）
- 付款超过提供商限制

**解决方案**：
- 在**提供商响应**字段中查看错误
- 确认联盟会员的付款详细信息正确
- 向提供商账户添加资金
- 要求联盟会员检查其账户状态
- 检查提供商的国家和货币支持
- 如果超过限制，拆分大额付款

### 模式不匹配

**症状**：测试付款有效但生产付款失败

**原因**：
- 提供商设置为沙箱模式但使用生产联盟会员账户
- API凭证来自错误环境

**解决方案**：
- 将提供商模式切换为生产
- 重新生成生产API凭证
- 确认webhook URL指向生产域名

## 安全最佳实践

使用这些安全措施保护您的付款集成：

### 凭证存储

- **永远不要将凭证提交到版本控制** — 使用环境变量或安全存储
- **每季度轮换API密钥** — 每3个月生成新密钥
- **为沙箱和生产使用单独的密钥** — 永远不要混合环境
- **限制API权限** — 仅授予付款访问权限，而不是完整账户控制

Spwig在数据库中加密存储提供商凭证。确保您的数据库备份安全。

### webhook安全

- **始终验证签名** — 永远不要跳过签名验证
- **仅使用HTTPS** — 不支持HTTP webhook
- **实施IP白名单** — 限制webhook到提供商IP范围
- **记录所有webhook** — 监控可疑活动
- **对webhook端点进行速率限制** — 防止滥用

### 访问控制

- **限制员工访问** — 仅受信任的员工应处理付款
- **使用双因素认证** — 要求员工账户使用2FA
- **审核付款操作** — 审查谁处理了哪些付款
- **职责分离** — 不同员工负责批准与处理

### 监控

- **每天检查失败的付款** — 及时解决问题
- **监控提供商账户余额** — 确保有足够的资金
- **每周审查交易日志** — 早期发现异常
- **设置警报** — 通过电子邮件通知大额或失败的付款

## 建议

- 在切换到生产环境之前，在沙箱模式下彻底测试您的集成 — 使用虚拟货币发现问题。
- 配置PayPal和Airwallex，以提供联盟会员付款选择 — 不同的联盟会员偏好不同的方法。
- 在初始设置期间设置webhook URL并验证它们是否正确触发 — webhook对于自动化至关重要。
- 保持提供商账户余额充足，以避免在批量处理期间出现付款失败。
- 如果配置了多个提供商，请使用描述性的账户名称（例如，"PayPal USD"，"PayPal EUR"）。
- 作为安全最佳实践，每季度轮换API凭证。
