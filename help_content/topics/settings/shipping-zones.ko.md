---
title: 배송 지역
---

배송 지역은 특정한 배송 비용을 위한 지리적 지역을 정의합니다. 국가, 주/도, 우편번호를 지역으로 그룹화한 후, 특정 지역에 해당하는 배송 방식을 연결하여 정확한 비용을 제어할 수 있습니다. 주소가 여러 지역에 해당할 경우 우선순위 기반으로 일치하는 지역이 결정됩니다(가장 높은 우선순위가 승리). 이 시스템은 복잡한 가격 전략을 가능하게 합니다: 원격 지역에 더 비싼 비용을 부과하거나, 국내 배송은 무료로 제공하거나, 특정 지역에 할인된 비용을 제공할 수 있습니다.

지리적 지역에 따라 다른 배송 비용이 필요한 경우, 간단한 국내 대 국제 분할부터 복잡한 다중 지역 계층 가격까지 지역을 사용하세요.

## 배송 지역 이해

**지역이란 무엇인가**: 국가, 주/도, 우편번호 패턴으로 정의된 이름이 있는 지리적 지역입니다.

**지역이 어떻게 작동하는가**:
1. 고객이 결제 시 배송 주소를 입력합니다
2. 시스템은 모든 활성 지역을 평가합니다
3. 고객의 주소와 일치하는 지역이 후보로 됩니다
4. 여러 지역이 일치하는 경우, 가장 높은 우선순위 지역이 승리합니다
5. 승리한 지역에 연결된 배송 방식이 표시됩니다
6. 어떤 지역에도 연결되지 않았거나 일치하는 지역에 연결된 방식도 표시됩니다

**지역 구성 요소**:
- **이름**: 지역 식별자(예: "국내", "EU", "원격 지역")
- **국가**: 포함된 국가 코드 목록(빈 값 = 모든 국가)
- **주/도**: 국가별 주/도 제한(선택 사항)
- **우편번호 패턴**: ZIP/우편번호 일치를 위한 정규식 패턴(선택 사항)
- **우선순위**: 여러 지역이 일치할 때 더 높은 숫자 = 더 높은 우선순위

---

## 지역 일치 논리

지역은 주소와 일치하기 위해 **점진적 좁히기**를 사용합니다:

### 레벨 1: 국가 일치

**빈 국가 목록** → 모든 국가에 해당하는 지역

**국가 목록 제공** → 주소 국가가 목록에 포함되어야 함

예시:
```
지역: "국내"
국가: ["US"]
→ 일치: 미국 주소
→ 불일치: 캐나다, 영국 등
```

### 레벨 2: 주/도 일치

**정의된 주/도가 없음** → 허용된 국가의 모든 주/도에 해당하는 지역


**특정 국가에 정의된 주** → 주소의 주가 일치해야 함

예시:
```
Zone: "West Coast"
Countries: ["US"]
States: {"US": ["CA", "OR", "WA"]}
→ 일치: 캐リフォ니아, 오레곤, 워싱턴 주소
→ 일치하지 않음: 뉴욕, 텍사스 등
```

### 레벨 3: 우편번호 일치

**정의된 패턴 없음** → 허용된 국가/주에 있는 모든 우편번호와 일치

**패턴 정의됨** → 주소의 우편번호가 적어도 하나의 패턴과 일치해야 함

예시:
```
Zone: "Los Angeles Metro"
Countries: ["US"]
States: {"US": ["CA"]}
Postal Patterns: ["^90[0-9]{3}$", "^91[0-9]{3}$"]
→ 일치: 90001, 91210, 90245
→ 일치하지 않음: 94102 (샌프란시스코)
```

**정규식 패턴 예시**:
- `^90[0-9]{3}$` - 로스앤젤레스 지역 (90000-90999)
- `^[A-Z][0-9][A-Z] [0-9][A-Z][0-9]$` - 캐나다 우편번호 형식 (K1A 0B1)
- `^SW[0-9]{1,2}` - 영국 런던 우편번호 중 SW로 시작하는 것

## 우선순위 기반 존 선택

다중 존이 주소와 일치할 경우, **우선순위**가 적용되는 존을 결정합니다:

**우선순위 작동 방식**:
- 숫자가 높을수록 우선순위가 높음
- 주소가 우선순위 100과 50을 가진 존과 모두 일치할 경우, 우선순위 100이 적용됨
- 오직 승리한 존의 배송 방법만 사용 가능함

**사용 사례**:

**시나리오 1: 구체적인 것이 일반적인 것을 대체함**
```
Zone A: "Remote Alaska"
  Countries: ["US"]
  States: {"US": ["AK"]}
  Priority: 100

Zone B: "Domestic USA"
  Countries: ["US"]
  Priority: 50

Address: Anchorage, AK
→ 두 존 모두 일치
→ 우선순위 100이 승리
→ "Remote Alaska" 존이 적용됨 (높은 배송 비용)
```

**시나리오 2: 우편번호가 주를 대체함**
```
Zone A: "Manhattan Premium"
  Countries: ["US"]
  States: {"US": ["NY"]}
  Postal Patterns: ["^100[0-2][0-9]$"]
  Priority: 100

Zone B: "New York State"
  Countries: ["US"]
  States: {"US": ["NY"]}
  Priority: 50

Address: New York, NY 10001
→ 두 존 모두 일치
→ 우선순위 100이 승리
→ "Manhattan Premium"이 적용됨 (프리미엄 배송 서비스)
```

## 배송 존 생성

**단계별 워크플로**:


1. **지역으로 이동**
   - 설정 > 배송 > 배송 지역으로 이동
   - "배송 지역 추가"를 클릭

2. **기본 설정**
   - **이름**: 설명적인 식별자 (예: "유럽 연합", "서부 해안", "원격 지역")
   - **우선순위**: 상대적 중요도 설정 (구체적: 100, 일반: 50, 대체: 1)
   - **활성화**: 활성화/비활성화 전환

3. **지리적 범위 정의**

   **옵션 A: 모든 국가** (국가 목록을 비워둠)
   - 지역은 전 세계 모든 주소와 일치함
   - 기본/대체 지역에 사용

   **옵션 B: 특정 국가**
   - "국가 추가"를 클릭
   - 드롭다운에서 국가 선택 (US, CA, UK 등)
   - 포함된 모든 국가에 반복

   **옵션 C: 특정 주/도**
   - 국가를 추가한 후, 각 국가에 대해 "주 추가"를 클릭
   - 드롭다운에서 주 선택
   - 예: US → CA, OR, WA 서부 해안

   **옵션 D: 우편번호 패턴** (고급)
   - 정규 표현식 패턴 입력 (한 줄에 하나씩)
   - 샘플 우편번호로 패턴 테스트
   - "패턴 검증" 클릭하여 구문 확인

4. **배송 방법과 연결**
   - 방법은 방법 편집 시에 연결할 수 있음 (지역 설정에서 아님)
   - 또는 기존 방법에 지역 연결: 방법 편집 → 배송 지역 → 지역 선택

5. **표시 우선순위 설정**
   - 여러 지역이 일치할 경우, 높은 우선순위 지역이 낮은 우선순위를 덮음
   - 권장: 구체적 지역 (100), 지역별 지역 (50), 기본 지역 (1)

6. **지역 활성화**
   - "활성화" 전환 = 예
   - 저장

## 일반적인 지역 설정

### 설정 1: 국내 vs 국제

**목표**: 국내와 다른 모든 국가에 대한 다른 요금.

```
지역 1: "국내"
  국가: [귀하의 국가 코드]
  우선순위: 50

지역 2: "국제"
  국가: [비워둠 또는 다른 모든 국가 선택]
  우선순위: 1
```

**배송 방법**:
- "국내 표준" → 국내 지역과 연결
- "국제 배송" → 국제 지역과 연결

---

### 설정 2: 다국적 국제

**목표**: EU, 북미, 아시아, 세계 나머지 지역에 대한 다른 요금.

```
Zone 1: "European Union"
  Countries: [AT, BE, BG, CY, CZ, DE, DK, EE, ES, FI, FR, GR, HR, HU, IE, IT, LT, LU, LV, MT, NL, PL, PT, RO, SE, SI, SK]
  Priority: 100

Zone 2: "North America"
  Countries: [US, CA, MX]
  Priority: 100

Zone 3: "Asia Pacific"
  Countries: [AU, CN, HK, IN, JP, KR, NZ, SG, TH, TW]
  Priority: 100

Zone 4: "Rest of World"
  Countries: [Leave empty]
  Priority: 1
```

**배송 방법**:
- "EU 배송" → EU 지역
- "북미 배송" → 북미 지역
- "아시아 태평양 배송" → 아시아 태평양 지역
- "국제 표준" → 세계 나머지 지역

---

### 설정 3: 원격 지역 추가 요금

**목표**: 국내 지역 내 원격 우편 번호에 추가 요금을 부과합니다.

```
Zone 1: "Remote Domestic"
  Countries: [US]
  Postal Patterns: ["^99[0-9]{3}$", "^96[7-9][0-9]{2}$"]  # Alaska, Hawaii
  Priority: 100

Zone 2: "Standard Domestic"
  Countries: [US]
  Priority: 50
```

**배송 방법**:
- "원격 배송" → 원격 국내 지역 (높은 비용)
- "표준 배송" → 표준 국내 지역

---

### 설정 4: 주별 구역

**목표**: 미국 지역별로 다른 요금을 적용합니다.

```
Zone 1: "West Coast"
  Countries: [US]
  States: {"US": ["CA", "OR", "WA"]}
  Priority: 100

Zone 2: "East Coast"
  Countries: [US]
  States: {"US": ["NY", "NJ", "CT", "MA", "PA"]}
  Priority: 100

Zone 3: "Midwest"
  Countries: [US]
  States: {"US": ["IL", "IN", "OH", "MI", "WI"]}
  Priority: 100

Zone 4: "South"
  Countries: [US]
  States: {"US": ["TX", "FL", "GA", "NC", "SC"]}
  Priority: 100

Zone 5: "Other US States"
  Countries: [US]
  Priority: 50
```

## 우편 번호 패턴 예시

우편 번호는 **정규식**(정규 표현식)을 사용하여 패턴 일치를 수행합니다:

### 미국 (ZIP 코드)

**형식**: 5자리 숫자 (예: 90210)

```
California (90000-96199):  ^9[0-6][0-9]{3}$
New York (10000-14999):    ^1[0-4][0-9]{3}$
Texas (75000-79999, 88500-88599):  ^(7[5-9]|885)[0-9]{2}$
Alaska (99500-99999):      ^99[5-9][0-9]{2}$
```

### 캐나다 (우편번호)

**형식**: A1A 1A1 (문자-숫자-문자 공백 숫자-문자-숫자)

```
모든 캐나다 우편번호:  ^[A-Z][0-9][A-Z] [0-9][A-Z][0-9]$
온타리오 (K, L, M, N, P):    ^[KLMNP][0-9][A-Z] [0-9][A-Z][0-9]$
퀘벡 (G, H, J):           ^[GHJ][0-9][A-Z] [0-9][A-Z][0-9]$
```

### 영국 (우편번호)

**형식**: AA1A 1AA 또는 A1A 1AA

```
런던 (E, EC, N, NW, SE, SW, W, WC):  ^(E|EC|N|NW|SE|SW|W|WC)[0-9]{1,2}
맨체스터 (M):                        ^M[0-9]{1,2}
버밍엄 (B):                        ^B[0-9]{1,2}
```

### 호주 (우편번호)

**형식**: 4자리 숫자 (예: 2000)

```
뉴사우스웨일스 (1000-2999):  ^[12][0-9]{3}$
빅토리아 (3000-3999, 8000-8999):  ^[38][0-9]{3}$
퀸즐랜드 (4000-4999, 9000-9999):  ^[49][0-9]{3}$
```

### 패턴 테스트

**패턴을 저장하기 전에**, 알려진 우편번호로 테스트하세요:

1. 패턴 입력: `^90[0-9]{3}$`
2. 테스트 입력: "90210" → 일치해야 함
3. 테스트 입력: "10001" → 일치하지 않아야 함
4. 테스트 입력: "9021" → 일치하지 않아야 함 (4자리 숫자만 허용)

복잡한 패턴을 검증하려면 온라인 정규표현식 테스터(예: regex101.com)를 사용하세요.

## 지역 커버리지 요약

지역은 관리 목록 뷰에서 **커버리지 요약**을 표시하여 포함된 내용을 보여줍니다:

**예시**:
- "모든 국가" → 국가 제한 없음
- "US, CA, MX" → 3개 국가
- "US (CA, OR, WA)" → 미국에 포함된 3개 주
- "US (90xxx-91xxx)" → 미국에 포함된 우편번호 패턴

**요약 사용 목적**:
- 열람 없이 지역 커버리지를 빠르게 확인
- 커버리지 중복 또는 공백을 발견
- 지역 구성 상태를 한눈에 확인

## 지역과 배송 방법 연결

지역과 방법은 **다대다 관계**를 가집니다:

**방법 측에서**(권장):
1. 배송 방법 편집
2. "배송 지역" 섹션으로 스크롤
3. 적용 가능한 지역 선택 (다중 선택)
4. 방법 저장


- Zones don't directly link to methods
- Linking is always done from method configuration

```
Method: "Domestic Standard"
Linked Zones: ["Domestic USA"]
→ Only shown to US addresses

Method: "International Express"
Linked Zones: ["EU", "Asia Pacific", "Rest of World"]
→ Shown to all non-US addresses
```

- Use addresses in different zones
- Verify correct zone matches

- Use address that matches multiple zones
- Verify highest priority zone wins
- Confirm expected shipping methods appear

- Border postal codes (e.g., 90999 vs 91000)
- State boundaries
- International addresses with similar postal codes

- Enter test address
- See which zone(s) match
- View priority resolution

- Lower priority zone selected despite higher priority zone matching
- Postal code pattern syntax error (pattern fails silently)
- State code mismatch (CA vs California)

- Verify priority values (higher number = higher priority)
- Test postal code patterns with regex validator
- Use 2-letter state codes (CA, not California)

- 메서드에 연결된 지역이 없음(모든 곳에서 사용 가능)
- 여러 지역이 일치하지만, 예상치 못한 지역이 우선순위가 높음
- 지역 범위가 의도치 않게 중복됨

- 메서드에 연결된 지역을 검토하세요
- 일치하는 지역의 우선순위를 확인하세요
- 중복을 확인하기 위해 지역 범위 요약을 감사하세요

- **2개의 지역부터 시작하세요** - 국내 및 국제, 필요에 따라 확장하세요
- **우선순위를 신중하게 사용하세요** - 구체적인 지역은 100, 지역은 50, 대체 지역은 1
- **우편 번호 패턴을 철저히 테스트하세요** - 정규식 오류는 조용히 실패하여 지역이 일치하지 않게 됩니다
- **지역 논리를 문서화하세요** - 지역 설명에 노트를 추가하여 범위 의도를 설명하세요
- **과도한 지역은 피하세요** - 너무 많은 지역은 설정을 복잡하게 만듭니다. 복잡한 시나리오에는 배송 프로모션을 사용하세요
- **상태 코드를 사용하세요, 이름은 사용하지 마세요** - "CA"가 아닌 "California", "NY"가 아닌 "New York"
- **대체 지역을 생성하세요** - 모든 국가, 우선순위 1, 최소한 하나의 배송 옵션이 항상 사용 가능하도록 보장합니다
- **지역 성능을 모니터링하세요** - 많은 고객이 "배송 옵션이 없음"을 보고하는 경우 지역 범위를 감사하세요
