---
title: 배송 지역
---

배송 지역은 특정한 배송 비용을 위한 지리적 지역을 정의합니다. 국가, 주/도, 우편번호를 지역으로 그룹화한 후, 특정 지역에 해당하는 배송 방식을 연결하여 정확한 비용을 제어할 수 있습니다. 여러 지역에 해당하는 주소의 경우 우선순위 기반으로 일치시키며(가장 높은 우선순위가 승리), 이 시스템은 복잡한 가격 전략을 가능하게 합니다: 원격 지역에 더 많은 비용을 부과하거나, 국내 배송을 무료로 제공하거나, 특정 지역에 할인된 비용을 제공할 수 있습니다.

다른 지리적 지역에 따라 다른 배송 비용이 필요한 경우에 지역을 사용하세요. 단순한 국내 대 국제 분할에서 복잡한 다중 지역 계층 가격까지 다양한 경우에 적용할 수 있습니다.

## 배송 지역 이해

**지역이란 무엇인가**: 국가, 주/도, 우편번호 패턴으로 정의된 이름이 있는 지리적 지역입니다.

**지역이 어떻게 작동하는가**:
1. 고객이 결제 시 배송 주소를 입력합니다
2. 시스템이 모든 활성 지역을 평가합니다
3. 고객의 주소와 일치하는 지역이 후보가 됩니다
4. 여러 지역이 일치하는 경우, 가장 높은 우선순위 지역이 승리합니다
5. 승리한 지역에 연결된 배송 방식이 표시됩니다
6. 어떤 지역에도 연결되지 않은(또는 일치하는 지역에 연결된) 방식도 표시됩니다

**지역 구성 요소**:
- **이름**: 지역 식별자 (예: "국내", "EU", "원격 지역")
- **국가**: 포함된 국가 코드 목록 (빈 값 = 모든 국가)
- **주/도**: 특정 국가별 주 제한 (선택 사항)
- **우편번호 패턴**: ZIP/우편번호 일치를 위한 정규식 패턴 (선택 사항)
- **우선순위**: 여러 지역이 일치할 때 더 높은 숫자 = 더 높은 우선순위

---

## 지역 일치 논리

지역은 주소 일치를 위해 **점진적 좁히기**를 사용합니다:

### 레벨 1: 국가 일치

**빈 국가 목록** → 모든 국가에 해당하는 지역

**제공된 국가 목록** → 주소 국가가 목록에 포함되어야 함

예시:
```
지역: "국내"
국가: ["US"]
→ 일치: 미국 주소
→ 일치하지 않음: 캐나다, 영국 등
```

### 레벨 2: 주/도 일치

**정의된 주가 없음** → 허용된 국가의 모든 주에 해당하는 지역

**특정 국가에 대한 주가 정의됨** → 주소 주가 일치해야 함

예시:
```
지역: "서부 해안"
국가: ["US"]
주: {"US": ["CA", "OR", "WA"]}
→ 일치: 캘리포니아, 오regon, 워싱턴 주소
→ 일치하지 않음: 뉴욕, 텍사스 등
```

### 레벨 3: 우편번호 일치

**정의된 패턴이 없음** → 허용된 국가/주에 해당하는 모든 우편번호

**패턴이 정의됨** → 주소 우편번호가 적어도 하나의 패턴과 일치해야 함

예시:
```
지역: "로스앤젤레스 메트로"
국가: ["US"]
주: {"US": ["CA"]}
우편 패턴: ["^90[0-9]{3}$", "^91[0-9]{3}$"]
→ 일치: 90001, 91210, 90245
→ 일치하지 않음: 94102 (샌프란시스코)
```

**정규식 패턴 예시**:
- `^90[0-9]{3}$` - 로스앤젤레스 지역 (90000-90999)
- `^[A-Z][0-9][A-Z] [0-9][A-Z][0-9]$` - 캐나다 우편번호 형식 (K1A 0B1)
- `^SW[0-9]{1,2}` - 영국 런던 우편번호의 SW로 시작하는 것

## 우선순위 기반 지역 선택

여러 지역이 주소에 일치하는 경우, **우선순위**가 어떤 지역이 적용되는지를 결정합니다:

**우선순위 작동 방식**:
- 더 높은 숫자 = 더 높은 우선순위
- 주소가 우선순위 100과 50의 지역에 모두 일치하는 경우, 우선순위 100이 승리합니다
- 승리한 지역의 배송 방식만 사용할 수 있습니다

**사용 사례**:

**시나리오 1: 특정 지역이 일반 지역을 대체**
```
지역 A: "원격 알래스카"
  국가: ["US"]
  주: {"US": ["AK"]}
  우선순위: 100

지역 B: "국내 미국"
  국가: ["US"]
  우선순위: 50

주소: 앵커리지, AK
→ 두 지역 모두 일치
→ 우선순위 100이 승리
→ "원격 알래스카" 지역이 적용됨 (더 높은 배송 비용)
```

**시나리오 2: 우편번호가 주를 대체**
```
지역 A: "맨하튼 프리미엄"
  국가: ["US"]
  주: {"US": ["NY"]}
  우편 패턴: ["^100[0-2][0-9]$"]
  우선순위: 100

지역 B: "뉴욕 주"
  국가: ["US"]
  주: {"US": ["NY"]}
  우선순위: 50

주소: 뉴욕, NY 10001
→ 두 지역 모두 일치
→ 우선순위 100이 승리
→ "맨하튼 프리미엄"이 적용됨 (프리미엄 배송 서비스)
```

## 배송 지역 생성

**단계별 워크플로**:

1. **지역으로 이동**
   - 설정 > 배송 > 배송 지역으로 이동
   - "배송 지역 추가"를 클릭합니다

2. **기본 구성**
   - **이름**: 설명적인 식별자 (예: "유럽 연합", "서부 해안", "원격 지역")
   - **우선순위**: 상대적 중요도 설정 (100은 특정, 50은 일반, 1은 대체)
   - **활성**: 활성화/비활성화 전환

3. **지리적 범위 정의**

   **옵션 A: 모든 국가** (국가 목록을 비워둠)
   - 전 세계 모든 주소에 해당하는 지역
   - 기본/대체 지역에 사용

   **옵션 B: 특정 국가**
   - "국가 추가"를 클릭합니다
   - 드롭다운에서 국가 선택 (US, CA, UK 등)
   - 포함할 모든 국가에 반복

   **옵션 C: 특정 주/도**
   - 국가를 추가한 후, 각 국가에 대해 "주 추가"를 클릭합니다
   - 드롭다운에서 주를 선택합니다
   - 예: US → CA, OR, WA (서부 해안)

   **옵션 D: 우편번호 패턴** (고급)
   - 정규식 패턴 입력 (한 줄에 하나씩)
   - 샘플 우편번호로 패턴을 테스트합니다
   - "패턴 검증"을 클릭하여 구문을 확인합니다

4. **배송 방식에 연결**
   - 방법은 방법 편집 시에 연결할 수 있으며 (지역 구성에서 아님)
   - 또는 기존 방법에 지역 연결: 방법 편집 → 배송 지역 → 지역 선택

5. **표시 우선순위 설정**
   - 여러 지역이 일치할 때 더 높은 우선순위 지역이 더 낮은 우선순위를 대체합니다
   - 권장: 특정 지역 (100), 지역별 지역 (50), 기본 지역 (1)

6. **지역 활성화**
   - "활성" 전환 = 예
   - 저장

## 일반적인 지역 설정

### 설정 1: 국내 대 국제

**목표**: 국내 대 모든 다른 국가에 대한 다른 비용.

```
지역 1: "국내"
  국가: [당신의 국가 코드]
  우선순위: 50

지역 2: "국제"
  국가: [빈 값 또는 다른 모든 국가 선택]
  우선순위: 1
```

**배송 방식**:
- "국내 표준" → 국내 지역에 연결
- "국제 배송" → 국제 지역에 연결

---

### 설정 2: 다중 지역 국제

**목표**: 유럽, 북미, 아시아, 나머지 세계에 대한 다른 비용.

```
지역 1: "유럽 연합"
  국가: [AT, BE, BG, CY, CZ, DE, DK, EE, ES, FI, FR, GR, HR, HU, IE, IT, LT, LU, LV, MT, NL, PL, PT, RO, SE, SI, SK]
  우선순위: 100

지역 2: "북미"
  국가: [US, CA, MX]
  우선순위: 100

지역 3: "아시아 태평양"
  국가: [AU, CN, HK, IN, JP, KR, NZ, SG, TH, TW]
  우선순위: 100

지역 4: "나머지 세계"
  국가: [빈 값]
  우선순위: 1
```

**배송 방식**:
- "유럽 배송" → 유럽 연합 지역
- "북미 배송" → 북미 지역
- "아시아 태평양 배송" → 아시아 태평양 지역
- "국제 표준" → 나머지 세계 지역

---

### 설정 3: 원격 지역 추가 비용

**목표**: 국내 지역 내 원격 우편번호에 추가 비용을 부과합니다.

```
지역 1: "원격 국내"
  국가: [US]
  우편 패턴: ["^99[0-9]{3}$", "^96[7-9][0-9]{2}$"]  # 알래스카, 하와이
  우선순위: 100

지역 2: "표준 국내"
  국가: [US]
  우선순위: 50
```

**배송 방식**:
- "원격 배송" → 원격 국내 지역 (더 높은 비용)
- "표준 배송" → 표준 국내 지역

---

### 설정 4: 주별 특정 지역

**목표**: 미국의 각 지역에 대한 다른 비용.

```
지역 1: "서부 해안"
  국가: [US]
  주: {"US": ["CA", "OR", "WA"]}
  우선순위: 100

지역 2: "동부 해안"
  국가: [US]
  주: {"US": ["NY", "NJ", "CT", "MA", "PA"]}
  우선순위: 100

지역 3: "미드웨스트"
  국가: [US]
  주: {"US": ["IL", "IN", "OH", "MI", "WI"]}
  우선순위: 100

지역 4: "남부"
  국가: [US]
  주: {"US": ["TX", "FL", "GA", "NC", "SC"]}
  우선순위: 100

지역 5: "기타 미국 주"
  국가: [US]
  우선순위: 50
```

## 우편번호 패턴 예시

우편번호는 **정규식**(정규 표현식)을 사용하여 패턴 일치를 수행합니다:

### 미국 (ZIP 코드)

**형식**: 5자리 숫자 (예: 90210)

```
캘리포니아 (90000-96199):  ^9[0-6][0-9]{3}$
뉴욕 (10000-14999):    ^1[0-4][0-9]{3}$
텍사스 (75000-79999, 88500-88599):  ^(7[5-9]|885)[0-9]{2}$
알래스카 (99500-99999):      ^99[5-9][0-9]{2}$
```

### 캐나다 (우편번호)

**형식**: A1A 1A1 (문자-숫자-문자 공백 숫자-문자-숫자)

```
모든 캐나다 우편번호:  ^[A-Z][0-9][A-Z] [0-9][A-Z][0-9]$
온타리오 (K, L, M, N, P):    ^[KLMNP][0-9][A-Z] [0-9][A-Z][0-9]$
퀘벡 (G, H, J):           ^[GHJ][0-9][A-Z] [0-9][A-Z][0-9]$
```

### 영국 (우편번호)

**형식**: AA1A 1AA 또는 A1A 1AA

```
런던 (E, EC, N, NW, SE, SW, W, WC):  ^(E|EC|N|NW|SE|SW|W|WC)[0-9]{1,2}
맨체스터 (M):                        ^M[0-9]{1,2}
버밍엄 (B):                        ^B[0-9]{1,2}
```

### 호주 (우편번호)

**형식**: 4자리 숫자 (예: 2000)

```
뉴사우스웨일스 (1000-2999):  ^[12][0-9]{3}$
빅토리아 (3000-3999, 8000-8999):  ^[38][0-9]{3}$
퀸즐랜드 (4000-4999, 9000-9999):  ^[49][0-9]{3}$
```

### 패턴 테스트

**패턴을 저장하기 전에**, 알려진 우편번호로 테스트하세요:

1. 패턴 입력: `^90[0-9]{3}$`
2. 테스트 입력: "90210" → 일치해야 함
3. 테스트 입력: "10001" → 일치하지 않아야 함
4. 테스트 입력: "9021" → 4자리 숫자만 허용되므로 일치하지 않아야 함

복잡한 패턴을 검증하려면 온라인 정규식 테스터 (regex101.com)를 사용하세요.

## 지역 범위 요약

지역은 관리 목록 뷰에서 **범위 요약**을 표시하여 포함된 내용을 보여줍니다:

**예시**:
- "모든 국가" → 국가 제한 없음
- "US, CA, MX" → 3개 국가
- "US (CA, OR, WA)" → 미국에 포함된 3개 주
- "US (90xxx-91xxx)" → 미국에 포함된 우편번호 패턴

**요약 사용 목적**:
- 열람 없이 지역 범위를 빠르게 확인
- 범위 중복 또는 공백을 발견
- 지역 구성 요약을 통해 빠르게 감사

## 배송 방식에 지역 연결

지역과 방식은 **다대다 관계**를 가집니다:

**방식 측에서 연결** (권장):
1. 배송 방식 편집
2. "배송 지역" 섹션으로 스크롤
3. 적용 가능한 지역 선택 (다중 선택)
4. 방식 저장

**지역 측에서 연결**:
- 지역은 직접적으로 방식에 연결되지 않습니다
- 연결은 항상 방식 구성에서 수행됩니다

**방식-지역 행동**:

**연결된 지역이 없음** → 방식은 모든 주소에 사용 가능

**연결된 지역이 있음** → 고객 주소가 적어도 하나의 연결된 지역과 일치해야 방식이 사용 가능

**예시**:
```
방식: "국내 표준"
연결된 지역: ["국내 미국"]
→ 미국 주소에만 표시됨

방식: "국제 익스프레스"
연결된 지역: ["EU", "아시아 태평양", "나머지 세계"]
→ 모든 미국 이외 주소에 표시됨
```

## 지역 일치 테스트

실제 운영 전에 지역 구성 설정을 테스트하세요:

1. **테스트 주문 생성**
   - 다른 지역에 있는 주소를 사용하세요
   - 올바른 지역 일치를 확인하세요

2. **우선순위 해결 확인**
   - 여러 지역에 일치하는 주소를 사용하세요
   - 가장 높은 우선순위 지역이 승리하는지 확인하세요
   - 예상되는 배송 방식이 표시되는지 확인하세요

3. **경계 사례 테스트**
   - 경계 우편번호 (예: 90999 대 91000)
   - 주 경계

## 문제 해결

**문제 1: 결제 시 배송 방식이 없음**

**원인**:
- 고객 주소가 어떤 지역에도 일치하지 않음
- 모든 방식이 일치하지 않는 지역에 연결됨
- 지역 제한이 없는 방식이 존재하지 않음

**해결 방법**:
- 대체 지역 생성 (모든 국가, 우선순위 1)
- 또는 최소한 하나의 방식에서 지역 제한을 제거하세요
- 지역 국가/주/우편 패턴 확인

---

**문제 2: 잘못된 지역 일치**

**원인**:
- 더 높은 우선순위 지역이 일치하지만 더 낮은 우선순위 지역이 선택됨
- 우편번호 패턴 구문 오류 (패턴이 조용히 실패)
- 주 코드 불일치 (CA 대 캘리포니아)

**해결 방법**:
- 우선순위 값 확인 (더 높은 숫자 = 더 높은 우선순위)
- 정규식 검증기로 우편번호 패턴 테스트
- 2자리 주 코드 사용 (CA, 아닌 캘리포니아, NY, 아닌 뉴욕)

---

**문제 3: 예상치 못한 방식 표시**

**원인**:
- 방식이 연결된 지역이 없음 (모든 곳에서 사용 가능)
- 여러 지역이 일치하고 예상치 못한 지역이 더 높은 우선순위를 가짐
- 지역 범위가 의도치 않게 중복됨

**해결 방법**:
- 방식의 연결된 지역 확인
- 일치하는 지역의 우선순위 확인
- 지역 범위 요약을 통해 중복 확인

- **지역 논리를 문서화하세요** - 지역 설명에 노트를 추가하여 범위 목적을 설명

- **주 코드를 사용하세요, 이름은 사용하지 마세요** - "CA" 대신 "캘리포니아", "NY" 대신 "뉴욕"

- **지역 성능을 모니터링하세요** - 많은 고객이 "배송 불가"를 보는 경우, 지역 범위를 감사하세요

- **실제 주소로 테스트하세요** - 테스트 시 고객의 실제 주소를 사용하세요, 가상 주소는 사용하지 마세요
