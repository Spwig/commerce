title: Налоговая настройка
---

Ставки налогов определяют налог с продаж, НДС и другие налоги на потребление, применяемые при оформлении заказа на основе местоположения клиента и типа продукта — настройте ставки на уровне страны/штата/города с необязательными исключениями для категорий продуктов. Spwig поддерживает составной налог (налог с налога), выбор ставок на основе приоритета и группы предустановленных налогов для быстрой настройки региональных систем налогообложения (НДС ЕС, налог с продаж США). Ставки могут освобождать определенные типы продуктов (еда, книги, цифровые товары) или категории для соблюдения местных налоговых законов.

Используйте настройку налогов, чтобы обеспечить соответствие требованиям сбора налогов в вашем регионе продаж.

## Настройка ставок налогов

Каждая ставка налога определяет:

**Географический охват**:
- Страна (обязательно)
- Штат/провинция (необязательно)
- Город (необязательно)
- Шаблон почтового индекса (необязательно, регулярное выражение)

**Детали ставки**:
- **Ставка налога**: Процент (например, 8,5%)
- **Название**: Отображаемое название (например, "Налог с продаж Калифорнии")
- **Приоритет**: Высший приоритет выигрывает, когда сопоставляются несколько ставок
- **Активный**: Переключатель без удаления

**Исключения**:
- **Исключенные типы продуктов**: Цифровые товары, физические товары, услуги
- **Исключенные категории**: Определенные категории продуктов (Пищевые продукты, Книги, Медицинские)

**Составной налог**:
- **Составной налог**: Примените эту ставку поверх предыдущих налогов (налог с налога)
- Пример: PST Квебека применяется к GST

## Распространенные сценарии налогообложения

### Налог с продаж США (на уровне штата)

```
Название: Налог с продаж Калифорнии
Страна: США
Штат: CA
Ставка: 7,25%
Приоритет: 50
```

### НДС ЕС (на уровне страны)

```
Название: НДС Великобритании
Страна: GB
Ставка: 20%
Приоритет: 50

Название: НДС Германии
Страна: DE
Ставка: 19%
Приоритет: 50
```

### GST/PST Канады (составной)

```
Ставка 1: Федеральный GST
Страна: CA
Ставка: 5%
Приоритет: 100
Составной: Нет

Ставка 2: PST Квебека
Страна: CA
Штат: QC
Ставка: 9,975%
Приоритет: 50
Составной: Да  (применяется к подытогу + GST)
```

### Налог на уровне города

```
Название: Налог с продаж Сиэтла
Страна: США
Штат: WA
Город: Сиэтл
Ставка: 10,1%
Приоритет: 100
```

## Исключения из налогообложения

### Исключения по типам продуктов

Исключить целые типы продуктов:

- **Цифровые товары**: Программное обеспечение, электронные книги, музыка
- **Физические товары**: Осязаемые продукты
- **Услуги**: Консультации, установка

Пример: НДС ЕС не применяется к цифровым товарам для потребителей (в некоторых случаях)

### Исключения по категориям продуктов

Исключить определенные категории продуктов:

- Продукты питания и продукты первой необходимости (часто исключаются или применяется пониженная ставка)
- Книги и учебные материалы
- Медицинские изделия и лекарства
- Одежда (в некоторых юрисдикциях)

Настройка:
```
Название: Налог с продаж Калифорнии
Ставка: 7,25%
Исключенные категории: ["Продукты питания и напитки", "Препараты"]
```

## Предустановленные группы налогов

Быстрый способ загрузить распространенные настройки налогов:

**Предустановленный налог с продаж США**:
- Все 50 штатов + DC
- Ставки на уровне штата
- Автоматически обновляется при изменении ставок

**Предустановленный НДС ЕС**:
- Все 27 стран-членов ЕС
- Стандартные ставки НДС
- Логика обратного взимания для B2B

**Как использовать предустановки**:
1. Настройки > Корзина > Предустановки налогов
2. Выберите группу предустановок (например, "Налог с продаж США 2026")
3. Нажмите "Загрузить предустановку"
4. Ставки загружаются автоматически
5. Настройте их по мере необходимости

---

## Решение конфликтов приоритетов

Когда несколько ставок совпадают, выигрывает ставка с наивысшим приоритетом:

Пример:
```
Клиент в Сиэтле, штат Вашингтон:

Ставка A: Федеральная США (Приоритет 1) - 0%
Ставка B: Штат Вашингтон (Приоритет 50) - 6,5%
Ставка C: Город Сиэтл (Приоритет 100) - 3,6%

Результат: применяется ставка Сиэтла (итого 10,1%)
```

## Настройки отображения налогов

Настройте в Настройках > Корзина > Настройки налогов:

- **Цены включают налог**: Отображать цены с учетом налога (стиль ЕС)
- **Отображать налог отдельно**: Показывать налог как отдельную позицию (стиль США)
- **Округление налога**: По каждому товару или по заказу
- **Название налога**: Настройте метку ("НДС", "Налог с продаж", "GST")

---

## Тестирование настройки налогов

Перед запуском:

1. Создайте тестовые заказы из разных юрисдикций
2. Проверьте, применяется ли правильная ставка налога
3. Проверьте, работают ли исключения для исключенных категорий
4. Проверьте расчеты составного налога
5. Проверьте элементы налога в счетах-фактурах

---

## Замечания по соблюдению норм

- **США**: Правила Nexus требуют сбора налогов в штатах, где у вас есть физическое присутствие или экономический Nexus
- **ЕС**: Компании, зарегистрированные в НДС, должны собирать НДС у клиентов из ЕС
- **Канада**: GST/HST/PST различаются в зависимости от провинции
- **Обратитесь к налоговому специалисту**: Налоговые законы часто меняются, уточните текущие требования

---

## Советы

- **Используйте предустановки налогов** — быстрее, чем вводить вручную, автоматически обновляются
- **Следите за порогами Nexus** — отслеживайте продажи по штатам для экономического Nexus в США
- **Установите приоритет правильно** — Город > Штат > Страна
- **Проверьте составной налог** — убедитесь, что расчеты соответствуют ожидаемым суммам
- **Обновляйте ежегодно** — ставки налогов меняются, проверяйте каждый январь
- **Документируйте исключения** — сохраняйте записи о причинах исключений категорий
- **Используйте описательные названия** — "Налог с продаж Калифорнии 2026" лучше, чем "Налог 1"
- **Включите налог по умолчанию** — безопаснее, чем забыть применить налог
