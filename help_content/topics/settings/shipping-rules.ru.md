---
title: Правила доставки
---

Правила доставки применяют условные корректировки стоимости к методам доставки на основе содержимого корзины, атрибутов клиента и зон доставки — автоматически предлагают бесплатную доставку при заказе свыше 50 долларов, добавляют надбавки за удаленные районы или предоставляют скидки на доставку для клиентов VIP-категории. Правила используют выполнение на основе приоритета (высший приоритет сначала) с необязательными флагами остановки для предотвращения дальнейшей обработки. Каждое правило оценивает несколько условий (сумма корзины, вес, зоны, товары, группы клиентов) и выполняет одну из 6 типов корректировок, когда все условия совпадают.

Используйте правила доставки, когда вам нужны динамические стоимости доставки, которые меняются в зависимости от контекста заказа, а не только статические ставки из методов доставки.

## Типы правил доставки

Правила доставки применяют 6 типов корректировок стоимости:

### Процентная скидка

**Что это делает**: Уменьшает стоимость доставки на определенный процент (например, 25% скидка).

**Формула**: `new_cost = base_cost × (1 - percent/100)`

**Пример**:
```
Базовая стоимость: $20
Скидка: 25%
Результат: $15
```

**Сценарии использования**:
- Скидка на доставку для клиентов VIP (20% скидка на все доставки)
- Сезонные акции (15% скидка на доставку в декабре)
- Скидка на доставку для крупных заказов (10% скидка на доставку при заказе 5 и более товаров)

---

### Фиксированная скидка

**Что это делает**: Вычитает фиксированную сумму из стоимости доставки.

**Формула**: `new_cost = base_cost - amount` (минимум $0)

**Пример**:
```
Базовая стоимость: $15
Скидка: $5
Результат: $10
```

**Сценарии использования**:
- Бонус для новых клиентов ($5 скидка на доставку первого заказа)
- Вознаграждение за регистрацию в рассылке ($3 скидка на доставку)
- Преимущество программы лояльности ($10 скидка на доставку в месяц)

---

### Установить стоимость

**Что это делает**: Заменяет стоимость доставки на конкретную сумму.

**Формула**: `new_cost = fixed_amount`

**Пример**:
```
Базовая стоимость: $25
Установить: $9.99
Результат: $9.99
```

**Сценарии использования**:
- Акция с фиксированной стоимостью доставки (плоская стоимость доставки $5 для всех заказов сегодня)
- Специфичная доставка по категориям (книги всегда доставляются за $3.99)
- Акция по времени (ограничение доставки до $9.99 на этой неделе)

---

### Бесплатная доставка

**Что это делает**: Устанавливает стоимость доставки в $0.

**Формула**: `new_cost = $0`

**Пример**:
```
Базовая стоимость: $18
Применяется правило
Результат: $0
```

**Сценарии использования**:
- Бесплатная доставка при заказе свыше $50
- Бесплатная доставка для определенных товаров (рекламные товары)
- Бесплатная доставка для клиентов VIP
- Бесплатная доставка для заказов с 3 и более товарами

---

### Надбавка (фиксированная)

**Что это делает**: Добавляет фиксированную сумму к стоимости доставки.

**Формула**: `new_cost = base_cost + amount`

**Пример**:
```
Базовая стоимость: $12
Надбавка: $5
Результат: $17
```

**Сценарии использования**:
- Плата за доставку в удаленные районы
- Плата за обработку крупногабаритных товаров
- Надбавка за доставку в субботу
- Плата за упаковку хрупких товаров

---

### Надбавка (процентная)

**Что это делает**: Увеличивает стоимость доставки на определенный процент.

**Формула**: `new_cost = base_cost × (1 + percent/100)`

**Пример**:
```
Базовая стоимость: $20
Надбавка: 15%
Результат: $23
```

**Сценарии использования**:
- Надбавка в пиковые сезоны (20% в праздничные дни)
- Премиум-доставка с ускорением (надбавка 50%)
- Надбавка за топливо (переменная в зависимости от текущих ставок)

## Условия правила

Правила оценивают **Все условия должны быть выполнены**, чтобы правило применялось:

### Временная действительность

- **Дата начала**: Правило активно только после этой даты
- **Дата окончания**: Правило активно только до этой даты
- **Сценарий использования**: Сезонные акции, ограниченные по времени предложения

**Пример**: Бесплатная доставка только в выходные Black Friday
```
Дата начала: 2026-11-27 00:00
Дата окончания: 2026-11-30 23:59
```

---

### Диапазон суммы корзины

- **Минимальная сумма корзины**: Общая сумма корзины должна быть ≥ сумма
- **Максимальная сумма корзины**: Общая сумма корзины должна быть ≤ сумма
- **Сценарий использования**: Пороги бесплатной доставки, поэтапные скидки

**Пример**: Бесплатная доставка для заказов от $50 до $200
```
Минимум: $50
Максимум: $200
```

---

### Диапазон веса корзины

- **Минимальный вес**: Общий вес корзины должен быть ≥ сумма
- **Максимальный вес**: Общий вес корзины должен быть ≤ сумма
- **Сценарий использования**: Скидки на легкие посылки, надбавки за тяжелые товары

**Пример**: Надбавка в $5 для заказов свыше 20 кг
```
Минимальный вес: 20 кг
Максимальный вес: null (без ограничений)
```

---

### Диапазон количества товаров

- **Минимальное количество товаров**: В корзине должно быть ≥ количество товаров
- **Максимальное количество товаров**: В корзине должно быть ≤ количество товаров
- **Сценарий использования**: Скидки на крупные заказы, плата за одиночные товары

**Пример**: Бесплатная доставка при заказе 5 и более товаров
```
Минимальное количество товаров: 5
Максимальное количество товаров: null
```

---

### Зона доставки

- **Зоны**: Правило применяется только если адрес клиента совпадает с хотя бы одной из выбранных зон
- **Пустой выбор**: Правило применяется ко всем зонам
- **Сценарий использования**: Надбавки или скидки по зонам

**Пример**: Бесплатная доставка только для зоны Domestic
```
Зоны: ["Domestic USA"]
```

---

### Метод доставки

- **Методы**: Правило применяется только к определенным методам доставки
- **Пустой выбор**: Правило применяется ко всем методам
- **Сценарий использования**: Акции по методам доставки

**Пример**: Скидка 25% на доставку Express
```
Методы: ["Express Delivery"]
```

---

### Требования к товарам

**Требуется товары**: В корзине должно быть как минимум один из этих товаров

**Требуется категории**: В корзине должно быть как минимум один товар из этих категорий

**Сценарий использования**: Бесплатная доставка для определенных товаров, рекламные наборы

**Пример**: Бесплатная доставка, если в корзине есть товар "Promotion Item A"
```
Требуется товары: [Product ID 123]
```

---

### Исключения товаров

**Исключить товары**: Правило не применяется, если в корзине есть любой из этих товаров

**Исключить категории**: Правило не применяется, если в корзине есть товары из этих категорий

**Сценарий использования**: Исключение тяжелых/крупногабаритных товаров из бесплатной доставки

**Пример**: Бесплатная доставка, кроме категории мебели
```
Исключить категории: [Furniture]
```

---

### Группы клиентов

- **Группы клиентов**: Правило применяется только для клиентов в выбранных группах (VIP, оптовые и т.д.)
- **Пустой выбор**: Правило применяется ко всем группам клиентов
- **Сценарий использования**: Преимущества VIP, скидки для оптовых покупателей

**Пример**: Скидка на доставку 15% для клиентов VIP
```
Группы клиентов: ["VIP"]
```

---

### Первый заказ клиента

- **Первый заказ клиента**: Переключатель для ограничения правила только для клиентов без предыдущих заказов
- **Сценарий использования**: Предложение для новых клиентов

**Пример**: Скидка $5 на доставку первого заказа
```
Первый заказ клиента: Да
```

## Приоритет правила и выполнение

Правила выполняются в **порядке приоритета** (высшее число = более раннее выполнение):

### Механика приоритета

**Пример выполнения**:
```
Правило A (Приоритет 100): Бесплатная доставка, если сумма корзины > $50
Правило B (Приоритет 50): 10% скидка на все доставки
Правило C (Приоритет 1): $2 надбавка за удаленные зоны

Корзина: $60, удаленная зона
Базовая стоимость доставки: $15

Шаг 1: Правило A оценивается (Приоритет 100)
  Сумма корзины > $50? ДА
  Применить: Установить стоимость $0
  Стоимость теперь: $0

Шаг 2: Правило B оценивается (Приоритет 50)
  Применить 10% скидку к $0
  Стоимость теперь: $0 (все равно бесплатная)

Шаг 3: Правило C оценивается (Приоритет 1)
  Добавить $2 надбавку к $0
  Стоимость теперь: $2

Итоговая стоимость: $2
```

**Флаг остановки дальнейших правил**:

Если правило A имеет `stop_further_rules = True`:
```
Правило A (Приоритет 100, stop_further_rules=True): Бесплатная доставка, если сумма корзины > $50
Правило B (Приоритет 50): 10% скидка
Правило C (Приоритет 1): $2 надбавка за удаленные зоны

Корзина: $60
Базовая стоимость: $15

Шаг 1: Применяется правило A, устанавливает стоимость $0
        stop_further_rules = True → ОСТАНОВИТЬ

Итоговая стоимость: $0 (Правила B и C никогда не выполняются)
```

## Создание правил доставки

**Пошаговая рабочая процедура**:

1. **Перейти к правилам**
   - Настройки > Доставка > Правила доставки
   - Нажать "Добавить правило доставки"

2. **Базовая настройка**
   - **Название**: Внутренний идентификатор (например, "Бесплатная доставка при заказе свыше $50")
   - **Описание**: Необязательные примечания (не отображаются клиентам)
   - **Активно**: Переключатель для включения/отключения
   - **Приоритет**: Установить порядок выполнения (100 для высокого приоритета, 1 для низкого)

3. **Выбрать тип правила**
   - Выбрать тип корректировки (скидка %, фиксированная скидка, установить стоимость, бесплатная, надбавка %, фиксированная надбавка)
   - Ввести сумму или процент

4. **Установить флаг остановки** (Необязательно)
   - Отметить "Остановить дальнейшие правила", если это правило должно предотвратить выполнение правил с низким приоритетом
   - Использовать для окончательных/абсолютных правил (например, бесплатная доставка не должна иметь надбавок после нее)

5. **Определить условия** (Необязательно - оставить пустым для "всегда применять")
   - Временная действительность: даты начала/окончания
   - Сумма корзины: минимум/максимум
   - Вес корзины: минимум/максимум
   - Количество товаров: минимум/максимум
   - Зоны: выбрать применимые зоны
   - Методы: выбрать применимые методы
   - Товары: требуемые или исключенные
   - Клиент: группы или только для новых клиентов

6. **Сохранить правило**
   - Нажать Сохранить
   - Правило становится активным сразу (если переключатель "Активно" включен)

## Стратегии комбинирования правил

### Стратегия 1: Сложение скидок

**Разрешить нескольким скидкам, чтобы они накапливались**:
```
Правило A (Приоритет 100): 10% скидка для VIP → stop_further_rules=Нет
Правило B (Приоритет 50): 15% скидка на заказы >$100 → stop_further_rules=Нет

Клиент VIP с заказом $120:
Базовая стоимость: $15
После правила A: $13.50 (10% скидка)
После правила B: $11.48 (15% скидка от $13.50)
```

### Стратегия 2: Исключительные правила

**Применяется только одно правило** (самое высокое приоритет):
```
Правило A (Приоритет 100): Бесплатная доставка >$50 → stop_further_rules=Да
Правило B (Приоритет 50): 20% скидка на все доставки → stop_further_rules=Да

Корзина > $50:
Правило A применяется → Бесплатная доставка → ОСТАНОВИТЬ
Правило B никогда не выполняется
```

### Стратегия 3: Условные надбавки

**Скидки сначала, надбавки в конце**:
```
Правило A (Приоритет 100): Бесплатная доставка >$75
Правило B (Приоритет 75): 15% скидка для VIP
Правило C (Приоритет 50): 10% общая скидка
Правило D (Приоритет 25): $5 надбавка за удаленные районы
Правило E (Приоритет 1): 10% надбавка за топливо

Заказ: $80, удаленная зона, клиент VIP
Базовая стоимость: $20
A: $80 > $75 → Бесплатная ($0)
B: VIP → 15% скидка от $0 = $0
C: 10% скидка от $0 = $0
D: Удаленная зона +$5 = $5
E: Топливо +10% от $5 = $5.50

Итог: $5.50 (не бесплатная из-за надбавок)
```

**Чтобы избежать этого, используйте stop_further_rules=Да**:
```
Правило A (Приоритет 100, stop=Да): Бесплатная доставка >$75

Тот же заказ:
A: $80 > $75 → Бесплатная ($0) → ОСТАНОВИТЬ
Итог: $0 (на самом деле бесплатная)
```

## Тестирование правил доставки

**Перед запуском**:

1. **Создать тестовые корзины**
   - Корзина A: $25 (ниже порога)
   - Корзина B: $55 (выше порога)
   - Корзина C: $200 + удаленная зона
   - Корзина D: клиент VIP

2. **Тестировать каждое правило**
   - Перейти к оформлению заказа
   - Проверить, отображается ли правильная стоимость доставки
   - Проверить выполнение правил

3. **Тестировать разрешение приоритета**
   - Несколько совпадающих правил
   - Проверить, что правило с высшим приоритетом выполняется первым
   - Проверить поведение флага stop_further_rules

4. **Тестировать крайние случаи**
   - Сумма корзины точно на пороге
   - Несколько совпадающих условий
   - Противоречивые правила

## Устранение неполадок

**Проблема 1: Правило не применяется**

**Причины**:
- Правило не активно
- Не выполнено одно или несколько условий
- Правило с более высоким приоритетом установило stop_further_rules=Yes
- Временная действительность вне текущей даты

**Решение**: Проверьте все условия, проверьте приоритет, убедитесь, что правило активно.

---

**Проблема 2: Неожиданная сумма скидки**

**Причины**:
- Несколько правил, которые накапливаются
- Процент применяется к уже скиданной стоимости
- Неправильный приоритет правила

**Решение**: Проверьте порядок приоритета, проверьте флаги stop_further_rules, вручную проверьте выполнение.

---

**Проблема 3: Бесплатная доставка не работает**

**Причины**:
- Надбавка с низким приоритетом добавляет стоимость после правила бесплатной доставки
- Сумма корзины не достигает минимального порога
- В корзине есть исключенные товары

**Решение**: Используйте stop_further_rules=Yes для правила бесплатной доставки, проверьте условия, проверьте исключения.

## Советы

- **Используйте высокий приоритет для бесплатной доставки** - Приоритет 100 гарантирует, что она выполняется до других корректировок
- **Установите stop_further_rules для абсолютных правил** - Бесплатная доставка должна остановить дальнейшую обработку
- **Тестируйте комбинации правил** - Несколько правил могут неожиданно взаимодействовать
- **Используйте описательные названия** - "VIP 20% скидка (приоритет 75)" лучше, чем "Правило 3"
- **Документируйте сложную логику** - Добавьте примечания в поле описания
- **Начните с простых правил** - Постепенно добавляйте сложность
- **Следите за эффективностью правил** - Проверьте, используются ли правила или вызывают ли они путаницу
- **Избегайте излишнего количества правил** - Слишком много правил замедляют оформление заказа, используйте 5-10 максимум
- **Используйте зоны для географии** - Лучше, чем несколько похожих правил на страну
- **Комбинируйте с методами** - Правила + Методы работают вместе для сложной ценовой политики
- **Установите четкие временные окна** - Всегда включайте даты окончания для акций
- **Тестируйте крайние случаи** - Точно $50, точно 5 товаров и т.д.
