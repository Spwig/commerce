การจัดการอุปกรณ์อ่านบัตรติดตามอุปกรณ์ฮาร์ดแวร์การชำระเงินทางกายภาพ มอบหมายให้กับเครื่อง POS และตรวจสอบสถานะการดำเนินการของพวกเขา แต่ละอุปกรณ์อ่านบัตรแสดงฮาร์ดแวร์จริง (Stripe S700, WisePOS E หรือ P400) ที่ลงทะเบียนกับผู้ให้บริการชำระเงิน อุปกรณ์อ่านบัตรมีความสัมพันธ์แบบตัวต่อตัวกับเครื่องปลายทาง—แต่ละเครื่องมีอุปกรณ์อ่านบัตรเฉพาะของตนเอง ตรวจสอบสถานะอุปกรณ์อ่านบัตร (ออนไลน์ ไม่ออนไลน์ กำลังทำงาน) แบบเรียลไทม์ ปรับแต่งหน้าจอเริ่มต้นด้วยแบรนด์ของคุณ และแก้ไขปัญหาการเชื่อมต่อได้ก่อนที่จะส่งผลกระทบต่อประสบการณ์การชำระเงินของลูกค้า

ใช้การจัดการอุปกรณ์อ่านบัตรเพื่อให้แน่ใจว่าฮาร์ดแวร์การชำระเงินได้รับการตั้งค่า มอบหมาย และทำงานได้ทุกสาขา

## การทำความเข้าใจอุปกรณ์อ่านบัตร

อุปกรณ์อ่านบัตรคืออุปกรณ์ฮาร์ดแวร์ที่ประมวลผลการชำระเงินด้วยบัตรเครดิตและบัตรเดบิต:

**องค์ประกอบฮาร์ดแวร์**:
- ช่องสำหรับบัตร EMV chip
- แอนเทนา NFC (การชำระเงินแบบไม่สัมผัส/แตะเพื่อชำระเงิน)
- อุปกรณ์อ่านแถบแม่เหล็ก (แบบเก่า ใช้ไม่มากนัก)
- หน้าจอแสดงผล (แสดงจำนวนเงิน ขอ PIN หรือลายเซ็น)
- ความเชื่อมต่อเครือข่าย (Wi-Fi หรือ Ethernet ขึ้นอยู่กับรุ่น)

**การผสานรวมซอฟต์แวร์**:
- อุปกรณ์อ่านบัตรเชื่อมต่อกับ Stripe Terminal API (บนคลาวด์ ไม่ใช่การเชื่อมต่อโดยตรงกับอุปกรณ์ POS)
- เครื่องปลายทาง POS ขอชำระเงินผ่าน API
- Stripe ส่งคำขอไปยังอุปกรณ์อ่านบัตรที่ลงทะเบียนไว้
- อุปกรณ์อ่านบัตรประมวลผลบัตรและส่งผลลัพธ์กลับไปยัง POS
- ไม่จำเป็นต้องมีการเชื่อมต่อ USB/Bluetooth ระหว่าง POS และอุปกรณ์อ่านบัตร

**อุปกรณ์อ่านบัตรหนึ่งเครื่องต่อเครื่องปลายทางหนึ่งเครื่อง**:
- แต่ละเครื่องปลายทางควรจะมีอุปกรณ์อ่านบัตรที่มอบหมายไว้เพียงเครื่องเดียว
- ความสัมพันธ์แบบตัวต่อตัวช่วยให้การรับผิดชอบชัดเจนและแก้ไขปัญหาง่ายขึ้น
- เครื่องปลายทางหลายเครื่องไม่สามารถใช้อุปกรณ์อ่านบัตรเครื่องเดียวกันได้ (จะเกิดความขัดแย้ง)

## ประเภทของอุปกรณ์อ่านบัตร

Spwig POS รองรับอุปกรณ์อ่านบัตร Stripe Terminal:

**BBPOS WisePOS E** (`bbpos_wisepos_e`):
- อุปกรณ์ปลายทาง Android แบบครบวงจรพร้อมหน้าจอสัมผัสขนาด 5"
- ตัวพิมพ์แบบความร้อนในตัว (ใบเสร็จ)
- เหมาะสำหรับ: การชำระเงินแบบครบวงจรสำหรับร้านค้า ร้านอาหาร (แสดงการขอคำแนะนำการให้ทิปบนหน้าจอสี)
- ความเชื่อมต่อ: Wi-Fi เท่านั้น
- หน้าจอเริ่มต้น: สีเต็มรูปแบบ 480×800 แนวตั้ง

**อุปกรณ์อ่านบัตร Stripe S700** (`stripe_s700`):
- อุปกรณ์อ่านบัตรแบบตั้งโต๊ะพร้อมหน้าจอ LCD ขาวดำ
- ออกแบบกะทัดรัด ทนน้ำ
- เหมาะสำหรับ: ร้านค้ามาตรฐาน โต๊ะชำระเงินแบบกะทัดรัด
- ความเชื่อมต่อ: Wi-Fi หรือ Ethernet
- หน้าจอเริ่มต้น: ขาวดำ 480×800 แนวตั้ง

**Verifone P400** (`verifone_p400`):
- อุปกรณ์อ่านบัตรแบบตั้งโต๊ะรุ่นเก่า (รุ่นเก่า)
- ยังคงได้รับการสนับสนุน แต่ไม่แนะนำให้ใช้ในระบบใหม่
- เหมาะสำหรับ: การติดตั้งที่มีอยู่แล้ว (อย่าแทนที่ฮาร์ดแวร์ที่ทำงานได้)
- ความเชื่อมต่อ: Wi-Fi หรือ Ethernet
- หน้าจอเริ่มต้น: ขาวดำ 480×800 แนวตั้ง

**ความเข้ากันได้ในอนาคต**:
- อาจมีรุ่นอุปกรณ์อ่านบัตรเพิ่มเติมเมื่อ Stripe Terminal ขยายการให้บริการฮาร์ดแวร์
- กล่องเลือกประเภทอุปกรณ์อ่านบัตรจะอัปเดตอัตโนมัติจากความสามารถของผู้ให้บริการ

## ขั้นตอนการลงทะเบียนอุปกรณ์อ่านบัตร

**ขั้นตอนที่ 1: ซื้อและรับฮาร์ดแวร์**
- ซื้ออุปกรณ์อ่านบัตรจาก Stripe (stripe.com/terminal) หรือผู้ขายที่ได้รับอนุญาต
- ถอดกล่องและเปิดอุปกรณ์อ่านบัตร
- เชื่อมต่อกับเครือข่าย Wi-Fi (ทำตามขั้นตอนตั้งค่าบนหน้าจออุปกรณ์อ่านบัตร)

**ขั้นตอนที่ 2: ลงทะเบียนในแดชบอร์ด Stripe**
- ไปที่ **Stripe Dashboard > Terminal > Readers**
- คลิก **Register New Reader**
- ทำตามขั้นตอนการจับคู่บนหน้าจอ (อุปกรณ์อ่านบัตรแสดงรหัสการลงทะเบียน)
- มอบหมายอุปกรณ์อ่านบัตรให้กับตำแหน่ง Stripe (ต้องตรงกับตำแหน่งในการตั้งค่าผู้ให้บริการชำระเงิน)
- บันทึก **Reader ID** (ดูเหมือน `tmr_ABC123...`)

**ขั้นตอนที่ 3: ส 同步到 Spwig (自动)**
- Spwig 会自动发现注册到您的 Stripe 位置的读卡器
- 每30分钟执行一次后台任务同步
- 新的读卡器将在30分钟内出现在 **POS > Card Readers** 列表中

**ขั้นตอนที่ 4: มอบหมายให้กับเครื่องปลายทาง (ทำได้ด้วยตนเอง)**
- ไปที่ **POS > Card Readers**
- หาอุปกรณ์อ่านบัตรที่ถูกค้นพบใหม่ในรายการ
- คลิกเพื่อแก้ไข
- เลือก **Terminal** เพื่อมอบหมายอุปกรณ์อ่านบัตร
- บันทึก

**ขั้นตอนที่ 5: ทดสอบการชำระเงิน**
- ที่เครื่องปลายทาง ประมวลผลธุรกรรมทดสอบ
- เลือกวิธีการชำระเงินด้วยบัตร
- POS ควรค้นพบอุปกรณ์อ่านบัตรที่มอบหมายไว้
- ใช้บัตรทดสอบ Stripe (4242 4242 4242 4242) เพื่อทำให้การทดสอบสำเร็จ
- ตรวจสอบว่าการชำระเงินสำเร็จอย่างสมบูรณ์

หากอุปกรณ์อ่านบัตรไม่ปรากฏขึ้นระหว่างการทดสอบ ให้ตรวจสอบการมอบหมายเครื่องปลายทางและสถานะของอุปกรณ์อ่านบัตร

## การตรวจสอบสถานะอุปกรณ์อ่านบัตร

อุปกรณ์อ่านบัตรรายงานสถานะไปยัง Stripe Terminal API ซึ่ง Spwig จะส 同步 every 5 minutes:

**ออนไลน์** (สีเขียว) - อุปกรณ์อ่านบัตรเปิดใช้งานอยู่ เชื่อมต่อกับเครือข่าย และพร้อมที่จะรับชำระเงิน

**ไม่ออนไลน์** (สีแดง) - อุปกรณ์อ่านบัตรปิดอยู่ ไม่ได้เชื่อมต่อกับเครือข่าย หรือไม่สามารถเข้าถึงได้

**กำลังทำงาน** (สีเหลือง) - อุปกรณ์อ่านบัตรกำลังประมวลผลธุรกรรมการชำระเงินอยู่

**Last Seen** - เวลาที่อุปกรณ์อ่านบัตรเช็คอินล่าสุดกับ API ของ Stripe
- อัปเดตทุก ~2 นาทีเมื่ออุปกรณ์อ่านบัตรออนไลน์
- มีประโยชน์ในการวินิจฉัยปัญหาการเชื่อมต่อ ("อุปกรณ์อ่านบัตรไม่ออนไลน์เมื่อวานนี้" = ปัญหาด้านพลังงานหรือเครือข่ายในช่วงเวลาทำการ)

**สถานการณ์การใช้งานสถานะ**:
- **การตรวจสอบก่อนเปิดร้าน**: ตรวจสอบว่าอุปกรณ์อ่านบัตรทั้งหมดในร้านออนไลน์ก่อนเปิดประตู
- **แก้ไขปัญหา**: "เครื่อง 3 ไม่ยอมรับบัตร" → ตรวจสอบสถานะอุปกรณ์อ่านบัตร → แสดงว่าไม่ออนไลน์ → ตรวจสอบพลังงาน/เครือข่าย
- **ตรวจสอบบัญชี**: "มีการชำระเงินที่เครื่องปลายทาง 5 เมื่อวานหรือไม่?" → ตรวจสอบเวลาที่เห็นล่าสุด

## การมอบหมายเครื่องปลายทาง

อุปกรณ์อ่านบัตรใช้ความสัมพันธ์แบบตัวต่อตัวกับเครื่องปลายทาง:

**เหตุผลที่การมอบหมายมีความสำคัญ**:
- ระหว่างการชำระเงิน POS ต้องทราบว่าอุปกรณ์อ่านบัตรใดที่ต้องสื่อสาร
- การแบ่งปันอุปกรณ์อ่านบัตรเดียวกันระหว่างเครื่องปลายทางหลายเครื่องจะก่อให้เกิดความขัดแย้ง (พนักงานแคชเชียร์สองคนไม่สามารถใช้อุปกรณ์อ่านบัตรเครื่องเดียวกันพร้อมกันได้)
- อุปกรณ์อ่านบัตรที่ไม่มีการมอบหมายจะไม่ได้รับการใช้งาน (ฮาร์ดแวร์ที่ไม่มีการใช้งาน)

**กฎการมอบหมาย**:
- แต่ละเครื่องปลายทางสามารถมี **อุปกรณ์อ่านบัตรที่มอบหมายไว้เพียงเครื่องเดียว**
- แต่ละอุปกรณ์อ่านบัตรสามารถมอบหมายให้กับ **เครื่องปลายทางเพียงเครื่องเดียว**
- การมอบหมายอุปกรณ์อ่านบัตรให้กับเครื่องปลายทาง A จะทำให้อุปกรณ์อ่านบัตรถูกยกเลิกการมอบหมายจากเครื่องปลายทางก่อนหน้า

**การเปลี่ยนการมอบหมาย**:
- แก้ไขบันทึกอุปกรณ์อ่านบัตร
- เปลี่ยนฟิลด์ **Terminal** เป็นเครื่องปลายทางใหม่
- บันทึก
- เครื่องปลายทางเดิมจะสูญเสียการมอบหมายอุปกรณ์อ่านบัตร (จะแสดงข้อความผิดพลาด "ไม่มีอุปกรณ์อ่านบัตรที่มอบหมาย" ระหว่างการชำระเงิน)

**อุปกรณ์อ่านบัตรที่ไม่มีการมอบหมาย**:
- อุปกรณ์อ่านบัตรใหม่ที่ถูกค้นพบจะเริ่มต้นโดยไม่มีการมอบหมาย
- อุปกรณ์อ่านบัตรที่ไม่มีการมอบหมายจะปรากฏในรายการ แต่ไม่สามารถใช้งานได้
- มอบหมายให้กับเครื่องปลายทางเพื่อเปิดใช้งาน

## การปรับแต่งหน้าจอเริ่มต้น

หน้าจอเริ่มต้นของอุปกรณ์อ่านบัตรแสดงแบรนด์บนหน้าจอที่ต่อหน้าลูกค้าเมื่อไม่มีการดำเนินการ:

**หน้าจอเริ่มต้นคืออะไร?**
- ภาพที่แสดงบนหน้าจออุปกรณ์อ่านบัตรเมื่อไม่ได้ประมวลผลการชำระเงิน
- แทนโลโก้ Stripe ที่กำหนดไว้ด้วยแบรนด์ของคุณ
- แสดงให้ลูกค้าเห็นขณะรอการชำระเงิน

**หน้าจอเริ่มต้นอัตโนมัติ vs ปรับแต่ง**:

**หน้าจอเริ่มต้นอัตโนมัติ** (ค่าเริ่มต้น):
- Spwig สร้างหน้าจอเริ่มต้นจากโลโก้ร้านค้าของคุณ (ถ้ามีการตั้งค่าโลโก้ในตั้งค่าร้านค้า)
- ปรับขนาดอัตโนมัติให้ตรงกับข้อกำหนดของอุปกรณ์อ่านบัตร (480×800 แนวตั้ง)
- ขาวดำสำหรับ S700/P400, สีเต็มรูปแบบสำหรับ WisePOS E
- ไม่จำเป็นต้องตั้งค่าใดๆ

**หน้าจอเริ่มต้นแบบปรับแต่ง** (ขั้นสูง):
- อัปโหลดภาพหน้าจอเริ่มต้นที่ออกแบบเอง
- ควบคุมการออกแบบและแบรนด์ได้ทั้งหมด
- ต้องตรงกับข้อกำหนดของภาพ (ดูข้อมูลด้านล่าง)

**ข้อกำหนดหน้าจอเริ่มต้นแบบปรับแต่ง**:
- **ความละเอียด**: 480×800 พิกเซล (แนวตั้ง) แน่นอน
- **รูปแบบ**: PNG หรือ JPG
- **S700/P400**: ขาวดำเท่านั้น (ดำและขาว ไม่มีสีเทา)
- **WisePOS E**: รองรับสีเต็มรูปแบบ
- **ขนาดไฟล์**: <200KB

**การตั้งค่าหน้าจอเริ่มต้นแบบปรับแต่ง**:
1. แก้ไขบันทึกอุปกรณ์อ่านบัตร
2. อัปโหลดภาพไปยังฟิลด์ **Splash Override Image** (หรือเลือกจากไลบรารีภาพ)
3. บันทึก
4. หน้าจอเริ่มต้นจะส 同步ไปยังอุปกรณ์อ่านบัตรภายใน 5 นาที

**การลบหน้าจอเริ่มต้นแบบปรับแต่ง**:
- ลบฟิลด์ **Splash Override Image**
- บันทึก
- อุปกรณ์อ่านบัตรจะกลับไปใช้หน้าจอเริ่มต้นที่สร้างอัตโนมัติ (หรือใช้โลโก้ Stripe ที่กำหนดไว้หากไม่มีโลโก้ร้านค้า)

**การทดสอบหน้าจอเริ่มต้น**:
- หลังจากอัปโหลด รอ 5 นาทีเพื่อให้ส 同步
- เยี่ยมชมอุปกรณ์อ่านบัตร
- ตรวจสอบว่าหน้าจอเริ่มต้นปรากฏบนหน้าจอเมื่อไม่มีการดำเนินการ
- ตรวจสอบคุณภาพภาพ การจัดวาง และความคมชัด

## การตั้งค่าหน้าจอเริ่มต้นของ Stripe

เบื้องหลัง  Spwig จัดการการตั้งค่าหน้าจอเริ่มต้นของ Stripe Terminal:

**stripe_splash_file_id** - ID ภายในของ Stripe สำหรับไฟล์ภาพหน้าจอที่อัปโหลด
- ตั้งค่าอัตโนมัติเมื่อหน้าจอเริ่มต้นถูกอัปโหลด
- ใช้เพื่ออ้างอิงหน้าจอเริ่มต้นใน API ของ Stripe

**stripe_splash_config_id** - ID ภายในของ Stripe สำหรับการตั้งค่าหน้าจอเริ่มต้น
- เชื่อมโยงไฟล์หน้าจอเริ่มต้นกับอุปกรณ์อ่านบัตร
- ถูกจัดการอัตโนมัติเมื่อตั้งค่าหน้าจอเริ่มต้นให้กับอุปกรณ์อ่านบัตร

ฟิลด์เหล่านี้เป็นแบบอ่านได้และถูกจัดการอัตโนมัติ คุณไม่จำเป็นต้องโต้ตอบกับพวกเขาโดยตรง

## การแก้ไขปัญหาทั่วไป

**ปัญหา 1: อุปกรณ์อ่านบัตรแสดงว่าไม่ออนไลน์ แต่เปิดใช้งานอยู่**
- **สาเหตุ**: ปัญหาการเชื่อมต่อเครือข่าย รหัสผ่าน Wi-Fi ถูกเปลี่ยน อุปกรณ์อ่านบัตรอยู่นอกช่วงการรับสัญญาณ
- **วิธีแก้ไข**: ตรวจสอบการตั้งค่าเครือข่ายของอุปกรณ์อ่านบัตร รีเชื่อมต่อกับ Wi-Fi ตรวจสอบว่า API ของ Stripe สามารถเข้าถึงได้จากเครือข่าย

**ปัญหา 2: POS แสดงข้อความ "ไม่มีอุปกรณ์อ่านบัตรที่มอบหมาย" ระหว่างการชำระเงิน**
- **สาเหตุ**: อุปกรณ์อ่านบัตรไม่ได้ถูกมอบหมายให้กับเครื่องปลายทาง หรือการมอบหมายยังไม่สมบูรณ์
- **วิธีแก้ไข**: แก้ไขอุปกรณ์อ่านบัตร มอบหมายให้กับเครื่องปลายทาง บันทึก ทดสอบการชำระเงินอีกครั้ง

**ปัญหา 3: อุปกรณ์อ่านบัตรทำงานต่อเนื่อง (ติดอยู่ที่หน้าจอชำระเงิน)**
- **สาเหตุ**: ธุรกรรมหมดเวลาหรือเกิดข้อผิดพลาด สถานะอุปกรณ์อ่านบัตรไม่ได้รีเซ็ต
- **วิธีแก้ไข**: รีสตาร์ทอุปกรณ์อ่านบัตร (เปิดและปิดใหม่) หากยังคงมีปัญหา ติดต่อฝ่ายสนับสนุนของ Stripe

**ปัญหา 4: หน้าจอเริ่มต้นแบบปรับแต่งไม่แสดงขึ้น**
- **สาเหตุ**: ความละเอียดของภาพไม่ถูกต้อง ยังไม่ได้ส 同步 หรือไม่ตรงกับข้อกำหนดของสีขาวดำ (S700/P400)
- **วิธีแก้ไข**: ตรวจสอบว่าภาพมีความละเอียด 480×800 พิกเซลแน่นอน รอ 5 นาทีเพื่อให้ส 同步 ตรวจสอบว่าเป็นสีขาวดำสำหรับอุปกรณ์อ่านบัตรที่ไม่ใช่สี

**ปัญหา 5: อุปกรณ์อ่านบัตรถูกลงทะเบียนใน Stripe แต่ไม่ปรากฏใน Spwig**
- **สาเหตุ**: อุปกรณ์อ่านบัตรถูกลงทะเบียนกับตำแหน่ง Stripe ที่ต่างจากตำแหน่งในการตั้งค่าผู้ให้บริการ
- **วิธีแก้ไข**: ใน Stripe Dashboard ตรวจสอบว่าตำแหน่งของอุปกรณ์อ่านบัตรตรงกับ ID ตำแหน่งของผู้ให้บริการ

## เคล็ดลับ

- **อุปกรณ์อ่านบัตรหนึ่งเครื่องต่อเครื่องปลายทางหนึ่งเครื่อง** - อย่าแบ่งปันอุปกรณ์อ่านบัตรระหว่างเครื่องปลายทาง เพื่อป้องกันความขัดแย้งและทำให้การรับผิดชอบง่ายขึ้น
- **ลงทะเบียนอุปกรณ์อ่านบัตรก่อนนำไปใช้งาน** - ให้เสร็จสิ้นการลงทะเบียน Stripe และการมอบหมาย Spwig ก่อนที่จะนำอุปกรณ์อ่านบัตรไปยังเคาน์เตอร์ชำระเงิน
- **ทดสอบหน้าจอเริ่มต้นในร้าน** - ความแตกต่างของสีขึ้นอยู่กับรุ่นอุปกรณ์อ่านบัตรและแสงสว่าง ตรวจสอบว่าหน้าจอเริ่มต้นดูดีในสภาพแวดล้อมจริง
- **ตรวจสอบสถานะก่อนเปิดร้าน** - ตรวจสอบรายการอุปกรณ์อ่านบัตรทุกเช้าเพื่อให้แน่ใจว่าอุปกรณ์อ่านบัตรทั้งหมดออนไลน์ก่อนเปิดร้าน
- **ติดป้ายระบุฮาร์ดแวร์ทางกายภาพ** - ใช้เครื่องทำป้ายเพื่อระบุอุปกรณ์อ่านบัตรด้วยชื่อเครื่องปลายทาง ("เครื่องปลายทาง 1 Reader") เพื่อให้ระบุได้ง่ายระหว่างการแก้ไขปัญหา
- **รักษาอุปกรณ์อ่านบัตรให้ใช้พลังงานที่ไม่ขัดจังหวะ** - การไฟฟ้าขัดข้องระหว่างธุรกรรมอาจทำให้สถานะของอุปกรณ์อ่านบัตรเสียหาย แนะนำให้ใช้ระบบจ่ายไฟสำรอง (UPS)
- **บันทึกหมายเลขซีเรียลของอุปกรณ์อ่านบัตร** - บันทึกหมายเลขซีเรียลเพื่อใช้ในการรับประกันและสนับสนุน (พบได้บนป้ายของฮาร์ดแวร์อุปกรณ์อ่านบัตร)
- **อัปเดตเฟิร์มแวร์ของอุปกรณ์อ่านบัตร** - Stripe จะอัปเดตเฟิร์มแวร์อัตโนมัติ แต่ตรวจสอบว่าอุปกรณ์อ่านบัตรอยู่ที่เวอร์ชันล่าสุดเป็นระยะ (ตรวจสอบใน Stripe Dashboard)
