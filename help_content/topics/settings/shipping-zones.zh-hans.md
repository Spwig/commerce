---
title: 配送区域
---

配送区域定义了用于目标配送费率的地理区域——将国家、州或邮政编码分组到区域中，然后将配送方式链接到特定区域以实现精确的费率控制。当地址符合多个区域时，区域使用基于优先级的匹配（优先级最高的区域胜出）。此系统支持复杂的定价策略：对偏远地区收取更高费用，提供国内免费配送，或为特定区域提供折扣费率。

在需要为不同地理区域设置不同配送费用时使用区域，从简单的国内与国际划分到复杂的多区域分层定价。

## 了解配送区域

**什么是区域**：通过国家、州/省和邮政编码模式定义的命名地理区域。

**区域的工作方式**：
1. 客户在结账时输入配送地址
2. 系统评估所有活动区域
3. 与客户地址匹配的区域成为候选
4. 如果有多个区域匹配，优先级最高的区域胜出
5. 显示与胜出区域链接的配送方式
6. 显示未链接到任何区域（或链接到匹配区域）的配送方式

**区域组件**：
- **名称**：区域标识符（例如，"国内"、"欧盟"、"偏远地区"）
- **国家**：包含的国家代码列表（空 = 所有国家）
- **州/省**：按国家划分的州限制（可选）
- **邮政编码模式**：用于邮政编码匹配的正则表达式模式（可选）
- **优先级**：数字越高，当多个区域匹配时优先级越高

## 区域匹配逻辑

区域使用 **逐步缩小** 的方式匹配地址：

### 第1级：国家匹配

**国家列表为空** → 区域匹配所有国家

**提供国家列表** → 地址国家必须在列表中

示例：
```
区域："国内"
国家：["US"]
→ 匹配：任何美国地址
→ 不匹配：加拿大、英国等
```

### 第2级：州/省匹配

**未定义州** → 区域匹配允许国家中的所有州

**特定国家定义的状态** → 地址状态必须匹配

示例：
```
Zone: "West Coast"
Countries: ["US"]
States: {"US": ["CA", "OR", "WA"]}
→ Matches: California, Oregon, Washington addresses
→ No match: New York, Texas, etc.
```

### Level 3: 邮政编码匹配

**未定义模式** → 区域匹配允许国家/州内的所有邮政编码

**定义了模式** → 地址邮政编码必须匹配至少一个模式

示例：
```
Zone: "Los Angeles Metro"
Countries: ["US"]
States: {"US": ["CA"]}
Postal Patterns: ["^90[0-9]{3}$", "^91[0-9]{3}$"]
→ Matches: 90001, 91210, 90245
→ No match: 94102 (San Francisco)
```

**正则表达式模式示例**：
- `^90[0-9]{3}$` - 洛杉矶地区（90000-90999）
- `^[A-Z][0-9][A-Z] [0-9][A-Z][0-9]$` - 加拿大邮政编码格式（K1A 0B1）
- `^SW[0-9]{1,2}` - 以 SW 开头的英国伦敦邮政编码

## 基于优先级的区域选择

当多个区域匹配地址时，**优先级**决定适用哪个区域：

**优先级工作原理**：
- 数字越大，优先级越高
- 如果地址匹配优先级为 100 和 50 的区域，优先级 100 胜出
- 仅适用区域的配送方式可用

**使用场景**：

**场景 1：特定覆盖一般**
```
Zone A: "Remote Alaska"
  Countries: ["US"]
  States: {"US": ["AK"]}
  Priority: 100

Zone B: "Domestic USA"
  Countries: ["US"]
  Priority: 50

Address: Anchorage, AK
→ Matches both zones
→ Priority 100 wins
→ "Remote Alaska" zone applies (higher shipping cost)
```

**场景 2：邮政编码覆盖状态**
```
Zone A: "Manhattan Premium"
  Countries: ["US"]
  States: {"US": ["NY"]}
  Postal Patterns: ["^100[0-2][0-9]$"]
  Priority: 100

Zone B: "New York State"
  Countries: ["US"]
  States: {"US": ["NY"]}
  Priority: 50

Address: New York, NY 10001
→ Matches both zones
→ Priority 100 wins
→ "Manhattan Premium" applies (premium delivery service)
```

## 创建配送区域

**逐步工作流程**：

- 转到 设置 > 运输 > 运输区域

- 点击 "添加运输区域"

- **名称**: 描述性标识符（例如，"欧洲联盟"，"西海岸"，"偏远地区"）

- **优先级**: 设置相对重要性（100 用于具体区域，50 用于通用区域，1 用于备用区域）

- **启用**: 切换以启用或禁用

**选项 A: 所有国家**（留空国家列表）

- 该区域匹配全球所有地址

- 用于默认/备用区域

**选项 B: 特定国家**

- 点击 "添加国家"

- 从下拉菜单中选择国家（如 US, CA, UK 等）

- 对所有包含的国家重复此操作

**选项 C: 特定州/省**

- 添加国家后，为每个国家点击 "添加州/省"

- 从下拉菜单中选择州/省

- 示例：US → CA, OR, WA（西海岸）

**选项 D: 邮政编码模式**（高级）

- 输入正则表达式模式（每行一个）

- 使用示例邮政编码测试模式

- 点击 "验证模式" 以检查语法

- 在编辑运输方法时可以链接（不在区域配置中）

- 或将区域链接到现有方法：编辑方法 → 运输区域 → 选择区域

- 当多个区域匹配时，优先级较高的区域会覆盖优先级较低的区域

- 推荐：具体区域（100），区域（50），默认区域（1）

- 切换 "启用" = 是

- 保存

### 设置 2: 多区域国际

**目标**: 欧盟、北美、亚洲和世界其他地区的不同运费率。

```
Zone 1: "European Union"
  Countries: [AT, BE, BG, CY, CZ, DE, DK, EE, ES, FI, FR, GR, HR, HU, IE, IT, LT, LU, LV, MT, NL, PL, PT, RO, SE, SI, SK]
  Priority: 100

Zone 2: "North America"
  Countries: [US, CA, MX]
  Priority: 100

Zone 3: "Asia Pacific"
  Countries: [AU, CN, HK, IN, JP, KR, NZ, SG, TH, TW]
  Priority: 100

Zone 4: "Rest of World"
  Countries: [Leave empty]
  Priority: 1
```

**配送方式**:
- "EU Shipping" → 欧盟区域
- "North America Shipping" → 北美区域
- "Asia Pacific Shipping" → 亚太区域
- "International Standard" → 世界其他区域

---

### 设置 3: 远程地区附加费

**目标**: 在国内区域中对远程邮政编码添加附加费。

```
Zone 1: "Remote Domestic"
  Countries: [US]
  Postal Patterns: ["^99[0-9]{3}$", "^96[7-9][0-9]{2}$"]  # 阿拉斯加, 夏威夷
  Priority: 100

Zone 2: "Standard Domestic"
  Countries: [US]
  Priority: 50
```

**配送方式**:
- "Remote Shipping" → 远程国内区域（费用较高）
- "Standard Shipping" → 标准国内区域

---

### 设置 4: 按州划分的区域

**目标**: 为美国每个地区设置不同的运费。

```
Zone 1: "West Coast"
  Countries: [US]
  States: {"US": ["CA", "OR", "WA"]}
  Priority: 100

Zone 2: "East Coast"
  Countries: [US]
  States: {"US": ["NY", "NJ", "CT", "MA", "PA"]}
  Priority: 100

Zone 3: "Midwest"
  Countries: [US]
  States: {"US": ["IL", "IN", "OH", "MI", "WI"]}
  Priority: 100

Zone 4: "South"
  Countries: [US]
  States: {"US": ["TX", "FL", "GA", "NC", "SC"]}
  Priority: 100

Zone 5: "Other US States"
  Countries: [US]
  Priority: 50
```

## 邮政编码模式示例

邮政编码使用 **正则表达式**（regular expressions）进行模式匹配：

### 美国（ZIP 编码）

**格式**: 5 位数字（例如，90210）

```
California (90000-96199):  ^9[0-6][0-9]{3}$
New York (10000-14999):    ^1[0-4][0-9]{3}$
Texas (75000-79999, 88500-88599):  ^(7[5-9]|885)[0-9]{2}$
Alaska (99500-99999):      ^99[5-9][0-9]{2}$
```

### 加拿大（邮政编码）

**格式**: A1A 1A1（字母-数字-字母 空格 数字-字母-数字）

```
所有加拿大邮政编码:  ^[A-Z][0-9][A-Z] [0-9][A-Z][0-9]$\n安大略省 (K, L, M, N, P):    ^[KLMNP][0-9][A-Z] [0-9][A-Z][0-9]$\n魁北克省 (G, H, J):           ^[GHJ][0-9][A-Z] [0-9][A-Z][0-9]$\n```

### 英国（邮政编码）

**格式**: AA1A 1AA 或 A1A 1AA

```
伦敦 (E, EC, N, NW, SE, SW, W, WC):  ^(E|EC|N|NW|SE|SW|W|WC)[0-9]{1,2}\n曼彻斯特 (M):                        ^M[0-9]{1,2}\n伯明翰 (B):                        ^B[0-9]{1,2}\n```

### 澳大利亚（邮政编码）

**格式**: 4 位数字（例如，2000）

```
新南威尔士州 (1000-2999):  ^[12][0-9]{3}$\n维多利亚州 (3000-3999, 8000-8999):  ^[38][0-9]{3}$\n昆士兰州 (4000-4999, 9000-9999):  ^[49][0-9]{3}$\n```

### 模式测试

**在保存模式之前**, 使用已知的邮政编码进行测试:

1. 输入模式: `^90[0-9]{3}$`
2. 测试输入: "90210" → 应该匹配
3. 测试输入: "10001" → 应该不匹配
4. 测试输入: "9021" → 应该不匹配（只有 4 位数字）

使用在线正则表达式测试工具（regex101.com）验证复杂模式。

## 区域覆盖摘要

区域在管理列表视图中显示 **覆盖摘要**, 显示包含的内容:

**示例**:
- "所有国家" → 没有国家限制
- "US, CA, MX" → 3 个国家
- "US (CA, OR, WA)" → 美国包含 3 个州
- "US (90xxx-91xxx)" → 美国包含邮政编码模式

**使用摘要**:
- 快速验证区域覆盖，无需打开
- 发现覆盖的重叠或空白
- 一目了然地审核区域配置

## 将区域链接到运输方式

区域和运输方式具有 **多对多关系**:

**从方法侧**（推荐）:
1. 编辑运输方式
2. 滚动到 "运输区域" 部分
3. 选择适用的区域（多选）
4. 保存方法


- 区域不会直接链接到配送方式
- 链接总是从配送方式配置中完成

**未链接任何区域** → 配送方式对所有地址可用

**已链接区域** → 仅当客户地址匹配至少一个链接的区域时，配送方式才可用

**示例**

```
Method: "Domestic Standard"
Linked Zones: ["Domestic USA"]
→ 仅显示给美国地址

Method: "International Express"
Linked Zones: ["EU", "Asia Pacific", "Rest of World"]
→ 显示给所有非美国地址
```

## 测试区域匹配

上线前，请测试区域配置：

1. **创建测试订单**

- 使用不同区域中的地址

- 验证正确的区域匹配

2. **检查优先级解析**

- 使用匹配多个区域的地址

- 验证优先级最高的区域胜出

- 确认预期的配送方式显示

3. **测试边缘情况**

- 边界邮政编码（例如，90999 vs 91000）

- 州边界

- 具有相似邮政编码的国际地址

4. **使用区域预览工具**（如果可用）

- 输入测试地址

- 查看匹配的区域

- 查看优先级解析

## 故障排除

**问题 1：结账时没有可用的配送方式**

**原因**

- 客户地址不匹配任何区域

- 所有配送方式都链接到不匹配的区域

- 没有不受区域限制的配送方式

**解决方案**

- 创建一个备用区域（所有国家，优先级 1）

- 或从至少一个配送方式中移除区域限制

- 验证区域国家/州/邮政编码模式

**问题 2：区域匹配错误**

- 尽管有更高优先级的区域匹配，但选择了更低优先级的区域

- 邮政编码模式语法错误（模式静默失败）

- 州代码不匹配（CA vs California）

- 验证优先级值（数字越大，优先级越高）

- 使用正则表达式验证器测试邮政编码模式

- 使用两位州代码（CA，而不是 California）

**问题 3：显示了意外的配送方式**

**原因**:
- 方法未关联任何区域（适用于所有地方）
- 多个区域匹配，意外区域的优先级更高
- 区域覆盖范围意外重叠

**解决方案**:
- 检查方法关联的区域
- 检查匹配区域的优先级
- 审核区域覆盖范围摘要以查找重叠

## 小贴士

- **从 2 个区域开始** - 国内和国际，按需扩展
- **明智使用优先级** - 具体区域 100，区域 50，备用区域 1
- **彻底测试邮政模式** - 正则表达式错误会静默失败，导致区域无法匹配
- **记录区域逻辑** - 在区域描述中添加注释，解释覆盖意图
- **避免过多区域** - 太多区域会使配置复杂；使用运费促销处理复杂场景
- **使用州代码，而非名称** - "CA" 而非 "California"，"NY" 而非 "New York"
- **创建备用区域** - 所有国家，优先级 1，确保始终至少有一个运费选项可用
- **监控区域性能** - 如果许多客户看到 "无可用运费"，请审核区域覆盖范围
- **为新区域更新区域** - 当新成员加入时，将国家添加到欧盟区域
- **使用描述性名称** - "EU (排除英国)" 比 "区域 3" 更好
- **使用真实地址进行测试** - 测试时使用客户的实际地址，而非虚构的地址
