---
title: 运输区域
---

运输区域定义了针对特定运输费率的地理区域——将国家、州或邮政编码分组到区域中，然后将运输方式链接到特定区域以实现精确的费率控制。当地址符合多个区域时，区域使用基于优先级的匹配（优先级最高的区域胜出）。此系统支持复杂的定价策略：对偏远地区收取更高费用，在国内提供免费运输，或为特定区域提供折扣费率。

在需要不同地理区域的运输成本时使用区域，从简单的国内与国际划分到复杂的多区域分层定价。

## 理解运输区域

**什么是区域**：通过国家、州/省和邮政编码模式定义的命名地理区域。

**区域的工作原理**：
1. 顾客在结账时输入运输地址
2. 系统评估所有活动区域
3. 与顾客地址匹配的区域是候选区域
4. 如果多个区域匹配，优先级最高的区域胜出
5. 与获胜区域链接的运输方式将显示出来
6. 未链接到任何区域（或链接到匹配区域）的运输方式将显示出来

**区域组件**：
- **名称**：区域标识符（例如，"国内"、"欧盟"、"偏远地区"）
- **国家**：包含的国家代码列表（空=所有国家）
- **州/省**：按国家限制的州（可选）
- **邮政编码模式**：用于邮政编码匹配的正则表达式模式（可选）
- **优先级**：数字越高，当多个区域匹配时优先级越高

## 区域匹配逻辑

区域使用**逐步缩小**来匹配地址：

### 第1级：国家匹配

**国家列表为空** → 区域匹配所有国家

**提供国家列表** → 地址国家必须在列表中

示例：
```
区域："国内"
国家：["US"]
→ 匹配：任何美国地址
→ 不匹配：加拿大、英国等
```

### 第2级：州/省匹配

**未定义州** → 区域匹配允许国家的所有州

**为特定国家定义州** → 地址州必须匹配

示例：
```
区域："西海岸"
国家：["US"]
州：{"US": ["CA", "OR", "WA"]}
→ 匹配：加利福尼亚州、俄勒冈州、华盛顿州地址
→ 不匹配：纽约州、德克萨斯州等
```

### 第3级：邮政编码匹配

**未定义模式** → 区域匹配允许国家/州的所有邮政编码

**定义模式** → 地址邮政编码必须匹配至少一个模式

示例：
```
区域："洛杉矶都会区"
国家：["US"]
州：{"US": ["CA"]}
邮政模式：["^90[0-9]{3}$", "^91[0-9]{3}$"]
→ 匹配：90001、91210、90245
→ 不匹配：94102（旧金山）
```

**正则表达式模式示例**：
- `^90[0-9]{3}$` - 洛杉矶地区（90000-90999）
- `^[A-Z][0-9][A-Z] [0-9][A-Z][0-9]$` - 加拿大邮政编码格式（K1A 0B1）
- `^SW[0-9]{1,2}` - 以SW开头的英国伦敦邮政编码

## 基于优先级的区域选择

当多个区域匹配一个地址时，**优先级**决定了哪个区域适用：

**优先级的工作原理**：
- 数字越大，优先级越高
- 如果地址匹配优先级为100和50的区域，优先级100的区域胜出
- 仅显示获胜区域的运输方式

**使用场景**：

**场景1：特定区域覆盖一般区域**
```
区域A："偏远阿拉斯加"
  国家：["US"]
  州：{"US": ["AK"]}
  优先级：100

区域B："美国国内"
  国家：["US"]
  优先级：50

地址：阿拉斯加安克雷奇
→ 匹配两个区域
→ 优先级100胜出
→ "偏远阿拉斯加"区域适用（运输成本更高）
```

**场景2：邮政编码覆盖州**
```
区域A："曼哈顿高级"
  国家：["US"]
  州：{"US": ["NY"]}
  邮政模式：["^100[0-2][0-9]$"]
  优先级：100

区域B："纽约州"
  国家：["US"]
  州：{"US": ["NY"]}
  优先级：50

地址：纽约，NY 10001
→ 匹配两个区域
→ 优先级100胜出
→ "曼哈顿高级"适用（高级配送服务）
```

## 创建运输区域

**逐步工作流程**：

1. **导航到区域**
   - 前往设置 > 运输 > 运输区域
   - 点击"添加运输区域"

2. **基本配置**
   - **名称**：描述性标识符（例如，"欧洲联盟"、"西海岸"、"偏远地区"）
   - **优先级**：设置相对重要性（100用于特定区域，50用于一般区域，1用于备用）
   - **启用**：切换以启用或禁用

3. **定义地理覆盖范围**

   **选项A：所有国家**（留空国家列表）
   - 区域匹配全球所有地址
   - 用于默认/备用区域

   **选项B：特定国家**
   - 点击"添加国家"
   - 从下拉菜单中选择国家（US、CA、UK等）
   - 重复选择所有包含的国家

   **选项C：特定州/省**
   - 在添加国家后，为每个国家点击"添加州"
   - 从下拉菜单中选择州
   - 示例：US → CA、OR、WA（西海岸）

   **选项D：邮政编码模式**（高级）
   - 输入正则表达式模式（每行一个）
   - 使用示例邮政编码测试模式
   - 点击"验证模式"以检查语法

4. **链接到运输方式**
   - 在编辑运输方式时可以链接（不在区域配置中）
   - 或者将区域链接到现有方式：编辑方式 → 运输区域 → 选择区域

5. **设置显示优先级**
   - 当多个区域匹配时，优先级较高的区域覆盖优先级较低的区域
   - 推荐：特定区域（100）、区域（50）、默认区域（1）

6. **激活区域**
   - 切换"启用" = 是
   - 保存

## 常见区域设置

### 设置1：国内与国际

**目标**：国内与所有其他国家的不同费率。

```
区域1："国内"
  国家：[您的国家代码]
  优先级：50

区域2："国际"
  国家：[留空或选择所有其他国家]
  优先级：1
```

**运输方式**：
- "国内标准" → 链接到国内区域
- "国际运输" → 链接到国际区域

---

### 设置2：多区域国际

**目标**：欧盟、北美、亚洲、世界其他地区的不同费率。

```
区域1："欧洲联盟"
  国家：[AT, BE, BG, CY, CZ, DE, DK, EE, ES, FI, FR, GR, HR, HU, IE, IT, LT, LU, LV, MT, NL, PL, PT, RO, SE, SI, SK]
  优先级：100

区域2："北美"
  国家：[US, CA, MX]
  优先级：100

区域3："亚太"
  国家：[AU, CN, HK, IN, JP, KR, NZ, SG, TH, TW]
  优先级：100

区域4："世界其他地区"
  国家：[留空]
  优先级：1
```

**运输方式**：
- "欧盟运输" → 欧盟区域
- "北美运输" → 北美区域
- "亚太运输" → 亚太区域
- "国际标准" → 世界其他地区区域

---

### 设置3：偏远地区附加费

**目标**：为国内区域内的偏远邮政编码添加附加费。

```
区域1："偏远国内"
  国家：[US]
  邮政模式：["^99[0-9]{3}$", "^96[7-9][0-9]{2}$"]  # 阿拉斯加、夏威夷
  优先级：100

区域2："标准国内"
  国家：[US]
  优先级：50
```

**运输方式**：
- "偏远运输" → 偏远国内区域（费用更高）
- "标准运输" → 标准国内区域

---

### 设置4：州特定区域

**目标**：为每个美国地区设置不同费率。

```
区域1："西海岸"
  国家：[US]
  州：{"US": ["CA", "OR", "WA"]}
  优先级：100

区域2："东海岸"
  国家：[US]
  州：{"US": ["NY", "NJ", "CT", "MA", "PA"]}
  优先级：100

区域3："中西部"
  国家：[US]
  州：{"US": ["IL", "IN", "OH", "MI", "WI"]}
  优先级：100

区域4："南部"
  国家：[US]
  州：{"US": ["TX", "FL", "GA", "NC", "SC"]}
  优先级：100

区域5："其他美国州"
  国家：[US]
  优先级：50
```

## 邮政编码模式示例

邮政编码使用**正则表达式**（正则表达式）进行模式匹配：

### 美国（邮政编码）

**格式**：5位数字（例如，90210）

```
加利福尼亚州（90000-96199）：  ^9[0-6][0-9]{3}$
纽约州（10000-14999）：    ^1[0-4][0-9]{3}$
德克萨斯州（75000-79999, 88500-88599）：  ^(7[5-9]|885)[0-9]{2}$
阿拉斯加（99500-99999）：      ^99[5-9][0-9]{2}$
```

### 加拿大（邮政编码）

**格式**：A1A 1A1（字母-数字-字母 空格 数字-字母-数字）

```
所有加拿大邮政编码：  ^[A-Z][0-9][A-Z] [0-9][A-Z][0-9]$ 
安大略省（K, L, M, N, P）：    ^[KLMNP][0-9][A-Z] [0-9][A-Z][0-9]$ 
魁北克省（G, H, J）：           ^[GHJ][0-9][A-Z] [0-9][A-Z][0-9]$ 
```

### 英国（邮政编码）

**格式**：AA1A 1AA 或 A1A 1AA

```
伦敦（E, EC, N, NW, SE, SW, W, WC）：  ^(E|EC|N|NW|SE|SW|W|WC)[0-9]{1,2}
曼彻斯特（M）：                        ^M[0-9]{1,2}
伯明翰（B）：                        ^B[0-9]{1,2}
```

### 澳大利亚（邮政编码）

**格式**：4位数字（例如，2000）

```
新南威尔士州（1000-2999）：  ^[12][0-9]{3}$
维多利亚州（3000-3999, 8000-8999）：  ^[38][0-9]{3}$
昆士兰州（4000-4999, 9000-9999）：  ^[49][0-9]{3}$
```

### 模式测试

**在保存模式之前**，使用已知的邮政编码进行测试：

1. 输入模式：`^90[0-9]{3}$`
2. 测试输入："90210" → 应该匹配
3. 测试输入："10001" → 应该不匹配
4. 测试输入："9021" → 应该不匹配（只有4位数字）

使用在线正则表达式测试工具（regex101.com）验证复杂模式。

## 区域覆盖摘要

区域在管理列表视图中显示**覆盖摘要**，显示包含的内容：

**示例**：
- "所有国家" → 没有国家限制
- "US, CA, MX" → 3个国家
- "US (CA, OR, WA)" → 美国包含3个州
- "US (90xxx-91xxx)" → 美国包含邮政编码模式

**使用摘要**：
- 快速验证区域覆盖，无需打开
- 发现覆盖范围的重叠或空白
- 一目了然地审核区域配置

## 将区域链接到运输方式

区域和方式具有**多对多关系**：

**从方式角度**（推荐）：
1. 编辑运输方式
2. 滚动到"运输区域"部分
3. 选择适用的区域（多选）
4. 保存方式

**从区域角度**：
- 区域不直接链接到方式
- 链接始终从方式配置完成

**方式-区域行为**：

**未链接任何区域** → 方式适用于所有地址

**链接了区域** → 方式仅在客户地址匹配至少一个链接区域时可用

**示例**：
```
方式："国内标准"
链接区域：["国内美国"]
→ 仅显示给美国地址

方式："国际快递"
链接区域：["欧盟", "亚太", "世界其他地区"]
→ 显示给所有非美国地址
```

## 测试区域匹配

在上线前测试区域配置：

1. **创建测试订单**
   - 使用不同区域的地址
   - 验证正确的区域匹配

2. **检查优先级解决**
   - 使用匹配多个区域的地址
   - 验证优先级最高的区域胜出
   - 确认预期的运输方式出现

3. **测试边缘情况**
   - 边界邮政编码（例如，90999 vs 91000）
   - 州边界
   - 国际地址具有相似的邮政编码

4. **使用区域预览工具**（如果可用）
   - 输入测试地址
   - 查看匹配的区域（s）
   - 查看优先级解决

## 故障排除

**问题1：结账时没有可用的运输方式**

**原因**：
- 顾客地址不匹配任何区域
- 所有方式链接到不匹配的区域
- 没有无区域限制的方式存在

**解决方案**：
- 创建备用区域（所有国家，优先级1）
- 或从至少一个方式中移除区域限制
- 验证区域国家/州/邮政编码模式

---

**问题2：区域匹配错误**

**原因**：
- 尽管有更高优先级的区域匹配，但选择了较低优先级的区域
- 邮政编码模式语法错误（模式静默失败）
- 州代码不匹配（CA vs 加利福尼亚）

**解决方案**：
- 验证优先级值（数字越大，优先级越高）
- 使用正则表达式验证器测试邮政编码模式
- 使用2个字母的州代码（CA，而不是加利福尼亚，NY，而不是纽约）

---

**问题3：显示了意外的运输方式**

**原因**：
- 方式没有链接到任何区域（适用于所有地方）
- 多个区域匹配，意外区域具有更高优先级
- 区域覆盖范围意外重叠

**解决方案**：
- 审核方式的链接区域
- 检查匹配区域的优先级
- 审核区域覆盖范围摘要以查找重叠

## 小贴士

- **从2个区域开始** - 国内和国际，根据需要扩展
- **明智地使用优先级** - 特定区域100，区域50，备用1
- **彻底测试邮政编码模式** - 正则表达式错误静默失败，导致区域不匹配
- **记录区域逻辑** - 在区域描述中添加注释，解释覆盖意图
- **避免过多区域** - 太多区域会使配置复杂化；使用运输规则处理复杂场景
- **使用州代码，而不是名称** - "CA"而不是"加利福尼亚"，"NY"而不是"纽约"
- **创建备用区域** - 所有国家，优先级1，确保始终有至少一个运输选项可用
- **监控区域性能** - 如果许多客户看到"没有可用运输"，审核区域覆盖范围
- **更新区域以包含新地区** - 当新成员加入时，将国家添加到欧盟区域
- **使用描述性名称** - "欧盟（不含英国）"比"区域3"更好
- **使用真实地址进行测试** - 在测试时使用客户的实际地址，而不是虚构的地址
