---
title: Способы доставки
---

Способы доставки — это варианты доставки, которые отображаются покупателям на этапе оформления заказа. Каждый способ рассчитывает стоимость доставки с использованием различных стратегий ценообразования. Spwig поддерживает 7 типов способов доставки, варьирующихся от простых фиксированных ставок до сложных, рассчитываемых в реальном времени, с использованием данных от перевозчиков. Способы доставки могут быть ограничены минимальной/максимальной суммой заказа, весом и географическими зонами. Покупатели выбирают предпочтительный способ доставки при оформлении заказа, а рассчитанная стоимость добавляется к общей сумме заказа.

Используйте это руководство, чтобы настроить способы доставки, соответствующие вашей бизнес-модели, от базовых фиксированных ставок до сложных многоуровневых тарифов, основанных на зонах.

## Типы способов доставки

Spwig предоставляет 7 типов способов доставки, каждый из которых имеет разную логику расчета стоимости:

### Способ доставки с фиксированной стоимостью

**Что это такое**: Фиксированная стоимость, не зависящая от содержимого корзины, места назначения или веса.

**Когда использовать**:
- Простые магазины с предсказуемыми затратами на доставку
- Один тип товара (похожий по размеру/весу)
- Только внутренняя доставка со стандартными тарифами перевозчика
- Акции бесплатной доставки (используйте с правилами доставки)

**Настройка**:
- Установите **Тип способа доставки** = Способ доставки с фиксированной стоимостью
- Введите **Фиксированную стоимость** (например, $9.99)
- Необязательно: Установите ограничения минимальной/максимальной суммы заказа

**Пример**: "Стандартная доставка - $9.99" для всех внутренних заказов.

---

### Бесплатная доставка

**Что это такое**: Вариант доставки без оплаты (без оплаты со стороны покупателя).

**Когда использовать**:
- Акции бесплатной доставки
- Высокозначимые заказы (используйте с минимальной суммой заказа)
- Альтернатива самовывозу
- Преимущества программы лояльности

**Настройка**:
- Установите **Тип способа доставки** = Бесплатная доставка
- Необязательно: Установите **Минимальную сумму заказа** (например, бесплатная доставка при заказе от $50)
- Хорошо сочетается с правилами доставки для условной бесплатной доставки

**Пример**: "Бесплатная доставка при заказе от $50" с min_order_value = $50.

---

### Способ доставки, основанный на весе

**Что это такое**: Стоимость рассчитывается на основе таблицы тарифов с уровневыми ставками, основанной на общем весе корзины.

**Когда использовать**:
- Товары с переменным весом (книги, оборудование, продукты питания)
- Модели тарифов перевозчика, основанные на весе
- Предсказуемая пропорция веса к стоимости

**Настройка**:
1. Установите **Тип способа доставки** = Способ доставки, основанный на весе
2. Создайте **Таблицу тарифов на доставку** с basis_type = "weight"
3. Добавьте **Уровни тарифов на доставку** (например, 0-5кг = $10, 5-10кг = $15, 10-20кг = $25)
4. Необязательно: Ограничить использование определенными зонами

**Пример**:
```
0-2кг: $8
2-5кг: $12
5-10кг: $18
10кг+: $25
```

**Как это работает**: Корзина рассчитывает общий вес → находит соответствующий уровень → возвращает ставку уровня.

---

### Способ доставки, основанный на цене

**Что это такое**: Стоимость рассчитывается на основе таблицы тарифов с уровневыми ставками, основанной на общей сумме корзины.

**Когда использовать**:
- Стоимость доставки коррелирует с ценой заказа
- Стимулирует более высокую цену корзины (ниже ставка на доллар при более высоких уровнях)
- Простая альтернатива способу доставки на основе веса для товаров с похожей ценой

**Настройка**:
1. Установите **Тип способа доставки** = Способ доставки, основанный на цене
2. Создайте **Таблицу тарифов на доставку** с basis_type = "price"
3. Добавьте **Уровни тарифов на доставку** (например, $0-$50 = $9.99, $50-$100 = $14.99, $100+ = $19.99)

**Пример**:
```
$0-$25: $6.99
$25-$75: $9.99
$75-$150: $12.99
$150+: Бесплатно
```

**Как это работает**: Корзина рассчитывает общую сумму → находит соответствующий уровень → возвращает ставку уровня.

---

### Реальные тарифы перевозчика

**Что это такое**: Живые тарифы, полученные из API-интерфейсов перевозчиков (FedEx, UPS, DHL) на этапе оформления заказа.

**Когда использовать**:
- Переменные затраты на доставку в зависимости от места назначения
- Несколько вариантов перевозчиков для покупателей
- Точное ценообразование перевозчика без ручных таблиц тарифов
- Международная доставка с сложным ценообразованием

**Настройка**:
1. Установите **Тип способа доставки** = Реальный
2. Создайте **Учетную запись провайдера** (Настройки > Доставка > Учетные записи провайдеров)
3. Введите учетные данные API перевозчика (номер учетной записи, API-ключ, секрет)
4. Свяжите учетную запись провайдера со способом доставки
5. Необязательно: Добавьте процентную наценку или фиксированную наценку

**Требования**:
- Активная учетная запись перевозчика (FedEx, UPS, DHL и т.д.)
- Учетные данные API от перевозчика
- Определенные упаковки для доставки (для расчета объемного веса)

**Пример**: Метод "FedEx Ground" получает живые тарифы FedEx на основе веса корзины, размеров и места назначения при оформлении заказа.

**Как это работает**:
1. Покупатель вводит адрес при оформлении заказа
2. Система вызывает API-интерфейс перевозчика с информацией об отправке, месте назначения, размерах упаковки и весе
3. Перевозчик возвращает предложение по тарифу
4. Применяется необязательная наценка
5. Тариф отображается покупателю

---

### Самовывоз

**Что это такое**: Покупатель сам забирает заказ в физическом месте (без затрат на доставку).

**Когда использовать**:
- Розничные магазины, предлагающие самовывоз
- Варианты самовывоза с склада
- События или ярмарки
- Устранение затрат на доставку для местных клиентов

**Настройка**:
1. Установите **Тип способа доставки** = Самовывоз
2. Создайте **Местоположение** (Настройки > Доставка > Местоположения)
   - Установите адрес, часы работы, пропускную способность для самовывоза
3. Свяжите местоположение(я) со способом доставки
4. Необязательно: Установите время подготовки самовывоза (например, "Готово через 2 часа")

**Опыт клиента**:
- Выбирает "Самовывоз" при оформлении заказа
- Выбирает место самовывоза (если несколько)
- Выбирает дату/время самовывоза в зависимости от доступности
- Получает уведомление, когда заказ готов

**Пример**: "Самовывоз в магазине - Бесплатно" с 3 розничными магазинами, готовыми в течение 24 часов.

---

### Способ доставки по таблице тарифов

**Что это такое**: Гибкая многоуровневая цена, основанная на весе, цене или количестве с продвинутой настройкой зон.

**Когда использовать**:
- Сложное ценообразование (разные тарифы по зонам и весу)
- Нужна большая степень контроля, чем у способа доставки на основе веса или цены
- Несколько факторов ценообразования (например, вес + место назначения + количество)

**Настройка**:
1. Установите **Тип способа доставки** = Способ доставки по таблице тарифов
2. Создайте **Таблицу тарифов на доставку**
3. Определите **basis_type**: вес, цена или количество
4. Добавьте **Уровни тарифов на доставку** с минимальными/максимальными значениями
5. Необязательно: Ограничить уровни определенными зонами или странами

**Разница с способом доставки на основе веса/цены**: Способ доставки по таблице тарифов поддерживает географические ограничения на уровне, позволяя разным тарифам для одного веса/цены в разных зонах.

**Пример**:
```
Зона A (Внутренняя):
  0-5кг: $10
  5-10кг: $15

Зона B (Отдаленная):
  0-5кг: $18
  5-10кг: $25
```

**Как это работает**: Корзина рассчитывает значение основы (вес/цена/количество) → находит соответствующий уровень для зоны клиента → возвращает ставку уровня.

## Настройка способов доставки

Все способы доставки имеют общие настройки:

### Базовые настройки

- **Название**: Внутреннее обозначение (не отображается покупателям)
- **Отображаемое название**: Название, отображаемое покупателям при оформлении заказа (например, "Стандартная доставка", "Экспресс-доставка")
- **Описание**: Необязательный текст помощи, отображаемый при оформлении заказа (например, "Доставка в течение 3-5 рабочих дней")
- **Тип способа доставки**: Один из 7 типов выше
- **Активный**: Переключатель для включения/отключения способа без удаления

### Настройки стоимости

- **Фиксированная стоимость**: Только для способов доставки с фиксированной стоимостью
- **Таблица тарифов**: Для способов доставки на основе веса, цены, таблицы тарифов
- **Учетная запись провайдера**: Для способов доставки в реальном времени
- **Класс налога**: Применить налог к стоимости доставки (если применимо)

### Ограничения

**Ограничения по сумме заказа**:
- **Минимальная сумма заказа**: Способ доступен только если сумма корзины ≥ сумма (например, бесплатная доставка при заказе от $50)
- **Максимальная сумма заказа**: Способ скрыт, если сумма корзины > сумма (например, фиксированная стоимость только для заказов менее $100)

**Ограничения по весу**:
- **Минимальный вес**: Способ доступен только если вес корзины ≥ сумма
- **Максимальный вес**: Способ скрыт, если вес корзины > сумма (обычно для легких способов доставки)

**Географические ограничения**:
- **Зоны доставки**: Свяжите способ с определенными зонами (внутренние, международные, региональные)
- Пустые зоны = доступны для всех адресов
- Несколько зон = доступны для любой совпадающей зоны

### Расширенные настройки

- **Приоритет**: Порядок отображения в оформлении заказа (ниже число = выше в списке)
- **Плата за обработку**: Дополнительная фиксированная плата, добавляемая к рассчитанной стоимости
- **Порог бесплатной доставки**: Автоматически устанавливает стоимость в $0, если сумма корзины ≥ порог (альтернатива min_order_value)

## Создание способа доставки

**Пошаговая процедура**:

1. **Перейдите к способам доставки**
   - Перейдите в Настройки > Корзина > Способы доставки
   - Нажмите "Добавить способ доставки"

2. **Выберите тип способа доставки**
   - Выберите подходящий тип в зависимости от вашей стратегии ценообразования
   - Тип определяет доступные поля настройки стоимости

3. **Настройте базовую информацию**
   - Название: Внутреннее обозначение (например, "domestic_ground")
   - Отображаемое название: Для покупателей (например, "Стандартная доставка")
   - Описание: Время доставки (например, "5-7 рабочих дней")

4. **Установите расчет стоимости**
   - **Фиксированная стоимость**: Введите фиксированную стоимость
   - **Вес/Цена/Таблица тарифов**: Создайте таблицу тарифов (см. ниже)
   - **Реальный**: Свяжите учетную запись провайдера
   - **Бесплатная/Самовывоз**: Нет необходимости настраивать стоимость

5. **Добавьте ограничения (необязательно)**
   - Минимальная/максимальная сумма заказа
   - Минимальный/максимальный вес
   - Зоны доставки

6. **Установите приоритет**
   - Чем ниже число, тем выше способ в списке при оформлении заказа
   - Рекомендуемый порядок: Бесплатная (1), Самовывоз (2), Стандартная (3), Экспресс (4)

7. **Активируйте способ**
   - Переключите "Активный" = Да
   - Сохраните

## Создание таблиц тарифов

Для способов доставки на основе веса, цены и таблицы тарифов:

**Шаг 1: Создайте таблицу тарифов**
- Перейдите в Настройки > Доставка > Таблицы тарифов
- Нажмите "Добавить таблицу тарифов"
- Установите **Название** (например, "Тарифы на вес внутри страны")
- Установите **Тип основы**: вес, цена или количество

**Шаг 2: Добавьте уровни**
- Нажмите "Добавить уровень"
- Установите **Минимальное значение** и **Максимальное значение** (диапазон для совпадения)
- Установите **Тариф** (стоимость для этого уровня)
- Необязательно: Ограничить определенными зонами или странами
- Сохраните уровень

**Шаг 3: Повторите для всех уровней**
- Охватите полный диапазон (от 0 до максимального ожидаемого значения)
- Убедитесь, что нет промежутков (например, 0-5, 5-10, 10-20, 20+)
- Используйте `null` для максимального значения в последнем уровне (без ограничений)

**Шаг 4: Свяжите с способом доставки**
- Редактируйте способ доставки
- Выберите таблицу тарифов из выпадающего списка
- Сохраните

**Пример таблицы тарифов на основе веса**:
```
Название: Тарифы на вес внутри страны
Тип основы: Вес

Уровни:
1. Мин: 0г, Макс: 2000г, Тариф: $8
2. Мин: 2000г, Макс: 5000г, Тариф: $12
3. Мин: 5000г, Макс: 10000г, Тариф: $18
4. Мин: 10000г, Макс: null, Тариф: $25
```

## Распространенные сценарии доставки

### Сценарий 1: Базовая внутренняя доставка

**Цель**: Простая фиксированная стоимость $9.99 для всех внутренних заказов.

**Решение**:
- Тип способа доставки: Фиксированная стоимость
- Фиксированная стоимость: $9.99
- Зона доставки: "Внутренняя" (только ваша страна)

---

### Сценарий 2: Бесплатная доставка при заказе от $50

**Цель**: Стимулировать более высокую цену корзины с порогом бесплатной доставки.

**Вариант решения A** (Рекомендуется):
- Тип способа доставки: Бесплатная доставка
- Минимальная сумма заказа: $50
- Отображаемое название: "Бесплатная доставка (заказы $50+)")

**Вариант решения B** (С использованием правил):
- Тип способа доставки: Фиксированная стоимость
- Фиксированная стоимость: $9.99
- Создайте правило доставки:
  - Условие: Сумма корзины ≥ $50
  - Действие: Установите стоимость в $0

---

### Сценарий 3: Внутренняя доставка на основе веса + международная доставка

**Цель**: Разные тарифы для внутренней и международной доставки на основе веса.

**Решение**:
1. Создайте 2 зоны: "Внутренняя", "Международная"
2. Создайте 2 таблицы тарифов: "Внутренние тарифы на вес", "Международные тарифы на вес"
3. Создайте 2 способа доставки:
   - "Внутренняя доставка" → связана с зоной "Внутренняя" + таблицей тарифов "Внутренние тарифы на вес"
   - "Международная доставка" → связана с зоной "Международная" + таблицей тарифов "Международные тарифы на вес"

---

### Сценарий 4: Несколько вариантов перевозчиков

**Цель**: Позволить покупателям выбирать между FedEx Ground, FedEx Express, UPS Ground.

**Решение**:
1. Создайте учетную запись провайдера для API FedEx
2. Создайте учетную запись провайдера для API UPS
3. Создайте 3 способа доставки в реальном времени:
   - "FedEx Ground" → провайдер FedEx, код услуги = "FEDEX_GROUND"
   - "FedEx Express" → провайдер FedEx, код услуги = "FEDEX_EXPRESS"
   - "UPS Ground" → провайдер UPS, код услуги = "UPS_GROUND"
4. Все 3 способа доставки запрашивают API-интерфейсы перевозчиков при оформлении заказа и отображают живые тарифы

---

### Сценарий 5: Самовывоз + доставка

**Цель**: Розничный магазин предлагает как самовывоз, так и доставку.

**Решение**:
1. Создайте местоположение: "Основной магазин" с адресом, часами работы и временем подготовки
2. Создайте 2 способа доставки:
   - "Самовывоз" → тип Самовывоз, связанный с местоположением "Основной магазин"
   - "Стандартная доставка" → фиксированная стоимость $9.99
3. Покупатели видят оба варианта при оформлении заказа

## Тестирование способов доставки

Перед запуском в продажу протестируйте все способы доставки:

1. **Создайте тестовую корзину**
   - Добавьте товары с разным весом/ценой
   - Перейдите к оформлению заказа

2. **Проверьте каждый способ доставки**
   - Введите адреса в разных зонах
   - Убедитесь, что отображаются правильные способы доставки
   - Проверьте, совпадают ли рассчитанные стоимости с ожиданиями

3. **Проверьте ограничения**
   - Добавьте товары до достижения минимальной суммы заказа → убедитесь, что появляется бесплатная доставка
   - Добавьте тяжелые товары → убедитесь, что уровни тарифов на основе веса работают
   - Проверьте ограничения по зонам → убедитесь, что способы доставки скрыты для исключенных зон

4. **Проверьте способы доставки в реальном времени** (если применимо)
   - Используйте тестовые учетные данные провайдера
   - Убедитесь, что тарифы возвращаются успешно
   - Проверьте точность тарифов по сравнению с веб-сайтом перевозчика

## Устранение неполадок

**Проблема 1: Способ доставки не отображается при оформлении заказа**

**Причины**:
- Способ неактивен
- Сумма корзины не соответствует минимальной/максимальной сумме заказа
- Сумма корзины не соответствует минимальному/максимальному весу
- Адрес клиента не соответствует ни одной из связанных зон
- Нет уровней таблицы тарифов, охватывающих вес/цену корзины

**Решение**: Проверьте ограничения, убедитесь, что способ активен, убедитесь, что зоны/уровни охватывают сценарий клиента.

---

**Проблема 2: Неудачный расчет реальных тарифов**

**Причины**:
- Неверные учетные данные API
- Учетная запись провайдера неактивна
- Не определены упаковки для доставки (перевозчику нужны размеры)
- Не установлен адрес отправки
- API-интерфейс перевозчика недоступен

**Решение**: Проверьте соединение с провайдером, убедитесь, что учетные данные верны, убедитесь, что упаковки настроены, проверьте адрес отправки в настройках.

---

**Проблема 3: Неправильно рассчитанная стоимость**

**Причины**:
- Уровни таблицы тарифов имеют промежутки или пересечения
- Минимальные/максимальные значения уровней в неправильных единицах (граммы против кг)
- Неожиданно добавляется плата за обработку
- Правило доставки изменяет стоимость

**Решение**: Проверьте уровни таблицы тарифов, убедитесь, что единицы измерения верны, проверьте приоритет правил доставки.

## Советы

- **Начните с простого** - Используйте фиксированную ставку для первого способа, добавляйте сложность по мере необходимости
- **Тщательно протестируйте** - Проверьте, что все способы работают в тестовой среде, прежде чем включать их в продажу
- **Используйте описательные названия** - "Стандартная доставка (5-7 дней)" лучше, чем "Способ 1"
- **Установите реалистичные сроки доставки** - Подписывайте меньше, чем обещаете, чтобы доставлять быстрее для удовлетворения клиентов
- **Предложите самовывоз, если это возможно** - Снижает затраты на доставку, улучшает удобство для клиентов
- **Контролируйте надежность API перевозчика** - Используйте фиксированную ставку в качестве резервного варианта, если не удается получить тарифы в реальном времени
- **Используйте зоны для международных доставок** - Разные тарифы по регионам предотвращают потери на дорогих направлениях
- **Комбинируйте с правилами доставки** - Правила добавляют условную логику (акции бесплатной доставки, наценки на удаленные районы)

- **Обновляйте таблицы тарифов по сезонам** - Тарифы перевозчиков меняются, проверяйте ежегодно
- **Используйте приоритеты с умом** - Поместите бесплатные/дешевые варианты в начале, дорогие в конце
