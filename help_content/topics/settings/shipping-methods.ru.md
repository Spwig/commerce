title: Способы доставки
---

Способы доставки — это варианты доставки, отображаемые клиентам на этапе оформления заказа. Каждый способ рассчитывает стоимость доставки с использованием различных стратегий ценообразования. Spwig поддерживает 7 типов способов доставки, от простых фиксированных ставок до сложных реальных цен, рассчитываемых носителями. Способы доставки могут быть ограничены минимальной/максимальной стоимостью заказа, весом и географическими зонами. Клиенты выбирают предпочтительный способ доставки при оформлении заказа, а рассчитанная стоимость добавляется к общей сумме заказа.

Используйте это руководство для настройки способов доставки, соответствующих вашей бизнес-модели, от базовых фиксированных ставок до сложных многоуровневых цен, основанных на зонах.

## Типы способов доставки

Spwig предоставляет 7 типов способов доставки, каждый из которых имеет разную логику расчета стоимости:

### Ставка по фиксированной цене

**Что это такое**: Фиксированная стоимость, не зависящая от содержимого корзины, места назначения или веса.

**Когда использовать**:
- Простые магазины с предсказуемыми затратами на доставку
- Один тип продукта (похожий по размеру/весу)
- Только внутренняя доставка со стандартными тарифами перевозчика
- Акции бесплатной доставки (используйте с акциями на доставку)

**Настройка**:
- Установите **Тип метода** = Ставка по фиксированной цене
- Введите **Фиксированную стоимость** (например, $9.99)
- Необязательно: Установите ограничения минимальной/максимальной стоимости заказа

**Пример**: "Стандартная доставка - $9.99" для всех внутренних заказов.

---

### Бесплатная доставка

**Что это такое**: Бесплатный вариант доставки (без оплаты со стороны клиента).

**Когда использовать**:
- Акции бесплатной доставки
- Высокозначимые заказы (используйте с минимальной стоимостью заказа)
- Альтернатива самовывозу
- Преимущества программы лояльности

**Настройка**:
- Установите **Тип метода** = Бесплатная доставка
- Необязательно: Установите **Минимальную стоимость заказа** (например, бесплатная при заказе свыше $50)
- Хорошо сочетается с акциями на доставку для условной бесплатной доставки

**Пример**: "Бесплатная доставка при заказе свыше $50" с min_order_value = $50.

---

### Доставка на основе веса

**Что это такое**: Стоимость рассчитывается на основе таблицы тарифов с уровневыми ставками, основанных на общем весе корзины.

**Когда использовать**:
- Продукты с переменным весом (книги, оборудование, продукты питания)
- Модели тарифов перевозчика на основе веса
- Предсказуемая пропорция веса к стоимости

**Настройка**:
1. Установите **Тип метода** = Доставка на основе веса
2. Создайте **Таблицу тарифов на доставку** с basis_type = "weight"
3. Добавьте **Уровни тарифов на доставку** (например, 0-5кг = $10, 5-10кг = $15, 10-20кг = $25)
4. Необязательно: Ограничить определенными зонами

**Пример**:
```
0-2кг: $8
2-5кг: $12
5-10кг: $18
10кг+: $25
```

**Как это работает**: Корзина рассчитывает общий вес → находит соответствующий уровень → возвращает ставку уровня.

---

### Доставка на основе цены

**Что это такое**: Стоимость рассчитывается на основе таблицы тарифов с уровневыми ставками, основанных на подытоженной сумме корзины.

**Когда использовать**:
- Стоимость доставки коррелирует с ценой заказа
- Стимулировать более высокую сумму корзины (более низкая ставка на доллар при более высоких уровнях)
- Простая альтернатива доставке на основе веса для товаров с похожей ценой

**Настройка**:
1. Установите **Тип метода** = Доставка на основе цены
2. Создайте **Таблицу тарифов на доставку** с basis_type = "price"
3. Добавьте **Уровни тарифов на доставку** (например, $0-$50 = $9.99, $50-$100 = $14.99, $100+ = $19.99)

**Пример**:
```
$0-$25: $6.99
$25-$75: $9.99
$75-$150: $12.99
$150+: Бесплатно
```

**Как это работает**: Корзина рассчитывает подытоженную сумму → находит соответствующий уровень → возвращает ставку уровня.

---

### Реальные тарифы перевозчика

**Что это такое**: Живые тарифы, полученные из API-интерфейсов перевозчика (FedEx, UPS, DHL) на этапе оформления заказа.

**Когда использовать**:
- Переменные затраты на доставку в зависимости от места назначения
- Несколько вариантов перевозчиков для клиентов
- Точная стоимость перевозчика без ручных таблиц тарифов
- Международная доставка со сложной ценовой политикой

**Настройка**:
1. Установите **Тип метода** = Реальный
2. Создайте **Учетную запись провайдера** (Настройки > Доставка > Учетные записи провайдеров)
3. Введите учетные данные API перевозчика (номер учетной записи, API-ключ, секрет)
4. Свяжите учетную запись провайдера со способом доставки
5. Необязательно: Добавьте процентную наценку или фиксированную наценку

**Требования**:
- Активная учетная запись перевозчика (FedEx, UPS, DHL и т.д.)
- Учетные данные API от перевозчика
- Определенные упаковки для доставки (для расчета объемного веса)


**Пример**: Метод "FedEx Ground" получает актуальные тарифы FedEx на основе веса, размеров и места назначения корзины при оформлении заказа.

**Как это работает**:
1. Покупатель вводит адрес при оформлении заказа
2. Система обращается к API перевозчика с информацией об отправной точке, месте назначения, размерах и весе посылки
3. Перевозчик возвращает оценку стоимости
4. Необязательная наценка применяется
5. Стоимость отображается покупателю

---

### Локальная выдача

**Что это такое**: Покупатель сам забирает заказ в физическом месте (без стоимости доставки).

**Когда использовать**:
- Магазины розничной торговли, предлагающие выдачу
- Варианты выдачи на складе
- События или ярмарки
- Устранение стоимости доставки для местных клиентов

**Настройка**:
1. Установите **Тип метода** = Локальная выдача
2. Создайте **Локацию** (Настройки > Доставка > Локации)
   - Установите адрес, часы работы, пропускную способность выдачи
3. Свяжите локацию(ы) с методом
4. Необязательно: Установите время подготовки выдачи (например, "Готово через 2 часа")

**Опыт клиента**:
- Выбирает "Локальная выдача" при оформлении заказа
- Выбирает место выдачи (если их несколько)
- Выбирает дату и время выдачи в зависимости от доступности
- Получает уведомление, когда заказ готов

**Пример**: "Выдача в магазине - Бесплатно" с 3 розничными локациями, готовыми в течение 24 часов.

---

### Тарифы доставки по таблице

**Что это такое**: Гибкая многоуровневая ценовая политика, основанная на весе, цене или количестве с расширенной настройкой зон.

**Когда использовать**:
- Сложные тарифы (разные ставки по зонам И весу)
- Нужна большая степень контроля, чем у методов, основанных только на весе или цене
- Несколько факторов ценообразования (например, вес + место назначения + количество)

**Настройка**:
1. Установите **Тип метода** = Тарифы по таблице
2. Создайте **Таблицу тарифов доставки**
3. Определите **basis_type**: вес, цена или количество
4. Добавьте **Уровни тарифов доставки** с минимальными/максимальными значениями
5. Необязательно: Ограничьте уровни определенными зонами или странами

**Разница с методами на основе веса/цены**: Тарифы по таблице поддерживают географические ограничения на уровне каждого уровня, позволяя разные тарифы для одного и того же веса/цены в разных зонах.

**Пример**:
```
Зона A (Домашняя):
  0-5кг: $10
  5-10кг: $15

Зона B (Отдаленная):
  0-5кг: $18
  5-10кг: $25
```

**Как это работает**: Корзина рассчитывает значение основы (вес/цена/количество) → находит соответствующий уровень для зоны клиента → возвращает ставку уровня.

## Настройка метода доставки

Все методы доставки имеют эти общие настройки:

### Основные настройки

- **Название**: Внутренний идентификатор (не отображается клиентам)
- **Отображаемое название**: Название, видимое клиентам при оформлении заказа (например, "Стандартная доставка", "Экспресс-доставка")
- **Описание**: Необязательный текст помощи, отображаемый при оформлении заказа (например, "Доставка в течение 3-5 рабочих дней")
- **Тип метода**: Один из 7 типов выше
- **Активен**: Переключатель для включения/отключения метода без удаления

### Настройки стоимости

- **Фиксированная стоимость**: Только для методов с фиксированной стоимостью
- **Таблица тарифов**: Для методов на основе веса, цены, тарифов по таблице
- **Учетная запись провайдера**: Для методов доставки в реальном времени
- **Класс налога**: Применить налог к стоимости доставки (если применимо)

### Ограничения

**Ограничения по стоимости заказа**:
- **Минимальная стоимость заказа**: Метод доступен только если сумма корзины ≥ сумма (например, бесплатная доставка при заказе от $50)
- **Максимальная стоимость заказа**: Метод скрыт, если сумма корзины > сумма (например, фиксированная стоимость только для заказов до $100)

**Ограничения по весу**:
- **Минимальный вес**: Метод доступен только если вес корзины ≥ значение
- **Максимальный вес**: Метод скрыт, если вес корзины > значение (обычно для легких вариантов доставки)

**Географические ограничения**:
- **Зоны доставки**: Свяжите метод с определенными зонами (домашние, международные, региональные)
- Пустые зоны = доступны для всех адресов
- Несколько зон = доступны для любой совпадающей зоны

### Расширенные настройки

- **Приоритет**: Порядок отображения в оформлении заказа (меньшее число = выше в списке)
- **Плата за обработку**: Дополнительная фиксированная плата, добавляемая к рассчитанной стоимости
- **Порог бесплатной доставки**: Автоматически устанавливает стоимость в $0, если сумма корзины ≥ порог (альтернатива min_order_value)

## Создание метода доставки

**Пошаговая инструкция**:

1. **Перейдите к методам доставки**
   - Перейдите в Настройки > Корзина > Методы доставки
   - Нажмите "Добавить метод доставки"

2. **Выберите тип метода**
   - Выберите подходящий тип в зависимости от вашей стратегии ценообразования
   - Тип определяет доступные поля настройки стоимости

3. **Настройка базовой информации**
   - Название: Внутренняя ссылка (например, "domestic_ground")
   - Отображаемое название: Для клиентов (например, "Ground Shipping")
   - Описание: Срок доставки (например, "5-7 рабочих дней")

4. **Настройка расчета стоимости**
   - **Фиксированная стоимость**: Введите фиксированную стоимость
   - **Вес/Цена/Таблица стоимостей**: Создайте таблицу стоимостей (см. ниже)
   - **Реальное время**: Свяжите аккаунт провайдера
   - **Бесплатно/Самовывоз**: Нет необходимости настраивать стоимость

5. **Добавление ограничений (опционально)**
   - Минимальная/максимальная сумма заказа
   - Минимальный/максимальный вес
   - Зоны доставки

6. **Настройка приоритета**
   - Меньшие числа отображаются первыми на этапе оформления заказа
   - Рекомендуемый порядок: Бесплатно (1), Локальный самовывоз (2), Стандартный (3), Экспресс (4)

7. **Активация метода**
   - Переключите "Активный" = Да
   - Сохранить

## Создание таблиц стоимостей

Для методов, основанных на весе, цене и таблице стоимостей:

**Шаг 1: Создание таблицы стоимостей**
- Перейдите в Настройки > Доставка > Таблицы стоимостей
- Нажмите "Добавить таблицу стоимостей"
- Установите **Название** (например, "Domestic Weight Tiers")
- Установите **Тип основы**: вес, цена или количество

**Шаг 2: Добавление уровней**
- Нажмите "Добавить уровень"
- Установите **Минимальное значение** и **Максимальное значение** (диапазон для соответствия)
- Установите **Стоимость** (стоимость для этого уровня)
- Необязательно: Ограничить определенными зонами или странами
- Сохранить уровень

**Шаг 3: Повторить для всех уровней**
- Охватите полный диапазон (от 0 до максимального ожидаемого значения)
- Убедитесь, что нет промежутков (например, 0-5, 5-10, 10-20, 20+)
- Используйте `null` для максимального значения в последнем уровне (без ограничений)

**Шаг 4: Связать с методом доставки**
- Редактировать метод доставки
- Выберите таблицу стоимостей из выпадающего списка
- Сохранить

**Пример таблицы стоимостей на основе веса**:
```
Название: Domestic Weight Tiers
Основа: Вес

Уровни:
1. Мин: 0г, Макс: 2000г, Стоимость: $8
2. Мин: 2000г, Макс: 5000г, Стоимость: $12
3. Мин: 5000г, Макс: 10000г, Стоимость: $18
4. Мин: 10000г, Макс: null, Стоимость: $25
```

## Распространенные сценарии доставки

### Сценарий 1: Базовая национальная доставка

**Цель**: Простая фиксированная стоимость $9.99 для всех национальных заказов.

**Решение**:
- Тип метода: Фиксированная стоимость
- Фиксированная стоимость: $9.99
- Зона доставки: "Национальная" (только ваша страна)

---

### Сценарий 2: Бесплатная доставка при заказе от $50

**Цель**: Стимулировать более высокую сумму заказа с порогом бесплатной доставки.

**Вариант решения A** (Рекомендуется):
- Тип метода: Бесплатная доставка
- Минимальная сумма заказа: $50
- Отображаемое название: "Бесплатная доставка (заказы от $50+)")

**Вариант решения B** (С использованием правил):
- Тип метода: Фиксированная стоимость
- Фиксированная стоимость: $9.99
- Создать акцию доставки:
  - Условие: Сумма заказа ≥ $50
  - Действие: Установить стоимость в $0

---

### Сценарий 3: Национальная и международная доставка на основе веса

**Цель**: Разные ставки для национальной и международной доставки в зависимости от веса.

**Решение**:
1. Создайте 2 зоны: "Национальная", "Международная"
2. Создайте 2 таблицы стоимостей: "Национальные весовые ставки", "Международные весовые ставки"
3. Создайте 2 метода:
   - "Национальная доставка" → связана с зоной "Национальная" + таблицей "Национальные весовые ставки"
   - "Международная доставка" → связана с зоной "Международная" + таблицей "Международные весовые ставки"

---

### Сценарий 4: Несколько вариантов доставки

**Цель**: Позволить клиентам выбирать между FedEx Ground, FedEx Express, UPS Ground.

**Решение**:
1. Создайте аккаунт провайдера для API FedEx
2. Создайте аккаунт провайдера для API UPS
3. Создайте 3 метода в режиме реального времени:
   - "FedEx Ground" → провайдер FedEx, код услуги = "FEDEX_GROUND"
   - "FedEx Express" → провайдер FedEx, код услуги = "FEDEX_EXPRESS"
   - "UPS Ground" → провайдер UPS, код услуги = "UPS_GROUND"
4. Все три метода запрашивают API провайдера при оформлении заказа и отображают актуальные ставки

---

### Сценарий 5: Самовывоз и доставка

**Цель**: Розничный магазин предлагает как самовывоз, так и доставку.

**Решение**:
1. Создайте локацию: "Главный магазин" с адресом, временем работы и временем подготовки
2. Создайте 2 метода:
   - "Локальный самовывоз" → тип Локальный самовывоз, связанный с локацией "Главный магазин"
   - "Стандартная доставка" → фиксированная стоимость $9.99
3. Клиенты видят оба варианта при оформлении заказа

## Тестирование методов доставки

Перед запуском проверьте все методы:


1. **Создать тестовую корзину**
   - Добавить товары с различным весом/ценой
   - Перейти к оформлению заказа

2. **Проверить каждый метод**
   - Ввести адреса в разных зонах
   - Убедиться, что отображаются правильные методы
   - Проверить, совпадают ли рассчитанные стоимости с ожиданиями

3. **Проверить ограничения**
   - Добавить товары до достижения минимальной суммы заказа → убедиться, что появляется бесплатная доставка
   - Добавить тяжелые товары → убедиться, что работают уровни на основе веса
   - Проверить ограничения по зонам → убедиться, что методы скрыты для исключенных зон

4. **Проверить методы в реальном времени** (если применимо)
   - Использовать тестовые учетные данные перевозчика
   - Убедиться, что ставки успешно возвращаются
   - Проверить точность ставок по сравнению с веб-сайтом перевозчика

## Устранение неполадок

**Проблема 1: Метод не отображается при оформлении заказа**

**Причины**:
- Метод не активен
- Корзина не соответствует минимальной/максимальной сумме заказа
- Корзина не соответствует минимальному/максимальному весу
- Адрес клиента не совпадает с любыми связанными зонами
- Нет таблиц ставок, охватывающих вес/цену корзины

**Решение**: Проверьте ограничения, убедитесь, что статус активен, проверьте, охватывают ли зоны/уровни сценарий клиента.

---

**Проблема 2: Не удается получить ставки в реальном времени**

**Причины**:
- Неверные учетные данные API
- Учетная запись поставщика не активна
- Не определены упаковки для доставки (перевозчику нужны размеры)
- Не задан адрес отправки
- API перевозчика недоступен

**Решение**: Проверьте соединение с поставщиком, убедитесь, что учетные данные верны, проверьте, настроены ли упаковки, проверьте адрес отправки в настройках.

---

**Проблема 3: Неверно рассчитанная стоимость**

**Причины**:
- В таблице ставок есть промежутки или пересечения
- Минимальные/максимальные значения уровней указаны в неправильных единицах (граммы против кг)
- Неожиданно добавлена плата за обработку
- Правило доставки изменяет стоимость

**Решение**: Проверьте уровни таблицы ставок, убедитесь, что единицы измерения верны, проверьте приоритетность акций доставки.

## Советы

- **Начните с простого** - Используйте фиксированную ставку для первого метода, добавляйте сложность по мере необходимости
- **Тщательно тестируйте** - Проверьте, что все методы работают в тестовой среде, прежде чем включать их в продажу
- **Используйте описательные названия** - "Стандартная доставка (5-7 дней)" лучше, чем "Метод 1"
- **Установите реалистичные сроки доставки** - Слишком мало обещайте, слишком много выполняйте для удовлетворения клиентов
- **Предложите самовывоз, если это возможно** - Снижает расходы на доставку, улучшает удобство для клиентов
- **Следите за надежностью API перевозчика** - Используйте фиксированную ставку в качестве резервного варианта, если ставки в реальном времени не работают
- **Используйте зоны для международных доставок** - Разные ставки по регионам предотвращают потери на дорогих направлениях
- **Комбинируйте с акциями доставки** - Правила добавляют условную логику (бесплатные акции доставки, дополнительные сборы за удаленные районы)
- **Ограничьте количество методов** - 2-4 варианта на этапе оформления заказа предотвращают паралич решения
- **Обновляйте таблицы ставок по сезонам** - Ставки перевозчиков меняются, проверяйте ежегодно
- **Используйте приоритеты с умом** - Размещайте бесплатные/дешевые варианты в начале, дорогие варианты в конце
