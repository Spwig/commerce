---
title: 管理POS终端
---

管理POS终端是您零售运营的基础。每个终端代表一个物理设备（平板电脑、计算机或专用POS硬件），员工在此处理销售。使用仓库分配、员工授权、硬件集成和离线同步设置来配置终端。通过实时心跳跟踪监控终端状态，并在出现问题时远程解锁终端。正确的终端管理确保店内运营顺畅，并防止各地点的配置冲突。

导航至 **POS > 终端** 以注册新终端、查看在线/离线状态并管理所有终端设置。

![终端列表](/static/core/admin/img/help/managing-pos-terminals/terminal-list.webp)

## 终端列表视图

终端列表显示所有已注册的终端及其关键状态信息：

**终端名称** - 终端的描述性标签（例如，"结账1"，"主收银机"，"移动终端"）

**UUID** - 创建时自动生成的唯一标识符（用于内部设备识别）

**仓库** - 分配给此终端的物理位置（决定库存可用性和订单归属）

**在线状态** - 实时指示终端当前是否连接：
- **绿色圆点** - 在线（最近5分钟内收到心跳）
- **红色圆点** - 离线（超过5分钟未收到心跳）
- **灰色圆点** - 从未配对（终端已创建但设备从未连接）

**最后心跳** - 终端最近一次ping的时间戳（在线时每5分钟更新一次）

**配对码** - 用于初始设备配对的8位字母数字代码（首次使用后隐藏）

**分配用户** - 被授权使用此终端的员工人数

## 创建新终端

点击 **+ 添加终端** 以注册新POS设备：

![添加终端表单](/static/core/admin/img/help/managing-pos-terminals/terminal-add-form.webp)

### 基本配置

**终端名称** - 选择一个描述性名称，表明：
- 物理位置："北入口收银机"
- 功能："退货台终端"
- 序列："结账1"，"结账2"，"结账3"

名称帮助员工在轮班分配和故障排除时识别终端。在所有位置使用一致的命名约定。

**仓库** - **必填** - 选择此终端运营的仓库：
- 决定哪些库存可供销售
- 在此终端上放置的订单归属于此仓库
- 库存预订检查分配仓库的库存可用性
- **没有仓库分配将无法处理销售**

如果您有多个零售地点，请为每个地点创建单独的仓库并相应地分配终端。

**是否启用** - 切换以启用或禁用终端而不删除配置：
- 禁用的终端无法配对
- 禁用终端上的现有会话立即过期
- 用于临时禁用被盗或损坏的终端

### 员工分配

**分配用户** - 选择可以访问此终端的员工：
- 仅分配用户可以登录终端
- 用户还必须在其员工角色中具有POS权限
- 分配零用户实际上会锁定终端
- 常见模式：将所有商店员工分配给所有商店终端

**使用案例示例**：
- **普通商店**：将所有员工分配给所有终端（任何收银员都可以操作任何收银机）
- **百货商店**：将特定部门的员工分配给部门终端
- **多地点商店**：将特定地点的员工分配给地点终端
- **管理人员**：将管理分配给所有终端以获得监督访问权限

未分配终端的用户在尝试登录时会看到"未授权使用此终端"错误。

### 硬件配置

**硬件配置**字段是一个定义外围设备的JSON结构：

**热敏打印机**：
```json
{
  "printer": {
    "type": "network",
    "ip": "192.168.1.100",
    "port": 9100,
    "paper_width": 80
  }
}
```

**USB条码扫描器**：
```json
{
  "scanner": {
    "type": "usb",
    "vendor_id": "0x05e0",
    "product_id": "0x1200"
  }
}
```

**现金抽屉**（连接到打印机）：
```json
{
  "cash_drawer": {
    "enabled": true,
    "trigger": "printer_pulse"
  }
}
```

**完整示例**：
```json
{
  "printer": {
    "type": "network",
    "ip": "192.168.1.100",
    "port": 9100,
    "paper_width": 80
  },
  "scanner": {
    "type": "usb",
    "vendor_id": "0x05e0",
    "product_id": "0x1200"
  },
  "cash_drawer": {
    "enabled": true,
    "trigger": "printer_pulse"
  }
}
```

如果终端没有外围硬件（适用于没有打印机/扫描仪的移动终端或平板电脑），请留空。

### 离线缓存设置

配置终端在离线操作时缓存多少数据：

**订单同步天数**（7-30天，默认：14）：
- 本地缓存最近订单的天数
- 更高的值 = 离线时可访问更多历史数据
- 更低的值 = 同步更快，使用的存储更少
- **建议**：高流量终端使用7天，正常使用14天，审计密集型操作使用30天

**订单同步限制**（200-1000订单，默认：500）：
- 无论日期范围如何，缓存订单的最大数量
- 防止高流量终端使用过多存储
- **建议**：存储有限的平板电脑使用200，标准终端使用500，专用POS设备使用1000

**权衡**：
- **更高的设置**：更好的离线访问历史数据，初始同步更慢，使用更多存储
- **更低的设置**：更快的同步，更少的存储，有限的离线历史

终端在每次同步周期下载最近的X个订单（在Y天内）。如果终端每天处理50个订单，sync_days为14，预计缓存约700个订单（可能会达到sync_limit上限）。

## 终端配对工作流程

创建终端后，配对物理设备：

1. **生成配对码** - 保存终端时自动生成（8个字母数字字符）

2. **记录代码** - 在终端列表和详细视图中显示（首次成功配对后过期）

3. **导航到终端设备** - 在物理设备（平板电脑/计算机）上，打开浏览器并前往：`https://yourstore.com/pos/`

4. **输入配对码** - 当提示时输入8个字符的代码

5. **终端下载配置** - 设备接收：
   - 仓库分配
   - 硬件配置（打印机、扫描器、抽屉）
   - 离线缓存设置
   - 分配用户列表
   - 初始产品目录同步

6. **出现登录提示** - 终端显示分配用户的登录屏幕

7. **员工登录** - 输入分配给此终端的用户的凭据

8. **初始同步完成** - 终端下载：
   - 最近订单（按sync_days和sync_limit）
   - 分配仓库的完整产品目录
   - 客户数据库
   - 促销配置

9. **终端准备就绪** - 出现带有搜索栏的"准备销售"屏幕

10. **配对码已使用** - 代码从管理界面中移除；如果需要重新配对，请生成新代码

**配对码重新生成**：如果需要重新配对终端（设备重置、浏览器缓存清除、新硬件），使用 **重新生成配对码** 管理操作。这将使旧代码失效并生成新代码。

## 监控终端状态

### 心跳系统

终端每 **5分钟** 向服务器发送一次心跳信号，包含：
- 终端UUID
- 当前时间戳
- 在线用户数量
- 最后同步时间戳
- 服务工作线程状态

**在线状态指示器**：
- **绿色** - 最近5分钟内收到心跳（终端在线且正常运行）
- **红色** - 超过5分钟未收到心跳（终端离线或断开）
- **灰色** - 终端从未配对（从未收到心跳）

**使用案例**：
- **日常开放**：在商店开放前检查所有终端是否在线
- **故障排除**：识别哪些终端存在连接问题
- **审计**：验证营业时间内终端是否处于活动状态

### 最后心跳时间戳

显示最近一次心跳的确切日期和时间。使用此功能：
- 确定终端离线了多长时间
- 识别模式（例如，终端每天晚上关门时离线）
- 验证同步频率（在线时应每~5分钟更新一次）

## 远程解锁功能

当终端无响应或卡在某个屏幕上（软件崩溃、会话超时问题、浏览器挂起），使用 **远程解锁** 管理操作：

**工作原理**：
1. 在管理列表中选择有问题的终端
2. 从管理操作下拉菜单中选择 **远程解锁**
3. 确认操作
4. 服务器通过心跳响应发送解锁信号
5. 终端在下次心跳周期（<5分钟）接收信号
6. 终端强制登出当前用户并返回登录屏幕

**使用场景**：
- 终端卡在交易屏幕上
- 员工无法登出（登出按钮无响应）
- 会话显示活动但终端无响应
- 浏览器崩溃但会话cookie仍然存在

**重要**：远程解锁不会重启设备或浏览器——它只会强制登出并清除会话。如果终端完全冻结，员工可能需要手动重启浏览器或设备。

## 编辑终端配置

点击列表中的终端以编辑其配置：

![编辑终端表单](/static/core/admin/img/help/managing-pos-terminals/terminal-edit-form.webp)

**终端在线时可以安全更改**：
- 终端名称
- 分配用户
- 硬件配置（在终端重启应用后生效）
- 离线缓存设置（在下次同步时生效）

**需要重新配对**：
- 仓库分配（更改仓库需要重新配对以同步新库存）

**不能更改**：
- UUID（不可变标识符）

大多数设置更改将在下次心跳/同步周期生效。硬件配置更改需要员工关闭并重新打开POS应用（或刷新浏览器）。

## 常见问题排查

**终端登录时显示"未授权"**：
- 确认用户在此终端的 **分配用户** 列表中
- 确认用户在 **员工与权限 > 角色** 中具有POS权限
- 检查终端是否标记为 **启用**

**终端无法配对（无效代码）**：
- 配对码在首次使用后过期——如需，请重新生成
- 代码区分大小写——确认大小写
- 检查终端是否标记为 **启用**

**终端显示离线（红色圆点）**：
- 确认设备有互联网连接
- 检查终端是否实际在运行（浏览器打开到/pos/ URL）
- 确保防火墙没有阻止心跳请求
- 等待5分钟，直到下一个心跳周期

**终端同步缓慢**：
- 将 **订单同步天数** 从30减少到7
- 将 **订单同步限制** 从1000减少到200
- 检查终端位置的网络速度
- 确认服务器没有过载

**打印机无法工作**：
- 确认 **硬件配置** 中的打印机IP和端口
- 从终端设备测试打印机连接（ping IP地址）
- 检查打印机是否兼容ESC/POS
- 确认打印机已通电并在线

## 小贴士

- **命名约定很重要** - 使用一致的命名（位置 + 编号）以简化大规模管理
- **在配对前始终分配仓库** - 没有仓库分配，终端无法处理销售
- **在部署前测试硬件配置** - 打印测试收据以验证打印机/抽屉集成
- **每天监控心跳** - 设置例行检查以确保所有终端在商店开放时在线
- **为移动终端降低同步限制** - 平板电脑和手机从sync_days: 7, sync_limit: 200中受益
- **谨慎使用远程解锁** - 强制登出会中断正在进行的交易；确认终端确实卡住后再使用
- **记录配对码** - 在将终端部署到零售现场前写下代码（以防设置时间比预期长）
- **将管理员分配给所有终端** - 确保监督人员可以访问任何收银机以进行作废、退款和故障排除
