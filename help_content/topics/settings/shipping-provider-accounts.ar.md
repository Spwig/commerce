---
title: حسابات مزودي الشحن
---

تربط حسابات مزودي الشحن متجرك بواجهات برمجة التطبيقات (APIs) للمستثمرين (FedEx، UPS، DHL) لحساب التكلفة في الوقت الفعلي وشراء الملصقات تلقائيًا. تخزن كل حساب بيانات الاعتماد المشفرة لواجهة برمجة التطبيقات، وتراقب صحة الاتصال، وترتبط بطرق الشحن في الوقت الفعلي. تُحصل المزودون على أسعار حية أثناء الدفع بناءً على أبعاد الحزمة، والوزن، والمكان الأصلي، والوجهة - مما يلغي الحاجة إلى إعداد جداول الأسعار يدويًا ويضمن تسعير المستثمرين الدقيق.

استخدم حسابات المزودين عندما تحتاج إلى حساب تكلفة الشحن من قبل المزود أو إنشاء ملصقات تلقائيًا بدلًا من إنشاء الشحن يدويًا.

## مزودي الشحن المدعومين

يدعم Spwig المزودين الرئيسيين من خلال مكونات مزودين قابلة للتركيب:

### FedEx

**الخدمات**: الأرض، التعبير، الدولي
**واجهة برمجة التطبيقات (API)**: خدمات الويب لـ FedEx
**الميزات**: أسعار في الوقت الفعلي، شراء الملصقات، تتبع، الجمارك الدولية

### UPS

**الخدمات**: الأرض، الجو، العالم
**واجهة برمجة التطبيقات (API)**: واجهة برمجة التطبيقات لتطوير UPS
**الميزات**: أسعار في الوقت الفعلي، إنشاء الملصقات، تتبع، تحقق من العنوان

### DHL

**الخدمات**: التعبير، التجارة الإلكترونية، الدولي
**واجهة برمجة التطبيقات (API)**: واجهة برمجة التطبيقات لـ DHL Express
**الميزات**: أسعار دولية، وثائق الجمارك، تتبع

### مزودين إضافيين

قم بتثبيت المكونات من سوق المكونات حسب الحاجة (USPS، Canada Post، Australia Post، إلخ.)

## إعداد حساب مزود

تتطلب كل حساب لـ مزود:

### المعلومات الأساسية

- **اسم العرض**: كيف يظهر الحساب في لوحة الإدارة (مثلاً: "حساب FedEx الإنتاج")
- **المزود**: اختر مكون المزود المثبت من القائمة المنسدلة
- **الحالة النشطة**: مفتاح لتمكين أو تعطيل الحساب دون حذف بيانات الاعتماد
- **الافتراضي**: اضبطه كحساب افتراضي لهذا المزود (يكون فقط حساب افتراضي واحد لكل مزود)

### بيانات الاعتماد (المشفرة)

**تختلف حسب المزود**، وعادة ما تشمل:

**FedEx**:
- رقم الحساب
- رقم المتر
- مفتاح الواجهة
- سر الواجهة

**UPS**:
- رقم ترخيص الوصول
- معرف المستخدم
- كلمة المرور
- رقم الحساب

**DHL**:
- معرف الموقع
- كلمة المرور
- رقم الحساب

**كل بيانات الاعتماد مشفرة أثناء الراحة** وتنشأ فقط عند إجراء مكالمات API.

### عنوان الأصل

- **العنوان الافتراضي للشحن من**: عنوان المستودع / الأصل لحساب التكلفة
- تتطلب بعض المزودين إعدادًا محددًا للأصل في لوحة تحكمهم

### الإعدادات

خيارات مخصصة للمزود (تختلف حسب المزود):

- **وضع الاختبار**: استخدم نقاط نهاية API اختبارية / تجريبية للمزود
- **الأسعار المتفاوض عليها**: استخدم الأسعار المتفاوض عليها مع المزود (إذا كانت متاحة)
- **تضمين التأمين**: اقتباس التأمين تلقائيًا في الأسعار
- **رسوم السكن**: تطبيق رسوم التسليم إلى السكن
- **التوقيع مطلوب**: متطلبات التوقيع الافتراضية

## إنشاء حساب مزود

**عملية الإعداد بـ 6 خطوات**:

**الخطوة 1: احصل على وصول API للمزود**
1. أنشئ حساب مع المزود (FedEx.com، UPS.com، DHL.com)
2. اطلب الوصول إلى API / واجهة برمجة التطبيقات المطور
3. أكمل إعداد واجهة برمجة التطبيقات للمزود (قد يستغرق 1-3 أيام عمل)
4. تلقي بيانات الاعتماد عبر البريد الإلكتروني أو من خلال بوابة المطور

**الخطوة 2: تثبيت مكون المزود** (إذا لم يتم تثبيته مسبقًا)
1. انتقل إلى الإعدادات > المكونات > سوق المكونات
2. ابحث عن اسم المزود (مثلاً: "FedEx")
3. تثبيت مكون مزود الشحن
4. انتظر حتى اكتمال التثبيت

**الخطوة 3: إنشاء حساب مزود في Spwig**
1. انتقل إلى الإعدادات > الشحن > حسابات المزود
2. اضغط على "إضافة حساب مزود"
3. اختر المزود من القائمة المنسدلة
4. أدخل اسم العرض

**الخطوة 4: إدخال بيانات الاعتماد**
1. أملأ حقول بيانات الاعتماد (تختلف حسب المزود)
2. تُشفّر بيانات الاعتماد تلقائيًا عند الحفظ
3. خياري: تفعيل وضع الاختبار للاختبار الأولي

**الخطوة 5: اختبار الاتصال**
1. اضغط على زر "اختبار الاتصال"
2. يحاول النظام إجراء مكالمة API إلى المزود
3. تحقق من ظهور حالة "مرتبط"
4. تحقق من توقيت آخر اختبار

**الخطوة 6: ربطه بوسيلة الشحن**
1. أنشئ أو قم بتحرير وسيلة الشحن (الإعدادات > السلة > وسائل الشحن)
2. ضع method_type = "Real-Time"
3. اختر حساب المزود من القائمة المنسدلة
4. احفظ الوسيلة

## مراقبة حالة الاتصال

تتبع حسابات المزود حالة صحة الاتصال:

### قيم الحالة

**غير معروف** (رمادي): لم يتم اختباره أبدًا أو لم يتم الاتصال به بعد

**مرتبط** (أخضر): كانت المكالمة الأخيرة ناجحة، بيانات الاعتماد صحيحة

**خطأ** (أحمر): كانت المكالمة الأخيرة فاشلة، بيانات الاعتماد قد تكون غير صحيحة

### آخر اختبار

- **الوقت**: متى تم التحقق من الاتصال
- **تحديث تلقائي**: كلما تم استخدام المزود (استعلام التكلفة، شراء الملصق)
- **اختبار يدوي**: اضغط على زر "اختبار الاتصال" في أي وقت

### إصلاح الاتصالات الفاشلة

**الأسباب الشائعة**:
- بيانات الاعتماد غير صحيحة (خطأ إملائي، نسخ مع مساحة إضافية)
- مفتاح API للمزود منتهي الصلاحية أو مُلغى
- تم تفعيل وضع الاختبار ولكن يتم استخدام بيانات الاعتماد الإنتاجية (أو العكس)
- لم يتم تضمين عنوان IP مع المزود
- توقف API للمزود

**خطوات الحل**:
1. تأكد من أن بيانات الاعتماد تتطابق تمامًا مع لوحة تحكم المزود
2. تحقق من أن إعداد وضع الاختبار يتطابق مع نوع بيانات الاعتماد
3. تحقق من صفحة حالة API للمزود للحصول على تعطل
4. تواصل مع دعم المزود للتحقق من الحساب

## سير العمل لاستعلام التكلفة

كيف تعمل التكلفة في الوقت الفعلي أثناء الدفع:

**1. يدخل العميل عنوانه**
- يُدخل عنوان الشحن
- تُحسب السلة إجمالي الوزن + الأبعاد

**2. يجهز النظام طلب استعلام التكلفة**
- يحصل على بيانات الاعتماد الخاصة بالمزود (مُشفّرة)
- تُحسب أبعاد الحزمة من عناصر السلة (تستخدم حزم الشحن إذا تم تعريفها)
- يجهز طلب API مع الأصل، الوجهة، الحزم

**3. تم استدعاء API للمزود**
- يتم إرسال الطلب إلى واجهة برمجة التطبيقات للمزود مع بيانات الاعتماد
- يحسب المزود التكلفة بناءً على المنطقة، الوزن، الأبعاد
- يحتوي الاستجابة على خيارات الخدمة (الأرض، التعبير، إلخ.)

**4. تُعرض التكلفة**
- يحلل النظام استجابة المزود
- يُحولها إلى تنسيق موحد
- تُطبّق إضافات (إذا تم تكوينها)
- تُعرض التكلفة للعميل أثناء الدفع

**5. يختار العميل الخدمة**
- يختار العميل الخيار المفضل
- تُحفظ التكلفة المُحددة في الطلب

**تدفق API المثال**:
```
طلب إلى API لـ FedEx:
{
  "origin": {"postal_code": "90210", "country": "US"},
  "destination": {"postal_code": "10001", "country": "US"},
  "parcels": [{
    "weight": 2500,  // جرام
    "dimensions": {"length": 30, "width": 20, "height": 15}  // سم
  }]
}

استجابة FedEx:
[
  {"service": "FEDEX_GROUND", "rate": 12.50, "delivery_days": 5},
  {"service": "FEDEX_EXPRESS", "rate": 28.75, "delivery_days": 2}
]
```

## شراء الملصق (اختياري)

إذا دعم المزود إنشاء الملصقات:

**العملية**:
1. يكمل العميل الطلب
2. ينشئ البائع الشحن (الطلبات > تفاصيل الطلب > إنشاء الشحن)
3. اختر حساب المزود + الخدمة
4. يقوم النظام بطلب API للملصق من المزود
5. تُنشئ ملف PDF للملصق ويرتبط بالشحن
6. يُملأ رقم تتبع تلقائيًا
7. جاهز الملصق للطباعة

**المزايا**:
- لا حاجة لتسجيل الدخول إلى موقع المزود يدويًا
- يتم تزامن تتبع تلقائيًا
- تُنشئ وثائق الجمارك تلقائيًا (الدولية)
- من الممكن إنشاء ملصقات بالجملة

## إضافة التكلفة

أضف تكلفة البائع إلى تكلفة المزود:

**الإعداد** (في وسيلة الشحن، وليس حساب المزود):
- **نوع الإضافة**: نسبة مئوية أو ثابتة
- **مقدار الإضافة**: مثلاً 15% أو 2.50 دولار

**مثال**:
```
تكلفة المزود: 12.50 دولار
الإضافة: 15%
العميل يدفع: 14.38 دولار

أو

تكلفة المزود: 12.50 دولار
الإضافة: 2.50 دولار (ثابتة)
العميل يدفع: 15.00 دولار
```

**حالات الاستخدام**:
- تغطية تكاليف التعبئة والمعالجة
- إضافة هامش ربح للشحن
- تعويض رسوم البطاقات الائتمانية على الشحن

## حسابات مزود متعددة

يمكنك إنشاء حسابات متعددة لنفس المزود:

**حالات الاستخدام**:
1. **اختبار مقابل الإنتاج**
   - حساب الاختبار: بيانات الاعتماد لاختبار المزود
   - حساب الإنتاج: بيانات الاعتماد الحية

2. **مستودعات متعددة**
   - حساب المستودع A: الأصل = لوس أنجلوس
   - حساب المستودع B: الأصل = نيويورك

3. **أسعار متفاوض عليها مختلفة**
   - حساب A: أسعار عادية
   - حساب B: أسعار خصومات حجمية

**يمكن لكل حساب الارتباط بوسيلة شحن مختلفة** للتكوين المرن.

## نصائح

- **ابدأ بالاختبار في البيئة التجريبية** - استخدم بيانات الاعتماد التجريبية للمزود قبل الانتقال إلى الإنتاج
- **راقب حالة الاتصال** - تحقق من لوحة التحكم بانتظام للحصول على حالات الأخطاء
- **حدد حزم الشحن** - تحسين التقديرات من خلال أبعاد دقيقة
- **استخدم الأسعار المتفاوض عليها** - تفعيلها إذا كانت لديك خصومات حجمية مع المزود
- **حدد أصلًا واقعيًا** - استخدم عنوان الشحن الفعلي للمناطق الدقيقة
- **احتفظ ببيانات الاعتماد بأمان** - لا تشارك مفاتيح API، وقم بتغييرها بشكل دوري
- **احتفظ بوسيلة بديلة** - احتفظ بوسيلة بأسعار ثابتة نشطة إذا فشل API للمزود
- **راقب حدود API للمزود** - يحد بعض المزودين عدد مكالمات API يوميًا
- **تحديث بيانات الاعتماد فورًا** - عند تغيير مفاتيح المزود، قم بتحديثها فورًا
- **استخدم أسماء وصفية** - "FedEx LA Warehouse" أفضل من "FedEx 1"
