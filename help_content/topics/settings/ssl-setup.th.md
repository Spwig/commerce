title: การตั้งค่า SSL
---

SSL (Secure Sockets Layer) จะเข้ารหัสการเชื่อมต่อระหว่างเบราว์เซอร์ของลูกค้ากับร้านค้าของคุณ เมื่อ SSL เปิดใช้งาน URL ของร้านค้าของคุณจะเริ่มต้นด้วย `https://` และเบราว์เซอร์จะแสดงไอคอนล็อก คุณจำเป็นต้องใช้ SSL เพื่อการรับชำระเงิน การปกป้องข้อมูลลูกค้า และการมีอันดับที่ดีในเครื่องมือค้นหา

Spwig รองรับโหมด SSL หลายโหมดเพื่อให้เหมาะกับการตั้งค่าโฮสติ้งต่างๆ คู่มือนี้อธิบายแต่ละโหมดและช่วยให้คุณเลือกโหมดที่เหมาะสม

## การเลือกโหมด SSL

| โหมด | เหมาะสำหรับ | ค่าใช้จ่ายของใบรับรอง | การต่ออายุ |
|------|----------|-----------------|---------|
| **Let's Encrypt** | ร้านค้าส่วนใหญ่ | ฟรี | อัตโนมัติ |
| **Cloudflare Origin CA** | ร้านค้าที่ใช้ Cloudflare proxy | ฟรี | ทำด้วยมือ (สูงสุด 15 ปี) |
| **Custom Certificate** | ร้านค้าที่มีใบรับรองที่ซื้อมา | แตกต่างกัน | ทำด้วยมือ |
| **Managed Externally** | Load balancers, Cloudflare Flexible | N/A | N/A |
| **Self-Signed** | การพัฒนาและการทดสอบ | ฟรี | ทำด้วยมือ |
| **None (HTTP)** | การพัฒนาภายในเครือข่ายเท่านั้น | N/A | N/A |

หากคุณไม่แน่ใจว่าควรใช้โหมดใด **Let's Encrypt** เป็นตัวเลือกที่ดีที่สุดสำหรับร้านค้าส่วนใหญ่ มันฟรี ทำงานอัตโนมัติ และได้รับการไว้วางใจจากเบราว์เซอร์ทั้งหมด

## Let's Encrypt

Let's Encrypt ให้ใบรับรอง SSL ฟรีที่ต่ออายุอัตโนมัติทุก 60-90 วัน นี่คือตัวเลือกที่แนะนำสำหรับผู้ขายส่วนใหญ่

**ข้อกำหนด:**
- โดเมนของคุณต้องชี้ไปยังเซิร์ฟเวอร์ของคุณ (บันทึก A ใน DNS)
- พอร์ต 80 ต้องสามารถเข้าถึงได้จากอินเทอร์เน็ต (สำหรับการตรวจสอบใบรับรอง)
- อีเมลที่ใช้แจ้งเตือนเมื่อใบรับรองหมดอายุ

**ขั้นตอนการตั้งค่า:**
1. ไปที่ **Settings > Site Settings** และเปิดแท็บ **Domain & SSL**
2. ใส่ชื่อโดเมนของคุณ
3. เลือก **Let's Encrypt**
4. ใส่ที่อยู่อีเมลของผู้ดูแลระบบ
5. คลิก **Apply Configuration**

Spwig จะจัดการทุกสิ่งอื่นอัตโนมัติ: การตรวจสอบโดเมน การขอใบรับรอง การตั้งค่า NGINX และการตั้งค่าการต่ออายุอัตโนมัติ

## Cloudflare Origin CA

ใบรับรอง Cloudflare Origin CA จะเข้ารหัสการเชื่อมต่อระหว่างเซิร์ฟเวอร์ขอบของ Cloudflare กับร้านค้าของคุณ ใบรับรองเหล่านี้ฟรีและสามารถใช้งานได้ถึง 15 ปี แต่ใบรับรองเหล่านี้ **ได้รับการไว้วางใจโดย Cloudflare เท่านั้น** -- เบราว์เซอร์ที่เชื่อมต่อกับเซิร์ฟเวอร์ของคุณโดยตรงจะเห็นข้อความเตือนเกี่ยวกับใบรับรอง

โหมดนี้เหมาะสำหรับการใช้ Cloudflare เป็นโพรキซี่ (เปิดไอคอนเมฆสีส้ม) สำหรับโดเมนของคุณ Cloudflare จะแสดงใบรับรองที่ได้รับการไว้วางใจให้กับผู้เยี่ยมชม และใบรับรอง Origin CA จะเข้ารหัสการเชื่อมต่อระหว่าง Cloudflare กับเซิร์ฟเวอร์ของคุณ

**ข้อกำหนด:**
- บัญชี Cloudflare ที่มีโดเมนของคุณเพิ่มเข้าไปแล้ว
- ใบรับรอง Origin CA และคีย์ส่วนตัวที่สร้างจากแดชบอร์ดของ Cloudflare
- โหมด SSL/TLS ของ Cloudflare ต้องตั้งเป็น **Full (Strict)**

**การสร้างใบรับรอง Origin CA:**
1. เข้าสู่แดชบอร์ดของ Cloudflare
2. เลือกโดเมนของคุณ
3. ไปที่ **SSL/TLS > Origin Server**
4. คลิก **Create Certificate**
5. เลือก RSA หรือ ECC (RSA มีความเข้ากันได้ดีที่สุด)
6. เพิ่มโดเมนของคุณ (เช่น `example.com` และ `*.example.com`)
7. เลือกช่วงเวลาความถูกต้อง (แนะนำให้ใช้ 15 ปี)
8. คลิก **Create** และคัดลอกทั้งใบรับรองและคีย์ส่วนตัว

**การตั้งค่าใน Spwig:**
1. ไปที่ **Settings > Site Settings** และเปิดแท็บ **Domain & SSL**
2. ใส่ชื่อโดเมนของคุณ
3. เลือก **Cloudflare Origin CA**
4. วางใบรับรองลงในช่อง **Certificate (PEM)**
5. วางคีย์ส่วนตัวลงในช่อง **Private Key (PEM)**
6. คลิก **Apply Configuration**

**หลังจากตั้งค่า:**
- ใน Cloudflare ตั้งโหมด SSL/TLS เป็น **Full (Strict)**
- เปิดโพรกซี่ของ Cloudflare (ไอคอนเมฆสีส้ม) สำหรับบันทึก DNS ของโดเมนของคุณ
- ร้านค้าของคุณจะสามารถเข้าถึงได้ผ่าน HTTPS โดยใช้ใบรับรองที่ได้รับการไว้วางใจของ Cloudflare

## Custom Certificate

ใช้โหมดนี้หากคุณซื้อใบรับรอง SSL จากผู้ให้บริการใบรับรอง (CA) เช่น DigiCert, Sectigo หรือ GoDaddy หรือหากผู้ให้บริการโฮสติ้งของคุณออกใบรับรองให้คุณแล้ว

**ขั้นตอนการตั้งค่า:**
1.

ไปที่ **Settings > Site Settings** และเปิดแท็บ **Domain & SSL**
2.

ใส่ชื่อโดเมนของคุณ
3.

เลือก **Custom Certificate**
4.

รักษาการจัดรูปแบบ markdown, ค่าเส้นทางรูปภาพ, บล็อกโค้ด และคำศัพท์ทางเทคนิคทั้งหมดไว้

วางใบรับรองทั้งหมด (รวมใบรับรองอินเตอร์มีเดียต) ลงในช่อง **Certificate (PEM)**
5.

วางคีย์ส่วนตัวของคุณลงในช่อง **Private Key (PEM)**
6.

คลิก **Apply Configuration**

ใบรับรองของคุณควรมีชุดทั้งหมด: ใบรับรองโดเมนของคุณตามด้วยใบรับรองอินเตอร์มีเดียตใด ๆ คีย์ส่วนตัวควรอยู่ในรูปแบบ PEM (เริ่มต้นด้วย `-----BEGIN PRIVATE KEY-----` หรือ `-----BEGIN RSA PRIVATE KEY-----`)

## Managed Externally

เลือกโหมดนี้เมื่อ SSL ถูกปิดใช้งานโดยบริการภายนอกก่อนที่การจราจรจะไปถึงเซิร์ฟเวอร์ของคุณ ในการตั้งค่านี้ เซิร์ฟเวอร์ของคุณจะได้รับการจราจร HTTP แบบธรรมดาเท่านั้น — ไม่มีใบรับรองติดตั้งบนเซิร์ฟเวอร์เอง

**สถานการณ์ที่พบบ่อย:**
- **Cloudflare Flexible SSL** -- Cloudflare ทำการเข้ารหัสการจราจรจากเบราว์เซอร์ไปยัง Cloudflare แต่ส่ง HTTP ไปยังเซิร์ฟเวอร์ของคุณ
- **Cloud load balancers** -- AWS ALB, Google Cloud Load Balancer หรือ DigitalOcean Load Balancer ปิดใช้งาน SSL และส่ง HTTP
- **Reverse proxy** -- เซิร์ฟเวอร์อื่นที่อยู่ด้านหน้าของ Spwig ทำการจัดการ SSL

**ขั้นตอนการตั้งค่า:**
1. ไปที่ **Settings > Site Settings** และเปิดแท็บ **Domain & SSL**
2. ใส่ชื่อโดเมนของคุณ
3. เลือก **Managed Externally**
4. คลิก **Apply Configuration**

Spwig จะตั้งค่า NGINX ให้ส่ง HTTP เฉพาะและไว้ใจหัวข้อ `X-Forwarded-Proto` จากโพรกซีของคุณเพื่อตรวจจับผู้ใช้ HTTPS ได้อย่างถูกต้อง

## Self-Signed Certificate

ใบรับรองที่เซิร์ฟเวอร์ออกเองจะเข้ารหัสการเชื่อมต่อ แต่ไม่ได้รับการรับรองจากเบราว์เซอร์ ผู้เยี่ยมชมจะเห็นคำเตือนด้านความปลอดภัยที่ต้องเลี่ยงด้วยตนเอง โหมดนี้เหมาะสำหรับเซิร์ฟเวอร์พัฒนาและทดสอบภายในเท่านั้น

**ขั้นตอนการตั้งค่า:**
1. ไปที่ **Settings > Site Settings** และเปิดแท็บ **Domain & SSL**
2. ใส่ชื่อโดเมนของคุณ
3. เลือก **Self-Signed**
4. คลิก **Apply Configuration**

Spwig จะสร้างใบรับรองที่เซิร์ฟเวอร์ออกเองโดยอัตโนมัติ อย่าใช้โหมดนี้กับร้านค้าที่ใช้งานจริง

## Troubleshooting

**ใบรับรองไม่ทำงานหลังจากตั้งค่า:**
- ตรวจสอบว่า A record ของโดเมนของคุณชี้ไปยัง IP ของเซิร์ฟเวอร์ของคุณ
- ตรวจสอบให้แน่ใจว่าพอร์ต 80 และ 443 เปิดในไฟร์วอลล์ของคุณ
- รอสักพักเพื่อให้การเปลี่ยนแปลง DNS แพร่กระจาย

**Let's Encrypt ล้มเหลวในการออกใบรับรอง:**
- ตรวจสอบว่าโดเมนของคุณแก้ไขไปยัง IP ของเซิร์ฟเวอร์นี้
- ตรวจสอบให้แน่ใจว่าพอร์ต 80 ไม่ถูกบล็อกโดยไฟร์วอลล์
- หากคุณอยู่ด้านหลัง Cloudflare ให้ตั้ง DNS เป็น "DNS only" (คลาวด์สีเทา) ชั่วคราวขณะออกใบรับรอง

**Cloudflare แสดง "Error 526" (Invalid SSL Certificate):**
- ตรวจสอบให้แน่ใจว่าคุณเลือกโหมด **Cloudflare Origin CA** (ไม่ใช่ Managed Externally)
- ตรวจสอบให้แน่ใจว่าโหมด SSL/TLS ของ Cloudflare ตั้งค่าเป็น **Full (Strict)**
- ตรวจสอบว่าใบรับรอง Origin CA ยังไม่หมดอายุ

**เบราว์เซอร์แสดง "Not Secure" แม้จะมี SSL:**
- บางหน้าอาจโหลดรูปภาพหรือสคริปต์ผ่าน HTTP (เนื้อหาผสม) ตรวจสอบคอนโซลพัฒนาของเบราว์เซอร์ของคุณเพื่อหาคำเตือนเกี่ยวกับเนื้อหาผสม
- ตรวจสอบให้แน่ใจว่า URL ของเว็บไซต์ใน Settings ใช้ `https://`
