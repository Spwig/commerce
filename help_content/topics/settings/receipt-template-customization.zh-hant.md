---
title: 收據模板自定義
---

收據模板控制您POS終端打印的熱敏收據的外觀和內容。自定義標題和頁腳文字，添加您的商標，配置合規字段（稅號、商業登記號），並包含促銷二維碼。模板支持範圍定位——為所有商店創建默認模板，為區域創建特定群組的模板，或為個別位置創建特定商店的模板。系統使用範圍優先級規則來確定打印收據時應用哪個模板。

使用收據模板來保持品牌一致性，滿足區域合規要求，並通過促銷元素提高客戶參與度。

![收據模板列表](/static/core/admin/img/help/receipt-template-customization/receipt-list.webp)

## 收據模板基礎知識

收據模板定義了從ESC/POS熱敏打印機打印的收據的結構和內容。每個模板指定：

**物理配置**：
- 紙張寬度（58mm或80mm）
- 商標圖像（熱敏打印的單色）
- 字體大小和間距

**內容部分**：
- 標題文字（商店名稱、地址、聯繫信息）
- 動態交易數據（商品、價格、總計、支付方式）
- 頁腳文字（退貨政策、感謝信息、社交媒體）
- 合規字段（稅號、商業登記號）
- 帶標籤的促銷二維碼

**範圍定位**：
- 默認模板（除非被覆蓋，否則應用於所有商店）
- 群組模板（應用於群組中的所有商店）
- 商店模板（應用於特定商店/倉庫）

## 範圍優先級規則

當終端打印收據時，系統使用此層級（從最高優先級到最低）選擇模板：

| 優先級 | 範圍 | 示例 | 使用場景 |
|--------|-----|------|---------|
| **1** | 商店特定 | 巴黎商店模板 | 獨特的法國稅務合規要求 |
| **2** | 群組特定 | 歐洲商店模板 | 所有歐盟位置的增值稅顯示 |
| **3** | 默認 | 全球模板 | 所有未配置商店的備用 |

**運作方式**：
1. 檢查商店是否有專用模板（倉庫特定）
2. 如果沒有，檢查商店的群組是否有群組模板
3. 如果沒有，使用默認模板

**示例**：
- 默認模板："標準收據"（無範圍分配）
- 群組模板："歐洲收據"（分配給歐洲商店群組）- 包含增值稅登記
- 商店模板："巴黎收據"（分配給巴黎倉庫）- 包含法國SIRET號

**結果**：
- 巴黎商店終端：使用"巴黎收據"（最特定）
- 緬因州商店終端（在歐洲商店群組中，無商店模板）：使用"歐洲收據"（群組級別）
- 紐約商店終端（無群組，無商店模板）：使用"標準收據"（默認備用）

## 紙張寬度配置

熱敏收據打印機使用58mm或80mm紙張寬度。根據您的打印機硬件選擇：

| 紙張寬度 | 每行字符數 | 最佳用途 | 典型用途 |
|----------|------------|----------|-----------|
| **58mm** | ~32個字符 | 小型佔地面積，便攜 | 食品卡車、移動POS、自動販賣機 |
| **80mm** | ~48個字符 | 標準零售 | 大多數零售商店、餐廳 |

**不能混合寬度**：使用相同模板的所有終端必須具有相同紙張寬度的打印機。如果您有混合打印機類型，請為每種寬度創建單獨的模板。

**商標尺寸限制**：
- **58mm**：最大寬度384像素（建議：350px）
- **80mm**：最大寬度576像素（建議：550px）

超出最大寬度的商標會自動縮小，這可能會降低質量。

## 商標配置

收據商標必須為**單色**（僅黑色和白色）以兼容熱敏打印機：

**商標要求**：
- 文件格式：PNG、JPG或WebP
- 色彩模式：單色（黑色像素在白色背景上）
- 建議尺寸：
  - 58mm紙張：350px寬 × 100-150px高
  - 80mm紙張：550px寬 × 150-200px高
- 文件大小：<100KB（熱敏打印機內存有限）

**創建單色商標**：
1. 從您的常規商標（彩色或灰階）開始
2. 使用圖像編輯器將其轉換為純黑色和白色（無灰階）
3. 增加對比度以確保黑色元素是實體的
4. 導出為PNG格式，背景為透明或白色

**商標定位**：
- 始終水平居中
- 打印在收據頂部（在標題文字上方）
- 後跟自動間距（防止與內容過於密集）

**選擇商標**：
- 在模板表單中點擊**瀏覽媒體庫**
- 選擇單色商標資源
- 預覽顯示商標在收據上的顯示方式

**無商標**：如果希望僅使用文字品牌（標題文字可以包含商店名稱），請留空商標字段。

## 標題文字

標題文字在商標後立即顯示（或在無商標時顯示在頂部）。典型內容：

**商店名稱和地址**：
```
Your Store Name
123 Main Street
City, State 12345
Phone: (555) 123-4567
```

**營業時間**：
```
星期一至星期五：上午9點至晚上9點
星期六至星期日：上午10點至下午6點
```

**口號或口號**：
```
優質產品，卓越服務
```

**格式**：
- 使用換行符分隔信息
- 自動水平居中
- 保持行數在紙張寬度的字符限制內（58mm為32個字符，80mm為48個字符）

**可用變量**（可選）：
- `{store_name}` - 替換為倉庫名稱
- `{order_date}` - 替換為交易日期
- `{order_number}` - 替換為訂單ID

大多數商家使用靜態文本而不是變量以保持標題一致性。

## 頁腳文字

頁腳文字在交易詳情（商品、總計、支付）後顯示。典型內容：

**退貨政策**：
```
30天內憑收據退貨
僅限退貨或換貨
```

**感謝信息**：
```
感謝您與我們購物！
關注我們 @yourstore
```

**客戶服務**：
```
有問題嗎？請致電 (555) 123-4567
或郵件 support@yourstore.com
```

**格式技巧**：
- 將最重要的資訊放在前面（退貨政策、聯繫方式）
- 使用換行符提高可讀性
- 考慮在各部分之間添加分隔線（`---`）

## 合規字段

許多管轄區要求收據上包含特定資訊：

**稅號標籤** - 可自定義的稅務識別號標籤：
- 美國："Tax ID" 或 "EIN"
- 歐盟："VAT Number" 或 "VAT Reg No"
- 加拿大："GST/HST Number"
- 澳大利亞："ABN"

**稅號值** - 實際的識別號：
- 在模板中輸入一次，出現在所有收據上
- 示例："VAT Number: GB123456789"

**商業登記標籤** - 可自定義的商業登記標籤：
- 法國："SIRET"
- 德國："Handelsregister"
- 英國："Company Registration Number"

**商業登記值** - 實際的登記號：
- 示例："SIRET: 123 456 789 00010"

**顯示由Spwig提供支持** - 切換以顯示或隱藏"Powered by Spwig"品牌：
- 默認為啟用（支持平台開發）
- 禁用以進行白標操作

**各區域的合規示例**：

**歐盟**：
- 稅號標籤："VAT Number"
- 稅號值："GB123456789"
- 如果國家要求，顯示公司登記號

**美國**：
- 一般沒有收據稅號要求（因州而異）
- 可能包含B2B交易的EIN

**法國（特定）**：
- 所有收據必須包含SIRET
- 商業登記標籤："SIRET"
- 商業登記值："123 456 789 00010"

**澳大利亞**：
- 建議GST登記企業使用ABN（澳大利亞企業編號）
- 稅號標籤："ABN"

在上線前請檢查您所在地的收據要求。

## 二維碼促銷

在收據底部包含二維碼以推動客戶參與：

**二維碼URL** - 掃描後的目標地：
- 評論請求：`https://yourstore.com/reviews/leave-review`
- 忠誠度計劃：`https://yourstore.com/loyalty/join`
- 下一筆購買折扣：`https://yourstore.com/discount/THANKYOU`
- 社交媒體：`https://instagram.com/yourstore`
- 網站首頁：`https://yourstore.com`

**二維碼標籤** - 顯示在二維碼上方的文字：
- "掃描以留下評論並獲得下一筆購買的10％折扣"
- "加入我們的忠誠度計劃 - 這裡掃描"
- "關注我們的Instagram - 掃描以連接"
- "評價您的體驗"

**二維碼最佳實踐**：
- 使用短URL（長URL會產生密集、難以掃描的二維碼）
- 在部署前使用多部手機相機測試二維碼
- 在標籤中包含清晰的價值提議（掃描後客戶能得到什麼）
- 跟蹤二維碼掃描以衡量效果（使用帶追蹤參數的URL）

**動態二維碼**（高級）：
- 使用二維碼重定向服務（bit.ly, tinyurl）創建短URL
- 將重定向指向不同目的地，而不必重新打印收據
- 示例：`https://bit.ly/yourstoreqr` → 重定向到當前促銷

## 為不同範圍創建模板

**默認模板**（推薦起點）：
1. 導航至 **POS > 收據模板**
2. 點擊 **+ 添加收據模板**
3. 留空 **倉庫** 和 **商店群組** 字段（這使其成為默認模板）
4. 配置與您最常見打印機類型匹配的紙張寬度
5. 添加商標、標題、頁腳
6. 配置主要市場的合規字段
7. 儲存

此模板應用於所有商店，除非被覆蓋。

**群組模板**（用於區域差異）：
1. 創建新模板
2. 選擇 **商店群組**（例如，"歐洲商店"）
3. 留空 **倉庫**
4. 調整區域合規字段（例如，增值稅格式）
5. 調整標題文字（例如，區域地址）
6. 儲存

此模板應用於群組中的所有商店。

**商店模板**（用於位置特定需求）：
1. 創建新模板
2. 選擇 **倉庫**（例如，"巴黎商店"）
3. 調整所有字段以適應此特定位置
4. 儲存

此模板僅應用於此一個商店。

**測試模板**：
- 在終端上處理測試交易
- 打印收據
- 驗證商標清晰度、文字對齊、合規字段、二維碼可掃描性
- 如果需要，調整模板並重新測試

## 常見收據佈局

**最小收據**（食品卡車、臨時攤位）：
- 無商標（節省空間）
- 標題：僅商店名稱和電話
- 頁腳：感謝信息
- 無二維碼

**標準零售收據**：
- 商標（單色品牌標誌）
- 標題：完整商店名稱、地址、營業時間
- 合規：稅號
- 頁腳：退貨政策、感謝信息
- 二維碼：評論請求

**高級零售收據**：
- 商標（完整品牌字標）
- 標題：口號、地址、聯繫方式
- 合規：稅號、商業登記
- 頁腳：退貨政策、客戶服務、社交媒體
- 二維碼：忠誠度計劃註冊

**多位置連鎖店**：
- 默認模板：企業品牌、標準政策
- 群組模板：區域合規（歐盟的VAT、加拿大的GST）
- 商店模板：位置特定的地址和電話

## 管理多個模板

**模板命名約定**：
- 在名稱中使用範圍："Default Receipt", "EU Group Receipt", "Paris Store Receipt"
- 有助於在查看列表時識別哪個模板適用於哪個位置

**模板更改**：
- 更改會立即應用於未來的收據
- 已打印的過去收據不受影響
- 在部署到所有商店之前，在低流量終端上測試更改

**模板複製**：
- 當創建與現有模板類似的模板時，複製現有模板並修改
- 避免從頭開始創建

**刪除模板**：
- 當存在終端時，不能刪除默認模板（必須有一個備用）
- 可以刪除群組/商店模板（終端會回退到層級中的下一級）
- 刪除前確認沒有終端正在主動使用該模板

## 小技巧

- **不確定時從80mm開始** - 標準紙張寬度適用於大多數零售；58mm是專業用途
- **在實際打印機上測試商標** - 屏幕上看起來不錯的商標可能打印效果差；盡早測試
- **保持合規字段更新** - 收據上的過期稅務登記會導致法律問題
- **包含價值提議的二維碼掃描效果更好** - "掃描以獲得10％折扣"的表現比"掃描這裡"好10倍
- **審查字符限制** - 文字換行會破壞格式；在部署前計算每行的字符數
- **每種紙張寬度一個模板** - 不要將80mm模板分配給使用58mm打印機的終端（商標無法適配）
- **每月打印測試收據** - 打印機會隨著時間推移而退化；驗證質量是否仍然可接受
- **盡量少用變量** - 靜態文本比動態變量更可靠（失敗點更少）
- **備份模板配置** - 在進行重大更改前截圖或導出模板設置（方便回滾）
- **區域合規要求各不相同** - 在部署前研究當地收據要求；不遵守可能會導致嚴重罰款
