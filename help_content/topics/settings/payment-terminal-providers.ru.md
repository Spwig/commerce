Поставщики терминалов оплаты позволяют принимать оплату по кредитным и дебетовым картам на ваших терминалах POS. Stripe Terminal — основной поддерживаемый поставщик, который предлагает современные считыватели карт (S700, WisePOS E, P400), конкурентоспособные ставки обработки, и бесшовную интеграцию. Настройте учетные записи поставщиков с учетными данными API, отслеживайте статус подключения в реальном времени, и управляйте несколькими поставщиками, если вы работаете в разных регионах. Система поставщиков расширяема — дополнительные платежные обработчики могут быть интегрированы через фреймворк поставщиков, если Stripe Terminal не работает в вашем регионе.

Используйте поставщиков оплаты для безопасного принятия оплаты по картам, отслеживания статуса обработки платежей и управления назначением считывателей на терминалах.

## Обзор поставщиков оплаты

Поставщики оплаты — это сторонние сервисы, которые обрабатывают платежи по картам от имени вашего бизнеса:

**Обязанности поставщика**:
- Авторизация транзакций по картам в реальном времени
- Общение с физическими считывателями карт
- Обеспечение безопасности платежей (соответствие PCI, шифрование)
- Перечисление средств на ваш банковский счет (рассчет)
- Предоставление отчетности по транзакциям и управление спорами

**Роль Spwig**:
- Перенаправляет запросы на оплату на настроенный поставщик
- Хранит зашифрованные учетные данные поставщика
- Отслеживает статус подключения
- Связывает считыватели с терминалами
- Записывает результаты оплаты в заказы

## Stripe Terminal (основной поставщик)

Stripe Terminal — рекомендуемый поставщик оплаты для большинства продавцов:

**Функции**:
- Современные считыватели карт с чипом EMV
- Поддержка безконтактных платежей (NFC) (Apple Pay, Google Pay, карты с касанием)
- Интегрированное управление спорами
- Авторизация в реальном времени
- Дружественный для разработчиков API
- Доступен в 40+ странах

**Тарифы** (на 2024 год, проверьте текущие ставки):
- Ставки за транзакции: 2,7% + $0,05 за каждую личную транзакцию (США)
- Нет ежемесячных платежей, нет платы за настройку, нет платы за соответствие PCI
- Считыватели карт: одноразовая покупка ($59–$299 в зависимости от модели)

**Поддерживаемые регионы**:
- США, Канада, Великобритания, Европейский Союз, Австралия, Сингапур и другие
- Проверьте доступность Stripe: https://stripe.com/terminal

**Поддерживаемые считыватели**:
- BBPOS WisePOS E (все в одном Android-терминале)
- Stripe Reader S700 (считыватель для стойки)
- Verifone P400 (устаревший считыватель, но все еще поддерживается)

## Настройка Stripe Terminal

**Шаг 1: Создание учетной записи Stripe**
- Зарегистрируйтесь на stripe.com
- Завершите проверку бизнеса (банковский счет, ИНН)
- Активируйте платежи

**Шаг 2: Включение Stripe Terminal**
- В панели управления Stripe перейдите в **Products > Terminal**
- Нажмите **Get Started**
- Примите условия использования Terminal

**Шаг 3: Создание локации**
- Stripe Terminal требует "Location", представляющей вашу физическую торговую точку
- Перейдите в **Terminal > Locations**
- Нажмите **Create Location**
- Введите адрес магазина и детали
- Сохраните идентификатор локации (выглядит как `tml_1ABC123...`)

**Шаг 4: Генерация ключа API**
- Перейдите в **Developers > API Keys**
- Найдите ваш **Secret Key** (начинается с `sk_live_...` для продаж, `sk_test_...` для тестирования)
- Скопируйте секретный ключ (не делитесь его публично)

**Шаг 5: Настройка в Spwig**
- Перейдите в **POS > Payment Providers**
- Нажмите **+ Add Payment Provider**
- Выберите **Provider**: "Stripe Terminal"
- Введите **API Secret Key** (из шага 4)
- Введите **Location ID** (из шага 3)
- Сохраните

**Шаг 6: Тестирование подключения**
- После сохранения статус поставщика должен измениться на "Connected" (зеленый)
- Если статус показывает "Error" (красный), проверьте ключ API и идентификатор локации
- Проверьте сообщение об ошибке в представлении деталей поставщика

## Поля настройки поставщика

**Provider Key** - Выберите платежный обработчик:
- **stripe_terminal** - Stripe Terminal (рекомендуется)
- **manual** - Ручной ввод платежа (только для тестирования, без фактической обработки)
- Дополнительные поставщики могут появиться, если установлены через систему компонентов

**Credentials (Encrypted)** - Структура JSON, содержащая учетные данные API:
- Автоматически шифруется перед хранением
- Никогда не отображается в виде обычного текста после сохранения
- Пример структуры (Stripe Terminal):
```json
{
  "api_key": "sk_live_ABC123...",
  "location_id": "tml_1ABC123..."
}
```

**Provider Settings** - Дополнительные настройки (специфичные для поставщика):
- Описание выписки (отображается на счете кредитной карты клиента)
- Автоматическое захвачение (немедленное захват авторизованных платежей вместо ручного захвата)
- Переопределение валюты (если учетная запись поставщика использует другую валюту, чем магазин)

**Connection Status** - Индикатор статуса в реальном времени:
- **Connected** (зеленый) - Поставщик доступен и настроен правильно
- **Error** (красный) - Подключение не удалось или учетные данные неверны
- **Unknown** (серый) - Еще не протестировано (немедленно после создания)

**Last Tested** - Временная метка последнего теста подключения
- Обновляется автоматически при обработке транзакций
- Вручную запустите тест через административное действие **Test Connection**

## Мониторинг статуса подключения

Система отслеживает подключение к поставщику, чтобы предупредить вас о проблемах до того, как клиенты попытаются оплатить:

**Автоматические тесты**:
- Каждая транзакция оплаты запускает тест подключения (по необходимости)
- Задача в фоновом режиме проверяет подключение каждые 6 часов (предотвращающее мониторинг)

**Значения статуса**:

**Connected** - API поставщика доступен, учетные данные действительны, готовы к обработке платежей

**Error** - Общие причины:
- Недействительный ключ API (отозван, истек срок действия, или неправильный)
- Недействительный идентификатор локации (локация удалена в Stripe, введен неправильный ID)
- Проблемы с сетевым подключением (ограничение брандмауэра для API Stripe)
- Выход из строя сервиса Stripe (редко)

**Unknown** - Поставщик никогда не тестировался (новая учетная запись, ожидает первой транзакции)

**Решение проблемы с ошибкой**:
1. Проверьте сообщение об ошибке в представлении деталей поставщика (объясняет конкретную проблему)
2. Проверьте, действителен ли ключ API в панели управления Stripe
3. Проверьте, существует ли идентификатор локации в панели управления Stripe
4. Проверьте подключение вручную через административное действие **Test Connection**
5. Обновите учетные данные при необходимости

## Сравнение поддерживаемых считывателей карт

Stripe Terminal предлагает несколько вариантов оборудования считывателей карт:

| Модель | Тип | Методы оплаты | Дисплей | Лучше всего подходит | Цена |
|-------|------|-----------------|---------|----------|-------|
| **WisePOS E** | Все в одном | EMV чип, NFC, прокрутка | 5" цветной сенсорный экран | Полнофункциональный розничный POS | ~$299 |
| **S700** | Стол | EMV чип, NFC, прокрутка | Черно-белый LCD | Стандартная розничная касса | ~$249 |
| **P400** | Стол | EMV чип, NFC, прокрутка | Черно-белый LCD | Устаревшие развертывания | ~$299 |

**Преимущества WisePOS E**:
- На базе Android (запускает приложения, может отображать пользовательский контент)
- Цветной сенсорный экран (лучший UX для запросов чаевых, захвата подписи)
- Интегрированный принтер чеков (опционально)
- Самый высокий скорость транзакции

**Преимущества S700**:
- Ниже стоимость, чем у WisePOS E
- Компактный размер
- Устойчивый к брызгам дизайн

**P400** (старая модель):
- Все еще поддерживается, но не рекомендуется для новых развертываний
- Медленнее обработка карт с чипом, чем у S700/WisePOS E

Все считыватели подключаются к POS Spwig через API Stripe Terminal (не требуется прямое подключение USB/Bluetooth к устройству POS).

## Рассмотрение безопасности

**Шифрование учетных данных**:
- Все учетные данные поставщика шифруются в базе данных
- Шифрование использует секретный ключ приложения (определен в настройках приложения)
- Учетные данные никогда не отображаются в журналах или сообщениях об ошибках

**Разрешения ключа API**:
- Используйте **ограниченные ключи API** в продажах (ограничьте разрешения только для Terminal)
- Не используйте неограниченные секретные ключи (широкий доступ, чем требуется, представляет риск безопасности)
- В панели управления Stripe создайте ограниченный ключ с разрешениями только для **Terminal**

**Соответствие PCI**:
- Stripe Terminal обрабатывает соответствие PCI (данные карты никогда не касаются серверов Spwig)
- Номера карт обрабатываются полностью на оборудовании считывателя → серверы Stripe → сети карт
- Spwig хранит только результаты оплаты (одобрены/отклонены), никогда не детали карт

**Обновление ключей**:
- Обновляйте ключи API ежегодно в качестве лучшей практики безопасности
- При обновлении обновите учетные данные в настройке поставщика
- Старые ключи могут быть отозваны в панели управления Stripe после подтверждения, что новый ключ работает

## Несколько поставщиков

Некоторые продавцы нуждаются в нескольких учетных записях поставщиков:

**Операции с несколькими валютами**:
- Магазины в США используют учетную запись Stripe US (обрабатывает USD)
- Магазины в Европе используют учетную запись Stripe EU (обрабатывает EUR)
- Настройте отдельный поставщик на каждый язык

**Резервные поставщики**:
- Основной поставщик (Stripe Terminal)
- Резервный поставщик (ручной ввод) при сбое считывателя
- Кассир выбирает поставщика при инициировании оплаты

**Тестирование против продажи**:
- Поставщик для тестирования с API-ключом `sk_test_...`
- Поставщик для продажи с API-ключом `sk_live_...`
- Переключите поставщиков после фазы тестирования

## Устранение распространенных проблем

**Проблема 1: Статус показывает "Error" с сообщением "Недействительный ключ API"**
- **Причина**: Ключ API отозван или неправильно скопирован
- **Решение**: Сгенерируйте новый ключ API в панели управления Stripe, обновите учетные данные поставщика, протестируйте подключение

**Проблема 2: Считыватель не обнаружен во время оплаты**
- **Причина**: Считыватель не зарегистрирован в локации поставщика
- **Решение**: В панели управления Stripe проверьте, зарегистрирован ли считыватель в той же локации, что и используется в конфигурации поставщика

**Проблема 3: Оплата отклонена, несмотря на действительную карту**
- **Причина**: Учетная запись Stripe не полностью активирована (ожидается проверка)
- **Решение**: Завершите проверку бизнеса в панели управления Stripe (банковский счет, ИНН)

**Проблема 4: Статус подключения показывает "Unknown" и никогда не обновляется**
- **Причина**: Поставщик никогда не тестировался (не было попыток транзакции)
- **Решение**: Используйте административное действие **Test Connection**, чтобы вручную запустить тест подключения

## Советы

- **Тестовый режим перед продажей** - Используйте тестовые API-ключи Stripe (`sk_test_...`) для первоначальной настройки и тестирования
- **Один поставщик на валюту** - Не пытайтесь обрабатывать EUR с учетной записью Stripe, ориентированной на USD; создайте отдельные поставщики
- **Еженедельный мониторинг статуса подключения** - Предотвращение сбоев оплаты в кассе через проактивный мониторинг
- **Ограничение разрешений ключа API** - Ограничьте ключи API Stripe только разрешениями Terminal (принцип наименьших привилегий)
- **Документирование идентификаторов локаций** - Сохраните запись о том, какая локация Stripe соответствует какой физической магазину
- **Тестирование назначения считывателя** - После настройки поставщика протестируйте оплату с использованием реального считывателя, чтобы проверить полный поток
- **Обновление контактной информации Stripe** - Убедитесь, что контактная информация в Stripe соответствует текущей (важно для споров, соответствия)
