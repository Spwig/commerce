---
title: Панель аналитики поиска
---

Панель аналитики поиска отслеживает каждый поисковый запрос на вашем магазине, предоставляя информацию о том, что ищут клиенты, какие поиски успешны или неудачны, и насколько быстро ваша система поиска реагирует. Используйте эти данные для выявления популярных продуктов, обнаружения недостающих товаров, создания синонимов и оптимизации производительности поиска.

Для отображения данных необходимо включить отслеживание аналитики в **Настройки поиска > вкладка Аналитика**.

![Панель аналитики](/static/core/admin/img/help/search-analytics-dashboard/analytics-dashboard.webp)

## Обзор панели

Перейдите в **Поиск > Аналитика поиска**, чтобы получить доступ к панели. Страница показывает:

**Карты статистики** - Быстрые метрики за сегодня и за прошлую неделю:
- Общее количество поисков сегодня
- Общее количество поисков за эту неделю
- Запросы без результатов (поиски, возвращающие ноль продуктов)
- Среднее время отклика в миллисекундах

**Таблица популярных запросов** - самые частые поисковые термины с количеством результатов

**Запросы без результатов** - поиски, которые не возвращали результатов (критично для улучшения)

**Список запросов** - все отдельные записи поиска с фильтрами

## Статистика за сегодня

**Общее количество поисков сегодня** - количество всех запросов поиска с полуночи по вашему часовому поясу магазина. Включает как автодополнение, так и полные запросы страницы поиска.

**Уникальные запросы сегодня** - количество различных поисковых терминов, использованных сегодня. Если 5 клиентов ищут "ноутбук", это считается 1 уникальным запросом, несмотря на 5 общих поисков.

**Запросы без результатов сегодня** - поиски сегодня, которые не возвращали продуктов. Высокое количество запросов без результатов указывает на отсутствие продуктов или плохое покрытие синонимами.

Данные обновляются в реальном времени по мере выполнения поисков.

## Статистика за неделю

**Общее количество за неделю** - общее количество поисков за последние 7 дней

**Уникальные запросы** - различные поисковые термины, использованные этой неделей

**Рост по сравнению с предыдущей неделей** - процентное изменение по сравнению с предыдущей неделей (если отображается)

Используйте еженедельные данные для выявления трендов: увеличение объема поиска часто коррелирует с ростом трафика или маркетинговыми кампаниями.

## Среднее время отклика

⚠️ **МОНИТОРИНГ ПРОИЗВОДИТЕЛЬНОСТИ**

Среднее время (в миллисекундах) выполнения поисковых запросов. Целевые времена отклика:

| Тип запроса | Цель | Порог предупреждения |
|-------------|------|----------------------|
| Автодополнение | < 200мс | > 300мс постоянно |
| Полный поиск | < 500мс | > 800мс постоянно |

Если среднее время отклика превышает пороги предупреждений:
1. Проверьте **Настройки поиска > вкладка Кэширование** - увеличьте TTL кэша
2. Просмотрите **вкладка Глубокого индексирования** - отключите затратные функции (индексирование документов, индексирование отзывов на больших каталогах)
3. Посмотрите руководство [Оптимизация производительности поиска](/en/admin/help/search-performance-optimization/)

## Популярные запросы

Таблица популярных запросов показывает наиболее часто используемые поисковые термины:

**Используйте эти данные для**:
- **Продвижения популярных продуктов** - если "беспроводные наушники" являются популярным запросом, выделите эти продукты на главной странице
- **Решение о запасах** - высокий объем поиска в категории указывает на спрос
- **Выявление трендов** - сезонные поиски раскрывают, что сейчас популярно
- **Создание контента** - напишите блоги или руководства о часто искомых темах

Рассматривайте популярные запросы ежемесячно, чтобы выровнять вашу торговлю с интересами клиентов.

## Запросы без результатов

**КРИТИЧНО ДЛЯ УЛУЧШЕНИЯ** - запросы без результатов - это золотая жила для оптимизации вашего магазина.

Запросы без результатов возникают по трём основным причинам:

### 1. Отсутствие продуктов

Клиенты ищут продукты, которые вы не продаете.

**Пример**: Повторяющиеся поиски "фитнес-маты", но вы продаете только спортивное оборудование, а не товары для йоги.

**Действие**: Рассмотрите возможность добавления этих продуктов в ваш каталог, если поиски часто повторяются.

### 2. Отсутствие синонимов

Клиенты используют термины, которые не совпадают с описаниями ваших продуктов.

**Пример**: Клиенты ищут "ноутбук", но все ваши продукты называются "портативный компьютер".

**Действие**: Создайте синонимы, сопоставляющие термины клиентов с вашим языком продуктов. См. [Управление синонимами и перенаправлениями](/en/admin/help/managing-synonyms-redirects/).

### 3. Плохое фазовое сопоставление

Опечатки или ошибки в написании не совпадают даже при включённом фазовом поиске.

**Пример**: Поиск "accomodate" не находит продукты "accommodate".

**Действие**:
- Уменьшите порог схожести в **Настройки поиска > вкладка Фазовое сопоставление** (с 0,80 до 0,75)
- Добавьте однонаправленные синонимы для распространённых опечаток

**Еженедельная рабочая нагрузка**:
1. Проверяйте запросы без результатов каждую понедельник
2. Классифицируйте: отсутствие продуктов, отсутствие синонимов или опечатки
3. Добавьте синонимы для часто искомых терминов
4. Заметьте пробелы в ассортименте для планирования запасов

## Детали запроса

Нажмите на любой запрос в списке, чтобы просмотреть полные детали:

**Отслеживаемые поля**:
- **Текст запроса** - что ищет клиент
- **Время** - когда произошёл поиск
- **Количество результатов** - сколько результатов было возвращено
- **Время отклика** - миллисекунды для выполнения (мониторинг производительности)
- **Пользователь** - зарегистрированный клиент (если включено отслеживание пользователей)
- **Идентификатор сеанса** - анонимный идентификатор сеанса
- **Язык** - язык магазина во время поиска
- **Двигатель** - какой движок поиска обработал запрос

## Фильтрация и поиск

Используйте фильтры для анализа конкретных сегментов:

**Иерархия дат** - фильтрация по дате, месяцу или году

**Фильтр языка** - просмотр поисков по языку (ценный для магазинов с несколькими языками)

**Фильтр двигателя** - сравнение поведения поиска между различными двигателями

**Переключатель запросов без результатов** - показывать только запросы, которые возвращали ноль результатов

**Поле поиска** - найти конкретный текст запроса

## Экспорт данных

Нажмите **Экспортировать**, чтобы загрузить данные запросов в формате CSV для более глубокого анализа в Excel или инструментах обработки данных.

**CSV включает**:
- Все тексты запросов
- Время
- Количество результатов
- Время отклика
- Данные о языке и двигателе

Используйте экспорты для:
- Анализа трендов со временем
- Выявления сезонных паттернов поиска
- Аудита производительности
- Представления данных заинтересованным сторонам

## Рассмотрение конфиденциальности

Отслеживание аналитики поиска уважает конфиденциальность:

**Отслеживание пользователей** (опционально) - связывает поиски с учетными записями зарегистрированных клиентов. Отключите для соответствия GDPR/CCPA в **Настройки поиска > вкладка Аналитика**.

**Отслеживание сеансов** (по умолчанию) - использует анонимные идентификаторы сеансов для отслеживания паттернов поиска без идентификации клиентов. Конфиденциальность обеспечена.

**Хранение данных** - поисковые запросы остаются в базе данных вечно. Реализуйте собственную политику хранения, если требуется соответствие требованиям.

## Использование аналитики для улучшения поиска

Действенные инсайты из аналитики поиска:

**Еженедельные задачи**:
- Проверьте запросы без результатов и добавьте синонимы для часто используемых терминов
- Отслеживайте время отклика и оптимизируйте, если оно постоянно медленное
- Определите популярные поиски и убедитесь, что эти продукты хорошо запасены

**Ежемесячные задачи**:
- Проанализируйте популярные запросы для информирования выбора продуктов
- Экспортируйте данные для выявления сезонных трендов
- Проверьте паттерны поиска, специфичные для языка
- Отслеживайте количество переходов по перенаправлениям для оптимизации сокращений в навигации

**Ежеквартальные задачи**:
- Проверьте эффективность синонимов (снизились ли запросы без результатов?)
- Сравните рост объема поиска с общим трафиком
- Проведите A/B-тестирование изменений веса и измерьте релевантность результатов
- Проверьте, следует ли добавлять новые категории продуктов на основе спроса поиска

## Советы

- **Запросы без результатов - это золотая жила для улучшения** - они напрямую говорят вам о том, чего клиенты хотят, но вы не предоставляете
- **Проверяйте аналитику в понедельник утром** - начните неделю с оптимизации на основе данных предыдущей недели
- **Время отклика >300мс постоянно = исследуйте** - сначала проверьте настройки кэширования, затем функции глубокого индексирования
- **Экспортируйте CSV для анализа трендов** - анализ в электронных таблицах раскрывает паттерны, которые неочевидны в интерфейсе администратора
- **Создайте синонимы до добавления продуктов** - если клиенты ищут "чехлы для планшетов", но вы называете их "защитные обложки", сначала добавьте синоним
- **Отслеживайте сезонные паттерны поиска** - "Зимние сапоги" в октябре, "купальники" в марте - запасайте товары соответственно
