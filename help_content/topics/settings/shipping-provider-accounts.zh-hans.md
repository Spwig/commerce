---
title: 运输提供商账户
---

运输提供商账户将您的商店连接到承运商API（如FedEx、UPS、DHL），以实现实时运费计算和自动标签购买。每个账户存储加密的API凭据，监控连接状态，并链接到实时运输方式。提供商根据包裹尺寸、重量、原产地和目的地在结账时获取实时运费——消除了手动维护运费表的需要，并确保准确的承运商定价。

当您需要承运商计算的运费或自动标签生成，而不是手动创建运输时，请使用提供商账户。

## 受支持的运输提供商

Spwig通过可安装的提供商组件支持主要承运商：

### FedEx

**服务**: 地面运输、快递、国际运输
**API**: FedEx Web Services
**功能**: 实时运费、标签购买、跟踪、国际清关

### UPS

**服务**: 地面运输、空运、全球运输
**API**: UPS Developer API
**功能**: 实时运费、标签生成、跟踪、地址验证

### DHL

**服务**: 快递、电子商务、国际运输
**API**: DHL Express API
**功能**: 国际运费、清关文件、跟踪

### 其他提供商

需要时从组件市场安装（如USPS、加拿大邮政、澳大利亚邮政等）

## 提供商账户配置

每个提供商账户需要：

### 基本信息

- **显示名称**: 账户在管理界面中的显示方式（例如，"FedEx生产账户"）
- **提供商**: 从下拉菜单中选择已安装的提供商组件
- **启用**: 切换以启用或禁用账户，而无需删除凭据
- **设为默认**: 将此提供商设为默认账户（每个提供商只有一个默认账户）

### API凭据（加密）

**因提供商而异**，通常包括：

**FedEx**:
- 账户编号
- 计量编号
- API密钥
- API密钥

**UPS**:
- 访问许可证编号
- 用户ID
- 密码
- 账户编号

**DHL**:
- 站点ID
- 密码
- 账户编号

**所有凭据在静态存储时均加密**，仅在进行API调用时解密。

### 原始地址

- **默认发货地址**: 用于运费计算的仓库/原产地地址
- 一些提供商需要在他们的仪表板中设置特定的原产地

### 设置

提供商特定的选项（因承运商而异）：

- **测试模式**: 使用承运商的沙箱/测试API端点
- **协商费率**: 使用您与承运商协商的费率（如果可用）
- **包含保险**: 在费率中自动报价保险
- **住宅附加费**: 应用住宅配送费用
- **需要签名**: 默认签名要求

## 创建提供商账户

**6步设置流程**：

**步骤1：获取承运商API访问权限**
1. 在承运商处创建账户（FedEx.com、UPS.com、DHL.com）
2. 申请API/开发人员访问权限
3. 完成承运商的API入职流程（可能需要1-3个工作日）
4. 通过电子邮件或开发人员门户接收API凭据

**步骤2：安装提供商组件**（如果未预安装）
1. 转到设置 > 组件 > 市场
2. 搜索承运商名称（例如，"FedEx"）
3. 安装运输提供商组件
4. 等待安装完成

**步骤3：在Spwig中创建提供商账户**
1. 导航到设置 > 运输 > 提供商账户
2. 点击"添加提供商账户"
3. 从下拉菜单中选择提供商
4. 输入显示名称

**步骤4：输入API凭据**
1. 填写凭证字段（因提供商而异）
2. 凭证在保存时自动加密
3. 可选：启用测试模式进行初始测试

**步骤5：测试连接**
1. 点击"测试连接"按钮
2. 系统尝试向承运商发出API调用
3. 验证"已连接"状态是否出现
4. 检查last_tested_at时间戳

**步骤6：链接到运输方式**
1. 创建或编辑运输方式（设置 > 购物车 > 运输方式）
2. 设置method_type = "实时"
3. 从下拉菜单中选择提供商账户
4. 保存方法

## 连接状态监控

提供商账户跟踪连接状态：

### 状态值

**未知**（灰色）：从未测试过或尚未连接

**已连接**（绿色）：最后一次API调用成功，凭据有效

**错误**（红色）：最后一次API调用失败，凭据可能无效

### 最后测试时间

- **时间戳**：连接最后验证的时间
- **自动更新**：每次使用提供商时（获取运费、购买标签）
- **手动测试**：随时点击"测试连接"按钮

### 修复失败的连接

**常见原因**：
- 错误的API凭据（拼写错误，复制时带有额外空格）
- 承运商API密钥过期或被撤销
- 启用了测试模式但使用生产凭据（或相反）
- IP地址未在承运商处白名单
- 承运商API停机

**解决方案步骤**：
1. 确认凭据与承运商仪表板完全匹配
2. 检查测试模式设置是否与凭据类型匹配
3. 查看承运商的API状态页面以了解停机情况
4. 联系承运商支持以验证账户

## 运费查询工作流程

结账时实时运费的工作方式：

**1. 客户输入地址**
- 输入运输地址
- 购物车计算总重量 + 尺寸

**2. 系统准备运费请求**
- 获取提供商账户凭据（解密）
- 从购物车商品计算包裹尺寸（如果定义了运输包裹）
- 准备包含原产地、目的地、包裹的API请求

**3. 调用提供商API**
- 向承运商API发送请求并附上认证凭据
- 承运商根据区域、重量、尺寸计算运费
- 响应包括服务选项（地面运输、快递等）

**4. 显示运费**
- 系统解析承运商响应
- 标准化为标准格式
- 如果配置了，应用附加费
- 在结账时向客户显示运费

**5. 客户选择服务**
- 客户选择首选选项
- 选定的运费保存到订单中

**示例API流程**：
```
向FedEx API的请求：
{
  "origin": {"postal_code": "90210", "country": "US"},
  "destination": {"postal_code": "10001", "country": "US"},
  "parcels": [{
    "weight": 2500,  // 克
    "dimensions": {"length": 30, "width": 20, "height": 15}  // 厘米
  }]
}

FedEx响应：
[
  {"service": "FEDEX_GROUND", "rate": 12.50, "delivery_days": 5},
  {"service": "FEDEX_EXPRESS", "rate": 28.75, "delivery_days": 2}
]
```

## 标签购买（可选）

如果提供商支持标签生成：

**工作流程**：
1. 客户完成订单
2. 商家创建运输（订单 > 订单详情 > 创建运输）
3. 选择提供商账户 + 服务
4. 系统调用提供商的标签API
5. 生成标签PDF并附加到运输
6. 自动填写跟踪号
7. 标签准备打印

**优势**：
- 无需手动登录承运商网站
- 跟踪信息自动同步
- 自动生成清关文件（国际运输）
- 可以批量生成标签

## 运费附加费

向承运商运费添加商家附加费：

**配置**（在运输方式中，而不是提供商账户中）：
- **附加费类型**: 百分比或固定金额
- **附加费金额**: 例如，15%或$2.50

**示例**：
```
承运商运费：$12.50
附加费：15%
客户支付：$14.38

或者

承运商运费：$12.50
附加费：$2.50（固定）
客户支付：$15.00
```

**使用场景**：
- 覆盖包装/处理成本
- 向运费添加利润边际
- 抵消运费上的信用卡费用

## 多个提供商账户

您可以为同一提供商创建多个账户：

**使用场景**：
1. **测试与生产**
   - 测试账户：承运商沙箱凭据
   - 生产账户：实时凭据

2. **多个仓库**
   - 仓库A账户：原产地 = 洛杉矶
   - 仓库B账户：原产地 = 纽约

3. **不同的协商费率**
   - 账户A：标准费率
   - 账户B：批量折扣费率

**每个账户可以链接到不同的运输方式**，以实现灵活的配置。

## 小贴士

- **先在沙箱中测试** - 在使用实时凭据之前，使用承运商的测试凭据
- **监控连接状态** - 定期检查仪表板上的错误状态
- **定义运输包裹** - 准确的尺寸可提高运费报价
- **使用协商费率** - 如果您与承运商有批量折扣，请启用
- **设置现实的原产地** - 使用实际的发货地址以获得准确的区域
- **保持凭据安全** - 永远不要共享API密钥，定期轮换
- **保留备用方法** - 如果承运商API失败，请保持固定费率方法启用
- **监控承运商API限制** - 一些承运商限制每天的API调用次数
- **及时更新凭据** - 当承运商轮换密钥时，请立即更新
- **使用描述性名称** - "FedEx洛杉矶仓库"比"FedEx 1"更好
