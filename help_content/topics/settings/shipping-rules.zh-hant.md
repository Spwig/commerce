---
title: 運費規則
---

運費規則根據購物車內容、客戶屬性和配送區域對運費方法應用條件性費用調整，自動提供超過 $50 免運費、為遠程地區增加附加費用或為 VIP 顧客提供運費折扣。規則使用基於優先級的執行方式（優先級越高越先執行），並具有可選的停止標誌以防止進一步處理。每個規則評估多個條件（購物車金額、重量、區域、產品、客戶群組），並在所有條件匹配時執行其中一種 6 種調整類型。

當您在需要根據訂單上下文動態調整運費，而不僅僅是靜態的運費方法時，請使用運費規則。

## 運費規則類型

運費規則應用 6 種費用調整類型：

### 百分比折扣

**功能**：按百分比降低運費（例如 25% 折扣）。

**公式**：`new_cost = base_cost × (1 - percent/100)`

**範例**：
```
基礎費用：$20
折扣：25%
結果：$15
```

**使用場景**：
- VIP 顧客折扣（所有運費 20% 折扣）
- 季節性促銷（12 月運費 15% 折扣）
- 批量訂購折扣（5 項以上運費 10% 折扣）

---

### 固定折扣

**功能**：從運費中減去固定金額。

**公式**：`new_cost = base_cost - amount`（最低 $0）

**範例**：
```
基礎費用：$15
折扣：$5
結果：$10
```

**使用場景**：
- 首次顧客獎勵（首筆訂單運費 $5 折扣）
- 訂閱電子報獎勵（運費 $3 折扣）
- 會員制度好處（每月運費 $10 折扣）

---

### 設定費用

**功能**：將運費設定為特定金額。

**公式**：`new_cost = fixed_amount`

**範例**：
```
基礎費用：$25
設定為：$9.99
結果：$9.99
```

**使用場景**：
- 限時促銷（今天所有訂單平價 $5 運費）
- 分類特定運費（書籍運費始終為 $3.99）
- 時間基準促銷（本周運費上限為 $9.99）

---

### 免運費

**功能**：將運費設定為 $0。

**公式**：`new_cost = $0`

**範例**：
```
基礎費用：$18
規則適用
結果：$0
```

**使用場景**：
- 超過 $50 免運費
- 特定產品免運費（促銷產品）
- VIP 顧客免運費
- 3 項以上訂單免運費

---

### 附加費用（固定）

**功能**：向運費增加固定金額。

**公式**：`new_cost = base_cost + amount`

**範例**：
```
基礎費用：$12
附加費用：$5
結果：$17
```

**使用場景**：
- 遠程地區配送費用
- 超大物品處理費
- 周六配送附加費用
- 易碎物品包裝費用

---

### 附加費用（百分比）

**功能**：按百分比增加運費。

**公式**：`new_cost = base_cost × (1 + percent/100)`

**範例**：
```
基礎費用：$20
附加費用：15%
結果：$23
```

**使用場景**：
- 高峰季節附加費用（節日期間 20%）
- 快速配送附加費用（50% 附加費用）
- 燃料附加費用（根據當前費用變動）

---

## 規則條件

規則評估 **所有條件必須通過** 才能應用：

### 時間有效性

- **開始日期**：規則僅在該日期後生效
- **結束日期**：規則僅在該日期前生效
- **使用場景**：季節性促銷、限時優惠

**範例**：僅限黑色星期五週末免運費
```
開始：2026-11-27 00:00
結束：2026-11-30 23:59
```

---

### 購物車金額範圍

- **最低購物車金額**：購物車小計必須 ≥ 金額
- **最高購物車金額**：購物車小計必須 ≤ 金額
- **使用場景**：免運費門檻、分層折扣

**範例**：訂單 $50-$200 免運費
```
最低：$50
最高：$200
```

---

### 購物車重量範圍

- **最低重量**：購物車總重量必須 ≥ 金額
- **最高重量**：購物車總重量必須 ≤ 金額
- **使用場景**：輕量貨物折扣、重物附加費用

**範例**：超過 20kg 的訂單附加 $5 費用
```
最低重量：20kg
最高重量：null（無限制）
```

---

### 項目數量範圍

- **最低項目數量**：購物車必須包含 ≥ 數量的項目
- **最高項目數量**：購物車必須包含 ≤ 數量的項目
- **使用場景**：批量訂購折扣、單項目費用

**範例**：5 項以上免運費
```
最低項目：5
最高項目：null
```

---

### 配送區域

- **區域**：僅當客戶地址與至少一個選定區域匹配時，規則才適用
- **空選擇**：規則適用於所有區域
- **使用場景**：區域特定附加費用或折扣

**範例**：僅限國內區域免運費
```
區域：["Domestic USA"]
```

---

### 配送方式

- **方式**：規則僅適用於特定配送方式
- **空選擇**：規則適用於所有方式
- **使用場景**：特定配送方式的促銷

**範例**：快速配送 25% 折扣
```
方式：["Express Delivery"]
```

---

### 產品要求

**需要產品**：購物車必須包含至少一個這些產品

**需要分類**：購物車必須包含至少一個這些分類的產品

**使用場景**：特定產品免運費、促銷組合

**範例**：購物車包含 "Promotion Item A" 時免運費
```
需要產品：[Product ID 123]
```

---

### 產品排除

**排除產品**：如果購物車包含任何這些產品，規則不適用

**排除分類**：如果購物車包含任何這些分類的產品，規則不適用

**使用場景**：排除重型/超大物品免運費

**範例**：除家具分類外免運費
```
排除分類：[Furniture]
```

---

### 顧客群組

- **顧客群組**：規則僅適用於選定群組的顧客（VIP、批發等）
- **空選擇**：規則適用於所有顧客群組
- **使用場景**：VIP 優惠、批發折扣

**範例**：VIP 會員 15% 運費折扣
```
顧客群組：["VIP"]
```

---

### 首次顧客

- **首次顧客**：切換以限制規則僅適用於沒有先前訂單的顧客
- **使用場景**：新顧客歡迎優惠

**範例**：首筆訂單運費 $5 折扣
```
首次顧客：是
```

---

## 規則優先級與執行

規則按 **優先級順序** 執行（數字越高，越早執行）：

### 優先級機制

**執行範例**：
```
規則 A（優先級 100）：購物車 > $50 免運費
規則 B（優先級 50）：所有運費 10% 折扣
規則 C（優先級 1）：遠程區域 $2 附加費用

購物車：$60，遠程區域
基礎運費：$15

步驟 1：評估規則 A（優先級 100）
  購物車 > $50？ 是
  應用：設定費用為 $0
  當前費用：$0

步驟 2：評估規則 B（優先級 50）
  對 $0 應用 10% 折扣
  當前費用：$0（仍然免運費）

步驟 3：評估規則 C（優先級 1）
  對 $0 增加 $2 附加費用
  當前費用：$2

最終費用：$2
```

**停止進一步規則標誌**：

如果規則 A 有 `stop_further_rules = True`：
```
規則 A（優先級 100，stop_further_rules=True）：購物車 > $50 免運費
規則 B（優先級 50）：10% 折扣
規則 C（優先級 1）：$2 附加費用

購物車：$60
基礎：$15

步驟 1：應用規則 A，設定費用為 $0
        stop_further_rules = True → 停止

最終費用：$0（規則 B 和 C 從未執行）
```

## 建立運費規則

**逐步工作流程**：

1. **導航至規則**
   - 設定 > 運費 > 運費規則
   - 點擊 "新增運費規則"

2. **基本配置**
   - **名稱**：內部識別碼（例如，"超過 $50 免運費"）
   - **描述**：可選備註（不顯示給顧客）
   - **啟用**：切換以啟用/停用
   - **優先級**：設定執行順序（100 為高優先級，1 為低）

3. **選擇規則類型**
   - 選擇調整類型（折扣 %、固定折扣、設定費用、免運費、百分比附加費用、固定附加費用）
   - 輸入金額或百分比

4. **設定停止標誌**（可選）
   - 如果此規則應阻止低優先級規則執行，請勾選 "停止進一步規則"
   - 用於最終/絕對規則（例如，免運費後不應再有附加費用）

5. **定義條件**（可選 - 留空表示 "總是應用"）
   - 時間有效性：開始/結束日期
   - 購物車金額：最低/最高
   - 購物車重量：最低/最高
   - 項目數量：最低/最高
   - 區域：選擇適用區域
   - 方式：選擇適用方式
   - 產品：所需或排除
   - 顧客：群組或僅首次顧客

6. **保存規則**
   - 點擊保存
   - 規則立即生效（如果啟用切換為 Yes）

```
  最低購物車金額：$50
停止進一步規則：是
```

## 規則組合策略

### 策略 1：折扣疊加

**允許多個折扣疊加**：
```
規則 A（優先級 100）：VIP 10% 折扣 → stop_further_rules=否
規則 B（優先級 50）：訂單 >$100 15% 折扣 → stop_further_rules=否

VIP 顧客 $120 訂單：
基礎：$15
應用規則 A：$13.50（10% 折扣）
應用規則 B：$11.48（15% 折扣 $13.50）
```

### 策略 2：專屬規則

**僅有一個規則適用**（最高優先級）：
```
規則 A（優先級 100）：購物車 >$50 免運費 → stop_further_rules=是
規則 B（優先級 50）：所有運費 20% 折扣 → stop_further_rules=是

購物車 > $50：
規則 A 應用 → 免運費 → 停止
規則 B 從未執行
```

### 策略 3：條件附加費用

**折扣優先，附加費用最後**：
```
規則 A（優先級 100）：購物車 >$75 免運費
規則 B（優先級 75）：VIP 15% 折扣
規則 C（優先級 50）：一般 10% 折扣
規則 D（優先級 25）：遠程區域 $5 附加費用
規則 E（優先級 1）：燃料 10% 附加費用

訂單：$80，遠程區域，VIP 顧客
基礎：$20
A：$80 > $75 → 免運費（$0）
B：VIP → 15% 折扣 $0 = $0
C：10% 折扣 $0 = $0
D：遠程區域 +$5 = $5
E：燃料 +10% 的 $5 = $5.50

最終：$5.50（因附加費用而未免運費）
```

**要避免此情況，請使用 stop_further_rules=是**：
```
規則 A（優先級 100，stop=是）：購物車 >$75 免運費

相同訂單：
A：$80 > $75 → 免運費（$0）→ 停止
最終：$0（真正免運費）
```

## 測試運費規則

**上線前**：

1. **建立測試購物車**
   - 購物車 A：$25（低於門檻）
   - 購物車 B：$55（高於門檻）
   - 購物車 C：$200 + 遠程區域
   - 購物車 D：VIP 顧客

2. **測試每個規則**
   - 前往結帳
   - 驗證顯示正確的運費
   - 檢查規則執行順序

3. **測試優先級解決方案**
   - 多個匹配規則
   - 驗證最高優先級先執行
   - 檢查 stop_further_rules 行為

4. **測試邊界情況**
   - 購物車金額正好在門檻
   - 多個條件匹配
   - 冲突規則

## 故障排除

**問題 1：規則未應用**

**原因**：
- 規則未啟用
- 一個或多個條件未滿足
- 更高優先級規則設置 stop_further_rules=Yes
- 時間有效性超出當前日期

**解決方案**：審查所有條件，檢查優先級，驗證啟用狀態。

---

**問題 2：意外折扣金額**

**原因**：
- 多個規則疊加
- 百分比應用於已折扣費用
- 規則優先級錯誤

**解決方案**：檢查優先級順序，審查 stop_further_rules 標誌，手動追蹤執行過程。

---

**問題 3：免運費未生效**

**原因**：
- 更低優先級的附加費用規則在免運費規則後增加費用
- 購物車未達到最低金額門檻
- 購物車包含排除產品

**解決方案**：在免運費規則上使用 stop_further_rules=是，驗證條件，檢查排除項目。

## 小技巧

- **為免運費使用高優先級** - 優先級 100 確保其在其他調整前執行
- **為絕對規則設置 stop_further_rules** - 免運費應停止進一步處理
- **測試規則組合** - 多個規則可能產生意外互動
- **使用描述性名稱** - "VIP 20% 折扣（優先級 75）" 比 "規則 3" 更好
- **記錄複雜邏輯** - 在描述欄中添加備註
- **從簡單規則開始** - 逐步增加複雜度
- **監控規則表現** - 檢查規則是否被使用或導致混淆
- **避免過多規則** - 過多規則會減慢結帳，最多使用 5-10 個
- **使用區域進行地理區分** - 比每個國家設置多個相似規則更好
- **與配送方式組合使用** - 規則 + 配送方式可實現複雜定價
- **設置明確的時間窗口** - 促銷時總是包含結束日期
- **測試邊界情況** - 恰好 $50、恰好 5 項等
