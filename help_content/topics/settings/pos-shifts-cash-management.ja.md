---
title: POSシフトと現金管理
---

POSシフトは、レジ担当者の作業期間を追跡し、正確な現金会計を確保します。各シフトは、レジを開けて開始現金を数え、シフトを終了して最終的な数えと調整を行う1人のレジ担当者の端末での時間です。システムは、実際の現金売上に基づいて予想される現金を自動的に計算し、物理的な数えと比較して、調査が必要な不一致を強調表示します。シフト中の現金の動き（浮動金の追加、小切手の引き出し）は、完全な監査トレースのために理由が記録されます。

**POS > シフト**に移動して、すべてのシフトを表示し、アクティブなシフトを監視し、現金調整レポートを確認し、過去のアクティビティを監査します。

![シフト一覧](/static/core/admin/img/help/pos-shifts-cash-management/shift-list.webp)

## POSシフトの理解

シフトは、1人のレジ担当者が1つの端末を操作する作業期間です。シフトは現金の責任を強制します—シフト中のレジ担当者は、自分のレジの現金に責任を持っています。

**シフトライフサイクル**:
1. **開店** - レジ担当者がシフトを開始し、開店時の現金を数え、金額を記録します
2. **シフト中** - 売上を処理し、支払いを受け取り、返金を発行します
3. **閉店** - レジ担当者が現金を数え、終了金額を記録し、システムが不一致を計算します
4. **調整済み** - シフトが確定し、監査のためにロックされます

**追跡される主なメトリクス**:
- **開店現金** - シフト開始時のレジの現金
- **閉店現金** - シフト終了時のレジの現金
- **予想現金** - 計算: 開店現金 + 現金売上 - 現金返金 + 現金の動き
- **現金差額** - 不一致: 閉店現金 - 予想現金（正 = 超過、負 = 不足）
- **総売上** - シフト中のすべての売上取引の合計
- **総返金** - シフト中のすべての返金取引の合計
- **取引数** - 処理された注文数

## シフト一覧ビュー

シフト一覧は、すべてのシフトと主な情報を表示します:

**シフトステータス**:
- **開店中** (緑のバッジ) - 現在アクティブなシフト
- **閉店済み** (灰色のバッジ) - 完了したシフト
- **調整済み** (青のバッジ) - 確定し、監査のためにロックされています

**端末** - シフトが行われたPOS端末

**レジ担当者** - シフトを担当したスタッフ

**開店現金** - シフト開始時の現金額

**閉店現金** - シフト終了時の現金額（シフトがまだ開いている場合は空白）

**予想現金** - トランザクションに基づいてシステムで計算された予想額

**現金差額** - 不一致（負の場合は赤色で強調、正の場合は緑色、ゼロの場合は黒色）

**期間** - シフトの長さ（開始時間から終了時間）

**総売上** - シフト中に発生した収益

フィルタを使用して表示:
- みなし開店シフト（アクティブな端末を監視）
- 不一致のあるシフト（現金差額 ≠ 0）
- 日付範囲によるシフト（日常的な調整レポート）
- レジ担当者によるシフト（パフォーマンス監査）

## シフトの開店

レジ担当者はPOS端末から直接シフトを開店します（管理画面から開店することはできません）。端末でのワークフロー:

1. **スタッフがログイン** - 端末にアクセスするために資格情報を入力します

2. **開店現金を数える** - レジ内のすべての現金（紙幣と硬貨）を実際に数えます

3. **開店金額を入力** - POSアプリに数えた金額を記録します

4. **シフト開始** - 端末は売上処理が可能になります

**開店現金のガイドライン**:
- 標準的な開店現金（浮動金）は、店舗の規模によって$100-$300です
- 数えを2回行い、正確性を確保します—開店時のエラーは閉店時の不一致に連鎖します
- レジが空の場合、開店現金は$0.00（浮動金は現金の動きを通じて追加されます）
- $50以上の大きな紙幣は別途記録し、その動きを追跡します

![シフト追加フォーム](/static/core/admin/img/help/pos-shifts-cash-management/shift-add-form.webp)

## シフト中の処理

シフトが開いている間、システムは自動的に追跡します:

**現金売上** - 顧客が現金で支払うすべての取引（予想現金に加算）

**現金返金** - 現金で発行されたすべての返金（予想現金から減算）

**カード売上** - クレジット/デビットカード取引（現金に影響なし）

**分割支払い** - 現金とカードの一部（現金部分のみが予想現金に影響）

**ギフトカード＆チケット** - 現金以外の支払い方法（現金に影響なし）

レジ担当者は通常通り売上処理を続けます。システムは、予想現金の連続的な計算を裏側で維持します。

## 現金の動き

シフト中の現金の動きは、現金引き出物の調整です:

**浮動金の追加** - レジに現金を追加:
- 理由: "大きな紙幣のための浮動金の追加"
- 金額: +$100.00
- 予想現金は$100.00増加します

**小切手の引き出し** - 支出のために現金を引き出す:
- 理由: "事務用品の購入"
- 金額: -$25.00
- 予想現金は$25.00減少します

**銀行への預け入れ** - セキュリティのために余分な現金を引き出す:
- 理由: "セーフティードロップ - レジに$500以上"
- 金額: -$300.00
- 予想現金は$300.00減少します

**端末での現金の動きの記録**:
1. **メニュー** > **現金の動き**をタップします
2. タイプを選択: 追加または削除
3. 金額を入力
4. 理由を入力（監査トレースのために必須）
5. 確認

すべての現金の動きは、タイムスタンプ、金額、理由とともにシフト詳細レポートに表示されます。

## シフトの閉店

レジ担当者が作業期間を終了した場合、シフトを閉店します:

1. **シフトを閉店** - 端末メニューからタップします

2. **残りの取引を処理** - 保留中のカートや保留中の売上を完了します

3. **閉店現金を数える** - レジ内のすべての現金を実際に数えます
   - 紙幣は額面ごとに数えます（$100s, $50s, $20s, $10s, $5s, $1s）
   - 硬貨は種類ごとに数えます（クォータ、ダイム、ニッケル、ペンニ）
   - 合計 = 閉店現金額

4. **閉店金額を入力** - 数えた合計を記録します

5. **システムが不一致を計算**:
   - 予想現金 = 開店現金 + 現金売上 - 現金返金 + 現金の動き
   - 現金差額 = 閉店現金 - 予想現金
   - 例: 閉店 $485.00 - 予想 $480.00 = +$5.00 超過

6. **不一致を確認** - 端末は差額を表示します:
   - **正確 ($0.00)** - 完璧な調整
   - **小さな超過 (+$1 to +$5)** - 可能な四捨五入または顧客のチップ
   - **小さな不足 (-$1 to -$5)** - 小さな数え間違い、許容可能
   - **大きな不一致 (>$5)** - 再数えが必要

7. **必要に応じて再数え** - 不一致が$10以上の場合、シフトを確定する前にレジ担当者は閉店現金を再数えます

8. **シフトを確定** - 閉店金額を確認し、シフトのステータスは「閉店済み」に変更されます

9. **シフトレポートを印刷** - 端末はレジ担当者の記録用に現金調整のレシートを印刷します

![シフト詳細](/static/core/admin/img/help/pos-shifts-cash-management/shift-detail.webp)

## 現金調整の計算式

システムはこの式を使って予想現金を計算します:

```
予想現金 = 開店現金
          + 現金売上
          - 現金返金
          + 現金追加（動き）
          - 現金引き出し（動き）
```

**例**:
- 開店現金: $200.00
- 現金売上: $450.00（15件の取引）
- 現金返金: -$30.00（1件の返品）
- 現金追加: +$100.00（シフト中盤に浮動金を追加）
- 現金引き出し: -$50.00（小切手の引き出し）
- **予想現金: $200 + $450 - $30 + $100 - $50 = $670.00**

もしレジ担当者が閉店時に$675.00を数えた場合:
- 現金差額: $675.00 - $670.00 = **+$5.00 超過**

## シフトレポートと監査

シフトレポートは、詳細な調整情報を提供します:

**概要セクション**:
- 開店と閉店現金
- 予想現金の計算
- 現金差額（超過/不足）
- 総売上と返金
- 取引数
- シフト期間

**取引詳細**:
- シフト中のすべての売上（注文ID、金額、支払い方法）
- すべての返金
- 各取引のタイムスタンプ

**現金の動きログ**:
- すべての追加と引き出し
- 提供された理由
- タイムスタンプ

**使用例**:
- **日常的な調整** - 営業日の終わりにすべてのシフトを確認します
- **レジ担当者のパフォーマンス** - スタッフごとに不一致のパターンを特定します
- **盗難の検出** - 一貫した大きな不足は盗難を示唆する可能性があります
- **トレーニングの必要性** - 頻繁な小さな不一致は数えの正確性の問題を示唆します
- **監査トレース** - 会計および税務目的の完全な記録

## 複数端末の現金管理

複数の端末で同時にシフトが実行されている店舗の場合:

**個別の引き出物**：各端末には個別の現金引き出物があり、シフトは独立しています。端末1のレジ担当者Aと端末2のレジ担当者Bは、個別のシフトと個別の調整を行います。

**共有引き出物**：一部の店舗では、複数の端末にわたって1つの現金引き出物を共有しています（推奨されません）。このような場合:
- 共有引き出物には一度に1つのシフトを開けることができます
- レジ担当者は次のレジ担当者に引き渡すときにシフトを閉じる必要があります
- 引き渡し中のすべての追加/引き出しを現金の動きで追跡します
- 不一致は特定のレジ担当者に帰属させるのが難しくなります

**ベストプラクティス**：1つの端末に1つの現金引き出物、1つのシフトを1人のレジ担当者が1セッションごとに実行します。これにより、明確な責任と簡易な調整が保証されます。

## 不一致の処理

閉店現金が予想現金と一致しない場合:

**小さな不一致（< $5）**:
- 四捨五入、数え間違い、顧客のチップによるため許容されます
- シフトのノートに記録します
- パターンが発生しない限り、さらなる対応は必要ありません

**中規模の不一致（$5 - $20）**:
- シフトを確定する前に現金を再数えます
- トランザクションログを確認してエラーをチェックします（誤ったおつり、処理されていないキャンセル取引）
- シフトのノートに状況を記録します
- マネージャーによる確認が推奨されます

**大きな不一致（> $20）**:
- 再数えが必須です
- シフトを確定するためにはマネージャーの承認が必要です
- すべてのトランザクションと現金の動きを確認します
- 潜在的な原因（盗難、引き出物の誤操作、開店現金の誤記）を調査します
- 情報によっては懲戒処分が必要になる場合があります

**継続的な不足**:
- 同じレジ担当者による継続的な負の不一致は、トレーニングの問題または盗難を示唆します
- トレーニングの追加監視（シフト中にマネージャーによるスポットチェック）を実施します
- POSトレーニング手順を確認します
- 現金処理ポリシーの更新を検討します

## ヒント

- **開店現金を2回数える** - 開店時のエラーは閉店時の不一致に連鎖します。開始時の正確性は、終了時の問題を防ぎます
- **現金の動きをすぐに記録する** - 浮動金の追加や小切手の引き出しを閉店時に記録しないでください
- **常に動きの理由を提供する** - "$100を追加"は監査に無駄ですが、"$100を追加（$5紙幣が不足しているため）"は実行可能な情報です
- **不一致が$10以上の場合、再数えを行う** - 大きな不一致がある場合、再数えを行わずにシフトを確定しないでください
- **毎日シフトレポートを印刷する** - 会計用の毎日の調整書類に添付します
- **個々の不一致ではなくパターンを確認する** - 1つの-$3.00の不足は問題ありませんが、5回連続の-$3.00の不足は問題です
- **営業日の終わりにシフトを閉じる** - シフトを夜通し開けたままにしないでください。最近の不一致は調査が簡単です
- **レジ担当者に額面の数え方をトレーニングする** - 多くのエラーは紙幣の数え間違い（$5を$10と誤認する）から来ます
- **硬貨のラッピングを使用する** - 事前にラッピングされた硬貨は数え間違いを減らし、調整を速やかにします
