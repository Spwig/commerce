---
title: Настройка шаблонов чеков
---

Шаблоны чеков управляют внешним видом и содержанием печатных термических чеков на ваших терминалах POS. Настройте текст заголовка и подзаголовка, добавьте ваш логотип, настройте поля соответствия (налоговые идентификаторы, номера регистрации бизнеса) и включите промо QR-коды. Шаблоны поддерживают целевую настройку области — создайте шаблон по умолчанию для всех магазинов, шаблоны для групп по регионам или шаблоны для отдельных магазинов. Система использует правила приоритета области, чтобы определить, какой шаблон применяется при печати чека.

Используйте шаблоны чеков для поддержания целостности бренда, соблюдения региональных требований и повышения вовлеченности клиентов через промо-элементы.

## Основы шаблонов чеков

Шаблоны чеков определяют структуру и содержание печатных чеков с термических принтеров ESC/POS. Каждый шаблон указывает:

**Физическая настройка**:
- Ширина бумаги (58 мм или 80 мм)
- Изображение логотипа (черно-белое для термопечати)
- Размер шрифта и интервалы

**Разделы содержания**:
- Текст заголовка (название магазина, адрес, контактная информация)
- Динамические данные транзакции (товары, цены, итоги, методы оплаты)
- Текст подзаголовка (политика возврата, сообщение благодарности, социальные сети)
- Поля соответствия (налоговые идентификаторы, номера регистрации бизнеса)
- Промо QR-код с меткой

**Целевая настройка области**:
- Шаблон по умолчанию (применяется ко всем магазинам, если не переопределен)
- Шаблон группы (применяется ко всем магазинам в группе)
- Шаблон магазина (применяется к конкретному магазину/складу)

## Правила приоритета области

Когда терминал печатает чек, система выбирает шаблон с использованием этой иерархии (от высшего приоритета к низшему):

| Приоритет | Область | Пример | Сценарий использования |
|-----------|--------|--------|----------------------|
| **1** | Шаблон конкретного магазина | Шаблон магазина Париж | Уникальные требования соответствия налогам во Франции |
| **2** | Шаблон группы | Шаблон магазинов Европы | Отображение НДС для всех местоположений в ЕС |
| **3** | По умолчанию | Глобальный шаблон | Резервный вариант для всех неподключенных магазинов |

**Как это работает**:
1. Проверьте, есть ли у магазина собственный шаблон (специфичный для склада)
2. Если нет, проверьте, есть ли у группы магазина групповой шаблон
3. Если нет, используйте шаблон по умолчанию

**Пример**:
- Шаблон по умолчанию: "Стандартный чек" (без назначения области)
- Групповой шаблон: "Чек ЕС" (назначен группе магазинов Европы) - включает регистрацию НДС
- Шаблон магазина: "Чек Парижа" (назначен складу Париж) - включает французский номер SIRET

**Результат**:
- Терминал магазина Париж: Использует "Чек Парижа" (наиболее конкретный)
- Терминал магазина Берлин (в группе магазинов Европы, без шаблона магазина): Использует "Чек ЕС" (уровень группы)
- Терминал магазина Нью-Йорк (без группы, без шаблона магазина): Использует "Стандартный чек" (резервный по умолчанию)

## Настройка ширины бумаги

Термические принтеры для чеков используют либо ширину бумаги 58 мм, либо 80 мм. Выберите в зависимости от вашего оборудования принтера:

| Ширина бумаги | Количество символов в строке | Лучше подходит для | Типичное применение |
|----------------|-----------------------------|----------------------|----------------------|
| **58 мм** | ~32 символа | Малый размер, переносной | Автобусы-фургоны, мобильные POS, киоски |
| **80 мм** | ~48 символов | Стандартная розничная торговля | Большинство розничных магазинов, ресторанов |

**Нельзя смешивать ширину**: Все терминалы, использующие один и тот же шаблон, должны иметь принтеры с одинаковой шириной бумаги. Если у вас есть разные типы принтеров, создайте отдельные шаблоны для каждой ширины.

**Ограничения размера логотипа**:
- **58 мм**: Максимальная ширина 384 пикселя (рекомендуется: 350 пикселей)
- **80 мм**: Максимальная ширина 576 пикселей (рекомендуется: 550 пикселей)

Логотипы, превышающие максимальную ширину, автоматически уменьшаются в размере, что может снизить качество.

## Настройка логотипа

Логотипы чеков должны быть **черно-белыми** (только черный и белый) для совместимости с термическими принтерами:

**Требования к логотипу**:
- Формат файла: PNG, JPG или WebP
- Режим цвета: Черно-белый (черные пиксели на белом фоне)
- Рекомендуемые размеры:
  - Бумага 58 мм: 350 пикселей ширины × 100-150 пикселей высоты
  - Бумага 80 мм: 550 пикселей ширины × 150-200 пикселей высоты
- Размер файла: <100 КБ (термические принтеры имеют ограниченную память)

**Создание черно-белых логотипов**:
1. Начните с вашего обычного логотипа (цветного или серого)
2. Используйте редактор изображений, чтобы преобразовать его в чисто черно-белый (без серых оттенков)
3. Увеличьте контраст, чтобы убедиться, что черные элементы были твердыми
4. Экспортируйте как PNG с прозрачным или белым фоном

**Расположение логотипа**:
- Всегда центрируется горизонтально
- Печатается в верхней части чека (выше текста заголовка)
- Следует автоматическое пространство (предотвращает переполнение содержимого)

**Выбор логотипа**:
- Нажмите **Обзор медиатеки** в форме шаблона
- Выберите черно-белый актив логотипа
- Предварительный просмотр показывает, как будет выглядеть логотип на чеке

**Без логотипа**: Оставьте поле логотипа пустым, если вы предпочитаете текстовую маркировку (текст заголовка может включать название магазина).

## Текст заголовка

Текст заголовка отображается сразу после логотипа (или в верхней части, если логотипа нет). Обычное содержание:

**Название магазина и адрес**:
```
Ваше название магазина
123 Main Street
City, State 12345
Phone: (555) 123-4567
```

**Часы работы**:
```
Понедельник-Пятница: с 9 утра до 9 вечера
Суббота-Воскресенье: с 10 утра до 6 вечера
```

**Слоган или девиз**:
```
Качественные товары, исключительное обслуживание
```

**Форматирование**:
- Используйте переносы строк для разделения информации
- Автоматически выровнено по центру
- Сохраняйте строки под лимитом символов для ширины бумаги (32 символа для 58 мм, 48 для 80 мм)

**Доступные переменные** (опционально):
- `{store_name}` - Заменяется на название склада
- `{order_date}` - Заменяется на дату транзакции
- `{order_number}` - Заменяется на номер заказа

Большинство продавцов используют статический текст вместо переменных для последовательности заголовка.

## Текст подзаголовка

Текст подзаголовка отображается после деталей транзакции (товары, итоги, оплата). Обычное содержание:

**Политика возврата**:
```
Возврат в течение 30 дней с чеком
Только кредит магазина или обмен
```

**Сообщение благодарности**:
```
Спасибо, что покупали у нас!
Следите за нами @yourstore
```

**Служба поддержки клиентов**:
```
Вопросы? Звоните (555) 123-4567
или отправьте электронное письмо support@yourstore.com
```

**Советы по форматированию**:
- Держите наиболее важную информацию в начале (политика возврата, контакт)
- Используйте переносы строк для удобочитаемости
- Рассмотрите добавление разделительной линии (`---`) между разделами

## Поля соответствия

Многие юрисдикции требуют определенной информации на чеках:

**Метка налогового идентификатора** - Настраиваемая метка для номера налогового идентификатора:
- США: "Tax ID" или "EIN"
- ЕС: "VAT Number" или "VAT Reg No"
- Канада: "GST/HST Number"
- Австралия: "ABN"

**Значение налогового идентификатора** - фактический номер идентификатора:
- Вводится один раз в шаблоне и отображается на всех чеках
- Пример: "VAT Number: GB123456789"

**Метка регистрации бизнеса** - Настраиваемая метка для регистрации бизнеса:
- Франция: "SIRET"
- Германия: "Handelsregister"
- Великобритания: "Company Registration Number"

**Значение регистрации бизнеса** - фактический номер регистрации:
- Пример: "SIRET: 123 456 789 00010"

**Показать "Powered by Spwig"** - Переключатель для отображения или скрытия брендинга "Powered by Spwig":
- Включено по умолчанию (поддерживает развитие платформы)
- Отключите для операций с белой меткой

**Примеры соответствия по регионам**:

**Европейский Союз**:
- Метка налогового идентификатора: "VAT Number"
- Значение налогового идентификатора: "GB123456789"
- Отображайте номер регистрации компании, если это требуется в стране

**Соединенные Штаты**:
- Обычно не требуется налоговый идентификатор на чеке (варьируется по штатам)
- Может включать EIN для B2B-транзакций

**Франция (специфическая)**:
- Обязательный SIRET на всех чеках
- Метка регистрации бизнеса: "SIRET"
- Значение регистрации бизнеса: "123 456 789 00010"

**Австралия**:
- Рекомендуется ABN (австралийский номер бизнеса) для бизнесов, зарегистрированных на GST
- Метка налогового идентификатора: "ABN"

Проверьте требования к чекам в вашем местном юрисдикции перед запуском.

## Промо-коды QR

Включите QR-код в нижней части чеков для повышения вовлеченности клиентов:

**URL QR-кода** - Цель, на которую перейдете после сканирования:
- Отзыв: `https://yourstore.com/reviews/leave-review`
- Программа лояльности: `https://yourstore.com/loyalty/join`
- Скидка на следующую покупку: `https://yourstore.com/discount/THANKYOU`
- Социальные сети: `https://instagram.com/yourstore`
- Главная страница сайта: `https://yourstore.com`

**Метка QR-кода** - Текст, отображаемый над QR-кодом:
- "Сканируйте, чтобы оставить отзыв и получить 10% скидки на следующую покупку"
- "Присоединяйтесь к нашей программе лояльности - сканируйте здесь"
- "Следите за нами в Instagram - сканируйте, чтобы подключиться"
- "Оцените свой опыт"

**Рекомендации по QR-кодам**:
- Используйте короткие URL (длинные URL создают плотные, трудные для сканирования коды)
- Тестируйте QR-код с несколькими телефонными камерами перед развертыванием
- Включите четкое предложение ценности в метке (что клиент получит при сканировании)
- Отслеживайте сканирования QR-кодов, чтобы измерить эффективность (используйте URL с параметром отслеживания)

**Динамические QR-коды** (расширенные):
- Используйте сервис перенаправления QR (bit.ly, tinyurl), чтобы создать короткий URL
- Направьте перенаправление на разные цели по сезонам без повторной печати чеков
- Пример: `https://bit.ly/yourstoreqr` → перенаправляет на текущую акцию

## Создание шаблонов для разных областей

**Шаблон по умолчанию** (рекомендуемая начальная точка):
1. Перейдите в **POS > Шаблоны чеков**
2. Нажмите **+ Добавить шаблон чека**
3. Оставьте поля **Склад** и **Группа магазинов** пустыми (это делает его шаблоном по умолчанию)
4. Настройте ширину бумаги, соответствующую вашему наиболее распространенному типу принтера
5. Добавьте логотип, заголовок, подзаголовок
6. Настройте поля соответствия для вашего основного рынка
7. Сохраните

Этот шаблон применяется ко всем магазинам, если не переопределен.

**Групповой шаблон** (для региональных различий):
1. Создайте новый шаблон
2. Выберите **Группу магазинов** (например, "European Stores")
3. Оставьте поле **Склад** пустым
4. Настройте поля соответствия для региона (например, форматирование НДС)
5. Настройте текст заголовка (например, региональный адрес)
6. Сохраните

Этот шаблон применяется ко всем магазинам в группе.

**Шаблон магазина** (для специфических нужд локации):
1. Создайте новый шаблон
2. Выберите **Склад** (например, "Paris Store")
3. Настройте все поля для этой конкретной локации
4. Сохраните

Этот шаблон применяется только к одному магазину.

**Тестирование шаблонов**:
- Обработайте тестовую транзакцию на терминале
- Распечатайте чек
- Проверьте четкость логотипа, выравнивание текста, поля соответствия, сканируемость QR-кода
- При необходимости отрегулируйте шаблон и повторно протестируйте

## Распространенные макеты чеков

**Минимальный чек** (автобусы-фургоны, выездные мероприятия):
- Без логотипа (экономия места)
- Заголовок: только название магазина и телефон
- Подзаголовок: сообщение благодарности
- Без QR-кода

**Стандартный розничный чек**:
- Логотип (черно-белый бренд)
- Заголовок: полное название магазина, адрес, часы работы
- Соответствие: налоговый идентификатор
- Подзаголовок: политика возврата, сообщение благодарности
- QR-код: запрос отзывов

**Премиумный розничный чек**:
- Логотип (полный бренд-логотип)
- Заголовок: слоган, адрес, контакт
- Соответствие: налоговый идентификатор, регистрация бизнеса
- Подзаголовок: политика возврата, служба поддержки клиентов, социальные сети
- QR-код: регистрация в программе лояльности

**Многофилиалная сеть**:
- Шаблон по умолчанию: корпоративный бренд, стандартные политики
- Групповые шаблоны: региональное соответствие (НДС для ЕС, GST для Канады)
- Шаблоны магазинов: адрес и телефон для конкретной локации

## Управление несколькими шаблонами

**Соглашение по именованию шаблонов**:
- Используйте область в имени: "Default Receipt", "EU Group Receipt", "Paris Store Receipt"
- Это помогает определить, какой шаблон применяется где при просмотре списка

**Изменения шаблонов**:
- Изменения применяются сразу к будущим чекам
- Прошлые чеки (уже напечатанные) не затрагиваются
- Тестируйте изменения на терминале с низким трафиком перед развертыванием во все магазины

**Дублирование шаблонов**:
- При создании нового шаблона, похожего на существующий, дублируйте существующий шаблон и вносите изменения
- Это предотвращает необходимость начинать с нуля

**Удаление шаблонов**:
- Нельзя удалить шаблон по умолчанию, если существуют терминалы (должен быть один резервный)
- Можно удалить групповые/шаблоны магазинов (терминалы переходят к следующему уровню иерархии)
- Подтвердите, что терминалы не используют шаблон активно, прежде чем удалять

## Советы

- **Начните с 80 мм, если не уверены** - Стандартная ширина бумаги подходит для большинства розничных магазинов; 58 мм - специализированная
- **Тестируйте логотип на реальном принтере** - То, что выглядит хорошо на экране, может плохо печататься; тестирование должно быть ранним
- **Обновляйте поля соответствия** - Истекшие налоговые регистрации на чеках вызывают юридические проблемы
- **QR-коды с предложением ценности сканируются лучше** - "Сканируйте для 10% скидки" превосходит "Сканируйте здесь" в 10 раз
- **Проверьте лимиты символов** - Обтекание текста портит форматирование; подсчитайте количество символов на строку перед развертыванием
- **Один шаблон на ширину бумаги** - Не назначайте шаблон 80 мм на терминал с принтером 58 мм (логотип не поместится)
- **Печатайте тестовые чеки ежемесячно** - Принтеры со временем ухудшаются; проверьте, остается ли качество приемлемым
- **Используйте переменные с осторожностью** - Статический текст более надежен, чем динамические переменные (менее точек отказа)
- **Создайте резервную копию настройки шаблона** - Сделайте скриншот или экспортируйте настройки шаблона перед крупными изменениями (легкий возврат)
- **Соответствие регионам различается** - Исследуйте местные требования к чекам перед развертыванием; штрафы за несоответствие могут быть серьезными
