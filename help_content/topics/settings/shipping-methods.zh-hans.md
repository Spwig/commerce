---
title: 配送方式
---

配送方式是结账时面向客户的配送选项——每种方式都使用不同的定价策略来计算运费。Spwig 支持 7 种配送方式，从简单的固定费率到复杂的承运商实时计算的定价。配送方式可以通过最低/最高订单金额、重量和地理区域进行限制。客户在结账时选择其偏好的配送方式，计算出的费用将被添加到订单总额中。

使用本指南配置符合您商业模式的配送方式，从基本的固定费率配送到复杂的基于区域的分层定价。

## 配送方式类型

Spwig 提供 7 种配送方式类型，每种方式都有不同的费用计算逻辑：

### 固定费率配送

**是什么**：无论购物车内容、目的地或重量如何，费用都是固定的。

**何时使用**：
- 预测运费的简单商店
- 单一产品类型（尺寸/重量相似）
- 仅限国内配送，使用标准承运商费率
- 免运费门槛促销（与运费促销结合使用）

**配置**：
- 设置 **配送方式类型** = 固定费率
- 输入 **固定费用**（例如，9.99 美元）
- 可选：设置最低/最高订单金额限制

**示例**：所有国内订单使用“标准配送 - 9.99 美元”。

---

### 免运费

**是什么**：零费用的配送选项（客户无需支付费用）。

**何时使用**：
- 免运费促销
- 高价值订单（与最低订单金额结合使用）
- 本地自提的替代方案
- 会员计划福利

**配置**：
- 设置 **配送方式类型** = 免运费
- 可选：设置 **最低订单金额**（例如，订单满 50 美元免运费）
- 与运费促销结合使用，实现条件性免运费

**示例**：最低订单金额为 50 美元时，“订单满 50 美元免运费”。

---

### 基于重量的配送

**是什么**：根据购物车总重量，从分层费率表中计算费用。

**使用场景**:
- 具有可变重量的产品（书籍、硬件、杂货）
- 基于重量的承运商定价模型
- 可预测的重量与成本比例

**配置**:
1. 设置 **方法类型** = 基于重量
2. 创建 **运费费率表**，并设置 basis_type = "weight"
3. 添加 **运费费率层级**（例如，0-5kg = 10美元，5-10kg = 15美元，10-20kg = 25美元）
4. 可选：限制到特定区域

**示例**:
```
0-2kg: 8美元
2-5kg: 12美元
5-10kg: 18美元
10kg+: 25美元
```

**工作原理**：购物车计算总重量 → 找到匹配的层级 → 返回该层级的费率。

---

### 基于价格的运费

**是什么**：根据购物车小计从分层费率表中计算成本。

**使用场景**:
- 运费与订单价值相关
- 鼓励更高的购物车金额（在更高层级中每美元费率更低）
- 与基于重量的运费类似商品的简单替代方案

**配置**:
1. 设置 **方法类型** = 基于价格
2. 创建 **运费费率表**，并设置 basis_type = "price"
3. 添加 **运费费率层级**（例如，0-50美元 = 9.99美元，50-100美元 = 14.99美元，100+美元 = 19.99美元）

**示例**:
```
0-25美元: 6.99美元
25-75美元: 9.99美元
75-150美元: 12.99美元
150+美元: 免费
```

**工作原理**：购物车计算小计 → 找到匹配的层级 → 返回该层级的费率。

---

### 实时承运商费率

**是什么**：在结账时从承运商API（FedEx、UPS、DHL）实时获取的费率。

**使用场景**:
- 根据目的地不同运费成本变化
- 为客户提供多个承运商选项
- 在无需手动费率表的情况下准确获取承运商定价
- 国际运输，具有复杂的定价

**配置**:
1. 设置 **方法类型** = 实时
2. 创建 **提供商账户**（设置 > 运费 > 提供商账户）
3. 输入承运商API凭据（账户号、API密钥、密钥）
4. 将提供商账户链接到运费方法
5. 可选：添加利润率百分比或固定利润率

**要求**:
- 活动的承运商账户（FedEx、UPS、DHL 等）
- 承运商的 API 凭证
- 已定义的发货包裹（用于计算体积重量）

**示例**："FedEx Ground" 方法在结账时根据购物车重量、尺寸和目的地获取实时 FedEx 运费。

**工作原理**:
1. 顾客在结账时输入地址
2. 系统使用原产地、目的地、包裹尺寸和重量调用承运商 API
3. 承运商返回运费报价
4. 可选：应用加价
5. 运费显示给顾客

---

### 本地自提

**是什么**：顾客在实体地点自提订单（无配送费用）。

**何时使用**:
- 提供自提服务的零售店
- 仓库自提选项
- 活动或市集摊位
- 为本地顾客消除运费

**配置**:
1. 设置 **方法类型** = 本地自提
2. 创建 **位置**（设置 > 配送 > 位置）
   - 设置地址、营业时间、自提容量
3. 将位置链接到方法
4. 可选：设置自提准备时间（例如，"2 小时内准备好"）

**客户体验**:
- 在结账时选择 "本地自提"
- 如果有多个位置，选择自提地点
- 根据可用性选择自提日期/时间
- 订单准备就绪时收到通知

**示例**："门店自提 - 免费"，有 3 个零售地点，24 小时内准备好。

---

### 表格费率配送

**是什么**：基于重量、价格或数量的灵活分层定价，具有高级区域定位功能。

**何时使用**:
- 复杂定价（按区域和重量不同费率）
- 需要比基于重量或价格的定价有更多控制
- 多个定价因素（例如，重量 + 目的地 + 数量）

**配置**:
1. 设置 **方法类型** = 表格费率
2. 创建 **运费费率表**
3. 定义 **basis_type**：重量、价格或数量
4. 添加 **运费费率层级** 并设置最小/最大值
5. 可选：将层级限制在特定区域或国家

**与按重量/价格计费的区别**：阶梯计费支持按层级设置地理限制，允许对同一重量/价格在不同区域设置不同费率。

**示例**：
```
区域 A（国内）：
  0-5kg：10 美元
  5-10kg：15 美元

区域 B（偏远地区）：
  0-5kg：18 美元
  5-10kg：25 美元
```

**工作原理**：购物车计算基准值（重量/价格/数量）→ 找到客户所在区域的匹配层级 → 返回该层级的费率。

## 运输方式配置

所有运输方式共享以下通用设置：

### 基本设置

- **名称**：内部标识符（不显示给客户）
- **显示名称**：结账时面向客户的名称（例如，“标准运输”，“快递配送”）
- **描述**：结账时显示的可选帮助文本（例如，“3-5 个工作日送达”）
- **方法类型**：以上 7 种类型之一
- **启用**：切换以启用或禁用方法而不删除

### 费用设置

- **固定费用**：仅适用于固定费用方法
- **费率表**：适用于基于重量、基于价格、阶梯计费的方法
- **提供商账户**：适用于实时承运商方法
- **税率类别**：对运费收取税（如适用）

### 限制

**订单价值限制**：
- **最低订单价值**：当购物车小计 ≥ 金额时，该方法才可用（例如，订单满 50 美元免运费）
- **最高订单价值**：当购物车小计 > 金额时，该方法隐藏（例如，仅适用于低于 100 美元的订单）

**重量限制**：
- **最低重量**：当购物车重量 ≥ 金额时，该方法才可用
- **最高重量**：当购物车重量 > 金额时，该方法隐藏（常见于轻型运输选项）

**地理限制**：
- **运输区域**：将方法链接到特定区域（国内、国际、区域）
- 空区域 = 可用于所有地址
- 多个区域 = 可用于任何匹配的区域

### 高级设置

- **优先级**: 在结账时的显示顺序（数字越小，列表中位置越高）
- **处理费**: 添加到计算成本的附加固定费用
- **免费送货门槛**: 如果购物车小计 ≥ 阈值，自动将成本设为 $0（替代 min_order_value）

## 创建配送方式

**逐步操作流程**:

1. **导航到配送方式**
   - 前往 设置 > 购物车 > 配送方式
   - 点击 "添加配送方式"

2. **选择方法类型**
   - 根据您的定价策略选择合适的类型
   - 类型决定了可用的成本配置字段

3. **配置基本信息**
   - 名称: 内部参考（例如，"domestic_ground"）
   - 显示名称: 面向客户（例如，"标准配送"）
   - 描述: 配送时间范围（例如，"5-7 个工作日"）

4. **设置成本计算**
   - **固定费用**: 输入固定费用
   - **重量/价格/表格费率**: 创建费率表（见下文）
   - **实时费率**: 链接提供商账户
   - **免费/自提**: 无需成本配置

5. **添加限制（可选）**
   - 最低/最高订单价值
   - 最低/最高重量
   - 配送区域

6. **设置优先级**
   - 数字越小，在结账时显示越靠前
   - 推荐顺序: 免费（1），本地自提（2），标准（3），加急（4）

7. **激活方法**
   - 切换 "激活" = 是
   - 保存

---

## 创建费率表

对于基于重量、基于价格和表格费率的方法:

**步骤 1: 创建费率表**
- 前往 设置 > 配送 > 费率表
- 点击 "添加费率表"
- 设置 **名称**（例如，"国内重量分层"）
- 设置 **基准类型**: 重量、价格或数量

**步骤 2: 添加分层**
- 点击 "添加分层"
- 设置 **最小值** 和 **最大值**（用于匹配的范围）
- 设置 **费率**（此分层的成本）
- 可选: 限制到特定区域或国家
- 保存分层

**步骤 3: 为所有分层重复操作**
- 覆盖完整范围（0 到最大预期值）
- 确保没有间隙（例如，0-5, 5-10, 10-20, 20+）
- 在最后一个分层中使用 `null` 作为最大值（无限）

**步骤 4：链接到配送方式**
- 编辑配送方式
- 从下拉菜单中选择费率表
- 保存

**示例基于重量的费率表**：
```
Name: Domestic Weight Tiers
Basis: Weight

Tiers:
1. Min: 0g, Max: 2000g, Rate: $8
2. Min: 2000g, Max: 5000g, Rate: $12
3. Min: 5000g, Max: 10000g, Rate: $18
4. Min: 10000g, Max: null, Rate: $25
```

## 常见配送场景

### 场景 1：基本国内配送

**目标**：为所有国内订单设置简单的 9.99 美元固定运费。

**解决方案**：
- 配送方式类型：固定运费
- 固定费用：9.99 美元
- 配送区域："Domestic"（仅您所在国家）

---

### 场景 2：满 50 美元免运费

**目标**：通过设置免运费门槛，鼓励客户提高购物车金额。

**解决方案 A**（推荐）：
- 配送方式类型：免运费
- 最低订单金额：50 美元
- 显示名称："Free Shipping (Orders $50+)"

**解决方案 B**（使用规则）：
- 配送方式类型：固定运费
- 固定费用：9.99 美元
- 创建配送促销：
  - 条件：购物车金额 ≥ 50 美元
  - 动作：将费用设置为 0 美元

---

### 场景 3：基于重量的国内 + 国际配送

**目标**：根据重量对国内和国际配送设置不同的费率。

**解决方案**：
1. 创建 2 个区域："Domestic", "International"
2. 创建 2 个费率表："Domestic Weight", "International Weight"
3. 创建 2 个配送方式：
   - "Domestic Shipping" → 链接到 Domestic 区域 + Domestic Weight 表
   - "International Shipping" → 链接到 International 区域 + International Weight 表

---

### 场景 4：多种承运商选项

**目标**：让客户可以选择 FedEx Ground、FedEx Express、UPS Ground。

**解决方案**：
1. 为 FedEx API 创建提供商账户
2. 为 UPS API 创建提供商账户
3. 创建 3 个实时配送方式：
   - "FedEx Ground" → FedEx 提供商，服务代码 = "FEDEX_GROUND"
   - "FedEx Express" → FedEx 提供商，服务代码 = "FEDEX_EXPRESS"
   - "UPS Ground" → UPS 提供商，服务代码 = "UPS_GROUND"
4. 所有 3 个配送方式在结账时查询承运商 API 并显示实时费率

### 情景 5：本地自提 + 配送

**目标**：零售商店提供自提和配送两种选项。

**解决方案**：
1. 创建位置："主店"，包含地址、营业时间、准备时间
2. 创建 2 种方法：
   - "本地自提" → 本地自提类型，链接到主店位置
   - "标准配送" → 固定费用 $9.99
3. 顾客在结账时看到这两种选项

## 测试配送方法

上线前测试所有方法：

1. **创建测试购物车**
   - 添加不同重量/价格的产品
   - 进入结账流程

2. **测试每种方法**
   - 在不同区域输入地址
   - 验证是否显示正确的配送方法
   - 检查计算费用是否符合预期

3. **测试限制条件**
   - 添加商品直到达到最低订单金额 → 验证是否显示免费配送
   - 添加重物 → 验证基于重量的分层是否生效
   - 测试区域限制 → 验证被排除区域的方法是否隐藏

4. **测试实时方法**（如适用）
   - 使用承运商测试凭据
   - 验证是否成功返回费率
   - 检查费率与承运商网站是否一致

## 故障排除

**问题 1：结账时方法未显示**

**原因**：
- 方法未激活
- 购物车未达到最低/最高订单金额
- 购物车未达到最低/最高重量
- 顾客地址不匹配任何关联区域
- 没有费率表分层覆盖购物车重量/价格

**解决方案**：检查限制条件，验证激活状态，确保区域/分层覆盖顾客的场景。

---

**问题 2：实时费率失败**

**原因**：
- 无效的 API 凭据
- 提供商账户未激活
- 未定义配送包裹（承运商需要尺寸）
- 未设置原地址
- 承运商 API 故障

**解决方案**：测试提供商连接，验证凭据，确保已配置包裹，检查设置中的原地址。

---

**问题 3：计算费用不正确**

**原因**：
- 费率表分层存在间隙或重叠
- 分层的最小/最大值单位错误（克 vs 千克）
- 意外添加了处理费
- 配送规则修改了费用


**解决方案**：检查费率表层级，确认单位，检查运费促销优先级。

---

## 小贴士

- **从简单开始** - 第一个方法使用固定费率，按需增加复杂度
- **彻底测试** - 在生产环境中启用前，在测试环境中验证所有方法
- **使用描述性名称** - "标准运输（5-7 天）" 比 "方法 1" 更好
- **设置现实的配送时间** - 低估时间，超预期交付以提高客户满意度
- **如果可能，提供自提服务** - 降低运费成本，提高客户便利性
- **监控承运商 API 的可靠性** - 如果实时费率失败，使用固定费率作为后备
- **对国际运输使用区域划分** - 按地区划分费率，避免在昂贵目的地产生损失
- **与运费促销结合使用** - 规则添加条件逻辑（免费运费促销，偏远地区附加费）
- **保持方法有限** - 结账时提供 2-4 个选项，避免决策瘫痪
- **按季节更新费率表** - 承运商费率会变化，每年审查一次
- **明智地使用优先级** - 将免费/便宜的选项放在前面，昂贵的选项放在最后
