---
title: 配送方式
---

配送方式是结账时面向客户的配送选项，每种方式使用不同的定价策略计算运费。Spwig 支持 7 种配送方式，从简单的固定费率到复杂的承运商实时计算的定价。配送方式可以通过最低/最高订单价值、重量和地理区域进行限制。客户在结账时选择首选的配送方式，计算出的费用将添加到订单总额中。

使用本指南配置符合您业务模式的配送方式，从基本的固定费率配送到复杂的基于区域的分层定价。

## 配送方式类型

Spwig 提供 7 种配送方式类型，每种类型有不同的费用计算逻辑：

### 固定费率配送

**是什么**：无论购物车内容、目的地或重量，费用都是固定的。

**何时使用**：
- 预测运费的简单商店
- 单一产品类型（类似尺寸/重量）
- 仅国内配送，使用标准承运商费率
- 免运费门槛促销（与配送规则一起使用）

**配置**：
- 设置 **配送方式类型** = 固定费率
- 输入 **固定费用**（例如，9.99 美元）
- 可选：设置最低/最高订单价值限制

**示例**：所有国内订单使用“标准配送 - 9.99 美元”。

---

### 免运费

**是什么**：零费用的配送选项（客户无需支付费用）。

**何时使用**：
- 免运费促销
- 高价值订单（与最低订单价值结合使用）
- 本地自提的替代方案
- 忠诚度计划福利

**配置**：
- 设置 **配送方式类型** = 免运费
- 可选：设置 **最低订单价值**（例如，订单满 50 美元免运费）
- 与配送规则结合使用，实现条件性免运费

**示例**：最低订单价值 = 50 美元，显示“订单满 50 美元免运费”。

---

### 基于重量的配送

**是什么**：根据购物车总重量，从分层费率表中计算费用。

**何时使用**：
- 重量可变的产品（书籍、硬件、杂货）
- 基于重量的承运商定价模型
- 预测的重量与费用比率

**配置**：
1. 设置 **配送方式类型** = 基于重量
2. 创建 **配送费率表**，basis_type = "weight"
3. 添加 **配送费率分层**（例如，0-5kg = 10 美元，5-10kg = 15 美元，10-20kg = 25 美元）
4. 可选：限制到特定区域

**示例**：
```
0-2kg: 8 美元
2-5kg: 12 美元
5-10kg: 18 美元
10kg+: 25 美元
```

**工作原理**：购物车计算总重量 → 找到匹配的分层 → 返回该分层的费率。

---

### 基于价格的配送

**是什么**：根据购物车小计，从分层费率表中计算费用。

**何时使用**：
- 配送费用与订单价值相关
- 鼓励更高的购物车价值（在更高分层中每美元费率较低）
- 与基于重量的配送相比，适用于类似价格的商品的简单替代方案

**配置**：
1. 设置 **配送方式类型** = 基于价格
2. 创建 **配送费率表**，basis_type = "price"
3. 添加 **配送费率分层**（例如，0-50 美元 = 9.99 美元，50-100 美元 = 14.99 美元，100+ 美元 = 19.99 美元）

**示例**：
```
0-25 美元: 6.99 美元
25-75 美元: 9.99 美元
75-150 美元: 12.99 美元
150+ 美元: 免费
```

**工作原理**：购物车计算小计 → 找到匹配的分层 → 返回该分层的费率。

---

### 实时承运商费率

**是什么**：结账时从承运商 API（FedEx、UPS、DHL）获取实时费率。

**何时使用**：
- 根据目的地不同，运费变化
- 为客户提供多种承运商选项
- 在不使用手动费率表的情况下准确获取承运商定价
- 国际配送，具有复杂的定价

**配置**：
1. 设置 **配送方式类型** = 实时
2. 创建 **提供商账户**（设置 > 配送 > 提供商账户）
3. 输入承运商 API 凭据（账户号、API 密钥、密钥）
4. 将提供商账户链接到配送方式
5. 可选：添加利润率百分比或固定利润率

**要求**：
- 激活的承运商账户（FedEx、UPS、DHL 等）
- 承运商的 API 凭据
- 定义的配送包裹（用于计算尺寸重量）

**示例**："FedEx Ground" 方法在结账时根据购物车重量、尺寸和目的地获取实时 FedEx 费率。

**工作原理**：
1. 客户在结账时输入地址
2. 系统使用原产地、目的地、包裹尺寸和重量调用承运商 API
3. 承运商返回费率报价
4. 可选利润率应用
5. 费率显示给客户

---

### 本地自提

**是什么**：客户在物理地点自提订单（无配送费用）。

**何时使用**：
- 提供自提服务的零售店
- 仓库自提选项
- 活动或市集摊位
- 为本地客户消除配送费用

**配置**：
1. 设置 **配送方式类型** = 本地自提
2. 创建 **位置**（设置 > 配送 > 位置）
   - 设置地址、营业时间、自提容量
3. 将位置链接到配送方式
4. 可选：设置自提准备时间（例如，"2 小时后准备就绪"）

**客户体验**：
- 在结账时选择“本地自提”
- 如果有多个位置，选择自提位置
- 根据可用性选择自提日期和时间
- 在订单准备就绪时收到通知

**示例**：带有 3 个零售位置，准备时间在 24 小时内的“在商店自提 - 免费”。

---

### 表格费率配送

**是什么**：基于重量、价格或数量的灵活分层定价，具有高级区域定位功能。

**何时使用**：
- 复杂定价（按区域和重量不同费率）
- 需要比基于重量或价格的配送方式有更多控制
- 多个定价因素（例如，重量 + 目的地 + 数量）

**配置**：
1. 设置 **配送方式类型** = 表格费率
2. 创建 **配送费率表**
3. 定义 **basis_type**：重量、价格或数量
4. 添加 **配送费率分层**，带有最小/最大值
5. 可选：将分层限制到特定区域或国家

**与基于重量/价格的差异**：表格费率支持按分层的地理限制，允许在不同区域中相同重量/价格的不同费率。

**示例**：
```
区域 A（国内）：
  0-5kg: 10 美元
  5-10kg: 15 美元

区域 B（偏远地区）：
  0-5kg: 18 美元
  5-10kg: 25 美元
```

**工作原理**：购物车计算基础值（重量/价格/数量） → 找到客户区域的匹配分层 → 返回该分层的费率。

## 配送方式配置

所有配送方式共享这些常见设置：

### 基本设置

- **名称**：内部标识符（不显示给客户）
- **显示名称**：结账时面向客户的名称（例如，"标准配送"，"特快专递"）
- **描述**：结账时显示的可选帮助文本（例如，"3-5 个工作日送达"）
- **配送方式类型**：上述 7 种类型之一
- **启用**：切换以启用或禁用方法而不删除

### 费用设置

- **固定费用**：仅适用于固定费率方式
- **费率表**：适用于基于重量、基于价格、表格费率方式
- **提供商账户**：适用于实时承运商方式
- **税率类别**：对配送费用征税（如适用）

### 限制

**订单价值限制**：
- **最低订单价值**：当购物车小计 ≥ 金额时，该方式才可用（例如，订单满 50 美元免运费）
- **最高订单价值**：当购物车小计 > 金额时，该方式隐藏（例如，固定费率仅适用于低于 100 美元的订单）

**重量限制**：
- **最低重量**：当购物车重量 ≥ 金额时，该方式才可用
- **最高重量**：当购物车重量 > 金额时，该方式隐藏（常见于轻量配送选项）

**地理限制**：
- **配送区域**：将方式链接到特定区域（国内、国际、区域）
- 空区域 = 可用于所有地址
- 多个区域 = 可用于任何匹配的区域

### 高级设置

- **优先级**：在结账时的显示顺序（数字越低，列表中位置越高）
- **处理费**：添加到计算费用的附加固定费用
- **免运费门槛**：如果购物车小计 ≥ 阈值，自动将费用设为 0 美元（替代最低订单价值）

## 创建配送方式

**逐步工作流程**：

1. **导航到配送方式**
   - 前往 设置 > 购物车 > 配送方式
   - 点击 "添加配送方式"

2. **选择配送方式类型**
   - 根据您的定价策略选择适当类型
   - 类型决定了可用的费用配置字段

3. **配置基本信息**
   - 名称：内部参考（例如，"domestic_ground"）
   - 显示名称：面向客户（例如，"地面配送"）
   - 描述：配送时间（例如，"5-7 个工作日送达"）

4. **设置费用计算**
   - **固定费率**：输入固定费用
   - **基于重量/价格/表格费率**：创建费率表（见下文）
   - **实时**：链接提供商账户
   - **免运费/自提**：无需费用配置

5. **添加限制（可选）**
   - 最低/最高订单价值
   - 最低/最高重量
   - 配送区域

6. **设置优先级**
   - 数字越低，在结账时越靠前
   - 推荐顺序：免运费（1），本地自提（2），标准（3），特快（4）

7. **激活方式**
   - 切换 "启用" = 是
   - 保存

---

## 创建费率表

对于基于重量、基于价格和表格费率方式：

**步骤 1：创建费率表**
- 前往 设置 > 配送 > 费率表
- 点击 "添加费率表"
- 设置 **名称**（例如，"国内重量分层"）
- 设置 **基础类型**：重量、价格或数量

**步骤 2：添加分层**
- 点击 "添加分层"
- 设置 **最小值** 和 **最大值**（匹配范围）
- 设置 **费率**（该分层的费用）
- 可选：限制到特定区域或国家
- 保存分层

**步骤 3：为所有分层重复操作**
- 覆盖完整范围（0 到最大预期值）
- 确保没有间隙（例如，0-5，5-10，10-20，20+）
- 在最后一层使用 `null` 表示最大值（无限制）

**步骤 4：链接到配送方式**
- 编辑配送方式
- 从下拉菜单中选择费率表
- 保存

**示例基于重量的表**：
```
名称：国内重量分层
基础：重量

分层：
1. 最小：0g，最大：2000g，费率：8 美元
2. 最小：2000g，最大：5000g，费率：12 美元
3. 最小：5000g，最大：10000g，费率：18 美元
4. 最小：10000g，最大：null，费率：25 美元
```

## 常见配送场景

### 场景 1：基本国内配送

**目标**：所有国内订单使用简单的 9.99 美元固定费率。

**解决方案**：
- 配送方式类型：固定费率
- 固定费用：9.99 美元
- 配送区域："国内"（仅您所在国家）

---

### 场景 2：订单满 50 美元免运费

**目标**：通过免运费门槛鼓励更高的购物车价值。

**解决方案 A（推荐）**：
- 配送方式类型：免运费
- 最低订单价值：50 美元
- 显示名称："订单满 50 美元免运费"

**解决方案 B（使用规则）**：
- 配送方式类型：固定费率
- 固定费用：9.99 美元
- 创建配送规则：
  - 条件：购物车价值 ≥ 50 美元
  - 动作：将费用设为 0 美元

---

### 场景 3：基于重量的国内 + 国际配送

**目标**：根据重量，国内和国际配送使用不同的费率。

**解决方案**：
1. 创建 2 个区域："国内"，"国际"
2. 创建 2 个费率表："国内重量"，"国际重量"
3. 创建 2 个方法：
   - "国内配送" → 链接到国内区域 + 国内重量表
   - "国际配送" → 链接到国际区域 + 国际重量表

---

### 场景 4：多个承运商选项

**目标**：让客户在 FedEx Ground、FedEx Express、UPS Ground 之间选择。

**解决方案**：
1. 创建 FedEx API 的提供商账户
2. 创建 UPS API 的提供商账户
3. 创建 3 个实时方法：
   - "FedEx Ground" → FedEx 提供商，服务代码 = "FEDEX_GROUND"
   - "FedEx Express" → FedEx 提供商，服务代码 = "FEDEX_EXPRESS"
   - "UPS Ground" → UPS 提供商，服务代码 = "UPS_GROUND"
4. 所有 3 个方法在结账时查询承运商 API 并显示实时费率

---

### 场景 5：本地自提 + 配送

**目标**：零售店提供自提和配送选项。

**解决方案**：
1. 创建位置："主店"，带有地址、营业时间、准备时间
2. 创建 2 个方法：
   - "本地自提" → 本地自提类型，链接到主店位置
   - "标准配送" → 固定费率 9.99 美元
3. 客户在结账时看到两种选项

## 测试配送方式

在上线之前，测试所有方法：

1. **创建测试购物车**
   - 添加具有不同重量/价格的产品
   - 前往结账

2. **测试每种方法**
   - 在不同区域输入地址
   - 验证正确的配送方式出现
   - 检查计算出的费用是否符合预期

3. **测试限制**
   - 添加商品直到满足最低订单价值 → 验证免运费是否出现
   - 添加重物 → 验证基于重量的分层是否正常工作
   - 测试区域限制 → 验证排除区域的方法是否隐藏

4. **测试实时方法**（如适用）
   - 使用承运商测试凭据
   - 验证是否成功返回费率
   - 检查费率与承运商网站的准确性

## 故障排除

**问题 1：配送方式在结账时未显示**

**原因**：
- 配送方式未启用
- 购物车未满足最低/最高订单价值
- 购物车未满足最低/最高重量
- 客户地址不匹配任何链接的区域
- 没有费率表分层覆盖购物车重量/价格

**解决方案**：检查限制，验证启用状态，确保区域/分层覆盖客户场景。

---

**问题 2：实时费率失败**

**原因**：
- 无效的 API 凭据
- 提供商账户未启用
- 未定义配送包裹（承运商需要尺寸）
- 未设置原产地地址
- 承运商 API 故障

**解决方案**：测试提供商连接，验证凭据，确保已配置包裹，检查设置中的原产地地址。

---

**问题 3：计算出的费用不正确**

**原因**：
- 费率表分层存在间隙或重叠
- 分层的最小/最大值单位错误（克 vs 千克）
- 意外添加处理费
- 配送规则修改了费用

**解决方案**：审查费率表分层，验证单位，检查配送规则优先级。

## 建议

- **从简单开始** - 使用固定费率作为第一个方法，根据需要添加复杂性
- **彻底测试** - 在启用生产环境之前，在测试环境中验证所有方法
- **使用描述性名称** - "标准配送（5-7 天）" 比 "方法 1" 更好
- **设置现实的配送时间** - 低估，超时交付以提高客户满意度
- **如果可能，提供自提** - 减少配送费用，提高客户便利性
- **监控承运商 API 的可靠性** - 如果实时费率失败，使用固定费率作为后备
- **使用区域进行国际配送** - 按区域不同费率，防止在昂贵目的地产生损失
- **与配送规则结合使用** - 规则添加条件逻辑（免运费促销，偏远地区附加费）
- **保持方法有限** - 结账时 2-4 个选项可防止决策瘫痪
- **定期更新费率表** - 承运商费率会变化，每年审查一次
- **明智地使用优先级** - 将免费/低价选项放在前面，昂贵选项放在最后
