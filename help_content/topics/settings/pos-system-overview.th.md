ระบบ POS ของ Spwig ช่วยเปลี่ยนร้านค้าของคุณให้เป็นโซลูชันค้าปลีกที่สมบูรณ์แบบด้วยเทอร์มินัลจุดขายที่ทันสมัย ติดตั้งเทอร์มินัลไม่จำกัดจำนวนที่สถานที่ไม่จำกัดจำนวนด้วยค่าธรรมเนียมใบอนุญาตคงที่ €499/ปี แต่ละเทอร์มินัลเป็น Progressive Web App (PWA) ที่ทำงานได้โดยไม่มีอินเทอร์เน็ต ซิงค์อัตโนมัติ และผสานรวมกับสต็อกสินค้า ข้อมูลลูกค้า และการประมวลผลการชำระเงิน จัดการทุกอย่างจากแดชบอร์ดผู้ดูแลระบบ—การตั้งค่าเทอร์มินัล การตรวจสอบความถูกต้องของช่วงเวลาการทำงาน การปรับแต่งใบเสร็จ และการผสานรวมฮาร์ดแวร์

ใช้ระบบ POS เมื่อคุณมีสถานที่ค้าปลีกทางกายภาพ ร้านค้าชั่วคราว งานแสดงสินค้า หรือสภาพแวดล้อมใด ๆ ที่ลูกค้าซื้อสินค้าโดยตรงแทนที่จะซื้อออนไลน์

## Spwig POS คืออะไร?

Spwig POS เป็นระบบจุดขายที่ผสานรวมกันอย่างสมบูรณ์ ที่ออกแบบมาสำหรับผู้ขายที่ขายทั้งออนไลน์และในสถานที่ทางกายภาพ ต่างจากระบบ POS ของบุคคลที่สามที่ต้องการการผสานรวมที่ซับซ้อน Spwig POS ถูกสร้างขึ้นโดยตรงในแพลตฟอร์มของคุณ ทำให้ข้อมูลสอดคล้องกันอย่างสมบูรณ์ในทุกช่องทางการขาย

**คุณลักษณะหลัก**:
- **เทอร์มินัลไม่จำกัดจำนวน** - ติดตั้งเทอร์มินัลได้ตามที่คุณต้องการโดยไม่มีค่าใช้จ่ายเพิ่มเติม
- **สถาปัตยกรรมที่เน้นการทำงานแบบออฟไลน์** - ยังคงประมวลผลการขายได้แม้จะสูญเสียการเชื่อมต่ออินเทอร์เน็ต
- **Progressive Web App** - ไม่จำเป็นต้องติดตั้งจากแอปสโตร์ สามารถเข้าถึงได้ผ่านเบราว์เซอร์บนอุปกรณ์ใด ๆ (แท็บเล็ต คอมพิวเตอร์ เทอร์มินัลเฉพาะ)
- **การซิงค์สต็อกจริง** - การจองสต็อก (TTL 15 นาที) ป้องกันการขายเกินจำนวนที่มีอยู่ในช่องทางต่าง ๆ
- **การสนับสนุนการชำระเงินแบบแยกส่วน** - ยอมรับวิธีการชำระเงินหลายแบบต่อรายการ (เงินสด + บัตร + บัตรของขวัญ)
- **การผสานรวมฮาร์ดแวร์** - พิมพ์ใบเสร็จแบบเทอร์มอล ESC/POS สแกนเนอร์บาร์โค้ด ลิ้นชักเงิน หน้าจอแสดงผลลูกค้า
- **การจัดการช่วงเวลาการทำงาน** - การตรวจสอบความถูกต้องของเงินสดกับการนับเงินเริ่มต้น/สิ้นสุดและติดตามความคลาดเคลื่อน
- **พร้อมสำหรับหลายสถานที่** - กลุ่มร้านค้าที่มีการกำหนดค่าที่สืบทอดเพื่อจัดการแฟรนไชส์และภูมิภาค

## การอนุญาตและการเปิดใช้งาน

**ราคาแบบคงที่**: €499 ต่อปีครอบคลุมเทอร์มินัลไม่จำกัดจำนวนที่สถานที่ไม่จำกัดจำนวน ไม่มีค่าธรรมเนียมต่อเทอร์มินัล ไม่มีค่าธรรมเนียมการซื้อขาย ไม่มีค่าใช้จ่ายแฝง

**รูปแบบใบอนุญาต**: `POS-XXXX-XXXX-XXXX-XXXX` (จะได้รับหลังจากการซื้อ)

**การเปิดใช้งาน**: ใส่คีย์ใบอนุญาตของคุณใน **Settings > POS Licensing** ระบบจะตรวจสอบกับเซิร์ฟเวอร์ใบอนุญาตของ Spwig และเปิดใช้งานคุณสมบัติ POS ทั้งหมดทันที ใบอนุญาตมีช่วงเวลา 14 วันหลังจากหมดอายุเพื่อให้สามารถประมวลผลการชำระเงินได้ทันเวลา

**สิ่งที่คุณได้รับ**:
- การลงทะเบียนเทอร์มินัลไม่จำกัดจำนวน
- การกำหนดงานพนักงานไม่จำกัดจำนวน
- คุณสมบัติ POS ทั้งหมด (ช่วงเวลาการทำงาน การจัดการเงินสด การปรับแต่งใบเสร็จ หน้าจอแสดงผลลูกค้า)
- การผสานรวมผู้ให้บริการชำระเงิน (Stripe Terminal และระบบผู้ให้บริการที่ขยายได้)
- การสนับสนุนการผสานรวมฮาร์ดแวร์
- การอัปเดตและแก้ไขข้อผิดพลาดในช่วงเวลาใบอนุญาต

ไม่มีคุณสมบัติ POS ใดสามารถเข้าถึงได้โดยไม่มีใบอนุญาตที่ถูกต้อง—อินเทอร์เฟซการจับคู่เทอร์มินัล การจัดการช่วงเวลาการทำงาน และหน้า POS ผู้ดูแลระบบล้วนต้องการการเปิดใช้งาน

## สถาปัตยกรรมระบบ

**Frontend** - Progressive Web App ด้วย React 18:
- เน้นการทำงานแบบออฟไลน์ด้วยการแคช Service Worker (ทำงานได้โดยไม่มีอินเทอร์เน็ต)
- ระบบสร้าง Vite สำหรับการโหลดที่รวดเร็ว
- CSS Modules + โทเคนการออกแบบ (สอดคล้องกับธีมร้านค้าของคุณ)
- IndexedDB สำหรับการเก็บข้อมูลท้องถิ่น
- 10 ภาษาที่รองรับ (อังกฤษ จีน (แบบง่าย/แบบซับซ้อน) ฝรั่งเศส เยอรมัน สเปน โปรตุเกส ญี่ปุ่น รัสเซีย อารบิก)

**Backend** - การผสานรวมด้านหลัง:
- 13 โมเดล POS (POSTerminal POSShift CashMovement ReceiptTemplate PromoSlide ฯลฯ)
- 43+ จุดปลายทาง REST API สำหรับการดำเนินการเทอร์มินัล
- ระบบการจองสต็อกที่มีการจัดการ TTL
- งาน Celery สำหรับการซิงค์ในพื้นหลัง
- การจัดเก็บข้อมูลผู้ให้บริการชำระเงินที่เข้ารหัส

**ความปลอดภัย**:
- การจับคู่เทอร์มินัลผ่านรหัส 8 ตัวอักษร (สร้างขึ้นด้วยเซิร์ฟเวอร์ หมดอายุหลังจากใช้งาน)
- การควบคุมการกำหนดงานพนักงานที่กำหนดว่าผู้ใช้ใดสามารถเข้าถึงเทอร์มินัลใดได้
- ความสามารถในการล็อก/ปลดล็อกแบบระยะไกลสำหรับกรณีฉุกเฉินของผู้ดูแลระบบ
- การจัดเก็บข้อมูลผู้ให้บริการชำระเงินที่เข้ารหัส
- การตรวจสอบความถูกต้องแบบเซสชันพร้อมการปลดล็อกด้วยการตรวจสอบชีวภาพ (ขึ้นอยู่กับเบราว์เซอร์)

## ขั้นตอนการเริ่มต้น

ทำตามขั้นตอน 5 ขั้นตอนต่อไปนี้เพื่อติดตั้งเทอร์มินัล POS แรกของคุณ:

**ขั้นตอนที่ 1: เปิดใช้งานใบอนุญาต POS**
- ไปที่ **Settings > POS Licensing**
- ใส่คีย์ใบอนุญาตของคุณ (`POS-XXXX-XXXX-XXXX-XXXX`)
- ตรวจสอบใบอนุญาต (ต้องการการเชื่อมต่ออินเทอร์เน็ต)
- ยืนยันการเปิดใช้งาน

**ขั้นตอนที่ 2: สร้างคลังสินค้า**
- ไปที่ **Catalog > Warehouses**
- สร้างคลังสินค้าที่แสดงถึงสถานที่ค้าปลีกของคุณ
- ตั้งค่าที่อยู่และข้อมูลการติดต่อ
- คลังสินค้านี้จะติดตามสต็อกสินค้าทางกายภาพสำหรับการขาย POS

**ขั้นตอนที่ 3: ลงทะเบียนเทอร์มินัล**
- ไปที่ **POS > Terminals**
- คลิก **+ Add Terminal**
- ตั้งชื่อเทอร์มินัล (เช่น "Main Register", "Checkout 1")
- ระบุคลังสินค้าจากขั้นตอนที่ 2
- ตั้งค่าการตั้งค่าฮาร์ดแวร์ (เครื่องพิมพ์ ตัวสแกนบาร์โค้ด ลิ้นชักเงิน)
- บันทึกเพื่อสร้างรหัสจับคู่ 8 ตัวอักษร

**ขั้นตอนที่ 4: กำหนดงานพนักงาน**
- ในการตั้งค่าเทอร์มินัล ให้เลื่อนไปที่ **Assigned Users**
- เลือกพนักงานที่ได้รับอนุญาตให้ใช้เทอร์มินัลนี้
- เพียงผู้ใช้ที่ได้รับการกำหนดงานเท่านั้นที่สามารถเข้าสู่ระบบเทอร์มินัลได้
- ผู้ใช้ต้องมีสิทธิ์ POS ที่เหมาะสมในบทบาทพนักงานของพวกเขา

**ขั้นตอนที่ 5: จับคู่อุปกรณ์**
- บนอุปกรณ์เทอร์มินัลของคุณ (แท็บเล็ต/คอมพิวเตอร์) ไปที่ URL `/pos/`
- ใส่รหัสจับคู่ 8 ตัวอักษรจากขั้นตอนที่ 3
- เทอร์มินัลจะดาวน์โหลดการตั้งค่าและซิงค์ข้อมูลเริ่มต้น
- เข้าสู่ระบบด้วยข้อมูลผู้ใช้ที่ได้รับการกำหนดงาน
- เทอร์มินัลพร้อมสำหรับการขายแล้ว

หลังจากจับคู่ เทอร์มินัลจะซิงค์อัตโนมัติทุก 5 นาที (ปรับแต่งได้) โหมดออฟไลน์ช่วยให้ดำเนินการต่อได้เมื่อไม่มีการเชื่อมต่ออินเทอร์เน็ต—การขายจะซิงค์อัตโนมัติเมื่อการเชื่อมต่อกลับมา

## คุณสมบัติหลักของ POS

**การประมวลผลการขาย**:
- ค้นหาสินค้าตามชื่อ SKU หรือบาร์โค้ด
- การชำระเงินแบบแยกส่วน (วิธีการชำระเงินหลายแบบต่อรายการ)
- ตะกร้าที่ถูกเก็บไว้ (บันทึกการซื้อขายที่ไม่สมบูรณ์)
- การคืนเงินและยกเลิกการขายพร้อมการติดตามเหตุผล
- การใช้ส่วนลด (คูปอง บัตรของขวัญ การส่งเสริมการขาย)
- การค้นหาลูกค้าและการแลกคะแนนสะสม

**การจัดการเงินสด**:
- การเปิดช่วงเวลาการทำงานพร้อมการนับเงินเริ่มต้น
- การปิดช่วงเวลาการทำงานพร้อมการตรวจสอบความถูกต้องระหว่างคาดการณ์กับจริง
- การเคลื่อนไหวของเงินสด (การเพิ่มเงินสำรอง การถอนเงินเล็กน้อยพร้อมเหตุผล)
- การคำนวณเงินสดที่คาดการณ์โดยอัตโนมัติจากยอดขายเงินสด
- การติดตามและรายงานความคลาดเคลื่อน

**การผสานรวมฮาร์ดแวร์**:
- เครื่องพิมพ์ใบเสร็จแบบเทอร์มอล ESC/POS (เครือข่ายหรือซีเรียล)
- สแกนเนอร์บาร์โค้ดผ่าน USB
- การเปิดลิ้นชักเงินผ่านการกระตุ้นจากเครื่องพิมพ์
- หน้าจอแสดงผลสำหรับลูกค้า (การเลื่อนภาพส่งเสริมการขายเมื่อไม่มีการใช้งาน)
- ตัวอ่านบัตร Stripe Terminal (S700, WisePOS E, P400)

**ความสามารถในการทำงานแบบออฟไลน์**:
- Service Worker แคชทรัพยากรทั้งหมดของเทอร์มินัล
- IndexedDB เก็บคำสั่งซื้อล่าสุด (ปรับแต่งได้: 7-30 วัน 200-1000 คำสั่งซื้อ)
- การจองสต็อกพร้อม TTL 15 นาทีป้องกันการขายเกินจำนวน
- คิวการขายสำหรับซิงค์เมื่อการเชื่อมต่อกลับมา
- การตรวจจับการเชื่อมต่ออัตโนมัติ

## หน้าผู้ดูแลระบบ POS

เข้าถึงหน้าผู้ดูแลระบบเหล่านี้เพื่อจัดการทุกด้านของการติดตั้ง POS ของคุณ:

**แดชบอร์ด POS** (`/admin/pos/`)
- ภาพรวมระบบและสถิติที่รวดเร็ว
- การใช้งานเทอร์มินัลล่าสุด
- สรุปช่วงเวลาการทำงานที่ใช้งานอยู่
- สถานะใบอนุญาตและวันหมดอายุ

**การจัดการเทอร์มินัล** (`/admin/pos_app/posterminal/`)
- ลงทะเบียนและตั้งค่าเทอร์มินัล
- กำหนดงานพนักงานและคลังสินค้า
- ตรวจสอบสถานะออนไลน์/ออฟไลน์ (การติดตามการเต้นหัวใจ)
- ปลดล็อกเทอร์มินัลจากระยะไกล
- [เรียนรู้เพิ่มเติม: การจัดการเทอร์มินัล POS](managing-pos-terminals)

**การจัดการช่วงเวลาการทำงาน** (`/admin/pos_app/posshift/`)
- ดูช่วงเวลาการทำงานทั้งหมด (เปิด ปิด ประวัติ)
- ตรวจสอบรายงานการตรวจสอบความถูกต้องของเงินสด
- ติดตามการเคลื่อนไหวของเงินสดและความคลาดเคลื่อน
- ตรวจสอบกิจกรรมช่วงเวลาการทำงาน
- [เรียนรู้เพิ่มเติม: ช่วงเวลาการทำงาน POS และการจัดการเงินสด](pos-shifts-cash-management)

**กลุ่มร้านค้า** (`/admin/pos_app/storegroup/`)
- จัดกลุ่มเทอร์มินัลตามสถานที่/ภูมิภาค
- ตั้งค่าการตั้งค่าระดับกลุ่ม (สกุลเงิน ภาษา เวลาพื้นที่)
- ดำเนินการลำดับชั้นการสืบทอดการตั้งค่า
- [เรียนรู้เพิ่มเติม: กลุ่มร้านค้า POS](pos-store-groups)

**เทมเพลตใบเสร็จ** (`/admin/pos_app/receipttemplate/`)
- ปรับแต่งใบเสร็จที่พิมพ์ (ความกว้างของกระดาษ โลโก้ หัว/ท้าย)

- กำหนดขอบเขตเทมเพลตให้กับร้านค้าหรือกลุ่มเฉพาะ

## การติดตั้งหลายสถานที่

สำหรับผู้ขายที่มีสถานที่ค้าปลีกหลายแห่ง Spwig POS สนับสนุนลำดับชั้นการสืบทอดการตั้งค่า:

**ลำดับชั้นการตั้งค่า** (ลำดับความสำคัญสูงสุดถึงต่ำสุด):
1. การตั้งค่าเฉพาะเทอร์มินัล (ทับซ้อนทุกอย่าง)
2. การตั้งค่าเฉพาะร้านค้า (ทับซ้อนกลุ่มและไซต์)
3. การตั้งค่ากลุ่ม (ทับซ้อนค่าเริ่มต้นของไซต์)
4. ค่าเริ่มต้นของไซต์ (ค่าเริ่มต้นสำหรับทุกอย่าง)

ตั้งค่าการตั้งค่าที่ใช้ร่วมกันที่ระดับกลุ่ม (เช่น สกุลเงินภูมิภาค ภาษา) และทับซ้อนสำหรับร้านค้าหรือเทอร์มินัลเฉพาะตามที่จำเป็น ดู [กลุ่มร้านค้า POS](pos-store-groups) สำหรับคำแนะนำการตั้งค่าที่ละเอียดอ่อน

## เคล็ดลับ

- **เริ่มต้นด้วยเทอร์มินัลหนึ่งเครื่อง** - ทดสอบการตั้งค่าและขั้นตอนการทำงาน POS กับเทอร์มินัลเดียวก่อนที่จะติดตั้งทั่วทั้งฟลีต
- **กำหนดคลังสินค้าก่อนจับคู่** - เทอร์มินัลไม่สามารถประมวลผลการขายได้หากไม่มีการกำหนดคลังสินค้า
- **ตั้งค่าเทมเพลตใบเสร็จตั้งแต่เนิ่นๆ** - ฟิลด์ที่เกี่ยวข้องกับกฎหมาย (หมายเลขภาษี) แตกต่างกันไปตามภูมิภาค; ตั้งค่าก่อนเริ่มต้นใช้งาน
- **ทดสอบโหมดออฟไลน์** - ตัดการเชื่อมต่ออินเทอร์เน็ตและตรวจสอบว่าการขายยังดำเนินต่อได้; ยืนยันการซิงค์เมื่อเชื่อมต่ออีกครั้ง
- **ใช้กลุ่มร้านค้าสำหรับหลายสถานที่** - ทำให้การจัดการการตั้งค่าง่ายขึ้นสำหรับการติดตั้งแฟรนไชส์หรือภูมิภาค
- **ตรวจสอบสถานะการเต้นหัวใจ** - เทอร์มินัลจะส่งข้อความไปยังเซิร์ฟเวอร์ทุก 5 นาที; เทอร์มินัลที่อยู่ในโหมดออฟไลน์จะปรากฏในแดชบอร์ดผู้ดูแลระบบ
- **ตั้งค่าขีดจำกัดการซิงค์เพื่อประสิทธิภาพ** - เทอร์มินัลที่มีการเชื่อมต่อช้าจะได้ประโยชน์จากค่าการตั้งค่า sync_days/sync_limit ที่ต่ำกว่า
e- **สำรองการตั้งค่าฮาร์ดแวร์** - จัดทำเอกสาร IP ของเครื่องพิมพ์ การตั้งค่าสแกนเนอร์ และการตั้งค่าลิ้นชักเงินเพื่อการฟื้นฟูข้อมูลในกรณีเกิดภัยพิบัติ
