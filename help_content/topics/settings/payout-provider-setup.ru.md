---
title: Настройка провайдера выплат
---

Настройка провайдера выплат позволяет настроить PayPal и Airwallex для автоматических выплат аффилиатам. Это руководство показывает, как подключить аккаунты провайдеров выплат, настроить вебхуки и протестировать интеграцию.

## Поддерживаемые провайдеры выплат

Spwig интегрируется с двумя провайдерами выплат для автоматизации выплат аффилиатам:

| Провайдер | Метод оплаты | Обработка | Поддержка пакетов | Лучше подходит для |
|----------|----------------|------------|---------------|----------|
| **PayPal** | Переводы через аккаунт PayPal | На основе API | Да (до 15 000) | Большинству аффилиатов, глобальное охват |
| **Airwallex** | Международные банковские переводы | На основе API | Нет (индивидуальные) | Банковские переводы, международные платежи |

### Основные различия

**Выплаты PayPal**:
- Требуется, чтобы у аффилиата был аккаунт PayPal (почта для оплаты)
- Обрабатывает пакеты до 15 000 выплат за раз
- Быстрая обработка (1-2 рабочих дня)
- Низкая сложность настройки
- Сборы: ~2% или $0.25-$1.00 за каждую выплату
- Один вебхук для всего пакета

**Airwallex**:
- Поддерживает прямые банковские переводы
- Обрабатывает индивидуальные выплаты по одной
- Дольше обрабатывает (2-5 рабочих дней)
- Поддерживает несколько валют и стран
- Сборы различаются в зависимости от страны назначения
- Индивидуальный вебхук на каждую выплату

Вы можете настроить оба провайдера и позволить аффилиатам выбирать предпочитаемый метод оплаты.

## Почему использовать провайдеров выплат?

Интеграция провайдеров оплаты предлагает значительные преимущества по сравнению с ручными выплатами:

- **Автоматическая обработка** — Нет необходимости вводить данные вручную или выполнять выплаты
- **Эффективность пакетов** — Обрабатывайте десятки или сотни выплат с одного щелчка
- **Подтверждения через вебхуки** — Автоматические обновления статуса при завершении выплат
- **Снижение ошибок** — Система проверяет детали аккаунта перед обработкой
- **Аудит-трек** — Полный список транзакций и ответов от провайдера
- **Быстрые выплаты** — Аффилиаты получают средства быстрее
- **Масштабируемость** — Обрабатывать растущие программы аффилиатов без пропорционального увеличения административной работы

Без интеграции провайдера, вам нужно обрабатывать каждую выплату вручную через ваш банк или панель управления PayPal, а затем возвращаться в Spwig, чтобы отметить выплаты как завершенные.

## Настройка PayPal

Следуйте этим шагам, чтобы настроить выплаты PayPal для автоматических выплат аффилиатам.

### Предварительные требования

Перед началом вам нужно:
- Аккаунт PayPal Business (личные аккаунты не могут использовать API выплат)
- Доступ к [Панели разработчика PayPal](https://developer.paypal.com/dashboard/)
- Одобрение для использования API выплат (после тестирования в песочнике)

### Шаг 1: Создание приложения PayPal

1. **Перейдите** в [Панель разработчика PayPal](https://developer.paypal.com/dashboard/)
2. **Войдите** в свой аккаунт PayPal Business
3. **Нажмите** **My Apps & Credentials** в левой боковой панели
4. **Выберите** вкладку **Live** (или Sandbox для тестирования)
5. **Нажмите** **Create App**
6. **Введите имя приложения** (например, "Spwig Affiliate Payouts")
7. **Выберите тип приложения**: Merchant
8. **Нажмите** **Create App**

PayPal генерирует ваши учетные данные.

### Шаг 2: Получение учетных данных API

После создания приложения:

1. **Скопируйте Client ID** — длинная строка из алфавитно-цифровых символов
2. **Нажмите** **Show** под разделом Secret
3. **Скопируйте Client Secret** — держите это в секрете
4. **Заметьте режим** — Sandbox или Live

### Шаг 3: Включение функции выплат

Приложения PayPal требуют явного разрешения для использования выплат:

1. **Прокрутите** до раздела **Features** в вашем приложении
2. **Найдите** функцию **Payouts**
3. **Нажмите** **Add**, если она еще не включена
4. **Отправьте на одобрение**, если используете режим Live (одобрение занимает 1-2 рабочих дня)

### Шаг 4: Добавление провайдера в Spwig

Теперь добавьте аккаунт PayPal в Spwig:

1. **Перейдите** в **Settings > Payout Providers**
2. **Нажмите** **+ Add PayPal Account**
3. **Заполните форму**:
   - **Account Name**: Описательная метка (например, "Main PayPal Account")
   - **Client ID**: Вставьте из панели разработчика PayPal
   - **Client Secret**: Вставьте из панели разработчика PayPal
   - **Mode**: Выберите Sandbox (тестирование) или Production (активный режим)
   - **Is Active**: Отметьте, чтобы включить
4. **Нажмите Save**

Spwig проверяет учетные данные, запрашивая токен доступа. Если проверка не прошла, проверьте еще раз свой Client ID и Secret.

### Шаг 5: Тестирование соединения

Проверьте интеграцию PayPal:

1. Создайте тестовую выплату в **Affiliate Program > Payouts**
2. Используйте свою собственную почту PayPal как получателя
3. Установите сумму $0.01 (если в режиме Production) или любую сумму (если Sandbox)
4. Обработайте через провайдера
5. Проверьте аккаунт PayPal на входящие платежи
6. Проверьте обновления вебхука статуса выплаты в Spwig

Если вы используете режим Sandbox, создайте тестовый аккаунт PayPal на [PayPal Sandbox](https://developer.paypal.com/dashboard/accounts), чтобы получить тестовые выплаты.

## Настройка Airwallex

Airwallex поддерживает международные банковские переводы для аффилиатов, предпочитающих прямой депозит.

### Предварительные требования

Перед началом вам нужно:
- Аккаунт Airwallex (создайте на [airwallex.com](https://www.airwallex.com))
- Подтвержденный статус бизнес-аккаунта
- Включенный доступ к API (свяжитесь с поддержкой Airwallex, если это необходимо)
- Достаточно средств на вашем аккаунте Airwallex

### Шаг 1: Генерация учетных данных API

1. **Войдите** в [Панель управления Airwallex](https://www.airwallex.com/app/)
2. **Перейдите** в **Settings > API Keys**
3. **Нажмите** **Create API Key**
4. **Введите описание**: "Spwig Affiliate Payouts"
5. **Выберите разрешения**: Включите **Payouts** (чтение и запись)
6. **Нажмите** **Generate**
7. **Скопируйте API Key** — показывается только один раз
8. **Скопируйте Client ID** — отображается вместе с ключом

### Шаг 2: Заметьте вашу среду

Airwallex предоставляет две среды:

- **Demo**: Для тестирования с фиктивными транзакциями
- **Production**: Для реальных денежных переводов

Убедитесь, что вы знаете, к какой среде относится ваш API-ключ.

### Шаг 3: Добавление провайдера в Spwig

Добавьте аккаунт Airwallex в Spwig:

1. **Перейдите** в **Settings > Payout Providers**
2. **Нажмите** **+ Add Airwallex Account**
3. **Заполните форму**:
   - **Account Name**: Описательная метка (например, "Airwallex EUR Account")
   - **API Key**: Вставьте из панели управления Airwallex
   - **Client ID**: Вставьте из панели управления Airwallex
   - **Environment**: Выберите Demo или Production
   - **Is Active**: Отметьте, чтобы включить
4. **Нажмите Save**

Spwig проверяет учетные данные, запрашивая баланс вашего аккаунта.

### Шаг 4: Проверка поддерживаемых стран

Airwallex поддерживает переводы в многие страны, но не все. Проверьте страницу [Airwallex coverage](https://www.airwallex.com/global-business-account/global-transfers), чтобы убедиться, что страны ваших аффилиатов поддерживаются.

Часто поддерживаемые страны включают:
- США
- Великобритания
- Страны Европейского союза
- Австралия
- Канада
- Сингапур
- Гонконг

### Шаг 5: Тестирование банковского перевода

Проверьте интеграцию Airwallex:

1. Создайте тестовую выплату для аффилиата с банковскими деталями
2. Используйте небольшую сумму ($1-$5), если в режиме Production
3. Обработайте через провайдера
4. Проверьте панель управления Airwallex на транзакцию
5. Дождитесь подтверждения вебхука
6. Проверьте завершение выплаты в Spwig

Режим Demo обрабатывает мгновенно. Режим Production занимает 2-5 рабочих дней.

## Логика выбора провайдера

Когда вы обрабатываете выплату, Spwig автоматически выбирает подходящего провайдера на основе метода оплаты аффилиата.

### Поток выбора

1. **Проверьте метод оплаты аффилиата**:
   - Если `payment_email` установлен → аффилиат предпочитает PayPal
   - Если установлены банковские детали → аффилиат предпочитает банковский перевод
2. **Сопоставьте с провайдером**:
   - Электронная почта PayPal → Используйте активный аккаунт провайдера PayPal
   - Банковские детали → Используйте активный аккаунт провайдера Airwallex
3. **Перейдите к первому доступному**, если предпочтительный провайдер не настроен
4. **Показать ошибку**, если не существует подходящего провайдера

### Несколько аккаунтов провайдера

Вы можете настроить несколько аккаунтов для одного провайдера (например, два аккаунта PayPal для разных регионов). Spwig выбирает первый активный аккаунт, который соответствует методу оплаты. Чтобы контролировать, какой аккаунт используется, измените их порядок в списке администратора или установите только один как активный.

## Тестирование интеграции выплат

Всегда тестируйте интеграцию провайдера перед обработкой реальных выплат аффилиатам.

### Тестирование в режиме песочника/демо

1. **Установите провайдера в режим песочника** (PayPal Sandbox или Airwallex Demo)
2. **Создайте тестового аффилиата** с тестовыми данными оплаты
3. **Создайте тестовые комиссии** и утвердите их
4. **Создайте тестовую выплату**, включающую эти комиссии
5. **Обработайте через провайдера** с помощью меню действий
6. **Проследите логи Celery** для запросов API
7. **Проверьте панель провайдера** на транзакцию
8. **Дождитесь вебхука** для обновления статуса выплаты
9. **Проверьте, что комиссии отмечены как оплаченные**

### Тестирование в режиме производства

Перед запуском:

1. **Переключитесь в режим производства** в настройках провайдера
2. **Создайте небольшую тестовую выплату** себе ($0.01-$1.00)
3. **Обработайте** и подождите завершения
4. **Проверьте получение средств** на вашем собственном аккаунте
5. **Проверьте, что сработал вебхук** и обновлен статус
6. **Просмотрите сборы провайдера** по транзакциям

### Распространенные проблемы тестирования

| Проблема | Причина | Решение |
|-------|-------|----------|
| "Неверные учетные данные" | Неправильный API-ключ или несоответствие режима | Проверьте учетные данные, убедитесь, что режим песочника и производства совпадает |
| Вебхук никогда не срабатывает | URL не настроен в провайдере | Добавьте URL вебхука в панель управления провайдера |
| Выплата остается в обработке | Неудачный подписи вебхука | Проверьте, совпадает ли секрет вебхука |
| Нет доступного провайдера | Нет активного провайдера для метода оплаты | Включите как минимум один аккаунт провайдера |

## Обработка пакетов (PayPal)

PayPal поддерживает обработку пакетов для повышения эффективности и экономии затрат.

### Как работает пакетирование

Когда вы выбираете несколько выплат и нажимаете **Process with Provider**:

1. Spwig группирует все выплаты PayPal в один пакет
2. Система отправляет один запрос API со всеми деталями выплат (до 15 000)
3. PayPal обрабатывает весь пакет как одну транзакцию
4. Вебхук возвращает результаты пакета
5. Spwig обновляет все выплаты на основе ответа пакета

### Преимущества пакетирования

- **Снижение количества вызовов API** — один запрос для сотен выплат
- **Снижение сборов** — некоторые структуры сборов PayPal предпочтительны для пакетирования
- **Быстрая обработка** — параллельное выполнение всего пакета
- **Один вебхук** — проще мониторинг и логирование

### Ограничения пакета

PayPal накладывает следующие ограничения:
- Максимум 15 000 получателей на пакет
- Максимум $100 000 суммарно на пакет
- Обработка обычно завершается в течение нескольких минут

Если вы превышаете 15 000 выплат, Spwig автоматически разбивает на несколько пакетов.

## Индивидуальная обработка (Airwallex)

Airwallex обрабатывает выплаты по одной, что дает другие компромиссы.

### Как работает индивидуальная обработка

Когда вы обрабатываете выплаты Airwallex:

1. Система отправляет отдельный запрос API для каждой выплаты
2. Airwallex ставит переводы в очередь по отдельности
3. Каждый перевод завершается независимо (2-5 дней)
4. Индивидуальный вебхук срабатывает, когда каждый перевод завершается
5. Spwig обновляет выплаты по мере поступления вебхуков

### Преимущества индивидуальной обработки

- **Лучшая изоляция ошибок** — одна ошибка не блокирует другие
- **Отслеживание по каждой выплате** — уникальные идентификаторы транзакций
- **Более подробные данные по выплатам** — информация по каждому переводу
- **Гибкое время** — переводы завершаются с разной скоростью

### Время обработки

В отличие от мгновенного пакетирования PayPal, переводы Airwallex занимают больше времени:
- Внутренние переводы: 1-2 рабочих дня
- Международные переводы: 3-5 рабочих дней
- Некоторые страны: до 7 рабочих дней

Установите ожидания аффилиатов соответствующим образом в условиях вашей программы.

## Настройка вебхуков

Вебхуки позволяют автоматически обновлять статус выплат, когда провайдеры завершают транзакции.

### Формат URL вебхука

Настройте этот URL в панели управления провайдера:

```
https://yourdomain.com/api/payout-providers/{provider}/webhook/
```

Замените `{provider}` на:
- `paypal` для вебхуков PayPal
- `airwallex` для вебхуков Airwallex

Примеры:
- `https://shop.example.com/api/payout-providers/paypal/webhook/`
- `https://shop.example.com/api/payout-providers/airwallex/webhook/`

### Настройка вебхука PayPal

1. **Перейдите** в [Панель разработчика PayPal](https://developer.paypal.com/dashboard/)
2. **Нажмите** на имя вашего приложения
3. **Прокрутите** до раздела **Webhooks**
4. **Нажмите** **Add Webhook**
5. **Введите URL вебхука** (формат выше)
6. **Выберите события**:
   - `PAYMENT.PAYOUTSBATCH.SUCCESS`
   - `PAYMENT.PAYOUTSBATCH.DENIED`
   - `PAYMENT.PAYOUTS-ITEM.SUCCEEDED`
   - `PAYMENT.PAYOUTS-ITEM.FAILED`
7. **Нажмите Save**

PayPal предоставляет ключ подписи вебхука. Spwig использует его для проверки подлинности вебхука.

### Настройка вебхука Airwallex

1. **Перейдите** в [Панель управления Airwallex](https://www.airwallex.com/app/)
2. **Перейдите** в **Settings > Webhooks**
3. **Нажмите** **Create Webhook**
4. **Введите URL вебхука** (формат выше)
5. **Выберите события**:
   - `transfer.created`
   - `transfer.completed`
   - `transfer.failed`
6. **Нажмите Create**

Airwallex подписывает вебхуки с вашим секретным ключом API.

### Безопасность вебхуков

Вебхуки проверяются с помощью следующих механизмов:

- **Проверка подписи** — провайдер подписывает полезную нагрузку вебхука секретным ключом
- **Проверка временной метки** — отклоняются старые вебхуки (предотвращает повторные атаки)
- **Список разрешенных IP-адресов** (опционально) — ограничение до диапазонов IP провайдера
- **Требуется HTTPS** — вебхуки работают только через SSL

Никогда не отключайте проверку подписи в режиме производства.

### Тестирование вебхуков

Большинство провайдеров предлагают инструменты для тестирования вебхуков:

**PayPal**: Используйте "Simulator" в панели разработчика, чтобы запустить тестовые вебхуки

**Airwallex**: Создайте тестовый перевод в режиме Demo и наблюдайте за вебхуком

Вы также можете проверить журналы вебхуков в Spwig в разделе **Settings > System Logs** (если включено логирование).

## Устранение неполадок

### Ошибка неверных учетных данных

**Симптом**: "Ошибка аутентификации" при сохранении аккаунта провайдера

**Причины**:
- Неверный Client ID или Secret
- Использование учетных данных песочника в режиме производства (или наоборот)
- Истекший или отозванный API-ключ
- Не подтвержденный аккаунт

**Решения**:
- Скопируйте учетные данные заново из панели управления провайдера
- Проверьте, совпадает ли режим (песочник vs производство)
- Сгенерируйте новые API-ключи
- Свяжитесь с поддержкой провайдера, чтобы проверить статус аккаунта

### Вебхук не получен

**Симптом**: Выплата застревает в статусе "Processing" вечно

**Причины**:
- URL вебхука не настроен в панели управления провайдера
- Недействительный сертификат SSL
- Брандмауэр блокирует IP-адреса провайдера
- Проверка подписи вебхука не прошла

**Решения**:
- Проверьте URL вебхука в настройках провайдера
- Убедитесь, что сертификат SSL действителен
- Разрешите диапазоны IP провайдера в брандмауэре
- Проверьте логи Celery на ошибки подписи
- Проверьте вебхук с помощью инструментов симуляции провайдера

### Выплата завершилась неудачно

**Симптом**: Статус выплаты меняется на "Failed" с сообщением об ошибке

**Причины**:
- Неверные данные оплаты аффилиата (неверная почта или банковский аккаунт)
- Недостаточно средств на аккаунте провайдера
- Аккаунт получателя не может принимать платежи
- Страна не поддерживается (Airwallex)
- Выплата превышает лимиты провайдера

**Решения**:
- Проверьте ошибку в поле **Provider Response**
- Убедитесь, что данные оплаты аффилиата корректны
- Добавьте средства на аккаунт провайдера
- Попросите аффилиата проверить статус своего аккаунта
- Проверьте поддержку страны и валюты провайдера
- Разделите крупные выплаты, если они превышают лимиты

### Несоответствие режима

**Симптом**: Тестовые выплаты работают, но выплаты в режиме производства не работают

**Причины**:
- Провайдер установлен в режим песочника, но используются аккаунты аффилиатов в режиме производства
- API-ключи из неправильной среды

**Решения**:
- Переключите режим провайдера на Production
- Сгенерируйте API-ключи в режиме производства
- Убедитесь, что URL вебхука указывает на домен производства

## Рекомендации по безопасности

Защитите интеграцию выплат следующими мерами безопасности:

### Хранение учетных данных

- **Никогда не сохраняйте учетные данные в системе контроля версий** — используйте переменные окружения или безопасное хранение
- **Обновляйте API-ключи ежеквартально** — генерируйте новые ключи каждые 3 месяца
- **Используйте отдельные ключи для песочника и производства** — никогда не смешивайте среды
- **Ограничьте разрешения API** — предоставляйте только доступ к выплатам, а не полный контроль над аккаунтом

Spwig хранит учетные данные провайдера зашифрованными в базе данных. Обеспечьте безопасность резервных копий вашей базы данных.

### Безопасность вебхуков

- **Всегда проверяйте подписи** — никогда не пропускайте проверку подписи
- **Используйте исключительно HTTPS** — вебхуки через HTTP не поддерживаются
- **Реализуйте разрешение IP-адресов** — ограничьте вебхуки диапазонами IP провайдера
- **Записывайте все вебхуки** — мониторьте подозрительную активность
- **Ограничьте частоту запросов к конечным точкам вебхуков** — предотвратите злоупотребление

### Контроль доступа

- **Ограничьте доступ сотрудников** — только доверенные сотрудники должны обрабатывать выплаты
- **Используйте двухфакторную аутентификацию** — требуйте 2FA для аккаунтов сотрудников
- **Проверяйте действия по выплатам** — просматривайте, кто обработал какие выплаты
- **Разделяйте обязанности** — разные сотрудники для утверждения и обработки выплат

### Мониторинг

- **Проверяйте неудачные выплаты ежедневно** — оперативно решайте проблемы
- **Мониторьте балансы аккаунтов провайдера** — убедитесь, что средств достаточно
- **Рассматривайте журналы транзакций еженедельно** — выявляйте аномалии на ранних этапах
- **Настройте оповещения** — электронные письма для крупных или неудачных выплат

## Советы

- Тщательно протестируйте интеграцию в режиме песочника перед переходом в режим производства — поймайте проблемы с фиктивными деньгами.
- Настройте как PayPal, так и Airwallex, чтобы дать аффилиатам выбор метода оплаты — разные аффилиаты предпочитают разные методы.
- Настройте URL вебхуков во время первоначальной настройки и проверьте, что они срабатывают правильно — вебхуки критически важны для автоматизации.
- Держите балансы аккаунтов провайдера на высоком уровне, чтобы избежать неудачных выплат во время пакетной обработки.
- Используйте описательные имена аккаунтов, если настроены несколько провайдеров (например, "PayPal USD", "PayPal EUR").
- Обновляйте API-ключи каждые квартал как часть рекомендаций по безопасности.
- Документируйте URL вебхуков и учетные данные в безопасном менеджере паролей, который делится с вашей командой.
- Немедленно проверяйте неудачные выплаты — задержки раздражают аффилиатов и повреждают репутацию программы.
- Всегда используйте HTTPS для установки Spwig — вебхуки требуют сертификатов SSL.
- Свяжитесь с поддержкой провайдера, если сталкиваетесь с постоянными ошибками — они могут проверить статус вашего аккаунта и разрешения.
