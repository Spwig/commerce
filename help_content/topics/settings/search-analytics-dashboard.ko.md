---
title: 검색 분석 대시보드
---

검색 분석 대시보드는 가게의 모든 검색 쿼리를 추적하여 고객이 무엇을 검색하는지, 어떤 검색이 성공하거나 실패하는지, 그리고 검색 시스템이 얼마나 빠르게 응답하는지를 제공합니다. 이 데이터를 사용하여 인기 있는 제품을 식별하고, 누락된 재고를 발견하며, 동의어를 생성하고, 검색 성능을 최적화할 수 있습니다.

데이터가 표시되기 위해 **검색 설정 > 분석 탭**에서 분석 추적을 활성화해야 합니다.

![분석 대시보드](/static/core/admin/img/help/search-analytics-dashboard/analytics-dashboard.webp)

## 대시보드 개요

**검색 > 검색 분석**으로 이동하여 대시보드에 액세스합니다. 이 페이지는 다음과 같은 내용을 표시합니다:

**통계 카드** - 오늘과 지난 주의 빠른 지표:
- 오늘의 총 검색
- 이번 주의 총 검색
- 결과 없는 쿼리(제품을 반환하지 않는 검색)
- 평균 응답 시간(밀리초 단위)

**상위 쿼리 테이블** - 결과 수와 함께 가장 자주 검색되는 용어

**결과 없는 쿼리** - 결과가 없는 검색(개선을 위한 중요한 요소)

**쿼리 목록** - 필터 기능을 통해 모든 개별 검색 기록을 확인할 수 있습니다.

## 오늘의 통계

**오늘의 총 검색** - 가게의 시간대 기준으로 자정 이후의 모든 검색 요청 수. 자동 완성 및 전체 검색 페이지 요청 모두 포함.

**오늘의 고유 쿼리** - 오늘 사용된 고유한 검색 용어 수. 5명의 고객이 모두 "노트북"을 검색하면, 총 5개의 검색에도 불구하고 고유 쿼리는 1개로 계산됩니다.

**오늘의 결과 없는 검색** - 오늘 결과가 없는 검색. 결과 없는 검색이 많으면 제품이 부족하거나 동의어 커버리지가 부족함을 나타냅니다.

실시간 데이터는 검색이 발생할 때마다 업데이트됩니다.

## 주간 통계

**주간 총 검색** - 지난 7일 동안의 총 검색

**주간 고유 쿼리** - 이번 주에 사용된 고유한 검색 용어

**전주 대비 성장률** - 이전 주 대비 퍼센트 변화(표시되는 경우)

주간 데이터를 사용하여 트렌드를 파악하세요: 검색량 증가가 일반적으로 트래픽 성장 또는 마케팅 캠페인과 상관관계가 있습니다.

## 평균 응답 시간

⚠️ **성능 모니터링**

검색 쿼리를 실행하는 평균 시간(밀리초 단위). 목표 응답 시간:

| 쿼리 유형 | 목표 | 경고 임계값 |
|------------|--------|-------------------|
| 자동 완성 | < 200ms | 300ms 이상 지속적으로 |
| 전체 검색 | < 500ms | 800ms 이상 지속적으로 |

평균 응답 시간이 경고 임계값을 초과하는 경우:
1. **검색 설정 > 캐싱 탭**을 확인하세요 - 캐시 TTL을 증가시킵니다.
2. **심층 인덱싱 탭**을 검토하세요 - 비용이 많이 들는 기능(문서 인덱싱, 대량 카탈로그의 리뷰 인덱싱)을 비활성화합니다.
3. [검색 성능 최적화](/en/admin/help/search-performance-optimization/) 가이드를 참조하세요.

## 상위 쿼리

상위 쿼리 테이블은 가장 자주 검색된 용어를 보여줍니다:

**이 데이터를 다음과 같이 사용하세요**:
- **인기 제품 강조** - "무선 이어폰"이 상위 검색어라면, 해당 제품을 홈페이지에 두드러지게 표시하세요.
- **재고 결정** - 특정 카테고리에 대한 높은 검색량은 수요를 나타냅니다.
- **트렌드 식별** - 계절별 검색은 현재 인기 있는 항목을 드러냅니다.
- **콘텐츠 생성** - 자주 검색되는 주제에 대한 블로그 게시물이나 가이드를 작성하세요.

매달 상위 쿼리를 검토하여 고객의 관심사와 상품 판매를 맞추세요.

## 결과 없는 쿼리

**개선을 위한 핵심 요소** - 결과 없는 쿼리는 가게를 최적화하는 데 있어 금광입니다.

결과 없는 쿼리는 주로 세 가지 이유로 발생합니다:

### 1. 누락된 제품

고객은 판매하지 않는 제품을 검색합니다.

**예**: "요가 매트"에 대한 반복적인 검색이 있지만, 귀하의 가게는 요가 용품이 아닌 피트니스 장비만 판매합니다.

**조치**: 검색이 빈번한 경우, 이러한 제품을 카탈로그에 추가하는 것을 고려하세요.

### 2. 누락된 동의어

고객은 제품 설명과 일치하지 않는 용어를 사용합니다.

**예**: 고객이 "노트북"을 검색하지만, 귀하의 제품은 모두 "노트북 컴퓨터"라고 표기합니다.

**조치**: 고객의 용어를 귀하의 제품 언어에 매핑하는 동의어를 생성하세요. [동의어 및 리디렉션 관리](/en/admin/help/managing-synonyms-redirects/) 가이드를 참조하세요.

### 3. 부족한 퍼지 매칭

오타나 오타가 퍼지 검색이 활성화되어 있어도 일치하지 않습니다.

**예**: "accomodate" 검색은 "accommodate" 제품을 찾지 못합니다.

**조치**:
- **검색 설정 > 퍼지 매칭 탭**에서 유사도 임계값을 낮추세요 (0.80에서 0.75로)
- 자주 발생하는 오타에 대한 단방향 동의어를 추가하세요

**주간 워크플로**:
1. 매주 월요일에 결과 없는 쿼리를 검토하세요
2. 분류: 누락된 제품, 누락된 동의어, 또는 오타
3. 자주 검색되는 용어에 대한 동의어를 추가하세요
4. 재고 계획을 위한 제품 누락을 기록하세요

## 쿼리 세부 정보

리스트에서 쿼리를 클릭하여 전체 세부 정보를 확인할 수 있습니다:

**추적 필드**:
- **쿼리 텍스트** - 고객이 검색한 내용
- **타임스탬프** - 검색이 발생한 시간
- **결과 수** - 반환된 결과 수
- **응답 시간** - 실행 시간(밀리초 단위, 성능 모니터링)
- **사용자** - 로그인한 고객(사용자 추적이 활성화된 경우)
- **세션 ID** - 익명 세션 식별자
- **언어** - 검색 시점의 가게 언어
- **엔진** - 쿼리를 처리한 검색 엔진

## 필터링 및 검색

필터를 사용하여 특정 세그먼트를 분석하세요:

**날짜 계층** - 날짜, 월 또는 연도로 필터링

**언어 필터** - 언어별 검색을 확인(다국어 가게에 유용)

**엔진 필터** - 다양한 엔진 간의 검색 행동을 비교

**결과 없는 토글** - 결과가 없는 쿼리만 표시

**검색 상자** - 특정 쿼리 텍스트를 찾기

## 데이터 내보내기

**내보내기**를 클릭하여 Excel 또는 데이터 도구에서 더 깊은 분석을 위해 쿼리 데이터를 CSV로 다운로드합니다.

**CSV에는 포함됨**:
- 모든 쿼리 텍스트
- 타임스탬프
- 결과 수
- 응답 시간
- 언어 및 엔진 데이터

내보내기를 다음과 같이 사용하세요:
- 시간에 따른 트렌드 분석
- 계절별 검색 패턴 식별
- 성능 감사
- 이해관계자에게 보고

## 개인정보 보호 고려사항

검색 분석 추적은 개인정보 보호를 존중합니다:

**사용자 추적**(선택 사항) - 로그인한 고객 계정에 검색을 연결합니다. GDPR/CCPA 준수를 위해 **검색 설정 > 분석 탭**에서 비활성화하세요.

**세션 추적**(기본값) - 익명 세션 ID를 사용하여 고객을 식별하지 않고 검색 패턴을 추적합니다. 개인정보 보호에 친화적입니다.

**데이터 보존** - 검색 쿼리는 데이터베이스에 영구적으로 남아 있습니다. 필요 시 맞춤형 보존 정책을 구현하세요.

## 분석을 통한 검색 개선

검색 분석에서 얻은 실행 가능한 통찰력:

**주간 작업**:
- 결과 없는 쿼리를 검토하고 자주 검색되는 용어에 대한 동의어를 추가하세요
- 응답 시간을 모니터링하고 지속적으로 느릴 경우 최적화하세요
- 상위 검색을 식별하고 해당 제품이 잘 재고되어 있는지 확인하세요

**월간 작업**:
- 상위 쿼리를 분석하여 제품 선택에 영향을 줍니다
- 데이터를 내보내어 계절별 트렌드를 식별합니다
- 언어별 검색 패턴을 검토합니다
- 리디렉션 클릭 수를 추적하여 네비게이션 단축키를 최적화합니다

**분기별 작업**:
- 동의어 효과를 감사합니다(결과 없는 쿼리가 감소했는가?)
- 검색량 성장률과 전체 트래픽을 비교합니다
- 가중치 변경을 A/B 테스트하고 결과 관련성을 측정합니다
- 검색 수요에 따라 새로운 제품 카테고리를 추가해야 하는지 검토합니다

## 팁

- **결과 없는 쿼리는 개선을 위한 금광입니다** - 고객이 원하는 것이 무엇인지 직접 알려줍니다.
- **월요일 아침에 분석을 검토하세요** - 이전 주의 데이터를 기반으로 주를 시작하여 최적화하세요.
- **응답 시간이 지속적으로 300ms 이상인 경우 조사하세요** - 캐싱 설정을 먼저 확인한 후 심층 인덱싱 기능을 확인하세요.
- **트렌드 분석을 위해 CSV를 내보내세요** - 스프레드시트 분석은 관리 인터페이스에서 명확하지 않은 패턴을 드러냅니다.
- **제품을 추가하기 전에 동의어를 생성하세요** - 고객이 "태블릿 케이스"를 검색하지만 귀하의 제품은 "보호 커버"라고 부르는 경우, 먼저 동의어를 추가하세요.
- **계절별 검색 패턴을 추적하세요** - 10월에 "겨울 신발", 3월에 "수영복" - 재고를 적절히 관리하세요.

기억하세요: 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어는 보존 규칙에 따라 정확히 유지하세요.
