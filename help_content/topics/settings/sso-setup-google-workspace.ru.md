---
title: 'Настройка SSO: Google Workspace'
---

Этот руководство поможет вам подключить Spwig к Google Workspace для однократной авторизации администратора. После настройки сотрудники смогут войти в панель администратора Spwig, используя свой аккаунт Google Workspace.

**Примечание:** Google может обновлять интерфейс Cloud Console со временем. Эти инструкции были написаны на основе интерфейса, существовавшего в начале 2026 года. Если какие-либо шаги отличаются от того, что вы видите, обратитесь к официальной документации Google о [настройке OAuth 2.0](https://support.google.com/cloud/answer/6158849).

## Предварительные условия

- Подписка на Google Workspace (Google Workspace Business, Enterprise или Education)
- Доступ администратора к [Google Cloud Console](https://console.cloud.google.com)
- URL вашего магазина Spwig (например, `https://your-store.com`)
- У сотрудников должны быть электронные адреса в Spwig, совпадающие с их аккаунтами Google Workspace

## Шаг 1: Создайте или выберите проект Google Cloud

1. Перейдите в [Google Cloud Console](https://console.cloud.google.com)
2. Нажмите на селектор проекта в верхней панели
3. Нажмите **New Project** (или выберите существующий проект, если предпочитаете)
4. Введите имя проекта (например, `Spwig SSO`)
5. Выберите вашу организацию
6. Нажмите **Create**

## Шаг 2: Настройка экрана согласия OAuth

1. В Cloud Console перейдите в **APIs & Services > OAuth consent screen**
2. Выберите **Internal** в качестве типа пользователя — это ограничивает вход пользователям внутри вашей организации Google Workspace
3. Нажмите **Create**
4. Заполните необходимые поля:

| Поле | Значение |
|-------|-------|
| **Название приложения** | `Spwig Admin` (или название вашего магазина) |
| **Электронная почта поддержки пользователя** | Ваш адрес электронной почты администратора |
| **Разрешенные домены** | `your-store.com` (домен вашего магазина, без `https://`) |
| **Электронная почта разработчика** | Ваш адрес электронной почты администратора |

5. Нажмите **Save and Continue**
6. На странице **Scopes** нажмите **Add or Remove Scopes** и добавьте:
   - `openid`
   - `email`
   - `profile`
7. Нажмите **Save and Continue**
8. Просмотрите обзор и нажмите **Back to Dashboard**

## Шаг 3: Создание учетных данных OAuth

1. Перейдите в **APIs & Services > Credentials**
2. Нажмите **Create Credentials > OAuth client ID**
3. Настройте клиент:

| Поле | Значение |
|-------|-------|
| **Тип приложения** | Web application |
| **Имя** | `Spwig SSO` |
| **Разрешенные URI перенаправления** | `https://your-store.com/oidc/callback/` |

4. Нажмите **Create**
5. Диалог отображает ваш **Client ID** и **Client Secret** — скопируйте оба значения. Вы также можете скачать их в формате JSON для безопасного хранения.

**Важно:** URI перенаправления должен точно совпадать с `https://your-store.com/oidc/callback/` — включая завершающий слеш и схему `https://`. Замените `your-store.com` на фактический домен вашего магазина.

## Шаг 4: Получение URL Discovery

Google использует один стандартный URL Discovery для всех арендаторов Workspace:

```
https://accounts.google.com/.well-known/openid-configuration
```

Этот URL одинаков для каждой организации Google Workspace — вам не нужно настраивать его с использованием арендатора или домена.

## Шаг 5: Настройка в Spwig

1. В админ-панели Spwig перейдите в **Enterprise SSO > SSO Provider Configuration**
2. Установите **Provider Name** на `Google Workspace`
3. Введите URL Discovery: `https://accounts.google.com/.well-known/openid-configuration`
4. Нажмите **Auto-Discover** — это заполнит все поля конечных точек автоматически
5. Введите **Client ID** из шага 3
6. Введите **Client Secret** из шага 3
7. Нажмите **Save**

### Сопоставление утверждений

Google использует стандартные названия утверждений OIDC, поэтому стандартная настройка Spwig работает без изменений:

| Настройка Spwig | Утверждение Google | Значение по умолчанию |
|---------------|-------------|---------------|
| Утверждение электронной почты | `email` | `email` |
| Утверждение имени | `given_name` | `given_name` |
| Утверждение фамилии | `family_name` | `family_name` |

Изменения в сопоставлении утверждений не требуются.

## Шаг 6: Включение и тестирование

1.

Перейдите в **Site Settings > Security** tab
2.

Проверьте **Enable SSO for admin login**
3.

Нажмите **Save**
4.

Сохраните все форматирование markdown, пути к изображениям, блоки кода и технические термины.

Откройте страницу входа в админ-панель в **частном/инкогнито окне**
5.

Вы должны увидеть кнопку **Войти через Google Workspace**
6.

Нажмите на нее — вас перенаправят на страницу входа Google
7.

Войдите с помощью аккаунта Google Workspace, электронная почта которого совпадает с сотрудником в Spwig
8.

Вы должны быть перенаправлены обратно на панель управления администратором Spwig

## Групповая сопоставление ролей

В отличие от Microsoft Entra ID или Okta, Google по умолчанию не включает членство в группах в стандартных токенах OIDC. Реализация утверждений групп с Google требует использования Google Workspace Directory API и дополнительной настройки, выходящей за рамки базового OIDC.

Для большинства развертываний Google Workspace мы рекомендуем управлять статусом сотрудников и суперпользователей напрямую в Spwig, а не через автоматическое сопоставление ролей:

1. Создайте учетные записи сотрудников в Spwig с соответствующими разрешениями
2. Используйте систему ролей сотрудников Spwig для управления уровнями доступа
3. Сотрудники заходят через SSO, и Spwig использует их существующие разрешения

Если вам требуется автоматическое сопоставление ролей на основе групп, обратитесь к документации [Google Workspace Admin SDK Directory API](https://developers.google.com/admin-sdk/directory) для настройки пользовательских утверждений.

## Распространенные проблемы

| Проблема | Причина | Решение |
|---------|-------|----------|
| **Ошибка 400: redirect_uri_mismatch** | URI перенаправления в Google Cloud не совпадает точно | Проверьте, что URI перенаправления равен `https://your-store.com/oidc/callback/` с завершающим слешем. Проверьте HTTP против HTTPS. |
| **Ошибка 403: access_denied** | Пользователь не является частью организации Google Workspace | С типом пользователя "Внутренний" только пользователи вашей организации могут войти. Убедитесь, что учетная запись пользователя является частью вашей доменной зоны Workspace. |
| **На экране согласия OAuth отображается "Это приложение не проверено"** | Нормально для внутренних приложений | Это предупреждение ожидаемо для внутренних приложений и не влияет на функциональность. Пользователи в вашей организации все равно могут войти. |
| **Успешный вход в Google, но неудачный в Spwig** | Нет совпадающего пользователя в Spwig | Убедитесь, что в Spwig существует учетная запись сотрудника с той же электронной почтой, что и у аккаунта Google Workspace. Проверьте, правильно ли настроено ограничение для сотрудников. |
| **"Доступ заблокирован: запрос этого приложения недействителен"** | Области не настроены должным образом | Убедитесь, что области `openid`, `email` и `profile` добавлены на экран согласия OAuth. |

## Советы

- **Используйте тип пользователя "Внутренний"** — это ограничивает вход в вашу организацию Google Workspace и не требует процесса проверки приложения Google.
- **Секреты клиента Google не истекают** — в отличие от Microsoft Entra ID, секреты клиента OAuth Google не имеют даты истечения срока действия. Однако вы можете вращать их в любое время с помощью страницы "Учетные записи".
- **Один проект для нескольких приложений** — вы можете создать несколько идентификаторов клиентов OAuth в одном проекте Google Cloud, если у вас несколько установок Spwig.
- **Тестирование с неадмин-аккаунтом** — создайте тестовую учетную запись сотрудника в Spwig и используйте обычного пользователя Google Workspace (не суперадминистратора), чтобы проверить, работает ли SSO как ожидается.
