title: CDN 設置

內容分發網絡 (CDN) 會在全球各地的伺服器上儲存您商店的圖片、樣式表和腳本的副本。當客戶訪問您的商店時，這些文件會從距離客戶最近的伺服器提供，而不是從您的主要主機伺服器提供。這可以減少頁面加載時間，特別是對於遠離您商店主機位置的客戶。

Spwig 預設已使用 Brotli 和 gzip 預壓縮、帶有 1 年不可變標頭的指紋化資源緩存，以及正確的內容協商來優化靜態資源的傳輸。添加 CDN 是可選的，但它可以進一步提高國際客戶商店的速度。

## 您的商店是否需要 CDN？

並非每家商店都能從 CDN 中獲得相同的好處。請使用以下指南來決定：

**建議使用 CDN 的情況**：
- 您的客戶分布在多個國家或大洲
- 您的商店包含許多產品圖片或媒體內容豐富的頁面
- 您希望實現全球最快的頁面加載速度
- 您向遠離主機伺服器的地區銷售（例如，伺服器在歐洲，客戶在亞洲）

**可能不需要 CDN 的情況**：
- 您的客戶主要為本地或與伺服器位於同一國家
- 您的商店目錄規模較小，圖片較少
- 您的主機提供商已經內建了 CDN

如有疑問，CDN 不會影響性能。像 Cloudflare 這樣的服務提供免費層，因此可以無成本地嘗試。

## Spwig 如何與 CDN 協作

Spwig 預設即支援 CDN。您不需要在 Spwig 管理面板中更改任何程式碼或設置。以下是 Spwig 已為您做的事情：

- **帶有指紋的靜態文件** -- 每個 CSS、JavaScript 和圖片文件的檔名中都包含唯一的版本哈希。這意味著 CDN 可以安全地緩存這些文件很長時間，而無需提供過時的內容。
- **長效緩存標頭** -- 靜態資源使用 1 年不可變緩存標頭進行傳輸，告訴 CDN 和瀏覽器積極緩存這些資源。
- **預壓縮文件** -- Spwig 使用 Brotli 和 gzip 預壓縮資源，因此您的 CDN 可以在不進行額外處理的情況下傳輸更小的文件。
- **正確的內容協商** -- Spwig 會傳送正確的內容類型和編碼標頭，CDN 依賴這些標頭進行正確的緩存。

您所需要做的就是將您的域名的 DNS 指向 CDN 提供商，一切就會自動運作。

## 設置 Cloudflare

Cloudflare 是最受歡迎的 CDN，它提供一個對大多數商店都適用的免費層。請按照以下步驟操作：

**步驟 1：創建 Cloudflare 帳戶**
- 訪問 cloudflare.com 並註冊一個免費帳戶

**步驟 2：添加您的域名**
- 點擊 **Add a Site** 並輸入您的商店域名
- 選擇 **Free** 計劃（對大多數商店來說已經足夠）

**步驟 3：更新您的 DNS 名稱伺服器**
- Cloudflare 會顯示給您兩個名稱伺服器（例如，`anna.ns.cloudflare.com`）
- 登入您的域名註冊商（您購買域名的地方）
- 將您目前的名稱伺服器替換為 Cloudflare 名稱伺服器
- DNS 更改可能需要最多 24 小時才能生效

**步驟 4：配置 SSL/TLS**
- 在 Cloudflare 仪表板中，前往 **SSL/TLS**
- 將加密模式設置為 **Full (strict)**
- 這確保 Cloudflare 和您的伺服器之間的所有流量都保持加密

**步驟 5：驗證是否已生效**
- 一旦 DNS 传播完成，訪問您的商店並在瀏覽器中檢查 `cf-cache-status` 標頭（請參閱下面的驗證您的 CDN 以了解詳細信息）

## 設置 AWS CloudFront

如果您已經使用 Amazon Web Services，CloudFront 可以自然地與您的基礎設施整合：

1. 在您的 AWS 帳戶中打開 **CloudFront** 控制台
2. 使用您的商店域名作為源創建一個新的 **Distribution**
3. 將 **Origin Protocol Policy** 設置為 "HTTPS Only"
4. 在 **Cache Behavior** 下，將 **Cache Policy** 設置為 "CachingOptimized" 以優化靜態資源的緩存
5. 將您的商店域名添加為 **Alternate Domain Name (CNAME)**
6. 從 AWS Certificate Manager 綁定 SSL 證書
7. 更新您的域名 DNS 以指向 CloudFront 分發 URL

CloudFront 的定價是根據使用量計算的。

對於大多數商店來說，成本都很低，因為 Spwig 的指紋化資源會長時間緩存。

## 推薦的 CDN 設定

為取得最佳效果，請設定您的 CDN 來緩存正確的內容，並跳過其餘內容。

**應緩存的內容**（靜態資源）：
- `/static/` -- 所有樣式表、腳本、字型和主題資源
- `/media/` -- 產品圖片和上傳的媒體文件
- 圖像文件（`.jpg`, `.png`, `.webp`, `.svg`, `.gif`）
- 字型文件（`.woff`, `.woff2`）

**不應緩存的內容**（動態頁面）：
- `/admin/` -- 管理面板必須始終提供新內容
- `/cart/` -- 購物車頁面包含會話特定的數據
- `/checkout/` -- 結帳頁面絕不能緩存以確保安全性
- `/accounts/` -- 顧客帳號頁面包含私人數據
- 任何需要登入或顯示個人化內容的頁面

**一般緩存規則**：
- **尊重來源緩存標頭** -- Spwig 會為每種類型的內容傳送正確的 cache-control 標頭。請設定您的 CDN 來遵循這些標頭，而不是覆蓋它們。
- **啟用 Brotli 壓縮** -- Cloudflare 和 CloudFront 都支援 Brotli。啟用它以利用 Spwig 的預壓縮資源。
- **將瀏覽器緩存 TTL 設為 "尊重現有標頭"** -- 這樣讓 Spwig 內建的緩存策略來驅動行為。

## 驗證您的 CDN

設定完成後，確認 CDN 是否正確提供您的內容：

**步驟 1：開啟瀏覽器開發人員工具**
- 在 Chrome 或 Firefox 中，按下 **F12** 以開啟開發人員工具
- 點擊 **Network**（網路）標籤

**步驟 2：載入您的商店**
- 在開發人員工具開啟的情況下，訪問您的商店首頁
- 點擊任何靜態文件請求（例如 `.css` 或 `.js` 文件）

**步驟 3：檢查回應標頭**
- **Cloudflare**：查找 `cf-cache-status` 標頭。值為 `HIT` 表示文件是從 CDN 緩存中提供的。`MISS` 表示它是從您的伺服器取得（僅首次請求）。
- **CloudFront**：查找 `x-cache` 標頭。值為 `Hit from cloudfront` 表示 CDN 提供了內容。

**步驟 4：從其他位置進行測試**
- 使用類似 gtmetrix.com 或 webpagetest.org 的免費工具，從不同地理位置測試您的商店
- 比較 CDN 設定前後載入時間

## 常見問題

### 更改主題後內容過時

**問題**：在更新主題或進行設計更改後，客戶仍然看到舊版本。

**解決方案**：清除 CDN 緩存。在 Cloudflare 中，前往 **Caching > Configuration > Purge Everything**。在 CloudFront 中，為 `/*` 建立 **Invalidation**。請注意，Spwig 的指紋化資源通常會防止此問題，因為更新後的文件會自動獲得新的檔名。此問題最常影響非指紋化資源，例如自定義上傳文件。

---

### 混合內容警告

**問題**：啟用 CDN 後，瀏覽器顯示有關 "混合內容" 的安全警告。

**解決方案**：確保您的 CDN 的 SSL 模式設定為 **Full (strict)**，而不是 "Flexible"。Flexible 模式可能會導致伺服器收到 HTTP 請求而不是 HTTPS，進而導致混合內容警告。在 Cloudflare 中，檢查 **SSL/TLS > Overview** 並確認模式。

---

### 管理面板運行緩慢

**問題**：添加 CDN 後，管理面板感覺變慢。

**解決方案**：CDN 不應該緩存管理面板頁面。建立一個 **Page Rule**（Cloudflare）或 **Cache Behavior**（CloudFront），將緩存設定為 "Bypass"，以匹配任何 `/admin/*` 的 URL。這樣可以確保管理請求直接到您的伺服器，而不經過 CDN 的負荷。

---

### 圖片無法載入

**問題**：在 CDN 設定後，產品圖片或媒體文件返回錯誤。

**解決方案**：確認您的 CDN 的來源設定為正確的協議（HTTPS）和埠。同時檢查您的伺服器防火牆是否允許來自 CDN IP 範圍的連接。

## 小技巧

保留所有 markdown 格式、圖片路徑、程式碼區塊和技術術語。

- **從 Cloudflare 的免費層開始** -- 它涵蓋了大多數商店的需求，僅需幾分鐘即可設置
- **始終使用完整的（嚴格）SSL 模式** -- 使用彈性模式會產生安全漏洞，並可能導致結帳流程中斷
- **在進行重大主題更新後清除 CDN 缓存** -- 雖然 Spwig 的帶版本號的文件可以處理大多數情況，但完整的缓存清除可確保沒有過時的內容殘留
- **不要缓存結帳或購物車頁面** -- 缓存這些頁面可能會導致一個客戶的資料被另一個客戶看到
- **從您的客戶所在地進行測試** -- 使用 webpagetest.org 等免費工具，從客戶購物的地區測試實際性能
- **監控您的 CDN 分析數據** -- Cloudflare 和 CloudFront 都提供了顯示缓存命中率、節省的頻寬和按國家分類的流量的儀表板
- **在設置期間保持 DNS TTL 值較低** -- 在切換到 CDN 時，將 DNS TTL 設置為 300 秒（5 分鐘），一旦確認一切運作正常，再提高這個值
- **CDN 不會取代良好的主機服務** -- 您的源站服務器對於動態頁面（如結帳、購物車和管理後台）仍然很重要
