title: 運費優惠
---

運費規則根據購物車內容、客戶屬性和配送區域對運費方法應用條件性費用調整—自動提供滿 $50 免運費、為偏遠地區增加附加費用，或為 VIP 顧客折扣運費。規則使用基於優先順序的執行方式（優先順序較高者先執行），並可選擇停止標誌以防止進一步處理。每個規則會評估多個條件（購物車金額、重量、區域、產品、客戶群組），並在所有條件都符合時執行其中一種 6 種調整類型之一。

當您在需要根據訂單上下文動態調整運費，而不僅僅是來自運費方法的靜態費用時，請使用運費優惠。

## 運費優惠類型

運費規則應用 6 種費用調整類型：

### 百分比折扣

**功能說明**：按百分比降低運費（例如：減 25%）。

**公式**：`new_cost = base_cost × (1 - percent/100)`

**範例**：
```
基礎費用：$20
折扣：25%
結果：$15
```

**使用情境**：
- VIP 顧客折扣（所有運費減 20%）
- 季節性促銷（12 月運費減 15%）
- 批量訂購折扣（5 項以上運費減 10%）

---

### 固定折扣

**功能說明**：從運費中扣除固定金額。

**公式**：`new_cost = base_cost - amount`（最低 $0）

**範例**：
```
基礎費用：$15
折扣：$5
結果：$10
```

**使用情境**：
- 首次購物顧客優惠（首筆訂單運費減 $5）
- 訂閱電子報獎勵（運費減 $3）
- 會員忠誠度方案（每月運費減 $10）

---

### 覆蓋費用

**功能說明**：將運費設定為特定金額。

**公式**：`new_cost = fixed_amount`

**範例**：
```
基礎費用：$25
設定為：$9.99
結果：$9.99
```

**使用情境**：
- 限時促銷（今天所有訂單運費 $5）
- 特定類別運費（書籍運費永遠 $3.99）
- 時間基準促銷（本周運費上限 $9.99）

---

### 免運費

**功能說明**：將運費設定為 $0。

**公式**：`new_cost = $0`

```
Base cost: $20
Surcharge: 15%
Result: $23
```

旺季加價（節日期間 20%）

加急配送附加費用（50% 加價）

油價附加費用（根據當前油價變動）

---

### 加價（固定）

**功能**：按百分比增加運費。

**公式**：`new_cost = base_cost × (1 + percent/100)`

## 促銷條件

促銷活動評估 **所有條件都必須符合** 才能套用規則

### 時間有效性

- **開始日期**：規則僅在這日期之後生效

- **結束日期**：規則僅在這日期之前生效

- **使用場景**：季節性促銷、限定時間優惠

**範例**：僅限黑色星期五週末免運費

```
Start: 2026-11-27 00:00
End: 2026-11-30 23:59
```

### 購物車金額範圍

- **最低購物車金額**：購物車小計必須 ≥ 金額

- **最高購物車金額**：購物車小計必須 ≤ 金額

- **使用場景**：免運費門檻、分層折扣

**範例**：購物金額 $50-$200 免運費

```
Min: $50
Max: $200
```

### 購物車重量範圍

- **最低重量**：購物車總重量必須 ≥ 金額

- **最高重量**：購物車總重量必須 ≤ 金額

- **使用場景**：輕量貨件折扣、重物附加費用

**範例**：購物車重量超過 20kg 加 $5

```
Min Weight: 20kg
Max Weight: null (unlimited)
```

### 商品數量範圍

- **最小商品數量**: 購物車必須包含 ≥ 指定數量的商品
- **最大商品數量**: 購物車必須包含 ≤ 指定數量的商品
- **使用場景**: 批量訂購折扣、單件費用

**範例**: 5件以上免運費
```
Min Items: 5
Max Items: null
```

---

### 運送區域

- **區域**: 規則僅適用於客戶地址與至少一個選定區域匹配的情況
- **未選擇**: 規則適用於所有區域
- **使用場景**: 特定區域的附加費用或折扣

**範例**: 僅適用於國內區域的免運費
```
Zones: ["Domestic USA"]
```

---

### 運送方式

- **方式**: 規則僅適用於特定的運送方式
- **未選擇**: 規則適用於所有方式
- **使用場景**: 特定運送方式的促銷

**範例**: Express Shipping 享有 25% 折扣
```
Methods: ["Express Delivery"]
```

---

### 商品要求

**需要商品**: 購物車必須包含至少一個這些商品

**需要分類**: 購物車必須包含至少一個這些分類的商品

**使用場景**: 特定商品的免運費、促銷組合

**範例**: 當購物車包含 "Promotion Item A" 時免運費
```
Requires Products: [Product ID 123]
```

---

### 商品排除

**排除商品**: 如果購物車包含這些商品，則規則不適用

**排除分類**: 如果購物車包含這些分類的商品，則規則不適用

**使用場景**: 排除重型/超大商品免運費

**範例**: 除家具分類外免運費
```
Excludes Categories: [Furniture]
```

---

### 顧客群組

- **顧客群組**: 規則僅適用於選定群組的顧客（VIP、批發等）
- **未選擇**: 規則適用於所有顧客群組
- **使用場景**: VIP 優惠、批發折扣

**範例**: VIP 會員享有 15% 運費折扣
```
Customer Groups: ["VIP"]
```

---

### 首次顧客


- **新客戶**: 切換以限制規則僅適用於沒有先前訂單的客戶
- **使用場景**: 新客戶歡迎優惠

**範例**: 首次訂購減 $5 運費
```
First Time Customer: Yes
```

## 優惠優先順序與執行

優惠會按照 **優先順序** 執行（數字越高表示越早執行）：

### 優先順序機制

**執行範例**：
```
優惠 A（優先順序 100）：購物車金額超過 $50 免運費
優惠 B（優先順序 50）：所有運費 10% 折扣
優惠 C（優先順序 1）：遠程地區加 $2 費用

購物車：$60，遠程地區
基本運費：$15

步驟 1：執行優惠 A（優先順序 100）
  購物車金額 > $50？是
  應用：將費用設為 $0
  當前費用：$0

步驟 2：執行優惠 B（優先順序 50）
  對 $0 進行 10% 折扣
  當前費用：$0（仍然免運費）

步驟 3：執行優惠 C（優先順序 1）
  對 $0 加上 $2 費用
  當前費用：$2

最終費用：$2
```

**停止後續優惠旗標**：

如果優惠 A 設有 `stop_further_promotions = True`：
```
優惠 A（優先順序 100，stop_further_promotions=True）：購物車金額超過 $50 免運費
優惠 B（優先順序 50）：10% 折扣
優惠 C（優先順序 1）：遠程地區加 $2 費用

購物車：$60
基本費用：$15

步驟 1：應用優惠 A，將費用設為 $0
        stop_further_promotions = True → 停止

最終費用：$0（優惠 B 和 C 從未執行）
```

## 建立運費優惠

**逐步操作流程**：

1. **導航至規則**
   - 設定 > 運費 > 運費優惠
   - 點擊 "新增運費優惠"

2. **基本設定**
   - **名稱**：內部識別碼（例如："購物金額超過 $50 免運費"）
   - **描述**：可選備註（不會顯示給客戶）
   - **啟用**：切換以啟用或停用
   - **優先順序**：設定執行順序（100 表示高優先順序，1 表示低）

3. **選擇優惠類型**
   - 選擇調整類型（折扣百分比、固定折扣、設定費用、免費用、附加費用百分比、固定附加費用）
   - 輸入金額或百分比

4. **設定停止旗標**（選項）
   - 如果此規則應阻止較低優先級的促銷執行，請勾選 "停止進一步促銷"
   - 用於最終/絕對規則（例如，免運費應不允許在之後添加附加費用）

5. **定義條件**（選項 - 留空表示 "總是應用"）
   - 時間有效性：開始/結束日期
   - 購物車價值：最小/最大
   - 購物車重量：最小/最大
   - 商品數量：最小/最大
   - 區域：選擇適用區域
   - 方法：選擇適用方法
   - 商品：必需或排除
   - 顧客：群組或僅首次購買者

6. **保存規則**
   - 點擊保存
   - 規則立即生效（如果啟用開關為 Yes）

## 優惠組合策略

### 策略 1：折扣疊加

**允許多個折扣疊加**

```
優惠 A（優先級 100）：VIP 享 10% 折扣 → stop_further_promotions=No
優惠 B（優先級 50）：訂購金額超過 $100 享 15% 折扣 → stop_further_promotions=No

VIP 用戶訂購 $120：
原價：$15
應用優惠 A 後：$13.50（10% 折扣）
應用優惠 B 後：$11.48（15% 折扣 $13.50）
```

### 策略 2：獨佔規則

**僅有一個規則適用**（最高優先級）

```
優惠 A（優先級 100）：購物金額超過 $50 免運費 → stop_further_promotions=Yes
優惠 B（優先級 50）：所有運費享 20% 折扣 → stop_further_promotions=Yes
```

購物車 > $50:
促銷 A 適用 → 免運費 → 停止
促銷 B 永不執行
```

### 策略 3: 條件附加費用

**先折扣，後附加費用**：
```
促銷 A (優先級 100): 購物車 >$75 免運費
促銷 B (優先級 75): 15% VIP 折扣
促銷 C (優先級 50): 10% 一般折扣
促銷 D (優先級 25): $5 遠離地區附加費用
促銷 E (優先級 1): 10% 燃料附加費用

訂單: $80, 遠離地區, VIP 顧客
基準: $20
A: $80 > $75 → 免運費 ($0)
B: VIP → 15% 折扣 $0 = $0
C: 10% 折扣 $0 = $0
D: 遠離地區 +$5 = $5
E: 燃料 +10% 的 $5 = $5.50

最終: $5.50 (因附加費用而未免運費)
```

**為避免此情況，使用 stop_further_promotions=Yes**：
```
促銷 A (優先級 100, stop=Yes): 購物車 >$75 免運費

相同訂單：
A: $80 > $75 → 免運費 ($0) → 停止
最終: $0 (真正免運費)
```

## 測試運費促銷

**上線前**：

1. **建立測試購物車**
   - 購物車 A: $25 (低於門檻)
   - 購物車 B: $55 (高於門檻)
   - 購物車 C: $200 + 遠離地區
   - 購物車 D: VIP 顧客

2. **測試每個規則**
   - 前往結帳
   - 確認顯示正確的運費
   - 檢查規則執行順序

3. **測試優先級解決方案**
   - 多個匹配規則
   - 確認優先級最高的先執行
   - 檢查 stop_further_promotions 的行為

4. **測試邊界情況**
   - 購物車價值正好在門檻
   - 多個條件匹配
   - 冲突規則

## 故障排除

**問題 1: 促銷未套用**

**原因**：
- 規則未啟用
- 一個或多個條件未滿足
- 更高優先級的規則設置了 stop_further_promotions=Yes
- 時間有效性超出當前日期

**解決方案**：檢查所有條件，確認優先級，驗證啟用狀態。

---

**問題 2: 折扣金額異常**

**原因**：
- 多個促銷疊加
- 百分比應用於已折扣的費用
- 規則優先級錯誤

**解決方案**：檢查優先級順序，審查 stop_further_promotions 標誌，手動追蹤執行過程。

---

**問題 3：免運費功能無法正常運作**

**原因**:
- 免運費促銷的優先級較低，導致後續費用調整覆蓋
- 購物車金額未達最低門檻
- 購物車中包含被排除的產品

**解決方案**: 在免運費促銷中設定 stop_further_promotions=Yes，驗證條件，檢查排除項目。

## 小技巧

- **為免運費設定高優先級** - 優先級 100 可確保其在其他調整之前執行
- **為絕對規則設定 stop_further_promotions** - 免運費應停止後續處理
- **測試規則組合** - 多個促銷可能會產生預料之外的互動
- **使用具描述性的名稱** - "VIP 20% 折扣（優先級 75）" 比 "促銷 3" 更好
- **記錄複雜邏輯** - 在描述欄中添加註解
- **從簡單的促銷開始** - 逐步增加複雜度
- **監控規則表現** - 檢查規則是否被使用或導致混淆
- **避免過多促銷** - 過多促銷會拖慢結帳流程，建議使用 5-10 個
- **使用地理區域** - 比為每個國家設定多個類似規則更好
- **與方法結合使用** - 規則 + 方法可共同實現更複雜的定價
- **設定明確的時間範圍** - 促銷時永遠包含結束日期
- **測試邊界情況** - 恰好 $50、恰好 5 項等
