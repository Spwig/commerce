title: 배송 제공자 계정
---

배송 제공자 계정은 FedEx, UPS, DHL과 같은 운송사의 API에 스토어를 연결하여 실시간 요금 계산 및 자동 라벨 구매를 가능하게 합니다. 각 계정은 암호화된 API 자격 증명을 저장하고 연결 상태를 모니터링하며 실시간 배송 방법에 연결됩니다. 제공자들은 결제 시 패키지의 크기, 무게, 출발지 및 목적지를 기반으로 실시간 요금을 가져옵니다. 이는 수동 요금표 유지 관리의 필요성을 제거하고 정확한 운송사 요금을 보장합니다.

운송사 계산된 배송 요금 또는 자동 라벨 생성이 필요한 경우 제공자 계정을 사용합니다. 수동 배송 생성 대신에.

## 지원되는 배송 제공자

Spwig은 설치 가능한 제공자 구성 요소를 통해 주요 운송사를 지원합니다:

### FedEx

**서비스**: Ground, Express, International
**API**: FedEx Web Services
**기능**: 실시간 요금, 라벨 구매, 추적, 국제 관세

### UPS

**서비스**: Ground, Air, Worldwide
**API**: UPS Developer API
**기능**: 실시간 요금, 라벨 생성, 추적, 주소 검증

### DHL

**서비스**: Express, eCommerce, International
**API**: DHL Express API
**기능**: 국제 요금, 관세 문서, 추적

### 추가 제공자

필요에 따라 구성 요소 시장에서 설치 (USPS, Canada Post, Australia Post 등)

## 제공자 계정 구성

각 제공자 계정은 다음과 같은 정보가 필요합니다:

### 기본 정보

- **표시 이름**: 관리자에서 계정이 표시되는 방식 (예: "FedEx Production Account")
- **제공자**: 드롭다운에서 설치된 제공자 구성 요소 선택
- **활성**: 계정을 활성화/비활성화할 수 있으며 자격 증명은 삭제되지 않습니다.
- **기본 계정**: 이 제공자에 대한 기본 계정으로 설정 (제공자당 하나의 기본 계정만 가능)

### API 자격 증명 (암호화됨)

**제공자에 따라 다름**, 일반적으로 포함:

**FedEx**:
- 계정 번호
- 미터 번호
- API 키
- API 비밀

**UPS**:
- 액세스 라이선스 번호
- 사용자 ID
- 비밀번호
- 계정 번호

**DHL**:
- 사이트 ID
- 비밀번호
- 계정 번호

**모든 자격 증명은 저장 시 암호화**되고 API 호출 시에만 복호화됩니다.

### 출발지 주소

- **기본 배송 출발지**: 요금 계산을 위한 창고/출발지 주소
- 일부 제공자는 그들의 대시보드에서 특정 출발지 설정이 필요합니다.

### 설정

제공자별 옵션 (운송사에 따라 다름):

- **테스트 모드**: 운송사의 샌드박스/테스트 API 엔드포인트 사용
- **협상된 요금**: 협상된 운송사 요금 사용 (사용 가능 시)
- **보험 포함**: 요금에 보험 자동 견적 포함
- **주거지 추가 요금**: 주거지 배송 요금 적용
- **서명 필요**: 기본 서명 요구 사항 설정

## 제공자 계정 생성

**6단계 설정 프로세스**:

**단계 1: 운송사 API 액세스 취득**
1. 운송사(예: FedEx.com, UPS.com, DHL.com)와 계정 생성
2. API/개발자 액세스 신청
3. 운송사의 API 온보딩 완료 (1-3 영업일 소요)
4. 이메일 또는 개발자 포털을 통해 API 자격 증명 수령

**단계 2: 제공자 구성 요소 설치** (사전 설치되지 않은 경우)
1. 설정 > 구성 요소 > 시장으로 이동
2. 운송사 이름 검색 (예: "FedEx")
3. 배송 제공자 구성 요소 설치
4. 설치 완료까지 기다림

**단계 3: Spwig에서 제공자 계정 생성**
1. 설정 > 배송 > 제공자 계정으로 이동
2. "제공자 계정 추가" 클릭
3. 드롭다운에서 제공자 선택
4. 표시 이름 입력

**단계 4: API 자격 증명 입력**
1. 자격 증명 필드 입력 (제공자에 따라 다름)
2. 저장 시 자동 암호화
3. 선택 사항: 초기 테스트를 위해 테스트 모드 활성화

**단계 5: 연결 테스트**
1. "연결 테스트" 버튼 클릭
2. 시스템이 운송사에 API 호출 시도
3. "연결됨" 상태 확인
4. last_tested_at 타임스탬프 확인

**단계 6: 배송 방법에 연결**
1. 배송 방법 생성 또는 편집 (설정 > 장바구니 > 배송 방법)
2. method_type = "Real-Time" 설정
3. 드롭다운에서 제공자 계정 선택
4. 방법 저장

## 연결 상태 모니터링

제공자 계정은 연결 상태를 추적합니다:

### 상태 값

**알 수 없음** (회색): 테스트되지 않거나 아직 연결되지 않음

**연결됨** (초록색): 마지막 API 호출 성공, 자격 증명 유효

**에러** (빨강색): 마지막 API 호출 실패, 자격 증명이 유효하지 않을 수 있음

### 마지막 테스트

- **타임스탬프**: 연결이 마지막으로 확인된 시간
- **자동 업데이트**: 제공자가 사용될 때마다 (요금 조회, 라벨 구매)
- **수동 테스트**: 언제든지 "연결 테스트" 버튼 클릭

### 실패한 연결 문제 해결

**일반적인 원인**:
- 잘못된 API 자격 증명 (오타, 추가 공백과 함께 복사)
- 운송사 API 키 만료 또는 취소됨
- 테스트 모드 활성화되었지만 생산 자격 증명을 사용하거나 (반대로도)
- 운송사와의 IP 주소가 허용 목록에 없음
- 운송사 API 다운타임

**해결 단계**:
1. 자격 증명이 운송사 대시보드와 정확히 일치하는지 확인
2. 테스트 모드 설정이 자격 증명 유형과 일치하는지 확인
3. 운송사의 API 상태 페이지에서 중단 사항 확인
4. 운송사 지원에 문의하여 계정 확인

## 요금 조회 워크플로우

체크아웃 시 실시간 요금 작동 방식:

**1. 고객이 주소 입력**
- 배송 주소 입력
- 장바구니에서 총 무게 + 크기 계산

**2. 시스템이 요금 요청 준비**
- 제공자 계정 자격 증명 가져오기 (복호화)
- 장바구니 항목에서 패키지 크기 계산 (배송 패키지가 정의된 경우 사용)
- 출발지, 목적지, 패키지와 함께 API 요청 준비

**3. 제공자 API 호출**
- 인증 자격 증명과 함께 운송사 API에 요청 전송
- 운송사가 구역, 무게, 크기에 기반하여 요금 계산
- 응답에는 서비스 옵션 (Ground, Express 등) 포함

**4. 요금 표시**
- 시스템이 운송사 응답 파싱
- 표준 형식으로 정규화
- 구성된 경우 마크업 적용 (선택 사항)
- 고객에게 체크아웃 시 요금 표시

**5. 고객이 서비스 선택**
- 고객이 선호 옵션 선택
- 선택된 요금 주문에 저장

**예시 API 흐름**:
```
FedEx API에 대한 요청:
{
  "origin": {"postal_code": "90210", "country": "US"},
  "destination": {"postal_code": "10001", "country": "US"},
  "parcels": [{
    "weight": 2500,  // 그램
    "dimensions": {"length": 30, "width": 20, "height": 15}  // cm
  }]
}

FedEx 응답:
[
  {"service": "FEDEX_GROUND", "rate": 12.50, "delivery_days": 5},
  {"service": "FEDEX_EXPRESS", "rate": 28.75, "delivery_days": 2}
]
```

## 라벨 구매 (선택 사항)

제공자가 라벨 생성을 지원하는 경우:

**워크플로우**:
1. 고객이 주문 완료
2. 상인은 배송 생성 (주문 > 주문 세부 정보 > 배송 생성)
3. 제공자 계정 + 서비스 선택
4. 시스템이 제공자의 라벨 API 호출
5. 라벨 PDF 생성 및 배송에 첨부
6. 추적 번호 자동 입력
7. 인쇄 가능한 라벨 준비 완료

**이점**:
- 수동으로 운송사 웹사이트에 로그인하지 않음
- 추적 자동 동기화
- 국제 운송 시 관세 양식 자동 생성
- 배치 라벨 생성 가능

## 요금 마크업

운송사 요금에 상인 마크업 추가:

**설정** (배송 방법에서, 제공자 계정이 아닌):
- **마크업 유형**: 퍼센트 또는 고정
- **마크업 금액**: 예: 15% 또는 $2.50

**예시**:
```
운송사 요금: $12.50
마크업: 15%
고객이 지불하는 금액: $14.38

OR

운송사 요금: $12.50
마크업: $2.50 (고정)
고객이 지불하는 금액: $15.00
```

**사용 사례**:
- 포장/처리 비용 충당
- 배송에 이익 마진 추가
- 신용 카드 수수료 상쇄

## 여러 제공자 계정

동일한 제공자에 대해 여러 계정을 생성할 수 있습니다:

**사용 사례**:
1. **테스트 vs 프로덕션**
   - 테스트 계정: 운송사 샌드박스 자격 증명
   - 프로덕션 계정: 실제 자격 증명

2. **다중 창고**
   - 창고 A 계정: 출발지 = 로스앤젤레스
   - 창고 B 계정: 출발지 = 뉴욕

3. **다른 협상된 요금**
   - 계정 A: 표준 요금
   - 계정 B: 볼륨 할인 요금

**각 계정은 다른 배송 방법에 연결될 수 있어** 유연한 구성이 가능합니다.

## 팁

- **샌드박스에서 테스트** - 실제 운영 전에 운송사 테스트 자격 증명 사용
- **연결 상태 모니터링** - 대시보드에서 정기적으로 오류 상태 확인
- **배송 패키지 정의** - 정확한 크기로 요금 견적 향상
- **협상된 요금 사용** - 운송사와 볼륨 할인이 있는 경우 활성화
- **현실적인 출발지 설정** - 정확한 구역을 위해 실제 배송 출발지 주소 사용
- **자격 증명 보안 유지** - API 키 공유 금지, 주기적으로 교체
- **대체 방법 보유** - 운송사 API 실패 시 평균 요금 방법 유지
- **운송사 API 제한 모니터링** - 일부 운송사는 하루 API 호출 제한
- **자격 증명 즉시 업데이트** - 운송사가 키를 교체할 때 즉시 업데이트
- **설명적인 이름 사용** - "FedEx LA Warehouse"가 "FedEx 1"보다 더 좋음

기억하세요: 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어는 보존 규칙에 따라 정확히 유지합니다.
