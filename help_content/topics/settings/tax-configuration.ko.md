---
title: 세금 설정
---

세율은 고객의 위치와 제품 유형에 따라 체크아웃 시 부과되는 판매세, 부가세 및 기타 소비세를 정의합니다. 국가/주/도시 수준의 세율을 설정할 수 있으며, 제품 카테고리의 예외 적용 여부를 선택할 수 있습니다. Spwig은 복합세(세금에 대한 세금), 우선순위 기반 세율 선택 및 지역별 세금 시스템(EU 부가세, 미국 판매세)을 위한 세율 사전 설정 그룹을 지원합니다. 특정 제품 유형(음식, 책, 디지털 상품) 또는 카테고리에 대해 세율을 예외로 지정하여 지역별 세법 규정을 준수할 수 있습니다.

세금 설정을 사용하여 판매 지역의 세금 징수 요구사항에 대한 법적 준수를 보장하세요.

## 세율 설정

각 세율은 다음과 같이 정의합니다:

**지리적 범위**:
- 국가 (필수)
- 주/도 (선택)
- 도시 (선택)
- 우편 번호 패턴 (선택, 정규식)

**세율 세부 정보**:
- **세율**: 백분율 (예: 8.5%)
- **이름**: 표시 이름 (예: "캘리포니아 판매세")
- **우선순위**: 여러 세율이 일치할 경우 높은 우선순위가 적용됨
- **활성화**: 삭제 없이 토글 가능

**예외**:
- **예외 제품 유형**: 디지털 상품, 물리적 상품, 서비스
- **예외 카테고리**: 특정 제품 카테고리 (음식, 책, 의료)

**복합세**:
- **복합세 여부**: 이전 세금에 이 세율을 추가 적용 (세금에 대한 세금)
- 예시: 퀘벡 PST는 GST에 복합 적용됩니다.

## 일반적인 세금 시나리오

### 미국 주 수준 판매세

```
이름: 캘리포니아 판매세
국가: 미국
주: CA
세율: 7.25%
우선순위: 50
```

### 유럽 연합 부가세 (국가 수준)

```
이름: 영국 부가세
국가: GB
세율: 20%
우선순위: 50

이름: 독일 부가세
국가: DE
세율: 19%
우선순위: 50
```

### 캐나다 GST/PST (복합)

```
세율 1: 연방 GST
국가: CA
세율: 5%
우선순위: 100
복합세 여부: 아니오

세율 2: 퀘벡 PST
국가: CA
주: QC
세율: 9.975%
우선순위: 50
복합세 여부: 예 (소계 + GST에 적용)
```

### 도시 수준 세금

```
이름: 시애틀 판매세
국가: 미국
주: WA
도시: 시애틀
세율: 10.1%
우선순위: 100
```

## 세금 예외

### 제품 유형 예외

전체 제품 유형을 예외로 지정합니다:

- **디지털 상품**: 소프트웨어, 전자책, 음악
- **물리적 상품**: 유형 제품
- **서비스**: 컨설팅, 설치

예시: EU 부가세는 소비자에게 디지털 상품에 적용되지 않습니다 (일부 경우)

### 카테고리 예외

특정 제품 카테고리를 예외로 지정합니다:

- 식품 및 식료품 (종종 예외 또는 감면율 적용)
- 책 및 교육 자료
- 의약품 및 의약품
- 의류 (일부 지역)

설정:
```
이름: 캘리포니아 판매세
세율: 7.25%
예외 카테고리: ["식품 및 음료", "처방약"]
```

## 세금 사전 설정 그룹

일반적인 세금 설정을 빠르게 불러옵니다:

**미국 판매세 사전 설정**:
- 모든 50개 주 + DC
- 주 수준 세율
- 세율이 변경되면 자동 업데이트

**유럽 연합 부가세 사전 설정**:
- 모든 27개 EU 회원국
- 표준 부가세율
- B2B를 위한 역전 계산 논리

**사전 설정 사용 방법**:
1. 설정 > 장바구니 > 세금 사전 설정
2. 사전 설정 그룹 선택 (예: "2026년 미국 판매세")
3. "사전 설정 불러오기" 클릭
4. 세율이 자동으로 불러옵니다.
5. 필요에 따라 사용자 정의

## 우선순위 해결

여러 세율이 일치할 경우, 가장 높은 우선순위가 적용됩니다:

예시:
```
시애틀, WA에 있는 고객:

세율 A: 미국 연방 (우선순위 1) - 0%
세율 B: 워싱턴 주 (우선순위 50) - 6.5%
세율 C: 시애틀 도시 (우선순위 100) - 3.6%

결과: 시애틀 세율 (총 10.1%) 적용
```

## 세금 표시 옵션

설정 > 장바구니 > 세금 설정에서 구성:

- **세금 포함 가격**: 세금이 포함된 가격 표시 (유럽 스타일)
- **세금 별도 표시**: 세금을 항목으로 표시 (미국 스타일)
- **세금 반올림**: 항목별 또는 주문별
- **세금 제목**: 라벨을 사용자 정의 ("부가세", "판매세", "GST")

---

## 세금 설정 테스트

라이브 환경에 적용하기 전에:

1. 다른 지역에서 테스트 주문 생성
2. 올바른 세금율이 적용되었는지 확인
3. 제외된 카테고리에 대한 예외가 작동하는지 확인
4. 복합세 계산 테스트
5. 세금 항목이 영수증에 표시되는지 확인

---

## 준수 참고 사항

- **미국**: Nexus 규칙은 물리적 존재 또는 경제적 Nexus가 있는 주에서의 세금 징수를 요구합니다.
- **유럽**: 부가세 등록 기업은 EU 고객에게 부가세를 징수해야 합니다.
- **캐나다**: GST/HST/PST는 주에 따라 다릅니다.
- **세무 전문가 상담**: 세금 법규가 자주 변경되므로 현재 요구사항을 확인하세요.

## 팁

- **세금 사전 설정 사용** - 수동 입력보다 빠르고 자동 업데이트
- **Nexus 임계값 모니터링** - 미국 경제적 Nexus를 위해 주별 판매 추적
- **우선순위 올바르게 설정** - 도시 > 주 > 국가
- **복합세 테스트** - 계산이 예상 금액과 일치하는지 확인
- **연간 업데이트** - 세금율이 변경되므로 매년 1월에 검토
- **예외 기록** - 카테고리가 예외로 지정된 이유를 기록해 두세요
- **설명적인 이름 사용** - "2026년 캘리포니아 판매세"가 "세금 1"보다 더 좋음
- **기본적으로 세금 활성화** - 세금을 적용하는 것을 잊지 않도록 안전한 방법
