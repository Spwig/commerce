title: 영수증 템플릿 커스터마이징

영수증 템플릿은 POS 단말기에서 인쇄되는 열전사 영수증의 외형과 내용을 제어합니다. 헤더 및 푸터 텍스트를 커스터마이징하고 로고를 추가하며, 규제 필드(세금 번호, 사업 등록 번호)를 구성하고 프로모션 QR 코드를 포함할 수 있습니다. 템플릿은 범위 대상 설정을 지원합니다. 모든 매장에 적용되는 기본 템플릿, 지역별 그룹 템플릿, 개별 위치에 적용되는 매장별 템플릿을 생성할 수 있습니다. 시스템은 범위 우선순위 규칙을 사용하여 영수증을 인쇄할 때 어떤 템플릿이 적용되는지 결정합니다.

영수증 템플릿을 사용하여 브랜드 일관성을 유지하고 지역 규제 요구 사항을 충족하며 프로모션 요소를 통해 고객 참여를 향상시킬 수 있습니다.

![영수증 템플릿 목록](/static/core/admin/img/help/receipt-template-customization/receipt-list.webp)

## 영수증 템플릿 기초

영수증 템플릿은 ESC/POS 열전사 프린터에서 인쇄되는 영수증의 구조와 내용을 정의합니다. 각 템플릿은 다음을 지정합니다:

**물리적 구성**:
- 종이 너비(58mm 또는 80mm)
- 로고 이미지(열 인쇄용 흑백)
- 글꼴 크기 및 간격

**내용 섹션**:
- 헤더 텍스트(매장 이름, 주소, 연락처 정보)
- 동적 거래 데이터(상품, 가격, 합계, 결제 방식)
- 푸터 텍스트(환불 정책, 감사 메시지, 소셜 미디어)
- 규제 필드(세금 번호, 사업 등록 번호)
- 라벨이 있는 프로모션 QR 코드

**범위 대상 설정**:
- 기본 템플릿(모든 매장에 적용되며, 덮어쓰지 않는 한)
- 그룹 템플릿(그룹 내 모든 매장에 적용)
- 매장 템플릿(특정 매장/창고에 적용)

## 범위 우선순위 규칙

단말기가 영수증을 인쇄할 때 시스템은 다음과 같은 계층 구조를 사용하여 템플릿을 선택합니다(가장 높은 우선순위에서 가장 낮은 우선순위):

| 우선순위 | 범위 | 예시 | 사용 사례 |
|----------|-------|---------|----------|
| **1** | 매장별 | 파리 매장 템플릿 | 독특한 프랑스 세금 규제 요구 사항 |
| **2** | 그룹별 | 유럽 매장 템플릿 | 모든 유럽 지역에서의 부가가치세 표시 |
| **3** | 기본 | 글로벌 템플릿 | 설정되지 않은 모든 매장의 대체용 |

**작동 방식**:
1. 매장에 전용 템플릿(창고별)이 있는지 확인합니다.
2. 없으면 매장의 그룹에 그룹 템플릿이 있는지 확인합니다.
3. 없으면 기본 템플릿을 사용합니다.

**예시**:
- 기본 템플릿: "표준 영수증" (범위 할당 없음)
- 그룹 템플릿: "유럽 영수증" (유럽 매장 그룹에 할당됨) - 부가가치세 등록 포함
- 매장 템플릿: "파리 영수증" (파리 창고에 할당됨) - 프랑스 SIRET 번호 포함

**결과**:
- 파리 매장 단말기: "파리 영수증" 사용(가장 구체적)
- 베를린 매장 단말기(유럽 매장 그룹에 속함, 매장 템플릿 없음): "유럽 영수증" 사용(그룹 수준)
- 뉴욕 매장 단말기(그룹 없음, 매장 템플릿 없음): "표준 영수증" 사용(기본 대체)

## 종이 너비 구성

열전사 영수증 프린터는 58mm 또는 80mm의 종이 너비를 사용합니다. 프린터 하드웨어에 따라 선택합니다:

| 종이 너비 | 줄당 문자 수 | 최적 사용 | 일반적인 사용 |
|-------------|---------------------|----------|-------------|
| **58mm** | ~32 문자 | 작은 공간, 이동 가능 | 음식 트럭, 이동형 POS, 키오스크 |
| **80mm** | ~48 문자 | 표준 소매 | 대부분의 소매점, 레스토랑 |

**너비 혼합 불가**: 동일한 템플릿을 사용하는 모든 단말기는 동일한 종이 너비 프린터를 사용해야 합니다. 혼합된 프린터 유형이 있는 경우 각 너비에 대해 별도의 템플릿을 생성합니다.

**로고 크기 제한**:
- **58mm**: 최대 너비 384 픽셀(권장: 350px)
- **80mm**: 최대 너비 576 픽셀(권장: 550px)

최대 너비를 초과한 로고는 자동으로 축소되며, 이는 품질 저하를 초래할 수 있습니다.

## 로고 구성

영수증 로고는 열 프린터 호환성을 위해 **흑백** (검은색과 흰색만)이어야 합니다:

**로고 요구 사항**:
- 파일 형식: PNG, JPG 또는 WebP
- 색상 모드: 흑백(흰색 배경에 검은색 픽셀)
- 권장 크기:
  - 58mm 종이: 350px 너비 × 100-150px 높이
  - 80mm 종이: 550px 너비 × 150-200px 높이
- 파일 크기: <100KB (열 프린터는 메모리가 제한되어 있음)

**흑백 로고 생성**:
1. 일반적인 로고(컬러 또는 그레이스케일)로 시작합니다.
2. 이미지 편집기로 순수 검은색과 흰색으로 변환합니다(그레이스케일 없음)
3. 대비를 높여 검은색 요소가 단단하게 되도록 합니다.
4. 투명하거나 흰색 배경을 가진 PNG로 내보냅니다.

**로고 위치**:
- 항상 수평으로 중앙 정렬
- 영수증 상단에 인쇄(헤더 텍스트 위)
- 자동 간격이 따르며(내용과의 충돌 방지)

**로고 선택**:
- 템플릿 폼에서 **미디어 라이브러리 탐색**을 클릭합니다.
- 흑백 로고 자산을 선택합니다.
- 미리보기에서 로고가 영수증에 어떻게 표시되는지 확인합니다.

**로고 없음**: 로고 필드를 비워 두고 텍스트만으로 브랜딩을 원하는 경우(헤더 텍스트에 매장 이름을 포함할 수 있음).

## 헤더 텍스트

로고 바로 다음에 헤더 텍스트가 표시됩니다(로고가 없는 경우 상단에 표시됨). 일반적인 내용:

**매장 이름 및 주소**:
```
Your Store Name
123 Main Street
City, State 12345
Phone: (555) 123-4567
```

**영업 시간**:
```
월~금: 오전 9시~오후 9시
토~일: 오전 10시~오후 6시
```

**슬로건 또는 슬로건**:
```
품질 있는 제품, 뛰어난 서비스
```

**포맷팅**:
- 정보를 분리하기 위해 줄 바꿈을 사용합니다.
- 자동으로 중앙 정렬됩니다.
- 줄 수를 종이 너비의 문자 수 제한 이하로 유지합니다(58mm는 32자, 80mm는 48자).

**사용 가능한 변수**(선택 사항):
- `{store_name}` - 창고 이름으로 대체됩니다.
- `{order_date}` - 거래 날짜로 대체됩니다.
- `{order_number}` - 주문 ID로 대체됩니다.

대부분의 상인은 헤더 일관성을 위해 정적 텍스트를 사용합니다.

## 푸터 텍스트

거래 세부 정보(상품, 합계, 결제) 후에 푸터 텍스트가 표시됩니다. 일반적인 내용:

**환불 정책**:
```
영수증과 함께 30일 이내 환불
매장 신용 또는 교환만 가능
```

**감사 메시지**:
```
우리와 쇼핑해 주셔서 감사합니다!
@yourstore를 팔로우해 주세요
```

**고객 서비스**:
```
질문이 있으십니까? 전화 (555) 123-4567
또는 이메일 support@yourstore.com
```

**포맷팅 팁**:
- 가장 중요한 정보(환불 정책, 연락처)를 먼저 배치합니다.
- 가독성을 위해 줄 바꿈을 사용합니다.
- 섹션 사이에 구분선(`---`)을 추가하는 것을 고려합니다.

## 규제 필드

많은 지역에서는 영수증에 특정 정보가 필요합니다:

**세금 번호 라벨** - 세금 식별 번호를 위한 사용자 정의 라벨:
- 미국: "Tax ID" 또는 "EIN"
- 유럽: "VAT Number" 또는 "VAT Reg No"
- 캐나다: "GST/HST Number"
- 호주: "ABN"

**세금 번호 값** - 실제 식별 번호:
- 템플릿에서 한 번 입력하면 모든 영수증에 표시됩니다.
- 예: "VAT Number: GB123456789"

**사업 등록 라벨** - 사업 등록을 위한 사용자 정의 라벨:
- 프랑스: "SIRET"
- 독일: "Handelsregister"
- 영국: "Company Registration Number"

**사업 등록 값** - 실제 등록 번호:
- 예: "SIRET: 123 456 789 00010"

**Spwig Powered By 표시** - "Powered by Spwig" 브랜딩을 표시하거나 숨기는 전환 스위치:
- 기본적으로 활성화되어 있음(플랫폼 개발 지원)
- 화이트 라벨 운영을 위해 비활성화 가능

**지역별 규제 예시**:

**유럽 연합**:
- 세금 번호 라벨: "VAT Number"
- 세금 번호 값: "GB123456789"
- 국가별 요구 사항에 따라 회사 등록 번호 표시

**미국**:
- 일반적으로 영수증 세금 번호 요구 사항 없음(주별로 다름)
- B2B 거래에 EIN 포함 가능

**프랑스(특정)**:
- 모든 영수증에 SIRET 필수
- 사업 등록 라벨: "SIRET"
- 사업 등록 값: "123 456 789 00010"

**호주**:
- GST 등록 사업체에 ABN(호주 사업 번호) 권장
- 세금 번호 라벨: "ABN"

실제 운영 전에 지역의 영수증 요구 사항을 확인하세요.

## QR 코드 프로모션

고객 참여를 유도하기 위해 영수증 하단에 QR 코드를 포함합니다:

**QR 코드 URL** - 스캔 시 목적지:
- 리뷰 요청: `https://yourstore.com/reviews/leave-review`
- 충성도 프로그램: `https://yourstore.com/loyalty/join`
- 다음 구매 할인: `https://yourstore.com/discount/THANKYOU`
- 소셜 미디어: `https://instagram.com/yourstore`
- 웹사이트 메인 페이지: `https://yourstore.com`

**QR 코드 라벨** - QR 코드 위에 표시되는 텍스트:
- "리뷰를 남기고 다음 구매 시 10% 할인 받기"
- "충성도 프로그램에 가입하세요 - 여기를 스캔하세요"
- "인스타그램을 팔로우하세요 - 연결하려면 스캔하세요"
- "경험 평가"

**QR 코드 최선의 실천**:
- 짧은 URL 사용(긴 URL은 밀도가 높고 스캔하기 어려운 코드 생성)
- 배포 전 여러 스마트폰 카메라로 QR 코드 테스트
- 라벨에 명확한 가치 제안 포함(스캔으로 고객이 무엇을 얻는지)
- QR 코드 스캔을 추적하여 효과 측정(추적 파라미터가 포함된 URL 사용)

**동적 QR 코드**(고급):
- QR 리디렉션 서비스(bit.ly, tinyurl)를 사용하여 짧은 URL 생성
- 리디렉션을 계절별로 다른 목적지로 설정하여 영수증 재인쇄 없이도 가능
- 예: `https://bit.ly/yourstoreqr` → 현재 프로모션으로 리디렉션

## 다른 범위를 위한 템플릿 생성

**기본 템플릿**(추천 시작점):
1. **POS > 영수증 템플릿**으로 이동합니다.
2. **+ 영수증 템플릿 추가**를 클릭합니다.
3. **창고** 및 **매장 그룹** 필드를 비워 두세요(이렇게 하면 기본 템플릿이 됩니다).
4. 대부분의 프린터 유형과 일치하는 종이 너비를 구성합니다.
5. 로고, 헤더, 푸터를 추가합니다.
6. 주요 시장에 대한 규제 필드를 구성합니다.
7. 저장합니다.

이 템플릿은 모든 매장에 적용되며 덮어쓰지 않는 한입니다.

**그룹 템플릿**(지역 차이를 위한):
1. 새 템플릿을 생성합니다.
2. **매장 그룹**을 선택합니다(예: "유럽 매장")
3. **창고**를 비워 두세요.
4. 지역에 맞게 규제 필드를 조정합니다(예: 부가가치세 형식)
5. 헤더 텍스트를 조정합니다(예: 지역 주소)
6. 저장합니다.

이 템플릿은 그룹 내 모든 매장에 적용됩니다.

**매장 템플릿**(위치별 요구 사항을 위한):
1. 새 템플릿을 생성합니다.
2. **창고**를 선택합니다(예: "파리 매장")
3. 이 특정 위치에 대한 모든 필드를 조정합니다.
4. 저장합니다.

이 템플릿은 이 하나의 매장에만 적용됩니다.

**템플릿 테스트**:
- 단말기에서 테스트 거래를 처리합니다.
- 영수증을 인쇄합니다.
- 로고의 명확성, 텍스트 정렬, 규제 필드, QR 코드의 스캔 가능성 등을 확인합니다.
- 필요한 경우 템플릿을 수정하고 다시 테스트합니다.

## 일반적인 영수증 레이아웃

**최소 영수증**(음식 트럭, 팝업):
- 로고 없음(공간 절약)
- 헤더: 매장 이름 및 전화 번호만
- 푸터: 감사 메시지
- QR 코드 없음

**표준 소매 영수증**:
- 로고(흑백 브랜드 마크)
- 헤더: 전체 매장 이름, 주소, 영업 시간
- 규제: 세금 번호
- 푸터: 환불 정책, 감사 메시지
- QR 코드: 리뷰 요청

**프리미엄 소매 영수증**:
- 로고(전체 브랜드 워드마크)
- 헤더: 슬로건, 주소, 연락처
- 규제: 세금 번호, 사업 등록
- 푸터: 환불 정책, 고객 서비스, 소셜 미디어
- QR 코드: 충성도 프로그램 가입

**다중 위치 체인**:
- 기본 템플릿: 기업 브랜딩, 표준 정책

- 매장 템플릿: 위치별 주소 및 전화 번호

## 여러 템플릿 관리

**템플릿 이름 규칙**:
- 범위를 이름에 포함: "기본 영수증", "유럽 그룹 영수증", "파리 매장 영수증"
- 목록을 검토할 때 어떤 템플릿이 어디에 적용되는지 쉽게 식별할 수 있도록 합니다.

**템플릿 변경**:
- 변경 사항은 미래 영수증에 즉시 적용됩니다.
- 과거에 인쇄된 영수증은 영향을 받지 않습니다.
- 모든 매장에 배포하기 전에 낮은 트래픽 단말기에서 변경 사항을 테스트합니다.

**템플릿 복제**:
- 기존 템플릿과 유사한 새 템플릿을 생성할 때 기존 템플릿을 복제하고 수정합니다.
- 처음부터 시작하는 것을 방지합니다.

**템플릿 삭제**:
- 단말기가 존재하는 동안 기본 템플릿을 삭제할 수 없습니다(대체 템플릿이 있어야 함)
- 그룹/매장 템플릿은 삭제할 수 있습니다(단말기는 계층 구조의 다음 수준으로 되돌아갑니다)
- 템플릿을 삭제하기 전에 해당 템플릿을 사용하는 단말기가 없는지 확인합니다.

- **실제 프린터에서 로고 테스트** - 화면에서 잘 보이는 것이 프린터에서 잘 인쇄되지 않을 수 있음; 조기에 테스트
- **규제 필드를 최신 상태로 유지하세요** - 영수증에 만료된 세금 등록은 법적 문제를 일으킬 수 있음
- **가치 제안이 있는 QR 코드는 더 잘 스캔됩니다** - "10% 할인을 받으려면 스캔하세요"는 "여기를 스캔하세요"보다 10배 더 효과적
- **문자 수 제한을 확인하세요** - 텍스트 줄이 끊어지면 포맷이 망가짐; 배포 전 줄당 문자 수를 확인하세요
- **종이 너비별로 하나의 템플릿만 사용하세요** - 58mm 프린터를 사용하는 단말기에 80mm 템플릿을 할당하지 마세요(로고가 맞지 않음)
- **매달 테스트 영수증을 인쇄하세요** - 프린터는 시간이 지남에 따라 악화됨; 품질이 여전히 적절한지 확인하세요
