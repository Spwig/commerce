---
title: مزوّجي أجهزة نقاط البيع
---

مزوّجي أجهزة نقاط البيع يمكّنون من قبول بطاقات الائتمان والخصم في أجهزة نقاط البيع الخاصة بك. Stripe Terminal هو المزوّج الرئيسي المدعوم، ويقدم قارئات بطاقات حديثة (S700، WisePOS E، P400)، ونسبة معالجة تنافسية، وتكامل سلس. اضبط حسابات المزوّجين مع بيانات الاعتماد الخاصة بـ API، وراقب حالة الاتصال في الوقت الفعلي، وإدارة مزوّجين متعددين إذا كنت تعمل في مناطق مختلفة. نظام المزوّجين قابل للتوسيع - يمكن دمج معالجي دفع إضافيين عبر إطار عمل المزوّج إذا لم يكن Stripe Terminal يعمل في سوقك.

استخدم مزوّجي الدفع لقبول الدفعات بالبطاقات بشكل آمن، ومتابعة حالة معالجة الدفع، وإدارة تعيينات القارئ عبر أجهزة نقاط البيع.

![قائمة مزوّجي الدفع](/static/core/admin/img/help/payment-terminal-providers/provider-list.webp)

## لمحة عامة عن مزوّجي الدفع

مزوّجي الدفع هي خدمات ثالثة تتعامل مع دفعات البطاقات نيابة عن عملك:

**مهام المزوّج**:
- تفويض معاملات البطاقات في الوقت الفعلي
- التواصل مع قارئات البطاقات المادية
- التعامل مع أمن الدفع (الامتثال لـ PCI، التشفير)
- تحويل الأموال إلى حسابك المصرفي (الاستقرار)
- تقديم تقارير المعاملات وإدارة النزاعات

**دور Spwig**:
- توجيه طلبات الدفع إلى المزوّج المُكوّن
- تخزين بيانات الاعتماد المشفرة للمزوّج
- مراقبة حالة الاتصال
- ربط القارئات بأجهزة نقاط البيع
- تسجيل نتائج الدفع في الطلبات

## Stripe Terminal (المزوّج الرئيسي)

Stripe Terminal هو مزوّج الدفع الموصى به لمعظم التجار:

**المزايا**:
- قارئات بطاقات شريحة EMV الحديثة
- دعم الدفع بدون لمس (NFC) (Apple Pay، Google Pay، بطاقات الدفع باللمس)
- إدارة النزاعات المتكاملة
- التفويض في الوقت الفعلي
- واجهة برمجة تطبيقات سهلة الاستخدام
- متاح في أكثر من 40 دولة

**الأسعار** (اعتبارًا من عام 2024، يرجى التحقق من الأسعار الحالية):
- رسوم المعاملات: 2.7% + 0.05 دولار أمريكي لكل معاملة حضورية (الولايات المتحدة)
- لا توجد رسوم شهرية، ولا رسوم إعداد، ولا رسوم امتثال لـ PCI
- أجهزة قارئات البطاقات: شراء مرة واحدة (59 دولارًا أمريكيًا إلى 299 دولارًا أمريكيًا حسب النموذج)

**المناطق المدعومة**:
- الولايات المتحدة، كندا، المملكة المتحدة، الاتحاد الأوروبي، أستراليا، سنغافورة، وغيرها
- تحقق من توفر Stripe: https://stripe.com/terminal

**القارئات المدعومة**:
- BBPOS WisePOS E (جهاز Android متعدد الاستخدامات)
- Stripe Reader S700 (قارئ مكتبية)
- Verifone P400 (قارئ قديم، لا يزال مدعومًا)

## إعداد Stripe Terminal

**الخطوة 1: إنشاء حساب Stripe**
- سجّل الدخول على stripe.com
- أكمل التحقق من العمل (حساب بنكي، رقم هوية ضريبية)
- تفعيل الدفعات

**الخطوة 2: تمكين Stripe Terminal**
- في لوحة تحكم Stripe، انتقل إلى **المنتجات > الطرف النهائي**
- اضغط على **ابدأ**
- قبول شروط الخدمة الخاصة بالطرف النهائي

**الخطوة 3: إنشاء الموقع**
- يتطلب Stripe Terminal "الموقع" الذي يمثل موقع بيعك المادي
- انتقل إلى **الطرف النهائي > المواقع**
- اضغط على **إنشاء موقع**
- أدخل عنوان المتجر والتفاصيل
- احفظ معرف الموقع (يبدو مثل `tml_1ABC123...`)

**الخطوة 4: إنشاء مفتاح API**
- انتقل إلى **المطورين > مفاتيح API**
- ابحث عن **المفتاح السري** (يبدأ بـ `sk_live_...` للإنتاج، `sk_test_...` للاختبار)
- نسخ المفتاح السري (لا تشاركه علنًا)

**الخطوة 5: إعداد في Spwig**
- انتقل إلى **نقاط البيع > مزوّجي الدفع**
- اضغط على **+ إضافة مزوّج الدفع**
- اختر **المزوّج**: "Stripe Terminal"
- أدخل **مفتاح API السري** (من الخطوة 4)
- أدخل **معرف الموقع** (من الخطوة 3)
- احفظ

**الخطوة 6: اختبار الاتصال**
- بعد الحفظ، يجب أن تتغير حالة المزوّج إلى "مُتصل" (أخضر)
- إذا ظهرت حالة "خطأ" (أحمر)، تحقق من مفتاح API ومعرف الموقع
- تحقق من رسالة الخطأ في عرض تفاصيل المزوّج

## الحقول الخاصة بإعداد المزوّج

**مفتاح المزوّج** - اختر معالج الدفع:
- **stripe_terminal** - Stripe Terminal (الموصى به)
- **manual** - إدخال دفع يدوي (للاختبار فقط، لا معالجة فعليّة)
- قد تظهر مزوّجين إضافيين إذا تم تثبيتها عبر نظام المكونات

**الاعتماد (مشفر)** - هيكل JSON يحتوي على بيانات الاعتماد الخاصة بـ API:
- يتم تشفيره تلقائيًا قبل التخزين
- لا يظهر أبدًا في النص العادي بعد الحفظ
- مثال على الهيكل (Stripe Terminal):
```json
{
  "api_key": "sk_live_ABC123...",
  "location_id": "tml_1ABC123..."
}
```

**إعدادات المزوّج** - إعدادات إضافية (تختلف حسب المزوّج):
- وصف البيان (يظهر على وثيقة بطاقة الائتمان الخاصة بالعميل)
- التقاط تلقائي (التقاط الدفعات المُفوضة فورًا مقابل التقاط يدوي)
- تجاوز العملة (إذا كانت حساب المزوّج يستخدم عملة مختلفة عن المتجر)

**حالة الاتصال** - مؤشر حالة في الوقت الفعلي:
- **مُتصل** (أخضر) - المزوّج قابل للوصول وتم إعداده بشكل صحيح
- **خطأ** (أحمر) - فشل الاتصال أو بيانات الاعتماد غير صحيحة
- **غير معروف** (رمادي) - لم يتم اختباره بعد (فورًا بعد الإنشاء)

**المرة الأخيرة التي تم اختبارها** - توقيت آخر اختبار للاتصال:
- يتم تحديثه تلقائيًا عند معالجة المعاملات
- قم بتشغيل الاختبار يدويًا عبر إجراء **اختبار الاتصال** في أداة الإدارية

## مراقبة حالة الاتصال

يقوم النظام بمراقبة اتصال المزوّج لإعلامك بالمشاكل قبل محاولة العملاء الدفع:

**الاختبار التلقائي**:
- كل معاملة دفع تُحفّز اختبار الاتصال (بضرورة)
- يقوم وظيفة خلفية بفحص الاتصال كل 6 ساعات (مراقبة وقائية)

**معاني الحالة**:

**مُتصل** - واجهة برمجة تطبيقات المزوّج قابلة للوصول، بيانات الاعتماد صحيحة، جاهزة لمعالجة الدفعات

**خطأ** - أسباب شائعة:
- مفتاح API غير صحيح (تم إلغاؤه، انتهت صلاحيته، أو غير صحيح)
- معرف الموقع غير صحيح (تم حذف الموقع في Stripe، تم إدخال معرف خاطئ)
- مشاكل في الاتصال الشبكي (حظر جدار الحماية لواجهة برمجة تطبيقات Stripe)
- خدمة Stripe متوقفة (نادر)

**غير معروف** - لم يتم اختبار المزوّج بعد (حساب جديد ينتظر أول معاملة)

**حل حالة الخطأ**:
1. تحقق من رسالة الخطأ في عرض تفاصيل المزوّج (تشرح المشكلة المحددة)
2. تأكد من أن مفتاح API لا يزال صحيحًا في لوحة تحكم Stripe
3. تأكد من أن معرف الموقع لا يزال موجودًا في لوحة تحكم Stripe
4. قم بتشغيل اختبار الاتصال يدويًا عبر إجراء **اختبار الاتصال** في أداة الإدارية
5. قم بتحديث بيانات الاعتماد إذا لزم الأمر

## مقارنة أجهزة قارئات البطاقات المدعومة

يقدم Stripe Terminal خيارات متعددة لجهاز قارئ البطاقات:

| النموذج | النوع | طرق الدفع | الشاشة | الأفضل لـ | السعر |
|---------|-------|------------|--------|-----------|-------|
| **WisePOS E** | متعدد الاستخدامات | شريحة EMV، NFC، تمرير | شاشة لمس بحجم 5 بوصات | نقاط بيع تجارية متكاملة | ~299 دولار أمريكي |
| **S700** | مكتبية | شريحة EMV، NFC، تمرير | شاشة لمس أحادية اللون | نقاط بيع تجارية عادية | ~249 دولار أمريكي |
| **P400** | مكتبية | شريحة EMV، NFC، تمرير | شاشة لمس أحادية اللون | الانتشارات القديمة | ~299 دولار أمريكي |

**مزايا WisePOS E**:
- بنظام Android (يعمل التطبيقات، يمكن عرض المحتوى المخصص)
- شاشة لمس بцвет (تجربة مستخدم أفضل لطلبات الديون، التقاط التوقيع)
- طابعة فواتير مدمجة (اختيارية)
- أسرع سرعة معاملة

**مزايا S700**:
- تكلفة أقل من WisePOS E
- حجم صغير
- تصميم مقاوم للماء

**P400** (نموذج أقدم):
- لا يزال مدعومًا ولكن لا يُنصح به للاستخدام الجديد
- معالجة البطاقات ببطء أكثر من S700/WisePOS E

تتصل جميع القارئات بـ Spwig POS عبر واجهة برمجة تطبيقات Stripe Terminal (لا حاجة لاتصال مباشر عبر USB/Bluetooth مع جهاز نقاط البيع).

## اعتبارات الأمان

**تشفير بيانات الاعتماد**:
- يتم تشفير جميع بيانات الاعتماد للمزوّج في قاعدة البيانات
- يستخدم التشفير مفتاحًا سريًا للتطبيق (المحدد في إعدادات التطبيق)
- لا تظهر بيانات الاعتماد أبدًا في السجلات أو رسائل الخطأ

**أذونات مفاتيح API**:
- استخدم مفاتيح API **المحدودة** في الإنتاج (حد الأذونات فقط إلى الطرف النهائي)
- لا تستخدم مفاتيح سرية غير محدودة (الوصول الأوسع من اللازم = خطر أمني)
- في لوحة تحكم Stripe، أنشئ مفتاحًا محدودًا فقط بـ **الطرف النهائي** الأذونات

**امتثال PCI**:
- Stripe Terminal يتعامل مع امتثال PCI (بيانات البطاقات لا تصل إلى خوادم Spwig)
- يتم معالجة أرقام البطاقات بالكامل على معدات القارئ → خوادم Stripe → شبكات البطاقات
- يخزن Spwig فقط نتائج الدفع (الموافقة/الرفض)، لا تفاصيل البطاقات أبدًا

**تغيير المفاتيح**:
- تغيير مفاتيح API سنويًا كممارسات أمنية
- عند التغيير، قم بتحديث بيانات الاعتماد في إعدادات المزوّج
- يمكن إلغاء المفاتيح القديمة في لوحة تحكم Stripe بعد التأكد من أن المفتاح الجديد يعمل

## مزوّجين متعددين

بعض التجار يحتاجون إلى حسابات مزوّجين متعددة:

**العمليات متعددة العملات**:
- المتاجر الأمريكية تستخدم حساب Stripe الأمريكي (تتعامل مع USD)
- المتاجر الأوروبية تستخدم حساب Stripe الأوروبي (تتعامل مع EUR)
- اضبط مزوّجًا منفصلًا لكل عملة

**المزوّجين الاحتياطية**:
- المزوّج الرئيسي (Stripe Terminal)
- مزوّج احتياطي (الإدخال اليدوي) عند عطل القارئ
- يختار الموظف المزوّج عند بدء الدفع

**الاختبار مقابل الإنتاج**:
- استخدم مزوّجًا لاختبار بـ `sk_test_...` مفاتيح API
- استخدم مزوّجًا للإنتاج بـ `sk_live_...` مفاتيح API
- قم بتغيير المزوّجين بعد مرحلة الاختبار

## إصلاح المشكلات الشائعة

**المشكلة 1: تظهر الحالة "خطأ" مع رسالة "مفتاح API غير صحيح"**
- **السبب**: تم إلغاء مفتاح API أو تم نسخه بشكل خاطئ
- **الحل**: أنشئ مفتاح API جديد في لوحة تحكم Stripe، قم بتحديث بيانات الاعتماد للمزوّج، اختبر الاتصال

**المشكلة 2: لم يتم اكتشاف القارئ أثناء الدفع**
- **السبب**: القارئ لم يتم تسجيله في موقع المزوّج
- **الحل**: في لوحة تحكم Stripe، تأكد من أن القارئ مسجّل في نفس معرف الموقع المستخدم في إعدادات المزوّج

**المشكلة 3: تم رفض الدفع رغم وجود بطاقة صالحة**
- **السبب**: لم يتم تفعيل حساب Stripe بالكامل (التحقق معلق)
- **الحل**: أكمل التحقق من العمل في لوحة تحكم Stripe (حساب بنكي، رقم هوية ضريبية)

**المشكلة 4: حالة الاتصال تظهر "غير معروف" ولا تتحديث أبدًا**
- **السبب**: لم يتم اختبار المزوّج أبدًا (لم تتم المحاولة أي معاملات)
- **الحل**: استخدم إجراء **اختبار الاتصال** في أداة الإدارية لتشغيل اختبار الاتصال يدويًا

## نصائح

- **الاختبار قبل الإنتاج** - استخدم مفاتيح API لاختبار Stripe (`sk_test_...`) للاعداد الأولي واختبار
- **مزوّج واحد لكل عملة** - لا تحاول معالجة EUR باستخدام حساب Stripe الأمريكي؛ أنشئ مزوّجين منفصلين
- **راقب حالة الاتصال أسبوعيًا** - المراقبة المبكرة تمنع فشل الدفع عند نقطة البيع
- **حد من أذونات مفاتيح API** - قم بتقييد مفاتيح Stripe API فقط إلى أذونات الطرف النهائي (مبدأ أقل صلاحية)
- **وثّق معرفات المواقع** - احتفظ بسجل لمعرف Stripe الموقع الذي يناظر أي متجر مادي
- **اختبار تعيين القارئ** - بعد إعداد المزوّج، قم بإجراء دفع باستخدام قارئ البطاقات الفعلي لتأكيد تدفق كامل
- **احتفظ بتحديث معلومات الاتصال مع Stripe** - تأكد من أن معلومات الاتصال الخاصة بالعمل في Stripe تتطابق مع الحالية (مهمة للنزاعات، الامتثال)
