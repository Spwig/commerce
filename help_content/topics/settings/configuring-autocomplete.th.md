---
title: การตั้งค่าการเสริมคำค้นหาอัตโนมัติ
---

การเสริมคำค้นหาอัตโนมัติ หรือเรียกอีกอย่างว่า การค้นหาแบบคาดการณ์ หรือ การค้นหาขณะพิมพ์ แสดงผลลัพธ์ขณะลูกค้าพิมพ์คำค้นหาของพวกเขา ฟีเจอร์นี้ช่วยปรับปรุงประสบการณ์ผู้ใช้อย่างมีนัยสำคัญโดยช่วยให้ลูกค้าค้นหาสินค้าได้เร็วขึ้น และลดการค้นหาที่ไม่พบผลลัพธ์ คู่มือนี้อธิบายวิธีการตั้งค่าพฤติกรรมการเสริมคำค้นหา ตัวเลือกการแสดงผล และผลต่อประสิทธิภาพ

การเสริมคำค้นหาเปิดใช้งานโดยค่าเริ่มต้นพร้อมการตั้งค่าที่เหมาะสม ปรับแต่งเฉพาะนี้ได้ก็ต่อเมื่อคุณมีความกังวลเรื่องประสิทธิภาพหรือตัวเลือกการแสดงผลที่เฉพาะเจาะจง

![การตั้งค่าการเสริมคำค้นหา](/static/core/admin/img/help/configuring-autocomplete/autocomplete-settings-main.webp)

## เปิดใช้งานการเสริมคำค้นหา

ไปที่ **ค้นหา > การตั้งค่าการค้นหา** และคลิกแท็บ **การเสริมคำค้นหา**

**เปิดใช้งานการเสริมคำค้นหา** - ปุ่มสลับหลักสำหรับการค้นหาแบบคาดการณ์ เมื่อเปิดใช้งาน ช่องค้นหาจะแสดงผลลัพธ์แบบเลื่อนลงขณะลูกค้าพิมพ์

**จำนวนผลลัพธ์สูงสุดต่อประเภท** - ค่าเริ่มต้น: 8 รายการ จำนวนผลลัพธ์ที่จะแสดงสำหรับแต่ละประเภทเนื้อหา (สินค้า หมวดหมู่ แบรนด์ โพสต์บล็อก) ค่าที่ต่ำกว่า (5-6) ลดขนาดข้อมูล API และแสดงผลได้เร็วขึ้น ค่าที่สูงกว่า (10-12) ให้ลูกค้ามีตัวเลือกมากขึ้น แต่ทำให้การตอบสนองช้าลง

## การตั้งค่าการล่าช้า

⚠️ **คำเตือนเรื่องประสิทธิภาพ** - การตั้งค่าการล่าช้ามีผลอย่างมากต่อภาระของเซิร์ฟเวอร์

**ระยะเวลาการล่าช้า** - ค่าเริ่มต้น: 300ms ระยะเวลาที่ต้องรอหลังจากปุ่มพิมพ์สุดท้ายก่อนที่จะเรียกใช้การเสริมคำค้นหา

การตั้งค่านี้ช่วยสมดุลระหว่างความเร็วในการตอบสนองกับภาระของเซิร์ฟเวอร์:

| เวลา | ประสบการณ์ผู้ใช้ | ผลกระทบต่อเซิร์ฟเวอร์ |
|-------|----------------|---------------|
| **100ms** | ตอบสนองได้ดีมาก | มีการเรียกใช้ API มากกว่า 3 เท่าเมื่อเทียบกับ 300ms - ภาระสูง |
| **200ms** | ตอบสนองได้ดี | มีการเรียกใช้ API มากกว่า 1.5 เท่าเมื่อเทียบกับ 300ms |
| **300ms** | สมดุลที่ดี (แนะนำ) | ค่าพื้นฐาน |
| **400ms** | ตอบสนองได้ช้าเล็กน้อย | มีการเรียกใช้ API น้อยลง - ภาระต่ำ |
| **500ms** | ล่าช้าอย่างเห็นได้ชัด | มีการเรียกใช้ API น้อยลง 50% แต่รู้สึกช้า |

**คำแนะนำ**: รักษาไว้ระหว่าง 250-350ms เพิ่มขึ้นเกิน 350ms ก็ต่อเมื่อเซิร์ฟเวอร์ของคุณมีปัญหากับภาระการเสริมคำค้นหา อย่าตั้งค่าต่ำกว่า 200ms เว้นแต่คุณมีเซิร์ฟเวอร์ที่เร็วมากและแคตตาล็อกที่เล็ก

## การตั้งค่าการแสดงผลสำหรับสินค้า

ปุ่มสลับเหล่านี้ควบคุมข้อมูลใดที่แสดงในผลลัพธ์การเสริมคำค้นหาสำหรับสินค้า:

**แสดงรูปภาพย่อ** - ค่าเริ่มต้น: เปิดใช้งาน แสดงรูปภาพสินค้าข้างๆ ผลลัพธ์ **ผลกระทบต่อประสิทธิภาพ**: เพิ่มการสอบถามรูปภาพและเพิ่มขนาดข้อมูล JSON ปิดใช้งานเพื่อให้การเสริมคำค้นหาเร็วขึ้นเมื่อการเชื่อมต่อช้า

**แสดงคำอธิบาย** - ค่าเริ่มต้น: ปิดใช้งาน แสดงคำอธิบายสั้นๆ ของสินค้า **ผลกระทบต่อประสิทธิภาพ**: เพิ่มการประมวลผลข้อความและเพิ่มขนาดข้อมูลอย่างมาก ควรปิดใช้งานเว้นแต่คำอธิบายจะจำเป็นต่อการเลือกสินค้า

**แสดงราคา** - ค่าเริ่มต้น: เปิดใช้งาน แสดงราคาสินค้า **ผลกระทบต่อประสิทธิภาพ**: ต่ำ - ข้อมูลราคาถูกโหลดมาพร้อมกับสินค้า ปลอดภัยที่จะเปิดใช้งานไว้

**แสดง SKU** - ค่าเริ่มต้น: เปิดใช้งาน แสดง SKU ของสินค้า **ผลกระทบต่อประสิทธิภาพ**: ต่ำ - SKU ถูกจัดทำดัชนีไว้แล้ว จำเป็นสำหรับร้านค้า B2B

**แสดงสถานะสต็อก** - ค่าเริ่มต้น: ปิดใช้งาน ⚠️ **คำเตือนเรื่องประสิทธิภาพอย่างมาก**

แสดงป้ายกำกับ "มีสินค้า", "สินค้าใกล้หมด", หรือ "สินค้าหมด" **อย่าเปิดใช้งานนี้สำหรับแคตตาล็อกขนาดใหญ่**

สถานะสต็อกต้องการการรวม `with_stock_totals()` - การคำนวณจำนวนสต็อกที่มีอยู่ในทุกคลังสินค้าสำหรับสินค้าแต่ละชิ้นในผลลัพธ์การเสริมคำค้นหา นี่เพิ่ม:
- ภาระฐานข้อมูลอย่างมาก (การสอบถามการรวม)
- เวลาเพิ่ม 200-500ms สำหรับแคตตาล็อกที่มีมากกว่า 1,000 สินค้า
- ความล่าช้าที่อาจเกิดขึ้นสำหรับแคตตาล็อกที่มีมากกว่า 10,000 สินค้า

เปิดใช้งานก็ต่อเมื่อจำเป็นอย่างยิ่งและคุณมีสินค้าน้อยกว่า 500 ชิ้น

## การตั้งค่าการแสดงผลสำหรับโพสต์บล็อก

**แสดงรูปภาพเด่น** - ค่าเริ่มต้น: เปิดใช้งาน รูปภาพย่อของโพสต์บล็อกในผลลัพธ์การเสริมคำค้นหา

**แสดงข้อความย่อ** - ค่าเริ่มต้น: เปิดใช้งาน ข้อความย่อจากเนื้อหาของโพสต์

**ความยาวข้อความย่อ** - ค่าเริ่มต้น: 60 ตัวอักษร จำนวนข้อความย่อที่แสดง

การตั้งค่านี้มีผลกระทบต่อประสิทธิภาพน้อยมาก เนื่องจากโพสต์บล็อกมักมีจำนวนน้อยเมื่อเทียบกับสินค้า

## การตั้งค่าการแสดงผลสำหรับหมวดหมู่และแบรนด์

**แสดงรูปภาพ/โลโก้** - ค่าเริ่มต้น: เปิดใช้งาน รูปภาพหรือโลโก้ของหมวดหมู่หรือแบรนด์ในผลลัพธ์

**แสดงจำนวนสินค้า** - ค่าเริ่มต้น: ปิดใช้งาน ⚠️ **คำเตือนเรื่องประสิทธิภาพ**

แสดงจำนวนสินค้าในแต่ละหมวดหมู่หรือแบรนด์ (เช่น "อิเล็กทรอนิกส์ (234)")

**อย่าเปิดใช้งานนี้สำหรับแคตตาล็อกขนาดใหญ่** การนับจำนวนสินค้าจะถูกคำนวณใหม่ในแต่ละคำขอการเสริมคำค้นหา:
- แต่ละประเภทเนื้อหาที่เปิดใช้งานการนับจะเพิ่มคำขอ 2 ครั้งเพิ่มเติม
- คำขอรวมการเชื่อมต่อและการรวมข้อมูล
- เวลาเพิ่ม 100-300ms ทั่วไป
- เพิ่มขึ้นตามจำนวนหมวดหมู่/แบรนด์

เปิดใช้งานก็ต่อเมื่อคุณมีน้อยกว่า 50 หมวดหมู่/แบรนด์ และมีสินค้ารวมน้อยกว่า 1,000 ชิ้น

## การแคช

**ระยะเวลาการหมดอายุแคชการเสริมคำค้นหา** - ค่าเริ่มต้น: 60 วินาที (ตั้งค่าในแท็บการแคช)

ผลลัพธ์การเสริมคำค้นหาถูกแคชเพื่อปรับปรุงประสิทธิภาพ ระยะเวลาการหมดอายุ 60 วินาทีหมายถึง:
- ลูกค้าคนแรกที่ค้นหา "laptop" จะกระตุ้นการสอบถามฐานข้อมูล
- 59 วินาทีถัดไป การค้นหา "laptop" ทั้งหมดจะแสดงผลลัพธ์ที่แคชไว้
- หลังจาก 60 วินาที แคชจะหมดอายุ และการค้นหาครั้งต่อไปจะอัปเดตข้อมูล

**คำแนะนำสำหรับระยะเวลาการหมดอายุ**:
- **45-60 วินาที**: สมดุลที่ดีสำหรับร้านค้าส่วนใหญ่ (ค่าเริ่มต้น)
- **90-120 วินาที**: เพิ่มประสิทธิภาพมากขึ้นหากสินค้าในสต็อกเปลี่ยนแปลงน้อย
- **30 วินาที**: ผลลัพธ์ที่ทันสมัยมากขึ้นหากคุณเพิ่มสินค้าบ่อยๆ

การเพิ่มระยะเวลาการหมดอายุของแคชเป็นวิธีที่ง่ายที่สุดในการปรับปรุงประสิทธิภาพของการเสริมคำค้นหา

## การเสริมคำค้นหาหลายภาษา

หากคุณมีการตั้งค่าหลายภาษา การเสริมคำค้นหาจะค้นหาเนื้อหาที่แปลแล้วที่เก็บไว้ใน JSONField ของการแปล

**วิธีการทำงาน**:
- ลูกค้าค้นหาในภาษาสเปน: "zapatos"
- ระบบค้นหาชื่อสินค้าที่แปลเป็นภาษาสเปน
- ผลลัพธ์แสดงชื่อสินค้าที่แปลเป็นภาษาสเปนจากข้อมูล JSONField
- หากไม่มีการแปลภาษาสเปน ระบบจะกลับไปใช้ภาษาพื้นฐาน

**ประสิทธิภาพ**: มีการเพิ่มภาระเล็กน้อยสำหรับ 1-3 ภาษา แต่เมื่อมี 5 ภาษาขึ้นไป จะมีความซับซ้อนในการสอบถามเพิ่มขึ้นเล็กน้อย

## การทดสอบการเสริมคำค้นหา

หลังจากตั้งค่าเสร็จแล้ว ทดสอบประสบการณ์การเสริมคำค้นหา:

1. **เปิดหน้าแรกของร้านค้า** ของคุณในหน้าต่าง Incognito
2. **คลิกช่องค้นหา** เพื่อโฟกัส
3. **พิมพ์ชื่อสินค้าที่พบบ่อย** ช้าๆ (เช่น "laptop")
4. **สังเกต**:
   - ความเร็วที่ผลลัพธ์ปรากฏหลังจากคุณหยุดพิมพ์ (การล่าช้าทำงานหรือไม่?)
   - ข้อมูลใดที่แสดง (รูปภาพย่อ ราคา SKU ตามที่ตั้งไว้)
   - ผลลัพธ์มีความเกี่ยวข้องหรือไม่ (ตรวจสอบน้ำหนักความเกี่ยวข้องหากไม่เกี่ยวข้อง)
5. **ทดสอบบนมือถือ** - ตรวจสอบว่าเมนูเลื่อนลงสามารถใช้งานได้ด้วยการสัมผัสและอ่านได้ชัดเจน

## เคล็ดลับ

- **ปิดใช้งานคำอธิบายสินค้าเพื่อความเร็ว** - คำอธิบายเพิ่มขนาดข้อมูลอย่างมากโดยมีค่าใช้จ่ายน้อยในบริบทของการเสริมคำค้นหา
- **อย่าเปิดใช้งานสถานะสต็อกสำหรับแคตตาล็อกขนาดใหญ่** - การรวมข้อมูลสต็อกทำลายประสิทธิภาพของการเสริมคำค้นหา
- **ทดสอบบนมือถือด้วยเป้าหมายการสัมผัส** - ผลลัพธ์การเสริมคำค้นหาต้องสามารถแตะได้ง่ายบนโทรศัพท์
- **ตรวจสอบเวลาตอบสนองรายสัปดาห์** - ตั้งเป้าหมายให้การขอใช้เวลาต่ำกว่า 200ms
- **เพิ่มระยะเวลาการหมดอายุของแคชหากช้า** - เป็นการปรับปรุงประสิทธิภาพที่ง่ายที่สุด
- **จำนวนสินค้าเป็นเรื่องที่ใช้ทรัพยากรสูง - ปิดใช้งานเว้นแต่จำเป็น** - จำนวนหมวดหมู่/แบรนด์แต่ละรายการเพิ่มคำขอ 2 ครั้งในแต่ละคำขอการเสริมคำค้นหา
