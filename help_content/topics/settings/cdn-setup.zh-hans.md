内容分发网络 (CDN) 会在全球各地的服务器上存储您商店的图片、样式表和脚本的副本。当客户访问您的商店时，这些文件将从离他们最近的服务器提供，而不是从您的主托管服务器提供。这可以减少页面加载时间，特别是对于远离您商店托管位置的客户。

Spwig 默认已通过 Brotli 和 gzip 预压缩、带 1 年不可变头的指纹资产缓存以及适当的内容协商来优化静态资源的交付。添加 CDN 是可选的，但对于拥有国际客户群的商店，可以进一步提高速度。

## 您的商店是否需要 CDN？

并非每家商店都能从 CDN 中获得同等的好处。请使用以下指南进行判断：

**建议使用 CDN 的情况**：
- 您的客户分布在多个国家或大洲
- 您的商店包含大量产品图片或媒体密集型页面
- 您希望实现全球最快的页面加载速度
- 您向远离托管服务器的地区销售（例如，服务器在欧洲，客户在亚洲）

**可能不需要 CDN 的情况**：
- 您的客户主要本地或与服务器位于同一国家
- 您的商店产品目录较小，图片较少
- 您的托管提供商已包含内置的 CDN

如有疑问，使用 CDN 不会影响性能。像 Cloudflare 这样的服务提供免费层级，因此尝试使用不会产生任何成本。

## Spwig 如何与 CDN 配合工作

Spwig 默认已准备好与 CDN 配合使用。您无需在 Spwig 管理面板中更改任何代码或设置。以下是 Spwig 为您已经完成的工作：

- **带指纹的静态文件** -- 每个 CSS、JavaScript 和图像文件的文件名中都包含一个唯一的版本哈希。这意味着 CDN 可以安全地长时间缓存这些文件，而不会提供过时的内容。
- **长期缓存头** -- 静态资源使用 1 年不可变缓存头进行分发，告诉 CDN 和浏览器积极缓存这些资源。
- **预压缩文件** -- Spwig 使用 Brotli 和 gzip 预压缩资源，因此您的 CDN 可以在不进行额外处理的情况下交付更小的文件。
- **适当的内容协商** -- Spwig 发送正确的 content-type 和编码头，这是 CDN 正确缓存所依赖的。

您需要做的只是将您的域名的 DNS 指向 CDN 提供商，一切都会自动工作。

## 设置 Cloudflare

Cloudflare 是最受欢迎的 CDN，其免费层级对大多数商店都适用。请按照以下步骤操作：

**步骤 1：创建 Cloudflare 账户**
- 访问 cloudflare.com 并注册一个免费账户

**步骤 2：添加您的域名**
- 点击 **Add a Site** 并输入您的商店域名
- 选择 **Free** 计划（对大多数商店已足够）

**步骤 3：更新您的 DNS 名称服务器**
- Cloudflare 会显示两个名称服务器（例如，`anna.ns.cloudflare.com`）
- 登录到您的域名注册商（您购买域名的地方）
- 将当前的名称服务器替换为 Cloudflare 的名称服务器
- DNS 更改可能需要最多 24 小时才能生效

**步骤 4：配置 SSL/TLS**
- 在 Cloudflare 仪表板中，进入 **SSL/TLS**
- 将加密模式设置为 **Full (strict)**
- 这确保了 Cloudflare 与您的服务器之间的所有流量保持加密

**步骤 5：验证是否已生效**
- 一旦 DNS 传播完成，请访问您的商店，并在浏览器中检查 `cf-cache-status` 头（请参阅下文的验证您的 CDN）

## 设置 AWS CloudFront

如果您已经在使用 Amazon Web Services，CloudFront 可以自然地与您的基础设施集成：

1. 在您的 AWS 账户中打开 **CloudFront** 控制台
2. 使用您的商店域名作为源创建一个新的 **Distribution**
3. 将 **Origin Protocol Policy** 设置为 "HTTPS Only"
4. 在 **Cache Behavior** 下，将 **Cache Policy** 设置为 "CachingOptimized" 以用于静态资源
5. 将您的商店域名添加为 **Alternate Domain Name (CNAME)**
6. 从 AWS Certificate Manager 附加 SSL 证书
7. 更新您的域名的 DNS 以指向 CloudFront 分发 URL

CloudFront 的定价基于使用情况。

对于大多数商店来说，成本都很低，因为 Spwig 的指纹资产会被缓存很长时间。

## 推荐的 CDN 设置

为了获得最佳效果，请配置您的 CDN 以缓存适当的内容并跳过其他内容。

**需要缓存的内容**（静态资源）：
- `/static/` -- 所有样式表、脚本、字体和主题资源
- `/media/` -- 产品图片和上传的媒体文件
- 图像文件（`.jpg`，`.png`，`.webp`，`.svg`，`.gif`）
- 字体文件（`.woff`，`.woff2`）

**不需要缓存的内容**（动态页面）：
- `/admin/` -- 管理面板必须始终提供最新内容
- `/cart/` -- 购物车页面包含会话特定数据
- `/checkout/` -- 结账页面绝不能缓存以确保安全
- `/accounts/` -- 客户账户页面包含私人数据
- 任何需要登录或显示个性化内容的页面

**通用缓存规则**：
- **尊重源站缓存头信息** -- Spwig 会为每种内容类型发送正确的缓存控制头信息。请配置您的 CDN 以遵循这些头信息，而不是覆盖它们。
- **启用 Brotli 压缩** -- Cloudflare 和 CloudFront 都支持 Brotli。启用它以利用 Spwig 的预压缩资源。
- **将浏览器缓存 TTL 设置为 "尊重现有头信息"** -- 这可以让 Spwig 的内置缓存策略驱动行为。

## 验证您的 CDN

设置完成后，确认 CDN 是否正确提供您的内容：

**步骤 1：打开浏览器开发者工具**
- 在 Chrome 或 Firefox 中，按 **F12** 打开开发者工具
- 点击 **Network**（网络）标签

**步骤 2：加载您的商店**
- 在开发者工具打开的情况下访问您的商店首页
- 点击任意静态文件请求（例如，一个 `.css` 或 `.js` 文件）

**步骤 3：检查响应头信息**
- **Cloudflare**：查找 `cf-cache-status` 头信息。值为 `HIT` 表示文件是从 CDN 缓存中提供的。`MISS` 表示文件是从您的服务器获取的（仅首次请求）。
- **CloudFront**：查找 `x-cache` 头信息。值为 `Hit from cloudfront` 表示 CDN 提供了内容。

**步骤 4：从其他位置进行测试**
- 使用免费工具如 gtmetrix.com 或 webpagetest.org 从不同地理位置测试您的商店
- 比较启用 CDN 前后加载时间

## 常见问题

### 主题更改后内容过时

**问题**：在更新主题或进行设计更改后，客户仍然看到旧版本。

**解决方案**：清除 CDN 缓存。在 Cloudflare 中，前往 **Caching > Configuration > Purge Everything**。在 CloudFront 中，为 `/*` 创建 **Invalidation**。请注意，Spwig 的指纹资源通常可以防止此问题，因为更新的文件会自动获得新的文件名。此问题最常见于非指纹资源，如自定义上传。

---

### 混合内容警告

**问题**：启用 CDN 后，浏览器会显示关于 "混合内容" 的安全警告。

**解决方案**：确保您的 CDN 的 SSL 模式设置为 **Full (strict)**，而不是 "Flexible"。Flexible 模式可能导致您的服务器收到 HTTP 请求而不是 HTTPS，从而导致混合内容警告。在 Cloudflare 中，检查 **SSL/TLS > Overview** 并确认模式。

---

### 管理面板运行缓慢

**问题**：添加 CDN 后，管理面板感觉变慢。

**解决方案**：CDN 不应缓存管理面板页面。创建一个 **Page Rule**（Cloudflare）或 **Cache Behavior**（CloudFront），将缓存设置为 "Bypass"，以匹配任何 `/admin/*` 的 URL。这确保管理请求直接发送到您的服务器，而无需经过 CDN 的开销。

---

### 图片无法加载

**问题**：启用 CDN 后，产品图片或媒体文件返回错误。

**解决方案**：确认您的 CDN 的源站配置了正确的协议（HTTPS）和端口。同时检查您的服务器防火墙是否允许来自 CDN IP 范围的连接。

## 小贴士

保留所有 markdown 格式、图片路径、代码块和专业术语。

- **从 Cloudflare 的免费套餐开始** -- 它可以满足大多数商店的需求，设置过程只需几分钟
- **始终使用完整的（严格）SSL 模式** -- 灵活模式会创建安全漏洞，并可能导致结账流程中断
- **在进行重大主题更新后清除 CDN 缓存** -- 尽管 Spwig 的指纹文件可以处理大多数情况，但完全清除缓存可以确保没有过时内容残留
- **不要缓存结账或购物车页面** -- 缓存这些页面可能会导致一个客户的数据暴露给另一个客户
- **从客户所在地区进行测试** -- 使用 webpagetest.org 等免费工具，从客户购物的地区测量实际性能
- **监控 CDN 分析数据** -- Cloudflare 和 CloudFront 都提供了显示缓存命中率、节省的带宽和按国家划分的流量的仪表板
- **在设置期间保持 DNS TTL 较低** -- 在切换到 CDN 时，将 DNS TTL 设置为 300 秒（5 分钟），一旦确认一切正常运行，再将其增加
- **CDN 不能替代优质的托管服务** -- 对于结账、购物车和管理等动态页面，您的源服务器仍然很重要
