---
title: 배송 방법
---

배송 방법은 고객이 체크아웃 시 보는 배송 옵션으로, 각 방법은 다른 가격 책정 전략을 사용하여 배송 비용을 계산합니다. Spwig은 단순한 고정 가격에서 복잡한 운송업체 계산 실시간 가격까지 총 7가지 방법 유형을 지원합니다. 방법은 최소/최대 주문 금액, 무게, 지리적 지역에 따라 제한될 수 있습니다. 고객은 체크아웃 시 선호하는 방법을 선택하고, 계산된 비용은 주문 총액에 추가됩니다.

이 가이드를 사용하여 기본적인 고정 가격 배송에서 복잡한 지역 기반 계층 가격까지 비즈니스 모델에 맞는 배송 방법을 구성하세요.

## 배송 방법 유형

Spwig은 각각 다른 비용 계산 논리가 있는 7가지 배송 방법 유형을 제공합니다:

### 고정 가격 배송

**어떤 것인가**: 장바구니 내용, 목적지, 무게와 관계없이 고정된 비용입니다.

**사용 시기**:
- 예측 가능한 배송 비용이 있는 간단한 가게
- 단일 제품 유형 (유사한 크기/무게)
- 표준 운송업체 비용으로 국내 배송만
- 무료 배송 임계값 프로모션 (배송 규칙과 함께 사용)

**설정**:
- **방법 유형** = 고정 가격 설정
- **고정 비용** 입력 (예: $9.99)
- 선택 사항: 최소/최대 주문 금액 제한 설정

**예시**: "표준 배송 - $9.99"은 모든 국내 주문에 적용됩니다.

---

### 무료 배송

**어떤 것인가**: 고객에게 비용이 없는 무료 배송 옵션입니다.

**사용 시기**:
- 무료 배송 프로모션
- 고가의 주문 (최소 주문 금액과 함께 사용)
- 지역 픽업 대안
- 충성도 프로그램 혜택

**설정**:
- **방법 유형** = 무료 배송 설정
- 선택 사항: **최소 주문 금액** 설정 (예: $50 이상 무료)
- 배송 규칙과 함께 사용하여 조건부 무료 배송 가능

**예시**: "$50 이상 주문 시 무료 배송" 최소 주문 금액 = $50.

---

### 무게 기반 배송

**어떤 것인가**: 총 장바구니 무게를 기반으로 계층 가격 표에 따라 비용이 계산됩니다.

**사용 시기**:
- 가변 무게의 제품 (도서, 하드웨어, 식료품)
- 무게 기반 운송업체 가격 모델
- 예측 가능한 무게 대 비용 비율

**설정**:
1. **방법 유형** = 무게 기반 설정
2. **배송 가격 표** 생성, basis_type = "weight"
3. **배송 가격 계층** 추가 (예: 0-5kg = $10, 5-10kg = $15, 10-20kg = $25)
4. 선택 사항: 특정 지역에 제한

**예시**:
```
0-2kg: $8
2-5kg: $12
5-10kg: $18
10kg+: $25
```

**작동 방식**: 장바구니가 총 무게를 계산 → 해당 계층을 찾음 → 계층의 가격을 반환합니다.

---

### 가격 기반 배송

**어떤 것인가**: 장바구니 소계를 기반으로 계층 가격 표에 따라 비용이 계산됩니다.

**사용 시기**:
- 배송 비용이 주문 금액과 상관관계
- 높은 장바구니 금액 유도 (고계층에서는 달러당 낮은 비율)
- 유사한 가격의 제품에 대한 무게 기반 대안

**설정**:
1. **방법 유형** = 가격 기반 설정
2. **배송 가격 표** 생성, basis_type = "price"
3. **배송 가격 계층** 추가 (예: $0-$50 = $9.99, $50-$100 = $14.99, $100+ = $19.99)

**예시**:
```
$0-$25: $6.99
$25-$75: $9.99
$75-$150: $12.99
$150+: 무료
```

**작동 방식**: 장바구니가 소계를 계산 → 해당 계층을 찾음 → 계층의 가격을 반환합니다.

---

### 실시간 운송업체 비용

**어떤 것인가**: 체크아웃 시 운송업체 API (FedEx, UPS, DHL)에서 실시간으로 가져오는 비용입니다.

**사용 시기**:
- 목적지에 따라 가변적인 배송 비용
- 고객에게 여러 운송업체 옵션 제공
- 수동 가격 표 없이 정확한 운송업체 가격
- 복잡한 가격이 있는 국제 배송

**설정**:
1. **방법 유형** = 실시간 설정
2. **공급업체 계정** 생성 (설정 > 배송 > 공급업체 계정)
3. 운송업체 API 자격 증명 입력 (계정 번호, API 키, 비밀)
4. 공급업체 계정을 배송 방법에 연결
5. 선택 사항: 마크업 비율 또는 고정 마크업 추가

**요구 사항**:
- 활성화된 운송업체 계정 (FedEx, UPS, DHL 등)
- 운송업체에서 제공한 API 자격 증명
- 배송 패키지 정의 (차원 무게 계산을 위해)

**예시**: "FedEx Ground" 방법은 체크아웃 시 장바구니 무게, 차원, 목적지에 따라 실시간 FedEx 비용을 가져옵니다.

**작동 방식**:
1. 고객이 체크아웃 시 주소를 입력합니다.
2. 시스템이 출발지, 목적지, 패키지 차원, 무게를 기반으로 운송업체 API를 호출합니다.
3. 운송업체가 비용 견적을 반환합니다.
4. 선택 사항 마크업 적용
5. 고객에게 비용 표시

---

### 지역 픽업

**어떤 것인가**: 고객이 물리적 위치에서 주문을 직접 수령하는 방법 (배송 비용 없음).

**사용 시기**:
- 픽업을 제공하는 유통업체 가게
- 창고 픽업 옵션
- 이벤트 또는 시장 부스
- 지역 고객에게 배송 비용을 없애는 것

**설정**:
1. **방법 유형** = 지역 픽업 설정
2. **위치** 생성 (설정 > 배송 > 위치)
   - 주소, 영업 시간, 픽업 용량 설정
3. 위치를 방법에 연결
4. 선택 사항: 픽업 준비 시간 설정 (예: "2시간 내 준비 완료")

**고객 경험**:
- 체크아웃 시 "지역 픽업" 선택
- 여러 위치 중 픽업 위치 선택 (선택 사항)
- 가용성 기반 픽업 날짜/시간 선택
- 주문이 준비되었을 때 알림을 받음

**예시**: "가게에서 픽업 - 무료" 3개의 유통업체 위치, 24시간 이내 준비 완료.

---

### 표 기반 배송

**어떤 것인가**: 무게, 가격 또는 수량을 기반으로 한 유연한 계층 가격이며 고급 지역 타겟팅을 지원합니다.

**사용 시기**:
- 복잡한 가격 (지역별 AND 무게별 다른 가격)
- 무게 기반 또는 가격 기반보다 더 많은 제어가 필요한 경우
- 여러 가격 요인 (예: 무게 + 목적지 + 수량)

**설정**:
1. **방법 유형** = 표 기반 설정
2. **배송 가격 표** 생성
3. **basis_type** 정의: 무게, 가격 또는 수량
4. **배송 가격 계층** 추가, 최소/최대 값 설정
5. 선택 사항: 계층을 특정 지역 또는 국가에 제한

**무게/가격 기반과의 차이점**: 표 기반은 계층별로 지리적 제한을 지원하여 동일한 무게/가격이 다른 지역에서 다른 가격을 적용할 수 있습니다.

**예시**:
```
Zone A (국내):
  0-5kg: $10
  5-10kg: $15

Zone B (원격지):
  0-5kg: $18
  5-10kg: $25
```

**작동 방식**: 장바구니가 기준 값 (무게/가격/수량)을 계산 → 고객의 지역에 해당하는 계층을 찾음 → 계층의 가격을 반환합니다.

## 배송 방법 설정

모든 배송 방법은 다음 공통 설정을 공유합니다:

### 기본 설정

- **이름**: 내부 식별자 (고객에게 표시되지 않음)
- **표시 이름**: 체크아웃 시 고객에게 표시되는 이름 (예: "표준 배송", "익스프레스 배송")
- **설명**: 체크아웃 시 표시되는 선택적 도움말 텍스트 (예: "3-5 영업일 내 배송")
- **방법 유형**: 위의 7가지 유형 중 하나
- **활성화**: 방법을 활성화/비활성화할 수 있는 스위치 (삭제 없이)

### 비용 설정

- **고정 비용**: 고정 가격 방법에만 해당
- **가격 표**: 무게 기반, 가격 기반, 표 기반 방법에 해당
- **공급업체 계정**: 실시간 운송업체 방법에 해당
- **세금 분류**: 배송 비용에 세금 적용 (적용 가능한 경우)

### 제한

**주문 금액 제한**:
- **최소 주문 금액**: 장바구니 소계가 금액 이상일 때만 방법이 사용 가능 (예: $50 이상 무료 배송)
- **최대 주문 금액**: 장바구니 소계가 금액을 초과하면 방법이 숨김 (예: $100 미만의 주문에만 고정 가격)

**무게 제한**:
- **최소 무게**: 장바구니 무게가 금액 이상일 때만 방법이 사용 가능
- **최대 무게**: 장바구니 무게가 금액을 초과하면 방법이 숨김 (경량 배송 옵션에 흔함)

**지리적 제한**:
- **배송 지역**: 방법을 특정 지역 (국내, 국제, 지역별)에 연결
- 빈 지역 = 모든 주소에 사용 가능
- 여러 지역 = 해당 지역 중 하나에 사용 가능

### 고급 설정

- **우선순위**: 체크아웃 시 표시 순서 (숫자가 낮을수록 목록 상단에 표시됨)
- **처리 비용**: 계산된 비용에 추가되는 고정 비용
- **무료 배송 임계값**: 장바구니 소계가 임계값 이상일 경우 비용을 $0으로 자동 설정 (min_order_value 대안)

## 배송 방법 생성

**단계별 워크플로우**:

1. **배송 방법으로 이동**
   - 설정 > 장바구니 > 배송 방법으로 이동
   - "배송 방법 추가" 클릭

2. **방법 유형 선택**
   - 가격 전략에 따라 적절한 유형 선택
   - 유형은 사용 가능한 비용 설정 필드를 결정합니다.

3. **기본 정보 설정**
   - 이름: 내부 참조 (예: "domestic_ground")
   - 표시 이름: 고객에게 보이는 이름 (예: "지상 배송")
   - 설명: 배송 기간 (예: "5-7 영업일")

4. **비용 계산 설정**
   - **고정 가격**: 고정 비용 입력
   - **무게/가격/표 기반**: 가격 표 생성 (아래 참조)
   - **실시간**: 공급업체 계정 연결
   - **무료/픽업**: 비용 설정 필요 없음

5. **제한 추가 (선택 사항)**
   - 최소/최대 주문 금액
   - 최소/최대 무게
   - 배송 지역

6. **우선순위 설정**
   - 숫자가 낮을수록 체크아웃 시 먼저 표시됨
   - 권장 순서: 무료 (1), 지역 픽업 (2), 표준 (3), 익스프레드 (4)

7. **방법 활성화**
   - "활성화" = 예
   - 저장

---

## 가격 표 생성

무게 기반, 가격 기반, 표 기반 방법에 대해:

**단계 1: 가격 표 생성**
- 설정 > 배송 > 가격 표로 이동
- "가격 표 추가" 클릭
- **이름** 설정 (예: "국내 무게 계층")
- **기준 유형** 설정: 무게, 가격 또는 수량

**단계 2: 계층 추가**
- "계층 추가" 클릭
- **최소 값** 및 **최대 값** 설정 (일치 범위)
- **가격** 설정 (이 계층의 비용)
- 선택 사항: 특정 지역 또는 국가에 제한
- 계층 저장

**단계 3: 모든 계층 반복**
- 전체 범위 커버 (0에서 최대 예상 값)
- 간격 없음 보장 (예: 0-5, 5-10, 10-20, 20+)
- 최종 계층에서 **최대 값**에 `null` 사용 (무한대)

**단계 4: 배송 방법에 연결**
- 배송 방법 편집
- 드롭다운에서 가격 표 선택
- 저장

**예시 무게 기반 표**:
```
이름: 국내 무게 계층
기준: 무게

계층:
1. 최소: 0g, 최대: 2000g, 가격: $8
2. 최소: 2000g, 최대: 5000g, 가격: $12
3. 최소: 5000g, 최대: 10000g, 가격: $18
4. 최소: 10000g, 최대: null, 가격: $25
```

## 일반적인 배송 시나리오

### 시나리오 1: 기본 국내 배송

**목표**: 모든 국내 주문에 대해 간단한 $9.99 고정 가격.

**해결책**:
- 방법 유형: 고정 가격
- 고정 비용: $9.99
- 배송 지역: "국내" (국가만)

---

### 시나리오 2: $50 이상 무료 배송

**목표**: 무료 배송 임계값을 통해 더 높은 장바구니 금액 유도.

**해결책 A** (권장):
- 방법 유형: 무료 배송
- 최소 주문 금액: $50
- 표시 이름: "$50 이상 주문 시 무료 배송"

**해결책 B** (규칙 사용):
- 방법 유형: 고정 가격
- 고정 비용: $9.99
- 배송 규칙 생성:
  - 조건: 장바구니 금액 ≥ $50
  - 조치: 비용을 $0으로 설정

### 시나리오 3: 국내 + 국제 무게 기반

## 배송 방법 테스트

라이브로 출시하기 전에 모든 방법을 테스트하세요:

1. **테스트 장바구니 생성**
   - 다양한 무게/가격을 가진 제품 추가
   - 체크아웃으로 이동

2. **각 방법 테스트**
   - 다른 지역의 주소 입력
   - 올바른 방법이 표시되는지 확인
   - 계산된 비용이 예상과 일치하는지 확인

3. **제한 테스트**
   - min_order_value 달성까지 항목 추가 → 무료 배송이 표시되는지 확인
   - 무거운 항목 추가 → 무게 기반 계층이 작동하는지 확인
   - 지역 제한 테스트 → 제외된 지역에 대한 방법이 숨겨지는지 확인

4. **실시간 방법 테스트** (적용 가능한 경우)
   - 운송업체 테스트 자격 증명 사용
   - 비용이 성공적으로 반환되는지 확인
   - 운송업체 웹사이트와 비용 정확도 확인

## 문제 해결

**문제 1: 체크아웃 시 방법이 나타나지 않음**

**원인**:
- 방법이 비활성화됨
- 장바구니가 최소/최대 주문 금액을 충족하지 않음
- 장바구니가 최소/최대 무게를 충족하지 않음
- 고객 주소가 연결된 지역과 일치하지 않음
- 가격 표 계층이 장바구니 무게/가격을 커버하지 않음

**해결책**: 제한 확인, 활성화 상태 확인, 고객 시나리오에 해당하는 지역/계층이 있는지 확인.

---

**문제 2: 실시간 비용 실패**

**원인**:
- 잘못된 API 자격 증명
- 공급업체 계정 비활성화
- 정의되지 않은 배송 패키지 (운송업체는 차원이 필요함)
- 출발지 주소 설정되지 않음
- 운송업체 API 다운

**해결책**: 공급업체 연결 테스트, 자격 증명 확인, 패키지 설정 확인, 설정에서 출발지 주소 확인.

---

**문제 3: 계산된 비용이 잘못됨**

**원인**:
- 가격 표 계층에 간격 또는 중복이 있음
- 계층 최소/최대 값이 잘못된 단위 (그램 대 kg)
- 처리 비용이 예상치 않게 추가됨
- 배송 규칙이 비용을 수정함

**해결책**: 가격 표 계층 검토, 단위 확인, 배송 규칙 우선순위 확인.
