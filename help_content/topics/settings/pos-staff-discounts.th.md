---
title: ส่วนลดพนักงาน POS และความปลอดภัยของเทอร์มินัล
---

การตั้งค่าส่วนลดพนักงาน POS ช่วยให้คุณควบคุมว่าพนักงานแต่ละคนสามารถใช้ส่วนลดได้เท่าไรที่จุดขาย (POS) เหตุการณ์การล็อกเทอร์มินัลให้บันทึกการล็อกหรือปลดล็อกเทอร์มินัลทุกครั้ง ช่วยให้คุณติดตามว่าใครเข้าถึงเทอร์มินัล และมีการพยายามเข้าสู่ระบบล้มเหลวหรือไม่

## ข้อจำกัดของส่วนลดพนักงาน

พนักงานที่ใช้ POS แต่ละคนสามารถมีสิทธิ์ในการใช้ส่วนลดได้เป็นรายบุคคล โดยค่าเริ่มต้น พนักงานสามารถใช้ส่วนลดได้สูงสุด 10% สำหรับสินค้าหรือทั้งตะกร้าสินค้า คุณสามารถเพิ่มหรือลดข้อจำกัดนี้ได้ตามแต่ละบุคคล หรือกำหนดให้พนักงานเป็นผู้จัดการที่สามารถอนุมัติส่วนลดที่เกินข้อจำกัดมาตรฐาน

### การตั้งค่าข้อจำกัดส่วนลดของพนักงาน

1. ไปที่ **POS > ส่วนลดพนักงาน**
2. คลิก **+ เพิ่มส่วนลดพนักงาน POS** หรือคลิกพนักงานที่มีอยู่เพื่อแก้ไข
3. เลือก **พนักงาน** จากรายการ
4. ตั้งค่าข้อจำกัดส่วนลด:

| ฟิลด์ | คำอธิบาย |
|-------|-------------|
| **ส่วนลดสูงสุด (%)** | ร้อยละสูงสุดของส่วนลดที่บุคคลนี้สามารถใช้ได้ (เช่น `10` สำหรับ 10%) |
| **จำนวนเงินส่วนลดสูงสุด** | จำนวนเงินคงที่สูงสุดต่อรายการธุรกรรม (เว้นว่างหากไม่ต้องการข้อจำกัดคงที่) |
| **สามารถใช้ส่วนลดสินค้าได้** | อนุญาตให้ใช้ส่วนลดกับรายการสินค้าแต่ละรายการ |
| **สามารถใช้ส่วนลดตะกร้าได้** | อนุญาตให้ใช้ส่วนลดกับยอดรวมตะกร้าทั้งหมด |
| **ต้องระบุเหตุผล** | เมื่อเปิดใช้งาน พนักงานจะต้องพิมพ์เหตุผลก่อนใช้ส่วนลดใด ๆ |

5. คลิก **บันทึก**

### วิธีที่ข้อจำกัดส่วนลดทำงานที่ POS

เมื่อพนักงานแคชเชียร์พยายามใช้ส่วนลด:
- หากส่วนลดอยู่ในข้อจำกัดของพวกเขา ส่วนลดจะถูกใช้ทันที
- หากส่วนลดเกินข้อจำกัดของพวกเขา เทอร์มินัลจะขอ **การอนุมัติจากผู้จัดการ**
- ผู้จัดการจะป้อน PIN เพื่ออนุมัติการยกเว้น และส่วนลดจะถูกใช้

กระบวนการทำงานนี้ช่วยป้องกันการใช้ส่วนลดมูลค่าสูงโดยไม่ได้รับอนุญาต ในขณะเดียวกันก็ยังคงความยืดหยุ่นเมื่อส่วนลดที่แท้จริงจำเป็น

## บทบาทผู้จัดการ

พนักงานที่มีเครื่องหมาย **Is Manager** สามารถอนุมัติส่วนลดที่เกินข้อจำกัดของพนักงานคนอื่นได้

ผู้จัดการจะถูกระบุที่ปลายทางโดยใช้ PIN ที่พวกเขาป้อนเมื่อขออนุมัติ

### การตั้งค่าผู้จัดการ

1. เปิดบันทึกส่วนลดของพนักงาน
2. ตรวจสอบ **Is Manager**
3. ป้อน **Manager PIN** (4-6 หลัก) — นี่จะถูกแฮชอย่างปลอดภัยเมื่อบันทึก
4. คลิก **Save**

Manager PIN แยกจาก PIN แคชเชียร์ที่ใช้สำหรับการล็อก/ปลดล็อกปลายทาง ผู้จัดการสามารถมีทั้ง Manager PIN (สำหรับการอนุมัติส่วนลด) และ PIN แคชเชียร์ (สำหรับการเข้าถึงปลายทาง)

### ความปลอดภัยของ Manager PIN

เมื่อคุณป้อน PIN ในฟอร์มผู้ดูแลระบบและบันทึก Spwig จะแฮชมันอัตโนมัติ — PIN ที่ไม่ได้แฮชจะไม่ถูกเก็บไว้ ฟิลด์ PIN ที่ไม่ได้แฮชจะถูกลบหลังจากบันทึก ซึ่งเป็นพฤติกรรมตามคาด

## PIN แคชเชียร์และการเข้าถึงบัตร

พนักงานแต่ละคนยังสามารถมี **Cashier PIN** สำหรับการล็อกและปลดล็อกปลายทาง:

- **Cashier PIN** — PIN 4-6 หลักที่ใช้ปลดล็อกปลายทางหลังจากที่มันล็อกอัตโนมัติหรือถูกล็อกด้วยมือ
- **Card Identifier** — บัตรที่ลงทะเบียน (บัตรสแกนหรือ NFC) ยังสามารถใช้เพื่อปลดล็อกปลายทางได้ด้วย

เพื่อตั้งค่า PIN แคชเชียร์ ให้ป้อนมันในฟิลด์ **Cashier PIN** และบันทึก คล้ายกับ Manager PIN มันจะถูกแฮชอัตโนมัติเมื่อบันทึก

## เหตุการณ์การล็อกปลายทาง

ทุกครั้งที่ปลายทางถูกล็อกหรือปลดล็อก Spwig จะบันทึกเหตุการณ์การล็อกปลายทาง ซึ่งสร้างประวัติการตรวจสอบความปลอดภัยที่สมบูรณ์

### การดูเหตุการณ์การล็อก

ไปที่ **POS > Terminal Lock Events** เพื่อดูประวัติทั้งหมด คุณสามารถกรองเหตุการณ์ได้ตาม:
- Terminal
- ประเภทเหตุการณ์
- ช่วงวันที่

### ประเภทเหตุการณ์


| เหตุการณ์ | ความหมาย |
|-------|---------|
| **Manual Lock** | พนักงานได้ล็อกเทอร์มินัลโดยเจตนา |
| **Auto-Lock (Idle Timeout)** | เทอร์มินัลถูกล็อกโดยอัตโนมัติเนื่องจากไม่มีการใช้งาน |
| **Unlock by Cashier** | พนักงานขายได้ยืนยันตัวตนและปลดล็อกเทอร์มินัล |
| **Unlock by Manager** | ผู้จัดการได้ใช้ข้อมูลประจำตัวของตนเองเพื่อปลดล็อก |
| **Unlock by Card** | เทอร์มินัลถูกล็อกโดยใช้บัตรที่ลงทะเบียนไว้ |
| **Unlock by Biometric** | เทอร์มินัลถูกล็อกโดยใช้ลายนิ้วมือหรือการจดจำใบหน้า |
| **Failed Unlock Attempt** | มีการพยายามปลดล็อกโดยใช้ข้อมูลประจำตัวที่ไม่ถูกต้อง |
| **Lockout (3+ failures)** | เทอร์มินัลถูกล็อกหลังจากพยายามเข้าถึงซ้ำๆ หลายครั้ง |

### ข้อมูลที่บันทึกในเหตุการณ์ล็อก

แต่ละเหตุการณ์บันทึกข้อมูลดังนี้:
- **เทอร์มินัล** ที่เกี่ยวข้อง
- **ประเภทเหตุการณ์**
- ผู้ที่ดำเนินการ (**Performed By**) และผู้ที่เข้าสู่ระบบเมื่อเกิดเหตุการณ์ล็อก (**Locked By**)
- ว่ามีการใช้ **Manager Override** หรือไม่
- **วิธีการปลดล็อก** (PIN, บัตร หรือการตรวจสอบชีวภาพ)
- **จำนวนการพยายามล้มเหลว** ก่อนเหตุการณ์นี้ (มีประโยชน์ในการตรวจจับรูปแบบการโจมตีด้วยการทายรหัส)
- **ยอดเงินรวม** และจำนวนสินค้าในตะกร้าในขณะเกิดเหตุการณ์
- **ที่อยู่ IP** ของคำขอ

### การสืบสวนปัญหาด้านความปลอดภัย

หากคุณสงสัยว่ามีการเข้าถึงเทอร์มินัลโดยไม่ได้รับอนุญาต:

1. ไปที่ **POS > Terminal Lock Events**
2. กรองตามเทอร์มินัลที่เกี่ยวข้อง
3. ค้นหาเหตุการณ์ประเภท **Failed Unlock Attempt** หรือ **Lockout** — เหล่านี้บ่งบอกถึงการพยายามเข้าถึงซ้ำๆ หลายครั้ง
4. ตรวจสอบฟิลด์ **Performed By** สำหรับการปลดล็อกที่สำเร็จ เพื่อดูว่าใครได้รับการเข้าถึง
5. ตรวจสอบบันทึกกะงาน (**POS > Shifts**) เพื่อยืนยันว่าพนักงานขายคนใดที่ควรอยู่ในกะนั้น

## เคล็ดลับ

- ตั้งค่าข้อจำกัดการให้ส่วนลดตามระดับของพนักงาน — พนักงานใหม่อาจเริ่มต้นที่ 5%, พนักงานที่มีประสบการณ์ที่ 10-15%, และผู้จัดการสามารถอนุมัติส่วนลดที่สูงกว่านี้ได้
- เปิดใช้งาน **Requires Reason** สำหรับพนักงานที่มีข้อจำกัดการให้ส่วนลดที่สูงกว่า

# การมีเหตุผลที่บันทึกไว้ช่วยให้คุณวิเคราะห์รูปแบบการให้ส่วนลดและระบุการใช้โดยไม่เหมาะสม
- ตรวจสอบเหตุการณ์การล็อกอุปกรณ์ทุกสัปดาห์หากร้านค้าของคุณมีพนักงานหลายคนหรือมีการเปลี่ยนพนักงานบ่อยครั้ง — รูปแบบการเข้าถึงที่ผิดปกติจะถูกสังเกตเห็นได้ง่ายขึ้นก่อนที่จะกลายเป็นปัญหา
- หากพนักงานออกจากงาน ให้ลบ PIN ของพนักงานและตัวระบุบัตรทันทีเพื่อป้องกันการเข้าถึงอุปกรณ์
- ใช้เหตุการณ์การล็อกอุปกรณ์เพื่อระบุอุปกรณ์ที่อาจต้องปรับค่าเวลาล็อกอัตโนมัติ — หากลูกค้ามักจะกระตุ้นการล็อกโดยไม่ตั้งใจ เวลาที่กำหนดไว้สำหรับการล็อกเมื่อไม่มีการใช้งานอาจตั้งไว้สั้นเกินไป
- PIN ของผู้จัดการควรเปลี่ยนเป็นระยะ

อัปเดต PIN ใหม่ในบันทึกส่วนลดของพนักงาน — PIN ใหม่จะถูกแฮชเมื่อบันทึก
