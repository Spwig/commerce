---
title: 관련성 가중치 및 심층 색인
---

관련성 가중치와 심층 색인은 검색 결과의 순위를 결정하고 어떤 제품 데이터가 검색되는지 제어합니다. 가중치는 중요도의 배수이며, 2.0의 가중치는 해당 필드에서의 일치가 1.0의 가중치보다 두 배 중요하다는 의미입니다. 심층 색인은 검색이 기본 제품 이름을 넘어 SKU, 속성, 리뷰, 심지어 문서 내용까지 포함하는지 여부를 결정합니다. 이 가이드는 두 시스템을 설명하고, 언제 조정해야 하는지, 그리고 성능에 미치는 중요한 영향을 설명합니다.

대부분의 전자상거래 스토어에 적합한 기본값이 있습니다. 특정한 순위 또는 색인 요구사항이 있는 경우에만 조정하세요.

![Weights Tab](/static/core/admin/img/help/search-settings-overview/search-settings-weights.webp)

## 가중치 이해

가중치는 다양한 필드에서 텍스트 일치가 발생할 때 적용되는 배수(0.0-2.0 범위). 더 높은 가중치는 해당 필드에서의 일치가 결과에서 더 높은 순위를 차지한다는 의미입니다.

**예시**: 제품이 "laptop"을 이름(가중치 1.50)과 설명(가중치 0.80)에 모두 포함하고 있다면:
- 이름 일치는 관련성 점수에 1.50을 기여합니다.
- 설명 일치는 0.80을 기여합니다.
- 합산 점수가 다른 제품과의 순위를 결정합니다.

가중치는 검색 결과 순위를 결정할 때 특정 필드를 다른 필드보다 우선시할 수 있도록 합니다.

## 가중치 범주 및 기본값

**Search Settings > Weights tab**으로 이동하여 모든 가중치 설정을 확인하세요:

| 필드 | 기본 가중치 | 근거 |
|-------|---------------|-----------|
| **weight_name** | 1.50 | 제품 이름이 가장 중요합니다 - 고객은 정확한 이름 일치가 상위에 나타나기를 기대합니다 |
| **weight_sku** | 1.20 | SKU는 구체적인 식별자입니다 - B2B 및 재방문 고객에게 중요합니다 |
| **weight_description** | 0.80 | 설명은 맥락을 제공하지만 정확한 이름 일치보다 덜 중요합니다 |
| **weight_categories** | 0.80 | 카테고리 일치는 탐색에 도움이 되지만 이름/SKU만큼 구체적이지 않습니다 |
| **weight_attributes** | 0.70 | 색상, 사이즈, 재질 검색 - 유용하지만 보조 정보입니다 |
| **weight_brands** | 0.70 | 브랜드 필터링은 중요하지만 대부분의 스토어에서는 주요 검색 기준이 아닙니다 |
| **weight_blog_posts** | 0.60 | 블로그 콘텐츠는 전자상거래 중심의 검색에서 덜 중요합니다(가장 낮은 우선순위) |
| **weight_reviews** | 0.50 | 사용자 생성 콘텐츠는 제어가 가장 덜 되어 있으며, 가장 낮은 가중치입니다 |

이러한 기본값은 제품 발견이 주요 검색 목표인 일반적인 전자상거래 스토어를 가정합니다.

## 가중치 조정 시기

스토어의 우선순위가 일반적인 전자상거래 패턴과 다를 때 가중치를 조정하세요:

**SKU 중심 스토어 (B2B, 도매)** - `weight_sku`를 1.8-2.0으로 증가시켜 제품 코드 검색이 결과에 우세하게 되도록 합니다. B2B 고객은 일반적으로 정확한 SKU로 검색합니다.

**브랜드 중심 스토어** - 고객이 주로 브랜드로 쇼핑하는 경우 (디자이너 의류, 럭셔리 상품) `weight_brands`를 1.2-1.5로 증가시킵니다.

**콘텐츠 중심 스토어** - 블로그 게시물이 제품만큼 중요한 콘텐츠 출판사 또는 교육용 리테일러인 경우 `weight_blog_posts`를 0.9-1.2로 증가시킵니다.

**속성 중심 스토어 (패션)** - 고객이 자주 색상, 사이즈, 스타일 속성으로 검색하는 경우 `weight_attributes`를 1.0-1.2로 증가시킵니다.

## 가중치 조정 예시

| 스토어 유형 | 권장 조정 |
|-----------|------------------------|
| **B2B 도매** | weight_sku: 2.0, weight_name: 1.3, weight_description: 0.6 - 제품 코드 우선 |
| **패션 부티크** | weight_attributes: 1.2, weight_brands: 1.2, weight_name: 1.4 - 색상/스타일/브랜드 중요 |
| **콘텐츠 출판사** | weight_blog_posts: 1.2, weight_name: 1.3, weight_reviews: 0.7 - 콘텐츠가 제품만큼 중요 |
| **일반 전자상거래** | 기본값 사용 - 일반 온라인 스토어에 균형 잡힌 설정 |

한 번에 하나의 가중치만 조정하고 추가 변경 전에 테스트하세요.

## 심층 색인 개요

⚠️ **성능 경고** - 각 심층 색인 옵션은 쿼리 복잡성과 오버헤드를 증가시킵니다.

심층 색인은 기본 제품 이름/설명을 넘어 추가 데이터로 검색을 확장합니다:

![Deep Indexing Tab](/static/core/admin/img/help/search-settings-overview/search-settings-deep-indexing.webp)

**Search Settings > Deep Indexing tab**으로 이동하여 구성하세요.

## SKU 색인

**기본값**: ON, **성능 영향**: 낮음

색인에 제품 및 변형 SKU를 포함합니다. 변형 JOIN을 트리거합니다(소량의 비용).

**ON 유지 시기**: B2B 스토어에서 고객이 제품 코드를 알고 있는 경우 필수입니다. 이전 주문에서 SKU를 기억한 재방문 고객에게도 도움이 됩니다.

**비활성화 시기**: SKU가 하나도 없는 경우에만 비활성화하세요. 성능 영향은 거의 없습니다.

## 속성 색인

**기본값**: ON, **성능 영향**: 중간

색인에 제품 속성(색상, 사이즈, 재질, 사용자 정의 속성)을 포함합니다. 속성 테이블과 JOIN합니다.

**ON 유지 시기**: 패션, 구성 가능한 제품, 또는 고객이 제품 특성("빨간 드레스", "대형 티셔츠")으로 검색하는 경우에 중요합니다.

**비활성화 시기**: 제품당 많은 속성을 가진 20,000개 이상의 제품 카탈로그에서는 50-100ms의 오버헤드가 발생할 수 있습니다. 성능이 매우 중요하고 고객이 속성으로 검색하지 않는 경우에만 비활성화하세요.

## 사용자 정의 필드 색인

**기본값**: ON, **성능 영향**: 중간

색인에 판매자 정의 사용자 정의 필드를 포함합니다. JSONField를 탐색해야 합니다.

**ON 유지 시기**: 사용자 정의 필드를 검색 가능한 제품 데이터(보증 정보, 사양, 호환성 세부 정보)로 사용하는 경우.

**비활성화 시기**: 사용자 정의 필드를 사용하지 않거나, 사용자 정의 필드에 검색 불가능한 데이터(내부 메모, 회계 코드)가 포함된 경우. 비활성화하면 JSONField 처리 오버헤드를 절약할 수 있습니다.

## 리뷰 색인

**기본값**: ON, **성능 영향**: 중간-높음

승인된 리뷰 제목과 주석을 검색에 포함합니다. 리뷰 테이블과 JOIN하고 텍스트 검색 오버헤드를 추가합니다.

**ON 유지 시기**: 리뷰 중심의 카탈로그에서 고객이 리뷰 내용을 기반으로 제품을 검색하는 경우("방수 노트북 가방"이 리뷰 텍스트에 나타날 수 있음).

**비활성화 시기**: 제품당 많은 리뷰를 가진 20,000개 이상의 제품 카탈로그 또는 스토어. 대형 카탈로그에 100-200ms의 오버헤드가 추가됩니다.

## 문서 색인

**기본값**: OFF, **성능 영향**: 매우 높음 🚨

**경고: 무심코 켜지 말 것** - 가장 비용이 많이 들는 검색 기능입니다.

문서 색인은 디지털 제품에 첨부된 PDF, DOCX, XLSX 파일에서 텍스트를 추출하여 파일 내용을 검색할 수 있도록 합니다.

**기술적 세부 사항**:
- PyPDF2, python-docx, openpyxl 라이브러리 사용
- 검색 시 동기식 파일 I/O 및 텍스트 추출
- MD5 해시를 통해 파일 추적(파일이 변경될 때만 재색인)
- 대형 파일(10MB 이상의 PDF)에서 시간 초과 가능성

**성능 영향**:
- 매우 비용이 많이 들는 초기 색인(대규모 라이브러리의 경우 분~시간 소요)
- 쿼리 오버헤드가 크게 증가(100-500ms 추가 지연)
- 대형 문서에 대한 메모리 소모가 큼

**활성화할 경우**:
- 디지털 제품을 판매하고 문서 내용을 검색할 수 있는 경우(ebooks, 보고서, 매뉴얼)
- 카탈로그가 작음(500개 미만의 디지털 제품)
- 서버에 충분한 자원이 있음
- 영향을 철저히 테스트했음

**디지털 제품 스토어에 대해**: 고객이 실제로 문서 내용을 검색해야 하는지, 아니면 제품 이름/설명을 검색하는 것으로 충분한지 고려하세요.

## 성능 영향 표

| 기능 | 기본값 | 영향 | 사용 시기 |
|---------|---------|--------|----------|
| SKU 색인 | ON | 낮음 | 항상 (B2B 필수) |
| 속성 색인 | ON | 중간 | 구성 가능한 제품 |
| 사용자 정의 필드 색인 | ON | 중간 | 사용자 정의 필드 사용 |
| 리뷰 색인 | ON | 중간-높음 | 리뷰 중심 스토어 |
| 문서 색인 | OFF | 매우 높음 | 디지털 제품만 (먼저 테스트) |

영향은 일반적인 카탈로그를 가정합니다. 대규모 카탈로그(50,000개 이상의 제품)는 비례적으로 더 높은 오버헤드를 경험합니다.

## 가중치 변경 테스트

가중치를 조정할 때는 다음 테스트 워크플로를 따르세요:

1. **한 번에 하나의 가중치만 변경** - 여러 가중치를 동시에 조정하지 마세요. 결과에 영향을 미친 변경 사항을 알 수 없습니다.
2. **작은 증분** - ±0.2 단위로 조정하세요(예: 1.0 → 1.2, 1.0 → 1.8이 아님)
3. **실제 쿼리로 테스트** - 분석에서 실제 고객 검색어를 사용하세요. 무작위 테스트는 피하세요.
4. **분석 모니터링** - 상위 쿼리 사용 전후의 결과 관련성을 비교하세요.
5. **1-2주 기다리기** - 고객이 새로운 순위와 상호작용할 시간을 주세요.
6. **클릭-through 비율 측정** - 고객이 결과를 더 많이/적게 클릭하는지 확인하세요.

## 성능 대 정확도의 트레이드오프

더 많은 색인 = 더 나은 검색 결과지만 더 느린 성능:

**시나리오: 작은 카탈로그 (<1,000 제품)**
- 모든 색인 옵션(SKU, 속성, 사용자 정의 필드, 리뷰)을 활성화하세요
- 성능 영향은 최소합니다
- 포괄적인 검색 기능

**시나리오: 중간 카탈로그 (1,000-10,000 제품)**
- SKU, 속성, 사용자 정의 필드를 ON으로 유지하세요
- 제품당 평균 리뷰가 10개 이상인 경우 리뷰 색인을 고려하여 비활성화하세요
- 응답 시간을 모니터링하세요

**시나리오: 대규모 카탈로그 (>10,000 제품)**
- SKU를 ON으로 유지하세요(낮은 영향)
- 리뷰 색인을 비활성화하세요(높은 영향)
- 사용하지 않는 경우 사용자 정의 필드 색인을 비활성화하세요
- 문서 색인을 절대 활성화하지 마세요
- 제품이 50,000개 이상인 경우 Elasticsearch를 고려하세요

카탈로그 크기와 서버 자원에 따라 균형을 맞추세요.

## 엔진별 가중치 오버라이드

검색 엔진을 마법사(단계 3)를 통해 생성할 때 해당 엔진에 대한 전역 가중치를 오버라이드할 수 있습니다.

**사용 사례**: 블로그 중심 엔진
- "blog" 엔진을 생성하세요
- `weight_blog_posts`를 1.5로 오버라이드(전역 0.60 대비)
- 블로그 엔진 검색에서 블로그 콘텐츠가 더 높은 순위를 차지하게 됩니다

대부분의 엔진은 가중치를 오버라이드하지 않아야 합니다 - 빈칸으로 남겨두면 전역 설정을 상속받습니다.

## 팁

- **절대 문서 색인을 활성화하지 마세요, 만약 정말 필수적인 경우가 아니면** - 모든 검색 기능 중에서 성능 비용이 가장 높습니다
- **B2B 스토어: weight_sku를 2.0으로 증가하세요** - 제품 코드는 주요 검색 방법입니다
- **저조한 트래픽 시간대에 가중치 변경을 테스트하세요** - 피크 시간 전에 성능 영향을 관찰하세요
- **색인을 활성화한 후 응답 시간을 모니터링하세요** - 분석 대시보드에서 느려짐을 확인하세요
- **20,000개 이상의 제품 카탈로그에서 리뷰 색인을 비활성화하세요** - 성능에 큰 영향을 줍니다
- **테스트 시 한 번에 하나의 가중치만 변경하세요** - 동시에 변경하면 원인/결과를 판단할 수 없습니다
- **문서 추출은 PyPDF2/docx/openpyxl이 필요합니다** - 문서 색인을 활성화하기 전에 이러한 라이브러리가 설치되어 있는지 확인하세요

기억하세요: 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어를 보존 규칙에 따라 정확히 유지하세요.
