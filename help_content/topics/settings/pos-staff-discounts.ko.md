---
title: POS 직원 할인 및 터미널 보안
---

POS 직원 할인 설정은 각 직원이 판매 시점에서 적용할 수 있는 할인 금액을 제어할 수 있도록 합니다. 터미널 잠금 이벤트는 터미널이 잠겨 있거나 잠금이 해제된 모든 시점에 대한 감사 추적을 제공하여 터미널에 접근한 사람과 실패한 로그인 시도가 있었는지 추적하는 데 도움이 됩니다.

## 직원 할인 한도

POS를 사용하는 각 직원은 개별 할인 권한을 가질 수 있습니다. 기본적으로 직원은 항목 또는 전체 장바구니에 대해 최대 10%의 할인을 적용할 수 있습니다. 사람별로 이 한도를 높일 수 있고, 또는 직원을 관리자로 지정하여 표준 한도를 초과하는 할인을 승인할 수 있도록 할 수 있습니다.

### 직원의 할인 한도 설정

1. **POS > 직원 할인**으로 이동합니다.
2. **+ 직원 할인 추가**를 클릭하거나 기존 직원을 클릭하여 편집합니다.
3. 목록에서 **직원**을 선택합니다.
4. 할인 한도를 설정합니다:

| 필드 | 설명 |
|-------|-------------|
| **최대 할인 %** | 이 사람이 적용할 수 있는 최대 할인 비율 (예: `10`은 10%) |
| **최대 할인 금액** | 거래당 고정된 최대 금액 (고정한도 없이 비워둘 수 있음) |
| **항목 할인 적용 가능** | 개별 항목에 할인을 적용할 수 있도록 허용 |
| **장바구니 할인 적용 가능** | 전체 장바구니 금액에 할인을 적용할 수 있도록 허용 |
| **이유 필요** | 체크되면 직원이 할인을 적용하기 전에 이유를 입력해야 함 |

5. **저장**을 클릭합니다.

### POS에서 할인 한도 작동 방식

계산원이 할인을 적용하려 할 때:
- 할인이 한도 내에 있는 경우 즉시 적용됩니다.
- 할인이 한도를 초과하는 경우 터미널에서 **관리자 승인**을 요청합니다.
- 관리자는 PIN을 입력하여 오버라이드를 승인하고 할인을 적용합니다.

이 워크플로는 정당한 할인이 필요한 경우 유연성을 제공하면서도 비정상적인 고가 할인을 방지합니다.

## 관리자 역할

**Is Manager** 플래그를 가진 직원은 다른 직원의 한도를 초과하는 할인을 승인할 수 있습니다.

매니저는 승인 요청 시 터미널에서 PIN을 입력하여 식별됩니다.

### 매니저 설정

1. 직원의 할인 기록을 열어주세요
2. **매니저 여부**를 확인하세요
3. **매니저 PIN** (4~6자리)을 입력하세요 — 이 값은 저장 시 안전하게 해시 처리됩니다
4. **저장**을 클릭하세요

매니저 PIN은 터미널 잠금/해제에 사용되는 캐셔 PIN과 별도입니다. 매니저는 할인 승인에 사용되는 매니저 PIN과 터미널 접근에 사용되는 캐셔 PIN을 모두 가질 수 있습니다.

### 매니저 PIN 보안

관리자 폼에서 PIN을 입력하고 저장하면 Spwig는 자동으로 이를 해시 처리합니다 — 원본 PIN은 절대 저장되지 않습니다. 원본 PIN 필드는 저장 후 자동으로 지워지며, 이는 예상되는 동작입니다.

## 캐셔 PIN 및 카드 접근

각 직원은 터미널 잠금/해제에 사용할 수 있는 **캐셔 PIN**도 가질 수 있습니다:

- **캐셔 PIN** — 터미널이 자동으로 잠금되거나 수동으로 잠금된 후 터미널을 해제하기 위해 사용되는 4~6자리 PIN
- **카드 식별자** — 등록된 카드(스와이프 카드 또는 NFC)도 터미널을 해제하는 데 사용할 수 있습니다

캐셔 PIN을 설정하려면 **캐셔 PIN** 필드에 입력한 후 저장하세요. 매니저 PIN과 마찬가지로 이 값은 저장 시 자동으로 해시 처리됩니다.

## 터미널 잠금 이벤트

터미널이 잠금되거나 해제될 때마다 Spwig는 터미널 잠금 이벤트를 기록합니다. 이는 완전한 보안 감사 추적을 생성합니다.

### 잠금 이벤트 보기

**POS > 터미널 잠금 이벤트**로 이동하여 전체 기록을 확인할 수 있습니다. 이벤트를 다음으로 필터링할 수 있습니다:
- 터미널
- 이벤트 유형
- 날짜 범위

### 이벤트 유형


| 이벤트 | 의미 |
|-------|---------|
| **수동 잠금** | 직원이 의도적으로 터미널을 잠궜습니다 |
| **자동 잠금 (사용자 비활성화 타임아웃)** | 터미널이 비활성화로 인해 자동으로 잠겼습니다 |
| **계산원에 의한 잠금 해제** | 계산원이 인증하여 터미널을 잠금 해제했습니다 |
| **매니저에 의한 잠금 해제** | 매니저가 자격 증명을 사용하여 잠금 해제했습니다 |
| **카드에 의한 잠금 해제** | 등록된 스와이프 카드를 사용하여 터미널이 잠금 해제되었습니다 |
| **생체 인식에 의한 잠금 해제** | 지문 또는 얼굴 인식을 사용하여 터미널이 잠금 해제되었습니다 |
| **실패한 잠금 해제 시도** | 잘못된 자격 증명으로 잠금 해제 시도가 이루어졌습니다 |
| **차단 (3회 이상 실패)** | 반복된 실패 시도 후 터미널이 차단되었습니다 |

### 잠금 이벤트 기록에 포함된 내용

각 이벤트는 다음을 기록합니다:
- 관여된 **터미널**
- **이벤트 유형**
- 작업을 수행한 사람 (**작업자**) 및 잠금 발생 시 로그인한 사람 (**잠금자**)
- **매니저 오버라이드**가 사용되었는지 여부
- **잠금 해제 방법** (PIN, 카드 또는 생체 인식)
- 이 이벤트 전의 **실패 시도** (브루트포스 패턴을 감지하는 데 유용)
- 이벤트 발생 시의 **카트 총액** 및 항목 수
- 요청의 IP 주소

### 보안 문제 조사

터미널에 대한 비인가된 접근이 의심된다면:

1. **POS > 터미널 잠금 이벤트**로 이동합니다
2. 조사하고자 하는 터미널로 필터링합니다
3. **실패한 잠금 해제 시도** 또는 **차단** 유형의 이벤트를 확인합니다 — 이는 반복된 실패 접근을 나타냅니다
4. 성공적인 잠금 해제의 **작업자** 필드를 확인하여 접근한 사람을 파악합니다
5. **POS > Shifts**에 있는 근무 기록과 비교하여 해당 근무 중이어야 했던 계산원을 확인합니다

## 팁

- 직원의 경력 수준에 따라 할인 한도를 설정하세요 — 신입 직원은 5%에서 시작하고, 경력이 있는 직원은 10-15%, 매니저는 더 높은 할인을 승인할 수 있습니다.
- 더 높은 할인 한도를 가진 직원에게 **이유 입력 필수**를 활성화하세요.

# 파일에 저장된 사유가 있으면 할인 패턴을 분석하고 모든 악용을 식별하는 데 도움이 됩니다.
- 매장에 여러 직원이 있거나 직원 이탈률이 높은 경우, 주간 단위로 터미널 락아웃 이벤트를 검토하세요 — 불규칙한 접근 패턴은 문제가 되기 전에 더 쉽게 발견할 수 있습니다.
- 직원이 퇴사할 경우, 즉시 그들의 캐셔 PIN과 카드 식별자를 제거하여 터미널 접근을 방지하세요.
- 락아웃 이벤트를 사용하여 자동 락 타임아웃을 조정해야 할 터미널을 식별하세요 — 고객들이 자주 실수로 락아웃을 유발하는 경우, 유휴 타임아웃이 너무 짧게 설정되어 있을 수 있습니다.
- 매니저 PIN은 주기적으로 변경해야 합니다.

직원 할인 기록에서 업데이트하세요 — 새 PIN은 저장 시 해시됩니다.
