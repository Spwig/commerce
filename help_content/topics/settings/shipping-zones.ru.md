title: Зоны доставки
---

Зоны доставки определяют географические регионы для целевых тарифов на доставку — группируйте страны, штаты или почтовые индексы в зоны, а затем связывайте методы доставки с конкретными зонами для точного контроля над тарифами. При совпадении адреса с несколькими зонами зоны используют приоритетное сопоставление (выигрывает зона с наивысшим приоритетом). Эта система позволяет использовать продвинутые стратегии ценообразования: взимать больше за отдаленные районы, предлагать бесплатную доставку внутри страны или предоставлять скидки для определенных регионов.

Используйте зоны, когда вам нужны разные тарифы на доставку для разных географических зон, от простого разделения между национальным и международным до сложной многоуровневой ценовой политики для нескольких регионов.

## Понимание зон доставки

**Что такое зоны**: Названные географические регионы, определенные с помощью кодов стран, штатов/провинций и шаблонов почтовых индексов.

**Как работают зоны**:
1. Покупатель вводит адрес доставки на этапе оформления заказа
2. Система оценивает все активные зоны
3. Зоны, совпадающие с адресом клиента, становятся кандидатами
4. Если несколько зон совпадают, зона с наивысшим приоритетом выигрывает
5. Методы доставки, связанные с выигравшей зоной, отображаются
6. Методы, не связанные с какой-либо зоной (или связанные с совпадающей зоной), отображаются

**Компоненты зоны**:
- **Название**: Идентификатор зоны (например, "Домашняя", "ЕС", "Отдаленные районы")
- **Страны**: Список включенных кодов стран (пустой = все страны)
- **Штаты/провинции**: Ограничения по штатам для конкретных стран (опционально)
- **Шаблоны почтовых индексов**: Регулярные выражения для сопоставления почтовых индексов (опционально)
- **Приоритет**: Чем выше число, тем выше приоритет при совпадении нескольких зон


## Логика сопоставления зон

Зоны используют **постепенное сужение** для сопоставления адресов:

### Уровень 1: Сопоставление по стране

**Пустой список стран** → Зона совпадает со всеми странами

**Список стран предоставлен** → Страна адреса должна быть в списке

Пример:
```
Зона: "Домашняя"
Страны: ["US"]
→ Совпадает: Любой адрес в США
→ Не совпадает: Канада, Великобритания и т.д.
```

### Уровень 2: Сопоставление по штату/провинции

**Не определены штаты** → Зона совпадает со всеми штатами в разрешенных странах

**Определены штаты для конкретных стран** → Штат адреса должен совпадать

Пример:
```
Зона: "Западное побережье"
Страны: ["US"]
Штаты: {"US": ["CA", "OR", "WA"]}
→ Совпадает: Адреса Калифорнии, Орегона, Вашингтона
→ Не совпадает: Нью-Йорк, Техас и т.д.
```

### Уровень 3: Сопоставление по почтовому индексу

**Не определены шаблоны** → Зона совпадает со всеми почтовыми индексами в разрешенных странах/штатах

**Определены шаблоны** → Почтовый индекс адреса должен совпадать хотя бы с одним шаблоном

Пример:
```
Зона: "Метрополитен Лос-Анджелес"
Страны: ["US"]
Штаты: {"US": ["CA"]}
Шаблоны почтовых индексов: ["^90[0-9]{3}$", "^91[0-9]{3}$"]
→ Совпадает: 90001, 91210, 90245
→ Не совпадает: 94102 (Сан-Франциско)
```

**Примеры шаблонов регулярных выражений**:
- `^90[0-9]{3}$` - Район Лос-Анджелеса (90000-90999)
- `^[A-Z][0-9][A-Z] [0-9][A-Z][0-9]$` - Формат почтового индекса Канады (K1A 0B1)
- `^SW[0-9]{1,2}` - Почтовые индексы Лондона, начинающиеся с SW


## Выбор зоны на основе приоритета

Когда несколько зон совпадают с адресом, **приоритет** определяет, какая зона применяется:

**Как работает приоритет**:
- Чем выше число, тем выше приоритет
- Если адрес совпадает с зонами с приоритетом 100 и 50, выигрывает зона с приоритетом 100
- Доступны только методы доставки выигравшей зоны

**Сценарии использования**:

**Сценарий 1: Конкретная зона переопределяет общую**
```
Зона A: "Отдаленный Аляска"
  Страны: ["US"]
  Штаты: {"US": ["AK"]}
  Приоритет: 100

Зона B: "Домашняя США"
  Страны: ["US"]
  Приоритет: 50

Адрес: Анкоридж, Аляска
→ Совпадает с обеими зонами
→ Приоритет 100 выигрывает
→ Применяется зона "Отдаленный Аляска" (высокая стоимость доставки)
```

**Сценарий 2: Почтовый индекс переопределяет штат**
```
Зона A: "Премиум Манхэттен"
  Страны: ["US"]
  Штаты: {"US": ["NY"]}
  Шаблоны почтовых индексов: ["^100[0-2][0-9]$"]
  Приоритет: 100

Зона B: "Штат Нью-Йорк"
  Страны: ["US"]
  Штаты: {"US": ["NY"]}
  Приоритет: 50

Адрес: Нью-Йорк, Нью-Йорк 10001
→ Совпадает с обеими зонами
→ Приоритет 100 выигрывает
→ Применяется "Премиум Манхэттен" (премиум-услуга доставки)
```


## Создание зон доставки

**Пошаговая инструкция**:

1. **Перейдите к зонам**
   - Перейдите в Настройки > Доставка > Зоны доставки
   - Нажмите "Добавить зону доставки"

2. **Базовая настройка**
   - **Название**: Описательный идентификатор (например, «Европейский Союз», «Западное побережье», «Отдаленные районы»)
   - **Приоритет**: Установите относительную важность (100 для конкретных, 50 для общих, 1 для резервного)
   - **Активен**: Переключатель для включения/отключения

3. **Определение географического охвата**

   **Вариант A: Все страны** (оставьте список стран пустым)
   - Зона соответствует всем адресам по всему миру
   - Используется для дефолтных/резервных зон

   **Вариант B: Конкретные страны**
   - Нажмите «Добавить страну»
   - Выберите страны из выпадающего списка (США, Канада, Великобритания и т.д.)
   - Повторите для всех включенных стран

   **Вариант C: Конкретные штаты/провинции**
   - После добавления стран нажмите «Добавить штаты» для каждой страны
   - Выберите штаты из выпадающего списка
   - Пример: США → CA, OR, WA для Западного побережья

   **Вариант D: Модели почтовых индексов** (расширенный)
   - Введите регулярные выражения (по одному на строку)
   - Тестируйте модели с образцовыми почтовыми индексами
   - Нажмите «Проверить модели», чтобы проверить синтаксис

4. **Связь с методами доставки**
   - Методы можно связать при редактировании метода (не в настройках зоны)
   - Или связать зоны с существующими методами: Редактировать метод → Зоны доставки → Выбрать зоны

5. **Установка приоритета отображения**
   - Зоны с более высоким приоритетом переопределяют зоны с более низким приоритетом, если несколько совпадают
   - Рекомендуется: конкретные зоны (100), региональные зоны (50), дефолтная зона (1)

6. **Активировать зону**
   - Переключатель «Активен» = Да
   - Сохранить

## Распространенные настройки зон

### Настройка 1: Внутренняя vs международная доставка

**Цель**: Разные тарифы для внутренней доставки и всех других стран.

```
Зона 1: «Внутренняя»
  Страны: [Ваш код страны]
  Приоритет: 50

Зона 2: «Международная»
  Страны: [Оставьте пустым или выберите все остальные страны]
  Приоритет: 1
```

**Методы доставки**:
- «Стандартная внутренняя доставка» → Связана с зоной Внутренняя
- «Международная доставка» → Связана с зоной Международная

---

### Настройка 2: Многообластная международная доставка

**Цель**: Разные тарифы для ЕС, Северной Америки, Азии, Остального мира.

```
Зона 1: «Европейский Союз»
  Страны: [AT, BE, BG, CY, CZ, DE, DK, EE, ES, FI, FR, GR, HR, HU, IE, IT, LT, LU, LV, MT, NL, PL, PT, RO, SE, SI, SK]
  Приоритет: 100

Зона 2: «Северная Америка»
  Страны: [US, CA, MX]
  Приоритет: 100

Зона 3: «Азиатско-тихоокеанский регион»
  Страны: [AU, CN, HK, IN, JP, KR, NZ, SG, TH, TW]
  Приоритет: 100

Зона 4: «Остальной мир»
  Страны: [Оставьте пустым]
  Приоритет: 1
```

**Методы доставки**:
- «Доставка в ЕС» → Зона ЕС
- «Доставка в Северную Америку» → Зона Северная Америка
- «Доставка в Азиатско-тихоокеанский регион» → Зона Азиатско-тихоокеанский регион
- «Стандартная международная доставка» → Зона Остальной мир

---

### Настройка 3: Дополнительная плата за отдаленные районы

**Цель**: Добавить дополнительную плату за отдаленные почтовые индексы в пределах внутренней зоны.

```
Зона 1: «Отдаленная внутренняя»
  Страны: [US]
  Модели почтовых индексов: ["^99[0-9]{3}$", "^96[7-9][0-9]{2}$"]  # Аляска, Гавайи
  Приоритет: 100

Зона 2: «Стандартная внутренняя»
  Страны: [US]
  Приоритет: 50
```

**Методы доставки**:
- «Доставка в отдаленные районы» → Зона Отдаленная внутренняя (высокая стоимость)
- «Стандартная доставка» → Зона Стандартная внутренняя

---

### Настройка 4: Зоны по штатам

**Цель**: Разные тарифы для каждой области США.

```
Зона 1: «Западное побережье»
  Страны: [US]
  Штаты: {"US": ["CA", "OR", "WA"]}
  Приоритет: 100

Зона 2: «Восточное побережье»
  Страны: [US]
  Штаты: {"US": ["NY", "NJ", "CT", "MA", "PA"]}
  Приоритет: 100

Зона 3: «Средний Запад»
  Страны: [US]
  Штаты: {"US": ["IL", "IN", "OH", "MI", "WI"]}
  Приоритет: 100

Зона 4: «Юг»
  Страны: [US]
  Штаты: {"US": ["TX", "FL", "GA", "NC", "SC"]}
  Приоритет: 100

Зона 5: «Другие штаты США»
  Страны: [US]
  Приоритет: 50
```

## Примеры моделей почтовых индексов

Почтовые индексы используют **регулярные выражения** (регулярные выражения) для сопоставления моделей:

### США (ZIP-коды)

**Формат**: 5 цифр (например, 90210)

```
Калифорния (90000-96199):  ^9[0-6][0-9]{3}$
Нью-Йорк (10000-14999):    ^1[0-4][0-9]{3}$
Техас (75000-79999, 88500-88599):  ^(7[5-9]|885)[0-9]{2}$
Аляска (99500-99999):      ^99[5-9][0-9]{2}$
```

### Канада (почтовые коды)

**Формат**: A1A 1A1 (буква-цифра-буква пробел цифра-буква-цифра)


Все канадские почтовые индексы:  ^[A-Z][0-9][A-Z] [0-9][A-Z][0-9]$

- **Начните с 2 зон** - Внутренняя и Международная, расширьте при необходимости
- **Используйте приоритеты с умом** - Конкретные зоны 100, региональные 50, резервная 1
- **Тщательно протестируйте постальные шаблоны** - Ошибки регулярных выражений не отображаются, что приводит к тому, что зоны не совпадают
- **Документируйте логику зон** - Добавьте примечания в описание зоны, объясняющие намерение охвата
- **Избегайте излишнего количества зон** - Слишком много зон усложняет настройку; используйте акции доставки для сложных сценариев
- **Используйте коды штатов, а не названия** - "CA" вместо "California", "NY" вместо "New York"
- **Создайте резервную зону** - Все страны, приоритет 1, гарантирует, что всегда будет доступна как минимум одна опция доставки
- **Контролируйте эффективность зон** - Если много клиентов видят "нет доступной доставки", проверьте охват зон
- **Обновляйте зоны для новых регионов** - Добавляйте страны в зону ЕС, когда новые члены присоединяются
- **Используйте описательные названия** - "ЕС (без Великобритании)" лучше, чем "Зона 3"
- **Тестируйте с реальными адресами** - Используйте реальные адреса клиентов во время тестирования, а не вымышленные
