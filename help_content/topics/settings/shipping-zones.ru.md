---
title: Зоны доставки
---

Зоны доставки определяют географические регионы для целевых тарифов на доставку — группируйте страны, штаты или почтовые индексы в зоны, а затем связывайте методы доставки с конкретными зонами для точного контроля тарифов. При совпадении адреса с несколькими зонами зоны используют приоритетное сопоставление (высший приоритет выигрывает). Эта система позволяет реализовывать продвинутые стратегии ценообразования: взимать больше за доставку в удаленные районы, предлагать бесплатную доставку внутри страны или предоставлять скидки на доставку в определенных регионах.

Используйте зоны, когда вам нужны разные тарифы на доставку для разных географических регионов, от простого разделения между национальным и международным до сложной многоуровневой ценовой политики для нескольких регионов.

## Понимание зон доставки

**Что такое зоны**: Названные географические регионы, определенные с помощью кодов стран, штатов/провинций и шаблонов почтовых индексов.

**Как работают зоны**:
1. Покупатель вводит адрес доставки на этапе оформления заказа
2. Система оценивает все активные зоны
3. Зоны, совпадающие с адресом покупателя, становятся кандидатами
4. Если совпадают несколько зон, зона с наивысшим приоритетом выигрывает
5. Методы доставки, связанные с выигравшей зоной, отображаются
6. Методы, не связанные с какой-либо зоной (или связанные с совпадающей зоной), отображаются

**Компоненты зоны**:
- **Название**: Идентификатор зоны (например, "Домашняя", "ЕС", "Удаленные районы")
- **Страны**: Список включенных кодов стран (пустой = все страны)
- **Штаты/провинции**: Ограничения по штатам для конкретных стран (опционально)
- **Шаблоны почтовых индексов**: Регулярные выражения для сопоставления почтовых индексов (опционально)
- **Приоритет**: Чем выше число, тем выше приоритет при совпадении нескольких зон

## Логика сопоставления зон

Зоны используют **постепенное сужение** для сопоставления адресов:

### Уровень 1: Сопоставление по стране

**Пустой список стран** → Зона совпадает со ВСЕМИ странами

**Список стран предоставлен** → Страна адреса должна быть в списке

Пример:
```
Зона: "Домашняя"
Страны: ["US"]
→ Совпадает: Любой адрес США
→ Не совпадает: Канада, Великобритания и т.д.
```

### Уровень 2: Сопоставление по штату/провинции

**Не определены штаты** → Зона совпадает со ВСЕМИ штатами в разрешенных странах

**Определены штаты для конкретных стран** → Штат адреса должен совпадать

Пример:
```
Зона: "Западное побережье"
Страны: ["US"]
Штаты: {"US": ["CA", "OR", "WA"]}
→ Совпадает: Адреса Калифорнии, Орегона, Вашингтона
→ Не совпадает: Нью-Йорк, Техас и т.д.
```

### Уровень 3: Сопоставление по почтовому индексу

**Не определены шаблоны** → Зона совпадает со ВСЕМИ почтовыми индексами в разрешенных странах/штатах

**Определены шаблоны** → Почтовый индекс адреса должен совпадать хотя бы с одним шаблоном

Пример:
```
Зона: "Метрополитен Лос-Анджелеса"
Страны: ["US"]
Штаты: {"US": ["CA"]}
Шаблоны почтовых индексов: ["^90[0-9]{3}$", "^91[0-9]{3}$"]
→ Совпадает: 90001, 91210, 90245
→ Не совпадает: 94102 (Сан-Франциско)
```

**Примеры шаблонов регулярных выражений**:
- `^90[0-9]{3}$` - Район Лос-Анджелеса (90000-90999)
- `^[A-Z][0-9][A-Z] [0-9][A-Z][0-9]$` - Формат почтовых индексов Канады (K1A 0B1)
- `^SW[0-9]{1,2}` - Почтовые индексы Лондона, начинающиеся с SW

## Выбор зоны на основе приоритета

Когда несколько зон совпадают с адресом, **приоритет** определяет, какая зона применяется:

**Как работает приоритет**:
- Чем выше число, тем выше приоритет
- Если адрес совпадает со зонами с приоритетом 100 и 50, выигрывает зона с приоритетом 100
- Доступны только методы доставки выигравшей зоны

**Сценарии использования**:

**Сценарий 1: Конкретная зона переопределяет общую**
```
Зона A: "Удаленная Аляска"
  Страны: ["US"]
  Штаты: {"US": ["AK"]}
  Приоритет: 100

Зона B: "Домашняя США"
  Страны: ["US"]
  Приоритет: 50

Адрес: Анкоридж, АК
→ Совпадает с обеими зонами
→ Приоритет 100 выигрывает
→ Применяется зона "Удаленная Аляска" (высокая стоимость доставки)
```

**Сценарий 2: Почтовый индекс переопределяет штат**
```
Зона A: "Премиум Манхэттен"
  Страны: ["US"]
  Штаты: {"US": ["NY"]}
  Шаблоны почтовых индексов: ["^100[0-2][0-9]$"]
  Приоритет: 100

Зона B: "Штат Нью-Йорк"
  Страны: ["US"]
  Штаты: {"US": ["NY"]}
  Приоритет: 50

Адрес: Нью-Йорк, Нью-Йорк, 10001
→ Совпадает с обеими зонами
→ Приоритет 100 выигрывает
→ Применяется "Премиум Манхэттен" (премиум-услуга доставки)
```

## Создание зон доставки

**Пошаговая инструкция**:

1. **Перейдите к зонам**
   - Перейдите в Настройки > Доставка > Зоны доставки
   - Нажмите "Добавить зону доставки"

2. **Базовая настройка**
   - **Название**: Описательный идентификатор (например, "Европейский Союз", "Западное побережье", "Удаленные районы")
   - **Приоритет**: Установите относительную важность (100 для конкретных, 50 для общих, 1 для резервной)
   - **Активна**: Включите/отключите

3. **Определите географическое охват**

   **Вариант A: Все страны** (оставьте список стран пустым)
   - Зона совпадает со всеми адресами в мире
   - Используйте для зон по умолчанию/резервных

   **Вариант B: Конкретные страны**
   - Нажмите "Добавить страну"
   - Выберите страны из выпадающего списка (US, CA, UK и т.д.)
   - Повторите для всех включенных стран

   **Вариант C: Конкретные штаты/провинции**
   - После добавления стран нажмите "Добавить штаты" для каждой страны
   - Выберите штаты из выпадающего списка
   - Пример: US → CA, OR, WA для Западного побережья

   **Вариант D: Шаблоны почтовых индексов** (расширенный)
   - Введите шаблоны регулярных выражений (по одному на строку)
   - Тестируйте шаблоны с образцовыми почтовыми индексами
   - Нажмите "Проверить шаблоны" для проверки синтаксиса

4. **Связать с методами доставки**
   - Методы могут быть связаны при редактировании метода (не в настройке зоны)
   - Или связать зоны с существующими методами: Редактировать метод → Зоны доставки → Выбрать зоны

5. **Установить приоритет отображения**
   - Зоны с более высоким приоритетом переопределяют зоны с более низким приоритетом при совпадении нескольких зон
   - Рекомендуется: Конкретные зоны (100), Региональные зоны (50), Зона по умолчанию (1)

6. **Активировать зону**
   - Включите "Активна" = Да
   - Сохраните

## Распространенные настройки зон

### Настройка 1: Внутри страны и международная доставка

**Цель**: Разные тарифы для доставки внутри страны и всех других стран.

```
Зона 1: "Внутри страны"
  Страны: [Ваш код страны]
  Приоритет: 50

Зона 2: "Международная"
  Страны: [Оставьте пустым или выберите все остальные страны]
  Приоритет: 1
```

**Методы доставки**:
- "Стандартная доставка внутри страны" → Связана с зоной "Внутри страны"
- "Международная доставка" → Связана с зоной "Международная"

Зона 1: "Европейский Союз"

## Примеры шаблонов почтовых индексов

Почтовые индексы используют **регулярные выражения** для сопоставления шаблонов:

### США (ZIP-коды)

**Формат**: 5 цифр (например, 90210)

```
Калифорния (90000-96199):  ^9[0-6][0-9]{3}$
Нью-Йорк (10000-14999):    ^1[0-4][0-9]{3}$
Техас (75000-79999, 88500-88599):  ^(7[5-9]|885)[0-9]{2}$
Аляска (99500-99999):      ^99[5-9][0-9]{2}$
```

### Канада (Почтовые индексы)

**Формат**: A1A 1A1 (буква-цифра-буква пробел цифра-буква-цифра)

```
Все канадские почтовые индексы:  ^[A-Z][0-9][A-Z] [0-9][A-Z][0-9]$ 
Онтарио (K, L, M, N, P):    ^[KLMNP][0-9][A-Z] [0-9][A-Z][0-9]$ 
Квебек (G, H, J):           ^[GHJ][0-9][A-Z] [0-9][A-Z][0-9]$ 
```

### Великобритания (Почтовые индексы)

**Формат**: AA1A 1AA или A1A 1AA

```
Лондон (E, EC, N, NW, SE, SW, W, WC):  ^(E|EC|N|NW|SE|SW|W|WC)[0-9]{1,2}
Манчестер (M):                        ^M[0-9]{1,2}
Бирмингем (B):                        ^B[0-9]{1,2}
```

### Австралия (Почтовые индексы)

**Формат**: 4 цифры (например, 2000)

```
Новый Южный Уэльс (1000-2999):  ^[12][0-9]{3}$
Виктория (3000-3999, 8000-8999):  ^[38][0-9]{3}$
Квинсленд (4000-4999, 9000-9999):  ^[49][0-9]{3}$
```

### Тестирование шаблонов

**Перед сохранением шаблонов** протестируйте их с известными почтовыми индексами:

1. Введите шаблон: `^90[0-9]{3}$`
2. Тестовый ввод: "90210" → Должен совпадать
3. Тестовый ввод: "10001" → Не должен совпадать
4. Тестовый ввод: "9021" → Не должен совпадать (только 4 цифры)

Используйте онлайн-тестеры регулярных выражений (regex101.com), чтобы проверить сложные шаблоны.

## Общая сводка охвата зон

Зоны отображают **сводку охвата** в представлении списка администратора, показывающую, что включено:

**Примеры**:
- "Все страны" → Нет ограничений по странам
- "US, CA, MX" → 3 страны
- "US (CA, OR, WA)" → США с 3 штатами
- "US (90xxx-91xxx)" → США со шаблонами почтовых индексов

**Используйте сводку для**:
- Быстрого проверения охвата зон без открытия
- Обнаружения перекрытий или пробелов в охвате
- Проверки конфигурации зон в один взгляд

## Связывание зон с методами доставки

Зоны и методы имеют **многие ко многим** отношения:

**С точки зрения метода** (Рекомендуется):
1. Редактировать метод доставки
2. Прокрутить до раздела "Зоны доставки"
3. Выбрать применимые зоны (множественный выбор)
4. Сохранить метод

**С точки зрения зоны**:
- Зоны не связаны напрямую с методами
- Связывание всегда выполняется из настройки метода

**Поведение метод-зона**:

**Нет связанных зон** → Метод доступен для ВСЕХ адресов

**Связанные зоны** → Метод доступен только если адрес покупателя совпадает с хотя бы одной связанной зоной

**Пример**:
```
Метод: "Стандартная доставка внутри страны"
Связанные зоны: ["Домашняя США"]
→ Отображается только для адресов США

Метод: "Международная экспресс-доставка"
Связанные зоны: ["ЕС", "Азиатско-тихоокеанский регион", "Остальной мир"]
→ Отображается для всех ненацональных адресов
```

## Тестирование сопоставления зон

Перед запуском проверьте настройку зон:

1. **Создайте тестовые заказы**
   - Используйте адреса в разных зонах
   - Проверьте правильное сопоставление зон

2. **Проверьте разрешение приоритета**
   - Используйте адрес, совпадающий с несколькими зонами
   - Проверьте, что зона с наивысшим приоритетом выигрывает
   - Убедитесь, что отображаются ожидаемые методы доставки

3. **Проверьте крайние случаи**
   - Границы почтовых индексов (например, 90999 vs 91000)
   - Границы штатов
   - Международные адреса с похожими почтовыми индексами

4. **Используйте инструмент предварительного просмотра зон** (если доступно)
   - Введите тестовый адрес
   - Посмотрите, какие зоны совпадают
   - Посмотрите разрешение приоритета

## Устранение неполадок

**Проблема 1: Нет доступных методов доставки при оформлении заказа**

**Причины**:
- Адрес покупателя не совпадает ни с одной зоной
- Все методы связаны с зонами, которые не совпадают
- Нет методов без ограничений по зонам

**Решение**:
- Создайте резервную зону (все страны, приоритет 1)
- Или удалите ограничения по зонам хотя бы у одного метода
- Проверьте шаблоны стран/штатов/почтовых индексов зоны

---

**Проблема 2: Неправильное сопоставление зон**

**Причины**:
- Выбрана зона с низким приоритетом, несмотря на то, что есть зона с высоким приоритетом, совпадающая с адресом
- Ошибка синтаксиса шаблона почтового индекса (шаблон молча не работает)
- Несоответствие кода штата (CA vs Калифорния)

**Решение**:
- Проверьте значения приоритета (чем выше число, тем выше приоритет)
- Тестируйте шаблоны почтовых индексов с помощью проверяющего регулярные выражения
- Используйте двухбуквенные коды штатов (CA, а не Калифорния)

---

**Проблема 3: Отображается неожиданный метод доставки**

**Причины**:
- Метод не связан с зонами (доступен везде)
- Совпадают несколько зон, и неожиданная зона имеет более высокий приоритет
- Непреднамеренное перекрытие охвата зон

**Решение**:
- Проверьте связанные зоны метода
- Проверьте приоритет совпадающих зон
- Проверьте сводку охвата зон на наличие перекрытий

---

## Советы

- **Начните с 2 зон** - Внутри страны и международная, расширьте по мере необходимости
- **Используйте приоритет с умом** - Конкретные зоны 100, региональные 50, резервная 1
- **Тщательно тестируйте шаблоны почтовых индексов** - Ошибки регулярных выражений не работают молча, вызывая несоответствие зон
- **Документируйте логику зон** - Добавьте примечания в описание зоны, объясняющие намерение охвата
- **Избегайте излишнего количества зон** - Слишком много зон усложняет настройку; используйте правила доставки для сложных сценариев
