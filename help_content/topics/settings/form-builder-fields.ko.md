---
title: 폼 빌더 필드 및 검증
---

폼 필드는 사용자로부터 하나의 데이터를 수집하는 빌딩 블록입니다. Form Builder는 단순한 텍스트 입력에서부터 고급 평가 척도 및 제품 선택기까지 총 22가지 필드 유형을 제공합니다. 각 필드를 라벨, 검증 규칙, 도움말 텍스트 및 조건부 논리로 구성하여 사용자의 응답에 따라 적응하는 동적인 폼을 생성할 수 있습니다. 필드는 필수 또는 선택 사항으로 설정할 수 있으며, 정규 표현식 패턴으로 검증하고 사용자 정의 CSS 클래스로 스타일을 지정할 수 있습니다.

이 가이드를 통해 사용 가능한 모든 필드 유형을 이해하고, 각 필드를 사용하는 시기와 검증 및 조건부 논리를 어떻게 구성하는지 알아보세요.

## 필드 구성 기초

모든 필드는 다음 공통 설정을 공유합니다:

**식별자**:
- **필드 이름** - 데이터 저장을 위한 기계 이름 (공백 없음, 밑줄 사용: `email_address`)
- **필드 유형** - 입력 행동 및 렌더링을 결정합니다.
- **단계 할당** - 이 필드가 속한 단계 (다단계 폼만 해당)

**표시**:
- **라벨** - 사용자에게 표시되는 질문 또는 프롬프트 (예: "What is your email address?" (당신의 이메일 주소는 무엇인가요?))
- **보조 텍스트** - 입력 필드 내부의 힌트 텍스트 (예: "you@example.com")
- **도움말 텍스트** - 필드 아래에 표시되는 추가 가이드 (예: "우리는 결코 당신의 이메일을 공유하지 않습니다")
- **기본값** - 사전에 채워진 값 (사용자가 변경할 수 있음)

**레이아웃**:
- **너비** - 전체 (100%), 반 (50%), 또는 1/3 (33%)의 폼 너비
- **CSS 클래스** - 사용자 정의 디자인을 위한 추가 스타일링 클래스
- **순서** - 단계 내 위치 (드래그하여 재정렬)

**검증**:
- **필수** - 필수 상태 전환 (라벨 옆에 빨간 별표 표시)
- **최소/최대 길이** - 문자 수 제한 (텍스트 필드)
- **최소/최대 값** - 숫자 제한 (숫자 필드)
- **검증 패턴** - 복잡한 검증을 위한 사용자 정의 정규식
- **에러 메시지** - 검증 실패 시 표시되는 사용자 정의 텍스트

## 텍스트 입력 필드

**단일 줄 텍스트** (`text`):
- 짧은 응답을 위한 기본 텍스트 입력
- 검증: 최소/최대 길이, 정규식 패턴
- 사용 사례: 이름, 주소, 제품 코드, 짧은 답변
- 예: "Full Name", "Street Address", "Company Name"

**다중 줄 텍스트** (`textarea`):
- 더 긴 내용을 위한 확장 가능한 텍스트 영역 (3-10 줄)
- 검증: 최소/최대 길이
- 사용 사례: 댓글, 피드백, 상세 설명, 메시지
- 예: "Tell us about your experience", "Additional notes"

**이메일 주소** (`email`):
- 이메일 전용 검증 (이름과 도메인 필요)
- 모바일 키보드는 @ 키를 두드러지게 표시
- 사용 사례: 연락처 이메일, 뉴스레터 가입, 계정 생성
- 예: "Email Address", "Work Email"

**전화 번호** (`phone`):
- 전화 번호를 자동으로 형식화
- 모바일 키보드는 숫자 레이아웃을 표시
- 검증: 구성 가능한 패턴 (국제 형식 지원)
- 사용 사례: 연락처 전화, 응급 연락처, 예약 일정
- 예: "Phone Number", "Mobile", "Contact Number"

**숫자** (`number`):
- 증가/감소 조절기와 함께 숫자 입력
- 검증: 최소/최대 값, 단계 증가
- 응답에서 숫자(문자열이 아님)를 반환
- 사용 사례: 수량, 나이, 경력 연수, 예산 금액
- 예: "How many employees?", "Your age", "Years in business"

**URL** (`url`):
- URL 검증 (http:// 또는 https:// 필요)
- 모바일 키보드는 .com 키를 표시
- 사용 사례: 웹사이트, LinkedIn 프로필, 포트폴리오 링크
- 예: "Company Website", "Portfolio URL"

## 선택 필드

**드롭다운 선택** (`select`):
- 드롭다운 메뉴에서 단일 옵션 선택
- 구성: {value, label} 옵션 배열
- 기본 선택 지원
- 사용 사례: 카테고리, 주/국가, 상태 선택
- 예: "Select your country", "Department", "How did you hear about us?"
- 최적 사용: 5개 이상의 옵션 (적은 옵션은 라디오 대신 사용)

**라디오 버튼** (`radio`):
- 표시된 모든 옵션 중 단일 선택
- 구성: {value, label} 옵션 배열
- 2-4개 옵션에 대해 드롭다운보다 UX가 더 좋음
- 사용 사례: 예/아니요 질문, 성별, 선택지가 적은 선호도
- 예: "Would you recommend us?", "Preferred contact method"

**체크박스** (`checkbox`):
- 단일 토글 체크박스 (켜기/끄기)
- 응답에서 true/false를 반환
- 사용 사례: 약관 동의, 합의, 단일 선호도
- 예: "I agree to terms and conditions", "Subscribe to newsletter"

**체크박스 그룹** (`checkbox_group`):
- 옵션 중 여러 선택 (사용자가 0, 1, 또는 여러 개 선택 가능)
- 구성: {value, label} 옵션 배열
- 응답에서 선택된 값 배열을 반환
- 사용 사례: 다중 선택 선호도, 관심사, 필요한 기능
- 예: "Which topics interest you?", "Select all that apply"

## 평가 필드

**별점 평가** (`rating_stars`):
- 시각적 별점 척도 (보통 1-5개 별)
- 구성:
  - `max_stars`: 3-10개 별 (기본값: 5)
  - `allow_half`: true/false로 반별 허용 여부
  - `icon`: fa-star (기본값) 또는 fa-heart
  - `color`: 16진 색상 코드 (기본값: #FFD700 금색)
- 사용 사례: 제품 평가, 서비스 품질, 만족도 점수
- 예: "Rate your experience", "How was our service?"

**리커트 척도** (`rating_likert`):
- 문장 평가 척도: 강하게 동의 → 강하게 불동의
- 구성:
  - `scale_type`: 5_point (1-5) 또는 7_point (1-7)
  - `labels`: 끝점 텍스트 커스터마이징 (왼쪽: "Strongly Disagree", 오른쪽: "Strongly Agree")
- 응답에서 숫자 값(1-5 또는 1-7)을 반환
- 사용 사례: 설문 문장, 동의 척도, 감정 측정
- 예: "The product meets my needs", "Customer service was helpful"

**순수 추천 점수 (NPS)** (`rating_nps`):
- 0-10 척도: "Not at all likely"에서 "Extremely likely"
- 구성:
  - `low_label`: 왼쪽 끝점 텍스트 (기본값: "Not at all likely")
  - `high_label`: 오른쪽 끝점 텍스트 (기본값: "Extremely likely")
- 응답에서 0-10 값 (0-6 = detractors, 7-8 = passives, 9-10 = promoters)
- 사용 사례: NPS 설문, 추천 가능성, 충성도 측정
- 예: "How likely are you to recommend us to a friend?"

## 고급 필드

**파일 업로드** (`file`):
- 단일 또는 다중 파일 업로드
- 구성:
  - `max_size_mb`: 파일당 파일 크기 제한 (기본값: 5MB)
  - `allowed_types`: 허용된 확장자 배열 (예: ["pdf", "doc", "docx", "jpg", "png"])
  - `max_files`: 최대 파일 수 (1개는 단일, 2개 이상은 다중)
- 응답에서 파일 경로(들)을 반환
- 파일은 `/media/form_uploads/{form-slug}/`에 저장됨
- 사용 사례: 이력서 업로드, 문서 제출, 사진 첨부
- 예: "Upload your resume", "Attach supporting documents"

**제품 선택기** (`product_select`):
- 제품 카탈로그에서 다중 선택
- 구성:
  - `category_filters`: 특정 카테고리 제한 (카테고리 ID 배열)
  - `max_selections`: 1개는 단일 제품, 2개 이상은 다중
  - `display_mode`: "list" (기본값) 또는 "grid" (썸네일 포함)
- 응답에서 제품 ID/SKU를 반환
- 사용 사례: 제품 추천, 위시리스트, 피드백 설문, 번들
- 예: "Which products are you interested in?", "Select your favorite items"

**날짜** (`date`):
- 날짜 선택 인터페이스 (달력 팝업)
- 응답에서 ISO 형식 (YYYY-MM-DD)을 반환
- 검증: 최소/최대 날짜
- 사용 사례: 생년월일, 이벤트 날짜, 예약 일정, 마감일
- 예: "Date of Birth", "Preferred Appointment Date"

**시간** (`time`):
- 시간 선택기 (시간과 분)
- 응답에서 ISO 시간 형식 (HH:MM)을 반환
- 사용 사례: 예약 시간, 가용성 창
- 예: "Preferred Time", "Available After"

**날짜 및 시간** (`datetime`):
- 날짜 및 시간 선택기 결합
- 응답에서 전체 ISO 날짜 시간을 반환
- 사용 사례: 이벤트 일정, 예약 예약
- 예: "Event Start Time", "Delivery Window"

## 레이아웃 필드 (입력 없음)

**섹션 헤드** (`heading`):
- 폼 섹션을 구성하기 위한 헤드 텍스트
- 구성: 헤드 레벨 (h2, h3, h4)
- 데이터 수집 없음
- 사용 사례: 긴 폼을 논리적 섹션으로 나누기
- 예: "Personal Information", "Contact Details", "Preferences"

**설명 문단** (`paragraph`):
- 지시사항 또는 정보를 위한 풍부한 텍스트 블록
- 데이터 수집 없음
- 기본 포맷 지원 (볼드, 이탈릭, 링크)
- 사용 사례: 단계 지시사항, 법적 면책 조항, 설명
- 예: 개인정보 보호 정책 알림, GDPR 동의 설명

**구분선** (`divider`):
- 섹션 간 시각적 구분선
- 데이터 수집 없음
- 사용 사례: 섹션 간 시각적 구성

**숨은 필드** (`hidden`):
- 프로그래밍 값이 있는 보이지 않는 필드
- 구성: `default_value` (필수)
- 사용자에게 라벨 또는 도움말 텍스트 표시 없음
- 사용 사례: UTM 파라미터, 추적 데이터, 세션 ID, 추천 코드
- 예: URL 파라미터 값이 있는 숨은 필드

## 필드 검증 규칙

**필수 필드**:
- 필드 설정에서 "필수" 체크박스 토글
- 라벨 옆에 빨간 별표 (*) 표시
- 필수 필드가 비어 있으면 폼 제출 불가
- 사용자 정의 오류: "This field is required" (또는 사용자 정의 메시지)

**최소/최대 길이** (텍스트 필드):
- 최소 문자 수 설정: 너무 짧은 응답 방지
- 최대 문자 수 설정: 과도한 입력 방지
- 예: 메시지 필드는 최소 10자 ("ok" 응답 방지)

**최소/최대 값** (숫자 필드):
- 최소 숫자 값 설정: 음수 나이, 수량 방지
- 최대 숫자 값 설정: 합리적인 범위로 입력 제한
- 예: 나이 필드는 최소 18, 최대 120

**검증 패턴** (정규식):
- 복잡한 검증을 위한 사용자 정의 정규식
- 일반적인 패턴:
  - 우편번호: `^×{5}(-×{4})?$` (미국 형식)
  - 전화: `^(×{3}) ×{3}-×{4}$` (미국 형식)
  - 제품 코드: `^[A-Z]{2}×{4}$` (2개 대문자, 4개 숫자)
- 패턴 사용 시 사용자 정의 오류 메시지 필요

**파일 검증**:
- 최대 파일 크기: 대용량 업로드 방지 (기본값 5MB)
- 허용된 유형: 특정 확장자 흰색리스트 (보안)
- 예: 이력서 필드는 ["pdf", "doc", "docx"], 최대 2MB 허용

## 조건부 논리

사용자 응답에 따라 필드가 나타나거나 사라지는 동적 폼을 생성하세요:

**조건부 규칙 작동 방식**:
1. 사용자가 "소스 필드" (트리거)에 답변
2. 시스템이 규칙을 평가: 연산자 + 비교 값
3. 조건이 참이면 액션 실행 (필드 또는 단계 표시/숨기기/필수로 만들기)
4. 여러 규칙이 캐스케이드 될 수 있음 (규칙 A가 규칙 B를 트리거)

**사용 가능한 연산자**:
- **같음** (`equals`): 정확한 일치 (예: 국가가 "US")
- **같지 않음** (`not_equals`): 값 외의 모든 것
- **포함** (`contains`): 텍스트가 하위 문자열 포함 (대소문자 구분 없음)
- **보다 큼** (`greater_than`): 숫자 비교 (예: 나이 > 18)
- **보다 작음** (`less_than`): 숫자 비교 (예: 평점 < 3)
- **비어 있음** (`is_empty`): 필드에 값 없음
- **비어 있지 않음** (`is_not_empty`): 필드에 값 있음
- **리스트에 포함** (`in_list`): 값이 ["Option1", "Option2"] 중 하나

**사용 가능한 액션**:
- **필드 표시** - 숨겨진 필드 표시
- **필드 숨기기** - 필드 숨기기 (숨기면 값이 지워짐)
- **필드 필수로 만들기** - 필드를 필수로 설정
- **필드 선택 사항으로 만들기** - 필드를 선택 사항으로 설정
- **값 설정** - 필드에 값을 채우기
- **단계 표시** - 숨겨진 단계 표시 (다단계 폼만 해당)
- **단계 숨기기** - 단계 숨기기 (다단계 폼만 해당)
- **단계로 건너뛰기** - 특정 단계로 이동 (다단계 폼만 해당)

**예시 규칙**:
- IF `contact_method` EQUALS "phone" THEN show_field `phone_number`
- IF `rating` LESS_THAN "3" THEN require_field `improvement_feedback`
- IF `country` IN_LIST ["US", "CA"] THEN show_step `shipping_details`
- IF `budget` GREATER_THAN "10000" THEN show_field `enterprise_features`

**조건부 규칙 생성**:
1. 오른쪽 패널의 "조건부 규칙" 탭 클릭
2. "규칙 추가" 클릭
3. 소스 필드 (트리거) 선택
4. 연산자 (비교 방법) 선택
5. 비교 값 (비교 대상) 입력
6. 액션 (실행할 작업) 선택
7. 대상 (영향을 받는 필드 또는 단계) 선택
8. 선택 사항: 우선순위 설정 (높은 우선순위 규칙이 먼저 평가됨)
9. 규칙 저장

**규칙 우선순위**:
- 숫자가 높을수록 먼저 평가됨 (우선순위 100이 우선순위 10보다 먼저)
- 규칙이 충돌하거나 캐스케이드될 때 우선순위 사용
- 예: 규칙 A (우선순위 100)이 필드를 표시하고, 규칙 B (우선순위 50)이 필수로 만든다면 (A가 먼저 실행되고, 그 후 B가 실행됨)

## 일반적인 필드 패턴

**연락처 폼**:
- 전체 이름 (text, 필수)
- 이메일 (email, 필수)
- 전화 (phone)
- 주제 (select: "Sales", "Support", "Partnership")
- 메시지 (textarea, 필수, 최소 10자)

**제품 피드백**:
- 제품 (product_select, 단일 선택)
- 전체 평가 (rating_stars, 5개 별)
- 조건: IF 평가 < 3 THEN require "What can we improve?" (textarea)
- 추천 (rating_nps)

**채용 신청**:
- 단계 1: 개인 정보 (이름, 이메일, 전화)
- 단계 2: 이력서 (파일 업로드, 허용 ["pdf", "doc"], 최대 2MB)
- 단계 3: 가용성 (시작 날짜, 작업 일 선택)
- 조건: IF "years_experience" > 5 THEN show_field "leadership_experience"

## 팁

- **적절한 필드 유형 사용** - 이메일 필드는 이메일에 사용 (text가 아님), 검증 및 더 나은 모바일 키보드 제공
- **라벨을 짧게 유지** - 세부 사항은 도움말 텍스트에, 라벨에는 사용하지 마세요
- **관련 필드 그룹화** - 헤드와 구분선을 사용하여 시각적 구성
- **검증 테스트** - 폼 미리보기 및 유효하지 않은 데이터로 제출 시도
- **파일 업로드 크기 제한** - 최대 5MB로 대용량 파일로 인한 서버 과부하 방지
- **조건부 논리 적게 사용** - 너무 많은 규칙은 사용자를 혼란스키게 함, 폼을 간단하게 유지
- **현실적인 최대 값 설정** - 나이 최대 120, 수량 최대 100 (1000과 같은 오타 방지)
- **패턴 예시 제공** - 정규식 검증을 사용하는 경우, 도움말 텍스트에 예시를 표시
- **명확한 필드를 필수로 설정** - 연락처 폼의 이름 및 이메일은 항상 필수
- **2-4개 옵션은 라디오 사용** - 5개 이상은 드롭다운 (UX 개선)
- **짧은 입력은 반 너비 사용** - 전화 및 우편번호는 반 너비로, 수직 공간 절약
- **제품 선택기는 위시리스트에 사용** - 고객이 추천을 위해 여러 제품을 선택할 수 있도록 함
