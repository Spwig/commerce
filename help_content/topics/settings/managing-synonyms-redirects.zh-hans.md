---
title: 管理同义词和重定向
---

同义词和重定向通过处理等效术语和将特定查询路由到目标页面，使您的搜索更加智能。同义词可将搜索扩展到包含相关术语（例如“laptop”也会找到“notebook”），而重定向则会将类似“sale”的查询直接发送到您的销售页面。本指南将解释如何创建和管理这两种功能，以提高搜索的相关性和客户体验。

使用同义词进行术语等效，使用重定向进行导航快捷方式。

![同义词列表](/static/core/admin/img/help/managing-synonyms-redirects/synonym-list.webp)

## 了解同义词

同义词告诉搜索系统某些术语应被视为等效。当客户搜索一个术语时，系统会自动包括与同义词匹配的结果。

**示例**：创建同义词映射“laptop”→“notebook”，“portable computer”。现在当有人搜索“laptop”时，也会得到包含“notebook”或“portable computer”名称或描述的产品结果。

同义词特别有价值的情况包括：
- 英式英语与美式英语（jumper/sweater, trainers/sneakers）
- 品牌名称与通用术语（tissues/Kleenex）
- 常见拼写错误（accommodate/accomodate）
- 行业术语与普通语言（CPU/processor）

## 创建同义词

导航到 **搜索 > 同义词** 并点击 **+ 添加同义词**。

![添加同义词表单](/static/core/admin/img/help/managing-synonyms-redirects/synonym-form.webp)

**术语** - 触发同义词扩展的原始搜索术语

**同义词** - 等效术语的 JSON 数组，例如 `['sweater', 'pullover', 'jumper']`

**双向** - 默认：已勾选。启用后，同义词关系双向生效：
- 搜索“laptop”会找到“notebook”产品
- 搜索“notebook”会找到“laptop”产品

取消勾选以创建单向映射（见下文）。

**语言** - 可选。将此同义词限制为特定语言的搜索。留空以适用于所有语言。

**引擎** - 可选。将此同义词限制为特定搜索引擎。留空以全局适用。

**启用** - 此同义词当前是否被使用。取消勾选可临时禁用而不删除。

## 双向示例

大多数同义词应为双向 - 真正的等效项，双向生效：

| 术语 | 同义词 | 使用场景 |
|------|----------|----------|
| laptop | notebook, portable computer | 美式/英式英语 + 通用术语 |
| sofa | couch, settee | 地域差异 |
| trainers | sneakers, running shoes | 英国/美国英语 |
| mobile | cell phone, cellular | 国际差异 |

启用双向后，无论客户使用哪个术语，这些术语都会找到相同的产品。

## 单向示例

取消勾选“双向”以创建单向关系：

**常见使用场景**：
- **拼写错误**：术语：“acco
mmodate” → 同义词：`['accommodate']`（单向，因此正确拼写不会找到拼写错误）
- **具体 → 通用**：术语：“MacBook” → 同义词：`['laptop']`（MacBooks 是笔记本电脑，但并非所有笔记本电脑都是 MacBooks）
- **缩写**：术语：“CPU” → 同义词：`['processor']`（CPU 会找到处理器产品，但处理器搜索不应总是包含 CPU）

## 语言特定同义词

使用语言字段创建适合特定地区的同义词：

**示例**：英式英语商店
- 术语：“jumper”，同义词：`['sweater', 'pullover']`，语言：英语（英国）
- 术语：“trainers”，同义词：`['sneakers']`，语言：英语（英国）

**示例**：多语言商店
- 术语：“ordinateur portable”，同义词：`['laptop', 'notebook']`，语言：法语
- 术语：“zapatos”，同义词：`['shoes']`，语言：西班牙语

语言特定同义词仅在客户使用该语言浏览时适用。

## 引擎特定同义词

大多数同义词应全局适用（留空引擎字段）。仅在不同搜索上下文需要不同术语映射时使用引擎特定同义词：

**示例**：你有独立的 "shop" 和 "blog" 引擎
- 博客同义词：术语："tutorial" → 同义词：`["guide", "how-to"]`，引擎：blog
- 此同义词仅适用于博客搜索，不适用于产品搜索

## 理解重定向

搜索重定向会将特定查询直接发送到指定页面，绕过正常的搜索结果。当你知道客户应该去哪个页面时，请使用重定向。

**示例**：创建 "sale" → "/products/sale/" 的重定向。现在当有人搜索 "sale" 时，他们会跳过搜索结果，直接进入你的销售页面。

重定向非常适合：
- 常见导航快捷方式（"returns" → 返还政策页面）
- 季节性促销（"summer sale" → 夏季系列）
- 流行分类（"laptops" → 笔记本电脑分类页面）
- 政策页面（"shipping" → 运输信息）

## 匹配类型

重定向支持四种匹配类型，控制搜索查询必须多严格地匹配：

**精确匹配** - 不区分大小写的精确匹配。查询必须完全匹配术语（忽略大小写）。
- 术语："sale"
- 匹配："sale", "SALE", "Sale"
- 不匹配："summer sale", "on sale"

**包含** - 查询包含术语的任何位置。
- 术语："sizing"
- 匹配："sizing guide", "help with sizing", "what sizing"
- 不匹配："size chart"（不同词语）

**以...开头** - 查询以术语开头。
- 术语："return"
- 匹配："returns", "return policy", "returning items"
- 不匹配："how to return"（不以术语开头）

**正则表达式** - 使用正则表达式进行模式匹配。**⚠️ 性能警告** - 复杂的正则表达式模式会减慢搜索速度。请谨慎使用。
- 模式：`^(laptop|notebook)s?$`
- 匹配："laptop", "laptops", "notebook", "notebooks"
- 仅在其他匹配类型不起作用时使用

## 创建重定向

导航到 **搜索 > 重定向** 并点击 **+ 添加重定向**。

![添加重定向表单](/static/core/admin/img/help/managing-synonyms-redirects/redirect-form.webp)

**术语** - 要匹配的搜索查询

**匹配类型** - 精确匹配、包含、以...开头或正则表达式（见上文）

**重定向 URL** - 将客户发送到哪里。可以是相对路径（`/products/sale/`）或绝对路径（`https://example.com/page/`）

**重定向类型** - HTTP 状态码：
- **302（临时）**：推荐。浏览器不会缓存，之后可以更改目标
- **301（永久）**：浏览器和搜索引擎会缓存。仅用于永久重定向

**引擎** - 可选。限制到特定搜索引擎

**命中次数** - 每次使用此重定向时自动递增。有助于识别最受欢迎的快捷方式。

**启用** - 启用或禁用此重定向

## 重定向示例

| 术语 | 匹配类型 | URL | 用例 |
|------|-----------|-----|----------|
| sale | 精确匹配 | "/products/sale/" | 将直接搜索 "sale" 引导至销售页面 |
| clearance | 精确匹配 | "/clearance/" | 跳过清仓商品的搜索 |
| sizing | 包含 | "/pages/size-guide/" | 任何关于尺寸的查询都引导至指南 |
| return | 以...开头 | "/pages/returns/" | 与返回相关的查询引导至政策 |

所有使用 302（临时）重定向以获得灵活性。

## 重定向类型：302 与 301

**302（临时）** - 推荐用于大多数重定向
- 浏览器每次都会发出新的请求
- 你可以随时更改目标 URL
- 如果不确定，这是更安全的选择

**301（永久）** - 请谨慎使用
- 浏览器会缓存重定向
- 搜索引擎会更新其索引
- 之后更改起来更困难

**建议**：除非你绝对确定重定向永远不会更改，请使用 302。

## 命中次数分析

命中次数字段每次重定向触发时会自动递增。使用此功能：
- 识别最常用的导航快捷方式
- 找出从未使用过的重定向（考虑删除）
- 发现流行的搜索模式

每月查看命中次数以优化你的重定向策略。

## 寻找同义词机会

**使用零结果查询**：导航到 **搜索 > 搜索分析** 并筛选零结果查询。

这些查询揭示了：
- 客户使用的与产品描述不匹配的术语
- 你尚未考虑的地区差异
- 常见拼写错误

**工作流程**：
1. 每周查看零结果查询
2. 识别模式（相同术语反复出现）
3. 添加同义词，将客户语言映射到你的产品名称
4. 监控零结果是否减少

## 小贴士

- **每周监控零结果查询以获取同义词灵感** - 它们揭示了客户语言与产品描述之间的差距
- **从常见同义词开始，根据数据扩展** - 从明显的地区差异开始，然后根据实际搜索行为添加
- **使用双向同义词以获得真正的等价关系** - 大多数同义词应该双向工作（笔记本电脑 ↔ 笔记本）
- **避免复杂的正则表达式模式** - 正则表达式匹配比其他匹配类型更慢；仅在必要时使用
- **默认使用302重定向（临时）** - 给你灵活性，以后可以更改目标
- **使用真实查询测试同义词** - 搜索同义词以验证它们是否返回预期结果
- **多语言商店使用语言特定的同义词** - 为每种支持的语言创建适合该地区的术语映射
