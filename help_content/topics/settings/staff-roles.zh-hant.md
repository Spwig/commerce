---
title: 員工角色與權限
---

員工角色讓您可以精確控制每位團隊成員在管理面板和POS終端機中可以看到和可以做的事情。定義具有特定權限的角色，然後將其分配給員工。一個用戶可以擁有多個角色，其有效的權限是所有分配角色的組合。

![員工角色](/static/core/admin/img/help/staff-roles/role-list.webp)

## 如何運作

1. 您創建**角色**，用以定義一組權限（例如，"訂單管理員"，"收銀員"）
2. 每個角色控制兩種類型的訪問權限：**管理面板權限**和**POS權限**
3. 您**分配角色**時，從員工的個人資料頁面進行
4. 員工的有效權限是其**所有角色**的組合——如果任何角色授予訪問權限，用戶就擁有該權限
5. 權限會**緩存**以提高性能，並在角色更改時自動刷新

## 風險預設角色

Spwig包含7個內建角色，涵蓋了大多數常見的團隊結構。這些角色不能刪除，但您可以創建自定義角色以滿足更特定的需求。

| 角色 | 訪問 | 描述 |
|------|--------|-------------|
| **商店所有者** | 管理 + POS | 完全訪問所有內容。主要商店管理員使用。 |
| **商店經理** | 管理 + POS | 日常運營——對產品、訂單、客戶、營銷和搜索有完全訪問權限。設計、電子郵件、付款和設置僅限查看。 |
| **內容編輯員** | 管理 | 管理頁面、博客文章、設計和媒體。產品僅限查看。 |
| **訂單管理員** | 管理 | 處理訂單、運輸、退貨和客戶服務。產品僅限查看。 |
| **營銷管理員** | 管理 | 管理促銷、優惠券、聯營、忠誠度和推薦計劃。產品、客戶和媒體僅限查看。 |
| **收銀員** | 僅限POS | 前線POS員工。可以處理銷售和查看禮品卡餘額。不能處理折扣、退款或現金管理。 |
| **高級收銀員** | 僅限POS | 有經驗的POS員工。可以處理退款、應用折扣（最多25%），管理現金和關閉班次。 |

## 創建自定義角色

導航至 **設置 > 員工角色** 並點擊 **添加角色**。

### 一般設置

| 設置 | 描述 |
|---------|-------------|
| **顯示名稱** | 在管理面板中顯示的角色名稱（例如，"倉庫員工"） |
| **描述** | 這個角色的簡要說明 |
| **排序順序** | 控制角色列表中的顯示順序 |
| **圖標** | 從20個圖標中選擇，以視覺方式識別角色 |
| **徽章顏色** | 用於角色徽章的顏色（藍色、綠色、橙色、紅色、青色、灰色） |
| **管理面板** | 切換此角色是否授予管理後台訪問權限 |
| **POS終端機** | 切換此角色是否授予POS終端機訪問權限 |

### 管理權限分類

管理權限選項卡將所有平台功能分為13個分類。對於每個分類，您可以設置三種訪問等級之一：

- **無** — 無法訪問此區域（菜單項被隱藏）
- **查看** — 只讀訪問（可以查看數據但不能更改）
- **完整** — 完整訪問（可以查看、創建、編輯和刪除）

![權限分類](/static/core/admin/img/help/staff-roles/permission-categories.webp)

| 分類 | 它控制什麼 |
|----------|-----------------|
| **產品目錄** | 產品、分類、品牌、屬性、庫存、倉庫、數字資產 |
| **訂單與履行** | 訂單、退款、退貨、運送、運輸配置 |
| **客戶** | 客戶個人資料、分段、分析 |
| **內容與頁面** | 頁面、博客文章、公告、表單 |
| **設計與主題** | 主題、首頁/頁腳模板、菜單、設計令牌、自定義CSS |
| **營銷與促銷** | 促銷、優惠券、聯營、忠誠度、推薦、產品供應 |
| **媒體庫** | 圖片、視頻、文件夾、標籤 |
| **電子郵件系統** | 電子郵件賬戶、模板、傳遞隊列 |
| **付款與賬單** | 支付提供商、交易、webhook、訂閱、匯率 |
| **搜索** | 搜索設置、同義詞、重定向、分析 |
| **商店設置** | 網站設置、地理定位、國家映射、業務規則 |
| **POS管理** | POS終端機、班次、現金流動、收據模板 |
| **用戶與角色** | 員工用戶賬戶、角色、API令牌 |

當用戶擁有多個角色時，**最高**的訪問等級會生效。例如，如果角色A授予產品的"查看"，而角色B授予"完整"，用戶將獲得"完整"訪問權限。

### POS權限標誌

如果角色授予POS訪問權限，POS權限選項卡讓您可以精確調整POS操作員可以執行的操作。這些與管理權限是分離的，並在POS終端機中進行檢查。

![POS權限](/static/core/admin/img/help/staff-roles/pos-permissions.webp)

| 群組 | 權限 | 描述 |
|-------|-----------|-------------|
| **一般** | POS訪問 | 可以在所有POS系統中使用 |
| **銷售與折扣** | 手動折扣 | 可以應用手動的項目級或購物車級折扣 |
| | 最大折扣% | 允許的最高折扣百分比（0–100） |
| | 價格覆蓋 | 可以在收銀機上覆蓋產品價格 |
| **退款與作廢** | 處理退款 | 可以處理POS訂單的退款 |
| | 作廢訂單 | 可以從當前班次中作廢POS訂單 |
| **禮品卡** | 發行禮品卡 | 可以在收銀機上發行新禮品卡 |
| | 查看禮品卡餘額 | 可以查看禮品卡餘額 |
| **現金管理** | 現金管理 | 可以執行現金收入和現金支出操作 |
| | 開啟現金抽屜 | 可以在沒有銷售的情況下打開現金抽屜 |
| | 關閉班次 | 可以關閉班次並執行現金對賬 |
| **報告** | 查看POS報告 | 可以查看班次報告和銷售摘要 |
| **庫存** | 庫存調整 | 可以調整庫存水平（接收、損壞、重新計算、退回） |

對於布爾權限，如果**任何**用戶的角色啟用了它，用戶就擁有該權限。對於最大折扣%，所有角色中**最高**的值適用。

## 管理員工

導航至 **設置 > 員工管理** 以查看和管理您的團隊。

### 員工列表

員工列表顯示所有具有員工訪問權限的用戶。對於每個成員，您可以看到：
- **姓名和電子郵件**
- **分配的角色**（顯示為彩色徽章）
- **訪問類型** — 管理員專用、POS專用或兩者皆有
- **2FA狀態** — 是否啟用了雙因素驗證
- **啟用/停用**狀態

使用過濾器按角色、訪問類型或2FA狀態篩選。

### 為員工分配角色

1. 點擊員工以打開其個人資料
2. 在**角色**部分，您會看到每個可用角色的卡片
3. 點擊任何角色卡片上的切換開關以分配或移除它
4. 更改立即生效——不需要保存按鈕
5. 下方的**有效權限**摘要顯示所有分配角色的組合結果

### 添加新員工

1. 導航至 **設置 > 員工管理** 並點擊 **添加員工**
2. 輸入用戶的電子郵件、名字和姓氏
3. 設置臨時密碼
4. 分配一個或多個角色
5. 現在用戶可以使用其角色提供的訪問權限登錄

## 複製角色

要根據現有角色創建新角色：

1. 打開您要複製的角色
2. 在頁面底部點擊 **複製角色**
3. 將創建一個具有相同所有權限的新角色
4. 將其重命名並根據需要調整權限
5. 保存新角色

當您在需要一個與現有角色類似但有微小差異的角色時，這很有用——例如，基於"商店經理"的"高級經理"，但權限較少。

## 權限如何應用

### 管理面板

- **菜單可見性** — 對於用戶有"無"訪問權限的分類，側邊欄部分將被隱藏
- **頁面訪問** — 尝試訪問受限制的頁面會顯示權限錯誤
- **操作限制** — 具有"查看"訪問權限時，編輯和刪除按鈕將被隱藏，並阻止保存操作
- **超級用戶繞過** — 超級用戶帳戶始終具有完整訪問權限，與角色分配無關

### POS終端機

- **登錄閘門** — 只有至少有一個角色啟用了"POS終端機"的用戶才能登錄到POS
- **功能開關** — POS按鈕和操作（退款、折扣、作廢等）根據用戶合併的POS權限顯示或隱藏
- **折扣上限** — 最大折扣%設置了POS操作員可以應用的折扣的硬性上限
- **API強制執行** — 所有POS權限都在API層進行服務器端檢查，而不僅僅是在UI中

## 小技巧

- **從預設角色開始** — 7個內建角色涵蓋了大多數團隊結構。只有在需要更特定的訪問控制時才創建自定義角色。
- **使用複製功能** — 當您需要一個與現有角色類似的角色時，複製它並進行調整，而不是從頭開始構建。
- **需要時分配多個角色** — 處理訂單和營銷的員工可以同時分配"訂單管理員"和"營銷管理員"角色。權限會自動組合。
- **分離管理員和POS訪問** — 收銀員通常不需要管理員訪問，而辦公人員不需要POS訪問。使用訪問開關來保持整潔。
- **為POS員工設置折扣限制** — 最大折扣%防止收銀員應用過度折扣。設置為0以禁用折扣，或為高級員工設置合理的上限，如10–25%。
- **定期審查角色** — 隨著團隊的成長，審計角色分配以確保員工擁有完成工作所需的最低訪問權限。當人員變更職位時，移除角色。
- **為敏感角色啟用2FA** — 具有付款、設置或用戶管理訪問權限的員工應啟用雙因素驗證以提高安全性。
