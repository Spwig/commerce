---
title: Управление терминалами POS
---

Управление терминалами POS является основой ваших розничных операций. Каждый терминал представляет собой физическое устройство (планшет, компьютер или специализированное оборудование POS), где сотрудники обрабатывают продажи. Настройте терминалы с назначением складов, авторизацией сотрудников, интеграцией оборудования и настройками синхронизации в автономном режиме. Контролируйте состояние терминалов с помощью отслеживания в реальном времени и удаленно разблокируйте терминалы, когда возникают проблемы. Правильное управление терминалами обеспечивает плавные операции в магазине и предотвращает конфликты настроек между локациями.

Перейдите в **POS > Терминалы**, чтобы зарегистрировать новые терминалы, просмотреть статус онлайн/оффлайн и управлять всеми настройками терминалов.

![Список терминалов](/static/core/admin/img/help/managing-pos-terminals/terminal-list.webp)

## Вид списка терминалов

Список терминалов отображает все зарегистрированные терминалы с ключевой информацией о статусе:

**Название терминала** - Описательная метка для терминала (например, "Оплата 1", "Основной кассовый аппарат", "Мобильный терминал")

**UUID** - Уникальный идентификатор, автоматически созданный при создании (используется для внутренней идентификации устройства)

**Склад** - Физическое место, назначенное этому терминалу (определяет доступность запасов и приписку заказов)

**Статус онлайн** - Живой индикатор, показывающий, подключен ли терминал:
- **Зеленая точка** - Онлайн (последний пинг получен в течение последних 5 минут)
- **Красная точка** - Офлайн (нет пинга более 5 минут)
- **Серая точка** - Никогда не подключался (терминал создан, но устройство никогда не подключалось)

**Последний пинг** - Время последнего сигнала от терминала (обновляется каждые 5 минут, когда терминал онлайн)

**Код сопряжения** - 8-символьный алфавитно-цифровой код, используемый для первоначального сопряжения устройства (скрывается после первого использования)

**Назначенные пользователи** - Количество сотрудников, авторизованных для использования этого терминала

## Создание нового терминала

Нажмите **+ Добавить терминал**, чтобы зарегистрировать новое устройство POS:

![Форма добавления терминала](/static/core/admin/img/help/managing-pos-terminals/terminal-add-form.webp)

### Основная настройка

**Название терминала** - Выберите описательное имя, указывающее:
- Физическое место: "Северный вход касса"
- Функция: "Касса возврата"
- Последовательность: "Оплата 1", "Оплата 2", "Оплата 3"

Имена помогают сотрудникам идентифицировать терминалы при назначении смен и устранении неполадок. Используйте последовательные соглашения об именовании во всех локациях.

**Склад** - **Требуется** - Выберите склад, с которого работает этот терминал:
- Определяет, какие запасы доступны для продажи
- Заказы, сделанные на этом терминале, приписываются этому складу
- Проверка запасов проверяет доступность в назначенном складе
- **Невозможно обрабатывать продажи без назначения склада**

Если у вас есть несколько розничных локаций, создайте отдельный склад для каждой локации и назначьте терминалы соответственно.

**Активен** - Переключатель для включения/отключения терминала без удаления настройки:
- Неактивные терминалы не могут быть сопряжены
- Сеансы на неактивных терминалах истекают немедленно
- Используйте для временного отключения украденных или поврежденных терминалов

### Назначение сотрудников

**Назначенные пользователи** - Выберите, какие сотрудники могут получить доступ к этому терминалу:
- Только назначенные пользователи могут войти в терминал
- Пользователи также должны иметь разрешения POS в своей роли сотрудника
- Назначение ноль пользователей фактически блокирует терминал
- Обычный паттерн: назначить всех сотрудников магазина всем терминалам магазина

**Примеры использования**:
- **Обычный магазин**: назначить всех сотрудников всем терминалам (любой кассир может работать на любой кассе)
- **Многоотделенный магазин**: назначить специфических сотрудников отдела на терминалы отдела
- **Многолокационный**: назначить специфических сотрудников локации на терминалы локации
- **Менеджеры**: назначить менеджеров на все терминалы для доступа супервайзера

Пользователи без назначения терминала видят ошибку "Не авторизованы для этого терминала" при попытке входа.

### Настройка оборудования

Поле **Настройка оборудования** является структурой JSON, определяющей периферийные устройства:

**Термопечать**:
```json
{
  "printer": {
    "type": "network",
    "ip": "192.168.1.100",
    "port": 9100,
    "paper_width": 80
  }
}
```

**USB-сканер штрих-кодов**:
```json
{
  "scanner": {
    "type": "usb",
    "vendor_id": "0x05e0",
    "product_id": "0x1200"
  }
}
```

**Кассовый ящик** (подключен к принтеру):
```json
{
  "cash_drawer": {
    "enabled": true,
    "trigger": "printer_pulse"
  }
}
```

**Полный пример**:
```json
{
  "printer": {
    "type": "network",
    "ip": "192.168.1.100",
    "port": 9100,
    "paper_width": 80
  },
  "scanner": {
    "type": "usb",
    "vendor_id": "0x05e0",
    "product_id": "0x1200"
  },
  "cash_drawer": {
    "enabled": true,
    "trigger": "printer_pulse"
  }
}
```

Оставьте поле пустым, если терминал не имеет периферийного оборудования (подходит для мобильных терминалов или планшетов без принтера/сканера).

### Настройки кэша в автономном режиме

Настройте, сколько данных терминал кэширует для автономной работы:

**Количество дней синхронизации заказов** (7-30 дней, по умолчанию: 14):
- Количество дней недавних заказов для локального кэширования
- Более высокие значения = больше исторических данных доступно в автономном режиме
- Более низкие значения = более быстрая синхронизация, меньше используемого хранилища
- **Рекомендация**: 7 дней для терминалов с высоким объемом, 14 дней для обычного использования, 30 дней для операций с аудитом

**Лимит синхронизации заказов** (200-1000 заказов, по умолчанию: 500):
- Максимальное количество заказов для кэширования, независимо от диапазона дат
- Предотвращает излишнее использование хранилища на терминалах с высоким объемом
- **Рекомендация**: 200 для планшетов с ограниченным хранилищем, 500 для стандартных терминалов, 1000 для специализированных устройств POS

**Торговые компромиссы**:
- **Высокие настройки**: лучший доступ к историческим данным в автономном режиме, более медленная первоначальная синхронизация, больше используемого хранилища
- **Низкие настройки**: более быстрая синхронизация, меньше хранилища, ограниченная история в автономном режиме

Терминал загружает самые последние X заказов (в пределах Y дней) на каждом цикле синхронизации. Если терминал обрабатывает 50 заказов в день и sync_days составляет 14, ожидается около 700 заказов в кэше (может достичь ограничения sync_limit).

## Рабочий процесс сопряжения терминала

После создания терминала сопрягите физическое устройство:

1. **Создание кода сопряжения** - Автоматически создается, когда вы сохраняете терминал (8 алфавитно-цифровых символов)

2. **Запишите код** - Отображается в списке терминалов и в детальном представлении (истекает после первого успешного сопряжения)

3. **Перейдите к физическому устройству терминала** - На физическом устройстве (планшет/компьютер), откройте браузер и перейдите по адресу: `https://yourstore.com/pos/`

4. **Введите код сопряжения** - Введите 8-символьный код, когда вас попросят

5. **Терминал загружает конфигурацию** - Устройство получает:
   - Назначение склада
   - Конфигурация оборудования (принтер, сканер, ящик)
   - Настройки кэша в автономном режиме
   - Список назначенных пользователей
   - Первичная синхронизация каталога продуктов

6. **Появляется экран входа** - Терминал отображает экран входа для назначенных пользователей

7. **Сотрудник входит в систему** - Введите учетные данные пользователя, назначенного на этот терминал

8. **Завершается первичная синхронизация** - Терминал загружает:
   - Недавние заказы (в соответствии с sync_days и sync_limit)
   - Полный каталог продуктов для назначенного склада
   - Базу данных клиентов
   - Настройки акций

9. **Готов терминал** - Появляется экран "Готов к продажам" со строкой поиска

10. **Код сопряжения использован** - Код удаляется из админ-панели; сгенерируйте новый код, если требуется повторное сопряжение

**Перегенерация кода сопряжения**: Если вам нужно повторно сопрячьте терминал (сброс устройства, очистка кэша браузера, новое оборудование), используйте административное действие **Перегенерация кода сопряжения**. Это делает старый код недействительным и создает новый.

## Мониторинг состояния терминала

### Система пинга

Терминалы отправляют пинг серверу каждые **5 минут** с сигналом пинга, содержащим:
- UUID терминала
- Текущее время
- Количество онлайн-пользователей
- Время последней синхронизации
- Статус сервис-воркера

**Индикатор онлайн-статуса**:
- **Зеленый** - Получен пинг в течение последних 5 минут (терминал онлайн и работает)
- **Красный** - Нет пинга более 5 минут (терминал оффлайн или отключен)
- **Серый** - Терминал никогда не сопрягался (никогда не получал пинга)

**Сценарии использования**:
- **Ежедневный запуск**: Проверьте, что все терминалы онлайн перед открытием магазина
- **Устранение неполадок**: Определите, какие терминалы сталкиваются с проблемами подключения
- **Аудит**: Проверьте, что терминалы активны в рабочее время

### Время последнего пинга

Отображает точную дату и время последнего пинга. Используйте это для:
- Определения, как долго терминал был оффлайн
- Выявления паттернов (например, терминал оффлайн каждый вечер при закрытии)
- Проверки частоты синхронизации (должно обновляться каждые ~5 минут при онлайн-режиме)

## Функция удаленной разблокировки

Когда терминал становится неактивным или застревает на экране (программная ошибка, проблемы с тайм-аутом сеанса, зависший браузер), используйте административное действие **Удаленная разблокировка**:

**Как это работает**:
1. Выберите проблемный терминал в списке администратора
2. Выберите **Удаленная разблокировка** из выпадающего меню административных действий
3. Подтвердите действие
4. Сервер отправляет сигнал разблокировки через ответ пинга
5. Терминал получает сигнал на следующем цикле пинга (<5 мин)
6. Терминал принудительно выходит из текущего пользователя и возвращается на экран входа

**Когда использовать**:
- Терминал застрял на экране транзакции
- Сотрудник не может выйти (кнопка выхода не отвечает)
- Сеанс выглядит активным, но терминал не отвечает
- Браузер завис, но куки сеанса все еще активны

**Важно**: Удаленная разблокировка не перезагружает устройство или браузер — она только принудительно выходит из системы и очищает сеанс. Если терминал полностью завис, сотрудники могут вручную перезагрузить браузер или устройство.

## Редактирование настройки терминала

Нажмите терминал в списке, чтобы отредактировать его настройку:

![Форма редактирования терминала](/static/core/admin/img/help/managing-pos-terminals/terminal-edit-form.webp)

**Безопасно изменять, когда терминал онлайн**:
- Название терминала
- Назначенные пользователи
- Настройка оборудования (вступает в силу после перезапуска приложения терминала)
- Настройки кэша в автономном режиме (вступают в силу на следующей синхронизации)

**Требуется повторное сопряжение**:
- Назначение склада (изменение склада требует повторного сопряжения для синхронизации нового запаса)

**Нельзя изменить**:
- UUID (неизменный идентификатор)

Изменения большинства настроек применяются на следующем цикле пинга/синхронизации. Изменения настройки оборудования требуют, чтобы сотрудники закрыли и снова открыли приложение POS (или обновили браузер).

## Устранение распространенных проблем

**Терминал показывает "Не авторизован" при входе**:
- Проверьте, что пользователь находится в списке **Назначенные пользователи** для этого терминала
- Проверьте, что у пользователя есть разрешения POS в **Сотрудники и разрешения > Роли**
- Проверьте, что терминал помечен как **Активен**

**Терминал не сопрягается (неверный код)**:
- Коды сопряжения истекают после первого использования — перегенерируйте, если это необходимо
- Коды чувствительны к регистру — проверьте заглавные буквы
- Проверьте, что терминал помечен как **Активен**

**Терминал показывает оффлайн (красная точка)**:
- Проверьте, есть ли у устройства интернет-подключение
- Проверьте, действительно ли терминал работает (браузер открыт на URL /pos/)
- Убедитесь, что брандмауэр не блокирует запросы пинга
- Подождите 5 минут до следующего цикла пинга

**Терминал медленно синхронизируется**:
- Уменьшите **Количество дней синхронизации заказов** с 30 до 7
- Уменьшите **Лимит синхронизации заказов** с 1000 до 200
- Проверьте скорость сети на месте терминала
- Убедитесь, что сервер не перегружен

**Принтер не работает**:
- Проверьте IP-адрес и порт принтера в **Настройка оборудования**
- Проверьте подключение принтера с устройства терминала (ping IP-адрес)
- Убедитесь, что принтер совместим с ESC/POS
- Проверьте, что принтер включен и онлайн

## Советы

- **Соглашение об именовании имеет значение** - Используйте последовательные имена (место + номер), чтобы упростить управление в масштабе
- **Всегда назначайте склад перед сопряжением** - Терминалы не могут обрабатывать продажи без назначения склада
- **Проверьте настройку оборудования перед развертыванием** - Распечатайте тестовый чек, чтобы проверить интеграцию принтера/ящика
- **Ежедневно отслеживайте пинг** - Настройте регулярную проверку, чтобы убедиться, что все терминалы онлайн при открытии магазина
- **Снижайте лимиты синхронизации для мобильных терминалов** - Планшеты и телефоны лучше работают с sync_days: 7, sync_limit: 200
- **Используйте удаленную разблокировку с осторожностью** - Принудительный выход прерывает активные транзакции; убедитесь, что терминал действительно застрял
- **Запишите коды сопряжения** - Запишите код перед развертыванием терминала на торговой площадке (на случай, если настройка займет больше времени, чем ожидалось)
- **Назначьте менеджера всем терминалам** - Обеспечивает доступ супервайзеров к любому кассовому аппарату для аннулирования, возврата и устранения неполадок
