---
title: 運輸區域
---

運輸區域用於定義地理區域以進行目標定價—將國家、州或郵政編碼分組為區域，然後將運輸方式與特定區域連結，以實現精確的定價控制。當地址符合多個區域時，區域會根據優先順序進行匹配（優先順序最高的區域獲勝）。此系統可實現複雜的定價策略：對偏遠地區收取更高費用、在國內提供免運費，或為特定區域提供折扣價格。

當您在不同地理區域需要不同的運輸費用時，請使用區域，從簡單的國內對國際分區到複雜的多區域階梯定價。

## 理解運輸區域

**區域是什麼**：以國家、州/省和郵政編碼模式為基礎定義的命名地理區域。

**區域如何運作**：
1. 顧客在結帳時輸入運輸地址
2. 系統評估所有活動區域
3. 與顧客地址匹配的區域是候選區域
4. 如果多個區域匹配，優先順序最高的區域獲勝
5. 與獲勝區域連結的運輸方式會顯示出來
6. 未與任何區域連結（或與匹配區域連結）的方式也會顯示

**區域組成部分**：
- **名稱**：區域識別碼（例如，"Domestic"、"EU"、"Remote Areas"）
- **國家**：包含的國家代碼清單（留空 = 所有國家）
- **州/省**：每個國家的州限制（可選）
- **郵政編碼模式**：用於郵遞編碼匹配的正則表達式模式（可選）
- **優先順序**：數字越高，當多個區域匹配時優先順序越高

## 區域匹配邏輯

區域使用 **漸進縮小** 來匹配地址：

### 第1層：國家匹配

**國家清單為空** → 區域匹配所有國家

**提供國家清單** → 地址國家必須在清單中

範例：
```
區域："Domestic"
國家：["US"]
→ 匹配：任何美國地址
→ 不匹配：加拿大、英國等
```

### 第2層：州/省匹配

**未定義州** → 區域匹配允許國家的所有州

**為特定國家定義州** → 地址州必須匹配

範例：
```
區域："West Coast"
國家：["US"]
州：{"US": ["CA", "OR", "WA"]}
→ 匹配：加州、俄勒岡州、華盛頓州地址
→ 不匹配：紐約州、德州等
```

### 第3層：郵政編碼匹配

**未定義模式** → 區域匹配允許國家/州的所有郵政編碼

**定義模式** → 地址郵政編碼必須匹配至少一個模式

範例：
```
區域："Los Angeles Metro"
國家：["US"]
州：{"US": ["CA"]}
郵政模式：["^90[0-9]{3}$", "^91[0-9]{3}$"]
→ 匹配：90001、91210、90245
→ 不匹配：94102（舊金山）
```

**正則表達式模式範例**：
- `^90[0-9]{3}$` - 洛杉磯地區（90000-90999）
- `^[A-Z][0-9][A-Z] [0-9][A-Z][0-9]$` - 加拿大郵政編碼格式（K1A 0B1）
- `^SW[0-9]{1,2}` - 以 SW 開頭的英國倫敦郵編

## 基於優先順序的區域選擇

當多個區域匹配地址時，**優先順序** 確定哪個區域適用：

**優先順序如何運作**：
- 數字越高，優先順序越高
- 如果地址匹配優先順序為 100 和 50 的區域，優先順序 100 獲勝
- 只有獲勝區域的運輸方式可用

**使用案例**：

**場景1：特定區域覆蓋一般區域**
```
區域A："Remote Alaska"
  國家：["US"]
  州：{"US": ["AK"]}
  優先順序：100

區域B："Domestic USA"
  國家：["US"]
  優先順序：50

地址：阿拉斯加安克雷奇
→ 匹配兩個區域
→ 優先順序100獲勝
→ "Remote Alaska"區域適用（運費較高）
```

**場景2：郵政編碼覆蓋州**
```
區域A："Manhattan Premium"
  國家：["US"]
  州：{"US": ["NY"]}
  郵政模式：["^100[0-2][0-9]$"]
  優先順序：100

區域B："New York State"
  國家：["US"]
  州：{"US": ["NY"]}
  優先順序：50

地址：紐約，NY 10001
→ 匹配兩個區域
→ 優先順序100獲勝
→ "Manhattan Premium"適用（高級配送服務）
```

## 建立運輸區域

**逐步工作流程**：

1. **導航到區域**
   - 前往設定 > 運輸 > 運輸區域
   - 點擊"新增運輸區域"

2. **基本配置**
   - **名稱**：描述性識別碼（例如，"European Union"、"West Coast"、"Remote Areas"）
   - **優先順序**：設置相對重要性（100為特定，50為一般，1為預設）
   - **啟用**：切換以啟用/停用

3. **定義地理範圍**

   **選項A：所有國家**（留空國家清單）
   - 區域匹配全球所有地址
   - 用於預設/預設區域

   **選項B：特定國家**
   - 點擊"新增國家"
   - 從下拉選單中選擇國家（US、CA、UK 等）
   - 重複選擇所有包含的國家

   **選項C：特定州/省**
   - 在新增國家後，為每個國家點擊"新增州"
   - 從下拉選單中選擇州
   - 範例：US → CA、OR、WA（西海岸）

   **選項D：郵政編碼模式**（高階）
   - 輸入正則表達式模式（每行一個）
   - 使用範例郵政編碼測試模式
   - 點擊"驗證模式"以檢查語法

4. **連結到運輸方式**
   - 當編輯方式時可以連結（不在區域配置中）
   - 或連結區域到現有方式：編輯方式 → 運輸區域 → 選擇區域

5. **設置顯示優先順序**
   - 當多個匹配時，較高優先順序的區域會覆蓋較低優先順序的區域
   - 建議：特定區域（100）、區域（50）、預設區域（1）

6. **啟用區域**
   - 切換"啟用" = 是
   - 儲存

## 常見區域設定

### 設定1：國內對國際

**目標**：為國內與所有其他國家設定不同費用。

```
區域1："Domestic"
  國家：[您的國家代碼]
  優先順序：50

區域2："International"
  國家：[留空或選擇所有其他國家]
  優先順序：1
```

**運輸方式**：
- "Domestic Standard" → 連結到 Domestic 區域
- "International Shipping" → 連結到 International 區域

---

### 設定2：多區域國際

**目標**：為歐盟、北美、亞洲、世界其他地區設定不同費用。

```
區域1："European Union"
  國家：[AT, BE, BG, CY, CZ, DE, DK, EE, ES, FI, FR, GR, HR, HU, IE, IT, LT, LU, LV, MT, NL, PL, PT, RO, SE, SI, SK]
  優先順序：100

區域2："North America"
  國家：[US, CA, MX]
  優先順序：100

區域3："Asia Pacific"
  國家：[AU, CN, HK, IN, JP, KR, NZ, SG, TH, TW]
  優先順序：100

區域4："Rest of World"
  國家：[留空]
  優先順序：1
```

**運輸方式**：
- "EU Shipping" → EU 區域
- "North America Shipping" → North America 區域
- "Asia Pacific Shipping" → Asia Pacific 區域
- "International Standard" → Rest of World 區域

---

### 設定3：偏遠地區附加費用

**目標**：為國內區域內的偏遠郵政編碼增加附加費用。

```
區域1："Remote Domestic"
  國家：[US]
  郵政模式：["^99[0-9]{3}$", "^96[7-9][0-9]{2}$"]  # 阿拉斯加、夏威夷
  優先順序：100

區域2："Standard Domestic"
  國家：[US]
  優先順序：50
```

**運輸方式**：
- "Remote Shipping" → Remote Domestic 區域（費用較高）
- "Standard Shipping" → Standard Domestic 區域

---

### 設定4：州特定區域

**目標**：為每個美國地區設定不同費用。

```
區域1："West Coast"
  國家：[US]
  州：{"US": ["CA", "OR", "WA"]}
  優先順序：100

區域2："East Coast"
  國家：[US]
  州：{"US": ["NY", "NJ", "CT", "MA", "PA"]}
  優先順序：100

區域3："Midwest"
  國家：[US]
  州：{"US": ["IL", "IN", "OH", "MI", "WI"]}
  優先順序：100

區域4："South"
  國家：[US]
  州：{"US": ["TX", "FL", "GA", "NC", "SC"]}
  優先順序：100

區域5："Other US States"
  國家：[US]
  優先順序：50
```

## 郵政編碼模式範例

郵政編碼使用 **正則表達式**（正則表達式）進行模式匹配：

### 美國（ZIP 編碼）

**格式**：5位數字（例如，90210）

```
加州（90000-96199）：  ^9[0-6][0-9]{3}$
紐約州（10000-14999）：    ^1[0-4][0-9]{3}$
德州（75000-79999, 88500-88599）：  ^(7[5-9]|885)[0-9]{2}$
阿拉斯加（99500-99999）：      ^99[5-9][0-9]{2}$
```

### 加拿大（郵政編碼）

**格式**：A1A 1A1（字母-數字-字母 空格 數字-字母-數字）

```
所有加拿大郵政編碼：  ^[A-Z][0-9][A-Z] [0-9][A-Z][0-9]$ 
安大略省（K, L, M, N, P）：    ^[KLMNP][0-9][A-Z] [0-9][A-Z][0-9]$ 
魁北克省（G, H, J）：           ^[GHJ][0-9][A-Z] [0-9][A-Z][0-9]$ 
```

### 英國（郵編）

**格式**：AA1A 1AA 或 A1A 1AA

```
倫敦（E, EC, N, NW, SE, SW, W, WC）：  ^(E|EC|N|NW|SE|SW|W|WC)[0-9]{1,2}
曼徹斯特（M）：                        ^M[0-9]{1,2}
伯明翰（B）：                        ^B[0-9]{1,2}
```

### 澳大利亞（郵編）

**格式**：4位數字（例如，2000）

```
新南威爾士州（1000-2999）：  ^[12][0-9]{3}$
維多利亞州（3000-3999, 8000-8999）：  ^[38][0-9]{3}$
昆士蘭州（4000-4999, 9000-9999）：  ^[49][0-9]{3}$
```

### 模式測試

**在儲存模式之前**，使用已知的郵政編碼進行測試：

1. 輸入模式：`^90[0-9]{3}$`
2. 測試輸入："90210" → 應該匹配
3. 測試輸入："10001" → 應該不匹配
4. 測試輸入："9021" → 應該不匹配（只有4位數字）

使用線上正則表達式測試工具（regex101.com）來驗證複雜模式。

## 區域覆蓋總覽

區域在管理員清單視圖中顯示 **覆蓋總覽**，顯示包含內容：

**範例**：
- "All countries" → 無國家限制
- "US, CA, MX" → 3個國家
- "US (CA, OR, WA)" → 美國包含3個州
- "US (90xxx-91xxx)" → 美國包含郵政編碼模式

**使用總覽來**：
- 快速驗證區域覆蓋，而無需打開
- 發現覆蓋重疊或缺口
- 一目了然地審核區域配置

## 將區域連結到運輸方式

區域和方式有 **多對多關係**：

**從方式側**（建議）：
1. 編輯運輸方式
2. 滾動到"運輸區域"部分
3. 選擇適用的區域（多選）
4. 儲存方式

**從區域側**：
- 區域不直接連結到方式
- 連結始終從方式配置完成

**方式-區域行為**：

**未連結任何區域** → 方式可用於所有地址

**已連結區域** → 方式僅在顧客地址匹配至少一個連結區域時可用

**範例**：
```
方式："Domestic Standard"
連結區域：["Domestic USA"]
→ 僅顯示給美國地址

方式："International Express"
連結區域：["EU", "Asia Pacific", "Rest of World"]
→ 顯示給所有非美國地址
```

## 測試區域匹配

在上線前測試區域配置：

1. **建立測試訂單**
   - 使用不同區域的地址
   - 驗證正確的區域匹配

2. **檢查優先順序解決方案**
   - 使用匹配多個區域的地址
   - 驗證優先順序最高的區域獲勝
   - 確認預期的運輸方式出現

3. **測試邊界案例**
   - 邊界郵政編碼（例如，90999 vs 91000）
   - 州界
   - 國際地址與類似的郵政編碼

4. **使用區域預覽工具**（如果可用）
   - 輸入測試地址
   - 查看哪些區域匹配
   - 查看優先順序解決方案

## 故障排除

**問題1：結帳時沒有可用的運輸方式**

**原因**：
- 顧客地址不匹配任何區域
- 所有方式都連結到不匹配的區域
- 没有任何方式沒有區域限制

**解決方案**：
- 建立預設區域（所有國家，優先順序1）
- 或從至少一個方式中移除區域限制
- 驗證區域國家/州/郵政編碼模式

---

**問題2：錯誤的區域匹配**

**原因**：
- 選擇了優先順序較低的區域，儘管有更高優先順序的區域匹配
- 郵政編碼模式語法錯誤（模式靜默失敗）
- 州代碼不匹配（CA vs 加州）

**解決方案**：
- 驗證優先順序值（數字越高，優先順序越高）
- 使用正則表達式驗證器測試郵政編碼模式
- 使用2個字母的州代碼（CA，而不是加州，NY 而不是紐約）

---

**問題3：顯示了意外的運輸方式**

**原因**：
- 方式沒有連結任何區域（可用於所有地址）
- 多個區域匹配，意外區域有更高的優先順序
- 區域覆蓋意外重疊

**解決方案**：
- 檢查方式的連結區域
- 檢查匹配區域的優先順序
- 審核區域覆蓋總覽以查找重疊

## 小技巧

- **從2個區域開始** - 國內和國際，根據需要擴展
- **明智地使用優先順序** - 特定區域100，區域50，預設1
- **徹底測試郵政模式** - 正則表達式錯誤會靜默失敗，導致區域不匹配
- **記錄區域邏輯** - 在區域描述中添加筆記，說明覆蓋意圖
- **避免過多區域** - 過多區域會使配置複雜；使用運輸規則處理複雜場景
- **使用州代碼，而不是名稱** - "CA"而不是"California"，"NY"而不是"New York"
- **建立預設區域** - 所有國家，優先順序1，確保至少有一個運輸選項始終可用
- **監控區域表現** - 如果許多顧客看到"無運輸可用"，審核區域覆蓋
- **更新區域以包含新區域** - 當新成員加入時，將國家添加到歐盟區域
- **使用描述性名稱** - "EU (Excluding UK)"比"Zone 3"更好
- **使用真實地址進行測試** - 測試時使用顧客的實際地址，而不是虛構的地址
