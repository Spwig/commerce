Система POS Spwig превращает ваш магазин в полное решение для розничной торговли с современными терминалами продаж. Разверните неограниченное количество терминалов в неограниченном количестве местоположений за ежегодную фиксированную плату в размере €499. Каждый терминал представляет собой прогрессивное веб-приложение (PWA), которое работает автономно, синхронизируется автоматически и интегрируется с вашим ассортиментом, данными о клиентах и обработкой платежей. Управляйте всем из панели администратора — настройка терминалов, сверка смен, настройка чеков и интеграция с оборудованием.

Используйте систему POS, когда у вас есть физические розничные точки продаж, временные магазины, выставки, или любая среда, где клиенты совершают покупки лично, а не онлайн.

## Что такое Spwig POS?

Spwig POS — это полностью интегрированная система продаж, предназначенная для продавцов, которые продают как онлайн, так и в физических точках продаж. В отличие от сторонних систем POS, требующих сложной интеграции, Spwig POS встроен напрямую в вашу платформу, обеспечивая идеальную синхронизацию данных по всем каналам продаж.

**Основные характеристики**:
- **Неограниченное количество терминалов** - Развертывайте столько терминалов, сколько необходимо, без дополнительных затрат
- **Архитектура с приоритетом автономной работы** - Продолжает обрабатывать продажи даже при потере соединения с интернетом
- **Прогрессивное веб-приложение** - Нет необходимости устанавливать в магазине приложений; доступ через браузер на любом устройстве (планшеты, компьютеры, специализированные терминалы)
- **Синхронизация реального запаса** - Бронирование запасов (TTL 15 минут) предотвращает перепродажу по каналам
- **Поддержка разделения оплаты** - Принимайте несколько методов оплаты за одну транзакцию (наличные + карта + подарочная карта)
- **Интеграция с оборудованием** - Термопечати ESC/POS, сканеры штрих-кодов, денежные ящики, дисплеи для клиентов
- **Управление сменами** - Сверка наличных с начальными и конечными подсчетами и отслеживанием расхождений
- **Готов к работе с несколькими точками продаж** - Группы магазинов с наследованием настроек для управления франшизами и региональными настройками

## Лицензирование и активация

**Ежегодная фиксированная плата**: €499 в год покрывает неограниченное количество терминалов в неограниченном количестве местоположений. Нет дополнительных платежей за терминал, нет платежей за транзакции, нет скрытых затрат.

**Формат лицензии**: `POS-XXXX-XXXX-XXXX-XXXX` (предоставляется после покупки)

**Активация**: Введите свой ключ лицензии в **Настройки > Лицензирование POS**. Система проверяет лицензию на сервере Spwig и сразу включает все функции POS. Лицензии включают 14-дневный период грейс-времени после истечения срока действия, чтобы позволить обработку оплаты с задержкой.

**Что вы получаете**:
- Неограниченная регистрация терминалов
- Неограниченное назначение сотрудников
- Все функции POS (смены, управление наличными, настройка чеков, дисплеи для клиентов)
- Интеграции с поставщиками платежей (Stripe Terminal и расширяемая система поставщиков)
- Поддержка интеграции с оборудованием
- Обновления и исправления ошибок в течение периода действия лицензии

Нет функций POS без действительной лицензии — интерфейс сопоставления терминалов, управление сменами и страницы администратора POS требуют активации.

## Архитектура системы

**Frontend** - Прогрессивное веб-приложение на React 18:
- Приоритет автономной работы с кэшированием через Service Worker (работает без интернета)
- Система сборки Vite для быстрой загрузки
- CSS Modules + токены дизайна (соответствуют теме вашего магазина)
- IndexedDB для локального хранения данных
- 10 поддерживаемых языков (английский, китайский упрощенный/традиционный, французский, немецкий, испанский, португальский, японский, русский, арабский)

**Backend** - Интеграция с бэкендом:
- 13 моделей POS (POSTerminal, POSShift, CashMovement, ReceiptTemplate, PromoSlide и т.д.)
- 43+ конечных точек REST API для операций с терминалами
- Система бронирования запасов с управлением TTL
- Задачи Celery для фоновой синхронизации
- Зашифрованное хранение учетных данных для поставщиков платежей

**Безопасность**:
- Сопоставление терминалов через коды из 8 символов (создаются на стороне сервера, истекают после использования)
- Контроль назначения сотрудников, определяющий, какие пользователи могут получить доступ к каким терминалам
- Возможность удаленного блокировки/разблокировки для экстренных случаев администратора
- Зашифрованные учетные данные поставщиков платежей
- Аутентификация на основе сеанса с поддержкой разблокировки биометрией (зависит от браузера)

## Рабочий процесс начала работы

Следуйте этим 5 шагам, чтобы развернуть ваш первый терминал POS:

**Шаг 1: Активация лицензии POS**
- Перейдите в **Настройки > Лицензирование POS**
- Введите свой ключ лицензии (`POS-XXXX-XXXX-XXXX-XXXX`)
- Проверьте лицензию (требуется подключение к интернету)
- Подтвердите активацию

**Шаг 2: Создание склада**
- Перейдите в **Каталог > Склады**
- Создайте склад, представляющий ваше место розничной продажи
- Настройте адрес и контактную информацию
- Этот склад будет отслеживать физический запас для продаж POS

**Шаг 3: Регистрация терминала**
- Перейдите в **POS > Терминалы**
- Нажмите **+ Добавить терминал**
- Установите имя терминала (например, "Основной кассовый аппарат", "Оплата 1")
- Назначьте склад из шага 2
- Настройте параметры оборудования (печать, сканер, денежный ящик)
- Сохраните, чтобы сгенерировать 8-символьный код сопоставления

**Шаг 4: Назначение сотрудников**
- В настройках терминала прокрутите до **Назначенные пользователи**
- Выберите сотрудников, которым разрешено использовать этот терминал
- Только назначенные пользователи могут войти в терминал
- Пользователям должно быть присвоено соответствующее разрешение POS в их роли сотрудника

**Шаг 5: Сопоставление устройства**
- На вашем устройстве терминала (планшет/компьютер) перейдите по URL `/pos/`
- Введите 8-символьный код сопоставления из шага 3
- Терминал загружает конфигурацию и синхронизирует начальные данные
- Войдите с учетными данными назначенного сотрудника
- Терминал готов к продажам

После сопоставления терминалы автоматически синхронизируются каждые 5 минут (настраивается). Режим автономной работы позволяет продолжать работу, когда интернет недоступен — продажи автоматически синхронизируются, когда соединение восстановлено.

## Основные функции POS

**Обработка продаж**:
- Поиск товаров по названию, SKU или штрих-коду
- Разделение оплаты (несколько методов оплаты за заказ)
- Сохраненные корзины (сохранение незавершенных транзакций)
- Возвраты и аннулирования с отслеживанием причин
- Применение скидок (купоны, подарочные карты, акции)
- Поиск клиентов и использование баллов лояльности

**Управление наличными**:
- Открытие смены с начальным подсчетом наличных
- Закрытие смены с проверкой ожидаемого и фактического баланса
- Движение наличных (дополнение кассы, выдача наличных с указанием причин)
- Автоматический расчет ожидаемого наличного баланса на основе продаж наличными
- Отслеживание расхождений и отчеты

**Интеграция с оборудованием**:
- Термопечати ESC/POS (сетевые или последовательные)
- Сканеры штрих-кодов по USB
- Триггер денежного ящика через импульс принтера
- Дисплеи для клиентов (рекламный карусель при простое)
- Считывающие устройства карт Stripe Terminal (S700, WisePOS E, P400)

**Возможности автономной работы**:
- Service Worker кэширует все активы терминала
- IndexedDB хранит последние заказы (настраивается: 7-30 дней, 200-1000 заказов)
- Бронирование запасов с TTL 15 минут предотвращает перепродажу
- Очередь продаж для синхронизации при восстановлении соединения
- Автоматическое обнаружение повторного подключения

**Управление терминалами** (`/admin/pos_app/posterminal/`)

- Отслеживание движений наличных и расхождений

## Развертывание для нескольких местоположений

Для продавцов с несколькими розничными точками продаж, Spwig POS поддерживает иерархическое наследование настроек:

**Иерархия настроек** (от высшего приоритета к низшему):
1. Настройки, специфичные для терминала (переопределяют все)
2. Настройки, специфичные для магазина (переопределяют группу и сайт)
3. Настройки группы (переопределяют настройки сайта)
4. Настройки сайта по умолчанию (обратный ход для всех)

Настройте общие параметры на уровне группы (например, региональная валюта, язык) и переопределяйте их для конкретных магазинов или терминалов по мере необходимости. См. [Группы магазинов POS](pos-store-groups) для получения подробных инструкций по настройке.

## Советы

- **Начните с одного терминала** - протестируйте настройку POS и рабочий процесс с одним терминалом перед развертыванием на всю флоту
- **Назначьте склад до сопоставления** - терминалы не могут обрабатывать продажи без назначения склада
- **Настройте шаблоны чеков заранее** - поля соответствия (идентификаторы налогов) различаются по регионам; настройте их до запуска
- **Проверьте режим автономной работы** - отключите интернет и проверьте, продолжаются ли продажи; подтвердите синхронизацию при восстановлении соединения
- **Используйте группы магазинов для нескольких местоположений** - упрощает управление настройками для франшиз или региональных развертываний
- **Отслеживайте статус пульса** - терминалы отправляют пинг серверу каждые 5 минут; терминалы в режиме оффлайн отображаются в панели администратора
- **Настройте лимиты синхронизации для производительности** - терминалы с медленным подключением выигрывают от более низких настроек sync_days/sync_limit
- **Создайте резервную копию настроек оборудования** - документируйте IP-адреса принтеров, настройки сканеров, конфигурацию денежных ящиков для восстановления после катастрофы
