---
title: วิธีการจัดส่ง
---

วิธีการจัดส่งคือตัวเลือกการจัดส่งที่แสดงให้ลูกค้าเห็นขณะชำระเงิน—แต่ละวิธีจะคำนวณค่าใช้จ่ายการจัดส่งโดยใช้กลยุทธ์การกำหนดราคาที่แตกต่างกัน Spwig รองรับวิธีการ 7 แบบ ตั้งแต่อัตราคงที่ง่ายๆ ไปจนถึงราคาแบบเรียลไทม์ที่คำนวณโดยผู้ให้บริการจัดส่งที่ซับซ้อน วิธีการสามารถถูกจำกัดตามค่าสั่งซื้อขั้นต่ำ/สูงสุด น้ำหนัก และโซนภูมิศาสตร์ ลูกค้าจะเลือกวิธีการที่ต้องการขณะชำระเงิน และค่าใช้จ่ายที่คำนวณจะถูกเพิ่มเข้าไปในยอดรวมของคำสั่งซื้อ

ใช้คู่มือนี้เพื่อตั้งค่าวิธีการจัดส่งที่ตรงกับโมเดลธุรกิจของคุณ ตั้งแต่การจัดส่งอัตราคงที่พื้นฐานไปจนถึงการกำหนดราคาแบบชั้นเชิงตามโซนที่ซับซ้อน

## ประเภทของวิธีการจัดส่ง

Spwig ให้บริการ 7 ประเภทของวิธีการจัดส่ง แต่ละประเภทมีตรรกะการคำนวณค่าใช้จ่ายที่แตกต่างกัน:

### การจัดส่งอัตราคงที่

**คืออะไร**: ค่าใช้จ่ายคงที่ไม่ว่าจะมีเนื้อหาในตะกร้าสินค้า จุดหมายปลายทาง หรือน้ำหนัก

**เมื่อใช้**:
- ร้านค้าที่ง่ายและมีค่าใช้จ่ายการจัดส่งที่คาดการณ์ได้
- สินค้าประเภทเดียว (ขนาด/น้ำหนักใกล้เคียงกัน)
- การจัดส่งภายในประเทศเท่านั้นโดยใช้อัตราค่าบริการมาตรฐาน
- การส่งฟรีตามเงื่อนไข (ใช้ร่วมกับโปรโมชันการจัดส่ง)

**การตั้งค่า**:
- ตั้งค่า **ประเภทวิธีการ** = การจัดส่งอัตราคงที่
- ใส่ **ค่าคงที่** (เช่น $9.99)
- ตัวเลือก: ตั้งค่าข้อจำกัดค่าสั่งซื้อขั้นต่ำ/สูงสุด

**ตัวอย่าง**: "การจัดส่งมาตรฐาน - $9.99" สำหรับคำสั่งซื้อภายในประเทศทั้งหมด

---

### การจัดส่งฟรี

**คืออะไร**: ตัวเลือกการจัดส่งที่ไม่มีค่าใช้จ่าย (ไม่มีค่าใช้จ่ายสำหรับลูกค้า)

**เมื่อใช้**:
- การส่งฟรีในโปรโมชัน
- คำสั่งซื้อที่มีมูลค่าสูง (ใช้ร่วมกับค่าสั่งซื้อขั้นต่ำ)
- ทางเลือกการรับสินค้าที่สถานที่ของคุณ
- สิทธิประโยชน์ของโปรแกรมสะสมแต้ม

**การตั้งค่า**:
- ตั้งค่า **ประเภทวิธีการ** = การจัดส่งฟรี
- ตัวเลือก: ตั้งค่า **ค่าสั่งซื้อขั้นต่ำ** (เช่น ฟรีเมื่อสั่งซื้อเกิน $50)
- ทำงานได้ดีกับโปรโมชันการจัดส่งสำหรับการส่งฟรีตามเงื่อนไข

**ตัวอย่าง**: "การจัดส่งฟรีสำหรับคำสั่งซื้อที่มีมูลค่าเกิน $50" โดยมี min_order_value = $50

---

### การจัดส่งตามน้ำหนัก

**คืออะไร**: ค่าใช้จ่ายคำนวณจากตารางอัตราตามชั้นที่กำหนดตามน้ำหนักรวมของตะกร้าสินค้า

**เมื่อใดที่ควรใช้**:
- สินค้าที่มีน้ำหนักเปลี่ยนแปลงได้ (หนังสือ, อุปกรณ์, ของชำ)
- แบบจำลองการกำหนดราคาของผู้ให้บริการตามน้ำหนัก
- อัตราส่วนน้ำหนักต่อต้นทุนที่คาดการณ์ได้

**การตั้งค่า**:
1. ตั้งค่า **ประเภทวิธี** = ตามน้ำหนัก
2. สร้าง **ตารางอัตราค่าจัดส่ง** ด้วย basis_type = "weight"
3. เพิ่ม **ระดับอัตราค่าจัดส่ง** (เช่น 0-5kg = $10, 5-10kg = $15, 10-20kg = $25)
4. ตัวเลือก: จำกัดเฉพาะโซนที่กำหนด

**ตัวอย่าง**:
```
0-2kg: $8
2-5kg: $12
5-10kg: $18
10kg+: $25
```

**ทำงานอย่างไร**: ตะกร้าคำนวณน้ำหนักรวม → ค้นหาระดับที่ตรงกัน → คืนค่าอัตราของระดับนั้น

---

### การจัดส่งตามราคา

**คืออะไร**: ค่าใช้จ่ายคำนวณจากตารางอัตราตามระดับที่กำหนดโดยใช้ยอดรวมของตะกร้า

**เมื่อใดที่ควรใช้**:
- ค่าจัดส่งสัมพันธ์กับมูลค่าของคำสั่งซื้อ
- ส่งเสริมให้มีมูลค่าตะกร้าสูงขึ้น (อัตราต่อดอลลาร์ต่ำลงในระดับที่สูงขึ้น)
- ทางเลือกที่ง่ายกว่าการจัดส่งตามน้ำหนักสำหรับสินค้าที่มีราคาใกล้เคียงกัน

**การตั้งค่า**:
1. ตั้งค่า **ประเภทวิธี** = ตามราคา
2. สร้าง **ตารางอัตราค่าจัดส่ง** ด้วย basis_type = "price"
3. เพิ่ม **ระดับอัตราค่าจัดส่ง** (เช่น $0-$50 = $9.99, $50-$100 = $14.99, $100+ = $19.99)

**ตัวอย่าง**:
```
$0-$25: $6.99
$25-$75: $9.99
$75-$150: $12.99
$150+: Free
```

**ทำงานอย่างไร**: ตะกร้าคำนวณยอดรวม → ค้นหาระดับที่ตรงกัน → คืนค่าอัตราของระดับนั้น

---

### อัตราค่าจัดส่งแบบเรียลไทม์จากผู้ให้บริการ

**คืออะไร**: อัตราค่าจัดส่งแบบสดที่ดึงมาจาก API ของผู้ให้บริการ (FedEx, UPS, DHL) ที่เคาน์เตอร์ชำระเงิน

**เมื่อใดที่ควรใช้**:
- ค่าจัดส่งเปลี่ยนแปลงตามจุดหมายปลายทาง
- มีตัวเลือกผู้ให้บริการหลายรายสำหรับลูกค้า
- ราคาผู้ให้บริการที่แม่นยำโดยไม่ต้องตั้งค่าตารางอัตราด้วยตนเอง
- การจัดส่งระหว่างประเทศที่มีราคาซับซ้อน

**การตั้งค่า**:
1. ตั้งค่า **ประเภทวิธี** = เรียลไทม์
2. สร้าง **บัญชีผู้ให้บริการ** (การตั้งค่า > การจัดส่ง > บัญชีผู้ให้บริการ)
3. ใส่ข้อมูลการเข้าถึง API ของผู้ให้บริการ (หมายเลขบัญชี, คีย์ API, ความลับ)
4. เชื่อมต่อบัญชีผู้ให้บริการกับวิธีการจัดส่ง
5. ตัวเลือก: เพิ่มเปอร์เซ็นต์เพิ่มค่าหรือค่าเพิ่มคงที่

**ข้อกำหนด**:
- บัญชีผู้ให้บริการที่ใช้งานอยู่ (FedEx, UPS, DHL, ฯลฯ)
- ข้อมูลการเข้าถึง API จากผู้ให้บริการ
- กำหนดแพ็กเกจการจัดส่ง (สำหรับการคำนวณน้ำหนักตามขนาด)

**ตัวอย่าง**: วิธีการ "FedEx Ground" จะดึงอัตราค่าจัดส่งจาก FedEx แบบเรียลไทม์ ตามน้ำหนักสินค้า ขนาด และปลายทางในขั้นตอนชำระเงิน

**วิธีการทำงาน**:
1. ลูกค้ากรอกที่อยู่ในขั้นตอนชำระเงิน
2. ระบบเรียกใช้ API ของผู้ให้บริการโดยใช้จุดเริ่มต้น ปลายทาง ขนาด น้ำหนักของแพ็กเกจ
3. ผู้ให้บริการส่งอัตราค่าจัดส่งกลับมา
4. สามารถเพิ่มค่าที่ตั้งไว้ได้ (ถ้าต้องการ)
5. แสดงอัตราค่าจัดส่งให้ลูกค้าเห็น

---

### การรับสินค้าที่จุดรับของตนเอง

**คืออะไร**: ลูกค้าสามารถรับสินค้าได้ที่สถานที่จริง (ไม่มีค่าใช้จ่ายในการจัดส่ง)

**ควรใช้เมื่อ**:
- ร้านค้าปลีกที่ให้บริการรับสินค้าที่ร้าน
- ตัวเลือกรับสินค้าที่คลังสินค้า
- กิจกรรมหรือตลาดนัด
- ลดค่าใช้จ่ายในการจัดส่งสำหรับลูกค้าในพื้นที่

**การตั้งค่า**:
1. ตั้งค่า **ประเภทวิธี** = การรับสินค้าที่จุดรับของตนเอง
2. สร้าง **สถานที่** (การตั้งค่า > การจัดส่ง > สถานที่)
   - ตั้งค่าที่อยู่ เวลาทำการ ความจุในการรับสินค้า
3. เชื่อมโยงสถานที่กับวิธี
4. ตัวเลือก: ตั้งค่าเวลาเตรียมสินค้า (เช่น "พร้อมรับภายใน 2 ชั่วโมง")

**ประสบการณ์ของลูกค้า**:
- เลือก "การรับสินค้าที่จุดรับของตนเอง" ในขั้นตอนชำระเงิน
- เลือกสถานที่รับสินค้า (ถ้ามีหลายแห่ง)
- เลือกวันและเวลาในการรับสินค้าตามความพร้อม
- ได้รับการแจ้งเตือนเมื่อสินค้าพร้อมรับ

**ตัวอย่าง**: "รับสินค้าที่ร้าน - ฟรี" พร้อม 3 สาขาค้าปลีก พร้อมรับภายใน 24 ชั่วโมง

---

### การจัดส่งตามตารางอัตราค่าจัดส่ง

**คืออะไร**: ราคาแบบชั้นเชิงที่ยืดหยุ่น ขึ้นอยู่กับน้ำหนัก ราคา หรือจำนวน พร้อมเป้าหมายโซนที่ซับซ้อน

**ควรใช้เมื่อ**:
- ราคาที่ซับซ้อน (อัตราค่าจัดส่งแตกต่างกันตามโซน และน้ำหนัก)
- ต้องการควบคุมมากกว่าการกำหนดตามน้ำหนักหรือราคาเพียงอย่างเดียว
- ปัจจัยหลายอย่างในการกำหนดราคา (เช่น น้ำหนัก + ปลายทาง + จำนวน)

**การตั้งค่า**:
1. ตั้งค่า **ประเภทวิธี** = การจัดส่งตามตารางอัตราค่าจัดส่ง
2. สร้าง **ตารางอัตราค่าจัดส่ง**
3. กำหนด **basis_type**: น้ำหนัก ราคา หรือจำนวน
4. เพิ่ม **ช่วงอัตราค่าจัดส่ง** พร้อมค่าต่ำสุด/สูงสุด
5. ตัวเลือก: จำกัดช่วงตามโซนหรือประเทศเฉพาะ

**ความแตกต่างจาก Weight/Price-Based**: Table rate รองรับการจำกัดภูมิศาสตร์ตามระดับ ทำให้สามารถกำหนดอัตราค่าจัดส่งที่แตกต่างกันสำหรับน้ำหนัก/ราคาเดียวกันในโซนต่างๆ

**ตัวอย่าง**:
```
Zone A (ภายในประเทศ):
  0-5kg: $10
  5-10kg: $15

Zone B (พื้นที่ห่างไกล):
  0-5kg: $18
  5-10kg: $25
```

**วิธีการทำงาน**: ตะกร้าคำสั่งซื้อคำนวณค่าพื้นฐาน (น้ำหนัก/ราคา/จำนวน) → ค้นหาระดับที่ตรงกับโซนของลูกค้า → คืนค่าอัตราของระดับนั้น

## การตั้งค่าวิธีการจัดส่ง

วิธีการจัดส่งทั้งหมดมีการตั้งค่าที่เหมือนกันดังนี้:

### การตั้งค่าพื้นฐาน

- **ชื่อ**: ตัวระบุภายใน (ไม่แสดงให้ลูกค้าเห็น)
- **ชื่อที่แสดง**: ชื่อที่ลูกค้าเห็นในขั้นตอนชำระเงิน (เช่น "Standard Shipping", "Express Delivery")
- **คำอธิบาย**: ข้อความช่วยเหลือที่แสดงในขั้นตอนชำระเงิน (เช่น "จัดส่งภายใน 3-5 วันทำการ")
- **ประเภทวิธี**: เลือกจากประเภท 7 ประเภทข้างต้น
- **เปิดใช้งาน**: ปิด/เปิดใช้งานวิธีโดยไม่ต้องลบ

### การตั้งค่าค่าใช้จ่าย

- **ค่าคงที่**: สำหรับวิธี flat-rate เท่านั้น
- **ตารางอัตรา**: สำหรับวิธีน้ำหนัก, ราคา, table-rate
- **บัญชีผู้ให้บริการ**: สำหรับวิธี real-time carrier
- **ประเภทภาษี**: ใช้ภาษีกับค่าจัดส่ง (หากมีผล)

### การจำกัด

**การจำกัดค่าสั่งซื้อ**:
- **ค่าสั่งซื้อขั้นต่ำ**: วิธีนี้จะใช้งานได้ก็ต่อเมื่อค่ารวมตะกร้า ≥ จำนวน (เช่น จัดส่งฟรีเมื่อสั่งซื้อเกิน $50)
- **ค่าสั่งซื้อขั้นสูงสุด**: วิธีนี้จะซ่อนหากค่ารวมตะกร้า > จำนวน (เช่น ค่าคงที่สำหรับสั่งซื้อที่ต่ำกว่า $100)

**การจำกัดน้ำหนัก**:
- **น้ำหนักขั้นต่ำ**: วิธีนี้จะใช้งานได้ก็ต่อเมื่อน้ำหนักตะกร้า ≥ จำนวน
- **น้ำหนักขั้นสูงสุด**: วิธีนี้จะซ่อนหากน้ำหนักตะกร้า > จำนวน (พบได้บ่อยในตัวเลือกจัดส่งสำหรับสินค้าเบา)

**การจำกัดภูมิศาสตร์**:
- **โซนจัดส่ง**: เชื่อมโยงวิธีกับโซนเฉพาะ (ภายในประเทศ, ระหว่างประเทศ, ภูมิภาค)
- โซนว่าง = ใช้งานได้กับทุกที่อยู่
- โซนหลายแห่ง = ใช้งานได้กับโซนที่ตรงกันใดๆ

### การตั้งค่าขั้นสูง

- **ลำดับความสำคัญ**: ลำดับการแสดงในหน้าชำระเงิน (จำนวนที่ต่ำกว่า = แสดงอยู่ด้านบนในรายการ)
- **ค่าธรรมเนียมการจัดส่ง**: ค่าใช้จ่ายเพิ่มเติมคงที่ที่เพิ่มเข้ากับต้นทุนที่คำนวณแล้ว
- **เกณฑ์การจัดส่งฟรี**: ตั้งค่าต้นทุนเป็น $0 อัตโนมัติหากยอดสินค้าในตะกร้า ≥ เกณฑ์ (เป็นทางเลือกแทน min_order_value)

## การสร้างวิธีการจัดส่ง

**ขั้นตอนการทำงานแบบทีละขั้น**:

1. **ไปยังวิธีการจัดส่ง**
   - ไปที่ Settings > Cart > Shipping Methods
   - คลิก "Add Shipping Method"

2. **เลือกประเภทวิธีการ**
   - เลือกประเภทที่เหมาะสมตามกลยุทธ์การกำหนดราคาของคุณ
   - ประเภทจะกำหนดฟิลด์การตั้งค่าต้นทุนที่สามารถใช้งานได้

3. **ตั้งค่าข้อมูลพื้นฐาน**
   - Name: ชื่ออ้างอิงภายใน (เช่น, "domestic_ground")
   - Display Name: ชื่อที่แสดงให้ลูกค้าเห็น (เช่น, "Ground Shipping")
   - Description: เวลาจัดส่ง (เช่น, "5-7 วันทำการ")

4. **ตั้งค่าการคำนวณต้นทุน**
   - **อัตราคงที่**: ใส่ต้นทุนคงที่
   - **น้ำหนัก/ราคา/ตารางอัตรา**: สร้างตารางอัตรา (ดูข้างล่าง)
   - **แบบเรียลไทม์**: เชื่อมต่อบัญชีผู้ให้บริการ
   - **ฟรี/รับที่ร้าน**: ไม่จำเป็นต้องตั้งค่าต้นทุน

5. **เพิ่มข้อจำกัด (ไม่จำเป็น)**
   - ค่าสั่งซื้อขั้นต่ำ/สูงสุด
   - น้ำหนักขั้นต่ำ/สูงสุด
   - พื้นที่จัดส่ง

6. **ตั้งค่าลำดับความสำคัญ**
   - จำนวนที่ต่ำกว่าจะแสดงก่อนในหน้าชำระเงิน
   - ลำดับที่แนะนำ: ฟรี (1), รับที่ร้าน (2), มาตรฐาน (3), เอ็กซ์เพรส (4)

7. **เปิดใช้งานวิธีการ**
   - เปิดสวิตช์ "Active" = ใช่
   - บันทึก

## การสร้างตารางอัตรา

สำหรับวิธีการที่ใช้ตามน้ำหนัก ตามราคา และตารางอัตรา:

**ขั้นตอนที่ 1: สร้างตารางอัตรา**
- ไปที่ Settings > Shipping > Rate Tables
- คลิก "Add Rate Table"
- ตั้งค่า **ชื่อ** (เช่น, "Domestic Weight Tiers")
- ตั้งค่า **ประเภทฐาน**: weight, price, หรือ quantity

**ขั้นตอนที่ 2: เพิ่มช่วงอัตรา**
- คลิก "Add Tier"
- ตั้งค่า **ค่าขั้นต่ำ** และ **ค่าสูงสุด** (ช่วงที่ใช้ในการจับคู่)
- ตั้งค่า **อัตรา** (ต้นทุนสำหรับช่วงนี้)
- ตัวเลือก: จำกัดเฉพาะโซนหรือประเทศที่กำหนด
- บันทึกช่วง

**ขั้นตอนที่ 3: ทำซ้ำสำหรับช่วงทั้งหมด**
- ครอบคลุมช่วงทั้งหมด (0 ถึงค่าสูงสุดที่คาดการณ์ได้)
- ตรวจสอบให้แน่ใจว่าไม่มีช่องว่าง (เช่น, 0-5, 5-10, 10-20, 20+)
- ใช้ `null` สำหรับค่าสูงสุดในช่วงสุดท้าย (ไม่จำกัด)

**ขั้นตอนที่ 4: ลิงก์ไปยังวิธีการจัดส่ง**
- แก้ไขวิธีการจัดส่ง
- เลือกตารางอัตราค่าจัดส่งจากเมนูแบบเลื่อนลง
- บันทึก

**ตัวอย่างตารางอัตราค่าจัดส่งตามน้ำหนัก**:
```
ชื่อ: Domestic Weight Tiers
ฐาน: Weight

Tiers:
1. Min: 0g, Max: 2000g, Rate: $8
2. Min: 2000g, Max: 5000g, Rate: $12
3. Min: 5000g, Max: 10000g, Rate: $18
4. Min: 10000g, Max: null, Rate: $25
```

## สถานการณ์การจัดส่งที่พบบ่อย

### สถานการณ์ที่ 1: การจัดส่งภายในประเทศพื้นฐาน

**เป้าหมาย**: อัตราค่าจัดส่งคงที่ $9.99 สำหรับคำสั่งซื้อภายในประเทศทั้งหมด

**วิธีแก้ไข**:
- ประเภทวิธี: Flat Rate
- ค่าใช้จ่ายคงที่: $9.99
- พื้นที่การจัดส่ง: "Domestic" (เฉพาะประเทศของคุณเท่านั้น)

---

### สถานการณ์ที่ 2: การจัดส่งฟรีเมื่อสั่งซื้อครบ $50

**เป้าหมาย**: ส่งเสริมให้ลูกค้าเพิ่มมูลค่าตะกร้าด้วยเกณฑ์การจัดส่งฟรี

**ตัวเลือกวิธีแก้ไข A** (แนะนำ):
- ประเภทวิธี: Free Shipping
- ค่าสั่งซื้อขั้นต่ำ: $50
- ชื่อแสดง: "Free Shipping (Orders $50+)"

**ตัวเลือกวิธีแก้ไข B** (ใช้กฎ):
- ประเภทวิธี: Flat Rate
- ค่าใช้จ่ายคงที่: $9.99
- สร้างโปรโมชันการจัดส่ง:
  - ข้อกำหนด: มูลค่าตะกร้า ≥ $50
  - การกระทำ: ตั้งค่าค่าใช้จ่ายเป็น $0

---

### สถานการณ์ที่ 3: การจัดส่งตามน้ำหนักภายในประเทศ + ระหว่างประเทศ

**เป้าหมาย**: อัตราค่าจัดส่งที่แตกต่างกันสำหรับภายในประเทศและระหว่างประเทศตามน้ำหนัก

**วิธีแก้ไข**:
1. สร้าง 2 พื้นที่: "Domestic", "International"
2. สร้าง 2 ตารางอัตราค่าจัดส่ง: "Domestic Weight", "International Weight"
3. สร้าง 2 วิธี:
   - "Domestic Shipping" → ลิงก์ไปยังพื้นที่ Domestic + ตารางอัตราค่าจัดส่ง Domestic Weight
   - "International Shipping" → ลิงก์ไปยังพื้นที่ International + ตารางอัตราค่าจัดส่ง International Weight

---

### สถานการณ์ที่ 4: ตัวเลือกผู้ให้บริการหลายราย

**เป้าหมาย**: ให้ลูกค้าเลือกระหว่าง FedEx Ground, FedEx Express, UPS Ground

**วิธีแก้ไข**:
1. สร้างบัญชีผู้ให้บริการสำหรับ API ของ FedEx
2. สร้างบัญชีผู้ให้บริการสำหรับ API ของ UPS
3. สร้าง 3 วิธีแบบเรียลไทม์:
   - "FedEx Ground" → ผู้ให้บริการ FedEx, service code = "FEDEX_GROUND"
   - "FedEx Express" → ผู้ให้บริการ FedEx, service code = "FEDEX_EXPRESS"
   - "UPS Ground" → ผู้ให้บริการ UPS, service code = "UPS_GROUND"
4. ทั้ง 3 วิธีจะสอบถาม API ของผู้ให้บริการที่เคาน์เตอร์ชำระเงินและแสดงอัตราค่าใช้จ่ายแบบเรียลไทม์

### สถานการณ์ที่ 5: การรับสินค้าที่ร้าน + การจัดส่ง

**เป้าหมาย**: ร้านค้าปลีกเสนอทั้งตัวเลือกรับสินค้าที่ร้านและจัดส่ง

**แนวทางแก้ไข**:
1. สร้างสถานที่: "Main Store" พร้อมที่อยู่ เวลาทำการ และเวลาเตรียมสินค้า
2. สร้างวิธี 2 วิธี:
   - "Local Pickup" → ประเภท Local Pickup ที่เชื่อมโยงกับสถานที่ Main Store
   - "Standard Delivery" → ค่าจัดส่งคงที่ $9.99
3. ลูกค้าจะเห็นตัวเลือกทั้งสองที่เคาน์เตอร์ชำระเงิน

## การทดสอบวิธีการจัดส่ง

ก่อนเปิดให้บริการ ทดสอบวิธีการทั้งหมด:

1. **สร้างตะกร้าทดสอบ**
   - เพิ่มสินค้าที่มีน้ำหนักและราคาต่างกัน
   - ดำเนินการไปยังเคาน์เตอร์ชำระเงิน

2. **ทดสอบวิธีการแต่ละวิธี**
   - ใส่ที่อยู่ในโซนต่างๆ
   - ตรวจสอบว่าวิธีการที่เหมาะสมปรากฏขึ้น
   - ตรวจสอบว่าค่าที่คำนวณตรงกับความคาดหวัง

3. **ทดสอบข้อจำกัด**
   - เพิ่มสินค้าจนถึงมูลค่าขั้นต่ำ → ตรวจสอบว่าการจัดส่งฟรีปรากฏขึ้น
   - เพิ่มสินค้าที่มีน้ำหนักมาก → ตรวจสอบว่าวิธีการตามน้ำหนักทำงานได้
   - ทดสอบข้อจำกัดโซน → ตรวจสอบว่าวิธีการซ่อนสำหรับโซนที่ถูกตัดออก

4. **ทดสอบวิธีการแบบเรียลไทม์** (หากใช้ได้)
   - ใช้ข้อมูลการเข้าถึงทดสอบของผู้ให้บริการ
   - ตรวจสอบว่าอัตราค่าที่ได้รับกลับมาสำเร็จ
   - ตรวจสอบความถูกต้องของอัตราค่ากับเว็บไซต์ผู้ให้บริการ

## การแก้ไขปัญหา

**ปัญหาที่ 1: วิธีการไม่ปรากฏที่เคาน์เตอร์ชำระเงิน**

**สาเหตุ**:
- วิธีการไม่ได้เปิดใช้งาน
- ตะกร้าไม่ตรงตามมูลค่าขั้นต่ำ/สูงสุด
- ตะกร้าไม่ตรงตามน้ำหนักขั้นต่ำ/สูงสุด
- ที่อยู่ลูกค้าไม่ตรงกับโซนที่เชื่อมโยง
- ไม่มีช่วงอัตราค่าที่ครอบคลุมน้ำหนัก/ราคาตะกร้า

**แนวทางแก้ไข**: ตรวจสอบข้อจำกัด ตรวจสอบสถานะการเปิดใช้งาน ตรวจสอบว่าโซน/ช่วงครอบคลุมสถานการณ์ของลูกค้า

---

**ปัญหาที่ 2: อัตราค่าแบบเรียลไทม์ล้มเหลว**

**สาเหตุ**:
- ข้อมูล API ไม่ถูกต้อง
- บัญชีผู้ให้บริการไม่ได้เปิดใช้งาน
- ไม่มีแพ็กเกจการจัดส่งที่กำหนด (ผู้ให้บริการต้องการมิติ)
- ไม่ได้ตั้งที่อยู่ต้นทาง
- API ผู้ให้บริการล่ม

**แนวทางแก้ไข**: ทดสอบการเชื่อมต่อกับผู้ให้บริการ ตรวจสอบข้อมูลการเข้าถึง ตรวจสอบว่าแพ็กเกจได้กำหนดแล้ว ตรวจสอบที่อยู่ต้นทางในการตั้งค่า

---

**ปัญหาที่ 3: ค่าที่คำนวณไม่ถูกต้อง**

**สาเหตุ**:
- ช่วงอัตราค่ามีช่องว่างหรือทับซ้อนกัน
- ค่าขั้นต่ำ/สูงสุดในช่วงไม่ถูกต้อง (กรัม vs กิโลกรัม)
- ค่าธรรมเนียมการจัดการถูกเพิ่มโดยไม่คาดคิด
- กฎการจัดส่งที่ปรับเปลี่ยนค่า

**วิธีแก้ไข**: ตรวจสอบระดับตารางอัตราค่าจัดส่ง ยืนยันหน่วย ตรวจสอบลำดับความสำคัญของโปรโมชันการจัดส่ง

---

## เคล็ดลับ

- **เริ่มจากง่ายๆ** - ใช้อัตราค่าจัดส่งแบบคงที่สำหรับวิธีแรก เพิ่มความซับซ้อนเมื่อจำเป็น
- **ทดสอบอย่างละเอียด** - ตรวจสอบว่าวิธีทั้งหมดทำงานได้ในสภาพแวดล้อมทดสอบก่อนเปิดใช้งานในสภาพแวดล้อมการผลิต
- **ใช้ชื่อที่อธิบายได้ชัดเจน** - "การจัดส่งมาตรฐาน (5-7 วัน)" ดีกว่า "วิธี 1"
- **ตั้งค่าเวลาจัดส่งที่เป็นจริง** - ให้คำมั่นสัญญาที่ต่ำกว่าและส่งตรงเวลาเพื่อความพึงพอใจของลูกค้า
- **เสนอการรับสินค้าเองหากเป็นไปได้** - ลดค่าใช้จ่ายในการจัดส่งและเพิ่มความสะดวกสบายให้ลูกค้า
- **ตรวจสอบความน่าเชื่อถือของ API ผู้ให้บริการจัดส่ง** - ใช้อัตราค่าจัดส่งแบบคงที่เป็นทางเลือกหากอัตราค่าจัดส่งแบบเรียลไทม์ล้มเหลว
- **ใช้โซนสำหรับการจัดส่งระหว่างประเทศ** - อัตราค่าจัดส่งที่แตกต่างกันตามภูมิภาคช่วยป้องกันการสูญเสียในจุดหมายปลายทางที่มีค่าใช้จ่ายสูง
- **รวมกับโปรโมชันการจัดส่ง** - กฎเพิ่มตรรกะแบบมีเงื่อนไข (โปรโมชันการจัดส่งฟรี ค่าธรรมเนียมเพิ่มสำหรับพื้นที่ห่างไกล)
- **จำกัดวิธีให้น้อย** - 2-4 ตัวเลือกที่เคาน์เตอร์ช่วยป้องกันการตัดสินใจที่ซับซ้อน
- **อัปเดตตารางอัตราตามฤดูกาล** - อัตราค่าจัดส่งของผู้ให้บริการเปลี่ยนแปลง ตรวจสอบทุกปี
- **ใช้ลำดับความสำคัญอย่างชาญฉลาด** - วางตัวเลือกที่ฟรี/ถูกที่สุดข้างหน้า และตัวเลือกที่มีค่าใช้จ่ายสูงสุดไว้ท้าย
