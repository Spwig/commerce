---
title: วิธีการจัดส่ง
---

วิธีการจัดส่งคือตัวเลือกการจัดส่งที่แสดงให้ลูกค้าเห็นในขั้นตอนการชำระเงิน แต่ละวิธีจะคำนวณค่าใช้จ่ายการจัดส่งโดยใช้กลยุทธ์การกำหนดราคาที่แตกต่างกัน Spwig สนับสนุนวิธีการ 7 แบบตั้งแต่ราคาคงที่ง่ายๆ ไปจนถึงราคาแบบเรียลไทม์ที่คำนวณโดยผู้ให้บริการจัดส่งที่ซับซ้อน วิธีการสามารถจำกัดได้ตามค่าสั่งซื้อขั้นต่ำ/สูงสุด น้ำหนัก และโซนภูมิศาสตร์ ลูกค้าเลือกวิธีการที่ต้องการในขั้นตอนการชำระเงิน และค่าใช้จ่ายที่คำนวณจะถูกเพิ่มเข้าไปในยอดรวมของคำสั่งซื้อ

ใช้คู่มือนี้เพื่อตั้งค่าวิธีการจัดส่งที่ตรงกับโมเดลธุรกิจของคุณ ตั้งแต่การจัดส่งแบบราคาคงที่พื้นฐานไปจนถึงการกำหนดราคาแบบชั้นเชิงตามโซนที่ซับซ้อน

## ประเภทของวิธีการจัดส่ง

Spwig ให้บริการ 7 ประเภทของวิธีการจัดส่ง แต่ละประเภทมีตรรกะการคำนวณค่าใช้จ่ายที่แตกต่างกัน:

### การจัดส่งแบบราคาคงที่

**คืออะไร**: ค่าใช้จ่ายคงที่ไม่ว่าจะมีเนื้อหาในตะกร้าสินค้า ปลายทาง หรือน้ำหนัก

**เมื่อใช้**:
- ร้านค้าที่ง่ายและมีค่าใช้จ่ายการจัดส่งที่คาดการณ์ได้
- ประเภทสินค้าเดียว (ขนาด/น้ำหนักคล้ายกัน)
- การจัดส่งภายในประเทศเท่านั้นโดยใช้อัตราค่าบริการมาตรฐาน
- การส่งฟรีตามโปรโมชั่น (ใช้ร่วมกับกฎการจัดส่ง)

**การตั้งค่า**:
- ตั้งค่า **ประเภทวิธีการ** = ราคาคงที่
- ใส่ **ค่าคงที่** (เช่น $9.99)
- ตัวเลือก: ตั้งค่าข้อจำกัดค่าสั่งซื้อขั้นต่ำ/สูงสุด

**ตัวอย่าง**: "การจัดส่งมาตรฐาน - $9.99" สำหรับคำสั่งซื้อภายในประเทศทั้งหมด

---

### การจัดส่งฟรี

**คืออะไร**: ตัวเลือกการจัดส่งที่ไม่มีค่าใช้จ่าย (ไม่มีค่าใช้จ่ายสำหรับลูกค้า)

**เมื่อใช้**:
- การส่งฟรีตามโปรโมชั่น
- คำสั่งซื้อที่มีค่าสูง (ใช้ร่วมกับค่าสั่งซื้อขั้นต่ำ)
- ทางเลือกการรับสินค้าที่สถานที่ท้องถิ่น
- สิทธิประโยชน์ของโปรแกรมสะสมแต้ม

**การตั้งค่า**:
- ตั้งค่า **ประเภทวิธีการ** = การจัดส่งฟรี
- ตัวเลือก: ตั้งค่า **ค่าสั่งซื้อขั้นต่ำ** (เช่น ฟรีเมื่อสั่งซื้อเกิน $50)
- ทำงานได้ดีกับกฎการจัดส่งสำหรับการส่งฟรีแบบมีเงื่อนไข

**ตัวอย่าง**: "การจัดส่งฟรีสำหรับคำสั่งซื้อที่สูงกว่า $50" โดยมี min_order_value = $50

---

### การจัดส่งตามน้ำหนัก

**คืออะไร**: ค่าใช้จ่ายคำนวณจากตารางอัตราชั้นเชิงตามน้ำหนักรวมของตะกร้าสินค้า

**เมื่อใช้**:
- สินค้าที่มีน้ำหนักต่างกัน (หนังสือ อุปกรณ์ฮาร์ดแวร์ ของชำ)
- แบบจำลองการกำหนดราคาตามน้ำหนักของผู้ให้บริการจัดส่ง
- อัตราส่วนน้ำหนักต่อค่าใช้จ่ายที่คาดการณ์ได้

**การตั้งค่า**:
1. ตั้งค่า **ประเภทวิธีการ** = ตามน้ำหนัก
2. สร้าง **ตารางอัตราการจัดส่ง** โดยมี basis_type = "weight"
3. เพิ่ม **ชั้นอัตราการจัดส่ง** (เช่น 0-5kg = $10, 5-10kg = $15, 10-20kg = $25)
4. ตัวเลือก: จำกัดเฉพาะโซนที่กำหนด

**ตัวอย่าง**:
```
0-2kg: $8
2-5kg: $12
5-10kg: $18
10kg+: $25
```

**วิธีการทำงาน**: ตะกร้าคำนวณน้ำหนักรวม → ค้นหาชั้นที่ตรงกัน → คืนค่าอัตราของชั้น

---

### การจัดส่งตามราคา

**คืออะไร**: ค่าใช้จ่ายคำนวณจากตารางอัตราชั้นเชิงตามยอดรวมตะกร้าสินค้า

**เมื่อใช้**:
- ค่าใช้จ่ายการจัดส่งสัมพันธ์กับค่าของคำสั่งซื้อ
- ส่งเสริมให้ยอดตะกร้าสูงขึ้น (อัตราต่ำกว่าต่อเงินในชั้นที่สูงกว่า)
- ทางเลือกที่ง่ายกว่าการจัดส่งตามน้ำหนักสำหรับสินค้าที่มีราคาใกล้เคียงกัน

**การตั้งค่า**:
1. ตั้งค่า **ประเภทวิธีการ** = ตามราคา
2. สร้าง **ตารางอัตราการจัดส่ง** โดยมี basis_type = "price"
3. เพิ่ม **ชั้นอัตราการจัดส่ง** (เช่น $0-$50 = $9.99, $50-$100 = $14.99, $100+ = $19.99)

**ตัวอย่าง**:
```
$0-$25: $6.99
$25-$75: $9.99
$75-$150: $12.99
$150+: ฟรี
```

**วิธีการทำงาน**: ตะกร้าคำนวณยอดรวม → ค้นหาชั้นที่ตรงกัน → คืนค่าอัตราของชั้น

---

### อัตราค่าจัดส่งแบบเรียลไทม์จากผู้ให้บริการ

**คืออะไร**: อัตราค่าใช้จ่ายแบบเรียลไทม์ที่ดึงมาจาก API ของผู้ให้บริการจัดส่ง (FedEx, UPS, DHL) ในขั้นตอนการชำระเงิน

**เมื่อใช้**:
- ค่าใช้จ่ายการจัดส่งเปลี่ยนแปลงตามปลายทาง
- มีตัวเลือกผู้ให้บริการจัดส่งหลายตัวสำหรับลูกค้า
- ราคาผู้ให้บริการที่แม่นยำโดยไม่ต้องใช้ตารางอัตราค่าใช้จ่ายด้วยตนเอง
- การจัดส่งระหว่างประเทศที่มีราคาซับซ้อน

**การตั้งค่า**:
1. ตั้งค่า **ประเภทวิธีการ** = เรียลไทม์
2. สร้าง **บัญชีผู้ให้บริการ** (Settings > Shipping > Provider Accounts)
3. ใส่ข้อมูลการเข้าถึง API ของผู้ให้บริการ (หมายเลขบัญชี, คีย์ API, ความลับ)
4. เชื่อมต่อบัญชีผู้ให้บริการกับวิธีการจัดส่ง
5. ตัวเลือก: เพิ่มเปอร์เซ็นต์เพิ่มค่าหรือค่าเพิ่มคงที่

**ข้อกำหนด**:
- บัญชีผู้ให้บริการที่ใช้งานอยู่ (FedEx, UPS, DHL, ฯลฯ)
- ข้อมูลการเข้าถึง API จากผู้ให้บริการ
- กำหนดแพ็กเกจการจัดส่ง (สำหรับการคำนวณน้ำหนักตามมิติ)

**ตัวอย่าง**: วิธีการ "FedEx Ground" ดึงอัตราค่าใช้จ่ายแบบเรียลไทม์จาก FedEx ตามน้ำหนัก ขนาด และปลายทางในขั้นตอนการชำระเงิน

**วิธีการทำงาน**:
1. ลูกค้าใส่ที่อยู่ในขั้นตอนการชำระเงิน
2. ระบบเรียกใช้ API ของผู้ให้บริการโดยใช้จุดเริ่มต้น ปลายทาง ขนาดแพ็กเกจ และน้ำหนัก
3. ผู้ให้บริการคืนค่าอัตราค่าใช้จ่าย
4. ตัวเลือกเพิ่มค่าถูกนำไปใช้
5. อัตราค่าใช้จ่ายแสดงให้ลูกค้าเห็น

---

### การรับสินค้าที่สถานที่ท้องถิ่น

**คืออะไร**: ลูกค้าเลือกสินค้าที่สถานที่ทางกายภาพ (ไม่มีค่าใช้จ่ายในการจัดส่ง)

**เมื่อใช้**:
- ร้านค้าปลีกที่เสนอการรับสินค้า
- ตัวเลือกการรับสินค้าที่คลังสินค้า
- เหตุการณ์หรือตลาดนัด
- ลดค่าใช้จ่ายการจัดส่งสำหรับลูกค้าท้องถิ่น

**การตั้งค่า**:
1. ตั้งค่า **ประเภทวิธีการ** = การรับสินค้าท้องถิ่น
2. สร้าง **สถานที่** (Settings > Shipping > Locations)
   - ตั้งค่าที่อยู่ เวลาทำการ และความสามารถในการรับสินค้า
3. เชื่อมต่อสถานที่กับวิธีการ
4. ตัวเลือก: ตั้งค่าเวลาเตรียมการรับสินค้า (เช่น "พร้อมใช้งานภายใน 2 ชั่วโมง")

**ประสบการณ์ของลูกค้า**:
- เลือก "การรับสินค้าท้องถิ่น" ในขั้นตอนการชำระเงิน
- เลือกสถานที่รับสินค้า (หากมีหลายแห่ง)
- เลือกวันและเวลาในการรับสินค้าตามความพร้อมใช้งาน
- ได้รับการแจ้งเตือนเมื่อสินค้าพร้อมใช้งาน

**ตัวอย่าง**: "รับสินค้าที่ร้านค้า - ฟรี" โดยมีร้านค้าปลีก 3 แห่ง พร้อมใช้งานภายใน 24 ชั่วโมง

---

### การจัดส่งตามตารางอัตรา

**คืออะไร**: การกำหนดราคาแบบชั้นเชิงที่ยืดหยุ่นตามน้ำหนัก ราคา หรือจำนวนสินค้าโดยมีการกำหนดเป้าหมายโซนที่ซับซ้อนมากขึ้น

**เมื่อใช้**:
- การกำหนดราคาที่ซับซ้อน (อัตราต่างกันตามโซนและน้ำหนัก)
- ต้องการควบคุมมากกว่าการจัดส่งตามน้ำหนักหรือราคาเพียงอย่างเดียว
- ปัจจัยการกำหนดราคาหลายอย่าง (เช่น น้ำหนัก + ปลายทาง + จำนวน)

**การตั้งค่า**:
1. ตั้งค่า **ประเภทวิธีการ** = ตารางอัตรา
2. สร้าง **ตารางอัตราการจัดส่ง**
3. กำหนด **basis_type**: น้ำหนัก ราคา หรือจำนวน
4. เพิ่ม **ชั้นอัตราการจัดส่ง** พร้อมค่าขั้นต่ำ/สูงสุด
5. ตัวเลือก: จำกัดชั้นตามโซนหรือประเทศเฉพาะ

**ความแตกต่างจากตามน้ำหนัก/ราคา**: ตารางอัตราสนับสนุนการจำกัดภูมิศาสตร์ต่อชั้น ทำให้สามารถกำหนดอัตราต่างกันสำหรับน้ำหนัก/ราคาเดียวกันในโซนต่างกันได้

**ตัวอย่าง**:
```
โซน A (ภายในประเทศ):
  0-5kg: $10
  5-10kg: $15

โซน B (พื้นที่ห่างไกล):
  0-5kg: $18
  5-10kg: $25
```

**วิธีการทำงาน**: ตะกร้าคำนวณค่าพื้นฐาน (น้ำหนัก/ราคา/จำนวน) → ค้นหาชั้นที่ตรงกับโซนของลูกค้า → คืนค่าอัตราของชั้น

---

## การตั้งค่าวิธีการจัดส่ง

วิธีการจัดส่งทั้งหมดมีการตั้งค่าที่เหมือนกันดังนี้:

### การตั้งค่าพื้นฐาน

- **ชื่อ**: ชื่ออ้างอิงภายใน (ไม่แสดงให้ลูกค้าเห็น)
- **ชื่อแสดงผล**: ชื่อที่ลูกค้าเห็นในขั้นตอนการชำระเงิน (เช่น "การจัดส่งมาตรฐาน", "การจัดส่งแบบเร่งด่วน")
- **คำอธิบาย**: ข้อความช่วยเหลือที่แสดงในขั้นตอนการชำระเงิน (เช่น "จัดส่งภายใน 3-5 วันทำการ")
- **ประเภทวิธีการ**: หนึ่งใน 7 ประเภทข้างต้น
- **เปิดใช้งาน**: ปิด/เปิดวิธีการโดยไม่ต้องลบ

### การตั้งค่าค่าใช้จ่าย

- **ค่าคงที่**: สำหรับวิธีการราคาคงที่เท่านั้น
- **ตารางอัตรา**: สำหรับวิธีการตามน้ำหนัก ตามราคา และตารางอัตรา
- **บัญชีผู้ให้บริการ**: สำหรับวิธีการเรียลไทม์ผู้ให้บริการ
- **ประเภทภาษี**: ใช้ภาษีกับค่าใช้จ่ายการจัดส่ง (หากมีผล)

### ข้อจำกัด

**ข้อจำกัดค่าสั่งซื้อ**:
- **ค่าสั่งซื้อขั้นต่ำ**: วิธีการจะสามารถใช้งานได้ก็ต่อเมื่อค่ารวมตะกร้า ≥ จำนวน (เช่น การจัดส่งฟรีเมื่อสั่งซื้อเกิน $50)
- **ค่าสั่งซื้อสูงสุด**: วิธีการจะถูกซ่อนหากค่ารวมตะกร้า > จำนวน (เช่น ราคาคงที่สำหรับคำสั่งซื้อที่ต่ำกว่า $100)

**ข้อจำกัดน้ำหนัก**:
- **น้ำหนักขั้นต่ำ**: วิธีการจะสามารถใช้งานได้ก็ต่อเมื่อน้ำหนักตะกร้า ≥ จำนวน
- **น้ำหนักสูงสุด**: วิธีการจะถูกซ่อนหากน้ำหนักตะกร้า > จำนวน (พบได้บ่อยในตัวเลือกการจัดส่งน้ำหนักเบา)

**ข้อจำกัดภูมิศาสตร์**:
- **โซนการจัดส่ง**: เชื่อมต่อวิธีการกับโซนเฉพาะ (ภายในประเทศ ระหว่างประเทศ ภูมิภาค)
- โซนว่าง = สามารถใช้งานได้กับที่อยู่ทั้งหมด
- โซนหลายแห่ง = สามารถใช้งานได้กับโซนที่ตรงกันใดๆ

### การตั้งค่าขั้นสูง

- **ลำดับความสำคัญ**: ลำดับการแสดงในขั้นตอนการชำระเงิน (จำนวนที่ต่ำกว่า = แสดงในรายการสูงกว่า)
- **ค่าธรรมเนียมการจัดการ**: ค่าใช้จ่ายเพิ่มเติมที่ถูกเพิ่มเข้าไปในค่าใช้จ่ายที่คำนวณ
- **เกณฑ์การจัดส่งฟรี**: ตั้งค่าค่าใช้จ่ายเป็น $0 หากค่ารวมตะกร้า ≥ เกณฑ์ (ทางเลือกอื่นแทน min_order_value)

---

## การสร้างวิธีการจัดส่ง

**ขั้นตอนการทำงานแบบทีละขั้นตอน**:

1. **ไปที่วิธีการจัดส่ง**
   - ไปที่ Settings > Cart > Shipping Methods
   - คลิก "Add Shipping Method"

2. **เลือกประเภทวิธีการ**
   - เลือกประเภทที่เหมาะสมตามกลยุทธ์การกำหนดราคาของคุณ
   - ประเภทกำหนดฟิลด์การตั้งค่าค่าใช้จ่ายที่สามารถใช้งานได้

3. **ตั้งค่าข้อมูลพื้นฐาน**
   - ชื่อ: อ้างอิงภายใน (เช่น "domestic_ground")
   - ชื่อแสดงผล: สำหรับลูกค้า (เช่น "การจัดส่งพื้นฐาน")
   - คำอธิบาย: เวลาจัดส่ง (เช่น "5-7 วันทำการ")

4. **ตั้งค่าการคำนวณค่าใช้จ่าย**
   - **ราคาคงที่**: ใส่ค่าคงที่
   - **ตามน้ำหนัก/ราคา/ตารางอัตรา**: สร้างตารางอัตรา (ดูข้างล่าง)
   - **เรียลไทม์**: เชื่อมต่อบัญชีผู้ให้บริการ
   - **ฟรี/รับสินค้า**: ไม่จำเป็นต้องตั้งค่าค่าใช้จ่าย

5. **เพิ่มข้อจำกัด (ตัวเลือก)**
   - ค่าสั่งซื้อขั้นต่ำ/สูงสุด
   - น้ำหนักขั้นต่ำ/สูงสุด
   - โซนการจัดส่ง

6. **ตั้งค่าลำดับความสำคัญ**
   - จำนวนที่ต่ำกว่าจะแสดงก่อนในขั้นตอนการชำระเงิน
   - ลำดับที่แนะนำ: ฟรี (1), การรับสินค้าท้องถิ่น (2), มาตรฐาน (3), เร่งด่วน (4)

7. **เปิดใช้งานวิธีการ**
   - เปิดใช้งาน = ใช่
   - บันทึก

## การสร้างตารางอัตรา

สำหรับวิธีการตามน้ำหนัก ตามราคา และตารางอัตรา:

**ขั้นตอนที่ 1: สร้างตารางอัตรา**
- ไปที่ Settings > Shipping > Rate Tables
- คลิก "Add Rate Table"
- ตั้งค่า **ชื่อ** (เช่น "ชั้นน้ำหนักภายในประเทศ")
- ตั้งค่า **ประเภทพื้นฐาน**: น้ำหนัก ราคา หรือจำนวน

**ขั้นตอนที่ 2: เพิ่มชั้น**
- คลิก "Add Tier"
- ตั้งค่า **ค่าขั้นต่ำ** และ **ค่าสูงสุด** (ช่วงที่ตรงกัน)
- ตั้งค่า **อัตรา** (ค่าใช้จ่ายสำหรับชั้นนี้)
- ตัวเลือก: จำกัดเฉพาะโซนหรือประเทศ
- บันทึกชั้น

**ขั้นตอนที่ 3: ทำซ้ำสำหรับชั้นทั้งหมด**
- ครอบคลุมช่วงทั้งหมด (0 ถึงค่าสูงสุดที่คาดการณ์ได้)
- ตรวจสอบให้แน่ใจว่าไม่มีช่องว่าง (เช่น 0-5, 5-10, 10-20, 20+)
- ใช้ `null` สำหรับค่าสูงสุดในชั้นสุดท้าย (ไม่จำกัด)

**ขั้นตอนที่ 4: เชื่อมต่อกับวิธีการจัดส่ง**
- แก้ไขวิธีการจัดส่ง
- เลือกตารางอัตราจากเมนูลงล่าง
- บันทึก

**ตัวอย่างตารางอัตราตามน้ำหนัก**:
```
ชื่อ: ชั้นน้ำหนักภายในประเทศ
ประเภทพื้นฐาน: น้ำหนัก

ชั้น:
1. ค่าขั้นต่ำ: 0g, ค่าสูงสุด: 2000g, อัตรา: $8
2. ค่าขั้นต่ำ: 2000g, ค่าสูงสุด: 5000g, อัตรา: $12
3. ค่าขั้นต่ำ: 5000g, ค่าสูงสุด: 10000g, อัตรา: $18
4. ค่าขั้นต่ำ: 10000g, ค่าสูงสุด: null, อัตรา: $25
```

## สถานการณ์การจัดส่งที่พบบ่อย

### สถานการณ์ที่ 1: การจัดส่งภายในประเทศพื้นฐาน

**เป้าหมาย**: ราคาคงที่ $9.99 สำหรับคำสั่งซื้อภายในประเทศทั้งหมด

**แนวทางแก้ไข**:
- ประเภทวิธีการ: ราคาคงที่
- ค่าคงที่: $9.99
- โซนการจัดส่ง: "ภายในประเทศ" (ประเทศของคุณเท่านั้น)

---

### สถานการณ์ที่ 2: การจัดส่งฟรีเมื่อสั่งซื้อเกิน $50

**เป้าหมาย**: ส่งเสริมให้ยอดตะกร้าสูงขึ้นด้วยเกณฑ์การจัดส่งฟรี

**แนวทางแก้ไข ตัวเลือก A** (แนะนำ):
- ประเภทวิธีการ: การจัดส่งฟรี
- ค่าสั่งซื้อขั้นต่ำ: $50
- ชื่อแสดงผล: "การจัดส่งฟรี (คำสั่งซื้อ $50+)")

**แนวทางแก้ไข ตัวเลือก B** (ใช้กฎ):
- ประเภทวิธีการ: ราคาคงที่
- ค่าคงที่: $9.99
- สร้างกฎการจัดส่ง:
  - เงื่อนไข: ค่าตะกร้า ≥ $50
  - การกระทำ: ตั้งค่าค่าใช้จ่ายเป็น $0

---

### สถานการณ์ที่ 3: การจัดส่งตามน้ำหนักภายในประเทศ + ระหว่างประเทศ

**เป้าหมาย**: อัตราค่าใช้จ่ายต่างกันสำหรับภายในประเทศและระหว่างประเทศตามน้ำหนัก

**แนวทางแก้ไข**:
1. สร้างโซน 2 โซน: "ภายในประเทศ", "ระหว่างประเทศ"
2. สร้างตารางอัตรา 2 ตาราง: "น้ำหนักภายในประเทศ", "น้ำหนักระหว่างประเทศ"
3. สร้างวิธีการ 2 วิธี:
   - "การจัดส่งภายในประเทศ" → เชื่อมต่อกับโซนภายในประเทศ + ตารางอัตราภายในประเทศ
   - "การจัดส่งระหว่างประเทศ" → เชื่อมต่อกับโซนระหว่างประเทศ + ตารางอัตราระหว่างประเทศ

---

### สถานการณ์ที่ 4: ตัวเลือกผู้ให้บริการจัดส่งหลายตัว

**เป้าหมาย**: ให้ลูกค้าเลือกระหว่าง FedEx Ground, FedEx Express, UPS Ground

**แนวทางแก้ไข**:
1. สร้างบัญชีผู้ให้บริการสำหรับ API ของ FedEx
2. สร้างบัญชีผู้ให้บริการสำหรับ API ของ UPS
3. สร้างวิธีการแบบเรียลไทม์ 3 วิธี:
   - "FedEx Ground" → ผู้ให้บริการ FedEx, รหัสบริการ = "FEDEX_GROUND"
   - "FedEx Express" → ผู้ให้บริการ FedEx, รหัสบริการ = "FEDEX_EXPRESS"
   - "UPS Ground" → ผู้ให้บริการ UPS, รหัสบริการ = "UPS_GROUND"
4. วิธีการทั้ง 3 วิธีจะสอบถาม API ของผู้ให้บริการในขั้นตอนการชำระเงินและแสดงอัตราค่าใช้จ่ายแบบเรียลไทม์

---

### สถานการณ์ที่ 5: การรับสินค้าท้องถิ่น + การจัดส่ง

**เป้าหมาย**: ร้านค้าปลีกเสนอตัวเลือกการรับสินค้าและจัดส่ง

**แนวทางแก้ไข**:
1. สร้างสถานที่: "ร้านค้าหลัก" โดยมีที่อยู่ เวลาทำการ และเวลาเตรียมการรับสินค้า
2. สร้างวิธีการ 2 วิธี:
   - "การรับสินค้าท้องถิ่น" → ประเภทการรับสินค้าท้องถิ่น, เชื่อมต่อกับสถานที่ร้านค้าหลัก
   - "การจัดส่งมาตรฐาน" → ราคาคงที่ $9.99
3. ลูกค้าจะเห็นตัวเลือกทั้งสองในขั้นตอนการชำระเงิน

## การทดสอบวิธีการจัดส่ง

ก่อนเปิดใช้งาน ทดสอบวิธีการทั้งหมด:

1. **สร้างตะกร้าทดสอบ**
   - เพิ่มสินค้าที่มีน้ำหนัก/ราคาต่างกัน
   - ดำเนินการไปยังขั้นตอนการชำระเงิน

2. **ทดสอบวิธีการแต่ละวิธี**
   - ใส่ที่อยู่ในโซนต่างๆ
   - ตรวจสอบว่าวิธีการที่เหมาะสมปรากฏขึ้น
   - ตรวจสอบให้แน่ใจว่าค่าใช้จ่ายที่คำนวณตรงกับความคาดหวัง

3. **ทดสอบข้อจำกัด**
   - เพิ่มสินค้าจนถึงค่าสั่งซื้อขั้นต่ำ → ตรวจสอบให้แน่ใจว่าการจัดส่งฟรีปรากฏขึ้น
   - เพิ่มสินค้าที่มีน้ำหนักมาก → ตรวจสอบให้แน่ใจว่าชั้นตามน้ำหนักทำงานได้
   - ทดสอบข้อจำกัดโซน → ตรวจสอบให้แน่ใจว่าวิธีการถูกซ่อนสำหรับโซนที่ถูกตัดออก

4. **ทดสอบวิธีการแบบเรียลไทม์** (หากจำเป็น):
   - ใช้ข้อมูลการเข้าถึง API ของผู้ให้บริการทดสอบ
   - ตรวจสอบให้แน่ใจว่าอัตราค่าใช้จ่ายถูกคืนค่าได้สำเร็จ
   - ตรวจสอบความถูกต้องของอัตราค่าใช้จ่ายกับเว็บไซต์ของผู้ให้บริการ

## การแก้ไขปัญหา

**ปัญหา 1: วิธีการไม่ปรากฏในขั้นตอนการชำระเงิน**

**สาเหตุ**:
- วิธีการไม่ได้เปิดใช้งาน
- ตะกร้าไม่ตรงกับค่าสั่งซื้อขั้นต่ำ/สูงสุด
- ตะกร้าไม่ตรงกับน้ำหนักขั้นต่ำ/สูงสุด
- ที่อยู่ลูกค้าไม่ตรงกับโซนที่เชื่อมต่อ
- ไม่มีชั้นตารางอัตราที่ครอบคลุมน้ำหนัก/ราคาของตะกร้า

**แนวทางแก้ไข**: ตรวจสอบข้อจำกัด ตรวจสอบสถานะการเปิดใช้งาน ตรวจสอบให้แน่ใจว่าโซน/ชั้นครอบคลุมสถานการณ์ของลูกค้า

---

**ปัญหา 2: อัตราค่าใช้จ่ายแบบเรียลไทม์ล้มเหลว**

**สาเหตุ**:
- ข้อมูลการเข้าถึง API ไม่ถูกต้อง
- บัญชีผู้ให้บริการไม่ได้เปิดใช้งาน
- ไม่มีแพ็กเกจการจัดส่งที่กำหนด (ผู้ให้บริการต้องการมิติ)
- ไม่ได้ตั้งค่าที่อยู่จุดเริ่มต้น
- API ของผู้ให้บริการล่ม

**แนวทางแก้ไข**: ทดสอบการเชื่อมต่อบัญชีผู้ให้บริการ ตรวจสอบข้อมูลการเข้าถึง ตรวจสอบให้แน่ใจว่าแพ็กเกจถูกกำหนด ตรวจสอบที่อยู่จุดเริ่มต้นในการตั้งค่า

---

**ปัญหา 3: ค่าใช้จ่ายที่คำนวณไม่ถูกต้อง**

**สาเหตุ**:
- ชั้นตารางอัตรามีช่องว่างหรือทับซ้อนกัน
- ค่าขั้นต่ำ/สูงสุดของชั้นอยู่ในหน่วยที่ผิด (กรัม vs กิโลกรัม)
- ค่าธรรมเนียมการจัดการถูกเพิ่มโดยไม่คาดคิด
- กฎการจัดส่งเปลี่ยนแปลงค่าใช้จ่าย

**แนวทางแก้ไข**: ตรวจสอบชั้นตารางอัตรา ตรวจสอบหน่วย ตรวจสอบลำดับความสำคัญของกฎการจัดส่ง

---

## คำแนะนำ

- **เริ่มต้นง่ายๆ** - ใช้ราคาคงที่สำหรับวิธีการแรก แล้วเพิ่มความซับซ้อนตามที่จำเป็น
- **ทดสอบอย่างละเอียด** - ตรวจสอบให้แน่ใจว่าวิธีการทั้งหมดทำงานได้ในสภาพแวดล้อมทดสอบก่อนเปิดใช้งานในสภาพแวดล้อมการผลิต
- **ใช้ชื่อที่อธิบายได้** - "การจัดส่งมาตรฐาน (5-7 วัน)") ดีกว่า "วิธีการ 1"
- **ตั้งค่าเวลาจัดส่งที่เป็นจริง** - ให้คำมั่นสัญญาที่ต่ำกว่าและส่งมอบที่ดีกว่าเพื่อความพึงพอใจของลูกค้า
- **เสนอการรับสินค้าหากเป็นไปได้** - ลดค่าใช้จ่ายการจัดส่ง ปรับปรุงความสะดวกในการใช้งานของลูกค้า
- **ตรวจสอบความน่าเชื่อถือของ API ผู้ให้บริการ** - มีวิธีการราคาคงที่เป็นทางเลือกหากอัตราค่าใช้จ่ายแบบเรียลไทม์ล้มเหลว
- **ใช้โซนสำหรับการจัดส่งระหว่างประเทศ** - อัตราต่างกันตามภูมิภาคช่วยป้องกันการสูญเสียในปลายทางที่มีค่าใช้จ่ายสูง
- **รวมกับกฎการจัดส่ง** - กฎเพิ่มตรรกะเงื่อนไข (โปรโมชั่นการจัดส่งฟรี ค่าเพิ่มสำหรับพื้นที่ห่างไกล)
- **จำกัดวิธีการให้น้อย** - 2-4 ตัวเลือกในขั้นตอนการชำระเงินช่วยป้องกันการตัดสินใจที่ซับซ้อน
- **อัปเดตตารางอัตราตามฤดูกาล** - อัตราค่าใช้จ่ายของผู้ให้บริการเปลี่ยนแปลง ตรวจสอบทุกปี
- **ใช้ลำดับความสำคัญอย่างชาญฉลาด** - วางตัวเลือกฟรี/ราคาถูกข้างหน้า ตัวเลือกที่มีราคาสูงข้างหลัง
