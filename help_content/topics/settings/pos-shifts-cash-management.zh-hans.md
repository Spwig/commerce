---
title: POS班次与现金管理
---

POS班次跟踪收银员的工作时段并确保准确的现金核算。每个班次代表单个收银员在终端上的时间——从用起始现金数打开钱箱到用最终数和对账关闭班次。系统会根据实际现金销售计算预期现金，并与实物清点数进行比较，突出显示差异以供调查。班次期间的现金流动（如增加备用金、提取零用现金）会记录原因，以实现完整的审计追踪。

导航到 **POS > 班次** 以查看所有班次、监控活动班次、查看现金对账报告和审计历史活动。

![班次列表](/static/core/admin/img/help/pos-shifts-cash-management/shift-list.webp)

## 了解POS班次

一个班次是单个收银员操作一个终端的工作时段。班次强制执行现金责任——每个收银员在其班次期间对其钱箱中的现金负责。

**班次生命周期**：
1. **开启** - 收银员开始班次，清点初始现金，记录金额
2. **班次期间** - 处理销售，接受付款，发放退款
3. **关闭** - 收银员清点现金，记录关闭金额，系统计算差异
4. **对账完成** - 班次最终确定并锁定以供审计

**跟踪的关键指标**：
- **初始现金** - 班次开始时钱箱中的起始现金
- **结束现金** - 班次结束时钱箱中的实物现金
- **预期现金** - 计算得出：初始现金 + 现金销售 - 现金退款 + 现金流动
- **现金差异** - 差异：结束现金 - 预期现金（正数 = 超额，负数 = 短缺）
- **总销售额** - 班次期间所有销售交易的总和
- **总退款** - 班次期间所有退款交易的总和
- **交易数量** - 处理的订单数量

## 班次列表视图

班次列表显示所有班次的关键信息：

**班次状态**：
- **开启**（绿色徽章）- 当前活动班次
- **关闭**（灰色徽章）- 已完成的班次
- **对账完成**（蓝色徽章）- 已最终确定并锁定以供审计

**终端** - 班次所在的POS终端

**收银员** - 执行班次的员工

**初始现金** - 起始现金金额

**结束现金** - 结束现金金额（如果班次仍开启则为空）

**预期现金** - 基于交易计算出的预期金额

**现金差异** - 差异（如果为负数则用红色突出显示，正数则用绿色，零则用黑色）

**持续时间** - 班次时长（从开始时间到结束时间）

**总销售额** - 班次期间产生的收入

使用筛选器查看：
- 仅开启的班次（监控活动终端）
- 存在差异的班次（现金差异 ≠ 0）
- 按日期范围筛选班次（每日对账报告）
- 按收银员筛选班次（绩效审计）

## 开启班次

收银员直接从POS终端开启班次（不能从管理员界面开启）。终端上的工作流程：

1. **员工登录** - 输入凭据以访问终端

2. **清点初始现金** - 实际清点钱箱中的所有现金（纸币和硬币）

3. **输入初始金额** - 在POS应用中记录清点的金额

4. **班次开始** - 终端准备好处理销售

**初始现金指南**：
- 标准初始现金（备用金）通常为$100-$300，具体取决于商店规模
- 清点两次以确保准确性——初始错误会导致关闭时的差异
- 如果钱箱为空，初始现金为$0.00（通过现金流动添加备用金）
- 单独记录大额纸币（> $50）以跟踪其流动

## 班次期间

当班次开启时，系统会自动跟踪：

**现金销售** - 任何客户用现金支付的交易（增加预期现金）

**现金退款** - 任何以现金形式发放的退款（减少预期现金）

**卡销售** - 信用卡/借记卡交易（不影响现金）

**分账支付** - 部分现金 + 部分卡（仅现金部分影响预期现金）

**礼品卡和优惠券** - 非现金支付方式（不影响现金）

收银员继续正常处理销售。系统在后台维护预期现金的持续计算。

## 现金流动

现金流动是在班次期间对钱箱的调整：

**备用金增加** - 向钱箱中添加现金：
- 原因："为大额纸币添加零钱"
- 金额：+$100.00
- 预期现金增加$100.00

**零用现金提取** - 为支出移除现金：
- 原因："购买办公用品"
- 金额：-$25.00
- 预期现金减少$25.00

**银行存款** - 为安全移除超额现金：
- 原因："安全存款 - 钱箱内超过$500"
- 金额：-$300.00
- 预期现金减少$300.00

**在终端上记录现金流动**：
1. 点击 **菜单** > **现金流动**
2. 选择类型：添加或移除
3. 输入金额
4. 输入原因（审计追踪所需）
5. 确认

所有现金流动都会在班次详细报告中显示，包括时间戳、金额和原因。

## 关闭班次

当收银员完成工作时段后，他们关闭班次：

1. **点击关闭班次** - 在终端菜单上

2. **处理剩余交易** - 完成任何挂起的购物车或待处理的销售

3. **清点结束现金** - 实际清点钱箱中的所有现金
   - 按面额清点纸币（$100s, $50s, $20s, $10s, $5s, $1s）
   - 按类型清点硬币（25美分、10美分、5美分、1美分）
   - 总计 = 结束现金金额

4. **输入结束金额** - 记录清点的总额

5. **系统计算差异**：
   - 预期现金 = 初始现金 + 现金销售 - 现金退款 + 现金流动
   - 现金差异 = 结束现金 - 预期现金
   - 示例：结束$485.00 - 预期$480.00 = +$5.00超额

6. **查看差异** - 终端显示差异：
   - **精确（$0.00）** - 完全对账
   - **小额超额（+$1 到 +$5）** - 可接受的四舍五入或顾客小费
   - **小额短缺（-$1 到 -$5）** - 微小计数错误，可接受
   - **大额差异（> $5）** - 需要重新计数

7. **如需重新计数** - 如果差异较大（> $10），收银员应在最终确认前重新清点结束现金

8. **最终确认班次** - 确认结束金额，班次状态变为"关闭"

9. **打印班次报告** - 终端打印现金对账收据供收银员记录

## 现金对账公式

系统使用以下公式计算预期现金：

```
预期现金 = 初始现金
            + 现金销售
            - 现金退款
            + 现金增加（流动）
            - 现金减少（流动）
```

**示例**：
- 初始现金：$200.00
- 现金销售：$450.00（来自15笔交易）
- 现金退款：-$30.00（1笔退货）
- 现金增加：+$100.00（班次中途添加备用金）
- 现金减少：-$50.00（提取零用现金）
- **预期现金：$200 + $450 - $30 + $100 - $50 = $670.00**

如果收银员在结束时清点$675.00：
- 现金差异：$675.00 - $670.00 = **+$5.00超额**

## 班次报告与审计

班次报告提供详细的对账信息：

**摘要部分**：
- 初始和结束现金
- 预期现金计算
- 现金差异（超额/短缺）
- 总销售额和退款
- 交易数量
- 班次时长

**交易详情**：
- 班次期间的所有销售（订单ID、金额、支付方式）
- 所有发出的退款
- 每笔交易的时间戳

**现金流动日志**：
- 所有增加和减少
- 提供的原因
- 时间戳

**使用场景**：
- **每日对账** - 在业务日结束时查看所有班次
- **收银员绩效** - 通过员工识别差异模式
- **盗窃检测** - 一致的大额短缺可能表明盗窃
- **培训需求** - 频繁的小额差异表明计数准确性问题
- **审计追踪** - 用于会计和税务的完整记录

## 多终端现金管理

对于有多个终端同时运行班次的商店：

**独立的钱箱**：每个终端都有自己的钱箱——班次是独立的。终端1的收银员A和终端2的收银员B运行独立的班次，各自进行对账。

**共享的钱箱**：一些商店在多个终端之间共享一个钱箱（不推荐）。如果这样做：
- 每个共享钱箱一次只能开启一个班次
- 收银员在交接给下一个收银员时必须关闭班次
- 现金流动跟踪交接期间的所有增加/减少
- 差异更难归因于特定收银员

**最佳实践**：每个终端一个钱箱，每个会话每个收银员一个班次。这确保了清晰的责任归属和简化的对账。

## 处理差异

当结束现金与预期现金不匹配时：

**小额差异（< $5）**：
- 由于四舍五入、计数错误或顾客小费，可以接受
- 在班次备注中记录
- 除非出现模式，否则不需要进一步操作

**中等差异（$5 - $20）**：
- 在最终确认班次前重新计数现金
- 检查交易日志是否有错误（错误找零、未处理的取消交易）
- 在班次备注中记录情况
- 建议经理审核

**大额差异（> $20）**：
- 必须重新计数
- 需要经理批准才能关闭班次
- 检查所有交易和现金流动
- 调查潜在原因（盗窃、钱箱被触碰、初始现金错误）
- 根据具体情况可能需要纪律处分

**持续短缺**：
- 同一收银员的持续负差异 = 培训问题或盗窃
- 实施额外的监督（经理在班次期间随机检查）
- 审核POS培训程序
- 考虑更新现金处理政策

## 小贴士

- **清点初始现金两次** - 初始错误会引发结束差异；起始时的准确性可防止结束时的问题
- **立即记录现金流动** - 不要等到结束时才记录备用金增加或零用现金提取
- **始终提供流动原因** - "添加$100" 对审计无用；"添加$100用于零钱（$5纸币不足）" 是可操作的
- **如果差异 > $10，重新计数** - 在没有重新计数的情况下不要最终确认有大额差异的班次
- **每天打印班次报告** - 附加到每日对账文件供会计使用
- **查看模式而非个别差异** - 一个 -$3.00 短缺是可以接受的；连续五个 -$3.00 短缺是问题
- **在一天结束时关闭班次** - 不要让班次在夜间保持开启；近期的差异更容易调查
- **培训收银员按面额计数** - 大多数错误来自纸币计数错误（将$5误认为$10）
- **使用硬币包装纸** - 预先包装的硬币可减少计数错误并加快对账速度
