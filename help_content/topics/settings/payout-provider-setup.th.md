title: การตั้งค่าผู้ให้บริการการจ่ายเงิน

การตั้งค่าผู้ให้บริการการจ่ายเงินช่วยให้คุณสามารถกำหนดค่า PayPal และ Airwallex เพื่อจ่ายเงินให้กับพันธมิตรอัตโนมัติ คู่มือนี้แสดงวิธีการเชื่อมต่อบัญชีผู้ให้บริการการจ่ายเงิน กำหนดค่า webhook และทดสอบการผสานรวมของคุณ

## ผู้ให้บริการการจ่ายเงินที่รองรับ

Spwig ผสานรวมกับผู้ให้บริการการจ่ายเงินสองรายเพื่อจ่ายเงินให้กับพันธมิตรอัตโนมัติ:

| ผู้ให้บริการ | วิธีการชำระเงิน | การประมวลผล | การสนับสนุนแบบแบตช์ | เหมาะสำหรับ |
|----------------|------------------|----------------|---------------------|------------|
| **PayPal** | การโอนเงินผ่านบัญชี PayPal | ผ่าน API | ใช่ (สูงสุด 15,000) | พันธมิตรส่วนใหญ่ ครอบคลุมทั่วโลก |
| **Airwallex** | การโอนเงินผ่านธนาคารระหว่างประเทศ | ผ่าน API | ไม่ (รายบุคคล) | การโอนเงินผ่านธนาคาร การชำระเงินระหว่างประเทศ |

### ความแตกต่างหลัก

**การจ่ายเงินผ่าน PayPal**:
- ต้องการให้พันธมิตรมีบัญชี PayPal (อีเมลสำหรับการชำระเงิน)
- ประมวลผลแบบแบตช์ได้สูงสุด 15,000 รายการในครั้งเดียว
- ใช้เวลาในการประมวลผลที่เร็วขึ้น (1-2 วันทำการ)
- ความซับซ้อนในการตั้งค่าต่ำกว่า
- ค่าธรรมเนียม: ~2% หรือ $0.25-$1.00 ต่อการชำระเงิน
- webhook เดียวสำหรับแบบแบตช์ทั้งหมด

**Airwallex**:
- สนับสนุนการโอนเงินผ่านธนาคารโดยตรง
- ประมวลผลการจ่ายเงินรายบุคคลทีละรายการ
- ใช้เวลาในการประมวลผลนานกว่า (2-5 วันทำการ)
- สนับสนุนหลายสกุลเงินและประเทศ
- ค่าธรรมเนียมแตกต่างกันไปตามประเทศปลายทาง
- webhook รายบุคคลต่อการจ่ายเงิน

คุณสามารถตั้งค่าผู้ให้บริการทั้งสองรายและให้พันธมิตรเลือกวิธีการชำระเงินที่ต้องการ

## ทำไมถึงควรใช้ผู้ให้บริการการจ่ายเงิน?

การผสานรวมผู้ให้บริการการจ่ายเงินมีประโยชน์มากกว่าการชำระเงินแบบมือถือ:

- **การประมวลผลอัตโนมัติ** — ไม่ต้องป้อนข้อมูลหรือดำเนินการชำระเงินด้วยมือ
- **ประสิทธิภาพแบบแบตช์** — ประมวลผลการจ่ายเงินหลายสิบหรือหลายร้อยรายการด้วยการคลิกเดียว
- **การยืนยันผ่าน webhook** — อัปเดตสถานะอัตโนมัติเมื่อการชำระเงินเสร็จสิ้น
- **ลดข้อผิดพลาด** — ระบบตรวจสอบข้อมูลบัญชีก่อนการประมวลผล
- **บันทึกการตรวจสอบ** — บันทึกการดำเนินการและคำตอบจากผู้ให้บริการทั้งหมด
- **การชำระเงินที่รวดเร็วขึ้น** — พันธมิตรจะได้รับเงินเร็วขึ้น
- **การขยายตัวได้ดี** — จัดการกับโปรแกรมพันธมิตรที่เติบโตโดยไม่ต้องเพิ่มงานดูแลตามสัดส่วน

หากไม่มีการผสานรวมผู้ให้บริการ คุณจะต้องดำเนินการชำระเงินแต่ละรายการผ่านบัญชีธนาคารหรือแดชบอร์ด PayPal ของคุณ แล้วคืนกลับไปยัง Spwig เพื่อทำเครื่องหมายว่าการจ่ายเงินเสร็จสิ้น

## การตั้งค่า PayPal

ทำตามขั้นตอนต่อไปนี้เพื่อตั้งค่า PayPal Payouts สำหรับการจ่ายเงินพันธมิตรอัตโนมัติ

### ข้อกำหนดเบื้องต้น

ก่อนเริ่มต้น คุณต้องมี:
- บัญชี PayPal Business (บัญชีส่วนตัวไม่สามารถใช้ API Payouts ได้)
- การเข้าถึงแดชบอร์ดผู้พัฒนา PayPal
- การอนุมัติสำหรับ API Payouts (หลังจากทดสอบในสภาพแวดล้อม Sandbox)

### ขั้นตอนที่ 1: สร้างแอป PayPal

1. **เข้าไปที่** [PayPal Developer Dashboard](https://developer.paypal.com/dashboard/)
2. **ลงชื่อเข้าใช้** ด้วยบัญชี PayPal Business ของคุณ
3. **คลิก** **My Apps & Credentials** ที่ด้านซ้ายของแถบด้านข้าง
4. **เลือก** แท็บ **Live** (หรือ Sandbox สำหรับการทดสอบ)
5. **คลิก** **Create App**
6. **ใส่ชื่อแอป** (เช่น "Spwig Affiliate Payouts")
7. **เลือกประเภทแอป**: Merchant
8. **คลิก** **Create App**

PayPal จะสร้างข้อมูลการรับรองของคุณ

### ขั้นตอนที่ 2: รับข้อมูลการรับรอง API

หลังจากสร้างแอป:

1. **คัดลอก Client ID** — ตัวอักษรยาวที่เป็นตัวเลข
2. **คลิก** **Show** ภายใต้ Secret
3. **คัดลอก Client Secret** — รักษาความลับนี้ไว้
4. **บันทึกโหมด** — Sandbox หรือ Live

### ขั้นตอนที่ 3: เปิดใช้งานคุณสมบัติการจ่ายเงิน

แอป PayPal ต้องการสิทธิ์ในการใช้คุณสมบัติการจ่ายเงิน:

1. **เลื่อนลงไปที่** หัวข้อ **Features** ในแอปของคุณ
2. **ค้นหา** คุณสมบัติ **Payouts**
3. **คลิก** **Add** หากยังไม่ได้เปิดใช้งาน
4. **ส่งคำขออนุมัติ** หากใช้โหมด Live (การอนุมัติใช้เวลา 1-2 วันทำการ)

### ขั้นตอนที่ 4: เพิ่มผู้ให้บริการใน Spwig

ตอนนี้เพิ่มบัญชี PayPal ลงใน Spwig:

1. **เข้าไปที่** **Settings > Payout Providers**
2. **คลิก** **+ Add PayPal Account**
3. **กรอกแบบฟอร์ม**:
   - **ชื่อบัญชี**: ป้ายกำกับที่อธิบายได้ (เช่น "Main PayPal Account")
   - **Client ID**: คัดลอกจากแดชบอร์ดผู้พัฒนา PayPal
   - **Client Secret**: คัดลอกจากแดชบอร์ดผู้พัฒนา PayPal
   - **โหมด**: เลือก Sandbox (สำหรับการทดสอบ) หรือ Production (สำหรับการใช้งานจริง)
   - **Is Active**: ติ๊กเพื่อเปิดใช้งาน
4. **คลิก Save**

Spwig จะตรวจสอบข้อมูลการรับรองโดยการขอโทเค็นการเข้าถึง หากการตรวจสอบล้มเหลว โปรดตรวจสอบ Client ID และ Secret ของคุณอีกครั้ง

### ขั้นตอนที่ 5: ทดสอบการเชื่อมต่อ

ตรวจสอบการผสานรวม PayPal ของคุณ:

1. สร้างการจ่ายเงินทดสอบใน **Affiliate Program > Payouts**
2. ใช้อีเมล PayPal ของคุณเองเป็นผู้รับ
3. ตั้งจำนวนเงินเป็น $0.01 (หากใช้โหมด Production) หรือจำนวนใดก็ได้ (หากใช้ Sandbox)
4. ดำเนินการผ่านผู้ให้บริการ
5. ตรวจสอบบัญชี PayPal ของคุณเพื่อดูการชำระเงินที่เข้ามา
6. ตรวจสอบว่า webhook อัปเดตสถานะการจ่ายเงินใน Spwig

หากใช้โหมด Sandbox ให้สร้างบัญชี PayPal ทดสอบที่ [PayPal Sandbox](https://developer.paypal.com/dashboard/accounts) เพื่อรับการจ่ายเงินทดสอบ

## การตั้งค่า Airwallex

Airwallex สนับสนุนการโอนเงินผ่านธนาคารระหว่างประเทศสำหรับพันธมิตรที่ต้องการการโอนเงินโดยตรง

### ข้อกำหนดเบื้องต้น

ก่อนเริ่มต้น คุณต้องมี:
- บัญชี Airwallex (สร้างที่ [airwallex.com](https://www.airwallex.com))
- สถานะบัญชีธุรกิจที่ได้รับการยืนยัน
- เปิดใช้งานการเข้าถึง API (ติดต่อฝ่ายสนับสนุน Airwallex หากจำเป็น)
- ยอดเงินเพียงพอในบัญชี Airwallex ของคุณ

### ขั้นตอนที่ 1: สร้างข้อมูลการรับรอง API

1. **ลงชื่อเข้าใช้** ที่ [Airwallex Dashboard](https://www.airwallex.com/app/)
2. **เข้าไปที่** **Settings > API Keys**
3. **คลิก** **Create API Key**
4. **ใส่คำอธิบาย**: "Spwig Affiliate Payouts"
5. **เลือกสิทธิ์**: เปิดใช้งาน **Payouts** (อ่านและเขียน)
6. **คลิก** **Generate**
7. **คัดลอก API Key** — แสดงเพียงครั้งเดียว
8. **คัดลอก Client ID** — แสดงร่วมกับคีย์

### ขั้นตอนที่ 2: บันทึกสภาพแวดล้อมของคุณ

Airwallex ให้สภาพแวดล้อมสองแบบ:

- **Demo**: สำหรับการทดสอบด้วยธุรกรรมปลอม
- **Production**: สำหรับการโอนเงินจริง

ตรวจสอบให้แน่ใจว่าคุณทราบว่าคีย์ API ของคุณอยู่ในสภาพแวดล้อมใด

### ขั้นตอนที่ 3: เพิ่มผู้ให้บริการใน Spwig

เพิ่มบัญชี Airwallex ลงใน Spwig:

1. **เข้าไปที่** **Settings > Payout Providers**
2. **คลิก** **+ Add Airwallex Account**
3. **กรอกแบบฟอร์ม**:
   - **ชื่อบัญชี**: ป้ายกำกับที่อธิบายได้ (เช่น "Airwallex EUR Account")
   - **API Key**: คัดลอกจากแดชบอร์ด Airwallex
   - **Client ID**: คัดลอกจากแดชบอร์ด Airwallex
   - **สภาพแวดล้อม**: เลือก Demo หรือ Production
   - **Is Active**: ติ๊กเพื่อเปิดใช้งาน
4. **คลิก Save**

Spwig จะตรวจสอบข้อมูลการรับรองโดยการสอบถามยอดเงินในบัญชีของคุณ

### ขั้นตอนที่ 4: ตรวจสอบประเทศที่รองรับ

Airwallex สนับสนุนการโอนเงินไปยังประเทศหลายประเทศ แต่ไม่ใช่ทุกประเทศ ตรวจสอบหน้า [Airwallex coverage](https://www.airwallex.com/global-business-account/global-transfers) เพื่อยืนยันว่าประเทศของพันธมิตรของคุณได้รับการสนับสนุน

ประเทศที่รองรับทั่วไป ได้แก่:
- สหรัฐอเมริกา
- สหราชอาณาจักร
- ประเทศในสหภาพยุโรป
- ออสเตรเลีย
- แคนาดา
- สิงคโปร์
- ฮ่องกง

### ขั้นตอนที่ 5: ทดสอบการโอนเงินผ่านธนาคาร

ทดสอบการผสานรวม Airwallex ของคุณ:

1. สร้างการจ่ายเงินทดสอบสำหรับพันธมิตรที่มีรายละเอียดบัญชีธนาคาร
2. ใช้จำนวนเงินเล็กน้อย ($1-$5) หากอยู่ในโหมด Production
3. ดำเนินการผ่านผู้ให้บริการ
4. ตรวจสอบแดชบอร์ด Airwallex สำหรับธุรกรรม
5. รอให้ webhook ยืนยัน
6. ตรวจสอบว่าการจ่ายเงินเสร็จสิ้นใน Spwig

โหมด Demo ดำเนินการทันที โหมด Production ใช้เวลา 2-5 วันทำการ

## ตรรกะการเลือกผู้ให้บริการ

เมื่อคุณดำเนินการจ่ายเงิน Spwig จะเลือกผู้ให้บริการที่เหมาะสมโดยอัตโนมัติโดยพิจารณาจากวิธีการชำระเงินของพันธมิตร

### ขั้นตอนการเลือก

1. **ตรวจสอบวิธีการชำระเงินของพันธมิตร**:
   - หากตั้งค่า `payment_email` → พันธมิตรต้องการใช้ PayPal
   - หากตั้งค่ารายละเอียดบัญชีธนาคาร → พันธมิตรต้องการใช้การโอนเงินผ่านธนาคาร
2. **จับคู่กับผู้ให้บริการ**:
   - อีเมล PayPal → ใช้บัญชีผู้ให้บริการ PayPal ที่เปิดใช้งาน
   - รายละเอียดบัญชีธนาคาร → ใช้บัญชีผู้ให้บริการ Airwallex ที่เปิดใช้งาน
3. **กลับไปใช้ผู้ให้บริการแรกที่พร้อมใช้งาน** หากผู้ให้บริการที่ต้องการไม่ได้ตั้งค่า
4. **แสดงข้อความผิดพลาด** หากไม่มีผู้ให้บริการที่ตรงกัน

### บัญชีผู้ให้บริการหลายตัว

คุณสามารถตั้งค่าบัญชีหลายตัวสำหรับผู้ให้บริการเดียวกัน (เช่น บัญชี PayPal สองตัวสำหรับภูมิภาคต่างกัน) Spwig จะเลือกบัญชีแรกที่เปิดใช้งานและตรงกับวิธีการชำระเงิน ในการควบคุมว่าบัญชีใดถูกใช้ ให้จัดลำดับใหม่ในรายการผู้ดูแลระบบ หรือตั้งค่าเพียงบัญชีเดียวให้เปิดใช้งาน

## การทดสอบการผสานรวมการจ่ายเงิน

ตรวจสอบการผสานรวมผู้ให้บริการของคุณเสมอ ก่อนดำเนินการชำระเงินให้พันธมิตรจริง

### การทดสอบในโหมด Sandbox/Demo

1. **ตั้งค่าผู้ให้บริการให้อยู่ในโหมด Sandbox** (PayPal Sandbox หรือ Airwallex Demo)
2. **สร้างพันธมิตรทดสอบ** พร้อมรายละเอียดการชำระเงินทดสอบ
3. **สร้างคอมมิชชันทดสอบ** และอนุมัติ
4. **สร้างการจ่ายเงินทดสอบ** ที่รวมคอมมิชชันเหล่านั้น
5. **ดำเนินการผ่านผู้ให้บริการ** โดยใช้เมนูการดำเนินการ
6. **ตรวจสอบบันทึก Celery** สำหรับคำขอ API
7. **ตรวจสอบแดชบอร์ดผู้ให้บริการ** สำหรับธุรกรรม
8. **รอให้ webhook** อัปเดตสถานะการจ่ายเงิน
9. **ตรวจสอบว่าคอมมิชชันถูกทำเครื่องหมายว่าได้รับการชำระเงินแล้ว**

### การทดสอบในโหมด Production

ก่อนเริ่มใช้งานจริง:

1. **เปลี่ยนไปใช้โหมด Production** ในการตั้งค่าผู้ให้บริการ
2. **สร้างการจ่ายเงินทดสอบเล็กน้อย** ให้กับตัวคุณเอง ($0.01-$1.00)
3. **ดำเนินการ** และรอให้เสร็จสิ้น
4. **ตรวจสอบว่าได้รับเงิน** ในบัญชีของคุณเอง
5. **ตรวจสอบว่า webhook ถูกส่ง** และสถานะได้รับการอัปเดต
6. **ตรวจสอบค่าธรรมเนียมการดำเนินการของผู้ให้บริการ**

### ปัญหาที่พบบ่อยในการทดสอบ

| ปัญหา | สาเหตุ | วิธีแก้ไข |
|--------|--------|-----------|
| "ข้อมูลการรับรองไม่ถูกต้อง" | คีย์ API ผิด หรือโหมดไม่ตรงกัน | ตรวจสอบข้อมูลการรับรองอีกครั้ง ตรวจสอบว่าโหมด Sandbox และ Production ตรงกัน |
| webhook ไม่เคยทำงาน | URL ไม่ได้ตั้งค่าในผู้ให้บริการ | เพิ่ม URL webhook ในแดชบอร์ดผู้ให้บริการ |
| การจ่ายเงินยังอยู่ในสถานะ "Processing" | การตรวจสอบลายเซ็น webhook ล้มเหลว | ตรวจสอบว่าลายเซ็น webhook ตรงกัน |
| ไม่มีผู้ให้บริการใดที่พร้อมใช้งาน | ไม่มีผู้ให้บริการที่เปิดใช้งานสำหรับวิธีการชำระเงิน | เปิดใช้งานบัญชีผู้ให้บริการอย่างน้อยหนึ่งบัญชี |

## การประมวลผลแบบแบตช์ (PayPal)

PayPal สนับสนุนการประมวลผลแบบแบตช์เพื่อเพิ่มประสิทธิภาพและประหยัดค่าใช้จ่าย

### การทำงานของแบบแบตช์

เมื่อคุณเลือกการจ่ายเงินหลายรายการและคลิก **Process with Provider**:

1. Spwig จัดกลุ่มการจ่ายเงินผ่าน PayPal ทั้งหมดเข้าเป็นแบบแบตช์เดียว
2. ระบบส่งคำขอ API เดียวพร้อมรายละเอียดการจ่ายเงินทั้งหมด (สูงสุด 15,000)
3. PayPal ประมวลผลแบบแบตช์ทั้งหมดเป็นธุรกรรมเดียว
4. webhook จะคืนผลลัพธ์แบบแบตช์
5. Spwig อัปเดตการจ่ายเงินทั้งหมดตามผลลัพธ์แบบแบตช์

### ข้อดีของการประมวลผลแบบแบตช์

- **ลดการเรียกใช้ API** — คำขอเดียวสำหรับการจ่ายเงินหลายร้อยรายการ
- **ค่าธรรมเนียมต่ำกว่า** — โครงสร้างค่าธรรมเนียมบางส่วนของ PayPal สนับสนุนการประมวลผลแบบแบตช์
- **การประมวลผลที่รวดเร็วขึ้น** — ดำเนินการแบบแบตช์ทั้งหมดพร้อมกัน
- **webhook เดียว** — ง่ายต่อการตรวจสอบและบันทึกข้อมูล

### ข้อจำกัดของการประมวลผลแบบแบตช์

PayPal กำหนดข้อจำกัดเหล่านี้:
- สูงสุด 15,000 ผู้รับต่อแบบแบตช์
- สูงสุด $100,000 รวมต่อแบบแบตช์
- การประมวลผลมักจะเสร็จสิ้นภายในไม่กี่นาที

หากคุณมีการจ่ายเงินเกิน 15,000 รายการ Spwig จะแบ่งเป็นแบบแบตช์หลายแบบโดยอัตโนมัติ

## การประมวลผลแบบรายบุคคล (Airwallex)

Airwallex ประมวลผลการจ่ายเงินทีละรายการ ซึ่งมีข้อได้เปรียบและข้อเสียที่แตกต่างกัน

### การทำงานของการประมวลผลแบบรายบุคคล

เมื่อคุณดำเนินการจ่ายเงินผ่าน Airwallex:

1. ระบบส่งคำขอ API แยกต่างหากสำหรับแต่ละการจ่ายเงิน
2. Airwallex เพิ่มการโอนเงินแต่ละรายการเข้าคิว
3. แต่ละการโอนเงินจะเสร็จสิ้นได้โดยอิสระ (2-5 วันทำการ)
4. webhook แต่ละรายการจะทำงานเมื่อแต่ละการโอนเงินเสร็จสิ้น
5. Spwig อัปเดตการจ่ายเงินเมื่อ webhook ถึง

### ข้อดีของการประมวลผลแบบรายบุคคล

- **การแยกข้อผิดพลาดได้ดีขึ้น** — การล้มเหลวหนึ่งรายการไม่ได้ขัดขวางรายการอื่น
- **ติดตามรายการจ่ายเงิน** — รหัสธุรกรรมต่อรายการ
- **ข้อมูลการชำระเงินมากขึ้น** — ข้อมูลเฉพาะธนาคารต่อการโอนเงิน
- **เวลาที่ยืดหยุ่นได้** — การโอนเงินเสร็จสิ้นในอัตราที่แตกต่างกัน

### เวลาในการประมวลผล

ต่างจาก PayPal ที่ดำเนินการแบบแบตช์ทันที Airwallex ใช้เวลานานกว่า:
- การโอนเงินภายในประเทศ: 1-2 วันทำการ
- การโอนเงินระหว่างประเทศ: 3-5 วันทำการ
- บางประเทศ: ถึง 7 วันทำการ

ตั้งความคาดหวังของพันธมิตรให้เหมาะสมในข้อตกลงของโปรแกรมของคุณ

## การตั้งค่า webhook

webhook ช่วยให้อัปเดตสถานะการจ่ายเงินอัตโนมัติเมื่อผู้ให้บริการดำเนินการเสร็จสิ้น

### รูปแบบ URL webhook

ตั้งค่า URL นี้ในแดชบอร์ดผู้ให้บริการของคุณ:

```
https://yourdomain.com/api/payout-providers/{provider}/webhook/
```

แทน `{provider}` ด้วย:
- `paypal` สำหรับ webhook ของ PayPal
- `airwallex` สำหรับ webhook ของ Airwallex

ตัวอย่าง:
- `https://shop.example.com/api/payout-providers/paypal/webhook/`
- `https://shop.example.com/api/payout-providers/airwallex/webhook/`

### การตั้งค่า webhook ของ PayPal

1. **เข้าไปที่** [PayPal Developer Dashboard](https://developer.paypal.com/dashboard/)
2. **คลิก** ชื่อแอปของคุณ
3. **เลื่อนลงไปที่** หัวข้อ **Webhooks**
4. **คลิก** **Add Webhook**
5. **ใส่ URL webhook** (รูปแบบด้านบน)
6. **เลือกเหตุการณ์**:
   - `PAYMENT.PAYOUTSBATCH.SUCCESS`
   - `PAYMENT.PAYOUTSBATCH.DENIED`
   - `PAYMENT.PAYOUTS-ITEM.SUCCEEDED`
   - `PAYMENT.PAYOUTS-ITEM.FAILED`
7. **คลิก Save**

PayPal ให้คีย์การลงนาม webhook Spwig ใช้คีย์นี้เพื่อตรวจสอบความถูกต้องของ webhook

### การตั้งค่า webhook ของ Airwallex

1. **เข้าไปที่** [Airwallex Dashboard](https://www.airwallex.com/app/)
2. **ไปที่** **Settings > Webhooks**
3. **คลิก** **Create Webhook**
4. **ใส่ URL webhook** (รูปแบบด้านบน)
5. **เลือกเหตุการณ์**:
   - `transfer.created`
   - `transfer.completed`
   - `transfer.failed`
6. **คลิก Create**

Airwallex ลงนาม webhook ด้วยคีย์ลับของคุณ

### ความปลอดภัยของ webhook

webhook ได้รับการตรวจสอบความถูกต้องโดยใช้กลไกต่อไปนี้:

- **การตรวจสอบลายเซ็น** — ผู้ให้บริการลงนาม payload webhook ด้วยคีย์ลับ
- **การตรวจสอบเวลา** — ปฏิเสธ webhook ที่เก่า (ป้องกันการโจมตีแบบ replay)
- **การอนุญาต IP (ตัวเลือก)** — จำกัดเฉพาะช่วง IP ของผู้ให้บริการ
- **ต้องใช้ HTTPS** — webhook ทำงานได้เฉพาะผ่าน SSL

อย่าปิดการตรวจสอบลายเซ็นในโหมด Production

### การทดสอบ webhook

ผู้ให้บริการส่วนใหญ่มีเครื่องมือทดสอบ webhook:

**PayPal**: ใช้ "Simulator" ในแดชบอร์ดผู้พัฒนาเพื่อส่ง webhook ทดสอบ

**Airwallex**: สร้างการโอนเงินทดสอบในโหมด Demo และสังเกตการแจ้ง webhook

คุณยังสามารถตรวจสอบบันทึก webhook ใน Spwig ที่ **Settings > System Logs** (หากเปิดใช้งานการบันทึก)

## การแก้ไขปัญหา

### ข้อผิดพลาดการรับรองไม่ถูกต้อง

**อาการ**: ได้รับข้อความ "Authentication failed" เมื่อบันทึกบัญชีผู้ให้บริการ

**สาเหตุ**:
- Client ID หรือ Secret ผิด
- ใช้คีย์ Sandbox ในโหมด Production (หรือตรงกันข้าม)
- คีย์ API หมดอายุหรือถูกยกเลิก
- บัญชีไม่ได้รับการยืนยัน

**วิธีแก้ไข**:
- คัดลอกข้อมูลการรับรองใหม่จากแดชบอร์ดผู้ให้บริการ
- ตรวจสอบว่าโหมดตรงกัน (Sandbox vs Production)
- สร้างคีย์ API ใหม่
- ติดต่อฝ่ายสนับสนุนผู้ให้บริการเพื่อตรวจสอบสถานะบัญชี

### ไม่ได้รับ webhook

**อาการ**: การจ่ายเงินติดอยู่ในสถานะ "Processing" เป็นเวลานาน

**สาเหตุ**:
- ไม่ได้ตั้งค่า URL webhook ในแดชบอร์ดผู้ให้บริการ
- ใบรับรอง SSL ไม่ถูกต้อง
- ไฟร์วอลล์บล็อก IP ของผู้ให้บริการ
- การตรวจสอบลายเซ็น webhook ล้มเหลว

**วิธีแก้ไข**:
- ตรวจสอบ URL webhook ในการตั้งค่าผู้ให้บริการอีกครั้ง
- ตรวจสอบว่าใบรับรอง SSL ถูกต้อง
- เพิ่มช่วง IP ของผู้ให้บริการในไฟร์วอลล์
- ตรวจสอบบันทึก Celery สำหรับข้อผิดพลาดลายเซ็น
- ทดสอบ webhook ด้วยเครื่องมือ Simulator ของผู้ให้บริการ

### การจ่ายเงินล้มเหลว

**อาการ**: สถานะการจ่ายเงินเปลี่ยนเป็น "Failed" พร้อมข้อความข้อผิดพลาด

**สาเหตุ**:
- รายละเอียดการชำระเงินของพันธมิตรไม่ถูกต้อง (อีเมลหรือบัญชีธนาคารผิด)
- ยอดเงินไม่เพียงพอในบัญชีผู้ให้บริการ
- บัญชีผู้รับไม่สามารถรับการชำระเงินได้
- ประเทศไม่ได้รับการสนับสนุน (Airwallex)
- การจ่ายเงินเกินข้อจำกัดของผู้ให้บริการ

**วิธีแก้ไข**:
- ตรวจสอบข้อผิดพลาดในฟิลด์ **Provider Response**
- ตรวจสอบว่ารายละเอียดการชำระเงินของพันธมิตรถูกต้อง
- เพิ่มเงินลงในบัญชีผู้ให้บริการ
- ขอให้พันธมิตรตรวจสอบสถานะบัญชีของพวกเขา
- ตรวจสอบการสนับสนุนประเทศและสกุลเงินของผู้ให้บริการ
- แบ่งการจ่ายเงินขนาดใหญ่หากเกินข้อจำกัด

### โหมดไม่ตรงกัน

**อาการ**: การจ่ายเงินทดสอบทำงานได้ แต่การจ่ายเงินในโหมด Production ล้มเหลว

**สาเหตุ**:
- ผู้ให้บริการตั้งค่าเป็นโหมด Sandbox แต่ใช้บัญชีพันธมิตรในโหมด Production
- คีย์ API จากสภาพแวดล้อมที่ผิด

**วิธีแก้ไข**:
- เปลี่ยนโหมดผู้ให้บริการเป็น Production
- สร้างคีย์ API สำหรับโหมด Production ใหม่
- ตรวจสอบว่า URL webhook ชี้ไปยังโดเมน Production

## แนวทางปฏิบัติที่ปลอดภัยที่สุด

ปกป้องการผสานรวมการจ่ายเงินของคุณด้วยมาตรการความปลอดภัยต่อไปนี้:

### การจัดเก็บข้อมูลการรับรอง

- **อย่าเพิ่มข้อมูลการรับรองลงในระบบควบคุมเวอร์ชัน** — ใช้ตัวแปรสภาพแวดล้อมหรือการจัดเก็บที่ปลอดภัย
- **หมุนเวียนคีย์ API ทุกไตรมาส** — สร้างคีย์ใหม่ทุก 3 เดือน
- **ใช้คีย์ที่แตกต่างกันสำหรับ Sandbox และ Production** — อย่าผสมสภาพแวดล้อม
- **จำกัดสิทธิ์ API** — อนุญาตเฉพาะการเข้าถึง Payouts ไม่ใช่การควบคุมบัญชีทั้งหมด

Spwig จัดเก็บข้อมูลการรับรองผู้ให้บริการในฐานข้อมูลแบบเข้ารหัส รักษาสำเนาสำรองของฐานข้อมูลให้ปลอดภัย

### ความปลอดภัยของ webhook

- **ตรวจสอบลายเซ็นเสมอ** — อย่าข้ามการตรวจสอบลายเซ็น
- **ใช้ HTTPS เฉพาะ** — ไม่สนับสนุน webhook ผ่าน HTTP
- **ดำเนินการอนุญาต IP** — จำกัด webhook ให้เฉพาะช่วง IP ของผู้ให้บริการ
- **บันทึก webhook ทั้งหมด** — ตรวจสอบกิจกรรมที่น่าสงสัย
- **จำกัดอัตราการใช้งาน webhook** — ป้องกันการใช้ในทางที่ผิด

### การควบคุมการเข้าถึง

- **จำกัดการเข้าถึงพนักงาน** — เพียงพนักงานที่ไว้ใจได้เท่านั้นที่ควรดำเนินการจ่ายเงิน
- **ใช้การยืนยันสองปัจจัย** — ต้องการ 2FA สำหรับบัญชีพนักงาน
- **ตรวจสอบการดำเนินการจ่ายเงิน** — ตรวจสอบว่าใครดำเนินการจ่ายเงินใด
- **แบ่งหน้าที่** — ให้พนักงานคนต่างกันรับผิดชอบการอนุมัติและดำเนินการ

### การตรวจสอบ

- **ตรวจสอบการจ่ายเงินที่ล้มเหลวทุกวัน** — แก้ไขปัญหาทันที
- **ตรวจสอบยอดเงินในบัญชีผู้ให้บริการ** — รับรองว่ามีเงินเพียงพอ
- **ตรวจสอบบันทึกธุรกรรมรายสัปดาห์** — จับความผิดปกติได้เร็วขึ้น
- **ตั้งค่าการแจ้งเตือน** — แจ้งเตือนทางอีเมลสำหรับการจ่ายเงินขนาดใหญ่หรือที่ล้มเหลว

## คำแนะนำ

- ทดสอบการผสานรวมอย่างละเอียดในโหมด Sandbox ก่อนเปลี่ยนไปใช้โหมด Production — จับปัญหาด้วยเงินปลอม
- ตั้งค่าทั้ง PayPal และ Airwallex เพื่อให้พันธมิตรเลือกวิธีการชำระเงินที่ต้องการ — พันธมิตรต่างกันมีความชอบที่แตกต่างกัน
- ตั้งค่า URL webhook ขณะตั้งค่าเบื้องต้นและตรวจสอบว่าทำงานได้ถูกต้อง — webhook เป็นสิ่งสำคัญสำหรับการอัตโนมัติ
- รักษายอดเงินในบัญชีผู้ให้บริการให้เพียงพอเพื่อหลีกเลี่ยงการจ่ายเงินที่ล้มเหลวระหว่างการประมวลผลแบบแบตช์
- ใช้ชื่อบัญชีที่อธิบายได้หากตั้งค่าผู้ให้บริการหลายราย (เช่น "PayPal USD", "PayPal EUR")
- หมุนเวียนคีย์ API ทุกไตรมาสตามแนวทางปฏิบัติความปลอดภัย
- บันทึก URL webhook และข้อมูลการรับรองในผู้จัดการรหัสผ่านที่ปลอดภัยและแบ่งปันกับทีมของคุณ
- ตรวจสอบการจ่ายเงินที่ล้มเหลวทันที — ความล่าช้าทำให้พันธมิตรไม่พอใจและทำลายชื่อเสียงของโปรแกรม
- ใช้ HTTPS เสมอสำหรับการติดตั้ง Spwig ของคุณ — webhook ต้องการใบรับรอง SSL
- ติดต่อฝ่ายสนับสนุนผู้ให้บริการหากพบข้อผิดพลาดที่ยังคงอยู่ — พวกเขาสามารถตรวจสอบสถานะบัญชีและสิทธิ์ของคุณได้
