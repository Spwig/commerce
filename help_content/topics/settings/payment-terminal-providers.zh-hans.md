支付终端提供商使您能够在POS终端上接受信用卡和借记卡支付。Stripe Terminal 是主要支持的提供商，提供现代的卡读取器（S700、WisePOS E、P400）、有竞争力的处理费率以及无缝集成。通过API凭证配置提供商账户，实时监控连接状态，并在不同地区运营时管理多个提供商。该提供商系统是可扩展的——如果Stripe Terminal在您的市场中不可用，可以通过提供商框架集成其他支付处理器。

使用支付提供商安全地接受卡支付，跟踪支付处理状态，并管理读取器在终端上的分配。

![支付提供商列表](/static/core/admin/img/help/payment-terminal-providers/provider-list.webp)

## 支付提供商概述

支付提供商是代表您的企业处理卡支付的第三方服务：

**提供商职责**：
- 实时授权卡交易
- 与物理卡读取器通信
- 处理支付安全（PCI合规性、加密）
- 将资金转入您的银行账户（结算）
- 提供交易报告和争议管理

**Spwig的角色**：
- 将支付请求路由到配置的提供商
- 存储加密的提供商凭证
- 监控连接状态
- 将读取器与终端关联
- 在订单中记录支付结果

## Stripe Terminal（主要提供商）

Stripe Terminal 是大多数商家推荐的支付提供商：

**功能**：
- 现代EMV芯片卡读取器
- 支持非接触式（NFC）支付（Apple Pay、Google Pay、轻触支付卡）
- 集成的争议管理
- 实时授权
- 开发者友好的API
- 在40多个国家/地区可用

**定价**（截至2024年，请核实当前费率）：
- 交易费用：2.7% + 每笔现场交易0.05美元（美国）
- 无月费、无设置费、无PCI合规性费
- 卡读取器硬件：一次性购买（根据型号不同，价格在59至299美元之间）

**支持的地区**：
- 美国、加拿大、英国、欧洲联盟、澳大利亚、新加坡等
- 检查Stripe的可用性：https://stripe.com/terminal

**支持的读取器**：
- BBPOS WisePOS E（一体化安卓终端）
- Stripe Reader S700（台式读取器）
- Verifone P400（旧款读取器，仍受支持）

## 设置Stripe Terminal

**步骤1：创建Stripe账户**
- 在stripe.com注册
- 完成企业验证（银行账户、税号）
- 激活支付

**步骤2：启用Stripe Terminal**
- 在Stripe仪表板中，导航到 **产品 > 终端**
- 点击 **开始使用**
- 接受终端服务条款

**步骤3：创建位置**
- Stripe Terminal需要一个“位置”来代表您的实体零售站点
- 导航到 **终端 > 位置**
- 点击 **创建位置**
- 输入商店地址和详细信息
- 保存位置ID（看起来像 `tml_1ABC123...`）

**步骤4：生成API密钥**
- 导航到 **开发者 > API密钥**
- 找到您的 **密钥**（以 `sk_live_...` 开头用于生产环境，以 `sk_test_...` 开头用于测试）
- 复制密钥（不要公开分享）

**步骤5：在Spwig中配置**
- 导航到 **POS > 支付提供商**
- 点击 **+ 添加支付提供商**
- 选择 **提供商**："Stripe Terminal"
- 输入 **API密钥**（来自步骤4）
- 输入 **位置ID**（来自步骤3）
- 保存

**步骤6：测试连接**
- 保存后，提供商状态应变为“已连接”（绿色）
- 如果状态显示“错误”（红色），请验证API密钥和位置ID
- 在提供商详细信息视图中检查错误信息

![支付提供商添加表单](/static/core/admin/img/help/payment-terminal-providers/provider-add-form.webp)

## 提供商配置字段

**提供商密钥** - 选择支付处理器：
- **stripe_terminal** - Stripe Terminal（推荐）
- **manual** - 手动支付输入（仅用于测试，不进行实际处理）
- 如果通过组件系统安装了其他提供商，可能会出现更多选项

**凭证（加密）** - 包含API凭证的JSON结构：
- 在存储之前自动加密
- 保存后永远不会以明文形式显示
- 示例结构（Stripe Terminal）：
```json
{
  "api_key": "sk_live_ABC123...",
  "location_id": "tml_1ABC123..."
}
```

**提供商设置** - 其他配置（特定于提供商）：
- 陈述描述符（显示在客户的信用卡账单上）
- 自动捕获（立即捕获授权的支付，而不是手动捕获）
- 货币覆盖（如果提供商账户使用的货币与商店不同）

**连接状态** - 实时状态指示器：
- **已连接**（绿色）- 提供商可访问且配置正确，准备处理支付
- **错误**（红色）- 连接失败或凭证无效
- **未知**（灰色）- 尚未测试（创建后立即出现）

**最后测试** - 最近连接测试的时间戳
- 在处理交易时自动更新
- 通过 **测试连接** 管理操作手动触发测试

## 连接状态监控

系统监控提供商连接，以便在客户尝试支付前提醒您出现问题：

**自动测试**：
- 每笔支付交易都会触发连接测试（必要）
- 每6小时后台作业测试连接（预防性监控）

**状态含义**：

**已连接** - 提供商API可达，凭证有效，准备处理支付

**错误** - 常见原因：
- 无效的API密钥（已撤销、过期或错误）
- 无效的位置ID（在Stripe中删除了位置，输入了错误的ID）
- 网络连接问题（防火墙阻止Stripe API）
- Stripe服务中断（罕见）

**未知** - 提供商尚未测试（新创建的账户等待首次交易）

**解决错误状态**：
1. 在提供商详细信息视图中检查错误信息（解释具体问题）
2. 在Stripe仪表板中确认API密钥是否仍然有效
3. 在Stripe仪表板中确认位置ID是否仍然存在
4. 通过 **测试连接** 管理操作手动测试连接
5. 如有必要，更新凭证

## 支持的卡读取器比较

Stripe Terminal 提供多种读取器硬件选项：

| 型号 | 类型 | 支付方式 | 显示屏 | 最佳用途 | 价格 |
|-------|------|-----------------|---------|----------|-------|
| **WisePOS E** | 一体化 | EMV芯片、NFC、刷卡 | 5英寸彩色触摸屏 | 功能齐全的零售POS | ~299美元 |
| **S700** | 台式 | EMV芯片、NFC、刷卡 | 单色LCD | 标准零售结账 | ~249美元 |
| **P400** | 台式 | EMV芯片、NFC、刷卡 | 单色LCD | 旧部署 | ~299美元 |

**WisePOS E 优势**：
- 基于安卓（可运行应用，可显示自定义内容）
- 彩色触摸屏（更好的用户体验，用于提示小费、签名捕获）
- 集成收据打印机（可选）
- 最快的交易速度

**S700 优势**：
- 比WisePOS E成本更低
- 占地面积小
- 防溅设计

**P400**（旧型号）：
- 仍受支持，但不推荐用于新部署
- 比S700/WisePOS E的芯片卡处理速度慢

所有读取器通过Stripe Terminal API连接到Spwig POS（不需要直接通过USB/蓝牙连接到POS设备）。

## 安全注意事项

**凭证加密**：
- 所有提供商凭证在数据库中以静止状态加密
- 加密使用应用程序密钥（在应用程序设置中定义）
- 凭证不会出现在日志或错误消息中

**API密钥权限**：
- 在生产环境中使用 **受限API密钥**（仅限Terminal权限）
- 不要使用无限制的密钥（不必要的广泛访问会带来安全风险）
- 在Stripe仪表板中，创建仅具有 **Terminal** 权限的受限密钥

**PCI合规性**：
- Stripe Terminal 处理PCI合规性（卡数据不会触及Spwig服务器）
- 卡号在读取器硬件上完全处理 → Stripe服务器 → 卡网络
- Spwig 仅存储支付结果（批准/拒绝），从不存储卡详细信息

**密钥轮换**：
- 按照安全最佳实践，每年轮换API密钥
- 轮换时，更新提供商配置中的凭证
- 在Stripe仪表板中撤销旧密钥，确认新密钥已生效后

## 多个提供商

一些商家需要多个提供商账户：

**多币种操作**：
- 美国商店使用Stripe美国账户（处理美元）
- 欧洲商店使用Stripe欧洲账户（处理欧元）
- 每种币种配置单独的提供商

**备用提供商**：
- 主提供商（Stripe Terminal）
- 当读取器出现故障时，备用提供商（手动输入）
- 收银员在发起支付时选择提供商

**测试与生产环境**：
- 使用 `sk_test_...` API密钥进行测试提供商
- 使用 `sk_live_...` API密钥进行生产提供商
- 测试阶段后切换提供商

## 常见问题排查

**问题1：状态显示“错误”并带有消息“无效的API密钥”**
- **原因**：API密钥被撤销或复制错误
- **解决方案**：在Stripe仪表板中生成新API密钥，更新提供商凭证，测试连接

**问题2：支付过程中未发现读取器**
- **原因**：读取器未注册到提供商的位置
- **解决方案**：在Stripe仪表板中，确认读取器已注册到提供商配置中使用的位置ID

**问题3：尽管卡有效，支付仍被拒绝**
- **原因**：Stripe账户未完全激活（验证待处理）
- **解决方案**：在Stripe仪表板中完成企业验证（银行账户、税号）

**问题4：连接状态显示“未知”且从未更新**
- **原因**：提供商从未测试过（未尝试交易）
- **解决方案**：使用 **测试连接** 管理操作手动触发连接性测试

## 小贴士

- **生产前测试模式** - 使用Stripe测试API密钥（`sk_test_...`）进行初始设置和测试
- **每个币种一个提供商** - 不要尝试用基于美元的Stripe账户处理欧元；创建单独的提供商
- **每周监控连接状态** - 主动监控可防止结账时的支付失败
- **限制API密钥权限** - 将Stripe API密钥限制为仅Terminal权限（最小特权原则）
- **记录位置ID** - 保留记录，了解哪个Stripe位置对应哪个实体商店
- **测试读取器分配** - 在提供商设置后，使用实际卡读取器测试支付以验证端到端流程
- **保持Stripe联系信息更新** - 确保Stripe中的企业联系信息与当前信息一致（对争议和合规性很重要）
