---
title: 运输规则
---

运输规则根据购物车内容、客户属性和配送区域对运输方式应用条件性费用调整，例如自动提供超过50美元的免费运输、为偏远地区添加附加费或为VIP客户提供运输折扣。规则基于优先级执行（优先级高的先执行），并带有可选的停止标志以防止进一步处理。每条规则评估多个条件（购物车金额、重量、区域、产品、客户组），并在所有条件匹配时执行六种调整类型之一。

当您需要基于订单上下文动态调整运输费用，而不是仅仅使用运输方式的静态费率时，请使用运输规则。

## 运输规则类型

运输规则应用6种类型的费用调整：

### 百分比折扣

**作用**：按百分比减少运输费用（例如，减少25%）。

**公式**：`new_cost = base_cost × (1 - percent/100)`

**示例**：
```
基础费用：20美元
折扣：25%
结果：15美元
```

**使用场景**：
- VIP客户折扣（所有运输费用减少20%）
- 节日促销（12月运输费用减少15%）
- 批量订单折扣（5件以上商品运输费用减少10%）

---

### 固定折扣

**作用**：从运输费用中减去固定金额。

**公式**：`new_cost = base_cost - amount`（最低0美元）

**示例**：
```
基础费用：15美元
折扣：5美元
结果：10美元
```

**使用场景**：
- 首次客户优惠（首次订单运输费用减5美元）
- 邮件订阅奖励（运输费用减3美元）
- 会员计划优惠（每月运输费用减10美元）

---

### 设置费用

**作用**：将运输费用覆盖为特定金额。

**公式**：`new_cost = fixed_amount`

**示例**：
```
基础费用：25美元
设置为：9.99美元
结果：9.99美元
```

**使用场景**：
- 限时促销（今天所有订单平价5美元运输）
- 分类特定运输（书籍始终为3.99美元运输）
- 基于时间的促销（本周运输费用上限为9.99美元）

---

### 免费运输

**作用**：将运输费用设置为0美元。

**公式**：`new_cost = $0`

**示例**：
```
基础费用：18美元
规则适用
结果：0美元
```

**使用场景**：
- 超过50美元免费运输
- 特定产品免费运输（促销商品）
- VIP客户免费运输
- 3件以上商品订单免费运输

---

### 附加费（固定）

**作用**：向运输费用添加固定金额。

**公式**：`new_cost = base_cost + amount`

**示例**：
```
基础费用：12美元
附加费：5美元
结果：17美元
```

**使用场景**：
- 偏远地区配送费
- 超大件物品处理费
- 周六配送附加费
- 易碎物品包装费

---

### 附加费（百分比）

**作用**：按百分比增加运输费用。

**公式**：`new_cost = base_cost × (1 + percent/100)`

**示例**：
```
基础费用：20美元
附加费：15%
结果：23美元
```

**使用场景**：
- 高峰期附加费（节假日期间增加20%）
- 快递配送溢价（附加费50%）
- 燃油附加费（根据当前费率变化）

## 规则条件

规则评估**所有条件必须通过**才能应用：

### 时间有效性

- **开始日期**：规则仅在此日期之后生效
- **结束日期**：规则仅在此日期之前生效
- **使用场景**：节日促销，限时优惠

**示例**：仅黑色星期五周末免费运输
```
开始：2026-11-27 00:00
结束：2026-11-30 23:59
```

---

### 购物车金额范围

- **最低购物车金额**：购物车小计必须≥金额
- **最高购物车金额**：购物车小计必须≤金额
- **使用场景**：免费运输门槛，分层折扣

**示例**：订单金额50-200美元免费运输
```
最低：50美元
最高：200美元
```

---

### 购物车重量范围

- **最低重量**：购物车总重量必须≥金额
- **最高重量**：购物车总重量必须≤金额
- **使用场景**：轻量货物折扣，重物附加费

**示例**：订单超过20公斤加5美元附加费
```
最低重量：20公斤
最高重量：null（无限制）
```

---

### 商品数量范围

- **最低商品数量**：购物车中商品数量必须≥数量
- **最高商品数量**：购物车中商品数量必须≤数量
- **使用场景**：批量订单折扣，单件商品费用

**示例**：5件以上商品免费运输
```
最低商品数：5
最高商品数：null
```

---

### 配送区域

- **区域**：仅当客户地址匹配至少一个选定区域时，规则才适用
- **空选择**：规则适用于所有区域
- **使用场景**：特定区域的附加费或折扣

**示例**：仅适用于国内区域的免费运输
```
区域：["Domestic USA"]
```

---

### 配送方式

- **方式**：规则仅适用于特定配送方式
- **空选择**：规则适用于所有方式
- **使用场景**：特定配送方式的促销

**示例**：快递配送25%折扣
```
方式：["Express Delivery"]
```

---

### 产品要求

**要求产品**：购物车中必须包含这些产品中的至少一个

**要求分类**：购物车中必须包含这些分类中的至少一个产品

**使用场景**：特定产品的免费运输，促销捆绑包

**示例**：购物车包含“促销商品A”时免费运输
```
要求产品：[产品ID 123]
```

---

### 产品排除

**排除产品**：如果购物车包含这些产品中的任何一个，规则不适用

**排除分类**：如果购物车包含这些分类中的任何产品，规则不适用

**使用场景**：排除重/超大件商品的免费运输

**示例**：除家具分类外免费运输
```
排除分类：[家具]
```

---

### 客户组

- **客户组**：规则仅适用于选定组的客户（VIP，批发等）
- **空选择**：规则适用于所有客户组
- **使用场景**：VIP优惠，批发折扣

**示例**：VIP会员15%运输折扣
```
客户组：["VIP"]
```

---

### 首次客户

- **首次客户**：切换以将规则限制为没有先前订单的客户
- **使用场景**：新客户欢迎优惠

**示例**：首次订单运输费用减5美元
```
首次客户：是
```

---

## 规则优先级与执行

规则按**优先级顺序**执行（数字越高，越早执行）：

### 优先级机制

**执行示例**：
```
规则A（优先级100）：购物车金额>50美元免费运输
规则B（优先级50）：所有运输10%折扣
规则C（优先级1）：偏远区域加2美元附加费

购物车：60美元，偏远区域
基础运输费用：15美元

步骤1：评估规则A（优先级100）
  购物车金额>50美元？是
  应用：设置费用为0美元
  费用现在：0美元

步骤2：评估规则B（优先级50）
  对0美元应用10%折扣
  费用现在：0美元（仍然免费）

步骤3：评估规则C（优先级1）
  向0美元添加2美元附加费
  费用现在：2美元

最终费用：2美元
```

**停止进一步规则标志**：

如果规则A有`stop_further_rules = True`：
```
规则A（优先级100，stop_further_rules=True）：购物车金额>50美元免费运输
规则B（优先级50）：10%折扣
规则C（优先级1）：2美元附加费

购物车：60美元
基础：15美元

步骤1：应用规则A，设置费用为0美元
        stop_further_rules = True → 停止

最终费用：0美元（规则B和C从未执行）
```

## 创建运输规则

**逐步工作流程**：

1. **导航到规则**
   - 设置 > 运输 > 运输规则
   - 点击“添加运输规则”

2. **基本配置**
   - **名称**：内部标识符（例如，“超过50美元免费运输”）
   - **描述**：可选备注（不显示给客户）
   - **启用**：切换以启用/禁用
   - **优先级**：设置执行顺序（100为高优先级，1为低）

3. **选择规则类型**
   - 选择调整类型（折扣百分比，固定折扣，设置费用，免费，百分比附加费，固定附加费）
   - 输入金额或百分比

4. **设置停止标志**（可选）
   - 如果此规则应阻止低优先级规则执行，请勾选“停止进一步规则”
   - 用于最终/绝对规则（例如，免费运输后不应添加附加费）

5. **定义条件**（可选 - 留空表示“始终适用”）
   - 时间有效性：开始/结束日期
   - 购物车金额：最低/最高
   - 购物车重量：最低/最高
   - 商品数量：最低/最高
   - 区域：选择适用区域
   - 方式：选择适用方式
   - 产品：要求或排除
   - 客户：组或仅首次客户

6. **保存规则**
   - 点击保存
   - 如果启用切换为“是”，规则立即生效

## 常见运输规则场景

### 场景1：超过50美元免费运输

**目标**：当购物车小计≥50美元时提供免费运输。

```

类型：免费运输
优先级：100

  最低购物车金额：50美元
停止进一步规则：是
```

  区域：["偏远地区"]

## 规则组合策略

### 策略1：叠加折扣

**允许多个折扣叠加**：
```
规则A（优先级100）：VIP客户10%折扣 → stop_further_rules=否
规则B（优先级50）：订单金额>100美元15%折扣 → stop_further_rules=否

VIP客户订单金额120美元：
基础：15美元
应用规则A后：13.50美元（10%折扣）
应用规则B后：11.48美元（15%折扣13.50美元）
```

### 策略2：独占规则

**仅一个规则适用**（最高优先级）：
```
规则A（优先级100）：订单金额>50美元免费运输 → stop_further_rules=是
规则B（优先级50）：所有运输20%折扣 → stop_further_rules=是

订单金额>50美元：
规则A适用 → 免费运输 → 停止
规则B从未执行
```

### 策略3：条件附加费

**折扣优先，附加费最后**：
```
规则A（优先级100）：订单金额>75美元免费运输
规则B（优先级75）：VIP客户15%折扣
规则C（优先级50）：一般客户10%折扣
规则D（优先级25）：偏远地区5美元附加费
规则E（优先级1）：燃油附加费10%

订单：80美元，偏远地区，VIP客户
基础：20美元
A：80美元>75美元 → 免费（0美元）
B：VIP客户 → 15%折扣0美元=0美元
C：10%折扣0美元=0美元
D：偏远地区+5美元=5美元
E：燃油+10%的5美元=5.50美元

最终：5.50美元（由于附加费，不是真正的免费）
```

**要防止这种情况，请使用stop_further_rules=是**：
```
规则A（优先级100，stop=是）：订单金额>75美元免费运输

相同订单：
A：80美元>75美元 → 免费（0美元）→ 停止
最终：0美元（真正免费）
```

3. **测试优先级解决**
   - 多个匹配规则
   - 验证最高优先级先执行
   - 检查stop_further_rules行为

4. **测试边缘情况**
   - 购物车金额正好达到门槛
   - 多个条件匹配

## 故障排除

**问题1：规则未应用**

**原因**：
- 规则未启用
- 一个或多个条件未满足
- 更高优先级规则设置stop_further_rules=是
- 时间有效性不在当前日期范围内

**解决方案**：检查所有条件，查看优先级，确认启用状态。

---

**问题2：折扣金额异常**

**原因**：
- 多个规则叠加
- 百分比应用于已折扣的费用
- 规则优先级错误

**解决方案**：检查优先级顺序，查看stop_further_rules标志，手动跟踪执行过程。

---

**问题3：免费运输未生效**

**原因**：
- 更低优先级的附加费规则在免费运输规则后添加费用
- 购物车未达到最低金额门槛
- 购物车包含排除产品

**解决方案**：在免费运输规则上使用stop_further_rules=是，验证条件，检查排除项。

## 建议

- **为免费运输使用高优先级** - 优先级100确保其在其他调整之前执行
- **为绝对规则设置stop_further_rules** - 免费运输应停止进一步处理
- **测试规则组合** - 多个规则可能意外交互
- **使用描述性名称** - “VIP 20%折扣（优先级75）”比“规则3”更好
- **记录复杂逻辑** - 在描述字段中添加注释
- **从简单规则开始** - 逐步增加复杂性
- **监控规则性能** - 检查规则是否被使用或导致混淆
- **避免过多规则** - 太多规则会减慢结账，最多使用5-10个
- **使用区域进行地理划分** - 比按国家设置多个类似规则更好
- **与配送方式结合使用** - 规则+配送方式可实现复杂定价
- **设置明确的时间窗口** - 促销时始终包含结束日期
- **测试边缘情况** - 正好50美元，正好5件商品等
