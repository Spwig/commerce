title: Form Builder 개요
---

Form Builder는 데이터 수집을 위한 맞춤형 폼을 생성합니다—연락처 폼, 설문조사, 신청서, 등록, 기타 등. 드래그 앤 드롭 필드를 사용하여 시각적으로 폼을 생성하고, 검증 규칙을 구성하고, 다단계 워크플로를 활성화하고, 상세한 분석을 통해 응답을 수집할 수 있습니다. 폼은 Page Builder 요소와 원활하게 통합되어 사이트의 anywhere에 embedding할 수 있습니다. 모든 제출은 데이터베이스에 저장되며, 분석 및 내보내기용으로 전체 메타데이터 (IP 주소, 브라우저, 완료 시간)가 포함됩니다.

고객으로부터 구조화된 데이터를 수집해야 할 때 Form Builder를 사용합니다. 간단한 연락처 정보 또는 복잡한 다단계 신청서 모두 가능합니다.

## Form Builder란 무엇인가요?

Form Builder는 코드 없이 맞춤형 폼을 생성하기 위한 시각적 드래그 앤 드롭 도구입니다:

**지원하는 폼 유형**:
- 연락처 폼 (이름, 이메일, 메시지)
- 고객 설문조사 (평점, 피드백, NPS)
- 제품 등록 (보증, 지원)
- 일자리 신청 (이력서 업로드, 다단계)
- 이벤트 등록 (참가자 정보, 선호도)
- 서비스 요청 (상세한 요구사항)
- 뉴스레터 등록 (선호도를 위한 체크박스)

**주요 기능**:
- **22개의 필드 유형** - 텍스트, 이메일, 전화, 파일 업로드, 평점, 제품 선택기 등
- **다단계 폼** - 긴 폼을 논리적인 단계로 나누고 진행 상황을 추적
- **조건부 논리** - 사용자 응답에 따라 필드를 표시하거나 숨기기
- **검증 규칙** - 필수 필드, 최소/최대 길이, 사용자 정의 정규식 패턴
- **스팸 보호** - 허니팟 필드 또는 Google reCAPTCHA v3
- **응답 분석** - 완료 시간, IP 주소, 브라우저, 추천자 추적
- **CSV 내보내기** - Excel/Google Sheets에서 분석을 위해 모든 응답 다운로드
- **다국어** - 활성화된 모든 언어로 폼 라벨 및 메시지 번역

## 첫 번째 폼 생성

**Settings > Pages > Forms**으로 이동하여 폼 관리자에 접근합니다:

**단계 1: 새 폼 생성**
- **+ 새 폼 생성** 클릭
- 폼 이름 입력 (내부 식별자, 고객에게 표시되지 않음)
- 폼 제목 입력 (폼 위에 표시되는 제목)
- 선택 사항: 설명 추가 (제목 아래에 표시되는 도움말 텍스트)

**단계 2: 필드 추가**
- **폼 디자인 편집** 클릭하여 시각적 빌더 열기
- 왼쪽 사이드바에서 필드 유형을 캔버스로 드래그
- 필드 클릭하여 오른쪽 패널에서 구성
- 라벨, 플레이스홀더, 도움말 텍스트 설정
- 필수 상태 전환
- 검증 규칙 추가

**단계 3: 폼 설정 구성**
- 제출 버튼 텍스트 설정 (기본값: "Submit")
- 성공 메시지 커스터마이징 (제출 후 표시)
- 스팸 보호 선택 (허니팟 권장)
- 필요 시 "Require Login" 전환
- 복잡한 폼을 위해 "Multi-step Form" 활성화

**단계 4: 폼 활성화**
- **Active** 상태 전환 활성화
- 활성화된 폼만 제출을 받습니다.
- 폼 저장

**단계 5: Page Builder에서 사용**
- 어떤 페이지에도 **Form** 요소 추가
- 드롭다운에서 폼 선택
- 폼은 페이지 스타일을 상속합니다.
- 제출은 백엔드로 자동 전송됩니다.

## 단일 페이지 vs 다단계 폼

**단일 페이지 폼** (기본값):
- 모든 필드 한 번에 표시
- 스크롤하여 모든 필드 확인
- 하단에 제출 버튼
- 최적 사용: 연락처 폼, 짧은 설문, 간단한 데이터 수집

**다단계 폼**:
- 필드를 번호로 표시된 단계로 구성
- 진행 상황 표시줄로 현재 단계 표시
- 뒤로/다음 탐색 버튼
- 최종 단계에서만 제출
- 선택 사항: 부분 응답 저장 (초안 모드)
- 최적 사용: 일자리 신청, 등록, 복잡한 설문, 결제 흐름

**다단계 활성화**:
1. 폼 설정에서 "Multi-step Form" 전환
2. 오른쪽 패널에서 "Steps" 탭 클릭
3. 단계 추가 (예: "개인 정보", "연락처", "선호도")
4. 필드 편집 시 단계 드롭다운을 사용하여 필드를 단계에 할당
5. 드래그하여 단계 재정렬
6. 단계 속성 설정: 제목, 설명, 건너뛸 수 있는지 여부

**다단계 이점**:
- 폼 포기 감소 (심리적: "이 페이지에는 오직 3개의 질문만 있습니다")
- 논리적 그룹화로 UX 개선
- 진행 표시줄로 완료 유도
- 긴 폼에 대한 선택적 초안 저장

## 폼 설정 설명

**기본 설정**:
- **내부 이름** - 관리자에서 폼을 식별하는 방법 (고객에게 표시되지 않음)
- **Slug** - URL 친화적인 식별자 (자동 생성, API 엔드포인트에서 사용)
- **폼 제목** - 폼 위에 표시되는 제목
- **설명** - 선택적 도움말 텍스트 (제목 아래에 표시)
- **제출 버튼 텍스트** - 버튼 라벨 커스터마이징 (예: "메시지 보내기", "지금 신청")

**메시지**:
- **성공 메시지** - 성공적인 제출 후 표시 (기본값: "제출 감사합니다!")
- **에러 메시지** - 제출 실패 시 표시 (기본값: "오류 발생. 다시 시도해 주세요.")

**보안 및 접근**:
- **Active** - 활성화된 폼만 제출을 받습니다 (비활성화된 폼은 "폼 불가능" 표시)
- **Require Login** - 인증된 사용자만 제한 (익명 사용자는 로그인 요청 표시)

**스팸 보호**:
- **None** - 보호 없음 (권장하지 않음, 봇이 스팸)
- **Honeypot Field** - 보이지 않는 필드로 봇을 포착 (대부분의 상인에게 권장)
- **Google reCAPTCHA v3** - Google에서 사이트 키 및 비밀 키 필요 (가장 강력한 보호)

**고급 기능**:
- **Multi-step Form** - 단계별 워크플로 활성화
- **Save Partial Responses** - 사용자가 진행 상황을 저장하고 나중에 다시 시작할 수 있도록 허용 (다단계만)

## 스팸 보호 옵션

**Honeypot Field (권장)**:
- 폼에 보이지 않는 필드 추가
- 봇이 채우고 (사람은 보지 못함)
- 채워진 honeypot 제출 거부
- 구성 필요 없음
- 사용자에게 CAPTCHA 불편함 없음
- 95% 이상의 스팸 봇에 효과적

**Google reCAPTCHA v3**:
- 보이지 않는 배경 점수 (0.0-1.0)
- "트래픽 라이트 클릭" 도전 없음
- 설정 필요:
  1. google.com/recaptcha/admin에서 계정 생성
  2. 사이트 키 및 비밀 키 생성
  3. 폼 빌더 설정에서 키 입력
- honeypot보다 더 견고함
- honeypot이 부족할 때 사용

**None**:
- 스팸 보호 없음
- 내부 폼 또는 테스트에만 사용
- 공개 폼은 스팸이 심할 수 있음

## 폼 응답 관리

**Settings > Pages > Forms > [Form Name] > Responses**에서 모든 제출 확인:

**응답 목록 보기**:
- 상태: 초안, 제출, 완료
- 제출자: 이메일 (로그인 시) 또는 "익명"
- IP 주소 및 위치 (GeoIP 활성화 시)
- 제출 날짜/시간
- 완료 시간 (초)

**응답 세부 정보**:
- 모든 필드 값과 라벨
- 메타데이터: 브라우저, 추천자, 언어
- 진행 상황 추적 (다단계): 현재 단계, 완료된 단계
- 액션 결과 (폼이 액션을 트리거할 경우)

**응답 필터링**:
- 폼, 상태, 날짜 범위로 필터
- 제출자 이메일 또는 IP 주소로 검색
- 제출 날짜, 완료 시간으로 정렬

**응답 내보내기**:
- **CSV로 내보내기** 버튼 클릭
- `{form-slug}_responses_{date}.csv` 다운로드
- 헤더 행: Submitted At, User, IP, Status, [필드 라벨]
- 한 응답씩 행으로 표시
- Excel, Google Sheets 또는 데이터 분석 도구에서 열기

## 페이지에서 폼 사용

**폼 embedding**:
1. Page Builder에서 페이지 열기
2. 요소 패널에서 **Form** 요소 추가
3. 드롭다운에서 폼 선택
4. 폼 컨테이너 스타일 커스터마이징 (배경, 패딩, 테두리)
5. 페이지 저장 및 게시

**폼 렌더링**:
- 폼 제목 및 설명 (폼 설정에서)
- 순서대로 모든 필드 (단일 페이지) 또는 현재 단계 (다단계)
- 커스터마이징된 텍스트를 가진 제출 버튼
- 제출 후 성공/에러 메시지

**스타일 상속**:
- 폼은 페이지 테마 스타일을 상속
- 버튼은 테마 버튼 스타일 사용
- 입력 필드는 테마 입력 스타일 사용
- 특정 스타일을 위해 필드에 사용자 정의 CSS 클래스 추가 가능

## Form Builder 인터페이스

**왼쪽 사이드바 - 필드 라이브러리**:
- 카테고리별로 정렬 (텍스트, 선택, 평점, 고급)
- 필드를 캔버스로 드래그하거나 클릭하여 추가
- 필드 유형을 빠르게 찾기 위해 검색

**메인 캔버스 - 필드 편집기**:
- 드래그 핸들 (≡)으로 필드 순서 변경
- 필드 클릭하여 선택 및 편집
- 각 필드에 삭제 버튼 (×)
- 구성된 필드의 시각적 미리보기
- 빈 상태 및 드롭 존 지시사항

**오른쪽 사이드바 - 속성 패널**:
- **폼 설정 탭** - 기본 정보, 메시지, 스팸 보호
- **필드 설정 탭** - 선택된 필드 구성 (라벨, 검증 등)
- **단계 탭** - 단계 관리 (다단계 폼만)
- **조건부 규칙 탭** - 응답에 따라 표시/숨기기 논리 추가

**도구 모음 기능**:
- **되돌리기/재실행** - 전체 편집 기록
- **미리보기** - 폼 기능 테스트
- **저장** - 편집 중 3초마다 자동 저장
- **번역** - 다른 언어로 폼 텍스트 번역

## 일반적인 폼 예시

**연락처 폼**:
- 필드: 전체 이름 (필수), 이메일 (필수), 전화, 메시지 (필수)
- 제출 버튼: "메시지 보내기"
- 성공: "우리와 연락 주셔서 감사합니다! 24시간 이내에 답변 드리겠습니다."

**제품 피드백 설문**:
- 단계 1: 별점 평가, 리커트 척도 동의
- 단계 2: NPS 점수, 개선 제안
- 조건: 평점 < 3인 경우 개선 피드백 필수

**일자리 신청**:
- 단계 1: 개인 정보 (이름, 이메일, 전화)
- 단계 2: 경험 (이력서 업로드, 경력 연수, 추천자)
- 단계 3: 가용성 (시작 날짜, 급여 기대치)
- 부분 저장 활성화 (신청자가 나중에 다시 시작 가능)

**선호도를 포함한 뉴스레터 등록**:
- 이메일 (필수)
- 체크박스 그룹: 관심사 (제품, 판매, 블로그 업데이트)
- reCAPTCHA 활성화 (가짜 등록 방지)

## 팁

- **단일 페이지부터 시작** - 필드가 10개 이상인 경우에만 다단계 추가
- **먼저 honeypot 사용** - 스팸이 지속되면 reCAPTCHA로 업그레이드
- **게시 전 테스트** - 미리보기 모드로 검증 및 흐름 확인
- **정기적으로 내보내기** - 주간으로 응답 CSV 다운로드하여 백업
- **완료 시간 모니터링** - 평균이 5분 이상인 경우 폼이 너무 길 수 있음
- **조건부 논리 사용** - 관련 없는 필드 숨기기로 폼 길이 인식 감소
- **긴 폼에 부분 저장 활성화** - 다단계 신청에서 포기 감소
- **폼 라벨 번역** - 다국어 사이트를 위해 내장 번역 시스템 사용
- **민감한 데이터에 로그인 요구** - 익명 스팸 방지 및 제출을 사용자 계정과 연결
- **성공 메시지 구체화** - "24시간 이내에 답변 드리겠습니다"가 "감사합니다"보다 더 좋음
