---
title: 员工角色与权限
---

员工角色可让您精确控制每位团队成员在管理面板和POS终端中可以看到和可以做的事情。定义具有特定权限的角色，然后将其分配给员工。一个用户可以拥有多个角色，其有效权限是所有分配角色的组合。

![员工角色](/static/core/admin/img/help/staff-roles/role-list.webp)

## 工作原理

1. 您创建**角色**，定义一组权限（例如，"订单管理员"，"收银员"）
2. 每个角色控制两种类型的访问权限：**管理面板权限**和**POS权限**
3. 您从员工的个人资料页面**分配角色**给员工
4. 员工的有效权限是其**所有角色的并集**——如果任何角色授予访问权限，用户就拥有该权限
5. 权限**被缓存**以提高性能，并在角色更改时自动刷新

## 预定义角色

Spwig包含7个内置角色，涵盖最常见的团队结构。这些角色不能被删除，但您可以为更具体的需求创建自定义角色。

| 角色 | 访问权限 | 描述 |
|------|--------|-------------|
| **商店所有者** | 管理 + POS | 完全访问所有内容。适用于主要商店管理员。 |
| **商店经理** | 管理 + POS | 日常运营——对产品、订单、客户、营销和搜索有完全访问权限。设计、电子邮件、支付和设置仅限查看。 |
| **内容编辑器** | 管理 | 管理页面、博客文章、设计和媒体。对产品仅限查看。 |
| **订单管理员** | 管理 | 处理订单、发货、退货和客户服务。对产品仅限查看。 |
| **营销经理** | 管理 | 管理促销、优惠券、联盟、忠诚度和推荐计划。对产品、客户和媒体仅限查看。 |
| **收银员** | 仅POS | 前线POS员工。可以处理销售和检查礼品卡余额。无折扣、退款或现金管理权限。 |
| **高级收银员** | 仅POS | 有经验的POS员工。可以处理退款，应用折扣（最高25%），管理现金和关闭班次。 |

## 创建自定义角色

导航到 **设置 > 员工角色** 并点击 **添加角色**。

### 一般设置

| 设置 | 描述 |
|---------|-------------|
| **显示名称** | 在管理面板中显示的角色名称（例如，"仓库员工"） |
| **描述** | 说明此角色的用途 |
| **排序顺序** | 控制角色列表中的显示顺序 |
| **图标** | 从20个图标中选择以直观识别角色 |
| **徽章颜色** | 用于角色徽章的颜色（蓝色、绿色、橙色、红色、青色、灰色） |
| **管理面板** | 切换此角色是否授予管理后台访问权限 |
| **POS终端** | 切换此角色是否授予POS终端访问权限 |

### 管理权限类别

管理权限标签将所有平台功能组织到13个类别中。对于每个类别，您可以设置三种访问级别之一：

- **无** — 没有访问权限（菜单项隐藏）
- **查看** — 只读访问（可以看到数据但不能更改）
- **完整** — 完整访问（可以查看、创建、编辑和删除）

![权限类别](/static/core/admin/img/help/staff-roles/permission-categories.webp)

| 类别 | 控制内容 |
|----------|-----------------|
| **产品目录** | 产品、分类、品牌、属性、库存、仓库、数字资产 |
| **订单与履约** | 订单、退款、退货、发货、运输配置 |
| **客户** | 客户资料、客户群、分析 |
| **内容与页面** | 页面、博客文章、公告、表单 |
| **设计与主题** | 主题、页眉/页脚模板、菜单、设计标记、自定义CSS |
| **营销与促销** | 促销、优惠券、联盟、忠诚度、推荐、产品馈送 |
| **媒体库** | 图片、视频、文件夹、标签 |
| **电子邮件系统** | 电子邮件账户、模板、投递队列 |
| **支付与账单** | 支付提供商、交易、网络钩子、订阅、汇率 |
| **搜索** | 搜索设置、同义词、重定向、分析 |
| **商店设置** | 网站设置、地理定位、国家映射、业务规则 |
| **POS管理** | POS终端、班次、现金流动、收据模板 |
| **用户与角色** | 员工用户账户、角色、API令牌 |

当用户拥有多个角色时，**最高**的访问级别生效。例如，如果角色A对产品授予"查看"，而角色B授予"完整"，用户将获得"完整"访问权限。

### POS权限标志

如果角色授予POS访问权限，POS权限标签允许您精确调整POS操作员可以执行的操作。这些权限与管理权限分开，并在POS终端进行检查。

![POS权限](/static/core/admin/img/help/staff-roles/pos-permissions.webp)

| 组 | 权限 | 描述 |
|-------|-----------|-------------|
| **通用** | POS访问 | 可以使用所有POS系统 |
| **销售与折扣** | 手动折扣 | 可以应用手动的项目级或购物车级折扣 |
|  | 最大折扣% | 允许的最高折扣百分比（0–100） |
|  | 价格覆盖 | 可以在收银机上覆盖产品价格 |
| **退款与作废** | 处理退款 | 可以处理POS订单的退款 |
|  | 作废订单 | 可以从当前班次中作废POS订单 |
| **礼品卡** | 发行礼品卡 | 可以在收银机上发行新的礼品卡 |
|  | 检查礼品卡余额 | 可以查询礼品卡余额 |
| **现金管理** | 现金管理 | 可以执行现金存入和现金取出操作 |
|  | 打开钱箱 | 可以在没有销售的情况下打开钱箱 |
|  | 关闭班次 | 可以关闭班次并执行现金对账 |
| **报告** | 查看POS报告 | 可以查看班次报告和销售摘要 |
| **库存** | 库存调整 | 可以调整库存水平（接收、损坏、重新计数、退货） |

对于布尔权限，如果**任何**用户的角色启用了它，用户就拥有该权限。对于最大折扣%，所有角色中的**最高**值适用。

## 管理员工

导航到 **设置 > 员工管理** 以查看和管理您的团队。

### 员工列表

员工列表显示所有具有员工访问权限的用户。对于每个成员，您可以查看：
- **姓名和电子邮件**
- **分配的角色**（显示为彩色徽章）
- **访问类型** — 仅管理、仅POS或两者
- **2FA状态** — 是否启用了双因素认证
- **激活/停用**状态

使用过滤器按角色、访问类型或2FA状态进行筛选。

### 为员工分配角色

1. 点击员工以打开其个人资料
2. 在**角色**部分，您会看到每个可用角色的卡片
3. 点击任何角色卡片上的切换按钮以分配或移除它
4. 更改立即生效——不需要保存按钮
5. 下面的**有效权限**摘要显示了所有分配角色的综合结果

### 添加新员工

1. 导航到 **设置 > 员工管理** 并点击 **添加员工**
2. 输入用户的电子邮件、名和姓
3. 设置一个临时密码
4. 分配一个或多个角色
5. 用户现在可以使用其角色提供的访问权限登录

## 克隆角色

要基于现有角色创建新角色：

1. 打开您要复制的角色
2. 在页面底部点击 **克隆角色**
3. 创建一个具有相同所有权限的新角色
4. 重命名它并根据需要调整权限
5. 保存新角色

当您需要一个与现有角色相似但有细微差别的角色时，这很有用——例如，基于"商店经理"的"初级经理"，但权限较少。

## 权限如何应用

### 管理面板

- **菜单可见性** — 对于用户具有"无"访问权限的类别，隐藏侧边栏部分
- **页面访问** — 访问受限页面会显示权限错误
- **操作限制** — 具有"查看"访问权限时，隐藏编辑和删除按钮并阻止保存操作
- **超级用户绕过** — 超级用户账户始终拥有完整访问权限，无论角色分配如何

### POS终端

- **登录门禁** — 只有拥有至少一个启用了"POS终端"的角色的用户才能登录到POS
- **功能开关** — POS按钮和操作（退款、折扣、作废等）根据用户合并的POS权限显示或隐藏
- **折扣上限** — 最大折扣%强制对POS操作员可应用的折扣大小施加硬性限制
- **API执行** — 所有POS权限在API层服务器端进行检查，而不仅仅是在UI中

## 小贴士

- **从预定义角色开始** — 7个内置角色涵盖了大多数团队结构。只有在需要更具体的访问控制时才创建自定义角色。
- **使用克隆功能** — 当您需要一个与现有角色相似的角色时，克隆它并进行调整，而不是从头开始构建。
- **需要时分配多个角色** — 处理订单和营销的员工可以同时分配"订单管理员"和"营销管理员"角色。权限会自动合并。
- **分离管理与POS访问** — 收银员通常不需要管理访问权限，办公室员工不需要POS访问权限。使用访问开关保持整洁。
- **为POS员工设置折扣限制** — 最大折扣%防止收银员应用过高的折扣。将其设置为0以禁止折扣，或为高级员工设置合理的上限，如10–25%。
- **定期审查角色** — 随着团队的发展，审查角色分配以确保员工拥有完成工作所需的最低访问权限。当人员职位变动时，移除相应角色。
- **为敏感角色启用2FA** — 允许访问支付、设置或用户管理的员工应启用双因素认证以确保安全。
