title: การจัดการเครื่อง POS

การจัดการเครื่อง POS เป็นรากฐานของการดำเนินงานค้าปลีกของคุณ แต่ละเครื่องแทนอุปกรณ์ทางกายภาพ (แท็บเล็ต คอมพิวเตอร์ หรือฮาร์ดแวร์ POS ที่ใช้เฉพาะ) ที่พนักงานใช้ในการดำเนินการขาย กำหนดค่าเครื่องต่าง ๆ ด้วยการกำหนดคลังสินค้า การอนุญาตให้ใช้งานพนักงาน การผสานรวมฮาร์ดแวร์ และการตั้งค่าการซิงค์แบบออฟไลน์ ติดตามสถานะเครื่องแบบเรียลไทม์ด้วยระบบ heartbeat และปลดล็อกเครื่องจากระยะไกลเมื่อเกิดปัญหา การจัดการเครื่องที่เหมาะสมจะช่วยให้การดำเนินงานในร้านค้าเป็นไปอย่างราบรื่นและป้องกันการขัดแย้งของค่ากำหนดในแต่ละสาขา

ไปที่ **POS > เครื่อง POS** เพื่อลงทะเบียนเครื่องใหม่ ดูสถานะออนไลน์/ออฟไลน์ และจัดการการตั้งค่าเครื่องทั้งหมด

![รายการเครื่อง POS](/static/core/admin/img/help/managing-pos-terminals/terminal-list.webp)

## รายการเครื่อง POS

รายการเครื่องแสดงเครื่องที่ลงทะเบียนทั้งหมดพร้อมข้อมูลสถานะสำคัญ:

**ชื่อเครื่อง** - ป้ายชื่อที่อธิบายเครื่อง (เช่น "Checkout 1", "Main Register", "Mobile Terminal")

**UUID** - ตัวระบุที่ไม่ซ้ำกันที่สร้างขึ้นโดยอัตโนมัติเมื่อสร้าง (ใช้ภายในสำหรับการระบุอุปกรณ์)

**คลังสินค้า** - สถานที่ทางกายภาพที่กำหนดให้กับเครื่องนี้ (กำหนดว่าสินค้าใดพร้อมขายและสั่งซื้อจะถูกจัดสรรให้กับคลังสินค้าใด)

**สถานะออนไลน์** - ตัวชี้วัดสถานะที่แสดงว่าเครื่องอยู่ในสถานะออนไลน์หรือไม่:
- **จุดสีเขียว** - ออนไลน์ (ได้รับ heartbeat ภายใน 5 นาทีที่ผ่านมา)
- **จุดสีแดง** - ออฟไลน์ (ไม่มี heartbeat มากกว่า 5 นาที)
- **จุดสีเทา** - ไม่เคยคู่กับอุปกรณ์ (เครื่องถูกสร้างขึ้นแล้วแต่ไม่เคยเชื่อมต่อกับอุปกรณ์)

**Heartbeat ล่าสุด** - เวลาที่ล่าสุดที่เครื่องส่ง heartbeat (อัปเดตทุก 5 นาทีเมื่อออนไลน์)

**รหัสคู่กับอุปกรณ์** - รหัสอัลฟabet 8 ตัวที่ใช้ในการคู่กับอุปกรณ์ครั้งแรก (ซ่อนหลังจากใช้ครั้งแรก)

**ผู้ใช้ที่ได้รับการกำหนด** - จำนวนพนักงานที่ได้รับการอนุญาตให้ใช้เครื่องนี้

## การสร้างเครื่องใหม่

คลิก **+ เพิ่มเครื่อง** เพื่อลงทะเบียนอุปกรณ์ POS ใหม่:

![ฟอร์มเพิ่มเครื่อง](/static/core/admin/img/help/managing-pos-terminals/terminal-add-form.webp)

### การตั้งค่าพื้นฐาน

**ชื่อเครื่อง** - เลือกชื่อที่อธิบายได้ชัดเจนที่บ่งบอก:
- สถานที่ทางกายภาพ: "เครื่องที่เคาน์เตอร์เหนือ"
- หน้าที่: "เครื่องที่เคาน์เตอร์คืนสินค้า"
- ลำดับ: "Checkout 1", "Checkout 2", "Checkout 3"

ชื่อช่วยให้พนักงานระบุเครื่องได้ง่ายขึ้นในระหว่างการจัดสรรกะและแก้ไขปัญหา ใช้ชื่อที่มีรูปแบบที่สอดคล้องกันทั่วทั้งสาขา

**คลังสินค้า** - **จำเป็น** - เลือกคลังสินค้าที่เครื่องนี้ทำงาน:
- กำหนดว่าสินค้าใดพร้อมขาย
- สั่งซื้อที่ทำผ่านเครื่องนี้จะถูกจัดสรรให้กับคลังสินค้านี้
- การจองสินค้าจะตรวจสอบความพร้อมใช้งานในคลังสินค้าที่กำหนด
- **ไม่สามารถดำเนินการขายได้หากไม่ได้กำหนดคลังสินค้า**

หากคุณมีหลายสาขาค้าปลีก สร้างคลังสินค้าแยกต่างหากสำหรับแต่ละสาขาและกำหนดเครื่องตามลำพัง

**เปิดใช้งาน** - เปิด/ปิดเครื่องโดยไม่ต้องลบการตั้งค่า:
- เครื่องที่ไม่ได้เปิดใช้งานไม่สามารถคู่กับอุปกรณ์ได้
- ชุดข้อมูลที่มีอยู่ในเครื่องที่ไม่ได้เปิดใช้งานจะหมดอายุทันที
- ใช้เพื่อปิดใช้งานชั่วคราวเครื่องที่ถูกขโมยหรือเสียหาย

### การกำหนดผู้ใช้พนักงาน

**ผู้ใช้ที่ได้รับการกำหนด** - เลือกพนักงานที่สามารถเข้าถึงเครื่องนี้ได้:
- เพียงผู้ใช้ที่ได้รับการกำหนดเท่านั้นที่สามารถเข้าสู่ระบบเครื่องนี้ได้
- ผู้ใช้ต้องมีสิทธิ์ POS ในบทบาทพนักงานของตนเองด้วย
- การกำหนดผู้ใช้เป็นศูนย์จะทำให้เครื่องถูกปิดล็อก
- รูปแบบที่พบบ่อย: กำหนดทุกพนักงานในร้านให้เข้าถึงทุกเครื่องในร้าน

**ตัวอย่างการใช้งาน**:
- **ร้านค้าทั่วไป**: กำหนดทุกพนักงานให้เข้าถึงทุกเครื่อง (พนักงานแคชเชียร์สามารถทำงานที่เครื่องใดก็ได้)
- **ร้านค้าแบบแผนก**: กำหนดพนักงานเฉพาะแผนกให้เข้าถึงเครื่องในแผนกนั้น
- **หลายสาขา**: กำหนดพนักงานเฉพาะสาขาให้เข้าถึงเครื่องในสาขาของตนเอง
- **ผู้จัดการ**: กำหนดผู้จัดการให้เข้าถึงทุกเครื่องเพื่อการเข้าถึงแบบตรวจสอบ

ผู้ใช้ที่ไม่มีการกำหนดเครื่องจะเห็นข้อความแสดงข้อผิดพลาด "Not authorized for this terminal" เมื่อพยายามเข้าสู่ระบบ

### การตั้งค่าฮาร์ดแวร์

ฟิลด์ **Hardware Config** เป็นโครงสร้าง JSON ที่กำหนดอุปกรณ์เสริม:

**เครื่องพิมพ์ความร้อน**:
```json
{
  "printer": {
    "type": "network",
    "ip": "192.168.1.100",
    "port": 9100,
    "paper_width": 80
  }
}
```

**เครื่องสแกนบาร์โค้ดผ่าน USB**:
```json
{
  "scanner": {
    "type": "usb",
    "vendor_id": "0x05e0",
    "product_id": "0x1200"
  }
}
```

**ตู้เงินสด** (เชื่อมต่อกับเครื่องพิมพ์):
```json
{
  "cash_drawer": {
    "enabled": true,
    "trigger": "printer_pulse"
  }
}
```

**ตัวอย่างทั้งหมด**:
```json
{
  "printer": {
    "type": "network",
    "ip": "192.168.1.100",
    "port": 9100,
    "paper_width": 80
  },
  "scanner": {
    "type": "usb",
    "vendor_id": "0x05e0",
    "product_id": "0x1200"
  },
  "cash_drawer": {
    "enabled": true,
    "trigger": "printer_pulse"
  }
}
```

หากเครื่องไม่มีอุปกรณ์เสริม ให้ทิ้งว่างไว้ (เหมาะสำหรับเครื่องมือเคลื่อนที่หรือแท็บเล็ตที่ไม่มีเครื่องพิมพ์/สแกนบาร์โค้ด)

### การตั้งค่าแคชออฟไลน์

กำหนดจำนวนข้อมูลที่เครื่องแคชสำหรับการทำงานแบบออฟไลน์:

**จำนวนวันซิงค์คำสั่งซื้อ** (7-30 วัน ค่าเริ่มต้น: 14):
- จำนวนวันของคำสั่งซื้อล่าสุดที่จะแคชในท้องถิ่น
- ค่าที่สูงกว่า = มีข้อมูลประวัติมากขึ้นที่สามารถเข้าถึงได้แบบออฟไลน์
- ค่าที่ต่ำกว่า = การซิงค์เร็วขึ้น ใช้พื้นที่จัดเก็บน้อยลง
- **คำแนะนำ**: 7 วันสำหรับเครื่องที่มีปริมาณสูง 14 วันสำหรับการใช้งานปกติ 30 วันสำหรับการตรวจสอบบัญชีที่เน้น

**ขีดจำกัดการซิงค์คำสั่งซื้อ** (200-1000 คำสั่งซื้อ ค่าเริ่มต้น: 500):
- จำนวนคำสั่งซื้อสูงสุดที่จะแคชไม่ว่าช่วงเวลาจะเป็นเท่าไร
- ป้องกันการใช้พื้นที่จัดเก็บมากเกินไปในเครื่องที่มีปริมาณสูง
- **คำแนะนำ**: 200 สำหรับแท็บเล็ตที่มีพื้นที่จัดเก็บจำกัด 500 สำหรับเครื่องมาตรฐาน 1000 สำหรับอุปกรณ์ POS ที่ใช้เฉพาะ

**ข้อได้เปรียบ/ข้อเสีย**:
- **การตั้งค่าที่สูงกว่า**: การเข้าถึงข้อมูลประวัติที่ดีกว่าเมื่ออยู่ในโหมดออฟไลน์ การซิงค์เริ่มต้นช้าลง ใช้พื้นที่จัดเก็บมากขึ้น
- **การตั้งค่าที่ต่ำกว่า**: การซิงค์เร็วขึ้น ใช้พื้นที่น้อยลง ประวัติออฟไลน์จำกัด

เครื่องจะดาวน์โหลดคำสั่งซื้อ X รายการล่าสุด (ภายใน Y วัน) ในแต่ละรอบการซิงค์ หากเครื่องประมวลผลคำสั่งซื้อ 50 รายการ/วัน และ sync_days เป็น 14 คาดว่าจะมีคำสั่งซื้อประมาณ 700 รายการที่ถูกแคช (อาจถูกจำกัดโดย sync_limit)

## กระบวนการทำงานคู่กับอุปกรณ์เครื่อง POS

หลังจากสร้างเครื่อง ให้คู่กับอุปกรณ์ทางกายภาพ:

1. **สร้างรหัสคู่กับอุปกรณ์** - สร้างโดยอัตโนมัติเมื่อคุณบันทึกเครื่อง (8 ตัวอักษรที่เป็นตัวอักษรและตัวเลข)

2. **จดจำรหัส** - แสดงในรายการเครื่องและมุมมองรายละเอียด (หมดอายุหลังจากคู่กับอุปกรณ์สำเร็จครั้งแรก)

3. **ไปที่อุปกรณ์เครื่อง POS** - บนอุปกรณ์ทางกายภาพ (แท็บเล็ต/คอมพิวเตอร์) เปิดเบราว์เซอร์และไปที่: `https://yourstore.com/pos/`

4. **ป้อนรหัสคู่กับอุปกรณ์** - ป้อนรหัส 8 ตัวอักษรเมื่อถูกขอ

5. **เครื่องดาวน์โหลดการตั้งค่า** - อุปกรณ์จะได้รับ:
   - การกำหนดคลังสินค้า
   - การตั้งค่าฮาร์ดแวร์ (เครื่องพิมพ์ สแกนบาร์โค้ด ตู้เงินสด)
   - การตั้งค่าแคชออฟไลน์
   - รายการผู้ใช้ที่ได้รับการกำหนด
   - การซิงค์แคตตาล็อกสินค้าเริ่มต้น

6. **หน้าจอเข้าสู่ระบบปรากฏขึ้น** - เครื่องแสดงหน้าจอเข้าสู่ระบบสำหรับผู้ใช้ที่ได้รับการกำหนด

7. **พนักงานเข้าสู่ระบบ** - ป้อนข้อมูลผู้ใช้ที่ได้รับการกำหนดให้กับเครื่องนี้

8. **การซิงค์เริ่มต้นเสร็จสิ้น** - เครื่องดาวน์โหลด:
   - คำสั่งซื้อล่าสุด (ตาม sync_days และ sync_limit)
   - แคตตาล็อกสินค้าทั้งหมดสำหรับคลังสินค้าที่กำหนด
   - ฐานข้อมูลลูกค้า
   - การตั้งค่าโปรโมชัน

9. **เครื่องพร้อมใช้งาน** - หน้าจอ "Ready to Sell" ปรากฏขึ้นพร้อมกับแถบค้นหา

10. **รหัสคู่กับอุปกรณ์ถูกใช้แล้ว** - รหัสจะถูกลบออกจากหน้าแอดมิน สร้างรหัสใหม่หากต้องการคู่กับอุปกรณ์อีกครั้ง

**การสร้างรหัสคู่กับอุปกรณ์ใหม่**: หากคุณต้องการคู่กับอุปกรณ์อีกครั้ง (รีเซ็ตอุปกรณ์ ล้างแคชเบราว์เซอร์ ฮาร์ดแวร์ใหม่) ให้ใช้การดำเนินการแอดมิน **Regenerate Pairing Code** การดำเนินการนี้จะทำให้รหัสเก่าไม่สามารถใช้งานได้และสร้างรหัสใหม่

## การตรวจสอบสถานะเครื่อง POS

### ระบบ heartbeat

เครื่องจะส่งสัญญาณ heartbeat ไปยังเซิร์ฟเวอร์ทุก **5 นาที** ซึ่งประกอบด้วย:
- UUID ของเครื่อง
- เวลาปัจจุบัน
- จำนวนผู้ใช้ออนไลน์
- เวลาซิงค์ล่าสุด
- สถานะ Service Worker

**ตัวชี้วัดสถานะออนไลน์**:
- **สีเขียว** - ได้รับ heartbeat ภายใน 5 นาทีที่ผ่านมา (เครื่องออนไลน์และทำงานได้ปกติ)
- **สีแดง** - ไม่มี heartbeat มากกว่า 5 นาที (เครื่องออฟไลน์หรือไม่ได้เชื่อมต่อ)
- **สีเทา** - เครื่องไม่เคยคู่กับอุปกรณ์ (ไม่เคยได้รับ heartbeat)

**กรณีการใช้งาน**:
- **เปิดร้านค้า**: ตรวจสอบว่าเครื่องทั้งหมดออนไลน์ก่อนเปิดร้านค้า
- **แก้ไขปัญหา**: ระบุเครื่องที่มีปัญหาการเชื่อมต่อ
- **ตรวจสอบบัญชี**: ตรวจสอบว่าเครื่องทั้งหมดยังคงออนไลน์ในช่วงเวลาทำการ

### เวลาที่เกิด heartbeat ล่าสุด

แสดงเวลาที่แน่นอนของ heartbeat ล่าสุด ใช้เพื่อ:
- ระบุว่าเครื่องออฟไลน์มานานแค่ไหน
- ระบุรูปแบบ (เช่น เครื่องออฟไลน์ทุกคืนเมื่อปิดร้าน)
- ตรวจสอบความถี่ในการซิงค์ (ควรอัปเดตทุก ~5 นาทีเมื่อออนไลน์)

## คุณสมบัติการปลดล็อกจากระยะไกล

เมื่อเครื่องกลายเป็นไม่ตอบสนองหรือติดอยู่ที่หน้าจอ (เกิดข้อผิดพลาดของซอฟต์แวร์ ปัญหาการหมดอายุของเซสชัน หรือเบราว์เซอร์ติด) ให้ใช้การดำเนินการแอดมิน **Remote Unlock**:

**วิธีการทำงาน**:
1. เลือกเครื่องที่มีปัญหาในรายการแอดมิน
2. เลือก **Remote Unlock** จากเมนูการดำเนินการแอดมิน
3. ยืนยันการดำเนินการ
4. เซิร์ฟเวอร์ส่งสัญญาณปลดล็อกผ่านการตอบสนอง heartbeat
5. เครื่องจะได้รับสัญญาณในรอบ heartbeat ถัดไป (<5 นาที)
6. เครื่องจะทำการล็อกเอาต์ผู้ใช้ปัจจุบันโดยบังคับและกลับไปที่หน้าจอเข้าสู่ระบบ

**เมื่อใช้**:
- เครื่องติดอยู่ที่หน้าจอการทำธุรกรรม
- พนักงานไม่สามารถออกจากระบบได้ (ปุ่มออกจากระบบไม่ตอบสนอง)
- เซสชันดูเหมือนยังคงใช้งานอยู่แต่เครื่องไม่ตอบสนอง
- เบราว์เซอร์ล่มแต่คุกกี้เซสชันยังคงอยู่

**สำคัญ**: การปลดล็อกจากระยะไกลไม่ได้รีสตาร์ทอุปกรณ์หรือเบราว์เซอร์ มันเพียงแค่บังคับให้ล็อกเอาต์และล้างเซสชัน หากเครื่องติดอยู่ทั้งหมด พนักงานอาจต้องรีสตาร์ทเบราว์เซอร์หรืออุปกรณ์ด้วยตนเอง

## การแก้ไขการตั้งค่าเครื่อง POS

คลิกเครื่องในรายการเพื่อแก้ไขการตั้งค่า:

![ฟอร์มแก้ไขเครื่อง](/static/core/admin/img/help/managing-pos-terminals/terminal-edit-form.webp)

**สามารถแก้ไขได้ขณะที่เครื่องออนไลน์**:
- ชื่อเครื่อง
- ผู้ใช้ที่ได้รับการกำหนด
- การตั้งค่าฮาร์ดแวร์ (มีผลหลังจากเครื่องรีสตาร์ทแอป)
- การตั้งค่าแคชออฟไลน์ (มีผลเมื่อซิงค์ครั้งต่อไป)

**ต้องคู่กับอุปกรณ์อีกครั้ง**:
- การกำหนดคลังสินค้า (การเปลี่ยนคลังสินค้าต้องการคู่กับอุปกรณ์อีกครั้งเพื่อซิงค์สินค้าใหม่)

**ไม่สามารถแก้ไขได้**:
- UUID (ตัวระบุที่ไม่สามารถเปลี่ยนแปลงได้)

การเปลี่ยนแปลงต่อการตั้งค่าส่วนใหญ่จะมีผลในรอบ heartbeat/sync ถัดไป การเปลี่ยนแปลงการตั้งค่าฮาร์ดแวร์ต้องให้พนักงานปิดและเปิดแอป POS อีกครั้ง (หรือรีเฟรชเบราว์เซอร์)

## การแก้ไขปัญหาทั่วไป

**เครื่องแสดงข้อความ "Not Authorized" เมื่อเข้าสู่ระบบ**:
- ตรวจสอบว่าผู้ใช้อยู่ในรายการ **ผู้ใช้ที่ได้รับการกำหนด** สำหรับเครื่องนี้
- ตรวจสอบว่าผู้ใช้มีสิทธิ์ POS ใน **Staff & Permissions > Roles**
- ตรวจสอบว่าเครื่องมีการตั้งค่า **เปิดใช้งาน**

**เครื่องไม่สามารถคู่กับอุปกรณ์ได้ (รหัสไม่ถูกต้อง)**:
- รหัสคู่กับอุปกรณ์หมดอายุหลังจากใช้ครั้งแรก สร้างใหม่หากจำเป็น
- รหัสเป็นตัวอักษรที่ไวต่อการตั้งค่าตัวพิมพ์ใหญ่/เล็ก ตรวจสอบการตั้งค่าตัวพิมพ์ใหญ่/เล็ก
- ตรวจสอบว่าเครื่องมีการตั้งค่า **เปิดใช้งาน**

**เครื่องแสดงสถานะออฟไลน์ (จุดสีแดง)**:
- ตรวจสอบว่าอุปกรณ์มีการเชื่อมต่ออินเทอร์เน็ต
- ตรวจสอบว่าเครื่องกำลังทำงานอยู่ (เบราว์เซอร์เปิดไปที่ URL /pos/)
- ตรวจสอบว่าไฟร์วอลล์ไม่ได้บล็อกคำขอ heartbeat
- รอ 5 นาทีสำหรับรอบ heartbeat ถัดไป

**เครื่องซิงค์ช้า**:
- ลด **จำนวนวันซิงค์คำสั่งซื้อ** จาก 30 เป็น 7
- ลด **ขีดจำกัดการซิงค์คำสั่งซื้อ** จาก 1000 เป็น 200
- ตรวจสอบความเร็วเครือข่ายที่เครื่องอยู่
- ตรวจสอบว่าเซิร์ฟเวอร์ไม่ได้รับโหลดหนักเกินไป

**เครื่องพิมพ์ไม่ทำงาน**:
- ตรวจสอบ IP และพอร์ตของเครื่องพิมพ์ใน **Hardware Config**
- ทดสอบการเชื่อมต่อเครื่องพิมพ์จากอุปกรณ์เครื่อง (ping IP address)
- ตรวจสอบว่าเครื่องพิมพ์เป็นแบบ ESC/POS
- ตรวจสอบว่าเครื่องพิมพ์เปิดและออนไลน์อยู่

## เคล็ดลับ

- **รูปแบบชื่อสำคัญ** - ใช้รูปแบบชื่อที่สอดคล้องกัน (สถานที่ + จำนวน) เพื่อให้การจัดการในระดับใหญ่เป็นไปได้ง่ายขึ้น
- **กำหนดคลังสินค้าก่อนคู่กับอุปกรณ์เสมอ** - เครื่องไม่สามารถดำเนินการขายได้หากไม่ได้กำหนดคลังสินค้า
- **ทดสอบการตั้งค่าฮาร์ดแวร์ก่อนนำไปใช้งาน** - พิมพ์ใบเสร็จทดสอบเพื่อตรวจสอบการผสานรวมเครื่องพิมพ์/ตู้เงินสด
- **ตรวจสอบ heartbeat ทุกวัน** - ตั้งค่าให้ตรวจสอบว่าเครื่องทั้งหมดออนไลน์ก่อนเปิดร้านค้า
- **ลดขีดจำกัดการซิงค์สำหรับเครื่องมือเคลื่อนที่** - แท็บเล็ตและโทรศัพท์มีประโยชน์จากการตั้งค่า sync_days: 7, sync_limit: 200
- **ใช้การปลดล็อกจากระยะไกลอย่างระมัดระวัง** - การบังคับล็อกเอาต์จะขัดขวางการทำธุรกรรมที่กำลังดำเนินอยู่ ตรวจสอบก่อนว่าเครื่องติดอยู่จริง
- **จดบันทึกรหัสคู่กับอุปกรณ์** - เขียนรหัสลงในกระดาษก่อนนำไปใช้งานที่ร้านค้า (หากการตั้งค่าใช้เวลานานกว่าที่คาดไว้)
- **กำหนดผู้จัดการให้เข้าถึงทุกเครื่อง** - ทำให้ผู้กำกับดูแลสามารถเข้าถึงเครื่องใดก็ได้เพื่อการยกเลิก การคืนเงิน และการแก้ไขปัญหา
