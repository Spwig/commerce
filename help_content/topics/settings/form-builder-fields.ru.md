---
title: Поля формы и валидация
---

Поля форм являются основными строительными блоками ваших форм — каждое поле собирает одну часть данных от пользователей. Form Builder предлагает 22 типа полей, от простых текстовых полей ввода до продвинутых шкал рейтинга и селекторов продуктов. Настройте каждое поле с помощью меток, правил валидации, справочного текста и условной логики, чтобы создать динамические формы, которые адаптируются в зависимости от ответов пользователей. Поля могут быть обязательными или необязательными, валидироваться с помощью регулярных выражений и стилизоваться с помощью пользовательских CSS-классов.

Используйте это руководство, чтобы понять все доступные типы полей, когда использовать каждый из них и как настроить валидацию и условную логику.

## Основы настройки полей

Каждое поле имеет общие настройки:

**Идентификация**:
- **Название поля** - машинное имя для хранения данных (без пробелов, используйте подчеркивания: `email_address`)
- **Тип поля** - определяет поведение ввода и отображение
- **Назначение шага** - к какому шагу относится это поле (только для многошаговых форм)

**Отображение**:
- **Метка** - вопрос или приглашение, отображаемое пользователям (например, «Какой у вас адрес электронной почты?»)
- **Подсказка** - текст-подсказка внутри поля ввода (например, «you@example.com»)
- **Справочный текст** - дополнительные инструкции под полем (например, «Мы никогда не поделимся вашей электронной почтой»)
- **Значение по умолчанию** - предварительно заполненное значение (пользователи могут его изменить)

**Макет**:
- **Ширина** - полная (100%), половина (50%) или треть (33%) ширины формы
- **CSS-класс** - дополнительные классы стилей для пользовательского дизайна
- **Порядок** - позиция внутри шага (перетаскивайте, чтобы изменить порядок)

**Валидация**:
- **Обязательное** - переключатель для обязательного статуса (появляется красная звездочка рядом с меткой)
- **Мин/макс длина** - ограничения длины (текстовые поля)
- **Мин/макс значение** - числовые ограничения (числовые поля)
- **Паттерн валидации** - пользовательское регулярное выражение для сложной валидации
- **Сообщение об ошибке** - пользовательский текст, отображаемый при неудачной валидации

## Текстовые поля ввода

**Однорядный текст** (`text`):
- Базовое текстовое поле ввода для коротких ответов
- Валидация: минимальная/максимальная длина, паттерн регулярного выражения
- Сценарий использования: имена, адреса, коды продуктов, короткие ответы
- Пример: «Полное имя», «Адрес улицы», «Название компании»

**Многострочный текст** (`textarea`):
- Расширяемая область текста для более длинного содержимого (3–10 строк)
- Валидация: минимальная/максимальная длина
- Сценарий использования: комментарии, отзывы, подробные описания, сообщения
- Пример: «Расскажите нам о вашем опыте», «Дополнительные примечания»

**Адрес электронной почты** (`email`):
- Специфическая валидация для электронной почты (требуется @ и домен)
- На мобильных клавиатурах отображается клавиша @
- Сценарий использования: контактная почта, подписка на рассылку, создание аккаунта
- Пример: «Адрес электронной почты», «Рабочая почта»

**Номер телефона** (`phone`):
- Автоматически форматирует номера телефонов
- На мобильных клавиатурах отображается числовая раскладка
- Валидация: настраиваемый паттерн (поддерживаются международные форматы)
- Сценарий использования: контактный телефон, экстренный контакт, планирование встреч
- Пример: «Номер телефона», «Мобильный», «Контактный номер»

**Число** (`number`):
- Числовое ввод с контролем увеличения/уменьшения
- Валидация: минимальное/максимальное значение, шаг увеличения
- Возвращает число (не строку) в ответах
- Сценарий использования: количества, возраст, годы опыта, суммы бюджета
- Пример: «Сколько сотрудников?», «Ваш возраст», «Годы в бизнесе»

**URL** (`url`):
- Валидация URL (требуется http:// или https://)
- На мобильных клавиатурах отображается клавиша .com
- Сценарий использования: веб-сайт, профиль LinkedIn, ссылка на портфолио
- Пример: «Веб-сайт компании», «URL портфолио»

## Поля выбора

**Список выбора** (`select`):
- Выбор одного варианта из выпадающего списка
- Настройка: массив {value, label} вариантов
- Поддерживает выбор по умолчанию
- Сценарий использования: категории, штаты/страны, выбор статуса
- Пример: «Выберите вашу страну», «Отдел», «Как вы узнали о нас?»
- Лучше всего подходит для: 5+ вариантов (для меньшего количества вариантов используйте радиокнопки)

**Радиокнопки** (`radio`):
- Единичный выбор из видимых вариантов (все варианты отображаются)
- Настройка: массив {value, label} вариантов
- Лучший UX по сравнению с списком для 2–4 вариантов
- Сценарий использования: вопросы да/нет, пол, предпочтения с небольшим количеством вариантов
- Пример: «Рекомендуете ли вы нас?», «Предпочтительный способ связи»

**Чекбокс** (`checkbox`):
- Единичный переключатель чекбокса (вкл/выкл)
- Возвращает true/false в ответах
- Сценарий использования: принятие условий, соглашения, одиночное предпочтение
- Пример: «Я согласен с условиями и положениями», «Подписаться на рассылку»

**Группа чекбоксов** (`checkbox_group`):
- Множественный выбор из вариантов (пользователи могут выбрать 0, 1 или несколько)
- Настройка: массив {value, label} вариантов
- Возвращает массив выбранных значений
- Сценарий использования: множественный выбор предпочтений, интересы, необходимые функции
- Пример: «Какие темы вас интересуют?», «Выберите все, что подходит»

## Поля рейтинга

**Рейтинг звезд** (`rating_stars`):
- Визуальная шкала рейтинга звезд (обычно 1–5 звезд)
- Настройка:
  - `max_stars`: 3–10 звезд (по умолчанию: 5)
  - `allow_half`: true/false для рейтинга с половинной звездой
  - `icon`: fa-star (по умолчанию) или fa-heart
  - `color`: код цвета в формате hex (по умолчанию: #FFD700 золотой)
- Сценарий использования: рейтинги продуктов, качество обслуживания, оценки удовлетворенности
- Пример: «Оцените ваш опыт», «Как вы оцениваете наш сервис?»

**Шкала Лайкерта** (`rating_likert`):
- Шкала оценки утверждений: «Сильно не согласен» → «Сильно согласен»
- Настройка:
  - `scale_type`: 5_point (1–5) или 7_point (1–7)
  - `labels`: настройка текста конечных точек (слева: «Сильно не согласен», справа: «Сильно согласен»)
- Возвращает числовое значение (1–5 или 1–7)
- Сценарий использования: утверждения опроса, шкалы согласия, измерение настроения
- Пример: «Продукт соответствует моим потребностям», «Обслуживание клиентов было полезным»

**Чистый показатель лояльности (NPS)** (`rating_nps`):
- Шкала 0–10: «Не очень вероятно» до «Очень вероятно»
- Настройка:
  - `low_label`: текст левой конечной точки (по умолчанию: «Не очень вероятно»)
  - `high_label`: текст правой конечной точки (по умолчанию: «Очень вероятно»)
- Возвращает значение 0–10 (0–6 = негативные отзывы, 7–8 = пассивные, 9–10 = сторонники)
- Сценарий использования: опросы NPS, вероятность рекомендации, измерение лояльности
- Пример: «Как вероятно, что вы порекомендуете нас другу?»

## Расширенные поля

**Загрузка файла** (`file`):
- Загрузка одного или нескольких файлов
- Настройка:
  - `max_size_mb`: ограничение размера файла на файл (по умолчанию: 5 МБ)
  - `allowed_types`: массив расширений (например, ["pdf", "doc", "docx", "jpg", "png"])
  - `max_files`: максимальное количество файлов (1 для одного файла, 2+ для нескольких)
- Возвращает путь(ы) файла в ответах
- Файлы хранятся в `/media/form_uploads/{form-slug}/`
- Сценарий использования: загрузка резюме, подача документов, прикрепление фотографий
- Пример: "Загрузите свое резюме", "Прикрепите подтверждающие документы"

**Выбор продукта** (`product_select`):
- Многофункциональный выбор из вашей каталога продуктов
- Настройка:
  - `category_filters`: ограничение до определенных категорий (массив идентификаторов категорий)
  - `max_selections`: 1 для одного продукта, 2+ для нескольких
  - `display_mode`: "list" (по умолчанию) или "grid" (с миниатюрами)
- Возвращает идентификаторы продуктов/SKU в ответах
- Сценарий использования: рекомендации по продуктам, списки желаний, опросы обратной связи, наборы
- Пример: "Какие продукты вас интересуют?", "Выберите ваши любимые товары"

**Дата** (`date`):
- Интерфейс выбора даты (окно календаря)
- Возвращает формат ISO (ГГГГ-ММ-ДД)
- Проверка: минимальная/максимальная дата
- Сценарий использования: даты рождения, даты событий, планирование встреч, сроки
- Пример: "Дата рождения", "Предпочтительная дата встречи"

**Время** (`time`):
- Выбор времени (часы и минуты)
- Возвращает формат времени ISO (ЧЧ:ММ)
- Сценарий использования: время встречи, окно доступности
- Пример: "Предпочтительное время", "Доступно после"

**Дата и время** (`datetime`):
- Комбинированный выбор даты и времени
- Возвращает полный формат ISO даты и времени
- Сценарий использования: планирование событий, бронирование встреч
- Пример: "Дата начала события", "Окно доставки"

## Поля макета (Не ввод)

**Заголовок раздела** (`heading`):
- Текст заголовка для организации разделов формы
- Настройка: уровень заголовка (h2, h3, h4)
- Нет сбора данных
- Сценарий использования: разделение длинных форм на логические разделы
- Пример: "Личная информация", "Контактные данные", "Предпочтения"

**Описательный абзац** (`paragraph`):
- Блок богатого текста для инструкций или информации
- Нет сбора данных
- Поддерживает базовое форматирование (жирный, курсив, ссылки)
- Сценарий использования: инструкции по шагам, юридические уведомления, объяснения
- Пример: уведомление о политике конфиденциальности, объяснение согласия по GDPR

**Разделительная линия** (`divider`):
- Визуальная горизонтальная линия-разделитель
- Нет сбора данных
- Сценарий использования: визуальная организация между разделами

**Скрытное поле** (`hidden`):
- Невидимое поле с программным значением
- Настройка: `default_value` (обязательно)
- Нет метки или текста помощи для пользователей
- Сценарий использования: параметры UTM, отслеживаемые данные, идентификаторы сеансов, реферальные коды
- Пример: скрытое поле со значением из параметра URL

## Правила проверки полей

**Обязательные поля**:
- Включите флажок "Обязательное" в настройках поля
- Красная звездочка (*) появляется рядом с меткой
- Форма не может быть отправлена, если обязательные поля пусты
- Пользовательское сообщение об ошибке: "Это поле обязательно" (или пользовательское сообщение)

**Минимальная/максимальная длина** (текстовые поля):
- Установите минимальное количество символов: предотвращает слишком короткие ответы
- Установите максимальное количество символов: предотвращает избыточный ввод
- Пример: поле сообщения требует минимум 10 символов (предотвращает ответы вроде "ок")

**Минимальное/максимальное значение** (числовые поля):
- Установите минимальное числовое значение: предотвращает отрицательные возрасты, количества
- Установите максимальное числовое значение: ограничивает ввод в разумный диапазон
- Пример: поле возраста требует минимум 18, максимум 120

**Правило проверки** (регулярное выражение):
- Пользовательское регулярное выражение для сложной проверки
- Распространенные паттерны:
  - Почтовый индекс: `^\d{5}(-\d{4})?$` (формат США)
  - Телефон: `^\(\d{3}\) \d{3}-\d{4}$` (формат США)
  - Код продукта: `^[A-Z]{2}\d{4}$` (2 буквы, 4 цифры)
- Требуется пользовательское сообщение об ошибке при использовании паттернов

**Проверка файла**:
- Максимальный размер файла: предотвращает загрузку больших файлов (по умолчанию 5 МБ)
- Разрешенные типы: белый список определенных расширений (безопасность)
- Пример: поле резюме разрешает ["pdf", "doc", "docx"], максимум 2 МБ

## Условная логика

Создание динамических форм, где поля появляются/исчезают в зависимости от ответов пользователя:

**Как работают условные правила**:
1. Пользователь отвечает на "источниковое поле" (триггер)
2. Система оценивает правило: оператор + значение сравнения
3. Если условие истинно, выполняется действие (показать/скрыть/обязательное поле или шаг)
4. Могут быть несколько правил, которые каскадируются (правило A запускает правило B)

**Доступные операторы**:
- **Равно** (`equals`): точное совпадение (например, страна равна "US")
- **Не равно** (`not_equals`): все, кроме значения
- **Содержит** (`contains`): текст включает подстроку (без учета регистра)
- **Больше чем** (`greater_than`): числовое сравнение (например, возраст > 18)
- **Меньше чем** (`less_than`): числовое сравнение (например, рейтинг < 3)
- **Пусто** (`is_empty`): поле не имеет значения
- **Не пусто** (`is_not_empty`): поле имеет любое значение
- **В списке** (`in_list`): значение является одним из ["Option1", "Option2"]

**Доступные действия**:
- **Показать поле** - отобразить скрытое поле
- **Скрыть поле** - скрыть поле (значение очищается, если скрыто)
- **Обязательное поле** - сделать поле обязательным
- **Необязательное поле** - сделать поле необязательным
- **Установить значение** - заполнить поле значением
- **Показать шаг** - отобразить скрытый шаг (только для многошаговых форм)
- **Скрыть шаг** - скрыть шаг (только для многошаговых форм)
- **Перейти к шагу** - перейти к конкретному шагу (только для многошаговых форм)

**Примеры правил**:
- ЕСЛИ `contact_method` РАВНО "phone" ТОГДА show_field `phone_number`
- ЕСЛИ `rating` МЕНЬШЕ "3" ТОГДА require_field `improvement_feedback`
- ЕСЛИ `country` В СПИСКЕ ["US", "CA"] ТОГДА show_step `shipping_details`
- ЕСЛИ `budget` БОЛЬШЕ "10000" ТОГДА show_field `enterprise_features`

**Создание условных правил**:
1. Нажмите вкладку "Conditional Rules" в правой панели
2. Нажмите "Add Rule"
3. Выберите исходное поле (триггер)
4. Выберите оператор (как сравнивать)
5. Введите значение сравнения (что сравнивать)
6. Выберите действие (что делать)
7. Выберите целевой объект (поле или шаг, на который влияет)
8. Необязательно: Установите приоритет (правила с более высоким приоритетом оцениваются первыми)
9. Сохраните правило

**Приоритет правила":
- Более высокие числа оцениваются первыми (приоритет 100 перед приоритетом 10)
- Используйте приоритет, когда правила конфликтуют или каскадируются
- Пример: Правило A (приоритет 100) отображает поле, Правило B (приоритет 50) требует его (A выполняется первым, затем B)

## Распространенные шаблоны полей

**Форма контакта":
- Полное имя (текст, обязательно)
- Email (email, обязательно)
- Телефон (телефон)
- Тема (выпадающий список с вариантами: "Продажи", "Поддержка", "Партнерство")
- Сообщение (текстовая область, обязательно, минимум 10 символов)

**Обратная связь по продукту":
- Продукт (product_select, одиночный выбор)
- Общий рейтинг (rating_stars, 5 звезд)
- Условие: ЕСЛИ рейтинг < 3 ТО требуется "Что мы можем улучшить?" (текстовая область)
- Рекомендация (rating_nps)

**Заявка на работу":
- Этап 1: Личная информация (имя, email, телефон)
- Этап 2: Резюме (загрузка файла, разрешенные форматы ["pdf", "doc"], максимум 2 МБ)
- Этап 3: Доступность (дата начала, группа флажков для рабочих дней)
- Условие: ЕСЛИ "years_experience" > 5 ТО показать поле "leadership_experience"

## Советы

- **Используйте соответствующие типы полей" - поле email для email (а не текст), обеспечивает валидацию и лучшие мобильные клавиатуры
- **Держите метки короткими" - используйте текст подсказок для деталей, а не метки
- **Группируйте связанные поля" - используйте заголовки и разделители для визуальной организации
- **Тестируйте валидацию" - предварительный просмотр формы и попытка отправки с недопустимыми данными
- **Ограничьте размеры загрузки файлов" - максимум 5 МБ предотвращает перегрузку сервера большими файлами
