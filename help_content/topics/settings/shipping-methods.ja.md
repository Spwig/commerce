---
title: 配送方法
---

配送方法は、チェックアウト時に顧客に表示される配達オプションで、それぞれの方法は異なる価格設定戦略を使用して配送費用を計算します。Spwigは、単純な固定料金から複雑なカリアー計算リアルタイム価格まで、7つの方法タイプをサポートしています。方法は最小/最大注文価値、重量、および地理的ゾーンによって制限できます。顧客はチェックアウト時に好む方法を選択し、計算された費用が注文合計に追加されます。

このガイドを使用して、基本的な固定料金配送から高度なゾーンベースの段階的な価格設定まで、ビジネスモデルに合った配送方法を構成してください。

## 配送方法の種類

Spwigは、それぞれ異なるコスト計算ロジックを持つ7つの配送方法の種類を提供しています：

### 固定料金配送

**概要**：カートの内容、目的地、または重量に関係なく固定費用。

**使用するタイミング**：
- 見通しの良い配送費用を持つ単純なストア
- 単一の製品タイプ（類似したサイズ/重量）
- 標準的なカリアー料金でのみ国内配送
- フリー配送の閾値プロモーション（配送ルールと併用）

**設定**：
- **方法タイプ** = 固定料金
- **固定費用**を入力（例：$9.99）
- オプション：最小/最大注文価値の制限を設定

**例**：すべての国内注文に対して「Standard Shipping - $9.99」。

---

### フリー配送

**概要**：顧客に無料の配送オプション（無料）。

**使用するタイミング**：
- フリー配送プロモーション
- 高価な注文（最小注文価値と併用）
- ローカルピックアップの代替
- ロイヤルティプログラムの特典

**設定**：
- **方法タイプ** = フリー配送
- オプション：**最小注文価値**を設定（例：$50以上無料）
- 配送ルールと併用して条件付き無料配送が効果的

**例**：min_order_value = $50で「$50以上の注文で無料配送」。

---

### 重量ベース配送

**概要**：カートの総重量に基づく段階的な料金表からコストが計算されます。

**使用するタイミング**：
- 変動する重量を持つ製品（本、ハードウェア、食品）
- 重量ベースのカリアー価格モデル
- 重量とコストの比率が予測可能

**設定**：
1. **方法タイプ** = 重量ベース
2. **配送料金表**を作成し、basis_type = "weight"に設定
3. **配送料金段階**を追加（例：0-5kg = $10、5-10kg = $15、10-20kg = $25）
4. オプション：特定のゾーンに制限

**例**：
```
0-2kg: $8
2-5kg: $12
5-10kg: $18
10kg+: $25
```

**動作原理**：カートが総重量を計算 → 対応する段階を検索 → 段階の料金を返す。

---

### 価格ベース配送

**概要**：カートのサブトータルに基づく段階的な料金表からコストが計算されます。

**使用するタイミング**：
- 配送費用が注文価値と関連する
- 高いカート価値を促進（上位段階ではドルあたりの料金が低い）
- 類似価格の商品に対して重量ベースの代替として単純な方法

**設定**：
1. **方法タイプ** = 価格ベース
2. **配送料金表**を作成し、basis_type = "price"に設定
3. **配送料金段階**を追加（例：$0-$50 = $9.99、$50-$100 = $14.99、$100+ = $19.99）

**例**：
```
$0-$25: $6.99
$25-$75: $9.99
$75-$150: $12.99
$150+: Free
```

**動作原理**：カートがサブトータルを計算 → 対応する段階を検索 → 段階の料金を返す。

---

### リアルタイムカリアー料金

**概要**：チェックアウト時にカリアーAPI（FedEx、UPS、DHL）からライブ料金を取得します。

**使用するタイミング**：
- 宛先によって変動する配送費用
- 顧客向けの複数のカリアー選択肢
- 手動の料金表なしで正確なカリアー価格
- 複雑な価格設定の国際配送

**設定**：
1. **方法タイプ** = リアルタイム
2. **プロバイダアカウント**を作成（設定 > 配送 > プロバイダアカウント）
3. カリアーAPI資格情報（アカウント番号、APIキー、シークレット）を入力
4. プロバイダアカウントを配送方法にリンク
5. オプション：マーカップ％または固定マーカップを追加

**要件**：
- 有効なカリアーアカウント（FedEx、UPS、DHLなど）
- カリアーからのAPI資格情報
- 配送パッケージの定義（寸法重量計算用）

**例**：チェックアウト時にカートの重量、寸法、宛先に基づいてリアルタイムのFedEx料金を取得する「FedEx Ground」方法。

**動作原理**：
1. 顧客がチェックアウト時に住所を入力
2. システムが発送元、宛先、パッケージの寸法、重量をもとにカリアーAPIを呼び出す
3. カリアーが料金見積もりを返す
4. オプションのマーカップが適用される
5. 料金が顧客に表示される

---

### ローカルピックアップ

**概要**：顧客が物理的な場所で注文をピックアップ（配送費用なし）。

**使用するタイミング**：
- ピックアップを提供する小売店
- 倉庫ピックアップオプション
- イベントや市場の屋台
- 地域の顧客の配送費用を削減

**設定**：
1. **方法タイプ** = ローカルピックアップ
2. **場所**を作成（設定 > 配送 > 場所）
   - 住所、営業時間、ピックアップ容量を設定
3. 場所を方法にリンク
4. オプション：ピックアップ準備時間を設定（例：「2時間で準備完了」）

**顧客体験**：
- チェックアウト時に「ローカルピックアップ」を選択
- 複数のピックアップ場所がある場合は場所を選択
- 有効な日時を選択
- 注文が準備完了したときに通知を受ける

**例**：3つの小売店の場所があり、24時間以内に準備完了する「Store Pickup - Free」。

---

### テーブルレート配送

**概要**：重量、価格、または数量に基づく柔軟な段階的な価格設定、および高度なゾーンターゲティング。

**使用するタイミング**：
- 複雑な価格設定（ゾーンごとに異なる料金と重量）
- 重量ベースまたは価格ベースだけでは制御が不十分な場合
- 複数の価格要因（例：重量 + 宛先 + 数量）

**設定**：
1. **方法タイプ** = テーブルレート
2. **配送料金表**を作成
3. **basis_type**を定義：重量、価格、または数量
4. **配送料金段階**を追加し、最小/最大値を設定
5. オプション：段階を特定のゾーンまたは国に制限

**重量/価格ベースとの違い**：テーブルレートは、ゾーンごとに同じ重量/価格でも異なる料金をサポートします。

**例**：
```
ゾーンA（国内）：
  0-5kg: $10
  5-10kg: $15

ゾーンB（遠方）：
  0-5kg: $18
  5-10kg: $25
```

**動作原理**：カートが基準値（重量/価格/数量）を計算 → 顧客のゾーンに一致する段階を検索 → 段階の料金を返す。

## 配送方法の設定

すべての配送方法はこれらの共通設定を共有しています：

### 基本設定

- **名前**：内部識別子（顧客には表示されません）
- **表示名**：チェックアウト時に顧客向けの名前（例：「Standard Shipping」、「Express Delivery」）
- **説明**：チェックアウト時に表示されるオプションのヘルプテキスト（例：「3-5営業日で配達」）
- **方法タイプ**：上記の7つのタイプのいずれか
- **アクティブ**：方法を有効/無効に切り替え、削除せずに

### コスト設定

- **固定費用**：固定料金方法のみ
- **料金表**：重量ベース、価格ベース、テーブルレート方法
- **プロバイダアカウント**：リアルタイムカリアー方法
- **税クラス**：配送費用に税を適用（該当する場合）

### 制限

**注文価値制限**：
- **最小注文価値**：カートサブトータルが金額以上の場合のみ方法が利用可能（例：$50以上の無料配送）
- **最大注文価値**：カートサブトータルが金額を超えると方法が非表示（例：$100未満の注文にのみ固定料金）

**重量制限**：
- **最小重量**：カート重量が金額以上の場合のみ方法が利用可能
- **最大重量**：カート重量が金額を超えると方法が非表示（軽量配送オプションで一般的）

**地理的制限**：
- **配送ゾーン**：方法を特定のゾーン（国内、国際、地域）にリンク
- 空のゾーン = すべての住所に利用可能
- 複数のゾーン = 任意の一致するゾーンに利用可能

### 高度な設定

- **優先度**：チェックアウトでの表示順（数字が小さいほどリスト上位に表示）
- **処理手数料**：計算された費用に追加される固定手数料
- **無料配送閾値**：カートサブトータルが閾値以上の場合、費用を$0に自動設定（min_order_valueの代わり）

## 配送方法の作成

**ステップバイステップワークフロー**：

1. **配送方法に移動**
   - 設定 > カート > 配送方法に移動
   - 「配送方法を追加」をクリック

2. **方法タイプを選択**
   - 価格戦略に基づき適切なタイプを選択
   - タイプがコスト設定の利用可能なフィールドを決定

3. **基本情報を設定**
   - 名前：内部参照（例："domestic_ground"）
   - 表示名：顧客向け（例："Ground Shipping"）
   - 説明：配達期間（例："5-7営業日"）

4. **コスト計算を設定**
   - **固定料金**：固定費用を入力
   - **重量/価格/テーブルレート**：料金表を作成（下記を参照）
   - **リアルタイム**：プロバイダアカウントをリンク
   - **無料/ピックアップ**：コスト設定は必要ありません

5. **制限を追加（オプション）**
   - 最小/最大注文価値
   - 最小/最大重量
   - 配送ゾーン

6. **優先度を設定**
   - 数字が小さいほどチェックアウトで先に表示される
   - おすすめの順序：無料（1）、ローカルピックアップ（2）、標準（3）、エクスプレス（4）

7. **方法を有効化**
   - 「アクティブ」を「はい」に設定
   - 保存

---

## レート表の作成

重量ベース、価格ベース、およびテーブルレート方法用：

**ステップ1：レート表を作成**
- 設定 > 配送 > レート表に移動
- 「レート表を追加」をクリック
- **名前**を設定（例："Domestic Weight Tiers"）
- **基準タイプ**を設定：重量、価格、または数量

**ステップ2：段階を追加**
- 「段階を追加」をクリック
- **最小値**と**最大値**を設定（一致する範囲）
- **料金**を設定（この段階のコスト）
- オプション：特定のゾーンまたは国に制限
- 段階を保存

**ステップ3：すべての段階を繰り返す**
- 全範囲をカバー（0から最大予想値）
- ギャップがないことを確認（例：0-5、5-10、10-20、20+）
- 最後の段階で**最大値**に`null`を使用（無制限）

**ステップ4：配送方法にリンク**
- 配送方法を編集
- ドロップダウンからレート表を選択
- 保存

**例：重量ベースの表**：
```
名前：Domestic Weight Tiers
基準：重量

段階：
1. 最小：0g、最大：2000g、料金：$8
2. 最小：2000g、最大：5000g、料金：$12
3. 最小：5000g、最大：10000g、料金：$18
4. 最小：10000g、最大：null、料金：$25
```

## 一般的な配送シナリオ

### シナリオ1：基本的な国内配送

**目的**：すべての国内注文に対して単純な$9.99固定料金。

**解決策**：
- 方法タイプ：固定料金
- 固定費用：$9.99
- 配送ゾーン："国内"（あなたの国のみ）

---

### シナリオ2：$50以上の無料配送

**目的**：無料配送の閾値で高いカート価値を促進。

**解決策オプションA**（推奨）：
- 方法タイプ：無料配送
- 最小注文価値：$50
- 表示名："Free Shipping (Orders $50+)"

**解決策オプションB**（ルールを使用）：
- 方法タイプ：固定料金
- 固定費用：$9.99
- 配送ルールを作成：
  - 条件：カート価値 ≥ $50
  - アクション：費用を$0に設定

---

### シナリオ3：国内と国際の重量ベース

**目的**：国内と国際の重量に基づいて異なる料金。

**解決策**：
1. 2つのゾーンを作成："国内"、"国際"
2. 2つの料金表を作成："国内重量"、"国際重量"
3. 2つの方法を作成：
   - "国内配送" → 国内ゾーン + 国内重量表にリンク
   - "国際配送" → 国際ゾーン + 国際重量表にリンク

---

### シナリオ4：複数のカリアー選択肢

**目的**：顧客がFedEx Ground、FedEx Express、UPS Groundのいずれかを選択可能。

**解決策**：
1. FedEx API用のプロバイダアカウントを作成
2. UPS API用のプロバイダアカウントを作成
3. 3つのリアルタイム方法を作成：
   - "FedEx Ground" → FedExプロバイダ、サービスコード = "FEDEX_GROUND"
   - "FedEx Express" → FedExプロバイダ、サービスコード = "FEDEX_EXPRESS"
   - "UPS Ground" → UPSプロバイダ、サービスコード = "UPS_GROUND"
4. すべての3つの方法はチェックアウト時にカリアーAPIをクエリし、ライブ料金を表示します。

---

### シナリオ5：ローカルピックアップ＋配送

**目的**：小売店がピックアップと配送の両方のオプションを提供。

**解決策**：
1. 住所、時間、準備時間を設定した"Main Store"場所を作成
2. 2つの方法を作成：
   - "ローカルピックアップ" → ローカルピックアップタイプ、Main Store場所にリンク
   - "標準配送" → 固定料金$9.99
3. 顧客はチェックアウト時に両方のオプションを表示します。

## 配送方法のテスト

本番環境にリリースする前に、すべての方法をテストしてください：

1. **テストカートを作成**
   - 異なる重量/価格の製品を追加
   - チェックアウトに進む

2. **各方法をテスト**
   - 異なるゾーンの住所を入力
   - 一致する方法が表示されることを確認
   - 計算された費用が予期通りであることを確認

3. **制限をテスト**
   - min_order_valueを満たすまで商品を追加 → 無料配送が表示されることを確認
   - 重い商品を追加 → 重量ベースの段階が動作することを確認
   - ゾーン制限をテスト → 除外されたゾーンで方法が非表示されることを確認

4. **リアルタイム方法をテスト**（該当する場合）
   - カリアーのテスト資格情報を使用
   - 料金が正常に返されることを確認
   - カリアーのウェブサイトと料金の正確性を確認

## トラブルシューティング

**問題1：チェックアウトで方法が表示されない**

**原因**：
- 方法が非アクティブ
- カートが最小/最大注文価値を満たしていない
- カートが最小/最大重量を満たしていない
- 顧客の住所がリンクされたゾーンと一致しない
- レート表の段階がカートの重量/価格をカバーしていない

**解決策**：制限を確認し、アクティブ状態を確認し、顧客のシナリオをカバーするゾーン/段階があることを確認してください。

---

**問題2：リアルタイム料金が失敗**

**原因**：
- 無効なAPI資格情報
- プロバイダアカウントが非アクティブ
- 配送パッケージが定義されていない（カリアーは寸法が必要）
- 出発地住所が設定されていない
- カリアーAPIが停止中

**解決策**：プロバイダ接続をテストし、資格情報を確認し、パッケージが設定されていることを確認し、設定で出発地住所を確認してください。

---

**問題3：計算された費用が不正確**

**原因**：
- レート表の段階にギャップまたは重複がある
- 段階の最小/最大値が単位が間違っている（グラム vs kg）
- 処理手数料が予期せず追加されている
- 配送ルールが費用を変更している

**解決策**：レート表の段階を確認し、単位を確認し、配送ルールの優先度を確認してください。

---

## ヒント

- **シンプルから始めましょう** - 最初の方法として固定料金を使用し、必要に応じて複雑さを追加
- **徹底的にテスト** - 本番環境で有効にする前に、すべての方法がステージング環境で動作することを確認
- **説明的な名前を使用** - "Standard Shipping (5-7 days)" は "Method 1" よりも良い
- **現実的な配達時間を設定** - 顧客満足度を高めるために、過小評価し、過大達成する
- **ピックアップを提供可能なら** - 配送費用を削減し、顧客の利便性を向上
- **カリアーAPIの信頼性を監視** - リアルタイム料金が失敗した場合に備えて、固定料金の代替手段を用意
- **国際配送にはゾーンを使用** - 高価な宛先で損失を防ぐために、地域ごとに異なる料金
- **配送ルールと組み合わせる** - ルールで条件付き論理を追加（無料配送プロモーション、遠方地域の追加料金）
- **方法を限定** - チェックアウトで2-4つのオプションを提供し、意思決定の麻痺を防ぐ
- **レート表を定期的に更新** - カリアーの料金が変動するため、年次に確認
- **優先度を賢く使用** - 無料/安価なオプションを先に、高価なオプションを最後に配置
