---
title: 配送方法
---

配送方法は、チェックアウト時に顧客に表示される配達オプションで、それぞれの方法は異なる価格設定戦略を使用して配送コストを計算します。Spwigは、単純な固定料金から複雑なキャリアによるリアルタイム価格計算まで、7種類の配送方法をサポートしています。方法は、最小/最大注文金額、重量、地理的ゾーンによって制限できます。顧客はチェックアウト時に希望の方法を選択し、計算されたコストが注文合計に追加されます。

このガイドを使用して、基本的な固定料金配送から高度なゾーンベースの段階的な価格設定まで、ビジネスモデルに合った配送方法を構成してください。

## 配送方法の種類

Spwigは、それぞれ異なるコスト計算ロジックを持つ7種類の配送方法を提供しています：

### 固定料金配送

**概要**: カートの内容、目的地、または重量に関係なく固定のコストです。

**使用するタイミング**:
- 配送コストが予測可能なシンプルなストア
- 単一の製品タイプ（類似のサイズ/重量）
- 標準的なキャリア料金での国内配送のみ
- フリーシippingキャンペーン（配送キャンペーンと併用）

**設定**:
- **方法タイプ** = 固定料金
- **固定コスト**を入力（例: $9.99）
- オプション: 最小/最大注文金額の制限を設定

**例**: 国内注文すべてに適用される「標準配送 - $9.99」。

---

### フリーシipping

**概要**: 顧客に料金を請求しないゼロコストの配送オプションです。

**使用するタイミング**:
- フリーシippingキャンペーン
- 高価な注文（最小注文金額と併用）
- ローカルピックアップの代替
- ロイヤルティプログラムの特典

**設定**:
- **方法タイプ** = フリーシipping
- オプション: **最小注文金額**を設定（例: $50以上無料）
- 配送キャンペーンと組み合わせて条件付きフリーシippingに適しています

**例**: 最小注文金額 = $50で「$50以上注文でフリーシipping」。

---

### 重量ベースの配送

**概要**: カートの総重量に基づいて段階的な料金表からコストが計算されます。

**使用するタイミング**:
- 変動する重量を持つ商品（本、ハードウェア、食品など）
- 重量に基づく配送料金モデル
- 重量とコストの予測可能な比率

**設定**:
1. **方法タイプ**をWeight-Basedに設定
2. basis_type = "weight"とした**配送料金表**を作成
3. **配送料金段階**を追加（例: 0-5kg = $10, 5-10kg = $15, 10-20kg = $25）
4. 省略可能: 特定のゾーンに限定

**例**:
```
0-2kg: $8
2-5kg: $12
5-10kg: $18
10kg+: $25
```

**仕組み**：カートが総重量を計算 → 対応する段階を検索 → 段階の料金を返す。

---

### 価格ベースの配送

**何であるか**：カートの小計に基づいて段階的な料金表からコストを計算。

**使用するタイミング**:
- 配送コストが注文金額と関連する
- 高いカート金額を促進（上位段階ではドルあたりの料金が低い）
- 価格が類似している商品に対して、重量ベースの代替としてシンプル

**設定**:
1. **方法タイプ**をPrice-Basedに設定
2. basis_type = "price"とした**配送料金表**を作成
3. **配送料金段階**を追加（例: $0-$50 = $9.99, $50-$100 = $14.99, $100+ = $19.99）

**例**:
```
$0-$25: $6.99
$25-$75: $9.99
$75-$150: $12.99
$150+: 無料
```

**仕組み**：カートが小計を計算 → 対応する段階を検索 → 段階の料金を返す。

---

### 実時間の配送業者料金

**何であるか**：チェックアウト時に配送業者のAPI（FedEx、UPS、DHLなど）からライブ料金を取得。

**使用するタイミング**:
- 宛先によって変動する配送コスト
- 顧客向けの複数の配送業者オプション
- 手動の料金表なしで正確な配送業者料金
- 複雑な料金体系を持つ国際配送

**設定**:
1. **方法タイプ**をReal-Timeに設定
2. **プロバイダーアカウント**を作成（**設定** > **配送** > **プロバイダーアカウント**）
3. 配送業者のAPI資格情報を入力（アカウント番号、APIキー、シークレット）
4. プロバイダーアカウントを配送方法にリンク
5. 省略可能: マーカップ％または固定マーカップを追加

**要件**:
- アクティブな配送業者アカウント (FedEx, UPS, DHL など)
- 配送業者から取得した API 認証情報
- 配送パッケージの定義 (寸法重量計算用)

**例**: "FedEx Ground" メソッドは、チェックアウト時にカートの重量、寸法、目的地に基づいてリアルタイムの FedEx 料金を取得します。

**仕組み**:
1. 顧客がチェックアウト時に住所を入力
2. システムが原点、目的地、パッケージの寸法、重量をもとに配送業者の API を呼び出します
3. 配送業者が料金を返します
4. 必要に応じてマーカップが適用されます
5. 料金が顧客に表示されます

---

### ローカルピックアップ

**概要**: 顧客が物理的な場所で注文をピックアップ (配送費用なし)

**使用するタイミング**:
- ピックアップを提供する小売店
- 倉庫のピックアップオプション
- イベントやマーケットストalls
- 地域の顧客に配送費用を省く

**設定**:
1. **メソッドタイプ** = ローカルピックアップ に設定
2. **ロケーション** を作成 (設定 > 配送 > ロケーション)
   - 住所、営業時間、ピックアップ容量を設定
3. メソッドにロケーションをリンク
4. 必要に応じてピックアップ準備時間を設定 (例: "2時間で準備完了")

**顧客体験**:
- チェックアウト時に "ローカルピックアップ" を選択
- 複数のピックアップ場所がある場合は場所を選択
- 有効な日時を選択
- 注文が準備完了したときに通知を受け取る

**例**: "店舗でのピックアップ - 無料" で、3つの小売店舗があり、24時間以内に準備完了。

---

### テーブルレート配送

**概要**: 重量、価格、または数量に基づく柔軟な段階的な価格設定、高度なゾーンターゲティング付き。

**使用するタイミング**:
- 複雑な価格設定 (ゾーンと重量による異なる料金)
- 重量ベースまたは価格ベースだけでは制御が不十分な場合
- 複数の価格要因 (例: 重量 + 宛先 + 数量)

**設定**:
1. **メソッドタイプ** = テーブルレート に設定
2. **配送料金テーブル** を作成
3. **basis_type** を定義: 重量、価格、または数量
4. 最小/最大値を持つ **配送料金段階** を追加
5. 必要に応じて段階を特定のゾーンまたは国に制限

**Weight/Price-Basedとの違い**: Table rateは、階層ごとに地理的制限をサポートし、同じ重量/価格でも異なるゾーンで異なる料金を設定できます。

**例**:
```
Zone A (国内):
  0-5kg: $10
  5-10kg: $15

Zone B (遠方):
  0-5kg: $18
  5-10kg: $25
```

**仕組み**: カートは基準値（重量/価格/数量）を計算 → 顧客のゾーンに該当する階層を検索 → 階層の料金を返します。

## 配送方法の設定

すべての配送方法は、以下の共通設定を共有しています:

### 基本設定

- **名前**: 内部識別子（顧客には表示されません）
- **表示名**: チェックアウト時に顧客に表示される名前（例: "Standard Shipping", "Express Delivery"）
- **説明**: チェックアウト時に表示されるオプションのヘルプテキスト（例: "3〜5営業日で到着"）
- **方法タイプ**: 上記の7つのタイプのいずれか
- **有効**: メソッドを有効/無効に切り替え、削除せずに

### コスト設定

- **固定コスト**: フラットレート方法のみ
- **料金表**: 重量ベース、価格ベース、テーブルレート方法用
- **プロバイダーアカウント**: 実時間のカーレル方法用
- **税クラス**: 配送コストに税を適用（該当する場合）

### 制限

**注文金額制限**:
- **最低注文金額**: カートのサブトータルが金額以上の場合のみ、この方法が利用可能（例: $50以上で無料配送）
- **最高注文金額**: カートのサブトータルが金額を超えると、この方法は非表示（例: $100未満の注文のみにフラットレートを適用）

**重量制限**:
- **最低重量**: カートの重量が金額以上の場合のみ、この方法が利用可能
- **最高重量**: カートの重量が金額を超えると、この方法は非表示（軽量配送オプションで一般的）

**地理的制限**:
- **配送ゾーン**: メソッドを特定のゾーン（国内、国際、地域）にリンク
- 空のゾーン = すべての住所に利用可能
- 複数のゾーン = 任意の一致するゾーンに利用可能

### 高度な設定

- **Priority**: チェックアウトでの表示順 (数字が小さいほどリスト上位に表示)
- **Handling Fee**: 計算されたコストに追加される固定料金
- **Free Shipping Threshold**: カートのサブトータルがしきい値以上の場合、自動的にコストを $0 に設定 (min_order_value の代替)

## シップングメソッドの作成

**ステップバイステップワークフロー**:

1. **シップングメソッドへの移動**
   - 設定 > カート > シップングメソッド にアクセス
   - "Add Shipping Method" をクリック

2. **メソッドタイプの選択**
   - 価格戦略に基づいて適切なタイプを選択
   - タイプが利用可能なコスト設定フィールドを決定

3. **基本情報の設定**
   - 名前: 内部参照 (例: "domestic_ground")
   - 表示名: 顧客向け (例: "Ground Shipping")
   - 説明: 配達期間 (例: "5-7 営業日")

4. **コスト計算の設定**
   - **Flat Rate**: 固定コストを入力
   - **Weight/Price/Table Rate**: レートテーブルを作成 (下記を参照)
   - **Real-Time**: プロバイダーアカウントをリンク
   - **Free/Pickup**: コスト設定は不要

5. **制限の追加 (任意)**
   - 最小/最大注文金額
   - 最小/最大重量
   - 配送地域

6. **Priority の設定**
   - 数字が小さいほどチェックアウトで先に表示される
   - おすすめの順序: Free (1), Local Pickup (2), Standard (3), Express (4)

7. **メソッドの有効化**
   - "Active" をトグルして Yes に設定
   - 保存

---

## レートテーブルの作成

重量ベース、価格ベース、テーブルレートメソッド用:

**ステップ1: レートテーブルの作成**
- 設定 > シップング > レートテーブル にアクセス
- "Add Rate Table" をクリック
- **名前** を設定 (例: "Domestic Weight Tiers")
- **Basis Type** を設定: weight, price, または quantity

**ステップ2: ティアの追加**
- "Add Tier" をクリック
- **Min Value** と **Max Value** を設定 (一致する範囲)
- **Rate** を設定 (このティアのコスト)
- 任意: 特定の地域または国に制限
- ティアを保存

**ステップ3: すべてのティアの追加**
- 全範囲をカバー (0 から最大予想値)
- 空間がないことを確認 (例: 0-5, 5-10, 10-20, 20+)
- 最後のティアで **Max Value** に `null` を使用 (無制限)

**ステップ4: 配送方法へのリンク**
- 配送方法の編集
- ドロップダウンからレートテーブルを選択
- 保存

**例: 重量ベースのテーブル**:
```
Name: Domestic Weight Tiers
Basis: Weight

Tiers:
1. Min: 0g, Max: 2000g, Rate: $8
2. Min: 2000g, Max: 5000g, Rate: $12
3. Min: 5000g, Max: 10000g, Rate: $18
4. Min: 10000g, Max: null, Rate: $25
```

## 一般的な配送シナリオ

### シナリオ1: 基本的な国内配送

**目的**: 国内注文すべてに $9.99 の固定料金を適用する。

**解決策**:
- メソッドタイプ: 固定料金
- 固定コスト: $9.99
- 配送ゾーン: "Domestic" (あなたの国のみ)

---

### シナリオ2: $50 以上の無料配送

**目的**: 注文金額を高めるために無料配送のしきい値を設定する。

**解決策オプションA** (推奨):
- メソッドタイプ: 無料配送
- 最低注文金額: $50
- 表示名: "Free Shipping (Orders $50+)"

**解決策オプションB** (ルールを使用):
- メソッドタイプ: 固定料金
- 固定コスト: $9.99
- 配送プロモーションを作成:
  - 条件: カート金額 ≥ $50
  - アクション: コストを $0 に設定

---

### シナリオ3: 重量ベースの国内 + 国際配送

**目的**: 国内と国際配送で重量に基づいて異なる料金を設定する。

**解決策**:
1. 2つのゾーンを作成: "Domestic", "International"
2. 2つのレートテーブルを作成: "Domestic Weight", "International Weight"
3. 2つのメソッドを作成:
   - "Domestic Shipping" → Domestic ゾーン + Domestic Weight テーブルにリンク
   - "International Shipping" → International ゾーン + International Weight テーブルにリンク

---

### シナリオ4: 複数の配送会社オプション

**目的**: 顧客が FedEx Ground、FedEx Express、UPS Ground の選択肢を提供する。

**解決策**:
1. FedEx API 用のプロバイダーアカウントを作成
2. UPS API 用のプロバイダーアカウントを作成
3. 3つのリアルタイムメソッドを作成:
   - "FedEx Ground" → FedEx プロバイダー、サービスコード = "FEDEX_GROUND"
   - "FedEx Express" → FedEx プロバイダー、サービスコード = "FEDEX_EXPRESS"
   - "UPS Ground" → UPS プロバイダー、サービスコード = "UPS_GROUND"
4. すべての3つのメソッドはチェックアウト時にカレントAPIをクエリし、ライブレートを表示する

### シナリオ5: ローカルピックアップ＋配送

**目的**: 小売店がピックアップと配送の両方のオプションを提供します。

**解決策**:
1. 位置を作成: "Main Store" に住所、営業時間、準備時間を設定
2. 2つの方法を作成:
   - "Local Pickup" → ローカルピックアップタイプ、Main Storeの位置にリンク
   - "Standard Delivery" → フラットレート $9.99
3. 顧客はチェックアウト時に両方のオプションを表示します

## 配送方法のテスト

本番環境に公開する前に、すべての方法をテストしてください:

1. **テストカートの作成**
   - 異なる重量/価格の商品を追加
   - チェックアウトへ進む

2. **各方法のテスト**
   - 異なるゾーンの住所を入力
   - 適切な方法が表示されるか確認
   - 計算されたコストが予想と一致するか確認

3. **制限のテスト**
   - 最小注文金額に達するまで商品を追加 → 無料配送が表示されるか確認
   - 重い商品を追加 → 重量に基づく階層が機能するか確認
   - ゾーン制限をテスト → 排除されたゾーンでは方法が非表示になるか確認

4. **リアルタイム方法のテスト** (適用される場合)
   - キャリアのテスト資格情報を使用
   - レートが正常に返されるか確認
   - キャリアのウェブサイトとレートの正確性を確認

## トラブルシューティング

**問題1: チェックアウト時に方法が表示されない**

**原因**:
- 方法が非アクティブ
- カートが最小/最大注文金額に達していない
- カートが最小/最大重量に達していない
- 顧客の住所がリンクされたゾーンと一致しない
- レートテーブルの階層がカートの重量/価格をカバーしていない

**解決策**: 制限を確認し、アクティブ状態を確認し、顧客の状況をカバーするゾーン/階層が設定されているか確認してください。

---

**問題2: リアルタイムレートの失敗**

**原因**:
- 無効なAPI資格情報
- プロバイダーアカウントが非アクティブ
- 配送パッケージが定義されていない (キャリアは寸法が必要)
- 出発地の住所が設定されていない
- キャリアAPIが停止中

**解決策**: プロバイダーコネクションをテストし、資格情報を確認し、パッケージが設定されているか確認し、設定で出発地住所を確認してください。

---

**問題3: 計算されたコストが不正確**

**原因**:
- レートテーブルの階層にギャップや重複がある
- 階層の最小/最大値が単位が誤っている (グラム vs kg)
- 予期せぬ処理手数料が追加されている
- 配送ルールがコストを変更している


**解決策**: 輸送料率表の階層を確認し、単位を確認し、配送プロモーションの優先順位を確認してください。

---

## ヒント

- **シンプルに始めましょう** - 最初の方法にはフラットレートを使用し、必要に応じて複雑さを追加してください
- **徹底的にテストしてください** - 本番環境で有効にする前に、すべての方法がステージング環境で動作することを確認してください
- **説明的な名前を使用してください** - "Standard Shipping (5-7 days)" は "Method 1" よりも良いです
- **現実的な納品時間を設定してください** - 顧客満足度を高めるために、過小評価して過大達成してください
- **可能であればピックアップを提供してください** - 配送コストを削減し、顧客の利便性を向上させます
- **キャリアのAPIの信頼性を監視してください** - 実際のレートが失敗した場合にフラットレートの代替手段を用意してください
- **国際配送にはゾーンを使用してください** - 地域ごとに異なる料金を設定することで、高額な配送先での損失を防ぎます
- **配送プロモーションと組み合わせて使用してください** - ルールは条件付き論理を追加します（無料配送プロモーション、遠隔地域への追加料金など）
- **方法を限定してください** - チェックアウト時に2〜4つのオプションを提供することで、意思決定のパラリスを防ぎます
- **料金表を季節ごとに更新してください** - キャリアの料金は変動するため、年次に確認してください
- **優先順位を賢く使用してください** - 無料/安価なオプションを最初に、高価なオプションを最後に配置してください
