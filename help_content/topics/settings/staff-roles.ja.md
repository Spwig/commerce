---
title: スタッフの役割と権限
---

スタッフの役割は、各チームメンバーが管理パネルとPOS端末の両方で何を表示できるか、何ができるかを正確に制御するためのものです。特定の権限を持つ役割を定義し、それをスタッフメンバーに割り当てます。ユーザーは複数の役割を持つことができ、その有効な権限はすべての割り当てられた役割の組み合わせになります。

![スタッフの役割](/static/core/admin/img/help/staff-roles/role-list.webp)

## 仕組み

1. **役割**を作成し、一連の権限を定義します（例: "注文管理者", "レジ担当"）
2. 各役割は2種類のアクセスを制御します: **管理パネルの権限**と**POSの権限**
3. **役割を割り当て**は、スタッフメンバーのプロフィールページから行います
4. スタッフメンバーの有効な権限は、すべての役割の**合併**です — 任意の役割がアクセスを許可する場合、ユーザーはそれを取得します
5. 権限は**キャッシュ**され、パフォーマンスのために、役割が変更されたときに自動的に更新されます

## 事前に定義された役割

Spwigには、最も一般的なチーム構造をカバーする7つの組み込み役割が含まれています。これらは削除できませんが、より具体的なニーズのためにカスタム役割を作成できます。

| 役割 | アクセス | 説明 |
|------|--------|-------------|
| **店舗オーナー** | 管理 + POS | すべてへのフルアクセス。主な店舗管理者用 |
| **店舗マネージャー** | 管理 + POS | 日常的な運用 — プロダクト、注文、顧客、マーケティング、検索へのフルアクセス。デザイン、メール、支払い、設定は閲覧専用 |
| **コンテンツ編集者** | 管理 | ページ、ブログ投稿、デザイン、メディアを管理。プロダクトは閲覧専用 |
| **注文管理者** | 管理 | 注文、配送、返品、カスタマーサービスを処理。プロダクトは閲覧専用 |
| **マーケティングマネージャー** | 管理 | プロモーション、割引券、アフィリエイト、ロイヤルティ、リファラルプログラムを管理。プロダクト、顧客、メディアは閲覧専用 |
| **レジ担当** | POSのみ | フロントラインのPOSスタッフ。販売処理とギフトカード残高の確認が可能。割引、返金、現金管理は不可 |
| **シニアレジ担当** | POSのみ | 経験豊富なPOSスタッフ。返金処理、割引の適用（最大25％）、現金管理、シフトの終了が可能 |

## カスタム役割の作成

**設定 > スタッフ役割**に移動し、**役割を追加**をクリックします。

### 一般的な設定

| 設定 | 説明 |
|---------|-------------|
| **表示名** | 管理パネルに表示される役割名（例: "倉庫スタッフ"） |
| **説明** | この役割の目的の簡単な説明 |
| **並び順** | 役割リストでの表示順序を制御 |
| **アイコン** | 20のアイコンから選んで、視覚的に役割を識別 |
| **バッジカラー** | 役割バッジに使用される色（青、緑、オレンジ、赤、テール、グレー） |
| **管理パネル** | この役割が管理パネルへのアクセスを許可するかどうかを切り替え |
| **POS端末** | この役割がPOS端末へのアクセスを許可するかどうかを切り替え |

### 管理権限のカテゴリ

管理権限タブは、すべてのプラットフォーム機能を13のカテゴリに整理します。各カテゴリに対して、3つのアクセスレベルのいずれかを設定します:

- **なし** — このエリアへのアクセスなし（メニュー項目は非表示）
- **閲覧** — 読み取り専用（データは見ることができるが変更はできない）
- **フル** — 完全なアクセス（表示、作成、編集、削除が可能）

![権限カテゴリ](/static/core/admin/img/help/staff-roles/permission-categories.webp)

| カテゴリ | 何を制御するか |
|----------|-----------------|
| **商品カタログ** | 商品、カテゴリ、ブランド、属性、在庫、倉庫、デジタルアセット |
| **注文と履行** | 注文、返金、返品、出荷、配送設定 |
| **顧客** | 顧客プロファイル、セグメント、分析 |
| **コンテンツとページ** | ページ、ブログ投稿、お知らせ、フォーム |
| **デザインとテーマ** | テーマ、ヘッダー/フッターテンプレート、メニュー、デザイントークン、カスタムCSS |
| **マーケティングとプロモーション** | プロモーション、割引券、アフィリエイト、ロイヤルティ、リファラル、商品フィード |
| **メディアライブラリ** | 画像、動画、フォルダ、タグ |
| **メールシステム** | メールアカウント、テンプレート、配信キュー |
| **支払いと請求** | 支払いプロバイダー、トランザクション、ウェブフック、サブスクリプション、為替レート |
| **検索** | 検索設定、同義語、リダイレクト、分析 |
| **店舗設定** | サイト設定、地理的場所、国マッピング、ビジネスルール |
| **POS管理** | POS端末、シフト、現金の動き、レシートテンプレート |
| **ユーザーと役割** | スタッフユーザーのアカウント、役割、APIトークン |

ユーザーが複数の役割を持つ場合、**最高**のアクセスレベルが適用されます。例えば、役割Aが商品に対して"閲覧"を許可し、役割Bが"フル"を許可する場合、ユーザーは"フル"のアクセスを得ます。

### POS権限フラグ

役割がPOSアクセスを許可する場合、POS権限タブはPOSオペレーターが何ができるかを正確に調整するためのものです。これらは管理権限とは別であり、POS端末でチェックされます。

![POS権限](/static/core/admin/img/help/staff-roles/pos-permissions.webp)

| グループ | 権限 | 説明 |
|-------|-----------|-------------|
| **一般** | POSアクセス | すべてのPOSシステムを使用可能 |
| **販売と割引** | 手動割引 | 注文の項目レベルまたはカートレベルの手動割引を適用可能 |
| | 最大割引％ | 許可される最大割引％（0–100） |
| | 価格上書き | 登録で商品価格を上書き可能 |
| **返金とキャンセル** | 返金処理 | POS注文の返金を処理可能 |
| | キャンセル注文 | 現在のシフトからPOS注文をキャンセル可能 |
| **ギフトカード** | ギフトカード発行 | 登録で新しいギフトカードを発行可能 |
| | ギフトカード残高確認 | ギフトカード残高を確認可能 |
| **現金管理** | 現金管理 | 現金の入出金操作可能 |
| | 現金引き出し | 売上なしで現金引き出しが可能 |
| | シフト終了 | シフトを終了し、現金の照合を実行可能 |
| **レポート** | POSレポートの閲覧 | シフトレポートと販売サマリを閲覧可能 |
| **在庫** | 在庫調整 | 在庫レベルを調整可能（受領、損傷、再計算、返品） |

ブール型の権限については、**任意の**ユーザーの役割がそれを有効にしている場合、ユーザーはそれを取得できます。最大割引％については、**すべての役割**の中で最も高い値が適用されます。

## スタッフメンバーの管理

**設定 > スタッフ管理**に移動して、チームを表示および管理します。

### スタッフリスト

スタッフリストは、スタッフアクセスを持つすべてのユーザーを表示します。各メンバーについて、以下を確認できます:
- **名前とメールアドレス**
- **割り当てられた役割**（色付きバッジとして表示）
- **アクセスタイプ** — 管理のみ、POSのみ、または両方
- **2FAステータス** — 2要素認証が有効になっているかどうか
- **アクティブ/非アクティブ**ステータス

フィルタを使用して、役割、アクセスタイプ、または2FAステータスで絞り込みます。

### スタッフに役割を割り当て

1. スタッフメンバーをクリックしてプロフィールを開きます
2. **役割**セクションで、利用可能な各役割のカードが表示されます
3. 任意の役割カードのトグルをクリックして、割り当てまたは削除します
4. 変更は即座に反映されます — 保存ボタンは必要ありません
5. 下の**有効な権限**サマリは、すべての割り当てられた役割の組み合わせ結果を表示します

### 新しいスタッフメンバーの追加

1. **設定 > スタッフ管理**に移動し、**スタッフメンバーを追加**をクリックします
2. ユーザーのメールアドレス、名前、姓を入力します
3. 一時パスワードを設定します
4. 1つ以上の役割を割り当てます
5. ユーザーは、その役割が提供するアクセスを使用してログインできるようになります

## 役割の複製

既存の役割に基づいて新しい役割を作成するには:

1. 複製したい役割を開きます
2. ページの下部にある**役割を複製**をクリックします
3. すべての権限が同じ新しい役割が作成されます
4. 名前を変更し、必要に応じて権限を調整します
5. 新しい役割を保存します

これは、既存の役割とわずかな違いがある役割が必要な場合に役立ちます — 例えば、"ジュニアマネージャー"は"店舗マネージャー"に基づいていますが、権限が少なくなっています。

## 権限の適用方法

### 管理パネル

- **メニューの可視性** — ユーザーが"なし"のアクセスを持つカテゴリでは、サイドバーのセクションが非表示になります
- **ページアクセス** — 制限されたページにアクセスしようとすると、権限エラーが表示されます
- **アクション制限** — "閲覧"のアクセスを持つ場合、編集および削除ボタンは非表示になり、保存アクションはブロックされます
- **スーパーユーザーのバイパス** — スーパーユーザーアカウントは、役割の割り当てに関係なく常にフルアクセスを持っています

### POS端末

- **ログインゲート** — POSにログインできるのは、少なくとも1つの役割で"POS端末"が有効になっているユーザーのみです
- **機能トグル** — POSのボタンやアクション（返金、割引、キャンセルなど）は、ユーザーの統合されたPOS権限に基づいて表示または非表示されます
- **割引上限** — 最大割引％は、POSオペレーターが適用できる割引の最大値を強制的に設定します
- **APIの実行** — すべてのPOS権限は、APIレイヤーでサーバー側でチェックされ、UIのみでなく、APIレイヤーでチェックされます

## ヒント

- **事前に定義された役割から始める** — 7つの組み込み役割は、ほとんどのチーム構造をカバーしています。より具体的なアクセス制御が必要な場合にのみ、カスタム役割を作成してください。
- **複製機能を使用する** — 既存の役割と類似した役割が必要な場合は、それを複製して調整するのではなく、から作成してください。
- **必要に応じて複数の役割を割り当て** — 注文とマーケティングの両方を担当するスタッフメンバーには、"注文管理者"と"マーケティング管理者"の両方の役割を割り当てることができます。権限は自動的に組み合わされます。
- **管理とPOSアクセスを分離** — レジ担当は通常管理アクセスを必要としませんし、事務スタッフはPOSアクセスを必要としません。アクセスのトグルを使用して、整理を保つようにしてください。
- **POSスタッフの割引上限を設定** — 最大割引％は、レジ担当が過剰な割引を適用することを防ぎます。0に設定して割引を無効にしたり、シニアスタッフに10–25％などの合理的な上限を設定したりしてください。
- **定期的に役割を確認** — チームが成長するにつれて、役割の割り当てを監査し、スタッフが仕事に必要な最小限のアクセスを持っていることを確認してください。ポジションが変更された場合、役割を削除してください。
- **敏感な役割で2FAを有効にする** — 支払い、設定、ユーザー管理にアクセスするスタッフは、セキュリティのために2要素認証を有効にしてください。
