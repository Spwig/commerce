भुगतान टर्मिनल प्रदाता अपने पॉजिशन (POS) टर्मिनल पर क्रेडिट और डेबिट कार्ड की स्वीकृति की अनुमति देते हैं। स्ट्राइप टर्मिनल मुख्य रूप से समर्थित प्रदाता है, जो आधुनिक कार्ड रीडर (S700, WisePOS E, P400), प्रतिस्पर्धी प्रोसेसिंग दर, और अच्छी तरह से एकीकरण प्रदान करता है। एपीआई योग्यताओं के साथ प्रदाता खातों को सेट करें, वास्तविक समय में कनेक्शन के स्थिति की निगरानी करें, और अगर अलग-अलग क्षेत्रों में काम कर रहे हैं तो कई प्रदाताओं को प्रबंधित करें। प्रदाता प्रणाली विस्तारित करने योग्य है- अगर स्ट्राइप टर्मिनल आपके बाजार में संचालित नहीं होता है तो प्रदाता फ्रेमवर्क के माध्यम से अतिरिक्त भुगतान प्रोसेसर एकीकृत किए जा सकते हैं।

भुगतान प्रदाताओं का उपयोग बिना किसी खतरे के कार्ड भुगतान की स्वीकृति के लिए किया जा सकता है, भुगतान प्रोसेसिंग के स्थिति की ट्रैकिंग करें, और टर्मिनल के बीच रीडर के आवंटन को प्रबंधित करें।

![भुगतान प्रदाता सूची](/static/core/admin/img/help/payment-terminal-providers/provider-list.webp)

## भुगतान प्रदाता समीक्षा

भुगतान प्रदाता आपके व्यवसाय के लिए कार्ड भुगतान के प्रक्रिया के लिए तीसरे पक्ष के सेवा हैं:

**प्रदाता के दायित्व**:
- वास्तविक समय में कार्ड लेनदेन के लिए अनुमोदन करें
- भौतिक कार्ड रीडर्स के साथ संचार करें
- भुगतान सुरक्षा का प्रबंधन करें (PCI पालन, एन्क्रिप्शन)
- अपने बैंक खाते में धनराशि का ट्रांसफर करें (सेटलमेंट)
- लेनदेन रिपोर्टिंग और विवाद प्रबंधन प्रदान करें

**Spwig की भूमिका**:
- भुगतान अनुरोध को निर्धारित प्रदाता के लिए रूट करें
- एन्क्रिप्टेड प्रदाता एक्रेडेंटियल्स के भंडारण करें
- कनेक्शन स्थिति की निगरानी करें
- रीडर्स को टर्मिनल्स से संबद्ध करें
- आदेश में भुगतान परिणामों के रिकॉर्ड करें

## Stripe Terminal (मुख्य प्रदाता)

Stripe Terminal अधिकांश व्यापारियों के लिए भुगतान प्रदाता के रूप में अनुशंसित है:

**विशेषताएं**:
- आधुनिक EMV चिप कार्ड रीडर्स
- बेलॉन्टलेस (NFC) भुगतान समर्थन (Apple Pay, Google Pay, टैप-टू-पेई कार्ड)
- समावेशी विवाद प्रबंधन
- वास्तविक समय अनुमोदन
- विकासकर्ता-अनुकूल API
- 40+ देशों में उपलब्ध

**मूल्य** (2024 के अनुसार, वर्तमान दरों की पुष्टि करें):
- लेनदेन शुल्क: 2.7% + $0.05 प्रति व्यक्तिगत लेनदेन (यूएस)
- कोई मासिक शुल्क, कोई स्थापना शुल्क, कोई पीसीआई अनुगमन शुल्क नहीं
- कार्ड रीडर हार्डवेयर: एक बार की खरीद (मॉडल के आधार पर $59-$299)

**समर्थित क्षेत्र**:
- संयुक्त राज्य अमेरिका, कनाडा, यूनाइटेड किंगडम, यूरोपीय संघ, ऑस्ट्रेलिया, सिंगापुर, और अधिक
- स्ट्राइप की उपलब्धता की जांच करें: https://stripe.com/terminal

**समर्थित रीडर**:
- BBPOS WisePOS E (एक-सभी-एंड्रॉइड टर्मिनल)
- स्ट्राइप रीडर S700 (काउंटर रीडर)
- वेरिफोन P400 (पुराना रीडर, अभी भी समर्थित)

## स्ट्राइप टर्मिनल की स्थापना

**चरण 1: स्ट्राइप खाता बनाएं**
- stripe.com पर रजिस्टर करें
- व्यवसाय के सत्यापन को पूरा करें (बैंक खाता, कर आईडी)
- भुगतान को सक्रिय करें

**चरण 2: स्ट्राइप टर्मिनल को सक्रिय करें**
- स्ट्राइप डैशबोर्ड में, **उत्पाद > टर्मिनल** जाएं
- **शुरू करें** पर क्लिक करें
- टर्मिनल की सेवा के शर्तों को स्वीकृत करें

**चरण 3: स्थान बनाएं**
- Stripe Terminal को एक "Location" की आवश्यकता होती है जो आपके भौतिक खुदरा स्थल का प्रतिनिधित्व करता है
- **Terminal > Locations** पर जाएं
- **Create Location** पर क्लिक करें
- दुकान के पता और विवरण दर्ज करें
- स्थान के ID को सहेजें (जैसे `tml_1ABC123...`)

**चरण 4: API कुंजी उत्पन्न करें**
- **Developers > API Keys** पर जाएं
- अपनी **Secret Key** को खोजें (उत्पादन के लिए `sk_live_...` से शुरू होता है, परीक्षण के लिए `sk_test_...` से शुरू होता है)
- सीक्रेट कुंजी को कॉपी करें (सार्वजनिक रूप से साझा न करें)

**चरण 5: Spwig में विन्यास करें**
- **POS > Payment Providers** पर जाएं
- **+ Add Payment Provider** पर क्लिक करें
- **Provider** चुनें: "Stripe Terminal"
- **API Secret Key** दर्ज करें (चरण 4 से)
- **Location ID** दर्ज करें (चरण 3 से)
- सहेजें

**चरण 6: कनेक्शन की जांच करें**
- सहेजने के बाद, प्रदाता की स्थिति "Connected" (हरा) में बदल जानी चाहिए
- यदि स्थिति "Error" (लाल) दिखाती है, तो API कुंजी और स्थान के ID की पुष्टि करें
- प्रदाता विवरण दृश्य में त्रुटि संदेश की जांच करें

[![Payment Provider Add Form](/static/core/admin/img/help/payment-terminal-providers/provider-add-form.webp)](https://example.com)

## Provider Configuration Fields

**Provider Key** - Select the payment processor:
- **stripe_terminal** - Stripe Terminal (recommended)
- **manual** - Manual payment entry (for testing only, no actual processing)
- Additional providers may appear if installed via component system

**Credentials (Encrypted)** - JSON structure containing API credentials:
- Automatically encrypted before storage
- Never visible in plain text after saving
- Example structure (Stripe Terminal):
```json
{
  "api_key": "sk_live_ABC123...",
  "location_id": "tml_1ABC123..."
}
```

**Provider Settings** - Additional configuration (provider-specific):
- Statement descriptor (appears on customer's credit card statement)
- Auto-capture (immediately capture authorized payments vs manual capture)
- Currency override (if provider account uses different currency than store)

**संयोजन स्थिति** - वास्तविक समय स्थिति संकेतक:
- **संयोजित** (हरा) - प्रदाता अभिगमन और सही रूप से विन्यासित है
- **त्रुटि** (लाल) - संयोजन विफल या अमान्य अधिकार प्रमाण है
- **अज्ञात** (ग्रे) - अभी तक परीक्षण नहीं किया गया है (निर्माण के तुरंत बाद)

**अंतिम परीक्षण** - सबसे हालिस्त परीक्षण के समय छपा हुआ है
- लेनदेन प्रक्रिया के समय स्वचालित रूप से अपडेट होता है
- **संयोजन परीक्षण** प्रशासन कार्रवाई के माध्यम से मनमाने से परीक्षण ट्रिगर करें

## संयोजन स्थिति निगरानी

प्रणाली प्रदाता कनेक्टिविटी की निगरानी करती है ताकि ग्राहक भुगतान के प्रयास के पहले समस्याओं के बारे में आपको सूचना दे सके:

**स्वचालित परीक्षण**:
- प्रत्येक भुगतान लेनदेन के बाद संयोजन परीक्षण ट्रिगर होता है (आवश्यकता के अनुसार)
- पृष्ठभूमि कार्य 6 घंटे के बाद संयोजन के परीक्षण करता है (रोकथाम निगरानी)

**स्थिति के अर्थ**:

**संयोजित** - प्रदाता API अभिगमन योग्य है, अधिकार प्रमाण वैध हैं, भुगतान प्रक्रिया के लिए तैयार हैं

**त्रुटि** - सामान्य कारण:
- अमान्य API कुंजी (वापस ले लिया गया, अतीत के, या गलत)
- अमान्य स्थान के आईडी (स्ट्राइप में स्थान हटा दिया गया, गलत आईडी दर्ज की गई)
- नेटवर्क कनेक्टिविटी समस्याएं (फायरवॉल स्ट्राइप API को ब्लॉक कर रहा है)
- स्ट्राइप सेवा बंद (दुर्लभ)

**अज्ञात** - प्रदाता कभी नहीं परीक्षण किया गया (नए बनाए गए खाता पहले लेनदेन के लिए अपेक्षा में)

**त्रुटि स्थिति के समाधान**:
1. प्रदाता विवरण दृश्य में त्रुटि संदेश की जांच करें (विशिष्ट समस्या का विवरण देता है)
2. स्ट्राइप डैशबोर्ड में यह जांचें कि API कुंजी अभी भी वैध है
3. स्ट्राइप डैशबोर्ड में यह जांचें कि स्थान के आईडी अभी भी मौजूद है
4. **टेस्ट कनेक्शन** प्रशासन कार्रवाई के माध्यम से कनेक्शन की जांच करें
5. आवश्यकता होने पर अपने विवरण को अपडेट करें

## समर्थित कार्ड पढ़ने वाले तुलना

स्ट्राइप टर्मिनल कई पठन उपकरण विकल्प प्रदान करता है:

| मॉडल | प्रकार | भुगतान विधियाँ | प्रदर्शन | अच्छा है | मूल्य |
|-------|------|-----------------|---------|----------|-------|
| **WisePOS E** | सभी-एक-कदम | EMV चिप, NFC, स्वाइप | 5" रंगीन टच स्क्रीन | पूर्ण-विशेषता खुदरा POS | ~$299 |
| **S700** | काउंटरटॉप | EMV चिप, NFC, स्वाइप | मोनोक्रोम एलसीडी | मानक खुदरा चेकआउट | ~$249 |
| **P400** | काउंटरटॉप | EMV चिप, NFC, स्वाइप | मोनोक्रोम एलसीडी | पुराने तैनातियों के लिए | ~$299 |

**WisePOS E के लाभ**:
- एंड्रॉइड आधारित (एप्लिकेशन चलाता है, कस्टम कंटेंट प्रदर्शित कर सकता है)
- रंगीन टच स्क्रीन (टिप प्रम्प्ट और साइन एक्सप्रेस के लिए बेहतर UX)
- एकीकृत रसीद प्रिंटर (वैकल्पिक)
- सबसे तेज लेनदेन गति

**S700 के लाभ**:
- WisePOS E की तुलना में कम कीमत
- छोटा फुटप्रिंट
- जलरोधी डिज़ाइन

**P400** (पुराना मॉडल):
- अभी भी समर्थित है लेकिन नई तैनातियों के लिए अनिवार्य नहीं है
- S700/WisePOS E की तुलना में धीमा चिप कार्ड प्रोसेसिंग

सभी पाठक स्पविग पीओएस के माध्यम से स्ट्रीप टर्मिनल एपीआई के माध्यम से जुड़ते हैं (पीओएस उपकरण के सीधे यूएसबी/ब्लूटूथ कनेक्शन की आवश्यकता नहीं होती है)।

## सुरक्षा पर विचार

**अधिकार प्रमाण पत्र एन्क्रिप्शन**:
- सभी प्रदाता क्रेडेंशियल्स डेटाबेस में विश्राम पर एन्क्रिप्ट किए जाते हैं
- एन्क्रिप्शन एप्लिकेशन सीक्रेट की (एप्लिकेशन सेटिंग्स में परिभाषित) उपयोग करता है
- क्रेडेंशियल्स कभी लॉग्स या त्रुटि संदेशों में नहीं दिखाई देते हैं

**एपीआई कुंजी अधिकार**:
- उत्पादन में **सीमित एपीआई कुंजी** का उपयोग करें (टर्मिनल के लिए केवल अधिकार की सीमा लगाएं)
- असीमित सीक्रेट कुंजी का उपयोग मत करें (अधिक अनुमति जो आवश्यकता से अधिक है = सुरक्षा जोखिम)
- स्ट्रीप डैशबोर्ड में केवल **टर्मिनल** अधिकार के साथ सीमित कुंजी बनाएं

**पीसीआई अनुगमन**:
- स्ट्रीप टर्मिनल पीसीआई अनुगमन के लिए जिम्मेदार है (कार्ड डेटा कभी भी स्पविग सर्वरों तक नहीं पहुंचता)
- कार्ड संख्या पूरी तरह से रीडर हार्डवेयर पर प्रोसेस की जाती है → स्ट्रीप सर्वर → कार्ड नेटवर्क
- स्पविग केवल भुगतान परिणाम (अनुमोदित/अस्वीकृत) को संग्रहित करता है, कभी भी कार्ड विवरण नहीं

**की रोटेशन**:
- एक वर्ष में API कुंजियों को घुमाएं, सुरक्षा के लिए सर्वोत्तम प्रथा के रूप में
- रोटेशन करते समय, प्रदाता विन्यास में अपनाए गए आईडी को अपडेट करें
- पुरानी कुंजियों को स्ट्राइप डैशबोर्ड में रद्द कर सकते हैं, जब नई कुंजी काम करती है तो इसे पुष्टि करें

## एकाधिक प्रदाता

कुछ व्यापारी कई प्रदाता खातों की आवश्यकता रखते हैं:

**बहु-मुद्रा संचालन**:
- संयुक्त राज्य के दुकानें स्ट्राइप यूएस खाता उपयोग करती हैं (यूएसडी प्रक्रिया करती हैं)
- यूरोपीय दुकानें स्ट्राइप यूई खाता उपयोग करती हैं (ईयूआर प्रक्रिया करती हैं)
- मुद्रा के लिए पृथक प्रदाता को विन्यास करें

**प्रमुख प्रदाता**:
- मुख्य प्रदाता (स्ट्राइप टर्मिनल)
- प्रदाता के बैकअप (मैनुअल प्रविष्टि) जब पठक खराब हो जाते हैं
- कैशियर भुगतान शुरू करते समय प्रदाता का चयन करता है

**परीक्षण विरोधी उत्पादन**:
- परीक्षण प्रदाता `sk_test_...` API कुंजी के साथ
- उत्पादन प्रदाता `sk_live_...` API कुंजी के साथ
- परीक्षण चरण के बाद प्रदाताओं को बदलें

## सामान्य समस्याओं के त्रुटि निवारण

**समस्या 1: स्थिति में "Error" दिखाई देता है और संदेश "Invalid API key" होता है**
- **कारण**: API कुंजी वापस ले ली गई है या गलत तरीके से कॉपी की गई है
- **समाधान**: Stripe डैशबोर्ड में नई API कुंजी जनरेट करें, प्रदाता क्रेडेंशियल्स अपडेट करें, कनेक्शन की जांच करें

**समस्या 2: भुगतान के दौरान रीडर खोजा नहीं गया**
- **कारण**: रीडर प्रदाता के स्थान में रजिस्टर नहीं है
- **समाधान**: Stripe डैशबोर्ड में जांच करें कि रीडर समान स्थान ID में रजिस्टर है जो प्रदाता कॉन्फ़िग में उपयोग किया गया है

**समस्या 3: वैध कार्ड के बावजूद भुगतान अस्वीकृत हो जाता है**
- **कारण**: Stripe खाता पूरी तरह से सक्रिय नहीं है (सत्यापन प्रतीक्षा में)
- **समाधान**: Stripe डैशबोर्ड में व्यवसाय सत्यापन पूरा करें (बैंक खाता, कर आईडी)

**समस्या 4: कनेक्शन स्थिति में "Unknown" दिखाई देता है और कभी अपडेट नहीं होता है**
- **कारण**: प्रदाता कभी टेस्ट नहीं किया गया है (कोई लेनदेन प्रयास नहीं किया गया है)
- **समाधान**: **Test Connection** प्रशासन कार्रवाई का उपयोग करके मनमाने से कनेक्टिविटी परीक्षण ट्रिगर करें

## सुझाव

- **उत्पादन से पहले परीक्षण मोड** - प्रारंभिक सेटअप और परीक्षण के लिए Stripe परीक्षण API कुंजियों (`sk_test_...`) का उपयोग करें
- **एक मुद्रा के लिए एक प्रदाता** - USD आधारित Stripe खाते के साथ EUR के भुगतान के लिए प्रयास न करें; अलग-अलग प्रदाताओं के निर्माण करें
- **सप्ताह में कनेक्शन की स्थिति की निगरानी करें** - चेकआउट पर भुगतान विफलता को रोकने के लिए प्रारंभिक निगरानी
- **API कुंजी की अनुमतियों को सीमित करें** - Stripe API कुंजियों को केवल टर्मिनल अनुमतियों तक सीमित करें (कम से कम अनुमति के सिद्धांत)
- **स्थान के ID के बारे में दस्तावेज़ बनाए रखें** - यह रिकॉर्ड रखें कि Stripe के कौन-सा स्थान किस भौतिक दुकान के साथ संबंधित है
- **पठनकर्ता के आवंटन का परीक्षण करें** - प्रदाता सेटअप के बाद, वास्तविक कार्ड पठनकर्ता के साथ भुगतान का परीक्षण करें ताकि अंत-से-अंत फ्लो की पुष्टि की जा सके
- **Stripe संपर्क की जानकारी अपडेट रखें** - Stripe में व्यवसाय संपर्क जानकारी को वर्तमान के साथ मेल खाना सुनिश्चित करें (विवादों और अनुपालन के लिए महत्वपूर्ण)
