---
title: SSL 配置
---

SSL（安全套接层）用于加密客户浏览器与您的商店之间的连接。当启用 SSL 时，您的商店 URL 以 `https://` 开头，并且浏览器会显示一个锁形图标。SSL 对于接受付款、保护客户数据以及在搜索引擎中获得良好排名至关重要。

Spwig 支持多种 SSL 模式，以适应不同的托管环境。本指南将解释每种模式，并帮助您选择合适的模式。

## 选择 SSL 模式

| 模式 | 最佳适用 | 证书费用 | 续订 |
|------|----------|-----------------|---------|
| **Let's Encrypt** | 大多数商店 | 免费 | 自动 |
| **Cloudflare Origin CA** | 使用 Cloudflare 代理的商店 | 免费 | 手动（最多 15 年） |
| **自定义证书** | 拥有购买证书的商店 | 各不相同 | 手动 |
| **外部管理** | 负载均衡器、Cloudflare 灵活模式 | 无 | 无 |
| **自签名证书** | 开发和测试 | 免费 | 手动 |
| **无（HTTP）** | 本地开发 | 无 | 无 |

如果您不确定使用哪种模式，**Let's Encrypt** 是大多数商店的最佳选择。它是免费的、自动的，并且受到所有浏览器的信任。

## Let's Encrypt

Let's Encrypt 提供免费且受信任的 SSL 证书，每 60-90 天自动续订。这是大多数商家的推荐选项。

**要求：**
- 您的域名必须指向您的服务器（DNS 中的 A 记录）
- 端口 80 必须可以从互联网访问（用于证书验证）
- 用于证书过期通知的电子邮件地址

**配置步骤：**
1. 转到 **设置 > 网站设置**，并打开 **域名与 SSL** 选项卡
2. 输入您的域名
3. 选择 **Let's Encrypt**
4. 输入您的管理员电子邮件地址
5. 点击 **应用配置**

Spwig 会自动处理其余一切：验证您的域名、获取证书、配置 NGINX 以及设置自动续订。

## Cloudflare Origin CA

Cloudflare Origin CA 证书用于加密 Cloudflare 的边缘服务器与您的商店之间的连接。这些证书是免费的，最多可使用 15 年，但它们**仅被 Cloudflare 信任**——直接连接到您服务器的浏览器会看到证书警告。

如果您的域名使用 Cloudflare 作为代理（启用橙色云），此模式是理想的。Cloudflare 向访客展示其自己的受信任证书，而 Origin CA 证书则用于加密 Cloudflare 与您服务器之间的连接。

**要求：**
- 一个包含您域名的 Cloudflare 账户
- 从 Cloudflare 控制台生成的 Origin CA 证书和私钥
- 将 Cloudflare SSL/TLS 模式设置为 **Full (Strict)**

**生成 Origin CA 证书：**
1. 登录您的 Cloudflare 控制台
2. 选择您的域名
3. 转到 **SSL/TLS > 原始服务器**
4. 点击 **创建证书**
5. 选择 RSA 或 ECC（RSA 兼容性最好）
6. 添加您的域名（例如，`example.com` 和 `*.example.com`）
7. 选择有效期（建议 15 年）
8. 点击 **创建**，并复制证书和私钥

**在 Spwig 中设置：**
1. 转到 **设置 > 网站设置**，并打开 **域名与 SSL** 选项卡
2. 输入您的域名
3. 选择 **Cloudflare Origin CA**
4. 将证书粘贴到 **证书（PEM）** 字段中
5. 将私钥粘贴到 **私钥（PEM）** 字段中
6. 点击 **应用配置**

**配置完成后：**
- 在 Cloudflare 中，将 SSL/TLS 模式设置为 **Full (Strict)**
- 为您的域名的 DNS 记录启用 Cloudflare 代理（橙色云）
- 您的商店将通过 HTTPS 访问，并使用 Cloudflare 的受信任证书

## 自定义证书

如果您从证书颁发机构（CA）如 DigiCert、Sectigo 或 GoDaddy 购买了 SSL 证书，或者您的托管提供商为您颁发了证书，请使用此模式。

**配置步骤：**
1.

转到 **设置 > 网站设置**，并打开 **域名与 SSL** 选项卡
2.

输入您的域名
3.

选择 **自定义证书**
4.

保留所有 markdown 格式、图片路径、代码块和技术术语。

将证书链（包括中间证书）粘贴到 **证书 (PEM)** 字段中
5.

将您的私钥粘贴到 **私钥 (PEM)** 字段中
6.

点击 **应用配置**

您的证书应包含完整的链：您的域名证书后跟任何中间证书。私钥应为 PEM 格式（以 `-----BEGIN PRIVATE KEY-----` 或 `-----BEGIN RSA PRIVATE KEY-----` 开头）。

## 由外部管理

选择此模式当 SSL 在流量到达您的服务器之前由外部服务终止。在此设置中，您的服务器仅接收纯 HTTP 流量——服务器本身不安装证书。

**常见场景：**
- **Cloudflare 灵活 SSL** -- Cloudflare 加密浏览器到 Cloudflare 的流量，但向您的服务器发送 HTTP
- **云负载均衡器** -- AWS ALB、Google Cloud Load Balancer 或 DigitalOcean Load Balancer 终止 SSL 并转发 HTTP
- **反向代理** -- Spwig 前面的另一台服务器处理 SSL

**设置步骤：**
1. 转到 **设置 > 网站设置** 并打开 **域名与 SSL** 选项卡
2. 输入您的域名
3. 选择 **由外部管理**
4. 点击 **应用配置**

Spwig 将配置 NGINX 仅提供 HTTP 并信任来自代理的 `X-Forwarded-Proto` 头，以正确检测 HTTPS 访客。

## 自签名证书

自签名证书可以加密连接，但不受浏览器信任。访客将看到需要手动绕过的安全警告。此模式仅适用于开发服务器和内部测试。

**设置步骤：**
1. 转到 **设置 > 网站设置** 并打开 **域名与 SSL** 选项卡
2. 输入您的域名
3. 选择 **自签名**
4. 点击 **应用配置**

Spwig 会自动生成自签名证书。不要在生产商店中使用此模式。

## 故障排除

**配置后证书无法工作：**
- 验证您的域名的 A 记录指向您的服务器的 IP 地址
- 确保防火墙中已打开 80 和 443 端口
- 等待几分钟以使 DNS 更改生效

**Let's Encrypt 无法颁发证书：**
- 检查您的域名是否解析到此服务器的 IP 地址
- 确保防火墙未阻止 80 端口
- 如果您使用 Cloudflare，请在证书颁发期间临时将 DNS 设置为 "仅 DNS"（灰色云）

**Cloudflare 显示 "错误 526"（无效的 SSL 证书）：**
- 确保您选择了 **Cloudflare Origin CA** 模式（而不是由外部管理）
- 检查您的 Cloudflare SSL/TLS 模式是否设置为 **Full (Strict)**
- 验证 Origin CA 证书是否已过期

**尽管已启用 SSL，浏览器显示 "不安全"：**
- 某些页面可能会通过 HTTP 加载图像或脚本（混合内容）。检查浏览器的开发者控制台以查看混合内容警告。
- 确保设置中的站点 URL 使用 `https://`
