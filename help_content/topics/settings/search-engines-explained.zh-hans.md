---
title: 搜索引擎详解
---

Spwig 中的搜索引擎不是像 Elasticsearch 或 Algolia 这样的外部服务 - 它们是您商店数据库原生搜索系统中的配置上下文。每个引擎定义要搜索哪些内容类型、要排除哪些内容，以及如何对结果进行排序。本指南将解释什么是搜索引擎，何时创建多个引擎，以及如何配置它们。

大多数商家使用一个默认的 "shop" 引擎。只有在需要不同内容组合或排除不同使用场景时，才创建多个引擎。

![搜索引擎列表](/static/core/admin/img/help/search-engines-explained/search-engines-list.webp)

## 什么是搜索引擎？

Spwig 中的搜索引擎是一个命名配置，指定以下内容：

- **要搜索的内容类型**（产品、分类、品牌、博客文章）
- **要排除的内容**（您希望从搜索中隐藏的特定分类或品牌）
- **自定义相关性权重**（可选的每引擎权重覆盖）
- **活动状态**（引擎可以临时禁用）

每个引擎都有一个唯一的 slug，用于 API 调用和前端代码，以指定哪个引擎应处理搜索请求。

## 何时创建多个引擎

大多数商店只需要一个引擎。在以下情况下创建额外的引擎：

| 使用场景 | 示例 |
|----------|---------|
| **不同的内容组合** | 店铺引擎仅搜索产品；博客引擎仅搜索博客文章 |
| **选择性排除** | 主店铺引擎隐藏清仓分类；清仓引擎仅显示清仓商品 |
| **部门特定搜索** | 电子产品引擎排除服装分类；服装引擎排除电子产品 |
| **B2B 与 B2C 分离** | 批发引擎仅显示批量产品；零售引擎显示消费产品 |

如果您不确定是否需要多个引擎，请坚持使用一个。除非有特定的使用场景，否则添加引擎会增加复杂性而没有好处。

## 4 步向导

![向导步骤 1 - 基本信息](/static/core/admin/img/help/search-engines-explained/wizard-step1-basic.webp)

导航到 **搜索 > 设置向导**，通过一个引导式的 4 步流程创建新引擎：

### 步骤 1：基本信息

**引擎名称** - 友好的显示名称（例如，"店铺搜索"，"博客搜索"）。仅在管理界面中使用。

**Slug** - URL 安全的标识符（例如，"shop-search"，"blog-search"）。用于 API 调用和前端代码。如果留空，将从名称自动生成。

**活动** - 该引擎是否可用于搜索。非活动引擎不会返回任何结果。

### 步骤 2：内容类型

选择此引擎将搜索哪些类型的内容：

- 产品（包括所有产品类型：实物、数字、订阅）
- 分类
- 品牌
- 博客文章

**提示**：仅选择与此引擎目的相关的内容类型。专注于博客的引擎不需要启用产品。

### 步骤 3：权重（可选）

![向导步骤 3 - 权重](/static/core/admin/img/help/search-engines-explained/wizard-step3-weights.webp)

可选地，为这个特定引擎自定义相关性权重。如果跳过，该引擎将继承来自 SearchSettings 的全局权重。

大多数引擎应跳过此步骤并使用全局默认值。只有在该引擎有独特的排名需求时，才自定义权重（例如，博客引擎可能会将 weight_blog_posts 增加到 1.2）。

### 步骤 4：查看和创建

查看您的配置，然后点击 **创建引擎** 以保存。

## 引擎配置字段

如果您直接编辑引擎（绕过向导），您将看到这些字段：

**名称和 Slug** - 显示名称和 URL 标识符

**活动状态** - 开关以启用或禁用

**内容类型** - 类似 `['product', 'category']` 的 JSON 数组

**权重覆盖** - 类似 `{'weight_name': 1.8}` 的 JSON 对象（如果使用全局权重则为空）

**排除的分类** - 与 Category 模型的 M2M 关系。这些分类中的产品不会出现在搜索结果中。

**排除的品牌** - 与 Brand 模型的 M2M 关系。这些品牌的产品不会出现在搜索结果中。

## 使用排除

排除功能会隐藏此引擎搜索结果中的特定内容：

**示例：隐藏清仓商品**

1. 创建一个 "Main Shop" 引擎
2. 在 "排除的分类" 字段中，选择您的 "清仓" 分类
3. 在 "排除的品牌" 字段中，选择您想要隐藏的任何预算品牌
4. 保存

现在通过 "Main Shop" 引擎的搜索将不会返回清仓产品，即使它们在您的网站上是可见的。您可以创建一个单独的 "清仓" 引擎，该引擎仅搜索清仓商品。

## 在前端使用引擎

您的前端代码通过 API 调用指定使用哪个引擎：

```javascript
// 使用 "shop" 引擎（最常见）
fetch('/api/search/?q=laptop&engine=shop')

// 使用 "blog" 引擎
fetch('/api/search/?q=ecommerce tips&engine=blog')

// 如果未指定引擎参数，则使用默认引擎
fetch('/api/search/?q=laptop')
```

引擎 slug 变成查询参数。如果没有指定引擎，Spwig 会使用按字母顺序排列的第一个活动引擎。

## 与引擎相关的同义词和重定向

Synonym 和 SearchRedirect 模型都有一个可选的 `engine` 外键。如果设置，该同义词或重定向仅适用于通过该特定引擎进行的搜索。

**示例**：博客引擎可以有像 "tutorial" → "guide" 这样的同义词，这些同义词不适用于产品搜索。

大多数同义词和重定向不应是引擎特定的 - 将 engine 字段留空以全局应用它们。

## 小贴士

- **从一个引擎开始** - 创建默认的 "shop" 引擎，并在有明确需要多个引擎之前使用它进行所有操作
- **使用描述性的 slug** - 选择像 "shop"、"blog"、"wholesale" 这样的 slug，明确表示引擎的用途
- **在激活前测试引擎** - 以非活动状态创建引擎，通过 API 进行测试，然后激活
- **除非必要，否则不要创建引擎** - 更多的引擎意味着更多的配置复杂性，如果它们都做同样的事情，就没有好处
- **按引擎查看分析** - 搜索分析仪表板可以按引擎筛选，以查看哪些引擎使用最多
