title: 결제 단말기 제공업체
---

결제 단말기 제공업체는 귀하의 POS 단말기에서 신용 및 대출 카드 수락을 가능하게 합니다. Stripe Terminal은 주요 지원되는 제공업체로, 현대적인 카드 리더기(S700, WisePOS E, P400), 경쟁력 있는 처리 비용, 그리고 원활한 통합을 제공합니다. API 자격 증명을 사용하여 제공업체 계정을 구성하고, 실시간으로 연결 상태를 모니터링하며, 여러 제공업체를 관리할 수 있습니다. 제공업체 시스템은 확장 가능합니다. Stripe Terminal이 귀하의 시장에서 운영되지 않는 경우, 제공업체 프레임워크를 통해 추가 결제 처리업체를 통합할 수 있습니다.

결제 제공업체를 사용하여 카드 결제를 안전하게 수락하고, 결제 처리 상태를 추적하며, 단말기 간 리더기 할당을 관리할 수 있습니다.

## 결제 제공업체 개요

결제 제공업체는 귀하의 사업을 대신하여 카드 결제를 처리하는 제3자 서비스입니다:

**제공업체의 책임**:
- 실시간으로 카드 거래를 승인합니다
- 물리적 카드 리더기와 통신합니다
- 결제 보안을 처리합니다 (PCI 준수, 암호화)
- 자금을 귀하의 은행 계좌로 이체합니다 (정산)
- 거래 보고 및 분쟁 관리를 제공합니다

**Spwig의 역할**:
- 구성된 제공업체로 결제 요청을 라우팅합니다
- 암호화된 제공업체 자격 증명을 저장합니다
- 연결 상태를 모니터링합니다
- 리더기를 단말기와 연결합니다
- 결제 결과를 주문에 기록합니다

## Stripe Terminal (주요 제공업체)

Stripe Terminal은 대부분의 상인에게 권장되는 결제 제공업체입니다:

**기능**:
- 현대적인 EMV 칩 카드 리더기
- 비접촉식 (NFC) 결제 지원 (Apple Pay, Google Pay, 탭 투 페이 카드)
- 통합 분쟁 관리
- 실시간 승인
- 개발자 친화적인 API
- 40개 이상의 국가에서 사용 가능

**가격** (2024년 기준, 현재 가격 확인):
- 거래 수수료: 대면 거래당 2.7% + $0.05 (미국)
- 월별 수수료 없음, 설정 수수료 없음, PCI 준수 수수료 없음
- 카드 리더기 하드웨어: 일회성 구매 ($59-$299, 모델에 따라 다름)

**지원 지역**:
- 미국, 캐나다, 영국, 유럽 연합, 호주, 싱가포르 및 기타 지역
- Stripe의 가용성 확인: https://stripe.com/terminal

**지원 리더기**:
- BBPOS WisePOS E (모든 기능이 포함된 안드로이드 단말기)
- Stripe Reader S700 (데스크탑 리더기)
- Verifone P400 (기존 리더기, 여전히 지원됨)

## Stripe Terminal 설정

**단계 1: Stripe 계정 생성**
- stripe.com에서 가입합니다
- 사업 확인 완료 (은행 계좌, 세금 ID)
- 결제 활성화

**단계 2: Stripe Terminal 활성화**
- Stripe 대시보드에서 **제품 > 단말기**로 이동합니다
- **시작하기**를 클릭합니다
- 단말기 이용 약관을 수락합니다

**단계 3: 위치 생성**
- Stripe Terminal은 귀하의 물리적 유통 시설을 나타내는 "위치"가 필요합니다
- **단말기 > 위치**로 이동합니다
- **위치 생성**을 클릭합니다
- 가게 주소 및 세부 정보를 입력합니다
- 위치 ID를 저장합니다 (tml_1ABC123...와 같은 형식)

**단계 4: API 키 생성**
- **개발자 > API 키**로 이동합니다
- **비밀 키**를 찾습니다 (생산 환경은 `sk_live_...`로 시작, 테스트는 `sk_test_...`로 시작)
- 비밀 키를 복사합니다 (공개하지 마세요)

**단계 5: Spwig에서 구성**
- **POS > 결제 제공업체**로 이동합니다
- **+ 결제 제공업체 추가**를 클릭합니다
- **제공업체**: "Stripe Terminal" 선택
- **API 비밀 키** 입력 (단계 4에서)
- **위치 ID** 입력 (단계 3에서)
- 저장

**단계 6: 연결 테스트**
- 저장 후, 제공업체 상태가 "연결됨"(녹색)으로 변경되어야 합니다
- 상태가 "에러"(빨간색)를 표시하는 경우, API 키 및 위치 ID를 확인합니다
- 제공업체 세부 정보 뷰에서 오류 메시지를 확인합니다

![결제 제공업체 추가 폼](/static/core/admin/img/help/payment-terminal-providers/provider-add-form.webp)

## 제공업체 구성 필드

**제공업체 키** - 결제 처리업체를 선택합니다:
- **stripe_terminal** - Stripe Terminal (권장)
- **manual** - 수동 결제 입력 (테스트용으로만, 실제 처리 없음)
- 구성 시스템을 통해 설치된 추가 제공업체가 나타날 수 있습니다

**자격 증명 (암호화됨)** - API 자격 증명을 포함하는 JSON 구조:
- 저장 전에 자동으로 암호화됩니다
- 저장 후에는 평문으로 보이지 않습니다
- 예시 구조 (Stripe Terminal):
```json
{
  "api_key": "sk_live_ABC123...",
  "location_id": "tml_1ABC123..."
}
```

**제공업체 설정** - 추가 구성 (제공업체별):
- 명세 설명 (고객의 신용 카드 명세에 표시됨)
- 자동 승인 (승인된 결제를 즉시 승인하는 것 대신 수동 승인)
- 통화 오버라이드 (제공업체 계정이 가게와 다른 통화를 사용하는 경우)

**연결 상태** - 실시간 상태 지시자:
- **연결됨** (녹색) - 제공업체가 도달 가능하며 올바르게 구성됨
- **에러** (빨간색) - 연결 실패 또는 자격 증명이 잘못됨
- **알 수 없음** (회색) - 아직 테스트되지 않음 (처음 생성 직후)

**마지막 테스트** - 가장 최근 연결 테스트의 타임스탬프
- 거래가 처리될 때 자동으로 업데이트됩니다
- **연결 테스트** 관리 작업을 통해 수동으로 테스트를 트리거할 수 있습니다

## 연결 상태 모니터링

시스템은 제공업체 연결성을 모니터링하여 고객이 결제를 시도하기 전에 문제를 알립니다:

**자동 테스트**:
- 모든 결제 거래는 연결 테스트를 트리거합니다 (필수)
- 배경 작업은 6시간마다 연결을 테스트합니다 (예방적 모니터링)

**상태 의미**:

**연결됨** - 제공업체 API가 도달 가능하며, 자격 증명이 유효하며, 결제를 처리할 준비가 되어 있습니다

**에러** - 일반적인 원인:
- 유효하지 않은 API 키 (취소됨, 만료됨 또는 잘못됨)
- 유효하지 않은 위치 ID (Stripe에서 위치가 삭제됨, 잘못된 ID 입력됨)
- 네트워크 연결 문제 (화이트가 방화벽으로 Stripe API 차단)
- Stripe 서비스 중단 (드물음)

**알 수 없음** - 제공업체가 아직 테스트되지 않았음 (새로 생성된 계정이 첫 거래 대기 중)

**에러 상태 해결**:
1. 제공업체 세부 정보 뷰에서 오류 메시지를 확인합니다 (구체적인 문제를 설명함)
2. Stripe 대시보드에서 API 키가 여전히 유효한지 확인합니다
3. Stripe 대시보드에서 위치 ID가 여전히 존재하는지 확인합니다
4. **연결 테스트** 관리 작업을 통해 연결을 수동으로 테스트합니다
5. 필요한 경우 자격 증명을 업데이트합니다

## 지원되는 카드 리더기 비교

Stripe Terminal은 여러 리더기 하드웨어 옵션을 제공합니다:

| 모델 | 유형 | 결제 방법 | 디스플레이 | 최적 사용 | 가격 |
|-------|------|-----------------|---------|----------|-------|
| **WisePOS E** | 통합형 | EMV 칩, NFC, 스와이프 | 5" 컬러 터치 스크린 | 기능이 풍부한 유통 POS | ~$299 |
| **S700** | 데스크탑 | EMV 칩, NFC, 스와이프 | 단색 LCD | 표준 유통 체크아웃 | ~$249 |
| **P400** | 데스크탑 | EMV 칩, NFC, 스와이프 | 단색 LCD | 기존 배포 | ~$299 |

**WisePOS E의 장점**:
- 안드로이드 기반 (앱 실행, 커스텀 콘텐츠 표시 가능)
- 컬러 터치 스크린 (팁 요청, 서명 수집 등 사용자 경험 향상)
- 통합 영수증 프린터 (선택 사항)
- 가장 빠른 거래 속도

**S700의 장점**:
- WisePOS E보다 비용이 낮음
- 작은 공간 점유
- 스플래시 방지 설계

**P400** (구형 모델):
- 여전히 지원되지만 새로운 배포에는 권장되지 않음
- S700/WisePOS E보다 칩 카드 처리 속도가 느림

모든 리더기는 Stripe Terminal API를 통해 Spwig POS에 연결됩니다 (POS 장치에 직접 USB/Bluetooth 연결이 필요하지 않음).

## 보안 고려사항

**자격 증명 암호화**:
- 모든 제공업체 자격 증명은 데이터베이스에서 정지 시 암호화됩니다
- 암호화는 애플리케이션 비밀 키(애플리케이션 설정에서 정의됨)를 사용합니다
- 자격 증명은 로그 또는 오류 메시지에 나타나지 않습니다

**API 키 권한**:
- 생산 환경에서는 **제한된 API 키**를 사용하세요 (단말기만 허용)
- 무제한 비밀 키를 사용하지 마세요 (필요 이상의 접근 권한은 보안 위험)
- Stripe 대시보드에서 단말기만 허용하는 **제한된 키**를 생성하세요

**PCI 준수**:
- Stripe Terminal은 PCI 준수를 처리합니다 (카드 데이터는 Spwig 서버에 닿지 않음)
- 카드 번호는 리더기 하드웨어 → Stripe 서버 → 카드 네트워크에서 처리됩니다
- Spwig은 결제 결과(승인/거절)만 저장하며, 카드 세부 정보는 저장하지 않습니다

**키 회전**:
- 보안 최선 실천으로 연간 API 키를 회전하세요
- 키를 회전할 때 제공업체 구성에서 자격 증명을 업데이트하세요
- 새 키가 작동하는 것을 확인한 후 Stripe 대시보드에서 이전 키를 취소할 수 있습니다

## 여러 제공업체

일부 상인은 여러 제공업체 계정이 필요합니다:

**다중 통화 운영**:
- 미국 가게는 Stripe 미국 계정을 사용하여 USD를 처리합니다
- 유럽 가게는 Stripe 유럽 계정을 사용하여 EUR를 처리합니다
- 통화별로 별도의 제공업체를 구성합니다

**대체 제공업체**:
- 주요 제공업체 (Stripe Terminal)
- 리더기 고장 시 대체 제공업체 (수동 입력)
- 결제를 시작할 때 점원이 제공업체를 선택합니다

**테스트 vs 운영**:
- `sk_test_...` API 키로 테스트 제공업체를 사용합니다
- `sk_live_...` API 키로 운영 제공업체를 사용합니다
- 테스트 단계 후 제공업체를 전환합니다

## 일반적인 문제 해결

**문제 1: 상태가 "에러"로 표시되고 메시지가 "잘못된 API 키"입니다**
- **원인**: API 키가 취소되었거나 잘못 복사되었습니다
- **해결책**: Stripe 대시보드에서 새 API 키를 생성하고 제공업체 자격 증명을 업데이트한 후 연결을 테스트합니다

**문제 2: 결제 중 리더기가 발견되지 않습니다**
- **원인**: 리더기가 제공업체의 위치에 등록되지 않았습니다
- **해결책**: Stripe 대시보드에서 리더기가 동일한 위치 ID를 사용하여 제공업체 구성에 등록되었는지 확인합니다

**문제 3: 유효한 카드임에도 불구하고 결제가 거부됩니다**
- **원인**: Stripe 계정이 완전히 활성화되지 않았습니다 (확인 대기 중)
- **해결책**: Stripe 대시보드에서 사업 확인을 완료합니다 (은행 계좌, 세금 ID)

**문제 4: 연결 상태가 "알 수 없음"으로 표시되고 업데이트되지 않습니다**
- **원인**: 제공업체가 테스트되지 않았습니다 (거래 시도 없음)
- **해결책**: **연결 테스트** 관리 작업을 사용하여 연결 테스트를 수동으로 트리거합니다

## 팁

- **운영 전 테스트 모드 사용** - Stripe 테스트 API 키(`sk_test_...`)를 사용하여 초기 설정 및 테스트를 수행합니다
- **통화별 하나의 제공업체** - 미국 기반 Stripe 계정으로 EUR을 처리하지 마세요; 별도의 제공업체를 생성하세요
- **주간 연결 상태 모니터링** - 주간 모니터링은 체크아웃 시 결제 실패를 예방합니다
- **API 키 권한 제한** - Stripe API 키를 단말기만 허용하도록 제한하세요 (최소 권한 원칙)
- **위치 ID 기록** - Stripe 위치가 어떤 물리적 가게에 해당하는지 기록해 두세요
- **리더기 할당 테스트** - 제공업체 설정 후 실제 카드 리더기로 결제 테스트를 수행하여 엔드 투 엔드 흐름을 확인하세요
- **Stripe 연락처 업데이트** - Stripe에 있는 사업 연락처 정보가 현재와 일치하는지 확인하세요 (분쟁 및 준수에 중요함)
