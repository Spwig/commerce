---
title: Роли сотрудников и разрешения
---

Роли сотрудников позволяют вам точно контролировать, что каждый член команды может видеть и делать — как в панели администратора, так и в терминале POS. Определите роли со специфическими разрешениями, а затем назначьте их сотрудникам. Пользователь может иметь несколько ролей, и их действительные разрешения являются комбинацией всех назначенных ролей.

![Роли сотрудников](/static/core/admin/img/help/staff-roles/role-list.webp)

## Как это работает

1. Вы создаете **роли**, которые определяют набор разрешений (например, "Менеджер заказов", "Кассир")
2. Каждая роль контролирует два типа доступа: **разрешения панели администратора** и **разрешения POS**
3. Вы **назначаете роли** сотрудникам с их страницы профиля
4. Действительные разрешения сотрудника — это **объединение** всех его ролей — если любая роль предоставляет доступ, пользователь имеет его
5. Разрешения **кэшируются** для повышения производительности и автоматически обновляются при изменении ролей

## Предопределенные роли

Spwig включает 7 встроенных ролей, которые охватывают наиболее распространенные структуры команды. Эти роли нельзя удалить, но вы можете создать пользовательские роли для более конкретных нужд.

| Роль | Доступ | Описание |
|------|--------|-------------|
| **Владелец магазина** | Админ + POS | Полный доступ ко всему. Для основного администратора магазина. |
| **Менеджер магазина** | Админ + POS | Ежедневные операции — полный доступ к товарам, заказам, клиентам, маркетингу и поиску. Только для просмотра: дизайн, электронная почта, платежи и настройки. |
| **Редактор контента** | Админ | Управляет страницами, блогами, дизайном и медиа. Только для просмотра: товары. |
| **Менеджер заказов** | Админ | Обрабатывает заказы, доставку, возвраты и службу поддержки клиентов. Только для просмотра: товары. |
| **Менеджер маркетинга** | Админ | Управляет акциями, купонами, аффилиат-программой, программой лояльности и реферальной программой. Только для просмотра: товары, клиенты и медиа. |
| **Кассир** | Только POS | Основной персонал POS. Может обрабатывать продажи и проверять балансы подарочных карт. Нет скидок, возвратов или управления наличными. |
| **Старший кассир** | Только POS | Опытный персонал POS. Может обрабатывать возвраты, применять скидки (до 25%), управлять наличными и закрывать смены. |

## Создание пользовательской роли

Перейдите в **Настройки > Роли сотрудников** и нажмите **Добавить роль**.

### Общие настройки

| Настройка | Описание |
|---------|-------------|
| **Отображаемое имя** | Имя роли, отображаемое в админ-панели (например, "Складской персонал") |
| **Описание** | Краткое объяснение того, для чего предназначена эта роль |
| **Порядок сортировки** | Управляет порядком отображения в списке ролей |
| **Иконка** | Выберите одну из 20 иконок для визуального определения роли |
| **Цвет значка** | Цвет, используемый для значков ролей (Синий, Зеленый, Оранжевый, Красный, Бирюзовый, Серый) |
| **Панель администратора** | Переключатель, определяющий, предоставляет ли эта роль доступ к задней панели администратора |
| **POS-терминалы** | Переключатель, определяющий, предоставляет ли эта роль доступ к терминалам POS |

### Категории разрешений администратора

Вкладка разрешений администратора организует все функции платформы в 13 категорий. Для каждой категории вы устанавливаете один из трех уровней доступа:

- **Нет** — Нет доступа к этой области (пункты меню скрыты)
- **Просмотр** — Только для чтения (можно видеть данные, но не изменять их)
- **Полный** — Полный доступ (можно просматривать, создавать, редактировать и удалять)

![Категории разрешений](/static/core/admin/img/help/staff-roles/permission-categories.webp)

| Категория | Что она контролирует |
|----------|-----------------|
| **Каталог товаров** | Товары, категории, бренды, атрибуты, запасы, склады, цифровые активы |
| **Заказы и выполнение** | Заказы, возвраты, возвраты, отгрузки, настройки доставки |
| **Клиенты** | Профили клиентов, сегменты, аналитика |
| **Контент и страницы** | Страницы, блоги, объявления, формы |
| **Дизайн и тема** | Темы, шаблоны заголовка/подвал, меню, токены дизайна, пользовательский CSS |
| **Маркетинг и акции** | Акции, купоны, аффилиат, лояльность, рефералы, товарные фиды |
| **Библиотека медиа** | Изображения, видео, папки, теги |
| **Система электронной почты** | Аккаунты электронной почты, шаблоны, очередь доставки |
| **Платежи и оплата** | Провайдеры платежей, транзакции, вебхуки, подписки, курсы обмена |
| **Поиск** | Настройки поиска, синонимы, перенаправления, аналитика |
| **Настройки магазина** | Настройки сайта, геолокация, сопоставления стран, бизнес-правила |
| **Управление POS** | Терминалы POS, смены, движения наличных, шаблоны чеков |
| **Пользователи и роли** | Аккаунты сотрудников, роли, токены API |

Когда у пользователя есть несколько ролей, **наивысший** уровень доступа выигрывает. Например, если Роль A предоставляет "Просмотр" для Товаров, а Роль B предоставляет "Полный", пользователь получает "Полный" доступ.

### Флаги разрешений POS

Если роль предоставляет доступ к POS, вкладка Разрешения POS позволяет точно настроить, что может делать оператор POS. Это отдельно от разрешений администратора и проверяется на терминале POS.

![Разрешения POS](/static/core/admin/img/help/staff-roles/pos-permissions.webp)

| Группа | Разрешение | Описание |
|-------|-----------|-------------|
| **Общее** | Доступ к POS | Может использовать систему POS в целом |
| **Продажи и скидки** | Ручные скидки | Может применять ручные скидки на товары или корзину |
| | Максимальный процент скидки | Наивысший процент скидки, разрешенный (0–100) |
| | Переопределение цены | Может переопределять цены товаров на кассе |
| **Возвраты и аннулирования** | Обработка возвратов | Может обрабатывать возвраты по заказам POS |
| | Аннулировать заказы | Может аннулировать заказы POS с текущей смены |
| **Подарочные карты** | Выдавать подарочные карты | Может выдавать новые подарочные карты на кассе |
| | Проверить баланс подарочной карты | Может проверить баланс подарочной карты |
| **Управление наличными** | Управление наличными | Может выполнять операции снятия и пополнения наличных |
| | Открыть денежный ящик | Может открыть денежный ящик без продажи |
| | Закрыть смены | Может закрывать смены и выполнять сверку наличных |
| **Отчеты** | Просмотр отчетов POS | Может просматривать отчеты по сменам и сводки продаж |
| **Инвентарь** | Корректировка запасов | Может корректировать уровни запасов (получение, повреждение, пересчет, возврат) |

Для булевых разрешений, если **любая** из ролей пользователя включает его, у пользователя есть это разрешение. Для Максимального процента скидки применяется **наивысшее** значение среди всех ролей.

## Управление сотрудниками

Перейдите в **Настройки > Управление сотрудниками**, чтобы просмотреть и управлять вашей командой.

### Список сотрудников

Список сотрудников показывает всех пользователей с доступом к сотрудникам. Для каждого члена команды вы можете увидеть:
- **Имя и электронная почта**
- **Назначенные роли** (показаны как цветные значки)
- **Тип доступа** — Только админ, Только POS или Оба
- **Статус 2FA** — Включено ли двухфакторное подтверждение
- **Статус активности** — Активный/Неактивный

Используйте фильтры, чтобы сузить поиск по роли, типу доступа или статусу 2FA.

### Назначение ролей сотрудникам

1. Нажмите на сотрудника, чтобы открыть его профиль
2. В разделе **Роли** вы увидите карточки для каждой доступной роли
3. Нажмите переключатель на любой карточке роли, чтобы назначить или удалить ее
4. Изменения вступают в силу сразу — не требуется кнопка "Сохранить"
5. Ниже указанная сводка **Действительных разрешений** показывает комбинированный результат всех назначенных ролей

### Добавление нового сотрудника

1. Перейдите в **Настройки > Управление сотрудниками** и нажмите **Добавить сотрудника**
2. Введите электронную почту, имя и фамилию пользователя
3. Установите временный пароль
4. Назначьте одну или несколько ролей
5. Теперь пользователь может войти с доступом, предоставляемым его ролями

## Клонирование ролей

Чтобы создать новую роль на основе существующей:

1. Откройте роль, которую хотите скопировать
2. Нажмите **Клонировать роль** внизу страницы
3. Создается новая роль со всеми теми же разрешениями
4. Переименуйте ее и настройте разрешения по мере необходимости
5. Сохраните новую роль

Это полезно, когда вам нужна роль, которая похожа на существующую, но с небольшими отличиями — например, "Младший менеджер", основанный на "Менеджере магазина", но с меньшими разрешениями.

## Как применяются разрешения

### Панель администратора

- **Видимость меню** — Разделы боковой панели скрыты для категорий, где у пользователя "Нет" доступ
- **Доступ к страницам** — Попытка посетить ограниченную страницу показывает ошибку разрешения
- **Ограничения действий** — При наличии "Просмотра" скрыты кнопки редактирования и удаления, а действия сохранения заблокированы
- **Обход суперпользователя** — Учетные записи суперпользователя всегда имеют полный доступ, независимо от назначения ролей

### Терминал POS

- **Блокировка входа** — Только пользователи с хотя бы одной ролью, у которой включено "POS-терминалы", могут войти в POS
- **Переключатели функций** — Кнопки и действия POS (возврат, скидка, аннулирование и т.д.) отображаются или скрываются в зависимости от объединенных разрешений POS пользователя
- **Предел скидки** — Максимальный процент скидки накладывает жесткий лимит на размер скидки, которую может применить оператор POS
- **Выполнение API** — Все разрешения POS проверяются на сервере на уровне API, а не только в интерфейсе пользователя

## Советы

- **Начните с предопределенных ролей** — 7 встроенных ролей охватывают большинство структур команды. Создавайте пользовательские роли только тогда, когда вам нужны более конкретные ограничения доступа.
- **Используйте функцию клонирования** — Когда вам нужна роль, похожая на существующую, скопируйте ее и внесите изменения вместо создания с нуля.
- **Назначайте несколько ролей при необходимости** — Сотрудник, который занимается и заказами, и маркетингом, может быть назначен как "Менеджер заказов", так и "Менеджер маркетинга". Разрешения автоматически объединяются.
- **Разделите доступ администратора и POS** — Кассирам обычно не нужен доступ к админ-панели, а офисному персоналу не нужен доступ к POS. Используйте переключатели доступа, чтобы держать все чисто.
- **Установите лимиты скидок для сотрудников POS** — Максимальный процент скидки предотвращает применение чрезмерных скидок кассирами. Установите его на 0, чтобы запретить скидки, или разумный предел, например, 10–25% для старших сотрудников.
- **Регулярно проверяйте роли** — По мере роста вашей команды аудит назначений ролей обеспечит, что сотрудники имеют минимальный доступ, необходимый для их работы. Удалите роли, когда люди меняют должности.
- **Включите 2FA для чувствительных ролей** — Сотрудники, имеющие доступ к платежам, настройкам или управлению пользователями, должны иметь включено двухфакторное подтверждение для обеспечения безопасности.
