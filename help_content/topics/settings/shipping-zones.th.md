โซนการจัดส่งกำหนดภูมิศาสตร์สำหรับอัตราค่าจัดส่งที่เป้าหมาย—จัดกลุ่มประเทศ รัฐ หรือรหัสไปรษณีย์เป็นโซน แล้วเชื่อมโยงวิธีการจัดส่งกับโซนเฉพาะเพื่อควบคุมอัตราค่าใช้จ่ายอย่างแม่นยำ โซนใช้การจับคู่ตามลำดับความสำคัญเมื่อที่อยู่ตรงกับหลายโซน (โซนที่มีลำดับความสำคัญสูงสุดชนะ) ระบบนี้ช่วยให้สามารถกำหนดกลยุทธ์การกำหนดราคาที่ซับซ้อนได้ เช่น คิดค่าจัดส่งสูงสำหรับพื้นที่ห่างไกล ให้บริการจัดส่งฟรีภายในประเทศ หรือให้ราคาส่วนลดสำหรับภูมิภาคเฉพาะ

ใช้โซนเมื่อคุณต้องการค่าใช้จ่ายการจัดส่งที่แตกต่างกันสำหรับพื้นที่ภูมิศาสตร์ต่างกัน ตั้งแต่การแบ่งแยกภายในประเทศกับต่างประเทศที่ง่ายไปจนถึงการกำหนดราคาแบบหลายระดับในหลายภูมิภาคที่ซับซ้อน

## การเข้าใจโซนการจัดส่ง

**โซนคืออะไร**: ภูมิศาสตร์ที่มีชื่อระบุโดยประเทศ รัฐ/จังหวัด และรูปแบบรหัสไปรษณีย์

**โซนทำงานอย่างไร**:
1. ลูกค้ากรอกที่อยู่จัดส่งขณะชำระเงิน
2. ระบบประเมินโซนที่ใช้งานทั้งหมด
3. โซนที่ตรงกับที่อยู่ของลูกค้าจะเป็นตัวเลือก
4. หากมีโซนหลายโซนตรงกัน โซนที่มีลำดับความสำคัญสูงสุดจะชนะ
5. วิธีการจัดส่งที่เชื่อมโยงกับโซนที่ชนะจะถูกแสดง
6. วิธีการที่ไม่ได้เชื่อมโยงกับโซนใด (หรือเชื่อมโยงกับโซนที่ตรงกัน) จะถูกแสดง

**องค์ประกอบของโซน**:
- **ชื่อ**: ตัวระบุโซน (เช่น "ภายในประเทศ", "ยุโรป", "พื้นที่ห่างไกล")
- **ประเทศ**: รายการของรหัสประเทศที่รวมไว้ (ว่าง = ทุกประเทศ)
- **รัฐ/จังหวัด**: ข้อจำกัดของรัฐในแต่ละประเทศ (ไม่จำเป็นต้องระบุ)
- **รูปแบบรหัสไปรษณีย์**: รูปแบบ Regex สำหรับการจับคู่รหัสไปรษณีย์ (ไม่จำเป็นต้องระบุ)
- **ลำดับความสำคัญ**: จำนวนที่สูงกว่า = ลำดับความสำคัญสูงกว่าเมื่อมีหลายโซนตรงกัน

## ตรรกะการจับคู่โซน

โซนใช้ **การจับคู่แบบค่อยๆ จำกัด** เพื่อจับคู่ที่อยู่:

### ระดับ 1: การจับคู่ประเทศ

**รายการประเทศว่าง** → โซนตรงกับทุกประเทศ

**รายการประเทศที่ให้มา** → ประเทศในที่อยู่ต้องอยู่ในรายการ

ตัวอย่าง:
```
โซน: "ภายในประเทศ"
ประเทศ: ["US"]
→ ตรงกัน: ที่อยู่ใดๆ ในสหรัฐอเมริกา
→ ไม่ตรงกัน: แคนาดา, สหราชอาณาจักร, เป็นต้น
```

### ระดับ 2: การจับคู่รัฐ/จังหวัด

**ไม่มีรัฐที่กำหนด** → โซนตรงกับทุกรัฐในประเทศที่อนุญาต


**States defined for specific countries** → Address state must match

Example:
```
Zone: "West Coast"
Countries: ["US"]
States: {"US": ["CA", "OR", "WA"]}
→ Matches: California, Oregon, Washington addresses
→ No match: New York, Texas, etc.
```

### Level 3: Postal Code Matching

**No patterns defined** → Zone matches ALL postal codes in allowed country/states

**Patterns defined** → Address postal code must match at least one pattern

Example:
```
Zone: "Los Angeles Metro"
Countries: ["US"]
States: {"US": ["CA"]}
Postal Patterns: ["^90[0-9]{3}$", "^91[0-9]{3}$"]
→ Matches: 90001, 91210, 90245
→ No match: 94102 (San Francisco)
```

**Regex Pattern Examples**:
- `^90[0-9]{3}$` - Los Angeles area (90000-90999)
- `^[A-Z][0-9][A-Z] [0-9][A-Z][0-9]$` - Canadian postal code format (K1A 0B1)
- `^SW[0-9]{1,2}` - London UK postcodes starting with SW

## Priority-Based Zone Selection

When multiple zones match an address, **priority** determines which zone applies:

**How Priority Works**:
- Higher number = higher priority
- If address matches zones with priority 100 and 50, priority 100 wins
- Only winning zone's shipping methods are available

**Use Cases**:

**Scenario 1: Specific Overrides General**
```
Zone A: "Remote Alaska"
  Countries: ["US"]
  States: {"US": ["AK"]}
  Priority: 100

Zone B: "Domestic USA"
  Countries: ["US"]
  Priority: 50

Address: Anchorage, AK
→ Matches both zones
→ Priority 100 wins
→ "Remote Alaska" zone applies (higher shipping cost)
```

**Scenario 2: Postal Code Overrides State**
```
Zone A: "Manhattan Premium"
  Countries: ["US"]
  States: {"US": ["NY"]}
  Postal Patterns: ["^100[0-2][0-9]$"]
  Priority: 100

Zone B: "New York State"
  Countries: ["US"]
  States: {"US": ["NY"]}
  Priority: 50

Address: New York, NY 10001
→ Matches both zones
→ Priority 100 wins
→ "Manhattan Premium" applies (premium delivery service)
```

## Creating Shipping Zones

**Step-by-Step Workflow**:

- ไปที่การตั้งค่า > การจัดส่ง > พื้นที่การจัดส่ง

- คลิก "เพิ่มพื้นที่การจัดส่ง"

- **ชื่อ**: ตัวระบุที่อธิบายได้ (เช่น "สหภาพยุโรป", "ฝั่งตะวันตก", "พื้นที่ห่างไกล")

- **ลำดับความสำคัญ**: ตั้งค่าความสำคัญที่สัมพันธ์กัน (100 สำหรับเฉพาะเจาะจง, 50 สำหรับทั่วไป, 1 สำหรับใช้เป็นตัวสำรอง)

- **เปิดใช้งาน**: ใช้ปุ่มสลับเพื่อเปิดหรือปิดใช้งาน

- วิธีการสามารถเชื่อมโยงได้ขณะแก้ไขวิธี (ไม่ใช่ในการตั้งค่าพื้นที่)

- หรือเชื่อมโยงพื้นที่กับวิธีที่มีอยู่แล้ว: แก้ไขวิธี → พื้นที่การจัดส่ง → เลือกพื้นที่

- พื้นที่ที่มีลำดับความสำคัญสูงกว่าจะทับซ้อนพื้นที่ที่มีลำดับความสำคัญต่ำกว่าเมื่อมีหลายพื้นที่ตรงกัน

- แนะนำ: พื้นที่เฉพาะเจาะจง (100), พื้นที่ภูมิภาค (50), พื้นที่เริ่มต้น (1)

- เปิดใช้งาน "เปิดใช้งาน" = ใช่

- บันทึก

### Setup 2: Multi-Region International

**เป้าหมาย**: อัตราค่าจัดส่งที่แตกต่างกันสำหรับยุโรป ทวีปอเมริกาเหนือ เอเชีย และโลกอื่นๆ

```
Zone 1: "European Union"
  Countries: [AT, BE, BG, CY, CZ, DE, DK, EE, ES, FI, FR, GR, HR, HU, IE, IT, LT, LU, LV, MT, NL, PL, PT, RO, SE, SI, SK]
  Priority: 100

Zone 2: "North America"
  Countries: [US, CA, MX]
  Priority: 100

Zone 3: "Asia Pacific"
  Countries: [AU, CN, HK, IN, JP, KR, NZ, SG, TH, TW]
  Priority: 100

Zone 4: "Rest of World"
  Countries: [Leave empty]
  Priority: 1
```

**วิธีการจัดส่ง**:
- "EU Shipping" → โซนยุโรป
- "North America Shipping" → โซนอเมริกาเหนือ
- "Asia Pacific Shipping" → โซนแปซิฟิกตะวันออก
- "International Standard" → โซนโลกอื่นๆ

### การตั้งค่า 3: ค่าธรรมเนียมพิเศษสำหรับพื้นที่ห่างไกล

**เป้าหมาย**: เพิ่มค่าธรรมเนียมสำหรับรหัสไปรษณีย์ที่อยู่ในพื้นที่ห่างไกลภายในโซนในประเทศ

```
Zone 1: "Remote Domestic"
  Countries: [US]
  Postal Patterns: ["^99[0-9]{3}$", "^96[7-9][0-9]{2}$"]  # Alaska, Hawaii
  Priority: 100

Zone 2: "Standard Domestic"
  Countries: [US]
  Priority: 50
```

**วิธีการจัดส่ง**:
- "Remote Shipping" → โซนพื้นที่ห่างไกลในประเทศ (ค่าใช้จ่ายสูงกว่า)
- "Standard Shipping" → โซนในประเทศมาตรฐาน

---

### การตั้งค่า 4: โซนตามรัฐ

**เป้าหมาย**: อัตราค่าจัดส่งที่แตกต่างกันสำหรับแต่ละภูมิภาคของสหรัฐอเมริกา

```
Zone 1: "West Coast"
  Countries: [US]
  States: {"US": ["CA", "OR", "WA"]}
  Priority: 100

Zone 2: "East Coast"
  Countries: [US]
  States: {"US": ["NY", "NJ", "CT", "MA", "PA"]}
  Priority: 100

Zone 3: "Midwest"
  Countries: [US]
  States: {"US": ["IL", "IN", "OH", "MI", "WI"]}
  Priority: 100

Zone 4: "South"
  Countries: [US]
  States: {"US": ["TX", "FL", "GA", "NC", "SC"]}
  Priority: 100

Zone 5: "Other US States"
  Countries: [US]
  Priority: 50
```

## ตัวอย่างรูปแบบรหัสไปรษณีย์

รหัสไปรษณีย์ใช้ **regex** (regular expressions) สำหรับการจับคู่รูปแบบ:

### สหรัฐอเมริกา (ZIP Codes)

**รูปแบบ**: 5 หลัก (เช่น 90210)

แคลิฟอร์เนีย (90000-96199):  ^9[0-6][0-9]{3}$
นิวยอร์ก (10000-14999):    ^1[0-4][0-9]{3}$
เท็กซัส (75000-79999, 88500-88599):  ^(7[5-9]|885)[0-9]{2}$
อลาสกา (99500-99999):      ^99[5-9][0-9]{2}$


- เขตไม่เชื่อมโยงโดยตรงกับวิธี
- การเชื่อมโยงจะทำเสมอจากตั้งค่าวิธี

**ไม่มีเขตที่เชื่อมโยง** → วิธีสามารถใช้ได้กับที่อยู่ทั้งหมด

**เขตที่เชื่อมโยง** → วิธีสามารถใช้ได้ก็ต่อเมื่อที่อยู่ของลูกค้าตรงกับเขตที่เชื่อมโยงอย่างน้อยหนึ่งเขต

```
Method: "Domestic Standard"
Linked Zones: ["Domestic USA"]
→ แสดงเฉพาะที่อยู่ในสหรัฐอเมริกา

Method: "International Express"
Linked Zones: ["EU", "Asia Pacific", "Rest of World"]
→ แสดงที่อยู่ที่ไม่ใช่สหรัฐอเมริกาทั้งหมด
```

## การทดสอบการจับคู่เขต

ก่อนเปิดใช้งาน การทดสอบการตั้งค่าเขต:

1. **สร้างคำสั่งซื้อทดสอบ**

- ใช้ที่อยู่ในเขตต่างๆ

- ตรวจสอบการจับคู่เขตที่ถูกต้อง

2. **ตรวจสอบการแก้ไขลำดับความสำคัญ**

- ใช้ที่อยู่ที่ตรงกับเขตหลายเขต

- ตรวจสอบว่าเขตที่มีลำดับความสำคัญสูงสุดชนะ

- ยืนยันว่าวิธีการจัดส่งที่คาดไว้ปรากฏขึ้น

3. **ทดสอบกรณีขอบเขต**

- รหัสไปรษณีย์ที่อยู่ใกล้ขอบเขต (เช่น 90999 vs 91000)

- ขอบเขตของรัฐ

- ที่อยู่ต่างประเทศที่มีรหัสไปรษณีย์คล้ายกัน

4. **ใช้เครื่องมือแสดงตัวอย่างเขต** (ถ้ามี)

- ป้อนที่อยู่ทดสอบ

- ดูว่าเขตใดที่ตรงกัน

- ดูการแก้ไขลำดับความสำคัญ

## การแก้ไขปัญหา

**ปัญหา 1: ไม่มีวิธีการจัดส่งที่สามารถใช้ได้ที่หน้าชำระเงิน**

**สาเหตุ**

- ที่อยู่ของลูกค้าไม่ตรงกับเขตใดๆ

- วิธีทั้งหมดที่เชื่อมโยงกับเขตที่ไม่ตรงกัน

- ไม่มีวิธีที่ไม่มีข้อจำกัดเขต

**วิธีแก้ไข**

- สร้างเขตสำรอง (ทุกประเทศ, ลำดับความสำคัญ 1)

- หรือลบข้อจำกัดเขตจากวิธีอย่างน้อยหนึ่งวิธี

- ตรวจสอบรูปแบบประเทศ/รัฐ/รหัสไปรษณีย์ของเขต

**ปัญหา 2: การจับคู่เขตผิดพลาด**

- เลือกเขตที่มีลำดับความสำคัญต่ำกว่าแม้ว่าเขตที่มีลำดับความสำคัญสูงกว่าจะตรงกัน

- ข้อผิดพลาดในไวยากรณ์รูปแบบรหัสไปรษณีย์ (รูปแบบล้มเหลวโดยเงียบ)

- รหัสรัฐไม่ตรงกัน (CA vs California)

- ตรวจสอบค่าลำดับความสำคัญ (จำนวนที่สูงกว่า = ลำดับความสำคัญสูงกว่า)

- ทดสอบรูปแบบรหัสไปรษณีย์ด้วยเครื่องมือตรวจสอบ regex

- ใช้รหัสรัฐ 2 ตัวอักษร (CA, ไม่ใช่ California)

**ปัญหา 3: วิธีที่ไม่คาดคิดปรากฏขึ้น**

- วิธีนี้ไม่มีโซนที่เชื่อมโยง (สามารถใช้ได้ทุกที่)
- มีหลายโซนที่ตรงกัน โซนที่ไม่คาดคิดมีลำดับความสำคัญสูงกว่า
- การครอบคลุมของโซนทับซ้อนกันโดยไม่ตั้งใจ

- ตรวจสอบโซนที่เชื่อมโยงกับวิธี
- ตรวจสอบลำดับความสำคัญของโซนที่ตรงกัน
- ตรวจสอบสรุปการครอบคลุมของโซนเพื่อหาการทับซ้อน

- **เริ่มต้นด้วย 2 โซน** - ภายในประเทศและระหว่างประเทศ ขยายตามที่จำเป็น
- **ใช้ลำดับความสำคัญอย่างชาญฉลาด** - โซนเฉพาะ 100, โซนภูมิภาค 50, โซนสำรอง 1
- **ทดสอบรูปแบบการจัดส่งอย่างละเอียด** - ข้อผิดพลาดของ Regex จะล้มเหลวโดยเงียบ ทำให้โซนไม่ตรงกัน
- **จัดทำเอกสารเกี่ยวกับตรรกะของโซน** - เพิ่มหมายเหตุในคำอธิบายโซนเพื่ออธิบายจุดประสงค์ของการครอบคลุม
- **หลีกเลี่ยงการใช้โซนมากเกินไป** - โซนมากเกินไปทำให้การตั้งค่าซับซ้อน; ใช้โปรโมชันการจัดส่งสำหรับสถานการณ์ที่ซับซ้อน
- **ใช้รหัสสถานะ ไม่ใช่ชื่อ** - "CA" ไม่ใช่ "California", "NY" ไม่ใช่ "New York"
- **สร้างโซนสำรอง** - ทุกประเทศ ลำดับความสำคัญ 1 ทำให้มีตัวเลือกการจัดส่งอย่างน้อยหนึ่งตัวเสมอ
- **ตรวจสอบประสิทธิภาพของโซน** - หากลูกค้าจำนวนมากเห็นข้อความ "ไม่มีตัวเลือกการจัดส่ง" ให้ตรวจสอบการครอบคลุมของโซน
- **อัปเดตโซนสำหรับภูมิภาคใหม่** - เพิ่มประเทศในโซนยุโรปเมื่อมีประเทศใหม่เข้าร่วม
- **ใช้ชื่อที่อธิบายได้ดี** - "EU (Excluding UK)" ดีกว่า "Zone 3"
- **ทดสอบด้วยที่อยู่จริง** - ใช้ที่อยู่จริงของลูกค้าในระหว่างการทดสอบ ไม่ใช่ที่อยู่ที่คิดขึ้นมา
