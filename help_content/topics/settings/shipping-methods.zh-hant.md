---
title: 運送方式
---

運送方式是在結帳時顯示給客戶的配送選項，每種方式使用不同的定價策略來計算運費。Spwig 支援 7 種運送方式，從簡單的固定費用到複雜的運輸商即時定價。運送方式可以根據最低/最高訂單金額、重量和地理區域進行限制。客戶在結帳時選擇他們偏好的方式，計算出的費用會加到訂單總額中。

使用本指南來配置符合您業務模式的運送方式，從基本的固定費用運送，到複雜的基於區域的階梯定價。

## 運送方式類型

Spwig 提供 7 種運送方式類型，每種方式有不同的費用計算邏輯：

### 固定費用運送

**是什麼**：與購物車內容、目的地或重量無關的固定費用。

**使用場景**：
- 預測運費的簡單商店
- 單一產品類型（類似尺寸/重量）
- 僅限國內運送，使用標準運輸商費用
- 免運促銷（與運費規則一起使用）

**配置**：
- 設定 **方式類型** = 固定費用
- 輸入 **固定費用**（例如，$9.99）
- 可選：設定最低/最高訂單金額限制

**範例**：所有國內訂單使用 "Standard Shipping - $9.99"。

---

### 免運

**是什麼**：零費用的運送選項（客戶無需支付費用）。

**使用場景**：
- 免運促銷
- 高價值訂單（與最低訂單金額結合使用）
- 當地自取的替代方案
- 會員忠誠度計劃的福利

**配置**：
- 設定 **方式類型** = 免運
- 可選：設定 **最低訂單金額**（例如，滿 $50 免運）
- 與運費規則搭配使用，實現條件式免運

**範例**："Orders Over $50 Free Shipping"，最低訂單金額 = $50。

---

### 基於重量的運送

**是什麼**：根據總購物車重量，從階梯費用表中計算費用。

**使用場景**：
- 重量不固定的產品（書籍、硬體、雜貨）
- 基於重量的運輸商費用模型
- 預測的重量與費用比例

**配置**：
1. 設定 **方式類型** = 基於重量
2. 建立 **運費費用表**，basis_type = "weight"
3. 添加 **運費費用階梯**（例如，0-5kg = $10，5-10kg = $15，10-20kg = $25）
4. 可選：限制特定區域

**範例**：
```
0-2kg: $8
2-5kg: $12
5-10kg: $18
10kg+: $25
```

**運作方式**：購物車計算總重量 → 找到對應的階梯 → 返回階梯費用。

---

### 基於價格的運送

**是什麼**：根據購物車小計，從階梯費用表中計算費用。

**使用場景**：
- 運費與訂單價值相關
- 鼓勵更高的購物車價值（高階梯的每美元費用較低）
- 當產品價格相近時，作為基於重量的簡單替代方案

**配置**：
1. 設定 **方式類型** = 基於價格
2. 建立 **運費費用表**，basis_type = "price"
3. 添加 **運費費用階梯**（例如，$0-$50 = $9.99，$50-$100 = $14.99，$100+ = $19.99）

**範例**：
```
$0-$25: $6.99
$25-$75: $9.99
$75-$150: $12.99
$150+: Free
```

**運作方式**：購物車計算小計 → 找到對應的階梯 → 返回階梯費用。

---

### 實時運輸商費用

**是什麼**：在結帳時從運輸商 API（FedEx、UPS、DHL）即時獲取費用。

**使用場景**：
- 根據目的地變動的運費
- 為客戶提供多個運輸商選項
- 無需手動費用表即可準確獲取運輸商費用
- 國際運送，費用複雜

**配置**：
1. 設定 **方式類型** = 實時
2. 建立 **運輸商帳號**（設置 > 運送 > 運輸商帳號）
3. 輸入運輸商 API 權限（帳號號碼、API 金鑰、密鑰）
4. 將運輸商帳號連結到運送方式
5. 可選：添加利潤率百分比或固定利潤

**需求**：
- 活動的運輸商帳號（FedEx、UPS、DHL 等）
- 運輸商提供的 API 權限
- 定義運送包裝（用於維度重量計算）

**範例**："FedEx Ground" 方式在結帳時根據購物車重量、尺寸和目的地即時獲取 FedEx 費用。

**運作方式**：
1. 客戶在結帳時輸入地址
2. 系統使用起點、目的地、包裝尺寸和重量呼叫運輸商 API
3. 運輸商返回費用報價
4. 可選添加利潤
5. 費用顯示給客戶

---

### 當地自取

**是什麼**：客戶在實體地點自取訂單（無配送費用）。

**使用場景**：
- 提供自取服務的零售商店
- 倉庫自取選項
- 活動或市集攤位
- 為本地客戶消除配送費用

**配置**：
1. 設定 **方式類型** = 當地自取
2. 建立 **地點**（設置 > 運送 > 地點）
   - 設定地址、營業時間、自取容量
3. 將地點連結到方式
4. 可選：設定自取準備時間（例如，"2小時內準備好"）

**客戶體驗**：
- 在結帳時選擇 "當地自取"
- 如果有多個地點，選擇自取地點
- 根據可用性選擇自取日期/時間
- 當訂單準備好時收到通知

**範例**："Store Pickup - Free"，有 3 個零售地點，24 小時內準備好。

---

### 表格費用運送

**是什麼**：基於重量、價格或數量的靈活階梯定價，並支援先進的區域定位。

**使用場景**：
- 複雜的定價（根據區域和重量不同費用）
- 需要比僅基於重量或價格有更多控制權
- 多個定價因素（例如，重量 + 目的地 + 數量）

**配置**：
1. 設定 **方式類型** = 表格費用
2. 建立 **運費費用表**
3. 定義 **basis_type**：weight、price 或 quantity
4. 添加 **運費費用階梯**，包含最小/最大值
5. 可選：將階梯限制於特定區域或國家

**與基於重量/價格的差異**：表格費用支援每個階梯的地理限制，允許在不同區域中相同重量/價格有不同的費用。

**範例**：
```
Zone A (Domestic):
  0-5kg: $10
  5-10kg: $15

Zone B (Remote):
  0-5kg: $18
  5-10kg: $25
```

**運作方式**：購物車計算 basis 值（重量/價格/數量） → 找到客戶區域對應的階梯 → 返回階梯費用。

## 運送方式配置

所有運送方式都共享這些常見設置：

### 基本設置

- **名稱**：內部識別碼（不顯示給客戶）
- **顯示名稱**：結帳時顯示給客戶的名稱（例如，"Standard Shipping", "Express Delivery"）
- **描述**：結帳時顯示的可選幫助文字（例如，"Delivery in 3-5 business days"）
- **方式類型**：上述 7 種類型之一
- **啟用**：切換以啟用/停用方式而不刪除

### 費用設置

- **固定費用**：僅限於固定費用方式
- **費用表**：用於基於重量、基於價格、表格費用方式
- **運輸商帳號**：用於實時運輸商方式
- **稅類**：對運費費用徵稅（如適用）

### 限制

**訂單價值限制**：
- **最低訂單價值**：當購物車小計 ≥ 金額時，方式才可用（例如，滿 $50 免運）
- **最高訂單價值**：當購物車小計 > 金額時，方式隱藏（例如，固定費用僅適用於 $100 以下的訂單）

**重量限制**：
- **最低重量**：當購物車重量 ≥ 金額時，方式才可用
- **最高重量**：當購物車重量 > 金額時，方式隱藏（常見於輕型運送選項）

**地理限制**：
- **運送區域**：將方式連結到特定區域（國內、國際、區域）
- 空區域 = 可用於所有地址
- 多個區域 = 可用於任何匹配的區域

### 高級設置

- **優先順序**：在結帳時顯示順序（數字越小，列表中越靠前）
- **處理費用**：添加到計算費用的額外固定費用
- **免運閾值**：如果購物車小計 ≥ 閾值，自動將費用設為 $0（替代最低訂單價值）

## 建立運送方式

**逐步工作流程**：

1. **導航到運送方式**
   - 前往設置 > 購物車 > 運送方式
   - 點擊 "Add Shipping Method"

2. **選擇方式類型**
   - 根據您的定價策略選擇合適的類型
   - 類型決定可用的費用配置欄位

3. **配置基本資訊**
   - 名稱：內部參考（例如，"domestic_ground"）
   - 顯示名稱：面向客戶（例如，"Ground Shipping"）
   - 描述：配送時間（例如，"5-7 business days"）

4. **設定費用計算**
   - **固定費用**：輸入固定費用
   - **重量/價格/表格費用**：建立費用表（見下文）
   - **實時**：連結運輸商帳號
   - **免運/自取**：無需費用配置

5. **添加限制（可選）**
   - 最低/最高訂單價值
   - 最低/最高重量
   - 運送區域

6. **設定優先順序**
   - 數字越小，在結帳時越靠前
   - 推薦順序：免運（1），當地自取（2），標準（3），快遞（4）

7. **啟用方式**
   - 切換 "啟用" = 是
   - 儲存

---

## 建立費用表

對於基於重量、基於價格和表格費用方式：

**步驟 1：建立費用表**
- 前往設置 > 運送 > 費用表
- 點擊 "Add Rate Table"
- 設定 **名稱**（例如，"Domestic Weight Tiers"）
- 設定 **Basis Type**：weight、price 或 quantity

**步驟 2：添加階梯**
- 點擊 "Add Tier"
- 設定 **最小值** 和 **最大值**（匹配範圍）
- 設定 **費用**（此階梯的費用）
- 可選：限制於特定區域或國家
- 儲存階梯

**步驟 3：重複所有階梯**
- 覆蓋完整範圍（0 到預期最大值）
- 確保無間隙（例如，0-5、5-10、10-20、20+）
- 在最後階梯中使用 `null` 作為最大值（無限）

**步驟 4：連結到運送方式**
- 編輯運送方式
- 從下拉選單中選擇費用表
- 儲存

**基於重量的費用表範例**：
```
名稱：Domestic Weight Tiers
Basis：Weight

階梯：
1. 最小值：0g，最大值：2000g，費用：$8
2. 最小值：2000g，最大值：5000g，費用：$12
3. 最小值：5000g，最大值：10000g，費用：$18
4. 最小值：10000g，最大值：null，費用：$25
```

## 常見運送場景

### 情況 1：基本國內運送

**目標**：所有國內訂單使用簡單的 $9.99 固定費用。

**解決方案**：
- 方式類型：固定費用
- 固定費用：$9.99
- 運送區域："Domestic"（僅您國家）

---

### 情況 2：滿 $50 免運

**目標**：透過免運閾值鼓勵更高的購物車價值。

**解決方案 A（推薦）**：
- 方式類型：免運
- 最低訂單價值：$50
- 顯示名稱："Free Shipping (Orders $50+)"

**解決方案 B（使用規則）**：
- 方式類型：固定費用
- 固定費用：$9.99
- 建立運送規則：
  - 條件：購物車價值 ≥ $50
  - 動作：將費用設為 $0

---

### 情況 3：基於重量的國內 + 國際

**目標**：根據重量，國內與國際有不同的費用。

**解決方案**：
1. 建立 2 個區域："Domestic", "International"
2. 建立 2 個費用表："Domestic Weight", "International Weight"
3. 建立 2 個方式：
   - "Domestic Shipping" → 連結到 Domestic 區域 + Domestic Weight 費用表
   - "International Shipping" → 連結到 International 區域 + International Weight 費用表

---

### 情況 4：多個運輸商選項

**目標**：讓客戶在 FedEx Ground、FedEx Express、UPS Ground 之間選擇。

**解決方案**：
1. 建立 FedEx API 的運輸商帳號
2. 建立 UPS API 的運輸商帳號
3. 建立 3 個實時方式：
   - "FedEx Ground" → FedEx 提供商，service_code = "FEDEX_GROUND"
   - "FedEx Express" → FedEx 提供商，service_code = "FEDEX_EXPRESS"
   - "UPS Ground" → UPS 提供商，service_code = "UPS_GROUND"
4. 所有 3 個方式在結帳時查詢運輸商 API 並顯示即時費用

---

### 情況 5：當地自取 + 配送

**目標**：零售商店提供自取和配送選項。

**解決方案**：
1. 建立地點："Main Store"，包含地址、營業時間、準備時間
2. 建立 2 個方式：
   - "Local Pickup" → 當地自取類型，連結到 Main Store 地點
   - "Standard Delivery" → 固定費用 $9.99
3. 客戶在結帳時看到這兩個選項

## 測試運送方式

在上線前，測試所有方式：

1. **建立測試購物車**
   - 添加不同重量/價格的產品
   - 前往結帳

2. **測試每個方式**
   - 在不同區域輸入地址
   - 確認正確的方式出現
   - 檢查計算費用是否符合預期

3. **測試限制**
   - 添加商品直到達到最低訂單價值 → 確認免運出現
   - 添加重物 → 確認基於重量的階梯有效
   - 測試區域限制 → 確認被排除區域的方式隱藏

4. **測試實時方式**（如適用）
   - 使用運輸商測試憑證
   - 確認費用成功返回
   - 檢查費用準確性與運輸商網站

## 故障排除

**問題 1：方式在結帳時不顯示**

**原因**：
- 方式未啟用
- 購物車未達到最低/最高訂單價值
- 購物車未達到最低/最高重量
- 客戶地址不匹配任何連結區域
- 無費用表階梯覆蓋購物車重量/價格

**解決方案**：檢查限制，確認啟用狀態，確保區域/階梯覆蓋客戶場景。

---

**問題 2：實時費用失敗**

**原因**：
- 無效的 API 權限
- 運輸商帳號未啟用
- 未定義運送包裝（運輸商需要尺寸）
- 未設定起點地址
- 運輸商 API 故障

**解決方案**：測試運輸商連接，確認權限，確保包裝配置，檢查設置中的起點地址。

---

**問題 3：計算費用錯誤**

**原因**：
- 費用表階梯有間隙或重疊
- 階梯最小/最大值單位錯誤（克 vs 公斤）
- 意外添加處理費用
- 運費規則修改費用

**解決方案**：審查費用表階梯，確認單位，檢查運費規則的優先順序。

## 小技巧

- **從簡單開始** - 使用固定費用作為第一種方式，根據需要增加複雜度
- **徹底測試** - 在啟用生產環境前，在測試環境中驗證所有方式
- **使用描述性名稱** - "Standard Shipping (5-7 days)" 比 "Method 1" 更好
- **設定實際的配送時間** - 對客戶滿意度而言，少承諾，多達成
- **如果可能，提供自取選項** - 減少運費成本，提高客戶便利性
- **監控運輸商 API 的可靠性** - 如果實時費用失敗，準備固定費用作為備用
- **使用區域處理國際運送** - 根據區域設定不同費用，避免在昂貴目的地產生虧損
- **與運費規則結合使用** - 規則可添加條件邏輯（免運促銷、遠程地區附加費用）
- **保持方式數量有限** - 結帳時 2-4 個選項可避免決策困難
- **定期更新費用表** - 運輸商費用會變動，每年審查一次
- **明智使用優先順序** - 將免運/便宜選項放在前面，昂貴選項放在最後
