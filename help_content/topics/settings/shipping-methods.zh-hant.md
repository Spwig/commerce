---
title: 運送方式
---

運送方式是結帳時顯示給客戶的配送選項——每種方式都使用不同的定價策略來計算運費。Spwig 支援 7 種運送方式，從簡單的固定費用到複雜的運輸商即時計價。運送方式可以根據最低/最高訂單金額、重量和地理區域進行限制。客戶在結帳時選擇他們偏好的方式，計算出的費用會加到訂單總額中。

使用本指南來配置符合您業務模式的運送方式，從基本的固定費用運送，到複雜的基於區域的階梯定價。

## 運送方式類型

Spwig 提供 7 種運送方式類型，每種方式都有不同的費用計算邏輯：

### 固定費用運送

**是什麼**：與購物車內容、目的地或重量無關的固定費用。

**使用場合**：
- 運費預測簡單的商店
- 單一產品類型（尺寸/重量相似）
- 僅限國內運送，使用標準運輸商費用
- 免運門檻促銷（與運送促銷搭配使用）

**設定方式**：
- 設定 **方式類型** = 固定費用
- 輸入 **固定費用**（例如，$9.99）
- 可選：設定最低/最高訂單金額限制

**範例**：所有國內訂單使用「標準運送 - $9.99」。

---

### 免運

**是什麼**：零費用運送選項（客戶無需支付費用）。

**使用場合**：
- 免運促銷
- 高價值訂單（與最低訂單金額搭配使用）
- 當地自取的替代方案
- 會員忠誠度方案的優惠

**設定方式**：
- 設定 **方式類型** = 免運
- 可選：設定 **最低訂單金額**（例如，滿 $50 免運）
- 與運送促銷搭配使用，可實現條件式免運

**範例**：最低訂單金額為 $50 的「滿 $50 免運」。

---

### 基於重量的運送

**是什麼**：根據購物車總重量，從階梯費用表中計算費用。

**何時使用**
- 具有變動重量的產品（書籍、硬體、雜貨）
- 基於重量的運費定價模型
- 可預測的重量與成本比例

**設定**
1. 設定 **方法類型** = 基於重量
2. 建立 **運費率表**，basis_type = "weight"
3. 增加 **運費率階梯**（例如，0-5kg = $10，5-10kg = $15，10-20kg = $25）
4. 可選：限制於特定區域

**範例**
```
0-2kg: $8
2-5kg: $12
5-10kg: $18
10kg+: $25
```

**運作方式**：購物車計算總重量 → 找到對應階梯 → 返回階梯的費用。

---

### 基於價格的運費

**這是什麼**：根據購物車小計從階梯率表計算費用。

**何時使用**
- 運費與訂單價值相關
- 鼓勵更高的購物車金額（高階梯的每美元費用較低）
- 與基於重量的運費類似的商品的簡單替代方案

**設定**
1. 設定 **方法類型** = 基於價格
2. 建立 **運費率表**，basis_type = "price"
3. 增加 **運費率階梯**（例如，$0-$50 = $9.99，$50-$100 = $14.99，$100+ = $19.99）

**範例**
```
$0-$25: $6.99
$25-$75: $9.99
$75-$150: $12.99
$150+: 免費
```

**運作方式**：購物車計算小計 → 找到對應階梯 → 返回階梯的費用。

---

### 實時運費提供商

**這是什麼**：在結帳時從運費提供商 API（FedEx、UPS、DHL）即時獲取費用。

**何時使用**
- 不同目的地的運費費用不同
- 為客戶提供多個運費提供商選項
- 無需手動費用表即可精確的提供商費用
- 國際運費具有複雜的費用結構

**設定**
1. 設定 **方法類型** = 實時
2. 建立 **提供商帳號**（**設定** > **運費** > **提供商帳號**）
3. 輸入提供商 API 認證（帳號號碼、API 金鑰、密鑰）
4. 將提供商帳號連結到運費方法
5. 可選：增加利潤百分比或固定利潤


**需求**:
- 活動的運輸商帳號 (FedEx, UPS, DHL 等)
- 來自運輸商的 API 權限
- 已定義的運輸包裝 (用於計算體積重量)

**範例**："FedEx Ground" 方法會根據購物車重量、尺寸和目的地在結帳時取得即時的 FedEx 費率。

**運作方式**:
1. 顧客在結帳時輸入地址
2. 系統根據起點、目的地、包裝尺寸和重量呼叫運輸商 API
3. 運輸商回傳費用報價
4. 可選：套用附加費用
5. 費率顯示給顧客

---

### 當地自取

**是什麼**：顧客在實體地點自取訂單（無運費）。

**何時使用**:
- 提供自取服務的零售商店
- 倉庫自取選項
- 活動或市集攤位
- 為本地顧客消除運費

**設定方式**:
1. 設定 **方法類型** = 當地自取
2. 建立 **位置** (設定 > 運送 > 位置)
   - 設定地址、營業時間、自取容量
3. 將位置連結到方法
4. 可選：設定自取準備時間 (例如，"2小時內準備好")

**顧客體驗**:
- 在結帳時選擇 "當地自取"
- 如果有多個位置，選擇自取位置
- 根據可用性選擇自取日期/時間
- 當訂單準備好時收到通知

**範例**："在商店自取 - 免費"，有 3 個零售位置，24 小時內準備好。

---

### 標準運費表

**是什麼**：根據重量、價格或數量的靈活階梯定價，並具有先進的區域定位功能。

**何時使用**:
- 複雜的定價 (按區域和重量不同收費)
- 需要比僅依重量或價格更強的控制
- 多個定價因素 (例如，重量 + 目的地 + 數量)

**設定方式**:
1. 設定 **方法類型** = 標準運費表
2. 建立 **運費表**
3. 定義 **basis_type**：重量、價格或數量
4. 添加 **運費階梯**，並設定最小/最大值
5. 可選：限制階梯至特定區域或國家

**與按重量/價格計算的差異**：分表計費支援根據階層設定地理限制，允許相同重量/價格在不同區域有不同的費用。

**範例**：
```
Zone A (Domestic):
  0-5kg: $10
  5-10kg: $15

Zone B (Remote):
  0-5kg: $18
  5-10kg: $25
```

**運作方式**：購物車計算基礎值（重量/價格/數量）→ 找出客戶所在區域對應的階層 → 返回該階層的費用。

## 運費方式設定

所有運費方式共用這些基本設定：

### 基本設定

- **名稱**：內部識別碼（不會顯示給客戶）
- **顯示名稱**：結帳時顯示給客戶的名稱（例如，「標準運費」、「快遞服務」）
- **描述**：結帳時顯示的選項說明（例如，「3-5個工作天內送達」）
- **方式類型**：上述7種類型之一
- **啟用**：切換以啟用或停用方式，而無需刪除

### 費用設定

- **固定費用**：僅適用於固定費用方式
- **費用表**：適用於按重量、按價格、分表計費方式
- **供應商帳號**：適用於即時運輸商方式
- **稅務分類**：對運費費用徵稅（如適用）

### 限制

**訂單價值限制**：
- **最低訂單價值**：只有當購物車小計 ≥ 金額時，此方式才可用（例如，滿 $50 免運費）
- **最高訂單價值**：當購物車小計 > 金額時，此方式將隱藏（例如，僅適用於 $100 以下的訂單）

**重量限制**：
- **最低重量**：只有當購物車重量 ≥ 金額時，此方式才可用
- **最高重量**：當購物車重量 > 金額時，此方式將隱藏（常見於輕型運輸選項）

**地理限制**：
- **運送區域**：將方式連結到特定區域（國內、國際、區域）
- 空區域 = 可用於所有地址
- 多個區域 = 可用於任何匹配的區域

### 高階設定

- **Priority**: 在結帳時的顯示順序（數字越小，列表中越靠前）
- **Handling Fee**: 加到計算成本上的額外固定費用
- **Free Shipping Threshold**: 如果購物車小計 ≥ 閾值，自動將成本設為 $0（替代 min_order_value）

## 建立運費方式

**逐步操作流程**：

1. **導航至運費方式**
   - 前往 設定 > 購物車 > 運費方式
   - 點擊 "新增運費方式"

2. **選擇方式類型**
   - 根據您的定價策略選擇合適的類型
   - 類型決定可用的成本配置欄位

3. **配置基本資訊**
   - 名稱：內部參考（例如，"domestic_ground"）
   - 顯示名稱：面向客戶（例如，"Ground Shipping"）
   - 說明：配送時間範圍（例如，"5-7 個工作天"）

4. **設定成本計算**
   - **固定費用**：輸入固定費用
   - **重量/價格/表格費用**：建立費用表（請參閱下方）
   - **即時費用**：連結供應商帳戶
   - **免費/自取**：不需要費用配置

5. **新增限制（可選）**
   - 最低/最高訂單價值
   - 最低/最高重量
   - 運送區域

6. **設定優先順序**
   - 數字越小，在結帳時顯示越靠前
   - 推薦順序：免費（1），本地自取（2），標準（3），快遞（4）

7. **啟用方式**
   - 切換 "啟用" = 是
   - 儲存

---

## 建立費用表

對於基於重量、價格和表格費用的方式：

**步驟 1：建立費用表**
- 前往 設定 > 運費 > 費用表
- 點擊 "新增費用表"
- 設定 **名稱**（例如，"Domestic Weight Tiers"）
- 設定 **基礎類型**：weight、price 或 quantity

**步驟 2：新增階梯**
- 點擊 "新增階梯"
- 設定 **最小值** 和 **最大值**（匹配範圍）
- 設定 **費用**（此階梯的費用）
- 可選：限制於特定區域或國家
- 儲存階梯

**步驟 3：重複所有階梯**
- 覆蓋完整範圍（0 到預期最大值）
- 確保沒有間隔（例如，0-5、5-10、10-20、20+）
- 最後階梯使用 `null` 作為最大值（無限）

**步驟 4：連結到運費方式**
- 編輯運費方式
- 從下拉選單中選擇費用表
- 儲存

**範例基於重量的費用表**：
```
Name: Domestic Weight Tiers
Basis: Weight

Tiers:
1. Min: 0g, Max: 2000g, Rate: $8
2. Min: 2000g, Max: 5000g, Rate: $12
3. Min: 5000g, Max: 10000g, Rate: $18
4. Min: 10000g, Max: null, Rate: $25
```

## 常見運費情境

### 情境 1：基本國內運費

**目標**：為所有國內訂單提供簡單的 $9.99 固定費用。

**解決方案**：
- 方法類型：固定費用
- 固定費用：$9.99
- 運費區域："Domestic"（僅限您所在國家）

---

### 情境 2：滿 $50 免運費

**目標**：透過免運費門檻鼓勵更高的購物金額。

**解決方案 A**（建議）：
- 方法類型：免運費
- 最低訂單價值：$50
- 顯示名稱："Free Shipping (Orders $50+)"

**解決方案 B**（使用規則）：
- 方法類型：固定費用
- 固定費用：$9.99
- 建立運費促銷：
  - 條件：購物金額 ≥ $50
  - 動作：將費用設為 $0

---

### 情境 3：基於重量的國內 + 國際

**目標**：根據重量為國內和國際訂單設定不同的費用。

**解決方案**：
1. 建立 2 個區域："Domestic", "International"
2. 建立 2 個費用表："Domestic Weight", "International Weight"
3. 建立 2 個方法：
   - "Domestic Shipping" → 連結到 Domestic 區域 + Domestic Weight 費用表
   - "International Shipping" → 連結到 International 區域 + International Weight 費用表

---

### 情境 4：多個運遞商選項

**目標**：讓客戶在 FedEx Ground、FedEx Express、UPS Ground 之間選擇。

**解決方案**：
1. 建立 FedEx API 的提供商帳號
2. 建立 UPS API 的提供商帳號
3. 建立 3 個即時方法：
   - "FedEx Ground" → FedEx 提供商，服務代碼 = "FEDEX_GROUND"
   - "FedEx Express" → FedEx 提供商，服務代碼 = "FEDEX_EXPRESS"
   - "UPS Ground" → UPS 提供商，服務代碼 = "UPS_GROUND"
4. 所有 3 個方法在結帳時查詢運遞商 API 並顯示即時費用

### 情境 5：本地自提 + 配送

**目標**：零售商店提供自提和配送兩種選項。

**解決方案**：
1. 建立位置："Main Store"，包含地址、營業時間和準備時間
2. 建立 2 種方法：
   - "Local Pickup" → 本地自提類型，連結到 Main Store 位置
   - "Standard Delivery" → 固定費用 $9.99
3. 顧客在結帳時可以看到這兩種選項

## 測試運費方法

在上線前，測試所有方法：

1. **建立測試購物車**
   - 加入不同重量/價格的產品
   - 繼續結帳

2. **測試每種方法**
   - 在不同區域輸入地址
   - 確認正確的方法出現
   - 檢查計算費用是否符合預期

3. **測試限制條件**
   - 加入商品直到達到最低訂購金額 → 確認免運費用出現
   - 加入重的商品 → 確認基於重量的費用層級正確運作
   - 測試區域限制 → 確認被排除區域的方法不會顯示

4. **測試即時方法**（如適用）
   - 使用運輸商測試憑證
   - 確認費用成功返回
   - 檢查費用準確性與運輸商網站一致

## 故障排除

**問題 1：結帳時方法未顯示**

**原因**：
- 方法未啟用
- 購物車未達到最低/最高訂購金額
- 購物車未達到最低/最高重量
- 顧客地址與任何連結區域不匹配
- 無費用表層級涵蓋購物車重量/價格

**解決方案**：檢查限制條件，確認啟用狀態，確保區域/層級涵蓋顧客的情況。

---

**問題 2：即時費用失敗**

**原因**：
- 無效的 API 憑證
- 提供商帳戶未啟用
- 未定義運輸包裝（運輸商需要尺寸）
- 未設定起點地址
- 運輸商 API 故障

**解決方案**：測試提供商連線，確認憑證，確保包裝已設定，檢查設定中的起點地址。

---

**問題 3：計算費用錯誤**

**原因**：
- 費用表層級存在間隔或重疊
- 層級的最小/最大值單位錯誤（克 vs 公斤）
- 意外添加處理費用
- 運費規則修改費用


**解決方案**：審查運費表層級，確認單位，檢查運費促銷的優先順序。

---

## 小技巧

- **從簡單開始** - 對第一種方法使用固定費用，根據需要逐步增加複雜度
- **徹底測試** - 在生產環境啟用之前，在測試環境中驗證所有方法是否正常運作
- **使用描述性名稱** - "標準運送（5-7 天）" 比 "方法 1" 更好
- **設定實際的配送時間** - 過度承諾會導致客戶不滿，應謹慎承諾
- **如果可能，提供自取服務** - 可降低運費成本，提高客戶便利性
- **監控運輸商 API 的可靠性** - 如果即時運費失敗，請使用固定費用作為備用方案
- **對國際運送使用區域** - 按地區設定不同費用，可避免在昂貴目的地產生虧損
- **與運費促銷結合使用** - 規則可增加條件邏輯（免運費促銷、偏遠地區附加費用）
- **保持方法數量有限** - 收銀結帳時提供 2-4 個選項，可避免決策困難
- **按季更新費用表** - 運輸商費用會變動，每年審查一次
- **明智地使用優先順序** - 將免費用或費用較低的選項放在前面，費用較高的選項放在最後
