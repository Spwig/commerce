---
title: 配送ゾーン
---

配送ゾーンは、ターゲットとした配送料金のための地理的地域を定義します。国、州、または郵便番号をゾーンにグループ化し、特定のゾーンに配送方法をリンクさせることで、正確な料金管理が可能です。住所が複数のゾーンに該当する場合、ゾーンは優先順位に基づいてマッチングされます（最も高い優先順位が勝ちます）。このシステムにより、複雑な価格戦略が可能になります。例として、遠隔地域に高額を請求し、国内配送を無料にしたり、特定の地域に割引料金を提供したりすることが可能です。

異なる地理的地域に異なる配送費用が必要な場合にゾーンを使用してください。単純な国内対国際の分割から、複雑な多地域階層価格体系まで、幅広い用途に適しています。

## 配送ゾーンの理解

**ゾーンとは**: 国、州/県、郵便番号のパターンによって定義された名前付きの地理的地域。

**ゾーンの動作方法**:
1. 顧客がチェックアウト時に配送住所を入力
2. システムはすべてのアクティブなゾーンを評価
3. 顧客の住所に該当するゾーンが候補に選出される
4. 複数のゾーンが該当する場合、最も高い優先順位のゾーンが選ばれる
5. 勝利したゾーンにリンクされた配送方法が表示される
6. どのゾーンにもリンクされていない（または該当するゾーンにリンクされている）配送方法も表示される

**ゾーンの構成要素**:
- **名前**: ゾーンの識別子（例: "国内", "EU", "遠隔地域"）
- **国**: 含まれる国コードのリスト（空 = すべての国）
- **州/県**: 国ごとの州の制限（任意）
- **郵便番号パターン**: ZIP/郵便番号のマッチング用の正規表現パターン（任意）
- **優先順位**: 複数のゾーンが該当する場合、数値が高いほど優先順位が高くなる

---

## ゾーンのマッチングロジック

ゾーンは、**段階的な絞り込み**を使用して住所にマッチします。

### レベル1: 国のマッチング

**国リストが空の場合** → ゾーンはすべての国にマッチ

**国リストが提供されている場合** → 住所の国がリスト内に含まれている必要があります

例:
```
ゾーン: "国内"
国: ["US"]
→ マッチ: 任意のUS住所
→ マッチしない: カナダ、英国など
```

### レベル2: 州/県のマッチング

**州が定義されていない場合** → ゾーンは許可された国すべての州にマッチ

**特定の国で定義された州** → 住所の州が一致する必要があります

例:
```
Zone: "West Coast"
Countries: ["US"]
States: {"US": ["CA", "OR", "WA"]}
→ 匹配: カリフォルニア州、オレゴン州、ワシントン州の住所
→ 不一致: ニューヨーク州、テキサス州など
```

### レベル3: 郵便番号の一致

**パターンが定義されていない** → ゾーンは許可された国/州のすべての郵便番号に一致します

**パターンが定義されている** → 住所の郵便番号が少なくとも1つのパターンと一致する必要があります

例:
```
Zone: "Los Angeles Metro"
Countries: ["US"]
States: {"US": ["CA"]}
Postal Patterns: ["^90[0-9]{3}$", "^91[0-9]{3}$"]
→ 匹配: 90001, 91210, 90245
→ 不一致: 94102 (サンフランシスコ)
```

**正規表現パターンの例**:
- `^90[0-9]{3}$` - ロサンゼルス地域 (90000-90999)
- `^[A-Z][0-9][A-Z] [0-9][A-Z][0-9]$` - カナダの郵便番号形式 (K1A 0B1)
- `^SW[0-9]{1,2}` - ロンドンUKのSWで始まる郵便番号

## 優先順位に基づくゾーン選択

複数のゾーンが住所に一致する場合、**優先順位**がどのゾーンが適用されるかを決定します:

**優先順位の仕組み**:
- 数字が大きいほど優先順位が高い
- 住所が優先順位100と50のゾーンに一致する場合、優先順位100が勝つ
- 勝利したゾーンのみの配送方法が利用可能

**使用例**:

**シナリオ1: 特定が一般を上書き**
```
Zone A: "Remote Alaska"
  Countries: ["US"]
  States: {"US": ["AK"]}
  Priority: 100

Zone B: "Domestic USA"
  Countries: ["US"]
  Priority: 50

Address: Anchorage, AK
→ 両方のゾーンに一致
→ 優先順位100が勝つ
→ "Remote Alaska" ゾーンが適用 (高い配送費用)
```

**シナリオ2: 郵便番号が州を上書き**
```
Zone A: "Manhattan Premium"
  Countries: ["US"]
  States: {"US": ["NY"]}
  Postal Patterns: ["^100[0-2][0-9]$"]
  Priority: 100

Zone B: "New York State"
  Countries: ["US"]
  States: {"US": ["NY"]}
  Priority: 50

Address: New York, NY 10001
→ 両方のゾーンに一致
→ 優先順位100が勝つ
→ "Manhattan Premium" が適用 (プレミアム配送サービス)
```

## 配送ゾーンの作成

**ステップバイステップワークフロー**:


1. **ゾーンへの移動**
   - 設定 > 配送 > 配送ゾーンにアクセス
   - "配送ゾーンを追加"をクリック

2. **基本設定**
   - **名前**: 説明的な識別子（例: "ヨーロッパ連合", "西海岸", "遠隔地域"）
   - **優先度**: 相対的な重要度を設定（特定: 100, 一般的: 50, デフォルト: 1）
   - **有効**: 有効/無効を切り替え

3. **地理的範囲の定義

   **オプションA: すべての国**（国リストを空のままにする）
   - ゾーンはグローバルにすべての住所に一致
   - デフォルト/フォールバックゾーンとして使用

   **オプションB: 特定の国**
   - "国を追加"をクリック
   - ドロップダウンから国を選択（US, CA, UKなど）
   - 含めるすべての国に対して繰り返す

   **オプションC: 特定の州/県**
   - 国を追加した後、各国に対して"州を追加"をクリック
   - 州をドロップダウンから選択
   - 例: US → CA, OR, WA（西海岸）

   **オプションD: 郵便番号パターン**（高度）
   - 1行ごとに正規表現パターンを入力
   - サンプル郵便番号でパターンをテスト
   - "パターンを検証"をクリックして構文を確認

4. **配送方法へのリンク**
   - 方法は編集中の方法でリンクできます（ゾーン設定内ではできません）
   - または既存の方法にゾーンをリンク: 方法を編集 → 配送ゾーン → ゾーンを選択

5. **表示優先度の設定**
   - 複数のゾーンが一致する場合、優先度が高いゾーンが優先される
   - おすすめ: 特定ゾーン（100）、地域ゾーン（50）、デフォルトゾーン（1）

6. **ゾーンの有効化**
   - "有効"をトグルしてはいに設定
   - 保存

## 一般的なゾーン設定

### 設定1: 国内 vs 国際

**目的**: 国内とその他のすべての国との間で異なる料金を設定する。

```
ゾーン1: "国内"
  国: [あなたの国コード]
  優先度: 50

ゾーン2: "国際"
  国: [空のままにするか、すべての他の国を選択]
  優先度: 1
```

**配送方法**:
- "国内標準" → 国内ゾーンにリンク
- "国際配送" → 国際ゾーンにリンク

---

### 設定2: 多地域国際

**目的**: EU、北米、アジア、世界の他の地域向けの異なる料金。

```
Zone 1: "European Union"
  Countries: [AT, BE, BG, CY, CZ, DE, DK, EE, ES, FI, FR, GR, HR, HU, IE, IT, LT, LU, LV, MT, NL, PL, PT, RO, SE, SI, SK]
  Priority: 100

Zone 2: "North America"
  Countries: [US, CA, MX]
  Priority: 100

Zone 3: "Asia Pacific"
  Countries: [AU, CN, HK, IN, JP, KR, NZ, SG, TH, TW]
  Priority: 100

Zone 4: "Rest of World"
  Countries: [Leave empty]
  Priority: 1
```

**配送方法**:
- "EU Shipping" → EU地域
- "North America Shipping" → 北米地域
- "Asia Pacific Shipping" → アジア太平洋地域
- "International Standard" → 世界の他の地域

---

### 設定3: 遠隔地域料金

**目的**: 国内地域内の遠隔郵便番号に料金を追加。

```
Zone 1: "Remote Domestic"
  Countries: [US]
  Postal Patterns: ["^99[0-9]{3}$", "^96[7-9][0-9]{2}$"]  # アラスカ、ハワイ
  Priority: 100

Zone 2: "Standard Domestic"
  Countries: [US]
  Priority: 50
```

**配送方法**:
- "Remote Shipping" → 遠隔地域 (高コスト)
- "Standard Shipping" → 標準地域

---

### 設定4: 州ごとのゾーン

**目的**: 各US地域向けの異なる料金。

```
Zone 1: "West Coast"
  Countries: [US]
  States: {"US": ["CA", "OR", "WA"]}
  Priority: 100

Zone 2: "East Coast"
  Countries: [US]
  States: {"US": ["NY", "NJ", "CT", "MA", "PA"]}
  Priority: 100

Zone 3: "Midwest"
  Countries: [US]
  States: {"US": ["IL", "IN", "OH", "MI", "WI"]}
  Priority: 100

Zone 4: "South"
  Countries: [US]
  States: {"US": ["TX", "FL", "GA", "NC", "SC"]}
  Priority: 100

Zone 5: "Other US States"
  Countries: [US]
  Priority: 50
```

## 郵便番号パターンの例

郵便番号は **regex** (正規表現) を使用してパターンマッチングを行います。

### アメリカ合衆国 (ZIPコード)

**形式**: 5桁 (例: 90210)

```
カリフォルニア (90000-96199):  ^9[0-6][0-9]{3}$
ニューヨーク (10000-14999):    ^1[0-4][0-9]{3}$
テキサス (75000-79999, 88500-88599):  ^(7[5-9]|885)[0-9]{2}$
アラスカ (99500-99999):      ^99[5-9][0-9]{2}$
```

### カナダ (郵便番号)

**形式**: A1A 1A1 (文字-数字-文字 空白 数字-文字-数字)

```
すべてのカナダの郵便番号:  ^[A-Z][0-9][A-Z] [0-9][A-Z][0-9]$\nオンタリオ (K, L, M, N, P):    ^[KLMNP][0-9][A-Z] [0-9][A-Z][0-9]$\nケベック (G, H, J):           ^[GHJ][0-9][A-Z] [0-9][A-Z][0-9]$\n```

### 英国 (郵便番号)

**形式**: AA1A 1AA または A1A 1AA

```
ロンドン (E, EC, N, NW, SE, SW, W, WC):  ^(E|EC|N|NW|SE|SW|W|WC)[0-9]{1,2}\nマンチェスター (M):                        ^M[0-9]{1,2}\nバーミンガム (B):                        ^B[0-9]{1,2}\n```

### オーストラリア (郵便番号)

**形式**: 4桁 (例: 2000)

```
ニューサウスウェールズ (1000-2999):  ^[12][0-9]{3}$\nビクトリア (3000-3999, 8000-8999):  ^[38][0-9]{3}$\nクイーンズランド (4000-4999, 9000-9999):  ^[49][0-9]{3}$\n```

### パターンテスト

**パターンを保存する前**に、既知の郵便番号でテストしてください:

1. パターンを入力: `^90[0-9]{3}$`
2. テスト入力: "90210" → 匹配する必要があります
3. テスト入力: "10001" → 匹配しない必要があります
4. テスト入力: "9021" → 匹配しない必要があります (4桁のみ)

複雑なパターンを検証するには、オンラインの正規表現テストツール (regex101.com) を使用してください。

## ゾーンカバレッジサマリー

ゾーンは、管理リストビューで **カバレッジサマリー** を表示し、何が含まれているかを示します:

**例**:
- "すべての国" → 国の制限なし
- "US, CA, MX" → 3か国
- "US (CA, OR, WA)" → 米国で3州
- "US (90xxx-91xxx)" → 米国で郵便番号パターン

**サマリーの使用**:
- ゾーンカバレッジを確認するのに迅速に
- カバレッジの重複やギャップを発見する
- ゾーン構成を一目で確認して監査する

## ゾーンを配送方法にリンクする

ゾーンと方法は **多対多の関係** です:

**方法側から** (推奨):
1. 配送方法を編集
2. "配送ゾーン"セクションにスクロール
3. 適用可能なゾーンを選択 (マルチ選択)
4. 方法を保存


- Zones don't directly link to methods

- Linking is always done from method configuration

**No zones linked** → Method available to ALL addresses

**Zones linked** → Method only available if customer address matches at least one linked zone

```
Method: "Domestic Standard"
Linked Zones: ["Domestic USA"]
→ Only shown to US addresses
```

```
Method: "International Express"
Linked Zones: ["EU", "Asia Pacific", "Rest of World"]
→ Shown to all non-US addresses
```

Before going live, test zone configuration:

- Use addresses in different zones

- Verify correct zone matches

- Use address that matches multiple zones

- Verify highest priority zone wins

- Confirm expected shipping methods appear

- Border postal codes (e.g., 90999 vs 91000)

- State boundaries

- International addresses with similar postal codes

- Enter test address

- See which zone(s) match

- View priority resolution

- Lower priority zone selected despite higher priority zone matching

- Postal code pattern syntax error (pattern fails silently)

- State code mismatch (CA vs California)

- Verify priority values (higher number = higher priority)

- Test postal code patterns with regex validator

- Use 2-letter state codes (CA, not California)

**原因**:
- メソッドにリンクされたゾーンが存在しない（どこでも利用可能）
- 複数のゾーンが一致するが、予期せぬゾーンの優先度が高すぎる
- ゾーンのカバレッジが意図せず重複している

**解決策**:
- メソッドにリンクされているゾーンを確認する
- 一致するゾーンの優先度を確認する
- カバレッジの要約を確認し、重複をチェックする

## ヒント

- **2つのゾーンから開始する** - 国内と国際、必要に応じて拡張する
- **優先度を賢く使用する** - 特定のゾーンは100、地域は50、フォールバックは1
- **郵便番号パターンを徹底的にテストする** - Regexエラーは静かに失敗し、ゾーンが一致しない原因になる
- **ゾーンのロジックを文書化する** - ゾーンの説明にノートを追加し、カバレッジの意図を説明する
- **過剰なゾーンを避ける** - ゾーンが多すぎると設定が複雑になる。複雑なシナリオでは配送プロモーションを使用する
- **州コードを使用し、名前は使用しない** - "CA"ではなく"California"、"NY"ではなく"New York"
- **フォールバックゾーンを作成する** - すべての国、優先度1、常に少なくとも1つの配送オプションが利用可能になるようにする
- **ゾーンのパフォーマンスを監視する** - 許可されていない配送オプションが多数表示される場合は、ゾーンのカバレッジを確認する
- **新しい地域に対応するためゾーンを更新する** - 新しい加盟国がEUゾーンに追加されるたびに、国を追加する
- **説明的な名前を使用する** - "EU (UKを除く)"よりも"Zone 3"の方が良い
- **実際の住所でテストする** - テスト時には顧客の実際の住所を使用し、架空の住所は使用しない
