---
title: 配送ゾーン
---

配送ゾーンは、ターゲットとした配送料金のための地理的領域を定義します。国、州、または郵便番号をゾーンにグループ化し、特定のゾーンに配送方法をリンクさせることで、正確な料金管理が可能です。複数のゾーンに該当する住所がある場合、ゾーンは優先順位に基づいてマッチングされます（最高優先順位が勝ちます）。このシステムにより、複雑な価格設定戦略が可能になります。例として、遠隔地域に高い料金を設定したり、国内配送を無料にしたり、特定の地域に割引料金を提供したりすることが挙げられます。

異なる地理的領域に異なる配送費用が必要な場合にゾーンを使用してください。単純な国内対国際の分割から、複雑な多地域段階的な価格設定まで、幅広い用途に適しています。

## 配送ゾーンの理解

**ゾーンとは**：国、州/県、郵便番号のパターンによって定義された名前付きの地理的領域。

**ゾーンの動作方法**：
1. 顧客がチェックアウト時に配送住所を入力
2. システムがすべてのアクティブなゾーンを評価
3. 顧客の住所に該当するゾーンが候補に
4. 複数のゾーンに該当する場合、最高優先順位のゾーンが勝利
5. 勝利したゾーンにリンクされた配送方法が表示される
6. どのゾーンにもリンクされていない（または該当するゾーンにリンクされている）配送方法は表示される

**ゾーンの構成要素**：
- **名前**：ゾーンの識別子（例："国内"、"EU"、"遠隔地域"）
- **国**：含まれる国コードのリスト（空 = すべての国）
- **州/県**：特定の国ごとの州の制限（任意）
- **郵便番号パターン**：ZIP/郵便番号のマッチング用の正規表現パターン（任意）
- **優先順位**：複数のゾーンが該当する場合、数値が高いほど優先順位が高い

## ゾーンのマッチングロジック

ゾーンは、住所のマッチングに**段階的な絞り込み**を使用します。

### レベル1: 国のマッチング

**国リストが空の場合** → ゾーンはすべての国にマッチ

**国リストが提供されている場合** → 住所の国がリストに含まれている必要があります

例：
```
ゾーン: "国内"
国: ["US"]
→ マッチ: 任意のUS住所
→ マッチしない: カナダ、イギリスなど
```

### レベル2: 州/県のマッチング

**州が定義されていない場合** → ゾーンは許可された国すべての州にマッチ

**特定の国に対して州が定義されている場合** → 住所の州が一致する必要があります

例：
```
ゾーン: "西海岸"
国: ["US"]
州: {"US": ["CA", "OR", "WA"]}
→ マッチ: カリフォルニア州、オレゴン州、ワシントン州の住所
→ マッチしない: ニューヨーク州、テキサス州など
```

### レベル3: 郵便番号のマッチング

**パターンが定義されていない場合** → ゾーンは許可された国/州すべての郵便番号にマッチ

**パターンが定義されている場合** → 住所の郵便番号が少なくとも1つのパターンと一致する必要があります

例：
```
ゾーン: "ロサンゼルスメトロ"
国: ["US"]
州: {"US": ["CA"]}
郵便番号パターン: ["^90[0-9]{3}$", "^91[0-9]{3}$"]
→ マッチ: 90001, 91210, 90245
→ マッチしない: 94102（サンフランシスコ）
```

**正規表現パターンの例**：
- `^90[0-9]{3}$` - ロサンゼルス地域（90000-90999）
- `^[A-Z][0-9][A-Z] [0-9][A-Z][0-9]$` - カナダの郵便番号形式（K1A 0B1）
- `^SW[0-9]{1,2}` - ロンドンUKのSWで始まる郵便番号

## 優先順位に基づくゾーン選択

複数のゾーンが住所にマッチする場合、**優先順位**がどのゾーンが適用されるかを決定します。

**優先順位の動作方法**：
- 数値が高いほど優先順位が高い
- 住所が優先順位100と50のゾーンにマッチする場合、優先順位100が勝利
- 勝利したゾーンの配送方法のみが利用可能

**使用例**：

**シナリオ1: 特定のゾーンが一般的なゾーンを上書き**
```
ゾーンA: "遠隔アラスカ"
  国: ["US"]
  州: {"US": ["AK"]}
  優先順位: 100

ゾーンB: "国内USA"
  国: ["US"]
  優先順位: 50

住所: アンカレッジ、AK
→ 両方のゾーンにマッチ
→ 優先順位100が勝利
→ "遠隔アラスカ"ゾーンが適用（高い配送費用）
```

**シナリオ2: 郵便番号が州を上書き**
```
ゾーンA: "マンハッタンプレミアム"
  国: ["US"]
  州: {"US": ["NY"]}
  郵便番号パターン: ["^100[0-2][0-9]$"]
  優先順位: 100

ゾーンB: "ニューヨーク州"
  国: ["US"]
  州: {"US": ["NY"]}
  優先順位: 50

住所: ニューヨーク、NY 10001
→ 両方のゾーンにマッチ
→ 優先順位100が勝利
→ "マンハッタンプレミアム"が適用（プレミアム配送サービス）
```

## 配送ゾーンの作成

**ステップバイステップワークフロー**：

1. **ゾーンへの移動**
   - 設定 > 配送 > 配送ゾーンに移動
   - "配送ゾーンを追加"をクリック

2. **基本設定**
   - **名前**：説明的な識別子（例："ヨーロッパ連合"、"西海岸"、"遠隔地域"）
   - **優先順位**：相対的な重要度を設定（100で特定、50で一般的、1でフォールバック）
   - **アクティブ**：有効/無効を切り替え

3. **地理的範囲の定義**

   **オプションA: すべての国**（国リストを空にします）
   - ゾーンはグローバルにすべての住所にマッチ
   - デフォルト/フォールバックゾーンとして使用

   **オプションB: 特定の国**
   - "国を追加"をクリック
   - ドロップダウンから国を選択（US、CA、UKなど）
   - 含まれるすべての国に対して繰り返す

   **オプションC: 特定の州/県**
   - 国を追加した後、各国に対して"州を追加"をクリック
   - 州をドロップダウンから選択
   - 例：US → CA、OR、WA（西海岸）

   **オプションD: 郵便番号パターン**（高度な機能）
   - 正規表現パターンを入力（1行ごとに1つ）
   - サンプル郵便番号でパターンをテスト
   - "パターンを検証"をクリックして構文を確認

4. **配送方法へのリンク**
   - 方法は、方法を編集中にリンクできます（ゾーン構成内ではできません）
   - または、ゾーンを既存の方法にリンク：方法を編集 → 配送ゾーン → ゾーンを選択

5. **表示優先順位の設定**
   - 複数のマッチがある場合、高い優先順位のゾーンが低い優先順位を上書き
   - おすすめ: 特定のゾーン（100）、地域のゾーン（50）、デフォルトゾーン（1）

6. **ゾーンの有効化**
   - "アクティブ"をトグル = はい
   - 保存

## 一般的なゾーン設定

### 設定1: 国内対国際

**目的**：国内とその他のすべての国に対して異なる料金を設定。

```
ゾーン1: "国内"
  国: [あなたの国コード]
  優先順位: 50

ゾーン2: "国際"
  国: [空に残すか、他のすべての国を選択]
  優先順位: 1
```

**配送方法**：
- "国内標準" → 国内ゾーンにリンク
- "国際配送" → 国際ゾーンにリンク

---

### 設定2: 多地域国際

**目的**：EU、北米、アジア、世界の他の地域に対して異なる料金を設定。

```
ゾーン1: "ヨーロッパ連合"
  国: [AT, BE, BG, CY, CZ, DE, DK, EE, ES, FI, FR, GR, HR, HU, IE, IT, LT, LU, LV, MT, NL, PL, PT, RO, SE, SI, SK]
  優先順位: 100

ゾーン2: "北米"
  国: [US, CA, MX]
  優先順位: 100

ゾーン3: "アジア太平洋"
  国: [AU, CN, HK, IN, JP, KR, NZ, SG, TH, TW]
  優先順位: 100

ゾーン4: "世界の他の地域"
  国: [空に残す]
  優先順位: 1
```

**配送方法**：
- "EU配送" → EUゾーン
- "北米配送" → 北米ゾーン
- "アジア太平洋配送" → アジア太平洋ゾーン
- "国際標準" → 世界の他の地域ゾーン

---

### 設定3: 遠隔地域の追加料金

**目的**：国内ゾーン内の遠隔郵便番号に追加料金を設定。

```
ゾーン1: "遠隔国内"
  国: [US]
  郵便番号パターン: ["^99[0-9]{3}$", "^96[7-9][0-9]{2}$"]  # アラスカ、ハワイ
  優先順位: 100

ゾーン2: "標準国内"
  国: [US]
  優先順位: 50
```

**配送方法**：
- "遠隔配送" → 遠隔国内ゾーン（高コスト）
- "標準配送" → 標準国内ゾーン

---

### 設定4: 州ごとのゾーン

**目的**：米国の各地域に異なる料金を設定。

```
ゾーン1: "西海岸"
  国: [US]
  州: {"US": ["CA", "OR", "WA"]}
  優先順位: 100

ゾーン2: "東海岸"
  国: [US]
  州: {"US": ["NY", "NJ", "CT", "MA", "PA"]}
  優先順位: 100

ゾーン3: "ミッドウエスト"
  国: [US]
  州: {"US": ["IL", "IN", "OH", "MI", "WI"]}
  優先順位: 100

ゾーン4: "南"
  国: [US]
  州: {"US": ["TX", "FL", "GA", "NC", "SC"]}
  優先順位: 100

ゾーン5: "その他のUS州"
  国: [US]
  優先順位: 50
```

## 郵便番号パターンの例

郵便番号は**正規表現**（正規表現）を使用してパターンマッチングを行います。

### アメリカ合衆国（ZIPコード）

**形式**：5桁（例：90210）

```
カリフォルニア州（90000-96199）:  ^9[0-6][0-9]{3}$
ニューヨーク州（10000-14999）:    ^1[0-4][0-9]{3}$
テキサス州（75000-79999, 88500-88599）:  ^(7[5-9]|885)[0-9]{2}$
アラスカ州（99500-99999）:      ^99[5-9][0-9]{2}$
```

### カナダ（郵便番号）

**形式**：A1A 1A1（文字-数字-文字 空白 数字-文字-数字）

```
すべてのカナダの郵便番号:  ^[A-Z][0-9][A-Z] [0-9][A-Z][0-9]$ 
オンタリオ州（K, L, M, N, P）:    ^[KLMNP][0-9][A-Z] [0-9][A-Z][0-9]$ 
ケベック州（G, H, J）:           ^[GHJ][0-9][A-Z] [0-9][A-Z][0-9]$ 
```

### 英国（郵便番号）

**形式**：AA1A 1AA または A1A 1AA

```
ロンドン（E, EC, N, NW, SE, SW, W, WC）:  ^(E|EC|N|NW|SE|SW|W|WC)[0-9]{1,2}
マンチェスター（M）:                        ^M[0-9]{1,2}
バーミンガム（B）:                        ^B[0-9]{1,2}
```

### オーストラリア（郵便番号）

**形式**：4桁（例：2000）

```
ニューサウスウェールズ州（1000-2999）:  ^[12][0-9]{3}$
ビクトリア州（3000-3999, 8000-8999）:  ^[38][0-9]{3}$
クイーンズランド州（4000-4999, 9000-9999）:  ^[49][0-9]{3}$
```

### パターンのテスト

**パターンを保存する前**に、既知の郵便番号でテストしてください：

1. パターンを入力: `^90[0-9]{3}$`
2. テスト入力: "90210" → マッチする必要があります
3. テスト入力: "10001" → マッチしない必要があります
4. テスト入力: "9021" → マッチしない必要があります（4桁のみ）

複雑なパターンの検証には、オンラインの正規表現テストツール（regex101.com）を使用してください。

## ゾーンカバレッジのサマリ

ゾーンは、管理画面のリストビューで**カバレッジサマリ**を表示します。これは、含まれている内容を示します：

**例**：
- "すべての国" → 国の制限なし
- "US, CA, MX" → 3か国
- "US (CA, OR, WA)" → USに3つの州が含まれる
- "US (90xxx-91xxx)" → USに郵便番号パターンが含まれる

**サマリの使用**：
- 画面を開かずに、迅速にゾーンのカバレッジを確認
- カバレッジの重複やギャップを発見
- 一目でゾーン構成を監査

## 配送方法へのゾーンのリンク

ゾーンと方法は**多対多の関係**を持っています：

**方法側から**（推奨）：
1. 配送方法を編集
2. "配送ゾーン"セクションにスクロール
3. 適用可能なゾーンを選択（マルチ選択）
4. 方法を保存

**ゾーン側から**：
- ゾーンは直接的に方法にリンクできません
- リンクは常に方法構成から行われます

**方法-ゾーンの動作**：

**リンクされたゾーンがない** → 方法はすべての住所に利用可能

**リンクされたゾーンがある** → 方法は、顧客の住所が少なくとも1つのリンクされたゾーンにマッチする場合にのみ利用可能

**例**：
```
方法: "国内標準"
リンクされたゾーン: ["国内USA"]
→ US住所のみに表示

方法: "国際エクスプレス"
リンクされたゾーン: ["EU", "アジア太平洋", "世界の他の地域"]
→ すべての非US住所に表示
```

## ゾーンマッチングのテスト

本番環境にリリースする前に、ゾーン構成をテストしてください：

1. **テスト注文の作成**
   - 異なるゾーンの住所を使用
   - 正しいゾーンマッチが確認される

2. **優先順位の解決を確認**
   - 複数のゾーンにマッチする住所を使用
   - 最高優先順位のゾーンが勝利することを確認
   - 期待される配送方法が表示されることを確認

3. **エッジケースのテスト**
   - 境界の郵便番号（例：90999 vs 91000）
   - 州の境界
   - 類似した郵便番号を持つ国際住所

4. **ゾーンプレビューツールの使用**（利用可能であれば）
   - テスト住所を入力
   - どのゾーンがマッチするかを確認
   - 優先順位の解決を確認

## トラブルシューティング

**問題1: チェックアウト時に利用可能な配送方法がない**

**原因**：
- 顧客の住所がどのゾーンにもマッチしない
- すべての方法がマッチしないゾーンにリンクされている
- ゾーン制限なしの方法が存在しない

**解決策**：
- フォールバックゾーン（すべての国、優先順位1）を作成
- または、少なくとも1つの方法からゾーン制限を削除
- ゾーンの国/州/郵便番号パターンを確認

---

**問題2: 間違ったゾーンがマッチ**

**原因**：
- 高優先順位のゾーンがマッチしているにもかかわらず、低優先順位のゾーンが選択されている
- 郵便番号パターンの構文エラー（パターンが静かに失敗）
- 州コードの不一致（CA vs カリフォルニア）

**解決策**：
- 優先順位値を確認（数値が高いほど優先順位が高い）
- 郵便番号パターンを正規表現検証ツールでテスト
- 2文字の州コードを使用（CA、ではなくカリフォルニア、NY、ではなくニューヨーク）

---

**問題3: 予期しない方法が表示**

**原因**：
- 方法にリンクされたゾーンがない（すべての住所に利用可能）
- 複数のゾーンがマッチし、予期しないゾーンが優先順位が高い
- ゾーンのカバレッジが意図せず重複している

**解決策**：
- 方法のリンクされたゾーンを確認
- マッチするゾーンの優先順位を確認
- ゾーンカバレッジサマリで重複を監査

---

## ヒント

- **2つのゾーンから始める** - 国内と国際、必要に応じて拡張
- **優先順位を賢く使用** - 特定のゾーン100、地域50、フォールバック1
- **郵便番号パターンを徹底的にテスト** - 正規表現エラーは静かに失敗し、ゾーンがマッチしない原因になる
- **ゾーンロジックを文書化** - ゾーンの説明にノートを追加してカバレッジの意図を説明
- **過剰なゾーンを避ける** - ゾーンが多すぎると構成が複雑になる。複雑なシナリオには配送ルールを使用
- **州コードではなく名前を使用** - "CA"ではなく"カリフォルニア"、"NY"ではなく"ニューヨーク"
- **フォールバックゾーンを作成** - すべての国、優先順位1、これにより常に少なくとも1つの配送オプションが利用可能
- **ゾーンのパフォーマンスを監視** - 複数の顧客が"配送不可"を表示する場合、ゾーンのカバレッジを監査
- **新しい地域に応じてゾーンを更新** - 新しいEU加盟国が加わった場合、EUゾーンに国を追加
- **説明的な名前を使用** - "EU（イギリスを除く）"の方が"ゾーン3"よりも良い
- **実際の住所でテスト** - テスト時には顧客の実際の住所を使用し、架空の住所は使用しない
