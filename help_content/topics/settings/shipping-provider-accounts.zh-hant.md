---
title: 運遞服務商帳號
---

運遞服務商帳號可將您的商店連接到運遞商 API（如 FedEx、UPS、DHL），以進行即時運費計算和自動標籤購買。每個帳號會儲存加密的 API 權限資料，監控連線狀態，並連結到即時運遞方式。運遞商會根據包裹尺寸、重量、出發地和目的地在結帳時取得即時運費——消除手動維護運費表的需要，並確保準確的運遞商定價。

當您在需要運遞商計算的運費或自動標籤生成，而不是手動建立運送時，請使用服務商帳號。

## 支援的運遞服務商

Spwig 透過可安裝的服務商組件支援主要的運遞商：

### FedEx

**服務**: 地面運送、快遞、國際運送
**API**: FedEx Web Services
**功能**: 即時運費、標籤購買、追蹤、國際海關文件

### UPS

**服務**: 地面運送、航空運送、全球運送
**API**: UPS Developer API
**功能**: 即時運費、標籤生成、追蹤、地址驗證

### DHL

**服務**: 快遞、電商、國際運送
**API**: DHL Express API
**功能**: 國際運費、海關文件、追蹤

### 其他服務商

根據需要從組件市場安裝（USPS、加拿大郵政、澳洲郵政等）

## 服務商帳號設定

每個服務商帳號需要：

### 基本資訊

- **顯示名稱**: 帳號在管理介面中顯示的方式（例如，"FedEx 生產帳號"）
- **服務商**: 從下拉選單中選擇已安裝的服務商組件
- **啟用**: 切換以啟用或停用帳號，而無需刪除權限資料
- **設為預設**: 設為此服務商的預設帳號（每個服務商只能有一個預設）

### API 權限資料（加密）

**因服務商而異**，通常包括：

**FedEx**:
- 帳號
- 測量編號
- API 金鑰
- API 密碼

**UPS**:
- 接入許可編號
- 使用者 ID
- 密碼
- 帳號

**DHL**:
- 網站 ID
- 密碼
- 帳號

**所有權限資料在靜態狀態下均加密**，僅在進行 API 呼叫時解密。

### 出發地址

- **預設出貨地址**: 用於運費計算的倉庫/出發地址
- 有些服務商需要在他們的儀表板中設定特定的出發地

### 設定

因服務商而異的選項（視運遞商而定）：

- **測試模式**: 使用運遞商的沙盒/測試 API 端點
- **談判後的運費**: 使用您與運遞商談判後的運費（如果可用）
- **包含保險**: 在運費中自動報價保險
- **住宅附加費用**: 套用住宅配送費用
- **需要簽收**: 預設簽收要求

## 建立服務商帳號

**6 步驟設定流程**：

**步驟 1：取得運遞商 API 權限**
1. 在運遞商處建立帳號（FedEx.com、UPS.com、DHL.com）
2. 申請 API/開發者權限
3. 完成運遞商的 API 建立流程（可能需要 1-3 個工作天）
4. 透過電子郵件或開發者入口網站接收 API 權限資料

**步驟 2：安裝服務商組件**（如果尚未預先安裝）
1. 前往設定 > 組件 > 市場
2. 搜尋運遞商名稱（例如，"FedEx"）
3. 安裝運遞商組件
4. 等待安裝完成

**步驟 3：在 Spwig 中建立服務商帳號**
1. 前往設定 > 運送 > 服務商帳號
2. 點擊 "新增服務商帳號"
3. 從下拉選單中選擇服務商
4. 輸入顯示名稱

**步驟 4：輸入 API 權限資料**
1. 填寫權限資料欄位（因服務商而異）
2. 權限資料在儲存時會自動加密
3. 可選：啟用測試模式以進行初步測試

**步驟 5：測試連線**
1. 點擊 "測試連線" 按鈕
2. 系統會嘗試向運遞商發出 API 呼叫
3. 確認顯示 "已連線" 狀態
4. 檢查 last_tested_at 時間戳記

**步驟 6：連結到運送方式**
1. 建立或編輯運送方式（設定 > 紙箱 > 運送方式）
2. 設定 method_type = "即時"
3. 從下拉選單中選擇服務商帳號
4. 儲存運送方式

## 連線狀態監控

服務商帳號會追蹤連線狀態：

### 狀態值

**未知**（灰色）：從未測試過或尚未連線

**已連線**（綠色）：上次 API 呼叫成功，權限資料有效

**錯誤**（紅色）：上次 API 呼叫失敗，權限資料可能無效

### 最後測試時間

- **時間戳記**：連線最後驗證的時間
- **自動更新**：每次使用服務商時（運費查詢、標籤購買）
- **手動測試**：隨時點擊 "測試連線" 按鈕

### 解決連線失敗

**常見原因**：
- 錯誤的 API 權限資料（拼寫錯誤、複製時有額外空格）
- 運遞商 API 金鑰過期或被撤銷
- 啟用了測試模式但使用的是生產環境權限資料（或相反）
- IP 位址未在運遞商處白名單
- 運遞商 API 當機

**解決步驟**：
1. 確認權限資料與運遞商儀表板完全一致
2. 檢查測試模式設定是否與權限資料類型相符
3. 查看運遞商的 API 狀態頁面以確認是否有中斷
4. 聯絡運遞商支援以驗證帳號

## 運費查詢工作流程

結帳時即時運費的運作方式：

**1. 顧客輸入地址**
- 輸入運送地址
- 計算購物車總重量 + 維度

**2. 系統準備運費請求**
- 取得服務商帳號權限資料（解密後）
- 從購物車項目計算包裹維度（如果定義了運送包裹）
- 準備包含出發地、目的地、包裹的 API 請求

**3. 呼叫服務商 API**
- 向運遞商 API 發送請求，附上驗證權限資料
- 運遞商根據區域、重量、維度計算運費
- 回應包含服務選項（地面運送、快遞等）

**4. 顯示運費**
- 系統解析運遞商回應
- 正規化為標準格式
- 如果設定，套用附加費用
- 在結帳時顯示運費給顧客

**5. 顧客選擇服務**
- 顧客選擇偏好的選項
- 選定的運費儲存到訂單中

**API 流程範例**：
```
向 FedEx API 的請求：
{
  "origin": {"postal_code": "90210", "country": "US"},
  "destination": {"postal_code": "10001", "country": "US"},
  "parcels": [{
    "weight": 2500,  // 克
    "dimensions": {"length": 30, "width": 20, "height": 15}  // 公分
  }]
}

FedEx 回應：
[
  {"service": "FEDEX_GROUND", "rate": 12.50, "delivery_days": 5},
  {"service": "FEDEX_EXPRESS", "rate": 28.75, "delivery_days": 2}
]
```

## 標籤購買（可選）

如果服務商支援標籤生成：

**工作流程**：
1. 顧客完成訂單
2. 商家建立運送（訂單 > 訂單明細 > 建立運送）
3. 選擇服務商帳號 + 服務
4. 系統呼叫服務商的標籤 API
5. 產生標籤 PDF 並附加到運送
6. 自動填寫追蹤號碼
7. 標籤準備列印

**優點**：
- 無需手動登入運遞商網站
- 追蹤自動同步
- 自動產生海關文件（國際運送）
- 可批量產生標籤

## 運費附加費用

向運遞商運費添加商家附加費用：

**設定**（在運送方式中，而非服務商帳號中）：
- **附加費用類型**：百分比或固定金額
- **附加費用金額**：例如，15% 或 $2.50

**範例**：
```
運遞商運費：$12.50
附加費用：15%
顧客支付：$14.38

或

運遞商運費：$12.50
附加費用：$2.50（固定）
顧客支付：$15.00
```

**使用案例**：
- 覆蓋包裝/處理成本
- 向運送添加利潤邊際
- 抵消運送的信用卡費用

## 多個服務商帳號

您可以為同一服務商建立多個帳號：

**使用案例**：
1. **測試 vs 生產**
   - 測試帳號：運遞商沙盒權限資料
   - 生產帳號：實際權限資料

2. **多個倉庫**
   - 倉庫 A 帳號：出發地 = 洛杉磯
   - 倉庫 B 帳號：出發地 = 紐約

3. **不同的談判後運費**
   - 帳號 A：標準運費
   - 帳號 B：批量折扣運費

**每個帳號可以連結到不同的運送方式**，以實現靈活的設定。

## 小技巧

- **先在沙盒測試** - 在實際使用前使用運遞商測試權限資料
- **監控連線狀態** - 定期檢查儀表板上的錯誤狀態
- **定義運送包裹** - 精確的維度可提高運費報價
- **使用談判後的運費** - 如果您與運遞商有批量折扣，請啟用此功能
- **設定實際的出發地** - 使用實際的出貨地址以獲得準確的區域
- **保持權限資料安全** - 永遠不要分享 API 金鑰，定期輪換
- **保留備用方法** - 如果運遞商 API 失敗，請保持固定運費方式啟用
- **監控運遞商 API 限制** - 有些運遞商限制每天的 API 呼叫次數
- **及時更新權限資料** - 當運遞商輪換金鑰時，請立即更新
- **使用描述性的名稱** - "FedEx LA 倉庫" 比 "FedEx 1" 更好
