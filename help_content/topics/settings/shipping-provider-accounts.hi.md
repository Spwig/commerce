title: वितरण प्रदाता खाता

वितरण प्रदाता खाता आपके स्टोर को वितरक API (FedEx, UPS, DHL) से जोड़ता है जिससे वास्तविक समय में दर की गणना और लेबल की स्वचालित खरीद हो सके। प्रत्येक खाता एन्क्रिप्टेड API प्रमाणपत्रों को संग्रहित करता है, कनेक्शन के स्वास्थ्य की निगरानी करता है, और वास्तविक समय में वितरण विधियों से जुड़ा रहता है। प्रदाता चेकआउट के आधार पर भार, मूल, गंतव्य और पैकेज के आयाम के आधार पर वास्तविक समय में दर ले आते हैं - जिससे मानक दर तालिका बनाए रखने की आवश्यकता खत्म हो जाती है और वितरक की दर के सही अनुमान बनाए रखे जाते हैं।

जब आपको वितरक द्वारा गणना की गई वितरण दर या स्वचालित लेबल उत्पादन की आवश्यकता होती है, तो प्रदाता खाता का उपयोग करें।

## समर्थित वितरण प्रदाता

Spwig इस्तेमाल करने योग्य प्रदाता घटकों के माध्यम से प्रमुख वितरकों का समर्थन करता है:

### FedEx

**सेवाएं**: ग्राउंड, एक्सप्रेस, अंतरराष्ट्रीय
**API**: FedEx वेब सेवाएं
**विशेषताएं**: वास्तविक समय में दर, लेबल खरीद, ट्रैकिंग, अंतरराष्ट्रीय कर नियम

### UPS

**सेवाएं**: ग्राउंड, एयर, वर्ल्डवाइड
**API**: UPS डेवलपर API
**विशेषताएं**: वास्तविक समय में दर, लेबल उत्पादन, ट्रैकिंग, पता सत्यापन

### DHL

**सेवाएं**: एक्सप्रेस, ई-कॉमर्स, अंतरराष्ट्रीय
**API**: DHL एक्सप्रेस API
**विशेषताएं**: अंतरराष्ट्रीय दर, कस्टम दस्तावेज, ट्रैकिंग

### अतिरिक्त प्रदाता

जैसा आवश्यक हो, कॉम्पोनेंट मार्केटप्लेस से इन्स्टॉल करें (USPS, कैनाडा पोस्ट, ऑस्ट्रेलिया पोस्ट, आदि)

## प्रदाता खाता सेटअप

प्रत्येक प्रदाता खाता के लिए:

### बुनियादी जानकारी

- **प्रदर्शन नाम**: खाता प्रशासन में कैसे दिखता है (उदाहरण के लिए, "FedEx प्रोडक्शन अकाउंट")
- **प्रदाता**: ड्रॉपडाउन से इंस्टॉल किए गए प्रदाता कॉम्पोनेंट का चयन करें
- **सक्रिय**: बिना एक्रेडेंशियल्स के डिलीट किए बिना एनेबल/डिसएबल करने के लिए स्विच करें
- **डिफ़ॉल्ट है**: इस प्रदाता के लिए डिफ़ॉल्ट खाता के रूप में सेट करें (प्रत्येक प्रदाता के लिए केवल एक डिफ़ॉल्ट)

### API एक्रेडेंशियल्स (एन्क्रिप्टेड)

**प्रदाता द्वारा भिन्न**, आमतौर पर शामिल होता है:

**FedEx**:
- खाता संख्या
- मीटर संख्या
- API कुंजी
- API रहस्य

**UPS**:
- एक्सेस लाइसेंस संख्या
- उपयोगकर्ता के आईडी
- पासवर्ड
- खाता संख्या

**DHL**:
- साइट ID
- पासवर्ड
- खाता संख्या

**सभी एकाउंट विश्राम पर एन्क्रिप्ट किए गए हैं** और केवल API कॉल करते समय डिक्रिप्ट किए जाते हैं।

### मूल पता

- **डिफ़ॉल्ट शिप फ्रॉम**: दर गणना के लिए वर्कशॉप/मूल पता
- कुछ प्रदाताओं के डैशबोर्ड में विशिष्ट मूल सेटअप की आवश्यकता होती है

### सेटिंग्स

प्रदाता-विशिष्ट विकल्प (वाहक द्वारा भिन्न):

- **टेस्ट मोड**: वाहक के सैंडबॉक्स/टेस्ट API अंतबिंदुओं का उपयोग करें
- **सौदा दर**: अपनी सौदा वाहक दर का उपयोग करें (यदि उपलब्ध है)
- **बीमा शामिल करें**: दरों में बीमा के लिए स्वचालित उद्धरण शामिल करें
- **निवासी शुल्क**: निवासी डिलीवरी शुल्क लागू करें
- **हस्ताक्षर आवश्यक**: डिफ़ॉल्ट हस्ताक्षर आवश्यकताएं

## एक प्रदाता एकाउंट बनाएं

**6-चरण सेटअप प्रक्रिया**:

**चरण 1: वाहक API प्रवेश प्राप्त करें**
1. वाहक के साथ एकाउंट बनाएं (FedEx.com, UPS.com, DHL.com)
2. API/विकासक एक्सेस के लिए आवेदन करें
3. वाहक के API ऑनबोर्डिंग को पूरा करें (1-3 व्यावसायिक दिनों के लिए लग सकता है)
4. ईमेल या विकासक पोर्टल के माध्यम से API एकाउंट प्रमाणपत्र प्राप्त करें

**चरण 2: प्रदाता घटक इंस्टॉल करें** (यदि पूर्व में इंस्टॉल नहीं किया गया है)
1.

सेटिंग्स > कॉम्पोनेंट्स > मार्केटप्लेस जाएं
2.

कैरियर के नाम के लिए खोजें (उदाहरण के लिए, "FedEx")
3.

शिपिंग प्रोवाइडर कॉम्पोनेंट को इंस्टॉल करें
4.

इंस्टॉलेशन पूरा होने तक प्रतीक्षा करें

**चरण 3: Spwig में प्रोवाइडर खाता बनाएं**
1. सेटिंग्स > शिपिंग > प्रोवाइडर खाता जाएं
2. "Add Provider Account" पर क्लिक करें
3. ड्रॉपडाउन से प्रोवाइडर का चयन करें
4. डिस्प्ले नाम दर्ज करें

**चरण 4: API एक्रेडेंशियल्स दर्ज करें**
1. एक्रेडेंशियल फील्ड्स भरें (प्रोवाइडर द्वारा भिन्न)
2. बचाने पर एक्रेडेंशियल्स स्वचालित रूप से एन्क्रिप्ट हो जाते हैं
3. वैकल्पिक: प्रारंभिक परीक्षण के लिए परीक्षण मोड को चालू करें

**चरण 5: कनेक्शन की जांच करें**
1. "Test Connection" बटन पर क्लिक करें
2. प्रणाली कैरियर के एपीआई के लिए एपीआई कॉल करने की कोशिश करती है
3. यह जांचें कि "Connected" स्थिति दिखाई देती है
4. अंतिम परीक्षण के समयसीमा की जांच करें

**चरण 6: शिपिंग विधि के साथ लिंक करें**
1. शिपिंग विधि बनाएं या संपादित करें (सेटिंग्स > कार्ट > शिपिंग विधियां)
2. method_type = "Real-Time" रखें
3. ड्रॉपडाउन से प्रोवाइडर खाता चुनें
4. विधि को सहेजें

## कनेक्शन स्थिति निगरानी

प्रोवाइडर खाता कनेक्शन स्वास्थ्य की ट्रैक करता है:

### स्थिति मान


**अज्ञात** (ग्रे): कभी टेस्ट नहीं किया गया या अभी तक कनेक्ट नहीं किया गया

**कनेक्ट किया गया** (हरा): अंतिम API कॉल सफल रहा, एक्रेडेंटियल्स वैध हैं

**त्रुटि** (लाल): अंतिम API कॉल विफल रहा, एक्रेडेंटियल्स अमान्य हो सकते हैं

### अंतिम टेस्ट किया गया

- **समय अंतराल**: जब कनेक्शन की जांच की गई थी
- **स्वचालित अपडेट**: हर बार जब प्रोवाइडर का उपयोग किया गया (दर ले आओ, लेबल खरीद)
- **मैन्युअल टेस्ट**: कोई भी समय "कनेक्शन टेस्ट" बटन दबाएं

### विफल कनेक्शन के त्रुटि निवारण

**सामान्य कारण**:
- गलत API एक्रेडेंटियल्स (लेखन त्रुटि, अतिरिक्त जगह के साथ कॉपी किया गया)
- कैरियर API की अवधि समाप्त हो गई या रद्द कर दी गई
- परीक्षण मोड योग्य लेकिन उत्पादन एक्रेडेंटियल्स का उपयोग कर रहे हैं (या विपरीत)
- कैरियर के साथ आईपी पता सफेद सूची में नहीं है
- कैरियर API डाउनटाइम

**समाधान कदम**:
1. एक्रेडेंटियल्स की जांच करें कि कैरियर डैशबोर्ड के साथ पूरी तरह से मेल खाते हैं
2. परीक्षण मोड सेटिंग की जांच करें कि एक्रेडेंटियल टाइप के साथ मेल खाता है
3. कैरियर के API स्थिति पृष्ठ की जांच करें बंद नहीं हो रहा है
4. कैरियर समर्थन से संपर्क करें खाता सत्यापन के लिए

## दर खोज वर्कफ्लो

चेकआउट पर वास्तविक समय दर कैसे काम करता है:

**1.

**ग्राहक पता दर्ज करता है**
- शिपिंग पता दर्ज किया गया
- कार्ट वजन + आयाम की कुल राशि की गणना करता है

**2. प्रणाली दर अनुरोध तैयार करती है**
- प्रदाता खाता विवरण प्राप्त करता है (डिक्रिप्ट किया गया)
- कार्ट आइटम से पैकेज आयाम की गणना करता है (यदि परिभाषित किया गया है तो शिपिंग पैकेज का उपयोग करता है)
- मूल, गंतव्य और भार वाले डेटा के साथ API अनुरोध तैयार करता है

**3. प्रदाता API को कॉल किया जाता है**
- वहां एक वितरक API के साथ अनुरोध भेजा जाता है जिसमें प्रमाणीकरण विवरण शामिल होता है
- वितरक जोन, वजन और आयाम के आधार पर दर की गणना करता है
- प्रतिक्रिया में सेवा विकल्प (जमीनी, एक्सप्रेस, आदि) शामिल होता है

**4. दर दिखाई गई हैं**
- प्रणाली वितरक प्रतिक्रिया का विश्लेषण करती है
- मानक फॉर्मेट में सामान बनाती है
- यदि निर्धारित किया गया है तो वैकल्पिक मार्कअप लागू किया जाता है
- चेकआउट पर ग्राहक को दर दिखाई जाती है

**5. ग्राहक सेवा चुनता है**
- ग्राहक अपने पसंदीदा विकल्प को चुनता है
- चुनी गई दर को ऑर्डर में सहेजा जाता है

**उदाहरण API प्रवाह**:
```
FedEx API के लिए अनुरोध:
{
  "origin": {"postal_code": "90210", "country": "US"},
  "destination": {"postal_code": "10001", "country": "US"},
  "parcels": [{
    "weight": 2500,  // grams
    "dimensions": {"length": 30, "width": 20, "height": 15}  // cm
  }]
}

FedEx प्रतिक्रिया:
[
  {"service": "FEDEX_GROUND", "rate": 12.50, "delivery_days": 5},
  {"service": "FEDEX_EXPRESS", "rate": 28.75, "delivery_days": 2}
]
```

## लेबल खरीद (वैकल्पिक)

यदि प्रदाता लेबल उत्पादन के समर्थन के साथ है:

**कार्यप्रवाह**:
1. ग्राहक ऑर्डर पूरा करता है
2. व्यापारी भेजी गई वस्तु बनाता है (ऑर्डर > ऑर्डर विवरण > भेजी गई वस्तु बनाएं)
3. प्रदाता खाता + सेवा चुनें
4. प्रणाली प्रदाता के लेबल API को कॉल करती है
5. लेबल PDF उत्पन्न किया जाता है और भेजी गई वस्तु से जुड़ा होता है
6. ट्रैकिंग संख्या स्वचालित रूप से भर जाता है
7. लेबल प्रिंट करने के लिए तैयार है

**लाभ**:
- कोई मानक वाहक वेबसाइट लॉगिन नहीं
- ट्रैकिंग स्वचालित रूप से सिंक्रनाइज़ किया जाता है
- अंतरराष्ट्रीय शुल्क फॉर्म स्वचालित रूप से उत्पन्न किए जाते हैं
- बैच लेबल उत्पादन संभव है

## दर अतिरिक्त

वहां व्यापक लाभ के लिए वाहक दरों में एक अतिरिक्त लागत जोड़ें:

**सेटअप** (वितरण विधि में, प्रदाता खाता के बजाय):
- **लाभ के प्रकार**: प्रतिशत या निश्चित
- **लाभ की राशि**: उदाहरण के लिए, 15% या $2.50

**उदाहरण**:
```
वाहक दर: $12.50
लाभ: 15%
ग्राहक द्वारा भुगतान: $14.38

या

वाहक दर: $12.50
लाभ: $2.50 (निश्चित)
ग्राहक द्वारा भुगतान: $15.00
```

**उपयोग के मामले**:
- पैकेजिंग/हैंडलिंग लागत को कवर करें
- वितरण के लिए लाभ मार्जिन जोड़ें
- वितरण पर क्रेडिट कार्ड शुल्क को बराबर करें

## एकाधिक प्रदाता खाता

आप एक ही प्रदाता के लिए एकाधिक खाता बना सकते हैं:

**उपयोग के मामले**:
1. **परीक्षण विरोधी उत्पादन**
   - परीक्षण खाता: वाहक सैंडबॉक्स योग्यता
   - उत्पादन खाता: जीवित योग्यता

2. **एकाधिक वर्कशॉप**
   - वर्कशॉप A खाता: मूल = लॉस एंजिल्स
   - वर्कशॉप B खाता: मूल = न्यूयॉर्क

3. **अलग-अलग निर्धारित दर**
   - खाता A: मानक दर
   - खाता B: आय छूट दर

**प्रत्येक खाता अलग-अलग वितरण विधि से जुड़ सकता है** अधिक लचीला सेटअप के लिए।

## सुझाव

सभी मार्कडाउन फॉर्मेटिंग, इमेज पथ, कोड ब्लॉक और तकनीकी शब्दों को संरक्षित करें।

- **पहले सैंडबॉक्स में परीक्षण करें** - लाइव होने से पहले कैरियर के परीक्षण प्रमाणपत्र का उपयोग करें
- **संयोजन के स्थिति की निगरानी करें** - डैशबोर्ड पर त्रुटि स्थितियों की नियमित जांच करें
- **वितरण पैकेज परिभाषित करें** - सटीक आयाम दर उद्धरण में सुधार करते हैं
