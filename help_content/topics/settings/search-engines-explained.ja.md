---
title: 検索エンジンの説明
---

Spwigの検索エンジンは、ElasticsearchやAlgoliaのような外部サービスではなく、ストアのデータベースネイティブ検索システム内の構成コンテキストです。各エンジンは、検索対象のコンテンツタイプ、除外する内容、および結果のランキング方法を定義します。このガイドでは、検索エンジンとは何か、複数のエンジンを作成するタイミング、および構成方法について説明します。

ほとんどの販売業者は、1つのデフォルトの"shop"エンジンを使用します。異なるコンテンツミックスや除外が必要なユースケースがある場合にのみ、複数のエンジンを作成してください。

![検索エンジン一覧](/static/core/admin/img/help/search-engines-explained/search-engines-list.webp)

## 検索エンジンとは？

Spwigの検索エンジンは、名前付きの構成で、以下を指定します：

- **検索対象のコンテンツタイプ**（商品、カテゴリ、ブランド、ブログ投稿）
- **除外する内容**（検索から隠したい特定のカテゴリやブランド）
- **カスタムな関連性の重み**（エンジンごとのオプションの重みの上書き）
- **アクティブ状態**（エンジンは一時的に無効にできます）

各エンジンには、API呼び出しやフロントエンドコードで検索リクエストを処理するエンジンを指定するために使用される一意のslugがあります。

## 複数のエンジンを作成するタイミング

ほとんどのストアでは1つのエンジンで十分です。以下のようなシナリオで追加のエンジンを作成してください：

| ユースケース | 例 |
|-------------|-----|
| **異なるコンテンツミックス** | Shopエンジンは商品のみを検索；Blogエンジンはブログ投稿のみを検索 |
| **選択的な除外** | メインのShopエンジンはセールカテゴリを非表示；セールエンジンはセール商品のみを表示 |
| **部門別検索** | 電子機器エンジンは服飾カテゴリを除外；服飾エンジンは電子機器を除外 |
| **B2BとB2Cの分離** | 販売用エンジンはバルク商品のみを表示；小売用エンジンは消費者向け商品のみを表示 |

複数のエンジンが必要かどうかが分からない場合は、1つだけ使用してください。特定のユースケースがない限り、エンジンを追加しても利点はなく、複雑さが増します。

## 4ステップのウィザード

![Wizard Step 1 - Basic Info](/static/core/admin/img/help/search-engines-explained/wizard-step1-basic.webp)

**Search > Setup Wizard**にアクセスして、ガイド付き4ステップのプロセスを通じて新しいエンジンを作成します:

### Step 1: Basic Information

**Engine Name** - 友好的な表示名（例: "Shop Search", "Blog Search"）。管理インターフェースでのみ使用されます。

**Slug** - URLに安全な識別子（例: "shop-search", "blog-search"）。API呼び出しとフロントエンドコードで使用されます。名前を空のままにした場合、自動生成されます。

**Active** - このエンジンが検索に利用可能かどうか。非アクティブなエンジンは検索結果を返しません。

### Step 2: Content Types

このエンジンが検索するコンテンツタイプを選択してください:

- Products (すべての製品タイプを含む: 物理的、デジタル、サブスクリプション)
- Categories
- Brands
- Blog Posts

**Tip**: このエンジンの目的に合ったコンテンツタイプのみを選択してください。ブログに特化したエンジンは製品を有効にする必要はありません。

### Step 3: Weights (Optional)

![Wizard Step 3 - Weights](/static/core/admin/img/help/search-engines-explained/wizard-step3-weights.webp)

この特定のエンジンの関連性の重みを任意でカスタマイズできます。スキップした場合、エンジンはSearchSettingsからグローバルな重みを継承します。

ほとんどのエンジンはこのステップをスキップし、グローバルなデフォルトを使用する必要があります。このエンジンがユニークなランキングの必要性がある場合のみ、重みをカスタマイズしてください（例: ブログエンジンではweight_blog_postsを1.2に増やす場合）。

### Step 4: Review & Create

設定内容を確認し、**Create Engine**をクリックして保存してください。

## Engine Configuration Fields

ウィザードをバイパスしてエンジンを直接編集する場合、これらのフィールドが表示されます:

**Name and Slug** - 表示名とURL識別子

**Active Status** - 有効/無効を切り替えます

**Content Types** - JSON配列例: `['product', 'category']`

**Weight Overrides** - JSONオブジェクト例: `{'weight_name': 1.8}`（グローバルな重みを使用している場合は空）

**Excluded Categories** - CategoryモデルへのM2M関係。これらのカテゴリに属する製品は検索結果に表示されません。

**除外ブランド** - BrandモデルとのM2M関係。

これらのブランドを持つ商品は、検索結果に表示されません。

## 除外の使用方法

除外は、このエンジンの検索結果から特定のコンテンツを非表示にします。

**例: クリアランス商品を非表示にする**

1. "Main Shop"エンジンを作成します。
2. "Excluded Categories"（除外カテゴリ）フィールドに、あなたの"Clearance"（クリアランス）カテゴリを選択します。
3. "Excluded Brands"（除外ブランド）フィールドに、非表示にしたい予算ブランドを選択します。
4. 保存します。

"Main Shop"エンジンを通じた検索は、サイト上では表示されているにもかかわらず、クリアランス商品を返しません。別途"Clearance"（クリアランス）エンジンを作成し、それだけを検索するようにすることもできます。

## フロントエンドでのエンジンの使用

フロントエンドコードは、API呼び出しを通じてどのエンジンを使用するかを指定します。

```javascript
// "shop"エンジンを使用（最も一般的）
fetch('/api/search/?q=laptop&engine=shop')

// "blog"エンジンを使用
fetch('/api/search/?q=ecommerce tips&engine=blog')

// engineパラメータが指定されていない場合のデフォルトエンジン
fetch('/api/search/?q=laptop')
```

エンジンのスラッグはクエリパラメータになります。エンジンが指定されていない場合、Spwigはアルファベット順で最初のアクティブなエンジンを使用します。

## エンジン固有の同義語とリダイレクト

Synonym（同義語）とSearchRedirect（検索リダイレクト）モデルの両方には、オプションの`engine`の外部キーがあります。設定されている場合、その同義語またはリダイレクトは、特定のエンジンを通じた検索にのみ適用されます。

**例**: ブログエンジンには、"tutorial" → "guide"などの、製品検索には適用されない同義語があるかもしれません。

ほとんどの同義語とリダイレクトはエンジン固有であるべきではありません - engineフィールドを空にしてグローバルに適用するようにしてください。

## ヒント

すべてのマークダウンの書式、画像のパス、コードブロック、および技術用語を保持してください。

- **1つのエンジンから開始する** - デフォルトの "shop" エンジンを作成し、複数のエンジンが必要になる明確なニーズが出てくるまでそれにすべてを使用する
- **説明的なスラッグを使用する** - "shop"、"blog"、"wholesale" のような、エンジンの目的を明確に示すスラッグを選ぶ
- **アクティブ化する前にエンジンをテストする** - エンジンを非アクティブ状態で作成し、API経由でテストした後、アクティブ化する
- **必要がない限りエンジンを作成しない** - エンジンがすべて同じことをする場合、より多くのエンジンは構成の複雑さを増すだけの利点がない
- **エンジンごとに分析を確認する** - 検索分析ダッシュボードはエンジンでフィルタをかけ、どのエンジンが最も使用されているかを確認できる
