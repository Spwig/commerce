---
title: POS端末の管理
---

POS端末の管理は、小売業務の基礎です。各端末は、スタッフが販売を処理する物理的なデバイス（タブレット、コンピュータ、または専用のPOSハードウェア）を表します。倉庫の割り当て、スタッフの承認、ハードウェア統合、オフライン同期設定で端末を構成します。リアルタイムのheartbeat（心拍）追跡で端末の状態を監視し、問題が発生したときにリモートで端末を解除します。適切な端末管理は、店舗内でのスムーズな運用を確保し、複数の場所にまたがる構成の競合を防ぎます。

**POS > 端末**に移動して、新しい端末を登録し、オンライン/オフライン状態を表示し、すべての端末設定を管理します。

![端末リスト](/static/core/admin/img/help/managing-pos-terminals/terminal-list.webp)

## 端末リストビュー

端末リストは、登録されたすべての端末と主要な状態情報を表示します：

**端末名** - 端末の記述ラベル（例："チェックアウト1"、"メインレジ"、"モバイル端末"）

**UUID** - 作成時に自動生成される一意の識別子（デバイス識別に内部的に使用）

**倉庫** - この端末に割り当てられた物理的な場所（在庫の可用性と注文の属性を決定）

**オンライン状態** - 端末が現在接続されているかどうかを示すライブインジケーター：
- **緑色のドット** - オンライン（過去5分以内にheartbeatが受信）
- **赤色のドット** - オフライン（5分以上heartbeatが受信されていない）
- **灰色のドット** - これまでにペアリングされていない（端末は作成されたがデバイスがこれまでに接続されていない）

**最後のheartbeat** - 端末から受信した最新のパングのタイムスタンプ（オンライン時に5分ごとに更新）

**ペアリングコード** - 初回のデバイスペアリングに使用される8文字のアルファベット数字コード（初回使用後は非表示）

**割り当てられたユーザー** - この端末を使用するために承認されたスタッフの数

## 新しい端末の作成

**+ 端末の追加**をクリックして、新しいPOSデバイスを登録します：

![端末追加フォーム](/static/core/admin/img/help/managing-pos-terminals/terminal-add-form.webp)

### 基本設定

**端末名** - 説明的な名前を選択して、以下を示します：
- 物理的な場所："北口レジ"
- 関数："返品デスク端末"
- シーケンス："チェックアウト1"、"チェックアウト2"、"チェックアウト3"

名前は、シフト割り当てやトラブルシューティング中にスタッフが端末を識別するのに役立ちます。すべての場所で一貫した名前付けの規則を使用してください。

**倉庫** - **必須** - この端末が運用する倉庫を選択します：
- 売り可能な在庫を決定します
- この端末で行われた注文はこの倉庫に属性されます
- 在庫予約は割り当てられた倉庫での可用性を確認します
- **倉庫の割り当てがないと販売処理はできません**

複数の小売店舗がある場合、各店舗に個別の倉庫を作成し、端末を適切に割り当てます。

**有効** - 端末を有効/無効に切り替えます（構成を削除せずに）：
- 無効な端末はペアリングできません
- 無効な端末上の既存セッションは直ちに期限切れになります
- 賃貸や破損した端末を一時的に無効にします

### スタッフの割り当て

**割り当てられたユーザー** - この端末にアクセスできるスタッフを選択します：
- 割り当てられたユーザーのみが端末にログインできます
- ユーザーはスタッフロールでPOS権限を持っている必要があります
- 割り当てられたユーザーがゼロの場合、端末は実質的にロックされます
- 一般的なパターン：すべての店舗スタッフをすべての店舗端末に割り当てます

**使用例**：
- **一般店舗**：すべてのスタッフをすべての端末に割り当てます（どのレジ係もどのレジでも作業できます）
- **デパート**：部門専用のスタッフを部門端末に割り当てます
- **複数店舗**：店舗専用のスタッフを店舗端末に割り当てます
- **管理者**：すべての端末に管理を割り当てて、監督アクセスを提供します

端末に割り当てられていないユーザーは、ログインを試みると"この端末に認証されていません"というエラーを表示します。

### ハードウェア構成

**ハードウェア構成**フィールドは、周辺デバイスを定義するJSON構造です：

**熱伝票プリンタ**：
```json
{
  "printer": {
    "type": "network",
    "ip": "192.168.1.100",
    "port": 9100,
    "paper_width": 80
  }
}
```

**USBバーコードスキャナ**：
```json
{
  "scanner": {
    "type": "usb",
    "vendor_id": "0x05e0",
    "product_id": "0x1200"
  }
}
```

**キャッシュドロワー**（プリンタに接続）：
```json
{
  "cash_drawer": {
    "enabled": true,
    "trigger": "printer_pulse"
  }
}
```

**完全な例**：
```json
{
  "printer": {
    "type": "network",
    "ip": "192.168.1.100",
    "port": 9100,
    "paper_width": 80
  },
  "scanner": {
    "type": "usb",
    "vendor_id": "0x05e0",
    "product_id": "0x1200"
  },
  "cash_drawer": {
    "enabled": true,
    "trigger": "printer_pulse"
  }
}
```

周辺ハードウェアがない場合は空白のままにしてください（プリンタ/スキャナが搭載されていないモバイル端末やタブレットに適しています）。

### オフラインキャッシュ設定

端末がオフラインで動作するためにどのくらいのデータをキャッシュするかを構成します：

**注文同期日数**（7-30日、デフォルト：14）：
- 最近の注文をローカルにキャッシュする日数
- 高い値 = オフラインでより多くの履歴データが利用可能
- 低い値 = 同期が速く、ストレージ使用量が少ない
- **推奨**：高頻度の端末には7日、通常使用には14日、監査が多ければ30日

**注文同期上限**（200-1000注文、デフォルト：500）：
- 日付範囲に関係なくキャッシュする注文の最大数
- 高頻度の端末でストレージ使用量を過剰に防ぐ
- **推奨**：ストレージが限られているタブレットには200、標準端末には500、専用POSデバイスには1000

**トレードオフ**：
- **高い設定**：過去のデータへのオフラインアクセスが向上、初期同期が遅く、ストレージ使用量が増加
- **低い設定**：同期が速く、ストレージ使用量が少なく、オフライン履歴が限定的

端末は、各同期サイクルで最新のX注文（Y日以内）をダウンロードします。端末が1日50注文を処理し、sync_daysが14の場合、約700注文がキャッシュされます（sync_limitの上限に達する可能性があります）。

## 端末ペアリングワークフロー

端末を作成した後、物理的なデバイスをペアリングします：

1. **ペアリングコードの生成** - 端末を保存するときに自動的に作成されます（8文字のアルファベット数字）

2. **コードの記録** - 端末リストと詳細ビューに表示されます（最初の成功したペアリング後、期限切れ）

3. **端末デバイスに移動** - 物理的なデバイス（タブレット/コンピュータ）で、ブラウザを開き、`https://yourstore.com/pos/`にアクセスします。

4. **ペアリングコードの入力** - プロンプトが表示されたときに8文字のコードを入力します。

5. **端末が構成をダウンロード** - デバイスは以下を受信します：
   - 倉庫の割り当て
   - ハードウェア構成（プリンタ、スキャナ、ドロワー）
   - オフラインキャッシュ設定
   - 割り当てられたユーザー一覧
   - 初期商品カタログ同期

6. **ログインプロンプトの表示** - 端末は割り当てられたユーザーのためのログイン画面を表示します。

7. **スタッフがログイン** - この端末に割り当てられたユーザーの資格情報を入力します。

8. **初期同期の完了** - 端末は以下をダウンロードします：
   - 最近の注文（sync_daysとsync_limitに従って）
   - 割り当てられた倉庫のフル商品カタログ
   - 顧客データベース
   - プロモーション構成

9. **端末が準備完了** - 検索バー付きの"販売準備完了"画面が表示されます。

10. **ペアリングコードの使用済み** - コードは管理者から削除され、再ペアリングが必要な場合は新しいコードを生成してください。

**ペアリングコードの再生成**：端末を再ペアリングする必要がある場合（デバイスのリセット、ブラウザキャッシュのクリア、新しいハードウェア）は、**ペアリングコードの再生成**管理アクションを使用してください。これにより、古いコードが無効になり、新しいコードが作成されます。

## 端末ステータスの監視

### Heartbeatシステム

端末は、**5分ごとに**サーバーにheartbeatシグナルを送信します。このシグナルには以下が含まれます：
- 端末UUID
- 現在のタイムスタンプ
- オンラインユーザー数
- 最後の同期タイムスタンプ
- サービスワーカーの状態

**オンライン状態インジケーター**：
- **緑色** - 最後の5分以内にheartbeatが受信（端末はオンラインかつ運用可能）
- **赤色** - 5分以上heartbeatが受信されていない（端末がオフラインまたは切断）
- **灰色** - 端末がこれまでにペアリングされていない（これまでにheartbeatが受信されたことがない）

**使用例**：
- **毎日のオープン**：店舗のオープン前にすべての端末がオンラインであることを確認します
- **トラブルシューティング**：接続問題を経験している端末を特定します
- **監査**：営業時間中に端末がアクティブであることを確認します

### 最後のheartbeatタイムスタンプ

最も最近のheartbeatの正確な日時を表示します。これを使用して：
- 端末がオフラインになっている時間を把握する
- パターンを特定する（例：毎晩閉店時に端末がオフラインになる）
- 同期頻度を確認する（オンライン時に約5分ごとに更新されるべき）

## リモート解除機能

端末が応答しない、または画面に固まる（ソフトウェアのクラッシュ、セッションタイムアウトの問題、ブラウザのハング）場合、**リモート解除**管理アクションを使用してください：

**仕組み**：
1. 管理リストで問題のある端末を選択します
2. 管理アクションドロップダウンから**リモート解除**を選択します
3. アクションを確認します
4. サーバーはheartbeat応答を通じて解除シグナルを送信します
5. 端末は次のheartbeatサイクル（<5分）でシグナルを受信します
6. 端末は現在のユーザーを強制的にログアウトし、ログイン画面に戻ります

**使用するタイミング**：
- 端末が取引画面で凍結している
- スタッフがログアウトできない（ログアウトボタンが応答しない）
- セッションがアクティブだが端末が応答していない
- ブラウザがクラッシュしているがセッションクッキーが残っている

**重要**：リモート解除はデバイスやブラウザを再起動しません。ログアウトとセッションのクリアを強制するだけです。端末が完全に凍結している場合、スタッフはブラウザまたはデバイスを手動で再起動する必要があります。

## 端末構成の編集

リストにある端末をクリックして構成を編集します：

![端末編集フォーム](/static/core/admin/img/help/managing-pos-terminals/terminal-edit-form.webp)

**端末がオンラインである間でも安全に変更可能**：
- 端末名
- 割り当てられたユーザー
- ハードウェア構成（アプリを再起動した後に効果を発揮）
- オフラインキャッシュ設定（次の同期時に効果を発揮）

**再ペアリングが必要**：
- 倉庫の割り当て（倉庫を変更する必要がある場合は、新しい在庫を同期するために再ペアリングが必要）

**変更不可**：
- UUID（変更不可の一意の識別子）

ほとんどの設定の変更は次のheartbeat/同期サイクルで適用されます。ハードウェア構成の変更は、スタッフがPOSアプリを閉じて再開する（またはブラウザを再読み込みする）必要があります。

## 一般的な問題のトラブルシューティング

**端末がログイン時に"認証されていません"と表示される**：
- この端末の**割り当てられたユーザー**リストにユーザーが含まれていることを確認してください
- ユーザーが**スタッフと権限 > ロール**でPOS権限を持っていることを確認してください
- 端末が**有効**にマークされていることを確認してください

**端末がペアリングできない（無効なコード）**：
- ペアリングコードは最初の使用後に期限切れになります。必要に応じて再生成してください
- コードは大文字と小文字を区別します。大文字の使用を確認してください
- 端末が**有効**にマークされていることを確認してください

**端末がオフライン（赤いドット）**：
- デバイスがインターネット接続を持っていることを確認してください
- 端末が実際に実行されていることを確認してください（/pos/ URLにブラウザが開いている）
- ファイアウォールがheartbeatリクエストをブロックしていないことを確認してください
- 次のheartbeatサイクル（<5分）を待ってください

**端末の同期が遅い**：
- **注文同期日数**を30から7に減らしてください
- **注文同期上限**を1000から200に減らしてください
- 端末の場所でのネットワーク速度を確認してください
- サーバーが過負荷になっていないことを確認してください

**プリンタが動作しない**：
- **ハードウェア構成**でプリンタのIPアドレスとポートを確認してください
- 端末デバイスからプリンタの接続性をテストしてください（IPアドレスをpingしてみる）
- プリンタがESC/POS互換であることを確認してください
- プリンタが電源が入っており、オンラインであることを確認してください

## ヒント

- **名前付けの規則が重要** - 一貫した名前付け（場所 + 番号）を使用して、スケールでの管理を簡素化してください
- **ペアリングする前に常に倉庫を割り当ててください** - 倉庫の割り当てがないと、端末は販売処理ができません
- **デプロイする前にハードウェア構成をテストしてください** - テストレシートを印刷してプリンタ/ドロワーの統合を確認してください
- **毎日のheartbeatを監視してください** - 店舗オープン時にすべての端末がオンラインであることを確認するルーチンを設定してください
- **モバイル端末の同期上限を低くしてください** - タブレットやスマートフォンはsync_days: 7、sync_limit: 200が有益です
- **リモート解除は慎重に使用してください** - アクティブな取引を中断する可能性があります。端末が実際に固まっていることを確認してください
- **ペアリングコードを記録してください** - 端末を小売フロアに配置する前にコードを書き留めてください（セットアップが予想より長くかかる場合を考慮）
- **すべての端末に管理者を割り当ててください** - 管理者が返品、返金、トラブルシューティングのためにどのレジにもアクセスできるようにします
