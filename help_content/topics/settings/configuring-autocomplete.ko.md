---
title: 자동 완성 설정
---

자동 완성(예측 검색 또는 입력 시 검색)은 고객이 쿼리를 입력하는 동안 결과를 표시합니다. 이는 고객이 제품을 더 빠르게 찾고, 결과 없는 검색을 줄여 사용자 경험을 크게 개선합니다. 이 가이드는 자동 완성 동작, 표시 설정 및 성능의 트레이드오프를 구성하는 방법을 설명합니다.

자동 완성은 기본적으로 합리적인 설정으로 활성화되어 있습니다. 특정 성능 문제나 표시 선호도가 있는 경우에만 이러한 설정을 조정하세요.

![자동 완성 설정](/static/core/admin/img/help/configuring-autocomplete/autocomplete-settings-main.webp)

## 자동 완성 활성화

**검색 > 검색 설정**으로 이동하고 **자동 완성** 탭을 클릭합니다.

**자동 완성 활성화** - 예측 검색의 주 마스터 스위치. 활성화되면 고객이 입력할 때 검색 입력란에 결과 드롭다운이 표시됩니다.

**유형별 최대 결과 수** - 기본값: 8개 항목. 각 콘텐츠 유형(제품, 카테고리, 브랜드, 블로그 게시물)에 대해 표시할 결과 수. 낮은 값(5-6)은 API 데이터 전송량을 줄이고 빠르게 렌더링합니다. 높은 값(10-12)은 고객에게 더 많은 선택지를 제공하지만 응답 속도를 느리게 합니다.

## 지연 시간 설정

⚠️ **성능 경고** - 지연 시간 설정은 서버 부하에 큰 영향을 미칩니다.

**지연 시간** - 기본값: 300ms. 마지막 키스토크 이후 자동 완성 요청을 트리거하기 전에 기다리는 시간.

이 설정은 응답성과 서버 부하 사이의 균형을 맞춥니다:

| 지연 시간 | 사용자 경험 | 서버 영향 |
|-----------|------------|----------|
| **100ms** | 매우 빠름 | 300ms 대비 API 호출이 3배 많음 - 높은 부하 |
| **200ms** | 빠름 | 300ms 대비 API 호출이 1.5배 많음 |
| **300ms** | 좋은 균형(권장) | 기준선 |
| **400ms** | 약간 느림 | API 호출이 적음 - 낮은 부하 |
| **500ms** | 분명히 지연됨 | API 호출이 50% 적지만 느리게 느껴짐 |

**권장 사항**: 250-350ms 사이를 유지하세요. 자동 완성 부하로 인해 서버가 곤란할 경우에만 350ms 이상으로 증가시키세요. 매우 빠른 서버와 작은 카탈로그를 제외하고는 절대 200ms 이하로 설정하지 마세요.

## 제품 표시 설정

이 스위치는 제품 자동 완성 결과에 나타나는 정보를 제어합니다:

**썸네일 표시** - 기본값: ON. 결과 옆에 제품 이미지를 표시합니다. **성능 영향**: 이미지 쿼리와 JSON 데이터 전송량을 증가시킵니다. 느린 연결에서 더 빠른 자동 완성을 위해 비활성화하세요.

**설명 표시** - 기본값: OFF. 제품의 짧은 설명을 표시합니다. **성능 영향**: 텍스트 처리와 데이터 전송량을 크게 증가시킵니다. 제품 선택에 설명이 필수적인 경우를 제외하고는 비활성화 상태를 유지하세요.

**가격 표시** - 기본값: ON. 제품 가격을 표시합니다. **성능 영향**: 낮음 - 제품과 함께 이미 로드된 가격 데이터입니다. 활성화 상태를 유지하는 것이 안전합니다.

**SKU 표시** - 기본값: ON. 제품 SKU를 표시합니다. **성능 영향**: 낮음 - SKU는 이미 인덱싱되어 있습니다. B2B 매장에 필수적입니다.

**재고 상태 표시** - 기본값: OFF. **⚠️ 주요 성능 경고**

"재고 있음", "재고 부족", "재고 없음" 배지 표시. **대량 카탈로그에 절대 활성화하지 마세요**.

재고 상태는 `with_stock_totals()` 집계가 필요합니다 - 모든 창고에서의 재고 수량을 자동 완성 결과의 모든 제품에 대해 계산합니다. 이는 다음과 같은 영향을 미칩니다:
- 데이터베이스에 대한 큰 부하(집계 쿼리)
- 1,000개 이상의 제품이 있는 카탈로그의 경우 200-500ms 추가 지연
- 10,000개 이상의 제품이 있는 카탈로그의 경우 시간 초과 가능성

절대적으로 필수적이며 제품 수가 500개 미만인 경우에만 활성화하세요.

## 블로그 게시물 표시 설정

**대표 이미지 표시** - 기본값: ON. 블로그 게시물의 썸네일을 자동 완성 결과에 표시합니다.

**요약 표시** - 기본값: ON. 게시물 내용에서 간단한 미리보기 텍스트를 표시합니다.

**요약 길이** - 기본값: 60자. 표시할 미리보기 텍스트의 양.

제품에 비해 블로그 게시물이 일반적으로 적기 때문에 이러한 설정은 성능에 거의 영향을 미치지 않습니다.

## 카테고리 및 브랜드 표시 설정

**썸네일/로고 표시** - 기본값: ON. 카테고리 또는 브랜드 이미지를 결과에 표시합니다.

**제품 수 표시** - 기본값: OFF. **⚠️ 성능 경고**

각 카테고리 또는 브랜드에 있는 제품 수를 표시합니다(예: "전자기기 (234)").

**대량 카탈로그에 절대 활성화하지 마세요**. 제품 수는 자동 완성 요청마다 재계산됩니다:
- 각 콘텐츠 유형에 대해 제품 수 표시가 활성화되면 2개 추가 쿼리가 발생합니다
- 쿼리에는 조인 및 집계가 포함됩니다
- 100-300ms 추가 지연이 일반적입니다
- 카테고리/브랜드 수가 증가할수록 영향이 커집니다

제품 수가 50개 미만이고 총 제품 수가 1,000개 미만인 경우에만 활성화하세요.

## 캐싱

**자동 완성 캐시 TTL** - 기본값: 60초(캐싱 탭에서 설정).

자동 완성 결과는 성능을 향상시키기 위해 캐시됩니다. 60초 TTL은 다음과 같은 의미입니다:
- 첫 번째 고객이 "노트북"을 검색하면 데이터베이스 쿼리가 실행됩니다
- 다음 59초 동안 "노트북" 검색은 캐시된 결과를 반환합니다
- 60초 후 캐시가 만료되고 다음 검색은 데이터를 새로고침합니다

**TTL 권장 사항**:
- **45-60초**: 대부분의 매장에 적합한 균형(기본값)
- **90-120초**: 제품 재고가 드물게 변경되는 경우 성능이 더 좋습니다
- **30초**: 제품을 자주 추가하는 경우 더 최신의 결과를 제공합니다

캐시 TTL을 증가시키는 것은 자동 완성 성능을 향상시키는 가장 쉬운 방법입니다.

## 다국어 자동 완성

다중 언어가 구성된 경우 자동 완성은 JSONField 번역에 저장된 번역된 콘텐츠를 자동으로 검색합니다.

**작동 방식**:
- 고객이 스페인어로 "zapatos"를 검색합니다
- 시스템은 스페인어 제품 이름 번역을 검색합니다
- 결과는 JSONField 데이터에서 스페인어 제품 이름을 표시합니다
- 스페인어 번역이 없으면 기본 언어로 전환합니다

**성능**: 1-3개 언어의 경우 거의 추가 부하가 없습니다. 5개 이상의 언어인 경우 쿼리 복잡도가 약간 증가합니다.

## 자동 완성 테스트

설정을 구성한 후 자동 완성 경험을 테스트하세요:

1. **익명 창에서 매장의 홈페이지를 열고**
2. **검색 상자를 클릭하여 포커스를 맞춥니다**
3. **일반적인 제품 이름을 천천히 입력합니다**(예: "laptop")
4. **관찰**:
   - 입력을 멈춘 후 결과가 얼마나 빠르게 나타나는지(지연 시간이 작동하는지 확인)
   - 어떤 정보가 표시되는지(썸네일, 가격, SKU가 구성된 대로)
   - 결과가 관련성이 있는지(관련성 가중치를 확인해야 하는 경우)
5. **모바일 테스트** - 드롭다운이 터치에 친화적이고 가독성이 있는지 확인하세요

## 팁

- **속도를 위해 제품 설명 비활성화** - 설명은 자동 완성 컨텍스트에서 거의 가치가 없으면서 데이터 전송량을 크게 증가시킵니다
- **대량 카탈로그에 절대 재고 상태 활성화하지 마세요** - 재고 집계는 자동 완성 성능을 해칩니다
- **터치 대상과 함께 모바일에서 테스트** - 자동 완성 결과는 전화에서 쉽게 탭할 수 있어야 합니다
- **주간 응답 시간 모니터링** - 자동 완성 요청은 200ms 미만을 목표로 하세요
- **느릴 경우 캐시 TTL 증가** - 가장 쉬운 성능 최적화 방법
- **제품 수는 비용이 많이 들며, 필수적이지 않으면 비활성화하세요** - 각 카테고리/브랜드 수는 자동 완성 요청마다 2개 쿼리를 추가합니다

기억하세요: 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어는 보존 규칙에 따라 정확히 유지하세요.
