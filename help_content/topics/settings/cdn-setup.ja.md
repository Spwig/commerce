title: CDNの設定
---

コンテンツ配信ネットワーク（CDN）は、あなたのストアの画像、スタイルシート、スクリプトのコピーを世界中のサーバーに保存します。顧客がストアを訪問する際、これらのファイルはあなたのメインのホスティングサーバーではなく、顧客に最も近いサーバーから提供されます。これにより、特にストアがホスティングされている場所から遠く離れた顧客にとって、ページの読み込み時間が短縮されます。

Spwigは、Brotliおよびgzipによる事前圧縮、1年間有効な不変ヘッダー付きのフィンガープリント付きアセットキャッシュ、適切なコンテンツ交渉を含む静的アセットの配信をデフォルトで最適化しています。CDNの追加は任意ですが、国際的な顧客層を持つストアにとってはさらに速度の向上につながります。

## ストアはCDNが必要ですか？

すべてのストアがCDNから同等に利益を得るわけではありません。以下のガイドラインを参考に判断してください。

**CDNが推奨される場合**:
- 顧客が複数の国や大陸に分散している
- ストアが多くの商品画像やメディアが豊富なページを含んでいる
- 世界中で最も速いページ読み込み時間を望んでいる
- ホスティングサーバーから遠く離れた地域に販売している（例：ヨーロッパのサーバー、アジアの顧客）

**CDNが不要な場合**:
- 顧客が主にローカルまたはサーバーと同じ国にいる
- ストアが画像が少ない小規模なカタログを持っている
- ホスティングプロバイダーがすでに組み込みのCDNを提供している

迷った場合は、CDNはパフォーマンスに悪影響を及ぼしません。Cloudflareなどのサービスは無料プランを提供しており、試すコストはかかりません。

## SpwigとCDNの連携方法

SpwigはデフォルトでCDNに対応しています。Spwigの管理パネル内でのコードや設定の変更は必要ありません。Spwigがすでに以下を行っています。

- **フィンガープリント付きの静的ファイル** -- すべてのCSS、JavaScript、画像ファイルには、ファイル名にユニークなバージョンハッシュが含まれています。これにより、CDNはこれらのファイルを長期間キャッシュしても、古いコンテンツを提供することなく安全です。
- **長期間有効なキャッシュヘッダー** -- 静的アセットは1年間有効な不変キャッシュヘッダーで提供され、CDNやブラウザに積極的にキャッシュするよう指示されます。
- **事前圧縮されたファイル** -- SpwigはBrotliおよびgzipを使用してアセットを事前に圧縮しており、CDNは追加の処理なしに小さなファイルを配信できます。
- **適切なコンテンツ交渉** -- SpwigはCDNが適切なキャッシュに必要な正しいコンテンツタイプとエンコーディングヘッダーを送信します。

すべてあなたに必要なのは、ドメインのDNSをCDNプロバイダーにポイントすることだけです。その後、すべてが自動的に動作します。

## Cloudflareの設定

Cloudflareは最も人気のあるCDNであり、ほとんどのストアにとって十分に機能する無料プランを提供しています。以下の手順に従ってください。

**ステップ1: Cloudflareアカウントの作成**
- cloudflare.comにアクセスし、無料アカウントを登録します

**ステップ2: ドメインの追加**
- **Add a Site**をクリックし、ストアのドメイン名を入力します
- **Free**プランを選択します（ほとんどのストアにとって十分です）

**ステップ3: DNSネームサーバーの更新**
- Cloudflareは2つのネームサーバー（例: `anna.ns.cloudflare.com`）を表示します
- ドメインレジストラ（ドメインを購入した場所）にログインします
- 現在のネームサーバーをCloudflareのネームサーバーに置き換えます
- DNSの変更は最大24時間かかることがあります

**ステップ4: SSL/TLSの設定**
- Cloudflareダッシュボードで **SSL/TLS** に移動します
- 暗号化モードを **Full (strict)** に設定します
- これにより、Cloudflareとあなたのサーバー間のすべてのトラフィックが暗号化されたままになります

**ステップ5: それが動作しているかの確認**
- DNSがプロパゲートされたら、ストアにアクセスし、ブラウザで `cf-cache-status` ヘッダーを確認してください（下記のCDNの確認方法を参照）

## AWS CloudFrontの設定

すでにAmazon Web Servicesを使用している場合、CloudFrontはあなたのインフラと自然に統合されます。

1. AWSアカウントの **CloudFront** コンソールを開きます
2. ストアのドメインをオリジンとして使用して新しい **Distribution** を作成します
3. **Origin Protocol Policy** を「HTTPS Only」に設定します
4. **Cache Behavior** で、静的アセットに対して **Cache Policy** を「CachingOptimized」に設定します
5. ストアのドメインを **Alternate Domain Name (CNAME)** として追加します
6. AWS Certificate ManagerからSSL証明書を取得してアタッチします
7. ドメインのDNSをCloudFrontディストリビューションURLにポイントするように更新します

CloudFrontの料金は使用量に基づいています。

ほとんどのストアでは、Spwigのファイングリップされたアセットが長期間キャッシュされるため、コストは最小限です。

## おすすめのCDN設定

最高の結果を得るために、CDNを正しいコンテンツをキャッシュし、他のコンテンツはスキップするように設定してください。

**キャッシュするべきもの** (静的アセット):
- `/static/` -- すべてのスタイルシート、スクリプト、フォント、テーマアセット
- `/media/` -- 商品画像およびアップロードされたメディアファイル
- 画像ファイル (`.jpg`, `.png`, `.webp`, `.svg`, `.gif`)
- フォントファイル (`.woff`, `.woff2`)

**キャッシュしないでください** (動的ページ):
- `/admin/` -- 管理パネルは常に新鮮なコンテンツを提供する必要があります
- `/cart/` -- ショッピングカートページにはセッション固有のデータが含まれています
- `/checkout/` -- 支払いページはセキュリティ上の理由で決してキャッシュしてはいけません
- `/accounts/` -- 顧客アカウントページには個人情報が含まれています
- ログインが必要なページまたはパーソナライズされたコンテンツを表示するページ

**一般的なキャッシュルール**:
- **オリジンのキャッシュヘッダーを尊重する** -- Spwigは各コンテンツタイプに対して正しいキャッシュ制御ヘッダーを送信します。CDNをこれらのヘッダーを上書きするのではなく、尊重するように設定してください。
- **Brotli圧縮を有効にする** -- CloudflareとCloudFrontの両方がBrotliをサポートしています。Spwigの事前に圧縮されたアセットを活用するために有効にしてください。
- **ブラウザキャッシュTTLを"既存のヘッダーを尊重"に設定する** -- これにより、Spwigの組み込みキャッシュポリシーが動作します。

## CDNの確認

セットアップ後、CDNがコンテンツを正しく提供していることを確認してください:

**ステップ1: ブラウザの開発者ツールを開く**
- ChromeまたはFirefoxで **F12** を押して開発者ツールを開きます
- **Network** タブをクリックします

**ステップ2: ストアをロードする**
- 開発者ツールを開いたままストアのホームページにアクセスします
- 任意の静的ファイルリクエスト (例: `.css` または `.js` ファイル) をクリックします

**ステップ3: レスポンスヘッダーを確認する**
- **Cloudflare**: `cf-cache-status` ヘッダーを確認してください。値が `HIT` の場合、ファイルはCDNキャッシュから提供されています。`MISS` の場合、サーバーから取得されたことを示します (最初のリクエストのみ)。
- **CloudFront**: `x-cache` ヘッダーを確認してください。値が `Hit from cloudfront` の場合、CDN経由での配信が確認されます。

**ステップ4: 他の場所からテストする**
- gtmetrix.com または webpagetest.org のような無料ツールを使用して、ストアを異なる地理的場所からテストしてください
- CDNのセットアップ前後でロード時間を比較してください

## 一般的な問題

### テーマ変更後の古いコンテンツ

**問題**: テーマを更新したりデザイン変更を行った後、顧客が古いバージョンをまだ表示しています。

**解決策**: CDNキャッシュをクリアしてください。Cloudflareでは **Caching > Configuration > Purge Everything** にアクセスしてください。CloudFrontでは `/*` に対する **Invalidation** を作成してください。Spwigのファイングリップされたアセットは通常、この問題を防ぐため、更新されたファイルは自動的に新しいファイル名が割り当てられます。この問題は主にファイングリップされていないアセット (例: カスタムアップロード) に影響を与えます。

---

### 混合コンテンツ警告

**問題**: CDNを有効にした後、ブラウザが"混合コンテンツ"に関するセキュリティ警告を表示します。

**解決策**: CDNのSSLモードが **Full (strict)** に設定されていることを確認してください。"Flexible"モードは、サーバーがHTTPリクエストを代わりに受け取る可能性があり、混合コンテンツ警告を引き起こします。Cloudflareでは **SSL/TLS > Overview** を確認し、モードを確認してください。

---

### 管理パネルが遅く感じる

**問題**: CDNを追加した後、管理パネルが遅く感じます。

**解決策**: CDNは管理ページをキャッシュしてはいけません。Cloudflareでは **Page Rule**、CloudFrontでは **Cache Behavior** を作成し、`/admin/*` に一致するURLに対してキャッシュを "Bypass" に設定してください。これにより、管理リクエストはCDNのオーバーヘッドなしに直接サーバーに送信されます。

---

### 画像が表示されない

**問題**: CDNをセットアップした後、商品画像やメディアファイルがエラーを返します。

**解決策**: CDNのオリジンが正しいプロトコル (HTTPS) とポートで設定されていることを確認してください。また、サーバーのファイアウォールがCDNのIP範囲からの接続を許可していることも確認してください。

## ヒント

すべてのマークダウンフォーマット、画像パス、コードブロック、および技術用語を保持してください。

- **Cloudflareの無料プランから開始する** -- これはほとんどのストアのニーズをカバーし、数分でセットアップできます
- **常にFull (strict) SSLモードを使用する** -- Flexibleモードはセキュリティ上の脆弱性を引き起こし、チェックアウトフローを破損する可能性があります
- **テーマの主要な更新後、CDNキャッシュをクリアする** -- Spwigのファイングリービングファイルはほとんどのケースを処理しますが、完全なキャッシュのクリアにより、古いコンテンツが残らないことを確実にします
- **チェックアウトやカートページをキャッシュしない** -- これらのページをキャッシュすると、1人の顧客のデータが別の顧客に公開される可能性があります
- **顧客の所在地からテストする** -- webpagetest.orgなどの無料ツールを使って、顧客がショッピングする地域からのリアルワールドパフォーマンスを測定してください
- **CDNの分析を監視する** -- CloudflareとCloudFrontの両方とも、キャッシュヒット率、節約された帯域幅、国別トラフィックを表示するダッシュボードを提供しています
- **CDNへの切り替え中にDNS TTLを低く保つ** -- DNS TTLを300秒（5分）に設定し、すべてが正常に動作することを確認した後で増やすようにしてください
- **CDNは優れたホスティングを置き換えるものではない** -- チェックアウト、カート、管理などの動的なページでは、オリジンサーバーが依然として重要です
