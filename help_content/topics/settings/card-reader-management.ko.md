---
title: 카드 리더 관리
---

카드 리더 관리는 물리적 결제 하드웨어 장치를 추적하고, 이를 POS 단말기와 연결하며, 운영 상태를 모니터링합니다. 각 카드 리더는 실제 하드웨어(Stripe S700, WisePOS E 또는 P400)로, 결제 제공업체에 등록된 장치를 나타냅니다. 리더는 단말기와 일대일 관계를 가지며, 각 단말기는 자체 전용 카드 리더를 가지고 있습니다. 실시간으로 리더 상태(온라인, 오프라인, 바쁜 상태)를 모니터링하고, 브랜딩을 사용하여 스플래시 화면을 커스터마이징하며, 고객 체크아웃 경험에 영향을 미치기 전에 연결 문제를 해결합니다.

카드 리더 관리를 사용하여 모든 위치에서 결제 하드웨어가 올바르게 구성, 할당 및 운영되고 있는지 확인합니다.

![카드 리더 목록](/static/core/admin/img/help/card-reader-management/reader-list.webp)

## 카드 리더 이해

카드 리더는 신용 및 대출 카드 결제를 처리하는 물리적 하드웨어 장치입니다:

**하드웨어 구성 요소**:
- EMV 칩 카드 슬롯
- NFC 안테나(비접촉식/탭 페이)
- 자석 대형 읽기기(구형, 거의 사용되지 않음)
- 디스플레이 화면(금액 표시, PIN 입력, 서명 요청)
- 네트워크 연결(Wi-Fi 또는 이더넷, 모델에 따라 다름)

**소프트웨어 통합**:
- 리더는 Stripe Terminal API에 연결됩니다(클라우드 기반, POS 장치와 직접 연결되지 않음)
- POS 단말기는 API를 통해 결제를 요청합니다
- Stripe는 요청을 등록된 리더에게 라우팅합니다
- 리더는 카드를 처리하고 결과를 POS로 반환합니다
- POS와 리더 간의 USB/블루투스 연결은 필요하지 않습니다

**단말기당 하나의 리더**:
- 각 POS 단말기는 정확히 하나의 할당된 카드 리더를 가져야 합니다
- 일대일 관계는 명확한 책임감과 간소화된 문제 해결을 보장합니다
- 여러 단말기는 하나의 리더를 공유할 수 없습니다(충돌을 일으킴)

## 카드 리더 유형

Spwig POS는 Stripe Terminal 카드 리더를 지원합니다:

**BBPOS WisePOS E** (`bbpos_wisepos_e`):
- 5" 컬러 터치 스크린이 있는 모든 인 한 Android 단말기
- 내장 프린터 옵션(열 프린터)
- 최적 사용: 전면 기능이 있는 소매 체크아웃, 레스토랑(컬러 스크린에서 팁 요청)
- 연결: Wi-Fi만
- 스플래시 화면: 전체 컬러 480×800 세로

**Stripe Reader S700** (`stripe_s700`):
- 단일 화면 디스플레이가 있는 카운터탑 리더
- 컴팩트한 디자인, 스플래시 방수 처리
- 최적 사용: 표준 소매, 컴팩트한 체크아웃 카운터
- 연결: Wi-Fi 또는 이더넷
- 스플래시 화면: 모노크롬 480×800 세로

**Verifone P400** (`verifone_p400`):
- 구형 카운터탑 리더(구형 모델)
- 여전히 지원되지만 새로운 배포에는 권장하지 않습니다
- 최적 사용: 기존 배포(작동 중인 하드웨어를 교체하지 않음)
- 연결: Wi-Fi 또는 이더넷
- 스플래시 화면: 모노크롬 480×800 세로

**미래 호환성**:
- Stripe Terminal이 하드웨어 제공을 확장할 경우 추가 리더 모델이 추가될 수 있습니다
- 리더 유형 드롭다운은 제공업체 기능에서 자동으로 채워집니다

## 리더 등록 워크플로우

**단계 1: 하드웨어 구매 및 수령**
- Stripe(stripe.com/terminal) 또는 인증된 리셀러로부터 리더를 주문합니다
- 리더를 언박싱하고 전원을 켜세요
- Wi-Fi 네트워크에 연결합니다(리더의 화면 설정을 따르세요)

**단계 2: Stripe 대시보드에서 등록**
- **Stripe 대시보드 > Terminal > Readers**로 이동합니다
- **새로운 리더 등록**을 클릭합니다
- 화면에 표시된 페어링 프로세스를 따릅니다(리더는 등록 코드를 표시합니다)
- 리더를 Stripe 위치에 할당합니다(결제 제공업체 구성의 위치와 일치해야 합니다)
- **리더 ID**를 확인합니다(예: `tmr_ABC123...`)

**단계 3: Spwig으로 동기화(자동)**
- Spwig은 Stripe 위치에 등록된 리더를 자동으로 발견합니다
- 백그라운드 작업은 30분마다 동기화합니다
- 새 리더는 **POS > Card Readers** 목록에 30분 이내에 나타납니다

**단계 4: 단말기 할당(수동)**
- **POS > Card Readers**로 이동합니다
- 목록에서 새로 발견된 리더를 찾습니다
- 편집을 클릭합니다
- 할당할 단말기를 선택합니다
- 저장합니다

**단계 5: 결제 테스트**
- POS 단말기에서 테스트 거래를 처리합니다
- 카드 결제 방법을 선택합니다
- POS는 할당된 리더를 발견해야 합니다
- Stripe 테스트 카드(4242 4242 4242 4242)를 사용하여 테스트를 완료합니다
- 결제가 성공적으로 완료되었는지 확인합니다

테스트 중 리더가 나타나지 않으면 단말기 할당 및 리더 상태를 확인합니다.

## 리더 상태 모니터링

리더는 Stripe Terminal API에 상태를 보고하며, Spwig은 5분마다 동기화합니다:

**온라인**(녹색) - 리더는 전원이 켜지고 네트워크에 연결되어 있으며 결제를 받을 준비가 되어 있습니다

**오프라인**(빨강) - 리더는 전원이 꺼져 있거나 네트워크에서 분리되었거나 접근할 수 없습니다

**바쁘다**(노랑) - 리더는 현재 결제 거래를 처리하고 있습니다

**최근 확인** - 리더가 Stripe API에 마지막으로 확인한 시간
- 리더가 온라인 상태일 때 약 2분마다 업데이트됩니다
- 연결 문제 진단에 유용합니다("리더는 3시간 전에 오프라인 상태가 되었습니다" = 영업 시간 중 전원 또는 네트워크 문제)

**상태 사용 사례**:
- **사전 개방 확인**: 문을 열기 전에 모든 매장 리더가 온라인 상태인지 확인합니다
- **문제 해결**: "단말기 3은 카드를 받지 않습니다" → 리더 상태 확인 → 오프라인 상태를 보여줌 → 전원/네트워크 확인
- **감사**: "어제 단말기 5에서 결제가 처리되었나요?" → 마지막 확인 시간 확인

## 단말기 할당

카드 리더는 단말기와 **일대일 관계**를 사용합니다:

**할당이 중요한 이유**:
- 결제 중 POS는 어떤 리더와 통신해야 하는지 알아야 합니다
- 하나의 리더를 여러 단말기에서 공유하면 충돌이 발생합니다(두 명의 점원이 동시에 동일한 리더를 사용할 수 없음)
- 할당되지 않은 리더는 사용되지 않습니다(고립된 하드웨어)

**할당 규칙**:
- 각 단말기는 **정확히 하나**의 카드 리더를 할당할 수 있습니다
- 각 카드 리더는 **정확히 하나**의 단말기에 할당될 수 있습니다
- 리더를 단말기 A에 할당하면 이전 단말기에서 자동으로 할당이 해제됩니다

**할당 변경**:
- 리더 기록을 편집합니다
- **단말기** 필드를 새로운 단말기로 변경합니다
- 저장합니다
- 이전 단말기는 카드 리더 할당을 잃게 됩니다(결제 중 "No reader assigned" 오류가 나타납니다)

**할당되지 않은 리더**:
- 새로 발견된 리더는 할당되지 않은 상태로 시작합니다
- 할당되지 않은 리더는 목록에 나타나지만 사용할 수 없습니다
- 단말기에 할당하여 활성화합니다

## 스플래시 화면 커스터마이징

리더 스플래시 화면은 고객 대면 화면에 브랜딩을 표시합니다:

**스플래시 화면이 무엇인가요?**
- 결제 처리 중이지 않은 경우 리더 디스플레이에 표시되는 이미지
- 기본 Stripe 로고를 사용자의 브랜딩으로 대체합니다
- 체크아웃을 기다리는 동안 고객에게 표시됩니다

**자동 생성 vs 커스터마이징**:

**자동 생성**(기본값):
- Spwig은 스토어 로고에서 스플래시 화면을 생성합니다(스토어 설정에서 로고가 구성된 경우)
- 리더 사양에 따라 자동으로 크기 조정됩니다(480×800 세로)
- S700/P400은 모노크롬, WisePOS E는 컬러
- 구성이 필요 없습니다

**커스텀 스플래시**(고급):
- 자체 설계된 스플래시 화면 이미지를 업로드합니다
- 디자인과 브랜딩에 대한 완전한 제어
- 이미지 요구 사항을 충족해야 합니다(아래 참조)

**커스텀 스플래시 요구 사항**:
- **해상도**: 정확히 480×800 픽셀(세로 방향)
- **포맷**: PNG 또는 JPG
- **S700/P400**: 모노크롬만(검은색과 흰색, 회색 없음)
- **WisePOS E**: 전체 컬러 지원
- **파일 크기**: <200KB

**커스텀 스플래시 설정**:
1. 카드 리더 기록을 편집합니다
2. **Splash Override Image** 필드에 이미지를 업로드하거나 미디어 라이브러리에서 선택합니다
3. 저장합니다
4. 스플래시는 리더에 5분 이내에 동기화됩니다

**커스텀 스플래시 제거**:
- **Splash Override Image** 필드를 지웁니다
- 저장합니다
- 리더는 자동 생성 스플래시로 돌아갑니다(스토어 로고가 없는 경우 Stripe 기본값으로)

**스플래시 테스트**:
- 업로드 후 5분 동안 동기화를 기다립니다
- 리더 장치를 방문합니다
- 스플래시가 대기 화면에 나타나는지 확인합니다
- 이미지 품질, 중심 정렬 및 대비를 확인합니다

## Stripe 스플래시 설정

내부적으로 Spwig은 Stripe Terminal 스플래시 화면 설정을 관리합니다:

**stripe_splash_file_id** - Stripe에서 업로드된 스플래시 이미지 파일의 내부 ID
- 스플래시가 업로드될 때 자동으로 설정됩니다
- Stripe API에서 스플래시를 참조하는 데 사용됩니다

**stripe_splash_config_id** - Stripe에서 스플래시 설정의 내부 ID
- 스플래시 파일을 리더에 연결합니다
- 리더에 스플래시를 할당할 때 자동으로 관리됩니다

이 필드는 읽기 전용이며 자동으로 관리됩니다. 직접적으로 상호작용할 필요가 없습니다.

## 일반적인 문제 해결

**문제 1: 리더가 전원이 켜져 있지만 오프라인 상태를 표시함**
- **원인**: 네트워크 연결 문제, Wi-Fi 비밀번호 변경, 리더가 범위 밖에 있음
- **해결 방법**: 리더의 네트워크 설정을 확인하고 Wi-Fi에 다시 연결하며, 네트워크에서 Stripe API에 도달할 수 있는지 확인합니다

**문제 2: 결제 중 POS가 "No reader assigned"라고 표시함**
- **원인**: 리더가 단말기에 할당되지 않았거나 할당이 완료되지 않았음
- **해결 방법**: 리더를 편집하고 단말기에 할당하고 저장한 후 결제를 다시 테스트합니다

**문제 3: 리더가 계속 바쁜 상태(결제 화면에 멈춤)**
- **원인**: 거래가 타임아웃 또는 충돌했고, 리더 상태가 재설정되지 않았음
- **해결 방법**: 리더를 재부팅(전원 주기)하고 지속되면 Stripe 지원에 문의합니다

**문제 4: 커스텀 스플래시가 나타나지 않음**
- **원인**: 이미지 해상도가 잘못됨, 동기화가 아직 안됨, 모노크롬 요구 사항 미충족(S700/P400)
- **해결 방법**: 이미지가 정확히 480×800인지 확인하고 5분 동안 동기화를 기다리며, 색상이 없는 리더에 대해 모노크롬인지 확인합니다

**문제 5: Stripe에 등록된 리더가 Spwig에 나타나지 않음**
- **원인**: 리더가 제공업체 구성과 다른 Stripe 위치에 등록됨
- **해결 방법**: Stripe 대시보드에서 리더의 위치가 제공업체의 위치 ID와 일치하는지 확인합니다

## 팁

- **단말기당 하나의 리더** - 단말기 간 리더 공유를 피하여 충돌을 방지하고 책임감을 간소화합니다
- **리더를 배치하기 전에 등록** - 리더를 체크아웃에 배치하기 전에 Stripe 등록 및 Spwig 할당을 완료합니다
- **매장 내 스플래시 화면 테스트** - 리더 모델과 조명에 따라 대비가 달라지므로 실제 환경에서 스플래시가 잘 보이는지 확인합니다
- **사전 개방 상태 모니터링** - 매일 아침 리더 목록을 확인하여 매장이 열리기 전에 모든 리더가 온라인 상태인지 확인합니다
- **하드웨어 물리적으로 라벨링** - 단말기 이름("단말기 1 리더")을 표시하여 문제 해결 시 쉽게 식별할 수 있도록 라벨 메이커를 사용합니다
- **리더에 무중단 전원 공급 유지** - 거래 중 전원 차단은 리더 상태를 손상시킬 수 있으므로 UPS를 권장합니다
- **리더 시리얼 번호 기록** - 보증 및 지원을 위해 시리얼 번호 기록을 유지합니다(리더 하드웨어 라벨에서 찾을 수 있음)
