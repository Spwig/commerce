title: Счета поставщиков доставки
---

Счета поставщиков доставки связывают ваш магазин с API перевозчиков (FedEx, UPS, DHL) для расчета тарифов в реальном времени и автоматической покупки этикеток. Каждый аккаунт хранит зашифрованные учетные данные API, отслеживает состояние соединения и связывается с методами доставки в реальном времени. Поставщики получают актуальные тарифы на этапе оформления заказа на основе размеров, веса, места отправки и места доставки — это исключает ручной ввод таблиц тарифов и гарантирует точные цены перевозчиков.

Используйте учетные записи поставщиков, когда вам нужны тарифы на доставку, рассчитанные поставщиком, или автоматическая генерация этикеток вместо ручного создания доставки.

## Поддерживаемые поставщики доставки

Spwig поддерживает основных перевозчиков через устанавливаемые компоненты поставщиков:

### FedEx

**Услуги**: Земля, Экспресс, Международные
**API**: FedEx Web Services
**Функции**: Тарифы в реальном времени, покупка этикеток, отслеживание, международные таможенные пошлины

### UPS

**Услуги**: Земля, Воздушные, Всемирные
**API**: UPS Developer API
**Функции**: Тарифы в реальном времени, генерация этикеток, отслеживание, проверка адресов

### DHL

**Услуги**: Экспресс, Электронная коммерция, Международные
**API**: DHL Express API
**Функции**: Международные тарифы, таможенные документы, отслеживание

### Дополнительные поставщики

Установите из рынка компонентов по мере необходимости (USPS, Canada Post, Australia Post и т.д.)

## Настройка учетной записи поставщика

Каждая учетная запись поставщика требует:

### Основная информация

- **Отображаемое имя**: Как отображается учетная запись в админ-панели (например, "FedEx Production Account")
- **Поставщик**: Выберите установленный компонент поставщика из выпадающего списка
- **Активный**: Переключатель для включения/отключения без удаления учетных данных
- **По умолчанию**: Установите как учетную запись по умолчанию для этого поставщика (только одна учетная запись по умолчанию на поставщика)

### Учетные данные API (зашифрованные)

**Варьируются в зависимости от поставщика**, обычно включают:

**FedEx**:
- Номер учетной записи
- Номер счетчика
- Ключ API
- Секрет API

**UPS**:
- Номер лицензии доступа
- Имя пользователя
- Пароль
- Номер учетной записи

**DHL**:
- Идентификатор сайта
- Пароль
- Номер учетной записи

**Все учетные данные зашифрованы в покое** и расшифровываются только при выполнении вызовов API.

### Адрес отправителя

- **По умолчанию отправлять из**: Адрес склада/отправки для расчета тарифов
- Некоторые поставщики требуют специальной настройки отправки в их панели управления

### Настройки

Поставщик-специфичные параметры (варьируются в зависимости от перевозчика):

- **Режим тестирования**: Используйте тестовые/песочные API-конечные точки поставщика
- **Согласованные тарифы**: Используйте согласованные тарифы поставщика (если доступно)
- **Включить страховку**: Автоматически включить страховку в тарифы
- **Доплата за жилые адреса**: Применить дополнительные расходы на доставку на жилые адреса
- **Требуется подпись**: Задайте стандартные требования к подписи

## Создание учетной записи поставщика

**6-шаговый процесс настройки**:

**Шаг 1: Получение доступа к API перевозчика**
1. Создайте учетную запись у перевозчика (FedEx.com, UPS.com, DHL.com)
2. Запросите доступ к API/разработке
3. Завершите регистрацию API перевозчика (может занять 1-3 рабочих дня)
4. Получите учетные данные API по электронной почте или через портал разработчика

**Шаг 2: Установка компонента поставщика** (если не предустановлен)
1. Перейдите в Настройки > Компоненты > Рынок
2. Найдите имя перевозчика (например, "FedEx")
3. Установите компонент поставщика доставки
4. Подождите, пока завершится установка

**Шаг 3: Создание учетной записи поставщика в Spwig**
1. Перейдите в Настройки > Доставка > Учетные записи поставщиков
2. Нажмите "Добавить учетную запись поставщика"
3. Выберите поставщика из выпадающего списка
4. Введите отображаемое имя

**Шаг 4: Введите учетные данные API**
1. Заполните поля учетных данных (варьируются в зависимости от поставщика)
2. Учетные данные автоматически зашифровываются при сохранении
3. Необязательно: Включите режим тестирования для первоначального тестирования

**Шаг 5: Тестирование соединения**
1. Нажмите кнопку "Тестирование соединения"
2. Система пытается выполнить вызов API к перевозчику
3. Проверьте, появляется ли статус "Подключено"
4. Проверьте временной штамп last_tested_at

**Шаг 6: Связь с методом доставки**
1. Создайте или измените метод доставки (Настройки > Корзина > Методы доставки)
2. Установите method_type = "Real-Time"
3. Выберите учетную запись поставщика из выпадающего списка
4. Сохраните метод

## Мониторинг состояния соединения

Учетные записи поставщиков отслеживают состояние соединения:

### Значения состояния

**Неизвестно** (серый): Никогда не тестировалось или еще не подключено

**Подключено** (зеленый): Последний вызов API успешен, учетные данные действительны

**Ошибка** (красный): Последний вызов API завершился неудачно, учетные данные могут быть недействительны

### Последний тест

- **Временная метка**: Когда соединение было последний раз проверено
- **Автоматическое обновление**: Каждый раз, когда используется поставщик (запрос тарифа, покупка этикетки)
- **Ручной тест**: Нажмите кнопку "Тестирование соединения" в любое время

### Устранение неполадок с соединением

**Частые причины**:
- Неправильные учетные данные API (опечатка, скопированы с лишним пробелом)
- Ключ API перевозчика истек или отозван
- Включен режим тестирования, но используются учетные данные для продаж (или наоборот)
- IP-адрес не добавлен в белый список перевозчиком
- Перерыв в работе API перевозчика

**Шаги решения**:
1. Проверьте, что учетные данные точно совпадают с панелью управления перевозчика
2. Проверьте, что настройка режима тестирования совпадает с типом учетных данных
3. Проверьте страницу статуса API перевозчика на наличие сбоев
4. Свяжитесь с поддержкой перевозчика для проверки учетной записи

## Рабочий процесс поиска тарифов

Как работают тарифы в реальном времени при оформлении заказа:

**1. Покупатель вводит адрес**
- Вводится адрес доставки
- Корзина рассчитывает общий вес + размеры

**2. Система готовит запрос на тариф**
- Получает учетные данные поставщика (расшифрованные)
- Рассчитывает размеры посылки из элементов корзины (использует посылки доставки, если определены)
- Подготавливает запрос API с адресом отправки, адресом доставки, посылками

**3. Вызов API поставщика**
- Запрос отправляется в API перевозчика с учетными данными аутентификации
- Перевозчик рассчитывает тариф на основе зоны, веса, размеров
- Ответ включает варианты услуг (Земля, Экспресс и т.д.)

**4. Отображение тарифов**
- Система анализирует ответ перевозчика
- Приводит к стандартному формату
- Применяется дополнительная наценка (если настроено)
- Тарифы отображаются покупателю при оформлении заказа

**5. Покупатель выбирает услугу**
- Покупатель выбирает предпочтительный вариант
- Выбранный тариф сохраняется в заказе

**Пример потока API**:
```
Запрос в API FedEx:
{
  "origin": {"postal_code": "90210", "country": "US"},
  "destination": {"postal_code": "10001", "country": "US"},
  "parcels": [{
    "weight": 2500,  // граммы
    "dimensions": {"length": 30, "width": 20, "height": 15}  // см
  }]
}

Ответ FedEx:
[
  {"service": "FEDEX_GROUND", "rate": 12.50, "delivery_days": 5},
  {"service": "FEDEX_EXPRESS", "rate": 28.75, "delivery_days": 2}
]
```

## Покупка этикетки (опционально)

Если поставщик поддерживает генерацию этикеток:

**Рабочий процесс**:
1. Покупатель завершает заказ
2. Продавец создает отправку (Заказы > Детали заказа > Создать отправку)
3. Выберите учетную запись поставщика и услугу
4. Система вызывает API поставщика для этикетки
5. Генерируется PDF-файл этикетки и прикрепляется к отправке
6. Номер отслеживания автоматически заполняется
7. Этикетка готова к печати

**Преимущества**:
- Нет необходимости вручную входить в сайт перевозчика
- Автоматическое синхронизирование отслеживания
- Автоматически генерируются таможенные формы (международные)
- Возможна генерация пакетных этикеток

## Добавление наценки к тарифам

Добавьте наценку продавца к тарифам перевозчика:

**Настройка** (в методе доставки, а не в учетной записи поставщика):
- **Тип наценки**: Процент или Фиксированный
- **Размер наценки**: Например, 15% или $2.50

**Пример**:
```
Тариф перевозчика: $12.50
Наценка: 15%
Покупатель платит: $14.38

ИЛИ

Тариф перевозчика: $12.50
Наценка: $2.50 (фиксированная)
Покупатель платит: $15.00
```

**Сценарии использования**:
- Охват расходов на упаковку/обслуживание
- Добавление маржи прибыли к доставке
- Компенсация комиссий за кредитные карты при доставке

## Несколько учетных записей поставщика

Вы можете создать несколько учетных записей для одного поставщика:

**Сценарии использования**:
1. **Тестирование против производства**
   - Тестовая учетная запись: Учетные данные песочницы перевозчика
   - Учетная запись для производства: Активные учетные данные

2. **Несколько складов**
   - Учетная запись склада A: Адрес отправки = Лос-Анджелес
   - Учетная запись склада B: Адрес отправки = Нью-Йорк

3. **Разные согласованные тарифы**
   - Учетная запись A: Стандартные тарифы
   - Учетная запись B: Тарифы со скидками по объему

**Каждая учетная запись может быть связана с разными методами доставки** для гибкой настройки.

## Советы

- **Сначала протестируйте в песочнике** - Используйте тестовые учетные данные перевозчика перед запуском
- **Мониторьте состояние соединения** - Регулярно проверяйте панель управления на наличие ошибок
- **Определите посылки доставки** - Точные размеры улучшают оценки тарифов
- **Используйте согласованные тарифы** - Включите, если у вас есть скидки по объему у перевозчика
- **Установите реалистичный адрес отправки** - Используйте фактический адрес отправки для точных зон
- **Сохраняйте учетные данные в безопасности** - Никогда не делитесь ключами API, регулярно обновляйте их
- **Имейте резервный метод** - Сохраните метод с фиксированной ставкой активным, если API перевозчика не работает
- **Мониторьте лимиты API перевозчика** - Некоторые перевозчики ограничивают количество вызовов API в день
- **Обновляйте учетные данные вовремя** - Когда перевозчик меняет ключи, обновите их немедленно
- **Используйте описательные имена** - "FedEx LA Warehouse" лучше, чем "FedEx 1"
