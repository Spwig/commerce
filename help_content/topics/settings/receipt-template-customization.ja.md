---
title: レシートテンプレートのカスタマイズ
---

レシートテンプレートは、POS端末で印刷される熱転写レシートの外観と内容を制御します。ヘッダーとフッターのテキストをカスタマイズし、ロゴを追加し、コンプライアンスフィールド（税識別番号、事業登記番号）を構成し、プロモーション用のQRコードを含めることができます。テンプレートはスコープターゲティングをサポートしており、すべての店舗に適用されるデフォルトテンプレート、地域ごとのグループ固有テンプレート、または個々の店舗に適用される店舗固有テンプレートを作成できます。システムはスコープ優先順位ルールを使用して、レシートを印刷するときにどのテンプレートが適用されるかを決定します。

レシートテンプレートを使用してブランドの一貫性を維持し、地域ごとのコンプライアンス要件を満たし、プロモーション要素を通じて顧客の関与を高めることができます。

## レシートテンプレートの基本

レシートテンプレートは、ESC/POS熱転写プリンタから印刷されるレシートの構造と内容を定義します。各テンプレートは次のものを指定します：

**物理的な構成**:
- 紙の幅（58mmまたは80mm）
- ロゴ画像（熱転写印刷用のモノクロ）
- フォントのサイズと間隔

**コンテンツセクション**:
- ヘッダーテキスト（店舗名、住所、連絡先情報）
- 動的な取引データ（商品、価格、合計、支払い方法）
- フッターテキスト（返品ポリシー、ありがとうメッセージ、ソーシャルメディア）
- コンプライアンスフィールド（税識別番号、事業登記番号）
- ラベル付きプロモーションQRコード

**スコープターゲティング**:
- デフォルトテンプレート（すべての店舗に適用される、上書きされない限り）
- グループテンプレート（グループ内のすべての店舗に適用される）
- 店舗テンプレート（特定の店舗/倉庫に適用される）

## スコープ優先順位ルール

端末がレシートを印刷するとき、システムはこの階層を使用してテンプレートを選択します（最も高い優先度から最も低い優先度）:

| 優先度 | スコープ | 例 | 使用例 |
|--------|--------|-----|--------|
| **1** | 店舗固有 | パリ店舗テンプレート | フランスの税コンプライアンス要件の独自性 |
| **2** | グループ固有 | ヨーロッパ店舗テンプレート | EUのすべてのロケーションでのVAT表示 |
| **3** | デフォルト | グローバルテンプレート | すべての設定されていない店舗の代替 |

**仕組み**:
1. 店舗に専用のテンプレート（倉庫固有）があるかを確認します
2. ない場合は、店舗のグループにグループテンプレートがあるかを確認します
3. ない場合は、デフォルトテンプレートを使用します

**例**:
- デフォルトテンプレート: "Standard Receipt"（スコープの割り当てなし）
- グループテンプレート: "EU Receipt"（ヨーロッパ店舗グループに割り当て済み）- VAT登録を含む
- 店舗テンプレート: "Paris Receipt"（パリ倉庫に割り当て済み）- フランスのSIRET番号を含む

**結果**:
- パリ店舗端末: "Paris Receipt"（最も具体的）を使用
- ベルリン店舗端末（ヨーロッパ店舗グループに所属、店舗テンプレートなし）: "EU Receipt"（グループレベル）を使用
- ニューヨーク店舗端末（グループなし、店舗テンプレートなし）: "Standard Receipt"（デフォルト代替）を使用

## 紙の幅の設定

熱転写レシートプリンタは58mmまたは80mmの紙幅を使用します。プリンタハードウェアに基づいて選択してください:

| 紙の幅 | 1行の文字数 | 最適な用途 | 一般的な用途 |
|--------|------------|-----------|-------------|
| **58mm** | ~32文字 | 小型、ポータブル | フードトラック、モバイルPOS、キオスク |
| **80mm** | ~48文字 | 標準小売 | 多くの小売店、レストラン |

**幅の混在不可**: 同じテンプレートを使用するすべての端末は、同じ紙幅のプリンタを使用する必要があります。混合プリンタタイプがある場合は、各幅用に別々のテンプレートを作成してください。

**ロゴのサイズ制限**:
- **58mm**: 最大幅384ピクセル（推奨: 350px）
- **80mm**: 最大幅576ピクセル（推奨: 550px）

最大幅を超えるロゴは自動的に縮小されるため、品質が低下する可能性があります。

## ロゴの設定

レシートのロゴは、熱転写プリンタとの互換性のために**モノクロ**（黒と白のみ）でなければなりません:

**ロゴの要件**:
- ファイル形式: PNG、JPG、またはWebP
- カラーモード: モノクロ（白背景の黒いピクセル）
- 推奨サイズ:
  - 58mm紙: 350px幅 × 100-150px高さ
  - 80mm紙: 550px幅 × 150-200px高さ
- ファイルサイズ: <100KB（熱転写プリンタのメモリは限られている）

**モノクロロゴの作成**:
1. 通常のロゴ（カラーまたはグレースケール）から開始します
2. 画像エディタを使用して、純粋な黒と白に変換します（グレースケールなし）
3. コントラストを高めて、黒い要素が実体であることを確認します
4. 透過または白背景のPNGとしてエクスポートします

**ロゴの配置**:
- いつも水平方向に中央に配置されます
- レシートの上部に印刷されます（ヘッダーテキストの上）
- 自動的にスペースが追加されます（コンテンツと混雑を防ぐ）

**ロゴの選択**:
- テンプレートフォームで**メディアライブラリをブラウズ**をクリックします
- モノクロロゴアセットを選択します
- プレビューはレシート上でのロゴの表示方法を示します

**ロゴなし**: ロゴフィールドを空に残して、テキストのみのブランド（ヘッダーテキストに店舗名を含めることもできます）を好む場合があります。

## ヘッダーテキスト

ロゴの直後にヘッダーテキストが表示されます（ロゴがない場合は上部に表示されます）。典型的な内容:

**店舗名と住所**:
```
Your Store Name
123 Main Street
City, State 12345
Phone: (555) 123-4567
```

**営業時間**:
```
月曜日-金曜日: 9時-21時
土曜日-日曜日: 10時-18時
```

**キャッチコピーまたはスローガン**:
```
Quality Products, Exceptional Service
```

**フォーマット**:
- 情報を分離するために改行を使用します
- 自動的に中央揃えされます
- 紙幅の文字数制限（58mmは32文字、80mmは48文字）以内に保つようにしてください

**利用可能な変数**（オプション）:
- `{store_name}` - これは倉庫名に置き換えられます
- `{order_date}` - これは取引日時に置き換えられます
- `{order_number}` - これは注文IDに置き換えられます

多くの商人はヘッダーテキストの一貫性のために静的テキストを使用します。

## フッターテキスト

取引の詳細（商品、合計、支払い）の後にフッターテキストが表示されます。典型的な内容:

**返品ポリシー**:
```
30日以内にレシート付きで返品
返品はストアクレジットまたは交換のみ
```

**ありがとうメッセージ**:
```
ご購入ありがとうございます！
@yourstoreをフォローしてください
```

**カスタマーサービス**:
```
ご質問は (555) 123-4567 に電話ください
または support@yourstore.com にメールしてください
```

**フォーマットのヒント**:
- 最も重要な情報を最初に配置してください（返品ポリシー、連絡先）
- 改行を使用して読みやすさを保つ
- セクションの間に区切り線（`---`）を追加することを検討してください

## コンプライアンスフィールド

多くの管轄区域では、レシートに特定の情報が必要です:

**税識別番号ラベル** - 税識別番号のカスタマイズ可能なラベル:
- 米国: "Tax ID" または "EIN"
- EU: "VAT Number" または "VAT Reg No"
- カナダ: "GST/HST Number"
- オーストラリア: "ABN"

**税識別番号値** - 実際の識別番号:
- テンプレート内で一度入力し、すべてのレシートに表示されます
- 例: "VAT Number: GB123456789"

**事業登記ラベル** - 事業登記のカスタマイズ可能なラベル:
- フランス: "SIRET"
- ドイツ: "Handelsregister"
- 英国: "Company Registration Number"

**事業登記値** - 実際の登記番号:
- 例: "SIRET: 123 456 789 00010"

**Spwigのロゴを表示** - "Powered by Spwig"のブランド表示を表示または非表示に切り替える:
- デフォルトで有効（プラットフォーム開発をサポート）
- ホワイトラベル運用の場合は無効にします

**地域ごとのコンプライアンス例**:

**ヨーロッパ連合**:
- 税識別番号ラベル: "VAT Number"
- 税識別番号値: "GB123456789"
- 国によって要求される場合は、会社登記番号を表示

**アメリカ合衆国**:
- 一般的にレシートの税識別番号は必要ありません（州によって異なる）
- B2B取引にはEINを含めることもあります

**フランス（特定）**:
- すべてのレシートにSIRETが必須
- 事業登記ラベル: "SIRET"
- 事業登記値: "123 456 789 00010"

**オーストラリア**:
- GST登録企業にはABN（オーストラリア企業番号）が推奨されます
- 税識別番号ラベル: "ABN"

本番運用前に、あなたの地域の管轄区域のレシート要件を確認してください。

## QRコードプロモーション

レシートの下部にQRコードを含めることで、顧客の関与を促進します:

**QRコードURL** - スキャンされた後の目的地:
- レビューの依頼: `https://yourstore.com/reviews/leave-review`
- ロイヤルティプログラム: `https://yourstore.com/loyalty/join`
- 次回購入割引: `https://yourstore.com/discount/THANKYOU`
- ソーシャルメディア: `https://instagram.com/yourstore`
- ウェブサイトのホーム: `https://yourstore.com`

**QRコードラベル** - QRコードの上に表示されるテキスト:
- "スキャンしてレビューを残し、次回購入で10％割引を獲得"
- "ロイヤルティプログラムに参加 - ここをスキャン"
- "Instagramでフォロー - ここをスキャンして接続"
- "ご経験を評価してください"

**QRコードのベストプラクティス**:
- 短いURLを使用してください（長いURLは、スキャンが困難な密集したQRコードを作成します）
- 配置前に複数のスマートフォンカメラでQRコードをテストしてください
- ラベルに明確な価値提案を含めてください（スキャンすることで顧客が何を得るか）
- QRコードのスキャンを追跡して効果を測定してください（トラッキングパラメータ付きのURLを使用）

**動的QRコード**（高度な機能）:
- QRリダイレクトサービス（bit.ly、tinyurl）を使用して短いURLを作成します
- 季節ごとにリダイレクト先を変更することなく、レシートを再印刷せずに
- 例: `https://bit.ly/yourstoreqr` → 現在のプロモーションにリダイレクト

## 異なるスコープ用のテンプレートの作成

**デフォルトテンプレート**（推奨された開始点）:
1. **POS > レシートテンプレート**に移動します
2. **+ レシートテンプレートの追加**をクリックします
3. **倉庫**と**ストアグループ**フィールドを空のままにします（これによりデフォルトになります）
4. あなたの最も一般的なプリンタタイプに合った紙幅を設定します
5. ロゴ、ヘッダー、フッターを追加します
6. 主要市場向けのコンプライアンスフィールドを構成します
7. 保存します

このテンプレートはすべての店舗に適用されます（上書きされない限り）。

**グループテンプレート**（地域の違いに対応）:
1. 新しいテンプレートを作成します
2. **ストアグループ**（例: "European Stores"）を選択します
3. **倉庫**を空のままにします
4. 地域のコンプライアンスフィールドを調整します（例: VATのフォーマット）
5. ヘッダーテキストを調整します（例: 地域の住所）
6. 保存します

このテンプレートはグループ内のすべての店舗に適用されます。

**店舗テンプレート**（場所固有のニーズ）:
1. 新しいテンプレートを作成します
2. **倉庫**（例: "Paris Store"）を選択します
3. この特定の場所のすべてのフィールドを調整します
4. 保存します

このテンプレートはこの1つの店舗にのみ適用されます。

**テンプレートのテスト**:
- 端末でテスト取引を処理します
- レシートを印刷します
- ロゴの明確さ、テキストの配置、コンプライアンスフィールド、QRコードのスキャン可能性を確認します
- 必要に応じてテンプレートを調整し、再テストします

## 一般的なレシートレイアウト

**最小限のレシート**（フードトラック、ポップアップ）:
- ロゴなし（スペースの節約）
- ヘッダー: 店舗名と電話番号のみ
- フッター: ありがとうメッセージ
- QRコードなし

**標準小売レシート**:
- ロゴ（モノクロブランドマーク）
- ヘッダー: 全店舗名、住所、営業時間
- コンプライアンス: 税識別番号
- フッター: 返品ポリシー、ありがとうメッセージ
- QRコード: レビュー依頼

**プレミアム小売レシート**:
- ロゴ（フルブランドワードマーク）
- ヘッダー: キャッチコピー、住所、連絡先
- コンプライアンス: 税識別番号、事業登記
- フッター: 返品ポリシー、カスタマーサービス、ソーシャルメディア
- QRコード: ロイヤルティプログラム登録

**マルチロケーションチェーン**:
- デフォルトテンプレート: コーポレートブランド、標準ポリシー
- グループテンプレート: 地域コンプライアンス（EUのVAT、カナダのGST）
- 店舗テンプレート: 地域固有の住所と電話番号

## 複数テンプレートの管理

**テンプレートの命名規則**:
- スコープを名前に含める: "Default Receipt", "EU Group Receipt", "Paris Store Receipt"
- リストを確認するときにどのテンプレートがどこに適用されるかを識別しやすくなります

**テンプレートの変更**:
- 変更は今後のレシートに即座に適用されます
- 過去のレシート（既に印刷されたもの）は影響を受けません
- すべての店舗に展開する前に、低交通量の端末で変更をテストしてください

**テンプレートの複製**:
- 既存のテンプレートと類似した新しいテンプレートを作成する場合、既存のテンプレートを複製して変更してください
- から始める必要はありません

**テンプレートの削除**:
- 端末が存在する間はデフォルトテンプレートを削除できません（代替が必要です）
- グループ/店舗テンプレートは削除できます（端末は階層の次のレベルにフォールバックします）
- 削除する前に、テンプレートがアクティブに使用されている端末がないことを確認してください

## ヒント

- **80mmから始めることをおすすめします** - 標準的な紙幅はほとんどの小売店に適しています。58mmは専門的な用途です
- **実際のプリンタでロゴをテストしてください** - スクリーン上では良い見た目でも、印刷時に品質が低下する可能性があります。早期にテストしてください
- **コンプライアンスフィールドを最新の状態に保つ** - レシート上の有効期限切れの税登録は法的問題を引き起こします
- **価値提案付きQRコードはスキャンされやすい** - "10％割引を獲得するためにスキャン"は"ここをスキャン"の10倍の効果があります
- **文字数制限を確認してください** - テキストの折り返しはフォーマットを崩します。展開する前に1行あたりの文字数を数えてください
- **1つのテンプレートを1つの紙幅に割り当ててください** - 58mmプリンタの端末に80mmテンプレートを割り当てると、ロゴが収まらない可能性があります
- **月に1回テストレシートを印刷してください** - プリンタは時間が経つにつれて劣化します。品質が依然として許容範囲内であることを確認してください
- **変数は控えめに使用してください** - 静的テキストは動的変数よりも信頼性が高いため、失敗ポイントが少ないです
- **テンプレート構成をバックアップしてください** - 大規模な変更を行う前に、スクリーンショットまたはテンプレート設定をエクスポートしてください（簡単にロールバックできます）
- **地域ごとのコンプライアンスは異なります** - 本番運用前に地域ごとのレシート要件を調査してください。コンプライアンスが不十分な場合、罰金が厳しくなる可能性があります
