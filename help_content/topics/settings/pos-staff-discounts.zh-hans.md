---
title: POS员工折扣与终端安全
---

POS员工折扣设置让您控制每位员工在收银时可以应用的折扣幅度。终端锁定事件可提供每次终端被锁定或解锁的审计记录，帮助您跟踪谁访问了终端，以及是否发生过登录失败的情况。

## 员工折扣限制

使用POS的每位员工都可以拥有独立的折扣权限。默认情况下，员工可以对商品或整个购物车应用最高10%的折扣。您可以根据个人情况提高或降低此限制，或指定员工为经理，使其能够批准超出标准限制的折扣。

### 配置员工的折扣限制

1. 导航到 **POS > 员工折扣**
2. 点击 **+ 添加POS员工折扣** 或点击现有员工以进行编辑
3. 从列表中选择 **员工**
4. 设置折扣限制：

| 字段 | 描述 |
|-------|-------------|
| **最大折扣百分比** | 该员工可以应用的最大折扣百分比（例如，`10` 表示 10%） |
| **最大折扣金额** | 每笔交易的最大固定金额（留空表示无固定上限） |
| **可应用商品折扣** | 允许对单个商品行应用折扣 |
| **可应用购物车折扣** | 允许对整个购物车总额应用折扣 |
| **需要原因** | 勾选后，员工在应用任何折扣前必须输入原因 |

5. 点击 **保存**

### POS上的折扣限制如何工作

当收银员尝试应用折扣时：
- 如果折扣在其限制范围内，将立即应用
- 如果折扣超出其限制，终端将提示 **经理批准**
- 经理输入其PIN以授权覆盖，折扣将被应用

此工作流程可防止未经授权的高额折扣，同时在需要真实折扣时提供灵活性。

## 经理角色

带有 **是经理** 标志的员工可以批准超出其他员工限制的折扣。

经理在终端上通过输入PIN码来确认审批请求。

### 设置经理

1. 打开员工的折扣记录
2. 勾选 **是经理**
3. 输入 **经理PIN**（4-6位数字）—— 保存时会安全地进行哈希处理
4. 点击 **保存**

经理PIN与用于终端锁定/解锁的收银员PIN是分开的。一个经理可以同时拥有经理PIN（用于折扣审批）和收银员PIN（用于终端访问）。

### 经理PIN安全

当你在管理表单中输入PIN并保存时，Spwig会自动对其进行哈希处理——原始PIN永远不会被存储。保存后，原始PIN字段会清空，这是预期的行为。

## 收银员PIN和卡片访问

每个员工还可以拥有一个 **收银员PIN**，用于锁定和解锁终端：

- **收银员PIN** —— 用于在终端自动锁定或手动锁定后解锁的4-6位PIN
- **卡片标识符** —— 注册的卡片（刷卡卡或NFC）也可以用于解锁终端

要设置收银员PIN，请在 **收银员PIN** 字段中输入并保存。与经理PIN一样，保存时会自动进行哈希处理。

## 终端锁定事件

每次终端被锁定或解锁时，Spwig都会记录一个终端锁定事件。这会创建一个完整的安全审计跟踪。

### 查看锁定事件

导航到 **POS > 终端锁定事件** 以查看完整历史记录。你可以按以下条件筛选事件：
- 终端
- 事件类型
- 日期范围

### 事件类型


| 事件 | 含义 |
|-------|---------|
| **手动锁定** | 员工故意锁定了终端 |
| **自动锁定（空闲超时）** | 由于不活动，终端被自动锁定 |
| **收银员解锁** | 收银员通过身份验证解锁了终端 |
| **经理解锁** | 经理使用其凭证解锁 |
| **刷卡解锁** | 终端使用已注册的刷卡进行解锁 |
| **生物识别解锁** | 终端使用指纹或人脸识别进行解锁 |
| **解锁失败** | 使用错误的凭证尝试解锁 |
| **锁定（3次以上失败）** | 在多次失败尝试后终端被锁定 |

### 锁事件记录包含的内容

每个事件记录：
- 涉及的 **终端**
- **事件类型**
- 执行操作的人 (**执行者**) 以及锁定发生时登录的用户 (**锁定者**)
- 是否使用了 **经理覆盖**
- **解锁方式**（PIN、卡片或生物识别）
- 此事件之前的 **失败尝试**（有助于发现暴力破解模式）
- 事件发生时的 **购物车总额** 和商品数量
- 请求的 IP 地址

### 调查安全问题

如果您怀疑终端存在未经授权的访问：

1. 导航到 **POS > 终端锁定事件**
2. 按照您要调查的终端进行筛选
3. 查找类型为 **解锁失败** 或 **锁定** 的事件 —— 这些表示重复的失败访问
4. 检查成功解锁的 **执行者** 字段，以查看谁获得了访问权限
5. 与班次记录 (**POS > 班次**) 进行交叉核对，以验证应值班的收银员

## 小贴士

- 根据员工资历设置折扣限制 —— 新员工可能从 5% 开始，有经验的员工为 10-15%，经理可以批准更高的折扣。
- 对于任何拥有更高折扣权限的员工，启用 **需要原因**。

# 保留原因记录

保留原因记录有助于您分析折扣模式并识别任何滥用情况。
- 如果您的商店有多个员工或员工流动率较高，请每周查看终端锁定事件 —— 在问题发生之前，更容易发现不规则的访问模式。
- 如果员工离职，请立即删除其收银员PIN和卡片标识符，以防止终端访问。
- 使用锁定事件来识别可能需要调整自动锁定超时的终端 —— 如果客户经常触发意外锁定，空闲超时可能设置得太短。
- 管理员PIN应定期更改。

在员工折扣记录中更新它们 —— 新的PIN在保存时会被哈希处理。
