---
title: POS 단말기 관리
---

POS 단말기 관리는 유통업체의 운영 기반입니다. 각 단말기는 직원이 판매를 처리하는 물리적 장치(태블릿, 컴퓨터 또는 전용 POS 하드웨어)를 나타냅니다. 단말기를 창고 할당, 직원 권한, 하드웨어 통합 및 오프라인 동기화 설정으로 구성합니다. 실시간 페어링 추적으로 단말기 상태를 모니터링하고 문제가 발생할 경우 원격으로 단말기를 해제합니다. 올바른 단말기 관리는 매장 내 운영을 원활하게 하고 위치별 설정 충돌을 방지합니다.

**POS > 단말기**로 이동하여 새 단말기를 등록하고 온라인/오프라인 상태를 확인하며 모든 단말기 설정을 관리합니다.

![단말기 목록](/static/core/admin/img/help/managing-pos-terminals/terminal-list.webp)

## 단말기 목록 보기

단말기 목록은 등록된 모든 단말기와 주요 상태 정보를 표시합니다:

**단말기 이름** - 단말기의 설명적인 라벨(예: "체크아웃 1", "메인 레지스터", "모바일 단말기")

**UUID** - 생성 시 자동으로 생성된 고유 식별자(내부적으로 장치 식별에 사용)

**창고** - 이 단말기에 할당된 물리적 위치(재고 가용성 및 주문 속성을 결정함)

**온라인 상태** - 단말기의 현재 연결 상태를 나타내는 실시간 지시자:
- **초록색 점** - 온라인(최근 5분 이내에 페어링 신호 수신)
- **빨간색 점** - 오프라인(5분 이상 페어링 신호 없음)
- **회색 점** - 결코 페어링되지 않음(단말기 생성되었지만 장치가 결코 연결되지 않음)

**마지막 페어링 신호** - 단말기에서 수신된 가장 최근의 펑(핑) 시간戳(온라인 상태일 때 5분마다 업데이트됨)

**페어링 코드** - 초기 장치 페어링에 사용되는 8자리 영문자 및 숫자 코드(최초 사용 후 숨김)

**할당된 사용자** - 이 단말기를 사용하도록 권한이 부여된 직원 수

## 새 단말기 생성

**+ 단말기 추가**를 클릭하여 새 POS 장치를 등록합니다:

![단말기 추가 폼](/static/core/admin/img/help/managing-pos-terminals/terminal-add-form.webp)

### 기본 설정

**단말기 이름** - 다음을 나타내는 설명적인 이름을 선택하세요:
- 물리적 위치: "북쪽 입구 레지스터"
- 기능: "환불 데스크 단말기"
- 순서: "체크아웃 1", "체크아웃 2", "체크아웃 3"

이름은 직원이 교대 배정 및 문제 해결 시 단말기를 식별하는 데 도움이 됩니다. 모든 위치에서 일관된 이름 규칙을 사용하세요.

**창고** - **필수** - 이 단말기가 운영되는 창고를 선택하세요:
- 판매 가능한 재고를 결정함
- 이 단말기에서 생성된 주문은 이 창고에 속함
- 할당된 창고에서 재고 예약을 확인함
- **창고 할당 없이는 판매를 처리할 수 없음**

여러 유통업체 위치가 있다면 각 위치에 대해 별도의 창고를 생성하고 단말기를 해당 창고에 할당하세요.

**활성화 여부** - 단말기를 삭제하지 않고 활성화/비활성화를 전환합니다:
- 비활성화된 단말기는 페어링할 수 없음
- 비활성화된 단말기의 기존 세션은 즉시 만료됨
- 도난되거나 손상된 단말기를 일시적으로 비활성화할 때 사용

### 직원 할당

**할당된 사용자** - 이 단말기를 사용할 수 있는 직원을 선택하세요:
- 할당된 사용자만 단말기에 로그인할 수 있음
- 사용자는 직원 역할에서 POS 권한을 가져야 함
- 할당된 사용자가 0인 경우 단말기를 효과적으로 잠금 처리함
- 일반적인 패턴: 모든 매장 직원을 모든 매장 단말기에 할당함

**사용 사례 예시**:
- **일반 매장**: 모든 직원을 모든 단말기에 할당함(어떤 점원도 어떤 레지스터를 사용할 수 있음)
- **부문 매장**: 부문별 직원을 부문 단말기에 할당함
- **다중 위치**: 위치별 직원을 위치 단말기에 할당함
- **관리자**: 모든 단말기에 관리자 권한을 부여하여 감독 접근을 제공함

단말기 할당이 없는 사용자는 로그인 시 "이 단말기에는 권한이 없습니다" 오류를 보게 됩니다.

### 하드웨어 설정

**하드웨어 구성** 필드는 주변 장치를 정의하는 JSON 구조입니다:

**열전도 프린터**:
```json
{
  "printer": {
    "type": "network",
    "ip": "192.168.1.100",
    "port": 9100,
    "paper_width": 80
  }
}
```

**USB 바코드 스캐너**:
```json
{
  "scanner": {
    "type": "usb",
    "vendor_id": "0x05e0",
    "product_id": "0x1200"
  }
}
```

**현금 캐비닛**(프린터에 연결):
```json
{
  "cash_drawer": {
    "enabled": true,
    "trigger": "printer_pulse"
  }
}
```

**완전한 예시**:
```json
{
  "printer": {
    "type": "network",
    "ip": "192.168.1.100",
    "port": 9100,
    "paper_width": 80
  },
  "scanner": {
    "type": "usb",
    "vendor_id": "0x05e0",
    "product_id": "0x1200"
  },
  "cash_drawer": {
    "enabled": true,
    "trigger": "printer_pulse"
  }
}
```

단말기에 주변 하드웨어가 없는 경우 빈칸으로 남겨두세요(프린터/스캐너가 없는 모바일 단말기 또는 태블릿에 적합함).

### 오프라인 캐시 설정

단말기가 오프라인 작업을 위해 얼마나 많은 데이터를 캐시하는지 설정합니다:

**주문 동기화 일수**(7-30일, 기본값: 14):
- 최근 주문을 로컬에 캐시할 일수
- 더 높은 값 = 오프라인 시 더 많은 역사적 데이터 사용 가능
- 더 낮은 값 = 더 빠른 동기화, 저장 공간 사용 감소
- **추천**: 고부하 단말기에는 7일, 일반 사용에는 14일, 감사 작업이 많은 경우 30일

**주문 동기화 제한**(200-1000 주문, 기본값: 500):
- 날짜 범위와 관계없이 캐시할 주문의 최대 수
- 고부하 단말기에서 과도한 저장 공간 사용을 방지함
- **추천**: 제한된 저장 공간의 태블릿에는 200, 표준 단말기에는 500, 전용 POS 장치에는 1000

**트레이드오프**:
- **높은 설정**: 역사적 데이터에 대한 더 나은 오프라인 접근, 초기 동기화 느림, 더 많은 저장 공간 사용
- **낮은 설정**: 더 빠른 동기화, 적은 저장 공간, 오프라인 역사 제한

단말기는 각 동기화 주기마다 가장 최근의 X 주문(최대 Y 일)을 다운로드합니다. 단말기가 하루에 50개의 주문을 처리하고 sync_days가 14일이라면 약 700개의 주문이 캐시되며 sync_limit 제한에 도달할 수 있습니다.

## 단말기 페어링 워크플로우

단말기를 생성한 후 물리적 장치를 페어링합니다:

1. **페어링 코드 생성** - 단말기를 저장할 때 자동 생성됨(8자리 영문자 및 숫자)

2. **코드 기록** - 단말기 목록 및 세부 정보 뷰에서 표시됨(첫 번째 성공적인 페어링 후 만료됨)

3. **단말기 장치 이동** - 물리적 장치(태블릿/컴퓨터)에서 브라우저를 열고 이동: `https://yourstore.com/pos/`

4. **페어링 코드 입력** - 요청 시 8자리 코드를 입력합니다

5. **단말기 구성 다운로드** - 장치는 다음을 받습니다:
   - 창고 할당
   - 하드웨어 구성(프린터, 스캐너, 캐비닛)
   - 오프라인 캐시 설정
   - 할당된 사용자 목록
   - 초기 제품 카탈로그 동기화

6. **로그인 화면 표시** - 단말기는 할당된 사용자에게 로그인 화면을 표시합니다

7. **직원 로그인** - 이 단말기에 할당된 사용자의 자격 증명을 입력합니다

8. **초기 동기화 완료** - 단말기는 다음을 다운로드합니다:
   - 최근 주문(동기화 일수 및 제한에 따라)
   - 할당된 창고의 전체 제품 카탈로그
   - 고객 데이터베이스
   - 프로모션 구성

9. **단말기 준비 완료** - 검색 바와 함께 "판매 준비 완료" 화면이 나타납니다

10. **페어링 코드 소비** - 코드는 관리자에서 제거되며, 재페어링이 필요한 경우 새 코드를 생성합니다

**페어링 코드 재생성**: 단말기를 다시 페어링해야 하는 경우(장치 재설정, 브라우저 캐시 삭제, 새 하드웨어) **페어링 코드 재생성** 관리 작업을 사용합니다. 이 작업은 이전 코드를 무효화하고 새 코드를 생성합니다.

## 단말기 상태 모니터링

### 페어링 시스템

단말기는 **5분마다** 서버에 페어링 신호를 보내며 다음을 포함하는 페어링 신호를 포함합니다:
- 단말기 UUID
- 현재 시간戳
- 온라인 사용자 수
- 마지막 동기화 시간戳
- 서비스 워커 상태

**온라인 상태 지시자**:
- **초록색** - 최근 5분 이내에 페어링 신호 수신(단말기 온라인 및 작동 중)
- **빨간색** - 5분 이상 페어링 신호 없음(단말기 오프라인 또는 연결 끊김)
- **회색** - 단말기 결코 페어링되지 않음(페어링 신호 없음)

**사용 사례**:
- **매일 오픈**: 매장 개장 전 모든 단말기의 온라인 상태 확인
- **문제 해결**: 연결 문제를 겪는 단말기를 식별
- **감사**: 영업 시간 동안 단말기들이 활성화되어 있는지 확인

### 마지막 페어링 신호 시간戳

가장 최근의 페어링 신호의 정확한 날짜 및 시간을 표시합니다. 이를 사용하여:
- 단말기가 얼마나 오랫동안 오프라인인지 확인
- 패턴 식별(예: 매일 밤 문을 닫을 때 단말기가 오프라인 상태가 됨)
- 동기화 빈도 확인(온라인 상태일 때 약 5분마다 업데이트되어야 함)

## 원격 해제 기능

단말기가 응답하지 않거나 화면에 멈춰 있는 경우(소프트웨어 충돌, 세션 타임아웃 문제, 브라우저 중단) **원격 해제** 관리 작업을 사용합니다:

**작동 방식**:
1. 관리 목록에서 문제 있는 단말기를 선택합니다
2. 관리 작업 드롭다운에서 **원격 해제**를 선택합니다
3. 작업을 확인합니다
4. 서버는 페어링 응답을 통해 해제 신호를 보냅니다
5. 단말기는 다음 페어링 신호 주기(<5분)에 신호를 수신합니다
6. 단말기는 현재 사용자를 강제로 로그아웃하고 로그인 화면으로 돌아갑니다

**사용 시기**:
- 단말기 거래 화면에서 멈춰 있음
- 직원이 로그아웃할 수 없음(로그아웃 버튼이 응답하지 않음)
- 세션이 활성화되어 있지만 단말기가 응답하지 않음
- 브라우저가 충돌했지만 세션 쿠키가 여전히 남아 있음

**중요**: 원격 해제는 장치 또는 브라우저를 재시작하지 않습니다. 로그아웃 및 세션 정리만 강제합니다. 단말기가 완전히 멈춘 경우 직원이 브라우저 또는 장치를 수동으로 재시작해야 할 수 있습니다.

## 단말기 구성 편집

목록에서 단말기를 클릭하여 구성 편집합니다:

![단말기 편집 폼](/static/core/admin/img/help/managing-pos-terminals/terminal-edit-form.webp)

**단말기가 온라인 상태일 때 안전하게 변경할 수 있는 항목**:
- 단말기 이름
- 할당된 사용자
- 하드웨어 구성(앱 재시작 후 적용됨)
- 오프라인 캐시 설정(다음 동기화 시 적용됨)

**재페어링이 필요한 변경**:
- 창고 할당(창고 변경은 새 재고 동기화를 위해 재페어링이 필요함)

**변경 불가 항목**:
- UUID(불변 식별자)

대부분의 설정 변경은 다음 페어링/동기화 주기에서 적용됩니다. 하드웨어 구성 변경은 직원이 POS 앱을 닫고 다시 열거나 브라우저를 새로 고침해야 합니다.

## 일반적인 문제 해결

**단말기 로그인 시 "권한 없음" 표시**:
- 이 단말기의 **할당된 사용자** 목록에 사용자가 있는지 확인하세요
- 사용자가 **직원 및 권한 > 역할**에서 POS 권한을 가지고 있는지 확인하세요
- 단말기가 **활성화됨** 상태인지 확인하세요

**단말기 페어링 불가(잘못된 코드)**:
- 페어링 코드는 최초 사용 후 만료됩니다. 필요 시 재생성하세요
- 코드는 대소문자를 구분합니다. 대문자 여부를 확인하세요
- 단말기가 **활성화됨** 상태인지 확인하세요

**단말기 오프라인(빨간 점)**:
- 단말기 장치가 인터넷 연결이 있는지 확인하세요
- 단말기가 실제로 실행되고 있는지 확인하세요(브라우저에서 /pos/ URL 열림)
- 방화벽이 페어링 요청을 차단하지 않는지 확인하세요
- 다음 페어링 신호 주기(5분 이내)를 기다리세요

**단말기 동기화 느림**:
- **주문 동기화 일수**를 30에서 7로 줄이세요
- **주문 동기화 제한**을 1000에서 200으로 줄이세요
- 단말기 위치의 네트워크 속도를 확인하세요
- 서버가 과부하 상태인지 확인하세요

**프린터 작동하지 않음**:
- **하드웨어 구성**에서 프린터 IP 및 포트를 확인하세요
- 단말기 장치에서 프린터 연결을 테스트하세요(IP 주소 펑(핑) 테스트)
- 프린터가 ESC/POS 호환인지 확인하세요
- 프린터가 켜져 있고 온라인 상태인지 확인하세요

## 팁

- **이름 규칙 중요** - 위치 + 번호와 같은 일관된 이름 규칙을 사용하여 대규모 관리를 간소화하세요
- **페어링 전에 항상 창고 할당** - 창고 할당 없이는 단말기가 판매를 처리할 수 없습니다
- **배포 전 하드웨어 구성 테스트** - 테스트 영수증을 인쇄하여 프린터/캐비닛 통합을 확인하세요
- **매일 페어링 신호 모니터링** - 매장 개장 시 모든 단말기의 온라인 상태를 확인하는 루틴을 설정하세요
- **모바일 단말기의 동기화 제한 낮춤** - 태블릿 및 스마트폰은 sync_days: 7, sync_limit: 200이 유리합니다
- **원격 해제은 신중하게 사용** - 활성 거래를 중단하는 강제 로그아웃은 단말기가 실제로 멈춘 경우에만 사용하세요
- **페어링 코드 기록** - 단말기를 매장에 배치하기 전에 코드를 기록하세요(설치에 더 오래 걸릴 수 있는 경우를 대비하세요)
- **모든 단말기에 관리자 할당** - 감독자가 void, 환불 및 문제 해결을 위해 모든 레지스터에 접근할 수 있도록 보장하세요
