title: บทบาทและสิทธิ์การใช้งานของพนักงาน
---

บทบาทของพนักงานช่วยให้คุณควบคุมได้อย่างแม่นยำว่าแต่ละสมาชิกในทีมสามารถมองเห็นและทำอะไรได้บ้าง — ทั้งในแดชบอร์ดผู้ดูแลระบบและที่เครื่อง POS กำหนดบทบาทที่มีสิทธิ์เฉพาะ แล้วจัดสรรให้กับพนักงาน ผู้ใช้สามารถมีบทบาทหลายบทบาท และสิทธิ์ที่มีผลต่อผู้ใช้นั้นคือการรวมของบทบาททั้งหมดที่ได้รับการกำหนด

![บทบาทของพนักงาน](/static/core/admin/img/help/staff-roles/role-list.webp)

## วิธีการทำงาน

1. คุณสร้าง **บทบาท** ที่กำหนดชุดของสิทธิ์ (เช่น "ผู้จัดการคำสั่งซื้อ", "แคชเชียร์")
2. แต่ละบทบาทควบคุมการเข้าถึงสองประเภท: **สิทธิ์ในแดชบอร์ดผู้ดูแลระบบ** และ **สิทธิ์ในเครื่อง POS**
3. คุณ **กำหนดบทบาท** ให้กับพนักงานจากหน้าโปรไฟล์ของพวกเขา
4. สิทธิ์ที่มีผลต่อพนักงานคือ **การรวม** ของบทบาททั้งหมดของพวกเขา — หากบทบาทใดให้สิทธิ์การเข้าถึง ผู้ใช้จะได้รับสิทธิ์นั้น
5. สิทธิ์จะถูก **แคช** เพื่อเพิ่มประสิทธิภาพ และจะถูกอัปเดตอัตโนมัติเมื่อบทบาทเปลี่ยนแปลง

## บทบาทที่กำหนดไว้ล่วงหน้า

Spwig มีบทบาทที่ติดตั้งไว้ 7 บทบาทที่ครอบคลุมโครงสร้างทีมที่พบได้บ่อยที่สุด บทบาทเหล่านี้ไม่สามารถลบได้ แต่คุณสามารถสร้างบทบาทที่กำหนดเองเพื่อความต้องการที่เฉพาะเจาะจงมากขึ้น

| บทบาท | การเข้าถึง | คำอธิบาย |
|------|--------|-------------|
| **เจ้าของร้านค้า** | แดชบอร์ดผู้ดูแลระบบ + เครื่อง POS | มีสิทธิ์เข้าถึงทุกสิ่ง สำหรับผู้ดูแลระบบหลักของร้านค้า |
| **ผู้จัดการร้านค้า** | แดชบอร์ดผู้ดูแลระบบ + เครื่อง POS | การดำเนินงานประจำวัน — มีสิทธิ์เข้าถึงสินค้า คำสั่งซื้อ ลูกค้า การตลาด และการค้นหา สามารถดูข้อมูลได้เท่านั้นสำหรับการออกแบบ อีเมล การชำระเงิน และการตั้งค่า |
| **ผู้แก้ไขเนื้อหา** | แดชบอร์ดผู้ดูแลระบบ | จัดการหน้าเพจ บล็อก ออกแบบ และสื่อ สามารถดูข้อมูลได้เท่านั้นสำหรับสินค้า |
| **ผู้จัดการคำสั่งซื้อ** | แดชบอร์ดผู้ดูแลระบบ | จัดการคำสั่งซื้อ การจัดส่ง การคืนสินค้า และบริการลูกค้า สามารถดูข้อมูลได้เท่านั้นสำหรับสินค้า |
| **ผู้จัดการการตลาด** | แดชบอร์ดผู้ดูแลระบบ | จัดการโปรโมชั่น คูปอง ระบบพันธมิตร โปรแกรมสะสมแต้ม และการแนะนำ สามารถดูข้อมูลได้เท่านั้นสำหรับสินค้า ลูกค้า และสื่อ |
| **แคชเชียร์** | เครื่อง POS เท่านั้น | พนักงาน POS ระดับหน้าแรก สามารถดำเนินการขายและตรวจสอบยอดเงินของบัตรของขวัญ ไม่มีสิทธิ์ในการให้ส่วนลด การคืนเงิน หรือการจัดการเงินสด |
| **แคชเชียร์ระดับสูง** | เครื่อง POS เท่านั้น | พนักงาน POS ที่มีประสบการณ์ สามารถดำเนินการคืนเงิน ใช้ส่วนลด (สูงสุด 25%) จัดการเงินสด และปิดกะ |

## การสร้างบทบาทที่กำหนดเอง

ไปที่ **การตั้งค่า > บทบาทของพนักงาน** และคลิก **เพิ่มบทบาท**

### การตั้งค่าทั่วไป

| การตั้งค่า | คำอธิบาย |
|---------|-------------|
| **ชื่อแสดงผล** | ชื่อบทบาทที่แสดงในแดชบอร์ดผู้ดูแลระบบ (เช่น "พนักงานคลังสินค้า") |
| **คำอธิบาย** | คำอธิบายสั้น ๆ ว่าบทบาทนี้ใช้เพื่ออะไร |
| **ลำดับการจัดเรียง** | ควบคุมลำดับการแสดงผลในรายการบทบาท |
| **ไอคอน** | เลือกจากไอคอน 20 ไอคอนเพื่อระบุบทบาทอย่างเห็นได้ชัดทางสายตา |
| **สีของป้ายกำกับ** | สีที่ใช้สำหรับป้ายกำกับบทบาท (สีฟ้า สีเขียว สีส้ม สีแดง สีเทา สีเทาเขียว) |
| **แดชบอร์ดผู้ดูแลระบบ** | เปิด/ปิดว่าบทบาทนี้ให้สิทธิ์เข้าถึงแดชบอร์ดผู้ดูแลระบบ |
| **เครื่อง POS** | เปิด/ปิดว่าบทบาทนี้ให้สิทธิ์เข้าถึงเครื่อง POS |

### หมวดหมู่สิทธิ์ในแดชบอร์ดผู้ดูแลระบบ

แท็บสิทธิ์ในแดชบอร์ดผู้ดูแลระบบจัดกลุ่มคุณสมบัติทั้งหมดของแพลตฟอร์มเป็น 13 หมวดหมู่ สำหรับแต่ละหมวดหมู่ คุณตั้งค่าระดับการเข้าถึงหนึ่งในสามระดับ:

- **ไม่มี** — ไม่มีสิทธิ์เข้าถึงพื้นที่นี้ (รายการเมนูจะถูกซ่อน)
- **ดู** — มีสิทธิ์อ่านข้อมูลเท่านั้น (สามารถดูข้อมูลได้แต่ไม่สามารถเปลี่ยนแปลงได้)
- **เต็ม** — มีสิทธิ์เข้าถึงทั้งหมด (สามารถดู สร้าง แก้ไข และลบข้อมูลได้)

![หมวดหมู่สิทธิ์](/static/core/admin/img/help/staff-roles/permission-categories.webp)

| หมวดหมู่ | สิ่งที่ควบคุม |
|----------|-----------------|
| **แคตตาล็อกสินค้า** | สินค้า หมวดหมู่ แบรนด์ คุณสมบัติ สต็อก คลังสินค้า ทรัพยากรดิจิทัล |
| **คำสั่งซื้อและการจัดส่ง** | คำสั่งซื้อ การคืนเงิน การคืนสินค้า การจัดส่ง การตั้งค่าการจัดส่ง |
| **ลูกค้า** | โปรไฟล์ลูกค้า กลุ่มลูกค้า การวิเคราะห์ |
| **เนื้อหาและหน้าเพจ** | หน้าเพจ บล็อก ประกาศ ฟอร์ม |
| **การออกแบบและธีม** | ธีม แม่แบบหัวข้อ/ฟุตเตอร์ เมนู โทเค็นการออกแบบ CSS ที่กำหนดเอง |
| **การตลาดและการส่งเสริมการขาย** | การส่งเสริมการขาย คูปอง พันธมิตร โปรแกรมสะสมแต้ม การแนะนำ ฟีดสินค้า |
| **ไลบรารีสื่อ** | รูปภาพ วิดีโอ โฟลเดอร์ แท็ก |
| **ระบบอีเมล** | บัญชีอีเมล แม่แบบ คิวส่งอีเมล |
| **การชำระเงินและการเรียกเก็บเงิน** | ผู้ให้บริการชำระเงิน การทำธุรกรรม webhook การสมัครสมาชิก อัตราแลกเปลี่ยน |
| **การค้นหา** | การตั้งค่าการค้นหา คำสังเคราะห์ การเปลี่ยนเส้นทาง การวิเคราะห์ |
| **การตั้งค่าร้านค้า** | การตั้งค่าเว็บไซต์ การกำหนดตำแหน่งทางภูมิศาสตร์ การจับคู่ประเทศ กฎทางธุรกิจ |
| **การจัดการเครื่อง POS** | เครื่อง POS กะงาน การเคลื่อนไหวของเงินสด แม่แบบใบเสร็จ |
| **ผู้ใช้และบทบาท** | บัญชีผู้ใช้พนักงาน บทบาท โทเค็น API |

เมื่อผู้ใช้มีบทบาทหลายบทบาท ระดับการเข้าถึงที่ **สูงสุด** จะเป็นผู้ชนะ ตัวอย่างเช่น หากบทบาท A ให้สิทธิ์ "ดู" สำหรับสินค้า และบทบาท B ให้สิทธิ์ "เต็ม" ผู้ใช้จะได้รับสิทธิ์ "เต็ม" ในการเข้าถึง

### แฟล็กสิทธิ์ในเครื่อง POS

หากบทบาทนั้นให้สิทธิ์เข้าถึงเครื่อง POS แท็บสิทธิ์ในเครื่อง POS จะช่วยให้คุณปรับแต่งได้อย่างแม่นยำว่าผู้ปฏิบัติงานเครื่อง POS สามารถทำอะไรได้ นี่คือสิทธิ์ที่แยกจากสิทธิ์ในแดชบอร์ดผู้ดูแลระบบ และจะถูกตรวจสอบที่เครื่อง POS

![สิทธิ์ในเครื่อง POS](/static/core/admin/img/help/staff-roles/pos-permissions.webp)

| กลุ่ม | สิทธิ์ | คำอธิบาย |
|-------|-----------|-------------|
| **ทั่วไป** | การเข้าถึงเครื่อง POS | สามารถใช้ระบบเครื่อง POS ได้ |
| **การขายและส่วนลด** | ส่วนลดแบบมือถือ | สามารถใช้ส่วนลดแบบรายการหรือระดับตะกร้าได้ |
| | ส่วนลดสูงสุด % | ค่าเปอร์เซ็นต์ส่วนลดสูงสุดที่อนุญาต (0–100) |
| | การกำหนดราคาใหม่ | สามารถกำหนดราคาสินค้าใหม่ได้ที่เคาน์เตอร์ |
| **การคืนเงินและการยกเลิก** | ดำเนินการคืนเงิน | สามารถดำเนินการคืนเงินสำหรับคำสั่งซื้อเครื่อง POS ได้ |
| | การยกเลิกคำสั่งซื้อ | สามารถยกเลิกคำสั่งซื้อเครื่อง POS จากกะปัจจุบันได้ |
| **บัตรของขวัญ** | ออกบัตรของขวัญ | สามารถออกบัตรของขวัญใหม่ได้ที่เคาน์เตอร์ |
| | ตรวจสอบยอดเงินบัตรของขวัญ | สามารถตรวจสอบยอดเงินบัตรของขวัญได้ |
| **การจัดการเงินสด** | การจัดการเงินสด | สามารถดำเนินการฝากเงินและถอนเงินได้ |
| | เปิดกล่องเงิน | สามารถเปิดกล่องเงินโดยไม่มีการขาย |
| | ปิดกะ | สามารถปิดกะและดำเนินการตรวจสอบเงินสดได้ |
| **รายงาน** | ดูรายงานเครื่อง POS | สามารถดูรายงานกะและสรุปยอดขายได้ |
| **สต็อก** | การปรับปรุงสต็อก | สามารถปรับปรุงระดับสต็อก (รับสินค้า ความเสียหาย นับใหม่ คืนสินค้า) |

สำหรับสิทธิ์ที่เป็นบูลีน หาก **บทบาทใด ๆ** ของผู้ใช้เปิดใช้งานมัน ผู้ใช้จะมีสิทธิ์นั้น สำหรับค่าส่วนลดสูงสุด % ค่าที่ **สูงสุด** ทั้งหมดของบทบาทจะถูกนำไปใช้

## การจัดการพนักงาน

ไปที่ **การตั้งค่า > การจัดการพนักงาน** เพื่อดูและจัดการทีมของคุณ

### รายการพนักงาน

รายการพนักงานแสดงผู้ใช้ทั้งหมดที่มีสิทธิ์เข้าถึงพนักงาน สำหรับแต่ละสมาชิก คุณสามารถดู:
- **ชื่อและอีเมล**
- **บทบาทที่กำหนด** (แสดงเป็นป้ายกำกับสี)
- **ประเภทการเข้าถึง** — เฉพาะแดชบอร์ดผู้ดูแลระบบ เฉพาะเครื่อง POS หรือทั้งสองอย่าง
- **สถานะ 2FA** — ว่าการยืนยันตัวตนแบบสองขั้นตอนได้เปิดใช้งานหรือไม่
- **สถานะการใช้งาน** — เปิดใช้งานหรือไม่เปิดใช้งาน

ใช้ตัวกรองเพื่อจำกัดตามบทบาท ประเภทการเข้าถึง หรือสถานะ 2FA

### การกำหนดบทบาทให้กับพนักงาน

1. คลิกที่พนักงานเพื่อเปิดโปรไฟล์ของพวกเขา
2. ในส่วน **บทบาท** คุณจะเห็นการ์ดสำหรับบทบาทที่มีอยู่ทุกบทบาท
3. คลิกปุ่มสลับบนการ์ดบทบาทใด ๆ เพื่อกำหนดหรือลบบทบาทนั้น
4. การเปลี่ยนแปลงจะมีผลทันที — ไม่จำเป็นต้องคลิกปุ่มบันทึก
5. สรุป **สิทธิ์ที่มีผล** ด้านล่างแสดงผลรวมของบทบาททั้งหมดที่ได้รับการกำหนด

### การเพิ่มพนักงานใหม่

1. ไปที่ **การตั้งค่า > การจัดการพนักงาน** และคลิก **เพิ่มพนักงานใหม่**
2. ใส่อีเมล ชื่อ และนามสกุลของผู้ใช้
3. ตั้งรหัสผ่านชั่วคราว
4. กำหนดบทบาทหนึ่งหรือหลายบทบาท
5. ผู้ใช้สามารถเข้าสู่ระบบด้วยสิทธิ์ที่บทบาทของพวกเขาให้ได้แล้ว

## การคัดลอกบทบาท

เพื่อสร้างบทบาทใหม่ที่มีฐานจากบทบาทที่มีอยู่:

1. เปิดบทบาทที่คุณต้องการคัดลอก
2. คลิก **คัดลอกบทบาท** ที่ด้านล่างของหน้า
3. บทบาทใหม่จะถูกสร้างขึ้นมาพร้อมกับสิทธิ์เดียวกันทั้งหมด
4. เปลี่ยนชื่อและปรับแต่งสิทธิ์ตามที่จำเป็น
5. บันทึกบทบาทใหม่

มีประโยชน์เมื่อคุณต้องการบทบาทที่คล้ายกับบทบาทที่มีอยู่แต่มีความแตกต่างเล็กน้อย — เช่น บทบาท "ผู้จัดการระดับต้น" ที่มีฐานจากบทบาท "ผู้จัดการร้านค้า" แต่มีสิทธิ์ที่น้อยกว่า

## วิธีที่สิทธิ์ถูกนำไปใช้

### แดชบอร์ดผู้ดูแลระบบ

- **การมองเห็นเมนู** — สำหรับหมวดหมู่ที่ผู้ใช้มีสิทธิ์ "ไม่มี" หัวข้อด้านข้างจะถูกซ่อน
- **การเข้าถึงหน้าเพจ** — การพยายามเข้าถึงหน้าเพจที่ถูกจำกัดจะแสดงข้อความข้อผิดพลาดเกี่ยวกับสิทธิ์
- **การจำกัดการดำเนินการ** — เมื่อมีสิทธิ์ "ดู" ปุ่มแก้ไขและลบจะถูกซ่อน และการดำเนินการบันทึกจะถูกบล็อก
- **การข้ามผู้ใช้ระดับสูงสุด** — บัญชีผู้ใช้ระดับสูงสุดจะมีสิทธิ์เต็มที่เสมอไม่ว่าบทบาทที่กำหนดจะเป็นอย่างไร

### เครื่อง POS

- **การเข้าสู่ระบบ** — ผู้ใช้ที่มีบทบาทอย่างน้อยหนึ่งบทบาทที่มีการเปิดใช้งาน "เครื่อง POS" สามารถเข้าสู่ระบบเครื่อง POS ได้
- **การเปิดใช้งานคุณสมบัติ** — ปุ่มและการดำเนินการเครื่อง POS (การคืนเงิน ส่วนลด การยกเลิก ฯลฯ) จะแสดงหรือซ่อนตามสิทธิ์ที่รวมกันในเครื่อง POS ของผู้ใช้
- **ข้อจำกัดส่วนลด** — ค่าเปอร์เซ็นต์ส่วนลดสูงสุดกำหนดข้อจำกัดที่เข้มงวดสำหรับขนาดส่วนลดที่ผู้ปฏิบัติงานเครื่อง POS สามารถนำไปใช้ได้
- **การบังคับใช้ API** — ทุกสิทธิ์ในเครื่อง POS จะถูกตรวจสอบที่ระดับ API ด้านเซิร์ฟเวอร์ ไม่ใช่แค่ใน UI เท่านั้น

## เคล็ดลับ

- **เริ่มต้นด้วยบทบาทที่กำหนดไว้ล่วงหน้า** — บทบาทที่ติดตั้งไว้ 7 บทบาทครอบคลุมโครงสร้างทีมส่วนใหญ่ สร้างบทบาทที่กำหนดเองเพียงเมื่อคุณต้องการการควบคุมการเข้าถึงที่เฉพาะเจาะจงมากขึ้น
- **ใช้คุณสมบัติการคัดลอก** — เมื่อคุณต้องการบทบาทที่คล้ายกับบทบาทที่มีอยู่ ให้คัดลอกบทบาทนั้นและปรับแต่งแทนที่จะสร้างจากศูนย์
- **กำหนดบทบาทหลายบทบาทเมื่อจำเป็น** — พนักงานที่ดูแลทั้งคำสั่งซื้อและการตลาดสามารถได้รับบทบาท "ผู้จัดการคำสั่งซื้อ" และ "ผู้จัดการการตลาด" ได้ สิทธิ์จะรวมกันโดยอัตโนมัติ
- **แยกการเข้าถึงแดชบอร์ดผู้ดูแลระบบและเครื่อง POS** — แคชเชียร์มักไม่จำเป็นต้องมีสิทธิ์เข้าถึงแดชบอร์ดผู้ดูแลระบบ และพนักงานในสำนักงานไม่จำเป็นต้องมีสิทธิ์เข้าถึงเครื่อง POS ใช้ตัวสลับการเข้าถึงเพื่อรักษาความสะอาด
- **ตั้งค่าข้อจำกัดส่วนลดสำหรับพนักงานเครื่อง POS** — ค่าเปอร์เซ็นต์ส่วนลดสูงสุดช่วยป้องกันไม่ให้แคชเชียร์ใช้ส่วนลดที่มากเกินไป ตั้งค่าเป็น 0 หากไม่ต้องการส่วนลด หรือตั้งค่าข้อจำกัดที่สมเหตุสมผล เช่น 10–25% สำหรับพนักงานระดับสูง
- **ตรวจสอบบทบาทเป็นระยะ** — เมื่อทีมของคุณเติบโต ตรวจสอบการกำหนดบทบาทเพื่อให้แน่ใจว่าพนักงานมีสิทธิ์เข้าถึงขั้นต่ำที่จำเป็นสำหรับงานของพวกเขา ลบบทบาทเมื่อผู้คนเปลี่ยนตำแหน่ง
- **เปิดใช้งาน 2FA สำหรับบทบาทที่ไว้ใจไม่ได้** — พนักงานที่มีสิทธิ์เข้าถึงการชำระเงิน การตั้งค่า หรือการจัดการผู้ใช้ควรเปิดใช้งานการยืนยันตัวตนแบบสองขั้นตอนเพื่อความปลอดภัย
