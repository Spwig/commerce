---
title: 多幣種設定
---

多幣種功能讓您的客戶可以使用其偏好的幣種瀏覽產品並完成結帳。價格會根據連接的提供商或手動設定的匯率，從您的基準幣種自動轉換。

## 程序開始前

在啟用多幣種功能之前，您需要：

1. **一個活躍的匯率提供商** - 請前往 **設定 > 多幣種 > 匯率儀表板** 並連接至少一個提供商（例如 Open Exchange Rates、Fixer.io 或 ExchangeRate-API）。提供商必須處於活躍狀態並同步匯率。
2. **至少兩種幣種** - 您的基準幣種加上一種或多種您想要支援的額外幣種。

## 啟用多幣種

前往 **設定 > 多幣種** 並勾選 **啟用多幣種**。啟用後，請配置以下選項：

| 設定 | 說明 |
|---------|-------------|
| **幣種選擇模式** | 客戶選擇幣種的方式。*自動* 會根據他們的位置自動偵測，*手動* 讓他們從切換器中選擇，*兩者皆有* 結合這兩種方式。 |
| **顯示幣種切換器** | 在您的商店前端顯示幣種選擇器，讓客戶可以手動更改幣種。 |
| **切換器位置** | 貨幣切換器出現的位置（標題列、頁尾或側邊欄）。 |
| **顯示匯率資訊** | 向客戶顯示價格是從您的基準幣種轉換而來的近似值。 |
| **啟用地區格式** | 根據每個客戶的地區格式化數字和貨幣符號（例如歐洲格式的 1.234,56）。 |

## 結帳模式

選擇多幣種在結帳時的運作方式：

| 模式 | 說明 |
|------|-------------|
| **完整多幣種** | 客戶以選擇的幣種瀏覽、加入購物車並付款。匯率在結帳時鎖定並記錄在訂單中。這是預設選項。 |
| **僅顯示** | 價格以客戶的幣種顯示，方便查看，但購物車和付款始終以您的基準幣種處理。在結帳時，客戶會看到一個提示，顯示近似轉換金額以及實際收費金額（以您的基準幣種顯示）。 |

**僅顯示** 適用於當您的付款提供商僅支援您的基準幣種，或當您想要完全避免匯率風險時。客戶在瀏覽時仍會看到本地化的價格，讓他們以自己的幣種了解成本。

## 匯率同步間隔

控制您的商店從連接的提供商處獲取最新匯率的頻率：

| 間隔 | 說明 |
|----------|-------------|
| **即時** | 每15分鐘一次。適合國際銷售量高的商店。 |
| **每小時** | 每小時一次。新鮮度和 API 使用之間的良好平衡。 |
| **每天** | 每天一次。適合大多數商店。這是預設選項。 |
| **每週** | 每週一次。適合價格穩定的商店。 |
| **每月 / 每季** | 對於很少更改匯率的商店，更新頻率較低。 |
| **僅手動** | 汇率從不自動獲取。您需手動管理所有匯率。 |

同步間隔會影響後台任務從提供商處獲取匯率的頻率。在同步之間，會使用緩存的匯率。如果您需要立即強制同步，請使用 **Exchange Rates Dashboard** 上的 **Sync Now** 按鈕，或在 **Manual Exchange Rates** 頁面上使用 **Sync from Provider**。

## 手動匯率

手動匯率讓您可以為特定幣種組設定精確的轉換率。它們會優先於從提供商獲取的匯率，讓您完全掌控定價。

前往 **Exchange Rates > Manual Exchange Rates** 來管理它們。

### 手動設定匯率

點擊 **Add Rate** 來為幣種組建立一個匯率。指定基準幣種、目標幣種和匯率。例如，將 USD/EUR 設為 0.92 表示 1 USD = 0.92 EUR。

### 從提供商同步

點擊 **Sync from Provider** 來自動從連接的提供商的最新匯率填充手動匯率。

這會為所有支援的貨幣創建手動匯率，為您提供一個起點來進行微調。

在同步過程中會跳過已鎖定的匯率，因此您手動調整的任何匯率都不會被覆蓋。

### 停用匯率

點擊任何匯率旁邊的鎖定圖示，以防止在提供商同步時被覆蓋。當您在特定條件下談妥了特定匯率，或希望維持固定匯率而不受市場波動影響時，這非常有用。

- **已停用**的匯率會顯示鎖定徽章，並排除在自動同步之外。
- **未停用**的匯率可以在點擊「從提供商同步」時進行更新。

### 提供商比較

每個手動匯率會顯示當前提供商的匯率，並附上百分比差異。這有助於您快速查看手動匯率與市場匯率的比較：

- **綠色**百分比表示您的匯率高於提供商的匯率。
- **紅色**百分比表示您的匯率低於提供商的匯率。

## 匯率附加利潤

您可以為匯率添加百分比附加利潤，以覆蓋貨幣轉換費用，並保護在客戶下單與您收到付款之間的匯率波動。

例如，在 1.18 USD/EUR 的匯率上添加 2% 的附加利潤，會將其調整為約 1.20 USD/EUR。這個小幅度的緩衝有助於確保您不會在貨幣轉換中虧損。

## 匯率選擇策略

當您在多個匯率提供商之間連接時，可以選擇如何選擇匯率：

- **主要提供商** - 始終使用您指定的主要提供商的匯率。這確保了商店內的價格一致性。如果主要提供商沒有某種貨幣對的數據，則會回退到任何提供商的最新可用匯率。
- **最新可用** - 使用任何活動提供商的最新同步匯率。這會讓您獲得最新數據，但不同提供商之間的匯率可能會略有差異。

對於大多數商店，**主要提供商**是推薦的選擇，因為它能提供最可預測的定價。

## 支援的貨幣

使用拖放貨幣管理員來選擇您商店支援的貨幣：

1. **可用貨幣**（左欄）顯示所有您可以啟用的貨幣。
2. **活動貨幣**（右欄）顯示目前在您商店中啟用的貨幣。
3. 在欄位之間拖動貨幣以啟用或停用它們。
4. 在活動欄內拖動以重新排序貨幣在切換器中的顯示順序。
5. 點擊 **儲存貨幣設定** 以套用您的更改。

您的基準貨幣始終處於啟用狀態，無法移除。

## 如何解析匯率

當需要將價格轉換時，系統會按照以下順序檢查匯率：

1. **手動匯率** - 如果存在某個貨幣對的活動手動匯率，則始終優先使用它。
2. **提供商匯率** - 如果沒有手動匯率，則使用連接提供商的最新匯率。

這意味著您可以為大多數貨幣使用提供商，並在需要精確控制的地方覆蓋特定貨幣對的手動匯率。

## 重要：此設定是永久性的

一旦啟用多幣種功能，且客戶以外幣下單，此設定 **無法停用**。這是因為：

- 訂單會永久儲存客戶選擇的貨幣和購買時使用的匯率。
- 財務報告和退款計算依賴於這項歷史貨幣資料。
- 停用多幣種功能會導致現有的多幣種訂單處於不一致的狀態。

如果尚未有任何外幣訂單，您仍然可以停用多幣種功能。

## 小提示

保留所有標記語言格式、圖片路徑、程式碼塊和技術術語。

- **先以小額訂單測試** - 在外幣中下一個測試訂單，以驗證結帳流程，並確保匯率正確套用。
- **定期監控匯率** - 周期性查看匯率儀表板，確保您的供應商正在同步匯率，且數字看起來合理。
- **考慮波動性貨幣的利潤空間** - 如果您支援波動性較高的貨幣，稍微提高一點利潤空間（2-3%）可以保護您的利潤。
- **從主要貨幣開始** - 從廣泛使用的貨幣（EUR、GBP、JPY、CAD、AUD）開始，根據客戶需求逐步擴展。
- **審查支付供應商的相容性** - 並非所有支付供應商都支援所有貨幣。

查看您的支付供應商文件，確認他們處理哪些貨幣。
- **如果不確定，請使用僅顯示模式** - 如果您不確定支付供應商是否支援多幣種結帳，請先使用僅顯示模式開始。

之後可以切換到完整的多幣種模式。
- **促銷前鎖定匯率** - 如果您正在進行促銷活動，請在活動開始前鎖定匯率，以確保促銷期間價格一致。
