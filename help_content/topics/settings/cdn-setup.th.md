---
title: การตั้งค่า CDN
---

เครือข่ายการจัดส่งเนื้อหา (CDN) จะเก็บสำเนาของรูปภาพ ไฟล์สไตล์ และสคริปต์ของร้านคุณไว้บนเซิร์ฟเวอร์ทั่วโลก เมื่อลูกค้าเข้าเยี่ยมชมร้านคุณ ไฟล์เหล่านี้จะถูกส่งจากเซิร์ฟเวอร์ที่อยู่ใกล้ที่สุดแทนที่จะส่งจากเซิร์ฟเวอร์หลักของคุณ ซึ่งช่วยลดเวลาในการโหลดหน้าเว็บ โดยเฉพาะสำหรับลูกค้าที่อยู่ห่างไกลจากที่ร้านคุณได้รับการโฮสต์

Spwig ได้ปรับปรุงการส่งไฟล์สติชทั้งหมดโดยอัตโนมัติโดยใช้การบีบอัดล่วงหน้าด้วย Brotli และ gzip การแคชไฟล์ที่มีลายนิ้วมือพร้อมหัวข้อที่ไม่เปลี่ยนแปลงเป็นเวลา 1 ปี และการเจรจาเนื้อหาที่เหมาะสม การเพิ่ม CDN เป็นทางเลือก แต่สามารถเพิ่มความเร็วได้มากยิ่งขึ้นสำหรับร้านค้าที่มีลูกค้าทั่วโลก

## ร้านคุณจำเป็นต้องใช้ CDN หรือไม่?

ไม่ใช่ทุกร้านค้าที่ได้รับประโยชน์เท่ากันจาก CDN ใช้แนวทางเหล่านี้เพื่อตัดสินใจ:

**แนะนำให้ใช้ CDN หาก**:
- ลูกค้าของคุณกระจายตัวอยู่หลายประเทศหรือทวีป
- ร้านคุณมีรูปภาพจำนวนมากหรือหน้าเว็บที่เน้นสื่อ
- คุณต้องการเวลาในการโหลดหน้าเว็บที่เร็วที่สุดในโลก
- คุณขายสินค้าไปยังภูมิภาคที่อยู่ห่างไกลจากเซิร์ฟเวอร์ของคุณ (เช่น เซิร์ฟเวอร์ในยุโรป ลูกค้าในเอเชีย)

**ไม่จำเป็นต้องใช้ CDN หาก**:
- ลูกค้าของคุณเป็นส่วนใหญ่เป็นท้องถิ่นหรืออยู่ในประเทศเดียวกับเซิร์ฟเวอร์ของคุณ
- ร้านคุณมีแคตตาล็อกขนาดเล็กพร้อมรูปภาพจำนวนน้อย
- ผู้ให้บริการโฮสต์ของคุณมี CDN ที่รวมไว้แล้ว

เมื่อไม่แน่ใจ ไม่มีผลต่อประสิทธิภาพ การใช้ CDN บริการเช่น Cloudflare มีแผนฟรี ดังนั้นจึงไม่มีค่าใช้จ่ายในการลองใช้

## การทำงานของ Spwig กับ CDN

Spwig พร้อมใช้งานกับ CDN โดยค่าเริ่มต้น คุณไม่จำเป็นต้องเปลี่ยนโค้ดหรือการตั้งค่าใดๆ ภายในแดชบอร์ดของ Spwig นี่คือสิ่งที่ Spwig ทำให้คุณแล้ว:

- **ไฟล์สติชที่มีลายนิ้วมือ** -- ทุกไฟล์ CSS, JavaScript และรูปภาพมีแฮชเวอร์ชันที่เป็นเอกลักษณ์ในชื่อไฟล์ ซึ่งหมายความว่า CDN สามารถแคชไฟล์เหล่านี้ได้เป็นเวลานานโดยไม่ต้องส่งเนื้อหาที่ล้าสมัย
- **หัวข้อแคชที่มีอายุยาวนาน** -- ไฟล์สติชถูกส่งพร้อมกับหัวข้อแคชที่ไม่เปลี่ยนแปลงเป็นเวลา 1 ปี ซึ่งบอก CDN และเบราว์เซอร์ให้แคชไฟล์เหล่านี้อย่างเข้มงวด
- **ไฟล์ที่บีบอัดล่วงหน้า** -- Spwig บีบอัดไฟล์ล่วงหน้าด้วย Brotli และ gzip ดังนั้น CDN ของคุณจึงสามารถส่งไฟล์ขนาดเล็กลงโดยไม่ต้องประมวลผลเพิ่มเติม
- **การเจรจาเนื้อหาที่เหมาะสม** -- Spwig ส่งหัวข้อ content-type และการเข้ารหัสที่ถูกต้องซึ่ง CDN ใช้เพื่อการแคชที่เหมาะสม

สิ่งที่คุณต้องทำคือการตั้งค่า DNS ของโดเมนของคุณไปยังผู้ให้บริการ CDN และทุกอย่างจะทำงานโดยอัตโนมัติ

## การตั้งค่า Cloudflare

Cloudflare เป็น CDN ที่ได้รับความนิยมสูงสุดและมีแผนฟรีที่เหมาะกับร้านค้าส่วนใหญ่ โปรดปฏิบัติตามขั้นตอนต่อไปนี้:

**ขั้นตอนที่ 1: สร้างบัญชี Cloudflare**
- เยี่ยมชม cloudflare.com และสมัครสมาชิกบัญชีฟรี

**ขั้นตอนที่ 2: เพิ่มโดเมนของคุณ**
- คลิก **Add a Site** และป้อนชื่อโดเมนร้านคุณ
- เลือกแผน **Free** (เพียงพอสำหรับร้านค้าส่วนใหญ่)

**ขั้นตอนที่ 3: อัปเดต Nameserver ของ DNS**
- Cloudflare จะแสดง Nameserver สองตัว (เช่น `anna.ns.cloudflare.com`)
- เข้าสู่ระบบไปยังผู้จดทะเบียนโดเมนของคุณ (ที่คุณซื้อโดเมน)
- แทน Nameserver ปัจจุบันด้วย Nameserver ของ Cloudflare
- การเปลี่ยน DNS อาจใช้เวลาสูงสุด 24 ชั่วโมงในการทำงาน

**ขั้นตอนที่ 4: ตั้งค่า SSL/TLS**
- ในแดชบอร์ดของ Cloudflare ไปที่ **SSL/TLS**
- ตั้งค่าโหมดการเข้ารหัสเป็น **Full (strict)**
- นี่จะทำให้การสื่อสารระหว่าง Cloudflare และเซิร์ฟเวอร์ของคุณมีการเข้ารหัสทั้งหมด

**ขั้นตอนที่ 5: ตรวจสอบว่าทำงานได้จริง**
- เมื่อ DNS ได้รับการเผยแพร่ ให้เข้าเยี่ยมชมร้านคุณและตรวจสอบหัวข้อ `cf-cache-status` ในเบราว์เซอร์ของคุณ (ดูที่ การตรวจสอบ CDN ของคุณ ด้านล่าง)

## การตั้งค่า AWS CloudFront

หากคุณใช้ Amazon Web Services อยู่แล้ว CloudFront จะรวมกับโครงสร้างพื้นฐานของคุณได้อย่างเป็นธรรมชาติ:

1. เปิด **CloudFront** ในแดชบอร์ด AWS ของคุณ
2. สร้าง **Distribution** ใหม่โดยใช้โดเมนร้านคุณเป็นแหล่งข้อมูล
3. ตั้งค่า **Origin Protocol Policy** เป็น "HTTPS Only"
4. ภายใต้ **Cache Behavior** ตั้งค่า **Cache Policy** เป็น "CachingOptimized" สำหรับไฟล์สติช
5. เพิ่มโดเมนร้านคุณเป็น **Alternate Domain Name (CNAME)**
6. ติดตั้งใบรับรอง SSL จาก AWS Certificate Manager
7. อัปเดต DNS ของโดเมนของคุณให้ชี้ไปยัง URL ของ CloudFront distribution


การกำหนดราคา CloudFront ขึ้นอยู่กับการใช้งาน

สำหรับร้านค้าส่วนใหญ่ ค่าใช้จ่ายจะอยู่ในระดับต่ำ เนื่องจากทรัพยากรของ Spwig ถูกแคชไว้เป็นระยะเวลานาน

## การตั้งค่า CDN ที่แนะนำ

เพื่อให้ได้ผลลัพธ์ที่ดีที่สุด ให้ตั้งค่า CDN ของคุณให้แคชเนื้อหาที่เหมาะสมและข้ามเนื้อหาที่เหลือ

**สิ่งที่ควรแคช** (ทรัพยากรสถิต):
- `/static/` -- ไฟล์สไตล์, สคริปต์, ฟอนต์ และทรัพยากรธีมทั้งหมด
- `/media/` -- รูปภาพสินค้าและไฟล์สื่อที่อัปโหลด
- ไฟล์รูปภาพ (`.jpg`, `.png`, `.webp`, `.svg`, `.gif`)
- ไฟล์ฟอนต์ (`.woff`, `.woff2`)

**สิ่งที่ไม่ควรแคช** (หน้าเว็บแบบไดนามิก):
- `/admin/` -- แผงควบคุมต้องส่งเนื้อหาที่สดเสมอ
- `/cart/` -- หน้าตะกร้าสินค้ามีข้อมูลเฉพาะเซสชัน
- `/checkout/` -- หน้าชำระเงินไม่ควรแคชเพื่อความปลอดภัย
- `/accounts/` -- หน้าบัญชีลูกค้ามีข้อมูลส่วนตัว
- หน้าใดๆ ที่ต้องการการเข้าสู่ระบบหรือแสดงเนื้อหาเฉพาะบุคคล

**กฎการแคชทั่วไป**:
- **ให้ความสำคัญกับหัวข้อแคชจากแหล่งต้นทาง** -- Spwig ส่งหัวข้อ cache-control ที่ถูกต้องสำหรับแต่ละประเภทของเนื้อหา ให้ตั้งค่า CDN ของคุณให้เคารพหัวข้อนี้แทนที่จะทับ書き
- **เปิดใช้งานการบีบอัด Brotli** -- ทั้ง Cloudflare และ CloudFront สนับสนุน Brotli เปิดใช้งานเพื่อใช้ประโยชน์จากทรัพยากรที่ถูกบีบอัดล่วงหน้าของ Spwig
- **ตั้งค่า TTL แคชเบราว์เซอร์เป็น "Respect Existing Headers"** -- นี่จะให้นโยบายแคชที่สร้างขึ้นโดย Spwig กำหนดพฤติกรรม

## การตรวจสอบ CDN ของคุณ

หลังจากตั้งค่า เข้าตรวจสอบว่า CDN กำลังให้เนื้อหาของคุณได้อย่างถูกต้อง:

**ขั้นตอนที่ 1: เปิดเครื่องมือพัฒนาเบราว์เซอร์**
- ใน Chrome หรือ Firefox กด **F12** เพื่อเปิดเครื่องมือพัฒนา
- คลิกแท็บ **Network**

**ขั้นตอนที่ 2: โหลดร้านค้าของคุณ**
- เยี่ยมชมหน้าแรกของร้านค้าของคุณโดยมีเครื่องมือพัฒนาเปิดอยู่
- คลิกที่การร้องขอไฟล์สถิตใดๆ (เช่น ไฟล์ `.css` หรือ `.js`)

**ขั้นตอนที่ 3: ตรวจสอบหัวข้อการตอบกลับ**
- **Cloudflare**: ค้นหาหัวข้อ `cf-cache-status` ค่า `HIT` หมายถึงไฟล์ถูกส่งจากแคชของ CDN `MISS` หมายถึงไฟล์ถูกดึงจากเซิร์ฟเวอร์ของคุณ (เฉพาะการร้องขอครั้งแรกเท่านั้น)
- **CloudFront**: ค้นหาหัวข้อ `x-cache` ค่า `Hit from cloudfront` ยืนยันการส่งผ่านจาก CDN

**ขั้นตอนที่ 4: ทดสอบจากสถานที่อื่น**
- ใช้เครื่องมือฟรีเช่น gtmetrix.com หรือ webpagetest.org เพื่อทดสอบร้านค้าของคุณจากสถานที่ทางภูมิศาสตร์ต่างๆ
- เปรียบเทียบเวลาโหลดก่อนและหลังการตั้งค่า CDN

## ปัญหาที่พบบ่อย

### เนื้อหาเก่าหลังจากเปลี่ยนธีม

**ปัญหา**: หลังจากอัปเดตธีมหรือทำการเปลี่ยนแปลงดีไซน์ ลูกค้ายังคงเห็นเวอร์ชันเก่า

**วิธีแก้ไข**: ล้างแคชของ CDN ของคุณ ใน Cloudflare ไปที่ **Caching > Configuration > Purge Everything** ใน CloudFront สร้าง **Invalidation** สำหรับ `/*` โปรดทราบว่าทรัพยากรที่ถูกทำเครื่องหมายด้วยลายนิ้วมือของ Spwig มักจะป้องกันปัญหานี้ เนื่องจากไฟล์ที่อัปเดตจะได้รับชื่อไฟล์ใหม่โดยอัตโนมัติ ปัญหานี้มักเกิดกับทรัพยากรที่ไม่ได้ทำเครื่องหมายด้วยลายนิ้วมือ เช่น การอัปโหลดแบบกำหนดเอง

---

### ข้อความเตือนเนื้อหาแบบผสม

**ปัญหา**: บราวเซอร์ของคุณแสดงข้อความเตือนด้านความปลอดภัยเกี่ยวกับ "เนื้อหาแบบผสม" หลังจากเปิดใช้งาน CDN

**วิธีแก้ไข**: ตรวจสอบให้แน่ใจว่าโหมด SSL ของ CDN ของคุณตั้งค่าเป็น **Full (strict)** ไม่ใช่ "Flexible" โหมด Flexible อาจทำให้เซิร์ฟเวอร์ของคุณได้รับคำขอ HTTP แทน HTTPS ซึ่งนำไปสู่ข้อความเตือนเนื้อหาแบบผสม ใน Cloudflare ตรวจสอบ **SSL/TLS > Overview** และตรวจสอบโหมด

---

### แผงควบคุมทำงานช้าลง

**ปัญหา**: แผงควบคุมดูช้าลงหลังจากเพิ่ม CDN

**วิธีแก้ไข**: CDN ไม่ควรแคชหน้าแผงควบคุม สร้าง **Page Rule** (Cloudflare) หรือ **Cache Behavior** (CloudFront) ที่ตั้งค่าการแคชเป็น "Bypass" สำหรับ URL ใดๆ ที่ตรงกับ `/admin/*` นี่จะทำให้การร้องขอแผงควบคุมไปยังเซิร์ฟเวอร์โดยตรงโดยไม่มีการรบกวนจาก CDN

---

### รูปภาพไม่โหลด

**ปัญหา**: รูปภาพสินค้าหรือไฟล์สื่อแสดงข้อผิดพลาดหลังจากตั้งค่า CDN

**วิธีแก้ไข**: ตรวจสอบให้แน่ใจว่า origin ของ CDN ของคุณตั้งค่าด้วยโปรโตคอลที่ถูกต้อง (HTTPS) และพอร์ต ตรวจสอบอีกครั้งว่าไฟร์วอลล์ของเซิร์ฟเวอร์ของคุณอนุญาตให้เชื่อมต่อจากช่วง IP ของ CDN

## เคล็ดลับ

รักษาการจัดรูปแบบ markdown, ค่าเส้นทางภาพ, บล็อกโค้ด และคำศัพท์ทางเทคนิคไว้ทั้งหมด

- **เริ่มต้นด้วยแผนฟรีของ Cloudflare** -- มันครอบคลุมความต้องการของร้านค้าส่วนใหญ่ และใช้เวลาเพียงไม่กี่นาทีในการตั้งค่า
- **ใช้โหมด SSL แบบ Full (strict) เสมอ** -- โหมดที่ยืดหยุ่นอาจสร้างช่องโหว่ด้านความปลอดภัย และอาจทำให้ขั้นตอนการชำระเงินล้มเหลว
- **ล้างแคช CDN หลังอัปเดตธีมสำคัญ** -- แม้ว่าไฟล์ที่มีการติดตั้งลายน้ำของ Spwig จะจัดการกรณีส่วนใหญ่ แต่การล้างแคชทั้งหมดจะช่วยให้มั่นใจว่าไม่มีเนื้อหาเก่าที่เหลืออยู่
- **อย่าแคชหน้าชำระเงินหรือตะกร้าสินค้า** -- การแคชหน้าเหล่านี้อาจทำให้ข้อมูลของลูกค้าคนหนึ่งถูกเปิดเผยให้ลูกค้าอีกคนเห็น
- **ทดสอบจากตำแหน่งของลูกค้าของคุณ** -- ใช้เครื่องมือฟรีเช่น webpagetest.org เพื่อวัดประสิทธิภาพในโลกจริงจากภูมิภาคที่ลูกค้าของคุณช้อปปิ้ง
- **ตรวจสอบสถิติของ CDN ของคุณ** -- ทั้ง Cloudflare และ CloudFront ให้แดชบอร์ดที่แสดงอัตราการเข้าถึงแคช ปริมาณแบนด์วิธที่ประหยัดได้ และการจราจรตามประเทศ
- **รักษา TTL DNS ให้ต่ำในช่วงการตั้งค่า** -- ตั้งค่า TTL DNS เป็น 300 วินาที (5 นาที) ขณะเปลี่ยนไปใช้ CDN แล้วเพิ่มค่าเมื่อทุกอย่างได้รับการยืนยันว่าทำงานได้ดี
- **CDN ไม่สามารถแทนที่การโฮสต์ที่ดีได้** -- เซิร์ฟเวอร์ต้นทางของคุณยังมีความสำคัญสำหรับหน้าที่มีการเปลี่ยนแปลง เช่น การชำระเงิน ตะกร้าสินค้า และการจัดการระบบ

เลือกการโฮสต์คุณภาพสูงร่วมกับ CDN
