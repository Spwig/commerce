---
title: บัญชีผู้ให้บริการจัดส่ง
---

บัญชีผู้ให้บริการจัดส่งจะเชื่อมต่อร้านค้าของคุณกับ API ของผู้ให้บริการจัดส่ง (FedEx, UPS, DHL) เพื่อคำนวณอัตราค่าจัดส่งแบบเรียลไทม์และซื้อฉลากอัตโนมัติ บัญชีแต่ละบัญชีจะเก็บข้อมูลการเข้าถึง API ที่เข้ารหัส ตรวจสอบสุขภาพการเชื่อมต่อ และเชื่อมโยงกับวิธีการจัดส่งแบบเรียลไทม์ ผู้ให้บริการจะดึงอัตราค่าจัดส่งแบบสดที่เคาน์เตอร์ชำระเงินโดยพิจารณาจากขนาด น้ำหนัก จุดเริ่มต้น และจุดปลายทางของแพ็กเกจ ซึ่งช่วยลดการบำรุงรักษาตารางอัตราค่าจัดส่งแบบมือและรับประกันราคาผู้ให้บริการที่ถูกต้อง

ใช้บัญชีผู้ให้บริการเมื่อคุณต้องการอัตราค่าจัดส่งที่คำนวณโดยผู้ให้บริการหรือการสร้างฉลากอัตโนมัติแทนการสร้างการจัดส่งแบบมือ

## ผู้ให้บริการจัดส่งที่รองรับ

Spwig รองรับผู้ให้บริการหลักผ่านส่วนประกอบผู้ให้บริการที่ติดตั้งได้:

### FedEx

**บริการ**: Ground, Express, International
**API**: FedEx Web Services
**คุณสมบัติ**: อัตราค่าจัดส่งแบบเรียลไทม์, การซื้อฉลาก, การติดตาม, ค่าธรรมเนียมระหว่างประเทศ

### UPS

**บริการ**: Ground, Air, Worldwide
**API**: UPS Developer API
**คุณสมบัติ**: อัตราค่าจัดส่งแบบเรียลไทม์, การสร้างฉลาก, การติดตาม, การตรวจสอบที่อยู่

### DHL

**บริการ**: Express, eCommerce, International
**API**: DHL Express API
**คุณสมบัติ**: อัตราค่าจัดส่งระหว่างประเทศ, เอกสารค่าธรรมเนียมระหว่างประเทศ, การติดตาม

### ผู้ให้บริการเพิ่มเติม

ติดตั้งได้ตามความจำเป็นจากตลาดส่วนประกอบ (USPS, Canada Post, Australia Post, ฯลฯ)

## การตั้งค่าบัญชีผู้ให้บริการ

แต่ละบัญชีผู้ให้บริการต้องการ:

### ข้อมูลพื้นฐาน

- **ชื่อแสดง**: วิธีที่บัญชีปรากฏในแดชบอร์ดผู้ดูแลระบบ (เช่น "FedEx Production Account")
- **ผู้ให้บริการ**: เลือกส่วนประกอบผู้ให้บริการที่ติดตั้งแล้วจากเมนูแบบเลื่อนลง
- **เปิดใช้งาน**: เปิด/ปิดใช้งานโดยไม่ต้องลบข้อมูลการเข้าถึง
- **เป็นค่าเริ่มต้น**: ตั้งค่าเป็นบัญชีผู้ให้บริการค่าเริ่มต้นสำหรับผู้ให้บริการนี้ (มีเพียงหนึ่งบัญชีค่าเริ่มต้นต่อผู้ให้บริการ)

### ข้อมูลการเข้าถึง API (เข้ารหัส)

**ขึ้นอยู่กับผู้ให้บริการ** โดยทั่วไปจะรวม:

**FedEx**:
- หมายเลขบัญชี
- หมายเลขมิเตอร์
- คีย์ API
- รหัสลับ API

**UPS**:
- หมายเลขใบอนุญาตการเข้าถึง
- ชื่อผู้ใช้
- รหัสผ่าน
- หมายเลขบัญชี

**DHL**:
- ไอดีไซต์
- รหัสผ่าน
- หมายเลขบัญชี

**ข้อมูลการเข้าถึงทั้งหมดถูกเข้ารหัสเมื่ออยู่ในพื้นที่** และถูกถอดรหัสเพียงเมื่อทำการเรียกใช้ API

### ที่อยู่จุดเริ่มต้น

- **ที่อยู่จุดเริ่มต้นเริ่มต้น**: ที่อยู่คลังสินค้า/จุดเริ่มต้นสำหรับการคำนวณอัตราค่าจัดส่ง
- ผู้ให้บริการบางรายต้องการการตั้งค่าจุดเริ่มต้นเฉพาะในแดชบอร์ดของตนเอง

### การตั้งค่า

ตัวเลือกเฉพาะผู้ให้บริการ (ขึ้นอยู่กับผู้ให้บริการ):

- **โหมดทดสอบ**: ใช้จุดปลายทาง API ทดสอบ/สแตนด์บี้ของผู้ให้บริการ
- **อัตราค่าที่เจรจา**: ใช้อัตราค่าที่เจรจาได้กับผู้ให้บริการ (หากมี)
- **รวมประกัน**: ตั้งค่าประกันอัตโนมัติในอัตราค่าจัดส่ง
- **ค่าธรรมเนียมสำหรับที่อยู่ที่อยู่อาศัย**: ใช้ค่าธรรมเนียมการจัดส่งสำหรับที่อยู่อาศัย
- **ต้องการลายเซ็น**: ตั้งค่าความต้องการลายเซ็นเริ่มต้น

## การสร้างบัญชีผู้ให้บริการ

**กระบวนการตั้งค่า 6 ขั้นตอน**:

**ขั้นตอนที่ 1: ขอเข้าถึง API ของผู้ให้บริการ**
1. สร้างบัญชีกับผู้ให้บริการ (FedEx.com, UPS.com, DHL.com)
2. ขอเข้าถึง API/Developer
3. สมัครใช้งาน API ของผู้ให้บริการ (อาจใช้เวลา 1-3 วันทำการ)
4. รับข้อมูลการเข้าถึง API ผ่านอีเมลหรือพอร์ทัลผู้พัฒนา

**ขั้นตอนที่ 2: ติดตั้งส่วนประกอบผู้ให้บริการ** (หากยังไม่ได้ติดตั้งไว้ล่วงหน้า)
1. ไปที่ Settings > Components > Marketplace
2. ค้นหาชื่อผู้ให้บริการ (เช่น "FedEx")
3. ติดตั้งส่วนประกอบผู้ให้บริการจัดส่ง
4. รอจนกว่าการติดตั้งจะเสร็จสิ้น

**ขั้นตอนที่ 3: สร้างบัญชีผู้ให้บริการใน Spwig**
1. ไปที่ Settings > Shipping > Provider Accounts
2. คลิก "Add Provider Account"
3. เลือกผู้ให้บริการจากเมนูแบบเลื่อนลง
4. ป้อนชื่อแสดง

**ขั้นตอนที่ 4: ป้อนข้อมูลการเข้าถึง API**
1. กรอกช่องข้อมูลการเข้าถึง (ขึ้นอยู่กับผู้ให้บริการ)
2. ข้อมูลการเข้าถึงจะถูกเข้ารหัสอัตโนมัติเมื่อบันทึก
3. ตัวเลือก: เปิดใช้งานโหมดทดสอบสำหรับการทดสอบเบื้องต้น

**ขั้นตอนที่ 5: ทดสอบการเชื่อมต่อ**
1. คลิกปุ่ม "Test Connection"
2. ระบบจะพยายามเรียกใช้ API กับผู้ให้บริการ
3. ตรวจสอบว่าสถานะ "Connected" ปรากฏขึ้น
4. ตรวจสอบเวลาที่บันทึกครั้งสุดท้าย (last_tested_at)

**ขั้นตอนที่ 6: เชื่อมโยงกับวิธีการจัดส่ง**
1. สร้างหรือแก้ไขวิธีการจัดส่ง (Settings > Cart > Shipping Methods)
2. ตั้งค่า method_type = "Real-Time"
3. เลือกบัญชีผู้ให้บริการจากเมนูแบบเลื่อนลง
4. บันทึกวิธีการจัดส่ง

## การตรวจสอบสถานะการเชื่อมต่อ

บัญชีผู้ให้บริการติดตามสุขภาพการเชื่อมต่อ:

### ค่าสถานะ

**ไม่ทราบ** (สีเทา): ยังไม่ได้ทดสอบหรือยังไม่เชื่อมต่อ

**เชื่อมต่อ** (สีเขียว): การเรียกใช้ API ครั้งสุดท้ายสำเร็จ ข้อมูลการเข้าถึงถูกต้อง

**ข้อผิดพลาด** (สีแดง): การเรียกใช้ API ครั้งสุดท้ายล้มเหลว ข้อมูลการเข้าถึงอาจไม่ถูกต้อง

### ครั้งสุดท้ายที่ทดสอบ

- **เวลาที่บันทึก**: เมื่อการเชื่อมต่อได้รับการตรวจสอบ
- **อัปเดตอัตโนมัติ**: ทุกครั้งที่ใช้ผู้ให้บริการ (การดึงอัตราค่าจัดส่ง, การซื้อฉลาก)
- **ทดสอบด้วยตนเอง**: คลิกปุ่ม "Test Connection" ได้ตลอดเวลา

### การแก้ไขปัญหาการเชื่อมต่อที่ล้มเหลว

**สาเหตุที่พบบ่อย**:
- ข้อมูลการเข้าถึง API ไม่ถูกต้อง (สะกดผิด คัดลอกพร้อมช่องว่างเพิ่มเติม)
- คีย์ API ของผู้ให้บริการหมดอายุหรือถูกยกเลิก
- เปิดใช้งานโหมดทดสอบแต่ใช้ข้อมูลการเข้าถึงการผลิต (หรือตรงกันข้าม)
- ที่อยู่ IP ไม่ได้รับการอนุมัติจากผู้ให้บริการ
- ผู้ให้บริการ API อยู่ในช่วงหยุดให้บริการ

**ขั้นตอนการแก้ไข**:
1. ตรวจสอบให้แน่ใจว่าข้อมูลการเข้าถึงตรงกับแดชบอร์ดของผู้ให้บริการอย่างถูกต้อง
2. ตรวจสอบการตั้งค่าโหมดทดสอบให้ตรงกับประเภทของข้อมูลการเข้าถึง
3. ตรวจสอบหน้าสถานะ API ของผู้ให้บริการสำหรับการหยุดให้บริการ
4. ติดต่อฝ่ายสนับสนุนของผู้ให้บริการเพื่อตรวจสอบบัญชี

## วิธีการทำงานในการค้นหาอัตราค่าจัดส่ง

วิธีการทำงานของอัตราค่าจัดส่งแบบเรียลไทม์ที่เคาน์เตอร์ชำระเงิน:

**1. ลูกค้าป้อนที่อยู่**
- ป้อนที่อยู่จัดส่ง
- คำนวณน้ำหนักรวม + ขนาดจากตะกร้าสินค้า

**2. ระบบเตรียมคำขออัตราค่าจัดส่ง**
- ดึงข้อมูลการเข้าถึงผู้ให้บริการ (ถอดรหัส)
- คำนวณขนาดแพ็กเกจจากสินค้าในตะกร้า (ใช้แพ็กเกจจัดส่งหากกำหนดไว้)
- เตรียมคำขอ API พร้อมที่อยู่จุดเริ่มต้น จุดปลายทาง และแพ็กเกจ

**3. เรียกใช้ API ผู้ให้บริการ**
- ส่งคำขอไปยัง API ของผู้ให้บริการพร้อมข้อมูลการเข้าถึง
- ผู้ให้บริการคำนวณอัตราค่าจัดส่งตามโซน น้ำหนัก และขนาด
- ผลตอบกลับจะรวมตัวเลือกบริการ (Ground, Express, ฯลฯ)

**4. แสดงอัตราค่าจัดส่ง**
- ระบบวิเคราะห์ผลตอบกลับจากผู้ให้บริการ
- ปรับให้เป็นรูปแบบมาตรฐาน
- ใช้ค่าเพิ่ม (หากตั้งค่าไว้) ได้
- แสดงอัตราค่าจัดส่งให้ลูกค้าที่เคาน์เตอร์ชำระเงิน

**5. ลูกค้าเลือกบริการ**
- ลูกค้าเลือกตัวเลือกที่ต้องการ
- อัตราค่าจัดส่งที่เลือกจะถูกบันทึกในคำสั่งซื้อ

**ตัวอย่างการไหลของ API**:
```
คำขอไปยัง API ของ FedEx:
{
  "origin": {"postal_code": "90210", "country": "US"},
  "destination": {"postal_code": "10001", "country": "US"},
  "parcels": [{
    "weight": 2500,  // กรัม
    "dimensions": {"length": 30, "width": 20, "height": 15}  // ซม.
  }]
}

ผลตอบกลับจาก FedEx:
[
  {"service": "FEDEX_GROUND", "rate": 12.50, "delivery_days": 5},
  {"service": "FEDEX_EXPRESS", "rate": 28.75, "delivery_days": 2}
]
```

## การซื้อฉลาก (ไม่จำเป็น)

หากผู้ให้บริการรองรับการสร้างฉลาก:

**วิธีการทำงาน**:
1. ลูกค้าสร้างคำสั่งซื้อเสร็จสิ้น
2. ผู้ขายสร้างการจัดส่ง (Orders > Order Detail > Create Shipment)
3. เลือกบัญชีผู้ให้บริการ + บริการ
4. ระบบเรียกใช้ API สร้างฉลากของผู้ให้บริการ
5. สร้าง PDF ฉลากและแนบกับการจัดส่ง
6. หมายเลขติดตามจะถูกเติมอัตโนมัติ
7. ฉลากพร้อมสำหรับการพิมพ์

**ประโยชน์**:
- ไม่จำเป็นต้องเข้าสู่เว็บไซต์ผู้ให้บริการด้วยตนเอง
- การติดตามจะซิงค์อัตโนมัติ
- สร้างแบบฟอร์มค่าธรรมเนียมระหว่างประเทศอัตโนมัติ (ระหว่างประเทศ)
- สร้างฉลากแบบกลุ่มได้

## การเพิ่มค่าอัตราค่าจัดส่ง

เพิ่มค่าค้าของผู้ขายไปยังอัตราค่าจัดส่งของผู้ให้บริการ:

**การตั้งค่า** (ในวิธีการจัดส่ง ไม่ใช่บัญชีผู้ให้บริการ):
- **ประเภทค่าเพิ่ม**: เปอร์เซ็นต์หรือค่าคงที่
- **จำนวนค่าเพิ่ม**: เช่น 15% หรือ $2.50

**ตัวอย่าง**:
```
อัตราค่าจัดส่งจากผู้ให้บริการ: $12.50
ค่าเพิ่ม: 15%
ลูกค้าจ่าย: $14.38

หรือ

อัตราค่าจัดส่งจากผู้ให้บริการ: $12.50
ค่าเพิ่ม: $2.50 (ค่าคงที่)
ลูกค้าจ่าย: $15.00
```

**กรณีการใช้งาน**:
- คุ้มครองค่าใช้จ่ายในการบรรจุภัณฑ์/การจัดการ
- เพิ่มกำไรจากการจัดส่ง
- ชดเชยค่าธรรมเนียมบัตรเครดิตในการจัดส่ง

## บัญชีผู้ให้บริการหลายบัญชี

คุณสามารถสร้างบัญชีผู้ให้บริการหลายบัญชีสำหรับผู้ให้บริการเดียวกัน:

**กรณีการใช้งาน**:
1. **ทดสอบ vs การผลิต**
   - บัญชีทดสอบ: ข้อมูลการเข้าถึงของผู้ให้บริการในสแตนด์บี้
   - บัญชีการผลิต: ข้อมูลการเข้าถึงแบบสด

2. **คลังสินค้าหลายแห่ง**
   - บัญชีคลังสินค้า A: จุดเริ่มต้น = ลอสแองเจลิส
   - บัญชีคลังสินค้า B: จุดเริ่มต้น = นิวยอร์ก

3. **อัตราค่าที่เจรจาต่างกัน**
   - บัญชี A: อัตราค่ามาตรฐาน
   - บัญชี B: อัตราค่าส่วนลดตามปริมาณ

## เคล็ดลับ

- **ทดสอบในสแตนด์บี้ก่อน** - ใช้ข้อมูลการเข้าถึงทดสอบของผู้ให้บริการก่อนเริ่มใช้งานจริง
- **ตรวจสอบสถานะการเชื่อมต่อ** - ตรวจสอบแดชบอร์ดเป็นประจำเพื่อตรวจสอบสถานะข้อผิดพลาด
- **กำหนดแพ็กเกจจัดส่ง** - ขนาดที่ถูกต้องช่วยเพิ่มการเสนออัตราค่าจัดส่ง
- **ใช้อัตราค่าที่เจรจา** - เปิดใช้งานหากคุณมีส่วนลดตามปริมาณกับผู้ให้บริการ
- **ตั้งค่าจุดเริ่มต้นที่เป็นจริง** - ใช้ที่อยู่จุดเริ่มต้นจริงเพื่อให้โซนถูกต้อง
- **รักษาความปลอดภัยของข้อมูลการเข้าถึง** - ไม่เคยแบ่งปันคีย์ API และเปลี่ยนเป็นระยะ
- **มีวิธีการสำรอง** - รักษาวิธีการค่าคงที่ไว้หาก API ของผู้ให้บริการล้มเหลว
- **ตรวจสอบข้อจำกัดของ API ผู้ให้บริการ** - บางผู้ให้บริการจำกัดการเรียกใช้ API ต่อวัน
