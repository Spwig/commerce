---
title: Понимание настроек поиска
---

Интерфейс SearchSettings управляет всем глобальным поведением поиска в вашем магазине Spwig. Эта единственная страница настройки использует интерфейс с 8 вкладками, чтобы организовать параметры поиска, от базовой активации до продвинутой настройки производительности. Изменения здесь применяются ко всем поисковым системам, если только они не переопределены на уровне поисковой системы.

Этот руководство проходит через каждую вкладку, объясняя, что делает каждая настройка и когда её нужно изменять.

![General Tab of Search Settings](/static/core/admin/img/help/search-settings-overview/search-settings-general.webp)

## Интерфейс с 8 вкладками

SearchSettings - это модель singleton, только одна запись конфигурации существует (pk=1) для вашего всего магазина. Интерфейс разделён на восемь вкладок:

| Вкладка | Назначение |
|--------|-----------|
| **Общие** | Включение/выключение поиска, установка базовых параметров |
| **Автодополнение** | Настройка поведения выпадающего списка автодополнения |
| **Типы контента** | Выбор типов контента, которые можно искать |
| **Глубокое индексирование** | Контроль данных продуктов, которые индексируются (влияние на производительность) |
| **Нечёткое совпадение** | Толерантность к опечаткам и пороги схожести |
| **Веса** | Множители релевантности для ранжирования результатов |
| **Кэширование** | Баланс между временем отклика и свежестью |
| **Аналитика** | Отслеживание запросов и настройки приватности |

Каждая вкладка фокусируется на конкретном аспекте настройки поиска.

## Вкладка Общие

Вкладка Общие содержит основные настройки, которые влияют на все поиски:

**Включить поиск** - Главный переключатель для системы поиска. Когда он отключен, все функции поиска неактивны по всему магазину, включая автодополнение и страницу результатов поиска.

**Минимальная длина запроса** - По умолчанию: 2 символа. Поиски короче этого значения отвергаются. Установка этого значения на 1 позволяет выполнять поиски с одним символом (например, "A"), но увеличивает нагрузку на сервер.

**Результаты на странице** - По умолчанию: 20 элементов. Управляет пагинацией для страниц результатов поиска. Более высокие значения (30-50) уменьшают количество кликов по пагинации, но увеличивают время загрузки страницы.

## Вкладка Типы контента

![Settings for Content Types](/static/core/admin/img/help/search-settings-overview/search-settings-content-types.webp)

Переключите, какие типы контента отображаются в результатах поиска:

- **Товары** - Физические, цифровые и подписочные товары
- **Категории** - Категории товаров
- **Бренды** - Бренды товаров
- **Посты блога** - Контент блога

**Примечание по производительности**: Меньше типов контента = более быстрые поиски. Каждый включённый тип добавляет дополнительные запросы к базе данных. Если у вас нет блога, отключите Посты блога, чтобы улучшить время отклика.

## Вкладка Глубокое индексирование

⚠️ **ПРЕДУПРЕЖДЕНИЕ О ПРОИЗВОДИТЕЛЬНОСТИ** - Эти настройки имеют значительное влияние на производительность.

![Settings for Deep Indexing](/static/core/admin/img/help/search-settings-overview/search-settings-deep-indexing.webp)

Глубокое индексирование контролирует, какие данные, связанные с товарами, включаются в поиски:

**Индексировать артикульные номера** - По умолчанию: ВКЛЮЧЁНО, Низкое влияние. Включает артикульные номера товаров и вариантов в поиски. Необходимо для B2B-магазинов, где клиенты ищут по кодам товаров.

**Индексировать атрибуты** - По умолчанию: ВКЛЮЧЁНО, Среднее влияние. Включает атрибуты товаров (цвет, размер, материал) в поиски. Добавляет JOIN к таблице атрибутов. Важно для модной одежды и настраиваемых товаров.

**Индексировать пользовательские поля** - По умолчанию: ВКЛЮЧЁНО, Среднее влияние. Включает пользовательские поля, определённые продавцом, в результаты поиска. Требует обхода JSONField.

**Индексировать отзывы** - По умолчанию: ВКЛЮЧЁНО, Среднее-Высокое влияние ⚠️

Индексация документов извлекает текст из PDF, DOCX и XLSX-файлов, прикреплённых к цифровым товарам. Эта функция:

- Требует очень дорогой начальной индексации
- Добавляет значительную нагрузку на запросы при каждом поиске
- Может вызывать тайм-ауты при работе с большими файлами
- **Должна быть включена только для цифровых магазинов с индексируемыми документами**
- **Никогда не включайте её без необходимости** - тщательно протестируйте влияние на производительность

## Вкладка Нечёткое совпадение

![Settings for Fuzzy Matching](/static/core/admin/img/help/search-settings-overview/search-settings-fuzzy-matching.webp)

Нечёткое совпадение использует расстояние Левенштейна для обработки опечаток:

**Включить нечёткое совпадение** - Позволяет поиску совпадать с похожими терминами (например, "laptop" совпадает с "labtop")

**Порог схожести** - По умолчанию: 0.80 (80% схожести). Диапазон: 0.0-1.0. Более высокие значения требуют более точных совпадений и работают быстрее. Низкие значения захватывают больше опечаток, но могут возвращать несущественные результаты.

**Максимальное расстояние редактирования** - По умолчанию: 2 изменения символов. Максимальное количество вставок, удалений или замен, разрешённых. Низкие значения (1) улучшают производительность, но захватывают меньше опечаток.

## Вкладка Веса

Веса управляют оценкой релевантности - как ранжируются результаты. Вкладка Веса показывает значения умножителей по умолчанию для каждого поискового поля:

- weight_name: 1.50 (название товара наиболее важно)
- weight_sku: 1.20
- weight_description: 0.80
- weight_categories: 0.80
- weight_attributes: 0.70
- weight_brands: 0.70
- weight_blog_posts: 0.60
- weight_reviews: 0.50

Эти значения по умолчанию хорошо работают для большинства интернет-магазинов. Для получения подробной информации о настройке весов и понимании их влияния см. тему [Веса релевантности и глубокое индексирование](/en/admin/help/relevance-weights-deep-indexing/).

## Вкладка Кэширование

![Settings for Caching](/static/core/admin/img/help/search-settings-overview/search-settings-caching.webp)

Кэширование значительно улучшает производительность поиска, сохраняя последние результаты:

**TTL кэша автодополнения** - По умолчанию: 60 секунд. Как долго результаты автодополнения кэшируются. Короткий TTL (30-45с) = более свежие результаты, но больше запросов к базе данных. Длинный TTL (90-120с) = более быстрый, но потенциально устаревший результат.

**TTL кэша результатов** - По умолчанию: 300 секунд (5 минут). Продолжительность кэша страницы полного результата поиска. Длинный TTL значительно улучшает производительность, но задерживает видимость новых товаров.

**Торговля**: Кэширование - это самый эффективный способ оптимизации производительности. Если поиски медленные, увеличьте эти значения перед отключением функций.

## Вкладка Аналитика

![Settings for Analytics](/static/core/admin/img/help/search-settings-overview/search-settings-analytics.webp)

**Отслеживать запросы поиска** - Включает панель мониторинга поиска. Записывает текст запроса, количество результатов, время отклика и временной штамп.

**Отслеживать информацию пользователя** - Связывает поиски с авторизованными пользователями. Отключите для соблюдения конфиденциальности (GDPR, CCPA).

**Отслеживать информацию сеанса** - Использует идентификаторы сеансов для отслеживания анонимных поисковых запросов. Полезно для выявления паттернов поиска без использования личных данных.

## Модель Singleton

SearchSettings использует паттерн Singleton - только одна запись настроек существует в вашей базе данных (pk=1). Когда вы навигируете к Настройкам поиска в админке, вы всегда редактируете ту же запись.

Нет опции "Добавить" или "Удалить" - только "Изменить". Все поисковые системы наследуют эти настройки, если только они не указывают переопределения на уровне поисковой системы (редко).

## Советы

- **Сохраняйте настройки по умолчанию, если у вас нет специфических потребностей** - Настройки по умолчанию оптимизированы для типичных интернет-магазинов
- **Никогда не включайте индексацию документов без необходимости** - Только для цифровых магазинов с индексируемыми документами, и сначала протестируйте влияние на производительность
- **Следите за временем отклика в аналитике** - Цель: <200мс для автодополнения, <500мс для полного поиска
- **Увеличьте TTL кэша, если производительность медленная** - Кэширование - это самый простой выигрыш в производительности
- **Рассмотрите запросы без результатов еженедельно** - Они раскрывают отсутствующие товары или необходимые синонимы
- **Отключите ненужные типы контента** - Если у вас нет блога, отключите Посты блога, чтобы ускорить поиски

Помните: Сохраните все форматирование markdown, пути к изображениям, блоки кода и технические термины в точности, как указано в правилах сохранения.
