---
title: Управление синонимами и перенаправлениями
---

Синонимы и перенаправления делают ваш поиск умнее, обрабатывая эквивалентные термины и направляя конкретные запросы на целевые страницы. Синонимы расширяют поиски, включая связанные термины («laptop» также находит «notebook»), в то время как перенаправления отправляют запросы вроде «sale» напрямую на вашу страницу с акциями. Это руководство объясняет, как создавать и управлять обоими функциями, чтобы улучшить релевантность поиска и опыт клиентов.

Используйте синонимы для эквивалентности терминов и перенаправления для коротких путей навигации.

![Список синонимов](/static/core/admin/img/help/managing-synonyms-redirects/synonym-list.webp)

## Понимание синонимов

Синонимы сообщают системе поиска, что определенные термины должны рассматриваться как эквивалентные. Когда клиент ищет один термин, система автоматически включает результаты, соответствующие синонимам.

**Пример**: Создайте сопоставление синонимов «laptop» → «notebook», «portable computer». Теперь, когда кто-то ищет «laptop», он также получает результаты для продуктов, содержащих «notebook» или «portable computer» в их названиях или описаниях.

Синонимы особенно ценны для:
- Британского против американского английского (jumper/sweater, trainers/sneakers)
- Брендовых против общих терминов (tissues/Kleenex)
- Распространенных опечаток (accommodate/accomodate)
- Специфической терминологии отрасли против простого языка (CPU/processor)

## Создание синонимов

Перейдите в **Search > Synonyms** и нажмите **+ Add Synonym**.

![Форма добавления синонима](/static/core/admin/img/help/managing-synonyms-redirects/synonym-form.webp)

**Term** - Исходный термин поиска, который запускает расширение синонимов

**Synonyms** - Массив эквивалентных терминов в формате JSON, например `['sweater', 'pullover', 'jumper']`

**Bidirectional** - По умолчанию: Отмечено. При включении, отношения синонимов работают в обе стороны:
- Поиск «laptop» находит продукты «notebook»
- Поиск «notebook» находит продукты «laptop»

Снимите галочку для односторонних сопоставлений (см. ниже).

**Language** - Необязательно. Ограничьте этот синоним поисками на определенном языке. Оставьте пустым, чтобы применить ко всем языкам.

**Engine** - Необязательно. Ограничьте этот синоним определенным движком поиска. Оставьте пустым, чтобы применить глобально.

**Active** - Используется ли этот синоним в настоящее время. Снимите галочку, чтобы временно отключить, не удаляя.

## Примеры двусторонних синонимов

Большинство синонимов должны быть двусторонними - истинными эквивалентами, которые работают в обе стороны:

| Term | Synonyms | Use Case |
|------|----------|----------|
| laptop | notebook, portable computer | American/British English + generic terms |
| sofa | couch, settee | Regional variations |
| trainers | sneakers, running shoes | UK/US English |
| mobile | cell phone, cellular | International variations |

С включенным двусторонним режимом все эти термины находят одни и те же продукты, независимо от того, какой термин использует клиент.

## Примеры односторонних синонимов

Снимите галочку «Bidirectional» для односторонних отношений:

**Часто используемые случаи**:
- **Опечатки**: Термин: «acco

mmodate» → Синонимы: `['accommodate']` (одностороннее, поэтому правильное написание не находит опечатку)
- **Специфичный → Общий**: Термин: «MacBook» → Синонимы: `['laptop']` (MacBooks - это ноутбуки, но не все ноутбуки - это MacBooks)
- **Сокращения**: Термин: «CPU» → Синонимы: `['processor']` (CPU находит продукты с процессором, но поиски по процессору не должны всегда включать CPU)

## Синонимы, специфичные для языка

Используйте поле Language, чтобы создать синонимы, соответствующие региону:

**Пример**: Магазин на британском английском
- Термин: «jumper», Синонимы: `['sweater', 'pullover']`, Язык: English (UK)
- Термин: «trainers», Синонимы: `['sneakers']`, Язык: English (UK)

**Пример**: Магазин с несколькими языками
- Термин: «ordinateur portable», Синонимы: `['laptop', 'notebook']`, Язык: French
- Термин: «zapatos», Синонимы: `['shoes']`, Язык: Spanish

Синонимы, специфичные для языка, применяются только тогда, когда клиент просматривает в этом языке.

## Синонимы, специфичные для движка

Большинство синонимов должны применяться глобально (оставьте поле Engine пустым). Используйте специфичные для движка синонимы только тогда, когда разным контекстам поиска нужны разные сопоставления терминов:

**Пример**: У вас есть отдельные движки «shop» и «blog»
- Синоним блога: Термин: «tutorial» → Синонимы: `['guide', 'how-to']`, Движок: blog
- Этот синоним применяется только к поиску в блоге, а не к поиску продуктов

## Понимание перенаправлений

Перенаправления поиска отправляют конкретные запросы напрямую на определенные страницы, обходя обычные результаты поиска. Используйте перенаправления, когда вы точно знаете, куда должен пойти клиент.

**Пример**: Создайте перенаправление для «sale» → `/products/sale/`. Теперь, когда кто-то ищет «sale», он пропускает результаты поиска и попадает напрямую на вашу страницу с акциями.

Перенаправления идеальны для:
- Общих коротких путей навигации («returns» → страница с политикой возврата)
- Сезонных акций («summer sale» → летняя коллекция)
- Популярных категорий («laptops» → страница категории ноутбуков)
- Страниц политик («shipping» → информация о доставке)

![Список перенаправлений](/static/core/admin/img/help/managing-synonyms-redirects/redirect-list.webp)

## Типы совпадений

Перенаправления поддерживают четыре типа совпадений, которые контролируют, насколько строго должен совпадать запрос поиска:

**Exact** - Точное совпадение, не зависящее от регистра. Запрос должен точно совпадать с термином (игнорируя заглавные буквы).
- Термин: «sale»
- Совпадения: «sale», «SALE», «Sale»
- Не совпадает: «summer sale», «on sale»

**Contains** - Запрос содержит термин в любом месте.
- Термин: «sizing»
- Совпадения: «sizing guide», «help with sizing», «what sizing»
- Не совпадает: «size chart» (разное слово)

**Starts With** - Запрос начинается с термина.
- Термин: «return»
- Совпадения: «returns», «return policy», «returning items»
- Не совпадает: «how to return» (не начинается с термина)

**Regex** - Совпадение по шаблону с использованием регулярных выражений. **⚠️ Осторожно с производительностью** - сложные регулярные выражения замедляют поиск. Используйте редко.
- Шаблон: `^(laptop|notebook)s?$`
- Совпадения: «laptop», «laptops», «notebook», «notebooks»
- Используйте только если другие типы совпадений не работают

## Создание перенаправлений

Перейдите в **Search > Redirects** и нажмите **+ Add Redirect**.

![Форма добавления перенаправления](/static/core/admin/img/help/managing-synonyms-redirects/redirect-form.webp)

**Term** - Запрос поиска, который нужно сопоставить

**Match Type** - Точный, содержит, начинается с, или Регулярное выражение (см. выше)

**Redirect URL** - Куда отправить клиента. Может быть относительным (`/products/sale/`) или абсолютным (`https://example.com/page/`)

**Redirect Type** - Код состояния HTTP:
- **302 (Временное)**: Рекомендуется. Браузер не кэширует, вы можете изменить назначение позже
- **301 (Постоянное)**: Браузер и поисковые системы кэшируют. Используйте только для постоянных перенаправлений

**Engine** - Необязательно. Ограничьте определенным движком поиска

**Hit Count** - Автоматически увеличивается каждый раз, когда это перенаправление используется. Помогает определить популярные сокращения.

**Active** - Включить/отключить это перенаправление

## Примеры перенаправлений

| Term | Match Type | URL | Use Case |
|------|-----------|-----|----------|
| sale | Exact | `/products/sale/` | Прямой поиск «sale» на страницу акций |
| clearance | Exact | `/clearance/` | Пропустить поиск по акционным товарам |
| sizing | Contains | `/pages/size-guide/` | Любой запрос о размерах переходит к руководству |
| return | Starts With | `/pages/returns/` | Запросы, связанные с возвратом, переходят к политике |

Все используют 302 (временные) перенаправления для гибкости.

## Тип перенаправления: 302 против 301

**302 (Временное)** - Рекомендуется для большинства перенаправлений
- Браузер делает свежий запрос каждый раз
- Вы можете изменить URL назначения в любое время
- Безопасный выбор, если вы не уверены

**301 (Постоянное)** - Используйте редко
- Браузер кэширует перенаправление
- Поисковые системы обновляют свои индексы
- Сложнее изменить позже

**Рекомендация**: Используйте 302, если вы абсолютно уверены, что перенаправление никогда не изменится.

## Анализ количества попаданий

Поле Hit Count автоматически увеличивается каждый раз, когда срабатывает перенаправление. Используйте это, чтобы:
- Определить самые часто используемые сокращения навигации
- Найти перенаправления, которые никогда не используются (рассмотрите их удаление)
- Открыть популярные паттерны поиска

Проверяйте количества попаданий ежемесячно, чтобы оптимизировать стратегию перенаправления.

## Поиск возможностей синонимов

**Используйте запросы без результатов**: Перейдите в **Search > Search Analytics** и отфильтруйте запросы без результатов. Это раскрывает:
- Термины, которые используют клиенты, но не совпадают с описаниями ваших продуктов
- Региональные вариации, которые вы не рассматривали
- Распространенные опечатки

**Рабочий процесс**:
1. Рассмотрите запросы без результатов еженедельно
2. Определите паттерны (повторяющиеся термины)
3. Добавьте синонимы, чтобы сопоставить язык клиентов с названиями ваших продуктов
4. Отслеживайте, уменьшается ли количество запросов без результатов

## Советы

- **Рассматривайте запросы без результатов еженедельно для идей синонимов** - Они раскрывают разрывы между языком клиентов и описаниями ваших продуктов
- **Начните с распространенных синонимов, расширяйте на основе данных** - Начните с очевидных региональных вариаций, затем добавьте на основе реального поведения поиска
- **Используйте двусторонние синонимы для истинных эквивалентов** - Большинство синонимов должны работать в обе стороны (laptop ↔ notebook)
- **Избегайте сложных регулярных выражений** - Совпадение по регулярным выражениям медленнее, чем другие типы совпадений; используйте только когда это необходимо
- **Используйте временные перенаправления (302) по умолчанию** - Это дает вам гибкость для изменения целей позже
- **Тестируйте синонимы с реальными запросами** - Ищите синонимы, чтобы проверить, возвращаются ли ожидаемые результаты
- **Специфичные синонимы для магазинов с несколькими языками** - Создавайте сопоставления терминов, соответствующие региону, для каждого языка, который вы поддерживаете
