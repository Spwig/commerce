อินเทอร์เฟซ SearchSettings ควบคุมพฤติกรรมการค้นหาทั่วไปในร้านค้า Spwig ของคุณ หน้าตั้งค่าเดียวนี้ใช้โครงสร้าง 8 แท็บเพื่อจัดระเบียบตัวเลือกการค้นหา ตั้งแต่การเปิดใช้งานพื้นฐานไปจนถึงการปรับแต่งประสิทธิภาพขั้นสูง การเปลี่ยนแปลงที่นี่จะใช้กับเครื่องมือค้นหาทั้งหมด เว้นแต้จะมีการกำหนดค่าที่ระดับเครื่องมือค้นหา

คู่มือนี้จะอธิบายแต่ละแท็บ พร้อมทั้งอธิบายว่าแต่ละการตั้งค่ามีผลอย่างไร และเมื่อใดควรปรับแต่ง

![แท็บการตั้งค่าทั่วไป](/static/core/admin/img/help/search-settings-overview/search-settings-general.webp)

## อินเทอร์เฟซ 8 แท็บ

SearchSettings เป็นโมเดลแบบ singleton - มีเพียงหนึ่งรายการการตั้งค่าเท่านั้น (pk=1) สำหรับร้านค้าของคุณทั้งหมด อินเทอร์เฟซถูกแบ่งออกเป็นแปดแท็บ:

| แท็บ | วัตถุประสงค์ |
|-----|---------|
| **ทั่วไป** | เปิด/ปิดการค้นหา ตั้งค่าพารามิเตอร์พื้นฐาน |
| **การแนะนำ** | กำหนดพฤติกรรมของการแสดงผลลัพธ์การค้นหาแบบคาดการณ์ |
| **ประเภทเนื้อหา** | เลือกประเภทเนื้อหาที่สามารถค้นหาได้ |
| **การจัดทำดัชนีลึก** | ควบคุมข้อมูลสินค้าใดที่จะถูกจัดทำดัชนี (ผลกระทบต่อประสิทธิภาพ) |
| **การจับคู่แบบคลุมเครือ** | ความอดทนต่อการพิมพ์ผิดและเกณฑ์ความคล้ายคลึง |
| **น้ำหนัก** | ตัวคูณความเกี่ยวข้องสำหรับการจัดอันดับผลลัพธ์ |
| **การแคช** | การแลกเปลี่ยนระหว่างเวลาตอบสนองกับความสดใหม่ |
| **การวิเคราะห์** | การติดตามคำค้นหาและการตั้งค่าความเป็นส่วนตัว |

แต่ละแท็บมุ่งเน้นไปที่ด้านหนึ่งของการตั้งค่าการค้นหา

## แท็บทั่วไป

แท็บทั่วไปมีการตั้งค่าหลักที่มีผลต่อการค้นหาทั้งหมด:

**เปิดใช้งานการค้นหา** - ปุ่มเปิด/ปิดหลักสำหรับระบบการค้นหา เมื่อถูกปิดการใช้งาน คุณสมบัติการค้นหาทั้งหมดจะไม่ทำงานในร้านค้าของคุณ รวมถึงการแนะนำและการแสดงผลลัพธ์การค้นหา

**ความยาวคำค้นหาขั้นต่ำ** - ค่าเริ่มต้น: 2 ตัวอักษร การค้นหาที่สั้นกว่านี้จะถูกปฏิเสธ การตั้งค่าเป็น 1 จะอนุญาตให้ค้นหาด้วยตัวอักษรเดียว (เช่น "A") แต่เพิ่มภาระการทำงานของเซิร์ฟเวอร์

**จำนวนผลลัพธ์ต่อหน้า** - ค่าเริ่มต้น: 20 รายการ ควบคุมการแบ่งหน้าสำหรับหน้าผลลัพธ์การค้นหา ค่าที่สูงขึ้น (30-50) ลดการคลิกแบ่งหน้า แต่เพิ่มเวลาในการโหลดหน้า

## แท็บประเภทเนื้อหา

![การตั้งค่าประเภทเนื้อหา](/static/core/admin/img/help/search-settings-overview/search-settings-content-types.webp)

เปิด/ปิดประเภทเนื้อหาที่ปรากฏในผลลัพธ์การค้นหา:

- **สินค้า** - สินค้าทางกายภาพ ดิจิทัล และแบบสมัครสมาชิก
- **หมวดหมู่** - หมวดหมู่สินค้า
- **แบรนด์** - แบรนด์สินค้า
- **บทความบล็อก** - เนื้อหาบล็อก

**หมายเหตุเรื่องประสิทธิภาพ**: จำนวนประเภทเนื้อหาที่น้อยลง = การค้นหาที่เร็วขึ้น แต่ละประเภทที่เปิดใช้งานเพิ่มการสอบถามฐานข้อมูลเพิ่มเติม หากคุณไม่มีบล็อก ให้ปิดการใช้งานบทความบล็อกเพื่อปรับปรุงเวลาการตอบสนอง

## แท็บการจัดทำดัชนีลึก

⚠️ **คำเตือนเรื่องประสิทธิภาพ** - การตั้งค่าเหล่านี้มีผลกระทบต่อประสิทธิภาพอย่างมาก

![การตั้งค่าการจัดทำดัชนีลึก](/static/core/admin/img/help/search-settings-overview/search-settings-deep-indexing.webp)

การจัดทำดัชนีลึกควบคุมข้อมูลที่เกี่ยวข้องกับสินค้าใดที่จะถูกรวมในการค้นหา:

**จัดทำดัชนี SKU** - ค่าเริ่มต้น: เปิดใช้งาน ผลกระทบต่ำ รวม SKU ของสินค้าและตัวเลือกสินค้าในการค้นหา จำเป็นสำหรับร้านค้า B2B ที่ลูกค้าค้นหาด้วยรหัสสินค้า

**จัดทำดัชนีคุณสมบัติ** - ค่าเริ่มต้น: เปิดใช้งาน ผลกระทบปานกลาง รวมคุณสมบัติของสินค้า (สี ขนาด วัสดุ) ในการค้นหา เพิ่มการ JOIN ตารางคุณสมบัติ สำคัญสำหรับสินค้าแฟชั่นและสินค้าที่สามารถกำหนดค่าได้

**จัดทำดัชนีฟิลด์เฉพาะ** - ค่าเริ่มต้น: เปิดใช้งาน ผลกระทบปานกลาง รวมฟิลด์เฉพาะที่ผู้ขายกำหนดในการค้นหา จำเป็นต้องใช้ JSONField traversal

**จัดทำดัชนีรีวิว** - ค่าเริ่มต้น: เปิดใช้งาน ผลกระทบปานกลาง-สูง ⚠️

การจัดทำดัชนีเอกสารจะดึงข้อความจากไฟล์ PDF, DOCX และ XLSX ที่แนบกับสินค้าดิจิทัล คุณสมบัติ:

- ต้องใช้การจัดทำดัชนีเริ่มต้นที่มีค่าใช้จ่ายสูงมาก
- เพิ่มภาระการสอบถามอย่างมากในทุกการค้นหา
- อาจทำให้เกิดการหมดเวลาสำหรับไฟล์ขนาดใหญ่
- **ควรเปิดใช้งานเพียงร้านค้าสินค้าดิจิทัลที่มีเอกสารที่สามารถค้นหาได้เท่านั้น**
- **ห้ามเปิดใช้งานโดยไม่จำเป็น** - ทดสอบผลกระทบต่อประสิทธิภาพอย่างละเอียดก่อน

## แท็บการจับคู่แบบคลุมเครือ

![การตั้งค่าการจับคู่แบบคลุมเครือ](/static/core/admin/img/help/search-settings-overview/search-settings-fuzzy-matching.webp)

การจับคู่แบบคลุมเครือใช้ระยะห่างของ Levenshtein เพื่อจัดการกับการพิมพ์ผิด:

**เปิดใช้งานการจับคู่แบบคลุมเครือ** - อนุญาตให้การค้นหาตรงกับคำที่คล้ายกัน (เช่น "laptop" ตรงกับ "labtop")

**เกณฑ์ความคล้ายคลึง** - ค่าเริ่มต้น: 0.80 (80% ความคล้ายคลึง) ช่วง: 0.0-1.0 ค่าที่สูงขึ้นต้องการการตรงกันมากขึ้นและทำงานได้เร็วขึ้น ค่าที่ต่ำกว่าจะจับการพิมพ์ผิดได้มากขึ้น แต่อาจคืนผลลัพธ์ที่ไม่เกี่ยวข้อง

**ระยะการแก้ไขสูงสุด** - ค่าเริ่มต้น: 2 การเปลี่ยนตัวอักษร สูงสุดจำนวนการแทรก การลบ หรือการแทนที่ที่อนุญาต ค่าที่ต่ำกว่า (1) ปรับปรุงประสิทธิภาพ แต่จับการพิมพ์ผิดได้น้อยลง

## แท็บน้ำหนัก

น้ำหนักควบคุมคะแนนความเกี่ยวข้อง - วิธีการจัดอันดับผลลัพธ์ แท็บน้ำหนักแสดงตัวคูณค่าเริ่มต้นสำหรับแต่ละฟิลด์ที่สามารถค้นหาได้:

- weight_name: 1.50 (ชื่อสินค้ามีความสำคัญที่สุด)
- weight_sku: 1.20
- weight_description: 0.80
- weight_categories: 0.80
- weight_attributes: 0.70
- weight_brands: 0.70
- weight_blog_posts: 0.60
- weight_reviews: 0.50

ค่าเริ่มต้นเหล่านี้เหมาะกับร้านค้าอีคอมเมิร์ซส่วนใหญ่ สำหรับข้อมูลเพิ่มเติมเกี่ยวกับการปรับแต่งน้ำหนักและการเข้าใจผลกระทบของมัน ดูหัวข้อ [น้ำหนักความเกี่ยวข้องและการจัดทำดัชนีลึก](/en/admin/help/relevance-weights-deep-indexing/)

## แท็บการแคช

![การตั้งค่าการแคช](/static/core/admin/img/help/search-settings-overview/search-settings-caching.webp)

การแคชปรับปรุงประสิทธิภาพการค้นหาอย่างมากโดยการจัดเก็บผลลัพธ์ล่าสุด:

**ระยะเวลาการแคชการแนะนำ** - ค่าเริ่มต้น: 60 วินาที ระยะเวลาที่ผลลัพธ์การแนะนำถูกจัดเก็บ ระยะเวลาการแคชที่สั้นกว่า (30-45 วินาที) = ผลลัพธ์ที่สดใหม่กว่า แต่มีการสอบถามฐานข้อมูลมากขึ้น ระยะเวลาการแคชที่ยาวกว่า (90-120 วินาที) = ทำงานได้เร็วขึ้น แต่ผลลัพธ์อาจล้าสมัย

**ระยะเวลาการแคชผลลัพธ์** - ค่าเริ่มต้น: 300 วินาที (5 นาที) ระยะเวลาที่หน้าผลลัพธ์การค้นหาทั้งหมดถูกจัดเก็บ ระยะเวลาที่ยาวขึ้นจะปรับปรุงประสิทธิภาพอย่างมาก แต่จะทำให้การปรากฏของสินค้าใหม่ล่าช้าลง

**ข้อแลกเปลี่ยน**: การแคชคือการปรับปรุงประสิทธิภาพที่มีประสิทธิภาพที่สุด หากการค้นหาช้า ให้เพิ่มค่าเหล่านี้ก่อนที่จะปิดใช้งานคุณสมบัติ

## แท็บการวิเคราะห์

![การตั้งค่าการวิเคราะห์](/static/core/admin/img/help/search-settings-overview/search-settings-analytics.webp)

**ติดตามคำค้นหา** - เปิดใช้งานแดชบอร์ดการวิเคราะห์การค้นหา บันทึกข้อความคำค้นหา จำนวนผลลัพธ์ เวลาตอบสนอง และเวลาที่บันทึก

**ติดตามข้อมูลผู้ใช้** - ผูกคำค้นหาเข้ากับผู้ใช้ที่เข้าสู่ระบบ ปิดใช้งานเพื่อปฏิบัติตามข้อกำหนดความเป็นส่วนตัว (GDPR, CCPA)

**ติดตามข้อมูลเซสชัน** - ใช้ ID เซสชันเพื่อติดตามการค้นหาของผู้ใช้ที่ไม่ระบุตัวตน มีประโยชน์ในการระบุรูปแบบการค้นหาโดยไม่ต้องใช้ข้อมูลส่วนบุคคล

## รูปแบบ Singleton

SearchSettings ใช้รูปแบบ singleton - มีเพียงหนึ่งรายการการตั้งค่าเท่านั้นในฐานข้อมูลของคุณ (pk=1) เมื่อคุณเข้าไปที่การตั้งค่าการค้นหาในแดชบอร์ด คุณจะแก้ไขรายการเดียวกันเสมอ

ไม่มีตัวเลือก "เพิ่ม" หรือ "ลบ" - เพียง "แก้ไข" เท่านั้น ทั้งหมดเครื่องมือค้นหาจะสืบทอดการตั้งค่าเหล่านี้ เว้นแต้เครื่องมือค้นหาจะกำหนดการตั้งค่าเฉพาะตัวเอง (ไม่พบบ่อย)

## เคล็ดลับ

- **รักษาค่าเริ่มต้นไว้หากไม่มีความจำเป็นเฉพาะ** - ค่าเริ่มต้นได้รับการปรับแต่งสำหรับร้านค้าอีคอมเมิร์ซทั่วไป
- **ห้ามเปิดใช้งานการจัดทำดัชนีเอกสารโดยไม่จำเป็น** - เปิดใช้งานเฉพาะสำหรับร้านค้าสินค้าดิจิทัลที่มีเอกสารที่สามารถค้นหาได้ และทดสอบผลกระทบต่อประสิทธิภาพก่อน
- **ตรวจสอบเวลาตอบสนองในแดชบอร์ดการวิเคราะห์** - ตั้งเป้าหมายให้การแนะนำน้อยกว่า 200 มิลลิวินาที และการค้นหาทั้งหมดน้อยกว่า 500 มิลลิวินาที
- **เพิ่มระยะเวลาการแคชหากประสิทธิภาพช้า** - การแคชคือการปรับปรุงประสิทธิภาพที่ง่ายที่สุด
- **ตรวจสอบคำค้นหาที่ไม่พบผลลัพธ์รายสัปดาห์** - พวกเขาเปิดเผยสินค้าที่หายไปหรือคำสังเคราะห์ที่จำเป็น
- **ปิดใช้งานประเภทเนื้อหาที่ไม่ได้ใช้** - หากคุณไม่มีบล็อก ปิดการใช้งานบทความบล็อกเพื่อเร่งความเร็วการค้นหา

จำไว้ว่า: รักษาการจัดรูปแบบมาร์กดาวน์ ค่าเส้นทางภาพ บล็อกโค้ด และคำศัพท์ทางเทคนิคตามที่แสดงในกฎการรักษาไว้แน่นอน
