---
title: 검색 설정 이해
---

SearchSettings 인터페이스는 Spwig 상점에서의 모든 전역 검색 동작을 제어합니다. 이 단일 설정 페이지는 기본적인 활성화부터 고급 성능 조정까지 검색 옵션을 정리하기 위해 8개 탭 인터페이스를 사용합니다. 여기서의 변경 사항은 엔진 수준에서 오버라이드되지 않는 한 모든 검색 엔진에 적용됩니다.

이 가이드는 각 탭을 순회하며 각 설정이 무엇을 하고 언제 조정해야 하는지 설명합니다.

![Search Settings General Tab](/static/core/admin/img/help/search-settings-overview/search-settings-general.webp)

## 8-Tab 인터페이스

SearchSettings는 싱글톤 모델입니다 - 전체 상점에 대해 하나의 설정 기록만 존재합니다 (pk=1). 인터페이스는 8개 탭으로 나뉩니다:

| 탭 | 목적 |
|-----|---------|
| **일반** | 검색 활성화/비활성화, 기본 파라미터 설정 |
| **자동 완성** | 예측 검색 드롭다운 동작 구성 |
| **콘텐츠 유형** | 검색 가능한 콘텐츠 유형 선택 |
| **심층 인덱싱** | 제품 데이터 중 어떤 것이 인덱싱되는지 제어 (성능 영향) |
| **부정확한 일치** | 오타 허용 범위 및 유사성 임계값 |
| **가중치** | 결과 순위에 대한 관련성 곱셈 계수 |
| **캐싱** | 응답 시간 대신 신선도의 트레이드오프 |
| **분석** | 쿼리 추적 및 개인정보 설정 |

각 탭은 검색 설정의 특정 측면에 집중합니다.

## 일반 탭

일반 탭에는 모든 검색에 영향을 주는 핵심 설정이 포함되어 있습니다:

**검색 활성화** - 검색 시스템의 마스터 스위치입니다. 비활성화되면 상점 전체에서 검색 기능이 비활성화되며, 자동 완성 및 검색 결과 페이지도 포함됩니다.

**최소 쿼리 길이** - 기본값: 2자. 이보다 짧은 쿼리는 거부됩니다. 이 값을 1로 설정하면 단일 문자 검색(예: "A")이 가능하지만 서버 부하가 증가합니다.

**페이지당 결과 수** - 기본값: 20개 항목. 검색 결과 페이지의 페이징을 제어합니다. 더 높은 값(30-50)은 페이징 클릭을 줄이지만 페이지 로드 시간을 증가시킵니다.

## 콘텐츠 유형 탭

![콘텐츠 유형 설정](/static/core/admin/img/help/search-settings-overview/search-settings-content-types.webp)

검색 결과에 나타나는 콘텐츠 유형을 토글합니다:

- **제품** - 물리적, 디지털 및 구독 제품
- **카테고리** - 제품 카테고리
- **브랜드** - 제품 브랜드
- **블로그 게시물** - 블로그 콘텐츠

**성능 참고 사항**: 콘텐츠 유형이 적을수록 검색이 빠릅니다. 각 활성화된 유형은 추가 데이터베이스 쿼리를 추가합니다. 블로그가 없으면 블로그 게시물을 비활성화하여 응답 시간을 개선하세요.

## 심층 인덱싱 탭

⚠️ **성능 경고** - 이러한 설정은 성능에 큰 영향을 미칩니다.

![심층 인덱싱 설정](/static/core/admin/img/help/search-settings-overview/search-settings-deep-indexing.webp)

심층 인덱싱은 제품 관련 데이터 중 어떤 것이 검색에 포함되는지 제어합니다:

**SKU 인덱싱** - 기본값: ON, 낮은 영향. 제품 및 변형 SKU를 검색에 포함합니다. 고객이 제품 코드로 검색하는 B2B 상점에 필수적입니다.

**속성 인덱싱** - 기본값: ON, 중간 영향. 제품 속성(색상, 크기, 재질)을 검색에 포함합니다. 속성 테이블에 JOIN을 추가합니다. 패션 및 구성 가능한 제품에 중요합니다.

**사용자 정의 필드 인덱싱** - 기본값: ON, 중간 영향. 판매자 정의 사용자 정의 필드를 검색 결과에 포함합니다. JSONField 탐색이 필요합니다.

**리뷰 인덱싱** - 기본값: ON, 중간-높은 영향. 승인된 리뷰 제목 및 주석을 검색에 포함합니다. 리뷰 테이블에 JOIN을 추가하고 텍스트 검색 오버헤드를 추가합니다. 리뷰 중심 카탈로그에 유용합니다.

**문서 인덱싱** - 기본값: OFF, **매우 높은 영향** ⚠️

문서 인덱싱은 디지털 제품에 첨부된 PDF, DOCX 및 XLSX 파일에서 텍스트를 추출합니다. 이 기능은 다음과 같은 특징을 가지고 있습니다:

- 매우 비싼 초기 인덱싱이 필요합니다
- 모든 검색에서 쿼리 오버헤드를 크게 증가시킵니다
- 대량의 파일에서 타임아웃을 유발할 수 있습니다
- **디지털 제품 상점에서 검색 가능한 문서가 있는 경우에만 활성화해야 합니다**
- **경고: 무심코 활성화하지 마세요** - 성능 영향을 철저히 테스트하세요

## 부정확한 일치 탭

![부정확한 일치 설정](/static/core/admin/img/help/search-settings-overview/search-settings-fuzzy-matching.webp)

부정확한 일치는 라벤슈타인 거리 알고리즘을 사용하여 오타를 처리합니다:

**부정확한 일치 활성화** - 유사한 용어(예: "laptop"이 "labtop"과 일치)로 검색을 허용합니다.

**유사성 임계값** - 기본값: 0.80 (80% 유사성). 범위: 0.0-1.0. 더 높은 값은 더 가까운 일치를 요구하고 실행 속도가 빠릅니다. 더 낮은 값은 더 많은 오타를 포착하지만 관련 없는 결과도 포함할 수 있습니다.

**최대 편집 거리** - 기본값: 2자 변경. 허용되는 삽입, 삭제 또는 대체의 최대 수. 더 낮은 값(1)은 성능을 향상시키지만 더 적은 오타를 포착합니다.

## 가중치 탭

가중치는 관련성 점수를 제어합니다 - 결과가 어떻게 순위를 매기는지. 가중치 탭은 각 검색 가능한 필드에 대한 기본 곱셈 계수를 보여줍니다:

- weight_name: 1.50 (제품 이름이 가장 중요함)
- weight_sku: 1.20
- weight_description: 0.80
- weight_categories: 0.80
- weight_attributes: 0.70
- weight_brands: 0.70
- weight_blog_posts: 0.60
- weight_reviews: 0.50

이 기본값은 대부분의 전자상거래 상점에 잘 작동합니다. 가중치 조정 및 그 영향에 대한 자세한 정보는 [관련성 가중치 및 심층 인덱싱](/en/admin/help/relevance-weights-deep-indexing/) 주제를 참조하세요.

## 캐싱 탭

![캐싱 설정](/static/core/admin/img/help/search-settings-overview/search-settings-caching.webp)

캐싱은 최근 결과를 저장함으로써 검색 성능을 크게 향상시킵니다:

**자동 완성 캐시 TTL** - 기본값: 60초. 자동 완성 결과가 캐시되는 시간. 더 짧은 TTL(30-45초)은 더 신선한 결과를 제공하지만 더 많은 데이터베이스 쿼리를 유발합니다. 더 긴 TTL(90-120초)은 더 빠르지만 잠재적으로 오래된 결과를 제공합니다.

**결과 캐시 TTL** - 기본값: 300초(5분). 전체 검색 결과 페이지의 캐시 기간. 더 긴 TTL은 성능을 크게 향상시키지만 새로운 제품의 가시성을 지연시킵니다.

**트레이드오프**: 캐싱은 성능 최적화에 가장 효과적인 방법입니다. 검색이 느릴 경우, 기능을 비활성화하기 전에 이러한 값을 증가시켜 보세요.

## 분석 탭

![분석 설정](/static/core/admin/img/help/search-settings-overview/search-settings-analytics.webp)

**검색 쿼리 추적** - 검색 분석 대시보드를 활성화합니다. 쿼리 텍스트, 결과 수, 응답 시간 및 타임스탬프를 기록합니다.

**사용자 정보 추적** - 로그인한 사용자와의 검색을 연결합니다. 개인정보 보호 규정(GDPR, CCPA) 준수를 위해 비활성화하세요.

**세션 정보 추적** - 세션 ID를 사용하여 익명 사용자의 검색을 추적합니다. 개인 정보 없이 검색 패턴을 식별하는 데 유용합니다.

## 싱글톤 패턴

SearchSettings는 싱글톤 패턴을 사용합니다 - 데이터베이스에 검색 설정 기록이 하나만 존재합니다 (pk=1). 관리자에서 검색 설정으로 이동하면 항상 동일한 기록을 편집하고 있습니다.

"추가" 또는 "삭제" 옵션이 없습니다 - 오직 "변경"만 있습니다. 모든 검색 엔진은 이 설정을 상속하지만, 엔진별로 오버라이드를 지정할 수 있습니다(드물게).

## 팁

- **특정 필요가 없는 한 기본값을 유지하세요** - 기본 설정은 일반적인 전자상거래 상점에 최적화되어 있습니다
- **문서 인덱싱을 무심코 활성화하지 마세요** - 디지털 제품 상점에서 검색 가능한 문서가 있는 경우에만 활성화하고, 먼저 성능 영향을 테스트하세요
- **분석에서 응답 시간을 모니터링하세요** - 자동 완성은 <200ms, 전체 검색은 <500ms를 목표로 하세요
- **성능이 느릴 경우 캐시 TTL을 증가시키세요** - 캐싱은 가장 쉬운 성능 향상 방법입니다
- **매주 zero-result 쿼리를 검토하세요** - 이는 누락된 제품 또는 필요한 동의어를 드러냅니다
- **사용되지 않는 콘텐츠 유형을 비활성화하세요** - 블로그가 없으면 블로그 게시물을 비활성화하여 검색 속도를 높이세요

기억하세요: 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어는 보존 규칙에 따라 정확히 유지하세요.
