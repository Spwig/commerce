เครื่องมือค้นหาใน Spwig ไม่ใช่บริการภายนอกเช่น Elasticsearch หรือ Algolia - มันคือบริบทการตั้งค่าภายในระบบค้นหาของฐานข้อมูลร้านค้าของคุณเอง แต่ละเครื่องมือกำหนดว่าจะค้นหาเนื้อหาประเภทใด อะไรที่ควรจะไม่รวม และผลลัพธ์ควรจะถูกจัดอันดับอย่างไร คู่มือนี้อธิบายว่าเครื่องมือค้นหาคืออะไร ควรสร้างเครื่องมือหลายเครื่องมือเมื่อใด และวิธีการตั้งค่ามัน

ผู้ขายส่วนใหญ่ใช้เครื่องมือค้นหาหนึ่งเครื่องมือคือ "shop" สร้างเครื่องมือหลายเครื่องมือก็ต่อเมื่อคุณต้องการเนื้อหาที่แตกต่างกันหรือการตัดสินใจที่แตกต่างกันสำหรับกรณีการใช้งานที่แตกต่างกัน

![รายการเครื่องมือค้นหา](/static/core/admin/img/help/search-engines-explained/search-engines-list.webp)

## เครื่องมือค้นหาคืออะไร?

เครื่องมือค้นหาใน Spwig คือการตั้งค่าที่มีชื่อระบุว่า:

- **ประเภทเนื้อหาที่จะค้นหา** (สินค้า หมวดหมู่ แบรนด์ โพสต์บล็อก)
- **สิ่งที่ควรจะไม่รวม** (หมวดหมู่หรือแบรนด์เฉพาะที่คุณต้องการซ่อนจากการค้นหา)
- **น้ำหนักความเกี่ยวข้องแบบกำหนดเอง** (การปรับน้ำหนักแบบต่อเครื่องมือ ตัวเลือก)
- **สถานะการใช้งาน** (เครื่องมือสามารถถูกปิดใช้งานชั่วคราวได้)

แต่ละเครื่องมือมี slug ที่เป็นเอกลักษณ์ที่ใช้ใน API และโค้ดหน้าแรกเพื่อกำหนดว่าเครื่องมือใดควรจะจัดการคำขอค้นหา

## เมื่อใดควรสร้างเครื่องมือหลายเครื่องมือ

ร้านค้าส่วนใหญ่ต้องการเครื่องมือเพียงเครื่องเดียว สร้างเครื่องมือเพิ่มเติมสำหรับสถานการณ์ต่อไปนี้:

| กรณีการใช้งาน | ตัวอย่าง |
|----------------|---------|
| **เนื้อหาที่แตกต่างกัน** | เครื่องมือค้นหาในร้านค้าค้นหาสินค้าเพียงอย่างเดียว; เครื่องมือค้นหาบล็อกค้นหาโพสต์บล็อกเพียงอย่างเดียว |
| **การตัดสินใจที่เลือกได้** | เครื่องมือค้นหาหลักซ่อนหมวดหมู่สินค้าลดราคา; เครื่องมือค้นหาสินค้าลดราคาแสดงเฉพาะสินค้าลดราคา |
| **การค้นหาเฉพาะแผนก** | เครื่องมือค้นหาอิเล็กทรอนิกส์ไม่รวมหมวดหมู่เสื้อผ้า; เครื่องมือค้นหาเสื้อผ้าไม่รวมอิเล็กทรอนิกส์ |
| **การแยก B2B และ B2C** | เครื่องมือค้นหาส่วนรวมแสดงเฉพาะสินค้าจำนวนมาก; เครื่องมือค้นหาค้าปลีกแสดงเฉพาะสินค้าสำหรับผู้บริโภค |

หากคุณไม่แน่ใจว่าต้องการเครื่องมือหลายเครื่องมือหรือไม่ ให้ใช้เครื่องมือเพียงเครื่องเดียว การเพิ่มเครื่องมือจะสร้างความซับซ้อนโดยไม่มีประโยชน์ เว้นแต่คุณจะมีกรณีการใช้งานเฉพาะ

## วิザาร์ด 4 ขั้นตอน

![ขั้นตอนที่ 1 ของวิザาร์ด - ข้อมูลพื้นฐาน](/static/core/admin/img/help/search-engines-explained/wizard-step1-basic.webp)

ไปที่ **Search > Setup Wizard** เพื่อสร้างเครื่องมือใหม่ผ่านกระบวนการที่มีการนำทาง 4 ขั้นตอน:

### ขั้นตอนที่ 1: ข้อมูลพื้นฐาน

**ชื่อเครื่องมือ** - ชื่อแสดงที่เป็นมิตร (เช่น "Shop Search", "Blog Search"). ใช้ในอินเทอร์เฟซผู้ดูแลระบบเท่านั้น

**Slug** - ตัวระบุที่ปลอดภัยสำหรับ URL (เช่น "shop-search", "blog-search"). ใช้ใน API และโค้ดหน้าแรก ถ้าปล่อยว่างจะสร้างอัตโนมัติจากชื่อ

**Active** - ระบุว่าเครื่องมือนี้พร้อมใช้งานสำหรับการค้นหา ถ้าเครื่องมือไม่ได้ใช้งาน จะไม่คืนผลลัพธ์ใดๆ

### ขั้นตอนที่ 2: ประเภทเนื้อหา

เลือกประเภทเนื้อหาที่เครื่องมือนี้จะค้นหา:

- สินค้า (รวมทุกประเภทสินค้า: สินค้าทางกายภาพ สินค้าดิจิทัล การสมัครสมาชิก)
- หมวดหมู่
- แบรนด์
- โพสต์บล็อก

**เคล็ดลับ**: เลือกเฉพาะประเภทเนื้อหาที่เกี่ยวข้องกับวัตถุประสงค์ของเครื่องมือนี้ เครื่องมือที่เน้นบล็อกไม่จำเป็นต้องเปิดใช้งานสินค้า

### ขั้นตอนที่ 3: น้ำหนัก (ตัวเลือก)

![ขั้นตอนที่ 3 ของวิザาร์ด - น้ำหนัก](/static/core/admin/img/help/search-engines-explained/wizard-step3-weights.webp)

ปรับแต่งน้ำหนักความเกี่ยวข้องสำหรับเครื่องมือนี้ได้ตามต้องการ หากข้ามขั้นตอนนี้ เครื่องมือจะสืบทอดน้ำหนักทั่วไปจาก SearchSettings

เครื่องมือส่วนใหญ่ควรข้ามขั้นตอนนี้และใช้น้ำหนักทั่วไป ปรับแต่งน้ำหนักเฉพาะก็ต่อเมื่อเครื่องมือนี้มีความต้องการการจัดอันดับที่แตกต่างกัน (เช่น เครื่องมือบล็อกอาจเพิ่มน้ำหนัก weight_blog_posts เป็น 1.2)

### ขั้นตอนที่ 4: ตรวจสอบและสร้าง

ตรวจสอบการตั้งค่าของคุณและคลิก **Create Engine** เพื่อบันทึก

**ชื่อและ Slug** - ชื่อแสดงและตัวระบุ URL

**สถานะการใช้งาน** - ปิด/เปิดใช้งาน
