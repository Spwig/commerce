---
title: Объяснение поисковых систем
---

Поисковые системы в Spwig не являются внешними сервисами, такими как Elasticsearch или Algolia - это контексты настройки в вашей собственной системе поиска в базе данных магазина. Каждая система определяет, какие типы контента искать, что исключить, и как ранжировать результаты. Это руководство объясняет, что такое поисковые системы, когда создавать несколько систем и как их настраивать.

Большинство продавцов используют одну стандартную систему поиска "shop". Создавайте несколько систем только в том случае, если вам нужны разные комбинации контента или исключения для разных сценариев использования.

![Список поисковых систем](/static/core/admin/img/help/search-engines-explained/search-engines-list.webp)

## Что такое поисковые системы?

Поисковая система в Spwig - это настроенная конфигурация с именем, которая указывает:

- **Какие типы контента искать** (товары, категории, бренды, блог-посты)
- **Что исключить** (конкретные категории или бренды, которые вы хотите скрыть от поиска)
- **Пользовательские веса релевантности** (опциональные перекрытия веса для каждой системы)
- **Активный статус** (системы могут быть временно отключены)

Каждая система имеет уникальный slug, который используется в API-запросах и клиентском коде для указания, какая система должна обрабатывать запрос поиска.

## Когда создавать несколько систем

Большинству магазинов требуется только одна система. Создавайте дополнительные системы для следующих сценариев:

| Сценарий использования | Пример |
|----------------------|--------|
| **Разные комбинации контента** | Система поиска магазина ищет только товары; Система поиска блога ищет только блог-посты |
| **Выборочное исключение** | Основная система магазина скрывает категорию распродаж; Система распродаж показывает только товары распродаж |
| **Поиск по отделам** | Система электроники исключает категории одежды; Система одежды исключает электронику |
| **Разделение B2B и B2C** | Система оптовой торговли показывает только товары оптовой продажи; Система розничной торговли показывает товары для конечных потребителей |

Если вы не уверены, нужны ли вам несколько систем, используйте одну. Добавление систем создает сложность без преимуществ, если у вас нет конкретного сценария использования.

## Мастер из 4 шагов

![Шаг 1 мастера - Основная информация](/static/core/admin/img/help/search-engines-explained/wizard-step1-basic.webp)

Перейдите в **Поиск > Мастер настройки** для создания новой системы через пошаговый процесс из 4 шагов:

### Шаг 1: Основная информация

**Название системы** - дружественное отображаемое имя (например, "Поиск магазина", "Поиск блога"). Используется только в интерфейсе администратора.

**Slug** - идентификатор, безопасный для URL (например, "shop-search", "blog-search"). Используется в API-запросах и клиентском коде. Если оставить пустым, он будет автоматически сгенерирован из названия.

**Активный** - определяет, доступна ли эта система для поиска. Неактивные системы не возвращают результаты.

### Шаг 2: Типы контента

Выберите, какие типы контента будет искать эта система:

- Товары (включает все типы товаров: физические, цифровые, подписки)
- Категории
- Бренды
- Блог-посты

**Совет**: выбирайте только типы контента, которые соответствуют цели этой системы. Система поиска блога не нуждается в включенных товарах.

### Шаг 3: Веса (опционально)

![Шаг 3 мастера - Веса](/static/core/admin/img/help/search-engines-explained/wizard-step3-weights.webp)

Опционально настройте веса релевантности для этой конкретной системы. Если пропустить, система наследует глобальные веса из SearchSettings.

Большинству систем следует пропустить этот шаг и использовать глобальные значения по умолчанию. Настройте веса только в том случае, если у этой системы есть уникальные требования к ранжированию (например, система поиска блога может увеличить weight_blog_posts до 1.2).

### Шаг 4: Проверка и создание

Проверьте свою конфигурацию и нажмите **Создать систему** для сохранения.
