---
title: SSL 설정
---

SSL (Secure Sockets Layer)은 고객의 브라우저와 가게 사이의 연결을 암호화합니다. SSL이 활성화되면 가게의 URL은 `https://`로 시작하고 브라우저는 잠금 아이콘을 표시합니다. 결제 수락, 고객 데이터 보호 및 검색 엔진에서의 높은 순위를 위해 SSL은 필수입니다.

Spwig은 다양한 호스팅 환경에 맞게 여러 SSL 모드를 지원합니다. 이 가이드는 각 모드를 설명하고 적절한 모드를 선택하는 방법을 도와줍니다.

## SSL 모드 선택

| 모드 | 최적 사용 | 인증서 비용 | 갱신 |
|------|----------|-----------------|---------|
| **Let's Encrypt** | 대부분의 가게 | 무료 | 자동 |
| **Cloudflare Origin CA** | Cloudflare 프록시를 사용하는 가게 | 무료 | 수동 (최대 15년) |
| **사용자 정의 인증서** | 구매한 인증서를 사용하는 가게 | 다름 | 수동 |
| **외부에서 관리** | 로드 밸런서, Cloudflare 유연 | N/A | N/A |
| **자체 서명** | 개발 및 테스트 | 무료 | 수동 |
| **없음 (HTTP)** | 로컬 개발만 | N/A | N/A |

모든 모드 중 어떤 것을 사용해야 할지 확신이 없으면, **Let's Encrypt**은 대부분의 가게에 최적입니다. 무료이며 자동 갱신되며 모든 브라우저에서 신뢰받습니다.

## Let's Encrypt

Let's Encrypt는 60-90일마다 자동으로 갱신되는 무료이고 신뢰받는 SSL 인증서를 제공합니다. 이는 대부분의 상인에게 권장되는 옵션입니다.

**요구사항:**
- 도메인은 서버로 가리켜야 합니다 (DNS의 A 레코드)
- 인터넷에서 포트 80에 접근할 수 있어야 합니다 (인증서 확인을 위해)
- 인증서 만료 알림을 위한 이메일 주소

**설정 단계:**
1. **설정 > 사이트 설정**으로 이동하고 **도메인 및 SSL** 탭을 열어주세요
2. 도메인 이름을 입력하세요
3. **Let's Encrypt**을 선택하세요
4. 관리자 이메일 주소를 입력하세요
5. **설정 적용**을 클릭하세요

Spwig은 나머지 모든 작업을 자동으로 처리합니다: 도메인 확인, 인증서 획득, NGINX 구성, 자동 갱신 설정.

## Cloudflare Origin CA

Cloudflare Origin CA 인증서는 Cloudflare의 엣지 서버와 가게 사이의 연결을 암호화합니다. 이러한 인증서는 무료이며 최대 15년까지 유효하지만, **Cloudflare만 신뢰**합니다 - 서버에 직접 연결하는 브라우저는 인증서 경고를 표시합니다.

이 모드는 도메인에 대해 Cloudflare 프록시(주황색 구름 활성화)를 사용하는 경우에 이상적입니다. Cloudflare은 방문자에게 자체 신뢰 인증서를 제공하고, Origin CA 인증서는 Cloudflare과 서버 사이의 연결을 보호합니다.

**요구사항:**
- 도메인을 추가한 Cloudflare 계정
- Cloudflare 대시보드에서 생성한 Origin CA 인증서 및 개인 키
- Cloudflare SSL/TLS 모드를 **Full (Strict)**로 설정

**Origin CA 인증서 생성:**
1. Cloudflare 대시보드에 로그인하세요
2. 도메인을 선택하세요
3. **SSL/TLS > Origin Server**로 이동하세요
4. **인증서 생성**을 클릭하세요
5. RSA 또는 ECC를 선택하세요 (RSA는 가장 호환성이 좋음)
6. 도메인을 추가하세요 (예: `example.com` 및 `*.example.com`)
7. 유효 기간을 선택하세요 (15년이 권장됨)
8. **생성**을 클릭하고 인증서와 개인 키를 복사하세요

**Spwig에서 설정:**
1. **설정 > 사이트 설정**으로 이동하고 **도메인 및 SSL** 탭을 열어주세요
2. 도메인 이름을 입력하세요
3. **Cloudflare Origin CA**를 선택하세요
4. **인증서 (PEM)** 필드에 인증서를 붙여넣으세요
5. **개인 키 (PEM)** 필드에 개인 키를 붙여넣으세요
6. **설정 적용**을 클릭하세요

**설정 후:**
- Cloudflare에서 SSL/TLS 모드를 **Full (Strict)**로 설정하세요
- 도메인의 DNS 레코드에 대해 Cloudflare 프록시(주황색 구름)를 활성화하세요
- Cloudflare의 신뢰 인증서를 통해 가게는 HTTPS로 접근 가능하게 됩니다

## 사용자 정의 인증서

도메인에 대한 인증서를 DigiCert, Sectigo 또는 GoDaddy와 같은 인증 기관(CA)에서 구입했거나 호스팅 제공업체가 제공한 경우 이 모드를 사용하세요.

**설정 단계:**
1.

**설정 > 사이트 설정**으로 이동하고 **도메인 및 SSL** 탭을 열어주세요
2.

도메인 이름을 입력하세요
3.

**사용자 정의 인증서**를 선택하세요
4.

모든 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어를 유지하세요

인증서 체인(중간 인증서 포함)을 **인증서 (PEM)** 필드에 붙여넣으세요
5.

자신의 개인 키를 **개인 키 (PEM)** 필드에 붙여넣으세요
6.

**설정 적용**을 클릭하세요

인증서는 전체 체인을 포함해야 합니다: 도메인 인증서 다음에 중간 인증서가 포함되어야 합니다. 개인 키는 PEM 형식이어야 하며 (`-----BEGIN PRIVATE KEY-----` 또는 `-----BEGIN RSA PRIVATE KEY-----`로 시작해야 합니다).

## 외부에서 관리

SSL이 서버에 도달하기 전에 외부 서비스에 의해 종료될 때 이 모드를 선택하세요. 이 설정에서는 서버가 일반 HTTP 트래픽만 수신하게 되며, 서버 자체에 인증서가 설치되어 있지 않습니다.

**일반적인 시나리오:**
- **Cloudflare Flexible SSL** -- Cloudflare는 브라우저에서 Cloudflare로의 트래픽을 암호화하지만, 서버로 HTTP를 전달합니다
- **클라우드 로드 밸런서** -- AWS ALB, Google Cloud Load Balancer 또는 DigitalOcean Load Balancer가 SSL을 종료하고 HTTP를 전달합니다
- **역방향 프록시** -- Spwig 앞에 있는 다른 서버가 SSL을 처리합니다

**설정 단계:**
1. **설정 > 사이트 설정**으로 이동하고 **도메인 및 SSL** 탭을 열어주세요
2. 도메인 이름을 입력하세요
3. **외부에서 관리**를 선택하세요
4. **설정 적용**을 클릭하세요

Spwig은 NGINX를 구성하여 HTTP만 제공하고 프록시에서 전달된 `X-Forwarded-Proto` 헤더를 신뢰하여 HTTPS 방문자를 올바르게 감지합니다.

## 자체 서명 인증서

자체 서명 인증서는 연결을 암호화하지만 브라우저에 의해 신뢰되지 않습니다. 방문자는 수동으로 우회해야 하는 보안 경고를 보게 됩니다. 이 모드는 개발 서버 및 내부 테스트에만 적합합니다.

**설정 단계:**
1. **설정 > 사이트 설정**으로 이동하고 **도메인 및 SSL** 탭을 열어주세요
2. 도메인 이름을 입력하세요
3. **자체 서명**을 선택하세요
4. **설정 적용**을 클릭하세요

Spwig은 자동으로 자체 서명 인증서를 생성합니다. 이 모드는 프로덕션 스토어에 사용하지 마세요.

## 문제 해결

**설정 후 인증서가 작동하지 않는 경우:**
- 도메인의 A 레코드가 서버의 IP 주소를 가리키는지 확인하세요
- 방화벽에서 포트 80과 443이 열려 있는지 확인하세요
- DNS 변경이 전파되기를 위해 몇 분 기다려보세요

**Let's Encrypt 인증서 발급에 실패한 경우:**
- 도메인이 이 서버의 IP 주소로 해석되는지 확인하세요
- 방화벽에서 포트 80이 차단되지 않았는지 확인하세요
- Cloudflare 뒤에 있다면 인증서 발급 중 DNS를 "DNS 전용" (회색 구름)으로 일시적으로 설정하세요

**Cloudflare에서 "Error 526" (잘못된 SSL 인증서)이 나타나는 경우:**
- **Cloudflare Origin CA** 모드를 선택했는지 확인하세요 (Managed Externally가 아닌)
- Cloudflare SSL/TLS 모드가 **Full (Strict)**로 설정되어 있는지 확인하세요
- Origin CA 인증서가 만료되지 않았는지 확인하세요

**SSL을 사용했음에도 불구하고 브라우저에서 "Not Secure"가 표시되는 경우:**
- 일부 페이지는 HTTP를 통해 이미지나 스크립트를 로드할 수 있습니다 (혼합 콘텐츠). 브라우저의 개발자 콘솔에서 혼합 콘텐츠 경고를 확인하세요.
- 설정에서 사이트 URL이 `https://`를 사용하는지 확인하세요
