---
title: 'Настройка SSO: Microsoft Entra ID'
---

Этот руководство поможет вам подключить Spwig к Microsoft Entra ID (ранее Azure Active Directory) для однократного входа администратора. После настройки ваш персонал сможет войти в панель администратора Spwig, используя свой рабочий аккаунт Microsoft.

**Примечание:** Microsoft может обновлять интерфейс центра администрирования Entra со временем. Эти инструкции были написаны на основе интерфейса, доступного в начале 2026 года. Если какие-либо шаги отличаются от того, что вы видите, обратитесь к официальной документации Microsoft о [регистрации приложения с платформой идентификации Microsoft](https://learn.microsoft.com/en-us/entra/identity-platform/quickstart-register-app).

## Предварительные условия

- Подписка Azure с доступом к Microsoft Entra ID
- Роль **Application Administrator** или **Global Administrator** в вашем арендованном арендодателе Entra ID
- URL-адрес вашего магазина Spwig (например, `https://your-store.com`)
- У сотрудников должны быть электронные адреса в Spwig, совпадающие с их аккаунтами Microsoft

## Шаг 1: Регистрация приложения

1. Войдите в [центр администрирования Microsoft Entra](https://entra.microsoft.com)
2. Перейдите в **Identity > Applications > App registrations**
3. Нажмите **New registration**
4. Настройте регистрацию:

| Field | Value |
|-------|-------|
| **Name** | `Spwig Admin SSO` (или любое имя по вашему выбору) |
| **Supported account types** | **Accounts in this organizational directory only** (Single tenant) |
| **Redirect URI** | Platform: **Web**, URI: `https://your-store.com/oidc/callback/` |

5. Нажмите **Register**

**Важно:** URI перенаправления должен точно совпадать с `https://your-store.com/oidc/callback/` — включая завершающий слеш. Замените `your-store.com` на фактический домен вашего магазина.

## Шаг 2: Запишите идентификаторы приложения

После регистрации вы увидите страницу **Overview** приложения. Запишите эти два значения — вам понадобятся они позже:

| Value | Where to Find It | What It's For |
|-------|-----------------|---------------|
| **Application (client) ID** | Overview page, top section | Enter as **Client ID** in Spwig |
| **Directory (tenant) ID** | Overview page, top section | Used to build the Discovery URL |

## Шаг 3: Создайте секрет клиента

1. В регистрации приложения перейдите в **Certificates & secrets**
2. Нажмите **New client secret**
3. Введите описание (например, `Spwig SSO`) и выберите срок действия
4. Нажмите **Add**
5. **Сразу скопируйте значение** — оно отображается только один раз. Это секретный ключ, который вы введете в Spwig.

**Не копируйте идентификатор секрета** — вам нужен столбец **Value**, а не столбец **ID**.

**Установите напоминание** о том, чтобы обновить секрет до его истечения срока действия. Когда секрет истекает, SSO перестанет работать, пока вы не создадите новый и не обновите его в Spwig.

## Шаг 4: Настройка разрешений API

1. Перейдите в **API permissions**
2. Проверьте, что **Microsoft Graph > User.Read** (delegated) указан. Это добавляется по умолчанию.
3. Если разрешения `openid`, `email` и `profile` не указаны, нажмите **Add a permission > Microsoft Graph > Delegated permissions** и добавьте их.
4. Нажмите **Grant admin consent for [your organization]**, если это требуется.

## Шаг 5: Создание URL-адреса обнаружения

URL-адрес обнаружения OIDC имеет следующий формат:

```
https://login.microsoftonline.com/{tenant-id}/v2.0/.well-known/openid-configuration
```

Замените `{tenant-id}` на **Directory (tenant) ID** из шага 2.

Пример: если ваш ID арендодателя `a1b2c3d4-e5f6-7890-abcd-ef1234567890`, URL-адрес обнаружения будет:

```
https://login.microsoftonline.com/a1b2c3d4-e5f6-7890-abcd-ef1234567890/v2.0/.well-known/openid-configuration
```

## Шаг 6: Настройка утверждений групп (опционально)

Если вы хотите, чтобы Spwig автоматически назначал статус сотрудника или суперпользователя на основе членства в группе Entra ID:

1. В регистрации приложения перейдите в **Token configuration**
2. Нажмите **Add groups claim**
3. Выберите типы групп для включения (обычно **Security groups**)
4. В разделе **Customize token properties by type**, для **ID** токена выберите **Group ID**
5. Нажмите **Add**

**Важно:** Entra ID отправляет **Object IDs** (UUIDs вроде `a1b2c3d4-...`), а не отображаемые имена групп.

При настройке сопоставления ролей в Spwig вы должны использовать эти Object IDs.

Чтобы найти Object ID группы:
1. В центре администрирования Entra перейдите в **Identity > Groups > All groups**
2. Нажмите на группу
3. Скопируйте **Object ID** с страницы обзора группы

### Ограничение групп

Microsoft Entra ID включает в токен максимум **200 групп**. Если пользователь принадлежит к более чем 200 группам, утверждение групп заменяется на ссылку на Microsoft Graph API. Для организаций с большим количеством групп рассмотрите возможность создания отдельной группы безопасности для доступа к Spwig и использования [фильтрации групп](https://learn.microsoft.com/en-us/entra/identity-platform/optional-claims-reference), чтобы ограничить, какие группы включаются.

## Шаг 7: Настройка в Spwig

1. В административной панели Spwig перейдите в **Enterprise SSO > SSO Provider Configuration**
2. Установите **Provider Name** на `Microsoft Entra ID`
3. Вставьте URL-адрес открытия из шага 5 в **OIDC Discovery URL**
4. Нажмите **Auto-Discover** — это автоматически заполнит все поля конечных точек
5. Введите **Client ID** из шага 2
6. Введите **Client Secret** (значение) из шага 3
7. Если вы настроили утверждения групп в шаге 6:
   - Установите **Groups Claim** на `groups`
   - В **Staff Groups** введите Object IDs групп, чьи члены должны быть сотрудниками (разделенные запятыми)
   - В **Superuser Groups** введите Object IDs групп, чьи члены должны быть суперпользователями (разделенные запятыми)
8. Нажмите **Save**

## Шаг 8: Включение и тестирование

1. Перейдите в **Site Settings > Security**
2. Поставьте галочку **Enable SSO for admin login**
3. Нажмите **Save**
4. Откройте страницу входа в админ-панель в **частном/инкогнито окне**
5. Вы должны увидеть кнопку **Sign in with Microsoft Entra ID**
6. Нажмите на нее — вы должны быть перенаправлены на страницу входа Microsoft
7. Войдите с учетной записью Microsoft, электронная почта которой совпадает с электронной почтой сотрудника в Spwig
8. Вы должны быть перенаправлены обратно на админ-панель Spwig

## Распространенные проблемы

| Проблема | Причина | Решение |
|---------|-------|----------|
| **AADSTS50011: URI перенаправления не совпадает** | URI перенаправления в Entra не совпадает точно | Проверьте, что URI перенаправления равен `https://your-store.com/oidc/callback/` с завершающим слешем. Проверьте, нет ли несоответствия между HTTP и HTTPS. |
| **AADSTS700016: Приложение не найдено** | Неправильный Client ID или арендатор | Проверьте Client ID и убедитесь, что URL открытия использует правильный идентификатор арендатора |
| **Вход успешен в Microsoft, но неудачен в Spwig** | Нет совпадающего пользователя в Spwig | Убедитесь, что в Spwig существует учетная запись сотрудника с той же электронной почтой, что и у учетной записи Microsoft. Проверьте, что у пользователя есть статус сотрудника, если включено ограничение только для сотрудников. |
| **Утверждение групп пустое** | Утверждения групп не настроены | Следуйте шагу 6, чтобы добавить утверждение групп в конфигурацию токена |
| **Утверждение групп возвращает URL вместо ID** | Пользователь принадлежит к более чем 200 группам | Используйте фильтрацию групп, чтобы ограничить группы в токене, или назначьте конкретные группы |
| **SSO перестает работать после нескольких месяцев** | Срок действия секрета клиента истек | Создайте новый секрет клиента в Entra и обновите его в конфигурации провайдера SSO Spwig |

## Советы

- **Используйте группы безопасности** для сопоставления ролей, а не группы Microsoft 365 или распределительные списки.

Группы безопасности предназначены для управления доступом и работают наиболее надежно с утверждениями OIDC.
- **Рекомендуется использовать однотенантную конфигурацию** — выбор «Accounts in this organizational directory only» ограничивает SSO только пользователям вашей организации.

Многотенантные конфигурации требуют дополнительной проверки.
- **Установите длительный срок истечения срока действия секрета** — выберите 24 месяца при создании секрета клиента и установите напоминание календаря на 22 месяца, чтобы обновить его.
- **Условный доступ** — вы можете создать политики условного доступа в Entra ID, которые применяются только к регистрации приложения Spwig.


Например, можно потребовать использование MFA, блокировать вход с непроверенных местоположений или требовать использование соответствующих устройств.
- **Проверьте с помощью неадмин-аккаунта** — создайте тестовый аккаунт сотрудника в Spwig, чтобы убедиться, что SSO работает, прежде чем внедрить его для всей вашей команды.
