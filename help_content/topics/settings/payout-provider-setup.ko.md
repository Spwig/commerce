title: 지급 서비스 제공업체 설정
---

지급 서비스 제공업체 설정은 자동화된 파트너 지급을 위해 PayPal 및 Airwallex를 구성할 수 있도록 합니다. 이 가이드는 제공업체 계정을 연결하고 웹후크를 구성하며 통합을 테스트하는 방법을 보여줍니다.

## 지원되는 지급 서비스 제공업체

Spwig은 두 가지 지급 서비스 제공업체와 통합하여 파트너 지급을 자동화합니다:

| 제공업체 | 지급 방법 | 처리 | 배치 지원 | 최적의 사용처 |
|----------|----------------|------------|---------------|----------|
| **PayPal** | PayPal 계정 이체 | API 기반 | 예 (최대 15,000) | 대부분의 파트너, 글로벌 범위 |
| **Airwallex** | 국제 은행 이체 | API 기반 | 아니요 (개별) | 은행 이체, 국제 지급 |

### 주요 차이점

**PayPal 지급**:
- 파트너가 PayPal 계정(지급 이메일)을 가져야 함
- 최대 15,000개의 지급을 한 번에 처리
- 더 빠른 처리 (1-2 영업일)
- 낮은 설정 복잡도
- 수수료: ~2% 또는 지급당 $0.25-$1.00
- 전체 배치에 대한 단일 웹후크

**Airwallex**:
- 직접 은행 이체 지원
- 개별 지급을 하나씩 처리
- 더 긴 처리 시간 (2-5 영업일)
- 여러 통화 및 국가 지원
- 수수료는 목적지 국가에 따라 다름
- 지급당 개별 웹후크

두 제공업체를 모두 구성하고 파트너가 선호하는 지급 방법을 선택하도록 할 수 있습니다.

## 왜 지급 서비스 제공업체를 사용해야 하나요?

지급 서비스 제공업체를 통합하면 수동 지급보다 다음과 같은 이점이 있습니다:

- **자동 처리** — 수동 데이터 입력 또는 지급 실행 필요 없음
- **배치 효율성** — 클릭 하나로 수십 개 또는 수백 개의 지급 처리
- **웹후크 확인** — 지급이 완료되면 자동 상태 업데이트
- **오류 감소** — 처리 전에 시스템이 계좌 정보를 검증
- **감사 추적** — 거래 및 제공업체 응답의 전체 기록
- **빠른 지급** — 파트너가 더 빠르게 자금을 받음
- **확장성** — 파트너 프로그램이 성장하면서도 관리 작업이 비례하지 않음

제공업체 통합 없이도 은행 또는 PayPal 대시보드를 통해 각 지급을 수동으로 처리해야 하며, 이후 Spwig으로 돌아가 지급을 완료 상태로 표시해야 합니다.

## PayPal 설정

자동화된 파트너 지급을 위해 PayPal 지급을 구성하는 단계를 따르세요.

### 사전 조건

시작하기 전에 다음이 필요합니다:
- PayPal Business 계정 (개인 계정은 지급 API 사용 불가)
- PayPal 개발자 대시보드에 대한 액세스
- 지급 API에 대한 프로덕션 승인 (샌드박스 테스트 후)

### 단계 1: PayPal 앱 생성

1. **이동** [PayPal 개발자 대시보드](https://developer.paypal.com/dashboard/)
2. **로그인** PayPal Business 계정으로
3. **클릭** 왼쪽 사이드바의 **My Apps & Credentials**
4. **선택** **Live** 탭 (테스트용으로는 Sandbox)
5. **클릭** **Create App**
6. **입력 앱 이름** (예: "Spwig 파트너 지급")
7. **앱 유형 선택**: 판매자
8. **클릭** **Create App**

PayPal은 앱의 자격 증명을 생성합니다.

### 단계 2: API 자격 증명 가져오기

앱을 생성한 후:

1. **복사 Client ID** — 긴 영문 숫자 문자열
2. **클릭** **Show** 비밀 아래
3. **복사 Client Secret** — 이 정보를 기밀로 유지하세요
4. **모드 노트** — 샌드박스 또는 라이브

### 단계 3: 지급 기능 활성화

PayPal 앱은 지급을 사용하려면 명시적인 허가가 필요합니다:

1. **스크롤** 앱의 **Features** 섹션으로
2. **찾기** **Payouts** 기능
3. **추가** 아직 활성화되지 않은 경우 클릭
4. **승인 요청** 라이브 모드 사용 시 (승인은 1-2 영업일 소요)

### 단계 4: Spwig에 제공업체 추가

이제 PayPal 계정을 Spwig에 추가하세요:

1. **이동** **Settings > Payout Providers**
2. **클릭** **+ Add PayPal Account**
3. **폼 작성**:
   - **계정 이름**: 설명적인 라벨 (예: "Main PayPal Account")
   - **Client ID**: PayPal 개발자 대시보드에서 복사
   - **Client Secret**: PayPal 개발자 대시보드에서 복사
   - **모드**: 테스트용으로는 Sandbox, 라이브용으로는 Production 선택
   - **활성화**: 활성화하려면 체크
4. **클릭 Save**

Spwig은 액세스 토큰을 요청하여 자격 증명을 검증합니다. 검증이 실패하면 Client ID 및 Secret을 다시 확인하세요.

### 단계 5: 연결 테스트

PayPal 통합을 확인하세요:

1. **Affiliate Program > Payouts**에서 테스트 지급 생성
2. 자신의 PayPal 이메일을 수신자로 사용
3. 금액을 $0.01로 설정 (프로덕션 모드일 경우) 또는 임의 금액 (샌드박스 모드일 경우)
4. 제공업체와 함께 처리
5. PayPal 계정에서 입금 확인
6. Spwig에서 웹후크가 지급 상태를 업데이트하는지 확인

샌드박스 모드를 사용하는 경우, [PayPal Sandbox](https://developer.paypal.com/dashboard/accounts)에서 테스트 지급을 받을 수 있는 테스트 PayPal 계정을 생성하세요.

## Airwallex 설정

Airwallex은 직접 입금을 선호하는 파트너에게 국제 은행 이체를 지원합니다.

### 사전 조건

시작하기 전에 다음이 필요합니다:
- Airwallex 계정 (사이트 [airwallex.com](https://www.airwallex.com)에서 생성)
- 확인된 사업자 계정 상태
- API 액세스 활성화 (필요 시 Airwallex 지원팀에 문의)
- Airwallex 계정에 충분한 잔액

### 단계 1: API 자격 증명 생성

1. **로그인** [Airwallex 대시보드](https://www.airwallex.com/app/)
2. **이동** **Settings > API Keys**
3. **클릭** **Create API Key**
4. **설명 입력**: "Spwig 파트너 지급"
5. **권한 선택**: **Payouts** (읽기 및 쓰기) 활성화
6. **클릭** **Generate**
7. **API 키 복사** — 한 번만 표시됨
8. **Client ID 복사** — 키와 함께 표시됨

### 단계 2: 환경 확인

Airwallex은 두 가지 환경을 제공합니다:

- **Demo**: 가상 거래로 테스트
- **Production**: 실제 돈 이체

API 키가 속한 환경을 확인하세요.

### 단계 3: Spwig에 제공업체 추가

Airwallex 계정을 Spwig에 추가하세요:

1. **이동** **Settings > Payout Providers**
2. **클릭** **+ Add Airwallex Account**
3. **폼 작성**:
   - **계정 이름**: 설명적인 라벨 (예: "Airwallex EUR 계정")
   - **API 키**: Airwallex 대시보드에서 복사
   - **Client ID**: Airwallex 대시보드에서 복사
   - **환경**: Demo 또는 Production 선택
   - **활성화**: 활성화하려면 체크
4. **클릭 Save**

Spwig은 계좌 잔액을 조회하여 자격 증명을 검증합니다.

### 단계 4: 지원되는 국가 확인

Airwallex은 많은 국가에 이체를 지원하지만 모든 국가를 지원하지는 않습니다. [Airwallex 커버리지](https://www.airwallex.com/global-business-account/global-transfers) 페이지를 확인하여 파트너의 국가가 지원되는지 확인하세요.

일반적으로 지원되는 국가에는 다음과 같은 것이 포함됩니다:
- 미국
- 영국
- 유럽 연합 국가
- 호주
- 캐나다
- 싱가포르
- 홍콩

### 단계 5: 은행 이체 테스트

Airwallex 통합을 테스트하세요:

1. 은행 정보가 있는 파트너를 위한 테스트 지급 생성
2. 프로덕션 모드일 경우 작은 금액 ($1-$5) 사용
3. 제공업체와 함께 처리
4. Airwallex 대시보드에서 거래 확인
5. 웹후크 확인을 기다림
6. Spwig에서 지급 완료 확인

Demo 모드는 즉시 처리됩니다. 프로덕션 모드는 2-5 영업일이 소요됩니다.

## 제공업체 선택 논리

지급을 처리할 때 Spwig은 파트너의 지급 방법에 따라 적절한 제공업체를 자동으로 선택합니다.

### 선택 흐름

1. **파트너 지급 방법 확인**:
   - `payment_email`이 설정된 경우 → 파트너는 PayPal을 선호함
   - 은행 정보가 설정된 경우 → 파트너는 은행 이체를 선호함
2. **제공업체에 매칭**:
   - PayPal 이메일 → 활성화된 PayPal 제공업체 계정 사용
   - 은행 정보 → 활성화된 Airwallex 제공업체 계정 사용
3. **대체 제공업체로 전환** → 선호 제공업체가 구성되지 않은 경우
4. **에러 표시** → 매칭 제공업체가 없는 경우

### 여러 제공업체 계정

동일한 제공업체에 대해 여러 계정을 구성할 수 있습니다 (예: 다른 지역을 위한 두 개의 PayPal 계정). Spwig은 지급 방법에 맞는 첫 번째 활성화된 계정을 선택합니다. 사용할 계정을 제어하려면 관리 목록에서 순서를 재정렬하거나 하나만 활성화하세요.

## 지급 통합 테스트

라이브 지급을 처리하기 전에 제공업체 통합을 항상 테스트하세요.

### 샌드박스/데모 모드 테스트

1. **제공업체를 샌드박스 모드로 설정** (PayPal 샌드박스 또는 Airwallex 데모)
2. **테스트 파트너**를 생성하고 테스트 지급 세부 정보를 입력하세요
3. **테스트 수익**을 생성하고 승인하세요
4. **테스트 지급**을 생성하고 해당 수익을 포함하세요
5. **제공업체를 사용하여 처리** (액션 메뉴를 통해)
6. **Celery 로그 모니터링** (API 요청 확인)
7. **제공업체 대시보드 확인** (거래 확인)
8. **웹후크 대기** (지급 상태 업데이트 확인)
9. **수익이 지급된 상태로 표시되는지 확인**

### 프로덕션 테스트

라이브로 전환하기 전:

1. **제공업체 설정에서 프로덕션 모드로 전환**
2. **자신에게 작은 테스트 지급** 생성 ($0.01-$1.00)
3. **처리**하고 완료를 기다림
4. **자신의 계좌에 자금 수령 확인**
5. **웹후크 발동 확인** 및 상태 업데이트
6. **제공업체 거래 수수료 검토**

### 일반적인 테스트 문제

| 문제 | 원인 | 해결책 |
|-------|-------|----------|
| "잘못된 자격 증명" | 잘못된 API 키 또는 모드 불일치 | 자격 증명 다시 확인, 샌드박스 대 프로덕션 확인 |
| 웹후크가 실행되지 않음 | 제공업체에서 URL이 구성되지 않음 | 제공업체 대시보드에 웹후크 URL 추가 |
| 지급이 처리 상태에 머무름 | 웹후크 서명 실패 | 웹후크 비밀이 일치하는지 확인 |
| 사용 가능한 제공업체 없음 | 지급 방법에 대한 활성화된 제공업체 없음 | 적어도 하나의 제공업체 계정을 활성화하세요 |

## 배치 처리 (PayPal)

PayPal은 효율성과 비용 절감을 위해 배치 처리를 지원합니다.

### 배치 처리 작동 방식

여러 지급을 선택하고 **Provider와 함께 처리**를 클릭하면:

1. Spwig은 모든 PayPal 지급을 단일 배치로 그룹화합니다
2. 시스템은 모든 지급 세부 정보를 포함한 단일 API 요청을 보냅니다 (최대 15,000)
3. PayPal은 전체 배치를 단일 거래로 처리합니다
4. 웹후크는 배치 결과와 함께 반환됩니다
5. Spwig은 배치 응답에 따라 모든 지급을 업데이트합니다

### 배치 처리의 장점

- **API 요청 감소** — 수십 개의 지급에 대해 단일 요청
- **낮은 수수료** — 일부 PayPal 수수료 구조는 배치 처리를 선호함
- **빠른 처리** — 전체 배치에 대한 병렬 실행
- **단일 웹후크** — 모니터링 및 로깅이 더 쉬움

### 배치 제한

PayPal은 다음과 같은 제한을 적용합니다:
- 배치당 최대 15,000 수신자
- 배치당 최대 $100,000 총액
- 처리는 일반적으로 몇 분 내에 완료됨

15,000개 이상의 지급을 처리하면 Spwig은 자동으로 여러 배치로 나뉩니다.

## 개별 처리 (Airwallex)

Airwallex은 지급을 하나씩 처리하며 이는 다른 트레이드오프를 제공합니다.

### 개별 처리 작동 방식

Airwallex 지급을 처리할 때:

1. 시스템은 각 지급에 대해 별도의 API 요청을 보냅니다
2. Airwallex은 이체를 개별적으로 대기열에 추가합니다
3. 각 이체는 독립적으로 완료됩니다 (2-5일)
4. 각 이체가 완료되면 개별 웹후크가 실행됩니다
5. Spwig은 웹후크가 도착할 때 지급을 업데이트합니다

### 개별 처리의 장점

- **더 나은 오류 격리** — 하나의 실패가 다른 것들을 막지 않음
- **개별 지급 추적** — 개별 거래 ID
- **더 많은 지급 세부 정보** — 이체당 은행별 정보
- **유연한 타이밍** — 이체가 다른 속도로 완료됨

### 처리 시간

PayPal의 즉시 배치와 달리 Airwallex 이체는 더 오래 걸립니다:
- 국내 이체: 1-2 영업일
- 국제 이체: 3-5 영업일
- 일부 국가: 최대 7 영업일

프로그램 조건에 따라 파트너의 기대를 적절히 설정하세요.

## 웹후크 구성

웹후크는 제공업체가 거래를 완료할 때 자동 지급 상태 업데이트를 가능하게 합니다.

### 웹후크 URL 형식

제공업체 대시보드에서 이 URL을 구성하세요:

```
https://yourdomain.com/api/payout-providers/{provider}/webhook/
```

{provider}을 다음과 같이 대체하세요:
- `paypal`은 PayPal 웹후크를 위한 것
- `airwallex`은 Airwallex 웹후크를 위한 것

예시:
- `https://shop.example.com/api/payout-providers/paypal/webhook/`
- `https://shop.example.com/api/payout-providers/airwallex/webhook/`

### PayPal 웹후크 설정

1. **이동** [PayPal 개발자 대시보드](https://developer.paypal.com/dashboard/)
2. **클릭** 앱 이름
3. **스크롤** **Webhooks** 섹션으로
4. **클릭** **Add Webhook**
5. **웹후크 URL 입력** (위 형식)
6. **이벤트 선택**:
   - `PAYMENT.PAYOUTSBATCH.SUCCESS`
   - `PAYMENT.PAYOUTSBATCH.DENIED`
   - `PAYMENT.PAYOUTS-ITEM.SUCCEEDED`
   - `PAYMENT.PAYOUTS-ITEM.FAILED`
7. **클릭 Save**

PayPal은 웹후크 서명 키를 제공합니다. Spwig은 이를 사용하여 웹후크의 진위를 확인합니다.

### Airwallex 웹후크 설정

1. **이동** [Airwallex 대시보드](https://www.airwallex.com/app/)
2. **이동** **Settings > Webhooks**
3. **클릭** **Create Webhook**
4. **웹후크 URL 입력** (위 형식)
5. **이벤트 선택**:
   - `transfer.created`
   - `transfer.completed`
   - `transfer.failed`
6. **클릭 Create**

Airwallex은 웹후크를 API 비밀 키로 서명합니다.

### 웹후크 보안

웹후크는 다음 메커니즘을 사용하여 검증됩니다:

- **서명 검증** — 제공업체는 서명 키로 웹후크 페이로드를 서명합니다
- **타임스탬프 확인** — 오래된 웹후크 거부 (재생 공격 방지)
- **IP 허용 목록** (선택 사항) — 제공업체 IP 범위에 제한
- **HTTPS 필수** — 웹후크는 SSL을 통해만 작동

생산 환경에서는 서명 검증을 비활성화하지 마세요.

### 웹후크 테스트

대부분의 제공업체는 웹후크 테스트 도구를 제공합니다:

**PayPal**: 개발자 대시보드의 "Simulator"를 사용하여 테스트 웹후크를 실행하세요

**Airwallex**: 데모 모드에서 테스트 이체를 생성하고 웹후크가 실행되는지 확인하세요

Spwig의 **Settings > System Logs**에서 웹후크 로그를 확인할 수도 있습니다 (로그가 활성화된 경우).

## 문제 해결

### 잘못된 자격 증명 오류

**증상**: 제공업체 계정을 저장할 때 "인증 실패"

**원인**:
- 잘못된 Client ID 또는 Secret
- 샌드박스 자격 증명을 프로덕션 모드에서 사용 (또는 그 반대)
- API 키 만료 또는 취소됨
- 계정이 확인되지 않음

**해결책**:
- 제공업체 대시보드에서 자격 증명 다시 복사
- 모드가 일치하는지 확인 (샌드박스 대 프로덕션)
- API 키 다시 생성
- 제공업체 지원팀에 문의하여 계정 상태 확인

### 웹후크 수신 실패

**증상**: 지급이 "Processing" 상태에 영원히 머무름

**원인**:
- 제공업체 대시보드에서 웹후크 URL이 구성되지 않음
- HTTPS 인증서가 유효하지 않음
- 방화벽이 제공업체 IP를 차단
- 웹후크 서명 검증 실패

**해결책**:
- 제공업체 설정에서 웹후크 URL 다시 확인
- SSL 인증서가 유효한지 확인
- 방화벽에서 제공업체 IP 범위 허용
- Celery 로그에서 서명 오류 확인
- 제공업체의 시뮬레이터 도구로 웹후크 테스트

### 지급 실패

**증상**: 지급 상태가 "Failed"로 변경되고 오류 메시지가 있음

**원인**:
- 잘못된 파트너 지급 세부 정보 (잘못된 이메일 또는 은행 계좌)
- 제공업체 계정 잔액 부족
- 수신자 계좌가 지급을 받을 수 없음
- 국가가 지원되지 않음 (Airwallex)
- 지급이 제공업체 제한을 초과함

**해결책**:
- **Provider Response** 필드에서 오류 확인
- 파트너의 지급 세부 정보가 올바른지 확인
- 제공업체 계정에 자금 추가
- 파트너에게 계좌 상태 확인 요청
- 제공업체의 국가 및 통화 지원 확인
- 제한을 초과하는 대규모 지급을 분할

### 모드 불일치

**증상**: 테스트 지급은 작동하지만 프로덕션 지급은 실패함

**원인**:
- 제공업체가 샌드박스 모드로 설정되어 있지만 프로덕션 파트너 계정을 사용함
- 잘못된 환경의 API 자격 증명 사용

**해결책**:
- 제공업체 모드를 프로덕션으로 전환
- 프로덕션 API 자격 증명 다시 생성
- 웹후크 URL이 프로덕션 도메인을 가리키는지 확인

## 보안 최선 실천

지급 통합을 보호하기 위해 다음 보안 조치를 취하세요:

### 자격 증명 저장

- **버전 관리에 자격 증명을 커밋하지 마세요** — 환경 변수 또는 보안 저장소 사용
- **분기별 API 키 회전** — 3개월마다 새로운 키 생성
- **샌드박스 및 프로덕션용 별도 키 사용** — 환경을 혼합하지 마세요
- **API 권한 제한** — 지급에 대한 액세스만 부여하고 전체 계좌 제어는 허용하지 마세요

Spwig은 데이터베이스에 제공업체 자격 증명을 암호화하여 저장합니다. 데이터베이스 백업을 보호하세요.

### 웹후크 보안

- **항상 서명 검증** — 서명 검증을 건너뛰지 마세요
- **HTTPS만 사용** — HTTP 웹후크는 지원되지 않음
- **IP 허용 목록 구현** — 웹후크를 제공업체 IP 범위에 제한
- **모든 웹후크 로그** — 이상한 활동 감시
- **웹후크 엔드포인트에 대한 요청 제한** — 남용 방지

### 접근 제어

- **직원 접근 제한** — 지급을 처리할 수 있는 직원만 제한하세요
- **2FA 사용** — 직원 계정에 2FA를 요구하세요
- **지급 작업 감사** — 누가 어떤 지급을 처리했는지 검토하세요
- **업무 분리** — 승인과 처리를 담당하는 다른 직원을 지정하세요

### 모니터링

- **매일 실패한 지급 확인** — 문제를 신속히 해결하세요
- **제공업체 계좌 잔액 모니터링** — 충분한 자금이 있는지 확인하세요
- **주간 거래 로그 검토** — 이상을 조기에 발견하세요
- **알림 설정** — 대규모 또는 실패한 지급에 대한 이메일 알림

## 팁

- 라이브 모드로 전환하기 전에 샌드박스 모드에서 통합을 철저히 테스트하세요 — 가상 돈으로 문제를 포착하세요.
- PayPal 및 Airwallex 모두를 구성하여 파트너가 선호하는 지급 방법을 선택하도록 하세요 — 다른 파트너는 다른 방법을 선호합니다.
- 초기 설정 시 웹후크 URL을 구성하고 올바르게 실행되는지 확인하세요 — 웹후크는 자동화에 필수적입니다.
- 배치 처리 중 지급 실패를 피하기 위해 제공업체 계좌 잔액을 항상 충분히 유지하세요.
- 여러 제공업체를 구성할 경우 설명적인 계정 이름을 사용하세요 (예: "PayPal USD", "PayPal EUR").
- 보안 최선 실천으로 분기별 API 자격 증명을 회전하세요.
- 웹후크 URL 및 자격 증명을 보안 암호 관리자에 문서화하고 팀과 공유하세요.
- 실패한 지급을 즉시 모니터링하세요 — 지연은 파트너에게 짜증을 주고 프로그램의 평판을 해칩니다.
- Spwig 설치에 항상 HTTPS를 사용하세요 — 웹후크는 SSL 인증서가 필요합니다.
- 지속적인 오류가 발생할 경우 제공업체 지원팀에 문의하세요 — 계정 상태 및 권한을 확인할 수 있습니다.
