---
title: 配送プロバイダーアカウント
---

配送プロバイダーアカウントは、リアルタイムの料金計算とラベルの自動購入のために、ストアをカーレットのAPI（フェデックス、UPS、DHLなど）に接続します。各アカウントは暗号化されたAPI資格情報を保存し、接続の状態を監視し、リアルタイムの配送方法にリンクします。プロバイダーは、パッケージの寸法、重量、出発地、目的地に基づいてチェックアウト時にライブレートを取得し、手動でレートテーブルを管理する必要をなくし、正確なカーレット料金を保証します。

カーレットが計算した配送料金やラベルの自動生成が必要な場合にプロバイダーアカウントを使用してください。

## サポートされている配送プロバイダー

Spwigは、インストール可能なプロバイダー コンポーネントを通じて主要なカーレットをサポートしています。

### フェデックス

**サービス**: グラウンド、エクスプレス、国際
**API**: フェデックス ウェブ サービス
**機能**: リアルタイム料金、ラベル購入、追跡、国際関税

### UPS

**サービス**: グラウンド、航空便、世界中
**API**: UPS デベロッパーアプリケーションプログラミングインターフェース（API）
**機能**: リアルタイム料金、ラベル生成、追跡、住所検証

### DHL

**サービス**: エクスプレス、eコマース、国際
**API**: DHL エクスプレス API
**機能**: 国際料金、関税書類、追跡

### その他のプロバイダー

必要に応じてコンポーネント マーケットプレイスからインストールしてください（USPS、カナダ ポスト、オーストラリア ポストなど）。

## プロバイダー アカウントの構成

各プロバイダー アカウントには以下が必要です。

### 基本情報

- **表示名**: 管理画面に表示されるアカウント名（例: "フェデックス プロダクション アカウント"）
- **プロバイダー**: ドロップダウンからインストール済みのプロバイダー コンポーネントを選択
- **アクティブ**: アカウントを有効/無効に切り替え、資格情報を削除せずに
- **デフォルト**: このプロバイダーのデフォルトアカウントとして設定（プロバイダーごとに1つだけ）

### API 資格情報（暗号化済み）

**プロバイダーごとに異なります**。通常は以下を含みます。

**フェデックス**:
- アカウント番号
- メーター番号
- API キー
- API シークレット

**UPS**:
- アクセスライセンス番号
- ユーザーID
- パスワード
- アカウント番号

**DHL**:
- サイトID
- パスワード
- アカウント番号

**すべての資格情報は静止時に暗号化され**、API呼び出しを行うときにのみ復号されます。

### 出発地の住所

- **デフォルトの発送元**: レート計算用の倉庫/出発地の住所
- 一部のプロバイダーは、そのダッシュボードで特定の出発地の設定が必要です。

### 設定

プロバイダー固有のオプション（カーレットごとに異なります）:

- **テストモード**: カーレットのサンドボックス/テストAPIエンドポイントを使用
- **交渉済み料金**: 交渉済みのカーレット料金を使用（利用可能であれば）
- **保険を含める**: レートに自動で保険を提示
- **住宅追加料金**: 住宅配達料金を適用
- **署名が必要**: デフォルトの署名要件

## プロバイダー アカウントの作成

**6ステップのセットアッププロセス**:

**ステップ1: カーレットのAPIアクセスを取得**
1. カーレット（フェデックス.com、UPS.com、DHL.com）でアカウントを作成
2. API/デベロッパーアクセスを申請
3. カーレットのAPIオンボーディングを完了（1〜3営業日かかる場合があります）
4. エメールまたは開発者ポータル経由でAPI資格情報を取得

**ステップ2: プロバイダー コンポーネントをインストール**（事前にインストールされていない場合）
1. 設定 > コンポーネント > マーケットプレイスに移動
2. カーレット名（例: "フェデックス"）を検索
3. 配送プロバイダー コンポーネントをインストール
4. インストールが完了するのを待ちます

**ステップ3: Spwigでプロバイダー アカウントを作成**
1. 設定 > 配送 > プロバイダー アカウントに移動
2. "プロバイダー アカウントを追加"をクリック
3. ドロップダウンからプロバイダーを選択
4. 表示名を入力

**ステップ4: API資格情報を入力**
1. 資格情報フィールドを入力（プロバイダーごとに異なります）
2. 保存時に資格情報が自動で暗号化されます
3. 任意: 初期テストのためにテストモードを有効にします

**ステップ5: 接続をテスト**
1. "接続をテスト"ボタンをクリック
2. システムがカーレットのAPI呼び出しを試行
3. "接続済み"ステータスが表示されることを確認
4. last_tested_at タイムスタンプを確認

**ステップ6: 配送方法にリンク**
1. 配送方法を作成または編集（設定 > カート > 配送方法）
2. method_type = "Real-Time" に設定
3. ドロップダウンからプロバイダー アカウントを選択
4. メソッドを保存

## 接続状態の監視

プロバイダー アカウントは接続の状態を監視します。

### ステータス値

**不明**（灰色）: これまでテストされていないか、まだ接続されていない

**接続済み**（緑）: 最後のAPI呼び出しが成功し、資格情報が有効

**エラー**（赤）: 最後のAPI呼び出しが失敗し、資格情報が無効である可能性があります

### 最後にテストされた

- **タイムスタンプ**: 接続が最後に確認された時間
- **自動更新**: プロバイダーが使用されるたびに（レート取得、ラベル購入）
- **手動テスト**: いつでも "接続をテスト" ボタンをクリック

### 接続の失敗をトラブルシューティングする

**一般的な原因**:
- 間違ったAPI資格情報（タイプミス、余分なスペースを含んでコピー）
- カーレットのAPIキーが期限切れまたは取り消された
- テストモードが有効になっているが、生産資格情報を使用している（またはその逆）
- IPアドレスがカーレットとホワイトリストに登録されていない
- カーレットのAPIがダウンしている

**解決手順**:
1. 資格情報がカーレットのダッシュボードと完全に一致することを確認
2. テストモード設定が資格情報タイプと一致することを確認
3. カーレットのAPIステータスページで障害を確認
4. カーレットのサポートに連絡してアカウントの確認を依頼

## レート照会ワークフロー

チェックアウト時にリアルタイムレートがどのように機能するか:

**1. 顧客が住所を入力**
- 配送住所が入力される
- カートで総重量 + 尺寸が計算される

**2. システムがレートリクエストを準備**
- プロバイダー アカウント資格情報を取得（復号化）
- カートアイテムからパッケージの寸法を計算（配送パッケージが定義されている場合）
- オリジン、目的地、パッケージを含むAPIリクエストを準備

**3. プロバイダーAPIが呼び出される**
- 認証資格情報を含むカーレットAPIにリクエストを送信
- カーレットはゾーン、重量、寸法に基づいてレートを計算
- レスポンスにはサービスオプション（グラウンド、エクスプレスなど）が含まれます

**4. レートが表示される**
- システムがカーレットの応答を解析
- 標準形式に正規化
- 設定されている場合、オプションのマーカップが適用
- 顧客にチェックアウト時にレートが表示される

**5. 顧客がサービスを選択**
- 顧客が希望のオプションを選択
- 選択されたレートが注文に保存される

**APIフローの例**:
```
フェデックスAPIへのリクエスト:
{
  "origin": {"postal_code": "90210", "country": "US"},
  "destination": {"postal_code": "10001", "country": "US"},
  "parcels": [{
    "weight": 2500,  // グラム
    "dimensions": {"length": 30, "width": 20, "height": 15}  // cm
  }]
}

フェデックス応答:
[
  {"service": "FEDEX_GROUND", "rate": 12.50, "delivery_days": 5},
  {"service": "FEDEX_EXPRESS", "rate": 28.75, "delivery_days": 2}
]
```

## ラベル購入（オプション）

プロバイダーがラベル生成をサポートしている場合:

**ワークフロー**:
1. 顧客が注文を完了
2. 店舗が配送を作成（注文 > 注文詳細 > 配送を作成）
3. プロバイダー アカウント + サービスを選択
4. システムがプロバイダーのラベルAPIを呼び出す
5. ラベルPDFが生成され、配送に添付される
6. 追跡番号が自動で入力される
7. ラベルが印刷準備完了

**利点**:
- カーレットのウェブサイトに手動でログインする必要なし
- 追跡が自動で同期
- 国際配送の場合、関税書類が自動生成
- バッチラベル生成が可能

## レートマーアップ

カーレットレートに店舗のマーアップを追加:

**設定**（配送方法内、プロバイダー アカウント内ではない）:
- **マーアップタイプ**: パーセンテージまたは固定
- **マーアップ額**: 例: 15% または $2.50

**例**:
```
カーレットレート: $12.50
マーアップ: 15%
顧客が支払う金額: $14.38

または

カーレットレート: $12.50
マーアップ: $2.50（固定）
顧客が支払う金額: $15.00
```

**使用例**:
- 包装/取り扱い費用を補填
- 配送に利益率を追加
- 配送のクレジットカード手数料を補填

## 複数のプロバイダー アカウント

同じプロバイダーに対して複数のアカウントを作成できます。

**使用例**:
1. **テスト vs 生産**
   - テストアカウント: カーレットのサンドボックス資格情報
   - 生産アカウント: 実際の資格情報

2. **複数の倉庫**
   - 倉庫Aアカウント: 出発地 = ロサンゼルス
   - 倉庫Bアカウント: 出発地 = ニューヨーク

3. **異なる交渉済みレート**
   - アカウントA: 標準レート
   - アカウントB: 量割引レート

**各アカウントは異なる配送方法にリンクできます**。柔軟な構成が可能です。

## ヒント

- **サンドボックスでテストを実施** - 実際に運用する前に、カーレットのテスト資格情報を使用してください
- **接続状態を監視** - ダッシュボードで定期的にエラー状態を確認してください
- **配送パッケージを定義** - 尺寸の正確さによりレートの提示が向上します
- **交渉済みレートを使用** - カーレットと量割引がある場合は有効にしてください
- **現実的な出発地を設定** - 実際の発送元住所を使用して正確なゾーンを確保してください
- **資格情報を安全に保つ** - APIキーを共有しないでください、定期的にローテーションしてください
- **バックアップ方法を保持** - カーレットAPIが失敗した場合に、フラットレート方法を保持してください
- **カーレットAPIの制限を監視** - 一部のカーレットは1日のAPI呼び出し数に制限があります
- **資格情報を迅速に更新** - カーレットがキーをローテーションした場合、すぐに更新してください
- **記述的な名前を使用** - "フェデックス LA倉庫" が "フェデックス 1" よりも良い
