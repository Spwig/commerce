---
title: 搜索引擎說明
---

Spwig 中的搜索引擎不是像 Elasticsearch 或 Algolia 這樣的外部服務 - 它們是您商店數據庫原生搜索系統中的配置上下文。每個引擎定義要搜索的內容類型、要排除的內容以及結果的排序方式。本指南解釋了什麼是搜索引擎，什麼時候需要創建多個引擎，以及如何配置它們。

大多數商家使用一個預設的 "shop" 引擎。只有在需要不同內容組合或排除的情況下，才需要創建多個引擎。

![搜索引擎列表](/static/core/admin/img/help/search-engines-explained/search-engines-list.webp)

## 什麼是搜索引擎？

Spwig 中的搜索引擎是一個命名的配置，指定：

- **要搜索的內容類型**（產品、分類、品牌、部落格文章）
- **要排除的內容**（您希望從搜索中隱藏的特定分類或品牌）
- **自定義相關性權重**（可選的每個引擎權重覆蓋）
- **活動狀態**（引擎可以臨時停用）

每個引擎都有唯一的 slug，用於 API 調用和前端代碼，以指定哪個引擎應處理搜索請求。

## 什麼時候創建多個引擎

大多數商店只需要一個引擎。在以下情況下創建附加引擎：

| 使用情況 | 範例 |
|----------|---------|
| **不同的內容組合** | Shop 引擎只搜索產品；Blog 引擎只搜索部落格文章 |
| **選擇性排除** | 主商店引擎隱藏清倉分類；清倉引擎只顯示清倉商品 |
| **部門特定搜索** | 電子產品引擎排除服裝分類；服裝引擎排除電子產品 |
| **B2B 與 B2C 分離** | 批發引擎只顯示批量產品；零售引擎顯示消費產品 |

如果您不確定是否需要多個引擎，請堅持使用一個。除非有特定的使用情況，否則添加引擎會增加複雜性而沒有好處。

## 4 步驟向導

![向導步驟 1 - 基本資訊](/static/core/admin/img/help/search-engines-explained/wizard-step1-basic.webp)

導航到 **Search > Setup Wizard** 通過一個引導式的 4 步驟過程來創建新引擎：

### 步驟 1：基本資訊

**引擎名稱** - 友好的顯示名稱（例如，"Shop Search"，"Blog Search"）。僅在管理介面中使用。

**Slug** - URL 安全的識別符（例如，"shop-search"，"blog-search"）。用於 API 調用和前端代碼。如果留空，將從名稱自動生成。

**Active** - 這個引擎是否可用於搜索。停用的引擎不會返回任何結果。

### 步驟 2：內容類型

選擇這個引擎將搜索哪些類型的內容：

- 產品（包括所有產品類型：實體、數字、訂閱）
- 分類
- 品牌
- 部落格文章

**提示**：僅選擇與此引擎目的相關的內容類型。專注於部落格的引擎不需要啟用產品。

### 步驟 3：權重（可選）

![向導步驟 3 - 權重](/static/core/admin/img/help/search-engines-explained/wizard-step3-weights.webp)

可選地為此特定引擎自定義相關性權重。如果跳過，引擎將繼承來自 SearchSettings 的全局權重。

大多數引擎應跳過此步驟並使用全局默認值。只有在這個引擎有獨特的排名需求時才自定義權重（例如，部落格引擎可能會將 weight_blog_posts 增加到 1.2）。

### 步驟 4：審查與創建

審查您的配置並點擊 **Create Engine** 以保存。

## 引擎配置字段

如果您直接編輯引擎（繞過向導），您將看到這些字段：

**名稱和 Slug** - 顯示名稱和 URL 識別符

**活動狀態** - 切換以啟用/停用

**內容類型** - JSON 數組，如 `['product', 'category']`

**權重覆蓋** - JSON 物件，如 `{'weight_name': 1.8}`（如果使用全局權重則為空）

**排除的分類** - 與 Category 模型的 M2M 關係。這些分類中的產品不會出現在搜索結果中。

**排除的品牌** - 與 Brand 模型的 M2M 關係。這些品牌的产品不會出現在搜索結果中。

## 使用排除

排除會隱藏此引擎搜索結果中的特定內容：

**範例：隱藏清倉商品**

1. 創建一個 "Main Shop" 引擎
2. 在排除的分類欄位中，選擇您的 "Clearance" 分類
3. 在排除的品牌欄位中，選擇您想要隱藏的任何預算品牌
4. 儲存

現在通過 "Main Shop" 引擎的搜索不會返回清倉產品，即使它們在您的網站上是可見的。您可以創建一個專門的 "Clearance" 引擎，只搜索清倉商品。

## 在前端使用引擎

您的前端代碼通過 API 調用指定使用哪個引擎：

```javascript
// 使用 "shop" 引擎（最常見）
fetch('/api/search/?q=laptop&engine=shop')

// 使用 "blog" 引擎
fetch('/api/search/?q=ecommerce tips&engine=blog')

// 如果未指定引擎，使用預設引擎
fetch('/api/search/?q=laptop')
```

引擎的 slug 變成查詢參數。如果未指定引擎，Spwig 會使用按字母順序排列的第一個活動引擎。

## 引擎特定的同義詞和重定向

Synonym 和 SearchRedirect 模型都有可選的 `engine` 外鍵。如果設置，該同義詞或重定向僅適用於通過該特定引擎的搜索。

**範例**：Blog 引擎可以有像 "tutorial" → "guide" 這樣的同義詞，這些同義詞不適用於產品搜索。

大多數同義詞和重定向不應該是引擎特定的 - 留空 engine 欄位以全局應用它們。

## 小技巧

- **從一個引擎開始** - 創建預設的 "shop" 引擎，並在有明確需求之前使用它來處理所有事情
- **使用描述性的 slug** - 選擇像 "shop"、"blog"、"wholesale" 這樣的 slug，清楚地表明引擎的用途
- **在啟用前測試引擎** - 在非啟用狀態下創建引擎，通過 API 測試，然後啟用
- **除非必要，否則不要創建引擎** - 更多的引擎會增加配置複雜性，如果它們都做同樣的事情，則沒有好處
- **按引擎審核分析** - 搜索分析儀表板可以按引擎過濾，以查看哪些引擎被使用最多
