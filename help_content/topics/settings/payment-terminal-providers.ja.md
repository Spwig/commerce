---
title: 支払い端末プロバイダ
---

支払い端末プロバイダは、POS端末でクレジットおよびデビットカードの受け入れを可能にします。Stripe Terminal は、主にサポートされているプロバイダであり、現代的なカードリーダー（S700、WisePOS E、P400）、競争力のある処理手数料、およびシームレスな統合を提供しています。API資格情報を使用してプロバイダアカウントを構成し、リアルタイムで接続状態を監視し、複数のプロバイダを管理する必要がある場合があります（異なる地域で運用している場合）。プロバイダシステムは拡張可能です—Stripe Terminal が市場で動作しない場合、プロバイダフレームワークを通じて追加の支払いプロセッサを統合できます。

支払いプロバイダを使用して、カード支払いを安全に受け入れ、支払い処理の状態を追跡し、端末間のリーダーの割り当てを管理します。

![支払いプロバイダ一覧](/static/core/admin/img/help/payment-terminal-providers/provider-list.webp)

## 支払いプロバイダ概要

支払いプロバイダは、あなたのビジネスの代わりにカード支払いを処理する第三者サービスです：

**プロバイダの責任**:
- リアルタイムでカード取引を承認する
- 物理的なカードリーダーと通信する
- 支払いセキュリティ（PCI準拠、暗号化）を処理する
- あなたの銀行口座に資金を移転する（決済）
- トランザクションレポートと紛争管理を提供する

**Spwigの役割**:
- 設定されたプロバイダに支払いリクエストをルーティングする
- 暗号化されたプロバイダ資格情報を保存する
- 接続状態を監視する
- リーダーを端末に関連付ける
- 注文に支払い結果を記録する

## Stripe Terminal（主なプロバイダ）

Stripe Terminal は、ほとんどの小売業者にとって推奨される支払いプロバイダです：

**機能**:
- 現代的なEMVチップカードリーダー
- マイクロペイメント（NFC）のサポート（Apple Pay、Google Pay、タップして支払いカード）
- 紛争管理の統合
- リアルタイム承認
- 開発者向けのAPI
- 40か国以上で利用可能

**料金**（2024年時点、現在の料金を確認してください）:
- トランザクション手数料：対面取引ごとに2.7％＋0.05ドル（米国）
- 月額料金なし、設定料金なし、PCI準拠料金なし
- カードリーダーハードウェア：1回限りの購入（モデルによって59ドル〜299ドル）

**サポート地域**:
- アメリカ合衆国、カナダ、イギリス、ヨーロッパ連合、オーストラリア、シンガポール、その他
- Stripeの利用可能性を確認する：https://stripe.com/terminal

**サポートされているリーダー**:
- BBPOS WisePOS E（すべてを備えたAndroid端末）
- Stripe Reader S700（カウンタートップリーダー）
- Verifone P400（レガシーリーダー、依然としてサポートされています）

## Stripe Terminalの設定

**ステップ1：Stripeアカウントの作成**
- stripe.comでサインアップ
- ビジネス検証を完了（銀行口座、税ID）
- 支払いを有効にする

**ステップ2：Stripe Terminalの有効化**
- Stripeダッシュボードで、**Products > Terminal** に移動
- **Get Started** をクリック
- Terminalの利用規約を承諾する

**ステップ3：Locationの作成**
- Stripe Terminalは、あなたの物理的な小売サイトを表す「Location」が必要です
- **Terminal > Locations** に移動
- **Create Location** をクリック
- 店舗の住所と詳細を入力
- 位置IDを保存（`tml_1ABC123...` のような形式）

**ステップ4：APIキーの生成**
- **Developers > API Keys** に移動
- あなたの**Secret Key**（本番環境では`sk_live_...`で始まり、テスト環境では`sk_test_...`で始まる）を確認
- 秘密鍵をコピー（公開しないでください）

**ステップ5：Spwigでの設定**
- **POS > Payment Providers** に移動
- **+ Add Payment Provider** をクリック
- **Provider**：「Stripe Terminal」を選択
- **API Secret Key**（ステップ4から）を入力
- **Location ID**（ステップ3から）を入力
- 保存

**ステップ6：接続のテスト**
- 保存後、プロバイダのステータスは「Connected」（緑）に変更される必要があります
- ステータスが「Error」（赤）を示す場合、APIキーと位置IDを確認してください
- プロバイダの詳細ビューでエラーメッセージを確認してください

## プロバイダ設定フィールド

**Provider Key** - 支払いプロセッサを選択：
- **stripe_terminal** - Stripe Terminal（推奨）
- **manual** - 手動支払い入力（テスト用のみ、実際の処理なし）
- コンポーネントシステムを通じてインストールされた場合、追加のプロバイダが表示される可能性があります

**Credentials (Encrypted)** - API資格情報を含むJSON構造：
- 保存前に自動的に暗号化されます
- 保存後、平文で決して表示されません
- 例（Stripe Terminal）:
```json
{
  "api_key": "sk_live_ABC123...",
  "location_id": "tml_1ABC123..."
}
```

**Provider Settings** - その他の設定（プロバイダ固有）：
- ステートメント記述子（顧客のクレジットカード明細に表示されます）
- 自動キャプチャ（承認された支払いをすぐにキャプチャするか、手動キャプチャ）
- 通貨の上書き（プロバイダアカウントがストアの通貨と異なる場合）

**Connection Status** - リアルタイムステータスインジケータ：
- **Connected**（緑）- プロバイダに到達可能で、正しく構成されています
- **Error**（赤）- 接続失敗または資格情報が無効
- **Unknown**（グレー）- まだテストされていない（作成直後）

**Last Tested** - 最新の接続テストのタイムスタンプ
- トランザクションが処理されるたびに自動的に更新されます
- **Test Connection** 管理アクションを通じて手動でテストをトリガーできます

## 接続ステータスの監視

システムは、顧客が支払いを試みる前に問題を通知するためにプロバイダの接続性を監視します：

**自動テスト**:
- すべての支払いトランザクションが接続テストをトリガーします（必要性上）
- バックグラウンドジョブが6時間ごとに接続をテストします（予防的な監視）

**ステータスの意味**:

**Connected** - プロバイダAPIは到達可能で、資格情報は有効で、支払いを処理する準備ができています

**Error** - 一般的な原因：
- 無効なAPIキー（失効、期限切れ、または誤った）
- 無効なロケーションID（Stripeでロケーションが削除された、または間違ったIDが入力された）
- ネットワーク接続問題（ファイアウォールがStripe APIをブロック）
- Stripeサービスの障害（まれ）

**Unknown** - プロバイダがまだテストされていない（新しく作成されたアカウントで最初のトランザクションが待機中）

**Errorステータスの解決**:
1. プロバイダの詳細ビューでエラーメッセージを確認し、特定の問題を説明します
2. StripeダッシュボードでAPIキーが依然として有効であることを確認します
3. StripeダッシュボードでロケーションIDが依然として存在することを確認します
4. **Test Connection** 管理アクションを通じて接続を手動でテストします
5. 必要に応じて資格情報を更新します

## サポートされているカードリーダーの比較

Stripe Terminalは、複数のリーダーハードウェアオプションを提供しています：

| モデル | タイプ | 支払い方法 | 表示 | 最適な用途 | 価格 |
|-------|------|-----------------|---------|----------|-------|
| **WisePOS E** | すべてを備えた | EMVチップ、NFC、スワイプ | 5インチカラータッチスクリーン | フル機能の小売POS | ~$299 |
| **S700** | カウンタートップ | EMVチップ、NFC、スワイプ | モノクロLCD | 標準的な小売チェックアウト | ~$249 |
| **P400** | カウンタートップ | EMVチップ、NFC、スワイプ | モノクロLCD | レガシーデプロイメント | ~$299 |

**WisePOS Eの利点**:
- Androidベース（アプリを実行し、カスタムコンテンツを表示可能）
- カラータッチスクリーン（チップ提示、署名取得のUX向上）
- 結合されたレシートプリンター（オプション）
- 最も高速なトランザクション速度

**S700の利点**:
- WisePOS Eよりもコストが低い
- コンパクトな足場
- スプラッシュ耐性設計

**P400**（古いモデル）:
- 依然としてサポートされていますが、新しいデプロイメントには推奨されません
- S700/WisePOS Eよりもチップカード処理が遅い

すべてのリーダーは、Stripe Terminal APIを通じてSpwig POSに接続されます（POSデバイスへの直接のUSB/Bluetooth接続は必要ありません）。

## セキュリティ上の考慮事項

**資格情報の暗号化**:
- すべてのプロバイダ資格情報は、データベースで静的に暗号化されています
- 暗号化にはアプリケーションの秘密鍵（アプリケーション設定で定義）が使用されます
- 資格情報はログやエラーメッセージに決して表示されません

**APIキーの権限**:
- 本番環境では**制限付きAPIキー**を使用してください（Terminalのみへの権限を制限）
- 不制限の秘密鍵を使用しないでください（必要なアクセスよりも広範囲なアクセスはセキュリティリスク）
- Stripeダッシュボードで、**Terminal**のみの権限を持つ制限付きキーを作成してください

**PCI準拠**:
- Stripe TerminalはPCI準拠を処理します（カードデータはSpwigサーバーに触れません）
- カード番号はリーダーハードウェア→Stripeサーバー→カードネットワークで完全に処理されます
- Spwigは支払い結果（承認/拒否）のみを保存し、カードの詳細は保存しません

**キーの回転**:
- セキュリティのベストプラクティスとして、毎年APIキーを回転してください
- キーを回転する際、プロバイダ設定で資格情報を更新してください
- 新しいキーが動作することを確認した後、Stripeダッシュボードで古いキーを失効させることができます

## 複数のプロバイダ

一部の小売業者は複数のプロバイダアカウントが必要です：

**複数通貨の運用**:
- 米国店舗はStripe USアカウントを使用（USD処理）
- ヨーロッパ店舗はStripe EUアカウントを使用（EUR処理）
- 通貨ごとに個別のプロバイダを構成してください

**バックアッププロバイダ**:
- プライマリプロバイダ（Stripe Terminal）
- リーダーが故障した場合のバックアッププロバイダ（手動入力）
- 店員が支払いを開始する際にプロバイダを選択します

**テスト対本番**:
- テストプロバイダは`sk_test_...` APIキーを使用
- 本番プロバイダは`sk_live_...` APIキーを使用
- テストフェーズ後にプロバイダを切り替えてください

## 一般的な問題のトラブルシューティング

**問題1：ステータスが「Error」でメッセージ「Invalid API key」**
- **原因**：APIキーが失効または誤ってコピーされた
- **解決策**：Stripeダッシュボードで新しいAPIキーを生成し、プロバイダ資格情報を更新し、接続をテストしてください

**問題2：支払い中にリーダーが検出されない**
- **原因**：リーダーがプロバイダのロケーションに登録されていない
- **解決策**：Stripeダッシュボードで、リーダーがプロバイダ構成で使用されている同じロケーションIDに登録されていることを確認してください

**問題3：有効なカードにもかかわらず支払いが拒否される**
- **原因**：Stripeアカウントが完全にアクティブになっていない（検証が保留中）
- **解決策**：Stripeダッシュボードで銀行口座、税IDのビジネス検証を完了してください

**問題4：接続ステータスが「Unknown」で更新されない**
- **原因**：プロバイダがまだテストされていない（トランザクションが試行されていない）
- **解決策**：**Test Connection** 管理アクションを使用して、手動で接続性テストをトリガーしてください

## ヒント

- **本番環境前のテストモード** - StripeテストAPIキー（`sk_test_...`）を使用して初期設定とテストを実施してください
- **通貨ごとに1つのプロバイダ** - USDベースのStripeアカウントでEURを処理しないでください；個別のプロバイダを作成してください
- **週に一度接続ステータスを監視** - チェックアウトでの支払い失敗を防ぐための積極的な監視
- **Stripe APIキーの権限を制限** - Stripe APIキーをTerminalのみの権限に制限してください（最小特権の原則）
- **ロケーションIDを記録** - Stripeのロケーションがどの物理的なストアに該当するかを記録してください
- **リーダーの割り当てをテスト** - プロバイダ設定後、実際のカードリーダーを使用して支払いをテストし、エンドツーエンドフローを確認してください
- **Stripeの連絡先情報を更新** - Stripe内のビジネス連絡先情報を現在のものに保証してください（紛争、準拠のため重要）
