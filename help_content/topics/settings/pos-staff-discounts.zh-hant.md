---
title: POS員工折扣與終端機安全
---

POS員工折扣設定讓您可以控制每位員工在結帳時能應用的折扣幅度。終端機鎖定事件可提供每次終端機被鎖定或解鎖的審計記錄，幫助您追蹤誰曾經存取過終端機，以及是否有任何登入失敗的情況發生。

## 員工折扣限制

每位使用POS的員工都可以有個別的折扣權限。預設情況下，員工可以對商品或整個購物車應用最高10%的折扣。您可以根據個人調整這個限制，或指定某些員工為經理，讓他們可以批准超出標準限制的折扣。

### 設定員工的折扣限制

1. 點擊 **POS > 員工折扣**
2. 點擊 **+ 新增POS員工折扣** 或點擊現有員工以進行編輯
3. 從清單中選擇 **員工**
4. 設定折扣限制：

| 欄位 | 說明 |
|-------|-------------|
| **最大折扣百分比** | 這位員工可以應用的最大折扣百分比（例如，`10` 表示 10%） |
| **最大折扣金額** | 每筆交易的固定金額上限（留空表示無固定上限） |
| **可應用商品折扣** | 允許對個別商品項目應用折扣 |
| **可應用購物車折扣** | 允許對整個購物車總額應用折扣 |
| **需要說明** | 勾選後，員工在應用任何折扣前必須輸入說明 |

5. 點擊 **儲存**

### 在POS上折扣限制的運作方式

當收銀員嘗試應用折扣時：
- 如果折扣在其限制範圍內，將立即套用
- 如果折扣超出其限制，終端機會提示 **經理批准**
- 經理輸入其PIN以授權覆蓋，折扣將被套用

此流程可防止未經授權的高價值折扣，同時在需要真正折扣時提供靈活性。

## 經理角色

具有 **是經理** 標記的員工可以批准超出其他員工限制的折扣。

經理會在終端機上通過輸入PIN碼來識別，當需要批准時。

### 設置經理

1. 打開員工的折扣記錄
2. 勾選 **是經理**
3. 輸入 **經理PIN**（4-6位數字）— 保存時會安全地進行哈希處理
4. 點擊 **保存**

經理PIN與用於終端機鎖定/解鎖的收銀員PIN是分開的。經理可以同時擁有經理PIN（用於折扣批准）和收銀員PIN（用於終端機訪問）。

### 經理PIN安全性

當您在管理員表單中輸入PIN並保存時，Spwig會自動對其進行哈希處理 — 原始PIN從來不會被存儲。保存後，原始PIN欄位會自動清除，這是預期的行為。

## 收銀員PIN與卡片訪問

每個員工還可以設置 **收銀員PIN** 來鎖定和解鎖終端機：

- **收銀員PIN** — 用於在終端機自動鎖定或手動鎖定後解鎖的4-6位數字PIN
- **卡片識別碼** — 也可以使用已註冊的卡片（刷卡卡或NFC）來解鎖終端機

要設置收銀員PIN，請在 **收銀員PIN** 欄位中輸入並保存。與經理PIN一樣，保存時會自動進行哈希處理。

## 終端機鎖定事件

每次終端機被鎖定或解鎖時，Spwig都會記錄一個終端機鎖定事件。這會創建一個完整的安全審計軌跡。

### 查看鎖定事件

導航至 **POS > 終端機鎖定事件** 以查看完整的歷史記錄。您可以按以下條件過濾事件：
- 終端機
- 事件類型
- 日期範圍

### 事件類型

| 事件 | 含義 |
|-------|---------|
| **手動鎖定** | 員工刻意鎖定了終端機 |
| **自動鎖定（閒置逾時）** | 終端機因閒置而自動鎖定 |
| ** cashier 解鎖** | cashier 驗證並解鎖了終端機 |
| **經理解鎖** | 經理使用其憑證解鎖 |
| **刷卡解鎖** | 終端機使用已註冊的刷卡進行解鎖 |
| **生物解鎖** | 終端機使用指紋或面部識別進行解鎖 |
| **解鎖失敗** | 使用錯誤憑證嘗試解鎖 |
| **鎖定（3次以上失敗）** | 終端機在多次失敗後被鎖定 |

### 什麼鎖事件記錄包含

每個事件記錄：
- 涉及的 **終端機**
- **事件類型**
- 執行操作的人 (**Performed By**) 以及鎖定時登入的用戶 (**Locked By**)
- 是否使用了 **經理覆蓋**
- **解鎖方式**（PIN、卡片或生物識別）
- 此事件之前的 **失敗嘗試**（有助於發現暴力破解模式）
- 事件發生時的 **購物車總額** 和商品數量
- 請求的 IP 地址

### 調查安全問題

如果您懷疑有未經授權的終端機訪問：

1. 進入 **POS > 終端機鎖定事件**
2. 透過相關終端機過濾
3. 查看類型為 **解鎖失敗** 或 **鎖定** 的事件 — 這些表示重複的訪問失敗
4. 檢查成功解鎖的 **Performed By** 欄位，以查看誰獲得了訪問權
5. 與班次記錄 (**POS > 班次**) 交叉核對，以確認當時應該在崗的 cashier

## 小提示

- 根據員工資歷設定折扣限制 — 新員工可能從 5% 開始，有經驗的員工為 10-15%，經理可以批准更高的折扣。
- 對於任何享有更高折扣限制的員工，啟用 **需要原因**。

# 保留原因記錄

保留原因記錄可幫助您分析折扣模式並識別任何濫用情況。
- 如果您的商店有多位員工或員工流動率較高，請每週審核終端鎖定事件 — 在問題變得嚴重之前，更容易發現不規則的訪問模式。
- 如果員工離職，請立即刪除其收銀員 PIN 和卡片識別碼，以防止終端訪問。
- 使用鎖定事件來識別可能需要調整自動鎖定超時的終端 — 如果客戶經常觸發意外鎖定，閒置超時可能設置得太短。
- 管理員 PIN 應定期更改。

在員工折扣記錄中更新它們 — 新的 PIN 在保存時會被哈希處理。
