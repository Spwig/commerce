---
title: 安裝指南
---

本指南將引導您在自己的伺服器上安裝 Spwig。整個過程是自動化的 —— 單一命令即可處理 Docker 設定、資料庫建立、服務配置和 SSL 證書。

## 安裝前的準備

您需要：

- 執行 **Ubuntu 22.04 或 24.04** 的伺服器（也支援 Debian 12）
- 伺服器的 **root 或 sudo 權限**
- 至少 **4 GB RAM** 和 **20 GB 硬碟空間**（建議使用 8 GB RAM）
- 您從 Spwig 購買獲得的 **授權令牌**（請查看您的電子郵件收據）
- 可選，一個指向您伺服器 IP 位址的 **網域名稱**

> **提示：** 您可以在沒有網域名稱的情況下進行安裝，之後使用網域名稱設定工具添加一個。在這段時間內，您的商店將透過伺服器的 IP 位址進行訪問。

## 執行安裝程式

透過 SSH 連接到您的伺服器，並執行來自購買確認電子郵件的安裝命令。它看起來像這樣：

```bash
curl -fsSL https://spwig.com/install.sh | sudo bash -s -- --token YOUR_LICENSE_TOKEN
```

將 `YOUR_LICENSE_TOKEN` 替換為您電子郵件中的令牌。

安裝程式會自動執行八個階段：

1. **預飛行檢查** —— 驗證您的伺服器是否符合要求（作業系統、硬碟、RAM、埠）
2. **令牌驗證** —— 確認您的授權並提取您的商店配置
3. **模式檢測** —— 確定適合您伺服器的最佳安裝模式（請參閱下文）
4. **配置** —— 生成安全密碼、資料庫憑證和服務配置
5. **圖像下載** —— 從倉庫中拉取 Spwig 應用程式圖像
6. **服務啟動** —— 按順序啟動資料庫、快取、應用程式和背景工作人員
7. **SSL 設定** —— 如果您配置了網域名稱，則會取得 SSL 證書
8. **最終化** —— 建立您的管理員帳號並生成便利腳本

根據您的伺服器網際網路速度，這個過程需要 5–15 分鐘。

## 安裝模式

安裝程式會自動檢測您的伺服器環境並選擇最佳模式。

您也可以使用 `--mode` 标志手动指定一个模式。

### 独立模式

**最适合：** 专用服务器和 VPS 实例，其中 Spwig 是唯一的网络应用程序。

- 直接使用 80 和 443 端口
- 通过 Let's Encrypt 自动处理 SSL 证书
- 这是最常见且推荐的模式

### 附加组件模式

**最适合：** 已在 80/443 端口上运行其他网络应用程序（如 WordPress 或公司网站）的服务器。

- Spwig 在备用端口上运行（自动检测，通常为 8080 或 8443）
- 安装程序会为您生成一个 nginx 代理配置块，供您添加到现有的网络服务器中
- 您的现有网络服务器处理 SSL 并将流量代理到 Spwig

### 本地模式

**最适合：** 在您自己的计算机上进行开发和测试。

- 仅可通过 `localhost` 或 `127.0.0.1` 访问
- 使用自签名 SSL 证书（您的浏览器会显示安全警告 — 这是正常的）
- 启用调试功能
- 不需要许可证验证

## 安装过程中会发生什么

### Docker

如果尚未安装 Docker，安装程序会提供安装选项。Spwig 完全在 Docker 容器中运行 —— 除了 Docker 之外，不会在您的服务器操作系统上直接安装任何内容。

### 创建的服务

安装程序会创建以下服务：

| 服務 | 功能 |
|---------|---------|
| **資料庫** (PostgreSQL 16) | 儲存您商店的所有資料 — 產品、訂單、客戶、設定 |
| **快取** (Redis) | 提升頁面載入速度並管理後台任務佇列 |
| **連線池** (PgBouncer) | 高效管理資料庫連線 |
| **物件儲存** (MinIO) | 儲存上傳的圖片、檔案和媒體 |
| **應用程式** (Spwig) | 商店本身 — 管理面板和商店前端 |
| **網頁伺服器** (Nginx) | 借助壓縮和快取功能，將您的商店提供給訪客 |
| **後台工作程式** (Celery) | 處理電子郵件、翻譯、分析和其他後台任務 |
| **任務排程器** (Celery Beat) | 執行自動備份和電子郵件營銷等排程任務 |
| **翻譯器** | 多語言商店的 AI 驅動翻譯服務 |
| **升級器** | 從 Spwig 市場處理元件更新 |

### 管理員帳號

在安裝結束時，您會被提示創建一個管理員帳號。這就是您用來登入商店管理面板的帳號。

### 維護模式

您的商店會以 **維護模式** 啟動 — 訪客會看到「即將推出」頁面。這讓您有時間配置商店（添加產品、設置支付方式、自定義主題）再上線。

當您準備好時，執行安裝程式創建的便利腳本：

```bash
./go-live.sh
```

或從 **管理員 > 商店設定 > 維護** 中停用維護模式。

## 安裝後

一旦安裝程式完成，您將看到包含以下內容的摘要：

- 您商店的 URL
- 您管理面板的 URL（通常為 `https://yourdomain.com/en/admin/`）
- 您配置文件的位置
- 可用的便利腳本

### 便利腳本

安裝程式會在您的安裝目錄中創建這些腳本：

- **`./go-live.sh`** — 將您的商店從維護模式中移除
- **`./configure-domain.sh`** — 添加或更改您的域名並取得 SSL 證書

### 下一步

1.

登入你的管理面板
2.

完成 **設定嚮導** — 它會引導你設定商店名稱、貨幣、時區和基本設定
3.

新增你的產品
4.

設定支付方式
5.

選擇並自訂一個主題
6.

準備好時執行 `./go-live.sh`

## 在雲端市場安裝

Spwig 可作為一個點擊應用程式在幾個雲端供應商上使用：

- **DigitalOcean** — 從 DigitalOcean 市場place 部署
- **Akamai (Linode)** — 從 Linode 市場place 部署
- **Vultr** — 從 Vultr 市場place 部署

這些市場place影像都預先載入了安裝程式。建立伺服器後，透過 SSH 登入並按照螢幕上的指示，使用你的授權令牌完成設定。

## 獲得幫助

如果安裝失敗或遇到錯誤：

1. 執行 **診斷工具**：`./doctor.sh`（在安裝期間建立）
2. 該診斷工具會檢查所有服務、連線性、SSL 和常見問題
3. 使用 `./doctor.sh --fix` 嘗試自動修復
4. 如果問題仍然存在，請將診斷工具的輸出結果傳送給 Spwig 支援團隊
