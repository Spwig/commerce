---
title: คู่มือการติดตั้ง
---

คู่มือนี้จะพาคุณผ่านขั้นตอนการติดตั้ง Spwig บนเซิร์ฟเวอร์ของคุณเอง กระบวนการทั้งหมดเป็นระบบอัตโนมัติ — คำสั่งเดียวสามารถจัดการการตั้งค่า Docker การสร้างฐานข้อมูล การกำหนดค่าบริการ และใบรับรอง SSL ได้

## ก่อนเริ่มต้น

คุณต้องมี:

- เซิร์ฟเวอร์ที่กำลังทำงานด้วย **Ubuntu 22.04 หรือ 24.04** (Debian 12 ยังรองรับ)
- **สิทธิ์ root หรือ sudo** สำหรับเซิร์ฟเวอร์
- อย่างน้อย **4 GB RAM** และ **20 GB พื้นที่จัดเก็บข้อมูล** (แนะนำให้ใช้ 8 GB RAM)
- **โทเค็นใบอนุญาต** จากการซื้อ Spwig ของคุณ (ตรวจสอบใบเสร็จอีเมลของคุณ)
- แบบเลือกใช้ **ชื่อโดเมน** ที่ชี้ไปยัง IP ของเซิร์ฟเวอร์ของคุณ

> **เคล็ดลับ:** คุณสามารถติดตั้งโดยไม่ใช้โดเมน และเพิ่มโดเมนภายหลังด้วยเครื่องมือกำหนดค่าโดเมน ร้านค้าของคุณจะสามารถเข้าถึงได้ผ่าน IP ของเซิร์ฟเวอร์ในช่วงเวลานั้น

## การเริ่มต้นติดตั้ง

เชื่อมต่อกับเซิร์ฟเวอร์ผ่าน SSH และรันคำสั่งติดตั้งจากอีเมลยืนยันการซื้อของคุณ ดูเหมือนนี้:

```bash
curl -fsSL https://spwig.com/install.sh | sudo bash -s -- --token YOUR_LICENSE_TOKEN
```

แทนที่ `YOUR_LICENSE_TOKEN` ด้วยโทเค็นจากอีเมลของคุณ

ตัวติดตั้งจะดำเนินการผ่าน 8 ขั้นตอนอัตโนมัติ:

1. **การตรวจสอบก่อนติดตั้ง** — ตรวจสอบว่าเซิร์ฟเวอร์ของคุณตรงตามข้อกำหนด (ระบบปฏิบัติการ, ดิสก์, RAM, พอร์ต)
2. **การตรวจสอบโทเค็น** — ยืนยันใบอนุญาตของคุณและดึงข้อมูลการกำหนดค่าร้านค้าของคุณ
3. **การตรวจจับโหมด** — กำหนดโหมดการติดตั้งที่เหมาะสมที่สุดสำหรับเซิร์ฟเวอร์ของคุณ (ดูข้อมูลด้านล่าง)
4. **การกำหนดค่า** — สร้างรหัสผ่านที่ปลอดภัย ข้อมูลการเข้าถึงฐานข้อมูล และการกำหนดค่าบริการ
5. **ดาวน์โหลดภาพ** — ดึงภาพแอปพลิเคชัน Spwig จาก registry
6. **การเริ่มต้นบริการ** — เริ่มฐานข้อมูล แคช แอปพลิเคชัน และงานพื้นหลังตามลำดับ
7. **การตั้งค่า SSL** — รับใบรับรอง SSL หากคุณมีโดเมนที่กำหนดไว้
8. **การสรุป** — สร้างบัญชีผู้ดูแลระบบของคุณและสร้างสคริปต์ที่สะดวก

กระบวนการใช้เวลา 5–15 นาที ขึ้นอยู่กับความเร็วอินเทอร์เน็ตของเซิร์ฟเวอร์ของคุณ

## โหมดการติดตั้ง

ตัวติดตั้งจะตรวจจับสภาพแวดล้อมของเซิร์ฟเวอร์ของคุณโดยอัตโนมัติและเลือกโหมดที่เหมาะสมที่สุด


คุณยังสามารถระบุได้ด้วยตนเองโดยใช้คำสั่ง `--mode` ด้วย

### โหมด Standalone

**เหมาะที่สุดสำหรับ:** เซิร์ฟเวอร์ที่ใช้เฉพาะและ VPS ที่ Spwig เป็นแอปเว็บเพียงตัวเดียว

- ใช้พอร์ต 80 และ 443 โดยตรง
- จัดการใบรับรอง SSL อัตโนมัติผ่าน Let's Encrypt
- นี่คือโหมดที่พบได้บ่อยที่สุดและแนะนำ

### โหมด Sidecar

**เหมาะที่สุดสำหรับ:** เซิร์ฟเวอร์ที่มีแอปเว็บอื่นกำลังทำงานอยู่แล้ว (เช่น WordPress, เว็บไซต์ของบริษัท ฯลฯ) บนพอร์ต 80/443

- Spwig จะทำงานบนพอร์ตอื่น (ตรวจจับอัตโนมัติ, โดยทั่วไปคือ 8080 หรือ 8443)
- ตัวติดตั้งจะสร้างบล็อกการกำหนดค่า nginx proxy ให้คุณเพื่อเพิ่มเข้ากับเซิร์ฟเวอร์เว็บที่มีอยู่ของคุณ
- เซิร์ฟเวอร์เว็บที่มีอยู่ของคุณจะจัดการ SSL และส่งผ่านการจราจรไปยัง Spwig

### โหมด Local

**เหมาะที่สุดสำหรับ:** การพัฒนาและการทดสอบบนคอมพิวเตอร์ของคุณเอง

- สามารถเข้าถึงได้เฉพาะที่ `localhost` หรือ `127.0.0.1`
- ใช้ใบรับรอง SSL ที่เซลฟ์-เซ็น (เบราว์เซอร์ของคุณจะแสดงข้อเตือนด้านความปลอดภัย — นี่เป็นเรื่องปกติ)
- เปิดใช้งานคุณสมบัติสำหรับการตรวจสอบข้อผิดพลาด
- ไม่จำเป็นต้องตรวจสอบการรับรองสิทธิ์

## สิ่งที่เกิดขึ้นระหว่างการติดตั้ง

### Docker

หาก Docker ยังไม่ได้ติดตั้งไว้แล้ว ตัวติดตั้งจะเสนอให้ติดตั้ง Docker ให้คุณ Spwig ทำงานทั้งหมดภายในคอนเทนเนอร์ Docker — ไม่มีสิ่งใดถูกติดตั้งโดยตรงบนระบบปฏิบัติการของเซิร์ฟเวอร์ของคุณนอกเหนือจาก Docker

### บริการที่สร้างขึ้น

ตัวติดตั้งสร้างบริการต่อไปนี้:

| บริการ | วัตถุประสงค์ |
|---------|---------|
| **ฐานข้อมูล** (PostgreSQL 16) | เก็บข้อมูลของร้านคุณทั้งหมด — สินค้า, คำสั่งซื้อ, ลูกค้า, การตั้งค่า |
| **แคช** (Redis) | เพิ่มความเร็วในการโหลดหน้าและจัดการคิวงานในพื้นหลัง |
| **ตัวจัดการการเชื่อมต่อ** (PgBouncer) | จัดการการเชื่อมต่อกับฐานข้อมูลอย่างมีประสิทธิภาพ |
| **การจัดเก็บข้อมูล** (MinIO) | เก็บรูปภาพ, ไฟล์, และสื่อที่อัปโหลด |
| **แอปพลิเคชัน** (Spwig) | ร้านคุณเอง — แดชบอร์ดผู้ดูแลระบบและหน้าร้าน |
| **เซิร์ฟเวอร์เว็บ** (Nginx) | ส่งข้อมูลไปยังผู้เยี่ยมชมโดยใช้การบีบอัดและแคช |
| **ตัวประมวลผลงานพื้นหลัง** (Celery) | ประมวลผลอีเมล, การแปล, การวิเคราะห์, และงานพื้นหลังอื่น ๆ |
| **ตัวจัดตารางงาน** (Celery Beat) | ทำงานที่กำหนดเวลา เช่น การสำรองข้อมูลอัตโนมัติและแคมเปญอีเมล |
| **ตัวแปลภาษา** | บริการแปลภาษาที่ใช้ AI สำหรับร้านค้าหลายภาษา |
| **ตัวอัปเดต** | จัดการการอัปเดตส่วนประกอบจากตลาด Spwig |

### บัญชีผู้ดูแลระบบ

เมื่อการติดตั้งเสร็จสิ้น คุณจะถูกขอให้สร้างบัญชีผู้ดูแลระบบ นี่คือบัญชีที่คุณจะใช้เพื่อเข้าสู่แดชบอร์ดผู้ดูแลระบบของร้านคุณ

### โหมดบำรุงรักษา

ร้านคุณเริ่มต้นใน **โหมดบำรุงรักษา** — ผู้เยี่ยมชมจะเห็นหน้า "Coming Soon" นี่คือช่วงเวลาที่คุณสามารถตั้งค่าร้านคุณ (เพิ่มสินค้า, ตั้งค่าวิธีการชำระเงิน, ปรับแต่งธีม) ก่อนเปิดให้บริการ

เมื่อคุณพร้อมแล้ว ให้รันสคริปต์ที่ติดตั้งโดยตัวติดตั้ง:

```bash
./go-live.sh
```

หรือปิดโหมดบำรุงรักษาจาก **Admin > Store Settings > Maintenance**

## หลังการติดตั้ง

เมื่อตัวติดตั้งเสร็จสิ้น คุณจะเห็นสรุปที่มี:

- URL ของร้านคุณ
- URL ของแดชบอร์ดผู้ดูแลระบบ (โดยทั่วไปคือ `https://yourdomain.com/en/admin/`)
- ตำแหน่งของไฟล์การตั้งค่า
- สคริปต์ที่ใช้สะดวก

### สคริปต์ที่ใช้สะดวก

ตัวติดตั้งสร้างสคริปต์เหล่านี้ในไดเรกทอรีการติดตั้งของคุณ:

- **`./go-live.sh`** — นำร้านคุณออกจากการบำรุงรักษา
- **`./configure-domain.sh`** — เพิ่มหรือเปลี่ยนโดเมนและรับใบรับรอง SSL

### ขั้นตอนต่อไป

1.

เข้าสู่ระบบในแดชบอร์ดผู้ดูแลระบบ
2.

สมบูรณ์แบบ **Setup Wizard** — มันจะนำคุณผ่านการตั้งค่าพื้นฐาน เช่น ชื่อร้านค้า เหรียญเงิน เวลาท้องถิ่น และการตั้งค่าพื้นฐาน
3.

เพิ่มสินค้าของคุณ
4.

ตั้งค่าวิธีการชำระเงิน
5.

เลือกและปรับแต่งธีม
6.

รัน `./go-live.sh` เมื่อพร้อม

## การติดตั้งบนตลาดคลาวด์

Spwig มีให้ใช้งานเป็นแอปแบบคลิกเดียวบนผู้ให้บริการคลาวด์ต่าง ๆ:

- **DigitalOcean** — ติดตั้งจาก DigitalOcean Marketplace
- **Akamai (Linode)** — ติดตั้งจาก Linode Marketplace
- **Vultr** — ติดตั้งจาก Vultr Marketplace

ภาพถ่ายตลาดเหล่านี้มีตัวติดตั้งที่โหลดไว้แล้ว หลังจากสร้างเซิร์ฟเวอร์ ให้ SSH เข้าและทำตามคำแนะนำบนหน้าจอเพื่อสมบูรณ์การตั้งค่าด้วยโทเค็นใบอนุญาตของคุณ

## การขอความช่วยเหลือ

หากการติดตั้งล้มเหลวหรือคุณพบข้อผิดพลาด:

1. รันเครื่องมือ **diagnostic tool**: `./doctor.sh` (สร้างขึ้นระหว่างการติดตั้ง)
2. เครื่องมือ doctor จะตรวจสอบบริการทั้งหมด ความเชื่อมต่อ SSL และปัญหาทั่วไป
3. ใช้ `./doctor.sh --fix` เพื่อพยายามซ่อมแซมอัตโนมัติ
4. หากปัญหายังคงอยู่ ให้ติดต่อฝ่ายสนับสนุนของ Spwig พร้อมกับผลลัพธ์จากเครื่องมือ doctor
