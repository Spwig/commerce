---
title: 설치 가이드
---

이 가이드는 Spwig을 자신의 서버에 설치하는 방법을 안내합니다. 전체 과정은 자동화되어 있으며, 단일 명령어로 Docker 설정, 데이터베이스 생성, 서비스 구성 및 SSL 인증서 처리가 이루어집니다.

## 시작하기 전에

필요한 사항:

- **Ubuntu 22.04 또는 24.04**를 실행하는 서버 (Debian 12도 지원)
- 서버에 **Root 또는 sudo 액세스**가 필요합니다
- 최소 **4 GB RAM** 및 **20 GB 디스크 공간** (8 GB RAM 권장)
- Spwig 구매에서 받은 **라이선스 토큰** (이메일 영수증 확인)
- 선택적으로, 서버 IP 주소를 가리키는 **도메인 이름**

> **팁:** 도메인 없이 설치할 수 있으며, 나중에 도메인 구성 도구를 사용해 도메인을 추가할 수 있습니다. 이 기간 동안 가게는 서버의 IP 주소를 통해 접근할 수 있습니다.

## 설치 실행

SSH를 통해 서버에 연결하고 구매 확인 이메일에서 설치 명령어를 실행합니다. 다음과 같습니다:

```bash
curl -fsSL https://spwig.com/install.sh | sudo bash -s -- --token YOUR_LICENSE_TOKEN
```

`YOUR_LICENSE_TOKEN`을 이메일에서 받은 토큰으로 대체합니다.

설치 프로그램은 자동으로 8단계를 실행합니다:

1. **사전 점검** — 서버가 요구 사항(OS, 디스크, RAM, 포트)을 충족하는지 확인합니다
2. **토큰 검증** — 라이선스를 확인하고 가게 구성 정보를 추출합니다
3. **모드 감지** — 서버에 가장 적합한 설치 모드를 결정합니다 (아래 참조)
4. **설정** — 보안 비밀번호, 데이터베이스 자격 증명 및 서비스 설정을 생성합니다
5. **이미지 다운로드** — 레지스트리에서 Spwig 애플리케이션 이미지를 가져옵니다
6. **서비스 시작** — 데이터베이스, 캐시, 애플리케이션 및 백그라운드 워커를 순서대로 시작합니다
7. **SSL 설정** — 도메인을 구성한 경우 SSL 인증서를 취득합니다
8. **최종화** — 관리자 계정을 생성하고 편의 스크립트를 생성합니다

프로세스는 서버의 인터넷 속도에 따라 5~15분 정도 소요됩니다.

## 설치 모드

설치 프로그램은 자동으로 서버 환경을 감지하고 최적의 모드를 선택합니다.

수동으로 `--mode` 플래그를 사용하여 하나를 지정할 수도 있습니다.

### 독립 모드

**가장 적합한 경우:** Spwig이 유일한 웹 애플리케이션인 전용 서버 및 VPS 인스턴스.

- 80 및 443 포트를 직접 사용
- Let's Encrypt를 통해 SSL 인증서를 자동으로 처리
- 가장 일반적이고 권장되는 모드입니다.

### 사이드카 모드

**가장 적합한 경우:** 이미 80/443 포트에서 다른 웹 애플리케이션(예: WordPress, 회사 웹사이트 등)을 실행 중인 서버.

- Spwig은 대체 포트에서 실행됩니다(자동 감지, 일반적으로 8080 또는 8443)
- 설치 프로그램은 기존 웹 서버에 추가할 수 있는 nginx 프록시 구성 블록을 생성합니다
- 기존 웹 서버가 SSL을 처리하고 트래픽을 Spwig으로 프록시합니다

### 로컬 모드

**가장 적합한 경우:** 자신의 컴퓨터에서 개발 및 테스트.

- `localhost` 또는 `127.0.0.1`에서만 접근 가능
- 자체 서명된 SSL 인증서를 사용(브라우저에서 보안 경고가 표시됩니다 — 이는 정상입니다)
- 디버그 기능이 활성화됨
- 라이선스 검증이 필요하지 않음

## 설치 중에 발생하는 일

### Docker

Docker가 이미 설치되어 있지 않다면, 설치 프로그램은 대신 설치할 수 있습니다. Spwig은 Docker 컨테이너 내부에서 완전히 실행되며, Docker 외부에서는 서버 운영 체제에 아무것도 설치되지 않습니다.

### 생성된 서비스

설치 프로그램은 다음 서비스를 생성합니다:

| 서비스 | 목적 |
|---------|---------|
| **데이터베이스** (PostgreSQL 16) | 모든 상점 데이터를 저장합니다 — 제품, 주문, 고객, 설정 등 |
| **캐시** (Redis) | 페이지 로딩 속도를 빠르게 하고 백그라운드 작업 큐를 관리합니다 |
| **연결 풀러** (PgBouncer) | 데이터베이스 연결을 효율적으로 관리합니다 |
| **객체 저장소** (MinIO) | 업로드된 이미지, 파일 및 미디어를 저장합니다 |
| **애플리케이션** (Spwig) | 상점 자체 — 관리자 패널과 상점 프론트엔드 |
| **웹 서버** (Nginx) | 압축 및 캐싱을 통해 방문자에게 상점을 제공합니다 |
| **백그라운드 워커** (Celery) | 이메일, 번역, 분석 및 기타 백그라운드 작업을 처리합니다 |
| **작업 스케줄러** (Celery Beat) | 자동 백업 및 이메일 캠페인과 같은 예약된 작업을 실행합니다 |
| **번역기** | 다국어 상점을 위한 AI 기반 번역 서비스 |
| **업그레이더** | Spwig 마켓플레이스에서 구성 요소 업데이트를 처리합니다 |

### 관리자 계정

설치가 끝나면 관리자 계정을 생성하도록 요청됩니다. 이 계정은 상점의 관리자 패널에 로그인하는 데 사용할 것입니다.

### 유지보수 모드

상점은 **유지보수 모드**에서 시작됩니다 — 방문자는 "Coming Soon" 페이지를 보게 됩니다. 이는 상점을 공개하기 전에 상점을 구성할 시간을 줍니다(제품 추가, 결제 방법 설정, 테마 커스터마이징 등).

준비가 되면 설치 프로그램이 생성한 편의 스크립트를 실행합니다:

```bash
./go-live.sh
```

또는 **관리자 > 상점 설정 > 유지보수**에서 유지보수 모드를 비활성화할 수 있습니다.

## 설치 후

설치 프로그램이 완료되면 다음 요약 정보를 볼 수 있습니다:

- 상점의 URL
- 관리자 패널 URL(보통 `https://yourdomain.com/en/admin/`)
- 구성 파일의 위치
- 사용 가능한 편의 스크립트

### 편의 스크립트

설치 프로그램은 설치 디렉터리에 다음 스크립트를 생성합니다:

- **`./go-live.sh`** — 유지보수 모드에서 상점을 해제합니다
- **`./configure-domain.sh`** — 도메인을 추가하거나 변경하고 SSL 인증서를 취득합니다

## 클라우드 마켓플레이스에서 설치

Spwig은 여러 클라우드 제공업체에서 한 클릭 애플리케이션으로 제공됩니다:

- **DigitalOcean** — DigitalOcean 마켓플레이스에서 배포
- **Akamai (Linode)** — Linode 마켓플레이스에서 배포
- **Vultr** — Vultr 마켓플레이스에서 배포

이 마켓플레이스 이미지는 설치 프로그램이 사전에 로드되어 있습니다. 서버를 생성한 후 SSH로 접속하여 라이선스 토큰으로 설정을 완료하기 위해 화면에 표시된 지시사항을 따르세요.

## 도움 받기

설치가 실패하거나 오류가 발생하는 경우:

1. **진단 도구**를 실행하세요: `./doctor.sh` (설치 중 생성됨)
2. 진단 도구는 모든 서비스, 연결성, SSL 및 일반적인 문제를 확인합니다
3. `./doctor.sh --fix`를 사용하여 자동 수리 시도
4. 문제가 지속되면 Spwig 지원팀에 진단 도구의 출력 결과를 함께 제공하세요
