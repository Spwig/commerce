---
title: インストールガイド
---

このガイドでは、独自のサーバーにSpwigをインストールする手順を説明します。このプロセスはすべて自動化されており、単一のコマンドでDockerの設定、データベースの作成、サービスの構成、SSL証明書の取得が行われます。

## はじめに

必要なもの:

- **Ubuntu 22.04 または 24.04** で動作するサーバー（Debian 12もサポート）
- サーバーへの **rootまたはsudoアクセス権**
- **4 GB RAM** と **20 GBディスク容量**（8 GB RAMが推奨）
- Spwig購入から取得した **ライセンストークン**（メールの領収書を確認）
- オプション: サーバーのIPアドレスを指す **ドメイン名**

> **ヒント:** ドメインなしでインストールすることもでき、後でドメイン設定ツールを使ってドメインを追加できます。その間、ストアはサーバーのIPアドレスを通じてアクセス可能です。

## インストーラーの実行

SSHでサーバーに接続し、購入確認メールからインストールコマンドを実行してください。コマンドは以下のようになります:

```bash
curl -fsSL https://spwig.com/install.sh | sudo bash -s -- --token YOUR_LICENSE_TOKEN
```

`YOUR_LICENSE_TOKEN`をメールから取得したトークンに置き換えてください。

インストーラーは自動的に8つのフェーズを実行します:

1. **事前チェック** — サーバーが要件を満たしているかを確認します（OS、ディスク、RAM、ポート）
2. **トークン検証** — ライセンスを確認し、ストア構成を抽出します
3. **モード検出** — サーバーに最適なインストールモードを決定します（下記を参照）
4. **構成** — セキュアなパスワード、データベース資格情報、サービス構成を生成します
5. **イメージのダウンロード** — レジストリからSpwigアプリケーションイメージを取得します
6. **サービスの起動** — データベース、キャッシュ、アプリケーション、バックグラウンドワーカーを順番に起動します
7. **SSL設定** — ドメインが設定されている場合、SSL証明書を取得します
8. **最終処理** — 管理者アカウントを作成し、便利なスクリプトを生成します

プロセスには、サーバーのインターネット速度に応じて5〜15分かかります。

## インストールモード

インストーラーは自動的にサーバー環境を検出し、最適なモードを選択します。

手動で `--mode` フラグを使用して指定することもできます。

スタンドアロンモード

Spwig が唯一のウェブアプリケーションである専用サーバーおよび VPS インスタンス向けです。

サイドカー モード

既にポート 80/443 で別のウェブアプリケーション（WordPress、会社のウェブサイトなど）を実行しているサーバー向けです。

ローカル モード

自分のコンピュータで開発およびテストを行う場合に最適です。

インストール時の処理

| サービス | 目的 |
|---------|---------|
| **データベース** (PostgreSQL 16) | すべての店舗データを保存 — 商品、注文、顧客、設定 |
| **キャッシュ** (Redis) | ページの読み込みを高速化し、バックグラウンドタスクキューを管理 |
| **コネクションプーラー** (PgBouncer) | データベース接続を効率的に管理 |
| **オブジェクトストレージ** (MinIO) | アップロードされた画像、ファイル、メディアを保存 |
| **アプリケーション** (Spwig) | 店舗自体 — 管理パネルと販売サイト |
| **Webサーバー** (Nginx) | 圧縮とキャッシュを使用して訪問者に店舗を提供 |
| **バックグラウンドワーカー** (Celery) | メール、翻訳、分析、その他のバックグラウンドタスクを処理 |
| **タスクスケジューラ** (Celery Beat) | 自動バックアップやメールキャンペーンなどのスケジュールされたタスクを実行 |
| **翻訳者** | 多言語店舗用のAI搭載翻訳サービス |
| **アップグレーダー** | Spwigマーケットプレイスからコンポーネントの更新を処理 |

### 管理アカウント

インストールの最後に、管理アカウントを作成するよう求められます。これは、店舗の管理パネルにログインするために使用するアカウントです。

### 保守モード

店舗は**保守モード**で開始します — 訪問者は「Coming Soon（近日公開）」ページを表示します。これは、店舗を公開する前に設定（商品の追加、支払い方法の設定、テーマのカスタマイズ）を行う時間を与えてくれます。

準備ができた場合は、インストーラーが作成した便利なスクリプトを実行してください:

```bash
./go-live.sh
```

または、**管理 > 店舗設定 > 保守モード**から保守モードを無効にできます。

## インストール後の手順

インストーラーが終了すると、以下が表示されます:

- あなたの店舗のURL
- あなたの管理パネルのURL（通常は `https://yourdomain.com/en/admin/`）
- あなたの設定ファイルの場所
- 利便性スクリプト

### 利便性スクリプト

インストーラーは、インストールディレクトリにこれらのスクリプトを作成します:

- **`./go-live.sh`** — 保守モードから店舗を外す
- **`./configure-domain.sh`** — ドメインを追加または変更し、SSL証明書を取得

### 次のステップ

1.


管理パネルにログイン
2.

**セットアップウィザード**を完了してください — これはストア名、通貨、タイムゾーン、基本設定を通じてガイドします
3.

製品を追加
4.

支払い方法を構成
5.

準備ができたときに `./go-live.sh` を実行してください

## クラウドマーケットプレイスでのインストール

Spwig は、いくつかのクラウドプロバイダーでワンクリックアプリケーションとして利用可能です:

- **DigitalOcean** — DigitalOcean マーケットプレイスからデプロイ
- **Akamai (Linode)** — Linode マーケットプレイスからデプロイ
- **Vultr** — Vultr マーケットプレイスからデプロイ

これらのマーケットプレイスイメージには、インストーラーが事前にロードされています。サーバーを作成した後、SSH で接続し、ライセンストークンを使用してセットアップを完了するための画面表示の指示に従ってください。

## ヘルプの取得

インストールが失敗したか、エラーに遭遇した場合:

1. **診断ツール**を実行してください: `./doctor.sh` (インストール中に作成されます)
2. doctor はすべてのサービス、接続性、SSL、および一般的な問題をチェックします
3. 問題が解決しない場合は、`./doctor.sh --fix` を使用して自動修復を試してください
4. 問題が解決しない場合は、doctor の出力を添えて Spwig サポートに連絡してください
