---
title: 系统要求
---

Spwig 可以在大多数现代 Linux 服务器上运行。本页涵盖最低和推荐的规格，较小服务器上会发生什么，以及哪些云服务提供商表现良好。

## 最低要求

| 资源 | 最低 | 推荐 |
|----------|---------|-------------|
| **操作系统** | Ubuntu 22.04 LTS、Ubuntu 24.04 LTS 或 Debian 12 | Ubuntu 24.04 LTS |
| **RAM** | 4 GB | 8 GB 或更多 |
| **磁盘空间** | 20 GB | 40 GB 或更多 |
| **CPU** | 1 vCPU | 2+ vCPUs |
| **架构** | x86_64 (AMD64) | x86_64 |
| **网络** | 公共 IP 地址（用于独立模式） | 静态公共 IP |
| **端口** | 80 和 443（独立）或任何替代端口（侧车） | 80 和 443 |

> **注意：** 基于 ARM 的服务器（例如 AWS Graviton、Oracle Ampere）目前不支持。

## 资源层级

安装程序会自动检测服务器的可用 RAM 并选择适当的资源层级。

### 标准层级（6 GB+ RAM）

所有服务都以完整功能运行：

- 启用 AI 驱动的 **翻译服务** —— 通过您的管理面板直接将产品描述、页面内容和 SEO 文本翻译成多种语言
- 应用程序、数据库和后台工作程序的完整内存分配
- 后台工作程序的并发性针对您的 CPU 数量进行了优化

### 小型层级（4–6 GB RAM）

安装程序会进行调整以节省内存：

- **禁用** AI 翻译服务以节省约 2 GB 的 RAM。您仍然可以手动管理翻译或使用外部翻译工具 —— 仅内置的 AI 翻译器受到影响。
- 应用程序和工作程序的内存限制降低
- 所有其他功能与标准层级完全相同

> **提示：** 如果您从小型服务器开始，之后升级到 6 GB+ RAM，请重新运行安装程序以启用翻译服务。

## 磁盘空间使用

一个全新的 Spwig 安装大约使用 8 GB 的磁盘空间：

| 组件 | 大小 |
|-----------|------|
| Docker 镜像 | ~4 GB |
| 数据库（空商店） | ~200 MB |
| AI 翻译模型（如果启用） | ~2 GB |
| 应用程序和配置文件 | ~500 MB |
| 操作系统和 Docker 引擎 | ~3 GB |

为以下内容规划额外空间：

- **产品图片和媒体** — 取决于您的产品目录大小。对于一个拥有数百个产品的典型商店，预算 1–5 GB。
- **数据库增长** — 随着订单、客户和分析数据的增长而增长。一个每天处理 100 个订单的商店通常每年增长约 1 GB。
- **备份** — 如果本地存储备份，每个完整备份的大小大致等于您的数据库加上媒体。如果保留 30 天的备份，预算为当前数据大小的 2–3 倍。

## 域名和 DNS

在安装期间域名是可选的，但生产使用时是必需的。您需要：

- 一个域名或子域名（例如 `shop.example.com`）
- 一个 **A 记录** 指向您的服务器的公共 IP 地址
- DNS 传播完成（通常在添加记录后 5–60 分钟内完成）

当检测到有效的域名时，安装程序会自动从 Let's Encrypt 获取免费的 SSL 证书。您也可以在安装后使用 `./configure-domain.sh` 脚本添加域名。

## 防火墙

如果您的服务器有防火墙（大多数云提供商默认启用），请确保以下端口是开放的：

| 端口 | 协议 | 用途 |
|------|----------|---------|
| **22** | TCP | SSH 访问（用于您管理服务器） |
| **80** | TCP | HTTP（用于 Let's Encrypt 证书验证） |
| **443** | TCP | HTTPS（您的商店安全流量） |

在 sidecar 模式下，打开安装程序分配的替代端口，而不是 80/443。

## 软件前提条件

安装程序会自动处理所有软件安装。作为参考，这些是它安装或验证的组件：

- **Docker Engine** — 容器运行时（如果缺失，将自动安装）
- **Docker Compose** — 服务编排（包含在 Docker Engine 中）
- **curl** — 由安装程序本身使用（几乎所有 Linux 系统都自带）

不需要预先安装其他软件。Spwig 不需要您手动安装 Python、Node.js、PostgreSQL、Redis 或 Nginx — 一切都在 Docker 容器中运行。
