---
title: Руководство по установке
---

Это руководство поможет вам установить Spwig на вашем собственном сервере. Весь процесс автоматизирован — одна команда выполняет настройку Docker, создание базы данных, настройку служб и получение SSL-сертификатов.

## Перед началом

Вам понадобится:

- Сервер, на котором работает **Ubuntu 22.04 или 24.04** (также поддерживается Debian 12)
- **Доступ с правами root или sudo** к серверу
- Не менее **4 ГБ ОЗУ** и **20 ГБ дискового пространства** (рекомендуется 8 ГБ ОЗУ)
- **Лицензионный токен** из вашей покупки Spwig (проверьте электронный чек)
- Опционально, **доменное имя**, сопоставленное с IP-адресом вашего сервера

> **Совет:** Вы можете установить Spwig без доменного имени и добавить его позже с помощью инструмента настройки домена. Временно ваш магазин будет доступен через IP-адрес сервера.

## Запуск установщика

Подключитесь к своему серверу через SSH и выполните команду установки из письма подтверждения покупки. Она выглядит так:

```bash
curl -fsSL https://spwig.com/install.sh | sudo bash -s -- --token YOUR_LICENSE_TOKEN
```

Замените `YOUR_LICENSE_TOKEN` на токен из вашего электронного письма.

Установщик автоматически проходит через восемь этапов:

1. **Предварительные проверки** — проверяет, соответствует ли ваш сервер требованиям (ОС, диск, ОЗУ, порты)
2. **Проверка токена** — подтверждает вашу лицензию и извлекает настройки вашего магазина
3. **Определение режима** — определяет лучший режим установки для вашего сервера (см. ниже)
4. **Настройка** — генерирует безопасные пароли, учетные данные базы данных и настройки служб
5. **Загрузка образов** — загружает образы приложения Spwig из реестра
6. **Запуск служб** — запускает базу данных, кэш, приложение и фоновые рабочие процессы в указанном порядке
7. **Настройка SSL** — получает SSL-сертификат, если у вас настроен домен
8. **Завершение** — создает ваш аккаунт администратора и генерирует утилиты для удобства

Процесс занимает 5–15 минут в зависимости от скорости интернета вашего сервера.

## Режимы установки

Установщик автоматически определяет окружение вашего сервера и выбирает лучший режим. Вы также можете указать один вручную с помощью флага `--mode`.

### Режим Standalone

**Рекомендуется для:** Выделенных серверов и инстансов VPS, где Spwig — это единственное веб-приложение.

- Использует порты 80 и 443 напрямую
- Автоматически обрабатывает SSL-сертификаты через Let's Encrypt
- Это самый распространенный и рекомендуемый режим

### Режим Sidecar

**Рекомендуется для:** Серверов, которые уже запускают другое веб-приложение (WordPress, корпоративный сайт и т.д.) на портах 80/443.

- Spwig работает на альтернативном порту (автоматически определяется, обычно 8080 или 8443)
- Установщик генерирует блок конфигурации nginx, который вы должны добавить в существующий веб-сервер
- Ваш существующий веб-сервер обрабатывает SSL и проксирует трафик к Spwig

### Режим Local

**Рекомендуется для:** Разработки и тестирования на вашем собственном компьютере.

- Доступен только по адресу `localhost` или `127.0.0.1`
- Использует самоподписанный SSL-сертификат (браузер покажет предупреждение о безопасности — это нормально)
- Включены функции отладки
- Проверка лицензии не требуется

## Что происходит во время установки

### Docker

Если Docker еще не установлен, установщик предлагает установить его для вас. Spwig работает полностью внутри контейнеров Docker — ничего не устанавливается напрямую на операционную систему вашего сервера, кроме Docker.

### Созданные службы

Установщик создает следующие службы:

| Сервис | Назначение |
|---------|---------|
| **База данных** (PostgreSQL 16) | Хранит все данные вашего магазина — товары, заказы, клиентов, настройки |
| **Кэш** (Redis) | Ускоряет загрузку страниц и управляет очередями фоновых задач |
| **Коннектор** (PgBouncer) | Эффективно управляет соединениями с базой данных |
| **Объектное хранилище** (MinIO) | Хранит загруженные изображения, файлы и медиа |
| **Приложение** (Spwig) | Сам магазин — панель администратора и магазин |
| **Веб-сервер** (Nginx) | Обслуживает ваш магазин для посетителей с использованием сжатия и кэширования |
| **Фоновый рабочий процесс** (Celery) | Обрабатывает электронную почту, переводы, аналитику и другие фоновые задачи |
| **Планировщик задач** (Celery Beat) | Запускает запланированные задачи, такие как автоматические резервные копии и кампании по электронной почте |
| **Переводчик** | Сервис автоматического перевода для многоязычных магазинов |
| **Обновитель** | Обрабатывает обновления компонентов из магазина Spwig |

### Админ-аккаунт

В конце установки вас попросят создать аккаунт администратора. Это аккаунт, который вы будете использовать для входа в панель администратора вашего магазина.

### Режим обслуживания

Ваш магазин запускается в **режиме обслуживания** — посетители видят страницу «Скоро откроется». Это даёт вам время настройте магазин (добавьте товары, настройте методы оплаты, настройте тему) перед запуском.

Когда вы будете готовы, запустите удобный скрипт, созданный установщиком:

```bash
./go-live.sh
```

Или отключите режим обслуживания через **Админ > Настройки магазина > Режим обслуживания**.

## После установки

После завершения установщика вы увидите сводку с:

- URL вашего магазина
- URL панели администратора (обычно `https://yourdomain.com/en/admin/`)
- Расположение ваших файлов конфигурации
- Доступные удобные скрипты

### Удобные скрипты

Установщик создаёт эти скрипты в вашей директории установки:

- **`./go-live.sh`** — выводит ваш магазин из режима обслуживания
- **`./configure-domain.sh`** — добавляет или изменяет ваш домен и получает сертификат SSL

### Следующие шаги

1. Войдите в свою панель администратора
2. Завершите **Мастер настройки** — он проведёт вас через настройку названия магазина, валюты, часового пояса и базовых настроек
3. Добавьте свои товары
4. Настройте метод оплаты
5. Выберите и настройте тему
6. Запустите `./go-live.sh`, когда будете готовы

## Установка на облачных рынках

Spwig доступен как одно-клик приложение на нескольких облачных провайдерах:

- **DigitalOcean** — разверните из DigitalOcean Marketplace
- **Akamai (Linode)** — разверните из Linode Marketplace
- **Vultr** — разверните из Vultr Marketplace

Эти образы рынка поставляются с предустановленным установщиком. После создания сервера подключитесь по SSH и следуйте инструкциям на экране, чтобы завершить настройку с использованием вашего токена лицензии.

## Получение помощи

Если установка не удалась или вы столкнулись с ошибкой:

1. Запустите **инструмент диагностики**: `./doctor.sh` (созданный во время установки)
2. Инструмент проверяет все службы, соединение, SSL и распространённые проблемы
3. Используйте `./doctor.sh --fix`, чтобы попытаться автоматически исправить проблемы
4. Свяжитесь с поддержкой Spwig с выводом инструмента диагностики, если проблема сохраняется
