---
title: 安装指南
---

本指南将引导您在自己的服务器上安装 Spwig。整个过程是自动化的——一条命令即可处理 Docker 设置、数据库创建、服务配置和 SSL 证书。

## 安装前的准备

您需要：

- 运行 **Ubuntu 22.04 或 24.04** 的服务器（也支持 Debian 12）
- 服务器的 **root 或 sudo 权限**
- 至少 **4 GB RAM** 和 **20 GB 磁盘空间**（推荐 8 GB RAM）
- 您从 Spwig 购买获得的 **许可证密钥**（请查看您的电子邮件收据）
- 可选，一个指向您服务器 IP 地址的 **域名**

> **提示：** 您可以在没有域名的情况下进行安装，并稍后通过域名配置工具添加一个。在此期间，您的商店可通过服务器的 IP 地址访问。

## 运行安装程序

通过 SSH 连接到您的服务器，并运行从购买确认电子邮件中获得的安装命令。它看起来像这样：

```bash
curl -fsSL https://spwig.com/install.sh | sudo bash -s -- --token YOUR_LICENSE_TOKEN
```

将 `YOUR_LICENSE_TOKEN` 替换为您电子邮件中的密钥。

安装程序会自动运行八个阶段：

1. **预检** — 验证您的服务器是否满足要求（操作系统、磁盘、RAM、端口）
2. **令牌验证** — 确认您的许可证并提取您的商店配置
3. **模式检测** — 确定最适合您服务器的安装模式（见下文）
4. **配置** — 生成安全密码、数据库凭据和服务配置
5. **镜像下载** — 从注册表中拉取 Spwig 应用程序镜像
6. **服务启动** — 按顺序启动数据库、缓存、应用程序和后台工作进程
7. **SSL 配置** — 如果您配置了域名，则获取 SSL 证书
8. **最终化** — 创建您的管理员账户并生成便捷脚本

根据您的服务器网络速度，整个过程需要 5–15 分钟。

## 安装模式

安装程序会自动检测您的服务器环境并选择最佳模式。


你也可以通过 `--mode` 标志手动指定一个模式。

### 独立模式

**最适合：** 专用服务器和 VPS 实例，其中 Spwig 是唯一的网络应用程序。

- 直接使用 80 和 443 端口
- 通过 Let's Encrypt 自动处理 SSL 证书
- 这是最常见且推荐的模式

### 附加组件模式

**最适合：** 已经在 80/443 端口上运行其他网络应用程序（如 WordPress 或公司网站）的服务器。

- Spwig 在备用端口上运行（自动检测，通常为 8080 或 8443）
- 安装程序会为你生成一个 nginx 代理配置块，供你添加到现有的网络服务器中
- 你的现有网络服务器处理 SSL 并将流量代理到 Spwig

### 本地模式

**最适合：** 在你自己的计算机上进行开发和测试。

- 仅可通过 `localhost` 或 `127.0.0.1` 访问
- 使用自签名 SSL 证书（你的浏览器会显示安全警告 — 这是正常的）
- 启用调试功能
- 不需要许可证验证

## 安装过程中会发生什么

### Docker

如果尚未安装 Docker，安装程序会提供安装选项。Spwig 完全在 Docker 容器中运行 —— 除了 Docker 之外，不会在你的服务器操作系统上直接安装任何内容。

### 创建的服务

安装程序会创建以下服务：

| 服务 | 用途 |
|---------|---------|
| **数据库** (PostgreSQL 16) | 存储您商店的所有数据 —— 产品、订单、客户、设置 |
| **缓存** (Redis) | 加快页面加载速度并管理后台任务队列 |
| **连接池** (PgBouncer) | 高效管理数据库连接 |
| **对象存储** (MinIO) | 存储上传的图片、文件和媒体 |
| **应用** (Spwig) | 商店本身 —— 后台管理面板和商店前端 |
| **Web 服务器** (Nginx) | 通过压缩和缓存将您的商店提供给访客 |
| **后台工作器** (Celery) | 处理电子邮件、翻译、分析和其他后台任务 |
| **任务调度器** (Celery Beat) | 运行计划任务，如自动备份和电子邮件活动 |
| **翻译器** | 用于多语言商店的 AI 驱动翻译服务 |
| **升级器** | 从 Spwig 市场处理组件更新 |

### 管理员账户

在安装结束时，您将被提示创建一个管理员账户。您将使用此账户登录到商店的管理面板。

### 维护模式

您的商店默认处于 **维护模式** —— 访客会看到一个“即将上线”的页面。这为您提供了配置商店的时间（添加产品、设置支付方式、自定义主题）在正式上线之前。

当您准备就绪时，运行安装程序创建的便捷脚本：

```bash
./go-live.sh
```

或从 **管理员 > 商店设置 > 维护** 禁用维护模式。

## 安装后

一旦安装程序完成，您将看到一个摘要，其中包含：

- 您商店的 URL
- 您管理员面板的 URL（通常为 `https://yourdomain.com/en/admin/`）
- 您配置文件的位置
- 可用的便捷脚本

### 便捷脚本

安装程序会在您的安装目录中创建这些脚本：

- **`./go-live.sh`** —— 将您的商店从维护模式中移除
- **`./configure-domain.sh`** —— 添加或更改您的域名并获取 SSL 证书

### 下一步

1.

登录您的管理面板
2.

完成 **设置向导** —— 它将引导您完成商店名称、货币、时区和基本设置
3.

添加您的产品
4.

配置支付方式
5.

选择并自定义一个主题
6.

准备就绪时运行 `./go-live.sh`

## 在云市场安装

Spwig 可作为一键应用程序在多个云提供商上使用：

- **DigitalOcean** —— 从 DigitalOcean 市场部署
- **Akamai (Linode)** —— 从 Linode 市场部署
- **Vultr** —— 从 Vultr 市场部署

这些市场镜像都预装了安装程序。创建服务器后，通过 SSH 登录并按照屏幕上的说明使用您的许可证令牌完成设置。

## 获取帮助

如果安装失败或遇到错误：

1. 运行 **诊断工具**：`./doctor.sh`（在安装期间创建）
2. 诊断工具会检查所有服务、连接性、SSL 和常见问题
3. 使用 `./doctor.sh --fix` 尝试自动修复
4. 如果问题仍然存在，请将诊断工具的输出结果发送给 Spwig 支持团队
