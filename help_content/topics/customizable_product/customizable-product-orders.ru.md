---
title: Выполнение заказов на настраиваемые продукты
---

Когда клиент создает продукт и оформляет заказ, его дизайн фиксируется и сохраняется вместе с заказом. Это руководство объясняет, как пользовательские дизайны проходят через жизненный цикл заказа, и как получить печатные файлы, необходимые для выполнения заказа.

## Жизненный цикл дизайна

Дизайн клиента проходит несколько этапов от создания до выполнения:

### 1. Создание дизайна

Клиент использует визуальный редактор на сайте магазина, чтобы создать свой дизайн. По мере работы прогресс сохраняется автоматически в браузере. Зарегистрированные клиенты также могут сохранять дизайны в своем аккаунте для последующего редактирования.

### 2. Черновик дизайна

Когда клиент нажимает **Добавить в корзину**, текущее состояние дизайна сохраняется как **черновик дизайна**. Черновик включает в себя:

- Полное состояние холста для каждой поверхности (позиции элементов, текстовое содержание, загруженные изображения, клипарт, стили)
- Разбивку стоимости, показывающую все применимые сборы за дизайн
- Предварительные изображения в миниатюрах для каждой поверхности

Черновик связан с элементом корзины через уникальный токен. Это гарантирует, что точный дизайн, созданный клиентом, сохраняется, даже если он продолжит покупки перед оформлением заказа.

**Истечение срока действия черновика:** Черновики дизайнов автоматически истекают через 7 дней, если клиент не завершит заказ. Это предотвращает накопление заброшенных дизайнов.

### 3. Снимок дизайна

Когда клиент завершает оформление заказа, черновик дизайна преобразуется в **неизменный снимок дизайна**. Это постоянная запись дизайна:

- Снимок не может быть изменен клиентом после покупки
- Он содержит ту же самую дизайнерскую информацию, что и черновик
- Он постоянно связан с конкретным элементом заказа

Эта неизменность важна — она гарантирует, что то, что клиент заказал, точно соответствует тому, что вы производите и отправляете, без возможности изменений после оплаты.

### 4. Генерация файлов для выполнения заказа

После оформления заказа система автоматически генерирует **высококачественные файлы для выполнения заказа** для каждой поверхности дизайна. Это составные изображения, объединяющие все элементы дизайна (текст, изображения, клипарт) в один печатный файл с разрешением DPI, настроенным для каждой поверхности.

Генерация происходит асинхронно в фоновом режиме. Для большинства дизайнов генерация завершается в течение нескольких секунд. Статус **Отрендерено** снимка указывает, готовы ли файлы для выполнения заказа.

## Получение данных о дизайне в заказах

### Страница деталей заказа

Когда вы просматриваете заказ, содержащий настраиваемые продукты, в панели администратора:

1. Перейдите в **Заказы > Все заказы**
2. Откройте заказ с настраиваемым продуктом
3. Элемент заказа с настраиваемым продуктом показывает информацию о дизайне, включая предварительные изображения поверхностей и ссылку на снимок дизайна

### Список снимков дизайна

Вы также можете просматривать все снимки дизайна напрямую:

1. Перейдите в **Настраиваемые продукты > Снимки дизайна**
2. Список показывает все снимки, связанные с элементами заказа
3. Нажмите на снимок, чтобы просмотреть полные данные о дизайне, отрендеренные изображения и файлы для выполнения заказа

Каждый снимок показывает:

| Параметр | Описание |
|---------|----------|
| **Элемент заказа** | Ссылка на связанный элемент заказа |
| **Дизайнерские данные** | Полное состояние холста (JSON) |
| **Отрендеренные изображения** | Предварительные изображения поверхностей |
| **Файлы для выполнения заказа** | Высококачественные составные файлы для печати |
| **Отрендерено** | Готово ли отображение |
| **Время завершения отображения** | Временная метка, когда были сгенерированы файлы |

## Загрузка файлов для выполнения заказа

Файлы для выполнения заказа — это те файлы, которые вы отправляете своему печатному поставщику или используете в своем производственном процессе.

**Для заказа на настраиваемую футболку:**
- Скачайте файл поверхности **Спереди** (например, составное изображение PNG с разрешением 300 DPI)
- Скачайте файл поверхности **Сзади**
- Скачайте файл поверхности **Рукав** (если был создан)
- Отправьте все файлы своему печатному оператору или принтеру DTG (напрямую на одежду)



**Для заказа на собственную постерную печать:**
- Скачайте файл **Front** (передняя сторона) в печатном разрешении
- Файл включает зону обрезки, если она была настроена для этой стороны
- Отправьте его в вашу типографию для постеров/карт

Каждый файл представляет собой одну составную изображение, содержащее все элементы дизайна, объединенные вместе, отрендеренные в DPI, которое вы настроили для этой стороны.

## Сохраненные дизайны

Зарегистрированные клиенты могут сохранять свои дизайны в своем аккаунте для последующего редактирования. Как продавец, вы можете просматривать эти сохраненные дизайны в списке только для чтения:

1. Перейдите в **Customizable Products > Saved Designs**
2. Список показывает все дизайны, сохраненные клиентами, с именем клиента, продуктом, названием дизайна и датой

Сохраненные дизайны:
- **Владеют клиентом** — Они принадлежат аккаунту клиента
- **Только для чтения для продавцов** — Вы можете просматривать их, но не можете их изменять
- **Отдельно от заказов** — Сохраненный дизайн становится заказом только тогда, когда клиент добавляет его в корзину и оформляет заказ
- **Можно повторно использовать** — Клиенты могут загрузить сохраненный дизайн, изменить его и оформить заказ несколько раз

## Процесс выполнения заказов

### Стандартный процесс

1. **Получение заказа** — Заказ появляется в вашем списке заказов с персонализированными товарами
2. **Проверка рендеринга** — Проверьте, что снимок дизайна показывает **Rendered: Yes**. Если рендеринг еще не завершен, подождите несколько минут и обновите страницу
3. **Скачайте файлы** — Скачайте файл для выполнения заказа для каждой стороны с дизайном
4. **Проверьте качество** — Откройте файлы и убедитесь, что дизайн соответствует вашим стандартам печати (проверьте DPI, расположение элементов и читаемость текста)
5. **Отправьте на производство** — Передайте файлы вашему поставщику печати или команде по производству
6. **Отправьте и завершите** — После производства отправьте продукт клиенту и пометьте заказ как выполненный

### Пример выполнения заказа на футболку

1. Получен заказ: "Custom Team T-shirt" с дизайном на передней и задней стороне
2. Откройте заказ → посмотрите снимок дизайна
3. Скачайте `front.png` (300 DPI, 300x400 мм) и `back.png` (300 DPI, 300x400 мм)
4. Отправьте оба файла в ваш DTG-принтер с цветом изделия и размером из выбора варианта заказа
5. После печати и проверки качества отправьте клиенту

### Пример выполнения заказа на постер

1. Получен заказ: "Custom A4 Poster" с одной стороной с дизайном
2. Откройте заказ → посмотрите снимок дизайна
3. Скачайте `front.png` (300 DPI, 210x297 мм с зоной обрезки 3 мм)
4. Отправьте в вашу службу печати постеров
5. После печати и обрезки отправьте клиенту

## Решение проблем

**Проблема:** Снимок дизайна показывает "Rendered: No" и рендеринг еще не завершен

- **Причина:** Задача фона может не завершиться или все еще обрабатывается
- **Решение:** Подождите несколько минут. Если рендеринг не завершится, проверьте журналы фоновых задач. Вы также можете просмотреть данные дизайна напрямую в снимке, чтобы убедиться, что дизайн клиента сохранен

**Проблема:** Файл для выполнения выглядит хуже, чем ожидалось

- **Причина:** Клиент мог загрузить изображения низкого разрешения
- **Решение:** Проверьте настройки DPI стороны. Если были настроены предупреждения о минимальном DPI, клиент был предупрежден во время процесса создания дизайна. Для будущих продуктов рассмотрите увеличение минимального требования к DPI

**Проблема:** Клиент запрашивает изменения в своем дизайне после оформления заказа

- **Решение:** Снимки дизайна неизменны по конструкции. Если клиенту нужны изменения, он должен оформить новый заказ с обновленным дизайном. Если вы согласны сделать исключение, клиент может использовать свой сохраненный дизайн (если он сохранил его) как отправную точку для нового заказа

## Советы

- Всегда проверяйте, завершен ли рендеринг перед началом производства.

Проверьте поле **Rendered** на снимке дизайна.
- Сохраняйте настройки DPI, соответствующие вашему методу печати.

Более высокое DPI обеспечивает лучшее качество, но увеличивает размер файла. 300 DPI — стандарт для большинства профессиональных печатных продуктов.
- Поощряйте клиентов сохранять свои дизайны перед оформлением заказа.

Сохраните все форматирование markdown, пути к изображениям, блоки кода и технические термины.

# Если возникла проблема на производстве и заказ нужно перезаказать, сохраненный дизайн упрощает повторный заказ.
- Включите буфер в ваш график производства для персонализируемых продуктов.

В отличие от стандартных продуктов, каждый товар требует индивидуальной обработки файлов.
- Если вы обрабатываете большие объемы персонализируемых заказов, рассмотрите автоматизацию шага загрузки файлов, интегрируя API вашего поставщика печати.
