title: 미디어 라이브러리
---

미디어 라이브러리는 가게 전반에 걸쳐 사용되는 모든 이미지, 비디오, 3D 모델 및 파일을 관리하는 중심 허브입니다. 파일을 끌어다 놓아 업로드하고 폴더와 태그로 정리하며, 시스템이 이미지를 자동으로 최적화하여 빠른 로딩을 가능하게 합니다.

![미디어 갤러리](/static/core/admin/img/help/media-library/media-gallery.webp)

## 갤러리 인터페이스

사이드바에서 **미디어 라이브러리**로 이동하여 갤러리를 엽니다. 인터페이스는 세 가지 영역으로 구성되어 있습니다:

| 영역 | 위치 | 목적 |
|------|----------|---------|
| **업로드 영역** | 좌측 사이드바 상단 | 파일을 끌어다 놓아 업로드 (이미지, 비디오, 3D 모델 최대 100MB) |
| **폴더 및 태그** | 좌측 사이드바 하단 | 폴더를 탐색하고 태그로 필터링하며, 쓰레기통에 접근 |
| **미디어 그리드** | 메인 영역 | 모든 자산을 검색, 필터링, 탐색 및 관리 |

### 툴바 컨트롤

미디어 그리드 위의 툴바는 다음과 같은 기능을 제공합니다:

- **검색** — 제목, 대체 텍스트, 설명 또는 태그 이름으로 자산을 찾습니다
- **유형 필터** — 이미지, 비디오 또는 3D 모델만 표시
- **크기 필터** — 파일 크기로 필터링 (작은, 중간, 큰)
- **대량 작업** — 항목 선택, 세부 정보 편집, 선택 항목 삭제
- **보기 모드** — 그리드 (대형), 작은 그리드 또는 목록 보기 (세션 간 유지됨)

## 파일 업로드

좌측 사이드바의 **업로드** 영역에 하나 이상의 파일을 끌어다 놓거나, 영역을 클릭하여 파일 선택기를 엽니다.

### 지원되는 형식

| 유형 | 형식 |
|------|---------|
| **이미지** | JPEG, PNG, GIF, WebP, SVG, BMP, TIFF |
| **비디오** | MP4, WebM, MOV, MKV, AVI |
| **3D 모델** | GLB, glTF |

### 업로드 대기열

여러 파일을 업로드할 때, 업로드 대기열 관리자가 나타나며 다음을 표시합니다:

- 각 파일의 이름과 업로드 진행률 표시줄
- 동시 업로드 (성능을 위해 최대 2개)
- 업로드 후 파일이 최적화되는 처리 상태
- 개별 업로드를 취소하거나 완료된 항목을 지우는 옵션

대기열은 드래그 가능하며, 업로드가 완료될 동안 작업을 계속할 수 있도록 최소화할 수 있습니다.

## 자동 이미지 최적화

업로드한 모든 이미지는 자동으로 최적화됩니다:

- **WebP 변환** — 원본과 함께 WebP 버전이 생성됩니다 (품질 85%) 빠른 로딩을 위해
- **썸네일 생성** — 이미지 프리셋에 따라 여러 크기의 버전이 생성됩니다
- **EXIF 방향** — 이미지가 자동으로 올바른 방향으로 회전됩니다

### 시스템 이미지 프리셋

플랫폼에는 일반적인 사용 사례를 다루는 21개의 내장 프리셋이 포함되어 있습니다:

| 프리셋 | 크기 | 자르기 | 사용처 |
|--------|-----------|------|---------|
| **썸네일** | 150 x 150 | 커버 | 관리자 목록, 빠른 미리보기 |
| **작은** | 300 x 300 | 커버 | 작은 제품 카드 |
| **중간** | 600 x 600 | 포함 | 제품 카드, 블로그 썸네일 |
| **대형** | 1200 x 1200 | 포함 | 제품 상세 페이지 |
| **갤러리** | 800 x 800 | 포함 | 이미지 갤러리 |
| **헤로** | 1920 x 1080 | 커버 | 헤로 섹션, 페이지 배너 |
| **배너** | 1200 x 400 | 커버 | 프로모션 배너 |
| **카드** | 400 x 300 | 커버 | 기능 카드, 콘텐츠 카드 |
| **아바타** | 200 x 200 | 자르기 | 고객 및 직원 아바타 |
| **제품 목록** | 400 x 400 | 커버 | 제품 그리드 카드 |
| **제품 상세** | 1200 x 1200 | 커버 | 전체 제품 이미지 |
| **제품 썸네일** | 100 x 100 | 커버 | 변형 선택자, 미니 카트 |
| **카테고리 배너** | 1920 x 480 | 커버 | 카테고리 페이지 헤더 |
| **카테고리 썸네일** | 300 x 200 | 커버 | 카테고리 카드 |
| **로고 헤더** | 300 x 80 | 패드 | 사이트 헤더 로고 |
| **로고 푸터** | 200 x 60 | 패드 | 사이트 푸터 로고 |
| **이메일 로고** | 400 x 100 | 패드 | 이메일 템플릿 로고 |
| **로고 정사각형** | 160 x 160 | 패드 | 정사각형 로고 배치 |
| **브랜드 로고** | 200 x 100 | 패드 | 브랜드/파트너 로고 |
| **공지 배너** | 800 x 300 | 커버 | 공지 이미지 |
| **공지 배경** | 1200 x 800 | 커버 | 공지 배경 |

시스템 프리셋은 이름을 바꾸거나 삭제할 수 없습니다. 기본값으로 제공되지 않는 크기를 필요로 한다면 **미디어 라이브러리 > 이미지 크기 프리셋**에서 추가적인 사용자 정의 프리셋을 생성할 수 있습니다.

### 자르기 모드

| 모드 | 동작 |
|------|----------|
| **커버** | 전체 영역을 채우고 필요할 경우 가장자리를 자릅니다 — 카드 및 배너에 적합 |
| **포함** | 전체 이미지를 영역에 맞추고 필요할 경우 투명 공간을 추가합니다 — 제품 이미지에 적합 |
| **자르기** | 정확한 크기에 맞춰 중앙을 자릅니다 |
| **패드** | 이미지를 맞추고 패딩(투명, 흰색 또는 검은색)을 추가합니다 — 로고에 적합 |

## 파일 정리

### 폴더

폴더를 생성하여 미디어를 논리적인 그룹으로 정리합니다. 폴더는 임의의 깊이까지 중첩될 수 있습니다. 좌측 사이드바의 폴더를 클릭하여 해당 폴더에 있는 자산만 표시합니다. **모든 파일** 링크는 모든 자산을 표시합니다.

### 태그

자산에 태그를 추가하여 폴더 간 유연한 정리가 가능합니다. 태그는 좌측 사이드바의 클라우드에 나타납니다. 태그를 클릭하여 해당 태그로 자산을 필터링합니다. 자산은 여러 태그를 가질 수 있습니다.

### 검색

검색 바는 제목, 대체 텍스트, 설명 또는 태그 이름으로 자산을 찾습니다. 검색을 유형 및 크기 필터와 함께 사용하여 정확한 결과를 얻을 수 있습니다.

## 자산 세부 정보

자산을 클릭하여 대형 미리보기와 전체 메타데이터가 있는 세부 정보 뷰를 엽니다.

![자산 세부 정보](/static/core/admin/img/help/media-library/media-detail.webp)

세부 정보 뷰는 다음과 같은 항목을 표시합니다:

- **미리보기** — 원본 크기의 대형 이미지 미리보기
- **파일 정보** — 유형, 크기, 파일 크기, 업로드 날짜
- **편집을 위한 탭**:

| 탭 | 필드 |
|-----|--------|
| **일반** | 제목, 대체 텍스트, 설명 (다국어 가게의 경우 모두 번역 가능) |
| **기술** | MIME 유형, 파일 해시, 원본 파일 이름, WebP 버전 상태 |
| **정리** | 폴더 할당, 태그, 공개/비공개 전환 |
| **고급** | 초점 지점 좌표, 외부 ID, 메타데이터 JSON |

### 번역 가능한 필드

제목, 대체 텍스트 및 설명은 번역을 지원합니다. 각 필드 옆의 번역 아이콘을 클릭하여 활성화된 언어에 대한 번역을 추가합니다. 이는 이미지가 SEO 및 접근성에 적절하게 로컬화된 대체 텍스트 및 설명을 보장합니다.

### 사용 추적

시스템은 플랫폼 전반에서 각 자산이 사용되는 위치를 추적합니다. 하단의 **미디어 사용** 섹션은 이 자산을 참조하는 모든 모델 및 필드를 표시하여 변경 또는 삭제하기 전에 영향을 이해할 수 있도록 합니다.

## 비디오 지원

미디어 라이브러리에 업로드된 비디오는 자동으로 분석됩니다:

- **메타데이터 추출** — 길이, 해상도, 프레임 속도, 비트 전송률 및 코덱이 수집됩니다
- **포스터 이미지** — 비디오에서 미리보기용 썸네일이 생성됩니다
- **스트리밍** — 비디오는 전체 파일을 다운로드하지 않고도 범위 요청을 지원하여 검색이 가능합니다
- **선택적 변환** — 비디오는 최적화된 WebM/AV1 형식으로 변환될 수 있어 더 빠른 전송이 가능합니다

## 쓰레기통

자산을 삭제하면 **쓰레기통**으로 이동하여 영구적으로 삭제되지 않고 보호됩니다. 이는 실수로 인한 삭제를 방지합니다.

| 작업 | 수행 내용 |
|--------|-------------|
| **삭제** | 자산을 쓰레기통으로 이동 (부드러운 삭제) |
| **복원** | 삭제된 자산을 원래 위치로 되돌립니다 |
| **영구 삭제** | 자산 및 모든 썸네일을 저장소에서 영구적으로 제거합니다 |
| **쓰레기통 비우기** | 쓰레기통에 있는 모든 항목을 영구적으로 삭제합니다 |

좌측 사이드바의 **쓰레기통**을 클릭하여 삭제된 자산을 보고 관리합니다.

## 미디어 라이브러리 사용처

미디어 라이브러리는 플랫폼 전반에 걸쳐 통합되어 있습니다:

| 기능 | 미디어 사용 방법 |
|---------|------------------|
| **제품 카탈로그** | 제품 이미지, 변형 이미지, 카테고리 배너 |
| **블로그** | 특징 이미지, CKEditor를 통한 콘텐츠 내 이미지 |
| **페이지 빌더** | 이미지 요소, 헤로 배경, 갤러리 구성 요소 |
| **헤더/푸터 빌더** | 로고 이미지, 배경 이미지 |
| **사이트 설정** | 사이트 로고 및 브라우저 아이콘 |
| **공지사항** | 공지 이미지 및 배경 |
| **CKEditor** | 모든 풍부한 텍스트 이미지 업로드는 미디어 라이브러리를 통해 이루어집니다 |
| **로열티 프로그램** | 보상 및 등급 이미지 |

이 기능들 중 하나에서 이미지를 선택할 때, 미디어 라이브러리 갤러리가 모달 창으로 열려서 쉽게 탐색 및 선택이 가능합니다.

## 팁

- **설명적인 제목과 대체 텍스트 사용** — 좋은 메타데이터는 SEO 및 접근성을 개선합니다. 시스템은 전반적인 가게에서 이미지 태그에 대체 텍스트를 사용합니다.
- **파일 업로드 전 폴더 구조 생성** — 많은 파일을 업로드하기 전에 폴더 구조 (예: 제품, 블로그, 배너, 로고)를 생성하세요. 진행하면서 정리하는 것이 나중에 재정리하는 것보다 훨씬 쉽습니다.
- **교차 범주를 위한 태그 사용** — "계절", "할인", 또는 "라이프스타일"과 같은 태그는 여러 폴더에 걸친 자산을 찾는 데 도움이 됩니다.
- **삭제 전 사용 추적 확인** — 사용 추적 섹션은 자산이 참조되는 위치를 표시합니다. 사용 중인 자산을 삭제하면 가게 전반에 걸쳐 이미지가 손상될 수 있습니다.
- **WebP 사용** — 자동 WebP 변환은 일반적으로 JPEG에 비해 파일 크기를 25-35% 줄이며, 시각적 품질 손실 없이 이루어집니다. 업로드 전에 이미지를 수동으로 변환할 필요가 없습니다.
- **사용자 정의 프리셋 생성** — 고유한 레이아웃이 특정 이미지 크기를 필요로 한다면, 사용자 정의 프리셋을 생성하여 이미지를 수동으로 리사이징하는 대신 사용하세요.

기억하세요: 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어는 보존 규칙에 따라 정확히 유지하세요.
