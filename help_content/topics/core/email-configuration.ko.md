---
title: 이메일 구성
---

이메일 구성은 상점이 거래 이메일을 보내는 방식을 제어합니다. 주문 확인, 배송 알림, 비밀번호 재설정 등이 포함됩니다. Spwig에는 내장된 SMTP 서버가 포함되어 있으며, 더 높은 전달률을 위해 외부 이메일 제공업체도 지원합니다.

![이메일 계정](/static/core/admin/img/help/email-configuration/email-accounts.webp)

## 사용 가능한 제공업체

| 제공업체 | 설명 |
|----------|-------------|
| **내장 SMTP** | Spwig에 포함된 무료, 자체 호스팅 이메일 서버. 자동 DKIM 서명. |
| **Gmail API** | Google OAuth 인증을 통해 Gmail 또는 Google Workspace 계정을 사용하여 이메일을 보냅니다. |
| **일반 SMTP** | SendGrid, Mailgun, Amazon SES 또는 자체 이메일 서버를 포함한 모든 SMTP 서버 연결. |

## 이메일 설정

**설정 > 이메일 계정**으로 이동하여 설정 마법사를 시작하려면 **이메일 계정 추가**를 클릭합니다.

### 단계 1: 제공업체 선택

이메일 제공업체를 선택합니다. 내장 SMTP 서버는 시작하는 데 가장 간단한 옵션입니다. 외부 계정이 필요하지 않습니다.

### 단계 2: 자격 증명 구성

선택한 제공업체의 자격 증명을 입력합니다:

- **내장 SMTP** — 자격 증명이 필요하지 않습니다. 서버는 Spwig 설치에 실행됩니다.
- **Gmail API** — Google OAuth를 통해 인증합니다. Google 계정으로 로그인하라는 메시지가 표시됩니다.
- **일반 SMTP** — SMTP 서버 주소, 포트, 사용자 이름 및 암호를 입력합니다.

### 단계 3: 발신자 구성

외부 이메일의 발신자 신원을 설정합니다:

- **From 이메일** — "From" 필드에 표시되는 이메일 주소 (예: orders@yourstore.com)
- **From 이름** — 이메일 주소 옆에 표시되는 표시 이름 (예: "Your Store Name")
- **Reply-To 이메일** — 고객의 답변이 전달되는 위치 (From 주소와 다를 수 있음)

### 단계 4: DNS 검증

도메인의 이메일 인증 레코드를 확인합니다. 마법사는 세 가지 DNS 레코드를 확인합니다:

| 레코드 | 목적 |
|--------|---------|
| **SPF** | 도메인 대신 이메일을 보내는 서버를 승인합니다 |
| **DKIM** | 이메일을 디지털 서명하여 조작되지 않았음을 입증합니다 |
| **DMARC** | SPF/DKIM 검사에 실패한 이메일에 대해 수신 서버에 무엇을 해야 하는지 말해줍니다 |

각 레코드에 대해 마법사는 다음을 표시합니다:
- **현재 상태** — 레코드가 올바르게 구성되었는지 여부
- **필요한 값** — 도메인 등록업체에서 추가해야 할 정확한 DNS 레코드
- **전파 상태** — 최근 변경 사항이 적용되었는지 여부 (DNS 변경은 최대 48시간까지 소요될 수 있음)

내장 SMTP 서버는 도메인을 위한 DKIM 키를 자동으로 생성합니다.

### 단계 5: 테스트 이메일 보내기

모든 것이 작동하는지 확인하기 위해 테스트 이메일을 보내세요:
1. 수신자 이메일 주소를 입력합니다
2. **테스트 보내기**를 클릭합니다
3. 테스트 메시지가 수신함에 표시되어 있는지 확인합니다
4. 이메일이 스팸 경고 없이 도착하는지 확인합니다

### 단계 6: 저장 및 활성화

설정을 저장하고 계정을 활성화합니다. 이 계정이 기본 이메일 계정이 되어야 한다면 **기본**으로 표시합니다.

## 이메일 템플릿

Spwig에는 모든 거래 이벤트에 대해 30개 이상의 이메일 템플릿이 포함되어 있습니다. **설정 > 이메일 템플릿**로 이동하여 관리합니다.

### 템플릿 유형

템플릿은 모든 상점 이벤트를 다룹니다:
- **주문 주기** — 확인, 처리, 배송, 전달, 취소
- **결제** — 영수증, 환불 확인, 결제 실패
- **고객 계정** — 환영, 비밀번호 재설정, 이메일 확인
- **선물 카드** — 배송, 잔액 알림
- **배송** — 추적 업데이트, 배송 확인
- **디지털 제품** — 다운로드 링크, 라이선스 키
- **마케팅** — 포기한 장바구니 복구, 리뷰 요청

### 템플릿 커스터마이징

1. 템플릿 목록으로 이동합니다
2. 편집하려는 템플릿을 클릭합니다
3. 제목 줄, 헤더, 본문 내용 및 푸터를 수정합니다
4. 동적 콘텐츠를 위해 템플릿 변수 (예: `{{ order.number }}`, `{{ customer.name }}`)를 사용합니다
5. 저장하기 전에 이메일 미리보기를 합니다

### 다국어 지원

이메일 템플릿은 여러 언어를 지원합니다:
- 각 템플릿은 모든 활성화된 상점 언어에 대한 번역을 가질 수 있습니다
- 시스템은 고객의 선호 언어로 이메일을 보냅니다
- **언어 대체 체인** — 번역이 없는 경우 시스템은 상점의 기본 언어로 대체합니다
- **AI 번역** 기능을 사용하여 템플릿을 다른 언어로 자동 번역합니다

### 템플릿 복제

시스템 템플릿의 맞춤 버전을 만들려면:
1. 수정하려는 템플릿을 엽니다
2. **템플릿 복제**를 클릭합니다
3. 복제된 버전을 편집합니다
4. 복제본은 원본 시스템 템플릿보다 우선순위가 높습니다

## 이메일 대기열

**설정 > 이메일 대기열**에서 외부 이메일을 모니터링합니다:

- **대기 중** — 보내기 위해 대기 중인 이메일
- **보내는 중** — 현재 전송 중
- **보냄** — 성공적으로 전달됨
- **실패** — 전달할 수 없음 (오류 세부 정보 포함)
- **반송** — 수신자의 메일 서버에 의해 거부됨

이메일을 클릭하여 수신자, 제목, 발송 시간 및 전달 상태를 포함한 전체 세부 정보를 확인합니다.

## 전달 추적

이메일 참여를 추적합니다:
- **열람** — 이메일을 열람한 수신자 수
- **클릭** — 이메일 내 링크 클릭
- **반송** — 하드 및 소프트 반송 추적
- **불만** — 수신자로부터의 스팸 보고

## 여러 계정

여러 이메일 계정을 구성할 수 있습니다:
- **기본 계정** — 모든 외부 이메일에 사용되며, 오버라이드되지 않는 한
- **대체** — 기본 계정이 실패할 경우 이메일이 대기열에 들어가 재시도합니다
- 다른 목적에 따라 다른 계정을 사용합니다 (예: 하나는 거래 이메일, 다른 하나는 마케팅)

## 팁

- 빠른 설정을 위해 **내장 SMTP** 서버부터 시작한 후, 더 높은 전송량이나 더 나은 전달률이 필요할 경우 외부 제공업체로 전환합니다.
- 항상 **SPF, DKIM 및 DMARC** 레코드를 구성하세요. 이 레코드가 없으면 이메일이 스팸 폴더에 더 많이 들어갈 가능성이 큽니다.
- 구성 변경 후 **테스트 이메일**을 보내어 전달이 작동하는지 확인합니다.
- 정기적으로 이메일 대기열을 확인하여 **실패** 또는 **반송**된 이메일을 확인하세요. 이는 전달 문제를 나타냅니다.
- **전문적인 발신자 주소** (예: orders@yourstore.com)를 사용하여, 무료 이메일 주소보다 신뢰도와 전달률이 더 좋습니다.
- 템플릿을 간결하게 유지하세요. 거래 이메일은 정보를 빠르게 전달해야 하며, 마케팅 뉴스레터가 되어서는 안 됩니다.
