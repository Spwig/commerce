---
title: API 令牌
---

API 令牌是安全密钥，允許外部服務和整合與您的商店進行通訊。當第三方服務或工具需要訪問您的商店數據或觸發操作時，它會在每次請求中發送 API 令牌，以便您的商店驗證請求是否已授權。您可以在管理員的 API 令牌部分創建和管理所有令牌。

## 當您需要 API 令牌時

您通常需要創建 API 令牌的情況包括：

- 連接需要讀取或寫入您商店的外部服務或自動化工具
- 設置需要驗證來電的 Webhook 接收器
- 為您的安裝配置 Spwig 帮助系统
- 使用 Spwig API 建立自定義整合
- 在您的 Spwig 商店與其他系統之間同步數據

每個整合應有其自己的令牌，這樣您可以撤銷某個服務的訪問權限而不影響其他服務。

## 令牌類型

創建令牌時，您選擇一個類型來描述其用途。類型僅供您參考，幫助您跟蹤每個令牌的作用。

| 類型 | 用途 |
|------|---------|
| **Help System** | 用於 Spwig 帮助文档系统 |
| **External Integration** | 第三方服務、自動化工具（例如 Zapier）或數據同步工具 |
| **Webhook** | Webhook 接收器或端點的驗證 |
| **Custom** | 任何不屬於上述類別的其他用途 |
| **Instance Sync** | Spwig 安裝之間或與外部 Spwig 服務之間的同步 |

## 創建 API 令牌

1. 進入 **Settings > API Tokens**
2. 點擊 **+ Add API Token**
3. 輸入一個 **Name**，清楚描述令牌的用途（例如，`Zapier Product Sync` 或 `Help System API`）
4. 選擇適當的 **Token Type**
5. 可選地添加 **Description**，提供更多關於整合的細節
6. 根據需要配置 **Active** 狀態、**Expiry Date** 和 **Allowed IPs**（請參閱下文）
7. 點擊 **Save**

保存後，完整令牌值會顯示在詳情頁面上。**立即複製它** — 由於安全原因，清單視圖中令牌會被遮蔽，離開此頁後無法再次取得完整的令牌值。

## 令牌值安全性

Spwig 只顯示完整的令牌值一次：即在您保存新令牌後立即顯示。之後，清單視圖只會顯示遮蔽版本（例如，`spw_••••••••••••••••••••3f8a`）。

如果您遺失了令牌值，將無法恢復。您需要刪除舊令牌並創建一個新令牌，然後更新使用該令牌的整合。

**切勿在電子郵件、聊天訊息或源代碼中分享令牌值。** 請將其視為密碼。

## 設置過期日期

**過期時間**欄位設置令牌停止工作的日期和時間。如果令牌不應過期，請留空。

過期日期適用於：

- 有固定截止日期的臨時整合
- 提供給第三方的令牌，您希望自動移除訪問權限
- 為高權限整合增加額外的安全層次

當令牌過期後，使用它的請求將被拒絕。您可以通過更新 **過期時間** 日期或創建一個替代令牌來延長訪問權限。

## 限制到特定 IP 地址

**允許的 IP** 欄位接受 IP 地址列表。當列表不為空時，令牌僅在請求來自這些地址之一時才有效。

例如，如果您的分析工具運行在 IP 為 `203.0.113.42` 的服務器上，添加該 IP 可確保即使令牌洩露，也無法從其他位置濫用。

請留空 **允許的 IP** 以允許來自任何 IP 地址的請求。

## 監控令牌使用情況

令牌列表顯示：

- **使用次數** — 令牌已被使用的總次數
- **最後使用時間** — 令牌最後一次用於發起請求的時間

這些欄位可幫助您識別未使用的令牌（可撤銷的候選對象），並發現異常活動。

使用次數的突然增加可能表示該令牌被除預期的整合外的其他人使用。

## 撤銷令牌

若要立即停止令牌的運作，但不刪除它：

1. 點擊令牌名稱
2. 取消勾選 **Active**
3. 儲存

該令牌仍會保留在您的清單中供參考，但會拒絕任何後續請求。當您在調查問題時需要臨時中止某個整合時，這很有幫助。

若要永久刪除令牌：

1. 在清單中選中其複選框
2. 從操作選單中選擇 **Delete selected API tokens**
3. 確認刪除

一旦刪除，令牌無法恢復。如果整合仍需要訪問權限，請創建一個新令牌並更新整合的配置。

## 示例：設置 Zapier 整合

**場景：** 您希望將商店連接到 Zapier 以自動化訂單通知。

| Field | Value |
|-------|-------|
| Name | `Zapier Order Automation` |
| Token Type | External Integration |
| Description | Used by Zapier to read new orders and trigger notifications |
| Active | Yes |
| Expires At | *(leave blank)* |
| Allowed IPs | *(leave blank — Zapier uses dynamic IPs)* |

保存後，複製完整的令牌值並粘貼到 Zapier 的 Spwig 整合設置中。

## 小技巧

保留所有 markdown 格式、圖片路徑、程式碼塊和技術術語。

- 為每個令牌指定一個清晰且具體的名稱 — 當你幾個月後進行故障排除時，`Shopify Sync v2` 比 `Token 3` 有用得多
- 為每個整合建立一個令牌 — 如果某個整合遭到入侵，你可以只撤銷該令牌，而不會影響到其他令牌
- 為用於一次性專案或臨時整合的令牌設定到期日期 — 這能降低遺忘的令牌持續處於啟用狀態的風險
- 每幾個月審查一次你的令牌清單，停用任何 **最後使用** 日期異常古老的令牌，因為這些可能屬於已經不再運行的整合
- 如果你懷疑某個令牌已經外洩，請立即停用它，建立一個替代令牌，並在重新啟用存取權之前更新受影響的整合
