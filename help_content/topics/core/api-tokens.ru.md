---
title: Токены API
---

Токены API — это безопасные ключи, которые позволяют внешним сервисам и интеграциям общаться со вашим магазином. Когда сторонний сервис или инструмент нуждается в доступе к данным вашего магазина или в выполнении действий, он отправляет токен API вместе с каждым запросом, чтобы ваш магазин мог проверить, авторизован ли запрос. Вы создаете и управляете всеми токенами в разделе **Токены API** вашего административного интерфейса.

## Когда вам понадобится токен API

Вы обычно будете создавать токен API, когда:

- Подключаете внешний сервис или инструмент автоматизации, который должен читать или записывать данные в ваш магазин
- Настраиваете получатель веб-хука, который должен аутентифицировать входящие вызовы
- Настраиваете Систему помощи Spwig для вашей установки
- Создаете пользовательскую интеграцию с использованием API Spwig
- Синхронизируете данные между вашим магазином Spwig и другим системой

Каждая интеграция должна иметь свой собственный токен, чтобы вы могли отозвать доступ для одного сервиса, не влияя на другие.

## Типы токенов

При создании токена вы выбираете тип, который описывает его назначение. Тип предназначен для вашей информации и помогает вам отслеживать, что делает каждый токен.

| Тип | Назначение |
|------|---------|
| **Система помощи** | Используется системой помощи Spwig |
| **Внешняя интеграция** | Сторонние сервисы, инструменты автоматизации (например, Zapier) или инструменты синхронизации данных |
| **Веб-хук** | Аутентификация для получателей веб-хуков или конечных точек |
| **Пользовательский** | Любой другой цели, которая не подходит в вышеуказанные категории |
| **Синхронизация экземпляров** | Синхронизация между установками Spwig или внешними сервисами Spwig |

## Создание токена API

1. Перейдите в **Настройки > Токены API**
2. Нажмите **+ Добавить токен API**
3. Введите **Название**, которое ясно описывает назначение токена (например, `Синхронизация продуктов Zapier` или `API Системы помощи`)
4. Выберите подходящий **Тип токена**
5. Дополнительно добавьте **Описание** с дополнительной информацией о интеграции
6. Настройте статус **Активен**, **Дата истечения срока действия** и **Разрешенные IP-адреса** по мере необходимости (см. ниже)
7. Нажмите **Сохранить**

После сохранения полное значение токена отображается на странице деталей. **Скопируйте его немедленно** — токен маскируется в представлении списка для обеспечения безопасности и не может быть снова полностью извлечен после того, как вы покинете эту страницу.

## Безопасность значения токена

Spwig показывает полное значение токена только один раз: сразу после сохранения нового токена. После этого в представлении списка отображается только маскированная версия (например, `spw_••••••••••••••••••••3f8a`).

Если вы потеряете значение токена, его невозможно восстановить. Вам придется удалить старый токен и создать новый, а затем обновить интеграцию, которая его использовала.

**Никогда не делитесь значениями токенов по электронной почте, в сообщениях в чате или в исходном коде.** Обрабатывайте их как пароли.

## Установка даты истечения срока действия

Поле **Истекает к** задает дату и время, после которых токен перестанет работать автоматически. Оставьте его пустым для токенов, которые не должны истекать.

Даты истечения срока действия полезны для:

- Временных интеграций с фиксированной датой окончания
- Токенов, переданных третьим лицам, где вы хотите автоматически удалить доступ
- Добавления дополнительного уровня безопасности для интеграций с высокими привилегиями

Когда токен истекает, запросы, использующие его, отвергаются. Вы можете продлить доступ, обновив дату **Истекает к** или создав заменяющий токен.

## Ограничение по конкретным IP-адресам

Поле **Разрешенные IP-адреса** принимает список IP-адресов. Когда список не пуст, токен работает только тогда, когда запрос поступает с одного из этих адресов.

Например, если ваш инструмент аналитики работает на сервере с IP-адресом `203.0.113.42`, добавление этого IP означает, что токен не может быть использован из любого другого места, даже если он утечет.

Оставьте поле **Разрешенные IP-адреса** пустым, чтобы разрешить запросы с любого IP-адреса.

## Мониторинг использования токенов

Список токенов показывает:

- **Количество использований** — общее количество раз, когда токен был использован
- **Последнее использование** — когда токен последний раз использовался для выполнения запроса

Эти поля помогают вам определить неиспользуемые токены (кандидаты на отзыв) и выявить неожиданную активность.

Резкое увеличение количества использований может указывать на то, что токен используется кем-то другим, кроме предназначенной интеграции.

## Отзыв токена

Чтобы немедленно остановить работу токена без его удаления:

1. Нажмите на имя токена
2. Снимите галочку с **Active**
3. Сохраните

Токен остается в вашем списке для справки, но отклоняется при любых последующих запросах. Это полезно, когда вам нужно временно приостановить интеграцию во время расследования проблемы.

Чтобы полностью удалить токен:

1. Выберите его чекбокс в списке
2. Выберите **Delete selected API tokens** в меню действий
3. Подтвердите удаление

После удаления токен нельзя восстановить. Если интеграция всё ещё нуждается в доступе, создайте новый токен и обновите настройки интеграции.

## Пример: настройка интеграции Zapier

**Сценарий:** Вы хотите подключить свой магазин к Zapier для автоматизации уведомлений об заказах.

| Field | Value |
|-------|-------|
| Name | `Zapier Order Automation` |
| Token Type | External Integration |
| Description | Used by Zapier to read new orders and trigger notifications |
| Active | Yes |
| Expires At | *(leave blank)* |
| Allowed IPs | *(leave blank — Zapier uses dynamic IPs)* |

После сохранения скопируйте полное значение токена и вставьте его в настройки интеграции Spwig в Zapier.

## Советы

- Дайте каждому токену понятное и конкретное имя — `Shopify Sync v2` намного полезнее, чем `Token 3`, когда вы будете проверять проблему спустя месяцы
- Создавайте один токен на каждую интеграцию — если интеграция скомпрометирована, вы можете отозвать только этот токен, не нарушая работу других
- Установите срок действия для токенов, используемых в одноразовых проектах или временных интеграциях — это снижает риск забытых токенов, которые остаются активными вечно
- Проверяйте свой список токенов каждые несколько месяцев и деактивируйте те токены, у которых дата **Last Used** неожиданно старая, так как они могут принадлежать интеграциям, которые больше не работают
- Если вы подозреваете, что токен был раскрыт, немедленно деактивируйте его, создайте замену и обновите затронутую интеграцию перед тем, как снова включить доступ
