---
title: APIトークン
---

APIトークンは、外部サービスや統合がストアと通信できるようにするためのセキュアなキーです。第三者のサービスやツールがストアのデータにアクセスしたり、アクションをトリガーしたりする必要がある場合、各リクエストにAPIトークンを添えて送信します。これにより、ストアはリクエストが承認されていることを確認できます。APIトークンは、管理画面のAPIトークンセクションから作成および管理します。

## APIトークンが必要になるとき

通常、以下のような場合にAPIトークンを作成する必要があります。

- ストアから読み取りまたは書き込みが必要な外部サービスや自動化ツールを接続するとき
- 入力されたコールを認証する必要があるウェブフックレシーバーを設定するとき
- ご自分のインストール用のSpwigヘルプシステムを構成するとき
- SpwigのAPIを使用してカスタム統合を構築するとき
- Spwigストアと他のシステムの間でデータを同期するとき

各統合には独自のトークンを持つべきです。これにより、あるサービスのアクセスを無効にしても他のサービスに影響を与えることなくできます。

## トークンの種類

トークンを作成する際、その目的を示すタイプを選択します。このタイプは、ご自身の参考のために、各トークンの役割を把握するのに役立ちます。

| タイプ | 目的 |
|------|---------|
| **ヘルプシステム** | Spwigヘルプドキュメンテーションシステムで使用されます |
| **外部統合** | 第三者サービス、自動化ツール（例：Zapier）、またはデータ同期ツール |
| **ウェブフック** | ウェブフックレシーバーまたはエンドポイントの認証 |
| **カスタム** | 上記のカテゴリに該当しない他の目的 |
| **インスタンス同期** | Spwigインストール間または外部Spwigサービス間の同期 |

## APIトークンの作成

1. **設定 > APIトークン** に移動します
2. **+ APIトークンの追加** をクリックします
3. トークンの目的を明確に示す **名前** を入力します（例：`Zapier製品同期` または `ヘルプシステムAPI`）
4. 適切な **トークンタイプ** を選択します
5. 統合に関する詳細情報を含む **説明** を任意で追加します
6. 必要に応じて **アクティブ** ステータス、**有効期限日**、および **許可されたIPアドレス** を構成します（下記を参照）
7. **保存** をクリックします

保存後、トークンの完全な値は詳細ページに表示されます。**すぐにコピーしてください** — リストビューではセキュリティのためトークンがマスキングされて表示され、このページを離れた後では再び完全な値を取得することはできません。

## トークン値のセキュリティ

Spwigは、新しいトークンを保存した直後にのみ、トークンの完全な値を表示します。その後、リストビューではマスキングされたバージョン（例: `spw_••••••••••••••••••••3f8a`）のみが表示されます。

トークン値を紛失した場合、それを復元することはできません。古いトークンを削除し、新しいトークンを作成して、それを使用していた統合を更新する必要があります。

**メール、チャットメッセージ、ソースコードでトークン値を共有しないでください**。パスワードのように扱ってください。

## 有効期限の設定

**Expires At**フィールドは、トークンが自動的に停止する日時を設定します。有効期限のないトークンの場合は、このフィールドを空のままにしてください。

有効期限の設定は次のケースで役立ちます:

- 一定の終了日を持つ一時的な統合
- 自動的にアクセスを削除したい第三者に与えるトークン
- 高権限を持つ統合に追加のセキュリティ層を提供

トークンが期限切れになると、そのトークンを使用したリクエストは拒否されます。アクセスを延長するには、**Expires At**の日時を更新するか、代替トークンを作成してください。

## 特定のIPアドレスへの制限

**Allowed IPs**フィールドはIPアドレスのリストを受け入れます。リストが空でない場合、トークンはそのリストに含まれるアドレスからのみリクエストを処理します。

たとえば、アナリティクスツールが`203.0.113.42`のサーバーで実行されている場合、そのIPを追加すると、他の場所からトークンが漏洩した場合でも誤って使用されることはできません。

**Allowed IPs**を空のままにすると、すべてのIPアドレスからのリクエストが許可されます。

## トークンの使用状況の監視

トークンリストは次の情報を表示します:

- **Usage Count** — トークンが使用された総回数
- **Last Used** — トークンが最後にリクエストを作成した時刻

これらのフィールドは、使用されていないトークン（失効の候補）を特定し、予期せぬアクティビティを発見するのに役立ちます。

使用回数の急激な増加は、トークンが意図された統合以外の人物によって使用されている可能性を示しています。

## トークンの失効

トークンを削除せずにすぐに使用を停止するには:

1. トークン名をクリックします
2. **Active** を外します
3. 保存します

トークンはリストに残っており、参照のために保持されますが、その後のすべてのリクエストで拒否されます。これは、問題の調査中に統合を一時的に停止する必要がある場合に役立ちます。

トークンを永久に削除するには:

1. リストからそのチェックボックスを選択します
2. アクションメニューから **Delete selected API tokens** を選択します
3. 削除を確認します

一度削除されると、トークンは復元できません。統合がまだアクセスを必要とする場合は、新しいトークンを作成し、統合の設定を更新してください。

## 例: Zapier統合の設定

**シナリオ:** 店舗をZapierに接続して、注文通知を自動化したい場合です。

| Field | Value |
|-------|-------|
| Name | `Zapier Order Automation` |
| Token Type | External Integration |
| Description | Zapierが新しい注文を読み取り、通知をトリガーするために使用されます |
| Active | Yes |
| Expires At | *(空白のままにします)* |
| Allowed IPs | *(空白のままにします — Zapierは動的IPを使用します)* |

保存後、トークンの完全な値をコピーし、ZapierのSpwig統合設定に貼り付けます。

## Tips

すべてのMarkdownの書式、画像のパス、コードブロック、技術用語を保持してください。

- すべてのトークンに明確で具体的な名前を付ける — ブラウザのトラブルシューティングを数カ月後に実施する場合、`Shopify Sync v2` は `Token 3` よりもはるかに有用である
- 1つの統合ごとに1つのトークンを作成する — 統合が侵害された場合、他のトークンに影響を与えることなくそのトークンのみを無効にできる
- 一時的なプロジェクトや一時的な統合で使用されるトークンに有効期限を設定する — これにより、忘れられたトークンが無期限に有効なままになるリスクが低減される
- 数か月に一度トークンリストを確認し、**Last Used** 日が予期せぬほど古いトークンを無効にする — これらはもはや実行されていない統合に属している可能性がある
- トークンが漏洩した可能性がある場合は、直ちに無効にし、代替トークンを作成し、影響を受けた統合を更新した後でアクセスを再び有効にする
