---
title: 付款設定
---

付款服務商將您的商店連接到付款網關，讓您可以在結帳時接受信用卡、數位錢包和其他付款方式。Spwig 支援同時使用多個服務商，為您的客戶提供靈活的付款選項。

![付款服務商](/static/core/admin/img/help/payment-setup/payment-dashboard.webp)

## 可用服務商

| 服務商 | 說明 |
|----------|-------------|
| **Stripe** | 信用卡、Apple Pay、Google Pay 和 135+ 種貨幣 |
| **PayPal** | PayPal 帳戶餘額、信用卡/借記卡，以及延後付款選項 |
| **Airwallex** | 為跨境電商優化的多幣種付款 |
| **Adyen** | 全球擁有 250+ 種付款方式的企業級付款 |
| **Square** | 支援現場和線上付款，並整合 POS 系統 |
| **Revolut** | 快速的歐洲付款，具有競爭性的外匯匯率 |

## 連接服務商

前往 **設定 > 付款服務商**，並點擊 **連接服務商** 來啟動設定向導。

### 第 1 步：選擇服務商

從可用的付款服務商中選擇。每張卡片顯示服務商支援的功能和地區。

### 第 2 步：設定指南

查看特定服務商的設定指南。這包括：
- 如何在服務商處創建帳戶（如果您還沒有帳戶）
- 如何在服務商的儀表板中找到您的 API 權限憑證
- 任何前提條件（例如，商業驗證）

### 第 3 步：輸入憑證

輸入您的 API 權限憑證：
- **API 金鑰 / 秘密金鑰** — 從服務商的儀表板獲得的驗證憑證
- **結帳模式** — 選擇客戶如何與付款表單互動：

| 模式 | 說明 |
|------|-------------|
| **Hosted** | 客戶被導向服務商的付款頁面（例如，Stripe Checkout）。最簡單的設定，PCI 合規性由服務商處理。 |
| **Integrated** | 支付表單直接嵌入到您的結帳頁面中。無縫體驗，但需要服務商的 JavaScript SDK。 |

- **沙盒 / 真實模式** — 先使用沙盒模式進行測試，準備好後切換到真實模式

### 第 4 步：測試連接

點擊 **測試連接** 來驗證您的憑證是否有效。向導會檢查：
- API 金鑰驗證
- 帳戶權限
- Webhook 端點的可訪問性

### 第 5 步：配置和保存

完成服務商設定：
- **啟用** — 啟用或停用服務商
- **預設服務商** — 設為結帳時的主要付款方式
- **顯示名稱** — 結帳時顯示給客戶的名稱
- **排序順序** — 控制結帳時服務商顯示的順序（數字較小的會先顯示）

## 付款儀表板

前往 **設定 > 付款儀表板** 以查看付款活動概覽：

### 需要處理的動作

頂部的警示卡片突出顯示需要關注的問題：
- **失敗交易** — 無法處理的付款
- **待捕獲交易** — 已授權但尚未捕獲的付款
- **連接錯誤** — 存在連接問題的服務商

### 收入分析

- **收入圖表** — 隨時間推移的付款體積視覺化分析，按天、週或月分組
- **效能指標** — 總收入、成功率、平均交易價值和退款率
- **服務商比較** — 每個連接的服務商並排顯示的效能卡片

### 交易分析

- **狀態分佈** — 已完成、待處理、失敗和退款的交易數量
- **付款方式組合** — 客戶最常使用的付款方式（信用卡、PayPal、數位錢包）

## 管理付款方式

每個服務商支援不同的付款方式。您可以根據國家啟用或停用特定的付款方式：

1. 前往服務商的設定頁面
2. 滾動到 **付款方式** 區段
3. 啟用或停用個別的付款方式
4. 使用國家級控制來限制特定市場的付款方式

當您在一個地區很受歡迎但在另一個地區不那麼受歡迎的付款方式時，這很有用（例如荷蘭的 iDEAL，比利時的 Bancontact）。

## Webhook

Webhook 可以讓您的商店與付款服務商實時同步。它們處理以下事件：
- 付款完成或失敗
- 退款處理
- 開啟爭議和退單
- 訂閱續費

### 自動設定

當您連接一個服務商時，Spwig 會自動在服務商處註冊一個 Webhook 端點。Webhook 的 URL 會顯示在服務商的設定頁面上供參考。

### Webhook 監控

每個進來的 Webhook 都會被記錄，包括：
- **事件類型**（例如，payment_intent.succeeded）
- **時間戳** 和處理狀態
- **負載** 用於除錯

如果 Webhook 處理失敗，會被記錄為錯誤，以便您進行調查。

## 使用多個服務商

您可以同時連接多個付款服務商：

- **預設服務商** — 在結帳時預設選擇的服務商。在服務商的設定中標記一個服務商為預設。
- **排序順序** — 控制結帳時的顯示順序。客戶可以看到所有啟用的服務商，並選擇他們偏好的一個。
- **故障轉移** — 如果某個服務商出現停機，客戶仍然可以使用其他服務商進行付款。

## 小技巧

- 從 **Stripe** 或 **PayPal** 開始 — 它們支援最廣泛的付款方式和地區。
- 使用 **沙盒/測試模式** 來處理測試交易，再上線。每個服務商的文檔中都有測試信用卡號。
- 啟用 **多個服務商**，以便在某個服務商出現問題時，客戶有備用的付款選項。
- 為您偏好的服務商設置 **較低的排序順序**，使其在結帳時首先顯示。
- 每週查看付款儀表板，以及早發現失敗交易和連接問題。
- 保護好您的 API 權限憑證 — 它們在資料庫中加密存儲，但絕不應該分享。
