ผู้ให้บริการชำระเงินเชื่อมต่อร้านค้าของคุณกับผู้ให้บริการชำระเงิน เพื่อให้คุณสามารถรับบัตรเครดิต กระเป๋าเงินดิจิทัล และวิธีการชำระเงินอื่นๆ ที่เคาน์เตอร์ชำระเงิน Spwig รองรับผู้ให้บริการหลายรายพร้อมกัน ทำให้ลูกค้าของคุณมีตัวเลือกการชำระเงินที่ยืดหยุ่น

![ผู้ให้บริการชำระเงิน](/static/core/admin/img/help/payment-setup/payment-dashboard.webp)

## ผู้ให้บริการที่มีอยู่

| ผู้ให้บริการ | คำอธิบาย |
|----------|-------------|
| **Stripe** | บัตรเครดิต Apple Pay Google Pay และ 135+ สกุลเงิน |
| **PayPal** | ยอดเงิน PayPal บัตรเครดิต/เดบิต และตัวเลือกชำระเงินภายหลัง |
| **Airwallex** | การชำระเงินหลายสกุลเงินที่ได้รับการปรับแต่งสำหรับการค้าข้ามพรมแดน |
| **Adyen** | การชำระเงินระดับองค์กรที่มีวิธีการชำระเงิน 250+ วิธีทั่วโลก |
| **Square** | การชำระเงินแบบตัวต่อตัวและออนไลน์พร้อมการสนับสนุน POS ที่ผสานรวม |
| **Revolut** | การชำระเงินในยุโรปที่รวดเร็วพร้อมอัตราแลกเปลี่ยนเงินตราที่มีการแข่งขัน |

## การเชื่อมต่อกับผู้ให้บริการ

ไปที่ **การตั้งค่า > ผู้ให้บริการชำระเงิน** และคลิก **เชื่อมต่อกับผู้ให้บริการ** เพื่อเปิดตัวเครื่องมือตั้งค่า

### ขั้นตอนที่ 1: เลือกผู้ให้บริการ

เลือกผู้ให้บริการชำระเงินที่มีอยู่ แต่ละบัตรแสดงคุณสมบัติและภูมิภาคที่ผู้ให้บริการรองรับ

### ขั้นตอนที่ 2: คำแนะนำการตั้งค่า

ตรวจสอบคู่มือการตั้งค่าเฉพาะผู้ให้บริการ ซึ่งรวมถึง:
- วิธีสร้างบัญชีกับผู้ให้บริการ (หากคุณยังไม่มี)
- ตำแหน่งที่คุณสามารถหาได้ API ข้อมูลการรับรองในแดชบอร์ดของผู้ให้บริการ
- ข้อกำหนดเบื้องต้น (เช่น การตรวจสอบธุรกิจ)

### ขั้นตอนที่ 3: ป้อนข้อมูลการรับรอง

ป้อนข้อมูลการรับรอง API ของคุณ:
- **API Key / Secret Key** — ข้อมูลการรับรองการเข้าถึงของคุณจากแดชบอร์ดผู้ให้บริการ
- **โหมดการชำระเงิน** — เลือกวิธีที่ลูกค้าจะมีปฏิสัมพันธ์กับฟอร์มการชำระเงิน:

| โหมด | คำอธิบาย |
|------|-------------|
| **Hosted** | ลูกค้าจะถูกเปลี่ยนเส้นทางไปยังหน้าชำระเงินของผู้ให้บริการ (เช่น Stripe Checkout) ตั้งค่าที่ง่ายที่สุด ผู้ให้บริการจัดการความเป็นไปตามมาตรฐาน PCI |
| **Integrated** | ฟอร์มการชำระเงินจะถูกฝังโดยตรงในหน้าชำระเงินของคุณ ประสบการณ์ที่ราบรื่น แต่ต้องการ SDK JavaScript ของผู้ให้บริการ |

- **โหมดทดลอง / โหมดจริง** — เริ่มต้นในโหมดทดลองสำหรับการทดสอบ แล้วเปลี่ยนไปโหมดจริงเมื่อพร้อม

### ขั้นตอนที่ 4: ทดสอบการเชื่อมต่อ

คลิก **ทดสอบการเชื่อมต่อ** เพื่อตรวจสอบว่าข้อมูลการรับรองของคุณถูกต้อง เครื่องมือตรวจสอบ:
- การรับรอง API key
- สิทธิ์การใช้งานบัญชี
- ความสามารถในการเข้าถึงจุดปลายทางเว็บฮุค

### ขั้นตอนที่ 5: ตั้งค่าและบันทึก

กำหนดค่าผู้ให้บริการ:
- **เปิดใช้งาน** — เปิดหรือปิดการใช้งานผู้ให้บริการ
- **ผู้ให้บริการเริ่มต้น** — ตั้งเป็นวิธีการชำระเงินหลักที่เคาน์เตอร์ชำระเงิน
- **ชื่อที่แสดง** — ชื่อที่แสดงให้ลูกค้าเห็นขณะชำระเงิน
- **ลำดับการจัดเรียง** — ควบคุมลำดับที่ผู้ให้บริการปรากฏที่เคาน์เตอร์ชำระเงิน (ตัวเลขที่ต่ำกว่าจะปรากฏก่อน)

## แดชบอร์ดการชำระเงิน

ไปที่ **การตั้งค่า > แดชบอร์ดการชำระเงิน** เพื่อดูภาพรวมกิจกรรมการชำระเงินของคุณ:

### การดำเนินการที่จำเป็น

การ์ดแจ้งเตือนด้านบนเน้นปัญหาที่ต้องการความสนใจ:
- **การทำธุรกรรมที่ล้มเหลว** — การชำระเงินที่ไม่สามารถดำเนินการได้
- **การทำธุรกรรมที่รอการจับคู่** — การชำระเงินที่ได้รับการอนุมัติแต่ยังรอการจับคู่
- **ข้อผิดพลาดในการเชื่อมต่อ** — ผู้ให้บริการที่มีปัญหาการเชื่อมต่อ

### การวิเคราะห์รายได้

- **แผนภูมิรายได้** — การวิเคราะห์ภาพรวมของปริมาณการชำระเงินตามเวลา แบ่งตามวัน สัปดาห์ หรือเดือน
- **ตัวชี้วัดประสิทธิภาพ** — รายได้รวม อัตราความสำเร็จ มูลค่าธุรกรรมเฉลี่ย และอัตราคืนเงิน
- **การเปรียบเทียบผู้ให้บริการ** — บัตรประสิทธิภาพที่เปรียบเทียบกันของผู้ให้บริการที่เชื่อมต่อแต่ละราย

### การแยกประเภทการทำธุรกรรม

- **การกระจายสถานะ** — จำนวนการทำธุรกรรมที่สำเร็จ รออยู่ ล้มเหลว และคืนเงิน
- **ส่วนผสมวิธีการชำระเงิน** — วิธีการชำระเงินที่ลูกค้าใช้บ่อยที่สุด (บัตรเครดิต PayPal กระเป๋าเงินดิจิทัล)

## การจัดการวิธีการชำระเงิน

ผู้ให้บริการแต่ละรายรองรับวิธีการชำระเงินที่แตกต่างกัน คุณสามารถเปิดหรือปิดวิธีการเฉพาะตามประเทศ:

1. ไปที่หน้าตั้งค่าผู้ให้บริการ
2. ค้นหาส่วน **วิธีการชำระเงิน**
3. เปิดหรือปิดวิธีการเฉพาะแต่ละรายการ
4. ใช้การควบคุมระดับประเทศเพื่อจำกัดวิธีการเฉพาะในตลาดที่กำหนด

มีประโยชน์เมื่อวิธีการชำระเงินเป็นที่นิยมในภูมิภาคหนึ่งแต่ไม่ใช่ในอีกภูมิภาคหนึ่ง (เช่น iDEAL ในเนเธอร์แลนด์ Bancontact ในเบลเยียม)

## Webhooks

Webhooks ช่วยให้ร้านค้าของคุณสอดคล้องกับผู้ให้บริการชำระเงินแบบเรียลไทม์ พวกเขาจัดการเหตุการณ์เช่น:
- การชำระเงินสำเร็จหรือล้มเหลว
- การคืนเงินที่ดำเนินการแล้ว
- การเปิดข้อพิพาทและคืนเงิน
- การต่ออายุสมาชิก

### การตั้งค่าอัตโนมัติ

เมื่อคุณเชื่อมต่อกับผู้ให้บริการ Spwig จะลงทะเบียนจุดปลายทางเว็บฮุคกับผู้ให้บริการโดยอัตโนมัติ URL ของเว็บฮุคจะแสดงบนหน้าตั้งค่าผู้ให้บริการเพื่อใช้อ้างอิง

### การตรวจสอบเว็บฮุค

แต่ละเว็บฮุคที่เข้ามาจะถูกบันทึกพร้อมกับ:
- **ประเภทเหตุการณ์** (เช่น payment_intent.succeeded)
- **เวลาประทับ** และสถานะการประมวลผล
- **ข้อมูลบรรทัด** สำหรับการแก้ไขปัญหา

หากเว็บฮุคล้มเหลวในการประมวลผล จะถูกบันทึกเป็นข้อผิดพลาดเพื่อให้คุณสามารถตรวจสอบได้

## การใช้ผู้ให้บริการหลายราย

คุณสามารถเชื่อมต่อกับผู้ให้บริการชำระเงินหลายรายพร้อมกัน:

- **ผู้ให้บริการเริ่มต้น** — ผู้ให้บริการที่ถูกเลือกเป็นค่าเริ่มต้นที่เคาน์เตอร์ชำระเงิน ตั้งค่าผู้ให้บริการหนึ่งรายเป็นค่าเริ่มต้นในตั้งค่าของมัน
- **ลำดับการจัดเรียง** — ควบคุมลำดับการแสดงที่เคาน์เตอร์ชำระเงิน ลูกค้าจะเห็นผู้ให้บริการที่เปิดใช้งานทั้งหมดและสามารถเลือกวิธีการที่ต้องการได้
- **การสำรอง** — หากผู้ให้บริการมีปัญหาด้านการเชื่อมต่อ ลูกค้าสามารถชำระเงินได้โดยใช้ผู้ให้บริการอื่นแทน

## เคล็ดลับ

- เริ่มต้นด้วย **Stripe** หรือ **PayPal** — พวกเขาครอบคลุมช่วงวิธีการชำระเงินและภูมิภาคที่กว้างที่สุด
- ใช้ **โหมดทดลอง/ทดสอบ** เพื่อประมวลผลการทำธุรกรรมทดสอบก่อนที่จะเปิดใช้งานจริง แต่ละผู้ให้บริการมีเลขบัตรทดสอบในเอกสารของพวกเขา
- เปิดใช้งาน **ผู้ให้บริการหลายราย** เพื่อให้ลูกค้ามีทางเลือกในการชำระเงินหากผู้ให้บริการหนึ่งมีปัญหา
- ตั้งค่า **ลำดับการจัดเรียงต่ำ** สำหรับผู้ให้บริการที่คุณชอบเพื่อให้แสดงขึ้นแรกที่เคาน์เตอร์ชำระเงิน
- ตรวจสอบแดชบอร์ดการชำระเงินรายสัปดาห์เพื่อจับความล้มเหลวในการชำระเงินและปัญหาการเชื่อมต่อได้ตั้งแต่เนิ่นๆ
- รักษาข้อมูลการรับรอง API ให้ปลอดภัย — แม้ว่าจะถูกเก็บไว้ในฐานข้อมูลแบบเข้ารหัส แต่ไม่ควรแบ่งปันกับผู้อื่น
