---
title: メールの設定
---

メールの設定は、ストアがトランザクショナルメール（注文確認、配送通知、パスワードリセットなど）を送信する方法を制御します。Spwigには組み込みのSMTPサーバーが含まれており、より高い配信性を確保するために外部のメールプロバイダもサポートしています。

![メールアカウント](/static/core/admin/img/help/email-configuration/email-accounts.webp)

## 利用可能なプロバイダ

| プロバイダ | 説明 |
|----------|-------------|
| **組み込みSMTP** | Spwigに含まれる無料のセルフホストメールサーバー。自動DKIM署名をサポートしています。 |
| **Gmail API** | Google OAuth認証を使用して、GmailまたはGoogle Workspaceアカウント経由でメールを送信します。 |
| **汎用SMTP** | 任意のSMTPサーバー（SendGrid、Mailgun、Amazon SES、または独自のメールサーバー）に接続できます。 |

## メールの設定

**設定 > メールアカウント**に移動し、**メールアカウントの追加**をクリックして設定ウィザードを起動します。

### ステップ1: プロバイダの選択

メールプロバイダを選択します。組み込みのSMTPサーバーは、すぐに開始するための最も簡単なオプションです — 外部アカウントは必要ありません。

### ステップ2: 処理情報の設定

選択したプロバイダの処理情報を入力します:

- **組み込みSMTP** — 処理情報は必要ありません。サーバーはSpwigのインストール上に実行されます。
- **Gmail API** — Google OAuthを介して認証します。Googleアカウントでサインインするようにリダイレクトされます。
- **汎用SMTP** — SMTPサーバーのアドレス、ポート、ユーザー名、パスワードを入力します。

### ステップ3: 送信者設定

送信メールの送信者情報を設定します:

- **From Email** — 「From」フィールドに表示されるメールアドレス（例: orders@yourstore.com）
- **From Name** — メールアドレスの隣に表示される表示名（例: 「Your Store Name」）
- **Reply-To Email** — 顧客からの返信先（Fromアドレスとは異なる場合があります）

### ステップ4: DNS検証

ドメインのメール認証レコードを確認します。ウィザードは3つのDNSレコードをチェックします:

| レコード | 目的 |
|--------|---------|
| **SPF** | あなたのサーバーがドメインの代わりにメールを送信することを許可します |
| **DKIM** | メールをデジタル署名して、改ざんされていないことを証明します |
| **DMARC** | SPF/DKIMチェックに失敗したメールの処理方法を受信サーバーに指示します |

各レコードに対して、ウィザードは以下を表示します:
- **現在の状態** — レコードが正しく設定されているかどうか
- **必要な値** — ドメインレジスターで追加する必要のある正確なDNSレコード
- **プロパゲーション状態** — 最近の変更が効果を発揮しているかどうか（DNS変更は最大48時間かかる場合があります）

組み込みのSMTPサーバーは、あなたのドメイン用のDKIMキーを自動的に生成します。

### ステップ5: テストメールの送信

すべてが正常に動作することを確認するためにテストメールを送信します:
1. 受信者メールアドレスを入力します
2. **テストメールの送信**をクリックします
3. テストメッセージが届くまでメールボックスを確認します
4. メールがスパム警告なしに届くことを確認します

### ステップ6: 保存と有効化

設定を保存し、アカウントを有効にします。このアカウントを**デフォルト**としてマークする場合は、それがプライマリメールアカウントになるようにします。

## メールテンプレート

Spwigには、すべてのトランザクショナルイベント用に30以上のメールテンプレートが含まれています。**設定 > メールテンプレート**に移動して管理してください。

### テンプレートタイプ

テンプレートは、すべてのストアイベントをカバーしています:
- **注文ライフサイクル** — 確認、処理、出荷、配達、キャンセル
- **支払い** — 領収書、返金確認、支払い失敗
- **顧客アカウント** — ウェルカム、パスワードリセット、メール確認
- **ギフトカード** — 配送、残高通知
- **配送** — 配送状況の更新、配達確認
- **デジタル製品** — ダウンロードリンク、ライセンスキー
- **マーケティング** — カート放棄の回復、レビュー依頼

### テンプレートのカスタマイズ

1. テンプレート一覧に移動します
2. 編集するテンプレートをクリックします
3. 件名、ヘッダー、本文、フッターを編集します
4. テンプレート変数（例: `{{ order.number }}`, `{{ customer.name }}`）を使用して動的なコンテンツを作成します
5. 保存する前にメールをプレビューします

### 多言語サポート

メールテンプレートは複数の言語をサポートしています:
- 各テンプレートは、ストアでアクティブなすべての言語の翻訳を含むことができます
- システムは、顧客の希望する言語でメールを送信します
- **言語の代替チェーン** — 翻訳が利用できない場合、システムはストアのデフォルト言語にフォールバックします
- **AI翻訳**機能を使用して、テンプレートを他の言語に自動的に翻訳できます

### テンプレートの複製

システムテンプレートのカスタマイズされたバージョンを作成するには:
1. 編集したいテンプレートを開きます
2. **テンプレートの複製**をクリックします
3. 複製したバージョンを編集します
4. 複製版は、元のシステムテンプレートよりも優先されます

## メールキュー

**設定 > メールキュー**で、送信中のメールを監視できます:

- **キューイング** — 送信を待っているメール
- **送信中** — 現在送信中のメール
- **送信済み** — 成功裏に配信されたメール
- **失敗** — 配信できなかったメール（エラーの詳細付き）
- **バウンド** — 受信者のメールサーバーによって拒否されたメール

メールをクリックして、受信者、件名、送信時間、配信状態などの詳細情報を表示できます。

## 配信の追跡

メールのエンゲージメントを追跡します:
- **開封数** — メールを開封した受信者の数
- **クリック数** — メール内のリンククリック数
- **バウンド数** — ハードバウンドとソフトバウンドの追跡
- **苦情数** — 受信者からのスパム報告数

## 複数のアカウント

複数のメールアカウントを設定できます:
- **デフォルトアカウント** — すべての送信メールに使用されます（上書きされない限り）
- **代替アカウント** — デフォルトアカウントが失敗した場合、メールは再試行のためキューイングされます
- 異なる目的で異なるアカウントを使用できます（例: トランザクショナルメール用の1つ、マーケティング用のもう1つ）

## ヒント

- すぐに設定が必要な場合は、**組み込みSMTP**サーバーから始め、より高い送信量やより良い配信性が必要な場合は、外部プロバイダに切り替えてください。
- **SPF、DKIM、およびDMARC**レコードを常に設定してください — これらがないと、メールはスパムフォルダに届く可能性がはるかに高くなります。
- 任意の設定変更の後、**テストメール**を送信して配信が正常に動作することを確認してください。
- **失敗**または**バウンド**メールのキューを定期的に監視してください — これらは配信性の問題を示しています。
- **専門的な送信者アドレス**（例: orders@yourstore.com）を使用して、信頼性と配信性を高めます。無料のメールアドレスよりもお勧めします。
- テンプレートは簡潔に保つ — トランザクショナルメールは情報を迅速に提供する必要がありますが、マーケティングニュースレターではありません。
