title: Настройка платежей
---

Платежные провайдеры подключают ваш магазин к платежным шлюзам, чтобы вы могли принимать кредитные карты, цифровые кошельки и другие методы оплаты на этапе оформления заказа. Spwig поддерживает несколько провайдеров одновременно, предоставляя вашим клиентам гибкие варианты оплаты.

![Платежные провайдеры](/static/core/admin/img/help/payment-setup/payment-dashboard.webp)

## Доступные провайдеры

| Провайдер | Описание |
|----------|-------------|
| **Stripe** | Кредитные карты, Apple Pay, Google Pay и 135+ валют |
| **PayPal** | Баланс PayPal, кредитные/дебетовые карты и опции оплаты позже |
| **Airwallex** | Многонациональные платежи, оптимизированные для международной торговли |
| **Adyen** | Предприятие-уровень платежей с 250+ методами оплаты по всему миру |
| **Square** | Оплата в магазине и онлайн с поддержкой POS |
| **Revolut** | Быстрые европейские платежи с конкурентоспособными курсами обмена |

## Подключение провайдера

Перейдите в **Настройки > Платежные провайдеры** и нажмите **Подключить провайдера**, чтобы запустить мастер настройки.

### Шаг 1: Выберите провайдера

Выберите из доступных платежных провайдеров. Каждая карточка показывает поддерживаемые функции и регионы провайдера.

### Шаг 2: Инструкции по настройке

Просмотрите специфическую для провайдера инструкцию по настройке. Это включает в себя:
- Как создать учетную запись у провайдера (если у вас еще нет такой)
- Где найти свои учетные данные API в панели управления провайдера
- Любые предварительные условия (например, проверка бизнеса)

### Шаг 3: Введите учетные данные

Введите свои учетные данные API:
- **Ключ API / Секретный ключ** — ваши учетные данные для аутентификации из панели управления провайдера
- **Режим оформления** — выберите, как клиенты взаимодействуют с формой оплаты:

| Режим | Описание |
|------|-------------|
| **Хостированный** | Клиенты перенаправляются на страницу оплаты провайдера (например, Stripe Checkout). Самый простой способ настройки, соответствие требованиям PCI обеспечивается провайдером. |
| **Интегрированный** | Форма оплаты встроена напрямую в вашу страницу оформления заказа. Плавный опыт, но требуется JavaScript SDK провайдера. |

- **Режим песочницы / Режим живых данных** — начните с режима песочницы для тестирования, затем переключитесь на режим живых данных, когда будет готово

### Шаг 4: Тестирование подключения

Нажмите **Проверить подключение**, чтобы проверить, действительны ли ваши учетные данные. Мастер проверяет:
- Аутентификацию ключа API
- Разрешения учетной записи
- Доступность конечной точки веб-_hooks

### Шаг 5: Настройка и сохранение

Завершите настройки провайдера:
- **Активный** — включить или отключить провайдера
- **Основной провайдер** — установить как основной метод оплаты при оформлении заказа
- **Отображаемое имя** — имя, отображаемое клиентам при оформлении заказа
- **Порядок сортировки** — управляет порядком отображения провайдеров при оформлении заказа (меньшие числа отображаются первыми)

## Платежная панель управления

Перейдите в **Настройки > Платежная панель управления** для получения обзора вашей платежной активности:

### Требуемые действия

Карты предупреждений вверху выделяют проблемы, требующие внимания:
- **Неудачные транзакции** — платежи, которые не удалось обработать
- **Ожидающие захвата** — авторизованные платежи, ожидающие захвата
- **Ошибки подключения** — провайдеры с проблемами подключения

### Анализ доходов

- **Диаграмма доходов** — визуальный разбор объема платежей со временем, сгруппированный по дню, неделе или месяц
- **Показатели производительности** — общий доход, уровень успеха, среднее значение транзакции и уровень возвратов
- **Сравнение провайдеров** — карточки производительности для каждого подключенного провайдера, расположенные рядом

### Разбор транзакций

- **Распределение статусов** — количество завершенных, ожидающих, неудачных и возвращенных транзакций
- **Смесь методов оплаты** — какие методы оплаты чаще всего используют клиенты (кредитные карты, PayPal, цифровые кошельки)

## Управление методами оплаты

Каждый провайдер поддерживает разные методы оплаты. Вы можете включить или отключить конкретные методы по стране:

1. Перейдите на страницу настройки провайдера
2. Прокрутите до раздела **Методы оплаты**
3. Включите или отключите отдельные методы
4. Используйте настройки на уровне страны, чтобы ограничить методы оплаты определенными рынками

Это полезно, когда метод оплаты популярен в одной области, но не в другой (например, iDEAL в Нидерландах, Bancontact в Бельгии).

## Webhooks

Webhooks синхронизируют ваш магазин с платежным провайдером в реальном времени. Они обрабатывают события, такие как:
- Завершение или неудача платежа
- Обработка возвратов
- Открытие споров и возвратов
- Обновления подписок

### Автоматическая настройка

Когда вы подключаете провайдера, Spwig автоматически регистрирует конечную точку веб-_hooks у провайдера. URL веб-_hooks отображается на странице настройки провайдера для справки.

### Мониторинг веб-_hooks

Каждый входящий веб-_hook записывается с:
- **Тип события** (например, payment_intent.succeeded)
- **Время** и статус обработки
- **Полезная нагрузка** для отладки

Если веб-_hook не удается обработать, он записывается как ошибка, чтобы вы могли расследовать.

## Использование нескольких провайдеров

Вы можете подключить несколько платежных провайдеров одновременно:

- **Основной провайдер** — провайдер, выбранный по умолчанию при оформлении заказа. Отметьте один провайдер как основной в его настройках.
- **Порядок сортировки** — управляет порядком отображения при оформлении заказа. Клиенты видят все активные провайдеры и могут выбрать свой предпочтительный.
- **Резервное соединение** — если провайдер испытывает сбои, клиенты все равно могут платить с помощью альтернативного провайдера.

## Советы

- Начните с **Stripe** или **PayPal** — они охватывают самый широкий спектр методов оплаты и регионов.
- Используйте **режим песочницы/теста** для обработки тестовых транзакций перед запуском. У каждого провайдера есть номера тестовых карт в их документации.
- Включите **несколько провайдеров**, чтобы у клиентов был резервный вариант оплаты, если у одного провайдера возникнут проблемы.
- Установите **низкий порядок сортировки** для вашего предпочтительного провайдера, чтобы он отображался первым при оформлении заказа.
- Регулярно проверяйте Платежную панель управления еженедельно, чтобы выявлять неудачные транзакции и проблемы с подключением на ранних стадиях.
- Сохраняйте свои учетные данные API в безопасности — они хранятся зашифрованными в базе данных, но никогда не должны делиться.
