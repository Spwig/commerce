---
title: 支付设置
---

支付提供商将您的商店连接到支付网关，使您能够在结账时接受信用卡、数字钱包和其他支付方式。Spwig 可以同时支持多个提供商，为您的客户提供灵活的支付选项。

![支付提供商](/static/core/admin/img/help/payment-setup/payment-dashboard.webp)

## 可用提供商

| 提供商 | 描述 |
|----------|-------------|
| **Stripe** | 信用卡、Apple Pay、Google Pay 和 135+ 种货币 |
| **PayPal** | PayPal 余额、信用卡/借记卡和稍后付款选项 |
| **Airwallex** | 优化多币种支付的跨境电子商务 |
| **Adyen** | 全球拥有 250+ 种支付方式的企业级支付 |
| **Square** | 集成 POS 支持的面对面和在线支付 |
| **Revolut** | 快速的欧洲支付和具有竞争力的外汇汇率 |

## 连接提供商

导航到 **设置 > 支付提供商**，然后点击 **连接提供商** 以启动设置向导。

### 第一步：选择提供商

从可用的支付提供商中选择。每张卡片显示提供商支持的功能和区域。

### 第二步：设置说明

查看特定提供商的设置指南。这包括：
- 如果您还没有账户，如何在提供商处创建账户
- 如何在提供商的仪表板中找到您的 API 凭据
- 任何前提条件（例如，商业验证）

### 第三步：输入凭据

输入您的 API 凭据：
- **API 密钥 / 秘密密钥** — 您从提供商的仪表板获得的认证凭据
- **结账模式** — 选择客户如何与支付表单进行交互：

| 模式 | 描述 |
|------|-------------|
| **托管** | 客户被重定向到提供商的支付页面（例如，Stripe 结账）。设置最简单，PCI 合规性由提供商处理。 |
| **集成** | 支付表单直接嵌入到您的结账页面中。无缝体验，但需要提供商的 JavaScript SDK。 |

- **沙箱 / 生产模式** — 从沙箱模式开始进行测试，然后在准备就绪时切换到生产模式

### 第四步：测试连接

点击 **测试连接** 以验证您的凭据是否有效。向导检查：
- API 密钥认证
- 账户权限
- Webhook 端点的可访问性

### 第五步：配置和保存

最终确定提供商的设置：
- **启用** — 启用或禁用提供商
- **默认提供商** — 在结账时设置为主要支付方式
- **显示名称** — 结账时向客户显示的名称
- **排序顺序** — 控制提供商在结账时的显示顺序（数字较小的优先显示）

## 支付仪表板

导航到 **设置 > 支付仪表板** 以查看您的支付活动概览：

### 需要执行的操作

顶部的警报卡片突出显示需要关注的问题：
- **失败的交易** — 无法处理的支付
- **待捕获的交易** — 已授权但等待捕获的支付
- **连接错误** — 存在连接问题的提供商

### 收入分析

- **收入图表** — 随时间推移的支付量的可视化分解，按天、周或月分组
- **性能指标** — 总收入、成功率、平均交易价值和退款率
- **提供商比较** — 每个连接提供商的并排性能卡片

### 交易分解

- **状态分布** — 已完成、待处理、失败和退款的交易数量
- **支付方式组合** — 客户最常使用的支付方式（信用卡、PayPal、数字钱包）

## 管理支付方式

每个提供商支持不同的支付方式。您可以按国家/地区启用或禁用特定的支付方式：

1. 导航到提供商的配置页面
2. 滚动到 **支付方式** 部分
3. 打开或关闭单个支付方式
4. 使用国家/地区级别的控制来限制特定市场的支付方式

当某个支付方式在一个地区很流行但在另一个地区不流行时，这很有用（例如，荷兰的 iDEAL，比利时的 Bancontact）。

## Webhook

Webhook 使您的商店能够实时与支付提供商保持同步。它们处理以下事件：
- 支付完成或失败
- 退款处理
- 争议和拒付的开启
- 订阅续订

### 自动设置

当您连接一个提供商时，Spwig 会自动在提供商处注册一个 webhook 端点。webhook URL 会显示在提供商的配置页面上，供参考。

### Webhook 监控

每个传入的 webhook 都会记录以下信息：
- **事件类型**（例如，payment_intent.succeeded）
- **时间戳** 和处理状态
- **有效负载** 用于调试

如果 webhook 处理失败，它将被记录为错误，以便您进行调查。

## 使用多个提供商

您可以同时连接多个支付提供商：

- **默认提供商** — 在结账时默认选择的提供商。在提供商的配置中将其中一个提供商标记为默认。
- **排序顺序** — 控制结账时的显示顺序。客户可以看到所有启用的提供商，并可以选择他们偏好的一个。
- **故障转移** — 如果某个提供商出现停机，客户仍然可以使用其他提供商进行支付。

## 小贴士

- 从 **Stripe** 或 **PayPal** 开始 — 它们覆盖的支付方式和区域最广泛。
- 使用 **沙箱/测试模式** 在上线前处理测试交易。每个提供商的文档中都有测试卡号。
- 启用 **多个提供商** 以便在某个提供商出现问题时，客户有备用的支付选项。
- 为您的首选提供商设置 **较低的排序顺序**，以便在结账时首先显示。
- 每周查看一次 **支付仪表板**，以便尽早发现失败的交易和连接问题。
- 保护好您的 API 凭据 — 它们在数据库中是加密存储的，但绝不要与他人共享。
