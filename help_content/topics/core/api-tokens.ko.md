---
title: API 토큰
---

API 토큰은 외부 서비스 및 통합이 가게와 통신할 수 있도록 해주는 보안 키입니다. 제3자 서비스나 도구가 가게의 데이터에 접근하거나 액션을 트리거해야 할 때, 요청에 함께 API 토큰을 보내어 요청이 승인되었는지 확인할 수 있습니다. 모든 토큰은 관리자 계정의 API 토큰 섹션에서 생성하고 관리합니다.

## API 토큰이 필요한 경우

다음과 같은 경우 API 토큰을 생성해야 합니다:

- 외부 서비스나 자동화 도구를 연결할 때, 해당 도구가 가게의 데이터를 읽거나 쓰는 경우
- 수신하는 웹훅이 들어오는 요청을 인증해야 하는 경우
- 설치된 Spwig 도움 시스템을 설정할 때
- Spwig의 API를 사용하여 커스텀 통합을 구축할 때
- Spwig 가게와 다른 시스템 간 데이터를 동기화할 때

각 통합에는 자체 토큰이 있어야 하며, 하나의 서비스에 대한 액세스를 취소해도 다른 서비스에 영향을 주지 않도록 할 수 있습니다.

## 토큰 유형

토큰을 생성할 때, 해당 토큰의 목적을 설명하는 유형을 선택합니다. 이 유형은 참고용이며, 각 토큰의 기능을 추적하는 데 도움이 됩니다.

| 유형 | 목적 |
|------|---------|
| **Help System** | Spwig 도움 문서 시스템에서 사용됨 |
| **External Integration** | 제3자 서비스, 자동화 도구(예: Zapier), 또는 데이터 동기화 도구 |
| **Webhook** | 웹훅 수신기 또는 엔드포인트의 인증 |
| **Custom** | 위의 범주에 해당하지 않는 기타 목적 |
| **Instance Sync** | Spwig 설치 간 또는 외부 Spwig 서비스 간 동기화 |

## API 토큰 생성

1. **설정 > API 토큰**으로 이동합니다
2. **+ API 토큰 추가**를 클릭합니다
3. 토큰의 목적을 명확히 설명하는 **이름**을 입력합니다(예: `Zapier 제품 동기화` 또는 `Help System API`)
4. 적절한 **토큰 유형**을 선택합니다
5. 통합에 대한 추가 정보를 포함하는 **설명**을 선택적으로 입력합니다
6. 필요에 따라 **활성 상태**, **만료 날짜**, **허용된 IP 주소**를 구성합니다(아래 참조)
7. **저장**을 클릭합니다

저장 후, 전체 토큰 값은 상세 페이지에서 표시됩니다. **즉시 복사하세요** — 보안상 목록 보기에서는 토큰이 마스킹되어 있고, 이 페이지를 떠나면 다시 전체 토큰 값을 얻을 수 없습니다.

## 토큰 값 보안

Spwig은 새 토큰을 저장한 직후에만 전체 토큰 값을 표시합니다. 이후 목록 보기에서는 마스킹된 버전만 표시됩니다(예: `spw_••••••••••••••••••••3f8a`).

토큰 값을 잃어버리면 복구할 수 없습니다. 이전 토큰을 삭제하고 새 토큰을 생성한 후, 해당 토큰을 사용하던 통합을 업데이트해야 합니다.

**이메일, 채팅 메시지, 소스 코드에 토큰 값을 공유하지 마세요.** 비밀번호처럼 대우하세요.

## 만료 날짜 설정

**Expires At** 필드는 토큰이 더 이상 작동하지 않도록 자동으로 설정할 날짜와 시간을 지정합니다. 토큰이 만료되어야 하지 않는 경우 이 필드를 비워 두세요.

만료 날짜는 다음과 같은 경우에 유용합니다:

- 특정 종료 날짜가 있는 일시적인 통합
- 제3자에게 제공하는 토큰으로 자동으로 액세스가 제거되도록 하고 싶은 경우
- 고권한 통합에 추가 보안 층을 제공하는 경우

토큰이 만료되면 해당 토큰으로 이루어진 요청은 거부됩니다. **Expires At** 날짜를 업데이트하거나 대체 토큰을 생성하여 액세스를 연장할 수 있습니다.

## 특정 IP 주소로 제한

**Allowed IPs** 필드는 IP 주소 목록을 입력할 수 있습니다. 목록이 비어 있지 않은 경우, 토큰은 해당 주소 중 하나에서 요청이 오는 경우에만 작동합니다.

예를 들어, 분석 도구가 `203.0.113.42` 서버에서 실행된다면, 해당 IP를 추가하면 토큰이 다른 위치에서 유출되어도 악용될 수 없습니다.

**Allowed IPs**을 비워 두면 모든 IP 주소에서 요청을 허용합니다.

## 토큰 사용 모니터링

토큰 목록은 다음과 같은 정보를 표시합니다:

- **Usage Count** — 토큰이 사용된 총 횟수
- **Last Used** — 토큰이 요청을 위해 마지막으로 사용된 시점

이 필드는 사용되지 않는 토큰(회수 후보)을 식별하고 예상치 못한 활동을 감지하는 데 도움이 됩니다.

사용 횟수의 갑작스러운 증가가 토큰이 의도된 인테그레이션 외의 누군가에 의해 사용되고 있음을 나타낼 수 있습니다.

## 토큰 회수

토큰을 삭제하지 않고 즉시 작동을 중단하려면:

1. 토큰 이름을 클릭합니다.
2. **활성화**를 선택 해제합니다.
3. 저장합니다.

토큰은 참조용으로 목록에 남아 있지만 이후 요청에서는 거부됩니다. 이는 문제를 조사하는 동안 인테그레이션을 일시적으로 중단할 때 유용합니다.

토큰을 영구적으로 제거하려면:

1. 목록에서 해당 체크박스를 선택합니다.
2. 액션 메뉴에서 **선택한 API 토큰 삭제**를 선택합니다.
3. 삭제를 확인합니다.

한 번 삭제된 토큰은 복구할 수 없습니다. 인테그레이션이 여전히 액세스가 필요하다면 새 토큰을 생성하고 인테그레이션의 설정을 업데이트합니다.

## 예시: Zapier 인테그레이션 설정

**시나리오:** 스토어를 Zapier에 연결하여 주문 알림을 자동화하고 싶습니다.

| 필드 | 값 |
|-------|-------|
| 이름 | `Zapier 주문 자동화` |
| 토큰 유형 | 외부 인테그레이션 |
| 설명 | Zapier이 새 주문을 읽고 알림을 트리거하는 데 사용됨 |
| 활성화 | 예 |
| 만료 시간 | *(빈칸으로 둡니다)* |
| 허용된 IP 주소 | *(빈칸으로 둡니다 — Zapier은 동적 IP를 사용합니다)* |

저장한 후 전체 토큰 값을 복사하여 Zapier의 Spwig 인테그레이션 설정에 붙여넣습니다.

## 팁

모든 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어를 유지합니다.

- 모든 토큰에 명확하고 구체적인 이름을 부여하세요 — 수개월 후 문제를 해결할 때 `Shopify Sync v2`는 `Token 3`보다 훨씬 유용합니다
- 하나의 통합에 대해 하나의 토큰을 생성하세요 — 통합이 해킹된 경우 다른 토큰에 영향을 주지 않고 해당 토큰만 취소할 수 있습니다
- 일회성 프로젝트나 임시 통합에서 사용되는 토큰에 만료 날짜를 설정하세요 — 이는 잊혀진 토큰이 영구히 활성 상태로 남는 위험을 줄입니다
- 몇 달에 한 번 토큰 목록을 검토하고 **마지막 사용** 날짜가 예상보다 오래된 토큰을 비활성화하세요. 이는 더 이상 실행되지 않는 통합에 속할 수 있습니다
- 토큰이 노출되었을 가능성이 있다고 생각되면 즉시 비활성화하고 대체 토큰을 생성한 후 영향을 받은 통합을 업데이트한 후에 다시 접근 권한을 활성화하세요
