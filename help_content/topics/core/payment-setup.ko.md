---
title: 결제 설정
---

결제 제공업체는 귀하의 가게를 결제 게이트웨이에 연결하여 고객이 체크아웃 시 신용카드, 디지털 지갑 및 기타 결제 방식을 사용할 수 있도록 합니다. Spwig은 여러 제공업체를 동시에 지원하여 고객에게 유연한 결제 옵션을 제공합니다.

![결제 제공업체](/static/core/admin/img/help/payment-setup/payment-dashboard.webp)

## 사용 가능한 제공업체

| 제공업체 | 설명 |
|----------|-------------|
| **Stripe** | 신용카드, Apple Pay, Google Pay 및 135개 이상의 통화 |
| **PayPal** | PayPal 잔액, 신용/체크카드 및 나중에 결제 옵션 |
| **Airwallex** | 해외 무역을 위한 최적화된 다국통화 결제 |
| **Adyen** | 전 세계 250개 이상의 결제 방식을 지원하는 기업용급 결제 |
| **Square** | 통합된 POS 지원을 갖춘 대면 및 온라인 결제 |
| **Revolut** | 경쟁력 있는 외환 비율을 갖춘 빠른 유럽 내 결제 |

## 제공업체 연결

**설정 > 결제 제공업체**로 이동하여 **제공업체 연결**을 클릭하여 설정 마법사를 시작합니다.

### 단계 1: 제공업체 선택

사용 가능한 결제 제공업체 중에서 선택합니다. 각 카드는 제공업체가 지원하는 기능과 지역을 보여줍니다.

### 단계 2: 설정 지침

제공업체별 설정 가이드를 확인합니다. 이는 다음과 같은 내용을 포함합니다:
- 제공업체 계정을 만드는 방법(계정이 없는 경우)
- 제공업체 대시보드에서 API 자격 증명을 찾는 위치
- 필요한 사전 조건(예: 사업 확인)

### 단계 3: 자격 증명 입력

API 자격 증명을 입력합니다:
- **API 키 / 비밀 키** - 제공업체 대시보드에서 받은 인증 자격 증명
- **체크아웃 모드** - 고객이 결제 양식과 상호작용하는 방식을 선택합니다:

| 모드 | 설명 |
|------|-------------|
| **호스팅** | 고객이 제공업체의 결제 페이지로 리디렉션됩니다(예: Stripe Checkout). 가장 간단한 설정이며, 제공업체가 PCI 준수를 처리합니다. |
| **통합** | 결제 양식이 체크아웃 페이지에 직접 삽입됩니다. 매끄러운 경험을 제공하지만, 제공업체의 JavaScript SDK가 필요합니다. |

- **샌드박스 / 라이브 모드** - 테스트용으로 샌드박스 모드에서 시작한 후 준비가 되면 라이브 모드로 전환합니다.

### 단계 4: 연결 테스트

**연결 테스트**를 클릭하여 자격 증명이 유효한지 확인합니다. 마법사는 다음을 확인합니다:
- API 키 인증
- 계정 권한
- 웹후크 엔드포인트의 접근성

### 단계 5: 설정 및 저장

제공업체 설정을 마무리합니다:
- **활성화** - 제공업체를 활성화하거나 비활성화합니다
- **기본 제공업체** - 체크아웃 시 기본 결제 방식으로 설정합니다
- **표시 이름** - 체크아웃 시 고객에게 표시되는 이름입니다
- **정렬 순서** - 체크아웃 시 제공업체가 나타나는 순서를 제어합니다(숫자가 낮을수록 먼저 나타납니다)

## 결제 대시보드

**설정 > 결제 대시보드**로 이동하여 결제 활동의 개요를 확인합니다:

### 처리해야 할 작업

상단의 경고 카드는 주의가 필요한 문제를 강조합니다:
- **실패한 거래** - 처리할 수 없는 결제
- **대기 중인 캡처** - 캡처를 기다리는 승인된 결제
- **연결 오류** - 연결 문제를 겪는 제공업체

### 수익 분석

- **수익 차트** - 시간에 따른 결제량의 시각적 분석, 일, 주, 월 단위로 그룹화됨
- **성능 지표** - 총 수익, 성공률, 평균 거래 금액 및 환불 비율
- **제공업체 비교** - 연결된 각 제공업체의 성능 카드를 병렬적으로 비교

### 거래 분석

- **상태 분포** - 완료, 대기, 실패, 환불된 거래 수
- **결제 방식 혼합** - 고객이 가장 많이 사용하는 결제 방식(신용카드, PayPal, 디지털 지갑)

## 결제 방식 관리

각 제공업체는 다른 결제 방식을 지원합니다. 국가별로 특정 방식을 활성화하거나 비활성화할 수 있습니다:

1. 제공업체의 설정 페이지로 이동합니다
2. **결제 방식** 섹션으로 스크롤합니다
3. 개별 방식을 켜거나 끕니다
4. 국가 수준의 제어를 사용하여 특정 시장에 방식을 제한합니다

이 기능은 특정 지역에서는 인기 있는 결제 방식이 다른 지역에서는 그렇지 않은 경우에 유용합니다(예: 네덜란드의 iDEAL, 벨기에의 Bancontact).

## 웹후크

웹후크는 결제 제공업체와 실시간으로 가게를 동기화합니다. 다음 이벤트를 처리합니다:
- 결제 완료 또는 실패
- 환불 처리
- 분쟁 및 환불 요청 개시
- 구독 갱신

### 자동 설정

제공업체를 연결할 때 Spwig은 자동으로 제공업체에 웹후크 엔드포인트를 등록합니다. 웹후크 URL은 제공업체의 설정 페이지에 표시되어 참조용으로 사용할 수 있습니다.

### 웹후크 모니터링

각 들어오는 웹후크는 다음과 같이 기록됩니다:
- **이벤트 유형** (예: payment_intent.succeeded)
- **시간戳** 및 처리 상태
- **페이로드** 디버깅용

웹후크가 처리에 실패하면 오류로 기록되어 조사할 수 있습니다.

## 여러 제공업체 사용

여러 결제 제공업체를 동시에 연결할 수 있습니다:

- **기본 제공업체** - 체크아웃 시 기본으로 선택되는 제공업체입니다. 제공업체의 설정에서 하나의 제공업체를 기본으로 표시합니다.
- **정렬 순서** - 체크아웃 시 표시 순서를 제어합니다. 고객은 모든 활성 제공업체를 볼 수 있으며 선호하는 방식을 선택할 수 있습니다.
- **대체** - 제공업체가 다운타임을 겪는 경우에도 고객은 대체 제공업체를 사용하여 결제할 수 있습니다.

## 팁

- **Stripe** 또는 **PayPal**부터 시작하세요 - 이들은 가장 넓은 범위의 결제 방식과 지역을 커버합니다.
- **샌드박스/테스트 모드**를 사용하여 라이브 전에 테스트 거래를 처리합니다. 각 제공업체는 문서에 테스트 카드 번호를 제공합니다.
- **여러 제공업체**를 활성화하여 고객이 하나의 제공업체에 문제가 있을 경우 대체 결제 옵션을 제공할 수 있도록 합니다.
- 선호하는 제공업체에 **낮은 정렬 순서**를 설정하여 체크아웃 시 먼저 나타나도록 합니다.
- 매주 **결제 대시보드**를 확인하여 실패한 거래 및 연결 문제를 조기에 발견합니다.
- API 자격 증명을 안전하게 유지하세요 - 데이터베이스에 암호화되어 저장되지만 공유해서는 안 됩니다.

기억하세요: 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어는 보존 규칙에 따라 정확히 유지하세요.
