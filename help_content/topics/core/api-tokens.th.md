---
title: โทเค็น API
---

โทเค็น API เป็นกุญแจที่ปลอดภัยซึ่งอนุญาตให้บริการภายนอกและระบบการผสานรวมสามารถสื่อสารกับร้านค้าของคุณได้ เมื่อบริการภายนอกหรือเครื่องมือที่สามต้องการเข้าถึงข้อมูลของร้านค้าของคุณหรือกระตุ้นการดำเนินการ มันจะส่งโทเค็น API ไปพร้อมกับแต่ละคำขอ เพื่อให้ร้านค้าของคุณสามารถตรวจสอบว่าคำขอได้รับการอนุญาตแล้ว คุณสามารถสร้างและจัดการโทเค็นทั้งหมดจากส่วน API Tokens ในแดชบอร์ดผู้ดูแลระบบของคุณ

## เมื่อคุณต้องการโทเค็น API

คุณมักจะต้องสร้างโทเค็น API เมื่อ:

- เชื่อมต่อกับบริการภายนอกหรือเครื่องมืออัตโนมัติที่ต้องการอ่านหรือเขียนข้อมูลในร้านค้าของคุณ
- ตั้งค่าตัวรับเวิร์กฮุคที่ต้องการตรวจสอบการเรียกใช้ที่เข้ามา
- ตั้งค่าระบบช่วยเหลือ Spwig สำหรับการติดตั้งของคุณ
- สร้างการผสานรวมแบบกำหนดเองโดยใช้ API ของ Spwig
- สังเคราะห์ข้อมูลระหว่างร้านค้า Spwig ของคุณกับระบบอื่น

แต่ละการผสานรวมควรจะมีโทเค็นของตัวเอง เพื่อให้คุณสามารถยกเลิกการเข้าถึงสำหรับบริการหนึ่งโดยไม่กระทบต่ออื่นๆ

## ประเภทของโทเค็น

เมื่อคุณสร้างโทเค็น คุณเลือกประเภทที่อธิบายวัตถุประสงค์ของมัน ประเภทนี้เป็นเพียงข้อมูลอ้างอิงและช่วยให้คุณติดตามว่าโทเค็นแต่ละตัวทำอะไร

| ประเภท | วัตถุประสงค์ |
|--------|-------------|
| **Help System** | ใช้โดยระบบเอกสารช่วยเหลือ Spwig |
| **External Integration** | บริการภายนอก เครื่องมืออัตโนมัติ (เช่น Zapier) หรือเครื่องมือซิงค์ข้อมูล |
| **Webhook** | การตรวจสอบสำหรับตัวรับเวิร์กฮุคหรือปลายทาง |
| **Custom** | วัตถุประสงค์อื่นใดที่ไม่ตรงกับประเภทข้างต้น |
| **Instance Sync** | การสังเคราะห์ระหว่างการติดตั้ง Spwig หรือบริการ Spwig ภายนอก |

## การสร้างโทเค็น API

1. ไปที่ **Settings > API Tokens**
2. คลิก **+ Add API Token**
3. ใส่ **Name** ที่อธิบายชัดเจนว่าโทเค็นนี้ใช้สำหรับอะไร (เช่น `Zapier Product Sync` หรือ `Help System API`)
4. เลือก **Token Type** ที่เหมาะสม
5. เพิ่ม **Description** ที่มีรายละเอียดเพิ่มเติมเกี่ยวกับการผสานรวม (ถ้าต้องการ)
6. ตั้งค่า **Active** สถานะ **Expiry Date** และ **Allowed IPs** ตามที่จำเป็น (ดูข้อมูลด้านล่าง)
7. คลิก **Save**

หลังจากบันทึกข้อมูลแล้ว ค่าโทเคนทั้งหมดจะแสดงบนหน้ารายละเอียด **คัดลอกมันทันที** — โทเคนจะถูกบังไว้ในมุมมองรายการเพื่อความปลอดภัย และไม่สามารถเรียกคืนค่าทั้งหมดได้อีกครั้งหลังจากที่คุณออกจากหน้านี้

## ความปลอดภัยของค่าโทเคน

Spwig จะแสดงค่าโทเคนทั้งหมดเพียงครั้งเดียว: ทันทีหลังจากที่คุณบันทึกโทเคนใหม่ หลังจากนั้น มุมมองรายการจะแสดงเฉพาะรูปแบบที่ถูกบังไว้เท่านั้น (เช่น `spw_••••••••••••••••••••3f8a`)

หากคุณสูญเสียค่าโทเคน คุณจะไม่สามารถกู้คืนมันได้ คุณจะต้องลบโทเคนเก่าและสร้างโทเคนใหม่ แล้วอัปเดตการผสานที่ใช้โทเคนนั้น

**อย่าแชร์ค่าโทเคนผ่านอีเมล ข้อความแชท หรือโค้ดแหล่ง** ให้รักษาไว้เหมือนกับรหัสผ่าน

## การตั้งค่าวันหมดอายุ

ฟิลด์ **Expires At** กำหนดวันและเวลาที่โทเคนจะหยุดทำงานโดยอัตโนมัติ ปล่อยว่างไว้หากโทเคนไม่ควรหมดอายุ

วันหมดอายุมีประโยชน์ในกรณีต่อไปนี้:

- การผสานชั่วคราวที่มีวันสิ้นสุดที่กำหนดไว้
- โทเคนที่ให้กับบุคคลที่สามที่คุณต้องการให้การเข้าถึงถูกยกเลิกโดยอัตโนมัติ
- เพิ่มชั้นความปลอดภัยเพิ่มเติมสำหรับการผสานที่มีสิทธิ์สูง

เมื่อโทเคนหมดอายุ การร้องขอที่ใช้โทเคนนั้นจะถูกปฏิเสธ คุณสามารถขยายการเข้าถึงได้โดยการอัปเดตวันหมดอายุในฟิลด์ **Expires At** หรือสร้างโทเคนแทนที่

## การจำกัดให้ใช้กับที่อยู่ IP ที่เฉพาะเจาะจง

ฟิลด์ **Allowed IPs** รับรายการที่อยู่ IP เมื่อรายการไม่ว่าง โทเคนจะทำงานได้ก็ต่อเมื่อคำขอมาจากที่อยู่เหล่านั้นเท่านั้น

ตัวอย่างเช่น หากเครื่องมือวิเคราะห์ของคุณทำงานบนเซิร์ฟเวอร์ที่ `203.0.113.42` การเพิ่มที่อยู่ IP นี้หมายความว่าโทเคนจะไม่ถูกใช้ผิดพลาดจากสถานที่อื่นใด แม้จะถูกหลบหลีก

ปล่อย **Allowed IPs** ว่างไว้เพื่ออนุญาตให้คำขอจากที่อยู่ IP ใด ๆ

## การติดตามการใช้โทเคน

รายการโทเคนแสดง:

- **Usage Count** — จำนวนครั้งทั้งหมดที่โทเคนถูกใช้งาน
- **Last Used** — เมื่อโทเคนถูกใช้ล่าสุดเพื่อทำคำขอ

ฟิลด์เหล่านี้ช่วยให้คุณระบุโทเคนที่ไม่ได้ใช้งาน (เป็นตัวเลือกสำหรับการยกเลิก) และตรวจจับกิจกรรมที่ไม่คาดคิดได้

การเพิ่มขึ้นอย่างกะทันหันของจำนวนการใช้งานอาจบ่งบอกว่าโทเคนถูกใช้งานโดยบุคคลอื่นนอกเหนือจากอินเทอร์เฟซที่ตั้งใจไว้

## การยกเลิกโทเคน

เพื่อหยุดโทเคนจากการทำงานโดยไม่ต้องลบมัน:

1. คลิกที่ชื่อโทเคน

2. ถอดการเลือก **Active**

3. บันทึก

โทเคนยังคงอยู่ในรายการของคุณเพื่อใช้อ้างอิง แต่จะถูกปฏิเสธในคำขอที่ตามมา ซึ่งมีประโยชน์เมื่อคุณต้องการระงับอินเทอร์เฟซชั่วคราวขณะสืบสวนปัญหา

เพื่อลบโทเคนออกอย่างถาวร:

1. เลือกช่องถูกต้องในรายการ

2. เลือก **Delete selected API tokens** จากเมนูการดำเนินการ

3. ยืนยันการลบ

เมื่อโทเคนถูกลบออกแล้ว จะไม่สามารถกู้คืนได้ หากอินเทอร์เฟซยังต้องการการเข้าถึง ให้สร้างโทเคนใหม่และอัปเดตการตั้งค่าของอินเทอร์เฟซ

## ตัวอย่าง: การตั้งค่าอินเทอร์เฟซ Zapier

**สถานการณ์:** คุณต้องการเชื่อมต่อร้านค้าของคุณกับ Zapier เพื่อให้สามารถแจ้งเตือนคำสั่งซื้ออัตโนมัติได้

| Field | Value |

|-------|-------|

| Name | `Zapier Order Automation` |

| Token Type | External Integration |

| Description | Used by Zapier to read new orders and trigger notifications |

| Active | Yes |

| Expires At | *(leave blank)* |

| Allowed IPs | *(leave blank — Zapier uses dynamic IPs)* |

หลังจากบันทึกแล้ว ให้คัดลอกค่าโทเคนทั้งหมดและวางไว้ในการตั้งค่าอินเทอร์เฟซ Spwig ของ Zapier

## เคล็ดลับ

รักษาการจัดรูปแบบ markdown ทั้งหมด ค่าเส้นทางรูปภาพ บล็อกโค้ด และคำศัพท์ทางเทคนิคไว้

- ให้ทุก token มีชื่อที่ชัดเจนและเฉพาะเจาะจง — `Shopify Sync v2` มีประโยชน์มากกว่า `Token 3` มากเมื่อคุณกำลังแก้ไขปัญาในภายหลัง
- สร้าง token หนึ่งต่อการผสาน (integration) — หากการผสานถูกบุกรุก คุณสามารถยกเลิก token นั้นเพียงตัวเดียวโดยไม่กระทบต่อ token อื่นๆ
- ตั้งวันหมดอายุสำหรับ token ที่ใช้ในโครงการครั้งเดียวหรือการผสานชั่วคราว — นี่จะลดความเสี่ยงที่ token ที่ถูกลืมจะยังคงใช้งานได้ตลอดไป
- ตรวจสอบรายการ token ทุกไม่กี่เดือน และปิดใช้งาน token ใดๆ ที่มีวันที่ **Last Used** ที่เก่าผิดปกติ เนื่องจากอาจเป็น token ที่ใช้กับการผสานที่ไม่ได้ทำงานอีกต่อไปแล้ว
- หากคุณสงสัยว่า token ถูกเปิดเผย ให้ปิดใช้งานทันที สร้าง token แทนที่ และอัปเดตการผสานที่ได้รับผลกระทบก่อนเปิดใช้งานอีกครั้ง
