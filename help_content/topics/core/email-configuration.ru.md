---
title: Настройка электронной почты
---

Настройка электронной почты управляет тем, как ваш магазин отправляет транзакционные электронные письма — подтверждения заказов, уведомления о доставке, сброс паролей и многое другое. Spwig включает встроенного SMTP-сервер и поддерживает внешние провайдеры электронной почты для более высокой доставляемости.

![Email accounts](/static/core/admin/img/help/email-configuration/email-accounts.webp)

## Доступные провайдеры

| Провайдер | Описание |
|----------|-------------|
| **Встроенный SMTP** | Бесплатный, самозагружаемый сервер электронной почты, включенный в Spwig. Автоматическая подпись DKIM. |
| **Gmail API** | Отправка через ваш аккаунт Gmail или Google Workspace с использованием аутентификации OAuth. |
| **Общий SMTP** | Подключите любой SMTP-сервер (SendGrid, Mailgun, Amazon SES или ваш собственный почтовый сервер). |

## Настройка электронной почты

Перейдите в **Настройки > Аккаунты электронной почты** и нажмите **Добавить аккаунт электронной почты**, чтобы запустить мастер настройки.

### Шаг 1: Выберите провайдера

Выберите своего провайдера электронной почты. Встроенный SMTP-сервер — это самый простой способ начать — он не требует внешних аккаунтов.

### Шаг 2: Настройка учетных данных

Введите учетные данные для выбранного провайдера:

- **Встроенный SMTP** — Учетные данные не требуются. Сервер работает на вашей установке Spwig.
- **Gmail API** — Аутентифицируйтесь через Google OAuth. Вас перенаправят для входа в ваш аккаунт Google.
- **Общий SMTP** — Введите адрес SMTP-сервера, порт, имя пользователя и пароль.

### Шаг 3: Настройка отправителя

Установите идентичность отправителя для исходящих электронных писем:

- **Email отправителя** — Адрес электронной почты, который отображается в поле "От" (например, orders@yourstore.com)
- **Имя отправителя** — Отображаемое имя рядом с адресом электронной почты (например, "Название вашего магазина")
- **Email для ответа** — Адрес, куда будут направлены ответы клиентов (может отличаться от адреса отправителя)

### Шаг 4: Проверка DNS

Проверьте записи аутентификации электронной почты вашего домена. Мастер проверяет три DNS-записи:

| Запись | Назначение |
|--------|---------|
| **SPF** | Разрешает вашему серверу отправлять электронные письма от имени вашего домена |
| **DKIM** | Цифровая подпись электронных писем для доказательства, что они не были изменены |
| **DMARC** | Сообщает принимающим серверам, что делать с электронными письмами, которые не прошли проверки SPF/DKIM |

Для каждой записи мастер показывает:
- **Текущий статус** — настроен ли запись правильно
- **Требуемое значение** — точная DNS-запись, которую нужно добавить в ваш регистратор домена
- **Статус распространения** — настроены ли недавние изменения (изменения DNS могут занять до 48 часов)

Встроенный SMTP-сервер автоматически генерирует ключи DKIM для вашего домена.

### Шаг 5: Отправка тестового электронного письма

Отправьте тестовое электронное письмо, чтобы проверить, все ли работает:
1. Введите адрес электронной почты получателя
2. Нажмите **Отправить тест**
3. Проверьте свою почту на наличие тестового сообщения
4. Убедитесь, что электронное письмо пришло без предупреждений о спаме

### Шаг 6: Сохранить и активировать

Сохраните настройки и установите аккаунт как активный. Отметьте его как **По умолчанию**, если он должен быть основным аккаунтом электронной почты.

## Шаблоны электронной почты

Spwig включает более 30 шаблонов электронной почты для каждого транзакционного события. Перейдите в **Настройки > Шаблоны электронной почты**, чтобы управлять ими.

### Типы шаблонов

Шаблоны охватывают все события магазина, включая:
- **Жизненный цикл заказа** — подтверждение, обработка, отправка, доставка, отмена
- **Оплата** — чек, подтверждение возврата, провал оплаты
- **Аккаунт клиента** — приветствие, сброс пароля, проверка электронной почты
- **Подарочные карты** — доставка, уведомление о балансе
- **Доставка** — обновления отслеживания, подтверждение доставки
- **Цифровые продукты** — ссылки для загрузки, лицензионные ключи
- **Маркетинг** — восстановление забытой корзины, запросы отзывов

### Настройка шаблонов

1. Перейдите в список шаблонов
2. Нажмите на шаблон, чтобы отредактировать
3. Измените тему, заголовок, содержимое тела и подвал
4. Используйте переменные шаблона (например, `{{ order.number }}`, `{{ customer.name }}`) для динамического содержимого
5. Предварительный просмотр электронного письма перед сохранением

### Поддержка нескольких языков

Шаблоны электронной почты поддерживают несколько языков:
- Каждый шаблон может иметь переводы на все активные языки вашего магазина
- Система отправляет электронные письма на предпочитаемом языке клиента
- **Цепочка падения языка** — если перевод недоступен, система переходит к языку по умолчанию магазина
- Используйте функцию **Автоматического перевода** для автоматического перевода шаблонов на другие языки

### Клонирование шаблонов

Чтобы создать настроенную версию системного шаблона:
1. Откройте шаблон, который хотите изменить
2. Нажмите **Клонировать шаблон**
3. Редактируйте склонированную версию
4. Клон имеет приоритет над оригинальным системным шаблоном

## Очередь электронной почты

Отслеживайте исходящие электронные письма в **Настройках > Очереди электронной почты**:

- **В очереди** — электронные письма, ожидающие отправки
- **Отправка** — в настоящее время передаются
- **Отправлено** — успешно доставлено
- **Не удалось** — не удалось доставить (с подробностями ошибки)
- **Отброшено** — отклонено почтовым сервером получателя

Нажмите на любое электронное письмо, чтобы просмотреть его полные детали, включая получателя, тему, время отправки и статус доставки.

## Отслеживание доставки

Отслеживайте вовлеченность в электронной почте:
- **Открытия** — сколько получателей открыли электронное письмо
- **Клики** — клики по ссылкам в электронном письме
- **Отказы** — отслеживание жестких и мягких отказов
- **Жалобы** — жалобы на спам от получателей

## Несколько аккаунтов

Вы можете настроить несколько аккаунтов электронной почты:
- **Основной аккаунт** — используется для всех исходящих электронных писем, если не указано иное
- **Резервный** — если основной аккаунт не удается использовать, электронные письма помещаются в очередь для повторной попытки отправки
- Используйте разные аккаунты для разных целей (например, один для транзакционных электронных писем, другой для маркетинга)

## Советы

- Начните с **встроенного SMTP-сервера** для быстрой настройки, а затем переключитесь на внешнего провайдера, если вам нужны более высокие объемы отправки или лучшая доставляемость.
- Всегда настраивайте **SPF, DKIM и DMARC** — без них электронные письма гораздо вероятнее окажутся в папке спама.
- Отправьте **тестовое электронное письмо** после любого изменения настройки, чтобы проверить, работает ли доставка.
- Регулярно отслеживайте очередь электронной почты на наличие **неудачных** или **отброшенных** электронных писем — это указывает на проблемы с доставкой.
- Используйте **профессиональный адрес отправителя** (например, orders@yourstore.com) вместо бесплатного адреса электронной почты для лучшего доверия и доставляемости.
- Держите свои шаблоны краткими — транзакционные электронные письма должны быстро передавать информацию, а не быть рассылками маркетинга.
