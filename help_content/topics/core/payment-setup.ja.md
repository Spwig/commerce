---
title: 支払いの設定
---

支払いプロバイダは、ストアを支払いゲートウェイに接続し、チェックアウト時にクレジットカード、デジタルウォレット、その他の支払い方法を受け取れるようにします。Spwigは複数のプロバイダを同時にサポートしており、顧客に柔軟な支払いオプションを提供します。

![支払いプロバイダ](/static/core/admin/img/help/payment-setup/payment-dashboard.webp)

## 利用可能なプロバイダ

| プロバイダ | 説明 |
|----------|-------------|
| **Stripe** | クレジットカード、Apple Pay、Google Pay、および135以上の通貨 |
| **PayPal** | PayPal残高、クレジット/デビットカード、および後払いオプション |
| **Airwallex** | 輸出入ビジネス向けに最適化された多通貨支払い |
| **Adyen** | 世界中で250以上の支払い方法をサポートする企業向けの支払い |
| **Square** | POS端末を統合した、店頭およびオンライン支払い |
| **Revolut** | 競争力のある為替レートを備えた高速なヨーロッパ支払い |

## プロバイダの接続

**設定 > 支払いプロバイダ**に移動し、**プロバイダを接続**をクリックして設定ウィザードを起動します。

### ステップ1: プロバイダの選択

利用可能な支払いプロバイダから選択します。各カードには、プロバイダがサポートする機能と地域が表示されます。

### ステップ2: 設定手順

プロバイダ固有の設定ガイドを確認してください。これは以下を含みます:
- プロバイダのアカウントを作成する方法（まだアカウントを持っていない場合）
- プロバイダのダッシュボードでAPI資格情報をどこで見つけるか
- 事前条件（例: ビジネスの確認）

### ステップ3: 資格情報の入力

API資格情報を入力してください:
- **APIキー / シークレットキー** — プロバイダのダッシュボードから取得した認証資格情報
- **チェックアウトモード** — 顧客が支払いフォームとどのようにやり取りするかを選択します:

| モード | 説明 |
|------|-------------|
| **ホスト済み** | 顧客はプロバイダの支払いページにリダイレクトされます（例: Stripe Checkout）。最も簡単な設定で、PCI準拠はプロバイダが処理します。 |
| **統合済み** | 支払いフォームはチェックアウトページに直接埋め込まれます。スムーズな体験ですが、プロバイダのJavaScript SDKが必要です。 |

- **サンドボックス/ライブモード** — テストのためにサンドボックスモードから開始し、準備ができたらライブモードに切り替えてください

### ステップ4: 接続のテスト

**接続をテスト**をクリックして、資格情報が有効であることを確認します。ウィザードは以下を確認します:
- APIキー認証
- アカウントの権限
- Webhookエンドポイントの可達性

### ステップ5: 設定と保存

プロバイダの設定を確定します:
- **有効** — プロバイダを有効または無効にします
- **デフォルトプロバイダ** — チェックアウト時にプライマリ支払い方法として設定します
- **表示名** — チェックアウト時に顧客に表示される名前
- **並べ替え順序** — チェックアウト時にプロバイダが表示される順序を制御します（数字が小さいほど先に表示されます）

## 支払いダッシュボード

**設定 > 支払いダッシュボード**に移動して、支払い活動の概要を確認してください:

### 必要なアクション

上部のアラートカードは、注意が必要な問題を強調表示します:
- **失敗したトランザクション** — 処理できなかった支払い
- **キャプチャ待ち** — 承認された支払いがキャプチャ待ち
- **接続エラー** — 接続問題のあるプロバイダ

### 收益分析

- **収益チャート** — 時間ごとの支払い量の視覚的な分解、日、週、または月ごとにグループ化されます
- **パフォーマンスメトリクス** — 総収益、成功率、平均取引額、および返金率
- **プロバイダ比較** — 接続された各プロバイダのパフォーマンスカードを並べて表示

### トランザクションの分解

- **ステータス分布** — 完了、保留、失敗、および返金されたトランザクションの数
- **支払い方法の組み合わせ** — 顧客が最もよく使用する支払い方法（クレジットカード、PayPal、デジタルウォレット）

## 支払い方法の管理

各プロバイダは異なる支払い方法をサポートしています。国ごとに特定の方法を有効または無効にできます:

1. プロバイダの設定ページに移動します
2. **支払い方法**セクションにスクロールします
3. 個々の方法をオン/オフに切り替えます
4. 国レベルのコントロールを使用して、特定の市場に方法を制限します

これは、ある地域では人気のある支払い方法が他の地域ではそうではない場合に役立ちます（例: オランダのiDEAL、ベルギーのBancontact）。

## Webhook

Webhookは、支払いプロバイダとリアルタイムでストアを同期します。以下のようなイベントを処理します:
- 支払いの完了または失敗
- 返金の処理
- 紛争およびチャージバックの開設
- サブスクリプションの更新

### 自動設定

プロバイダを接続すると、SpwigはプロバイダにWebhookエンドポイントを自動的に登録します。Webhook URLは、プロバイダの設定ページに表示されます。

### Webhookの監視

各入力されたWebhookは、以下とともに記録されます:
- **イベントタイプ**（例: payment_intent.succeeded）
- **タイムスタンプ**と処理ステータス
- **ペイロード**でデバッグ

Webhookが処理に失敗した場合、エラーとして記録され、調査できます。

## 複数のプロバイダの使用

複数の支払いプロバイダを同時に接続できます:

- **デフォルトプロバイダ** — チェックアウト時にデフォルトで選択されるプロバイダ。設定で1つのプロバイダをデフォルトに指定してください。
- **並べ替え順序** — チェックアウト時の表示順序を制御します。顧客はすべての有効なプロバイダを表示し、希望のものを選択できます。
- **フェイルオーバー** — プロバイダがダウンしている場合でも、顧客は代替のプロバイダを使用して支払いできます。

## ヒント

- **Stripe**または**PayPal**から開始してください — これらは最も幅広い支払い方法と地域をカバーしています。
- **サンドボックス/テストモード**を使用して、本番環境に移行する前にテストトランザクションを処理してください。各プロバイダのドキュメントにはテスト用のカード番号が含まれています。
- **複数のプロバイダ**を有効にして、1つのプロバイダに問題がある場合に顧客に代替の支払いオプションを提供します。
- ご希望のプロバイダに**低い並べ替え順序**を設定して、チェックアウト時に最初に表示されるようにしてください。
- 週に一度**支払いダッシュボード**を確認して、失敗したトランザクションや接続問題を早期に発見してください。
- **API資格情報**を安全に保つ — データベースに暗号化して保存されていますが、共有してはいけません。
