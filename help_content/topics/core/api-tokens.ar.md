---
title: مفاتيح API
---

مفاتيح API هي مفاتيح آمنة تسمح للخدمات الخارجية والتكاملات بالاتصال بمخزنتك. عندما تحتاج خدمة ثالثة أو أداة إلى الوصول إلى بيانات مخزنتك أو تشغيل أفعال، فإنها ترسل مفتاح API مع كل طلب بحيث يمكن لمخزنتك التحقق من أن الطلب مُصادق عليه. تقوم بإنشاء وإدارة جميع المفاتيح من قسم مفاتيح API في لوحة التحكم الخاصة بك.

## متى تحتاج إلى مفتاح API

عادةً ما تحتاج إلى إنشاء مفتاح API عندما:

- تربط خدمة خارجية أو أداة تلقائية تحتاج إلى قراءة أو كتابة في مخزنتك
- تنشئ مستقبل webhook يحتاج إلى مصادقة المكالمات الواردة
- تقوم بتكوين نظام مساعدة Spwig لتركيبك
- تبني تكاملًا مخصصًا باستخدام API الخاص بـ Spwig
- تقوم بتوسيع البيانات بين مخزنتك في Spwig ونظام آخر

يجب أن يكون لكل تكامل مفتاحه الخاص بحيث يمكنك إلغاء إذن الوصول لخدمة واحدة دون التأثير على غيرها.

## أنواع المفاتيح

عند إنشاء مفتاح، تختار نوعًا يصف غرضه. النوع هو لمرجعيتك فقط ويُساعدك على تتبع ما يفعله كل مفتاح.

| النوع | الغرض |
|------|---------|
| **نظام المساعدة** | يُستخدم من قبل نظام وثائق المساعدة في Spwig |
| **التكامل الخارجي** | الخدمات الخارجية، أدوات التلقائية (مثلاً Zapier)، أو أدوات التزامن |
| **Webhook** | مصادقة مستقبلات webhook أو نقاط النهاية |
| **مخصص** | أي غرض آخر لا ينطبق على الفئات أعلاه |
| **التوافق بين المثيلات** | التزامن بين تركيبات Spwig أو الخدمات الخارجية لـ Spwig |

## إنشاء مفتاح API

1. انتقل إلى **الإعدادات > مفاتيح API**
2. انقر على **+ إضافة مفتاح API**
3. أدخل **اسمًا** يصف بوضوح الغرض من المفتاح (مثلاً `Zapier Product Sync` أو `Help System API`)
4. اختر النوع المناسب للمفتاح
5. أضف **وصفًا** مع تفاصيل إضافية عن التكامل (اختياري)
6. أعد تكوين حالة **الفعالية**، **تاريخ الانتهاء**، و**العناوين IP المسموح بها** حسب الحاجة (انظر أدناه)
7. انقر على **حفظ**

بعد الحفظ، يظهر القيمة الكاملة للرمز على صفحة التفاصيل. **انسخها فورًا** — يتم إخفاء الرمز في عرض القائمة لأسباب أمنية، ولا يمكن استرجاعه بالكامل بعد مغادرتك لهذه الصفحة.

## أمان قيمة الرمز

يعرض Spwig قيمة الرمز الكاملة مرة واحدة فقط: فورًا بعد حفظ رمز جديد. بعد ذلك، يعرض عرض القائمة فقط نسخة مخفية منه (مثلاً: `spw_••••••••••••••••••••3f8a`).

إذا فقدت قيمة الرمز، فلن تستطيع استعادتها. ستحتاج إلى حذف الرمز القديم وإنشاء رمز جديد، ثم تحديث التكامل الذي كان يستخدمه.

**لا تشارك قيم الرموز عبر البريد الإلكتروني أو الرسائل الخاصة أو الشيفرة المصدرية.** عاملها ككلمات مرور.

## تحديد تاريخ انتهاء الصلاحية

حقل **يُنتهي في** يحدد تاريخًا ووقتًا بعد ذلك سيتوقف عن العمل تلقائيًا. اتركه فارغًا للرموز التي لا يجب أن تنتهي صلاحيتها.

تُعد تواريخ الانتهاء مفيدة ل:

- التكاملات المؤقتة ذات التاريخ النهائي الثابت
- الرموز المقدمة لطرف ثالث حيث ترغب في إزالة الوصول تلقائيًا
- إضافة طبقة أمان إضافية للتكاملات ذات الصلاحيات العالية

عند انتهاء صلاحية الرمز، تُرفض الطلبات التي تستخدمه. يمكنك تمديد الوصول بتحديث تاريخ **يُنتهي في** أو بإنشاء رمز بديل.

## تقييد الوصول إلى عناوين IP معينة

حقل **العناوين IP المسموح بها** يقبل قائمة من عناوين IP. عندما تكون القائمة غير فارغة، يعمل الرمز فقط عندما يأتي الطلب من أحد هذه العناوين.

على سبيل المثال، إذا كانت أداة تحليلاتك تعمل على خادم بعنوان `203.0.113.42`، فإن إضافة هذا العنوان يعني أن الرمز لا يمكن استخدامه من أي مكان آخر، حتى لو تم تسريبه.

اترك حقل **العناوين IP المسموح بها** فارغًا للسماح بالطلبات من أي عنوان IP.

## مراقبة استخدام الرموز

تُظهر قائمة الرموز:

- **عدد الاستخدامات** — عدد مرات استخدام الرمز الإجمالي
- **الاستخدام الأخير** — متى تم استخدام الرمز لإجراء طلب

تساعد هذه الحقول في تحديد الرموز غير المستخدمة (المرشحة للإلغاء) وتحديد الأنشطة غير المتوقعة.

ارتفاع مفاجئ في عدد الاستخدامات قد يشير إلى أن الرمز يتم استخدامه من قبل شخص آخر غير التكامل المقصود.

## إلغاء رمز

لإيقاف الرمز فورًا دون حذفه:

1. اضغط على اسم الرمز
2. قم بإلغاء تحديد **النشاط**
3. احفظ

يبقى الرمز في قائمتك كمرجع ولكن يتم رفضه في أي طلبات لاحقة. هذا مفيد عندما تحتاج إلى تعطيل تكامل مؤقتًا أثناء التحقيق في مشكلة ما.

للحذف الدائم لرمز:

1. اختر مربع الاختيار الخاص به في القائمة
2. اختر **حذف الرموز المحددة لواجهة برمجة التطبيقات** من قائمة الأفعال
3. تأكيد الحذف

بعد الحذف، لا يمكن استعادة الرمز. إذا كان التكامل لا يزال يحتاج إلى الوصول، أنشئ رمزًا جديدًا وتحديث إعدادات التكامل.

## مثال: إعداد تكامل Zapier

**الموقف:** ترغب في ربط متجرك بـ Zapier لتوسيع إشعارات الطلبات.

| الحقل | القيمة |
|-------|-------|
| الاسم | `Zapier Order Automation` |
| نوع الرمز | تكامل خارجي |
| الوصف | يستخدمه Zapier لقراءة الطلبات الجديدة وإرسال إشعارات |
| النشاط | نعم |
| ينتهي في | *(اتركه فارغًا)* |
| الـIP المسموح به | *(اتركه فارغًا — Zapier يستخدم أكواد IP ديناميكية)* |

بعد الحفظ، قم بنسخ قيمة الرمز الكامل ولصقه في إعدادات تكامل Spwig في Zapier.

## نصائح

احتفظ بجميع التنسيقات المرتبطة بالماكرو، مسارات الصور، كتل الشفرة، والكلمات الفنية.

- امنح كل رمز اسمًا واضحًا ومحددًا - `Shopify Sync v2` أكثر فائدة من `Token 3` عندما تكون في عملية استكشاف الأخطاء الشائعة بعد شهور
- أنشئ رمزًا واحدًا لكل تكامل - إذا تمت اختراق إحدى التكاملات، يمكنك إلغاء تفعيل ذلك الرمز فقط دون إحداث أي تأثير على غيره
- حدد تاريخ انتهاء الصلاحية للرموز المستخدمة في مشاريع مؤقتة أو تكاملات مؤقتة - هذا يقلل من خطر وجود رموز مهملة تظل نشطة إلى الأبد
- تحقق من قائمة الرموز كل بضعة أشهر وتعطيل أي رموز لها تاريخ **آخر استخدام** غير متوقع، حيث قد تعود إلى تكاملات لم تعد تعمل
- إذا كنت تشك في أن رمزًا قد تم تسريبه، تعطيله فورًا، أنشئ استبدالًا، وتحديث التكامل المتضرر قبل إعادة تفعيل الوصول
