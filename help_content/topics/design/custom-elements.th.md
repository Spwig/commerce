องค์ประกอบแบบกำหนดเองช่วยให้คุณสร้างบล็อกของตัวปรับแต่งหน้าที่สามารถใช้ซ้ำได้ตามความต้องการของร้านค้าของคุณ คุณสามารถออกแบบองค์ประกอบแบบกำหนดเองได้ด้วยเครื่องมือที่มีอยู่ในตัวของตัวปรับแต่งหน้า จากนั้นเชื่อมต่อกับข้อมูลของร้านค้าจริง เช่น ชื่อสินค้า ราคา หรือรูปภาพ เพื่อให้องค์ประกอบนี้เติมข้อมูลจริงโดยอัตโนมัติเมื่อวางไว้ในหน้า หลังจากสร้างองค์ประกอบแบบกำหนดเองแล้ว องค์ประกอบเหล่านี้จะปรากฏในไลบรารีองค์ประกอบของตัวปรับแต่งหน้าร่วมกับบล็อกที่มีอยู่ในระบบ

## ใช้งานองค์ประกอบแบบกำหนดเองเมื่อใด

องค์ประกอบแบบกำหนดเองมีประโยชน์มากเมื่อคุณพบว่าตัวเองต้องสร้างรูปแบบเดียวกันซ้ำๆ แทนที่จะสร้าง "บัตรสินค้าแนะนำ" ใหม่ทุกครั้งในแต่ละหน้า คุณสร้างมันเพียงครั้งเดียวเป็นองค์ประกอบแบบกำหนดเองและวางไว้ที่ใดก็ได้ที่คุณต้องการ หากองค์ประกอบนั้นเชื่อมต่อกับข้อมูล องค์ประกอบจะดึงข้อมูลสินค้าล่าสุดโดยอัตโนมัติ — ไม่จำเป็นต้องอัปเดตด้วยมือเมื่อราคาหรือชื่อเปลี่ยนไป

การใช้งานที่พบบ่อย:

- บัตรแสดงสินค้าที่แสดงชื่อ ราคา และรูปภาพหลัก
- บล็อกโปรโมชันหมวดหมู่ที่มีแบนเนอร์ หัวข้อ และลิงก์
- แผงแสดงแบรนด์ที่มีโลโก้และคำอธิบาย
- ข้อความย่อของบทความบล็อกที่มีรูปภาพหลัก หัวข้อ และข้อความย่อ

## การสร้างองค์ประกอบแบบกำหนดเองใหม่

1. ไปที่ **Design > Custom Elements**
2. คลิก **+ Add Custom Element**
3. Spwig จะสร้างองค์ประกอบแบบกำหนดเองร่างเริ่มต้นทันทีและเปิด **Visual Builder** — คุณไม่จำเป็นต้องกรอกฟอร์มก่อน
4. ใน Visual Builder สร้างรูปแบบขององค์ประกอบโดยใช้เครื่องมือตัวปรับแต่งหน้าที่มีอยู่
5. เมื่อคุณพอใจกับการออกแบบ ตั้งค่าองค์ประกอบ (ชื่อ การเชื่อมต่อกับข้อมูล ไอคอน) ในแถบด้านข้าง
6. เปิดใช้งาน **Active** เมื่อคุณพร้อมที่จะเผยแพร่องค์ประกอบไปยังไลบรารี
7. บันทึกองค์ประกอบ

องค์ประกอบนี้จะพร้อมใช้งานในแผงองค์ประกอบของตัวปรับแต่งหน้าภายใต้หมวดหมู่ที่คุณกำหนดไว้

## Visual Builder

Visual Builder เป็นแคนวาสเฉพาะสำหรับการออกแบบองค์ประกอบของคุณ

มันทำงานเหมือนกับเครื่องมือสร้างหน้ามาตรฐาน แต่เน้นที่องค์ประกอบเดียวแทนที่จะเป็นหน้าทั้งหน้า

คุณสามารถ:

- เพิ่มและจัดวางองค์ประกอบย่อย (บล็อกข้อความ รูปภาพ คอนเทนเนอร์ ฯลฯ)
- ตั้งค่าสไตล์ ระยะห่าง และการจัดวางสำหรับแต่ละองค์ประกอบย่อย
- ดูตัวอย่างว่าองค์ประกอบจะดูอย่างไรเมื่อมีข้อมูลตัวอย่าง

การเปลี่ยนแปลงใน Visual Builder จะถูกบันทึกโดยตรงไปยังการกำหนดค่าขององค์ประกอบ ไม่มีขั้นตอนการเผยแพร่แยกต่างหาก — การบันทึกใน builder จะอัปเดตองค์ประกอบทันทีสำหรับหน้าที่ใช้งานอยู่แล้ว

## การตั้งค่าองค์ประกอบ

แต่ละองค์ประกอบที่กำหนดเองมีการตั้งค่าดังนี้:

| Field | Description |
|-------|-------------|
| **Name** | ชื่อที่แสดงในไลบรารีองค์ประกอบ |
| **Slug** | ตัวระบุที่ปลอดภัยสำหรับ URL สร้างโดยอัตโนมัติจากชื่อ |
| **Description** | หมายเหตุเพิ่มเติมเกี่ยวกับว่าองค์ประกอบนี้ใช้เพื่ออะไร |
| **Target Model** | โมเดลร้านค้าที่จะใช้เพื่อดึงข้อมูล (ดูข้อมูลด้านล่าง) |
| **Icon** | ไอคอนที่แสดงในไลบรารีองค์ประกอบ |
| **Category** | จัดกลุ่มองค์ประกอบที่เกี่ยวข้องกันในไลบรารี |
| **Active** | ระบุว่าองค์ประกอบนี้สามารถใช้งานได้ในเครื่องมือสร้างหน้าหรือไม่ |

## การผูกข้อมูล

การผูกข้อมูลเชื่อมโยงส่วนต่าง ๆ ของรูปแบบองค์ประกอบของคุณกับข้อมูลร้านค้าที่กำลังทำงานอยู่ เมื่อผู้แก้ไขหน้าวางองค์ประกอบที่ผูกข้อมูลไว้ในหน้า เขาจะเลือกบันทึกเฉพาะ (เช่น สินค้า) และทุกฟิลด์ที่ผูกจะถูกเติมข้อมูลอัตโนมัติจากบันทึกนั้น

### การเลือกโมเดลเป้าหมาย

การตั้งค่า **Target Model** กำหนดประเภทของข้อมูลร้านค้าที่องค์ประกอบสามารถแสดงได้ โมเดลที่มีอยู่คือ:

| Model | What it provides |
|-------|-----------------|
| **Product** | ชื่อ ราคา สถานะสต็อก รูปภาพ คำอธิบาย SKU หมวดหมู่ แบรนด์ และอื่น ๆ |
| **Category** | ชื่อ คำอธิบาย รูปภาพ แบนเนอร์ จำนวนสินค้า และ URL |
| **Brand** | ชื่อ โลโก้ คำอธิบาย ประวัติแบรนด์ และ URL |
| **Blog Post** | หัวข้อ บทสรุป รูปภาพเด่น ผู้เขียน วันที่เผยแพร่ และ URL |

ปล่อยให้ **Target Model** ว่างไว้เพื่อสร้างองค์ประกอบแบบสถิตที่ไม่มีข้อมูลที่เปลี่ยนแปลงได้

องค์ประกอบแบบคงที่มีประโยชน์สำหรับองค์ประกอบการออกแบบที่ไม่เปลี่ยนแปลง เช่น แบนเนอร์ตกแต่ง หรือส่วนเว้นวรรคของเค้าโครง

### การทำงานของการผูกข้อมูล

ภายใน Visual Builder คุณสามารถทำเครื่องหมายองค์ประกอบย่อยแต่ละตัวว่าเป็นข้อมูลที่ผูกไว้ได้โดยการเลือกฟิลด์โมเดลที่ควรแสดง ตัวอย่างเช่น:

- องค์ประกอบย่อย **ข้อความ** สามารถผูกกับ **ชื่อสินค้า** เพื่อให้แสดงชื่อสินค้าที่เลือก
- องค์ประกอบย่อย **รูปภาพ** สามารถผูกกับ **รูปภาพหลัก** เพื่อให้แสดงรูปภาพหลักของสินค้า
- องค์ประกอบย่อย **ข้อความ** สามารถผูกกับ **ราคา** เพื่อให้แสดงราคาปัจจุบันเสมอ

การผูกแต่ละตัวจะจับคู่ฟิลด์เนื้อหาขององค์ประกอบหนึ่งกับฟิลด์โมเดลหนึ่ง คุณสามารถเพิ่มการผูกหลายตัวในองค์ประกอบที่กำหนดเองได้ — เช่น การผูกบล็อกข้อความกับ **ชื่อสินค้า** และบล็อกรูปภาพแยกต่างหากกับ **รูปภาพหลัก** พร้อมกัน

### การตั้งค่ารูปภาพย่อหน้า

สำหรับการผูกรูปภาพ คุณสามารถระบุ **ตัวตั้งค่ารูปภาพย่อหน้า** (เช่น `thumbnail` หรือ `medium`) ได้เป็นตัวเลือก ซึ่งจะควบคุมขนาดของรูปภาพที่โหลด ช่วยให้หน้าเว็บโหลดเร็วขึ้นโดยการส่งรูปภาพที่มีขนาดเหมาะสมกับเค้าโครงขององค์ประกอบ

## การปิดใช้งานและเปิดใช้งานองค์ประกอบอีกครั้ง

การปิดใช้งานองค์ประกอบจะลบมันออกจากไลบรารีองค์ประกอบ ทำให้ไม่สามารถเพิ่มลงในหน้าใหม่ได้ หน้าที่มีอยู่แล้วที่ใช้องค์ประกอบนี้จะไม่ได้รับผลกระทบ — องค์ประกอบยังคงแสดงผลบนหน้าเหล่านั้น

เพื่อปิดใช้งาน:

1. ไปที่ **Design > Custom Elements**
2. คลิกที่ชื่อองค์ประกอบ
3. ยกเลิกการเลือก **Active**
4. บันทึก

เพื่อเปิดใช้งานอีกครั้ง ให้ทำตามขั้นตอนเดียวกันและเลือก **Active** อีกครั้ง

## การกรองไลบรารีองค์ประกอบ

รายการองค์ประกอบรองรับการกรองตาม:

- **Active / Inactive** — แสดงเฉพาะองค์ประกอบที่เผยแพร่หรือเฉพาะร่างเท่านั้น
- **Target Model** — กรองตามโมเดลที่องค์ประกอบผูกไว้
- **Category** — กรองตามหมวดหมู่ขององค์ประกอบ
- **Search** — ค้นหาตามชื่อ, slug หรือคำอธิบาย

นี่ช่วยได้เมื่อคุณมีองค์ประกอบที่กำหนดเองจำนวนมากและต้องการค้นหาองค์ประกอบเฉพาะอย่างรวดเร็ว

## ตัวอย่าง: บัตรเน้นสินค้า

**เป้าหมาย:** องค์ประกอบแบบได้รับการจัดรูปแบบที่แสดงภาพหลัก ชื่อ และราคาของสินค้า

| ตั้งค่า | ค่า |
|---------|-------|
| ชื่อ | Product Highlight Card |
| Target Model | Product |
| หมวดหมู่ | Products |
| ไอคอน | fas fa-box |

ใน Visual Builder ให้เพิ่ม:
- องค์ประกอบ **Image** ที่ผูกกับ **Main Image** พร้อมตั้งค่ารูปย่อ `medium`
- องค์ประกอบ **Text** ที่ผูกกับ **Product Name**
- องค์ประกอบ **Text** ที่ผูกกับ **Price**

เมื่อบันทึกและเปิดใช้งานแล้ว องค์ประกอบนี้จะปรากฏในตัวสร้างหน้าภายใต้หมวดหมู่ Products เมื่อผู้แก้ไขหน้าเพิ่มมันเข้าไปในหน้า เขาจะเลือกสินค้าที่ต้องการนำเสนอ และการ์ดจะอัปเดตข้อมูลอัตโนมัติ

## เคล็ดลับ

- ให้ชื่อองค์ประกอบที่อธิบายถึงวัตถุประสงค์และประเภทข้อมูลของมัน เช่น "Product Highlight Card" แทนที่จะใช้ "Card 1" เพื่อให้ไลบรารียังคงง่ายต่อการนำทางเมื่อขยายตัว
- ใช้ฟิลด์ **Category** เพื่อจัดกลุ่มองค์ประกอบที่เกี่ยวข้องกัน (Products, Blog, Promotions) — นี่ช่วยให้ไลบรารีองค์ประกอบมีการจัดระเบียบสำหรับผู้แก้ไขหน้าของคุณ
- ทดสอบองค์ประกอบที่ผูกกับข้อมูลโดยการเพิ่มมันลงในหน้าร่างและเลือกบันทึกจริงก่อนเผยแพร่ เพื่อยืนยันว่าการผูกข้อมูลดึงข้อมูลที่ถูกต้อง
- ปิดใช้งานองค์ประกอบที่ล้าสมัยแทนที่จะลบ — นี่ช่วยรักษาหน้าที่ยังคงอ้างอิงถึงมันไว้ และให้คุณมีตัวเลือกในการเปิดใช้งานอีกครั้งในภายหลัง
- องค์ประกอบแบบสถิต (ไม่มีโมเดลเป้าหมาย) เหมาะสำหรับรูปแบบการจัดวางที่คุณใช้ซ้ำทั่วทั้งเว็บไซต์ เช่น ตัวแบ่ง ปุ่มกระตุ้นการดำเนินการ หรือช่องว่างที่มีแบรนด์
