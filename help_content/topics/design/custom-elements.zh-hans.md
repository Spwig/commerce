---
title: 自定义元素
---

自定义元素可帮助您创建符合商店需求的可重复使用的页面构建器模块。您可以使用页面构建器现有的工具视觉化地设计一个元素，然后选择将其连接到实时商店数据（如产品名称、价格或图片），这样当元素放置在页面上时，它会自动填充真实内容。创建后，您的自定义元素将与内置模块一起出现在页面构建器的元素库中。

![自定义元素库](/static/core/admin/img/help/custom-elements/custom-elements-list.webp)

## 何时使用自定义元素

当您发现自己反复构建相同布局时，自定义元素最有价值。您不需要在每一页上从头开始重新创建“特色产品卡片”，而是将其构建一次作为自定义元素，并在需要时将其拖放到位。如果该元素与数据绑定，它会自动获取当前产品信息——当价格或名称更改时，无需手动更新。

常见用途：

- 显示名称、价格和主图片的产品亮点卡片
- 带横幅、标题和链接的分类促销模块
- 带标志和描述的品牌展示面板
- 带特色图片、标题和摘要的博客文章预告

## 创建新的自定义元素

1. 导航到 **设计 > 自定义元素**
2. 点击 **+ 添加自定义元素**
3. Spwig 会立即创建一个草稿元素并打开 **视觉构建器** — 您不需要先填写表单
4. 在视觉构建器中，使用可用的页面构建器工具构建元素的布局
5. 当您对设计满意时，在侧边栏中配置元素的设置（名称、数据绑定、图标）
6. 当您准备好将元素发布到库中时，将 **激活** 设置为开启
7. 保存元素

该元素现在将在页面构建器的元素面板中，出现在您分配的类别下。

## 视觉构建器

视觉构建器是专门用于设计元素的画布。

它的工作方式与标准页面构建器类似，但专注于单个元素，而不是整个页面。

你可以：

- 添加和排列子元素（文本块、图片、容器等）
- 为每个子元素设置样式、间距和布局
- 预览元素在示例数据下的外观

视觉构建器中的更改会直接保存到元素定义中。没有单独的发布步骤——在构建器中保存会立即更新已使用该元素的任何页面。

## 配置元素设置

每个自定义元素都有以下设置：

| 字段 | 描述 |
|-------|-------------|
| **名称** | 在元素库中显示的名称 |
| **别名** | URL 安全的标识符，从名称自动生成 |
| **描述** | 关于此元素用途的可选说明 |
| **目标模型** | 要从中绑定数据的商店模型（见下文） |
| **图标** | 在元素库中显示的图标 |
| **分类** | 在库中将相关元素分组 |
| **启用** | 元素是否在页面构建器中可用 |

## 数据绑定

数据绑定将元素布局的各个部分连接到实时商店数据。当页面编辑器将数据绑定元素放置在页面上时，他们会选择一个特定记录（例如，一个产品），所有绑定字段会自动从该记录填充。

### 选择目标模型

**目标模型**设置决定了元素可以显示的商店数据类型。可用模型包括：

| 模型 | 提供内容 |
|-------|-----------------|
| **产品** | 名称、价格、库存状态、图片、描述、SKU、分类、品牌等 |
| **分类** | 名称、描述、图片、横幅、产品数量和 URL |
| **品牌** | 名称、标志、描述、品牌故事和 URL |
| **博客文章** | 标题、摘要、特色图片、作者、发布日期和 URL |

将 **目标模型** 留空以创建一个没有动态数据的静态元素。


静态元素对于固定设计组件（如装饰横幅或布局间隔符）很有用。

### 绑定的工作原理

在可视化构建器中，你可以通过选择子元素应显示的模型字段，将单个子元素标记为数据绑定。例如：

- 一个 **文本** 子元素可以绑定到 **产品名称**，从而显示所选产品的名称
- 一个 **图像** 子元素可以绑定到 **主图**，从而显示产品的主图
- 一个 **文本** 子元素可以绑定到 **价格**，从而始终反映当前价格

每个绑定将一个元素内容字段映射到一个模型字段。你可以为单个自定义元素添加多个绑定——例如，同时将一个文本块绑定到 **产品名称**，并将一个单独的图像块绑定到 **主图**。

### 图像缩略图预设

对于图像绑定，你可以选择性地指定一个 **缩略图预设**（如 `thumbnail` 或 `medium`）。这会控制加载的图像大小，通过为元素布局提供适当大小的图像，有助于加快页面加载速度。

## 停用和重新启用元素

停用元素会将其从元素库中移除，因此无法添加到新页面中。已使用该元素的现有页面不受影响——该元素仍会在这些页面上渲染。

要停用：

1. 导航到 **设计 > 自定义元素**
2. 点击元素名称
3. 取消勾选 **启用**
4. 保存

要重新启用，请按照相同步骤操作，并再次勾选 **启用**。

## 过滤元素库

元素列表支持按以下条件进行过滤：

- **启用 / 停用** — 仅显示已发布或仅显示草稿元素
- **目标模型** — 按元素绑定的模型进行过滤
- **分类** — 按元素分类进行过滤
- **搜索** — 按名称、slug 或描述进行搜索

这在你有许多自定义元素且需要快速找到特定元素时很有帮助。

## 示例：产品高亮卡片

**目标：** 显示产品主图、名称和价格的卡片元素。

| 设置 | 值 |
|---------|-------|
| 名称 | 产品突出卡片 |
| 目标模型 | 产品 |
| 分类 | 产品 |
| 图标 | fas fa-box |

在视觉构建器中添加：
- 一个绑定到 **主图** 并使用 `medium` 缩略图预设的 **图片** 元素
- 一个绑定到 **产品名称** 的 **文本** 元素
- 一个绑定到 **价格** 的 **文本** 元素

保存并激活后，该元素会出现在页面构建器的“产品”分类下。当页面编辑器将其添加到页面时，他们可以选择要突出显示的产品，卡片会自动填充数据。

## 提示

- 为元素命名时，应包含其用途和数据类型，例如使用“产品突出卡片”而不是“卡片 1”，这样随着库的扩展，库仍易于导航
- 使用 **分类** 字段将相关元素分组（产品、博客、促销等）——这有助于页面编辑器更好地组织元素库
- 通过将数据绑定元素添加到草稿页面并选择真实记录（在发布前），来测试数据绑定是否能正确获取所需信息
- 将过时的元素设为停用而不是删除——这可以保留仍引用它们的页面，并且以后可以重新启用
- 没有目标模型的静态元素适用于跨网站重复使用的布局模式，例如分隔符、CTA 面板或品牌间隔符
