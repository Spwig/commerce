---
title: 自定義元件
---

自定義元件讓您可以建立符合您商店需求的可重複使用頁面構建塊。您可以使用頁面構建器現有的工具視覺化設計元件，然後選擇性地將其連接到實時商店資料——例如產品名稱、價格或圖片——這樣當元件放置在頁面上時，會自動填充真實內容。一旦建立，您的自定義元件會與內建塊一起出現在頁面構建器的元件庫中。

![自定義元件庫](/static/core/admin/img/help/custom-elements/custom-elements-list.webp)

## 什麼時候使用自定義元件

當您發現自己反覆建立相同佈局時，自定義元件最有價值。您不需要在每個頁面上從頭開始重新建立「特色產品卡片」，而是只需建立一次作為自定義元件，然後在需要的地方拖放即可。如果元件是資料綁定的，它會自動提取當前產品資訊——當價格或名稱更改時，無需手動更新。

常見用途：

- 顯示名稱、價格和主要圖片的產品亮點卡片
- 帶有橫幅、標題和連結的類別促銷塊
- 帶有標誌和描述的品牌展示面板
- 帶有特色圖片、標題和摘要的部落格文章預覽

## 建立新的自定義元件

1. 進入 **設計 > 自定義元件**
2. 點擊 **+ 添加自定義元件**
3. Spwig 會立即建立一個草稿元件並打開 **視覺構建器** — 您不需要先填寫表單
4. 在視覺構建器中，使用可用的頁面構建工具來建立元件的佈局
5. 當您對設計感到滿意時，在側邊欄配置元件的設定（名稱、資料綁定、圖示）
6. 當您準備好將元件發佈到庫中時，將 **啟用** 設為開啟
7. 儲存元件

元件現在已在頁面構建器的元件面板中，位於您指定的分類下。

## 視覺構建器

視覺構建器是專門用於設計您元件的畫布。

它的工作方式與標準頁面構建器類似，但專注於單個元素，而不是整個頁面。

您可以：

- 添加和排列子元素（文字塊、圖片、容器等）
- 為每個子元素設置樣式、間距和佈局
- 預覽元素在樣本數據下的顯示效果

視覺構建器中的更改會直接保存到元素定義中。沒有單獨的發布步驟——在構建器中保存會立即更新已經使用該元素的任何頁面。

## 設置元素選項

每個自定義元素都有以下設置：

| 欄位 | 說明 |
|-------|-------------|
| **名稱** | 在元素庫中顯示的名稱 |
| **Slug** | URL 安全的識別符，從名稱自動生成 |
| **描述** | 關於此元素的可選說明 |
| **目標模型** | 要綁定數據的商店模型（請參見下文） |
| **圖示** | 在元素庫中顯示的圖示 |
| **分類** | 在庫中將相關元素分組 |
| **啟用** | 元素是否可在頁面構建器中使用 |

## 數據綁定

數據綁定將您元素佈局的部分與實時商店數據連接起來。當頁面編輯器將數據綁定的元素放置在頁面上時，他們會選擇一個特定的記錄（例如，一個產品），所有綁定的字段會自動從該記錄填充。

### 選擇目標模型

**目標模型**設置決定了元素可以顯示的商店數據類型。可用的模型有：

| 模型 | 提供內容 |
|-------|-----------------|
| **產品** | 名稱、價格、庫存狀態、圖片、描述、SKU、分類、品牌等 |
| **分類** | 名稱、描述、圖片、橫幅、產品數量和 URL |
| **品牌** | 名稱、標誌、描述、品牌故事和 URL |
| **部落格文章** | 標題、摘要、特色圖片、作者、發布日期和 URL |

將 **目標模型** 留空以創建一個沒有動態數據的靜態元素。

靜態元素對於固定設計組件（如裝飾橫幅或佈局間隔）很有用。

### 綁定是如何運作的

在視覺構建器中，您可以通過選擇它們應顯示的模型欄位，將單個子元素標記為數據綁定。例如：

- 一個 **文字** 子元素可以綁定到 **產品名稱**，這樣就會顯示所選產品的名稱
- 一個 **圖片** 子元素可以綁定到 **主圖片**，這樣就會顯示產品的主要照片
- 一個 **文字** 子元素可以綁定到 **價格**，這樣就會始終反映當前價格

每個綁定會將一個元素內容欄位映射到一個模型欄位。您可以向單個自定義元素添加多個綁定——例如，同時將一段文字綁定到 **產品名稱**，並將一個獨立的圖片塊綁定到 **主圖片**。

### 圖片縮略圖預設

對於圖片綁定，您可以選擇性地指定一個 **縮略圖預設**（例如 `thumbnail` 或 `medium`）。這會控制加載的圖片大小，通過為元素的佈局提供適當大小的圖片，有助於加快頁面加載速度。

## 停用和啟用元素

停用一個元素會將其從元素庫中移除，因此無法添加到新頁面。已經使用該元素的現有頁面不受影響——該元素會繼續在這些頁面上渲染。

停用步驟：

1. 進入 **設計 > 自定義元素**
2. 點擊元素名稱
3. 取消勾選 **啟用**
4. 儲存

要重新啟用，請按照相同的步驟操作，並再次勾選 **啟用**。

## 過濾元素庫

元素列表支援以下過濾方式：

- **啟用 / 停用** — 顯示僅已發布或僅草稿元素
- **目標模型** — 按元素綁定的模型過濾
- **分類** — 按元素分類過濾
- **搜尋** — 按名稱、slug 或描述搜尋

這在您有很多自定義元素且需要快速找到特定元素時很有幫助。

## 示例：產品重點卡片

**目標：** 顯示產品主要圖片、名稱和價格的卡片元件。

| 設定 | 值 |
|---------|-------|
| 名稱 | Product Highlight Card |
| 目標模型 | Product |
| 分類 | Products |
| 圖示 | fas fa-box |

在視覺建構器中新增：
- 一個與 **Main Image** 綁定的 **圖片** 元件，並使用 `medium` 的縮略圖預設設定
- 一個與 **Product Name** 綁定的 **文字** 元件
- 一個與 **Price** 綁定的 **文字** 元件

保存並啟用後，該元件會出現在頁面建構器的 Products 分類下。當頁面編輯者將其新增到頁面時，他們會選擇要展示的產品，卡片會自動填入資料。

## 小技巧

- 為元件命名時，請包含其用途和資料類型，例如 "Product Highlight Card" 而非 "Card 1"，這樣可以讓元件庫在擴展時仍易於導覽
- 使用 **Category** 欄位來將相關元件分組（Products、Blog、Promotions）——這有助於讓元件庫對頁面編輯者來說更整齊有序
- 通過將資料綁定元件新增到草稿頁面並選擇真實記錄，來測試資料綁定元件，確認綁定是否正確提取了所需資訊
- 將過時的元件停用而不是刪除——這可以保留仍引用這些元件的頁面，並讓您有機會後續重新啟用
- 無目標模型的靜態元件適合用於跨網站重複使用的佈局模式，例如分隔線、CTA 面板或品牌間距
