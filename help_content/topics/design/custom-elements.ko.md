---
title: 사용자 정의 요소
---

사용자 정의 요소는 가게의 요구사항에 맞게 재사용 가능한 페이지 빌더 블록을 만들 수 있게 해줍니다. 페이지 빌더의 기존 도구를 사용하여 요소를 시각적으로 설계한 후, 선택적으로 제품 이름, 가격, 이미지와 같은 실제 가게 데이터에 연결할 수 있습니다. 이 요소는 페이지에 배치되면 실제 콘텐츠로 자동으로 채워집니다. 생성된 사용자 정의 요소는 내장된 블록과 함께 페이지 빌더의 요소 라이브러리에 나타납니다.

![사용자 정의 요소 라이브러리](/static/core/admin/img/help/custom-elements/custom-elements-list.webp)

## 사용자 정의 요소를 사용할 때

사용자 정의 요소는 동일한 레이아웃을 반복적으로 만들 때 가장 유용합니다. 매 페이지마다 "추천 제품 카드"를 처음부터 다시 만드는 대신, 사용자 정의 요소로 한 번만 만들고 필요한 곳에 끌어다 사용하면 됩니다. 요소가 데이터와 연결되어 있다면 제품 정보가 자동으로 업데이트되므로 가격이나 이름이 변경될 때 수동으로 업데이트할 필요가 없습니다.

일반적인 사용 사례:

- 제품 이름, 가격, 메인 이미지를 표시하는 제품 강조 카드
- 배너, 제목, 링크를 포함한 카테고리 프로모션 블록
- 로고와 설명을 포함한 브랜드 전시 패널
- 특징 이미지, 제목, 요약을 포함한 블로그 게시물 미리보기

## 새 사용자 정의 요소 생성

1. **디자인 > 사용자 정의 요소**로 이동
2. **+ 사용자 정의 요소 추가**를 클릭
3. Spwig는 즉시 초안 요소를 생성하고 **시각적 빌더**를 엽니다. 먼저 양식을 채우지 않아도 됩니다.
4. 시각적 빌더에서 사용 가능한 페이지 빌더 도구를 사용하여 요소의 레이아웃을 만듭니다.
5. 디자인에 만족하면 사이드바에서 요소의 설정(이름, 데이터 연결, 아이콘)을 구성합니다.
6. 요소를 라이브러리에 게시할 준비가 되면 **활성화**를 켜줍니다.
7. 요소를 저장합니다.

이제 요소는 페이지 빌더의 요소 패널에서 당신이 할당한 카테고리 아래에 사용할 수 있습니다.

## 시각적 빌더

시각적 빌더는 요소를 설계하기 위한 전용 캔버스입니다.

모든 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어를 유지합니다.

표준 페이지 빌더와 동일하게 작동하지만, 전체 페이지가 아닌 단일 요소에 초점을 맞추고 있습니다.

사용할 수 있습니다:

- 자식 요소(텍스트 블록, 이미지, 컨테이너 등)를 추가하고 배열
- 각 자식 요소의 스타일, 간격 및 레이아웃 설정
- 샘플 데이터와 함께 요소가 어떻게 표시되는지 미리보기

시각적 빌더에서 변경 사항은 요소 정의에 직접 저장됩니다. 별도의 게시 단계는 없습니다 — 빌더에서 저장하면 해당 요소를 이미 사용하는 모든 페이지에 즉시 업데이트됩니다.

## 요소 설정 구성

각 사용자 정의 요소에는 다음과 같은 설정이 있습니다:

| 필드 | 설명 |
|-------|-------------|
| **이름** | 요소 라이브러리에서 표시되는 이름 |
| **Slug** | 이름에서 자동 생성된 URL 안전 식별자 |
| **설명** | 이 요소가 무엇을 위한 것인지에 대한 선택적 메모 |
| **대상 모델** | 데이터를 바인딩할 스토어 모델 (아래 참조) |
| **아이콘** | 요소 라이브러리에 표시되는 아이콘 |
| **카테고리** | 라이브러리에서 관련된 요소를 그룹화 |
| **활성** | 이 요소가 페이지 빌더에서 사용 가능한지 여부 |

## 데이터 바인딩

데이터 바인딩은 요소 레이아웃의 일부를 실시간 스토어 데이터에 연결합니다. 페이지 편집자가 데이터가 바인딩된 요소를 페이지에 배치할 때, 특정 레코드(예: 제품)를 선택하고, 모든 바인딩된 필드는 해당 레코드에서 자동으로 채워집니다.

### 대상 모델 선택

**대상 모델** 설정은 요소가 표시할 수 있는 스토어 데이터 유형을 결정합니다. 사용 가능한 모델은 다음과 같습니다:

| 모델 | 제공 내용 |
|-------|-----------------|
| **제품** | 이름, 가격, 재고 상태, 이미지, 설명, SKU, 카테고리, 브랜드 및 기타 |
| **카테고리** | 이름, 설명, 이미지, 배너, 제품 수, URL |
| **브랜드** | 이름, 로고, 설명, 브랜드 스토리, URL |
| **블로그 게시물** | 제목, 요약, 특징 이미지, 작성자, 게시 날짜, URL |

**대상 모델**을 비워두면 동적 데이터가 없는 정적 요소를 생성할 수 있습니다.

정적 요소는 장식용 배너나 레이아웃 간격과 같은 고정된 디자인 구성 요소에 유용합니다.

### 바인딩의 작동 방식

시각적 빌더(Visual Builder)에서 개별 자식 요소를 데이터 바인딩으로 표시할 수 있도록 선택하여 모델 필드를 지정할 수 있습니다. 예를 들어:

- **텍스트** 자식 요소는 **제품 이름**에 바인딩되어 선택된 제품의 이름을 표시할 수 있습니다.
- **이미지** 자식 요소는 **메인 이미지**에 바인딩되어 제품의 주요 사진을 표시할 수 있습니다.
- **텍스트** 자식 요소는 **가격**에 바인딩되어 항상 현재 가격을 반영할 수 있습니다.

각 바인딩은 하나의 요소 콘텐츠 필드를 하나의 모델 필드에 매핑합니다. 하나의 커스텀 요소에 여러 바인딩을 추가할 수 있습니다. 예를 들어, 텍스트 블록을 **제품 이름**에 바인딩하고, 별도의 이미지 블록을 **메인 이미지**에 동시에 바인딩할 수 있습니다.

### 이미지 썸네일 사전 설정

이미지 바인딩에 대해 선택적으로 **썸네일 사전 설정**(예: `thumbnail` 또는 `medium`)을 지정할 수 있습니다. 이는 요소 레이아웃에 적합한 크기의 이미지를 제공하여 페이지 로딩 속도를 높이는 데 도움이 됩니다.

## 요소 비활성화 및 재활성화

요소를 비활성화하면 요소 라이브러리에서 제거되어 새 페이지에 추가할 수 없습니다. 기존 페이지에서 이미 해당 요소를 사용하고 있는 경우 영향을 받지 않으며, 해당 페이지에서는 요소가 계속 렌더링됩니다.

비활성화 방법:

1. **디자인 > 커스텀 요소**로 이동합니다.
2. 요소 이름을 클릭합니다.
3. **활성**을 선택 취소합니다.
4. 저장합니다.

재활성화하려면 동일한 단계를 따르고 **활성**을 다시 선택합니다.

## 요소 라이브러리 필터링

요소 목록은 다음으로 필터링할 수 있습니다:

- **활성 / 비활성** — 출판된 요소만 또는 초안 요소만 표시
- **대상 모델** — 요소가 바인딩된 모델로 필터링
- **카테고리** — 요소 카테고리로 필터링
- **검색** — 이름, 슬러그 또는 설명으로 검색

이 기능은 많은 커스텀 요소가 있는 경우 특정 요소를 빠르게 찾는 데 도움이 됩니다.

## 예시: 제품 강조 카드

**목표:** 제품의 주 이미지, 이름 및 가격을 표시하는 카드 요소입니다.

| 설정 | 값 |
|---------|-------|
| 이름 | 제품 강조 카드 |
| 대상 모델 | 제품 |
| 범주 | 제품 |
| 아이콘 | fas fa-box |

시각적 빌더에서 추가:
- **이미지** 요소를 추가하고 **메인 이미지**에 바인딩하여 썸네일 사전 설정 `medium` 사용
- **텍스트** 요소를 추가하고 **제품 이름**에 바인딩
- **텍스트** 요소를 추가하고 **가격**에 바인딩

저장 및 활성화 후, 이 요소는 페이지 빌더의 제품 범주 아래에 나타납니다. 페이지 편집자가 이를 페이지에 추가할 때, 어떤 제품을 강조할지 선택하면 카드가 자동으로 채워집니다.

## 팁

- 요소 이름은 목적과 데이터 유형을 포함하는 설명적인 이름을 사용하세요. 예를 들어, "제품 강조 카드"가 "카드 1"보다 더 좋습니다. 이는 라이브러리가 커질수록 탐색이 쉬워지기 때문입니다.
- **범주** 필드를 사용하여 관련된 요소(제품, 블로그, 프로모션 등)를 그룹화하세요. 이는 페이지 편집자가 요소 라이브러리를 쉽게 정리할 수 있도록 합니다.
- 데이터 바인딩 요소를 테스트하려면 초안 페이지에 추가하고 출판 전에 실제 레코드를 선택하여 바인딩이 올바른 정보를 가져오는지 확인하세요.
- 사용하지 않는 요소는 삭제하기보다 비활성화하세요. 이는 해당 요소를 참조하는 페이지를 보존하고 나중에 다시 활성화할 수 있는 선택지를 제공합니다.
- 대상 모델이 없는 정적 요소는 사이트 전반에서 반복적으로 사용하는 레이아웃 패턴(예: 구분선, CTA 패널, 브랜드 간격 등)에 이상적입니다.
