---
title: Пользовательские элементы
---

Пользовательские элементы позволяют создавать повторно используемые блоки для построителя страниц, адаптированные под нужды вашего магазина. Вы проектируете элемент визуально с помощью существующих инструментов построителя страниц, а затем, по желанию, подключаете его к живым данным магазина — таким как названия продуктов, цены или изображения — чтобы элемент автоматически заполнялся реальным содержимым, когда он размещается на странице. После создания ваши пользовательские элементы появляются в библиотеке элементов построителя страниц вместе с встроенными блоками.

![Библиотека пользовательских элементов](/static/core/admin/img/help/custom-elements/custom-elements-list.webp)

## Когда использовать пользовательские элементы

Пользовательские элементы наиболее полезны, когда вы часто создаете одинаковую макетку. Вместо того чтобы создавать «карточку популярного продукта» с нуля на каждой странице, вы создаете ее один раз в виде пользовательского элемента и используете ее там, где это необходимо. Если элемент привязан к данным, он автоматически получает текущую информацию о продукте — не требуется вручную обновлять при изменении цен или названий.

Часто используемые случаи:

- Карточки выделения продуктов, показывающие название, цену и главное изображение
- Блоки продвижения категорий с баннером, заголовком и ссылкой
- Панели демонстрации брендов с логотипом и описанием
- Краткие анонсы статей блога с основным изображением, заголовком и выдержкой

## Создание нового пользовательского элемента

1. Перейдите в **Дизайн > Пользовательские элементы**
2. Нажмите **+ Добавить пользовательский элемент**
3. Spwig сразу создает черновик элемента и открывает **Визуальный редактор** — вам не нужно заполнять форму заранее
4. В Визуальном редакторе создайте макет вашего элемента с помощью доступных инструментов построителя страниц
5. Когда вы будете довольны дизайном, настройте параметры элемента (название, привязку к данным, иконку) в боковой панели
6. Включите **Активный**, когда будете готовы опубликовать элемент в библиотеку
7. Сохраните элемент

Теперь элемент доступен в панели элементов построителя страниц в категории, которую вы назначили.

## Визуальный редактор

Визуальный редактор — это отдельная область для проектирования вашего элемента. Он работает как стандартный построитель страниц, но фокусируется на одном элементе, а не на всей странице. Вы можете:

- Добавлять и размещать дочерние элементы (текстовые блоки, изображения, контейнеры и т.д.)
- Настраивать стиль, отступы и макет для каждого дочернего элемента
- Предварительно просматривать, как будет выглядеть элемент с образцовыми данными

Изменения в Визуальном редакторе сохраняются напрямую в определении элемента. Нет отдельного шага публикации — сохранение в редакторе сразу обновляет элемент для всех страниц, которые уже используют его.

## Настройка параметров элемента

Каждый пользовательский элемент имеет следующие параметры:

| Параметр | Описание |
|---------|----------|
| **Название** | Отображаемое имя, показываемое в библиотеке элементов |
| **Slug** | URL-совместимый идентификатор, автоматически генерируемый из названия |
| **Описание** | Необязательная заметка о том, для чего предназначен этот элемент |
| **Целевая модель** | Модель магазина, от которой будут получаться данные (см. ниже) |
| **Иконка** | Иконка, отображаемая в библиотеке элементов |
| **Категория** | Группирует связанные элементы в библиотеке |
| **Активный** | Определяет, доступен ли элемент в построителе страниц |

## Привязка данных

Привязка данных соединяет части макета вашего элемента с живыми данными магазина. Когда редактор страницы размещает элемент с привязкой данных на странице, он выбирает конкретную запись (например, продукт), и все привязанные поля автоматически заполняются из этой записи.

### Выбор целевой модели

Настройка **Целевой модели** определяет, какой тип данных магазина может отображать элемент. Доступные модели:

| Модель | Что она предоставляет |
|-------|------------------|
| **Продукт** | Название, цена, статус наличия, изображения, описание, SKU, категория, бренд и многое другое |
| **Категория** | Название, описание, изображение, баннер, количество продуктов и URL |
| **Бренд** | Название, логотип, описание, история бренда и URL |
| **Статья блога** | Заголовок, выдержка, основное изображение, автор, дата публикации и URL |

Оставьте **Целевую модель** пустой, чтобы создать статический элемент без динамических данных. Статические элементы полезны для фиксированных компонентов дизайна, таких как декоративные баннеры или элементы макета.

### Как работает привязка


Внутри Visual Builder вы можете пометить отдельные дочерние элементы как привязанные к данным, выбрав поле модели, которое они должны отображать.

Например:
- Дочерний элемент **текст** можно привязать к **Product Name**, чтобы он отображал имя выбранного продукта
- Дочерний элемент **изображение** можно привязать к **Main Image**, чтобы он отображал основное изображение продукта
- Дочерний элемент **текст** можно привязать к **Price**, чтобы он всегда отражал текущую цену

Каждая привязка сопоставляет одно поле содержимого элемента с одним полем модели. Вы можете добавить несколько привязок к одному пользовательскому элементу — например, привязать блок текста к **Product Name** и отдельный блок изображения к **Main Image** одновременно.

### Предустановки миниатюр изображений

Для привязок изображений вы можете дополнительно указать **Предустановку миниатюры** (например, `thumbnail` или `medium`). Это управляет размером загружаемого изображения, что помогает страницам быстрее загружаться, предоставляя изображение подходящего размера для макета элемента.

## Отключение и повторное включение элементов

Отключение элемента удаляет его из библиотеки элементов, поэтому его нельзя добавить на новые страницы. Существующие страницы, которые уже используют этот элемент, не затрагиваются — элемент продолжает отображаться на этих страницах.

Чтобы отключить:
1. Перейдите в **Design > Custom Elements**
2. Нажмите на имя элемента
3. Снимите флажок **Active**
4. Сохраните

Чтобы повторно включить, выполните те же шаги и снова установите флажок **Active**.

## Фильтрация библиотеки элементов

Список элементов поддерживает фильтрацию по:
- **Active / Inactive** — показывать только опубликованные или только черновики элементов
- **Target Model** — фильтровать по модели, к которой привязан элемент
- **Category** — фильтровать по категории элемента
- **Search** — искать по названию, slug или описанию

Это помогает, когда у вас много пользовательских элементов, и вам нужно быстро найти конкретный.

## Пример: карточка выделения продукта

**Цель:** элемент-карточка, который отображает основное изображение продукта, его имя и цену.

| Настройка | Значение |
|---------|-------|
| Name | Product Highlight Card |
| Target Model | Product |
| Category | Products |
| Icon | fas fa-box |

В Visual Builder добавьте:
- Элемент **Image**, привязанный к **Main Image** с предустановкой миниатюры `medium`
- Элемент **Text**, привязанный к **Product Name**
- Элемент **Text**, привязанный к **Price**

После сохранения и активации элемент появляется в конструкторе страниц в категории Products. Когда редактор страницы добавляет его на страницу, он выбирает, какой продукт выделить, и карточка автоматически заполняется.

## Советы

- Дайте элементам описательные названия, включающие их назначение и тип данных — например, «Product Highlight Card» вместо «Card 1» — это поможет сохранить библиотеку легко навигируемой по мере её роста
- Используйте поле **Category**, чтобы сгруппировать связанные элементы (Products, Blog, Promotions) — это поможет организовать библиотеку элементов для ваших редакторов страниц
- Тестируйте элементы с привязкой данных, добавляя их на черновую страницу и выбирая ре cord перед публикацией, чтобы убедиться, что привязка получает правильную информацию
- Отключайте устаревшие элементы вместо их удаления — это сохранит любые страницы, которые всё ещё ссылаются на них, и даст вам возможность повторно включить их позже
- Статические элементы (без целевой модели) идеальны для макетов, которые вы повторно используете по всей веб-сайте, например, разделители, панели CTA или брендовые промежутки
