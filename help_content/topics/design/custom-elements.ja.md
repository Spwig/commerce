---
title: カスタム要素
---

カスタム要素を使用すると、ストアのニーズに合わせた再利用可能なページビルダーのブロックを作成できます。ページビルダーの既存のツールを使って要素を視覚的にデザインし、オプションでそれをライブストアデータ（例：商品名、価格、または画像）に接続できます。これにより、ページに配置されたときに要素が自動的に実際のコンテンツで埋め込まれます。一度作成すると、カスタム要素は既存のブロックとともにページビルダーの要素ライブラリに表示されます。

![カスタム要素ライブラリ](/static/core/admin/img/help/custom-elements/custom-elements-list.webp)

## カスタム要素の使用タイミング

カスタム要素は、同じレイアウトを繰り返し作成しているときに最も価値があります。毎回ページごとに「特集商品カード」をから頭で作り直す代わりに、一度カスタム要素として作成し、必要な場所に配置するだけで済みます。要素がデータバインドされている場合、価格や名前が変更されても手動で更新する必要がありません。価格や名前が変更されても手動で更新する必要がありません。

一般的な用途:

- 名前、価格、およびメイン画像を表示する商品ハイライトカード
- バナー、タイトル、およびリンクを含むカテゴリプロモーションブロック
- ロゴと説明を含むブランド展示パネル
- 特集画像、タイトル、および抜粋を含むブログ投稿のプレビュー

## 新しいカスタム要素の作成

1. **デザイン > カスタム要素**に移動します
2. **+ カスタム要素の追加**をクリックします
3. Spwigはすぐにドラフト要素を作成し、**ビジュアルビルダー**を開きます — フォームを事前に記入する必要はありません
4. ビジュアルビルダーで、利用可能なページビルダーのツールを使って要素のレイアウトを構築します
5. デザインに満足したら、サイドバーで要素の設定（名前、データバインディング、アイコン）を構成します
6. 要素をライブラリに公開する準備が整ったら、**アクティブ**をオンにします
7. 要素を保存します

要素は、割り当てたカテゴリの下でページビルダーの要素パネルに利用可能になります。

## ビジュアルビルダー

ビジュアルビルダーは、要素をデザインするための専用のキャンバスです。

標準のページビルダーと同様に動作しますが、ページ全体ではなく単一の要素に焦点を当てています。

以下ができます：

- 子要素（テキストブロック、画像、コンテナなど）を追加し、配置を調整
- 各子要素のスタイル、間隔、レイアウトを設定
- サンプルデータを使用して要素の表示方法をプレビュー

ビジュアルビルダーで行った変更は、要素の定義に直接保存されます。別途の公開ステップは必要ありません — ビルダー内で保存すると、すでにその要素を使用しているページにも即座に反映されます。

## 要素設定の構成

各カスタム要素には、以下の設定があります：

| フィールド | 説明 |
|-------|-------------|
| **名前** | 要素ライブラリに表示される表示名 |
| **スラッグ** | 名前から自動生成されるURLに安全な識別子 |
| **説明** | この要素の用途に関するオプションのノート |
| **対象モデル** | データをバインドするためのストアモデル（下記を参照） |
| **アイコン** | 要素ライブラリに表示されるアイコン |
| **カテゴリ** | ライブラリ内で関連する要素をグループ化 |
| **アクティブ** | 要素がページビルダーで利用可能かどうか |

## データバインディング

データバインディングは、要素のレイアウトの一部をライブストアデータに接続します。ページエディタがページ上にデータバインドされた要素を配置する場合、特定のレコード（例えば、商品）を選択し、すべてのバインドされたフィールドがそのレコードから自動的に埋め込まれます。

### 対象モデルの選択

**対象モデル**の設定は、要素が表示できるストアデータのタイプを決定します。利用可能なモデルは以下の通りです：

| モデル | 提供内容 |
|-------|-----------------|
| **商品** | 名前、価格、在庫状態、画像、説明、SKU、カテゴリ、ブランド、その他 |
| **カテゴリ** | 名前、説明、画像、バナー、商品数、URL |
| **ブランド** | 名前、ロゴ、説明、ブランドストーリー、URL |
| **ブログ投稿** | タイトル、抜粋、特記事項、著者、掲載日、URL |

**対象モデル**を空のままにすると、動的なデータがない静的な要素を作成できます。

静的要素は、装飾用のバナーまたはレイアウトのスペースなどの固定されたデザインコンポーネントに役立ちます。

### バインディングの仕組み

ビジュアルビルダー内では、個々の子要素をデータバインディング対象としてマークし、表示するモデルフィールドを選択できます。例えば：

- **テキスト**の子要素は **製品名** にバインディングできます。これにより、選択された製品の名前が表示されます。
- **画像**の子要素は **メイン画像** にバインディングできます。これにより、製品の主な写真が表示されます。
- **テキスト**の子要素は **価格** にバインディングできます。これにより、常に現在の価格が反映されます。

各バインディングは、1つの要素コンテンツフィールドを1つのモデルフィールドにマッピングします。1つのカスタム要素に複数のバインディングを追加できます。たとえば、テキストブロックを **製品名** に、別途の画像ブロックを **メイン画像** に同時にバインディングできます。

### 画像サムネイルのプリセット

画像バインディングでは、オプションで **サムネイルプリセット**（例: `thumbnail` または `medium`）を指定できます。これにより、読み込む画像のサイズを制御でき、要素のレイアウトに適したサイズの画像を提供することで、ページの読み込み速度が向上します。

## 要素の非アクティブ化と再アクティブ化

要素を非アクティブ化すると、要素ライブラリから削除され、新しいページに追加できなくなります。既存のページでその要素がすでに使用されている場合、それらのページには影響を与えません。要素は引き続きそのページにレンダリングされます。

非アクティブ化するには：

1. **Design > Custom Elements** に移動します。
2. 要素名をクリックします。
3. **Active** を外します。
4. 保存します。

再アクティブ化するには、同じ手順に従い、**Active** を再度チェックします。

## 要素ライブラリのフィルタリング

要素リストは以下でフィルタリングできます：

- **Active / Inactive** — 公開済みの要素またはドラフトの要素のみを表示
- **Target Model** — 要素がバインディングされているモデルでフィルタリング
- **Category** — 要素カテゴリでフィルタリング
- **Search** — 名前、スラッグ、または説明で検索

これは、多くのカスタム要素がある場合に特定の要素を素早く見つけるのに役立ちます。

## 例：製品ハイライトカード

**目的:** 商品のメイン画像、名前、価格を表示するカード要素。

| 設定 | 値 |
|---------|-------|
| 名前 | Product Highlight Card |
| 対象モデル | Product |
| カテゴリ | Products |
| アイコン | fas fa-box |

ビジュアルビルダーで以下を追加してください:
- **画像**要素を **メイン画像** にバインドし、サムネイルプリセットを `medium` に設定
- **テキスト**要素を **商品名** にバインド
- **テキスト**要素を **価格** にバインド

保存して有効化すると、この要素はページビルダーの **Products** カテゴリに表示されます。ページエディタがこれをページに追加する際、どの商品を特集するかを選択し、カードが自動的に情報を表示します。

## Tips

- 要素の名前は、その目的とデータタイプを含めるようにしてください。たとえば、「Product Highlight Card」のように「Card 1」といった名前よりもわかりやすい名前を使用すると、ライブラリが拡大しても簡単にナビゲートできます。
- **カテゴリ**フィールドを使って関連する要素（Products, Blog, Promotionsなど）をグループ化してください。これにより、ページエディタにとって要素ライブラリが整理されやすくなります。
- データバインドされた要素をテストするには、ドラフトページに追加し、公開する前に実際のレコードを選択して確認してください。これにより、バインディングが正しい情報を取得しているかを確認できます。
- 過去の要素は削除するのではなく、無効化してください。これにより、まだその要素を参照しているページが保持され、後で再アクティブ化するオプションも得られます。
- 対象モデルがない静的要素は、サイト全体で再利用するレイアウトパターン（セパレータ、CTAパネル、ブランド用のスペースなど）に最適です。
