---
title: Kategorien-Seiten-Vorlagen
---

Kategorien-Seiten-Vorlagen steuern, wie Ihre Produkte angezeigt werden, wenn Kunden eine Kategorie durchsuchen — ob es sich dabei um ein klassisches Raster, eine horizontale Liste, einen scrollbaren Karussell oder eine dramatische Hauptvorschau handelt. Sie können einen einzelnen Standard festlegen, der für den gesamten Store gilt, und diesen dann auf jede einzelne Kategorie überschreiben, die eine andere Behandlung benötigt.

## Wo sich die Einstellungen befinden

Es gibt zwei Orte, um Kategorien-Vorlagen zu konfigurieren:

- **Site-wide default** — Navigieren Sie zu **Design > Page Templates** und scrollen Sie zu dem Abschnitt **Category**. Die hier ausgewählte Vorlage gilt für jede Kategorie-Seite, es sei denn, sie wird überschrieben.
- **Per-category override** — Navigieren Sie zu **Catalog > Categories**, öffnen Sie jede Kategorie und klicken Sie auf den Reiter **Display**. Die hier festgelegte Vorlage überschreibt den Site-wide Standard für diese Kategorie. Alle anderen Kategorien verwenden weiterhin den Standard.

## Die sechs Vorlagen

### Grid

Die Grid-Vorlage ordnet Produkte in einem responsiven Kartenraster an — das bekannteste Layout für den Online-Einkauf. Produkte werden als einheitliche Karten angezeigt, die Bild, Name, Preis und den Hinzufüge-Button für den Warenkorb enthalten.

**Best for:** Allgemeine Stores, Mode, Haushaltswaren, Elektronik — jede Store, bei der eine konsistente Produktpräsentation wichtig ist.

| Option | Beschreibung |
|--------|-------------|
| **Products per row** | Wie viele Karten pro Zeile auf dem Desktop (2, 3, 4 oder 5) |
| **Show category banner** | Zeigt das Kategorie-Bannerbild oben auf der Seite an |
| **Show subcategories** | Zeigt klickbare Unterkategorie-Chips unter dem Banner an |
| **Show category description** | Zeigt den Beschreibungstext der Kategorie an |
| **Card image fallback** | Zeigt ein Platzhalterbild an, wenn ein Produkt keine Fotos hat |

### List

Die List-Vorlage zeigt Produkte als horizontale Zeilen an — Bild auf einer Seite, Produktinformationen (Name, Beschreibung, Preis) auf der anderen. Dies gibt jedem Produkt mehr Platz und macht es einfacher für Kunden, sie im Vergleich zu sehen.

**Best for:** Stores mit komplexen Produkten, die von einer kurzen Beschreibung profitieren, wie z. B. Werkzeuge, Haushaltsgeräte oder B2B-Kataloge.

| Option | Beschreibung |
|--------|-------------|
| **Show product image** | Zeigt das Produktbild in jeder Zeile an |
| **Image position** | Ob das Bild auf der linken oder rechten Seite angezeigt wird |
| **Show short description** | Zeigt einen Produkt-Auszug unter dem Produkt-Namen an |
| **Show category banner** | Zeigt das Kategorie-Bannerbild oben auf der Seite an |
| **Show subcategories** | Zeigt klickbare Unterkategorie-Chips unter dem Banner an |
| **Show category description** | Zeigt den Beschreibungstext der Kategorie an |

### Carousel

Die Carousel-Vorlage gruppiert Produkte in horizontal scrollbaren Zeilen — ähnlich wie die „weiter durchsuchen“-Abschnitte auf Streaming-Plattformen. Kunden swipen oder klicken auf Pfeile, um durch Produkte zu navigieren, ohne die Seite zu verlassen.

**Best for:** Große Kataloge mit vielen Produkten in einer Kategorie, Geschenkeläden, Buchläden oder jede Store, bei der Durchsuchen und Entdecken eine Priorität sind.

| Option | Beschreibung |
|--------|-------------|
| **Slides per view** | Wie viele Produkte auf dem Desktop gleichzeitig sichtbar sind (3, 4 oder 5) |
| **Row size** | Wie viele Produkte pro Karussell-Zeile, bevor eine neue Zeile unterhalb begonnen wird (4, 6, 8 oder 12) |
| **Show navigation arrows** | Zeigt Pfeiltasten für Vorwärts/Rückwärts auf jeder Zeile an |
| **Show dots** | Zeigt einen Fortschrittsbalken und eine Seitenzahl (z. B. „1 / 3“) unter jeder Zeile an |
| **Autoplay** | Automatisch die Folien weiterlaufen |
| **Autoplay speed** | Zeit in Millisekunden zwischen jedem automatischen Weiterlauf |
| **Infinite loop** | Zurück zur ersten Folie springen, nachdem die letzte erreicht wurde |
| **Show category banner** | Zeigt das Kategorie-Bannerbild oben auf der Seite an |
| **Show subcategories** | Zeigt klickbare Unterkategorie-Chips unter dem Banner an |

### Masonry

Die Masonry-Vorlage erstellt ein Pinterest-artiges, gestapeltes Raster, bei dem jede Karte sich an die natürliche Höhe ihres Bildes anpasst. Das Ergebnis ist ein vielfältiges, redaktionelles Gefühl anstelle eines einheitlichen Rasters.

**Best for:** Fotografie, Kunstdrucke, handgefertigte Produkte, Innendesign oder jede Kategorie, bei der das visuelle Erscheinungsbild der Produkte variiert und Sie sie dynamischer präsentieren möchten.

| Option | Beschreibung |
|--------|-------------|
| **Columns** | Anzahl der Spalten über die gesamte Seite (2, 3 oder 4) |
| **Gap size** | Abstand zwischen Karten — klein, mittel oder groß |
| **Show category banner** | Zeigt das Kategorie-Bannerbild oben auf der Seite an |
| **Show subcategories** | Zeigt klickbare Unterkategorie-Chips unter dem Banner an |
| **Show category description** | Zeigt den Beschreibungstext der Kategorie an |

### Featured

Die Featured-Vorlage bewirbt eine kleine Anzahl von Produkten als große Hauptkarten oben auf der Seite, während die restlichen Produkte in einem unterstützenden Raster darunter angezeigt werden. Sie lenkt sofort die Aufmerksamkeit auf Ihre prioritären Produkte.

**Best for:** Neue Artikelabschnitte, Bestsellerseiten, kurierte Sammlungen oder saisonale Kategorien, bei denen Sie bestimmte Produkte hervorheben möchten.

| Option | Beschreibung |
|--------|-------------|
| **Featured products count** | Wie viele Produkte eine Hauptbehandlung oben erhalten (1, 2 oder 3) |
| **Hero layout** | Zeigt die Hauptkarten in voller Breite oder als geteiltes Layout neben einander an |
| **Show category banner** | Zeigt das Kategorie-Bannerbild oben auf der Seite an |
| **Show subcategories** | Zeigt klickbare Unterkategorie-Chips unter dem Banner an |
| **Show category description** | Zeigt den Beschreibungstext der Kategorie an |

### Accordion

Die Accordion-Vorlage ist speziell für die **root categories page** — die Seite, die alle Ihre Hauptkategorien anzeigt. Sie zeigt jede Kategorie als interaktiven Panel an; das Überfahren oder Tippen eines Panels vergrößert es horizontal, während die anderen sich verkleinern. Auf individuellen Kategorie-Seiten fällt die Accordion-Vorlage auf ein Standardraster zurück.

**Best for:** Stores mit starken visuellen Kategoriebildern, bei denen Sie die Kategorien-Seite selbst als immersiven Erlebnis wünschen — Mode-Marken, Lebensstil-Store und Stores mit starken redaktionellen Identitäten.

| Option | Beschreibung |
|--------|-------------|
| **Panels per row** | Maximale Kategorien pro Accordion-Zeile, bevor zur nächsten Zeile unterhalb gewechselt wird (4–8, Standard 6) |
| **Height** | Wie hoch die Accordion-Panel sind — klein, mittel, groß oder extra groß |
| **Expand ratio** | Wie viel breiter das aktive (überfahren) Panel im Vergleich zu inaktiven Panels wird (2–5×) |
| **Transition speed** | Animationsgeschwindigkeit in Millisekunden |
| **Mobile layout** | Wie Kategorien auf Mobilgeräten angezeigt werden — vertikal gestapelt, als Karussell oder als Raster |
| **Show product count** | Zeigt eine Produktanzahl-Badge auf jedem Kategorie-Panel an |

## Common options (all templates)

Jede Vorlage teilt sich eine Reihe von Optionen, die das umgebende Seitenchrome steuern:

| Option | Beschreibung |
|--------|-------------|
| **Show breadcrumb** | Zeigt die Navigation „Home > Categories > Name“ oben auf der Kategorie an |
| **Show sort bar** | Zeigt das Dropdownmenü „Sort by“ an, damit Kunden die Ergebnisse neu ordnen können |
| **Show product count** | Zeigt die Gesamtzahl der Produkte an (z. B. „29 products“) |
| **Show help section** | Zeigt einen Abschnitt an, der fragt „Kann nicht finden, wonach Sie suchen?“ am unteren Rand an |
| **Default sort** | Die Sortierung, die verwendet wird, wenn ein Kunde erstmals auf die Seite gelangt |
| **Products per page** | Wie viele Produkte pro Seite geladen werden |
| **Pagination style** | Wie zusätzliche Produkte geladen werden: paginierte Seiten, ein „Mehr laden“-Button oder unendlicher Scroll |

## Quick comparison

Verwenden Sie diese Tabelle, um die richtige Vorlage auf einen Blick auszuwählen:

| Vorlage | Produkte pro Zeile | Visuelles Erscheinungsbild | Beste Produkttypen |
|----------|-----------------|------------------|--------------------|
| **Grid** | 2–5 konfigurierbar | Einheitlich, sauber | Alles — der sichere Standard |
| **List** | 1 (vollbreitige Zeilen) | Raum, informativ | Komplexe oder technische Produkte |
| **Carousel** | 3–5 gleichzeitig sichtbar | Durchsuchbar, dynamisch | Große Kataloge, Entdeckung |
| **Masonry** | 2–4 (gestapelt) | Redaktionell, vielfältig | Visuelle/Handwerksprodukte |
| **Featured** | Hero + Raster darunter | Werbend | Kurierte Auswahl, neue Artikel |
| **Accordion** | N/A (Kategorien) | Immersiv, dramatisch | Root-Kategorien-Seite |

## Setting a site-wide default template

1. Navigieren Sie zu **Design > Page Templates**.
2. Scrollen Sie zu dem Abschnitt **Category**.
3. Wählen Sie Ihre bevorzugte Vorlage aus dem Dropdown **Template** aus.
4. Konfigurieren Sie die Vorlagenoptionen, die sich unter dem Dropdown befinden.
5. Konfigurieren Sie die allgemeinen Optionen (Sortierleiste, Breadcrumb, Pagination usw.).
6. Klicken Sie auf **Save Configuration**.

Die ausgewählte Vorlage und ihre Einstellungen gelten nun für jede Kategorie-Seite in Ihrem Store.

## Overriding the template for a specific category

Sie können jeder einzelnen Kategorie eine andere Vorlage geben, ohne den Store-weiten Standard zu ändern.

1. Navigieren Sie zu **Catalog > Categories**.
2. Klicken Sie auf die Kategorie, die Sie anpassen möchten.
3. Klicken Sie auf den Reiter **Display** oben im Bearbeitungsformular.
4. Unter **Template** wählen Sie **Custom**, um die Kategorie-spezifischen Einstellungen zu aktivieren.
5. Wählen Sie die Vorlage aus, die Sie für diese Kategorie verwenden möchten.
6. Konfigurieren Sie die Vorlagenoptionen entsprechend.
7. Klicken Sie auf **Save**.

Nur diese Kategorie wird die benutzerdefinierte Vorlage verwenden. Alle anderen Kategorien verwenden weiterhin den Store-weiten Standard. Um eine Kategorie zurück auf den Standard zu setzen, kehren Sie zu ihrem **Display**-Reiter zurück und setzen Sie die Vorlage auf **Use site default**.

## Example: mixing templates across categories

Hier ist ein praktisches Beispiel, wie verschiedene Vorlagen verschiedene Teile des gleichen Stores bedienen können:

| Kategorie | Vorlage | Grund |
|----------|----------|--------|
| All categories (root page) | Accordion | Dramatischer visueller Einstieg |
| Clothing | Grid (4 pro Zeile) | Schnelle, vertraute Durchsuchung |
| Sale | Featured | Hervorhebt die besten Angebote sofort |
| Books | Carousel | Großer Katalog, einfach durchsuchbar |
| Handmade Gifts | Masonry | Vielfältige Bildgrößen, handwerkliches Gefühl |
| Industrial Tools | List | Kunden müssen Beschreibungen sehen |

## Tips

- Beginnen Sie mit der **Grid**-Vorlage als Store-weiten Standard — sie funktioniert gut für fast alle Produkttypen und ist das vertrauteste Layout für Kunden. Personalisieren Sie einzelne Kategorien von dort aus.
- Verwenden Sie die **Featured**-Vorlage sparsam. Sie funktioniert am besten, wenn Sie tatsächlich 1–3 herausragende Produkte für diese Kategorie kuriert haben. Wenn alle Produkte gleich wichtig wirken, verliert die Hauptbehandlung ihre Wirkung.
- Die **Accordion**-Vorlage erzeugt nur ihre volle visuelle Wirkung auf der Root-Kategorien-Seite. Wenn Sie sie einer Produktkategorie zuweisen, sehen Kunden eine einfache Rasteransicht — behalten Sie sie daher für Ihre Hauptkategorie-Liste.
- Setzen Sie **products per page** sorgfältig. 24–36 Produkte pro Seite funktionieren gut für die Grid- und Masonry-Vorlagen. Für den Carousel sind die Zeilenbreite und die Einstellungen für Folien pro Ansicht, die dieses Erlebnis direkt steuern.
- Aktivieren Sie **Show subcategories** auf Elternkategorien, damit Kunden direkt zu einer engeren Auswahl springen können, ohne Sortier- oder Filterwerkzeuge verwenden zu müssen.
- Testen Sie Ihre gewählte Vorlage auf Mobilgeräten. Die Masonry- und Carousel-Vorlagen sehen auf kleineren Bildschirmen sehr unterschiedlich aus — verwenden Sie die **mobile layout**-Option auf Accordion und prüfen Sie, ob die Masonry-Spalten auf Handys nicht zu schmal sind.
- Wenn Sie die List-Vorlage mit **Show short description** aktiviert verwenden, stellen Sie sicher, dass Ihre Produkte tatsächlich kurze Beschreibungen haben. Leere Beschreibungszeilen wirken unbeabsichtigt.
