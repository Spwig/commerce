---
title: カテゴリページテンプレート
---

カテゴリページテンプレートは、顧客がカテゴリを閲覧する際の製品の表示方法を制御します。これは、クラシックなグリッド、水平リスト、スクロール可能なカーニュール、またはドラマチックな特集レイアウトなどです。ストア全体に適用される単一のデフォルトを設定し、必要に応じて個々のカテゴリでそれを上書きできます。

## 設定の場所

カテゴリテンプレートを構成するには2つの場所があります:

- **サイト全体のデフォルト** — **Design > Page Templates** に移動し、**Category** セクションまでスクロールします。ここに選択したテンプレートは、上書きされない限り、すべてのカテゴリページに適用されます。
- **カテゴリごとの上書き** — **Catalog > Categories** に移動し、任意のカテゴリを開き、**Display** タブをクリックします。ここにテンプレートを設定すると、そのカテゴリのサイト全体のデフォルトを上書きします。他のすべてのカテゴリは引き続きデフォルトを使用します。

## 六つのテンプレート

### Grid

Grid テンプレートは、レスポンシブなカードグリッドで製品を配置します。オンラインショッピングで最もなじみのあるレイアウトです。製品は、画像、名前、価格、カートに追加ボタンを備えた均一なカードとして表示されます。

**最適な用途:** 一般的な用途のストア、ファッション、ホームウェア、電子機器 — 一貫した製品のプレゼンテーションが重要なすべてのストア。

| オプション | 説明 |
|--------|-------------|
| **1行あたりの製品数** | デスクトップで1行あたりに表示されるカードの数（2、3、4、または5） |
| **カテゴリバナーの表示** | ページの上部にカテゴリのバナー画像を表示 |
| **サブカテゴリの表示** | バナーの下にクリック可能なサブカテゴリチップを表示 |
| **カテゴリ説明の表示** | カテゴリの説明テキストを表示 |
| **カード画像の代替表示** | 製品に写真がない場合にプレースホルダー画像を表示 |

### List

List テンプレートは、製品を水平な行として表示します — 画像は片側に、製品の詳細（名前、説明、価格）はもう片側に。これにより、各製品に余白が確保され、顧客が一目で比較しやすくなります。

**最適な用途:** 複雑な製品で短い説明のプレビューが役立つストア、例えば、工具、家電、またはB2Bカタログ。

| オプション | 説明 |
|--------|-------------|
| **製品画像の表示** | 各行に製品画像を含める |
| **画像の位置** | 画像が左側に表示されるか右側に表示されるか |
| **短い説明の表示** | 製品名の下に製品の抜粋を表示 |
| **カテゴリバナーの表示** | ページの上部にカテゴリのバナー画像を表示 |
| **サブカテゴリの表示** | バナーの下にクリック可能なサブカテゴリチップを表示 |
| **カテゴリ説明の表示** | カテゴリの説明テキストを表示 |

### Carousel

Carousel テンプレートは、製品を水平スクロール可能な行にグループ化します — ストリーミングプラットフォームの「継続して閲覧」セクションに類似しています。顧客はスワイプまたは矢印をクリックして、ページを離さずに製品を閲覧できます。

**最適な用途:** 単一のカテゴリに多くの製品がある大規模なカタログ、ギフトショップ、書店、または閲覧と発見が優先されるすべてのストア。

| オプション | 説明 |
|--------|-------------|
| **1ビューあたりのスライド数** | デスクトップで一度に表示される製品数（3、4、または5） |
| **行のサイズ** | 1つのカーニュール行に表示される製品数で、下に新しい行を開始する（4、6、8、または12） |
| **ナビゲーション矢印の表示** | 各行に前の/次の矢印ボタンを表示 |
| **ドットの表示** | 各行の下に進行状況バーとページカウンター（例: "1 / 3"）を表示 |
| **自動再生** | スライドを自動的に進める |
| **自動再生速度** | 各自動進捗の間隔（ミリ秒） |
| **無限ループ** | 最後のスライドに達した後、最初のスライドに戻る |
| **カテゴリバナーの表示** | ページの上部にカテゴリのバナー画像を表示 |
| **サブカテゴリの表示** | バナーの下にクリック可能なサブカテゴリチップを表示 |

### Masonry

Masonry テンプレートは、Pinterestスタイルの階段状のグリッドを作成し、各カードは画像の自然な高さに合わせて調整されます。その結果、一様なグリッドではなく、多様で編集的な雰囲気が得られます。

**最適な用途:** 写真、アートプリント、ハンドメイド商品、インテリアデザイン、または製品の視覚的特徴が異なるカテゴリで、よりダイナミックにそれらを紹介したい場合。

| オプション | 説明 |
|--------|-------------|
| **列数** | ページ全体にわたる列数（2、3、または4） |
| **ギャップサイズ** | カード間のスペース — 小、中、または大 |
| **カテゴリバナーの表示** | ページの上部にカテゴリのバナー画像を表示 |
| **サブカテゴリの表示** | バナーの下にクリック可能なサブカテゴリチップを表示 |
| **カテゴリ説明の表示** | カテゴリの説明テキストを表示 |

### Featured

Featured テンプレートは、ページの上部に少数の製品を大規模なヒーローカードとして表示し、残りの製品は下部のサポートグリッドで表示します。これにより、優先製品に即座の注目が集まります。

**最適な用途:** 新着商品セクション、ベストセラーページ、キュレーションされたコレクション、または特定の製品をスポットライトしたい季節カテゴリ。

| オプション | 説明 |
|--------|-------------|
| **特集製品数** | ページの上部でヒーローとして扱われる製品数（1、2、または3） |
| **ヒーローレイアウト** | ヒーローカードをフル幅で表示するか、または並列して表示するか |
| **カテゴリバナーの表示** | ページの上部にカテゴリのバナー画像を表示 |
| **サブカテゴリの表示** | バナーの下にクリック可能なサブカテゴリチップを表示 |
| **カテゴリ説明の表示** | カテゴリの説明テキストを表示 |

### Accordion

Accordion テンプレートは、**ルートカテゴリページ** — すべてのトップレベルカテゴリを表示するページ — に特化して設計されています。これは、各カテゴリをインタラクティブなパネルとして表示します。パネルをホバーまたはタップすると、そのパネルが水平に拡張され、他のパネルは圧縮されます。個々のカテゴリページでは、Accordion テンプレートは標準のグリッドにフォールバックします。

**最適な用途:** カテゴリ画像が視覚的に強調されているストアで、カテゴリページ自体が没入感のある体験になるようにしたい場合 — ファッションブランド、ライフスタイルストア、または編集的なアイデンティティが強いストア。

| オプション | 説明 |
|--------|-------------|
| **1行あたりのパネル数** | 1つのアコーディオン行に表示されるカテゴリの最大数（4–8、デフォルト6） |
| **高さ** | アコーディオンパネルの高さ — 小、中、大、またはエクストラ大 |
| **拡張比率** | アクティブ（ホバー）パネルが非アクティブパネルに比べてどのくらい広がるか（2–5×） |
| **トランジション速度** | アニメーション速度（ミリ秒） |
| **モバイルレイアウト** | モバイルでのカテゴリ表示方法 — 垂直に積み重ね、カーニュール、またはグリッド |
| **製品数の表示** | 各カテゴリパネルに製品数のバッジを表示 |

## 一般的なオプション（すべてのテンプレート）

すべてのテンプレートは、周囲のページの外枠を制御する一連のオプションを共有しています:

| オプション | 説明 |
|--------|-------------|
| **パンくずリストの表示** | カテゴリの上部に "Home > Categories > Name" ナビゲーションを表示 |
| **ソートバーの表示** | "Sort by" ドロップダウンを表示し、顧客が結果を再並べ替えできるようにします |
| **製品数の表示** | 総製品数（例: "29 products"）を表示 |
| **ヘルプセクションの表示** | "Can't find what you're looking for?" セクションをページの下部に表示 |
| **デフォルトソート** | 顧客がページに最初にアクセスしたときに使用されるソート順序 |
| **1ページあたりの製品数** | 1ページあたりに読み込む製品数 |
| **ページネーションスタイル** | その他の製品がどのように読み込まれるか: ページネーションされたページ、"Load more" ボタン、または無限スクロール |

## クイック比較

この表を使って、一目で正しいテンプレートを選択してください:

| テンプレート | 1行あたりの製品数 | 視覚的特徴 | 最適な製品タイプ |
|----------|-----------------|------------------|--------------------|
| **Grid** | 2–5 設定可能 | 一様、清潔 | すべて — セーフなデフォルト |
| **List** | 1（フル幅の行） | 空間があり、情報量が多い | 複雑または技術的な製品 |
| **Carousel** | 3–5 一度に表示 | 閲覧可能、動的 | 大規模なカタログ、発見 |
| **Masonry** | 2–4（階段状） | 編集的、多様 | 視覚的/ハンドメイド製品 |
| **Featured** | ヒーロー + 下部のグリッド | プロモーション | カリキュレーションされた選択、新着商品 |
| **Accordion** | N/A（カテゴリ） | 没入的、ドラマチック | ルートカテゴリページ |

## サイト全体のデフォルトテンプレートの設定

1. **Design > Page Templates** に移動します。
2. **Category** セクションまでスクロールします。
3. **Template** ドロップダウンから、お好みのテンプレートを選択します。
4. ドロップダウンの下に表示されるテンプレートオプションを構成します。
5. 一般的なオプション（ソートバー、パンくずリスト、ページネーションなど）を構成します。
6. **Save Configuration** をクリックします。

選択したテンプレートとその設定は、今後ストアのすべてのカテゴリページに適用されます。

## 特定のカテゴリのテンプレートを上書き

ストア全体のデフォルトを変更することなく、任意のカテゴリに異なるテンプレートを割り当てることができます。

1. **Catalog > Categories** に移動します。
2. カスタマイズしたいカテゴリをクリックします。
3. 編集フォームの上部にある **Display** タブをクリックします。
4. **Template** の下で **Custom** を選択して、カテゴリごとの設定を有効にします。
5. このカテゴリに使用したいテンプレートを選択します。
6. 必要に応じてテンプレートオプションを構成します。
7. **Save** をクリックします。

このカテゴリのみがカスタムテンプレートを使用します。他のすべてのカテゴリは、サイト全体のデフォルトを引き続き使用します。カテゴリをデフォルトに戻すには、Display タブに戻り、テンプレートを **Use site default** に設定してください。

## 例: カテゴリ間でのテンプレートの混在

同じストアの異なる部分で異なるテンプレートがどのように使用できるかの実用的な例です:

| カテゴリ | テンプレート | 理由 |
|----------|----------|--------|
| All categories (root page) | Accordion | 視覚的にドラマチックなエントリーポイント |
| Clothing | Grid (4 per row) | 速く、なじみのある閲覧 |
| Sale | Featured | トップの取引をすぐに強調 |
| Books | Carousel | 大規模なカタログ、簡単な閲覧 |
| Handmade Gifts | Masonry | 多様な画像サイズ、ハンドメイドの雰囲気 |
| Industrial Tools | List | 顧客は説明を確認する必要がある |

## ヒント

- サイト全体のデフォルトとして **Grid** テンプレートから始めましょう — これはほぼすべての製品タイプに適しており、ショッパーにとって最もなじみのあるレイアウトです。そこから個々のカテゴリをカスタマイズしてください。
- **Featured** テンプレートは控えめに使用してください。このテンプレートは、そのカテゴリに1–3の際立った製品を実際にキュレーションできる場合に最適です。すべての製品が同等に重要に見える場合、ヒーローの扱いは効果を失います。
- **Accordion** テンプレートは、ルートカテゴリページでのみその完全な視覚的効果を発揮します。これを製品カテゴリに割り当てると、顧客は単なるグリッドを見ることになります — そのため、トップレベルのカテゴリリストにのみ予約してください。
- **1ページあたりの製品数** を慎重に設定してください。Grid および Masonry テンプレートでは、1ページあたり24–36製品が効果的です。Carousel テンプレートでは、行のサイズとスライド数の設定がこの体験をより直接的に制御します。
- 親カテゴリで **Show subcategories** を有効にして、顧客がソートやフィルタツールを使用せずに、より狭い選択肢に直接ジャンプできるようにしてください。
- モバイルで選択したテンプレートをテストしてください。Masonry および Carousel テンプレートは特に、小さな画面で見た目が大きく異なります — Accordion の **モバイルレイアウト** オプションを使用し、Masonry の列が電話で過度に狭くならないように確認してください。
- List テンプレートを使用して **Show short description** を有効にした場合、製品に実際に短い説明が設定されていることを確認してください。空の説明行は意図せずに見える可能性があります。
