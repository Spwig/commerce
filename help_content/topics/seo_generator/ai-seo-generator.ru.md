title: Генератор SEO с использованием ИИ
---

Генератор SEO с использованием ИИ автоматически создает мета-заголовки, мета-описания и другое SEO-содержание для ваших продуктов с использованием поставщика ИИ. Вместо того чтобы вручную писать SEO-текст для каждого продукта, вы можете генерировать точное и оптимизированное содержание в пакетном режиме с помощью одного действия.

Ваш магазин поставляется с встроенным генератором SEO, который работает сразу. Вы также можете установить дополнительные компоненты поставщика ИИ из магазина компонентов Spwig для доступа к более мощным языкам модели.

## Как работает генератор SEO

Генератор SEO считывает имя, описание, категорию и атрибуты вашего продукта, а затем использует настроенного поставщика ИИ, чтобы создать SEO-содержание, соответствующее этому продукту. Сгенерированное содержание сохраняется напрямую в поля SEO продукта.

Вы можете генерировать SEO-содержание для отдельных продуктов с экрана редактирования продукта или запускать пакетную генерацию для нескольких продуктов с экрана списка продуктов.

## Настройка поставщика SEO

### Использование встроенного поставщика

Ваш магазин включает встроенный поставщик SEO, который генерирует SEO-содержание детерминированно на основе данных вашего продукта — не требуется внешние ключи API. Он автоматически устанавливается как основной поставщик при новых установках.

Чтобы проверить, что он активен:

1. Перейдите в **Маркетинг > Поставщики SEO**
2. Проверьте, что встроенный поставщик отображается с меткой **ПЕРВОЙ СТУПЕНИ** и статусом **АКТИВНО**
3. Если поставщики не отображаются, нажмите **+ Добавить учетную запись поставщика SEO** и установите **Ключ поставщика** в `deterministic`

### Подключение компонента поставщика ИИ

Для более богатого и контекстуального SEO-содержания вы можете установить компонент поставщика ИИ (например, на основе OpenAI или Claude) из магазина компонентов Spwig.

1. Установите компонент поставщика через систему обновления компонентов (спросите у администратора магазина)
2. Перейдите в **Маркетинг > Поставщики SEO**
3. Нажмите **+ Добавить учетную запись поставщика SEO**
4. Заполните форму:

**Раздел информации о поставщике:**
- **Сайт** — выберите ваш магазин
- **Компонент поставщика** — выберите установленный компонент поставщика ИИ
- **Ключ поставщика** — оставьте пустым при использовании компонентного поставщика
- **Имя учетной записи** — описательное имя, например, `Поставщик SEO OpenAI`

**Раздел настройки:**
- **Активен** — установите флажок, чтобы включить этого поставщика
- **Основной** — установите флажок, чтобы использовать этого поставщика как стандартный для всех генераций SEO
- **Приоритет** — меньшие числа проверяются первыми в цепочке резервного копирования
- **Настройки** — настройки, специфичные для поставщика, в виде объекта JSON (например, имя модели, стиль, язык)

5. Нажмите **Сохранить**

Только один поставщик может быть установлен как основной. Если вы помечаете новый поставщик как основной, предыдущий основной автоматически понижается в приоритете.

### Цепочка резервного копирования поставщика

Если ваш основной поставщик не работает (например, из-за сбоя API), ваш магазин автоматически переходит к следующему активному поставщику в порядке приоритета. Это гарантирует, что генерация SEO продолжается работать, даже если один поставщик временно недоступен.

## Генерация SEO-содержания для продукта

### Отдельный продукт

1. Перейдите в **Продукты > Продукты** и откройте любой продукт
2. Прокрутите до раздела **SEO** в форме продукта
3. Нажмите кнопку **Создать SEO**
4. Поставщик ИИ генерирует мета-заголовок и мета-описание на основе деталей продукта
5. Проверьте сгенерированное содержание и внесите изменения, если это необходимо
6. Нажмите **Сохранить**, чтобы применить изменения

### Пакетная генерация

Чтобы сгенерировать или обновить SEO-содержание для нескольких продуктов сразу:

1. Перейдите в **Продукты > Продукты**
2. Выберите продукты, которые вы хотите обновить, с помощью их флажков, или выберите все
3. Откройте выпадающий список **Действие**
4. Выберите **Создать SEO-содержание** (или похожее название действия — проверьте выпадающий список для точного метки)
5. Нажмите **Go**

Spwig помещает задачи генерации в очередь и обрабатывает их в фоновом режиме. Обновите список продуктов через минуту или две, чтобы увидеть обновленные поля SEO.

## Настройки провайдера

Поле **Настройки** на аккаунте провайдера SEO принимает объект JSON с настройками, специфичными для провайдера. Общие параметры включают:

```json
{
  "language": "en",
  "tone": "professional",
  "max_title_length": 60,
  "max_description_length": 160
}
```

Эти настройки различаются в зависимости от компонента провайдера. Для получения полного списка доступных параметров обратитесь к документации провайдера.

## Управление несколькими провайдерами

Если у вас настроено более одного аккаунта провайдера SEO, список провайдеров покажет их статус в кратком виде:

- **PRIMARY badge** — этот провайдер используется по умолчанию для всех операций генерации SEO
- **ACTIVE badge** — провайдер включен
- **INACTIVE badge** — провайдер отключен и не будет использоваться

Чтобы изменить, какой провайдер является основным, откройте аккаунт провайдера, которого вы хотите сделать основным, поставьте галочку **Is Primary** и сохраните. Система автоматически гарантирует, что только один провайдер имеет флаг основного статуса в любой момент времени.

## Советы

- Генерируйте SEO-контент для новых продуктов сразу после их создания — это занимает всего секунды и дает поисковым системам что-то полезное для индексации сразу
- Проверьте AI-сгенерированные мета-описания перед публикацией, если у ваших продуктов нетипичные или технические названия; генератор работает лучше всего с четкими и описательными названиями продуктов
- Установите в настройках провайдера `"max_title_length": 60` и `"max_description_length": 160`, чтобы сохранить сгенерированный контент в пределах рекомендованных Google ограничений по количеству символов
- Выполните массовую генерацию SEO после импорта большого каталога продуктов, чтобы быстро заполнить все поля SEO
- Если вы значительно обновите описание продукта, перегенерируйте его SEO-контент, чтобы мета-теги соответствовали новому тексту
- Встроенный детерминированный провайдер — хороший старт; обновитесь до компонента на основе ИИ, когда ваш каталог будет настроен, и вы захотите более богатый и естественный SEO-текст
