---
title: ソーシャルシェアリング
---

ソーシャルシェアボタンを使用すると、顧客はショッピングサイトから直接、製品、ブログ記事、ページをソーシャルネットワークに共有できます。どのプラットフォームが表示されるか、ボタンの見た目、配置場所、共有アクティビティの追跡とカウントの有無は、あなたが制御できます。

## ソーシャルシェアリング設定の構成

すべてのソーシャルシェアリングの動作は、単一の設定ページから制御されます。**マーケティング > ソーシャルシェアリング設定**に移動してください（ページは自動的に設定フォームにリダイレクトされます — 設定レコードは1つだけです）。

### 配置: ボタンが表示される場所

**配置**セクションは、どの種類のコンテンツに共有ボタンが自動的に表示されるかを制御します。

| 設定 | 説明 |
|---------|-------------|
| **製品で有効** | 製品詳細ページに共有ボタンを表示 |
| **カテゴリで有効** | カテゴリ一覧ページに共有ボタンを表示 |
| **ブログ投稿で有効** | ブログ投稿ページに共有ボタンを表示 |
| **カスタムページで有効** | カスタムストアページに共有ボタンを表示 |

表示させたいコンテンツタイプにチェックを入れてください。任意の組み合わせを有効にできます — たとえば、製品とブログ投稿のみ。

**配置位置**は、ページ上でのボタンの表示位置を制御します:

| オプション | 説明 |
|--------|-------------|
| **コンテンツの下** (デフォルト) | メインコンテンツの後に表示 |
| **コンテンツの上** | メインコンテンツの前に表示 |
| **サイドバー** | ページのサイドバーに表示 |
| **フローティング（ステッキィ）** | 視覚的にスクロールするにつれて画面の側面に固定表示 |

### 外観: ボタンの見た目

**外観**セクションは、どのプラットフォームが表示されるか、ボタンのスタイルがどうなるかを制御します。

**有効なプラットフォーム** — 空のままにすると、すべてのサポートされているプラットフォームが表示されます。または、JSON配列を入力して表示されるプラットフォームを制限できます:

```json
["facebook", "twitter", "pinterest", "whatsapp", "email"]
```

サポートされているプラットフォームキー: `facebook`, `twitter`, `linkedin`, `pinterest`, `whatsapp`, `telegram`, `email`

**ボタンスタイル**オプション:

すべてのマークダウンフォーマット、画像パス、コードブロック、技術用語を保持してください。

| スタイル | 説明 |
|-------|-------------|
| **アイコンのみ** (デフォルト) | プラットフォームのアイコンのみを表示 |
| **アイコン + ラベル** | アイコンとプラットフォーム名を表示 |
| **ラベルのみ** | プラットフォーム名のみをテキストで表示 |

**ボタンのサイズ** — ストアフロントのデザインに合わせて **小**, **中** (デフォルト), または **大** を選択してください。

**レイアウト方向** — ボタンを **水平** (デフォルト、並べて表示) または **垂直** (積み重ねて表示) に配置します。

**タイトルの表示** — これをチェックすると、ボタングループの上に "共有" という見出しが表示されます。

**モバイルでの表示** は、小画面でのボタンの表示を制御します：

| オプション | 説明 |
|--------|-------------|
| **常に表示** (デフォルト) | すべてのデバイスでボタンが表示される |
| **モバイルで非表示** | モバイルデバイスでボタンが非表示になる |
| **モバイル専用** | モバイルデバイスのみでボタンが表示される |

### トレーキング設定

**共有数の表示** — これをチェックすると、各ボタンにそのプラットフォームが共有された回数を示すバッジが表示されます。共有が記録されるたびに、カウントはリアルタイムで更新されます。

**共有のトレーキング** — これをチェックすると、すべての共有クリックが共有分析に記録されます。これを無効にすると、新しいレコードの保存は停止しますが、既存のデータは削除されません。トレーキングは、共有した顧客にロイヤルティバッジを授与します（ロイヤルティプログラムが有効な場合）。

フォームの下部にある **保存** をクリックして変更を適用してください。設定は直ちに反映されます。

## 共有アクティビティの確認

### 個別の共有イベント

**マーケティング > ソーシャル共有** に移動して、記録されたすべての共有イベントのログを確認してください。各行には以下が表示されます：

- **プラットフォーム** — どのソーシャルネットワークが使用されたか（カラーコード付きバッジで表示）
- **共有されたコンテンツ** — 共有されたコンテンツの種類と名前（例: `product: Blue Widget`）
- **ユーザー** — 共有した顧客、またはログインしていない訪問者には "匿名" と表示されます
- **デバイスの種類** — デスクトップ、モバイル、またはタブレット
- **共有日時** — 共有の日時

共有ログは読み取り専用です — 顧客が共有ボタンをクリックすると、エントリが自動的に作成されます。

**プラットフォーム**と**デバイスの種類**のフィルタを使用して、共有パターンを確認し、日付階層を使用して特定の期間を調べることができます。

### コンテンツごとの共有数

**マーケティング > 共有数**に移動して、コンテンツ項目とプラットフォームごとに集計された共有総数を確認してください。このビューにより、最も共有された製品や投稿を簡単に特定できます。

各行には以下が表示されます：
- **コンテンツ** — アイテムの種類と名前（例：`product: Blue Widget`）
- **プラットフォーム** — ソーシャルネットワーク
- **共有数** — そのプラットフォームでの記録された合計共有数
- **最終更新** — そのカウントが最後に再計算された時刻

リストは共有数の降順で並べられており、最もウイルスのように広がるコンテンツが上に表示されます。共有数は、新しい共有イベントが記録されるたびに自動的に更新されます — 手動で更新する必要はありません。

## 共有のトラッキング方法の理解

顧客が共有ボタンをクリックすると、Spwigは以下を記録します：

1. どのプラットフォームで共有したか
2. どのコンテンツが共有されたか（製品、ブログ投稿、ページなど）
3. ログインしているかどうか（ログインしている場合、共有はそのアカウントに関連付けられ、ロイヤルティ統合に使用されます）
4. そのデバイスの種類
5. 共有されたURL

そのプラットフォームとコンテンツ項目の共有数は自動的にインクリメントされます。**共有数を表示**が有効になっている場合、ページが次に読み込まれたときに、ボタンに更新された数が表示されます。

## ロイヤルティ統合

ロイヤルティプログラムが有効で、**共有をトラッキング**が有効になっている場合、ログインしている顧客はコンテンツを共有したときにロイヤルティバッジを獲得します。ソーシャル共有バッジは、ロイヤルティプログラムのアクションベースのルールの一部です。

共有に対するポイントの設定を構成するには、**顧客 > ロイヤルティルール**に移動し、**アクションベース**タイプと**ソーシャル共有**アクションタイプのルールを探してください。

## ヒント

すべてのマークダウンの書式、画像のパス、コードブロック、技術用語を保持してください。

- プロダクトやブログ投稿で共有を最初に有効にしてください — これらは顧客が自然に共有する可能性が高いコンテンツタイプです
- Pinterestはファッション、インテリア、食品などの視覚的な製品カテゴリで特に価値があります — これらのストアでは`enabled_platforms`リストで優先的に有効にしてください
- WhatsAppの共有は、モバイルでの温かいリファラルから強いコンバージョンをもたらします — WhatsAppでは**Mobile Only**の表示モードを使用し、他のプラットフォームはすべてのデバイスで表示されたままにしてください
- 共有数が膨張していることに気付いた場合は、**Is Admin Traffic**フラグが完全に動作する前に行われた管理者セッションからのテストトラフィックがカウントされている可能性があります — 共有分析からエントリをクリアしてカウントをリセットできます
- 共有数リストを毎月確認し、最も共有されている製品を特定して、ホームページやマーケティングメールでより目立たせるようにしてください
