---
title: 소셜 공유
---

소셜 공유 버튼은 고객이 상품, 블로그 게시물 및 페이지를 직접 판매점에서 소셜 네트워크에 공유할 수 있도록 합니다. 어떤 플랫폼이 표시되는지, 버튼이 어떻게 생겼는지, 어디에 배치되는지, 공유 활동이 추적되고 계산되는지 등을 제어할 수 있습니다.

## 소셜 공유 설정 구성

모든 소셜 공유 동작은 단일 설정 페이지에서 제어됩니다. **마케팅 > 소셜 공유 설정**으로 이동하세요 (이 페이지는 자동으로 설정 폼으로 리디렉션됩니다 — 설정 기록은 하나뿐입니다).

### 위치: 버튼이 나타나는 곳

**위치** 섹션은 어떤 유형의 콘텐츠가 자동으로 공유 버튼을 표시하는지 제어합니다.

| 설정 | 설명 |
|---------|-------------|
| **제품에서 활성화** | 상품 상세 페이지에 공유 버튼 표시 |
| **카테고리에서 활성화** | 카테고리 목록 페이지에 공유 버튼 표시 |
| **블로그 게시물에서 활성화** | 블로그 게시물 페이지에 공유 버튼 표시 |
| **커스텀 페이지에서 활성화** | 커스텀 스토어 페이지에 공유 버튼 표시 |

원하는 콘텐츠 유형에서 체크박스를 선택합니다. 어떤 조합도 가능합니다 — 예를 들어, 상품과 블로그 게시물만 활성화할 수 있습니다.

**위치 위치**는 버튼이 페이지의 어디에 표시되는지를 제어합니다:

| 옵션 | 설명 |
|--------|-------------|
| **내용 아래** (기본값) | 주요 콘텐츠 이후에 표시 |
| **내용 위** | 주요 콘텐츠 이전에 표시 |
| **사이드바** | 페이지의 사이드바에 표시 |
| **부유 (고정)** | 방문자가 스크롤할 때 화면의 측면에 고정 |

### 외관: 버튼이 어떻게 생겼는지

**외관** 섹션은 어떤 플랫폼이 표시되고 버튼이 어떻게 스타일링되는지를 제어합니다.

**활성화된 플랫폼** — 빈 상태로 두면 모든 지원되는 플랫폼을 표시하거나, JSON 배열을 입력하여 표시할 플랫폼을 제한할 수 있습니다:

```json
["facebook", "twitter", "pinterest", "whatsapp", "email"]
```

지원되는 플랫폼 키: `facebook`, `twitter`, `linkedin`, `pinterest`, `whatsapp`, `telegram`, `email`

**버튼 스타일** 옵션:

모든 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어를 유지합니다.

| 스타일 | 설명 |
|-------|-------------|
| **아이콘만** (기본값) | 플랫폼 아이콘만 표시 |
| **아이콘 + 라벨** | 아이콘과 플랫폼 이름을 함께 표시 |
| **라벨만** | 플랫폼 이름만 텍스트로 표시 |

**버튼 크기** — **작은**, **중간** (기본값), 또는 **대형**을 선택하여 가게 디자인과 일치시킵니다.

**레이아웃 방향** — 버튼을 **가로** (기본값, 옆으로 나란히) 또는 **세로** (스택)로 배치합니다.

**제목 표시** — 선택한 경우, 버튼 그룹 위에 "공유" 제목이 나타납니다.

**모바일 표시 설정**은 소형 화면에서 버튼 표시를 제어합니다:

| 옵션 | 설명 |
|--------|-------------|
| **항상 표시** (기본값) | 모든 기기에서 버튼이 표시됨 |
| **모바일에서 숨기기** | 모바일 기기에서 버튼이 숨김 |
| **모바일 전용** | 모바일 기기에서만 버튼이 표시됨 |

### 추적 설정

**공유 횟수 표시** — 선택한 경우, 각 버튼에 해당 플랫폼이 몇 번 공유되었는지 표시하는 횟수 배지가 나타납니다. 공유가 기록될 때마다 실시간으로 업데이트됩니다.

**공유 추적** — 선택한 경우, 모든 공유 클릭이 공유 분석에 기록됩니다. 이 기능을 비활성화하면 새로운 기록이 저장되지 않지만 기존 데이터는 삭제되지 않습니다. 추적 기능은 로열티 프로그램이 활성화된 경우, 공유한 고객에게 로열티 배지를 수여합니다.

폼 하단의 **저장**을 클릭하여 변경 사항을 적용합니다. 설정은 즉시 적용됩니다.

## 공유 활동 보기

### 개별 공유 이벤트

**마케팅 > 소셜 공유**로 이동하여 기록된 모든 공유 이벤트 로그를 볼 수 있습니다. 각 항목은 다음과 같은 정보를 표시합니다:

- **플랫폼** — 사용된 소셜 네트워크 (색상으로 표시된 배지로 표시됨)
- **공유된 콘텐츠** — 공유된 콘텐츠의 유형과 이름 (예: `제품: 블루 위지트`)
- **사용자** — 공유한 고객 또는 로그인하지 않은 방문객은 "익명"으로 표시됨
- **기기 유형** — 데스크탑, 모바일 또는 태블릿
- **공유 시간** — 공유가 이루어진 날짜 및 시간

공유 기록은 읽기 전용입니다 — 고객이 공유 버튼을 클릭할 때 자동으로 항목이 생성됩니다.

**플랫폼** 및 **기기 유형** 필터를 사용하여 공유 패턴을 탐색하고, 날짜 계층 구조를 사용하여 특정 시간 기간을 살펴보세요.

### 콘텐츠별 공유 횟수

**마케팅 > 공유 횟수**로 이동하여 콘텐츠 항목과 플랫폼별로 그룹화된 집계된 공유 총계를 확인할 수 있습니다. 이 뷰는 가장 많이 공유된 제품과 게시물을 쉽게 식별할 수 있도록 도와줍니다.

각 항목은 다음과 같이 표시됩니다:
- **콘텐츠** — 항목의 유형과 이름 (예: `product: Blue Widget`)
- **플랫폼** — 소셜 네트워크
- **공유 횟수** — 해당 플랫폼에서 기록된 총 공유 횟수
- **마지막 업데이트** — 횟수가 마지막으로 재계산된 시점

리스트는 공유 횟수 내림차순으로 정렬되어 있으므로, 가장 바이럴적인 콘텐츠가 상단에 표시됩니다. 공유 횟수는 새로운 공유 이벤트가 기록될 때마다 자동으로 업데이트되며, 수동으로 새로 고치는 필요가 없습니다.

## 공유가 어떻게 추적되는지 이해하기

고객이 공유 버튼을 클릭하면 Spwig은 다음과 같은 정보를 기록합니다:

1. 고객이 어떤 플랫폼에서 공유했는지
2. 어떤 콘텐츠가 공유되었는지 (제품, 블로그 게시물, 페이지 등)
3. 로그인했는지 여부 (로그인한 경우, 공유는 그들의 계정과 연결되어 충성도 통합에 사용됩니다)
4. 고객의 기기 유형
5. 공유된 URL

그 플랫폼과 콘텐츠 항목의 공유 횟수는 자동으로 증가합니다. **공유 횟수 표시**가 활성화된 경우, 페이지가 다시 로드될 때 해당 버튼에 업데이트된 횟수가 표시됩니다.

## 충성도 통합

충성도 프로그램이 활성화되어 있고 **공유 추적**이 활성화된 경우, 로그인한 고객은 콘텐츠를 공유할 때 충성도 배지가 부여됩니다. 소셜 공유 배지는 충성도 프로그램의 액션 기반 규칙의 일부입니다.

공유에 대한 포인트 부여를 구성하려면 **고객 > 충성도 규칙**으로 이동하고 **액션 기반** 유형과 **소셜 공유** 액션 유형을 가진 규칙을 찾아보세요.

## 팁

모든 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어를 그대로 유지하세요.

- 먼저 제품과 블로그 게시물에서 공유 기능을 활성화하세요 — 이는 고객들이 가장 자연스럽게 공유할 가능성이 높은 콘텐츠 유형입니다
- Pinterest는 패션, 홈 인테리어, 음식과 같은 시각적인 제품 카테고리에 특히 유용합니다 — 이러한 매장에서는 `enabled_platforms` 목록에서 Pinterest를 우선적으로 활성화하세요
- WhatsApp 공유는 모바일에서 따뜻한 추천을 통해 강력한 전환율을 유도합니다 — WhatsApp은 **Mobile Only** 표시 모드로 설정하면서 다른 플랫폼은 모든 기기에서 표시되도록 유지하는 것이 좋습니다
- 공유 횟수가 부풀려졌다고 느끼면, **Is Admin Traffic** 플래그가 완전히 작동하기 전에 관리자 세션에서 발생한 테스트 트래픽이 계산되었는지 확인하세요 — 공유 분석에서 항목을 삭제하여 카운트를 재설정할 수 있습니다
- 매달 공유 횟수 목록을 검토하여 가장 많이 공유된 제품을 확인하고, 홈페이지나 마케팅 이메일에서 더 두드러지게 특징을 내세우세요
