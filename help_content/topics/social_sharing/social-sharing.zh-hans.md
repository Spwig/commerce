---
title: 社交分享
---

社交分享按钮可以让客户直接从您的商店前端将您的产品、博客文章和页面分享到社交网络。您可以控制哪些平台显示、按钮的外观、放置的位置，以及是否跟踪和统计分享活动。

## 配置社交分享设置

所有社交分享行为都通过一个设置页面进行控制。导航到 **营销 > 社交分享设置**（页面会自动重定向到设置表单——只有一条设置记录）。

### 位置：按钮显示的位置

**位置**部分控制哪些类型的内容会自动显示分享按钮。

| 设置 | 描述 |
|---------|-------------|
| **在产品上启用** | 在产品详情页面显示分享按钮 |
| **在分类上启用** | 在分类列表页面显示分享按钮 |
| **在博客文章上启用** | 在博客文章页面显示分享按钮 |
| **在自定义页面上启用** | 在自定义商店页面显示分享按钮 |

勾选您希望显示按钮的内容类型。您可以启用任何组合——例如，仅产品和博客文章。

**位置位置**控制按钮在页面上的显示位置：

| 选项 | 描述 |
|--------|-------------|
| **内容下方**（默认） | 在主要内容之后显示 |
| **内容上方** | 在主要内容之前显示 |
| **侧边栏** | 在页面侧边栏中显示 |
| **浮动（固定）** | 当访客滚动时，按钮会固定在视口的一侧 |

### 外观：按钮的样式

**外观**部分控制显示哪些平台以及按钮的样式。

**启用的平台**——留空以显示所有支持的平台，或输入一个 JSON 数组以限制显示的平台：

```json
["facebook", "twitter", "pinterest", "whatsapp", "email"]
```

支持的平台键：`facebook`，`twitter`，`linkedin`，`pinterest`，`whatsapp`，`telegram`，`email`

**按钮样式**选项：

保留所有markdown格式、图片路径、代码块和专业术语。

| 风格 | 描述 |
|-------|-------------|
| **图标仅** (默认) | 仅显示平台图标 |
| **图标 + 标签** | 显示图标和平台名称 |
| **仅标签** | 仅以文本形式显示平台名称 |

**按钮大小** — 选择 **小**, **中** (默认) 或 **大**, 以匹配您的商店设计。

**布局方向** — 将按钮排列为 **水平** (默认, 并排) 或 **垂直** (堆叠)。

**显示标题** — 勾选后, 在按钮组上方会出现一个“分享”标题。

**移动设备可见性** 控制按钮在小屏幕上的显示:

| 选项 | 描述 |
|--------|-------------|
| **始终显示** (默认) | 所有设备上都显示按钮 |
| **在移动设备上隐藏** | 在移动设备上隐藏按钮 |
| **仅移动设备** | 仅在移动设备上显示按钮 |

### 跟踪设置

**显示分享次数** — 勾选后, 每个按钮上会出现一个徽章, 显示该平台被分享的次数。次数会随着分享记录实时更新。

**跟踪分享** — 勾选后, 每次分享点击都会记录在分享分析中。禁用此功能将停止保存新记录, 但不会删除现有数据。跟踪功能还会向分享的客户授予忠诚徽章(如果启用了忠诚计划)。

点击底部表单的 **保存** 以应用您的更改。设置会立即生效。

## 查看分享活动

### 单个分享事件

导航到 **营销 > 社交分享** 以查看所有记录的分享事件日志。每条记录显示:

- **平台** — 使用的社交网络(以彩色徽章形式显示)
- **分享内容** — 被分享内容的类型和名称(例如, `product: Blue Widget`)
- **用户** — 分享的客户, 或“匿名”表示未登录的访客
- **设备类型** — 台式机、移动设备或平板电脑
- **分享时间** — 分享的日期和时间

分享日志为只读 — 当客户点击分享按钮时，条目会自动创建。

使用 **平台** 和 **设备类型** 过滤器来探索分享模式，并使用日期层级查看特定时间段的数据。

### 按内容分类的分享次数

导航到 **营销 > 分享次数**，查看按内容项和平台分组的汇总分享总数。此视图可轻松识别您最受欢迎的产品和帖子。

每条记录显示：
- **内容** — 项目类型和名称（例如，`product: Blue Widget`）
- **平台** — 社交网络
- **分享次数** — 在该平台上记录的总分享次数
- **最后更新** — 计数最后一次重新计算的时间

列表按分享次数降序排列，因此您最热门的内容会出现在顶部。分享次数在每次记录新的分享事件时会自动更新 — 无需手动刷新。

## 了解分享是如何被追踪的

当客户点击分享按钮时，Spwig 会记录：

1. 他们分享的平台
2. 被分享的内容（产品、博客文章、页面等）
3. 他们是否登录（如果登录，分享将与他们的账户关联以进行忠诚度集成）
4. 他们的设备类型
5. 被分享的 URL

该平台和内容项的分享次数会自动递增。如果启用了 **显示分享次数**，则在页面下次加载时，更新后的次数会显示在按钮上。

## 忠诚度集成

如果您的忠诚度计划处于活动状态且 **追踪分享** 已启用，已登录的客户在分享内容时将获得忠诚度徽章。社交分享徽章是忠诚度计划基于行动的规则的一部分。

要配置分享的积分奖励，请导航到 **客户 > 忠诚度规则**，并查找类型为 **基于行动** 且操作类型为 **社交分享** 的规则。

## 小贴士

保留所有 Markdown 格式、图片路径、代码块和专业术语。

- 首先在产品和博客文章上启用分享功能 —— 这些是客户最可能自发分享的内容类型
- Pinterest 对于视觉类产品类别（如时尚、家居装饰和食品）特别有价值 —— 对于这些店铺，请在 `enabled_platforms` 列表中优先启用它
- WhatsApp 分享能带来来自温暖推荐的高转化率，尤其是在移动设备上；可以考虑为 WhatsApp 使用 **仅限移动设备** 的显示模式，同时在所有设备上保持其他平台的可见性
- 如果你发现分享次数被夸大，请检查是否在 **Is Admin Traffic** 标志完全生效之前，统计了来自管理员会话的测试流量 —— 你可以通过清除分享分析中的条目来重置次数
- 每月查看一次分享次数列表，以识别最受欢迎的产品，并在首页或营销邮件中更突出地展示它们
