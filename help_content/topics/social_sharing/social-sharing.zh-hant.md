---
title: 社交分享
---

社交分享按鈕讓客戶可以直接從您的商店前台將您的產品、部落格文章和頁面分享到社交網絡。您可以控制哪些平台顯示、按鈕的外觀、按鈕的位置，以及是否追蹤和計算分享活動。

## 設置社交分享選項

所有社交分享行為都由一個設置頁面控制。導航至 **行銷 > 社交分享設置**（頁面會自動重定向到設置表單——只有一個設置記錄）。

### 放置位置：按鈕顯示的位置

**放置位置** 這個部分控制哪些類型的內容會自動顯示分享按鈕。

| 設置 | 說明 |
|---------|-------------|
| **在產品上啟用** | 在產品詳情頁顯示分享按鈕 |
| **在分類上啟用** | 在分類清單頁顯示分享按鈕 |
| **在部落格文章上啟用** | 在部落格文章頁顯示分享按鈕 |
| **在自定義頁面上啟用** | 在自定義商店頁面顯示分享按鈕 |

勾選您希望顯示按鈕的內容類型。您可以啟用任何組合——例如，僅產品和部落格文章。

**放置位置** 控制按鈕在頁面上顯示的位置：

| 選項 | 說明 |
|--------|-------------|
| **內容下方**（預設） | 在主要內容之後顯示 |
| **內容上方** | 在主要內容之前顯示 |
| **側邊欄** | 在頁面側邊欄顯示 |
| **浮動（固定）** | 當訪客滾動時，按鈕會固定在視口側邊 |

### 外觀：按鈕的樣式

**外觀** 這個部分控制顯示哪些平台以及如何對按鈕進行樣式設計。

**啟用的平台** —— 留空以顯示所有支援的平台，或輸入 JSON 陣列以限制顯示哪些平台：

```json
["facebook", "twitter", "pinterest", "whatsapp", "email"]
```

支援的平台鍵：`facebook`、`twitter`、`linkedin`、`pinterest`、`whatsapp`、`telegram`、`email`

**按鈕樣式** 選項：

保留所有 markdown 格式、圖片路徑、程式碼塊和技術術語。

| 標式 | 說明 |
|-------|-------------|
| **圖示僅** (預設) | 僅顯示平台圖示 |
| **圖示 + 標籤** | 顯示圖示和平台名稱 |
| **僅標籤** | 僅顯示平台名稱為文字 |

**按鈕大小** — 選擇 **小**、**中** (預設) 或 **大**，以配合您的商店設計。

**佈局方向** — 將按鈕排列為 **水平** (預設，並排) 或 **垂直** (堆疊)。

**顯示標題** — 勾選後，按鈕組上方會顯示一個 "分享" 標題。

**行動裝置顯示** 控制按鈕在小螢幕上的顯示方式：

| 選項 | 說明 |
|--------|-------------|
| **總是顯示** (預設) | 所有裝置上都顯示按鈕 |
| **在行動裝置上隱藏** | 在行動裝置上隱藏按鈕 |
| **僅行動裝置** | 僅在行動裝置上顯示按鈕 |

### 追蹤設定

**顯示分享數量** — 勾選後，每個按鈕上會顯示一個數量徽章，顯示該平台被分享的次數。數量會隨著分享的記錄即時更新。

**追蹤分享** — 勾選後，每次點擊分享都會被記錄在分享分析中。停用此功能會停止保存新的記錄，但不會刪除現有的資料。追蹤功能也會向分享的客戶頒發忠誠徽章（如果忠誠計劃處於啟用狀態）。

點擊底部表單的 **儲存** 以套用您的更改。設定會立即生效。

## 查看分享活動

### 個別分享事件

前往 **行銷 > 社交分享**，查看所有已記錄的分享事件日誌。每個項目顯示：

- **平台** — 使用的社交網絡（以彩色徽章顯示）
- **分享內容** — 被分享的內容類型和名稱（例如：`product: Blue Widget`）
- **用戶** — 分享的客戶，或對於未登入的訪客顯示 "匿名"
- **裝置類型** — 桌上型、行動裝置或平板電腦
- **分享時間** — 分享的日期和時間

分享記錄為只讀 — 當顧客點擊分享按鈕時，會自動創建條目。

使用 **平台** 和 **設備類型** 篩選器來探索分享模式，並使用日期層級來查看特定時間段的數據。

### 按內容分類的分享次數

導航至 **行銷 > 分享次數**，可查看按內容項目和平台分組的聚合分享總數。此視圖可輕鬆識別最受歡迎的產品和帖子。

每個條目顯示：
- **內容** — 項目的類型和名稱（例如，`product: Blue Widget`）
- **平台** — 社交網絡
- **分享次數** — 該平台記錄的總分享次數
- **最後更新** — 次數最後重新計算的時間

清單按分享次數降序排列，因此最受歡迎的內容會出現在最上方。分享次數在每次記錄新的分享事件時會自動更新 — 無需手動刷新。

## 了解分享是如何被追蹤的

當顧客點擊分享按鈕時，Spwig 會記錄：

1. 他們在什麼平台分享
2. 分享的內容（產品、部落格文章、頁面等）
3. 他們是否已登入（如果是，分享會與其帳戶連結，以進行忠誠度整合）
4. 他們的設備類型
5. 被分享的 URL

該平台和內容項的分享次數會自動增加。如果啟用了 **顯示分享次數**，則在頁面下次加載時，更新後的次數會出現在按鈕上。

## 忠誠度整合

如果您的忠誠度計劃已啟用，且 **追蹤分享** 已啟用，已登入的顧客在分享內容時會獲得忠誠度徽章。社交分享徽章是忠誠度計劃中基於行動的規則的一部分。

要配置分享的積分獎勵，請導航至 **顧客 > 忠誠度規則**，並查找類型為 **基於行動** 且行動類型為 **社交分享** 的規則。

## 小技巧

保留所有 markdown 格式、圖片路徑、程式碼塊和技術術語。

- 首先啟用產品和部落格文章的分享功能 — 這些是客戶最有可能自然分享的內容類型
- Pinterest 對於視覺化產品類別（如時尚、家居裝飾和美食）特別有價值 — 對於這些商店，請在 `enabled_platforms` 清單中優先啟用 Pinterest
- WhatsApp 分享能從溫暖推薦中帶來強勁的轉化率，尤其是在行動裝置上；考慮為 WhatsApp 使用 **僅限行動裝置** 的顯示模式，同時讓其他平台在所有裝置上都可見
- 如果你發現分享次數被誇大，請檢查是否在 **Is Admin Traffic** 標誌完全啟用之前，已計算了管理員會話的測試流量 — 你可以透過清除分享分析中的條目來重置次數
- 每月審查一次分享次數清單，以識別最受分享的產品，並在首頁或營銷郵件中更突出地展示這些產品
