---
title: SMS 아웃박스
---

SMS 아웃박스는 가게가 시도한 모든 텍스트 메시지의 완전한 기록입니다. 이 기능을 사용하여 알림이 고객에게 전달되었는지 확인하고, 전송 실패를 조사하며, 전체 메시징 활동을 이해할 수 있습니다.

**SMS 시스템 > SMS 아웃박스**로 이동하여 메시지 로그를 확인합니다.

![SMS 아웃박스 목록과 상태 배지](/static/core/admin/img/help/sms-outbox/outbox-list.webp)

## 아웃박스 보기

아웃박스의 각 행은 하나의 메시지 시도를 나타내며 다음을 표시합니다:

- **전화번호** — 수신자의 전화번호
- **메시지 유형** — SMS 또는 WhatsApp
- **상태** — 현재 전송 상태(아래 참조)
- **생성 시간** — 메시지가 생성된 시간
- **전송 시간** — 메시지가 제공업체로 전송된 시간

상단의 요약 바는 가장 중요한 상태에 대한 집계 수치를 한눈에 볼 수 있도록 표시합니다.

## 메시지 상태

| 상태 | 의미 |
|------|------|
| 대기 중 | 메시지가 전송 대기열에서 처리될 준비가 되어 있습니다 |
| 대기 목록 | 메시지가 대기 목록에 추가되었으며 곧 전송될 예정입니다 |
| 전송됨 | 제공업체가 메시지 전송을 수락했습니다 |
| 전달됨 | 제공업체가 메시지가 수신자의 장치에 도달했음을 확인했습니다 |
| 실패 | 제공업체가 메시지를 거부하거나 전송하지 못했습니다 |
| 건너뛰기 | 의도적으로 전송을 건너뛰았습니다(아래의 건너뛰기 이유 참조) |
| 샌드박스 기록 | 메시지가 기록되었지만만 (가게가 테스트/샌드박스 모드에 있습니다) |

> **전송됨 vs 전달됨:** **전송됨** 상태는 메시지가 가게에서 제공업체에 의해 수락되었음을 의미합니다. **전달됨** 상태는 제공업체가 운송업체로부터 전달 확인을 받았음을 의미합니다. 모든 제공업체가 전달 확인을 지원하지는 않으며, 제공업체가 지원하지 않는 경우 메시지가 **전송됨** 상태로 표시되지만 **전달됨** 상태로 진행되지 않을 수 있으며, 이는 정상입니다.

## 메시지 세부 정보 보기

아웃박스의 행을 클릭하여 해당 메시지의 전체 세부 정보를 볼 수 있습니다:

- 전송된 **Message**의 전체 텍스트
- **Provider Message ID** — SMS 제공업체에서 제공하는 참조 번호 (제공업체 지원팀에 문의할 때 유용함)
- **Error Message** (실패한 메시지에 한함) — 제공업체에서 반환한 정확한 오류 메시지
- **Retry Count** — Spwig이 메시지를 전송하려고 시도한 횟수
- 모든 타임스탬프 (created, queued, sent, delivered)

## Outbox 필터링

오른쪽의 필터를 사용하여 목록을 좁힐 수 있습니다:

- **Status** — 특정 상태의 메시지만 표시
- **Message Type** — SMS만 또는 WhatsApp 메시지만 표시
- **Date** — 메시지가 생성된 날짜로 필터링

상단의 검색 상자에서 전화번호, 메시지 내용 또는 제공업체 메시지 ID로 검색할 수 있습니다.

## Skip 사유 이해

스킵된 메시지는 Spwig이 전송이 적절하거나 불필요하다고 판단하여 전송되지 않았습니다. 일반적인 스킵 사유:

| Skip 사유 | 의미 |
|-------------|---------------|
| `user_preference_disabled` | 고객이 계정 설정에서 SMS 알림을 비활성화했습니다 |
| `unsubscribed` | 고객이 SMS 메시지에서 구독을 해제했습니다 |
| `no_provider` | 활성화된 기본 SMS 제공업체 계정이 구성되지 않았습니다 |
| `template_inactive` | 이 알림 유형의 템플릿이 비활성화되어 있습니다 |

스킵된 메시지는 실패가 아닙니다 — 시스템이 예상대로 작동했다는 의미입니다. 그러나 `no_provider` 스킵이 많다면 SMS 제공업체 계정을 구성하고 활성화해야 합니다.

## 전송 실패 문제 해결

메시지가 **Failed** 상태를 보일 경우 다음 단계를 따르세요:

1. 실패한 메시지를 클릭하여 **Error Message**를 확인합니다
2. 일반적인 오류 원인:


   | 오류 | 가능한 원인 |
   |-------|-------------|
   | 유효하지 않은 전화번호 | 고객의 전화번호가 누락되었거나 E.164 형식이 아닙니다 |
   | 인증 실패 | 제공업체 자격 증명이 잘못되었거나 만료되었습니다 — **SMS 제공업체 계정**에서 업데이트하세요 |
   | 계정 일시 중지 | 제공업체 계정이 일시 중지되었습니다 — 제공업체 대시보드에 로그인하세요 |
   | 자금 부족 | 제공업체 계정 잔액이 너무 낮습니다 — 충전하세요 |
   | 통신사 거부 | 목적지 통신사가 메시지를 차단했습니다(보통 콘텐츠 필터링 때문입니다) |

3. 근본적인 문제를 수정한 후에는 향후 메시지가 정상적으로 전송됩니다 — 아웃박스는 읽기 전용 로그이며 개별 메시지를 수동으로 다시 보내는 것은 불가능합니다

## 아웃박스는 읽기 전용입니다

SMS 아웃박스는 기록만 저장하는 것입니다. 수동으로 메시지를 아웃박스에 추가할 수 없으며 여기서 개별 메시지를 다시 보내는 것도 불가능합니다. 메시지는 Spwig이 관련 이벤트가 발생할 때 자동으로 전송됩니다(예: 주문이 생성될 때).

## 팁

- 바쁜 기간 후 아웃박스를 확인하여 모든 주문 확인 메시지가 성공적으로 전송되었는지 확인하세요
- 고객이 SMS를 받지 못했다고 말하면 전화번호로 아웃박스를 검색하여 메시지가 전송되었는지, 실패했는지, 건너뛰었는지 확인하세요
- **실패** 메시지의 급격한 증가량은 제공업체 자격 증명 또는 계정 잔액에 문제가 있음을 나타냅니다 — 즉시 이를 확인하세요
- 많은 **건너뛰기** 메시지가 있고 원인 `no_provider`인 경우 **SMS 시스템 > SMS 제공업체 계정**으로 이동하여 활성화된 기본 계정이 구성되어 있는지 확인하세요
- 목록 상단의 날짜 계층 구조를 통해 날짜, 월, 연도별로 편리하게 이전 메시지를 검토할 수 있습니다
