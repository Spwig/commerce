---
title: เทมเพลต SMS
---

เทมเพลต SMS ควบคุมข้อความของการแจ้งเตือนทุกประเภทที่ร้านค้าของคุณส่งไปยังลูกค้าผ่านข้อความ แต่ละเทมเพลตสอดคล้องกับเหตุการณ์เฉพาะ เช่น การยืนยันการสั่งซื้อ หรือการอัปเดตการจัดส่ง และใช้ตัวแปรแทนที่ที่ Spwig จะแทนที่ด้วยรายละเอียดการสั่งซื้อจริงเมื่อข้อความถูกส่ง

ไปที่ **ระบบ SMS > เทมเพลต SMS** เพื่อดูและแก้ไขเทมเพลตของคุณ

![รายการเทมเพลต SMS](/static/core/admin/img/help/sms-templates/templates-list.webp)

## ประเภทเทมเพลตที่มีอยู่

Spwig มีประเภทเทมเพลตต่อไปนี้ที่ติดตั้งไว้:

| ประเภทเทมเพลต | ส่งเมื่อใด |
|---------------|-----------------|
| การยืนยันการสั่งซื้อ | เมื่อลูกค้าสั่งซื้อ |
| การอัปเดตการจัดส่ง | เมื่อสถานะการติดตามการสั่งซื้อเปลี่ยน |
| การแจ้งเตือนการจัดส่ง | เมื่อการสั่งซื้อถูกทำเครื่องหมายว่าถูกจัดส่ง |
| การรีเซ็ตรหัสผ่าน | เมื่อลูกค้าขอรีเซ็ตรหัสผ่าน |
| รหัสยืนยัน | เมื่อต้องการรหัสที่ใช้ครั้งเดียวสำหรับการยืนยันบัญชี |
| ใบเสร็จ POS | เมื่อการขายถูกประมวลผลที่เครื่องจุดขาย |
| การตลาด | สำหรับแคมเปญส่งเสริมการขาย (ต้องการการยินยอมแยกต่างหาก) |
| เทมเพลตแบบกำหนดเอง | สำหรับการแจ้งเตือนใด ๆ ที่คุณสร้าง |

## การแก้ไขเทมเพลต

1. ไปที่ **ระบบ SMS > เทมเพลต SMS**
2. คลิกที่เทมเพลตที่คุณต้องการแก้ไข
3. อัปเดตฟิลด์ **ข้อความ** ด้วยข้อความที่คุณต้องการ
4. ใช้ตัวแทนที่เป็น `{ตัวแปร}` เพื่อรวมข้อมูลเฉพาะการสั่งซื้อ (ดูตัวแปรด้านล่าง)
5. ตรวจสอบ **เปิดใช้งาน** เพื่อเปิดใช้งานเทมเพลต — เทมเพลตที่ไม่ได้เปิดใช้งานจะไม่ถูกส่ง
6. คลิก **บันทึก**

![การแก้ไขเทมเพลต SMS](/static/core/admin/img/help/sms-templates/template-edit.webp)

## การใช้ตัวแปร

ตัวแปรคือตัวแทนที่เขียนในวงเล็บปีกกา — ตัวอย่างเช่น `{name}` หรือ `{order_number}` เมื่อ Spwig ส่งข้อความ มันจะแทนที่ตัวแทนแต่ละตัวด้วยค่าจริงสำหรับลูกค้าหรือการสั่งซื้อที่เกี่ยวข้อง

### ตัวแปรที่พบบ่อย

รักษาการจัดรูปแบบ markdown ทั้งหมด ค่าเส้นทางรูปภาพ บล็อกโค้ด และคำศัพท์ทางเทคนิคไว้

| ตัวแปร | ถูกแทนด้วย |
|----------|---------------|
| `{name}` | ชื่อของลูกค้า |
| `{order_number}` | หมายเลขอ้างอิงคำสั่งซื้อ |
| `{total}` | จำนวนเงินรวมของคำสั่งซื้อ |
| `{tracking_number}` | หมายเลขติดตามการจัดส่ง |
| `{store_name}` | ชื่อร้านของคุณ |
| `{code}` | รหัสยืนยันหรือรีเซ็ต |

**ตัวอย่างข้อความ:**

```
Hi {name}, your order #{order_number} has been confirmed. Total: {total}. We'll update you when it ships. - {store_name}
```

เมื่อส่งข้อความนี้จะกลายเป็น:

```
Hi Sarah, your order #10045 has been confirmed. Total: $89.00. We'll update you when it ships. - The Garden Shop
```

> ใส่ตัวแปรเฉพาะที่มีอยู่ในประเภทเทมเพลตที่กำหนดเท่านั้น ตัวอย่างเช่น `{tracking_number}` มีอยู่ในเทมเพลตอัปเดตการจัดส่ง แต่ไม่มีในเทมเพลตรีเซ็ตรหัสผ่าน หากคุณใช้ตัวแปรที่ไม่มีอยู่ มันจะปรากฏในข้อความตามที่มันเป็น (ไม่ได้รับการแทนที่)

## ข้อจำกัดของตัวอักษรและการกำหนดความยาวข้อความ

ข้อความ SMS มาตรฐานมีข้อจำกัด **160 ตัวอักษร** สำหรับหนึ่งส่วน ข้อความที่ยาวกว่านี้จะถูกแบ่งเป็นหลายส่วนและส่งเป็น SMS แบบรวมกัน (concatenated SMS) แต่ผู้ให้บริการจะนับแต่ละส่วนแยกกันเพื่อการเรียกเก็บค่าบริการ

**คำแนะนำในการอยู่ในข้อจำกัด:**
- รักษาข้อความให้กระชับ — หนึ่งจุดประสงค์ต่อข้อความ
- ย่อคำหรือประโยคที่ใช้บ่อยเมื่อเป็นธรรมชาติ (เช่น "Ord" แทน "Order")
- หลีกเลี่ยงคำฟุ่มเฟือยที่ไม่จำเป็น

Spwig ไม่บังคับใช้ข้อจำกัดตัวอักษรที่เข้มงวดในตัวแก้ไข ดังนั้นจึงนับตัวอักษรของคุณ (รวมค่าตัวแปร) ก่อนบันทึก

## การเปิดใช้งานและปิดใช้งานเทมเพลต

ปุ่ม **Active** ที่เทมเพลตแต่ละตัวควบคุมว่าประเภทการแจ้งเตือนนั้นจะถูกส่งหรือไม่ หากเทมเพลตไม่ได้เปิดใช้งาน Spwig จะข้ามการส่งการแจ้งเตือนนั้นโดยสิ้นเชิง — ข้อความจะปรากฏเป็น **Skipped** ในกล่องจดหมาย SMS พร้อมเหตุผล `template_inactive`

เพื่อเปิดใช้งานเทมเพลต:
1. เปิดเทมเพลต
2. ตรวจสอบกล่อง **Active**
3. บันทึก

เพื่อปิดใช้งาน (หยุดการส่งการแจ้งเตือนโดยไม่ต้องลบเทมเพลต):
1.

เปิดเทมเพลต
2.


ยกเลิกการเลือก **Active**
3.

บันทึก

## เคล็ดลับ

- เขียนข้อความในสไตล์เดียวกับแบรนด์ของคุณ — SMS เป็นช่องทางสื่อสารที่ตรงไปตรงมาและส่วนตัว ดังนั้นโทนเสียงที่เป็นมิตรจึงเหมาะมาก
- รวมชื่อร้านค้าของคุณในข้อความเสมอ เพื่อให้ลูกค้าทราบว่าใครที่กำลังส่งข้อความมาหาพวกเขา
- รักษาข้อความยืนยันการสั่งซื้อให้กระชับ: หมายเลขการสั่งซื้อ ยอดรวม และหมายเหตุเกี่ยวกับขั้นตอนต่อไปก็เพียงพอแล้ว
- ทดสอบข้อความโดยการสั่งซื้อแบบทดสอบที่ร้านค้าของคุณเอง (โดยใช้เบอร์โทรศัพท์ที่คุณควบคุม) เพื่อดูข้อความที่ลูกค้าจะได้รับโดยตรง
- หากการแจ้งเตือนทำให้เกิดความสับสนหรือการร้องเรียน ให้ปิดการใช้งานเทมเพลตและแก้ไขมันแทนที่จะลบมัน — ด้วยวิธีนี้คุณสามารถเปิดใช้งานมันอีกครั้งเมื่อได้รับการอัปเดตแล้ว
- เทมเพลตสำหรับการตลาดสามารถส่งได้เฉพาะลูกค้าที่ได้ให้ความยินยอมอย่างชัดเจนในการรับข้อความผ่าน SMS เท่านั้น เนื่องจากตามข้อกำหนดของกฎหมายโทรคมนาคมในประเทศส่วนใหญ่
