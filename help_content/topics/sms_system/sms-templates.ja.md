---
title: SMSテンプレート
---

SMSテンプレートは、ストアが顧客にSMSで通知を送信する際のテキストを制御します。各テンプレートは特定のイベント（注文の確認や配送の更新など）に対応し、Spwigがメッセージを送信する際に実際の注文詳細に置き換えるプレースホルダー変数を使用しています。

**SMSシステム > SMSテンプレート**にアクセスして、テンプレートを表示および編集してください。

![SMSテンプレート一覧](/static/core/admin/img/help/sms-templates/templates-list.webp)

## 利用可能なテンプレートタイプ

Spwigには以下の組み込みテンプレートタイプが含まれています：

| テンプレートタイプ | 送信されるタイミング |
|------------------|---------------------|
| 注文確認         | 顧客が注文を置いたとき |
| 配送状況の更新   | 注文の追跡ステータスが変更されたとき |
| 配達通知         | 注文が「配達済み」とマークされたとき |
| パスワードのリセット | 顧客がパスワードのリセットを要求したとき |
| 検証コード       | アカウントの検証にワンタイムコードが必要なとき |
| POSレシート      | ポイントオブセール端末で売上が処理されたとき |
| マーケティング   | プロモーションキャンペーン用（別途同意が必要） |
| カスタム        | その他、自分で作成した通知用 |

## テンプレートの編集

1. **SMSシステム > SMSテンプレート**にアクセスします
2. 編集したいテンプレートをクリックします
3. **メッセージ**フィールドを、希望するテキストで更新します
4. `{変数}`形式のプレースホルダーを使用して、注文固有の情報を含めます（下記の変数を参照）
5. **有効**をチェックしてテンプレートを有効にします — 無効なテンプレートは送信されません
6. **保存**をクリックします

![SMSテンプレートの編集](/static/core/admin/img/help/sms-templates/template-edit.webp)

## 変数の使用

変数は、波かっ括で囲まれたプレースホルダーです — 例えば、`{name}`や`{order_number}`です。Spwigがメッセージを送信する際、各プレースホルダーをその顧客または注文の実際の値に置き換えます。

### 一般的な変数

すべてのMarkdownの書式、画像のパス、コードブロック、技術用語は保持してください。

| 変数 | 代替される内容 |
|----------|---------------|
| `{name}` | 顧客の名前 |
| `{order_number}` | 注文参照番号 |
| `{total}` | 注文の合計金額 |
| `{tracking_number}` | 配送の追跡番号 |
| `{store_name}` | あなたの店舗の名前 |
| `{code}` | 認証またはリセットコード |

**例メッセージ：**

```
Hi {name}, your order #{order_number} has been confirmed. Total: {total}. We'll update you when it ships. - {store_name}
```

送信時にこれは以下になります：

```
Hi Sarah, your order #10045 has been confirmed. Total: $89.00. We'll update you when it ships. - The Garden Shop
```

> あるテンプレートタイプにのみ利用可能な変数を含めるだけにしてください。たとえば、`{tracking_number}`は配送状況更新テンプレートで利用可能ですが、パスワードリセットテンプレートでは利用できません。利用できない変数を使用すると、メッセージにそのままで（置き換えられずに）表示されます。

## 文字数制限とメッセージの長さ

標準的なSMSメッセージは、**160文字**の1セグメントに制限されています。より長いメッセージは複数のセグメントに分割され、1つの（連結された）SMSとして送信されます。ただし、キャリアは各セグメントを個別に料金計算します。

**制限内に収めるためのヒント：**
- メッセージを簡潔に保つ — 1つの目的を持つメッセージにしましょう
- 自然に感じられる場合、一般的なフレーズを略して使用してください（例："Ord" ではなく "Order"）
- 不要な埋め合わせ語を避けてください

Spwigはエディタで厳密な文字数制限を課しませんので、保存する前に変数の値を含めた文字数を数えてください。

## テンプレートの有効化と無効化

各テンプレートの **Active**（有効）トグルは、その通知タイプが送信されるかどうかを制御します。テンプレートが無効になっている場合、Spwigはその通知を完全に送信しません — SMS送信ボックスにメッセージが **Skipped**（スキップ）として表示され、理由は `template_inactive` になります。

テンプレートを有効にするには：
1. テンプレートを開く
2. **Active** チェックボックスをチェック
3. 保存

テンプレートを無効にする（通知タイプの送信を停止するが、テンプレートを削除しない）には：
1.

Open the template
2.


アクティブを外す
3.

保存

## ヒント

- ブランドの声に合わせてメッセージを書く — SMSは直接的で個人的なチャネルなので、フレンドリーなトーンが効果的です
- メッセージに常に店舗名を含めて、顧客が誰からメッセージを受けているかを明確にしましょう
- 注文確認メッセージは簡潔に保つ — 注文番号、合計金額、次の手順に関する注意書きだけがあれば十分です
- 自分の店舗でテスト注文を実施し（コントロール可能な電話番号を使用して）、顧客が実際に受け取る内容を確認してみましょう
- 通知が混乱や苦情を引き起こしている場合は、テンプレートを無効化して修正するようにし、削除しないでください — これにより、更新後に再び有効化できます
- マーケティングテンプレートは、ほとんどの国で電気通信規制により、SMSマーケティングに明示的に同意した顧客だけに送信する必要があります
