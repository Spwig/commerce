---
title: SMS オーダー
---

SMS オーダーは、店舗が送信しようとしたすべてのテキストメッセージの完全な記録です。通知が顧客に届いたかを確認したり、配信失敗を調査したり、全体的なメッセージ活動を理解するために使用してください。

**SMS システム > SMS オーダー**に移動して、メッセージログを表示してください。

![SMS オーダー一覧とステータスバッジ](/static/core/admin/img/help/sms-outbox/outbox-list.webp)

## オーダーの確認

オーダーの各行は、1つのメッセージの送信を表し、以下を表示します：

- **電話番号** — 受信者の電話番号
- **メッセージタイプ** — SMS または WhatsApp
- **ステータス** — 現在の配信ステータス（以下を参照）
- **作成日時** — メッセージが作成された時刻
- **送信日時** — メッセージがプロバイダーに送信された時刻

上部のサマリーバーは、重要なステータスの集計数を一目で確認できるように表示されます。

## メッセージステータス

| ステータス | 意味 |
|----------|------|
| 保留中 | メッセージは送信キューによって取得待ちです |
| キューに追加済み | メッセージはキューに追加され、すぐに送信されます |
| 送信済み | プロバイダーがメッセージの配信を受け入れました |
| 配信済み | プロバイダーがメッセージが受信者のデバイスに届いたことを確認しました |
| 失敗 | プロバイダーがメッセージを拒否または配信できませんでした |
| スキップ | 送信が意図的にスキップされました（下記のスキップ理由を参照） |
| サンクスタックログ | メッセージはログにのみ記録されました（ストアはテスト/サンクスタックモードです） |

> **送信済み vs 配信済み:** **送信済み**のステータスは、メッセージが店舗を離れ、プロバイダーによって受け入れられたことを意味します。**配信済み**のステータスは、プロバイダーがキャリアから配信確認を受け取ったことを意味します。すべてのプロバイダーが配信確認をサポートしているわけではありません — ご使用のプロバイダーがサポートしていない場合、メッセージは**送信済み**のままになり、**配信済み**に進まない場合がありますが、これは正常です。

## メッセージの詳細を表示

オーダーの各行をクリックして、そのメッセージの詳細情報を表示してください：

- 送信された**Message**の完全なテキスト
- **Provider Message ID** — SMSプロバイダーからの参照番号（プロバイダーのサポートに連絡する際に役立ちます）
- **Error Message**（失敗したメッセージ用） — プロバイダーから返された正確なエラーメッセージ
- **Retry Count** — Spwigがメッセージを送信しようとした回数
- すべてのタイムスタンプ（created, queued, sent, delivered）

## 出力リストのフィルタリング

右側のフィルタを使用してリストを絞り込みます：

- **Status** — 特定のステータスを持つみメッセージのみを表示
- **Message Type** — SMSのみまたはWhatsAppのみのメッセージを表示
- **Date** — メッセージが作成された日でフィルタリング

上部の検索ボックスを使用して、電話番号、メッセージ内容、またはプロバイダーのメッセージIDで検索できます。

## スキップ理由の理解

スキップされたメッセージは、Spwigが送信が不適切または不要であると判断したため送信されませんでした。一般的なスキップ理由は以下の通りです：

| スキップ理由 | その意味 |
|-------------|---------------|
| `user_preference_disabled` | 顧客がアカウント設定でSMS通知をオフにしました |
| `unsubscribed` | 顧客がSMSメッセージから解除しています |
| `no_provider` | 有効なデフォルトのSMSプロバイダーのアカウントが設定されていません |
| `template_inactive` | この通知タイプのテンプレートが非アクティブです |

スキップされたメッセージは失敗ではありません — システムが意図通りに動作したことを意味します。ただし、`no_provider`のスキップ数が高ければ、SMSプロバイダーのアカウントを設定し、アクティブ化する必要があります。

## 配送失敗のトラブルシューティング

メッセージが**Failed**のステータスを示している場合、以下の手順に従ってください：

1. 失敗したメッセージをクリックして**Error Message**を表示
2. 一般的なエラーの原因：

   | エラー | おそらく原因 |
   |-------|-------------|
   | 無効な電話番号 | 顧客の電話番号が欠けている、またはE.164形式でない |
   | 認証失敗 | プロバイダの資格情報が無効または期限切れです — **SMSプロバイダアカウント**で更新してください |
   | アカウントの一時停止 | プロバイダのアカウントが一時停止されています — プロバイダのダッシュボードにログインしてください |
   | 資金不足 | プロバイダのアカウント残高が低すぎる — 補充してください |
   | キャリアの拒否 | 宛先のキャリアがメッセージをブロックしました（コンテンツフィルタリングのためなど） |

3. 本来の問題を修正した後、今後のメッセージは通常通り送信されます — オーダーBOXは読み取り専用のログであり、個別のメッセージを手動で再送信することはできません

## オーダーBOXは読み取り専用

SMSオーダーBOXは記録用のみです。オーダーBOXにメッセージを手動で追加することはできず、ここから個別のメッセージを再送信することもできません。メッセージは、Spwigが関連するイベントが発生したときに自動的に送信されます（例: 注文が行われた時）。

## ヒント

- 多忙な時期の後、オーダー確認メッセージがすべて正常に配信されたかを確認するためにオーダーBOXを確認してください
- 顧客がSMSを受信しなかったと述べている場合、電話番号でオーダーBOXを検索して、メッセージが送信された、失敗した、またはスキップされたかを確認してください
- **失敗**メッセージの急激な増加は、プロバイダの資格情報またはアカウント残高に問題があることを示しています — すぐにこれらを確認してください
- **スキップ**メッセージが多数あり、理由が`no_provider`である場合、**SMSシステム > SMSプロバイダアカウント**に移動し、アクティブなデフォルトアカウントが設定されているか確認してください
- リストの上部にある日付階層を使用して、日、月、年ごとに履歴メッセージを迅速に確認できます
