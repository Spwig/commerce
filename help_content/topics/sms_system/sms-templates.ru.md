---
title: Шаблоны SMS
---

Шаблоны SMS управляют текстом каждой уведомления, которые ваш магазин отправляет клиентам через текстовые сообщения. Каждый шаблон соответствует определенному событию — например, подтверждению заказа или обновлению доставки — и использует заполнители переменных, которые Spwig заменяет на реальные данные заказа, когда сообщение отправляется.

Перейдите в **Система SMS > Шаблоны SMS**, чтобы просмотреть и редактировать свои шаблоны.

![Список шаблонов SMS](/static/core/admin/img/help/sms-templates/templates-list.webp)

## Доступные типы шаблонов

Spwig включает следующие встроенные типы шаблонов:

| Тип шаблона | Когда он отправляется |
|---------------|-----------------|
| Подтверждение заказа | Когда клиент оформляет заказ |
| Обновление доставки | Когда статус отслеживания заказа изменяется |
| Уведомление о доставке | Когда заказ помечается как доставленный |
| Сброс пароля | Когда клиент запрашивает сброс пароля |
| Код проверки | Когда требуется одноразовый код для проверки учетной записи |
| Чек из POS | Когда продажа обрабатывается на терминале POS |
| Маркетинг | Для рекламных кампаний (требуется отдельное согласие) |
| Пользовательский | Для любого другого уведомления, которое вы создаете |

## Редактирование шаблона

1. Перейдите в **Система SMS > Шаблоны SMS**
2. Нажмите на шаблон, который вы хотите отредактировать
3. Обновите поле **Сообщение** своим желаемым текстом
4. Используйте заполнители `{переменная}`, чтобы включить информацию, специфичную для заказа (см. переменные ниже)
5. Поставьте галочку **Активный**, чтобы включить шаблон — неактивные шаблоны не отправляются
6. Нажмите **Сохранить**

![Редактирование шаблона SMS](/static/core/admin/img/help/sms-templates/template-edit.webp)

## Использование переменных

Переменные — это заполнители, написанные в фигурных скобках — например, `{имя}` или `{номер_заказа}`. Когда Spwig отправляет сообщение, он заменяет каждый заполнитель на реальное значение для этого клиента или заказа.

### Общие переменные

| Переменная | Заменяется на |
|----------|---------------|
| `{имя}` | Имя клиента |
| `{номер_заказа}` | Номер ссылки на заказ |
| `{итого}` | Общая сумма заказа |
| `{номер_отслеживания}` | Номер отслеживания посылки |
| `{название_магазина}` | Название вашего магазина |
| `{код}` | Код проверки или сброса |

**Пример сообщения:**

```
Привет {имя}, ваш заказ #{номер_заказа} подтвержден. Итого: {итого}. Мы сообщим вам, когда он будет отправлен. - {название_магазина}
```

Когда сообщение отправляется, это становится:

```
Привет Сара, ваш заказ #10045 подтвержден. Итого: $89,00. Мы сообщим вам, когда он будет отправлен. - The Garden Shop
```

> Включайте только те переменные, которые доступны для определенного типа шаблона. Например, `{номер_отслеживания}` доступен в шаблоне обновления доставки, но не в шаблоне сброса пароля. Если вы используете недоступную переменную, она будет отображаться как есть (не замененная) в сообщении.

## Ограничения по количеству символов и длина сообщения

Стандартные сообщения SMS ограничены **160 символами** для одного сегмента. Более длинные сообщения разбиваются на несколько сегментов и отправляются как одно (сцепленное SMS), но операторы считают каждый сегмент отдельно для целей оплаты.

**Советы по соблюдению лимита:**
- Держите сообщения краткими — одна цель на сообщение
- Сокращайте распространенные фразы, где это естественно (например, «Ord» вместо «Order»)
- Избегайте лишних заполнительных слов

Spwig не накладывает жесткий лимит на количество символов в редакторе, поэтому подсчитайте свои символы (включая значения переменных) перед сохранением.

## Активация и деактивация шаблонов

Переключатель **Активный** на каждом шаблоне управляет тем, отправляется ли этот тип уведомления. Если шаблон неактивен, Spwig полностью пропускает отправку этого уведомления — сообщение будет отображаться как **Пропущено** в исходящем ящике SMS с причиной `template_inactive`.

Чтобы активировать шаблон:
1. Откройте шаблон
2. Поставьте галочку **Активный**
3. Сохраните

Чтобы деактивировать (остановить отправку типа уведомления без удаления шаблона):
1. Откройте шаблон
2. Снимите галочку **Активный**
3. Сохраните

## Советы

Сохраните всю разметку Markdown, пути к изображениям, блоки кода и технические термины.

- Используйте стиль, соответствующий вашему бренду — SMS — это прямой и личный канал, поэтому дружелюбный тон будет хорошо работать
- Всегда включайте название вашего магазина в сообщение, чтобы клиенты знали, кто им пишет
- Держите сообщения подтверждения заказа краткими: достаточно номера заказа, общей суммы и примечания о следующих шагах
- Тестируйте сообщения, размещая тестовый заказ на вашем собственном магазине (используя номер телефона, который вы контролируете), чтобы увидеть, что именно получают клиенты
- Если уведомление вызывает путаницу или жалобы, отключите шаблон и внесите в него правки вместо удаления — таким образом вы сможете снова включить его после обновления
- Маркетинговые шаблоны могут отправляться только клиентам, которые явно согласились на получение SMS-рассылок, как это требуется по законодательству в большинстве стран
