---
title: 簡訊發件匣
---

簡訊發件匣是您商店嘗試發送的每條文字訊息的完整記錄。您可以使用它來確認通知是否已傳達給客戶、調查遞送失敗的原因，並了解整體訊息活動情況。

導航至 **簡訊系統 > 簡訊發件匣** 以查看訊息日誌。

![包含狀態徽章的簡訊發件匣列表](/static/core/admin/img/help/sms-outbox/outbox-list.webp)

## 阅讀發件匣

發件匣中的每一行代表一次訊息嘗試，並顯示：

- **電話** — 收件人的電話號碼
- **訊息類型** — 簡訊或 WhatsApp
- **狀態** — 當前遞送狀態（請參閱下文）
- **建立時間** — 訊息建立的時間
- **發送時間** — 訊息發送到提供商的時間

頂部的摘要欄顯示最重要的狀態的聚合數量，一目了然。

## 訊息狀態

| 狀態 | 含義 |
|------|------|
| 待處理 | 訊息正在等待發送佇列處理 |
| 排隊中 | 訊息已排隊，將很快發送 |
| 已發送 | 提供商已接受訊息進行遞送 |
| 已遞送 | 提供商確認訊息已到達收件人的設備 |
| 失敗 | 提供商拒絕或無法遞送訊息 |
| 已跳過 | 故意跳過發送（請參閱下方跳過原因） |
| 沙盒記錄 | 訊息僅被記錄（商店處於測試/沙盒模式） |

> **已發送 vs 已遞送：** **已發送** 狀態表示訊息已離開您的商店，並被提供商接受。**已遞送** 狀態表示提供商已從運營商收到遞送回執。並非所有提供商都支援遞送回執 — 如果您的提供商不支援，訊息可能會顯示為 **已發送**，但永遠不會進展到 **已遞送**，這屬於正常情況。

## 查看訊息詳情

點擊發件匣中的任何一行，以查看該訊息的完整詳情：

- 已發送的完整 **訊息** 文本
- **提供商訊息 ID** — 來自 SMS 提供商的參考編號（在聯繫提供商支持時很有用）
- **錯誤訊息**（針對失敗的訊息）— 提供商返回的確切錯誤
- **重試次數** — Spwig 已嘗試發送訊息的次數
- 所有時間戳（創建、排隊、發送、已遞送）

## 過濾出站訊息

使用右側的過濾器來縮小列表：

- **狀態** — 僅顯示具有特定狀態的訊息
- **訊息類型** — 僅顯示 SMS 或僅顯示 WhatsApp 訊息
- **日期** — 按訊息創建的日期過濾

頂部的搜索框可讓您按電話號碼、訊息內容或提供商訊息 ID 進行搜索。

## 理解跳過原因

跳過的訊息未被發送，因為 Spwig 判斷發送是不恰當或不必要的。常見的跳過原因：

| 跳過原因 | 含義 |
|---------|------|
| `user_preference_disabled` | 顧客在其帳戶設置中關閉了 SMS 通知 |
| `unsubscribed` | 顧客已取消訂閱 SMS 訊息 |
| `no_provider` | 無活動的預設 SMS 提供商帳戶已配置 |
| `template_inactive` | 這個通知類型的模板處於非活動狀態 |

跳過的訊息不是失敗 — 這表示系統按預期運作。然而，大量 `no_provider` 跳過表示您需要配置並啟用 SMS 提供商帳戶。

## 解決遞送失敗的問題

如果訊息顯示 **失敗** 狀態，請按照以下步驟操作：

1. 點擊失敗的訊息以查看其 **錯誤訊息**
2. 常見的錯誤原因：

   | 錯誤 | 可能原因 |
   |-------|-------------|
   | 無效的電話號碼 | 顧客的電話號碼遺漏或不是 E.164 格式 |
   | 認證失敗 | 你的供應商憑證無效或已過期 — 在 **SMS 供應商帳號** 中更新這些憑證 |
   | 帳號被停用 | 你的供應商帳號已被停用 — 登入供應商的儀表板 |
   | 帳戶餘額不足 | 你的供應商帳號餘額過低 — 補充餘額 |
   | 載體拒絕 | 目的地載體阻止了訊息（通常由於內容過濾） |

3. 在解決根本問題後，未來的訊息將會正常發送 — 訊息佇列僅為只讀日誌，無法手動重新發送個別訊息

## 訊息佇列為只讀

SMS 訊息佇列僅為記錄用途。你無法手動新增訊息到佇列，也無法從此處重新發送個別訊息。當相關事件發生時（例如下單），Spwig 會自動發送訊息。

## 小提示

- 在忙碌時期後檢查訊息佇列，確認所有訂單確認訊息都已成功傳送
- 如果顧客表示沒有收到簡訊，請透過他們的電話號碼在訊息佇列中搜索，查看訊息是否已發送、失敗或跳過
- **失敗** 訊息的突然增加通常表示你的供應商憑證或帳號餘額有問題 — 立即檢查這些項目
- 如果看到許多標記為 **跳過** 的訊息，且原因為 `no_provider`，請前往 **SMS 系統 > SMS 供應商帳號**，並確保已設定一個有效的預設帳號
- 清單頂部的日期層級可讓你快速按天、月或年瀏覽歷史訊息
