---
title: กล่องจดหมาย SMS
---

กล่องจดหมาย SMS เป็นบันทึกที่สมบูรณ์ของข้อความทุกข้อที่ร้านของคุณพยายามส่ง ใช้เพื่อยืนยันว่าการแจ้งเตือนถึงลูกค้า ตรวจสอบความล้มเหลวในการส่ง และเข้าใจกิจกรรมการสื่อสารทั่วไปของคุณ

ไปที่ **ระบบ SMS > กล่องจดหมาย SMS** เพื่อดูบันทึกข้อความ

![รายการกล่องจดหมาย SMS พร้อมกับป้ายสถานะ](/static/core/admin/img/help/sms-outbox/outbox-list.webp)

## การอ่านกล่องจดหมาย

แต่ละแถวในกล่องจดหมายแสดงถึงการพยายามส่งข้อความหนึ่งข้อ และแสดง:

- **เบอร์โทรศัพท์** — หมายเลขโทรศัพท์ของผู้รับ
- **ประเภทข้อความ** — SMS หรือ WhatsApp
- **สถานะ** — สถานะการส่งที่กำลังดำเนินอยู่ (ดูด้านล่าง)
- **สร้างเมื่อ** — เมื่อข้อความถูกสร้างขึ้น
- **ส่งเมื่อ** — เมื่อข้อความถูกส่งไปยังผู้ให้บริการ

แถบสรุปด้านบนแสดงจำนวนรวมสำหรับสถานะที่สำคัญที่สุดเพื่อให้เห็นภาพรวมได้ทันที

## สถานะข้อความ

| สถานะ | ความหมาย |
|--------|---------|
| รอการส่ง | ข้อความกำลังรอให้ระบบคิวการส่งรับข้อมูล |
| คิว | ข้อความถูกเพิ่มเข้าคิวและจะถูกส่งเร็ว ๆ นี้ |
| ส่งแล้ว | ผู้ให้บริการยอมรับข้อความเพื่อส่ง |
| ส่งถึงผู้รับแล้ว | ผู้ให้บริการยืนยันว่าข้อความถึงอุปกรณ์ของผู้รับ |
| ล้มเหลว | ผู้ให้บริการปฏิเสธหรือไม่สามารถส่งข้อความได้ |
| ข้าม | การส่งถูกข้ามโดยเจตนา (ดูเหตุผลการข้ามด้านล่าง) |
| บันทึกในสภาพแวดล้อมทดสอบ | ข้อความถูกบันทึกเพียงอย่างเดียว (ร้านค้าอยู่ในโหมดทดสอบ/สแตนด์บี้) |

> **ส่งแล้ว vs ส่งถึงผู้รับแล้ว:** สถานะ **ส่งแล้ว** หมายถึงข้อความออกจากร้านคุณและได้รับการยอมรับจากผู้ให้บริการ สถานะ **ส่งถึงผู้รับแล้ว** หมายถึงผู้ให้บริการได้รับหลักฐานการส่งจากผู้ให้บริการ ไม่ใช่ผู้ให้บริการทุกตัวที่สนับสนุนการยืนยันการส่ง — หากผู้ให้บริการของคุณไม่สนับสนุน การส่งอาจแสดงเป็น **ส่งแล้ว** แต่ไม่เคยก้าวไปถึง **ส่งถึงผู้รับแล้ว** ซึ่งเป็นเรื่องปกติ

## การดูรายละเอียดข้อความ

คลิกที่แถวใดแถวหนึ่งในกล่องจดหมายเพื่อดูรายละเอียดทั้งหมดของข้อความนั้น:

- ข้อความ **Message** ที่ส่งไปทั้งหมด
- **Provider Message ID** — เลขอ้างอิงจากผู้ให้บริการ SMS (มีประโยชน์เมื่อติดต่อฝ่ายสนับสนุนของผู้ให้บริการ)
- **Error Message** (สำหรับข้อความที่ล้มเหลว) — ข้อความข้อผิดพลาดที่ผู้ให้บริการคืนค่ามา
- **Retry Count** — จำนวนครั้งที่ Spwig ได้พยายามส่งข้อความ
- เวลาทั้งหมด (created, queued, sent, delivered)

## การกรองข้อความที่ส่งแล้ว

ใช้ตัวกรองทางด้านขวาเพื่อลดรายการ:

- **Status** — แสดงเฉพาะข้อความที่มีสถานะเฉพาะ
- **Message Type** — แสดงเฉพาะ SMS หรือเฉพาะข้อความ WhatsApp
- **Date** — กรองตามวันที่ข้อความถูกสร้างขึ้น

ช่องค้นหาที่ด้านบนช่วยให้คุณค้นหาตามเบอร์โทรศัพท์ ข้อความ หรือ Provider Message ID

## การเข้าใจเหตุผลที่ข้ามข้อความ

ข้อความที่ถูกข้ามไม่ได้ถูกส่งเพราะ Spwig ตัดสินใจว่าการส่งไม่เหมาะสมหรือไม่จำเป็น เหตุผลที่พบบ่อย:

| Skip Reason | ความหมาย |
|-------------|---------------|
| `user_preference_disabled` | ลูกค้าปิดการแจ้งเตือน SMS ในการตั้งค่าบัญชีของตนเอง |
| `unsubscribed` | ลูกค้าได้ยกเลิกการสมัครรับข้อความ SMS |
| `no_provider` | ไม่มีบัญชีผู้ให้บริการ SMS ที่เปิดใช้งานอยู่ |
| `template_inactive` | แม่แบบสำหรับประเภทการแจ้งเตือนนี้ไม่ได้เปิดใช้งาน |

ข้อความที่ถูกข้ามไม่ถือว่าเป็นความล้มเหลว — มันหมายความว่าระบบทำงานตามที่ตั้งใจไว้ อย่างไรก็ตาม จำนวน `no_provider` ที่สูงแสดงว่าคุณจำเป็นต้องตั้งค่าและเปิดใช้งานบัญชีผู้ให้บริการ SMS

## การแก้ไขปัญหาการส่งที่ล้มเหลว

หากข้อความแสดงสถานะ **Failed** ให้ทำตามขั้นตอนต่อไปนี้:

1. คลิกข้อความที่ล้มเหลวเพื่อดู **Error Message**
2. สาเหตุของข้อผิดพลาดที่พบบ่อย:

   | ข้อผิดพลาด | สาเหตุที่เป็นไปได้ |
   |-------|-------------|
   | หมายเลขโทรศัพท์ไม่ถูกต้อง | หมายเลขโทรศัพท์ของลูกค้าหายไป หรือไม่ได้อยู่ในรูปแบบ E.164 |
   | การตรวจสอบความถูกต้องล้มเหลว | ข้อมูลผู้ให้บริการของคุณไม่ถูกต้องหรือหมดอายุ — อัปเดตใน **SMS Provider Accounts** |
   | บัญชีถูกระงับ | บัญชีผู้ให้บริการของคุณถูกระงับ — เข้าสู่แดชบอร์ดของผู้ให้บริการ |
   | เงินไม่เพียงพอ | ยอดเงินในบัญชีผู้ให้บริการของคุณต่ำเกินไป — จ่ายเงินเพิ่ม |
   | ผู้ให้บริการปฏิเสธ | ผู้ให้บริการปลายทางบล็อกข้อความ (มักเกิดจากตัวกรองเนื้อหา) |

3. หลังจากแก้ไขปัญหาที่แท้จริงแล้ว ข้อความในอนาคตจะถูกส่งตามปกติ — กล่องจดหมายออกเป็นแค่บันทึกอ่านเท่านั้น และไม่สามารถส่งข้อความเดี่ยวได้ด้วยตนเอง

## Outbox is read-only

กล่องจดหมายออกเป็นแค่บันทึกอ่านเท่านั้น

กล่องจดหมายออกเป็นแค่บันทึกอ่านเท่านั้น คุณไม่สามารถเพิ่มข้อความลงในกล่องจดหมายออกได้ด้วยตนเอง และคุณไม่สามารถส่งข้อความเดี่ยวจากที่นี่ได้ ข้อความจะถูกส่งโดย Spwig อัตโนมัติเมื่อเหตุการณ์ที่เกี่ยวข้องเกิดขึ้น (เช่น มีการสั่งซื้อ)

## Tips

- ตรวจสอบกล่องจดหมายออกหลังช่วงเวลาที่ยุ่งเพื่อยืนยันว่าข้อความยืนยันการสั่งซื้อทั้งหมดถูกส่งไปแล้ว
- หากลูกค้าบอกว่าเขาไม่ได้รับ SMS ค้นหาในกล่องจดหมายออกโดยใช้หมายเลขโทรศัพท์ของเขาเพื่อดูว่าข้อความถูกส่ง ล้มเหลว หรือข้ามไปแล้ว
- การเพิ่มขึ้นอย่างกะทันหันของข้อความ **ล้มเหลว** มักบ่งบอกว่ามีปัญหากับข้อมูลผู้ให้บริการหรือยอดเงินบัญชีของคุณ — ตรวจสอบทันที
- หากคุณเห็นข้อความ **ข้าม** จำนวนมากที่มีเหตุผล `no_provider` ให้ไปที่ **SMS System > SMS Provider Accounts** และตรวจสอบให้แน่ใจว่ามีบัญชีผู้ให้บริการที่เปิดใช้งานอยู่ตั้งค่าเป็นค่าเริ่มต้น
- ลำดับชั้นของวันที่ด้านบนของรายการช่วยให้คุณสามารถนำทางไปยังแต่ละวัน เดือน หรือปีเพื่อตรวจสอบข้อความในอดีตได้อย่างรวดเร็ว
