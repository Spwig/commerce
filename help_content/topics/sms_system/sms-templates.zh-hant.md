---
title: 簡訊模板
---

簡訊模板控制您商店透過簡訊發送到客戶的每則通知的內容。每個模板對應一個特定事件——例如訂單確認或運送更新——並使用佔位符變數，當訊息發送時，Spwig 會用實際的訂單細節取代這些變數。

導航至 **簡訊系統 > 簡訊模板** 以查看和編輯您的模板。

![簡訊模板列表](/static/core/admin/img/help/sms-templates/templates-list.webp)

## 可用的模板類型

Spwig 包含以下內建的模板類型：

| 模板類型 | 發送時機 |
|---------------|-----------------|
| 訂單確認 | 當客戶下訂時 |
| 運送更新 | 當訂單的追蹤狀態變更時 |
| 配送通知 | 當訂單標記為已送達時 |
| 密碼重設 | 當客戶請求重設密碼時 |
| 驗證碼 | 當需要一次性驗證碼進行帳號驗證時 |
| POS 收據 | 當在 POS 終端機處理銷售時 |
| 廣告行銷 | 用於促銷活動（需要單獨同意） |
| 自訂 | 用於您創建的任何其他通知 |

## 編輯模板

1. 導航至 **簡訊系統 > 簡訊模板**
2. 點擊您要編輯的模板
3. 在 **訊息** 欄位中更新您想要的文本
4. 使用 `{variable}` 占位符來包含與訂單相關的資訊（請參閱下方的變數）
5. 勾選 **啟用** 以啟用模板 — 未啟用的模板不會發送
6. 點擊 **儲存**

![編輯簡訊模板](/static/core/admin/img/help/sms-templates/template-edit.webp)

## 使用變數

變數是以花括號寫出的佔位符——例如，`{name}` 或 `{order_number}`。當 Spwig 發送訊息時，它會將每個佔位符替換為該客戶或訂單的實際值。

### 常見變數

保留所有 markdown 格式、圖片路徑、程式碼塊和技術術語。

| 變數 | 被替換為 |
|----------|---------------|
| `{name}` | 顧客的名字 |
| `{order_number}` | 訂單參考編號 |
| `{total}` | 訂單總金額 |
| `{tracking_number}` | 運送追蹤編號 |
| `{store_name}` | 您的商店名稱 |
| `{code}` | 驗證或重設碼 |

**範例訊息：**

```
Hi {name}, your order #{order_number} has been confirmed. Total: {total}. We'll update you when it ships. - {store_name}
```

當傳送時，這會變成：

```
Hi Sarah, your order #10045 has been confirmed. Total: $89.00. We'll update you when it ships. - The Garden Shop
```

> 只包含可用於特定模板類型的變數。例如，`{tracking_number}` 可用於運送更新模板，但不能用於密碼重設模板。如果您使用了不可用的變數，它會以原樣（未替換）出現在訊息中。

## 字元限制和訊息長度

標準簡訊訊息限制為 **160 個字元** 一個段落。較長的訊息會被拆分成多個段落並作為一個（連接簡訊）傳送，但電信業者會分別計算每個段落作為計費依據。

**保持在限制內的技巧：**
- 保持訊息簡潔 — 每個訊息只有一個目的
- 在自然的情況下縮寫常見短語（例如 "Ord" 代替 "Order"）
- 避免不必要的填充詞

Spwig 編輯器中不會強制執行硬性字元限制，因此在儲存前請計算您的字元數（包括變數值）。

## 啟用和停用模板

每個模板上的 **Active** 開關控制是否傳送該通知類型。如果模板未啟用，Spwig 會完全跳過傳送該通知 — 該訊息會在簡訊發送匣中顯示為 **Skipped**，原因為 `template_inactive`。

要啟用模板：
1. 開啟模板
2. 勾選 **Active** 方塊
3. 儲存

要停用（停止傳送通知類型但不刪除模板）：
1.

開啟模板
2.


取消勾選 **啟用**
3.

儲存

## 小提示

- 以與您品牌一致的語氣撰寫訊息 —— 短訊是直接且個人化的溝通管道，因此使用親切的語氣效果會很好
- 訊息中務必包含您的商店名稱，讓顧客知道是誰在與他們聯絡
- 保持訂單確認訊息簡潔：訂單編號、總金額，以及關於下一步的說明就足夠了
- 透過在您自己的商店下一個測試訂單（使用您可控的電話號碼）來測試訊息，以確認顧客實際收到的內容
- 如果某則通知引起混淆或投訴，請停用該模板並進行修改，而不是直接刪除它 —— 這樣在更新後可以重新啟用
- 市場營銷模板只能發送給已明確同意接收短信營銷的客戶，這符合大多數國家電信法規的要求
