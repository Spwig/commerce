title: SMSプロバイダの設定

SMS通知は、顧客が注文の各ステップ（確認から配達まで）について常に情報を提供します。店舗からSMSまたはWhatsAppメッセージを送信するには、SMSプロバイダのアカウントをあなたの資格情報を使用して接続します。接続後、Spwigはそのアカウントを使用してすべての出力テキストメッセージを送信します。

**SMSシステム > SMSプロバイダアカウント**にナビゲートして、SMSプロバイダを管理してください。

![SMSプロバイダアカウント一覧](/static/core/admin/img/help/sms-setup/provider-list.webp)

## SMSプロバイダの追加

初回設定の場合は**セットアップウィザード**（推奨）を使用してプロバイダを追加できます。または、手動フォームを使用することもできます。

### セットアップウィザードの使用

1. **SMSシステム > SMSプロバイダアカウント**にナビゲートします
2. ツールバーにある**セットアップウィザード**をクリックします
3. ガイドされた手順に従います：
   - **ステップ1**：利用可能なプロバイダのリストからプロバイダを選択します
   - **ステップ2**：プロバイダの資格情報を入力します（APIキー、アカウントSIDなど）
   - **ステップ3**：表示名とデフォルト設定を設定し、保存します
4. ウィザードは保存する前に自動的に接続をテストします

### 手動でプロバイダを追加する

1. **SMSシステム > SMSプロバイダアカウント**にナビゲートします
2. 利用可能なSMSプロバイダを確認するには**プロバイダをブラウズ**をクリックするか、**+ SMSプロバイダアカウントの追加**を直接クリックします
3. **プロバイダ**フィールドで、ドロップダウンからSMSプロバイダを選択します
4. プロバイダを選択すると、そのプロバイダが要求するものに基づいて資格情報フィールドが自動的に表示されます
5. 必要な資格情報フィールドを入力します（プロバイダごとに異なります — 一般的なプロバイダについては下記のセクションを参照してください）
6. このアカウントを識別する**表示名**を入力します（例：`Twilio — メイン`）
7. **デフォルト設定**を設定します（下記を参照）
8. **保存**をクリックします

## プロバイダ資格情報

### Twilio

| フィールド | 見つける場所 |
|---------|------------|
| Account SID | Twilioコンソール → ダッシュボード |
| Auth Token | Twilioコンソール → ダッシュボード |
| From Number | E.164形式のあなたのTwilio電話番号（例：`+15551234567`） |

### その他のプロバイダ


他のインストール済みSMSプロバイダーコンポーネントは、選択されたときに独自の資格情報フィールドを表示します。

プロバイダーのドキュメントを参照して、必要な正確な値を確認してください — 通常はAPIキーまたはアクセストークンと送信者識別子です。

## デフォルト設定

資格情報を入力した後、このアカウントの使用方法を構成します:

- **Active** — このアカウントを有効または無効にします。デフォルトとして設定されている場合でも、無効なアカウントはSMSの送信には使用されません
- **Default SMS Account** — これをチェックすると、店舗からのすべてのSMS通知がこのアカウントを使用します。一度に1つのアカウントのみがデフォルトのSMSアカウントとして設定できます
- **Default WhatsApp Account** — このプロバイダーがWhatsAppをサポートしている場合（例: Twilio via WhatsApp Business API）、これをチェックしてWhatsAppメッセージのデフォルトとして使用します

## 接続のテスト

プロバイダーアカウントを保存した後、資格情報が正常に動作するかをテストします:

1. **SMS System > SMS Provider Accounts** に移動します
2. あなたのプロバイダーアカウントをクリックして開きます
3. **Test Connection** ボタンをクリックします
4. Spwig はプロバイダーに対してテストリクエストを送信し、**Connection Status** フィールドを更新します

| Status | Meaning |
|--------|---------|
| Connected | 資格情報が有効で、プロバイダーにアクセス可能です |
| Connection Failed | 資格情報が不正、またはプロバイダーにアクセスできません |
| Untested | 接続がまだテストされていません |

テストが失敗した場合は、資格情報を再度確認し、プロバイダーのダッシュボードで必要な権限がアカウントに付与されていることを確認してください。

## 接続ステータス列

SMS Provider Accounts リストは、各アカウントに対して色分けされた **Connection** バッジを表示します:

- **Connected** (緑) — アカウントは正常に動作しています
- **Connection Failed** (赤) — 資格情報の検証に失敗しました — 更新してください
- **Untested** (灰色) — アカウントはまだテストされていません

## ヒント

- 最初のプロバイダーの設定には **Setup Wizard** を使用してください — これはすべてのフィールドをガイドし、保存する前に接続をテストします
- 一度に1つのアカウントのみがデフォルトのSMSアカウントとして設定できます。

# 通知設定

2番目のアカウントを追加し、それをデフォルトに設定すると、以前のデフォルト設定は自動的に解除されます
- プロバイダのAPI資格情報を安全な場所に記録しておきましょう

資格情報が変更された場合は、ここにすぐに更新し、通知の失敗を防ぎましょう
- 有効でないアカウントはリストに残りますが、通知の送信には使用されません — 有効化せずにバックアップ用の資格情報を保持するのに便利です
- 多くのプロバイダは送信されたメッセージごとに料金を請求します — プロバイダのダッシュボードで使用量を確認し、予期せぬ請求を防ぎましょう
