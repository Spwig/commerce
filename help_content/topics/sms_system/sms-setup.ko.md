---
title: SMS 제공업체 설정
---

SMS 알림은 고객의 주문 과정에서 각 단계에 대해 알리게 합니다. 주문 확인부터 배송까지. 가게에서 SMS 또는 WhatsApp 메시지를 보내려면 SMS 제공업체 계정을 자격 증명과 연결해야 합니다. 연결되면 Spwig은 해당 계정을 사용하여 모든 외부 텍스트 메시지를 보냅니다.

**SMS 시스템 > SMS 제공업체 계정**으로 이동하여 SMS 제공업체를 관리합니다.

![SMS 제공업체 계정 목록](/static/core/admin/img/help/sms-setup/provider-list.webp)

## SMS 제공업체 추가

처음 설정하는 경우 **설치 마법사**(권장) 또는 수동 양식을 사용하여 제공업체를 추가할 수 있습니다.

### 설치 마법사 사용

1. **SMS 시스템 > SMS 제공업체 계정**으로 이동합니다.
2. 도구 모음에서 **설치 마법사**를 클릭합니다.
3. 안내된 단계를 따릅니다:
   - **단계 1**: 사용 가능한 제공업체 목록에서 제공업체를 선택합니다.
   - **단계 2**: 제공업체 자격 증명을 입력합니다 (API 키, 계정 SID 등).
   - **단계 3**: 표시 이름과 기본 설정을 설정한 후 저장합니다.
4. 마법사는 저장하기 전에 연결을 자동으로 테스트합니다.

### 수동으로 제공업체 추가

1. **SMS 시스템 > SMS 제공업체 계정**으로 이동합니다.
2. 사용 가능한 SMS 제공업체를 탐색하려면 **제공업체 보기**를 클릭하거나 **+ SMS 제공업체 계정 추가**를 직접 클릭합니다.
3. **제공업체** 필드에서 드롭다운에서 SMS 제공업체를 선택합니다.
4. 제공업체를 선택하면 해당 제공업체가 요구하는 자격 증명 필드가 자동으로 나타납니다.
5. 필요한 자격 증명 필드를 입력합니다 (이 필드는 제공업체에 따라 다릅니다 - 아래의 일반적인 제공업체 섹션을 참조하십시오).
6. 이 계정을 식별할 수 있는 **표시 이름**을 입력합니다 (예: `Twilio — Main`).
7. **기본 설정**을 설정합니다 (아래를 참조하십시오).
8. **저장**을 클릭합니다.

## 제공업체 자격 증명

### Twilio

| 필드 | 어디에서 찾을 수 있는지 |
|-------|-----------------|
| 계정 SID | Twilio 콘솔 → 대시보드 |
| 인증 토큰 | Twilio 콘솔 → 대시보드 |
| From Number | E.164 형식의 Twilio 전화 번호 (예: `+15551234567`) |

### 기타 제공업체


선택된 다른 설치된 SMS 제공업체 구성 요소는 각각의 특정 인증 필드를 표시합니다.

제공업체의 문서를 참조하여 필요한 정확한 값을 확인하세요 — 일반적으로 API 키 또는 액세스 토큰과 발신자 식별자가 필요합니다.

## 기본 설정

자격 증명을 입력한 후 이 계정이 어떻게 사용되는지 구성합니다:

- **활성화** — 이 계정을 활성화하거나 비활성화합니다. 비활성화된 계정은 기본으로 설정되어 있더라도 SMS 전송에 사용되지 않습니다.
- **기본 SMS 계정** — 선택하면, 가게에서 모든 SMS 알림이 이 계정을 사용합니다. 한 번에 하나의 계정만 기본 SMS 계정이 될 수 있습니다.
- **기본 WhatsApp 계정** — 이 제공업체가 WhatsApp을 지원하는 경우(예: WhatsApp Business API를 통해 Twilio), 이 옵션을 선택하여 WhatsApp 메시지의 기본 계정으로 사용합니다.

## 연결 테스트

제공업체 계정을 저장한 후 자격 증명이 작동하는지 테스트합니다:

1. **SMS 시스템 > SMS 제공업체 계정**으로 이동합니다.
2. 제공업체 계정을 클릭하여 열어주세요.
3. **연결 테스트** 버튼을 클릭합니다.
4. Spwig는 제공업체에게 테스트 요청을 보내고 **연결 상태** 필드를 업데이트합니다.

| 상태 | 의미 |
|------|------|
| 연결됨 | 자격 증명이 유효하고 제공업체에 도달할 수 있음 |
| 연결 실패 | 자격 증명이 잘못되었거나 제공업체에 도달할 수 없음 |
| 테스트되지 않음 | 연결이 아직 테스트되지 않았음 |

테스트가 실패하면 자격 증명을 다시 확인하고 제공업체의 대시보드에서 필요한 권한이 있는지 확인하세요.

## 연결 상태 열

SMS 제공업체 계정 목록은 각 계정에 대해 색상으로 표시된 **연결** 배지를 보여줍니다:

- **연결됨** (초록색) — 계정이 작동 중입니다.
- **연결 실패** (빨강색) — 자격 증명이 실패했습니다 — 업데이트하세요.
- **테스트되지 않음** (회색) — 계정이 아직 테스트되지 않았습니다.

## 팁

- 첫 번째 제공업체에 대해 설정 마법사를 사용하세요 — 이는 모든 필드를 안내하고 저장하기 전에 연결을 테스트합니다.
- 한 번에 하나의 계정만 기본 SMS 계정이 될 수 있습니다.

두 번째 계정을 추가하고 기본으로 표시하면 이전 기본 계정은 자동으로 해제됩니다
- 제공업체의 API 자격 증명을 안전한 장소에 기록해 두세요.

자격 증명이 변경되면 여기에서 즉시 업데이트하여 알림 실패를 방지하세요
- 비활성화된 계정은 목록에 남아 있지만 발송에는 사용되지 않음 — 비활성화된 상태로 백업 자격 증명을 유지하는 데 유용함
- 대부분의 제공업체는 전송된 메시지당 요금을 청구함 — 제공업체의 대시보드에서 사용량을 모니터링하여 예상치 못한 요금 청구를 피하세요
