title: การตั้งค่าผู้ให้บริการ SMS
---

การแจ้งเตือนผ่าน SMS จะช่วยให้ลูกค้าของคุณได้รับข้อมูลที่จำเป็นทุกขั้นตอนของการสั่งซื้อ — ตั้งแต่การยืนยันจนถึงการจัดส่งสินค้า ในการส่งข้อความ SMS หรือ WhatsApp จากร้านค้าของคุณ คุณจำเป็นต้องเชื่อมต่อบัญชีผู้ให้บริการ SMS ด้วยข้อมูลประจำตัวของคุณ เมื่อเชื่อมต่อแล้ว Spwig จะใช้บัญชีนั้นเพื่อส่งข้อความทั้งหมดที่ส่งออก

ไปที่ **ระบบ SMS > บัญชีผู้ให้บริการ SMS** เพื่อจัดการผู้ให้บริการ SMS ของคุณ

![รายการบัญชีผู้ให้บริการ SMS](/static/core/admin/img/help/sms-setup/provider-list.webp)

## การเพิ่มผู้ให้บริการ SMS

คุณสามารถเพิ่มผู้ให้บริการโดยใช้ **เครื่องมือตั้งค่าแบบมีคำแนะนำ** (แนะนำสำหรับการตั้งค่าครั้งแรก) หรือใช้แบบฟอร์มแบบมือถือ

### การใช้เครื่องมือตั้งค่าแบบมีคำแนะนำ

1. ไปที่ **ระบบ SMS > บัญชีผู้ให้บริการ SMS**
2. คลิกที่ **เครื่องมือตั้งค่าแบบมีคำแนะนำ** ในแถบเครื่องมือ
3. ทำตามขั้นตอนที่มีคำแนะนำ:
   - **ขั้นตอนที่ 1**: เลือกผู้ให้บริการจากรายการผู้ให้บริการที่มีอยู่
   - **ขั้นตอนที่ 2**: ป้อนข้อมูลประจำตัวของผู้ให้บริการ (คีย์ API, SID บัญชี, ฯลฯ)
   - **ขั้นตอนที่ 3**: ตั้งชื่อแสดงและตั้งค่าเริ่มต้น แล้วบันทึก
4. เครื่องมือจะทดสอบการเชื่อมต่อโดยอัตโนมัติก่อนที่จะบันทึก

### การเพิ่มผู้ให้บริการด้วยตนเอง

1. ไปที่ **ระบบ SMS > บัญชีผู้ให้บริการ SMS**
2. คลิกที่ **ค้นหาผู้ให้บริการ** เพื่อสำรวจผู้ให้บริการ SMS ที่มีอยู่ หรือคลิกที่ **+ เพิ่มบัญชีผู้ให้บริการ SMS** โดยตรง
3. ในช่อง **ผู้ให้บริการ** เลือกผู้ให้บริการ SMS ของคุณจากเมนูแบบเลื่อนลง
4. เมื่อคุณเลือกผู้ให้บริการ ช่องข้อมูลประจำตัวจะปรากฏขึ้นโดยอัตโนมัติ ตามที่ผู้ให้บริการนั้นต้องการ
5. กรอกช่องข้อมูลประจำตัวที่จำเป็น (ข้อมูลเหล่านี้จะแตกต่างกันไปตามผู้ให้บริการ — ดูส่วนที่เกี่ยวข้องด้านล่างสำหรับผู้ให้บริการทั่วไป)
6. ใส่ **ชื่อแสดง** เพื่อระบุบัญชีนี้ (เช่น `Twilio — Main`)
7. ตั้งค่า **ตั้งค่าเริ่มต้น** (ดูข้อมูลด้านล่าง)
8. คลิก **บันทึก**

## ข้อมูลประจำตัวผู้ให้บริการ

### Twilio

| ช่อง | ที่ไหนสามารถหาได้ |
|-----|------------------|
| Account SID | คอนโซล Twilio → แดชบอร์ด |
| Auth Token | คอนโซล Twilio → แดชบอร์ด |
| From Number | เบอร์โทรศัพท์ Twilio ของคุณในรูปแบบ E.164 (เช่น `+15551234567`)

### ผู้ให้บริการอื่น ๆ


ส่วนประกอบผู้ให้บริการ SMS อื่น ๆ จะแสดงฟิลด์ข้อมูลการรับรองเฉพาะของตนเองเมื่อเลือกแล้ว

โปรดอ้างอิงเอกสารของผู้ให้บริการเพื่อหาค่าที่จำเป็นอย่างถูกต้อง — โดยทั่วไปจะเป็น API key หรือ access token และตัวระบุผู้ส่ง

## การตั้งค่าเริ่มต้น

หลังจากป้อนข้อมูลการรับรอง ให้กำหนดว่าบัญชีนี้จะถูกใช้งานอย่างไร:

- **Active** — เปิดหรือปิดการใช้งานบัญชีนี้ บัญชีที่ไม่ได้เปิดใช้งานจะไม่ถูกใช้สำหรับการส่งข้อความ แม้จะตั้งเป็นบัญชีเริ่มต้น
- **Default SMS Account** — เมื่อติ๊กเลือก ข้อความ SMS ทั้งหมดจากร้านค้าของคุณจะใช้บัญชีนี้ บัญชีเพียงหนึ่งบัญชีเท่านั้นที่สามารถเป็นบัญชี SMS เริ่มต้นได้ในเวลาเดียวกัน
- **Default WhatsApp Account** — หากผู้ให้บริการนี้รองรับ WhatsApp (เช่น Twilio ผ่าน WhatsApp Business API) ให้ติ๊กเลือกเพื่อใช้เป็นบัญชีเริ่มต้นสำหรับข้อความ WhatsApp

## การทดสอบการเชื่อมต่อ

หลังจากบันทึกบัญชีผู้ให้บริการ ให้ทดสอบว่าข้อมูลการรับรองทำงานได้:

1. ไปที่ **SMS System > SMS Provider Accounts**
2. คลิกที่บัญชีผู้ให้บริการของคุณเพื่อเปิดมัน
3. คลิกปุ่ม **Test Connection**
4. Spwig ส่งคำขอทดสอบไปยังผู้ให้บริการและอัปเดตฟิลด์ **Connection Status**

| Status | Meaning |
|--------|---------|
| Connected | ข้อมูลการรับรองถูกต้อง และผู้ให้บริการสามารถเข้าถึงได้ |
| Connection Failed | ข้อมูลการรับรองไม่ถูกต้อง หรือผู้ให้บริการไม่สามารถเข้าถึงได้ |
| Untested | การเชื่อมต่อยังไม่ได้รับการทดสอบ |

หากการทดสอบล้มเหลว ตรวจสอบข้อมูลการรับรองอีกครั้ง และตรวจสอบให้แน่ใจว่าบัญชีของคุณมีสิทธิ์ที่จำเป็นในแดชบอร์ดของผู้ให้บริการ

## คอลัมน์สถานะการเชื่อมต่อ

รายการ SMS Provider Accounts แสดง **Connection** badge ที่ใช้สีเพื่อระบุสถานะสำหรับแต่ละบัญชี:

- **Connected** (สีเขียว) — บัญชีทำงานได้
- **Connection Failed** (สีแดง) — ข้อมูลการรับรองล้มเหลว — อัปเดตข้อมูลการรับรอง
- **Untested** (สีเทา) — บัญชียังไม่ได้รับการทดสอบ

## เคล็ดลับ

- ใช้ Wizard ตั้งค่าสำหรับผู้ให้บริการของคุณครั้งแรก — มันจะนำคุณผ่านทุกฟิลด์ และทดสอบการเชื่อมต่อก่อนบันทึก
- มีเพียงบัญชีเดียวเท่านั้นที่สามารถเป็น Default SMS Account ได้ในเวลาเดียวกัน

หากคุณเพิ่มบัญชีที่สองและตั้งเป็นค่าเริ่มต้น บัญชีค่าเริ่มต้นเดิมจะถูกยกเลิกอัตโนมัติ
- จดบันทึกข้อมูลการเข้าถึง API ของผู้ให้บริการไว้ในสถานที่ที่ปลอดภัย

หากข้อมูลการเข้าถึงมีการเปลี่ยนแปลง ให้อัปเดตที่นี่ทันทีเพื่อหลีกเลี่ยงการแจ้งเตือนที่ล้มเหลว
- บัญชีที่ไม่ได้ใช้งานจะยังคงอยู่ในรายการ แต่จะไม่ถูกใช้สำหรับการส่ง — มีประโยชน์ในการเก็บข้อมูลสำรองโดยไม่ต้องเปิดใช้งาน
- ผู้ให้บริการส่วนใหญ่จะคิดค่าบริการตามจำนวนข้อความที่ส่ง — ตรวจสอบการใช้งานในแดชบอร์ดของผู้ให้บริการเพื่อหลีกเลี่ยงค่าใช้จ่ายที่ไม่คาดคิด
