---
title: 簡訊服務供應商設定
---

簡訊通知會在客戶下訂的每個階段通知他們，從確認到配送。若要從您的商店發送簡訊或WhatsApp訊息，請使用您的憑證連接簡訊服務供應商帳戶。一旦連接，Spwig將使用該帳戶發送所有出站文字訊息。

導覽至 **簡訊系統 > 簡訊服務供應商帳戶** 來管理您的簡訊服務供應商。

![簡訊服務供應商帳戶清單](/static/core/admin/img/help/sms-setup/provider-list.webp)

## 添加簡訊服務供應商

您可以使用 **設定向導**（首次設定建議使用）或手動表單來添加供應商。

### 使用設定向導

1. 導覽至 **簡訊系統 > 簡訊服務供應商帳戶**
2. 點擊工具欄中的 **設定向導**
3. 跟隨引導步驟：
   - **步驟 1**：從可用供應商清單中選擇您的供應商
   - **步驟 2**：輸入您的供應商憑證（API金鑰、帳戶SID等）
   - **步驟 3**：設定顯示名稱和預設設定，然後保存
4. 向導在保存前會自動測試連線

### 手動添加供應商

1. 導覽至 **簡訊系統 > 簡訊服務供應商帳戶**
2. 點擊 **瀏覽供應商** 來查看可用的簡訊服務供應商，或直接點擊 **+ 添加簡訊服務供應商帳戶**
3. 在 **供應商** 欄位中，從下拉選單中選擇您的簡訊服務供應商
4. 一旦選擇了供應商，憑證欄位會根據該供應商的要求自動出現
5. 填寫所需的憑證欄位（這些欄位因供應商而異 — 請參閱下方常見供應商部分）
6. 輸入 **顯示名稱** 來識別此帳戶（例如，`Twilio — 主帳號`）
7. 設定 **預設設定**（請參閱下方）
8. 點擊 **保存**

## 供應商憑證

### Twilio

| 欄位 | 找到位置 |
|-----|---------|
| 帳戶SID | Twilio 控制台 → 儀表板 |
| 認證令牌 | Twilio 控制台 → 儀表板 |
| 發送號碼 | 您的Twilio電話號碼，以E.164格式（例如，`+15551234567`） |

### 其他供應商


選擇其他已安裝的簡訊服務提供商組件時，會顯示其特定的憑證欄位。

請參閱您提供商的文件，以獲取所需的精確值 — 通常是一個 API 金鑰或存取令牌，以及發送者識別碼。

## 預設設定

輸入憑證後，配置此帳戶的使用方式：

- **啟用** — 啟用或停用此帳戶。停用的帳戶即使設為預設，也不會用於發送簡訊
- **預設簡訊帳戶** — 勾選後，您商店的所有簡訊通知都會使用此帳戶。一次只能有一個帳戶作為預設簡訊帳戶
- **預設 WhatsApp 帳戶** — 如果此提供商支援 WhatsApp（例如透過 WhatsApp Business API 的 Twilio），勾選此選項可將其設為 WhatsApp 訊息的預設帳戶

## 測試連線

保存提供商帳戶後，測試憑證是否有效：

1. 進入 **簡訊系統 > 簡訊提供商帳戶**
2. 點擊您的提供商帳戶以打開它
3. 點擊 **測試連線** 按鈕
4. Spwig 會向提供商發送測試請求，並更新 **連線狀態** 欄位

| 狀態 | 含義 |
|------|------|
| 已連線 | 憑證有效且提供商可達 |
| 連線失敗 | 憑證錯誤或提供商無法連線 |
| 未測試 | 還未測試連線 |

如果測試失敗，請再次檢查您的憑證，並確保您的帳戶在提供商的儀表板上具有必要的權限。

## 連線狀態欄位

簡訊提供商帳戶清單會為每個帳戶顯示一個彩色標記的 **連線** 標籤：

- **已連線**（綠色） — 帳戶正常運作
- **連線失敗**（紅色） — 憑證驗證失敗 — 請更新憑證
- **未測試**（灰色） — 帳戶尚未測試

## 小提示

- 首次使用提供商時，請使用設定向導 — 它會引導您完成每個欄位，並在保存前測試連線
- 一次只能有一個帳戶作為預設簡訊帳戶。

如果您新增一個帳號並標記為預設，先前的預設帳號會自動取消
- 請將您的提供商 API 權限資料保存在安全的地方。

如果權限資料有所更改，請立即在此處更新，以避免通知失敗
- 非活躍的帳號仍會保留在清單中，但不會用於發送郵件 — 這對於保留備用權限資料而不啟用帳號很有幫助
- 大多數提供商會根據發送的訊息數量收費 — 請在提供商的儀表板中監控使用情況，以避免產生意外費用
