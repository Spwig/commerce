---
title: SMS 템플릿
---

SMS 템플릿은 가게가 고객에게 텍스트 메시지를 통해 보내는 모든 알림의 텍스트를 제어합니다. 각 템플릿은 특정 이벤트(예: 주문 확인 또는 배송 상태 업데이트)에 해당하며, 메시지가 전송될 때 Spwig가 실제 주문 정보로 대체할 수 있는 플레이스홀더 변수를 사용합니다.

**SMS 시스템 > SMS 템플릿**으로 이동하여 템플릿을 보고 편집할 수 있습니다.

![SMS 템플릿 목록](/static/core/admin/img/help/sms-templates/templates-list.webp)

## 사용 가능한 템플릿 유형

Spwig에는 다음의 기본 제공 템플릿 유형이 포함되어 있습니다:

| 템플릿 유형 | 전송 시점 |
|---------------|-----------------|
| 주문 확인 | 고객이 주문을 할 때 |
| 배송 상태 업데이트 | 주문의 추적 상태가 변경될 때 |
| 배송 알림 | 주문이 배송 완료로 표시될 때 |
| 비밀번호 재설정 | 고객이 비밀번호 재설정을 요청할 때 |
| 인증 코드 | 계정 인증을 위해 일회용 코드가 필요한 경우 |
| POS 영수증 | 판매가 포인트 오브 세일 터미널에서 처리될 때 |
| 마케팅 | 프로모션 캠페인용 (별도의 동의 필요) |
| 사용자 정의 | 다른 알림을 생성할 때 사용 |

## 템플릿 편집

1. **SMS 시스템 > SMS 템플릿**으로 이동합니다.
2. 편집하려는 템플릿을 클릭합니다.
3. **메시지** 필드를 원하는 텍스트로 업데이트합니다.
4. `{변수}` 플레이스홀더를 사용하여 주문 관련 정보를 포함합니다(아래 변수를 참조하십시오).
5. **활성화**를 선택하여 템플릿을 활성화합니다. 비활성화된 템플릿은 전송되지 않습니다.
6. **저장**을 클릭합니다.

![SMS 템플릿 편집](/static/core/admin/img/help/sms-templates/template-edit.webp)

## 변수 사용

변수는 중괄호로 표시된 플레이스홀더입니다. 예를 들어 `{이름}` 또는 `{주문번호}`입니다. Spwig가 메시지를 전송할 때 각 플레이스홀더를 해당 고객 또는 주문의 실제 값으로 대체합니다.

### 일반적인 변수

모든 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어는 그대로 유지합니다.

| 변수 | 대체 내용 |
|----------|---------------|
| `{name}` | 고객의 이름 |
| `{order_number}` | 주문 참조 번호 |
| `{total}` | 주문 총 금액 |
| `{tracking_number}` | 배송 추적 번호 |
| `{store_name}` | 귀하의 가게 이름 |
| `{code}` | 확인 또는 재설정 코드 |

**예시 메시지:**

```
Hi {name}, your order #{order_number} has been confirmed. Total: {total}. We'll update you when it ships. - {store_name}
```

보내면 다음과 같이 됩니다:

```
Hi Sarah, your order #10045 has been confirmed. Total: $89.00. We'll update you when it ships. - The Garden Shop
```

> 템플릿 유형에 따라 사용 가능한 변수만 포함하세요. 예를 들어, `{tracking_number}`는 배송 업데이트 템플릿에서 사용할 수 있지만 비밀번호 재설정 템플릿에서는 사용할 수 없습니다. 사용 가능한 변수가 아닌 경우, 메시지에 그대로 (대체되지 않은 상태로) 나타납니다.

## 문자 수 제한 및 메시지 길이

표준 SMS 메시지는 **160자**로 한 세그먼트에 제한됩니다. 더 긴 메시지는 여러 세그먼트로 나뉘어 하나의 메시지로 전송되지만, 통신사들은 각 세그먼트를 별도로 요금에 반영합니다.

**제한을 지키는 팁:**
- 메시지를 간결하게 유지하세요 — 메시지 하나당 하나의 목적만 포함하세요
- 자연스러운 경우 일반적인 표현을 약자로 줄이세요 (예: "Ord" 대신 "Order")
- 불필요한 채움 단어를 피하세요

Spwig은 편집기에서 강제적인 문자 수 제한을 적용하지 않으므로, 저장하기 전에 변수 값 포함한 문자 수를 확인하세요.

## 템플릿 활성화 및 비활성화

각 템플릿의 **Active** 토글은 해당 알림 유형이 전송되는지 여부를 제어합니다. 템플릿이 비활성화되면 Spwig은 해당 알림을 완전히 보내지 않으며, SMS 발신함에서 이 메시지는 **Skipped**로 표시되고 원인은 `template_inactive`입니다.

템플릿을 활성화하려면:
1. 템플릿을 열어주세요
2. **Active** 확인란을 선택하세요
3. 저장하세요

템플릿을 비활성화하려면 (알림 유형을 보내지 않되 템플릿을 삭제하지 않고 유지하려면):
1.

템플릿을 열어주세요
2.


활성화 해제 **Active**
3.

저장

## 팁

- 브랜드의 목소리와 일치하는 메시지를 작성하세요 — SMS는 직접적이고 개인적인 채널이므로 친근한 어조가 잘 어울립니다
- 메시지에 항상 가게 이름을 포함하여 고객이 누가 자신에게 메시지를 보�는지 알 수 있도록 해주세요
- 주문 확인 메시지는 간결하게 유지하세요: 주문 번호, 총 금액, 그리고 다음 단계에 대한 참고사항만 포함하면 됩니다
- 자신의 가게에서 테스트 주문을 진행하여 정확히 고객이 받는 메시지를 확인해 보세요 (제어할 수 있는 전화번호를 사용하세요)
- 알림이 혼란을 일으키거나 불만을 유발하는 경우 템플릿을 비활성화하고 수정하세요. 삭제하지 말고 수정한 후 다시 활성화할 수 있도록 해야 합니다
- 마케팅 템플릿은 대부분의 국가에서 통신 규제에 따라 SMS 마케팅에 명시적으로 동의한 고객에게만 전송해야 합니다
