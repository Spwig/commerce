---
title: 短信发送箱
---

短信发送箱是您店铺尝试发送的每条短信的完整记录。您可以使用它来确认通知是否已送达客户，调查发送失败的情况，并了解整体短信活动情况。

导航至 **短信系统 > 短信发送箱** 以查看短信日志。

![带有状态徽章的短信发送箱列表](/static/core/admin/img/help/sms-outbox/outbox-list.webp)

## 查看发送箱

发送箱中的每一行代表一次短信发送尝试，并显示以下信息：

- **电话** — 接收者的电话号码
- **消息类型** — 短信或WhatsApp
- **状态** — 当前的发送状态（见下文）
- **创建时间** — 消息创建的时间
- **发送时间** — 消息发送给提供商的时间

顶部的摘要栏显示了最重要的状态的汇总数量，便于快速查看。

## 消息状态

| 状态 | 含义 |
|------|------|
| 等待中 | 消息正在等待发送队列处理 |
| 已排队 | 消息已排队，即将发送 |
| 已发送 | 提供商已接受该消息进行发送 |
| 已送达 | 提供商确认消息已送达接收者的设备 |
| 已失败 | 提供商拒绝或无法发送该消息 |
| 已跳过 | 故意跳过发送（请参见下方跳过原因） |
| 沙盒记录 | 仅记录消息（店铺处于测试/沙盒模式） |

> **已发送 vs 已送达：** **已发送** 状态表示消息已离开您的店铺并被提供商接受。**已送达** 状态表示提供商已从运营商处收到送达回执。并非所有提供商都支持送达回执 — 如果您的提供商不支持，消息可能会显示为 **已发送**，但永远不会进入 **已送达** 状态，这是正常现象。

## 查看消息详情

点击发送箱中的任何一行，查看该消息的完整详情：

- 已发送的完整 **消息** 文本
- **提供商消息ID** — 来自短信提供商的参考编号（在联系提供商支持时很有用）
- **错误消息**（针对失败的消息）— 提供商返回的确切错误
- **重试次数** — Spwig 尝试发送消息的次数
- 所有时间戳（创建时间、排队时间、发送时间、送达时间）

## 过滤出站消息

使用右侧的筛选器来缩小列表：

- **状态** — 仅显示具有特定状态的消息
- **消息类型** — 仅显示短信或仅显示WhatsApp消息
- **日期** — 按消息创建的日期进行筛选

顶部的搜索框允许您通过电话号码、消息内容或提供商消息ID进行搜索。

## 理解跳过原因

被跳过的消息未被发送，因为Spwig认为发送是不适当或不必要的。常见的跳过原因：

| 跳过原因 | 含义 |
|---------|------|
| `user_preference_disabled` | 客户在其账户设置中关闭了短信通知 |
| `unsubscribed` | 客户已取消订阅短信消息 |
| `no_provider` | 未配置任何活动的默认短信提供商账户 |
| `template_inactive` | 此通知类型的模板处于非活动状态 |

被跳过的消息不是失败 — 这意味着系统按预期工作。但是，`no_provider`跳过次数较高表明您需要配置并激活一个短信提供商账户。

## 故障排除发送失败

如果消息显示 **失败** 状态，请按照以下步骤操作：

1. 点击失败的消息以查看其 **错误消息**
2. 常见的错误原因：

   | 错误 | 可能原因 |
   |-------|-------------|
   | 无效的电话号码 | 顾客的电话号码缺失或不是E.164格式 |
   | 认证失败 | 你的提供商凭证无效或已过期 —— 在 **短信提供商账户** 中更新它们 |
   | 账户被暂停 | 你的提供商账户已被暂停 —— 登录提供商的仪表板 |
   | 资金不足 | 你的提供商账户余额太低 —— 充值 |
   | 运营商拒绝 | 目标运营商阻止了消息（通常由于内容过滤） |

3. 在解决根本问题后，未来的消息将正常发送 —— 发件箱是一个只读日志，不能手动重新发送单个消息

## 发件箱是只读的

短信发件箱仅用于记录。你不能手动向发件箱添加消息，也不能从此处重新发送单个消息。当相关事件发生时（例如下单），Spwig会自动发送消息。

## 提示

- 在繁忙时段后查看发件箱，确认所有订单确认消息是否已成功送达
- 如果客户表示未收到短信，请通过他们的电话号码在发件箱中搜索，查看消息是否已发送、失败或跳过
- **失败**消息的突然激增通常表明提供商凭证或账户余额存在问题 —— 立即检查这些内容
- 如果看到许多原因标记为 `no_provider` 的 **跳过**消息，请前往 **短信系统 > 短信提供商账户**，确保已配置一个活动的默认账户
- 列表顶部的日期层级可帮助你按天、月或年快速导航，查看历史消息
