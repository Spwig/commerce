---
title: Настройка SMS-провайдера
---

SMS-уведомления информируют ваших клиентов на каждом этапе их заказа — от подтверждения до доставки. Чтобы отправлять SMS или сообщения WhatsApp из вашего магазина, подключите учетную запись SMS-провайдера с вашими учетными данными. После подключения Spwig использует эту учетную запись для отправки всех исходящих текстовых сообщений.

Перейдите в **Система SMS > Учетные записи SMS-провайдеров**, чтобы управлять своими SMS-провайдерами.

![Список учетных записей SMS-провайдеров](/static/core/admin/img/help/sms-setup/provider-list.webp)

## Добавление SMS-провайдера

Вы можете добавить провайдера с помощью **Мастера настройки** (рекомендуется для первого использования) или вручную.

### Использование мастера настройки

1. Перейдите в **Система SMS > Учетные записи SMS-провайдеров**
2. Нажмите **Мастер настройки** в панели инструментов
3. Следуйте пошаговым инструкциям:
   - **Шаг 1**: Выберите своего провайдера из списка доступных провайдеров
   - **Шаг 2**: Введите свои учетные данные провайдера (ключи API, SID аккаунта и т.д.)
   - **Шаг 3**: Установите имя отображения и настройки по умолчанию, затем сохраните
4. Мастер автоматически проверяет соединение перед сохранением

### Добавление провайдера вручную

1. Перейдите в **Система SMS > Учетные записи SMS-провайдеров**
2. Нажмите **Обзор провайдеров**, чтобы изучить доступных SMS-провайдеров, или нажмите **+ Добавить учетную запись SMS-провайдера** напрямую
3. В поле **Провайдер** выберите своего SMS-провайдера из выпадающего списка
4. После выбора провайдера, поля учетных данных появятся автоматически в зависимости от требований этого провайдера
5. Заполните необходимые поля учетных данных (они различаются в зависимости от провайдера — см. разделы ниже для распространенных провайдеров)
6. Введите **Имя отображения**, чтобы идентифицировать эту учетную запись (например, `Twilio — Основной`)
7. Установите **Настройки по умолчанию** (см. ниже)
8. Нажмите **Сохранить**

## Учетные данные провайдера

### Twilio

| Параметр | Где найти |
|---------|----------|
| Account SID | Консоль Twilio → Панель управления |
| Auth Token | Консоль Twilio → Панель управления |
| From Number | Ваш номер телефона Twilio в формате E.164 (например, `+15551234567`) |

### Другие провайдеры

Другие установленные компоненты SMS-провайдеров будут отображать свои собственные специфические поля учетных данных при выборе. Обратитесь к документации вашего провайдера для получения точных значений — обычно это API-ключ или токен доступа и идентификатор отправителя.

## Настройки по умолчанию

После ввода учетных данных настройте, как будет использоваться эта учетная запись:

- **Активный** — включить или отключить эту учетную запись. Неактивные учетные записи не используются для отправки, даже если они установлены как по умолчанию
- **Основная учетная запись SMS** — при включении все SMS-уведомления из вашего магазина будут использовать эту учетную запись. Только одна учетная запись может быть основной учетной записью SMS одновременно
- **Основная учетная запись WhatsApp** — если этот провайдер поддерживает WhatsApp (например, Twilio через WhatsApp Business API), включите это, чтобы использовать его как основную для сообщений WhatsApp

## Проверка соединения

После сохранения учетной записи провайдера проверьте, работают ли учетные данные:

1. Перейдите в **Система SMS > Учетные записи SMS-провайдеров**
2. Нажмите на учетную запись провайдера, чтобы открыть ее
3. Нажмите кнопку **Проверить соединение**
4. Spwig отправляет тестовый запрос провайдеру и обновляет поле **Состояние соединения**

| Состояние | Значение |
|----------|--------|
| Подключено | Учетные данные действительны, и провайдер доступен |
| Ошибка подключения | Учетные данные неверны или провайдер недоступен |
| Не проверено | Соединение еще не проверялось |

Если проверка не удалась, дважды проверьте свои учетные данные и убедитесь, что ваша учетная запись имеет необходимые разрешения в панели управления провайдера.

## Столбец состояния соединения

Список учетных записей SMS-провайдеров отображает цветную метку **Соединение** для каждой учетной записи:

- **Подключено** (зеленый) — учетная запись работает
- **Ошибка подключения** (красный) — учетные данные не прошли проверку — обновите их
- **Не проверено** (серый) — учетная запись еще не проверялась

## Советы

- Используйте мастер настройки для первого провайдера — он поможет вам пройти через каждое поле и проверит соединение перед сохранением
- Только одна учетная запись может быть основной учетной записью SMS одновременно.

Если вы добавите второй аккаунт и установите его как основной, предыдущий основной автоматически снимется
- Запишите свои учетные данные API провайдера в безопасном месте.

Если учетные данные изменятся, обновите их здесь немедленно, чтобы избежать сбоев уведомлений
- Неактивные аккаунты остаются в списке, но не используются для отправки — это полезно для хранения резервных учетных данных без их активации
- Большинство провайдеров взимают плату за каждое отправленное сообщение — отслеживайте использование в панели управления провайдера, чтобы избежать неожиданных счетов
