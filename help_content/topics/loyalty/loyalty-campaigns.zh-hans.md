---
title: 忠诚度活动
---

忠诚度活动让您能够运行限时促销和自动奖励，这些奖励超越了您日常的积分规则。使用它们来运行双倍积分周末、在客户生日时奖励客户、赢回不活跃的购物者，并向特定成员群体提供定向奖金。

每个活动定义了一个触发器或计划、适用的成员以及要执行的操作。一旦激活，活动会自动触发——您只需设置一次，Spwig 会处理其余部分。

## 活动类型

| 类型 | 触发时机 |
|------|---------------|
| **基于触发的** | 当发生特定事件时（例如，下单、检测到生日） |
| **定时的** | 在重复的时间表上（每天、每周、每月） |
| **手动的** | 仅当您从管理后台明确运行时 |
| **基于行为的** | 当客户匹配某种行为模式时（例如，浏览但未购买） |

## 创建活动

导航到 **促销 > 忠诚度活动**，然后点击 **+ 添加忠诚度活动**。

### 第 1 步：基本信息

- **名称** —— 一个清晰且具有描述性的名称，仅在管理后台中可见（例如，`生日奖金 — 200 积分`）
- **Slug** —— 从名称自动生成；用于内部用途
- **描述** —— 关于活动目的的可选说明
- **活动类型** —— 从上表中选择类型

### 第 2 步：触发器或计划

**对于基于触发的活动**，设置触发活动的 **触发事件**。可用的触发器包括：

你可以添加 **触发条件** 作为 JSON 对象，以进一步筛选活动触发的条件。例如，仅当订单金额超过 100 美元时触发：

{
  "min_order_amount": 100
}

**对于定时活动**，设置 **调度类型**（每日、每周、每月或自定义 Cron），并在 **调度配置** 字段中配置时间：

{
  "hour": 9,
  "minute": 0
}

### 第 3 步: actions

**Actions** 字段定义了活动触发时会发生什么。输入一个动作对象的 JSON 数组。最常见的动作是奖励积分：

[
  {
    "type": "award_points",
    "points": 200,
    "description": "生日奖励 — 感谢您成为会员！"
  }
]

其他可用动作包括发送电子邮件通知或奖励徽章。请参阅您的提供商组件文档以获取完整列表。

### 第 4 步: targeting

使用目标字段控制活动适用于哪些会员：

- **Target All Members** — 默认勾选；活动适用于所有活跃的忠诚会员
- **Target Segment** — 限制活动仅适用于特定分段的会员（参见下方的 [Segments](#managing-member-segments)）
- **Target Tiers** — 限制活动仅适用于特定忠诚等级的会员

### 第5步：限制和冷却时间

- **每个会员的最大触发次数** —— 同一个会员可以从中受益的次数。设置为 `1` 以实现一次性奖励，例如生日奖励。留空表示无限制。
- **冷却天数** —— 同一个会员两次活动触发之间的最小天数。例如，设置为 `365` 以防止生日活动每年触发超过一次。

### 第6步：活动日期

设置 **开始日期** 和 **结束日期** 以使活动具有时间限制。两个都留空表示活动持续进行。

活动可以处于以下状态之一：

| 状态 | 描述 |
|--------|-------------|
| **草稿** | 已创建但尚未激活；可以安全地配置和测试 |
| **激活中** | 正在运行，并在满足条件时触发 |
| **暂停** | 暂时停止，但不会丢失配置 |
| **已结束** | 已过结束日期；不再触发 |
| **已归档** | 从活动列表中隐藏，但保留记录 |

填写完所有字段后，点击 **保存**。然后将状态更改为 **激活中** 以启动活动。

## 实际示例

### 示例：周末双倍积分

**场景：** 在特定周末期间的所有订单购买时奖励2倍积分。

| 字段 | 值 |
|-------|-------|
| 名称 | `Double Points Weekend — March` |
| 活动类型 | 基于触发 |
| 触发事件 | 订单创建 |
| 动作 | `["{\"type\": \"award_points_multiplier\", \"multiplier\": 2.0}"]` |
| 开始日期 | 周五晚上 |
| 结束日期 | 周日午夜 |
| 目标所有会员 | 勾选 |

### 示例：生日奖励

**场景：** 在会员生日当天，给每个忠诚会员奖励200积分。

| 字段 | 值 |
|-------|-------|
| 名称 | `Birthday Bonus` |
| 活动类型 | 基于触发 |
| 触发事件 | 客户生日 |
| 动作 | `["{\"type\": \"award_points\", \"points\": 200, \"description\": \"Happy birthday from us!\"}"]` |
| 每个会员的最大触发次数 | 1 |
| 冷却天数 | 365 |
| 目标所有会员 | 勾选 |

### 示例：挽回客户活动


**场景：** 向90天内未购买的会员发送100个积分奖励。

| 字段 | 值 |
|-------|-------|
| 名称 | `90天回流奖励` |
| 活动类型 | 基于触发 |
| 触发事件 | 90天未活跃 |
| 操作 | `["{\"type\": \"award_points\", \"points\": 100, \"description\": \"We miss you — here are some bonus points\"}"]` |
| 每位会员最大触发次数 | 1 |
| 冷却天数 | 180 |
| 针对所有会员 | 已勾选 |

## 管理会员分组

分组功能允许您针对特定的忠诚会员群体进行活动投放。导航至 **促销 > 忠诚分组** 来管理它们。

### 分组类型

| 类型 | 描述 |
|------|-------------|
| **基于规则** | 会员资格由规则决定（例如，拥有超过1,000积分的会员） |
| **动态计算** | 会员资格根据实时条件按需计算 |
| **手动分配** | 会员手动添加到该分组 |

### 创建分组

1. 导航至 **促销 > 忠诚分组**，然后点击 **+ 添加忠诚分组**
2. 填写以下内容：
   - **名称** — 描述性名称（例如，`高价值客户`，`银级会员`）
   - **Slug** — 自动生成
   - **条件类型** — 会员资格的确定方式
   - **条件配置** — 定义会员规则的JSON对象
3. 点击 **保存**

#### 示例：拥有500+积分的会员分组

```json
{
  "min_available_points": 500
}
```

#### 示例：仅限金级会员的分组

```json
{
  "tier_slugs": ["gold"]
}
```

分组列表中的 **会员数量** 列显示当前匹配的会员数量。点击某个分组并使用 **刷新会员数量** 操作，如果数据已更改，可以重新计算该数量。

## 跟踪活动效果

### 活动执行历史

导航至 **促销 > 活动执行**，查看每次活动对任何会员触发的记录。每条执行记录会显示哪个活动运行了，为哪个会员运行，以及结果。

### 查看活动覆盖范围

打开任意活动记录，查看 **触发次数** 的计数以及活动最后一次触发的时间。

这能让你快速了解有多少会员从该活动中受益。

## 小贴士

- 首先以 **草稿** 状态创建活动，以便在活动上线前检查所有设置
- 在所有一次性奖励活动（生日、首次购买、注册）中使用 **每位会员最大触发次数**，以防止客户多次获得奖励
- 将 **目标分群** 与基于触发的活动结合使用，以运行专属等级的促销活动 —— 例如，仅限金卡和白金卡会员的购买双倍积分
- 在挽回客户活动上设置 **冷却天数**，以防止会员在进行小额购买后短时间内再次失活时收到过多通知
- 活动列表是你跟踪当前活动促销的最佳工具 —— 在推出新优惠前查看它，以确保活动不会无意中叠加
- 将已结束的活动归档而不是删除，以便保留你运行过哪些促销以及何时运行的历史记录
