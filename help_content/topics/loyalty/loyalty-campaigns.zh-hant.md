---
title: 忠誠度活動
---

忠誠度活動讓您可以運行有限時間的促銷活動和自動化獎勵，這些活動超越您日常的積分規則。您可以使用這些活動來運行雙倍積分的週末活動、在顧客生日時贈送禮物、重新吸引不活躍的購物者，並向特定群組的成員提供針對性的獎勵。

每個活動都會定義觸發條件或時間表、適用的會員，以及要執行的操作。一旦啟用，活動會自動觸發——您只需設置一次，Spwig 會處理其餘部分。

## 活動類型

| 類型 | 觸發時機 |
|------|---------------|
| **基於觸發的活動** | 當發生特定事件時（例如，下單、檢測到生日） |
| **定時活動** | 在重複的時間表上（每日、每周、每月） |
| **手動活動** | 僅當您從管理員處明確運行時 |
| **行為活動** | 當顧客符合某種行為模式時（例如，瀏覽但未購買） |

## 創建活動

導航至 **Promotions > Loyalty Campaigns** 並點擊 **+ Add Loyalty Campaign**。

### 第 1 步：基本資訊

- **名稱** — 一個清晰且具描述性的名稱，僅在管理員介面中顯示（例如，`Birthday Bonus — 200 Points`）
- **Slug** — 從名稱自動生成；用於內部用途
- **描述** — 關於活動目的的可選備註
- **活動類型** — 從上表中選擇類型

### 第 2 步：觸發條件或時間表

**對於基於觸發的活動**，設置觸發活動的 **觸發事件**。可用的觸發條件包括：

您可以新增 **觸發條件** 為 JSON 物件，以進一步過濾活動觸發的時機。例如，僅針對金額超過 $100 的訂單觸發：

{
  "min_order_amount": 100
}

**對於預定時活動**，請設定 **排程類型**（每日、每週、每月或自定義 Cron），並在 **排程設定** 欄位中設定時間：

{
  "hour": 9,
  "minute": 0
}

### 第 3 步：動作

**動作** 欄位定義了活動觸發時會發生什麼。請輸入一個動作物件的 JSON 陣列。最常見的動作是贈送額外積分：

[
  {
    "type": "award_points",
    "points": 200,
    "description": "生日禮遇 — 謝謝您成為我們的會員！"
  }
]

其他可用的動作包括發送電子郵件通知或贈送徽章。請參閱您的供應商組件文件以取得完整清單。

### 第 4 步：目標對象

使用目標對象欄位來控制活動適用於哪些會員：

- **針對所有會員** — 預設已勾選；活動適用於所有活躍的忠誠度會員
- **針對特定分群** — 限制活動僅適用於特定分群的會員（請參閱下方的 [分群](#managing-member-segments)）
- **針對等級** — 限制活動僅適用於特定忠誠度等級的會員

### 第5步：限制與冷卻時間

- **每位會員的最大觸發次數** — 同一位會員可以從此活動中受益的次數。設為 `1` 可用於一次性獎勵，例如生日獎勵。留空表示無限制。
- **冷卻天數** — 同一位會員之間活動觸發的最小天數。例如，設為 `365` 可以防止生日活動每年觸發超過一次。

### 第6步：活動日期

設定 **開始日期** 和 **結束日期** 以使活動具有時間限制。兩者都留空表示活動持續進行。

活動可以處於以下狀態之一：

| 狀態 | 說明 |
|------|------|
| **草稿** | 已建立但尚未啟用；可以安全地配置和測試 |
| **啟用** | 正在運行，當條件滿足時會觸發 |
| **暫停** | 暫時停止，但不會丟失配置 |
| **已結束** | 已過結束日期；不再觸發 |
| **已歸檔** | 從活動清單中隱藏，但保留用於記錄 |

填寫所有欄位後，點擊 **保存**。然後將狀態更改為 **啟用** 以開始活動。

## 實際例子

### 例子：週末雙倍積分

**情境：** 在特定週末期間的所有訂購，贈送雙倍積分。

| 欄位 | 值 |
|------|----|
| 名稱 | `Double Points Weekend — March` |
| 活動類型 | 基於觸發 |
| 觸發事件 | 訂單下單 |
| 動作 | `["{\"type\": \"award_points_multiplier\", \"multiplier\": 2.0}"]` |
| 開始日期 | 星期五晚上 |
| 結束日期 | 星期日午夜 |
| 目標所有會員 | 勾選 |

### 例子：生日獎勵

**情境：** 為所有忠誠會員在其生日贈送 200 個額外積分。

| 欄位 | 值 |
|------|----|
| 名稱 | `Birthday Bonus` |
| 活動類型 | 基於觸發 |
| 觸發事件 | 顧客生日 |
| 動作 | `["{\"type\": \"award_points\", \"points\": 200, \"description\": \"Happy birthday from us!\"}"]` |
| 每位會員的最大觸發次數 | 1 |
| 冷卻天數 | 365 |
| 目標所有會員 | 勾選 |

### 例子：挽回客戶活動


**情境：** 向90天內未購買的會員發送100個獎勵積分。

| 欄位 | 值 |
|-------|-------|
| 名稱 | `90-Day Win-Back Bonus` |
| 活動類型 | 基於觸發 |
| 觸發事件 | 90天未活躍 |
| 動作 | `["{\"type\": \"award_points\", \"points\": 100, \"description\": \"We miss you — here are some bonus points\"}"]` |
| 每位會員最大觸發次數 | 1 |
| 冷卻天數 | 180 |
| 目標所有會員 | 已勾選 |

## 管理會員分群

分群讓您可以針對特定的忠誠度會員群組進行活動。前往 **Promotions > Loyalty Segments** 來管理它們。

### 分群類型

| 類型 | 說明 |
|------|-------------|
| **基於規則** | 會員資格由規則決定（例如，擁有超過1,000個積分的會員） |
| **動態計算** | 會員資格根據即時條件動態計算 |
| **手動指定** | 會員手動加入分群 |

### 建立分群

1. 前往 **Promotions > Loyalty Segments**，點擊 **+ Add Loyalty Segment**
2. 填寫：
   - **名稱** — 描述性名稱（例如，`High-Value Customers`，`Silver Tier Members`）
   - **Slug** — 自動產生
   - **條件類型** — 會員資格如何決定
   - **條件設定** — 定義會員規則的JSON物件
3. 點擊 **Save**

#### 範例：擁有500個積分以上的會員分群

```json
{
  "min_available_points": 500
}
```

#### 範例：僅限黃金等級會員的分群

```json
{
  "tier_slugs": ["gold"]
}
```

分群列表中的 **Member Count** 欄位顯示目前符合條件的會員數量。點擊分群並使用 **Refresh Member Count** 動作來重新計算，如果您的資料已更改。

## 跟蹤活動表現

### 活動執行歷史

前往 **Promotions > Campaign Executions** 可查看每次活動對任何會員觸發的記錄。每個執行記錄會顯示哪個活動執行了，是為哪位會員執行的，以及結果。

### 檢查活動範圍

打開任何活動記錄，查看 **觸發次數** 的數量以及活動上次觸發的時間。

這讓你可以快速了解有多少會員從該活動中受益。

## 小技巧

- 首先以 **草稿** 狀態建立活動，這樣可以在活動上線前審查所有設定
- 在所有一次性獎勵活動（生日、首次購買、註冊）中使用 **每位會員最大觸發次數**，以防止客戶多次獲得獎勵
- 將 **目標群組** 與基於觸發的活動結合使用，來運行階梯專屬促銷活動 — 例如，僅限黃金和白金會員的雙倍積分購買優惠
- 在回顧客戶活動中設定 **冷卻天數**，以避免會員在進行小額購買後不久再次失活時收到過多通知
- 活動列表是你追蹤目前活躍促銷的最佳工具 — 在發佈新優惠前查看它，以確保活動不會意外疊加
- 將已結束的活動歸檔而不是刪除，這樣你可以保留歷史記錄，了解曾經運行過哪些促銷以及時間
