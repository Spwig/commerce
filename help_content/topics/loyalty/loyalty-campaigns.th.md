---
title: แคมเปญสะสมแต้ม
---

แคมเปญสะสมแต้มช่วยให้คุณสามารถจัดโปรโมชันแบบมีระยะเวลาจำกัดและมอบรางวัลแบบอัตโนมัติที่เหนือกว่ากฎการสะสมแต้มทั่วไป ใช้แคมเปญเหล่านี้เพื่อจัดโปรโมชันเพิ่มแต้มในช่วงวันสุดสัปดาห์ มอบรางวัลให้ลูกค้าในวันเกิด ดึงกลับลูกค้าที่ไม่ได้ใช้งาน และมอบโบนัสแบบเฉพาะเจาะจงให้กับกลุ่มสมาชิกที่กำหนด

แต่ละแคมเปญจะกำหนดเหตุการณ์หรือตารางเวลา สมาชิกที่ใช้ได้ และการกระทำที่ต้องดำเนินการ เมื่อแคมเปญเปิดใช้งานแล้ว จะทำงานอัตโนมัติ — คุณตั้งค่าเพียงครั้งเดียว และ Spwig จะจัดการส่วนที่เหลือให้คุณ

## ประเภทของแคมเปญ

| ประเภท | เวลาที่ทำงาน |
|------|---------------|
| **แบบกระตุ้น** | เมื่อเกิดเหตุการณ์เฉพาะ (เช่น มีการสั่งซื้อ ตรวจพบวันเกิด) |
| **แบบกำหนดเวลา** | ตามตารางเวลาที่ซ้ำกัน (รายวัน รายสัปดาห์ รายเดือน) |
| **แบบมือถือ** | เมื่อคุณเปิดใช้งานจากแดชบอร์ดเท่านั้น |
| **แบบพฤติกรรม** | เมื่อลูกค้าตรงกับรูปแบบพฤติกรรม (เช่น ดูสินค้าแต่ไม่ซื้อ) |

## การสร้างแคมเปญ

ไปที่ **Promotions > Loyalty Campaigns** และคลิก **+ Add Loyalty Campaign**

### ขั้นตอนที่ 1: ข้อมูลพื้นฐาน

- **ชื่อ** — ชื่อที่ชัดเจนและอธิบายได้ ใช้ได้เฉพาะในแดชบอร์ด (เช่น `Birthday Bonus — 200 Points`)
- **Slug** — สร้างอัตโนมัติจากชื่อ; ใช้ภายในระบบ
- **คำอธิบาย** — หมายเหตุเพิ่มเติมเกี่ยวกับวัตถุประสงค์ของแคมเปญ
- **ประเภทแคมเปญ** — เลือกประเภทจากตารางข้างต้น

### ขั้นตอนที่ 2: กระตุ้นหรือกำหนดเวลา

**สำหรับแคมเปญแบบกระตุ้น** ตั้งค่า **เหตุการณ์กระตุ้น** ที่ทำให้แคมเปญทำงาน ตัวกระตุ้นที่มีให้ใช้งานได้รวมถึง:

คุณสามารถเพิ่ม **เงื่อนไขการกระตุ้น** เป็นวัตถุ JSON เพื่อกรองเพิ่มเติมว่าแคมเปญจะทำงานเมื่อใด ตัวอย่างเช่น เพื่อให้ทำงานเฉพาะคำสั่งซื้อที่มีมูลค่าเกิน $100:

{
  "min_order_amount": 100
}

**สำหรับแคมเปญที่กำหนดเวลา** ให้ตั้งค่า **ประเภทการกำหนดเวลา** (รายวัน รายสัปดาห์ รายเดือน หรือ Cron แบบกำหนดเอง) และตั้งค่าเวลาในช่อง **การกำหนดเวลา**:

{
  "hour": 9,
  "minute": 0
}

ขั้นตอนที่ 3: การกระทำ

ช่อง **การกระทำ** กำหนดว่าเกิดอะไรขึ้นเมื่อแคมเปญถูกกระตุ้น ใส่รายการวัตถุการกระทำในรูปแบบ JSON การกระทำที่พบบ่อยที่สุดคือการให้คะแนนโบนัส:

[
  {
    "type": "award_points",
    "points": 200,
    "description": "ของขวัญวันเกิด — ขอบคุณที่เป็นสมาชิก!"
  }
]

การกระทำอื่น ๆ ที่มีอยู่รวมถึงการส่งการแจ้งเตือนทางอีเมลหรือการให้เครื่องหมาย โปรดดูเอกสารประกอบขององค์ประกอบผู้ให้บริการของคุณสำหรับรายการทั้งหมด

ขั้นตอนที่ 4: การกำหนดเป้าหมาย

ควบคุมว่าแคมเปญจะใช้กับสมาชิกใดโดยใช้ช่องกำหนดเป้าหมาย:

- **กำหนดเป้าหมายทุกสมาชิก** — เลือกโดยค่าเริ่มต้น; แคมเปญจะใช้กับสมาชิกความภักดีที่ใช้งานอยู่ทุกคน
- **กำหนดเป้าหมายกลุ่ม** — จำกัดแคมเปญให้กับสมาชิกในกลุ่มเฉพาะ (ดู [กลุ่ม](#managing-member-segments) ด้านล่าง)
- **กำหนดเป้าหมายระดับ** — จำกัดแคมเปญให้กับสมาชิกในระดับความภักดีเฉพาะ

รักษาการจัดรูปแบบมาร์กดาวน์ ชื่อเส้นทางภาพ บล็อกโค้ด และคำศัพท์ทางเทคนิคไว้ทั้งหมด

### ขั้นตอนที่ 5: ข้อจำกัดและช่วงเวลาการรีเซ็ต

- **จำนวนการกระตุ้นสูงสุดต่อสมาชิก** — จำนวนครั้งที่สมาชิกคนเดียวกันสามารถได้รับประโยชน์จากแคมเปญนี้ ตั้งค่าเป็น `1` สำหรับโบนัสแบบครั้งเดียว เช่น ของขวัญวันเกิด ทิ้งว่างไว้หากไม่ต้องการจำกัด
- **ช่วงเวลาการรีเซ็ต (วัน)** — จำนวนวันขั้นต่ำระหว่างการกระตุ้นแคมเปญสำหรับสมาชิกคนเดียวกัน ตัวอย่างเช่น ตั้งค่าเป็น `365` เพื่อป้องกันไม่ให้แคมเปญวันเกิดทำงานมากกว่าหนึ่งครั้งต่อปี

### ขั้นตอนที่ 6: วันที่ของแคมเปญ

ตั้งค่า **วันเริ่มต้น** และ **วันสิ้นสุด** เพื่อทำให้แคมเปญมีระยะเวลาจำกัด ทิ้งว่างทั้งสองช่องหากต้องการให้แคมเปญดำเนินต่อไป

แคมเปญสามารถอยู่ในสถานะใดสถานะหนึ่งต่อไปนี้:

| สถานะ | คำอธิบาย |
|--------|-------------|
| **ร่าง (Draft)** | สร้างแล้วแต่ยังไม่เปิดใช้งาน ปลอดภัยสำหรับการตั้งค่าและการทดสอบ |
| **เปิดใช้งาน (Active)** | กำลังทำงานและจะทำงานเมื่อเงื่อนไขถูกกระตุ้น |
| **ระงับ (Paused)** | หยุดชั่วคราวโดยไม่สูญเสียการตั้งค่า |
| **สิ้นสุด (Ended)** | เกินวันสิ้นสุดแล้ว ไม่ทำงานอีกต่อไป |
| **เก็บถาวร (Archived)** | ซ่อนจากรายการที่ใช้งานอยู่แต่ยังคงเก็บไว้ในประวัติ |

หลังจากกรอกข้อมูลทั้งหมด ให้คลิก **บันทึก (Save)** จากนั้นเปลี่ยนสถานะเป็น **เปิดใช้งาน (Active)** เพื่อเริ่มต้นแคมเปญ

## ตัวอย่างการใช้งานจริง

### ตัวอย่าง: ได้คะแนนเพิ่ม 2 เท่าในช่วงวันหยุดสุดสัปดาห์

**สถานการณ์:** ให้คะแนน 2 เท่าสำหรับการซื้อทุกครั้งที่ทำในช่วงสุดสัปดาห์ที่กำหนด

| ฟิลด์ | ค่า |
|-------|-------|
| ชื่อ | `Double Points Weekend — March` |
| ประเภทแคมเปญ | Trigger-Based |
| เหตุการณ์การกระตุ้น | Order Placed |
| การกระทำ | `["{\"type\": \"award_points_multiplier\", \"multiplier\": 2.0}"]` |
| วันเริ่มต้น | เวลาเย็นวันศุกร์ |
| วันสิ้นสุด | เที่ยงคืนวันอาทิตย์ |
| กำหนดเป้าหมายทุกสมาชิก | ติ๊ก |

### ตัวอย่าง: โบนัสวันเกิด

**สถานการณ์:** ให้คะแนนโบนัส 200 คะแนนแก่สมาชิกสะสมแต้มทุกคนในวันเกิดของพวกเขา

| ฟิลด์ | ค่า |
|-------|-------|
| ชื่อ | `Birthday Bonus` |
| ประเภทแคมเปญ | Trigger-Based |
| เหตุการณ์การกระตุ้น | Customer Birthday |
| การกระทำ | `["{\"type\": \"award_points\", \"points\": 200, \"description\": \"Happy birthday from us!\"}"]` |
| จำนวนการกระตุ้นสูงสุดต่อสมาชิก | 1 |
| ช่วงเวลาการรีเซ็ต (วัน) | 365 |
| กำหนดเป้าหมายทุกสมาชิก | ติ๊ก |

### ตัวอย่าง: แคมเปญเรียกคืนลูกค้า


**สถานการณ์:** ส่งคะแนนโบนัส 100 คะแนนให้สมาชิกที่ไม่ได้ซื้อสินค้าในช่วง 90 วันที่ผ่านมา

| Field | Value |
|-------|-------|
| Name | `90-Day Win-Back Bonus` |
| Campaign Type | Trigger-Based |
| Trigger Event | Inactive 90 Days |
| Actions | `["{\"type\": \"award_points\", \"points\": 100, \"description\": \"We miss you — here are some bonus points\"}"]` |
| Max Triggers Per Member | 1 |
| Cooldown Days | 180 |
| Target All Members | Checked |

## Managing member segments

Segments ช่วยให้คุณสามารถกำหนดเป้าหมายแคมเปญไปยังกลุ่มสมาชิกความภักดีที่เฉพาะเจาะจงได้ ให้ไปที่ **Promotions > Loyalty Segments** เพื่อจัดการกับมัน

### Segment types

| Type | Description |
|------|-------------|
| **Rule-Based** | การเป็นสมาชิกถูกกำหนดโดยกฎ (เช่น สมาชิกที่มีคะแนนเกิน 1,000 คะแนน) |
| **Dynamic Calculation** | การเป็นสมาชิกถูกคำนวณตามเกณฑ์แบบเรียลไทม์เมื่อต้องการ |
| **Manual Assignment** | สมาชิกถูกเพิ่มเข้าไปในกลุ่มด้วยมือ |

### Creating a segment

1. ให้ไปที่ **Promotions > Loyalty Segments** และคลิก **+ Add Loyalty Segment**
2. กรอกข้อมูลดังนี้:
   - **Name** — ชื่อที่อธิบายได้ชัดเจน (เช่น `High-Value Customers`, `Silver Tier Members`)
   - **Slug** — สร้างโดยอัตโนมัติ
   - **Criteria Type** — วิธีที่การเป็นสมาชิกถูกกำหนด
   - **Criteria Config** — วัตถุ JSON ที่กำหนดกฎการเป็นสมาชิก
3. คลิก **Save**

#### Example: segment for members with 500+ points

```json
{
  "min_available_points": 500
}
```

#### Example: segment for Gold tier members only

```json
{
  "tier_slugs": ["gold"]
}
```

คอลัมน์ **Member Count** ในรายการกลุ่มจะแสดงจำนวนสมาชิกที่ตรงกับเงื่อนไขในขณะนี้ คลิกที่กลุ่มและใช้การกระทำ **Refresh Member Count** เพื่อคำนวณใหม่หากข้อมูลของคุณมีการเปลี่ยนแปลง

## Tracking campaign performance

### Campaign execution history

ให้ไปที่ **Promotions > Campaign Executions** เพื่อดูบันทึกทุกครั้งที่แคมเปญถูกดำเนินการสำหรับสมาชิกใดๆ บันทึกการดำเนินการแต่ละรายการจะแสดงแคมเปญใดที่ถูกดำเนินการ สมาชิกใดที่ถูกดำเนินการ และผลลัพธ์

### Reviewing a campaign's reach

เปิดดูบันทึกแคมเปญใด ๆ เพื่อดูจำนวน **Times Triggered** และเวลาที่แคมเปญถูกเปิดใช้งานครั้งสุดท้าย

นี่ให้คุณมีมุมมองอย่างรวดเร็วว่ามีสมาชิกกี่คนที่ได้รับประโยชน์จากแคมเปญ

## เคล็ดลับ

- สร้างแคมเปญในสถานะ **Draft** ก่อน เพื่อที่คุณจะได้ตรวจสอบการตั้งค่าทั้งหมดก่อนที่จะเปิดใช้งาน
- ใช้ **Max Triggers Per Member** สำหรับแคมเปญโบนัสแบบครั้งเดียว (วันเกิด, การซื้อครั้งแรก, การสมัคร) เพื่อป้องกันไม่ให้ลูกค้าได้รับโบนัสมากกว่าหนึ่งครั้ง
- รวม **Target Segment** กับแคมเปญที่ใช้การกระตุ้นเพื่อเปิดตัวโปรโมชันเฉพาะระดับ — เช่น ให้คะแนนเพิ่มสองเท่าสำหรับการซื้อเฉพาะสมาชิก Gold และ Platinum
- ตั้งค่า **Cooldown Days** สำหรับแคมเปญดึงกลับลูกค้า เพื่อไม่ให้สมาชิกได้รับการส่งข้อความมากเกินไปหากพวกเขาทำการซื้อเล็กน้อยแล้วกลับไปไม่ใช้งานอีกครั้งในระยะเวลาสั้น ๆ
- รายการแคมเปญคือเครื่องมือที่ดีที่สุดของคุณในการติดตามโปรโมชันที่กำลังเปิดใช้งานอยู่ — ตรวจสอบรายการนี้ก่อนเปิดตัวข้อเสนอใหม่เพื่อให้มั่นใจว่าแคมเปญไม่ได้ถูกเปิดใช้งานโดยไม่ตั้งใจ
- จัดเก็บแคมเปญที่สิ้นสุดลงแทนที่จะลบเพื่อให้คุณมีบันทึกทางประวัติศาสตร์ของโปรโมชันที่คุณเคยดำเนินการและเวลาที่ดำเนินการ
