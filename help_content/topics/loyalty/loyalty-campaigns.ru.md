---
title: Кампании лояльности
---

Кампании лояльности позволяют запускать ограниченные по времени акции и автоматические награды, которые выходят за рамки обычных правил начисления баллов. Используйте их для проведения акций с двойным начислением баллов в выходные дни, награждения клиентов в день их рождения, возвращения неактивных покупателей и предоставления целевых бонусов определенным группам членов.

Каждая кампания определяет триггер или расписание, членов, к которым она применяется, и действия, которые необходимо выполнить. После активации кампании срабатывают автоматически — вы настраиваете их один раз, а Spwig обрабатывает остальное.

## Типы кампаний

| Тип | Когда срабатывает |
|------|---------------|
| **На основе триггера** | Когда происходит конкретное событие (например, оформлен заказ, обнаружен день рождения) |
| **Расписание** | По повторяющемуся графику (ежедневно, еженедельно, ежемесячно) |
| **Ручная** | Только тогда, когда вы явно запускаете её из административной панели |
| **Поведенческая** | Когда клиент соответствует определенному поведенческому паттерну (например, просматривает товары, но не покупает) |

## Создание кампании

Перейдите в **Promotions > Loyalty Campaigns** и нажмите **+ Add Loyalty Campaign**.

### Шаг 1: базовая информация

- **Name** — понятное и описательное имя, видимое только в административной панели (например, `Birthday Bonus — 200 Points`)
- **Slug** — автоматически генерируется из имени; используется внутри системы
- **Description** — необязательные примечания о цели кампании
- **Campaign Type** — выберите тип из таблицы выше

### Шаг 2: триггер или расписание

**Для кампаний на основе триггера**, установите **Trigger Event**, который запускает кампанию. Доступные триггеры включают:

| Триггер | Описание |
|---------|-------------|
| Order Placed | Срабатывает, когда член завершает заказ |
| First Purchase | Срабатывает при первом заказе члена |
| Customer Birthday | Срабатывает в день рождения члена |
| Membership Anniversary | Срабатывает ежегодно в день вступления члена |
| Cart Abandoned | Срабатывает, когда корзина оставлена без оплаты |
| Tier Promotion | Срабатывает, когда член переходит на более высокий уровень |
| Points Expiring Soon | Срабатывает, когда у члена баллы скоро истекут |
| Inactive 90 Days | Срабатывает, когда член не совершал покупок в течение 90 дней |
| Review Submitted | Срабатывает, когда член отправляет отзыв о товаре |
| Referral Converted | Срабатывает, когда рекомендованный клиент совершает покупку |

Вы можете добавить **Trigger Conditions** в виде объекта JSON, чтобы дополнительно фильтровать, когда срабатывает кампания. Например, чтобы запускать только для заказов свыше 100 долларов:

```json
{
  "min_order_amount": 100
}
```

**Для расписанных кампаний**, установите **Schedule Type** (Daily, Weekly, Monthly или Custom Cron) и настройте время в поле **Schedule Config**:

```json
{
  "hour": 9,
  "minute": 0
}
```

### Шаг 3: действия

Поле **Actions** определяет, что происходит, когда кампания срабатывает. Введите массив объектов действий в формате JSON. Наиболее распространенным действием является начисление бонусных баллов:

```json
[
  {
    "type": "award_points",
    "points": 200,
    "description": "Birthday bonus — thank you for being a member!"
  }
]
```

Другие доступные действия включают отправку электронной почты или начисление значка. Для получения полного списка обратитесь к документации компонента вашего провайдера.

### Шаг 4: целевая аудитория

Управляйте теми членами, к кому применяется кампания, с помощью полей целевой аудитории:

- **Target All Members** — по умолчанию отмечено; кампания применяется ко всем активным членам лояльности
- **Target Segment** — ограничьте кампанию членами определенного сегмента (см. [Segments](#managing-member-segments) ниже)
- **Target Tiers** — ограничьте кампанию членами определенных уровней лояльности

### Шаг 5: ограничения и охлаждение

- **Max Triggers Per Member** — сколько раз один и тот же член может получить выгоду от этой кампании. Установите значение `1` для одноразовых бонусов, например, бонуса в день рождения. Оставьте поле пустым для неограниченного количества.
- **Cooldown Days** — минимальное количество дней между запусками кампании для одного и того же члена. Например, установите значение `365`, чтобы предотвратить запуск кампании в день рождения более одного раза в год.

### Шаг 6: даты кампании

Установите **Start Date** и **End Date**, чтобы сделать кампанию ограниченной по времени. Оставьте оба поля пустыми для кампании, которая длится неограниченно долго.

Кампании могут находиться в одном из следующих статусов:

| Статус | Описание |
|--------|-------------|
| **Черновик** | Создана, но еще не активна; безопасно настраивать и тестировать |
| **Активна** | Запущена и сработает, когда будут выполнены условия |
| **Приостановлена** | Временно остановлена без потери настроек |
| **Завершена** | Прошла дату окончания; больше не срабатывает |
| **Архивирована** | Скрыта из списка активных, но сохранена для архива |

После заполнения всех полей нажмите **Сохранить**. Затем измените статус на **Активна**, чтобы запустить кампанию.

## Практические примеры

### Пример: удвоение баллов на выходных

**Сценарий:** Начислять 2x баллов на все покупки, сделанные в определенный выходной.

| Поле | Значение |
|-------|-------|
| Название | `Double Points Weekend — March` |
| Тип кампании | На основе триггера |
| Событие триггера | Заказ оформлен |
| Действия | `["{\"type\": \"award_points_multiplier\", \"multiplier\": 2.0}"]` |
| Дата начала | Вечер пятницы |
| Дата окончания | Полночь воскресенья |
| Целевая группа всех членов | Отмечено |

### Пример: бонус на день рождения

**Сценарий:** Начислять 200 бонусных баллов каждому члену программы лояльности в день его рождения.

| Поле | Значение |
|-------|-------|
| Название | `Birthday Bonus` |
| Тип кампании | На основе триггера |
| Событие триггера | День рождения клиента |
| Действия | `["{\"type\": \"award_points\", \"points\": 200, \"description\": \"Happy birthday from us!\"}"]` |
| Максимальное количество триггеров на члена | 1 |
| Срок ожидания | 365 |
| Целевая группа всех членов | Отмечено |

### Пример: кампания для возвращения клиентов

**Сценарий:** Начислять 100 бонусных баллов членам, которые не совершали покупок в течение 90 дней.

| Поле | Значение |
|-------|-------|
| Название | `90-Day Win-Back Bonus` |
| Тип кампании | На основе триггера |
| Событие триггера | Неактивен 90 дней |
| Действия | `["{\"type\": \"award_points\", \"points\": 100, \"description\": \"We miss you — here are some bonus points\"}"]` |
| Максимальное количество триггеров на члена | 1 |
| Срок ожидания | 180 |
| Целевая группа всех членов | Отмечено |

## Управление сегментами членов

Сегменты позволяют нацеливать кампании на определенные группы членов программы лояльности. Перейдите в **Promotions > Loyalty Segments**, чтобы управлять ими.

### Типы сегментов

| Тип | Описание |
|------|-------------|
| **На основе правил** | Членство определяется правилами (например, члены с более чем 1000 баллами) |
| **Динамический расчет** | Членство рассчитывается по запросу на основе критериев в реальном времени |
| **Ручное назначение** | Члены добавляются в сегмент вручную |

### Создание сегмента

1. Перейдите в **Promotions > Loyalty Segments** и нажмите **+ Add Loyalty Segment**
2. Заполните:
   - **Название** — описательное название (например, `High-Value Customers`, `Silver Tier Members`)
   - **Slug** — автоматически сгенерировано
   - **Тип критерия** — как определяется членство
   - **Конфигурация критерия** — объект JSON, определяющий правила членства
3. Нажмите **Сохранить**

#### Пример: сегмент для членов с более чем 500 баллами

```json
{
  "min_available_points": 500
}
```

#### Пример: сегмент только для членов Gold

```json
{
  "tier_slugs": ["gold"]
}
```

Столбец **Member Count** в списке сегментов показывает, сколько членов в настоящее время соответствуют критериям. Нажмите на сегмент и используйте действие **Refresh Member Count**, чтобы пересчитать его, если данные изменились.

## Отслеживание эффективности кампаний

### История выполнения кампаний

Перейдите в **Promotions > Campaign Executions**, чтобы увидеть запись каждого раза, когда кампания сработала для любого члена. Каждая запись о выполнении показывает, какая кампания сработала, для какого члена и результат.

### Просмотр охвата кампании

Откройте любую запись кампании, чтобы увидеть счетчик **Times Triggered** и время последнего срабатывания кампании. Это дает быстрый обзор того, сколько членов получили выгоду от кампании.

## Советы

Сохраните всю разметку Markdown, пути к изображениям, блоки кода и технические термины.

- Создайте кампании со статусом **Черновик**, чтобы вы могли проверить все настройки перед тем, как они запустятся
- Используйте **Максимальное количество триггеров на клиента** для всех кампаний с одноразовой премией (день рождения, первая покупка, регистрация), чтобы предотвратить получение бонуса клиентами более одного раза
- Сочетайте **Целевую группу** с кампанией на основе триггера, чтобы запускать промоакции, доступные только для определенных уровней — например, удвоение баллов за покупки только для клиентов Gold и Platinum
- Установите значение **Период охлаждения (дней)** для кампаний по восстановлению клиентов, чтобы клиенты не получали слишком много уведомлений, если они совершают небольшую покупку, а затем снова становятся неактивными в течение короткого времени
- Список кампаний — это ваш лучший инструмент для отслеживания текущих активных акций — проверьте его перед запуском новых предложений, чтобы убедиться, что кампании не накладываются друг на друга случайно
- Архивируйте завершенные кампании вместо их удаления, чтобы сохранить исторический учет запущенных вами акций и их даты
