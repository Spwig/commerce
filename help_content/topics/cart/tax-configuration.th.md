---
title: การตั้งค่าภาษี
---

ตั้งค่ากฎเกณฑ์ภาษีสำหรับร้านค้าของคุณเพื่อให้ภาษีที่ถูกต้องถูกนำไปใช้อัตโนมัติสำหรับคำสั่งซื้อตามตำแหน่งของลูกค้า คุณสามารถโหลดการตั้งค่าภาษีสำหรับภูมิภาคต่าง ๆ ได้ด้วยการคลิกเพียงครั้งเดียว หรือสร้างกฎเกณฑ์ที่กำหนดเองสำหรับประเทศ รัฐ เมือง หรือรหัสไปรษณีย์ใด ๆ

![Tax Dashboard](/static/core/admin/img/help/tax-configuration/tax-dashboard.webp)

## Tax Dashboard

ไปที่ **Orders > Shipments > Tax Rates** เพื่อเปิดแดชบอร์ดภาษี หน้าเว็บแสดง:

- **Statistics panel** — 4 บัตรแสดงผลรวมกฎ, กฎที่ใช้งานอยู่, ประเทศที่ครอบคลุม และประเภทภาษีที่ใช้งานอยู่
- **Filters** — ค้นหาตามชื่อ ประเทศ หรือรัฐ และกรองตามประเทศ ประเภทภาษี (Sales Tax, VAT, GST, Custom) หรือสถานะ (Active/Inactive)
- **Tax rule cards** — แต่ละบัตรแสดงธงประเทศ ชื่อกฎ ตำแหน่ง ร้อยละอัตราภาษี ป้ายประเภทภาษี ป้ายสถานะ ลำดับความสำคัญ และจำนวนการยกเว้นภาษี

## Loading Tax Presets

คลิก **Load Presets** เพื่อเปิดหน้าต่างการตั้งค่าภาษีล่วงหน้า การตั้งค่าล่วงหน้าเป็นชุดอัตราภาษีมาตรฐานสำหรับภูมิภาคต่าง ๆ ที่พร้อมสำหรับการโหลดเข้าสู่ร้านค้าของคุณด้วยการคลิกเพียงครั้งเดียว

![Load Presets](/static/core/admin/img/help/tax-configuration/tax-presets-modal.webp)

การตั้งค่าล่วงหน้าถูกจัดกลุ่มตามภูมิภาคของโลก:

| Region | Preset Groups |
|--------|--------------|
| **Africa** | Africa VAT (25 rates) |
| **Asia Pacific** | Asia-Pacific VAT/GST (24 rates), Central Asia VAT (6 rates) |
| **Europe** | EU VAT Rates, UK VAT, Other European VAT |
| **Latin America** | Latin America VAT |
| **Middle East** | Middle East VAT |
| **North America** | US State Sales Tax, Canadian GST/HST |
| **Oceania** | Oceania GST/VAT |

### How Presets Work

1. คลิก **Load** บนกลุ่มการตั้งค่าล่วงหน้าที่คุณต้องการ
2. ระบบจะสร้างกฎเกณฑ์ภาษีสำหรับประเทศหรือรัฐทั้งหมดในกลุ่มนั้น
3. กฎเกณฑ์ที่มีอยู่เดิมที่มีประเทศ รัฐ และประเภทภาษีเดียวกันจะถูกข้ามโดยอัตโนมัติเพื่อป้องกันการซ้ำกัน
4. หลังจากโหลด กฎเกณฑ์แต่ละข้อสามารถแก้ไขได้อย่างเต็มที่ — ปรับอัตราภาษี เพิ่มการยกเว้น หรือปิดใช้งานกฎเกณฑ์ที่คุณไม่จำเป็นต้องใช้

คุณสามารถโหลดกลุ่มการตั้งค่าล่วงหน้าหลายกลุ่มได้ ตัวอย่างเช่น โหลดทั้ง EU VAT และ UK VAT หากคุณขายสินค้าให้ลูกค้าทั่วทั้งยุโรป

## Creating Tax Rules Manually

คลิก **Add Tax Rate** เพื่อสร้างกฎเกณฑ์ภาษีที่กำหนดเอง ฟอร์มมีส่วนประกอบ 4 ส่วน:

![Tax Rate Form](/static/core/admin/img/help/tax-configuration/tax-rate-form.webp)

### Basic Information

| Field | Description |
|-------|-------------|
| **Name** | ชื่อแสดงของกฎเกณฑ์ (เช่น "California Sales Tax") |
| **Is Active** | ปิด/เปิดใช้งานกฎเกณฑ์ |
| **Tax Type** | Sales Tax, VAT, GST หรือ Custom Tax |
| **Rate (%)** | อัตราภาษีเป็นเปอร์เซ็นต์ (เช่น ใส่ 8.25 สำหรับ 8.25%) |
| **Priority** | จำนวนที่สูงกว่าจะมีลำดับความสำคัญมากกว่าเมื่อมีกฎเกณฑ์หลายข้อตรงกับตำแหน่งเดียวกัน |

### Geographic Scope

| Field | Description |
|-------|-------------|
| **Country** | รหัส ISO 3166-1 alpha-2 (เช่น US, GB, DE) |
| **State** | รัฐหรือจังหวัด (ปล่อยว่างเพื่อใช้กับประเทศทั้งหมด) |
| **City** | ชื่อเมือง (ไม่บังคับ สำหรับกฎเกณฑ์ภาษีระดับเมือง) |
| **Postal Codes** | รายการรหัสไปรษณีย์เฉพาะ (ไม่บังคับ สำหรับกฎเกณฑ์ภาษีระดับรหัสไปรษณีย์) |

กฎเกณฑ์จะถูกจับคู่จากที่เฉพาะเจาะจงที่สุดไปยังที่น้อยที่สุด กฎเกณฑ์สำหรับรหัสไปรษณีย์เฉพาะจะมีลำดับความสำคัญมากกว่ากฎเกณฑ์สำหรับรัฐเดียวกัน ซึ่งจะมีลำดับความสำคัญมากกว่ากฎเกณฑ์ทั่วประเทศ

### Application Rules

| Field | Description |
|-------|-------------|
| **Applies to Shipping** | เมื่อเปิดใช้งาน ภาษีนี้จะถูกนำไปใช้กับค่าจัดส่งด้วย |
| **Compound Tax** | เมื่อเปิดใช้งาน ภาษีนี้จะถูกคำนวณเพิ่มจากภาษีอื่น ๆ (จำนวนพื้นฐาน บวกภาษีที่ถูกนำไปใช้ก่อนหน้า) |

### Product Exemptions

| Field | Description |
|-------|-------------|
| **Exempt Product Types** | ประเภทสินค้าที่ได้รับการยกเว้นภาษีนี้ (เช่น ดิจิทัล, บริการ) |
| **Exempt Categories** | ประเภทสินค้าเฉพาะที่ได้รับการยกเว้นภาษีนี้ |

## Tax Types

| Type | Used For | Examples |
|------|----------|---------|
| **Sales Tax** | US, Canada | ภาษีขายระดับรัฐและจังหวัด |
| **VAT** | Europe, UK, much of Asia and Africa | ภาษีมูลค่าเพิ่ม |
| **GST** | Australia, New Zealand, India, Singapore | ภาษีสินค้าและบริการ |
| **Custom Tax** | Special cases | ค่าธรรมเนียมท้องถิ่น ภาษีสิ่งแวดล้อม ภาษีหรูหรา |

## How Tax Calculation Works

เมื่อลูกค้าไปถึงขั้นตอนชำระเงิน ระบบจะคำนวณภาษีอัตโนมัติโดยอิงจากที่อยู่จัดส่งของลูกค้า:

1. **Geographic matching** — ค้นหากฎเกณฑ์ที่ใช้งานอยู่ทั้งหมดที่ตรงกับประเทศของลูกค้า แล้วค่อยจำกัดตามรัฐ เมือง และรหัสไปรษณีย์
2. **Specificity scoring** — กฎเกณฑ์ที่เฉพาะเจาะจงมากกว่า (รหัสไปรษณีย์ > เมือง > รัฐ > ประเทศ) จะได้รับการจัดลำดับให้สูงกว่า
3. **Priority ordering** — ภายในระดับความเฉพาะเจาะจงเดียวกัน กฎเกณฑ์ที่มีลำดับความสำคัญสูงกว่าจะมีความสำคัญมากกว่า
4. **Product exemptions** — สินค้าที่ได้รับการยกเว้นจะถูกตัดออกจากการนำไปใช้ของกฎเกณฑ์ที่เกี่ยวข้อง
5. **Non-compound taxes** — ถูกคำนวณก่อนจากมูลค่าพื้นฐานของสินค้าแต่ละชิ้น
6. **Compound taxes** — ถูกคำนวณจากมูลค่าพื้นฐาน บวกกับภาษีที่ไม่รวมกันทั้งหมดที่ถูกนำไปใช้ก่อนหน้า
7. **Shipping tax** — หากกฎเกณฑ์มี "Applies to Shipping" เปิดใช้งาน ค่าจัดส่งจะถูกรวมเข้าไปในจำนวนที่ต้องเสียภาษี

การแยกประเภทภาษีจะถูกบันทึกไว้กับคำสั่งซื้อเพื่อให้คุณสามารถเห็นได้อย่างชัดเจนว่ากฎเกณฑ์ใดถูกนำไปใช้ และแต่ละกฎเกณฑ์มีส่วนร่วมเท่าไร

## Common Setups

### EU Store

1. คลิก **Load Presets** และโหลดกลุ่ม **EU VAT Rates**
2. นี่จะสร้างกฎเกณฑ์ภาษี VAT สำหรับประเทศสมาชิกทั้งหมดในสหภาพยุโรปโดยใช้อัตรามาตรฐานปัจจุบัน
3. โหลด **UK VAT** เพิ่มเติมหากคุณยังขายสินค้าให้กับสหราชอาณาจักรอีกด้วย

### US Store

1. คลิก **Load Presets** และโหลดกลุ่ม **US State Sales Tax**
2. นี่จะสร้างกฎเกณฑ์ภาษีขายสำหรับรัฐทั้งหมดในสหรัฐอเมริกาที่เก็บภาษีขาย
3. สำหรับภาษีระดับเมือง ให้เพิ่มกฎเกณฑ์ด้วยตนเองโดยกรอกชื่อเมืองและตั้งค่าลำดับความสำคัญให้สูงกว่า

### Multi-Region Store

1. โหลดกลุ่มการตั้งค่าล่วงหน้าหลายกลุ่มสำหรับแต่ละตลาดที่คุณขายสินค้า
2. ระบบจะนำไปใช้ภาษีที่ถูกต้องตามตำแหน่งของลูกค้าแต่ละคน
3. ปรับกฎเกณฑ์เฉพาะตามความต้องการทางธุรกิจของคุณ

## Tips

- **Start with presets** — โหลดกลุ่มการตั้งค่าล่วงหน้าสำหรับตลาดเป้าหมายของคุณ แล้วปรับอัตราภาษีเฉพาะแต่ละข้อแทนที่จะสร้างกฎเกณฑ์ทั้งหมดจากศูนย์
- **Use priority wisely** — ตั้งค่าลำดับความสำคัญให้สูงกว่าสำหรับกฎเกณฑ์ท้องถิ่นที่เฉพาะเจาะจงมากกว่าเพื่อให้สามารถทับซ้อนกฎเกณฑ์ภูมิภาคทั่วไปได้อย่างถูกต้อง
- **Check compound tax carefully** — ภาษีที่รวมกันนั้นพบได้น้อย แทบทุกเขตอำนาจศาลใช้ภาษีแบบง่าย (ไม่รวมกัน) โปรดเปิดใช้งานภาษีที่รวมกันเฉพาะเมื่อข้อบังคับท้องถิ่นของคุณกำหนดให้ต้องคำนวณภาษีจากภาษีอื่น ๆ อย่างชัดเจน
- **Keep rules active/inactive** — แทนที่จะลบกฎเกณฑ์ภาษีสำหรับการเปลี่ยนแปลงตามฤดูกาลหรือชั่วคราว ให้ปิดใช้งานกฎเกณฑ์และเปิดใช้งานใหม่เมื่อจำเป็น
- **Test before going live** — หลังจากตั้งค่ากฎเกณฑ์ภาษีของคุณแล้ว ให้สั่งซื้อสินค้าทดสอบจากที่อยู่ต่าง ๆ เพื่อตรวจสอบว่าภาษีที่ถูกต้องถูกนำไปใช้หรือไม่
