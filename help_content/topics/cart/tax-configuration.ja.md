---
title: 税の設定
---

ストアの税ルールを構成して、顧客の所在地に基づいて注文に正しい税が自動的に適用されるようにします。1クリックで地域のプリセットを読み込むか、任意の国、州、市、郵便番号に対してカスタムルールを作成できます。

![Tax Dashboard](/static/core/admin/img/help/tax-configuration/tax-dashboard.webp)

## 税ダッシュボード

**注文 > 輸送 > 税率**に移動して、税ダッシュボードを開きます。このページには以下が表示されます:

- **統計パネル** — 総ルール数、アクティブなルール数、カバーされている国、使用中の税種類を表示する4つのカード
- **フィルタ** — 名前、国、州で検索し、国、税種類（Sales Tax、VAT、GST、Custom）やステータス（Active/Inactive）でフィルタリングできます
- **税ルールカード** — 各カードには、国旗、ルール名、場所、税率パーセンテージ、税種類バッジ、ステータスバッジ、優先度、免除数が表示されます

## 税プリセットの読み込み

**プリセットを読み込む**をクリックして、プリセットモーダルを開きます。プリセットは、地域の標準税率のコレクションで、1クリックでストアに読み込む準備が整っています。

![Load Presets](/static/core/admin/img/help/tax-configuration/tax-presets-modal.webp)

プリセットは世界の地域ごとに整理されています:

| Region | Preset Groups |
|--------|--------------|
| **Africa** | Africa VAT (25 rates) |
| **Asia Pacific** | Asia-Pacific VAT/GST (24 rates), Central Asia VAT (6 rates) |
| **Europe** | EU VAT Rates, UK VAT, Other European VAT |
| **Latin America** | Latin America VAT |
| **Middle East** | Middle East VAT |
| **North America** | US State Sales Tax, Canadian GST/HST |
| **Oceania** | Oceania GST/VAT |

### プリセットの仕組み

1. 読み込みたいプリセットグループの**読み込み**をクリックします
2. システムは、そのグループ内のすべての国または州に対して税ルールを作成します
3. 同じ国、州、税種類を持つ既存のルールは自動的にスキップされ、重複を防ぎます
4. 読み込み後、各ルールは完全に編集可能になります — レートを調整し、免除を追加するか、必要ないルールを無効にできます

複数のプリセットグループを読み込むことができます。たとえば、ヨーロッパ全域に顧客がいる場合、EU VATとUK VATの両方を読み込むことができます。

## 手動で税ルールを作成する

**税率を追加**をクリックして、カスタムルールを作成します。フォームには4つのセクションがあります:

![Tax Rate Form](/static/core/admin/img/help/tax-configuration/tax-rate-form.webp)

### 基本情報

| Field | Description |
|-------|-------------|
| **Name** | ルールの表示名（例: "California Sales Tax"） |
| **Is Active** | ルールを有効または無効にする切り替えスイッチ |
| **Tax Type** | Sales Tax、VAT、GST、またはCustom Tax |
| **Rate (%)** | パーセンテージとしての税率（例: 8.25%を入力する場合は8.25を入力） |
| **Priority** | 複数のルールが同じ場所に一致する場合、より高い数字が優先されます |

### 地理的範囲

| Field | Description |
|-------|-------------|
| **Country** | ISO 3166-1 alpha-2コード（例: US、GB、DE） |
| **State** | 州または県（空白にすると、国全体に適用されます） |
| **City** | 市名（オプション、市レベルの税ルール用） |
| **Postal Codes** | 特定の郵便番号のリスト（オプション、郵便番号レベルの税ルール用） |

ルールは最も具体的な順に一致します。特定の郵便番号のルールは、同じ州のルールよりも優先され、州のルールは国全体のルールよりも優先されます。

### 適用ルール

| Field | Description |
|-------|-------------|
| **Applies to Shipping** | にチェックを入れると、この税は配送費用にも適用されます |
| **Compound Tax** | にチェックを入れると、この税は他の税の上に計算されます（基本金額にすでに適用された税を含む） |

### 商品免除

| Field | Description |
|-------|-------------|
| **Exempt Product Types** | この税から免除される商品タイプ（例: デジタル、サービス） |
| **Exempt Categories** | この税から免除される特定の商品カテゴリ |

## 税種類

| Type | Used For | Examples |
|------|----------|---------|
| **Sales Tax** | US、Canada | 州および省の消費税 |
| **VAT** | Europe、UK、アジアおよびアフリカの大部分 | 增値税 |
| **GST** | Australia、New Zealand、India、Singapore | 商品およびサービス税 |
| **Custom Tax** | 特殊なケース | 地方追加税、環境税、高級品税 |

## 税計算の仕組み

顧客がチェックアウト画面に到達すると、システムはその配送先住所に基づいて税を自動的に計算します:

1. **地理的マッチング** — 顧客の国に一致するすべてのアクティブなルールを検索し、州、市、郵便番号で絞り込みます
2. **具体的性スコア** — より具体的なルール（郵便番号 > 市 > 州 > 国）は順位が高くなります
3. **優先度順序** — 同じ具体的性レベル内では、優先度の高いルールが優先されます
4. **商品免除** — 免除された商品は、適用可能な各ルールから除外されます
5. **非複合税** — 各商品の基本価格に対して最初に計算されます
6. **複合税** — 基本価格にすでに適用されたすべての非複合税を加えて計算されます
7. **配送税** — ルールに"Applies to Shipping"が有効になっている場合、配送費用は課税対象額に含まれます

税の明細は注文に保存されるため、どのルールが適用され、それぞれがどのくらい寄与したかを確認できます。

## 一般的な設定

### EUストア

1. **プリセットを読み込む**をクリックし、**EU VAT Rates**グループを読み込みます
2. これにより、EU加盟国の現在の標準税率に基づいてVATルールがすべて作成されます
3. UKにも販売している場合、**UK VAT**をオプションで読み込むことができます

### USストア

1. **プリセットを読み込む**をクリックし、**US State Sales Tax**グループを読み込みます
2. これにより、州の消費税を徴収するすべてのUS州の消費税ルールが作成されます
3. 市レベルの税がある場合は、市名を入力し、優先度を高く設定して手動でルールを追加します

### 複数地域ストア

1. 各販売市場ごとに複数のプリセットグループを読み込みます
2. システムは、顧客の所在地に基づいて正しい税を適用します
3. ご自分のビジネス要件に応じて、必要に応じて個々のルールを調整します

## ヒント

- **プリセットから始める** — ターゲット市場のプリセットグループを読み込み、すべてのルールをから作成するのではなく、個々の税率をカスタマイズします。
- **優先度を賢く使用する** — より具体的な地域のルールに高い優先度値を設定して、広範な地域のルールを正しく上書きできるようにします。
- **複合税を慎重に確認する** — 複合税は珍しいです。ほとんどの管轄区域は単純（非複合）税を使用しています。税の上での税の計算が地域の規制によって明示的に要求されている場合にのみ、複合税を有効にしてください。
- **アクティブ/非アクティブなルールを維持する** — 季節的または一時的な変更のために税ルールを削除するのではなく、非アクティブに切り替え、必要に応じて再アクティブしてください。
- **本番環境に公開する前にテストする** — 税ルールを設定した後、さまざまな住所からテスト注文を置き、正しい税が適用されていることを確認してください。
