---
title: 批量导入优惠券代码
---

优惠券导入向导允许您通过上传CSV或XLSX电子表格一次性创建数百个优惠券代码。当您有预先打印的代码、第三方系统的忠诚度计划代码，或需要启动大型活动而无需手动添加每个代码时，这非常理想。

![带有导入按钮的优惠券列表](/static/core/admin/img/help/voucher-import/voucher-list-import-button.webp)

## 开始导入

导航到 **营销 > 优惠券**，然后点击页面右上角的 **导入** 按钮。这将打开三步导入向导。

## 第1步：上传文件并设置批量设置

![导入上传表单](/static/core/admin/img/help/voucher-import/import-upload.webp)

第一页有两个部分：文件上传和批量折扣设置。

### 准备文件

上传一个最大为5 MB的`.csv`或`.xlsx`文件。文件必须有标题行作为第一行。最低要求是包含优惠券代码的单列——其他所有列都是可选的。

导入器会自动识别常见的列名。如果您的文件使用了以下任何名称，Spwig将在下一页自动预选正确的映射，无需额外点击：

| 您的列名 | 映射到 |
|---------|-------|
| `code`, `voucher_code`, `coupon_code`, `promo_code` | 优惠券代码 |
| `name`, `title`, `campaign` | 内部名称 |
| `description`, `details`, `note` | 面向客户的描述 |
| `external_id`, `member_id`, `reference` | 外部ID |

**提示：** 首先下载XLSX模板（参见下方的[将优惠券导出为模板](#exporting-vouchers-as-a-template)）——它使用导入器预期的确切列名，因此列映射是自动的。

### 文件限制

- 最大文件大小：**5 MB**
- 每次导入的最大行数：**5,000 个代码**

### 设置批量折扣设置

批量中的每个优惠券将共享您在此页面上配置的相同折扣设置。

# 第2步：映射列并查看

填写字段的方式与创建单个优惠券时相同：

**折扣部分**

| 字段 | 描述 |
|-------|-------------|
| **折扣类型** | 百分比、固定金额或免运费 |
| **折扣值** | 要扣除的百分比（0–100）或固定金额 |
| **最大折扣金额** | 百分比折扣的上限（例如，将20%折扣上限设为50美元） |
| **应用范围** | 整个购物车、特定产品或特定分类 |

**有效期部分**

| 字段 | 描述 |
|-------|-------------|
| **开始日期** | 代码生效的日期（如果留空，默认为现在） |
| **结束日期** | 代码过期的日期（留空表示无过期时间） |
| **有效天数** | 结束日期的替代选项 —— 代码在创建后此天数内有效 |

**使用限制部分**

| 字段 | 描述 |
|-------|-------------|
| **总使用次数上限** | 所有客户可以兑换的总次数（留空表示无限制） |
| **每个客户的最大使用次数** | 一个客户可以使用此批次中任何代码的次数 |
| **最低订单金额** | 代码生效前所需的最低购物车总额 |

**限制条件**

可选中任意组合：
- **不能应用于促销商品** —— 防止代码与已打折的商品叠加使用
- **不能与其他优惠券组合使用** —— 防止客户在同一订单中使用两个代码
- **不能与促销商品组合使用** —— 类似于上述，但针对促销价格商品
- **仅限新客户** —— 将代码限制为没有完成订单历史的客户
- **立即生效** —— 勾选此选项，使代码在导入时立即生效

当您对设置满意时，点击 **继续预览**。

## 第2步：映射列并查看

![列映射和预览页面](/static/core/admin/img/help/voucher-import/import-preview.webp)

预览页面在顶部显示四个摘要计数器：

- **已解析的行数** — 在您的文件中找到的总数据行数

- **将导入** — 将要创建的新代码

- **重复项** — 已经存在于您的商品目录中的代码

- **将跳过（无效）** — 因验证错误被拒绝的行（代码为空、代码过长等）

### 列映射

通过 **列映射** 表，您可以告诉 Spwig 您的文件中的哪一列对应每个优惠券字段。Spwig 会自动检测常见的标题名称（请参阅上表），但您可以通过每行的下拉菜单更改任何映射。

只有 **优惠券代码** 列是必需的。其他字段 — **内部名称**、**面向客户的描述** 和 **外部 ID** — 是可选的。如果您跳过这些字段，Spwig 会使用合理的默认值（内部名称默认为 "Imported voucher {code}"）。

### 重复代码策略

如果您的文件中的任何代码已经在您的商品目录中存在，您必须选择如何处理它们：

| 策略 | 会发生什么 |

|----------|-------------|

| **跳过重复项** | 现有的代码将保持原样。只有新代码会被创建。 |

| **覆盖设置** | 现有的代码将使用此批次的折扣设置进行更新。它们的代码、使用次数和创建日期将被保留。 |

| **导入失败** | 如果发现任何一个重复项，整个导入将被取消。当您需要确保没有任何现有代码受到影响时使用此选项。 |

发现的任何重复代码都会列在一个可展开的面板中，以便您在决定前进行查看。

### 数据预览表

页面底部显示了您文件中的前 20 行，以便您在提交前确认列映射是否正确。与现有代码匹配的行会被突出显示。

当一切看起来正确时，点击 **导入 N 张优惠券** 以提交批次。

## 第 3 步：查看结果

![导入结果页面](/static/core/admin/img/help/voucher-import/import-result.webp)

导入完成后，您将看到一个显示以下内容的摘要：

- **已导入** — 成功创建的代码

- **已跳过** — 未创建的代码（重复或无效行）

- **已处理行数** — 从您的文件中评估的总行数

- **失败** — 遇到意外错误的行

点击 **查看已导入的优惠券**，打开仅包含此批次代码的优惠券列表，便于检查结果或批量激活新代码。

如果发现任何问题——例如应用了错误的折扣类型——您可以在重新导入时使用 **覆盖设置** 策略来修正批次，而无需删除并重新创建代码。

点击 **导入另一批次** 开始新的上传，或点击 **返回优惠券列表** 返回您的完整目录。

## 作为模板导出优惠券

优惠券列表支持 XLSX 导出操作，生成的文件列顺序与导入器预期的完全一致。这是获取正确格式模板的最简单方法：

1. 导航到 **营销 > 优惠券**

2. 选择您要导出的优惠券（或全选）

3. 从 **操作** 下拉菜单中选择 **将所选优惠券导出为 XLSX**

4. 点击 **执行**

下载的文件包含导入器理解的所有 21 列，包括导入向导中为批次级别设置的字段（折扣类型、日期、使用限制等）。您可以将此文件作为参考，或使用 **覆盖设置** 策略通过编辑 → 重新导入的循环来处理现有代码。

## 小贴士

保留所有 Markdown 格式、图片路径、代码块和专业术语。

- 首先下载一个 XLSX 导出文件作为模板 —— 列名已预先格式化，因此在预览页面上无需任何调整即可自动映射。
- 在导入数百个代码之前，先运行一个小的测试批次（5–10 个代码），以验证您的列映射和批次设置是否正确。
- 如果代码将随时间分发，请使用 **Days valid** 而不是固定的 **End date** —— 这样每个代码的过期时间将从导入时间开始计算，而不是一个固定的日历日期。
- 如果您从第三方忠诚度系统接收代码，请将供应商的会员或客户参考编号映射到 **External ID** 列，以便之后可以对冲销进行核对。
- 在进行大规模导入后，点击结果页面上的 **View imported vouchers**，将列表筛选为仅新批次 —— 然后您可以批量编辑、激活或停用它们。
- 使用 **Fail** 重复策略进行导入失败时，您的目录将保持不变，因此可以安全地修复文件并根据需要多次重试。
