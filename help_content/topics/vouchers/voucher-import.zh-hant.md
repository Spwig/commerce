---
title: 批量導入優惠券代碼
---

優惠券導入向導讓您可以通過上傳CSV或XLSX電子表格一次創建數百個優惠券代碼。當您有預印的代碼、第三方系統的忠誠度計劃代碼，或僅需發起一個大型活動而無需手動添加每個代碼時，這非常理想。

![優惠券列表與導入按鈕](/static/core/admin/img/help/voucher-import/voucher-list-import-button.webp)

## 開始導入

導航至 **行銷 > 優惠券**，並點擊頁面上右側的 **導入** 按鈕。這將打開三步導入向導。

## 第1步：上傳文件並設置批量折扣設置

![導入上傳表單](/static/core/admin/img/help/voucher-import/import-upload.webp)

第一頁包含兩個部分：文件上傳和批量折扣設置。

### 準備您的文件

上傳一個 `.csv` 或 `.xlsx` 文件，最大為 5 MB。文件必須有標題行作為第一行。最低要求是包含優惠券代碼的單個列——其他所有列都是可選的。

導入器會自動識別常見的列名。如果您的文件使用了以下任何名稱，Spwig 將在下一個頁面自動預選正確的映射，無需額外點擊：

| 您的列名 | 對應到 |
|---------|-------|
| `code`, `voucher_code`, `coupon_code`, `promo_code` | 優惠券代碼 |
| `name`, `title`, `campaign` | 內部名稱 |
| `description`, `details`, `note` | 面向客戶的描述 |
| `external_id`, `member_id`, `reference` | 外部ID |

**提示：** 首先下載 XLSX 模板（請參閱下方的 [將優惠券導出為模板](#exporting-vouchers-as-a-template)）——它使用導入器預期的精確列名，因此列映射是自動的。

### 文件限制

- 最大文件大小：**5 MB**
- 每次導入的最大行數：**5,000 個代碼**

### 設置批量折扣設置

批次中的每個優惠券將共享您在此頁配置的相同折扣設置。

# 填寫欄位方式與建立單一優惠券相同：

**折扣部分**

| 欄位 | 說明 |
|-------|-------------|
| **折扣類型** | 百分比、固定金額或免運費 |
| **折扣值** | 要扣除的百分比（0–100）或固定金額 |
| **最大折扣金額** | 百分比折扣的上限（例如將20%折扣上限設定為50美元） |
| **應用範圍** | 全部購物車、特定商品或特定類別 |

**有效性部分**

| 欄位 | 說明 |
|-------|-------------|
| **開始日期** | 輸入碼生效的日期（若留空則預設為現在） |
| **結束日期** | 輸入碼過期的日期（留空表示無期限） |
| **有效天數** | 結束日期的替代選項 — 輸入碼在創建後這許多天過期 |

**使用限制部分**

| 欄位 | 說明 |
|-------|-------------|
| **總使用次數上限** | 所有客戶可兌換的總次數（留空表示無限制） |
| **每位客戶最大使用次數** | 一位客戶可以使用此批次中任何碼的次數 |
| **最低訂單金額** | 使用此碼前所需的最低購物車總金額 |

**限制**

勾選任何組合：
- **不可應用於促銷商品** — 防止碼與已經折扣的商品疊加使用
- **不可與其他優惠券組合使用** — 防止客戶在同一筆訂單中使用兩個碼
- **不可與促銷商品組合使用** — 與上述類似，但針對促銷價格商品
- **僅限新客戶** — 將碼限制為沒有先前完成訂單的客戶
- **立即啟用** — 勾選此選項，使碼在導入後立即生效

當您對設定感到滿意時，請點擊 **繼續預覽**。

## 第2步：映射欄位並審查

![欄位映射和預覽頁面](/static/core/admin/img/help/voucher-import/import-preview.webp)

預覽頁面顯示頂部四個摘要計數器：

- **已解析的行數** — 在您的檔案中找到的總資料行數

- **將導入** — 將要建立的新代碼

- **重複項目** — 已經存在於您的目錄中的代碼

- **將跳過（無效）** — 因驗證錯誤而被拒絕的行（代碼為空、代碼過長等）

### 欄位對應

「**欄位對應**」表格讓您可以告訴 Spwig 您的檔案中哪一欄對應到每個優惠券欄位。Spwig 會自動偵測常見的標題名稱（請參閱上表），但您可以使用每一行的下拉選單來更改任何對應關係。

只有 **優惠券代碼** 欄是必填的。其他欄位 — **內部名稱**、**面向客戶的描述** 和 **外部 ID** — 是可選的。如果您跳過這些欄位，Spwig 會使用合理的預設值（內部名稱預設為 "Imported voucher {code}"）。

### 重複代碼策略

如果您的檔案中存在任何已經在您的目錄中的代碼，您必須選擇如何處理它們：

| 策略 | 會發生什麼 |

|----------|-------------|

| **跳過重複項目** | 現有的代碼會保持原樣。只有新代碼會被建立。 |

| **覆寫設定** | 現有的代碼會使用此批次的折扣設定進行更新。其代碼、使用次數和建立日期會被保留。 |

| **導入失敗** | 如果發現任何一個重複項目，整個導入將被取消。當您需要確保沒有現有的代碼受到影響時，請使用此選項。 |

發現的任何重複代碼都會列在一個可展開的面板中，以便您在做出決定前進行審查。

### 資料預覽表格

頁面底部會顯示您檔案中的前 20 行，以便您在提交前確認欄位對應是否正確。與現有代碼匹配的行會被標記。

當一切看起來正確時，點擊 **導入 N 張優惠券** 來提交批次。

## 第 3 步：查看結果

![導入結果頁面](/static/core/admin/img/help/voucher-import/import-result.webp)

導入完成後，您將看到一個顯示以下內容的摘要：

- **已導入** — 成功創建的代碼

- **跳過** — 未創建的代碼（重複或無效行）

- **已處理的行數** — 從您的文件中評估的總行數

- **失敗** — 遇到意外錯誤的行

點擊 **查看已導入的優惠券**，可打開僅過濾出此批次代碼的優惠券列表，方便您檢查結果或批量啟用新代碼。

如果發現任何問題——例如套用了錯誤的折扣類型——您可以在重新導入時使用 **覆寫設定** 策略，修正批次而無需刪除並重新創建代碼。

點擊 **導入另一個批次** 以開始新的上傳，或點擊 **返回優惠券列表** 以返回完整的目錄。

## 將優惠券導出為模板

優惠券列表支援 XLSX 導出功能，產生的文件與導入器預期的欄位順序完全一致。這是獲得正確格式模板的最簡單方法：

1. 點擊導航到 **行銷 > 優惠券**

2. 選擇您要導出的優惠券（或全選）

3. 從 **操作** 下拉選單中選擇 **將選中的優惠券導出為 XLSX**

4. 點擊 **執行**

下載的文件包含導入器理解的所有 21 個欄位，包括在導入向導中為批次級的欄位（折扣類型、日期、使用限制等）。您可以將此文件作為參考，或使用 **覆寫設定** 策略，通過編輯 → 重新導入的週期來處理現有的代碼。

## 小提示

保留所有 Markdown 格式、圖片路徑、程式碼塊和技術術語。

- 首先下載 XLSX 格式的匯出檔案作為範本 — 欄位名稱已經預先格式化，因此自動對應會直接識別，無需在預覽頁面進行任何調整。
- 在匯入數百個代碼之前，先執行一小批 5–10 個代碼的測試，以驗證您的欄位對應和批次設定是否正確。
- 如果代碼會在一段時間內分階段發放，請使用 **有效天數** 而非固定的 **結束日期** — 這樣每個代碼的過期時間將從其匯入時間開始計算，而不是單一的日曆日期。
- 如果您從第三方忠誠度系統接收代碼，請將供應商的會員或客戶參考編號對應到 **外部 ID** 欄位，這樣之後就可以對應紅利記錄。
- 在進行大量匯入後，點擊結果頁面上的 **查看已匯入的優惠券**，將列表篩選為僅顯示新批次 — 然後您可以批量編輯、啟用或停用它們。
- 使用 **失敗** 的重複策略進行匯入失敗時，您的目錄將保持不變，因此可以安全地修正檔案並重試所需的次數。
