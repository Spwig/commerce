title: クーポンの例
---

このガイドでは、最も一般的なクーポンタイプのフィールドごとの具体的な例を提供します。各例は、**マーケティング > クーポン** → **+ クーポンの追加**でクーポンを作成するときに、入力すべき内容を正確に示しています。

![クーポンカード](/static/core/admin/img/help/voucher-examples/voucher-card.webp)

## 例1: パーセンテージ割引（最大割引額あり）

**シナリオ:** カート全体に対して20％の割引を提供しますが、最大50ドルまでに設定して、高価な注文が利益を保証するようにします。有効期限は設定しません。

| フィールド | 値 |
|-------|-------|
| コード | `SAVE20` |
| 名前 | 20％割引 — 最大50ドル |
| 割引タイプ | パーセンテージ |
| 割引値 | 20 |
| 最大割引額 | 50 |
| 適用範囲 | カート全体 |
| 最大使用回数（全体） | *(空 — 無制限)* |
| 最大使用回数（顧客ごと） | 1 |
| 最低注文額 | *(空 — 最低額なし)* |

**最大割引額の仕組み:** 200ドルの注文では割引額は40ドルになります。300ドルの注文では60ドルの割引になりますが、最大額の制限により50ドルに抑えられます。500ドルの注文では割引額は50ドルのままです。これは、実際の割引額を予測可能にしながら、魅力的なプロモーションを実施できるようにします。

## 例2: 最低額以上の固定金額割引

**シナリオ:** 75ドル以上の注文に対して10ドルの割引を提供し、より大きな注文を促進します。

| フィールド | 値 |
|-------|-------|
| コード | `TAKE10` |
| 名前 | 75ドル以上の注文で10ドル割引 |
| 割引タイプ | 固定金額 |
| 割引値 | 10 |
| 適用範囲 | カート全体 |
| 最低注文額 | 75 |
| 最大使用回数（顧客ごと） | 0 *(無制限)* |
| 終了日 | *(空 — 有効期限なし)* |

> **注意:** 最低注文額を設定することで、利益率を守ることができます。これを設定しないと、顧客が12ドルの注文にこのコードを適用して、利益を失う可能性があります。固定金額のクーポンは、常に合理的な最低額と組み合わせて使用してください。

## 例3: 無料配送

**シナリオ:** 最低購入額を設定せずに、すべての注文に対して無料配送を提供します。

| フィールド | 値 |
|-------|-------|
| コード | `FREESHIP` |
| 名前 | 無料配送 |
| 割引タイプ | 無料配送 |
| 適用範囲 | カート全体 |
| 最大使用回数（全体） | *(空 — 無制限)* |
| 最大使用回数（顧客ごと） | 1 |
| 最低注文額 | *(空 — 最低額なし)* |

> **注意:** **無料配送**の割引タイプを選択すると、注文から配送料が自動的に削除されます。これは、顧客がどの配送方法を選択しても、最もクリーンなアプローチです。

## 例4: 初回顧客向けのウェルカムコード

**シナリオ:** 新規顧客に初回注文に対して15％の割引を提供し、コンバージョンを促進します。

| フィールド | 値 |
|-------|-------|
| コード | `WELCOME15` |
| 名前 | ウェルカム — 初回注文15％割引 |
| 割引タイプ | パーセンテージ |
| 割引値 | 15 |
| 適用範囲 | カート全体 |
| 最大使用回数（顧客ごと） | 1 |
| 初回顧客のみ | 有効 |

システムは、顧客が以前の注文を完了しているかどうかを確認して、初回顧客のステータスを検証します。注文履歴がある顧客がこのコードを適用しようとした場合、チェックアウト時に明確なエラーメッセージが表示されます。

## 例5: 商品固有のクーポン

**シナリオ:** 選んだ商品に対して5ドルの割引を提供します — 例えば、在庫の少ない商品を販売促進するために。

| フィールド | 値 |
|-------|-------|
| コード | `PICK5` |
| 名前 | 選んだ商品で5ドル割引 |
| 割引タイプ | 固定金額 |
| 割引値 | 5 |
| 適用範囲 | 特定の商品 |
| 対象商品 | *(対象商品を選択)* |
| 最大使用回数（顧客ごと） | 1 |

> **注意:** 特定の商品を対象にしたい場合は、**商品範囲**を使用してください。特定の部門のすべての商品を対象にしたい場合は、次の例の**カテゴリ範囲**を使用してください。商品範囲は正確な制御を可能にし、カタログが頻繁に変更される場合、カテゴリ範囲の方が管理が簡単です。

## 例6: カテゴリクーポン

**シナリオ:** 電子機器カテゴリのすべての商品に対して25％の割引キャンペーンを実施します。

| フィールド | 値 |
|-------|-------|
| コード | `ELEC25` |
| 名前 | 電子機器25％割引 |
| 割引タイプ | パーセンテージ |
| 割引値 | 25 |
| 適用範囲 | 特定のカテゴリ |
| 対象カテゴリ | 電子機器 |
| 最大使用回数（全体） | *(空 — 無制限)* |
| 最大使用回数（顧客ごと） | 1 |


カテゴリにスコープが設定されている場合、割引はカート内の対象商品のみに適用されます。

電子機器以外の商品は、通常価格で請求されます。

## 割引タイプの比較

| タイプ | 仕組み | 最適な用途 | 例 |
|------|-------------|----------|---------|
| **パーセンテージ** | 対象となる合計金額のパーセンテージを控除 | 注文規模に応じて拡大するスケーリング割引 | カート全体から20％割引 |
| **固定金額** | 固定されたドル単位の金額を控除 | 簡単で予測可能なプロモーション | 75ドル以上の注文に10ドル割引 |
| **送料無料** | 注文の送料を削除 | 支払い時にカート放棄を減らす | 送料無料、最低購入額なし |

## スコープの比較

| スコープ | 仕組み | 最適な用途 |
|-------|-------------|----------|
| **カート全体** | 割引は注文全体の合計に適用されます | 店舗全体のプロモーションやウェルカムコード |
| **特定の商品** | カート内の選択された商品のみに割引が適用されます | 特定の在庫を処分したり、特典商品を強調したりする場合 |
| **特定のカテゴリ** | 選択されたカテゴリの商品のみに割引が適用されます | 部門全体のセールや季節限定のプロモーション |

## ヒント

- **覚えやすいコードを使用する** — `SUMMER20` は `COUPONX1600406498` よりも効果的です。バッチキャンペーン用に自動生成されたコードを保存してください。
- **配布する前にテストする** — ボーナスコードを使ってテスト注文を行い、それが正しく適用され、すべての制限を遵守しているかを確認してください。
- **使用状況を監視する** — 各ボーナスコードカードの「Redemptions（利用回数）」カウントを確認し、キャンペーンの実績をリアルタイムで追跡してください。
- **アノンサーションバーと組み合わせる** — サイト全体のアノンサーションでボーナスコードを宣伝し、顧客が買い物を開始する前に目に留まるようにしてください。
