---
title: バッチインポートでクーポンコードを追加する
---

クーポンコードのインポートウィザードを使用すると、CSVまたはXLSX形式のスプレッドシートをアップロードすることで、数百のクーポンコードを一度に作成できます。これは、事前に印刷されたコード、サードパーティのシステムからのロイヤルティプログラムコード、または手動で各コードを追加せずに大規模なキャンペーンを開始する必要がある場合に最適です。

![クーポンリストにインポートボタン](/static/core/admin/img/help/voucher-import/voucher-list-import-button.webp)

## インポートの開始

**マーケティング > クーポン**に移動し、ページの右上部にある**インポート**ボタンをクリックします。これにより、3ステップのインポートウィザードが開きます。

## ステップ1: ファイルをアップロードし、バッチ設定を指定する

![インポートアップロードフォーム](/static/core/admin/img/help/voucher-import/import-upload.webp)

最初のページには2つの部分があります: ファイルのアップロードとバッチ割引設定。

### ファイルの準備

5 MBまでの`.csv`または`.xlsx`ファイルをアップロードしてください。ファイルには、最初の行がヘッダーロウである必要があります。最低限の要件は、クーポンコードを含む単一の列です。他の列はすべて任意です。

インポーターは、一般的な列名を自動的に認識します。ファイルが以下のいずれかの名前を使用している場合、Spwigは次のページで正しいマッピングを事前に選択し、追加のクリックなしで処理します:

| あなたの列名 | マッピング先 |
|-----------------|---------|
| `code`, `voucher_code`, `coupon_code`, `promo_code` | クーポンコード |
| `name`, `title`, `campaign` | 内部名 |
| `description`, `details`, `note` | 顧客向けの説明 |
| `external_id`, `member_id`, `reference` | 外部ID |

**ヒント:** まずXLSXテンプレートをダウンロードしてください（下記の[テンプレートとしてクーポンをエクスポートする](#exporting-vouchers-as-a-template)を参照）—これはインポーターが期待する正確な列名を使用しており、列のマッピングが自動的になります。

### ファイルの制限

- 最大ファイルサイズ: **5 MB**
- 1回のインポートあたりの最大行数: **5,000 コード**

### バッチ割引設定の指定

バッチ内のすべてのクーポンは、このページで設定した同じ割引設定を共有します。


# チケットの作成時に通常入力するフィールドを埋めてください:

**割引セクション**

| フィールド | 説明 |
|-------|-------------|
| **割引タイプ** | パーセンテージ、固定金額、または無料配送 |
| **割引値** | 減額するパーセンテージ（0–100）または固定金額 |
| **最大割引額** | パーセンテージ割引のオプション上限（例：20％割引を50ドルに上限設定） |
| **適用範囲** | カート全体、特定の商品、または特定のカテゴリ |

**有効期間セクション**

| フィールド | 説明 |
|-------|-------------|
| **開始日** | コードが有効になる日時（空白の場合は現在時刻がデフォルト） |
| **終了日** | コードが期限切れになる日時（空白の場合は期限なし） |
| **有効日数** | 終了日代替 — コードが作成からこの日数後に期限切れになります |

**使用制限セクション**

| フィールド | 説明 |
|-------|-------------|
| **最大使用回数（全体）** | すべての顧客にわたって許可される合計使用回数（空白 = 無制限） |
| **最大使用回数（1顧客あたり）** | 1つの顧客がこのバッチのコードを使用できる回数 |
| **最低注文金額** | コードが適用される前のカートの最低合計金額 |

**制限**

以下の任意の組み合わせをチェックしてください:
- **セール商品には適用不可** — 既に割引されている商品とコードが重複して適用されないようにします
- **他のチケットと併用不可** — 顧客が同じ注文で2つのコードを使用できないようにします
- **セール価格商品と併用不可** — 上記と同様ですが、セール価格商品を対象にしています
- **初回購入者限定** — 以前の注文が完了していない顧客にのみコードを適用します
- **即時有効** — チェックしたままにすると、コードがインポートされた瞬間に有効になります

設定に満足したら、**プレビューへ進む**をクリックしてください。

## ステップ2: 列をマッピングし、確認する

![列のマッピングとプレビュー画面](/static/core/admin/img/help/voucher-import/import-preview.webp)

プレビュー画面には、上部に4つのサマリーカウンターが表示されます:

- **Rows parsed** — ファイルに含まれるデータ行の合計数

- **Will import** — 新しく作成されるコード

- **Duplicates** — カタログに既に存在するコード

- **Will skip (invalid)** — 検証エラー（コードが空、コードが長すぎるなど）により拒否された行



### Column mapping



**Column mapping** テーブルを使用して、Spwig にファイル内のどの列が各割引券のフィールドに対応するかを指定できます。Spwig は一般的なヘッダー名を自動検出します（上記の表を参照）が、各行のドロップダウンを使用して任意のマッピングを変更できます。



**Voucher code** 列のみが必須です。他のフィールド — **Internal name**、**Customer-facing description**、および **External ID** — はオプションです。これらをスキップした場合、Spwig は合理的なデフォルト値を使用します（内部名はデフォルトで "Imported voucher {code}" になります）。



### Duplicate code strategy



ファイル内のコードがカタログに既に存在する場合、どのように処理するかを選択する必要があります:



| Strategy | What happens |

|----------|-------------|

| **Skip duplicates** | 既存のコードはそのままであり、新しいコードのみが作成されます。 |

| **Overwrite settings** | 既存のコードはこのバッチの割引設定で更新されます。コード、使用回数、作成日は保持されます。 |

| **Fail the import** | 1つの重複が見つかった場合、全体のインポートがキャンセルされます。既存のコードが影響を受けないことを保証する必要がある場合に使用してください。 |



見つかった重複コードは、拡張可能なパネルに一覧表示されるため、決定する前に確認できます。



### Data preview table



ページの下部には、ファイルの最初の20行が表示され、コミットする前に列のマッピングが正しいかを確認できます。既存のコードと一致する行はハイライトされます。



すべてが正しいように見える場合は、**Import N vouchers** をクリックしてバッチをコミットしてください。



## Step 3: Review the result



![Import result page](/static/core/admin/img/help/voucher-import/import-result.webp)



インポートが完了すると、以下が表示されます:



- **インポート済み** — 成功裏に作成されたコード

- **スキップ** — 作成されなかったコード（重複または無効な行）

- **処理された行数** — ファイルから評価された総行数

- **失敗** — 予期せぬエラーに遭遇した行

**インポートされた割引券を表示**をクリックして、このバッチからコードだけにフィルタされた割引券リストを開きます。これにより、結果の確認や新しいコードのバッチアクティベーションが簡単になります。

何かが間違っているように見える場合（例えば、誤った割引タイプが適用されている場合など）は、再インポート時に**設定を上書き**戦略を使用して、コードを削除し直す必要なくバッチを修正できます。

**別のバッチをインポート**をクリックして、新しいアップロードを開始するか、**割引券リストに戻る**をクリックして、フルカタログに戻ります。

## 割引券をテンプレートとしてエクスポートする

割引券リストは、インポーターが期待するのと同じ列の順序でファイルを作成するXLSXエクスポートアクションをサポートしています。正しいフォーマットのテンプレートを取得する最も簡単な方法です:

1. **マーケティング > 割引券**に移動します

2. エクスポートしたい割引券を選択する（またはすべて選択）

3. **Action**ドロップダウンから**選択した割引券をXLSXとしてエクスポート**を選択します

4. **Go**をクリックします

ダウンロードされたファイルには、インポーターが理解するすべての21列が含まれており、インポートウィザードでバッチレベルのフィールド（割引タイプ、日付、使用制限など）も含まれています。このファイルを参照として使用するか、**設定を上書き**戦略を使用して、既存のコードを編集 → 再インポートのサイクルを通じてラウンドトリップできます。

## ヒント

すべてのMarkdownの書式、画像のパス、コードブロック、および技術用語を保持してください。

- まず、列名が事前にフォーマットされているため、プレビュー画面で自動マッピングが行われるよう、XLSXエクスポートをダウンロードしてテンプレートとして使用してください。
- 数百個をインポートする前に、5〜10個のコードを小さなテストバッチとして実行し、列のマッピングとバッチ設定が正しいか確認してください。
- コードが時間にわたって配布される場合、**有効期間（日数）**を使用し、**終了日（固定）**ではなくしてください — これにより、各コードの有効期限はインポートされた時刻から計算され、単一のカレンダー日ではなくなります。
- 第三者ロイヤルティシステムからコードを受け取る場合、サプライヤーの会員または顧客参照を**外部ID**列にマッピングしてください — これにより、後でリデメーションを照合できます。
- 大規模なインポート後、結果ページで **インポートされたバウチャーを表示** をクリックして、リストを新しいバッチのみにフィルタリングできます — その後、グループとして一括編集、アクティベート、または非アクティベートできます。
- **失敗**の重複戦略を使用した失敗したインポートはカタログに変更をもたらさないため、ファイルを修正して必要に応じて何度でも再試行することが安全です。
