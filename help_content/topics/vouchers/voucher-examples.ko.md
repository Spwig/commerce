이 가이드는 가장 일반적인 쿠폰 유형에 대해 필드별로 구체적인 예시를 제공합니다. 각 예시는 **마케팅 > 쿠폰** → **+ 쿠폰 추가**에서 쿠폰을 생성할 때 입력해야 할 내용을 정확히 보여줍니다.

![쿠폰 카드](/static/core/admin/img/help/voucher-examples/voucher-card.webp)

## 예시 1: 할인 한도가 있는 비율 할인

**시나리오:** 전체 주문에 대해 20% 할인을 제공하지만, 할인 금액은 최대 $50으로 제한하여 고가의 주문이 수익성 있는 상태를 유지하도록 합니다. 만료일은 없습니다.

| 필드 | 값 |
|-------|-------|
| 코드 | `SAVE20` |
| 이름 | 20% 할인 — 최대 $50 |
| 할인 유형 | 비율 |
| 할인 금액 | 20 |
| 최대 할인 금액 | 50 |
| 적용 범위 | 전체 주문 |
| 총 최대 사용 횟수 | *(빈칸 — 무제한)* |
| 고객당 최대 사용 횟수 | 1 |
| 최소 주문 금액 | *(빈칸 — 최소 없음)* |

**할인 한도 작동 방식:** $200 주문의 경우 할인 금액은 $40입니다. $300 주문의 경우 $60이 되지만, 한도로 인해 $50으로 제한됩니다. $500 주문의 경우 할인 금액은 여전히 $50입니다. 이는 대규모 할인을 제공하면서도 실제 할인 금액을 예측 가능한 상태로 유지할 수 있는 방법입니다.

## 예시 2: 최소 주문 금액 이상 시 고정 금액 할인

**시나리오:** $75 이상의 주문에 대해 $10 할인을 제공하여 더 큰 주문을 유도합니다.

| 필드 | 값 |
|-------|-------|
| 코드 | `TAKE10` |
| 이름 | $75 이상 주문에 $10 할인 |
| 할인 유형 | 고정 금액 |
| 할인 금액 | 10 |
| 적용 범위 | 전체 주문 |
| 최소 주문 금액 | 75 |
| 고객당 최대 사용 횟수 | 0 *(무제한)* |
| 종료일 | *(빈칸 — 만료 없음)* |

> **참고:** 최소 주문 금액을 설정하면 이익 마진을 보호할 수 있습니다. 이 설정이 없으면 고객이 $12 주문에 이 코드를 사용하여 수익을 모두 잃을 수 있습니다. 항상 고정 금액 쿠폰은 합리적인 최소 주문 금액과 함께 사용해야 합니다.

## 예시 3: 무료 배송

**시나리오:** 최소 구매 금액 없이 모든 주문에 무료 배송을 제공합니다.

| 필드 | 값 |
|-------|-------|
| 코드 | `FREESHIP` |
| 이름 | 무료 배송 |
| 할인 유형 | 무료 배송 |
| 적용 범위 | 전체 주문 |
| 총 최대 사용 횟수 | *(빈칸 — 무제한)* |
| 고객당 최대 사용 횟수 | 1 |
| 최소 주문 금액 | *(빈칸 — 최소 없음)* |

> **참고:** **무료 배송** 할인 유형을 선택하면 주문에서 배송비가 자동으로 제거됩니다. 이는 고객이 어떤 배송 방식을 선택하더라도 작동합니다.

## 예시 4: 신규 고객 환영 쿠폰 코드

**시나리오:** 신규 고객에게 첫 주문에 대해 15% 할인을 제공하여 전환을 유도합니다.

| 필드 | 값 |
|-------|-------|
| 코드 | `WELCOME15` |
| 이름 | 환영 — 첫 주문 15% 할인 |
| 할인 유형 | 비율 |
| 할인 금액 | 15 |
| 적용 범위 | 전체 주문 |
| 고객당 최대 사용 횟수 | 1 |
| 신규 고객만 적용 | 체크됨 |

시스템은 고객이 이전에 완료된 주문이 있는지 확인하여 첫 방문 여부를 검증합니다. 기존 주문 내역이 있는 고객이 이 코드를 적용하려 할 경우, 결제 시 명확한 오류 메시지가 표시됩니다.

## 예시 5: 특정 제품 쿠폰

**시나리오:** 선택된 제품에 대해 $5 할인을 제공합니다 — 예를 들어, 판매가 느린 재고를 처리하기 위해.

| 필드 | 값 |
|-------|-------|
| 코드 | `PICK5` |
| 이름 | 선택된 항목에 $5 할인 |
| 할인 유형 | 고정 금액 |
| 할인 금액 | 5 |
| 적용 범위 | 특정 제품 |
| 해당 제품 | *(대상 제품 선택)* |
| 고객당 최대 사용 횟수 | 1 |

> **참고:** 특정 제품 범위를 선택하면 개별 SKU를 할인할 수 있습니다. 특정 카테고리 범위(다음 예시)를 선택하면 부서의 모든 항목을 할인할 수 있습니다. 제품 범위는 정확한 제어를 제공하지만, 카탈로그가 자주 변경될 경우 카테고리 범위가 유지하기 더 쉬운 선택입니다.

## 예시 6: 카테고리 쿠폰

**시나리오:** 전자 제품 카테고리의 모든 항목에 대해 25% 할인을 제공합니다.

| 필드 | 값 |
|-------|-------|
| 코드 | `ELEC25` |
| 이름 | 전자 제품 25% 할인 |
| 할인 유형 | 비율 |
| 할인 금액 | 25 |
| 적용 범위 | 특정 카테고리 |
| 해당 카테고리 | 전자 제품 |
| 총 최대 사용 횟수 | *(빈칸 — 무제한)* |
| 고객당 최대 사용 횟수 | 1 |


카테고리에 적용된 경우, 할인은 장바구니에 있는 해당 상품에만 적용됩니다.

전자제품이 아닌 상품은 정가로 요금이 부과됩니다.

## 할인 유형 비교

| 유형 | 작동 방식 | 최적 사용처 | 예시 |
|------|-------------|----------|---------|
| **백분율** | 해당 총액의 백분율을 차감 | 주문 규모가 커질수록 증가하는 할인 | 전체 장바구니에 20% 할인 |
| **고정 금액** | 고정된 달러 금액을 차감 | 간단하고 예측 가능한 프로모션 | $75 이상 주문 시 $10 할인 |
| **무료 배송** | 주문의 배송비를 제거 | 결제 단계에서 장바구니 포기율을 줄이기 위해 | 최소 금액 없이 무료 배송 |

## 범위 비교

| 범위 | 작동 방식 | 최적 사용처 |
|-------|-------------|----------|
| **전체 장바구니** | 할인이 전체 주문 총액에 적용됨 | 전체 스토어 프로모션 및 환영 코드 |
| **특정 상품** | 할인이 장바구니에 있는 선택된 상품에만 적용됨 | 특정 재고를 정리하거나 특별한 할인을 제공할 때 |
| **특정 카테고리** | 할인이 선택된 카테고리에 있는 상품에만 적용됨 | 부서별 할인 및 계절별 프로모션 |

## 팁

- **기억하기 쉬운 코드 사용** — `SUMMER20`은 `COUPONX1600406498`보다 더 잘 작동합니다. 대량 캠페인에 자동 생성된 코드를 사용하세요.
- **배포 전 테스트** — 바우처 코드를 사용하여 테스트 주문을 생성하여 할인이 올바르게 적용되고 모든 제한을 준수하는지 확인하세요.
- **사용량 모니터링** — 각 바우처 카드의 사용 횟수를 확인하여 캠페인 성과를 실시간으로 추적하세요.
- **공지 표시와 결합** — 사이트 전체 공지에 바우처 코드를 홍보하여 고객이 쇼핑을 시작하기 전에 보게 하세요.
