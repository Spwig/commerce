---
title: Ограничения купонов
---

Ограничения купонов управляют тем, кто может использовать купон, когда и как часто. Настройте эти параметры при создании или редактировании купона в **Маркетинг > Купоны**.

![Правила ограничений](/static/core/admin/img/help/voucher-restrictions/restriction-rules.webp)

## Ограничения использования

Установите глобальные и ограничения на использование по каждому клиенту в разделе **Ограничения использования** формы купона.

- **Максимальное количество использований в целом** — Максимальное количество раз, которое этот купон может быть использован всеми клиентами. Оставьте поле пустым для неограниченного использования.
- **Максимальное количество использований на клиента** — Сколько раз один клиент может использовать этот купон. Установите значение 1 для большинства кампаний.

| Модель | Максимальное общее количество | На клиента | Сценарий использования |
|---------|-----------|--------------|----------|
| Ограниченная кампания | 100 | 1 | "Первые 100 клиентов" — дефицит |
| Неограниченный общий код | (пустое) | 1 | Продвижение, продолжающееся в течение времени |
| Неограниченное многократное использование | (пустое) | (пустое) | Внутренний/скидка для сотрудников |
| Уникальные коды для однократного использования | 1 | 1 | Коды для кампаний, созданные в большом количестве |

## Минимальная сумма заказа

Поле **Минимальная сумма заказа** защищает вашу маржу, требуя определенной суммы в корзине, прежде чем купон будет применен. Например, "Скидка 10 долларов на заказы свыше 50 долларов" гарантирует, что вы никогда не снизите цену небольшого заказа до неприбыльного уровня.

| Скидка | Рекомендуемый минимум | Соотношение |
|----------|-------------------|-------|
| 5 долларов скидки | 30+ | ~6:1 |
| 10 долларов скидки | 50+ | ~5:1 |
| 20 долларов скидки | 100+ | ~5:1 |
| 15% скидки | 40+ | Зависит от каталога |

## Ограничение скидки (максимальная сумма скидки)

Поле **Максимальная сумма скидки** в разделе **Настройка скидки** ограничивает, насколько может быть снижена сумма купона в процентах. Это применяется только к купонам в процентах и предотвращает чрезмерные скидки на заказы с высокой стоимостью.

Пример: "Скидка 20%, максимальная сумма скидки 50 долларов"
- Корзина на 200 долларов = скидка 40 долларов (20%)
- Корзина на 300 долларов = скидка 50 долларов (ограничено)
- Корзина на 1000 долларов = скидка 50 долларов (ограничено)

Добавьте ограничение скидки на любой купон в процентах, который вы делитесь публично.

## Правила комбинирования

Раздел **Ограничения и правила** (нажмите, чтобы раскрыть) содержит флажки, которые управляют тем, как купоны взаимодействуют с другими скидками.

| Настройка | Что это делает | Когда включать |
|---------|--------------|----------------|
| **Исключить товары со скидкой** | Купон пропускает товары, которые уже находятся на распродаже | Для большинства кампаний — защищает маржу распродаж |
| **Нельзя комбинировать с другими купонами** | Только один купон на заказ | По умолчанию для большинства купонов |
| **Нельзя комбинировать с товарами со скидкой** | Блокирует купон, если в корзине есть ЛЮБОЙ товар со скидкой | Для строгих кампаний, где купон заменяет цену распродажи |
| **Только для новых клиентов** | Только клиенты с нулевыми предыдущими заказами | Кампании привлечения/получения новых клиентов |

## Ограничения клиентов

Для простой настройки целевой аудитории установите флажок **Только для новых клиентов** в разделе **Ограничения и правила**.

Для продвинутой настройки целевой аудитории используйте таблицу **Ограничения купонов** в нижней части формы. Нажмите **+ Добавить еще одно ограничение купона**, чтобы добавить строки. Каждое ограничение имеет три поля:

- **Тип** — Категория ограничения (список)
- **Значение** — Соответствующее значение (разделенное запятыми или JSON)
- **Включить** — Отмечено = клиент должен соответствовать; не отмечено = клиент должен НЕ соответствовать

| Тип | Значение | Включить | Эффект |
|------|-------|-----------|--------|
| user_email_domain | @company.com | Да | Только сотрудники компании могут использовать его |
| shipping_country | US,CA | Да | Только клиенты из США и Канады |
| shipping_country | RU | Нет | Все, КРОМЕ России |
| day_of_week | monday,tuesday | Да | Только действует в понедельник и вторник |
| payment_method | stripe | Да | Только для платежей через Stripe |

Соедините несколько строк для многоуровневых ограничений. Все включенные ограничения должны совпадать, и ни одно исключительное ограничение не должно совпадать, чтобы купон применялся.

## Стратегии истечения срока

Управляйте сроком действия купона с помощью полей даты и срока действия.

- **Дата окончания** — Жесткая дата окончания (например, 31 декабря 2026 года).

Купон перестанет работать в полночь.
- **Количество дней действия** — Срок действия, рассчитываемый с момента создания или первого использования купона.

Переопределяет дату окончания, если задано.


Полезно для приветственных кодов: "действителен 30 дней с момента получения".

| Стратегия | Дата окончания | Дней действия | Сценарий использования |
|----------|----------|------------|----------|
| Жесткий срок окончания | Установлено | (пусто) | Сезонные кампании, события |
| Скользящее окно | (пусто) | 30 | Приветственные коды, купоны с скидками |
| Без срока окончания | (пусто) | (пусто) | Постоянные коды, скидки для сотрудников |

## Предотвращение злоупотреблений

Следуйте этой проверке-списку, чтобы сохранить безопасность ваших купонов:

- Всегда устанавливайте **Максимальное количество использований на клиента** в 1, если нет специальной причины не делать этого.
- Установите **Минимальную сумму заказа** для всех купонов с фиксированной суммой скидки.
- Добавьте **Максимальную сумму скидки** для публичных купонов с процентной скидкой.
- Используйте сложные для угадывания коды для купонов с высокой стоимостью — избегайте очевидных кодов, таких как "DISCOUNT50".
- Отслеживайте аналитику использования на каждой карточке купона в панели управления.
- Отключите купон немедленно, если вы заметите необычные паттерны использования.
- Для кампаний с высокой стоимостью используйте уникальные коды, сгенерированные в пакетном режиме, вместо одного общего кода.

## Советы

- Начните с ограничений и снимите их, если уровень использования слишком низкий — легче ослабить правила, чем ужесточить их после того, как коды уже в ходу.
- Тестируйте каждый купон с реальным оформлением заказа перед тем, как распределить его среди клиентов.
- Регулярно проверяйте панель аналитики купонов, чтобы выявлять проблемы на ранних этапах.
- Комбинируйте несколько ограничений для многоуровневой защиты — например, лимит на клиента + минимальная сумма заказа + ограничение скидки + исключение товаров со скидкой.
