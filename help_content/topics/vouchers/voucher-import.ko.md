---
title: 대량으로 바우처 코드 가져오기
---

바우처 가져오기 마법사는 CSV 또는 XLSX 스프레드시트를 업로드하여 수백 개의 바우처 코드를 한 번에 생성할 수 있습니다. 이는 미리 인쇄된 코드가 있는 경우, 제3자 시스템에서 제공하는 충성도 프로그램 코드가 있는 경우, 또는 단순히 대규모 캠페인을 시작해야 하는 경우에 이상적입니다.

![바우처 목록과 가져오기 버튼](/static/core/admin/img/help/voucher-import/voucher-list-import-button.webp)

## 가져오기 시작

**마케팅 > 바우처**로 이동하고 페이지 상단 오른쪽 영역의 **가져오기** 버튼을 클릭합니다. 이는 세 단계의 가져오기 마법사를 열게 됩니다.

## 단계 1: 파일 업로드 및 배치 설정 지정

![가져오기 업로드 폼](/static/core/admin/img/help/voucher-import/import-upload.webp)

첫 번째 페이지에는 두 가지 부분이 있습니다: 파일 업로드 및 배치 할인 설정.

### 파일 준비

5MB 이하의 `.csv` 또는 `.xlsx` 파일을 업로드합니다. 파일은 첫 번째 행이 헤더 행이어야 합니다. 최소 요구 사항은 바우처 코드를 포함하는 단일 열이며, 다른 열은 선택 사항입니다.

가져오기 도구는 일반적인 열 이름을 자동으로 인식합니다. 파일에 아래 이름 중 하나를 사용하는 경우 Spwig은 다음 페이지에서 추가 클릭 없이 올바른 매핑을 사전에 선택합니다:

| 당신의 열 이름 | 매핑 대상 |
|-----------------|---------|
| `code`, `voucher_code`, `coupon_code`, `promo_code` | 바우처 코드 |
| `name`, `title`, `campaign` | 내부 이름 |
| `description`, `details`, `note` | 고객 대면 설명 |
| `external_id`, `member_id`, `reference` | 외부 ID |

**팁:** 먼저 XLSX 템플릿을 다운로드하세요(아래 [바우처를 템플릿으로 내보내기](#exporting-vouchers-as-a-template) 참조) - 이 템플릿은 가져오기 도구가 예상하는 정확한 열 이름을 사용하므로 열 매핑이 자동으로 이루어집니다.

### 파일 제한

- 최대 파일 크기: **5 MB**
- 한 번에 가져올 수 있는 최대 행 수: **5,000 개의 코드**

### 배치 할인 설정 지정

배치 내의 모든 바우처는 이 페이지에서 구성한 동일한 할인 설정을 공유합니다.


단일 쿠폰을 생성할 때와 같이 필드를 입력합니다:

**할인 섹션**

| 필드 | 설명 |
|-------|-------------|
| **할인 유형** | 비율, 고정 금액, 또는 무료 배송 |
| **할인 금액** | 차감할 비율(0–100) 또는 고정 금액 |
| **최대 할인 금액** | 비율 할인의 선택적 상한(예: 20% 할인을 $50으로 제한) |
| **적용 범위** | 전체 장바구니, 특정 상품, 또는 특정 카테고리 |

**유효성 섹션**

| 필드 | 설명 |
|-------|-------------|
| **시작 날짜** | 코드가 활성화되는 시점(빈 경우 기본값은 현재 시간) |
| **종료 날짜** | 코드가 만료되는 시점(빈 경우 만료 없음) |
| **유효 기간(일)** | 종료 날짜 대신 사용 — 생성 후 이만큼의 일 수 만큼 유효 |

**사용 제한 섹션**

| 필드 | 설명 |
|-------|-------------|
| **총 최대 사용 횟수** | 모든 고객을 통해 허용되는 총 사용 횟수(빈 경우 무제한) |
| **고객당 최대 사용 횟수** | 한 고객이 이 배치의 코드를 사용할 수 있는 횟수 |
| **최소 주문 금액** | 코드가 적용되기 전에 필요한 최소 장바구니 금액 |

**제한 사항**

다음 중 하나 이상을 선택할 수 있습니다:
- **할인 상품에 적용 불가** — 이미 할인된 상품과 중복 적용을 방지합니다.
- **다른 쿠폰과 중복 사용 불가** — 고객이 동일한 주문에 두 개의 코드를 사용하지 못하도록 합니다.
- **할인 상품과 중복 사용 불가** — 위와 유사하지만 할인 가격 상품에 적용됩니다.
- **처음 방문 고객만 해당** — 이전에 완료된 주문이 없는 고객에게만 코드를 적용합니다.
- **즉시 활성화** — 체크박스를 선택하여 코드가 임포트되는 즉시 활성화됩니다.

설정에 만족하면 **미리보기로 이동**을 클릭합니다.

## 단계 2: 열 매핑 및 검토

![열 매핑 및 미리보기 페이지](/static/core/admin/img/help/voucher-import/import-preview.webp)

미리보기 페이지는 상단에 네 개의 요약 카운터를 표시합니다:

- **Rows parsed** — 파일에 있는 전체 데이터 행 수

- **Will import** — 새로 생성될 코드

- **Duplicates** — 카탈로그에 이미 있는 코드

- **Will skip (invalid)** — 검증 오류(코드가 비어 있음, 코드가 너무 길음 등)로 인해 거부된 행



### Column mapping



**Column mapping** 표는 Spwig에게 파일의 각 열이 바우처 필드 중 어떤 것에 해당하는지 알려줍니다. Spwig는 일반적인 헤더 이름을 자동으로 감지합니다(위 표를 참조하십시오), 하지만 각 행의 드롭다운을 사용하여 매핑을 변경할 수 있습니다.



**Voucher code** 열만 필수입니다. 다른 필드인 **Internal name**, **Customer-facing description**, **External ID**는 선택 사항입니다. 이를 건너뛰면 Spwig는 합리적인 기본값을 사용합니다(내부 이름은 "Imported voucher {code}"로 기본값이 설정됩니다).



### Duplicate code strategy



파일에 있는 코드 중 카탈로그에 이미 있는 코드가 있다면, 어떻게 처리할지 선택해야 합니다:



| Strategy | What happens |

|----------|-------------|

| **Skip duplicates** | 기존 코드는 그대로 유지됩니다. 새로운 코드만 생성됩니다. |

| **Overwrite settings** | 기존 코드는 이 배치의 할인 설정으로 업데이트됩니다. 코드, 사용 횟수, 생성 날짜는 유지됩니다. |

| **Fail the import** | 중복된 코드가 하나라도 발견되면 전체 임포트가 취소됩니다. 기존 코드가 영향을 받지 않도록 보장해야 할 때 이 옵션을 사용합니다. |



파일에 있는 중복된 코드는 확장 가능한 패널에 나열되어 있어 결정하기 전에 확인할 수 있습니다.



### Data preview table



페이지 하단에는 파일의 처음 20개 행이 표시되어, 컬럼 매핑이 올바른지 확인할 수 있습니다. 기존 코드와 일치하는 행은 강조 표시됩니다.



모든 것이 올바르게 보인다면, **Import N vouchers**를 클릭하여 배치를 확정합니다.



## Step 3: Review the result



![Import result page](/static/core/admin/img/help/voucher-import/import-result.webp)



임포트가 완료되면, 다음 요약 정보를 볼 수 있습니다:

- **등록됨** — 성공적으로 생성된 코드

- **건너뜀** — 생성되지 않은 코드(중복 또는 유효하지 않은 행)

- **처리된 행** — 파일에서 평가된 총 행 수

- **실패** — 예상치 못한 오류가 발생한 행

**등록된 쿠폰 보기**를 클릭하여 이 배치에서 생성된 코드만 포함한 쿠폰 목록을 열 수 있습니다. 이렇게 하면 결과를 쉽게 확인하거나 새 코드를 일괄적으로 활성화할 수 있습니다.

어떤 문제가 있는 경우 — 예를 들어 잘못된 할인 유형이 적용된 경우 — 기존 코드를 삭제하고 다시 생성할 필요 없이 **덮어쓰기 설정** 전략을 사용하여 재등록을 통해 배치를 수정할 수 있습니다.

**다른 배치 등록**을 클릭하여 새 업로드를 시작하거나 **쿠폰 목록으로 돌아가기**를 클릭하여 전체 쿠폰 목록으로 돌아갈 수 있습니다.

## 쿠폰을 템플릿으로 내보내기

쿠폰 목록은 임포터가 예상하는 정확한 열 순서로 파일을 생성하는 XLSX 내보내기 기능을 지원합니다. 이는 올바르게 포맷된 템플릿을 얻는 가장 쉬운 방법입니다:

1. **마케팅 > 쿠폰**으로 이동합니다

2. 내보내고자 하는 쿠폰을 선택하거나(또는 전체 선택)

3. **Action** 드롭다운에서 **선택한 쿠폰을 XLSX로 내보내기**를 선택합니다

4. **Go**를 클릭합니다

다운로드한 파일에는 임포터가 이해하는 21개의 모든 열이 포함되어 있으며, 임포트 마법사에서 배치 수준으로 처리되는 필드(할인 유형, 날짜, 사용 제한 등)도 포함되어 있습니다. 이 파일을 참조로 사용하거나 **덮어쓰기 설정** 전략을 사용하여 기존 코드를 편집 → 재등록 사이클을 통해 다시 처리할 수 있습니다.

## 팁

모든 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어를 유지합니다.

- 먼저 XLSX 형식의 내보내기를 다운로드하여 템플릿으로 사용하세요 — 열 이름은 사전에 포맷되어 있어 미리보기 페이지에서 자동 매핑이 조정 없이 이루어집니다.
- 수백 개의 코드를 가져오기 전에 5~10개의 작은 테스트 배치를 실행하여 열 매핑과 배치 설정이 올바른지 확인하세요.
- 코드가 시간에 따라 분배될 경우 **유효 기간** 대신 고정된 **종료 날짜**를 사용하지 마세요 — 각 코드의 만료는 가져온 시점부터 시작되며 단일 달력 날짜가 아닙니다.
- 제3자 충성도 시스템에서 코드를 받는 경우, 공급업체의 회원 또는 고객 참조를 **외부 ID** 열에 매핑하여 나중에 사용 내역을 대조할 수 있도록 하세요.
- 대량의 가져오기 후 결과 페이지에서 **가져온 쿠폰 보기**를 클릭하여 새 배치만 필터링할 수 있습니다 — 이후 그룹으로 일괄 편집, 활성화 또는 비활성화할 수 있습니다.
- 실패한 가져오기(**Fail** 중복 전략 사용 시)는 카탈로그를 변경하지 않으므로 파일을 수정하고 필요한 만큼 다시 시도하는 것이 안전합니다.
