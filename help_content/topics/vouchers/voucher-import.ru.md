---
title: Массовое импорт купонов
---

Мастер импорта купонов позволяет создать сотни кодов купонов за раз, загружая файл CSV или XLSX. Это идеально, если у вас уже есть напечатанные коды, коды программы лояльности из сторонней системы или просто нужно запустить масштабную кампанию, не добавляя каждый код вручную.

![Список купонов с кнопкой импорта](/static/core/admin/img/help/voucher-import/voucher-list-import-button.webp)

## Начало импорта

Перейдите в **Маркетинг > Купоны** и нажмите кнопку **Импорт** в верхнем правом углу страницы. Это открывает мастер импорта из трёх шагов.

## Шаг 1: Загрузите файл и настройте параметры пакета

![Форма загрузки импорта](/static/core/admin/img/help/voucher-import/import-upload.webp)

Первая страница имеет две части: загрузка файла и настройки скидки пакета.

### Подготовка файла

Загрузите файл `.csv` или `.xlsx` до 5 МБ. Файл должен иметь заголовок в первой строке. Минимальное требование — один столбец, содержащий коды купонов — все остальные столбцы необязательны.

Импорт распознаёт распространённые названия столбцов автоматически. Если ваш файл использует любые из названий ниже, Spwig предварительно выберет правильное сопоставление на следующей странице без дополнительных кликов:

| Название вашего столбца | Сопоставляется с |
|------------------------|------------------|
| `code`, `voucher_code`, `coupon_code`, `promo_code` | Код купона |
| `name`, `title`, `campaign` | Внутреннее имя |
| `description`, `details`, `note` | Описание для клиентов |
| `external_id`, `member_id`, `reference` | Внешний ID |

**Совет:** Сначала скачайте шаблон XLSX (см. [Экспорт купонов как шаблона](#exporting-vouchers-as-a-template) ниже) — он использует точные названия столбцов, которые ожидает импорт, поэтому сопоставление столбцов будет автоматическим.

### Ограничения файла

- Максимальный размер файла: **5 МБ**
- Максимальное количество строк на импорт: **5 000 кодов**

### Настройка параметров скидки пакета

Каждый купон в пакете будет делиться одинаковыми настройками скидки, которые вы настроите на этой странице. Заполните поля так, как вы бы делали при создании одного купона:

**Раздел скидки**

| Поле | Описание |
|------|----------|
| **Тип скидки** | Процент, Фиксированная сумма или Бесплатная доставка |
| **Значение скидки** | Процент (0–100) или фиксированная сумма, которую нужно вычесть |
| **Максимальная сумма скидки** | Необязательный лимит для скидок в процентах (например, ограничить скидку 20% до 50 долларов) |
| **Область применения** | Весь заказ, Конкретные товары или Конкретные категории |

**Раздел действительности**

| Поле | Описание |
|------|----------|
| **Дата начала** | Когда коды становятся активными (по умолчанию сейчас, если оставить пустым) |
| **Дата окончания** | Когда коды истекают (оставьте пустым, чтобы не истекали) |
| **Количество дней действия** | Альтернатива дате окончания — коды истекают через это количество дней после создания |

**Раздел ограничений использования**

| Поле | Описание |
|------|----------|
| **Максимальное количество использований в целом** | Общее количество использования по всем клиентам (пустое = неограниченно) |
| **Максимальное количество использований на клиента** | Сколько раз один клиент может использовать любой код из этого пакета |
| **Минимальная сумма заказа** | Минимальная сумма заказа, необходимая для применения кода |

**Ограничения**

Проверьте любую комбинацию:
- **Не применяется к товарам со скидкой** — предотвращает накопление кода с уже скиданными товарами
- **Не может сочетаться с другими купонами** — предотвращает использование клиентами двух кодов в одном заказе
- **Не может сочетаться с товарами со скидкой** — аналогично выше, но направлено на товары со скидкой
- **Только для новых клиентов** — ограничивает код клиентам без предыдущих завершённых заказов
- **Активировать сразу** — оставьте отмеченным, чтобы сделать коды активными сразу после импорта

Когда вы удовлетворены настройками, нажмите **Продолжить к предварительному просмотру**.

## Шаг 2: Сопоставление столбцов и проверка

![Страница сопоставления столбцов и предварительного просмотра](/static/core/admin/img/help/voucher-import/import-preview.webp)

Страница предварительного просмотра показывает четыре сводных счётчика вверху:

- **Обработанные строки** — общее количество строк данных, найденных в вашем файле

- **Будут импортированы** — новые коды, которые будут созданы

- **Дубликаты** — коды, которые уже существуют в вашем каталоге

- **Будут пропущены (недопустимые)** — строки, отклоненные из-за ошибок проверки (пустой код, код слишком длинный и т.д.)

### Сопоставление столбцов

Таблица **Сопоставление столбцов** позволяет указать Spwig, какой столбец в вашем файле соответствует каждому полю купона. Spwig автоматически определяет распространенные названия заголовков (см. таблицу выше), но вы можете изменить любое сопоставление, используя выпадающий список на каждой строке.

Только столбец **Код купона** является обязательным. Остальные поля — **Внутреннее имя**, **Описание для клиента** и **Внешний идентификатор** — являются необязательными. Если вы их пропустите, Spwig будет использовать разумные значения по умолчанию (внутреннее имя по умолчанию будет "Импортированный купон {code}").

### Стратегия обработки дубликатов кодов

Если какие-либо коды в вашем файле уже существуют в вашем каталоге, вы должны выбрать, как их обрабатывать:

| Стратегия | Что происходит |

|----------|-------------|

| **Пропустить дубликаты** | Существующие коды остаются такими, какими они есть. Создаются только новые коды. |

| **Перезаписать настройки** | Существующие коды обновляются настройками скидок этой партии. Их коды, счетчики использования и даты создания сохраняются. |

| **Отказаться от импорта** | Если найден хотя бы один дубликат, весь импорт отменяется. Используйте это, когда вам нужно гарантировать, что существующие коды не будут затронуты. |

Найденные дубликаты кодов перечислены в раскрывающемся панели, чтобы вы могли их просмотреть перед принятием решения.

### Предварительный просмотр таблицы данных

В нижней части страницы отображаются первые 20 строк вашего файла, чтобы вы могли подтвердить, что сопоставление столбцов выглядит правильно, прежде чем подтвердить. Строки, совпадающие с существующими кодами, выделены.

Когда все будет выглядеть правильно, нажмите **Импортировать N купонов**, чтобы подтвердить партию.

## Шаг 3: Проверка результата

![Страница результата импорта](/static/core/admin/img/help/voucher-import/import-result.webp)

После завершения импорта вы увидите сводку, показывающую:

- **Импортировано** — коды, успешно созданные

- **Пропущено** — коды, которые не были созданы (дубликаты или недопустимые строки)

- **Обработанные строки** — общее количество строк из вашего файла, которые были оценены

- **Неудачно** — строки, которые столкнулись с неожиданной ошибкой

Нажмите **Просмотреть импортированные купоны**, чтобы открыть список купонов, отфильтрованный только кодами из этой партии, что упрощает проверку результата или массовое активирование новых кодов.

Если что-то выглядит неправильно — например, была применена неправильная тип скидки — вы можете использовать стратегию **Перезаписать настройки** при повторном импорте, чтобы исправить партию без необходимости удаления и повторного создания кодов.

Нажмите **Импортировать новую партию**, чтобы начать новый загрузку, или **Назад к списку купонов**, чтобы вернуться к вашему полному каталогу.

## Экспорт купонов как шаблона

Список купонов поддерживает действие экспорта в формате XLSX, которое создает файл в том же порядке столбцов, который ожидает импортер. Это самый простой способ получить правильно отформатированный шаблон:

1. Перейдите в **Маркетинг > Купоны**

2. Выберите купоны, которые вы хотите экспортировать (или выберите все)

3. Выберите **Экспортировать выбранные купоны в XLSX** из выпадающего списка **Действие**

4. Нажмите **Go**

Скачанный файл содержит все 21 столбца, которые понимает импортер, включая поля, которые являются уровнями партии в мастере импорта (тип скидки, даты, лимиты использования и т.д.). Вы можете использовать этот файл в качестве справочного материала или пройти через цикл редактирования → повторный импорт своих существующих кодов с использованием стратегии **Перезаписать настройки**.

## Советы

Сохраните всю разметку Markdown, пути к изображениям, блоки кода и технические термины.

- Сначала скачайте шаблон XLSX, чтобы использовать его как шаблон — названия столбцов предварительно отформатированы, поэтому автоматическое сопоставление распознает их без каких-либо изменений на странице предварительного просмотра.
- Запустите небольшую тестовую партию из 5–10 кодов перед импортом сотен, чтобы проверить, что ваше сопоставление столбцов и настройки партии корректны.
- Используйте **Дней действия** вместо фиксированной **Дата окончания**, когда коды будут распределяться со временем — тогда срок действия каждого кода начнется с момента его импорта, а не с одного календарного дня.
- Если вы получаете коды из третьей стороны системы лояльности, сопоставьте идентификатор члена или клиента поставщика с колонкой **Внешний ID**, чтобы позже можно было свести купоны.
- После крупного импорта нажмите **Просмотреть импортированные купоны** на странице результатов, чтобы отфильтровать список, оставив только новую партию — затем вы можете массово редактировать, активировать или деактивировать их как группу.
- Неудачный импорт (с использованием стратегии **Fail** для дубликатов) оставляет ваш каталог без изменений, поэтому безопасно исправить файл и повторить попытку столько раз, сколько необходимо.
