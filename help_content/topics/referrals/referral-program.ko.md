---
title: 추천 프로그램
---

추천 프로그램은 기존 고객이 가족 및 친구와 함께 고유한 추천 링크를 공유할 수 있도록 합니다. 추천된 친구가 첫 번째 자격 있는 구매를 하면, 추천자와 새 고객 모두가 보상을 받을 수 있습니다. 이는 구두로 전파되는 방식으로 새로운 고객을 유치하게 됩니다.

## 추천 프로그램 작동 방식

1. 고객이 친구와 고유한 추천 링크(또는 코드)를 공유합니다.
2. 친구가 링크를 클릭하면 최대 30일 동안 쿠키를 통해 추적됩니다(설정 가능).
3. 친구가 가입하고 첫 번째 자격 있는 주문을 작성합니다.
4. 시스템은 추천 기여 기록을 생성하고 사기 및 자격 검사를 수행합니다.
5. 기여가 승인되면 양측 모두에게 보상이 지급됩니다.

귀하의 상점에는 단일 추천 프로그램 설정이 있습니다. **마케팅 > 추천 프로그램**으로 이동하여 설정할 수 있습니다.

## 추천 프로그램 설정

### 프로그램 상태

프로그램은 세 가지 상태를 가집니다:

- **초안** — 프로그램이 구성 중이지만 아직 활성화되지 않았습니다. 추천 링크는 비활성 상태입니다.
- **활성** — 프로그램이 활성화되어 있습니다. 고객은 링크를 공유하고 보상을 얻을 수 있습니다.
- **일시 중지** — 프로그램이 일시적으로 중지되었습니다. 기존 기여는 여전히 처리되지만 새로운 추천은 추적되지 않습니다.

프로그램을 출시할 준비가 되었을 때 **상태**를 **활성**으로 설정할 수 있습니다. 언제든지 일시 중지할 수 있습니다.

### 보상 설정

추천이 성공할 때 지급되는 보상을 정의합니다. 이 프로그램은 **양측 보상**을 지원합니다 — 즉, 링크를 공유한 추천자(고객)와 링크를 사용한 새 고객(추천 대상) 모두에게 보상을 지급할 수 있습니다.

**보상 설정** 필드에서 각 수신자의 보상을 구성합니다. 사용 가능한 보상 유형은 다음과 같습니다:

| 보상 유형 | 설명 |
|-------------|-------------|
| **스토어 크레딧** | 고객의 지갑에 크레딧을 추가하여 향후 주문에서 사용할 수 있도록 함 |
| **쿠폰 코드** | 고유한 할인 바우처 코드를 생성함 |
| **백분율 할인** | 체크아웃 시 사용할 수 있는 백분율 할인을 제공함 |
| **특별 혜택** | 고객에게 제공되는 커스텀 혜택 (예: 무료 선물, 우선 접근권) — 보상 설명 필드에 기술됨 |

**예시 구성** — 추천인에게 $10 스토어 크레딧, 신규 고객에게 $10 할인:

```json
{
  "referrer": {"kind": "credit", "amount": 10},
  "referee": {"kind": "discount", "amount": 10},
  "double_sided": true
}
```

추천인에게만 보상을 제공하려면 `"double_sided": false`로 설정하세요.

### 자격 규칙

자격 규칙은 어떤 추천이 보상 대상이 되는지 결정합니다. 이는 **자격 규칙** 필드에서 구성합니다:

| 규칙 | 기능 |
|------|--------------|
| `new_customer_only` | `true`인 경우, 추천된 친구는 기존 주문이 없는 신규 고객이어야 함 |
| `min_order_value` | 추천된 친구가 지출해야 하는 최소 주문 금액 (스토어 통화 단위) |
| `exclude_discounts` | `true`인 경우, 추천 고객이 바우처를 사용한 주문은 자격이 없음 |
| `exclude_staff` | `true`인 경우, 직원 계정은 추천인 또는 추천 대상이 될 수 없음 |

**예시** — 신규 고객만, 최소 $40 주문, 직원 제외:

```json
{
  "new_customer_only": true,
  "min_order_value": 40.0,
  "exclude_discounts": false,
  "exclude_staff": true
}
```

### 타이밍 설정

**타이밍 설정** 필드는 자격이 있는 주문 후 보상이 언제 제공되는지를 제어합니다:

| 설정 | 기능 |
|---------|--------------|
| `issue_on` | 보상을 언제 제공할지: `signup` (등록 시 즉시), `first_purchase` (주문 후 즉시), 또는 `post_refund` (환불 기간이 종료된 후) |
| `refund_window_days` | `post_refund`를 사용할 때 보상을 제공하기 전에 기다려야 하는 일수 (기본값: 14일) |

사용 `post_refund`는 가장 주의 깊은 접근 방식입니다 — 환불 기간이 지나기 전까지 보상이 지급되기를 기다리므로, 나중에 환불되는 주문에 보상을 지급할 위험을 줄입니다.

### Caps and limits

한 명의 추천인에게 무한한 보상을 얻는 것을 방지하기 위해 **Caps & Limits** 필드에서 제한을 설정합니다:

| Setting | What it does |
|---------|--------------|
| `monthly_per_referrer` | 월별로 각 추천인당 성공적으로 추천된 최대 수량 |
| `lifetime_per_referrer` | 각 추천인에게 제공된 성공적인 추천 총 최대 수량 |
| `max_reward_per_order` | 단일 추천 전환에 지급되는 최대 보상 금액 (상점 통화 기준) |

**예시** — 월 20건, 평생 200건, 전환당 최대 $50 보상:

```json
{
  "monthly_per_referrer": 20,
  "lifetime_per_referrer": 200,
  "max_reward_per_order": 50
}
```

### Tracking configuration

**Tracking Configuration** 필드에서 추천 링크의 추적 방식을 구성합니다:

| Setting | What it does |
|---------|--------------|
| `cookie_ttl_days` | 친구가 링크를 클릭한 후 추천 추적 쿠키가 활성 상태로 유지되는 일수 (기본값: 30) |
| `attribution` | 속성 방법 — 현재 `last_touch` (최근 추천 링크 클릭이 인정됨) |

### Fraud policy

사기 탐지 시스템은 승인 전에 각 추천 속성을 위험 점수로 자동 평가합니다. **Fraud Policy** 필드에서 정책을 구성합니다:

| 설정 | 기능 |
|---------|--------------|
| `policy` | 전체 엄격도: `strict`, `balanced`, 또는 `lenient` |
| `auto_reject_threshold` | 위험 점수(0–100)가 이 값보다 높으면 속성 분석이 자동으로 거부됨(기본값: 80) |
| `auto_approve_threshold` | 위험 점수가 이 값보다 낮으면 속성 분석이 자동으로 승인됨(기본값: 30) |
| `check_ip` | `true`인 경우, 추천자와 추천 고객이 동일한 IP 주소를 공유하는지 확인함 |
| `check_device` | `true`인 경우, 추천자와 추천 고객 간에 공유된 기기 지문을 확인함 |
| `check_velocity` | `true`인 경우, 단일 출처에서 비정상적으로 높은 추천률을 감지함 |
| `velocity_window_hours` | 속도 확인을 위한 시간 창(시간 단위) |
| `max_referrals_per_window` | 속도 창 내에서 하나의 출처로부터 허용되는 최대 추천 수 |

자동 거부 임계값과 자동 승인 임계값 사이의 위험 점수를 가진 속성 분석은 **대기 중** 상태가 되며, 수동 검토가 필요함.

### 이용 약관

**이용 약관** 필드에 프로그램에 대한 법적 이용 약관을 입력하세요. 이 텍스트는 고객이 추천 프로그램을 보는 경우에 표시됩니다. 마크다운 형식이 지원됨.

## 추천 속성 분석 보기

**마케팅 > 추천 속성 분석**으로 이동하여 모든 추천 사례를 볼 수 있습니다 — 추천자와 추천 고객 간의 연결 관계.

![추천 속성 분석 목록](/static/core/admin/img/help/referral-program/attribution-list.webp)

각 속성 분석은 추천자, 추천 고객, 그들이 처음으로 주문한 주문, 현재 상태, 위험 점수를 표시함.

### 속성 분석 상태

| 상태 | 의미 |
|--------|---------------|
| **대기 중** | 검토 대기 — 위험 점수가 수동 검토 범위에 있음 |
| **승인됨** | 추천이 유효함 — 보상이 이미 지급되었거나 지급될 예정임 |
| **거부됨** | 추천이 자격 요건을 충족하지 못하거나 사기로 표시됨 |
| **만료됨** | 추천이 추적 창 내에서 전환되지 않았음 |


### 수동으로 승인하거나 거부하는 어트리뷰션

**대기 중** 상태인 어트리뷰션은 어트리뷰션 기록을 열고 액션 버튼을 사용하여 수동으로 승인하거나 거부할 수 있습니다. 거부할 경우 **거부 사유**를 선택하세요:

- 자기 추천
- 신규 고객이 아님
- 최소 주문 금액 미달
- 일회용 이메일
- 캡 초과
- 사기 위험
- 주문 환불 또는 취소됨
- 수동 거부

자신의 기록을 위해 **거부 메모**를 추가할 수도 있습니다.

### 위험 수준으로 필터링

사이드바의 **위험 수준** 필터를 사용하여 검토가 필요한 고위험 어트리뷰션에 집중할 수 있습니다:

- 낮은 위험 (점수 0–30) — 자동 승인
- 중간 위험 (점수 31–70) — 수동 검토
- 높은 위험 (점수 71–89) — 수동 검토, 주의해서 처리하세요
- 매우 높은 위험 (점수 90+) — 자동 거부

## 발급된 보상 보기

**마케팅 > 발급된 보상**으로 이동하여 승인된 어트리뷰션으로 인해 발급된 모든 보상을 확인할 수 있습니다.

각 보상 항목은 고객, 추천자인지 참조자인지, 보상 유형 및 금액, 현재 사용 상태를 표시합니다.

### 보상 상태

| 상태 | 의미 |
|------|------|
| **대기 중** | 보상은 생성되었지만 고객에게 아직 전달되지 않았습니다 |
| **발급됨** | 보상은 활성화되어 고객이 사용할 수 있습니다 |
| **사용됨** | 고객이 보상을 사용했습니다 |
| **만료됨** | 보상은 사용되지 않고 만료일을 지났습니다 |
| **취소됨** | 보상은 수동으로 취소되었습니다 (예: 보상이 발급된 후 원래 주문이 환불된 경우) |

### 보상 취소

보상이 취소되어야 하는 경우 — 예를 들어, 자격 주문이 반품된 경우 — 보상 기록을 열고 **취소** 액션을 사용하세요. 보상이 취소된 이유를 설명하는 메모를 추가하여 기록에 남기세요.

## 팁

- `post_refund` 타이밍 설정부터 시작하세요.

환불 창이 만료되기 전에 보상 지급을 지연시키면 결국 환불되는 주문에 보상을 지급하는 것을 방지할 수 있습니다.
- `balanced` 사기 정책은 대부분의 매장에 적합한 기본 설정입니다.

계좌 수가 적은 계정에서의 추천 수가 비정상적으로 급증하는 것을 발견하면 `strict`로 전환하세요.
- 실현 가능한 월별 및 평생 한도를 설정하세요.

보상 가치가 높은 경우, 월별 추천자당 10–20개의 한도가 사기 방지에 적절합니다.
- **대기 중** 속성 지정을 매주 검토하세요.

너무 오랫동안 검토되지 않은 상태로 두면 보상을 기다리는 정당한 추천자들이 좌절할 수 있습니다.
- **위험 수준** 필터를 사용하여 수동 검토 대기 목록을 우선순위를 정하되, 매우 높은 위험 속성부터 시작하여 중간 위험으로 이동하세요.
- 이용 약관을 짧고 간단한 언어로 유지하세요.

고객이 규칙을 명확히 이해할 때 참여 가능성이 더 높아집니다.
