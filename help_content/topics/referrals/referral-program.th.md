---
title: โปรแกรมแนะนำเพื่อน
---

โปรแกรมแนะนำเพื่อนช่วยให้ลูกค้าที่มีอยู่สามารถแบ่งปันลิงก์แนะนำที่เป็นเอกลักษณ์กับเพื่อนและครอบครัวของพวกเขา เมื่อเพื่อนที่ถูกแนะนำทำการซื้อครั้งแรกที่ตรงตามเงื่อนไข ทั้งผู้แนะนำและลูกค้าใหม่สามารถได้รับรางวัล — ช่วยเพิ่มการดึงดูดลูกค้าใหม่ผ่านการบอกต่อ

## วิธีการทำงานของโปรแกรมแนะนำเพื่อน

1. ลูกค้าแบ่งปันลิงก์แนะนำที่เป็นเอกลักษณ์ (หรือรหัส) กับเพื่อน
2. เพื่อนคลิกลิงก์และจะถูกติดตามผ่านคุกกี้เป็นเวลาสูงสุด 30 วัน (ปรับแต่งได้)
3. เพื่อนลงทะเบียนและสั่งซื้อครั้งแรกที่ตรงตามเงื่อนไข
4. ระบบสร้างบันทึกการแนะนำและดำเนินการตรวจสอบความถูกต้องและคุณสมบัติ
5. หากการแนะนำได้รับการอนุมัติ รางวัลจะถูกมอบให้กับทั้งสองฝ่าย

ร้านค้าของคุณมีการตั้งค่าโปรแกรมแนะนำเพื่อนเพียงหนึ่งชุด เข้าสู่ **Marketing > Referral Program** เพื่อตั้งค่า

## การตั้งค่าโปรแกรมแนะนำเพื่อนของคุณ

### สถานะโปรแกรม

โปรแกรมมีสถานะสามแบบ:

- **Draft** — โปรแกรมกำลังถูกตั้งค่าแต่ยังไม่เปิดใช้งาน ลิงก์แนะนำยังไม่ได้รับการเปิดใช้งาน
- **Active** — โปรแกรมเปิดใช้งานแล้ว ลูกค้าสามารถแบ่งปันลิงก์และรับรางวัลได้
- **Paused** — โปรแกรมถูกหยุดชั่วคราว บันทึกการแนะนำที่มีอยู่ยังคงดำเนินการต่อ แต่ไม่มีการติดตามการแนะนำใหม่

ตั้งค่า **Status** เป็น **Active** เมื่อคุณพร้อมที่จะเปิดตัว คุณสามารถหยุดใช้งานได้ทุกเมื่อ

### การตั้งค่ารางวัล

กำหนดรางวัลที่จะได้รับเมื่อการแนะนำมีผล โปรแกรมรองรับ **รางวัลแบบสองฝ่าย** — หมายความว่าคุณสามารถให้รางวัลทั้งผู้แนะนำ (ลูกค้าที่แบ่งปันลิงก์) และผู้ถูกแนะนำ (ลูกค้าใหม่ที่ใช้ลิงก์)

กำหนดรางวัลสำหรับผู้รับแต่ละคนในช่อง **Reward Configuration** ประเภทของรางวัลที่มีให้คือ:

| ประเภทของรางวัล | คำอธิบาย |
|-------------|-------------|
| **เครดิตร้านค้า** | เพิ่มเครดิตเข้าไปในกระเป๋าเงินของลูกค้า สามารถใช้ได้กับคำสั่งซื้อในอนาคต |
| **รหัสคูปอง** | สร้างรหัสส่วนลดที่เป็นเอกลักษณ์ |
| **ส่วนลดเปอร์เซ็นต์** | ออกส่วนลดเปอร์เซ็นต์เพื่อใช้ที่เคาน์เตอร์ชำระเงิน |
| **สิทธิพิเศษ** | สิทธิพิเศษที่กำหนดเอง (เช่น ของฟรี การเข้าถึงก่อน) — อธิบายในช่องคำอธิบายของรางวัล |

**ตัวอย่างการตั้งค่า** — เครดิตร้านค้า $10 สำหรับผู้แนะนำ และส่วนลด $10 สำหรับลูกค้าใหม่:

```json
{
  "referrer": {"kind": "credit", "amount": 10},
  "referee": {"kind": "discount", "amount": 10},
  "double_sided": true
}
```

ตั้งค่า `"double_sided": false` หากคุณต้องการให้รางวัลเฉพาะผู้แนะนำเท่านั้น。

### กฎเกณฑ์การคุ้มครอง

กฎเกณฑ์การคุ้มครองกำหนดว่าการแนะนำใดที่มีสิทธิ์ได้รับรางวัล ตั้งค่าเหล่านี้ในช่อง **กฎเกณฑ์การคุ้มครอง**:

| กฎ | ทำอะไร |
|------|--------------|
| `new_customer_only` | หากเป็น `true` ลูกค้าที่ถูกแนะนำต้องเป็นลูกค้าใหม่ (ไม่มีคำสั่งซื้อก่อนหน้า) |
| `min_order_value` | จำนวนเงินขั้นต่ำ (ในสกุลเงินของร้านค้า) ที่ลูกค้าที่ถูกแนะนำต้องใช้จ่าย |
| `exclude_discounts` | หากเป็น `true` คำสั่งซื้อที่ลูกค้าที่ถูกแนะนำใช้คูปองจะไม่มีสิทธิ์ |
| `exclude_staff` | หากเป็น `true` บัญชีพนักงานจะไม่สามารถเป็นผู้แนะนำหรือผู้ถูกแนะนำได้ |

**ตัวอย่าง** — ลูกค้าใหม่เท่านั้น จำนวนขั้นต่ำ $40 ห้ามพนักงาน:

```json
{
  "new_customer_only": true,
  "min_order_value": 40.0,
  "exclude_discounts": false,
  "exclude_staff": true
}
```

### การตั้งค่าเวลา

ช่อง **การตั้งค่าเวลา** ควบคุมว่ารางวัลจะถูกออกเมื่อใดหลังจากคำสั่งซื้อที่คุ้มครอง:

| การตั้งค่า | ทำอะไร |
|---------|--------------|
| `issue_on` | กำหนดเวลาในการออกรางวัล: `signup` (ทันทีเมื่อลงทะเบียน), `first_purchase` (ทันทีหลังคำสั่งซื้อ), หรือ `post_refund` (หลังจากช่วงเวลาคืนเงินสิ้นสุด) |
| `refund_window_days` | จำนวนวันที่ต้องรอเพื่อออกรางวัลเมื่อใช้ `post_refund` (ค่าเริ่มต้น: 14 วัน) |

การใช้ `post_refund` เป็นวิธีที่ระมัดระวังที่สุด — มันจะรอจนกว่าช่วงเวลาคืนสินค้าจะผ่านไปแล้วจึงจะให้รางวัล ช่วยลดความเสี่ยงในการให้รางวัลกับคำสั่งซื้อที่อาจถูกคืนเงินในภายหลัง

### Caps และ limits

ป้องกันไม่ให้ผู้แนะนำรายเดียวได้รับรางวัลไม่จำกัดโดยการตั้งค่า caps ในช่อง **Caps & Limits**:

| Setting | What it does |
|---------|--------------|
| `monthly_per_referrer` | จำนวนสูงสุดของคำแนะนำที่สำเร็จและได้รับรางวัลต่อเดือนต่อผู้แนะนำ |
| `lifetime_per_referrer` | จำนวนสูงสุดของคำแนะนำที่สำเร็จและได้รับรางวัลตลอดเวลาต่อผู้แนะนำ |
| `max_reward_per_order` | ค่าเงินสูงสุด (ในสกุลเงินของร้านคุณ) ที่จะถูกให้เป็นรางวัลสำหรับการแนะนำหนึ่งครั้ง |

**ตัวอย่าง** — 20 คำแนะนำต่อเดือน, 200 ตลอดเวลา, 50 ดอลลาร์สูงสุดต่อการแปลง:

```json
{
  "monthly_per_referrer": 20,
  "lifetime_per_referrer": 200,
  "max_reward_per_order": 50
}
```

### การตั้งค่าการติดตาม

กำหนดวิธีการติดตามลิงก์การแนะนำในช่อง **Tracking Configuration**:

| Setting | What it does |
|---------|--------------|
| `cookie_ttl_days` | จำนวนวันที่คุกกี้การติดตามการแนะนำยังคงใช้งานได้หลังจากเพื่อนคลิกที่ลิงก์ (ค่าเริ่มต้น: 30) |
| `attribution` | วิธีการให้เครดิต — ปัจจุบันใช้ `last_touch` (การคลิกลิงก์การแนะนำครั้งล่าสุดจะได้รับเครดิต) |

### นโยบายการฉ้อโกง

ระบบตรวจจับการฉ้อโกงจะให้คะแนนการให้เครดิตการแนะนำแต่ละรายการเพื่อประเมินความเสี่ยงก่อนที่จะอนุมัติ กำหนดนโยบายในช่อง **Fraud Policy**:

| การตั้งค่า | ทำหน้าที่อะไร |
|---------|--------------|
| `policy` | ระดับความเข้มงวดโดยรวม: `strict`, `balanced`, หรือ `lenient` |
| `auto_reject_threshold` | คะแนนความเสี่ยง (0–100) ที่สูงกว่าซึ่งการให้เครดิตจะถูกปฏิเสธอัตโนมัติ (ค่าเริ่มต้น: 80) |
| `auto_approve_threshold` | คะแนนความเสี่ยงที่ต่ำกว่าซึ่งการให้เครดิตจะได้รับการอนุมัติอัตโนมัติ (ค่าเริ่มต้น: 30) |
| `check_ip` | หากเป็น `true` จะตรวจสอบว่าผู้แนะนำและผู้ถูกแนะนำมีที่อยู่ IP เดียวกันหรือไม่ |
| `check_device` | หากเป็น `true` จะตรวจสอบว่าผู้แนะนำและผู้ถูกแนะนำมีลายนิ้วมืออุปกรณ์เดียวกันหรือไม่ |
| `check_velocity` | หากเป็น `true` จะตรวจสอบอัตราการแนะนำที่สูงผิดปกติจากแหล่งเดียวกัน |
| `velocity_window_hours` | ช่วงเวลา (ในหน่วยชั่วโมง) สำหรับการตรวจสอบอัตราการแนะนำ |
| `max_referrals_per_window` | จำนวนการแนะนำสูงสุดที่อนุญาตจากแหล่งเดียวกันภายในช่วงเวลาอัตราการแนะนำ |


การให้เครดิตที่มีคะแนนความเสี่ยงอยู่ระหว่างช่วงของ auto-reject และ auto-approve จะอยู่ในสถานะ **Pending** และต้องการการตรวจสอบด้วยมือ

### เงื่อนไขและข้อตกลง

กรอกข้อกำหนดและเงื่อนไขทางกฎหมายสำหรับโปรแกรมในช่อง **Terms & Conditions** ข้อความนี้จะแสดงให้ลูกค้าเห็นเมื่อพวกเขาดูโปรแกรมการแนะนำ รองรับการจัดรูปแบบ Markdown

## การดูกรณีการแนะนำ

ไปที่ **Marketing > Referral Attributions** เพื่อดูกรณีการแนะนำทั้งหมด — ลิงก์ระหว่างผู้แนะนำและลูกค้าที่ถูกแนะนำ

![รายการการให้เครดิต](/static/core/admin/img/help/referral-program/attribution-list.webp)

แต่ละกรณีการให้เครดิตจะแสดงผู้แนะนำ ลูกค้าที่ถูกแนะนำ คำสั่งซื้อแรกที่พวกเขาทำ สถานะปัจจุบัน และคะแนนความเสี่ยง

### สถานะการให้เครดิต

| สถานะ | หมายถึงอะไร |
|--------|---------------|
| **Pending** | รอการตรวจสอบ — คะแนนความเสี่ยงอยู่ในช่วงที่ต้องตรวจสอบด้วยมือ |
| **Approved** | การแนะนำเป็นไปตามเงื่อนไข — รางวัลได้รับหรือจะได้รับการจ่ายแล้ว |
| **Rejected** | การแนะนำไม่ตรงตามเงื่อนไขหรือถูกตั้งข้อสงสัยว่าเป็นการหลอกลวง |
| **Expired** | การแนะนำไม่ได้ถูกแปลงภายในช่วงเวลาติดตาม |

### การอนุมัติหรือปฏิเสธการให้เครดิตด้วยตนเอง

สำหรับการให้เครดิตที่มีสถานะ **Pending** คุณสามารถอนุมัติหรือปฏิเสธได้ด้วยตนเองโดยการเปิดบันทึกการให้เครดิตและใช้ปุ่มดำเนินการ เมื่อปฏิเสธ ให้เลือก **เหตุผลการปฏิเสธ**:

- Self Referral
- Not New Customer
- Below Minimum Order Value
- Disposable Email
- Cap Exceeded
- Fraud Risk
- Order Refunded or Cancelled
- Manual Rejection

คุณยังสามารถเพิ่ม **หมายเหตุการปฏิเสธ** สำหรับบันทึกของคุณเองได้อีกด้วย

### การกรองตามระดับความเสี่ยง

ใช้ตัวกรอง **Risk Level** ทางด้านข้างเพื่อโฟกัสที่การให้เครดิตที่มีความเสี่ยงสูงซึ่งต้องรีวิว:

- Low Risk (score 0–30) — Auto-approved
- Medium Risk (score 31–70) — Manual review
- High Risk (score 71–89) — Manual review, treat with caution
- Very High Risk (score 90+) — Auto-rejected

## การดูรางวัลที่ได้รับ

ไปที่ **Marketing > Issued Rewards** เพื่อดูรางวัลทั้งหมดที่ได้รับจากการให้เครดิตที่ได้รับการอนุมัติ

แต่ละรายการรางวัลจะแสดงลูกค้า ว่าเป็นผู้แนะนำหรือผู้ถูกแนะนำ ประเภทและจำนวนรางวัล และสถานะการใช้สิทธิ์ในปัจจุบัน

### สถานะของรางวัล

| Status | What it means |
|--------|---------------|
| **Pending** | รางวัลถูกสร้างขึ้นแล้วแต่ยังไม่ได้ส่งให้ลูกค้า |
| **Issued** | รางวัลเป็นที่ใช้งานได้และพร้อมสำหรับลูกค้า |
| **Redeemed** | ลูกค้าได้ใช้รางวัลแล้ว |
| **Expired** | รางวัลหมดอายุโดยไม่ได้ใช้ |
| **Revoked** | รางวัลถูกยกเลิกด้วยตนเอง (เช่น หากการสั่งซื้อเดิมได้รับการคืนเงินหลังจากที่รางวัลถูกออกแล้ว) |

### การยกเลิกรางวัล

หากต้องการยกเลิกรางวัล — เช่น การสั่งซื้อที่เป็นคุณสมบัติถูกคืน — เปิดบันทึกรางวัลและใช้การดำเนินการ **Revoke** เพิ่มหมายเหตุอธิบายว่าทำไมถึงถูกยกเลิกสำหรับบันทึกของคุณ

## เคล็ดลับ

- เริ่มต้นด้วยการตั้งค่า `post_refund`

การรอให้ช่วงเวลาคืนสินค้าสิ้นสุดลงก่อนที่จะออกรางวัลช่วยป้องกันไม่ให้รางวัลสั่งซื้อที่ในที่สุดถูกคืน
- นโยบายการฉ้อโกงแบบ `balanced` เป็นค่าเริ่มต้นที่ดีสำหรับร้านค้าส่วนใหญ่

เปลี่ยนไปใช้ `strict` หากคุณสังเกตเห็นการเพิ่มขึ้นผิดปกติของการแนะนำจากบัญชีจำนวนน้อย
- ตั้งค่าข้อจำกัดรายเดือนและตลอดอายุการใช้งานให้เป็นจริงสมเหตุสมผล

หากค่ารางวัลของคุณสูง การตั้งค่าข้อจำกัด 10–20 ต่อเดือนต่อผู้แนะนำเป็นสิ่งที่เหมาะสมเพื่อป้องกันการใช้ประโยชน์
- ตรวจสอบ **Pending** การให้เครดิตรายสัปดาห์

การปล่อยให้การให้เครดิตเหล่านี้ไม่ได้รับการตรวจสอบเป็นเวลานานอาจทำให้ผู้แนะนำที่ถูกต้องรู้สึกหงุดหงิดเพราะรอรางวัลของพวกเขา
