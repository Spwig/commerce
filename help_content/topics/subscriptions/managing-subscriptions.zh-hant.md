# 管理客戶訂閱

客戶訂閱部分讓您可以全面查看商店中所有活躍、暫停和已取消的重複訂閱。從這裡您可以監控帳單狀況、查看單個訂閱細節，並在出現問題時採取行動。

## 查看客戶訂閱

導航至 **訂閱 > 客戶訂閱**，以查看所有客戶的完整訂閱清單。

![客戶訂閱列表](/static/core/admin/img/help/managing-subscriptions/subscription-list.webp)

清單顯示每個訂閱的客戶、方案名稱、當前狀態、下次帳單日期以及已完成的帳單週期數量。

### 過濾和搜尋

使用右側的過濾面板，按以下條件縮小訂閱範圍：

- **狀態** — 依活躍、試用期、逾期、暫停、已取消或已過期過濾
- **方案** — 查看特定方案的訂閱
- **供應商模式** — 原生（Stripe/PayPal 管理）或備用（內部帳單）

使用搜尋欄透過客戶電子郵件地址查找訂閱。

## 訂閱狀態

了解每個狀態有助於您識別需要關注的訂閱：

| 狀態 | 含義 |
|------|------|
| **試用期** | 客戶處於免費或減價試用期 |
| **活躍** | 訂閱狀態正常 — 帳單當前無欠款，且訪問權限處於活躍狀態 |
| **逾期** | 支付嘗試失敗 — 系統正在重試。在寬限期內，客戶仍保留訪問權限 |
| **暫停** | 訂閱暫時中止 — 無帳單，無訪問權限 |
| **已取消** | 已請求取消。客戶可能在期滿日期前仍擁有訪問權限 |
| **已過期** | 訂閱已完全結束 — 試用期過期、達到最大帳單週期數或取消期已過 |

**逾期** 的訂閱需要最關注 — 如果支付持續失敗且寬限期結束，訂閱將被中止。

## 查看訂閱細節

點擊任何訂閱以打開詳細視圖。

這會顯示：

### 當前計費週期

- **當前週期開始 / 結束** — 活動計費窗口的日期
- **下次計費日期** — 下一次收費將嘗試的時間
- **上次計費日期** 和 **上次計費狀態** — 最近一次計費嘗試的結果
- **計費週期數量** — 已完成的成功計費週期數量

### 訂閱資訊

- **方案** 和 **計費層級** — 用戶所處的方案和計費頻率
- **產品 / 型號** — 與此訂閱相關聯的目錄產品（如適用）
- **數量** — 座位或單位數量（針對基於數量的方案）
- **付款令牌** — 用於重複計費的已存儲付款方式

### 試用詳情

如果訂閱處於試用狀態，**試用結束日期** 會顯示用戶的試用何時到期並開始完整的計費。

### 取消詳情

對於已取消的訂閱，您可以查看：

- **取消類型** — 取消是立即生效、在週期結束時生效，還是已安排
- **取消時間** — 取消請求的時間
- **取消原因** — 用戶取消的說明（如已記錄）
- **恢復截止日期** — 用戶可以在不從頭開始重新訂閱的情況下恢復的最後日期

### 宽限期和承諾

- **宽限期結束日期** — 如果付款失敗，這會顯示在訪問被暫停之前的截止日期
- **最低承諾結束日期** — 對於有最低承諾的方案，最早的取消日期

## 暫停訂閱

暫停訂閱會暫時停止計費，同時也會暫停訪問。這對於想要休息而不完全取消的用戶來說很有用。

要查看已暫停的訂閱，請按 **狀態：已暫停** 篩選。詳細視圖會顯示：

- **暫停時間** — 暫停開始的時間
- **暫停原因** — 暫停的原因說明
- **自動恢復日期** — 如果設置，訂閱將在這一天自動恢復收費和訪問權限

訂閱會在自動恢復日期或客戶手動重新啟用時恢復。

## 計費週期日誌

每次計費嘗試（無論成功或失敗）都會記錄在計費週期日誌中。前往 **訂閱 > 計費週期日誌** 可查看此歷史記錄。

![計費週期日誌列表](/static/core/admin/img/help/managing-subscriptions/billing-cycle-log.webp)

### 閲讀計費週期日誌條目

每個日誌條目記錄以下資訊：

- **訂閱** — 這次計費嘗試屬於哪位客戶的訂閱
- **週期號** — 連續的計費週期（週期 1 = 試用期後的第一筆收費）
- **計費日期** — 收費嘗試的日期
- **狀態** — 挂起、處理中、成功、失敗或重試中
- **金額細節**：
  - **基本金額** — 在任何調整前的方案價格
  - **數量金額** — 單位數量的額外費用
  - **附加功能金額** — 活動附加功能的總費用
  - **折扣金額** — 所有折扣的總額
  - **總金額** — 最終收費（或嘗試收費）的總金額
- **付款方式** — 使用的信用卡或付款方式
- **提供商交易 ID** — 支付提供商的參考編號（用於退款查詢時很有用）
- **失敗原因** — 如果計費失敗，說明失敗原因（例如，信用卡被拒絕、資金不足）

### 診斷付款失敗

如果客戶聯繫您反映計費問題，請找到他們的訂閱並查看計費週期日誌。**失敗原因** 欄位會說明問題所在。常見的失敗原因包括：

- **卡片被拒絕** — 客戶的銀行拒絕了該卡片
- **餘額不足** — 計費時賬戶餘額過低
- **卡片過期** — 儲存的付款方式已過期
- **網絡錯誤** — 與付款提供商的臨時連接問題 — 通常重試後會解決

對於持續失敗的情況，請指引客戶在他們的賬戶設置中更新付款方式。

## 小技巧

- 每週檢查 **逾期未付** 過濾器，以發現可能流失的訂閱。向客戶發送一封簡短的電子郵件，通常可以在寬限期結束前解決付款問題。
- 計費週期日誌為只讀 — 它們是自動生成的，無法修改。這確保了可靠的審計軌跡。
- 如果客戶的訂閱顯示 **逾期未付**，但他們已經更新了付款方式，下一次自動重試將會使用新卡片。重試遵循在計劃中配置的寬限期安排。
- **已過期** 的訂閱不會被刪除 — 它們會保留以供報告使用。使用日期過濾器來專注於目前活躍的訂閱。
- 對於處於 **試用期** 的訂閱，請檢查 **試用結束日期**，以預測即將到來的首次收費，並主動解決任何付款方式問題。
