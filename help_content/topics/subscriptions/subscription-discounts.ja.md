# サブスクリプション割引

サブスクリプション割引は、個々の顧客のサブスクリプションに対して価格の削減を適用できる機能です。例えば、忠実なサブスクライバーへの報酬、プロモーションクーポンの適用、または善意のクレジットによる請求書の紛争解決などに使用できます。プランレベルの価格階層とは異なり、これらの割引は特定のサブスクリプションに直接適用されます。

## サブスクリプション割引の確認

**サブスクリプション > サブスクリプション割引**に移動して、現在のサブスクリプションに適用されているすべての割引を確認できます。

各行には、所属するサブスクリプション、割引の種類と金額、割引がどれくらい続くか、および現在アクティブかどうかが表示されます。

特定のサブスクリプションに添付された割引を確認するには、**サブスクリプション > 顧客サブスクリプション**を開き、サブスクリプションをクリックして、詳細ページの下部にある**割引**セクションにスクロールします。

## サブスクリプションへの割引の追加

新しい割引を追加するには:

1. **サブスクリプション > サブスクリプション割引**に移動します
2. **+ サブスクリプション割引の追加**をクリックします
3. 割引を適用したい**サブスクリプション**を選択します
4. 割引の設定を構成します（下記で説明）
5. **保存**をクリックします

割引は次の請求サイクルから適用されます。

## 割引の種類

割引の計算方法を選択します:

| 割引の種類 | 仕組み | 例 |
|---------------|--------------|---------|
| **百分比割引** | 請求額を百分比で減らす | `20` は $50 の請求額を $40 に減らす |
| **固定金額割引** | 請求額から固定金額を引く | `10` は $50 の請求額を $40 に減らす |
| **固定価格上書き** | 正常なプラン価格に関係なく、サブスクリプションを特定の価格に設定する | `29` は請求額を $29/サイクルに設定する |

**割引価値**フィールドに、選択したタイプ（百分比、ドル額、または固定価格）に対応する数値を入力します。

### 例: 保留オファー

顧客がキャンセルしたいと連絡してきます。あなたは、3か月間25％割引で継続することを提案します:


| フィールド | 値 |
|-------|-------|
| ディスカウントタイプ | パーセンテージ割引 |
| ディスカウント値 | `25` |
| 継続タイプ | 繰り返し |
| 継続期間（月） | `3` |

## ディスカウント期間

将来の請求サイクルにディスカウントが適用される期間を制御してください:

| 継続タイプ | 適用されるタイミング |
|---------------|-----------------|
| **一度だけ適用** | 次の請求サイクルの料金のみを減らし、その後自動的に期限切れになります |
| **永久に** | 手動で無効にするまで、すべての将来的な請求サイクルに適用されます |
| **繰り返し** | 一定の月数だけ適用され、その後期限切れになります |

**繰り返し**タイプのディスカウントでは、**継続期間（月）**フィールドにディスカウントが続く月数を入力してください。**残りのサイクル**フィールドは、残っているサイクル数を追跡します — それぞれの請求サイクルごとにカウントダウンされます。

## クーポンコード

ディスカウントがプロモーションクーポンコードによってトリガーされた場合、**クーポンコード**フィールドにそのコードを入力してください。これは情報提供用であり、ディスカウントがどのプロモーションから発生したかを記録するために使用されます。

## ディスカウントの無効化

ディスカウントが自然に期限切れになる前に停止したい場合は、ディスカウントレコードを開き、**アクティブ**チェックボックスを外し、保存してください。ディスカウントは将来的な請求サイクルには適用されなくなります。次の請求時にサブスクリプションは通常のプラン価格に戻ります。

ディスカウントを作成するときに**有効期限**日を設定することもできます — システムはその日以降自動的にディスカウントを無効にします。

## ヒント

- **一度だけ適用**タイプのディスカウントは、一時的な善意の行動（例: サービス障害に対するサブスクライバーへの補償）に使用してください。

これらは、自動的に期限切れになるため、シンプルです。
- **パーセンテージ割引**は、**固定金額割引**よりも変動価格のサブスクリプションに対して安全です。これは、実際の請求額に合わせて割引がスケーリングされるためです。
- 保留キャンペーンを提供する場合は、**繰り返し**タイプで3か月の継続期間を使用してください — これにより、顧客が継続する理由を提供しつつ、収益を永久に削減することなく、顧客にメリットを提供できます。
- **クーポンコード**フィールドは、顧客が使用したコードと一致させておく必要があります。

# 割引の適用

これは、サブスクリプションの収益を確認する際に、どのプロモーションがどの割引をもたらしたかを監査しやすくします。
- 割引はプランではなく、個々のサブスクリプションに適用されます。

プランの価格をすべての新規サブスクライバー向けに下げる必要がある場合は、代わりにプランの価格階層を更新してください。
