---
title: 顧客サブスクリプションの管理
---

顧客サブスクリプションセクションでは、ストア内のすべてのアクティブ、一時停止、キャンセルされた定期的なサブスクリプションの完全なビューが提供されます。ここから請求状況を監視し、個々のサブスクリプションの詳細を確認し、問題が発生したときにアクションを取ることができます。

## 顧客サブスクリプションの表示

**サブスクリプション > 顧客サブスクリプション**に移動して、すべての顧客のサブスクリプションのフルリストを表示します。

![顧客サブスクリプション一覧](/static/core/admin/img/help/managing-subscriptions/subscription-list.webp)

このリストは、各サブスクリプションの顧客、プラン名、現在のステータス、次の請求日、および完了した請求サイクルの数を表示します。

### フィルタリングと検索

右側のフィルタパネルを使用して、以下でサブスクリプションを絞り込みます:

- **ステータス** — アクティブ、トライアル、過去の請求、一時停止、キャンセル、期限切れでフィルタリング
- **プラン** — 特定のプランのサブスクリプションを表示
- **プロバイダモード** — ネイティブ（Stripe/PayPal管理）または フォールバック（内部請求）

検索バーを使用して、顧客のメールアドレスでサブスクリプションを検索します。

## サブスクリプションステータス

各ステータスを理解することで、注意が必要なサブスクリプションを特定できます:

| ステータス | 意味 |
|------------|------|
| **トライアル** | 顧客は無料または割引価格のトライアル期間中 |
| **アクティブ** | サブスクリプションは正常 — 請求は最新で、アクセスは有効 |
| **過去の請求** | 支払いが失敗しました — システムが再試行しています。恩恵期間中は顧客はアクセスを維持 |
| **一時停止** | サブスクリプションが一時的に停止 — 請求もアクセスもない |
| **キャンセル** | キャンセルが要求されました。期間終了日まで顧客はアクセスを維持する可能性があります |
| **期限切れ** | サブスクリプションが完全に終了 — トライアルが終了、最大請求サイクルに達した、またはキャンセル期間が終了 |

**過去の請求**のサブスクリプションは最も注意が必要です — 支払いが継続して失敗し、恩恵期間が終了するとサブスクリプションは停止されます。

## サブスクリプションの詳細の表示

サブスクリプションをクリックして、詳細ビューを開きます。

これは表示されます：

### 現在の請求期間

- **現在の期間の開始 / 終了** — アクティブな請求ウィンドウの日付
- **次の請求日** — 次の請求が試行される日
- **最後の請求日** および **最後の請求ステータス** — 最新の請求試行の結果
- **請求サイクル数** — 成功した請求サイクルの数

### サブスクリプション情報

- **プラン** および **料金レベル** — 顧客が利用しているプランと請求頻度
- **製品 / バリエーション** — このサブスクリプションに関連付けられたカタログ製品（該当する場合）
- **数量** — シートまたは単位数（数量ベースのプランの場合）
- **支払いトークン** — 再請求のために使用されている保存された支払い方法

### 試用期間の詳細

サブスクリプションが試用期間中である場合、**試用期間終了日** は顧客の試用期間が終了し、完全な請求が始まる日を表示します。

### 解約の詳細

解約されたサブスクリプションについては、以下を確認できます：

- **解約タイプ** — 解約が即時、期間終了時、または予定されているか
- **解約日時** — 解約が要求された日時
- **解約理由** — 顧客が解約した理由（記録されている場合）
- **再開期限** — 顧客が再サブスクリプションなしで再開できる最後の日付

### 猶予期間とコミットメント

- **猶予期間終了日** — 支払いが失敗した場合、アクセスが停止される前の期限を表示します
- **最低コミットメント終了日** — 最低コミットメントがあるプランの場合、最も早い解約日

## サブスクリプションの一時停止

一時停止されたサブスクリプションは、一時的に請求を停止し、アクセスも停止します。これは、完全に解約せずに休憩を取る必要がある顧客にとって便利です。

一時停止されたサブスクリプションを表示するには、**ステータス: 一時停止**でフィルタを適用してください。詳細ビューでは以下が表示されます：

- **Paused At** — パウズが開始された時刻
- **Pause Reason** — パウズの理由に関するメモ
- **Auto Resume Date** — 設定されている場合、サブスクリプションが自動的に請求とアクセスの再開が行われる日付

サブスクリプションは、自動再開日または顧客が手動で再アクティベートしたときに再開されます。

## Billing cycle logs

すべての請求試行 — 成功または失敗 — は請求サイクルログに記録されます。**Subscriptions > Billing Cycle Logs** にアクセスしてこの履歴を確認してください。

![Billing cycle log list](/static/core/admin/img/help/managing-subscriptions/billing-cycle-log.webp)

### Reading a billing cycle log entry

各ログエントリは以下を記録します:

- **Subscription** — この請求試行が所属する顧客のサブスクリプション
- **Cycle Number** — 連番の請求サイクル（サイクル1 = 試用期間後の最初の請求）
- **Billing Date** — 請求が試行された日時
- **Status** — 保留中、処理中、成功、失敗、または再試行
- **Amount breakdown**:
  - **Base Amount** — あらゆる調整前のプラン価格
  - **Quantity Amount** — シート/単位数の追加料金
  - **Add-ons Amount** — 有効なアドオンの合計コスト
  - **Discount Amount** — 適用された割引の合計
  - **Total Amount** — 最終的に請求（または試行）された金額
- **Payment Method** — 使用されたカードまたは支払い方法
- **Provider Transaction ID** — 支払いプロバイダーの参照番号（返金の検索に役立ちます）
- **Failure Reason** — 請求が失敗した場合、失敗した理由（例: カードが拒否された、資金不足）

### Diagnosing payment failures

顧客が請求に関する問題について連絡してきた場合、そのサブスクリプションを確認し、請求サイクルログをチェックしてください。**Failure Reason** フィールドは、何が問題になったかを説明します。一般的な失敗理由には以下が含まれます:

- **カードが拒否されました** — 顧客の銀行によってカードが拒否されました
- **資金不足** — 請求時に口座残高が低すぎました
- **有効期限切れのカード** — 保存された支払い方法が有効期限切れです
- **ネットワークエラー** — 支払いプロバイダーとの一時的な接続問題 — 再試行することで通常は解決します

持続的な失敗の場合、顧客にアカウント設定で支払い方法を更新するよう案内してください。

## Tips

- **Past Due**フィルターを週に1回確認して、退会のリスクがあるサブスクリプションを把握してください。顧客にメールを送信するだけでも、猶予期間が終了する前に支払い問題を解決できる場合があります。
- 請求サイクルログは読み取り専用です — これらは自動的に作成され、変更することはできません。これは信頼性の高い監査トレースを保証します。
- 顧客のサブスクリプションが**Past Due**を示している場合でも、すでに支払い方法を更新している場合は、次の自動再試行で新しいカードが使用されます。再試行は、プランで設定された猶予期間スケジュールに従って行われます。
- **有効期限切れ**のサブスクリプションは削除されません — 報告のために表示されたままです。日付フィルターを使用して、現在アクティブなサブスクリプションに焦点を当ててください。
- **Trial**状態のサブスクリプションについては、**Trial End Date**を確認して、最初の請求が近づいていることを予測し、支払い方法の問題を事前に解決してください。
