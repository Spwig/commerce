---
title: การจัดการการสมัครสมาชิกของลูกค้า
---

ส่วนการสมัครสมาชิกของลูกค้าให้คุณมีมุมมองที่ครบถ้วนเกี่ยวกับการสมัครสมาชิกแบบซ้ำที่เปิดใช้งานอยู่ ถูกระงับ และถูกยกเลิกในร้านของคุณ จากที่นี่คุณสามารถตรวจสอบสุขภาพการเรียกเก็บเงิน ดูรายละเอียดการสมัครสมาชิกแต่ละรายการ และดำเนินการเมื่อเกิดปัญหา

## การดูการสมัครสมาชิกของลูกค้า

ไปที่ **Subscriptions > Customer Subscriptions** เพื่อดูรายการการสมัครสมาชิกทั้งหมดของลูกค้า

![รายการการสมัครสมาชิกของลูกค้า](/static/core/admin/img/help/managing-subscriptions/subscription-list.webp)

รายการนี้แสดงรายละเอียดของลูกค้า การสมัครสมาชิก ชื่อแผน สถานะปัจจุบัน วันที่เรียกเก็บเงินครั้งต่อไป และจำนวนรอบการเรียกเก็บเงินที่เสร็จสิ้นแล้ว

### การกรองและการค้นหา

ใช้แผงกรองทางด้านขวาเพื่อจำกัดการสมัครสมาชิกตาม:

- **สถานะ** — กรองตาม Active, Trial, Past Due, Paused, Canceled, หรือ Expired
- **แผน** — ดูการสมัครสมาชิกสำหรับแผนเฉพาะ
- **โหมดผู้ให้บริการ** — Native (Stripe/PayPal-managed) หรือ Fallback (การเรียกเก็บเงินภายใน)

ใช้แถบค้นหาเพื่อค้นหาการสมัครสมาชิกตามที่อยู่อีเมลของลูกค้า

## สถานะการสมัครสมาชิก

การเข้าใจสถานะแต่ละสถานะช่วยให้คุณระบุการสมัครสมาชิกที่ต้องการความสนใจ:

| สถานะ | หมายถึงอะไร |
|--------|---------------|
| **Trial** | ลูกค้าอยู่ในช่วงทดลองใช้ฟรีหรือราคาลด |
| **Active** | การสมัครสมาชิกอยู่ในสภาพปกติ — การเรียกเก็บเงินเป็นปัจจุบันและสิทธิ์การเข้าถึงเปิดใช้งาน |
| **Past Due** | การชำระเงินล้มเหลว — ระบบกำลังพยายามอีกครั้ง ลูกค้ายังคงมีสิทธิ์การเข้าถึงในช่วงเวลาอภัย |
| **Paused** | การสมัครสมาชิกถูกระงับชั่วคราว — ไม่มีการเรียกเก็บเงินและไม่มีสิทธิ์การเข้าถึง |
| **Canceled** | การยกเลิกได้รับการขอแล้ว ลูกค้าอาจยังมีสิทธิ์การเข้าถึงจนถึงวันสิ้นสุดของช่วงการสมัครสมาชิก |
| **Expired** | การสมัครสมาชิกสิ้นสุดลงอย่างสมบูรณ์ — ช่วงทดลองสิ้นสุด จำนวนรอบการเรียกเก็บเงินสูงสุดถึง หรือช่วงเวลาการยกเลิกสิ้นสุด |

การสมัครสมาชิกที่ **Past Due** ต้องการความสนใจมากที่สุด — หากการชำระเงินยังคงล้มเหลวและช่วงเวลาอภัยสิ้นสุด การสมัครสมาชิกจะถูกระงับ

## การดูรายละเอียดการสมัครสมาชิก

คลิกที่การสมัครสมาชิกใดๆ เพื่อเปิดดูรายละเอียด

นี่คือสิ่งที่แสดง:

### ช่วงการเรียกเก็บเงินปัจจุบัน

- **Current Period Start / End** — วันที่ของช่วงการเรียกเก็บเงินที่ใช้งานอยู่
- **Next Billing Date** — วันที่จะมีการเรียกเก็บเงินครั้งต่อไป
- **Last Billing Date** และ **Last Billing Status** — ผลลัพธ์ของการเรียกเก็บเงินครั้งล่าสุด
- **Billing Cycle Count** — จำนวนรอบการเรียกเก็บเงินที่สำเร็จแล้ว

### ข้อมูลการสมัครสมาชิก

- **Plan** และ **Pricing Tier** — แผนและความถี่ในการเรียกเก็บเงินที่ลูกค้ากำลังใช้งานอยู่
- **Product / Variant** — ผลิตภัณฑ์ในแคตตาล็อกที่เชื่อมโยงกับการสมัครสมาชิกนี้ (ถ้ามี)
- **Quantity** — จำนวนที่นั่งหรือหน่วย (สำหรับแผนที่คำนวณตามปริมาณ)
- **Payment Token** — วิธีการชำระเงินที่ถูกเก็บไว้สำหรับการเรียกเก็บเงินแบบซ้ำๆ

### รายละเอียดทดลองใช้

หากการสมัครสมาชิกอยู่ในช่วงทดลองใช้ คุณจะเห็น **Trial End Date** ซึ่งแสดงวันที่การทดลองใช้ของลูกค้าจะสิ้นสุดและเริ่มการเรียกเก็บเงินเต็มรูปแบบ

### รายละเอียดการยกเลิก

สำหรับการสมัครสมาชิกที่ถูกยกเลิก คุณสามารถดูได้:

- **Cancellation Type** — ว่าการยกเลิกเป็นแบบทันที ที่สิ้นสุดช่วง หรือกำหนดไว้ล่วงหน้า
- **Canceled At** — วันที่ที่การยกเลิกถูกขอ
- **Cancellation Reason** — หมายเหตุเกี่ยวกับเหตุผลที่ลูกค้ายกเลิก (ถ้ามีการบันทึก)
- **Reactivation Deadline** — วันสุดท้ายที่ลูกค้าสามารถเปิดใช้งานใหม่โดยไม่ต้องสมัครสมาชิกใหม่จากศูนย์

### ช่วงเวลาอภัยและข้อผูกมัด

- **Grace Period End Date** — หากการชำระเงินล้มเหลว นี่คือวันสุดท้ายก่อนที่การเข้าถึงจะถูกระงับ
- **Minimum Commitment End Date** — สำหรับแผนที่มีข้อผูกมัดขั้นต่ำ วันที่เร็วที่สุดที่สามารถยกเลิกได้

## การหยุดการสมัครสมาชิก

การสมัครสมาชิกที่ถูกหยุดจะหยุดการเรียกเก็บเงินชั่วคราวพร้อมกับระงับการเข้าถึง นี่เป็นประโยชน์สำหรับลูกค้าที่ต้องการพักชั่วคราวโดยไม่ยกเลิกสมัครสมาชิกทั้งหมด

เพื่อดูการสมัครสมาชิกที่ถูกหยุด ให้กรองด้วย **Status: Paused** หน้ารายละเอียดแสดง:

- **Paused At** — เมื่อการพักเริ่มขึ้น
- **Pause Reason** — หมายเหตุว่าทำไมจึงถูกพัก
- **Auto Resume Date** — หากตั้งค่าไว้ วันที่การสมัครสมาชิกจะเริ่มต้นการเรียกเก็บเงินและเข้าถึงอีกครั้งโดยอัตโนมัติ

การสมัครสมาชิกจะเริ่มต้นใหม่ทั้งในวันที่กำหนดให้เริ่มต้นอัตโนมัติ หรือเมื่อลูกค้าเปิดใช้งานใหม่ด้วยตนเอง

## Billing cycle logs

ทุกครั้งที่มีการเรียกเก็บเงิน — ไม่ว่าจะสำเร็จหรือล้มเหลว — จะถูกบันทึกในบันทึกวงจรการเรียกเก็บเงิน ไปที่ **Subscriptions > Billing Cycle Logs** เพื่อดูประวัติการใช้งาน

![Billing cycle log list](/static/core/admin/img/help/managing-subscriptions/billing-cycle-log.webp)

### Reading a billing cycle log entry

แต่ละรายการบันทึกจะบันทึก:

- **Subscription** — การสมัครสมาชิกของลูกค้าที่การเรียกเก็บเงินครั้งนี้เกี่ยวข้อง
- **Cycle Number** — วงจรการเรียกเก็บเงินตามลำดับ (Cycle 1 = การเรียกเก็บเงินครั้งแรกหลังทดลองใช้)
- **Billing Date** — วันที่มีการเรียกเก็บเงิน
- **Status** — ค้างอยู่, กำลังดำเนินการ, สำเร็จ, ล้มเหลว, หรือกำลังลองอีกครั้ง
- **Amount breakdown**:
  - **Base Amount** — ราคาแผนก่อนการปรับเปลี่ยนใด ๆ
  - **Quantity Amount** — ค่าเพิ่มสำหรับจำนวนที่นั่ง/หน่วย
  - **Add-ons Amount** — ค่ารวมของ add-ons ที่ใช้งานอยู่
  - **Discount Amount** — ค่าส่วนลดทั้งหมดที่ถูกนำมาใช้
  - **Total Amount** — จำนวนเงินทั้งหมดที่ถูกเรียกเก็บ (หรือพยายามเรียกเก็บ)
- **Payment Method** — บัตรหรือวิธีการชำระเงินที่ใช้
- **Provider Transaction ID** — เลขอ้างอิงจากผู้ให้บริการชำระเงิน (มีประโยชน์สำหรับการค้นหาการคืนเงิน)
- **Failure Reason** — หากการเรียกเก็บเงินล้มเหลว สาเหตุที่เกิดขึ้น (เช่น บัตรถูกปฏิเสธ, เงินไม่เพียงพอ)

### Diagnosing payment failures

หากลูกค้าติดต่อคุณเกี่ยวกับปัญหาการเรียกเก็บเงิน ให้ค้นหาการสมัครสมาชิกของพวกเขาและตรวจสอบบันทึกวงจรการเรียกเก็บเงิน ฟิลด์ **Failure Reason** จะอธิบายว่าเกิดอะไรขึ้น สาเหตุการล้มเหลวที่พบบ่อยมีดังนี้:

- **การปฏิเสธบัตร** — บัตรของลูกค้าถูกธนาคารปฏิเสธ
- **เงินไม่เพียงพอ** — ยอดเงินในบัญชีต่ำเกินไปในขณะที่กำลังเรียกเก็บเงิน
- **บัตรหมดอายุ** — วิธีการชำระเงินที่บันทึกไว้มีอายุหมด
- **ข้อผิดพลาดของเครือข่าย** — ปัญหาการเชื่อมต่อชั่วคราวกับผู้ให้บริการชำระเงิน — มักจะแก้ไขได้เมื่อลองอีกครั้ง

หากมีความล้มเหลวที่เกิดขึ้นซ้ำ ๆ ให้แนะนำลูกค้าให้ไปอัปเดตวิธีการชำระเงินของพวกเขาในตั้งค่าบัญชีของพวกเขา

## เคล็ดลับ

- ตรวจสอบตัวกรอง **Past Due** รายสัปดาห์เพื่อจับการสมัครสมาชิกที่มีความเสี่ยงจะยกเลิก ข้อความอีเมลสั้น ๆ ถึงลูกค้ามักจะแก้ไขปัญหาการชำระเงินก่อนที่ช่วงเวลาอภัยโทษจะสิ้นสุดลง
- บันทึกการเรียกเก็บเงินเป็นแบบอ่านเฉพาะ — สร้างโดยอัตโนมัติและไม่สามารถแก้ไขได้ นี่ช่วยให้มั่นใจได้ว่ามีประวัติการตรวจสอบที่น่าเชื่อถือ
- หากการสมัครสมาชิกของลูกค้าแสดงสถานะ **Past Due** แต่ลูกค้าได้อัปเดตวิธีการชำระเงินแล้ว การลองอีกครั้งอัตโนมัติจะใช้บัตรใหม่ ความพยายามในการลองอีกครั้งจะดำเนินการตามตารางช่วงเวลาอภัยโทษที่กำหนดในแผน
- การสมัครสมาชิกที่ **หมดอายุ** จะไม่ถูกลบ — จะยังคงแสดงอยู่สำหรับการรายงาน ใช้ตัวกรองวันที่เพื่อโฟกัสกับการสมัครสมาชิกที่ยังใช้งานอยู่
- สำหรับการสมัครสมาชิกที่อยู่ใน **Trial** ให้ตรวจสอบ **Trial End Date** เพื่อคาดการณ์การเรียกเก็บเงินครั้งแรกที่จะเกิดขึ้นเร็ว ๆ นี้ และแก้ไขปัญหาการชำระเงินล่วงหน้า
