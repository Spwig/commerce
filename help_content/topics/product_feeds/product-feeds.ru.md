---
title: Продуктовые фиды
---

Продуктовые фиды позволяют экспортировать ваш каталог на платформы покупок, такие как Google Shopping и Facebook Catalog. После подключения данные о продуктах автоматически синхронизируются по расписанию, чтобы ваши объявления всегда отражали текущие цены, наличие товара и детали продуктов.

Ваш магазин использует систему компонентов-поставщиков для фидов. Каждый поставщик фидов (Google, Facebook или другие) устанавливается как компонент и затем подключается через учетную запись поставщика. Вы можете запускать несколько поставщиков фидов одновременно — например, один фид для Google Shopping и отдельный для Facebook.

## Подключение поставщика фидов

Перед тем как синхронизировать ваш каталог, вам нужно установить и подключить хотя бы один компонент поставщика фидов.

### Установка компонента поставщика

Компоненты поставщиков доступны в магазине компонентов Spwig. Администратор магазина устанавливает их через систему обновления компонентов. После установки компонента поставщика он появляется как вариант при создании учетной записи поставщика фидов.

### Создание учетной записи поставщика фидов

1. Перейдите в **Маркетинг > Поставщики фидов**
2. Нажмите **+ Добавить учетную запись поставщика фидов**
3. Заполните форму:

**Раздел Информация о поставщике:**
- **Сайт** — выберите ваш магазин (только один)
- **Компонент поставщика** — выберите установленный поставщик фидов (например, Google Shopping, Facebook Catalog)
- **Имя учетной записи** — описательное имя, например, `Google Shopping — Основной` или `Facebook Catalog — США`

**Раздел Настройка:**
- **Включено** — установите флажок, чтобы включить генерацию и синхронизацию фидов
- **Основной** — установите флажок, если это основной поставщик фидов для этого типа платформы
- **Приоритет** — управляет порядком сортировки в списке (меньшие числа отображаются первыми)
- **Настройка** — специфические настройки поставщика (см. ниже)

4. Нажмите **Сохранить**

## Параметры настройки фидов

Поле **Настройка** принимает объект JSON с следующими параметрами:

| Параметр | Значения | Описание |
|----------|---------|----------|
| `sync_interval` | `hourly`, `daily`, `weekly`, `manual` | Как часто фид автоматически регенерируется |
| `format_preference` | `xml`, `csv`, `json` | Формат вывода (наибольшее предпочтение у XML) |
| `include_variants` | `true` / `false` | Включать варианты продуктов как отдельные записи фидов |
| `target_country` | Код страны, например, `"US"` | Целевая страна для фидов |
| `content_language` | Код языка, например, `"en"` | Язык данных о продуктах |

#### Пример настройки ежедневного XML-фидов для США:

```json
{
  "sync_interval": "daily",
  "format_preference": "xml",
  "include_variants": true,
  "target_country": "US",
  "content_language": "en"
}
```

## Фильтрация продуктов, которые появляются в фиде

Вы можете точно контролировать, какие продукты включаются, добавив раздел `product_filter` в конфигурацию:

```json
{
  "product_filter": {
    "status": ["published"],
    "in_stock_only": true,
    "categories": [1, 5, 12]
  }
}
```

| Параметр фильтра | Описание |
|------------------|----------|
| `status` | Включать только продукты с этими статусами. Используйте `["published"]`, чтобы включить только активные продукты |
| `in_stock_only` | Установите `true`, чтобы исключить продукты, которых нет в наличии |
| `categories` | Ограничить определенными идентификаторами категорий |
| `brands` | Ограничить определенными идентификаторами брендов |

Вы также можете исключить определенные продукты по их идентификаторам с помощью `exclude_products`:

```json
{
  "exclude_products": [42, 87, 103]
}
```

## Отслеживание статуса синхронизации

Список учетных записей поставщиков фидов показывает статус синхронизации каждого подключенного фида:

- **PENDING** — синхронизация еще не выполнялась, или фид ожидает генерации
- **SYNCING** — синхронизация в настоящее время выполняется
- **SUCCESS** — последняя синхронизация завершилась без ошибок
- **ERROR** — последняя синхронизация завершилась с ошибкой; сообщение об ошибке отображается на странице деталей учетной записи

Список также показывает количество продуктов в текущем фиде и время последней синхронизации.

## Просмотр сгенерированных фидов

Перейдите в **Маркетинг > Продуктовые фиды**, чтобы увидеть сгенерированные файлы фидов. Каждая запись представляет собой один снимок сгенерированного фида и показывает:

- **Учетная запись провайдера** — к которой относится этот фид
- **Формат** — XML, CSV или JSON
- **Количество продуктов** — количество включенных продуктов
- **Размер** — размер сгенерированного фида
- **Создано** — когда он был создан
- **Истекает** — когда истечет срок действия этого кэшированного фида
- **Статус** — является ли фид все еще действительным или истек ли срок его действия
- **Количество загрузок** — сколько раз этот фид был загружен

Фиды только для чтения в административной панели — они генерируются автоматически процессом синхронизации.

## Просмотр истории синхронизации

Перейдите в **Маркетинг > Журналы синхронизации фидов**, чтобы увидеть полную историю всех попыток синхронизации для всех ваших учетных записей фидов. Каждый запись журнала фиксирует:

- Учетную запись провайдера, которая была синхронизирована
- Тип синхронизации (Полная, Инкрементальная, Ручная или Расписание)
- Статус (Успешно, Частично успешно, Неудачно и т.д.)
- Синхронизированные, неудачные и пропущенные продукты
- Продолжительность синхронизации
- Любые сообщения об ошибках

Панель журнала синхронизации в верхней части страницы показывает общие статистики: общее количество синхронизаций, процент успешных синхронизаций и средняя продолжительность синхронизации. Используйте фильтры **Учетная запись** и **Тип синхронизации**, чтобы сузить до конкретного фида.

### Что делать, если синхронизация завершилась неудачно

1. Перейдите в **Маркетинг > Журналы синхронизации фидов** и найдите неудачную запись
2. Нажмите на запись журнала, чтобы просмотреть полное **Сообщение об ошибке** и **Детали ошибки**
3. Общие причины включают:
   - Отсутствие обязательных полей продукта (название, цена, изображение)
   - Недействительные или истекшие учетные данные API — переустановите компонент провайдера, чтобы обновить учетные данные
   - Ошибки сети при подключении к API провайдера
4. После устранения проблемы следующая запланированная синхронизация будет выполняться автоматически, или вы можете запустить ручную синхронизацию с учетной записи провайдера

## Советы

- Установите `"sync_interval": "daily"` для большинства случаев использования — Google и Facebook не требуют более частных обновлений, если только у вас нет очень высокой волатильности цен
- Всегда включайте `"in_stock_only": true` в вашем фильтре продуктов, чтобы избежать рекламы продуктов, которые клиенты не могут купить
- Используйте описательное имя учетной записи, включающее платформу и целевой рынок (например, `Google Shopping — UK`), чтобы было легко управлять несколькими фидами
- Количество **Продуктов в фиде** на учетной записи провайдера сразу покажет, включаются ли в него меньше продуктов, чем ожидалось — проверьте настройки фильтра продуктов, если количество кажется низким
- Отметьте одну учетную запись как **Основной фид** для каждого типа провайдера; некоторые инструменты отчетности используют это, чтобы определить ваш основной фид
- Проверьте журнал синхронизации после любых крупных изменений в вашем каталоге продуктов, чтобы убедиться, что обновленные данные были правильно захвачены
