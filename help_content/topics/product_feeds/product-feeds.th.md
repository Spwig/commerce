---
title: การจัดหาข้อมูลสินค้า
---

การจัดหาข้อมูลสินค้าช่วยให้คุณสามารถส่งออกแคตตาล็อกของคุณไปยังแพลตฟอร์มการช้อปปิ้ง เช่น Google Shopping และ Facebook Catalog เมื่อเชื่อมต่อแล้ว ข้อมูลสินค้าของคุณจะถูกซิงค์อัตโนมัติในช่วงเวลาที่กำหนด เพื่อให้โฆษณาของคุณสะท้อนราคา จำนวนสต็อก และรายละเอียดสินค้าที่เป็นปัจจุบันเสมอ

ร้านค้าของคุณใช้ระบบส่วนประกอบผู้ให้บริการสำหรับการจัดหาข้อมูล แต่ละผู้ให้บริการจัดหาข้อมูล (Google, Facebook หรือผู้ให้บริการอื่น ๆ) จะถูกติดตั้งเป็นส่วนประกอบ และจากนั้นเชื่อมต่อผ่านบัญชีผู้ให้บริการ คุณสามารถใช้ผู้ให้บริการจัดหาข้อมูลหลายรายพร้อมกันได้ — ตัวอย่างเช่น หนึ่งสำหรับ Google Shopping และอีกหนึ่งแยกสำหรับ Facebook

## การเชื่อมต่อกับผู้ให้บริการจัดหาข้อมูล

ก่อนที่คุณจะซิงค์แคตตาล็อกของคุณ คุณจำเป็นต้องติดตั้งและเชื่อมต่อส่วนประกอบผู้ให้บริการอย่างน้อยหนึ่งตัว

### การติดตั้งส่วนประกอบผู้ให้บริการ

ส่วนประกอบผู้ให้บริการมีอยู่ในตลาดส่วนประกอบ Spwig ผู้ดูแลร้านค้าของคุณติดตั้งผ่านระบบอัปเดตส่วนประกอบ เมื่อส่วนประกอบผู้ให้บริการถูกติดตั้งแล้ว มันจะปรากฏเป็นตัวเลือกเมื่อสร้างบัญชีผู้ให้บริการจัดหาข้อมูล

### การสร้างบัญชีผู้ให้บริการจัดหาข้อมูล

1. ไปที่ **Marketing > Feed Providers**
2. คลิก **+ Add Feed Provider Account**
3. กรอกฟอร์ม:

**ส่วนข้อมูลผู้ให้บริการ:**
- **Site** — เลือกร้านค้าของคุณ (มีเพียงหนึ่งเดียว)
- **Provider Component** — เลือกส่วนประกอบผู้ให้บริการจัดหาข้อมูลที่ติดตั้งแล้ว (เช่น Google Shopping, Facebook Catalog)
- **Account Name** — ชื่อที่อธิบายได้ เช่น `Google Shopping — Main` หรือ `Facebook Catalog — US`

**ส่วนการตั้งค่า:**
- **Is Active** — เลือกเพื่อเปิดใช้งานการสร้างและซิงค์ข้อมูลจัดหาข้อมูล
- **Is Primary** — เลือกหากนี่คือผู้ให้บริการจัดหาข้อมูลหลักสำหรับประเภทแพลตฟอร์มนี้
- **Priority** — ควบคุมลำดับการจัดเรียงในรายการ (ตัวเลขที่ต่ำกว่าจะปรากฏก่อน)
- **Config** — ตั้งค่าเฉพาะผู้ให้บริการ (ดูข้อมูลด้านล่าง)

4. คลิก **Save**

### ตัวเลือกการตั้งค่าข้อมูลจัดหาข้อมูล

ฟิลด์ **Config** รับวัตถุ JSON ที่มีตัวเลือกต่อไปนี้:

รักษาการจัดรูปแบบมาร์กดาวน์ ค่าเส้นทางรูปภาพ บล็อกโค้ด และคำศัพท์ทางเทคนิคทั้งหมดไว้

| ตัวเลือก | ค่า | คำอธิบาย |
|--------|--------|-------------|
| `sync_interval` | `hourly`, `daily`, `weekly`, `manual` | ความถี่ที่ feed จะถูกสร้างขึ้นอัตโนมัติ |
| `format_preference` | `xml`, `csv`, `json` | รูปแบบการส่งออก (แพลตฟอร์มส่วนใหญ่ชอบ XML) |
| `include_variants` | `true` / `false` | รวมผลิตภัณฑ์แบบต่าง ๆ เป็นรายการ feed แยก |
| `target_country` | รหัสประเทศ เช่น `"US"` | ประเทศเป้าหมายสำหรับ feed |
| `content_language` | รหัสภาษา เช่น `"en"` | ภาษาของข้อมูลผลิตภัณฑ์ |

#### ตัวอย่างการตั้งค่าสำหรับ feed XML รายวันที่มุ่งเป้าไปที่สหรัฐอเมริกา:

```json
{
  "sync_interval": "daily",
  "format_preference": "xml",
  "include_variants": true,
  "target_country": "US",
  "content_language": "en"
}
```

## การกรองผลิตภัณฑ์ที่ปรากฏใน feed

คุณสามารถควบคุมผลิตภัณฑ์ที่ถูกรวมอยู่ได้อย่างแม่นยำโดยการเพิ่มส่วน `product_filter` ลงใน config:

```json
{
  "product_filter": {
    "status": ["published"],
    "in_stock_only": true,
    "categories": [1, 5, 12]
  }
}
```

| ตัวเลือกการกรอง | คำอธิบาย |
|---------------|-------------|
| `status` | รวมผลิตภัณฑ์ที่มีสถานะเหล่านี้เท่านั้น ใช้ `["published"]` เพื่อรวมผลิตภัณฑ์ที่เผยแพร่แล้วเท่านั้น |
| `in_stock_only` | ตั้งค่าเป็น `true` เพื่อไม่รวมผลิตภัณฑ์ที่หมดสต็อก |
| `categories` | จำกัดเฉพาะ ID หมวดหมู่ |
| `brands` | จำกัดเฉพาะ ID แบรนด์ |

คุณยังสามารถใช้ `exclude_products` เพื่อไม่รวมผลิตภัณฑ์เฉพาะตาม ID ได้อีกด้วย:

```json
{
  "exclude_products": [42, 87, 103]
}
```

## การตรวจสอบสถานะการซิงค์

รายการบัญชีผู้ให้บริการ feed จะแสดงสถานะการซิงค์ของ feed ที่เชื่อมต่อแต่ละตัวได้อย่างรวดเร็ว:

- **PENDING** — ยังไม่มีการซิงค์ใด ๆ หรือ feed กำลังรอการสร้าง |
- **SYNCING** — การซิงค์กำลังดำเนินอยู่ |
- **SUCCESS** — การซิงค์ครั้งสุดท้ายเสร็จสมบูรณ์โดยไม่มีข้อผิดพลาด |
- **ERROR** — การซิงค์ครั้งสุดท้ายล้มเหลว; ข้อความแสดงข้อผิดพลาดจะแสดงบนหน้ารายละเอียดบัญชี

รายการยังแสดงจำนวนผลิตภัณฑ์ใน feed ปัจจุบันและเวลาที่การซิงค์ครั้งสุดท้ายดำเนินการ


ไปที่ **การตลาด > Product Feeds** เพื่อดูไฟล์ feed ที่สร้างขึ้น

แต่ละรายการแสดงถึง快照ของ feed ที่สร้างขึ้น และแสดง:

- **Provider Account** — บัญชี feed ที่รายการนี้อยู่ใน
- **Format** — XML, CSV หรือ JSON
- **Product Count** — จำนวนสินค้าที่รวมอยู่
- **Size** — ขนาดไฟล์ของ feed ที่สร้างขึ้น
- **Generated At** — เมื่อใดที่สร้างขึ้น
- **Expires At** — เมื่อใดที่เวอร์ชันที่เก็บไว้นี้จะหมดอายุ
- **Status** — ว่า feed ยังมีความถูกต้องหรือไม่ หรือหมดอายุแล้ว
- **Download Count** — จำนวนครั้งที่ feed นี้ถูกดาวน์โหลด

feed ทั้งหมดเป็นเพียงแค่การอ่านในแดชบอร์ด — ถูกสร้างขึ้นโดยอัตโนมัติผ่านกระบวนการ sync

## การดูประวัติ sync

ไปที่ **การตลาด > Feed Sync Logs** เพื่อดูประวัติการ sync ทั้งหมดสำหรับบัญชี feed ทั้งหมดของคุณ แต่ละรายการบันทึก:

- บัญชี provider ที่ถูก sync
- ประเภท sync (Full, Incremental, Manual หรือ Scheduled)
- สถานะ (Success, Partial Success, Failed, ฯลฯ)
- สินค้าที่ sync, ล้มเหลว และข้าม
- ระยะเวลา sync
- ข้อความข้อผิดพลาดใด ๆ

แดชบอร์ด log sync ที่ด้านบนของหน้าแสดงสถิติโดยรวม: จำนวน sync ทั้งหมด, อัตราความสำเร็จ และระยะเวลา sync ค่าเฉลี่ย ใช้ตัวกรอง **Account** และ **Sync Type** เพื่อจำกัดให้เฉพาะ feed ที่ต้องการ

### ทำอะไรเมื่อ sync ล้มเหลว

1. ไปที่ **การตลาด > Feed Sync Logs** และหารายการที่ล้มเหลว
2. คลิกเข้าไปที่รายการ log เพื่อดู **ข้อความข้อผิดพลาด** และ **รายละเอียดข้อผิดพลาด** ทั้งหมด
3. สาเหตุที่พบบ่อยมีดังนี้:
   - ไม่มีฟิลด์สินค้าที่จำเป็น (ชื่อ, ราคา, รูปภาพ)
   - ข้อมูล API ที่ไม่ถูกต้องหรือหมดอายุ — ติดตั้งส่วนประกอบ provider ใหม่เพื่ออัปเดตข้อมูล
   - ข้อผิดพลาดเครือข่ายในการเชื่อมต่อกับ API ของ provider
4. เมื่อปัญหาได้รับการแก้ไขแล้ว sync ที่กำหนดไว้ครั้งต่อไปจะทำงานอัตโนมัติ หรือคุณสามารถเรียก sync แบบมือโดยตรงจากบัญชี provider

## เคล็ดลับ

รักษาการจัดรูปแบบ markdown, ค่าเส้นทางรูปภาพ, บล็อกโค้ด และคำศัพท์ทางเทคนิคไว้ทั้งหมด

- ตั้งค่า `"sync_interval": "daily"` สำหรับกรณีใช้งานทั่วไป — Google และ Facebook ไม่จำเป็นต้องอัปเดตบ่อยกว่านี้ เว้นแต่คุณมีความผันผวนของราคาที่สูงมาก
- รวม `"in_stock_only": true` เสมอในตัวกรองผลิตภัณฑ์ของคุณ เพื่อหลีกเลี่ยงการโฆษณาผลิตภัณฑ์ที่ลูกค้าไม่สามารถซื้อได้
- ใช้ชื่อบัญชีที่อธิบายได้ชัดเจนซึ่งรวมถึงแพลตฟอร์มและตลาดเป้าหมาย (เช่น `Google Shopping — UK`) เพื่อให้จัดการฟีดหลายตัวได้ง่ายขึ้น
- จำนวน **Products in Feed** บนบัญชีผู้ให้บริการจะแจ้งให้คุณทราบทันทีหากมีผลิตภัณฑ์ที่ถูกนำเข้ามาในฟีดน้อยกว่าที่คาดไว้ — ตรวจสอบการตั้งค่าตัวกรองผลิตภัณฑ์ของคุณหากจำนวนดูน้อยเกินไป
- ระบุบัญชีหนึ่งเป็น **Primary Feed** สำหรับแต่ละประเภทผู้ให้บริการ; เครื่องมือรายงานบางตัวใช้ข้อมูลนี้เพื่อระบุฟีดหลักของคุณ
- ตรวจสอบบันทึกการซิงค์หลังจากทำการเปลี่ยนแปลงจำนวนมากในแคตตาล็อกผลิตภัณฑ์ของคุณ เพื่อยืนยันว่าข้อมูลที่อัปเดตถูกจับได้อย่างถูกต้อง
