---
title: 製品フィード
---

製品フィードを使用すると、Google ShoppingやFacebook Catalogなどのショッピングプラットフォームにカタログをエクスポートできます。接続後、製品データはスケジュールに従って自動的に同期されるため、広告は常に現在の価格、在庫、製品の詳細を反映します。

あなたのストアは、フィード用のプロバイダーコンポーネントシステムを使用しています。各フィードプロバイダ（Google、Facebook、またはその他のプロバイダー）はコンポーネントとしてインストールされ、プロバイダーアカウントを通じて接続されます。複数のフィードプロバイダーを同時に実行できます。たとえば、Google Shopping用のフィードとFacebook用の別々のフィードを実行できます。

## フィードプロバイダーの接続

カタログを同期できるようになる前に、少なくとも1つのフィードプロバイダーコンポーネントをインストールし、接続する必要があります。

### プロバイダーコンポーネントのインストール

プロバイダーコンポーネントはSpwigコンポーネントマーケットプレイスで利用可能です。ストア管理者はコンポーネント更新システムを通じてインストールします。プロバイダーコンポーネントがインストールされると、フィードプロバイダー アカウントを作成するときにオプションとして表示されます。

### フィードプロバイダー アカウントの作成

1. **マーケティング > フィードプロバイダー** に移動します
2. **+ Feed Provider Account を追加** をクリックします
3. フォームを入力します:

**プロバイダー情報セクション:**
- **サイト** — あなたのストアを選択します（1つだけあります）
- **プロバイダーコンポーネント** — インストール済みのフィードプロバイダーを選択します（例: Google Shopping、Facebook Catalog）
- **アカウント名** — 説明的な名前を入力します。例: `Google Shopping — Main` または `Facebook Catalog — US`

**設定セクション:**
- **有効** — フィードの生成と同期を有効にするためにチェックします
- **プライマリ** — このプラットフォームタイプの主要なフィードプロバイダーである場合はチェックします
- **優先度** — リストでの並び順を制御します（数字が小さいほど先に表示されます）
- **設定** — プロバイダー固有の設定（以下を参照）

4. **保存** をクリックします

### フィード設定オプション

**設定** フィールドは、以下のオプションを持つJSONオブジェクトを受け入れます:

すべてのマークダウンフォーマット、画像パス、コードブロック、技術用語を保持してください。

| オプション | 値 | 説明 |
|--------|--------|-------------|
| `sync_interval` | `hourly`, `daily`, `weekly`, `manual` | チャネルが自動的に再生成される頻度 |
| `format_preference` | `xml`, `csv`, `json` | 出力形式（ほとんどのプラットフォームがXMLを好む） |
| `include_variants` | `true` / `false` | 製品のバリエーションを個別のチャネルエントリとして含める |
| `target_country` | 国コード例: "US" | チャネルの対象国 |
| `content_language` | 言語コード例: "en" | 製品データの言語 |

#### 米国を対象とした毎日のXMLチャネルの例:

```json
{
  "sync_interval": "daily",
  "format_preference": "xml",
  "include_variants": true,
  "target_country": "US",
  "content_language": "en"
}
```

## チャネルに表示される製品のフィルタリング

チャネルに表示される製品を正確に制御するには、configに`product_filter`セクションを追加します:

```json
{
  "product_filter": {
    "status": ["published"],
    "in_stock_only": true,
    "categories": [1, 5, 12]
  }
}
```

| フィルタオプション | 説明 |
|---------------|-------------|
| `status` | これらのステータスを持つみのみを含める。`["published"]`を使用して、ライブ製品のみを含める。
| `in_stock_only` | `true`に設定して、在庫切れの製品を除外する |
| `categories` | 特定のカテゴリIDに限定 |
| `brands` | 特定のブランドIDに限定 |

`exclude_products`を使用して、特定の製品IDを除外することもできます:

```json
{
  "exclude_products": [42, 87, 103]
}
```

## シンクステータスの確認

チャネルプロバイダーアカウント一覧は、接続されたチャネルのシンクステータスを一目で確認できます:

- **PENDING** — シンクがまだ実行されていない、またはチャネルが生成を待っている
- **SYNCING** — 現在シンクが進行中
- **SUCCESS** — 最後のシンクがエラーなく完了
- **ERROR** — 最後のシンクに失敗; エラーメッセージはアカウント詳細ページに表示されます

一覧には、現在のチャネルに含まれる製品数と、最後のシンクが実行された時間も表示されます。


 **マーケティング > プロダクトフィード** に移動して、生成されたフィードファイルを確認してください。

各行は1つの生成されたフィードスナップショットを表し、以下を表示します：

- **プロバイダーアカウント** — このフィードが所属するアカウント
- **形式** — XML、CSV、またはJSON
- **商品数** — 含まれている商品数
- **サイズ** — 生成されたフィードのファイルサイズ
- **生成日時** — 作成された時刻
- **有効期限** — このキャッシュバージョンの有効期限
- **ステータス** — フィードがまだ有効かどうか、または期限切れかどうか
- **ダウンロード回数** — このフィードが何回ダウンロードされたか

管理者画面ではフィードは読み取り専用です — これらは同期プロセスによって自動的に生成されます。

## シンクロ履歴の確認

 **マーケティング > フィードシンクロログ** に移動して、すべてのフィードアカウントのすべてのシンクロ試行の完全な履歴を確認してください。各行のログは以下を記録します：

- シンクロされたプロバイダーアカウント
- シンクロタイプ（フル、インクリメンタル、手動、またはスケジュール済み）
- ステータス（成功、部分的成功、失敗など）
- シンクロされた商品、失敗した商品、スキップされた商品
- シンクロの所要時間
- 任意のエラーメッセージ

ページの上部にあるシンクロログダッシュボードは、全体的な統計を表示します：総シンクロ数、成功率、平均シンクロ時間。 **アカウント** と **シンクロタイプ** のフィルタを使用して、特定のフィードに絞り込むことができます。

### シンクロが失敗した場合の対処法

1. **マーケティング > フィードシンクロログ** に移動し、失敗したエントリを確認します
2. ログエントリをクリックして、完全な **エラーメッセージ** と **エラーディテール** を表示します
3. 一般的な原因には以下が含まれます：
   - 必須の商品フィールドが欠落しています（タイトル、価格、画像）
   - 無効または期限切れのAPI資格情報 — プロバイダーコンポーネントを再インストールして資格情報を更新してください
   - プロバイダーアプリケーションのAPIに接続するネットワークエラー
4. 問題が解決したら、次のスケジュール済みシンクロが自動的に実行されます。または、プロバイダーアカウントから手動でシンクロをトリガーできます。

## ヒント

すべてのマークダウンの書式、画像のパス、コードブロック、および技術用語を保持してください。

- ほとんどのユースケースでは `"sync_interval": "daily"` を設定してください — Google および Facebook は価格の変動が非常に高い場合を除いて、より頻繁な更新は必要ありません
- プロダクトフィルタに常に `"in_stock_only": true` を含めてください。これにより、顧客が購入できない商品を広告しないようにします
- プラットフォームとターゲット市場を含む記述的なアカウント名（例: `Google Shopping — UK`）を使用してください。これにより、複数のフィードを管理するのが簡単になります
- プロバイダーアカウントの **Products in Feed** 数は、予想より少ない商品が含まれているかどうかをすぐに確認できます — 数が低く見える場合は、プロダクトフィルタ設定を確認してください
- 各プロバイダータイプに対して1つのアカウントを **Primary Feed** としてマークしてください。一部のレポートツールはこれを使ってあなたの主要なフィードを特定します
- 商品カタログに大規模な変更を行った後は、同期ログを確認して、更新されたデータが正しく取得されていることを確認してください
