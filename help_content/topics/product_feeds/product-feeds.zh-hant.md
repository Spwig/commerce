---
title: 產品清單
---

產品清單讓您可以將目錄匯出到購物平台，例如 Google Shopping 和 Facebook Catalog。一旦連接，您的產品資料會按照預設的時間表自動同步，因此您的廣告始終反映您目前的價格、庫存和產品細節。

您的商店使用提供者組件系統來管理清單。每個清單提供者（Google、Facebook 或其他）都作為組件安裝，然後通過提供者帳戶進行連接。您可以同時運行多個清單提供者——例如，一個用於 Google Shopping，另一個用於 Facebook。

## 連接清單提供者

在您可以同步目錄之前，需要安裝並連接至少一個清單提供者組件。

### 安裝提供者組件

提供者組件可在 Spwig 組件市場中找到。您的商店管理員通過組件更新系統安裝它們。一旦安裝了提供者組件，它就會在創建清單提供者帳戶時顯示為一個選項。

### 創建清單提供者帳戶

1. 進入 **行銷 > 清單提供者**
2. 點擊 **+ 添加清單提供者帳戶**
3. 填寫表單：

**提供者資訊部分：**
- **站點** — 選擇您的商店（只有一個）
- **提供者組件** — 選擇已安裝的清單提供者（例如，Google Shopping、Facebook Catalog）
- **帳戶名稱** — 一個描述性名稱，例如 `Google Shopping — 主要` 或 `Facebook Catalog — 美國`

**配置部分：**
- **是否啟用** — 勾選以啟用清單生成和同步
- **是否主要** — 勾選如果這是您此平台類型的主要清單提供者
- **優先順序** — 控制列表中的排序順序（數字較小的會先顯示）
- **配置** — 提供者特定的設置（請參閱下文）

4. 點擊 **保存**

查看生成的提要

前往 **行銷 > 產品供應源** 以查看生成的供應源文件。

每個項目代表一個生成的供應源快照，顯示：

- **供應商帳戶** — 這個供應源屬於哪個帳戶
- **格式** — XML、CSV 或 JSON
- **產品數量** — 包含的產品數量
- **大小** — 生成的供應源文件大小
- **生成時間** — 創建時間
- **過期時間** — 這個緩存版本的過期時間
- **狀態** — 供應源是否仍然有效或已過期
- **下載次數** — 這個供應源已被下載的次數

在管理介面中，供應源是只讀的 — 它們由同步過程自動生成。

## 查看同步歷史記錄

前往 **行銷 > 供應源同步日誌** 以查看所有供應源帳戶的完整同步嘗試歷史記錄。每個日誌項目會記錄：

- 被同步的供應商帳戶
- 同步類型（完整、增量、手動或預約）
- 狀態（成功、部分成功、失敗等）
- 同步的產品數量、失敗的產品數量和跳過的產品數量
- 同步持續時間
- 任何錯誤訊息

頁面頂部的同步日誌儀表板顯示整體統計數據：總共的同步次數、成功率和平均同步持續時間。使用 **帳戶** 和 **同步類型** 過濾器來縮小到特定的供應源。

### 當同步失敗時該怎麼辦

1. 前往 **行銷 > 供應源同步日誌** 並找到失敗的項目
2. 點擊日誌項目以查看完整的 **錯誤訊息** 和 **錯誤詳情**
3. 常見原因包括：
   - 缺少必要的產品欄位（標題、價格、圖片）
   - 無效或過期的 API 權限 — 重新安裝供應商組件以刷新權限
   - 連接到供應商 API 的網路錯誤
4. 一旦問題解決，下一次預約的同步將自動執行，或者您可以從供應商帳戶觸發手動同步

## 小技巧

保留所有 markdown 格式、圖片路徑、程式碼塊和技術術語。

- 將 `"sync_interval": "daily"` 用於大多數情況 — Google 和 Facebook 除非價格波動極大，否則不需要更頻繁的更新
- 在產品過濾條件中始終包含 `"in_stock_only": true`，以避免廣告推廣客戶無法購買的產品
- 使用包含平台和目標市場的描述性帳號名稱（例如：`Google Shopping — UK`），以便於管理多個供應商帳號
- 提供商帳號上的 **Products in Feed** 數量可立即告訴你是否有少於預期的產品被包含 — 如果數量看起來偏低，請檢查產品過濾條件設定
- 為每種供應商類型標記一個帳號為 **Primary Feed**；某些報告工具會使用這個標記來識別您的主要供應商
- 在對產品目錄進行批量更改後，請查看同步日誌以確認更新的資料是否已正確抓取
