---
title: 제품 피드
---

제품 피드를 사용하면 Google 쇼핑 및 Facebook 카탈로그와 같은 쇼핑 플랫폼으로 카탈로그를 내보낼 수 있습니다. 연결된 후 제품 데이터는 일정에 따라 자동으로 동기화되어 광고가 항상 현재 가격, 재고 및 제품 세부 정보를 반영합니다.

쇼핑몰은 피드에 제공업체 구성 요소 시스템을 사용합니다. 각 피드 제공업체(Google, Facebook 또는 기타)는 구성 요소로 설치되고 제공업체 계정을 통해 연결됩니다. 여러 피드 제공업체를 동시에 실행할 수 있습니다. 예를 들어 Google 쇼핑용 피드와 Facebook용 별도의 피드를 실행할 수 있습니다.

## 피드 제공업체 연결

카탈로그를 동기화할 수 있게 되기 전에 최소한 하나의 피드 제공업체 구성 요소를 설치하고 연결해야 합니다.

### 제공업체 구성 요소 설치

제공업체 구성 요소는 Spwig 구성 요소 시장에서 이용할 수 있습니다. 쇼핑몰 관리자는 구성 요소 업데이트 시스템을 통해 설치합니다. 제공업체 구성 요소가 설치되면 피드 제공업체 계정을 생성할 때 옵션으로 나타납니다.

### 피드 제공업체 계정 생성

1. **마케팅 > 피드 제공업체**로 이동합니다.
2. **+ 피드 제공업체 계정 추가**를 클릭합니다.
3. 양식을 작성합니다:

**제공업체 정보 섹션:**
- **사이트** — 쇼핑몰을 선택합니다(한 개만 있음)
- **제공업체 구성 요소** — 설치된 피드 제공업체를 선택합니다(예: Google 쇼핑, Facebook 카탈로그)
- **계정 이름** — `Google 쇼핑 — 메인` 또는 `Facebook 카탈로그 — US`와 같은 설명적인 이름

**설정 섹션:**
- **활성화** — 피드 생성 및 동기화를 활성화하려면 선택합니다
- **기본 제공업체** — 이 플랫폼 유형의 주 피드 제공업체인지 선택합니다
- **우선순위** — 목록에서의 정렬 순서를 결정합니다(숫자가 낮을수록 먼저 표시됨)
- **설정** — 제공업체별 설정(아래 참조)

4. **저장**을 클릭합니다.

### 피드 설정 옵션

**설정** 필드는 다음 옵션을 포함하는 JSON 객체를 수락합니다:

Preserve all markdown formatting, image paths, code blocks, and technical terms.

| 옵션 | 값 | 설명 |
|--------|--------|-------------|
| `sync_interval` | `hourly`, `daily`, `weekly`, `manual` | 피드가 자동으로 재생성되는 빈도 |
| `format_preference` | `xml`, `csv`, `json` | 출력 형식 (대부분의 플랫폼은 XML을 선호함) |
| `include_variants` | `true` / `false` | 제품 변형을 별도의 피드 항목으로 포함 |
| `target_country` | 국가 코드 예: "US" | 피드의 대상 국가 |
| `content_language` | 언어 코드 예: "en" | 제품 데이터의 언어 |

#### 미국을 대상으로 하는 매일 XML 피드의 예시 구성:

```json
{
  "sync_interval": "daily",
  "format_preference": "xml",
  "include_variants": true,
  "target_country": "US",
  "content_language": "en"
}
```

## 피드에 나타나는 제품을 필터링하기

구성을 통해 정확히 어떤 제품이 포함되는지 제어할 수 있습니다: `product_filter` 섹션을 추가하여:

```json
{
  "product_filter": {
    "status": ["published"],
    "in_stock_only": true,
    "categories": [1, 5, 12]
  }
}
```

| 필터 옵션 | 설명 |
|---------------|-------------|
| `status` | 이러한 상태를 가진 제품만 포함합니다. `["published"]`를 사용하여 활성 제품만 포함합니다. |
| `in_stock_only` | `true`로 설정하여 재고가 없는 제품을 제외합니다 |
| `categories` | 특정 카테고리 ID로 제한 |
| `brands` | 특정 브랜드 ID로 제한 |

`exclude_products`를 사용하여 특정 제품을 ID로 제외할 수도 있습니다:

```json
{
  "exclude_products": [42, 87, 103]
}
```

## 동기화 상태 모니터링

피드 제공업체 계정 목록은 연결된 각 피드의 동기화 상태를 한눈에 볼 수 있습니다:

- **PENDING** — 아직 동기화가 실행되지 않았거나 피드가 생성을 기다리고 있음
- **SYNCING** — 현재 동기화가 진행 중임
- **SUCCESS** — 마지막 동기화가 오류 없이 완료됨
- **ERROR** — 마지막 동기화가 실패함; 오류 메시지는 계정 세부 정보 페이지에 표시됨

목록은 현재 피드의 제품 수와 마지막 동기화가 실행된 시간도 표시합니다.


메뉴에서 **마케팅 > 제품 피드**로 이동하여 생성된 피드 파일을 확인할 수 있습니다.

각 항목은 생성된 피드 스냅샷을 나타내며 다음 정보를 표시합니다:

- **공급업체 계정** — 이 피드가 속한 계정
- **포맷** — XML, CSV 또는 JSON
- **제품 수** — 포함된 제품 수
- **크기** — 생성된 피드의 파일 크기
- **생성 시간** — 생성된 시점
- **만료 시간** — 이 캐시 버전이 만료되는 시점
- **상태** — 피드가 여전히 유효한지 또는 만료되었는지
- **다운로드 횟수** — 이 피드가 몇 번 다운로드되었는지

관리자에서 피드는 읽기 전용입니다 — 이는 동기화 프로세스에 의해 자동으로 생성됩니다.

## 동기화 기록 보기

메뉴에서 **마케팅 > 피드 동기화 로그**로 이동하여 모든 피드 계정의 모든 동기화 시도의 전체 기록을 확인할 수 있습니다. 각 로그 항목은 다음 정보를 기록합니다:

- 동기화된 공급업체 계정
- 동기화 유형 (전체, 증분, 수동, 예약)
- 상태 (성공, 부분 성공, 실패 등)
- 동기화된 제품, 실패한 제품, 건너뛴 제품
- 동기화 지속 시간
- 오류 메시지(있는 경우)

페이지 상단의 동기화 로그 대시보드는 전체 통계를 표시합니다: 총 동기화 횟수, 성공률, 평균 동기화 지속 시간. **계정** 및 **동기화 유형** 필터를 사용하여 특정 피드에 좁혀보세요.

### 동기화가 실패했을 때 해야 할 일

1. 메뉴에서 **마케팅 > 피드 동기화 로그**로 이동하여 실패한 항목을 찾습니다
2. 로그 항목을 클릭하여 전체 **오류 메시지** 및 **오류 세부 정보**를 확인합니다
3. 일반적인 원인은 다음과 같습니다:
   - 필수 제품 필드 누락 (제목, 가격, 이미지)
   - 유효하지 않거나 만료된 API 인증 정보 — 공급업체 구성 요소를 다시 설치하여 인증 정보를 새로 고칩니다
   - 공급업체의 API에 연결하는 네트워크 오류
4. 문제가 해결되면 다음 예약된 동기화가 자동으로 실행되거나, 공급업체 계정에서 수동 동기화를 트리거할 수 있습니다

## 팁

모든 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어를 유지하세요.

- 대부분의 경우 `"sync_interval": "daily"`를 설정하세요 — 가격 변동성이 매우 높지 않다면 Google 및 Facebook은 더 자주 업데이트할 필요가 없습니다
- 제품 필터에 항상 `"in_stock_only": true`를 포함하여 고객이 구매할 수 없는 제품을 광고하는 것을 피하세요
- 플랫폼과 타겟 시장을 포함한 설명적인 계정 이름(예: `Google Shopping — UK`)을 사용하여 여러 피드를 관리하는 것이 쉽도록 하세요
- 제공업체 계정의 **Products in Feed** 수는 예상보다 적은 제품이 포함되는지 즉시 알려줍니다 — 수치가 낮아 보이면 제품 필터 설정을 확인하세요
- 각 제공업체 유형에 대해 하나의 계정을 **Primary Feed**로 표시하세요; 일부 보고 도구는 이를 통해 주 피드를 식별합니다
- 제품 카탈로그에 대량 변경을 수행한 후에 동기화 로그를 확인하여 업데이트된 데이터가 올바르게 수집되었는지 확인하세요
