---
title: Обновления платформы
---

Ваша установка Spwig состоит из набора компонентов — тем, виджетов, интеграций, элементов конструктора страниц и подключений к поставщикам — каждый из которых имеет свою версию, которую можно обновлять независимо. Регистр компонентов предоставляет централизованный обзор всего установленного, показывает, какие компоненты имеют ожидающие обновления, и позволяет устанавливать или откатывать обновления в любое время.

![Обзор Регистра компонентов](/static/core/admin/img/help/platform-updates/component-registry-overview.webp)

## Понимание регистра компонентов

Перейдите в **Extensions > Регистр компонентов**, чтобы увидеть каждый установленный компонент. Каждая строка показывает:

- **Название** — отображаемое имя компонента
- **Тип** — какой тип компонента (тема, виджет, интеграция и т.д.)
- **Текущая версия** — версия, которая в настоящее время работает на вашем магазине
- **Состояние обновления** — доступно ли обновление
- **Канал** — какой канал обновления следует компоненту
- **Автоматическое обновление** — обновления устанавливаются автоматически
- **Заблокировано** — компонент заморожен на текущей версии

Панель в верхней части страницы показывает сводные счетчики: общее количество установленных компонентов, сколько из них имеют доступные обновления, и сколько из них актуальны.

### Типы компонентов

| Тип | Что это такое |
|------|------------|
| Тема | Визуальный дизайн вашего магазина |
| Виджет | Повторно используемые блоки конструктора страниц |
| Элемент конструктора страниц | Пользовательские элементы для конструктора страниц |
| Утилита конструктора страниц | Инструменты и утилиты редактора |
| Шаблон заголовка/подвал | Макеты заголовка и подвала |
| Поставщик доставки | Интеграции с перевозчиками (FedEx, UPS и т.д.) |
| Поставщик электронной почты | Сервисы доставки электронной почты |
| Поставщик оплаты | Интеграции с платежными шлюзами |
| Поставщик курса обмена | Источники данных курса обмена |
| Поставщик перевода | Сервисы искусственного интеллекта для перевода |
| Пакет языков | Файлы перевода интерфейса |

## Каналы обновлений

Каждый компонент следует за каналом обновлений, который определяет, какие выпуски он получает. Вы можете назначить каждый компонент разным каналам в зависимости от того, насколько вы готовы к риску.

| Канал | Описание | Лучше всего подходит |
|---------|-------------|----------|
| **Stable** | Готовые к производству, тщательно протестированные выпуски | Все компоненты на живых магазинах |
| **Beta** | Предварительные сборки для тестирования новых функций перед их стабилизацией | Не критические компоненты, которые вы хотите предварительно просмотреть |
| **Development** | Последние функции, которые могут быть нестабильными | Только для сред тестирования |
| **Security** | Только критические исправления безопасности, доставляемые с наивысшим приоритетом | Компоненты, где стабильность имеет первостепенное значение |

Чтобы изменить канал компонента, нажмите на его имя, чтобы открыть подробный вид, затем выберите новое значение в поле **Канал обновления** и сохраните.

## Проверка обновлений

Spwig автоматически проверяет наличие обновлений с интервалом, настроенным в настройках вашего сервера обновлений (по умолчанию: каждые 24 часа). Чтобы проверить немедленно:

1. Перейдите в **Extensions > Регистр компонентов**
2. Нажмите кнопку **Проверить обновления** в верхней части страницы
3. Система обращается к серверу обновлений Spwig и обновляет состояние обновления для всех компонентов
4. Компоненты с доступными обновлениями выделяются, и счетчик **Доступны обновления** обновляется

Вы также можете запустить проверку обновления для отдельных компонентов с помощью действия **Проверить обновления** в меню действий списка.

## Установка обновлений

### Обновление одного компонента

1. Перейдите в **Extensions > Регистр компонентов**
2. Найдите компонент, который вы хотите обновить — компоненты с доступными обновлениями показывают индикатор обновления рядом с их версией
3. Нажмите кнопку **Установить обновление** в строке этого компонента
4. Подтвердите обновление, когда будет предложено
5. Обновление загружается, проверяется и устанавливается — индикатор прогресса показывает каждый этап
6. После завершения версия компонента **Текущая версия** обновляется до нового номера версии

### Обновление нескольких компонентов

1.

Выберите флажки рядом с компонентами, которые вы хотите обновить
2.



Выберите **Установить обновления** в раскрывающемся списке **Действие**
3.

Нажмите **Go**, чтобы продолжить
4.

Обновления устанавливаются в порядке зависимости — компоненты, на которые зависят другие, обновляются первыми

### Что происходит во время обновления

Процесс обновления проходит через следующие этапы:

1. **Проверка** — подтверждает, что обновление доступно, и ваша лицензия действительна
2. **Загрузка** — получает пакет с сервера обновлений Spwig
3. **Проверка целостности** — проверяет целостность пакета с помощью контрольной суммы SHA-256
4. **Распаковка** — распаковывает новые файлы
5. **Развертывание** — активирует новую версию
6. **Проверка состояния** — проверяет, работает ли компонент после обновления

Если на каком-либо этапе произойдет сбой, система автоматически попытается восстановить предыдущую версию.

## Обновления на уровне платформы

Помимо отдельных компонентов, Spwig может получать обновления на уровне платформы, которые обновляют ядро магазина. Эти обновления проходят более тщательный процесс, включающий миграции базы данных и краткий период обслуживания.

История обновлений платформы доступна в разделе **Platform Updates** (Обновления платформы) в реестре. Каждая запись показывает переход версии (например, `v1.3.2 → v1.3.3`), статус и продолжительность процесса обновления.

Безопасные обновления помечаются отдельно, и если в настройках сервера обновлений включено **Auto Install Security Updates** (Автоматическая установка безопасных обновлений), они устанавливаются автоматически, без необходимости ручного вмешательства.

## Просмотр истории версий

Чтобы увидеть все ранее установленные версии компонента:

1. Нажмите на имя компонента, чтобы открыть его подробный вид
2. Прокрутите вниз до раздела **Component Versions** (Версии компонента) в нижней части страницы
3. Каждая запись версии показывает номер версии, время установки, метод установки и состояние работоспособности

Система сохраняет последние три установленные версии для отката. Версии, выходящие за этот предел, автоматически удаляются.

## Откат компонента

Если обновление вызывает проблемы, вы можете откатиться к предыдущей версии:

1. Откройте подробный вид компонента
2. Прокрутите вниз до раздела **Rollback** (Откат)
3. Выберите версию, которую вы хотите восстановить
4. Нажмите **Roll Back to this Version** (Откатиться к этой версии)

Только версии, помеченные как **Rollback Available** (Откат доступен), могут быть восстановлены. Запись журнала отката фиксирует, кто инициировал откат и когда это произошло.

## Блокировка компонентов

Блокировка компонента предотвращает установку любых обновлений, включая автоматические. Это полезно, если у вас есть пользовательские настройки или интеграции, зависящие от определенной версии.

1. Откройте подробный вид компонента
2. В разделе **Lock & Freeze** (Блокировка и заморозка) установите флажок **Locked** (Заблокировано)
3. В поле **Lock Reason** (Причина блокировки) введите причину, чтобы ваша команда понимала, почему компонент заблокирован
4. Сохраните запись

Блокированные компоненты отображаются с индикатором блокировки в списке реестра. Чтобы разблокировать, снимите флажок **Locked** (Заблокировано) и сохраните.

## Чтение журналов обновлений

Журнал обновлений фиксирует каждую установку, обновление, откат и проверку состояния:

1. Откройте подробный вид компонента
2. **Update Logs** (Журналы обновлений) отображаются в нижней части страницы
3. Каждая запись показывает: выполненное действие, время начала и окончания, старую и новую версии, автоматическое или ручное выполнение, а также любые сообщения об ошибках, если операция завершилась неудачно

Записи журналов с статусом **Failed** (Неудачно) включают полное сообщение об ошибке, чтобы помочь в устранении неполадок.

## Включение автоматических обновлений

Вы можете разрешить Spwig автоматически устанавливать обновления, когда они станут доступны:

1. Откройте подробный вид компонента
2. В разделе **Version & Update Status** (Версия и статус обновления) установите флажок **Auto Update** (Автоматическое обновление)
3. Сохраните запись

При включении автоматического обновления система устанавливает обновления в следующем расписании проверок. Безопасные обновления следуют глобальному настройке **Auto Install Security Updates** (Автоматическая установка безопасных обновлений), независимо от настроек отдельных компонентов.

## Советы

Сохраните всю разметку Markdown, пути к изображениям, блоки кода и технические термины.

- Обновляйте всегда через канал **Stable** для тем и платежных провайдеров — это самые клиентоориентированные компоненты, и стабильность здесь имеет первостепенное значение
- Зафиксируйте компонент перед внесением пользовательских изменений в него, и четко запишите причину, чтобы будущие члены команды знали, что не нужно обновлять его
- Перед установкой значительного обновления версии проверьте **Release Notes** в записи версии компонента — там отмечаются разрывные изменения
- После обновления перейдите в соответствующий раздел вашего магазина, чтобы убедиться, что всё выглядит и работает как ожидалось, прежде чем объявлять обновление завершенным
- Если для компонента включен автоматический обновление, периодически проверяйте **Update Logs**, чтобы убедиться, что автоматические обновления завершаются успешно
