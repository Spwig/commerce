---
title: 平台更新
---

您的 Spwig 安装由一组组件构成 —— 主题、小部件、集成、页面构建器元素和提供商连接 —— 每个组件都有自己的版本，可以独立更新。组件注册表为您提供一个集中视图，显示所有已安装的内容，展示哪些组件有等待更新的版本，并允许您随时安装或回滚更新。

![组件注册表概览](/static/core/admin/img/help/platform-updates/component-registry-overview.webp)

## 了解组件注册表

导航到 **扩展 > 组件注册表**，查看商店中安装的所有组件。每一行显示：

- **名称** — 组件的显示名称
- **类型** — 组件的种类（主题、小部件、集成等）
- **当前版本** — 当前在您的商店中运行的版本
- **更新状态** — 是否有可用更新
- **通道** — 组件遵循的更新通道
- **自动更新** — 是否自动安装更新
- **锁定** — 组件是否被固定在当前版本

页面顶部的仪表板显示汇总统计：已安装组件总数、有多少组件有可用更新，以及有多少组件已更新至最新版本。

### 组件类型

| 类型 | 说明 |
|------|------------|
| 主题 | 商店的视觉设计 |
| 小部件 | 可重复使用的页面构建器模块 |
| 页面构建器元素 | 页面构建器的自定义元素 |
| 页面构建器工具 | 编辑器工具和实用程序 |
| 页眉/页脚模板 | 页眉和页脚布局 |
| 配送提供商 | 运输商集成（如 FedEx、UPS 等） |
| 邮件提供商 | 邮件投递服务 |
| 支付提供商 | 支付网关集成 |
| 汇率提供商 | 货币汇率数据源 |
| 翻译提供商 | AI 翻译服务 |
| 语言包 | 界面翻译文件 |

## 更新通道

每个组件都遵循一个更新通道，该通道控制它接收哪些发布版本。

你可以根据你能接受的风险程度，将每个组件分配到不同的通道中。

| 通道 | 描述 | 最适合 |
|---------|-------------|----------|
| **稳定版** | 生产就绪、经过充分测试的版本 | 所有在生产环境中使用的组件 |
| **测试版** | 在版本稳定前用于测试新功能的预发布构建 | 你希望预览的非关键组件 |
| **开发版** | 最新功能，可能不稳定 | 仅限测试环境 |
| **安全版** | 仅包含关键的安全补丁，以最高优先级提供 | 稳定性至关重要的组件 |

要更改组件的通道，请点击其名称以打开详细视图，然后在 **更新通道** 字段中选择新值并保存。

## 检查更新

Spwig 会根据你在更新服务器设置中配置的时间间隔（默认：每 24 小时）自动检查更新。要立即检查：

1. 导航到 **扩展 > 组件注册表**
2. 点击页面顶部的 **检查更新** 按钮
3. 系统会联系 Spwig 更新服务器，并刷新所有组件的更新状态
4. 有可用更新的组件会高亮显示，并更新 **有可用更新** 的计数

你也可以通过列表操作菜单中的 **检查更新** 操作，触发单个组件的更新检查。

## 安装更新

### 更新单个组件

1. 导航到 **扩展 > 组件注册表**
2. 找到你想要更新的组件 —— 有可用更新的组件会在其版本旁边显示更新指示器
3. 点击该组件行上的 **安装更新** 按钮
4. 确认更新
5. 更新会下载、验证并安装 —— 进度指示器会显示每个阶段
6. 一旦完成，该组件的 **当前版本** 会更新为新的版本号

### 更新多个组件

1.

选择你想要更新的组件旁边的复选框
2.



从 **Action** 下拉菜单中选择 **Install updates**
3.

点击 **Go** 以继续
4.

更新按照依赖顺序进行 —— 被其他组件依赖的组件会优先更新

### 更新过程中会发生什么

更新过程会经历以下阶段：

1. **Checking** —— 确认更新可用且您的许可证有效
2. **Downloading** —— 从 Spwig 更新服务器获取软件包
3. **Verifying** —— 通过 SHA-256 校验和检查软件包的完整性
4. **Extracting** —— 解压新文件
5. **Deploying** —— 激活新版本
6. **Health check** —— 更新后验证组件是否正常工作

如果任何阶段失败，系统会自动尝试恢复到之前的版本。

## 平台级别的更新

除了单个组件外，Spwig 还可以接收平台级别的更新，这些更新会升级核心商店引擎。这些更新会经过更严格的流程，包括数据库迁移和短暂的维护窗口。

平台更新历史可在注册表的 **Platform Updates** 部分查看。每个条目显示版本转换（例如，`v1.3.2 → v1.3.3`）、状态和更新过程的持续时间。

安全更新会单独标记，如果在更新服务器配置中启用了 **Auto Install Security Updates**，则会自动安装，无需手动操作。

## 查看版本历史

要查看组件的所有先前安装版本：

1. 点击组件名称以打开其详细视图
2. 滚动到页面底部的 **Component Versions** 部分
3. 每个版本条目显示版本号、安装时间、安装方法和其健康状态

系统保留最近安装的三个版本以供回滚。超出此范围的版本会自动删除。

## 回滚组件

如果更新导致问题，您可以回滚到之前的版本：

1.

打开组件的详细视图
2.


滚动到 **回滚** 部分
3.

选择您要恢复的版本
4.

点击 **回滚到此版本**

只有标记为 **可回滚** 的版本才能恢复。回滚日志条目记录了谁发起的回滚以及时间。

## 锁定组件

锁定组件可防止安装任何更新，包括自动更新。当您有依赖特定版本的自定义或集成时，这非常有用。

1. 打开组件的详细视图
2. 在 **锁定与冻结** 部分勾选 **锁定** 复选框
3. 在 **锁定原因** 中输入原因，以便您的团队了解为何冻结
4. 保存记录

锁定的组件在注册表列表中会显示一个锁标识。要解锁，请取消勾选 **锁定** 并保存。

## 阅读更新日志

更新日志记录了每次安装、更新、回滚和健康检查操作：

1. 打开组件的详细视图
2. **更新日志** 会以内联方式显示在页面底部
3. 每个条目显示：执行的操作、开始和结束时间、旧版本和新版本、是自动还是手动执行的，以及如果操作失败时的任何错误信息

状态为 **失败** 的日志条目包含完整的错误信息，以帮助进行故障排除。

## 启用自动更新

您可以允许 Spwig 在更新可用时自动安装更新：

1. 打开组件的详细视图
2. 在 **版本与更新状态** 部分勾选 **自动更新**
3. 保存记录

启用自动更新后，系统将在下一个计划检查周期内安装更新。安全更新将遵循全局 **自动安装安全更新** 设置，而不管单个组件的设置如何。

## 小贴士

保留所有 Markdown 格式、图片路径、代码块和专业术语。

- 始终在 **Stable** 频道上更新主题和支付提供商 —— 这些是最面向客户的组件，稳定性最为重要
- 在对组件进行自定义修改之前，请先将其锁定，并清楚记录原因，以便未来的团队成员知道不要更新它
- 在安装主要版本升级之前，请查看组件版本条目中的 **Release Notes** —— 所有破坏性变更都会在那里标记
- 更新后，请访问您商店受影响的区域，确认一切看起来和功能正常，然后再声明更新完成
- 如果组件启用了自动更新，请定期查看 **Update Logs**，以确保自动更新能够成功完成
