---
title: 플랫폼 업데이트
---

Spwig 설치는 테마, 위젯, 통합, 페이지 빌더 요소 및 제공업체 연결과 같은 구성 요소의 모음으로 구성되어 있으며, 각 구성 요소는 독립적으로 업데이트할 수 있는 자체 버전을 가지고 있습니다. 구성 요소 레지스트리에서는 설치된 모든 항목에 대한 중앙 뷰를 제공하며, 업데이트가 대기 중인 구성 요소를 표시하고 언제든지 업데이트를 설치하거나 롤백할 수 있습니다.

![구성 요소 레지스트리 개요](/static/core/admin/img/help/platform-updates/component-registry-overview.webp)

## 구성 요소 레지스트리 이해

**확장 기능 > 구성 요소 레지스트리**로 이동하여 상점에 설치된 모든 구성 요소를 확인할 수 있습니다. 각 행은 다음과 같은 정보를 표시합니다:

- **이름** — 구성 요소의 표시 이름
- **유형** — 구성 요소의 종류(테마, 위젯, 통합 등)
- **현재 버전** — 현재 상점에서 실행 중인 버전
- **업데이트 상태** — 업데이트가 사용 가능한지 여부
- **채널** — 구성 요소가 따르는 업데이트 채널
- **자동 업데이트** — 업데이트가 자동으로 설치되는지 여부
- **고정됨** — 구성 요소가 현재 버전에 고정되어 있는지 여부

페이지 상단의 대시보드는 요약된 수치를 표시합니다: 설치된 구성 요소의 총 수, 업데이트가 사용 가능한 구성 요소 수, 최신 상태인 구성 요소 수.

### 구성 요소 유형

| 유형 | 설명 |
|------|------------|
| 테마 | 상점의 시각적 디자인 |
| 위젯 | 재사용 가능한 페이지 빌더 블록 |
| 페이지 빌더 요소 | 페이지 빌더용 커스텀 요소 |
| 페이지 빌더 유틸리티 | 편집기 도구 및 유틸리티 |
| 헤더/푸터 템플릿 | 헤더 및 푸터 레이아웃 |
| 배송 제공업체 | 운송업체 통합 (FedEx, UPS 등) |
| 이메일 제공업체 | 이메일 전송 서비스 |
| 결제 제공업체 | 결제 게이트웨이 통합 |
| 환율 제공업체 | 통화율 데이터 소스 |
| 번역 제공업체 | AI 번역 서비스 |
| 언어 패키지 | 인터페이스 번역 파일 |

## 업데이트 채널

각 구성 요소는 업데이트 채널을 따르며, 이 채널은 해당 구성 요소가 받는 릴리스를 제어합니다.

각 구성 요소를 안정성 수준에 따라 다른 채널에 할당할 수 있습니다.

| 채널 | 설명 | 최적의 사용 대상 |
|---------|-------------|----------|
| **Stable** | 프로덕션 환경에서 사용할 수 있으며, 철저히 테스트된 릴리스 | 실제 매장에서 사용하는 모든 구성 요소 |
| **Beta** | 안정 버전이 되기 전에 새로운 기능을 테스트하기 위한 사전 릴리스 빌드 | 미리 보기하고 싶은 비중요 구성 요소 |
| **Development** | 최신 기능, 불안정할 수 있음 | 테스트 환경에만 사용 |
| **Security** | 중요한 보안 패치만, 최고 우선순위로 제공 | 안정성이 필수적인 구성 요소 |

구성 요소의 채널을 변경하려면 해당 이름을 클릭하여 세부 정보 뷰를 열고, **Update Channel**(채널 업데이트) 필드에서 새 값을 선택한 후 저장합니다.

## 업데이트 확인

Spwig은 업데이트 서버 설정에서 구성된 간격(기본값: 24시간마다)에 따라 자동으로 업데이트를 확인합니다. 즉시 확인하려면:

1. **Extensions > Component Registry**(확장 기능 > 구성 요소 레지스트리)로 이동
2. 페이지 상단의 **Check for Updates**(업데이트 확인) 버튼을 클릭
3. 시스템이 Spwig 업데이트 서버에 연결하여 모든 구성 요소의 업데이트 상태를 새로 고침
4. 업데이트가 가능한 구성 요소는 강조 표시되고, **Updates Available**(가능한 업데이트) 카운트가 업데이트됩니다

리스트의 액션 메뉴에서 **Check for Updates**(업데이트 확인) 액션을 사용하여 개별 구성 요소에 대한 업데이트 확인을 트리거할 수도 있습니다.

## 업데이트 설치

### 단일 구성 요소 업데이트

1. **Extensions > Component Registry**(확장 기능 > 구성 요소 레지스트리)로 이동
2. 업데이트할 구성 요소를 찾습니다 — 업데이트가 가능한 구성 요소는 버전 번호 옆에 업데이트 지시 표시가 나타납니다
3. 해당 구성 요소의 행에서 **Install Update**(업데이트 설치) 버튼을 클릭
4. 확인 메시지가 표시되면 업데이트를 확인합니다
5. 업데이트가 다운로드, 검증 및 설치됩니다 — 각 단계에 대한 진행 표시가 나타납니다
6. 완료되면 구성 요소의 **Current Version**(현재 버전)이 새 버전 번호로 업데이트됩니다

### 여러 구성 요소 업데이트

1.

업데이트하려는 구성 요소 옆의 체크박스를 선택
2.


업데이트를 설치하려면 **Action** 드롭다운에서 **Install updates**를 선택하세요
3.

**Go**를 클릭하여 진행합니다
4.

의존성 순서에 따라 업데이트가 설치됩니다 — 다른 구성 요소에 의존하는 구성 요소가 먼저 업데이트됩니다

### 업데이트 중에 발생하는 일

업데이트 프로세스는 다음 단계를 거칩니다:

1. **Checking** — 업데이트가 사용 가능한지 확인하고 라이선스가 유효한지 확인합니다
2. **Downloading** — Spwig 업데이트 서버에서 패키지를 다운로드합니다
3. **Verifying** — SHA-256 체크섬을 기준으로 패키지의 무결성을 확인합니다
4. **Extracting** — 새 파일을 압축 해제합니다
5. **Deploying** — 새 버전을 활성화합니다
6. **Health check** — 업데이트 후 구성 요소가 정상 작동하는지 확인합니다

어떤 단계든 실패하면 시스템은 자동으로 이전 버전으로 복원을 시도합니다.

## 플랫폼 수준 업데이트

개별 구성 요소뿐만 아니라 Spwig는 핵심 스토어 엔진을 업데이트하는 플랫폼 수준 업데이트도 받을 수 있습니다. 이러한 업데이트는 데이터베이스 마이그레이션과 짧은 유지보수 시간을 포함한 더 철저한 프로세스를 거칩니다.

플랫폼 업데이트 기록은 레지스트리의 **Platform Updates** 섹션에서 확인할 수 있습니다. 각 항목은 버전 전환(예: `v1.3.2 → v1.3.3`), 상태, 업데이트 프로세스의 지속 시간을 표시합니다.

보안 업데이트는 별도로 표시되며, 업데이트 서버 구성에서 **Auto Install Security Updates**가 활성화되어 있으면 수동 작업 없이 자동으로 설치됩니다.

## 버전 기록 보기

구성 요소의 이전에 설치된 모든 버전을 보려면:

1. 구성 요소 이름을 클릭하여 세부 정보 뷰를 엽니다
2. 페이지 하단의 **Component Versions** 섹션으로 스크롤합니다
3. 각 버전 항목은 버전 번호, 설치 시간, 설치 방법, 건강 상태를 표시합니다

시스템은 마지막으로 설치된 세 버전을 롤백용으로 유지합니다. 그 이상의 버전은 자동으로 삭제됩니다.

## 구성 요소 롤백

업데이트가 문제가 발생하면 이전 버전으로 롤백할 수 있습니다:

1.

구성 요소의 세부 정보 뷰를 엽니다
2.


롤백 섹션으로 스크롤하세요
3.

복원할 버전을 선택하세요
4.

**이 버전으로 롤백**을 클릭하세요

**롤백 가능**으로 표시된 버전만 복원할 수 있습니다. 롤백 로그 항목은 롤백을 시작한 사람과 시간을 기록합니다.

## 구성 요소 잠금

구성 요소를 잠그면 자동 업데이트 포함 모든 업데이트가 설치되는 것을 방지합니다. 특정 버전에 의존하는 커스터마이징 또는 통합이 있는 경우 이 기능이 유용합니다.

1. 구성 요소의 상세 뷰를 엽니다
2. **Lock & Freeze** 섹션에서 **Locked** 체크박스를 선택합니다
3. **Lock Reason**에 잠그는 이유를 입력하여 팀이 왜 동결되었는지 이해하도록 합니다
4. 기록을 저장합니다

잠긴 구성 요소는 레지스트리 목록에서 잠금 지시자로 표시됩니다. 해제하려면 **Locked**를 선택 해제하고 저장합니다.

## 업데이트 로그 확인

업데이트 로그는 설치, 업데이트, 롤백 및 건강 상태 확인 작업을 기록합니다:

1. 구성 요소의 상세 뷰를 엽니다
2. **Update Logs**는 페이지 하단에 인라인으로 표시됩니다
3. 각 항목은 다음과 같은 정보를 보여줍니다: 수행된 작업, 시작 및 종료 시간, 이전 및 새 버전, 자동 또는 수동 여부, 작업이 실패한 경우 오류 메시지

**실패** 상태의 로그 항목은 문제 해결에 도움이 되는 전체 오류 메시지를 포함합니다.

## 자동 업데이트 활성화

Spwig이 업데이트가 제공될 때마다 자동으로 설치하도록 허용할 수 있습니다:

1. 구성 요소의 상세 뷰를 엽니다
2. **Version & Update Status** 섹션에서 **Auto Update**를 선택합니다
3. 기록을 저장합니다

자동 업데이트가 활성화되면 시스템은 다음 예약된 확인 주기 동안 업데이트를 설치합니다. 보안 업데이트는 개별 구성 요소 설정과 관계없이 전역 **Auto Install Security Updates** 설정을 따릅니다.

## 팁

모든 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어를 유지하세요.

- 테마와 결제 제공업체는 고객과 직접적으로 관련된 구성 요소이므로 **Stable** 채널에서 항상 업데이트하세요. 이 구성 요소들은 안정성이 가장 중요합니다.
- 구성 요소를 수정하기 전에 먼저 해당 구성 요소를 잠급니다. 그리고 수정 이유를 명확히 기록하여 향후 팀원들이 업데이트하지 않도록 알 수 있도록 해주세요.
- 주요 버전 업데이트를 설치하기 전에 구성 요소의 버전 정보에서 **릴리스 노트**를 확인하세요. 이곳에 중단 변경 사항이 표시되어 있습니다.
- 업데이트 후에 상점의 영향을 받은 영역으로 이동하여 모든 것이 예상대로 보이고 작동하는지 확인한 후 업데이트가 완료되었음을 선언하세요.
- 구성 요소에서 자동 업데이트가 활성화된 경우, 주기적으로 **업데이트 로그**를 확인하여 자동 업데이트가 성공적으로 완료되고 있는지 확인하세요.
