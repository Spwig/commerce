title: 平台更新
---

您的 Spwig 安裝是由一系列組件組成 — 主題、小部件、整合、頁面構建器元素和提供者連接 — 每個組件都有自己的版本，可以獨立更新。組件倉庫為您提供一個集中視圖，顯示所有已安裝的內容，顯示哪些組件有待更新，並讓您隨時安裝或回滾更新。

![組件倉庫概覽](/static/core/admin/img/help/platform-updates/component-registry-overview.webp)

## 理解組件倉庫

導航至 **Extensions > 組件倉庫** 以查看商店中安裝的每個組件。每一行顯示：

- **名稱** — 組件的顯示名稱
- **類型** — 組件的類型（主題、小部件、整合等）
- **當前版本** — 目前在您的商店中運行的版本
- **更新狀態** — 是否有可用更新
- **頻道** — 組件遵循的更新頻道
- **自動更新** — 是否自動安裝更新
- **鎖定** — 組件是否固定在當前版本

頁面頂部的儀表板顯示總結數量：已安裝的組件總數、有多少個有可用更新，以及有多少個已更新。

### 組件類型

| 類型 | 是什麼 |
|------|------------|
| 主題 | 您商店的視覺設計 |
| 小部件 | 可重複使用的頁面構建器區塊 |
| 頁面構建器元素 | 頁面構建器的自定義元素 |
| 頁面構建器工具 | 編輯器工具和實用程式 |
| 頁首/頁尾模板 | 頁首和頁尾的佈局 |
| 運輸提供者 | 載運商整合（FedEx、UPS 等） |
| 電子郵件提供者 | 電子郵件傳遞服務 |
| 支付提供者 | 支付網關整合 |
| 汇率提供者 | 貨幣匯率數據來源 |
| 翻譯提供者 | AI 翻譯服務 |
| 語言包 | 介面翻譯文件 |

## 更新頻道

每個組件都遵循一個更新頻道，該頻道控制它接收哪些發行版本。

你可以根據自己能接受的風險程度，將每個組件分配到不同的頻道。

| 頻道 | 說明 | 最適合用於 |
|---------|-------------|----------|
| **穩定版** | 已準備好用於生產環境，經過充分測試的版本 | 所有在線商店上的組件 |
| **測試版** | 在正式發佈前用於測試新功能的預發佈版本 | 你想預覽的非關鍵組件 |
| **開發版** | 最新的功能，可能不穩定 | 僅限測試環境使用 |
| **安全版** | 僅包含關鍵的安全修補程式，以最高優先順序提供 | 穩定性至關重要的組件 |

要更改組件的頻道，請點擊其名稱以打開詳細視圖，然後在 **更新頻道** 欄位中選擇新值並保存。

## 檢查更新

Spwig 會根據你在更新伺服器設定中配置的間隔自動檢查更新（預設：每 24 小時一次）。若要立即檢查：

1. 進入 **Extensions > Component Registry**
2. 點擊頁面上方的 **Check for Updates** 按鈕
3. 系統會聯繫 Spwig 更新伺服器，並刷新所有組件的更新狀態
4. 有可用更新的組件會被標記出來，並更新 **Updates Available** 的數量

你也可以透過列表的動作選單中使用 **Check for Updates** 動作，來觸發單個組件的更新檢查。

## 安裝更新

### 更新單個組件

1. 進入 **Extensions > Component Registry**
2. 找到你想要更新的組件 — 有可用更新的組件會在其版本旁顯示更新指示符
3. 點擊該組件行中的 **Install Update** 按鈕
4. 當提示時確認更新
5. 更新會下載、驗證並安裝 — 進度指示器會顯示每個階段
6. 一旦完成，該組件的 **Current Version** 會更新為新的版本號

### 更新多個組件

1.

選中你想要更新的組件旁的複選框
2.



從 **Action** 下拉選單中選擇 **安裝更新**
3.

點擊 **Go** 以繼續
4.

更新會按照依賴順序進行 — 其他組件所依賴的組件會先更新

### 更新過程中會發生什麼

更新流程會經過以下階段：

1. **檢查** — 確認更新可用且您的許可證有效
2. **下載** — 從 Spwig 更新伺服器獲取套件
3. **驗證** — 將套件完整性與 SHA-256 校驗和進行比對
4. **解壓** — 解壓新文件
5. **部署** — 啟用新版本
6. **健康檢查** — 更新後驗證組件是否正常運作

如果任何階段失敗，系統會自動嘗試恢復到之前的版本。

## 平台級更新

除了個別組件外，Spwig 還可以接收平台級更新，這些更新會更新核心商店引擎。這些更新會經過更嚴謹的流程，包括資料庫遷移和短暫的維護時間。

平台更新歷史可在註冊表的 **Platform Updates** 區域查看。每個條目會顯示版本過渡（例如，`v1.3.2 → v1.3.3`）、狀態以及更新過程的持續時間。

安全更新會單獨標記，如果在更新伺服器配置中啟用了 **Auto Install Security Updates**，則會自動安裝，無需手動操作。

## 查看版本歷史

要查看組件的所有先前安裝版本：

1. 點擊組件名稱以打開其詳細視圖
2. 向下滾動到頁面底部的 **Component Versions** 區域
3. 每個版本條目會顯示版本號、安裝時間、安裝方法及其健康狀態

系統會保留最後三個已安裝的版本以供回滾。超過這個範圍的版本會自動刪除。

## 回滾組件

如果更新導致問題，您可以回滾到之前的版本：

1.

打開組件的詳細視圖
2.


滾動到 **Rollback** 區段
3.

選擇您要還原的版本
4.

點擊 **Roll Back to this Version**

只有標記為 **Rollback Available** 的版本才能還原。還原日誌條目會記錄誰啟動了還原以及時間。

## Locking components

鎖定組件可防止安裝任何更新，包括自動更新。當您在特定版本上有自定義或整合時，這會很有用。

1. 開啟組件的詳細視圖
2. 在 **Lock & Freeze** 區段中勾選 **Locked** 核取方塊
3. 在 **Lock Reason** 中輸入原因，讓您的團隊了解為什麼被凍結
4. 儲存記錄

被鎖定的組件會在註冊表列表中顯示鎖定指示器。要解除鎖定，取消勾選 **Locked** 並儲存。

## Reading update logs

更新日誌會記錄每次安裝、更新、還原和健康檢查操作：

1. 開啟組件的詳細視圖
2. **Update Logs** 會在頁面底部內聯顯示
3. 每個條目顯示：採取的動作、開始和結束時間、舊版和新版、是否為自動或手動，以及如果操作失敗時的任何錯誤訊息

狀態為 **Failed** 的日誌條目會包含完整的錯誤訊息，以協助進行疑難排解。

## Enabling auto-updates

您可以允許 Spwig 在更新可用時自動安裝更新：

1. 開啟組件的詳細視圖
2. 在 **Version & Update Status** 區段中勾選 **Auto Update**
3. 儲存記錄

啟用自動更新後，系統會在下一次排程檢查週期中安裝更新。安全更新會遵循全域的 **Auto Install Security Updates** 設定，與個別組件設定無關。

## Tips

保留所有 markdown 格式、圖片路徑、程式碼區塊和技術術語。

始終在 **Stable** 頻道更新主題和支付提供商 —— 這些是最面向用戶的組件，穩定性至關重要

在對組件進行自定義修改之前，先將其鎖定，並清楚記錄原因，以便未來團隊成員知道不要更新它

在安裝主要版本升級之前，查看組件版本條目中的 **Release Notes** —— 重大更改會在那裡標記

更新後，瀏覽您商店中受影響的區域，確認一切看起來和運作都符合預期，再宣告更新完成

如果組件啟用了自動更新，請定期查看 **Update Logs**，以確保自動更新能夠成功完成
