---
title: การอัปเดตแพลตฟอร์ม
---

การติดตั้ง Spwig ของคุณถูกสร้างขึ้นจากชุดขององค์ประกอบ — ธีม, วิดเจ็ต, การผสาน, องค์ประกอบสร้างหน้า, และการเชื่อมต่อกับผู้ให้บริการ — แต่ละส่วนมีเวอร์ชันของตัวเองที่สามารถอัปเดตได้อย่างอิสระ รีจิสตรีองค์ประกอบให้คุณมีมุมมองแบบรวมศูนย์เกี่ยวกับสิ่งที่ติดตั้งไว้ทั้งหมด แสดงรายการองค์ประกอบที่มีการอัปเดตรออยู่ และอนุญาตให้คุณติดตั้งหรือย้อนกลับการอัปเดตได้ตลอดเวลา

![ภาพรวมรีจิสตรีองค์ประกอบ](/static/core/admin/img/help/platform-updates/component-registry-overview.webp)

## การทำความเข้าใจรีจิสตรีองค์ประกอบ

ไปที่ **Extensions > Component Registry** เพื่อดูองค์ประกอบทั้งหมดที่ติดตั้งไว้ในร้านค้าของคุณ แต่ละแถวแสดง:

- **ชื่อ** — ชื่อแสดงขององค์ประกอบ
- **ประเภท** — ประเภทขององค์ประกอบ (ธีม, วิดเจ็ต, การผสาน, ฯลฯ)
- **เวอร์ชันปัจจุบัน** — เวอร์ชันที่กำลังทำงานอยู่ในร้านค้าของคุณ
- **สถานะการอัปเดต** — ว่ามีการอัปเดตพร้อมใช้งานหรือไม่
- **ช่องทาง** — ช่องทางการอัปเดตที่องค์ประกอบติดตาม
- **อัปเดตอัตโนมัติ** — ว่าการอัปเดตจะติดตั้งอัตโนมัติหรือไม่
- **ล็อก** — ว่าองค์ประกอบถูกแช่แข็งไว้ที่เวอร์ชันปัจจุบันหรือไม่

แดชบอร์ดด้านบนของหน้าแสดงจำนวนสรุป: จำนวนองค์ประกอบทั้งหมดที่ติดตั้งไว้, จำนวนที่มีการอัปเดตพร้อมใช้งาน, และจำนวนที่อัปเดตแล้ว

### ประเภทองค์ประกอบ

| ประเภท | หมายถึงอะไร |
|-------|-------------|
| ธีม | ดีไซน์ภาพรวมของร้านค้า |
| วิดเจ็ต | บล็อกสร้างหน้าที่สามารถใช้ซ้ำได้ |
| องค์ประกอบสร้างหน้า | องค์ประกอบเฉพาะสำหรับสร้างหน้า |
| เครื่องมือสร้างหน้า | เครื่องมือและเครื่องมือสำหรับตัวแก้ไข |
| แม่แบบหัว/ท้าย | รูปแบบหัวและท้าย |
| ผู้ให้บริการการจัดส่ง | การผสานกับผู้ให้บริการจัดส่ง (FedEx, UPS, ฯลฯ) |
| ผู้ให้บริการอีเมล | บริการส่งอีเมล |
| ผู้ให้บริการชำระเงิน | การผสานกับระบบชำระเงิน |
| ผู้ให้บริการอัตราแลกเปลี่ยน | แหล่งข้อมูลอัตราแลกเปลี่ยนสกุลเงิน |
| ผู้ให้บริการแปล | บริการแปลโดยใช้ AI |
| ชุดภาษา | ไฟล์แปลอินเทอร์เฟซ |

## ช่องทางการอัปเดต

องค์ประกอบทุกตัวติดตามช่องทางการอัปเดตที่ควบคุมการปล่อยที่มันได้รับ

คุณสามารถกำหนดแต่ละส่วนประกอบให้อยู่ในช่องทางที่แตกต่างกันได้ ตามระดับความเสี่ยงที่คุณรับได้

| ช่องทาง | คำอธิบาย | เหมาะสำหรับ |
|---------|-------------|----------|
| **Stable** | การปล่อยที่พร้อมใช้งานในระบบผลิต ได้รับการทดสอบอย่างละเอียด | ส่วนประกอบทั้งหมดในร้านค้าที่เปิดให้บริการ |
| **Beta** | การสร้างก่อนการปล่อย เพื่อทดสอบคุณสมบัติใหม่ก่อนที่จะเป็นเวอร์ชัน Stable | ส่วนประกอบที่ไม่สำคัญที่คุณต้องการลองใช้ |
| **Development** | คุณสมบัติล่าสุด อาจไม่มั่นคง | แค่ทดสอบในสภาพแวดล้อมการทดสอบ |
| **Security** | แพตช์ด้านความปลอดภัยที่สำคัญเท่านั้น ถูกส่งด้วยความสำคัญสูงสุด | ส่วนประกอบที่ความมั่นคงเป็นสิ่งสำคัญที่สุด |

เพื่อเปลี่ยนช่องทางของส่วนประกอบ ให้คลิกที่ชื่อของมันเพื่อเปิดมุมมองรายละเอียด แล้วเลือกค่าใหม่ในช่อง **Update Channel** และบันทึก\ n
## การตรวจสอบการอัปเดต

Spwig จะตรวจสอบการอัปเดตโดยอัตโนมัติทุกช่วงเวลาที่กำหนดไว้ในการตั้งค่าเซิร์ฟเวอร์อัปเดตของคุณ (ค่าเริ่มต้น: ทุก 24 ชั่วโมง) เพื่อตรวจสอบทันที:

1. ไปที่ **Extensions > Component Registry**
2. คลิกปุ่ม **Check for Updates** ที่ด้านบนของหน้าจอ
3. ระบบจะติดต่อเซิร์ฟเวอร์อัปเดตของ Spwig และอัปเดตสถานะการอัปเดตสำหรับส่วนประกอบทั้งหมด
4. ส่วนประกอบที่มีการอัปเดตสามารถใช้งานได้ จะแสดงเป็นตัวชี้วัด และจำนวน **Updates Available** จะอัปเดตตามจำนวนที่มี

คุณยังสามารถเรียกใช้การตรวจสอบการอัปเดตสำหรับส่วนประกอบแต่ละตัวโดยใช้การกระทำ **Check for Updates** จากเมนูการกระทำของรายการ

## การติดตั้งการอัปเดต

### การอัปเดตส่วนประกอบเดี่ยว

1. ไปที่ **Extensions > Component Registry**
2. หาส่วนประกอบที่คุณต้องการอัปเดต — ส่วนประกอบที่มีการอัปเดตจะแสดงตัวชี้วัดการอัปเดตข้างเวอร์ชันของมัน
3. คลิกปุ่ม **Install Update** บนแถวของส่วนประกอบนั้น
4. ยืนยันการอัปเดตเมื่อถูกขอ
5. การอัปเดตจะดาวน์โหลด ตรวจสอบ และติดตั้ง — ตัวชี้วัดความคืบหน้าจะแสดงแต่ละขั้นตอน
6. เมื่อเสร็จสิ้น หมายเลขเวอร์ชันปัจจุบันของส่วนประกอบจะอัปเดตเป็นหมายเลขเวอร์ชันใหม่

### การอัปเดตส่วนประกอบหลายตัว

1.

เลือกช่องถัดไปของส่วนประกอบที่คุณต้องการอัปเดต
2.



เลือก **Install updates** จากเมนู **Action**
3.

คลิก **Go** เพื่อดำเนินการต่อ
4.

การอัปเดตจะถูกติดตั้งตามลำดับการพึ่งพา — องค์ประกอบที่มีคนอื่นพึ่งพาจะอัปเดตก่อน

### สิ่งที่เกิดขึ้นระหว่างการอัปเดต

กระบวนการอัปเดตจะดำเนินการผ่านขั้นตอนต่อไปนี้:

1. **Checking** — ยืนยันว่ามีการอัปเดตพร้อมใช้งานและใบอนุญาตของคุณมีความถูกต้อง
2. **Downloading** — ดาวน์โหลดแพ็กเกจจากเซิร์ฟเวอร์อัปเดตของ Spwig
3. **Verifying** — ตรวจสอบความสมบูรณ์ของแพ็กเกจโดยใช้ checksum SHA-256
4. **Extracting** — แตกไฟล์ใหม่ออกมา
5. **Deploying** — เปิดใช้งานเวอร์ชันใหม่
6. **Health check** — ตรวจสอบว่าองค์ประกอบทำงานได้หลังจากการอัปเดต

หากขั้นตอนใดล้มเหลว ระบบจะพยายามกู้คืนเวอร์ชันก่อนหน้าโดยอัตโนมัติ

## การอัปเดตระดับแพลตฟอร์ม

นอกจากองค์ประกอบแต่ละตัวแล้ว Spwig ยังสามารถรับการอัปเดตระดับแพลตฟอร์มที่อัปเดตเครื่องมือหลักของร้านค้า กระบวนการเหล่านี้ผ่านขั้นตอนที่ละเอียดมากขึ้น รวมถึงการย้ายฐานข้อมูลและช่วงเวลาบำรุงรักษาสั้นๆ

ประวัติการอัปเดตแพลตฟอร์มสามารถเห็นได้ในส่วน **Platform Updates** ของ registry แต่ละรายการแสดงการเปลี่ยนเวอร์ชัน (เช่น `v1.3.2 → v1.3.3`) สถานะ และระยะเวลาของการอัปเดต

การอัปเดตด้านความปลอดภัยจะถูกทำเครื่องหมายแยกต่างหาก และหาก **Auto Install Security Updates** เปิดใช้งานในการตั้งค่าเซิร์ฟเวอร์อัปเดตของคุณ การอัปเดตจะติดตั้งโดยอัตโนมัติโดยไม่ต้องการการดำเนินการด้วยมือ

## การดูประวัติเวอร์ชัน

เพื่อดูเวอร์ชันที่ติดตั้งไว้ก่อนหน้านี้ทั้งหมดขององค์ประกอบ:

1. คลิกที่ชื่อองค์ประกอบเพื่อเปิดมุมมองรายละเอียด
2. ค้นหาส่วน **Component Versions** ที่ด้านล่างของหน้า
3. แต่ละรายการเวอร์ชันแสดงหมายเลขเวอร์ชัน วันที่ติดตั้ง วิธีการติดตั้ง และสถานะความสมบูรณ์ของระบบ

ระบบจะเก็บเวอร์ชันที่ติดตั้งล่าสุด 3 เวอร์ชันไว้สำหรับการกู้คืน เวอร์ชันที่เกินกว่านั้นจะถูกลบโดยอัตโนมัติ

## การกู้คืนเวอร์ชันขององค์ประกอบ

หากการอัปเดตทำให้เกิดปัญหา คุณสามารถกู้คืนไปยังเวอร์ชันก่อนหน้า:

1.

เปิดมุมมองรายละเอียดขององค์ประกอบ
2.

รักษาการจัดรูปแบบ markdown รูปภาพ โค้ดบล็อก และคำศัพท์ทางเทคนิคไว้ทั้งหมด

เลื่อนไปยังส่วน **Rollback**
3.

เลือกเวอร์ชันที่คุณต้องการกู้คืน
4.

คลิก **Roll Back to this Version**

เวอร์ชันที่มีเครื่องหมาย **Rollback Available** เท่านั้นที่สามารถกู้คืนได้ เวอร์ชันที่กู้คืนจะบันทึกว่าใครเป็นผู้เริ่มต้นการกู้คืนและเมื่อใด

## การล็อกส่วนประกอบ

การล็อกส่วนประกอบจะป้องกันไม่ให้มีการอัปเดตใด ๆ ถูกติดตั้ง รวมถึงการอัปเดตอัตโนมัติ ซึ่งมีประโยชน์เมื่อคุณมีการปรับแต่งหรือการผสานรวมที่ขึ้นอยู่กับเวอร์ชันเฉพาะ

1. เปิดหน้าต่างรายละเอียดของส่วนประกอบ
2. ตรวจสอบกล่อง **Locked** ในส่วน **Lock & Freeze**
3. ใส่เหตุผลใน **Lock Reason** เพื่อให้ทีมของคุณเข้าใจว่าทำไมจึงถูกล็อก
4. บันทึกข้อมูล

ส่วนประกอบที่ถูกล็อกจะแสดงด้วยตัวบ่งชี้ล็อกในรายการลงทะเบียน เพื่อปลดล็อก ให้ยกเลิกการตรวจสอบ **Locked** และบันทึกข้อมูล

## การอ่านบันทึกการอัปเดต

บันทึกการอัปเดตบันทึกการดำเนินการทุกครั้งที่ติดตั้ง อัปเดต การกู้คืน และการตรวจสอบสุขภาพ:

1. เปิดหน้าต่างรายละเอียดของส่วนประกอบ
2. **Update Logs** จะแสดงในบรรทัดล่างของหน้าจอ
3. แต่ละรายการแสดง: การดำเนินการที่ทำ เวลาเริ่มต้นและสิ้นสุด เวอร์ชันเก่าและใหม่ ว่าเป็นการดำเนินการอัตโนมัติหรือมือ รวมถึงข้อความข้อผิดพลาดหากการดำเนินการล้มเหลว

รายการบันทึกที่มีสถานะ **Failed** จะรวมข้อความข้อผิดพลาดทั้งหมดเพื่อช่วยในการแก้ไขปัญหา

## การเปิดใช้งานการอัปเดตอัตโนมัติ

คุณสามารถอนุญาตให้ Spwig ติดตั้งการอัปเดตอัตโนมัติเมื่อมีการอัปเดตพร้อมใช้งาน:

1. เปิดหน้าต่างรายละเอียดของส่วนประกอบ
2. ตรวจสอบ **Auto Update** ในส่วน **Version & Update Status**
3. บันทึกข้อมูล

เมื่อเปิดใช้งานการอัปเดตอัตโนมัติ ระบบจะติดตั้งการอัปเดตในรอบการตรวจสอบที่กำหนดไว้ล่วงหน้า การอัปเดตด้านความปลอดภัยจะปฏิบัติตามการตั้งค่า **Auto Install Security Updates** ทั่วโลกโดยไม่คำนึงถึงการตั้งค่าของส่วนประกอบแต่ละตัว

## เคล็ดลับ

รักษาการจัดรูปแบบมาร์กดาวน์ ค่าเส้นทางรูปภาพ บล็อกโค้ด และคำศัพท์ทางเทคนิคไว้ทั้งหมด

- เสมออัปเดตผ่านช่องทาง **Stable** สำหรับธีมและผู้ให้บริการชำระเงิน — ส่วนประกอบเหล่านี้เป็นส่วนที่ลูกค้าเห็นมากที่สุด และความเสถียรจึงมีความสำคัญที่สุด
- ล็อกส่วนประกอบก่อนทำการแก้ไขเฉพาะตัวใด ๆ และบันทึกเหตุผลอย่างชัดเจน เพื่อให้สมาชิกทีมในอนาคตทราบว่าไม่ควรอัปเดตส่วนประกอบนี้
- ตรวจสอบ **Release Notes** ในรายการเวอร์ชันของส่วนประกอบก่อนติดตั้งการอัปเกรดเวอร์ชันหลัก — การเปลี่ยนแปลงที่ทำลายระบบจะถูกทำเครื่องหมายไว้ที่นั่น
- หลังจากอัปเดต ให้เข้าไปที่ส่วนที่ได้รับผลกระทบของร้านค้าของคุณ เพื่อยืนยันว่าทุกอย่างดูและทำงานตามที่คาดไว้ก่อนประกาศว่าการอัปเดตเสร็จสิ้นแล้ว
