Spwigのインストールは、テーマ、ウィジェット、統合、ページビルダーエレメント、プロバイダ接続などのコンポーネントのコレクションから構成されています。それぞれのコンポーネントには、独立して更新できる独自のバージョンがあります。コンポーネントレジストリは、インストールされているすべてのものを中央で表示し、更新待ちのコンポーネントを示し、いつでも更新をインストールまたはロールバックできます。

![コンポーネントレジストリ概要](/static/core/admin/img/help/platform-updates/component-registry-overview.webp)

## コンポーネントレジストリの理解

**拡張機能 > コンポーネントレジストリ**に移動して、ストアにインストールされているすべてのコンポーネントを確認してください。各行は次の情報を表示します：

- **名前** — コンポーネントの表示名
- **タイプ** — コンポーネントの種類（テーマ、ウィジェット、統合など）
- **現在のバージョン** — ストアで現在実行されているバージョン
- **更新ステータス** — 更新が利用可能かどうか
- **チャネル** — コンポーネントが従う更新チャネル
- **自動更新** — 更新が自動でインストールされるかどうか
- **ロック** — コンポーネントが現在のバージョンで固定されているかどうか

ページの上部にあるダッシュボードは、合計でインストールされているコンポーネント数、更新が利用可能なコンポーネント数、最新のコンポーネント数のサマリカウントを表示します。

### コンポーネントタイプ

| タイプ | 何を表すか |
|------|------------|
| テーマ | ストアのビジュアルデザイン |
| ウィジェット | 再利用可能なページビルダーブロック |
| ページビルダーエレメント | ページビルダー用のカスタム要素 |
| ページビルダーユーティリティ | エディターツールとユーティリティ |
| ヘッダー/フッターテンプレート | ヘッダーやフッターのレイアウト |
| 配送プロバイダ | 輸送業者の統合（FedEx、UPSなど） |
| メールプロバイダ | メール配信サービス |
| 支払いプロバイダ | 支払いゲートウェイの統合 |
| 通貨レートプロバイダ | 通貨レートデータのソース |
| 翻訳プロバイダ | AI翻訳サービス |
| 言語パック | インターフェースの翻訳ファイル |

## 更新チャネル

すべてのコンポーネントは、どのリリースを受け取るかを制御する更新チャネルに従います。

各コンポーネントを、ご自身が受け入れ可能なリスクレベルに応じて異なるチャネルに割り当てることができます。

| チャネル | 説明 | 最適な用途 |
|---------|-------------|----------|
| **Stable** | 本番環境で使用可能な、完全にテスト済みのリリース | ライブストア上のすべてのコンポーネント |
| **Beta** | 本番リリースになる前に新機能をテストするためのプレリリースビルド | プレビューしたい非重要なコンポーネント |
| **Development** | 最新の機能が含まれていますが、不安定である可能性があります | テスト環境でのみ使用 |
| **Security** | 重要なセキュリティパッチのみ、最も高い優先度で配信 | 安定性が最優先事項であるコンポーネント |

コンポーネントのチャネルを変更するには、その名前をクリックして詳細ビューを開き、**Update Channel**フィールドで新しい値を選択し、保存してください。

## 更新の確認

Spwigは、更新サーバー設定で構成された間隔（デフォルト: 24時間ごと）で自動的に更新を確認します。すぐに確認するには:

1. **Extensions > Component Registry** に移動します
2. ページ上部の **Check for Updates** ボタンをクリックします
3. システムはSpwigの更新サーバーに接続し、すべてのコンポーネントの更新ステータスを更新します
4. 更新が利用可能なコンポーネントはハイライトされ、**Updates Available** のカウントが更新されます

リストのアクションメニューから **Check for Updates** アクションを使用して、個々のコンポーネントの更新確認をトリガーすることもできます。

## 更新のインストール

### 単一のコンポーネントの更新

1. **Extensions > Component Registry** に移動します
2. 更新したいコンポーネントを探します — 更新が利用可能なコンポーネントはバージョンの隣に更新インジケーターが表示されます
3. そのコンポーネントの行にある **Install Update** ボタンをクリックします
4. プロンプトが表示されたら更新を確認します
5. 更新がダウンロードされ、検証され、インストールされます — 各段階の進行状況インジケーターが表示されます
6. 完了すると、コンポーネントの **Current Version** が新しいバージョン番号に更新されます

### 複数のコンポーネントの更新

1.

更新したいコンポーネントの横にあるチェックボックスをクリックします
2.


アクションドロップダウンから **Install updates** を選択します

進むには **Go** をクリックします

更新は依存関係の順序でインストールされます — 他のコンポーネントに依存しているものはまず更新されます

更新中に何が起こるか

更新プロセスは次のステージを経ます:

1. **Checking** — 更新が利用可能であり、ライセンスが有効であることを確認します

2. **Downloading** — Spwig 更新サーバーからパッケージを取得します

3. **Verifying** — SHA-256 チェックサムに対してパッケージの整合性を確認します

4. **Extracting** — 新しいファイルを展開します

5. **Deploying** — 新しいバージョンを有効にします

6. **Health check** — 更新後、コンポーネントが正常に動作していることを確認します

どのステージでも失敗した場合、システムは自動的に以前のバージョンへの復元を試みます。

## プラットフォームレベルの更新

個々のコンポーネントに加えて、Spwig はコアストアエンジンを更新するプラットフォームレベルの更新を受けることができます。これらの更新は、データベースマイグレーションや短いメンテナンスウィンドウを含むより詳細なプロセスを経ます。

レジストリの **Platform Updates** セクションにプラットフォーム更新履歴が表示されます。各エントリには、バージョンの変遷（例: `v1.3.2 → v1.3.3`）、ステータス、および更新プロセスの所要時間が表示されます。

セキュリティ更新は個別にマークされ、更新サーバー構成で **Auto Install Security Updates** が有効になっている場合、手動操作を必要とせずに自動的にインストールされます。

## バージョン履歴の表示

コンポーネントの以前にインストールされたすべてのバージョンを確認するには:

1. コンポーネント名をクリックして詳細ビューを開きます

2. ページの下部にある **Component Versions** セクションにスクロールします

3. 各バージョンエントリには、バージョン番号、インストールされた日時、インストール方法、およびその健康状態が表示されます

システムは、ロールバックのために最後の3つのインストール済みバージョンを保持します。それ以降のバージョンは自動的に削除されます。

## コンポーネントのロック

コンポーネントをロックすると、自動的な更新を含め、すべての更新のインストールが防止されます。これは、特定のバージョンに依存するカスタマイズや統合がある場合に役立ちます。

1. コンポーネントの詳細ビューを開く
2. **ロック & フリーズ**セクションで **ロック** チェックボックスをオンにする
3. **ロック理由**に理由を入力し、チームがなぜ凍結されているのかを理解できるようにする
4. レコードを保存

ロックされたコンポーネントは、レジストリリストでロックアイコンで表示されます。ロックを解除するには、**ロック**をオフにして保存します。

## 更新ログの閲覧

更新ログは、すべてのインストール、更新、ロールバック、およびヘルスチェック操作を記録します：

1. コンポーネントの詳細ビューを開く
2. **更新ログ**はページの下部でインラインで表示されます
3. 各エントリは、実施されたアクション、開始および終了時間、古いバージョンと新しいバージョン、自動または手動で実行されたかどうか、操作が失敗した場合のエラーメッセージを表示します

**失敗**ステータスのログエントリには、トラブルシューティングを助けるために完全なエラーメッセージが含まれています。

## 自動更新の有効化

Spwigが更新が利用可能になるたびに自動的に更新をインストールできるように許可できます：

1. コンポーネントの詳細ビューを開く
2. **バージョン & 更新ステータス**セクションで **自動更新**をチェック
3. レコードを保存

自動更新が有効になっている場合、システムは次の予定されたチェックサイクル中に更新をインストールします。セキュリティ更新は、個々のコンポーネント設定に関係なく、グローバルな **セキュリティ更新の自動インストール**設定に従います。

## ヒント

すべてのマークダウンフォーマット、画像パス、コードブロック、および技術用語を保持してください。

- テーマや支払いプロバイダーについては、常に **Stable** チャンネルで更新してください — これらは顧客に最も直接的に関係するコンポーネントであり、安定性が最も重要です
- コンポーネントをカスタム変更する前にロックし、理由を明確に記録して、将来的なチームメンバーがそれを更新しないようにする理由を理解できるようにしてください
- メジャーバージョンアップをインストールする前に、コンポーネントのバージョンエントリにある **Release Notes** を確認してください — 重大な変更はそこでフラグ付けされています
- 更新後は、ストアの影響を受けた部分にアクセスして、すべてが予期通り表示され、動作していることを確認した後で、更新が完了したと宣言してください
- コンポーネントで自動更新が有効になっている場合は、定期的に **Update Logs** を確認して、自動更新が正常に完了していることを確認してください
