---
title: 电子邮件出站箱
---

电子邮件出站箱是您商店已发送或尝试发送的每封电子邮件的完整日志——包括订单确认、发货更新、管理员报告以及所有其他交易消息。您可以使用它来确认邮件是否已送达、调查失败情况以及管理电子邮件队列。

导航到 **电子邮件系统 > 电子邮件出站箱** 以查看电子邮件日志。

![带有状态徽章的电子邮件出站箱列表](/static/core/admin/img/help/email-outbox/outbox-list.webp)

## 阅读出站箱

顶部的摘要栏显示每个状态类别的数量。下方的列表显示单个电子邮件，包括：

- **主题** — 电子邮件的主题行
- **收件人** — 收件人的电子邮件地址
- **发件人** — 使用的发件人地址
- **状态** — 当前的投递状态
- **排队时间** — 电子邮件进入队列的时间
- **发送时间** — 电子邮件发送给提供商的时间
- **重试次数** — 已尝试发送的次数

## 电子邮件状态

| 状态 | 含义 |
|------|------|
| 排队中 | 电子邮件正在队列中等待发送 |
| 正在发送 | 电子邮件当前正在发送给提供商 |
| 已发送 | 提供商已接受电子邮件 |
| 已暂停 | 电子邮件已暂停，直到释放后才会发送 |
| 已记录 | 电子邮件已记录但未发送（测试模式或仅记录设置） |
| 已失败 | 提供商拒绝或无法投递电子邮件 |
| 已退信 | 电子邮件已发送但被收件人的邮件服务器退回 |
| 已跳过 | 由于系统原因跳过了发送

## 查看电子邮件详情

点击列表中的任何电子邮件以查看完整详情：

- 电子邮件的完整 **HTML正文** 和 **文本正文**
- **提供商消息ID** — 来自您电子邮件提供商的参考编号（在联系提供商支持时使用此编号）
- **错误信息** — 失败或退信电子邮件的确切错误信息
- **重试次数** 和 **最大重试次数** — 发送尝试的次数
- 所有时间戳：创建时间、排队时间、发送时间和失败时间

## 过滤出站箱

使用右侧的筛选器来缩小您的视图：

- **状态** — 显示特定投递状态的电子邮件
- **日期** — 按电子邮件创建或发送的时间进行筛选
- **模板类型** — 仅显示特定通知类型的电子邮件（例如，仅订单确认）

顶部的搜索框可以按主题、收件人地址、发件人地址或提供商消息ID进行搜索。

## 释放已暂停的电子邮件

状态为 **已暂停** 的电子邮件会被暂停 — 在您释放它们之前，它们不会被发送。如果商店在生成电子邮件时处于维护模式，或者管理员操作将其暂停，电子邮件可能会被暂停。

要释放已暂停的电子邮件：
1. 选择要释放的电子邮件（左侧的复选框）
2. 从 **操作** 下拉菜单中选择 **释放已暂停的电子邮件以发送**
3. 点击 **继续**

释放的电子邮件将进入 **已排队** 状态，并在下一个队列处理周期中发送。

## 计划中的电子邮件

一些电子邮件计划在将来某个时间发送 — 例如，每周摘要报告计划在特定日期和时间发送。导航到 **电子邮件系统 > 计划中的电子邮件** 以查看即将发送的计划邮件。

计划中的电子邮件列表显示：
- **模板类型** — 计划发送的电子邮件类型
- **收件人电子邮件** — 它将被发送到的地址
- **计划发送时间** — 它计划发送的日期和时间
- **状态** — 待发送（尚未发送）、已发送或发送失败

当计划时间到达时，计划中的电子邮件会自动处理 — 不需要手动操作。

## 修复发送失败的问题

如果电子邮件显示 **发送失败** 状态，请点击进入以查看错误信息，并按照以下步骤操作：

### 常见原因和解决方案

| 症状 | 可能原因 | 应对方法 |
|---------|-------------|------------|
| "认证失败" | 电子邮件提供商的凭据无效 | 在 **电子邮件系统 > 电子邮件账户** 中更新凭据 |
| "连接被拒绝" / "超时" | 无法访问您的电子邮件服务器 | 检查电子邮件提供商的状态页面；在 **电子邮件账户** 中测试连接 |
| "无效收件人" | 客户的电子邮件地址格式不正确 | 检查客户的账户并更正其电子邮件 |
| 被退回的电子邮件 | 收件人的邮件服务器拒绝了电子邮件 | 该地址可能不存在或其收件箱已满；不要过度重试 |
| 突然高失败率 | 提供商问题或凭据过期 | 检查提供商状态；在 **电子邮件账户** 中重新测试连接 |

### 检查您的电子邮件账户连接

如果许多电子邮件失败，请测试您的电子邮件账户：

1. 导航到 **电子邮件系统 > 电子邮件账户**
2. 找到您的活动账户并检查其 **连接** 状态
3. 如果连接显示错误，请点击账户并使用 **测试连接** 选项来诊断问题

### 重试行为

Spwig 会自动重试失败的电子邮件，最多达到 **最大重试次数** 限制。每封电子邮件上显示的重试次数会告诉您已尝试的次数。一旦达到重试限制，电子邮件将保持在 **失败** 状态，不再进行自动重试。

## 被退回的电子邮件

**被退回** 的电子邮件已发送但被收件人的邮件服务器退回。有两种退回类型：

- **硬退回** — 电子邮件地址不存在或域名不接受电子邮件。不要重试硬退回；该地址无效
- **软退回** — 临时问题（收件箱已满，服务器暂时不可用）。重试可能会成功

对同一地址的重复退回可能会损害您与电子邮件提供商的发件人声誉。如果您看到对同一客户地址的重复退回，请更新或从客户的账户中删除该地址。

## 小贴士

保留所有Markdown格式、图片路径、代码块和专业术语。

- 在大促活动或大型产品发布等重大事件后，检查出站邮箱，确认所有订单确认邮件是否已成功发送
- 如果客户表示未收到邮件，请通过其电子邮件地址在出站邮箱中搜索，查看邮件是否已发送、失败或跳过
- 失败率的突然上升通常表明凭证或账户存在问题 — 立即检查 **电子邮件账户**
- **已保留** 状态不是失败 — 它只是表示邮件正在等待。

在准备好发送保留的邮件时释放它们
- 使用 **模板类型** 过滤器快速审核某一类型的邮件 — 例如，检查过去7天内所有订单确认邮件是否具有 **已发送** 状态
- 列表顶部的日期层级导航（天 / 月 / 年）对于查看特定时间段的出站邮箱非常有用
