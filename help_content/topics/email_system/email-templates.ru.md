---
title: Шаблоны электронной почты
---

Шаблоны электронной почты управляют дизайном и содержимым каждой автоматически отправляемой электронной почты, которую ваш магазин отправляет клиентам и вам — подтверждения заказов, обновления доставки, сброс пароля, уведомления о возврате и многое другое. Редактирование шаблона изменит все будущие письма этого типа; предыдущие письма, уже находящиеся в очереди, не затронуты.

Перейдите в **Система электронной почты > Шаблоны электронной почты**, чтобы просмотреть и управлять своими шаблонами.

![Список шаблонов электронной почты](/static/core/admin/img/help/email-templates/templates-list.webp)

## Типы шаблонов

Ваш магазин включает шаблоны для широкого круга событий. Они сгруппированы по категориям:

### Электронные письма для клиентов
| Шаблон | Отправляется, когда |
|----------|-----------|
| Подтверждение заказа | Клиент завершает покупку |
| Подтверждение оплаты | Оплата успешно обработана |
| Заказ отправлен | Заказ помечен как отправленный |
| Подтверждение доставки | Добавлен номер отслеживания доставки |
| Подтверждение доставки | Заказ помечен как доставленный |
| Заказ отменен | Заказ отменен |
| Уведомление о задержке заказа | Задержка записана для заказа |
| Уведомление о возврате | Выдан возврат |

### Электронные письма учетной записи
| Шаблон | Отправляется, когда |
|----------|-----------|
| Добро пожаловать в учетную запись | Клиент создает учетную запись |
| Приглашение в учетную запись | Вы приглашаете клиента создать учетную запись |
| Проверка электронной почты | Клиент проверяет свой адрес электронной почты |
| Сброс пароля | Клиент запрашивает сброс пароля |

### Возвраты
| Шаблон | Отправляется, когда |
|----------|-----------|
| Возвраты: Получен запрос | Клиент подает заявку на возврат |
| Возвраты: Одобрен | Заявка на возврат одобрена |
| Возвраты: Отклонен | Заявка на возврат отклонена |
| Возвраты: Получена посылка | Возвращенный товар прибывает на ваше место |
| Возвраты: Выдан возврат | Выдан возврат по возврату |

### Уведомления администратора (отправляются вам)
| Шаблон | Отправляется, когда |
|----------|-----------|
| Администратор: Новый заказ | Размещен новый заказ |
| Администратор: Ошибка оплаты | Попытка оплаты не удалась |
| Администратор: Ежедневный отчет о продажах | Сформирован ежедневный отчет о продажах |
| Администратор: Предупреждение о низком запасе | Товар опускается ниже порога запаса |
| Администратор: Еженедельный обзор | Сформирован еженедельный обзор магазина |

Дополнительные шаблоны охватывают этапы отслеживания доставки, активность программы аффилиатов, подтверждения бронирования (если функция бронирования включена) и события программы лояльности.

## Редактирование шаблона

1. Перейдите в **Система электронной почты > Шаблоны электронной почты**
2. Найдите шаблон, который вы хотите отредактировать. Вы можете фильтровать по **Типу шаблона**, **Языку** или **Состоянию** с помощью фильтров справа
3. Нажмите на шаблон, чтобы открыть его
4. Отредактируйте **Тема** (тема письма, отображаемая в почтовом ящике клиента)
5. Отредактируйте **HTML-содержимое** для полной версии дизайна письма
6. Необязательно отредактировать **Текстовое содержимое** — это резервная текстовая версия для клиентов электронной почты, которые не поддерживают HTML
7. Нажмите **Сохранить**

> **Электронные письма HTML:** Поле HTML-содержимого принимает стандартный HTML, включая встроенный CSS. Spwig отображает это в правильно отформатированном письме. Если вы используете разметку MJML, она автоматически компилируется при сохранении.

## Предварительный просмотр шаблона

Перед сохранением вы можете предварительно просмотреть, как будет выглядеть шаблон в клиенте электронной почты:

1. Откройте шаблон, который вы хотите предварительно просмотреть
2. Нажмите кнопку **Предварительный просмотр** (видна в списке шаблонов или на странице деталей шаблона)
3. Предварительный просмотр открывается в новой вкладке браузера, показывающей отрендеренное письмо

Это позволяет проверить макет, форматирование и внешний вид переменных-заполнителей до того, как шаблон будет опубликован.

## Переменные шаблона

Переменные — это заполнители в вашем шаблоне, которые Spwig заменяет на реальные данные при отправке письма. Они записываются как `{{ variable_name }}`.

Часто используемые переменные, доступные в большинстве шаблонов:

| Переменная | Заменяется на |
|----------|---------------|
| `{{ customer_name }}` | Полное имя клиента |
| `{{ order_number }}` | Номер заказа |
| `{{ order_total }}` | Общая сумма заказа |
| `{{ store_name }}` | Название вашего магазина |
| `{{ store_url }}` | Веб-адрес вашего магазина |
| `{{ tracking_number }}` | Номер отслеживания посылки |
| `{{ tracking_url }}` | Кликабельная ссылка для отслеживания посылки |

Доступные переменные зависят от типа шаблона. Переменные, которые актуальны для шаблона, связанного с заказом (например, `{{ order_number }}`), недоступны в шаблоне учетной записи (например, Сброс пароля). Если вы включите переменную, которая не подходит, она будет отображаться как пустая или не заменится.

## Поддержка языков

Каждый тип шаблона может иметь версию на каждом языке, который поддерживает ваш магазин. Поля **Язык** на каждом шаблоне определяет, какая версия языка активна.

Spwig автоматически выбирает правильную версию языка на основе предпочтений клиента при отправке. Если для языка клиента не существует шаблона, Spwig переключается на английскую версию.

Чтобы добавить шаблон для нового языка:
1. Откройте существующий шаблон
2. Нажмите **Клонировать шаблон** в меню **Действия**
3. Установите код языка на клоне в новый язык
4. Переведите содержимое
5. Активируйте склонированный шаблон

## Клонирование, активация и деактивация шаблонов

### Клонирование шаблона

Клонирование создает точную копию шаблона — это удобно для создания языковых вариантов или тестирования разных версий без влияния на активный шаблон.

1. Выберите один или несколько шаблонов в списке
2. Выберите **Клонировать выбранные шаблоны** в выпадающем меню **Действия**
3. Клон создается как неактивный — отредактируйте его и активируйте, когда будете готовы

### Активация и деактивация шаблонов

Шаблон должен быть **Активным**, чтобы его можно было использовать для отправки. В каждый момент времени используется только один активный шаблон на тип и комбинацию языка.

Чтобы активировать или деактивировать в пакете:
1. Выберите шаблоны
2. Выберите **Активировать выбранные шаблоны** или **Деактивировать выбранные шаблоны** в выпадающем меню **Действия**

Или откройте отдельный шаблон и переключите флажок **Активный**.

## Системные шаблоны

Шаблоны, помеченные значком **Система**, являются шаблонами по умолчанию, добавленными Spwig. Их нельзя удалить. Вы можете редактировать их напрямую или склонировать, чтобы создать пользовательскую версию.

## Советы

- Всегда предварительно просмотрите шаблон после редактирования, чтобы обнаружить проблемы с форматированием до того, как клиенты увидят их
- Держите темы короткими и конкретными — `Ваш заказ #10045 отправлен` работает лучше, чем общие темы вроде `Обновление от нашего магазина`
- Редактируйте текстовую версию содержимого тоже — некоторые клиенты электронной почты показывают только текстовую версию, а некоторые клиенты предпочитают ее
- Используйте английскую версию шаблона как отправную точку перед созданием переведенной версии
- Если вы хотите протестировать изменения без влияния на активные электронные письма, склонируйте шаблон, отредактируйте клон и оставьте оба активными на короткое время, пока проверяете предварительный просмотр — затем деактивируйте оригинал
- Шаблоны уведомлений администратора (например, **Администратор: Новый заказ**) отправляются на адрес электронной почты администратора вашего магазина — убедитесь, что этот адрес электронной почты указан правильно в настройках вашего магазина
