---
title: 电子邮件模板
---

电子邮件模板控制您的商店发送给客户和您本人的每封自动电子邮件的设计和内容——包括订单确认、发货更新、密码重置、退款通知等。编辑模板将更改该类型的所有未来电子邮件；已放入邮箱的先前电子邮件不受影响。

导航到 **电子邮件系统 > 电子邮件模板** 以查看和管理您的模板。

![电子邮件模板列表](/static/core/admin/img/help/email-templates/templates-list.webp)

## 模板类型

您的商店包含针对各种事件的模板。它们按类别分组：

### 面向客户的订单电子邮件
| 模板 | 发送时机 |
|----------|-----------|
| 订单确认 | 客户完成购买 |
| 支付确认 | 支付成功处理 |
| 订单已发货 | 订单标记为已发货 |
| 发货确认 | 添加了运单跟踪号 |
| 送达确认 | 订单标记为已送达 |
| 订单已取消 | 订单被取消 |
| 订单延迟通知 | 记录了订单的延迟 |
| 退款通知 | 发出退款 |

### 账户电子邮件
| 模板 | 发送时机 |
|----------|-----------|
| 欢迎账户 | 客户创建账户 |
| 账户邀请 | 您邀请客户创建账户 |
| 邮箱验证 | 客户验证其电子邮件地址 |
| 密码重置 | 客户请求密码重置 |

### 退货
| 模板 | 发送时机 |
|----------|-----------|
| 退货：收到请求 | 客户提交退货请求 |
| 退货：已批准 | 退货请求已批准 |
| 退货：已拒绝 | 退货请求被拒绝 |
| 退货：收到包裹 | 退货商品到达您的位置 |
| 退货：已处理退款 | 退货的退款已发出 |


### 管理员通知（发送给您）
| 模板 | 发送时机 |
|----------|-----------|
| 管理员：新订单 | 有新订单生成 |
| 管理员：支付失败 | 支付尝试失败 |
| 管理员：每日销售报告 | 生成每日销售摘要 |
| 管理员：库存不足提醒 | 产品库存低于设定阈值 |
| 管理员：每周摘要 | 生成每周店铺摘要 |

其他模板涵盖物流跟踪里程碑、联盟计划活动、预订确认（如果启用了预订功能）以及忠诚度计划事件。

## 编辑模板

1. 导航至 **电子邮件系统 > 电子邮件模板**
2. 找到您要编辑的模板。您可以使用右侧的筛选器按 **模板类型**、**语言** 或 **状态** 进行筛选
3. 点击模板以打开它
4. 编辑 **主题** 行（显示在客户收件箱中的电子邮件主题）
5. 编辑 **HTML 内容** 以创建电子邮件的完整设计版本
6. 可选地编辑 **文本内容** —— 作为不支持 HTML 的电子邮件客户端的纯文本备用内容
7. 点击 **保存**

> **HTML 邮件：** HTML 内容字段接受标准 HTML，包括内联 CSS。Spwig 会将其渲染为格式正确的电子邮件。如果您使用 MJML 标记，保存时会自动编译。

## 预览模板

保存之前，您可以预览模板在电子邮件客户端中的显示效果：

1. 打开您要预览的模板
2. 点击 **预览** 按钮（在模板列表或模板详情页面上可见）
3. 预览将在新浏览器标签中打开，显示渲染后的电子邮件

这使您可以在模板上线之前检查布局、格式和占位符变量的显示效果。

## 模板变量

变量是模板中的占位符，Spwig 在发送电子邮件时会将其替换为真实数据。它们以 `{{ variable_name }}` 的形式书写。

大多数模板中可用的常见变量：

| 变量 | 替换为 |
|----------|---------------|
| `{{ customer_name }}` | 客户的全名 |
| `{{ order_number }}` | 订单参考编号 |
| `{{ order_total }}` | 订单总金额 |
| `{{ store_name }}` | 您的店铺名称 |
| `{{ store_url }}` | 您的店铺网址 |
| `{{ tracking_number }}` | 运输跟踪编号 |
| `{{ tracking_url }}` | 用于跟踪运输的可点击链接 |

可用的变量取决于模板类型。与订单相关模板（如 `{{ order_number }}`）相关的变量在账户模板（如密码重置）中不可用。如果您包含不适用的变量，它将显示为空或未被替换。

## 语言支持

每种模板类型可以为您的店铺支持的每种语言提供一个版本。模板的 **Language** 字段控制哪个语言版本处于活动状态。

Spwig 会根据客户语言偏好自动选择正确的语言版本进行发送。如果客户语言没有对应的模板，Spwig 会回退到英文版本。

要为新语言添加模板：
1. 打开现有模板
2. 从 **Actions** 菜单中点击 **Clone Template**
3. 将克隆的模板的 **Language Code** 设置为新语言
4. 翻译内容
5. 激活克隆的模板

## 克隆、激活和停用模板

### 克隆模板

克隆会创建模板的精确副本——这对于创建语言变体或在不影响实时模板的情况下测试不同版本很有用。

1. 在列表中选择一个或多个模板
2. 从 **Actions** 下拉菜单中选择 **Clone selected templates**
3. 克隆将被创建为非活动状态——编辑它并在准备好时激活它

### 激活和停用模板

模板必须为 **Active** 才能用于发送。每次只使用一个活动模板，按类型和语言组合计算。

要批量激活或停用：
1.

选择模板
2.

从 **操作** 下拉菜单中选择 **激活所选模板** 或 **停用所选模板**

或打开单个模板并切换 **启用** 复选框。

## 系统模板

标记为 **系统** 的模板是 Spwig 默认预装的模板。它们不能被删除。您可以直接编辑它们或克隆它们以创建自定义版本。

## 小贴士

- 编辑模板后始终预览它，以在客户看到之前发现格式问题
- 保持主题行简短且具体 — `Your order #10045 has shipped` 的效果比通用主题如 `Update from our store` 更好
- 也要编辑纯文本内容 — 一些电子邮件客户端只显示纯文本版本，一些客户更喜欢它
- 克隆模板的英文版本作为创建翻译版本的起点
- 如果您想在不影响实时电子邮件的情况下测试更改，请克隆模板，编辑克隆版本，并在验证预览时短暂保持两个版本都启用 — 然后停用原始版本
- 管理员通知模板（如 **Admin: New Order**）会发送到您商店的管理员电子邮件地址 — 确保商店设置中该电子邮件地址是正确的
