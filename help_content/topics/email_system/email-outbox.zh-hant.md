---
title: 電子郵件發件箱
---

電子郵件發件箱是您商店已發送或嘗試發送的所有電子郵件的完整記錄 — 訂單確認、運送更新、管理員報告以及所有其他交易訊息。您可以使用它來確認郵件是否已遞送、調查失敗原因，並管理電子郵件佇列。

導航至 **電子郵件系統 > 電子郵件發件箱** 以查看電子郵件記錄。

![電子郵件發件箱列表與狀態徽章](/static/core/admin/img/help/email-outbox/outbox-list.webp)

## 阅讀發件箱

頂部的摘要欄顯示每個狀態類別的數量。下方的列表顯示個別電子郵件，包含：

- **主旨** — 電子郵件的主旨行
- **收件人** — 收件人的電子郵件地址
- **發件人** — 使用的發件人地址
- **狀態** — 當前的遞送狀態
- **加入佇列時間** — 電子郵件進入佇列的時間
- **發送時間** — 電子郵件發送到服務提供商的時間
- **重試次數** — 已嘗試發送的次數

## 電子郵件狀態

| 狀態 | 含義 |
|------|------|
| 已加入佇列 | 電子郵件正在佇列中等待發送 |
| 正在發送 | 電子郵件目前正在發送到服務提供商 |
| 已發送 | 服務提供商已接受電子郵件 |
| 已暫停 | 電子郵件已暫停，直到釋放後才會發送 |
| 已記錄 | 電子郵件已記錄但未發送（測試模式或僅記錄設置） |
| 已失敗 | 服務提供商拒絕或無法遞送電子郵件 |
| 已反彈 | 電子郵件已發送但被收件人的郵件伺服器反彈 |
| 已跳過 | 因系統原因跳過發送 |

## 查看電子郵件詳細資訊

點擊列表中的任何電子郵件以查看完整詳細資訊：

- 電子郵件的完整 **HTML 內容** 和 **文字內容**
- **服務提供商訊息 ID** — 來自您的電子郵件提供商的參考（在聯繫提供商支援時使用此資訊）
- **錯誤訊息** — 失敗或反彈電子郵件的確切錯誤
- **重試次數** 和 **最大重試次數** — 發送嘗試的次數
- 所有時間戳：建立、加入佇列、發送和失敗

## 過濾發件箱

使用右側的過濾器來縮小您的視圖：

- **狀態** — 顯示特定遞送狀態的郵件
- **日期** — 依郵件建立或發送時間過濾
- **範本類型** — 僅顯示特定通知類型的郵件（例如，僅顯示訂單確認郵件）

頂部的搜尋框可依郵件主旨、收件人地址、發件人地址或郵件服務商訊息 ID 進行搜尋。

## 發送已暫停的郵件

狀態為 **已暫停** 的郵件會被暫停 — 直到您釋放它們，否則不會發送。郵件可能被暫停，如果您的商店在生成時處於維護模式，或者管理員操作導致其被暫停。

要釋放已暫停的郵件：
1. 選取您要釋放的郵件（左側的方框勾選）
2. 從 **動作** 下拉選單中選擇 **釋放已暫停的郵件以發送**
3. 點擊 **執行**

已釋放的郵件會轉為 **已排隊** 狀態，並在下一次排隊處理週期中發送。

## 預約郵件

有些郵件會預約在未來時間發送 — 例如，每週摘要報告會預約在特定日期和時間發送。前往 **郵件系統 > 預約郵件** 以查看即將發送的預約郵件。

預約郵件列表顯示：
- **範本類型** — 預約的郵件類型
- **收件人郵件** — 它將發送到的地址
- **預約時間** — 它預計發送的日期和時間
- **狀態** — 待處理（尚未發送）、已發送或失敗

預約郵件會在預約時間到達時自動處理 — 不需要手動操作。

## 解決發送失敗的問題

如果郵件顯示 **失敗** 狀態，請點擊以查看錯誤訊息，並按照以下步驟操作：

### 常見原因和解決方法

| 症狀 | 可能原因 | 處理方式 |
|---------|-------------|------------|
| "驗證失敗" | 電子郵件服務商的憑證無效 | 在 **電子郵件系統 > 電子郵件帳號** 中更新憑證 |
| "連線被拒絕" / "逾時" | 你的電子郵件伺服器無法連線 | 檢查電子郵件服務商的狀態頁面；在 **電子郵件帳號** 中測試連線 |
| "無效收件人" | 顧客的電子郵件地址格式錯誤 | 檢查顧客的帳號並修正其電子郵件 |
| 被退回的電子郵件 | 收件人的郵件伺服器拒絕了電子郵件 | 電子郵件地址可能不存在，或其收件箱已滿；不要過度重試 |
| 突然的高失敗率 | 服務商問題或憑證過期 | 檢查服務商狀態；在 **電子郵件帳號** 中重新測試連線 |

### 檢查你的電子郵件帳號連線

如果許多電子郵件都失敗，請測試你的電子郵件帳號：

1. 進入 **電子郵件系統 > 電子郵件帳號**
2. 找到你的活動帳號並檢查其 **連線** 狀態
3. 如果連線顯示錯誤，點擊帳號並使用 **測試連線** 選項來診斷問題

### 重試行為

Spwig 會自動重試失敗的電子郵件，最多達到 **最大重試次數** 的限制。每個電子郵件上顯示的重試次數告訴你已經嘗試了多少次。一旦達到重試限制，電子郵件會保持在 **失敗** 狀態，不會再進行自動重試。

## 被退回的電子郵件

一個 **被退回** 的電子郵件已經被發送，但被收件人的郵件伺服器退回。有兩種退回類型：

- **硬性退回** — 電子郵件地址不存在或域名不接受電子郵件。不要重試硬性退回；該地址無效
- **軟性退回** — 暫時性問題（收件箱已滿、伺服器暫時不可用）。重試可能成功

反覆退回同一地址可能會損害你與電子郵件服務商的發件者聲譽。如果你看到同一顧客地址反覆退回，請更新或從顧客帳號中刪除該地址。

## 小技巧

保留所有 markdown 格式、圖片路徑、程式碼區塊和技術術語。

- 在閃購或大型產品發佈等重大事件後，請查看出站郵件箱，確認所有訂單確認郵件均已成功發送
- 如果客戶表示未收到郵件，請透過他們的電子郵件地址在出站郵件箱中進行搜索，查看郵件是否已發送、失敗或跳過
- 失敗數量突然增加通常表示憑證或帳戶問題 — 請立即檢查 **電子郵件帳戶**
- **保留中** 狀態並非失敗 — 只表示郵件正在等待中。

在準備發送時釋放保留中的郵件
- 使用 **郵件模板類型** 過濾器快速審計某一類型的所有郵件 — 例如，檢查過去7天內的所有訂單確認郵件是否都有 **已發送** 狀態
- 列表頂部的日期層級導覽（日 / 月 / 年）對於審查特定時期的出站郵件非常有用
