title: เทมเพลตอีเมล
---

เทมเพลตอีเมลควบคุมการออกแบบและเนื้อหาของอีเมลแบบอัตโนมัติทุกฉบับที่ร้านค้าของคุณส่งไปยังลูกค้าและส่งไปยังคุณ — การยืนยันการสั่งซื้อ, การอัปเดตการจัดส่ง, การรีเซ็ตรหัสผ่าน, การแจ้งเตือนการคืนเงิน, และอื่นๆ อีกมากมาย การแก้ไขเทมเพลตจะเปลี่ยนอีเมลในอนาคตทั้งหมดที่เป็นประเภทเดียวกัน; อีเมลที่ผ่านมาที่อยู่ในกล่องจดหมายแล้วจะไม่ได้รับผลกระทบ

ไปที่ **การตั้งค่าอีเมล > เทมเพลตอีเมล** เพื่อดูและจัดการเทมเพลตของคุณ

![รายการเทมเพลตอีเมล](/static/core/admin/img/help/email-templates/templates-list.webp)

## ประเภทของเทมเพลต

ร้านค้าของคุณมีเทมเพลตสำหรับเหตุการณ์หลากหลายประเภท พวกเขาถูกจัดกลุ่มตามหมวดหมู่:

### อีเมลสำหรับลูกค้า
| เทมเพลต | ส่งเมื่อ |
|----------|-----------|
| การยืนยันการสั่งซื้อ | ลูกค้าทำการซื้อสินค้า |
| การยืนยันการชำระเงิน | การชำระเงินถูกดำเนินการสำเร็จ |
| การส่งสินค้า | การสั่งซื้อถูกตั้งสถานะว่าถูกส่ง |
| การยืนยันการจัดส่ง | หมายเลขติดตามการจัดส่งถูกเพิ่ม |
| การยืนยันการจัดส่ง | การสั่งซื้อถูกตั้งสถานะว่าถูกจัดส่ง |
| การยกเลิกการสั่งซื้อ | การสั่งซื้อถูกยกเลิก |
| การแจ้งเตือนการล่าช้า | การล่าช้าถูกบันทึกในรายการสั่งซื้อ |
| การแจ้งเตือนการคืนเงิน | การคืนเงินถูกดำเนินการ |

### อีเมลบัญชี
| เทมเพลต | ส่งเมื่อ |
|----------|-----------|
| ยินดีต้อนรับ | ลูกค้าสร้างบัญชี |
| การเชิญชวน | คุณเชิญชวนลูกค้าให้สร้างบัญชี |
| การตรวจสอบอีเมล | ลูกค้าตรวจสอบอีเมลของตนเอง |
| การรีเซ็ตรหัสผ่าน | ลูกค้าขอรีเซ็ตรหัสผ่าน |

### การคืนสินค้า
| เทมเพลต | ส่งเมื่อ |
|----------|-----------|
| การคืนสินค้า: รับคำขอ | ลูกค้าส่งคำขอคืนสินค้า |
| การคืนสินค้า: อนุมัติ | คำขอคืนสินค้าได้รับการอนุมัติ |
| การคืนสินค้า: ปฏิเสธ | คำขอคืนสินค้าถูกปฏิเสธ |
| การคืนสินค้า: รับแพ็กเกจ | รายการคืนสินค้าถึงที่ของคุณ |
| การคืนสินค้า: ดำเนินการคืนเงิน | การคืนเงินสำหรับการคืนสินค้าถูกดำเนินการ |


### การแจ้งเตือนสำหรับผู้ดูแลระบบ (ส่งถึงคุณ)
| แม่แบบ | ส่งเมื่อ |
|----------|-----------|
| ผู้ดูแลระบบ: คำสั่งซื้อใหม่ | มีการสั่งซื้อใหม่ |
| ผู้ดูแลระบบ: การชำระเงินล้มเหลว | การพยายามชำระเงินล้มเหลว |
| ผู้ดูแลระบบ: รายงานยอดขายรายวัน | สร้างรายงานยอดขายรายวัน |
| ผู้ดูแลระบบ: การแจ้งเตือนสต็อกต่ำ | สินค้าต่ำกว่าระดับสต็อกที่กำหนด |
| ผู้ดูแลระบบ: สรุปรายสัปดาห์ | สร้างสรุปสถานะร้านค้ารายสัปดาห์ |

แม่แบบเพิ่มเติมครอบคลุมการติดตามการจัดส่ง, การดำเนินการของโปรแกรมพันธมิตร, การยืนยันการจอง (หากเปิดใช้งานฟีเจอร์การจอง), และเหตุการณ์ของโปรแกรมสะสมแต้ม

## การแก้ไขแม่แบบ

1. ไปที่ **ระบบอีเมล > แม่แบบอีเมล**
2. ค้นหาแม่แบบที่คุณต้องการแก้ไข คุณสามารถกรองตาม **ประเภทแม่แบบ**, **ภาษา**, หรือ **สถานะ** โดยใช้ตัวกรองทางด้านขวา
3. คลิกที่แม่แบบเพื่อเปิดมัน
4. แก้ไขบรรทัด **หัวข้อ** (หัวข้ออีเมลที่แสดงในกล่องจดหมายของลูกค้า)
5. แก้ไข **เนื้อหา HTML** สำหรับเวอร์ชันออกแบบเต็มรูปแบบของอีเมล
6. แก้ไข **เนื้อหาข้อความ** แบบไม่มีการตกแต่ง (สำรองใช้กับผู้ใช้ที่ไม่รองรับ HTML) ได้ตามต้องการ
7. คลิก **บันทึก**

> **อีเมล HTML:** ฟิลด์เนื้อหา HTML ยอมรับ HTML มาตรฐานรวมถึง CSS ที่ฝังอยู่ Spwig จะแสดงผลอีเมลในรูปแบบที่ถูกต้อง หากคุณใช้โค้ด MJML จะถูกคอมไพล์อัตโนมัติเมื่อบันทึก

## การดูตัวอย่างแม่แบบ

ก่อนบันทึก คุณสามารถดูตัวอย่างว่าแม่แบบจะแสดงผลอย่างไรในโปรแกรมอีเมล:

1. เปิดแม่แบบที่คุณต้องการดูตัวอย่าง
2. คลิกปุ่ม **ดูตัวอย่าง** (แสดงในรายการแม่แบบหรือในหน้ารายละเอียดแม่แบบ)
3. ตัวอย่างจะเปิดในแท็บเบราว์เซอร์ใหม่ แสดงอีเมลที่ถูกแสดงผลแล้ว

สิ่งนี้ช่วยให้คุณตรวจสอบการจัดวาง, การจัดรูปแบบ, และการปรากฏของตัวแปรแทนที่ก่อนที่แม่แบบจะถูกเผยแพร่จริง

## ตัวแปรแม่แบบ

ตัวแปรคือตัวแทนที่อยู่ในแม่แบบของคุณ Spwig จะแทนที่ด้วยข้อมูลจริงเมื่อส่งอีเมล ตัวแปรถูกเขียนในรูปแบบ `{{ ชื่อตัวแปร }}`

ตัวแปรทั่วไปที่มีในแม่แบบส่วนใหญ่:


| ตัวแปร | ถูกแทนด้วย |
|----------|---------------|
| `{{ customer_name }}` | ชื่อเต็มของลูกค้า |
| `{{ order_number }}` | หมายเลขอ้างอิงคำสั่งซื้อ |
| `{{ order_total }}` | จำนวนเงินรวมของคำสั่งซื้อ |
| `{{ store_name }}` | ชื่อร้านของคุณ |
| `{{ store_url }}` | ที่อยู่เว็บไซต์ของร้านคุณ |
| `{{ tracking_number }}` | หมายเลขติดตามการจัดส่ง |
| `{{ tracking_url }}` | ลิงก์ที่สามารถคลิกเพื่อติดตามการจัดส่ง |

ตัวแปรที่มีอยู่โดยแท้จริงขึ้นอยู่กับประเภทของเทมเพลต ตัวแปรที่เกี่ยวข้องกับเทมเพลตที่เกี่ยวข้องกับคำสั่งซื้อ (เช่น `{{ order_number }}`) จะไม่สามารถใช้งานได้ในเทมเพลตที่เกี่ยวข้องกับบัญชี (เช่น การรีเซ็ตรหัสผ่าน) หากคุณรวมตัวแปรที่ไม่เกี่ยวข้อง มันจะปรากฏเป็นช่องว่างหรือไม่ได้รับการแทนที่

## การสนับสนุนภาษา

แต่ละประเภทของเทมเพลตสามารถมีเวอร์ชันสำหรับแต่ละภาษาที่ร้านคุณสนับสนุน ฟิลด์ **Language** บนเทมเพลตแต่ละตัวควบคุมว่าเวอร์ชันภาษาใดที่เป็นปัจจุบัน

Spwig จะเลือกเวอร์ชันภาษาที่ถูกต้องโดยอัตโนมัติโดยพิจารณาจากความชอบของภาษาของลูกค้าขณะส่ง หากไม่มีเทมเพลตสำหรับภาษาของลูกค้า Spwig จะกลับไปใช้เวอร์ชันภาษาอังกฤษ

เพื่อเพิ่มเทมเพลตสำหรับภาษาใหม่:
1. เปิดเทมเพลตที่มีอยู่
2. คลิก **Clone Template** จากเมนู **Actions**
3. ตั้งค่า **Language Code** บนเทมเพลตสำเนาเป็นภาษาใหม่
4. แปลเนื้อหา
5. เปิดใช้งานเทมเพลตที่สำเนา

## การสำเนา ตั้งค่า และปิดใช้งานเทมเพลต

### การสำเนาเทมเพลต

การสำเนาสร้างสำเนาที่แม่นยำของเทมเพลต — มีประโยชน์ในการสร้างเวอร์ชันภาษาต่างๆ หรือทดสอบเวอร์ชันต่างๆ โดยไม่กระทบกับเทมเพลตที่ใช้งานอยู่

1. เลือกเทมเพลตหนึ่งหรือหลายตัวในรายการ
2. เลือก **Clone selected templates** จากเมนู **Actions**
3. เทมเพลตสำเนาจะถูกสร้างขึ้นในสถานะไม่ได้ใช้งาน — แก้ไขมันและเปิดใช้งานเมื่อพร้อม

### การเปิดใช้งานและปิดใช้งานเทมเพลต

เทมเพลตต้อง **Active** จึงจะสามารถใช้งานได้ในการส่ง ใช้เทมเพลตที่เป็นปัจจุบันเพียงตัวเดียวต่อประเภทและภาษาที่รวมกัน

เพื่อเปิดใช้งานหรือปิดใช้งานแบบหมู่:
1.

เลือกเทมเพลต
2.

เลือก **เปิดใช้งานเทมเพลตที่เลือก** หรือ **ปิดใช้งานเทมเพลตที่เลือก** จากเมนู **Actions**

หรือเปิดเทมเพลตแต่ละตัวและสลับปุ่ม **Active**

## System templates

เทมเพลตที่มีป้ายกำกับ **System** เป็นเทมเพลตเริ่มต้นที่ Spwig ติดตั้งไว้ให้ ไม่สามารถลบได้ คุณสามารถแก้ไขเทมเพลตเหล่านี้โดยตรง หรือคัดลอกเพื่อสร้างเวอร์ชันที่ปรับแต่งเอง

## Tips

- เสมอตรวจสอบเทมเพลตหลังจากแก้ไขเพื่อจับข้อผิดพลาดด้านรูปแบบก่อนที่ลูกค้าจะเห็น
- รักษาหัวข้อให้สั้นและเฉพาะเจาะจง — `Your order #10045 has shipped` ทำงานได้ดีกว่าหัวข้อทั่วไป เช่น `Update from our store`
- แก้ไขเนื้อหาแบบ plain-text ด้วย — บางโปรแกรมอีเมลจะแสดงเฉพาะเวอร์ชัน plain-text และลูกค้าบางคนชอบใช้เวอร์ชันนี้
- คัดลอกเวอร์ชันภาษาอังกฤษของเทมเพลตเป็นจุดเริ่มต้นก่อนสร้างเวอร์ชันที่แปลแล้ว
- หากคุณต้องการทดสอบการเปลี่ยนแปลงโดยไม่ส่งผลต่ออีเมลที่ใช้งานอยู่ ให้คัดลอกเทมเพลต แก้ไขเทมเพลตที่คัดลอก แล้วเปิดใช้งานทั้งสองเวอร์ชันชั่วคราวขณะตรวจสอบรูปแบบ — จากนั้นปิดใช้งานเวอร์ชันเดิม
- เทมเพลตการแจ้งเตือนสำหรับผู้ดูแลระบบ (เช่น **Admin: New Order**) จะถูกส่งไปยังที่อยู่อีเมลผู้ดูแลระบบของร้านคุณ — ตรวจสอบให้แน่ใจว่าที่อยู่อีเมลนั้นถูกต้องในการตั้งค่าร้านคุณ
