---
title: 이메일 아웃박스
---

이메일 아웃박스는 스토어가 보낸 이메일 또는 보내려 시도한 이메일의 전체 기록입니다. 주문 확인, 배송 업데이트, 관리자 보고서 및 기타 모든 거래 관련 메시지가 포함됩니다. 이 기능을 사용하여 배송 상태를 확인하고 실패 사항을 조사하며 이메일 대기열을 관리할 수 있습니다.

**이메일 시스템 > 이메일 아웃박스**로 이동하여 이메일 기록을 확인합니다.

![이메일 아웃박스 목록과 상태 배지](/static/core/admin/img/help/email-outbox/outbox-list.webp)

## 아웃박스 보기

상단의 요약 바는 각 상태 범주별 수량을 표시합니다. 아래 목록은 다음과 같은 정보를 포함합니다:

- **제목** — 이메일 제목 줄
- **받는 사람** — 수신자의 이메일 주소
- **발신자** — 사용된 발신자 주소
- **상태** — 현재 배송 상태
- **대기열에 추가된 시간** — 이메일이 대기열에 추가된 시간
- **발송 시간** — 이메일이 제공업체에 전송된 시간
- **재시도 횟수** — 시도된 발송 횟수

## 이메일 상태

| 상태 | 의미 |
|------|------|
| 대기 중 | 이메일은 아직 보내지 않고 대기열에 있습니다 |
| 전송 중 | 이메일은 현재 제공업체로 전송되고 있습니다 |
| 전송됨 | 제공업체가 이메일을 수락했습니다 |
| 일시 중지됨 | 이메일은 일시 중지되어 해제될 때까지 보내지 않습니다 |
| 기록됨 | 이메일은 기록되었지만 보내지 않았습니다 (테스트 모드 또는 기록만 하는 설정) |
| 실패 | 제공업체가 이메일을 거부하거나 전송하지 못했습니다 |
| 반송됨 | 이메일은 보내졌지만 수신자의 메일 서버에서 반송되었습니다 |
| 건너뜀 | 시스템의 이유로 전송이 건너뛰어졌습니다 |

## 이메일 세부 정보 보기

목록에서 이메일을 클릭하여 전체 세부 정보를 확인할 수 있습니다:

- 이메일의 전체 **HTML 본문** 및 **텍스트 본문**
- **제공업체 메시지 ID** — 이메일 제공업체에서 제공하는 참조 (제공업체 지원팀에 문의할 때 사용)
- **에러 메시지** — 실패하거나 반송된 이메일의 정확한 에러 메시지
- **재시도 횟수** 및 **최대 재시도 횟수** — 시도된 전송 횟수
- 모든 타임스탬프: 생성, 대기, 전송, 실패

## 아웃박스 필터링

오른쪽의 필터를 사용하여 보기 범위를 좁힐 수 있습니다:

- **상태** — 특정 배송 상태의 이메일만 표시
- **날짜** — 이메일이 생성되거나 전송된 시간으로 필터링
- **템플릿 유형** — 특정 알림 유형의 이메일만 표시(예: 주문 확인 이메일만)

상단의 검색 상자에서는 제목, 수신자 주소, 발신자 주소 또는 제공업체 메시지 ID로 검색할 수 있습니다.

## 보류된 이메일 해제

**보류** 상태의 이메일은 일시 중지되어 있습니다 — 해제할 때까지 전송되지 않습니다. 이메일이 보류될 수 있는 경우는 가게가 생성될 때 유지보수 모드에 있었거나, 관리자 작업으로 보류되었을 때입니다.

보류된 이메일을 해제하려면:
1. 해제하고자 하는 이메일을 선택합니다(왼쪽의 체크박스를 선택)
2. **Actions** 드롭다운에서 **Release held emails for delivery**를 선택합니다
3. **Go**를 클릭합니다

해제된 이메일은 **Queued** 상태로 이동하고 다음 대기열 처리 주기에서 전송됩니다.

## 예약된 이메일

일부 이메일은 미래의 특정 시간에 전송하도록 예약되어 있습니다 — 예를 들어, 주간 요약 보고서는 특정 날짜와 시간에 전송하도록 예약됩니다. **Email System > Scheduled Emails**로 이동하여 예약된 전송을 확인할 수 있습니다.

예약된 이메일 목록에는 다음과 같은 항목이 표시됩니다:
- **템플릿 유형** — 예약된 이메일의 유형
- **수신자 이메일** — 전송될 주소
- **예약 시간** — 전송이 예정된 날짜 및 시간
- **상태** — 미전송(Pending), 전송(Sent), 또는 실패(Failed)

예약된 이메일은 예약된 시간에 자동으로 처리됩니다 — 수동 작업이 필요하지 않습니다.

## 실패한 전송 문제 해결

이메일이 **실패** 상태를 표시하는 경우, 오류 메시지를 보고 다음 단계를 따르세요:

### 일반적인 원인 및 해결 방법

| 증상 | 원인 | 해결 방법 |
|---------|-------------|------------|
| "인증 실패" | 이메일 제공자의 자격 증명이 잘못되었습니다 | **이메일 시스템 > 이메일 계정**에서 자격 증명을 업데이트하세요 |
| "연결 거부" / "타임아웃" | 이메일 서버에 도달할 수 없습니다 | 이메일 제공자의 상태 페이지를 확인하세요; **이메일 계정**에서 연결을 테스트하세요 |
| "잘못된 수신자" | 고객의 이메일 주소가 잘못되었습니다 | 고객의 계정을 검토하고 이메일을 수정하세요 |
| 반송된 이메일 | 수신자의 메일 서버가 이메일을 거부했습니다 | 이메일 주소가 존재하지 않거나 수신함이 가득 찼을 수 있습니다; 과도하게 다시 시도하지 마세요 |
| 갑작스럽게 높아진 실패율 | 제공자 문제 또는 자격 증명이 만료됨 | 제공자의 상태를 확인하고 **이메일 계정**에서 연결을 다시 테스트하세요 |

### 이메일 계정 연결 확인

많은 이메일이 실패하는 경우 이메일 계정을 테스트하세요:

1. **이메일 시스템 > 이메일 계정**으로 이동하세요
2. 활성 계정을 찾아 해당 계정의 **연결** 상태를 확인하세요
3. 연결이 오류를 표시하는 경우 계정을 클릭하고 **연결 테스트** 옵션을 사용하여 문제를 진단하세요

### 재시도 행동

Spwig은 **최대 재시도 횟수** 제한까지 실패한 이메일을 자동으로 재시도합니다. 각 이메일에 표시된 재시도 횟수는 시도한 횟수를 알려줍니다. 최대 재시도 횟수에 도달하면 이메일은 **실패** 상태로 남고 추가적인 자동 재시도는 발생하지 않습니다.

## 반송된 이메일

**반송**된 이메일은 보내졌지만 수신자의 메일 서버에 의해 반환되었습니다. 두 가지 반송 유형이 있습니다:

- **하드 반송** — 이메일 주소가 존재하지 않거나 도메인이 이메일을 수락하지 않습니다. 하드 반송은 다시 시도하지 마세요; 주소는 유효하지 않습니다
- **소프트 반송** — 일시적인 문제(수신함이 가득 찼거나 서버가 일시적으로 사용 불가능함). 재시도 시 성공할 수 있습니다

같은 주소로 반복된 반송은 이메일 제공자와의 발신자 평판에 영향을 줄 수 있습니다. 동일한 고객 주소로 반복된 반송을 보는 경우 해당 주소를 고객의 계정에서 업데이트하거나 제거하세요.

## 팁

모든 마크다운 형식, 이미지 경로, 코드 블록 및 기술 용어를 유지하세요.

- 플래시 세일이나 대규모 제품 출시와 같은 주요 이벤트 후에 아웃박스를 확인하여 모든 주문 확인 이메일이 성공적으로 전송되었는지 확인하세요
- 고객이 이메일을 받지 못했다고 말하면 이메일 주소로 아웃박스를 검색하여 전송되었는지, 실패했는지, 건너뛰었는지 확인하세요
- 실패 수의 급격한 증가는 자격 증명 또는 계정 문제를 나타낼 수 있습니다 — 즉시 **이메일 계정**을 확인하세요
- **보류** 상태는 실패가 아닙니다 — 이메일이 대기 중임을 의미합니다.

보류된 이메일을 전송할 준비가 되었을 때 보류된 이메일을 해제하세요
- **템플릿 유형** 필터를 사용하여 한 유형의 모든 이메일을 신속하게 감사할 수 있습니다 — 예를 들어, 지난 7일 동안의 모든 주문 확인 이메일이 **전송됨** 상태인지 확인하세요
- 목록 상단의 날짜 계층 네비게이션(일 / 월 / 년)은 특정 기간 동안의 아웃박스를 검토하는 데 유용합니다
