---
title: 電子郵件範本
---

電子郵件範本控制您商店發送到客戶和您自己的每一封自動電子郵件的設計和內容——包括訂單確認、運輸更新、密碼重置、退款通知等等。編輯範本會更改未來所有相同類型的電子郵件；已經在發件匣中的舊電子郵件不會受到影響。

導航至 **電子郵件系統 > 電子郵件範本** 以查看和管理您的範本。

![電子郵件範本列表](/static/core/admin/img/help/email-templates/templates-list.webp)

## 範本類型

您的商店包含針對各種事件的範本。它們按類別分組：

### 面向客戶的訂單郵件
| 範本 | 當...時發送 |
|----------|-----------|
| 訂單確認 | 顧客完成購買 |
| 支付確認 | 支付成功處理 |
| 訂單已發貨 | 訂單標記為已發貨 |
| 運輸確認 | 添加了運輸追蹤號 |
| 配送確認 | 訂單標記為已交付 |
| 訂單已取消 | 訂單被取消 |
| 訂單延遲通知 | 訂單上記錄了延遲 |
| 退款通知 | 發出退款 |

### 帳戶郵件
| 範本 | 當...時發送 |
|----------|-----------|
| 帳戶歡迎 | 顧客創建帳戶 |
| 帳戶邀請 | 您邀請顧客創建帳戶 |
| 電子郵件驗證 | 顧客驗證其電子郵件地址 |
| 密碼重置 | 顧客請求重置密碼 |

### 退貨
| 範本 | 當...時發送 |
|----------|-----------|
| 退貨：收到請求 | 顧客提交退貨請求 |
| 退貨：已批准 | 退貨請求已批准 |
| 退貨：已拒絕 | 退貨請求被拒絕 |
| 退貨：收到包裹 | 退回的商品到達您的位置 |
| 退貨：已處理退款 | 退貨的退款已發出 |


### 管理員通知（發送到您）
| 範本 | 當以下情況發生時發送 |
|----------|-----------|
| 管理員：新訂單 | 有新訂單產生 |
| 管理員：付款失敗 | 付款嘗試失敗 |
| 管理員：每日銷售報告 | 生成每日銷售摘要 |
| 管理員：庫存不足提醒 | 產品庫存低於設定的門檻 |
| 管理員：每周摘要 | 生成每周商店摘要 |

其他範本涵蓋運送追蹤里程碑、聯營計劃活動、預訂確認（如果啟用了預訂功能）以及忠誠度計劃事件。

## 編輯範本

1. 點擊 **電子郵件系統 > 電子郵件範本**
2. 找到您要編輯的範本。您可以使用右側的過濾器，按 **範本類型**、**語言** 或 **狀態** 來篩選
3. 點擊範本以打開它
4. 編輯 **主旨**（顯示在客戶收件箱中的電子郵件主旨）
5. 編輯 **HTML 內容** 以建立電子郵件的完整設計版本
6. 可選地編輯 **文字內容** — 作為不支援 HTML 的電子郵件客戶端的備用純文字版本
7. 點擊 **儲存**

> **HTML 電子郵件：** HTML 內容欄位接受標準 HTML，包括內聯 CSS。Spwig 會將其渲染成格式正確的電子郵件。如果您使用 MJML 標記，它會在儲存時自動編譯。

## 預覽範本

儲存之前，您可以預覽範本在電子郵件客戶端中的顯示效果：

1. 打開您要預覽的範本
2. 點擊 **預覽** 按鈕（在範本列表或範本詳細頁面中可見）
3. 預覽會在新瀏覽器標籤中打開，顯示渲染後的電子郵件

這讓您可以在範本上線前檢查佈局、格式和佔位符變數的顯示效果。

## 範本變數

變數是您範本中的佔位符，Spwig 在發送電子郵件時會將其替換為實際資料。它們寫作 `{{ 變數名稱 }}`。

大多數範本中都可用的常見變數：

| 變數 | 被替換為 |
|----------|---------------|
| `{{ customer_name }}` | 顧客的全名 |
| `{{ order_number }}` | 訂單參考編號 |
| `{{ order_total }}` | 訂單總金額 |
| `{{ store_name }}` | 您的商店名稱 |
| `{{ store_url }}` | 您的商店網址 |
| `{{ tracking_number }}` | 運送追蹤編號 |
| `{{ tracking_url }}` | 可點擊的運送追蹤連結 |

可用的變數取決於模板類型。與訂單相關的模板（例如 `{{ order_number }}`）所使用的變數，在帳號相關的模板（例如密碼重設）中是不可用的。如果您包含不適用的變數，它將顯示為空白或未被替換。

## 語言支援

每種模板類型都可以為您商店支援的每種語言提供一個版本。每個模板上的 **語言** 欄位控制哪個語言版本處於啟用狀態。

Spwig 會根據顧客的語言偏好自動選擇正確的語言版本進行發送。如果沒有對應顧客語言的模板，Spwig 會預設使用英文版本。

要為新語言新增模板：
1. 開啟現有的模板
2. 從 **動作** 功能表中點擊 **複製模板**
3. 將複製的模板的 **語言代碼** 設為新語言
4. 翻譯內容
5. 啟用複製的模板

## 複製、啟用與停用模板

### 複製模板

複製會建立一個模板的精確複本——這對於建立語言變體或測試不同版本而不影響現行模板非常有用。

1. 在清單中選擇一個或多個模板
2. 從 **動作** 下拉選單中選擇 **複製所選模板**
3. 複製的模板會處於停用狀態——編輯它並在準備好時啟用它

### 啟用與停用模板

模板必須處於 **啟用** 狀態才能用於發送。每次只會使用一種語言組合的啟用模板。

要批量啟用或停用：
1.

選擇模板
2.

從 **Actions** 下拉選單中選擇 **啟用選定的模板** 或 **停用選定的模板**

或打開單個模板並切換 **啟用** 方框。

## 系統模板

標記為 **System** 的模板是 Spwig 預設的模板。這些模板無法刪除。您可以直接編輯它們或複製它們來創建自定義版本。

## 小技巧

- 編輯模板後始終預覽它，以在客戶看到之前發現格式問題
- 保持主題行簡短且具體 — `Your order #10045 has shipped` 的效果比通用主題如 `Update from our store` 更好
- 也要編輯純文字內容 — 一些電子郵件客戶端只顯示純文字版本，有些客戶更喜歡它
- 在創建翻譯版本之前，先複製模板的英文版本作為起點
- 如果您想在不影響實際電子郵件的情況下測試更改，請複製模板，編輯複製的版本，並在驗證預覽時短暫保持兩個版本都啟用 — 然後停用原始版本
- 管理員通知模板（如 **Admin: New Order**）會發送到您商店的管理員電子郵件地址 — 請確保商店設置中該電子郵件地址正確
