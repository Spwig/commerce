---
title: メールテンプレート
---

メールテンプレートは、ストアが顧客や自分に送信するすべての自動メールのデザインと内容を制御します。注文の確認、配送の更新、パスワードのリセット、返金の通知など、さまざまなメールが含まれます。テンプレートを編集すると、そのタイプのすべての将来的なメールが変更されますが、既にメール送信キューにある以前のメールは影響を受けません。

**メールシステム > メールテンプレート**に移動して、テンプレートを表示および管理してください。

![メールテンプレート一覧](/static/core/admin/img/help/email-templates/templates-list.webp)

## テンプレートタイプ

ストアには、さまざまなイベント用のテンプレートが含まれています。それらはカテゴリごとにグループ化されています:

### 顧客向けの注文メール
| テンプレート | 送信されるとき |
|----------|-----------|
| 注文確認 | 顧客が購入を完了したとき |
| 支払い確認 | 支払いが正常に処理されたとき |
| 注文が発送されました | 注文が発送済みとしてマークされたとき |
| 配送確認 | 配送追跡番号が追加されたとき |
| 配達確認 | 注文が配達済みとしてマークされたとき |
| 注文がキャンセルされました | 注文がキャンセルされたとき |
| 注文の遅延通知 | 注文に遅延が記録されたとき |
| 返金通知 | 返金が発行されたとき |

### アカウントメール
| テンプレート | 送信されるとき |
|----------|-----------|
| アカウント歓迎 | 顧客がアカウントを作成したとき |
| アカウント招待 | あなたが顧客にアカウントを作成するよう招待したとき |
| メール確認 | 顧客がメールアドレスを確認したとき |
| パスワードリセット | 顧客がパスワードリセットを要求したとき |

### 返品
| テンプレート | 送信されるとき |
|----------|-----------|
| 返品: 依頼が受信されました | 顧客が返品依頼を提出したとき |
| 返品: 承認済み | 返品依頼が承認されたとき |
| 返品: 拒否されました | 返品依頼が拒否されたとき |
| 返品: 包装が到着しました | 返品品があなたの場所に到着したとき |
| 返品: 返金が処理されました | 返品の返金が発行されたとき |


### 管理者通知（あなたに送信）
| テンプレート | 送信されるとき |
|----------|-----------|
| 管理者: 新規注文 | 新規注文が行われたとき |
| 管理者: 支払い失敗 | 支払い試行が失敗したとき |
| 管理者: 日次売上レポート | 日次売上サマリが生成されたとき |
| 管理者: 在庫不足アラート | 商品が在庫閾値を下回ったとき |
| 管理者: 週次ダイジェスト | 週次ストアサマリが生成されたとき |

追加のテンプレートは、配送追跡のマイルストーン、アフィリエイトプログラムの活動、予約確認（予約機能が有効になっている場合）、およびロイヤルティプログラムのイベントをカバーしています。

## テンプレートの編集

1. **メールシステム > メールテンプレート** に移動します
2. 編集したいテンプレートを探します。右側のフィルターを使用して **テンプレートタイプ**、**言語**、または **ステータス** でフィルターを適用できます
3. テンプレートをクリックして開きます
4. **件名** を編集します（顧客のメールボックスに表示されるメールの件名）
5. **HTMLコンテンツ** を編集して、メールのフルデザインバージョンを作成します
6. 必要に応じて **テキストコンテンツ** を編集します — HTMLをサポートしないメールクライアント向けのプレーンテキストの代替です
7. **保存** をクリックします

> **HTMLメール:** HTMLコンテンツフィールドは、標準のHTML（インラインCSSを含む）を受け入れます。Spwigはこれを適切にフォーマットされたメールにレンダリングします。MJMLマーアップを使用している場合、保存時に自動的にコンパイルされます。

## テンプレートのプレビュー

保存する前に、テンプレートがメールクライアントでどのように表示されるかをプレビューできます:

1. プレビューしたいテンプレートを開きます
2. **プレビュー** ボタンをクリックします（テンプレートリストまたはテンプレート詳細ページで表示されます）
3. レンダリングされたメールが新しいブラウザタブで開きます

これにより、テンプレートが本番環境に公開される前にレイアウト、フォーマット、プレースホルダ変数の表示を確認できます。

## テンプレート変数

変数は、テンプレート内のプレースホルダーで、Spwigがメールを送信するときに実際のデータに置き換えられます。`{{ 変数名 }}` という形式で記述されます。

ほとんどのテンプレートで利用可能な一般的な変数:


| 変数 | 代替される内容 |
|----------|---------------|
| `{{ customer_name }}` | 顧客のフルネーム |
| `{{ order_number }}` | 注文参照番号 |
| `{{ order_total }}` | 注文の合計金額 |
| `{{ store_name }}` | あなたの店舗の名前 |
| `{{ store_url }}` | あなたの店舗のウェブアドレス |
| `{{ tracking_number }}` | 配送の追跡番号 |
| `{{ tracking_url }}` | 配送を追跡するクリック可能なリンク |

テンプレートの種類によって利用可能な変数は異なります。注文に関連するテンプレート（例: `{{ order_number }}`）で使用される変数は、アカウントテンプレート（例: パスワードリセット）では利用できません。適用されない変数を含めても、空白または置き換えられません。

## 言語サポート

各テンプレートタイプは、店舗がサポートする各言語のバージョンを持つことができます。**言語**フィールドは、各テンプレートで有効な言語バージョンを制御します。

Spwigは、送信時に顧客の言語設定に基づいて正しい言語バージョンを自動的に選択します。顧客の言語に対応するテンプレートが存在しない場合、Spwigは英語バージョンにフォールバックします。

新しい言語用のテンプレートを追加するには:
1. 既存のテンプレートを開きます
2. **Actions** メニューから **Clone Template** をクリックします
3. クローンの **Language Code** を新しい言語に設定します
4. 内容を翻訳します
5. クローンされたテンプレートを有効化します

## テンプレートのクローン、有効化、無効化

### テンプレートのクローン

クローンはテンプレートの正確なコピーを作成します — 言語バリアントを作成したり、ライブテンプレートに影響を与えることなく、さまざまなバージョンをテストしたりするのに役立ちます。

1. リストから1つ以上のテンプレートを選択します
2. **Actions** ドロップダウンから **Clone selected templates** を選択します
3. クローンは無効状態で作成されます — 編集して準備ができたら有効化してください

### テンプレートの有効化と無効化

テンプレートは **Active** である必要があります。送信に使用されます。1つのタイプと言語の組み合わせに対して、1つのアクティブなテンプレートのみが使用されます。

バッチで有効化または無効化するには:
1.

テンプレートを選択
2.

**アクション**ドロップダウンから **選択したテンプレートを有効にする** または **選択したテンプレートを無効にする** を選択します。

または、個別のテンプレートを開き、**有効**のチェックボックスを切り替えます。

## システムテンプレート

**システム**のバッジがついたテンプレートは、Spwigによってデフォルトで提供されるテンプレートです。これらは削除できません。直接編集するか、カスタムバージョンを作成するために複製してください。

## ヒント

- テンプレートを編集した後は常にプレビューを確認し、顧客が見る前にフォーマットの問題を修正してください。
- 件名は短く具体的に保つようにしてください。`Your order #10045 has shipped` は、`Update from our store` などの一般的な件名よりも効果的です。
- プレーンテキストの内容も編集してください — 一部のメールクライアントはプレーンテキストバージョンのみを表示し、一部の顧客はそれをお好みです。
- テンプレートの翻訳バージョンを作成する前に、英語版を複製して作業の起点とすることが推奨されます。
- 本番のメールに影響を与えたくない場合は、テンプレートを複製し、複製を編集して、確認のために両方を一時的に有効にしたままにしてください。確認が終わったら、オリジナルを無効にします。
- 管理者通知テンプレート（例: **Admin: New Order**）は、ストアの管理者メールアドレスに送信されます — ストアの設定でそのメールアドレスが正しいことを確認してください。
