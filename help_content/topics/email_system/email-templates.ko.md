---
title: 이메일 템플릿
---

이메일 템플릿은 고객과 귀하에게 전송되는 모든 자동 이메일의 디자인과 내용을 제어합니다. 주문 확인, 배송 업데이트, 비밀번호 재설정, 환불 알림 등 다양한 이메일이 포함됩니다. 템플릿을 편집하면 해당 유형의 모든 미래 이메일이 변경되지만, 이미 발신 대기 중인 이전 이메일은 영향을 받지 않습니다.

**이메일 시스템 > 이메일 템플릿**로 이동하여 템플릿을 확인하고 관리할 수 있습니다.

![이메일 템플릿 목록](/static/core/admin/img/help/email-templates/templates-list.webp)

## 템플릿 유형

귀하의 상점에는 다양한 이벤트에 대한 템플릿이 포함되어 있습니다. 이들은 카테고리별로 그룹화되어 있습니다:

### 고객 대상 주문 이메일
| 템플릿 | 전송 시 |
|----------|-----------|
| 주문 확인 | 고객이 구매를 완료할 때 |
| 결제 확인 | 결제가 성공적으로 처리될 때 |
| 주문 배송됨 | 주문이 배송 상태로 표시될 때 |
| 배송 확인 | 배송 추적 번호가 추가될 때 |
| 배송 확인 | 주문이 배송 완료 상태로 표시될 때 |
| 주문 취소 | 주문이 취소될 때 |
| 주문 지연 알림 | 주문에 지연이 기록될 때 |
| 환불 알림 | 환불이 발급될 때 |

### 계정 이메일
| 템플릿 | 전송 시 |
|----------|-----------|
| 계정 환영 | 고객이 계정을 생성할 때 |
| 계정 초대 | 고객을 계정 생성하도록 초대할 때 |
| 이메일 확인 | 고객이 이메일 주소를 확인할 때 |
| 비밀번호 재설정 | 고객이 비밀번호 재설정을 요청할 때 |

### 반품
| 템플릿 | 전송 시 |
|----------|-----------|
| 반품: 요청 수신 | 고객이 반품 요청을 제출할 때 |
| 반품: 승인됨 | 반품 요청이 승인될 때 |
| 반품: 거부됨 | 반품 요청이 거부될 때 |
| 반품: 패키지 수령 | 반품 상품이 귀하의 위치에 도착할 때 |
| 반품: 환불 처리됨 | 반품에 대한 환불이 발급될 때 |


### 관리자 알림 (당신에게 전송됨)
| 템플릿 | 전송 시점 |
|----------|-----------|
| 관리자: 새 주문 | 새 주문이 생성됨 |
| 관리자: 결제 실패 | 결제 시도 실패 |
| 관리자: 일일 매출 보고서 | 일일 매출 요약이 생성됨 |
| 관리자: 재고 부족 경고 | 제품이 재고 임계값 이하로 떨어짐 |
| 관리자: 주간 요약 | 주간 매장 요약이 생성됨 |

추가 템플릿은 배송 추적 마일스톤, 파트너 프로그램 활동, 예약 확인 (예약 기능이 활성화된 경우), 충성도 프로그램 이벤트를 포함합니다.

## 템플릿 편집

1. **이메일 시스템 > 이메일 템플릿**으로 이동하세요
2. 편집하려는 템플릿을 찾습니다. 오른쪽의 필터를 사용하여 **템플릿 유형**, **언어**, 또는 **상태**로 필터링할 수 있습니다
3. 템플릿을 클릭하여 열어주세요
4. **제목** 줄을 편집합니다 (고객의 인보크에서 표시되는 이메일 제목)
5. 이메일의 전체 디자인 버전을 위해 **HTML 콘텐츠**를 편집합니다
6. 선택적으로 **텍스트 콘텐츠**를 편집합니다 - HTML을 지원하지 않는 이메일 클라이언트를 위한 기본 텍스트입니다
7. **저장**을 클릭합니다

> **HTML 이메일:** HTML 콘텐츠 필드는 표준 HTML과 인라인 CSS를 포함합니다. Spwig은 이 내용을 올바르게 포맷된 이메일로 렌더링합니다. MJML 마크업을 사용하는 경우, 저장 시 자동으로 컴파일됩니다.

## 템플릿 미리보기

저장하기 전에 템플릿이 이메일 클라이언트에서 어떻게 표시되는지 미리보기할 수 있습니다:

1. 미리보기하려는 템플릿을 엽니다
2. **미리보기** 버튼을 클릭합니다 (템플릿 목록 또는 템플릿 상세 페이지에서 보입니다)
3. 렌더링된 이메일을 보여주는 새 브라우저 탭이 열립니다

이 기능은 템플릿이 공식적으로 사용되기 전에 레이아웃, 포맷, 플레이스홀더 변수의 표시 방식을 확인할 수 있도록 합니다.

## 템플릿 변수

변수는 템플릿 내의 플레이스홀더로, Spwig이 이메일을 전송할 때 실제 데이터로 대체됩니다. 이는 `{{ 변수 이름 }}`과 같이 작성됩니다.

대부분의 템플릿에서 사용 가능한 일반적인 변수:


| 변수 | 대체 내용 |
|----------|---------------|
| `{{ customer_name }}` | 고객의 전체 이름 |
| `{{ order_number }}` | 주문 참조 번호 |
| `{{ order_total }}` | 주문 총 금액 |
| `{{ store_name }}` | 귀하의 가게 이름 |
| `{{ store_url }}` | 귀하의 가게 웹 주소 |
| `{{ tracking_number }}` | 배송 추적 번호 |
| `{{ tracking_url }}` | 배송 추적을 위한 클릭 가능한 링크 |

템플릿 유형에 따라 사용 가능한 변수가 다릅니다. 주문 관련 템플릿(예: `{{ order_number }}`)에 해당하는 변수는 계정 관련 템플릿(예: 비밀번호 재설정)에서는 사용할 수 없습니다. 적용되지 않는 변수를 포함하면 빈 상태나 대체되지 않은 상태로 나타납니다.

## 언어 지원

각 템플릿 유형은 가게가 지원하는 각 언어에 대해 버전을 가질 수 있습니다. 템플릿의 **언어** 필드는 활성화된 언어 버전을 제어합니다.

Spwig은 전송 시 고객의 언어 선호도에 따라 적절한 언어 버전을 자동으로 선택합니다. 고객의 언어에 해당하는 템플릿이 존재하지 않으면 Spwig은 영어 버전으로 대체합니다.

새로운 언어에 대한 템플릿을 추가하려면:
1. 기존 템플릿을 열어주세요
2. **Actions** 메뉴에서 **Clone Template**을 클릭하세요
3. 클론의 **Language Code**를 새 언어로 설정하세요
4. 내용을 번역하세요
5. 클론된 템플릿을 활성화하세요

## 템플릿 복제, 활성화 및 비활성화

### 템플릿 복제

복제는 템플릿의 정확한 사본을 생성합니다. 이는 언어 변형을 만들거나 라이브 템플릿에 영향을 주지 않고 다른 버전을 테스트하는 데 유용합니다.

1. 목록에서 하나 이상의 템플릿을 선택하세요
2. **Actions** 드롭다운에서 **Clone selected templates**을 선택하세요
3. 복제본은 비활성 상태로 생성됩니다. 준비가 되면 편집하고 활성화하세요

### 템플릿 활성화 및 비활성화

템플릿이 **활성화**되어 있어야 전송에 사용될 수 있습니다. 각 유형과 언어 조합에 대해 한 번에 하나의 활성 템플릿만 사용됩니다.

대량으로 활성화하거나 비활성화하려면:
1.

템플릿 선택
2.

**Actions** 드롭다운에서 **선택한 템플릿 활성화** 또는 **선택한 템플릿 비활성화**를 선택합니다.

또는 개별 템플릿을 열고 **활성화** 확인란을 토글합니다.

## 시스템 템플릿

**System** 배지로 표시된 템플릿은 Spwig에서 기본으로 제공하는 템플릿입니다. 이 템플릿은 삭제할 수 없습니다. 직접 편집하거나 복사하여 사용자 정의 버전을 만들 수 있습니다.

## 팁

- 템플릿을 편집한 후에는 항상 미리보기를 확인하여 고객이 보기 전에 포맷 문제를 잡습니다.
- 제목은 짧고 구체적으로 유지하세요 — `Your order #10045 has shipped`은 `Update from our store`와 같은 일반적인 제목보다 더 효과적입니다.
- 플레인 텍스트 내용도 편집하세요 — 일부 이메일 클라이언트는 플레인 텍스트 버전만 표시하며, 일부 고객은 이를 선호합니다.
- 템플릿의 영어 버전을 복사하여 번역 버전을 만드는 시작점으로 사용하세요.
- 라이브 이메일에 영향을 주지 않고 변경 사항을 테스트하고자 한다면, 템플릿을 복사한 후 복사본을 편집하고, 미리보기를 확인하는 동안 두 템플릿을 모두 활성화한 상태로 유지한 후 원본을 비활성화하세요.
- 관리자 알림 템플릿(예: **Admin: New Order**)은 가게의 관리자 이메일 주소로 전송됩니다 — 가게 설정에서 해당 이메일 주소가 올바르게 설정되어 있는지 확인하세요.
