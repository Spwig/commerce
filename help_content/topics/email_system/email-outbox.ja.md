---
title: メール送信履歴
---

メール送信履歴は、店舗が送信または送信しようとしたすべてのメールの完全なログです — 注文確認、配送状況の更新、管理者レポート、およびその他のすべてのトランザクショナルメッセージが含まれます。配信の確認、失敗の調査、メールキューの管理に使用できます。

**メールシステム > メール送信履歴**に移動してメールログを表示します。

![メール送信履歴リストとステータスバッジ](/static/core/admin/img/help/email-outbox/outbox-list.webp)

## 送信履歴の閲覧

上部のサマリーバーには、各ステータスカテゴリのカウントが表示されます。下のリストには以下が表示されます:

- **件名** — メールの件名
- **宛先** — 受信者のメールアドレス
- **送信元** — 使用された送信元アドレス
- **ステータス** — 現在の配信ステータス
- **キューイング日時** — メールがキューに入った時刻
- **送信日時** — メールがプロバイダーに送信された時刻
- **再試行回数** — 送信が試行された回数

## メールステータス

| ステータス | 意味 |
|----------|------|
| キューイング中 | メールは送信を待っている状態です |
| 送信中 | メールは現在プロバイダーに送信されています |
| 送信済み | プロバイダーがメールを受け入れました |
| 保留中 | メールは保留されており、解放されるまで送信されません |
| ログ済み | メールは記録されていますが送信されていません（テストモードまたはログ記録専用設定） |
| 失敗 | プロバイダーがメールを拒否または配信できませんでした |
| バウンド | メールは送信されましたがあの受信者のメールサーバーから戻ってきました |
| スキップ | システムの理由で送信がスキップされました |

## メールの詳細を表示

リスト内の任意のメールをクリックして、完全な詳細を表示します:

- メールの完全な **HTML本文** と **テキスト本文**
- **プロバイダメッセージID** — あなたのメールプロバイダからの参照（プロバイダサポートに連絡するときに使用してください）
- **エラーメッセージ** — 失敗またはバウンドしたメールの正確なエラー
- **再試行回数** と **最大再試行回数** — 送信が試行された回数
- すべてのタイムスタンプ: 作成日時、キューイング日時、送信日時、失敗日時

## 送信履歴のフィルタリング

右側のフィルタを使用して、表示を絞り込みます:

- **ステータス** — 特定の配信ステータスのメールのみを表示します
- **日付** — メールが作成または送信された日時でフィルタリングします
- **テンプレートタイプ** — 特定の通知タイプのメールのみを表示します（例：注文確認メールのみ）

上部の検索ボックスは、件名、受信者アドレス、送信者アドレス、またはプロバイダのメッセージIDで検索します。

## 保留中のメールのリリース

**保留中**ステータスのメールは一時停止されています — これらはリリースするまで送信されません。メールが保留される理由としては、生成されたときにストアがメンテナンスモードになっていた、または管理者の操作によって保留された場合があります。

保留中のメールをリリースするには:
1. リリースしたいメールを選択します（左側のチェックボックスをクリックします）
2. **Actions**（アクション）ドロップダウンから **Release held emails for delivery**（配信用に保留中のメールをリリース）を選択します
3. **Go**（実行）をクリックします

リリースされたメールは **Queued**（キューに）ステータスに移動し、次のキュー処理サイクルで送信されます。

## 予約メール

一部のメールは将来の時間に送信されるように予約されています — 週次ダイジェストレポートなどは、特定の日時で送信されるように予約されています。**Email System > Scheduled Emails**（メールシステム > 予約メール）に移動して、今後の予約送信を確認してください。

予約メール一覧には以下が表示されます:
- **Template Type**（テンプレートタイプ） — 予約されたメールのタイプ
- **Recipient Email**（受信者メール） — 送信先のアドレス
- **Scheduled For**（予約日時） — 送信予定の日時
- **Status**（ステータス） — 未送信（Pending）、送信済み（Sent）、または失敗（Failed）

予約メールは、予約された時間に自動的に処理されます — 手動での操作は必要ありません。

## 失敗した配信のトラブルシューティング

メールが **Failed**（失敗）ステータスを示している場合、エラーメッセージを確認するためにクリックし、以下の手順に従ってください:

### 一般的な原因と対処法


| 症状 | 起因 | 対処方法 |
|---------|-------------|------------|
| "認証失敗" | メールプロバイダの資格情報が無効 | **メールシステム > メールアカウント**で資格情報を更新 |
| "接続拒否" / "タイムアウト" | メールサーバに到達できません | メールプロバイダのステータスページを確認してください；**メールアカウント**で接続をテスト |
| "無効な受信者" | 顧客のメールアドレスが不正 | 顧客のアカウントを確認し、メールアドレスを修正 |
| バウンドしたメール | 受信者のメールサーバがメールを拒否 | アドレスが存在しないか、インボックスがいっぱいかもしれません；過剰に再試行しないでください |
| 突然の高い失敗率 | プロバイダの問題または資格情報が期限切れ | プロバイダのステータスを確認；**メールアカウント**で接続を再テスト |

### メールアカウントの接続を確認する

多くのメールが失敗している場合、メールアカウントをテストしてください：

1. **メールシステム > メールアカウント**に移動
2. 有効なアカウントを見つけ、その**接続**ステータスを確認
3. 接続にエラーが表示されている場合、アカウントをクリックし、**接続テスト**オプションを使用して問題を診断

### 再試行の動作

Spwigは、**最大再試行回数**の制限まで失敗したメールを自動的に再試行します。各メールに表示されている再試行回数は、これまでに試行された回数を示しています。再試行回数の上限に達すると、メールは**失敗**のステータスのままになり、その後の自動再試行は行われません。

## バウンドしたメール

**バウンド**したメールは送信されたが、受信者のメールサーバによって返されました。バウンドは2種類あります：

- **ハードバウンド** — メールアドレスが存在しない、またはドメインがメールを受け入れない。ハードバウンドは再試行しないでください；アドレスは無効です
- **ソフトバウンド** — 一時的な問題（インボックスがいっぱい、サーバーが一時的に利用不可）。再試行で成功する可能性があります

同じアドレスへの繰り返しのバウンドは、メールプロバイダとの送信者評判を損なう可能性があります。同じ顧客アドレスへの繰り返しのバウンドが見られる場合、そのアドレスを顧客のアカウントから更新または削除してください。

## ヒント

すべてのMarkdownの書式、画像のパス、コードブロック、および技術用語を保持してください。

- フラッシュセールや大規模な商品発売などの主要なイベントの後、すべての注文確認メールが正常に送信されたかを確認するために、出力ボックスを確認してください
- 顧客がメールを受信しなかったと述べている場合、そのメールアドレスで出力ボックスを検索し、送信されたか、失敗したか、スキップされたかを確認してください
- 失敗数の急激な増加は、資格情報やアカウントの問題を示している可能性があります — **メールアカウント**をすぐに確認してください
- **保留中**のステータスは失敗ではありません — これはメールが待機していることを意味します。

保留中のメールを送信準備ができたら解放してください
- **テンプレートタイプ**フィルターを使用して、特定のタイプのすべてのメールを迅速に監査できます — たとえば、過去7日間のすべての注文確認メールが**送信済み**のステータスになっているかを確認します
- リストの上部にある日/月/年の日付階層ナビゲーションは、特定の期間の出力ボックスを確認するために役立ちます
