---
title: 自定義欄位
---

自定義欄位讓您可以在不修改任何程式碼的情況下，為產品、分類、訂單和客戶個人資料添加額外資料。用於存儲特定於業務的資訊，例如外部API ID、倉庫位置、合規數據，或您商店所需的任何屬性。

## 瀏覽自定義欄位

在管理側邊欄導航到 **設定 > 自定義欄位**。

![自定義欄位頁面](/static/core/admin/img/help/custom-fields/custom-fields-page.webp)

## 關鍵概念

### 欄位群組

欄位被組織成 **群組** — 作為一個部分一起顯示的邏輯集合。例如，一個 "運輸資訊" 群組可能包含倉庫位置、包裝尺寸和危險品分類的欄位。

### 欄位定義

每個欄位定義控制：
- **名稱**：在表單中顯示的標籤
- **Slug**：在JSON存儲和API回應中使用的可讀機器鍵
- **欄位類型**：渲染的輸入類型（文字、數字、下拉選單等）
- **驗證**：如最小/最大值、最大長度、正則表達式或允許的選擇項等規則
- **可見性**：欄位是否出現在商店前端

### 支援的欄位類型

| 類型 | 說明 | 使用範例 |
|------|-------------|-------------|
| **文字** | 單行文字輸入 | 外部API ID、品牌代碼 |
| **文字區塊** | 多行文字 | 特殊處理說明 |
| **數字** | 整數值 | 最小訂購數量 |
| **小數** | 小數值 | 重量覆蓋、自定義尺寸 |
| **是/否** | 勾選框切換 | 是否易碎、需要簽收 |
| **日期** | 日期選擇器 | 發布日期、過期日期 |
| **日期和時間** | 日期和時間選擇器 | 預定的可用性 |
| **URL** | 網站地址 | 供應商連結、規格表URL |
| **電子郵件** | 電子郵件地址 | 廠商聯繫 |
| **下拉選單** | 單選列表 | 材料類型、原產國 |
| **多選** | 多選列表 | 認證、標籤 |
| **顏色** | 顏色選擇器 | 品牌顏色、標籤顏色 |

## 管理自定義欄位

### 建立欄位群組

1. 開啟 **設定 > 自定義欄位**
2. 選擇模型標籤（產品、分類、訂單或客戶個人資料）
3. 點擊 **新增群組**
4. 輸入 **群組名稱**（例如，"外部整合"）
5. 可選地啟用 **在商店前端顯示**，如果客戶應該看到這些欄位
6. 點擊 **儲存群組**

### 將欄位加入群組

1. 在群組卡片上點擊 **新增欄位**
2. 輸入 **欄位名稱** — slug 會自動生成
3. 選擇 **欄位類型**
4. 可選地設置 **幫助文字** 和 **預設值**
5. 設置驗證選項（根據欄位類型而異）：
   - 文字：最大長度、正則表達式模式
   - 數字/小數：最小和最大值
   - 下拉選單：定義選擇項列表
6. 設置欄位選項：
   - **必填**：商家在保存時必須填寫此欄位
   - **在商店前端顯示**：在客戶面對的頁面上顯示值
   - **可翻譯**：允許值被翻譯（僅限文字/文字區塊）
7. 點擊 **儲存欄位**

### 編輯和重新排序

- 點擊任何群組或欄位的 **鉛筆圖示** 來編輯它
- 拖動 **握把** 來重新排序群組或群組內的欄位
- 所有相關表單會立即生效這些更改

### 刪除群組和欄位

- 點擊群組或欄位的 **垃圾桶圖示** 來刪除它
- 刪除是 **軟刪除** — 資料在資料庫中保留但從表單中隱藏
- 這保護了現有資料免受意外丟失

## 在表單中使用自定義欄位

一旦為模型定義了自定義欄位，對應的編輯表單上會自動出現 **自定義欄位** 標籤。

### 產品和分類

1. 開啟任何產品或分類進行編輯
2. 點擊 **自定義欄位** 標籤
3. 根據需要填寫欄位
4. 點擊 **儲存** — 值會與記錄一起存儲

### 訂單

訂單的自定義欄位值在訂單詳情頁上顯示為 **只讀部分**。訂單自定義欄位通常通過API或結帳時設置。

### 客戶個人資料

1. 開啟客戶個人資料
2. 點擊 **自定義欄位** 標籤
3. 填寫欄位並保存

## API存取

### 列出欄位定義

獲取模型的所有自定義欄位定義：

```
GET /api/custom-fields/definitions/?model=product&app=catalog
```

**回應：**
```json
[
  {
    "id": 1,
    "name": "External API ID",
    "slug": "external_api_id",
    "field_type": "text",
    "is_required": false,
    "group": { "name": "External Integrations" }
  }
]
```

### 讀取自定義欄位值

自定義欄位值包含在模型API回應的 `custom_fields` JSON物件中：

```json
{
  "id": 42,
  "name": "Blue Widget",
  "custom_fields": {
    "external_api_id": "API-12345",
    "is_fragile": true
  }
}
```

### 寫入自定義欄位值

透過API創建或更新記錄時，包含 `custom_fields`：

```json
{
  "custom_fields": {
    "external_api_id": "API-67890",
    "warehouse_location": "WH-A3"
  }
}
```

值會根據欄位定義進行驗證。無效值會返回 `400` 錯誤，並附帶詳細資訊。

### 透過自定義欄位查詢

自定義欄位已建立索引，以便快速數據庫查詢。使用數據庫查詢過濾器篩選記錄：

```
GET /api/products/?custom_fields__warehouse_location=WH-A3
```

## 商店前端顯示

### 給主題開發者

使用 `render_custom_fields` 模板標籤在商店前端顯示自定義欄位：

```python
{% load custom_fields_tags %}

{# 呈現所有商店前端可見的欄位 #}
{% render_custom_fields product %}

{# 獲取特定欄位值 #}
{% get_custom_field product "warehouse_location" as location %}
<p>發貨地點：{{ location }}</p>
```

只有在群組和欄位層級都啟用了 **在商店前端顯示** 的欄位才會被渲染。

## 最佳實踐

- **使用描述性名稱** — 欄位名稱會出現在表單和商店前端
- **設置幫助文字** — 指導商家在每個欄位中輸入什麼內容
- **將相關欄位分組** — 保持表單組織和直覺
- **使用預設值** — 設置合理的預設值以減少數據輸入
- **在商店前端顯示時要謹慎** — 只顯示對客戶有意義的欄位
- **在整合中使用slug** — slug 是穩定的識別碼；欄位名稱可以更改

## 故障排除

**自定義欄位標籤未出現：**
- 確認該模型至少有一個活動的欄位群組存在
- 檢查管理類是否包含 `CustomFieldsAdminMixin`
- 清除緩存並重新整理頁面

**欄位值未保存：**
- 確保已填寫必填欄位
- 檢查驗證規則（最小/最大值、正則表達式模式、允許的選擇項）
- 確認欄位處於活動狀態且未被軟刪除

**API返回空的 custom_fields：**
- 確認模型具有 `CustomFieldsMixin`
- 檢查是否為正確的內容類型存在欄位定義
- 確保序列化器包含 `CustomFieldsSerializerMixin`
