---
title: 自定义字段
---

自定义字段允许你在不修改任何代码的情况下，向产品、分类、订单和客户资料中添加额外的数据。使用它们来存储特定于业务的信息，如外部API ID、仓库位置、合规数据或您商店所需的任何属性。

## 访问自定义字段

在管理侧边栏中导航到 **设置 > 自定义字段**。

![自定义字段页面](/static/core/admin/img/help/custom-fields/custom-fields-page.webp)

## 关键概念

### 字段组

字段被组织到 **组** 中——作为部分出现的逻辑集合。例如，一个“运输信息”组可能包含仓库位置、包裹尺寸和危险品分类的字段。

### 字段定义

每个字段定义控制：
- **名称**：在表单中显示的标签
- **Slug**：在JSON存储和API响应中使用的可读机器键
- **字段类型**：渲染的输入类型（文本、数字、下拉菜单等）
- **验证**：如最小/最大值、最大长度、正则表达式或允许选择的规则
- **可见性**：字段是否出现在商店前端

### 支持的字段类型

| 类型 | 描述 | 示例用法 |
|------|-------------|-------------|
| **文本** | 单行文本输入 | 外部API ID，品牌代码 |
| **文本区域** | 多行文本 | 特殊处理说明 |
| **数字** | 整数值 | 最小订单数量 |
| **小数** | 小数值 | 重量覆盖，自定义尺寸 |
| **是/否** | 复选框切换 | 是否易碎，需要签名 |
| **日期** | 日期选择器 | 发布日期，过期日期 |
| **日期和时间** | 日期和时间选择器 | 预定可用性 |
| **URL** | 网络地址 | 供应商链接，规格表URL |
| **电子邮件** | 电子邮件地址 | 制造商联系方式 |
| **下拉菜单** | 单选列表 | 材料类型，原产国 |
| **多选** | 多选列表 | 认证，标签 |
| **颜色** | 颜色选择器 | 品牌颜色，标签颜色 |

## 管理自定义字段

### 创建字段组

1. 打开 **设置 > 自定义字段**
2. 选择模型标签（产品、分类、订单或客户资料）
3. 点击 **添加组**
4. 输入 **组名称**（例如，“外部集成”）
5. 如果客户应看到这些字段，可选择启用 **在商店前端显示**
6. 点击 **保存组**

### 将字段添加到组中

1. 在组卡片上，点击 **添加字段**
2. 输入 **字段名称**——slug会自动生成
3. 选择 **字段类型**
4. 可选地设置 **帮助文本**和**默认值**
5. 配置验证选项（根据字段类型而异）：
   - 文本：最大长度，正则表达式模式
   - 数字/小数：最小和最大值
   - 下拉菜单：定义选择列表
6. 设置字段选项：
   - **必填**：商家在保存时必须填写此字段
   - **在商店前端显示**：在客户面向的页面上显示值
   - **可翻译**：允许值被翻译（仅限文本/文本区域）
7. 点击 **保存字段**

### 编辑和重新排序

- 点击任何组或字段的 **铅笔图标** 进行编辑
- 拖动 **手柄** 以重新排序组或组内的字段
- 所有相关表单会立即生效更改

### 删除组和字段

- 点击组或字段的 **垃圾图标** 进行删除
- 删除是 **软删除**——数据在数据库中保留但隐藏在表单中
- 这保护了现有数据免受意外丢失

## 在表单中使用自定义字段

一旦为模型定义了自定义字段，相应的编辑表单上会自动出现 **自定义字段** 选项卡。

### 产品和分类

1. 打开任何产品或分类进行编辑
2. 点击 **自定义字段** 选项卡
3. 根据需要填写字段
4. 点击 **保存**——值将与记录一起存储

### 订单

订单的自定义字段值在订单详情页面上显示为 **只读部分**。订单的自定义字段通常通过API或结账时设置。

### 客户资料

1. 打开客户资料
2. 点击 **自定义字段** 选项卡
3. 填写字段并保存

## API访问

### 列出字段定义

获取模型的所有自定义字段定义：

```
GET /api/custom-fields/definitions/?model=product&app=catalog
```

**响应：**
```json
[
  {
    "id": 1,
    "name": "External API ID",
    "slug": "external_api_id",
    "field_type": "text",
    "is_required": false,
    "group": { "name": "External Integrations" }
  }
]
```

### 读取自定义字段值

自定义字段值包含在模型API响应的 `custom_fields` JSON对象中：

```json
{
  "id": 42,
  "name": "Blue Widget",
  "custom_fields": {
    "external_api_id": "API-12345",
    "is_fragile": true
  }
}
```

### 写入自定义字段值

通过API创建或更新记录时，包含 `custom_fields`：

```json
{
  "custom_fields": {
    "external_api_id": "API-67890",
    "warehouse_location": "WH-A3"
  }
}
```

值会根据字段定义进行验证。无效的值会返回带有详细信息的 `400` 错误。

### 通过自定义字段查询

自定义字段已为快速数据库查询进行索引。使用数据库查询过滤器筛选记录：

```
GET /api/products/?custom_fields__warehouse_location=WH-A3
```

## 商店前端显示

### 对于主题开发者

使用 `render_custom_fields` 模板标签在商店前端显示自定义字段：

```python
{% load custom_fields_tags %}

{# 渲染所有商店前端可见的字段 #}
{% render_custom_fields product %}

{# 获取特定字段值 #}
{% get_custom_field product "warehouse_location" as location %}
<p>发货地：{{ location }}</p>
```

只有在组和字段级别都启用了 **在商店前端显示** 的字段才会被渲染。

## 最佳实践

- **使用描述性名称**——字段名称会出现在表单和商店前端
- **设置帮助文本**——指导商家在每个字段中输入什么内容
- **将相关字段分组**——保持表单有条理且直观
- **使用默认值**——设置合理的默认值以减少数据输入
- **在商店前端显示时要选择性**——只显示对客户有意义的字段
- **在集成中使用slug**——slug是稳定的标识符；字段名称可以更改

## 故障排除

**自定义字段选项卡未出现：**
- 确认该模型至少有一个活动的字段组存在
- 检查管理类是否包含 `CustomFieldsAdminMixin`
- 清除缓存并刷新页面

**字段值未保存：**
- 确保填写了必填字段
- 检查验证规则（最小/最大值、正则表达式模式、允许的选择）
- 确认字段处于活动状态且未被软删除

**API返回空的 custom_fields：**
- 确认模型具有 `CustomFieldsMixin`
- 检查是否为正确的数据内容类型存在字段定义
- 确保序列化器包含 `CustomFieldsSerializerMixin`
