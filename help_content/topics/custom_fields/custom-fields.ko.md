---
title: 사용자 정의 필드
---

사용자 정의 필드는 제품, 카테고리, 주문 및 고객 프로필에 추가 데이터를 추가할 수 있도록 해주며, 코드를 수정하지 않고도 사용할 수 있습니다. 외부 API ID, 창고 위치, 규제 데이터 또는 가게가 필요한 기타 속성을 저장하는 등 비즈니스 관련 정보를 저장하는 데 사용할 수 있습니다.

## 사용자 정의 필드에 접근

관리자 사이드바에서 **설정 > 사용자 정의 필드**로 이동합니다.

![사용자 정의 필드 페이지](/static/core/admin/img/help/custom-fields/custom-fields-page.webp)

## 핵심 개념

### 필드 그룹

필드는 **그룹**으로 구성되며, 논리적인 집합으로 표시됩니다. 예를 들어, "배송 정보" 그룹에는 창고 위치, 패키지 크기, 위험물 분류 필드가 포함될 수 있습니다.

### 필드 정의

각 필드 정의는 다음을 제어합니다:
- **이름**: 폼에서 표시되는 라벨
- **슬러그**: JSON 저장 및 API 응답에서 사용되는 기계가 읽을 수 있는 키
- **필드 유형**: 렌더링되는 입력 유형 (텍스트, 숫자, 드롭다운 등)
- **검증**: 최소/최대, 최대 길이, 정규식, 허용된 선택지 등 규칙
- **가시성**: 필드가 쇼핑몰에 표시되는지 여부

### 지원되는 필드 유형

| 유형 | 설명 | 예시 사용 |
|------|-------------|-------------|
| **텍스트** | 단일 줄 텍스트 입력 | 외부 API ID, 브랜드 코드 |
| **텍스트 영역** | 다중 줄 텍스트 | 특별한 처리 노트 |
| **숫자** | 정수 값 | 최소 주문 수량 |
| **소수** | 소수 값 | 무게 재설정, 사용자 정의 차원 |
| **예/아니요** | 확인란 전환 | 취약한지, 서명이 필요한지 |
| **날짜** | 날짜 선택기 | 출시 날짜, 유효 기한 |
| **날짜 및 시간** | 날짜 및 시간 선택기 | 예약된 가용성 |
| **URL** | 웹 주소 | 공급업체 링크, 사양 시트 URL |
| **이메일** | 이메일 주소 | 제조사 연락처 |
| **드롭다운** | 단일 선택 목록 | 재료 유형, 원산지 국가 |
| **다중 선택** | 다중 선택 목록 | 인증, 태그 |
| **색상** | 색상 선택기 | 브랜드 색상, 라벨 색상 |

## 사용자 정의 필드 관리

### 필드 그룹 생성

1. **설정 > 사용자 정의 필드**를 엽니다
2. 모델 탭(제품, 카테고리, 주문, 또는 고객 프로필)을 선택합니다
3. **그룹 추가**를 클릭합니다
4. **그룹 이름**을 입력합니다 (예: "외부 통합")
5. 고객이 이 필드를 보아야 한다면 **쇼핑몰에 표시**를 선택적으로 활성화합니다
6. **그룹 저장**을 클릭합니다

### 그룹에 필드 추가

1. 그룹 카드에서 **필드 추가**를 클릭합니다
2. **필드 이름**을 입력합니다 - 슬러그는 자동으로 생성됩니다
3. **필드 유형**을 선택합니다
4. 선택적으로 **도움말 텍스트** 및 **기본값**을 설정합니다
5. 검증 옵션을 구성합니다 (필드 유형에 따라 다름):
   - 텍스트: 최대 길이, 정규식 패턴
   - 숫자/소수: 최소 및 최대 값
   - 드롭다운: 선택지 목록을 정의합니다
6. 필드 옵션을 설정합니다:
   - **필수**: 판매자들이 저장할 때 이 필드를 채워야 합니다
   - **쇼핑몰에 표시**: 고객이 보는 페이지에 값을 표시합니다
   - **번역 가능**: 값이 번역될 수 있도록 허용합니다 (텍스트/텍스트 영역만 해당)
7. **필드 저장**을 클릭합니다

### 편집 및 재정렬

- 그룹 또는 필드의 **연필 아이콘**을 클릭하여 편집합니다
- **잡기 손잡이**를 드래그하여 그룹 내에서 그룹 또는 필드를 재정렬합니다
- 변경 사항은 관련된 모든 폼에서 즉시 적용됩니다

### 그룹 및 필드 삭제

- 그룹 또는 필드의 **쓰레기 아이콘**을 클릭하여 삭제합니다
- 삭제는 **소프트 삭제**입니다 - 데이터는 데이터베이스에 보존되지만 폼에서 숨겨집니다
- 이는 기존 데이터가 실수로 손실되는 것을 방지합니다

## 폼에서 사용자 정의 필드 사용

모델에 사용자 정의 필드를 정의하면 해당 편집 폼에 **사용자 정의 필드** 탭이 자동으로 나타납니다.

### 제품 및 카테고리

1. 편집할 제품 또는 카테고리를 엽니다
2. **사용자 정의 필드** 탭을 클릭합니다
3. 필요한 필드를 입력합니다
4. **저장**을 클릭합니다 - 값은 기록과 함께 저장됩니다

### 주문

주문의 사용자 정의 필드 값은 주문 세부 정보 페이지에서 **읽기 전용 섹션**으로 표시됩니다. 주문 사용자 정의 필드는 일반적으로 API 또는 체크아웃 시 설정됩니다.

### 고객 프로필

1. 고객 프로필을 엽니다
2. **사용자 정의 필드** 탭을 클릭합니다
3. 필드를 입력하고 저장합니다

## API 접근

### 필드 정의 목록

모델에 대한 모든 사용자 정의 필드 정의를 검색합니다:

```
GET /api/custom-fields/definitions/?model=product&app=catalog
```

**응답:**
```json
[
  {
    "id": 1,
    "name": "외부 API ID",
    "slug": "external_api_id",
    "field_type": "text",
    "is_required": false,
    "group": { "name": "외부 통합" }
  }
]
```

### 사용자 정의 필드 값 읽기

사용자 정의 필드 값은 모델 API 응답의 `custom_fields` JSON 객체에 포함됩니다:

```json
{
  "id": 42,
  "name": "Blue Widget",
  "custom_fields": {
    "external_api_id": "API-12345",
    "is_fragile": true
  }
}
```

### 사용자 정의 필드 값 작성

API를 통해 기록을 생성하거나 업데이트할 때 `custom_fields`를 포함합니다:

```json
{
  "custom_fields": {
    "external_api_id": "API-67890",
    "warehouse_location": "WH-A3"
  }
}
```

값은 필드 정의에 대해 검증됩니다. 유효하지 않은 값은 세부 정보와 함께 `400` 오류를 반환합니다.

### 사용자 정의 필드로 쿼리

사용자 정의 필드는 빠른 데이터베이스 쿼리에 인덱싱되어 있습니다. 데이터베이스 쿼리 필터를 사용하여 기록을 필터링합니다:

```
GET /api/products/?custom_fields__warehouse_location=WH-A3
```

## 쇼핑몰 표시

### 테마 개발자용

쇼핑몰에서 사용자 정의 필드를 표시하려면 `render_custom_fields` 템플릿 태그를 사용합니다:

```python
{% load custom_fields_tags %}

{# 쇼핑몰에서 표시되는 모든 필드 렌더링 #}
{% render_custom_fields product %}

{# 특정 필드 값 가져오기 #}
{% get_custom_field product "warehouse_location" as location %}
<p>발송 출처: {{ location }}</p>
```

그룹 및 필드 수준에서 **쇼핑몰에 표시**가 활성화된 필드만 렌더링됩니다.

## 최선의 실천

- **설명적인 이름 사용** - 필드 이름은 폼과 쇼핑몰에 표시됩니다
- **도움말 텍스트 설정** - 판매자가 각 필드에 입력해야 할 내용을 안내합니다
- **관련 필드 그룹화** - 폼을 정리하고 직관적으로 유지합니다
- **기본값 사용** - 데이터 입력을 줄이기 위해 합리적인 기본값을 설정합니다
- **쇼핑몰 가시성에 신중하게 접근** - 고객에게 의미 있는 필드만 표시합니다
- **통합에 슬러그 사용** - 슬러그는 안정적인 식별자입니다; 필드 이름은 변경될 수 있습니다

## 문제 해결

**사용자 정의 필드 탭이 나타나지 않음:**
- 해당 모델에 활성화된 필드 그룹이 최소한 하나 있는지 확인합니다
- 관리자 클래스가 `CustomFieldsAdminMixin`을 포함하는지 확인합니다
- 캐시를 지우고 페이지를 새로고침합니다

**필드 값이 저장되지 않음:**
- 필수 필드가 채워져 있는지 확인합니다
- 검증 규칙(최소/최대, 정규식 패턴, 허용된 선택지)을 확인합니다
- 필드가 활성화되어 있고 소프트 삭제되지 않았는지 확인합니다

**API가 빈 custom_fields를 반환함:**
- 모델이 `CustomFieldsMixin`을 포함하고 있는지 확인합니다
- 해당 콘텐츠 유형에 대한 필드 정의가 존재하는지 확인합니다
- 시리얼라이저가 `CustomFieldsSerializerMixin`을 포함하고 있는지 확인합니다
