---
title: カスタムフィールド
---

カスタムフィールドを使用すると、コードを変更せずに製品、カテゴリ、注文、顧客プロファイルに追加のデータを追加できます。外部API ID、倉庫の場所、コンプライアンスデータ、またはストアが必要とする任意の属性などの、ビジネス固有の情報を保存するために使用します。

## カスタムフィールドへのアクセス

管理サイドバーで **Settings > Custom Fields** に移動してください。

![カスタムフィールドページ](/static/core/admin/img/help/custom-fields/custom-fields-page.webp)

## 主な概念

### フィールドグループ

フィールドは **グループ** に整理されており、セクションとして一緒に表示されます。たとえば、「配送情報」グループには、倉庫の場所、梱包の寸法、危険物分類のフィールドが含まれる場合があります。

### フィールド定義

各フィールド定義は次の内容を制御します：
- **名前**：フォームに表示されるラベル
- **スラッグ**：JSONストレージおよびAPI応答で使用される機械読み取り可能なキー
- **フィールドタイプ**：レンダリングされる入力の種類（テキスト、数値、ドロップダウンなど）
- **検証**：最小/最大、最大長、正規表現、または許容される選択肢などのルール
- **可視性**：フィールドがストアフロントに表示されるかどうか

### サポートされているフィールドタイプ

| タイプ | 説明 | 例 |
|------|-------------|-------------|
| **テキスト** | 単一行のテキスト入力 | 外部API ID、ブランドコード |
| **テキストエリア** | 複数行のテキスト | 特別な取り扱いに関するノート |
| **数値** | 整数値 | 最小注文数量 |
| **小数** | 小数値 | 重量の上書き、カスタム寸法 |
| **はい/いいえ** | チェックボックス切り替え | 破損の可能性あり、署名が必要 |
| **日付** | 日付ピッカー | 発売日、有効期限 |
| **日付と時刻** | 日付と時刻ピッカー | 予定された在庫可能時間 |
| **URL** | ウェブアドレス | 仕入れ先リンク、仕様シートURL |
| **メール** | メールアドレス | 製造元の連絡先 |
| **ドロップダウン** | 単一選択リスト | 材質タイプ、原産国 |
| **複数選択** | 複数選択リスト | 認証、タグ |
| **色** | 色選択 | ブランドカラー、ラベルカラー |

## カスタムフィールドの管理

### フィールドグループの作成

1. **Settings > Custom Fields** を開きます
2. モデルタブ（製品、カテゴリ、注文、または顧客プロファイル）を選択します
3. **Add Group** をクリックします
4. **Group Name** を入力します（例: "外部統合"）
5. これらのフィールドを顧客が表示する必要がある場合は、**Show on storefront** を有効にする（オプション）
6. **Save Group** をクリックします

### グループにフィールドを追加

1. グループカードで **Add Field** をクリックします
2. **Field Name** を入力します — スラッグは自動的に生成されます
3. **Field Type** を選択します
4. 必要に応じて **Help Text** と **Default Value** を設定します
5. 検証オプションを構成します（フィールドタイプによって異なります）：
   - テキスト: 最大長、正規表現パターン
   - 数値/小数: 最小値と最大値
   - ドロップダウン: 選択肢のリストを定義
6. フィールドオプションを設定します：
   - **必須**: 商家が保存するときにこのフィールドを入力する必要があります
   - **ストアフロントに表示**: 顧客向けのページに値を表示します
   - **翻訳可能**: 値を翻訳できるようにします（テキスト/テキストエリアのみ）
7. **Save Field** をクリックします

### 編集と並び順の変更

- 任意のグループまたはフィールドの **編集アイコン** をクリックして編集します
- **グリップハンドル** をドラッグして、グループ内またはグループ間のフィールドの並び順を変更します
- 変更はすぐにすべての関連フォームに反映されます

### グループとフィールドの削除

- グループまたはフィールドの **ゴミ箱アイコン** をクリックして削除します
- 削除は **ソフト削除** です — データはデータベースに保持されますが、フォームから非表示になります
- これにより、既存データの誤った削除を防ぎます

## フォームでのカスタムフィールドの使用

モデルに対してカスタムフィールドを定義すると、対応する編集フォームに **カスタムフィールド** タブが自動的に表示されます。

### 製品とカテゴリ

1. 任意の製品またはカテゴリを編集します
2. **カスタムフィールド** タブをクリックします
3. 必要に応じてフィールドを入力します
4. **保存** をクリックします — 値はレコードとともに保存されます

### 注文

注文のカスタムフィールド値は、注文詳細ページで **読み取り専用セクション** として表示されます。注文のカスタムフィールドは通常、APIまたはチェックアウト時に設定されます。

### 顧客プロファイル

1. 顧客プロファイルを開きます
2. **カスタムフィールド** タブをクリックします
3. フィールドを入力して保存します

## API アクセス

### フィールド定義の取得

モデルのカスタムフィールド定義をすべて取得します：

```
GET /api/custom-fields/definitions/?model=product&app=catalog
```

**応答：**
```json
[
  {
    "id": 1,
    "name": "External API ID",
    "slug": "external_api_id",
    "field_type": "text",
    "is_required": false,
    "group": { "name": "External Integrations" }
  }
]
```

### カスタムフィールド値の読み取り

カスタムフィールド値は、モデルAPI応答の `custom_fields` JSONオブジェクトに含まれます：

```json
{
  "id": 42,
  "name": "Blue Widget",
  "custom_fields": {
    "external_api_id": "API-12345",
    "is_fragile": true
  }
}
```

### カスタムフィールド値の書き込み

API経由でレコードを作成または更新する際は、`custom_fields` を含めます：

```json
{
  "custom_fields": {
    "external_api_id": "API-67890",
    "warehouse_location": "WH-A3"
  }
}
```

値はフィールド定義に対して検証されます。無効な値は `400` エラーと詳細情報を返します。

### カスタムフィールドによるクエリ

カスタムフィールドは高速なデータベースクエリのためにインデックス化されています。データベースクエリフィルタを使用してレコードをフィルタリングします：

```
GET /api/products/?custom_fields__warehouse_location=WH-A3
```

## ストアフロント表示

### テーマ開発者向け

ストアフロントでカスタムフィールドを表示するには、`render_custom_fields` テンプレートタグを使用します：

```python
{% load custom_fields_tags %}

{# ストアフロントで表示可能なすべてのフィールドをレンダリング #}
{% render_custom_fields product %}

{# 特定のフィールド値を取得 #}
{% get_custom_field product "warehouse_location" as location %}
<p>出荷元: {{ location }}</p>
```

グループとフィールドレベルの両方で **ストアフロントに表示** が有効になっているフィールドのみがレンダリングされます。

## 最佳実践

- **説明的な名前を使用** — フィールド名はフォームとストアフロントに表示されます
- **ヘルプテキストを設定** — 商家が各フィールドに入力する内容をガイドします
- **関連フィールドをグループ化** — フォームを整理し、直感的になります
- **デフォルト値を使用** — データ入力の手間を減らすために合理的なデフォルト値を設定します
- **ストアフロントの可視性を慎重に選択** — 顧客にとって意味のあるフィールドのみを表示します
- **統合でスラッグを使用** — スラッグは安定した識別子です；フィールド名は変更される可能性があります

## トラブルシューティング

**カスタムフィールドタブが表示されない：**
- そのモデルに少なくとも1つのアクティブなフィールドグループが存在することを確認してください
- 管理クラスが `CustomFieldsAdminMixin` を含んでいるかを確認してください
- キャッシュをクリアし、ページを再読み込みしてください

**フィールド値が保存されない：**
- 必須フィールドが入力されていることを確認してください
- 検証ルール（最小/最大、正規表現パターン、許容される選択肢）を確認してください
- フィールドがアクティブで、ソフト削除されていないことを確認してください

**APIが空の custom_fields を返す：**
- モデルが `CustomFieldsMixin` を持っていることを確認してください
- 正しいコンテンツタイプのフィールド定義が存在することを確認してください
- シリアライザーが `CustomFieldsSerializerMixin` を含んでいることを確認してください
