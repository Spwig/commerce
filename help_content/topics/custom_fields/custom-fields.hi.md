---
title: कस्टम फील्ड्स
---

कस्टम फील्ड्स आपको कोई कोड बदले बिना उत्पादों, श्रेणियों, आदेशों और ग्राहक प्रोफाइल में अतिरिक्त डेटा जोड़ने की अनुमति देते हैं। उनका उपयोग बिजनेस-विशिष्ट जैसे बाहरी API आईडी, वर्कशॉप स्थान, अनुगमन डेटा या आपके स्टोर के लिए कोई भी गुण जैसी जानकारी के भंडारण के लिए किया जा सकता है।

## कस्टम फील्ड्स तक पहुँच

प्रशासन साइडबार में **सेटिंग्स > कस्टम फील्ड्स** जाएं।

![कस्टम फील्ड्स पृष्ठ](/static/core/admin/img/help/custom-fields/custom-fields-page.webp)

## महत्वपूर्ण अवधारणाएं

### फील्ड समूह

फील्ड्स को **समूहों** में व्यवस्थित किया जाता है - जो एक खंड के रूप में एक साथ दिखाई देते हैं। उदाहरण के लिए, एक "शिपिंग जानकारी" समूह में वर्कशॉप स्थान, पैकेज आयाम और हज़मैट वर्गीकरण के लिए फील्ड्स हो सकते हैं।

### फील्ड परिभाषाएं

# प्रत्येक क्षेत्र परिभाषा के नियंत्रण

प्रत्येक क्षेत्र परिभाषा निम्नलिखित के नियंत्रण करती है:
- **नाम**: फॉर्म में दिखाए गए लेबल
- **स्लग**: JSON संग्रहण और API प्रतिक्रिया में उपयोग किया गया मशीन-पठनीय कुंजी
- **क्षेत्र प्रकार**: जो प्रविष्टि दिखाई जाती है (टेक्स्ट, संख्या, ड्रॉप-डाउन, आदि)
- **सत्यापन**: नियम जैसे कि न्यूनतम/अधिकतम, अधिकतम लंबाई, रेगेक्स या अनुमत विकल्प
- **दृश्यता**: क्षेत्र का ऑनलाइन स्टोर पर दिखाया जाता है या नहीं

### समर्थित क्षेत्र प्रकार


| प्रकार | विवरण | उदाहरण प्रयोग |
|------|-------------|-------------|
| **पाठ** | एक-पंक्ति वाला पाठ इनपुट | बाहरी API ID, ब्रांड कोड |
| **पाठ बॉक्स** | बहु-पंक्ति वाला पाठ | विशेष नियंत्रण नोट्स |
| **संख्या** | पूर्णांक मान | न्यूनतम आदेश मात्रा |
| **दशमलव** | दशमलव मान | भार ओवरराइड, विशिष्ट आयाम |
| **हां/नहीं** | चेकबॉक्स स्विच | टूटा हुआ है, हस्ताक्षर की आवश्यकता है |
| **तिथि** | तिथि चयनकर्ता | जारी करने की तिथि, अवधि समाप्ति की तिथि |
| **तिथि और समय** | तिथि और समय चयनकर्ता | निर्धारित उपलब्धता |
| **URL** | वेब पता | आपूर्तिकर्ता लिंक, विशिष्ट शीट URL |
| **ईमेल** | ईमेल पता | निर्माता संपर्क |
| **ड्रॉपडाउन** | एक-चयन सूची | सामग्री प्रकार, मूल देश |
| **बहु-चयन** | बहु-चयन सूची | प्रमाणपत्र, टैग |
| **रंग** | रंग चयनकर्ता | ब्रांड रंग, लेबल रंग |

## कस्टम फील्ड्स का प्रबंधन

### एक फील्ड समूह बनाएं

1.

**सेटिंग्स > कस्टम फील्ड्स** खोलें
2.

मॉडल टैब का चयन करें (उत्पाद, श्रेणियाँ, आदेश या ग्राहक प्रोफाइल)
3.

समूह जोड़ें
4.

एक **समूह नाम** दर्ज करें (उदाहरण के लिए, "बाहरी इंटीग्रेशन्स")
5.

वैकल्पिक रूप से **स्टोरफ्रंट पर दिखाएं** को सक्षम करें यदि ग्राहकों को इन क्षेत्रों को देखना चाहिए
6.

**सेव ग्रुप** पर क्लिक करें

### एक समूह में एक क्षेत्र जोड़ें

1. समूह कार्ड पर **एड फील्ड** पर क्लिक करें
2. एक **फील्ड नाम** दर्ज करें - स्लग स्वचालित रूप से उत्पन्न किया जाता है
3. **फील्ड प्रकार** चुनें
4. वैकल्पिक रूप से **हेल्प टेक्स्ट** और **डिफ़ॉल्ट मान** निर्धारित करें
5. वैधता विकल्पों को समायोजित करें (फील्ड प्रकार के आधार पर भिन्न होता है):
   - टेक्स्ट: अधिकतम लंबाई, रेगेक्स पैटर्न
   - संख्या/दशमलव: न्यूनतम और अधिकतम मूल्य
   - ड्रॉपडाउन: विकल्पों की सूची परिभाषित करें
6. क्षेत्र विकल्पों को सेट करें:
   - **आवश्यक**: व्यापारी को बचाने पर इस क्षेत्र को भरना आवश्यक है
   - **स्टोरफ्रंट पर दिखाएं**: ग्राहक-मुखी पृष्ठ पर मान दिखाएं
   - **अनुवाद योग्य**: मान को अनुवाद करने की अनुमति दें (केवल टेक्स्ट/टेक्स्ट क्षेत्र)
7. **फील्ड सेव** पर क्लिक करें

### संपादित करना और पुनर्क्रमित करना


- किसी भी समूह या क्षेत्र पर **पेंसिल आइकॉन** पर क्लिक करके इसे संपादित करें
- किसी समूह या क्षेत्र के भीतर समूहों या क्षेत्रों के क्रम को बदलने के लिए **ग्रिप हैंडल** को खींचें
- परिवर्तन संबंधित फॉर्मों पर तत्काल प्रभाव देखे जा सकते हैं

### समूहों और क्षेत्रों को हटाना

- किसी समूह या क्षेत्र पर **कचरा आइकॉन** पर क्लिक करके इसे हटाएं
- हटाए गए आइटम **मृदु हटाए गए** होते हैं — डेटा डेटाबेस में संरक्षित रहता है लेकिन फॉर्म से छिपा रहता है
- यह अप्रचलित डेटा के अप्रत्याशित नुकसान से बचाता है

## फॉर्म में कस्टम क्षेत्रों का उपयोग करें

एक मॉडल के लिए कस्टम क्षेत्रों को परिभाषित करने के बाद, संबंधित संपादित फॉर्म पर एक **कस्टम क्षेत्र** टैब स्वचालित रूप से दिखाई देता है।

### उत्पाद और श्रेणियाँ

1. किसी भी उत्पाद या श्रेणी को संपादित करने के लिए खोलें
2. **कस्टम क्षेत्र** टैब पर क्लिक करें
3. आवश्यकता के अनुसार क्षेत्रों को भरें
4. **सेव करें** पर क्लिक करें — मान रिकॉर्ड के साथ संग्रहीत किए जाते हैं

### आदेश

आदेश के कस्टम क्षेत्र मान आदेश विवरण पृष्ठ पर एक **केवल पढ़ें** अनुभाग के रूप में प्रदर्शित किए जाते हैं। आदेश के कस्टम क्षेत्र मान आमतौर पर API के माध्यम से या चेकआउट पर सेट किए जाते हैं।

### ग्राहक प्रोफाइल

1.

एक ग्राहक प्रोफाइल खोलें
2.

**कस्टम फील्ड** टैब पर क्लिक करें
3.

फील्ड भरें और सेव करें

## API पहुँच

### क्षेत्र परिभाषाओं की सूची

एक मॉडल के लिए सभी कस्टम क्षेत्र परिभाषाओं को प्राप्त करें:

```
GET /api/custom-fields/definitions/?model=product&app=catalog
```

**प्रतिक्रिया:**
```json
[
  {
    "id": 1,
    "name": "बाहरी API आईडी",
    "slug": "external_api_id",
    "field_type": "text",
    "is_required": false,
    "group": { "name": "बाहरी इंटीग्रेशन्स" }
  }
]
```

### कस्टम क्षेत्र मान पढ़ें

कस्टम क्षेत्र मान मॉडल API प्रतिक्रियाओं पर `custom_fields` JSON ऑब्जेक्ट में शामिल होते हैं:

```json
{
  "id": 42,
  "name": "ब्लू विजेट",
  "custom_fields": {
    "external_api_id": "API-12345",
    "is_fragile": true
  }
}
```

### कस्टम क्षेत्र मान लिखें

API के माध्यम से रिकॉर्ड बनाते या अपडेट करते समय `custom_fields` शामिल करें:

```json
{
  "custom_fields": {
    "external_api_id": "API-67890",
    "warehouse_location": "WH-A3"
  }
}
```

क्षेत्र परिभाषाओं के खिलाफ मान सत्यापित किए जाते हैं।

अमान्य मान एक `400` त्रुटि के साथ वापस आते हैं जिसमें विवरण शामिल होते हैं।

### कस्टम फील्ड्स द्वारा प्रश्न करें

कस्टम फील्ड्स त्वरित डेटाबेस प्रश्नों के लिए सूचीबद्ध होते हैं। डेटाबेस प्रश्न फ़िल्टर्स का उपयोग करके रिकॉर्ड फ़िल्टर करें:

```
GET /api/products/?custom_fields__warehouse_location=WH-A3
```

## स्टोरफ्रंट डिस्प्ले

### थीम विकासकर्ताओं के लिए

स्टोरफ्रंट पर कस्टम फील्ड्स दिखाने के लिए `render_custom_fields` टेम्पलेट टैग का उपयोग करें:

```python
{% load custom_fields_tags %}

{# सभी स्टोरफ्रंट-दृश्य फील्ड्स रेंडर करें #}
{% render_custom_fields product %}

{# एक विशिष्ट फील्ड मान प्राप्त करें #}
{% get_custom_field product "warehouse_location" as location %}
<p>शिपमेंट: {{ location }}</p>
```

केवल उन फील्ड्स जिनके समूह और फील्ड स्तर पर **स्टोरफ्रंट पर दिखाएं** सक्षम हैं, रेंडर किए जाएंगे।

## सर्वोत्तम प्रथाएं

- **विवरण नामों का उपयोग करें** — फ़ील्ड नाम फॉर्म और स्टोरफ्रंट पर दिखाई देते हैं
- **मदद पाने के लिए टेक्स्ट सेट करें** — व्यापक को यह बताएं कि प्रत्येक फ़ील्ड में क्या दर्ज करना है
- **संबंधित फ़ील्ड्स के समूह बनाएं** — फॉर्म को संगठित और अंतर्निहित रखें
- **डिफ़ॉल्ट के साथ चुनौती दें** — डेटा दर्ज करने को कम करने के लिए तार्किक डिफ़ॉल्ट सेट करें
- **स्टोरफ्रंट दृश्यता के साथ चुनौती दें** — केवल ग्राहकों के लिए महत्वपूर्ण फ़ील्ड्स दिखाएं
- **एकीकरण में स्लग का उपयोग करें** — स्लग स्थिर पहचानकर्ता होते हैं; फ़ील्ड नाम बदल सकते हैं

## ट्राउबलशूटिंग

**कस्टम फ़ील्ड्स टैब दिखाई नहीं दे रहा है:**
- जांचें कि उस मॉडल के लिए कम से कम एक सक्रिय फ़ील्ड समूह मौजूद है
- जांचें कि ऐडमिन कक्ष में `CustomFieldsAdminMixin` शामिल है
- कैश को साफ करें और पृष्ठ को रिफ्रेश करें

**फ़ील्ड मान बचाए नहीं जा रहे हैं:**
- यह सुनिश्चित करें कि आवश्यक फ़ील्ड भरे गए हैं
- वैधनिक नियमों की जांच करें (मिन/मैक्स, रेगेक्स पैटर्न, अनुमत विकल्प)
- जांचें कि फ़ील्ड सक्रिय है और निष्क्रिय नहीं है

**API द्वारा खाली custom_fields लौट रहा है:**
- यह निश्चित करें कि मॉडल में `CustomFieldsMixin` है
- सही कंटेंट टाइप के लिए फील्ड परिभाषाएं मौजूद हैं
- यह निश्चित करें कि सीरियलाइजर में `CustomFieldsSerializerMixin` शामिल है
