---
title: Пользовательские поля
---

Пользовательские поля позволяют добавлять дополнительные данные к Товарам, Категориям, Заказам и Профилей Клиентов без изменения любого кода. Используйте их для хранения специфичной для вашего бизнеса информации, такой как внешние ID API, расположение склада, данные соответствия или любые атрибуты, которые ваш магазин может потребовать.

## Доступ к пользовательским полям

Перейдите в **Настройки > Пользовательские поля** в боковой панели администратора.

![Страница пользовательских полей](/static/core/admin/img/help/custom-fields/custom-fields-page.webp)

## Основные понятия

### Группы полей

Поля организованы в **группы** — логические коллекции, которые отображаются вместе как раздел. Например, группа "Информация о доставке" может содержать поля для расположения склада, размеров упаковки и классификации опасных грузов.

### Определения полей

Каждое определение поля управляет:
- **Название**: метка, отображаемая в формах
- **Slug**: ключ, используемый в хранилище JSON и ответах API
- **Тип поля**: какой тип ввода отображается (текст, число, выпадающий список и т.д.)
- **Проверка**: правила, такие как минимум/максимум, максимальная длина, регулярное выражение или разрешенные варианты
- **Видимость**: отображается ли поле на фронтенде

### Поддерживаемые типы полей

| Тип | Описание | Пример использования |
|------|-------------|-------------|
| **Текст** | Однострочный ввод текста | Внешний ID API, код бренда |
| **Текстовая область** | Многострочный текст | Примечания о специальной обработке |
| **Число** | Целые числа | Минимальное количество заказа |
| **Десятичное число** | Десятичные значения | Перекрытие веса, пользовательское измерение |
| **Да/Нет** | Переключатель флажка | Является ли хрупким, требуется подпись |
| **Дата** | Выбор даты | Дата выпуска, срок годности |
| **Дата и время** | Выбор даты и времени | Запланированная доступность |
| **URL** | Веб-адрес | Ссылка на поставщика, URL спецификации |
| **Email** | Адрес электронной почты | Контакт производителя |
| **Выпадающий список** | Список с одиночным выбором | Тип материала, страна происхождения |
| **Многострочный список** | Список с множественным выбором | Сертификаты, теги |
| **Цвет** | Выбор цвета | Цвет бренда, цвет метки |

## Управление пользовательскими полями

### Создание группы полей

1. Откройте **Настройки > Пользовательские поля**
2. Выберите вкладку модели (Товары, Категории, Заказы или Профили клиентов)
3. Нажмите **Добавить группу**
4. Введите **Название группы** (например, "Внешние интеграции")
5. Дополнительно включите **Показывать на фронтенде**, если клиенты должны видеть эти поля
6. Нажмите **Сохранить группу**

### Добавление поля в группу

1. На карточке группы нажмите **Добавить поле**
2. Введите **Название поля** — slug генерируется автоматически
3. Выберите **Тип поля**
4. Дополнительно установите **Помощь текст** и **Значение по умолчанию**
5. Настройте правила проверки (варьируются в зависимости от типа поля):
   - Текст: максимальная длина, регулярное выражение
   - Число/Десятичное число: минимальное и максимальное значения
   - Выпадающий список: определите список вариантов
6. Настройте параметры поля:
   - **Обязательное**: продавцы должны заполнять это поле при сохранении
   - **Показывать на фронтенде**: отображать значение на клиентской странице
   - **Переводимое**: разрешить перевод значения (только для текста/текстовой области)
7. Нажмите **Сохранить поле**

### Редактирование и сортировка

- Нажмите значок **символа карандаша** для редактирования любой группы или поля
- Перетащите **ручку захвата** для сортировки групп или полей внутри группы
- Изменения вступают в силу сразу на всех соответствующих формах

### Удаление групп и полей

- Нажмите значок **корзины** для удаления группы или поля
- Удаления являются **мягкими удалениями** — данные сохраняются в базе данных, но скрываются из форм
- Это защищает существующие данные от случайной потери

## Использование пользовательских полей в формах

После определения пользовательских полей для модели, вкладка **Пользовательские поля** автоматически появляется на соответствующей форме редактирования.

### Товары и Категории

1. Откройте любой товар или категорию для редактирования
2. Нажмите вкладку **Пользовательские поля**
3. Заполните поля по мере необходимости
4. Нажмите **Сохранить** — значения сохраняются вместе с записью

### Заказы

Значения пользовательских полей для заказов отображаются как **только для чтения раздел** на странице деталей заказа. Пользовательские поля заказов обычно устанавливаются через API или на этапе оформления заказа.

### Профили клиентов

1. Откройте профиль клиента
2. Нажмите вкладку **Пользовательские поля**
3. Заполните поля и сохраните

## Доступ к API

### Получение определений полей

Получить все пользовательские определения полей для модели:

```
GET /api/custom-fields/definitions/?model=product&app=catalog
```

**Ответ:**
```json
[
  {
    "id": 1,
    "name": "External API ID",
    "slug": "external_api_id",
    "field_type": "text",
    "is_required": false,
    "group": { "name": "External Integrations" }
  }
]
```

### Чтение значений пользовательских полей

Значения пользовательских полей включаются в объект `custom_fields` в ответах API модели:

```json
{
  "id": 42,
  "name": "Blue Widget",
  "custom_fields": {
    "external_api_id": "API-12345",
    "is_fragile": true
  }
}
```

### Запись значений пользовательских полей

Включите `custom_fields` при создании или обновлении записи через API:

```json
{
  "custom_fields": {
    "external_api_id": "API-67890",
    "warehouse_location": "WH-A3"
  }
}
```

Значения проверяются на соответствие определениям полей. Недопустимые значения возвращают ошибку `400` с деталями.

### Поиск по пользовательским полям

Пользовательские поля индексированы для быстрых запросов к базе данных. Фильтруйте записи с помощью фильтров запросов к базе данных:

```
GET /api/products/?custom_fields__warehouse_location=WH-A3
```

## Отображение на фронтенде

### Для разработчиков тем

Используйте тег шаблона `render_custom_fields`, чтобы отобразить пользовательские поля на фронтенде:

```python
{% load custom_fields_tags %}

{# Отобразить все видимые на фронтенде поля #}
{% render_custom_fields product %}

{# Получить значение конкретного поля #}
{% get_custom_field product "warehouse_location" as location %}
<p>Отправка с: {{ location }}</p>
```

Только поля, у которых включено **Показывать на фронтенде** на уровне группы и поля, будут отображаться.

## Рекомендации

- **Используйте описательные названия** — названия полей отображаются в формах и на фронтенде
- **Установите текст подсказки** — направьте продавцов, что вводить в каждом поле
- **Группируйте связанные поля** — сохраняйте формы организованными и интуитивно понятными
- **Используйте значения по умолчанию** — устанавливайте разумные значения по умолчанию, чтобы сократить ввод данных
- **Будьте селективны с видимостью на фронтенде** — показывайте только те поля, которые имеют значение для клиентов
- **Используйте slugs в интеграциях** — slugs являются стабильными идентификаторами; названия полей могут меняться

## Устранение неполадок

**Вкладка Пользовательские поля не отображается:**
- Проверьте, существует ли хотя бы одна активная группа полей для этой модели
- Убедитесь, что класс администратора включает `CustomFieldsAdminMixin`
- Очистите кэш и обновите страницу

**Значения полей не сохраняются:**
- Убедитесь, что заполнены обязательные поля
- Проверьте правила проверки (минимум/максимум, регулярные выражения, разрешенные варианты)
- Проверьте, что поле активно и не было мягким удалено

**API возвращает пустые custom_fields:**
- Подтвердите, что модель имеет `CustomFieldsMixin`
- Проверьте, существуют ли определения полей для правильного типа содержимого
- Убедитесь, что сериализатор включает `CustomFieldsSerializerMixin`
