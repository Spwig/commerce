title: ฟิลด์แบบกำหนดเอง
---

ฟิลด์แบบกำหนดเองช่วยให้คุณเพิ่มข้อมูลเพิ่มเติมให้กับสินค้า หมวดหมู่ คำสั่งซื้อ และโปรไฟล์ลูกค้าโดยไม่ต้องแก้ไขโค้ดใดๆ ใช้ฟิลด์เหล่านี้เพื่อจัดเก็บข้อมูลเฉพาะทางธุรกิจ เช่น ID ของ API ภายนอก ตำแหน่งคลังสินค้า ข้อมูลการปฏิบัติตามกฎหมาย หรือคุณสมบัติใดๆ ที่ร้านค้าของคุณต้องการ

## การเข้าถึงฟิลด์แบบกำหนดเอง

ไปที่ **การตั้งค่า > ฟิลด์แบบกำหนดเอง** บนแถบด้านข้างของผู้ดูแลระบบ

![หน้าฟิลด์แบบกำหนดเอง](/static/core/admin/img/help/custom-fields/custom-fields-page.webp)

## แนวคิดหลัก

### กลุ่มฟิลด์

ฟิลด์ถูกจัดระเบียบเป็น **กลุ่ม** — ชุดข้อมูลที่มีความหมายซึ่งแสดงร่วมกันเป็นส่วนหนึ่ง ตัวอย่างเช่น กลุ่ม "ข้อมูลการจัดส่ง" อาจมีฟิลด์สำหรับตำแหน่งคลังสินค้า ขนาดกล่อง และการจัดประเภทสินค้าอันตราย

### นิยามฟิลด์

แต่ละนิยามฟิลด์ควบคุม:
- **ชื่อ**: ป้ายกำกับที่แสดงในฟอร์ม
- **Slug**: คีย์ที่สามารถอ่านได้โดยเครื่องที่ใช้ในการจัดเก็บ JSON และการตอบสนอง API
- **ประเภทฟิลด์**: ประเภทของอินพุตที่แสดง (ข้อความ ตัวเลข รายการเลือกเดี่ยว ฯลฯ)
- **การตรวจสอบความถูกต้อง**: กฎเช่น ค่าต่ำสุด/สูงสุด ความยาวสูงสุด รูปแบบ regex หรือตัวเลือกที่อนุญาต
- **การมองเห็น**: ฟิลด์นี้จะปรากฏบนหน้าร้านค้าหรือไม่

### ประเภทฟิลด์ที่รองรับ

| ประเภท | คำอธิบาย | ตัวอย่างการใช้งาน |
|--------|-----------|------------------|
| **ข้อความ** | ฟิลด์ข้อความบรรทัดเดียว | ID ของ API ภายนอก รหัสแบรนด์ |
| **กล่องข้อความ** | ข้อความหลายบรรทัด | หมายเหตุการจัดการพิเศษ |
| **ตัวเลข** | ค่าจำนวนเต็ม | ปริมาณสั่งซื้อขั้นต่ำ |
| **ทศนิยม** | ค่าทศนิยม | น้ำหนักที่กำหนดเอง ขนาดที่กำหนดเอง |
| **ใช่/ไม่ใช่** | ปุ่มสลับ | เป็นสินค้าที่เปราะบาง ต้องลงนาม |
| **วันที่** | ตัวเลือกวันที่ | วันที่เปิดตัว วันหมดอายุ |
| **วันที่และเวลา** | ตัวเลือกวันที่และเวลา | ความพร้อมใช้งานที่กำหนด |
| **URL** | ที่อยู่เว็บ | ลิงก์ผู้ขาย ลิงก์เอกสารข้อมูล |
| **อีเมล** | ที่อยู่อีเมล | ที่อยู่ติดต่อผู้ผลิต |
| **รายการเลือกเดี่ยว** | รายการเลือกเดี่ยว | ประเภทวัสดุ ประเทศแหล่งกำเนิด |
| **รายการเลือกหลายตัว** | รายการเลือกหลายตัว | การรับรอง แท็ก |
| **สี** | ตัวเลือกสี | สีแบรนด์ สีป้าย |

## การจัดการฟิลด์แบบกำหนดเอง

### การสร้างกลุ่มฟิลด์

1. เปิด **การตั้งค่า > ฟิลด์แบบกำหนดเอง**
2. เลือกแท็บโมเดล (สินค้า หมวดหมู่ คำสั่งซื้อ หรือโปรไฟล์ลูกค้า)
3. คลิก **เพิ่มกลุ่ม**
4. ใส่ **ชื่อกลุ่ม** (เช่น "การผสานรวมภายนอก")
5. เปิดใช้งาน **แสดงบนหน้าร้านค้า** ได้หากต้องการให้ลูกค้าเห็นฟิลด์เหล่านี้
6. คลิก **บันทึกกลุ่ม**

### การเพิ่มฟิลด์ในกลุ่ม

1. บนบัตรกลุ่ม คลิก **เพิ่มฟิลด์**
2. ใส่ **ชื่อฟิลด์** — slug จะถูกสร้างอัตโนมัติ
3. เลือก **ประเภทฟิลด์**
4. ตั้งค่า **ข้อความช่วยเหลือ** และ **ค่าเริ่มต้น** ได้ตามต้องการ
5. ตั้งค่าตัวเลือกการตรวจสอบความถูกต้อง (แตกต่างกันไปตามประเภทฟิลด์):
   - ข้อความ: ความยาวสูงสุด รูปแบบ regex
   - ตัวเลข/ทศนิยม: ค่าต่ำสุดและสูงสุด
   - รายการเลือกเดี่ยว: กำหนดรายการตัวเลือก
6. ตั้งค่าตัวเลือกฟิลด์:
   - **จำเป็น**: ผู้ขายต้องกรอกฟิลด์นี้เมื่อบันทึก
   - **แสดงบนหน้าร้านค้า**: แสดงค่าบนหน้าที่ติดต่อลูกค้า
   - **แปลได้**: อนุญาตให้ค่าสามารถแปลได้ (เฉพาะข้อความ/กล่องข้อความ)
7. คลิก **บันทึกฟิลด์**

### การแก้ไขและการจัดลำดับใหม่

- คลิกไอคอน **ปากกา** บนกลุ่มหรือฟิลด์ใดๆ เพื่อแก้ไข
- ลาก **ที่จับ** เพื่อจัดลำดับใหม่ของกลุ่มหรือฟิลด์ภายในกลุ่ม
- การเปลี่ยนแปลงจะมีผลทันทีบนฟอร์มที่เกี่ยวข้องทั้งหมด

### การลบกลุ่มและฟิลด์

- คลิกไอคอน **ถังขยะ** บนกลุ่มหรือฟิลด์เพื่อลบ
- การลบเป็น **การลบแบบอ่อน** — ข้อมูลจะถูกเก็บไว้ในฐานข้อมูลแต่ซ่อนจากฟอร์ม
- นี่ช่วยปกป้องข้อมูลที่มีอยู่จากความสูญเสียโดยไม่ตั้งใจ

## การใช้ฟิลด์แบบกำหนดเองในฟอร์ม

เมื่อคุณกำหนดฟิลด์แบบกำหนดเองสำหรับโมเดล แท็บ **ฟิลด์แบบกำหนดเอง** จะปรากฏอัตโนมัติบนฟอร์มแก้ไขที่เกี่ยวข้อง

### สินค้าและหมวดหมู่

1. เปิดสินค้าหรือหมวดหมู่ใดๆ เพื่อแก้ไข
2. คลิกแท็บ **ฟิลด์แบบกำหนดเอง**
3. กรอกฟิลด์ตามที่จำเป็น
4. คลิก **บันทึก** — ค่าจะถูกจัดเก็บร่วมกับบันทึก

### คำสั่งซื้อ

ค่าของฟิลด์แบบกำหนดเองสำหรับคำสั่งซื้อจะแสดงเป็น **ส่วนที่อ่านได้** บนหน้ารายละเอียดคำสั่งซื้อ ฟิลด์แบบกำหนดเองสำหรับคำสั่งซื้อโดยทั่วไปจะตั้งค่าผ่าน API หรือที่เคาน์เตอร์

### โปรไฟล์ลูกค้า

1. เปิดโปรไฟล์ลูกค้า
2. คลิกแท็บ **ฟิลด์แบบกำหนดเอง**
3. กรอกฟิลด์และบันทึก

## การเข้าถึง API

### การแสดงรายการนิยามฟิลด์

ดึงข้อมูลนิยามฟิลด์แบบกำหนดเองทั้งหมดสำหรับโมเดล:

```
GET /api/custom-fields/definitions/?model=product&app=catalog
```

**การตอบกลับ: **
```json
[
  {
    "id": 1,
    "name": "External API ID",
    "slug": "external_api_id",
    "field_type": "text",
    "is_required": false,
    "group": { "name": "External Integrations" }
  }
]
```

### การอ่านค่าฟิลด์แบบกำหนดเอง

ค่าฟิลด์แบบกำหนดเองจะรวมอยู่ในวัตถุ JSON `custom_fields` บนการตอบสนอง API ของโมเดล:

```json
{
  "id": 42,
  "name": "Blue Widget",
  "custom_fields": {
    "external_api_id": "API-12345",
    "is_fragile": true
  }
}
```

### การเขียนค่าฟิลด์แบบกำหนดเอง

รวม `custom_fields` เมื่อสร้างหรืออัปเดตบันทึกผ่าน API:

```json
{
  "custom_fields": {
    "external_api_id": "API-67890",
    "warehouse_location": "WH-A3"
  }
}
```

ค่าจะถูกตรวจสอบตามนิยามฟิลด์ หากค่าไม่ถูกต้องจะคืนค่า `400` พร้อมรายละเอียด

### การค้นหาด้วยฟิลด์แบบกำหนดเอง

ฟิลด์แบบกำหนดเองถูกจัดทำดัชนีเพื่อการค้นหาฐานข้อมูลอย่างรวดเร็ว ตัวกรองบันทึกโดยใช้ตัวกรองการค้นหาฐานข้อมูล:

```
GET /api/products/?custom_fields__warehouse_location=WH-A3
```

## การแสดงผลบนหน้าร้านค้า

### สำหรับนักพัฒนาธีม

ใช้แท็กแม่แบบ `render_custom_fields` เพื่อแสดงฟิลด์แบบกำหนดเองบนหน้าร้านค้า:

```python
{% load custom_fields_tags %}

{# แสดงฟิลด์ที่สามารถมองเห็นได้บนหน้าร้านค้าทั้งหมด #}
{% render_custom_fields product %}

{# ดึงค่าฟิลด์เฉพาะ #}
{% get_custom_field product "warehouse_location" as location %}
<p>ส่งจาก: {{ location }}</p>
```

ฟิลด์ที่มี **แสดงบนหน้าร้านค้า** เปิดใช้งานทั้งในระดับกลุ่มและระดับฟิลด์จะถูกแสดงผล

## แนวทางปฏิบัติที่ดีที่สุด

- **ใช้ชื่อที่อธิบายได้** — ชื่อฟิลด์จะปรากฏในฟอร์มและบนหน้าร้านค้า
- **ตั้งค่าข้อความช่วยเหลือ** — ช่วยให้ผู้ขายทราบว่าควรกรอกข้อมูลในแต่ละฟิลด์อย่างไร
- **จัดกลุ่มฟิลด์ที่เกี่ยวข้องกัน** — รักษาฟอร์มให้เป็นระเบียบและเข้าใจง่าย
- **ใช้ค่าเริ่มต้น** — ตั้งค่าที่เหมาะสมเพื่อลดการป้อนข้อมูล
- **เลือกใช้การมองเห็นบนหน้าร้านค้าอย่างมีเลือดมีเหตุผล** — แสดงฟิลด์เฉพาะที่มีความหมายต่อลูกค้าเท่านั้น
- **ใช้ slug ในการผสานรวม** — slug เป็นตัวระบุที่มั่นคง; ชื่อฟิลด์สามารถเปลี่ยนได้

## การแก้ไขปัญหา

**แท็บฟิลด์แบบกำหนดเองไม่แสดงขึ้นมา:**
- ตรวจสอบว่ามีกลุ่มฟิลด์ที่ใช้งานได้อย่างน้อยหนึ่งกลุ่มสำหรับโมเดลนั้น
- ตรวจสอบว่าคลาสผู้ดูแลระบบรวม `CustomFieldsAdminMixin`
- ล้างแคชและรีเฟรชหน้าเว็บ

**ค่าฟิลด์ไม่ได้ถูกบันทึก:**
- ตรวจสอบให้แน่ใจว่ากรอกฟิลด์ที่จำเป็น
- ตรวจสอบกฎการตรวจสอบความถูกต้อง (ค่าต่ำสุด/สูงสุด รูปแบบ regex ตัวเลือกที่อนุญาต)
- ตรวจสอบว่าฟิลด์นั้นยังใช้งานได้และไม่ได้ถูกลบแบบอ่อน

**API คืนค่า custom_fields ว่างเปล่า:**
- ยืนยันว่าโมเดลมี `CustomFieldsMixin`
- ตรวจสอบว่ามีนิยามฟิลด์สำหรับประเภทเนื้อหาที่ถูกต้อง
- ตรวจสอบให้แน่ใจว่าซีเรียไลเซอร์รวม `CustomFieldsSerializerMixin`
