---
title: 협력 프로그램
---

협력 프로그램은 제품을 홍보하는 파트너를 모집하고, 그들이 생성한 판매에 대해 수수료를 지급할 수 있도록 합니다. 협력자는 고유한 추천 링크를 공유하고, Spwig은 클릭을 추적하고 주문을 연결하여 수수료를 계산합니다.

![협력 프로그램](/static/core/admin/img/help/affiliate-program/program-list.webp)

## 작동 방식

1. 수수료율과 규칙을 사용하여 하나 이상의 **협력 프로그램**을 생성합니다.
2. 협력자는 대중 포털을 통해 가입하거나 수동으로 추가될 수 있습니다.
3. 각 협력자는 추적 코드가 포함된 **고유한 추천 링크**를 받습니다.
4. 고객이 링크를 클릭하고 구매를 하면 **수수료**가 기록됩니다.
5. 수수료를 검토 및 승인한 후 **지급**을 처리합니다.

## 프로그램 생성

**마케팅 > 협력 프로그램**으로 이동하여 **프로그램 추가**를 클릭합니다.

### 프로그램 설정

| 설정 | 설명 |
|---------|-------------|
| **이름** | 협력자에게 보이는 프로그램 이름 (예: "파트너 프로그램") |
| **수수료 유형** | 주문 총액의 **백분율** 또는 판매당 **고정 금액** |
| **수수료율** | 협력자가 얻는 백분율 또는 고정 금액 |
| **쿠키 수명** | 추천 추적 쿠키가 지속되는 일수 (기본값: 30일) |
| **최소 지급 금액** | 협력자가 지급을 요청할 수 있는 최소 수익 |
| **협력자 자동 승인** | 새로운 협력자 신청을 자동으로 승인하거나 수동 승인을 요구할 수 있습니다. |
| **상태** | 활성, 일시 중지, 또는 종료 |

### 수수료 유형

- **백분율** — 협력자는 추천 주문의 소계의 백분율을 얻습니다 (예: $100 주문의 10% = $10 수수료)
- **고정 금액** — 협력자는 주문 금액과 관계없이 판매당 고정 금액을 얻습니다 (예: 판매당 $5)

## 협력자 관리

**마케팅 > 협력자**로 이동하여 협력자 계정을 확인하고 관리합니다.

### 협력자 세부 정보

각 협력자는 다음과 같은 정보를 가지고 있습니다:
- **협력자 코드** — 추천 URL에 사용되는 고유 코드 (자동 생성 또는 사용자 정의)
- **추천 링크** — 협력자가 공유하는 전체 추적 URL (예: `yourstore.com/?ref=CODE`)
- **상태** — 대기 중, 승인됨, 또는 거부됨
- **지급 방법** — 협력자가 지급을 받는 방식 (PayPal 또는 은행 이체)
- **프로그램 가입** — 협력자가 속한 프로그램

### 수동으로 협력자 추가

1. **협력자 추가**를 클릭합니다.
2. 기존 고객 계정을 선택하거나 새 계정을 생성합니다.
3. 협력자를 하나 이상의 프로그램에 할당합니다.
4. 협력자 코드를 설정하거나 빈칸을 두어 자동 생성합니다.

### 협력자 포털

협력자는 공개 포털에 접근하여 다음과 같은 작업을 수행할 수 있습니다:
- 수익 및 클릭 통계를 보여주는 대시보드를 확인합니다.
- 추천 링크를 복사합니다.
- 수수료 기록을 추적합니다.
- 지급을 요청합니다.

포털 URL은 귀하의 스토어에서 자동으로 `/affiliate/`에 제공됩니다.

## 추적 및 수수료

### 추적 작동 방식

1. 고객이 협력자의 추천 링크를 클릭합니다.
2. 고객의 브라우저에 추적 쿠키가 설정됩니다 (설정된 쿠키 수명만큼 지속됨).
3. 고객이 쿠키 수명 내에 주문을 하면 주문이 협력자에게 연결됩니다.
4. 상태가 **대기 중**인 수수료 기록이 생성됩니다.

### 수수료 상태

| 상태 | 설명 |
|--------|-------------|
| **대기 중** | 수수료가 기록되었으며 검토를 기다리고 있습니다. |
| **승인됨** | 확인되어 지급 준비가 되었습니다. |
| **거부됨** | 수수료가 거부되었습니다 (예: 사기 주문 또는 반품 항목) |
| **지급됨** | 수수료가 완료된 지급에 포함되었습니다. |

### 수수료 검토

**마케팅 > 수수료**로 이동하여 대기 중인 수수료를 검토합니다:

1. 주문 세부 정보를 확인하여 판매가 정당한지 확인합니다.
2. **승인**을 클릭하여 확인하거나 **거부**와 함께 이유를 입력합니다.
3. 승인된 수수료는 협력자의 지급 잔액에 누적됩니다.

## 지급

협력자의 승인된 수수료 잔액이 최소 지급 임계값에 도달하면 지급을 처리할 수 있습니다.

### 지급 처리

1. **마케팅 > 지급**으로 이동합니다.
2. 지급 잔액이 있는 협력자를 선택합니다.
3. 지급 방법을 선택합니다:
   - **PayPal** — 협력자의 PayPal 이메일로 직접 자금을 전송합니다.
   - **은행 이체** — 수동 은행 이체를 기록합니다.
4. 확인하고 지급을 처리합니다.
5. 지급 상태가 **완료됨**으로 업데이트되고 수수료는 **지급됨**으로 표시됩니다.

### 지급 제공업체

Spwig은 자동 지급을 위해 지급 제공업체와 통합됩니다:
- **PayPal** — PayPal API를 통해 자동 대량 지급
- **Airwallex** — 경쟁력 있는 환율을 통해 국제 지급
- **수동** — Spwig 외부에서 처리된 지급을 기록합니다.

## 추천 링크

각 협력자의 추천 링크는 다음과 같은 패턴을 따릅니다:

```
https://yourstore.com/?ref=AFFILIATE_CODE
```

협력자는 또한 특정 제품이나 카테고리에 대한 링크를 생성할 수 있습니다:

```
https://yourstore.com/products/shoe-name/?ref=AFFILIATE_CODE
```

`ref` 매개변수는 모든 페이지에서 작동합니다 — 추적 쿠키는 착륙 페이지와 관계없이 설정됩니다.

## 프로그램 분석

협력 프로그램 대시보드는 다음과 같은 정보를 보여줍니다:
- **총 클릭 수** — 추천 링크가 클릭된 횟수
- **총 주문 수** — 협력자에게 연결된 주문
- **총 수수료** — 모든 수수료의 합 (대기 중, 승인됨, 지급됨)
- **활성 협력자 수** — 현재 추천을 생성하고 있는 승인된 협력자의 수

## 팁

- **백분율 기반 수수료** (5–15%)로 시작하세요 — 주문 금액에 따라 자연스럽게 확장되며 협력자에게 이해하기 쉬운 방식입니다.
- **30일 쿠키 수명**을 기본값으로 설정하세요 — 이는 고객이 돌아와 구매를 완료할 시간을 주되, 여전히 판매를 협력자에게 연결할 수 있도록 합니다.
- 공개 프로그램에서는 **자동 승인**을 활성화하여 마찰을 줄이고, 초대 전용 프로그램에서는 각 협력자를 검토하려면 **수동 승인**을 사용하세요.
- **최소 지급 금액** (예: $25–$50)을 설정하여 많은 작은 거래를 처리하는 것을 피하세요.
- **협력자 포털**을 브랜드에 맞게 커스터마이즈하세요 — 협력자가 스토어 홍보에 더 관심을 가질 수 있도록 전문적인 경험을 제공합니다.
- **자기 추천, 비정상적으로 높은 반품률, 의심스러운 클릭량**과 같은 사기 패턴을 정기적으로 모니터링하세요.
