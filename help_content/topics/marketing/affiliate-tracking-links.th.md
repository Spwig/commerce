---
title: การติดตามพันธมิตร & ลิงก์
---

การติดตามพันธมิตรเป็นหัวใจของระบบค่าคอมมิชชั่นโดยการเชื่อมโยงการซื้อของลูกค้ากับพันธมิตรที่แนะนำพวกเขา คู่มือนี้อธิบายว่าลิงก์ติดตามทำงานอย่างไร ข้อมูลใดที่ Spwig บันทึกเมื่อลูกค้าคลิกที่ลิงก์เหล่านั้น และระบบกำหนดค่าคอมมิชชั่นให้กับพันธมิตรใดผ่านระบบคุกกี้อย่างไร

การเข้าใจกลไกการติดตามช่วยให้คุณแก้ไขปัญหาการกำหนดค่าคอมมิชชั่น วิเคราะห์ประสิทธิภาพของลิงก์ และให้ความรู้กับพันธมิตรของคุณว่าจะเพิ่มการแปลงได้อย่างไร

## ลิงก์ติดตามคืออะไร?

ลิงก์ติดตามคือ URL ที่เป็นเอกลักษณ์ซึ่งเปลี่ยนเส้นทางลูกค้าไปยังร้านค้าของคุณในขณะที่บันทึกตัวตนของพันธมิตรในคุกกี้ แต่ละพันธมิตรสามารถสร้างลิงก์ติดตามหลายตัวที่ชี้ไปยังจุดหมายปลายทางต่างกัน — หน้าแรก ผลิตภัณฑ์เฉพาะ หน้ารวม หรือหน้าลงทุน

ตัวอย่างรูปแบบลิงก์ติดตาม:
```
https://yourstore.com/affiliate/track/a2b7f8c4d1e9/
```

ลิงก์นี้จะเปลี่ยนเส้นทางไปยังจุดหมายปลายทางในขณะที่ตั้งคุกกี้การติดตามที่เชื่อมโยงการซื้อในอนาคตกับพันธมิตรที่เป็นเจ้าของรหัสลิงก์ `a2b7f8c4d1e9`

พันธมิตรสร้างลิงก์เหล่านี้จากแดชบอร์ดของพอร์ทัลของพวกเขา พวกเขาคัดลอก URL ทั้งหมดและแบ่งปันในโพสต์บล็อก โซเชียลมีเดีย อีเมล หรือช่องทางใด ๆ ที่พวกเขาเข้าถึงลูกค้าเป้าหมาย

## องค์ประกอบของลิงก์ติดตาม

ทุกลิงก์ติดตามประกอบด้วยองค์ประกอบต่อไปนี้:

| องค์ประกอบ | ตัวอย่าง | คำอธิบาย |
|-----------|---------|-------------|
| **URL ฐาน** | `https://yourstore.com` | โดเมนของร้านค้าของคุณ |
| **เส้นทางติดตาม** | `/affiliate/track/` | จุดปลายทางการติดตามของ Spwig |
| **รหัสลิงก์** | `a2b7f8c4d1e9` | ตัวระบุที่เป็นเอกลักษณ์ที่สร้างโดยอัตโนมัติ 12 ตัวอักษร |
| **จุดหมาย** | ตั้งค่าเมื่อสร้างลิงก์ | ที่ลูกค้าไปถึงหลังจากเปลี่ยนเส้นทาง (หน้าแรก ผลิตภัณฑ์ ฯลฯ) |

เมื่อพันธมิตรสร้างลิงก์ Spwig จะสร้างรหัสที่เป็นเอกลักษณ์ 12 ตัวอักษรโดยอัตโนมัติ พันธมิตรไม่จำเป็นต้องสร้างหรือแก้ไขรหัสนี้ด้วยตนเอง — พวกเขาเพียงแค่เลือกจุดหมายและ Spwig จะจัดการส่วนที่เหลือ

### ป้ายกำกับลิงก์ (ไม่จำเป็น)

พันธมิตรสามารถเพิ่มป้ายกำกับให้กับลิงก์แต่ละตัวเพื่อจัดระเบียบของตนเอง:
- "ลิงก์ในชีวิตส่วนตัวของ Instagram"
- "คำอธิบายใน YouTube"
- "แคมเปญอีเมล Black Friday"

ป้ายกำกับช่วยให้พันธมิตรติดตามว่าช่องทางการส่งเสริมการขายใดทำงานได้ดีที่สุด ป้ายกำกับจะแสดงให้เห็นถึงพันธมิตรและคุณเท่านั้น — ลูกค้าจะไม่เห็นป้ายกำกับ

## กลไกการติดตามทำงานอย่างไร

กระบวนการติดตามและการกำหนดค่าคอมมิชชั่นประกอบด้วย 5 ขั้นตอนตั้งแต่การคลิกไปจนถึงการสร้างค่าคอมมิชชั่น:

### 1. ลูกค้าคลิกที่ลิงก์

ลูกค้าที่เป็นไปได้คลิกที่ลิงก์ติดตามของพันธมิตรจากช่องทางการส่งเสริมการขายใด ๆ (โพสต์โซเชียลมีเดีย บทความบล็อก บล็อกอีเมล)

### 2. บันทึกการคลิก

จุดปลายทางการติดตามของ Spwig บันทึกรายละเอียดการคลิก:
- ที่อยู่ IP
- User agent (เบราว์เซอร์และอุปกรณ์)
- HTTP referrer (ที่มาของการคลิก)
- เวลาที่บันทึก
- ตัวระบุเซสชัน

ข้อมูลนี้จะปรากฏใน **Clicks** ของผู้ดูแลระบบที่ **Affiliate > Clicks** สำหรับการวิเคราะห์และการตรวจจับการฉ้อโกง

### 3. ตั้งคุกกี้

ระบบติดตามตั้งคุกกี้ในเบราว์เซอร์ของลูกค้าก่อนที่จะเปลี่ยนเส้นทางพวกเขา คุกกี้มี:
- ID พันธมิตร (ผู้ที่ควรได้รับค่าคอมมิชชั่น)
- ID โปรแกรม (โครงสร้างค่าคอมมิชชั่นใดที่ใช้)
- รหัสลิงก์ (ลิงก์เฉพาะที่ถูกคลิก)

### 4. ลูกค้าซื้อสินค้า

ลูกค้าท่องเว็บไซต์ของคุณและทำการซื้อสินค้า นี่สามารถเกิดขึ้นได้ทันที หรือหลังจากผ่านไปหลายวัน/หลายสัปดาห์ ตราบใดที่พวกเขาซื้อภายในช่วงเวลาที่คุกกี้ยังมีอายุ

### 5. สร้างค่าคอมมิชชั่น

ที่เคาน์เตอร์  Spwig ตรวจสอบคุกกี้ของพันธมิตร หากคุกกี้ถูกพบและยังมีอายุ (ภายในช่วงเวลาที่กำหนด) ระบบจะสร้างบันทึกค่าคอมมิชชั่นที่มีสถานะ **Pending** ที่เชื่อมโยงกับพันธมิตร โปรแกรม และคำสั่งซื้อ

## การกำหนดค่าคอมมิชชั่นโดยใช้คุกกี้

คุกกี้ติดตามเป็นกลไกหลักที่เชื่อมโยงการซื้อสินค้ากับพันธมิตร การเข้าใจว่าคุกกี้ทำงานอย่างไรช่วยให้คุณตั้งค่าช่วงเวลาการกำหนดค่าคอมมิชชั่นที่เหมาะสมและแก้ไขปัญหาการติดตาม

### โครงสร้างคุกกี้

| คุณสมบัติ | ค่า |
|----------|-------|
| **ชื่อ** | `aff_{program_id}` (เช่น `aff_7` สำหรับ ID โปรแกรม 7) |
| **ค่า** | JSON ที่มี ID พันธมิตร รหัสลิงก์ เวลาที่บันทึก |
| **โดเมน** | โดเมนของร้านค้าของคุณ |
| **เส้นทาง** | `/` (การเข้าถึงทั่วเว็บไซต์) |
| **ระยะเวลา** | ช่วงเวลาการใช้งานของคุกกี้ตามโปรแกรม (1–365 วัน) |
| **HttpOnly** | `true` (ป้องกันการเข้าถึง JavaScript สำหรับความปลอดภัย) |
| **SameSite** | `Lax` (อนุญาตให้ติดตามจากแหล่งอ้างอิงภายนอก) |
| **Secure** | `true` บนเว็บไซต์ HTTPS (แนะนำ) |

### ช่วงเวลาการใช้งานของคุกกี้

ช่วงเวลาการใช้งานของคุกกี้กำหนดว่าลูกค้ามีเวลาเท่าไรในการซื้อสินค้าหลังจากคลิกที่ลิงก์ของพันธมิตร ช่วงเวลาการใช้งานนี้ตั้งค่าตามโปรแกรมที่ **Marketing > Affiliate Programs** เมื่อคุณสร้างหรือแก้ไขโปรแกรม

ช่วงเวลาการใช้งานของคุกกี้ตามมาตรฐานอุตสาหกรรม:
- **7 วัน**: สินค้าที่ตัดสินใจซื้อได้รวดเร็ว (ของชำ ตั๋วเหตุการณ์)
- **30 วัน**: การค้าปลีกมาตรฐาน (การตั้งค่าที่พบบ่อยที่สุด)
- **60–90 วัน**: การซื้อที่ต้องพิจารณา (เฟอร์นิเจอร์ อุปกรณ์อิเล็กทรอนิกส์ สินค้า B2B)
- **365 วัน**: วงจรการขายที่ยาวนาน (สินค้าหรู บริการที่มีราคาสูง)

หากลูกค้าคลิกที่ลิงก์ของพันธมิตรในวันที่ 1 มกราคม และช่วงเวลาการใช้งานของคุกกี้คือ 30 วัน ค่าคอมมิชชั่นจะถูกเครดิตให้กับพันธมิตรนั้นสำหรับการซื้อที่เกิดขึ้นภายในวันที่ 30 มกราคม การซื้อในวันที่ 31 มกราคมหรือหลังจากนั้นจะไม่สร้างค่าคอมมิชชั่น เนื่องจากคุกกี้หมดอายุ

### แบบจำลองการกำหนดค่าคอมมิชชั่นตามการคลิกล่าสุด

Spwig ใช้แบบจำลอง **การกำหนดค่าคอมมิชชั่นตามการคลิกล่าสุด**: การคลิกล่าสุดเป็นผู้ชนะ นี่คือวิธีที่มันทำงาน:

**สถานการณ์**: ลูกค้าคลิกที่ลิงก์ของพันธมิตร A ในวันจันทร์ แล้วคลิกที่ลิงก์ของพันธมิตร B ในวันพุธ แล้วซื้อในวันศุกร์

**ผลลัพธ์**: พันธมิตร B จะได้รับค่าคอมมิชชั่น เนื่องจากลิงก์ของพวกเขาเป็นการคลิกล่าสุด

คุกกี้การคลิกล่าสุดจะทับคุกกี้พันธมิตรก่อนหน้า แบบจำลองนี้ง่ายต่อการเข้าใจและป้องกันการกำหนดค่าคอมมิชชั่นซ้ำ แม้ว่าจะหมายความว่าพันธมิตรเพียงคนเดียวจะได้รับเครดิตต่อคำสั่งซื้อ (คนสุดท้ายก่อนการซื้อ)

## การบันทึกการคลิก

Spwig บันทึกการคลิกทุกครั้งที่ลิงก์ของพันธมิตรเพื่อให้ข้อมูลการวิเคราะห์กับทั้งคุณและพันธมิตร การบันทึกข้อมูลการคลิกช่วยวัดประสิทธิภาพของลิงก์ ตรวจจับการฉ้อโกง และปรับปรุงกลยุทธ์การส่งเสริมการขาย

### ข้อมูลที่บันทึกต่อการคลิก

ไปที่ **Affiliate > Clicks** เพื่อดูการคลิกที่บันทึกไว้ทั้งหมด แต่ละรายการมี:

| ฟิลด์ | คำอธิบาย |
|-------|-------------|
| **ลิงก์** | ลิงก์ติดตามใดที่ถูกคลิก |
| **พันธมิตร** | ผู้ที่เป็นเจ้าของลิงก์ |
| **ที่อยู่ IP** | ที่อยู่ IP ของลูกค้า (สำหรับการตรวจจับการฉ้อโกง) |
| **User Agent** | ข้อมูลเบราว์เซอร์และอุปกรณ์ |
| **แหล่งอ้างอิง** | หน้าที่ลูกค้าคลิกที่ลิงก์ (เช่น "https://instagram.com") |
| **ID เซสชัน** | ตัวระบุที่เป็นเอกลักษณ์สำหรับเซสชันการท่องเว็บไซต์นี้ |
| **เวลาที่บันทึก** | วันและเวลาที่แน่นอนของการคลิก |

### การจำกัดอัตราการคลิก

เพื่อป้องกันการฉ้อโกงและการใช้งานของบอท Spwig จำกัดการคลิกเป็น **100 ครั้งต่อนาทีต่อที่อยู่ IP** หากที่อยู่ IP เดียวกันเกินเกณฑ์นี้ การคลิกเพิ่มเติมจะถูกเพิกเฉยและไม่เพิ่มจำนวนการคลิก

การป้องกันนี้ช่วยป้องกันผู้กระทำผิดจากการเพิ่มจำนวนการคลิกโดยไม่ต้องบล็อกการจราจรที่ถูกต้อง ลูกค้าจริงเกือบจะไม่เคยเกิน 100 ครั้งต่อนาที

### ข้อพิจารณาเรื่องความเป็นส่วนตัว

ข้อมูลการคลิกมีที่อยู่ IP และ user agent สำหรับการตรวจจับการฉ้อโกง ให้แน่ใจว่านโยบายความเป็นส่วนตัวของคุณระบุว่าคุณติดตามการแนะนำพันธมิตรและแบ่งปันข้อมูลประสิทธิภาพที่ไม่ระบุตัวตนกับพันธมิตร

## การดูลิงก์พันธมิตร

ลิงก์ติดตามที่สร้างโดยพันธมิตรทั้งหมดจะปรากฏในแดชบอร์ดผู้ดูแลระบบของคุณเพื่อการตรวจสอบและการจัดการ

### การเข้าถึงรายการลิงก์

ไปที่ **Affiliate > Links** เพื่อดูลิงก์ติดตามทั้งหมดที่เกี่ยวข้องกับพันธมิตรและโปรแกรมทั้งหมด รายการทัศนคติแสดง:

- **รหัสลิงก์**: ตัวระบุที่เป็นเอกลักษณ์ 12 ตัวอักษร
- **พันธมิตร**: ผู้ที่สร้างลิงก์
- **โปรแกรม**: โครงสร้างค่าคอมมิชชั่นใดที่ใช้
- **ป้ายกำกับ**: คำอธิบายที่พันธมิตรให้มา (ไม่จำเป็น)
- **จุดหมาย**: ที่ลิงก์เปลี่ยนเส้นทางลูกค้า
- **จำนวนการคลิกทั้งหมด**: จำนวนการคลิกทั้งหมดตลอดเวลา
- **สถานะการใช้งาน**: ลิงก์ยังคงติดตามหรือไม่

### การกรองลิงก์

ใช้ตัวกรองผู้ดูแลระบบเพื่อจำกัดรายการ:
- **ตามพันธมิตร**: ดูลิงก์สำหรับพันธมิตรเฉพาะ
- **ตามโปรแกรม**: ดูลิงก์ที่ส่งเสริมโครงสร้างค่าคอมมิชชั่นเฉพาะ
- **ตามสถานะการใช้งาน**: หาลิงก์ที่ถูกปิดใช้งาน

การกรองนี้ช่วยให้คุณวิเคราะห์การกระจายลิงก์ในเครือข่ายพันธมิตรของคุณและระบุลิงก์ที่มีประสิทธิภาพสูงสุด

## สถิติลิงก์

สถิติการติดตามของแต่ละลิงก์สะสมข้อมูลประสิทธิภาพที่ช่วยให้พันธมิตรปรับปรุงกลยุทธ์การส่งเสริมการขายและช่วยให้คุณระบุพันธมิตรที่มีประสิทธิภาพสูงสุด

### คลิกที่บันทึกสถิติเพื่อดูข้อมูลที่ละเอียด:

| ตัวชี้วัด | คำอธิบาย | การคำนวณ |
|--------|-------------|-------------|
| **จำนวนการคลิกทั้งหมด** | จำนวนการคลิกทั้งหมดตั้งแต่สร้างลิงก์ | จำนวนบันทึกการคลิก |
| **การคลิก (7 วัน)** | ตัวชี้วัดกิจกรรมล่าสุด | การคลิกใน 7 วันที่ผ่านมา |
| **การแปลง** | คำสั่งซื้อที่ถูกกำหนดให้กับลิงก์นี้ | จำนวนค่าคอมมิชชั่นจากรหัสลิงก์นี้ |
| **อัตราการแปลง** | ร้อยละของการคลิกที่นำไปสู่การซื้อ | (การแปลง ÷ จำนวนการคลิกทั้งหมด) × 100 |
| **รายได้ทั้งหมด** | ผลรวมของค่าของคำสั่งซื้อทั้งหมดจากลิงก์นี้ | ผลรวมของยอดรวมคำสั่งซื้อสำหรับการคลิกที่แปลงแล้ว |

### การใช้สถิติเพื่อปรับปรุง

**สำหรับพันธมิตร**: ตัวเลขเหล่านี้แสดงให้เห็นว่าช่องทางการส่งเสริมการขายใดทำงานได้ดีที่สุด หากลิงก์ชีวิตส่วนตัวของ Instagram มีอัตราการแปลง 5% แต่ลิงก์บทความบล็อกมี 15% พันธมิตรควรเน้นเนื้อหาบล็อกมากขึ้น

**สำหรับผู้ขาย**: สถิติของลิงก์เปิดเผยพันธมิตรที่สร้างการจราจรคุณภาพ จำนวนการคลิกสูงแต่การแปลงต่ำแสดงว่ากลุ่มเป้าหมายของพันธมิตรอาจไม่เหมาะกับสินค้าของคุณ

## การจัดการลิงก์

คุณสามารถจัดการลิงก์ของพันธมิตรจากแดชบอร์ดผู้ดูแลระบบเพื่อการบำรุงรักษาและการแก้ไขปัญหา

### การปิดใช้งานลิงก์

เพื่อป้องกันไม่ให้ลิงก์เฉพาะติดตามการคลิกใหม่ขณะที่ยังคงบันทึกข้อมูลการคลิกในอดีต:

1. ไปที่ **Affiliate > Links**
2. คลิกที่ลิงก์ที่คุณต้องการปิดใช้งาน
3. ถอดการเลือก **Active**
4. คลิก **Save**

ลิงก์ที่ถูกปิดใช้งานยังคงเปลี่ยนเส้นทางลูกค้าไปยังจุดหมาย แต่ไม่ตั้งคุกกี้การติดตามหรือบันทึกการคลิก นี่เป็นประโยชน์เมื่อพันธมิตรกำลังดำเนินแคมเปญชั่วคราว หรือคุณต้องการปิดช่องทางการส่งเสริมการขายเฉพาะ

### การแก้ไขรายละเอียดลิงก์

คุณสามารถแก้ไข:
- **ป้ายกำกับ**: อัปเดตคำอธิบายที่พันธมิตรให้มา
- **จุดหมาย**: เปลี่ยนที่ลิงก์เปลี่ยนเส้นทาง (มีประโยชน์หากคุณย้ายหน้าสินค้า)
- **สถานะการใช้งาน**: เปิดหรือปิดการติดตาม

คุณไม่สามารถแก้ไขรหัสลิงก์ — มันถาวรและเชื่อมโยงกับข้อมูลการคลิกและค่าคอมมิชชั่นในอดีตทั้งหมด

### การลบลิงก์ที่ไม่ได้ใช้งาน

ลบลิงก์ที่ไม่ได้ใช้งานอีกต่อไปและไม่มีการคลิกหรือการแปลงในอดีต นี่ช่วยให้รายการลิงก์ของคุณสะอาดโดยไม่สูญเสียข้อมูลการวิเคราะห์ที่มีค่า

**คำเตือน**: การลบลิงก์จะลบบันทึกการคลิกที่เกี่ยวข้องทั้งหมด ลบลิงก์เฉพาะเมื่อไม่มีการคลิกเลย หรือเมื่อคุณมั่นใจอย่างแน่ใจว่าไม่จำเป็นต้องใช้ข้อมูลในอดีต

## แบบจำลองการกำหนดค่าคอมมิชชั่น

การเข้าใจตรรกะการกำหนดค่าคอมมิชชั่นของ Spwig ช่วยให้คุณตั้งความคาดหวังกับพันธมิตรและแก้ไขข้อพิพาทเกี่ยวกับค่าคอมมิชชั่น

### การกำหนดค่าคอมมิชชั่นตามการคลิกล่าสุด

ตามที่กล่าวไว้ก่อนหน้านี้ Spwig ใช้การกำหนดค่าคอมมิชชั่นตามการคลิกล่าสุด: หากลูกค้าคลิกที่ลิงก์ของพันธมิตรหลายตัวก่อนซื้อเพียงพันธมิตรที่คลิกล่าสุดเท่านั้นที่ได้รับค่าคอมมิชชั่น

**ข้อดี**:
- ง่ายต่อการเข้าใจและอธิบาย
- ป้องกันการกำหนดค่าคอมมิชชั่นซ้ำ
- ให้ค่าคอมมิชชั่นกับพันธมิตรที่ปิดการขาย

**ข้อเสีย**:
- พันธมิตรที่แนะนำลูกค้าไม่ได้รับเครดิต
- ไม่สะท้อนการเดินทางของลูกค้าหลายครั้ง
- อาจส่งเสริมการ "การลักลอบใช้ลิงก์" (พันธมิตรที่มุ่งเน้นลูกค้าที่มีความตั้งใจสูงที่ถูกแนะนำโดยผู้อื่น)

### ช่วงเวลาการใช้งานของคุกกี้กำหนดคุณสมบัติ

เพียงการซื้อที่อยู่ในช่วงเวลาการใช้งานของคุกกี้เท่านั้นที่สร้างค่าคอมมิชชั่น หากคุกกี้หมดอายุก่อนการชำระเงิน ไม่มีค่าคอมมิชชั่นถูกสร้างขึ้นแม้ว่าลูกค้าจะกลับมาผ่านการบันทึกหน้าเว็บ

**ตัวอย่าง**: ช่วงเวลาการใช้งานของคุกกี้ 30 วัน
- ลูกค้าคลิกลิงก์ในวันที่ 1 มกราคม → คุกกี้ถูกตั้งค่า หมดอายุในวันที่ 31 มกราคม
- ลูกค้าซื้อในวันที่ 25 มกราคม → สร้างค่าคอมมิชชั่น
- ลูกค้าซื้อในวันที่ 5 กุมภาพันธ์ → ไม่มีค่าคอมมิชชั่น (คุกกี้หมดอายุ)

### การติดตามเซสชัน

นอกจากคุกกี้แล้ว Spwig ยังติดตาม ID เซสชันสำหรับแต่ละการคลิก นี่เปิดโอกาสให้กำหนดค่าคอมมิชชั่นสำหรับการเข้าชมหลายครั้งในเซสชันเดียวกัน แม้ว่าคุกกี้จะถูกบล็อกหรือล้าง

หากลูกค้าคลิกที่ลิงก์ แล้วท่องเว็บไซต์ของคุณ โหลดหน้าหลายหน้า และแล้วซื้อ — ทั้งหมดในเซสชันเดียวกัน — พันธมิตรจะได้รับเครดิตแม้ไม่มีคุกกี้ที่ยังคงอยู่

## การแก้ไขปัญหา

ปัญหาการติดตามที่พบบ่อยและวิธีแก้ไข:

### ลิงก์ไม่ติดตามการคลิก

**อาการ**: จำนวนการคลิกยังคงอยู่ที่ศูนย์แม้ว่าพันธมิตรจะรายงานว่าได้แบ่งปันลิงก์

**สาเหตุและวิธีแก้ไข**:
1. **ลิงก์ถูกปิดใช้งาน**: ตรวจสอบสถานะ **Active** ในหน้ารายละเอียดลิงก์
2. **โปรแกรมไม่ได้ใช้งาน**: ไปที่ **Affiliate > Programs** และตรวจสอบสถานะโปรแกรมว่าเป็น **Active** หรือไม่
3. **สถานะบัญชีพันธมิตรถูกปิดใช้งาน**: ตรวจสอบสถานะบัญชีพันธมิตรที่ **Affiliate > Affiliates**
4. **การจำกัดอัตราการคลิก**: ตรวจสอบว่าที่อยู่ IP เดียวกันกำลังสร้างการคลิกจำนวนมากเกินไป (การจราจรจากบอท)

### อัตราการแปลงต่ำ

**อาการ**: จำนวนการคลิกสูงแต่มีคำสั่งซื้อจำนวนน้อยที่ถูกกำหนด

**สาเหตุและวิธีแก้ไข**:
1. **ช่วงเวลาการใช้งานของคุกกี้สั้นเกินไป**: เพิ่มช่วงเวลาการใช้งานของโปรแกรมหากสินค้าของคุณต้องการการวิจัยและพิจารณา
2. **คุณภาพของหน้าจุดหมาย**: ตรวจสอบหน้าลงทุน — รองรับมือถือหรือไม่? โหลดได้เร็วหรือไม่? สินค้ามีสต็อกหรือไม่?
3. **กลุ่มเป้าหมายไม่ตรงกัน**: กลุ่มเป้าหมายของพันธมิตรอาจไม่เหมาะกับสินค้าของคุณ
4. **เบราว์เซอร์บล็อกคุกกี้**: เครื่องมือความเป็นส่วนตัวบางอย่างบล็อกคุกกี้ที่สาม แม้ว่า Spwig จะใช้คุกกี้ที่หนึ่งซึ่งมีแนวโน้มที่จะไม่ถูกบล็อก

### บันทึกการคลิกซ้ำ

**อาการ**: ลูกค้าเดียวกันสร้างบันทึกการคลิกหลายครั้งในเวลาอันสั้น

**สาเหตุ**: นี่เป็นพฤติกรรมปกติ แต่ละการโหลดหน้าของลิงก์ติดตามจะสร้างบันทึกการคลิก หากลูกค้าคลิก หน้าโหลดช้า และคลิกอีกครั้ง คุณจะเห็นบันทึกหลายรายการ

**วิธีแก้ไข**: ไม่จำเป็นต้องดำเนินการใด ๆ ตัวจำกัดอัตราการคลิกป้องกันการใช้ประโยชน์ (100 ครั้งต่อนาทีต่อที่อยู่ IP) และการคลิกซ้ำจากเซสชันเดียวกันไม่ส่งผลต่อการกำหนดค่าคอมมิชชั่น — คุกกี้เพียงตัวเดียวถูกตั้งค่า

## คำแนะนำ

- **ทดสอบการติดตามก่อนเปิดตัว** — สร้างบัญชีพันธมิตรทดสอบ สร้างลิงก์ติดตาม คลิกที่มันในเบราว์เซอร์แบบไม่ระบุตัวตน และทำรายการซื้อทดสอบ ตรวจสอบว่าค่าคอมมิชชั่นปรากฏขึ้นพร้อมกับการกำหนดค่าคอมมิชชั่นที่ถูกต้อง
- **ให้ความรู้กับพันธมิตรเกี่ยวกับช่วงเวลาการใช้งานของคุกกี้** — ให้แน่ใจว่าพันธมิตรเข้าใจว่าพวกเขาได้รับค่าคอมมิชชั่นเฉพาะสำหรับการซื้อที่อยู่ในช่วงเวลาของคุกกี้เท่านั้น นี่ช่วยให้พวกเขาตั้งความคาดหวังที่เป็นจริงและเน้นการจราจรที่มีความตั้งใจสูง
- **ตรวจสอบรูปแบบการคลิกเพื่อตรวจจับการฉ้อโกง** — จำนวนการคลิกที่สูงผิดปกติจากที่อยู่ IP เดียวกัน หรือการคลิกที่ไม่มีข้อมูล user agent อาจบ่งบอกถึงการจราจรจากบอท ตรวจสอบพันธมิตรเหล่านี้อย่างระมัดระวังก่อนอนุมัติค่าคอมมิชชั่น
- **ใช้ป้ายกำกับลิงก์อย่างสม่ำเสมอ** — กระตุ้นให้พันธมิตรระบุป้ายกำกับลิงก์ตามช่องทาง (Instagram, Blog, Email) เพื่อให้คุณทั้งสองสามารถวิเคราะห์ได้ว่าช่องทางการส่งเสริมการขายใดที่สร้างการแปลงที่ดีที่สุด
- **พิจารณาช่วงเวลาการใช้งานของคุกกี้ที่ยาวนานสำหรับสินค้าที่มีราคาสูง** — หากค่าเฉลี่ยของคำสั่งซื้อของคุณสูง และลูกค้ามักจะวิจัยก่อนซื้อ ขยายช่วงเวลาการใช้งานของคุกกี้เป็น 60–90 วันเพื่อจับการแปลงที่ล่าช้า
- **ตรวจสอบข้อมูลแหล่งอ้างอิงเพื่อให้ข้อมูลเชิงลึกเกี่ยวกับช่องทาง** — ฟิลด์แหล่งอ้างอิงแสดงให้เห็นว่าการคลิกมาจากที่ใด หากคุณเห็นการคลิกจำนวนมากจาก "instagram.com" หรือ "youtube.com" คุณจะทราบว่าพันธมิตรของคุณใช้แพลตฟอร์มโซเชียลมีเดียใดได้อย่างมีประสิทธิภาพมากที่สุด
