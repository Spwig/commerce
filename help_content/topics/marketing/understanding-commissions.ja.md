---
title: コミッションの理解
---

コミッションは、アフィリエイトが成功裏に売上をあなたのストアに駆動したときに作成される収益記録です。各コミッションは特定の注文、アフィリエイト、プログラムに関連付けられ、保留から支払い済みまでのライフサイクルを経ます。このガイドでは、コミッションの仕組み、計算方法、および効果的な管理方法について説明します。

## コミッションとは？

コミッションは、顧客が購入を完了させたことに対してアフィリエイトに支払われる金額を表します。顧客がアフィリエイトの紹介リンクをクリックし、クッキーの有効期間内に注文を完了した場合、Spwigは自動的にコミッション記録を作成します。

各コミッションには以下が含まれます：
- **アフィリエイト** — 顧客を紹介したパートナー
- **プログラム** — コミッションのルールを定義するアフィリエイトプログラム
- **注文** — コミッションを生成した注文
- **金額** — 計算されたコミッション値
- **ステータス** — コミッションライフサイクルの現在の段階
- **日付** — 作成日、承認/拒否日、支払い日

## コミッションの計算

コミッションは、プログラムのコミッションタイプとレートに基づいて自動的に計算されます。

| コミッションタイプ | 計算方法 | 例 |
|------------------|----------|-----|
| **パーセンテージ** | 注文合計 × コミッション% ÷ 100 | 注文：$200、レート：10% → **$20 コミッション** |
| **固定金額** | 注文ごとに固定金額 | レート：$15 → **$15 コミッション**（注文の金額に関係なく） |

### 計算例

**パーセンテージコミッション（10％）**：
- 顧客が$50の注文を置く → $5コミッション
- 顧客が$150の注文を置く → $15コミッション
- 顧客が$300の注文を置く → $30コミッション

**固定コミッション（$20）**：
- 顧客が$50の注文を置く → $20コミッション
- 顧客が$150の注文を置く → $20コミッション
- 顧客が$300の注文を置く → $20コミッション

コミッションは**注文のサブトータル**（送料や税金の前）に基づいて計算され、注文が置かれたときにすぐに作成されます。

## コミッションライフサイクル

すべてのコミッションは、作成から支払いに至るまで、一連のステータスを経ます：

```
保留 → 承認 → 支払い済み
   ↓
拒否
```

### ステータスの定義

| ステータス | 説明 | 何が起こるか |
|------------|------|-------------|
| **保留** | 注文が置かれ、コミッションが確認待ち | コミッションは作成されていますが、まだ確認されていません。アフィリエイトは見ることができますが、資金を引き出すことはできません。 |
| **承認済み** | 店舗が売上が有効であることを確認 | コミッションは確認され、アフィリエイトの利用可能な残高に追加されます。支払いが可能になります。 |
| **拒否** | 店舗がコミッションを拒否 | コミッションは拒否されます（例：注文が返金された、不正行為、または利用規約に違反した）。支払いが不可能です。 |
| **支払い済み** | コミッションが完了した支払いに含まれる | アフィリエイトに支払いが済んでいます。コミッションは確定し、変更できません。 |

## コミッションが作成されるタイミング

コミッションは、次のシーケンスに従って自動的に作成されます：

1. **顧客がアフィリエイトリンクをクリック** — 紹介URLにはアフィリエイトのユニークなトラッキングコードが含まれています（例：`?ref=JOHNSMITH`）
2. **クッキーが設定** — 顧客のブラウザにアフィリエイトコードを含むトラッキングクッキーが保存されます
3. **クッキー有効期間内での購入** — 顧客がクッキーが期限切れになる前に注文を完了します（デフォルト：30日）
4. **システムが注文を属性付ける** — Spwigはアクティブなトラッキングクッキーを確認し、紹介したアフィリエイトを特定します
5. **コミッションが自動作成** — ステータス**保留**でコミッション記録が生成されます

注文が置かれたときにコミッションが**すぐに**作成され、支払いが確認される前でもです。これにより、注文が処理されている間に店舗がコミッションを確認できます。

## トレーキングと属性付け

Spwigは、**最後のクリック属性付け**を使用して、どのアフィリエイトが販売を獲得するかを決定します。

### 属性付けの仕組み

- **最後のクリックモデル** — 最後にクリックされたアフィリエイトリンクが信用されます（複数のアフィリエイトが顧客を紹介した場合でも）
- **クッキーベースのトレーキング** — 顧客のブラウザにアフィリエイトコードを保存するクッキー
- **クッキー有効期間** — 販売を属性付けることができる期間（プログラムごとに設定、通常は30日）
- **IPおよびセッショントレーキング** — 不正行為のパターンを特定するための追加データ

### 属性付けの例

- 日1：顧客がアフィリエイトAのリンクをクリック → アフィリエイトAのクッキーが設定される
- 日5：顧客がアフィリエイトBのリンクをクリック → クッキーが**更新**されてアフィリエイトBに設定される（最後のクリックが優先）
- 日7：顧客が注文を置く → コミッションは**アフィリエイトB**に支払われる

顧客が日35（30日間のクッキー有効期間が終了した後）に再訪問して注文を置いた場合、**コミッションは作成されません**。これは、トレーキングウィンドウが終了しているためです。

## コミッションの詳細

**マーケティング > コミッション**に移動して、すべてのコミッション記録を表示してください。

### コミッションのフィールド

各コミッションには以下が表示されます：

| フィールド | 説明 |
|----------|------|
| **アフィリエイト** | アフィリエイトの名前とコード |
| **プログラム** | アフィリエイトプログラムの名前 |
| **注文** | 注文番号（クリック可能なリンクで詳細な注文情報を表示） |
| **金額** | 計算されたコミッション値 |
| **ステータス** | 現在の段階（保留、承認済み、拒否、支払い済み） |
| **作成日** | コミッションが生成された日時 |
| **承認/拒否日** | ステータスが更新された日時 |
| **支払い日** | 支払いが処理された日時 |
| **メモ** | コミッションに関する内部メモ |

### 注文詳細の表示

コミッション記録内の**注文番号**をクリックして、元の注文を表示します。これにより、以下を確認できます：
- 注文合計と購入された商品
- 顧客情報
- 支払い状況
- 配送状況
- いかなる返金または返品

このコンテキストは、コミッションを承認または拒否するかどうかを判断するのに役立ちます。

## コミッションの管理

このガイドはコミッションの理解に焦点を当てていますが、コミッションの承認、拒否、支払いの実際の手順は**コミッション管理**のヘルプトピックで詳細に説明されています。

### 簡易概要

- **承認** — 注文が正当であり、コミッションが有効であることを確認する
- **拒否** — 不正な注文、返金、またはポリシー違反のコミッションを拒否する
- **メモの追加** — 承認または拒否の理由を記録し、将来的に参照する
- **支払いの処理** — 承認されたコミッションをバッチ支払いにグループ化する

各管理タスクのステップバイステップの指示については、関連するヘルプトピックを参照してください。

## ヒント

- 最初の1か月は、**毎日**保留中のコミッションを確認して、リズムを確立し、トレーキングの問題を早期に発見してください
- **メール通知**を設定して、新しいコミッションが作成されたときに通知されるようにして、注文の詳細が新鮮なうちに確認できるようにしてください
- 注文の履行後（注文が置かれた直後にではなく）にコミッションを**承認**してください。これにより、キャンセルや返品を考慮できます
- **メモ欄**を活用して、特に拒否されたコミッションの理由を記録し、アフィリエイトが質問した場合に記録があるようにしてください
- **拒否のパターン**に注意してください — 1つのアフィリエイトが多くのコミッションが拒否されている場合、それは不正行為またはプログラムの条件の誤解を示している可能性があります
- **コミッション承認ポリシー**を作成し（例：「14日間の返品期間後」）、アフィリエイトに明確な期待値を設定してください
