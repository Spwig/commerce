---
title: Affiliate Tracking & Links
---

Affiliate tracking은 고객의 구매를 추적하여 해당 추천을 한 파트너에게 수수료를 지급하는 전체 수수료 시스템을 구동합니다. 이 가이드는 추적 링크가 어떻게 작동하는지, 고객이 추적 링크를 클릭할 때 Spwig이 어떤 데이터를 기록하는지, 그리고 쿠키 기반의 속성 분배 시스템이 각 수수료를 어떤 파트너에게 지급하는지 설명합니다.

추적 메커니즘을 이해하면 속성 분배 문제를 해결하고, 링크 성능을 분석하며, 파트너가 전환율을 극대화하는 방법을 교육하는 데 도움이 됩니다.

## 추적 링크란 무엇인가요?

추적 링크는 고객을 가맹점으로 리디렉션하면서 파트너의 신원을 쿠키에 기록하는 고유한 URL입니다. 각 파트너는 다른 목적지로 가는 여러 추적 링크를 생성할 수 있습니다 — 홈 페이지, 특정 제품, 컬렉션 페이지, 또는 랜딩 페이지.

추적 링크 예시 형식:
```
https://yourstore.com/affiliate/track/a2b7f8c4d1e9/
```

이 링크는 목적지로 리디렉션하면서 `a2b7f8c4d1e9` 링크 코드를 소유한 파트너와 연관된 추적 쿠키를 설정합니다.

파트너는 자신의 포털 대시보드에서 이러한 링크를 생성합니다. 전체 URL을 복사하여 블로그 게시물, 소셜 미디어, 이메일, 또는 잠재 고객을 도달할 수 있는 기타 채널에서 공유합니다.

## 추적 링크 구성 요소

모든 추적 링크는 다음 요소를 포함합니다:

| 구성 요소 | 예시 | 설명 |
|-----------|---------|-------------|
| **기본 URL** | `https://yourstore.com` | 가맹점 도메인 |
| **추적 경로** | `/affiliate/track/` | Spwig의 추적 엔드포인트 |
| **링크 코드** | `a2b7f8c4d1e9` | 자동 생성된 12자리 고유 식별자 |
| **목적지** | 링크 생성 시 설정 | 리디렉션 후 고객이 도착하는 위치 (홈 페이지, 제품 등) |

파트너가 링크를 생성할 때 Spwig은 고유한 12자리 코드를 자동으로 생성합니다. 파트너는 이 코드를 수동으로 생성하거나 편집할 필요가 없습니다 — 목적지를 선택하고 Spwig이 나머지를 처리합니다.

### 링크 라벨 (선택 사항)

파트너는 자신의 조직을 위해 각 링크에 라벨을 추가할 수 있습니다:
- "Instagram Bio Link"
- "YouTube Description"
- "Black Friday Email Campaign"

라벨은 파트너가 어떤 프로모션 채널이 가장 효과적인지 추적하는 데 도움이 됩니다. 이 라벨은 파트너와 당신만 보고, 고객은 보지 못합니다.

## 추적 작동 방식

추적 및 속성 분배 과정은 클릭에서 수수료 지급까지 5단계로 이루어집니다:

### 1. 고객이 링크를 클릭합니다

잠재 고객이 소셜 미디어 게시물, 블로그 기사, 이메일 뉴스레터 등 어떤 프로모션 채널에서 파트너의 추적 링크를 클릭합니다.

### 2. 클릭 기록

Spwig의 추적 엔드포인트는 클릭 세부 정보를 기록합니다:
- IP 주소
- 사용자 에이전트 (브라우저 및 장치)
- HTTP 리퍼러 (클릭이 어디서 온 것인지)
- 타임스탬프
- 세션 식별자

이 데이터는 **Affiliate > Clicks**에서 **Clicks** 관리자에 표시되어 분석 및 사기 탐지에 사용됩니다.

### 3. 쿠키 설정

추적 시스템은 고객의 브라우저에 쿠키를 설정한 후 리디렉션합니다. 쿠키는 다음과 같은 정보를 포함합니다:
- 파트너 ID (수수료를 받을 파트너)
- 프로그램 ID (어떤 수수료 구조가 적용되는지)
- 링크 코드 (어떤 특정 링크가 클릭되었는지)

### 4. 고객이 구매합니다

고객은 가맹점에서 상품을 둘러보고 구매를 완료합니다. 이는 즉시 또는 며칠, 몇 주 후에 발생할 수 있으며, 쿠키 유효 기간 내에 구매하면 됩니다.

### 5. 수수료 생성

체크아웃 시 Spwig은 파트너 쿠키를 확인합니다. 쿠키가 존재하고 여전히 유효하다면 (쿠키 유효 기간 내), 시스템은 파트너, 프로그램 및 주문과 연결된 **대기 중** 상태의 수수료 기록을 생성합니다.

## 쿠키 기반 속성 분배

추적 쿠키는 구매를 파트너와 연결하는 핵심 메커니즘입니다. 쿠키 작동 방식을 이해하면 최적의 속성 윈도우를 설정하고 추적 문제를 해결하는 데 도움이 됩니다.

### 쿠키 구조

| 속성 | 값 |
|----------|-------|
| **이름** | `aff_{program_id}` (예: `aff_7` for program ID 7) |
| **값** | 파트너 ID, 링크 코드, 타임스탬프를 포함하는 JSON |
| **도메인** | 가맹점 도메인 |
| **경로** | `/` (사이트 전체 접근 가능) |
| **기간** | 프로그램의 쿠키 유효 기간 (1–365일) |
| **HttpOnly** | `true` (보안을 위해 JavaScript 접근 방지) |
| **SameSite** | `Lax` (외부 리퍼러에서 추적 허용) |
| **Secure** | HTTPS 사이트에서 `true` (권장) |

### 쿠키 유효 기간

쿠키 유효 기간은 고객이 파트너 링크를 클릭한 후 구매할 수 있는 시간을 결정합니다. 이 윈도우는 프로그램별로 **Marketing > Affiliate Programs**에서 프로그램을 생성하거나 편집할 때 설정됩니다.

업계 표준 쿠키 유효 기간:
- **7일**: 신속한 결정 상품 (식료품, 이벤트 티켓)
- **30일**: 표준 전자상거래 (가장 일반적인 설정)
- **60–90일**: 고려 구매 (가구, 전자기기, B2B 상품)
- **365일**: 장기 판매 주기 (로우지, 고가 서비스)

고객이 1월 1일에 파트너 링크를 클릭하고 쿠키 유효 기간이 30일이라면, 1월 30일까지의 구매는 해당 파트너에게 신용됩니다. 1월 31일 이후의 구매는 쿠키가 만료되어 수수료가 생성되지 않습니다.

### 마지막 클릭 속성 모델

Spwig은 **마지막 클릭 속성**을 사용합니다: 가장 최근의 파트너 링크가 승리합니다. 작동 방식은 다음과 같습니다:

**시나리오**: 고객이 월요일에 파트너 A의 링크를 클릭하고, 수요일에 파트너 B의 링크를 클릭한 후, 금요일에 구매합니다.

**결과**: 파트너 B가 수수료를 받습니다. 그들의 링크가 가장 최근의 클릭이었기 때문입니다.

마지막 클릭 쿠키는 이전 파트너 쿠키를 덮어씁니다. 이 모델은 이해하기 쉽고, 중복 수수료를 방지하지만, 주문당 하나의 파트너만 신용을 받을 수 있다는 점을 고려해야 합니다 (구매 전 마지막 파트너).

## 클릭 기록

Spwig은 모든 파트너 링크의 클릭을 기록하여 파트너와 파트너 모두에게 분석 정보를 제공합니다. 클릭 데이터는 링크 성능을 측정하고, 사기 탐지 및 프로모션 전략 최적화에 도움이 됩니다.

### 클릭당 기록된 데이터

**Affiliate > Clicks**로 이동하여 모든 기록된 클릭을 확인합니다. 각 항목에는 다음과 같은 정보가 포함됩니다:

| 필드 | 설명 |
|-------|-------------|
| **링크** | 클릭된 추적 링크 |
| **파트너** | 링크를 생성한 파트너 |
| **IP 주소** | 고객의 IP (사기 탐지용) |
| **사용자 에이전트** | 브라우저 및 장치 정보 |
| **리퍼러** | 고객이 링크를 클릭한 페이지 (예: "https://instagram.com") |
| **세션 ID** | 이 브라우징 세션의 고유 식별자 |
| **타임스탬프** | 클릭의 정확한 날짜 및 시간 |

### 클릭 제한

클릭 사기 및 봇 악용을 방지하기 위해 Spwig은 **1분당 100개의 IP 주소당 클릭**을 제한합니다. 동일한 IP가 이 임계값을 초과하면 추가 클릭은 무시되고 클릭 수에 포함되지 않습니다.

이 보호 기능은 악의적인 행위자가 클릭 통계를 부풀리지 않으면서 정상적인 트래픽을 차단하지 않도록 합니다. 실제 고객은 거의 1분당 100개의 클릭을 초과하지 않습니다.

### 개인정보 보호 고려사항

클릭 데이터는 사기 탐지 목적의 IP 주소 및 사용자 에이전트를 포함합니다. 개인정보 보호 정책에서 파트너 추천을 추적하고 파트너에게 익명화된 성능 데이터를 공유한다는 점을 공개해야 합니다.

## 파트너 링크 보기

모든 파트너가 생성한 추적 링크는 관리자 패널에서 모니터링 및 관리할 수 있습니다.

### 링크 목록에 접근

**Affiliate > Links**로 이동하여 모든 파트너 및 프로그램의 추적 링크를 확인합니다. 목록 뷰는 다음과 같은 정보를 표시합니다:

- **링크 코드**: 고유한 12자리 식별자
- **파트너**: 링크를 생성한 파트너
- **프로그램**: 어떤 수수료 구조가 적용되는지
- **라벨**: 선택 사항인 파트너 제공 설명
- **목적지**: 링크가 고객을 리디렉션하는 위치
- **총 클릭 수**: 링크 생성 이후의 모든 클릭 수
- **활성 상태**: 링크가 현재 추적 중인지 여부

### 링크 필터링

관리자 필터를 사용하여 목록을 좁힐 수 있습니다:
- **파트너별**: 특정 파트너의 모든 링크를 확인
- **프로그램별**: 특정 수수료 구조를 홍보하는 링크를 확인
- **활성 상태별**: 비활성 링크를 확인

이 필터링은 파트너 네트워크에서 링크 분포를 분석하고 최고 성과를 내는 링크를 식별하는 데 도움이 됩니다.

## 링크 통계

각 추적 링크는 파트너가 프로모션 전략을 최적화하고, 당신은 최고 성과를 내는 파트너를 식별하는 데 도움이 되는 성능 지표를 누적합니다.

### 링크 기록을 클릭하여 상세 통계를 확인합니다:

| 지표 | 설명 | 계산 |
|--------|-------------|-------------|
| **총 클릭 수** | 링크 생성 이후 기록된 모든 클릭 | 클릭 기록 수 |
| **클릭 (7일)** | 최근 활동 지표 | 지난 7일 동안의 클릭 |
| **전환율** | 이 링크에 연결된 주문 | 이 링크 코드에서 생성된 수수료 수 |
| **전환율** | 클릭 중 구매로 이어진 비율 | (전환율 ÷ 총 클릭 수) × 100 |
| **총 매출** | 이 링크에서 생성된 모든 주문의 총 금액 | 전환된 클릭의 주문 총액 합계 |

### 통계를 활용한 최적화

**파트너에게**: 이러한 수치는 어떤 프로모션 채널이 가장 효과적인지를 보여줍니다. 인스타그램 바이오 링크가 5%의 전환율을 보이지만 블로그 게시물 링크가 15%의 전환율을 보인다면, 파트너는 블로그 콘텐츠에 더 집중해야 합니다.

**상인에게**: 링크 통계는 어떤 파트너가 품질 높은 트래픽을 유도하는지를 보여줍니다. 높은 클릭 수와 낮은 전환율은 파트너의 대상자가 당신의 상품에 적합하지 않다는 것을 나타냅니다.

## 링크 관리

관리자 패널에서 링크를 관리하여 유지보수 및 문제 해결을 수행할 수 있습니다.

### 링크 비활성화

특정 링크가 새로운 클릭을 추적하지 않도록 하되, 기존 데이터는 보존하려면:

1. **Affiliate > Links**로 이동합니다
2. 비활성화하려는 링크를 클릭합니다
3. **활성**을 선택 해제합니다
4. **저장**을 클릭합니다

비활성화된 링크는 여전히 고객을 목적지로 리디렉션하지만, 추적 쿠키를 설정하거나 클릭을 기록하지 않습니다. 이는 파트너가 일시적인 캠페인을 진행하거나 특정 프로모션 채널을 비활성화해야 할 때 유용합니다.

### 링크 세부 정보 편집

다음 항목을 수정할 수 있습니다:
- **라벨**: 파트너 제공 설명을 업데이트
- **목적지**: 링크가 리디렉션하는 위치를 변경 (제품 페이지가 이동했을 경우 유용)
- **활성 상태**: 추적을 활성화하거나 비활성화

링크 코드는 편집할 수 없습니다 — 이는 영구적이며, 모든 기존 클릭 및 수수료 데이터와 연결되어 있습니다.

### 비활성 링크 삭제

더 이상 사용되지 않으며 기존 클릭 또는 전환도 없는 링크를 삭제합니다. 이는 분석 데이터를 잃지 않고 링크 목록을 깔끔하게 유지하는 데 도움이 됩니다.

**경고**: 링크를 삭제하면 관련된 모든 클릭 기록도 함께 삭제됩니다. 클릭 수가 0인 링크만 삭제하거나, 기존 데이터가 절대 필요하지 않다고 확신할 때만 삭제해야 합니다.

## 속성 모델

Spwig의 속성 논리 이해는 파트너와의 기대치 설정 및 수수료 분쟁 문제 해결에 도움이 됩니다.

### 마지막 클릭 속성

이전에 언급했듯이, Spwig은 마지막 클릭 속성을 사용합니다: 고객이 구매하기 전에 여러 파트너 링크를 클릭한 경우, 마지막 클릭한 파트너만 수수료를 받습니다.

**장점**:
- 이해하고 설명하기 쉬움
- 중복 수수료 방지
- 판매를 완료한 파트너에게 보상

**단점**:
- 고객을 처음 소개한 이전 파트너는 신용을 받지 못함
- 다중 접촉 고객 여정을 반영하지 않음
- 다른 파트너가 이미 추천한 고의 고객을 대상으로 하는 "링크 해킹"을 유도할 수 있음

### 쿠키 유효 기간이 자격을 결정함

쿠키 유효 기간 내에 이루어진 구매만 수수료를 생성합니다. 쿠키가 체크아웃 전에 만료되면, 고객이 북마크를 통해 다시 방문하더라도 수수료가 생성되지 않습니다.

**예시**: 30일 쿠키 유효 기간
- 고객이 1월 1일에 링크를 클릭 → 쿠키 설정, 1월 31일에 만료
- 고객이 1월 25일에 구매 → 수수료 생성
- 고객이 2월 5일에 구매 → 수수료 없음 (쿠키 만료)

### 세션 추적

쿠키 외에도 Spwig은 각 클릭의 세션 ID를 추적합니다. 이는 쿠키가 차단되거나 삭제된 경우에도 동일한 세션 내에서 여러 방문의 속성을 가능하게 합니다.

고객이 링크를 클릭하고, 가맹점에서 둘러보며 여러 페이지를 로드하고, 그 후 구매를 한다면 — 모두 동일한 세션에서 이루어진 경우 — 파트너는 지속적인 쿠키 없이도 신용을 받습니다.

## 문제 해결

일반적인 추적 문제 및 해결 방법:

### 링크가 클릭을 추적하지 않음

**증상**: 클릭 수가 0으로 유지되지만 파트너가 링크를 공유했다고 보고합니다.

**원인 및 해결 방법**:
1. **링크가 비활성화됨**: 링크 세부 정보 페이지에서 **활성** 상태를 확인합니다.
2. **프로그램이 비활성화됨**: **Affiliate > Programs**로 이동하여 프로그램 상태가 **활성**인지 확인합니다.
3. **파트너 계정이 비활성화됨**: **Affiliate > Affiliates**에서 파트너의 계정 상태를 확인합니다.
4. **클릭 제한**: 동일한 IP가 과도한 클릭을 생성하는지 확인합니다 (봇 트래픽)

### 낮은 전환율

**증상**: 높은 클릭 수지만 매우 적은 주문이 연결되어 있습니다.

**원인 및 해결 방법**:
1. **쿠키 유효 기간이 너무 짧음**: 제품이 조사 및 고려가 필요하다면 프로그램의 쿠키 유효 기간을 늘립니다.
2. **목적지 페이지 품질**: 랜딩 페이지를 확인합니다. 모바일 친화적인가요? 빠르게 로드되나요? 제품이 재고에 있나요?
3. **대상자 불일치**: 파트너의 대상자가 당신의 제품에 적합하지 않을 수 있습니다.
4. **브라우저가 쿠키를 차단함**: 일부 프라이버시 도구가 제3자 쿠키를 차단하지만, Spwig은 제1자 쿠키를 사용하며 이는 차단될 가능성이 적습니다.

### 중복 클릭 기록

**증상**: 동일한 고객이 짧은 시간 내 여러 클릭 기록을 생성합니다.

**원인**: 이는 정상적인 행동입니다. 추적 링크의 각 페이지 로드는 클릭 기록을 생성합니다. 고객이 링크를 클릭하고 페이지가 느리게 로드되어 다시 클릭하면 여러 기록을 보게 됩니다.

**해결 방법**: 조치가 필요 없습니다. 클릭 제한기로 인해 악의적인 행위 (1분당 100개의 클릭/IP)가 방지되며, 동일한 세션에서의 중복 클릭은 속성에 영향을 주지 않습니다 — 하나의 쿠키만 설정됩니다.

## 팁

- **출시 전 추적 테스트**: 테스트 파트너 계정을 생성하고, 추적 링크를 생성한 후, 무인 브라우저에서 클릭하고 테스트 구매를 완료합니다. 올바른 파트너 속성으로 수수료가 생성되는지 확인합니다.
- **쿠키 유효 기간에 대한 파트너 교육**: 파트너가 쿠키 유효 기간 내에만 수수료를 받을 수 있음을 확실히 해야 합니다. 이는 파트너가 현실적인 기대치를 설정하고 고의 고객을 집중하는 데 도움이 됩니다.
- **사기 탐지용 클릭 패턴 모니터링**: 단일 IP에서 비정상적으로 높은 클릭 수 또는 사용자 에이전트 문자열이 없는 클릭은 봇 트래픽을 나타낼 수 있습니다. 수수료 승인 전에 이러한 파트너를 신중하게 검토해야 합니다.
- **링크 라벨을 일관되게 사용**: 파트너가 채널 (인스타그램, 블로그, 이메일)별로 링크를 라벨링하도록 장려하여, 당신은 어떤 프로모션 채널이 가장 효과적인지를 분석할 수 있습니다.
- **고가 상품에 더 긴 쿠키 유효 기간 고려**: 평균 주문 금액이 높고 고객이 일반적으로 구매 전에 조사하는 경우, 쿠키 유효 기간을 60–90일로 연장하여 지연된 전환을 포착할 수 있습니다.
- **리퍼러 데이터를 통해 채널 통찰력 확인**: 리퍼러 필드는 클릭이 어디서 오는지를 보여줍니다. "instagram.com" 또는 "youtube.com"에서 많은 클릭이 보인다면, 파트너가 가장 효과적으로 사용하는 소셜 플랫폼을 알 수 있습니다.
