---
title: 了解佣金
---

佣金是在联盟会员成功将销售引导至您的商店时创建的收入记录。每个佣金都与特定的订单、联盟会员和计划相关联，并从待处理状态过渡到已支付状态。本指南将解释佣金的工作原理、计算方式以及如何有效管理它们。

## 什么是佣金？

佣金代表联盟会员因推荐完成购买的客户而应得的金额。当客户点击联盟会员的推荐链接并在cookie生命周期窗口内下单时，Spwig会自动生成佣金记录。

每个佣金包含：
- **联盟会员** — 推荐客户的合作伙伴
- **计划** — 定义佣金规则的联盟计划
- **订单** — 生成佣金的订单
- **金额** — 计算出的佣金值
- **状态** — 佣金生命周期中的当前阶段
- **日期** — 创建日期、批准/拒绝日期和支付日期

## 佣金计算

佣金的计算基于计划的佣金类型和费率自动进行。

| 佣金类型 | 计算方式 | 示例 |
|----------|---------|------|
| **百分比** | 订单总额 × 佣金百分比 ÷ 100 | 订单：200美元，费率：10% → **20美元佣金** |
| **固定金额** | 每个订单固定金额 | 费率：15美元 → **15美元佣金**（无论订单价值如何） |

### 计算示例

**百分比佣金（10%）**：
- 客户下单50美元 → 5美元佣金
- 客户下单150美元 → 15美元佣金
- 客户下单300美元 → 30美元佣金

**固定佣金（20美元）**：
- 客户下单50美元 → 20美元佣金
- 客户下单150美元 → 20美元佣金
- 客户下单300美元 → 20美元佣金

佣金是基于**订单小计**（不含运费和税）计算的，并在订单下单时立即创建。

## 佣金生命周期

每个佣金从创建到付款都会经历一系列状态：

```
待处理 → 已批准 → 已支付
   ↓
已拒绝
```

### 状态定义

| 状态 | 描述 | 发生情况 |
|------|------|---------|
| **待处理** | 订单已下单，佣金等待审核 | 佣金已创建但尚未确认。联盟会员可以看到它，但无法提取资金。 |
| **已批准** | 商家确认销售有效 | 佣金已验证并添加到联盟会员的可用余额中。有资格获得付款。 |
| **已拒绝** | 商家拒绝佣金 | 佣金被拒绝（例如，订单被退款、欺诈或违反条款）。无资格获得付款。 |
| **已支付** | 佣金包含在已完成的付款中 | 联盟会员已获得付款。佣金最终确定且无法修改。 |

## 佣金何时创建

佣金按照以下顺序自动生成：

1. **客户点击联盟链接** — 推荐URL包含联盟会员的唯一跟踪代码（例如，`?ref=JOHNSMITH`）
2. **设置Cookie** — 在客户的浏览器中存储包含联盟代码的跟踪Cookie
3. **在Cookie生命周期内下单** — 客户在Cookie过期前完成订单（默认：30天）
4. **系统分配订单** — Spwig检查是否有活动的跟踪Cookie并识别推荐联盟会员
5. **自动生成佣金** — 创建状态为**待处理**的佣金记录

订单下单时佣金会**立即**创建，即使在支付确认之前。这使商家能够在订单处理过程中审核佣金。

## 跟踪与归属

Spwig使用**最后点击归属**来确定哪个联盟会员应获得销售的信用。

### 归属机制

- **最后点击模型** — 最近点击的联盟链接获得信用（即使客户被多个联盟会员推荐）
- **基于Cookie的跟踪** — Cookie在客户的浏览器中存储联盟代码
- **Cookie生命周期** — 确定销售可以归属的时间窗口（按计划配置，通常为30天）
- **IP和会话跟踪** — 额外数据有助于识别欺诈模式

### 归属示例

- 第1天：客户点击联盟会员A的链接 → 为联盟会员A设置Cookie
- 第5天：客户点击联盟会员B的链接 → Cookie**更新**为联盟会员B（最后点击胜出）
- 第7天：客户下单 → 佣金归属**联盟会员B**

如果客户在第35天（Cookie过期后）返回并下单，**不会创建佣金**，因为跟踪窗口已关闭。

## 佣金详情

导航至 **营销 > 佣金** 以查看所有佣金记录。

### 佣金字段

每个佣金显示：

| 字段 | 描述 |
|------|------|
| **联盟会员** | 联盟会员的名称和代码 |
| **计划** | 联盟计划名称 |
| **订单** | 订单编号（可点击链接查看完整订单详情） |
| **金额** | 计算出的佣金值 |
| **状态** | 当前阶段（待处理、已批准、已拒绝、已支付） |
| **创建时间** | 佣金生成的时间 |
| **批准/拒绝日期** | 状态更新的时间 |
| **支付日期** | 付款处理的时间 |
| **备注** | 关于佣金的内部备注 |

### 查看订单详情

点击佣金记录中的**订单编号**以查看原始订单。这使您可以验证：
- 订单总额和购买的商品
- 客户信息
- 支付状态
- 运输状态
- 任何退款或退货

此上下文有助于您决定是否批准或拒绝佣金。

## 管理佣金

虽然本指南重点在于理解佣金，但批准、拒绝和支付佣金的实际步骤在**佣金管理**帮助主题中有详细说明。

### 快速概述

- **批准** — 验证订单是合法的并确认佣金有效
- **拒绝** — 拒绝欺诈订单、退款或违反政策的佣金
- **添加备注** — 记录批准或拒绝的原因，供以后参考
- **处理付款** — 将已批准的佣金分组为批量付款

查看相关帮助主题，以获取每项管理任务的逐步说明。

## 小贴士

- 在您第一个月内**每天**审查待处理佣金，以建立节奏并尽早发现任何跟踪问题
- 设置**电子邮件通知**，当新佣金创建时通知您，以便在订单详情仍新鲜时进行审查
- 在**订单完成之后**（而不是在订单下单时立即）批准佣金，以考虑取消和退货
- 使用**备注字段**记录决策，特别是对于被拒绝的佣金，这样如果您有联盟会员提出问题，您有记录可查
- 寻找**拒绝的模式** — 如果某个联盟会员有大量被拒绝的佣金，这可能表明欺诈或对计划条款的误解
- 考虑创建**佣金批准政策**（例如，"在14天退货窗口后批准"）并告知联盟会员，以明确预期
