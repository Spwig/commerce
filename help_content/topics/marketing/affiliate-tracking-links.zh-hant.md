追蹤聯營夥伴與連結

聯營夥伴追蹤功能透過將客戶的購買行為與推薦他們的聯營夥伴連結起來，從而推動整個傭金系統。本指南說明追蹤連結的工作原理、Spwig 在客戶點擊這些連結時會記錄哪些資料，以及基於 Cookie 的歸因系統如何決定每個傭金由哪個聯營夥伴獲得。

了解追蹤機制有助於您解決歸因問題、分析連結表現，並教育您的聯營夥伴如何最大化他們的轉化率。

## 什麼是追蹤連結？

追蹤連結是一種獨特的 URL，它會將客戶重定向到您的商店，同時在 Cookie 中記錄聯營夥伴的身份。每個聯營夥伴可以創建多個追蹤連結，指向不同的目的地——首頁、特定產品、商品集合頁面或著陸頁面。

追蹤連結格式範例：
```
https://yourstore.com/affiliate/track/a2b7f8c4d1e9/
```

此連結會將客戶重定向到目的地，同時設置一個追蹤 Cookie，將未來的購買行為與擁有連結代碼 `a2b7f8c4d1e9` 的聯營夥伴關聯起來。

聯營夥伴會從他們的門戶儀表板生成這些連結。他們複製完整的 URL，並在部落格文章、社交媒體、電子郵件或任何接觸潛在客戶的渠道中分享。

## 追蹤連結組成部分

每個追蹤連結包含以下元素：

| 元件 | 範例 | 說明 |
|------|------|------|
| **基礎 URL** | `https://yourstore.com` | 您的商店域名 |
| **追蹤路徑** | `/affiliate/track/` | Spwig 的追蹤端點 |
| **連結代碼** | `a2b7f8c4d1e9` | 自動生成的 12 個字符唯一識別碼 |
| **目的地** | 創建連結時設置 | 客戶重定向後到達的位置（首頁、產品等） |

當聯營夥伴創建連結時，Spwig 會自動生成唯一的 12 個字符代碼。聯營夥伴無需手動創建或編輯此代碼——他們只需選擇目的地，Spwig 會處理其餘部分。

### 連結標籤（可選）

聯營夥伴可以為每個連結添加標籤，以便於他們自己的組織：
- "Instagram 個人簡介連結"
- "YouTube 描述"
- "黑色星期五電子郵件活動"

標籤幫助聯營夥伴追蹤哪些推廣渠道表現最佳。這些標籤僅對聯營夥伴和您可見，客戶永遠看不到標籤。

## 追蹤機制

從點擊到傭金的追蹤和歸因過程遵循五個步驟：

### 1. 客戶點擊連結

潛在客戶從任何推廣渠道（社交媒體帖子、部落格文章、電子郵件簡訊）點擊聯營夥伴的追蹤連結。

### 2. 記錄點擊

Spwig 的追蹤端點會記錄點擊細節：
- IP 地址
- User agent（瀏覽器和設備）
- HTTP referrer（點擊來源）
- 時間戳
- 會話識別碼

這些數據會出現在 **Affiliate > Clicks** 的 **Clicks** 管理介面中，用於分析和欺詐檢測。

### 3. 設置 Cookie

追蹤系統會在客戶瀏覽器中設置 Cookie，然後將其重定向。Cookie 包含：
- 聯營夥伴 ID（應獲得傭金的聯營夥伴）
- 程式 ID（應用的傭金結構）
- 連結代碼（點擊的具體連結）

### 4. 客戶購買

客戶瀏覽您的商店並完成購買。這可以立即發生，也可以在幾天或幾週後發生，只要他們在 Cookie 有效期限內完成購買。

### 5. 創建傭金

在結帳時，Spwig 會檢查是否存在聯營夥伴 Cookie。如果找到且仍然有效（在 Cookie 有效期限內），系統會創建一個與聯營夥伴、程式和訂單相關聯的狀態為 **Pending** 的傭金記錄。

## 基於 Cookie 的歸因

追蹤 Cookie 是將購買行為與聯營夥伴連結的核心機制。了解 Cookie 的工作原理有助於您設置最佳的歸因窗口並解決追蹤問題。

### Cookie 結構

| 屬性 | 值 |
|------|----|
| **名稱** | `aff_{program_id}`（例如，`aff_7` 對應程式 ID 7） |
| **值** | 包含聯營夥伴 ID、連結代碼和時間戳的 JSON |
| **域名** | 您的商店域名 |
| **路徑** | `/`（網站範圍的訪問權限） |
| **持續時間** | 程式 Cookie 有效期限（1–365 天） |
| **HttpOnly** | `true`（防止 JavaScript 進行安全訪問） |
| **SameSite** | `Lax`（允許從外部來源進行追蹤） |
| **Secure** | 在 HTTPS 網站上為 `true`（建議） |

### Cookie 有效期限窗口

Cookie 有效期限決定了客戶在點擊聯營夥伴連結後有多長時間可以完成購買。此窗口在 **Marketing > Affiliate Programs** 設置，當您創建或編輯程式時。

行業標準 Cookie 有效期限：
- **7 天**：快速決策產品（雜貨、活動門票）
- **30 天**：標準電商（最常見的設置）
- **60–90 天**：考慮性購買（家具、電子產品、B2B 產品）
- **365 天**：長銷售週期（奢侈品、高價服務）

如果客戶在 1 月 1 日點擊了聯營夥伴連結，且您的 Cookie 有效期限為 30 天，那麼他們在 1 月 30 日之前通過該連結完成的任何購買都會歸功於該聯營夥伴。1 月 31 日或之後的購買不會產生傭金，因為 Cookie 已過期。

### 最後點擊歸因模型

Spwig 使用 **最後點擊歸因**：最近的聯營夥伴連結獲得傭金。以下是其運作方式：

**情境**：客戶在週一點擊聯營夥伴 A 的連結，週三點擊聯營夥伴 B 的連結，週五完成購買。

**結果**：聯營夥伴 B 獲得傭金，因為他們的連結是最近的點擊。

最後點擊 Cookie 會覆蓋之前的聯營夥伴 Cookie。此模型簡單易懂，可防止重複傭金，但這意味著每個訂單只有一個聯營夥伴獲得傭金（購買前最近的一個）。

## 點擊記錄

Spwig 記錄每個聯營夥伴連結的每一次點擊，以提供您和聯營夥伴的分析數據。點擊數據有助於衡量連結表現、檢測欺詐行為，並優化推廣策略。

### 每次點擊記錄的數據

導航至 **Affiliate > Clicks** 以查看所有記錄的點擊。每個條目包含：

| 欄位 | 說明 |
|------|------|
| **連結** | 哪個追蹤連結被點擊 |
| **聯營夥伴** | 哪個聯營夥伴擁有該連結 |
| **IP 地址** | 客戶的 IP（用於欺詐檢測） |
| **User Agent** | 瀏覽器和設備資訊 |
| **來源** | 客戶點擊連結的頁面（例如，"https://instagram.com"） |
| **會話 ID** | 本次瀏覽會話的唯一識別碼 |
| **時間戳** | 點擊的確切日期和時間 |

### 速率限制

為防止點擊欺詐和機器人濫用，Spwig 將點擊限制為 **每分鐘每 IP 地址 100 次**。如果同一 IP 超過此閾值，額外的點擊將被忽略，且不會增加點擊計數。

此保護措施可防止惡意行為者虛構點擊數據，同時不阻礙合法流量。真正的客戶幾乎從來不會超過每分鐘 100 次點擊。

### 隱私考量

點擊數據包含 IP 地址和 User Agent，用於欺詐檢測。請確保您的隱私權政策披露您追蹤聯營夥伴推薦，並與聯營夥伴分享匿名化的表現數據。

## 查看聯營夥伴連結

所有由聯營夥伴生成的追蹤連結都會出現在您的管理面板中，以便監控和管理。

### 存取連結清單

導航至 **Affiliate > Links** 以查看所有聯營夥伴和程式的所有追蹤連結。清單視圖顯示：

- **連結代碼**：唯一的 12 個字符識別碼
- **聯營夥伴**：創建連結的聯營夥伴
- **程式**：應用的傭金結構
- **標籤**：聯營夥伴提供的可選描述
- **目的地**：連結重定向客戶的位置
- **總點擊數**：所有時間的點擊數
- **狀態**：連結目前是否在追蹤

### 過濾連結

使用管理面板的過濾器來縮小清單：
- **按聯營夥伴**：查看特定合作夥伴的所有連結
- **按程式**：查看推廣特定傭金結構的連結
- **按狀態**：查找已停用的連結

此過濾功能有助於您分析聯營夥伴網絡中的連結分佈，並識別表現最佳的連結。

## 連結統計數據

每個追蹤連結累積的表現指標有助於聯營夥伴優化他們的推廣策略，並幫助您識別表現最佳的合作伙伴。

### 點擊連結記錄以查看詳細統計數據：

| 指標 | 說明 | 計算方式 |
|------|------|---------|
| **總點擊數** | 自連結創建以來的所有記錄點擊 | 點擊記錄的數量 |
| **點擊數（7 天）** | 最近活動指標 | 最近 7 天的點擊數 |
| **轉化率** | 被歸因於此連結的訂單數 | 此連結代碼的傭金數量 |
| **轉化率** | 點擊導致購買的百分比 | (轉化率 ÷ 總點擊數) × 100 |
| **總營收** | 此連結所有訂單價值總和 | 轉化點擊的訂單總額總和 |

### 使用統計數據進行優化

**對於聯營夥伴**：這些數字顯示哪些推廣渠道效果最佳。如果 Instagram 個人簡介連結的轉化率為 5%，但部落格文章連結的轉化率為 15%，聯營夥伴應更加關注部落格內容。

**對於商家**：連結統計數據揭示哪些聯營夥伴帶來了高品質流量。高點擊數但低轉化率表明聯營夥伴的受眾與您的產品不匹配。

## 管理連結

您可以從管理面板管理聯營夥伴連結，以便於維護和故障排除。

### 停用連結

要防止特定連結追蹤新點擊，同時保留歷史數據：

1. 導航至 **Affiliate > Links**
2. 點擊您要停用的連結
3. 取消勾選 **Active**
4. 點擊 **Save**

停用的連結仍然會將客戶重定向到目的地，但不會設置追蹤 Cookie 或記錄點擊。這在聯營夥伴運行臨時活動或您需要停用特定推廣渠道時很有用。

### 編輯連結細節

您可以修改：
- **標籤**：更新聯營夥伴提供的描述
- **目的地**：更改連結重定向的位置（如果您移動了產品頁面時很有用）
- **狀態**：啟用或停用追蹤

您不能編輯連結代碼——它是永久性的，與所有歷史點擊和傭金數據相關聯。

### 刪除無效連結

刪除不再使用且沒有歷史點擊或轉化的連結。這有助於保持您的連結清單整潔，同時不丟失寶貴的分析數據。

**警告**：刪除連結會刪除所有相關的點擊記錄。只有在確認連結沒有點擊或完全確定不需要歷史數據時才刪除連結。

## 归因模型

了解 Spwig 的归因逻辑有助于您与联营伙伴设定预期，并解决佣金争议。

### 最后点击归因

如前所述，Spwig 使用最后点击归因：如果客户在购买前点击了多个联营伙伴链接，只有最近点击的联营伙伴会获得佣金。

**优点**：
- 易于理解和解释
- 防止重复佣金
- 奖励促成交易的联营伙伴

**缺点**：
- 早期介绍客户的联营伙伴得不到任何信用
- 无法反映多触点客户旅程
- 可能激励“链接劫持”（联营伙伴针对已被其他联营伙伴推荐的高意向客户）

### Cookie 有效期决定资格

只有在Cookie有效期窗口内的购买才会产生佣金。如果Cookie在结账前过期，即使客户通过书签返回，也不会产生佣金。

**示例**：30天Cookie有效期
- 客户1月1日点击链接→Cookie设置，1月31日过期
- 客户1月25日购买→佣金创建
- 客户2月5日购买→无佣金（Cookie已过期）

### 会话跟踪

除了Cookie，Spwig还会跟踪每个点击的会话ID。这使得即使Cookie被阻止或清除，也能在同一个会话中进行多访问归因。

如果客户点击链接，浏览您的商店触发多次页面加载，然后他们购买——所有都在同一个会话中——即使没有持久Cookie，联营伙伴也会获得信用。

## 故障排除

常见追踪问题及解决方法：

### 链接未追踪点击

**症状**：尽管联营伙伴报告分享了链接，但点击次数保持为零。

**原因和解决方法**：
1. **链接已停用**：检查链接详情页面的 **Active** 状态
2. **程序未激活**：导航到 **Affiliate > Programs** 并确认程序状态为 **Active**
3. **联营伙伴账户已停用**：检查联营伙伴的账户状态在 **Affiliate > Affiliates**
4. **速率限制**：检查同一IP是否生成了过多的点击（机器人流量）

### 转化率低

**症状**：点击次数高但归因的订单很少。

**原因和解决方法**：
1. **Cookie 有效期太短**：如果您的产品需要研究和考虑，请增加程序的Cookie有效期
2. **目的地页面质量**：检查着陆页——是否适合移动设备？加载速度快吗？产品是否有库存？
3. **受众不匹配**：联营伙伴的受众可能与您的产品不匹配
4. **浏览器阻止Cookie**：一些隐私工具会阻止第三方Cookie，尽管Spwig使用第一方Cookie，这不太可能被阻止

### 重复点击记录

**症状**：同一客户在短时间内生成多个点击记录。

**原因**：这是正常现象。每次加载跟踪链接的页面都会创建一个点击记录。如果客户点击链接，页面加载缓慢，他们再次点击，您会看到多个记录。

**解决方法**：无需操作。速率限制器防止滥用（每分钟每IP 100次点击），并且同一会话中的重复点击不会影响归因——只设置一个Cookie。

## 小贴士

- **在上线前测试追踪** — 创建一个测试联营伙伴账户，生成一个追踪链接，在无痕浏览器中点击它，并完成一个测试购买。确认佣金以正确的联营伙伴归因出现。
- **教育联营伙伴关于Cookie有效期** — 确保联营伙伴了解他们只能获得在Cookie窗口内完成的购买佣金。这有助于他们设定现实的期望并专注于高意向流量。
- **监控点击模式以检测欺诈** — 单个IP的异常高点击次数或没有User Agent字符串的点击可能表明机器人流量。在批准佣金前仔细审查这些联营伙伴。
- **一致使用链接标签** — 鼓励联营伙伴按渠道（Instagram、部落格、电子邮件）标记他们的链接，这样您都可以分析哪些推广渠道带来最佳转化。
- **高价值产品考虑更长的Cookie有效期** — 如果您的平均订单价值较高且客户通常在购买前进行研究，请将Cookie有效期延长至60-90天，以捕捉这些延迟转化。
- **检查来源数据以获得渠道洞察** — 来源字段显示点击来源。如果您看到很多来自“instagram.com”或“youtube.com”的点击，您就知道您的联营伙伴最有效地使用了哪些社交平台。

記住：保留所有 Markdown 格式、圖片路徑、程式碼塊和技術術語，完全按照保存規則所示。
