---
title: コミッション管理
---

コミッション管理は、アフィリエイトの収益を確認し、正当な販売のみを認証するプロセスです。本ガイドでは、保留中のコミッションを確認し、有効なものを承認し、詐欺や返品の注文を拒否し、バッチアクションを使用してコミッションを効率的に管理する方法を説明します。

## コミッションダッシュボード

**マーケティング > コミッション**に移動して、コミッション管理ダッシュボードにアクセスしてください。

ダッシュボードは、すべてのアフィリエイトプログラムにわたるコミッション活動の概要を提供します：

| 統計 | 説明 |
|-----------|-------------|
| **保留中のコミッション** | あなたの確認を待っているコミッションの数 |
| **承認済みコミッション** | 支払い準備が整ったコミッション |
| **支払い済みコミッション** | アフィリエイトに支払われたコミッション |
| **拒否されたコミッション** | 詐欺、返品、またはポリシー違反により拒否されたコミッション |
| **未払いコミッション額** | 承認済みだが未払いのコミッションの合計価値 |

これらの統計は、あなたの確認作業量を追跡し、アフィリエイトプログラムの財務影響を監視するのに役立ちます。

![コミッションダッシュボード](/static/core/admin/img/help/commission-management/commission-dashboard.webp)

## コミッションの表示

コミッションリストは、すべてのコミッション記録を時系列順に表示します。

### リストの列

| 列 | 説明 |
|--------|-------------|
| **アフィリエイト** | アフィリエイトの名前とユニークなコード |
| **プログラム** | このコミッションを生成したアフィリエイトプログラム |
| **注文** | 注文番号（クリックして完全な注文詳細を表示） |
| **金額** | あなたの店舗の通貨で表されるコミッション価値 |
| **ステータス** | 保留中、承認済み、拒否済み、または支払い済み |
| **作成日時** | コミッションが生成された日時 |

### コミッションのフィルタリング

フィルターサイドバーを使用してコミッションを絞り込みます：

- **ステータスでフィルター** — 保留中、承認済み、拒否済み、または支払い済みのコミッションのみを表示
- **アフィリエイトでフィルター** — 特定のパートナーのコミッションを表示
- **プログラムでフィルター** — 特定のアフィリエイトプログラムのコミッションを表示
- **日付範囲でフィルター** — 作成日でフィルター

### コミッションの検索

検索バーを使用して特定のコミッションを検索します：

- **注文番号**を入力して、特定の販売のコミッションを検索
- **アフィリエイトコード**を入力して、1つのパートナーのすべてのコミッションを表示

## コミッションの詳細

リスト内の任意のコミッションをクリックして、その完全な詳細情報を表示します。

### 詳細フィールド

詳細ビューでは以下が表示されます：

- **注文情報** — 注文番号をクリックして、新しいタブで注文の詳細を表示（商品、配送先住所、支払い状況、顧客情報など）
- **アフィリエイト情報** — アフィリエイトの名前、コード、支払い用メールアドレス、およびプログラム会員ステータス
- **プログラム詳細** — プログラム名、コミッションタイプ（パーセンテージまたは固定）、およびコミッション率
- **タイムスタンプ** — 作成日、承認/拒否日、および支払い日
- **ノートセクション** — 商家のみが見える内部ノート（下記で説明）

この情報は、コミッションを承認する前にその正当性を確認するために役立ちます。

## コミッションの承認

コミッションを承認すると、それが有効であることを確認し、アフィリエイトの利用可能な残高に追加され、支払い対象になります。

### 承認するタイミング

以下の場合にコミッションを承認してください：

- **注文が正常に履行された** — 商品が発送されたり、デジタル商品が配信されたりした場合
- **返品や返金が発生していない** — 顧客が返品を要求していない（履行後14〜30日待つことを検討）
- **品質基準が満たされている** — 販売がプログラムの条件を満たしている（例：セルフリファラルでない、顧客が本物の支払い方法を使用している）
- **詐欺が検出されていない** — 注文が詐欺スクリーニングを通過している（IPアドレス、請求/配送先住所の不一致、異常な注文パターンを確認）

### 承認方法

**単一コミッションの承認：**

1. **マーケティング > コミッション**に移動します
2. 承認したいコミッションをクリックします
3. 詳細ページの上部にある**承認**ボタンをクリックします
4. 任意でノートを追加します（例：「配信が成功した後に承認しました」）
5. ステータスが**承認済み**に変更され、コミッションがアフィリエイトの残高に追加されます

**バッチ承認：**

1. **マーケティング > コミッション**に移動します
2. 承認したいコミッションの横にあるチェックボックスをクリックします
3. **アクション**ドロップダウンから**選択したものを承認**を選択します
4. **Go**をクリックします
5. 選択したすべてのコミッションのステータスが**承認済み**に変更されます

承認されたコミッションは、アフィリエイトのダッシュボードで利用可能な残高として表示され、次の支払いバッチに含まれることができます。

## コミッションの拒否

コミッションを拒否すると、アフィリエイトの残高から削除され、支払い対象外となります。

### 拒否するタイミング

以下の場合にコミッションを拒否してください：

- **詐欺的な注文** — 注文に詐欺の兆候がある（盗難された支払い方法、IPアドレスの不一致、アフィリエイトが自身のリンクを使用している）
- **顧客が商品を返品した** — 顧客が全額返金のために商品を返品した
- **品質問題** — 販売がプログラムの条件を満たしていない（例：アフィリエイトが広告ガイドラインを違反している）
- **条件違反** — アフィリエイトが禁止されたプロモーション方法を使用している（スパム、商標入札、クッキー詰め放題）
- **注文がキャンセルされた** — 顧客が履行前に関わった

### 拒否方法

**単一コミッションの拒否：**

1. **マーケティング > コミッション**に移動します
2. 拒否したいコミッションをクリックします
3. 詳細ページの上部にある**拒否**ボタンをクリックします
4. **理由を説明するノート**を追加してください（紛争解決のために強く推奨されます）
5. ステータスが**拒否済み**に変更されます

**バッチ拒否：**

1. **マーケティング > コミッション**に移動します
2. 拒否したいコミッションの横にあるチェックボックスをクリックします
3. **アクション**ドロップダウンから**選択したものを拒否**を選択します
4. **Go**をクリックします
5. 選択したすべてのコミッションのステータスが**拒否済み**に変更されます

拒否されたコミッションは、アフィリエイトの残高から削除され、支払いできません。それらはコミッション履歴に表示され、記録のため残ります。

## バッチアクション

バッチアクションを使用すると、一度に複数のコミッションを承認または拒否でき、大量のバッチを処理するときに時間を節約できます。

### バッチアクションの使用方法

1. **マーケティング > コミッション**に移動します
2. 処理したいコミッションのみを表示するようにリストをフィルタリングします（例：ステータス**保留中**でフィルタリング）
3. 各コミッションの横にあるチェックボックスをクリックするか、現在のページですべてを選択するヘッダーのチェックボックスをクリックします
4. **アクション**ドロップダウンからアクションを選択します：
   - **選択したものを承認** — 選択したすべてのコミッションを承認状態にマークします
   - **選択したものを拒否** — 選択したすべてのコミッションを拒否状態にマークします
5. **Go**をクリックします
6. 更新されたコミッションの数を示す確認メッセージを確認します

### 効率的なバッチ処理

- **プログラムでフィルタリング** — 信頼できる高パフォーマンスのアフィリエイトのすべてのコミッションを一度に承認
- **日付範囲でフィルタリング** — 14日以上前のコミッション（返品期限を過ぎた）を処理
- **高価値コミッションを個別に確認** — 小額コミッションにバッチアクションを使用し、高価値コミッションは個別に確認

## コミッションノート

ノートフィールドは、あなたの決定を記録し、チームとコミュニケーションを取るのに役立ちます。

### ノートの追加

ノートは以下のように追加できます：

- **承認時に** — コミッションをクリックし、ノートフィールドにノートを追加してから**承認**をクリックします
- **拒否時に** — 拒否理由を説明するノートを追加します
- **いつでも** — コミッションをクリックし、ノートフィールドにノートを追加または編集してから保存します

### ノートの使用タイミング

- **拒否されたコミッション** — 常に理由を記録します（例：「顧客は2月10日26日に注文#12345を返品しました」）
- **高価値コミッション** — 検証手順を記録します（例：「トラッキング番号#ABC123で配達を確認しました」）
- **紛争中のコミッション** — アフィリエイトとのコミュニケーションを記録します
- **詐欺のパターン** — 今後の参考に、疑わしい活動を記録します

ノートは**内部用のみ**です — アフィリエイトはそれを見ることはできません。記録のためのあなたのツールとして機能します。

## コミッションフロー

ここに、コミッション管理ワークフローの全体像を示します：

```
注文が行われる → コミッションが作成される（保留中）
                      ↓
              商家による確認
                      ↓
                ┌─────┴─────┐
                ↓           ↓
            承認済み     拒否済み
                ↓           ↓
        支払い対象     支払い不可
                ↓
        支払いに含まれる
                ↓
              支払い済み
```

**タイムラインの例：**

- **1日目：** 顧客がアフィリエイトリンクを通じて$100の注文を置く → $10のコミッションが作成される（保留中）
- **15日目：** 注文が履行され、返品期限が過ぎる → 商家がコミッションを承認
- **20日目：** 商家が月次の支払いバッチを処理 → コミッションのステータスが支払い済みに変更される
- **21日目：** アフィリエイトがPayPalを通じて支払いを受ける

## 最佳実践

### 確認期間

一貫した確認スケジュールを確立してください：

- **毎日の確認** — 毎朝保留中のコミッションを処理（高ボリュームのプログラムには推奨）
- **週次の確認** — 毎週月曜日に前週のコミッションを承認するための時間を確保してください
- **2週間ごとの確認** — 支払いスケジュールと合わせて（中旬にコミッションを承認し、月末に支払いを処理）

### 品質保証チェック

コミッションを承認する前に確認してください：

1. **注文が履行されている** — 管理画面で注文のステータスを確認します
2. **支払いが確認されている** — 支払い方法が正常に処理されていることを確認します
3. **返品期限が過ぎている** — 配達後14〜30日間待って返品を考慮します
4. **詐欺フラグがない** — 注文を確認し、疑わしいパターン（住所の不一致、高リスクの国、同じIPから複数の注文）を確認します
5. **アフィリエイトが良好な状態にある** — アフィリエイトの履歴を確認し、以前の詐欺や違反がないか確認します

### 詐欺防止

以下の赤いフラグに注意してください：

- **自己リファラル** — アフィリエイトが自身のトラッキングリンクを使用して注文を置く
- **クッキー詰め放題** — 通常よりも高いクリックからコンバージョン比率で、注文価値が低い
- **重複注文** — 同じ顧客/IPから同じアフィリエイトリンクを通じて複数の注文
- **地理的不一致** — 国Aに所属するアフィリエイトが国Bでのみ販売を促進している
- **チャージバック** — アフィリエイトが紹介した注文のチャージバック率が高い

詐欺を検出した場合は、**コミッションを拒否**し、アフィリエイトのプログラム会員資格を終了させる検討をします。

### アフィリエイトとのコミュニケーション

- **期待値を設定** — プログラムの条件に明確に記載されたコミッション承認ポリシーをアフィリエイトに提供してください
- **透明性を保つ** — コミッションを拒否した場合は、アフィリエイトに理由を説明するメールを送信してください（ノートを参考に）
- **紛争への対応** — アフィリエイトが拒否に異議を唱える場合、ノートと注文の詳細を確認してください
- **ガイドラインの作成** — アフィリエイトポータルに「コミッション承認ポリシー」ページを作成して、混乱を避けてください

## ヒント

- 返品期限が終了した後にコミッションを承認してください（通常14〜30日） — これにより、顧客が後に返品する注文を承認してしまうことを防ぎます
- **フィルタリングとバッチアクション**を使用して、信頼できるアフィリエイトのコミッションを効率的に処理し、新しいまたは高リスクのパートナーは手動で確認してください
- **拒否理由をノートフィールドに記録** — これにより、アフィリエイトが決定に異議を唱えた場合に保護され、パターンの特定に役立ちます
- **自己リファラル**に注意してください — これは、アフィリエイトが自身のリンクを使用して個人的な購入でコミッションを獲得するという一般的な違反です
- **最小承認基準**を設定してください — 例えば、$10未満のコミッションは自動的に承認し、$50以上のものは手動で確認して、効率とリスクのバランスを取る
- **詐欺チェックリスト**を作成してください — 疑わしいフラグ（IPアドレスの不一致、疑わしい注文パターン、高リスクの支払い方法）を含むレビューのプロセスを標準化してください
- **アフィリエイトごとの拒否率を監視** — あるアフィリエイトが多くの拒否がある場合、詐欺の可能性や、プログラムの条件に関する追加のトレーニングが必要であることを示しているかもしれません
