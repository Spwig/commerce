---
title: アフィリエイトのトラッキングとリンク
---

アフィリエイトトラッキングは、顧客の購入を紹介したアフィリエイトと関連付けることで、全体のコミッションシステムを動かしています。このガイドでは、トラッキングリンクの仕組み、顧客がそのリンクをクリックした際にSpwigが記録するデータ、およびクッキーに基づく帰属システムがどのアフィリエイトが各コミッションを獲得するかをどのように決定するかを説明します。

トラッキングのメカニズムを理解することで、帰属の問題をトラブルシューティングし、リンクのパフォーマンスを分析し、アフィリエイトにコンバージョンを最大化する方法を教えることができます。

## トラッキングリンクとは？

トラッキングリンクとは、顧客をストアにリダイレクトしながら、アフィリエイトのIDをクッキーに記録する一意のURLです。各アフィリエイトは、ホーム画面、特定の商品、コレクションページ、またはランディングページなど、さまざまな目的地を指す複数のトラッキングリンクを作成できます。

トラッキングリンクの例:
```
https://yourstore.com/affiliate/track/a2b7f8c4d1e9/
```

このリンクは、目的地にリダイレクトするだけでなく、リンクコード `a2b7f8c4d1e9` を所有するアフィリエイトと関連付けるトラッキングクッキーを設定します。

アフィリエイトは、自分のポータルダッシュボードからこれらのリンクを生成します。彼らはフルURLをコピーし、ブログ投稿、ソーシャルメディア、メール、または潜在顧客に到達する他のチャネルで共有します。

## トラッキングリンクの構成要素

すべてのトラッキングリンクには、以下の要素が含まれています:

| コンポーネント | 例 | 説明 |
|---------------|-----|-------|
| **ベースURL** | `https://yourstore.com` | あなたのストアのドメイン |
| **トラッキングパス** | `/affiliate/track/` | Spwigのトラッキングエンドポイント |
| **リンクコード** | `a2b7f8c4d1e9` | 自動生成された12文字の一意な識別子 |
| **目的地** | リンクが作成されるときに設定 | リダイレクト後の顧客の着地先（ホーム画面、商品など） |

アフィリエイトがリンクを作成すると、Spwigは自動的に一意な12文字のコードを生成します。アフィリエイトはこのコードを手動で作成または編集する必要がありません——彼らは単に目的地を選択し、Spwigが残りの処理を行います。

### リンクラベル（オプション）

アフィリエイトは、自分の組織のために各リンクにラベルを追加できます:
- "Instagram Bio Link"
- "YouTube Description"
- "Black Friday Email Campaign"

ラベルは、アフィリエイトがどのプロモーションチャネルが最も効果的かを追跡するのに役立ちます。これらはアフィリエイトとあなただけに表示され、顧客は決してラベルを見ません。

## トラッキングの仕組み

トラッキングと帰属プロセスは、クリックからコミッションに至るまで5つのステップに従います:

### 1. 顧客がリンクをクリック

潜在顧客は、ソーシャルメディア投稿、ブログ記事、メールニュースレターなど、どのプロモーションチャネルからでもアフィリエイトのトラッキングリンクをクリックします。

### 2. クリックが記録される

Spwigのトラッキングエンドポイントは、クリックの詳細を記録します:
- IPアドレス
- User agent（ブラウザとデバイス）
- HTTPリファラ（クリックがどこから来たか）
- タイムスタンプ
- セッションID

このデータは、**クリック**管理画面の **アフィリエイト > クリック** に表示され、分析と不正検出に使用されます。

### 3. クッキーが設定される

トラッキングシステムは、顧客をリダイレクトする前にブラウザにクッキーを設定します。クッキーには以下が含まれます:
- アフィリエイトID（どのアフィリエイトがコミッションを獲得するか）
- プログラムID（どのコミッション構造が適用されるか）
- リンクコード（どの特定のリンクがクリックされたか）

### 4. 顧客が購入

顧客はストアを閲覧し、購入を完了します。これは即座に、または数日〜数週間後でも構いません。ただし、クッキーの有効期間内に購入する必要があります。

### 5. コミッションが作成される

チェックアウト時に、Spwigはアフィリエイトクッキーを確認します。クッキーが見つかり、有効期間内にあれば、システムはアフィリエイト、プログラム、注文に関連する **Pending** ステータスのコミッションレコードを作成します。

## クッキーに基づく帰属

トラッキングクッキーは、購入をアフィリエイトと関連付けるためのコアメカニズムです。クッキーの仕組みを理解することで、最適な帰属ウィンドウを設定し、トラッキングの問題をトラブルシューティングできます。

### クッキー構造

| プロパティ | 値 |
|-----------|----|
| **名前** | `aff_{program_id}`（例: `aff_7` for program ID 7） |
| **値** | アフィリエイトID、リンクコード、タイムスタンプを含むJSON |
| **ドメイン** | あなたのストアのドメイン |
| **パス** | `/`（サイト全体へのアクセス） |
| **期間** | プログラムのクッキー有効期間（1〜365日） |
| **HttpOnly** | `true`（セキュリティのためJavaScriptによるアクセスを防ぐ） |
| **SameSite** | `Lax`（外部リファラからトラッキングを許可） |
| **Secure** | HTTPSサイトでは `true`（推奨） |

### クッキー有効期間ウィンドウ

クッキー有効期間は、顧客がアフィリエイトリンクをクリックしてから購入するまでの時間を決定します。このウィンドウは、**マーケティング > アフィリエイトプログラム**でプログラムを作成または編集するときに、各プログラムごとに設定されます。

業界標準のクッキー有効期間:
- **7日**: 即決商品（食品、イベントチケット）
- **30日**: 標準的なEC（最も一般的な設定）
- **60〜90日**: 考察が必要な商品（家具、電子機器、B2B商品）
- **365日**: 長い販売サイクル（高級品、高額サービス）

顧客が1月1日にアフィリエイトリンクをクリックし、クッキー有効期間が30日である場合、1月30日までに購入すればそのアフィリエイトに報酬が支払われます。1月31日以降の購入はクッキーが期限切れのため、コミッションは生成されません。

### 最後のクリック帰属モデル

Spwigは **最後のクリック帰属** モデルを使用します: 最後のクリックが勝ちます。以下のように動作します:

**シナリオ**: 顧客が月曜日にアフィリエイトAのリンクをクリックし、水曜日にアフィリエイトBのリンクをクリックし、金曜日に購入します。

**結果**: アフィリエイトBがコミッションを獲得します。なぜなら、彼らのリンクが最後のクリックだったからです。

最後のクリッククッキーは、以前のアフィリエイトクッキーを上書きします。このモデルは理解しやすく、二重コミッションを防ぐことができますが、注文ごとに1人のアフィリエイト（購入前の最後の1人）だけが報酬を獲得することになります。

## クリック記録

Spwigは、すべてのアフィリエイトリンクのクリックを記録し、あなたとアフィリエイトの両方の分析を提供します。クリックデータは、リンクのパフォーマンスを測定し、不正を検出し、プロモーション戦略を最適化するのに役立ちます。

### クリックごとに記録されるデータ

**アフィリエイト > クリック** に移動して、すべての記録されたクリックを表示してください。各行には以下が含まれます:

| フィールド | 説明 |
|----------|------|
| **リンク** | どのトラッキングリンクがクリックされたか |
| **アフィリエイト** | リンクの作成者 |
| **IPアドレス** | 顧客のIP（不正検出用） |
| **User Agent** | ブラウザとデバイス情報 |
| **リファラ** | 顧客がリンクをクリックしたページ（例: "https://instagram.com"） |
| **セッションID** | このブラウジングセッションのユニークな識別子 |
| **タイムスタンプ** | クリックの正確な日時 |

### レート制限

クリック不正やボットの悪用を防ぐために、Spwigは **1分あたり1IPアドレスあたり100クリック** に制限します。同じIPがこのしきい値を超えると、追加のクリックは無視され、クリックカウントは増加しません。

この保護は、悪意のある人物がクリック統計を虚偽でインフレさせることを防ぎつつ、正当なトラフィックをブロックしないように設計されています。実際の顧客は、1分あたり100クリックを超えることはほとんどありません。

### プライバシーに関する考慮事項

クリックデータには、不正検出のためにIPアドレスとUser Agentが含まれています。プライバシーポリシーで、アフィリエイトの紹介を追跡し、アフィリエイトに匿名化されたパフォーマンスデータを共有していることを明記してください。

## アフィリエイトリンクの閲覧

すべてのアフィリエイトが生成したトラッキングリンクは、管理パネルで監視および管理できます。

### リンクリストへのアクセス

**アフィリエイト > リンク** に移動して、すべてのアフィリエイトとプログラムのトラッキングリンクを表示します。リストビューでは以下が表示されます:

- **リンクコード**: 一意の12文字の識別子
- **アフィリエイト**: リンクを作成した人
- **プログラム**: どのコミッション構造が適用されるか
- **ラベル**: オプションのアフィリエイト提供の説明
- **目的地**: リンクが顧客をリダイレクトする場所
- **総クリック数**: すべてのクリック数
- **アクティブ状態**: リンクが現在トラッキングしているかどうか

### リンクのフィルタリング

管理パネルのフィルタを使用してリストを絞り込みます:
- **アフィリエイトごと**: 特定のパートナーのすべてのリンクを表示
- **プログラムごと**: 特定のコミッション構造を促進するリンクを表示
- **アクティブ状態ごと**: 非アクティブなリンクを検索

このフィルタリングは、アフィリエイトネットワーク全体のリンク配分を分析し、最高のパフォーマンスを発揮するリンクを特定するのに役立ちます。

## リンク統計

各トラッキングリンクは、アフィリエイトがプロモーション戦略を最適化し、あなたが最高のパフォーマンスを発揮するパートナーを特定するのに役立つパフォーマンスメトリクスを蓄積します。

### リンクレコードをクリックして詳細な統計を表示:

| メトリク | 説明 | 計算方法 |
|--------|------|----------|
| **総クリック数** | リンク作成以来のすべての記録されたクリック | クリックレコードの数 |
| **クリック数（7日間）** | 最近の活動の指標 | 過去7日間のクリック数 |
| **コンバージョン** | このリンクに帰属された注文 | このリンクコードからのコミッション数 |
| **コンバージョン率** | クリックから購入に至った割合 | (コンバージョン ÷ 総クリック数) × 100 |
| **総売上高** | このリンクから得られたすべての注文の合計価値 | 転換されたクリックの注文合計 |

### 統計を用いた最適化

**アフィリエイト向け**: これらの数字は、どのプロモーションチャネルが最も効果的かを示します。インスタグラムのプロフィールリンクが5%のコンバージョン率で、ブログ投稿リンクが15%のコンバージョン率であれば、アフィリエイトはブログコンテンツに焦点を当てるべきです。

**販売者向け**: リンク統計は、どのアフィリエイトが質の高いトラフィックをもたらしているかを明らかにします。クリック数が高くてコンバージョン率が低い場合は、そのアフィリエイトの対象者があなたの商品に合っていない可能性があります。

## リンクの管理

メンテナンスやトラブルシューティングのために、管理パネルからアフィリエイトリンクを管理できます。

### リンクの非アクティブ化

特定のリンクが新しいクリックをトラッキングしないようにしながら、履歴データを保持するには:

1. **アフィリエイト > リンク** に移動します
2. 非アクティブ化したいリンクをクリックします
3. **アクティブ**を外します
4. **保存**をクリックします

非アクティブなリンクは、顧客を目的地にリダイレクトしますが、トラッキングクッキーを設定したり、クリックを記録したりしません。これは、アフィリエイトが一時的なキャンペーンを実施している場合や、特定のプロモーションチャネルを無効にする必要がある場合に役立ちます。

### リンク詳細の編集

以下を変更できます:
- **ラベル**: アフィリエイトが提供した説明を更新
- **目的地**: リンクがリダイレクトする場所を変更（商品ページを移動した場合に便利）
- **アクティブ状態**: トレーキングを有効または無効にする

リンクコードは編集できません——これは永久的であり、すべての履歴クリックとコミッションデータに結びついています。

### 使用していないリンクの削除

使用していないリンクで、履歴クリックやコンバージョンが一切ないものを削除します。これにより、リンクリストを清潔に保つことができますが、貴重な分析データは失いません。

**警告**: リンクを削除すると、関連するすべてのクリックレコードも削除されます。クリック数がゼロであるリンク、または履歴データが必要でないことを絶対に確信している場合にのみ削除してください。

## 帰属モデル

Spwigの帰属ロジックを理解することで、アフィリエイトとの期待値を設定し、コミッションの紛争をトラブルシューティングできます。

### 最後のクリック帰属

先に述べたように、Spwigは最後のクリック帰属モデルを使用します: 顧客が購入する前に複数のアフィリエイトリンクをクリックした場合、最後のクリックしたアフィリエイトだけがコミッションを獲得します。

**利点**:
- 簡単に理解しやすく説明できます
- 二重コミッションを防ぎます
- 販売をクローズしたアフィリエイトに報酬を支払います

**欠点**:
- 顧客を紹介した初期のアフィリエイトは報酬がもらえません
- マルチタッチの顧客旅程を反映していません
- 他の誰かに紹介された高意図の顧客を狙う「リンクハイジャック」を奨励する可能性があります

### クッキー有効期間が資格を決定する

クッキー有効期間内でのみ購入がコミッションを生成します。クッキーがチェックアウト前に期限切れになると、顧客がブックマークを通じて戻ってきてもコミッションは生成されません。

**例**: 30日間のクッキー有効期間
- 顧客が1月1日にリンクをクリック → クッキーが設定され、1月31日に期限切れ
- 顧客が1月25日に購入 → コミッションが生成される
- 顧客が2月5日に購入 → コミッションなし（クッキーが期限切れ）

### セッション追跡

クッキーに加えて、Spwigは各クリックのセッションIDを追跡します。これにより、クッキーがブロックされたりクリアされたりした場合でも、同じセッション内での複数回の訪問の帰属が可能になります。

顧客がリンクをクリックし、ストアを閲覧して複数のページを読み込み、その後購入する——すべて同じセッション内——アフィリエイトは、持続的なクッキーがなくても報酬を得ることができます。

## トラブルシューティング

一般的なトラッキングの問題とその解決方法:

### リンクがクリックをトラッキングしない

**症状**: クリック数がゼロのまま、アフィリエイトがリンクを共有したにもかかわらず。

**原因と解決策**:
1. **リンクが非アクティブ**: リンク詳細ページで **アクティブ** ステータスを確認してください
2. **プログラムが非アクティブ**: **アフィリエイト > プログラム** に移動し、プログラムのステータスが **アクティブ** であることを確認してください
3. **アフィリエイトアカウントが非アクティブ**: **アフィリエイト > アフィリエイト** でアフィリエイトのアカウントステータスを確認してください
4. **レート制限**: 同じIPが過剰なクリックを生成しているかを確認してください（ボットトラフィック）

### コンバージョン率が低い

**症状**: クリック数は高いが、帰属された注文は非常に少ない。

**原因と解決策**:
1. **クッキー有効期間が短すぎる**: 商品が研究や検討を必要とする場合、プログラムのクッキー有効期間を延長してください
2. **目的地ページの品質**: ランディングページを確認してください——モバイル対応していますか？読み込みは速いですか？商品は在庫がありますか？
3. **対象者の不一致**: アフィリエイトの対象者があなたの商品に合っていない可能性があります
4. **ブラウザがクッキーをブロック**: 一部のプライバシーツールがサードパーティのクッキーをブロックしますが、Spwigはファーストパーティのクッキーを使用しており、これによりブロックされる可能性は低いです

### 重複したクリックレコード

**症状**: 同じ顧客が短時間で複数のクリックレコードを生成します。

**原因**: これは正常な動作です。トラッキングリンクの各ページ読み込みがクリックレコードを作成します。顧客がリンクをクリックし、ページが遅く読み込まれ、再びクリックした場合、複数のレコードが表示されます。

**解決策**: 何もする必要はありません。レート制限により、悪意のある行為（1分あたり1IPあたり100クリック）が防止され、同じセッションからの重複クリックは帰属に影響を与えません——1つのクッキーのみが設定されます。

## ヒント

- **リリース前にはトラッキングをテストしてください** — テストアフィリエイトアカウントを作成し、トラッキングリンクを生成し、インコグニトブラウザでクリックし、テスト購入を完了してください。正しいアフィリエイト帰属とともにコミッションが表示されることを確認してください。
- **アフィリエイトにクッキー有効期間を教育してください** — アフィリエイトがクッキー有効期間内でのみコミッションを獲得できるようにすることを確認してください。これにより、彼らは現実的な期待値を設定し、高意図のトラフィックに焦点を当てることができます。
- **クリックパターンを監視して不正を検出してください** — 1つのIPから異常に高いクリック数、またはユーザーエージェント文字列がないクリックは、ボットトラフィックを示している可能性があります。コミッションを承認する前に、これらのアフィリエイトを慎重に確認してください。
- **リンクラベルを一貫して使用してください** — アフィリエイトにリンクをチャネル（Instagram、ブログ、メール）ごとにラベルを付けることを奨励してください。これにより、どちらもどのプロモーションチャネルが最も効果的なコンバージョンをもたらすかを分析できます。
- **高額商品には長いクッキー有効期間を考慮してください** — 平均注文価値が高くて、顧客が購入前に研究を必要とする場合、クッキー有効期間を60〜90日に延長して、遅延したコンバージョンをキャッチしてください。
- **リファラデータを用いてチャネルの洞察を得てください** — リファラフィールドは、クリックがどの場所から来たかを示します。"instagram.com"や"youtube.com"から多くのクリックがある場合、どのソーシャルプラットフォームがアフィリエイトによって最も効果的に使用されているかがわかります。
