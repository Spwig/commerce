---
title: 联盟计划
---

联盟计划允许您招募推广您产品的合作伙伴，并根据他们带来的销售获得佣金。联盟成员分享独特的推荐链接，Spwig会自动跟踪点击、归因订单并计算佣金。

![联盟计划](/static/core/admin/img/help/affiliate-program/program-list.webp)

## 如何运作

1. 您创建一个或多个**联盟计划**，设置佣金率和规则
2. 联盟成员通过公共门户**注册**，或手动添加
3. 每个联盟成员获得一个带有跟踪代码的**唯一推荐链接**
4. 当客户点击链接并下单时，将**记录佣金**
5. 您审核并批准佣金，然后处理**付款**

## 创建计划

导航到 **营销 > 联盟计划**，点击 **添加计划**。

### 计划设置

| 设置 | 描述 |
|---------|-------------|
| **名称** | 联盟成员可见的计划名称（例如："合作伙伴计划"） |
| **佣金类型** | 订单总额的**百分比**或每笔销售的**固定金额** |
| **佣金率** | 联盟成员获得的百分比或固定金额 |
| **Cookie有效期** | 推荐跟踪Cookie的持续时间（默认：30天） |
| **最低付款额** | 联盟成员可申请付款的最低收益 |
| **自动批准联盟成员** | 自动接受新联盟成员申请，或需要手动批准 |
| **状态** | 启用、暂停或关闭 |

### 佣金类型

- **百分比** — 联盟成员获得每笔推荐订单子总额的百分比（例如：100美元订单的10% = 10美元佣金）
- **固定** — 联盟成员每笔销售获得固定金额，与订单价值无关（例如：每笔销售5美元）

## 管理联盟成员

导航到 **营销 > 联盟成员** 以查看和管理联盟成员账户。

### 联盟成员详情

每个联盟成员有：
- **联盟代码** — 在推荐链接中使用的唯一代码（自动生成或自定义）
- **推荐链接** — 联盟成员分享的完整跟踪URL（例如：`yourstore.com/?ref=CODE`）
- **状态** — 等待中、已批准或已拒绝
- **付款方式** — 联盟成员如何接收付款（PayPal或银行转账）
- **计划成员资格** — 联盟成员所属的计划

### 手动添加联盟成员

1. 点击 **添加联盟成员**
2. 选择现有客户账户或创建新账户
3. 将联盟成员分配给一个或多个计划
4. 设置联盟代码（或留空以自动生成）

### 联盟成员门户

联盟成员可以访问一个面向公众的门户，他们可以在其中：
- 查看包含收入和点击统计信息的仪表板
- 复制他们的推荐链接
- 跟踪佣金历史记录
- 申请付款

门户URL在您的商店中自动可用，地址为 `/affiliate/`。

## 跟踪和佣金

### 跟踪机制

1. 客户点击联盟成员的推荐链接
2. 在客户的浏览器中设置跟踪Cookie（持续时间根据配置的Cookie有效期）
3. 如果客户在Cookie有效期内下单，订单将归因于该联盟成员
4. 创建佣金记录，状态为 **待处理**

### 佣金状态

| 状态 | 描述 |
|--------|-------------|
| **待处理** | 已记录佣金，等待审核 |
| **已批准** | 已验证并准备付款 |
| **已拒绝** | 佣金被拒绝（例如，欺诈订单或退货商品） |
| **已付款** | 佣金包含在已完成的付款中 |

### 审核佣金

导航到 **营销 > 佣金** 以审核待处理佣金：

1. 检查订单详情以确认销售是否合法
2. 点击 **批准** 以确认，或点击 **拒绝** 并输入原因
3. 已批准的佣金将累积到联盟成员的付款余额中

## 付款

当联盟成员的已批准佣金余额达到最低付款门槛时，您可以处理付款。

### 处理付款

1. 导航到 **营销 > 付款**
2. 选择有可用余额的联盟成员
3. 选择付款方式：
   - **PayPal** — 直接将资金发送到联盟成员的PayPal邮箱
   - **银行转账** — 记录手动银行转账
4. 确认并处理付款
5. 付款状态更新为 **已完成**，佣金标记为 **已付款**

### 付款服务提供商

Spwig与付款服务提供商集成以实现自动化付款：
- **PayPal** — 通过PayPal API进行自动化批量付款
- **Airwallex** — 国际付款，具有有竞争力的汇率
- **手动** — 记录在Spwig外部处理的付款

## 推荐链接

每个联盟成员的推荐链接遵循以下模式：

```
https://yourstore.com/?ref=AFFILIATE_CODE
```

联盟成员还可以创建指向特定产品或类别的链接：

```
https://yourstore.com/products/shoe-name/?ref=AFFILIATE_CODE
```

`ref`参数适用于任何页面 — 无论着陆页面是什么，都会设置跟踪Cookie。

## 计划分析

联盟计划仪表板显示：
- **总点击次数** — 推荐链接被点击的次数
- **总订单数** — 归因于联盟成员的订单
- **总佣金** — 所有佣金的总和（包括待处理、已批准和已付款）
- **活跃联盟成员** — 当前正在生成推荐的已批准联盟成员数量

## 小贴士

- 从**基于百分比的佣金**（5–15%）开始 — 它随着订单价值自然增长，且易于联盟成员理解。
- 将**30天Cookie有效期**作为基准 — 这为客户提供时间返回并完成购买，同时仍将销售归因于联盟成员。
- 对于公共计划，启用**自动批准**以减少摩擦，或对于邀请制计划，使用手动批准以审核每个联盟成员。
- 设置合理的**最低付款额**（例如：25–50美元）以避免处理大量小额交易。
- 自定义**联盟成员门户**以匹配您的品牌 — 当体验感觉专业时，联盟成员更有可能推广您的商店。
- 定期监控佣金，以发现**欺诈模式**，如自我推荐、异常高的退货率或可疑的点击量。
