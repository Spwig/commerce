---
title: アフィリエイトプログラム
---

アフィリエイトプログラムを使用すると、商品のプロモーションを行うパートナーを募集し、彼らが生成する売上の手数料を獲得できます。アフィリエイターはユニークな紹介リンクを共有し、Spwigはクリックを自動的に追跡し、注文を属性付けし、手数料を計算します。

![アフィリエイトプログラム](/static/core/admin/img/help/affiliate-program/program-list.webp)

## 仕組み

1. 手数料率とルールをもとに、1つ以上の**アフィリエイトプログラム**を作成します
2. アフィリエイターは、パブリックポータルを通じて**登録**するか、手動で追加されます
3. 各アフィリエイターは、トラッキングコード付きの**ユニークな紹介リンク**を受け取ります
4. 顧客がリンクをクリックして購入すると、**手数料**が記録されます
5. 手数料を確認し、承認した後、**支払い**を処理します

## プログラムの作成

**マーケティング > アフィリエイトプログラム**に移動し、**プログラムの追加**をクリックします。

### プログラム設定

| 設定 | 説明 |
|---------|-------------|
| **名前** | アフィリエイターに表示されるプログラム名（例: "パートナープログラム"） |
| **手数料タイプ** | 注文合計の**パーセンテージ**または**固定**金額ごとに売上 |
| **手数料率** | アフィリエイターが獲得するパーセンテージまたは固定金額 |
| **クッキーの有効期限** | 紹介リンクのトラッキングクッキーが保持される日数（デフォルト: 30日） |
| **最低支払い額** | アフィリエイターが支払いを要求できる前の最低収入 |
| **アフィリエイターの自動承認** | 新しいアフィリエイターの申請を自動的に承認するか、手動承認が必要か |
| **ステータス** | 有効、一時停止、または終了 |

### 手数料タイプ

- **パーセンテージ** — アフィリエイターは、紹介した注文のサブトータルのパーセンテージを獲得します（例: 100ドルの注文の10％ = 10ドルの手数料）
- **固定** — 注文の金額に関係なく、アフィリエイターは1件あたりの固定金額を獲得します（例: 1件あたり5ドル）

## アフィリエイターの管理

**マーケティング > アフィリエイター**に移動して、アフィリエイターのアカウントを表示および管理します。

### アフィリエイターの詳細

各アフィリエイターには以下があります：
- **アフィリエイターコード** — 紹介URLで使用されるユニークなコード（自動生成またはカスタム）
- **紹介リンク** — アフィリエイターが共有する完全なトラッキングURL（例: `yourstore.com/?ref=CODE`）
- **ステータス** — 保留中、承認済み、または拒否済み
- **支払い方法** — アフィリエイターが支払いを受け取る方法（PayPalまたは銀行振込）
- **プログラムメンバーシップ** — アフィリエイターが所属するプログラム

### 手動でアフィリエイターを追加する

1. **アフィリエイターの追加**をクリックします
2. 既存の顧客アカウントを選択するか、新しいアカウントを作成します
3. アフィリエイターを1つ以上のプログラムに割り当てます
4. アフィリエイターコードを設定するか、空白にして自動生成させます

### アフィリエイターのポータル

アフィリエイターは、パブリックなポータルにアクセスし、以下を行えます：
- 紹介リンクのクリック数や収益の統計を確認するダッシュボード
- 紹介リンクをコピーする
- 手数料履歴を追跡する
- 支払いを要求する

ポータルURLは、ストアの`/affiliate/`に自動的に表示されます。

## トレーキングと手数料

### トレーキングの仕組み

1. 顧客がアフィリエイターの紹介リンクをクリックします
2. 顧客のブラウザに、設定されたクッキー有効期限に応じたトレーキングクッキーが設定されます
3. 顧客がクッキー有効期限内に注文を確定した場合、その注文はアフィリエイターに帰属されます
4. 手数料の記録が作成され、ステータスは**保留中**になります

### 手数料ステータス

| ステータス | 説明 |
|--------|-------------|
| **保留中** | 手数料が記録され、確認待ちです |
| **承認済み** | 検証済みで、支払い準備ができています |
| **拒否** | 手数料が拒否されました（例: 詐欺的な注文または返品された商品） |
| **支払い済み** | 手数料が完了した支払いに含まれています |

### 手数料の確認

**マーケティング > 手数料**に移動して、保留中の手数料を確認します：

1. 注文の詳細を確認して、売上が正当であることを確認します
2. **承認**をクリックして確認するか、**拒否**と理由を入力します
3. 承認された手数料は、アフィリエイターの支払い残高に集積されます

## 支払い

アフィリエイターの承認された手数料残高が最低支払い額のしきい値に達すると、支払いを処理できます。

### 支払いの処理

1. **マーケティング > 支払い**に移動します
2. 利用可能な残高を持つアフィリエイターを選択します
3. 支払い方法を選択します：
   - **PayPal** — アフィリエイターのPayPalメールアドレスに直接資金を送金します
   - **銀行振込** — 手動の銀行振込を記録します
4. 支払いを確認し、処理します
5. 支払いのステータスは**完了**に更新され、手数料は**支払い済み**としてマークされます

### 支払いプロバイダ

Spwigは、支払いプロバイダと統合して自動支払いを処理します：
- **PayPal** — PayPal APIを通じた自動的な大規模支払い
- **Airwallex** — 競争的な為替レートで国際的な支払いを処理
- **手動** — Spwigの外で処理された支払いを記録

## 紹介リンク

各アフィリエイターの紹介リンクは次のパターンに従います：

```
https://yourstore.com/?ref=AFFILIATE_CODE
```

アフィリエイターは、特定の商品やカテゴリへのリンクも作成できます：

```
https://yourstore.com/products/shoe-name/?ref=AFFILIATE_CODE
```

`ref`パラメータはどのページでも動作します — ランディングページに関係なく、トラッキングクッキーが設定されます。

## プログラム分析

アフィリエイトプログラムダッシュボードでは以下が表示されます：
- **総クリック数** — 紹介リンクがクリックされた回数
- **総注文数** — アフィリエイターに帰属された注文数
- **総手数料** — すべての手数料の合計（保留中、承認済み、支払い済み）
- **アクティブなアフィリエイター** — 現在紹介リンクを生成している承認済みアフィリエイターの数

## ヒント

- 最初は**パーセンテージベースの手数料**（5〜15％）から始めましょう — 注文価格に自然にスケールし、アフィリエイターにとって理解しやすいです。
- **30日間のクッキー有効期限**を基準値として設定してください — これにより、顧客が戻って購入を完了する時間があり、それでも販売はアフィリエイターに帰属します。
- 公開プログラムでは**自動承認**を有効にして摩擦を減らすか、招待制のプログラムではアフィリエイターを事前に確認するために**手動承認**を使用してください。
- **最低支払い額**（例: 25〜50ドル）を適切に設定して、多くの小さな取引を処理するのを避けてください。
- **アフィリエイターのポータル**をブランドに合わせてカスタマイズしてください — アフィリエイターがプロモーションをしやすいように、体験がプロフェッショナルに感じるようにしてください。
- **自己紹介や異常に高い返品率、または疑わしいクリック量**などの詐欺パターンを定期的に監視してください。
