---
title: 커미션 관리
---

커미션 관리는 파트너의 수익을 검토하고 승인하여 정당한 판매만 신용되도록 하는 과정입니다. 이 가이드는 대기 중인 커미션을 검토하고, 유효한 커미션을 승인하며, 사기 또는 반품 주문을 거부하고, 대량 작업을 사용하여 커미션을 효율적으로 관리하는 방법을 보여줍니다.

## 커미션 대시보드

**마케팅 > 커미션**으로 이동하여 커미션 관리 대시보드에 액세스합니다.

대시보드는 모든 파트너 프로그램에 걸쳐 커미션 활동의 개요를 제공합니다:

| 통계 | 설명 |
|-----------|-------------|
| **대기 중인 커미션** | 검토를 기다리는 커미션 수 |
| **승인된 커미션** | 지급 준비가 된 확인된 커미션 |
| **지급된 커미션** | 파트너에게 지급된 커미션 |
| **거부된 커미션** | 사기, 반품 또는 정책 위반으로 인해 거부된 커미션 |
| **지급 대기 금액** | 승인되었지만 지급되지 않은 커미션의 총 가치 |

이 통계는 파트너 프로그램의 재정적 영향을 추적하고 검토 작업량을 모니터링하는 데 도움이 됩니다.

![커미션 대시보드](/static/core/admin/img/help/commission-management/commission-dashboard.webp)

## 커미션 보기

커미션 목록은 모든 커미션 기록을 시간 순서대로 표시합니다.

### 목록 열

| 열 | 설명 |
|--------|-------------|
| **파트너** | 파트너의 이름과 고유 코드 |
| **프로그램** | 이 커미션을 생성한 파트너 프로그램 |
| **주문** | 주문 번호 (클릭하여 전체 주문 세부 정보를 확인) |
| **금액** | 귀하의 상점 통화로 표시된 커미션 값 |
| **상태** | 대기 중, 승인됨, 거부됨 또는 지급됨 |
| **생성됨** | 커미션이 생성된 시점 |

### 커미션 필터링

필터 사이드바를 사용하여 커미션을 좁혀보세요:

- **상태별** — 대기 중, 승인됨, 거부됨 또는 지급된 커미션만 표시
- **파트너별** — 특정 파트너의 커미션을 확인
- **프로그램별** — 특정 파트너 프로그램의 커미션을 확인
- **날짜 범위별** — 생성 날짜로 필터링

### 커미션 검색

검색 바를 사용하여 특정 커미션을 찾을 수 있습니다:

- **주문 번호**를 입력하여 특정 판매의 커미션을 찾습니다
- **파트너 코드**를 입력하여 한 파트너의 모든 커미션을 확인합니다

## 커미션 세부 정보

목록에서 커미션을 클릭하여 전체 세부 정보를 확인합니다.

### 세부 필드

세부 정보 뷰는 다음과 같은 정보를 표시합니다:

- **주문 정보** — 주문 번호를 클릭하여 새 탭에서 전체 주문을 확인합니다. 이는 항목, 배송 주소, 결제 상태 및 고객 세부 정보를 포함합니다.
- **파트너 정보** — 파트너의 이름, 코드, 결제 이메일 및 프로그램 가입 상태
- **프로그램 세부 정보** — 프로그램 이름, 커미션 유형(백분율 또는 고정), 커미션 비율
- **타임스탬프** — 생성 날짜, 승인/거부 날짜 및 지급 날짜
- **메모 섹션** — 상인만 볼 수 있는 내부 메모(아래에서 설명됨)

이 정보는 커미션을 승인하기 전에 그 유효성을 확인하는 데 도움이 됩니다.

## 커미션 승인

커미션을 승인하면 유효하다고 확인하고 파트너의 사용 가능한 잔액에 추가하여 지급 대상이 되도록 합니다.

### 승인 시기

다음과 같은 경우 커미션을 승인하세요:

- **주문이 성공적으로 완료됨** — 제품이 배송되었거나 디지털 상품이 제공됨
- **반품 또는 환불 없음** — 고객이 반품을 요청하지 않았음 (배송 후 14-30일 기다리는 것이 좋음)
- **품질 기준 충족** — 판매가 프로그램의 조건을 충족함 (예: 자가 추천이 아님, 고객이 진짜 결제 방법을 사용함)
- **사기 감지 없음** — 주문이 사기 검사를 통과함 (IP, 청구/배송 주소 불일치, 비정상적인 주문 패턴 확인)

### 승인 방법

**단일 커미션 승인:**

1. **마케팅 > 커미션**으로 이동합니다
2. 승인하려는 커미션을 클릭합니다
3. 세부 정보 페이지 상단의 **승인** 버튼을 클릭합니다
4. 선택적으로 메모를 추가합니다 (예: "배송 완료 후 승인")
5. 상태가 **승인됨**으로 변경되고 커미션이 파트너의 잔액에 추가됩니다

**대량 승인:**

1. **마케팅 > 커미션**으로 이동합니다
2. 승인하려는 커미션 옆의 상자에 체크합니다
3. **Actions** 드롭다운에서 **선택한 항목 승인**을 선택합니다
4. **Go**를 클릭합니다
5. 선택한 모든 커미션의 상태가 **승인됨**으로 변경됩니다

승인된 커미션은 파트너 대시보드에서 사용 가능한 잔액으로 표시되고 다음 지급 배치에 포함될 수 있습니다.

## 커미션 거부

커미션을 거부하면 파트너의 잔액에서 제거하고 지급 대상이 되지 않도록 표시합니다.

### 거부 시기

다음과 같은 경우 커미션을 거부하세요:

- **사기 주문** — 주문이 사기 징후를 보임 (도난된 결제 방법, IP 불일치, 파트너가 자신의 링크를 사용함)
- **고객이 제품을 반품함** — 고객이 전체 환불을 요청함
- **품질 문제** — 판매가 프로그램 조건을 충족하지 않음 (예: 파트너가 광고 가이드라인을 위반함)
- **조건 위반** — 파트너가 금지된 홍보 방법을 사용함 (스팸, 상표 경매, 쿠키 채워넣기)
- **주문 취소** — 고객이 배송 전에 취소함

### 거부 방법

**단일 커미션 거부:**

1. **마케팅 > 커미션**으로 이동합니다
2. 거부하려는 커미션을 클릭합니다
3. 세부 정보 페이지 상단의 **거부** 버튼을 클릭합니다
4. **메모 추가** — 거부 이유를 설명하는 메모를 추가하세요 (분쟁 해결을 위해 권장됨)
5. 상태가 **거부됨**으로 변경됩니다

**대량 거부:**

1. **마케팅 > 커미션**으로 이동합니다
2. 거부하려는 커미션 옆의 상자에 체크합니다
3. **Actions** 드롭다운에서 **선택한 항목 거부**를 선택합니다
4. **Go**를 클릭합니다
5. 선택한 모든 커미션의 상태가 **거부됨**으로 변경됩니다

거부된 커미션은 파트너의 잔액에서 제거되고 지급될 수 없습니다. 이들은 커미션 기록에서 보관되어 기록을 위해 남아 있습니다.

## 대량 작업

대량 작업은 여러 커미션을 한 번에 승인하거나 거부할 수 있도록 하여 대량 배치를 처리할 때 시간을 절약할 수 있습니다.

### 대량 작업 사용

1. **마케팅 > 커미션**으로 이동합니다
2. 처리하려는 커미션만 표시하도록 목록을 필터링합니다 (예: 상태 **대기 중**으로 필터링)
3. 각 커미션 옆의 상자에 체크하거나, 현재 페이지의 헤더 상자에 체크하여 모든 항목을 선택합니다
4. **Actions** 드롭다운에서 작업을 선택합니다:
   - **선택한 항목 승인** — 선택한 모든 커미션을 승인 상태로 표시
   - **선택한 항목 거부** — 선택한 모든 커미션을 거부 상태로 표시
5. **Go**를 클릭합니다
6. 업데이트된 커미션 수를 표시하는 확인 메시지를 확인합니다

### 효율적인 대량 처리

- **프로그램별로 필터링** — 신뢰할 수 있는 고성과 파트너의 모든 커미션을 한 번에 승인
- **날짜 범위별로 필터링** — 14일 이상 된 커미션 처리 (반품 기간을 지나감)
- **고가 커미션 별도 검토** — 소액 커미션에 대량 작업을 사용하고, 고가 커미션은 수동으로 검토

## 커미션 메모

메모 필드는 결정을 기록하고 팀과 소통할 수 있도록 합니다.

### 메모 추가

메모는 다음과 같이 추가할 수 있습니다:

- **승인 시** — 커미션을 클릭하고, 메모 필드에 메모를 추가한 후 **승인**을 클릭합니다
- **거부 시** — 거부 이유를 설명하는 메모를 추가합니다
- **언제든지** — 커미션을 클릭하고, 메모 필드에 메모를 추가하거나 편집한 후 저장합니다

### 메모 사용 시기

- **거부된 커미션** — 항상 이유를 기록합니다 ("고객이 2/10/26에 주문 #12345를 반품함")
- **고가 커미션** — 확인 단계를 기록합니다 ("추적 번호 #ABC123을 통해 배송 확인함")
- **분쟁 커미션** — 파트너와의 소통을 기록합니다
- **사기 패턴** — 향후 참고를 위해 의심스러운 활동을 기록합니다

메모는 **내부 전용**입니다 — 파트너는 이를 볼 수 없습니다. 이는 귀하의 기록 도구로 사용됩니다.

## 커미션 흐름

이것은 전체 커미션 관리 워크플로입니다:

```
주문 완료 → 커미션 생성 (대기 중)
                      ↓
              상인 검토
                      ↓
                ┌─────┴─────┐
                ↓           ↓
            승인됨        거부됨
                ↓           ↓
        지급 대기        지급 불가
                ↓
        지급 포함
                ↓
              지급됨
```

**시간표 예시:**

- **1일차:** 고객이 파트너 링크를 통해 $100 주문을 완료함 → $10 커미션 생성 (대기 중)
- **15일차:** 주문 완료 및 반품 기간 종료 → 상인 커미션 승인
- **20일차:** 상인이 월별 지급 배치를 처리함 → 커미션 상태가 지급됨으로 변경됨
- **21일차:** 파트너가 PayPal을 통해 지급을 받음

## 최선의 실천 방법

### 검토 기간

일관된 검토 일정을 수립하세요:

- **매일 검토** — 매일 아침 대기 중인 커미션을 처리하세요 (고부하 프로그램에 권장됨)
- **주간 검토** — 매주 월요일에 이전 주의 커미션을 승인하는 시간을 할당하세요
- **2주간 검토** — 지급 일정과 맞추세요 (중간에 커미션 승인, 월말에 지급 처리)

### 품질 보증 검사

커미션을 승인하기 전에 확인하세요:

1. **주문이 완료됨** — 관리자에서 주문 상태 확인
2. **결제가 확인됨** — 결제 방법이 성공적으로 처리되었는지 확인
3. **반품 기간 종료** — 배송 후 14-30일 기다리세요 (반품을 고려함)
4. **사기 플래그 없음** — 주문을 의심스러운 패턴(주소 불일치, 고위험 국가, 동일한 IP에서의 다중 주문)을 확인하세요
5. **좋은 상태의 파트너** — 파트너의 이력을 확인하여 이전 사기 또는 위반 사항이 있는지 확인하세요

### 사기 예방

다음과 같은 빨간 신호를 주의하세요:

- **자기 추천** — 파트너가 자신의 추적 링크를 사용하여 주문을 배치함
- **쿠키 채워넣기** — 비정상적으로 높은 클릭 대 전환 비율과 낮은 주문 금액
- **중복 주문** — 동일한 고객/IP가 동일한 파트너 링크를 통해 여러 주문을 생성함
- **지리적 위치 불일치** — 국가 A에 소재한 파트너가 국가 B에서만 판매를 유도함
- **환불** — 파트너 추천 주문의 높은 환불 비율

사기를 감지하면 **커미션을 거부**하고 파트너 프로그램 가입을 종료하는 것을 고려하세요.

### 파트너와의 커뮤니케이션

- **기대 설정** — 프로그램 조건에 명확하게 기록된 커미션 승인 정책을 설정하세요
- **투명성 유지** — 커미션을 거부할 경우 파트너에게 이유를 설명하는 이메일을 보내세요 (메모를 참조하세요)
- **분쟁 대응** — 파트너가 거부에 대해 질문할 경우 메모와 주문 세부 정보를 검토하세요
- **가이드라인 게시** — 파트너 포털에 "커미션 승인 정책" 페이지를 생성하여 혼란을 피하세요

## 팁

- **반품 기간 종료 후** 커미션을 승인하세요 (보통 14-30일) — 이는 고객이 나중에 반품하는 주문을 승인하는 것을 피합니다
- **필터를 사용한 대량 작업**을 통해 신뢰할 수 있는 파트너의 커미션을 효율적으로 처리하고, 새로운 또는 고위험 파트너는 수동으로 검토하세요
