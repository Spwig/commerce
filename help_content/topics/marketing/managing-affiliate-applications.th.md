---
title: การจัดการการสมัครเป็นพันธมิตร
---

เมื่อคู่ค้าที่เป็นไปได้สมัครเข้าร่วมโปรแกรมพันธมิตรของคุณ พวกเขาจะปรากฏในคิวการสมัครของคุณรอการตรวจสอบ การคู่มือนี้จะแสดงวิธีการประเมิน อนุมัติ และปฏิเสธการสมัครของพันธมิตรเพื่อสร้างเครือข่ายคู่ค้าคุณภาพที่สอดคล้องกับแบรนด์ของคุณ

การจัดการการสมัครอย่างระมัดระวังช่วยให้คุณทำงานร่วมกับพันธมิตรที่น่าเชื่อถือซึ่งจะนำเสนอร้านค้าของคุณอย่างมืออาชีพและสร้างยอดขายที่แท้จริง

![Applications List View](/static/core/admin/img/help/managing-affiliate-applications/applications-list.webp)

## Application Sources

การสมัครพันธมิตรจะเข้ามาในคิวของคุณผ่านช่องทางต่างๆ ดังนี้:

### Public Portal Applications

การสมัครส่วนใหญ่มาจากพอร์ทัลพันธมิตรสาธารณะที่ `/affiliate/` บนร้านค้าของคุณ เมื่อคู่ค้าที่เป็นไปได้คลิก **Become an Affiliate** พวกเขาจะกรอกแบบฟอร์มการลงทะเบียนที่สร้างบันทึกการสมัคร

### Guest vs Registered Users

หากคุณเปิดใช้งาน **Allow Guest Registration** ในการตั้งค่าพันธมิตร ผู้ใช้ที่ไม่ใช่ลูกค้าสามารถสมัครได้โดยตรง แต่ถ้าไม่เปิดใช้งาน ผู้ใช้จะต้องสร้างบัญชีลูกค้าบนร้านค้าของคุณก่อนจึงจะสามารถสมัครเป็นพันธมิตรได้

### Manual Review Requirement

เมื่อเปิดใช้งาน **Require Approval** ในการตั้งค่าพันธมิตร (แนะนำ) ทุกการสมัครจะอยู่ในสถานะ **Pending** รอให้คุณตรวจสอบ หากไม่เปิดใช้งาน การสมัครจะได้รับการอนุมัติโดยอัตโนมัติ และพันธมิตรจะได้รับการเข้าถึงแดชบอร์ดของพวกเขาทันที

## Viewing Applications

ไปที่ **Affiliate Program > Applications** (หรือ **Marketing > Affiliate Memberships** ในแดชบอร์ดผู้ดูแลระบบ) เพื่อดูการสมัครทั้งหมดในโปรแกรม

มุมมองรายการจะแสดง:

| Column | Description |
|--------|-------------|
| **Affiliate** | ชื่อและอีเมลของผู้สมัคร |
| **Program** | โปรแกรมพันธมิตรที่พวกเขาต้องการสมัคร |
| **Status** | รอการตรวจสอบ, ได้รับการอนุมัติ หรือถูกปฏิเสธ |
| **Applied Date** | วันที่พวกเขาส่งการสมัคร |
| **Payment Method** | วิธีการชำระเงินที่พวกเขาต้องการ (PayPal หรือการโอนเงินผ่านธนาคาร) |

### Filtering Applications

ใช้ตัวกรองในแดชบอร์ดผู้ดูแลระบบเพื่อจำกัดการสมัคร:

- **Status**: ดูเฉพาะการสมัครที่รอการตรวจสอบ
- **Program**: กรองตามโปรแกรมเฉพาะหากคุณมีหลายโปรแกรมพันธมิตร
- **Date Range**: ค้นหาการสมัครจากช่วงเวลาที่เฉพาะเจาะจง

### Pending Applications Badge

แถบด้านข้างของแดชบอร์ดผู้ดูแลระบบจะแสดงจำนวนการแจ้งเตือนเมื่อคุณมีการสมัครที่ยังต้องดำเนินการ

## Reviewing an Application

คลิกที่การสมัครใดๆ เพื่อดูโปรไฟล์ผู้สมัครทั้งหมด ข้อมูลรายละเอียดนี้แสดงข้อมูลทั้งหมดที่คุณจำเป็นต้องตัดสินใจอนุมัติอย่างมีข้อมูล

![Application Detail Card](/static/core/admin/img/help/managing-affiliate-applications/application-detail-card.webp)

### Affiliate Profile Information

ตรวจสอบรายละเอียดสำคัญเกี่ยวกับผู้สมัคร:

**Basic Information**
- **Email Address**: ใช้สำหรับการเข้าสู่ระบบและการสื่อสาร
- **Company/Business Name**: ชื่อองค์กรของพวกเขา (หากมี)
- **Website URL**: แพลตฟอร์มการส่งเสริมหลักของพวกเขา
- **Phone Number**: ข้อมูลการติดต่อ

**Payment Information**
- **Payment Method**: PayPal หรือการโอนเงินผ่านธนาคาร
- **PayPal Email**: จำเป็นหากพวกเขาเลือกการชำระเงินผ่าน PayPal
- **Bank Details**: จำเป็นหากพวกเขาเลือกการโอนเงินผ่านธนาคาร (หมายเลขบัญชี, หมายเลขการโอน, รหัส SWIFT)

**Promotional Channels**
การสมัครหลายรายการมีหมายเหตุเกี่ยวกับสถานที่ที่พันธมิตรวางแผนจะส่งเสริมผลิตภัณฑ์ของคุณ — บัญชีโซเชียลมีเดีย, ช่องทาง YouTube, รายการอีเมล, หรือบล็อก

### Program Details

ตรวจสอบโปรแกรมที่พวกเขาสมัครและตรวจสอบอัตราค่าคอมมิชชั่น, ระยะเวลาคุกกี้, และเกณฑ์การชำระเงินขั้นต่ำ ให้แน่ใจว่าผู้สมัครเป็นผู้เหมาะสมกับกลุ่มเป้าหมายของโปรแกรม

### Application History

หากผู้สมัครเคยถูกปฏิเสธมาก่อนหรือสมัครหลายโปรแกรม ประวัติการสมัครนี้จะปรากฏในมุมมองรายละเอียด

## Approval Criteria

ใช้รายการตรวจสอบนี้เพื่อประเมินการสมัครแต่ละรายการอย่างสม่ำเสมอ:

### Business Legitimacy

- [ ] **Active Website or Social Media**: ผู้สมัครมีแพลตฟอร์มที่ใช้งานอยู่พร้อมเนื้อหาจริงหรือไม่
- [ ] **Relevant Audience**: กลุ่มเป้าหมายของพวกเขาตรงกับกลุ่มลูกค้าเป้าหมายของคุณหรือไม่
- [ ] **Quality Content**: เนื้อหาของพวกเขาเป็นมืออาชีพ ถูกต้อง และสอดคล้องกับค่านิยมของแบรนด์ของคุณหรือไม่
- [ ] **Established Platform**: พวกเขาเป็นผู้มีผู้ติดตามที่มีส่วนร่วมหรือมีการจราจรที่มีความหมายหรือไม่

### Payment Information

- [ ] **Valid Payment Details**: พวกเขาได้ให้อีเมล PayPal ที่ใช้งานได้หรือข้อมูลบัญชีธนาคารที่สมบูรณ์หรือไม่
- [ ] **Matching Identity**: ข้อมูลการชำระเงินตรงกับชื่อธุรกิจหรือข้อมูลส่วนตัวของพวกเขาหรือไม่

### Brand Alignment

- [ ] **Appropriate Fit**: เนื้อหา โทน และสไตล์ของพวกเขาสอดคล้องกับภาพลักษณ์ของแบรนด์ของคุณหรือไม่
- [ ] **No Conflicts**: พวกเขาได้ส่งเสริมคู่แข่งโดยตรงหรือไม่
- [ ] **Professional Standards**: พวกเขาปฏิบัติตามมาตรฐานคุณภาพที่คุณรู้สึกสบายใจที่จะร่วมมือหรือไม่

### Fraud Prevention

- [ ] **No Red Flags**: ตรวจสอบสัญญาณเช่นอีเมลทั่วไป โปรไฟล์ที่ไม่สมบูรณ์ หรือรูปแบบเว็บไซต์ที่น่าสงสัย
- [ ] **No Previous Violations**: พวกเขาเคยถูกปฏิเสธมาก่อนเนื่องจากฉ้อโกงหรือการละเมิดข้อตกลงหรือไม่
- [ ] **Reasonable Expectations**: แผนการส่งเสริมที่ระบุไว้ของพวกเขาเป็นไปได้และสามารถทำได้จริงหรือไม่

หากการสมัครตรงตามเกณฑ์ทั้งหมด ให้อนุมัติมัน หากมันล้มเหลวในการตรวจสอบใดๆ ที่สำคัญ (ความเสี่ยงด้านการฉ้อโกง การไม่สอดคล้องกับแบรนด์ ข้อมูลการชำระเงินที่ไม่ถูกต้อง) ปฏิเสธมันพร้อมเหตุผลที่ชัดเจน

## Approving Applications

ปฏิบัติตามขั้นตอนต่อไปนี้เพื่ออนุมัติการสมัครหนึ่งหรือหลายรายการ:

### Single Application Approval

1. เปิดหน้ารายละเอียดการสมัคร
2. ตรวจสอบข้อมูลโปรไฟล์ทั้งหมดอย่างระมัดระวัง
3. ยืนยันว่าข้อมูลการชำระเงินสมบูรณ์และถูกต้อง
4. คลิกปุ่ม **Save and Continue Editing** หากคุณต้องการเพิ่มหมายเหตุ
5. เลือก **Approve** จากเมนูแบบเลื่อนลงของสถานะ
6. คลิก **Save**

### Bulk Approval

สำหรับการสมัครที่คุณคัดเลือกหลายรายการ:

1. ไปที่รายการการสมัครที่ **Affiliate Program > Applications**
2. ตรวจสอบกล่องที่อยู่ติดกับการสมัครที่คุณต้องการอนุมัติ
3. เลือก **Approve selected applications** จากเมนูแบบเลื่อนลง **Actions**
4. คลิก **Go**
5. ยืนยันการดำเนินการแบบกลุ่มเมื่อถูกขอ

### What Happens After Approval

เมื่อคุณอนุมัติการสมัคร:

- สถานะของพันธมิตรจะเปลี่ยนเป็น **Approved**
- พวกเขาจะได้รับการแจ้งเตือนทางอีเมล (หากมีการตั้งค่าเทมเพลตอีเมล)
- พวกเขาจะได้รับการเข้าถึงแดชบอร์ดพันธมิตรเพื่อสร้างลิงก์ติดตาม
- พวกเขาสามารถเริ่มส่งเสริมผลิตภัณฑ์ของคุณและเริ่มรับค่าคอมมิชชั่นได้

พันธมิตรที่ได้รับการอนุมัติจะปรากฏในรายการพันธมิตรด้วยสถานะ **Active** และสามารถเริ่มส่งเสริมโปรแกรมของคุณได้ทันที

## Rejecting Applications

ปฏิเสธการสมัครที่ไม่ตรงตามเกณฑ์ของคุณเพื่อปกป้องแบรนด์และป้องกันการฉ้อโกง

### When to Reject

เหตุผลทั่วไปในการปฏิเสธ:

- **No Active Platform**: ผู้สมัครไม่มีเว็บไซต์ บล็อก หรือการมีอยู่บนโซเชียลมีเดีย
- **Competitor Conflicts**: พวกเขาส่งเสริมคู่แข่งโดยตรงเป็นหลัก
- **Brand Misalignment**: สไตล์ เนื้อหา หรือค่านิยมของพวกเขาขัดแย้งกับแบรนด์ของคุณ
- **Invalid Payment Info**: ข้อมูลการชำระเงินขาดหายไปหรือชัดเจนว่าเป็นเท็จ
- **Suspicious Activity**: อีเมลทั่วไป โปรไฟล์ไม่สมบูรณ์ หรือสัญญาณการฉ้อโกง
- **Terms Violations**: ผู้สมัครที่เคยละเมิดข้อกำหนดของโปรแกรมพันธมิตร

### How to Reject

1. เปิดหน้ารายละเอียดการสมัคร
2. ตรวจสอบข้อมูลผู้สมัครเพื่อยืนยันว่าการปฏิเสธเหมาะสม
3. เพิ่มหมายเหตุในช่อง **Notes** เพื่ออธิบายเหตุผล (สำหรับอ้างอิงภายในเท่านั้น)
4. เปลี่ยนเมนูแบบเลื่อนลง **Status** เป็น **Rejected**
5. คลิก **Save**

### After Rejection

เมื่อคุณปฏิเสธการสมัคร:

- สถานะของพันธมิตรจะเปลี่ยนเป็น **Rejected**
- พวกเขาจะสูญเสียการเข้าถึงแดชบอร์ดพันธมิตร (หากเคยมี)
- พวกเขาไม่สามารถสร้างลิงก์ติดตามหรือรับค่าคอมมิชชั่นได้
- ไม่มีการแจ้งเตือนอัตโนมัติที่ถูกส่ง (คุณสามารถปรับแต่งนี้ในเทมเพลตอีเมล)

พันธมิตรที่ถูกปฏิเสธยังคงอยู่ในฐานข้อมูลของคุณเพื่อการบันทึกข้อมูล คุณสามารถเปลี่ยนสถานะของพวกเขาเป็น **Approved** ได้ในภายหลังหากสถานการณ์เปลี่ยนไป

## Auto-Approve Settings

ควบคุมว่าการสมัครต้องการการตรวจสอบด้วยมือหรือไม่ในการตั้งค่าโปรแกรมพันธมิตรของคุณ:

### Manual Review (Recommended)

ไปที่โปรแกรมของคุณที่ **Marketing > Affiliate Programs** และตรวจสอบให้แน่ใจว่า **Auto-Approve** ยังไม่ได้ถูกเปิดใช้งาน การตั้งค่านี้หมายถึง:

- ทุกการสมัครเริ่มต้นเป็น **Pending**
- คุณตรวจสอบผู้สมัครแต่ละคนก่อนที่พวกเขาจะได้รับการเข้าถึง
- การควบคุมคุณภาพที่ดีขึ้นและการป้องกันการฉ้อโกง
- งานมากขึ้นสำหรับคุณ แต่ปลอดภัยกว่าสำหรับแบรนด์ของคุณ

ใช้การตรวจสอบด้วยมือเมื่อคุณต้องการตรวจสอบคู่ค้าอย่างระมัดระวัง ทำงานกับอินฟลูเอนเซอร์ที่เลือกแล้ว หรือรักษามาตรฐานแบรนด์ที่เข้มงวด

### Auto-Approve Mode

ตรวจสอบ **Auto-Approve** ในการตั้งค่าโปรแกรมของคุณเพื่ออนุมัติการสมัครทั้งหมดโดยอัตโนมัติ การตั้งค่านี้หมายถึง:

- การสมัครข้ามสถานะรอและไปยังสถานะ **Approved** ทันที
- พันธมิตรจะได้รับการเข้าถึงแดชบอร์ดของพวกเขาทันที
- งานน้อยลงสำหรับคุณ แต่มีความเสี่ยงด้านการฉ้อโกงสูงขึ้น
- เหมาะสำหรับโปรแกรมการแนะนำที่เปิดกว้างที่มีผู้สมัครจำนวนมาก

ใช้การอนุมัติอัตโนมัติสำหรับโปรแกรมการแนะนำทั่วไปที่คุณต้องการเพิ่มการมีส่วนร่วมและยอมรับความแตกต่างของคุณภาพบางส่วน

## Bulk Actions

ประมวลผลการสมัครหลายรายการอย่างมีประสิทธิภาพโดยใช้การดำเนินการแบบกลุ่มในอินเทอร์เฟซผู้ดูแลระบบ:

### Approve Multiple Applications

1. ไปที่ **Affiliate Program > Applications**
2. ใช้ตัวกรองเพื่อแสดงเฉพาะการสมัครที่ **Pending**
3. ตรวจสอบกล่องสำหรับทุกการสมัครที่คุณต้องการอนุมัติ
4. เลือก **Approve selected applications** จากเมนูแบบเลื่อนลง **Actions**
5. คลิก **Go** และยืนยัน

### Filter for Efficiency

รวมตัวกรองเพื่อประมวลผลการสมัครเป็นกลุ่ม:

- **Program + Status**: อนุมัติการสมัครที่ยังต้องดำเนินการทั้งหมดสำหรับโปรแกรมอินฟลูเอนเซอร์ของคุณ
- **Date + Status**: ตรวจสอบการสมัครจากสัปดาห์ที่ผ่านมา
- **Payment Method + Status**: ประมวลผลการสมัครผ่าน PayPal ทั้งหมดพร้อมกัน

วิธีการประมวลผลแบบแบตช์นี้ช่วยให้คุณตรวจสอบการสมัครที่คล้ายกันร่วมกัน ทำให้การใช้มาตรฐานที่สม่ำเสมอเป็นเรื่องง่ายขึ้น

## Tips

- **ตรวจสอบภายใน 24-48 ชั่วโมง** — การตอบสนองที่รวดเร็วสร้างความประทับใจเชิงบวกครั้งแรกและป้องกันไม่ให้ผู้สมัครสูญเสียความสนใจหรือสมัครกับคู่แข่ง
- **ตรวจสอบข้อมูลการชำระเงินตั้งแต่ต้น** — การตรวจจับอีเมล PayPal ที่ไม่ถูกต้องหรือข้อมูลบัญชีธนาคารที่ไม่สมบูรณ์ในขั้นตอนการสมัครช่วยลดปัญหาการจ่ายเงินในภายหลัง
- **ตรวจสอบเนื้อหาของพวกเขาเป็นลำดับแรก** — เยี่ยมชมเว็บไซต์หรือโปรไฟล์โซเชียลมีเดียของผู้สมัครก่อนอนุมัติเพื่อตรวจสอบว่าพวกเขามีเนื้อหาจริงและมีผู้ติดตามที่มีส่วนร่วม
- **บันทึกเหตุผลในการปฏิเสธ** — ใช้ช่องหมายเหตุเพื่อบันทึกเหตุผลที่คุณปฏิเสธการสมัคร ช่วยให้รักษาความสม่ำเสมอและปกป้องคุณหากผู้สมัครยื่นขอสมัครอีกครั้งในภายหลัง
- **ตั้งค่าข้อกำหนดของโปรแกรมให้ชัดเจน** — อัปเดตหน้าแรกของพอร์ทัลพันธมิตรของคุณเพื่อระบุอย่างชัดเจนว่าคุณกำลังมองหาอะไรในคู่ค้า (ขนาดผู้ชมขั้นต่ำ ประเภทเนื้อหา ฯลฯ) เพื่อลดการสมัครที่ไม่เหมาะสม
- **ตรวจสอบผู้สมัครซ้ำๆ** — บางครั้งผู้สมัครที่ถูกปฏิเสธก่อนหน้านี้จะยื่นขอสมัครอีกครั้งด้วยอีเมลที่แตกต่างกัน ตรวจสอบ URL เว็บไซต์ ชื่อบริษัท และข้อมูลการชำระเงินเพื่อจับผู้สมัครซ้ำ
- **เริ่มต้นด้วยการตรวจสอบที่เข้มงวด** — มันง่ายกว่าที่จะอนุมัติผู้สมัครมากขึ้นในภายหลังมากกว่าที่จะลบผู้สมัครที่มีปัญหาในภายหลัง เริ่มต้นด้วยเกณฑ์การอนุมัติที่เข้มงวดและคลายลงหากคุณต้องการผู้สมัครมากขึ้น
