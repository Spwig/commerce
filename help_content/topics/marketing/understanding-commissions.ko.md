title: 커미션 이해
---

커미션은 파트너가 성공적으로 판매를 유도할 때 생성되는 수익 기록입니다. 각 커미션은 특정 주문, 파트너, 프로그램과 연결되어 있으며, 대기 상태에서 지급 상태로 이동하는 커미션의 수명 주기를 거칩니다. 이 가이드는 커미션의 작동 방식, 계산 방법, 효과적인 관리 방법을 설명합니다.

## 커미션이란 무엇인가요?

커미션은 고객이 구매를 완료한 경우 해당 고객을 유도한 파트너에게 지급되어야 하는 금액을 의미합니다. 고객이 파트너의 추천 링크를 클릭하고 쿠키 유효 기간 동안 주문을 완료하면 Spwig은 자동으로 커미션 기록을 생성합니다.

각 커미션에는 다음과 같은 정보가 포함됩니다:
- **파트너** — 고객을 유도한 파트너
- **프로그램** — 커미션 규칙을 정의하는 파트너 프로그램
- **주문** — 커미션을 생성한 주문
- **금액** — 계산된 커미션 값
- **상태** — 커미션 수명 주기의 현재 단계
- **날짜** — 생성 날짜, 승인/거부 날짜, 지급 날짜

## 커미션 계산

커미션은 프로그램의 커미션 유형과 비율에 따라 자동으로 계산됩니다.

| 커미션 유형 | 계산 방식 | 예시 |
|-------------|-----------|------|
| **백분율** | 주문 총액 × 커미션 % ÷ 100 | 주문: $200, 비율: 10% → **$20 커미션** |
| **고정 금액** | 주문당 고정 금액 | 비율: $15 → **$15 커미션** (주문 금액과 관계없이) |

### 계산 예시

**백분율 커미션 (10%)**:
- 고객이 $50 주문을 하면 → $5 커미션
- 고객이 $150 주문을 하면 → $15 커미션
- 고객이 $300 주문을 하면 → $30 커미션

**고정 금액 커미션 ($20)**:
- 고객이 $50 주문을 하면 → $20 커미션
- 고객이 $150 주문을 하면 → $20 커미션
- 고객이 $300 주문을 하면 → $20 커미션

커미션은 **주문 소계** (배송비 및 세금 전)에 기반하여 계산되며, 주문이 생성될 때 즉시 생성됩니다.

## 커미션 수명 주기

모든 커미션은 생성부터 지급까지 일련의 상태를 거칩니다:

```
대기 → 승인 → 지급
   ↓
거부
```

### 상태 정의

| 상태 | 설명 | 발생하는 일자 |
|------|------|----------------|
| **대기** | 주문이 생성되고 커미션이 검토를 기다리는 상태 | 커미션은 생성되었지만 아직 확인되지 않았습니다. 파트너는 이를 볼 수 있지만 자금을 인출할 수 없습니다. |
| **승인** | 판매가 유효하다고 상인이 확인한 상태 | 커미션은 확인되어 파트너의 사용 가능한 잔액에 추가됩니다. 지급 대상이 됩니다. |
| **거부** | 상인이 커미션을 거부한 상태 | 커미션은 거부됩니다 (예: 주문이 환불되었거나 사기, 이용 약관 위반). 지급 대상이 아닙니다. |
| **지급** | 커미션이 완료된 지급에 포함된 상태 | 파트너에게 지급되었습니다. 커미션은 최종화되고 수정할 수 없습니다. |

## 커미션이 생성되는 시점

커미션은 다음 순서에 따라 자동으로 생성됩니다:

1. **고객이 파트너 링크를 클릭함** — 추천 URL에는 파트너의 고유 추적 코드가 포함되어 있습니다 (예: `?ref=JOHNSMITH`)
2. **쿠키가 설정됨** — 고객의 브라우저에 파트너 코드가 포함된 추적 쿠키가 저장됩니다
3. **쿠키 유효 기간 동안 구매** — 고객이 쿠키가 만료되기 전에 주문을 완료합니다 (기본값: 30일)
4. **시스템이 주문을 지정함** — Spwig은 활성 추적 쿠키를 확인하고 추천 파트너를 식별합니다
5. **커미션이 자동 생성됨** — 상태가 **대기**인 커미션 기록이 생성됩니다

주문이 생성될 때 커미션은 **즉시** 생성되며, 결제가 확인되기 전에 생성됩니다. 이는 주문이 처리되는 동안 상인들이 커미션을 검토할 수 있도록 합니다.

## 추적 및 지정

Spwig은 **마지막 클릭 지정**을 사용하여 어떤 파트너가 판매에 신용을 받는지 결정합니다.

### 지정 방식

- **마지막 클릭 모델** — 고객이 여러 파트너의 추천 링크를 클릭한 경우, 마지막으로 클릭한 파트너가 신용을 받습니다
- **쿠키 기반 추적** — 고객의 브라우저에 파트너 코드를 저장하는 쿠키
- **쿠키 유효 기간** — 판매가 지정될 수 있는 윈도우 기간 (프로그램별로 설정, 일반적으로 30일)
- **IP 및 세션 추적** — 사기 패턴을 식별하는 추가 데이터

### 지정 예시

- Day 1: 고객이 파트너 A의 링크를 클릭 → 파트너 A의 쿠키가 설정됨
- Day 5: 고객이 파트너 B의 링크를 클릭 → 쿠키가 **업데이트**되어 파트너 B로 변경됨 (마지막 클릭이 우선)
- Day 7: 고객이 주문을 완료함 → 커미션이 **파트너 B**에게 지급됨

고객이 Day 35 (30일 쿠키 유효 기간이 만료된 후)에 다시 돌아와 주문을 완료하면, **커미션이 생성되지 않습니다**. 추적 윈도우가 종료되었기 때문입니다.

## 커미션 세부 정보

**마케팅 > 커미션**으로 이동하여 모든 커미션 기록을 확인할 수 있습니다.

### 커미션 필드

각 커미션은 다음과 같은 정보를 표시합니다:

| 필드 | 설명 |
|------|------|
| **파트너** | 파트너의 이름 및 코드 |
| **프로그램** | 파트너 프로그램 이름 |
| **주문** | 주문 번호 (전체 주문 세부 정보를 보려면 클릭 가능한 링크) |
| **금액** | 계산된 커미션 값 |
| **상태** | 현재 단계 (대기, 승인, 거부, 지급) |
| **생성일** | 커미션이 생성된 시점 |
| **승인/거부일** | 상태가 업데이트된 시점 |
| **지급일** | 지급이 처리된 시점 |
| **메모** | 커미션에 대한 내부 메모 |

### 주문 세부 정보 보기

커미션 기록에서 **주문 번호**를 클릭하여 원래 주문을 확인할 수 있습니다. 이는 다음과 같은 정보를 확인할 수 있도록 합니다:
- 주문 총액 및 구매한 항목
- 고객 정보
- 결제 상태
- 배송 상태
- 환불 또는 반품

이 컨텍스트는 커미션을 승인하거나 거부할지 결정하는 데 도움이 됩니다.

## 커미션 관리

이 가이드는 커미션을 이해하는 데 초점을 맞추고 있지만, 커미션을 승인, 거부, 지급하는 실제 단계는 **커미션 관리** 도움말 주제에서 상세히 다룹니다.

### 간단한 개요

- **승인** — 주문이 정당하고 커미션이 유효하다는 것을 확인합니다
- **거부** — 사기 주문, 환불 또는 정책 위반에 대한 커미션을 거부합니다
- **메모 추가** — 승인 또는 거부 이유를 기록하여 향후 참조용으로 사용합니다
- **지급 처리** — 승인된 커미션을 일괄 지급으로 그룹화합니다

각 관리 작업에 대한 단계별 지침은 관련 도움말 주제를 참조하십시오.

## 팁

- 첫 달 동안 **매일** 대기 중인 커미션을 검토하여 리듬을 만들고 추적 문제를 조기에 발견하세요
- **이메일 알림**을 설정하여 새로운 커미션이 생성될 때 알림을 받으세요. 주문 세부 정보가 신선할 때 검토할 수 있습니다
- 주문이 완료된 후 **주문 처리가 완료된 후** 커미션을 승인하세요 (주문 생성 시 즉시 승인하지 마세요) 환불 및 취소를 고려할 수 있습니다
- **메모 필드**를 사용하여 결정을 기록하세요, 특히 거절된 커미션의 경우, 파트너가 질문을 할 경우 기록이 필요합니다
- **거절 패턴**을 확인하세요 — 만약 한 파트너가 많은 거절된 커미션을 가지고 있다면, 사기 또는 프로그램 조건에 대한 오해를 나타낼 수 있습니다
- **커미션 승인 정책**을 만드는 것을 고려하세요 (예: "14일 환불 기간 이후 승인") 파트너에게 이를 전달하여 명확한 기대를 설정하세요

기억하세요: 마크다운 포맷, 이미지 경로, 코드 블록, 기술 용어는 보존 규칙에 따라 정확히 유지하세요.
