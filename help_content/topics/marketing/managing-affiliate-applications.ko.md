---
title: 협력자 신청 관리
---

잠재적 파트너가 협력자 프로그램에 가입 신청을 하면, 신청 목록에 나타나서 검토를 기다립니다. 이 가이드는 신뢰할 수 있는 협력자를 통해 전문적인 대표와 진정한 판매를 이끌어내는 품질 높은 파트너 네트워크를 구축하기 위해 협력자 신청을 평가, 승인 및 거부하는 방법을 보여줍니다.

협력자 신청을 신중하게 관리하면, 귀하의 매장에 대한 전문적인 대표와 진정한 판매를 이끌어내는 신뢰할 수 있는 협력자와 협력할 수 있습니다.

![Applications List View](/static/core/admin/img/help/managing-affiliate-applications/applications-list.webp)

## Application Sources

협력자 신청은 여러 채널을 통해 귀하의 대기 목록에 도달합니다:

### Public Portal Applications

대부분의 신청은 귀하의 매장에서 `/affiliate/`에 있는 공개 협력자 포털에서 옵니다. 잠재적 협력자가 **Become an Affiliate**을 클릭하면, 등록 양식을 완료하여 신청 기록을 생성합니다.

### Guest vs Registered Users

협력자 설정에서 **Allow Guest Registration**을 활성화한 경우, 비고객이 직접 신청할 수 있습니다. 그렇지 않으면, 사용자는 먼저 귀하의 매장에서 고객 계정을 생성한 후 협력자 신청을 할 수 있습니다.

### Manual Review Requirement

협력자 설정에서 **Require Approval**을 활성화한 경우(권장), 모든 신청은 **Pending** 상태로 유지되어 귀하의 검토를 기다립니다. 비활성화된 경우, 신청은 자동 승인되며 협력자는 즉시 대시보드에 접근할 수 있습니다.

## Viewing Applications

**Affiliate Program > Applications**(또는 관리자에서 **Marketing > Affiliate Memberships**)로 이동하여 모든 프로그램 신청을 볼 수 있습니다.

목록 뷰는 다음과 같이 보입니다:

| Column | Description |
|--------|-------------|
| **Affiliate** | 신청자의 이름과 이메일 |
| **Program** | 그들이 신청한 협력자 프로그램 |
| **Status** | 대기, 승인, 거부 |
| **Applied Date** | 신청을 제출한 날짜 |
| **Payment Method** | 선호한 지급 방법 (PayPal 또는 은행 이체) |

### Filtering Applications

관리자 필터를 사용하여 신청을 좁힐 수 있습니다:

- **Status**: 검토를 기다리는 대기 신청만 보기
- **Program**: 여러 협력자 프로그램을 운영하는 경우 특정 프로그램으로 필터
- **Date Range**: 특정 시간 기간의 신청을 찾기

### Pending Applications Badge

관리자 사이드바는 처리가 필요한 대기 신청이 있을 때 **Affiliates** 옆에 배지 카운트를 표시합니다.

## Reviewing an Application

어떤 신청을 클릭하여 전체 신청자 프로필을 보세요. 이 세부 정보 뷰는 승인 결정을 내리는 데 필요한 모든 정보를 보여줍니다.

![Application Detail Card](/static/core/admin/img/help/managing-affiliate-applications/application-detail-card.webp)

### Affiliate Profile Information

신청자에 대한 다음 주요 세부 정보를 검토하세요:

**Basic Information**
- **Email Address**: 로그인 및 커뮤니케이션에 사용됨
- **Company/Business Name**: 그들의 조직 (적용 가능한 경우)
- **Website URL**: 그들의 주요 홍보 플랫폼
- **Phone Number**: 연락처 정보

**Payment Information**
- **Payment Method**: PayPal 또는 은행 이체
- **PayPal Email**: PayPal 지급을 선택한 경우 필수
- **Bank Details**: 은행 이체를 선택한 경우 필수 (계좌 번호, 라우팅 번호, SWIFT 코드)

**Promotional Channels**
많은 신청은 협력자가 귀하의 제품을 홍보할 계획인 장소에 대한 메모를 포함합니다 - 소셜 미디어 계정, 유튜브 채널, 이메일 목록 또는 블로그.

### Program Details

그들이 신청한 프로그램을 확인하고 수수료율, 쿠키 수명, 최소 지급 임계값을 검토하세요. 신청자가 프로그램의 타겟 고객층에 적합한지 확인하세요.

### Application History

신청자가 이전에 거부되었거나 여러 프로그램에 신청한 경우, 이 기록은 세부 정보 뷰에 나타납니다.

## Approval Criteria

각 신청을 일관되게 평가하기 위해 이 확인 목록을 사용하세요:

### Business Legitimacy

- [ ] **Active Website or Social Media**: 신청자가 실제 콘텐츠가 있는 활성 플랫폼을 가지고 있습니까?
- [ ] **Relevant Audience**: 그들의 대상 고객층과 일치합니까?
- [ ] **Quality Content**: 그들의 콘텐츠가 전문적이며 잘 작성되어 있고 귀하의 브랜드 가치와 일치합니까?
- [ ] **Established Platform**: 그들이 참여도 있는 팔로워나 의미 있는 트래픽을 가지고 있습니까?

### Payment Information

- [ ] **Valid Payment Details**: PayPal 이메일이나 완전한 은행 계좌 정보를 제공했습니까?
- [ ] **Matching Identity**: 지급 정보가 그들의 사업명이나 개인 정보와 일치합니까?

### Brand Alignment

- [ ] **Appropriate Fit**: 그들의 콘텐츠, 톤, 스타일이 귀하의 브랜드 이미지와 일치합니까?
- [ ] **No Conflicts**: 이미 직접 경쟁사를 홍보하고 있습니까?
- [ ] **Professional Standards**: 그들이 귀하가 협력하고자 하는 품질 기준을 유지하고 있습니까?

### Fraud Prevention

- [ ] **No Red Flags**: 일반적인 이메일 주소, 불완전한 프로필, 또는 의심스러운 웹사이트 패턴과 같은 징후를 확인하세요
- [ ] **No Previous Violations**: 이전에 사기 또는 이용 약관 위반으로 거부된 적이 있습니까?
- [ ] **Reasonable Expectations**: 그들이 언급한 홍보 계획이 현실적이며 달성 가능한가요?

모든 기준을 충족하는 신청은 승인하세요. 사기 위험, 브랜드 불일치, 무효한 지급 정보와 같은 어떤 중요한 검사를 실패하면, 명확한 이유를 표시하여 거부하세요.

## Approving Applications

하나 이상의 신청을 승인하려면 다음 단계를 따르세요:

### Single Application Approval

1. 신청 세부 정보 페이지를 열어주세요
2. 모든 프로필 정보를 신중히 검토하세요
3. 지급 세부 정보가 완전하고 유효한지 확인하세요
4. 메모를 추가해야 한다면 **Save and Continue Editing** 버튼을 클릭하세요
5. 상태 드롭다운에서 **Approve**를 선택하세요
6. **Save**를 클릭하세요

### Bulk Approval

여러 개의 자격을 갖춘 신청을 승인하려면:

1. **Affiliate Program > Applications**에서 신청 목록으로 이동하세요
2. 승인하고자 하는 신청 옆의 체크박스를 선택하세요
3. **Actions** 드롭다운에서 **Approve selected applications**을 선택하세요
4. **Go**를 클릭하세요
5. 요청 시 배치 작업을 확인하세요

### What Happens After Approval

신청을 승인하면 다음과 같은 일이 발생합니다:

- 협력자의 상태가 **Approved**로 변경됩니다
- 이메일 템플릿이 구성된 경우 이메일 알림을 받습니다
- 협력자 대시보드에 접근하여 추적 링크를 생성할 수 있습니다
- 귀하의 제품을 홍보하고 수수료를 벌 수 시작합니다

승인된 협력자는 협력자 목록에 **Active** 상태로 나타나며, 즉시 프로그램 홍보를 시작할 수 있습니다.

## Rejecting Applications

기준에 부합하지 않는 신청을 거부하여 브랜드를 보호하고 사기를 방지하세요.

### When to Reject

거부의 일반적인 이유:

- **No Active Platform**: 신청자가 웹사이트, 블로그 또는 소셜 미디어 존재가 없습니다
- **Competitor Conflicts**: 그들은 주로 직접 경쟁사를 홍보합니다
- **Brand Misalignment**: 그들의 콘텐츠 스타일, 언어 또는 가치가 귀하의 브랜드와 충돌합니다
- **Invalid Payment Info**: 누락되거나 명백히 가짜인 지급 세부 정보
- **Suspicious Activity**: 일반적인 이메일, 불완전한 프로필 또는 사기 지표
- **Terms Violations**: 이전에 프로그램 이용 약관을 위반한 협력자

### How to Reject

1. 신청 세부 정보 페이지를 열어주세요
2. 신청자의 정보를 검토하여 거부가 적절한지 확인하세요
3. **Notes** 필드에 거부 이유를 설명하는 메모를 추가하세요(내부 참조용)
4. **Status** 드롭다운을 **Rejected**로 변경하세요
5. **Save**를 클릭하세요

### After Rejection

신청을 거부하면 다음과 같은 일이 발생합니다:

- 협력자의 상태가 **Rejected**로 변경됩니다
- 그들은 협력자 대시보드에 접근할 수 없습니다(이전에 접근할 수 있었다면)
- 그들은 추적 링크를 생성하거나 수수료를 벌 수 없습니다
- 자동 알림은 보내지 않습니다(이메일 템플릿에서 이 설정을 커스터마이징할 수 있습니다)

거부된 협력자는 기록을 위해 데이터베이스에 남아 있습니다. 상황이 변경되면 나중에 그들의 상태를 **Approved**로 변경할 수 있습니다.

## Auto-Approve Settings

협력자 프로그램 구성에서 신청이 수동 검토를 필요로 하는지 제어하세요:

### Manual Review (Recommended)

**Marketing > Affiliate Programs**에서 프로그램으로 이동하여 **Auto-Approve**가 **unchecked** 상태인지 확인하세요. 이 설정은 다음과 같은 의미를 가지고 있습니다:

- 모든 신청은 **Pending** 상태로 시작합니다
- 그들이 접근을 얻기 전에 각 신청자를 검토합니다
- 더 높은 품질 관리 및 사기 방지
- 귀하에게 더 많은 작업이 필요하지만, 브랜드에 더 안전합니다

수동 검토를 사용하여 파트너를 신중하게 검토하고, 특정 인플루언서와 협력하거나, 엄격한 브랜드 표준을 유지하고자 할 때 사용하세요.

### Auto-Approve Mode

프로그램 설정에서 **Auto-Approve**를 확인하여 모든 신청을 자동으로 승인하세요. 이는 다음과 같은 의미를 가지고 있습니다:

- 신청은 대기 상태를 건너뛰고 바로 **Approved** 상태가 됩니다
- 협력자는 즉시 대시보드에 접근할 수 있습니다
- 귀하에게 더 적은 작업이 필요하지만, 사기 위험이 높습니다
- 많은 파트너가 있는 개방형 추천 프로그램에 적합합니다

일반 추천 프로그램에 사용하여 참여를 극대화하고 품질 차이를 수용하고자 할 때 자동 승인을 사용하세요.

## Bulk Actions

관리자 인터페이스의 배치 작업을 사용하여 여러 신청을 효율적으로 처리하세요:

### Approve Multiple Applications

1. **Affiliate Program > Applications**로 이동하세요
2. 필터를 사용하여 오직 **Pending** 상태의 신청만 보여주세요
3. 승인하고자 하는 모든 신청 옆의 체크박스를 선택하세요
4. **Actions** 드롭다운에서 **Approve selected applications**을 선택하세요
5. **Go**를 클릭하고 확인하세요

### Filter for Efficiency

필터를 조합하여 배치로 신청을 처리하세요:

- **Program + Status**: 인플루언서 프로그램에 대한 모든 대기 신청을 승인하세요
- **Date + Status**: 지난 주의 신청을 검토하세요
- **Payment Method + Status**: PayPal 신청자 모두를 함께 처리하세요

이 배치 접근 방식은 유사한 신청을 함께 검토하여 일관된 기준을 적용하는 것이 더 쉬워집니다.

## Tips

- **24-48시간 이내에 검토하세요** - 빠른 응답 시간은 긍정적인 첫인상을 만들고, 신청자가 흥미를 잃거나 경쟁사에게 신청하는 것을 방지합니다
- **초기 단계에서 지급 세부 정보를 확인하세요** - 신청 단계에서 무효한 PayPal 이메일이나 불완전한 은행 세부 정보를 포착하여 나중에 지급 곤란을 방지합니다
- **콘텐츠를 먼저 확인하세요** - 승인하기 전에 신청자의 웹사이트 또는 소셜 미디어 프로필을 방문하여 실제 콘텐츠와 참여도 있는 대상 고객이 있는지 확인하세요
- **거부 이유를 기록하세요** - **Notes** 필드를 사용하여 신청을 거부한 이유를 기록하세요. 이는 일관성을 유지하고, 신청자가 나중에 다시 신청할 경우 보호합니다
- **명확한 프로그램 요구사항을 설정하세요** - 협력자 포털 랜딩 페이지를 업데이트하여 파트너에게 무엇을 원하는지 명확히 표시하세요(최소 대상 고객 수, 콘텐츠 유형 등)하여 자격이 없는 신청을 줄이세요
- **반복 신청자를 주의하세요** - 일부 거부된 협력자는 다른 이메일 주소로 다시 신청합니다. 웹사이트 URL, 회사 이름 및 지급 세부 정보를 확인하여 중복을 포착하세요
- **엄격하게 시작하세요** - 나중에 문제 있는 협력자를 제거하는 것보다 시간이 지남에 따라 더 자유롭게 승인하는 것이 더 쉽습니다. 처음에는 엄격한 승인 기준을 설정하고, 더 많은 파트너가 필요하다면 기준을 완화하세요

기억하세요: 마크다운 포맷, 이미지 경로, 코드 블록 및 기술 용어는 보존 규칙에 따라 정확히 표시되어야 합니다.
