---
title: 联盟跟踪与链接
---

联盟跟踪通过将客户购买与推荐他们的联盟成员关联起来，为整个佣金系统提供动力。本指南将解释跟踪链接的工作原理，当客户点击这些链接时Spwig记录哪些数据，以及基于cookie的归因系统如何确定哪个联盟成员获得每笔佣金。

了解跟踪机制有助于您排查归因问题，分析链接性能，并教育您的联盟成员如何最大化他们的转化率。

## 什么是跟踪链接？

跟踪链接是一个唯一的URL，它将客户重定向到您的商店，同时在cookie中记录联盟成员的身份。每个联盟成员可以创建多个跟踪链接，指向不同的目的地——主页、特定产品、集合页面或落地页。

跟踪链接格式示例：
```
https://yourstore.com/affiliate/track/a2b7f8c4d1e9/
```

此链接将客户重定向到目标页面，同时设置一个跟踪cookie，将未来的购买与拥有代码`a2b7f8c4d1e9`的联盟成员关联起来。

联盟成员从他们的门户仪表板生成这些链接。他们复制完整的URL，并在博客文章、社交媒体、电子邮件或任何接触潜在客户渠道中分享。

## 跟踪链接组件

每个跟踪链接包含以下元素：

| 组件 | 示例 | 描述 |
|-------|------|------|
| **基础URL** | `https://yourstore.com` | 您的商店域名 |
| **跟踪路径** | `/affiliate/track/` | Spwig的跟踪端点 |
| **链接代码** | `a2b7f8c4d1e9` | 自动生成的12位唯一标识符 |
| **目标** | 创建链接时设置 | 客户重定向后到达的位置（主页、产品等） |

当联盟成员创建链接时，Spwig会自动生成唯一的12位代码。联盟成员无需手动创建或编辑此代码——他们只需选择目标，Spwig会处理其余部分。

### 链接标签（可选）

联盟成员可以为每个链接添加标签，以便于他们自己的组织：
- "Instagram简介链接"
- "YouTube描述"
- "黑色星期五电子邮件活动"

标签帮助联盟成员跟踪哪些推广渠道表现最佳。这些标签仅对联盟成员和您可见，客户永远不会看到标签。

## 跟踪机制

从点击到佣金的跟踪和归因过程遵循五个步骤：

### 1. 客户点击链接

潜在客户从任何推广渠道（社交媒体帖子、博客文章、电子邮件通讯）点击联盟成员的跟踪链接。

### 2. 记录点击

Spwig的跟踪端点记录点击详情：
- IP地址
- 用户代理（浏览器和设备）
- HTTP来源（点击来源）
- 时间戳
- 会话标识符

这些数据会出现在**点击**管理界面的**联盟 > 点击**中，用于分析和欺诈检测。

### 3. 设置Cookie

跟踪系统在将客户重定向之前在其浏览器中设置一个Cookie。该Cookie包含：
- 联盟成员ID（应获得佣金的成员）
- 程序ID（适用的佣金结构）
- 链接代码（点击的特定链接）

### 4. 客户购买

客户浏览您的商店并完成购买。这可以立即发生，也可以在几天或几周后，只要他们在Cookie有效期内完成购买。

### 5. 创建佣金

在结账时，Spwig会检查是否有联盟成员的Cookie。如果找到且仍然有效（在Cookie有效期之内），系统会创建一个与联盟成员、程序和订单相关的**待处理**状态的佣金记录。

## 基于Cookie的归因

跟踪Cookie是将购买与联盟成员关联的核心机制。了解Cookie的工作原理有助于您设置最佳归因窗口并排查跟踪问题。

### Cookie结构

| 属性 | 值 |
|------|----|
| **名称** | `aff_{program_id}`（例如，program_id为7时为`aff_7`） |
| **值** | 包含联盟成员ID、链接代码和时间戳的JSON |
| **域** | 您的商店域名 |
| **路径** | `/`（全站访问） |
| **持续时间** | 程序的Cookie有效期（1–365天） |
| **HttpOnly** | `true`（防止JavaScript访问以提高安全性） |
| **SameSite** | `Lax`（允许从外部来源进行跟踪） |
| **Secure** | 在HTTPS站点上为`true`（推荐） |

### Cookie有效期窗口

Cookie有效期决定了客户在点击联盟链接后有多长时间可以完成购买。此窗口在**营销 > 联盟计划**中设置，当您创建或编辑一个计划时。

行业标准Cookie有效期：
- **7天**：快速决策产品（杂货、活动门票）
- **30天**：标准电子商务（最常见的设置）
- **60–90天**：考虑性购买（家具、电子产品、B2B产品）
- **365天**：长销售周期（奢侈品、高价值服务）

如果客户在1月1日点击了联盟链接，而您的Cookie有效期为30天，那么他们在1月30日之前通过该链接完成的任何购买都会归功于该联盟成员。1月31日或之后的购买不会产生佣金，因为Cookie已过期。

### 最后点击归因模型

Spwig使用**最后点击归因**：最近的联盟链接获胜。以下是其工作原理：

**场景**：客户在周一点击联盟A的链接，然后在周三点击联盟B的链接，然后在周五完成购买。

**结果**：联盟B获得佣金，因为他们的链接是最近的点击。

最后点击Cookie会覆盖之前的联盟Cookie。此模型易于理解，可防止双重佣金，但意味着每个订单只有一个联盟成员获得信用（购买前最近的一个）。

## 点击记录

Spwig记录每个联盟链接上的每次点击，以提供您和联盟成员的分析数据。点击数据有助于衡量链接性能、检测欺诈和优化推广策略。

### 每次点击捕获的数据

导航到**联盟 > 点击**以查看所有记录的点击。每个条目包含：

| 字段 | 描述 |
|------|------|
| **链接** | 哪个跟踪链接被点击 |
| **联盟成员** | 拥有该链接的联盟成员 |
| **IP地址** | 客户的IP（用于欺诈检测） |
| **用户代理** | 浏览器和设备信息 |
| **来源** | 客户点击链接的页面（例如，"https://instagram.com"） |
| **会话ID** | 此浏览会话的唯一标识符 |
| **时间戳** | 点击的准确日期和时间 |

### 速率限制

为了防止点击欺诈和机器人滥用，Spwig将点击限制为**每分钟每个IP地址100次**。如果同一IP超过此阈值，额外的点击将被忽略，不会增加点击计数。

此保护措施可防止恶意行为者在不阻断合法流量的情况下夸大点击统计数据。真正的客户几乎不会超过每分钟100次点击。

### 隐私考虑

点击数据包含IP地址和用户代理，用于欺诈检测。确保您的隐私政策披露您跟踪联盟成员的推荐，并与联盟成员分享匿名化的性能数据。

## 查看联盟链接

所有由联盟成员生成的跟踪链接都会出现在您的管理面板中，以便监控和管理。

### 访问链接列表

导航到**联盟 > 链接**以查看所有联盟成员和计划的跟踪链接。列表视图显示：

- **链接代码**：唯一的12位标识符
- **联盟成员**：创建链接的联盟成员
- **计划**：适用的佣金结构
- **标签**：联盟成员提供的可选描述
- **目标**：链接将客户重定向到哪里
- **总点击数**：所有时间的点击计数
- **激活状态**：链接当前是否处于跟踪状态

### 过滤链接

使用管理面板的过滤器来缩小列表：
- **按联盟成员**：查看特定合作伙伴的所有链接
- **按计划**：查看推广特定佣金结构的链接
- **按激活状态**：查找已停用的链接

此过滤有助于您分析联盟网络中的链接分布，并识别表现最佳的链接。

## 链接统计

每个跟踪链接累积的性能指标有助于联盟成员优化其推广策略，并帮助您识别表现最佳的合作伙伴。

### 点击链接记录以查看详细统计信息：

| 指标 | 描述 | 计算方式 |
|------|------|----------|
| **总点击数** | 自链接创建以来的所有记录点击 | 点击记录的数量 |
| **7天内点击数** | 最近活动指标 | 最近7天内的点击数 |
| **转化** | 归因于此链接的订单 | 此链接代码的佣金数量 |
| **转化率** | 点击导致购买的百分比 | (转化 ÷ 总点击数) × 100 |
| **总营收** | 此链接所有订单价值的总和 | 转化点击的订单总额之和 |

### 使用统计信息进行优化

**对于联盟成员**：这些数字显示哪些推广渠道效果最佳。如果一个Instagram简介链接的转化率为5%，而一个博客文章链接的转化率为15%，联盟成员应更加关注博客内容。

**对于商家**：链接统计揭示哪些联盟成员带来了高质量流量。高点击数但低转化率表明该联盟成员的受众可能不适合您的产品。

## 管理链接

您可以从管理面板管理联盟链接，用于维护和故障排除。

### 停用链接

为了防止特定链接跟踪新点击，同时保留历史数据：

1. 导航到**联盟 > 链接**
2. 点击您要停用的链接
3. 取消勾选**激活**
4. 点击**保存**

停用的链接仍然将客户重定向到目标，但不会设置跟踪Cookie或记录点击。当联盟成员正在运行临时活动或您需要停用特定推广渠道时，这很有用。

### 编辑链接详情

您可以修改：
- **标签**：更新联盟成员提供的描述
- **目标**：更改链接重定向的位置（如果移动了产品页面很有用）
- **激活状态**：启用或禁用跟踪

您不能编辑链接代码——它是永久性的，并与所有历史点击和佣金数据相关联。

### 删除不活跃的链接

删除不再使用且没有历史点击或转化的链接。这有助于保持链接列表的整洁，同时不丢失有价值的分析数据。

**警告**：删除链接会删除所有相关点击记录。仅当链接的点击数为零或您完全确定不需要历史数据时才删除链接。

## 归因模型

了解Spwig的归因逻辑有助于您与联盟成员设定预期并解决佣金争议。

### 最后点击归因

如前所述，Spwig使用最后点击归因：如果客户在购买前点击了多个联盟链接，只有最近的联盟成员获得佣金。

**优点**：
- 易于理解和解释
- 防止双重佣金
- 奖励促成交易的联盟成员

**缺点**：
- 介绍客户给联盟成员的早期成员得不到任何信用
- 无法反映多触点客户旅程
- 可能激励"链接劫持"（联盟成员针对已被其他联盟成员推荐的高意向客户）

### Cookie有效期决定资格

只有在Cookie有效期窗口内的购买才会产生佣金。如果Cookie在结账前过期，即使客户通过书签返回，也不会创建佣金。

**示例**：30天Cookie有效期
- 客户1月1日点击链接 → Cookie设置，1月31日过期
- 客户1月25日购买 → 创建佣金
- 客户2月5日购买 → 无佣金（Cookie已过期）

### 会话跟踪

除了Cookie，Spwig还会跟踪每个点击的会话ID。这使得即使Cookie被阻止或清除，也能在同一个会话中进行多访问归因。

如果客户点击链接，浏览您的商店触发多次页面加载，然后他们进行购买——所有都在同一个会话中——即使没有持久Cookie，联盟成员也会获得信用。

## 故障排除

常见跟踪问题及解决方法：

### 链接未跟踪点击

**症状**：尽管联盟成员报告分享了链接，但点击数保持为零。

**原因和解决方法**：
1. **链接已停用**：检查链接详情页面的**激活**状态
2. **计划未激活**：导航到**联盟 > 计划**并确认计划状态为**激活**
3. **联盟成员账户已停用**：在**联盟 > 联盟成员**中检查联盟成员的账户状态
4. **速率限制**：检查是否同一IP生成了过多点击（机器人流量）

### 转化率低

**症状**：点击数很高，但归因的订单很少。

**原因和解决方法**：
1. **Cookie有效期太短**：如果您的产品需要研究和考虑，请增加计划的Cookie有效期
2. **目标页面质量**：检查落地页——是否适合移动设备？加载速度快吗？产品是否有库存？
3. **受众不匹配**：联盟成员的受众可能不适合您的产品
4. **浏览器阻止Cookie**：一些隐私工具会阻止第三方Cookie，尽管Spwig使用第一方Cookie，这些Cookie被阻止的可能性较低

### 重复点击记录

**症状**：同一客户在短时间内生成多个点击记录。

**原因**：这是正常行为。跟踪链接的每次页面加载都会创建一个点击记录。如果客户点击链接，页面加载缓慢，他们再次点击，您会看到多个记录。

**解决方法**：无需采取任何行动。速率限制器防止滥用（每分钟每个IP 100次点击），并且来自同一会话的重复点击不会影响归因——只设置一个Cookie。

## 建议

- **在上线前测试跟踪** — 创建一个测试联盟成员账户，生成一个跟踪链接，在无痕浏览器中点击它，并完成一个测试购买。验证佣金是否以正确的联盟成员归因出现。
- **教育联盟成员了解Cookie有效期** — 确保联盟成员了解他们只能获得在Cookie窗口内完成的购买佣金。这有助于他们设定现实的期望并专注于高意向流量。
- **监控点击模式以检测欺诈** — 来自单个IP的异常高点击量或没有用户代理字符串的点击可能表明机器人流量。在批准佣金前，请仔细审查这些联盟成员。
- **始终使用链接标签** — 鼓励联盟成员按渠道（Instagram、博客、电子邮件）标记他们的链接，这样您都可以分析哪些推广渠道带来最佳转化。
- **高价值产品考虑更长的Cookie有效期** — 如果您的平均订单价值较高且客户通常在购买前进行研究，请将Cookie有效期延长至60–90天，以捕捉这些延迟转化。
- **检查来源数据以获得渠道洞察** — 来源字段显示点击来源。如果您看到很多来自"instagram.com"或"youtube.com"的点击，您就知道您的联盟成员最有效地使用了哪些社交平台。
