---
title: 聯營計劃
---

聯營計劃讓您可以招募推廣您產品的夥伴，並根據他們產生的銷售額獲得傭金。聯營夥伴分享獨特的推薦連結，Spwig 會自動追蹤點擊、歸因訂單並計算傭金。

![聯營計劃](/static/core/admin/img/help/affiliate-program/program-list.webp)

## 如何運作

1. 您可以創建一個或多個 **聯營計劃**，設定傭金比例和規則
2. 聯營夥伴透過公開的門戶 **註冊**，或手動新增
3. 每位聯營夥伴會獲得一個帶有追蹤碼的 **獨特推薦連結**
4. 當顧客點擊連結並完成購買時，會記錄一筆 **傭金**
5. 您審核並批准傭金，然後處理 **付款**

## 創建計劃

前往 **行銷 > 聯營計劃**，並點擊 **新增計劃**。

### 計劃設定

| 設定 | 說明 |
|---------|-------------|
| **名稱** | 聯營夥伴可見的計劃名稱（例如："合作夥伴計劃"） |
| **傭金類型** | 訂單總額的 **百分比** 或每筆銷售的 **固定金額** |
| **傭金比例** | 聯營夥伴可獲得的百分比或固定金額 |
| **Cookie 有效期** | 推薦追蹤 Cookie 的持續天數（預設：30 天） |
| **最低付款額** | 聯營夥伴可要求付款的最低收益 |
| **自動批准聯營夥伴** | 自動接受新的聯營夥伴申請，或需要手動批准 |
| **狀態** | 啟用、暫停或關閉 |

### 傭金類型

- **百分比** — 聯營夥伴可獲得每筆推薦訂單子總額的百分比（例如：100 美元訂單的 10% = 10 美元傭金）
- **固定金額** — 聯營夥伴每筆銷售獲得固定金額，與訂單價值無關（例如：每筆銷售 5 美元）

## 管理聯營夥伴

前往 **行銷 > 聯營夥伴** 來查看和管理聯營夥伴帳號。

### 聯營夥伴詳情

每位聯營夥伴有：
- **聯營代碼** — 在推薦連結中使用的唯一代碼（自動產生或自訂）
- **推薦連結** — 聯營夥伴分享的完整追蹤連結（例如：`yourstore.com/?ref=CODE`）
- **狀態** — 等待處理、已批准或已拒絕
- **付款方式** — 聯營夥伴如何接收付款（PayPal 或銀行轉帳）
- **計畫成員資格** — 聯營夥伴屬於哪些計畫

### 手動新增聯營夥伴

1. 點擊 **新增聯營夥伴**
2. 選擇現有的客戶帳號或建立新的帳號
3. 將聯營夥伴指派至一個或多個計畫
4. 設定聯營代碼（或留空以自動產生）

### 聯營夥伴門戶

聯營夥伴可以訪問一個公開的門戶，他們可以在那裡：
- 查看包含收益和點擊統計的儀表板
- 複製他們的推薦連結
- 追蹤傭金歷史紀錄
- 要求付款

門戶的 URL 會自動在您商店的 `/affiliate/` 路徑下提供。

## 追蹤與傭金

### 追蹤是如何運作的

1. 顧客點擊聯營夥伴的推薦連結
2. 在顧客的瀏覽器中設定追蹤 Cookie（持續設定的 Cookie 有效期）
3. 如果顧客在 Cookie 有效期内下單，該訂單會歸因於聯營夥伴
4. 創建一筆狀態為 **待處理** 的傭金記錄

### 傭金狀態

| 狀態 | 說明 |
|--------|-------------|
| **待處理** | 傭金已記錄，等待審核 |
| **已批准** | 已驗證，準備付款 |
| **已拒絕** | 傭金被拒絕（例如：詐騙訂單或退貨） |
| **已付款** | 傭金已包含在已完成的付款中 |

### 審核傭金

前往 **行銷 > 傭金** 來審核待處理的傭金：

1. 檢查訂單詳情以確認銷售是否合法
2. 點擊 **批准** 以確認，或點擊 **拒絕** 並輸入原因
3. 已批准的傭金會累加至聯營夥伴的付款餘額

## 付款

當聯營夥伴的已批准傭金餘額達到最低付款門檻時，您可以處理付款。

### 處理付款

1. 前往 **行銷 > 付款**
2. 選擇有可用餘額的聯營夥伴
3. 選擇付款方式：
   - **PayPal** — 直接將資金發送到聯營夥伴的 PayPal 電子郵件
   - **銀行轉帳** — 記錄手動銀行轉帳
4. 確認並處理付款
5. 付款狀態更新為 **已完成**，傭金標記為 **已付款**

### 付款服務商

Spwig 與付款服務商整合以實現自動化付款：
- **PayPal** — 透過 PayPal API 自動批量付款
- **Airwallex** — 國際付款，具有競爭性的匯率
- **手動** — 記錄在 Spwig 之外處理的付款

## 推薦連結

每個聯營夥伴的推薦連結遵循以下格式：

```
https://yourstore.com/?ref=AFFILIATE_CODE
```

聯營夥伴也可以建立指向特定產品或類別的連結：

```
https://yourstore.com/products/shoe-name/?ref=AFFILIATE_CODE
```

`ref` 參數適用於任何頁面 — 無論著陸頁是什麼，都會設定追蹤 Cookie。

## 計畫分析

聯營計劃儀表板顯示：
- **總點擊次數** — 推薦連結被點擊的次數
- **總訂單數** — 被歸因於聯營夥伴的訂單數
- **總傭金** — 所有傭金總和（包括待處理、已批准和已付款）
- **活躍聯營夥伴** — 目前正在產生推薦的已批准聯營夥伴數量

## 小技巧

- 從 **基於百分比的傭金**（5–15%）開始 — 它會隨著訂單價值自然擴展，而且聯營夥伴很容易理解。
- 設定 **30 天 Cookie 有效期** 作為基準 — 這讓顧客有時間返回並完成購買，同時仍將銷售歸因於聯營夥伴。
- 對於公開的計劃，啟用 **自動批准** 以減少摩擦，或對邀請制的計劃使用手動批准，以便審核每個聯營夥伴。
- 設定合理的 **最低付款額**（例如：25–50 美元）以避免處理許多小額交易。
- 自定義 **聯營門戶** 以符合您的品牌 — 當體驗感覺專業時，聯營夥伴更有可能推廣您的商店。
- 定期監控傭金，以發現 **詐騙模式**，例如自我推薦、異常高的退貨率或可疑的點擊量。
