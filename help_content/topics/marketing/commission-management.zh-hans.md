---
title: 佣金管理
---

佣金管理是审核和批准联盟销售收益的过程，以确保只有合法的销售被计入。本指南将向您展示如何审核待处理的佣金，批准有效的佣金，拒绝欺诈或退货订单，并使用批量操作高效管理佣金。

## 佣金仪表板

导航到 **营销 > 佣金** 以访问佣金管理仪表板。

该仪表板提供了所有联盟计划中佣金活动的概览：

| 统计 | 描述 |
|-----------|-------------|
| **待处理佣金** | 等待您审核的佣金数量 |
| **已批准佣金** | 已确认并准备付款的佣金 |
| **已支付佣金** | 已支付给联盟成员的佣金 |
| **已拒绝佣金** | 因欺诈、退货或政策违规而被拒绝的佣金 |
| **待支付金额** | 已批准但尚未支付的佣金总价值 |

这些统计数据帮助您跟踪审核工作量并监控联盟计划的财务影响。

## 查看佣金

佣金列表按时间顺序显示所有佣金记录。

### 列表列

| 列 | 描述 |
|--------|-------------|
| **联盟成员** | 联盟成员的名称和唯一代码 |
| **计划** | 生成此佣金的联盟计划 |
| **订单** | 订单编号（点击查看完整订单详情） |
| **金额** | 以您店铺的货币表示的佣金价值 |
| **状态** | 待处理、已批准、已拒绝或已支付 |
| **创建时间** | 佣金生成的时间 |

### 过滤佣金

使用过滤侧边栏缩小佣金范围：

- **按状态** — 仅显示待处理、已批准、已拒绝或已支付的佣金
- **按联盟成员** — 查看特定合作伙伴的佣金
- **按计划** — 查看特定联盟计划的佣金
- **按日期范围** — 按创建日期过滤

### 搜索佣金

使用搜索栏查找特定佣金：

- 输入 **订单编号** 以查找特定销售的佣金
- 输入 **联盟成员代码** 以查看一个合作伙伴的所有佣金

## 佣金详情

点击列表中的任何佣金以查看其详细信息。

### 详细字段

详细视图显示：

- **订单信息** — 点击订单编号在新标签页中查看完整订单，包括商品、送货地址、付款状态和客户详细信息
- **联盟成员信息** — 联盟成员的名称、代码、付款电子邮件和计划会员状态
- **计划详情** — 计划名称、佣金类型（百分比或固定）和佣金率
- **时间戳** — 创建日期、批准/拒绝日期和付款日期
- **备注部分** — 仅对商家可见的内部备注（如下所述）

此信息帮助您在批准佣金前验证其合法性。

## 批准佣金

批准佣金确认其有效性，并将其添加到联盟成员的可用余额中，使其有资格获得付款。

### 何时批准

在以下情况下批准佣金：

- **订单成功完成** — 产品已发货或数字商品已交付
- **无退货或退款** — 客户未要求退货（考虑在完成订单后等待14-30天）
- **符合质量标准** — 销售符合您的计划条款（例如，不是自荐，客户使用了真实的支付方式）
- **未检测到欺诈** — 订单通过欺诈筛查（检查IP、账单/送货地址不匹配、异常订单模式）

### 如何批准

**单个佣金批准：**

1. 导航到 **营销 > 佣金**
2. 点击您要批准的佣金
3. 点击详细页面顶部的 **批准** 按钮
4. 可选地添加备注（例如，"在成功交付后批准"）
5. 状态更改为 **已批准**，佣金被添加到联盟成员的余额中

**批量批准：**

1. 导航到 **营销 > 佣金**
2. 勾选您要批准的佣金旁边的复选框
3. 从 **操作** 下拉菜单中选择 **批准所选**
4. 点击 **执行**
5. 所有选定的佣金状态更改为 **已批准**

已批准的佣金在联盟成员的仪表板中显示为可用余额，并可以包含在下一批付款中。

## 拒绝佣金

拒绝佣金会将其从联盟成员的余额中移除，并标记为不符合付款条件。

### 何时拒绝

在以下情况下拒绝佣金：

- **欺诈订单** — 订单显示欺诈迹象（被盗的支付方式、IP不匹配、联盟成员使用自己的链接）
- **客户退货** — 客户退回商品以获得全额退款
- **质量问题** — 销售不符合计划条款（例如，联盟成员违反了广告指南）
- **条款违规** — 联盟成员使用了禁止的促销方式（垃圾邮件、商标竞价、Cookie填充）
- **订单取消** — 客户在订单完成前取消

### 如何拒绝

**单个佣金拒绝：**

1. 导航到 **营销 > 佣金**
2. 点击您要拒绝的佣金
3. 点击详细页面顶部的 **拒绝** 按钮
4. **添加备注** 说明原因（强烈建议用于解决争议）
5. 状态更改为 **已拒绝**

**批量拒绝：**

1. 导航到 **营销 > 佣金**
2. 勾选您要拒绝的佣金旁边的复选框
3. 从 **操作** 下拉菜单中选择 **拒绝所选**
4. 点击 **执行**
5. 所有选定的佣金状态更改为 **已拒绝**

被拒绝的佣金将从联盟成员的余额中移除，且无法支付。它们在佣金历史记录中仍然可见，用于记录。

## 批量操作

批量操作允许您一次批准或拒绝多笔佣金，从而节省处理大批量时的时间。

### 使用批量操作

1. 导航到 **营销 > 佣金**
2. 过滤列表以仅显示您要处理的佣金（例如，按状态 **待处理** 过滤）
3. 勾选每个佣金旁边的复选框，或点击页眉复选框以选择当前页面上的所有佣金
4. 从 **操作** 下拉菜单中选择一个操作：
   - **批准所选** — 将所有选定的佣金标记为已批准
   - **拒绝所选** — 将所有选定的佣金标记为已拒绝
5. 点击 **执行**
6. 查看确认消息，显示已更新的佣金数量

### 高效批量处理

- **按计划过滤** — 一次性批准来自可信的高绩效联盟成员的所有佣金
- **按日期范围过滤** — 处理超过14天的佣金（已超过您的退货窗口）
- **单独审核高价值佣金** — 对小额佣金使用批量操作，手动审核大额佣金

## 佣金备注

备注字段允许您记录决策并与团队沟通。

### 添加备注

可以在以下情况下添加备注：

- **在批准时** — 点击佣金，添加备注到备注字段，然后点击 **批准**
- **在拒绝时** — 添加说明拒绝原因的备注
- **任何时候** — 点击佣金，添加或编辑备注字段中的备注，并保存

### 何时使用备注

- **已拒绝的佣金** — 始终记录原因（"客户于2月10日退回订单#12345"）
- **高价值佣金** — 记录采取的验证步骤（"通过跟踪编号ABC123验证了交付"）
- **有争议的佣金** — 记录与联盟成员的沟通
- **欺诈模式** — 记录可疑活动以供将来参考

备注是 **仅限内部** 的 — 联盟成员无法看到它们。它们作为您的记录工具。

## 佣金流程

以下是完整的佣金管理流程：

```
订单下单 → 佣金创建（待处理）
                      ↓
              商家审核
                      ↓
                ┌─────┴─────┐
                ↓           ↓
            已批准       已拒绝
                ↓           ↓
        准备付款       无法付款
                ↓
        包含在付款中
                ↓
              已支付
```

**时间线示例：**

- **第1天：** 客户通过联盟链接下单100美元订单 → 创建10美元佣金（待处理）
- **第15天：** 订单完成且退货窗口已过 → 商家批准佣金
- **第20天：** 商家处理月度付款批次 → 佣金状态更改为已支付
- **第21天：** 联盟成员通过PayPal收到付款

## 最佳实践

### 审核窗口

建立一致的审核时间表：

- **每日审核** — 每天早上处理待处理的佣金（推荐用于高流量计划）
- **每周审核** — 每周一预留时间批准上一周的佣金
- **每两周审核** — 与您的付款时间表对齐（月中批准佣金，月底处理付款）

### 质量保证检查

在批准佣金之前，请验证：

1. **订单已发货** — 在管理后台检查订单状态
2. **支付已确认** — 验证支付方式已成功处理
3. **退货窗口已过** — 在发货后等待14-30天以应对退货
4. **无欺诈标志** — 检查订单是否有可疑模式（地址不匹配、高风险国家、同一IP的多个订单）
5. **联盟成员状态良好** — 检查联盟成员的历史记录是否有之前的欺诈或违规行为

### 欺诈预防

留意这些红旗：

- **自我推荐** — 联盟成员使用自己的跟踪链接下单
- **Cookie填充** — 转化率异常高但订单价值低
- **重复订单** — 同一客户/IP通过同一联盟链接下单多次
- **地理位置不匹配** — 联盟成员来自国家A，但只在国家B销售
- **拒付** — 联盟成员推荐的订单有高拒付率

如果您发现欺诈行为，请 **拒绝佣金** 并考虑终止联盟成员的计划资格。

### 与联盟成员的沟通

- **设定预期** — 在计划条款中明确记录佣金批准政策
- **保持透明** — 如果您拒绝佣金，考虑发送电子邮件解释原因（使用备注作为参考）
- **处理争议** — 如果联盟成员对拒绝有疑问，请查看备注和订单详情
- **发布指南** — 在联盟门户上创建一个“佣金批准政策”页面以避免混淆

## 小贴士

- 在 **退货窗口关闭后** 批准佣金（通常为14-30天），以避免批准客户之后可能退货的订单
- 使用 **带过滤器的批量操作** 高效处理可信联盟成员的佣金，同时手动审核新或高风险合作伙伴
- 在 **备注字段中记录拒绝原因** — 这可以保护您，如果联盟成员对决定提出异议，并帮助您识别模式
- 注意 **自我推荐** — 这是常见的违规行为，联盟成员使用自己的链接在个人购买上赚取佣金
- 设置 **最低批准阈值** — 例如，自动批准低于10美元的佣金，但手动审核超过50美元的佣金，以平衡效率与风险
- 创建 **欺诈检查表** — 用红旗列表标准化您的审核流程（IP不匹配、可疑订单模式、高风险支付方式）
- 监控 **联盟成员的拒绝率** — 如果某个联盟成员有大量拒绝，可能表明存在欺诈或需要额外的计划条款培训
