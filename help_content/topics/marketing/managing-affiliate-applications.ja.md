---
title: アフィリエイト申請の管理
---

潜在的なパートナーがアフィリエイトプログラムに参加申請すると、申請キューに表示され、審査を待つことになります。本ガイドでは、質の高いブランドと一致するパートナーのネットワークを構築するために、アフィリエイト申請を評価・承認・拒否する方法を説明します。

申請を慎重に管理することで、あなたのストアをプロフェッショナルに代表し、本物の販売をもたらす信頼できるアフィリエイトと協業できます。

![Applications List View](/static/core/admin/img/help/managing-affiliate-applications/applications-list.webp)

## Application Sources

アフィリエイト申請は、いくつかのチャネルを通じてあなたのキューに到達します。

### Public Portal Applications

ほとんどの申請は、あなたのストアの`/affiliate/`にあるパブリックなアフィリエイトポータルから来ます。潜在的なアフィリエイトが**Become an Affiliate**をクリックすると、登録フォームを完了して申請レコードを作成します。

### Guest vs Registered Users

アフィリエイト設定で**Allow Guest Registration**を有効にしている場合、非顧客は直接申請できます。それ以外の場合は、ユーザーがまずストアで顧客アカウントを作成した後でなければ、アフィリエイトになる申請はできません。

### Manual Review Requirement

アフィリエイト設定で**Require Approval**を有効にしている場合（推奨）、すべての申請は**Pending**状態になり、あなたの審査を待つことになります。無効にしている場合、申請は自動的に承認され、アフィリエイトはすぐにダッシュボードにアクセスできます。

## Viewing Applications

**Affiliate Program > Applications**（または管理画面の**Marketing > Affiliate Memberships**）に移動して、すべてのプログラム申請を確認してください。

リストビューでは以下が表示されます：

| Column | Description |
|--------|-------------|
| **Affiliate** | 申請者の名前とメールアドレス |
| **Program** | どのアフィリエイトプログラムに申請しているか |
| **Status** | 保留中、承認済み、または拒否済み |
| **Applied Date** | 申請を提出した日時 |
| **Payment Method** | お支払いの希望方法（PayPalまたは銀行振込） |

### Filtering Applications

管理画面のフィルターを使用して、申請を絞り込みます：

- **Status**: 保留中の申請のみを表示
- **Program**: 複数のアフィリエイトプログラムを実行している場合、特定のプログラムでフィルター
- **Date Range**: 特定の時間枠の申請を検索

### Pending Applications Badge

管理画面のサイドバーには、保留中の申請がアクションを必要とする場合に**Affiliates**の横にバッジカウントが表示されます。

## Reviewing an Application

任意の申請をクリックして、申請者の完全なプロフィールを表示します。この詳細ビューでは、承認の決定を下すために必要なすべての情報を表示します。

![Application Detail Card](/static/core/admin/img/help/managing-affiliate-applications/application-detail-card.webp)

### Affiliate Profile Information

申請者に関する以下の重要な情報を確認してください：

**Basic Information**
- **Email Address**: ログインおよび通信に使用される
- **Company/Business Name**: その組織（該当する場合）
- **Website URL**: その主なプロモーションプラットフォーム
- **Phone Number**: 連絡先情報

**Payment Information**
- **Payment Method**: PayPalまたは銀行振込
- **PayPal Email**: PayPal支払いを選択した場合に必要
- **Bank Details**: 銀行振込を選択した場合に必要（口座番号、ルーティング番号、SWIFTコード）

**Promotional Channels**
多くの申請には、アフィリエイトがあなたの製品をどの場所でプロモーションするかに関するノートが含まれています — ソーシャルメディアアカウント、YouTubeチャンネル、メールリスト、またはブログ。

### Program Details

どのプログラムに申請したかを確認し、そのコミッション率、クッキーの有効期間、および最低支払い閾値を確認してください。申請者がプログラムのターゲットオーディエンスに適しているかを確認してください。

### Application History

申請者が以前に拒否されたことがあるか、複数のプログラムに申請したことがある場合、この履歴は詳細ビューに表示されます。

## Approval Criteria

このチェックリストを使用して、各申請を一貫して評価してください：

### Business Legitimacy

- [ ] **Active Website or Social Media**: 申請者は実際のコンテンツを持つライブプラットフォームを持っていますか？
- [ ] **Relevant Audience**: そのオーディエンスはあなたのターゲット顧客の人口統計に合っていますか？
- [ ] **Quality Content**: そのコンテンツはプロフェッショナルで、よく書かれていて、あなたのブランド価値と一致していますか？
- [ ] **Established Platform**: 信頼できるフォロワーまたは意味のあるトラフィックを持っているか？

### Payment Information

- [ ] **Valid Payment Details**: 有効なPayPalメールアドレスまたは完全な銀行口座情報を提供していますか？
- [ ] **Matching Identity**: 支払い情報は、彼らのビジネス名または個人情報と一致していますか？

### Brand Alignment

- [ ] **Appropriate Fit**: そのコンテンツ、トーン、スタイルはあなたのブランドイメージと一致していますか？
- [ ] **No Conflicts**: すでに直接の競合をプロモーションしていますか？
- [ ] **Professional Standards**: あなたが関連する品質基準を維持していますか？

### Fraud Prevention

- [ ] **No Red Flags**: 一般的なメールアドレス、プロフィールの不完全、または疑わしいウェブサイトのパターンなどの兆候を確認してください
- [ ] **No Previous Violations**: 以前に詐欺や利用規約の違反で拒否されたことがありますか？
- [ ] **Reasonable Expectations**: その述べたプロモーション計画は現実的で達成可能ですか？

すべての基準を満たす申請は承認してください。詐欺のリスク、ブランドの不一致、無効な支払い情報などの重要なチェックを失敗した場合は、明確な理由を添えて拒否してください。

## Approving Applications

1つ以上の申請を承認するには、以下の手順に従ってください：

### Single Application Approval

1. 申請の詳細ページを開く
2. すべてのプロフィール情報を慎重に確認する
3. 支払い情報が完全で有効であることを確認する
4. メモを残す必要がある場合は、**Save and Continue Editing**ボタンをクリックする
5. ステータスドロップダウンから**Approve**を選択する
6. **Save**をクリックする

### Bulk Approval

複数の資格のある申請に対して：

1. **Affiliate Program > Applications**に移動して、申請リストを表示する
2. 承認したい申請の横にあるチェックボックスをクリックする
3. **Actions**ドロップダウンから**Approve selected applications**を選択する
4. **Go**をクリックする
5. バッチアクションを確認するよう求められた場合に確認する

### What Happens After Approval

申請を承認した後：

- アフィリエイトのステータスは**Approved**に変更されます
- 申請者がメールテンプレートが構成されている場合、メール通知を受け取ります
- アフィリエイトダッシュボードにアクセスして、トラッキングリンクを生成できます
- あなたの製品をプロモーションし、コミッションを獲得できます

承認されたアフィリエイトは、アフィリエイトリストに**Active**ステータスで表示され、すぐにあなたのプログラムをプロモーションを開始できます。

## Rejecting Applications

基準に合致しない申請を拒否して、ブランドを保護し、詐欺を防止してください。

### When to Reject

拒否の一般的な理由：

- **No Active Platform**: 申請者がウェブサイト、ブログ、またはソーシャルメディアの存在を持っていない
- **Competitor Conflicts**: 彼らが主に直接の競合をプロモーションしている
- **Brand Misalignment**: そのコンテンツスタイル、言語、または価値観がブランドと衝突している
- **Invalid Payment Info**: 欠落している、または明らかに偽の支払い情報
- **Suspicious Activity**: 一般的なメール、不完全なプロフィール、または詐欺の兆候
- **Terms Violations**: 以前のアフィリエイトでプログラムの利用規約を違反した

### How to Reject

1. 申請の詳細ページを開く
2. 申請者の情報を確認して、拒否が適切であることを確認する
3. **Notes**フィールドに拒否理由を説明するメモを追加する（内部参照のみ）
4. **Status**ドロップダウンを**Rejected**に変更する
5. **Save**をクリックする

### After Rejection

申請を拒否した後：

- アフィリエイトのステータスは**Rejected**に変更されます
- 以前アクセスしていたアフィリエイトダッシュボードへのアクセス権を失います
- トレーキングリンクを作成したり、コミッションを獲得したりすることはできません
- 自動的に通知は送信されません（メールテンプレートでこれをカスタマイズできます）

拒否されたアフィリエイトは、記録のためにデータベースに残ります。状況が変化した場合、後でそのステータスを**Approved**に変更できます。

## Auto-Approve Settings

アフィリエイトプログラムの設定で、申請が手動のレビューを必要とするかどうかを制御してください：

### Manual Review (Recommended)

**Marketing > Affiliate Programs**に移動して、**Auto-Approve**が**unchecked**であることを確認してください。この設定では：

- すべての申請は**Pending**から始まります
- 申請者がアクセスを取得する前に、各申請者をレビューします
- より良い品質管理と詐欺防止
- あなたにとってより多くの作業が必要ですが、ブランドにとってより安全です

手動レビューを使用して、パートナーを慎重に選定し、特定のインフルエンサーと協業し、厳格なブランド基準を維持したい場合に使用してください。

### Auto-Approve Mode

プログラム設定で**Auto-Approve**をチェックして、すべての申請を自動的に承認します。これは意味します：

- 申請は保留状態をスキップして直接**Approved**に進みます
- アフィリエイトはすぐにダッシュボードにアクセスできます
- あなたにとっての作業量は少なくなりますが、詐欺のリスクは高まります
- 多くのパートナーを持つオープンなリファラルプログラムに最適です

オープンなリファラルプログラムで、参加を最大化し、ある程度の品質の変動を許容したい場合に、自動承認を使用してください。

## Bulk Actions

管理インターフェースのバッチアクションを使用して、複数の申請を効率的に処理してください：

### Approve Multiple Applications

1. **Affiliate Program > Applications**に移動します
2. フィルターを使用して、**Pending**状態の申請のみを表示します
3. 承認したいすべての申請のチェックボックスをクリックします
4. **Actions**ドロップダウンから**Approve selected applications**を選択します
5. **Go**をクリックして確認します

### Filter for Efficiency

フィルターを組み合わせて、申請をバッチ処理して効率化します：

- **Program + Status**: あなたのインフルエンサープログラムの保留中の申請すべてを承認
- **Date + Status**: 過去1週間の申請をレビュー
- **Payment Method + Status**: すべてのPayPal申請者を一緒に処理

このバッチ処理アプローチは、類似した申請を一緒にレビューし、一貫した基準を適用するのを容易にします。

## Tips

- **24〜48時間以内にレビュー** — 快速な応答時間は、ポジティブな第一印象を作り、申請者が関心を失ったり、競合に申請したりすることを防ぎます。
- **支払い情報を早期に確認** — 申請段階で無効なPayPalメールアドレスや不完全な銀行情報を見つけることで、後の支払いの問題を防ぎます。
- **コンテンツを最初に確認** — 承認する前に、申請者のウェブサイトやソーシャルメディアプロフィールを訪問して、本当にコンテンツがあり、関心のあるオーディエンスを持っているかを確認してください。
- **拒否理由を記録** — ノートフィールドを使用して、申請を拒否した理由を記録してください。これは、一貫性を保つのに役立ち、申請者が後で再申請した場合にあなたを保護します。
- **明確なプログラム要件を設定** — アフィリエイトポータルの着陸ページを更新して、パートナーに求めていることを明確に述べてください（最小オーディエンスサイズ、コンテンツタイプなど）これにより、不適格な申請を減らすことができます。
- **再申請者を注意深く見る** — 一部の拒否されたアフィリエイトは、異なるメールアドレスで再申請します。ウェブサイトURL、会社名、支払い情報を確認して、重複を検出してください。
- **厳格に開始する** — 後で問題のあるアフィリエイトを削除するよりも、時間とともにより自由に承認する方が簡単です。最初は厳格な承認基準から始め、より多くのパートナーが必要であれば、それを緩和してください。
