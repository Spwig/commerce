---
title: 管理联盟申请
---

当潜在合作伙伴申请加入您的联盟计划时，它们会出现在您的申请队列中，等待审核。本指南将向您展示如何评估、批准和拒绝联盟申请，以建立一个与您的品牌相匹配的高质量合作伙伴网络。

仔细管理申请可以确保您与值得信赖的联盟合作伙伴合作，他们将专业地代表您的商店并推动真实的销售。

![Applications List View](/static/core/admin/img/help/managing-affiliate-applications/applications-list.webp)

## Application Sources

联盟申请通过以下几种渠道进入您的队列：

### Public Portal Applications

大多数申请来自您商店的公共联盟门户，网址为 `/affiliate/`。当潜在联盟成员点击 **Become an Affiliate** 时，他们会完成一个注册表单，该表单会创建一个申请记录。

### Guest vs Registered Users

如果您在联盟设置中启用了 **Allow Guest Registration**，非客户可以直接申请。否则，用户必须首先在您的商店上创建客户账户，然后才能申请成为联盟成员。

### Manual Review Requirement

当在联盟设置中启用 **Require Approval**（推荐）时，所有申请都会进入 **Pending** 状态，等待您的审核。如果禁用此功能，申请将自动获得批准，联盟成员可以立即访问他们的仪表板。

## Viewing Applications

导航到 **Affiliate Program > Applications**（或在管理后台中 **Marketing > Affiliate Memberships**）以查看所有计划申请。

列表视图显示：

| Column | Description |
|--------|-------------|
| **Affiliate** | 申请人的姓名和电子邮件 |
| **Program** | 他们正在申请的联盟计划 |
| **Status** | 等待审核、已批准或已拒绝 |
| **Applied Date** | 他们提交申请的日期 |
| **Payment Method** | 他们偏好的付款方式（PayPal 或银行转账） |

### Filtering Applications

使用管理后台的筛选功能来缩小申请范围：

- **Status**: 仅查看等待审核的申请
- **Program**: 如果您运行多个联盟计划，可以按特定计划筛选
- **Date Range**: 查找特定时间段内的申请

### Pending Applications Badge

当您有需要处理的待处理申请时，管理后台侧边栏会在 **Affiliates** 旁边显示一个徽章计数。

## Reviewing an Application

点击任何申请以查看完整的申请人资料。此详细视图显示了您做出知情批准决定所需的所有信息。

![Application Detail Card](/static/core/admin/img/help/managing-affiliate-applications/application-detail-card.webp)

### Affiliate Profile Information

审查申请人以下关键信息：

**Basic Information**
- **Email Address**: 用于登录和通信
- **Company/Business Name**: 他们的组织（如适用）
- **Website URL**: 他们的主要推广平台
- **Phone Number**: 联系信息

**Payment Information**
- **Payment Method**: PayPal 或银行转账
- **PayPal Email**: 如果他们选择了 PayPal 支付方式，则需要提供
- **Bank Details**: 如果他们选择了银行转账，则需要提供（账户号码、路由号码、SWIFT 代码）

**Promotional Channels**
许多申请包括关于联盟成员计划推广您产品的渠道的说明——社交媒体账号、YouTube 频道、电子邮件列表或博客。

### Program Details

检查他们申请的计划并查看其佣金率、Cookie 有效期和最低付款门槛。确保申请人适合该计划的目标受众。

### Application History

如果申请人之前被拒绝过或申请了多个计划，此历史记录会显示在详细视图中。

## Approval Criteria

使用此检查表来评估每个申请，以确保一致性：

### Business Legitimacy

- [ ] **Active Website or Social Media**: 申请人是否有具有真实内容的实时平台？
- [ ] **Relevant Audience**: 他们的受众是否与您的目标客户人口统计相匹配？
- [ ] **Quality Content**: 他们的内容是否专业、写得不错，并与您的品牌价值观一致？
- [ ] **Established Platform**: 他们是否有参与度高的关注者或有意义的流量？

### Payment Information

- [ ] **Valid Payment Details**: 他们是否提供了有效的 PayPal 邮箱或完整的银行账户信息？
- [ ] **Matching Identity**: 支付信息是否与他们的公司名称或个人信息一致？

### Brand Alignment

- [ ] **Appropriate Fit**: 他们的内容、语气和风格是否与您的品牌形象相符？
- [ ] **No Conflicts**: 他们是否已经在推广直接竞争对手？
- [ ] **Professional Standards**: 他们是否保持您愿意与其关联的质量标准？

### Fraud Prevention

- [ ] **No Red Flags**: 检查是否有迹象，如通用电子邮件地址、不完整的个人资料或可疑的网站模式
- [ ] **No Previous Violations**: 他们之前是否因欺诈或条款违规而被拒绝？
- [ ] **Reasonable Expectations**: 他们陈述的推广计划是否现实且可实现？

如果申请符合所有标准，请批准。如果它未能通过任何关键检查（欺诈风险、品牌不匹配、无效的付款信息），请明确说明原因予以拒绝。

## Approving Applications

按照以下步骤批准一个或多个申请：

### Single Application Approval

1. 打开申请详情页面
2. 仔细审查所有个人资料信息
3. 验证付款信息是否完整且有效
4. 如果需要做笔记，请点击 **Save and Continue Editing** 按钮
5. 从状态下拉菜单中选择 **Approve**
6. 点击 **Save**

### Bulk Approval

对于多个合格的申请：

1. 导航到 **Affiliate Program > Applications** 的申请列表
2. 勾选您要批准的申请旁边的复选框
3. 从 **Actions** 下拉菜单中选择 **Approve selected applications**
4. 点击 **Go**
5. 在提示时确认批量操作

### What Happens After Approval

批准申请后：

- 联盟成员的状态更改为 **Approved**
- 如果已配置电子邮件模板，他们将收到电子邮件通知
- 他们可以访问联盟仪表板以生成跟踪链接
- 他们可以开始推广您的产品并赚取佣金

已批准的联盟成员会出现在联盟成员列表中，状态为 **Active**，并可以立即开始推广您的计划。

## Rejecting Applications

拒绝不符合您标准的申请，以保护您的品牌并防止欺诈。

### When to Reject

拒绝的常见原因：

- **No Active Platform**: 申请人没有网站、博客或社交媒体存在
- **Competitor Conflicts**: 他们主要推广直接竞争对手
- **Brand Misalignment**: 他们的内容风格、语言或价值观与您的品牌冲突
- **Invalid Payment Info**: 缺失或明显虚假的付款信息
- **Suspicious Activity**: 通用电子邮件、不完整的个人资料或欺诈迹象
- **Terms Violations**: 之前违反计划条款的联盟成员

### How to Reject

1. 打开申请详情页面
2. 审查申请人的信息以确认拒绝是合适的
3. 在 **Notes** 字段中添加说明以解释原因（仅用于内部参考）
4. 将 **Status** 下拉菜单更改为 **Rejected**
5. 点击 **Save**

### After Rejection

拒绝申请后：

- 联盟成员的状态更改为 **Rejected**
- 他们失去对联盟仪表板的访问权限（如果之前有访问权限）
- 他们无法创建跟踪链接或赚取佣金
- 不会自动发送通知（您可以在电子邮件模板中自定义此设置）

被拒绝的联盟成员仍保留在您的数据库中以供记录。如果情况发生变化，您可以稍后将他们的状态更改为 **Approved**。

## Auto-Approve Settings

控制您的联盟计划配置中申请是否需要人工审核：

### Manual Review (Recommended)

导航到您的计划 **Marketing > Affiliate Programs**，并确保 **Auto-Approve** 未选中。此设置意味着：

- 所有申请开始为 **Pending**
- 您在他们获得访问权限之前审查每个申请人
- 更好的质量控制和欺诈预防
- 您的工作量更大，但对您的品牌更安全

当您希望仔细审核合作伙伴、与特定影响者合作或保持严格的品牌标准时，使用人工审核。

### Auto-Approve Mode

在您的计划设置中勾选 **Auto-Approve** 以自动接受所有申请。这意味着：

- 申请跳过待处理状态，直接进入 **Approved**
- 联盟成员可以立即访问他们的仪表板
- 您的工作量更少，但欺诈风险更高
- 适用于开放推荐计划，拥有大量合作伙伴

对于一般推荐计划，当您希望最大化参与度并接受一些质量差异时，使用自动批准。

## Bulk Actions

使用管理界面的批量操作高效处理多个申请：

### Approve Multiple Applications

1. 导航到 **Affiliate Program > Applications**
2. 使用筛选器仅显示 **Pending** 申请
3. 勾选您要批准的所有申请的复选框
4. 从 Actions 下拉菜单中选择 **Approve selected applications**
5. 点击 **Go** 并确认

### Filter for Efficiency

结合筛选器以分批处理申请：

- **Program + Status**: 批准您影响者计划的所有待处理申请
- **Date + Status**: 审核过去一周的申请
- **Payment Method + Status**: 一起处理所有 PayPal 申请人

这种批量处理方法可以帮助您一起审核类似的申请，使您更容易应用一致的标准。

## Tips

- **在 24-48 小时内审核** — 快速的响应时间可以创造良好的第一印象，并防止申请人失去兴趣或申请竞争对手
- **尽早验证付款信息** — 在申请阶段发现无效的 PayPal 邮箱或不完整的银行信息，可以避免以后付款时的麻烦
- **先检查他们的内容** — 在批准之前访问申请人的网站或社交媒体资料，以确认他们有真实的内容和参与度高的受众
- **记录拒绝原因** — 使用备注字段记录拒绝申请的原因。这有助于保持一致性，并在申请人以后重新申请时保护您
- **设定明确的计划要求** — 更新您的联盟门户落地页，明确说明您在合作伙伴中寻找的内容（最低受众规模、内容类型等），以减少不合格的申请
- **留意重复申请者** — 一些被拒绝的联盟成员可能会使用不同的电子邮件地址重新申请。检查网站 URL、公司名称和付款信息以发现重复申请
- **从严开始** — 随着时间的推移，更容易更自由地批准申请，而不是以后移除有问题的联盟成员。从严格的批准标准开始，如果需要更多合作伙伴，可以逐渐放宽标准
