---
title: 管理聯營應用程式
---

當潛在合作夥伴申請加入您的聯營計劃時，他們會出現在您的應用程式佇列中，等待審核。本指南將教您如何評估、批准和拒絕聯營應用程式，以建立一個與您的品牌一致的高品質合作夥伴網絡。

仔細管理應用程式可確保您與值得信賴的聯營合作夥伴合作，他們會專業地代表您的商店並推動真實的銷售。

![應用程式列表視圖](/static/core/admin/img/help/managing-affiliate-applications/applications-list.webp)

## 應用程式來源

聯營應用程式通過多個渠道到達您的佇列中：

### 公眾門戶應用程式

大多數應用程式來自您商店的公眾聯營門戶，位於 `/affiliate/`。當潛在聯營合作夥伴點擊 **成為聯營合作夥伴** 時，他們會完成一個註冊表單，該表單會創建一個應用程式記錄。

### 訪客與註冊用戶

如果您在聯營設置中啟用了 **允許訪客註冊**，非客戶可以直接申請。否則，用戶必須首先在您的商店上創建一個客戶賬戶，然後才能申請成為聯營合作夥伴。

### 手動審核要求

當您在聯營設置中啟用 **需要批准**（建議）時，所有應用程式都會處於 **待處理** 狀態，等待您的審核。如果禁用，應用程式會自動批准，聯營合作夥伴可以立即訪問他們的儀表板。

## 查看應用程式

導航到 **聯營計劃 > 應用程式**（或 **營銷 > 聯營成員資格** 在管理員中）以查看所有計劃應用程式。

列表視圖顯示：

| 欄位 | 說明 |
|--------|-------------|
| **聯營合作夥伴** | 申請人的姓名和電子郵件 |
| **計劃** | 他們正在申請的聯營計劃 |
| **狀態** | 待處理、已批准或已拒絕 |
| **申請日期** | 他們提交申請的時間 |
| **付款方式** | 他們偏好的付款方式（PayPal 或銀行轉賬） |

### 過濾應用程式

使用管理員過濾器來縮小應用程式範圍：

- **狀態**：僅查看待處理的應用程式，等待審核
- **計劃**：如果您運營多個聯營計劃，可以按特定計劃過濾
- **日期範圍**：查找特定時間段內的應用程式

### 待處理應用程式徽章

當您有需要處理的待處理應用程式時，管理員側邊欄會在 **聯營合作夥伴** 旁顯示徽章計數。

## 審核應用程式

點擊任何應用程式以查看完整的申請人個人資料。此詳細視圖顯示了您做出有根據的批准決策所需的所有信息。

![應用程式詳細資料卡片](/static/core/admin/img/help/managing-affiliate-applications/application-detail-card.webp)

### 聯營合作夥伴個人資料信息

審核申請人的以下關鍵細節：

**基本資訊**
- **電子郵件地址**：用於登錄和溝通
- **公司/商號名稱**：他們的組織（如適用）
- **網站 URL**：他們的主要推廣平台
- **電話號碼**：聯繫信息

**付款資訊**
- **付款方式**：PayPal 或銀行轉賬
- **PayPal 電子郵件**：如果他們選擇 PayPal 支付，則需要此信息
- **銀行詳情**：如果他們選擇銀行轉賬，則需要此信息（賬戶號碼、路由號碼、SWIFT 編碼）

**推廣渠道**

許多應用程式包含關於聯營合作夥伴計劃推廣您產品的筆記——社交媒體賬號、YouTube 頻道、電子郵件列表或博客。

### 聯營計劃詳情

檢查他們申請的計劃，並審核其傭金率、cookie 有效期和最低付款門檻。確保申請人適合該計劃的目標受眾。

### 應用程式歷史記錄

如果申請人之前被拒絕過或申請了多個計劃，此歷史記錄會出現在詳細視圖中。

## 批准標準

使用此檢查清單來評估每個應用程式，以保持一致性：

### 商業合法性

- [ ] **活躍的網站或社交媒體**：申請人是否有具有真實內容的活躍平台？
- [ ] **相關受眾**：他們的受眾是否與您的目標客戶人口統計相符？
- [ ] **高品質內容**：他們的內容是否專業、寫作良好，並與您的品牌價值觀一致？
- [ ] **已建立的平台**：他們是否有參與度高的追隨者或有意義的流量？

### 付款資訊

- [ ] **有效的付款詳情**：他們是否提供了可用的 PayPal 電子郵件或完整的銀行賬戶資訊？
- [ ] **匹配的身份**：付款詳情是否與他們的商號名稱或個人資訊一致？

### 品牌一致性

- [ ] **適當的契合度**：他們的內容、語調和風格是否與您的品牌形象一致？
- [ ] **無衝突**：他們是否已經推廣直接競爭對手？
- [ ] **專業標準**：他們是否維持您願意關聯的品質標準？

### 防止詐騙

- [ ] **無紅旗**：檢查是否有跡象，如通用電子郵件地址、不完整的個人資料或可疑的網站模式
- [ ] **無先前違規**：他們之前是否因詐騙或條款違規而被拒絕？
- [ ] **合理的期望**：他們所聲稱的推廣計劃是否現實且可實現？

如果應用程式符合所有標準，請批准。如果它未能通過任何關鍵檢查（詐騙風險、品牌不一致、無效的付款資訊），請拒絕並提供明確的原因。

## 批准應用程式

按照以下步驟批准一個或多個應用程式：

### 批准單個應用程式

1. 打開應用程式詳細頁面
2. 認真審閱所有個人資料資訊
3. 驗證付款詳情是否完整且有效
4. 如果需要做筆記，點擊 **保存並繼續編輯** 按鈕
5. 從狀態下拉菜單中選擇 **批准**
6. 點擊 **保存**

### 批量批准

對於多個合格的應用程式：

1. 導航到 **聯營計劃 > 應用程式** 的應用程式列表
2. 勾選您要批准的應用程式旁的方框
3. 從 **操作** 下拉菜單中選擇 **批准選中的應用程式**
4. 點擊 **前往**
5. 當提示時確認批量操作

### 批准後發生什麼

當您批准應用程式時：

- 聯營合作夥伴的狀態變為 **已批准**
- 如果配置了電子郵件模板，他們會收到電子郵件通知
- 他們可以訪問聯營合作夥伴儀表板以生成追蹤鏈接
- 他們可以開始推廣您的產品並賺取傭金

已批准的聯營合作夥伴會出現在聯營合作夥伴列表中，狀態為 **活躍**，並可以立即開始推廣您的計劃。

## 拒絕應用程式

拒絕不符合您標準的應用程式，以保護您的品牌並防止詐騙。

### 當何時拒絕

拒絕的常見原因：

- **無活躍平台**：申請人沒有網站、博客或社交媒體存在
- **競爭對手衝突**：他們主要推廣直接競爭對手
- **品牌不一致**：他們的內容風格、語言或價值觀與您的品牌衝突
- **無效的付款資訊**：缺少或明顯虛假的付款詳情
- **可疑活動**：通用電子郵件、不完整的個人資料或詐騙指標
- **條款違規**：之前違反計劃條款的聯營合作夥伴

### 如何拒絕

1. 打開應用程式詳細頁面
2. 審閱申請人的資訊以確認拒絕是合適的
3. 在 **筆記** 欄中添加說明拒絕原因（僅限內部參考）
4. 將 **狀態** 下拉菜單更改為 **已拒絕**
5. 點擊 **保存**

### 拒絕後發生什麼

當您拒絕應用程式時：

- 聯營合作夥伴的狀態變為 **已拒絕**
- 他們失去對聯營合作夥伴儀表板的訪問權限（如果有的話）
- 他們無法創建追蹤鏈接或賺取傭金
- 不會自動發送通知（您可以在電子郵件模板中自定義此功能）

被拒絕的聯營合作夥伴會留在您的數據庫中以供記錄。如果情況改變，您可以後續將他們的狀態更改為 **已批准**。

## 自動批准設置

控制您的聯營計劃配置中應用程式是否需要手動審核：

### 手動審核（建議）

導航到您的計劃 **營銷 > 聯營計劃**，並確保 **自動批准** 未勾選。此設置表示：

- 所有應用程式開始為 **待處理**
- 您在他們獲得訪問權限之前審核每個申請人
- 更好的品質控制和詐騙預防
- 對您來說工作量更大，但對您的品牌更安全

當您在需要仔細審核合作夥伴、與特定影響者合作或維持嚴格品牌標準時，使用手動審核。

### 自動批准模式

在您的計劃設置中勾選 **自動批准** 以自動接受所有應用程式。這表示：

- 應用程式跳過待處理狀態，直接進入 **已批准**
- 聯營合作夥伴可以立即訪問他們的儀表板
- 對您來說工作量更少，但詐騙風險更高
- 最適合開放推薦計劃，擁有大量合作夥伴

對於一般推薦計劃，當您希望最大化參與度並接受一些品質差異時，使用自動批准。

## 批量操作

使用管理員介面的批量操作來高效處理多個應用程式：

### 批准多個應用程式

1. 導航到 **聯營計劃 > 應用程式**
2. 使用過濾器僅顯示 **待處理** 的應用程式
3. 勾選您要批准的所有應用程式旁的方框
4. 從操作下拉菜單中選擇 **批准選中的應用程式**
5. 點擊 **前往** 並確認

### 為提高效率過濾

結合過濾器以批次處理應用程式：

- **計劃 + 狀態**：批准所有待處理的影響者計劃應用程式
- **日期 + 狀態**：審核過去一周的應用程式
- **付款方式 + 狀態**：一起處理所有 PayPal 申請人

這種批次處理方法可幫助您一起審核類似的應用程式，使您更容易應用一致的標準。

## 小技巧

- **在 24-48 小時內審核** — 快速的回應時間會創造良好的第一印象，並防止申請人失去興趣或申請競爭對手
- **早期驗證付款詳情** — 在申請階段就發現無效的 PayPal 電子郵件或不完整的銀行詳情，可以避免後期付款的麻煩
- **先查看他們的內容** — 在批准之前訪問申請人的網站或社交媒體個人資料，以確認他們有真實的內容和參與度高的受眾
- **記錄拒絕原因** — 使用筆記欄來記錄拒絕應用程式的原因。這有助於保持一致性，並在申請人後續重新申請時保護您
- **設定明確的計劃要求** — 更新您的聯營門戶首頁，明確說明您在合作夥伴中尋找什麼（最低受眾規模、內容類型等），以減少不合格的應用程式
- **留意重複申請人** — 有些被拒絕的聯營合作夥伴會使用不同的電子郵件地址重新申請。檢查網站 URL、公司名稱和付款詳情以發現重複
- **開始時嚴格** — 隨著時間的推移，更容易批准更多的申請，而不是後期刪除有問題的聯營合作夥伴。從嚴格的批准標準開始，如果需要更多合作夥伴，再放寬標準
