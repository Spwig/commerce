# Built-in SMTP Server Provider

## Overview

The Built-in SMTP Server is a self-hosted email sending solution that provides **zero-configuration email delivery** for your Spwig e-commerce platform. Unlike external providers that charge monthly fees, the built-in SMTP server runs on your infrastructure, giving you complete control and privacy.

## Key Features

- **Zero Monthly Fees** - Completely free, you only pay for server infrastructure
- **Full Control** - Own your email infrastructure and data
- **Automatic DKIM Signing** - Professional email authentication out of the box
- **Privacy First** - Emails don't go through third-party services
- **No Rate Limits** - Only limited by your server capacity
- **localhost Security** - Server only accepts connections from localhost (port 2525)
- **Postfix Integration** - Production-ready relay architecture

## Architecture

```
Django Application
    ↓
Built-in SMTP Server (localhost:2525)
    ↓ DKIM Signing
Postfix MTA (port 25)
    ↓
Internet (Email Recipients)
```

The built-in SMTP server runs on `localhost:2525` for security (no authentication needed, localhost-only). It automatically signs all outgoing emails with DKIM, then relays them through **Postfix** for delivery to the internet.

## System Requirements

### Required Software

1. **Postfix MTA** (Mail Transfer Agent)
   ```bash
   sudo apt-get install postfix
   ```
   - Choose "Internet Site" during installation
   - Configure hostname as your domain name

2. **Python Dependencies** (already included in requirements.txt)
   - `aiosmtpd==1.4.6` - Modern asyncio SMTP server
   - `dkimpy==1.1.8` - DKIM signing library

### DNS Requirements

For reliable email delivery, you MUST configure these DNS records:

1. **SPF Record** - Authorizes your server to send emails
   - Type: `TXT`
   - Hostname: `@` (your domain)
   - Value: `v=spf1 ip4:YOUR_SERVER_IP ~all`

2. **DKIM Record** - Cryptographic email signature
   - Type: `TXT`
   - Hostname: `mail._domainkey.yourdomain.com`
   - Value: Auto-generated by `./manage.py generate_dkim_keys`

3. **DMARC Record** - Policy for failed authentication
   - Type: `TXT`
   - Hostname: `_dmarc.yourdomain.com`
   - Value: `v=DMARC1; p=quarantine; rua=mailto:postmaster@yourdomain.com`

4. **PTR Record (Reverse DNS)** - Maps IP to domain
   - **Must be configured by your hosting provider**
   - Critical for deliverability

## Quick Start Guide

### Step 1: Install Postfix

```bash
sudo apt-get update
sudo apt-get install postfix
```

During installation:
- Choose **"Internet Site"**
- Set hostname to your domain (e.g., `shop.example.com`)

### Step 2: Generate DKIM Keys

```bash
./manage.py generate_dkim_keys --domain yourdomain.com
```

This will:
- Generate RSA 2048-bit public/private key pair
- Store private key encrypted in database
- Display public key for DNS configuration

**Example Output:**
```
✓ DKIM keys generated successfully

Add this TXT record to your DNS:
  Hostname: mail._domainkey.yourdomain.com
  Type: TXT
  Value: v=DKIM1; k=rsa; p=MIIBIjANBgkqhkiG9w0BAQEFAAOCAQ8AMIIBCgK...
```

### Step 3: Configure DNS

Add the DNS records shown in the setup wizard to your DNS provider:

- **SPF** - Authorizes your server IP
- **DKIM** - Displays the public key from Step 2
- **DMARC** - Sets email authentication policy
- **PTR** - Contact your hosting provider

**DNS Propagation**: Can take 15 minutes to 48 hours depending on provider.

### Step 4: Start SMTP Server

```bash
./manage.py start_smtp_server
```

**Server Details:**
- Listen Address: `127.0.0.1:2525` (localhost only for security)
- Relay to: `127.0.0.1:25` (Postfix)
- DKIM Signing: Automatic

**Production Deployment:**
```bash
# Using systemd service (recommended)
sudo systemctl start spwig-smtp
sudo systemctl enable spwig-smtp

# Or use supervisor/screen for process management
```

### Step 5: Test Email Sending

```bash
./manage.py shell
```

```python
from django.core.mail import send_mail

send_mail(
    subject='Test Email',
    message='This is a test from the built-in SMTP server!',
    from_email='noreply@yourdomain.com',
    recipient_list=['recipient@example.com'],
    fail_silently=False,
)
```

Check the recipient inbox for the email. Inspect headers to verify DKIM signature.

## Configuration

### Django Settings

The built-in SMTP server requires minimal configuration in `settings.py`:

```python
# Email Backend
EMAIL_BACKEND = 'django.core.mail.backends.smtp.EmailBackend'
EMAIL_HOST = '127.0.0.1'
EMAIL_PORT = 2525
EMAIL_USE_TLS = False  # Not needed for localhost
EMAIL_USE_SSL = False  # Not needed for localhost

# Default From Email
DEFAULT_FROM_EMAIL = 'noreply@yourdomain.com'
```

### Email Account Configuration

Create an `EmailAccount` in the admin interface:

1. Navigate to **Email System** → **Email Accounts** → **Add Email Account**
2. Select **Built-in SMTP Server** as provider
3. Configure:
   - **From Email**: `noreply@yourdomain.com`
   - **Account Name**: "Default SMTP"
   - **DKIM Selector**: `mail` (default)
4. Save

### Advanced Options

**Custom SMTP Port:**
```bash
./manage.py start_smtp_server --port 3025
```

**Custom Postfix Relay:**
```bash
./manage.py start_smtp_server --postfix-host 192.168.1.100 --postfix-port 587
```

**Force DKIM Key Regeneration:**
```bash
./manage.py generate_dkim_keys --domain yourdomain.com --force
```

**Check SPF:**
```bash

```

**Check DKIM:**
```bash

```

**Check DMARC:**
```bash

```

**Check PTR:**
```bash
dig -x YOUR_SERVER_IP +short
```

## Troubleshooting

### Emails Not Sending

1. **Check SMTP server is running:**
   ```bash
   netstat -tuln | grep 2525
   ```
   Should show: `127.0.0.1:2525`

2. **Check Postfix is running:**
   ```bash
   sudo systemctl status postfix
   ```

3. **View Postfix logs:**
   ```bash
   sudo tail -f /var/log/mail.log
   ```

### Emails Going to Spam

1. **Verify all DNS records are valid** (SPF, DKIM, DMARC, PTR)
2. **Check IP reputation** at [MXToolbox](https://mxtoolbox.com/blacklists.aspx)
3. **Warm up your IP** by gradually increasing send volume
4. **Ensure PTR record is set** by hosting provider
5. **Check email content** isn't triggering spam filters

### DKIM Signature Failing

1. **Regenerate DKIM keys:**
   ```bash
   ./manage.py generate_dkim_keys --domain yourdomain.com --force
   ```

2. **Verify DNS record exactly matches:**
   ```bash
   dig mail._domainkey.yourdomain.com TXT +short
   ```

3. **Check for extra quotes or whitespace** in DNS TXT value

4. **Test DKIM signature** at [DKIM Validator](https://dkimvalidator.com)

### DNS Propagation Delays

- DNS changes can take **15 minutes to 48 hours** to propagate
- Use `dig @8.8.8.8 yourdomain.com TXT` to check specific resolvers
- Clear local DNS cache: `sudo systemd-resolve --flush-caches`
- Use [DNS Checker](https://dnschecker.org) for global propagation status

### Postfix Configuration Issues

**View Postfix config:**
```bash
sudo postconf -n
```

**Test Postfix directly:**
```bash
echo "Test email body" | mail -s "Test Subject" recipient@example.com
```

**Postfix queue:**
```bash
sudo postqueue -p  # View mail queue
sudo postsuper -d ALL  # Delete all queued mail (use cautiously)
```

## Security Considerations

### Localhost-only Binding

The SMTP server **MUST** only bind to `127.0.0.1` (localhost):

- No authentication required (safe because localhost-only)
- Prevents external connections
- Only Django application can send emails

**Security Warning:**
```bash
# DO NOT BIND TO PUBLIC IP - SECURITY RISK
./manage.py start_smtp_server --host 0.0.0.0  # ❌ NEVER DO THIS
```

### DKIM Private Key Storage

- Private keys are **encrypted** using Fernet (symmetric encryption)
- Stored in `EmailAccount.credentials` JSON field
- Encryption key derived from Django `SECRET_KEY`
- Keys never exposed in logs or API responses

### Rate Limiting

The built-in SMTP server has **no built-in rate limits** (only limited by server capacity). For production use:

1. Configure Postfix rate limits:
   ```bash
   sudo postconf -e 'smtp_destination_rate_delay = 1s'
   sudo postconf -e 'smtp_destination_concurrency_limit = 10'
   sudo systemctl reload postfix
   ```

2. Implement application-level throttling using Django ratelimit

3. Monitor send volume to avoid triggering ISP throttles

## Production Deployment

### systemd Service (Recommended)

Create `/etc/systemd/system/spwig-smtp.service`:

```ini
[Unit]
Description=Spwig Built-in SMTP Server
After=network.target postfix.service
Requires=postfix.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/var/www/spwig
ExecStart=/var/www/spwig/venv/bin/python manage.py start_smtp_server
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

**Enable and start:**
```bash
sudo systemctl daemon-reload
sudo systemctl enable spwig-smtp
sudo systemctl start spwig-smtp
sudo systemctl status spwig-smtp
```

### Supervisor Configuration

```ini
[program:spwig-smtp]
command=/var/www/spwig/venv/bin/python manage.py start_smtp_server
directory=/var/www/spwig
user=www-data
autostart=true
autorestart=true
redirect_stderr=true
stdout_logfile=/var/log/spwig/smtp-server.log
```

### Docker Deployment

```dockerfile
# In your Dockerfile
RUN apt-get update && apt-get install -y postfix

# Start both Postfix and SMTP server
CMD service postfix start && python manage.py start_smtp_server
```

## Advantages vs External Providers

| Feature | Built-in SMTP | External Providers |
|---------|---------------|-------------------|
| **Cost** | Free (server only) | $10-100/month |
| **Rate Limits** | Server capacity only | 100-10,000/month |
| **Privacy** | Full control | Third-party access |
| **Setup Complexity** | Medium (DNS + Postfix) | Easy (API keys) |
| **Deliverability** | Depends on IP reputation | Professional IPs |
| **OAuth Required** | No | Often yes |
| **Tracking** | Manual (Postfix logs) | Dashboard included |
| **Support** | Self-service | Provider support |

## When to Use Built-in SMTP

**Best for:**
- ✅ High-volume senders (>10,000 emails/month)
- ✅ Privacy-conscious merchants
- ✅ Dedicated servers with clean IP reputation
- ✅ Technical teams comfortable with server management
- ✅ Budget-conscious merchants avoiding monthly fees

**Not recommended for:**
- ❌ Shared hosting environments
- ❌ Servers with poor IP reputation
- ❌ Non-technical users (use external providers instead)
- ❌ When you need click/open tracking (use marketing providers)
